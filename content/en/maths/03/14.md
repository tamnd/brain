Chapter 14. Gödel Theorems

This chapter studies the limits of formal reasoning about arithmetic. Up to this point, formal systems have been presented as precise and powerful tools. Gödel’s theorems show that even well-designed systems have inherent boundaries.

The starting point is the idea that syntax can be encoded as arithmetic. Every symbol, formula, and proof can be assigned a natural number. This process is called arithmetization of syntax. Once this encoding is in place, statements about formulas and proofs become statements about numbers.

For example, a formula $A$ can be represented by a number $\ulcorner A \urcorner$. A relation such as “$A$ is provable” can then be expressed inside arithmetic itself. This allows a system to speak about its own proofs.

The key construction uses self-reference. Gödel showed how to build a sentence $G$ that indirectly refers to its own provability. Informally, $G$ asserts:

$$
\text{“$G$ is not provable.”}
$$

This sentence is not written in plain language. It is encoded entirely within arithmetic using the machinery developed earlier.

The first incompleteness theorem states that any sufficiently strong, consistent formal system cannot prove all true statements about natural numbers.

More precisely, if a system can express basic arithmetic and is consistent, then there exists a sentence $G$ such that:

* $G$ is true in the standard natural numbers
* $G$ is not provable in the system

So truth and provability diverge. There are true statements that no proof in the system can reach.

The second incompleteness theorem strengthens this result. It shows that such a system cannot prove its own consistency.

If the system could prove a statement expressing:

$$
\mathrm{Con}(\text{System})
$$

then the system would be inconsistent. Therefore, assuming consistency, this statement cannot be proved inside the system itself.

These results depend on three main ingredients.

First, the system must be able to represent arithmetic. It must encode statements about numbers and basic operations such as addition and multiplication.

Second, the system must be able to represent its own proofs. This is achieved through Gödel numbering and definable relations such as “$x$ is the code of a valid proof”.

Third, diagonalization is used to construct self-referential statements. This technique ensures that a formula can refer to its own code.

The consequences are far-reaching.

Formal systems cannot capture all mathematical truth. No consistent, sufficiently expressive system can be both complete and able to prove its own consistency.

This does not make formal systems useless. It clarifies their scope. Within those limits, formal systems remain precise and reliable tools.

The incompleteness theorems also connect logic with computation. The construction of self-referential statements is closely related to undecidability results such as the halting problem. Both rely on encoding syntax and using diagonal arguments.

Later sections refine these ideas. Arithmetization is developed in detail. The incompleteness theorems are proved formally. Variants and extensions are explored, including stronger systems and alternative formulations.

This chapter marks a turning point. Logic no longer studies only what can be proved. It also studies what cannot be proved, and why those limits arise from the structure of formal reasoning itself.
