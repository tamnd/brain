## 9.1 Forcing

Forcing is a method for constructing new models of set theory. It is one of the main tools used to prove that a statement is independent of ZFC.

A statement $\varphi$ is **independent of ZFC** if neither $\varphi$ nor its negation can be proved from ZFC, assuming ZFC is consistent:

$$
\mathrm{ZFC} \nvdash \varphi
$$

and

$$
\mathrm{ZFC} \nvdash ¬\varphi
$$

The most famous example is the continuum hypothesis:

$$
\mathrm{CH}: \quad 2^{\aleph_0} = \aleph_1
$$

This says that the cardinality of the real numbers is the first uncountable cardinal. Forcing provides models where $\mathrm{CH}$ holds and models where $\mathrm{CH}$ fails.

### The Basic Idea

Suppose we start with a model $M$ of ZFC.

Forcing adds a new object to $M$, often called a **generic object**, while preserving enough structure so that the enlarged universe is still a model of set theory.

We write the extension as:

$$
M[G]
$$

Here:

* $M$ is the original model
* $G$ is the generic object
* $M[G]$ is the enlarged model

You can think of $M[G]$ as $M$ plus carefully controlled new information.

The word “generic” means that $G$ meets every requirement that $M$ can describe. It avoids being too special from the viewpoint of $M$.

### Forcing Conditions

Forcing starts with a partially ordered set:

$$
(\mathbb{P}, \leq)
$$

The elements of $\mathbb{P}$ are called **conditions**.

A condition is a finite or partial approximation to the object we want to add.

The order $\leq$ means “stronger than”. If:

$$
q \leq p
$$

then $q$ contains at least as much information as $p$.

For example, to add a new subset of $\mathbb{N}$, a condition might be a finite binary sequence:

$$
p : n \to {0,1}
$$

A stronger condition extends the sequence by deciding more bits.

So if:

$$
p = 101
$$

then a stronger condition could be:

$$
q = 10110
$$

because $q$ gives more information.

### Generic Filters

A **filter** $G \subseteq \mathbb{P}$ is a set of compatible conditions that is closed upward in information.

Informally, $G$ contains many finite approximations that fit together into one complete object.

For binary sequences, a generic filter determines an infinite binary sequence:

$$
x \in 2^{\mathbb{N}}
$$

That sequence can be viewed as a new real number.

The filter must also meet every dense set from $M$.

A set $D \subseteq \mathbb{P}$ is **dense** if every condition can be strengthened to a condition in $D$:

$$
\forall p \in \mathbb{P},\exists q \in D,(q \leq p)
$$

Meeting every dense set means:

$$
G \cap D \neq \varnothing
$$

for each dense set $D$ that belongs to $M$.

Dense sets represent unavoidable requirements. A generic filter satisfies all requirements that the original model can state.

### Forcing Relation

Forcing has a formal relation written:

$$
p \Vdash \varphi
$$

This is read as:

$$
p \text{ forces } \varphi
$$

It means that every generic extension containing $p$ will satisfy $\varphi$.

So a condition $p$ does not merely suggest that $\varphi$ may become true. It guarantees that $\varphi$ will be true in any compatible generic extension.

For example, if $p$ decides that the first bit of the new binary sequence is $1$, then:

$$
p \Vdash \dot{x}(0)=1
$$

Here $\dot{x}$ is a name for the new real added by forcing.

### Names

The extension $M[G]$ contains new objects. To talk about them from inside $M$, forcing uses **names**.

A name is a formal object in $M$ that describes how an object will be interpreted once $G$ is chosen.

If $\dot{x}$ is a name, then its interpretation in the generic extension is written:

$$
\dot{x}^G
$$

So the forcing language can speak about future objects before the generic extension has been built.

This is important because the forcing relation must be defined inside the ground model $M$.

### Example: Adding a Cohen Real

A standard example is Cohen forcing.

Let:

$$
\mathbb{P} = 2^{<\omega}
$$

This is the set of all finite binary sequences.

The order is extension:

$$
q \leq p
$$

if $q$ extends $p$.

A generic filter $G$ determines an infinite binary sequence:

$$
c = \bigcup G
$$

where:

$$
c \in 2^\omega
$$

This $c$ is called a **Cohen real**.

From the viewpoint of $M$, the real $c$ is new. It is not one of the reals in the original model.

The forcing extension is:

$$
M[c]
$$

or more formally:

$$
M[G]
$$

### Independence Results

Forcing is used to prove independence by constructing different models.

To show that a statement $\varphi$ is not provable from ZFC, it is enough to build a model of:

$$
\mathrm{ZFC} + ¬\varphi
$$

assuming there is a model of ZFC.

To show that $¬\varphi$ is not provable from ZFC, it is enough to build a model of:

$$
\mathrm{ZFC} + \varphi
$$

So independence often has the form:

$$
\mathrm{Con}(\mathrm{ZFC})
\Longrightarrow
\mathrm{Con}(\mathrm{ZFC}+\varphi)
$$

and

$$
\mathrm{Con}(\mathrm{ZFC})
\Longrightarrow
\mathrm{Con}(\mathrm{ZFC}+¬\varphi)
$$

For the continuum hypothesis, forcing gives models where:

$$
2^{\aleph_0} = \aleph_1
$$

and models where:

$$
2^{\aleph_0} > \aleph_1
$$

### Why Forcing Works

Forcing works because it adds new sets in a controlled way.

The partial order $\mathbb{P}$ specifies what kind of information may be added. The generic filter $G$ assembles compatible approximations into a full object. Dense sets ensure the object is sufficiently generic. Names allow the ground model to describe objects in the extension. The forcing relation connects finite conditions with truth in the final model.

The result is a precise method for changing the universe while preserving the axioms of set theory.

### Intuition

A useful analogy is building an infinite object through finite commitments.

Each condition says:

$$
\text{the final object must at least look like this}
$$

A stronger condition says more.

The generic filter collects many compatible commitments. The union of those commitments produces the new object.

For Cohen forcing, the commitments are finite binary strings. The final object is an infinite binary sequence. Since the sequence meets every dense requirement from $M$, it behaves like a new real that was not definable or present inside $M$.
