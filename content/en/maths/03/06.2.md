## 6.2 Types and Realizations

Definable sets describe collections of elements that already exist in a structure. Types go further. They describe **possible elements** by listing properties those elements should satisfy.

Let $\mathcal M$ be a structure and $A \subseteq M$ a set of parameters.

### Types

A **type** over $A$ is a set of formulas with free variable $x$ (or a tuple $x_1,\dots,x_n$), where parameters may come from $A$.

A type is meant to describe a hypothetical element.

Example over $(\mathbb{N};<)$:

$$
p(x) = {x>0,\ x>1,\ x>2,\ x>3,\dots}
$$

This collection says: “$x$ is larger than every natural number”.

No actual natural number satisfies all these formulas. Still, the description itself is meaningful.

### Consistency

A type must be **consistent**.

This means that every finite subset of the formulas can be satisfied together in some structure.

Formally, $p(x)$ is consistent if for every finite subset:

$$
{\varphi_1(x),\dots,\varphi_n(x)} \subseteq p(x)
$$

there exists a structure $\mathcal N$ and an element $a$ such that:

$$
\mathcal N \models \varphi_i(a)
\quad \text{for all } i
$$

The infinite set:

$$
{x>0,\ x>1,\ x>2,\dots}
$$

is consistent because any finite part can be satisfied by choosing a sufficiently large number.

### Realization

A type $p(x)$ is **realized** in a structure $\mathcal M$ if there exists an element $a \in M$ such that:

$$
\mathcal M \models \varphi(a)
\quad \text{for every } \varphi(x) \in p(x)
$$

In this case, $a$ is said to **realize** the type.

Example:

Let:

$$
p(x) = {x>0,\ x>1}
$$

In $\mathbb{N}$, the element $2$ realizes this type.

However, the infinite type:

$$
{x>0,\ x>1,\ x>2,\dots}
$$

is not realized in $\mathbb{N}$.

In a larger structure, such as a nonstandard model of arithmetic, it may be realized by a “nonstandard” element.

### Complete Types

A type is **complete** if it decides every formula.

That means for every formula $\varphi(x)$, either:

$$
\varphi(x) \in p(x)
$$

or:

$$
¬\varphi(x) \in p(x)
$$

A complete type gives a full logical description of an element, as far as the language allows.

You can think of it as a maximal consistent set of properties.

### Types Over a Set

Types are usually considered **over a set of parameters** $A$.

This means formulas may include elements of $A$ as constants.

We write:

$$
p(x) \in S(A)
$$

to indicate that $p(x)$ is a type over $A$.

Example:

Let $A={a}$ in a structure with order.

The type:

$$
{x>a,\ x \ne a}
$$

describes elements strictly greater than $a$.

Types over larger parameter sets can express more refined distinctions.

### Space of Types

All complete types over $A$ form a set called the **type space**, written:

$$
S(A)
$$

Each element of $S(A)$ represents one possible way an element could behave relative to $A$.

Even if some types are not realized in the structure, they still exist in $S(A)$ as consistent descriptions.

### Example: Order Types

In a dense linear order without endpoints, consider a parameter $a$.

Possible types for $x$ include:

* $x < a$
* $x = a$
* $x > a$

But we can refine further. For example, we can describe an element strictly between two parameters:

$$
a < x < b
$$

Each such description corresponds to a type.

### Omitting Types

A type is **omitted** in a structure if it is not realized.

Example:

In $\mathbb{N}$, the type:

$$
{x>0,\ x>1,\ x>2,\dots}
$$

is omitted.

This reflects a limitation of the structure. It does not contain an element satisfying all these conditions.

The study of which types are realized or omitted is a central part of model theory.

### Connection to Definability

Definable sets describe sets of actual elements.

Types describe potential elements.

A definable set corresponds to a single formula.

A type corresponds to a possibly infinite collection of formulas.

So types generalize definability from sets to consistent descriptions.

### Perspective

Types provide a way to reason about elements that may not yet exist in the structure.

They allow us to extend structures, compare models, and understand how expressive a theory is.

In the next section, saturated models will appear as structures that realize as many types as possible, making this abstract notion concrete.
