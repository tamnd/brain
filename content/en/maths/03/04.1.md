## 4.1 Languages and Signatures

A first-order language provides the symbols used to write formulas. A **signature** specifies the non-logical part of that language. It fixes the vocabulary before any interpretation is given.

### Signatures

A **signature** $\mathcal L$ consists of:

* a set of constant symbols
* a set of function symbols, each with an arity
* a set of relation symbols, each with an arity

Formally, we can write:

$$
\mathcal L = (C, F, R)
$$

where:

* $C$ is a set of constants
* $F$ is a set of function symbols
* $R$ is a set of relation symbols

Each function symbol $f \in F$ has a fixed arity $n$, written:

$$
f : n
$$

Each relation symbol $R \in R$ has a fixed arity $n$:

$$
R : n
$$

Arity tells us how many arguments the symbol takes.

### Logical vs Non-logical Symbols

Every first-order language contains logical symbols:

$$
¬,\ \land,\ \lor,\ \to,\ \leftrightarrow,\ \forall,\ \exists,\ =,\ (, ),\ \text{variables}
$$

These are fixed and interpreted the same way in all structures.

The signature $\mathcal L$ introduces **non-logical symbols**. Their meaning depends on the structure.

### Terms

Terms represent elements of a structure. They are built from variables and function symbols.

The definition is inductive:

1. Every variable is a term.
2. Every constant symbol is a term.
3. If $f$ is an $n$-ary function symbol and $t_1, \dots, t_n$ are terms, then:
   $$
   f(t_1, \dots, t_n)
   $$
   is a term.
4. Nothing else is a term.

Examples:

$$
x,\ y,\ 0
$$

$$
f(x),\ g(x,y),\ x+1
$$

The last example uses infix notation for readability, but formally it is:

$$
+(x,1)
$$

### Atomic Formulas

Atomic formulas are the simplest formulas.

They are built from relation symbols and terms:

1. If $R$ is an $n$-ary relation symbol and $t_1, \dots, t_n$ are terms, then:
   $$
   R(t_1, \dots, t_n)
   $$
   is an atomic formula.

2. If $t_1$ and $t_2$ are terms, then:
   $$
   t_1 = t_2
   $$
   is an atomic formula.

Examples:

$$
R(x,y)
$$

$$
x = y
$$

$$
x < y
$$

### Formulas

Formulas are built from atomic formulas using logical connectives and quantifiers.

If $\varphi$ and $\psi$ are formulas, then:

$$
¬\varphi,\quad (\varphi \land \psi),\quad (\varphi \lor \psi),\quad (\varphi \to \psi)
$$

are formulas.

If $\varphi$ is a formula and $x$ is a variable, then:

$$
\forall x, \varphi
\qquad
\exists x, \varphi
$$

are formulas.

Example:

$$
\forall x, (P(x) \to \exists y, R(x,y))
$$

### Free and Bound Variables

Variables in a formula can be free or bound.

* A variable is **bound** if it is inside the scope of a quantifier.
* A variable is **free** if it is not bound.

Example:

$$
\forall x, (P(x,y))
$$

Here:

* $x$ is bound
* $y$ is free

A **sentence** is a formula with no free variables.

Example:

$$
\forall x, \exists y, R(x,y)
$$

### Example Signatures

Arithmetic:

$$
\mathcal L_{\mathrm{arith}} = {0, 1, +, \cdot, <}
$$

Group theory:

$$
\mathcal L_{\mathrm{grp}} = {\cdot, {}^{-1}, e}
$$

Graph theory:

$$
\mathcal L_{\mathrm{graph}} = {E}
$$

where $E$ is a binary relation symbol.

Ordered structures:

$$
\mathcal L = {<}
$$

Each signature determines the shape of formulas that can be written.

### Well-Formed Expressions

A formula is **well-formed** if it follows the rules of the language:

* function symbols applied to the correct number of terms
* relation symbols applied to the correct number of terms
* parentheses placed correctly
* quantifiers applied to variables

Example of a well-formed formula:

$$
\forall x, \exists y, (x < y)
$$

Example of an ill-formed expression:

$$
\forall (x < y)
$$

This fails because the quantifier is not applied to a variable correctly.

### Role of the Signature

The signature controls expressiveness.

* A richer signature can describe more structure.
* A smaller signature forces simpler descriptions.

For example, without $<$, arithmetic cannot express order. Without $+$ and $\cdot$, it cannot express arithmetic relations.

The same underlying set can support many different structures depending on the signature.
