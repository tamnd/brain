## 1.2 Semantics: Truth Tables

Syntax tells you how to build formulas. Semantics tells you how to evaluate them. In propositional logic, evaluation is simple and precise: every formula is either true or false.

We start with **truth values**:

$$
\mathrm{T} \quad \text{(true)}, \qquad \mathrm{F} \quad \text{(false)}
$$

A **valuation** assigns a truth value to each variable:

$$
v : \mathrm{Var} \to {\mathrm{T}, \mathrm{F}}
$$

You can think of a valuation as one possible “world” where each statement has already been decided.

Example:

$$
v(p)=\mathrm{T}, \quad v(q)=\mathrm{F}, \quad v(r)=\mathrm{T}
$$

Once variables are assigned, every larger formula gets a value by following fixed rules. These rules are what truth tables encode.

### Negation

Negation flips the truth value.

| $A$          | $¬A$         |
| ------------ | ------------ |
| $\mathrm{T}$ | $\mathrm{F}$ |
| $\mathrm{F}$ | $\mathrm{T}$ |

So when you see $¬A$, read it as “$A$ is not true”.

$$
v(¬A)=\mathrm{T} \quad \text{exactly when} \quad v(A)=\mathrm{F}
$$

### Conjunction

Conjunction requires both parts to be true.

| $A$          | $B$          | $A \land B$  |
| ------------ | ------------ | ------------ |
| $\mathrm{T}$ | $\mathrm{T}$ | $\mathrm{T}$ |
| $\mathrm{T}$ | $\mathrm{F}$ | $\mathrm{F}$ |
| $\mathrm{F}$ | $\mathrm{T}$ | $\mathrm{F}$ |
| $\mathrm{F}$ | $\mathrm{F}$ | $\mathrm{F}$ |

You can read $A \land B$ as “$A$ and $B$”.

$$
v(A \land B)=\mathrm{T} \quad \text{only if both } A \text{ and } B \text{ are true}
$$

### Disjunction

Disjunction requires at least one part to be true.

| $A$          | $B$          | $A \lor B$   |
| ------------ | ------------ | ------------ |
| $\mathrm{T}$ | $\mathrm{T}$ | $\mathrm{T}$ |
| $\mathrm{T}$ | $\mathrm{F}$ | $\mathrm{T}$ |
| $\mathrm{F}$ | $\mathrm{T}$ | $\mathrm{T}$ |
| $\mathrm{F}$ | $\mathrm{F}$ | $\mathrm{F}$ |

This “or” is inclusive. It allows both to be true.

$$
v(A \lor B)=\mathrm{T} \quad \text{if at least one of } A, B \text{ is true}
$$

### Implication

Implication is the connective that often feels less intuitive at first. The rule is:

| $A$          | $B$          | $A \to B$    |
| ------------ | ------------ | ------------ |
| $\mathrm{T}$ | $\mathrm{T}$ | $\mathrm{T}$ |
| $\mathrm{T}$ | $\mathrm{F}$ | $\mathrm{F}$ |
| $\mathrm{F}$ | $\mathrm{T}$ | $\mathrm{T}$ |
| $\mathrm{F}$ | $\mathrm{F}$ | $\mathrm{T}$ |

It is false only in one case: when $A$ is true but $B$ is false.

A useful way to remember:

* If the premise fails, the implication does not complain.
* It only fails when the premise holds but the conclusion does not.

Formally:

$$
v(A \to B)=\mathrm{F} \quad \text{exactly when} \quad v(A)=\mathrm{T},\ v(B)=\mathrm{F}
$$

You can also rewrite implication as:

$$
A \to B \equiv ¬A \lor B
$$

This identity is often used to simplify formulas.

### Biconditional

The biconditional checks whether both sides agree.

| $A$          | $B$          | $A \leftrightarrow B$ |
| ------------ | ------------ | --------------------- |
| $\mathrm{T}$ | $\mathrm{T}$ | $\mathrm{T}$          |
| $\mathrm{T}$ | $\mathrm{F}$ | $\mathrm{F}$          |
| $\mathrm{F}$ | $\mathrm{T}$ | $\mathrm{F}$          |
| $\mathrm{F}$ | $\mathrm{F}$ | $\mathrm{T}$          |

You can read it as “$A$ if and only if $B$”.

$$
v(A \leftrightarrow B)=\mathrm{T} \quad \text{when } A \text{ and } B \text{ have the same value}
$$

It can also be written as:

$$
(A \to B) \land (B \to A)
$$

### Truth Tables for Full Formulas

A truth table lists all possible valuations. For each row, you compute the value step by step.

Example:

$$
(p \land q) \to p
$$

| $p$          | $q$          | $p \land q$  | $(p \land q) \to p$ |
| ------------ | ------------ | ------------ | ------------------- |
| $\mathrm{T}$ | $\mathrm{T}$ | $\mathrm{T}$ | $\mathrm{T}$        |
| $\mathrm{T}$ | $\mathrm{F}$ | $\mathrm{F}$ | $\mathrm{T}$        |
| $\mathrm{F}$ | $\mathrm{T}$ | $\mathrm{F}$ | $\mathrm{T}$        |
| $\mathrm{F}$ | $\mathrm{F}$ | $\mathrm{F}$ | $\mathrm{T}$        |

The final column is always true. This means the formula is valid.

Another example:

$$
p \lor ¬p
$$

| $p$          | $¬p$         | $p \lor ¬p$  |
| ------------ | ------------ | ------------ |
| $\mathrm{T}$ | $\mathrm{F}$ | $\mathrm{T}$ |
| $\mathrm{F}$ | $\mathrm{T}$ | $\mathrm{T}$ |

This captures a basic logical principle: every statement is either true or false.

### Size of Truth Tables

If a formula has $n$ variables, there are:

$$
2^n
$$

possible valuations.

Each row corresponds to one assignment. As $n$ grows, the table doubles in size each time. This exponential growth explains why truth tables are practical for small formulas and less useful for large ones.

When you work through truth tables by hand, a good approach is to:

* list variables in a fixed order
* fill columns step by step
* reuse intermediate results

This keeps the process systematic and reduces errors.
