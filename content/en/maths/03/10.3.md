## 10.3 Church-Turing Thesis

The Church-Turing thesis explains why the formal models of computation introduced in this chapter are treated as precise models of algorithms.

It connects an informal idea with a formal one.

Informal idea:

```text
can be computed by an effective procedure
```

Formal idea:

```text
can be computed by a Turing machine
```

In short:

$$
\mathrm{EffectiveComputable} = \mathrm{TuringComputable}
$$

This statement is called a thesis because one side is informal. The phrase “effective procedure” means a finite, mechanical method that can be followed step by step.

A theorem can prove that two formal definitions are equivalent. For example, Turing machines and recursive functions compute the same class of functions. But a thesis relates a formal definition to an informal concept.

### Effective Procedures

An effective procedure has clear properties.

It has a finite description.

Each step is mechanical and follows fixed rules.

Each step uses finite data.

If it halts, it produces an answer after finitely many steps.

Examples:

$$
f(n)=n+1
$$

$$
g(x,y)=x+y
$$

$$
h(n)=n!
$$

Another example is primality testing. Given $n$, test whether any integer $d$ divides $n$, where:

$$
2 \le d \le n-1
$$

This method is finite and always halts.

### Turing-Computable Functions

A function

$$
f:\mathbb{N}\rightharpoonup\mathbb{N}
$$

is Turing-computable if a Turing machine computes it.

If the value is defined, the machine halts:

$$
f(n)\downarrow
$$

If the value is undefined, the machine does not halt:

$$
f(n)\uparrow
$$

For total functions

$$
f:\mathbb{N}\to\mathbb{N}
$$

the machine halts on every input.

### Why It Is a Thesis

The statement has the form:

```text
informal concept = formal model
```

The formal side can be studied exactly. The informal side cannot be fully formalized without choosing another model.

The thesis asserts that the formal definition captures the intended idea of computation.

### Evidence for the Thesis

Several independent models were developed:

| Model               | View                  |
| ------------------- | --------------------- |
| Turing machines     | symbol manipulation   |
| recursive functions | function generation   |
| lambda calculus     | function application  |
| register machines   | instruction execution |
| WHILE programs      | structured programs   |

They compute the same class of functions:

# $$ \mathrm{TuringComputable}

# \mathrm{PartialRecursive}

# \mathrm{LambdaComputable}

# \mathrm{RegisterComputable}

\mathrm{WhileComputable}
$$

### What the Thesis Allows

If no Turing machine decides a problem $P$, then no algorithm decides $P$.

Example:

```text
Given program M and input x, does M(x) halt?
```

No Turing machine solves this for all inputs. Therefore, no algorithm solves it.

### What the Thesis Does Not Say

Not every function

$$
f:\mathbb{N}\to\mathbb{N}
$$

is computable.

There are only countably many algorithms, but uncountably many functions.

Computability is different from efficiency.

```text
Computability: can it be done?
Complexity: how costly is it?
```

The thesis concerns idealized computation, not physical machines.

### Extended Church-Turing Thesis

A stronger version states that all reasonable models can be simulated efficiently by a probabilistic Turing machine, with polynomial overhead.

This concerns efficiency rather than computability.

Quantum computation may affect this stronger statement.
