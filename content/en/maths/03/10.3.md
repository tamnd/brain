## 10.3 Church-Turing Thesis

The Church-Turing thesis explains why the formal models of computation introduced in this chapter are taken seriously.

It connects an informal idea:

$$
\text{can be computed by an effective procedure}
$$

with a formal one:

$$
\text{can be computed by a Turing machine}
$$

In short:

$$
\text{effectively computable} = \text{Turing-computable}
$$

This statement is called a **thesis**, not a theorem, because one side of it is informal. The phrase “effective procedure” belongs to ordinary mathematical language. It means a finite, mechanical method that can be followed step by step.

A theorem can prove that two formal definitions are equivalent. For example, we can prove that Turing machines and recursive functions compute the same class of functions. But we cannot prove, in the same formal way, that a mathematical definition captures an informal human idea. We can only give evidence that the match is correct.

### Effective Procedures

An **effective procedure** is a method with clear rules.

It should have a finite description. The instructions must be expressible in a finite amount of text or symbols.

It should be mechanical. Each step follows from the current situation by a fixed rule. No creativity or intuition is needed during execution.

It should use finite data at each stage. A computation may run for a long time, but every individual step manipulates only finite information.

It should produce an answer after finitely many steps when it halts.

Simple examples include:

$$
f(n)=n+1
$$

$$
g(x,y)=x+y
$$

$$
h(n)=n!
$$

Each has a clear finite method. Given an input, we know how to compute the output.

A more interesting example is primality testing. Given $n$, one simple method is to test whether any integer $d$ with:

$$
2 \le d \le n-1
$$

divides $n$. This method may be slow, but it is still effective because it always stops.

Computability theory cares first about whether a method exists at all. Efficiency is a later question.

### Turing-Computable Functions

A function:

$$
f:\mathbb{N}\rightharpoonup\mathbb{N}
$$

is **Turing-computable** if there is a Turing machine that computes it.

This means:

$$
f(n)\downarrow
\quad \Longrightarrow \quad
\text{the machine halts with output } f(n)
$$

and:

$$
f(n)\uparrow
\quad \Longrightarrow \quad
\text{the machine does not halt}
$$

For a total function:

$$
f:\mathbb{N}\to\mathbb{N}
$$

the machine must halt on every input.

The Turing machine is intentionally simple. It has a finite set of states, a tape, a head that reads and writes symbols, and a transition rule. Yet this simple device can simulate any ordinary program, once the program and input are encoded as symbols.

That is the central insight. A very small model can capture the general notion of computation.

### Why It Is a Thesis

The Church-Turing thesis has the form:

$$
\text{informal concept} = \text{formal model}
$$

The formal side can be studied exactly. We can define Turing machines, prove theorems about them, and compare them with other formal systems.

The informal side cannot be fully formalized without choosing another formal model. It is the pre-theoretic idea of an algorithm.

So the thesis is a bridge. It says that the formal definition has captured the intended informal concept.

This is common in mathematics. We often begin with an intuitive idea, then choose a formal definition that preserves its essential behavior. The strength of the definition is judged by its consequences and by comparison with other definitions.

### Evidence for the Thesis

The strongest evidence is convergence.

Several models of computation were developed independently:

| Model               | View of computation                                       |
| ------------------- | --------------------------------------------------------- |
| Turing machines     | computation as mechanical symbol manipulation             |
| recursive functions | computation as generation from basic functions            |
| lambda calculus     | computation as function abstraction and application       |
| register machines   | computation as instructions over natural-number registers |
| WHILE programs      | computation as simple program execution                   |

These models look different. They use different syntax and different primitive operations.

But they compute the same class of partial functions:

$$
\text{Turing-computable}
========================

# \text{partial recursive}

# \lambda\text{-computable}

# \text{register-computable}

\text{WHILE-computable}
$$

This agreement would be surprising if the models were arbitrary. It suggests that they are all describing the same underlying phenomenon.

### What the Thesis Allows Us to Do

The thesis lets us move from machine results to general statements about algorithms.

Suppose we prove:

$$
\text{no Turing machine decides problem } P
$$

Using the Church-Turing thesis, we interpret this as:

$$
\text{no algorithm decides problem } P
$$

This is what makes undecidability results powerful.

For example, the halting problem asks:

$$
\text{given a program } M \text{ and input } x,\text{ does } M(x) \text{ halt?}
$$

Turing proved that no Turing machine can decide this problem for all programs and inputs.

By the Church-Turing thesis, we understand this as saying that no effective procedure can solve the halting problem in full generality.

### What the Thesis Does Not Say

The Church-Turing thesis does not say that every function is computable.

There are only countably many algorithms, because each algorithm has a finite description. But there are uncountably many functions:

$$
f:\mathbb{N}\to\mathbb{N}
$$

So most functions are not computable.

The thesis also does not say that every computable function is efficient.

A function may be computable but take too many steps to be useful in practice. Computability theory asks:

$$
\text{Can it be computed at all?}
$$

Complexity theory asks:

$$
\text{How much time or space does it require?}
$$

These are different questions.

The thesis also does not say that real physical computers are exactly Turing machines. Real computers have finite memory, hardware failures, timing behavior, and physical constraints. Turing machines are idealized mathematical objects. Their purpose is to model algorithmic possibility, not engineering detail.

### The Extended Church-Turing Thesis

There is also a stronger statement called the **extended Church-Turing thesis**.

Roughly, it says that every reasonable model of computation can be simulated efficiently by a probabilistic Turing machine, with at most polynomial overhead.

This stronger claim concerns efficiency, not just computability.

Quantum computation is important here. Quantum computers do not compute non-computable functions, but they may compute some functions faster than known classical methods. For this reason, the extended thesis is more delicate than the original Church-Turing thesis.

The original thesis concerns what can be computed in principle.

The extended thesis concerns what can be computed efficiently.

### Working Use

In ordinary computability theory, the Church-Turing thesis is used as a working principle.

When we give a clear algorithm in ordinary mathematical language, we treat it as Turing-computable.

When we prove that no Turing machine can solve a problem, we treat it as evidence that no algorithm can solve it.

This lets us use a precise formal model to answer broad questions about computation.
