## 10.3 Church-Turing Thesis

The Church-Turing thesis connects the informal idea of an algorithm with precise mathematical models of computation.

Informally, it says:

$$
\text{effectively computable} = \text{computable by a Turing machine}
$$

The phrase **effectively computable** means computable by a finite, mechanical procedure. Such a procedure can be followed step by step without intuition, guessing, or external judgment.

The Church-Turing thesis is not a theorem in the usual sense. A theorem proves a relation between formal objects. Here, one side is informal:

$$
\text{effectively computable}
$$

The other side is formal:

$$
\text{Turing-computable}
$$

So the thesis identifies an informal concept with a formal definition.

### Historical Motivation

Before modern computers, mathematicians wanted to understand the limits of formal calculation. They asked whether every mathematical problem could be solved by a fixed procedure.

This led to several independent models of computation:

| Model               | Main idea                                                 |
| ------------------- | --------------------------------------------------------- |
| Turing machines     | computation by a machine with states and a tape           |
| lambda calculus     | computation by function abstraction and application       |
| recursive functions | computation by initial functions and closure operations   |
| register machines   | computation by instructions over natural-number registers |
| formal grammars     | computation by symbolic rewriting rules                   |

These models were developed from different starting points, but they turned out to define the same class of computable functions.

That agreement is the main reason the thesis is accepted.

### Effective Procedures

An effective procedure should satisfy several conditions.

It must have a finite description. The instructions cannot be infinitely long.

It must be mechanical. Each step must be determined by fixed rules.

It must use finite data at each step. The computation may continue for many steps, but each individual configuration must be finite.

It must produce an answer after finitely many steps when it halts.

This matches the behavior of idealized algorithms.

For example, addition is effectively computable. We can give a finite procedure that takes two numbers and returns their sum.

Primality testing is also effectively computable. Given $n$, we can test whether any number from $2$ to $n-1$ divides it. This may be inefficient, but it halts and gives the correct answer.

### Turing-Computability

A function:

$$
f : \mathbb{N} \rightharpoonup \mathbb{N}
$$

is Turing-computable if there is a Turing machine that, on input $n$, behaves as follows:

$$
f(n)\downarrow
\quad \Longrightarrow \quad
\text{the machine halts with output } f(n)
$$

and:

$$
f(n)\uparrow
\quad \Longrightarrow \quad
\text{the machine does not halt}
$$

For total functions:

$$
f : \mathbb{N} \to \mathbb{N}
$$

the machine must halt on every input.

The machine model may look primitive, but it is powerful enough to simulate ordinary programs. Its simplicity is the point. It strips computation down to symbols, states, and local rules.

### Equivalent Models

The Church-Turing thesis is supported by equivalence theorems between formal models.

For example:

$$
\text{Turing-computable}
========================

# \text{lambda-computable}

\text{partial recursive}
$$

This equality is a theorem, because all three notions are formal.

The thesis then adds the informal claim:

$$
\text{these formal notions capture all effective computation}
$$

This is why the Church-Turing thesis is partly mathematical and partly conceptual.

### What the Thesis Does Not Say

The thesis does not say that every mathematical function is computable.

There are infinitely many functions:

$$
f : \mathbb{N} \to \mathbb{N}
$$

but only countably many algorithms, because each algorithm has a finite description.

So most functions are not computable.

The thesis also does not say that every computable function is efficiently computable.

A function may be computable but require enormous time or memory. Computability asks whether an answer can be obtained in principle. Complexity theory asks how expensive that computation is.

So these are different questions:

| Question                        | Field                |
| ------------------------------- | -------------------- |
| Can it be computed at all?      | computability theory |
| Can it be computed efficiently? | complexity theory    |

The thesis also does not depend on one physical computer architecture. It concerns idealized computation, not current hardware.

### Consequences

The Church-Turing thesis lets us prove impossibility results.

Once we accept that Turing machines capture all effective procedures, a proof that no Turing machine solves a problem becomes a proof that no algorithm solves it.

This is how undecidability results get their force.

For example, the halting problem asks whether a given program halts on a given input. Turing proved that no Turing machine can decide this problem for all programs and inputs.

Using the thesis, we interpret this as:

$$
\text{there is no algorithm that solves the halting problem in full generality}
$$

### Stronger and Weaker Versions

The basic Church-Turing thesis concerns what can be computed in principle.

A stronger version, sometimes called the **extended Church-Turing thesis**, concerns efficient computation. Roughly, it says that every reasonable model of computation can be simulated by a probabilistic Turing machine with only polynomial overhead.

This stronger thesis is more delicate. Quantum computation challenges some classical versions of it, because quantum computers may solve certain problems faster than known classical algorithms.

The original thesis remains about computability, not efficiency.

### Working Principle

In practice, computability theory uses the Church-Turing thesis as a bridge.

When we design an algorithm in ordinary mathematical language, we treat it as Turing-computable.

When we prove that no Turing machine can solve a problem, we treat that as showing no algorithm can solve it.

This bridge allows formal machine results to answer informal questions about what can and cannot be computed.
