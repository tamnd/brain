## 10.4 Formal Models of Computation

A formal model of computation gives a precise definition of what a computation is. It specifies how inputs are represented, how steps are performed, and how outputs are produced.

Different models describe computation in different ways. Despite these differences, they all aim to capture the same intuitive idea of an algorithm.

### Turing Machines

A Turing machine consists of:

| Component       | Role                     |
| --------------- | ------------------------ |
| tape            | stores symbols           |
| head            | reads and writes symbols |
| states          | control the computation  |
| transition rule | determines the next step |

A transition has the form:

$$
(q, a) \mapsto (q', b, D)
$$

This means:

- read symbol $a$ in state $q$
- write symbol $b$
- move in direction $D \in {L, R}$
- enter state $q'$

The machine starts with an input written on the tape. It proceeds step by step according to the transition rules. If it reaches a halting state, the computation stops and the tape contains the output.

### Lambda Calculus

Lambda calculus models computation using functions.

Expressions are built from:

$$
x
\quad
\lambda x.M
\quad
MN
$$

The main rule is function application:

$$
(\lambda x.M)N \to M[x := N]
$$

This replaces $x$ by $N$ inside $M$.

Lambda calculus has no built-in numbers or arithmetic. These are encoded using functions. Despite its simplicity, it can express all computable functions.

### Recursive Functions

Recursive functions define computation using equations.

They start with:

$$
Z(n)=0
$$

$$
S(n)=n+1
$$

$$
P_i^k(x_1,\dots,x_k)=x_i
$$

New functions are built using:

$$
f(\vec{x}) = h(g_1(\vec{x}),\dots,g_m(\vec{x}))
$$

$$
f(\vec{x},0)=g(\vec{x})
$$

$$
f(\vec{x},n+1)=h(\vec{x},n,f(\vec{x},n))
$$

$$
f(\vec{x})=\mu y,[g(\vec{x},y)=0]
$$

This approach avoids machines and describes computation purely through function construction.

### Register Machines

A register machine uses a finite number of registers that store natural numbers.

Typical instructions include:

| Instruction  | Meaning                |
| ------------ | ---------------------- |
| INC $r$      | increment register $r$ |
| DECJZ $r, L$ | decrement or jump      |
| JMP $L$      | jump to label          |
| HALT         | stop                   |

A configuration consists of the current instruction and register values.

Computation proceeds by executing instructions one by one.

### WHILE Programs

A WHILE program is a simple programming language.

Example:

```text
z := x
while y != 0 do
    z := z + 1
    y := y - 1
end
```

This computes addition.

The key feature is the while loop. It may run forever, so the model can represent partial functions.

### Rewriting Systems

A rewriting system transforms expressions using rules:

$$
l \to r
$$

An occurrence of $l$ may be replaced by $r$.

Example rules:

$$
x + 0 \to x
$$

$$
x + S(y) \to S(x+y)
$$

A computation is a sequence of such rewrites.

### Encodings

To compare models, we encode data as natural numbers.

Examples:

| Object       | Encoding idea     |
| ------------ | ----------------- |
| pair $(x,y)$ | pairing function  |
| list         | sequence encoding |
| program      | Gödel numbering   |

A pairing function maps two numbers to one:

# $$ \langle x,y\rangle

\frac{(x+y)(x+y+1)}{2}+y
$$

Encodings allow one model to simulate another.

### Simulation

A model $M_2$ simulates a model $M_1$ if it can reproduce each step of $M_1$.

The process is:

- encode configurations of $M_1$
- simulate each step using $M_2$
- decode the final result

If both models can simulate each other, they are equivalent.

Different models emphasize different views:

| Model               | View                    |
| ------------------- | ----------------------- |
| Turing machine      | mechanical process      |
| lambda calculus     | function evaluation     |
| recursive functions | algebraic definition    |
| register machine    | low-level program       |
| WHILE program       | structured program      |
| rewriting system    | symbolic transformation |

Each model gives a different perspective on computation, but all aim to capture the same underlying concept.
