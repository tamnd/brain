## 10.4 Formal Models of Computation

A formal model of computation gives an exact mathematical definition of what a computation is. The purpose is to replace the informal word “algorithm” with an object that can be studied by proofs.

A good model should answer questions like these:

| Question                     | Meaning                                           |
| ---------------------------- | ------------------------------------------------- |
| What counts as input?        | How data is represented before computation starts |
| What counts as a step?       | How one configuration changes into the next       |
| What counts as output?       | How the final answer is read                      |
| What counts as halting?      | When the computation stops                        |
| What functions are computed? | Which input-output behaviors the model defines    |

Different models answer these questions in different ways. The striking fact is that the standard models compute the same class of functions.

### Turing Machines

A Turing machine is an idealized machine with:

| Component           | Role                                     |
| ------------------- | ---------------------------------------- |
| tape                | stores symbols                           |
| head                | reads and writes one tape cell at a time |
| finite state set    | stores the current control state         |
| transition function | tells the machine what to do next        |
| start state         | where computation begins                 |
| halt states         | where computation stops                  |

At each step, the machine reads the current symbol and current state. Then it writes a symbol, moves left or right, and enters a new state.

A transition has the form:

$$
(q, a) \mapsto (q', b, D)
$$

where:

| Symbol | Meaning                       |
| ------ | ----------------------------- |
| $q$    | current state                 |
| $a$    | current tape symbol           |
| $q'$   | next state                    |
| $b$    | symbol to write               |
| $D$    | direction, usually $L$ or $R$ |

Turing machines are low-level, but they are precise. They make computation into a sequence of local mechanical updates.

### Lambda Calculus

Lambda calculus models computation using functions.

Its basic expressions are:

| Form          | Meaning              |
| ------------- | -------------------- |
| $x$           | variable             |
| $\lambda x.M$ | function abstraction |
| $MN$          | function application |

The central reduction rule is beta reduction:

$$
(\lambda x.M)N \to M[x := N]
$$

This means: apply the function $\lambda x.M$ to input $N$ by substituting $N$ for $x$ inside $M$.

For example:

$$
(\lambda x.x)x \to x
$$

A more concrete example:

$$
(\lambda x. x + 1)3 \to 4
$$

In pure lambda calculus, numbers and arithmetic are encoded as functions. This makes the system minimal, but still computationally complete.

Lambda calculus is important because it connects logic, programming languages, and computation. Functional programming languages are strongly influenced by it.

### Recursive Functions

Recursive function theory defines computation by building functions from simple starting functions.

The initial functions are:

$$
Z(n)=0
$$

$$
S(n)=n+1
$$

$$
P_i^k(x_1,\dots,x_k)=x_i
$$

Then new functions are generated by operations such as composition, primitive recursion, and minimization.

Composition has the form:

$$
f(\vec{x}) = h(g_1(\vec{x}),\dots,g_m(\vec{x}))
$$

Primitive recursion has the form:

$$
f(\vec{x},0)=g(\vec{x})
$$

$$
f(\vec{x},n+1)=h(\vec{x},n,f(\vec{x},n))
$$

Minimization has the form:

$$
f(\vec{x})=\mu y,[g(\vec{x},y)=0]
$$

Recursive functions are useful because they describe computation without machines. They use equations and closure rules instead.

### Register Machines

A register machine uses finitely many registers, each holding a natural number.

A simple register machine may have instructions such as:

| Instruction  | Meaning                                               |
| ------------ | ----------------------------------------------------- |
| `INC r`      | increment register $r$                                |
| `DECJZ r, L` | decrement $r$ if nonzero; otherwise jump to label $L$ |
| `JMP L`      | jump to label $L$                                     |
| `HALT`       | stop computation                                      |

A configuration consists of the current instruction and the current register values.

For example, a register state may be:

$$
(R_0,R_1,R_2)=(4,0,7)
$$

Register machines are closer to ordinary hardware than Turing machines. They also resemble assembly language.

Despite this different style, register machines compute the same partial computable functions.

### WHILE Programs

A WHILE language is a tiny programming language designed for computability theory.

It may contain:

```text
x := 0
x := y + 1
x := y
while x != 0 do
    ...
end
```

A program manipulates variables containing natural numbers.

Example:

```text
z := x
while y != 0 do
    z := z + 1
    y := y - 1
end
```

This computes addition if subtraction is understood as truncated subtraction.

WHILE programs are useful because they look like ordinary programs but remain simple enough for proofs.

The `while` loop is important because it may fail to terminate. That is what gives the language partial computability.

### Primitive Recursive Programs

If we restrict loops so that every loop has a fixed bound known before the loop starts, we get a weaker model.

For example:

```text
repeat n times:
    body
end
```

This kind of bounded iteration always terminates. It corresponds closely to primitive recursive functions.

The distinction is important:

| Program style         | Termination | Function class      |
| --------------------- | ----------- | ------------------- |
| bounded loops         | always halt | primitive recursive |
| unbounded while loops | may diverge | partial recursive   |

Adding unbounded search or unbounded loops gives more computational power.

### Rewriting Systems

A rewriting system computes by repeatedly replacing one expression with another according to rules.

A rule may have the form:

$$
l \to r
$$

meaning that an occurrence of pattern $l$ may be replaced by $r$.

For example, arithmetic can be represented by rewrite rules:

$$
x + 0 \to x
$$

$$
x + S(y) \to S(x+y)
$$

Then:

$$
2 + S(S(0))
$$

reduces step by step to:

$$
S(S(S(S(0))))
$$

Rewriting systems are important in logic, programming language semantics, and automated theorem proving.

### Comparing the Models

| Model               | Basic object               | Computation step      | Natural viewpoint   |
| ------------------- | -------------------------- | --------------------- | ------------------- |
| Turing machine      | tape and finite control    | read, write, move     | machine model       |
| lambda calculus     | expressions                | reduction             | functional model    |
| recursive functions | equations                  | closure construction  | algebraic model     |
| register machine    | registers and instructions | instruction execution | hardware-like model |
| WHILE programs      | variables and commands     | program execution     | programming model   |
| rewriting systems   | symbolic terms             | rule application      | syntactic model     |

Each model emphasizes a different intuition.

Turing machines show computation as mechanical symbol manipulation.

Lambda calculus shows computation as function application.

Recursive functions show computation as definability by equations.

Register machines show computation as instruction execution.

WHILE programs show computation as structured programming.

Rewriting systems show computation as transformation of expressions.

### Encodings

To compare models, we need encodings. A model may work with strings, numbers, tapes, expressions, or programs. Computability theory usually encodes all finite objects as natural numbers.

Examples of encodable objects include:

| Object         | Possible encoding               |
| -------------- | ------------------------------- |
| finite strings | number by base encoding         |
| pairs $(x,y)$  | pairing function                |
| finite lists   | product of prime powers         |
| programs       | Gödel numbering                 |
| formulas       | syntax trees encoded as numbers |

A pairing function is a computable bijection or injection:

$$
\langle x,y\rangle : \mathbb{N}^2 \to \mathbb{N}
$$

One standard example is Cantor pairing:

$$
\langle x,y\rangle
==================

\frac{(x+y)(x+y+1)}{2}+y
$$

With encodings, a model that computes on strings can be compared to a model that computes on natural numbers.

### Simulation

Two models are computationally equivalent when each can simulate the other.

For example, a register machine can simulate a Turing machine by encoding the tape contents into registers.

A Turing machine can simulate a register machine by storing register values on its tape.

A lambda calculus expression can encode natural numbers and simulate recursive definitions.

A WHILE program can simulate minimization by using an unbounded loop.

The details differ, but the pattern is the same:

$$
\text{one step system} \quad \leadsto \quad \text{another step system}
$$

Simulation usually requires encoding configurations and showing that each step of one model can be reproduced by finitely many steps of another.
