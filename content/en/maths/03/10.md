## Chapter 10. Computable Functions

Computability theory studies what it means for a function to be calculable by a finite mechanical procedure.

The central question is simple:

$$
\text{Which functions can be computed by an algorithm?}
$$

This question looks modern because we now think in terms of computers. But the mathematical problem came first. Before electronic computers, logicians wanted a precise definition of “effective calculation”. Computability theory gives that definition.

A function may be easy to describe but impossible to compute in full generality. For example, we may ask whether a program eventually stops. This question has a clear meaning. But there is no general algorithm that answers it correctly for every program and input.

So computability theory separates two ideas:

| Idea       | Meaning                                 |
| ---------- | --------------------------------------- |
| definable  | can be described in a formal language   |
| computable | can be calculated by a finite algorithm |

Every computable function is definable in a suitable formal system. But not every definable function is computable.

### Algorithms as Mathematical Objects

An algorithm is not just code written in a programming language. For computability theory, an algorithm must be described as a mathematical object.

It should have:

| Property                    | Meaning                                                                |
| --------------------------- | ---------------------------------------------------------------------- |
| finite description          | the procedure can be written using finitely many symbols               |
| mechanical steps            | each step follows from fixed rules                                     |
| deterministic behavior      | the next step is determined by the current state                       |
| finite input                | the procedure starts from finite data                                  |
| finite output when it halts | if it returns an answer, that answer appears after finitely many steps |

Different formal models capture this idea. The main ones include:

| Model                             | Basic idea                                                              |
| --------------------------------- | ----------------------------------------------------------------------- |
| recursive functions               | functions generated from simple initial functions by closure rules      |
| lambda calculus                   | computation as symbolic function application and reduction              |
| Turing machines                   | computation as state changes on a tape                                  |
| register machines                 | computation using numbered registers and instructions                   |
| programs in an idealized language | computation by a small programming language with loops and conditionals |

A major result is that these models define the same class of computable functions. This agreement supports the Church-Turing thesis.

### Partial and Total Functions

Computability theory often works with **partial functions**.

A total function gives an output for every input:

$$
f : \mathbb{N} \to \mathbb{N}
$$

A partial function may be undefined for some inputs:

$$
f : \mathbb{N} \rightharpoonup \mathbb{N}
$$

The symbol $\rightharpoonup$ indicates partiality.

Partial functions match real computation better than total functions. A program may halt and return an output, or it may run forever.

If a program halts on input $n$, we say:

$$
f(n)\downarrow
$$

If it does not halt on input $n$, we say:

$$
f(n)\uparrow
$$

This notation lets us distinguish two cases:

| Notation         | Meaning             |
| ---------------- | ------------------- |
| $f(n)\downarrow$ | $f(n)$ is defined   |
| $f(n)\uparrow$   | $f(n)$ is undefined |

### Recursive Functions

One way to define computability is to begin with very simple functions and close under safe operations.

The basic initial functions are:

| Function   | Definition                 |
| ---------- | -------------------------- |
| zero       | $Z(n)=0$                   |
| successor  | $S(n)=n+1$                 |
| projection | $P_i^k(x_1,\dots,x_k)=x_i$ |

From these, more functions are built using operations such as composition, primitive recursion, and minimization.

Composition lets us combine known computable functions:

$$
f(\vec{x}) = h(g_1(\vec{x}), \dots, g_m(\vec{x}))
$$

Primitive recursion defines a function by its value at $0$ and its value at $n+1$ in terms of its value at $n$.

A typical primitive recursive definition has the form:

$$
f(\vec{x},0)=g(\vec{x})
$$

$$
f(\vec{x},n+1)=h(\vec{x},n,f(\vec{x},n))
$$

This covers many familiar functions:

$$
x+y
$$

$$
x \times y
$$

$$
x^y
$$

factorials, bounded sums, bounded products, and many finite searches.

Minimization adds unbounded search. It defines:

$$
f(\vec{x}) = \mu y ,[g(\vec{x},y)=0]
$$

This means: search for the least $y$ such that $g(\vec{x},y)=0$.

If no such $y$ exists, the search never ends, so the resulting function may be partial.

### Church-Turing Thesis

The Church-Turing thesis says that the informal notion of effective computation is captured by the formal notion of Turing-computability.

Informally:

$$
\text{effectively computable} = \text{Turing-computable}
$$

This is not a theorem in the ordinary sense, because “effectively computable” is an informal concept. But it is supported by the convergence of many independent models.

Lambda calculus, recursive functions, Turing machines, register machines, and ordinary programming languages all lead to the same class of computable functions.

This convergence is one of the strongest pieces of evidence that the formal definition is the right one.

### Decidable Sets

A set $A \subseteq \mathbb{N}$ is **decidable** if there is an algorithm that always halts and correctly decides membership.

Its characteristic function is:

$$
\chi_A(n)=
\begin{cases}
1 & \text{if } n \in A,\
0 & \text{if } n \notin A.
\end{cases}
$$

The set $A$ is decidable exactly when $\chi_A$ is computable as a total function.

Examples of decidable sets include:

| Set                                 | Reason                              |
| ----------------------------------- | ----------------------------------- |
| even numbers                        | compute remainder modulo $2$        |
| prime numbers                       | test divisibility up to a bound     |
| finite sets                         | compare against each listed element |
| solutions to fixed bounded searches | check all cases                     |

Decidable sets are the computability-theoretic version of yes/no problems that can always be solved by an algorithm.

### Recursively Enumerable Sets

A set $A \subseteq \mathbb{N}$ is **recursively enumerable**, often abbreviated r.e., if there is an algorithm that lists its elements.

Equivalently, $A$ is r.e. if there is a partial computable function whose domain is exactly $A$.

So:

$$
n \in A
$$

means the computation eventually halts, while:

$$
n \notin A
$$

may cause the computation to run forever.

Every decidable set is recursively enumerable, but not every recursively enumerable set is decidable.

This difference is central. Decidable means we can always get a yes/no answer. Recursively enumerable means we can recognize yes-instances, but may wait forever on no-instances.

### The Shape of the Chapter

This chapter develops computable functions through five steps.

First, recursive functions give an algebraic way to generate computable functions from simple starting points.

Second, partial and total functions clarify the difference between computations that always halt and computations that may diverge.

Third, the Church-Turing thesis connects the formal models to the informal idea of algorithmic calculation.

Fourth, formal models of computation show that many different definitions give the same class of functions.

Fifth, the equivalence of models explains why computability is robust. It does not depend on the details of one machine model or one programming language.
