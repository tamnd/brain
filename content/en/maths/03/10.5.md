## 10.5 Equivalence of Models

The standard formal models of computation look very different, but they compute the same class of functions. This is one of the main facts behind computability theory.

The equivalence can be stated informally as:

$$
\text{Turing machines}
======================

# \text{recursive functions}

# \lambda\text{-calculus}

# \text{register machines}

\text{WHILE programs}
$$

This does not mean these models have the same syntax. It means they define the same input-output behaviors after suitable encodings.

### What Equivalence Means

Suppose model $M_1$ computes a partial function:

$$
f : \mathbb{N} \rightharpoonup \mathbb{N}
$$

Model $M_2$ is at least as strong as $M_1$ if every function computable in $M_1$ is also computable in $M_2$.

We write this informally as:

$$
M_1 \preceq M_2
$$

If both directions hold:

$$
M_1 \preceq M_2
\quad \text{and} \quad
M_2 \preceq M_1
$$

then the two models are computationally equivalent.

### Simulation

Equivalence is usually proved by simulation.

To show that $M_2$ can simulate $M_1$, we encode every configuration of $M_1$ as data inside $M_2$.

A configuration contains everything needed to continue the computation:

| Model            | Configuration contains               |
| ---------------- | ------------------------------------ |
| Turing machine   | state, tape contents, head position  |
| register machine | instruction pointer, register values |
| lambda calculus  | current expression                   |
| WHILE program    | command position, variable values    |
| rewriting system | current term                         |

Then we show that one computation step in $M_1$ can be reproduced by finitely many computation steps in $M_2$.

So if $M_1$ computes:

$$
x \mapsto y
$$

then $M_2$ computes the same function after decoding the final result.

### Turing Machines and Register Machines

A register machine can simulate a Turing machine by storing tape information in registers.

One common encoding splits the tape into two natural numbers:

| Register | Meaning                                        |
| -------- | ---------------------------------------------- |
| $L$      | symbols to the left of the head                |
| $R$      | symbol under the head and symbols to the right |
| $Q$      | current state                                  |

Moving the head left or right becomes arithmetic on the encodings.

Conversely, a Turing machine can simulate a register machine by writing register values on its tape:

$$
R_0 # R_1 # R_2 # \cdots # R_k
$$

Each register instruction can be simulated by scanning and rewriting the tape.

So:

$$
\text{Turing-computable}
========================

\text{register-machine-computable}
$$

### Turing Machines and Recursive Functions

To show that recursive functions are Turing-computable, we check the building blocks.

The initial functions are computable:

$$
Z(n)=0
$$

$$
S(n)=n+1
$$

$$
P_i^k(x_1,\dots,x_k)=x_i
$$

Composition is computable because machines can run subcomputations and feed their outputs into another computation.

Primitive recursion is computable because a machine can iterate a fixed recursive procedure $n$ times.

Minimization is computable because a machine can search:

$$
y=0,1,2,\dots
$$

until it finds:

$$
g(\vec{x},y)=0
$$

If no such $y$ exists, the search continues forever.

Thus every partial recursive function is Turing-computable.

The other direction is deeper but follows the same encoding idea. A Turing machine computation can be arithmetized. Its configurations can be encoded as natural numbers, and the transition relation can be expressed by recursive functions.

So every Turing-computable function is partial recursive.

Therefore:

$$
\text{Turing-computable}
========================

\text{partial recursive}
$$

### Turing Machines and Lambda Calculus

Lambda calculus can compute natural-number functions using encodings such as Church numerals.

A Church numeral represents $n$ by applying a function $n$ times:

$$
\overline{n}
============

\lambda f.\lambda x.\underbrace{f(f(\cdots f}_{n\text{ times}}(x)\cdots))
$$

For example:

$$
\overline{0}=\lambda f.\lambda x.x
$$

$$
\overline{1}=\lambda f.\lambda x.f(x)
$$

$$
\overline{2}=\lambda f.\lambda x.f(f(x))
$$

Basic arithmetic can be represented by lambda terms. More importantly, lambda calculus can encode recursion and simulate machines.

Conversely, a Turing machine can manipulate lambda expressions as finite strings and perform beta reduction.

So:

$$
\text{Turing-computable}
========================

\lambda\text{-computable}
$$

### WHILE Programs and Partial Recursive Functions

WHILE programs compute partial functions because loops may fail to terminate.

Initial functions are easy to program.

Composition is implemented by running one block of code after another.

Primitive recursion is implemented by bounded loops.

Minimization is implemented by unbounded search:

```text
y := 0
while g(x, y) != 0 do
    y := y + 1
end
return y
```

If some $y$ satisfies the condition, the loop halts. If none exists, the loop runs forever.

So every partial recursive function can be computed by a WHILE program.

Conversely, each WHILE program can be encoded and simulated by a Turing machine or by recursive functions. The state of the program consists of variable values and a program counter, both of which can be encoded as natural numbers.

Thus:

$$
\text{WHILE-computable}
=======================

\text{partial recursive}
$$

### Why the Equivalence Matters

The equivalence of models shows that computability is robust.

If only one model existed, we might worry that its definition was too narrow or too artificial. But many independent models reach the same boundary.

They differ in presentation:

| Model               | Looks like           |
| ------------------- | -------------------- |
| Turing machine      | mechanical device    |
| recursive functions | algebra of functions |
| lambda calculus     | symbolic functions   |
| register machine    | low-level program    |
| WHILE language      | structured program   |

Yet they agree on what can be computed.

This means computability does not depend on a particular syntax, machine design, or programming style.

### Consequence for Undecidability

Because the models are equivalent, an impossibility proof in one model transfers to all the others.

If no Turing machine decides a problem, then no WHILE program decides it, no register machine decides it, and no recursive function decides it.

This is why the halting problem is a general impossibility result, not a weakness of one formalism.

The equivalence theorem turns machine-specific results into statements about computation itself.
