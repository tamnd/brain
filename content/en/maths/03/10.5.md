## 10.5 Equivalence of Models

The standard models of computation look different, but they compute the same class of functions.

This is the main equivalence:

# $$ \mathrm{TuringComputable}

# \mathrm{PartialRecursive}

# \mathrm{LambdaComputable}

# \mathrm{RegisterComputable}

\mathrm{WhileComputable}
$$

This does not mean the models have the same syntax. A Turing machine uses tapes and states. Lambda calculus uses expressions. Recursive functions use equations. Register machines use instructions. WHILE programs use variables and loops.

The equivalence means that, after suitable encoding, they describe the same input-output behavior.

### What Equivalence Means

Let $M_1$ and $M_2$ be two models of computation.

We say $M_2$ can simulate $M_1$ if every function computed by $M_1$ can also be computed by $M_2$.

Write:

$$
M_1 \preceq M_2
$$

If both simulations hold,

$$
M_1 \preceq M_2
$$

and

$$
M_2 \preceq M_1
$$

then the models are equivalent.

So equivalence means:

$$
M_1 \equiv M_2
$$

The symbol $\equiv$ here means equality of computational power, not equality of notation.

### Simulation by Configurations

A simulation works by encoding configurations.

A **configuration** is a complete snapshot of a computation at one moment.

| Model               | Configuration contains                     |
| ------------------- | ------------------------------------------ |
| Turing machine      | state, tape contents, head position        |
| register machine    | instruction pointer, register values       |
| lambda calculus     | current expression                         |
| WHILE program       | current command, variable values           |
| recursive functions | current values in the defining computation |

To simulate a model, we encode each configuration as data in another model. Then we show that one step of the first model can be reproduced by finitely many steps of the second.

The general pattern is:

$$
C_0 \to C_1 \to C_2 \to \cdots \to C_k
$$

After encoding:

$$
\ulcorner C_0 \urcorner \to^* \ulcorner C_1 \urcorner \to^* \ulcorner C_2 \urcorner \to^* \cdots \to^* \ulcorner C_k \urcorner
$$

Here $\to^*$ means zero or more simulation steps.

### Turing Machines and Register Machines

A register machine can simulate a Turing machine by storing the tape in registers.

One common method stores the left and right parts of the tape as numbers:

| Register | Meaning                                             |
| -------- | --------------------------------------------------- |
| $L$      | symbols left of the head                            |
| $R$      | symbol under the head and symbols right of the head |
| $Q$      | current state                                       |

Moving the head becomes arithmetic on $L$ and $R$.

A Turing machine can simulate a register machine by writing register contents on its tape:

```text
R0 # R1 # R2 # ... # Rk
```

Each register instruction can be simulated by scanning, changing, and rewriting the tape.

Therefore:

# $$ \mathrm{TuringComputable}

\mathrm{RegisterComputable}
$$

### Turing Machines and Recursive Functions

Every partial recursive function is Turing-computable.

The initial functions are computable:

$$
Z(n)=0
$$

$$
S(n)=n+1
$$

$$
P_i^k(x_1,\dots,x_k)=x_i
$$

Composition is computable because machines can run subcomputations and feed their outputs into another computation.

Primitive recursion is computable because a machine can perform a bounded iteration.

Minimization is computable because a machine can search:

$$
y=0,1,2,3,\dots
$$

until it finds:

$$
g(\vec{x},y)=0
$$

If no such $y$ exists, the machine does not halt.

Conversely, every Turing machine computation can be encoded arithmetically. Its configurations can be represented by natural numbers, and its transition relation can be described by recursive functions.

Therefore:

# $$ \mathrm{TuringComputable}

\mathrm{PartialRecursive}
$$

### Turing Machines and Lambda Calculus

Lambda calculus can represent natural numbers using Church numerals.

The Church numeral for $n$ is:

# $$ \overline{n}

\lambda f.\lambda x.f^n(x)
$$

Examples:

$$
\overline{0}=\lambda f.\lambda x.x
$$

$$
\overline{1}=\lambda f.\lambda x.f(x)
$$

$$
\overline{2}=\lambda f.\lambda x.f(f(x))
$$

Lambda terms can represent arithmetic, conditionals, pairs, recursion, and machine configurations.

So lambda calculus can simulate Turing machines.

Conversely, a Turing machine can encode lambda terms as strings and perform beta reduction step by step.

Therefore:

# $$ \mathrm{TuringComputable}

\mathrm{LambdaComputable}
$$

### WHILE Programs and Recursive Functions

WHILE programs compute partial functions because loops may fail to terminate.

Initial functions can be programmed directly.

Composition is implemented by running code blocks in sequence.

Primitive recursion is implemented by bounded loops.

Minimization is implemented by an unbounded loop:

```text
y := 0
while g(x, y) != 0 do
    y := y + 1
end
return y
```

If a suitable $y$ exists, the program halts. If no such $y$ exists, it runs forever.

Thus every partial recursive function can be computed by a WHILE program.

Conversely, every WHILE program can be simulated by recursive functions by encoding its variables and program counter as natural numbers.

Therefore:

# $$ \mathrm{WhileComputable}

\mathrm{PartialRecursive}
$$

### Why Equivalence Matters

The equivalence of models shows that computability is robust.

It does not depend on one syntax, one machine design, or one programming language.

| Model               | Surface form        | Underlying power             |
| ------------------- | ------------------- | ---------------------------- |
| Turing machine      | tape and states     | partial computable functions |
| recursive functions | equations           | partial computable functions |
| lambda calculus     | symbolic functions  | partial computable functions |
| register machine    | instructions        | partial computable functions |
| WHILE program       | variables and loops | partial computable functions |

This agreement supports the Church-Turing thesis.

It also lets us choose whichever model is easiest for the proof at hand.

Use Turing machines for halting and simulation arguments.

Use recursive functions for arithmetic and formal logic.

Use lambda calculus for programming language semantics.

Use WHILE programs for algorithmic intuition.

### Consequence for Impossibility Results

Because the models are equivalent, an impossibility result in one model transfers to the others.

If no Turing machine decides a problem $P$, then no register machine, WHILE program, lambda term, or recursive function decides $P$.

So undecidability is not a limitation of one formalism. It is a limitation of computation itself.
