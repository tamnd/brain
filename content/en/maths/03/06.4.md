## 6.4 Stability Theory

Types measure how many different ways elements can behave. Stability theory studies when this space of types is controlled rather than chaotic.

The central question is: given a theory $T$, how complicated are the types over its models?

### Counting Types

Let $A$ be a set of parameters. Recall that:

$$
S(A)
$$

denotes the set of complete types over $A$.

A theory $T$ is **stable** if the number of types over any set $A$ does not grow too fast.

A basic form of stability says:

> for every infinite set $A$, the number of complete types over $A$ is at most $|A|$.

So:

$$
|S(A)| \le |A|
$$

for all infinite $A$.

If this holds, the theory avoids uncontrolled combinatorial explosion.

### Instability

A theory is **unstable** if it admits too many types.

This usually appears when the theory can encode arbitrary patterns.

A standard signal of instability is the **order property**.

A formula $\varphi(x,y)$ has the order property if there exist sequences:

$$
(a_i)*{i\in\mathbb{N}}, \quad (b_j)*{j\in\mathbb{N}}
$$

such that:

$$
\mathcal M \models \varphi(a_i,b_j)
\quad \text{if and only if} \quad i<j
$$

This encodes an infinite linear order inside the theory.

If a theory has the order property, it is unstable.

Example: the theory of linear orders is unstable, because the relation $x<y$ itself has this property.

### Stable Theories

A theory is stable when such patterns cannot be built.

Examples of stable theories include:

* algebraically closed fields
* vector spaces over a fixed field

These theories have well-behaved types. The possible behaviors of elements are limited and structured.

In stable theories, types often correspond to geometric objects such as varieties or linear subspaces.

### Forking (Idea)

In stable theories, independence between elements can be defined using types.

The key notion is **forking**.

Forking measures when a type depends on additional parameters in a nontrivial way.

Informally:

* a type does not fork over $A$ if it does not introduce new complexity beyond $A$
* a type forks if it encodes additional constraints that increase complexity

Forking behaves like independence in algebra.

For example, in vector spaces, linear independence matches the behavior of non-forking.

### Stability Hierarchy

Stability theory refines into a hierarchy.

| Class           | Description                                   |
| --------------- | --------------------------------------------- |
| Stable          | controlled number of types                    |
| Superstable     | stronger structural regularity                |
| $\omega$-stable | only countably many types over countable sets |

Each level imposes stricter limits on how types behave.

For example, $\omega$-stable theories are highly structured and often admit classification by dimension-like invariants.

### Example: Vector Spaces

Consider the theory of infinite-dimensional vector spaces over a fixed field.

Types correspond to linear conditions.

For example:

$$
x \notin \mathrm{span}(A)
$$

describes elements independent from $A$.

The number of such types over a set $A$ is controlled by the dimension of the space.

This theory is stable. It does not allow arbitrary combinatorial patterns.

### Example: Dense Linear Orders

Consider dense linear orders without endpoints.

This theory is unstable.

The relation $x<y$ allows arbitrary ordering patterns.

For any sequence:

$$
a_0 < a_1 < a_2 < \dots
$$

we can define many distinct types depending on how a new element compares to each $a_i$.

This produces many more types than the size of the parameter set.

### Why Stability Matters

Stability separates theories into two broad classes:

* those with manageable structure
* those with wild, unclassifiable behavior

In stable theories:

* types behave predictably
* independence is well-defined
* models can often be classified

In unstable theories:

* types proliferate
* independence is harder to control
* classification becomes difficult

### Perspective

Stability theory turns the study of models into the study of types.

Instead of focusing only on elements and sets, it analyzes the space of all possible behaviors.

Stable theories behave like geometric objects. Unstable theories behave like arbitrary combinatorial systems.

This distinction drives the classification program, which aims to organize theories based on their structural complexity.
