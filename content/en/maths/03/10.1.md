## 10.1 Recursive Functions

Recursive functions give a purely algebraic definition of computability. Instead of describing machines or programs, we start with a few basic functions and close under operations that build more complex ones.

The idea is simple: begin with functions that are obviously computable, then allow controlled ways to combine them.

### Initial Functions

The construction starts with three basic functions over the natural numbers.

**Zero function**

$$
Z(n) = 0
$$

This ignores the input and always returns $0$.

**Successor function**

$$
S(n) = n + 1
$$

This adds one to the input.

**Projection functions**

$$
P_i^k(x_1, x_2, \dots, x_k) = x_i
$$

This selects the $i$-th argument from a tuple of $k$ inputs.

These functions are simple, finite, and clearly computable.

### Closure Operations

From the initial functions, we build new functions using three operations.

#### Composition

If $g_1, \dots, g_m$ and $h$ are already defined, then we define:

$$
f(\vec{x}) = h\big(g_1(\vec{x}), \dots, g_m(\vec{x})\big)
$$

This means we first compute each $g_i(\vec{x})$, then apply $h$ to the results.

Example:

If:

$$
g(x) = x + 1, \quad h(x,y) = x + y
$$

then:

$$
f(x) = h(g(x), x) = (x+1) + x = 2x + 1
$$

#### Primitive Recursion

Primitive recursion defines a function by specifying:

* its value at $0$
* how to compute the next value from the previous one

The general form is:

$$
f(\vec{x}, 0) = g(\vec{x})
$$

$$
f(\vec{x}, n+1) = h(\vec{x}, n, f(\vec{x}, n))
$$

Here:

* $g$ gives the base case
* $h$ gives the step case

Example: addition

Define $x + y$ by recursion on $y$:

$$
x + 0 = x
$$

$$
x + (y+1) = S(x + y)
$$

Example: multiplication

$$
x \cdot 0 = 0
$$

$$
x \cdot (y+1) = (x \cdot y) + x
$$

Example: exponentiation

$$
x^0 = 1
$$

$$
x^{y+1} = x^y \cdot x
$$

Primitive recursion captures many familiar functions. It corresponds to bounded iteration where each step depends on the previous result.

#### Minimization

Minimization introduces unbounded search.

Given a function $g(\vec{x}, y)$, define:

$$
f(\vec{x}) = \mu y ,[g(\vec{x}, y) = 0]
$$

This means: find the smallest $y$ such that $g(\vec{x}, y)=0$.

If such a $y$ exists, $f(\vec{x})$ returns it. If no such $y$ exists, the computation never halts.

Minimization allows functions that may be undefined for some inputs. This is where partiality appears.

### Primitive Recursive Functions

Functions built using only:

* initial functions
* composition
* primitive recursion

are called **primitive recursive functions**.

They are always total. For every input, the computation finishes and returns a value.

Examples include:

* addition
* multiplication
* exponentiation
* factorial
* bounded sums and products

Example: factorial

$$
0! = 1
$$

$$
(n+1)! = (n+1) \cdot n!
$$

### General Recursive Functions

When we add minimization, we obtain the class of **general recursive functions**, also called **partial recursive functions**.

These functions may fail to terminate for some inputs.

So we distinguish:

| Class               | Property       |
| ------------------- | -------------- |
| primitive recursive | always total   |
| partial recursive   | may be partial |

Every primitive recursive function is partial recursive, but not the other way around.

### Why Recursive Definitions Matter

Recursive functions provide a machine-free definition of computation. They describe what can be computed using only symbolic rules.

This approach has several advantages:

* it is purely mathematical
* it avoids dependence on hardware or programming languages
* it aligns with formal proof systems
* it connects naturally to logic and arithmetic

Later, we will see that recursive functions define the same class of computable functions as Turing machines and other models. This agreement is a key part of the theory.
