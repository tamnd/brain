## 10.1 Recursive Functions

Recursive functions give one of the cleanest mathematical definitions of computation. Instead of starting with machines, tapes, memory, or programs, we start with functions on natural numbers.

The guiding idea is simple. Some functions are obviously computable. If we allow only precise rules for building new functions from old ones, then every function we obtain should also be computable.

We begin with three kinds of basic functions.

### Initial Functions

The **zero function** always returns $0$:

$$
Z(n)=0
$$

It ignores its input.

The **successor function** adds one:

$$
S(n)=n+1
$$

This is the formal version of counting one step forward.

The **projection functions** choose one input from a list of inputs:

$$
P_i^k(x_1,\dots,x_k)=x_i
$$

For example:

$$
P_2^3(x_1,x_2,x_3)=x_2
$$

These functions are not meant to be impressive. They are meant to be unquestionable starting points. Each can be computed by a finite, mechanical procedure.

### Composition

Composition builds a new function by feeding the outputs of some functions into another function.

If $g_1,\dots,g_m$ are computable functions, and $h$ is also computable, then:

$$
f(\vec{x}) = h(g_1(\vec{x}),\dots,g_m(\vec{x}))
$$

is computable.

This matches ordinary programming. First compute the intermediate values, then pass them to the next operation.

For example, suppose we already know how to compute addition and successor. Then the function:

$$
f(x,y)=S(x+y)
$$

is computable because it first computes $x+y$, then applies $S$.

### Primitive Recursion

Primitive recursion defines a function by giving a base case and a step case.

The general form is:

$$
f(\vec{x},0)=g(\vec{x})
$$

$$
f(\vec{x},n+1)=h(\vec{x},n,f(\vec{x},n))
$$

This says: to compute $f$ at $n+1$, use the previous value $f(\vec{x},n)$.

The variable $n$ acts like a counter. The computation proceeds for exactly $n$ steps, so it always stops.

A basic example is addition. Define $x+y$ by recursion on $y$:

$$
x+0=x
$$

$$
x+(y+1)=S(x+y)
$$

So adding $y$ means applying successor $y$ times to $x$.

Multiplication is defined from addition:

$$
x\cdot 0=0
$$

$$
x\cdot (y+1)=x\cdot y+x
$$

So multiplying by $y$ means adding $x$ repeatedly.

Exponentiation is defined from multiplication:

$$
x^0=1
$$

$$
x^{y+1}=x^y\cdot x
$$

This pattern is important. More complex computable functions are built from simpler ones by fixed rules.

### Primitive Recursive Functions

A function is **primitive recursive** if it can be built from the initial functions using composition and primitive recursion.

Primitive recursive functions are always total:

$$
f:\mathbb{N}^k \to \mathbb{N}
$$

For every input, the function returns a value.

This happens because primitive recursion uses bounded repetition. When computing $f(\vec{x},n)$, the number of recursive steps is controlled by the input $n$. There is no open-ended search.

Many familiar functions are primitive recursive:

| Function        | Recursive idea                          |
| --------------- | --------------------------------------- |
| addition        | repeated successor                      |
| multiplication  | repeated addition                       |
| exponentiation  | repeated multiplication                 |
| factorial       | repeated multiplication from $1$ to $n$ |
| bounded sum     | add finitely many values                |
| bounded product | multiply finitely many values           |

For example, factorial is defined by:

$$
0! = 1
$$

$$
(n+1)! = (n+1)\cdot n!
$$

The definition gives an immediate procedure for computation.

### Minimization

Primitive recursion only gives total functions. To model computations that may fail to halt, we add **minimization**.

If $g(\vec{x},y)$ is a computable function, define:

$$
f(\vec{x})=\mu y,[g(\vec{x},y)=0]
$$

This means: search for the least natural number $y$ such that $g(\vec{x},y)=0$.

The search checks:

$$
y=0,1,2,3,\dots
$$

If it finds a solution, it returns the first one. If no solution exists, the search continues forever.

So minimization may define a partial function:

$$
f:\mathbb{N}^k \rightharpoonup \mathbb{N}
$$

Example:

$$
f(x)=\mu y,[y^2=x]
$$

If $x=9$, then:

$$
f(9)=3
$$

If $x=10$, there is no natural number $y$ with $y^2=10$, so $f(10)$ is undefined.

### General Recursive Functions

When we allow initial functions, composition, primitive recursion, and minimization, we obtain the **partial recursive functions**, also called **general recursive functions**.

The hierarchy is:

$$
\text{primitive recursive} \subseteq \text{partial recursive}
$$

Primitive recursive functions always halt.

Partial recursive functions may fail to halt.

This matches the behavior of real programs. A program may compute an answer for some inputs and loop forever on others.

### Why This Definition Matters

Recursive functions define computation without mentioning a machine. Everything is expressed using functions, equations, and closure rules.

This is useful because it connects computability with arithmetic and logic. It lets us study algorithms inside mathematics itself.

The key lesson is this: computation can be characterized by how functions are generated. Start with simple functions. Close under safe operations. Add unbounded search to model possible non-termination. The resulting class matches the functions computable by Turing machines.
