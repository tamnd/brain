## Chapter 6. Definability and Types

Model theory studies structures through the formulas that hold inside them. Chapter 6 focuses on two central ideas: **definability** and **types**.

Definability asks which sets, relations, and functions can be described by formulas in a given language.

Suppose $\mathcal M$ is a structure with domain $M$. A subset $X \subseteq M$ is definable if there is a formula $\varphi(x)$ such that:

$$
X={a\in M : \mathcal M \models \varphi(a)}
$$

So a formula acts like a description. It selects exactly the elements satisfying it.

For example, in the ordered structure:

$$
(\mathbb{N}; <)
$$

the formula:

$$
x < 5
$$

defines the set:

$$
{0,1,2,3,4}
$$

assuming $5$ is available as a named parameter or definable term.

In arithmetic:

$$
(\mathbb{N};0,1,+,\times,<)
$$

the formula:

$$
\exists y,(x = y+y)
$$

defines the set of even natural numbers.

Definability is sensitive to language. The same underlying domain may have more or fewer definable sets depending on which symbols are available.

For example, the language:

$$
( \mathbb{N}; < )
$$

can express order properties.

The richer language:

$$
( \mathbb{N}; 0,1,+,\times,< )
$$

can express arithmetic properties.

Adding symbols gives formulas more expressive power.

Types describe possible behavior of elements, even when those elements are not already present in the structure. A type is a consistent collection of formulas describing what an element should satisfy.

For example, over the natural numbers with the usual order, we may consider the collection:

$$
{x>0,\ x>1,\ x>2,\ x>3,\dots}
$$

This describes an element larger than every standard natural number. No ordinary natural number realizes this collection in $\mathbb{N}$. But in a nonstandard model of arithmetic, such an element may exist.

This motivates the idea of **realization**.

A type is **realized** in a structure if some element of the structure satisfies all formulas in the type.

If no such element exists, the type is **omitted**.

Saturated models are structures that realize many types. They are large, rich models where every finitely consistent local description is represented by actual elements. Saturation gives model theory a powerful way to study completeness, homogeneity, and classification.

The chapter then moves to stability theory. Stability studies when a theory has a controlled space of types. In stable theories, types behave regularly enough that models can often be classified by structural invariants. In unstable theories, types proliferate in ways that make classification harder.

This leads to the broader **classification program** in model theory. The aim is to divide first-order theories according to how their models behave: stable or unstable, simple or complicated, classifiable or wild.

Chapter 6 develops these ideas in five stages:

| Section | Topic                        | Main question                                    |
| ------- | ---------------------------- | ------------------------------------------------ |
| 6.1     | Definable sets and functions | What can formulas describe inside a structure?   |
| 6.2     | Types and realizations       | What complete local descriptions are consistent? |
| 6.3     | Saturated models             | Which models realize enough types?               |
| 6.4     | Stability theory overview    | When do types behave in a controlled way?        |
| 6.5     | Classification programs      | How can theories be organized by model behavior? |

Definability connects syntax with geometry inside a structure. Types extend this view from actual elements to possible elements. Saturation and stability then turn those local descriptions into tools for comparing whole theories.
