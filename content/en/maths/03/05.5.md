## 5.5 Limitations of First-Order Logic

The main theorems of this chapter show both the strength and the limits of first-order logic. Compactness and the Löwenheim–Skolem theorems imply that certain natural properties cannot be expressed within first-order logic.

These limitations are structural. They do not depend on a particular theory, but on the expressive power of the language itself.

### Local Nature of First-Order Logic

First-order formulas describe structures using finitely many variables and finite combinations of connectives and quantifiers.

Every formula has a bounded “scope”. It can only inspect a finite portion of a structure at a time.

This leads to a general principle:

* first-order logic captures **local properties**
* it struggles with **global properties**

Compactness reinforces this. If a property requires checking infinitely many conditions at once, then first-order logic usually cannot express it as a single sentence.

### Failure to Express Finiteness

One of the most basic limitations is that first-order logic cannot express that a structure is finite.

We can write, for each $n$:

$$
\exists x_1,\dots,x_n \bigwedge_{i<j} x_i \neq x_j
$$

This says that the structure has at least $n$ elements.

But there is no single sentence saying “the structure is finite”.

If such a sentence existed, compactness would produce a contradiction, because we could combine it with all lower-bound sentences and still obtain a model.

So finiteness is not first-order definable.

### Failure to Control Cardinality

First-order logic also cannot control the size of infinite models.

By the Löwenheim–Skolem theorems:

* if a theory has one infinite model, it has countable models
* it also has arbitrarily large models

So there is no first-order sentence that says:

* “the structure has exactly $\aleph_0$ elements”
* “the structure is uncountable”

These properties depend on the size of the domain, which first-order logic cannot fix.

### Nonstandard Models

First-order theories often have unintended models.

For arithmetic, the axioms describe addition, multiplication, and order, but they do not exclude nonstandard elements.

So there are models of arithmetic that:

* satisfy all axioms
* contain elements larger than every standard natural number

This shows that first-order axioms cannot uniquely characterize structures like $\mathbb N$.

### Limits on Definability

Some properties cannot be expressed because first-order logic cannot quantify over all subsets or all functions.

For example:

* “every subset has a least element”
* “every function has a fixed point”

These require quantification over sets or functions, which is beyond first-order logic.

Such properties can be expressed in second-order logic, but not in first-order logic.

### Graph-Theoretic Example

Consider graphs in the language with one binary relation $E(x,y)$.

The property “the graph is connected” cannot be expressed by a single first-order sentence.

Connectivity requires saying that for any two vertices, there is a path of some finite length connecting them.

A first-order sentence can describe paths of a fixed length:

$$
\exists x_1,\dots,x_n
$$

but it cannot quantify over all possible path lengths at once.

So connectivity is not first-order definable.

### Order-Theoretic Example

In linear orders, the property of being a well-order cannot be captured in first-order logic.

A well-order requires that every nonempty subset has a least element.

This again involves quantifying over all subsets, which is not allowed in first-order logic.

As a result, there are non-well-ordered models that satisfy all first-order properties of well-orders.

### Compactness as a Source of Limitations

Compactness explains many of these failures.

If a property requires enforcing infinitely many constraints simultaneously, then:

* each finite subset may be satisfiable
* but the full condition may fail to hold uniquely

Compactness guarantees the existence of models that satisfy all finite constraints, even if they violate the intended global property.

### Summary of Limitations

| Property                 | First-order definable |
| ------------------------ | --------------------- |
| At least $n$ elements    | Yes                   |
| Finite                   | No                    |
| Countable                | No                    |
| Uncountable              | No                    |
| Connected graph          | No                    |
| Well-ordering            | No                    |
| Standard natural numbers | No                    |

These are not accidental gaps. They reflect a general boundary of expressive power.

### Perspective

First-order logic is balanced between expressiveness and control.

It is strong enough to formalize most of mathematics and to support powerful theorems like compactness and completeness.

At the same time, it is restricted enough to admit multiple models, nonstandard elements, and variation in size.

These limitations are not defects. They are part of what makes first-order logic manageable and widely applicable.

Understanding these limits helps clarify when first-order logic is the right tool, and when stronger systems are needed.
