## 7.3 Ordinals and Well-Ordering

Cardinality measures **how many** elements a set has. Ordinals measure **position** and **order**.

For finite sets, these ideas look almost the same. The set:

$$
{0,1,2}
$$

has three elements, and we often identify it with the number:

$$
3
$$

But in infinite set theory, size and order separate. Two sets can have the same cardinality but very different order structures.

### Well-Orders

A **linear order** on a set $A$ is a relation $\leq$ such that any two elements can be compared.

For all $a,b,c \in A$:

1. $a \leq a$
2. if $a \leq b$ and $b \leq a$, then $a=b$
3. if $a \leq b$ and $b \leq c$, then $a \leq c$
4. either $a \leq b$ or $b \leq a$

A **well-order** is a linear order in which every nonempty subset has a least element.

Formally, $\leq$ is a well-order on $A$ if every nonempty $B \subseteq A$ has some $b_0 \in B$ such that:

$$
b_0 \leq b
\quad \text{for all } b \in B
$$

The natural numbers with their usual order are well-ordered:

$$
0 < 1 < 2 < 3 < \cdots
$$

Every nonempty subset of $\mathbb{N}$ has a least element.

The integers with their usual order are not well-ordered:

$$
\cdots < -3 < -2 < -1 < 0 < 1 < 2 < \cdots
$$

For example, $\mathbb{Z}$ itself has no least element.

### Ordinals

An **ordinal** is a canonical order type of a well-ordered set.

You can think of an ordinal as a way of counting positions in a well-order.

The finite ordinals are:

$$
0 = \varnothing
$$

$$
1 = {0}
$$

$$
2 = {0,1}
$$

$$
3 = {0,1,2}
$$

In general:

$$
n = {0,1,\dots,n-1}
$$

So each ordinal is the set of all smaller ordinals.

The first infinite ordinal is:

$$
\omega
$$

It is the order type of the natural numbers:

$$
0 < 1 < 2 < 3 < \cdots
$$

As a set:

$$
\omega = {0,1,2,3,\dots}
$$

### Successor Ordinals

Given an ordinal $\alpha$, its **successor** is:

$$
\alpha + 1 = \alpha \cup {\alpha}
$$

For finite ordinals, this matches ordinary counting:

$$
0+1=1
$$

$$
1+1=2
$$

$$
2+1=3
$$

For the first infinite ordinal:

$$
\omega + 1
$$

has order type:

$$
0 < 1 < 2 < 3 < \cdots < \omega
$$

This is like all natural numbers followed by one new final point.

It has the same cardinality as $\omega$, but a different order type.

### Limit Ordinals

An ordinal is a **limit ordinal** if it is not $0$ and is not a successor ordinal.

The first limit ordinal is:

$$
\omega
$$

It has no immediate predecessor. There is no largest natural number right before $\omega$.

Other examples include:

$$
\omega + \omega
$$

and:

$$
\omega^2
$$

The order type $\omega+\omega$ looks like two copies of the natural numbers placed one after another:

$$
0,1,2,\dots,\omega,\omega+1,\omega+2,\dots
$$

Limit ordinals are important because they describe processes that continue through infinitely many earlier stages.

### Transfinite Recursion

Ordinals allow definitions by stages beyond the finite.

A construction by **transfinite recursion** usually has three cases:

1. Define the value at $0$.
2. Define the value at a successor stage $\alpha+1$ from the value at $\alpha$.
3. Define the value at a limit stage $\lambda$ from all earlier stages $\alpha<\lambda$.

This pattern generalizes ordinary recursion.

For example, suppose we build sets $A_\alpha$ by:

$$
A_0 = \varnothing
$$

$$
A_{\alpha+1} = A_\alpha \cup {\alpha}
$$

and for a limit ordinal $\lambda$:

$$
A_\lambda = \bigcup_{\alpha<\lambda} A_\alpha
$$

Then:

$$
A_1={0}
$$

$$
A_2={0,1}
$$

$$
A_3={0,1,2}
$$

and:

$$
A_\omega={0,1,2,\dots}
$$

### Transfinite Induction

Ordinals also support **transfinite induction**.

To prove that a property $P(\alpha)$ holds for all ordinals $\alpha$, it is enough to show:

$$
\forall \alpha,\bigl((\forall \beta<\alpha,\ P(\beta)) \to P(\alpha)\bigr)
$$

This says: if $P$ holds for every earlier ordinal, then it holds at $\alpha$.

This single rule covers the base case, successor case, and limit case.

In practice, we often split the proof into three parts:

1. Prove $P(0)$.
2. Prove $P(\alpha+1)$ assuming $P(\alpha)$.
3. Prove $P(\lambda)$ for limit $\lambda$ assuming $P(\alpha)$ for all $\alpha<\lambda$.

### Order Type

Two well-ordered sets have the same **order type** if there is an order-preserving bijection between them.

For example:

$$
0 < 2 < 4 < 6 < \cdots
$$

has the same order type as:

$$
0 < 1 < 2 < 3 < \cdots
$$

Both have order type $\omega$.

But:

$$
1 < 2 < 3 < \cdots < 0
$$

has order type:

$$
\omega + 1
$$

It has one final element after an infinite increasing sequence.

Even though both sets are countable, their order types differ.

### Ordinals vs Cardinals

A **cardinal** measures size.

An **ordinal** measures well-ordered position.

For finite sets, the distinction is small:

$$
3
$$

can mean both the number of elements and the order type of:

$$
0<1<2
$$

For infinite sets, the distinction matters:

$$
\omega
$$

and:

$$
\omega+1
$$

have the same cardinality:

$$
|\omega| = |\omega+1|
$$

but different order types:

$$
\omega \neq \omega+1
$$

Cardinals forget order. Ordinals remember order.
