## 6.5 Classification Programs

Model theory aims to understand theories by organizing them according to how their models behave. Classification programs provide this organization.

The central idea is to divide theories into classes where models can be described using structural invariants, and to separate them from theories where no such description is possible.

### Classifiable vs Non-Classifiable

A theory is **classifiable** if its models admit a systematic description.

This usually means:

* types are well-controlled
* independence behaves regularly
* models can be described using invariants such as dimension or rank

A theory is **non-classifiable** when its models exhibit too much combinatorial complexity.

In such theories:

* types proliferate
* independence breaks down
* no uniform description of models is available

This distinction is a guiding principle in modern model theory.

### Stability as the First Divide

Stability theory provides the first major boundary.

| Class    | Behavior                              |
| -------- | ------------------------------------- |
| Stable   | controlled types, geometric structure |
| Unstable | many types, combinatorial behavior    |

Stable theories often support classification.

Unstable theories tend to resist it.

For example:

* vector spaces form a stable, classifiable theory
* linear orders form an unstable, non-classifiable theory

### Refining Stability

Within stable theories, further distinctions appear.

| Class            | Feature                            |
| ---------------- | ---------------------------------- |
| $\omega$-stable  | very few types over countable sets |
| Superstable      | strong control over independence   |
| Stable (general) | controlled but broader behavior    |

These refinements correspond to increasing levels of structure.

In highly stable settings, models can often be described by dimension-like invariants.

### Morley Rank and Dimension

One approach to classification uses a notion of dimension.

In certain stable theories, definable sets can be assigned a rank, called **Morley rank**.

This behaves like dimension in geometry.

For example:

* finite sets have rank $0$
* infinite definable sets may have higher rank
* rank decreases under definable restrictions

This allows classification of models using geometric ideas.

### Example: Algebraically Closed Fields

The theory of algebraically closed fields is a central example of a classifiable theory.

In this setting:

* definable sets correspond to algebraic varieties
* types correspond to geometric conditions
* dimension corresponds to algebraic dimension

Models are classified by their transcendence degree.

So two algebraically closed fields are elementarily equivalent if they have the same characteristic, and their models are distinguished by size and transcendence structure.

### Unstable Theories

Unstable theories lack this structure.

The presence of the order property allows encoding arbitrary patterns.

This leads to:

* many distinct types
* no clear notion of independence
* failure of geometric classification

Examples include:

* linear orders
* random graphs
* many combinatorial structures

These theories behave more like arbitrary discrete systems than geometric objects.

### Dividing Lines

Modern classification theory identifies precise dividing lines between tame and wild behavior.

Some important dividing lines include:

* stability
* simplicity
* NIP (no independence property)

Each dividing line excludes certain combinatorial patterns.

For example:

* NIP theories exclude formulas that encode arbitrary subsets of large sets
* simple theories extend some ideas of stability but allow more flexibility

These dividing lines refine the classification landscape.

### Strategy of Classification

A typical classification approach follows these steps:

1. identify a dividing line that the theory satisfies
2. study types and independence within that class
3. define invariants that describe models
4. prove that models are determined by these invariants

This transforms the problem of understanding all models into a structured classification problem.

### Perspective

Classification programs aim to answer a global question:

> when can a theory be understood through structure, and when does it behave too freely?

Stable and highly structured theories resemble geometry. Their models can be described, compared, and classified.

Unstable theories resemble combinatorics. Their models vary too widely for a simple description.

This distinction shapes much of modern model theory and guides how new theories are analyzed.
