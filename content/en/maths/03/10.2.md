## 10.2 Partial vs Total Functions

Computability theory must account for a basic fact about programs: some programs halt, and some programs run forever.

This is why we distinguish **total functions** from **partial functions**.

### Total Functions

A function is **total** if it gives an output for every input.

For example:

$$
f:\mathbb{N}\to\mathbb{N}
$$

means that for every $n\in\mathbb{N}$, the value $f(n)$ is defined.

Examples:

$$
f(n)=n+1
$$

$$
g(n)=2n
$$

$$
h(n)=n^2
$$

Each input produces an output. There is no missing case.

In programming terms, a total computable function corresponds to a program that halts on every valid input.

### Partial Functions

A function is **partial** if it may be undefined for some inputs.

We write:

$$
f:\mathbb{N}\rightharpoonup\mathbb{N}
$$

The arrow $\rightharpoonup$ indicates that the function may fail to return a value.

For example:

$$
f(n)=
\begin{cases}
0 & \text{if } n \text{ is even},\
\text{undefined} & \text{if } n \text{ is odd}.
\end{cases}
$$

Then:

$$
f(4)=0
$$

but:

$$
f(5)
$$

has no value.

This does not mean the value is $0$. It also does not mean the value is false. It means the function gives no output for that input.

### Definedness

Computability theory uses notation for whether a computation halts.

If $f(n)$ is defined, we write:

$$
f(n)\downarrow
$$

This means the computation halts and returns a value.

If $f(n)$ is undefined, we write:

$$
f(n)\uparrow
$$

This means the computation does not halt.

| Notation         | Meaning             |
| ---------------- | ------------------- |
| $f(n)\downarrow$ | $f(n)$ is defined   |
| $f(n)\uparrow$   | $f(n)$ is undefined |

For several inputs, we write:

$$
f(x_1,\dots,x_k)\downarrow
$$

or:

$$
f(x_1,\dots,x_k)\uparrow
$$

### Why Partial Functions Are Needed

Partial functions are needed because unbounded computation may never finish.

Consider a search procedure:

```text
y := 0
while g(x, y) != 0 do
    y := y + 1
end
return y
```

This program searches for the least $y$ such that:

$$
g(x,y)=0
$$

If such a $y$ exists, the program eventually halts. If no such $y$ exists, the program runs forever.

So the function computed by this program is:

$$
f(x)=\mu y,[g(x,y)=0]
$$

This function is defined exactly when the search succeeds.

### A Simple Example

Define:

$$
f(x)=\mu y,[y^2=x]
$$

This searches for a natural number square root.

If:

$$
x=9
$$

then the search finds:

$$
y=3
$$

so:

$$
f(9)=3
$$

If:

$$
x=10
$$

then no natural number $y$ satisfies:

$$
y^2=10
$$

so the search never stops:

$$
f(10)\uparrow
$$

The function is partial because it is defined only on perfect squares.

### Domain of a Partial Function

The **domain** of a partial function is the set of inputs where it is defined:

$$
\operatorname{dom}(f)={n\in\mathbb{N}:f(n)\downarrow}
$$

For the square-root search above:

$$
\operatorname{dom}(f)={0,1,4,9,16,25,\dots}
$$

For a total function:

$$
\operatorname{dom}(f)=\mathbb{N}
$$

For a partial function:

$$
\operatorname{dom}(f)\subseteq\mathbb{N}
$$

The domain records exactly where the computation halts.

### Total Computable Functions

A function is **total computable** if there is an algorithm that halts on every input and returns the correct output.

Examples include:

$$
f(n)=n+1
$$

$$
f(x,y)=x+y
$$

$$
f(x,y)=x\cdot y
$$

$$
f(n)=n!
$$

These are total because the algorithm always finishes.

Total computability is the cleanest form of algorithmic computation. It says that the procedure always gives an answer.

### Partial Computable Functions

A function is **partial computable** if there is an algorithm that computes it whenever it is defined.

That means:

$$
f(n)\downarrow
\quad \Longrightarrow \quad
\text{the program halts with output } f(n)
$$

and:

$$
f(n)\uparrow
\quad \Longrightarrow \quad
\text{the program may run forever}
$$

The program does not have to announce that no value exists. In general, it cannot.

This is the key point. A partial computable function is allowed to diverge on inputs outside its domain.

### Undefined Is Not an Output

It is important to separate three ideas:

| Situation      | Meaning                                |
| -------------- | -------------------------------------- |
| $f(n)=0$       | the function halts and returns $0$     |
| $f(n)=1$       | the function halts and returns $1$     |
| $f(n)\uparrow$ | the function does not return any value |

Undefinedness is not a special value. It is absence of a value.

This matters because a program that returns $0$ has halted. A program that is undefined has not halted.

### Decision Problems

Many problems ask for a yes/no answer.

A set $A\subseteq\mathbb{N}$ is **decidable** if its characteristic function is total computable:

$$
\chi_A(n)=
\begin{cases}
1 & \text{if } n\in A,\
0 & \text{if } n\notin A.
\end{cases}
$$

The algorithm must halt in both cases.

It must say yes when $n\in A$.

It must say no when $n\notin A$.

### Recognizable Sets

A weaker notion is **recognizability**.

A set $A\subseteq\mathbb{N}$ is recognizable if there is a program that halts exactly on the elements of $A$.

That is:

$$
n\in A
\quad \Longrightarrow \quad
\text{the program halts}
$$

and:

$$
n\notin A
\quad \Longrightarrow \quad
\text{the program may run forever}
$$

So recognizability confirms membership, but it may fail to confirm non-membership.

Every decidable set is recognizable. If a program can always answer yes or no, then it can certainly halt on yes-instances.

But the converse fails. Some sets are recognizable without being decidable.

### Comparison

| Notion                      | Halts on yes? | Halts on no? | Result                 |
| --------------------------- | ------------: | -----------: | ---------------------- |
| decidable set               |           yes |          yes | yes/no answer          |
| recognizable set            |           yes | not required | confirmation only      |
| total computable function   |           yes |          yes | output for every input |
| partial computable function |  when defined | not required | output only on domain  |

The difference between total and partial functions is the mathematical form of the difference between guaranteed termination and possible non-termination.
