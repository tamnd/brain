### Variables and Constants

Programs need names for values.

Instead of repeating raw values everywhere, you store them in variables or constants and reuse them.

Zig separates this into two main concepts:

- `const` for values that do not change
- `var` for values that can change

This distinction is fundamental. Zig makes mutability explicit.

#### Constants with `const`

A constant is a value that cannot be reassigned after it is created.

Example:

```zig
const std = @import("std");

pub fn main() void {
    const age = 20;
    std.debug.print("Age: {}\n", .{age});
}
```

Here:

```zig
const age = 20;
```

means:

- create a name `age`
- store value `20`
- do not allow changes later

If you try this:

```zig
const age = 20;
age = 21;
```

Zig will reject it. Constants are immutable.

#### Why constants matter

Most values in real programs should not change.

Examples:

| Value | Should it change? |
|---|---|
| Configuration settings | No |
| Mathematical constants | No |
| Function parameters | No |
| Temporary loop counters | Sometimes yes |

Zig encourages you to default to `const`. Only use `var` when mutation is necessary.

This reduces accidental changes and makes programs easier to reason about.

#### Mutable values with `var`

A variable can change its value over time.

Example:

```zig
const std = @import("std");

pub fn main() void {
    var counter = 0;
    counter = counter + 1;
    counter = counter + 1;

    std.debug.print("Counter: {}\n", .{counter});
}
```

Output:

```text
Counter: 2
```

Here:

```zig
var counter = 0;
```

means:

- create a mutable value
- allow reassignment

Then:

```zig
counter = counter + 1;
```

updates the stored value.

#### Mutation is explicit

In many languages, mutation is default.

In Zig, it is explicit.

| Declaration | Meaning |
|---|---|
| `const x = 10;` | cannot change |
| `var x = 10;` | can change |

This forces you to think about whether a value should change or not.

#### Type inference

Zig usually infers types automatically.

Example:

```zig
const x = 42;
const y = 3.14;
const name = "zig";
```

These become:

| Value | Inferred type |
|---|---|
| `42` | integer type (default-sized) |
| `3.14` | floating point |
| `"zig"` | string literal |

You can also write types explicitly:

```zig
const x: i32 = 42;
const y: f64 = 3.14;
```

This is useful when you want to control size or precision.

#### Common integer types

Zig does not hide integer sizes.

You choose them explicitly when needed.

| Type | Meaning |
|---|---|
| `i8` | 8-bit signed integer |
| `u8` | 8-bit unsigned integer |
| `i32` | 32-bit signed integer |
| `u32` | 32-bit unsigned integer |
| `i64` | 64-bit signed integer |
| `u64` | 64-bit unsigned integer |

Example:

```zig
const small: u8 = 255;
const big: u64 = 1000000;
```

This is important for systems programming where memory layout matters.

#### Floating point values

Floating point numbers represent decimals.

Common types:

| Type | Meaning |
|---|---|
| `f32` | 32-bit float |
| `f64` | 64-bit float |

Example:

```zig
const pi: f64 = 3.14159;
const temperature: f32 = 36.6;
```

#### Shadowing is not allowed like in some languages

Zig does not allow redeclaring a variable in the same scope with the same name.

This is invalid:

```zig
const x = 10;
const x = 20;
```

Instead, you must use a new scope or a different name.

This avoids confusion about which value is active.

#### Scope rules

Variables exist inside the block where they are declared.

Example:

```zig
const std = @import("std");

pub fn main() void {
    const x = 10;

    {
        const y = 20;
        std.debug.print("{} {}\n", .{ x, y });
    }

    // y is not accessible here
}
```

Inside the inner block, `y` exists.

Outside it, `y` is gone.

This helps control lifetime and visibility.

#### Underscore discard

Sometimes you must define a value but do not use it.

Zig requires you to acknowledge that explicitly.

```zig
const _ = 10;
```

Or:

```zig
const x = 10;
_ = x;
```

This prevents silent unused-variable bugs.

#### Constants in real programs

Most Zig programs use many constants.

Example:

```zig
const std = @import("std");

pub fn main() void {
    const message = "Zig is explicit";
    const times = 3;

    var i: usize = 0;
    while (i < times) : (i += 1) {
        std.debug.print("{s}\n", .{message});
    }
}
```

Here:

- `message` never changes → `const`
- `times` never changes → `const`
- `i` changes in a loop → `var`

This is a typical Zig pattern.

#### Mental model

Think of Zig variables like labeled boxes:

- `const` is a sealed box
- `var` is an open box you can modify

The compiler enforces this strictly.

#### What to remember

Use `const` by default.

Use `var` only when a value must change.

Prefer explicit types when clarity or precision matters.

Understand that mutability is part of the type system, not an afterthought.
