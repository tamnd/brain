### Defining Functions

A function is a named piece of code.

You use a function when you want to give a task a name, run the same task more than once, or separate a larger program into smaller parts.

Here is a small function:

```zig
fn sayHello() void {
    std.debug.print("hello\n", .{});
}
```

This function is named `sayHello`.

It takes no input.

It returns nothing.

The return type is:

```zig
void
```

`void` means “no useful value.”

#### A Complete Example

```zig
const std = @import("std");

fn sayHello() void {
    std.debug.print("hello\n", .{});
}

pub fn main() void {
    sayHello();
}
```

This prints:

```text
hello
```

The program starts at `main`.

Inside `main`, this line calls the function:

```zig
sayHello();
```

Calling a function means running its body.

#### The Basic Shape

The basic shape of a Zig function is:

```zig
fn name(parameters) ReturnType {
    // body
}
```

Example:

```zig
fn addOne(x: u32) u32 {
    return x + 1;
}
```

This function is named `addOne`.

It has one parameter:

```zig
x: u32
```

It returns a `u32`.

The body is:

```zig
return x + 1;
```

#### Function Parameters

A parameter is an input to a function.

```zig
fn greet(name: []const u8) void {
    std.debug.print("hello, {s}\n", .{name});
}
```

The parameter is:

```zig
name: []const u8
```

This means `name` is a read-only slice of bytes. For now, you can think of it as text.

Use the function like this:

```zig
const std = @import("std");

fn greet(name: []const u8) void {
    std.debug.print("hello, {s}\n", .{name});
}

pub fn main() void {
    greet("Zig");
}
```

This prints:

```text
hello, Zig
```

#### Multiple Parameters

A function can take more than one parameter.

```zig
fn add(a: i32, b: i32) i32 {
    return a + b;
}
```

Use it like this:

```zig
const std = @import("std");

fn add(a: i32, b: i32) i32 {
    return a + b;
}

pub fn main() void {
    const result = add(10, 20);
    std.debug.print("{}\n", .{result});
}
```

This prints:

```text
30
```

The call:

```zig
add(10, 20)
```

passes `10` into `a` and `20` into `b`.

#### Return Values

A function returns a value with `return`.

```zig
fn square(x: u32) u32 {
    return x * x;
}
```

This function receives a number and returns its square.

```zig
const std = @import("std");

fn square(x: u32) u32 {
    return x * x;
}

pub fn main() void {
    const value = square(7);
    std.debug.print("{}\n", .{value});
}
```

This prints:

```text
49
```

The returned value is stored here:

```zig
const value = square(7);
```

#### A Function Must Match Its Return Type

If a function says it returns `u32`, it must return a `u32`.

This is valid:

```zig
fn getNumber() u32 {
    return 123;
}
```

This is invalid:

```zig
fn getNumber() u32 {
    return "hello";
}
```

The function promised a number, but it returned text.

Zig rejects this at compile time.

#### A `void` Function Does Not Return a Value

A function with return type `void` does not return useful data.

```zig
fn printLine() void {
    std.debug.print("line\n", .{});
}
```

You can write `return;` to leave early:

```zig
fn printIfPositive(x: i32) void {
    if (x <= 0) {
        return;
    }

    std.debug.print("{}\n", .{x});
}
```

This function prints only positive numbers.

If `x <= 0`, it returns early and prints nothing.

#### Function Names

Function names usually use lower camel case:

```zig
fn readFile() void {}
fn parseNumber() void {}
fn makeBuffer() void {}
```

The first word starts lowercase. Later words start uppercase.

Good function names say what the function does.

Better:

```zig
fn parseNumber(text: []const u8) u32 {
    // ...
}
```

Worse:

```zig
fn doThing(text: []const u8) u32 {
    // ...
}
```

A function name should reduce the reader’s work.

#### Function Order

In Zig, a function can call another function even if it appears later in the file.

```zig
const std = @import("std");

pub fn main() void {
    sayHello();
}

fn sayHello() void {
    std.debug.print("hello\n", .{});
}
```

This works.

You do not need to write a separate function declaration before the definition.

#### Public Functions

The keyword `pub` makes a function public.

```zig
pub fn main() void {
    // ...
}
```

The `main` function is public because the Zig runtime needs to find it as the program entry point.

For functions used only inside one file, leave off `pub`:

```zig
fn helper() void {
    // ...
}
```

Use `pub` when other files should be allowed to access the function.

#### Functions Make Code Smaller

Without functions, code repeats.

```zig
std.debug.print("hello, Ada\n", .{});
std.debug.print("hello, Bob\n", .{});
std.debug.print("hello, Cora\n", .{});
```

A function gives the repeated idea a name:

```zig
fn greet(name: []const u8) void {
    std.debug.print("hello, {s}\n", .{name});
}
```

Then the code becomes:

```zig
greet("Ada");
greet("Bob");
greet("Cora");
```

The program is shorter, but more importantly, the idea is clearer.

#### Functions Hide Details

A function can hide the steps behind a name.

```zig
fn isEven(n: u32) bool {
    return n % 2 == 0;
}
```

Now the caller can write:

```zig
if (isEven(10)) {
    std.debug.print("even\n", .{});
}
```

The caller does not need to think about `% 2 == 0` every time. The name `isEven` carries the meaning.

This is one of the main reasons functions exist.

#### Keep Functions Focused

A good beginner rule is: one function should do one clear task.

Good:

```zig
fn isEven(n: u32) bool {
    return n % 2 == 0;
}
```

Good:

```zig
fn printUserName(name: []const u8) void {
    std.debug.print("{s}\n", .{name});
}
```

Harder to read:

```zig
fn processEverything() void {
    // read input
    // validate input
    // print messages
    // write output
    // update state
}
```

Large functions are sometimes necessary, but small functions are easier to test, read, and change.

#### A Practical Example

Here is a complete program with several small functions:

```zig
const std = @import("std");

fn isAdult(age: u8) bool {
    return age >= 18;
}

fn printStatus(name: []const u8, age: u8) void {
    if (isAdult(age)) {
        std.debug.print("{s} is an adult\n", .{name});
    } else {
        std.debug.print("{s} is a minor\n", .{name});
    }
}

pub fn main() void {
    printStatus("Ada", 20);
    printStatus("Ben", 15);
}
```

This prints:

```text
Ada is an adult
Ben is a minor
```

The function `isAdult` answers one question.

The function `printStatus` prints one message.

The `main` function describes the program flow.

#### The Main Idea

A function gives a name to a piece of code.

The basic form is:

```zig
fn name(parameters) ReturnType {
    // body
}
```

Use parameters for input.

Use `return` for output.

Use `void` when the function returns nothing.

Functions help you split a program into small, named parts. A good function should make the program easier to read, not just shorter.
