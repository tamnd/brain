### `@truncate`

`@truncate` converts an integer to a smaller integer type by keeping only the low bits.

It does not check whether the original value fits.

It deliberately throws away the high bits.

```zig
const small: u8 = @truncate(big);
```

This means:

```text
Keep the lowest 8 bits of big.
Discard the rest.
```

#### A Simple Example

```zig
const std = @import("std");

pub fn main() void {
    const x: u16 = 300;
    const y: u8 = @truncate(x);

    std.debug.print("{}\n", .{y});
}
```

Output:

```text
44
```

Why?

The number `300` is `0x012c` in hexadecimal.

A `u8` can only hold 8 bits. The lowest 8 bits are `0x2c`.

`0x2c` is `44`.

So `@truncate` turns `300` into `44`.

#### `@truncate` Is Not a Safe Numeric Cast

This is the main beginner mistake.

If you want to preserve the numeric value, use `@intCast`.

```zig
const x: u16 = 200;
const y: u8 = @intCast(x);
```

This means:

```text
Convert x to u8, but only if the value fits.
```

But this:

```zig
const x: u16 = 300;
const y: u8 = @truncate(x);
```

means:

```text
Keep only the low 8 bits.
```

The result is not `300`. It cannot be. A `u8` has no room for that value.

#### Destination Type Comes from Context

Like `@intCast`, `@truncate` gets the destination type from context.

```zig
const y: u8 = @truncate(x);
```

The destination type is `u8`.

You can also write:

```zig
const y = @as(u8, @truncate(x));
```

This is useful when the destination type is not otherwise obvious.

#### When `@truncate` Is Correct

Use `@truncate` when you intentionally want low bits.

Common cases include:

```text
hashing
checksums
binary protocols
bit masks
random number generators
wrapping arithmetic
hardware registers
emulators
cryptographic primitives
```

For example, suppose a file format stores only the low 16 bits of a value:

```zig
fn low16(x: u32) u16 {
    return @truncate(x);
}
```

That is a correct use of `@truncate`.

The function name should make the behavior clear.

#### Example: Extracting Low Bits

```zig
const std = @import("std");

fn lowByte(x: u32) u8 {
    return @truncate(x);
}

pub fn main() void {
    const x: u32 = 0x12345678;
    const b = lowByte(x);

    std.debug.print("0x{x}\n", .{b});
}
```

Output:

```text
0x78
```

The low byte of `0x12345678` is `0x78`.

#### `@truncate` and Bit Masks

This code:

```zig
const y: u8 = @truncate(x);
```

is similar in meaning to:

```zig
const y: u8 = @intCast(x & 0xff);
```

Both keep the lowest 8 bits.

But `@truncate` says the intent directly.

It tells the reader: “I know this loses high bits.”

#### Signed Integers

`@truncate` can also be used with signed integers, but be careful.

The operation is still about bits.

Example:

```zig
const x: i16 = -1;
const y: u8 = @truncate(x);
```

The bit pattern for `-1` in two's complement has all bits set. The low 8 bits are `0xff`, so `y` becomes `255`.

This can be useful in low-level code, but it is rarely what you want in ordinary arithmetic.

For normal signed-to-unsigned conversion, use `@intCast` after checking the range.

#### `@truncate` vs Wrapping Arithmetic

`@truncate` only changes the width of an integer.

It does not perform addition, subtraction, or multiplication.

Wrapping arithmetic uses operators such as `+%`, `-%`, and `*%`.

Example:

```zig
const a: u8 = 250;
const b: u8 = a +% 10;
```

The result wraps around.

`@truncate` is different:

```zig
const x: u16 = 260;
const y: u8 = @truncate(x);
```

This keeps the low 8 bits of an already existing value.

Both are low-bit operations, but they happen at different stages.

#### Common Mistake

This is usually wrong:

```zig
fn shrink(x: u16) u8 {
    return @truncate(x);
}
```

If the function means “convert to `u8` only when possible,” use this:

```zig
fn shrink(x: u16) !u8 {
    if (x > std.math.maxInt(u8)) return error.TooLarge;
    return @intCast(x);
}
```

`@truncate` should not hide data loss. It should express intentional data loss.

#### How to Read `@truncate`

When you see:

```zig
const y: Small = @truncate(x);
```

read it as:

```text
Take the low bits of x that fit in Small.
```

Then ask:

```text
Is losing the high bits intentional?
Would @intCast be safer?
Should the code check the range first?
```

#### Key Idea

`@truncate` narrows an integer by discarding high bits.

It preserves low bits, not numeric meaning.

Use it only when losing high bits is the intended behavior.
