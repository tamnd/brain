### `if` Expressions

An `if` expression chooses between pieces of code.

It asks a boolean question. If the answer is `true`, one branch runs. If the answer is `false`, another branch may run.

```zig
const std = @import("std");

pub fn main() void {
    const age = 20;

    if (age >= 18) {
        std.debug.print("adult\n", .{});
    } else {
        std.debug.print("minor\n", .{});
    }
}
```

Output:

```text
adult
```

The condition is:

```zig
age >= 18
```

This produces a `bool`. In this case, it produces `true`, so the first branch runs.

#### Conditions Must Be Booleans

Zig is strict about conditions.

This is valid:

```zig
const ready = true;

if (ready) {
    // runs when ready is true
}
```

This is invalid:

```zig
if (1) {
    // error
}
```

Zig does not treat `1` as true or `0` as false. The condition must be a real `bool`.

This avoids a common source of bugs from C-like languages.

#### Basic `if`

The simplest form has only one branch.

```zig
if (condition) {
    // code runs only when condition is true
}
```

Example:

```zig
const std = @import("std");

pub fn main() void {
    const debug = true;

    if (debug) {
        std.debug.print("debug mode enabled\n", .{});
    }
}
```

Output:

```text
debug mode enabled
```

If `debug` were `false`, the block would be skipped.

#### `if` with `else`

Use `else` when you want an alternative branch.

```zig
if (condition) {
    // runs when condition is true
} else {
    // runs when condition is false
}
```

Example:

```zig
const std = @import("std");

pub fn main() void {
    const count = 0;

    if (count == 0) {
        std.debug.print("empty\n", .{});
    } else {
        std.debug.print("not empty\n", .{});
    }
}
```

Output:

```text
empty
```

Exactly one branch runs.

#### `else if`

Use `else if` when you have several cases.

```zig
const std = @import("std");

pub fn main() void {
    const score = 82;

    if (score >= 90) {
        std.debug.print("A\n", .{});
    } else if (score >= 80) {
        std.debug.print("B\n", .{});
    } else if (score >= 70) {
        std.debug.print("C\n", .{});
    } else {
        std.debug.print("D\n", .{});
    }
}
```

Output:

```text
B
```

Zig checks the branches from top to bottom.

The first true condition wins.

Here, `score >= 90` is false. Then `score >= 80` is true, so Zig runs that branch and skips the rest.

#### `if` Is an Expression

In Zig, `if` can produce a value.

```zig
const label = if (age >= 18) "adult" else "minor";
```

This means:

```text
if age is at least 18, label is "adult"
otherwise, label is "minor"
```

Complete example:

```zig
const std = @import("std");

pub fn main() void {
    const age = 17;

    const label = if (age >= 18) "adult" else "minor";

    std.debug.print("{s}\n", .{label});
}
```

Output:

```text
minor
```

This is useful when you want to choose a value directly.

#### Both Branches Must Fit the Same Result Type

When `if` returns a value, both branches must produce compatible types.

This is valid:

```zig
const value = if (enabled) 10 else 20;
```

Both branches produce integers.

This is invalid:

```zig
const value = if (enabled) 10 else "no";
```

One branch produces an integer. The other produces a string. Zig does not silently choose a vague common type.

Write code that returns one clear kind of value.

#### `if` Without `else` Does Not Produce a Useful Value

If you use `if` as a value, you usually need `else`.

This works:

```zig
const x = if (ok) 1 else 0;
```

This does not work as a value:

```zig
const x = if (ok) 1;
```

If `ok` is false, what value should `x` have? Zig will not guess.

#### Blocks Can Return Values

A branch can contain a block. If you want the block to produce a value, use `break` with a block label.

For many beginner cases, a simpler style is enough:

```zig
const result = if (score >= 50) "pass" else "fail";
```

But Zig also allows larger branches:

```zig
const std = @import("std");

pub fn main() void {
    const score = 85;

    const message = if (score >= 50) blk: {
        std.debug.print("checking passing score\n", .{});
        break :blk "pass";
    } else blk: {
        std.debug.print("checking failing score\n", .{});
        break :blk "fail";
    };

    std.debug.print("{s}\n", .{message});
}
```

Output:

```text
checking passing score
pass
```

The label `blk` gives the block a name. `break :blk "pass"` means the block produces the value `"pass"`.

You do not need this pattern constantly, but it is useful when each branch needs several statements before producing a result.

#### Nested `if`

An `if` can appear inside another `if`.

```zig
const std = @import("std");

pub fn main() void {
    const logged_in = true;
    const is_admin = false;

    if (logged_in) {
        if (is_admin) {
            std.debug.print("admin page\n", .{});
        } else {
            std.debug.print("user page\n", .{});
        }
    } else {
        std.debug.print("please log in\n", .{});
    }
}
```

Output:

```text
user page
```

Nested `if` is sometimes useful, but too much nesting makes code harder to read. When possible, combine simple conditions or return early from functions.

#### Combining Conditions

Use `and`, `or`, and `!` to combine boolean expressions.

```zig
const can_enter = age >= 18 and has_ticket;
```

Example:

```zig
const std = @import("std");

pub fn main() void {
    const age = 21;
    const has_ticket = true;
    const banned = false;

    if (age >= 18 and has_ticket and !banned) {
        std.debug.print("entry allowed\n", .{});
    } else {
        std.debug.print("entry denied\n", .{});
    }
}
```

Output:

```text
entry allowed
```

Read the condition as:

```text
age is at least 18
and has_ticket is true
and banned is false
```

#### Short-Circuiting

`and` and `or` short-circuit.

For `and`, if the left side is false, Zig does not need to evaluate the right side.

```zig
if (index < items.len and items[index] == 10) {
    // safe
}
```

The array access happens only after the bounds check succeeds.

For `or`, if the left side is true, Zig does not need to evaluate the right side.

```zig
if (is_admin or is_owner) {
    // access allowed
}
```

Short-circuiting is useful for safety and efficiency.

#### `if` with Optionals

An optional value may contain a value or `null`.

```zig
const maybe_number: ?i32 = 42;
```

You can use `if` to unwrap it:

```zig
const std = @import("std");

pub fn main() void {
    const maybe_number: ?i32 = 42;

    if (maybe_number) |number| {
        std.debug.print("number = {}\n", .{number});
    } else {
        std.debug.print("no number\n", .{});
    }
}
```

Output:

```text
number = 42
```

The syntax:

```zig
if (maybe_number) |number| {
```

means: if `maybe_number` contains a value, put that value into the name `number` and run the block.

If the optional is `null`, the `else` branch runs.

```zig
const maybe_number: ?i32 = null;
```

would print:

```text
no number
```

This is one of the most common uses of `if` in Zig.

#### `if` with Error Unions

An error union contains either a normal value or an error.

You can handle it with `if`.

```zig
const std = @import("std");

fn parseSmallNumber(text: []const u8) !u8 {
    return std.fmt.parseInt(u8, text, 10);
}

pub fn main() void {
    const result = parseSmallNumber("123");

    if (result) |value| {
        std.debug.print("value = {}\n", .{value});
    } else |err| {
        std.debug.print("error: {}\n", .{err});
    }
}
```

Output:

```text
value = 123
```

The syntax:

```zig
if (result) |value| {
```

means the operation succeeded.

The syntax:

```zig
} else |err| {
```

means the operation failed, and `err` is the error value.

You will usually use `try` for simple error propagation, but this `if` form is useful when you want to handle success and failure directly.

#### Common Beginner Mistakes

Do not write numeric conditions:

```zig
if (count) {
    // invalid
}
```

Write the comparison:

```zig
if (count != 0) {
    // valid
}
```

Do not confuse assignment and equality:

```zig
if (x = 10) {
    // invalid
}
```

Use equality comparison:

```zig
if (x == 10) {
    // valid
}
```

Do not return unrelated types from an `if` expression:

```zig
const value = if (ok) 1 else "no";
```

Choose one result type:

```zig
const value = if (ok) 1 else 0;
```

#### A Complete Example

```zig
const std = @import("std");

fn describeAge(age: u8) []const u8 {
    return if (age < 13)
        "child"
    else if (age < 20)
        "teenager"
    else
        "adult";
}

pub fn main() void {
    const age: u8 = 16;
    const description = describeAge(age);

    std.debug.print("{s}\n", .{description});
}
```

Output:

```text
teenager
```

The function returns a string chosen by an `if` expression.

```zig
return if (age < 13)
    "child"
else if (age < 20)
    "teenager"
else
    "adult";
```

This is compact, but still readable because every branch returns the same kind of value.

#### The Main Idea

`if` chooses code based on a boolean condition.

In Zig, `if` is stricter than in many languages. Conditions must be booleans. Numeric values are not automatically true or false. When `if` produces a value, the branches must produce compatible types.

This strictness makes control flow easier to trust.
