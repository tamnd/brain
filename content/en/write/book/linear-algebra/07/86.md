# Chapter 86. Conditioning and Stability

Conditioning and stability are two separate ideas. Conditioning describes the sensitivity of the mathematical problem. Stability describes the behavior of the algorithm used to solve the problem.

A problem is well-conditioned when small changes in the input produce small changes in the output. A problem is ill-conditioned when small changes in the input can produce large changes in the output. An algorithm is stable when the effects of floating point error remain controlled. These ideas are central in numerical linear algebra because floating point computation always introduces small perturbations.

## 86.1 Problems and Algorithms

A numerical problem may be viewed as a function

$$
f : X \to Y,
$$

where \(X\) is a space of input data and \(Y\) is a space of outputs.

For example, solving a linear system defines the problem

$$
(A,b) \mapsto x,
$$

where

$$
Ax=b.
$$

The problem itself is mathematical. It exists independently of any particular method.

An algorithm is a procedure for computing an approximate value of \(f(x)\). Different algorithms may solve the same problem with different accuracy, speed, and robustness.

Thus:

| Concept | Describes |
|---|---|
| Conditioning | The problem |
| Stability | The algorithm |
| Accuracy | The computed result |

A well-conditioned problem can be solved accurately by a stable algorithm. An ill-conditioned problem may produce inaccurate answers even when the algorithm is stable.

## 86.2 Perturbations

A perturbation is a small change in the input.

If the exact input is \(x\), a perturbed input is

$$
x+\Delta x.
$$

The output changes from

$$
f(x)
$$

to

$$
f(x+\Delta x).
$$

The corresponding output perturbation is

$$
\Delta f = f(x+\Delta x)-f(x).
$$

Conditioning asks how large \(\Delta f\) can be compared with \(\Delta x\).

If

$$
\|\Delta f\|
$$

is small whenever

$$
\|\Delta x\|
$$

is small, then the problem is well-conditioned near \(x\). If a very small \(\Delta x\) can produce a large \(\Delta f\), the problem is ill-conditioned near \(x\).

## 86.3 Absolute Condition Number

The absolute condition number measures absolute sensitivity.

For a scalar or vector problem \(f\), the local absolute condition number at \(x\) is

$$
\kappa_{\mathrm{abs}}(x) =
\lim_{\epsilon \to 0}
\sup_{\|\Delta x\|\le \epsilon}
\frac{\|f(x+\Delta x)-f(x)\|}{\|\Delta x\|}.
$$

When \(f\) is differentiable, this is the operator norm of the derivative:

$$
\kappa_{\mathrm{abs}}(x) =
\|Df(x)\|.
$$

This number tells us how much the output changes per unit change in the input.

Absolute conditioning is natural when the scale of the output matters directly.

## 86.4 Relative Condition Number

Relative conditioning compares fractional changes.

The relative condition number is approximately

$$
\kappa_{\mathrm{rel}}(x) =
\lim_{\epsilon \to 0}
\sup_{\|\Delta x\|\le \epsilon \|x\|}
\frac{
\|f(x+\Delta x)-f(x)\| / \|f(x)\|
}{
\|\Delta x\| / \|x\|
}.
$$

Equivalently,

$$
\frac{\text{relative output change}}{\text{relative input change}}
\approx
\kappa_{\mathrm{rel}}(x).
$$

A relative condition number near \(1\) means that relative input errors are not greatly magnified. A large relative condition number means that input errors may be strongly amplified.

Relative conditioning is especially important in floating point arithmetic, because rounding errors are often naturally modeled as small relative perturbations.

## 86.5 A Simple Scalar Example

Consider the function

$$
f(x)=\sqrt{x}.
$$

For \(x>0\),

$$
f'(x)=\frac{1}{2\sqrt{x}}.
$$

Thus the absolute condition number is

$$
\kappa_{\mathrm{abs}}(x) =
\frac{1}{2\sqrt{x}}.
$$

This becomes large near \(x=0\). Therefore the square root problem is absolutely ill-conditioned near zero.

The relative condition number is

$$
\kappa_{\mathrm{rel}}(x) =
\left|
\frac{x f'(x)}{f(x)}
\right| =
\left|
\frac{x}{\sqrt{x}}
\cdot
\frac{1}{2\sqrt{x}}
\right| =
\frac{1}{2}.
$$

Thus square root is relatively well-conditioned for positive \(x\). This example shows that absolute and relative conditioning can lead to different interpretations.

## 86.6 Conditioning of Linear Systems

Consider the linear system

$$
Ax=b,
$$

where \(A\) is nonsingular. The solution is

$$
x=A^{-1}b.
$$

If \(b\) is perturbed to

$$
b+\Delta b,
$$

then the new solution is

$$
x+\Delta x =
A^{-1}(b+\Delta b).
$$

Therefore

$$
\Delta x=A^{-1}\Delta b.
$$

Taking norms,

$$
\|\Delta x\|
\le
\|A^{-1}\|\|\Delta b\|.
$$

Since

$$
b=Ax,
$$

we also have

$$
\|b\|
\le
\|A\|\|x\|.
$$

Combining these inequalities gives

$$
\frac{\|\Delta x\|}{\|x\|}
\le
\|A\|\|A^{-1}\|
\frac{\|\Delta b\|}{\|b\|}.
$$

The factor

$$
\kappa(A)=\|A\|\|A^{-1}\|
$$

is the condition number of the matrix.

## 86.7 Matrix Condition Number

For a nonsingular matrix \(A\), the condition number in a chosen norm is

$$
\kappa(A)=\|A\|\|A^{-1}\|.
$$

In the Euclidean norm, the condition number is

$$
\kappa_2(A) =
\frac{\sigma_{\max}(A)}{\sigma_{\min}(A)}.
$$

Here \(\sigma_{\max}(A)\) is the largest singular value and \(\sigma_{\min}(A)\) is the smallest singular value.

If \(A\) is singular, then

$$
\sigma_{\min}(A)=0,
$$

so

$$
\kappa_2(A)=\infty.
$$

A large condition number means that the matrix is close to singular in the relevant norm.

## 86.8 Geometric Meaning

The condition number describes how unevenly a matrix stretches space.

Under the map

$$
x \mapsto Ax,
$$

the unit sphere is transformed into an ellipsoid. The largest semiaxis length is

$$
\sigma_{\max}(A),
$$

and the smallest semiaxis length is

$$
\sigma_{\min}(A).
$$

Thus

$$
\kappa_2(A) =
\frac{\text{largest stretch}}{\text{smallest stretch}}.
$$

If all directions are stretched by nearly the same amount, the condition number is close to \(1\). If some directions are stretched much less than others, the condition number is large.

A large condition number means that the inverse transformation must strongly amplify some directions. This is why solving \(Ax=b\) becomes sensitive.

## 86.9 Examples

The identity matrix has condition number

$$
\kappa_2(I)=1.
$$

It stretches every direction equally.

The diagonal matrix

$$
A =
\begin{bmatrix}
1 & 0 \\
0 & \epsilon
\end{bmatrix}
$$

has singular values

$$
1
\quad \text{and} \quad
\epsilon,
$$

when \(0<\epsilon<1\). Therefore

$$
\kappa_2(A)=\frac{1}{\epsilon}.
$$

If

$$
\epsilon = 10^{-8},
$$

then

$$
\kappa_2(A)=10^8.
$$

A relative error of size \(10^{-16}\) in the data may then produce a relative error of size roughly \(10^{-8}\) in the solution.

## 86.10 Residual and Error

For a computed solution \(\widehat{x}\), the residual is

$$
r=b-A\widehat{x}.
$$

The residual measures how well \(\widehat{x}\) satisfies the equations.

The forward error is

$$
\widehat{x}-x.
$$

These quantities are related by

$$
A(x-\widehat{x}) = r.
$$

Hence

$$
x-\widehat{x}=A^{-1}r.
$$

Taking norms,

$$
\|x-\widehat{x}\|
\le
\|A^{-1}\|\|r\|.
$$

A small residual implies a small error only when \(\|A^{-1}\|\) is not too large. Therefore a small residual alone does not guarantee an accurate solution for an ill-conditioned system.

## 86.11 Relative Residual Bound

A useful normalized residual is

$$
\frac{\|r\|}{\|A\|\|\widehat{x}\|+\|b\|}.
$$

This quantity measures the size of the equation error relative to the scale of the computation.

If this value is small, then \(\widehat{x}\) solves a nearby linear system. This is a backward error interpretation.

Forward error also depends on conditioning. A standard heuristic is

$$
\frac{\|\widehat{x}-x\|}{\|x\|}
\lesssim
\kappa(A)
\frac{\|r\|}{\|b\|}.
$$

Thus a small residual must be multiplied by the condition number to estimate possible solution error.

## 86.12 Stability

Stability describes whether an algorithm controls the effects of rounding error.

An unstable algorithm may turn small floating point errors into large output errors, even for a well-conditioned problem.

A stable algorithm produces the exact answer to a problem close to the original one. This is the idea of backward stability.

For a problem

$$
y=f(x),
$$

an algorithm produces a computed answer \(\widehat{y}\). The algorithm is backward stable if there exists a small perturbation \(\Delta x\) such that

$$
\widehat{y}=f(x+\Delta x).
$$

The computed answer is then exact for nearby data.

## 86.13 Backward Stability for Linear Systems

For a linear system, backward stability means that the computed solution \(\widehat{x}\) satisfies

$$
(A+\Delta A)\widehat{x}=b+\Delta b,
$$

where

$$
\frac{\|\Delta A\|}{\|A\|}
$$

and

$$
\frac{\|\Delta b\|}{\|b\|}
$$

are small.

This does not say that \(\widehat{x}\) is close to the exact solution \(x\). It says that the algorithm did not solve a wildly different problem.

The final forward accuracy is governed by both stability and conditioning.

## 86.14 Forward Stability

An algorithm is forward stable if the computed answer is close to the exact answer:

$$
\frac{\|\widehat{y}-y\|}{\|y\|}
$$

is small.

Forward stability is a direct accuracy statement. It is often harder to prove than backward stability.

Backward stability plus good conditioning usually implies forward accuracy. If the problem is ill-conditioned, backward stability may still allow a large forward error.

This distinction is one of the main organizing principles of numerical analysis.

## 86.15 A Stable Algorithm on an Ill-Conditioned Problem

Suppose an algorithm for solving \(Ax=b\) is backward stable. It computes \(\widehat{x}\) satisfying

$$
(A+\Delta A)\widehat{x}=b,
$$

with

$$
\frac{\|\Delta A\|}{\|A\|}\approx 10^{-16}.
$$

If

$$
\kappa(A)\approx 10^{12},
$$

then the relative forward error may be on the order of

$$
10^{-4}.
$$

The algorithm behaved well. The problem was sensitive.

This is why one should not blame an algorithm merely because the final answer has few correct digits. The condition number may be the limiting factor.

## 86.16 An Unstable Algorithm on a Well-Conditioned Problem

The opposite can also happen.

A problem may be well-conditioned, but an algorithm may compute it in a way that loses information through cancellation, overflow, or unnecessary squaring of condition numbers.

For example, solving least squares through the normal equations

$$
A^TAx=A^Tb
$$

can be less stable than solving the same problem by QR or SVD. The reason is that the condition number of \(A^TA\) in the Euclidean norm is

$$
\kappa_2(A^TA)=\kappa_2(A)^2.
$$

Thus the normal equations square the condition number. This can turn a moderately conditioned least squares problem into a much more sensitive linear system.

## 86.17 Conditioning of Least Squares

For the least squares problem

$$
\min_x \|Ax-b\|_2,
$$

conditioning depends on the geometry of the column space of \(A\), the size of the residual, and the singular values of \(A\).

If \(A\) has full column rank, then the solution can be characterized by

$$
A^TAx=A^Tb.
$$

However, this equation should be interpreted carefully. It is a mathematical characterization, not always the best numerical method.

QR and SVD methods are preferred when stability matters because they use orthogonal transformations and avoid explicitly forming \(A^TA\).

## 86.18 Orthogonal Transformations

Orthogonal transformations are central to stable numerical algorithms.

If \(Q\) is orthogonal, then

$$
\|Qx\|_2=\|x\|_2.
$$

For matrices,

$$
\|QA\|_F=\|A\|_F,
\qquad
\|AQ\|_F=\|A\|_F.
$$

Thus orthogonal transformations do not magnify errors by scaling. This makes Householder QR, Givens rotations, Schur decomposition, and SVD especially reliable.

Many stable algorithms are built from orthogonal transformations for this reason.

## 86.19 Pivoting and Stability

Gaussian elimination without pivoting can be unstable or fail when a pivot is zero or very small.

Partial pivoting chooses a pivot by selecting a large entry in the current column. This controls the multipliers

$$
\ell_{ik}=\frac{a_{ik}}{a_{kk}}.
$$

After partial pivoting,

$$
|\ell_{ik}| \le 1
$$

for the eliminated entries in that column.

This helps control roundoff growth. Gaussian elimination with partial pivoting is the standard dense direct method for general linear systems because it is reliable in ordinary practice.

## 86.20 Growth Factor

Even with pivoting, entries in the intermediate matrices may grow.

The growth factor measures how large intermediate entries become relative to the original matrix. A large growth factor can lead to loss of accuracy.

For Gaussian elimination, the growth factor is roughly

$$
\rho =
\frac{
\max \text{ entry produced during elimination}
}{
\max \text{ entry of } A
}.
$$

Backward error bounds for elimination with partial pivoting include this factor. In typical problems, \(\rho\) is moderate. In specially constructed examples, it can be large.

Thus pivoting improves stability but does not provide an absolute guarantee of small growth in every possible matrix.

## 86.21 Stability of QR

Householder QR is backward stable for least squares and related problems.

The algorithm applies a sequence of orthogonal reflectors:

$$
A \mapsto Q^TA.
$$

Each reflector preserves the Euclidean norm. Rounding errors occur, but they can be interpreted as small perturbations to the original matrix.

This is why QR is preferred over normal equations when accuracy matters. It avoids forming \(A^TA\) and avoids squaring the condition number.

## 86.22 Stability of Cholesky

Cholesky decomposition is stable for symmetric positive definite systems when the matrix is well-scaled and positive definite.

For

$$
A=LL^T,
$$

the algorithm exploits symmetry and positivity. It avoids pivoting in the standard SPD case because positive definiteness guarantees positive pivots in exact arithmetic.

However, if the matrix is nearly singular or not truly positive definite, Cholesky may fail or produce unreliable results. In such cases, the failure is often diagnostic: the input matrix may not satisfy the required assumptions.

## 86.23 Stability of SVD

The singular value decomposition is among the most reliable decompositions in numerical linear algebra.

It uses orthogonal or unitary transformations and exposes small singular values directly. This makes it effective for rank-deficient and ill-conditioned problems.

SVD is often more expensive than LU or QR, but it provides stronger diagnostic information:

| Quantity | Revealed by SVD |
|---|---|
| Rank | Number of nonzero singular values |
| Numerical rank | Number of singular values above tolerance |
| Conditioning | Ratio \(\sigma_{\max}/\sigma_{\min}\) |
| Null space | Right singular vectors for small singular values |
| Range | Left singular vectors for large singular values |

When accuracy and structural information matter more than speed, SVD is often the safest choice.

## 86.24 Accuracy as a Product of Two Effects

A useful rule is:

$$
\text{forward error}
\approx
\text{condition number}
\times
\text{backward error}.
$$

This is not an exact identity, but it is a good mental model.

It says that final accuracy depends on both the problem and the algorithm.

| Case | Expected result |
|---|---|
| Well-conditioned problem, stable algorithm | Accurate answer |
| Ill-conditioned problem, stable algorithm | Possibly inaccurate answer |
| Well-conditioned problem, unstable algorithm | Possibly inaccurate answer |
| Ill-conditioned problem, unstable algorithm | Unreliable answer |

This table is one of the simplest ways to organize numerical judgment.

## 86.25 Estimating Condition Numbers

Computing an exact condition number may be expensive. In practice, numerical libraries often estimate condition numbers.

For linear systems, one may estimate

$$
\|A^{-1}\|
$$

without forming \(A^{-1}\) explicitly. Since

$$
\kappa(A)=\|A\|\|A^{-1}\|,
$$

this gives an estimate of \(\kappa(A)\).

For smaller dense problems, the SVD gives a direct condition number in the Euclidean norm:

$$
\kappa_2(A) =
\frac{\sigma_1}{\sigma_n}.
$$

For large sparse problems, iterative estimates are often used.

## 86.26 Scaling and Conditioning

Scaling may improve numerical behavior, but it does not always improve the intrinsic conditioning of the original mathematical problem.

Row and column scaling can reduce the spread of matrix entries. This may improve pivoting behavior and reduce roundoff effects.

For example, if one row of a matrix has entries near

$$
10^{12}
$$

and another has entries near

$$
10^{-12},
$$

a solver may suffer from poor scaling. Rescaling rows can put equations on comparable numerical scales.

However, scaling should be interpreted carefully. It changes the representation of the problem. It may help the algorithm even when the underlying problem remains sensitive.

## 86.27 Componentwise Conditioning

Normwise conditioning measures perturbations using vector or matrix norms. Sometimes componentwise conditioning is more appropriate.

A componentwise perturbation satisfies

$$
|\Delta a_{ij}| \le \epsilon |a_{ij}|
$$

for each entry.

This is different from requiring only

$$
\|\Delta A\| \le \epsilon \|A\|.
$$

Componentwise analysis is useful when data entries have independent relative errors, or when entries vary greatly in scale.

Some algorithms may be normwise stable but not componentwise stable. Others may have stronger componentwise guarantees.

## 86.28 Mixed Precision and Stability

Mixed precision algorithms use lower precision for some operations and higher precision for correction or accumulation.

Their stability depends on whether low-precision errors can be controlled.

For example, iterative refinement may factor \(A\) in single precision but compute residuals in double precision. If \(A\) is not too ill-conditioned, the refinement can recover double precision accuracy.

The convergence condition depends on the product of the condition number and the lower-precision unit roundoff. If this product is too large, refinement may stagnate or fail.

## 86.29 Practical Diagnostic Questions

When a numerical result looks suspicious, ask these questions.

| Question | Purpose |
|---|---|
| Is the residual small? | Checks whether equations are nearly satisfied |
| Is the condition number large? | Checks sensitivity of the problem |
| Was a stable algorithm used? | Checks method reliability |
| Were the inputs badly scaled? | Checks avoidable numerical stress |
| Are there small singular values? | Checks rank deficiency or near singularity |
| Did cancellation occur? | Checks loss of significant digits |

These questions separate mathematical sensitivity from algorithmic failure.

## 86.30 Summary

Conditioning and stability are distinct.

Conditioning belongs to the problem. It measures how much the exact solution changes when the input data are perturbed. For a nonsingular matrix,

$$
\kappa(A)=\|A\|\|A^{-1}\|.
$$

In the Euclidean norm,

$$
\kappa_2(A) =
\frac{\sigma_{\max}(A)}{\sigma_{\min}(A)}.
$$

Stability belongs to the algorithm. A stable algorithm controls the effect of rounding error. A backward stable algorithm computes the exact solution to a nearby problem.

The practical rule is:

$$
\text{accuracy depends on conditioning and stability}.
$$

A stable algorithm cannot remove the sensitivity of an ill-conditioned problem. A well-conditioned problem can still be ruined by an unstable algorithm. Numerical linear algebra studies both sides because reliable computation requires both a well-posed problem and a stable method.
