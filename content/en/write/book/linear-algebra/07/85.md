# Chapter 85. Floating Point Arithmetic

Floating point arithmetic is the arithmetic used by computers to approximate real-number computation. It is essential in numerical linear algebra because matrices often contain real-valued measurements, coefficients, probabilities, errors, or physical quantities.

A computer cannot store most real numbers exactly. It stores a finite approximation. Floating point arithmetic defines how such approximations are represented, rounded, combined, and compared. Modern hardware commonly follows the IEEE 754 standard, which defines binary and decimal floating point formats, rounding rules, infinities, signed zeros, subnormal numbers, and NaN values.

## 85.1 Exact Arithmetic and Machine Arithmetic

In exact arithmetic, the real numbers form a continuous system. Between any two distinct real numbers, there is another real number. The set is infinite and uncountable.

A computer uses finite memory. Therefore it can represent only finitely many numbers in a fixed format. The arithmetic performed by the machine is not real arithmetic. It is rounded arithmetic.

For example, the real number

$$
\frac{1}{3}
$$

has no finite decimal expansion, and it also has no finite binary expansion. A floating point system stores only an approximation.

This distinction matters. A theorem proved over the real numbers may remain algebraically correct, but a numerical algorithm that implements it must also account for rounding error.

## 85.2 Floating Point Representation

A normalized floating point number has the form

$$
x = \pm m \times \beta^e.
$$

Here:

| Symbol | Meaning |
|---|---|
| \(\pm\) | Sign |
| \(m\) | Significand or mantissa |
| \(\beta\) | Base |
| \(e\) | Exponent |

Most modern scientific computation uses base

$$
\beta = 2.
$$

In binary floating point, a number is stored approximately as

$$
x = \pm (1.b_1b_2\cdots b_p)_2 \times 2^e.
$$

The number \(p\) is the precision. It counts the number of significant binary digits stored in the significand.

The exponent controls scale. The significand controls precision.

## 85.3 Floating Point as Scientific Notation

Floating point arithmetic is a finite version of scientific notation.

In decimal scientific notation, one writes

$$
6.022 \times 10^{23}
$$

or

$$
1.602 \times 10^{-19}.
$$

The coefficient stores significant digits. The exponent stores magnitude.

Floating point works the same way, usually in base \(2\). This allows one format to represent both very large and very small numbers.

The price is that the spacing between representable numbers is not uniform. Floating point numbers are dense near zero and sparse far from zero.

## 85.4 Binary64

The most common floating point format in numerical linear algebra is IEEE 754 binary64, often called double precision.

It uses 64 bits divided into three fields:

| Field | Bits |
|---|---:|
| Sign | 1 |
| Exponent | 11 |
| Fraction | 52 |

Because normalized binary numbers have an implicit leading \(1\), binary64 has 53 bits of precision in the significand.

This corresponds to about 16 decimal digits of precision.

Thus a double precision number typically stores around 15 to 16 meaningful decimal digits, depending on the number and the operation.

## 85.5 Binary32

IEEE 754 binary32, often called single precision, uses 32 bits:

| Field | Bits |
|---|---:|
| Sign | 1 |
| Exponent | 8 |
| Fraction | 23 |

With the implicit leading bit, it has 24 bits of significand precision. This corresponds to about 7 decimal digits of precision.

Single precision is faster and uses less memory than double precision on many systems. It is widely used in graphics, machine learning, and large-scale approximate computation.

Double precision remains standard in many parts of scientific computing because it gives substantially better accuracy.

## 85.6 Rounding

Most real numbers cannot be represented exactly in a finite floating point format. When an exact result is not representable, it must be rounded to a nearby representable number.

A common rounding mode is round to nearest, ties to even. This means that the exact value is rounded to the nearest floating point number. If it lies exactly halfway between two representable numbers, the one whose final significand bit is even is chosen. IEEE 754 defines rounding rules as part of the standard.

We often write the result of a floating point operation as

$$
\operatorname{fl}(x \circ y),
$$

where \(\circ\) is an arithmetic operation such as \(+\), \(-\), \(\times\), or \(/\). The notation means: compute the exact result, then round it to the floating point system.

## 85.7 Unit Roundoff

The unit roundoff measures the relative error introduced by one correctly rounded operation.

For a floating point system with precision \(p\) in base \(2\), the unit roundoff is approximately

$$
u = 2^{-p}.
$$

For binary64,

$$
u = 2^{-53}.
$$

Numerically,

$$
u \approx 1.11 \times 10^{-16}.
$$

For binary32,

$$
u = 2^{-24}.
$$

Numerically,

$$
u \approx 5.96 \times 10^{-8}.
$$

A standard model for one rounded operation is

$$
\operatorname{fl}(x \circ y) =
(x \circ y)(1+\delta),
\qquad
|\delta| \le u,
$$

provided no overflow, underflow, or exceptional case occurs.

This compact formula is the starting point of much of numerical error analysis.

## 85.8 Machine Epsilon

Machine epsilon is often defined as the distance from \(1\) to the next larger floating point number.

For binary64, this value is

$$
2^{-52}.
$$

For binary32, it is

$$
2^{-23}.
$$

Some authors use machine epsilon and unit roundoff interchangeably. Others define unit roundoff as one half of machine epsilon for round-to-nearest arithmetic.

To avoid ambiguity, this book uses:

| Quantity | Meaning |
|---|---|
| Machine epsilon | Gap between \(1\) and the next larger number |
| Unit roundoff | Maximum relative error from one rounded operation |

Under round-to-nearest,

$$
u = \frac{\varepsilon_{\text{mach}}}{2}.
$$

## 85.9 Nonrepresentable Numbers

Many ordinary decimal numbers are not exactly representable in binary floating point.

For example, the decimal number

$$
0.1
$$

has a repeating expansion in base \(2\). Therefore it cannot be stored exactly in a finite binary format. Floating point systems store the nearest representable approximation. This is why decimal-looking computations can produce small unexpected errors.

For example, a computer may evaluate

$$
0.1 + 0.2
$$

as a number slightly different from

$$
0.3.
$$

This does not mean the arithmetic is broken. It means the numbers being added are already binary approximations of the decimal values.

## 85.10 Relative Error and Absolute Error

Let \(x\) be the exact value and \(\widehat{x}\) be the computed value.

The absolute error is

$$
|\widehat{x}-x|.
$$

The relative error is

$$
\frac{|\widehat{x}-x|}{|x|},
\qquad x \ne 0.
$$

Absolute error measures error in the original units. Relative error measures error compared with the size of the quantity.

In numerical linear algebra, relative error is often more meaningful. An error of \(10^{-6}\) may be enormous when the true value is \(10^{-9}\), but negligible when the true value is \(10^6\).

## 85.11 Floating Point Addition

Floating point addition is more delicate than multiplication.

To add two numbers, the machine first aligns their exponents. If the numbers have very different magnitudes, the smaller number may lose significant digits during alignment.

For example, suppose we use decimal arithmetic with four significant digits. Then

$$
1.000 \times 10^6 + 1
$$

rounds to

$$
1.000 \times 10^6.
$$

The smaller term disappears.

This phenomenon is called absorption. It occurs when a small number is added to a much larger number and falls below the resolution of the larger number.

## 85.12 Cancellation

Cancellation occurs when nearly equal numbers are subtracted.

For example,

$$
1.000001 - 1.000000 = 0.000001.
$$

If the two input numbers were already rounded, their leading digits cancel and the remaining digits may contain mostly error.

Cancellation does not create error by itself in exact arithmetic. The problem is that subtraction reveals previous rounding errors.

This is one of the most important hazards in numerical computation.

## 85.13 Example of Cancellation

Suppose

$$
x = 1.234567
$$

and

$$
y = 1.234566.
$$

The exact difference is

$$
x-y = 0.000001.
$$

Now suppose both numbers are stored to six significant decimal digits:

$$
\widehat{x}=1.23457,
\qquad
\widehat{y}=1.23457.
$$

Then

$$
\widehat{x}-\widehat{y}=0.
$$

A small but meaningful difference has disappeared.

Algorithms should avoid subtracting nearly equal quantities when possible, especially if the result is later used in division or normalization.

## 85.14 Overflow

Overflow occurs when the exact result is too large in magnitude to be represented.

For example, if a system can represent numbers only up to \(M\), then any result with magnitude greater than \(M\) overflows.

In IEEE-style arithmetic, overflow may produce infinity:

$$
+\infty
\quad \text{or} \quad
-\infty.
$$

Overflow is usually a sign that a computation has exceeded the dynamic range of the format. It can occur in matrix norms, exponentials, products, determinants, and poorly scaled algorithms.

## 85.15 Underflow

Underflow occurs when a nonzero exact result is too small in magnitude to be represented as a normalized floating point number.

IEEE 754 includes subnormal numbers, which allow gradual underflow near zero. Subnormal numbers fill part of the gap between zero and the smallest normalized number.

Underflow can still cause loss of relative precision. Very small quantities may be rounded to zero, or represented with fewer significant digits than normal.

In numerical linear algebra, underflow often appears in scaling-sensitive computations, such as determinant products, norms, or iterative algorithms with very small residuals.

## 85.16 Infinities and NaNs

IEEE 754 includes special values:

| Value | Meaning |
|---|---|
| \(+\infty\) | Positive overflow or exact infinite result |
| \(-\infty\) | Negative overflow or exact infinite result |
| NaN | Not a Number |

A NaN may arise from undefined operations such as

$$
0/0
$$

or

$$
\sqrt{-1}
$$

in real arithmetic.

These values help computations proceed in a defined way after exceptional events. They also make it possible to detect invalid operations.

In numerical software, a NaN is often a symptom of an earlier failure: division by zero, invalid square root, overflow followed by invalid arithmetic, or uninitialized data.

## 85.17 Signed Zero

IEEE floating point arithmetic distinguishes

$$
+0
$$

and

$$
-0.
$$

They compare equal in ordinary equality tests, but they can behave differently in certain operations, such as division:

$$
\frac{1}{+0}=+\infty,
\qquad
\frac{1}{-0}=-\infty.
$$

Signed zero helps preserve limiting behavior in some numerical functions and complex arithmetic. It is usually invisible in elementary linear algebra, but it can appear in low-level numerical code.

## 85.18 Floating Point Matrix Computation

Matrix algorithms perform many arithmetic operations. Even if each operation has tiny relative error, the accumulated effect may become significant.

For example, matrix multiplication computes

$$
c_{ij} =
\sum_{k=1}^{n} a_{ik}b_{kj}.
$$

This dot product involves \(n\) multiplications and \(n-1\) additions. Each operation may round. The final result contains accumulated rounding error.

Thus the accuracy of matrix algorithms depends not only on the mathematical formula, but also on the order of operations, scaling, conditioning, and stability of the algorithm.

## 85.19 Dot Products

A dot product has the form

$$
x^Ty =
\sum_{i=1}^{n} x_i y_i.
$$

Floating point dot products can lose accuracy when terms have mixed signs or widely varying magnitudes.

If many positive and negative terms nearly cancel, the final result may be small compared with the intermediate sums. In that case, relative error can be large.

Improved summation methods, such as pairwise summation or compensated summation, can reduce error. These methods are especially useful when summing long vectors.

## 85.20 Summation Order

Floating point addition is generally not associative.

In exact arithmetic,

$$
(a+b)+c = a+(b+c).
$$

In floating point arithmetic, the two sides can differ because rounding occurs after each addition.

Thus

$$
\operatorname{fl}(\operatorname{fl}(a+b)+c)
$$

may differ from

$$
\operatorname{fl}(a+\operatorname{fl}(b+c)).
$$

This affects reductions, parallel computation, matrix multiplication, and iterative solvers. A parallel program may produce slightly different results from a sequential program because it sums terms in a different order.

## 85.21 Conditioning

Conditioning is a property of the mathematical problem, not of the algorithm.

A problem is well-conditioned if small changes in the input cause small changes in the output. A problem is ill-conditioned if small input changes can cause large output changes.

For a linear system

$$
Ax=b,
$$

the condition number of \(A\) controls sensitivity. In the Euclidean norm,

$$
\kappa_2(A)=\frac{\sigma_{\max}(A)}{\sigma_{\min}(A)}.
$$

If \(\kappa_2(A)\) is large, then small perturbations in \(A\) or \(b\) may produce large changes in \(x\).

Floating point arithmetic always introduces small perturbations. Therefore ill-conditioned problems are inherently difficult.

## 85.22 Stability

Stability is a property of the algorithm.

A stable algorithm gives a computed answer that is close to the exact answer for a nearby problem. This is called backward stability.

For solving

$$
Ax=b,
$$

a backward stable algorithm computes \(\widehat{x}\) such that

$$
(A+\Delta A)\widehat{x}=b+\Delta b,
$$

where \(\Delta A\) and \(\Delta b\) are small.

This is a powerful notion. It says that the algorithm did not introduce a large artificial error. Any remaining error is explained by the conditioning of the problem.

## 85.23 Forward Error and Backward Error

Forward error measures the distance between the computed answer and the exact answer:

$$
\frac{\|\widehat{x}-x\|}{\|x\|}.
$$

Backward error asks how much the input must be changed to make the computed answer exact.

For numerical linear algebra, backward error is often easier to analyze and more meaningful. A small backward error means the computed result solves a nearby problem exactly.

The relationship is:

| Quantity | Depends mainly on |
|---|---|
| Forward error | Conditioning and stability |
| Backward error | Algorithmic stability |
| Condition number | Problem sensitivity |

A stable algorithm applied to a well-conditioned problem gives an accurate result. A stable algorithm applied to an ill-conditioned problem may still give a poor forward result.

## 85.24 Scaling

Scaling can improve numerical behavior.

For example, if a matrix contains entries ranging from

$$
10^{-100}
$$

to

$$
10^{100},
$$

then arithmetic may suffer from overflow, underflow, or severe loss of precision.

Scaling rows or columns can reduce this range. In linear systems, one often tries to balance the magnitudes of coefficients before factorization.

Scaling does not change the exact mathematical problem if done carefully. It changes the numerical representation of the problem to make it safer for finite precision arithmetic.

## 85.25 Pivoting and Floating Point Arithmetic

Pivoting in Gaussian elimination is partly a numerical device.

If a pivot is very small, then multipliers such as

$$
\ell_{ik}=\frac{a_{ik}}{a_{kk}}
$$

may be very large. Large multipliers can amplify rounding errors.

Partial pivoting chooses a large available pivot in the current column. This controls the size of multipliers and usually improves numerical stability.

Thus PLU decomposition is not merely a way to avoid zero pivots. It is also a practical response to floating point arithmetic.

## 85.26 Residuals

For a computed solution \(\widehat{x}\) of

$$
Ax=b,
$$

the residual is

$$
r=b-A\widehat{x}.
$$

A small residual means that \(\widehat{x}\) nearly satisfies the original equations.

However, a small residual does not always imply a small forward error. If \(A\) is ill-conditioned, a vector \(\widehat{x}\) can have a small residual but still be far from the exact solution.

The residual must be interpreted together with conditioning.

## 85.27 Mixed Precision

Mixed precision algorithms use more than one floating point format.

For example, a matrix factorization may be computed in single precision, while correction steps are performed in double precision. This can be faster than using double precision throughout, especially on hardware where lower precision is much faster.

Mixed precision is useful when the problem is not too ill-conditioned and when iterative refinement can correct the lower-precision factorization.

The key question is whether the lower-precision errors are small enough for the higher-precision correction process to converge.

## 85.28 Iterative Refinement

Iterative refinement improves a computed solution by repeatedly solving for corrections.

Start with an approximate solution

$$
\widehat{x}.
$$

Compute the residual

$$
r=b-A\widehat{x}.
$$

Then solve approximately

$$
Ad=r
$$

for a correction \(d\), and update

$$
\widehat{x} \leftarrow \widehat{x}+d.
$$

If the residual is computed accurately and the system is not too ill-conditioned, this process can improve the solution significantly.

Iterative refinement is a standard technique for combining speed and accuracy.

## 85.29 Practical Rules

Floating point arithmetic should be treated as approximate arithmetic.

Useful rules include:

| Rule | Reason |
|---|---|
| Avoid subtracting nearly equal numbers | Reduces cancellation |
| Avoid unnecessary matrix inverses | Solves are usually more stable |
| Scale problems when magnitudes vary widely | Reduces overflow and underflow risk |
| Prefer orthogonal transformations when possible | They preserve norms |
| Use residuals to check computed solutions | Detects equation-level error |
| Consider condition numbers | Separates problem sensitivity from algorithm error |

These rules are not cosmetic. They are part of the discipline of numerical linear algebra.

## 85.30 Summary

Floating point arithmetic is the finite-precision arithmetic used by computers to approximate real arithmetic. A floating point number has a sign, significand, base, and exponent:

$$
x = \pm m \times \beta^e.
$$

Since most real numbers are not exactly representable, operations are rounded. This produces small local errors that may accumulate through matrix algorithms.

The main numerical hazards are rounding error, absorption, cancellation, overflow, underflow, and ill-conditioning. The main tools for controlling them are scaling, pivoting, stable algorithms, orthogonal transformations, residual checks, and error analysis.

Numerical linear algebra is built on this fact: the exact mathematical problem and the finite precision computation are related, but not identical. A good algorithm respects both.
