Deep learning is a branch of machine learning that studies models built from many layers of learned computation. These models are usually called neural networks. A neural network receives data as input, transforms it through a sequence of mathematical operations, and produces an output such as a class label, a probability distribution, a generated image, or the next token in a sentence.

The word *deep* refers to the use of many layers. Each layer computes a new representation of the data. Early layers usually represent simple patterns. Later layers combine these patterns into more abstract ones. In an image model, early layers may respond to edges and colors. Later layers may respond to textures, object parts, and complete objects. In a language model, early computations may represent tokens and local syntax. Later computations may represent semantic relations, discourse structure, or task-specific information.

A deep learning model can be viewed as a parameterized function

$$
f_\theta : X \to Y,
$$

where \(X\) is the input space, \(Y\) is the output space, and \(\theta\) denotes the parameters learned from data. In PyTorch, these parameters are tensors stored inside modules such as `nn.Linear`, `nn.Conv2d`, and `nn.TransformerEncoderLayer`.

For example, a simple neural network classifier maps an input vector to class scores:

$$
x \in \mathbb{R}^d
\quad \longmapsto \quad
f_\theta(x) \in \mathbb{R}^K.
$$

Here \(d\) is the number of input features and \(K\) is the number of classes. The output vector contains one score for each class. These scores are often called *logits*. A loss function compares the logits with the correct answer and produces a scalar loss. Training adjusts the parameters so that this loss becomes smaller.

### Learning from Examples

Deep learning systems learn from examples rather than from hand-written rules. Suppose we want to build a classifier that recognizes handwritten digits. A traditional rule-based system might try to describe the shape of each digit using explicit geometric rules. Such rules are hard to write and brittle in practice. People write digits in many different ways.

A deep learning system takes a different approach. It is given many examples:

$$
(x_1, y_1), (x_2, y_2), \ldots, (x_N, y_N),
$$

where each \(x_i\) is an image and each \(y_i\) is the correct digit label. The model starts with parameters that are usually random. It makes predictions, measures its errors, and updates its parameters. Over many iterations, it learns internal representations that help solve the task.

The central training loop has four steps:

1. Compute predictions from inputs.
2. Compute a loss from predictions and targets.
3. Compute gradients of the loss with respect to parameters.
4. Update the parameters using an optimizer.

In PyTorch, this pattern appears constantly:

```python
import torch
from torch import nn

model = nn.Linear(784, 10)
loss_fn = nn.CrossEntropyLoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

x = torch.randn(32, 784)
y = torch.randint(0, 10, (32,))

logits = model(x)
loss = loss_fn(logits, y)

optimizer.zero_grad()
loss.backward()
optimizer.step()
```

This small example contains the main mechanism of deep learning. The model computes outputs. The loss measures error. Backpropagation computes gradients. The optimizer changes the parameters.

### Representations

The main strength of deep learning is representation learning. A representation is a transformed version of the data that makes a task easier.

Raw data is often hard to use directly. An image is a grid of pixel values. A sentence is a sequence of token IDs. An audio recording is a waveform. These raw forms contain useful information, but the useful structure is distributed across many entries.

A neural network builds representations layer by layer:

$$
x \to h_1 \to h_2 \to \cdots \to h_L \to y.
$$

The input \(x\) is transformed into hidden representation \(h_1\), then into \(h_2\), and so on until the output \(y\). Each hidden representation is learned for the task.

For a feedforward network, this may look like

$$
h_1 = \sigma(W_1x + b_1),
$$

$$
h_2 = \sigma(W_2h_1 + b_2),
$$

$$
y = W_3h_2 + b_3.
$$

The matrices \(W_1, W_2, W_3\) and vectors \(b_1, b_2, b_3\) are learned. The function \(\sigma\) is an activation function, such as ReLU or GELU. Without nonlinear activation functions, a stack of linear layers would collapse into a single linear transformation.

In PyTorch:

```python
model = nn.Sequential(
    nn.Linear(784, 256),
    nn.ReLU(),
    nn.Linear(256, 128),
    nn.ReLU(),
    nn.Linear(128, 10),
)
```

This model maps a 784-dimensional input to 10 output logits through two hidden representations.

### Parameters and Training

A neural network contains parameters. These are the numbers adjusted during training. In a linear layer

$$
y = Wx + b,
$$

the matrix \(W\) and vector \(b\) are parameters. In a convolutional layer, the convolution kernels are parameters. In a transformer, the attention projections, feedforward layers, embeddings, and normalization parameters are all learned tensors.

Training means finding parameters that make the model perform well on the training data and, more importantly, on new data. This is done by minimizing a loss function.

For a dataset of \(N\) examples, the empirical risk is often written as

$$
\frac{1}{N}\sum_{i=1}^{N} \ell(f_\theta(x_i), y_i),
$$

where \(\ell\) is the loss for one example. The goal is to find parameters \(\theta\) that make this quantity small.

In practice, deep learning rarely computes this sum over the whole dataset at every step. It uses mini-batches. A mini-batch is a small subset of examples, such as 32, 128, or 1024 examples. Mini-batch training makes learning computationally efficient and introduces noise that can help optimization.

### Why Depth Helps

Depth allows a model to build complex functions by composing simpler functions. Each layer performs a relatively simple transformation. The full network can represent a much richer transformation because these layers are composed.

A shallow model must express a complicated mapping in one or two large steps. A deep model can express it through many intermediate representations. This often matches the structure of real data. Images have edges, textures, parts, and objects. Language has characters, subwords, words, phrases, clauses, and meaning. Audio has waveforms, frequencies, phonemes, words, and speaker patterns.

Depth also allows parameter sharing and hierarchy. Convolutional networks share filters across spatial locations. Recurrent networks reuse computations across time. Transformers apply attention and feedforward blocks repeatedly across sequence positions. These design choices allow large models to learn from large datasets efficiently.

### Deep Learning as Differentiable Programming

Most deep learning models are differentiable programs. They are programs made of operations for which derivatives can be computed. This matters because training uses gradients.

A gradient tells us how a small change in a parameter changes the loss. If increasing a parameter increases the loss, the optimizer can decrease that parameter. If increasing a parameter decreases the loss, the optimizer can increase it.

PyTorch records operations on tensors when gradient tracking is enabled. It then uses automatic differentiation to compute gradients.

```python
x = torch.tensor(2.0, requires_grad=True)

y = x * x + 3 * x + 1
y.backward()

print(x.grad)
```

The derivative of

$$
y = x^2 + 3x + 1
$$

is

$$
\frac{dy}{dx} = 2x + 3.
$$

At \(x = 2\), the derivative is \(7\). PyTorch computes this automatically.

The same mechanism scales to networks with millions or billions of parameters. The mathematics is the chain rule. The implementation is automatic differentiation.

### What Deep Learning Models Learn

A trained model learns numerical structure. It does not store human-readable rules in the usual sense. Its knowledge is distributed across many parameters.

A vision model may learn filters, shapes, textures, and object-level representations. A language model may learn grammar, factual associations, style, reasoning patterns, and statistical regularities in text. A speech model may learn acoustic features and phonetic structure. A recommender system may learn user and item embeddings.

These learned representations are useful because they compress regularities in data. The model does not memorize every possible input. It learns patterns that allow it to respond to new inputs.

This distinction is central. A model that performs well only on its training examples has memorized. A model that performs well on unseen examples has generalized. Deep learning aims at generalization.

### Deep Learning Tasks

Deep learning is used for many kinds of tasks.

In classification, the model assigns an input to one of several classes. Image classification and sentiment classification are examples.

In regression, the model predicts a numerical value. Price prediction and temperature forecasting are examples.

In sequence modeling, the model predicts or transforms sequences. Language modeling, machine translation, speech recognition, and time-series forecasting are examples.

In generation, the model creates new data. Text generation, image generation, music generation, and video generation are examples.

In reinforcement learning, an agent learns to act in an environment. The model may learn policies, value functions, or world models.

In representation learning, the model learns embeddings that can be used for retrieval, clustering, transfer learning, or downstream prediction.

### PyTorch’s Role

PyTorch is a framework for building, training, and deploying deep learning models. It provides tensors, automatic differentiation, neural network modules, optimizers, data loading tools, distributed training support, and compilation tools.

The core PyTorch workflow is direct:

```python
model = ...
data = ...
loss_fn = ...
optimizer = ...

for x, y in data:
    logits = model(x)
    loss = loss_fn(logits, y)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

This loop is simple, but it scales. The same structure applies to a small linear classifier, a convolutional network, a transformer, or a diffusion model. Larger systems add data pipelines, checkpointing, logging, distributed computation, mixed precision, and deployment infrastructure, but the conceptual core remains the same.

### Limits of Deep Learning

Deep learning is powerful, but it has limits. Models can require large datasets, large compute budgets, and careful engineering. They can fail under distribution shift. They can produce overconfident wrong answers. They can encode biases in training data. They can be difficult to interpret. Large generative models can produce fluent outputs that are factually incorrect.

These limitations do not make deep learning unusable. They define the engineering and scientific problems that must be handled. A serious deep learning practitioner must understand not only how to train models, but also how to evaluate them, stress-test them, explain their failures, and deploy them responsibly.

### Core Idea

Deep learning studies parameterized differentiable models trained from data. A model maps inputs to outputs. A loss function measures error. Automatic differentiation computes gradients. An optimizer updates parameters. Repeated over many examples, this process produces learned representations that can solve classification, prediction, generation, control, and reasoning tasks.

The rest of this book develops this idea in detail, starting from tensors and automatic differentiation, then moving to neural network modules, optimization, computer vision, sequence models, transformers, generative models, and large-scale training.
