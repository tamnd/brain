## 1.2 The PyTorch Ecosystem

PyTorch is a deep learning platform built around tensors, automatic differentiation, and composable neural network modules. It is used for research, production training, inference, computer vision, natural language processing, audio, reinforcement learning, graph learning, and large-scale model development.

The central idea is simple: PyTorch lets you write numerical programs using tensors, then automatically computes gradients through those programs. This makes it suitable for deep learning, where training requires repeated gradient computation over many parameters.

A typical PyTorch project uses several layers of the ecosystem:

| Layer | Main tools | Purpose |
|---|---|---|
| Tensor computation | `torch.Tensor` | Store and compute numerical arrays |
| Automatic differentiation | `torch.autograd` | Compute gradients |
| Neural networks | `torch.nn` | Define model layers and modules |
| Optimization | `torch.optim` | Update parameters |
| Data loading | `torch.utils.data` | Build datasets and mini-batches |
| Vision | `torchvision` | Image datasets, transforms, models |
| Text | TorchText, Hugging Face | Tokenization and NLP pipelines |
| Audio | `torchaudio` | Waveforms, spectrograms, audio models |
| Graphs | PyTorch Geometric | Graph neural networks |
| Distributed training | `torch.distributed` | Multi-GPU and multi-node training |
| Compilation | `torch.compile` | Performance optimization |
| Deployment | TorchScript, ONNX, TorchServe | Model export and serving |

### 1.2.1 Tensors

The base object in PyTorch is the tensor. A tensor is a multidimensional array with a shape, data type, device, and optional gradient history.

```python
import torch

x = torch.randn(32, 128)
print(x.shape)
print(x.dtype)
print(x.device)
```

A tensor can live on the CPU or on an accelerator such as a GPU.

```python
device = "cuda" if torch.cuda.is_available() else "cpu"

x = torch.randn(32, 128, device=device)
```

Most deep learning computation consists of tensor operations: matrix multiplication, elementwise arithmetic, reductions, reshaping, indexing, convolution, attention, and normalization.

### 1.2.2 Automatic Differentiation

PyTorch includes an automatic differentiation engine called `autograd`. When tensors have `requires_grad=True`, PyTorch records operations on them and builds a computation graph.

```python
x = torch.tensor(2.0, requires_grad=True)

y = x * x + 3 * x + 1
y.backward()

print(x.grad)
```

The call to `backward()` computes the derivative of `y` with respect to `x`. For neural networks, the same mechanism computes gradients for every trainable parameter.

This is the foundation of training. You define the forward computation. PyTorch computes the backward computation.

### 1.2.3 Neural Network Modules

The `torch.nn` package provides building blocks for neural networks. A model is usually written as a subclass of `nn.Module`.

```python
from torch import nn

class MLP(nn.Module):
    def __init__(self, in_features, hidden_features, out_features):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_features, hidden_features),
            nn.ReLU(),
            nn.Linear(hidden_features, out_features),
        )

    def forward(self, x):
        return self.net(x)
```

An `nn.Module` tracks parameters automatically. This makes it easy to pass parameters to an optimizer.

```python
model = MLP(784, 256, 10)

for name, param in model.named_parameters():
    print(name, param.shape)
```

Common modules include:

| Module | Use |
|---|---|
| `nn.Linear` | Fully connected layers |
| `nn.Conv2d` | Image convolutions |
| `nn.Embedding` | Token and ID embeddings |
| `nn.LayerNorm` | Feature normalization |
| `nn.Dropout` | Regularization |
| `nn.MultiheadAttention` | Attention layers |
| `nn.Transformer` | Transformer building blocks |

### 1.2.4 Optimizers

The `torch.optim` package contains optimization algorithms. Optimizers update model parameters using gradients.

```python
optimizer = torch.optim.AdamW(model.parameters(), lr=3e-4)
```

A training step usually follows this structure:

```python
logits = model(x)
loss = loss_fn(logits, y)

optimizer.zero_grad()
loss.backward()
optimizer.step()
```

The call to `zero_grad()` clears old gradients. The call to `backward()` computes new gradients. The call to `step()` updates the parameters.

Common optimizers include:

| Optimizer | Typical use |
|---|---|
| `SGD` | Simple baseline, often for vision models |
| `SGD` with momentum | Stronger classical optimizer |
| `Adam` | General-purpose adaptive optimizer |
| `AdamW` | Standard choice for transformers |
| `RMSprop` | Sometimes used for recurrent or RL models |

### 1.2.5 Data Loading

Deep learning models train on mini-batches. PyTorch provides `Dataset` and `DataLoader` abstractions for this.

A `Dataset` represents indexed examples. A `DataLoader` turns a dataset into batches.

```python
from torch.utils.data import Dataset, DataLoader

class ToyDataset(Dataset):
    def __init__(self):
        self.x = torch.randn(1000, 20)
        self.y = torch.randint(0, 2, (1000,))

    def __len__(self):
        return len(self.x)

    def __getitem__(self, idx):
        return self.x[idx], self.y[idx]

dataset = ToyDataset()
loader = DataLoader(dataset, batch_size=32, shuffle=True)
```

The training loop then consumes batches:

```python
for x, y in loader:
    logits = model(x)
    loss = loss_fn(logits, y)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

This separation keeps data handling independent from model code.

### 1.2.6 TorchVision

`torchvision` provides tools for computer vision. It includes image datasets, image transforms, pretrained models, detection models, and utilities.

Common uses include loading standard datasets:

```python
from torchvision import datasets, transforms

transform = transforms.ToTensor()

train_set = datasets.MNIST(
    root="data",
    train=True,
    download=True,
    transform=transform,
)
```

It also provides pretrained models:

```python
from torchvision import models

model = models.resnet50(weights=models.ResNet50_Weights.DEFAULT)
```

These models are useful for transfer learning. Instead of training a large vision model from scratch, you can start from pretrained weights and fine-tune the model on a smaller dataset.

### 1.2.7 TorchAudio

`torchaudio` provides tools for audio processing. It supports waveform loading, transformations, feature extraction, and audio model components.

Audio models often convert raw waveforms into time-frequency representations such as spectrograms or mel spectrograms.

```python
import torchaudio

waveform, sample_rate = torchaudio.load("speech.wav")

mel = torchaudio.transforms.MelSpectrogram(
    sample_rate=sample_rate
)(waveform)
```

Audio deep learning tasks include speech recognition, speaker identification, music classification, speech enhancement, and text-to-speech.

### 1.2.8 NLP and Transformers

PyTorch is widely used for natural language processing. Many modern NLP systems use PyTorch through the Hugging Face ecosystem.

A language model workflow usually includes tokenization, embedding lookup, transformer layers, and output projection.

```python
from transformers import AutoTokenizer, AutoModelForSequenceClassification

tokenizer = AutoTokenizer.from_pretrained("distilbert-base-uncased")
model = AutoModelForSequenceClassification.from_pretrained(
    "distilbert-base-uncased"
)
```

The Hugging Face libraries build on PyTorch tensors and modules. They provide pretrained models, tokenizers, training utilities, datasets, and model hubs.

This ecosystem is especially important for transformers, large language models, retrieval-augmented generation, fine-tuning, and evaluation.

### 1.2.9 PyTorch Geometric

Graph neural networks operate on nodes, edges, and graph-level structures. PyTorch Geometric is a common library for graph deep learning.

A graph dataset typically contains node features, edge indices, labels, and graph metadata.

Graph neural networks are used in molecular modeling, recommendation systems, knowledge graphs, social networks, traffic modeling, and scientific machine learning.

The core idea is message passing. Nodes update their representations by aggregating information from neighboring nodes.

### 1.2.10 Distributed Training

Large models and large datasets often require multiple GPUs or multiple machines. PyTorch provides distributed training tools through `torch.distributed`.

The most common interface is Distributed Data Parallel, usually called DDP. Each GPU holds a model replica. Each replica processes a different mini-batch. Gradients are synchronized across replicas before the optimizer updates parameters.

The basic idea:

```text
GPU 0: batch A -> forward -> backward
GPU 1: batch B -> forward -> backward
GPU 2: batch C -> forward -> backward
GPU 3: batch D -> forward -> backward

Synchronize gradients
Update parameters
```

For larger models, PyTorch also supports sharding and parallelism strategies such as Fully Sharded Data Parallel, tensor parallelism, and pipeline parallelism through the broader ecosystem.

### 1.2.11 Compilation and Performance

PyTorch began as an eager execution framework. Eager execution means operations run immediately, which makes debugging easy.

Modern PyTorch also supports compilation:

```python
compiled_model = torch.compile(model)
```

Compilation can optimize the model graph, reduce Python overhead, fuse operations, and improve runtime performance. It is especially useful when the model structure is stable and repeated many times.

Performance tuning also involves:

| Technique | Purpose |
|---|---|
| Mixed precision | Reduce memory and increase throughput |
| Gradient checkpointing | Trade compute for memory |
| Operator fusion | Reduce memory traffic |
| Efficient data loading | Keep accelerators busy |
| Profiling | Locate bottlenecks |
| Quantization | Reduce inference cost |

### 1.2.12 Deployment

After training, a model must often be served. Deployment may mean batch inference, real-time APIs, mobile inference, edge inference, or embedded inference.

PyTorch models can be deployed in several ways:

| Method | Use |
|---|---|
| Plain PyTorch | Python-based inference services |
| TorchScript | Serialized executable model format |
| ONNX | Interoperability with other runtimes |
| TorchServe | Model serving infrastructure |
| Quantized models | Lower memory and faster inference |
| Mobile runtimes | On-device inference |

A simple inference step disables gradient tracking:

```python
model.eval()

with torch.no_grad():
    logits = model(x)
    predictions = logits.argmax(dim=-1)
```

The call to `eval()` switches layers such as dropout and batch normalization into inference mode. The context `torch.no_grad()` avoids storing gradient history and reduces memory use.

### 1.2.13 The Shape of a PyTorch Project

A practical PyTorch project usually separates model code, data code, training code, evaluation code, and configuration.

A simple structure:

```text
project/
  data.py
  model.py
  train.py
  eval.py
  config.py
  checkpoints/
  runs/
```

A larger project may use a package layout:

```text
project/
  pyproject.toml
  src/
    myproject/
      data/
      models/
      training/
      evaluation/
      inference/
  scripts/
  configs/
  tests/
```

This separation matters because deep learning projects grow quickly. Clear structure makes experiments reproducible and code reusable.

### 1.2.14 The Core PyTorch Mental Model

PyTorch can be understood through five concepts.

First, tensors store data and parameters.

Second, modules define differentiable computations.

Third, losses turn model outputs into scalar training objectives.

Fourth, autograd computes gradients.

Fifth, optimizers update parameters.

A minimal supervised training system combines these pieces:

```python
import torch
from torch import nn
from torch.utils.data import DataLoader, TensorDataset

X = torch.randn(1000, 20)
y = torch.randint(0, 3, (1000,))

loader = DataLoader(TensorDataset(X, y), batch_size=32, shuffle=True)

model = nn.Sequential(
    nn.Linear(20, 64),
    nn.ReLU(),
    nn.Linear(64, 3),
)

loss_fn = nn.CrossEntropyLoss()
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3)

for epoch in range(5):
    for xb, yb in loader:
        logits = model(xb)
        loss = loss_fn(logits, yb)

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
```

This pattern will reappear throughout the book. Later chapters add convolution, attention, generation, distributed training, compilation, and deployment. The underlying structure remains the same: tensors flow forward, gradients flow backward, and parameters are updated.
