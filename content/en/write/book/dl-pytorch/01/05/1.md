## 5.1 Linear Regression

Linear regression is the simplest useful model for supervised learning. It predicts a real-valued target from one or more input features by forming a weighted sum of those features.

Suppose each training example consists of an input vector

$$
x \in \mathbb{R}^d
$$

and a target value

$$
y \in \mathbb{R}.
$$

The goal is to learn a function

$$
f:\mathbb{R}^d \to \mathbb{R}
$$

that predicts \(y\) from \(x\). In linear regression, this function has the form

$$
\hat{y} = w^\top x + b.
$$

Here \(w \in \mathbb{R}^d\) is the weight vector, \(b \in \mathbb{R}\) is the bias, and \(\hat{y}\) is the model prediction.

The weight vector controls how much each input feature contributes to the prediction. The bias shifts the prediction by a constant amount. If \(x_j\) increases by one unit while all other features stay fixed, the prediction changes by \(w_j\).

### 5.1.1 The One-Dimensional Case

The simplest case has one input feature. Then the model is

$$
\hat{y} = wx + b.
$$

This is the equation of a line. The parameter \(w\) is the slope, and \(b\) is the intercept.

For example, if we want to predict the price of a house from its size, \(x\) may represent square meters and \(y\) may represent price. A positive value of \(w\) means larger houses tend to have higher predicted prices. The model assumes that each additional square meter changes the predicted price by a constant amount.

This assumption is restrictive, but useful. Linear regression often serves as the first model to try because it is simple, interpretable, and mathematically clear.

### 5.1.2 Multiple Input Features

Most real problems have more than one feature. A house price model may use size, number of rooms, distance to the city center, building age, and neighborhood features.

We collect these features into a vector:

$$
x =
\begin{bmatrix}
x_1 \\
x_2 \\
\vdots \\
x_d
\end{bmatrix}.
$$

The model becomes

$$
\hat{y} =
w_1x_1 + w_2x_2 + \cdots + w_dx_d + b.
$$

Equivalently,

$$
\hat{y} = w^\top x + b.
$$

This form is compact and important. Neural networks use this operation repeatedly. A linear layer in a neural network is a collection of weighted sums followed by optional nonlinear transformations.

### 5.1.3 Training Data

Let the training dataset contain \(n\) examples:

$$
(x_1, y_1), (x_2, y_2), \ldots, (x_n, y_n).
$$

Each \(x_i \in \mathbb{R}^d\), and each \(y_i \in \mathbb{R}\).

We can stack the inputs into a matrix

$$
X \in \mathbb{R}^{n \times d}.
$$

Each row of \(X\) is one input example. We can also stack the targets into a vector

$$
y \in \mathbb{R}^n.
$$

The model predictions for all examples are

$$
\hat{y} = Xw + b.
$$

Here \(b\) is added to every row. In PyTorch, this happens by broadcasting.

### 5.1.4 Loss Function

A model must have a criterion for measuring error. For linear regression, the usual loss is mean squared error:

$$
L(w,b) =
\frac{1}{n}
\sum_{i=1}^{n}
(\hat{y}_i - y_i)^2.
$$

Since

$$
\hat{y}_i = w^\top x_i + b,
$$

we can write

$$
L(w,b) =
\frac{1}{n}
\sum_{i=1}^{n}
(w^\top x_i + b - y_i)^2.
$$

The squared error penalizes large mistakes more strongly than small mistakes. The mean takes the average over the training set.

Training linear regression means finding parameters \(w\) and \(b\) that make this loss small.

### 5.1.5 Why Squared Error Is Natural

Squared error has several useful properties.

First, it is differentiable. Gradient-based optimization can compute how the loss changes when each parameter changes.

Second, it gives a convex objective for linear regression. A convex objective has no bad local minima. Any local minimum is also a global minimum.

Third, squared error has a probabilistic interpretation. If the target is generated by

$$
y = w^\top x + b + \epsilon,
$$

where the noise term \(\epsilon\) is Gaussian with mean zero, then minimizing squared error is equivalent to maximum likelihood estimation.

Thus linear regression connects three views of learning: geometry, optimization, and probability.

### 5.1.6 Gradients

To train the model, we need the derivatives of the loss with respect to the parameters.

For one example, define the prediction error

$$
e_i = \hat{y}_i - y_i.
$$

The loss for that example is

$$
\ell_i = e_i^2.
$$

The derivative with respect to the prediction is

$$
\frac{\partial \ell_i}{\partial \hat{y}_i} =
2e_i.
$$

Since

$$
\hat{y}_i = w^\top x_i + b,
$$

we have

$$
\frac{\partial \hat{y}_i}{\partial w} =
x_i,
$$

and

$$
\frac{\partial \hat{y}_i}{\partial b} =
1.
$$

Therefore,

$$
\frac{\partial \ell_i}{\partial w} =
2e_i x_i,
$$

and

$$
\frac{\partial \ell_i}{\partial b} =
2e_i.
$$

For the full dataset,

$$
\nabla_w L =
\frac{2}{n}X^\top(Xw + b - y),
$$

and

$$
\nabla_b L =
\frac{2}{n}\sum_{i=1}^n(\hat{y}_i - y_i).
$$

These formulas show the structure of learning. The model compares predictions with targets, computes errors, and adjusts parameters in the direction that reduces those errors.

### 5.1.7 Gradient Descent

Gradient descent updates parameters by moving opposite the gradient:

$$
w \leftarrow w - \eta \nabla_w L,
$$

$$
b \leftarrow b - \eta \nabla_b L.
$$

The scalar \(\eta > 0\) is the learning rate.

A small learning rate makes slow progress. A large learning rate may overshoot the minimum and cause unstable training. Choosing the learning rate is one of the basic practical tasks in deep learning.

For linear regression, the loss surface is shaped like a bowl. Gradient descent moves downhill until it reaches the bottom. For deep neural networks, the loss surface is far more complicated, but the same basic update principle remains.

### 5.1.8 Linear Regression in PyTorch from Scratch

The following code creates synthetic data and trains a linear regression model using PyTorch tensors.

```python
import torch

torch.manual_seed(0)

n = 1000
d = 3

true_w = torch.tensor([2.0, -3.0, 1.5])
true_b = 0.7

X = torch.randn(n, d)
noise = 0.1 * torch.randn(n)

y = X @ true_w + true_b + noise
```

We now initialize the model parameters:

```python
w = torch.randn(d, requires_grad=True)
b = torch.zeros((), requires_grad=True)
```

The argument `requires_grad=True` tells PyTorch to track operations involving these tensors so that gradients can be computed.

A simple training loop is:

```python
lr = 0.05
num_epochs = 100

for epoch in range(num_epochs):
    y_hat = X @ w + b
    loss = ((y_hat - y) ** 2).mean()

    loss.backward()

    with torch.no_grad():
        w -= lr * w.grad
        b -= lr * b.grad

        w.grad.zero_()
        b.grad.zero_()

    if (epoch + 1) % 20 == 0:
        print(f"epoch {epoch + 1}, loss {loss.item():.4f}")
```

Several details are important.

The expression

```python
y_hat = X @ w + b
```

computes the predictions for the whole batch. The expression

```python
loss = ((y_hat - y) ** 2).mean()
```

computes mean squared error. The call

```python
loss.backward()
```

computes gradients by automatic differentiation.

The update is placed inside

```python
with torch.no_grad():
```

because parameter updates should not themselves become part of the computational graph.

Finally, the gradients are reset:

```python
w.grad.zero_()
b.grad.zero_()
```

PyTorch accumulates gradients by default. If we do not clear them, gradients from different iterations will be added together.

### 5.1.9 Linear Regression with `nn.Linear`

PyTorch provides a standard linear layer through `torch.nn.Linear`.

```python
import torch
from torch import nn

model = nn.Linear(in_features=3, out_features=1)
```

This layer represents

$$
\hat{y} = xW^\top + b.
$$

For a batch input \(X \in \mathbb{R}^{B \times 3}\), the output has shape

$$
[B, 1].
$$

Training with a built-in optimizer is shorter:

```python
loss_fn = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.05)

for epoch in range(100):
    y_hat = model(X).squeeze(-1)
    loss = loss_fn(y_hat, y)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

This version performs the same conceptual steps:

1. compute predictions
2. compute loss
3. clear old gradients
4. compute new gradients
5. update parameters

The optimizer manages the update rule. The module manages parameters.

### 5.1.10 Shape Conventions

Shape errors are common in regression models. Suppose

$$
X \in \mathbb{R}^{B \times d}.
$$

If we use

```python
model = nn.Linear(d, 1)
```

then

```python
y_hat = model(X)
```

has shape

```python
[B, 1]
```

If the target tensor has shape

```python
[B]
```

then PyTorch may broadcast during loss computation, which can silently produce incorrect behavior. It is better to make shapes explicit:

```python
y_hat = model(X).squeeze(-1)
loss = loss_fn(y_hat, y)
```

or store targets with shape `[B, 1]`:

```python
y = y.unsqueeze(-1)
loss = loss_fn(model(X), y)
```

A good rule is simple: predictions and targets should have the same shape when passed to the loss function.

### 5.1.11 Interpretation of Parameters

Linear regression is interpretable because each weight has a direct meaning. The parameter \(w_j\) measures the change in prediction for a one-unit increase in feature \(x_j\), holding other features fixed.

This interpretation depends on feature scaling. If one feature is measured in meters and another in millimeters, their weights will have different units. Large or small weights may reflect feature scale rather than feature importance.

For this reason, linear regression often works better when features are standardized:

$$
x_j' = \frac{x_j - \mu_j}{\sigma_j}.
$$

Here \(\mu_j\) is the mean of feature \(j\), and \(\sigma_j\) is its standard deviation.

After standardization, features have comparable scale. Optimization usually becomes easier, and weight magnitudes become more meaningful.

### 5.1.12 Linear Regression as a Neural Network

Linear regression is a neural network with no hidden layers and no nonlinear activation. In PyTorch, this is exactly a single `nn.Linear` layer.

```python
model = nn.Linear(d, 1)
```

Although simple, this model introduces nearly all core ingredients of deep learning:

| Ingredient | Linear regression example |
|---|---|
| Data | Input matrix \(X\), target vector \(y\) |
| Parameters | Weight vector \(w\), bias \(b\) |
| Forward pass | \(\hat{y} = Xw + b\) |
| Loss | Mean squared error |
| Gradients | \(\nabla_w L\), \(\nabla_b L\) |
| Optimizer | Gradient descent or SGD |
| Training loop | Repeat prediction, loss, backward, update |

Deep networks extend this pattern by composing many layers and inserting nonlinear functions between them.

### 5.1.13 Limits of Linear Regression

Linear regression can only represent linear relationships between inputs and output. It cannot directly model curved decision surfaces, complex interactions, or hierarchical features.

For example, if the true relationship is

$$
y = x^2,
$$

a one-dimensional linear model

$$
\hat{y} = wx + b
$$

cannot fit it well over a wide range.

One solution is to manually add nonlinear features, such as \(x^2\), \(x^3\), or feature products. Another solution is to use neural networks. A multilayer network learns nonlinear transformations of the input automatically.

Linear regression therefore serves as the starting point. It gives the basic training machinery in its cleanest form. The rest of deep learning keeps this machinery and increases the expressive power of the model.
