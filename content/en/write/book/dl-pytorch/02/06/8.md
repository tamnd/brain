## 6.9 Chapter Summary

This chapter introduced linear models and the basic optimization methods used to train them in PyTorch. Although linear models are simple, they contain the same training structure used in larger neural networks: compute predictions, compute a scalar loss, backpropagate gradients, and update parameters.

Linear regression predicts real-valued outputs from weighted sums of input features:

$$
\hat{y} = w^\top x + b.
$$

For batches, the same computation becomes:

$$
\hat{Y} = XW + b.
$$

The usual loss is mean squared error. PyTorch can implement this manually with tensors and autograd, or directly with `nn.Linear`, `nn.MSELoss`, and an optimizer.

Logistic regression extends the same linear structure to classification. It computes a logit

$$
z = w^\top x + b,
$$

then interprets it as a probability through the sigmoid function for binary classification. In PyTorch, the stable training loss is `nn.BCEWithLogitsLoss`, which combines sigmoid and binary cross-entropy internally.

For multi-class classification, the model produces one logit per class. `nn.CrossEntropyLoss` should receive raw logits and integer class labels. It applies a stable log-softmax internally.

The main supervised loss functions are:

| Task | Output | Target | Common PyTorch loss |
|---|---|---|---|
| Regression | Real values | Real values | `nn.MSELoss`, `nn.L1Loss`, `nn.HuberLoss` |
| Binary classification | One logit | 0 or 1 float | `nn.BCEWithLogitsLoss` |
| Multi-class classification | Class logits | Integer class ID | `nn.CrossEntropyLoss` |
| Multi-label classification | Label logits | 0 or 1 matrix | `nn.BCEWithLogitsLoss` |

Gradient descent updates parameters by moving against the gradient of the loss:

$$
\theta \leftarrow \theta - \eta \nabla_\theta L(\theta).
$$

The learning rate \(\eta\) controls the step size. Small learning rates train slowly. Large learning rates can destabilize training.

Stochastic gradient descent estimates the full gradient from minibatches. This makes training efficient and scalable. A PyTorch training loop usually has this shape:

```python
for batch_x, batch_y in loader:
    pred = model(batch_x)
    loss = loss_fn(pred, batch_y)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

Momentum improves SGD by accumulating a velocity. Adaptive methods such as AdaGrad, RMSProp, Adam, and AdamW scale parameter updates using statistics of past gradients. AdamW is a common default for modern PyTorch models because it combines adaptive updates with decoupled weight decay.

Learning rate schedules change the learning rate during training. Common schedules include step decay, multi-step decay, exponential decay, cosine decay, warmup, ReduceLROnPlateau, and one-cycle training. Modern transformer-style training often uses warmup followed by cosine or linear decay.

Regularization reduces overfitting. Weight decay discourages large parameters. Dropout randomly removes activations during training. Early stopping halts training when validation performance stops improving. Data augmentation increases effective data diversity. Label smoothing reduces overconfidence.

A complete training setup therefore includes:

```python
model = ...
loss_fn = ...
optimizer = ...
scheduler = ...
train_loader = ...
val_loader = ...
```

and a training loop that carefully handles training mode, evaluation mode, gradients, optimizer updates, and validation metrics.

The concepts in this chapter are the foundation for the rest of the book. Multilayer neural networks, convolutional networks, transformers, diffusion models, and reinforcement learning systems all use the same basic components: parameters, losses, gradients, optimizers, regularization, and evaluation.
