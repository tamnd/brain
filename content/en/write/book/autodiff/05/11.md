Reverse mode automatic differentiation propagates derivative information backward through a computation. Forward mode starts from input perturbations and pushes tangents toward outputs. Reverse mode starts from output sensitivities and pulls adjoints back toward inputs.

For a function

$$
f : \mathbb{R}^n \to \mathbb{R}^m,
$$

reverse mode naturally computes vector-Jacobian products:

$$
w^\top J_f(x).
$$

When the output is scalar,

$$
f : \mathbb{R}^n \to \mathbb{R},
$$

a single reverse-mode pass computes the full gradient:

$$
\nabla f(x).
$$

This makes reverse mode the dominant method for training neural networks, optimizing scalar objectives, and differentiating large parameterized programs.

## Adjoint Propagation

The core object in reverse mode is the adjoint. For a program variable $v$, its adjoint is usually written as

$$
\bar{v}.
$$

The adjoint measures how a final output changes with respect to that variable.

For a scalar objective $y$, the adjoint of an intermediate variable $v$ is

$$
\bar{v} = \frac{\partial y}{\partial v}.
$$

This is different from a forward-mode tangent. A tangent tells us how $v$ changes when the input changes. An adjoint tells us how the output changes when $v$ changes.

## Forward and reverse perspectives

Suppose a program computes

$$
x \to v \to y.
$$

Forward mode propagates:

$$
\dot{x} \to \dot{v} \to \dot{y}.
$$

Reverse mode propagates:

$$
\bar{y} \to \bar{v} \to \bar{x}.
$$

The two directions correspond to two ways of applying the chain rule.

Forward mode computes:

$$
\dot{y} =
\frac{\partial y}{\partial v}
\frac{\partial v}{\partial x}
\dot{x}.
$$

Reverse mode computes:

$$
\bar{x} =
\bar{y}
\frac{\partial y}{\partial x}.
$$

For scalar output, we seed:

$$
\bar{y}=1.
$$

Then reverse propagation computes all partial derivatives of $y$ with respect to the inputs.

## Local adjoint rules

Consider a primitive operation:

$$
z = \phi(x_1,\ldots,x_k).
$$

If the adjoint of $z$ is known, then each input receives contribution:

$$
\bar{x}_i
\mathrel{+}=
\bar{z}
\frac{\partial z}{\partial x_i}.
$$

The symbol $\mathrel{+=}$ matters. A variable may influence the output through many paths. Reverse mode must accumulate all contributions.

For example, if

$$
z = xy,
$$

then

$$
\frac{\partial z}{\partial x}=y,
\qquad
\frac{\partial z}{\partial y}=x.
$$

So the reverse rule is:

$$
\bar{x} \mathrel{+}= \bar{z}y,
$$

$$
\bar{y} \mathrel{+}= \bar{z}x.
$$

For addition,

$$
z = x + y,
$$

the reverse rule is:

$$
\bar{x} \mathrel{+}= \bar{z},
$$

$$
\bar{y} \mathrel{+}= \bar{z}.
$$

For sine,

$$
z = \sin x,
$$

the reverse rule is:

$$
\bar{x} \mathrel{+}= \bar{z}\cos x.
$$

Each rule is local, but it runs in the reverse order of the primal computation.

## Example: scalar function

Let

$$
f(x) = x^2 + 3x.
$$

Write the computation as:

$$
a = x^2,
$$

$$
b = 3x,
$$

$$
y = a + b.
$$

The forward pass computes the primal values. The reverse pass begins with:

$$
\bar{y}=1.
$$

From

$$
y = a + b,
$$

we get:

$$
\bar{a} \mathrel{+}= \bar{y},
\qquad
\bar{b} \mathrel{+}= \bar{y}.
$$

So:

$$
\bar{a}=1,
\qquad
\bar{b}=1.
$$

From

$$
b=3x,
$$

we get:

$$
\bar{x} \mathrel{+}= 3\bar{b}=3.
$$

From

$$
a=x^2,
$$

we get:

$$
\bar{x} \mathrel{+}=2x\bar{a}=2x.
$$

Accumulate both contributions:

$$
\bar{x}=2x+3.
$$

This equals:

$$
\frac{dy}{dx}.
$$

At $x=5$,

$$
\bar{x}=13.
$$

The forward value is:

$$
y=40.
$$

The reverse result is:

$$
(y,\bar{x})=(40,13).
$$

## Example with shared variables

Reverse mode must accumulate contributions when a variable is reused.

Consider:

$$
y = xx + x.
$$

Use intermediates:

$$
a = xx,
$$

$$
y = a + x.
$$

Start:

$$
\bar{y}=1.
$$

From:

$$
y=a+x,
$$

we get:

$$
\bar{a} \mathrel{+}=1,
\qquad
\bar{x} \mathrel{+}=1.
$$

From:

$$
a=xx,
$$

both inputs are the same variable $x$. The multiplication rule gives two contributions:

$$
\bar{x} \mathrel{+}=x\bar{a},
$$

$$
\bar{x} \mathrel{+}=x\bar{a}.
$$

So:

$$
\bar{x}=1+2x.
$$

This is correct because:

$$
\frac{d}{dx}(x^2+x)=2x+1.
$$

The accumulation step is not an implementation detail. It is the mathematical mechanism that sums derivative contributions over all paths.

## Vector outputs and seed adjoints

For

$$
f : \mathbb{R}^n \to \mathbb{R}^m,
$$

reverse mode begins with an output adjoint vector:

$$
\bar{y}=w.
$$

The result at the inputs is:

$$
\bar{x}=J_f(x)^\top w.
$$

Equivalently:

$$
\bar{x}^\top = w^\top J_f(x).
$$

Thus one reverse pass computes one vector-Jacobian product.

For scalar output,

$$
m=1,
$$

choosing

$$
\bar{y}=1
$$

gives:

$$
\bar{x}=J_f(x)^\top.
$$

This is the gradient.

## Reverse mode as transposed linearization

Forward mode applies the local linearization in the same direction as execution.

If a primitive has local Jacobian:

$$
A,
$$

then forward mode propagates:

$$
\dot{z}=A\dot{x}.
$$

Reverse mode propagates through the transpose:

$$
\bar{x}=A^\top \bar{z}.
$$

For a chain of operations:

$$
x \xrightarrow{A_1} v \xrightarrow{A_2} y,
$$

forward mode computes:

$$
\dot{y}=A_2A_1\dot{x}.
$$

Reverse mode computes:

$$
\bar{x}=A_1^\top A_2^\top \bar{y}.
$$

The transpose reverses the order. This is why reverse mode traverses the computation backward.

## Need for primal values

Reverse rules often require primal values from the forward pass.

For multiplication:

$$
z=xy,
$$

the reverse rule uses $x$ and $y$:

$$
\bar{x} \mathrel{+}= \bar{z}y,
\qquad
\bar{y} \mathrel{+}= \bar{z}x.
$$

For sine:

$$
z=\sin x,
$$

the reverse rule uses $x$:

$$
\bar{x} \mathrel{+}= \bar{z}\cos x.
$$

Therefore reverse mode usually needs access to intermediate primal values during the backward pass.

There are two common strategies:

| Strategy | Description |
|---|---|
| store | save intermediate values during the forward pass |
| recompute | rerun parts of the forward computation during the backward pass |

This creates a memory-time tradeoff. Storing everything is fast but memory-heavy. Recomputation saves memory but increases runtime.

## Computational shape

For a scalar-output function:

$$
f : \mathbb{R}^n \to \mathbb{R},
$$

reverse mode computes all $n$ gradient components in one backward pass.

The cost is typically a small constant multiple of the primal computation:

$$
O(C_f).
$$

This is the central reason reverse mode is used for large-scale optimization.

For a vector-output function:

$$
f : \mathbb{R}^n \to \mathbb{R}^m,
$$

one reverse pass computes one weighted combination of Jacobian rows:

$$
w^\top J_f(x).
$$

To construct the full Jacobian, reverse mode generally needs $m$ passes, one for each basis vector in output space.

## Reverse mode versus forward mode

The two modes compute complementary products.

| Mode | Seed | Product | Best case |
|---|---|---|---|
| Forward | input tangent $v$ | $Jv$ | few input directions |
| Reverse | output adjoint $w$ | $J^\top w$ | few output directions |

For scalar-output objectives, reverse mode is usually superior because one pass gives the full gradient. For scalar-input functions with many outputs, forward mode is usually superior because one pass gives all output sensitivities.

## Implementation sketch

A minimal reverse-mode system records operations during the forward pass. Each recorded operation knows how to push its output adjoint back to its input adjoints.

Conceptually:

```go
type Node struct {
    Value float64
    Grad  float64

    Backward func()
}
```

For multiplication:

```go
func Mul(x, y *Node) *Node {
    z := &Node{
        Value: x.Value * y.Value,
    }

    z.Backward = func() {
        x.Grad += z.Grad * y.Value
        y.Grad += z.Grad * x.Value
    }

    return z
}
```

For addition:

```go
func Add(x, y *Node) *Node {
    z := &Node{
        Value: x.Value + y.Value,
    }

    z.Backward = func() {
        x.Grad += z.Grad
        y.Grad += z.Grad
    }

    return z
}
```

To compute gradients:

```go
y := F(x)
y.Grad = 1

for nodes in reverse_topological_order {
    node.Backward()
}
```

This is the basic structure behind many dynamic graph systems.

## Summary

Reverse mode automatic differentiation propagates adjoints backward through a computation. Each adjoint records how the final output changes with respect to an intermediate variable.

A local reverse rule takes the adjoint of an operation output and distributes contributions to the adjoints of its inputs. Because a variable may affect the output through many paths, adjoints are accumulated.

The natural product of reverse mode is:

$$
J_f(x)^\top w.
$$

For scalar-output functions, this yields the full gradient in one reverse pass, which makes reverse mode the method of choice for high-dimensional optimization and neural network training.
