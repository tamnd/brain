## Tensor Operations

Modern automatic differentiation systems operate primarily on tensors. A tensor is a multidimensional array with shape, dtype, layout, and device placement. Matrix calculus extends naturally to tensors, but tensor programs introduce additional concerns:

- broadcasting
- batching
- reductions
- reshaping
- permutation
- memory layout
- device execution
- sparse structure

AD systems differentiate tensor programs by applying local derivative rules to tensor primitives while preserving shape semantics.

## Tensors and Indices

A tensor of rank $k$ has components:

$$
X_{i_1 i_2 \cdots i_k}.
$$

Examples:

| Rank | Object |
|---|---|
| 0 | scalar |
| 1 | vector |
| 2 | matrix |
| 3+ | higher-order tensor |

A tensor operation defines relationships between indexed components.

For example:

$$
Y_{ij}=2X_{ij}
$$

has derivative:

$$
\frac{\partial Y_{ij}}{\partial X_{ab}} =
2\,\delta_{ia}\delta_{jb}.
$$

AD systems usually do not materialize such high-rank Jacobians explicitly. Instead, they propagate tangents or adjoints using tensor operations directly.

## Elementwise Operations

An elementwise operation applies independently to every tensor entry.

Example:

```text
Y = sin(X)
```

means:

$$
Y_{ij\cdots}=\sin(X_{ij\cdots}).
$$

The differential is:

$$
dY_{ij\cdots} =
\cos(X_{ij\cdots})\,dX_{ij\cdots}.
$$

So the reverse-mode rule is:

```text
X_bar += cos(X) * Y_bar
```

The multiplication is elementwise.

Elementwise operations are especially efficient because their Jacobians are diagonal with respect to flattened tensor coordinates.

## Tensor Addition

For:

$$
Z = X + Y,
$$

the differential is:

$$
dZ = dX + dY.
$$

Reverse mode gives:

```text
X_bar += Z_bar
Y_bar += Z_bar
```

Tensor addition therefore duplicates the adjoint into both parents.

This is the tensor analogue of scalar addition.

## Elementwise Multiplication

For Hadamard multiplication:

$$
Z = X \odot Y,
$$

the differential is:

$$
dZ = dX \odot Y + X \odot dY.
$$

Reverse mode:

```text
X_bar += Z_bar * Y
Y_bar += Z_bar * X
```

Again, all multiplications are elementwise.

This operation appears constantly in neural networks, masking, normalization, and gating.

## Tensor Contraction

Tensor contraction generalizes matrix multiplication.

Example:

$$
C_{ij} = \sum_k A_{ik}B_{kj}.
$$

This is ordinary matrix multiplication.

A more general contraction:

$$
Y_{abc} =
\sum_{ij}
A_{aij}B_{ib}C_{jc}
$$

contracts shared indices.

Frameworks often expose tensor contraction through Einstein notation:

```text
Y = einsum("aij,ib,jc->abc", A, B, C)
```

Differentiation follows the same contraction structure. Each backward rule contracts over complementary indices.

For matrix multiplication:

$$
C=AB,
$$

the reverse rules are:

$$
\bar{A}=\bar{C}B^T
$$

$$
\bar{B}=A^T\bar{C}.
$$

Tensor contraction backward rules generalize this pattern.

## Einstein Summation

Einstein notation expresses tensor operations compactly.

Repeated indices imply summation:

$$
y_i = A_{ij}x_j.
$$

Outer products:

$$
Z_{ij}=x_i y_j.
$$

Tensor contraction:

$$
C_{ij}=A_{ik}B_{kj}.
$$

AD systems internally lower many tensor programs into contraction-like kernels.

Einstein notation is useful because it makes derivative structure visible.

For example:

$$
L=A_{ij}B_{jk}C_{ki}.
$$

Differentiating with respect to $A$:

$$
\frac{\partial L}{\partial A_{ij}} =
B_{jk}C_{ki}.
$$

The derivative follows the surviving indices.

## Tensor Reshaping

Reshape changes tensor interpretation without changing data.

```text
Y = reshape(X, new_shape)
```

Mathematically, reshape is a bijection between index systems.

The derivative is therefore trivial:

```text
X_bar = reshape(Y_bar, original_shape)
```

No numerical computation changes. Only index interpretation changes.

Transpose and permutation behave similarly.

## Tensor Permutation

For:

```text
Y = transpose(X)
```

or:

```text
Y[i,j,k] = X[k,i,j]
```

the backward rule applies the inverse permutation:

```text
X_bar = transpose_back(Y_bar)
```

Permutation operations therefore rearrange adjoints rather than numerically transform them.

## Reductions

A reduction lowers tensor rank.

Example:

```text
y = sum(X)
```

means:

$$
y=\sum_{ij\cdots}X_{ij\cdots}.
$$

The differential is:

$$
dy=\sum_{ij\cdots}dX_{ij\cdots}.
$$

So reverse mode broadcasts the scalar adjoint:

```text
X_bar += ones_like(X) * y_bar
```

For mean reduction:

```text
y = mean(X)
```

the backward rule becomes:

```text
X_bar += y_bar / numel(X)
```

Reduction gradients expand back to the input shape.

## Max Reduction

Max is piecewise differentiable.

For:

```text
y = max(X)
```

the backward pass routes gradients only to maximal entries.

If the maximum is unique:

```text
X_bar[i] += y_bar   if X[i] is maximal
```

All other entries receive zero.

At ties, the derivative is non-unique. Systems choose conventions:

- first maximal index
- evenly split gradients
- arbitrary stable rule

The operation is non-smooth at ties.

## Broadcasting

Broadcasting expands smaller tensors virtually.

Example:

```text
X: [batch, features]
b: [features]

Y = X + b
```

The forward pass conceptually replicates `b` across the batch axis.

Backward propagation must undo this replication:

```text
X_bar += Y_bar
b_bar += sum(Y_bar, axis=batch)
```

The rule is:

```text
Gradient over broadcasted axes:
    sum over expanded dimensions.
```

Broadcast semantics are one of the most important tensor-level AD rules.

## Tensor Shapes

Every tensor operation imposes shape constraints.

Example:

```text
A: [m, k]
B: [k, n]

C = A @ B
```

Backward shapes:

```text
C_bar: [m, n]

A_bar = C_bar @ B.T   -> [m, k]
B_bar = A.T @ C_bar   -> [k, n]
```

AD systems rely heavily on shape propagation. Shape mismatches are often easier to detect than algebraic mistakes.

Shape inference is therefore part of differentiable program compilation.

## Tensor Layout and Memory

Tensor operations are not only algebraic. They also interact with memory layout.

Example layouts:

```text
contiguous
strided
channels-first
channels-last
blocked
tiled
sparse
```

A transpose may be represented as metadata rather than physical data movement.

Backward operations must preserve correct layout semantics.

Example:

```text
Y = transpose(X)
```

Backward:

```text
X_bar = transpose(Y_bar)
```

The system must know whether the transpose is logical or physical.

Layout affects performance but not mathematical derivatives.

## Batched Operations

Tensor systems often batch operations.

Example:

```text
X: [batch, in]
W: [out, in]

Y = X @ W.T
```

Mathematically:

$$
Y_b = W X_b
$$

for each batch element $b$.

Backward propagation accumulates contributions across the batch:

$$
\bar{W} =
\sum_b
\bar{Y}_b X_b^T.
$$

This is why minibatch gradients naturally average or sum over examples.

## Tensor Jacobians

For tensors, the full Jacobian becomes enormous.

If:

```text
X.shape = [1000,1000]
Y.shape = [1000,1000]
```

then:

$$
J \in \mathbb{R}^{10^6 \times 10^6}.
$$

Explicit construction is infeasible.

AD systems therefore rely on:

- JVPs
- VJPs
- local contraction rules
- fused kernels

rather than explicit Jacobian matrices.

## Tensor Programs as Graphs

A tensor program defines a graph of tensor transformations:

```text
X -> matmul -> add -> relu -> reduce -> loss
```

Each node contributes a local derivative rule.

Reverse mode propagates adjoints backward through the graph while preserving tensor shapes and reduction semantics.

The backward graph is itself a tensor program.

## Correctness Rule

Tensor differentiation extends matrix calculus to multidimensional arrays.

```text
Elementwise operation:
    backward is elementwise.

Reduction:
    backward expands or broadcasts.

Broadcast:
    backward sums over expanded axes.

Permutation and reshape:
    backward applies inverse index transformation.

Tensor contraction:
    backward uses complementary contractions.
```

Automatic differentiation systems therefore operate as tensor transformation engines. They propagate derivatives through tensor algebra while preserving shapes, layouts, and execution semantics.
