## Matrix Calculus

Matrix calculus studies derivatives of functions whose inputs or outputs are vectors, matrices, or higher-rank arrays. Automatic differentiation systems use matrix calculus constantly, even when the user writes ordinary tensor code.

A scalar function has derivative:

$$
f'(x)
$$

A vector function has a Jacobian:

$$
J_{ij} = \frac{\partial y_i}{\partial x_j}
$$

A matrix-valued function can be flattened into vectors, differentiated, and reshaped. AD systems usually avoid forming full Jacobians unless explicitly requested. Instead, they compute products with Jacobians.

## Vectors

Let

$$
y = f(x)
$$

where

$$
x \in \mathbb{R}^n,\qquad y \in \mathbb{R}^m.
$$

The Jacobian is:

$$
J_f(x) \in \mathbb{R}^{m \times n}
$$

with entries:

$$
(J_f)_{ij} =
\frac{\partial y_i}{\partial x_j}.
$$

Forward mode naturally computes Jacobian-vector products:

$$
J_f(x)v.
$$

Reverse mode naturally computes vector-Jacobian products:

$$
u^T J_f(x).
$$

This is why AD systems can handle large tensor programs without materializing huge derivative arrays.

## Matrices as Linear Maps

A matrix multiplication

$$
y = Ax
$$

has derivatives:

$$
dy = A\,dx + dA\,x.
$$

If $A$ is constant, then:

$$
dy = A\,dx.
$$

So the Jacobian with respect to $x$ is simply $A$.

If $x$ is constant and $A$ varies, then:

$$
dy = dA\,x.
$$

In reverse mode, if $\bar{y}$ is the adjoint of $y$, then:

$$
\bar{x} = A^T \bar{y}
$$

and

$$
\bar{A} = \bar{y}x^T.
$$

These formulas are central to neural network layers, least-squares problems, linear solvers, and scientific computing.

## Matrix-Matrix Multiplication

For

$$
C = AB,
$$

the differential is:

$$
dC = dA\,B + A\,dB.
$$

Given an output adjoint $\bar{C}$, reverse mode gives:

$$
\bar{A} = \bar{C}B^T
$$

$$
\bar{B} = A^T\bar{C}.
$$

These equations explain the backward pass of dense linear layers.

```text
C = A @ B

A_bar = C_bar @ B.T
B_bar = A.T @ C_bar
```

The shapes verify the formulas:

```text
A:     m x k
B:     k x n
C:     m x n

C_bar: m x n
A_bar: m x k
B_bar: k x n
```

Shape correctness is one of the best ways to check matrix derivative formulas.

## Scalar Losses and Gradients

Most optimization problems use a scalar loss:

$$
L = f(X)
$$

where $X$ is a vector or matrix.

The gradient has the same shape as $X$:

$$
\nabla_X L
$$

with entries:

$$
(\nabla_X L)_{ij} =
\frac{\partial L}{\partial X_{ij}}.
$$

This shape convention is practical. A parameter tensor and its gradient can be stored side by side.

```text
parameter.shape == gradient.shape
```

Reverse mode is especially efficient for scalar losses because it computes the full gradient with one backward pass.

## Differentials

Differentials give a clean notation for matrix derivatives.

For a scalar function

$$
L = f(X),
$$

the differential can be written as:

$$
dL = \langle \nabla_X L, dX\rangle
$$

where the matrix inner product is:

$$
\langle A,B\rangle = \operatorname{tr}(A^T B).
$$

This means:

$$
dL = \operatorname{tr}\left((\nabla_X L)^T dX\right).
$$

To derive a gradient, rewrite the differential until it has the form:

$$
dL = \operatorname{tr}(G^T dX).
$$

Then:

$$
\nabla_X L = G.
$$

This is the standard method for matrix calculus in AD derivations.

## Example: Quadratic Form

Let

$$
L = x^T A x.
$$

The differential is:

$$
dL = dx^T A x + x^T A\,dx.
$$

Rewrite both terms as inner products with $dx$:

$$
dx^T A x = (Ax)^T dx
$$

$$
x^T A\,dx = (A^T x)^T dx.
$$

So:

$$
dL = (Ax + A^T x)^T dx.
$$

Therefore:

$$
\nabla_x L = (A + A^T)x.
$$

If $A$ is symmetric:

$$
\nabla_x L = 2Ax.
$$

AD systems compute this by composing local rules for matrix multiplication and addition. They do not need a special global rule for quadratic forms.

## Example: Frobenius Norm

Let

$$
L = \|X\|_F^2
$$

where

$$
\|X\|_F^2 = \operatorname{tr}(X^T X).
$$

The differential is:

$$
dL = 2\operatorname{tr}(X^T dX).
$$

So:

$$
\nabla_X L = 2X.
$$

For the half-squared norm,

$$
L = \frac12\|X\|_F^2,
$$

the gradient is:

$$
\nabla_X L = X.
$$

This convention appears frequently because it removes a factor of $2$.

## Trace Identities

Trace notation is useful because it lets matrix expressions be rearranged into gradient form.

Common identities:

$$
\operatorname{tr}(A) = \operatorname{tr}(A^T)
$$

$$
\operatorname{tr}(ABC) = \operatorname{tr}(BCA) = \operatorname{tr}(CAB)
$$

$$
\operatorname{tr}(A^T B) = \langle A,B\rangle
$$

These identities are not AD rules. They are algebraic tools used to derive compact backward formulas for matrix operations.

## Broadcasting and Reductions

Matrix calculus in software also includes broadcasting.

Example:

```text
Y = X + b
```

where:

```text
X: batch x features
b: features
Y: batch x features
```

The forward operation broadcasts `b` across the batch dimension.

If $\bar{Y}$ is the output adjoint, then:

```text
X_bar = Y_bar
b_bar = sum(Y_bar, axis=0)
```

The gradient with respect to a broadcasted value must reduce along the broadcasted axes.

This rule is systems-level matrix calculus: gradients must match the original input shape.

## Avoiding Full Jacobians

For a layer

$$
y = Wx + b
$$

with $W\in\mathbb{R}^{m\times n}$, the Jacobian with respect to $W$ has size:

$$
m \times (mn).
$$

Materializing it is wasteful.

Reverse mode instead uses the adjoint formula:

$$
\bar{W} = \bar{y}x^T.
$$

This computes the needed gradient directly.

The same principle applies throughout tensor AD. Full Jacobians are rarely the right internal representation. Local adjoint rules and Jacobian products are usually enough.

## Correctness Rule

Matrix calculus gives shape-aware derivative rules for vector and matrix programs.

```text
Forward mode:
    computes JVPs such as Jv.

Reverse mode:
    computes VJPs such as u^T J.

Scalar loss:
    gradient has the same shape as the input parameter.

Matrix operation:
    derive backward rules using differentials and trace identities.
```

Automatic differentiation implements matrix calculus operationally. It applies local rules to tensor primitives, propagates adjoints with correct shapes, and avoids full Jacobian construction unless explicitly requested.
