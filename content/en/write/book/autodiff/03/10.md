## Local Derivative Propagation

Automatic differentiation is based on one principle:

> A complex derivative can be computed by composing local derivatives along the executed computation.

Every operation contributes a small derivative rule. The global derivative emerges from repeated application of the chain rule.

Consider:

$$
y = \sin(x_1x_2) + x_1.
$$

A straight-line program is:

```text
v1 = x1
v2 = x2
v3 = v1 * v2
v4 = sin(v3)
v5 = v4 + v1
y  = v5
```

Each line defines a local transformation. Automatic differentiation propagates derivative information through these transformations one step at a time.

## The Chain Rule as Local Composition

Suppose a function is written as a composition:

$$
f = f_n \circ f_{n-1} \circ \cdots \circ f_1.
$$

The chain rule states:

$$
Df(x) =
Df_n
Df_{n-1}
\cdots
Df_1.
$$

Automatic differentiation does not construct this full derivative symbolically. Instead, it evaluates the factors incrementally during execution.

Each program statement contributes one local derivative operator.

## Elementary Operations

A local derivative rule belongs to an elementary operation.

Examples:

| Operation | Local rule |
|---|---|
| $v = a + b$ | $dv = da + db$ |
| $v = ab$ | $dv = b\,da + a\,db$ |
| $v = \sin(a)$ | $dv = \cos(a)\,da$ |
| $v = \exp(a)$ | $dv = \exp(a)\,da$ |
| $v = \log(a)$ | $dv = \frac{1}{a}da$ |

Each rule is local because it depends only on:

1. the operation,
2. the operation inputs,
3. sometimes the operation output.

The rule does not depend on the entire program.

## Local Jacobians

Each elementary operation defines a small Jacobian.

For:

$$
v = ab,
$$

view the operation as:

$$
g(a,b)=ab.
$$

Its Jacobian is:

$$
Dg(a,b) =
\begin{bmatrix}
b & a
\end{bmatrix}.
$$

For:

$$
v = \sin(a),
$$

the Jacobian is:

$$
Dg(a)=\cos(a).
$$

Automatic differentiation propagates derivatives by composing these local Jacobians according to program structure.

## Forward Propagation

Forward mode propagates tangent information in execution order.

Suppose:

$$
\dot x_1 = 1,
\qquad
\dot x_2 = 0.
$$

Then:

```text
v3 = v1 * v2
```

propagates:

$$
\dot v_3 =
v_2 \dot v_1 + v_1 \dot v_2.
$$

Next:

```text
v4 = sin(v3)
```

propagates:

$$
\dot v_4 =
\cos(v_3)\dot v_3.
$$

Finally:

```text
v5 = v4 + v1
```

propagates:

$$
\dot v_5 =
\dot v_4 + \dot v_1.
$$

The tangent flows forward through the dependency graph.

## Reverse Propagation

Reverse mode propagates adjoints backward.

Start with:

$$
\bar y = 1.
$$

Then:

```text
v5 = v4 + v1
```

produces:

$$
\bar v_4 += \bar v_5,
\qquad
\bar v_1 += \bar v_5.
$$

Next:

```text
v4 = sin(v3)
```

produces:

$$
\bar v_3 += \bar v_4 \cos(v_3).
$$

Finally:

```text
v3 = v1 * v2
```

produces:

$$
\bar v_1 += \bar v_3 v_2,
\qquad
\bar v_2 += \bar v_3 v_1.
$$

The adjoint flows backward through reversed dependencies.

## Propagation as Graph Traversal

Derivative propagation follows the dependency graph.

Forward direction:

```text
x -> v1 -> v2 -> y
```

Reverse direction:

```text
y -> v2 -> v1 -> x
```

Forward mode pushes tangent vectors through local Jacobians.

Reverse mode pulls adjoints backward through transposed local Jacobians.

## Tangent Propagation

Forward mode computes directional derivatives.

If:

$$
x(t)=x+t\dot x,
$$

then:

$$
\dot y =
\frac{d}{dt}f(x+t\dot x)\Big|_{t=0}.
$$

Each local operation propagates this infinitesimal perturbation.

For multiplication:

$$
v = ab,
$$

the tangent rule is:

$$
\dot v =
b\dot a + a\dot b.
$$

This is precisely the differential of multiplication.

## Adjoint Propagation

Reverse mode propagates sensitivities.

If:

$$
\bar v = \frac{\partial y}{\partial v},
$$

then for:

$$
v = ab,
$$

the adjoint rules are:

$$
\bar a += \bar v b,
\qquad
\bar b += \bar v a.
$$

These arise from the chain rule:

$$
\frac{\partial y}{\partial a} =
\frac{\partial y}{\partial v}
\frac{\partial v}{\partial a}.
$$

Reverse mode therefore accumulates sensitivities of the final output with respect to intermediate variables.

## Locality

The defining property of AD is locality.

To differentiate:

```text
v = op(a, b)
```

the system needs only:

| Information | Purpose |
|---|---|
| Operation type | choose derivative rule |
| Input values | evaluate rule |
| Output adjoint or tangent | propagate derivatives |

No global symbolic manipulation is required.

This locality makes AD compositional and scalable.

## Locality Versus Symbolic Differentiation

Symbolic differentiation manipulates whole expressions.

Example:

$$
\frac{d}{dx}\sin(x^2+1).
$$

A symbolic system rewrites:

$$
\cos(x^2+1)(2x).
$$

AD instead evaluates:

```text
v1 = x * x
v2 = v1 + 1
v3 = sin(v2)
```

and propagates local derivatives during execution.

The expression structure is preserved rather than algebraically transformed.

This avoids expression explosion.

## Locality Versus Finite Differences

Finite differences approximate derivatives numerically.

Example:

$$
f'(x)
\approx
\frac{f(x+h)-f(x)}{h}.
$$

AD instead computes derivatives exactly up to floating point rounding by propagating local derivative rules.

Advantages:

| Method | Error source |
|---|---|
| Finite differences | truncation + cancellation |
| Symbolic differentiation | expression growth |
| Automatic differentiation | floating point only |

AD combines numerical evaluation with exact chain-rule propagation.

## Vector-Valued Operations

Local propagation generalizes naturally to vectors and tensors.

Suppose:

$$
y = Ax.
$$

The local Jacobian is:

$$
Dy = A.
$$

Forward mode propagates:

$$
\dot y = A\dot x.
$$

Reverse mode propagates:

$$
\bar x += A^T \bar y.
$$

Thus:

| Mode | Local linear algebra |
|---|---|
| Forward | Jacobian-vector product |
| Reverse | vector-Jacobian product |

This distinction determines computational efficiency.

## Accumulation

If multiple paths contribute to a variable, derivative contributions accumulate.

Example:

```text
v1 = x * x
v2 = sin(v1)
v3 = cos(v1)
y  = v2 + v3
```

The graph is:

```text
      -> v2 --
     /         \
x -> v1          -> y
     \         /
      -> v3 --
```

The derivative of $v_1$ receives contributions from both branches:

$$
\bar v_1 =
\bar v_2 \cos(v_1) -
\bar v_3 \sin(v_1).
$$

Reverse mode therefore performs summation at merge points.

## Differential View

Automatic differentiation can be viewed as propagating differentials.

For:

$$
v = f(u),
$$

the differential relation is:

$$
dv = Df(u)\,du.
$$

Forward mode propagates $du$-type objects.

Reverse mode propagates dual objects:

$$
\bar u = Df(u)^T \bar v.
$$

Thus forward and reverse modes are dual forms of local derivative propagation.

## Linearization

Every elementary operation is locally linearized.

For small perturbations:

$$
f(x+\delta x)
\approx
f(x)+Df(x)\delta x.
$$

AD propagates these local linear approximations through the computation graph.

Forward mode explicitly computes:

$$
Df(x)\delta x.
$$

Reverse mode computes:

$$
Df(x)^T \lambda.
$$

The derivative computation is therefore a structured linearization of the original program.

## Tensor Example

Consider:

```text
y = relu(matmul(x, w) + b)
```

The local derivative structure is:

| Operation | Local derivative |
|---|---|
| `matmul` | linear map |
| `add` | identity |
| `relu` | elementwise mask |

Forward propagation:

```text
dx -> matmul tangent
   -> add tangent
   -> relu tangent
   -> dy
```

Reverse propagation:

```text
dy_bar
 -> relu backward
 -> add backward
 -> matmul backward
 -> dx_bar, dw_bar
```

Large tensor programs are built from the same local propagation rules as scalar programs.

## Propagation Order

Correct derivative propagation requires dependency order.

Forward mode:

1. compute primal inputs,
2. compute local tangents,
3. move forward.

Reverse mode:

1. compute primal forward pass,
2. initialize output adjoints,
3. traverse backward,
4. accumulate input adjoints.

The dependency graph determines legal propagation order.

## Operational Interpretation

Automatic differentiation is not symbolic algebra. It is not numerical approximation. It is a program execution discipline.

Each executed operation performs two tasks:

| Task | Meaning |
|---|---|
| primal computation | ordinary program execution |
| derivative propagation | local chain-rule update |

The derivative computation is therefore interleaved with, or derived from, the original execution.

## Core Idea

Automatic differentiation works because derivatives propagate locally. Every operation contributes a small derivative rule. The global derivative emerges from repeated composition of these local rules along the dependency graph.

Forward mode propagates tangents forward through local Jacobians. Reverse mode propagates adjoints backward through transposed local Jacobians. Both are implementations of the chain rule expressed operationally over program structure.
