# 10.2 BFS Shortest Paths

## Problem

You need the shortest path from one source vertex to all reachable vertices in an unweighted graph, and you also want the proof idea that makes the algorithm reliable.

The graph may be directed or undirected. The only requirement is that every edge has the same cost. In that setting, "shortest" means "fewest edges."

## Solution

Use breadth-first search with a queue, a distance table, and a parent table.

```text
bfs_shortest_paths(G, s):
    for each vertex v in G:
        dist[v] = infinity
        parent[v] = none
        seen[v] = false

    dist[s] = 0
    seen[s] = true
    queue = empty queue
    push s into queue

    while queue is not empty:
        u = pop front of queue

        for each neighbor v of u:
            if seen[v] == false:
                seen[v] = true
                dist[v] = dist[u] + 1
                parent[v] = u
                push v into queue

    return dist, parent
```

The queue is the important data structure. It ensures that vertices are processed in the same order in which they are discovered. Because each newly discovered vertex has distance exactly one more than the vertex that discovered it, the queue processes all distance `d` vertices before any distance `d + 1` vertex is expanded.

## Discussion

Breadth-first search has two jobs. First, it discovers all vertices reachable from the source. Second, it assigns each reachable vertex the least number of edges needed to reach it. These two jobs happen together because the algorithm explores the graph in concentric layers.

The source is layer `0`. Its neighbors are layer `1`. The neighbors that have not already appeared are layer `2`, and so on. Once a vertex appears in a layer, it never needs to be moved to a later layer, because a later layer represents a longer path.

The parent table records the edge that first discovered each vertex. If `parent[v] = u`, then the algorithm selected the path to `u`, followed by the edge from `u` to `v`, as its path to `v`. Since first discovery happens in the earliest possible layer, this parent edge lies on a shortest path.

## Invariant

At the start of each loop iteration, the queue contains discovered but unprocessed vertices in nondecreasing order of distance from the source.

More precisely, if vertex `x` appears before vertex `y` in the queue, then:

```text
dist[x] <= dist[y]
```

Also, all vertices already removed from the queue have final distances. Their values will never change.

This invariant is enough to justify the algorithm. When the algorithm removes a vertex `u`, no shorter path to `u` can still be waiting elsewhere in the graph, because any such path would have to pass through a vertex from a smaller layer, and all smaller layers have already been processed.

## Correctness

Let `delta(s, v)` denote the true shortest path length from `s` to `v`, measured in number of edges. If no path exists, `delta(s, v)` is infinite.

We prove that when BFS terminates, `dist[v] = delta(s, v)` for every vertex `v`.

First, every finite value assigned by BFS corresponds to an actual path. The source has distance `0`, which corresponds to the empty path from `s` to itself. If BFS assigns `dist[v] = dist[u] + 1`, then `v` is adjacent to `u`, and by induction there is an actual path from `s` to `u` of length `dist[u]`. Appending edge `(u, v)` gives an actual path from `s` to `v` of length `dist[v]`. Therefore the true shortest path cannot be longer than the value assigned by BFS.

Second, BFS cannot miss a shorter path. Consider a shortest path:

```text
s = v0, v1, v2, ..., vk = v
```

where `k = delta(s, v)`. The source `v0` is discovered with distance `0`. Suppose `vi` is discovered with distance at most `i`. When `vi` is processed, BFS examines edge `(vi, vi+1)`. If `vi+1` is undiscovered, it receives distance at most `i + 1`. If it was already discovered, its distance is already no larger than `i + 1`. By induction, `v` is discovered with distance at most `k`.

The first part gives `delta(s, v) <= dist[v)`. The second part gives `dist[v] <= delta(s, v)`. Therefore `dist[v] = delta(s, v)` for every reachable vertex. Unreachable vertices remain at infinity because BFS only assigns distances through actual edges from already reachable vertices.

## Python Implementation

```python
from collections import deque

def bfs_shortest_paths(graph, source):
    """
    Compute shortest path distances from source in an unweighted graph.

    graph:
        Dictionary mapping each vertex to an iterable of neighbors.

    source:
        Starting vertex.

    Returns:
        dist:
            Dictionary mapping each reachable vertex to its distance.
            Unreachable listed vertices receive None.

        parent:
            Dictionary mapping each vertex to its predecessor on one
            shortest path from source. The source has parent None.
    """
    dist = {v: None for v in graph}
    parent = {v: None for v in graph}

    dist[source] = 0
    q = deque([source])

    while q:
        u = q.popleft()

        for v in graph.get(u, []):
            if v not in dist:
                dist[v] = None
                parent[v] = None

            if dist[v] is None:
                dist[v] = dist[u] + 1
                parent[v] = u
                q.append(v)

    return dist, parent
```

This implementation treats a missing adjacency list as an empty one through `graph.get(u, [])`. It also handles neighbors that were not originally listed as keys by inserting them lazily into `dist` and `parent`.

## Reconstructing a Path

```python
def reconstruct_path(parent, source, target):
    path = []
    cur = target

    while cur is not None:
        path.append(cur)
        if cur == source:
            path.reverse()
            return path
        cur = parent.get(cur)

    return None
```

If the function returns `None`, the target was not reached from the source. Otherwise, the returned list is one shortest path from `source` to `target`.

## Example

```python
graph = {
    "A": ["B", "C"],
    "B": ["D"],
    "C": ["D", "E"],
    "D": ["F"],
    "E": ["F"],
    "F": [],
}

dist, parent = bfs_shortest_paths(graph, "A")

print(dist["F"])
print(reconstruct_path(parent, "A", "F"))
```

One possible output is:

```text
3
['A', 'B', 'D', 'F']
```

The path through `C` and `D` is also valid if the adjacency order changes. BFS guarantees a shortest path, but when there are several shortest paths, the specific one depends on neighbor order.

## Complexity

For an adjacency-list graph, BFS runs in:

```text
O(|V| + |E|)
```

It stores:

```text
O(|V|)
```

extra information for the queue, distance table, parent table, and seen state.

For an adjacency matrix, scanning all possible neighbors costs `O(|V|)` per vertex, so the running time becomes:

```text
O(|V|^2)
```

## Common Pitfalls

Mark a vertex when it is enqueued, not when it is dequeued. If you wait until dequeue time, the same vertex may be inserted many times through different parents.

Do not use BFS for weighted shortest paths unless all weights are equal. If weights are `0` and `1`, use 0-1 BFS. If all weights are non-negative, use Dijkstra's algorithm. If negative weights appear, use Bellman-Ford or a related method.

Do not assume the parent tree is unique. The distances are unique, but shortest paths may not be.

Do not stop early when a vertex is first seen unless you only need one target. For all-source distances from the given source, let the queue empty.

## Recipe Pattern

Use BFS shortest paths when the cost of a path is exactly the number of edges. The core pattern is simple: initialize the source with distance `0`, push it into a FIFO queue, assign each unseen neighbor distance `dist[u] + 1`, and record the predecessor used for discovery.
