# 10.14 Distance Labels and Relaxation Framework

## Problem

All shortest path algorithms share a common structure: they maintain tentative distances and improve them through edge relaxations. You need a unified model that explains why different algorithms work and how to design new variants.

## Core Model

Associate each vertex `v` with a distance label `dist[v]`. The label represents the current best-known estimate of the shortest distance from the source `s` to `v`.

Initialize:

```text
dist[s] = 0
dist[v] = infinity for v != s
```

Define relaxation for an edge `(u, v, w)`:

```text
if dist[u] + w < dist[v]:
    dist[v] = dist[u] + w
```

This operation is the only way distances change.

## Key Properties

### Upper Bound Property

At all times:

```text
dist[v] >= delta(s, v)
```

Every label is an upper bound on the true shortest distance. This holds because labels are derived from actual paths.

### Convergence

If no negative cycle is reachable, repeated relaxation eventually produces:

```text
dist[v] = delta(s, v)
```

### Optimality Condition

A distance labeling is optimal if and only if:

```text
dist[v] <= dist[u] + w
```

for every edge `(u, v, w)`.

This condition states that no edge can improve any label. It characterizes a fixed point of relaxation.

## Algorithm Families

All shortest path algorithms differ only in how they schedule relaxations.

| Algorithm | Relaxation Order | Constraint |
|---|---|---|
| BFS | Layer order | unit weights |
| Dijkstra | increasing `dist` | non-negative weights |
| Bellman-Ford | all edges repeatedly | none |
| DAG shortest paths | topological order | acyclic graph |
| 0-1 BFS | deque order | weights in `{0,1}` |
| A* | priority by `g + h` | admissible heuristic |

Each method enforces a discipline that guarantees convergence without unnecessary work.

## Label-Setting vs Label-Correcting

Two broad categories exist.

### Label-Setting Algorithms

Once a vertex label becomes final, it never changes.

Examples:

- BFS  
- Dijkstra  
- DAG shortest paths  

These algorithms rely on structural constraints such as non-negative weights or acyclicity.

### Label-Correcting Algorithms

Labels may change multiple times before stabilizing.

Examples:

- Bellman-Ford  
- SPFA  

These algorithms are more general but may require more work.

## Queue-Based Relaxation

Many algorithms can be expressed as:

```text
initialize queue with source

while queue not empty:
    u = pop

    for each edge (u, v, w):
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w
            push v into queue
```

Different algorithms differ in:

- how the queue is implemented  
- when vertices are inserted  
- whether duplicates are allowed  

Examples:

| Method | Queue type |
|---|---|
| BFS | FIFO queue |
| Dijkstra | min-priority queue |
| 0-1 BFS | deque |
| SPFA | FIFO queue with heuristics |

## Termination

Termination occurs when no relaxation can improve any label. For label-setting algorithms, this happens naturally due to ordering constraints. For label-correcting algorithms, explicit iteration limits or empty queues ensure termination.

If a negative cycle exists, relaxation never stabilizes. Distances continue decreasing indefinitely.

## Correctness Criterion

A shortest path algorithm is correct if:

1. It maintains the upper bound property  
2. It terminates  
3. The final labeling satisfies the optimality condition  

From these, it follows that all labels equal true shortest distances.

## Example Framework

```python
def relax_all(vertices, edges, source):
    dist = {v: float("inf") for v in vertices}
    dist[source] = 0

    changed = True

    while changed:
        changed = False

        for u, v, w in edges:
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                changed = True

    return dist
```

This is a conceptual baseline. It is correct when no negative cycle exists but inefficient because it relaxes edges without structure.

## Complexity Perspective

Efficiency depends on avoiding unnecessary relaxations.

| Strategy | Complexity |
|---|---|
| Unstructured relaxation | `O(infinite)` or very large |
| Bellman-Ford | `O(nm)` |
| Dijkstra | `O(m log n)` |
| DAG method | `O(n + m)` |

The structure imposed by each algorithm reduces redundant relaxations.

## Common Pitfalls

Do not assume relaxation order is irrelevant. Incorrect ordering can break correctness or efficiency.

Do not ignore the optimality condition. A labeling may appear stable but still be incorrect if some edges violate it.

Do not mix label-setting assumptions with negative weights. This breaks invariants.

Do not terminate early without justification. Some algorithms require full propagation.

## Recipe Pattern

Model shortest path computation as repeated relaxation of edges. Choose a scheduling discipline that matches graph constraints. Ensure termination and verify that no edge can further improve any distance.
