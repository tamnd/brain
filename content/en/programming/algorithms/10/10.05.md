# 10.5 Negative Cycles

## Problem

You have a weighted directed graph, and some edges may have negative weight. You need to know whether a shortest path problem has a well-defined answer.

A negative cycle is a cycle whose total edge weight is less than zero. If such a cycle is reachable from the source, then shortest distances to some vertices are undefined, because you can loop around the cycle repeatedly and make the path cost smaller without bound.

## Core Idea

Shortest paths are meaningful only when the path cost has a lower bound. A negative cycle destroys that lower bound.

Suppose a path from `s` reaches a cycle `C`, and the total cost of `C` is `-5`. After reaching the cycle, you can traverse it once, twice, or many times.

```text
cost after 1 loop  = original cost - 5
cost after 2 loops = original cost - 10
cost after k loops = original cost - 5k
```

As `k` grows, the cost keeps decreasing. There is no minimum finite path cost.

## Detection with Bellman-Ford

Bellman-Ford detects negative cycles by running one extra relaxation pass after the normal `|V| - 1` passes.

```text
detect_negative_cycle(G, s):
    dist = shortest estimates from Bellman-Ford after |V| - 1 passes

    for each edge (u, v, w):
        if dist[u] != infinity and dist[u] + w < dist[v]:
            return true

    return false
```

The condition

```text
dist[u] + w < dist[v]
```

means that an edge can still improve a distance after all simple shortest paths should already have been found. Since any simple path has at most `|V| - 1` edges, a further improvement implies that some improving path repeats a vertex. That repeated portion is a cycle. Since the path improves, the cycle has negative total weight.

## Correctness Argument

Assume no reachable negative cycle exists. Then every shortest path can be chosen as a simple path. A simple path in a graph with `|V|` vertices contains at most `|V| - 1` edges. Bellman-Ford therefore finds all shortest paths after `|V| - 1` full edge-relaxation passes.

Now suppose an edge can still be relaxed after those passes. Then there is a path from `s` to some vertex that uses at least `|V|` edges and has smaller cost than the best known simple path. Any path with at least `|V|` edges must repeat a vertex. The repeated segment is a cycle. Since removing a non-negative cycle would not make the path worse, the repeated segment must have negative cost. Therefore a reachable negative cycle exists.

## Marking Affected Vertices

Detecting that some negative cycle exists is often insufficient. You may need to know which vertices have undefined shortest distances.

A vertex `x` has undefined distance if:

1. A negative cycle is reachable from `s`.
2. `x` is reachable from that negative cycle.

The standard method is:

```text
find_negative_affected_vertices(G, s):
    run Bellman-Ford for |V| - 1 passes

    bad = empty set

    for each edge (u, v, w):
        if dist[u] != infinity and dist[u] + w < dist[v]:
            add v to bad

    run DFS or BFS from all vertices in bad

    every reached vertex has distance -infinity
```

The extra graph search propagates the effect of the negative cycle to all vertices whose shortest path value is undefined.

## Python Implementation

```python
from collections import deque

def bellman_ford_with_negative_cycles(vertices, edges, source):
    dist = {v: float("inf") for v in vertices}
    parent = {v: None for v in vertices}

    dist[source] = 0

    for _ in range(len(vertices) - 1):
        updated = False

        for u, v, w in edges:
            if dist[u] != float("inf") and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u
                updated = True

        if not updated:
            break

    affected = set()

    for u, v, w in edges:
        if dist[u] != float("inf") and dist[u] + w < dist[v]:
            affected.add(v)

    graph = {v: [] for v in vertices}
    for u, v, _ in edges:
        graph[u].append(v)

    q = deque(affected)

    while q:
        u = q.popleft()

        for v in graph[u]:
            if v not in affected:
                affected.add(v)
                q.append(v)

    for v in affected:
        dist[v] = float("-inf")

    return dist, parent, affected
```

This function returns three objects. The `dist` table stores ordinary finite distances, `inf` for unreachable vertices, and `-inf` for vertices affected by reachable negative cycles. The `affected` set contains exactly the vertices whose shortest distance is undefined because cost can decrease without bound.

## Example

```python
vertices = ["s", "a", "b", "c", "t"]

edges = [
    ("s", "a", 1),
    ("a", "b", 2),
    ("b", "c", -5),
    ("c", "a", 1),
    ("c", "t", 3),
]

dist, parent, affected = bellman_ford_with_negative_cycles(vertices, edges, "s")

print(dist)
print(affected)
```

The cycle

```text
a -> b -> c -> a
```

has total weight:

```text
2 + (-5) + 1 = -2
```

It is reachable from `s`, and `t` is reachable from the cycle. Therefore `a`, `b`, `c`, and `t` all have distance `-inf`.

## Complexity

Let `n = |V|` and `m = |E|`.

| Step | Time |
|---|---:|
| Bellman-Ford relaxation | `O(n * m)` |
| Extra cycle check | `O(m)` |
| Propagation from affected vertices | `O(n + m)` |

Total time is `O(n * m)`. Extra space is `O(n + m)` if an adjacency list is built for propagation, or `O(n)` if the graph representation is already available.

## Common Pitfalls

Do not report every negative cycle in the graph. For single-source shortest paths, only cycles reachable from the source matter.

Do not mark only the vertices directly relaxed on the extra pass. Any vertex reachable from such a vertex also has undefined distance.

Do not reconstruct paths through affected vertices. Parent pointers may contain cycles and no shortest path exists.

Do not confuse a negative edge with a negative cycle. Negative edges are allowed in Bellman-Ford. Negative cycles are the obstruction.

## Recipe Pattern

Use one extra Bellman-Ford pass to detect reachable negative cycles. Then run graph search from every vertex whose distance can still improve. Mark all reached vertices as affected. These vertices have distance `-inf`, not an ordinary finite shortest path.
