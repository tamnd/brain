# 10.24 Testing and Verification

10.24 Testing and Verification
Problem

Shortest path implementations often appear correct on simple inputs but fail on edge cases such as disconnected graphs, zero-weight edges, negative edges, or numerical limits. You need systematic methods to validate correctness beyond ad hoc testing.

Testing Strategy

Combine three approaches:

deterministic test cases

randomized testing

cross-verification against a trusted baseline

Deterministic Test Cases

Design small graphs where the correct answer is known exactly.

Basic Cases
Case	Property
single vertex	trivial distance
disconnected graph	unreachable vertices
line graph	unique path
cycle	multiple paths
tree	no alternative paths
Edge Cases
Case	Purpose
zero-weight edges	test ordering logic
negative edges	test Bellman-Ford
large weights	test overflow
multiple shortest paths	test parent handling
isolated nodes	test initialization
Randomized Testing

Generate random graphs and compare results with a simple but correct baseline.

Example Strategy

generate random graph with n vertices

assign random weights

run both optimized algorithm and baseline

compare results

Baseline Choice

Use a slow but simple method such as Bellman-Ford for verification.

Python
def verify(graph, edges, source):
    dist1, _ = dijkstra(graph, source)
    dist2, _ = bellman_ford(graph.keys(), edges, source)

    return dist1 == dist2
Invariant Checks

During execution, validate invariants.

Upper Bound Check
dist[v] >= delta(s, v)

Not directly computable, but can be approximated by checking consistency.

Edge Relaxation Condition

After completion:

dist[v] <= dist[u] + w

for all edges (u, v, w).

Python
def check_optimality(dist, edges):
    for u, v, w in edges:
        if dist[u] + w < dist[v]:
            return False
    return True

If this fails, the result is incorrect.

Path Verification

Verify that reconstructed paths match distances.

Python
def check_path(graph, path, dist):
    total = 0
    for i in range(len(path) - 1):
        u, v = path[i], path[i+1]
        for x, w in graph[u]:
            if x == v:
                total += w
                break
    return total == dist[path[-1]]
Negative Cycle Detection Tests

Construct graphs with known negative cycles.

a -> b (1)
b -> c (-3)
c -> a (1)

Verify that Bellman-Ford detects the cycle and does not produce distances.

Stress Testing

Run many random tests with increasing size.

small graphs for correctness

large graphs for performance

Track:

runtime

memory usage

correctness failures

Property-Based Testing

Define properties that must always hold:

distances never decrease after finalization in Dijkstra

BFS distances differ by at most 1 along edges

no edge can further relax after completion

Debugging Techniques
Trace Relaxations

Log when distances change.

update: u -> v, old dist[v], new dist[v]
Visualize Graph

Small graphs can be drawn to verify manually.

Step-by-Step Execution

Run algorithm on small input and inspect each iteration.

Common Bugs
Bug	Cause
incorrect initialization	forgetting to set source distance
missing stale check	processing outdated queue entries
wrong edge direction	directed vs undirected confusion
overflow	using fixed-size integers
parent errors	incorrect reconstruction
infinite loop	negative cycle not handled
Automation

Integrate tests into continuous workflows:

unit tests for small cases

randomized tests for robustness

performance benchmarks

Recipe Pattern

Build a suite of deterministic edge cases, add randomized tests with a trusted baseline, enforce invariants after execution, and validate reconstructed paths. Combine correctness checks with stress testing to ensure reliability.
