# 10.4 Bellman–Ford Algorithm

## Problem

You have a graph that may contain negative edge weights. Given a source vertex `s`, compute shortest path distances to all vertices, or detect that no finite solution exists due to a negative cycle reachable from `s`.

Dijkstra’s algorithm fails in this setting because its monotonicity invariant breaks when negative edges exist.

## Solution

Use the Bellman–Ford algorithm. Repeatedly relax all edges. After at most `|V| - 1` passes, all shortest paths (if they exist) are found.

```text id="4o8y2n"
bellman_ford(G, s):
    for each vertex v in G:
        dist[v] = infinity
        parent[v] = none

    dist[s] = 0

    repeat |V| - 1 times:
        for each edge (u, v, w):
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u

    for each edge (u, v, w):
        if dist[u] + w < dist[v]:
            report "negative cycle"

    return dist, parent
```

The final pass checks for further improvements. Any improvement implies a negative cycle.

## Discussion

Bellman–Ford does not rely on a priority structure. It performs a systematic relaxation over all edges. The algorithm gradually propagates shortest path information outward from the source.

The key idea is that any shortest path in a graph without cycles uses at most `|V| - 1` edges. Therefore, after `k` passes, all shortest paths that use at most `k` edges have been correctly computed.

## Invariant

After the `k`-th full pass over all edges:

> For every vertex `v`, `dist[v]` is the minimum cost of any path from `s` to `v` that uses at most `k` edges.

This invariant is maintained because each relaxation step considers extending known paths by one more edge.

## Correctness

Let `delta(s, v)` be the true shortest path distance.

Any simple path has at most `|V| - 1` edges. Therefore, if no negative cycles exist, the shortest path from `s` to any `v` uses at most `|V| - 1` edges.

By the invariant, after `|V| - 1` passes:

```text id="c6t7xu"
dist[v] = delta(s, v)
```

for all vertices reachable from `s`.

If an additional pass still improves some `dist[v]`, then there exists a path with at least `|V|` edges that is shorter than the best path found so far. Such a path must contain a cycle. Since it improves distance, the cycle must have negative total weight. Therefore a negative cycle exists.

## Python Implementation

```python id="v7s0yc"
def bellman_ford(graph, edges, source):
    """
    graph: list of vertices
    edges: list of (u, v, weight)
    """
    dist = {v: float("inf") for v in graph}
    parent = {v: None for v in graph}

    dist[source] = 0

    for _ in range(len(graph) - 1):
        updated = False

        for u, v, w in edges:
            if dist[u] != float("inf") and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u
                updated = True

        if not updated:
            break

    for u, v, w in edges:
        if dist[u] != float("inf") and dist[u] + w < dist[v]:
            return None, None  # negative cycle detected

    return dist, parent
```

The `updated` flag allows early termination when no changes occur.

## Path Reconstruction

Same pattern as before:

```python id="9s2q0a"
def reconstruct(parent, s, t):
    path = []
    cur = t

    while cur is not None:
        path.append(cur)
        if cur == s:
            path.reverse()
            return path
        cur = parent[cur]

    return None
```

## Complexity

Let `n = |V|` and `m = |E|`.

| Phase | Time |
|---|---:|
| Relaxation passes | `O(n * m)` |
| Cycle check | `O(m)` |

Total time:

```text id="q5o4em"
O(n * m)
```

Space usage is `O(n)`.

## Common Pitfalls

Do not forget to check for negative cycles. Without this step, the algorithm may return incorrect distances.

Do not attempt to reconstruct paths if a negative cycle exists. Distances are undefined in that case.

Do not use Bellman–Ford when all weights are non-negative. Dijkstra is strictly faster in that setting.

Ensure that unreachable vertices remain at infinity. Avoid relaxing edges from vertices with infinite distance.

## Variants

Queue-based optimization (SPFA) processes only vertices whose distances changed. It often performs faster in practice but can degrade to the same worst-case complexity.

Early stopping reduces work when the graph stabilizes quickly.

Edge reordering may improve cache behavior but does not change asymptotic complexity.

## Recipe Pattern

Use Bellman–Ford when negative edge weights are present and correctness must be guaranteed. Relax all edges repeatedly, rely on the bounded path length property, and detect negative cycles through an additional pass.
