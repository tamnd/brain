# 10.9 Multi-Source Shortest Paths

## Problem

You have several starting vertices instead of one. For every vertex in the graph, you want the distance to the nearest source and, often, which source reaches it first.

This appears in problems such as nearest facility search, spreading processes, grid expansion, infection models, fire escape problems, and Voronoi regions on graphs.

## Solution

Initialize all sources with distance `0` and insert all of them into the same frontier. Then run the ordinary shortest path algorithm for the graph class.

For an unweighted graph, use multi-source BFS.

```text id="e1v7kq"
multi_source_bfs(G, sources):
    for each vertex v in G:
        dist[v] = infinity
        origin[v] = none
        parent[v] = none

    queue = empty queue

    for each source s in sources:
        dist[s] = 0
        origin[s] = s
        push s into queue

    while queue is not empty:
        u = pop front of queue

        for each neighbor v of u:
            if dist[v] == infinity:
                dist[v] = dist[u] + 1
                origin[v] = origin[u]
                parent[v] = u
                push v into queue

    return dist, origin, parent
```

For a weighted graph with non-negative weights, use multi-source Dijkstra. Insert all sources into the priority queue with distance `0`.

## Discussion

A multi-source shortest path problem can be reduced to a single-source shortest path problem by adding a new artificial source `q`, then adding a zero-cost edge from `q` to every original source. Running a shortest path algorithm from `q` gives the distance from the nearest source.

In practice, you usually do not need to materialize `q`. Initializing every source with distance `0` has the same effect.

The `origin` table records which source owns each vertex. If `origin[v] = s`, then `s` is one nearest source to `v`. When ties exist, the selected origin depends on queue order or priority queue tie-breaking.

## Correctness

Consider the artificial-source construction. Add a new vertex `q` and edges `(q, s)` of weight `0` for every source `s`.

Any path from `q` to a vertex `v` must first take one of these zero-cost edges to some source `s`, then follow a path from `s` to `v`. Therefore the shortest distance from `q` to `v` is:

```text id="n84ftr"
min over sources s of distance(s, v)
```

The direct initialization method simulates this graph without explicitly storing `q`. Each source begins in the frontier with distance `0`, exactly as if it had been discovered from `q`. Therefore the usual correctness proof for BFS or Dijkstra applies unchanged.

## Python: Multi-Source BFS

```python id="7xeqgk"
from collections import deque

def multi_source_bfs(graph, sources):
    dist = {v: None for v in graph}
    parent = {v: None for v in graph}
    origin = {v: None for v in graph}

    q = deque()

    for s in sources:
        if s not in dist:
            dist[s] = None
            parent[s] = None
            origin[s] = None

        dist[s] = 0
        origin[s] = s
        q.append(s)

    while q:
        u = q.popleft()

        for v in graph.get(u, []):
            if v not in dist:
                dist[v] = None
                parent[v] = None
                origin[v] = None

            if dist[v] is None:
                dist[v] = dist[u] + 1
                parent[v] = u
                origin[v] = origin[u]
                q.append(v)

    return dist, origin, parent
```

## Python: Multi-Source Dijkstra

```python id="cnviit"
import heapq

def multi_source_dijkstra(graph, sources):
    dist = {v: float("inf") for v in graph}
    parent = {v: None for v in graph}
    origin = {v: None for v in graph}

    pq = []

    for s in sources:
        if s not in dist:
            dist[s] = float("inf")
            parent[s] = None
            origin[s] = None

        dist[s] = 0
        origin[s] = s
        heapq.heappush(pq, (0, s))

    while pq:
        d, u = heapq.heappop(pq)

        if d > dist[u]:
            continue

        for v, w in graph.get(u, []):
            if v not in dist:
                dist[v] = float("inf")
                parent[v] = None
                origin[v] = None

            nd = d + w

            if nd < dist[v]:
                dist[v] = nd
                parent[v] = u
                origin[v] = origin[u]
                heapq.heappush(pq, (nd, v))

    return dist, origin, parent
```

## Path Reconstruction

The parent table reconstructs a path from the nearest source to a target.

```python id="q68pn6"
def reconstruct_from_nearest_source(parent, origin, target):
    if target not in origin or origin[target] is None:
        return None

    path = []
    cur = target

    while cur is not None:
        path.append(cur)
        if cur == origin[target]:
            path.reverse()
            return path
        cur = parent[cur]

    return None
```

## Complexity

For an unweighted graph represented with adjacency lists, multi-source BFS runs in:

```text id="3vab06"
O(|V| + |E|)
```

For a graph with non-negative weights and a binary heap, multi-source Dijkstra runs in:

```text id="lnc18b"
O((|V| + |E|) log |V|)
```

The number of sources does not multiply the running time. The algorithms run once, with a larger initial frontier.

## Common Pitfalls

Do not run BFS or Dijkstra separately from each source unless you need distances from every source independently. For nearest-source distances, one multi-source run is enough.

Do not forget that ties are possible. If two sources are equally close, the algorithm chooses one according to processing order unless you define a tie-breaking rule.

Do not use multi-source BFS on weighted graphs with unequal weights. Use multi-source Dijkstra for non-negative weights.

Do not treat the artificial source as part of the final graph unless the problem explicitly requires it.

## Recipe Pattern

When several starting points compete to reach vertices, initialize all sources at distance `0` and run the usual shortest path algorithm once. Keep an `origin` table when the identity of the nearest source matters.
