# 10.1 Unweighted Shortest Paths

## Problem

You have a graph where every edge has the same cost. Given a source vertex `s`, compute the minimum number of edges needed to reach every other vertex.

This is the unweighted shortest path problem. Since each edge has cost `1`, the distance from `s` to a vertex `v` is the length of the shortest path from `s` to `v`, measured by edge count.

## Solution

Use breadth-first search. BFS explores the graph in layers. First it visits vertices at distance `0`, then distance `1`, then distance `2`, and so on.

```text
unweighted_shortest_paths(G, s):
    for each vertex v in G:
        dist[v] = infinity
        parent[v] = none

    dist[s] = 0
    queue = empty queue
    push s into queue

    while queue is not empty:
        u = pop front of queue

        for each neighbor v of u:
            if dist[v] == infinity:
                dist[v] = dist[u] + 1
                parent[v] = u
                push v into queue

    return dist, parent
```

The array `dist` stores the shortest known distance from `s`. The array `parent` stores enough information to reconstruct one shortest path. If `parent[v] = u`, then the chosen shortest path to `v` goes through `u`.

## Why BFS Works

BFS works because the queue preserves layer order. When a vertex `u` is removed from the queue, all vertices with smaller distance have already been processed, and no undiscovered shorter path to `u` can still exist.

The essential invariant is:

> At every step, vertices in the queue are ordered by nondecreasing distance from the source.

The source starts with distance `0`. When BFS processes a vertex at distance `d`, every newly discovered neighbor receives distance `d + 1`. Since the queue is first-in-first-out, all vertices at distance `d` are processed before vertices at distance `d + 1`.

## Correctness

We prove that BFS assigns each reachable vertex its true shortest distance from `s`.

Let `dist[v]` be the value assigned by BFS, and let `delta(s, v)` be the true shortest path length from `s` to `v`.

First, BFS never assigns a distance smaller than the true distance. A vertex receives its distance only when it is discovered from some neighbor `u`. If `dist[u] = d`, then BFS assigns `dist[v] = d + 1`, and the parent pointers define an actual path of length `d + 1` from `s` to `v`. Therefore `delta(s, v) <= dist[v]`.

Second, BFS never assigns a distance larger than the true distance. Consider any shortest path

```text
s = v0, v1, v2, ..., vk = v
```

where `k = delta(s, v)`. BFS discovers `v0` at distance `0`. If BFS discovers `vi` at distance at most `i`, then when `vi` is processed, it examines edge `(vi, vi+1)`. If `vi+1` has not already been discovered, BFS assigns it distance at most `i + 1`. If it has already been discovered, its distance is already no larger. By induction, BFS discovers `v` with distance at most `k`.

Together, these two inequalities give:

```text
dist[v] = delta(s, v)
```

for every reachable vertex `v`.

## Path Reconstruction

The `parent` array reconstructs one shortest path from `s` to a target `t`.

```text
path_to(parent, s, t):
    path = empty list
    cur = t

    while cur is not none:
        append cur to path
        if cur == s:
            break
        cur = parent[cur]

    if last element of path is not s:
        return no path

    reverse path
    return path
```

If `dist[t]` is infinity, then `t` is unreachable from `s`. Otherwise, following parent pointers from `t` back to `s` gives a shortest path.

## Complexity

Let `n = |V|` and `m = |E|`.

| Representation | Time | Space |
|---|---:|---:|
| Adjacency list | `O(n + m)` | `O(n + m)` |
| Adjacency matrix | `O(n^2)` | `O(n^2)` |

With an adjacency list, BFS visits each vertex at most once and scans each edge a constant number of times. This gives linear time in the size of the graph representation.

## Common Mistakes

Do not use DFS for shortest paths in an unweighted graph. DFS may reach a vertex through a long path before seeing a shorter one.

Do not mark a vertex as visited only when it is popped from the queue. Mark it when it is pushed. Otherwise the same vertex can enter the queue many times.

Do not use BFS directly when edge weights differ. BFS assumes every edge has equal cost. For non-negative weighted graphs, use Dijkstra’s algorithm instead.

## Implementation Pattern

A compact implementation usually follows this shape:

```python
from collections import deque

def bfs_shortest_paths(graph, source):
    dist = {v: None for v in graph}
    parent = {v: None for v in graph}

    dist[source] = 0
    q = deque([source])

    while q:
        u = q.popleft()

        for v in graph[u]:
            if dist[v] is None:
                dist[v] = dist[u] + 1
                parent[v] = u
                q.append(v)

    return dist, parent
```

For graph problems, this is the default template for unweighted shortest paths. The queue gives the layer order. The `dist` map prevents repeated discovery. The `parent` map records a concrete shortest path.
