# 10.22 Correctness Proofs

## Problem

You need to prove that a shortest path algorithm returns the true shortest distances, not merely plausible distances.

For shortest path algorithms, correctness usually comes from three ingredients:

1. a definition of the true shortest distance  
2. an invariant maintained by the algorithm  
3. a termination argument showing that all necessary relaxations have occurred  

## Shortest Distance

Let `delta(s, v)` denote the true shortest distance from source `s` to vertex `v`.

If no path exists from `s` to `v`, then:

```text id="f5b8ap"
delta(s, v) = infinity
```

If a negative cycle reachable from `s` can also reach `v`, then no finite shortest distance exists.

The algorithm computes a table `dist[v]`. Correctness means:

```text id="u2nvra"
dist[v] = delta(s, v)
```

for every vertex whose shortest distance is well-defined.

## Upper Bound Lemma

Most shortest path algorithms maintain the same basic lemma.

**Lemma.** At every point in the algorithm, every finite label `dist[v]` is the cost of some actual path from `s` to `v`.

This implies:

```text id="n3cstg"
delta(s, v) <= dist[v]
```

The algorithm never invents a distance smaller than the true optimum, because each label comes from a real path.

The proof is by induction over relaxations. Initially, `dist[s] = 0`, which is the empty path from `s` to itself. Whenever an edge `(u, v, w)` relaxes `v`, the algorithm appends that edge to the path already represented by `dist[u]`.

## Lower Bound Lemma

The other direction depends on the algorithm.

You must prove:

```text id="mq8dfh"
dist[v] <= delta(s, v)
```

Usually this means showing that the algorithm discovers every shortest path in the correct order.

## BFS Correctness

For unweighted graphs, BFS expands vertices by distance layers.

**Invariant.** Before processing layer `k + 1`, all vertices at distance at most `k` have already been discovered with correct distances.

A shortest path of length `k` has the form:

```text id="judt1b"
s = v0, v1, ..., vk = v
```

BFS discovers `v0` at layer `0`, then `v1` at layer `1`, and so on. Therefore `v` is discovered no later than layer `k`.

Combined with the upper bound lemma, BFS distances are exact.

## Dijkstra Correctness

For non-negative weights, Dijkstra uses a label-setting proof.

**Invariant.** When a vertex `u` is extracted from the priority queue with minimum tentative distance, `dist[u]` is final.

Suppose this were false. Then some shorter path to `u` exists. Along that shortest path, let `x` be the first unfinalized vertex, and let `p` be its predecessor. Vertex `p` was already finalized. When `p` was processed, edge `(p, x)` was relaxed. Since all weights are non-negative, the tentative distance to `x` would be no greater than the true shortest distance to `u`. Therefore `x` should have been extracted before `u`, a contradiction.

## Bellman-Ford Correctness

Bellman-Ford uses a path-length induction.

**Invariant.** After `k` full passes over all edges, `dist[v]` is at most the cost of the shortest path from `s` to `v` using at most `k` edges.

If no negative cycle is reachable, every shortest path can be chosen simple, and every simple path has at most `|V| - 1` edges. After `|V| - 1` passes, all shortest paths have been considered.

The extra pass proves whether a negative cycle exists. If an edge can still relax, then an improving path uses at least `|V|` edges and therefore repeats a vertex. The repeated segment must have negative cost.

## Floyd-Warshall Correctness

Floyd-Warshall uses dynamic programming over allowed intermediate vertices.

Define `D_k[i][j]` as the shortest path from `i` to `j` using only vertices from `{1, ..., k}` as intermediate vertices.

The recurrence is:

```text id="au07e3"
D_k[i][j] = min(D_{k-1}[i][j], D_{k-1}[i][k] + D_{k-1}[k][j])
```

A shortest path either avoids vertex `k`, or it passes through `k` and splits into two smaller subpaths. Induction on `k` proves correctness.

## DAG Shortest Paths Correctness

A topological order places every edge before its destination.

**Invariant.** When vertex `u` is processed in topological order, all possible predecessors of `u` have already been processed.

Every path to `u` must end through some predecessor `p`. Since `p` appears earlier, the edge `(p, u)` has already been relaxed. Therefore `dist[u]` is final when `u` is processed.

## General Proof Template

A shortest path proof can often be written in this form:

```text id="zi954r"
1. Show that every finite dist[v] represents a real path.
2. Show that the algorithm processes enough candidate paths.
3. Show that no shorter path remains unprocessed at termination.
4. Conclude dist[v] = delta(s, v).
```

For label-setting algorithms, step 3 is usually proved by a "first unprocessed vertex on a shortest path" contradiction.

For label-correcting algorithms, step 3 is usually proved by induction over path length or by the optimality condition.

## Optimality Condition

A distance table is optimal when every edge satisfies:

```text id="as89hi"
dist[v] <= dist[u] + w(u, v)
```

If this holds and every `dist[v]` represents a real path, then all distances are shortest distances.

This condition is useful for verifying the final output of Bellman-Ford and other relaxation-based algorithms.

## Common Pitfalls

Do not prove only that the algorithm finds some path. Correctness requires the shortest path.

Do not ignore unreachable vertices. They should remain at infinity.

Do not use Dijkstra's proof when negative edges exist.

Do not skip termination. An invariant alone does not prove that the algorithm reaches the desired state.

Do not reconstruct paths from a distance table unless parent pointers were maintained consistently.

## Recipe Pattern

Prove shortest path algorithms by combining the upper bound lemma with an algorithm-specific lower bound argument. Use layer order for BFS, minimum-label extraction for Dijkstra, path-length induction for Bellman-Ford, intermediate-vertex induction for Floyd-Warshall, and topological dependency order for DAGs.
