# 10.7 Johnson Algorithm

## Problem

You need all-pairs shortest paths in a sparse weighted directed graph. Edge weights may be negative, but the graph must not contain a negative cycle.

Floyd-Warshall is simple, but its `O(n^3)` time is often too expensive for sparse graphs. Johnson’s algorithm solves the same all-pairs problem by combining Bellman-Ford with repeated Dijkstra runs.

## Solution

Add a new artificial source vertex `q`. Connect `q` to every vertex with a zero-weight edge. Run Bellman-Ford from `q` to compute a potential value `h[v]` for every vertex.

Then reweight every edge `(u, v, w)` as:

```text id="sn68m7"
w'(u, v) = w + h[u] - h[v]
```

After reweighting, all edge weights are non-negative. Run Dijkstra once from each original vertex using the reweighted edges. Convert distances back to original weights at the end:

```text id="4gkm8n"
dist(u, v) = dist'(u, v) - h[u] + h[v]
```

## Pseudocode

```text id="k9aqpd"
johnson(G):
    add new vertex q
    for each vertex v in G:
        add edge (q, v, 0)

    h = bellman_ford(G with q, q)

    if Bellman-Ford reports a negative cycle:
        report "negative cycle"

    for each edge (u, v, w):
        w2(u, v) = w + h[u] - h[v]

    for each vertex u in G:
        dist2[u] = dijkstra(G with weights w2, u)

        for each vertex v in G:
            dist[u][v] = dist2[u][v] - h[u] + h[v]

    return dist
```

The artificial source is used only to compute potentials. It is removed before running Dijkstra from the real vertices.

## Why Reweighting Works

The reweighted edge cost is chosen so that every path changes by a constant depending only on its endpoints.

For a path

```text id="34nui7"
P = v0 -> v1 -> ... -> vk
```

the reweighted cost is:

```text id="knxucr"
w'(P)
= sum over i of (w(vi, vi+1) + h[vi] - h[vi+1])
= w(P) + h[v0] - h[vk]
```

The middle potential terms cancel. Therefore, for a fixed source `u` and target `v`, every path from `u` to `v` is shifted by the same amount:

```text id="ydd1me"
h[u] - h[v]
```

So reweighting preserves which path is shortest.

## Why Weights Become Non-Negative

Bellman-Ford computes shortest distances `h[v]` from `q`. For every edge `(u, v, w)`, the shortest path estimate satisfies the triangle inequality:

```text id="e92lik"
h[v] <= h[u] + w
```

Rearranging gives:

```text id="801k6l"
w + h[u] - h[v] >= 0
```

So every reweighted edge is non-negative. This allows Dijkstra to be used safely.

## Correctness

Johnson’s algorithm is correct because it preserves shortest paths while making all edge weights non-negative.

First, Bellman-Ford either detects a negative cycle or computes valid shortest path potentials `h`. If it detects a negative cycle, then all-pairs shortest paths are not well-defined.

Second, the reweighting preserves path order between any fixed pair of vertices. Since all paths from `u` to `v` shift by the same constant, a path with minimum original weight also has minimum reweighted weight.

Third, after reweighting, every edge is non-negative. Therefore Dijkstra computes correct shortest paths in the reweighted graph.

Finally, subtracting the potential shift recovers the original distance.

## Python Implementation

```python id="lloyfw"
import heapq

def bellman_ford(vertices, edges, source):
    dist = {v: float("inf") for v in vertices}
    dist[source] = 0

    for _ in range(len(vertices) - 1):
        updated = False

        for u, v, w in edges:
            if dist[u] != float("inf") and dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                updated = True

        if not updated:
            break

    for u, v, w in edges:
        if dist[u] != float("inf") and dist[u] + w < dist[v]:
            return None

    return dist

def dijkstra(vertices, adj, source):
    dist = {v: float("inf") for v in vertices}
    dist[source] = 0

    pq = [(0, source)]

    while pq:
        d, u = heapq.heappop(pq)

        if d > dist[u]:
            continue

        for v, w in adj[u]:
            nd = d + w

            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))

    return dist

def johnson(vertices, edges):
    q = object()

    vertices_with_q = list(vertices) + [q]
    edges_with_q = list(edges)

    for v in vertices:
        edges_with_q.append((q, v, 0))

    h = bellman_ford(vertices_with_q, edges_with_q, q)
    if h is None:
        return None

    adj = {v: [] for v in vertices}

    for u, v, w in edges:
        rw = w + h[u] - h[v]
        adj[u].append((v, rw))

    result = {}

    for u in vertices:
        dist2 = dijkstra(vertices, adj, u)
        result[u] = {}

        for v in vertices:
            if dist2[v] == float("inf"):
                result[u][v] = float("inf")
            else:
                result[u][v] = dist2[v] - h[u] + h[v]

    return result
```

## Complexity

Let `n = |V|` and `m = |E|`.

| Step | Time |
|---|---:|
| Bellman-Ford once | `O(nm)` |
| Reweighting | `O(m)` |
| Dijkstra from every vertex | `O(nm log n)` with binary heap |

Total time:

```text id="1jujgr"
O(nm log n)
```

for sparse graphs, plus the Bellman-Ford preprocessing cost. Space usage is `O(n^2)` for the final all-pairs distance table, or `O(n + m)` if distances are streamed one source at a time.

## When to Use

Use Johnson’s algorithm when the graph is sparse, all-pairs distances are required, and negative edges may appear.

Use Floyd-Warshall when the graph is small or dense.

Use repeated Dijkstra when all weights are already non-negative.

Use Bellman-Ford from one source when only one source matters.

## Common Pitfalls

Do not skip the Bellman-Ford step when negative edges exist. Dijkstra requires non-negative weights.

Do not forget to convert distances back after Dijkstra. Reweighted distances are not the original distances.

Do not include the artificial source `q` in the final result.

Do not use Johnson’s algorithm when a negative cycle exists. Bellman-Ford must reject that case.

Do not assume reweighted edge weights are unique or strictly positive. They are guaranteed to be non-negative, not positive.

## Recipe Pattern

Use Bellman-Ford to compute vertex potentials. Reweight edges so that every edge becomes non-negative. Run Dijkstra from each source. Convert each distance back by removing the endpoint-dependent potential shift.
