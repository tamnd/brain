# 10.18 Grid Shortest Paths

## Problem

You work on a 2D grid. Each cell is a vertex. Moves are allowed to neighboring cells with some cost. You need shortest paths from a source cell to all cells or to a target cell.

This model appears in pathfinding, maps, games, robotics, and image processing.

## Graph Model

Convert the grid into a graph:

- each cell `(i, j)` is a vertex  
- edges connect neighboring cells  

Common neighbor sets:

| Type | Moves |
|---|---|
| 4-direction | up, down, left, right |
| 8-direction | add diagonals |

Edge weights depend on the problem:

- `1` for uniform movement  
- `0/1` for terrain types  
- arbitrary non-negative weights for costs  
- negative weights only if no cycles exist or special handling is used  

## Algorithm Selection

| Grid type | Algorithm |
|---|---|
| uniform cost | BFS |
| weights in `{0,1}` | 0-1 BFS |
| non-negative weights | Dijkstra |
| heuristic available | A* |

## BFS on Grid

For uniform cost:

```python id="l9e2vx"
from collections import deque

def grid_bfs(grid, start):
    n, m = len(grid), len(grid[0])
    dist = [[-1] * m for _ in range(n)]

    q = deque([start])
    dist[start[0]][start[1]] = 0

    dirs = [(1,0), (-1,0), (0,1), (0,-1)]

    while q:
        x, y = q.popleft()

        for dx, dy in dirs:
            nx, ny = x + dx, y + dy

            if 0 <= nx < n and 0 <= ny < m:
                if dist[nx][ny] == -1 and grid[nx][ny] != '#':
                    dist[nx][ny] = dist[x][y] + 1
                    q.append((nx, ny))

    return dist
```

`'#'` represents blocked cells.

## Dijkstra on Grid

For weighted grids:

```python id="f3g7ds"
import heapq

def grid_dijkstra(grid, start):
    n, m = len(grid), len(grid[0])
    dist = [[float("inf")] * m for _ in range(n)]

    pq = [(0, start[0], start[1])]
    dist[start[0]][start[1]] = 0

    dirs = [(1,0), (-1,0), (0,1), (0,-1)]

    while pq:
        d, x, y = heapq.heappop(pq)

        if d > dist[x][y]:
            continue

        for dx, dy in dirs:
            nx, ny = x + dx, y + dy

            if 0 <= nx < n and 0 <= ny < m:
                w = grid[nx][ny]
                nd = d + w

                if nd < dist[nx][ny]:
                    dist[nx][ny] = nd
                    heapq.heappush(pq, (nd, nx, ny))

    return dist
```

Here `grid[nx][ny]` represents movement cost into that cell.

## A* on Grid

For target-based search, A* reduces exploration.

Typical heuristic for 4-direction grids:

```text id="q0w2kd"
h(x, y) = |x - tx| + |y - ty|
```

This is Manhattan distance.

## Optimization Techniques

### Early Exit

Stop when target is reached (for Dijkstra or A*).

### Visited Bitset

Use a compact boolean array instead of sets.

### Flattened Index

Convert `(i, j)` to a single index for faster memory access.

### Direction Encoding

Store directions in arrays to avoid repeated allocation.

## Obstacles and Constraints

- blocked cells must be skipped  
- dynamic obstacles require recomputation or incremental updates  
- weighted terrain may vary widely  

## Complexity

Let grid size be `n x m`.

| Algorithm | Time |
|---|---:|
| BFS | `O(nm)` |
| 0-1 BFS | `O(nm)` |
| Dijkstra | `O(nm log(nm))` |
| A* | depends on heuristic |

Space usage is `O(nm)`.

## Common Pitfalls

Do not revisit cells unnecessarily. Use distance checks instead of separate visited flags when weights exist.

Do not use BFS when weights differ.

Do not ignore boundary checks.

Do not assume grid is small. Memory layout matters for large grids.

Do not use recursive DFS for shortest paths.

## Variants

### Multi-source grid BFS

Initialize multiple starting cells.

### Layered grids

Add extra dimension (time, state, direction).

### Weighted obstacles

Treat obstacles as high-cost cells instead of blocked.

### Teleport edges

Add extra edges between arbitrary cells.

## Recipe Pattern

Model the grid as a graph. Choose BFS, 0-1 BFS, Dijkstra, or A* based on weight structure. Use coordinate-based adjacency, careful boundary checks, and efficient memory layout for performance.
