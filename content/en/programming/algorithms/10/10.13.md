# 10.13 Path Reconstruction

## Problem

A shortest path algorithm often returns only distances. In practice, you usually need the actual path as well: the list of vertices or edges that realizes the shortest distance.

The standard method is to store a predecessor, also called a parent, whenever a vertex distance improves.

## Solution

Maintain a `parent` table during the shortest path algorithm.

When an edge relaxation improves the distance to `v`, record that the best known path to `v` comes through `u`.

```text id="d0rg1v"
if dist[u] + w < dist[v]:
    dist[v] = dist[u] + w
    parent[v] = u
```

After the algorithm finishes, reconstruct the path by walking backward from the target to the source.

```text id="fbq6gq"
reconstruct(parent, s, t):
    path = empty list
    cur = t

    while cur is not none:
        append cur to path

        if cur == s:
            reverse path
            return path

        cur = parent[cur]

    return no path
```

This works for BFS, Dijkstra, Bellman-Ford without negative-cycle contamination, 0-1 BFS, DAG shortest paths, and A*.

## Discussion

The parent table stores a shortest path tree rooted at the source. For each reachable vertex `v`, `parent[v]` points to the previous vertex on one selected shortest path from `s` to `v`.

If several shortest paths exist, the parent table records only one of them. Which one appears depends on implementation details: adjacency order, tie-breaking rules, heap ordering, or whether equal-distance updates are accepted.

For most applications, one shortest path is enough. If the problem needs all shortest paths, the parent structure must store multiple predecessors.

## Python Implementation

```python id="0lhib9"
def reconstruct_path(parent, source, target):
    path = []
    cur = target

    while cur is not None:
        path.append(cur)

        if cur == source:
            path.reverse()
            return path

        cur = parent.get(cur)

    return None
```

This version returns `None` when the target is unreachable from the source.

## Example

```python id="tcztnx"
parent = {
    "s": None,
    "a": "s",
    "b": "a",
    "t": "b",
}

print(reconstruct_path(parent, "s", "t"))
```

Output:

```text id="kvyrqp"
['s', 'a', 'b', 't']
```

The reconstructed path is:

```text id="itf7gs"
s -> a -> b -> t
```

## Edge Paths

Some applications need the edge sequence rather than only the vertex sequence. Store the full predecessor edge.

```text id="6vy3fu"
parent_edge[v] = (u, v, w)
```

Then reconstruction can return both vertices and edge weights.

```python id="ehv4nv"
def reconstruct_edges(parent_edge, source, target):
    edges = []
    cur = target

    while cur != source:
        if cur not in parent_edge or parent_edge[cur] is None:
            return None

        edge = parent_edge[cur]
        edges.append(edge)
        cur = edge[0]

    edges.reverse()
    return edges
```

For the edge table:

```python id="doebl9"
parent_edge = {
    "a": ("s", "a", 3),
    "b": ("a", "b", 4),
    "t": ("b", "t", 2),
}
```

the result is:

```text id="n2fe6n"
[('s', 'a', 3), ('a', 'b', 4), ('b', 't', 2)]
```

## All Shortest Paths

To reconstruct all shortest paths, store every predecessor that preserves optimal distance.

```text id="p5ngue"
if dist[u] + w < dist[v]:
    dist[v] = dist[u] + w
    parents[v] = {u}

else if dist[u] + w == dist[v]:
    add u to parents[v]
```

Then enumerate paths by backtracking.

```python id="as6cwm"
def all_shortest_paths(parents, source, target):
    result = []
    path = []

    def dfs(v):
        path.append(v)

        if v == source:
            result.append(list(reversed(path)))
        else:
            for p in parents.get(v, []):
                dfs(p)

        path.pop()

    dfs(target)
    return result
```

This can be expensive. The number of shortest paths may be exponential in the number of vertices, even when the graph itself is small.

## Detecting Invalid Parent Chains

Parent chains should end at the source. If they enter a cycle or stop early, reconstruction is invalid.

A defensive implementation can track seen vertices.

```python id="4vn5uh"
def reconstruct_path_checked(parent, source, target):
    path = []
    seen = set()
    cur = target

    while cur is not None:
        if cur in seen:
            raise ValueError("parent chain contains a cycle")

        seen.add(cur)
        path.append(cur)

        if cur == source:
            path.reverse()
            return path

        cur = parent.get(cur)

    return None
```

Cycles in a parent chain usually indicate that path reconstruction was attempted after detecting a negative cycle, or that parent updates were performed incorrectly.

## Complexity

Let `L` be the number of vertices in the returned path.

| Operation | Time | Space |
|---|---:|---:|
| One path reconstruction | `O(L)` | `O(L)` |
| Edge path reconstruction | `O(L)` | `O(L)` |
| All shortest paths | output-sensitive | output-sensitive |

The shortest path algorithm itself usually dominates the cost. Reconstruction is linear in the length of the path.

## Common Pitfalls

Do not reconstruct a path to an unreachable target. Check the distance table or allow reconstruction to return `None`.

Do not reconstruct paths through vertices affected by a negative cycle. No shortest path exists for those vertices.

Do not overwrite parents on equal-distance relaxations unless your tie-breaking policy requires it.

Do not store only distances if later stages need routes, edges, or explanations.

Do not assume the parent tree is unique. Shortest distances may be unique while shortest paths are not.

## Recipe Pattern

During every successful relaxation, store the predecessor that produced the improvement. After the algorithm finishes, walk backward from the target to the source, reverse the result, and validate that the chain reaches the source.
