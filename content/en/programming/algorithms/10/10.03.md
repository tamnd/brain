# 10.3 Dijkstra Algorithm

## Problem

You have a graph with non-negative edge weights. Given a source vertex `s`, compute the minimum cost path to every other vertex.

Unlike the unweighted case, distances now depend on accumulated weights rather than edge count. A path with fewer edges may still be more expensive.

## Solution

Use Dijkstra’s algorithm with a priority queue keyed by current best-known distance.

```text id="9w4qzt"
dijkstra(G, s):
    for each vertex v in G:
        dist[v] = infinity
        parent[v] = none

    dist[s] = 0
    pq = empty min-priority queue
    push (0, s) into pq

    while pq is not empty:
        (d, u) = extract-min from pq

        if d > dist[u]:
            continue

        for each edge (u, v, w):
            if dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
                parent[v] = u
                push (dist[v], v) into pq

    return dist, parent
```

Each entry in the queue represents a candidate distance. Outdated entries are ignored when extracted.

## Discussion

Dijkstra’s algorithm generalizes BFS by replacing the FIFO queue with a priority queue. Instead of expanding vertices by layer, it expands them in order of increasing distance.

The algorithm maintains a tentative distance `dist[v]` for each vertex. At each step, it selects the vertex with the smallest tentative distance and attempts to improve distances of its neighbors.

The operation:

```text id="f9jz0j"
dist[v] > dist[u] + w
```

is called relaxation. It checks whether the path through `u` improves the best known path to `v`.

## Invariant

At the moment a vertex `u` is extracted from the priority queue with value `dist[u]`, this value is final. No shorter path to `u` will be discovered later.

This invariant depends on all edge weights being non-negative. If a negative edge exists, a shorter path to `u` could appear after `u` has already been processed.

## Correctness

Let `delta(s, v)` be the true shortest path distance.

First, the algorithm never underestimates distances. Every assigned value corresponds to a real path, constructed by chaining relaxations. Therefore:

```text id="x2p9yr"
delta(s, v) <= dist[v]
```

Second, the algorithm never misses a shorter path. Suppose a vertex `u` is extracted with `dist[u] > delta(s, u)`. Consider a shortest path:

```text id="3l6h7o"
s = v0, v1, ..., vk = u
```

Let `vi` be the first vertex on this path that has not yet been finalized. Then `vi-1` has already been finalized with correct distance. When processing `vi-1`, the algorithm would relax edge `(vi-1, vi)` and assign:

```text id="2z8qls"
dist[vi] <= delta(s, vi)
```

This contradicts the assumption that `u` was extracted with a suboptimal distance. Therefore:

```text id="4q1mht"
dist[v] = delta(s, v)
```

for all vertices reachable from `s`.

## Priority Queue Choices

| Structure | Insert | Extract-min | Decrease-key | Notes |
|---|---:|---:|---:|---|
| Binary heap | `O(log n)` | `O(log n)` | `O(log n)` | Common choice |
| Fibonacci heap | `O(1)` amortized | `O(log n)` | `O(1)` amortized | Theoretical improvement |
| Pairing heap | `O(1)` amortized | `O(log n)` | `O(1)` amortized | Practical compromise |

In practice, many implementations avoid explicit decrease-key and instead push duplicate entries, relying on the `if d > dist[u]` check.

## Complexity

Let `n = |V|` and `m = |E|`.

| Representation | Time |
|---|---:|
| Binary heap | `O((n + m) log n)` |
| Fibonacci heap | `O(n log n + m)` |

Space usage is `O(n + m)`.

## Path Reconstruction

Same as BFS: follow `parent` pointers from target back to source.

```python id="5m3v0r"
def reconstruct(parent, s, t):
    path = []
    cur = t

    while cur is not None:
        path.append(cur)
        if cur == s:
            path.reverse()
            return path
        cur = parent[cur]

    return None
```

## Python Implementation

```python id="h7t1kx"
import heapq

def dijkstra(graph, source):
    dist = {v: float("inf") for v in graph}
    parent = {v: None for v in graph}

    dist[source] = 0
    pq = [(0, source)]

    while pq:
        d, u = heapq.heappop(pq)

        if d > dist[u]:
            continue

        for v, w in graph[u]:
            if dist[v] > dist[u] + w:
                dist[v] = dist[u] + w
                parent[v] = u
                heapq.heappush(pq, (dist[v], v))

    return dist, parent
```

Graph format: `graph[u]` is a list of `(v, weight)` pairs.

## Common Pitfalls

Do not use Dijkstra when negative edge weights exist. The correctness invariant fails.

Do not forget the stale entry check. Without it, the algorithm may process outdated distances and degrade performance.

Do not assume integer weights. The algorithm works for any non-negative real weights, but floating-point precision may affect equality comparisons.

Do not stop early unless you only need a single target. If you only need the distance to `t`, you can stop when `t` is extracted from the queue.

## Variants

Single-target optimization stops when the target is finalized.

Multi-source shortest paths can be computed by inserting multiple sources into the queue with distance `0`.

For small integer weights, bucket-based methods can replace the heap and improve performance.

## Recipe Pattern

Use Dijkstra when all edge weights are non-negative and path cost is the sum of weights. Replace the BFS queue with a priority queue, maintain tentative distances, and relax edges until all distances stabilize.
