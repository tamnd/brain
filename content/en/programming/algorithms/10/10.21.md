# 10.21 Specialized Graph Models

## Problem

General shortest path algorithms assume arbitrary graphs. Many real problems have additional structure that can be exploited to obtain faster or simpler solutions.

You need to recognize these special graph models and apply tailored algorithms instead of generic ones.

## Trees

### Properties

- exactly one simple path between any two vertices  
- no cycles  
- `m = n - 1`  

### Solution

Distances can be computed using simple traversal.

```text
dist(u, v) = depth[u] + depth[v] - 2 * depth[lca(u, v)]
```

Preprocessing with Lowest Common Ancestor enables fast queries.

### Use Case

- hierarchical data  
- file systems  
- spanning trees  

## DAGs

### Properties

- no directed cycles  
- topological ordering exists  

### Solution

Process vertices in topological order and relax edges once.

Time complexity is linear:

```text
O(n + m)
```

### Use Case

- scheduling  
- dependency graphs  
- pipelines  

## Grid Graphs

### Properties

- regular structure  
- local adjacency  
- implicit edges  

### Solution

Use BFS, 0-1 BFS, Dijkstra, or A* depending on weights.

### Optimization

- avoid explicit edge storage  
- compute neighbors on the fly  

## Planar Graphs

### Properties

- edges do not cross  
- `m = O(n)`  

### Techniques

- separator theorems  
- region decomposition  
- specialized shortest path algorithms  

### Use Case

- maps  
- geographic networks  

## Small Integer Weights

### Properties

- weights bounded by small constant  

### Solution

Use bucket-based methods instead of heaps.

Example:

```text
Dial's algorithm
```

Time complexity:

```text
O(m + nC)
```

where `C` is maximum weight.

### Use Case

- road networks with small integer costs  
- discrete systems  

## Unit Capacity or Binary Weights

### Properties

- weights in `{0,1}`  

### Solution

Use 0-1 BFS.

Time complexity:

```text
O(n + m)
```

## Complete Graphs

### Properties

- `m = Θ(n^2)`  

### Solution

Matrix-based methods such as Floyd-Warshall become practical.

### Use Case

- dense similarity graphs  
- clustering  

## Metric Graphs

### Properties

- triangle inequality holds  

### Implication

Heuristic methods like A* become effective.

### Use Case

- geometric routing  
- distance-based optimization  

## Dynamic Graphs

### Properties

- edges or weights change over time  

### Solution

- incremental updates  
- local recomputation  

## Weighted vs Unweighted

| Type | Best Method |
|---|---|
| Unweighted | BFS |
| Non-negative weights | Dijkstra |
| Negative weights | Bellman-Ford |
| Acyclic | Topological |

## Recognition Patterns

Identify structure early:

- check for cycles  
- inspect weight range  
- measure density  
- detect regular patterns  

## Complexity Benefits

Specialized models often reduce complexity:

| Model | General | Specialized |
|---|---|---|
| DAG | `O(m log n)` | `O(n + m)` |
| Tree | `O(m log n)` | `O(n)` preprocessing |
| 0-1 weights | `O(m log n)` | `O(n + m)` |

## Common Pitfalls

Do not treat structured graphs as general graphs. You lose performance.

Do not miss hidden structure. Many problems encode DAGs or trees implicitly.

Do not assume weights require Dijkstra when simpler methods exist.

Do not ignore constraints such as acyclicity or bounded weights.

## Recipe Pattern

Classify the graph before choosing an algorithm. Apply the simplest algorithm that matches the structure. Exploit constraints to reduce time and space complexity.
