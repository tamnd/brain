# 10.6 Floyd–Warshall Algorithm

## Problem

You need shortest path distances between every pair of vertices in a weighted graph. Edge weights may be positive, zero, or negative, but no negative cycle may be present.

Single-source methods such as Dijkstra or Bellman–Ford become inefficient when repeated for all vertices. A direct all-pairs method is preferable for dense graphs.

## Solution

Use the Floyd–Warshall algorithm. It is a dynamic programming method that progressively allows intermediate vertices in paths.

Initialize a distance matrix `dist[i][j]`:

- `0` if `i == j`
- weight of edge `(i, j)` if it exists
- infinity otherwise

Then update:

```text id="7m9b2k"
for k in V:
    for i in V:
        for j in V:
            dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

Each iteration of `k` allows paths that go through vertex `k`.

## Interpretation

The algorithm maintains:

```text id="h0z8ra"
dist[i][j] = shortest path from i to j using only vertices {1, ..., k} as intermediates
```

At step `k`, you decide whether going through `k` improves the path from `i` to `j`.

The recurrence is:

```text id="k6c4yo"
dist_k[i][j] = min(
    dist_{k-1}[i][j],
    dist_{k-1}[i][k] + dist_{k-1}[k][j]
)
```

This expresses a simple choice: either the shortest path does not use `k`, or it uses `k` once as a split point.

## Correctness

We prove by induction on `k`.

Base case `k = 0`: no intermediate vertices are allowed. The matrix correctly stores direct edge weights or infinity.

Inductive step: assume correctness for `k - 1`. At step `k`, any shortest path from `i` to `j` either:

1. Does not use `k`, so its length is `dist_{k-1}[i][j]`, or  
2. Uses `k` as the last intermediate vertex, so it splits into a path from `i` to `k` and from `k` to `j`, both using only vertices `{1, ..., k-1}`  

Therefore the recurrence captures all possibilities.

After processing all vertices, all intermediate vertices are allowed, so `dist[i][j]` equals the true shortest path distance.

## Detecting Negative Cycles

After the algorithm finishes, check the diagonal:

```text id="0y6m1n"
if dist[i][i] < 0 for some i:
    negative cycle exists
```

A negative value on the diagonal means a path from a vertex back to itself has negative cost, which implies a negative cycle.

## Path Reconstruction

Maintain a `next[i][j]` table:

```text id="z9wq8t"
if edge (i, j) exists:
    next[i][j] = j
else:
    next[i][j] = none
```

When updating:

```text id="c3x9ph"
if dist[i][k] + dist[k][j] < dist[i][j]:
    dist[i][j] = dist[i][k] + dist[k][j]
    next[i][j] = next[i][k]
```

To reconstruct a path from `i` to `j`, repeatedly follow `next[i][j]`.

## Python Implementation

```python id="p2y7zs"
def floyd_warshall(vertices, edges):
    dist = {i: {j: float("inf") for j in vertices} for i in vertices}
    nxt = {i: {j: None for j in vertices} for i in vertices}

    for v in vertices:
        dist[v][v] = 0

    for u, v, w in edges:
        dist[u][v] = w
        nxt[u][v] = v

    for k in vertices:
        for i in vertices:
            for j in vertices:
                if dist[i][k] + dist[k][j] < dist[i][j]:
                    dist[i][j] = dist[i][k] + dist[k][j]
                    nxt[i][j] = nxt[i][k]

    for v in vertices:
        if dist[v][v] < 0:
            return None, None  # negative cycle

    return dist, nxt
```

## Reconstructing a Path

```python id="q8y2rk"
def get_path(nxt, i, j):
    if nxt[i][j] is None:
        return None

    path = [i]
    while i != j:
        i = nxt[i][j]
        path.append(i)

    return path
```

## Complexity

Let `n = |V|`.

| Metric | Value |
|---|---:|
| Time | `O(n^3)` |
| Space | `O(n^2)` |

The algorithm is efficient for dense graphs or when all-pairs distances are required. For sparse graphs, repeated single-source algorithms may be preferable.

## Common Pitfalls

Do not use Floyd–Warshall on large sparse graphs. The cubic cost becomes prohibitive.

Do not forget to initialize diagonal entries to zero.

Do not ignore negative cycles. Results are invalid if one exists.

Be careful with overflow when using large weights or infinity values.

Ensure that the order of loops is `k`, then `i`, then `j`. Changing the order breaks correctness.

## When to Use

Use Floyd–Warshall when:

- You need distances between all pairs  
- The graph is small or dense  
- Edge weights may be negative but no negative cycles exist  

## Recipe Pattern

Represent the graph as a matrix. Initialize base distances. Iteratively allow more intermediate vertices. At each step, decide whether routing through a new vertex improves existing paths.
