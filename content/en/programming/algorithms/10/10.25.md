# 10.25 Common Pitfalls and Debugging Patterns

## Problem

Shortest path implementations fail in subtle ways. The algorithms are simple at a high level, but small mistakes in initialization, relaxation, or data structures lead to incorrect results or poor performance.

This section lists recurring failure modes and concrete debugging patterns.

## Initialization Errors

### Symptom

Distances remain incorrect or never update.

### Cause

Incorrect initial values.

```text
dist[s] != 0
dist[v] not set to infinity for v != s
```

### Fix

Always enforce:

```text
dist[s] = 0
dist[v] = infinity for all other v
```

Ensure all vertices are initialized, including those not explicitly listed as keys.

## Missing Relaxation Condition

### Symptom

Distances do not improve when better paths exist.

### Cause

Incorrect comparison:

```text
if dist[u] + w <= dist[v]   (wrong in some contexts)
```

or forgetting the condition entirely.

### Fix

Use strict improvement:

```text
if dist[u] + w < dist[v]
```

This avoids unnecessary updates and maintains correctness.

## Stale Priority Queue Entries

### Symptom

Algorithm runs slowly or produces incorrect intermediate states.

### Cause

Outdated entries remain in the priority queue.

### Fix

Always check:

```text
if current_distance > dist[u]:
    skip
```

This ensures only valid states are processed.

## Wrong Data Structure

### Symptom

Correct algorithm but poor performance.

### Cause

Using BFS where weights differ, or using Dijkstra for unit weights.

### Fix

Match algorithm to constraints:

| Condition | Method |
|---|---|
| unit weights | BFS |
| weights in `{0,1}` | 0-1 BFS |
| non-negative weights | Dijkstra |
| negative weights | Bellman-Ford |

## Incorrect Graph Direction

### Symptom

Paths appear missing or asymmetric.

### Cause

Treating directed graph as undirected or vice versa.

### Fix

Verify edge insertion:

```text
directed: add only (u, v)
undirected: add (u, v) and (v, u)
```

## Overflow and Precision Issues

### Symptom

Distances wrap around or become incorrect.

### Cause

Using fixed-size integers or floating-point errors.

### Fix

- use sufficiently large integer types  
- avoid subtracting large nearly equal floats  
- define infinity carefully  

## Parent Pointer Errors

### Symptom

Distances correct but reconstructed paths incorrect.

### Cause

Incorrect parent updates or overwriting parents on equal distances.

### Fix

Update parent only on strict improvement:

```text
if dist[u] + w < dist[v]:
    parent[v] = u
```

Validate path after reconstruction.

## Negative Weight Misuse

### Symptom

Dijkstra produces incorrect results.

### Cause

Negative edges used with label-setting algorithm.

### Fix

Switch to Bellman-Ford or reweight edges.

## Missing Negative Cycle Handling

### Symptom

Algorithm does not terminate or produces decreasing distances.

### Cause

Negative cycle not detected.

### Fix

After Bellman-Ford:

```text
check for further relaxation
```

Mark affected vertices if needed.

## Early Termination Errors

### Symptom

Algorithm returns suboptimal path.

### Cause

Stopping too early.

### Fix

- BFS: finish layer  
- Dijkstra: stop only when target is extracted  
- A*: ensure heuristic is admissible  

## Boundary and Index Errors

### Symptom

Grid or array-based graph fails.

### Cause

Out-of-bounds access or missing neighbors.

### Fix

Always check:

```text
0 <= x < n and 0 <= y < m
```

before accessing grid.

## Debugging Patterns

### Minimal Counterexample

Reduce failing input to smallest graph that reproduces the bug.

### Trace Execution

Log relaxations:

```text
u -> v, old dist, new dist
```

### Verify Invariants

After completion:

```text
dist[v] <= dist[u] + w
```

for all edges.

### Cross-Check

Compare with a slower algorithm:

- Dijkstra vs Bellman-Ford  
- BFS vs brute force  

### Stepwise Execution

Run on very small graph and inspect each iteration manually.

## Performance Debugging

### Symptom

Algorithm is too slow.

### Causes

- repeated relaxations  
- poor data structure  
- unnecessary work  

### Fix

- use appropriate queue structure  
- avoid duplicate processing  
- reduce graph size if possible  

## Checklist

Before finalizing an implementation:

- initialization correct  
- relaxation condition correct  
- data structure appropriate  
- graph representation correct  
- negative weights handled  
- parent pointers valid  
- invariants verified  

## Recipe Pattern

When debugging, isolate the failure with a minimal example, trace relaxations, verify invariants, and compare against a trusted baseline. Fix the root cause, not just the symptom.
