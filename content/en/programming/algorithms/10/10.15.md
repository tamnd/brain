# 10.15 Potentials and Reweighting

## Problem

Shortest path algorithms often fail or become inefficient when edge weights include negative values. You need a systematic way to transform weights so that efficient algorithms such as Dijkstra can be applied without changing the set of shortest paths.

## Core Idea

Introduce a potential function `h : V -> R` on vertices and reweight every edge `(u, v, w)` as:

```text id="k4m1dp"
w'(u, v) = w + h[u] - h[v]
```

This transformation shifts edge weights while preserving the relative cost of all paths between any fixed pair of vertices.

## Path Invariance

For a path

```text id="a7t9zx"
P = v0 -> v1 -> ... -> vk
```

the reweighted cost is:

```text id="u5y3lb"
w'(P)
= sum (w(vi, vi+1) + h[vi] - h[vi+1])
= w(P) + h[v0] - h[vk]
```

All intermediate potential terms cancel. Therefore, for any fixed endpoints `s` and `t`:

```text id="r2xq6n"
w'(P) = w(P) + h[s] - h[t]
```

Every path from `s` to `t` shifts by the same constant. Hence the shortest path under `w` remains the shortest under `w'`.

## Goal of Reweighting

Choose `h` such that all reweighted edges are non-negative:

```text id="j1n8bw"
w'(u, v) >= 0
```

This condition is equivalent to:

```text id="9x6kcf"
h[v] <= h[u] + w
```

for every edge `(u, v, w)`.

This is exactly the triangle inequality for shortest path distances.

## Constructing Potentials

If the graph has no negative cycles, compute potentials using Bellman-Ford.

Add a new vertex `q` with edges `(q, v, 0)` to all vertices. Run Bellman-Ford from `q` to obtain:

```text id="0c8lqa"
h[v] = distance from q to v
```

These distances satisfy:

```text id="6q3m1k"
h[v] <= h[u] + w
```

so all reweighted edges are non-negative.

## Interpretation

The potential function assigns a scalar value to each vertex. Reweighting adjusts each edge based on the difference of potentials at its endpoints.

This can be viewed as shifting the reference frame of distances. The cost of a path depends only on the difference in potentials between its endpoints.

## Reduced Costs

The reweighted value `w'(u, v)` is called the reduced cost. Reduced costs are widely used in optimization and flow algorithms.

Properties:

- Reduced costs are non-negative  
- Reduced costs preserve shortest paths  
- Reduced costs reflect slack in constraints  

## Example

Consider edges:

```text id="7g0xmt"
(a, b, 3)
(b, c, -2)
(a, c, 4)
```

Suppose Bellman-Ford yields:

```text id="p3w9eh"
h[a] = 0
h[b] = 3
h[c] = 1
```

Then reweighted edges:

```text id="8d2xla"
(a, b): 3 + 0 - 3 = 0
(b, c): -2 + 3 - 1 = 0
(a, c): 4 + 0 - 1 = 3
```

All edges are now non-negative.

## Python Snippet

```python id="9m5v8d"
def reweight_edges(edges, h):
    new_edges = []

    for u, v, w in edges:
        w2 = w + h[u] - h[v]
        if w2 < 0:
            raise ValueError("reweighting failed")
        new_edges.append((u, v, w2))

    return new_edges
```

## Applications

### Johnson’s Algorithm

Uses potentials to convert a graph with negative edges into one with non-negative edges, enabling Dijkstra for all-pairs shortest paths.

### Min-Cost Flow

Reduced costs guide augmenting path selection and maintain optimality conditions.

### Dual Interpretation

Potentials correspond to dual variables in linear programming formulations of shortest path problems.

## Correctness

Let `delta(s, v)` be the original shortest path distance, and let `delta'(s, v)` be the reweighted distance.

For any path:

```text id="w9y2ls"
delta'(s, v) = delta(s, v) + h[s] - h[v]
```

Since the shift depends only on endpoints, minimizing `delta'(s, v)` is equivalent to minimizing `delta(s, v)`.

## Complexity Impact

Reweighting itself costs `O(m)`. The main cost lies in computing potentials via Bellman-Ford:

```text id="q8n4dp"
O(n * m)
```

After that, faster algorithms such as Dijkstra can be used.

## Common Pitfalls

Do not use arbitrary potentials. They must satisfy the triangle inequality condition.

Do not apply reweighting if a negative cycle exists. Potentials are undefined in that case.

Do not forget to convert distances back after running algorithms on reweighted edges.

Do not assume reweighted edges are strictly positive. They are only guaranteed to be non-negative.

## Recipe Pattern

Compute vertex potentials using shortest paths from an artificial source. Reweight edges using potential differences. Run efficient non-negative shortest path algorithms. Adjust final distances to recover original values.
