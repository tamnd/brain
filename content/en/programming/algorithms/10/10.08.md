# 10.8 DAG Shortest Paths

## Problem

You have a weighted directed acyclic graph, usually called a DAG. Given a source vertex `s`, compute the shortest path distance from `s` to every other vertex.

Edge weights may be positive, zero, or negative. Because the graph has no directed cycles, negative cycles cannot exist. This makes shortest paths well-defined even when some edges have negative weight.

## Solution

Process vertices in topological order. A topological order lists vertices so that every directed edge goes from an earlier vertex to a later vertex.

```text id="3kq7ma"
dag_shortest_paths(G, s):
    order = topological_sort(G)

    for each vertex v in G:
        dist[v] = infinity
        parent[v] = none

    dist[s] = 0

    for each vertex u in order:
        if dist[u] == infinity:
            continue

        for each edge (u, v, w):
            if dist[u] + w < dist[v]:
                dist[v] = dist[u] + w
                parent[v] = u

    return dist, parent
```

The algorithm relaxes each edge exactly once, but in an order that guarantees all possible predecessors of a vertex have already been considered before the vertex is finalized.

## Discussion

A DAG removes the main difficulty of shortest paths with negative weights. In a general graph, negative edges force repeated relaxation because a better route may return through a cycle and improve an earlier vertex. In a DAG, no directed path can return to an earlier vertex in topological order. Information only flows forward.

This gives a simple rule: compute distances in dependency order. If an edge points from `u` to `v`, then `u` must be processed before `v`. By the time the algorithm reaches `v`, all paths that could end at `v` have already had the chance to relax into it.

## Topological Sort

A topological sort can be computed with DFS or with indegrees. The indegree method is often easier to implement iteratively.

```python id="p7wz19"
from collections import deque

def topological_sort(vertices, graph):
    indeg = {v: 0 for v in vertices}

    for u in vertices:
        for v, _ in graph[u]:
            indeg[v] += 1

    q = deque([v for v in vertices if indeg[v] == 0])
    order = []

    while q:
        u = q.popleft()
        order.append(u)

        for v, _ in graph[u]:
            indeg[v] -= 1
            if indeg[v] == 0:
                q.append(v)

    if len(order) != len(vertices):
        return None

    return order
```

If the returned order is `None`, the graph contains a directed cycle and the DAG shortest path algorithm does not apply.

## Python Implementation

```python id="1g7s2a"
from collections import deque

def dag_shortest_paths(vertices, graph, source):
    """
    vertices:
        Iterable of vertices.

    graph:
        Dictionary mapping u to a list of (v, weight) edges.

    source:
        Source vertex.

    Returns:
        (dist, parent), or None if the graph is not acyclic.
    """
    indeg = {v: 0 for v in vertices}

    for u in vertices:
        for v, _ in graph.get(u, []):
            indeg[v] += 1

    q = deque([v for v in vertices if indeg[v] == 0])
    order = []

    while q:
        u = q.popleft()
        order.append(u)

        for v, _ in graph.get(u, []):
            indeg[v] -= 1
            if indeg[v] == 0:
                q.append(v)

    if len(order) != len(vertices):
        return None

    dist = {v: float("inf") for v in vertices}
    parent = {v: None for v in vertices}

    dist[source] = 0

    for u in order:
        if dist[u] == float("inf"):
            continue

        for v, w in graph.get(u, []):
            nd = dist[u] + w
            if nd < dist[v]:
                dist[v] = nd
                parent[v] = u

    return dist, parent
```

## Correctness

Let the vertices be processed in topological order. We prove that when a vertex `u` is processed, `dist[u]` is already the true shortest distance from `s` to `u`.

The source starts with distance `0`, and every other vertex starts with infinity. Consider any vertex `u`. Every path from `s` to `u` must end with some edge `(p, u)`, unless `u = s`. Since the graph is a DAG and the order is topological, every predecessor `p` of `u` appears before `u` in the order.

By induction, when each predecessor `p` was processed, `dist[p]` was correct. The algorithm relaxed edge `(p, u)`, so `dist[u]` received the best value among all paths that end at `u`. Therefore, when `u` is reached in the order, no future vertex can improve it, because future vertices cannot have edges back to `u`.

Thus each reachable vertex receives its true shortest distance, and unreachable vertices remain at infinity.

## Complexity

Let `n = |V|` and `m = |E|`.

| Step | Time | Space |
|---|---:|---:|
| Topological sort | `O(n + m)` | `O(n)` |
| Edge relaxation | `O(n + m)` | `O(n)` |
| Total | `O(n + m)` | `O(n)` |

The graph storage itself requires `O(n + m)` space.

## Path Reconstruction

The parent table works the same way as in BFS and Dijkstra.

```python id="ygj8c0"
def reconstruct_path(parent, source, target):
    path = []
    cur = target

    while cur is not None:
        path.append(cur)
        if cur == source:
            path.reverse()
            return path
        cur = parent[cur]

    return None
```

If the target is unreachable, the function returns `None`.

## Example

```python id="0fs6o9"
vertices = ["s", "a", "b", "c", "t"]

graph = {
    "s": [("a", 2), ("b", 5)],
    "a": [("c", -4)],
    "b": [("c", 1)],
    "c": [("t", 3)],
    "t": [],
}

result = dag_shortest_paths(vertices, graph, "s")

if result is not None:
    dist, parent = result
    print(dist["t"])
    print(reconstruct_path(parent, "s", "t"))
```

The shortest path is:

```text id="p14n8k"
s -> a -> c -> t
```

with cost:

```text id="aa87fn"
2 + (-4) + 3 = 1
```

## Common Pitfalls

Do not use this algorithm on a graph with cycles. Topological order exists only for directed acyclic graphs.

Do not reject negative edge weights. Negative weights are allowed because cycles are impossible.

Do not assume the source appears first in topological order. Vertices before the source are simply skipped because their distance remains infinity.

Do not use a priority queue. The topological order already provides the correct processing order.

## Recipe Pattern

Use DAG shortest paths when the graph is directed, acyclic, and weighted. Compute a topological order, initialize the source, then relax outgoing edges in that order. This gives single-source shortest paths in linear time, even with negative edge weights.
