# 8.9 Rerooting Technique

## Problem

You need to compute a value for every node where the value depends on the entire tree, not just the subtree rooted at that node. A direct DFS per node gives `O(n^2)` time, which is too slow.

Typical examples:
- Sum of distances from each node to all others  
- Number of nodes at even distance  
- DP values that depend on both subtree and outside-subtree contributions  

## Solution

Use rerooting (also called DP on trees with two passes).

1. First DFS computes values for a fixed root (usually `0`).  
2. Second DFS propagates information from parent to child, effectively “moving the root”.

### Example: Sum of Distances

Let:
- `subtree_size[u]` = number of nodes in subtree of `u`  
- `subtree_sum[u]` = sum of distances from `u` to nodes in its subtree  

First pass:

```python id="b2u7sy"
subtree_size = [1] * n
subtree_sum = [0] * n

def dfs1(u, parent):
    for v in g[u]:
        if v == parent:
            continue
        dfs1(v, u)
        subtree_size[u] += subtree_size[v]
        subtree_sum[u] += subtree_sum[v] + subtree_size[v]

dfs1(0, -1)
```

Now `subtree_sum[0]` is the sum of distances from root `0` to all nodes.

Second pass computes result for all nodes:

```python id="k2qz7f"
result = [0] * n

def dfs2(u, parent):
    result[u] = subtree_sum[u]

    for v in g[u]:
        if v == parent:
            continue

        # save state
        su, sv = subtree_size[u], subtree_size[v]
        du, dv = subtree_sum[u], subtree_sum[v]

        # remove v contribution from u
        subtree_size[u] -= subtree_size[v]
        subtree_sum[u] -= subtree_sum[v] + subtree_size[v]

        # add u contribution to v
        subtree_size[v] += subtree_size[u]
        subtree_sum[v] += subtree_sum[u] + subtree_size[u]

        dfs2(v, u)

        # restore state
        subtree_size[u], subtree_size[v] = su, sv
        subtree_sum[u], subtree_sum[v] = du, dv

dfs2(0, -1)
```

## Key Transition

When moving root from `u` to child `v`:

- Nodes in `v` subtree get closer by `1`  
- All other nodes get farther by `1`  

This transformation is handled by rebalancing subtree sizes and sums.

## Discussion

Rerooting avoids recomputation by reusing results from one root to derive results for neighboring roots. The idea is to maintain a DP state that can be updated when the root moves along an edge.

The structure is always:

- Bottom-up pass to gather subtree information  
- Top-down pass to propagate external contributions  

This pattern appears in many problems where each node needs a global answer.

## General Template

Define two DP values:

- `dp_down[u]`: computed from children  
- `dp_all[u]`: final answer including parent contribution  

First pass:

```python id="7v0zjq"
def dfs1(u, parent):
    for v in g[u]:
        if v == parent:
            continue
        dfs1(v, u)
        dp_down[u] = merge(dp_down[u], transform(dp_down[v]))
```

Second pass:

```python id="9j7d2g"
def dfs2(u, parent, acc_from_parent):
    dp_all[u] = combine(dp_down[u], acc_from_parent)

    for v in g[u]:
        if v == parent:
            continue
        new_acc = compute_new_acc(u, v, acc_from_parent)
        dfs2(v, u, new_acc)
```

The challenge lies in defining:
- `merge`
- `transform`
- `combine`
- `compute_new_acc`

These depend on the problem.

## Correctness

The first DFS computes correct values assuming the tree is rooted at `0`. The second DFS ensures that every node receives contributions from nodes outside its subtree.

For any edge `(u, v)`, rerooting correctly redistributes contributions because:

- The tree is partitioned into two parts: subtree of `v` and the rest  
- The transformation adjusts values to reflect the change in root  

By induction over edges, all nodes receive correct global values.

## Complexity

Each edge is processed a constant number of times in both passes.

Total time: `O(n)`

Space: `O(n)`

## Common Pitfalls

Forgetting to restore state after recursion leads to incorrect results.

Mixing up subtree and global values causes double counting.

Incorrect transition formulas when moving root leads to subtle errors.

Not separating “child contribution” and “parent contribution” clearly makes code hard to debug.

## Recipe Summary

Compute subtree DP, then reroot by pushing information from parent to child. Maintain reversible transitions so that state can be restored after recursion.
