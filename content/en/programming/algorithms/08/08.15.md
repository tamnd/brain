# 8.15 Heavy-Light Decomposition

## Problem

You need to answer queries on paths in a tree, such as:

- Sum of values along a path  
- Maximum value on a path  
- Update values on a path  

A naive approach processes each query in `O(n)` time. This is too slow when there are many queries.

## Solution

Decompose the tree into chains so that any path can be represented as a small number of contiguous segments. Then use a segment tree or Fenwick tree on top of the flattened structure.

## Step 1. Compute Sizes and Heavy Child

```python
subtree_size = [0] * n
heavy = [-1] * n
parent = [-1] * n
depth = [0] * n

def dfs(u, p):
    parent[u] = p
    subtree_size[u] = 1
    max_size = 0

    for v in g[u]:
        if v == p:
            continue
        depth[v] = depth[u] + 1
        dfs(v, u)
        subtree_size[u] += subtree_size[v]

        if subtree_size[v] > max_size:
            max_size = subtree_size[v]
            heavy[u] = v

dfs(0, -1)
```

The heavy child of a node is the child with the largest subtree.

## Step 2. Decompose into Chains

```python
head = [0] * n
pos = [0] * n
cur_pos = 0

def decompose(u, h):
    global cur_pos
    head[u] = h
    pos[u] = cur_pos
    cur_pos += 1

    if heavy[u] != -1:
        decompose(heavy[u], h)

    for v in g[u]:
        if v == parent[u] or v == heavy[u]:
            continue
        decompose(v, v)

decompose(0, 0)
```

Each node is assigned:
- `head[u]`: top of its chain  
- `pos[u]`: position in the flattened array  

## Step 3. Build Segment Tree

```python
flat = [0] * n
for u in range(n):
    flat[pos[u]] = value[u]

seg = SegmentTree(flat)
```

## Step 4. Path Query

```python
def query(u, v):
    res = 0

    while head[u] != head[v]:
        if depth[head[u]] < depth[head[v]]:
            u, v = v, u

        res += seg.query(pos[head[u]], pos[u])
        u = parent[head[u]]

    if depth[u] > depth[v]:
        u, v = v, u

    res += seg.query(pos[u], pos[v])
    return res
```

## Discussion

Heavy-light decomposition splits the tree into paths such that each node belongs to exactly one chain. The heavy edges form long chains, and light edges connect different chains.

The key property:

```text
Any root-to-node path crosses at most O(log n) light edges
```

This implies that any path between two nodes can be broken into `O(log n)` segments.

Each segment corresponds to a contiguous range in the flattened array, so segment tree queries apply directly.

## Updates

To update a node value:

```python
def update(u, new_value):
    seg.update(pos[u], new_value)
```

To update a path, apply updates to each segment in the same way as queries.

## Correctness

Each node belongs to exactly one chain. Within a chain, nodes are contiguous in the flattened array.

When querying a path between `u` and `v`, the algorithm climbs the tree chain by chain. At each step:

- The deeper chain is processed first  
- The segment from `head[u]` to `u` is queried  
- `u` moves to the parent of the chain head  

Eventually both nodes lie in the same chain. The remaining segment is contiguous.

Because chains partition the tree and the algorithm processes all segments exactly once, the result is correct.

## Complexity

Preprocessing:

- DFS: `O(n)`  
- Decomposition: `O(n)`  
- Segment tree build: `O(n)`  

Query:

- At most `O(log n)` segments  
- Each segment query: `O(log n)`  

Total per query: `O(log^2 n)`

With optimizations (Fenwick tree or careful merging), it can be reduced to `O(log n)`.

## Common Pitfalls

Forgetting to process the heavy child first breaks contiguity.

Incorrect handling of `head` comparisons leads to infinite loops.

Mixing up `pos[u]` and original node index causes incorrect queries.

Not handling the final segment when `u` and `v` are in the same chain.

Using wrong segment tree intervals leads to off-by-one errors.

## Recipe Summary

Decompose the tree into heavy chains, map nodes to a linear array, and use a segment tree to answer path queries by splitting paths into logarithmic segments.
