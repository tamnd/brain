# 8.25 Case Studies

## Problem

You need complete solutions that combine multiple tree techniques. These examples emphasize composition: selecting the right representation, then applying the appropriate data structure or traversal.

## Case Study 1. K-th Smallest Value on a Path

Given a tree with values on nodes, answer:

```
```
kth u v k
```
```

Return the k-th smallest value on the path from `u` to `v`.

### Approach

Use:

- Coordinate compression on values
- Persistent segment trees on root-to-node paths
- LCA for path splitting

### Preprocessing

Build a persistent segment tree for each node that stores frequency counts from root to that node.

```
Python

```
# root[u] stores frequency structure for path root -> u  
root[u] = update(root[parent[u]], value[u])
```
```

### Query

Let:

- `ru = root[u]`
- `rv = root[v]`
- `rl = root[lca(u, v)]`
- `rpl = root[parent[lca(u, v)]]`

Then the frequency in path `(u, v)` is:

```
```
freq = ru + rv - rl - rpl
```
```

Use binary search on the segment tree to find k-th.

### Complexity

- Preprocessing: O(n log n)
- Query: O(log n)

## Case Study 2. Path Sum with Updates

Operations:

```
```
update u x  
query u v   sum along path
```
```

### Approach

Use heavy-light decomposition.

### Steps

1. Decompose tree into chains
2. Build segment tree on flattened array
3. Query path by splitting into chain segments

```
Python

```
while head[u] != head[v]:  
    if depth[head[u]] < depth[head[v]]:  
        swap  
    answer += seg.query(pos[head[u]], pos[u])  
    u = parent[head[u]]
```
```

Final segment handled separately.

### Complexity

- Preprocessing: O(n)
- Query: O(log^2 n)
- Update: O(log n)

## Case Study 3. Number of Nodes Within Distance K

For each node, count nodes within distance `K`.

### Approach

Use centroid decomposition.

### Steps

1. Choose centroid
2. Compute distances from centroid
3. Count pairs using sorted distance arrays
4. Recurse on subtrees

### Key Insight

Each path is counted exactly once at its highest centroid.

### Complexity

- O(n log n)

## Case Study 4. Dynamic Tree Coloring

Operations:

```
```
paint u  
query u   distance to nearest painted node
```
```

### Approach

Centroid decomposition.

Maintain:

```
Python

```
best[c] = minimum distance from centroid c to any painted node
```
```

### Query

```
Python

```
ans = INF  
x = u  
  
while x != -1:  
    ans = min(ans, best[x] + dist(u, x))  
    x = centroid_parent[x]
```
```

### Update

```
Python

```
x = u  
while x != -1:  
    best[x] = min(best[x], dist(u, x))  
    x = centroid_parent[x]
```
```

### Complexity

- Update: O(log n)
- Query: O(log n)

## Discussion

Each case follows a consistent pattern:

| Problem Type | Technique |
| --- | --- |
| Subtree queries | Euler tour |
| Path queries | Heavy-light decomposition |
| Order statistics | Persistent segment tree |
| Global distance constraints | Centroid decomposition |
| Dynamic subtree updates | Lazy segment tree |

The core strategy:

1. Transform tree into a structure (array, chains, hierarchy)
2. Apply appropriate data structure
3. Combine results carefully

## Correctness Notes

Each transformation preserves structure:

- Euler tour preserves subtree contiguity
- HLD preserves path decomposition
- Persistent trees preserve historical state
- Centroid decomposition partitions paths uniquely

Correctness follows from these invariants.

## Complexity Summary

| Technique | Build | Query | Update |
| --- | --- | --- | --- |
| Euler + Fenwick | O(n) | O(log n) | O(log n) |
| HLD + Segment Tree | O(n) | O(log^2 n) | O(log n) |
| Persistent Tree | O(n log n) | O(log n) | O(log n) |
| Centroid Decomposition | O(n log n) | O(log n) | O(log n) |

## Common Pitfalls

Combining techniques incorrectly:

- Using wrong indices after flattening
- Forgetting LCA in path queries
- Double counting in centroid decomposition
- Incorrect version usage in persistent trees

## Recipe Summary

Complex tree problems are solved by combining transformations and data structures. Identify the query shape, apply the correct decomposition, and use efficient structures on top.
