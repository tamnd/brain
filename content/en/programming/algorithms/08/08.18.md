# 8.18 Expression Trees

## Problem

You need to represent an expression so that it can be evaluated, transformed, simplified, printed, or compiled. A flat string is hard to analyze because operator precedence, nesting, and subexpression structure are implicit. An expression tree makes the structure explicit.

Typical uses include:

- Arithmetic evaluation
- Symbolic simplification
- Parsing expressions
- Compilers and interpreters
- Formula rewriting
- Query planners

## Solution

Represent each expression as a node. Leaf nodes store constants or variables. Internal nodes store operators and point to child expressions.

```python id="8br06t"
class Expr:
    pass

class Const(Expr):
    def __init__(self, value):
        self.value = value

class Var(Expr):
    def __init__(self, name):
        self.name = name

class BinOp(Expr):
    def __init__(self, op, left, right):
        self.op = op
        self.left = left
        self.right = right
```

The expression:

```text id="3u3ht8"
x + 2 * y
```

is represented as:

```python id="v1jub4"
expr = BinOp(
    "+",
    Var("x"),
    BinOp("*", Const(2), Var("y")),
)
```

The multiplication node is a child of the addition node, which captures precedence directly.

## Evaluating an Expression Tree

Evaluation is a postorder traversal. First evaluate children, then apply the operator.

```python id="n552ry"
def eval_expr(expr, env):
    if isinstance(expr, Const):
        return expr.value

    if isinstance(expr, Var):
        return env[expr.name]

    if isinstance(expr, BinOp):
        left = eval_expr(expr.left, env)
        right = eval_expr(expr.right, env)

        if expr.op == "+":
            return left + right
        if expr.op == "-":
            return left - right
        if expr.op == "*":
            return left * right
        if expr.op == "/":
            return left / right

        raise ValueError("unknown operator")

    raise TypeError("unknown expression")
```

Usage:

```python id="yf0g5q"
env = {"x": 3, "y": 4}
print(eval_expr(expr, env))  # 11
```

## Printing an Expression Tree

Printing requires care because parentheses depend on precedence.

```python id="xbvmv9"
precedence = {
    "+": 1,
    "-": 1,
    "*": 2,
    "/": 2,
}

def print_expr(expr, parent_prec=0):
    if isinstance(expr, Const):
        return str(expr.value)

    if isinstance(expr, Var):
        return expr.name

    if isinstance(expr, BinOp):
        prec = precedence[expr.op]
        left = print_expr(expr.left, prec)
        right = print_expr(expr.right, prec + 1)
        text = f"{left} {expr.op} {right}"

        if prec < parent_prec:
            return f"({text})"
        return text

    raise TypeError("unknown expression")
```

This prints compact output while preserving meaning.

## Simplification

Expression trees support local rewrite rules. For example:

```text id="uzj786"
x + 0 = x
0 + x = x
x * 1 = x
1 * x = x
x * 0 = 0
0 * x = 0
```

```python id="dot2tk"
def simplify(expr):
    if isinstance(expr, (Const, Var)):
        return expr

    if isinstance(expr, BinOp):
        left = simplify(expr.left)
        right = simplify(expr.right)

        if expr.op == "+":
            if isinstance(right, Const) and right.value == 0:
                return left
            if isinstance(left, Const) and left.value == 0:
                return right

        if expr.op == "*":
            if isinstance(right, Const) and right.value == 1:
                return left
            if isinstance(left, Const) and left.value == 1:
                return right
            if isinstance(right, Const) and right.value == 0:
                return Const(0)
            if isinstance(left, Const) and left.value == 0:
                return Const(0)

        return BinOp(expr.op, left, right)

    raise TypeError("unknown expression")
```

Simplification is also a postorder traversal. Children are simplified first, then the parent is rewritten.

## Discussion

Expression trees are abstract syntax trees for expressions. They remove ambiguity from textual notation. Once parsed, the expression is no longer a string with precedence rules. It is a tree whose shape determines evaluation order.

The core invariant is:

```text id="0mj5kq"
Each node represents the expression formed from its children.
```

For a constant or variable, the node is already complete. For an operator node, the meaning comes from recursively combining the meanings of its children.

This recursive structure makes expression trees suitable for interpreters, compilers, symbolic algebra, and optimization passes. Each pass is usually a traversal that either computes a value or returns a transformed tree.

## Correctness

Evaluation is correct by structural induction on the expression tree.

For a constant, the evaluator returns its stored value. For a variable, the evaluator returns its value from the environment. For a binary operator, assume recursive calls correctly evaluate the left and right subexpressions. The evaluator then applies the operator to those two values, which is exactly the meaning of the full expression.

Simplification is correct when each rewrite rule preserves meaning. For example, replacing `x + 0` by `x` is valid because adding zero does not change a value. Since simplification first preserves correctness for children and then applies meaning-preserving local rewrites, the final tree has the same value as the original tree for every environment.

## Complexity

Let `n` be the number of nodes.

| Operation | Time | Space |
|---|---:|---:|
| Evaluation | O(n) | O(h) recursion stack |
| Printing | O(n) plus output size | O(h) recursion stack |
| Simplification | O(n) | O(n) for rebuilt tree |

Here `h` is the height of the expression tree.

## Common Pitfalls

Do not evaluate a variable without checking that it exists in the environment.

Do not print expressions without considering precedence. Otherwise output may change meaning.

Do not mutate shared subtrees during simplification unless aliasing is intentional.

Do not assume all binary operators are associative. Rewriting `(a - b) - c` as `a - (b - c)` changes the result.

Do not store raw substrings in tree nodes when indexes or interned tokens are enough for large inputs.

## Recipe Summary

Use expression trees to make expression structure explicit. Evaluate and simplify them with postorder traversal. Print them with precedence-aware formatting. Treat each transformation as a local rewrite whose correctness follows from the correctness of its rewrite rules.
