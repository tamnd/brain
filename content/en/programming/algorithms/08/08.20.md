# 8.20 Testing Tree Algorithms

## Problem

Tree algorithms often look simple but fail on edge cases such as degenerate trees, incorrect parent handling, or off-by-one errors in flattened representations. You need a systematic way to verify correctness.

## Solution

Test with structured cases, randomized generators, and invariant checks.

## Basic Test Cases

Start with minimal trees.

```text
Single node
0
```

```text
Chain (path)
0 - 1 - 2 - 3 - 4
```

```text
Star
    0
  / | \\n 1  2  3
```

```text
Balanced binary tree
        0
      /   \\n     1     2
    / \   / \\n   3  4  5  6
```

These shapes expose different failure modes.

## Edge Cases

Check:

- `n = 1`  
- Deep tree (linked list shape)  
- Wide tree (large branching factor)  
- Root at different positions  
- Zero or negative values  
- Duplicate values  

## Invariant Checks

After each algorithm, verify invariants.

### DFS

```python
assert len(visited) == n
```

### Subtree Size

```python
assert subtree_size[u] >= 1
assert sum(subtree_size[v] for v in children[u]) + 1 == subtree_size[u]
```

### Euler Tour

```python
assert tin[u] <= tout[u]
assert len(set(tin)) == n
```

### LCA

```python
l = lca(u, v)
assert is_ancestor(l, u)
assert is_ancestor(l, v)
```

### Segment Tree

Compare against brute force:

```python
assert seg.query(l, r) == sum(arr[l:r+1])
```

## Random Testing

Generate random trees and compare results with a slow reference implementation.

### Random Tree Generator

```python
import random

def random_tree(n):
    edges = []
    for i in range(1, n):
        p = random.randint(0, i - 1)
        edges.append((p, i))
    return edges
```

### Brute Force Distance

```python
def brute_dist(u, v, parent):
    # walk up to root and compare paths
    path_u = []
    while u != -1:
        path_u.append(u)
        u = parent[u]

    path_v = []
    while v != -1:
        path_v.append(v)
        v = parent[v]

    i = len(path_u) - 1
    j = len(path_v) - 1

    while i >= 0 and j >= 0 and path_u[i] == path_v[j]:
        i -= 1
        j -= 1

    return (i + 1) + (j + 1)
```

Compare against optimized implementations.

## Property-Based Testing

Instead of checking exact values, check properties.

Examples:

- Distances are symmetric  
- Distance to self is zero  
- Tree diameter endpoints are farthest apart  
- Subtree size sum equals `n`  

```python
assert dist(u, v) == dist(v, u)
assert dist(u, u) == 0
```

## Stress Testing

Run many random tests:

```python
for _ in range(1000):
    n = 50
    edges = random_tree(n)
    # build graph
    # run both brute and optimized
    # compare results
```

Small `n` is enough because brute force is expensive.

## Debugging Strategy

When a test fails:

1. Reduce input size  
2. Print intermediate values  
3. Check invariants step by step  
4. Compare against brute force  

Use deterministic seeds for reproducibility.

```python
random.seed(42)
```

## Common Pitfalls

Using global arrays without resetting between tests.

Mixing 0-based and 1-based indexing.

Incorrect parent handling in DFS.

Forgetting to clear visited arrays.

Not testing degenerate trees.

## Recipe Summary

Test tree algorithms with structured cases, randomized inputs, and invariant checks. Always compare against a simple reference solution when possible.
