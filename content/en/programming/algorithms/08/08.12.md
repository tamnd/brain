# 8.12 Fenwick Trees

## Problem

You need a compact data structure for prefix sums, range sums, and point updates. A segment tree works, but it may be more code than necessary when the operation is simple and invertible, such as addition.

## Solution

Use a Fenwick tree, also called a binary indexed tree. It stores partial sums in an array and supports both updates and prefix queries in `O(log n)` time.

```python id="6b1hpr"
class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def update(self, i, delta):
        i += 1
        while i <= self.n:
            self.bit[i] += delta
            i += i & -i

    def prefix_sum(self, i):
        total = 0
        i += 1
        while i > 0:
            total += self.bit[i]
            i -= i & -i
        return total

    def range_sum(self, left, right):
        if left > right:
            return 0
        return self.prefix_sum(right) - self.prefix_sum(left - 1)
```

The public interface uses zero-based indexes. The internal array uses one-based indexes because the low-bit operation is simpler with positive indexes.

## Example

```python id="z0t15j"
arr = [5, 2, 7, 3, 6]
fw = Fenwick(len(arr))

for i, x in enumerate(arr):
    fw.update(i, x)

print(fw.range_sum(1, 3))  # 2 + 7 + 3 = 12

fw.update(2, 4)            # arr[2] changes from 7 to 11
print(fw.range_sum(1, 3))  # 2 + 11 + 3 = 16
```

The update takes a delta, not a replacement value. If you want to set `arr[i]` to a new value, store the current array value and update by the difference.

```python id="2ep94l"
def set_value(fw, arr, i, new_value):
    delta = new_value - arr[i]
    arr[i] = new_value
    fw.update(i, delta)
```

## Discussion

A Fenwick tree is built around the expression:

```python id="xjei6s"
i & -i
```

This extracts the lowest set bit of `i`. In the internal one-based array, `bit[i]` stores the sum of a block whose length is `i & -i`.

For example:

```text id="h8sc29"
bit[8] stores 8 elements
bit[6] stores 2 elements
bit[5] stores 1 element
```

A prefix query walks backward through these blocks until it reaches zero. An update walks forward through all blocks that contain the updated index.

The data structure is compact because it stores one value per array index. The price is that it is less general than a segment tree. Fenwick trees are best for operations where prefix aggregates can be combined and subtracted, such as sums.

## Building in O(n)

The simple build method performs `n` updates and costs `O(n log n)`. You can build in `O(n)` by pushing each partial sum to its next responsible index.

```python id="qb9uzy"
class Fenwick:
    def __init__(self, arr):
        self.n = len(arr)
        self.bit = [0] + arr[:]

        for i in range(1, self.n + 1):
            j = i + (i & -i)
            if j <= self.n:
                self.bit[j] += self.bit[i]
```

This constructor assumes `arr` is zero-based and stores its values directly into the one-based tree.

## Range Add, Point Query

A Fenwick tree can also support adding a value to a range and querying a single point by using a difference array.

```python id="6m12rz"
class RangeAddPointQuery:
    def __init__(self, n):
        self.fw = Fenwick(n)
        self.n = n

    def range_add(self, left, right, delta):
        self.fw.update(left, delta)
        if right + 1 < self.n:
            self.fw.update(right + 1, -delta)

    def point_query(self, i):
        return self.fw.prefix_sum(i)
```

Adding `delta` at `left` starts the effect. Adding `-delta` after `right` stops the effect.

## Subtree Updates

Using an Euler tour, every subtree becomes a contiguous interval. Therefore range add and point query can support subtree additions.

```python id="6d4ydu"
updates = RangeAddPointQuery(n)

def add_to_subtree(u, delta):
    updates.range_add(tin[u], tout[u], delta)

def value_at_node(u):
    return updates.point_query(tin[u])
```

This is useful when many operations add a value to every node in a subtree, and later queries ask for the value at one node.

## Correctness

For point update and prefix sum, `bit[i]` stores the sum of a fixed block ending at `i`. The update operation adds `delta` to every block that contains the changed position. Therefore every future prefix query that includes that position receives exactly one copy of `delta`, and every prefix query that excludes it receives none.

For range sums, the interval sum from `left` to `right` equals the prefix sum through `right` minus the prefix sum before `left`. Since both prefix sums are correct, their difference is correct.

For range add and point query, the Fenwick tree stores a difference array. The prefix sum of the difference array at index `i` equals the accumulated value applied to position `i`. Each range update starts the contribution at `left` and cancels it after `right`, so each point query returns the correct total.

## Complexity

| Operation | Time | Space |
|---|---:|---:|
| Build by repeated updates | O(n log n) | O(n) |
| Build by linear constructor | O(n) | O(n) |
| Point update | O(log n) | O(n) |
| Prefix sum | O(log n) | O(n) |
| Range sum | O(log n) | O(n) |
| Range add, point query | O(log n) | O(n) |

## Common Pitfalls

Do not call `update(i, new_value)` when you mean assignment. The method adds a delta.

Do not use internal index `0`. In the one-based representation, index `0` stops the loop.

Do not forget to guard `right + 1` in range updates. Updating past the end corrupts the tree.

Do not use a Fenwick tree when the operation has no inverse and you need arbitrary range queries. For range minimum with updates, a segment tree is usually the better choice.

Do not mix zero-based external indexes and one-based internal indexes in the same method. Convert at the boundary.

## Recipe Summary

Use a Fenwick tree for compact prefix sums, range sums, and point updates. It is smaller and simpler than a segment tree when the operation is addition. Combine it with Euler tour flattening to answer subtree queries and subtree updates on trees.
