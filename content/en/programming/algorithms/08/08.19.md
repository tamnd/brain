# 8.19 Serialization

## Problem

You need to convert a tree into a linear representation that can be stored, transmitted, or reconstructed later. The representation must preserve structure, not just values.

Typical uses:
- Save and load data structures  
- Send trees over a network  
- Persist state in files  
- Compare structures  

## Solution

Use traversal with markers for null or missing children.

For binary trees, preorder traversal with null markers is simple and sufficient.

## Binary Tree Serialization

```python id="9btxj1"
def serialize(root):
    result = []

    def dfs(node):
        if node is None:
            result.append("#")
            return

        result.append(str(node.value))
        dfs(node.left)
        dfs(node.right)

    dfs(root)
    return " ".join(result)
```

Example output:

```text id="91q3mb"
10 5 # # 20 15 # # 30 # #
```

## Binary Tree Deserialization

```python id="6p7c2n"
def deserialize(data):
    values = iter(data.split())

    def dfs():
        val = next(values)
        if val == "#":
            return None

        node = Node(int(val))
        node.left = dfs()
        node.right = dfs()
        return node

    return dfs()
```

## Discussion

The null marker ensures that the structure is uniquely determined. Without it, different trees could produce the same sequence.

Preorder works because:
- Each node is processed before its children  
- Null markers define when branches end  

Other traversals can also work, but they require additional information.

## Level-Order Serialization

For some applications, breadth-first encoding is more natural.

```python id="o3m6vc"
from collections import deque

def serialize_level(root):
    if not root:
        return ""

    result = []
    q = deque([root])

    while q:
        node = q.popleft()

        if node:
            result.append(str(node.value))
            q.append(node.left)
            q.append(node.right)
        else:
            result.append("#")

    return " ".join(result)
```

Deserialization follows the same structure using a queue.

## General Trees

For trees with arbitrary number of children, store the number of children.

```python id="v2f4rc"
class Node:
    def __init__(self, value):
        self.value = value
        self.children = []
```

Serialize:

```python id="o4bp8s"
def serialize(root):
    result = []

    def dfs(node):
        result.append(str(node.value))
        result.append(str(len(node.children)))
        for child in node.children:
            dfs(child)

    dfs(root)
    return " ".join(result)
```

Deserialize:

```python id="8h4hpg"
def deserialize(data):
    values = iter(data.split())

    def dfs():
        value = next(values)
        count = int(next(values))
        node = Node(value)

        for _ in range(count):
            node.children.append(dfs())

        return node

    return dfs()
```

## Discussion

Serialization must encode both:
- Node values  
- Tree structure  

For binary trees, structure is implicit in left and right positions, so null markers are enough.

For general trees, structure must be explicit, usually by storing child counts.

The key invariant:

```text id="5n9z1h"
The serialized sequence contains enough information to reconstruct the exact tree.
```

## Correctness

For preorder with null markers:

Each node produces:
- One value token  
- Two recursive subtrees  

Null nodes produce a marker.

The sequence uniquely determines the tree because the recursion structure is preserved. During deserialization, each marker signals a leaf boundary.

For general trees:

Each node records its number of children, so the parser knows exactly how many recursive calls to make. This guarantees correct reconstruction.

## Complexity

Let `n` be number of nodes.

| Operation      | Time | Space |
|----------------|-----:|------:|
| Serialize      | O(n) | O(n) |
| Deserialize    | O(n) | O(h) recursion stack |

## Common Pitfalls

Omitting null markers in binary trees leads to ambiguous reconstruction.

Using inconsistent traversal order between serialization and deserialization breaks correctness.

Forgetting to convert values to and from strings causes type errors.

Trailing nulls in level-order representation can be trimmed but must be handled carefully.

Using recursion on very deep trees may cause stack overflow.

## Recipe Summary

Serialize trees by recording values and structure. Use preorder with null markers for binary trees and child counts for general trees. Ensure the encoding is unambiguous and matches the decoding logic.
