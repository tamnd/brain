# 8.24 Case Studies

## Problem

You need to combine several tree techniques into complete solutions. Individual tools such as DFS, Euler tours, binary lifting, and segment trees are useful on their own, but real problems often require two or three of them together.

This section gives compact case studies that show how to select and compose tree patterns.

## Case Study 1. Subtree Sum with Updates

You are given a rooted tree. Each node has a value. You need two operations:

```
```
update u x     set value[u] = x  
query u        return sum of values in subtree of u
```
```

Use an Euler tour to flatten the tree. Then use a Fenwick tree or segment tree on the flattened array.

```
Python



```
timer = 0  
tin = [0] * n  
tout = [0] * n  
order = []  
  
def dfs(u, parent):  
    global timer  
    tin[u] = timer  
    order.append(u)  
    timer += 1  
  
    for v in g[u]:  
        if v == parent:  
            continue  
        dfs(v, u)  
  
    tout[u] = timer - 1
```
```

Build the flat array.

```
Python



```
flat = [0] * n  
for i, u in enumerate(order):  
    flat[i] = value[u]
```
```

Use a Fenwick tree for point update and range sum.

```
Python



```
fw = Fenwick(n)  
  
for i, x in enumerate(flat):  
    fw.update(i, x)  
  
def set_node(u, x):  
    delta = x - value[u]  
    value[u] = x  
    fw.update(tin[u], delta)  
  
def subtree_sum(u):  
    return fw.range_sum(tin[u], tout[u])
```
```

The tree problem becomes an array problem because every subtree is a contiguous interval.

## Case Study 2. Distance Between Two Nodes

You are given an unweighted tree and many queries:

```
```
distance u v
```
```

Use binary lifting for LCA. Then compute distance from depths.

```
Python



```
def dist(u, v):  
    w = lca(u, v)  
    return depth[u] + depth[v] - 2 * depth[w]
```
```

The formula follows from the fact that the path from `u` to `v` goes upward from `u` to `w`, then downward from `w` to `v`.

```
```
distance(u, v)  
= distance(u, w) + distance(w, v)  
= depth[u] - depth[w] + depth[v] - depth[w]
```
```

This gives `O(log n)` per query after `O(n log n)` preprocessing.

## Case Study 3. Path Maximum Query

Each node has a value. You need:

```
```
query u v      maximum value on path from u to v  
update u x     set value[u] = x
```
```

Use heavy-light decomposition plus a segment tree.

After decomposition:

```
Python



```
flat = [0] * n  
for u in range(n):  
    flat[pos[u]] = value[u]  
  
seg = MaxSegmentTree(flat)
```
```

Path query:

```
Python



```
def path_max(u, v):  
    ans = -10**30  
  
    while head[u] != head[v]:  
        if depth[head[u]] < depth[head[v]]:  
            u, v = v, u  
  
        ans = max(ans, seg.query(pos[head[u]], pos[u]))  
        u = parent[head[u]]  
  
    if depth[u] > depth[v]:  
        u, v = v, u  
  
    ans = max(ans, seg.query(pos[u], pos[v]))  
    return ans
```
```

Update:

```
Python



```
def update_node(u, x):  
    value[u] = x  
    seg.update(pos[u], x)
```
```

Heavy-light decomposition reduces a tree path into logarithmically many array intervals.

## Case Study 4. Tree Diameter After Reading Edges

You are given an unweighted tree. Return the length and endpoints of its diameter.

Use two BFS passes.

```
Python



```
a, _, _ = bfs_farthest(0)  
b, dist, parent = bfs_farthest(a)  
length = dist[b]
```
```

Reconstruct path:

```
Python



```
path = []  
cur = b  
  
while cur != -1:  
    path.append(cur)  
    cur = parent[cur]  
  
path.reverse()
```
```

The path from `a` to `b` is a diameter.

## Case Study 5. Nearest Active Node

You have a tree and two operations:

```
```
activate u  
query u        distance from u to nearest active node
```
```

Use centroid decomposition. Store, for each centroid, the nearest active node distance seen so far.

Precompute distance from each node to its centroid ancestors. Then update and query climb the centroid tree.

```
Python



```
INF = 10**30  
best = [INF] * n  
  
def activate(u):  
    x = u  
    while x != -1:  
        best[x] = min(best[x], distance(u, x))  
        x = centroid_parent[x]  
  
def query(u):  
    ans = INF  
    x = u  
    while x != -1:  
        ans = min(ans, best[x] + distance(u, x))  
        x = centroid_parent[x]  
    return ans
```
```

This is efficient because the centroid tree has depth `O(log n)`. With precomputed distances to centroid ancestors, each operation runs in `O(log n)`.

## Discussion

Each case study follows the same selection rule:

| Problem shape | Technique |
| --- | --- |
| Subtree query | Euler tour |
| Path query | Heavy-light decomposition |
| Ancestor query | Binary lifting |
| Distance query | LCA |
| Global all-root value | Rerooting |
| Dynamic distance query | Centroid decomposition |

The main skill is recognizing the query shape. Subtree problems usually want flattening. Path problems usually want decomposition. Ancestor problems usually want lifting. Global rerooted values usually want two passes.

## Correctness Notes

The subtree sum case is correct because DFS flattening maps each subtree to one contiguous interval.

The distance query case is correct because LCA splits the path into two root-to-node paths.

The path maximum case is correct because heavy-light decomposition partitions the path into disjoint chain segments.

The diameter case is correct because the farthest node from an arbitrary start is an endpoint of a diameter.

The nearest active node case is correct because every path from a query node to an active node is represented through some centroid ancestor in the decomposition hierarchy.

## Complexity Summary

| Case | Preprocessing | Query | Update |
| --- | --- | --- | --- |
| Subtree sum | O(n log n) or O(n) | O(log n) | O(log n) |
| Distance query | O(n log n) | O(log n) | None |
| Path maximum | O(n) plus segment tree | O(log^2 n) | O(log n) |
| Diameter | O(n) | None | None |
| Nearest active node | O(n log n) | O(log n) | O(log n) |

## Recipe Summary

Choose the representation by query shape. Flatten subtrees, lift ancestors, decompose paths, reroot global values, and use centroid decomposition for dynamic distance problems. Tree algorithms become manageable when the structural transformation matches the operation.
