# 8.22 Common Bugs

## Problem

Tree algorithms fail more often from small structural mistakes than from high level logic. Many bugs arise from incorrect assumptions about parent relationships, indexing, or traversal order.

This section catalogs failure patterns that appear repeatedly.

## Parent Handling Errors

In undirected trees, forgetting to skip the parent causes infinite recursion.

```python id="8f0mcz"
def dfs(u, parent):
    for v in g[u]:
        if v == parent:
            continue
        dfs(v, u)
```

Missing the `if v == parent` condition creates a loop.

A related bug is mixing `visited` and `parent` inconsistently. In trees, parent tracking is enough. In graphs, a visited array is required.

## Off-by-One Errors

Euler tour intervals are a frequent source of mistakes.

Correct pattern:

```python id="3m8lq2"
tin[u] = timer
timer += 1
...
tout[u] = timer - 1
```

Common bugs:
- Using `tout[u] = timer`
- Mixing inclusive `[l, r]` with half-open `[l, r)`

These errors break subtree queries.

## Incorrect Root Assumptions

Many algorithms assume a fixed root.

If the root changes:
- Depth values change  
- Parent relationships change  
- LCA preprocessing must be rebuilt  

Using a root-dependent structure without updating it leads to wrong answers.

## Uninitialized Arrays

Forgetting to reset global arrays between test cases leads to stale values.

```python id="6d4u2c"
subtree_size = [0] * n
visited = [False] * n
```

This must be done before every run.

## Mixing Indexing Conventions

Combining 0-based and 1-based indexing causes subtle bugs.

Example:

```python id="4y8h1v"
# Fenwick uses 1-based internally
i += 1
```

If you forget this conversion, updates and queries shift incorrectly.

## Recursion Depth Failures

A chain-shaped tree causes deep recursion.

```text id="8k2pvd"
0 - 1 - 2 - 3 - ... - n
```

Recursive DFS may overflow the stack.

Fix:
- Use iterative DFS  
- Increase recursion limit (language dependent)  

## Incorrect State Restoration

In rerooting and backtracking algorithms, failing to restore state after recursion corrupts future computations.

```python id="p9f6q1"
# save state
old = value[u]

# modify
value[u] = new

dfs(v)

# restore
value[u] = old
```

Without restoration, later branches see incorrect values.

## Double Counting

Common in path counting and centroid decomposition.

Mistake:
- Counting paths in each subtree separately  
- Not subtracting overlaps  

Correct approach:
- Count contributions per subtree  
- Combine carefully to avoid duplicates  

## Wrong Base Cases

Incorrect base cases break recursion.

Example in DFS:

```python id="6j0q1x"
if u == parent:
    return
```

This is incorrect. It stops traversal entirely. The correct condition belongs inside the loop.

## Lazy Propagation Errors

Typical mistakes:

- Not pushing lazy values before descending  
- Forgetting to clear lazy value after push  
- Applying updates twice  

```python id="3h0k8c"
if lazy[node] != 0:
    push(node)
```

Missing this step leads to incorrect queries.

## Binary Lifting Mistakes

Common issues:

- LOG too small  
- Parent of root not set properly  
- Accessing parent of `-1`  

```python id="2y7k8n"
if parent[k-1][v] != -1:
    parent[k][v] = parent[k-1][ parent[k-1][v] ]
```

## Segment Tree Errors

Frequent bugs:

- Wrong identity value  
- Incorrect merge function  
- Wrong mid calculation  
- Missing boundary checks  

```python id="n6z0vl"
mid = (l + r) // 2
```

## Testing Gaps

Algorithms pass simple tests but fail edge cases.

Missing tests:
- Single node  
- Deep chain  
- Star shape  
- Random trees  

## Debugging Strategy

When a bug appears:

1. Reduce input size  
2. Print intermediate arrays  
3. Check invariants  
4. Compare with brute force  

Example:

```python id="k5t7s2"
print("tin:", tin)
print("tout:", tout)
```

## Checklist

Before finalizing code:

- Parent handling correct  
- Indexing consistent  
- Arrays initialized  
- Edge cases tested  
- Recursion safe  
- Data structure invariants maintained  

## Recipe Summary

Most bugs come from small structural mistakes. Verify parent handling, indexing, traversal order, and invariants carefully. Use systematic testing to catch edge cases early.
