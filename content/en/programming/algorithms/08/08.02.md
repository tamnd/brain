# 8.2 DFS Traversal

## Problem

You need to visit every node in a tree, compute values from parent to child or child to parent, and avoid crossing the same undirected edge back and forth forever.

## Solution

Use depth first search. In an undirected tree, pass the parent node as an argument so the traversal can ignore the edge that leads back upward.

```python
def dfs(u, parent):
    for v in g[u]:
        if v == parent:
            continue
        dfs(v, u)
```

Here `u` is the current node, and `parent` is the node from which `u` was reached. The condition `v == parent` prevents the traversal from immediately returning along the same edge.

A useful DFS usually does work before or after visiting children.

```python
def dfs(u, parent):
    enter[u] = True

    for v in g[u]:
        if v == parent:
            continue
        dfs(v, u)

    exit[u] = True
```

The work before the recursive calls is preorder work. The work after the recursive calls is postorder work. Preorder is used when information flows from parent to child. Postorder is used when information flows from child to parent.

## Example: Computing Depth

Depth is naturally computed on entry. The root has depth `0`. Every child has depth one more than its parent.

```python
depth = [0] * n

def dfs(u, parent):
    for v in g[u]:
        if v == parent:
            continue
        depth[v] = depth[u] + 1
        dfs(v, u)

dfs(0, -1)
```

The invariant is that when `dfs(u, parent)` starts, `depth[u]` is already correct. The function then assigns correct depths to all children before entering them.

## Example: Computing Subtree Size

Subtree size is naturally computed on exit. Each node contributes `1` for itself, then adds the sizes of its children.

```python
subtree_size = [1] * n

def dfs(u, parent):
    for v in g[u]:
        if v == parent:
            continue
        dfs(v, u)
        subtree_size[u] += subtree_size[v]

dfs(0, -1)
```

The invariant is that after `dfs(v, u)` returns, `subtree_size[v]` is correct. Therefore it can be safely added into `subtree_size[u]`.

## Iterative DFS

Recursive DFS is short, but deep trees can overflow the language call stack. A path-shaped tree with many nodes is the usual failure case. Use an explicit stack when the input size is large or when the environment has a small recursion limit.

```python
def iterative_dfs(root):
    parent = [-1] * n
    order = []

    stack = [root]
    parent[root] = root

    while stack:
        u = stack.pop()
        order.append(u)

        for v in g[u]:
            if v == parent[u]:
                continue
            parent[v] = u
            stack.append(v)

    parent[root] = -1
    return parent, order
```

The array `order` records a preorder traversal. If you need postorder behavior, process the same order backward.

```python
parent, order = iterative_dfs(0)

subtree_size = [1] * n

for u in reversed(order):
    for v in g[u]:
        if v == parent[u]:
            continue
        subtree_size[u] += subtree_size[v]
```

This gives the same child to parent aggregation as recursive postorder DFS, without using the program call stack.

## Discussion

DFS works well on trees because every node except the root has exactly one parent in the rooted view induced by the traversal. Once DFS moves from `u` to `v`, the algorithm can treat `u` as the parent of `v`, even if the input tree was originally unrooted.

The parent argument is enough because the graph is a tree. In a general graph, a node may be reachable through multiple paths, so parent tracking alone does not prevent repeated visits. For a general graph, use a `visited` array. For a tree, the parent check is simpler and expresses the stronger invariant directly.

DFS can be understood as a way to impose direction on an undirected tree. Before traversal, an edge `{u, v}` has no parent child orientation. After rooting the tree at `r`, every edge points away from `r`. This rooted orientation is not stored in the input; it is produced by the traversal.

## Correctness

We prove that DFS visits every node exactly once when called on the root of a tree.

Because the input is a tree, there is a unique simple path from the root to every node. Consider any node `x`. If `x` is the root, it is visited at the initial call. Otherwise, let `p` be the node immediately before `x` on the unique path from the root to `x`. When DFS visits `p`, it examines all neighbors of `p`. The node `x` is one of those neighbors, and `x` is not the parent of `p`, so DFS calls `dfs(x, p)`. Thus every node is eventually visited.

Now suppose a node `x` is visited twice. Then there must be two different ways for DFS to reach `x` from the root. These two ways give two distinct simple paths from the root to `x`, contradicting the defining property of a tree. Therefore each node is visited exactly once.

The subtree size computation is correct by structural induction on the rooted tree. A leaf has no children, so its size remains `1`, which is correct. For an internal node `u`, assume each child `v` has correct `subtree_size[v]` after its recursive call returns. The subtree of `u` consists of `u` itself together with the disjoint subtrees of its children. The algorithm initializes `subtree_size[u] = 1` and adds each child subtree size once, so the result is correct.

## Complexity

DFS examines each undirected edge twice: once from each endpoint. Since a tree with `n` nodes has `n - 1` edges, the traversal takes `O(n)` time.

Recursive DFS uses `O(h)` call stack space, where `h` is the height of the rooted tree. In the worst case, `h = n`. Iterative DFS uses `O(n)` explicit stack and array space.

## Common Pitfalls

Do not omit the parent check in an undirected tree. Without it, DFS moves from a node to its parent and then immediately back to the child, causing infinite recursion.

Do not use a parent check as a substitute for `visited` in a general graph. A cycle with three or more nodes can still be revisited through a non-parent edge.

Do not compute subtree values before visiting children. Values that depend on child subtrees must be computed after the recursive call returns.

Do not assume recursive DFS is safe for very deep trees. Use an explicit stack when the tree can contain a long chain.

Do not mix preorder and postorder state updates. Depth, parent, and root-to-node path values are usually preorder values. Subtree size, subtree sum, and height are usually postorder values.
