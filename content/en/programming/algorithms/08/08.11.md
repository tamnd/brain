# 8.11 Segment Trees

## Problem

You need to answer range queries and process updates on an array. Typical operations include range sum, range minimum, range maximum, and range assignment or range addition with lazy propagation.

A segment tree is useful when the array changes over time. Static prefix sums answer range sums faster, but they do not support arbitrary updates efficiently.

## Solution

Build a binary tree over array intervals. Each node stores an aggregate value for one interval.

For range sum with point updates:

```python
class SegmentTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.tree = [0] * (4 * self.n)
        self._build(arr, 1, 0, self.n - 1)

    def _build(self, arr, node, left, right):
        if left == right:
            self.tree[node] = arr[left]
            return

        mid = (left + right) // 2
        self._build(arr, node * 2, left, mid)
        self._build(arr, node * 2 + 1, mid + 1, right)
        self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]

    def query(self, ql, qr):
        return self._query(1, 0, self.n - 1, ql, qr)

    def _query(self, node, left, right, ql, qr):
        if qr < left or right < ql:
            return 0

        if ql <= left and right <= qr:
            return self.tree[node]

        mid = (left + right) // 2
        return (
            self._query(node * 2, left, mid, ql, qr)
            + self._query(node * 2 + 1, mid + 1, right, ql, qr)
        )

    def update(self, index, value):
        self._update(1, 0, self.n - 1, index, value)

    def _update(self, node, left, right, index, value):
        if left == right:
            self.tree[node] = value
            return

        mid = (left + right) // 2
        if index <= mid:
            self._update(node * 2, left, mid, index, value)
        else:
            self._update(node * 2 + 1, mid + 1, right, index, value)

        self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]
```

Usage:

```python
arr = [5, 2, 7, 3, 6]
seg = SegmentTree(arr)

print(seg.query(1, 3))  # 2 + 7 + 3 = 12

seg.update(2, 10)
print(seg.query(1, 3))  # 2 + 10 + 3 = 15
```

## Discussion

A segment tree stores summaries of intervals. The root stores the whole array. Each internal node splits its interval into left and right halves. Leaves store individual elements.

The important invariant is:

```text
tree[node] stores the aggregate value of arr[left:right+1]
```

For a sum segment tree, the merge operation is addition. For a minimum segment tree, the merge operation is `min`. For a maximum segment tree, the merge operation is `max`.

Range queries are efficient because any query interval can be decomposed into a small number of stored intervals. At each tree level, only a constant number of nodes partially overlap the query range, so the total number of visited nodes is logarithmic up to a constant factor.

## Range Minimum Variant

To support range minimum query, change the identity value and merge operation.

```python
INF = 10**30

class MinSegmentTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.tree = [INF] * (4 * self.n)
        self._build(arr, 1, 0, self.n - 1)

    def _build(self, arr, node, left, right):
        if left == right:
            self.tree[node] = arr[left]
            return

        mid = (left + right) // 2
        self._build(arr, node * 2, left, mid)
        self._build(arr, node * 2 + 1, mid + 1, right)
        self.tree[node] = min(self.tree[node * 2], self.tree[node * 2 + 1])

    def query(self, ql, qr):
        return self._query(1, 0, self.n - 1, ql, qr)

    def _query(self, node, left, right, ql, qr):
        if qr < left or right < ql:
            return INF

        if ql <= left and right <= qr:
            return self.tree[node]

        mid = (left + right) // 2
        return min(
            self._query(node * 2, left, mid, ql, qr),
            self._query(node * 2 + 1, mid + 1, right, ql, qr),
        )
```

The shape of the algorithm remains unchanged. Only the aggregate operation changes.

## Lazy Propagation

Point updates are not enough when you need to update a whole interval many times. Lazy propagation delays updates until they are needed.

Example: range add and range sum.

```python
class LazySegmentTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.tree = [0] * (4 * self.n)
        self.lazy = [0] * (4 * self.n)
        self._build(arr, 1, 0, self.n - 1)

    def _build(self, arr, node, left, right):
        if left == right:
            self.tree[node] = arr[left]
            return

        mid = (left + right) // 2
        self._build(arr, node * 2, left, mid)
        self._build(arr, node * 2 + 1, mid + 1, right)
        self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]

    def _apply(self, node, left, right, delta):
        self.tree[node] += delta * (right - left + 1)
        self.lazy[node] += delta

    def _push(self, node, left, right):
        if self.lazy[node] == 0 or left == right:
            return

        mid = (left + right) // 2
        delta = self.lazy[node]

        self._apply(node * 2, left, mid, delta)
        self._apply(node * 2 + 1, mid + 1, right, delta)

        self.lazy[node] = 0

    def range_add(self, ql, qr, delta):
        self._range_add(1, 0, self.n - 1, ql, qr, delta)

    def _range_add(self, node, left, right, ql, qr, delta):
        if qr < left or right < ql:
            return

        if ql <= left and right <= qr:
            self._apply(node, left, right, delta)
            return

        self._push(node, left, right)

        mid = (left + right) // 2
        self._range_add(node * 2, left, mid, ql, qr, delta)
        self._range_add(node * 2 + 1, mid + 1, right, ql, qr, delta)

        self.tree[node] = self.tree[node * 2] + self.tree[node * 2 + 1]

    def range_sum(self, ql, qr):
        return self._range_sum(1, 0, self.n - 1, ql, qr)

    def _range_sum(self, node, left, right, ql, qr):
        if qr < left or right < ql:
            return 0

        if ql <= left and right <= qr:
            return self.tree[node]

        self._push(node, left, right)

        mid = (left + right) // 2
        return (
            self._range_sum(node * 2, left, mid, ql, qr)
            + self._range_sum(node * 2 + 1, mid + 1, right, ql, qr)
        )
```

Lazy propagation preserves the interval invariant while avoiding repeated work. If an update covers a node interval completely, the node is updated immediately and the change is recorded in `lazy`. Children are updated only when a later query or update needs to inspect them.

## Correctness

The basic segment tree is correct by induction on intervals.

For a leaf interval, the node stores exactly one array element, so the invariant holds. For an internal interval, the left child stores the aggregate over the left half, and the right child stores the aggregate over the right half. Merging these two values gives the aggregate over the whole interval.

A range query is correct because each returned node interval is fully contained in the query range, and the selected intervals are disjoint and cover exactly the queried positions. Combining their stored values gives the correct result.

For lazy propagation, `_apply` updates a node as if every element in its interval had received the update. `_push` distributes the delayed update to its children before descending. Therefore every query sees the same aggregate values it would see if all updates had been applied eagerly.

## Complexity

For an array of length `n`:

| Operation | Time | Space |
|---|---:|---:|
| Build | O(n) | O(n) |
| Range query | O(log n) | O(n) total structure |
| Point update | O(log n) | O(n) total structure |
| Range update with lazy propagation | O(log n) | O(n) total structure |

The common `4 * n` allocation is a simple upper bound for the recursive implementation.

## Common Pitfalls

Choosing the wrong identity value breaks partial overlap logic. For sum use `0`. For minimum use positive infinity. For maximum use negative infinity.

Forgetting to push lazy values before descending gives stale child values.

Using `mid = (left + right) // 2` is fine in Python, but in fixed-width integer languages prefer `left + (right - left) / 2` to avoid overflow.

Confusing inclusive and exclusive intervals causes off-by-one errors. This recipe uses inclusive intervals `[left, right]`.

Returning `0` from a minimum query on a non-overlapping segment is wrong unless all values are nonnegative and the query logic accounts for it. Use the true identity for the operation.

## Recipe Summary

Use a segment tree when you need range queries with updates. Define the aggregate operation, its identity value, and the update rule. Maintain the invariant that each node stores the correct aggregate for its interval. Use lazy propagation when updates affect whole ranges.
