# 8.3 BFS Traversal

## Problem

You need to traverse a tree level by level, compute shortest distances from a root, or process nodes in increasing order of depth. The traversal should avoid recursion and provide predictable order.

## Solution

Use breadth first search with a queue. In an undirected tree, track the parent to prevent revisiting the edge that leads upward.

```python id="m3h1ya"
from collections import deque

def bfs(root):
    parent = [-1] * n
    depth = [-1] * n

    q = deque([root])
    parent[root] = root
    depth[root] = 0

    while q:
        u = q.popleft()

        for v in g[u]:
            if v == parent[u]:
                continue
            parent[v] = u
            depth[v] = depth[u] + 1
            q.append(v)

    parent[root] = -1
    return parent, depth
```

The queue enforces level order. Nodes are processed in the order they are discovered.

## Example: Level Order Processing

You can group nodes by depth using BFS.

```python id="nrbqkl"
def levels(root):
    depth = [-1] * n
    q = deque([root])
    depth[root] = 0

    result = []

    while q:
        u = q.popleft()

        if depth[u] == len(result):
            result.append([])

        result[depth[u]].append(u)

        for v in g[u]:
            if depth[v] != -1:
                continue
            depth[v] = depth[u] + 1
            q.append(v)

    return result
```

Each list in `result` contains nodes at the same depth.

## Example: Shortest Paths in a Tree

In a tree, there is exactly one simple path between any two nodes. BFS from a root computes the shortest path distances in terms of number of edges.

```python id="6t9p38"
depth = [-1] * n

def bfs(root):
    q = deque([root])
    depth[root] = 0

    while q:
        u = q.popleft()

        for v in g[u]:
            if depth[v] != -1:
                continue
            depth[v] = depth[u] + 1
            q.append(v)

bfs(0)
```

The invariant is that when a node `u` is dequeued, `depth[u]` is already the minimum number of edges from the root to `u`.

## Example: Multi-Source BFS

Sometimes you need distances from multiple starting points. Initialize the queue with all sources.

```python id="y7rmpo"
def multi_source_bfs(sources):
    dist = [-1] * n
    q = deque()

    for s in sources:
        dist[s] = 0
        q.append(s)

    while q:
        u = q.popleft()

        for v in g[u]:
            if dist[v] != -1:
                continue
            dist[v] = dist[u] + 1
            q.append(v)

    return dist
```

This computes, for every node, the distance to the nearest source.

## Discussion

BFS imposes a layering on the tree. The root is at layer `0`. All nodes at distance `1` form the next layer, and so on. The queue ensures that nodes are expanded in increasing order of distance.

In contrast to DFS, BFS is inherently iterative. It does not rely on recursion, so it avoids stack depth issues. It also provides direct access to level structure, which is useful for problems involving breadth, radius, or minimum steps.

In a tree, BFS and DFS both visit all nodes in linear time. The choice depends on the direction of information flow. If you need shortest paths or level grouping, BFS is natural. If you need subtree aggregation, DFS is usually simpler.

## Correctness

We prove that BFS computes correct shortest path distances from the root.

At initialization, the root has distance `0`, which is correct. All other nodes are marked unvisited.

Assume that when a node `u` is dequeued, `depth[u]` is the shortest distance from the root. For each neighbor `v` of `u` that has not been visited, BFS sets `depth[v] = depth[u] + 1` and enqueues `v`.

Any path from the root to `v` must pass through some neighbor of `v`. The first time BFS discovers `v`, it does so from the neighbor with minimum distance. Therefore the assigned distance is minimal. Once `v` is assigned a distance, it is never updated again, so the value remains correct.

Because the tree has no cycles, each node is discovered exactly once, and the process terminates after all nodes are processed.

## Complexity

Each node is enqueued and dequeued once. Each edge is examined at most twice. The total time is `O(n)` for a tree with `n` nodes.

The queue stores at most one level of nodes at a time in a balanced tree, but in the worst case it can store `O(n)` nodes, for example when the root has many children.

Space usage is `O(n)` due to the queue and auxiliary arrays.

## Common Pitfalls

Using a parent check and a visited check together can be redundant. In BFS, it is often simpler to use a visited or distance array and avoid a separate parent argument.

Forgetting to mark a node as visited before enqueuing can cause multiple insertions into the queue.

Assuming BFS works the same on weighted trees is incorrect. BFS computes shortest paths only when all edges have equal weight. For weighted trees, use DFS with accumulation or a priority-based method.

Mixing DFS-style logic into BFS can break invariants. BFS relies on the queue order to guarantee minimal distances.

## Recipe Summary

Use BFS when you need level order traversal or shortest path distances in an unweighted tree. Initialize the queue with one or more sources, mark nodes when they are enqueued, and propagate distances outward layer by layer.
