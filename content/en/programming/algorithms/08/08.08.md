# 8.8 Subtree Queries

## Problem

You need to compute or update values over subtrees efficiently. Typical tasks include subtree sum, minimum, maximum, counting nodes with a property, or applying updates to all nodes in a subtree.

## Solution

Flatten the tree using an Euler tour (entry-only) so that each subtree becomes a contiguous segment in an array. Then apply a range data structure such as a Fenwick tree or segment tree.

Assume you already have:

- `tin[u]`, `tout[u]` from DFS  
- `order` array mapping index to node  

Build a flat array:

```python id="7czw2f"
flat = [0] * n
for i, u in enumerate(order):
    flat[i] = value[u]
```

Now subtree of `u` corresponds to:

```python id="9pl9nt"
l = tin[u]
r = tout[u]
```

## Fenwick Tree (Binary Indexed Tree)

Efficient for prefix sums and point updates.

```python id="k5dxn7"
class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def update(self, i, delta):
        i += 1
        while i <= self.n:
            self.bit[i] += delta
            i += i & -i

    def query(self, i):
        s = 0
        i += 1
        while i > 0:
            s += self.bit[i]
            i -= i & -i
        return s

    def range_query(self, l, r):
        return self.query(r) - self.query(l - 1)
```

Initialize:

```python id="tvh6t5"
fw = Fenwick(n)
for i in range(n):
    fw.update(i, flat[i])
```

Subtree sum:

```python id="r3r1m0"
def subtree_sum(u):
    return fw.range_query(tin[u], tout[u])
```

## Segment Tree

Supports more general operations such as range minimum or range updates.

```python id="0g79hy"
class SegmentTree:
    def __init__(self, arr):
        self.n = len(arr)
        self.tree = [0] * (4 * self.n)
        self.build(arr, 1, 0, self.n - 1)

    def build(self, arr, node, l, r):
        if l == r:
            self.tree[node] = arr[l]
            return
        mid = (l + r) // 2
        self.build(arr, node*2, l, mid)
        self.build(arr, node*2+1, mid+1, r)
        self.tree[node] = self.tree[node*2] + self.tree[node*2+1]

    def query(self, node, l, r, ql, qr):
        if qr < l or r < ql:
            return 0
        if ql <= l and r <= qr:
            return self.tree[node]
        mid = (l + r) // 2
        return self.query(node*2, l, mid, ql, qr) + \\n               self.query(node*2+1, mid+1, r, ql, qr)
```

Subtree query:

```python id="r3l9lz"
seg = SegmentTree(flat)

def subtree_sum(u):
    return seg.query(1, 0, n-1, tin[u], tout[u])
```

## Subtree Updates

To add a value `x` to all nodes in subtree `u`, update the range `[tin[u], tout[u]]`.

With a segment tree, use lazy propagation. With Fenwick, use a difference trick.

Example with difference array using Fenwick:

```python id="z0gchp"
fw = Fenwick(n)

def range_add(l, r, x):
    fw.update(l, x)
    if r + 1 < n:
        fw.update(r + 1, -x)

def point_query(i):
    return fw.query(i)
```

Subtree update:

```python id="3hs1hb"
range_add(tin[u], tout[u], x)
```

## Discussion

The key transformation is:

Tree → Euler tour → Array → Range queries

This reduces hierarchical problems to linear problems. Once flattened, you can reuse well-optimized array data structures.

Fenwick trees are simpler and faster for sums. Segment trees are more flexible and support complex queries and updates.

This method works because DFS guarantees that all nodes in a subtree appear consecutively in the Euler order.

## Correctness

From Euler tour properties:

- `tin[u]` marks the first visit to `u`
- All descendants of `u` are visited before leaving `u`
- Therefore all nodes in subtree of `u` occupy `[tin[u], tout[u]]`

Thus any aggregate over the subtree is equivalent to a range aggregate over this interval.

## Complexity

Preprocessing DFS: `O(n)`

Fenwick tree:
- Update: `O(log n)`
- Query: `O(log n)`

Segment tree:
- Build: `O(n)`
- Query: `O(log n)`
- Update: `O(log n)`

## Common Pitfalls

Using incorrect `tout[u]` definition breaks subtree ranges.

Forgetting that Fenwick tree is 1-based internally causes indexing bugs.

Mixing point updates with range queries incorrectly in Fenwick without proper transformation.

Using segment tree without lazy propagation for range updates leads to incorrect results.

Not rebuilding `flat` array when values change externally.

## Recipe Summary

Flatten the tree with DFS, map each subtree to a contiguous segment, and use range data structures to answer queries efficiently.
