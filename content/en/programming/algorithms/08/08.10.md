# 8.10 Tries

## Problem

You need to store and query a large set of strings efficiently, especially for prefix queries such as:

- Does a string exist  
- How many strings share a prefix  
- Find all strings with a given prefix  
- Longest prefix match  

## Solution

Use a trie (prefix tree). Each edge represents a character, and each path from root to a node represents a prefix.

```python id="d7r4ks"
class TrieNode:
    def __init__(self):
        self.children = {}
        self.end = False
```

Insert a word:

```python id="l2s8qp"
def insert(root, word):
    node = root
    for ch in word:
        if ch not in node.children:
            node.children[ch] = TrieNode()
        node = node.children[ch]
    node.end = True
```

Search for a word:

```python id="k3d9af"
def search(root, word):
    node = root
    for ch in word:
        if ch not in node.children:
            return False
        node = node.children[ch]
    return node.end
```

Check prefix:

```python id="p8t0nx"
def starts_with(root, prefix):
    node = root
    for ch in prefix:
        if ch not in node.children:
            return False
        node = node.children[ch]
    return True
```

## Counting Prefix Matches

Store a counter in each node.

```python id="a7m1vy"
class TrieNode:
    def __init__(self):
        self.children = {}
        self.count = 0
        self.end = 0
```

Insert updates counts:

```python id="o4q2jk"
def insert(root, word):
    node = root
    for ch in word:
        if ch not in node.children:
            node.children[ch] = TrieNode()
        node = node.children[ch]
        node.count += 1
    node.end += 1
```

Count prefix:

```python id="z3n6qp"
def count_prefix(root, prefix):
    node = root
    for ch in prefix:
        if ch not in node.children:
            return 0
        node = node.children[ch]
    return node.count
```

## Discussion

A trie stores strings in a tree where each level corresponds to a character position. This avoids repeated storage of prefixes and enables fast prefix-based queries.

The complexity depends on string length, not on the number of stored strings. This makes tries effective when strings are long or when prefix queries dominate.

The main tradeoff is memory. A trie can use more space than a hash table because each node stores a map of children.

## Optimizations

### Fixed Alphabet

If the alphabet is small (e.g., lowercase letters), use arrays instead of maps.

```python id="y6w2sf"
class TrieNode:
    def __init__(self):
        self.children = [None] * 26
        self.end = False
```

### Compressed Trie

Merge chains of single-child nodes into one edge labeled with a string. This reduces memory and improves cache locality.

### Bitwise Trie

For integers, represent numbers in binary and build a trie over bits.

```python id="q9w3rm"
def insert(root, num):
    node = root
    for i in reversed(range(32)):
        bit = (num >> i) & 1
        if bit not in node.children:
            node.children[bit] = TrieNode()
        node = node.children[bit]
```

Used for problems like maximum XOR pair.

## Correctness

Each node represents a unique prefix. Insertion ensures that every prefix of a word corresponds to a path in the tree. The `end` flag marks complete words.

Search succeeds if and only if the path exists and the final node is marked as an end node.

Prefix queries succeed if the path exists, regardless of the `end` flag.

## Complexity

Let `L` be the length of the string.

| Operation      | Time   |
|----------------|--------|
| Insert         | O(L)   |
| Search         | O(L)   |
| Prefix query   | O(L)   |

Space depends on the number of nodes, which is bounded by total characters inserted.

## Common Pitfalls

Using a map for children can be slow if the alphabet is small. Prefer arrays when possible.

Forgetting to distinguish between prefix and full word leads to incorrect results.

Not handling duplicate insertions correctly when counting.

Memory usage can grow quickly with many distinct prefixes.

## Recipe Summary

Use a trie when prefix structure matters. Store counts or flags in nodes to support queries. Optimize representation based on alphabet size and memory constraints.
