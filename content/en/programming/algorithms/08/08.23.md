# 8.23 Implementation Patterns

## Problem

Tree algorithms are often correct in theory but fail in practice due to inconsistent structure, duplicated logic, or unclear data flow. You need stable patterns that reduce cognitive load and prevent errors.

## Pattern 1. Standard DFS Skeleton

Use a consistent DFS template with explicit parent handling.

```
Python



```
def dfs(u, parent):  
    # pre-processing  
  
    for v in g[u]:  
        if v == parent:  
            continue  
        dfs(v, u)  
  
    # post-processing
```
```

This separates preorder and postorder logic clearly.

## Pattern 2. Build Once, Query Many

Separate preprocessing from queries.

```
Python



```
build()  
  
for query in queries:  
    answer(query)
```
```

Examples:

- Binary lifting
- Heavy-light decomposition
- Segment trees

Avoid recomputing structures per query.

## Pattern 3. Flatten Then Solve

Convert tree problems into array problems.

```
Python



```
dfs(root)  
# build flat array  
# use segment tree or Fenwick
```
```

Used for:

- Subtree queries
- Range updates
- Frequency counting

## Pattern 4. Two-Pass DP

Split computation into:

1. Bottom-up pass
2. Top-down pass

```
Python



```
dfs1(u, parent)  
dfs2(u, parent)
```
```

Used in:

- Rerooting
- Distance sums
- Global DP problems

## Pattern 5. Parent + Depth Arrays

Always compute these early.

```
Python



```
parent[u]  
depth[u]
```
```

They support:

- LCA
- Distance computation
- Path decomposition

Distance formula:

```
Python



```
dist(u, v) = depth[u] + depth[v] - 2 * depth[lca(u, v)]
```
```

## Pattern 6. Iterative DFS

Avoid recursion when depth is large.

```
Python



```
stack = [(root, -1)]  
  
while stack:  
    u, parent = stack.pop()  
  
    for v in g[u]:  
        if v == parent:  
            continue  
        stack.append((v, u))
```
```

Add additional fields to simulate recursion state if needed.

## Pattern 7. Edge List to Adjacency List

Normalize input immediately.

```
Python



```
g = [[] for _ in range(n)]  
  
for u, v in edges:  
    g[u].append(v)  
    g[v].append(u)
```
```

Do this once and reuse.

## Pattern 8. Value Separation

Keep structure separate from values.

```
Python



```
g = [...]          # structure  
value = [...]      # node values
```
```

Avoid embedding values into adjacency lists unless required.

## Pattern 9. Reversible State Changes

For backtracking or rerooting:

```
Python



```
# save  
old = state  
  
# modify  
state = new  
  
dfs()  
  
# restore  
state = old
```
```

This prevents cross-branch contamination.

## Pattern 10. Small-to-Large Merging

When merging data from children:

```
Python



```
if len(A) < len(B):  
    A, B = B, A  
  
# merge B into A
```
```

Ensures total complexity stays near linear or `O(n log n)`.

Used in:

- DSU on tree
- Frequency maps

## Pattern 11. Layered Design

Break complex algorithms into layers:

1. Tree structure
2. Traversal
3. Data structure
4. Query logic

Example:

- HLD + segment tree
- Euler tour + Fenwick

Each layer is testable independently.

## Pattern 12. Explicit Invariants

Document invariants in code.

```
Python



```
# invariant: subtree_size[u] = 1 + sum(child sizes)
```
```

This helps debugging and verification.

## Pattern 13. Precompute Limits

Define constants clearly.

```
Python



```
LOG = (n).bit_length()
```
```

Avoid hardcoding values.

## Pattern 14. Clean Interfaces

Encapsulate data structures.

```
Python



```
class Tree:  
    def __init__(self, n):  
        self.g = [[] for _ in range(n)]  
  
    def add_edge(self, u, v):  
        self.g[u].append(v)  
        self.g[v].append(u)
```
```

Keeps logic organized and reusable.

## Pattern 15. Test Hooks

Add debug utilities.

```
Python



```
def check_tree():  
    assert len(edges) == n - 1
```
```

These can be disabled in production.

## Discussion

These patterns reduce error rates by standardizing how trees are handled. Instead of inventing structure for each problem, reuse known templates.

Most bugs arise when:

- Patterns are mixed incorrectly
- State is shared unintentionally
- Traversal logic is inconsistent

Consistency across implementations is more valuable than micro-optimizations.

## Recipe Summary

Use stable patterns: standard DFS, preprocessing, flattening, two-pass DP, and layered design. Keep structure, data, and logic separate. Maintain invariants explicitly.
