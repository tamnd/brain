# 8.7 Euler Tour

## Problem

You need to convert a tree into a linear structure so that subtree queries, ancestor checks, or path-related operations can be reduced to array problems.

## Solution

Perform a DFS and record entry and exit times. This produces an Euler tour ordering that maps each subtree to a contiguous segment.

```python id="kq2t9a"
timer = 0
tin = [0] * n
tout = [0] * n
order = []

def dfs(u, parent):
    global timer
    tin[u] = timer
    order.append(u)
    timer += 1

    for v in g[u]:
        if v == parent:
            continue
        dfs(v, u)

    tout[u] = timer - 1

dfs(0, -1)
```

Now each node `u` corresponds to a segment `[tin[u], tout[u]]` in the array `order`.

## Subtree Queries

Any subtree query becomes a range query.

Example: compute subtree sums.

```python id="1y2hkm"
value = [1, 2, 3, 4, 5]  # example values
flat = [0] * n

for i, u in enumerate(order):
    flat[i] = value[u]
```

Subtree of `u`:

```python id="2u0l4v"
l = tin[u]
r = tout[u]
# sum over flat[l:r+1]
```

This reduces tree aggregation to prefix sums or segment tree queries.

## Ancestor Check

Node `u` is an ancestor of node `v` if and only if:

```python id="2z0i3h"
def is_ancestor(u, v):
    return tin[u] <= tin[v] and tout[v] <= tout[u]
```

This works because DFS assigns entry times before visiting descendants and exit times after finishing them.

## Full Euler Tour (with Revisit)

Another variant records nodes on both entry and exit.

```python id="n9h2q1"
euler = []
depth = []

def dfs(u, parent, d):
    euler.append(u)
    depth.append(d)

    for v in g[u]:
        if v == parent:
            continue
        dfs(v, u, d + 1)
        euler.append(u)
        depth.append(d)
```

This produces a sequence of length `2n - 1`. It is used for LCA via RMQ.

## Discussion

The Euler tour technique transforms hierarchical structure into linear structure. This is powerful because arrays support efficient queries with prefix sums, segment trees, and sparse tables.

There are two main variants:

1. Entry-only (flattened tree)  
2. Full Euler tour (with backtracking)

The entry-only version is compact and ideal for subtree queries. The full tour preserves traversal structure and is used for LCA reductions.

The key invariant is that DFS visits all nodes in a subtree consecutively. Therefore the subtree corresponds to a contiguous segment in the flattened array.

## Correctness

We prove that the subtree of a node `u` corresponds to `[tin[u], tout[u]]`.

When DFS enters `u`, it assigns `tin[u]` and then recursively visits all descendants before assigning `tout[u]`. No node outside the subtree of `u` is visited between these two events.

Therefore all nodes in the subtree of `u` appear consecutively in `order`, starting at `tin[u]` and ending at `tout[u]`.

For ancestor checks, if `u` is an ancestor of `v`, then DFS enters `u` before `v` and exits `u` after `v`. This gives:

- `tin[u] <= tin[v]`  
- `tout[v] <= tout[u]`  

The converse also holds, so the condition is both necessary and sufficient.

## Complexity

DFS takes `O(n)` time.

All arrays (`tin`, `tout`, `order`) use `O(n)` space.

Each subtree query becomes a range query, which depends on the chosen data structure:

| Method          | Query Time |
|-----------------|-----------|
| Prefix sum      | O(1)      |
| Segment tree    | O(log n)  |
| Fenwick tree    | O(log n)  |

## Common Pitfalls

Forgetting to increment the timer correctly leads to overlapping intervals.

Using `tout[u] = timer` instead of `timer - 1` breaks inclusive range logic.

Mixing entry-only and full Euler tour logic leads to incorrect indexing.

Not resetting global variables between test cases causes incorrect results.

Assuming subtree nodes are contiguous without DFS flattening is incorrect.

## Recipe Summary

Run DFS to assign entry and exit times. Map each subtree to a contiguous array segment. Use range data structures to answer subtree queries efficiently.
