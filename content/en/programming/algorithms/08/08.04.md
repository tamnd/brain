# 8.4 Tree Diameter

## Problem

You need the longest simple path in a tree. The result is either the length of that path or the pair of endpoints. This path is called the diameter.

## Solution

Use two passes of BFS or DFS.

1. Start from an arbitrary node `s` and find the farthest node `a`.  
2. Start from `a` and find the farthest node `b`.  
3. The path from `a` to `b` is a diameter.

```python
from collections import deque

def bfs_farthest(start):
    dist = [-1] * n
    parent = [-1] * n

    q = deque([start])
    dist[start] = 0

    while q:
        u = q.popleft()
        for v in g[u]:
            if dist[v] != -1:
                continue
            dist[v] = dist[u] + 1
            parent[v] = u
            q.append(v)

    farthest = start
    for i in range(n):
        if dist[i] > dist[farthest]:
            farthest = i

    return farthest, dist, parent


a, _, _ = bfs_farthest(0)
b, dist, parent = bfs_farthest(a)

diameter_length = dist[b]
```

To reconstruct the path:

```python
path = []
cur = b
while cur != -1:
    path.append(cur)
    cur = parent[cur]
path.reverse()
```

## Discussion

A tree has a unique simple path between any two nodes. The diameter is the path with maximum length among all such pairs. The two-pass method works because of how distances behave in trees.

The first traversal identifies one endpoint of a longest path. The second traversal measures the maximum distance from that endpoint, which must reach the opposite endpoint.

This approach works for both DFS and BFS. BFS is preferred when edges are unweighted. DFS with accumulation works when edges have weights.

## Weighted Tree Variant

For weighted trees, replace BFS with DFS that accumulates distances.

```python
def dfs_farthest(u, parent, dist):
    best = (dist, u)

    for v, w in g[u]:
        if v == parent:
            continue
        cand = dfs_farthest(v, u, dist + w)
        if cand[0] > best[0]:
            best = cand

    return best


_, a = dfs_farthest(0, -1, 0)
diameter_length, b = dfs_farthest(a, -1, 0)
```

The invariant is that `dist` always represents the path length from the start node to the current node.

## Alternative: Single DFS with Two States

You can compute the diameter in one DFS by tracking the two largest downward paths at each node.

```python
diameter = 0

def dfs(u, parent):
    global diameter

    best1 = 0
    best2 = 0

    for v in g[u]:
        if v == parent:
            continue
        h = dfs(v, u) + 1

        if h > best1:
            best2 = best1
            best1 = h
        elif h > best2:
            best2 = h

    diameter = max(diameter, best1 + best2)
    return best1
```

Here `best1` and `best2` are the two largest heights among children. The candidate diameter through node `u` is `best1 + best2`.

## Correctness

For the two-pass method:

Take any longest path in the tree, with endpoints `x` and `y`. Starting BFS from any node `s`, one of the farthest nodes found must be either `x` or `y`. Call it `a`.

Now start BFS from `a`. The farthest node from `a` must be the other endpoint of the diameter. If it were not, then there would exist a longer path than the diameter, which is a contradiction.

For the single DFS method:

At each node `u`, the longest path that passes through `u` must go down into two different subtrees. The longest such path is the sum of the two largest child heights. Taking the maximum over all nodes gives the diameter.

## Complexity

Each traversal takes `O(n)` time. The two-pass method runs in `O(n)` total time. The single DFS method also runs in `O(n)` time.

Space complexity is `O(n)` for adjacency lists and auxiliary arrays. Recursive DFS uses `O(h)` stack space, where `h` is the tree height.

## Common Pitfalls

Using BFS on weighted trees gives incorrect results because edge weights are ignored.

Forgetting to reset arrays between the first and second traversal leads to incorrect distances.

Mixing parent checks and visited arrays inconsistently can skip valid edges or revisit nodes.

In the single DFS method, failing to track the top two heights correctly leads to undercounting the diameter.

## Recipe Summary

Use two traversals to compute the diameter directly and reconstruct the path when needed. Use a single DFS when only the length is required and you want a compact implementation.
