# 8.16 Centroid Decomposition

## Problem

You need to process queries or compute values over all paths in a tree, often involving distances or pair interactions. Direct solutions may take `O(n^2)` time. You need a divide and conquer structure over the tree.

Typical tasks:
- Count pairs of nodes satisfying a distance condition  
- Maintain dynamic updates with distance queries  
- Solve path problems with constraints  

## Solution

Decompose the tree recursively around centroids.

A centroid is a node whose removal splits the tree into components, each of size at most `n/2`.

## Step 1. Compute Subtree Sizes

```python id="e1x2mh"
subtree_size = [0] * n
removed = [False] * n

def dfs_size(u, p):
    subtree_size[u] = 1
    for v in g[u]:
        if v == p or removed[v]:
            continue
        dfs_size(v, u)
        subtree_size[u] += subtree_size[v]
```

## Step 2. Find Centroid

```python id="o9klp2"
def find_centroid(u, p, total):
    for v in g[u]:
        if v == p or removed[v]:
            continue
        if subtree_size[v] > total // 2:
            return find_centroid(v, u, total)
    return u
```

## Step 3. Decompose

```python id="p7zn3t"
centroid_parent = [-1] * n

def decompose(u, p):
    dfs_size(u, -1)
    c = find_centroid(u, -1, subtree_size[u])

    centroid_parent[c] = p
    removed[c] = True

    for v in g[c]:
        if not removed[v]:
            decompose(v, c)

decompose(0, -1)
```

This builds a centroid tree where each node points to its parent in the decomposition.

## Discussion

Centroid decomposition transforms a tree into a hierarchy where each level represents a balanced split. The depth of this decomposition is `O(log n)`.

At each centroid, you can process all paths that pass through it. Then recursively solve the same problem in each subtree.

The key idea:

```text id="1zq3ux"
Every path in the tree has a highest centroid where it is processed exactly once
```

This ensures no double counting.

## Example: Count Pairs with Distance ≤ K

At each centroid:

1. Collect distances from centroid to all nodes in each subtree  
2. Count valid pairs using these distances  
3. Merge results  

```python id="4jz8r2"
def collect_distances(u, p, dist, dists):
    dists.append(dist)
    for v, w in g[u]:
        if v == p or removed[v]:
            continue
        collect_distances(v, u, dist + w, dists)
```

Then use a two-pointer or binary search technique to count valid pairs.

## Dynamic Queries

Centroid decomposition supports dynamic queries such as:

- Activate a node  
- Query distance to nearest active node  

Maintain data at each centroid level:

```python id="5y1m2k"
dist_to_centroid = [[] for _ in range(n)]
```

Each node stores distances to its ancestors in the centroid tree.

Updates and queries climb the centroid tree in `O(log n)` time.

## Correctness

Each recursive step removes a centroid and processes all paths that pass through it. Remaining paths lie entirely within subtrees and are handled recursively.

Because each subtree has size at most half of the original, recursion depth is bounded by `O(log n)`.

Each path is considered at exactly one centroid, so no path is missed or counted multiple times.

## Complexity

Building decomposition:

- Each level processes all nodes once  
- Total: `O(n log n)`

Queries and updates:

- Each operation climbs centroid tree  
- Time: `O(log n)`

## Common Pitfalls

Forgetting to mark centroids as removed causes infinite recursion.

Mixing original tree edges with centroid tree edges leads to incorrect traversal.

Not clearing temporary data between processing subtrees leads to double counting.

Incorrect distance accumulation in weighted trees causes wrong results.

Using global arrays without resetting per decomposition step introduces bugs.

## Recipe Summary

Decompose the tree around centroids, process paths at each centroid, and recurse on subtrees. This converts global path problems into manageable local computations with logarithmic depth.
