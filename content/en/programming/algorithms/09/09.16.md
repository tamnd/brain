# 9.16 Euler Paths

## Problem

You need to determine whether a graph has a walk that uses every edge exactly once. Such a walk is called an Euler path. If it starts and ends at the same vertex, it is called an Euler circuit.

Euler paths appear in route inspection, puzzle solving, de Bruijn graph assembly, word-chain problems, and edge-covering tasks. The key point is that vertices may repeat, but edges may not.

## Solution

The test depends on whether the graph is undirected or directed.

For an undirected graph:

| Structure | Condition |
|---|---|
| Euler circuit | Every non-isolated vertex is connected, and every vertex has even degree |
| Euler path but not circuit | Every non-isolated vertex is connected, and exactly two vertices have odd degree |

For a directed graph:

| Structure | Condition |
|---|---|
| Euler circuit | Every non-isolated vertex belongs to one weakly connected component, and every vertex has `indeg[u] = outdeg[u]` |
| Euler path but not circuit | One vertex has `outdeg[u] = indeg[u] + 1`, one vertex has `indeg[u] = outdeg[u] + 1`, all others have equal in-degree and out-degree, and all non-isolated vertices are weakly connected |

After the condition is satisfied, construct the path with Hierholzer's algorithm.

## Undirected Degree Test

```python id="0h3jfp"
def has_euler_path_undirected(adj):
    odd = 0

    for neighbors in adj:
        if len(neighbors) % 2 == 1:
            odd += 1

    return odd == 0 or odd == 2
```

This only checks degree. A full test must also check that all vertices with nonzero degree lie in one connected component.

```python id="eo4l6b"
def nonzero_vertices_connected(adj):
    n = len(adj)
    start = -1

    for u in range(n):
        if len(adj[u]) > 0:
            start = u
            break

    if start == -1:
        return True

    seen = [False] * n
    stack = [start]
    seen[start] = True

    while stack:
        u = stack.pop()

        for v in adj[u]:
            if not seen[v]:
                seen[v] = True
                stack.append(v)

    for u in range(n):
        if len(adj[u]) > 0 and not seen[u]:
            return False

    return True
```

The complete undirected test is:

```python id="51f9mr"
def euler_status_undirected(adj):
    if not nonzero_vertices_connected(adj):
        return "none"

    odd = sum(1 for neighbors in adj if len(neighbors) % 2 == 1)

    if odd == 0:
        return "circuit"
    if odd == 2:
        return "path"
    return "none"
```

## Directed Degree Test

```python id="lpdo19"
def euler_status_directed(adj):
    n = len(adj)
    indeg = [0] * n
    outdeg = [0] * n

    for u in range(n):
        outdeg[u] = len(adj[u])
        for v in adj[u]:
            indeg[v] += 1

    start_like = 0
    end_like = 0

    for u in range(n):
        diff = outdeg[u] - indeg[u]

        if diff == 1:
            start_like += 1
        elif diff == -1:
            end_like += 1
        elif diff != 0:
            return "none"

    if start_like == 0 and end_like == 0:
        return "circuit"
    if start_like == 1 and end_like == 1:
        return "path"
    return "none"
```

As with the undirected case, this degree test must be paired with a connectivity check over the underlying undirected graph, restricted to non-isolated vertices.

## Hierholzer's Algorithm

Hierholzer's algorithm constructs an Euler circuit or path by walking unused edges and splicing cycles.

For a directed graph, the implementation is direct:

```python id="wqxqxa"
def euler_path_directed(adj, start):
    graph = [neighbors[:] for neighbors in adj]
    stack = [start]
    path = []

    while stack:
        u = stack[-1]

        if graph[u]:
            v = graph[u].pop()
            stack.append(v)
        else:
            path.append(stack.pop())

    path.reverse()
    return path
```

The returned list contains vertices. If the graph has `m` edges, a valid Euler path contains `m + 1` vertices.

For an undirected graph, each edge is stored twice in an adjacency list, so each physical edge needs an identifier to avoid using it twice.

```python id="o55vnv"
def build_undirected_with_edge_ids(n, edges):
    adj = [[] for _ in range(n)]

    for eid, (u, v) in enumerate(edges):
        adj[u].append((v, eid))
        adj[v].append((u, eid))

    return adj
```

Then run Hierholzer while marking edge IDs:

```python id="v98eqd"
def euler_path_undirected(n, edges, start):
    adj = build_undirected_with_edge_ids(n, edges)
    used = [False] * len(edges)
    stack = [start]
    path = []

    while stack:
        u = stack[-1]

        while adj[u] and used[adj[u][-1][1]]:
            adj[u].pop()

        if not adj[u]:
            path.append(stack.pop())
        else:
            v, eid = adj[u].pop()
            used[eid] = True
            stack.append(v)

    path.reverse()
    return path
```

## Choosing the Start Vertex

For an undirected Euler circuit, start at any vertex with nonzero degree.

For an undirected Euler path that is not a circuit, start at one of the two odd-degree vertices.

```python id="ac3g69"
def choose_start_undirected(adj):
    odd = [u for u, neighbors in enumerate(adj) if len(neighbors) % 2 == 1]

    if odd:
        return odd[0]

    for u, neighbors in enumerate(adj):
        if neighbors:
            return u

    return 0
```

For a directed Euler circuit, start at any vertex with nonzero out-degree.

For a directed Euler path, start at the unique vertex where:

```text id="kwu4zh"
outdeg[u] = indeg[u] + 1
```

## Correctness Argument

For an Euler circuit in an undirected graph, every time the walk enters a vertex, it must also leave it by a different unused edge. Thus all degrees must be even. For an Euler path with different endpoints, every internal vertex still has paired entry and exit edges, while the start has one extra exit and the end has one extra entry. This creates exactly two odd-degree vertices.

For directed graphs, the same pairing principle applies with in-degree and out-degree. Internal vertices need equal in-degree and out-degree. The start of a non-circuit path has one extra outgoing edge. The end has one extra incoming edge.

Hierholzer's algorithm is correct because it never gets stuck at an internal vertex while unused incident edges remain, assuming the Euler conditions hold. Each local closed walk or trail is added into the growing answer. Since every edge is removed exactly once, the final reversed stack output contains every edge exactly once.

## Complexity

With adjacency lists, the tests and construction run in:

$$
O(n + m)
$$

Each vertex is inspected a constant number of times. Each edge is removed once in the directed case. In the undirected case, each stored half-edge may be skipped or popped once, so the total work remains linear.

The extra space is:

$$
O(n + m)
$$

for copied adjacency lists, stacks, and edge markers.

## Common Pitfalls

A common mistake is checking degree but not connectivity. A graph may have all even degrees and still consist of several disconnected cycles.

Another mistake is treating isolated vertices as a connectivity failure. Vertices with degree zero do not affect the existence of an Euler path over edges.

A third mistake is using an undirected adjacency list without edge IDs. Removing only one direction of an edge allows the reverse copy to be used again.

A fourth mistake is confusing Euler paths with Hamiltonian paths. Euler paths use every edge exactly once. Hamiltonian paths use every vertex exactly once.

## Takeaway

Euler path problems are edge traversal problems. The existence test is based on degree balance and connectivity among non-isolated vertices. Hierholzer's algorithm constructs the path in linear time once those conditions hold.
