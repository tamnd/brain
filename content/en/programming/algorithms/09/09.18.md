# 9.18 Graph Coloring

## Problem

You need to assign labels (colors) to vertices so that adjacent vertices receive different labels. The objective varies by problem:

- Decide whether a graph can be colored with at most `k` colors.
- Find a coloring using as few colors as possible.
- Construct any valid coloring under given constraints.

Graph coloring appears in scheduling, register allocation, frequency assignment, and constraint satisfaction.

## Solution

Assign colors while maintaining the invariant:

```text
for every edge (u, v), color[u] != color[v]
```

### Greedy Coloring

Process vertices in some order and assign each vertex the smallest color not used by its neighbors.

```python id="3x3c6v"
def greedy_coloring(adj):
    n = len(adj)
    color = [-1] * n

    for u in range(n):
        used = set()

        for v in adj[u]:
            if color[v] != -1:
                used.add(color[v])

        c = 0
        while c in used:
            c += 1

        color[u] = c

    return color
```

This uses at most `Δ + 1` colors, where `Δ` is the maximum degree. The result depends on vertex order.

### Ordering Heuristics

Better results come from ordering vertices carefully.

- Largest degree first
- Smallest degree last (degeneracy ordering)
- DSATUR (choose vertex with most distinct neighbor colors)

Example of degree ordering:

```python id="z0y4zv"
def order_by_degree(adj):
    return sorted(range(len(adj)), key=lambda u: -len(adj[u]))
```

Then apply greedy coloring in that order.

### Backtracking for k-Coloring

To test whether a graph is k-colorable, use backtracking.

```python id="qk4b9k"
def k_coloring(adj, k):
    n = len(adj)
    color = [-1] * n

    def assign(u):
        if u == n:
            return True

        for c in range(k):
            ok = True
            for v in adj[u]:
                if color[v] == c:
                    ok = False
                    break

            if ok:
                color[u] = c
                if assign(u + 1):
                    return True
                color[u] = -1

        return False

    if assign(0):
        return color
    return None
```

This runs in exponential time in the worst case.

## Basic Example

Consider:

```text
0 - 1 - 2
 \  |
   3
```

A valid coloring is:

```text
0: 0
1: 1
2: 0
3: 2
```

Three colors are used.

For a bipartite graph, only two colors are needed.

## Chromatic Number

The chromatic number is the minimum number of colors needed.

- Complete graph with `n` vertices: `n` colors
- Bipartite graph: 2 colors
- Odd cycle: 3 colors

Computing the chromatic number is NP-hard.

## Correctness

Greedy coloring ensures correctness because it explicitly avoids assigning a color already used by neighbors.

Backtracking ensures correctness by exploring all assignments consistent with constraints.

## Complexity

| Method | Time | Space |
|---|---:|---:|
| Greedy | $O(n + m)$ | $O(n)$ |
| Backtracking | Exponential | $O(n)$ |

Heuristics improve practical performance but do not change worst-case complexity.

## Common Pitfalls

A common mistake is assuming greedy coloring gives the minimum number of colors. It does not guarantee optimality.

Another mistake is ignoring vertex ordering. Different orders can produce very different results.

A third mistake is confusing bipartite checking with general coloring. Bipartite graphs are a special case of 2-coloring.

A fourth mistake is using coloring when the problem is actually about partitioning edges or flows.

## Takeaway

Graph coloring assigns labels under adjacency constraints. Greedy methods are fast and simple. Exact solutions require exponential time in general. The structure of the graph determines how many colors are necessary.
