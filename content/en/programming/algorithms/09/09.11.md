# 9.11 Topological Sort

## Problem

You have a set of items with precedence constraints, and you need an ordering that respects those constraints. This appears in build systems, course prerequisites, task scheduling, dependency resolution, and compiler passes.

A topological order exists only for a directed acyclic graph. If the graph contains a directed cycle, no valid ordering exists.

## Solution

Model each item as a vertex. Add a directed edge:

```text
u -> v
```

when `u` must come before `v`.

Then compute a topological ordering.

One standard method is Kahn's algorithm. It repeatedly chooses vertices with in-degree zero.

```python
from collections import deque

def topological_sort(adj):
    n = len(adj)
    indeg = [0] * n

    for u in range(n):
        for v in adj[u]:
            indeg[v] += 1

    queue = deque()
    for u in range(n):
        if indeg[u] == 0:
            queue.append(u)

    order = []

    while queue:
        u = queue.popleft()
        order.append(u)

        for v in adj[u]:
            indeg[v] -= 1
            if indeg[v] == 0:
                queue.append(v)

    if len(order) != n:
        return None

    return order
```

If the function returns `None`, the graph has a directed cycle.

## Basic Example

Suppose a project has these dependencies:

```text
parse -> validate
validate -> compile
compile -> package
```

The graph is:

```text
0 -> 1 -> 2 -> 3
```

A valid topological order is:

```text
0, 1, 2, 3
```

For a graph with independent branches:

```text
0 -> 2
1 -> 2
2 -> 3
```

both of these are valid:

```text
0, 1, 2, 3
1, 0, 2, 3
```

Topological sort does not always produce a unique answer. It produces one order that respects all edges.

## In-Degree Meaning

The in-degree of a vertex counts how many unfinished prerequisites it has.

```text
indeg[v] = number of edges u -> v
```

A vertex with in-degree zero has no remaining prerequisites, so it can be placed next in the order.

When `u` is placed in the order, every outgoing edge `u -> v` is removed conceptually. This reduces the number of remaining prerequisites for `v`.

## Correctness Argument

Kahn's algorithm maintains the invariant that every vertex in the queue has no remaining incoming edges from unprocessed vertices. Therefore it is safe to place any queued vertex next.

When a vertex `u` is placed in the order, all edges leaving `u` are removed from consideration. If this causes `indeg[v]` to become zero, then all prerequisites of `v` have already been placed, so `v` becomes eligible.

If the algorithm places all vertices, then every edge points from an earlier vertex to a later vertex. This follows because an edge `u -> v` is removed only after `u` is placed, and `v` cannot become eligible through that edge until `u` is already in the order.

If the algorithm stops before placing all vertices, every remaining vertex has positive in-degree within the remaining subgraph. Following incoming edges forever in a finite graph must eventually repeat a vertex, which forms a directed cycle. Hence no topological order exists.

## DFS Method

Topological sort can also be computed with DFS. Visit every vertex, and append a vertex after all of its outgoing neighbors have been processed. Reverse the finish order.

```python
def topological_sort_dfs(adj):
    n = len(adj)
    state = [0] * n
    order = []

    def visit(u):
        if state[u] == 1:
            return False
        if state[u] == 2:
            return True

        state[u] = 1

        for v in adj[u]:
            if not visit(v):
                return False

        state[u] = 2
        order.append(u)
        return True

    for u in range(n):
        if state[u] == 0:
            if not visit(u):
                return None

    order.reverse()
    return order
```

The `state` array has three values:

```text
0 = unvisited
1 = currently visiting
2 = finished
```

A transition to a vertex with state `1` detects a directed cycle.

## Complexity

With an adjacency list, topological sort runs in:

$$
O(n + m)
$$

where `n` is the number of vertices and `m` is the number of edges.

Kahn's algorithm scans every edge once while computing in-degrees and once while removing edges conceptually. DFS also scans every edge once. Both use $O(n)$ extra space.

## Common Pitfalls

A common mistake is applying topological sort to an undirected graph. Topological order is defined for directed graphs.

Another mistake is forgetting to detect cycles. A partial order from Kahn's algorithm is not a valid answer when some vertices remain unprocessed.

A third mistake is reversing dependency direction. Decide whether `u -> v` means "`u` depends on `v`" or "`u` must come before `v`". This section uses the second convention.

A fourth mistake is expecting a unique order. Multiple valid topological orders are normal.

## Takeaway

Topological sort linearizes the constraints of a directed acyclic graph. Kahn's algorithm uses in-degrees to repeatedly select items with no remaining prerequisites. DFS uses finishing times. Both methods run in linear time and both must reject directed cycles.
