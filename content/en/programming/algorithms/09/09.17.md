# 9.17 Hamiltonian Paths

## Problem

You need to determine whether a graph contains a path that visits every vertex exactly once. This is a Hamiltonian path. If such a path starts and ends at the same vertex, it is a Hamiltonian cycle.

Unlike Euler paths, which are governed by local degree conditions, Hamiltonian paths depend on global structure. There is no simple necessary and sufficient condition that can be checked in linear time for general graphs.

## Solution

For general graphs, use search with pruning or dynamic programming on subsets.

### Backtracking (General Approach)

Try all possible paths, pruning when constraints are violated.

```python id="f3j3kv"
def has_hamiltonian_path(adj):
    n = len(adj)
    used = [False] * n

    def dfs(u, count):
        if count == n:
            return True

        for v in adj[u]:
            if not used[v]:
                used[v] = True
                if dfs(v, count + 1):
                    return True
                used[v] = False

        return False

    for start in range(n):
        used[start] = True
        if dfs(start, 1):
            return True
        used[start] = False

    return False
```

This runs in factorial time in the worst case. It is only practical for small graphs or when strong pruning is available.

## Dynamic Programming on Subsets

For graphs with up to about 20 vertices, use bitmask DP.

Define:

```text
dp[mask][v] = True if there is a path that visits exactly the vertices in mask and ends at v
```

```python id="t8o1pl"
def hamiltonian_path_dp(adj):
    n = len(adj)
    dp = [[False] * n for _ in range(1 << n)]

    for i in range(n):
        dp[1 << i][i] = True

    for mask in range(1 << n):
        for u in range(n):
            if not dp[mask][u]:
                continue

            for v in adj[u]:
                if mask & (1 << v):
                    continue

                dp[mask | (1 << v)][v] = True

    full = (1 << n) - 1

    for v in range(n):
        if dp[full][v]:
            return True

    return False
```

This runs in:

$$
O(n^2 2^n)
$$

and is feasible for small `n`.

## Basic Example

Consider:

```text
0 - 1 - 2
    |
    3
```

A Hamiltonian path is:

```text
3 -> 1 -> 0 -> 2
```

Every vertex appears exactly once.

Now consider:

```text
0 - 1 - 2
|
3
```

No Hamiltonian path exists that covers all vertices, because vertex `1` blocks movement between `2` and `3`.

## Directed Graphs

For directed graphs, edges must follow direction.

```text
0 -> 1 -> 2 -> 3
```

has a Hamiltonian path.

But:

```text
0 -> 1
2 -> 1
```

has no Hamiltonian path visiting all vertices, because `2` cannot be placed after `1`, and `1` must come after both `0` and `2`.

The same algorithms apply, but neighbor iteration respects direction.

## Special Cases

Some graph classes admit efficient solutions.

- DAGs: longest path DP can detect Hamiltonian path in linear time
- Complete graphs: always Hamiltonian
- Tournaments: always contain a Hamiltonian path
- Trees: Hamiltonian path exists only when the tree is a path

### DAG Case

For a DAG, compute topological order and check consecutive edges.

```python id="8u38r3"
def hamiltonian_in_dag(adj, topo):
    for i in range(len(topo) - 1):
        u = topo[i]
        v = topo[i + 1]
        if v not in adj[u]:
            return False
    return True
```

If every consecutive pair is connected, the topological order itself is a Hamiltonian path.

## Correctness Intuition

A Hamiltonian path must choose exactly one ordering of vertices such that every consecutive pair is connected.

Backtracking explores all such orderings, ensuring correctness by exhaustive search.

DP encodes partial paths using subsets. Each state records whether a path exists that visits a given subset and ends at a specific vertex. Transitions extend paths by adding one vertex at a time.

## Complexity

| Method | Time | Space |
|---|---:|---:|
| Backtracking | $O(n!)$ | $O(n)$ |
| DP (bitmask) | $O(n^2 2^n)$ | $O(n 2^n)$ |

This is fundamentally harder than Euler paths, which run in linear time.

## Common Pitfalls

A common mistake is confusing Hamiltonian and Euler paths. Euler paths use all edges once. Hamiltonian paths use all vertices once.

Another mistake is expecting a simple degree condition. No such condition exists for general graphs.

A third mistake is using BFS or DFS directly. These find some path, not necessarily one that covers all vertices.

A fourth mistake is ignoring direction in directed graphs. Edge direction restricts valid paths.

## Takeaway

Hamiltonian path problems are global ordering problems. Unlike Euler paths, they have no simple local characterization. General solutions require exponential time, with practical algorithms relying on backtracking or subset dynamic programming.
