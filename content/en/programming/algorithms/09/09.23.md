# 9.23 Common Bugs

## Problem

Graph algorithms are sensitive to small representation errors. A single missing reverse edge, wrong indexing convention, or late visited mark can make the algorithm return a plausible but wrong answer.

This section collects common graph bugs and shows how to detect them early.

## Bug 1: Forgetting Reverse Edges in an Undirected Graph

For an undirected graph, every edge must be stored in both directions.

```python id="o3fgmj"
adj[u].append(v)
adj[v].append(u)
```

If you store only one direction, the graph becomes directed by accident.

A simple test catches this:

```python id="ccdv7d"
def check_undirected_symmetry(adj):
    edge_set = set()

    for u in range(len(adj)):
        for v in adj[u]:
            edge_set.add((u, v))

    for u, v in edge_set:
        assert (v, u) in edge_set
```

## Bug 2: Adding Reverse Edges in a Directed Graph

For a directed graph, this is wrong unless the problem explicitly says edges are bidirectional:

```python id="cq6c1g"
adj[u].append(v)
adj[v].append(u)
```

This bug often hides in helper functions named `add_edge`. Use separate helpers:

```python id="yffvzy"
def add_directed(adj, u, v):
    adj[u].append(v)

def add_undirected(adj, u, v):
    adj[u].append(v)
    adj[v].append(u)
```

## Bug 3: One-Based vs Zero-Based Indexing

Many inputs number vertices from `1` to `n`, while arrays use indices from `0` to `n - 1`.

Convert once at the input boundary:

```python id="elhj4j"
u -= 1
v -= 1
```

Do not mix converted and unconverted IDs in the same program.

## Bug 4: Ignoring Isolated Vertices

An isolated vertex has no incident edges, but it still belongs to the graph.

```text id="t2nloe"
n = 4
edges = [(0, 1), (1, 2)]
```

Vertex `3` exists even though it appears in no edge.

Any algorithm that processes all components must loop over all vertices:

```python id="xgnfrk"
for u in range(n):
    if not seen[u]:
        dfs(u)
```

## Bug 5: Marking Visited Too Late

In DFS and BFS, mark a vertex when it is discovered, not after all its neighbors are processed.

Good BFS:

```python id="25b76r"
dist[start] = 0
queue.append(start)

while queue:
    u = queue.popleft()

    for v in adj[u]:
        if dist[v] == -1:
            dist[v] = dist[u] + 1
            queue.append(v)
```

If you delay marking, cycles may enqueue or recurse into the same vertex many times.

## Bug 6: Using DFS for Shortest Paths

DFS finds a path, not necessarily the shortest path.

Use BFS for shortest paths in unweighted graphs. Use Dijkstra or Bellman-Ford for weighted graphs.

```text id="ytcsnm"
DFS: reachability and structure
BFS: shortest path by number of edges
Dijkstra: shortest path with nonnegative weights
Bellman-Ford: shortest path with negative edges
```

## Bug 7: Using BFS on Weighted Graphs

BFS treats every edge as cost `1`. If weights differ, BFS gives the fewest-edge path, not the minimum-cost path.

Example:

```text id="mbutpg"
0 -> 1 weight 100
0 -> 2 weight 1
2 -> 1 weight 1
```

BFS may return `0 -> 1`, but the cheaper path is `0 -> 2 -> 1`.

## Bug 8: Using Dijkstra with Negative Edges

Dijkstra's algorithm assumes nonnegative weights. With negative edges, its greedy finalization step can be invalid.

Check the input contract:

```python id="h5hmpk"
for u, v, w in edges:
    assert w >= 0
```

Use Bellman-Ford if negative edges are allowed.

## Bug 9: Wrong Parent Check in Undirected Cycle Detection

In an undirected graph, seeing the parent again is normal. It does not prove a cycle.

Correct check:

```python id="51zdjp"
elif v != parent:
    return True
```

Without this condition, every undirected edge looks like a cycle.

## Bug 10: Over-Specifying Traversal Order in Tests

DFS and BFS output order depends on adjacency list order.

Instead of testing exact order:

```python id="4wg8k9"
assert order == [0, 1, 2, 3]
```

test the property you need:

```python id="882olb"
assert set(order) == {0, 1, 2, 3}
```

For BFS distances, exact distances are usually stable. For DFS order, exact order is often arbitrary unless neighbors are sorted.

## Bug 11: Reusing Mutable Rows in a Matrix

This Python code is wrong:

```python id="6dfef8"
matrix = [[False] * n] * n
```

It creates `n` references to the same row.

Use:

```python id="cof19m"
matrix = [[False for _ in range(n)] for _ in range(n)]
```

The same issue appears with adjacency lists:

```python id="t9g6iy"
adj = [[]] * n
```

Use:

```python id="yvuflj"
adj = [[] for _ in range(n)]
```

## Bug 12: Losing Edge IDs in Undirected Edge Algorithms

Some algorithms need to distinguish physical edges. This matters with bridges, Euler paths, and multigraphs.

If the graph has parallel edges:

```text id="ofda35"
0 - 1
0 - 1
```

then checking only `v == parent` may skip too much. Use edge IDs when parallel edges matter.

```python id="2gwtk2"
adj[u].append((v, edge_id))
adj[v].append((u, edge_id))
```

## Bug 13: Treating Duplicate Edges Accidentally

Duplicate edges may be legal or illegal depending on the graph model.

For a simple graph, normalize or reject duplicates.

```python id="kavfvs"
seen_edges = set()

for u, v in edges:
    key = (min(u, v), max(u, v))
    if key in seen_edges:
        raise ValueError("duplicate edge")
    seen_edges.add(key)
```

For a multigraph, keep duplicates.

## Bug 14: Forgetting Self Loops

A self loop is an edge from a vertex to itself.

```text id="un20q7"
u -> u
```

In a directed graph, a self loop is a cycle. In an undirected graph, it affects degree conventions and may invalidate assumptions for simple graphs.

Decide the policy explicitly:

```python id="j8ekib"
if u == v:
    raise ValueError("self loop not allowed")
```

## Bug 15: Not Checking Cycle Failure in Topological Sort

Kahn's algorithm must verify that every vertex was processed.

```python id="p7xh15"
if len(order) != n:
    return None
```

Returning a partial order for a cyclic graph is a bug.

## Debugging Checklist

Before blaming the algorithm, check the graph contract:

| Check | Question |
|---|---|
| Vertex range | Are IDs valid after indexing conversion? |
| Direction | Did you insert exactly the intended directions? |
| Weights | Are weights preserved and interpreted correctly? |
| Duplicates | Are parallel edges legal? |
| Self loops | Are loops legal? |
| Isolated vertices | Does `n` include them? |
| Connectivity | Are all components processed? |
| Traversal state | Are vertices marked at discovery time? |

## Takeaway

Most graph bugs are contract bugs. The algorithm assumes a model, and the representation must match that model exactly. Test direction, isolated vertices, duplicates, self loops, and disconnected graphs before testing large inputs.
