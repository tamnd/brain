# 9.7 Degree and Connectivity

## Problem

You need to understand the local and global structure of a graph. Degree measures how many edges touch a vertex. Connectivity measures whether vertices can reach each other through paths. These concepts appear in traversal, component detection, graph validation, scheduling, network analysis, and many correctness proofs.

## Solution

Compute degree from the graph representation, then use traversal to compute connectivity.

For an undirected graph, the degree of a vertex is the number of incident edges:

$$
\deg(u) = |\{v : \{u, v\} \in E\}|
$$

For a directed graph, degree splits into two values:

$$
\operatorname{outdeg}(u) = |\{v : (u, v) \in E\}|
$$

$$
\operatorname{indeg}(u) = |\{v : (v, u) \in E\}|
$$

Connectivity is defined by paths. In an undirected graph, two vertices are connected if there is a path between them. In a directed graph, reachability must respect edge direction.

## Computing Degree

For an undirected adjacency list, the degree of `u` is the length of its neighbor list:

```python id="l9sfn4"
def degrees_undirected(adj):
    return [len(neighbors) for neighbors in adj]
```

For a directed adjacency list, out-degree is immediate:

```python id="oi0i4q"
def out_degrees(adj):
    return [len(neighbors) for neighbors in adj]
```

In-degree requires scanning the edges:

```python id="1ypx1a"
def in_degrees(adj):
    n = len(adj)
    indeg = [0] * n

    for u in range(n):
        for v in adj[u]:
            indeg[v] += 1

    return indeg
```

The total cost is $O(n + m)$, where $n$ is the number of vertices and $m$ is the number of stored edges.

## Handshaking Principle

In an undirected graph, the sum of all degrees equals twice the number of edges:

$$
\sum_{v \in V} \deg(v) = 2|E|
$$

Each undirected edge contributes one to the degree of each endpoint.

In a directed graph, the total in-degree equals the total out-degree:

$$
\sum_{v \in V} \operatorname{indeg}(v)
=
\sum_{v \in V} \operatorname{outdeg}(v)
=
|E|
$$

Each directed edge contributes one outgoing edge at its source and one incoming edge at its target.

These identities are useful for sanity checks. If your computed degree totals violate them, the representation or counting logic is wrong.

## Connected Components

In an undirected graph, a connected component is a maximal group of vertices that can reach each other.

Use DFS or BFS to label components:

```python id="jhuc2d"
from collections import deque

def connected_components(adj):
    n = len(adj)
    comp = [-1] * n
    comp_id = 0

    for start in range(n):
        if comp[start] != -1:
            continue

        queue = deque([start])
        comp[start] = comp_id

        while queue:
            u = queue.popleft()

            for v in adj[u]:
                if comp[v] == -1:
                    comp[v] = comp_id
                    queue.append(v)

        comp_id += 1

    return comp, comp_id
```

The array `comp` stores the component identifier for each vertex. Two vertices `u` and `v` are in the same connected component exactly when:

```python id="0jk6no"
comp[u] == comp[v]
```

## Example

For this graph:

```text id="ip0rrx"
0 - 1 - 2

3 - 4

5
```

the components are:

```text id="ohd5et"
component 0: 0, 1, 2
component 1: 3, 4
component 2: 5
```

Vertex `5` is an isolated vertex. It has degree zero, but it still forms a component by itself.

## Directed Connectivity

Directed graphs have several different notions of connectivity.

A vertex `v` is reachable from `u` if there is a directed path from `u` to `v`.

A directed graph is strongly connected if every vertex can reach every other vertex.

A directed graph is weakly connected if the underlying undirected graph is connected after ignoring edge direction.

For example:

```text id="v2b4dc"
0 -> 1 -> 2
```

This graph is weakly connected, because ignoring direction gives one connected chain. It is not strongly connected, because `2` cannot reach `0`.

## Weak Connectivity

To compute weak components, treat each directed edge as undirected:

```python id="f2klam"
def weak_components(n, edges):
    adj = [[] for _ in range(n)]

    for u, v in edges:
        adj[u].append(v)
        adj[v].append(u)

    return connected_components(adj)
```

This is useful when direction matters for some algorithms but the first question is whether the graph is connected at all.

## Strong Connectivity Check

A simple way to test whether a directed graph is strongly connected is to run traversal twice: once on the graph and once on its reverse.

```python id="85nnmk"
def reachable_from(start, adj):
    n = len(adj)
    seen = [False] * n
    stack = [start]
    seen[start] = True

    while stack:
        u = stack.pop()

        for v in adj[u]:
            if not seen[v]:
                seen[v] = True
                stack.append(v)

    return seen
```

Build the reverse graph:

```python id="43bxv5"
def reverse_graph(adj):
    n = len(adj)
    rev = [[] for _ in range(n)]

    for u in range(n):
        for v in adj[u]:
            rev[v].append(u)

    return rev
```

Check strong connectivity:

```python id="qz8iea"
def is_strongly_connected(adj):
    if not adj:
        return True

    seen_forward = reachable_from(0, adj)
    if not all(seen_forward):
        return False

    rev = reverse_graph(adj)
    seen_reverse = reachable_from(0, rev)

    return all(seen_reverse)
```

If every vertex is reachable from `0`, and every vertex can reach `0`, then every vertex can reach every other vertex. The reverse traversal verifies the second condition.

## Correctness Argument

For undirected connected components, BFS or DFS starts from an unassigned vertex and follows exactly the edges that preserve reachability. Every vertex discovered from `start` is connected to `start` by the path used during traversal. Conversely, if a vertex is connected to `start`, then some path leads to it, and traversal eventually follows each edge on that path. Therefore the traversal labels exactly one component.

For the directed strong connectivity check, the first traversal proves that `0` can reach every vertex. The traversal on the reversed graph proves that every vertex can reach `0` in the original graph. For any two vertices `a` and `b`, vertex `a` can reach `0`, and `0` can reach `b`. Concatenating those paths gives a path from `a` to `b`. Hence the graph is strongly connected.

## Complexity

For adjacency lists:

| Operation | Cost |
|---|---:|
| Compute undirected degrees | $O(n)$ |
| Compute directed out-degrees | $O(n)$ |
| Compute directed in-degrees | $O(n + m)$ |
| Connected components | $O(n + m)$ |
| Reverse directed graph | $O(n + m)$ |
| Strong connectivity check | $O(n + m)$ |

The storage cost is $O(n + m)$.

## Common Pitfalls

A common error is forgetting isolated vertices. A vertex with no edges still belongs to the graph and forms its own connected component.

Another error is applying undirected connectivity logic to a directed graph. Weak connectivity and strong connectivity answer different questions.

A third error is computing in-degree by looking only at `len(adj[u])`. That gives out-degree, not in-degree.

A fourth error is double-counting undirected edges. In an adjacency list, each undirected edge appears twice. If you sum all neighbor list lengths, divide by two to recover the number of undirected edges.

## Takeaway

Degree is a local measure of how edges touch a vertex. Connectivity is a global measure of reachability. In undirected graphs, connected components partition the vertices. In directed graphs, distinguish reachability, weak connectivity, and strong connectivity before choosing an algorithm.
