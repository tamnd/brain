# 9.10 Connected Components

## Problem

You need to divide an undirected graph into maximal regions of mutual reachability. In many graph problems, the first useful operation is not to solve the whole graph at once, but to split it into independent connected components.

A connected component is a maximal set of vertices such that every vertex in the set can reach every other vertex in the set by following edges.

## Solution

Run BFS or DFS from every unvisited vertex. Each traversal discovers exactly one connected component. Assign the same component identifier to all vertices reached by that traversal.

```python
from collections import deque

def connected_components(adj):
    n = len(adj)
    comp = [-1] * n
    comp_count = 0

    for start in range(n):
        if comp[start] != -1:
            continue

        queue = deque([start])
        comp[start] = comp_count

        while queue:
            u = queue.popleft()

            for v in adj[u]:
                if comp[v] == -1:
                    comp[v] = comp_count
                    queue.append(v)

        comp_count += 1

    return comp, comp_count
```

The array `comp` stores the component id for each vertex. The value `comp_count` stores the number of connected components.

## Basic Example

Consider this graph:

```text
0 - 1 - 2

3 - 4

5
```

The adjacency list is:

```text
0: [1]
1: [0, 2]
2: [1]
3: [4]
4: [3]
5: []
```

The connected components are:

```text
component 0: 0, 1, 2
component 1: 3, 4
component 2: 5
```

Vertex `5` has no edges, but it is still a vertex. It forms a component of size one.

## Component Sizes

Many problems ask for the size of each component. Count vertices while traversing.

```python
from collections import deque

def component_sizes(adj):
    n = len(adj)
    comp = [-1] * n
    sizes = []

    for start in range(n):
        if comp[start] != -1:
            continue

        cid = len(sizes)
        size = 0
        queue = deque([start])
        comp[start] = cid

        while queue:
            u = queue.popleft()
            size += 1

            for v in adj[u]:
                if comp[v] == -1:
                    comp[v] = cid
                    queue.append(v)

        sizes.append(size)

    return comp, sizes
```

For the example above, `sizes` is:

```text
[3, 2, 1]
```

## Correctness Argument

When a traversal starts at an unassigned vertex `s`, every vertex it visits is connected to `s`, because each visit follows an edge from an already connected vertex. Therefore all discovered vertices belong in the same connected component.

Conversely, suppose a vertex `x` is connected to `s`. Then there is a path from `s` to `x`. The traversal examines every edge leaving each vertex it discovers. By induction along the path, the traversal eventually discovers every vertex on that path, including `x`. Therefore the traversal discovers all vertices in the component of `s`.

The outer loop starts a new traversal only from an unassigned vertex. Since previous traversals already assigned their entire components, each new traversal finds a different component. Hence the algorithm labels every vertex with exactly one connected component.

## Complexity

With an adjacency list, connected components run in:

$$
O(n + m)
$$

where `n` is the number of vertices and `m` is the number of stored edges.

Each vertex is assigned once. Each adjacency list is scanned once. In an undirected graph, each edge appears twice in the representation, but the total scan remains linear in the representation size.

The extra space is:

$$
O(n)
$$

for the component array and traversal queue or stack.

## Common Pitfalls

A common mistake is starting traversal only from vertex `0`. That finds only one component, not all components.

Another mistake is ignoring isolated vertices. Every vertex must be considered by the outer loop, even if its adjacency list is empty.

A third mistake is applying this exact algorithm to directed graphs and calling the result strongly connected components. Directed graphs need separate definitions: weak components ignore direction, while strongly connected components require mutual directed reachability.

## Takeaway

Connected components partition an undirected graph into independent reachable regions. The algorithm is a direct extension of BFS or DFS: start from each unvisited vertex, label everything reachable from it, then move to the next unvisited vertex.
