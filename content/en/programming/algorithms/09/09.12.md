# 9.12 Cycle Detection

## Problem

You need to determine whether a graph contains a cycle. The exact method depends on whether the graph is directed or undirected. This distinction matters because an undirected edge can be traversed in both directions, while a directed edge can only be followed from its source to its target.

A cycle is a path that starts and ends at the same vertex, using at least one edge. In a simple undirected graph, cycles usually require at least three distinct vertices. In a directed graph, a self loop is already a directed cycle.

## Solution

For an undirected graph, run DFS while remembering the parent vertex. If DFS reaches an already visited vertex that is not the parent, a cycle exists.

```python id="dpubtp"
def has_cycle_undirected(adj):
    n = len(adj)
    seen = [False] * n

    def visit(u, parent):
        seen[u] = True

        for v in adj[u]:
            if not seen[v]:
                if visit(v, u):
                    return True
            elif v != parent:
                return True

        return False

    for start in range(n):
        if not seen[start]:
            if visit(start, -1):
                return True

    return False
```

For a directed graph, run DFS with three states: unvisited, visiting, and finished. If DFS reaches a vertex that is currently visiting, a directed cycle exists.

```python id="92i8a6"
def has_cycle_directed(adj):
    n = len(adj)
    state = [0] * n

    def visit(u):
        state[u] = 1

        for v in adj[u]:
            if state[v] == 1:
                return True
            if state[v] == 0 and visit(v):
                return True

        state[u] = 2
        return False

    for start in range(n):
        if state[start] == 0:
            if visit(start):
                return True

    return False
```

The values in `state` mean:

```text id="3ukg3o"
0 = unvisited
1 = currently in the recursion stack
2 = fully processed
```

## Undirected Example

Consider:

```text id="5qdutq"
0 - 1
|   |
3 - 2
```

This graph has the cycle:

```text id="cbpetd"
0 - 1 - 2 - 3 - 0
```

When DFS enters `3` and sees neighbor `0`, vertex `0` is already visited and is not the parent of `3`. That non-parent visited edge proves a cycle.

Now consider:

```text id="0i4z5x"
0 - 1 - 2 - 3
```

This graph has no cycle. Every visited neighbor encountered during DFS is exactly the parent edge leading back to the previous vertex.

## Directed Example

Consider:

```text id="2h4pft"
0 -> 1 -> 2
     ^    |
     |    v
     4 <- 3
```

The directed cycle is:

```text id="vpwk86"
1 -> 2 -> 3 -> 4 -> 1
```

During DFS, reaching a vertex with state `1` means the vertex is already on the active recursion stack. The edge points back into the current path, which creates a directed cycle.

If DFS reaches a vertex with state `2`, that does not create a cycle. A finished vertex has already been fully explored and is no longer on the current path.

## Kahn's Algorithm Alternative

For directed graphs, topological sort can also detect cycles. If Kahn's algorithm cannot remove all vertices, then the graph contains a directed cycle.

```python id="oglllv"
from collections import deque

def has_cycle_directed_kahn(adj):
    n = len(adj)
    indeg = [0] * n

    for u in range(n):
        for v in adj[u]:
            indeg[v] += 1

    queue = deque()
    for u in range(n):
        if indeg[u] == 0:
            queue.append(u)

    removed = 0

    while queue:
        u = queue.popleft()
        removed += 1

        for v in adj[u]:
            indeg[v] -= 1
            if indeg[v] == 0:
                queue.append(v)

    return removed != n
```

This method is useful when cycle detection is part of dependency scheduling.

## Correctness Argument

In an undirected graph, DFS creates a forest. Each tree edge connects a vertex to the vertex that first discovered it. If DFS sees an already visited neighbor that is not the parent, then there are two distinct paths between the current vertex and that neighbor: the direct edge, and the path through the DFS tree. Together they form a cycle. Conversely, if the graph has a cycle, DFS will eventually traverse part of that cycle and encounter a previously visited vertex that is not the immediate parent.

In a directed graph, the recursion stack represents the current directed path from the DFS root to the active vertex. If an edge reaches a vertex already in the stack, then the path from that earlier vertex to the current vertex, plus the new edge back to the earlier vertex, forms a directed cycle. If no such edge is found, then every edge either goes to an unvisited vertex that is safely explored, or to a finished vertex that cannot be part of the current path. Hence no directed cycle exists.

## Complexity

With an adjacency list, both algorithms run in:

$$
O(n + m)
$$

where `n` is the number of vertices and `m` is the number of edges.

Each vertex is visited once. Each edge is inspected a constant number of times. The extra space is $O(n)$ for the visited or state array, plus recursion stack space.

## Common Pitfalls

A common undirected-graph mistake is forgetting the parent check. Without it, every edge back to the parent looks like a cycle.

A common directed-graph mistake is using only a Boolean visited array. That detects repeated visits but cannot distinguish an edge back into the active recursion stack from an edge to a finished vertex.

Another mistake is checking only from vertex `0`. Disconnected graphs require starting DFS from every unvisited vertex.

A final mistake is applying the undirected method to directed graphs. The parent rule does not detect all directed cycles and can also report the wrong result.

## Takeaway

Cycle detection depends on graph direction. In undirected graphs, a visited neighbor other than the parent proves a cycle. In directed graphs, an edge to a vertex currently on the DFS recursion stack proves a cycle. Both methods are linear in the size of the graph representation.
