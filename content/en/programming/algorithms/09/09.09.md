# 9.9 Breadth First Search

## Problem

You need to explore a graph in increasing distance from a starting vertex. This is the right traversal when every edge has the same cost and the task asks for the fewest number of edges, the nearest target, layers of reachability, or shortest paths in an unweighted graph.

Breadth first search, usually called BFS, visits all vertices at distance `0`, then all vertices at distance `1`, then all vertices at distance `2`, and so on.

## Solution

Use a queue. Mark the start vertex, assign distance `0`, and repeatedly remove the oldest vertex from the queue. When you discover an unvisited neighbor, assign its distance and append it to the back of the queue.

```python
from collections import deque

def bfs(start, adj):
    n = len(adj)
    dist = [-1] * n
    parent = [-1] * n

    queue = deque([start])
    dist[start] = 0

    while queue:
        u = queue.popleft()

        for v in adj[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + 1
                parent[v] = u
                queue.append(v)

    return dist, parent
```

The value `dist[v]` is the length of the shortest unweighted path from `start` to `v`. A value of `-1` means that `v` is not reachable from `start`.

## Basic Example

Consider this graph:

```text
0 - 1 - 3
|   |
2 - 4
```

One adjacency list is:

```text
0: [1, 2]
1: [0, 3, 4]
2: [0, 4]
3: [1]
4: [1, 2]
```

Starting from `0`, BFS assigns:

```text
dist[0] = 0
dist[1] = 1
dist[2] = 1
dist[3] = 2
dist[4] = 2
```

Vertices `1` and `2` are discovered before vertices `3` and `4` because they are closer to the start.

## Queue Invariant

BFS works because the queue preserves layer order. When a vertex `u` with distance `d` is removed from the queue, every undiscovered neighbor `v` receives distance `d + 1`.

The queue never contains a vertex of distance `d + 2` before all vertices of distance `d + 1` that were already discovered. This first-in, first-out rule is what separates BFS from DFS.

## Path Reconstruction

The `parent` array records how each vertex was first discovered.

```python
def reconstruct_path(parent, target):
    path = []

    while target != -1:
        path.append(target)
        target = parent[target]

    path.reverse()
    return path
```

To reconstruct a path from `start` to `target`:

```python
dist, parent = bfs(start, adj)

if dist[target] == -1:
    path = None
else:
    path = reconstruct_path(parent, target)
```

Because BFS first discovers each vertex by a shortest path, the reconstructed path has the minimum number of edges.

## Multi-Source BFS

Sometimes several starting vertices have distance `0`. This appears in problems involving nearest exits, nearest hospitals, infection spread, fire spread, or distance to any source.

```python
from collections import deque

def multi_source_bfs(sources, adj):
    n = len(adj)
    dist = [-1] * n
    queue = deque()

    for s in sources:
        dist[s] = 0
        queue.append(s)

    while queue:
        u = queue.popleft()

        for v in adj[u]:
            if dist[v] == -1:
                dist[v] = dist[u] + 1
                queue.append(v)

    return dist
```

This is equivalent to adding a new artificial source connected to all original sources by zero cost edges, then running BFS.

## BFS on a Grid

A grid can be treated as an implicit graph. Each cell is a vertex. Edges connect neighboring cells that can be entered.

```python
from collections import deque

def grid_bfs(grid, start):
    rows = len(grid)
    cols = len(grid[0])
    sr, sc = start

    dist = [[-1 for _ in range(cols)] for _ in range(rows)]
    queue = deque([(sr, sc)])
    dist[sr][sc] = 0

    directions = [(1, 0), (-1, 0), (0, 1), (0, -1)]

    while queue:
        r, c = queue.popleft()

        for dr, dc in directions:
            nr = r + dr
            nc = c + dc

            if not (0 <= nr < rows and 0 <= nc < cols):
                continue

            if grid[nr][nc] == "#":
                continue

            if dist[nr][nc] != -1:
                continue

            dist[nr][nc] = dist[r][c] + 1
            queue.append((nr, nc))

    return dist
```

The graph is never built explicitly. The neighbor list is generated from the current cell.

## Correctness Argument

BFS assigns `dist[start] = 0`, which is correct. Suppose BFS removes a vertex `u` with correct distance `d`. For each undiscovered neighbor `v`, the edge from `u` to `v` gives a path of length `d + 1`, so assigning `dist[v] = d + 1` is valid.

It remains to show that no shorter path to `v` exists. If there were a path to `v` of length less than `d + 1`, then the predecessor of `v` on that path would have distance less than `d`. BFS processes vertices in nondecreasing distance order, so that predecessor would have been processed before `u`, and `v` would already have been discovered. This contradicts the fact that `v` was undiscovered. Therefore the assigned distance is shortest.

## Complexity

With an adjacency list, BFS runs in:

$$
O(n + m)
$$

where `n` is the number of vertices and `m` is the number of stored edges.

Each vertex enters the queue at most once. Each adjacency list is scanned once. In an undirected graph, each edge appears twice in the adjacency representation, but the running time remains linear in the stored representation.

The extra space is:

$$
O(n)
$$

for the queue, distance array, and parent array.

## Common Pitfalls

A common mistake is marking a vertex when it is removed from the queue instead of when it is inserted. That can allow the same vertex to be enqueued many times.

Another mistake is using BFS on a weighted graph. BFS gives shortest paths by number of edges only. If edge weights differ, use a weighted shortest path algorithm.

A third mistake is forgetting unreachable vertices. Keep a sentinel such as `-1` to distinguish unreachable vertices from vertices at distance `0`.

A fourth mistake is using a list as a queue with `pop(0)` in Python. That makes each removal linear. Use `collections.deque`.

## Takeaway

BFS explores a graph by distance layers. It is the standard tool for reachability and shortest paths in unweighted graphs. The queue invariant is the essential idea: vertices are processed in nondecreasing distance from the source.
