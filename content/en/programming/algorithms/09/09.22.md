# 9.22 Common Graph Patterns

## Problem

Many graph problems look different on the surface but share the same underlying structure. Recognizing these patterns reduces problem solving time and prevents incorrect modeling.

## Solution

Map each problem to a small set of standard patterns. Then apply the corresponding algorithm.

## Pattern 1: Reachability

**Question form**

```text
Can we get from u to v?
```

**Model**

- Directed or undirected graph
- Unweighted

**Algorithm**

- DFS or BFS

**Example**

- Website navigation
- Game moves
- State transitions

---

## Pattern 2: Shortest Path (Unweighted)

**Question form**

```text
Minimum number of steps from u to v
```

**Model**

- Graph with equal edge cost

**Algorithm**

- BFS

**Example**

- Grid shortest path
- Minimum moves puzzle

---

## Pattern 3: Shortest Path (Weighted)

**Question form**

```text
Minimum cost path
```

**Model**

- Weighted graph

**Algorithm**

- Dijkstra if weights are nonnegative  
- Bellman-Ford if negative edges exist  

**Example**

- Road networks
- Cost optimization

---

## Pattern 4: Connected Components

**Question form**

```text
How many groups?
```

**Model**

- Undirected graph

**Algorithm**

- DFS or BFS

**Example**

- Social clusters
- Islands in grid

---

## Pattern 5: Cycle Detection

**Question form**

```text
Does a cycle exist?
```

**Model**

- Directed or undirected graph

**Algorithm**

- DFS with parent or recursion stack

**Example**

- Dependency loops
- Graph validity

---

## Pattern 6: Topological Ordering

**Question form**

```text
Find valid order of tasks
```

**Model**

- Directed acyclic graph

**Algorithm**

- Kahn or DFS topological sort

**Example**

- Build systems
- Course scheduling

---

## Pattern 7: Bipartite Graph

**Question form**

```text
Can nodes be split into two groups?
```

**Model**

- Undirected graph

**Algorithm**

- BFS or DFS coloring

**Example**

- Matching problems
- Conflict graphs

---

## Pattern 8: Strongly Connected Components

**Question form**

```text
Find mutually reachable groups
```

**Model**

- Directed graph

**Algorithm**

- Kosaraju or Tarjan

**Example**

- Compiler analysis
- Web graph clustering

---

## Pattern 9: Minimum Spanning Tree

**Question form**

```text
Connect all nodes with minimum cost
```

**Model**

- Weighted undirected graph

**Algorithm**

- Kruskal or Prim

**Example**

- Network design
- Infrastructure planning

---

## Pattern 10: Flow and Capacity

**Question form**

```text
Maximize flow from source to sink
```

**Model**

- Directed graph with capacities

**Algorithm**

- Ford-Fulkerson, Edmonds-Karp, Dinic

**Example**

- Network bandwidth
- Matching

---

## Pattern 11: Grid as Graph

**Question form**

```text
2D or 3D grid movement
```

**Model**

- Implicit graph

**Algorithm**

- BFS or DFS

**Example**

- Maze solving
- Image processing

---

## Pattern 12: State Space Graph

**Question form**

```text
Transitions between states
```

**Model**

- Nodes are states
- Edges are transitions

**Algorithm**

- BFS, DFS, or Dijkstra

**Example**

- Puzzle solving
- Game AI

---

## Pattern 13: Euler Path

**Question form**

```text
Use every edge exactly once
```

**Model**

- Graph with degree constraints

**Algorithm**

- Degree check + Hierholzer

**Example**

- Route planning
- Word chaining

---

## Pattern 14: Hamiltonian Path

**Question form**

```text
Visit every node exactly once
```

**Model**

- General graph

**Algorithm**

- Backtracking or DP

**Example**

- Traveling salesman
- Path covering

---

## Pattern 15: Graph Compression

**Question form**

```text
Reduce graph size while preserving structure
```

**Model**

- SCC or component contraction

**Algorithm**

- SCC + DAG

**Example**

- Optimization pipelines
- Dependency reduction

---

## Recognition Strategy

When you see a graph problem:

1. Identify whether edges have direction  
2. Check if weights exist  
3. Determine if you need paths, ordering, or grouping  
4. Match with a known pattern  

This mapping step is often more important than implementation.

## Common Pitfalls

A common mistake is solving a weighted problem with BFS. That ignores weights.

Another mistake is treating directed graphs as undirected. This changes reachability.

A third mistake is using DFS when shortest path is required.

A fourth mistake is not recognizing implicit graphs such as grids or states.

## Takeaway

Most graph problems reduce to a small set of patterns. Recognizing the pattern determines the correct algorithm. Implementation becomes routine once the mapping is correct.
