Case Study 1: Islands in a Grid

Problem:

Given a grid of land and water, count the number of islands.

Model:

Vertex: each land cell

Edge: between adjacent land cells

Direction: undirected

Weight: none

A land cell belongs to the same island as another land cell if they are connected by horizontal or vertical moves.

Python
def count_islands(grid):
    rows = len(grid)
    cols = len(grid[0])
    seen = [[False for _ in range(cols)] for _ in range(rows)]

    def dfs(r, c):
        seen[r][c] = True

        for dr, dc in [(1, 0), (-1, 0), (0, 1), (0, -1)]:
            nr = r + dr
            nc = c + dc

            if not (0 <= nr < rows and 0 <= nc < cols):
                continue

            if grid[nr][nc] == "0":
                continue

            if seen[nr][nc]:
                continue

            dfs(nr, nc)

    count = 0

    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "1" and not seen[r][c]:
                count += 1
                dfs(r, c)

    return count

The invariant is simple: after DFS starts from one unvisited land cell, it marks exactly the land cells in that island. The outer loop starts one DFS per island.

Case Study 2: Course Schedule

Problem:

Given courses and prerequisites, decide whether all courses can be completed.

Model:

Vertex: course

Edge: prerequisite to dependent course

Direction: directed

Weight: none

If course u must be completed before course v, add:

u -> v

Then check whether the graph has a directed cycle.

Python
from collections import deque

def can_finish_courses(n, prerequisites):
    adj = [[] for _ in range(n)]
    indeg = [0] * n

    for u, v in prerequisites:
        adj[u].append(v)
        indeg[v] += 1

    queue = deque()

    for u in range(n):
        if indeg[u] == 0:
            queue.append(u)

    finished = 0

    while queue:
        u = queue.popleft()
        finished += 1

        for v in adj[u]:
            indeg[v] -= 1
            if indeg[v] == 0:
                queue.append(v)

    return finished == n

If every course can be removed by the topological process, no cycle exists. If some courses remain, they depend on one another cyclically.

Case Study 3: Word Ladder

Problem:

Transform one word into another by changing one letter at a time.

Model:

Vertex: word

Edge: between words that differ by one character

Direction: undirected

Weight: one move

Use BFS because every transformation has the same cost.

Python
from collections import defaultdict, deque

def word_ladder_length(begin, end, words):
    words = set(words)
    if end not in words:
        return -1

    words.add(begin)
    buckets = defaultdict(list)

    for word in words:
        for i in range(len(word)):
            pattern = word[:i] + "*" + word[i + 1:]
            buckets[pattern].append(word)

    dist = {begin: 0}
    queue = deque([begin])

    while queue:
        word = queue.popleft()

        if word == end:
            return dist[word]

        for i in range(len(word)):
            pattern = word[:i] + "*" + word[i + 1:]

            for nxt in buckets[pattern]:
                if nxt not in dist:
                    dist[nxt] = dist[word] + 1
                    queue.append(nxt)

    return -1

The bucket structure avoids comparing every pair of words. The graph remains implicit: neighbors are generated by shared wildcard patterns.

Case Study 4: Detecting Build Cycles

Problem:

Given modules and import edges, detect whether the build graph has a cycle.

Model:

Vertex: module

Edge: module to imported module, or imported module to dependent module, depending on convention

Direction: directed

Weight: none

A DFS recursion stack detects cycles.

Python
def has_build_cycle(adj):
    n = len(adj)
    state = [0] * n

    def visit(u):
        state[u] = 1

        for v in adj[u]:
            if state[v] == 1:
                return True
            if state[v] == 0 and visit(v):
                return True

        state[u] = 2
        return False

    for u in range(n):
        if state[u] == 0:
            if visit(u):
                return True

    return False

The key invariant is that state 1 means the vertex is on the active DFS path. An edge to a state 1 vertex closes a directed cycle.

Case Study 5: Social Groups

Problem:

Given friendships, count social groups.

Model:

Vertex: person

Edge: friendship

Direction: undirected

Weight: none

This is connected components.

Python
def social_groups(n, friendships):
    adj = [[] for _ in range(n)]

    for u, v in friendships:
        adj[u].append(v)
        adj[v].append(u)

    seen = [False] * n
    groups = 0

    for start in range(n):
        if seen[start]:
            continue

        groups += 1
        stack = [start]
        seen[start] = True

        while stack:
            u = stack.pop()

            for v in adj[u]:
                if not seen[v]:
                    seen[v] = True
                    stack.append(v)

    return groups

The important edge case is an isolated person. They form a group of size one.

Case Study 6: Checking Whether Teams Can Be Split

Problem:

Given conflict pairs, decide whether people can be split into two teams so that no conflict is inside a team.

Model:

Vertex: person

Edge: conflict

Direction: undirected

Weight: none

This is bipartite checking.

Python
from collections import deque

def can_split_into_two_teams(n, conflicts):
    adj = [[] for _ in range(n)]

    for u, v in conflicts:
        adj[u].append(v)
        adj[v].append(u)

    color = [-1] * n

    for start in range(n):
        if color[start] != -1:
            continue

        color[start] = 0
        queue = deque([start])

        while queue:
            u = queue.popleft()

            for v in adj[u]:
                if color[v] == -1:
                    color[v] = 1 - color[u]
                    queue.append(v)
                elif color[v] == color[u]:
                    return False

    return True

A conflict within the same color class proves that no valid two-team split exists.

Case Study 7: Dependency Compression

Problem:

Group mutually dependent modules, then process groups in dependency order.

Model:

Vertex: module

Edge: dependency

Direction: directed

Weight: none

Compute SCCs, compress each SCC into one vertex, then topologically process the resulting DAG.

Python
def compress_dependencies(adj):
    comp, components = strongly_connected_components(adj)
    comp_count = len(components)

    dag = [set() for _ in range(comp_count)]

    for u in range(len(adj)):
        for v in adj[u]:
            cu = comp[u]
            cv = comp[v]

            if cu != cv:
                dag[cu].add(cv)

    return [list(neighbors) for neighbors in dag], components

The compression is useful because cycles inside one SCC become a single unit, while edges between units form an acyclic graph.

Testing the Cases

Use small structural tests before large examples.

Python
def test_course_schedule_cycle():
    assert not can_finish_courses(3, [(0, 1), (1, 2), (2, 0)])

def test_course_schedule_acyclic():
    assert can_finish_courses(3, [(0, 1), (1, 2)])

def test_social_groups():
    assert social_groups(5, [(0, 1), (1, 2), (3, 4)]) == 2

def test_team_split_triangle():
    assert not can_split_into_two_teams(3, [(0, 1), (1, 2), (2, 0)])

Each test targets one structural fact: a directed cycle, a directed acyclic chain, two undirected components, and an odd cycle.

Takeaway

Graph case studies follow a common sequence. First define vertices and edges. Then decide direction and weights. Then choose the representation that supports the algorithm's inner loop. Most graph problems become routine once the model is correct.
