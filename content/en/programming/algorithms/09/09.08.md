# 9.8 Depth First Search

## Problem

You need to explore a graph by following one path as far as possible before backtracking. This pattern is useful for reachability, connected components, cycle detection, topological sort, articulation points, bridges, and many graph decompositions.

Depth first search, usually called DFS, gives a disciplined way to perform this exploration.

## Solution

Maintain a visited set. Start from a vertex. Mark it as visited, then recursively visit each unvisited neighbor.

For an adjacency list:

```python
def dfs_recursive(u, adj, seen):
    seen[u] = True

    for v in adj[u]:
        if not seen[v]:
            dfs_recursive(v, adj, seen)
```

To search the whole graph, start DFS from every unvisited vertex:

```python
def dfs_all(adj):
    n = len(adj)
    seen = [False] * n

    for u in range(n):
        if not seen[u]:
            dfs_recursive(u, adj, seen)

    return seen
```

The outer loop is necessary because the graph may have several connected components.

## Basic Example

Consider the undirected graph:

```text
0 - 1 - 2
    |
    3
```

One possible adjacency list is:

```text
0: [1]
1: [0, 2, 3]
2: [1]
3: [1]
```

Starting at `0`, DFS may visit:

```text
0, 1, 2, 3
```

The exact order depends on the order of each neighbor list. If vertex `1` stores `[0, 3, 2]`, DFS may visit:

```text
0, 1, 3, 2
```

Both orders are valid. DFS specifies the exploration rule, not a unique order unless neighbor order is fixed.

## Recursive DFS

Recursive DFS mirrors the mathematical definition of graph reachability. A vertex is reachable from `u` if it is `u` itself or reachable from one of `u`'s neighbors.

```python
def reachable(start, adj):
    n = len(adj)
    seen = [False] * n

    def visit(u):
        seen[u] = True

        for v in adj[u]:
            if not seen[v]:
                visit(v)

    visit(start)
    return seen
```

After this function returns, `seen[v]` is true exactly when `v` is reachable from `start`.

## Iterative DFS

Recursive DFS can overflow the call stack on large graphs. An explicit stack avoids that problem.

```python
def dfs_iterative(start, adj):
    n = len(adj)
    seen = [False] * n
    stack = [start]
    seen[start] = True

    while stack:
        u = stack.pop()

        for v in adj[u]:
            if not seen[v]:
                seen[v] = True
                stack.append(v)

    return seen
```

This version marks a vertex when it is pushed. That prevents the same vertex from being pushed many times.

A second valid style marks a vertex when it is popped:

```python
def dfs_iterative_pop_mark(start, adj):
    n = len(adj)
    seen = [False] * n
    stack = [start]

    while stack:
        u = stack.pop()

        if seen[u]:
            continue

        seen[u] = True

        for v in adj[u]:
            if not seen[v]:
                stack.append(v)

    return seen
```

This style is sometimes simpler when the stack stores extra state, but it may push duplicates.

## Parent Array

DFS often records a parent for each discovered vertex. The parent array describes a DFS forest.

```python
def dfs_tree(start, adj):
    n = len(adj)
    seen = [False] * n
    parent = [-1] * n

    def visit(u):
        seen[u] = True

        for v in adj[u]:
            if not seen[v]:
                parent[v] = u
                visit(v)

    visit(start)
    return parent
```

If `parent[v] = u`, then DFS first discovered `v` from `u`.

This is useful for path reconstruction:

```python
def reconstruct_path(parent, target):
    path = []

    while target != -1:
        path.append(target)
        target = parent[target]

    path.reverse()
    return path
```

The reconstructed path is a path in the DFS tree, not necessarily a shortest path.

## Discovery and Finish Times

Some algorithms need to know when DFS enters and exits each vertex.

```python
def dfs_times(adj):
    n = len(adj)
    seen = [False] * n
    tin = [-1] * n
    tout = [-1] * n
    time = 0

    def visit(u):
        nonlocal time

        seen[u] = True
        tin[u] = time
        time += 1

        for v in adj[u]:
            if not seen[v]:
                visit(v)

        tout[u] = time
        time += 1

    for u in range(n):
        if not seen[u]:
            visit(u)

    return tin, tout
```

The entry time `tin[u]` records when DFS first discovers `u`. The exit time `tout[u]` records when all descendants of `u` have been processed.

These times support ancestor checks in a DFS tree:

```python
def is_ancestor(u, v, tin, tout):
    return tin[u] <= tin[v] and tout[v] <= tout[u]
```

This works because DFS processes a subtree as one contiguous interval.

## Correctness Argument

DFS maintains the invariant that every marked vertex is reachable from the starting vertex. The start vertex is reachable from itself. Whenever DFS moves from `u` to an unvisited neighbor `v`, there is already a path from the start to `u`, and the edge from `u` to `v` extends that path. Therefore `v` is reachable.

Conversely, suppose a vertex `x` is reachable from the start. Then there is a path:

$$
start = v_0, v_1, v_2, \ldots, v_k = x
$$

DFS examines all outgoing edges from every vertex it visits. Since it visits `v_0`, it eventually considers the edge to `v_1`; if `v_1` has not already been visited, DFS visits it. Repeating this argument along the path shows that DFS eventually visits `x`. Therefore DFS marks exactly the reachable vertices.

## Complexity

With an adjacency list, DFS runs in:

$$
O(n + m)
$$

where `n` is the number of vertices and `m` is the number of stored edges.

Each vertex is marked once. Each adjacency list is scanned once. In an undirected graph, each edge appears twice, but this is still linear in the representation size.

The extra space is:

$$
O(n)
$$

for the visited array and recursion stack or explicit stack.

## Common Pitfalls

A common mistake is marking vertices too late. If a vertex is marked only after all recursive calls return, cycles cause infinite recursion. Mark the vertex before visiting its neighbors.

Another mistake is assuming DFS finds shortest paths. DFS finds some path according to traversal order. Use BFS for shortest paths in unweighted graphs.

A third mistake is ignoring disconnected graphs. A single DFS from vertex `0` explores only the component reachable from `0`.

A fourth mistake is using recursive DFS on very deep graphs in languages with small recursion limits. Use an explicit stack for large inputs.

## Takeaway

DFS explores a graph by committing to one branch until it cannot go farther, then backtracking. It gives linear time reachability, component traversal, parent forests, and discovery intervals. Many deeper graph algorithms are refinements of this same pattern.
