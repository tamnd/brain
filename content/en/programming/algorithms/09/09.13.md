# 9.13 Bipartite Graphs

## Problem

You need to decide whether the vertices of an undirected graph can be split into two groups so that every edge goes between the groups, never inside one group. This structure appears in matching problems, scheduling problems, conflict graphs, assignment systems, and parity arguments.

A graph with such a split is called bipartite.

## Solution

Color the graph with two colors. Start from an uncolored vertex, give it color `0`, then give every neighbor the opposite color. Continue with BFS or DFS. If an edge ever connects two vertices with the same color, the graph is not bipartite.

```python id="w7h7wn"
from collections import deque

def is_bipartite(adj):
    n = len(adj)
    color = [-1] * n

    for start in range(n):
        if color[start] != -1:
            continue

        color[start] = 0
        queue = deque([start])

        while queue:
            u = queue.popleft()

            for v in adj[u]:
                if color[v] == -1:
                    color[v] = 1 - color[u]
                    queue.append(v)
                elif color[v] == color[u]:
                    return False, color

    return True, color
```

The array `color` stores the side of the bipartition:

```text id="5b69kv"
color[u] = 0 means u is on the left side
color[u] = 1 means u is on the right side
color[u] = -1 means u has not been assigned yet
```

The outer loop handles disconnected graphs.

## Basic Example

This graph is bipartite:

```text id="e2oirf"
0 - 1
|   |
3 - 2
```

One valid coloring is:

```text id="naz1p5"
color 0: 0, 2
color 1: 1, 3
```

Every edge crosses between the two color classes.

This graph is not bipartite:

```text id="umvp7u"
0
| \\n1 - 2
```

The triangle has three edges. If `0` has color `0`, then `1` and `2` must both have color `1`. But there is also an edge between `1` and `2`, so that edge connects equal colors.

## Why Odd Cycles Matter

An undirected graph is bipartite exactly when it has no odd cycle.

If a graph is bipartite, every edge alternates between the two sides. Along any cycle, the colors must alternate at each step. Returning to the starting vertex requires an even number of edges. Therefore every cycle has even length.

Conversely, if BFS coloring fails, it finds an edge whose endpoints already have the same color. The BFS tree paths from the component root to those endpoints have the same parity, and adding the edge between them creates an odd cycle. Thus a coloring conflict is not merely a local bug; it proves the structural obstruction.

## Returning the Two Parts

When the graph is bipartite, the color array directly gives the two parts.

```python id="0jvfcr"
def bipartition(adj):
    ok, color = is_bipartite(adj)
    if not ok:
        return None

    left = []
    right = []

    for u, c in enumerate(color):
        if c == 0:
            left.append(u)
        else:
            right.append(u)

    return left, right
```

For disconnected graphs, the split is not unique. Each connected component can have its colors flipped independently.

## DFS Version

The same idea can be written with DFS.

```python id="5v16vo"
def is_bipartite_dfs(adj):
    n = len(adj)
    color = [-1] * n

    def visit(u):
        for v in adj[u]:
            if color[v] == -1:
                color[v] = 1 - color[u]
                if not visit(v):
                    return False
            elif color[v] == color[u]:
                return False

        return True

    for start in range(n):
        if color[start] == -1:
            color[start] = 0
            if not visit(start):
                return False, color

    return True, color
```

BFS is often preferred because it makes the parity interpretation visible, but both versions are correct.

## Modeling Pattern

Bipartite graphs often arise when there are two kinds of objects.

Examples:

```text id="kwx1p6"
workers and jobs
students and courses
users and permissions
machines and tasks
```

Edges connect objects of different kinds:

```text id="5hsw64"
worker -> job
student -> course
user -> permission
machine -> task
```

In these cases the graph is bipartite by construction. The two vertex sets are already known.

Other times, the two sides are not known in advance. Then bipartite testing determines whether such a split exists.

## Correctness Argument

The algorithm maintains the invariant that every processed edge connects vertices of different colors. When an uncolored neighbor `v` is discovered from `u`, assigning `v` the opposite color makes the edge `u - v` valid. If `v` is already colored, the algorithm checks whether the existing color agrees with this requirement.

If the algorithm returns false, it has found an edge whose endpoints have the same color, so no two-coloring extending the previous assignments can exist. Since those assignments were forced by paths from the traversal root, the conflict proves that the component is not bipartite.

If the algorithm finishes without conflict, every edge has been inspected and every edge connects vertices of different colors. Therefore the color classes form a valid bipartition.

## Complexity

With an adjacency list, bipartite testing runs in:

$$
O(n + m)
$$

where `n` is the number of vertices and `m` is the number of stored edges.

Each vertex is colored at most once. Each edge is inspected a constant number of times. The extra space is $O(n)$ for the color array and the queue or recursion stack.

## Common Pitfalls

A common mistake is starting BFS from only one vertex. A disconnected graph can have several components, so every uncolored vertex must be considered.

Another mistake is treating a directed graph as if bipartiteness depended on edge direction. Bipartiteness is normally a property of the underlying undirected structure. For directed assignment graphs, ignore direction when checking whether the vertex set splits into two sides.

A third mistake is assuming the two sides have equal size. A bipartite graph may have parts of different sizes.

A fourth mistake is confusing bipartite graphs with complete bipartite graphs. A bipartite graph only requires all edges to cross between sides. It does not require every possible cross edge to exist.

## Takeaway

A graph is bipartite when its vertices can be colored with two colors so that every edge crosses colors. BFS or DFS coloring tests this in linear time. The central obstruction is an odd cycle.
