# 9.14 Strongly Connected Components

## Problem

You need to decompose a directed graph into regions where every vertex can reach every other vertex. Ordinary connected components do not preserve direction, so they are insufficient for directed graphs. Strongly connected components capture mutual directed reachability.

A strongly connected component, or SCC, is a maximal set of vertices $C$ such that for every pair of vertices $u, v \in C$, there is a directed path from $u$ to $v$ and a directed path from $v$ to $u$.

## Solution

Use Kosaraju's algorithm:

1. Run DFS on the original graph and record vertices in finishing order.
2. Reverse every edge to build the transpose graph.
3. Process vertices in reverse finishing order on the transpose graph.
4. Each DFS in the transpose graph gives one strongly connected component.

```python id="59gd0z"
def strongly_connected_components(adj):
    n = len(adj)
    seen = [False] * n
    order = []

    def dfs1(u):
        seen[u] = True

        for v in adj[u]:
            if not seen[v]:
                dfs1(v)

        order.append(u)

    for u in range(n):
        if not seen[u]:
            dfs1(u)

    rev = [[] for _ in range(n)]
    for u in range(n):
        for v in adj[u]:
            rev[v].append(u)

    comp = [-1] * n
    components = []

    def dfs2(u, cid):
        comp[u] = cid
        components[cid].append(u)

        for v in rev[u]:
            if comp[v] == -1:
                dfs2(v, cid)

    for u in reversed(order):
        if comp[u] == -1:
            components.append([])
            dfs2(u, len(components) - 1)

    return comp, components
```

The array `comp` maps each vertex to its component id. The list `components` stores the vertices in each component.

## Basic Example

Consider this directed graph:

```text id="u924xp"
0 -> 1 -> 2
^    |    |
|    v    v
3 <- 4 <- 5
```

Suppose the edges include:

```text id="22g4jt"
0 -> 1
1 -> 2
2 -> 5
5 -> 4
4 -> 1
1 -> 4
4 -> 3
3 -> 0
```

Here, every vertex can reach every other vertex, so the whole graph is one SCC.

Now remove the edge `3 -> 0`:

```text id="7duycu"
0 -> 1 -> 2
     |    |
     v    v
3 <- 4 <- 5
```

Then one SCC is:

```text id="xpvqdy"
1, 2, 4, 5
```

because those vertices can circulate through directed paths. Vertex `0` can reach that component but cannot be reached from it. Vertex `3` can be reached from the component but cannot reach back to it.

## Why Reversing Works

The transpose graph reverses every edge:

```text id="v2vczv"
u -> v becomes v -> u
```

Reversing edges does not change which vertices belong to the same SCC. If `u` can reach `v` and `v` can reach `u` in the original graph, then after reversing every edge, `v` can reach `u` and `u` can reach `v`. Mutual reachability remains mutual.

What changes is the direction between components. If one SCC has an edge to another SCC in the original graph, the edge goes the other way in the transpose graph. Kosaraju's algorithm uses finishing order from the first DFS to process source-like regions in the transpose graph without leaking into other components.

## Component Graph

After SCCs are computed, each component can be compressed into a single vertex. The result is called the condensation graph.

```python id="y7yalb"
def condensation_graph(adj, comp, comp_count):
    dag = [set() for _ in range(comp_count)]

    for u in range(len(adj)):
        for v in adj[u]:
            cu = comp[u]
            cv = comp[v]
            if cu != cv:
                dag[cu].add(cv)

    return [list(neighbors) for neighbors in dag]
```

The condensation graph is always a directed acyclic graph. If it had a directed cycle among components, those components would be mutually reachable and therefore should have been one SCC.

## Correctness Argument

The first DFS records vertices by finishing time. A vertex finishes only after all vertices reachable through its outgoing edges have been explored. At the component level, this means that sink regions in the reachable structure finish earlier, while source regions among remaining unprocessed components finish later.

When the second pass processes vertices in reverse finishing order on the transpose graph, it starts from a component that has no outgoing edge to any unassigned component in the transpose graph. Therefore the DFS cannot escape into another unassigned component. It visits exactly the vertices mutually reachable with the start vertex, which is one SCC.

Repeating this process assigns one SCC at a time. Since every vertex is assigned exactly once, the algorithm returns the full SCC decomposition.

## Tarjan's Algorithm

Tarjan's algorithm computes SCCs in one DFS pass using discovery indices and low-link values. It avoids explicitly constructing the reverse graph.

```python id="a7239e"
def tarjan_scc(adj):
    n = len(adj)
    index = 0
    stack = []
    on_stack = [False] * n
    disc = [-1] * n
    low = [0] * n
    comp = [-1] * n
    components = []

    def dfs(u):
        nonlocal index

        disc[u] = index
        low[u] = index
        index += 1

        stack.append(u)
        on_stack[u] = True

        for v in adj[u]:
            if disc[v] == -1:
                dfs(v)
                low[u] = min(low[u], low[v])
            elif on_stack[v]:
                low[u] = min(low[u], disc[v])

        if low[u] == disc[u]:
            cid = len(components)
            component = []

            while True:
                v = stack.pop()
                on_stack[v] = False
                comp[v] = cid
                component.append(v)

                if v == u:
                    break

            components.append(component)

    for u in range(n):
        if disc[u] == -1:
            dfs(u)

    return comp, components
```

Kosaraju is often easier to explain and debug. Tarjan is often more convenient when you want a single traversal and no transpose graph.

## Complexity

Both Kosaraju's algorithm and Tarjan's algorithm run in:

$$
O(n + m)
$$

where `n` is the number of vertices and `m` is the number of directed edges.

Kosaraju uses $O(n + m)$ extra space for the reverse graph. Tarjan uses $O(n)$ extra space beyond the input graph.

## Common Pitfalls

A common mistake is using undirected connected components on a directed graph. That computes weak components, not strongly connected components.

Another mistake is forgetting to process vertices in reverse finishing order in Kosaraju's algorithm. The order is essential.

A third mistake is reversing the graph incorrectly. For every edge `u -> v`, the transpose must contain `v -> u`.

A fourth mistake is assuming SCC output order has semantic meaning. Component ids depend on traversal order unless you explicitly sort or topologically order the condensation graph.

## Takeaway

Strongly connected components partition a directed graph into maximal regions of mutual reachability. Kosaraju's algorithm uses finishing times and the reversed graph. Tarjan's algorithm uses low-link values in one DFS. After compression, the component graph is always a DAG.
