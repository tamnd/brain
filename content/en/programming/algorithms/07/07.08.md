# 7.8 Interval Search

Interval search deals with locating, inserting, and querying ranges over ordered data. The core idea is to reduce interval problems to boundary searches using lower bound and upper bound. Once boundaries are located, intervals become simple slices or index ranges.

## Problem

Given a sorted list of disjoint intervals:

```text id="p5n7yr"
intervals = [(1, 3), (5, 7), (10, 15)]
```

Support the following operations:

1. Check whether a point `x` lies inside any interval  
2. Insert a new interval and merge overlaps  
3. Find all intervals overlapping a query range `[L, R]`  

## Point Query

To check whether a point lies inside any interval, search for the rightmost interval whose start is less than or equal to `x`.

```python id="j7rxsr"
def contains(intervals: list[tuple[int, int]], x: int) -> bool:
    lo = 0
    hi = len(intervals)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if intervals[mid][0] <= x:
            lo = mid + 1
        else:
            hi = mid

    i = lo - 1

    if i >= 0 and intervals[i][0] <= x <= intervals[i][1]:
        return True

    return False
```

The binary search finds the last interval whose start is less than or equal to `x`. Only that interval can contain `x`.

## Interval Overlap Query

To find all intervals overlapping `[L, R]`, first locate the first interval that could overlap.

```python id="ph8q7u"
def overlapping(intervals: list[tuple[int, int]], L: int, R: int):
    # find first interval with end >= L
    lo = 0
    hi = len(intervals)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if intervals[mid][1] >= L:
            hi = mid
        else:
            lo = mid + 1

    result = []
    i = lo

    while i < len(intervals) and intervals[i][0] <= R:
        result.append(intervals[i])
        i += 1

    return result
```

The binary search finds the first interval whose end is not less than `L`. From there, scan forward until intervals no longer overlap.

## Insertion and Merge

To insert a new interval `[L, R]`, merge all overlapping intervals and keep the list sorted and disjoint.

```python id="0k8u5q"
def insert_interval(intervals: list[tuple[int, int]], L: int, R: int):
    result = []

    i = 0
    n = len(intervals)

    # add all intervals before overlap
    while i < n and intervals[i][1] < L:
        result.append(intervals[i])
        i += 1

    # merge overlapping intervals
    start = L
    end = R

    while i < n and intervals[i][0] <= R:
        start = min(start, intervals[i][0])
        end = max(end, intervals[i][1])
        i += 1

    result.append((start, end))

    # add remaining intervals
    while i < n:
        result.append(intervals[i])
        i += 1

    return result
```

Binary search can be used to skip directly to the first overlapping interval, but the linear scan is often acceptable because only overlapping intervals are processed.

## Lower Bound Interpretation

Interval search can be expressed in terms of lower bounds.

For point query:

```text id="8x0y7y"
find largest i such that intervals[i].start <= x
```

For overlap query:

```text id="vt7r6z"
find smallest i such that intervals[i].end >= L
```

These are boundary searches over sorted keys.

## Example

```text id="u6z0qe"
intervals = [(1, 3), (5, 7), (10, 15)]
```

Query:

```text id="y9xtux"
x = 6
```

Binary search finds index `1`, interval `(5, 7)`. The point lies inside.

Query:

```text id="d8r5nk"
[L, R] = [6, 12]
```

Binary search finds index `1`. Overlapping intervals are:

```text id="s1kp5y"
(5, 7), (10, 15)
```

## Complexity

Point query:

```text id="dlx7zk"
O(log n)
```

Overlap query:

```text id="1nyo7w"
O(log n + k)
```

where `k` is the number of overlapping intervals.

Insertion:

```text id="7q3zy9"
O(n)
```

because intervals may need to be shifted or merged.

## Interval Trees

For frequent interval queries and updates, specialized structures such as interval trees or segment trees provide better asymptotic guarantees.

An interval tree stores intervals in a balanced tree augmented with maximum endpoint values. Queries can skip entire subtrees that cannot overlap.

Segment trees and Fenwick trees handle numeric ranges efficiently, especially when intervals correspond to numeric indices.

## Common Pitfalls

Assuming intervals are sorted by both start and end. Only one ordering is guaranteed. The code must rely on the chosen ordering consistently.

Forgetting that intervals may touch at endpoints. Decide whether `[a, b]` and `[b, c]` overlap or not.

Mixing inclusive and exclusive boundaries leads to off-by-one errors.

Using binary search without maintaining sorted invariants after insertion.

Scanning too far after binary search. Stop when the overlap condition fails.

## Takeaway

Interval problems reduce to boundary detection over sorted keys. Binary search locates candidate intervals in logarithmic time. Once the starting point is known, local scanning or merging completes the task.
