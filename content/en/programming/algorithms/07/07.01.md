# 7.1 Binary Search Template

Binary search is a method for finding a boundary in an ordered search space. The search space is usually an array of indices, but the same idea applies to integers, real numbers, capacities, times, costs, and any ordered domain where a yes-or-no test becomes true once and then stays true.

The safest way to understand binary search is to stop thinking of it as "looking for a value." It is better to think of it as "finding the first position where a predicate becomes true."

Let `P(i)` be a predicate over indices. Binary search applies when the values of `P` have this shape:

```text
false false false false true true true true
```

The goal is to find the first `true`.

## Problem

Given a sorted array `a` and a target value `x`, find the first index `i` such that:

```text
a[i] >= x
```

If no such index exists, return `len(a)`.

This operation is often called `lower_bound`.

## Solution

Use a half open interval `[lo, hi)`. Maintain the invariant that the answer is always inside this interval. At each step, test the middle index. If the middle value is large enough, the answer is at `mid` or to its left. Otherwise, the answer is strictly to the right of `mid`.

```python
def lower_bound(a: list[int], x: int) -> int:
    lo = 0
    hi = len(a)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if a[mid] >= x:
            hi = mid
        else:
            lo = mid + 1

    return lo
```

The interval is half open. The left endpoint `lo` is included. The right endpoint `hi` is excluded. This convention makes the empty interval natural: when `lo == hi`, there is no remaining position to inspect, and that position is the boundary.

## Example

Consider:

```text
a = [2, 4, 4, 7, 9]
x = 4
```

The predicate `a[i] >= 4` gives:

```text
i:       0      1      2      3      4
a[i]:    2      4      4      7      9
P(i):    false  true   true   true   true
```

The first true index is `1`, so `lower_bound(a, 4)` returns `1`.

For a target larger than every value:

```text
a = [2, 4, 4, 7, 9]
x = 10
```

The predicate is false everywhere:

```text
false false false false false
```

There is no valid array index, so the function returns `len(a)`, which is `5`.

## Invariant

The binary search loop maintains two facts.

First, every index strictly before `lo` is known to be false.

Second, every index at or after `hi` is known to be true, or lies outside the array as the artificial boundary at `len(a)`.

For `lower_bound`, this means:

```text
for all j < lo, a[j] < x
for all j >= hi, a[j] >= x
```

The second statement only applies to valid array indices. When `hi == len(a)`, it says that no true index has been found yet on the right side.

Initially:

```text
lo = 0
hi = len(a)
```

No index is before `lo`, so the first part is vacuously true. No valid index is at or after `hi`, so the second part is also vacuously true.

During the loop, choose:

```text
mid = lo + (hi - lo) // 2
```

If `a[mid] >= x`, then `mid` is true. Since the array is sorted, every index to the right of `mid` is also true. Therefore the first true index cannot be to the right of `mid`, and it is safe to set:

```text
hi = mid
```

If `a[mid] < x`, then `mid` is false. Since the array is sorted, every index to the left of `mid` is also false. Therefore the first true index must be strictly after `mid`, and it is safe to set:

```text
lo = mid + 1
```

Each step preserves the invariant and strictly shrinks the interval.

## Why the Loop Terminates

The interval length is:

```text
hi - lo
```

At the start of each iteration, this value is positive. After the update, the interval becomes smaller.

If the predicate is true at `mid`, then `hi` moves down to `mid`.

If the predicate is false at `mid`, then `lo` moves up to `mid + 1`.

In both cases, at least one candidate position is removed. Because the interval length is a nonnegative integer, the loop must eventually reach `lo == hi`.

At that point, the invariant says that every index before `lo` is false and every valid index from `lo` onward is true. Therefore `lo` is exactly the first true index.

## Generic Template

The same structure works for any monotone predicate.

```python
def first_true(n: int, pred) -> int:
    lo = 0
    hi = n

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if pred(mid):
            hi = mid
        else:
            lo = mid + 1

    return lo
```

This returns the first index `i` in `[0, n)` such that `pred(i)` is true. If no such index exists, it returns `n`.

For example, `lower_bound` can be written as:

```python
def lower_bound(a: list[int], x: int) -> int:
    return first_true(len(a), lambda i: a[i] >= x)
```

This form separates the search logic from the condition being searched. That separation is useful because most binary search bugs come from mixing these two concerns.

## Recipe: Search for an Exact Value

To find whether a sorted array contains `x`, use `lower_bound` first. Then check whether the returned index is inside the array and actually contains `x`.

```python
def binary_search(a: list[int], x: int) -> int:
    i = lower_bound(a, x)

    if i < len(a) and a[i] == x:
        return i

    return -1
```

This returns the first index containing `x`, or `-1` if `x` does not occur.

This is safer than writing a separate equality-based binary search because duplicate values are handled naturally. In the array:

```text
[2, 4, 4, 4, 9]
```

searching for `4` returns index `1`, the first occurrence.

## Common Variants

The first index where `a[i] >= x` is `lower_bound`.

```python
def lower_bound(a: list[int], x: int) -> int:
    return first_true(len(a), lambda i: a[i] >= x)
```

The first index where `a[i] > x` is `upper_bound`.

```python
def upper_bound(a: list[int], x: int) -> int:
    return first_true(len(a), lambda i: a[i] > x)
```

The last index where `a[i] <= x` can be derived from `upper_bound`.

```python
def last_leq(a: list[int], x: int) -> int:
    return upper_bound(a, x) - 1
```

The last index where `a[i] < x` can be derived from `lower_bound`.

```python
def last_lt(a: list[int], x: int) -> int:
    return lower_bound(a, x) - 1
```

These derived functions may return `-1` when no such index exists. That is not a bug. It is the natural boundary before the first array index.

## Common Pitfalls

Do not use `while lo <= hi` unless you deliberately choose a closed interval convention. Mixing closed intervals with half open updates is a common source of infinite loops.

Do not set `lo = mid` after a false predicate when `mid` can equal `lo`. Use `lo = mid + 1` so the interval shrinks.

Do not set `hi = mid - 1` in the half open template. The right endpoint is excluded, so the correct update is `hi = mid`.

Do not compute the midpoint as:

```python
mid = (lo + hi) // 2
```

in languages where integer addition can overflow. Python integers do not overflow, but the safer expression is portable:

```python
mid = lo + (hi - lo) // 2
```

Do not forget the final boundary check when searching for an exact value. `lower_bound` gives the first possible position, not proof that the target exists.

## Complexity

Each iteration cuts the remaining interval roughly in half. Starting with `n` candidate positions, after `k` iterations the remaining interval has size at most:

```text
n / 2^k
```

The loop stops when this quantity falls below `1`. Therefore the number of iterations is proportional to:

```text
log2(n)
```

The running time is:

```text
O(log n)
```

The algorithm uses only a constant number of variables, so the extra space is:

```text
O(1)
```

## Testing Checklist

Test an empty array.

```python
assert lower_bound([], 5) == 0
```

Test a target smaller than all elements.

```python
assert lower_bound([10, 20, 30], 5) == 0
```

Test a target larger than all elements.

```python
assert lower_bound([10, 20, 30], 40) == 3
```

Test duplicates.

```python
assert lower_bound([1, 2, 2, 2, 3], 2) == 1
assert upper_bound([1, 2, 2, 2, 3], 2) == 4
```

Test exact search.

```python
assert binary_search([1, 2, 2, 2, 3], 2) == 1
assert binary_search([1, 2, 2, 2, 3], 4) == -1
```
