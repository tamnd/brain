# 7.1 Binary Search Template

Binary search is often taught as a pattern to memorize, but it is better understood as a consequence of a loop invariant. When you derive the implementation from the invariant, the boundary updates and the termination condition are forced — there is nothing left to guess. This section establishes the template from first principles so that every later variant can be derived the same way.

## Problem

You are given a sorted array $A[0..n-1]$ (in non-decreasing order) and a target value $t$. Find the index of $t$ in $A$, or return $-1$ if it is absent.

## Solution

Maintain two indices, $lo$ and $hi$, with the **invariant**:

$$
A[lo - 1] < t \le A[hi]
$$

where we treat $A[-1] = -\infty$ and $A[n] = +\infty$ as sentinels. The invariant says: everything strictly left of $lo$ is less than $t$; everything from $hi$ onward is at least $t$. The target, if present, must lie in the interval $[lo, hi)$.

Initialise $lo = 0$ and $hi = n$. Both boundary conditions hold: nothing is left of index 0, so $A[-1] < t$ is vacuously true; and $A[n] = +\infty \ge t$ by convention.

At each step, compute $mid = lo + (hi - lo) / 2$ (integer division). The expression $lo + (hi - lo) / 2$ is used instead of $(lo + hi) / 2$ to avoid overflow when indices are large. Then:

```text
if A[mid] < t:
    lo = mid + 1      // everything up to mid is < t; lo can advance
else:
    hi = mid          // A[mid] >= t; hi can retreat
```

Both updates preserve the invariant. When $lo = hi$, the interval is empty and the loop terminates. At that point, $lo$ is the smallest index where $A[lo] \ge t$. If $lo < n$ and $A[lo] = t$, the target is present; otherwise it is absent.

```text
binary_search(A, n, t):
    lo = 0
    hi = n
    while lo < hi:
        mid = lo + (hi - lo) / 2
        if A[mid] < t:
            lo = mid + 1
        else:
            hi = mid
    if lo < n and A[lo] == t:
        return lo
    return -1
```

This is the **half-open interval** form: $lo$ is inclusive, $hi$ is exclusive. The loop terminates because $hi - lo$ strictly decreases each iteration: when $A[mid] < t$, $lo$ increases by at least 1; when $A[mid] \ge t$, $hi$ decreases by at least 1 (since $mid < hi$ whenever $lo < hi$).

The **closed interval** form initialises $hi = n - 1$ and loops while $lo \le hi$, updating $hi = mid - 1$ in the else branch. Both are correct; the half-open form generalises more cleanly to lower-bound and upper-bound variants because the termination point $lo = hi$ is directly an insertion position.

## Analysis

The loop halves the interval at each step. Starting from an interval of size $n$, it takes at most $\lceil \log_2 n \rceil$ iterations to reduce the interval to size 0, giving time complexity $O(\log n)$. Space is $O(1)$.

The only way to write an incorrect binary search is to break the invariant. The most common mistake is updating $lo = mid$ instead of $lo = mid + 1$: this leaves $mid$ in the interval when $A[mid] < t$, and when $hi = lo + 1$ the computation $mid = lo + (hi - lo)/2 = lo$ produces the same $mid$ forever, causing an infinite loop. The invariant makes this mistake impossible: if $A[mid] < t$, then $mid$ cannot satisfy the invariant's left side, so $lo$ must move strictly past it.

## Takeaway

Binary search is an invariant-maintenance algorithm, not a pattern. Fix the invariant — $A[lo-1] < t \le A[hi]$ — and the update rules and termination condition follow directly. Every variant (lower bound, upper bound, answer bisection) is obtained by changing the comparison predicate while keeping the invariant structure intact.
