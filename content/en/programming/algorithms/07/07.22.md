# 7.22 Tree Rotations

Tree rotations are local transformations used to repair balance in binary search trees. A rotation changes parent-child relationships while preserving the in-order order of keys. This makes rotations the basic operation behind AVL trees, red-black trees, treaps, and many augmented ordered structures.

## Problem

Given a binary search tree that has become locally unbalanced, transform it so that the height improves while preserving the binary search tree invariant.

The transformation must preserve this order:

```text id="pd846x"
left subtree keys < node key < right subtree keys
```

## Right Rotation

A right rotation is used when a node is too heavy on the left.

Before rotation:

```text id="6xr95q"
      y
     / \\n    x   C
   / \\n  A   B
```

After right rotation:

```text id="jh6ltn"
      x
     / \\n    A   y
       / \\n      B   C
```

The keys are ordered as:

```text id="pr9i92"
A < x < B < y < C
```

That order is unchanged by the rotation.

```python id="qizx0p"
def rotate_right(y):
    x = y.left
    B = x.right

    x.right = y
    y.left = B

    return x
```

## Left Rotation

A left rotation is symmetric. It is used when a node is too heavy on the right.

Before rotation:

```text id="6jxf1w"
    x
   / \\n  A   y
     / \\n    B   C
```

After left rotation:

```text id="mj7x82"
      y
     / \\n    x   C
   / \\n  A   B
```

The key order remains:

```text id="2muqe4"
A < x < B < y < C
```

```python id="3g5s2x"
def rotate_left(x):
    y = x.right
    B = y.left

    y.left = x
    x.right = B

    return y
```

## Updating Metadata

Balanced and augmented trees usually store metadata such as height, size, sum, priority, or color. Rotations must update this metadata after links change.

For an AVL tree with heights:

```python id="hyiqjv"
def height(node):
    return node.height if node else 0

def update_height(node):
    node.height = 1 + max(height(node.left), height(node.right))
```

Right rotation with height updates:

```python id="qr3g8j"
def rotate_right(y):
    x = y.left
    B = x.right

    x.right = y
    y.left = B

    update_height(y)
    update_height(x)

    return x
```

The lower node `y` is updated first because its children changed first. Then `x` is updated because it now contains `y` as a child.

Left rotation with height updates:

```python id="d9p0x9"
def rotate_left(x):
    y = x.right
    B = y.left

    y.left = x
    x.right = B

    update_height(x)
    update_height(y)

    return y
```

## Double Rotations

A single rotation fixes outside-heavy imbalance. An inside-heavy imbalance needs two rotations.

### Left-Right Case

Before:

```text id="h5v18j"
      z
     /
    x
     \\n      y
```

Fix by rotating left at `x`, then right at `z`.

```python id="uwoczt"
z.left = rotate_left(z.left)
return rotate_right(z)
```

### Right-Left Case

Before:

```text id="vixugx"
    z
     \\n      x
     /
    y
```

Fix by rotating right at `x`, then left at `z`.

```python id="u86akq"
z.right = rotate_right(z.right)
return rotate_left(z)
```

## Why Rotations Preserve Order

Consider right rotation:

```text id="eeunhz"
      y
     / \\n    x   C
   / \\n  A   B
```

The BST invariant gives:

```text id="q6g7ug"
all A < x
x < all B < y
y < all C
```

After rotation:

```text id="554u4k"
      x
     / \\n    A   y
       / \\n      B   C
```

The left subtree of `x` remains `A`, so those keys are still smaller than `x`. The right subtree of `x` becomes `y`, whose left subtree is `B`. Since all keys in `B` lie between `x` and `y`, they are valid as the left subtree of `y` and as part of the right subtree of `x`. Thus the BST invariant is preserved.

The left rotation proof is symmetric.

## Rotations in AVL Insertion

After ordinary BST insertion, AVL trees restore balance using rotations.

```python id="xc84yx"
def balance_factor(node):
    return height(node.left) - height(node.right)

def rebalance(node):
    update_height(node)
    bf = balance_factor(node)

    if bf > 1:
        if balance_factor(node.left) < 0:
            node.left = rotate_left(node.left)
        return rotate_right(node)

    if bf < -1:
        if balance_factor(node.right) > 0:
            node.right = rotate_right(node.right)
        return rotate_left(node)

    return node
```

This code chooses between single and double rotations based on the direction of imbalance.

## Rotations with Parent Pointers

If nodes store parent pointers, rotations must update them too.

```python id="jhnshb"
def rotate_left_with_parent(root, x):
    y = x.right
    B = y.left

    x.right = B
    if B:
        B.parent = x

    y.left = x
    y.parent = x.parent

    if x.parent is None:
        root = y
    elif x == x.parent.left:
        x.parent.left = y
    else:
        x.parent.right = y

    x.parent = y

    return root
```

Parent pointers make rotations more error-prone. Always update child links and parent links together.

## Rotations with Subtree Sizes

For order statistics trees, update sizes after rotation.

```python id="38pg2s"
def size(node):
    return node.size if node else 0

def update_size(node):
    node.size = 1 + size(node.left) + size(node.right)
```

Right rotation:

```python id="35h4sg"
def rotate_right(y):
    x = y.left
    B = x.right

    x.right = y
    y.left = B

    update_size(y)
    update_size(x)

    return x
```

The update order is the same as with heights: update the demoted node first, then the promoted node.

## Complexity

A rotation changes a constant number of pointers.

```text id="g8taln"
Time:  O(1)
Space: O(1)
```

Metadata updates remain constant time if the metadata is computed from immediate children.

## Common Pitfalls

Do not forget to return the new subtree root. After a rotation, the old root is no longer the root of that subtree.

Do not update metadata before changing pointers. Metadata must reflect the final structure.

Do not forget parent pointers if the tree stores them.

Do not use rotations to repair arbitrary ordering errors. Rotations preserve order; they do not fix a tree whose BST invariant is already broken.

Do not forget double rotations. A single rotation fixes outside-heavy cases, but inside-heavy cases need two steps.


## Takeaway

Rotations are constant-time local rewrites that preserve sorted order while changing tree shape. They are the mechanism that lets balanced trees keep logarithmic height after updates. The operation is simple, but every pointer and every piece of metadata must be updated in the correct order.
