# 7.9 Ordered Maps

Ordered maps store key value pairs while maintaining keys in sorted order. They support logarithmic time insertion, deletion, and search, along with order-based queries such as predecessor, successor, and range lookup. Unlike hash maps, they preserve structure that binary search can exploit.

## Problem

Design a data structure that supports:

- insert a key value pair  
- delete a key  
- find a key  
- find the smallest key greater than or equal to `x`  
- find the largest key less than or equal to `x`  
- iterate keys in sorted order  

All operations should run in logarithmic time.

## Structure

Ordered maps are typically implemented using balanced binary search trees. Each node satisfies:

```text id="c7f7wl"
all keys in left subtree < node.key < all keys in right subtree
```

To maintain logarithmic height, the tree must rebalance itself after updates. Common designs include AVL trees and red black trees.

## Basic Interface

A minimal ordered map interface looks like this:

```python id="t7z64z"
class OrderedMap:
    def insert(self, key, value): ...
    def delete(self, key): ...
    def find(self, key): ...
    def lower_bound(self, key): ...
    def upper_bound(self, key): ...
```

The `lower_bound` operation returns the smallest key not less than the input. The `upper_bound` operation returns the smallest key strictly greater than the input.

## Lower Bound in a Tree

Binary search in arrays becomes tree traversal in ordered maps.

```python id="j3y9c0"
def lower_bound(root, x):
    result = None

    while root:
        if root.key >= x:
            result = root
            root = root.left
        else:
            root = root.right

    return result
```

At each step, move left or right depending on the comparison. The variable `result` stores the best candidate seen so far.

## Upper Bound in a Tree

```python id="p1mljg"
def upper_bound(root, x):
    result = None

    while root:
        if root.key > x:
            result = root
            root = root.left
        else:
            root = root.right

    return result
```

These operations generalize array-based binary search to dynamic data.

## Predecessor and Successor

The predecessor of a key is the largest key strictly less than it. The successor is the smallest key strictly greater than it.

```python id="v0z9ph"
def predecessor(root, x):
    result = None

    while root:
        if root.key < x:
            result = root
            root = root.right
        else:
            root = root.left

    return result
```

```python id="6v6m9o"
def successor(root, x):
    result = None

    while root:
        if root.key > x:
            result = root
            root = root.left
        else:
            root = root.right

    return result
```

These operations rely on the same invariant as binary search: discard half of the remaining structure at each step.

## Balanced Trees

A naive binary search tree can become skewed, leading to linear time operations. Balanced trees enforce height constraints.

### AVL Trees

AVL trees maintain the invariant:

```text id="mtfb1v"
height(left subtree) - height(right subtree) ∈ {-1, 0, 1}
```

After insertion or deletion, rotations restore balance.

### Red Black Trees

Red black trees maintain color constraints:

```text id="s2v8dy"
no two red nodes are adjacent
every path from root to leaf has the same number of black nodes
```

These constraints guarantee logarithmic height.

## Range Queries

Ordered maps support efficient range queries using lower and upper bounds.

```python id="n7h96y"
def range_query(root, L, R):
    node = lower_bound(root, L)
    result = []

    while node and node.key <= R:
        result.append((node.key, node.value))
        node = successor(root, node.key)

    return result
```

This retrieves all keys in `[L, R]` in sorted order.

## Augmentation

Trees can store extra information to support advanced queries.

Example: subtree size for order statistics.

```text id="k3i7v3"
node.size = 1 + size(left) + size(right)
```

This allows operations such as:

- find k-th smallest element  
- count elements less than x  

Example: range sums.

```text id="txh44h"
node.sum = node.value + sum(left) + sum(right)
```

This supports efficient prefix and interval sum queries.

## Complexity

For balanced trees:

```text id="k7o4p0"
search:       O(log n)
insert:       O(log n)
delete:       O(log n)
lower_bound:  O(log n)
upper_bound:  O(log n)
```

Space:

```text id="lhtb0g"
O(n)
```

## Comparison with Arrays

| Structure     | Search        | Insert       | Order Queries |
|--------------|--------------|--------------|---------------|
| Sorted array | O(log n)     | O(n)         | O(log n)      |
| Ordered map  | O(log n)     | O(log n)     | O(log n)      |

Arrays provide better cache locality and simpler implementation. Ordered maps provide efficient updates.

## Common Pitfalls

Forgetting to rebalance after insertion or deletion leads to worst-case linear time.

Using recursion without tail control can cause stack overflows for deep trees.

Misunderstanding lower bound semantics leads to off-by-one errors in range queries.

Failing to update augmented data during rotations breaks correctness.

Mixing inclusive and exclusive bounds in range queries produces incorrect results.

## Testing Checklist

Insert and search:

```python id="h9v1xu"
m = OrderedMap()
m.insert(5, "a")
m.insert(2, "b")
m.insert(8, "c")

assert m.find(5) == "a"
assert m.find(3) is None
```

Bounds:

```python id="4h9q8u"
assert m.lower_bound(4).key == 5
assert m.upper_bound(5).key == 8
```

Deletion:

```python id="d2rcx6"
m.delete(5)
assert m.find(5) is None
```

Range query:

```python id="5m8trr"
# assuming range_query implemented
assert range_query(m.root, 2, 8) == [(2, "b"), (8, "c")]
```

## Takeaway

Ordered maps generalize binary search from static arrays to dynamic sets. The same boundary logic applies, but traversal replaces indexing. Balanced trees preserve logarithmic guarantees, and augmentation extends the structure to support richer queries.
