# 7.21 Binary Search Trees

A binary search tree stores keys in a recursive ordered structure. Each node has a key, an optional left child, and an optional right child. The left subtree contains smaller keys. The right subtree contains larger keys.

The basic invariant is:

```text
for every node:
  all keys in node.left  < node.key
  all keys in node.right > node.key
```

This invariant makes search simple. At each node, compare the target with the current key. If the target is smaller, go left. If it is larger, go right. If it is equal, the key has been found.

## Problem

Implement a binary search tree that supports:

1. search
2. insertion
3. deletion
4. minimum key
5. maximum key
6. in-order traversal

The tree does not rebalance itself. Therefore operations are `O(h)`, where `h` is the tree height.

## Node Structure

```python
class Node:
    def __init__(self, key: int):
        self.key = key
        self.left: Node | None = None
        self.right: Node | None = None
```

## Search

```python
def search(root: Node | None, key: int) -> Node | None:
    while root:
        if key == root.key:
            return root
        elif key < root.key:
            root = root.left
        else:
            root = root.right

    return None
```

The loop follows one path from root to leaf.

## Insert

```python
def insert(root: Node | None, key: int) -> Node:
    if root is None:
        return Node(key)

    if key < root.key:
        root.left = insert(root.left, key)
    elif key > root.key:
        root.right = insert(root.right, key)

    return root
```

This version ignores duplicate keys. If duplicates are required, choose a policy: store a count in each node, store equal keys on one side, or store a list of values per key.

## Minimum and Maximum

The minimum key is found by following left pointers.

```python
def minimum(root: Node) -> Node:
    while root.left:
        root = root.left
    return root
```

The maximum key is found by following right pointers.

```python
def maximum(root: Node) -> Node:
    while root.right:
        root = root.right
    return root
```

## In-Order Traversal

In-order traversal visits keys in sorted order.

```python
def inorder(root: Node | None) -> list[int]:
    if root is None:
        return []

    return inorder(root.left) + [root.key] + inorder(root.right)
```

For large trees, use a generator to avoid repeated list allocation.

```python
def inorder_iter(root: Node | None):
    stack = []
    cur = root

    while stack or cur:
        while cur:
            stack.append(cur)
            cur = cur.left

        cur = stack.pop()
        yield cur.key
        cur = cur.right
```

## Delete

Deletion has three cases.

If the node has no child, remove it.

If the node has one child, replace it with that child.

If the node has two children, replace its key with its in-order successor, then delete the successor from the right subtree.

```python
def delete(root: Node | None, key: int) -> Node | None:
    if root is None:
        return None

    if key < root.key:
        root.left = delete(root.left, key)
    elif key > root.key:
        root.right = delete(root.right, key)
    else:
        if root.left is None:
            return root.right

        if root.right is None:
            return root.left

        succ = minimum(root.right)
        root.key = succ.key
        root.right = delete(root.right, succ.key)

    return root
```

## Correctness

Search is correct because the BST invariant tells us which subtree can contain the key. If `key < root.key`, the right subtree cannot contain `key`. If `key > root.key`, the left subtree cannot contain `key`.

Insertion preserves the invariant because the new key is inserted only into the subtree where it belongs.

Deletion preserves the invariant in all cases. Removing a leaf changes no ordering relation. Replacing a node by its only child is safe because that child subtree already lies in the correct range. Replacing a node with its successor is safe because the successor is the smallest key in the right subtree, so it is larger than every key in the left subtree and no larger than the remaining keys in the right subtree.

## Complexity

Let `h` be the height of the tree.

```text
search:    O(h)
insert:    O(h)
delete:    O(h)
minimum:   O(h)
maximum:   O(h)
traversal: O(n)
```

If the tree is balanced, `h = O(log n)`. If the tree is skewed, `h = O(n)`.

A skewed tree can arise from sorted insertion order:

```text
1, 2, 3, 4, 5
```

This produces a chain rather than a branching tree.

## Common Pitfalls

Do not assume a plain binary search tree is automatically logarithmic. The height depends on insertion order.

Do not delete a two-child node by simply attaching one subtree to the other without preserving order.

Do not ignore duplicates without stating the policy.

Do not use recursive traversal on extremely deep unbalanced trees in languages with small call stacks.

Do not forget that in-order traversal depends on the BST invariant. If insertion or deletion breaks the invariant, traversal may still return values, but they will not be sorted.

## Testing Checklist

```python
root = None
for x in [5, 2, 8, 1, 3, 7, 9]:
    root = insert(root, x)

assert list(inorder_iter(root)) == [1, 2, 3, 5, 7, 8, 9]
assert search(root, 7).key == 7
assert search(root, 4) is None

root = delete(root, 2)
assert list(inorder_iter(root)) == [1, 3, 5, 7, 8, 9]

root = delete(root, 5)
assert list(inorder_iter(root)) == [1, 3, 7, 8, 9]
```

## Takeaway

A binary search tree is ordered recursion over keys. Its correctness comes from the subtree ordering invariant. Its performance depends on height. Plain BSTs are useful for teaching and simple cases, but production ordered maps usually require balancing.
