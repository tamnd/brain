# 7.17 Implementation Patterns

Binary search and ordered structures benefit from a small set of stable patterns. Reusing these patterns reduces boundary errors and keeps implementations consistent across problems.

This section consolidates the canonical templates and shows how to adapt them safely.

## Pattern 1. First True (Half-Open)

Use this when the predicate has shape:

```text id="f1yk8a"
False False False True True True
```

```python id="3h1n6k"
def first_true(lo: int, hi: int, pred) -> int:
    """
    Returns smallest x in [lo, hi) such that pred(x) is true.
    Requires: pred(hi) is true if hi is used as sentinel.
    """
    while lo < hi:
        mid = lo + (hi - lo) // 2

        if pred(mid):
            hi = mid
        else:
            lo = mid + 1

    return lo
```

Typical uses:

- lower_bound  
- minimum feasible value  
- boundary detection  

## Pattern 2. Last True (Closed Interval)

Use this when the predicate has shape:

```text id="o4e4ml"
True True True False False False
```

```python id="yfl2bh"
def last_true(lo: int, hi: int, pred) -> int:
    """
    Returns largest x in [lo, hi] such that pred(x) is true.
    Requires: pred(lo) is true.
    """
    while lo < hi:
        mid = lo + (hi - lo + 1) // 2

        if pred(mid):
            lo = mid
        else:
            hi = mid - 1

    return lo
```

The midpoint is biased upward to guarantee progress.

## Pattern 3. Lower Bound Wrapper

```python id="d2n4pt"
def lower_bound(a: list[int], x: int) -> int:
    return first_true(0, len(a), lambda i: a[i] >= x)
```

## Pattern 4. Upper Bound Wrapper

```python id="w0c1z9"
def upper_bound(a: list[int], x: int) -> int:
    return first_true(0, len(a), lambda i: a[i] > x)
```

## Pattern 5. Exact Search via Lower Bound

```python id="cnhm8h"
def binary_search(a: list[int], x: int) -> int:
    i = lower_bound(a, x)
    if i < len(a) and a[i] == x:
        return i
    return -1
```

This avoids duplicate-handling bugs.

## Pattern 8. Floating-Point Search

```python id="d9a0hm"
def binary_search_real(lo: float, hi: float, pred, steps: int = 80) -> float:
    for _ in range(steps):
        mid = lo + (hi - lo) / 2.0

        if pred(mid):
            hi = mid
        else:
            lo = mid

    return hi
```

Use fixed iterations to avoid termination issues.

## Pattern 9. Doubling to Find Upper Bound

```python id="q1u3v4"
def find_bound(pred):
    hi = 1

    while not pred(hi):
        hi *= 2

    return hi
```

Then apply binary search on `[0, hi]`.

## Pattern 10. Tree Lower Bound

```python id="8v9w2c"
def tree_lower_bound(root, x):
    result = None

    while root:
        if root.key >= x:
            result = root
            root = root.left
        else:
            root = root.right

    return result
```

## Pattern 11. Coordinate Compression

```python id="y7q2l0"
def compress(values):
    unique = sorted(set(values))
    index = {v: i for i, v in enumerate(unique)}
    return [index[v] for v in values], index, unique
```

## Pattern 12. Rank via Tree Traversal

```python id="t9v7lb"
def rank(node, key):
    result = 0

    while node:
        if key <= node.key:
            node = node.left
        else:
            left_size = node.left.size if node.left else 0
            result += left_size + 1
            node = node.right

    return result
```

## Pattern Selection Guide

| Goal                         | Pattern           |
|------------------------------|-------------------|
| first position satisfying    | first_true        |
| last position satisfying     | last_true         |
| exact match                  | lower_bound + check |
| minimum feasible             | first_true        |
| maximum feasible             | last_true         |
| real-valued optimization     | floating search   |
| unknown upper bound          | doubling + search |
| dynamic ordered set          | tree traversal    |

## Anti-Patterns

Avoid writing ad-hoc binary search loops:

```python id="2n0v0z"
# unclear invariant
while lo <= hi:
    mid = ...
    if condition:
        ...
```

Without a clear invariant, correctness becomes fragile.

Avoid mixing interval conventions. Choose either:

```text id="6dz7h4"
[lo, hi)
```

or:

```text id="v6i8pl"
[lo, hi]
```

and apply it consistently.

## Refactoring Strategy

When debugging:

1. rewrite using a standard template  
2. express the predicate explicitly  
3. check invariant  
4. test edge cases  

Most bugs disappear after switching to a known pattern.

## Testing Pattern

For each pattern, verify:

```python id="c0r3t5"
assert pred(result) is True
assert result == lo or pred(result - 1) is False
```

or the dual for last-true.

## Takeaway

Binary search is not a single algorithm but a family of patterns. Each pattern corresponds to a specific predicate shape and interval convention. Reusing these templates eliminates most boundary errors and makes implementations predictable.
