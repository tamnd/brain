# 7.24 Range Query Structures

Range query structures answer questions about intervals of an array. Instead of searching for a single value, we ask for an aggregate over a range, such as a sum, minimum, maximum, or count.

The common pattern is to preprocess data so that each query avoids scanning the full interval.

## Problem

Given an array `a`, support queries such as:

```
```
sum of a[L..R]  
minimum of a[L..R]  
maximum of a[L..R]  
number of updates in a[L..R]
```
```

The right data structure depends on whether the array is static or dynamic.

## Prefix Sums

For static range sum queries, prefix sums are simplest.

```
Python

```
def build_prefix(a: list[int]) -> list[int]:  
    prefix = [0] * (len(a) + 1)  
  
    for i, x in enumerate(a):  
        prefix[i + 1] = prefix[i] + x  
  
    return prefix
```
```

Query sum over inclusive range `[L, R]`:

```
Python

```
def range_sum(prefix: list[int], L: int, R: int) -> int:  
    return prefix[R + 1] - prefix[L]
```
```

The prefix array uses a half-open convention:

```
```
prefix[i] = sum of a[0..i)
```
```

This makes the query formula uniform.

## Fenwick Tree

A Fenwick tree supports point updates and prefix sum queries in logarithmic time.

```
Python

```
class Fenwick:  
    def __init__(self, n: int):  
        self.bit = [0] * (n + 1)  
  
    def add(self, i: int, delta: int) -> None:  
        i += 1  
        while i < len(self.bit):  
            self.bit[i] += delta  
            i += i & -i  
  
    def prefix_sum(self, i: int) -> int:  
        total = 0  
        i += 1  
        while i > 0:  
            total += self.bit[i]  
            i -= i & -i  
        return total  
  
    def range_sum(self, L: int, R: int) -> int:  
        if R < L:  
            return 0  
        return self.prefix_sum(R) - (self.prefix_sum(L - 1) if L > 0 else 0)
```
```

Fenwick trees are compact and fast for invertible prefix aggregates such as sum.

## Segment Tree

A segment tree supports more general range queries. It can handle sums, minimums, maximums, greatest common divisors, and many other associative operations.

```
Python

```
class SegmentTree:  
    def __init__(self, a: list[int]):  
        self.n = len(a)  
        self.tree = [0] * (2 * self.n)  
  
        for i in range(self.n):  
            self.tree[self.n + i] = a[i]  
  
        for i in range(self.n - 1, 0, -1):  
            self.tree[i] = self.tree[2 * i] + self.tree[2 * i + 1]  
  
    def update(self, i: int, value: int) -> None:  
        i += self.n  
        self.tree[i] = value  
  
        i //= 2  
        while i >= 1:  
            self.tree[i] = self.tree[2 * i] + self.tree[2 * i + 1]  
            i //= 2  
  
    def query(self, L: int, R: int) -> int:  
        # query half-open interval [L, R)  
        L += self.n  
        R += self.n  
  
        result = 0  
  
        while L < R:  
            if L % 2 == 1:  
                result += self.tree[L]  
                L += 1  
  
            if R % 2 == 1:  
                R -= 1  
                result += self.tree[R]  
  
            L //= 2  
            R //= 2  
  
        return result
```
```

This version answers range sum queries over half-open intervals `[L, R)`.

## Static Range Minimum Query

If the array never changes and only range minimum queries are needed, a sparse table gives `O(1)` queries after preprocessing.

```
Python

```
import math  
  
class SparseTableMin:  
    def __init__(self, a: list[int]):  
        self.n = len(a)  
        self.log = [0] * (self.n + 1)  
  
        for i in range(2, self.n + 1):  
            self.log[i] = self.log[i // 2] + 1  
  
        k = self.log[self.n] + 1  
        self.table = [a[:]]  
  
        j = 1  
        while (1 << j) <= self.n:  
            prev = self.table[j - 1]  
            length = 1 << j  
            half = length >> 1  
            row = []  
  
            for i in range(self.n - length + 1):  
                row.append(min(prev[i], prev[i + half]))  
  
            self.table.append(row)  
            j += 1  
  
    def query(self, L: int, R: int) -> int:  
        # inclusive range [L, R]  
        length = R - L + 1  
        j = self.log[length]  
        return min(self.table[j][L], self.table[j][R - (1 << j) + 1])
```
```

This works because minimum is idempotent:

```
```
min(x, x) = x
```
```

The query combines two overlapping blocks that cover the requested range.

## Choosing the Structure

| Structure | Updates | Query | Best for |
| --- | --- | --- | --- |
| Prefix sum | none | O(1) | static sums |
| Fenwick tree | O(log n) | O(log n) | dynamic sums |
| Segment tree | O(log n) | O(log n) | dynamic associative queries |
| Sparse table | none | O(1) | static min/max/gcd |

## Complexity

Prefix sums:

```
```
build: O(n)  
query: O(1)  
space: O(n)
```
```

Fenwick tree:

```
```
build:  O(n log n) or O(n)  
update: O(log n)  
query:  O(log n)  
space:  O(n)
```
```

Segment tree:

```
```
build:  O(n)  
update: O(log n)  
query:  O(log n)  
space:  O(n)
```
```

Sparse table:

```
```
build: O(n log n)  
query: O(1)  
space: O(n log n)
```
```

## Common Pitfalls

Do not mix inclusive and half-open intervals. Prefix sums and iterative segment trees are usually cleaner with half-open ranges.

Do not use a sparse table for updates. It is designed for static data.

Do not use a Fenwick tree for non-invertible range queries unless the operation fits the Fenwick model.

Do not forget the identity element. Segment trees need the correct neutral value, such as `0` for sum and positive infinity for minimum.

Do not ignore coordinate compression when array indices come from large sparse values.

```

```
fw = Fenwick(4)


## Takeaway

Range query structures extend ordered thinking from single positions to intervals. Prefix sums are best for static sums, Fenwick trees for dynamic prefix-based aggregates, segment trees for general dynamic associative operations, and sparse tables for static idempotent queries.
