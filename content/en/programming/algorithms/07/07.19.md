# 7.19 Case Studies

This section applies the patterns from the chapter to complete problems. Each case begins with the ordered structure hidden inside the problem, then states the predicate, proves monotonicity, and implements the search.

## Case Study 1. Minimum Eating Speed

Suppose there are piles of bananas, and a worker eats at speed `k` bananas per hour. In one hour, the worker chooses one pile and eats up to `k` bananas from it. Given `h` hours, find the minimum speed needed to finish all piles.

## Problem

```python
piles = [3, 6, 7, 11]
h = 8
```

The answer is `4`.

At speed `4`, the hours needed are:

```text
ceil(3 / 4) + ceil(6 / 4) + ceil(7 / 4) + ceil(11 / 4)
= 1 + 2 + 2 + 3
= 8
```

## Predicate

Define:

```text
P(k) = all piles can be eaten within h hours at speed k
```

If speed `k` works, then any larger speed also works. The predicate has false-to-true shape, so we search for the first true speed.

## Implementation

```python
def min_eating_speed(piles: list[int], h: int) -> int:
    def hours_needed(k: int) -> int:
        total = 0
        for p in piles:
            total += (p + k - 1) // k
        return total

    lo = 1
    hi = max(piles)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if hours_needed(mid) <= h:
            hi = mid
        else:
            lo = mid + 1

    return lo
```

## Correctness

The lower bound is `1`, the slowest positive speed. The upper bound is `max(piles)`, because at that speed each pile takes at most one hour. If a speed works, every larger speed works because each pile takes no more hours than before. Therefore the feasible speeds form one suffix of the search interval. The binary search returns the first feasible speed.

## Complexity

Each predicate check scans all piles.

```text
Time:  O(n log M)
Space: O(1)
```

where `M = max(piles)`.

## Case Study 2. Smallest Divisor Under a Threshold

Given an array `nums` and a threshold, choose the smallest positive divisor `d` such that:

```text
sum(ceil(nums[i] / d)) <= threshold
```

## Predicate

```text
P(d) = divisor d keeps the rounded sum at or below threshold
```

As `d` increases, each term `ceil(nums[i] / d)` decreases or stays the same. Therefore `P` is monotone false-to-true.

## Implementation

```python
def smallest_divisor(nums: list[int], threshold: int) -> int:
    def ok(d: int) -> bool:
        total = 0
        for x in nums:
            total += (x + d - 1) // d
        return total <= threshold

    lo = 1
    hi = max(nums)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if ok(mid):
            hi = mid
        else:
            lo = mid + 1

    return lo
```

## Case Study 3. Minimum Days to Make Bouquets

Given an array `bloomDay`, an integer `m`, and an integer `k`, find the minimum day when it is possible to make `m` bouquets, each using `k` adjacent bloomed flowers.

## Predicate

```text
P(day) = at least m bouquets can be made by this day
```

If bouquets can be made on day `d`, they can also be made on any later day. The predicate is false-to-true.

## Implementation

```python
def min_days_bouquets(bloomDay: list[int], m: int, k: int) -> int:
    if m * k > len(bloomDay):
        return -1

    def can_make(day: int) -> bool:
        bouquets = 0
        run = 0

        for b in bloomDay:
            if b <= day:
                run += 1
                if run == k:
                    bouquets += 1
                    run = 0
            else:
                run = 0

        return bouquets >= m

    lo = min(bloomDay)
    hi = max(bloomDay)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if can_make(mid):
            hi = mid
        else:
            lo = mid + 1

    return lo
```

## Correctness

The predicate counts how many disjoint adjacent groups of length `k` can be formed by a given day. Later days can only turn more flowers from unavailable to available, never the reverse. Therefore feasibility is monotone. Binary search returns the earliest feasible day.

## Case Study 4. Split Array Largest Sum

Given an array `nums` and an integer `k`, split the array into at most `k` nonempty contiguous parts. Minimize the largest part sum.

## Predicate

```text
P(limit) = nums can be split into at most k parts, each with sum at most limit
```

If a limit works, any larger limit also works. The predicate is false-to-true.

## Implementation

```python
def split_array_largest_sum(nums: list[int], k: int) -> int:
    def can_split(limit: int) -> bool:
        parts = 1
        current = 0

        for x in nums:
            if current + x <= limit:
                current += x
            else:
                parts += 1
                current = x

        return parts <= k

    lo = max(nums)
    hi = sum(nums)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if can_split(mid):
            hi = mid
        else:
            lo = mid + 1

    return lo
```

## Greedy Check

The predicate uses a greedy scan. It keeps adding numbers to the current part until adding another number would exceed `limit`, then starts a new part. This produces the fewest possible parts for that limit because delaying a split never hurts when all numbers are nonnegative.

## Case Study 5. K-th Missing Positive Number

Given a strictly increasing positive integer array `a`, find the `k`-th missing positive integer.

For index `i`, the number of missing positive integers before or at `a[i]` is:

```text
missing(i) = a[i] - (i + 1)
```

This value is monotone because the array is strictly increasing.

## Predicate

```text
P(i) = missing(i) >= k
```

We search for the first index where at least `k` numbers are missing.

## Implementation

```python
def kth_missing_positive(a: list[int], k: int) -> int:
    lo = 0
    hi = len(a)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        missing = a[mid] - (mid + 1)

        if missing >= k:
            hi = mid
        else:
            lo = mid + 1

    return lo + k
```

## Why `lo + k` Works

At termination, `lo` is the number of array elements before the answer. Among the positive integers up to the answer, exactly `lo` values are present in the array and `k` values are missing. Therefore:

```text
answer = lo + k
```

## Case Study 6. Median of Two Sorted Arrays

Given two sorted arrays, find the median of their union in logarithmic time.

This problem uses binary search over a partition. We choose how many elements to take from the first array, and the rest are taken from the second array.

## Implementation

```python
def median_two_sorted_arrays(a: list[int], b: list[int]) -> float:
    if len(a) > len(b):
        a, b = b, a

    m = len(a)
    n = len(b)
    total_left = (m + n + 1) // 2

    lo = 0
    hi = m

    while lo <= hi:
        i = lo + (hi - lo) // 2
        j = total_left - i

        a_left = a[i - 1] if i > 0 else float("-inf")
        a_right = a[i] if i < m else float("inf")
        b_left = b[j - 1] if j > 0 else float("-inf")
        b_right = b[j] if j < n else float("inf")

        if a_left <= b_right and b_left <= a_right:
            if (m + n) % 2 == 1:
                return max(a_left, b_left)
            return (max(a_left, b_left) + min(a_right, b_right)) / 2.0
        elif a_left > b_right:
            hi = i - 1
        else:
            lo = i + 1

    raise ValueError("input arrays must be sorted")
```

## Partition Condition

The correct partition satisfies:

```text
max(left side) <= min(right side)
```

The search moves left if too many elements were taken from `a`, and right if too few were taken from `a`.

## Complexity

The search is over the shorter array.

```text
Time:  O(log min(m, n))
Space: O(1)
```

## Case Study 7. First Bad Version

Given versions `1..n`, suppose there is a first bad version, and every version after it is also bad. Find the first bad version.

## Predicate

```text
P(v) = version v is bad
```

This is the purest form of first-true binary search.

```python
def first_bad_version(n: int, is_bad) -> int:
    lo = 1
    hi = n

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if is_bad(mid):
            hi = mid
        else:
            lo = mid + 1

    return lo
```

| Smallest divisor | first true | rounded sum within threshold |
| Bouquets | first true | enough bouquets by day |
| Split array | first true | split possible under limit |

## Takeaway

These case studies share the same structure. First state the candidate value, then define the feasibility predicate, then prove that the predicate changes direction only once. After that, the implementation is a standard boundary search.
