# 7.23 Heaps and Priority Queues

Heaps maintain a partial order that allows fast access to the smallest or largest element. Unlike binary search trees, heaps do not support full ordered traversal, but they provide efficient extraction and insertion of extremal elements.

A binary heap is the most common implementation. It is stored as an array and represents a complete binary tree.

## Structure

For a min-heap:

```
```
parent <= children
```
```

For a max-heap:

```
```
parent >= children
```
```

The tree is complete, so it can be stored in an array.

For index `i`:

```
```
left child  = 2*i + 1  
right child = 2*i + 2  
parent      = (i - 1) // 2
```
```

## Problem

Support operations:

- insert a value
- extract minimum (or maximum)
- peek at the minimum (or maximum)

All operations should be efficient.

## Push Operation

Insert a value at the end of the array, then restore the heap property by moving it upward.

```
Python

```
def push(heap: list[int], x: int) -> None:  
    heap.append(x)  
    i = len(heap) - 1  
  
    while i > 0:  
        p = (i - 1) // 2  
  
        if heap[p] <= heap[i]:  
            break  
  
        heap[p], heap[i] = heap[i], heap[p]  
        i = p
```
```

This process is called sift-up.

## Pop Operation

Remove the root, replace it with the last element, then restore the heap property by moving it downward.

```
Python

```
def pop(heap: list[int]) -> int:  
    if not heap:  
        raise IndexError("pop from empty heap")  
  
    root = heap[0]  
    last = heap.pop()  
  
    if heap:  
        heap[0] = last  
        sift_down(heap, 0)  
  
    return root
```
```

```
Python

```
def sift_down(heap: list[int], i: int) -> None:  
    n = len(heap)  
  
    while True:  
        left = 2 * i + 1  
        right = 2 * i + 2  
        smallest = i  
  
        if left < n and heap[left] < heap[smallest]:  
            smallest = left  
  
        if right < n and heap[right] < heap[smallest]:  
            smallest = right  
  
        if smallest == i:  
            break  
  
        heap[i], heap[smallest] = heap[smallest], heap[i]  
        i = smallest
```
```

This process is called sift-down.

## Peek

```
Python

```
def peek(heap: list[int]) -> int:  
    if not heap:  
        raise IndexError("peek from empty heap")  
    return heap[0]
```
```

## Build Heap

Convert an arbitrary array into a heap in linear time.

```
Python

```
def heapify(a: list[int]) -> None:  
    n = len(a)  
  
    for i in range(n // 2 - 1, -1, -1):  
        sift_down(a, i)
```
```

This works because leaves are already valid heaps, and internal nodes are fixed bottom-up.

## Complexity

```
```
push:    O(log n)  
pop:     O(log n)  
peek:    O(1)  
heapify: O(n)
```
```

The logarithmic cost comes from the height of the tree.

## Comparison with Binary Search

Heaps and binary search solve different problems.

| Operation | Binary Search | Heap |
| --- | --- | --- |
| find element | O(log n) | O(n) |
| min / max | O(1) if known | O(1) |
| insert | O(n) | O(log n) |
| delete min / max | O(n) | O(log n) |
| sorted iteration | O(n) | O(n log n) |

Heaps are optimized for repeated extraction of extremal elements.

## Applications

### Priority Queue

Process tasks in order of priority.

```
Python

```
heap = []  
push(heap, (priority, task))  
priority, task = pop(heap)
```
```

### Dijkstra Algorithm

Use a min-heap to extract the node with the smallest distance.

### Top K Elements

Find the k largest elements.

```
Python

```
import heapq  
  
def top_k(nums, k):  
    heap = []  
  
    for x in nums:  
        heapq.heappush(heap, x)  
        if len(heap) > k:  
            heapq.heappop(heap)  
  
    return heap
```
```

### Merge K Sorted Lists

Push the first element of each list, then repeatedly extract the smallest.

## Heaps vs Balanced Trees

| Feature | Heap | Balanced Tree |
| --- | --- | --- |
| min / max | O(1) | O(log n) |
| insert | O(log n) | O(log n) |
| delete arbitrary | O(n) | O(log n) |
| order queries | not supported | supported |

Heaps are specialized. Trees are general.

## Common Pitfalls

Do not assume a heap is sorted. Only the root is guaranteed to be the smallest or largest.

Do not modify heap elements without restoring heap property.

Do not use heap for binary search. The internal structure is not ordered globally.

Do not forget that heapify runs in linear time, not `O(n log n)`.

## Takeaway

Heaps maintain a partial order that supports efficient access to extremal elements. They complement binary search and trees by solving a different class of problems: repeated selection of minimum or maximum values.
