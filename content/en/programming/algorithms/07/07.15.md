# 7.15 Testing and Verification

Binary search and ordered data algorithms are short, but correctness depends on precise invariants. Testing must target boundaries, monotonicity assumptions, and termination behavior. Verification should connect the implementation to a stated invariant and postcondition.

## Problem

Given an implementation of binary search or an ordered operation, ensure that:

1. the invariant is preserved at every step  
2. the loop terminates  
3. the returned value satisfies the specification  
4. edge cases are handled correctly  

## Specification

For a first-true search over `[0, n)`, the specification is:

```text id="jzv4po"
result = smallest i such that pred(i) is true
if no such i exists, result = n
```

For lower bound:

```text id="s7q9t7"
result = smallest i such that a[i] >= x
```

For upper bound:

```text id="1l9h3w"
result = smallest i such that a[i] > x
```

## Invariant-Based Verification

Write the invariant explicitly.

For first-true:

```text id="m0t9pa"
all i < lo: pred(i) is false
all i >= hi: pred(i) is true
```

Initialization:

```text id="4n8y4m"
lo = 0
hi = n
```

Both parts hold vacuously.

Maintenance:

- if `pred(mid)` is true, set `hi = mid`
- if `pred(mid)` is false, set `lo = mid + 1`

Each update preserves the invariant and shrinks the interval.

Termination:

```text id="vq0r8j"
lo == hi
```

At that point, `lo` is the boundary.

## Property-Based Testing

Instead of checking a few fixed cases, generate many inputs and verify general properties.

### Property: Boundary Correctness

```python id="rw2d0p"
def check_first_true(n, pred):
    i = first_true(n, pred)

    assert 0 <= i <= n

    if i < n:
        assert pred(i) is True

    if i > 0:
        assert pred(i - 1) is False
```

### Property: Lower Bound

```python id="z3f4cn"
def check_lower_bound(a, x):
    i = lower_bound(a, x)

    assert 0 <= i <= len(a)

    if i < len(a):
        assert a[i] >= x

    if i > 0:
        assert a[i - 1] < x
```

### Property: Upper Bound

```python id="6h4c2o"
def check_upper_bound(a, x):
    i = upper_bound(a, x)

    assert 0 <= i <= len(a)

    if i < len(a):
        assert a[i] > x

    if i > 0:
        assert a[i - 1] <= x
```

These properties define correctness independent of implementation details.

## Exhaustive Testing on Small Inputs

For small arrays, test all possibilities.

```python id="6jzjpi"
from itertools import product

def brute_lower_bound(a, x):
    for i in range(len(a)):
        if a[i] >= x:
            return i
    return len(a)

def test_all_small():
    for n in range(6):
        for a in product(range(3), repeat=n):
            a = sorted(a)
            for x in range(5):
                assert lower_bound(list(a), x) == brute_lower_bound(list(a), x)
```

This catches edge cases that hand-written tests often miss.

## Monotonicity Testing

When using binary search on a predicate, test monotonicity directly.

```python id="l3r6wp"
def check_monotone(pred, lo, hi):
    seen_true = False

    for i in range(lo, hi + 1):
        if pred(i):
            seen_true = True
        else:
            assert not seen_true
```

This verifies that once `pred(i)` becomes true, it never becomes false again.

For last-true search, reverse the condition.

## Floating-Point Verification

For real-valued search, verify approximation quality.

```python id="0blz3l"
def check_sqrt(x):
    r = sqrt_binary(x)
    assert abs(r * r - x) <= 1e-9 * max(1.0, x)
```

Check both absolute and relative error when appropriate.

## Stress Testing

Generate large random inputs and compare against a slower reference implementation.

```python id="w6g6z8"
import random

def stress_test():
    for _ in range(10000):
        n = random.randint(0, 100)
        a = sorted(random.randint(0, 50) for _ in range(n))
        x = random.randint(0, 50)

        assert lower_bound(a, x) == brute_lower_bound(a, x)
```

Stress testing detects rare boundary errors.

## Runtime Assertions

Insert assertions into the implementation to verify invariants during execution.

```python id="4j2lpx"
def first_true_debug(n, pred):
    lo = 0
    hi = n

    while lo < hi:
        mid = lo + (hi - lo) // 2

        assert 0 <= lo <= mid < hi <= n

        if pred(mid):
            hi = mid
        else:
            lo = mid + 1

    return lo
```

These checks help catch incorrect updates early.

## Formal Verification Approach

To formally verify binary search:

1. define the predicate and its monotonicity  
2. define the invariant  
3. prove initialization  
4. prove maintenance  
5. prove termination  
6. prove postcondition  

This structure mirrors textbook correctness proofs.

## Common Pitfalls

Testing only typical cases and ignoring empty or one-element inputs.

Assuming monotonicity without checking it.

Testing only successful searches and not failure cases.

Using random tests without a reference implementation.

Ignoring floating-point precision issues.

## Testing Checklist

Basic cases:

```python id="v5qzv7"
assert lower_bound([], 5) == 0
assert lower_bound([5], 5) == 0
assert lower_bound([5], 6) == 1
```

Duplicates:

```python id="dwk9z6"
a = [1, 2, 2, 2, 3]
assert lower_bound(a, 2) == 1
assert upper_bound(a, 2) == 4
```

Monotone predicate:

```python id="fqdxlq"
check_monotone(lambda i: i >= 5, 0, 10)
```

Floating-point:

```python id="px0f6d"
check_sqrt(2.0)
check_sqrt(100.0)
```

Stress:

```python id="gk1wgh"
stress_test()
```

## Takeaway

Testing binary search requires more than checking outputs. It requires verifying invariants, monotonicity, and boundary behavior. Property-based testing and small exhaustive tests provide strong guarantees. Once the invariant is correct, the implementation becomes reliable.
