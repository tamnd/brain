# 7.18 Common Patterns and Reductions

Many problems that appear unrelated reduce to the same small set of ordered-search patterns. Recognizing these reductions is more valuable than memorizing specific solutions. This section catalogs common transformations that turn a problem into binary search or an ordered query.

## Pattern 1. Optimization to Feasibility

Convert:

```text
minimize f(x)
```

into:

```text
find smallest x such that P(x) is true
```

where `P(x)` checks whether a candidate `x` is feasible.

Examples:

- minimum capacity  
- minimum time  
- minimum budget  

The reduction replaces optimization with boundary detection.

## Pattern 2. Maximum Feasible Value

Convert:

```text
maximize f(x)
```

into:

```text
find largest x such that P(x) is true
```

Examples:

- maximum minimum distance  
- maximum throughput  
- maximum allowed constraint  

This uses last-true binary search.

## Pattern 3. Equality Search via Boundaries

Instead of searching directly for equality:

```text
find i such that a[i] == x
```

use:

```text
i = lower_bound(x)
check if a[i] == x
```

This avoids duplicate ambiguity and simplifies correctness.

## Pattern 4. Counting via Bounds

Count elements in `[L, R]`:

```text
count = upper_bound(R) - lower_bound(L)
```

This reduces counting to two boundary queries.

## Pattern 5. Range Extraction

Extract elements in `[L, R]`:

```text
l = lower_bound(L)
r = upper_bound(R)
result = a[l:r]
```

This is the same pattern as counting, but returns elements.

## Pattern 6. Prefix Transformation

Some problems become monotone only after transformation.

Example: maximum average subarray

Transform:

```text
b[i] = a[i] - x
```

Then check whether a subarray has nonnegative sum.

This converts a ratio problem into a sum problem.

## Pattern 7. Decision Tree Reduction

Binary search can be viewed as navigating a decision tree where each comparison halves the possibilities.

Example:

```text
find smallest i such that condition(i) holds
```

Each predicate evaluation discards half the remaining candidates.

## Pattern 8. Implicit Sorted Space

The search space may not be stored explicitly but still behaves as sorted.

Examples:

- time values  
- capacities  
- distances  
- answers to optimization problems  

Binary search operates on these implicit domains.

## Pattern 9. Binary Search on Graph Answer

Many graph problems reduce to checking feasibility with a threshold.

Example:

```text
minimum maximum edge weight such that graph is connected
```

Predicate:

```text
use only edges with weight ≤ x
check connectivity
```

This is monotone in `x`.

## Pattern 10. Scheduling Reduction

Scheduling problems often reduce to capacity or time feasibility.

Example:

```text
minimum time to finish jobs
```

Predicate:

```text
can assign jobs within time T
```

Binary search finds the smallest feasible `T`.

## Pattern 11. Geometry Thresholds

Geometric problems often reduce to distance thresholds.

Example:

```text
largest radius such that circles do not overlap
```

Predicate:

```text
placement feasible with radius r
```

Monotonicity follows from geometric constraints.

## Pattern 12. Compression + Binary Search

Large value ranges can be compressed, then searched.

Steps:

1. collect values  
2. compress to indices  
3. perform binary search or tree queries  

This enables efficient data structures.

## Pattern 13. Sliding Window + Binary Search

Sometimes the window size is unknown.

Example:

```text
maximum subarray length satisfying condition
```

Predicate:

```text
exists subarray of length ≥ k satisfying condition
```

Binary search over `k`.

## Pattern 14. Greedy + Binary Search

Greedy algorithms often provide feasibility checks.

Example:

```text
place k objects with distance ≥ d
```

Predicate uses greedy placement.

Binary search finds optimal `d`.

## Pattern 15. Monotone Matrix Search

In a matrix where rows and columns are sorted:

```text
search for value x
```

Binary search can be applied row-wise or using staircase traversal.

## Pattern 16. Bitmask Search Reduction

For some combinatorial problems:

```text
check if solution exists with cost ≤ x
```

Binary search over `x`, feasibility via DP or bitmask.

## Pattern 17. Prefix Sums + Binary Search

Use prefix sums to convert range queries into monotone checks.

Example:

```text
find earliest index where prefix sum ≥ target
```

Predicate:

```text
prefix[i] ≥ target
```

Binary search applies directly.

## Pattern 18. Two-Level Search

Combine binary search with another search.

Example:

- outer binary search on answer  
- inner linear or logarithmic check  

Total complexity:

```text
O(log R * cost(check))
```

## Pattern 19. Inverse Function Search

Given monotone function `f(x)`, find `x` such that:

```text
f(x) ≥ target
```

Binary search on `x`.

Examples:

- square root  
- cube root  
- inverse cumulative distribution  

## Pattern 20. Simulation as Predicate

Simulate a process and check if it succeeds.

Example:

```text
can process complete within time T
```

Simulation defines feasibility.

## Recognition Checklist

Before applying binary search, check:

```text
1. Is there an ordered parameter?
2. Can I define a yes/no predicate?
3. Is the predicate monotone?
4. Can I compute the predicate efficiently?
5. Can I bound the search space?
```

If all answers are yes, binary search is appropriate.

## Common Pitfalls

Forcing binary search when monotonicity does not hold.

Using a predicate that depends on multiple independent variables.

Ignoring cost of predicate.

Choosing bounds without proof.

Confusing value search with index search.

## Takeaway

Binary search is a reduction technique. Many problems reduce to boundary detection once an ordered parameter and a monotone predicate are identified. Recognizing these patterns turns complex problems into standard templates.
