# 7.11 Order Statistics Trees

Order statistics trees extend balanced binary search trees with the ability to answer rank and selection queries in logarithmic time. The key idea is to augment each node with the size of its subtree.

This augmentation turns an ordered map into a structure that supports both value-based queries and position-based queries.

## Problem

Maintain a dynamic set of keys with the following operations:

- insert a key  
- delete a key  
- find a key  
- find the k-th smallest element  
- find the rank of a key (number of elements strictly less than it)  

All operations must run in `O(log n)` time.

## Core Idea

Augment each node with a `size` field:

```text id="3hkg5m"
size(node) = 1 + size(left) + size(right)
```

This value represents the number of nodes in the subtree rooted at that node.

The BST invariant remains unchanged:

```text id="9r4l0s"
left subtree < node < right subtree
```

The `size` field allows navigation by position instead of by value.

## Select: k-th Smallest Element

To find the element with rank `k` (0-based):

```python id="5j7r1b"
def select(node, k):
    if not node:
        return None

    left_size = node.left.size if node.left else 0

    if k < left_size:
        return select(node.left, k)
    elif k == left_size:
        return node
    else:
        return select(node.right, k - left_size - 1)
```

At each step:

- compare `k` with the size of the left subtree  
- move left, return current node, or move right  

Each step discards half of the remaining nodes.

## Rank: Position of a Key

To compute how many elements are strictly less than a given key:

```python id="pxu1zz"
def rank(node, key):
    result = 0

    while node:
        if key <= node.key:
            node = node.left
        else:
            left_size = node.left.size if node.left else 0
            result += left_size + 1
            node = node.right

    return result
```

This accumulates counts as the traversal moves through the tree.

## Example

Consider the tree storing:

```text id="hfx2nn"
[2, 4, 6, 8, 10]
```

The structure might look like:

```text id="h91d9r"
        6 (size 5)
       / \\n   4 (2)  8 (2)
   /        \\n 2 (1)      10 (1)
```

To find the element with rank `2`:

- root: left size = 2  
- `k == left_size`, so return root (`6`)  

To find rank of `8`:

- move right from `6`, accumulate `2 + 1 = 3`  
- reach `8`, return `3`  

## Maintaining Size

The `size` field must be updated after every structural change.

After insertion or deletion:

```text id="9x1q7g"
node.size = 1 + size(left) + size(right)
```

During rotations:

```text id="j9u4qk"
update child sizes first, then parent
```

Failure to update sizes correctly breaks all order-statistics operations.

## Integration with Balanced Trees

Order statistics trees are usually built on top of balanced trees such as AVL or red black trees.

Insertion becomes:

1. insert as in BST  
2. update sizes along the path  
3. rebalance tree  
4. update sizes after rotations  

Deletion follows a similar pattern.

## Range Counting

Count elements in a range `[L, R]`:

```python id="9oy4jj"
def count_range(root, L, R):
    return rank(root, R + 1) - rank(root, L)
```

This uses two rank queries.

## Predecessor and Successor via Rank

Find predecessor:

```python id="b7xtq6"
def predecessor(root, key):
    r = rank(root, key)
    if r == 0:
        return None
    return select(root, r - 1)
```

Find successor:

```python id="3xq3km"
def successor(root, key):
    r = rank(root, key + 1)
    return select(root, r)
```

This reduces value-based queries to rank and select.

## Complexity

For a balanced tree:

```text id="u71r4u"
select: O(log n)
rank:   O(log n)
insert: O(log n)
delete: O(log n)
```

Space:

```text id="r2czd9"
O(n)
```

## Comparison with Arrays

| Operation        | Sorted Array | Order Statistics Tree |
|------------------|-------------|----------------------|
| Select k-th      | O(1)        | O(log n)             |
| Rank             | O(log n)    | O(log n)             |
| Insert           | O(n)        | O(log n)             |
| Delete           | O(n)        | O(log n)             |

Arrays are faster for static data. Trees support dynamic updates efficiently.

## Common Pitfalls

Forgetting to update `size` during rotations leads to incorrect ranks.

Using 1-based and 0-based indexing inconsistently causes off-by-one errors.

Assuming keys are distinct. If duplicates are allowed, the definition of rank must be clarified.

Not propagating size updates upward after insertion or deletion.

Using recursion without handling deep trees carefully.


## Takeaway

Order statistics trees augment balanced trees with subtree sizes to support position-based queries. The structure combines binary search with counting. Once sizes are maintained correctly, rank and selection become natural extensions of tree traversal.
