# 7.2 Lower Bound and Upper Bound

Binary search becomes reliable when expressed in terms of boundaries rather than equality. Two canonical operations capture this idea: **lower bound** and **upper bound**. These form the basis for most ordered queries, including counting, range extraction, and duplicate handling.

A lower bound locates the first position where a value could be inserted without violating order. An upper bound locates the first position strictly after all occurrences of a value. Together, they describe the exact interval occupied by equal elements.

## Problem

Given a sorted array `a` and a target `x`, compute:

- the first index `i` such that `a[i] >= x`
- the first index `j` such that `a[j] > x`

These indices define the half-open interval `[i, j)` that contains all occurrences of `x`.

## Solution

Both operations are direct applications of the monotone predicate template.

### Lower Bound

```python
def lower_bound(a: list[int], x: int) -> int:
    lo = 0
    hi = len(a)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if a[mid] >= x:
            hi = mid
        else:
            lo = mid + 1

    return lo
```

### Upper Bound

```python
def upper_bound(a: list[int], x: int) -> int:
    lo = 0
    hi = len(a)

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if a[mid] > x:
            hi = mid
        else:
            lo = mid + 1

    return lo
```

The difference lies only in the predicate. Lower bound uses `>=`, upper bound uses `>`.

## Example

Consider:

```text
a = [1, 2, 2, 2, 5, 7]
x = 2
```

Lower bound evaluates the predicate `a[i] >= 2`:

```text
false true true true true true
```

Upper bound evaluates `a[i] > 2`:

```text
false false false false true true
```

Therefore:

```text
lower_bound(a, 2) = 1
upper_bound(a, 2) = 4
```

The interval `[1, 4)` contains all elements equal to `2`.

## Counting Occurrences

The number of times `x` appears is:

```python
count = upper_bound(a, x) - lower_bound(a, x)
```

This works because the interval `[lower_bound, upper_bound)` includes exactly the indices where `a[i] == x`.

## Range Queries

To extract all elements equal to `x`:

```python
l = lower_bound(a, x)
r = upper_bound(a, x)
result = a[l:r]
```

To find all elements in a range `[L, R]`:

```python
l = lower_bound(a, L)
r = upper_bound(a, R)
result = a[l:r]
```

This pattern reduces range queries to two boundary searches.

## Invariant

Both functions maintain the same invariant structure:

```text
for all i < lo: predicate(i) is false
for all i >= hi: predicate(i) is true
```

For lower bound:

```text
predicate(i) = (a[i] >= x)
```

For upper bound:

```text
predicate(i) = (a[i] > x)
```

The correctness follows directly from the monotonicity of these predicates.

## Edge Cases

When all elements are smaller than `x`:

```text
a = [1, 2, 3]
x = 10
```

Both functions return `len(a)`.

When all elements are greater than or equal to `x`:

```text
a = [5, 6, 7]
x = 5
```

Lower bound returns `0`.

Upper bound returns the first index where `a[i] > 5`.

When `x` does not exist in the array:

```text
a = [1, 3, 5]
x = 4
```

Lower bound returns index `2`, the insertion point.

Upper bound also returns index `2`.

The interval is empty, so the count is zero.

## Derived Operations

These primitives support many common queries.

First occurrence:

```python
i = lower_bound(a, x)
if i < len(a) and a[i] == x:
    return i
return -1
```

Last occurrence:

```python
j = upper_bound(a, x) - 1
if j >= 0 and a[j] == x:
    return j
return -1
```

Insert position:

```python
pos = lower_bound(a, x)
```

This ensures the array remains sorted after insertion.

## Common Pitfalls

Using equality inside the loop instead of boundary predicates leads to fragile code. The boundary formulation is stable across duplicates and missing values.

Confusing lower and upper bound leads to off-by-one errors in counting. Always verify the interval `[l, r)` with a small example.

Forgetting that the returned index can be `len(a)` leads to out-of-bounds access. Always check before indexing.

## Complexity

Each operation runs in logarithmic time:

```text
O(log n)
```

Space usage remains constant.

## Takeaway

Lower bound and upper bound convert ordered search into boundary detection. They form a complete toolkit for working with sorted data. Once you adopt this view, many problems reduce to defining the correct predicate and applying the same binary search template.
