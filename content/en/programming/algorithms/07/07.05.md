# 7.5 Floating-Point Binary Search

Floating-point binary search is used when the answer lies in a continuous or nearly continuous domain. Instead of searching over integer indices, it searches over real numbers represented by floating-point values. The structure is the same as ordinary binary search: maintain an interval, test the midpoint, and keep the half that contains the answer.

The important difference is termination. Integer binary search can stop when `lo == hi` because there are only finitely many integer candidates. Floating-point search usually stops when the interval is small enough, or after a fixed number of iterations.

## Problem

Given a number `x >= 0`, compute an approximation of its square root.

We want a value `r` such that:

```text
r * r is approximately x
```

For example:

```text
sqrt(2) ≈ 1.41421356237
```

We can search for the smallest `r` such that:

```text
r * r >= x
```

This predicate is monotone for `r >= 0`.

## Solution

Use binary search over an interval that is guaranteed to contain the answer.

```python
def sqrt_binary(x: float, eps: float = 1e-12) -> float:
    if x < 0:
        raise ValueError("square root is undefined for negative input")

    lo = 0.0
    hi = max(1.0, x)

    while hi - lo > eps:
        mid = lo + (hi - lo) / 2.0

        if mid * mid >= x:
            hi = mid
        else:
            lo = mid

    return hi
```

The upper bound is `max(1.0, x)`. When `x >= 1`, the square root is at most `x`. When `0 <= x < 1`, the square root is larger than `x` but at most `1`.

## Why the Predicate Is Monotone

For nonnegative `r`, the function `r * r` increases as `r` increases. Therefore, if `r * r >= x`, then every larger value `s >= r` also satisfies `s * s >= x`.

The predicate:

```text
P(r) = r * r >= x
```

has this shape:

```text
False False False True True True
```

Binary search finds the boundary between values whose square is too small and values whose square is large enough.

## Fixed Iteration Version

A fixed number of iterations is often simpler and more predictable than an epsilon-based loop.

```python
def sqrt_binary_steps(x: float, steps: int = 80) -> float:
    if x < 0:
        raise ValueError("square root is undefined for negative input")

    lo = 0.0
    hi = max(1.0, x)

    for _ in range(steps):
        mid = lo + (hi - lo) / 2.0

        if mid * mid >= x:
            hi = mid
        else:
            lo = mid

    return hi
```

After `k` iterations, the interval length is at most:

```text
initial_length / 2^k
```

For double precision floating-point numbers, 60 to 100 iterations is usually enough for ordinary numeric ranges.

## Example

For `x = 2`, the interval begins as:

```text
lo = 0.0
hi = 2.0
```

The midpoint is `1.0`. Since `1.0 * 1.0 < 2`, the answer is to the right.

```text
lo = 1.0
hi = 2.0
```

The next midpoint is `1.5`. Since `1.5 * 1.5 >= 2`, the answer is at `1.5` or to the left.

```text
lo = 1.0
hi = 1.5
```

The process continues until the interval is small enough.

## Searching for Maximum Feasible Real Value

Some real-valued problems have a true-to-false predicate. For example, suppose `P(t)` means that a physical system can sustain load `t`. If load `t` is feasible, every smaller load is also feasible. The goal is to find the largest feasible `t`.

```python
def last_true_real(lo: float, hi: float, pred, steps: int = 80) -> float:
    for _ in range(steps):
        mid = lo + (hi - lo) / 2.0

        if pred(mid):
            lo = mid
        else:
            hi = mid

    return lo
```

Here `lo` should be a known feasible value, and `hi` should be a known infeasible value or at least a valid upper boundary beyond the answer.

## Choosing Between Epsilon and Fixed Steps

Use an epsilon when the problem statement asks for a specific absolute error, such as `1e-6`.

```python
while hi - lo > eps:
    ...
```

Use fixed steps when the numeric scale varies or when you want deterministic runtime.

```python
for _ in range(80):
    ...
```

In competitive programming, fixed steps are common because they avoid subtle termination issues. In production numeric code, the tolerance should match the domain and error requirements.

## Absolute and Relative Error

Absolute error measures the direct difference:

```text
abs(approx - exact)
```

Relative error measures the error compared with the size of the true value:

```text
abs(approx - exact) / max(1, abs(exact))
```

Absolute error is appropriate when answers have a fixed scale. Relative error is better when answers may vary over many orders of magnitude.

For example, an absolute error of `1e-6` is excellent for an answer near `1`, but may be meaningless for an answer near `10^12`.

## Common Pitfalls

Do not use `while lo < hi` for floating-point binary search. Floating-point values may stop changing when the interval becomes very small, causing an infinite loop.

Do not stop by checking `mid == lo` unless you deliberately handle machine precision. A fixed iteration count is simpler.

Do not use floating-point binary search when an exact integer answer is required. Search over integers instead.

Do not forget to prove monotonicity. Real-valued functions can increase and then decrease, or oscillate. Binary search works only when the predicate changes direction once.

Do not square very large values without considering overflow. For very large `x`, compare with division instead:

```python
if mid >= x / mid:
    hi = mid
else:
    lo = mid
```

This requires handling `mid == 0`.

## Complexity

Each iteration performs one predicate check. If the loop runs for `k` iterations and the predicate costs `T`, the total time is:

```text
O(kT)
```

For epsilon-based search over an initial interval of length `L`, the number of iterations is about:

```text
log2(L / eps)
```

The extra space usage is:

```text
O(1)
```

## Testing Checklist

Test zero.

```python
assert abs(sqrt_binary(0.0) - 0.0) < 1e-9
```

Test one.

```python
assert abs(sqrt_binary(1.0) - 1.0) < 1e-9
```

Test a non-square.

```python
assert abs(sqrt_binary(2.0) * sqrt_binary(2.0) - 2.0) < 1e-9
```

Test a number between zero and one.

```python
assert abs(sqrt_binary(0.25) - 0.5) < 1e-9
```

Test a large value.

```python
r = sqrt_binary(10**12)
assert abs(r * r - 10**12) / 10**12 < 1e-9
```

## Takeaway

Floating-point binary search is boundary search over real-valued candidates. The predicate must be monotone, the interval must contain the answer, and the loop must stop by tolerance or by a fixed number of iterations. The integer template remains conceptually valid, but the termination rule changes.
