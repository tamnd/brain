# 7.7 Peak Finding

Peak finding asks for an index whose value is not smaller than its neighbors. Unlike ordinary binary search, the array does not need to be sorted. The useful structure is directional: if the array is rising at position `mid`, then some peak exists to the right; if the array is falling or flat, then some peak exists to the left or at `mid`.

## Problem

Given an array `a`, find an index `i` such that:

```text id="dh4pdz"
a[i] >= a[i - 1] when i > 0
a[i] >= a[i + 1] when i + 1 < len(a)
```

For example:

```text id="h54yfv"
a = [1, 3, 5, 4, 2]
```

Index `2` is a peak because `a[2] = 5`, and it is greater than both neighbors.

For another example:

```text id="b78ewk"
a = [1, 2, 3, 4]
```

Index `3` is a peak because the right edge has no right neighbor.

## Solution

Compare `a[mid]` with `a[mid + 1]`.

```python id="tg1br1"
def find_peak(a: list[int]) -> int:
    if not a:
        raise ValueError("empty array has no peak")

    lo = 0
    hi = len(a) - 1

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if a[mid] < a[mid + 1]:
            lo = mid + 1
        else:
            hi = mid

    return lo
```

The loop uses the closed interval `[lo, hi]`. The expression `a[mid + 1]` is safe because `lo < hi`, so `mid < hi`.

## Why It Works

If `a[mid] < a[mid + 1]`, the array is rising from `mid` to `mid + 1`. In that case, a peak must exist in the right half. Starting at `mid + 1`, either the sequence keeps rising until the end, in which case the last element is a peak, or it eventually stops rising, in which case the first position where it stops is a peak.

So it is safe to discard `mid` and everything to its left:

```python id="9x9riu"
lo = mid + 1
```

If `a[mid] >= a[mid + 1]`, the array is falling or flat from `mid` to `mid + 1`. A peak must exist in the left half including `mid`. Starting at `mid`, either the sequence keeps rising as we move left backward, in which case the left edge is a peak, or it eventually stops, in which case there is a peak before or at `mid`.

So it is safe to keep `mid` and discard everything strictly to its right:

```python id="pzmt2a"
hi = mid
```

Each step preserves the fact that the current interval contains at least one peak.

## Example Walkthrough

Consider:

```text id="6r26q5"
a = [1, 3, 5, 4, 2]
```

Initial interval:

```text id="y0np5b"
lo = 0, hi = 4
```

First midpoint:

```text id="pyz70u"
mid = 2
a[mid] = 5
a[mid + 1] = 4
```

Since `5 >= 4`, a peak exists at `mid` or to the left.

```text id="xw5d13"
hi = 2
```

Second midpoint:

```text id="vcwx17"
lo = 0, hi = 2, mid = 1
a[mid] = 3
a[mid + 1] = 5
```

Since `3 < 5`, a peak exists to the right.

```text id="v8oqv3"
lo = 2
```

Now `lo == hi == 2`, so the algorithm returns `2`.

## Edge Cases

For a one-element array:

```python id="rqs2xq"
assert find_peak([10]) == 0
```

The only element is a peak.

For a strictly increasing array:

```python id="u0qr5i"
assert find_peak([1, 2, 3, 4]) == 3
```

The last element is a peak.

For a strictly decreasing array:

```python id="jbh2po"
assert find_peak([4, 3, 2, 1]) == 0
```

The first element is a peak.

For an array with repeated values:

```python id="6hc8w5"
i = find_peak([1, 2, 2, 2, 1])
assert 0 <= i < 5
assert (i == 0 or [1, 2, 2, 2, 1][i] >= [1, 2, 2, 2, 1][i - 1])
assert (i == 4 or [1, 2, 2, 2, 1][i] >= [1, 2, 2, 2, 1][i + 1])
```

With the `>=` definition of peak, a flat plateau may contain many valid answers.

## Variant: Strict Peak

Some problem statements define a peak as strictly greater than both neighbors. In that version, duplicates change the problem. An array such as:

```text id="5sc7v8"
[2, 2, 2]
```

has no strict peak. The simple binary search above solves the non-strict version. For strict peaks, the input usually includes an assumption such as adjacent elements are distinct, or artificial boundaries are negative infinity.

Under the assumption that adjacent elements are distinct, the same algorithm works and returns a strict peak.

```python id="47zpp9"
def find_strict_peak_distinct_neighbors(a: list[int]) -> int:
    if not a:
        raise ValueError("empty array has no peak")

    lo = 0
    hi = len(a) - 1

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if a[mid] < a[mid + 1]:
            lo = mid + 1
        else:
            hi = mid

    return lo
```

The proof is the same, but flat regions never occur.

## Two-Dimensional Peak Finding

The same idea extends to a matrix. A cell is a peak if it is not smaller than its up, down, left, and right neighbors.

One common method binary searches over columns. At each step, find the maximum element in the middle column. If the left neighbor is larger, a peak exists on the left. If the right neighbor is larger, a peak exists on the right. Otherwise, the middle-column maximum is a peak.

```python id="p92gb4"
def find_peak_2d(mat: list[list[int]]) -> tuple[int, int]:
    if not mat or not mat[0]:
        raise ValueError("empty matrix has no peak")

    rows = len(mat)
    cols = len(mat[0])

    lo = 0
    hi = cols - 1

    while lo <= hi:
        mid_col = lo + (hi - lo) // 2

        max_row = 0
        for r in range(1, rows):
            if mat[r][mid_col] > mat[max_row][mid_col]:
                max_row = r

        left = mat[max_row][mid_col - 1] if mid_col > 0 else float("-inf")
        right = mat[max_row][mid_col + 1] if mid_col + 1 < cols else float("-inf")
        cur = mat[max_row][mid_col]

        if cur >= left and cur >= right:
            return (max_row, mid_col)
        elif left > cur:
            hi = mid_col - 1
        else:
            lo = mid_col + 1

    raise RuntimeError("unreachable")
```

This returns a non-strict peak. Each iteration scans one column, so if the matrix has `m` rows and `n` columns, the running time is:

```text id="24k4xo"
O(m log n)
```

## Correctness Argument

For the one-dimensional algorithm, maintain this invariant:

```text id="oi7ue3"
There exists at least one peak inside [lo, hi].
```

Initially, every nonempty finite array has at least one peak. If the whole array is increasing, the last element is a peak. Otherwise, at the first position where the increase stops, there is a peak.

During the loop, compare `a[mid]` and `a[mid + 1]`.

If `a[mid] < a[mid + 1]`, a peak exists in `[mid + 1, hi]`, so the invariant holds after setting `lo = mid + 1`.

If `a[mid] >= a[mid + 1]`, a peak exists in `[lo, mid]`, so the invariant holds after setting `hi = mid`.

The interval length decreases each time. When one index remains, the invariant says that index is a peak.

## Complexity

The one-dimensional algorithm discards about half of the remaining interval at each step.

```text id="x7ik40"
Time:  O(log n)
Space: O(1)
```

The two-dimensional column-search algorithm scans one column per iteration.

```text id="p9ygq0"
Time:  O(m log n)
Space: O(1)
```

where `m` is the number of rows and `n` is the number of columns.

## Common Pitfalls

Do not compare `a[mid]` with both neighbors unless the boundary cases are handled carefully. The `a[mid]` versus `a[mid + 1]` comparison is enough.

Do not use `while lo <= hi` with the one-dimensional template above. The loop depends on accessing `mid + 1`, and the condition `lo < hi` makes that access safe.

Do not assume the returned peak is unique. Many arrays have several valid peaks.

Do not use this algorithm for finding the maximum. A peak can be local. The algorithm finds some peak, not necessarily the largest value.

## Testing Checklist

```python id="e39n2d"
def is_peak(a: list[int], i: int) -> bool:
    return (
        0 <= i < len(a)
        and (i == 0 or a[i] >= a[i - 1])
        and (i + 1 == len(a) or a[i] >= a[i + 1])
    )

for a in [
    [10],
    [1, 2, 3, 4],
    [4, 3, 2, 1],
    [1, 3, 5, 4, 2],
    [1, 2, 2, 2, 1],
    [1, 3, 2, 4, 1],
]:
    i = find_peak(a)
    assert is_peak(a, i)
```

## Takeaway

Peak finding uses binary search without sorted data. The reason it works is that a rising slope guarantees a peak to the right, while a falling or flat slope guarantees a peak to the left or at the current index. The invariant is not "the target remains inside the interval"; it is "some peak remains inside the interval."
