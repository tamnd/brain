# 7.6 Rotated Arrays

A rotated array is obtained by taking a sorted array and shifting a prefix to the end. The resulting array consists of two sorted segments. Many search problems on rotated arrays can be solved in logarithmic time by adapting binary search to account for the split.

## Problem

Given a rotated sorted array with distinct values, find the index of a target value `x`. If `x` does not exist, return `-1`.

Example:

```text
a = [4, 5, 6, 7, 0, 1, 2]
x = 0
```

The answer is index `4`.

## Structure

A rotated array has this form:

```text
sorted:   [0, 1, 2, 4, 5, 6, 7]
rotated:  [4, 5, 6, 7, 0, 1, 2]
```

There exists a pivot where the order breaks. The array can be viewed as two increasing segments:

```text
[4, 5, 6, 7] and [0, 1, 2]
```

At any index `mid`, at least one side of the array remains sorted. This observation drives the algorithm.

## Solution

Perform binary search while determining which half is sorted at each step.

```python
def search_rotated(a: list[int], x: int) -> int:
    lo = 0
    hi = len(a) - 1

    while lo <= hi:
        mid = lo + (hi - lo) // 2

        if a[mid] == x:
            return mid

        # left half is sorted
        if a[lo] <= a[mid]:
            if a[lo] <= x < a[mid]:
                hi = mid - 1
            else:
                lo = mid + 1
        else:
            # right half is sorted
            if a[mid] < x <= a[hi]:
                lo = mid + 1
            else:
                hi = mid - 1

    return -1
```

## Why It Works

At each step, one of the halves `[lo, mid]` or `[mid, hi]` must be sorted.

If the left half is sorted, then:

```text
a[lo] <= a[mid]
```

You can check whether `x` lies within this sorted interval. If it does, search the left half. Otherwise, search the right half.

If the right half is sorted, then:

```text
a[mid] <= a[hi]
```

You perform the symmetric check.

Each iteration discards half of the array, preserving logarithmic complexity.

## Example Walkthrough

```text
a = [4, 5, 6, 7, 0, 1, 2]
x = 0
```

Iteration 1:

```text
lo = 0, hi = 6, mid = 3, a[mid] = 7
```

Left half `[4, 5, 6, 7]` is sorted. Since `0` is not in `[4, 7)`, search right half.

```text
lo = 4, hi = 6
```

Iteration 2:

```text
mid = 5, a[mid] = 1
```

Left half `[0, 1]` is sorted. Since `0` is in `[0, 1)`, search left half.

```text
lo = 4, hi = 4
```

Iteration 3:

```text
mid = 4, a[mid] = 0
```

Found.

## Finding the Rotation Pivot

Sometimes the pivot itself is required. The pivot is the index of the smallest element.

```python
def find_pivot(a: list[int]) -> int:
    lo = 0
    hi = len(a) - 1

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if a[mid] > a[hi]:
            lo = mid + 1
        else:
            hi = mid

    return lo
```

The predicate here is:

```text
a[i] <= a[hi]
```

The smallest element lies in the region where this condition holds.

## Search via Pivot

An alternative approach is:

1. Find the pivot.
2. Perform standard binary search in the correct half.

```python
def search_rotated_v2(a: list[int], x: int) -> int:
    if not a:
        return -1

    pivot = find_pivot(a)

    if a[pivot] <= x <= a[-1]:
        lo, hi = pivot, len(a) - 1
    else:
        lo, hi = 0, pivot - 1

    while lo <= hi:
        mid = lo + (hi - lo) // 2

        if a[mid] == x:
            return mid
        elif a[mid] < x:
            lo = mid + 1
        else:
            hi = mid - 1

    return -1
```

This separates the concerns: first restore order, then apply standard search.

## Handling Duplicates

Duplicates complicate the structure. Consider:

```text
[2, 2, 2, 3, 4, 2]
```

At some points, it becomes impossible to determine which half is sorted because:

```text
a[lo] == a[mid] == a[hi]
```

In this case, reduce the search space conservatively:

```python
def search_rotated_duplicates(a: list[int], x: int) -> bool:
    lo = 0
    hi = len(a) - 1

    while lo <= hi:
        mid = lo + (hi - lo) // 2

        if a[mid] == x:
            return True

        if a[lo] == a[mid] == a[hi]:
            lo += 1
            hi -= 1
        elif a[lo] <= a[mid]:
            if a[lo] <= x < a[mid]:
                hi = mid - 1
            else:
                lo = mid + 1
        else:
            if a[mid] < x <= a[hi]:
                lo = mid + 1
            else:
                hi = mid - 1

    return False
```

This may degrade to linear time in the worst case, but remains efficient when duplicates are limited.

## Complexity

For distinct elements:

```text
Time:  O(log n)
Space: O(1)
```

With many duplicates:

```text
Worst-case time: O(n)
```

## Common Pitfalls

Assuming the entire array is sorted leads to incorrect comparisons. Always identify the sorted half before deciding.

Using strict inequalities incorrectly can exclude valid indices. Pay attention to boundary conditions such as `<=` and `<`.

Forgetting that the array may not be rotated at all. The algorithm still works because one half remains sorted.

Mixing pivot-based and direct methods leads to unnecessary complexity. Choose one approach and apply it consistently.

## Testing Checklist

No rotation:

```python
assert search_rotated([1, 2, 3, 4, 5], 3) == 2
```

Full rotation:

```python
assert search_rotated([1, 2, 3, 4, 5], 1) == 0
```

Typical rotation:

```python
assert search_rotated([4, 5, 6, 7, 0, 1, 2], 0) == 4
```

Missing element:

```python
assert search_rotated([4, 5, 6, 7, 0, 1, 2], 3) == -1
```

Duplicates:

```python
assert search_rotated_duplicates([2, 2, 2, 3, 4, 2], 3) is True
```

## Takeaway

Rotated arrays preserve partial order. At each step, one half remains sorted. Binary search can exploit this structure by detecting the sorted half and discarding the other. When duplicates appear, the structure weakens, and the algorithm must fall back to cautious shrinking.
