# 7.13 Parametric Search

Parametric search turns an optimization problem into a sequence of decision problems and then uses binary search to locate the optimal parameter. It is closely related to search on answer, but the emphasis is on structuring the optimization as a monotone feasibility problem with a clear parameter.

## Problem

Given an optimization objective, such as minimizing time, maximizing throughput, or minimizing cost under constraints, compute the optimal value efficiently.

Example:

You are given `n` machines, where machine `i` produces one item every `t[i]` units of time. Find the minimum time required to produce at least `k` items.

```text id="wqjpjp"
t = [2, 3, 7]
k = 10
```

## Reformulation

Define a predicate:

```text id="ox1h2g"
P(T) = machines can produce at least k items within time T
```

For a fixed time `T`, the number of items produced is:

```text id="r9i7t6"
sum(T // t[i])
```

This predicate is monotone:

```text id="v2z7d1"
False False False True True True
```

If time `T` is sufficient, any larger time is also sufficient.

## Solution

Binary search for the smallest `T` such that `P(T)` is true.

```python id="xk8g4n"
def min_time(t: list[int], k: int) -> int:
    def feasible(T: int) -> bool:
        total = 0
        for ti in t:
            total += T // ti
            if total >= k:
                return True
        return False

    lo = 0
    hi = min(t) * k

    while lo < hi:
        mid = lo + (hi - lo) // 2

        if feasible(mid):
            hi = mid
        else:
            lo = mid + 1

    return lo
```

The upper bound `min(t) * k` corresponds to the fastest machine producing all items alone.

## Structure of Parametric Search

Every parametric search has the same components:

1. parameter space  
2. feasibility predicate  
3. monotonicity proof  
4. bounds  

The parameter is the value being optimized. The predicate answers whether a given parameter value is sufficient.

## Monotonicity Argument

Assume time `T` is sufficient. Then machines produce at least `k` items in time `T`. For any `T' > T`, each machine produces at least as many items as before:

```text id="d0ot0h"
T' // t[i] >= T // t[i]
```

Therefore:

```text id="ruqz0l"
sum(T' // t[i]) >= sum(T // t[i]) >= k
```

So `P(T')` is also true.

## Another Example: Maximum Average Subarray

Given an array, find the maximum average of any subarray of length at least `L`.

Instead of searching directly, define:

```text id="zxgxsk"
P(x) = there exists a subarray of length ≥ L with average ≥ x
```

Transform the array:

```text id="c6ntak"
b[i] = a[i] - x
```

Then `P(x)` becomes:

```text id="fsr0jm"
there exists a subarray of length ≥ L with sum ≥ 0
```

This can be checked using prefix sums.

```python id="2j0jns"
def feasible(a: list[float], L: int, x: float) -> bool:
    n = len(a)
    prefix = [0.0] * (n + 1)

    for i in range(n):
        prefix[i + 1] = prefix[i] + (a[i] - x)

    min_prefix = 0.0

    for i in range(L, n + 1):
        if prefix[i] - min_prefix >= 0:
            return True
        min_prefix = min(min_prefix, prefix[i - L + 1])

    return False
```

Binary search on `x` gives the maximum average.

## Continuous Parametric Search

When the parameter is real-valued, use floating-point binary search.

```python id="uxg5xw"
def maximize_average(a: list[float], L: int) -> float:
    lo = min(a)
    hi = max(a)

    for _ in range(60):
        mid = (lo + hi) / 2.0

        if feasible(a, L, mid):
            lo = mid
        else:
            hi = mid

    return lo
```

## Choosing Bounds

The bounds must include the optimal answer.

For minimum search:

```text id="9r7v0u"
lo = smallest possible value
hi = guaranteed feasible value
```

For maximum search:

```text id="jsm6vb"
lo = guaranteed feasible value
hi = largest possible value
```

Poor bounds lead to incorrect results or wasted iterations.

## Complexity

Let `T_pred` be the cost of evaluating the predicate.

Binary search performs:

```text id="qtxc8c"
O(log range)
```

iterations.

Total complexity:

```text id="2hql1n"
O(T_pred * log range)
```

For integer domains, `range = hi - lo`.

For floating-point domains, the number of iterations is fixed.

## When to Use Parametric Search

Use it when:

- the problem asks for minimum or maximum value  
- a feasibility check is easier than direct optimization  
- the feasibility predicate is monotone  

Typical domains:

- scheduling  
- resource allocation  
- load balancing  
- geometric optimization  
- averages and ratios  

## Common Pitfalls

Failing to prove monotonicity leads to incorrect results.

Using a predicate that is too expensive defeats the purpose.

Choosing bounds that do not include the answer breaks correctness.

Mixing integer and floating-point domains incorrectly introduces precision issues.

Forgetting early exit in predicate increases runtime unnecessarily.

## Testing Checklist

Minimum time example:

```python id="q0qz7c"
assert min_time([2, 3, 7], 10) == 12
```

Edge case:

```python id="v8xxn5"
assert min_time([5], 1) == 5
```

Large case:

```python id="9z2v0l"
assert min_time([1, 2, 3], 1000000) >= 333334
```

Floating example:

```python id="8k2rtb"
a = [1.0, 12.0, -5.0, -6.0, 50.0, 3.0]
result = maximize_average(a, 4)
assert result > 10.0
```

## Takeaway

Parametric search reduces optimization to boundary detection over a parameter space. The key step is designing a monotone feasibility predicate. Once that is established, binary search locates the optimal value efficiently.
