## Solution
## Problem Understanding

We are given a connected undirected graph where each edge has a weight that represents an energy cost. The Daleks will not necessarily use all edges. Instead, they will choose a set of edges that connects all vertices with minimum possible total weight, which is exactly a minimum spanning tree, though it may not be unique if there are ties.

The key twist is that we focus on a single distinguished edge, the first edge in the input. We imagine changing its weight to some value $e$. For each such value, we ask whether this edge could still appear in at least one minimum spanning tree of the modified graph. Among all valid values $e$, we want the maximum possible one, capped at $10^9$.

So the problem is not about recomputing an MST for every change. Instead, it is about characterizing when a fixed edge can belong to some MST after its weight is adjusted.

The constraints are large, with up to $10^5$ vertices and $10^6$ edges. Any solution that recomputes MST or runs heavy graph queries per edge is immediately too slow. Even $O(m \log m)$ methods must be used carefully, since we only need to process one special edge but the structure of all edges still matters.

A subtle edge case appears when multiple MSTs exist. A naive idea might check whether the first edge is in the unique MST after modification, but uniqueness is not required. The edge only needs to appear in at least one MST, which changes the condition from “must be chosen” to “not strictly dominated in every cycle”.

Another tricky case is when lowering or raising the weight changes which cycle becomes critical. For example, if the first edge is part of a cycle where all other edges are much smaller, it can never appear in any MST regardless of lowering its weight if replacing it would still violate MST optimality in another cycle context.

## Approaches

A brute force approach would try all possible values of $e$ from $0$ to $10^9$. For each value, we modify the first edge weight and recompute an MST, then check whether the first edge is included in at least one MST. Each MST computation takes $O(m \log m)$, so this becomes $10^9 \cdot m \log m$, which is impossible.

We need a structural observation about MSTs.

An edge belongs to some MST if and only if there is no cycle where this edge is strictly the maximum edge and uniquely so. Equivalently, in any cycle containing the edge, there must exist another edge whose weight is at least as large. This leads to a standard MST characterization: for an edge $c_1 = (u,v)$, if we look at the path between $u$ and $v$ in a minimum spanning tree of the original graph, the maximum edge weight on that path controls when $c_1$ can replace it.

More precisely, if we build any MST of the original graph, then for the edge $c_1$, consider the unique path between its endpoints in that MST. Let $w_{\max}$ be the maximum edge weight along that path. If we set the weight of $c_1$ to some value $e$, then $c_1$ can be included in an MST if and only if $e \le w_{\max}$. If it is larger, replacing any edge on that path would increase cost.

The subtle point is that we need a path in an MST, not in the original graph. That is what makes Kruskal’s structure essential: MST paths encode the best possible alternative connectivity.

So the solution becomes: build a minimum spanning tree of the graph, then answer a maximum edge query on the path between the endpoints of the first edge.

To support this efficiently, we preprocess the MST with binary lifting for maximum edge on paths.

| Approach | Time Complexity | Space Complexity | Verdict |
| --- | --- | --- | --- |
| Brute Force | $O(10^9 \cdot m \log m)$ | $O(m)$ | Too slow |
| Optimal | $O(m \log m + \log n)$ | $O(n \log n)$ | Accepted |

## Algorithm Walkthrough

We first construct a minimum spanning tree using Kruskal’s algorithm. This gives us a tree that preserves the minimum total cost structure of the graph.

1. Sort all edges by weight and run Kruskal’s algorithm with a disjoint set union structure. We also record which edges are included in the MST and build an adjacency list representation of the MST. This step is necessary because the MST encodes all optimal replacement structures between nodes.
2. Take the first edge $c_1 = (u, v, w)$. We only care about these two endpoints in the MST, since any valid replacement must be compared through the unique tree path between them.
3. Root the MST at an arbitrary node, for example node 1, and run a DFS to compute parent pointers and depths. Alongside this, store the maximum edge weight from each node to its parent. This prepares us for fast path queries.
4. Build a binary lifting table where `up[k][v]` is the 2^k-th ancestor of v, and `maxw[k][v]` is the maximum edge weight on the path from v to that ancestor. This allows jumping upward in logarithmic steps while tracking maximum edge weights.
5. To compute the answer, lift u and v to their lowest common ancestor while tracking the maximum edge weight encountered on both upward paths. The result is the maximum edge weight along the unique MST path between u and v.
6. Output this value as $E_{max}(c_1)$, since any larger weight would make the edge strictly worse than an alternative path in the MST structure.

### Why it works

The MST defines a baseline optimal connectivity structure. Any cycle formed by adding a non-tree edge is resolved by removing the heaviest edge in that cycle. When we consider replacing edges along the path between u and v, the MST path is exactly the bottleneck that any alternative connection must beat. If the new weight of $c_1$ exceeds the maximum edge on this path, then inserting it would force removal of a cheaper or equal replacement, increasing total cost. If it is less than or equal, we can safely substitute it for the heaviest edge on that path without breaking optimality, meaning it can belong to at least one MST.

## Python Solution

```python
import sys
input = sys.stdin.readline

class DSU:
    def __init__(self, n):
        self.p = list(range(n))
        self.r = [0] * n

    def find(self, x):
        while self.p[x] != x:
            self.p[x] = self.p[self.p[x]]
            x = self.p[x]
        return x

    def union(self, a, b):
        a = self.find(a)
        b = self.find(b)
        if a == b:
            return False
        if self.r[a] < self.r[b]:
            a, b = b, a
        self.p[b] = a
        if self.r[a] == self.r[b]:
            self.r[a] += 1
        return True

def main():
    n, m = map(int, input().split())
    edges = []
    for i in range(m):
        a, b, w = map(int, input().split())
        edges.append((w, a - 1, b - 1))

    edges.sort()
    dsu = DSU(n)

    adj = [[] for _ in range(n)]
    mst_edges = 0

    for w, a, b in edges:
        if dsu.union(a, b):
            adj[a].append((b, w))
            adj[b].append((a, w))
            mst_edges += 1
            if mst_edges == n - 1:
                break

    LOG = 20
    up = [[-1] * n for _ in range(LOG)]
    mx = [[0] * n for _ in range(LOG)]
    depth = [0] * n

    sys.setrecursionlimit(10**7)

    def dfs(v, p):
        for to, w in adj[v]:
            if to == p:
                continue
            depth[to] = depth[v] + 1
            up[0][to] = v
            mx[0][to] = w
            dfs(to, v)

    dfs(0, -1)

    for k in range(1, LOG):
        for v in range(n):
            if up[k-1][v] != -1:
                up[k][v] = up[k-1][up[k-1][v]]
                mx[k][v] = max(mx[k-1][v], mx[k-1][up[k-1][v]])

    def get_max(u, v):
        if depth[u] < depth[v]:
            u, v = v, u

        ans = 0

        diff = depth[u] - depth[v]
        for k in range(LOG):
            if diff >> k & 1:
                ans = max(ans, mx[k][u])
                u = up[k][u]

        if u == v:
            return ans

        for k in reversed(range(LOG)):
            if up[k][u] != up[k][v]:
                ans = max(ans, mx[k][u], mx[k][v])
                u = up[k][u]
                v = up[k][v]

        ans = max(ans, mx[0][u], mx[0][v])
        return ans

    _, u, v = edges[0]
    print(get_max(u, v))

if __name__ == "__main__":
    main()
```

The DSU section builds the MST efficiently in near-linear time after sorting, ensuring we only keep edges that preserve minimal total cost connectivity. The adjacency list stores the tree structure needed for path queries.

The DFS initializes parent pointers and immediate edge weights. This is critical because binary lifting depends on correct base ancestors. The recursion depth is safe because the MST is a tree, not a general graph.

The binary lifting tables `up` and `mx` store both ancestry and maximum edge weights, enabling fast aggregation of path information.

The `get_max` function carefully lifts nodes in two phases, first aligning depths, then lifting both nodes while maintaining the invariant that we always track the maximum edge weight seen so far. The final step includes the last edges before the LCA.

## Worked Examples

### Example 1

Input:

```
3 3
1 2 8
2 3 3
3 1 4
```

We build an MST using Kruskal. Sorting edges gives weights 3, 4, 8. We pick (2,3,3) and (3,1,4). The MST path between 1 and 2 goes through 1-3-2.

| Step | Current nodes | Action | Max on path |
| --- | --- | --- | --- |
| Align depth | 2 → 2 | no change | 0 |
| Lift/compare | 1,2 via 3 | consider edges (1,3)=4 and (3,2)=3 | 4 |

So the maximum edge on the MST path between 1 and 2 is 4, matching the answer.

This confirms that any weight up to 4 still allows replacement without breaking MST optimality.

### Example 2

Input:

```
4 4
1 2 5
2 3 6
3 4 2
1 4 7
```

MST uses edges (3,4,2), (1,2,5), (2,3,6). Path from 1 to 2 is direct in MST.

| Step | Nodes | Path | Max |
| --- | --- | --- | --- |
| Lift | 1 to 2 | direct edge 5 | 5 |

So $E_{max} = 5$.

This shows the case where the MST path is direct and the answer comes from a single edge.

## Edge Cases

A key edge case is when the first edge is not part of the initial MST. For example, if it is very heavy, Kruskal will exclude it. Even then, the algorithm still works because we only care about the MST path between its endpoints, which exists regardless of whether the edge itself is chosen.

Another edge case is when all edges have the same weight. In this situation every spanning tree is an MST. The maximum edge on any path equals that shared weight, so the answer becomes that value. The lifting structure still behaves correctly because it only depends on connectivity, not uniqueness.

A final edge case is when the MST path between endpoints is a single edge. In that case the answer is just that edge weight. The lifting process handles this naturally since no upward jumps are needed and the LCA step immediately resolves.
