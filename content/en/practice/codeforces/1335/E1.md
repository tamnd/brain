## Solution

## Problem Understanding

We are given a sequence of integers and asked to extract a subsequence that has a very specific structure. The subsequence must look like a block of repeated value `x`, followed by a block of repeated value `y`, followed again by the same value `x`. The two values are allowed to be equal, which collapses the structure into a single uniform block.

The task is not to construct any valid subsequence, but to maximize its length under this constraint. Since we are free to delete elements while preserving order, the problem is really about choosing where the left block starts, where the middle block lies, and how much we can reuse matching elements from both ends.

The constraint `n ≤ 2000` per test case, with total sum also `≤ 2000`, suggests that an `O(n^3)` solution per test is too slow, but `O(n^2)` or `O(n^2 log n)` is acceptable. The small alphabet size (values up to 26) is an important hint: we can afford to try all value pairs `(x, y)`.

A naive mistake is to try all subsequences directly or attempt DP over all substrings without structure, which leads to cubic behavior or worse. Another subtle issue is assuming the middle block must be contiguous in the original array, which is false since we are working with subsequences.

Edge cases include arrays where all values are equal, arrays where optimal solution uses only one value, and cases where the best structure uses a very small middle block or even empty middle.

For example, in `[1, 1, 1, 1]`, the best answer is `4`, achieved by choosing `x = 1, y = 1`. A careless approach that forces `x ≠ y` would incorrectly reduce the answer.

Another example is `[1, 2, 3, 2, 1]`. It looks symmetric, but it is not valid because the middle cannot support a consistent block structure while preserving subsequence constraints.

## Approaches

The brute force idea is to choose values `x` and `y`, then try to construct the best subsequence of the form `x...x y...y x...x`. For fixed `(x, y)`, we would scan the array and decide how many `x` we can take from the left, how many `y` we can take in the middle, and how many `x` from the right. If we try to split positions explicitly, we end up checking all triples of split points, which is `O(n^3)` per pair, and with 26 values this becomes far too slow.

The key observation is that the outer blocks depend only on how many occurrences of `x` we can take from the prefix and suffix, while the middle block depends on a subarray between chosen boundaries. Instead of explicitly choosing boundaries, we fix the number of `x` taken from both ends, say `k`, and then look at the segment between the `k`-th occurrence from the left and the `k`-th occurrence from the right. Inside that segment, we only need the best frequency of some value `y`.

So for each value `x`, we precompute its occurrence positions. We try all possible `k` such that we can take `k` occurrences from both ends. For each such pair of boundary positions, we scan the middle segment and compute the maximum frequency of any value `y`. The answer is `2k + best_middle`.

This reduces the problem to `O(26 * n^2)`, since for each `x` we try up to `O(n)` values of `k`, and each middle scan is `O(n)`.

| Approach | Time Complexity | Space Complexity | Verdict |
| --- | --- | --- | --- |
| Brute Force | O(26² · n³) | O(n) | Too slow |
| Optimal | O(26 · n²) | O(n) | Accepted |

## Algorithm Walkthrough

We process each test case independently.

1. Build a list of positions for each value from 1 to 26. This allows us to quickly know where each value occurs, which is essential for fixing outer blocks.
2. For each value `x`, consider all possible ways to choose symmetric outer blocks. If `pos[x]` contains `m` occurrences, we try `k = 0` to `m // 2`. This represents taking `k` occurrences from the start and `k` from the end. This guarantees valid subsequence structure because indices remain increasing.
3. For each `k`, define the left boundary as `pos[x][k - 1]` and the right boundary as `pos[x][m - k]`. The middle segment is everything strictly between these two indices.
4. Scan the middle segment and compute frequency counts of all values `y`. The best possible middle block is the maximum frequency among all values in that segment. This works because the middle block must be a single repeated value.
5. Update the answer with `2 * k + best_middle`.
6. Also consider the case where we take no outer blocks, which reduces the problem to finding the most frequent element in the whole array.

### Why it works

The construction separates the subsequence into independent structural decisions. Once the outer `k` pairs of `x` are fixed, any valid completion depends only on the multiset of elements in the middle interval. Since the middle must be a single repeated value, the optimal choice is simply the most frequent element in that interval. Every valid solution corresponds to exactly one choice of `x`, `k`, and `y`, so enumerating all such triples covers the entire solution space without duplication or omission.

## Python Solution

```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n = int(input())
        a = list(map(int, input().split()))

        pos = [[] for _ in range(27)]
        for i, v in enumerate(a):
            pos[v].append(i)

        ans = 1

        # case: single block (x == y or y empty)
        freq = [0] * 27
        for v in a:
            freq[v] += 1
            ans = max(ans, freq[v])

        # try outer value x
        for x in range(1, 27):
            m = len(pos[x])
            if m == 0:
                continue

            for k in range(m // 2 + 1):
                l = pos[x][k - 1] + 1 if k > 0 else 0
                r = pos[x][m - k] - 1 if k < m else -1

                if l > r:
                    ans = max(ans, 2 * k)
                    continue

                cnt = [0] * 27
                best_mid = 0

                for i in range(l, r + 1):
                    cnt[a[i]] += 1
                    if cnt[a[i]] > best_mid:
                        best_mid = cnt[a[i]]

                ans = max(ans, 2 * k + best_mid)

        print(ans)

if __name__ == "__main__":
    solve()
```

The implementation first computes global frequencies to handle the degenerate case where the best answer is just a constant subsequence.

For each candidate outer value `x`, we iterate over possible symmetric counts `k`. The indices `l` and `r` carefully define the interior segment after removing `k` occurrences from both ends. The boundary logic is crucial: `pos[x][k - 1] + 1` ensures we exclude the left block, and `pos[x][m - k] - 1` excludes the right block.

The middle scan uses a fresh frequency array per configuration. Although this seems repetitive, the total complexity remains acceptable due to small constraints.

## Edge Cases

In a single-element array like `[5]`, every configuration reduces to either empty outer blocks or a single middle element. The algorithm naturally handles this because `k = 0` leads to the full range and the best frequency is 1.

In an all-equal array like `[7,7,7,7,7]`, every choice of `x` and `k` produces a valid structure, and the middle frequency equals the remaining count. The optimal value becomes the entire array length, which is achieved when `x = y = 7`.

In sparse arrays like `[1,2,3,4]`, no value repeats enough to form meaningful outer blocks, so all `k > 0` cases fail or give small contributions. The global frequency initialization ensures the answer correctly falls back to 1.
