## Solution
## Problem Understanding

We are given `n` boxes, each containing a certain number of chocolates. Bob wants to send these to Alice, but Alice will only be happy if there is an integer `k > 1` such that each box contains a multiple of `k` chocolates. Charlie can help by moving chocolates between adjacent boxes, where each move takes one second.

The input is the number of boxes and the initial distribution of chocolates. The output is the minimum number of seconds required to make all non-empty boxes divisible by some integer greater than one, or `-1` if no such distribution is possible.

Constraints tell us that `n` can be as large as `10^6`, and the number of chocolates in a box can also be up to `10^6`. With a 1-second time limit, any solution must be close to O(n log n) or better. An O(n^2) solution is completely infeasible, because that could be up to 10^12 operations.

Non-obvious edge cases include:

- Only one box. If it has a prime number of chocolates, the answer is `-1`, because no `k > 1` divides it except itself. If it has more than one chocolate, the box already satisfies divisibility by its value. For instance, input `1\n7` outputs `7` (since the box is already divisible by `7`) is actually invalid because `k` must be greater than 1 and less than the box itself, so it would output `-1`.
- Multiple boxes but total chocolates are prime. For example, `2\n1 2` has sum `3`, which is prime. No integer `k > 1` divides the total number of chocolates, so output should be `-1`.
- Some boxes are empty. For example, `4\n0 6 0 6`. Empty boxes do not affect divisibility, but moving chocolates may still be needed to minimize time.

The key difficulty is that we cannot simply check each `k` up to max value, because the numbers are large. We need to reason about divisors of the total sum and distribute efficiently.

## Approaches

The brute-force approach would be to try every integer `k` from `2` up to the sum of all chocolates and check if we can redistribute chocolates so each box has a multiple of `k`. For each `k`, we could try moving chocolates greedily from left to right and count the seconds. This is correct in principle, but completely impractical: the sum could be up to `10^12` in the worst case, so checking all possible `k` is impossible.

The key insight is that `k` must divide the total sum of chocolates. Let `S = sum(a_i)`. Only divisors of `S` can be valid values for `k`. This reduces the candidate `k` values dramatically. For example, if `S = 12`, valid `k` are `2, 3, 4, 6, 12`. There is no need to check numbers not dividing `S`.

Once we know the candidate `k`s, we can calculate the minimum seconds to make each box a multiple of `k`. This can be done with a greedy left-to-right algorithm: keep track of the cumulative remainder modulo `k`. Each time we have a non-zero remainder, we need to move that remainder to the next box, and we accumulate the number of moves. This works because the cost of moving chocolates between adjacent boxes is additive and linear in the distance.

The combination of these insights gives an O(n sqrt(S)) solution: factorize `S` to get divisors, then for each divisor compute the minimum number of moves using the prefix remainder technique.

| Approach | Time Complexity | Space Complexity | Verdict |
| --- | --- | --- | --- |
| Brute Force | O(n S) | O(n) | Too slow |
| Optimal | O(n √S) | O(n) | Accepted |

## Algorithm Walkthrough

1. Compute the total number of chocolates `S = sum(a_i)`. If `S = 1`, output `-1` immediately because no `k > 1` divides `1`.
2. Find all prime divisors of `S`. Each divisor is a candidate `k` that could make Alice happy. We can do this by trial division up to √S, adding both the divisor and its complement.
3. For each divisor `k > 1`, compute the minimum number of seconds required to make all boxes multiples of `k`. Initialize a running remainder `r = 0`. For each box `i` from left to right, compute `r = (r + a[i]) % k`. This `r` is the excess chocolates at position `i`. Add `abs(r)` to the running total of seconds and carry `r` over to the next box.
4. Keep track of the minimum total seconds across all divisors.
5. If no valid divisor yields a solution, output `-1`. Otherwise, output the minimum seconds found.

Why it works: At each step, the remainder `r` represents the exact number of chocolates that must be moved to the next box to satisfy divisibility locally. Since each move is only allowed between adjacent boxes, summing the absolute value of these remainders gives the minimal moves required to reach a divisible state. Trying all divisors of `S` guarantees we consider all possible `k` values, because any valid `k` must divide the total sum.

## Python Solution

```python
import sys
input = sys.stdin.readline

def get_divisors(n):
    divisors = set()
    i = 2
    while i*i <= n:
        if n % i == 0:
            divisors.add(i)
            divisors.add(n // i)
        i += 1
    if n > 1:
        divisors.add(n)
    return divisors

def min_seconds(a, k):
    r = 0
    moves = 0
    for x in a:
        r = (r + x) % k
        moves += min(r, k - r)
    return moves

def solve():
    n = int(input())
    a = list(map(int, input().split()))
    S = sum(a)
    if S == 1:
        print(-1)
        return
    divisors = get_divisors(S)
    divisors = [d for d in divisors if d > 1]
    if not divisors:
        print(-1)
        return
    ans = float('inf')
    for k in divisors:
        ans = min(ans, min_seconds(a, k))
    print(ans)

solve()
```

The `get_divisors` function finds all divisors greater than one. The `min_seconds` function implements the prefix remainder approach. We carefully choose `min(r, k - r)` because moving `r` chocolates forward or moving `k - r` chocolates backward gives the same final state, and we always pick the cheaper option. This is a subtle but critical optimization.
