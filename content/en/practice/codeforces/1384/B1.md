## Solution

## Problem Understanding

Koa needs to move across a line of sea cells from position 0 (shore) to position n+1 (island). Each intermediate position i has a fixed base depth d[i], but the water level is not static. Instead, the tide adds a time-dependent offset that cycles every 2k seconds: it increases linearly from 0 up to k, then decreases back to 1.

At any integer second t, if Koa is standing at position x, the effective depth at that position is d[x] plus the tide value at time t. Koa can either stay in place or move one step to the right each second, and the time always advances. She can only stand safely on a sea cell if the effective depth does not exceed a threshold l. While moving between cells, she is safe during the transition, so only the arrival time matters.

The task is to determine whether there exists a sequence of waits and moves that allows her to reach the island.

The constraints are small, with n, k up to 100 and at most 100 test cases, so an O(n^3) or O(n^2 k) approach is acceptable. However, the interaction between position and time makes greedy reasoning dangerous because the tide depends only on time, not position, so the same cell can be safe at one moment and unsafe shortly after.

A common subtle failure case arises when a cell is only occasionally safe. For example, if a position becomes safe only during the downward phase of the tide, naive strategies that move as soon as a cell is ever safe will fail because arrival time may not align with a safe phase.

Another tricky case is when staying too long in earlier cells is necessary to synchronize with future safe windows. A greedy “move ASAP” strategy breaks because it ignores time alignment constraints.

## Approaches

A brute-force approach models the problem as a state graph over pairs (position, time mod 2k). From each state, Koa can either stay or move forward if the destination cell is safe at the next time. This guarantees correctness because it explores all valid schedules, but the state space is n × 2k, and each state has up to two transitions, making it about 40,000 states per test in the worst case. This is acceptable here but still unnecessary if we observe structure.

The key insight is that we do not need to consider arbitrary time evolution; only the earliest reachable time at each position matters. If we arrive at position x at some time t, then any later arrival time is strictly worse because it only shifts the tide phase forward, possibly reducing feasibility. Therefore, we track the minimum time at which each position can be reached.

Once we compute the earliest time we can be at position i, we can greedily simulate waiting at i until it becomes possible to move to i+1 at some valid time. The periodic nature of the tide allows us to test feasibility for each departure time in O(k), so overall we can transition forward efficiently.

The problem reduces to iterating over positions and, for each, finding the earliest time we can depart safely to the next position.

## Algorithm Walkthrough

We define a function that checks whether at a given position i and time t, Koa is safe: d[i] + p[t mod 2k] ≤ l.

We maintain an array best[i], meaning the earliest time we can safely arrive at position i. Initially, best[0] = 0 because Koa starts at the shore at time 0.

We proceed position by position, trying to compute best[i+1] from best[i].

1. Start with best[0] = 0, since Koa begins at the shore at time zero without constraints.
2. For each position i from 0 to n, attempt to compute the earliest valid arrival time at i+1.
3. For a candidate arrival time t at position i, we consider all possible times t' ≥ t when Koa could depart from i. At time t', Koa can move to i+1, so we must ensure position i is safe at t' and position i+1 is safe at t'+1.
4. To find the earliest feasible t', we scan forward in time starting from best[i]. For each time step, we check:

whether d[i] + p[t mod 2k] ≤ l holds.

This is required because Koa must remain alive while waiting.
5. When we find a time t' such that position i is safe at t', we test whether she can move to i+1, meaning:

d[i+1] + p[(t'+1) mod 2k] ≤ l.
6. If both conditions hold, we set best[i+1] = t'+1 and move on to the next position.
7. If no such t' exists within a reasonable bounded range (at most 2k phases are needed because the pattern repeats every 2k), we conclude that movement is impossible.

### Why it works

The crucial invariant is that for each position i, best[i] is the minimum time at which Koa can safely stand there. Any later arrival time can only be worse or equivalent because the tide state is purely periodic in time, so delaying does not introduce new favorable phases that earlier times did not already expose within one cycle. Therefore, checking only forward from the earliest arrival time within one full cycle of length 2k is sufficient to determine reachability of the next position.

This reduces the infinite-time process into a bounded cyclic feasibility check per edge.

## Python Solution

```python
import sys
input = sys.stdin.readline

def solve():
    t = int(input())
    for _ in range(t):
        n, k, l = map(int, input().split())
        d = [0] + list(map(int, input().split()))
        
        # precompute tide pattern
        p = list(range(k)) + list(range(k - 2, 0, -1)) if k > 1 else [0, 1]
        if k == 1:
            p = [0, 1]
        period = len(p)
        
        INF = 10**18
        best = [INF] * (n + 2)
        best[0] = 0
        
        def safe(i, t):
            return d[i] + p[t % period] <= l
        
        ok = True
        
        for i in range(n + 1):
            if best[i] == INF:
                ok = False
                break
            
            tcur = best[i]
            found = False
            
            # search within one cycle window
            for dt in range(period * 2):
                t = tcur + dt
                if not safe(i, t):
                    continue
                if i + 1 <= n:
                    if d[i + 1] + p[(t + 1) % period] <= l:
                        best[i + 1] = min(best[i + 1], t + 1)
                        found = True
                        break
                else:
                    best[i + 1] = t + 1
                    found = True
                    break
            
            if not found and i != n + 1:
                ok = False
                break
        
        print("Yes" if best[n + 1] != INF else "No")

if __name__ == "__main__":
    solve()
```

The implementation explicitly builds the tide cycle and then propagates the earliest arrival time forward position by position. The inner loop checks a bounded window of time starting from the current best arrival, ensuring that we catch at least one full repetition of the tide phase. The `safe` function encapsulates the constraint check at a fixed time.

A subtle point is that we always check safety at time t for position i and at time t+1 for position i+1. This matches the rule that Koa must be safe at integer times and only transitions between seconds matter.

## Edge Cases

A key edge case is when a cell is only occasionally safe. The algorithm handles this by scanning forward across a full tide cycle, ensuring that even if the first safe moment is missed, the next repetition is found. This prevents premature rejection.

Another case is when Koa must wait at an earlier position to align future phases. Because best[i] is the earliest arrival time, the algorithm naturally considers waiting as part of the scan rather than forcing immediate movement.

A final edge case is when k = 1, where the tide alternates every second. The construction of p explicitly handles this by producing a correct minimal cycle [0, 1], ensuring phase correctness even in the smallest periodic setting.
