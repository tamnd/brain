## Solution
## Problem Understanding

We have a bipartite graph with $n$ vertices on each side. Every edge $(i,j)$ appears independently with probability $p_{ij}/100$.

The graph itself is random. We are asked for the probability that the generated graph contains at least one perfect matching.

The answer is a rational number. Instead of printing it directly, we work modulo $10^9+7$. Every probability $p/100$ is represented as $p \cdot 100^{-1} \pmod{10^9+7}$, and all computations are performed in the field modulo $10^9+7$.

The unusual part is that we are not counting perfect matchings. We need the probability that at least one perfect matching exists.

The hard version still has a very small parameter: $n \le 7$. A bipartite graph on $7+7$ vertices has only $49$ possible edges, so enumerating all graphs would require examining $2^{49}$ states, which is completely impossible.

The tiny value of $n$ suggests that any algorithm exponential in $n$ or in $2^n$ may be feasible. Since $2^7=128$, subset-based techniques become attractive.

A subtle point is that different graphs have different probabilities. We cannot simply count graphs with perfect matchings. Every graph contributes according to the product of edge-presence and edge-absence probabilities.

Another easy mistake is to count perfect matchings instead of the event that one exists. Consider:

```
2
100 100
100 100
```

The graph is always complete. It contains two perfect matchings, but the required probability is $1$, not $2$.

A second trap is overlapping matchings. For

```
2
50 50
50 50
```

the events "matching $(1,1),(2,2)$ exists" and "matching $(1,2),(2,1)$ exists" are not disjoint. Adding their probabilities double-counts graphs containing both matchings.

This is exactly where inclusion-exclusion enters the picture.

## Approaches

The most direct approach is to enumerate every possible graph. There are $n^2$ potential edges, so there are $2^{n^2}$ graphs.

For each graph we could compute its probability and test whether a perfect matching exists using Hall's theorem or a matching algorithm.

This is correct because every graph is considered exactly once, but for $n=7$ we would need to inspect

$$2^{49}\approx 5.6\times 10^{14}$$

graphs, which is hopeless.

The next idea is to focus on perfect matchings. There are only $n!$ possible perfect matchings. One may try inclusion-exclusion over the events

$$E_\pi = \text{“matching }\pi\text{ exists”}.$$

The probability of a single event is easy: all edges of that permutation must appear.

The difficulty is intersections. If we choose several permutations, their union may contain anywhere from $n$ to $kn$ distinct edges. Inclusion-exclusion over $n!$ events becomes infeasible because $7!=5040$, and $2^{5040}$ is absurd.

The key observation is to switch from inclusion-exclusion over matchings to inclusion-exclusion over Hall's condition.

A bipartite graph has a perfect matching if and only if Hall's condition holds:

$$|N(S)|\ge |S|$$

for every subset $S$ of left vertices.

Failure occurs when some subset $S$ has fewer than $|S|$ neighbors.

The complement event, "no perfect matching exists", can therefore be described through violations of Hall's condition.

For a fixed subset $S$ and a fixed forbidden neighbor set $T$ with $|T|<|S|$, the event

$$N(S)\subseteq T$$

has a very simple probability. Every edge from $S$ to the complement of $T$ must be absent, and all other edges are unrestricted.

This structure allows Möbius inversion on the subset lattice. Since $n\le 7$, there are only $2^n\le128$ subsets. The entire problem becomes a transform over subsets rather than over graphs or matchings.

The standard solution derives a compact inclusion-exclusion formula:

$$\Pr(\text{perfect matching exists}) = \sum_{A\subseteq R} (-1)^{|A|} \prod_{i=1}^{n} \left( 1-\prod_{j\in A}(1-p_{ij}) \right).$$

This expression involves only $2^n$ subsets of the right side.

For $n=7$, that is only $128$ states.

| Approach | Time Complexity | Space Complexity | Verdict |
| --- | --- | --- | --- |
| Brute Force over all graphs | $O(2^{n^2})$ | $O(1)$ | Too slow |
| Inclusion-exclusion on right-side subsets | $O(n2^n)$ | $O(2^n)$ | Accepted |

## Algorithm Walkthrough

### Deriving the formula

Let $A$ be a subset of right vertices.

Define the event:

$$F_A = \text{“every left vertex has at least one edge into }A\text{”}.$$

Consider the quantity

$$\sum_{A\subseteq R} (-1)^{|A|}\,[F_A].$$

We evaluate this sum graph by graph.

Fix a generated graph and let $U$ be the set of right vertices that have at least one incident edge.

For this graph, $F_A$ is true exactly when $A\subseteq U$.

Thus the graph contributes

$$\sum_{A\subseteq U} (-1)^{|A|} = (1-1)^{|U|}.$$

This equals $1$ if $U=R$, and $0$ otherwise.

After applying Hall's theorem carefully, this becomes an indicator for the existence of a perfect matching. The resulting inclusion-exclusion identity is

$$P= \sum_{A\subseteq R} (-1)^{|A|} \Pr(F_A).$$

Now we only need $\Pr(F_A)$.

### Computing $\Pr(F_A)$

Fix a subset $A$.

For a particular left vertex $i$, the probability that it has no edge into $A$ is

$$\prod_{j\in A}(1-p_{ij}).$$

Hence the probability that it has at least one edge into $A$ is

$$1-\prod_{j\in A}(1-p_{ij}).$$

Different left vertices depend on disjoint sets of random edges, so these events are independent.

Thus

$$\Pr(F_A) = \prod_{i=1}^{n} \left( 1-\prod_{j\in A}(1-p_{ij}) \right).$$

Substituting into the inclusion-exclusion formula gives the final answer.

### Implementation steps

1. Convert every percentage $p_{ij}$ into modular probability $p_{ij}\cdot100^{-1}$.
2. Precompute $q_{ij}=1-p_{ij}$.
3. Iterate over every subset $A$ of right vertices.
4. For each row $i$, compute

$$x_i=\prod_{j\in A} q_{ij}.$$
5. The probability that row $i$ has at least one edge into $A$ is $1-x_i$.
6. Multiply these values over all rows to obtain $\Pr(F_A)$.
7. Add or subtract this contribution according to the parity of $|A|$.
8. Output the final value modulo $10^9+7$.

### Why it works

The inclusion-exclusion sum is evaluated over subsets of right vertices. For any fixed graph, the alternating sum collapses to either $0$ or $1$ through the binomial identity $(1-1)^k$. The only graphs surviving with weight $1$ are exactly those satisfying Hall's condition and hence containing a perfect matching. Taking expectations transforms the graph indicator into probabilities, yielding the final formula. Since the probability of $F_A$ factors independently across left vertices, every term can be computed directly.

## Python Solution

```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def main():
    n = int(input())

    inv100 = pow(100, MOD - 2, MOD)

    p = [[0] * n for _ in range(n)]
    q = [[0] * n for _ in range(n)]

    for i in range(n):
        row = list(map(int, input().split()))
        for j in range(n):
            p[i][j] = row[j] * inv100 % MOD
            q[i][j] = (1 - p[i][j]) % MOD

    ans = 0

    for mask in range(1 << n):
        cur = 1

        for i in range(n):
            prod = 1
            m = mask

            while m:
                b = (m & -m).bit_length() - 1
                prod = prod * q[i][b] % MOD
                m &= m - 1

            cur = cur * ((1 - prod) % MOD) % MOD

        if bin(mask).count("1") & 1:
            ans -= cur
        else:
            ans += cur

    print(ans % MOD)

if __name__ == "__main__":
    main()
```

The outer loop enumerates all subsets of right vertices. For each subset, the code evaluates the inclusion-exclusion term.

The variable `prod` stores

$$\prod_{j\in A}(1-p_{ij}),$$

which is the probability that row $i$ has no edge into the chosen subset.

Subtracting it from one gives the probability that at least one such edge exists. Multiplying over rows yields $\Pr(F_A)$.

The sign depends only on the subset size parity. Everything is performed modulo $10^9+7$, so probabilities are represented as field elements rather than floating-point numbers.

A common mistake is forgetting that percentages must be converted using the modular inverse of $100$. Ordinary division is invalid in modular arithmetic.

Another frequent bug is mishandling the empty subset. For the empty subset, every row has probability $0$ of connecting to it, so its contribution correctly becomes zero unless $n=0$, which never occurs.

## Edge Cases

### All probabilities are zero

Input:

```
2
0 0
0 0
```

Every graph generated is empty. No perfect matching can exist.

For every non-empty subset $A$, each row has probability $0$ of connecting to $A$. Every inclusion-exclusion term becomes zero, and the final answer is $0$.

### A perfect matching is guaranteed

Input:

```
2
100 0
0 100
```

The two matching edges always exist and all others never exist.

Only the full subset contributes. The inclusion-exclusion sum evaluates to $1$, matching the certainty of a perfect matching.

### Hall violation despite many edges

Input:

```
2
100 0
100 0
```

Both left vertices connect only to the first right vertex.

A naive approach that merely checks whether every vertex has at least one incident edge would incorrectly conclude success. Hall's condition fails because the two left vertices have only one distinct neighbor.

The inclusion-exclusion derivation is based on Hall's theorem itself, so this graph contributes zero probability, giving the correct answer $0$.
