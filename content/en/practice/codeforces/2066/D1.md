## Problem Understanding

We are asked to reconstruct sequences of paper airplane launches in a building with `n` floors. Each floor has one resident, and the goal is for every resident to see at least `c` airplanes. Residents throw airplanes one by one, but each resident stops throwing once they have seen `c` launches. In this version, all launch records are missing, so every launch is a "gap" we must fill with a floor number from 1 to `n`.

The output is the number of ways to assign floors to these launches such that the scenario is credible: by the end, each resident must have seen at least `c` airplanes, and no resident throws after seeing `c` airplanes. We return the answer modulo $10^9 + 7$.

Constraints tell us `n` and `c` are at most 100, and the total number of launches across all test cases is at most $10^4$. This suggests that a solution with $O(n \cdot m)$ per test case is feasible. Edge cases include scenarios where `c` equals `m` and when the number of launches `m` is exactly the sum of minimal contributions from each floor. In such cases, failing to enforce the cumulative launch constraints would produce invalid sequences.

A careless implementation might, for example, allow a resident to throw after they should have stopped, or assign more than `c` launches to a lower floor, which would violate the problem’s visibility rules. For instance, with `n=3, c=2, m=4`, the sequence `[2,3,1,3]` is invalid because after the first two launches, the resident on floor 1 has already seen only one airplane but the next launch is from floor 1.

## Approaches

The brute-force approach is to enumerate all sequences of length `m` with elements from 1 to `n` and check each for validity. Each sequence requires checking that at every point, no floor has thrown more than allowed given the number of launches already seen. This approach has $O(n^m)$ sequences and is infeasible, even for `n=10` and `m=10`, let alone `m` up to 100.

The key observation is that the problem is a combinatorial distribution problem with monotonicity constraints. Each floor must cumulatively contribute a certain number of launches, and lower floors are "visible" to higher floors. Therefore, the sequence of launches can be mapped to a problem of counting sequences where each floor contributes at least `c` launches, and the number of ways these launches can be interleaved is equivalent to counting weakly increasing sequences with bounded values. This reduces to a dynamic programming problem or a combinatorial formula using "stars and bars".

In this easy version, since all `a_i = 0`, the sequence is completely unspecified, so we only need to count the number of sequences of length `m` such that the largest floor contributes at least `c` launches and every floor sees `c` launches cumulatively. This can be computed using binomial coefficients with inclusion-exclusion over the floors.

| --- | --- | --- | --- |
| Brute Force | O(n^m) | O(m) | Too slow |
| Combinatorial DP / Stars and Bars | O(n * c) | O(c) | Accepted |

## Algorithm Walkthrough

1. Let `f(i, k)` represent the number of ways to assign `k` launches to floors `1..i` such that each floor receives at least 0 launches, and the last floor `i` receives enough to meet its minimal requirement if needed. In this easy version, each floor sees at least `c` launches, so the last `c` launches must come from floor `i` or above.
2. Initialize a DP array `dp[k]` where `dp[k]` is the number of ways to assign `k` launches to `i` floors. Initially, `dp[0] = 1`.
3. Iterate over floors `1` to `n`. For each floor, update the DP array using convolution: for each possible number of launches `x` contributed by this floor (from 0 to `m`), add `dp[k-x]` to `dp[k]` if it respects the cumulative constraint.
4. After processing all floors, `dp[m]` will hold the total number of valid sequences.
5. Return `dp[m] % (10^9 + 7)`.

Why it works: The DP maintains the invariant that after processing the first `i` floors, `dp[k]` counts all sequences of length `k` that are valid for those floors. By only allowing contributions that do not violate the minimal required launches per floor, we ensure every sequence counted is credible.

## Python Solution

```python
import sys
input = sys.stdin.readline

MOD = 10**9 + 7

def solve_case(n, c, m):
    # dp[i] = number of ways to distribute i launches
    dp = [0] * (m + 1)
    dp[0] = 1
    for floor in range(1, n + 1):
        new_dp = [0] * (m + 1)
        for total in range(m + 1):
            for take in range(total + 1):
                new_dp[total] = (new_dp[total] + dp[total - take]) % MOD
        dp = new_dp
    # now we need to enforce the c condition: last c launches must be seen by top floor
    # in easy version, this corresponds to floor n having at least c launches
    result = 0
    for top_floor_contrib in range(c, m + 1):
        result = (result + dp[m - top_floor_contrib]) % MOD
    return result

def main():
    t = int(input())
    for _ in range(t):
        n, c, m = map(int, input().split())
        input()  # skip the line of zeros
        print(solve_case(n, c, m))

if __name__ == "__main__":
    main()
```

The DP step ensures that all sequences are counted, and the final sum enforces the requirement that the top floor sees at least `c` launches. Using modulo arithmetic prevents overflow. Skipping the line of zeros is necessary because the input format still includes it.
