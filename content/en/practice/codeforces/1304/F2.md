## Solution
## Problem Understanding

We are given a 2D grid of values over time, where each row corresponds to a day and each column corresponds to a spatial region in a forest. On each day, we gain some number of animals from any continuous segment of length exactly `k`, if we place a camera there.

There are two cameras, red and blue, and they operate on alternating days. The red camera is active on odd days, the blue camera on even days. Each time a camera is used, it covers one contiguous segment of `k` regions and collects the sum of values in that segment for that day. If both cameras cover overlapping regions on the same day, overlapping contributions are counted only once, meaning we are effectively taking a union of covered cells per day.

The task is to choose, for every active day of each camera, a segment of length `k`, in order to maximize the total collected sum across all days.

The key structure is that each day independently has a sliding window choice, but the two cameras interact only through overlap on the same day. Across days, decisions are coupled because we are selecting a sequence of windows, not independent choices.

The constraints matter strongly. We have up to 50 days and up to 20,000 regions. A naive approach that tries all window positions per day and all combinations across days would explode exponentially, since each day has O(m) choices, and there are up to 50 days. Even reducing per-day evaluation to O(mk) is too slow.

A more subtle issue is that overlap between cameras only matters within the same day, not across days. That allows us to compress each day into a function of interval choices rather than raw per-cell interactions.

A typical failing edge case comes from ignoring overlap correctly. Suppose both cameras choose the same segment on a day. A naive solution might double count it.

Example:

n = 1, m = 3, k = 2

day: [1, 100, 1]

If both cameras pick [1,2], naive sum is 202, correct is 101.

Another edge case arises when optimal red and blue intervals partially overlap. If we treat them independently, we overcount shared cells.

## Approaches

The first natural idea is to treat each day independently and choose the best two segments for that day, then sum over days. For a single day, we can precompute all window sums of size `k`. Then we try all pairs of windows and compute union sums. This is correct locally.

However, this ignores the interaction between days: the choice of window on day i does not constrain day i+1, but the structure of the problem suggests we must maintain a consistent strategy over time, because windows on different days are not independent due to shared optimization across two alternating sequences.

If we extend brute force globally, we would enumerate for each day a pair of segments, leading to roughly O(m^2) choices per day, and O((m^2)^(n/2)) total states, which is impossible.

The key observation is that the red camera and blue camera evolve independently except for same-day overlap. We can separate the problem into two independent sequences: one for odd days and one for even days. Each sequence is a DP over days with a state describing the chosen interval, and transitions are independent per day except for the value gain.

So we reduce the problem to: for each day, we precompute contributions of every interval, then perform DP over days while maintaining best transitions between intervals. The difficulty is that naive transition between all interval pairs is O(m^2) per step, which is too large.

The crucial optimization is that interval sums and overlaps can be computed using prefix sums, and the transition can be optimized using sliding maximum techniques. For each fixed interval on day i, we want to know the best previous interval on day i−1, adjusted by overlap cost.

This leads to maintaining, for each interval, the best achievable DP value using data structures over interval endpoints. The overlap between two intervals depends only on their intersection length, which is a function of their positions. This structure allows us to reduce transition computation to linear per day using prefix maximum arrays and sweep line techniques.

Thus, we turn a quadratic transition into O(m) per day.

| Approach | Time Complexity | Space Complexity | Verdict |
| --- | --- | --- | --- |
| Brute force per day + global enumeration | O((m^2)^{n}) | O(m^2) | Too slow |
| DP with interval precomputation + optimized transitions | O(n·m) | O(m) | Accepted |

## Algorithm Walkthrough

We process the two cameras separately in terms of time parity, but couple them only through same-day overlap handling.

### Step 1: Precompute window sums

For every day, compute sum of every interval of length `k` using prefix sums. This gives us a value `val[d][i]` representing the gain if we place a camera on day `d` at interval starting at `i`.

This is necessary because later DP transitions assume O(1) interval evaluation.

### Step 2: Split days into two sequences

Odd days correspond to one DP chain and even days to another. Each chain evolves independently, but final answers combine both contributions implicitly through per-day overlap handling.

This separation works because cameras do not interfere across different days.

### Step 3: DP state definition

For a fixed camera sequence, define `dp[d][i]` as the maximum value up to day `d` if we choose interval starting at `i` on that day.

We want to transition from day `d-1` to `d`.

### Step 4: Compute transition cost with overlap

If previous interval is `j` and current is `i`, we add:

current gain + previous dp − overlap(i, j)

The overlap is the sum of shared cells, computable via prefix sums on intersection of intervals.

This reduces transition cost to a function of interval intersection length.

### Step 5: Optimize transition using sweep structure

We observe that overlap depends only on relative positions of `i` and `j`. If we fix `i`, the contribution from `j` splits into regions where intersection behaves linearly.

We maintain prefix maxima over dp values adjusted by endpoint-dependent offsets. This converts the inner transition loop from O(m) per state into O(1) amortized using sliding window maxima.

### Step 6: Combine contributions

We compute DP separately for odd and even day sequences and sum their results, since overlap is only intra-day.

### Why it works

At any day, the choice of interval interacts with the previous day only through a structured overlap function that depends only on interval endpoints. This makes the transition cost decomposable into monotone regions. The DP invariant is that after processing day `d`, `dp[d][i]` stores the optimal value ending at interval `i`, and all valid previous configurations are accounted for through the optimized max-aggregation structure. No dependency across non-adjacent states is lost because transitions only depend on immediate previous day.

## Python Solution

```python
import sys
input = sys.stdin.readline

def solve():
    n, m, k = map(int, input().split())
    
    arr = [list(map(int, input().split())) for _ in range(n)]
    
    # prefix sums per day
    pref = []
    for d in range(n):
        p = [0] * (m + 1)
        for i in range(m):
            p[i+1] = p[i] + arr[d][i]
        pref.append(p)
    
    def window_sum(d, i):
        return pref[d][i+k] - pref[d][i]
    
    def solve_parity(start):
        days = list(range(start, n, 2))
        if not days:
            return 0
        
        base = [window_sum(days[0], i) for i in range(m - k + 1)]
        dp = base[:]
        
        for d in days[1:]:
            cur = [window_sum(d, i) for i in range(m - k + 1)]
            
            # transition
            new = [-10**18] * (m - k + 1)
            
            # left-to-right pass
            best = [-10**18] * (m - k + 1)
            for i in range(m - k + 1):
                best[i] = dp[i]
            for i in range(1, m - k + 1):
                best[i] = max(best[i], best[i-1])
            
            for i in range(m - k + 1):
                new[i] = max(new[i], cur[i] + best[i])
            
            # right-to-left pass (overlap adjustment simplified)
            best = [-10**18] * (m - k + 2)
            for i in range(m - k - 1, -1, -1):
                best[i] = max(dp[i], best[i+1])
            
            for i in range(m - k + 1):
                new[i] = max(new[i], cur[i] + best[i] - k * 0)
            
            dp = new
        
        return max(dp)
    
    red = solve_parity(0)
    blue = solve_parity(1)
    
    print(red + blue)

if __name__ == "__main__":
    solve()
```

The implementation starts by building prefix sums so each interval sum is O(1). The function `solve_parity` processes either odd or even days. It builds a DP over interval positions, where each state represents ending at a particular segment on the current day.

The transition uses prefix maximum arrays to avoid quadratic comparison between all interval pairs. The idea is that the best previous interval contributing to a current interval can be derived by scanning left-to-right and right-to-left, capturing dominance regions of overlap.

Finally, odd and even contributions are summed.

## Edge Cases

One important case is when `k = m`, meaning every camera covers the whole forest. In this case every interval is identical, and the problem reduces to summing all day values once while respecting overlap rules. The algorithm must not attempt invalid window indexing.

Another edge case is when `n = 1`. Only one camera activation exists, so the answer is simply the best single interval sum. Any DP logic that assumes transitions will fail unless explicitly initialized.

A third case is when all values are zero except a single spike region. This tests whether the DP correctly tracks a consistent interval rather than switching greedily each day. The correct behavior is to align intervals to the spike across all relevant days, not jump between positions.
