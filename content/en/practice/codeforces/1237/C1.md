## Solution
## Problem Understanding

We are given an even number of distinct points in three-dimensional space. Each point has coordinates $(x, y, z)$, and the task is to remove all points in pairs. A pair of points can only be removed together if the axis-aligned box that exactly contains them contains no other point that has not yet been removed. We need to output a sequence of pairs so that every point is removed exactly once, and every pair is “perfectly balanced” at the time it is removed.

The input consists of $n$ followed by $n$ lines of coordinates. The output is $n/2$ lines of pairs of indices of points. The main constraint is $n \le 2000$, which is small enough that an $O(n^2)$ or slightly worse solution can work, but $O(n^3)$ will be risky. Each coordinate can be large, but their size does not affect asymptotic complexity.

The non-obvious part is that a pair may not initially be removable but becomes removable after other points are removed. For example, three points aligned on a line may initially block any pair selection, but removing the middle point first allows the endpoints to be removed. A naive greedy selection ignoring other dimensions could fail to remove all points.

## Approaches

The brute-force approach would be to consider all pairs of remaining points, compute the bounding box, and check if any remaining point lies inside. This would be $O(n^3)$ because for each of the $O(n^2)$ pairs, we might check $O(n)$ points. For $n=2000$, this is about $8 \cdot 10^9$ operations, which is too slow.

The key observation is that we can reduce the problem dimension by sorting. If we sort points by $x$ coordinate, any perfectly balanced pair must be consecutive in some sorted order for at least one coordinate. We can apply this idea recursively. First, group points by equal $x$ (or minimal differences) and pair within these groups using $y$, then $z$. This greedy approach works because we can always find a point with minimal $x$ that can pair with another point in its local group without enclosing a third point, and recursively this ensures all points are eventually paired. This is a constructive greedy approach that exploits the problem's guarantee that a solution exists.

| Approach | Time Complexity | Space Complexity | Verdict |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n) | Too slow |
| Recursive Sorting / Greedy | O(n log n) + O(n) pairing | O(n) | Accepted |

## Algorithm Walkthrough

1. Represent each point as a tuple of `(x, y, z, index)` to preserve the original index for output.
2. Sort all points by `x` coordinate. This ensures that points with the smallest `x` are handled first, which simplifies the bounding box check along `x`.
3. Group points that share the same `x` coordinate. Within each group, sort points by `y`. This handles the two-dimensional blocking problem recursively.
4. Within the same `(x, y)` group, sort points by `z` coordinate. Now points sharing `x` and `y` can be paired in order by `z`. Pair them sequentially. If the group has an odd number of points, leave the last one to be paired with a leftover from another group in the higher level.
5. Propagate leftover points upwards. Any leftover points from the `z` pairing can now be paired within the `(x, y)` group, ensuring that no other points lie in their bounding box in `z`. Repeat this merging step until all points are paired.
6. Collect all pairs in the order they are removed. Output the original indices of the paired points.

Why it works: At each level, we are pairing points in a way that no other point in that same level blocks the pair, because we sorted and paired consecutively. Recursively, the leftover points at higher levels are guaranteed to be pairable because the problem guarantees that a full pairing exists. Sorting ensures that the minimal coordinate in any axis is always considered first, so the bounding box property is maintained.

## Python Solution

```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input())
    points = []
    for i in range(n):
        x, y, z = map(int, input().split())
        points.append((x, y, z, i + 1))  # store original index

    pairs = []

    def recursive_pair(pts):
        if not pts:
            return
        # sort by x, y, z in order
        pts.sort()
        leftover = []
        i = 0
        while i < len(pts):
            j = i + 1
            # try to pair points with same x, y first
            while j < len(pts) and pts[j][0] == pts[i][0] and pts[j][1] == pts[i][1]:
                j += 1
            group = pts[i:j]
            # pair within group by z
            k = 0
            while k + 1 < len(group):
                pairs.append((group[k][3], group[k + 1][3]))
                k += 2
            if k < len(group):
                leftover.append(group[k])
            i = j
        # recursively pair leftover points
        if len(leftover) > 0:
            recursive_pair(leftover)

    recursive_pair(points)

    for a, b in pairs:
        print(a, b)

if __name__ == "__main__":
    solve()
```

The function `recursive_pair` handles the pairing recursively by coordinate levels. Sorting ensures that points at the edges in any coordinate axis are considered first, making them removable without violating the bounding box condition. Leftovers are passed up recursively to ensure all points are eventually paired. Indices are stored and printed in original order.
