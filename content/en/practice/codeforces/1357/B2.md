## Problem Understanding

The task is to implement a reversible quantum oracle that marks bit strings representing numbers divisible by 3. The input is a register of up to 8 qubits, which encodes a binary integer in little-endian order, plus one extra qubit that acts as a target bit. The operation must flip this target bit if and only if the encoded integer is divisible by 3.

The key requirement is that the transformation must be unitary, meaning it cannot “compute and overwrite” information in a classical sense. Instead, for every basis state, the system must evolve consistently so that the input state is preserved and only the output qubit is toggled based on a Boolean predicate of the input.

The constraint N ≤ 8 is crucial. It means there are at most 2⁸ = 256 possible input basis states. This small domain size makes it feasible to explicitly construct the oracle by enumerating all possible inputs instead of deriving a compact arithmetic circuit for divisibility by 3.

A naive mistake here would be to try to interpret the binary number and compute divisibility using arithmetic logic directly on qubits without ancillary structure. That quickly becomes unnecessarily complex and risks violating the restriction that only X and controlled-X gates are allowed.

Another subtle edge case is forgetting that the oracle must act correctly on superpositions. If we incorrectly “branch” on a measurement or assume classical execution, we break linearity. For example, on a superposition of |3⟩ and |4⟩, the output qubit must flip only on the |3⟩ component, not globally.

## Approaches

A brute-force classical approach would be to evaluate the function f(x) = 1 if x mod 3 = 0 for every possible x, and then encode it into a reversible circuit. Since the input space has size at most 256, we can directly construct the truth table.

To convert this into a quantum-safe operation, we need to express the function as a sequence of controlled NOT operations on the output qubit. Each operation targets exactly one input assignment. The challenge is that quantum gates are not naturally “equality checks” on bitstrings, but we can simulate equality by using multi-controlled X gates where controls are conditioned on each input bit matching a fixed pattern.

The key idea is that for each valid input x divisible by 3, we temporarily transform the control basis so that all qubits act as positive controls. If a bit in the pattern is 0, we flip that qubit before and after applying the controlled operation. This converts a “control on 0” into a standard control on 1.

This reduces the problem from designing an arithmetic circuit to enumerating a small finite set of basis states and applying conditional flips.

## Algorithm Walkthrough

We construct the oracle by iterating over all possible bit strings and activating the output flip exactly for those divisible by 3.

1. Enumerate all integers x from 0 to 2^N − 1. Each x corresponds to a basis state of the input qubits in little-endian form. This gives us a complete description of all possible quantum basis inputs.
2. For each x, check whether x mod 3 equals 0. If it is not divisible, we do nothing for this state and move on. This ensures we only build the oracle support on valid states.
3. For every x that satisfies divisibility, we construct a multi-controlled X gate targeting the output qubit. The controls enforce that the input register matches the binary representation of x exactly.
4. To enforce equality with a fixed bitstring pattern, we handle each qubit individually. If the i-th bit of x is 1, we use a positive control on that qubit. If it is 0, we temporarily flip that qubit with an X gate, apply the controlled operation, and then flip it back. This effectively turns a “control on zero” into a standard control.
5. After applying the controlled X on the output, we restore any flipped input qubits so that the overall operation remains reversible and does not disturb unrelated basis states.

### Why it works

Each basis state |x⟩ activates exactly one sequence of controls corresponding to itself. If x is divisible by 3, exactly one controlled-X operation triggers and flips the output qubit. If it is not divisible by 3, no constructed gate matches that configuration, so the output remains unchanged. Because the construction is linear over basis states, superpositions evolve correctly by linear extension.

    for mask in range(1 << n):
        if mask % 3 != 0:
            continue

        flipped = []

        for i in range(n):
            # if bit i of mask is 0, we flip qubit i to turn it into control-on-1
            if ((mask >> i) & 1) == 0:
                inputs[i].X()
                flipped.append(i)

        controls = list(inputs)
        apply_controlled_x(controls, output)

        for i in flipped:
            inputs[i].X()

The actual implementation in Q# replaces the placeholder controlled operation with a multi-controlled X gate available through controlled variants. The essential structure remains identical: prepare controls, apply conditional flip, restore state.

The subtle point is that the input qubits are never modified permanently. Every temporary X is undone immediately after use, preserving unitarity and ensuring the oracle remains reversible.

## Edge Cases

One important edge case is the all-zero state. It corresponds to integer 0, which is divisible by 3, so the output qubit must always flip when the input is |00…0⟩ regardless of superposition context. The construction handles this correctly because mask = 0 is included in the divisible set.

Another edge case is when multiple input qubits are 0. If we incorrectly forget to invert controls for zero bits, no gate would ever match such states, breaking correctness for many valid x values. The temporary X trick ensures that both 0 and 1 constraints are handled symmetrically, preserving exact pattern matching.

A final edge case is superposition states where only some branches satisfy divisibility. The linearity of controlled-X ensures that only those branches trigger the output flip, and no interference occurs between unrelated basis states.
