## Solution
## Problem Understanding

Let a subarray have length $m$. If we sort it, then every integer between the two middle elements is a median.

For odd $m$, there is only one median.

For even $m$, if the middle two sorted values are $x$ and $y$, then every integer in $[x,y]$ is a median.

We are given an array whose values lie in $[1,n]$. For every integer $v$, we want to know whether there exists a subarray of length at least $k$ whose set of medians contains $v$. If such a subarray exists, $v$ is called a submedian. We must output every submedian together with one witnessing subarray.

The total sum of $n$ over all test cases is only $3 \cdot 10^5$, so $O(n^2)$ or $O(n \sqrt n)$ per test case is impossible. The intended solution is close to $O(n \log n)$.

The first subtle point is that "median $\ge x$" is much easier to test than "median $=x$". A standard transformation turns the problem into a maximum subarray style condition.

The second subtle point is that different subarrays can have overlapping median intervals. A value can be a median even if it never appears in the array. For example, the subarray $[5,3,7,9]$ has medians $5,6,7$. Any solution that only considers values present in the array misses valid answers.

The third subtle point is that the hard version asks for all submedians. Enumerating all subarrays is impossible, so we need a structural property of the answer set.

## Approaches

The brute force approach is straightforward. Enumerate every subarray of length at least $k$, sort it, compute its median interval, and mark every value inside that interval. There are $O(n^2)$ subarrays, and each one requires at least $O(n \log n)$ work. This is far beyond the limits.

The easy version already contains the key observation. Suppose we want to know whether there exists a subarray whose median is at least $x$.

Define

$$b_i =
\begin{cases}
1 & a_i \ge x\\
-1 & a_i < x
\end{cases}$$

A subarray contains at least as many elements $\ge x$ as elements $<x$ iff the sum of its transformed values is nonnegative.

So the question becomes:

"Does there exist a subarray of length at least $k$ with nonnegative sum?"

This can be checked in linear time using prefix sums and the minimum prefix seen $k$ positions earlier. The predicate is monotone in $x$, so binary search gives the maximum submedian and one witnessing subarray. The same idea also gives the minimum submedian.

The hard version adds one crucial theorem.

If $L$ and $R$ are submedians and $L \le z \le R$, then $z$ is also a submedian. In other words, the set of submedians is always one contiguous interval.

So it is enough to find the minimum and maximum submedians. Then we only need a way to recover a witness for every value between them.

The editorial observation is that if two subarrays differ by moving only one endpoint by one position, their median intervals always intersect. A path can be built between the minimum witness subarray and the maximum witness subarray while preserving length at least $k$. Along this path, the median interval changes continuously and covers every submedian value.

While traversing the path, we maintain the multiset of values inside the current subarray. From that multiset we can query the two middle order statistics and obtain the current median interval. Every value inside that interval receives the current subarray as its witness.

| Approach | Time Complexity | Space Complexity | Verdict |
| --- | --- | --- | --- |
| Brute Force | $O(n^3 \log n)$ | $O(n)$ | Too slow |
| Optimal | $O(n \log^2 n)$ | $O(n)$ | Accepted |

## Algorithm Walkthrough

1. Implement `check(x)`.

Transform the array into $+1/-1$ depending on whether each element is at least $x$.
2. Compute prefix sums.

A subarray $[l,r]$ has nonnegative transformed sum iff

$$pref[r] - pref[l-1] \ge 0.$$
3. Scan the right endpoint.

Maintain the minimum prefix value among positions up to $r-k$.

If

$$pref[r] \ge \min pref,$$

then a valid subarray of length at least $k$ exists.
4. Binary search the largest $x$ for which `check(x)` succeeds.

This gives the maximum submedian and one witness subarray.
5. Compute the minimum submedian.

Apply the same procedure to values

$$a'_i = n + 1 - a_i.$$

If $M$ is the maximum submedian in the transformed array, then the minimum submedian in the original array equals

$$n + 1 - M.$$
6. Let $[l_1,r_1]$ be a witness for the minimum submedian and $[l_2,r_2]$ a witness for the maximum submedian.
7. Build the editorial path between them.

First move $r$ until it reaches $l_2$.

Then move $r$ until it reaches $r_2$.

Finally move $l$ until it reaches $l_2$.

Every visited subarray has length at least $k$.
8. Maintain the current subarray's multiset using a Fenwick tree over values.
9. For every visited subarray, query the lower and upper middle order statistics.

If the sorted subarray has middle values $x$ and $y$, then every integer in $[x,y]$ is a median.
10. Use a DSU "next unassigned value" structure.

Whenever the current median interval is $[x,y]$, assign the current subarray as the witness for every still-unassigned value in that range.
11. After the path ends, every value in the submedian interval has been assigned exactly once.

### Why it works

The binary search predicate is exactly the easy-version characterization of "there exists a subarray whose median is at least $x$". Hence it correctly finds the minimum and maximum submedians.

The hard-version theorem states that all submedians form one contiguous interval. Furthermore, neighboring subarrays on the constructed path have intersecting median intervals, so the median interval evolves continuously while moving from the minimum witness to the maximum witness. Consequently every integer between the two extremes appears inside the median interval of some visited subarray. Recording the first such occurrence gives a valid witness for every submedian.

## Python Solution

```python
import sys
input = sys.stdin.readline

class Fenwick:
    def __init__(self, n):
        self.n = n
        self.bit = [0] * (n + 1)

    def add(self, idx, val):
        n = self.n
        bit = self.bit
        while idx <= n:
            bit[idx] += val
            idx += idx & -idx

    def sum(self, idx):
        res = 0
        bit = self.bit
        while idx:
            res += bit[idx]
            idx -= idx & -idx
        return res

    def kth(self, k):
        pos = 0
        bitmask = 1 << (self.n.bit_length() - 1)
        while bitmask:
            nxt = pos + bitmask
            if nxt <= self.n and self.bit[nxt] < k:
                k -= self.bit[nxt]
                pos = nxt
            bitmask >>= 1
        return pos + 1

def witness_max(a, k):
    n = len(a)

    def check(x):
        pref = [0] * (n + 1)
        for i in range(n):
            pref[i + 1] = pref[i] + (1 if a[i] >= x else -1)

        mn = 0
        mn_pos = 0

        for r in range(k, n + 1):
            if pref[r - k] < mn:
                mn = pref[r - k]
                mn_pos = r - k

            if pref[r] >= mn:
                return True, mn_pos + 1, r

        return False, -1, -1

    lo, hi = 1, n
    best = 1
    ans_lr = (1, k)

    while lo <= hi:
        mid = (lo + hi) // 2
        ok, l, r = check(mid)
        if ok:
            best = mid
            ans_lr = (l, r)
            lo = mid + 1
        else:
            hi = mid - 1

    return best, ans_lr

def solve_case(n, k, a):
    mx, seg_max = witness_max(a, k)

    b = [n + 1 - x for x in a]
    rev_mx, seg_min = witness_max(b, k)
    mn = n + 1 - rev_mx

    l1, r1 = seg_min
    l2, r2 = seg_max

    if l1 > l2:
        l1, l2 = l2, l1
        r1, r2 = r2, r1

    fw = Fenwick(n)

    def add_pos(i):
        fw.add(a[i - 1], 1)

    def rem_pos(i):
        fw.add(a[i - 1], -1)

    l, r = l1, r1
    for i in range(l, r + 1):
        add_pos(i)

    parent = list(range(n + 2))

    def find(x):
        while parent[x] != x:
            parent[x] = parent[parent[x]]
            x = parent[x]
        return x

    ans = [None] * (n + 1)

    def process():
        m = r - l + 1
        rk1 = (m + 1) // 2
        rk2 = (m + 2) // 2

        lo_v = fw.kth(rk1)
        hi_v = fw.kth(rk2)

        x = find(lo_v)
        while x <= hi_v:
            ans[x] = (l, r)
            parent[x] = find(x + 1)
            x = find(x)

    process()

    while r != l2:
        if r < l2:
            r += 1
            add_pos(r)
        else:
            rem_pos(r)
            r -= 1
        process()

    while r != r2:
        r += 1
        add_pos(r)
        process()

    while l != l2:
        rem_pos(l)
        l += 1
        process()

    out = []
    for v in range(mn, mx + 1):
        out.append((v, ans[v][0], ans[v][1]))

    return out

def main():
    t = int(input())
    res = []

    for _ in range(t):
        n, k = map(int, input().split())
        a = list(map(int, input().split()))

        cur = solve_case(n, k, a)

        res.append(str(len(cur)))
        for v, l, r in cur:
            res.append(f"{v} {l} {r}")

    sys.stdout.write("\n".join(res))

if __name__ == "__main__":
    main()
```
