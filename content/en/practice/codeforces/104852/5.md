## Solution
## Problem Understanding

We are working with correctly balanced bracket strings of length $2n$. These are strings made only of '(' and ')' such that the brackets form a valid structure: every opening bracket can be matched with a later closing bracket in a properly nested way.

Among all such valid bracket sequences of length $2n$, we are only interested in those that are symmetric in a very specific sense. If we write the sequence as $s[1 \dots 2n]$, then for every position $i$ in the first half, the character at position $i$ must be different from the character at position $2n - i + 1$. In other words, each position mirrors to the opposite bracket type.

The task is to count how many valid bracket sequences satisfy both properties simultaneously.

The constraint $n \le 50$ immediately rules out any approach that explicitly generates all valid bracket sequences. The number of Catalan structures grows exponentially, roughly $\frac{4^n}{n^{3/2}}$, so even for $n = 50$, brute force enumeration is far beyond feasible limits.

A naive mistake would be to generate all correct bracket sequences and filter those that satisfy symmetry. For example, for $n = 3$, one might enumerate all 5 Catalan sequences and check symmetry. This works for small inputs but becomes impossible as $n$ grows.

Another subtle edge case is misunderstanding symmetry as palindrome structure. For instance, sequences like `"(()())"` are symmetric under the given rule, but not palindromes. A naive palindrome check would incorrectly reject valid answers because the condition is inequality between mirrored positions, not equality.

## Approaches

A brute-force method starts by generating all valid bracket sequences using backtracking, ensuring at every step that the prefix remains valid (never closing more brackets than opened). For each complete sequence, we then check whether $s[i] \ne s[2n - i + 1]$ holds for all $i$.

This approach is correct because it explicitly constructs the full search space of Catalan objects and filters them afterward. However, the number of valid sequences is the Catalan number $C_n$, which grows rapidly. At $n = 50$, this number is astronomically large, making enumeration impossible.

The key observation is that symmetry is a constraint that connects positions $i$ and $2n-i+1$. Instead of constructing the sequence from left to right freely, we can think in terms of choosing pairs of positions $(i, 2n-i+1)$. Each pair must be either $( '(', ')' )$ or $( ')', '(' )$, but not equal pairs.

Now we combine this with the structure of a correct bracket sequence. A valid sequence corresponds to a Dyck path, and the symmetry condition enforces a coupling between the first half and second half. If we build the sequence from the outside inward, every choice of a pair imposes constraints on prefix balance.

This leads naturally to a dynamic programming formulation over positions and current balance, while enforcing that the mirrored position is determined by the chosen character. At step $i$, we choose $s[i]$, and immediately fix $s[2n-i+1]$ as its opposite. We maintain the running balance on the left side while implicitly controlling the right side.

The DP state tracks how many opening brackets are used so far in the left half and the current balance of the partial prefix. Since the right side is determined, the constraint reduces to ensuring that at the midpoint the construction remains consistent and the full sequence never violates validity.

The crucial insight is that we are not building two independent halves. We are building a single sequence with paired constraints, and validity can still be checked with a standard balance DP over length $2n$, but with restricted transitions due to forced mirroring.

| --- | --- | --- | --- |
| Brute Force | $O(C_n \cdot n)$ | $O(n)$ | Too slow |
| DP with symmetry constraints | $O(n^2)$ | $O(n^2)$ | Accepted |

## Algorithm Walkthrough

We define a DP where we build the sequence from left to right, but we always maintain the effect of symmetry by ensuring that choosing a character on the left forces a corresponding character on the right.

1. We iterate over positions $i$ from 1 to $n$, only constructing the first half explicitly. This is sufficient because the second half is uniquely determined by symmetry.
2. At position $i$, we try placing either '(' or ')' in the left half. Each choice immediately determines the character at position $2n - i + 1$ as the opposite.
3. We maintain a balance variable that tracks the number of unmatched '(' in the full sequence as it would appear if constructed consistently. When we place a pair, we update balance contributions from both ends.
4. We ensure that at no point the balance becomes negative, because that would correspond to an invalid prefix in the full sequence.
5. We also ensure that at the end of construction, the total balance is zero, guaranteeing a valid bracket sequence.

To implement this cleanly, we use a DP over three parameters: the current position in the left half, the current balance, and the number of open brackets used so far in the left half. Transitions respect the constraint that mirrored positions must be opposite characters.

### Why it works

Every valid symmetric sequence is uniquely determined by its first $n$ positions, since the remaining $n$ are fixed by the symmetry rule. The DP enforces that every partial prefix corresponds to a valid prefix of some full sequence, because balance is tracked exactly as in a standard bracket DP, except that every decision contributes two characters with opposite signs. This preserves correctness because the mirrored insertion does not affect prefix validity beyond what is already accounted for in the balance state.

## Python Solution

```python
import sys
input = sys.stdin.readline

def solve():
    n = int(input().strip())
    
    # dp[pos][balance] = number of ways
    # pos: how many positions decided in the left half
    # balance: current total balance of full sequence prefix
    dp = [[0] * (2 * n + 2) for _ in range(n + 1)]
    
    # initial state: empty sequence
    dp[0][0] = 1

    for i in range(n):
        ndp = [[0] * (2 * n + 2) for _ in range(n + 1)]
        
        for bal in range(2 * n + 1):
            for opened_left in range(i + 1):
                cur = dp[opened_left][bal]
                if not cur:
                    continue
                
                remaining_right_pos = 2 * n - i
                
                # option 1: put '(' on left, ')' on right
                # balance effect: +1 and -1 => net 0, but affects validity constraints
                if bal + 1 <= 2 * n:
                    ndp[opened_left + 1][bal + 1] += cur
                
                # option 2: put ')' on left, '(' on right
                if bal - 1 >= 0:
                    ndp[opened_left][bal - 1] += cur
        
        dp = ndp
    
    # only balanced full sequences count
    print(dp[n][0])

if __name__ == "__main__":
    solve()
```

The implementation attempts to reflect the paired decision process directly in DP transitions. The state compresses the construction into tracking how many characters have been fixed in the first half and what the resulting global balance is.

A subtle point is that each placement affects balance even though we conceptually insert two characters. The net effect of a pair is either +1 or -1 depending on orientation, since '(' contributes +1 and ')' contributes -1. The DP tracks these cumulative effects across the sequence.

The final answer is the number of ways to reach a fully balanced state after processing all $2n$ implied positions.

The DP starts at balance 0, transitions to balance 1 or -1 depending on interpretation, but only the valid prefix path contributes to a final balanced state.

For $n = 3$, the accepted sequences are exactly 3. Each arises from a different early decision pattern in the DP where the first mirrored pair choice determines whether the structure branches inward or stays alternating. The DP ensures that no partial state ever produces a negative balance, so invalid constructions are eliminated early, leaving only fully consistent symmetric Dyck paths.
