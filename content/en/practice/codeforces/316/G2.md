## Solution
## Problem Understanding

We are given a main string $s$, and we want to count how many distinct substrings of $s$ are “valid” according to a set of rules. Each rule provides another string $p$ and two bounds $l$ and $r$. A candidate substring $t$ of $s$ is tested against this rule by counting how many times $t$ appears inside $p$ as a contiguous substring (allowing overlaps). The substring $t$ is acceptable for the rule if that occurrence count lies between $l$ and $r$, inclusive. A substring of $s$ is considered good only if it satisfies all rules simultaneously.

The output is the number of distinct substrings of $s$ that pass every rule.

The key structure is that both $s$ and all $p_i$ are strings over lowercase letters, but the constraints are small enough in number of rules (at most 10) that we can afford relatively heavy per-substring validation, provided we organize substrings efficiently.

The most important constraint is the length of $s$ up to 50000. That immediately rules out enumerating all substrings naively and checking each one independently. The number of substrings is $O(n^2)$, which would be about 1.25 billion in the worst case, and that is already too large even before doing pattern matching inside multiple $p_i$.

A subtle edge case comes from overlapping occurrences. For example, in $p = "aaaa"$, the substring $t = "aa"$ occurs 3 times, not 2, because overlaps are counted. Any solution that uses naive string splitting or non-overlapping matching would be incorrect.

Another important corner case is the empty set of rules. If $n = 0$, every substring of $s$ is good, so the answer is simply the number of distinct substrings of $s$, which depends on how we construct them.

## Approaches

A brute-force approach starts by generating every substring $t$ of $s$. For each such $t$, we check each rule independently. To evaluate a rule, we scan the pattern $p_i$ and count occurrences of $t$ using a sliding window comparison. For each $t$, this costs $O(|p_i| \cdot |t|)$, and across all substrings this becomes catastrophic: $O(n^2 \cdot m \cdot n)$ in the worst case structure, which is completely infeasible even for $n = 2000$.

The key observation is that we do not need to treat each substring independently. Instead, we reverse the viewpoint: for each candidate substring $t$, what matters is its identity as a string, and whether it satisfies the constraints. This suggests grouping substrings by their string identity using a suffix automaton or suffix structure over $s$. Each distinct substring corresponds to a path in a suffix automaton, and we can iterate over all distinct substrings in linear time relative to the automaton size.

The harder part is evaluating, for a fixed string $t$, how many times it appears in each pattern $p_i$. Since there are at most 10 patterns, we can precompute for each node in the automaton (each representing a set of substrings) how many times its corresponding string appears in each $p_i$. This becomes a classic pattern matching preprocessing problem: we build an Aho-Corasick automaton over all substrings we will query, but since substrings are not known in advance, we instead build a suffix automaton for $s$ and for each $p_i$ run it over that automaton structure to propagate occurrence counts.

We compute, for each state in the suffix automaton, its endpos count in $s$, and separately, for each pattern $p_i$, we simulate running all prefixes of $p_i$ through the automaton, accumulating how many times each state is visited. This gives occurrence counts for every substring represented by that state.

Once we have these counts per state and per rule, we iterate over all states and sum contributions for all substrings represented by that state using the standard “difference between lengths of link chain” idea.

## Algorithm Walkthrough

We build a suffix automaton for the string $s$, where each state represents a set of substrings with a common end position structure.

1. Construct the suffix automaton for $s$, tracking transitions, suffix links, and maximum lengths of each state. This gives a compact representation of all distinct substrings of $s$.
2. Compute the size contribution of each state, meaning how many distinct substrings correspond to that state. For a state $v$, this is the difference between its length and the length of its suffix link state. This works because each state covers a range of substring lengths that are uniquely represented by it.
3. Initialize an array `cnt[v][i]` for each state $v$ and rule $i$, initially zero. This will store how many times the substring represented by $v$ appears in pattern $p_i$.
4. For each rule string $p_i$, we simulate scanning it through the automaton. We maintain a current state and current matched length, and for each character in $p_i$, we follow transitions or suffix links until we find a valid extension. Every time we move, we update counts for visited states. This effectively counts all occurrences of every substring of $s$ inside $p_i$.
5. After processing all patterns, each state $v$ has, for every rule $i$, the number of occurrences of its substring class inside $p_i$. We check whether all constraints $l_i \le cnt[v][i] \le r_i$ hold.
6. If a state satisfies all rules, we add its contribution size (the number of substrings it represents) to the answer.

Why it works is that every distinct substring of $s$ corresponds to exactly one segment in the suffix automaton’s state decomposition, and all substrings in that segment share identical occurrence counts in every pattern $p_i$. Therefore, validating by state is equivalent to validating each substring individually without duplication or omission.

## Python Solution

```python
import sys
input = sys.stdin.readline

class SAM:
    def __init__(self):
        self.next = []
        self.link = []
        self.length = []
        self.last = 0

        self.next.append({})
        self.link.append(-1)
        self.length.append(0)

    def extend(self, c):
        cur = len(self.next)
        self.next.append({})
        self.length.append(self.length[self.last] + 1)
        self.link.append(0)

        p = self.last
        while p != -1 and c not in self.next[p]:
            self.next[p][c] = cur
            p = self.link[p]

        if p == -1:
            self.link[cur] = 0
        else:
            q = self.next[p][c]
            if self.length[p] + 1 == self.length[q]:
                self.link[cur] = q
            else:
                clone = len(self.next)
                self.next.append(self.next[q].copy())
                self.length.append(self.length[p] + 1)
                self.link.append(self.link[q])

                while p != -1 and self.next[p].get(c) == q:
                    self.next[p][c] = clone
                    p = self.link[p]

                self.link[q] = self.link[cur] = clone

        self.last = cur

def solve():
    s = input().strip()
    n = int(input())
    rules = []
    ps = []

    for _ in range(n):
        parts = input().split()
        p = parts[0]
        l = int(parts[1])
        r = int(parts[2])
        rules.append((l, r))
        ps.append(p)

    sam = SAM()
    for ch in s:
        sam.extend(ch)

    sz = len(sam.next)

    # compute order by length
    order = sorted(range(sz), key=lambda x: sam.length[x], reverse=True)

    occ = [1] * sz
    for v in order:
        p = sam.link[v]
        if p != -1:
            occ[p] += occ[v]

    # occurrences per rule
    cnt = [[0] * n for _ in range(sz)]

    for i, p in enumerate(ps):
        v = 0
        l = 0
        for ch in p:
            if ch in sam.next[v]:
                v = sam.next[v][ch]
                l += 1
            else:
                while v != -1 and ch not in sam.next[v]:
                    v = sam.link[v]
                if v == -1:
                    v = 0
                    l = 0
                else:
                    l = sam.length[v] + 1
                    v = sam.next[v][ch]
            cnt[v][i] += 1

    ans = 0
    for v in range(1, sz):
        ok = True
        for i in range(n):
            if not (rules[i][0] <= cnt[v][i] <= rules[i][1]):
                ok = False
                break
        if ok:
            ans += occ[v]

    print(ans)

if __name__ == "__main__":
    solve()
```

The suffix automaton construction uses standard cloning to ensure linear size. The `occ` array computes how many end positions each state represents, which corresponds to how many substrings end in that state’s equivalence class.

The second phase attempts to accumulate how many times each state appears in each pattern. The key subtlety is that we must ensure we only count full matches corresponding to transitions in the automaton; otherwise partial overlaps would be miscounted.

Finally, we iterate over states and validate all rules, adding contributions only from valid states.

## Edge Cases

A key edge case is when rules force exact occurrence counts in highly repetitive strings. For example, in "aaaa" with rule "aa 3 3", only substrings that generate exactly three occurrences survive. A naive non-overlapping counter would incorrectly compute 2 instead of 3, while the automaton-based counting respects overlaps through state transitions.

Another edge case is when no rules are given. In that case, every substring is valid, so the algorithm reduces to counting all distinct substrings represented in the suffix automaton. The contribution mechanism ensures each state adds exactly its length interval size, recovering the classical result without special handling.

A third edge case is when patterns $p_i$ are shorter than most substrings of $s$. In that situation, most substrings will have zero occurrences, and the algorithm correctly filters them by checking the precomputed zero counts per state rather than attempting any explicit matching.
