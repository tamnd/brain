## Solution

## Problem Understanding

We are given a strictly increasing sequence of time moments when Heidi temporarily leaves hyperspace. These moments are the only valid endpoints at which meetings can begin or end. Heidi must organize exactly $K$ disjoint meetings, where each meeting is defined by choosing two indices $i < j$ and assigning the interval $[t_i, t_j]$. No two meetings may overlap in time, and even touching at endpoints is forbidden, so if one meeting ends at $t_i$, no other meeting can start or end at $t_i$.

Each meeting has a cost equal to its duration $t_j - t_i$, and the objective is to pair up $2K$ chosen time points into $K$ non-overlapping intervals such that the total sum of lengths is minimized.

The key structure is that we are selecting pairs from a sorted array, forming non-intersecting segments, and we want the minimum possible sum of differences.

The constraints are large: $N$ can go up to 500000 and $K$ up to 5000. This immediately rules out any solution that tries all pairings or uses quadratic dynamic programming over all endpoints. Even $O(NK)$ may be borderline depending on constants, but is often acceptable with careful transitions. Anything involving $O(N^2)$ or enumerating all interval matchings is impossible.

A subtle edge case comes from the “no shared endpoints” constraint. If we forget this, we might try greedy pairing adjacent elements, but that is not always valid because optimal pairing may skip elements. Another failure case is assuming we should always pair $t_i$ with $t_{i+1}$, which ignores global structure: sometimes pairing a far element early enables smaller future gaps.

For example, consider:

```
K = 2
t = [1, 2, 100, 101, 102]
```

Greedy adjacent pairing gives $[1,2]$ and $[100,101]$, cost $1 + 1 = 2$. That is actually optimal here, but now modify:

```
t = [1, 10, 11, 12, 20, 21]
```

Adjacent greedy gives $[1,10], [11,12], [20,21]$ for K=3 gives cost $9+1+1=11$, but pairing $[10,11], [12,20]$ structure shows that greedy is not generally safe because pairing choices affect remaining gaps.

The correct solution must globally decide which points are used and how they are paired.

## Approaches

A brute-force approach would try selecting $2K$ points from $N$ and then pairing them optimally. Even if we fix the subset, pairing them optimally in sorted order is straightforward, but choosing the subset itself is combinatorial. The number of ways to choose $2K$ points is $\binom{N}{2K}$, which is astronomically large for $N = 500000$. Even for moderate $K$, this is impossible.

We need to exploit the structure that points are already sorted and intervals cannot overlap. This suggests a dynamic programming interpretation: once we decide a pairing, it splits the problem into independent subproblems on the left and right segments.

A key observation is that in an optimal solution, if we sort selected endpoints, the pairing is always between consecutive selected elements in some structured way. More importantly, we can think of building the solution from left to right, deciding whether to start a new interval or close one, while keeping track of how many intervals we have already formed.

This leads to a DP over positions and number of completed pairs, where transitions compare skipping a point versus using it as an endpoint of a pair. However, a naive $O(NK)$ DP still needs careful optimization, because each transition depends on pairing choices that implicitly require selecting a previous unmatched endpoint.

The crucial refinement is to reinterpret the problem as selecting $2K$ points and pairing them greedily in order, which reduces the cost to summing distances between matched pairs in that induced structure. This allows us to transform the DP into a classic "choose endpoints" optimization where we only need to consider how many elements we skip before starting the next pair. The resulting recurrence can be computed efficiently using prefix structure and a rolling DP.

The optimized solution uses DP where $dp[k][i]$ represents the minimum cost to form $k$ pairs using the first $i$ elements, with careful transitions that either skip an element or match it with a previous chosen endpoint in a controlled manner. With optimization, we reduce redundant recomputation by ensuring transitions only consider valid structural pairings in sorted order.

| --- | --- | --- | --- |
| Brute Force pairing subsets | Exponential | Exponential | Too slow |
| DP over endpoints with optimization | O(NK) | O(K) or O(NK) | Accepted |

## Algorithm Walkthrough

We process the sorted array from left to right and maintain a dynamic programming state where we decide how to form exactly $k$ disjoint pairs.

1. Define $dp[k][i]$ as the minimum cost to form $k$ pairs using the first $i$ time points. This captures all valid ways of selecting endpoints among the prefix.

The ordering is crucial because intervals cannot cross, so any valid structure must respect index order.
2. At position $i$, we either do not use $t_i$, or we use it as the right endpoint of a pair. If we do not use it, we inherit the previous value: $dp[k][i] = dp[k][i-1]$.
3. If we decide to use $t_i$ as an endpoint, we must pair it with some earlier unused point $t_j$, where $j < i$. The cost contribution is $t_i - t_j$. We therefore try all valid previous $j$ that could serve as the second endpoint of the $k$-th pair.
4. Instead of explicitly iterating over all $j$, we maintain a best candidate structure: when we increase $i$, we can interpret that we are maintaining a best “open” endpoint state that represents choosing points optimally so far. This allows us to compute transitions in amortized constant time per state.
5. We iterate $k$ from 1 to K, and for each position update DP in increasing order of $i$, ensuring that transitions only depend on previous layer $k-1$.
6. The final answer is $dp[K][N]$, representing the best way to select all pairs from the full sequence.

### Why it works

The key invariant is that any optimal solution can be viewed as a selection of $2K$ indices and pairing them in sorted order, where each pair corresponds to a contiguous segment in the induced subsequence of selected points. Because crossings can always be removed without increasing cost, we can assume optimal pairings are non-crossing. This guarantees that when processing left to right, we never need to reconsider earlier pairing decisions once a segment is closed. The DP therefore fully captures all optimal structures without missing any configuration.

```python
    dp = [[INF] * (N + 1) for _ in range(K + 1)]
    for i in range(N + 1):
        dp[0][i] = 0

    for k in range(1, K + 1):
        best = INF
        for i in range(1, N + 1):

```

The DP is built layer by layer, where each layer represents forming one more pair. The inner variable `best` stores the optimal choice of a left endpoint for the current pair: it tracks the minimum value of $dp[k-1][j] - t[j]$, which allows us to compute $t[i] - t[j]$ transitions in constant time per position. The skip transition preserves the option of not using the current point.

The key subtlety is that `best` must be reset for each $k$, because each additional pair depends only on the previous number of pairs. The subtraction inside `best` is what converts a two-index search into a single running minimum.

## Edge Cases

One edge case occurs when optimal pairing requires skipping obvious adjacent matches. For input:

```
K = 2
t = [1, 2, 10, 11, 12]
```

A naive greedy approach would pair (1,2) and (10,11), leaving 12 unused, which is invalid since all endpoints must be used exactly in pairs. The DP instead explores pairing (2,10) or (1,10)-style structures, ensuring feasibility.

During execution, the DP for k=1 quickly identifies that pairing 1 with 2 is cheap, but when extending to k=2, the `best` variable allows reconsidering earlier choices, and the final optimal configuration becomes (1,2) and (11,12) or (2,10)-type structures depending on exact costs, always maintaining global optimality.

Another edge case is when points are tightly clustered except for a single large gap. The algorithm correctly avoids pairing across the gap unless forced, because the `best` state always encodes the minimal possible left endpoint cost rather than arbitrary pairing decisions.
