## Solution
## Problem Understanding

We are given a rectangular grid with $n$ rows and $m$ columns, containing $n \cdot m$ unit cells. The task is not to place dominoes explicitly, but to decide whether such a placement exists that covers every cell exactly once using $\frac{nm}{2}$ dominoes, where each domino covers two adjacent cells.

Among all these dominoes, we want exactly $k$ to be placed horizontally, meaning they connect left-right neighbors, while the remaining $\frac{nm}{2} - k$ must be vertical, connecting up-down neighbors.

A useful way to reinterpret the problem is that we are trying to decompose the grid into disjoint adjacent pairs, and we want to control how many pairs are taken horizontally versus vertically.

The constraint $n \cdot m$ is always even, which guarantees that a full tiling by dominoes is not immediately impossible due to parity. With $n, m \le 100$, the input size is small enough that any $O(1)$ per test case reasoning is sufficient, while any simulation or search over placements would still be feasible in principle but unnecessary.

The key difficulty is not placement itself but understanding how many horizontal dominoes can coexist with a full tiling.

A naive but misleading edge case appears when one dimension is 1. For example, if $n = 1, m = 4$, the grid is a single line. Only horizontal dominoes are possible, so $k$ must equal $m/2$. Any value smaller than that is impossible even though $nm/2$ is still valid.

Similarly, when $n = 2, m = 2$, we have exactly two dominoes total. Both orientations are possible but constrained, and some $k$ values like 1 are achievable, while others become impossible depending on parity of available horizontal slots.

These examples show that even though total area parity is sufficient for existence of a tiling, orientation constraints depend heavily on geometry.

## Approaches

A brute-force approach would try to construct all domino tilings of the grid and count how many horizontal dominoes each configuration uses. This is equivalent to enumerating perfect matchings in a grid graph and checking their horizontal edge count. Even for $n = m = 6$, the number of tilings grows exponentially, since each configuration corresponds to a perfect matching in a grid graph with many cycles. This quickly becomes infeasible beyond very small grids.

The key observation is that we do not need to construct a tiling at all. We only need to know whether some tiling with a given number of horizontal dominoes exists. This turns the problem into reasoning about local capacity constraints per row.

Each row of length $m$ can accommodate at most $\lfloor m/2 \rfloor$ horizontal dominoes, because each horizontal domino occupies two adjacent cells in the same row. Across all $n$ rows, this gives a total upper bound of $n \cdot \lfloor m/2 \rfloor$ horizontal dominoes.

Similarly, if we rotate perspective, we can bound vertical dominoes. Each column can contribute at most $\lfloor n/2 \rfloor$ vertical dominoes, giving a total of $m \cdot \lfloor n/2 \rfloor$.

Since every domino is either horizontal or vertical, and we need exactly $nm/2$ dominoes in total, the number of horizontal dominoes $k$ is feasible exactly when it lies in a range where both horizontal and vertical capacities are not violated.

The critical insight is that the grid behaves like a bipartite structure where horizontal and vertical matchings are independently bounded by row and column pairing capacities, and any distribution within those bounds can be realized by rearranging local pairings.

This reduces the problem to a simple range check.

## Algorithm Walkthrough

We reason about how many horizontal dominoes can exist in any valid full tiling.

1. Compute the total number of dominoes as $t = \frac{n \cdot m}{2}$. This is fixed because every domino covers exactly two cells.
2. Compute the maximum possible horizontal dominoes per row as $\lfloor m/2 \rfloor$. Each such domino consumes two adjacent cells in a single row, and no overlap is allowed, so a row cannot exceed this count.
3. Multiply by the number of rows to get the global horizontal capacity $H_{max} = n \cdot \lfloor m/2 \rfloor$. This is the absolute ceiling on horizontal dominoes in any tiling.
4. Compute the maximum possible vertical dominoes per column as $\lfloor n/2 \rfloor$, and multiply by columns to get $V_{max} = m \cdot \lfloor n/2 \rfloor$.
5. Convert vertical constraint into a horizontal lower bound. Since total dominoes is fixed, vertical count is $t - k$, so we must have $t - k \le V_{max}$, which becomes $k \ge t - V_{max}$.
6. Combine constraints: $k$ must satisfy both $t - V_{max} \le k \le H_{max}$.
7. Output "YES" if $k$ lies in this interval, otherwise output "NO".

### Why it works

Every domino in a tiling is either horizontal or vertical. Horizontal dominoes are constrained only within rows, and vertical dominoes only within columns. The grid decomposes into independent row pairings and column pairings, and any valid tiling corresponds to choosing a consistent mixture of these pairings.

The bounds derived from rows and columns are tight because we can always construct tilings that achieve any feasible split between horizontal and vertical dominance by locally rearranging 2×2 blocks. Since no global obstruction exists beyond row and column pairing capacity, the interval condition is both necessary and sufficient.

## Python Solution

```python
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    n, m, k = map(int, input().split())
    
    total = n * m // 2
    
    max_horizontal = n * (m // 2)
    max_vertical = m * (n // 2)
    
    min_horizontal = total - max_vertical
    
    if min_horizontal <= k <= max_horizontal:
        print("YES")
    else:
        print("NO")
```

The solution directly encodes the derived feasibility interval. The computation of $m // 2$ and $n // 2$ captures the pairing limits per row and column. The subtraction step converts the vertical capacity into a lower bound for horizontal dominoes, ensuring both constraints are enforced simultaneously.

All arithmetic is constant time per test case, so the solution is optimal for the given limits.

## Worked Examples

Consider the case $n = 2, m = 4, k = 2$.

We compute total dominoes as $2 \cdot 4 / 2 = 4$. Each row can host at most $4 // 2 = 2$ horizontal dominoes, so $H_{max} = 4$. Each column can host at most $2 // 2 = 1$ vertical domino, so $V_{max} = 4$. This yields a horizontal lower bound of $4 - 4 = 0$.

| V_max | 4 |
| k range | [0, 4] |

Since 2 lies inside the interval, the answer is YES. This confirms that intermediate mixes of orientations are achievable.

Now consider $n = 1, m = 4, k = 1$.

Total dominoes is 2. Horizontal capacity is $1 \cdot 2 = 2$. Vertical capacity is $4 \cdot 0 = 0$. The lower bound becomes $2 - 0 = 2$.

| Step | Value |
| --- | --- |
| total | 2 |
| H_max | 2 |
| V_max | 0 |
| k range | [2, 2] |
| k | 1 |

Since 1 is outside the allowed interval, the answer is NO. This captures the fact that a single row forces all dominoes to be horizontal.

## Edge Cases

For a grid with a single row such as $1 \times 4$, the algorithm computes total $=2$, $H_{max}=2$, $V_{max}=0$, and thus the valid range collapses to $k=2$. This directly matches the fact that no vertical placement exists in a single row, forcing all dominoes to be horizontal.

For a grid with a single column such as $4 \times 1$, the symmetric reasoning gives a valid range $k=0$, since horizontal placement is impossible and all dominoes must be vertical. The interval check naturally enforces this by producing $H_{max}=0$ and a non-zero lower bound equal to zero as well.
