## Solution

## Problem Understanding

The problem asks us to maximize the product of sums obtained by splitting a binary tree into two subtrees. We are given the `root` of a binary tree, where each node has a positive integer value. We can remove any one edge in the tree to form exactly two disjoint subtrees. For each possible split, we compute the sum of the nodes in each subtree and take their product. The goal is to find the maximum possible product of these sums and return it modulo `10^9 + 7`.

The input represents a standard binary tree structure. The constraints specify that there are up to `5 * 10^4` nodes, and each node value can be as large as `10^4`. This tells us that brute-force approaches that iterate over all edges and recompute subtree sums repeatedly would be too slow. The problem guarantees at least two nodes, so we never have to handle empty trees. Since node values are positive, the maximum product will always involve splitting somewhere meaningful in the tree, and no sum will be zero.

Edge cases include trees that are skewed (all left or all right), trees where all nodes have the same value, and very small trees with only two nodes, where the split is trivial.

## Approaches

The **brute-force approach** would involve trying to remove each edge individually, then computing the sum of the two resulting subtrees from scratch. While this guarantees correctness, it is inefficient because computing subtree sums from scratch is `O(n)` for each edge. With `n` edges, this results in `O(n^2)` time complexity, which is unacceptable for `n = 5 * 10^4`.

The **optimal approach** relies on two key observations. First, we can compute the total sum of the tree using a single traversal. Second, for each node, we can compute the sum of the subtree rooted at that node. If we conceptually “cut” the edge connecting this node to its parent, the product of the subtree sum and the remaining sum (total sum minus this subtree sum) can be computed in `O(1)`. By performing a single post-order traversal to compute subtree sums and track the maximum product, we can solve the problem efficiently in `O(n)` time and `O(n)` space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Computes subtree sums for every possible edge removal. |
| Optimal | O(n) | O(n) | Single post-order traversal computes subtree sums and maximum product. |

## Algorithm Walkthrough

1. Compute the total sum of all nodes in the binary tree using a post-order traversal. This ensures we know the sum of any potential remaining subtree after a split.
2. Initialize a variable to store the maximum product found so far.
3. Perform a second post-order traversal of the tree. At each node, compute the sum of its subtree. Consider the product obtained by “cutting” the connection to its parent: `(total sum - subtree sum) * subtree sum`. Update the maximum product if this product is larger.
4. After traversing all nodes, return the maximum product modulo `10^9 + 7`.

**Why it works:** The key invariant is that the sum of any subtree can be computed in post-order traversal without recomputation. By visiting every node exactly once, we consider every possible edge removal, guaranteeing that we find the maximum product.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def maxProduct(self, root: Optional[TreeNode]) -> int:
        MOD = 10**9 + 7
        
        # First traversal to compute total sum
        def subtree_sum(node: Optional[TreeNode]) -> int:
            if not node:
                return 0
            return node.val + subtree_sum(node.left) + subtree_sum(node.right)
        
        total_sum = subtree_sum(root)
        max_product = 0
        
        # Second traversal to compute maximum product
        def find_max_product(node: Optional[TreeNode]) -> int:
            nonlocal max_product
            if not node:
                return 0
            left_sum = find_max_product(node.left)
            right_sum = find_max_product(node.right)
            current_sum = node.val + left_sum + right_sum
            product = current_sum * (total_sum - current_sum)
            max_product = max(max_product, product)
            return current_sum
        
        find_max_product(root)
        return max_product % MOD
```

This Python implementation first calculates the total sum of the tree. Then, it performs a second post-order traversal, calculating the sum of each subtree and the corresponding product if that subtree were separated. We update the maximum product during this traversal, ensuring an `O(n)` runtime with only two traversals.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func maxProduct(root *TreeNode) int {
    const MOD = 1_000_000_007
    var totalSum int
    
    var subtreeSum func(node *TreeNode) int
    subtreeSum = func(node *TreeNode) int {
        if node == nil {
            return 0
        }
        return node.Val + subtreeSum(node.Left) + subtreeSum(node.Right)
    }
    
    totalSum = subtreeSum(root)
    
    maxProduct := 0
    
    var findMaxProduct func(node *TreeNode) int
    findMaxProduct = func(node *TreeNode) int {
        if node == nil {
            return 0
        }
        leftSum := findMaxProduct(node.Left)
        rightSum := findMaxProduct(node.Right)
        currentSum := node.Val + leftSum + rightSum
        product := currentSum * (totalSum - currentSum)
        if product > maxProduct {
            maxProduct = product
        }
        return currentSum
    }
    
    findMaxProduct(root)
    return maxProduct % MOD
}
```

In Go, we handle `nil` instead of `None`, and we use integer arithmetic carefully since Go does not have native big integers by default. We keep the logic identical to Python, with two recursive traversals to compute the total sum and then the maximum product.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Two traversals of the tree, each node visited once. |
| Space | O(n) | Recursive stack in the worst case of a skewed tree. |

The time complexity is linear because each node is processed exactly twice. Space complexity is dominated by the recursion stack for deep trees.
