## Problem Understanding

The problem asks us to compute a **running total of scores per gender, ordered by day**. Specifically, we are given a `Scores` table where each row represents a player scoring a certain number of points on a specific day. Each player belongs to a gender, either `'F'` for female or `'M'` for male. The combination `(gender, day)` is unique, so no two rows have the same gender on the same day.

The input is a table of scores with columns: `player_name`, `gender`, `day`, and `score_points`. The output should be a table with columns: `gender`, `day`, and `total`, where `total` is the **cumulative score for that gender up to that day**, inclusive. The output should be ordered by `gender` first, then `day` ascending.

Key points:

1. The cumulative sum is **per gender**, not per player.
2. Days may not be consecutive; the algorithm must handle gaps.
3. The primary key guarantees no duplicate `(gender, day)` pairs, simplifying aggregation.
4. Edge cases include a single gender, a single day, or no data at all.

## Approaches

A **brute-force approach** would iterate over each gender, sort all its days, and repeatedly sum all previous scores for that gender up to the current day. While correct, this requires repeatedly scanning previous days for each row, leading to an **O(n²)** time complexity, where `n` is the number of rows. This is inefficient for large datasets.

The **optimal approach** leverages SQL **window functions** or, in Python/Go implementations, a hash map combined with a running sum. By first **aggregating scores per day and gender**, then maintaining a cumulative total, we can achieve a **single-pass solution** with O(n log n) time for sorting and O(n) for cumulative sum calculation.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Sum previous days for each row; correct but slow |
| Optimal | O(n log n) | O(n) | Aggregate then compute cumulative sum per gender |

## Algorithm Walkthrough

1. **Aggregate scores by day and gender**: Combine all scores for the same gender on the same day. This ensures we work with one score per day per gender.
2. **Sort days per gender in ascending order**: Sorting guarantees we can compute running totals chronologically.
3. **Maintain cumulative sums per gender**: Initialize a hash map to track the running total. Iterate over the sorted rows and update the total for each gender.
4. **Output the result in sorted order**: Use the stored totals to produce the final table, ordered by `gender` and `day`.

**Why it works**: Sorting by day per gender ensures that each row's cumulative sum accounts for all previous scores. Maintaining a running total guarantees correctness without repeatedly scanning previous rows.

## Python Solution

```python
from typing import List
import collections

class Solution:
    def getCumulativeScores(self, scores: List[dict]) -> List[dict]:
        # Step 1: Aggregate scores per gender per day
        daily_scores = collections.defaultdict(int)
        for row in scores:
            key = (row['gender'], row['day'])
            daily_scores[key] += row['score_points']
        
        # Step 2: Sort by gender, then day
        sorted_keys = sorted(daily_scores.keys(), key=lambda x: (x[0], x[1]))
        
        # Step 3: Compute cumulative totals per gender
        cumulative_totals = collections.defaultdict(int)
        result = []
        for gender, day in sorted_keys:
            cumulative_totals[gender] += daily_scores[(gender, day)]
            result.append({
                'gender': gender,
                'day': day,
                'total': cumulative_totals[gender]
            })
        return result
```

The implementation first aggregates the scores to avoid multiple rows for the same day and gender. It sorts the aggregated results to ensure cumulative totals are computed correctly. Finally, it iterates through the sorted data and maintains a running total per gender.

## Go Solution

```go
package main

import (
	"sort"
)

type Score struct {
	PlayerName  string
	Gender      string
	Day         string
	ScorePoints int
}

type Result struct {
	Gender string
	Day    string
	Total  int
}

func GetCumulativeScores(scores []Score) []Result {
	// Step 1: Aggregate scores per gender per day
	dailyScores := make(map[string]map[string]int)
	for _, s := range scores {
		if _, ok := dailyScores[s.Gender]; !ok {
			dailyScores[s.Gender] = make(map[string]int)
		}
		dailyScores[s.Gender][s.Day] += s.ScorePoints
	}

	// Step 2: Prepare sorted output
	var result []Result
	for gender, dayMap := range dailyScores {
		var days []string
		for day := range dayMap {
			days = append(days, day)
		}
		sort.Strings(days)
		runningTotal := 0
		for _, day := range days {
			runningTotal += dayMap[day]
			result = append(result, Result{
				Gender: gender,
				Day:    day,
				Total:  runningTotal,
			})
		}
	}

	// Step 3: Sort final result by gender then day
	sort.Slice(result, func(i, j int) bool {
		if result[i].Gender == result[j].Gender {
			return result[i].Day < result[j].Day
		}
		return result[i].Gender < result[j].Gender
	})
	return result
}
```

Go requires explicit initialization of maps and slices. We use nested maps to aggregate scores, then sort days per gender before calculating the cumulative total. Finally, we sort the result slice to match the required output order.
