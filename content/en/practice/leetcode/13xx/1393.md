## Solution

## Problem Understanding

The problem is asking us to calculate the capital gain or loss for each stock based on a series of buy and sell operations. The input is a table `Stocks` with columns `stock_name`, `operation` (Buy or Sell), `operation_day`, and `price`. Each row represents either a purchase or a sale of a stock on a specific day at a certain price. The output should be a table with two columns: `stock_name` and `capital_gain_loss`, showing the total profit or loss for each stock after all transactions.

The problem guarantees that every `Sell` operation has a corresponding previous `Buy` operation and every `Buy` operation has a corresponding future `Sell` operation. This means we do not have to handle unmatched buys or sells, simplifying our implementation. The operations must be considered in chronological order (`operation_day`) because the gain or loss depends on buying before selling. Edge cases include multiple buy-sell cycles for the same stock, and negative capital gain when the stock is sold for less than it was bought.

## Approaches

### Brute Force

The brute-force approach is to iterate over each stock, sort its operations by `operation_day`, and then pair each `Buy` with its corresponding `Sell`. For each pair, calculate the gain/loss (`sell_price - buy_price`) and accumulate it. While this approach is correct, it requires sorting operations for every stock, which may be inefficient for a large number of records. Its complexity is dominated by sorting and iteration.

### Optimal Approach

The optimal approach leverages the fact that SQL can directly group by `stock_name` and sum the difference between the sell and buy prices using window functions or self-joins. We first join each `Sell` operation with the corresponding previous `Buy` operation on the same stock, ordered by `operation_day`. Once paired, we calculate `sell.price - buy.price` and sum it per stock. This avoids manual iteration and reduces complexity using SQL aggregation functions.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(n) | Sort operations per stock, then iterate to pair buys and sells |
| Optimal | O(n) | O(n) | Use SQL self-join and aggregation to compute gain/loss directly |

## Algorithm Walkthrough

1. Separate `Buy` and `Sell` operations into two subsets of the `Stocks` table.
2. For each `Sell`, find the corresponding previous `Buy` for the same stock. Ensure the join only pairs a `Sell` with a `Buy` where `buy.operation_day < sell.operation_day`.
3. Calculate the gain or loss for each pair as `sell.price - buy.price`.
4. Sum all gains and losses for each stock to get the total `capital_gain_loss`.
5. Return the result table with `stock_name` and `capital_gain_loss`.

**Why it works:** Every sell operation is guaranteed to have a matching buy operation earlier. By pairing sells with their corresponding buys and summing the differences, we capture the total capital gain or loss for each stock accurately.

## Python Solution

```python
import pandas as pd

def calculate_capital_gain_loss(stocks: pd.DataFrame) -> pd.DataFrame:
    # Separate buy and sell operations
    buys = stocks[stocks['operation'] == 'Buy']
    sells = stocks[stocks['operation'] == 'Sell']

    # Merge sell with corresponding buy on stock_name where buy is before sell
    merged = pd.merge(sells, buys, on='stock_name', suffixes=('_sell', '_buy'))
    merged = merged[merged['operation_day_buy'] < merged['operation_day_sell']]

    # Sort and pick the earliest unmatched buy for each sell
    merged = merged.sort_values(['stock_name', 'operation_day_sell', 'operation_day_buy'])
    merged['gain'] = merged['price_sell'] - merged['price_buy']
    
    # Sum gains for each stock
    result = merged.groupby('stock_name')['gain'].sum().reset_index()
    result.rename(columns={'gain': 'capital_gain_loss'}, inplace=True)
    return result
```

This code separates buys and sells, merges them on the stock name, filters to ensure the buy happened before the sell, sorts to pick the earliest valid buy for each sell, computes the gain, and aggregates the total per stock.

## Go Solution

```go
package main

import (
    "database/sql"
    _ "github.com/go-sql-driver/mysql"
    "fmt"
)

func CapitalGainLoss(db *sql.DB) {
    query := `
        SELECT stock_name, SUM(s.price - b.price) AS capital_gain_loss
        FROM Stocks b
        JOIN Stocks s ON b.stock_name = s.stock_name
        AND b.operation = 'Buy'
        AND s.operation = 'Sell'
        AND b.operation_day < s.operation_day
        GROUP BY stock_name;
    `
    rows, err := db.Query(query)
    if err != nil {
        panic(err)
    }
    defer rows.Close()

    for rows.Next() {
        var stock string
        var gain int
        rows.Scan(&stock, &gain)
        fmt.Printf("%s: %d\n", stock, gain)
    }
}
```

The Go implementation uses SQL to compute the gains directly, leveraging a self-join and aggregation. Differences from Python include no DataFrame operations, handling results through `rows.Next()` and scanning each row.

## Edge Cases

One edge case is a stock bought and sold at the same price, which should result in zero gain. Another is multiple overlapping buy-sell pairs; the algorithm ensures chronological matching, avoiding mismatches. A third case is only one stock in the table; the algorithm still computes correctly as it aggregates by `stock_name`. All cases are handled by pairing sells with the earliest valid previous buy and summing results.
