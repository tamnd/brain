## Solution

## Problem Understanding

This problem asks us to find the minimum cost to make at least one valid path from the top-left corner `(0, 0)` to the bottom-right corner `(m-1, n-1)` in a given `m x n` grid. Each cell in the grid contains a directional sign pointing to a neighbor cell: `1` for right, `2` for left, `3` for down, and `4` for up. A valid path is one where you follow these signs and reach the target cell.

You are allowed to **change the direction of a sign in any cell at a cost of 1**. The goal is to determine the **minimum total cost** to adjust the grid so that at least one path from `(0, 0)` to `(m-1, n-1)` exists.

The constraints ensure that the grid is at most 100x100, meaning there are up to 10,000 cells, which allows for algorithms with time complexity up to `O(m * n log(m * n))` using priority queues. The grid will always contain integers between 1 and 4, so there are no invalid signs, and there is no guarantee that a valid path initially exists.

Important edge cases include grids that already have a valid path (cost = 0), grids that require multiple direction changes, and grids where multiple optimal paths exist. Signs pointing outside the grid must be handled carefully to avoid index errors.

## Approaches

The **brute-force approach** would try to explore all possible paths from `(0, 0)` to `(m-1, n-1)` and keep track of the cost for changing directions. Essentially, it would perform a DFS or BFS where each step considers changing the direction or following the current direction. This approach is correct but extremely inefficient because the number of paths can grow exponentially in the grid size (`O(2^(m*n))` in the worst case) and is therefore not feasible.

The **key observation** is that this problem is a shortest-path problem on a weighted graph, where moving along the cell's current direction costs 0, and changing direction costs 1. This is a variant of Dijkstra's algorithm or 0-1 BFS, where edges have weight 0 or 1. Using a deque for 0-1 BFS ensures that nodes with cost 0 are processed immediately while nodes with cost 1 are added to the back, efficiently computing the minimum cost to reach the target.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^(m*n)) | O(m*n) | Explores all paths with direction changes, infeasible for large grids |
| Optimal (0-1 BFS) | O(m * n) | O(m * n) | Treats moving along direction as 0 cost, changing direction as cost 1, uses deque for BFS |

## Algorithm Walkthrough

1. Initialize a `distance` matrix of size `m x n` with `inf` values, representing the minimum cost to reach each cell.
2. Set the distance of the starting cell `(0, 0)` to 0.
3. Use a deque `dq` to perform 0-1 BFS, starting with `(0, 0)`.
4. Define direction vectors for `right`, `left`, `down`, and `up` to map the integers 1-4 to grid movements.
5. While the deque is not empty:

1. Pop a cell `(x, y)` from the front of the deque.
2. For each possible direction `(dx, dy)` and its corresponding index `k` from 1 to 4:

1. Compute the new cell `(nx, ny)` as `(x + dx, y + dy)`.
2. Skip if `(nx, ny)` is outside the grid.
3. Calculate the cost to move to `(nx, ny)`: `0` if `k` equals `grid[x][y]`, otherwise `1`.
4. If `distance[nx][ny]` can be improved by moving from `(x, y)`, update it and append `(nx, ny)` to the front if cost is 0, otherwise to the back.
6. Return `distance[m-1][n-1]` as the minimum cost to create a valid path.

**Why it works:** This algorithm guarantees that each cell is visited in increasing order of the total cost to reach it, because the deque ensures that 0-cost moves are prioritized. The distance array stores the minimum cost for each cell, so the value at the target `(m-1, n-1)` is the minimum cost overall.

## Python Solution

```python
from collections import deque
from typing import List

class Solution:
    def minCost(self, grid: List[List[int]]) -> int:
        m, n = len(grid), len(grid[0])
        distance = [[float('inf')] * n for _ in range(m)]
        distance[0][0] = 0
        
        # Direction mapping: 1->right, 2->left, 3->down, 4->up
        dirs = [(0,1), (0,-1), (1,0), (-1,0)]
        
        dq = deque([(0, 0)])
        
        while dq:
            x, y = dq.popleft()
            for k, (dx, dy) in enumerate(dirs, 1):
                nx, ny = x + dx, y + dy
                if 0 <= nx < m and 0 <= ny < n:
                    cost = 0 if grid[x][y] == k else 1
                    if distance[x][y] + cost < distance[nx][ny]:
                        distance[nx][ny] = distance[x][y] + cost
                        if cost == 0:
                            dq.appendleft((nx, ny))
                        else:
                            dq.append((nx, ny))
        
        return distance[m-1][n-1]
```

This implementation initializes the distance matrix, sets up the deque for BFS, and iteratively updates distances using 0-1 BFS. Movements that follow the current cell's direction are prioritized by adding them to the front of the deque, while direction changes are added to the back with cost 1.

## Go Solution

```go
package main

import (
    "container/list"
    "math"
)

func minCost(grid [][]int) int {
    m, n := len(grid), len(grid[0])
    distance := make([][]int, m)
    for i := 0; i < m; i++ {
        distance[i] = make([]int, n)
        for j := 0; j < n; j++ {
            distance[i][j] = math.MaxInt32
        }
    }
    distance[0][0] = 0

    dirs := [][2]int{{0,1},{0,-1},{1,0},{-1,0}}
    dq := list.New()
    dq.PushBack([2]int{0, 0})

    for dq.Len() > 0 {
        front := dq.Front()
        dq.Remove(front)
        x, y := front.Value.([2]int)[0], front.Value.([2]int)[1]
        for k, d := range dirs {
            nx, ny := x + d[0], y + d[1]
            if nx >= 0 && nx < m && ny >= 0 && ny < n {
                cost := 1
                if grid[x][y] == k + 1 {
                    cost = 0
                }
                if distance[x][y] + cost < distance[nx][ny] {
                    distance[nx][ny] = distance[x][y] + cost
                    if cost == 0 {
                        dq.PushFront([2]int{nx, ny})
                    } else {
                        dq.PushBack([2]int{nx, ny})
                    }
                }
            }
        }
    }

    return distance[m-1][n-1]
}
```

The Go implementation mirrors the Python version, using `container/list` as a deque. Costs are updated similarly, and movements in the correct direction are pushed to the front of the deque.

## Worked Examples

**Example 1:** `grid = [[1,1,1,1],[2,2,2,2],[1,1,1,1],[2,2,2,2]]`

Starting at `(0,0)` with distance 0. Move right along the first row (cost 0). At `(0,3)`, down movement required, cost 1, append to back. Continue BFS updating distances along paths that follow or change directions. Minimum cost eventually reaches `3` at `(3,3)`.

**Example 2:** `grid = [[1,1,3],[3,2,2],[1,1,4]]`

Following the signs leads naturally to `(2,2)` without changing any directions, so minimum cost is `0`.

**Example 3:** `grid = [[1,2],[4,3]]`

The shortest path requires changing one arrow to complete a valid path, so minimum cost is `1`.

## Complexity Analysis

| Measure | Complexity | Explanation |

| Time | O(m * n) | Each cell is visited at most once with 0-1 BFS, edges per cell are constant (4) |
| Space | O(m * n) | Distance matrix and deque storage for BFS |

The algorithm is efficient because 0-1 BFS processes each cell a limited number of times, avoiding unnecessary recomputation.
