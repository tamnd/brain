## Solution

## Problem Understanding

This problem asks us to compute the sum of the values of the _deepest leaves_ in a binary tree. A leaf is a node without children, and the deepest leaves are those farthest from the root. The input is a binary tree, represented by its root node. Each node contains an integer value, and left and right child references.

The expected output is a single integer, representing the sum of all node values that reside at the maximum depth in the tree. For example, if the tree has depth 4, all nodes at depth 4 are considered deepest leaves, and their values are summed.

The constraints specify that the tree contains between 1 and 10,000 nodes and node values are between 1 and 100. This tells us that the input can be moderately large, so algorithms with linear or near-linear time complexity are feasible, but anything quadratic in the number of nodes may be too slow. The tree is guaranteed to have at least one node, so we do not need to handle an empty tree.

Important edge cases include trees where all nodes form a straight line (maximum depth equals the number of nodes), trees with a single node (root is also a leaf), and trees where the deepest leaves are not on the same branch. Our solution must correctly handle these variations.

## Approaches

The brute-force approach is to traverse the tree twice. First, we compute the maximum depth of the tree using any traversal method, such as DFS or BFS. Then, in a second traversal, we sum the values of all nodes that are exactly at this maximum depth. This works because by definition, the deepest leaves reside at the maximum depth. However, it requires traversing all nodes twice, which can be inefficient.

A key insight for an optimal solution is that we can combine these steps into a single traversal. Using either a depth-first search (DFS) or a breadth-first search (BFS), we can keep track of the current depth and update a running sum whenever we reach a new maximum depth. BFS naturally processes the tree level by level, so the sum of the last level is the sum of the deepest leaves. DFS requires keeping track of the current depth and the maximum depth seen so far.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(h) | Two traversals of the tree: first to compute max depth, second to sum deepest leaves. h = tree height |
| Optimal | O(n) | O(w) | Single traversal; BFS uses a queue storing one level at a time, DFS uses recursion stack up to tree height. w = maximum width of tree |

## Algorithm Walkthrough

For the BFS-based optimal solution, the algorithm proceeds as follows:

1. Initialize a queue with the root node. This queue will store nodes level by level.
2. Initialize a variable `level_sum` to zero. This will store the sum of node values for the current level.
3. While the queue is not empty, do the following:

1. Reset `level_sum` to zero at the start of each level.
2. Determine the number of nodes at the current level (`level_size`) by checking the length of the queue.
3. Iterate through all nodes at this level:

1. Pop the node from the front of the queue.
2. Add the node's value to `level_sum`.
3. If the node has a left child, append it to the queue.
4. If the node has a right child, append it to the queue.
4. When the loop completes, `level_sum` contains the sum of the deepest leaves because BFS processes the tree level by level from top to bottom.

Why it works: BFS ensures that nodes are processed level by level. By overwriting `level_sum` at each level, we guarantee that after the last iteration, `level_sum` contains only the sum of the deepest leaves.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
from collections import deque
from typing import Optional

class Solution:
    def deepestLeavesSum(self, root: Optional[TreeNode]) -> int:
        if not root:
            return 0
        
        queue = deque([root])
        level_sum = 0
        
        while queue:
            level_sum = 0
            level_size = len(queue)
            for _ in range(level_size):
                node = queue.popleft()
                level_sum += node.val
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
        
        return level_sum
```

The implementation first handles the edge case of an empty tree, although the problem guarantees at least one node. The queue stores nodes for BFS traversal. For each level, `level_sum` is reset and computed, and child nodes are queued for the next level. After processing all levels, `level_sum` contains the sum of the deepest leaves.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func deepestLeavesSum(root *TreeNode) int {
    if root == nil {
        return 0
    }
    
    queue := []*TreeNode{root}
    var levelSum int
    
    for len(queue) > 0 {
        levelSum = 0
        levelSize := len(queue)
        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]
            levelSum += node.Val
            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }
    }
    
    return levelSum
}
```

In Go, the BFS queue is represented using a slice of pointers to `TreeNode`. We manage the queue manually by slicing off the first element and appending children. The logic mirrors the Python implementation closely. Go handles `nil` for empty children, and integer overflow is not a concern because node values are bounded.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited exactly once in BFS. |
| Space | O(w) | Queue stores at most the maximum width of the tree. |

The time complexity is linear because every node is processed once. Space complexity depends on the maximum number of nodes at any single level, which in a balanced tree is O(n/2) at the last level, simplifying to O(n) in the worst case.
