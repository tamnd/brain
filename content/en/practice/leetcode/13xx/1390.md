## Problem Understanding

The problem asks us to process an array of integers, `nums`, and compute the sum of divisors for those integers that have **exactly four divisors**. In other words, for each number in the array, we need to check how many positive integers divide it evenly, and if that count is exactly four, we sum all those divisors and add them to a running total. The final result is this total sum. If no integer has exactly four divisors, we return 0.

The input `nums` is an array of integers where each integer `nums[i]` is between 1 and 10^5, and the array length is up to 10^4. This indicates that a naive O(n * num) approach that checks every number up to `nums[i]` for divisibility could be too slow, especially for large values of `nums[i]`.

Edge cases include numbers with fewer than four divisors (like primes or perfect squares of primes), repeated numbers, and the number 1, which only has one divisor. Our solution must efficiently distinguish numbers with exactly four divisors without performing unnecessary calculations.

## Approaches

The **brute-force approach** would iterate over each number in `nums` and check all numbers from 1 to that number to count divisors. While it correctly counts divisors, it has poor performance for large inputs since checking divisibility up to `n` for each number leads to O(n * max(nums)) time complexity, which is too slow for the constraints.

The **key observation for an optimal solution** is that a number with exactly four divisors must be either of the form `p^3` (where `p` is prime, divisors are 1, p, p^2, p^3) or `p * q` (where `p` and `q` are distinct primes, divisors are 1, p, q, p*q). This allows us to avoid iterating through all numbers up to `num` to count divisors. We only need to check factors up to the square root of each number, and if exactly four divisors exist, sum them.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * max(nums)) | O(1) | Check all numbers from 1 to num for divisibility for each element in nums |
| Optimal | O(n * √max(nums)) | O(1) | Only check divisors up to √num, sum if exactly 4 divisors, using math property |

## Algorithm Walkthrough

1. Initialize a variable `total_sum` to 0. This will store the sum of divisors of all numbers with exactly four divisors.
2. Iterate over each number `num` in `nums`.
3. For each number, initialize a list `divisors` to store divisors found so far.
4. Loop `i` from 1 to the square root of `num` inclusive.
5. If `num % i == 0`, add `i` to the `divisors` list. If `i` is not equal to `num // i`, also add `num // i`.
6. If the length of `divisors` is exactly 4, sum all divisors and add to `total_sum`.
7. Continue to the next number.
8. After processing all numbers, return `total_sum`.

**Why it works**: By only iterating up to the square root of each number and adding both `i` and `num // i` when `i` divides `num`, we efficiently capture all divisors. Checking that the count of divisors is exactly 4 ensures we only include numbers meeting the problem's requirement.

## Python Solution

```python
from typing import List
import math

class Solution:
    def sumFourDivisors(self, nums: List[int]) -> int:
        total_sum = 0
        
        for num in nums:
            divisors = []
            for i in range(1, int(math.isqrt(num)) + 1):
                if num % i == 0:
                    divisors.append(i)
                    if i != num // i:
                        divisors.append(num // i)
                if len(divisors) > 4:
                    break
            if len(divisors) == 4:
                total_sum += sum(divisors)
        
        return total_sum
```

**Explanation**: We loop through each number in `nums`, collecting divisors up to its square root. If we detect more than 4 divisors, we break early to save time. Only numbers with exactly 4 divisors contribute to `total_sum`. Using `math.isqrt` ensures we compute the integer square root efficiently and correctly.

## Go Solution

```go
import (
    "math"
)

func sumFourDivisors(nums []int) int {
    totalSum := 0
    
    for _, num := range nums {
        divisors := []int{}
        sqrtNum := int(math.Sqrt(float64(num)))
        
        for i := 1; i <= sqrtNum; i++ {
            if num % i == 0 {
                divisors = append(divisors, i)
                if i != num / i {
                    divisors = append(divisors, num / i)
                }
            }
            if len(divisors) > 4 {
                break
            }
        }
        
        if len(divisors) == 4 {
            sum := 0
            for _, d := range divisors {
                sum += d
            }
            totalSum += sum
        }
    }
    
    return totalSum
}
```

**Explanation**: The Go version mirrors the Python approach. We compute integer square roots using `math.Sqrt` and convert to `int`. We append divisors to a slice and sum them if exactly four are found. The logic handles repeated elements and ensures numbers with fewer or more than four divisors are ignored.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * √max(nums)) | For each number, we loop only up to its square root |
| Space | O(√max(nums)) | Temporary list of divisors for each number |

The time complexity is efficient because even in the worst case of `nums[i] = 10^5`, √10^5 ≈ 316 iterations per number is manageable given `n ≤ 10^4`. Space usage is minimal since divisors are stored temporarily for each number.
