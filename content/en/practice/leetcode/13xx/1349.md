## Solution

## Problem Understanding

The problem gives us a classroom represented as a 2D matrix called `seats`. Each position in the matrix represents a seat. A seat can either be usable, represented by `'.'`, or broken, represented by `'#'`.

We want to place as many students as possible into the usable seats while ensuring that no student can cheat. According to the problem statement, a student can see the answers of students sitting:

- Directly left
- Directly right
- Upper left diagonal
- Upper right diagonal

A student cannot see the student directly in front or directly behind. That means vertical adjacency is completely allowed.

The goal is to compute the maximum number of students that can be seated without violating these visibility rules.

The matrix dimensions satisfy:

- `1 <= m <= 8`
- `1 <= n <= 8`

These constraints are extremely important. Since both dimensions are at most 8, each row contains at most 8 seats. This strongly suggests that bitmasking is feasible because every row configuration can be represented using an integer with at most 8 bits.

A naive brute-force solution would attempt all possible seat assignments across all cells. Since there can be up to 64 seats, that would require exploring up to `2^64` configurations, which is completely infeasible.

Several edge cases are important:

- A classroom where all seats are broken
- A classroom with only one row
- A classroom with only one column
- Sparse valid seats where many masks become invalid
- Cases where a locally optimal placement blocks better future rows
- Situations where diagonal conflicts matter more than horizontal conflicts

The problem guarantees that the matrix is non-empty and only contains `'.'` and `'#'`.

## Approaches

### Brute Force Approach

The brute-force solution considers every possible way to place students into usable seats. For every seat, we either place a student or do not place a student. After generating a configuration, we verify whether any cheating occurs.

This guarantees correctness because every possible arrangement is explored. The best valid arrangement is therefore the answer.

However, this approach is far too slow. With at most 64 seats, the total number of configurations becomes `2^64`, which is computationally impossible.

Even pruning invalid configurations early does not reduce the search space enough.

### Key Insight

The important observation is that the cheating constraints only involve:

- The current row
- The previous row

No interaction exists with rows further away.

This means we can process the classroom row by row using dynamic programming.

Because each row has at most 8 seats, we can represent a seating arrangement for a row as a bitmask. For example:

- `101001` means students sit in seats 0, 3, and 5
- `0` means no students sit in that row

For each row, we generate all valid masks:

- Students only occupy usable seats
- No two students sit adjacent horizontally

Then we transition between rows:

- Current row mask must not conflict diagonally with previous row mask

This converts the problem into a compact DP over row states.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^(m*n)) | O(m*n) | Tries every possible seating arrangement |
| Optimal Bitmask DP | O(m * 4^n) | O(2^n) | Uses row-state dynamic programming with bitmasks |

## Algorithm Walkthrough

1. Convert each row into a bitmask representing usable seats.

For every row, we create an integer where:

- `1` means the seat is usable
- `0` means the seat is broken

This allows fast bitwise validation later.
2. Generate all valid seating masks for each row.

A mask is valid if:

- Students only sit in usable seats
- No two students are adjacent horizontally

The horizontal condition is checked using:

$(mask\ \&\ (mask >> 1)) = 0$

If this expression is zero, no adjacent students exist.
3. Count the number of students in every valid mask.

We repeatedly need the number of set bits in a mask. Precomputing this avoids repeated work during DP transitions.
4. Define the DP state.

Let:

`dp[row][mask]`

represent the maximum number of students that can be seated up to `row`, where the current row uses seating configuration `mask`.
5. Initialize the first row.

Any valid mask for the first row can be used directly because there is no previous row to conflict with.
6. Transition between rows.

For each valid current-row mask and previous-row mask:

- Ensure no upper-left cheating
- Ensure no upper-right cheating

These checks are:

$(curr\ \&\ (prev >> 1)) = 0$

and

$((curr >> 1)\ \&\ prev) = 0$

If both conditions hold, the configurations are compatible.
7. Update the DP value.

The transition becomes:

`dp[row][curr] = max(dp[row][curr], dp[row-1][prev] + students_in_curr)`
8. Return the maximum value from the final row.

### Why it works

The DP works because the cheating constraints only depend on the current row and the immediately previous row. Every row configuration is evaluated against all compatible previous configurations. Since all valid row states are explored and all legal transitions are considered, the algorithm guarantees that the optimal arrangement is found.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def maxStudents(self, seats: List[List[str]]) -> int:
        rows = len(seats)
        cols = len(seats[0])

        usable_masks = []

        for row in seats:
            mask = 0
            for c in range(cols):
                if row[c] == '.':
                    mask |= (1 << c)
            usable_masks.append(mask)

        valid_masks_per_row = []

        for row_mask in usable_masks:
            valid_masks = []

            for mask in range(1 << cols):
                # Students only in usable seats
                if (mask & row_mask) != mask:
                    continue

                # No adjacent students in same row
                if mask & (mask >> 1):
                    continue

                valid_masks.append(mask)

            valid_masks_per_row.append(valid_masks)

        dp = defaultdict(int)

        # Initialize first row
        for mask in valid_masks_per_row[0]:
            dp[mask] = bin(mask).count('1')

        # Process remaining rows
        for r in range(1, rows):
            next_dp = defaultdict(int)

            for curr_mask in valid_masks_per_row[r]:
                curr_students = bin(curr_mask).count('1')

                for prev_mask, prev_value in dp.items():
                    # Check diagonal conflicts
                    if curr_mask & (prev_mask >> 1):
                        continue

                    if (curr_mask >> 1) & prev_mask:
                        continue

                    next_dp[curr_mask] = max(
                        next_dp[curr_mask],
                        prev_value + curr_students
                    )

            dp = next_dp

        return max(dp.values(), default=0)
```

The implementation begins by converting each row into a usable-seat bitmask. This preprocessing step allows fast validation using bitwise operations.

Next, all valid seating masks are generated for every row. Invalid masks are discarded immediately if they place students in broken seats or create horizontal adjacency conflicts.

The dynamic programming table is implemented using a dictionary where:

- The key is the current row mask
- The value is the best achievable student count

For each row, every valid current mask is checked against every previous-row mask. If no diagonal conflicts exist, the DP state is updated.

The final answer is the maximum value remaining after processing all rows.

## Go Solution

```go
package main

func maxStudents(seats [][]byte) int {
	rows := len(seats)
	cols := len(seats[0])

	usableMasks := make([]int, rows)

	for r := 0; r < rows; r++ {
		mask := 0

		for c := 0; c < cols; c++ {
			if seats[r][c] == '.' {
				mask |= (1 << c)
			}
		}

		usableMasks[r] = mask
	}

	validMasksPerRow := make([][]int, rows)

	for r := 0; r < rows; r++ {
		rowMask := usableMasks[r]

		for mask := 0; mask < (1 << cols); mask++ {
			// Students only in usable seats
			if (mask & rowMask) != mask {
				continue
			}

			// No adjacent students
			if (mask & (mask >> 1)) != 0 {
				continue
			}

			validMasksPerRow[r] = append(validMasksPerRow[r], mask)
		}
	}

	dp := map[int]int{}

	for _, mask := range validMasksPerRow[0] {
		dp[mask] = countBits(mask)
	}

	for r := 1; r < rows; r++ {
		nextDP := map[int]int{}

		for _, currMask := range validMasksPerRow[r] {
			currStudents := countBits(currMask)

			for prevMask, prevValue := range dp {
				if (currMask & (prevMask >> 1)) != 0 {
					continue
				}

				if ((currMask >> 1) & prevMask) != 0 {
					continue
				}

				candidate := prevValue + currStudents

				if candidate > nextDP[currMask] {
					nextDP[currMask] = candidate
				}
			}
		}

		dp = nextDP
	}

	answer := 0

	for _, value := range dp {
		if value > answer {
			answer = value
		}
	}

	return answer
}

func countBits(x int) int {
	count := 0

	for x > 0 {
		count += x & 1
		x >>= 1
	}

	return count
}
```

The Go implementation follows the same logic as the Python solution. Since Go does not provide a built-in popcount for plain integers without importing additional packages, a helper function `countBits` is used.

Maps are used for DP state storage, similar to Python dictionaries. Integer overflow is not a concern because the maximum answer is at most 64.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(m * 4^n) | For each row, all row masks may be compared against all previous-row masks |
| Space | O(2^n) | DP stores states for one row at a time |

The number of possible masks for a row is `2^n`. For each row, we may compare every valid current mask against every valid previous mask, leading to `O(2^n * 2^n) = O(4^n)` transitions per row.

Since `n <= 8`, the maximum number of masks is only 256, making this approach very efficient in practice.

## Edge Cases

### All Seats Are Broken

If every seat contains `'#'`, then no valid seating arrangement exists. A buggy implementation might accidentally assume at least one valid mask exists or fail when computing the maximum DP value.

This implementation handles the case correctly because every row still includes mask `0`, meaning no students seated. The final answer therefore becomes `0`.

### Single Column Classroom

In a one-column classroom, horizontal adjacency and diagonal conflicts disappear entirely because no neighboring columns exist.

A common mistake is accidentally treating vertical neighbors as invalid, even though the problem explicitly allows students directly in front or behind each other.

This implementation only checks horizontal and diagonal conflicts, so all usable seats in a single column can safely contain students.

### Dense Rows with Many Conflicts

Rows containing many usable seats can create a large number of possible masks. Incorrect implementations may allow adjacent students or diagonal conflicts because of faulty bit operations.

This solution explicitly validates:

- Horizontal adjacency using `mask & (mask >> 1)`
- Diagonal conflicts using shifted comparisons with the previous row

These checks guarantee that every DP transition remains valid.
