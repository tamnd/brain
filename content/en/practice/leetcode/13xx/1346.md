## Solution

## Problem Understanding

This problem asks us to determine whether, in a given integer array `arr`, there exist two distinct indices `i` and `j` such that the element at index `i` is exactly twice the element at index `j`. In other words, we want to check if for any pair of elements, one element is double the other.

The input array `arr` can contain both positive and negative integers, including zero, and its length ranges from 2 to 500. Each element is within the range `-10^3` to `10^3`. The output is a boolean: `true` if such a pair exists, and `false` otherwise.

Important edge cases include arrays containing zeros (since 0 doubled is still 0), negative numbers, or multiple duplicates. These can trip up naive implementations if one does not carefully check for distinct indices.

## Approaches

The brute-force approach is straightforward: for each element in the array, compare it with every other element to see if it is double. This approach is guaranteed to work because it checks all possible pairs, but it is inefficient with a time complexity of `O(n^2)`, which is acceptable for small arrays but unnecessary given the constraints.

The key insight for a better solution is that we can use a hash table to track seen elements. As we iterate through the array, for each element `x`, we check whether `2 * x` or `x / 2` (if `x` is even) has already been seen. This allows us to find a valid pair in `O(n)` time with `O(n)` extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check all pairs of elements for the condition |
| Optimal | O(n) | O(n) | Use a hash set to track seen elements and check for double or half |

## Algorithm Walkthrough

1. Initialize an empty hash set called `seen` to store elements we have already visited.
2. Iterate through each element `num` in the array.
3. For each `num`, check whether `2 * num` exists in the `seen` set. This checks if `num` is half of some previously seen element.
4. If `num` is even, check whether `num / 2` exists in the `seen` set. This checks if `num` is double some previously seen element.
5. If either condition is true, return `True` immediately since a valid pair has been found.
6. If neither condition is true, add `num` to the `seen` set and continue.
7. If the loop finishes without returning, no valid pair exists, so return `False`.

Why it works: At every iteration, the `seen` set contains all previous numbers. By checking for both `2 * num` and `num / 2` (when `num` is even), we ensure that any pair satisfying `arr[i] == 2 * arr[j]` will be detected. This avoids the need for nested loops while guaranteeing correctness.

## Python Solution

```python
from typing import List

class Solution:
    def checkIfExist(self, arr: List[int]) -> bool:
        seen = set()
        for num in arr:
            if 2 * num in seen or (num % 2 == 0 and num // 2 in seen):
                return True
            seen.add(num)
        return False
```

In this Python implementation, `seen` is a set that allows constant-time lookup. We iterate over each number, check for the double or half conditions, and immediately return `True` if found. Otherwise, we add the current number to the set and continue.

## Go Solution

```go
func checkIfExist(arr []int) bool {
    seen := make(map[int]struct{})
    for _, num := range arr {
        if _, ok := seen[2*num]; ok {
            return true
        }
        if num%2 == 0 {
            if _, ok := seen[num/2]; ok {
                return true
            }
        }
        seen[num] = struct{}{}
    }
    return false
}
```

In Go, we use a map as a set, storing each element as a key. The empty struct consumes no memory. The logic mirrors Python: check for double or half, then add the current number to the set.

## Edge Cases

One important edge case is when the array contains zero. Since 0 doubled is 0, the algorithm must check that there are at least two zeros to return `True`. Our set-based approach handles this because the first zero is added to `seen`, and the second zero triggers the `2*num` condition.

Another edge case is negative numbers. The algorithm correctly identifies pairs like `-4` and `-8` because the doubling and halving checks are valid for negative integers, and the set tracks all seen numbers regardless of sign.

A third edge case is arrays with duplicates. If a number appears multiple times, the set ensures we can match a double or half correctly. For example, `[1,1,2]` will return `True` because the second `1` will detect `2` in the array.

This comprehensive handling ensures correctness across the full range of inputs defined by the constraints.
