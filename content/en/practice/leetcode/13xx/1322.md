## Solution

## Problem Understanding

The problem asks us to calculate the **click-through rate (CTR)** for each advertisement in an `Ads` table. The `Ads` table contains three columns: `ad_id`, `user_id`, and `action`. Each row represents a user interaction with a specific ad. The `action` column can take one of three values: `'Clicked'`, `'Viewed'`, or `'Ignored'`. The **CTR** is defined as the ratio of clicks to total interactions that are either clicks or views, expressed as a percentage and rounded to two decimal places. Interactions with `'Ignored'` do not contribute to the CTR calculation.

We are required to return the result as a table with columns `ad_id` and `ctr`, sorted first by `ctr` in descending order and then by `ad_id` in ascending order to break ties.

Key points and constraints include:

1. **Primary key:** `(ad_id, user_id)` ensures each user interacts with a particular ad at most once.
2. **CTR calculation:** Only clicks and views count; ignored actions are excluded.
3. **Sorting:** CTR descending, ad_id ascending in case of ties.
4. **Rounding:** CTR must be rounded to two decimal points.
5. **Edge case:** Ads with no clicks or views should have a CTR of `0.00`.

Important edge cases include ads with only `'Ignored'` actions, ads with zero interactions, or ads where all interactions are clicks or all are views.

## Approaches

The **brute-force approach** involves iterating through all rows of the `Ads` table and maintaining two counters per ad: one for clicks and one for views. After processing the entire table, we compute the CTR for each ad by dividing the number of clicks by the sum of clicks and views and multiplying by 100. Finally, sort the result. This works, but if implemented in a programming language without SQL aggregation support, it requires extra bookkeeping with dictionaries or nested loops.

The **optimal approach** uses SQL aggregation functions. We can **group by `ad_id`**, count the clicks and views using conditional aggregation, and compute CTR directly in the query. This method leverages SQL's built-in optimizations and requires a single pass through the table. The query ensures rounding, and the `ORDER BY` clause handles the sorting requirements.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(a) | Iterate manually over all rows, track clicks and views per ad, then compute CTR |
| Optimal | O(n) | O(a) | Use SQL GROUP BY and conditional aggregation to compute CTR efficiently |

## Algorithm Walkthrough

1. **Filter actions:** Identify actions that contribute to CTR. Specifically, clicks and views matter, ignored actions are irrelevant.
2. **Aggregate per ad:** Use SQL `GROUP BY ad_id` to collect data for each ad. Within each group, count the number of `'Clicked'` actions and the number of `'Viewed'` actions.
3. **Compute CTR:** For each `ad_id`, calculate CTR as `(number of clicks) / (number of clicks + number of views) * 100`. Use `ROUND` to two decimal places to comply with the output format.
4. **Handle zero denominator:** Ads with no clicks or views will have zero CTR. Use `CASE WHEN` to avoid division by zero and ensure the CTR is `0.00`.
5. **Sort results:** Apply `ORDER BY ctr DESC, ad_id ASC` to meet the output requirements.

**Why it works:** Grouping by `ad_id` ensures all interactions for an ad are considered together. Conditional aggregation correctly counts clicks and views. The formula for CTR directly implements the problem definition. Rounding ensures the output matches the required format, and sorting guarantees the result order.

## Python Solution

```python
# SQL solution via Python interface (LeetCode executes the SQL directly)
# No class/method stub required for LeetCode SQL problem, just the query
SELECT
    ad_id,
    ROUND(
        CASE 
            WHEN SUM(CASE WHEN action IN ('Clicked', 'Viewed') THEN 1 ELSE 0 END) = 0 
            THEN 0
            ELSE SUM(CASE WHEN action = 'Clicked' THEN 1 ELSE 0 END) * 100.0 /
                 SUM(CASE WHEN action IN ('Clicked', 'Viewed') THEN 1 ELSE 0 END)
        END, 2
    ) AS ctr
FROM Ads
GROUP BY ad_id
ORDER BY ctr DESC, ad_id ASC;
```

**Explanation:**

This query first uses a `CASE` expression to count clicks and views per `ad_id`. The numerator counts `'Clicked'` actions, and the denominator counts all interactions that are either `'Clicked'` or `'Viewed'`. `ROUND` ensures the CTR is displayed to two decimal places. The `ORDER BY` clause ensures the correct ordering according to CTR and `ad_id`.

## Go Solution

```go
// This problem is SQL-based and typically does not require a Go solution.
// If implementing via Go with database/sql, the SQL query remains the same.
// Pseudocode in Go for executing the query:

import (
    "database/sql"
    "fmt"
    _ "github.com/go-sql-driver/mysql"
)

func GetAdCTR(db *sql.DB) ([]struct{
    AdID int
    CTR float64
}, error) {
    query := `
        SELECT
            ad_id,
            ROUND(
                CASE 
                    WHEN SUM(CASE WHEN action IN ('Clicked', 'Viewed') THEN 1 ELSE 0 END) = 0 
                    THEN 0
                    ELSE SUM(CASE WHEN action = 'Clicked' THEN 1 ELSE 0 END) * 100.0 /
                         SUM(CASE WHEN action IN ('Clicked', 'Viewed') THEN 1 ELSE 0 END)
                END, 2
            ) AS ctr
        FROM Ads
        GROUP BY ad_id
        ORDER BY ctr DESC, ad_id ASC;
    `
    rows, err := db.Query(query)
    if err != nil {
        return nil, err
    }
    defer rows.Close()

    var results []struct{
        AdID int
        CTR float64
    }
    for rows.Next() {
        var r struct{
            AdID int
            CTR float64
        }
        if err := rows.Scan(&r.AdID, &r.CTR); err != nil {
            return nil, err
        }
        results = append(results, r)
    }
    return results, nil
}
```

**Go-specific notes:** The Go version requires handling the SQL connection and scanning results into a struct. Unlike Python, Go is statically typed, so we must predefine the struct for results. SQL query logic remains identical.
