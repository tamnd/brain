## Solution

## Problem Understanding

The problem asks us to find the length of the longest substring in which every vowel appears an even number of times. The vowels we care about are `'a'`, `'e'`, `'i'`, `'o'`, and `'u'`.

A substring is a contiguous section of the string. For a substring to be valid, each vowel must occur either zero times or an even number of times. Consonants do not matter and can appear any number of times.

For example, consider the substring `"leetminicowor"` from the first example. Inside this substring:

- `'e'` appears 2 times
- `'i'` appears 2 times
- `'o'` appears 2 times
- `'a'` and `'u'` appear 0 times

All vowel counts are even, so the substring is valid.

The input consists of a single lowercase English string `s`. The output is a single integer representing the maximum possible length of a valid substring.

The constraints are important:

- `1 <= s.length <= 5 * 10^5`

This means the string can contain up to 500,000 characters. Any algorithm worse than linear or near-linear time will likely time out. In particular, checking all substrings explicitly is not feasible because there are `O(n^2)` substrings.

Several edge cases are important to recognize immediately:

- A string with no vowels at all should return the full string length because zero counts are even.
- A string where vowels appear odd numbers of times globally may still contain large valid substrings internally.
- Single-character vowel strings like `"a"` should return `0` because one occurrence is odd.
- Multiple repeated vowels like `"aa"` should return `2`.
- The empty-vowel state at the beginning of the string is significant and must be handled carefully.

## Approaches

### Brute Force Approach

The most direct approach is to examine every possible substring and check whether all vowel counts are even.

For each starting index `i`, we can extend the substring to every ending index `j`, maintaining counts of the five vowels. After constructing each substring, we verify whether every vowel count is even. If it is, we update the maximum length.

This approach is correct because it explicitly checks every substring and therefore cannot miss the optimal answer.

However, the performance is far too slow for the given constraints. There are `O(n^2)` substrings, and even if vowel counting is optimized incrementally, we still process quadratic numbers of substring boundaries. With `n = 500,000`, this becomes completely impractical.

### Optimal Approach

The key observation is that we do not actually care about the exact counts of vowels. We only care whether each count is even or odd.

This naturally suggests using parity tracking with bit manipulation.

We represent the parity state of the five vowels using a 5-bit integer:

- Bit 0 represents `'a'`
- Bit 1 represents `'e'`
- Bit 2 represents `'i'`
- Bit 3 represents `'o'`
- Bit 4 represents `'u'`

Each bit indicates whether the corresponding vowel count is currently odd (`1`) or even (`0`).

As we scan through the string:

- Encountering a vowel flips its corresponding bit using XOR.
- Consonants leave the state unchanged.

The crucial insight is this:

If the same parity state appears at two different indices, then the substring between those indices has even counts for all vowels.

Why? Because the parity changes between the two positions cancel out completely.

This transforms the problem into finding the largest distance between two identical states.

We use a hash map to store the earliest index where each state first appeared. Whenever we see the same state again, we compute the substring length.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every substring explicitly |
| Optimal | O(n) | O(1) | Uses prefix parity states with bit masking |

## Algorithm Walkthrough

1. Create a bitmask integer called `state` initialized to `0`.

A state of `0` means all vowel counts are currently even. Each bit in the integer tracks whether a vowel has appeared an odd number of times so far.
2. Create a hash map called `first_seen`.

This map stores the earliest index where each state appeared. Initialize it with:

```
{0: -1}
```

This is important because a valid substring may begin at index `0`.
3. Create a mapping from vowels to bit positions.

For example:

```
{
    'a': 0,
    'e': 1,
    'i': 2,
    'o': 3,
    'u': 4
}
```
4. Iterate through the string character by character.

At each index:

- If the character is a vowel, flip its corresponding bit using XOR.
- If it is a consonant, do nothing.

Example for `'e'`:

```
state ^= (1 << 1)
```
5. After updating the state, check whether this state has appeared before.

- If it has, then the substring between the previous occurrence and the current index has even counts for all vowels.
- Compute the length and update the answer.
6. If the state has not appeared before, store the current index.

We only store the first occurrence because earlier indices produce longer substrings.
7. Continue until the entire string has been processed.
8. Return the maximum substring length found.

### Why it works

The algorithm relies on a prefix parity invariant.

For every position, the bitmask represents whether each vowel has appeared an even or odd number of times from the start of the string up to that index.

If two positions have the same parity state, then the vowel parity changes between them must cancel out. Therefore, the substring between those positions contains even counts for every vowel.

By storing the earliest occurrence of each state, we maximize the possible substring length for every repeated state.

## Python Solution

```python
class Solution:
    def findTheLongestSubstring(self, s: str) -> int:
        vowel_to_bit = {
            'a': 0,
            'e': 1,
            'i': 2,
            'o': 3,
            'u': 4
        }

        first_seen = {0: -1}

        state = 0
        max_length = 0

        for index, char in enumerate(s):
            if char in vowel_to_bit:
                bit_position = vowel_to_bit[char]
                state ^= (1 << bit_position)

            if state in first_seen:
                current_length = index - first_seen[state]
                max_length = max(max_length, current_length)
            else:
                first_seen[state] = index

        return max_length
```

The implementation begins by creating a mapping from vowels to bit positions. This allows each vowel parity to be tracked inside a single integer.

The `first_seen` dictionary stores the earliest index where each parity state occurred. The initialization `{0: -1}` handles substrings that begin at the start of the string.

As we iterate through the string, vowels toggle their corresponding bits using XOR. XOR is ideal because it flips parity cleanly:

- Even → Odd
- Odd → Even

After updating the state, we check whether it has appeared before. If it has, the substring between the previous occurrence and the current index must contain even vowel counts.

We only store the first occurrence of a state because that produces the longest possible substring when the state reappears later.

The algorithm processes each character exactly once, giving linear runtime.

## Go Solution

```go
func findTheLongestSubstring(s string) int {
	vowelToBit := map[byte]int{
		'a': 0,
		'e': 1,
		'i': 2,
		'o': 3,
		'u': 4,
	}

	firstSeen := map[int]int{
		0: -1,
	}

	state := 0
	maxLength := 0

	for i := 0; i < len(s); i++ {
		char := s[i]

		if bitPosition, exists := vowelToBit[char]; exists {
			state ^= (1 << bitPosition)
		}

		if firstIndex, exists := firstSeen[state]; exists {
			currentLength := i - firstIndex

			if currentLength > maxLength {
				maxLength = currentLength
			}
		} else {
			firstSeen[state] = i
		}
	}

	return maxLength
}
```

The Go implementation follows exactly the same logic as the Python version.

One implementation detail is that Go strings are indexed as bytes, so we use `byte` keys in the vowel mapping.

Go does not provide Python-style dictionary membership checks directly, so we use the idiomatic two-value map lookup:

```go
value, exists := map[key]
```

The rest of the algorithm remains unchanged. Integer overflow is not a concern because the bitmask only uses 5 bits.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is processed exactly once |
| Space | O(1) | At most 32 possible parity states exist |

The runtime is linear because we perform constant-time operations for every character.

The space complexity is technically constant because there are only `2^5 = 32` possible vowel parity states. Even though we use a hash map, its size can never exceed 32 entries.

## Edge Cases

One important edge case is a string with no vowels at all, such as `"bcbcbc"`. A naive implementation might incorrectly assume vowels must appear at least once. However, zero is an even number, so the entire string is valid. In the parity-based solution, the state never changes from `00000`, so the algorithm correctly returns the full length.

Another critical edge case is a single vowel character like `"a"`. The vowel count is odd, so no valid non-empty substring exists. The algorithm handles this naturally because the parity state changes from `00000` to `00001` and never repeats, resulting in a maximum length of `0`.

A third subtle case is when the longest valid substring begins at index `0`. For example, `"aa"` should return `2`. Without initializing `first_seen[0] = -1`, the algorithm would fail to measure substrings starting from the beginning correctly. This initialization effectively treats the empty prefix before the string as having all-even vowel counts.

Another tricky scenario involves repeated parity states appearing multiple times. The algorithm must preserve the earliest occurrence of each state because earlier indices produce longer substrings. Overwriting earlier positions would lead to shorter results and incorrect answers.
