## Solution

## Problem Understanding

The problem is asking us to find the minimum number of jumps required to reach the last index of an integer array `arr`, starting from the first index. Each jump can either go to an adjacent index (`i + 1` or `i - 1`) or to any other index `j` in the array where the value matches the current index (`arr[i] == arr[j]`). The challenge lies in efficiently navigating both adjacency-based jumps and value-based jumps, especially given the potential for large arrays up to 50,000 elements.

The input is an integer array with potentially negative, zero, or positive values, and the output is a single integer representing the minimum number of steps to reach the end. Important edge cases include arrays of length 1 (already at the last index), arrays with repeated values that allow direct jumps to the end, and arrays where jumps to equal values must be chosen carefully to avoid unnecessary steps.

## Approaches

A brute-force approach would attempt to explore all possible paths using a recursive or depth-first search, keeping track of the number of jumps. While correct in theory, this approach is far too slow for the problem constraints because the number of possible paths grows exponentially with array length.

The key insight for an optimal solution is to use **Breadth-First Search (BFS)**. BFS is ideal because it explores all possible jumps level by level, ensuring the first time we reach the last index is the minimum number of steps. To handle jumps to equal values efficiently, we can maintain a hash map that maps each value to all indices containing that value. Once we use a value-based jump, we can clear its indices from the map to avoid redundant processing.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Recursively explores all jump combinations; impractical for large arrays |
| Optimal | O(n) | O(n) | BFS with a value-to-indices map ensures each index is visited once |

## Algorithm Walkthrough

1. Build a hash map where each key is a value in `arr` and the value is a list of all indices containing that value. This allows O(1) access to all possible same-value jumps.
2. Initialize a queue for BFS with the starting index 0, and a visited set to track indices we have already processed.
3. Initialize a variable `steps` to 0, which counts the number of jumps made.
4. While the queue is not empty, process all indices at the current BFS level:

1. Dequeue an index `i`.
2. If `i` is the last index, return `steps` because we have reached the goal in minimum jumps.
3. Explore index `i + 1` and `i - 1` if within bounds and not visited. Add them to the queue and mark as visited.
4. Explore all indices `j` in the hash map where `arr[j] == arr[i]` and `j` is not visited. Add them to the queue and mark as visited.
5. Clear the entry for `arr[i]` in the hash map to prevent redundant jumps in future BFS iterations.
5. Increment `steps` after processing all nodes at the current level.

**Why it works:** BFS guarantees that the first time we reach the last index, it is through the shortest path because BFS explores all nodes reachable in `k` steps before considering nodes reachable in `k + 1` steps. The visited set ensures that no index is processed more than once, and clearing the value-to-indices map prevents unnecessary revisiting.

## Python Solution

```python
from collections import deque, defaultdict
from typing import List

class Solution:
    def minJumps(self, arr: List[int]) -> int:
        if len(arr) == 1:
            return 0
        
        value_to_indices = defaultdict(list)
        for i, val in enumerate(arr):
            value_to_indices[val].append(i)
        
        queue = deque([0])
        visited = set([0])
        steps = 0
        
        while queue:
            for _ in range(len(queue)):
                current = queue.popleft()
                
                if current == len(arr) - 1:
                    return steps
                
                # Adjacent jumps
                for neighbor in [current - 1, current + 1]:
                    if 0 <= neighbor < len(arr) and neighbor not in visited:
                        visited.add(neighbor)
                        queue.append(neighbor)
                
                # Value-based jumps
                for neighbor in value_to_indices[arr[current]]:
                    if neighbor not in visited:
                        visited.add(neighbor)
                        queue.append(neighbor)
                
                # Prevent future redundant jumps
                value_to_indices[arr[current]].clear()
            
            steps += 1
        
        return -1
```

The Python code first constructs the value-to-indices map, then performs BFS starting from index 0. We explore both adjacent and equal-value jumps. Clearing the map ensures we do not revisit equal-value nodes unnecessarily. The algorithm terminates once the last index is dequeued.

## Go Solution

```go
package main

func minJumps(arr []int) int {
    n := len(arr)
    if n == 1 {
        return 0
    }
    
    valueToIndices := make(map[int][]int)
    for i, val := range arr {
        valueToIndices[val] = append(valueToIndices[val], i)
    }
    
    queue := []int{0}
    visited := make([]bool, n)
    visited[0] = true
    steps := 0
    
    for len(queue) > 0 {
        nextQueue := []int{}
        for _, current := range queue {
            if current == n-1 {
                return steps
            }
            
            neighbors := []int{current - 1, current + 1}
            for _, neighbor := range neighbors {
                if neighbor >= 0 && neighbor < n && !visited[neighbor] {
                    visited[neighbor] = true
                    nextQueue = append(nextQueue, neighbor)
                }
            }
            
            for _, neighbor := range valueToIndices[arr[current]] {
                if !visited[neighbor] {
                    visited[neighbor] = true
                    nextQueue = append(nextQueue, neighbor)
                }
            }
            
            valueToIndices[arr[current]] = nil
        }
        queue = nextQueue
        steps++
    }
    
    return -1
}
```

The Go version is similar to Python, using slices for the queue and a boolean slice for visited indices. The map tracks indices with the same value. The next level of BFS is built using a temporary slice `nextQueue`, and clearing the value map ensures no redundant jumps.

| 2 | [2,3,5] | {0,1,2,3,4,5} | Jump from 1 and 4 to neighbors and value matches |
| 3 | [9] | {0..9} | Jump from 3 or 404 match to last index |

Steps = 3

Steps = 1

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each index is visited at most once, adjacent jumps are constant time, value-based jumps are cleared after use |
| Space | O(n) | BFS queue, visited set, and value-to-indices map all scale linearly with array size |

The linear time complexity is guaranteed because after an index is visited, it is never revisited, and clearing the value-to-indices map ensures we do not repeatedly process indices with the same value.
