## Solution

## Problem Understanding

The problem is asking us to modify a given palindromic string by changing exactly **one character** so that the result is **no longer a palindrome**. Additionally, among all possible valid modifications, the output must be the **lexicographically smallest** string possible. A palindrome is a string that reads the same forwards and backwards, such as `"abba"` or `"racecar"`. The input string consists only of lowercase English letters, and its length can range from 1 to 1000 characters.

The expected output is a string where exactly one character has been changed such that the string is no longer a palindrome, and it is lexicographically smallest. If the string cannot be changed to meet the requirements (e.g., a single-character string), we return an empty string.

Key constraints and observations include:

- Single-character strings cannot be modified to become non-palindromes, so the output should be empty.
- The length of the string is manageable (up to 1000), allowing linear scans.
- Lexicographical order matters, meaning replacing a character with `'a'` is generally preferable to `'b'`, `'c'`, etc., if it maintains the non-palindrome property.
- The palindrome property ensures symmetry, so we only need to consider the first half of the string when trying to make minimal changes.

Important edge cases include:

- Single-character strings, which cannot be modified.
- Strings consisting entirely of `'a'` characters, which require changing the last character to `'b'`.
- Strings with an odd length where the middle character does not affect palindrome symmetry.

## Approaches

### Brute Force

A brute-force approach would involve iterating through each character of the palindrome and trying to replace it with every other lowercase letter (`'a'` to `'z'`) to check if the result is non-palindromic. For each replacement, we would compare the string to its reverse to determine if it is still a palindrome. We would then track the lexicographically smallest valid result. This approach is correct but inefficient because it requires `O(n * 26)` operations for a string of length `n` and generates many unnecessary comparisons.

### Optimal Approach

The key insight is that to achieve the **lexicographically smallest** result, we want to replace the first non-`'a'` character in the **first half of the string** with `'a'`. This guarantees the smallest lexicographical change while ensuring the string is no longer a palindrome. If the string is made entirely of `'a'` characters, changing the **last character** to `'b'` breaks the palindrome with minimal lexicographical impact.

This method reduces the problem to a single linear scan of the first half of the string, making it highly efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * 26) | O(n) | Try replacing each character with all 25 other letters and check for palindrome |
| Optimal | O(n) | O(n) | Scan first half, replace first non-'a' with 'a', otherwise change last character |

## Algorithm Walkthrough

1. If the length of the string is `1`, return an empty string because no valid change exists.
2. Convert the string to a mutable array of characters to allow modification.
3. Iterate through the first half of the string (`0` to `n//2 - 1`). For each character:

- If it is not `'a'`, replace it with `'a'` and return the joined string immediately. This guarantees the lexicographically smallest result.
4. If the loop completes without returning, all characters in the first half are `'a'`. Replace the **last character** with `'b'` to break the palindrome.
5. Return the modified string.

**Why it works:** The algorithm ensures that we make the **smallest possible lexicographical change** in the earliest position that can break the palindrome. The early exit guarantees optimality, and handling all `'a'` strings with the last-character change handles edge cases correctly.

## Python Solution

```python
class Solution:
    def breakPalindrome(self, palindrome: str) -> str:
        n = len(palindrome)
        if n == 1:
            return ""
        
        chars = list(palindrome)
        for i in range(n // 2):
            if chars[i] != 'a':
                chars[i] = 'a'
                return "".join(chars)
        
        chars[-1] = 'b'
        return "".join(chars)
```

**Explanation:** We first check for the single-character case. Then, by converting the string to a list, we can modify characters directly. We scan only the first half because changes in the second half would mirror the first, preserving the palindrome. The first non-`'a'` character is replaced with `'a'` for minimal lexicographical impact. If all first-half characters are `'a'`, we change the last character to `'b'`.

## Go Solution

```go
func breakPalindrome(palindrome string) string {
    n := len(palindrome)
    if n == 1 {
        return ""
    }

    chars := []byte(palindrome)
    for i := 0; i < n/2; i++ {
        if chars[i] != 'a' {
            chars[i] = 'a'
            return string(chars)
        }
    }

    chars[n-1] = 'b'
    return string(chars)
}
```

**Go notes:** Similar to Python, we handle single-character strings first. Strings in Go are immutable, so we convert to a byte slice for modification. We iterate only through the first half, and if no replacement is made, the last character is set to `'b'`.

## Edge Cases

1. **Single-character strings**: Strings like `"a"` or `"z"` cannot be modified to break the palindrome. The implementation immediately returns an empty string, handling this boundary correctly.
2. **Strings of all 'a' characters**: Palindromes like `"aaa"` or `"aaaaa"` need to have the **last character changed** to `'b'` since changing any first-half `'a'` does not produce a smaller lexicographical result. The code explicitly handles this by falling back to modifying the last character.
3. **Odd-length palindromes with middle character**: In palindromes like `"aba"` or `"racecar"`, the middle character does not affect symmetry. The algorithm only scans the first half, so the middle character is only changed if necessary. This ensures correctness while keeping the lexicographical result minimal.
