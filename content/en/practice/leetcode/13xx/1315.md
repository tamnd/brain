# LeetCode 1315 - Sum of Nodes with Even-Valued Grandparent

## Problem Understanding

The problem gives us the root of a binary tree and asks us to compute the sum of all nodes whose grandparent has an even value.

A grandparent of a node is defined as the parent of its parent. For any node in the tree, we need to determine whether its grandparent exists and whether that grandparent’s value is even. If both conditions are true, we add the node’s value to the final answer.

The input is represented as a binary tree. Each node contains:

- An integer value
- A left child pointer
- A right child pointer

The output is a single integer representing the total sum of all nodes that satisfy the condition.

The constraints tell us several important things:

- The tree can contain up to `10^4` nodes.
- Node values range from `1` to `100`.

Since the tree can become fairly large, we need an algorithm that visits each node efficiently. A solution worse than linear time would be unnecessary and potentially inefficient.

A few important observations and edge cases emerge immediately:

- The root node can never contribute to the answer because it has no parent or grandparent.
- Nodes at depth `1` also cannot contribute because they do not have grandparents.
- A tree with only one node should return `0`.
- The tree may be highly unbalanced, resembling a linked list.
- Multiple nodes can share the same even-valued grandparent, and all such descendants must be counted independently.

The core challenge is efficiently determining whether a node’s grandparent is even while traversing the tree.

## Approaches

### Brute Force Approach

A straightforward brute-force solution would examine every node individually and determine whether its grandparent is even.

One way to do this is:

1. Traverse every node in the tree.
2. For each node, search from the root again to determine its parent and grandparent.
3. If the grandparent exists and its value is even, add the node’s value to the answer.

This works because every node is checked independently and accurately. However, it is inefficient because determining parent and grandparent relationships repeatedly causes redundant traversals.

If there are `n` nodes, and each lookup potentially takes `O(n)` time, the total complexity becomes `O(n^2)` in the worst case.

This is unnecessarily slow because the ancestor information is already available during traversal.

### Optimal Approach

The key observation is that while traversing the tree, we already know the current node and can easily pass ancestor information downward.

Instead of recomputing grandparents repeatedly, we can carry:

- The parent value
- The grandparent value

during a Depth-First Search traversal.

At each node:

- If the grandparent exists and is even, add the current node’s value to the answer.
- Continue traversing children while updating ancestor information.

This allows every node to be processed exactly once.

The algorithm becomes linear because each node is visited one time and the work done per node is constant.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(h) | Repeatedly searches ancestor relationships |
| Optimal | O(n) | O(h) | Passes parent and grandparent information during DFS |

Here, `h` represents the height of the tree.

## Algorithm Walkthrough

### Optimal DFS Algorithm

1. Start a Depth-First Search from the root node.

We recursively traverse the binary tree because DFS naturally keeps track of ancestor relationships.
2. For each recursive call, pass three pieces of information:

- The current node
- The parent value
- The grandparent value

This allows every node to immediately know whether its grandparent is even.
3. Check whether the current node exists.

If the node is `None`, return `0` because there is nothing to contribute.
4. Determine whether the grandparent value is even.

If `grandparent % 2 == 0`, add the current node’s value to the result.
5. Recursively process the left subtree.

The current node becomes the parent for the next level, and the current parent becomes the grandparent.
6. Recursively process the right subtree using the same logic.
7. Return the sum accumulated from:

- The current node contribution
- The left subtree
- The right subtree

### Why it works

The algorithm maintains an important invariant throughout recursion:

- Every recursive call correctly knows the current node’s parent and grandparent.

Because of this invariant, when processing a node we can immediately determine whether its grandparent is even. Since every node is visited exactly once and evaluated using correct ancestor information, the final accumulated sum is correct.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def sumEvenGrandparent(self, root: Optional[TreeNode]) -> int:
        def dfs(
            node: Optional[TreeNode],
            parent: int,
            grandparent: int
        ) -> int:
            if not node:
                return 0

            current_sum = 0

            if grandparent % 2 == 0:
                current_sum += node.val

            current_sum += dfs(node.left, node.val, parent)
            current_sum += dfs(node.right, node.val, parent)

            return current_sum

        return dfs(root, 1, 1)
```

The implementation uses a recursive helper function named `dfs`.

The helper receives:

- The current node
- The parent value
- The grandparent value

The recursion terminates when a `None` node is reached.

For every valid node, the code checks whether the grandparent value is even. If it is, the current node’s value contributes to the total sum.

The recursive calls update ancestor information naturally:

- The current node value becomes the next parent value.
- The current parent value becomes the next grandparent value.

The initial call uses odd placeholder values (`1`) for both parent and grandparent so the root and its children are never incorrectly counted.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func sumEvenGrandparent(root *TreeNode) int {
    var dfs func(node *TreeNode, parent int, grandparent int) int

    dfs = func(node *TreeNode, parent int, grandparent int) int {
        if node == nil {
            return 0
        }

        currentSum := 0

        if grandparent%2 == 0 {
            currentSum += node.Val
        }

        currentSum += dfs(node.Left, node.Val, parent)
        currentSum += dfs(node.Right, node.Val, parent)

        return currentSum
    }

    return dfs(root, 1, 1)
}
```

The Go implementation mirrors the Python solution closely.

The primary difference is explicit handling of `nil` pointers instead of Python’s `None`.

Go integers are sufficient because the maximum possible sum is well within the range of a standard `int`.

The recursive closure is declared using `var dfs func(...)` so the function can recursively reference itself.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every node is visited exactly once |
| Space | O(h) | Recursive call stack proportional to tree height |

The DFS traversal processes each node exactly one time, so the runtime is linear in the number of nodes.

The auxiliary space comes from recursion. In the worst case of a completely skewed tree, the recursion depth becomes `O(n)`. In a balanced tree, it becomes `O(log n)`.

## Edge Cases

### Tree With Only One Node

A single-node tree has no parent and no grandparent relationships. A naive implementation might accidentally count the root if ancestor placeholders are not handled carefully.

This implementation avoids the issue by initializing parent and grandparent values with odd numbers. Since odd numbers fail the even check, the root never contributes incorrectly.

### Highly Skewed Tree

A tree may resemble a linked list where every node only has one child. Such structures are important because recursion depth becomes large.

The algorithm still works correctly because each recursive call maintains accurate ancestor tracking regardless of tree shape.

### Multiple Nodes Sharing the Same Even Grandparent

An even-valued grandparent may contribute several descendants to the final sum.

For example:

```
      2
     / \
    3   5
   / \
  7   9
```

Both `7` and `9` must be counted independently.

The DFS traversal naturally handles this because each descendant node performs its own grandparent check during traversal.
