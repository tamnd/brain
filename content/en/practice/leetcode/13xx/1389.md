## Solution

## Problem Understanding

This problem asks us to construct a _target_ array by inserting elements from one array `nums` into positions specified by another array `index`. Initially, the _target_ array is empty. For each element `nums[i]`, we insert it at position `index[i]` in the current _target_ array. The insertion is done sequentially from left to right, following the order of the arrays.

The input arrays `nums` and `index` are of equal length, ranging from 1 to 100 elements. Each `nums[i]` is an integer between 0 and 100, and `index[i]` is guaranteed to be a valid position within the current length of the _target_ array (`0 <= index[i] <= i`). The constraints are small enough that a straightforward approach is feasible, but we still want to be efficient.

Important edge cases include when the array has only one element, when all elements are inserted at the beginning (`index[i] = 0` for all `i`), or when elements are inserted at the end sequentially (`index[i] = i`). The problem guarantees no invalid insertions, so we do not need to handle out-of-bounds errors.

## Approaches

The simplest way to solve this problem is the brute-force approach: start with an empty list and iterate through each element of `nums` and `index`. For each pair `(nums[i], index[i])`, insert the element `nums[i]` into the current _target_ array at position `index[i]`. In Python, the `list.insert()` method performs this operation efficiently for small arrays. In Go, we need to manually shift elements in a slice to make room for the new element.

This approach works correctly because each insertion is done exactly as described, and the order of operations is preserved. The main downside is that each insertion into a list requires shifting elements to the right, which is an O(n) operation per insertion. Since we perform this for each of the `n` elements, the total time complexity is O(n^2).

Given the constraints (`n <= 100`), this brute-force approach is acceptable. There is no strictly faster method in this case because each insertion operation is inherently sequential and affects the positions of subsequent elements. The key insight is that the `insert()` method already handles the necessary shifting internally, so we do not need additional data structures or complex algorithms.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Insert each element at the specified index, shifting elements as needed |
| Optimal | O(n^2) | O(n) | Same as brute force; small constraints make this acceptable |

## Algorithm Walkthrough

1. Initialize an empty list called `target`. This will store the final array.
2. Iterate over the arrays `nums` and `index` simultaneously.
3. For each element `nums[i]` and its corresponding position `index[i]`, insert `nums[i]` into the `target` array at the specified position.
4. Continue until all elements have been processed.
5. Return the fully constructed `target` array.

Why it works: The algorithm maintains the invariant that after processing the first `i` elements, the `target` array contains exactly those elements in the correct positions. Each insertion operation preserves the order of previously inserted elements, and the guarantee that `index[i] <= i` ensures no invalid insertions occur.

## Python Solution

```python
from typing import List

class Solution:
    def createTargetArray(self, nums: List[int], index: List[int]) -> List[int]:
        target: List[int] = []
        for i in range(len(nums)):
            target.insert(index[i], nums[i])
        return target
```

The Python implementation uses a single list `target` and iterates through `nums` and `index`. The `insert()` method handles element shifting automatically. Each iteration places the current number at the correct position, maintaining the relative order of previously inserted numbers. After processing all elements, the `target` array is returned.

## Go Solution

```go
func createTargetArray(nums []int, index []int) []int {
    target := make([]int, 0, len(nums))
    for i := 0; i < len(nums); i++ {
        pos := index[i]
        target = append(target, 0) // increase length by 1
        copy(target[pos+1:], target[pos:]) // shift elements to the right
        target[pos] = nums[i]
    }
    return target
}
```

In Go, slices do not have a built-in insert method, so we manually shift elements to make space for the new element. We first append a dummy element to increase the slice length, then use `copy()` to shift elements from the insertion index to the right. Finally, we place the new element in the correct position. This preserves the relative order of all elements as in the Python solution.

## Edge Cases

One important edge case is when the array has only one element. In this situation, the algorithm must correctly handle an empty target array and insert the first element without error. Our implementation naturally handles this, since inserting at position 0 into an empty list is valid.

A second edge case occurs when all elements are inserted at the start (`index[i] = 0`). This requires shifting every previously inserted element one position to the right on each insertion. The Python `list.insert()` and Go `copy()` methods correctly handle this shifting, preserving the order of inserted elements.
