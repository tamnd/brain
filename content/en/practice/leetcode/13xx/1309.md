## Solution

## Problem Understanding

The problem asks us to take a string `s` composed of digits (`'0'` to `'9'`) and the `'#'` character and convert it into a string of lowercase English letters according to a defined mapping. The mapping has two parts. Single digits `'1'` to `'9'` map to letters `'a'` to `'i'`, while two-digit numbers followed by a `'#'`-specifically `'10#'` to `'26#'`-map to letters `'j'` to `'z'`. The string `s` is guaranteed to be valid, so we do not need to handle malformed inputs.

The input represents a compact encoding of letters. Each character in the decoded string may come from a single digit or a two-digit number with a `'#'`. The expected output is the resulting string after applying all these mappings.

The constraints specify that `1 <= s.length <= 1000`, meaning the input string can be long but not prohibitively so. Each string is valid, meaning that every `'#'` will appear in the context of a valid two-digit number and there are no ambiguous sequences. Edge cases could include strings composed entirely of single-digit letters, strings composed entirely of `'#'`-encoded letters, or mixed sequences ending with single digits. The guarantee of a unique mapping simplifies our solution because we do not need to consider ambiguities.

## Approaches

A brute-force approach would iterate through the string `s` from left to right and attempt to match substrings of length 3 whenever a `'#'` is encountered. This works by checking if the current character or a substring ending in `'#'` corresponds to a letter. While this is correct, it requires repeatedly slicing and parsing the string, which is slightly inefficient for longer strings. However, for the given constraints (length up to 1000), it is actually feasible.

The key observation for an optimal solution is that every `'#'` signifies a two-digit letter, and letters not followed by `'#'` are single-digit. By iterating **from the end of the string backward**, we can immediately identify whether a character belongs to a two-digit code or a single-digit code without extra lookahead checks. This approach allows direct parsing without repeatedly slicing substrings, which is cleaner and more efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Iterate left to right, check for `'#'`, slice substrings accordingly. |
| Optimal | O(n) | O(n) | Iterate right to left, detect `'#'`, convert numbers to characters, and build result efficiently. |

## Algorithm Walkthrough

1. Initialize an empty list `result` to store the decoded characters.
2. Start iterating through the string `s` **from right to left** using an index `i` initialized to the last character.
3. At each step, check if `s[i]` is `'#'`. If it is, the preceding two characters `s[i-2]` and `s[i-1]` form a two-digit number.
4. Convert the two-digit string to an integer, then convert it to the corresponding letter by computing `chr(96 + number)` since `'a'` is ASCII 97.
5. Append the resulting letter to `result` and decrement `i` by 3 to skip the two digits and `'#'`.
6. If `s[i]` is not `'#'`, it represents a single-digit letter. Convert it to an integer, map it to a character with `chr(96 + number)`, append to `result`, and decrement `i` by 1.
7. After the loop finishes, the `result` list contains characters in reverse order. Reverse it and join it to form the final decoded string.

Why it works: Iterating backward guarantees that any `'#'` encountered corresponds directly to the correct two-digit number preceding it. Single-digit numbers are naturally processed when no `'#'` follows. This ensures each character is correctly mapped, and the resulting string maintains the original order after reversing the `result` list.

## Python Solution

```python
class Solution:
    def freqAlphabets(self, s: str) -> str:
        result = []
        i = len(s) - 1
        while i >= 0:
            if s[i] == '#':
                number = int(s[i-2:i])
                result.append(chr(96 + number))
                i -= 3
            else:
                number = int(s[i])
                result.append(chr(96 + number))
                i -= 1
        return ''.join(reversed(result))
```

In the implementation, we initialize an empty list `result` to build the decoded string in reverse order. The while loop ensures we process every character. When a `'#'` is found, we take the two digits before it, convert to the corresponding ASCII letter, and skip past them. Single digits are handled similarly without the `'#'`. Finally, we reverse and join the list to return the correct string.

## Go Solution

```go
func freqAlphabets(s string) string {
    result := make([]byte, 0, len(s))
    i := len(s) - 1
    for i >= 0 {
        if s[i] == '#' {
            num := (s[i-2]-'0')*10 + (s[i-1]-'0')
            result = append(result, byte('a'+num-1))
            i -= 3
        } else {
            num := s[i] - '0'
            result = append(result, byte('a'+num-1))
            i--
        }
    }
    // Reverse the result slice
    for l, r := 0, len(result)-1; l < r; l, r = l+1, r-1 {
        result[l], result[r] = result[r], result[l]
    }
    return string(result)
}
```

In Go, we use a byte slice to store characters. ASCII arithmetic converts numbers to letters. The loop is identical to Python in logic, and we reverse the slice in-place before converting to a string. Go requires explicit handling of byte vs string, and slice preallocation improves performance.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We traverse the string once from end to start, performing constant-time operations at each step. |
| Space | O(n) | The output list stores n characters in the worst case and is reversed at the end. |

The algorithm is linear in time and space because each character is processed exactly once and the final reverse operation is also O(n).
