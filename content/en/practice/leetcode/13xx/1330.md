**Topics:** Array, Math, Greedy

## Solution

## Problem Understanding

The problem asks us to maximize the "value" of an integer array, where the value is defined as the sum of absolute differences between consecutive elements:

$$\text{value} = \sum_{i=0}^{n-2} |nums[i] - nums[i+1]|$$

You are allowed to select **any contiguous subarray** and reverse it **exactly once**, and the goal is to find the maximum possible value of the resulting array after such an operation.

The input `nums` is an array of integers, potentially including negative numbers, and its length can be up to 30,000, which rules out naive solutions that examine all possible subarrays explicitly. The output is a single integer representing the maximum achievable value.

Important edge cases include arrays with all identical elements, arrays already in strictly increasing or decreasing order, arrays with only two elements, or arrays where reversing the subarray at the boundaries yields the largest gain. The problem guarantees the input array has at least two elements, and the output fits in a 32-bit integer.

## Approaches

A brute-force approach would involve considering every possible subarray, reversing it, recalculating the sum of absolute differences, and keeping track of the maximum. While this works correctly, its time complexity is $O(n^3)$ because for each subarray (O(n²)), we would compute the sum of differences (O(n)), which is far too slow for $n = 30,000$.

The key observation for an optimal solution is that **only the elements adjacent to the reversed subarray contribute to a change in the array value**. Reversing a subarray internally preserves the differences between the subarray elements except at the boundaries. Specifically, if we reverse a subarray `nums[i:j]`, the differences affected are `nums[i-1]-nums[i]` and `nums[j]-nums[j+1]` (if they exist). By focusing on boundary differences, we can compute the maximum gain efficiently without explicitly reversing all subarrays.

We also notice two cases yield maximum gain: reversing a subarray starting or ending at the array boundaries, or reversing a subarray to swap a large and small element to maximize the absolute difference across the boundaries. By tracking the minimum and maximum adjacent differences, we can compute the maximum increase in value in O(n) time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Check all subarrays, reverse, and compute total value |
| Optimal | O(n) | O(1) | Track gains at boundaries and internal max/min differences to find best reversal |

## Algorithm Walkthrough

1. Compute the initial value of the array by summing `|nums[i] - nums[i+1]|` for all `i`. This serves as the base value.
2. Initialize a variable `maxGain` to 0. This will track the maximum increase in array value achievable by reversing one subarray.
3. Iterate through all pairs of consecutive elements `(nums[i], nums[i+1])` to compute possible gains for reversing subarrays at the boundaries. For each pair, calculate the potential increase in the absolute difference sum if this boundary were used as a reversal edge.
4. Track the global minimum of `min(nums[i], nums[i+1])` and maximum of `max(nums[i], nums[i+1])` to compute the largest internal gain possible when reversing a subarray entirely within the array (not touching the boundaries).
5. Update `maxGain` by comparing gains from internal and boundary reversals.
6. Return the sum of `initialValue + maxGain` as the result.

**Why it works:**

Reversing a subarray only changes the differences at its boundaries, so by analyzing all possible boundary gains, we capture the maximum achievable increment without explicitly testing every subarray. This reduces the problem from O(n³) to O(n).

## Python Solution

```python
from typing import List

class Solution:
    def maxValueAfterReverse(self, nums: List[int]) -> int:
        n = len(nums)
        initial_value = sum(abs(nums[i] - nums[i+1]) for i in range(n - 1))
        max_gain = 0

        # Track minimum and maximum for internal reversal gain
        min_adj = float('inf')
        max_adj = -float('inf')

        for i in range(n - 1):
            a, b = nums[i], nums[i+1]
            # Possible gain at boundaries
            max_gain = max(max_gain, abs(nums[0] - b) - abs(a - b))
            max_gain = max(max_gain, abs(nums[-1] - a) - abs(a - b))

            # Track for internal reversal potential
            min_adj = min(min_adj, max(a, b))
            max_adj = max(max_adj, min(a, b))

        # Internal reversal gain
        max_gain = max(max_gain, 2 * (max_adj - min_adj))
        return initial_value + max_gain
```

The implementation first calculates the initial array value. Then, it iterates through each pair of consecutive elements to calculate potential gains for boundary reversals. It also tracks internal maximum and minimum values for internal subarray reversals. Finally, it computes the largest gain and adds it to the initial value.

## Go Solution

```go
import "math"

func maxValueAfterReverse(nums []int) int {
    n := len(nums)
    initialValue := 0
    maxGain := 0

    minAdj := math.MaxInt32
    maxAdj := math.MinInt32

    for i := 0; i < n-1; i++ {
        a, b := nums[i], nums[i+1]
        initialValue += abs(a - b)

        // Boundary reversal gain
        maxGain = max(maxGain, abs(nums[0]-b)-abs(a-b))
        maxGain = max(maxGain, abs(nums[n-1]-a)-abs(a-b))

        // Internal reversal tracking
        minAdj = min(minAdj, max(a, b))
        maxAdj = max(maxAdj, min(a, b))
    }

    maxGain = max(maxGain, 2*(maxAdj - minAdj))
    return initialValue + maxGain
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

The Go version mirrors the Python logic. It uses helper functions for `abs`, `max`, and `min` to maintain clarity. Go-specific considerations include integer bounds and handling slices efficiently.

## Worked Examples

**Example 1:** `nums = [2,3,1,5,4]`

1. Initial value: `|2-3| + |3-1| + |1-5| + |5-4| = 1 + 2 + 4 + 1 = 8`
2. Boundary gains:

- Reversing `[2,3,1]` gives `abs(2-1) - abs(2-3) = 1-1 = 0` for the start boundary, no gain
- Reversing `[3,1,5]` gives maximum gain internally `2*(3-1) = 4`
3. Total value: `8 + 2 = 10` (matches expected output)

**Example 2:** `nums = [2,4,9,24,2,1,10]`

1. Initial value: `|2-4| + |4-9| + |9-24| + |24-2| + |2-1| + |1-10| = 2+5+15+22+1+9=54`
2. Maximum gain from internal reversal: `2*(9-2)=14`
3. Total value: `54 + 14 = 68`
