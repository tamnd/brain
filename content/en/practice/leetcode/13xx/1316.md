## Solution

## Problem Understanding

The problem asks us to count the number of **distinct echo substrings** in a given string `text`. An echo substring is defined as a non-empty substring that can be represented as `a + a`, where `a` is some string. Essentially, the substring is a repetition of some smaller string twice in a row. For example, in `"abcabcabc"`, `"abcabc"` is an echo substring because it is `"abc" + "abc"`.

The input is a string `text` of length up to 2000, containing only lowercase English letters. The output is a single integer representing the number of distinct echo substrings. By "distinct," we mean that if the same echo substring occurs multiple times in `text`, it should only be counted once.

Important constraints and observations include:

- Substrings must be **non-empty**.
- Only substrings of **even length** can be echo substrings because they are the concatenation of two identical parts.
- The string length limit (2000) allows solutions that are roughly O(n²) or O(n² log n) in complexity, but anything O(n³) is likely too slow.
- Edge cases could include very short strings, strings with no repeating patterns, or strings composed entirely of a single character.

## Approaches

The brute-force approach would examine all possible substrings of `text`, check if the length is even, split it into two halves, and compare them. While this guarantees correctness, it is too slow because it involves checking O(n²) substrings and potentially O(n) comparisons for each substring, resulting in O(n³) time complexity.

A better solution uses **string hashing** or a **Trie-based approach**. The key insight is that by using a **rolling hash** (like Rabin-Karp), we can compute and compare substring hashes in O(1) time. This allows us to efficiently detect whether two halves of a substring are equal and whether an echo substring is distinct.

Using a hash set ensures that we only count distinct substrings. We iterate through all possible even lengths and all possible starting positions, compute hashes for the first half and the second half, and check for equality.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(n²) | Compare each even-length substring by splitting into two halves |
| Optimal (Rolling Hash) | O(n²) | O(n²) | Use rolling hash to compare halves efficiently and hash set to track distinct substrings |

## Algorithm Walkthrough

1. Initialize a set `seen` to store distinct echo substrings.
2. Precompute powers of a chosen base modulo a large prime to support rolling hash calculations.
3. Iterate over all possible even lengths `l` of substrings from 2 up to `len(text)` with a step of 2.
4. For each starting index `i` where the substring of length `l` fits, split it into two halves.
5. Compute the rolling hashes for both halves in O(1) using precomputed prefix hashes.
6. If the hashes are equal, it indicates the halves are identical, so the substring is an echo substring.
7. Add the substring to the set `seen` to ensure distinctness.
8. After iterating through all possibilities, return the size of `seen` as the number of distinct echo substrings.

**Why it works**: Rolling hashes allow us to compare any substring in constant time, ensuring correctness in detecting repeated halves. The hash set guarantees that duplicates are not counted, so the final count represents all distinct echo substrings.

## Python Solution

```python
class Solution:
    def distinctEchoSubstrings(self, text: str) -> int:
        MOD = 10**9 + 7
        BASE = 31
        n = len(text)
        
        # Compute prefix hashes and powers of BASE
        prefix_hash = [0] * (n + 1)
        power = [1] * (n + 1)
        for i in range(n):
            prefix_hash[i + 1] = (prefix_hash[i] * BASE + ord(text[i]) - ord('a') + 1) % MOD
            power[i + 1] = (power[i] * BASE) % MOD

        def get_hash(l: int, r: int) -> int:
            # Returns hash of substring text[l:r] (exclusive)
            return (prefix_hash[r] - prefix_hash[l] * power[r - l] % MOD + MOD) % MOD

        seen = set()
        for length in range(2, n + 1, 2):
            half = length // 2
            for i in range(n - length + 1):
                if get_hash(i, i + half) == get_hash(i + half, i + length):
                    seen.add(text[i:i + length])
        return len(seen)
```

The Python code first precomputes prefix hashes and powers of a base to enable O(1) substring hash queries. It iterates over all even lengths and checks each substring by comparing the hashes of its halves. Any echo substring found is stored in a set for uniqueness. Finally, it returns the size of the set.

## Go Solution

```go
func distinctEchoSubstrings(text string) int {
    const MOD = 1_000_000_007
    const BASE = 31
    n := len(text)

    prefixHash := make([]int, n+1)
    power := make([]int, n+1)
    power[0] = 1
    for i := 0; i < n; i++ {
        prefixHash[i+1] = (prefixHash[i]*BASE + int(text[i]-'a'+1)) % MOD
        power[i+1] = (power[i] * BASE) % MOD
    }

    getHash := func(l, r int) int {
        return (prefixHash[r] - prefixHash[l]*power[r-l]%MOD + MOD) % MOD
    }

    seen := make(map[string]struct{})
    for length := 2; length <= n; length += 2 {
        half := length / 2
        for i := 0; i <= n-length; i++ {
            if getHash(i, i+half) == getHash(i+half, i+length) {
                seen[text[i:i+length]] = struct{}{}
            }
        }
    }
    return len(seen)
}
```

In Go, we use a map to track distinct substrings instead of a set. Integer arithmetic is carefully handled modulo a large prime. Slices are used to extract substrings when adding to the map, and the rolling hash logic mirrors the Python solution.
