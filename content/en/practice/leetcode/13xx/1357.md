## Solution

## Problem Understanding

This problem asks us to implement a `Cashier` system that tracks customers and applies a discount to every `nth` customer. The supermarket has a fixed catalog of products with associated prices. When a customer purchases items, the `getBill` function calculates their total cost, and, if it is the `nth` customer, applies a percentage discount to the subtotal.

The input to the `Cashier` constructor is `n`, which specifies the frequency of discount application, `discount`, the percentage of discount applied to eligible customers, and two parallel arrays `products` and `prices` representing the store catalog. Each call to `getBill` provides a customer's shopping cart via `product` and `amount` arrays, which list the product IDs and the quantity of each product bought. The output is the final bill as a floating-point number.

Constraints indicate that the maximum number of products is 200 and the number of `getBill` calls is at most 1000, so our solution must handle hundreds of calls efficiently. The problem guarantees all product IDs are unique and each product in a bill exists in the catalog, simplifying the lookup and avoiding edge cases with missing products.

Edge cases to consider include when `discount` is zero (no discount ever applies), when the discount applies to the very first customer, and when multiple products are purchased in large quantities.

## Approaches

A brute-force approach would iterate through the `products` array for each item in the customer's `product` list to look up its price. This would involve a nested iteration, giving a time complexity of O(m * p), where m is the number of items in the customer's cart and p is the total number of products. While correct, it is inefficient for repeated calls to `getBill`.

The optimal approach leverages a hash map to map product IDs to their prices. This allows constant-time price lookup per product in the customer's cart, reducing the per-bill computation to O(m), which is sufficient given the constraints.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m * p) | O(1) | Nested iteration over catalog for each product in bill |
| Optimal | O(m) | O(p) | Use hash map for O(1) price lookup, efficient for repeated calls |

## Algorithm Walkthrough

1. Initialize the `Cashier` object with `n`, `discount`, and the product catalog. Store a mapping from product ID to price for fast lookup.
2. Maintain a counter to track the number of customers processed so far.
3. In `getBill`, increment the customer counter.
4. Compute the subtotal by iterating over each product in the customer's cart, multiplying the quantity by the corresponding price from the hash map.
5. If the current customer count is a multiple of `n`, apply the discount by multiplying the subtotal by `(100 - discount) / 100`.
6. Return the final bill as a floating-point number.

**Why it works:** The hash map ensures constant-time access to product prices, guaranteeing correct subtotal calculation. The modulo operation on the customer counter correctly identifies which customers receive the discount. This invariant holds for any sequence of calls.

## Python Solution

```python
from typing import List

class Cashier:

    def __init__(self, n: int, discount: int, products: List[int], prices: List[int]):
        self.n = n
        self.discount = discount
        self.customer_count = 0
        self.price_map = {product: price for product, price in zip(products, prices)}

    def getBill(self, product: List[int], amount: List[int]) -> float:
        self.customer_count += 1
        total = sum(self.price_map[prod] * amt for prod, amt in zip(product, amount))
        if self.customer_count % self.n == 0:
            total *= (100 - self.discount) / 100
        return total
```

**Explanation:** The constructor initializes the hash map `price_map` from product IDs to prices and stores the discount frequency. `getBill` iterates over the products in the cart, multiplies quantities by prices using the hash map, checks if the discount applies, and returns the final bill.

## Go Solution

```go
type Cashier struct {
	n            int
	discount     int
	customerCount int
	priceMap     map[int]int
}

func Constructor(n int, discount int, products []int, prices []int) Cashier {
	priceMap := make(map[int]int)
	for i, product := range products {
		priceMap[product] = prices[i]
	}
	return Cashier{
		n:            n,
		discount:     discount,
		customerCount: 0,
		priceMap:     priceMap,
	}
}

func (this *Cashier) GetBill(product []int, amount []int) float64 {
	this.customerCount++
	total := 0
	for i, prod := range product {
		total += this.priceMap[prod] * amount[i]
	}
	if this.customerCount%this.n == 0 {
		return float64(total) * float64(100-this.discount) / 100.0
	}
	return float64(total)
}
```

**Go-specific notes:** The `priceMap` is a map[int]int for fast lookup. Integer arithmetic is safe here, and final conversion to float64 happens only at the end for discount computation. The customer counter is incremented before applying the discount modulo check.

## Worked Examples

### Example: Third customer with discount

Input: `product = [1,2,3,4,5,6,7]`, `amount = [1,1,1,1,1,1,1]`, `n = 3`, `discount = 50`

Step 1: Increment `customer_count` from 2 to 3.

Step 2: Compute subtotal:

```
1*100 + 1*200 + 1*300 + 1*400 + 1*300 + 1*200 + 1*100 = 1600
```

Step 3: Since `3 % 3 == 0`, apply 50% discount:

```
1600 * (100 - 50)/100 = 1600 * 0.5 = 800
```

Output: `800.0`

This pattern repeats for each call, with the modulo operation determining if discount applies.

| Time | O(m) | m is number of products in the customer's cart; hash map allows O(1) price lookup per item |
| Space | O(p) | p is number of products in catalog; hash map stores all product prices |

The algorithm is efficient because the most expensive operation is iterating over the customer's cart. The discount check is O(1), so the solution scales well even with 1000 calls to `getBill`.
