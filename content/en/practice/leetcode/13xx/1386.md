## Solution

## Problem Understanding

The problem asks us to maximize the number of four-person groups that can be seated in a cinema with `n` rows, each containing 10 seats. Some seats are already reserved, and the input `reservedSeats` lists the row and seat numbers for these reservations. A four-person group must occupy **four adjacent seats in a single row**. The adjacency rule is simple: seats across the aisle cannot be considered contiguous, except for the center split where two seats on either side of the aisle can form part of a group.

The input gives `n`, the number of rows, and `reservedSeats`, a list of already occupied seats. The output is a single integer representing the maximum number of four-person groups that can be placed in the cinema following the constraints.

Constraints are crucial here: `n` can be very large (up to 10^9), but `reservedSeats.length` is capped at `min(10*n, 10^4)`, meaning only a small fraction of seats are actually reserved. This suggests that we cannot iterate over all rows explicitly due to memory and runtime limits, and we need an approach that focuses only on rows with reservations.

Important edge cases include rows with no reservations at all, rows with scattered reserved seats preventing any group, and rows where only one group can fit in the middle segment.

## Approaches

### Brute Force

A brute-force solution would iterate over all rows from 1 to `n` and, for each row, check every possible four-seat segment to see if it is unreserved. For each segment, we would check whether seats 2-5, 4-7, or 6-9 are available, considering the aisle split for middle seats. While this guarantees the correct answer, it is **too slow** because `n` can be up to 10^9, making iteration over all rows infeasible.

### Optimal Approach

The key observation is that only rows with reserved seats need detailed checks. Any row with no reservations can seat **two groups** (one on the left 2-5 seats, one on the right 6-9 seats). Rows with reserved seats require checking which of the three possible groupings (left, middle, right) are still available. Using **bit masking** for each row, we can represent the reserved seats compactly and quickly check which seat blocks are free. This reduces the complexity from iterating over potentially billions of rows to only processing the reserved rows.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Check all rows sequentially, infeasible for large n |
| Optimal | O(r) | O(r) | Only process rows with reserved seats, r = len(reservedSeats) |

## Algorithm Walkthrough

1. **Group Mapping:** Define seat blocks for four-person groups. The left block is seats 2-5, the middle block is 4-7, and the right block is 6-9.
2. **Row Reservation Map:** Use a hash map where keys are row numbers and values are bit masks of reserved seats. Each bit corresponds to a seat being reserved or not.
3. **Mark Reserved Seats:** For each reserved seat, set the corresponding bit in the row mask. Only rows with reservations are stored.
4. **Count Groups per Reserved Row:** For each row in the map, check if the left, middle, or right block is free using bit masking.

- If both left and right blocks are free, assign **two groups**.
- Otherwise, assign **one group** if any block is free.
- Assign zero groups if all blocks intersect reserved seats.
5. **Add Fully Free Rows:** Rows not in the map are fully unreserved and can always host **two groups**.
6. **Sum Total Groups:** Combine counts from reserved and unreserved rows for the final answer.

**Why it works:** The solution leverages the sparsity of reservations. Rows with no reserved seats trivially accommodate two groups. For rows with reservations, bit masking ensures constant-time checking of the three possible groupings. This guarantees that all possible group allocations are considered efficiently.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def maxNumberOfFamilies(self, n: int, reservedSeats: List[List[int]]) -> int:
        row_map = defaultdict(int)
        for r, s in reservedSeats:
            row_map[r] |= 1 << (s - 1)
        
        total_groups = 0
        for mask in row_map.values():
            left = 0b00011110   # seats 2-5
            middle = 0b00111100 # seats 4-7
            right = 0b01111000  # seats 6-9
            count = 0
            if (mask & left) == 0:
                count += 1
            if (mask & right) == 0:
                count += 1
            if count == 0 and (mask & middle) == 0:
                count = 1
            total_groups += count
        
        # Add rows without any reservations
        total_groups += (n - len(row_map)) * 2
        return total_groups
```

The code first builds a mapping of reserved seats per row using bit masks. Then, for each row with reservations, it checks which of the three blocks are available. Fully unreserved rows are counted separately. Bit masks make the checks very efficient.

## Go Solution

```go
func maxNumberOfFamilies(n int, reservedSeats [][]int) int {
    rowMap := make(map[int]int)
    for _, seat := range reservedSeats {
        r, s := seat[0], seat[1]
        rowMap[r] |= 1 << (s - 1)
    }
    
    totalGroups := 0
    left, middle, right := 0b00011110, 0b00111100, 0b01111000
    
    for _, mask := range rowMap {
        count := 0
        if mask&left == 0 {
            count++
        }
        if mask&right == 0 {
            count++
        }
        if count == 0 && mask&middle == 0 {
            count = 1
        }
        totalGroups += count
    }
    
    totalGroups += (n - len(rowMap)) * 2
    return totalGroups
}
```

The Go implementation mirrors the Python logic. The main differences are the explicit use of slices and the syntax for bitwise operations. The map only stores rows with reservations, and bit masking efficiently checks the three blocks.
