## Solution

## Problem Understanding

The problem is asking us to maximize the sum of pizza slices you can pick when the pizza has `3n` slices arranged in a circle. Each turn, you choose one slice, then your two friends, Alice and Bob, pick the slices immediately adjacent to yours - Alice in the anti-clockwise direction and Bob in the clockwise direction. This continues until all slices are taken.

The input is an integer array `slices` of length `3n` where each element represents the size of a pizza slice in clockwise order. The output is a single integer, representing the maximum sum of slice sizes you can accumulate by choosing slices optimally.

Key constraints are: `3 * n == slices.length`, `1 <= slices.length <= 500`, and `1 <= slices[i] <= 1000`. These tell us the array can be moderately large, so an exponential or combinatorial approach is infeasible. Important edge cases include very small arrays (length 3), arrays with all slices equal, or arrays where the largest slices are consecutive. The circular nature of the array means that the first and last slices are considered adjacent, which is a subtlety that must be handled carefully.

## Approaches

### Brute Force Approach

The brute-force approach would be to try every possible sequence of selections. For each choice, you recursively remove your chosen slice and the slices your friends pick, then continue the process on the remaining slices until all are removed. This approach is guaranteed to produce the correct answer because it exhaustively explores all valid picking sequences, but it is computationally infeasible. The recursion would have complexity proportional to the number of combinations of `n` slices out of `3n` which grows extremely quickly (`O((3n choose n))`).

### Optimal Approach

The key insight is to reduce the problem to a dynamic programming problem by noticing that if you remove either the first slice or the last slice, the problem becomes a linear problem instead of a circular one. Essentially, you can solve two linear subproblems:

1. Consider slices from index `0` to `n-2` (excluding the last slice)
2. Consider slices from index `1` to `n-1` (excluding the first slice)

Then take the maximum result of these two subproblems. Each linear subproblem can be solved using a dynamic programming approach similar to the "house robber" problem, where you cannot take two adjacent elements, and you need to select exactly `n` elements.

The recurrence for the DP is:

```
dp[i][j] = max(dp[i-1][j], dp[i-2][j-1] + slices[i])
```

Here, `dp[i][j]` represents the maximum sum you can achieve using the first `i` slices while picking exactly `j` slices.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((3n choose n)) | O(3n) | Try all combinations of slices |
| Optimal | O(n^2) | O(n^2) | Dynamic programming with linearized subarray to handle circular array |

## Algorithm Walkthrough

1. Start by defining a helper function `calculate` that computes the maximum sum of slices in a linear array with exactly `n` slices to pick. This function uses dynamic programming to solve the problem.
2. Inside `calculate`, create a 2D array `dp` where `dp[i][j]` represents the maximum sum possible from the first `i` slices while picking exactly `j` slices. Initialize all values to 0.
3. Loop over all slices and for each number of slices to pick, fill the DP table using the recurrence `dp[i][j] = max(dp[i-1][j], dp[i-2][j-1] + slices[i])`.
4. Solve two subproblems: one excluding the first slice and one excluding the last slice, to handle the circular array.
5. Return the maximum value of these two subproblems.

Why it works: The DP recurrence ensures that at each slice, you either pick it (and therefore skip the previous one) or skip it, and by solving both linearized subproblems, you cover all possible optimal selections without violating the circular adjacency constraint. The invariant is that `dp[i][j]` always contains the maximum sum possible for that prefix and number of picks.

## Python Solution

```python
from typing import List

class Solution:
    def maxSizeSlices(self, slices: List[int]) -> int:
        def calculate(subslices: List[int]) -> int:
            n = len(subslices)
            pick_count = n // 3
            dp = [[0] * (pick_count + 1) for _ in range(n + 2)]
            
            for i in range(2, n + 2):
                for j in range(1, pick_count + 1):
                    dp[i][j] = max(dp[i-1][j], dp[i-2][j-1] + subslices[i-2])
            return dp[n+1][pick_count]
        
        return max(calculate(slices[:-1]), calculate(slices[1:]))
```

The implementation first defines a helper `calculate` to solve the linear DP problem. `dp[i][j]` stores the best sum using the first `i` slices while picking `j` slices. We offset `i` by 2 in the DP array to handle base cases easily. Finally, we solve two subproblems to handle circular adjacency and return the maximum.

## Go Solution

```go
func maxSizeSlices(slices []int) int {
    calculate := func(subslices []int) int {
        n := len(subslices)
        pickCount := n / 3
        dp := make([][]int, n+2)
        for i := range dp {
            dp[i] = make([]int, pickCount+1)
        }

        for i := 2; i < n+2; i++ {
            for j := 1; j <= pickCount; j++ {
                dp[i][j] = dp[i-1][j]
                if dp[i-2][j-1]+subslices[i-2] > dp[i][j] {
                    dp[i][j] = dp[i-2][j-1] + subslices[i-2]
                }
            }
        }
        return dp[n+1][pickCount]
    }

    n := len(slices)
    return max(calculate(slices[:n-1]), calculate(slices[1:]))
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

The Go implementation mirrors the Python approach, using slices and making a 2D slice for DP. The helper function `max` is used to simplify comparisons. Go-specific details include explicit slice creation and bounds handling.

## Worked Examples

**Example 1:** `slices = [1,2,3,4,5,6]`

Linear subproblems: `[1,2,3,4,5]` and `[2,3,4,5,6]`

- For `[1,2,3,4,5]`, pick slices 2 and 4 → sum = 2 + 4 = 6
- For `[2,3,4,5,6]`, pick slices 2 and 4 → sum = 4 + 6 = 10

Maximum = 10

**Example 2:** `slices = [8,9,8,6,1,1]`

Linear subproblems: `[8,9,8,6,1]` and `[9,8,6,1,1]`

- For `[8,9,8,6,1]`, pick 8 and 8 → sum = 16
- For `[9,8,6,1,1]`, pick 9 and 6 → sum = 15

Maximum = 16
