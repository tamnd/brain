## Solution

## Problem Understanding

In this problem, we are given a string `s` that contains only three possible characters: `'a'`, `'b'`, and `'c'`. We must count how many substrings contain at least one occurrence of all three characters.

A substring is a contiguous sequence of characters inside the string. For example, in the string `"abcabc"`, the substring `"bca"` is valid because it contains `'a'`, `'b'`, and `'c'`. However, `"ab"` is not valid because it does not contain `'c'`.

The task is not asking us to list the substrings, only to return the total count.

The input size can be as large as `5 * 10^4`, which means a naive solution that checks every substring individually may become too slow. Since the number of substrings in a string of length `n` is `O(n^2)`, and checking each substring may require additional work, we need a more efficient approach.

An important detail is that the string contains only three distinct characters. This greatly simplifies tracking character frequencies and enables an efficient sliding window solution.

Several edge cases are worth considering upfront:

- A string like `"abc"` contains exactly one valid substring, the entire string itself.
- A string with repeated characters at the beginning, such as `"aaacb"`, can still contain valid substrings once all three characters appear.
- Every valid substring may overlap with others, so we must count carefully without missing or double counting.
- The problem guarantees the string length is at least `3`, so we never need to handle empty strings.

## Approaches

### Brute Force Approach

The most direct approach is to generate every possible substring and check whether it contains all three characters.

We can use two nested loops:

- The outer loop selects the starting index.
- The inner loop selects the ending index.

For every substring, we can count whether `'a'`, `'b'`, and `'c'` are all present.

This approach is correct because it explicitly checks every substring and counts exactly those that satisfy the condition.

However, the performance is poor. There are `O(n^2)` substrings, and checking each substring may take up to `O(n)` time if done naively. Even with some optimizations, the total complexity remains too high for `n = 5 * 10^4`.

### Optimal Sliding Window Approach

The key observation is that once a window contains all three characters, every extension of that window to the right will also remain valid.

Suppose we have a valid substring from index `left` to `right`. Then:

- `s[left:right+1]` is valid.
- `s[left:right+2]` is also valid.
- Continuing further right will still preserve all three characters.

This means that when we find a valid window ending at position `right`, we can immediately count multiple substrings at once.

A sliding window works well here because:

- We expand the right pointer to include characters.
- Once the window contains all three characters, we shrink the left pointer while maintaining validity.
- Each valid window contributes additional substrings efficiently.

Since each character enters and leaves the window at most once, the algorithm runs in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) to O(n³) | O(1) | Checks every substring individually |
| Optimal | O(n) | O(1) | Sliding window with character frequency counting |

## Algorithm Walkthrough

### Step 1: Initialize the Sliding Window

We maintain:

- A frequency counter for `'a'`, `'b'`, and `'c'`
- A `left` pointer for the start of the window
- A running total `result`

The `right` pointer will expand across the string.

### Step 2: Expand the Window

For each character at index `right`:

- Add it to the frequency counter.
- This enlarges the current window.

At this point, the window is `s[left:right+1]`.

### Step 3: Check Whether the Window Is Valid

A window is valid if:

- Count of `'a'` > 0
- Count of `'b'` > 0
- Count of `'c'` > 0

As soon as all three are present, we know the current window satisfies the requirement.

### Step 4: Count All Valid Extensions

If the current window is valid, then every substring starting at `left` and ending at:

- `right`
- `right + 1`
- `right + 2`
- ...
- `n - 1`

will also be valid.

The number of such substrings is:

$n-right$

So we add `n - right` to the result.

### Step 5: Shrink the Window

After counting:

- Remove `s[left]` from the frequency counter.
- Increment `left`.

We continue shrinking while the window remains valid.

This ensures we count every valid starting position exactly once.

### Step 6: Continue Until the End

Repeat the process until the `right` pointer reaches the end of the string.

The accumulated result is the final answer.

### Why it works

The sliding window maintains the invariant that the current window always represents a contiguous substring of the input string.

Whenever the window contains all three characters, every longer substring obtained by extending the right boundary remains valid. By adding `n - right`, we count all such substrings in one operation.

Because each possible starting position is processed exactly once while shrinking the window, every valid substring is counted exactly once and none are missed.

## Python Solution

```python
class Solution:
    def numberOfSubstrings(self, s: str) -> int:
        n = len(s)
        
        count = {
            'a': 0,
            'b': 0,
            'c': 0
        }
        
        left = 0
        result = 0
        
        for right in range(n):
            count[s[right]] += 1
            
            while count['a'] > 0 and count['b'] > 0 and count['c'] > 0:
                result += n - right
                
                count[s[left]] -= 1
                left += 1
        
        return result
```

The implementation begins by storing the string length in `n`, since we repeatedly use it while counting valid substrings.

The `count` dictionary tracks how many times each character appears inside the current sliding window.

The `right` pointer expands the window one character at a time. Each new character is added to the frequency map.

The `while` loop checks whether the current window contains at least one `'a'`, `'b'`, and `'c'`. As long as it does:

- We add `n - right` to the answer because every extension to the right is valid.
- Then we shrink the window from the left to search for additional valid starting positions.

By shrinking immediately after counting, we avoid double counting and ensure all substrings are considered exactly once.

## Go Solution

```go
func numberOfSubstrings(s string) int {
	n := len(s)

	count := [3]int{}
	left := 0
	result := 0

	for right := 0; right < n; right++ {
		count[s[right]-'a']++

		for count[0] > 0 && count[1] > 0 && count[2] > 0 {
			result += n - right

			count[s[left]-'a']--
			left++
		}
	}

	return result
}
```

The Go implementation uses a fixed-size array instead of a hash map because there are only three possible characters. This is slightly more efficient than a map lookup.

The expression `s[right] - 'a'` converts:

- `'a'` to index `0`
- `'b'` to index `1`
- `'c'` to index `2`

Go strings are byte slices for ASCII characters, so indexing works naturally here.

The logic is otherwise identical to the Python version.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is processed at most twice, once by the right pointer and once by the left pointer |
| Space | O(1) | Only fixed-size character counts are stored |

The sliding window ensures that both pointers move only forward. Since neither pointer ever moves backward, the total number of operations is proportional to the string length.

The space usage remains constant because we only store counts for three characters regardless of input size.

## Edge Cases

One important edge case is the smallest valid input, `"abc"`. This case verifies that the algorithm correctly identifies exactly one valid substring and does not overcount. The sliding window becomes valid only once, and the formula `n - right` correctly contributes `1`.

Another important case involves repeated characters before the window becomes valid, such as `"aaaaacb"`. A buggy implementation might incorrectly count only one substring when the window first becomes valid. The sliding window correctly shrinks from the left multiple times, counting every valid starting position independently.

A third edge case is when valid substrings overlap heavily, such as `"abcabcabc"`. Many substrings share characters and valid regions. The sliding window handles this efficiently because each valid window contributes multiple substrings at once through the `n - right` calculation, while shrinking guarantees no duplicates are counted.
