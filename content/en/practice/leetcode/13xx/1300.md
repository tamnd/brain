## Solution

## Problem Understanding

This problem is asking us to find a single integer `value` such that if we replace all elements in the input array `arr` that are greater than `value` with `value` itself, the resulting array's sum is as close as possible to a given target integer `target`. The input array can contain any positive integers, and the output integer does not need to be an element of the array; it can be any integer that produces the closest sum.

For example, if `arr = [4,9,3]` and `target = 10`, choosing `value = 3` results in `[3,3,3]` with a sum of `9`, which is closer to `10` than other options like `4` or `9`. The goal is to minimize the absolute difference between the mutated array sum and the target, and if multiple integers achieve the same minimum difference, we select the smallest integer.

The constraints indicate that the array can be moderately large (`up to 10^4` elements) and elements and target values can go up to `10^5`. This rules out naive brute-force solutions that attempt to test every possible `value` between 1 and the maximum element in the array, as that could involve up to 10^9 iterations in the worst case.

Important edge cases include arrays with a single element, all elements already smaller than the target, very large elements compared to the target, and ties where multiple `value` choices produce the same closest sum.

## Approaches

### Brute Force

A straightforward brute-force approach would be to iterate over every possible `value` from 1 to the maximum element in the array. For each candidate `value`, we compute the sum of the mutated array, where all elements larger than `value` are replaced by `value`. We then keep track of the `value` that produces the minimum absolute difference from the target.

This approach is guaranteed to find the correct answer because it exhaustively tests all possible integers, but it is too slow for large arrays or large numbers because the time complexity is proportional to the product of the maximum element and the array length, potentially reaching O(10^9) operations.

### Optimal Approach

The key insight is that the sum of the mutated array is monotonic with respect to the replacement `value`. As `value` increases, the mutated array sum can only increase or stay the same because fewer elements are being capped. This monotonic property allows us to use **binary search** to efficiently find the best `value`.

Instead of testing every integer, we perform a binary search between 0 and the maximum element in `arr`. For each middle candidate, we compute the mutated sum and compare it with the target. If the sum is smaller than the target, we increase the candidate `value` to get closer; if it is larger, we decrease it. After the binary search converges, we check the final candidate and its neighbor to ensure we select the integer that gives the minimum absolute difference. This approach reduces the time complexity from linear in the maximum element to logarithmic in the maximum element multiplied by the array length.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(max(arr) * n) | O(1) | Iterate every value up to max element, compute sum each time |
| Optimal | O(n log(max(arr))) | O(1) | Binary search on value, compute mutated sum for each candidate |

## Algorithm Walkthrough

1. Sort the input array. Sorting is optional but simplifies reasoning; we can compute sums directly without scanning all elements repeatedly if needed.
2. Set the binary search range: `low = 0` and `high = max(arr)`.
3. While `low <= high`, compute the midpoint `mid = (low + high) // 2`.
4. Compute the sum of the array if all elements greater than `mid` are replaced with `mid`. This is done efficiently by iterating through the array and summing the minimum of `mid` and each element.
5. Compare the computed sum with `target`. If the sum is less than `target`, increase `low = mid + 1` to explore larger candidate values. If the sum is greater than `target`, decrease `high = mid - 1`.
6. After binary search, compare the sums for `low` and `high` candidates to determine which produces the minimum absolute difference to the target.
7. Return the smallest integer that achieves the closest sum to the target.

**Why it works:** The sum of the mutated array is a monotonic function of `value`. Binary search exploits this property to efficiently narrow down the candidate values. By checking both sides of the convergence point, we ensure we select the minimum integer in case of ties.

## Python Solution

```python
from typing import List

class Solution:
    def findBestValue(self, arr: List[int], target: int) -> int:
        def mutated_sum(value: int) -> int:
            return sum(min(x, value) for x in arr)
        
        low, high = 0, max(arr)
        best_value = high
        min_diff = float('inf')
        
        while low <= high:
            mid = (low + high) // 2
            current_sum = mutated_sum(mid)
            current_diff = abs(current_sum - target)
            
            if current_diff < min_diff or (current_diff == min_diff and mid < best_value):
                best_value = mid
                min_diff = current_diff
            
            if current_sum < target:
                low = mid + 1
            else:
                high = mid - 1
        
        return best_value
```

This Python implementation defines a helper function `mutated_sum` to compute the sum efficiently for any candidate value. We then perform binary search over the possible replacement values. The `best_value` variable tracks the integer that produces the closest sum to the target, and we ensure ties are broken by choosing the smaller integer.

## Go Solution

```go
func findBestValue(arr []int, target int) int {
    mutatedSum := func(value int) int {
        sum := 0
        for _, num := range arr {
            if num > value {
                sum += value
            } else {
                sum += num
            }
        }
        return sum
    }

    low, high := 0, 0
    for _, num := range arr {
        if num > high {
            high = num
        }
    }

    bestValue := high
    minDiff := int(1e9)

    for low <= high {
        mid := (low + high) / 2
        currentSum := mutatedSum(mid)
        currentDiff := currentSum - target
        if currentDiff < 0 {
            currentDiff = -currentDiff
        }

        if currentDiff < minDiff || (currentDiff == minDiff && mid < bestValue) {
            bestValue = mid
            minDiff = currentDiff
        }

        if currentSum < target {
            low = mid + 1
        } else {
            high = mid - 1
        }
    }

    return bestValue
}
```

In Go, we define an inline function `mutatedSum` to compute the sum after mutation. We also handle integer comparisons and absolute values manually. The rest follows the same binary search approach as in Python.
