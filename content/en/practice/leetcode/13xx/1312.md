## Solution

## Problem Understanding

This problem asks us to determine the minimum number of insertions needed to turn a given string `s` into a palindrome. A palindrome is a string that reads the same forwards and backwards, for example, `"racecar"` or `"level"`. The input is a string consisting only of lowercase English letters, with a maximum length of 500 characters. The output is a single integer representing the fewest insertions required to make `s` a palindrome.

The key constraints tell us that the input is relatively small in size (up to 500 characters), so algorithms with a time complexity up to `O(n^2)` are feasible. Edge cases include strings that are already palindromes (requiring 0 insertions), strings with no repeated characters (potentially requiring almost `n-1` insertions), and single-character strings (which are trivially palindromes).

The problem guarantees that the input string is non-empty and composed of lowercase letters, which simplifies character comparisons.

## Approaches

### Brute Force

A naive brute-force approach would attempt every possible insertion of characters at every possible position and check if the resulting string becomes a palindrome. We could recursively try to match the first and last character and explore all insertion possibilities when they do not match. While this approach is correct, it is exponentially slow (`O(2^n)`) because each mismatch can branch into two insertion possibilities. This is infeasible for strings up to length 500.

### Optimal Approach

The key insight for an optimal solution is to recognize that this problem is equivalent to finding the **Longest Palindromic Subsequence (LPS)** of the string. If we know the length of the LPS, the minimum insertions required to make the string a palindrome is:

```
min_insertions = len(s) - LPS_length
```

This is because characters not part of the LPS must be inserted in mirrored positions to create a palindrome.

To compute the LPS efficiently, we use **dynamic programming**. We define a 2D DP table `dp[i][j]` representing the length of the LPS in the substring `s[i:j+1]`. The recurrence relation is:

- If `s[i] == s[j]`, then `dp[i][j] = dp[i+1][j-1] + 2`
- If `s[i] != s[j]`, then `dp[i][j] = max(dp[i+1][j], dp[i][j-1])`

We fill this table iteratively for increasing substring lengths. The value `dp[0][n-1]` gives the length of the longest palindromic subsequence in the entire string.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Recursively try all insertions |
| Optimal | O(n^2) | O(n^2) | DP table computes LPS length |

## Algorithm Walkthrough

1. Initialize an `n x n` DP table `dp` where `dp[i][i] = 1` for all `i`, because a single character is always a palindrome of length 1.
2. Iterate over all substring lengths `l` from 2 to `n`.
3. For each substring of length `l`, define its start index `i` and end index `j = i + l - 1`.
4. If `s[i] == s[j]`, set `dp[i][j] = dp[i+1][j-1] + 2`, because the matching characters can extend the inner palindrome.
5. If `s[i] != s[j]`, set `dp[i][j] = max(dp[i+1][j], dp[i][j-1])`, because we must take the longer palindrome from either removing the start or end character.
6. After filling the table, compute `min_insertions = n - dp[0][n-1]`.

**Why it works:** Each DP entry accurately represents the longest palindromic subsequence of its corresponding substring. By considering matching or mismatched characters recursively, we ensure that all subsequences are accounted for. The subtraction from `n` gives exactly the characters that need insertion to achieve a full palindrome.

## Python Solution

```python
class Solution:
    def minInsertions(self, s: str) -> int:
        n = len(s)
        dp = [[0] * n for _ in range(n)]
        
        for i in range(n):
            dp[i][i] = 1
        
        for length in range(2, n + 1):
            for i in range(n - length + 1):
                j = i + length - 1
                if s[i] == s[j]:
                    dp[i][j] = dp[i + 1][j - 1] + 2
                else:
                    dp[i][j] = max(dp[i + 1][j], dp[i][j - 1])
        
        return n - dp[0][n - 1]
```

In this Python implementation, we first initialize a 2D list `dp` to store lengths of palindromic subsequences. We set the diagonal to 1, representing single-character palindromes. Then, for increasing substring lengths, we fill the table using the recurrence relation. Finally, we subtract the longest palindromic subsequence from the string length to find the minimum insertions required.

## Go Solution

```go
func minInsertions(s string) int {
    n := len(s)
    dp := make([][]int, n)
    for i := range dp {
        dp[i] = make([]int, n)
        dp[i][i] = 1
    }

    for length := 2; length <= n; length++ {
        for i := 0; i <= n - length; i++ {
            j := i + length - 1
            if s[i] == s[j] {
                if length == 2 {
                    dp[i][j] = 2
                } else {
                    dp[i][j] = dp[i+1][j-1] + 2
                }
            } else {
                dp[i][j] = max(dp[i+1][j], dp[i][j-1])
            }
        }
    }

    return n - dp[0][n-1]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

In Go, we allocate a slice of slices for the DP table. Go requires explicit allocation of the 2D array and initialization. The logic mirrors the Python implementation, with special handling for length 2 substrings to avoid out-of-bounds access.
