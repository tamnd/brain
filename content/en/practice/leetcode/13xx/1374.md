## Solution

## Problem Understanding

The problem asks us to construct a string of length `n` using only lowercase English letters, such that every distinct character in the string appears an odd number of times.

The input is a single integer `n`, representing the required length of the output string. The output can be any valid string that satisfies the condition. There is no requirement to minimize the number of distinct characters or produce a lexicographically smallest result.

The key condition is that every character frequency must be odd. For example:

- `"aaa"` is valid because `'a'` appears 3 times, and 3 is odd.
- `"ab"` is valid because both `'a'` and `'b'` appear once, and 1 is odd.
- `"aabb"` is invalid because both characters appear 2 times, and 2 is even.

The constraints are very small:

- `1 <= n <= 500`

This tells us that performance is not a major concern. Even inefficient solutions would likely pass. However, the problem is designed to test observation and reasoning rather than brute-force search.

An important observation is that:

- If `n` itself is odd, then repeating a single character `n` times works immediately.
- If `n` is even, then using one character `n - 1` times and another character once works, because both frequencies become odd.

Several edge cases are important:

- `n = 1`, the smallest possible input. A single character already has an odd count.
- Small even values such as `n = 2`, where using one repeated character is impossible because 2 is even.
- Large values such as `n = 500`, where the algorithm should still generate the string efficiently.

The problem guarantees that `n` is always positive, so we never need to handle invalid or zero-length inputs.

## Approaches

### Brute Force Approach

A brute-force solution would attempt to generate all possible lowercase strings of length `n` and check whether every character appears an odd number of times.

The validation step would involve counting character frequencies using a hash map and verifying that each count is odd.

This approach is correct because it eventually examines every possible string, so it must eventually find a valid one. However, it is computationally infeasible. There are `26^n` possible strings of length `n`, which grows exponentially.

Even for small values of `n`, the number of possibilities becomes enormous. Since the constraints allow `n` up to 500, brute force is completely impractical.

### Optimal Approach

The key insight is that we do not need to search for a valid arrangement because one can always be constructed directly.

If `n` is odd, then a string consisting entirely of the same character works. For example:

```
n = 5 -> "aaaaa"
```

The character `'a'` appears 5 times, which is odd.

If `n` is even, then using one character `n - 1` times and another character once guarantees both frequencies are odd:

```
n = 4 -> "aaab"
```

Here:

- `'a'` appears 3 times
- `'b'` appears 1 time

Both counts are odd.

This construction works for every valid input and requires only linear time to build the string.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(26^n * n) | O(n) | Generates all possible strings and validates frequencies |
| Optimal | O(n) | O(n) | Direct mathematical construction based on parity |

## Algorithm Walkthrough

1. Check whether `n` is odd or even.

This is the most important observation because the parity of `n` determines how we construct the string.
2. If `n` is odd, return a string containing `'a'` repeated `n` times.

Since `n` itself is odd, the frequency of `'a'` will also be odd, satisfying the requirement immediately.
3. If `n` is even, return a string containing `'a'` repeated `n - 1` times followed by `'b'`.

The value `n - 1` is odd because subtracting 1 from an even number produces an odd number. The character `'b'` appears exactly once, which is also odd.
4. Return the constructed string.

The resulting string always has length `n`, and every character frequency is odd.

### Why it works

The correctness relies entirely on parity.

- When `n` is odd, using one character `n` times produces one odd frequency.
- When `n` is even, splitting the counts into `n - 1` and `1` produces two odd frequencies.

Since every character frequency in the constructed string is odd, the output always satisfies the problem requirements.

## Python Solution

```python
class Solution:
    def generateTheString(self, n: int) -> str:
        if n % 2 == 1:
            return "a" * n
        
        return "a" * (n - 1) + "b"
```

The implementation follows the exact reasoning from the algorithm walkthrough.

The first condition checks whether `n` is odd using the modulo operator:

```python
n % 2 == 1
```

If this condition is true, the solution returns a string consisting entirely of `'a'` repeated `n` times. Since `n` is odd, the count of `'a'` is also odd.

If `n` is even, the solution constructs a string where:

- `'a'` appears `n - 1` times
- `'b'` appears once

Both frequencies are odd, so the string is valid.

String multiplication in Python makes this implementation concise and efficient.

## Go Solution

```go
func generateTheString(n int) string {
    if n%2 == 1 {
        result := ""
        
        for i := 0; i < n; i++ {
            result += "a"
        }
        
        return result
    }
    
    result := ""
    
    for i := 0; i < n-1; i++ {
        result += "a"
    }
    
    result += "b"
    
    return result
}
```

The Go implementation follows the same logic as the Python solution.

Go does not support direct string multiplication like Python, so the string is built using loops and concatenation.

The solution first checks whether `n` is odd. If it is, the code appends `"a"` exactly `n` times.

For even values of `n`, the code appends `"a"` `n - 1` times and then appends `"b"` once.

There are no concerns about integer overflow because `n` is at most 500. The resulting string is always non-empty because the constraints guarantee `n >= 1`.

## Edge Cases

One important edge case is `n = 1`. This is the smallest possible valid input. A naive implementation that always tries to use two characters could accidentally produce a string longer than necessary or introduce invalid counts. The implementation handles this correctly because `1` is odd, so it simply returns `"a"`.

Another important case is when `n` is even, especially the smallest even value `n = 2`. If we tried to use one repeated character, the frequency would be 2, which is even and therefore invalid. The implementation avoids this by splitting the counts into `1` and `1`, producing a valid result such as `"ab"`.

A third important edge case is the maximum constraint value `n = 500`. Large values can expose inefficient string-building logic in some languages. The algorithm remains efficient because it constructs the string directly in linear time. No nested loops or expensive searches are involved.

A subtle edge case is ensuring that all character frequencies remain odd simultaneously. Some naive constructions may accidentally produce one even count while fixing another. The implementation guarantees correctness mathematically by using either:

- one odd count when `n` is odd
- two odd counts when `n` is even

This invariant ensures the output is always valid.
