## Solution

## Problem Understanding

The problem is asking us to remove all **leaf nodes** from a binary tree that have a specific value, `target`. A leaf node is a node with no children, meaning both its left and right pointers are `null` (or `None` in Python). After removing such leaf nodes, their parent nodes may themselves become leaves with the value `target`, so the deletion process must be repeated until no leaf node has that value.

The input is a binary tree, represented by its root node, and an integer `target`. The output should be the root of the modified binary tree after all eligible leaf nodes have been removed.

Constraints tell us that the tree has at most 3000 nodes and node values are between 1 and 1000. This ensures that a depth-first traversal is feasible without running into performance issues. Important edge cases include trees where all nodes are leaves with the target value, trees with no nodes matching the target, and trees that become empty after deletions. The tree is guaranteed to be non-empty initially.

## Approaches

A naive approach is to repeatedly traverse the tree, check for leaf nodes with the target value, and remove them. You would continue this traversal until no leaf nodes with the target exist. This works but may repeatedly traverse the same nodes, leading to inefficiency.

The optimal approach leverages **post-order depth-first traversal**. By recursively visiting the left and right subtrees first, we ensure that we process children before their parent. When a node becomes a leaf after its children are removed, we check its value and delete it if it matches the target. This approach naturally handles the cascading deletions in a single traversal.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(h) | Repeatedly traverse tree and remove leaves; may reprocess nodes multiple times |
| Optimal | O(n) | O(h) | Post-order DFS processes children first; deletes leaves in one pass; h is tree height |

## Algorithm Walkthrough

1. Start at the root of the tree.
2. Recursively traverse the left subtree and update the left child pointer with the result of this traversal.
3. Recursively traverse the right subtree and update the right child pointer with the result of this traversal.
4. Check if the current node is a leaf (both children are null) and if its value equals the target.
5. If both conditions are true, return `null` (Python `None`) to remove the node.
6. If not, return the node itself with potentially updated children.
7. Continue this process until the recursion returns to the root.

**Why it works**: Post-order traversal ensures that we process all children before their parent. By removing leaves from the bottom up, any node that becomes a leaf as a result of child removal will also be correctly evaluated. This guarantees that all deletions, including cascading deletions, are handled in one traversal.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def removeLeafNodes(self, root: Optional[TreeNode], target: int) -> Optional[TreeNode]:
        if not root:
            return None
        
        root.left = self.removeLeafNodes(root.left, target)
        root.right = self.removeLeafNodes(root.right, target)
        
        if not root.left and not root.right and root.val == target:
            return None
        
        return root
```

This Python implementation recursively traverses the tree in post-order. Each recursive call updates the left and right children of the current node. If a node is a leaf and matches the target, it is removed by returning `None`. Otherwise, the node is returned with updated children.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func removeLeafNodes(root *TreeNode, target int) *TreeNode {
    if root == nil {
        return nil
    }
    
    root.Left = removeLeafNodes(root.Left, target)
    root.Right = removeLeafNodes(root.Right, target)
    
    if root.Left == nil && root.Right == nil && root.Val == target {
        return nil
    }
    
    return root
}
```

In Go, the approach is identical to Python. `nil` is used to represent absent children. The recursive post-order traversal ensures correct removal of leaves, including cascading deletions.

## Worked Examples

**Example 1:** `root = [1,2,3,2,null,2,4], target = 2`

| Step | Node | Left Child | Right Child | Action |
| --- | --- | --- | --- | --- |
| 1 | 2 (left) | null | null | Leaf with target -> removed |
| 2 | 2 (right of 3) | null | null | Leaf with target -> removed |
| 3 | 3 | null | 4 | Node not leaf or target -> keep |
| 4 | 1 | null | 3 | Node not leaf or target -> keep |

**Output:** `[1,null,3,null,4]`

**Example 2:** `root = [1,3,3,3,2], target = 3`

| Step | Node | Action |
| --- | --- | --- |
| 3 (left leaf) | 3 | Leaf with target -> removed |
| 3 (right leaf) | 3 | Leaf with target -> removed |
| 1 | 1 | Not leaf -> keep |

**Output:** `[1,3,null,null,2]`

**Example 3:** `root = [1,2,null,2,null,2], target = 2`

| Step | Node | Action |
| --- | --- | --- |
| Bottom-most 2 | 2 | Leaf with target -> removed |
| Middle 2 | 2 | Became leaf, value=target -> removed |
| Top 2 | 2 | Became leaf, value=target -> removed |
| 1 | 1 | Root remains |

**Output:** `[1]`

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited exactly once during post-order traversal. |
| Space | O(h) | Recursion stack depth is at most the height of the tree, h. |

The time complexity is linear because every node is visited once. Space complexity depends on the tree height due to the recursion stack; in the worst case of a skewed tree, this can be O(n).

## Edge Cases

**Single-node tree matching target**: The tree only has one node whose value matches the target. Without proper handling, code may leave the node in place. Correct handling returns `None`.

**Tree where all nodes are target leaves**: Every leaf node has the target value. As leaves are removed, their parents may become leaves themselves and must also be removed. Recursive post-order ensures these cascading deletions are handled.

**Skewed tree**: A tree where every node only has one child, either all left or all right. This tests the recursion stack depth and ensures that the algorithm does not rely on balanced trees. Proper post-order recursion still removes leaves correctly and handles depth up to 3000 safely.
