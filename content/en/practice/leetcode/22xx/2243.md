## Solution

## Problem Understanding

The problem gives us a numeric string `s` and an integer `k`. We repeatedly transform the string until its length becomes less than or equal to `k`.

In each round, we split the string into consecutive groups of size `k`. Every full group contains exactly `k` characters, except possibly the last group, which may contain fewer than `k` digits if the string length is not divisible by `k`.

For every group, we calculate the sum of its digits and convert that sum back into a string. After processing all groups, we concatenate all resulting sum strings together to form the next version of `s`.

We continue performing rounds while the current string length is greater than `k`. Once the string length is at most `k`, we stop and return the resulting string.

The input size is very small:

- `1 <= s.length <= 100`
- `2 <= k <= 100`

These constraints tell us that efficiency is not a major concern. Even a straightforward simulation is more than fast enough. The important part is implementing the transformation correctly.

Several edge cases are important:

- The last group may contain fewer than `k` characters.
- A digit sum can become multiple digits, such as `"999"` becoming `"27"`.
- The string may already satisfy `len(s) <= k`, in which case we return it immediately without processing.
- Strings containing many zeros must still preserve the correct number of groups, for example `"00000000"` becoming `"000"`.

## Approaches

### Brute Force Approach

The brute-force solution directly simulates the process exactly as described in the problem statement.

During each round:

1. Iterate through the string in chunks of size `k`.
2. Compute the digit sum for each chunk.
3. Append the string representation of the sum to a temporary result.
4. Replace `s` with the newly constructed string.
5. Repeat until `len(s) <= k`.

This approach is correct because it follows the problem specification step by step without any shortcuts or approximations.

Even though this is called "brute force", it is actually perfectly acceptable for the given constraints because the string length never exceeds 100.

### Key Insight for the Optimal Solution

The key observation is that the problem itself is inherently a simulation problem. There is no hidden mathematical shortcut or advanced data structure required.

The optimal solution is still simulation based, but implemented efficiently using string building techniques.

Instead of repeatedly creating many intermediate strings inefficiently, we process groups sequentially and build the next string incrementally.

Because each character is processed only a small number of times and the input size is tiny, this approach is both simple and optimal.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly rebuilds strings across rounds |
| Optimal | O(n²) | O(n) | Efficient simulation with incremental string construction |

## Algorithm Walkthrough

1. Start with the original string `s`.
2. Continue processing while the length of `s` is greater than `k`.
3. Create an empty list or string builder that will store the transformed groups for the current round.
4. Iterate through `s` in steps of size `k`.
5. For each group:

- Extract the substring from index `i` to `i + k`.
- Compute the sum of all digits in that group.
- Convert the sum back into a string.
- Append the resulting string to the builder.
6. After all groups are processed, concatenate the builder contents together to form the next version of `s`.
7. Repeat the process until `len(s) <= k`.
8. Return the final value of `s`.

### Why it works

Each round exactly matches the transformation rules described in the problem statement. Every group contributes its digit sum string to the next version of `s`, and the algorithm repeats until the stopping condition is satisfied.

Because the algorithm faithfully simulates every required operation in order, the produced result is guaranteed to be correct.

## Python Solution

```python
class Solution:
    def digitSum(self, s: str, k: int) -> str:
        while len(s) > k:
            next_parts = []

            for i in range(0, len(s), k):
                group = s[i:i + k]

                digit_sum = 0
                for ch in group:
                    digit_sum += int(ch)

                next_parts.append(str(digit_sum))

            s = "".join(next_parts)

        return s
```

The implementation directly mirrors the algorithm.

The outer `while` loop continues until the string length becomes at most `k`.

Inside each round, `next_parts` stores the transformed values for every group. We iterate through the string using a step size of `k`, which naturally creates consecutive groups.

For each group, we compute the sum of digits manually by converting every character into an integer and accumulating the total.

After calculating the group sum, we convert it back into a string and append it to `next_parts`.

At the end of the round, `"".join(next_parts)` creates the transformed string, which becomes the new value of `s`.

Finally, once the stopping condition is reached, we return the resulting string.

## Go Solution

```go
func digitSum(s string, k int) string {
	for len(s) > k {
		next := ""

		for i := 0; i < len(s); i += k {
			end := i + k
			if end > len(s) {
				end = len(s)
			}

			sum := 0

			for j := i; j < end; j++ {
				sum += int(s[j] - '0')
			}

			next += strconv.Itoa(sum)
		}

		s = next
	}

	return s
}
```

The Go implementation follows the same logic as the Python version.

One important difference is substring handling. Since Go strings are byte indexed, we explicitly calculate the `end` boundary for each group to avoid going out of range.

Digit conversion is done using `s[j] - '0'`, which converts a character digit into its integer value efficiently.

The function uses `strconv.Itoa` to convert sums back into strings, so the `strconv` package must be imported.

## Edge Cases

One important edge case occurs when the input string length is already less than or equal to `k`. In this situation, no rounds should be performed. A careless implementation might still attempt unnecessary processing. The solution avoids this issue because the `while` loop only executes when `len(s) > k`.

Another tricky case is when the last group contains fewer than `k` characters. Since the problem explicitly allows this, the implementation must not ignore or skip the remaining digits. Using slicing with `s[i:i+k]` in Python and boundary checks in Go ensures the final partial group is processed correctly.

A third important case involves groups whose digit sums produce multiple digits. For example, `"999"` becomes `"27"`, not a single digit. An incorrect implementation might assume every group maps to one character. The solution correctly converts sums into strings and appends the entire result, preserving all digits.

Finally, strings containing many zeros can be deceptive because repeated transformations still produce valid groups. The implementation correctly preserves zero sums as `"0"` strings and concatenates them normally, ensuring outputs like `"000"` are handled exactly as required.
