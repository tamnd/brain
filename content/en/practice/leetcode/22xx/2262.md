## Solution

## Problem Understanding

The problem asks us to calculate the **total appeal of all substrings** of a given string `s`. The **appeal** of a string is defined as the number of **distinct characters** in that string. For example, the string `"abbca"` has an appeal of `3` because it contains the distinct characters `'a'`, `'b'`, and `'c'`.

The input is a string `s` of lowercase English letters, with a length between 1 and 10^5. The output is a single integer representing the sum of the appeals of all possible substrings of `s`. A substring is any contiguous section of `s`, and because the number of substrings grows quadratically with string length, a naive approach of generating all substrings is not feasible for large `s`.

Important edge cases include strings with all identical characters, strings where each character is unique, and strings of length 1, since these can easily expose off-by-one errors or miscounting of appeals.

## Approaches

The **brute-force approach** would involve generating every substring of `s`, counting the number of distinct characters in each substring, and summing these counts. While this is correct in principle, generating all substrings explicitly leads to O(n^2) substrings, and counting distinct characters in each substring requires O(n) time. Therefore, this results in O(n^3) time complexity, which is infeasible for n up to 10^5.

The **optimal approach** relies on the observation that we can calculate each character's contribution to the total appeal based on the **substrings in which it appears as a new character**. By keeping track of the last index where each character appeared, we can determine how many substrings include that character as a new contribution. Specifically, for a character at index `i`, if its last occurrence was at `prev`, then it contributes to `(i - prev) * (n - i)` substrings, where `n` is the length of the string. Summing these contributions for all characters gives the total appeal in **linear time**.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n) | Generate all substrings and count distinct characters in each |
| Optimal | O(n) | O(26) ≈ O(1) | Use character last-seen index to compute contributions without generating substrings |

## Algorithm Walkthrough

1. Initialize a variable `total_appeal` to zero. This will store the cumulative sum of appeals.
2. Initialize an array or map `last_index` of size 26 (for each lowercase letter) with all values set to -1, representing the last index each character appeared.
3. Iterate through the string with index `i` and character `ch`.
4. For each character `ch`, compute its contribution to the total appeal. If the last occurrence of `ch` was at `prev = last_index[ch]`, the number of new substrings that include `ch` as a new distinct character is `(i - prev) * (n - i)`.
5. Add this contribution to `total_appeal`.
6. Update `last_index[ch] = i`.
7. After the loop, return `total_appeal`.

**Why it works:** The key invariant is that `(i - prev)` counts how many substrings start after the previous occurrence of the character and include the current one as new, while `(n - i)` counts how many substrings extend from the current position to the end of the string. Multiplying them gives the total number of substrings where this character contributes as a new distinct character, ensuring all contributions are counted exactly once.

## Python Solution

```python
class Solution:
    def appealSum(self, s: str) -> int:
        n = len(s)
        last_index = [-1] * 26
        total_appeal = 0
        
        for i, ch in enumerate(s):
            idx = ord(ch) - ord('a')
            total_appeal += (i - last_index[idx]) * (n - i)
            last_index[idx] = i
        
        return total_appeal
```

The implementation initializes `last_index` for all 26 letters. For each character, it calculates the number of substrings where it contributes as a new distinct character using its last occurrence. This avoids generating substrings explicitly. The final sum is returned as `total_appeal`.

## Go Solution

```go
func appealSum(s string) int64 {
    n := len(s)
    lastIndex := make([]int, 26)
    for i := range lastIndex {
        lastIndex[i] = -1
    }
    var total int64 = 0
    
    for i, ch := range s {
        idx := ch - 'a'
        total += int64((i - lastIndex[idx]) * (n - i))
        lastIndex[idx] = i
    }
    
    return total
}
```

In Go, the last seen indices are initialized to -1 using a slice of size 26. Type conversion to `int64` is necessary to avoid overflow for large strings. The rest of the algorithm mirrors the Python logic.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We traverse the string once, performing O(1) work per character. |
| Space | O(26) ≈ O(1) | We maintain an array of size 26 for last seen indices. |

The linear time complexity is optimal for strings of length up to 10^5, and the space requirement is constant.

1. **Single character string:** Input like `"a"` tests that the algorithm correctly handles minimal input and returns `1`.
2. **All identical characters:** Input like `"aaaaa"` ensures that repeated characters contribute correctly only once per substring and that no overcounting occurs.
3. **All unique characters:** Input like `"abcde"` verifies that each character contributes maximally to all substrings it appears in, and the algorithm correctly calculates contributions without missing any substrings.

This method systematically handles all these cases through the last-seen index logic, ensuring accurate appeal summation in linear time.
