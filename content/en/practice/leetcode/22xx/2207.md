## Solution

## Problem Understanding

The problem asks us to maximize the number of occurrences of a 2-character pattern as a subsequence in a string after inserting **exactly one character**, either `pattern[0]` or `pattern[1]`, at any position in the string. A **subsequence** is formed by deleting zero or more characters while maintaining the order of the remaining characters. The input `text` is a string of lowercase letters of length up to 10^5, and `pattern` is always of length 2. The output is an integer representing the maximum count of subsequences `pattern` that can be formed.

In other words, we are given a string and a 2-letter pattern, and we can insert a single extra letter anywhere to increase the count of this pattern as a subsequence. The task is to determine the **maximum achievable subsequences**.

Important constraints and observations include: the input string can be very long, so brute-force methods are not feasible. The pattern length is fixed at 2, which allows for efficient counting using prefix sums or greedy accumulation. Edge cases include strings that already maximize the pattern or strings consisting of a single repeated character.

## Approaches

A **brute-force approach** would attempt every possible insertion position for `pattern[0]` and `pattern[1]` in `text`, then count the resulting number of subsequences. This guarantees correctness but is too slow because for a string of length n, there are n+1 positions for insertion, and counting subsequences naively can be O(n^2). Therefore, the brute-force approach is impractical for n up to 10^5.

The **key insight** is that we only care about two counts: the number of `pattern[0]` characters seen so far and the number of `pattern[1]` characters remaining. By iterating through the string, we can calculate the number of subsequences `pattern[0] + pattern[1]` using a single pass. After computing the base number of subsequences, the maximum effect of a single insertion is to add the number of `pattern[1]` characters (if we insert `pattern[0]`) or add the number of `pattern[0]` characters (if we insert `pattern[1]`). This observation reduces the problem to O(n) time and O(1) space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Try every possible insertion and count subsequences; too slow for large n |
| Optimal | O(n) | O(1) | Count `pattern` subsequences in one pass; choose insertion that maximizes count |

## Algorithm Walkthrough

1. Initialize two counters: `count0` to track the number of `pattern[0]` characters seen so far and `subseq_count` to track the number of subsequences `pattern` formed.
2. Iterate through the string character by character.
3. For each character:

- If it matches `pattern[0]`, increment `count0`.
- If it matches `pattern[1]`, it can form new subsequences with all previous `pattern[0]` characters. Increment `subseq_count` by `count0`.
4. After completing the iteration, calculate two potential maximums:

- Inserting `pattern[0]` at the beginning increases subsequences by the total number of `pattern[1]` characters in the string.
- Inserting `pattern[1]` at the end increases subsequences by the total number of `pattern[0]` characters in the string.
5. Return the maximum between the original `subseq_count` plus either of the insertion effects.

**Why it works:** The algorithm works because the number of subsequences is determined by counting the number of `pattern[0]` characters before each `pattern[1]`. A single insertion only affects subsequences globally: inserting `pattern[0]` adds a new subsequence for each existing `pattern[1]`, and inserting `pattern[1]` adds a new subsequence for each existing `pattern[0]`. This guarantees we achieve the maximum possible count with one insertion.

## Python Solution

```python
class Solution:
    def maximumSubsequenceCount(self, text: str, pattern: str) -> int:
        count0 = 0
        subseq_count = 0
        total1 = text.count(pattern[1])

        for char in text:
            if char == pattern[0]:
                count0 += 1
            if char == pattern[1]:
                subseq_count += count0

        max_count = subseq_count + max(text.count(pattern[0]), text.count(pattern[1]))
        return max_count
```

**Implementation explanation:** We first count the total occurrences of `pattern[1]` to understand how many subsequences would result if we inserted `pattern[0]`. As we iterate, `count0` tracks the number of `pattern[0]` seen so far, and each `pattern[1]` contributes `count0` subsequences. Finally, we calculate the effect of inserting either character and return the maximum.

## Go Solution

```go
func maximumSubsequenceCount(text string, pattern string) int64 {
    var count0 int64 = 0
    var subseqCount int64 = 0
    var total1 int64 = 0

    for i := 0; i < len(text); i++ {
        if text[i] == pattern[1][0] {
            total1++
        }
    }

    for i := 0; i < len(text); i++ {
        if text[i] == pattern[0][0] {
            count0++
        }
        if text[i] == pattern[1][0] {
            subseqCount += count0
        }
    }

    var count0Total int64 = 0
    for i := 0; i < len(text); i++ {
        if text[i] == pattern[0][0] {
            count0Total++
        }
    }

    var count1Total int64 = 0
    for i := 0; i < len(text); i++ {
        if text[i] == pattern[1][0] {
            count1Total++
        }
    }

    if count0Total > count1Total {
        subseqCount += count0Total
    } else {
        subseqCount += count1Total
    }

    return subseqCount
}
```

**Go-specific notes:** Go requires explicit use of `int64` for large numbers to avoid overflow. The counting logic mirrors Python, but we must convert indices to bytes carefully using `[0]`.

## Worked Examples

**Example 1:** text = "abdcdbc", pattern = "ac"

| Step | char | count0 | subseq_count | Explanation |
| --- | --- | --- | --- | --- |
| 1 | a | 1 | 0 | 'a' matches pattern[0], increment count0 |
| 2 | b | 1 | 0 | no match |
| 3 | d | 1 | 0 | no match |
| 4 | c | 1 | 1 | 'c' matches pattern[1], adds count0 subsequences |
| 5 | d | 1 | 1 | no match |
| 6 | b | 1 | 1 | no match |
| 7 | c | 1 | 2 | 'c' matches pattern[1], adds count0 subsequences |

Inserting 'a' at the beginning: adds 2 (number of 'c's) → total 4. Maximum is 4.

**Example 2:** text = "aabb", pattern = "ab"

| Step | char | count0 | subseq_count |
| --- | --- | --- | --- |
| 1 | a | 1 | 0 |
| 2 | a | 2 | 0 |
| 3 | b | 2 | 2 |
| 4 | b | 2 | 4 |

Inserting 'a' at start adds 2 → total 6. Maximum is 6.
