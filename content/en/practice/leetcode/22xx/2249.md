## Solution

## Problem Understanding

The problem asks us to compute the number of **lattice points** (points with integer coordinates) that lie **inside at least one circle** from a given list of circles. Each circle is defined by its center `(xi, yi)` and radius `ri`. Lattice points that lie **on the circle’s boundary** are also considered inside the circle.

The input `circles` is a list of up to 200 elements, each describing a circle with integer coordinates `(xi, yi)` and a radius `ri`. The constraints ensure that all coordinates and radii are positive and relatively small (`1 <= xi, yi <= 100`, `1 <= ri <= min(xi, yi)`), which guarantees the problem is limited to a relatively small grid. This makes an enumeration-based solution feasible, since the total number of candidate points is bounded by the grid size (approximately 100x100).

Important edge cases include circles that are very small (radius 1), circles that overlap completely, circles that touch the grid edges, or multiple circles that partially overlap. The problem guarantees valid positive integers for coordinates and radii, so there are no degenerate circles or negative values.

## Approaches

### Brute Force

A brute-force approach would be to iterate over all potential lattice points in the entire grid, i.e., `(x, y)` with `1 <= x, y <= 100`. For each point, check whether it lies inside any circle using the formula:

$$(x - xi)^2 + (y - yi)^2 \le ri^2$$

If it does, count it as a lattice point. This approach guarantees correctness because it checks every integer coordinate in the bounding box, but it is inefficient, as it checks `100 * 100 = 10,000` points against up to 200 circles, resulting in up to 2 million checks.

### Optimal Approach

The key observation is that because the coordinates and radii are small (all <= 100), we can **enumerate only the lattice points within the bounding square of each circle** rather than the entire grid. For each circle `(xi, yi, ri)`, the candidate points lie within `x = xi - ri` to `xi + ri` and `y = yi - ri` to `yi + ri`. Checking only these points reduces unnecessary computations significantly. To avoid double counting points that are in multiple circles, we can store lattice points in a **set** (Python) or a **map** (Go).

This observation allows us to directly iterate over points that are **potentially inside a circle**, ensuring both correctness and efficiency.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(100*100 * N) = O(10,000 * N) | O(1) | Checks all points on the 100x100 grid against all circles |
| Optimal | O(N * r^2) ≈ O(200 * 100^2) | O(P) ≤ O(10,000) | Iterates over points in bounding box of each circle and uses a set to avoid double counting |

## Algorithm Walkthrough

1. Initialize an empty set `points` to store unique lattice points.
2. Iterate over each circle `(xi, yi, ri)` in the input list.
3. For each circle, iterate `x` from `xi - ri` to `xi + ri`.
4. For each `x`, iterate `y` from `yi - ri` to `yi + ri`.
5. For each candidate point `(x, y)`, check if it lies inside the circle using the condition `(x - xi)^2 + (y - yi)^2 <= ri^2`.
6. If it does, add `(x, y)` to the set `points`.
7. After processing all circles, the number of lattice points is simply the size of the set `points`.

**Why it works:** Each lattice point is checked exactly once per circle and stored uniquely in the set. Points inside multiple circles are naturally de-duplicated by the set, ensuring correct counting without double counting.

## Python Solution

```python
from typing import List

class Solution:
    def countLatticePoints(self, circles: List[List[int]]) -> int:
        points = set()
        
        for x0, y0, r in circles:
            for x in range(x0 - r, x0 + r + 1):
                for y in range(y0 - r, y0 + r + 1):
                    if (x - x0)**2 + (y - y0)**2 <= r**2:
                        points.add((x, y))
        
        return len(points)
```

**Implementation Walkthrough:** We use a set `points` to store unique lattice points. For each circle, we iterate only over candidate points within its bounding box. The check `(x - x0)**2 + (y - y0)**2 <= r**2` ensures the point lies inside or on the circle. After processing all circles, `len(points)` gives the total number of lattice points inside at least one circle.

## Go Solution

```go
func countLatticePoints(circles [][]int) int {
    points := make(map[[2]int]bool)
    
    for _, circle := range circles {
        x0, y0, r := circle[0], circle[1], circle[2]
        for x := x0 - r; x <= x0 + r; x++ {
            for y := y0 - r; y <= y0 + r; y++ {
                if (x - x0)*(x - x0) + (y - y0)*(y - y0) <= r*r {
                    points[[2]int{x, y}] = true
                }
            }
        }
    }
    
    return len(points)
}
```

**Go-specific notes:** Go uses a map with keys of type `[2]int` to represent lattice points. The boolean value is irrelevant; it is used to ensure uniqueness. Unlike Python, there is no built-in `set` type. Bounds and integer arithmetic are straightforward since the maximum coordinate and radius are small, preventing overflow.

The key optimization is bounding box iteration instead of checking all grid points, which keeps the algorithm efficient.
