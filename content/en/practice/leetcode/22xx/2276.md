## Solution

## Problem Understanding

The problem asks us to design a data structure that efficiently manages a **dynamic set of intervals** and can quickly return the number of distinct integers covered by all intervals. Specifically, we need a class `CountIntervals` that supports two operations: adding an interval `[left, right]` and counting all integers that appear in at least one interval.

The input to `add(left, right)` is two integers defining an inclusive interval. The `count()` method returns the total number of integers covered by **any** interval in the current set. The constraints are significant: `1 <= left <= right <= 10^9` and up to `10^5` operations. This rules out any approach that explicitly stores each integer in the interval because a single interval could contain up to a billion integers, which is infeasible in memory.

The key edge cases are overlapping intervals and adjacent intervals. Adding `[2, 5]` and `[4, 7]` should merge them into a single interval `[2, 7]`. Similarly, `[2, 5]` and `[6, 8]` should merge into `[2, 8]`. Properly managing these merges is essential to avoid double-counting integers in `count()`.

## Approaches

### Brute Force

A brute-force approach would store all integers explicitly in a set. Each `add(left, right)` operation would insert all integers from `left` to `right` into the set, and `count()` would return the size of the set. This is correct but infeasible for large intervals and high numbers of operations because inserting a billion integers is not practical. The time complexity for `add()` would be `O(right - left + 1)` and space complexity would also be proportional to the total number of integers added, which can reach up to `10^9`.

### Optimal Approach

The optimal solution uses a **sorted map** or **balanced binary search tree** (BST) to store intervals in a merged form. The key insight is that overlapping or adjacent intervals can be merged into a single interval, reducing storage to at most the number of non-overlapping intervals added. This allows `add(left, right)` to run in `O(log n + k)` time where `k` is the number of intervals merged, and `count()` to simply return the sum of all merged interval lengths. In Python, `SortedDict` from `sortedcontainers` or `bisect` can manage intervals efficiently. In Go, a custom BST or `map[int]int` with sorting can be used.

This approach avoids storing every integer and guarantees efficiency under the problem constraints.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(R) per add | O(R) | R = size of interval, impractical for large ranges |
| Optimal | O(log N + K) per add, O(1) per count | O(N) | N = number of merged intervals, K = intervals merged during add |

## Algorithm Walkthrough

1. Initialize an empty sorted dictionary `intervals` to store intervals with `start` as key and `end` as value, and a variable `total` to track the total covered integers.
2. To add an interval `[left, right]`, first find all existing intervals that overlap or are adjacent to `[left, right]`.
3. Merge overlapping intervals by taking the minimum of `left` and existing start, and the maximum of `right` and existing end.
4. Remove all overlapping intervals from `intervals` and adjust `total` by subtracting the lengths of these intervals.
5. Insert the new merged interval into `intervals` and increase `total` by its length `(merged_end - merged_start + 1)`.
6. For `count()`, simply return `total`.

**Why it works**: The algorithm maintains the invariant that `intervals` only contains non-overlapping intervals, and `total` always reflects the sum of all interval lengths. Merging ensures no double-counting, and removing old intervals ensures correctness when overlapping intervals are added.

## Python Solution

```python
from sortedcontainers import SortedDict

class CountIntervals:

    def __init__(self):
        self.intervals = SortedDict()
        self.total = 0

    def add(self, left: int, right: int) -> None:
        if not self.intervals:
            self.intervals[left] = right
            self.total += right - left + 1
            return
        
        # Find intervals that may overlap
        to_remove = []
        for start in self.intervals.irange(maximum=right):
            end = self.intervals[start]
            if end < left - 1:
                continue
            left = min(left, start)
            right = max(right, end)
            self.total -= end - start + 1
            to_remove.append(start)
        
        for start in to_remove:
            del self.intervals[start]
        
        self.intervals[left] = right
        self.total += right - left + 1

    def count(self) -> int:
        return self.total
```

**Explanation**: We maintain a `SortedDict` to quickly find intervals that might overlap with the new interval. For each overlapping interval, we adjust `left` and `right` to merge, subtract its length from the total, then delete it. Finally, we insert the new merged interval and update the total count. `count()` is constant-time because we track `total`.

## Go Solution

```go
import (
    "sort"
)

type Interval struct {
    start, end int
}

type CountIntervals struct {
    intervals []Interval
    total     int
}

func Constructor() CountIntervals {
    return CountIntervals{
        intervals: []Interval{},
        total:     0,
    }
}

func (this *CountIntervals) Add(left int, right int) {
    newStart, newEnd := left, right
    merged := []Interval{}
    removedTotal := 0

    for _, inter := range this.intervals {
        if inter.end < newStart-1 || inter.start > newEnd+1 {
            merged = append(merged, inter)
        } else {
            newStart = min(newStart, inter.start)
            newEnd = max(newEnd, inter.end)
            removedTotal += inter.end - inter.start + 1
        }
    }

    merged = append(merged, Interval{newStart, newEnd})
    this.intervals = merged
    this.total = this.total - removedTotal + (newEnd - newStart + 1)
}

func (this *CountIntervals) Count() int {
    return this.total
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

**Explanation**: Go does not have a built-in sorted map, so we store intervals in a slice. On each `Add`, we iterate through intervals to merge overlapping ones, adjusting the total covered length accordingly. This maintains the same invariant as Python: `intervals` are non-overlapping, and `total` is accurate.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(log N + K) per add | Searching for overlapping intervals and merging them, K = number of merged intervals |
| Space | O(N) | Stores at most N non-overlapping intervals |
| Count | O(1) | Total is tracked dynamically |

The dominant cost is merging overlapping intervals during `add`, which is proportional to the number of intervals that overlap with the new one. The `count()` operation is always constant time.
