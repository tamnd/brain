## Solution

## Problem Understanding

The problem describes a square `n x n` grid containing several rectangular artifacts buried under the surface. Each artifact occupies a small rectangular area, defined by its top-left and bottom-right coordinates. You are given a list of cells to dig, and any artifact whose every cell has been excavated can be extracted. The task is to compute the total number of artifacts that can be fully extracted after performing all digs.

In other words, the input consists of the grid size `n`, a list of artifacts represented as subgrid coordinates, and a list of excavation operations. Each artifact may cover up to four cells, and no two artifacts overlap. The output is simply the count of artifacts that are fully uncovered by the digs.

The constraints indicate that the grid can be large (`n` up to 1000), but the number of artifacts and digs is capped at the smaller of `n^2` and `10^5`. This guarantees that iterating over all artifacts or digs individually is feasible, but naive grid-based solutions that involve creating a full `n x n` array might be inefficient for space when `n` is near 1000. Since each artifact covers at most 4 cells, direct enumeration of their cells is possible. Edge cases include artifacts that are a single cell, artifacts fully outside the dig locations, or dig lists that do not uncover any artifact completely.

## Approaches

The brute-force approach would create an `n x n` boolean grid marking all dug cells. Then for each artifact, it would iterate over every cell in its rectangular area and check if all cells are dug. While correct, this approach requires `O(n^2)` space for the grid and `O(k * s)` time, where `k` is the number of artifacts and `s` is the maximum cells per artifact (up to 4). The high space complexity is unnecessary because the artifacts themselves occupy only a small fraction of the grid.

The optimal approach avoids creating the full grid and instead uses a set to store the dug cells. Each artifact is processed by checking its specific cells against the set of dug cells. If all cells of an artifact are present in the set, the artifact can be extracted. This method takes advantage of the fact that artifacts are small, digs are unique, and sets allow `O(1)` average-time membership checks. It drastically reduces space usage to `O(d)` for the dig set and ensures that checking each artifact is very fast.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2 + k * s) | O(n^2) | Uses a full grid to track dug cells and checks all artifact cells individually |
| Optimal | O(d + k * s) | O(d) | Uses a set for dug cells; checks artifact cells individually without building full grid |

## Algorithm Walkthrough

1. Convert the list of dug cells `dig` into a set of tuples. This ensures constant-time lookups when checking if a cell has been excavated.
2. Initialize a counter `count` to zero to track the number of fully extractable artifacts.
3. Iterate over each artifact in the `artifacts` list.
4. For each artifact, generate all cells it occupies. Since each artifact covers at most 4 cells, this is a small iteration.
5. Check if every cell of the artifact exists in the dug set.
6. If all cells are dug, increment `count`.
7. Return `count` after processing all artifacts.

Why it works: By keeping a set of dug cells, the algorithm ensures that the lookup of each cell is fast. Iterating only over the cells that belong to artifacts keeps the runtime proportional to the actual data size rather than the potentially huge grid. The invariant is that an artifact is counted only if every one of its cells is in the dug set, which guarantees correctness.

## Python Solution

```python
from typing import List

class Solution:
    def digArtifacts(self, n: int, artifacts: List[List[int]], dig: List[List[int]]) -> int:
        dug_set = set((r, c) for r, c in dig)
        count = 0
        
        for r1, c1, r2, c2 in artifacts:
            artifact_cells = [(r, c) for r in range(r1, r2 + 1) for c in range(c1, c2 + 1)]
            if all(cell in dug_set for cell in artifact_cells):
                count += 1
                
        return count
```

The Python implementation first constructs a set of dug cells for efficient membership checking. For each artifact, it enumerates all its constituent cells and verifies that each has been dug. If so, it increments the count. The list comprehension for `artifact_cells` ensures all artifact coordinates are included, even if the artifact is a single cell.

## Go Solution

```go
func digArtifacts(n int, artifacts [][]int, dig [][]int) int {
    dugSet := make(map[[2]int]struct{})
    for _, d := range dig {
        dugSet[[2]int{d[0], d[1]}] = struct{}{}
    }

    count := 0
    for _, art := range artifacts {
        r1, c1, r2, c2 := art[0], art[1], art[2], art[3]
        canExtract := true
        for r := r1; r <= r2; r++ {
            for c := c1; c <= c2; c++ {
                if _, found := dugSet[[2]int{r, c}]; !found {
                    canExtract = false
                    break
                }
            }
            if !canExtract {
                break
            }
        }
        if canExtract {
            count++
        }
    }
    return count
}
```

The Go implementation mirrors the Python version but uses a map with empty struct values for the dug set to minimize memory usage. Nested loops enumerate each cell of the artifact, and a flag `canExtract` tracks whether all cells are dug. Breaking early optimizes performance for partially dug artifacts.

## Worked Examples

**Example 1**

`n = 2`, `artifacts = [[0,0,0,0],[0,1,1,1]]`, `dig = [[0,0],[0,1]]`

Dug set: `{(0,0),(0,1)}`

Artifact 0 cells: `(0,0)` → fully dug, count incremented to 1

Artifact 1 cells: `(0,1),(1,1)` → `(1,1)` not dug, count remains 1

Output: `1`

**Example 2**

`n = 2`, `artifacts = [[0,0,0,0],[0,1,1,1]]`, `dig = [[0,0],[0,1],[1,1]]`

Dug set: `{(0,0),(0,1),(1,1)}`

Artifact 0 cells: `(0,0)` → dug, count = 1

Artifact 1 cells: `(0,1),(1,1)` → both dug, count = 2

Output: `2`

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(d + k * s) | Building the dug set is O(d), iterating artifacts is O(k * s) where s ≤ 4 |
| Space | O(d) | Storing the dug cells in a set/map |

The key to efficiency is that `s`, the maximum number of cells per artifact, is very small (≤4), so iterating over artifact cells is cheap. The set provides constant-time membership checks.
