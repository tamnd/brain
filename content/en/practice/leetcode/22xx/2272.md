## Solution

## Problem Understanding

This problem asks us to compute the maximum possible variance among all substrings of a given string.

The variance of a string is defined as the largest difference between the frequencies of any two characters that both appear in the string. For example:

- `"aabbb"` has:

- `a` appearing 2 times
- `b` appearing 3 times
- variance = `3 - 2 = 1`
- `"babbb"` has:

- `b` appearing 4 times
- `a` appearing 1 time
- variance = `4 - 1 = 3`

The important detail is that both characters must exist in the substring. We cannot compare a character count against zero unless that character actually appears in the substring.

The input is a lowercase English string `s` with length up to `10^4`. Since there are only 26 lowercase letters, the alphabet size is very small even though the string itself may be large.

The task is to examine all possible substrings and return the maximum variance achievable.

A naive implementation would generate every substring and compute character frequencies for each one. However, there are `O(n^2)` substrings, and computing frequencies repeatedly would make the solution far too slow for `n = 10^4`.

Several edge cases are important:

- Strings with all unique characters, such as `"abcde"`, always produce variance `0`.
- Strings containing only one distinct character also produce variance `0`, because we need two characters to form a positive difference.
- The optimal substring may not contain all occurrences of a character in the original string.
- The order of characters matters because substrings must remain contiguous.
- A valid variance requires both selected characters to appear at least once.

The key challenge is finding an efficient way to evaluate all relevant substrings without explicitly generating them.

## Approaches

### Brute Force Approach

The brute force solution considers every possible substring.

For each substring:

1. Compute the frequency of every character.
2. Find the largest and smallest positive frequencies.
3. Compute the variance.
4. Track the global maximum.

There are `O(n^2)` substrings. If we compute frequencies from scratch for each substring, each computation costs `O(n)`, producing an overall complexity of `O(n^3)`.

Even with prefix frequency optimization, we still need to examine all character pairs for every substring, leading to roughly `O(26 * n^2)` operations, which is still too expensive for `n = 10^4`.

Although brute force is straightforward and correct, it cannot meet the performance requirements.

### Key Insight Behind the Optimal Solution

The critical observation is that variance only depends on two characters at a time.

Suppose we pick two distinct characters:

- `major`
- `minor`

We want a substring where:

- `major` appears many times
- `minor` appears fewer times
- both characters appear at least once

For a fixed pair `(major, minor)`, the problem becomes:

> Find the maximum value of:
> 
> 
> 
> 
> `count(major) - count(minor)`
> 
> 
> 
> 
> over all substrings containing at least one `minor`.

This looks extremely similar to Kadane's algorithm for maximum subarray sum.

We can interpret:

- `major` as `+1`
- `minor` as `-1`
- all other characters as ignored

Now the problem becomes finding the maximum subarray sum with the constraint that at least one `-1` exists.

Since there are only 26 lowercase letters, we can enumerate all ordered character pairs:

- 26 × 26 = 676 pairs

For each pair, we run a modified Kadane scan over the string.

This transforms the problem into an efficient linear scan per character pair.

## Algorithm Walkthrough

### Step 1: Enumerate All Ordered Character Pairs

We try every ordered pair of distinct letters:

- `(a, b)`
- `(a, c)`
- `(b, a)`
- etc.

The order matters because:

- `count(a) - count(b)` is different from
- `count(b) - count(a)`

We therefore evaluate all possible directional variances.

### Step 2: Treat One Character as Positive and One as Negative

For the current pair:

- `major` contributes `+1`
- `minor` contributes `-1`
- all other characters are ignored

We scan the string from left to right.

### Step 3: Maintain Running Counts

During the scan, we track:

- `major_count`
- `minor_count`

Whenever we encounter:

- `major`, increment `major_count`
- `minor`, increment `minor_count`

### Step 4: Update the Answer Only When Minor Exists

A substring is only valid if it contains at least one `minor`.

Therefore:

- if `minor_count > 0`
- candidate variance is:

- `major_count - minor_count`

We update the global maximum accordingly.

### Step 5: Reset When the Running Difference Becomes Bad

If:

```
major_count < minor_count
```

then the current substring is hurting future results.

Just like Kadane's algorithm, we reset:

```
major_count = 0
minor_count = 0
```

However, we only reset if future `minor` characters still exist ahead in the string.

Otherwise, we might discard the only available `minor`.

### Step 6: Repeat for Every Pair

We repeat the scan for all ordered character pairs and return the maximum variance found.

### Why it works

For a fixed pair `(major, minor)`, every substring variance reduces to:

```
count(major) - count(minor)
```

The modified Kadane scan guarantees that we always keep the best possible prefix while discarding prefixes that can never improve future answers.

Enumerating all ordered pairs ensures that every possible dominant/minority character relationship is explored.

Because every valid variance depends on exactly two characters, this exhaustive pair enumeration is sufficient to find the global optimum.

## Python Solution

```python
class Solution:
    def largestVariance(self, s: str) -> int:
        answer = 0
        
        for major in range(26):
            for minor in range(26):
                if major == minor:
                    continue
                
                major_char = chr(ord('a') + major)
                minor_char = chr(ord('a') + minor)
                
                major_count = 0
                minor_count = 0
                
                remaining_minor = s.count(minor_char)
                
                for ch in s:
                    if ch != major_char and ch != minor_char:
                        continue
                    
                    if ch == major_char:
                        major_count += 1
                    else:
                        minor_count += 1
                        remaining_minor -= 1
                    
                    if minor_count > 0:
                        answer = max(answer, major_count - minor_count)
                    
                    if major_count < minor_count and remaining_minor > 0:
                        major_count = 0
                        minor_count = 0
        
        return answer
```

The solution iterates through every ordered pair of characters. Since the alphabet size is fixed at 26, this enumeration remains efficient.

For each pair:

- `major_count` tracks occurrences of the dominant character
- `minor_count` tracks occurrences of the minority character

Characters unrelated to the current pair are ignored because they do not affect the variance calculation.

The algorithm updates the answer only when at least one `minor` exists in the current substring. This enforces the problem requirement that both characters must appear.

The reset condition mirrors Kadane's algorithm. If the current substring contains more `minor` characters than `major` characters, then extending this substring cannot produce an optimal result later. Resetting allows a fresh start from a better future position.

The `remaining_minor` variable prevents invalid resets when no future `minor` characters remain.

## Go Solution

```go
func largestVariance(s string) int {
    answer := 0

    for major := 0; major < 26; major++ {
        for minor := 0; minor < 26; minor++ {
            if major == minor {
                continue
            }

            majorChar := byte('a' + major)
            minorChar := byte('a' + minor)

            majorCount := 0
            minorCount := 0

            remainingMinor := 0
            for i := 0; i < len(s); i++ {
                if s[i] == minorChar {
                    remainingMinor++
                }
            }

            for i := 0; i < len(s); i++ {
                ch := s[i]

                if ch != majorChar && ch != minorChar {
                    continue
                }

                if ch == majorChar {
                    majorCount++
                } else {
                    minorCount++
                    remainingMinor--
                }

                if minorCount > 0 {
                    variance := majorCount - minorCount
                    if variance > answer {
                        answer = variance
                    }
                }

                if majorCount < minorCount && remainingMinor > 0 {
                    majorCount = 0
                    minorCount = 0
                }
            }
        }
    }

    return answer
}
```

The Go implementation follows the same logic as the Python version.

Some implementation details differ slightly:

- Go strings are indexed as byte arrays because the input only contains lowercase English letters.
- We manually compute maximum values instead of using a built in `max` function.
- Integer overflow is not a concern because counts never exceed `10^4`.

The algorithm still uses constant extra space and linear scans per character pair.

## Edge Cases

### Strings With Only One Distinct Character

Consider:

```
"aaaaaa"
```

A common bug is accidentally treating missing characters as frequency zero and computing variance against them.

However, the definition requires both characters to exist in the substring.

The implementation handles this correctly because the variance is only updated when `minor_count > 0`.

### Resetting Too Aggressively

Suppose we have:

```
"aaab"
```

If we reset after seeing more minority characters without checking whether more future minority characters exist, we may discard the only valid substring containing both characters.

The `remaining_minor` counter prevents this issue by ensuring resets only happen when future minority characters remain available.

### Alternating Character Patterns

Strings like:

```
"abababab"
```

can expose bugs in Kadane style logic because the running difference frequently returns to zero.

The implementation correctly maintains valid substrings while continuously updating the best variance whenever both characters are present.

This ensures alternating patterns still produce the correct result instead of being reset unnecessarily.
