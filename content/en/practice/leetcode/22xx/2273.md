## Solution

## Problem Understanding

The problem asks us to process an array of strings `words` by repeatedly removing elements that are **anagrams of their immediate predecessor**. Formally, for any index `i` where `0 < i < len(words)`, if `words[i]` and `words[i-1]` are anagrams, `words[i]` should be removed. This process continues until no adjacent words are anagrams.

The input `words` is a list of lowercase strings. The output is a list of strings after all applicable deletions have been performed. Because the operation is **local to adjacent elements**, the order matters, and the resulting array always starts from the first word and keeps non-redundant words in sequence.

Constraints indicate that the length of `words` is small (`<= 100`) and the length of each word is very small (`<= 10`). This suggests that even operations like sorting or counting letters per word will be efficient.

Important edge cases include:

1. All words are distinct with no anagrams. In this case, the output is identical to the input.
2. All words are mutual anagrams. Only the first word survives.
3. Repeated words that are identical (which are trivially anagrams). The algorithm should remove duplicates except the first occurrence in a consecutive block.

## Approaches

### Brute Force

The brute-force approach would repeatedly scan the list and check every pair of consecutive words for being anagrams. If a pair is found, the latter word is removed and the scan starts over. To check for anagrams, we could sort each word and compare. This approach is correct because it directly implements the problem statement, but it is inefficient: each scan can take `O(n)` and checking anagrams takes `O(k log k)` per pair, potentially requiring `O(n^2 * k log k)` in the worst case.

### Optimal Approach

The key insight is that we only need to compare **consecutive words once** if we represent each word by a canonical form that identifies its letter composition. Sorting the characters of a word, or counting letters into a frequency array, produces such a canonical representation. Then we can iterate through the list **once**, compare each word to the previous kept word, and append it to the result only if it is not an anagram of the previous word. This reduces time complexity to `O(n * k log k)` using sorted strings or `O(n * k)` using letter frequency counts.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2 * k log k) | O(k) | Repeatedly scans array and checks consecutive pairs |
| Optimal | O(n * k) | O(n * k) | Iterates once, compares canonical forms of consecutive words |

## Algorithm Walkthrough

1. Initialize an empty list `result` that will hold the final words.
2. Iterate through each word `w` in `words`.
3. Convert `w` into a canonical form representing its character composition. This can be a tuple of counts of letters `'a'` to `'z'`.
4. If `result` is empty, append `w` and store its canonical form as `prev_form`.
5. For subsequent words, compare `w`'s canonical form with `prev_form`. If they are identical, `w` is an anagram of the previous word and should be skipped.
6. If they differ, append `w` to `result` and update `prev_form`.
7. Return `result`.

**Why it works**: By maintaining the canonical form of the last appended word, we guarantee that only consecutive anagrams are removed. This preserves the first occurrence and ensures correctness.

## Python Solution

```python
from typing import List

class Solution:
    def removeAnagrams(self, words: List[str]) -> List[str]:
        result = []
        prev_form = None
        
        for word in words:
            # Convert word to a tuple of letter counts
            count = [0] * 26
            for ch in word:
                count[ord(ch) - ord('a')] += 1
            current_form = tuple(count)
            
            if current_form != prev_form:
                result.append(word)
                prev_form = current_form
        
        return result
```

The implementation first initializes `result` and `prev_form`. Each word is converted into a tuple representing its letter counts, which allows O(1) comparison with the previous word. If the forms differ, the word is appended to `result`, ensuring only consecutive anagrams are removed.

## Go Solution

```go
func removeAnagrams(words []string) []string {
    result := []string{}
    var prevForm [26]int
    
    for i, word := range words {
        var count [26]int
        for _, ch := range word {
            count[ch-'a']++
        }
        
        if i == 0 || count != prevForm {
            result = append(result, word)
            prevForm = count
        }
    }
    
    return result
}
```

In Go, we use a fixed-size array of 26 integers to represent the letter counts. We compare arrays directly, which is allowed in Go since arrays of equal length are comparable. This is analogous to the Python tuple approach.

The algorithm scales linearly with the number of words, and the small constant `k` ensures it is efficient even at maximum constraints.

## Edge Cases

**All words identical**: If all words are `"a"`, the algorithm removes all but the first. Using canonical forms ensures duplicates are detected.

**Alternating anagrams**: Words like `["abc","def","cba","fed"]` require checking only consecutive elements. Non-adjacent anagrams do not trigger removal.

**Single word array**: A single-element input should be returned unchanged. Our algorithm handles this by starting with an empty `result` and appending the first word unconditionally.

These cases ensure the solution correctly implements the "adjacent anagram removal" requirement without over-removing or under-removing words.
