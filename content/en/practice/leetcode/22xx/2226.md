## Solution

## Problem Understanding

The problem asks us to distribute candies from a given list of piles to `k` children in a very specific way. Each element in the array `candies` represents a pile of candies. We are allowed to **split piles into smaller sub-piles** but **cannot merge piles together**. Each child must receive the **same number of candies**, and each child can only take candies from **one pile**. The goal is to determine the **maximum number of candies** each child can receive under these constraints.

The input consists of a list of integers `candies` and an integer `k`. The output is a single integer representing the maximum candies per child.

The constraints are important to consider. The number of piles can go up to `10^5`, individual piles can have up to `10^7` candies, and the number of children can be as large as `10^12`. This tells us that a naive approach that iterates through every possible allocation is infeasible due to both time and space limits.

Key edge cases include situations where there are more children than total candies, when piles have the same number of candies, or when all piles are smaller than the desired allocation.

## Approaches

### Brute Force

A brute-force approach would iterate through every possible number of candies per child, from `1` to the largest pile in `candies`. For each candidate number `x`, we would count how many children could be served by dividing each pile by `x` using integer division. The maximum `x` for which the total number of children served is at least `k` would be our answer.

This approach is correct because it exhaustively checks every possible allocation, but it is **too slow** given the constraints. For example, if the largest pile has `10^7` candies, iterating through all values up to `10^7` for `10^5` piles would be computationally prohibitive.

### Optimal Approach

The key insight is that we can use **binary search** on the number of candies per child. Since the function `canDistribute(x)` is **monotonic** - if `x` candies per child is possible, then any number less than `x` is also possible - we can efficiently find the maximum valid number using binary search.

The function `canDistribute(x)` iterates through all piles and calculates how many children each pile can serve if each child receives `x` candies. If the total count is at least `k`, then `x` is feasible; otherwise, it is not. Binary search narrows down the maximum feasible `x` in logarithmic time relative to the largest pile.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(max(candies) * n) | O(1) | Iterates every possible candies per child and counts children |
| Optimal | O(n * log(max(candies))) | O(1) | Uses binary search on candies per child with a monotonic feasibility check |

## Algorithm Walkthrough

1. Determine the range of possible candies per child. The minimum is `1`, and the maximum is the largest pile in `candies`.
2. Implement a helper function `canDistribute(x)` that returns `True` if it is possible to serve `k` children with `x` candies each:

- Initialize a counter `count` to zero.
- For each pile `pile` in `candies`, add `pile // x` to `count`. This calculates how many children can get `x` candies from this pile.
- If `count >= k`, return `True`; otherwise, return `False`.
3. Use binary search between the minimum and maximum possible candies per child:

- Calculate `mid` as the average of `low` and `high`.
- If `canDistribute(mid)` is `True`, move the `low` bound to `mid + 1` because we can potentially give more candies per child.
- If `canDistribute(mid)` is `False`, move the `high` bound to `mid - 1` because `mid` is too large.
4. After the loop terminates, `high` will contain the maximum number of candies each child can get.

**Why it works:** Binary search works here because the feasibility function is monotonic. If `x` candies per child is feasible, any smaller number of candies is also feasible. Conversely, if `x` is not feasible, any larger number is also not feasible. This guarantees that binary search converges to the maximum valid number.

## Python Solution

```python
from typing import List

class Solution:
    def maximumCandies(self, candies: List[int], k: int) -> int:
        def canDistribute(x: int) -> bool:
            count = 0
            for pile in candies:
                count += pile // x
            return count >= k
        
        low, high = 1, max(candies)
        result = 0
        while low <= high:
            mid = (low + high) // 2
            if canDistribute(mid):
                result = mid
                low = mid + 1
            else:
                high = mid - 1
        return result
```

The Python implementation defines the helper `canDistribute` which checks if `k` children can each get `x` candies. Binary search is then applied over the range from `1` to `max(candies)`. The variable `result` tracks the largest feasible number of candies per child.

## Go Solution

```go
func maximumCandies(candies []int, k int64) int {
    canDistribute := func(x int) bool {
        var count int64 = 0
        for _, pile := range candies {
            count += int64(pile / x)
        }
        return count >= k
    }
    
    low, high := 1, 0
    for _, pile := range candies {
        if pile > high {
            high = pile
        }
    }
    
    result := 0
    for low <= high {
        mid := (low + high) / 2
        if canDistribute(mid) {
            result = mid
            low = mid + 1
        } else {
            high = mid - 1
        }
    }
    return result
}
```

The Go version mirrors the Python solution but takes care to cast divisions and sums to `int64` to prevent overflow since `k` can be very large. The binary search logic and helper function are implemented identically.
