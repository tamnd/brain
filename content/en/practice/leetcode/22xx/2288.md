## Solution

## Problem Understanding

The problem gives us a sentence where words are separated by single spaces. Some words represent prices, and we must apply a percentage discount to those prices while leaving every other word unchanged.

A word is considered a valid price only if:

1. The word starts with the dollar sign `$`
2. Every remaining character is a digit
3. There is at least one digit after `$`

Examples of valid prices include `$1`, `$99`, and `$1234567890`.

Examples of invalid prices include:

- `$` because it has no digits
- `5$` because the dollar sign is not at the beginning
- `$1e5` because it contains letters
- `$12a` because it contains non-digit characters

For every valid price, we must:

1. Extract the numeric value
2. Apply the discount percentage
3. Format the result with exactly two decimal places
4. Put the `$` sign back in front

The final output is the modified sentence with all valid prices updated.

The constraints are important here:

- The sentence length can be up to `10^5`
- Prices can contain up to `10` digits
- We need to process efficiently in linear time

Because the input can be very large, repeatedly rebuilding strings inefficiently or using expensive parsing logic could become problematic. Fortunately, each word can be processed independently, which strongly suggests a single-pass solution.

Several edge cases are especially important:

- Words containing `$` but not representing valid prices
- A standalone `$`
- Prices with a `100%` discount, resulting in `$0.00`
- Very large numeric values
- Sentences with no prices at all
- Prices adjacent to letters or symbols that invalidate them

The problem guarantees that words are separated by single spaces and there are no leading or trailing spaces, which simplifies tokenization.

## Approaches

### Brute Force Approach

A brute force solution could examine every possible substring of the sentence to determine whether it forms a valid price, then attempt to replace it with the discounted version.

This approach would repeatedly scan portions of the sentence and perform unnecessary validations. Since substring extraction and repeated checks can become expensive, especially for long inputs, the total complexity could grow toward quadratic time.

Although this method would eventually produce the correct answer, it does much more work than necessary because the sentence structure already gives us natural word boundaries.

### Optimal Approach

The key observation is that prices are defined at the word level. Since the sentence is already separated by spaces, we only need to process each word independently.

The optimal strategy is:

1. Split the sentence into words
2. For each word:

- Check whether it is a valid price
- If valid, compute the discounted value
- Format it to exactly two decimal places
3. Rebuild the sentence

This works efficiently because each character is examined only a constant number of times.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n²) | Repeated substring scanning and reconstruction |
| Optimal | O(n) | O(n) | Single pass over all words |

## Algorithm Walkthrough

1. Split the sentence into individual words using spaces.

Since the problem guarantees single-space separation and no extra spaces, a simple split operation cleanly isolates every word.
2. Initialize an empty list to store processed words.

We will build the final sentence incrementally without modifying the original string directly.
3. Process each word independently.

For every word, determine whether it represents a valid price.
4. Check whether the word is a valid price.

A word is valid if:

- Its length is at least 2
- The first character is `$`
- Every remaining character is a digit

This ensures invalid patterns like `$`, `5$`, or `$1a` are ignored.
5. Extract the numeric portion.

Remove the leading `$` and convert the remaining digits into a numeric value.
6. Apply the discount.

If the original value is `price`, the discounted value becomes:

$\text{discounted} = \text{price} \times \frac{100 - \text{discount}}{100}$
7. Format the discounted value.

Convert the result into a string with exactly two decimal places using standard floating-point formatting.
8. Append the transformed word to the result list.

If the word was not a valid price, append it unchanged.
9. Join all processed words with spaces.

This reconstructs the final sentence in the required format.

### Why it works

The algorithm works because every valid price must occupy an entire word. By checking each word independently and validating the exact structure required by the problem, we guarantee that only legitimate prices are transformed.

Every word is either:

- A valid price and correctly discounted
- Not a valid price and preserved unchanged

Since every word is processed exactly once and reconstruction preserves ordering and spacing, the final sentence is correct.

## Python Solution

```python
class Solution:
    def discountPrices(self, sentence: str, discount: int) -> str:
        words = sentence.split()
        result = []

        multiplier = (100 - discount) / 100

        for word in words:
            if (
                len(word) >= 2
                and word[0] == '$'
                and word[1:].isdigit()
            ):
                original_price = int(word[1:])
                discounted_price = original_price * multiplier
                result.append(f"${discounted_price:.2f}")
            else:
                result.append(word)

        return " ".join(result)
```

The implementation begins by splitting the sentence into words. This gives direct access to each candidate token without needing manual parsing.

The `multiplier` variable stores the percentage remaining after the discount. For example, a `20%` discount means multiplying prices by `0.8`.

Each word is then validated carefully:

- `len(word) >= 2` prevents `$` from being treated as a price
- `word[0] == '$'` ensures the dollar sign is at the beginning
- `word[1:].isdigit()` guarantees the remaining characters are all digits

When a valid price is found, the numeric portion is converted into an integer, discounted, and formatted with exactly two decimal places using Python formatting syntax.

Finally, all processed words are joined back together into the final sentence.

## Go Solution

```go
package main

import (
	"fmt"
	"strconv"
	"strings"
	"unicode"
)

func discountPrices(sentence string, discount int) string {
	words := strings.Split(sentence, " ")
	result := make([]string, 0, len(words))

	multiplier := float64(100-discount) / 100.0

	for _, word := range words {
		if isValidPrice(word) {
			value, _ := strconv.ParseInt(word[1:], 10, 64)

			discounted := float64(value) * multiplier
			formatted := fmt.Sprintf("$%.2f", discounted)

			result = append(result, formatted)
		} else {
			result = append(result, word)
		}
	}

	return strings.Join(result, " ")
}

func isValidPrice(word string) bool {
	if len(word) < 2 || word[0] != '$' {
		return false
	}

	for _, ch := range word[1:] {
		if !unicode.IsDigit(ch) {
			return false
		}
	}

	return true
}
```

The Go implementation follows the same logic as the Python version but requires more explicit handling of strings and numeric conversion.

A helper function `isValidPrice` is used to keep validation logic clean and reusable. Since Go strings are byte-based, iterating with `range` ensures proper rune handling, although this problem only contains ASCII characters.

`strconv.ParseInt` converts the numeric portion safely into a 64-bit integer, which is sufficient because prices contain at most 10 digits.

Formatting is handled with `fmt.Sprintf("$%.2f", discounted)` to guarantee exactly two decimal places.

## Edge Cases

One important edge case is a standalone dollar sign such as `$`. A naive implementation might remove the `$` and attempt to parse an empty string as a number, causing an error. The solution avoids this by requiring the word length to be at least 2 before validating it as a price.

Another tricky case involves words that contain `$` but are not valid prices, such as `5$`, `$1a`, or `$10$`. These can easily fool implementations that only check whether a word contains a dollar sign. The algorithm correctly validates that the first character must be `$` and every remaining character must be a digit.

A third important edge case is a `100%` discount. In this situation, every valid price becomes zero. Some implementations may incorrectly produce `$0` instead of `$0.00`. Using fixed two-decimal formatting guarantees compliance with the output requirements.

Large numeric values are also worth considering. Prices may contain up to `10` digits, so implementations using small integer types could overflow in some languages. The Go solution uses `int64`, while Python integers automatically expand as needed.
