## Solution

## Problem Understanding

The problem asks us to determine how many positions on a street meet a minimum brightness requirement. The street is represented as a number line from `0` to `n - 1`. Each street lamp has a position and a range, and it illuminates all positions within that range. The brightness at a position is defined as the number of lamps illuminating it. A separate array `requirement` specifies, for each position, the minimum brightness needed. Our task is to count how many positions have brightness greater than or equal to their requirement.

The inputs are: `n`, the street length; `lights`, a 2D array with the position and range of each lamp; and `requirement`, a 1D array indicating minimum brightness per position. Constraints show that `n` and the number of lights can go up to `10^5`, and lamp ranges can also be large. This suggests a brute-force solution that checks every position for every light would be too slow. Edge cases include positions at the start or end of the street, lamps with zero range, or extremely high requirements.

## Approaches

### Brute Force

A naive approach iterates over each position `i` and counts how many lights illuminate it by iterating through the `lights` array. Then, it compares this count to `requirement[i]` and increments the result if the requirement is met. This approach is correct but inefficient, with a time complexity of `O(n * m)` where `m` is the number of lights. Given the constraints (`n, m ≤ 10^5`), this would be too slow.

### Optimal Approach

The key insight is that each lamp affects a **contiguous interval** on the street. We can leverage a **prefix sum array** to track brightness changes efficiently. For each lamp at position `p` with range `r`, we increase the brightness at `max(0, p-r)` by 1 and decrease at `min(n, p+r+1)` by 1. After processing all lamps, we compute the prefix sum to get the actual brightness at each position. Finally, we compare the brightness array with `requirement` to count positions meeting the requirement. This approach reduces time complexity to `O(n + m)` and uses `O(n)` space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m) | O(1) | Checks every light for every position |
| Optimal | O(n + m) | O(n) | Uses prefix sum for efficient interval updates |

## Algorithm Walkthrough

1. Initialize an array `brightness` of size `n+1` with zeros. The extra element handles interval subtraction without out-of-bounds errors.
2. For each lamp `[position, range]`, calculate the interval it illuminates as `[start, end] = [max(0, position - range), min(n - 1, position + range)]`.
3. Increment `brightness[start]` by 1 to indicate the start of illumination.
4. Decrement `brightness[end + 1]` by 1 to indicate the end of illumination (if `end + 1 < n`).
5. Compute the prefix sum of `brightness` to get the actual brightness at each position.
6. Iterate through positions `0` to `n-1` and compare `brightness[i]` to `requirement[i]`. Count the number of positions that meet or exceed the requirement.
7. Return the count.

Why it works: By using a prefix sum, each lamp contributes +1 to the start of its interval and -1 immediately after the interval. Computing the prefix sum accumulates all contributions correctly, ensuring that each position's brightness reflects all lamps that cover it. This approach efficiently handles overlapping lamp ranges.

## Python Solution

```python
from typing import List

class Solution:
    def meetRequirement(self, n: int, lights: List[List[int]], requirement: List[int]) -> int:
        brightness = [0] * (n + 1)  # Extra space for range subtraction
        
        for position, r in lights:
            start = max(0, position - r)
            end = min(n - 1, position + r)
            brightness[start] += 1
            if end + 1 < n:
                brightness[end + 1] -= 1
        
        # Compute prefix sum to get actual brightness
        for i in range(1, n):
            brightness[i] += brightness[i - 1]
        
        # Count positions meeting requirement
        count = 0
        for i in range(n):
            if brightness[i] >= requirement[i]:
                count += 1
                
        return count
```

The Python implementation first constructs the `brightness` array using interval updates. The prefix sum converts these interval increments into actual brightness values. Finally, the loop counts positions meeting the required brightness.

## Go Solution

```go
func meetRequirement(n int, lights [][]int, requirement []int) int {
    brightness := make([]int, n+1)

    for _, light := range lights {
        position, r := light[0], light[1]
        start := position - r
        if start < 0 {
            start = 0
        }
        end := position + r
        if end >= n {
            end = n - 1
        }
        brightness[start]++
        if end+1 < n {
            brightness[end+1]--
        }
    }

    for i := 1; i < n; i++ {
        brightness[i] += brightness[i-1]
    }

    count := 0
    for i := 0; i < n; i++ {
        if brightness[i] >= requirement[i] {
            count++
        }
    }
    return count
}
```

The Go solution mirrors the Python approach. Slice allocation uses `n+1` to simplify interval decrement handling. Prefix sum computation and final count are identical in logic.

## Worked Examples

### Example 1

Input: `n = 5`, `lights = [[0,1],[2,1],[3,2]]`, `requirement = [0,2,1,4,1]`

Step through each lamp:

- Lamp `[0,1]` updates interval `[0,1]`: brightness `[1,1,0,0,0,0]`
- Lamp `[2,1]` updates interval `[1,3]`: brightness `[1,2,0, -1,0,0]` (after incremental step)
- Lamp `[3,2]` updates interval `[1,4]`: brightness `[1,3,0,-1,-1,0]`

Prefix sum: `[1,4,4,3,2]`

Compare with `requirement`: `[1>=0, 4>=2, 4>=1, 3<4, 2>=1]` → positions `0,1,2,4` → count = 4

### Example 2

Input: `n = 1`, `lights = [[0,1]]`, `requirement = [2]`

Lamp `[0,1]` affects `[0,0]`: brightness `[1,0]`

Prefix sum: `[1]`

Compare with requirement: `1<2` → count = 0

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n + m) | `m` lamps are processed for interval updates, `n` positions for prefix sum and final comparison |
| Space | O(n) | Prefix sum array of size `n+1` |

The algorithm is efficient because each lamp contributes only two updates, and prefix sum converts this into actual brightness without iterating over each position for every lamp.

## Edge Cases

1. **Single-position street**: When `n = 1`, the algorithm must correctly handle prefix sum with minimal array size. Using `n+1` array prevents index errors when subtracting outside bounds.
2. **Lamp with zero range**: A lamp may only illuminate its own position. The interval calculation `start = position - r` and `end = position + r` correctly handles this and does not skip the position.
3. **High brightness requirements**: If `requirement[i]` is larger than the sum of all lamps covering a position, the position should not be counted
