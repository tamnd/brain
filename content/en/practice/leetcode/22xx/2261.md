## Solution

## Problem Understanding

The problem asks us to count the number of **distinct subarrays** of a given array `nums` that satisfy a specific divisibility constraint. A subarray is any contiguous, non-empty sequence of elements in `nums`. Each subarray we consider must have **at most `k` elements divisible by `p`**. Subarrays are considered **distinct** if they differ in length or have at least one element that differs at the same index.

The input consists of three parameters: `nums`, `k`, and `p`. `nums` is an array of integers with length up to 200. The integer `k` specifies the maximum number of elements in a subarray that can be divisible by `p`. The integer `p` is the divisor to check divisibility. The output is a single integer representing the number of distinct subarrays meeting the criteria.

The constraints indicate that a brute-force approach generating all possible subarrays is feasible in principle because `nums.length` is small, but we must also ensure uniqueness, which adds an additional challenge. Edge cases include arrays where all elements are divisible by `p`, `k` equals the array length, or repeated values create duplicate subarrays.

## Approaches

The brute-force approach generates **all possible subarrays**, counts the elements divisible by `p` for each subarray, and checks if the count is at most `k`. To ensure uniqueness, we can store each valid subarray in a set (using tuples as immutable representations). This method is correct but inefficient because it generates `O(n^2)` subarrays, and each subarray comparison for uniqueness can take up to `O(n)` time. For `n = 200`, this may still be acceptable, but it is not optimal.

The **optimal approach** uses the insight that we can combine **subarray enumeration with a hash set** to ensure uniqueness efficiently. Instead of storing the entire subarray, we can store a hash representation of the subarray, for example using Python tuples or rolling hash techniques. We iterate over all possible subarray starting points, extend them to the right while counting divisible elements, and stop extending once we exceed `k`. By storing each subarray representation in a set, we automatically count only distinct subarrays.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n^2) | Generate all subarrays, count divisibles, store each valid subarray in a set |
| Optimal | O(n^2) | O(n^2) | Enumerate subarrays while counting divisible elements, store each distinct subarray in a set |

## Algorithm Walkthrough

1. Initialize an empty set `distinct_subarrays` to store unique subarrays.
2. Iterate through the array with a variable `start` representing the starting index of a subarray.
3. For each `start`, initialize `count_divisible` to zero and an empty list `current_subarray`.
4. Extend the subarray by iterating `end` from `start` to the end of the array.
5. For each `nums[end]`, append it to `current_subarray`. If `nums[end]` is divisible by `p`, increment `count_divisible`.
6. If `count_divisible` exceeds `k`, break the inner loop because any further extension would also exceed `k`.
7. Convert `current_subarray` to a tuple and add it to `distinct_subarrays` to ensure uniqueness.
8. After iterating over all starting points, return the size of `distinct_subarrays`.

**Why it works:** This approach works because it guarantees that we consider all possible contiguous subarrays starting from each index. By counting divisible elements dynamically and stopping early, we avoid unnecessary computation. The set ensures that only distinct subarrays are counted.

## Python Solution

```python
from typing import List

class Solution:
    def countDistinct(self, nums: List[int], k: int, p: int) -> int:
        distinct_subarrays = set()
        
        for start in range(len(nums)):
            count_divisible = 0
            current_subarray = []
            
            for end in range(start, len(nums)):
                current_subarray.append(nums[end])
                
                if nums[end] % p == 0:
                    count_divisible += 1
                
                if count_divisible > k:
                    break
                
                distinct_subarrays.add(tuple(current_subarray))
        
        return len(distinct_subarrays)
```

This implementation uses a set of tuples to store each distinct subarray. The outer loop iterates through all possible starting indices. The inner loop builds subarrays incrementally, counting elements divisible by `p`. Once the count exceeds `k`, the inner loop stops. This ensures correctness while maintaining efficiency.

## Go Solution

```go
func countDistinct(nums []int, k int, p int) int {
    type key struct {
        arr []int
    }
    distinct := make(map[string]struct{})
    
    for start := 0; start < len(nums); start++ {
        countDivisible := 0
        current := []int{}
        
        for end := start; end < len(nums); end++ {
            current = append(current, nums[end])
            if nums[end]%p == 0 {
                countDivisible++
            }
            if countDivisible > k {
                break
            }
            // Use fmt.Sprintf to convert slice to string for map key
            keyStr := fmt.Sprintf("%v", current)
            distinct[keyStr] = struct{}{}
        }
    }
    
    return len(distinct)
}
```

In Go, slices are not hashable, so we convert the current subarray to a string key before inserting into the map. This achieves the same uniqueness guarantee as using tuples in Python.

## Edge Cases

1. **All elements divisible by `p`**. If `k` < array length, some subarrays will exceed `k`. The implementation correctly stops subarray extension once `k` is exceeded.
2. **Array with duplicates**. Duplicate elements can produce repeated subarrays. Using a set ensures only unique subarrays are counted, preventing overcounting.
3. **Single element arrays**. The minimum valid input must return 1 if it meets the divisibility constraint. The code handles this by initializing and checking each subarray individually.
4. **k = 0**. This tests that no subarray with elements divisible by `p` is counted. The inner loop correctly breaks when the first divisible element is encountered.

This approach guarantees correctness across all valid input scenarios.
