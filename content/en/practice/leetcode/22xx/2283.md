## Solution

## Problem Understanding

The problem gives us a string `num` consisting only of digits. The string is `0-indexed`, meaning the first character is at index `0`, the second at index `1`, and so on.

For every index `i`, the character `num[i]` represents how many times the digit `i` should appear in the entire string.

This is the key idea of the problem:

- The index represents a digit value
- The character at that index represents the expected frequency of that digit in the string

For example, consider:

```
num = "1210"
```

This means:

- At index `0`, the value is `'1'`, so digit `0` must appear exactly 1 time
- At index `1`, the value is `'2'`, so digit `1` must appear exactly 2 times
- At index `2`, the value is `'1'`, so digit `2` must appear exactly 1 time
- At index `3`, the value is `'0'`, so digit `3` must appear 0 times

If all these conditions are satisfied, we return `true`. Otherwise, we return `false`.

The constraints are very small:

- `1 <= n <= 10`
- The string contains only digits

Since the maximum length is only 10, even less efficient solutions would still run quickly. However, the problem is fundamentally about counting digit frequencies, so we should still design a clean and efficient approach.

An important observation is that the indices themselves correspond to digit values. Since the length is at most 10, the only relevant digits are `0` through `9`.

Some important edge cases include:

- Strings of length 1, such as `"0"` or `"1"`
- Digits that are expected to occur zero times
- Digits appearing more often than expected
- Digits missing entirely when a positive count is required
- Repeated digits in the string
- Cases where multiple indices fail simultaneously

The problem guarantees valid digit characters, so we do not need to handle invalid input types or non-digit characters.

## Approaches

### Brute Force Approach

The brute force method checks each index independently by scanning the entire string every time.

For each index `i`:

1. Convert `num[i]` into an integer representing the expected count
2. Traverse the whole string and count how many times digit `i` appears
3. Compare the actual count with the expected count
4. If they differ, immediately return `false`

If every index matches correctly, return `true`.

This works because we explicitly verify the condition for every digit index. However, it repeatedly scans the same string, which creates unnecessary repeated work.

Even though the constraints are tiny, this approach is less efficient conceptually because frequency counting can be done once and reused.

### Optimal Approach

The better solution counts all digit frequencies first, then validates each index using those counts.

The key insight is that the problem repeatedly asks:

> "How many times does digit `i` occur?"

Instead of recalculating this repeatedly, we can compute all digit frequencies once using a hash map or frequency array.

The algorithm becomes:

1. Count occurrences of every digit
2. Iterate through each index
3. Compare the expected frequency with the actual frequency
4. Return `false` immediately if any mismatch exists

This avoids redundant scanning and makes the logic cleaner.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Repeatedly scans the string for every index |
| Optimal | O(n) | O(1) | Counts frequencies once, then validates each index |

Since `n <= 10`, both approaches are practically fast, but the optimal approach is more scalable and cleaner.

## Algorithm Walkthrough

1. Create a frequency counter to store how many times each digit appears in the string.

We need quick access to the number of occurrences of every digit. A hash map or fixed-size array works well because digits range only from `0` to `9`.
2. Traverse the string once and populate the frequency counter.

For every character in `num`, increment its count in the frequency structure.

Example:

```
num = "1210"
```

Frequency counts become:

```
'0' -> 1
'1' -> 2
'2' -> 1
```
3. Iterate through every index `i` in the string.

Each index represents the digit whose frequency we must validate.
4. Convert `num[i]` into an integer.

This gives the expected number of occurrences for digit `i`.

Example:

```
i = 2
num[2] = '1'
expected count = 1
```
5. Look up how many times digit `i` actually appears.

Since indices are integers but the string stores characters, convert `i` into a character representation before checking the frequency map.

Example:

```
digit = '2'
actual count = 1
```
6. Compare expected and actual counts.

If they differ, return `false` immediately because the condition fails.
7. If all indices match successfully, return `true`.

### Why it works

The algorithm works because it directly verifies the exact condition required by the problem definition.

For every index `i`, we compare:

- The expected count, stored in `num[i]`
- The actual count, computed from the full string

Since every index is checked independently and accurately, the algorithm returns `true` if and only if all required digit frequencies match.

## Python Solution

```python
class Solution:
    def digitCount(self, num: str) -> bool:
        frequency = {}

        for digit in num:
            frequency[digit] = frequency.get(digit, 0) + 1

        for i in range(len(num)):
            expected_count = int(num[i])
            actual_count = frequency.get(str(i), 0)

            if expected_count != actual_count:
                return False

        return True
```

The implementation begins by building a frequency dictionary. Each digit character in the string is counted exactly once.

The second loop validates the problem condition. For every index `i`, the code:

- Reads the expected count from `num[i]`
- Converts `i` into a string because the dictionary keys are characters
- Retrieves the actual frequency using `.get()` with default value `0`

If any mismatch occurs, the function immediately returns `False`.

If the loop finishes successfully, every index satisfies the requirement, so the function returns `True`.

## Go Solution

```go
func digitCount(num string) bool {
    frequency := make(map[byte]int)

    for i := 0; i < len(num); i++ {
        frequency[num[i]]++
    }

    for i := 0; i < len(num); i++ {
        expectedCount := int(num[i] - '0')
        actualCount := frequency[byte(i)+'0']

        if expectedCount != actualCount {
            return false
        }
    }

    return true
}
```

The Go solution follows the same logic as the Python version but uses a `map[byte]int` for frequency counting.

A small Go-specific detail is converting characters into numeric values:

```go
int(num[i] - '0')
```

This converts a digit character like `'3'` into integer `3`.

To retrieve the frequency of digit `i`, we convert the integer index back into a character:

```go
byte(i) + '0'
```

Go maps return the zero value for missing keys automatically, so missing digits correctly produce frequency `0`.

## Edge Cases

One important edge case is a string of length 1. Consider `"0"`. The value at index `0` says digit `0` should appear zero times, but the string itself contains one `'0'`. This can easily cause mistakes if the implementation assumes the smallest input is automatically valid. The algorithm handles this correctly because it explicitly compares the expected and actual counts.

Another important case involves digits that should appear zero times. For example, in `"1210"`, digit `3` is expected to appear zero times. A buggy implementation might fail when looking up missing digits in the frequency map. Using `.get(..., 0)` in Python and Go's default map behavior ensures absent digits correctly count as zero.

A third edge case is when multiple conditions fail simultaneously, such as `"030"`. The algorithm should immediately return `false` upon detecting the first mismatch rather than continuing unnecessarily. This early termination keeps the implementation efficient and straightforward.

Another subtle case occurs when digits appear more times than expected. For example, `"0000"` expects digit `0` to appear zero times, but it actually appears four times. Since the algorithm counts actual frequencies first and compares exactly, these overcount situations are handled naturally and correctly.
