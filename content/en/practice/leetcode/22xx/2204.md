**Topics:** Depth-First Search, Breadth-First Search, Graph Theory, Topological Sort

## Solution

## Problem Understanding

This problem asks us to compute the **minimum distance from every node to a cycle** in a connected undirected graph that contains **exactly one cycle**. The input consists of the number of nodes `n` and a list of bidirectional edges `edges`. Each edge `[u, v]` indicates a connection between node `u` and node `v`. The nodes are numbered from `0` to `n - 1`. The output is an array `answer` of size `n`, where `answer[i]` is the minimum number of edges needed to reach any node that is part of the cycle.

The graph constraints are important: there is **exactly one cycle**, the graph is **connected**, and the number of nodes `n` can be as large as 100,000. This eliminates approaches that would repeatedly traverse the entire graph from each node, as they would be too slow. Also, the graph having exactly one cycle allows us to leverage **tree properties** outside the cycle, because all nodes not in the cycle form trees connected to the cycle.

Important edge cases include nodes that are **directly on the cycle**, nodes that are **far from the cycle** in a long branch, and graphs where the cycle is **small (3 nodes)** or **large (almost all nodes)**. The problem guarantees no self-loops and no multiple edges, so we do not need to handle those edge cases.

## Approaches

The brute-force approach would compute distances from each node to all other nodes using BFS or DFS until a cycle node is reached. For each node, we would check all possible paths to detect the cycle and compute distances. While this is correct, it would take **O(n²)** time in the worst case because we would perform a BFS from every node, which is infeasible for `n` up to 10^5.

The key insight for an optimal solution is to **first identify the cycle**, then compute distances from nodes outside the cycle using a **Breadth-First Search (BFS)** starting from all cycle nodes simultaneously. This works because nodes not in the cycle form trees attached to the cycle, and BFS from all cycle nodes guarantees the minimum distance is computed in one pass.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | BFS from each node to find nearest cycle node |
| Optimal | O(n) | O(n) | Detect cycle via DFS/topological pruning, then BFS from cycle nodes |

## Algorithm Walkthrough

1. **Build the adjacency list** from the edges. This allows efficient traversal of neighbors.
2. **Identify the cycle**. We can use a DFS with parent tracking. During DFS, if we encounter a visited node that is not the parent, we have found the cycle. Record all nodes in the cycle.
3. **Initialize distances**. For each node, set the distance to `0` if it is in the cycle and `-1` otherwise.
4. **Perform BFS** starting from all cycle nodes simultaneously. Push all cycle nodes into a queue with distance `0`.
5. **Expand BFS**. For each node popped from the queue, update the distance of all unvisited neighbors as `current_distance + 1` and push them to the queue.
6. **Return the distances** array once the queue is empty.

**Why it works:** The BFS ensures that distances propagate outward from the cycle. Since the graph is connected and acyclic outside the cycle, every node will be reached exactly once through the shortest path to the nearest cycle node.

## Python Solution

```python
from collections import deque
from typing import List

class Solution:
    def distanceToCycle(self, n: int, edges: List[List[int]]) -> List[int]:
        # Step 1: Build adjacency list
        adj = [[] for _ in range(n)]
        for u, v in edges:
            adj[u].append(v)
            adj[v].append(u)
        
        # Step 2: Detect cycle using DFS
        visited = [False] * n
        parent = [-1] * n
        cycle_nodes = set()
        
        def dfs(node, par):
            visited[node] = True
            for nei in adj[node]:
                if not visited[nei]:
                    parent[nei] = node
                    if dfs(nei, node):
                        return True
                elif nei != par:
                    # Cycle detected, backtrack to collect cycle nodes
                    cur = node
                    cycle_nodes.add(nei)
                    while cur != nei:
                        cycle_nodes.add(cur)
                        cur = parent[cur]
                    return True
            return False
        
        dfs(0, -1)
        
        # Step 3: Initialize distances
        dist = [-1] * n
        queue = deque()
        for node in cycle_nodes:
            dist[node] = 0
            queue.append(node)
        
        # Step 4: BFS from cycle nodes
        while queue:
            node = queue.popleft()
            for nei in adj[node]:
                if dist[nei] == -1:
                    dist[nei] = dist[node] + 1
                    queue.append(nei)
        
        return dist
```

The DFS detects the cycle and collects all nodes in the cycle. Then BFS propagates distances to all other nodes efficiently. This approach ensures linear time complexity with respect to the number of nodes.

## Go Solution

```go
package main

func distanceToCycle(n int, edges [][]int) []int {
    adj := make([][]int, n)
    for _, e := range edges {
        u, v := e[0], e[1]
        adj[u] = append(adj[u], v)
        adj[v] = append(adj[v], u)
    }

    visited := make([]bool, n)
    parent := make([]int, n)
    for i := 0; i < n; i++ {
        parent[i] = -1
    }
    cycleNodes := make(map[int]bool)

    var dfs func(int, int) bool
    dfs = func(node, par int) bool {
        visited[node] = true
        for _, nei := range adj[node] {
            if !visited[nei] {
                parent[nei] = node
                if dfs(nei, node) {
                    return true
                }
            } else if nei != par {
                cur := node
                cycleNodes[nei] = true
                for cur != nei {
                    cycleNodes[cur] = true
                    cur = parent[cur]
                }
                return true
            }
        }
        return false
    }

    dfs(0, -1)

    dist := make([]int, n)
    for i := 0; i < n; i++ {
        dist[i] = -1
    }

    queue := []int{}
    for node := range cycleNodes {
        dist[node] = 0
        queue = append(queue, node)
    }

    for len(queue) > 0 {
        node := queue[0]
        queue = queue[1:]
        for _, nei := range adj[node] {
            if dist[nei] == -1 {
                dist[nei] = dist[node] + 1
                queue = append(queue, nei)
            }
        }
    }

    return dist
}
```

In Go, we use a map to track cycle nodes instead of a set, and slices are used as queues. The logic mirrors the Python implementation.

| --- | --- | --- |

Linear complexity is achievable because the graph has exactly one cycle, allowing BFS to propagate distances efficiently.
