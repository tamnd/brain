## Solution

## Problem Understanding

The problem gives us a 2D integer array called `nums`. Each element of `nums` is itself an array containing distinct positive integers. Our task is to find every integer that appears in every single array and return those integers sorted in ascending order.

In simpler terms, we are looking for the intersection of all arrays. An integer belongs in the answer only if it exists in every subarray.

For example, if we have:

```
[[3,1,2,4,5],
 [1,2,3,4],
 [3,4,5,6]]
```

the integers `3` and `4` appear in all three arrays, so the result is:

```
[3,4]
```

The input guarantees that every individual array contains unique values. This is important because we do not need to worry about duplicate counting within the same subarray.

The constraints are relatively small:

- At most 1000 arrays
- Total number of integers across all arrays is at most 1000
- Integer values range from 1 to 1000

These constraints tell us several important things:

- A straightforward counting solution is fast enough.
- We do not need advanced optimization techniques.
- Hash maps or counting arrays are excellent choices.
- Sorting the final answer is inexpensive because the total amount of data is small.

Several edge cases are important to consider:

- Arrays may have no common elements, producing an empty result.
- There may be only one array, in which case every element belongs in the answer.
- Values may appear in many arrays but miss one array, meaning they must not be included.
- Since values inside each array are guaranteed distinct, we do not need deduplication logic within a single row.

## Approaches

### Brute Force Approach

A brute force solution would examine every distinct number and check whether it exists in every array.

One way to do this is:

1. Collect all distinct values from all arrays.
2. For each value, scan every array and check membership.
3. If the value exists in every array, add it to the answer.
4. Sort the result.

This approach is correct because it explicitly verifies the definition of intersection. A number is included only if every array contains it.

However, repeated membership checks can become inefficient. If membership is checked using linear search, each lookup costs `O(length of array)`, which leads to unnecessary repeated work.

Although the constraints are small enough that this would still pass, it is not the cleanest or most efficient solution.

### Optimal Approach

The key insight is that we only care about how many arrays contain each number.

Since each array already contains unique elements, we can safely count occurrences across arrays using a hash map.

The algorithm becomes:

1. Traverse every number in every array.
2. Count how many arrays contain each number.
3. Any number whose count equals `len(nums)` appears in every array.
4. Collect such numbers and sort them.

This works efficiently because each number is processed only once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(U × N × M) | O(U) | Repeatedly scans arrays for membership checks |
| Optimal | O(T + K log K) | O(K) | Counts occurrences using a hash map |

Where:

- `U` = number of unique values
- `N` = number of arrays
- `M` = average array length
- `T` = total number of integers across all arrays
- `K` = number of distinct integers

## Algorithm Walkthrough

1. Create a hash map called `frequency`.

This map stores how many arrays contain each integer. The key is the integer itself, and the value is its count.
2. Traverse every array in `nums`.

We process each subarray one at a time.
3. Traverse every number inside the current array.

Since values inside each array are guaranteed distinct, every appearance represents one valid contribution toward the global count.
4. Increment the frequency count for the current number.

If `3` appears in three different arrays, its count eventually becomes `3`.
5. After processing all arrays, create an empty result list.

This list will store integers present in every array.
6. Iterate through the frequency map.

For each `(number, count)` pair, check whether:

```
count == len(nums)
```

If true, the number appeared in every array.

1. Add qualifying numbers to the result list.
2. Sort the result in ascending order.

The problem explicitly requires sorted output.
3. Return the sorted result list.

### Why it works

The algorithm relies on a simple invariant:

> A number belongs in the final intersection if and only if it appears in every array.

Because we increment the count exactly once for each array containing the number, the final count precisely equals the number of arrays where it appears. Therefore, any number whose frequency equals `len(nums)` must exist in all arrays, guaranteeing correctness.

## Python Solution

```python
from typing import List
from collections import Counter

class Solution:
    def intersection(self, nums: List[List[int]]) -> List[int]:
        frequency = Counter()

        for array in nums:
            for number in array:
                frequency[number] += 1

        result = []

        for number, count in frequency.items():
            if count == len(nums):
                result.append(number)

        result.sort()

        return result
```

The implementation begins by importing `Counter` from the `collections` module. A `Counter` behaves like a hash map specialized for counting frequencies.

We then iterate through every array and every number inside it. Each number increases its count in the frequency map.

After counting is complete, we examine every `(number, count)` pair. If the count equals the total number of arrays, the number appears everywhere and is added to the result list.

Finally, the result list is sorted before being returned because the problem requires ascending order.

## Go Solution

```go
package main

import "sort"

func intersection(nums [][]int) []int {
	frequency := make(map[int]int)

	for _, array := range nums {
		for _, number := range array {
			frequency[number]++
		}
	}

	result := []int{}

	for number, count := range frequency {
		if count == len(nums) {
			result = append(result, number)
		}
	}

	sort.Ints(result)

	return result
}
```

The Go implementation follows the same logic as the Python version.

A `map[int]int` is used to store frequencies. We iterate through all arrays and increment counts for each number.

The result is stored in a slice. After collecting valid intersection elements, we sort the slice using `sort.Ints`.

One small language difference is that Go maps return zero values automatically for missing keys, making frequency counting concise without explicit initialization.

## Edge Cases

One important edge case occurs when there is only one array. In this situation, every element automatically belongs in the intersection because there are no other arrays to compare against. The implementation handles this naturally because each element's frequency becomes `1`, which equals `len(nums)`.

Another important case is when no numbers are shared across all arrays. A buggy implementation might accidentally include partially shared values. Our solution avoids this because only numbers whose count exactly matches the total number of arrays are included.

A third edge case involves arrays appearing in different orders. Since the algorithm uses frequency counting instead of positional comparison, ordering does not matter. The final sorting step ensures the returned answer is always in ascending order regardless of input arrangement.

A final subtle edge case is duplicate handling. The problem guarantees that values inside each array are unique. This matters because otherwise a number could be overcounted within a single array. Since uniqueness is guaranteed, incrementing the frequency directly is always correct.
