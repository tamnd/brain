[LeetCode Problem 2246](https://leetcode.com/problems/longest-path-with-different-adjacent-characters/)

**Difficulty:** 🔴 Hard  
**Topics:** Array, String, Tree, Depth-First Search, Graph Theory, Topological Sort

## Solution

## Problem Understanding

The problem gives us a **tree** structure, represented by a `parent` array where `parent[i]` indicates the parent of node `i`, and a string `s` where `s[i]` is the character assigned to node `i`. The root node is `0`, and `parent[0] == -1`. The task is to find the **length of the longest path** in the tree such that **no two adjacent nodes on the path share the same character**.

Restated, you can think of the tree as a set of nodes connected without cycles, and each node has a label. Starting from any node, you want to walk along the edges to maximize the number of nodes in a path while ensuring that neighboring nodes on the path always have **different characters**. The length of a path is the **number of nodes included**.

Constraints tell us that `n` can be up to 100,000, which rules out any solution that tries to enumerate all paths explicitly, as the number of paths grows exponentially. Each node has exactly one parent (except the root) and can have multiple children. The string `s` contains only lowercase letters.

Important edge cases include a tree where all nodes have the same character (so the longest path is just 1), a tree that is essentially a straight line (a degenerate tree), or a tree where every node has a unique character.

## Approaches

### Brute Force

The brute-force approach would involve generating **all paths** in the tree and checking if adjacent characters differ. For a tree with `n` nodes, this quickly becomes **exponentially large** because each node can lead to multiple children, so enumerating all paths is not feasible. While correct, it will time out for large `n`.

### Optimal Approach

The key insight is that this is a **tree problem**, and the longest path can be found using a **Depth-First Search (DFS)**. At each node, the longest path that includes this node can be formed by combining the **two longest child paths that do not conflict with the current node's character**.

1. Construct a tree as an adjacency list from the `parent` array.
2. Recursively compute the longest path starting from each child.
3. If a child has the same character as the current node, ignore its contribution.
4. Maintain the **top two longest paths** among children with different characters. The sum of these two paths plus one (for the current node) may update the global maximum path length.
5. Return the longest path from the current node to its parent to allow the parent to compute its longest path.

This method ensures **every node is visited once**, and we propagate only relevant path lengths upward.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Enumerates all paths; infeasible for n up to 1e5 |
| Optimal | O(n) | O(n) | DFS propagation, stores adjacency list and uses recursion |

## Algorithm Walkthrough

1. **Build adjacency list**: Convert the `parent` array to a dictionary mapping each node to its children. This allows easy traversal in DFS.
2. **DFS function**: Define a recursive DFS function that returns the **length of the longest path starting from the current node** going downwards with the constraint of adjacent characters differing.
3. **Track top paths**: For each node, iterate through its children and call DFS recursively. If the child's character differs from the current node's character, consider its path length for combination.
4. **Update global maximum**: Keep track of the **two largest child paths**. Their sum plus 1 (the current node itself) may form a path through this node. Update a global maximum variable if this is larger than the current maximum.
5. **Return single path**: Return the **longest single child path plus one** to the parent, as only one path can be extended upwards through the parent.
6. **Invoke DFS from root**: Start DFS at node 0. The global maximum will store the answer.

**Why it works**: This works because in a tree, the longest path is always either a single chain from the root down or a path that passes through some node and includes its two longest child paths with differing characters. By recursively combining the two longest child paths that satisfy the character constraint, we cover all possibilities efficiently.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def longestPath(self, parent: List[int], s: str) -> int:
        tree = defaultdict(list)
        n = len(parent)
        
        for i in range(1, n):
            tree[parent[i]].append(i)
        
        self.max_path = 1
        
        def dfs(node: int) -> int:
            longest = 0
            second_longest = 0
            
            for child in tree[node]:
                child_len = dfs(child)
                if s[child] != s[node]:
                    if child_len > longest:
                        second_longest = longest
                        longest = child_len
                    elif child_len > second_longest:
                        second_longest = child_len
            
            self.max_path = max(self.max_path, longest + second_longest + 1)
            return longest + 1
        
        dfs(0)
        return self.max_path
```

**Implementation Walkthrough**

We first build an adjacency list `tree` from the `parent` array. `self.max_path` keeps the global maximum length. The `dfs` function calculates the two longest child paths for each node, ignoring children with the same character. After processing all children, the sum of the two longest valid paths plus 1 is a candidate for the maximum path. Returning `longest + 1` allows the parent to consider extending the path through this node.

## Go Solution

```go
package main

func longestPath(parent []int, s string) int {
    n := len(parent)
    tree := make([][]int, n)
    for i := 1; i < n; i++ {
        tree[parent[i]] = append(tree[parent[i]], i)
    }
    
    maxPath := 1
    
    var dfs func(int) int
    dfs = func(node int) int {
        longest, secondLongest := 0, 0
        for _, child := range tree[node] {
            childLen := dfs(child)
            if s[child] != s[node] {
                if childLen > longest {
                    secondLongest = longest
                    longest = childLen
                } else if childLen > secondLongest {
                    secondLongest = childLen
                }
            }
        }
        if longest+secondLongest+1 > maxPath {
            maxPath = longest + secondLongest + 1
        }
        return longest + 1
    }
    
    dfs(0)
    return maxPath
}
```

**Go Notes**: In Go, slices represent the adjacency list. Recursive DFS is straightforward, and `maxPath` is updated as in Python. Integer overflow is not an issue here since the path length cannot exceed `n`.

## Worked Examples

| Node | Children | DFS Computation | Top Paths |
| --- | --- | --- | --- |

| 4 | [] | return 1 | longest=0, second_longest=0 |
| 1 | [3,4] | longest paths | longest=1, second_longest=1 |
| 5 | [] | return 1 | longest=0, second_longest=0 |
| 2 | [5] | longest=1 | second_longest=0 |
| 0 | [1,2] | combine 2 paths | longest=2, second_longest=1 -> max_path=3 |

Output: 3

**Example 2**: `parent = [-1,0,0,0], s = "aabc"`

| Node | Children | DFS Computation |
| --- | --- | --- |
| 1 | [] | return 1 |
| 2 | [] | return 1 |
| 3 | [] | return 1 |
| 0 | [1,2,3] | top two differing paths = 2+1 |

Output: 3

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited once in DFS |
| Space | O(n) | Adjacency list and recursion stack, both proportional to n |

The recursive DFS ensures that we do not explore any node more than once, making this optimal. Memory usage is linear due to tree representation and recursion depth.
