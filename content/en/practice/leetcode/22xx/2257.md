## Solution

## Problem Understanding

The problem asks us to determine how many cells in a grid are **unguarded**, given positions of guards and walls. The grid is defined by dimensions `m x n`, and both guards and walls are placed at unique coordinates. A guard can "see" in the four cardinal directions (north, east, south, west), and any cell in those directions is considered **guarded** unless blocked by another guard or a wall. The goal is to count the cells that are neither a wall nor a guard nor visible by any guard.

The input consists of the grid dimensions, a list of guard positions, and a list of wall positions. The output is a single integer representing the number of unguarded, unoccupied cells.

Constraints indicate that the grid can be quite large, up to `10^5 x 10^5` in theory, but the total number of cells is capped at `10^5`. This means a brute-force approach using a full 2D grid array is feasible. Additionally, the number of guards and walls is bounded, so operations per guard or wall need to be optimized to avoid redundant computations. Important edge cases include grids where guards or walls are at corners, multiple guards seeing overlapping areas, and walls blocking some directions completely.

## Approaches

The **brute-force approach** would involve creating a full `m x n` grid and marking each cell as guarded by iterating in four directions from each guard until a wall or another guard is encountered. While this is straightforward and guarantees correctness, it may repeatedly mark the same cells and can be inefficient when the number of guards is large.

The **optimal approach** relies on recognizing that a guard's line of sight in each direction can be simulated efficiently by scanning rows and columns separately. Instead of revisiting cells, we can maintain a single grid marking which cells are walls, guards, or already guarded. For each row, we scan left to right and right to left, marking all visible cells until a wall or guard is hit. Similarly, for each column, we scan top to bottom and bottom to top. This ensures each cell is checked at most twice per row and column, providing efficiency.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m * n * (guards)) | O(m * n) | Mark cells for each guard's sight direction explicitly |
| Optimal | O(m * n + guards + walls) | O(m * n) | Scan each row and column, marking cells efficiently without redundant work |

## Algorithm Walkthrough

1. Initialize a 2D `grid` of size `m x n`. Use integer values to represent the state of each cell: `0` for empty, `1` for guard, `2` for wall, `3` for guarded.
2. Place guards and walls in the grid using the input lists. Guards are `1`, walls are `2`.
3. For each row, perform two scans: left-to-right and right-to-left. During each scan, maintain a `seen_guard` flag. If a guard is encountered, set `seen_guard` to true. If a wall is encountered, reset `seen_guard` to false. Any empty cell with `seen_guard` true is marked as guarded (`3`).
4. Repeat the same logic for each column: top-to-bottom and bottom-to-top.
5. Finally, iterate through the grid and count cells that remain empty (`0`), as these are unguarded and unoccupied.
6. Return this count.

**Why it works:** By scanning in all four directions and stopping at walls or other guards, every visible cell from a guard is correctly marked. Empty cells that remain unmarked after all scans are guaranteed to be unguarded, fulfilling the problem requirements.

## Python Solution

```python
from typing import List

class Solution:
    def countUnguarded(self, m: int, n: int, guards: List[List[int]], walls: List[List[int]]) -> int:
        grid = [[0] * n for _ in range(m)]
        
        for r, c in guards:
            grid[r][c] = 1
        for r, c in walls:
            grid[r][c] = 2
        
        # Mark rows
        for r in range(m):
            seen_guard = False
            for c in range(n):
                if grid[r][c] == 1:
                    seen_guard = True
                elif grid[r][c] == 2:
                    seen_guard = False
                elif seen_guard:
                    grid[r][c] = 3
            seen_guard = False
            for c in range(n-1, -1, -1):
                if grid[r][c] == 1:
                    seen_guard = True
                elif grid[r][c] == 2:
                    seen_guard = False
                elif seen_guard:
                    grid[r][c] = 3
        
        # Mark columns
        for c in range(n):
            seen_guard = False
            for r in range(m):
                if grid[r][c] == 1:
                    seen_guard = True
                elif grid[r][c] == 2:
                    seen_guard = False
                elif seen_guard:
                    grid[r][c] = 3
            seen_guard = False
            for r in range(m-1, -1, -1):
                if grid[r][c] == 1:
                    seen_guard = True
                elif grid[r][c] == 2:
                    seen_guard = False
                elif seen_guard:
                    grid[r][c] = 3
        
        return sum(cell == 0 for row in grid for cell in row)
```

The implementation first initializes the grid and places guards and walls. Each row and column is scanned in both directions to mark cells seen by guards. Finally, the unguarded cells are counted by checking which cells remain `0`.

## Go Solution

```go
func countUnguarded(m int, n int, guards [][]int, walls [][]int) int {
    grid := make([][]int, m)
    for i := range grid {
        grid[i] = make([]int, n)
    }
    
    for _, g := range guards {
        grid[g[0]][g[1]] = 1
    }
    for _, w := range walls {
        grid[w[0]][w[1]] = 2
    }
    
    // Mark rows
    for r := 0; r < m; r++ {
        seen := false
        for c := 0; c < n; c++ {
            if grid[r][c] == 1 {
                seen = true
            } else if grid[r][c] == 2 {
                seen = false
            } else if seen {
                grid[r][c] = 3
            }
        }
        seen = false
        for c := n-1; c >= 0; c-- {
            if grid[r][c] == 1 {
                seen = true
            } else if grid[r][c] == 2 {
                seen = false
            } else if seen {
                grid[r][c] = 3
            }
        }
    }
    
    // Mark columns
    for c := 0; c < n; c++ {
        seen := false
        for r := 0; r < m; r++ {
            if grid[r][c] == 1 {
                seen = true
            } else if grid[r][c] == 2 {
                seen = false
            } else if seen {
                grid[r][c] = 3
            }
        }
        seen = false
        for r := m-1; r >= 0; r-- {
            if grid[r][c] == 1 {
                seen = true
            } else if grid[r][c] == 2 {
                seen = false
            } else if seen {
                grid[r][c] = 3
            }
        }
    }
    
    count := 0
    for r := 0; r < m; r++ {
        for c := 0; c < n; c++ {
            if grid[r][c] == 0 {
                count++
            }
        }
    }
    
    return count
}
```

The Go implementation mirrors the Python version, using slices to represent the grid. Loops handle marking cells seen by guards in all directions. Differences are mainly syntax: slice initialization, range loops, and explicit index handling.
