## Solution

## Problem Understanding

The problem asks us to find all **k-distant indices** in a given array `nums`. A k-distant index `i` is any index where there exists at least one index `j` such that `nums[j]` equals `key` and the distance between `i` and `j` is at most `k`, i.e., `|i - j| <= k`. Essentially, we are looking for indices that are “close enough” to any occurrence of the `key`.

The input consists of a list of integers `nums`, an integer `key` which is guaranteed to exist in the array, and an integer `k` defining the maximum distance to consider. The output is a sorted list of all indices meeting the k-distant condition.

Constraints indicate that the array length is moderate (`1 <= nums.length <= 1000`), so even O(n²) solutions may run, but a more efficient approach is preferred. Edge cases to consider include when all elements are equal to `key`, when `k` is very small (`k = 1`), or when `k` is equal to the array length.

## Approaches

The **brute-force approach** iterates through every index `i` of the array and, for each `i`, checks all indices `j` to see if `nums[j] == key` and `|i - j| <= k`. This is correct but inefficient, because it results in a nested loop: O(n²) time complexity.

The **optimal approach** leverages the fact that we only need to know the indices of `key` elements. First, we record all positions of `key`. Then, for each index `i`, we check if it is within `k` distance of any `key` index. This can be done efficiently by iterating over the key positions with a moving pointer since the key indices are sorted. Alternatively, we can mark all ranges `[max(0, j-k), min(n-1, j+k)]` for each `key` index `j` and then collect all indices marked, resulting in a simple O(n) pass.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every pair of indices to verify the k-distance condition. |
| Optimal | O(n) | O(n) | Marks ranges around key indices and collects all marked indices efficiently. |

## Algorithm Walkthrough

1. Initialize a boolean array `isKDistant` of size `n` (length of `nums`) to `False`. This will mark which indices are k-distant.
2. Iterate through `nums` to find every index `j` where `nums[j] == key`.
3. For each `key` index `j`, calculate the range `[start, end]` where `start = max(0, j - k)` and `end = min(n-1, j + k)`.
4. Mark all indices in this range as `True` in `isKDistant`.
5. After processing all key indices, iterate through `isKDistant` and collect all indices marked as `True`.
6. Return the collected indices as the final sorted result.

**Why it works:** Every index marked in `isKDistant` is guaranteed to be within `k` distance of at least one occurrence of `key`. By constructing ranges around each key index, we ensure no valid k-distant index is missed, and each index is included only once.

## Python Solution

```python
from typing import List

class Solution:
    def findKDistantIndices(self, nums: List[int], key: int, k: int) -> List[int]:
        n = len(nums)
        is_k_distant = [False] * n
        
        for j, val in enumerate(nums):
            if val == key:
                start = max(0, j - k)
                end = min(n - 1, j + k)
                for i in range(start, end + 1):
                    is_k_distant[i] = True
        
        result = [i for i, val in enumerate(is_k_distant) if val]
        return result
```

**Implementation Walkthrough:** First, we prepare a boolean list `is_k_distant` to track valid indices. We iterate through the array to find all key occurrences. For each key index, we calculate the valid range of k-distant indices and mark them. Finally, we enumerate the boolean list and collect indices that are marked `True`. This ensures correct, sorted output.

## Go Solution

```go
func findKDistantIndices(nums []int, key int, k int) []int {
    n := len(nums)
    isKDistant := make([]bool, n)
    
    for j, val := range nums {
        if val == key {
            start := j - k
            if start < 0 {
                start = 0
            }
            end := j + k
            if end >= n {
                end = n - 1
            }
            for i := start; i <= end; i++ {
                isKDistant[i] = true
            }
        }
    }
    
    result := []int{}
    for i, val := range isKDistant {
        if val {
            result = append(result, i)
        }
    }
    
    return result
}
```

**Go Notes:** The Go version is analogous to Python, using slices and a boolean array. The main differences are explicit slice creation and `append` for the result. Range checking ensures we do not go out of bounds.

## Complexity Analysis

| Measure | Complexity | Explanation |

| Time | O(n) | Iterate through the array once, and for each key index mark a range at most 2k+1 elements. Since total marks ≤ n * 2 (n <= 1000), effectively O(n). |
| Space | O(n) | Boolean array of size n to track k-distant indices. |

This is efficient for the given constraints and guarantees linear time performance.

## Edge Cases

**Case 1: Single-element array**

If `nums` has only one element, and it is the `key`, the function should return `[0]`. Implementation handles this because range calculation respects array bounds.

**Case 2: k equals array length**

If `k` is large enough to cover the entire array, all indices are valid. The `min` and `max` functions ensure that the range does not exceed array bounds, so the full array is correctly returned.

**Case 3: Consecutive keys**

When multiple keys appear consecutively, ranges may overlap. The boolean array approach naturally merges overlapping ranges, ensuring indices are not duplicated and all valid indices are included.
