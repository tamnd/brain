## Solution

## Problem Understanding

The problem asks us to find the **largest "good" integer** in a given string `num`. Here, a good integer is defined as a **3-character substring** of `num` in which all three characters are identical. The output should be the **string representation** of the largest such integer, or an empty string `""` if no such integer exists.

The input `num` is a string consisting of digits only, with a length between 3 and 1000 inclusive. Since the length is small, a solution that examines all substrings of length 3 is feasible, but we still want an efficient approach.

Key points to note are:

1. Substrings are contiguous, so we only consider sequences of exactly three consecutive characters.
2. Leading zeroes are valid, so "000" is a valid good integer.
3. The largest good integer is determined lexicographically since the substrings consist only of digits.
4. Edge cases include strings with no repeating triplets, strings consisting entirely of the same digit, and strings with multiple good integers where the largest one must be selected.

Important edge cases are:

- Input has no good integers: `"123456"`.
- Input has multiple good integers: `"6777133339"`.
- Input has leading zeros: `"2300019"`.

## Approaches

### Brute Force Approach

A brute-force approach iterates through every substring of length 3 in `num` and checks if all three characters are the same. If so, we compare it with the current maximum good integer found and update the maximum if necessary. This guarantees correctness because it examines all possible candidates.

However, this approach, while correct, does unnecessary comparisons and uses repeated string slicing, which can be slightly inefficient. For `num` of length `n`, there are `n-2` substrings to check, giving a time complexity of `O(n)` but with some string operations overhead.

### Optimal Approach

The optimal approach takes advantage of the fact that a good integer is defined by three **identical consecutive digits**. We can scan through the string once, checking if `num[i] == num[i+1] == num[i+2]`. If this condition is true, we form a substring and compare it with the current maximum good integer.

Since the string length is ≤ 1000, this single pass is efficient and avoids extra slicing or list construction. Lexicographical comparison of strings is sufficient because digit characters compare naturally (e.g., `'9' > '7' > '0'`).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Check all substrings of length 3 and update maximum |
| Optimal | O(n) | O(1) | Single pass, check consecutive triplets, update maximum |

## Algorithm Walkthrough

1. Initialize an empty string `max_good` to store the maximum good integer found.
2. Iterate through `num` from index `0` to `len(num) - 3` inclusive.
3. At each index `i`, check if `num[i]`, `num[i+1]`, and `num[i+2]` are identical.
4. If they are identical, form the substring `num[i:i+3]`.
5. Compare this substring with `max_good` lexicographically. If it is larger, update `max_good`.
6. Continue until the end of the string.
7. Return `max_good`. If no good integer was found, this will return an empty string.

**Why it works:** The algorithm maintains the invariant that `max_good` always holds the largest good integer found so far. By examining every consecutive triplet exactly once, we ensure no potential good integer is missed, and lexicographical comparison ensures we always store the largest one.

## Python Solution

```python
class Solution:
    def largestGoodInteger(self, num: str) -> str:
        max_good = ""
        n = len(num)
        for i in range(n - 2):
            if num[i] == num[i+1] == num[i+2]:
                candidate = num[i:i+3]
                if candidate > max_good:
                    max_good = candidate
        return max_good
```

**Implementation Explanation:** The variable `max_good` stores the largest good integer found during iteration. The loop goes from `0` to `n-3` to avoid index errors when checking triplets. Whenever three consecutive identical digits are found, they are compared to `max_good` to update the maximum. At the end, `max_good` is returned, or `""` if no triplet was found.

## Go Solution

```go
func largestGoodInteger(num string) string {
    maxGood := ""
    n := len(num)
    for i := 0; i <= n-3; i++ {
        if num[i] == num[i+1] && num[i] == num[i+2] {
            candidate := num[i:i+3]
            if candidate > maxGood {
                maxGood = candidate
            }
        }
    }
    return maxGood
}
```

**Go-specific Notes:** Go slices `num[i:i+3]` work similarly to Python slicing. Lexicographical comparison of strings in Go is straightforward with `>`. There is no need for explicit type conversion since `num` is already a string.

## Edge Cases

1. **No good integer exists:** Input such as `"123456"` has no three consecutive identical digits. The algorithm correctly returns `""` because the loop never finds a valid triplet.
2. **All digits identical:** Input such as `"111111"` contains overlapping triplets. The algorithm will evaluate each triplet, updating `max_good` as needed, and ultimately return `"111"` as expected.
3. **Leading zeros and largest integer not at start:** Input such as `"001122333444"` ensures that the algorithm correctly identifies the largest good integer (`"444"`) even when smaller triplets appear earlier. Lexicographical comparison ensures the correct selection.
