## Solution

## Problem Understanding

The problem asks us to find the distinct integers that appear in one array but not in the other. Specifically, given two integer arrays `nums1` and `nums2`, we must return a two-element list where the first element contains all distinct numbers in `nums1` that are not in `nums2`, and the second element contains all distinct numbers in `nums2` that are not in `nums1`. The order of numbers in each list does not matter.

The input arrays can have lengths up to 1000 and can contain negative numbers down to -1000 and positive numbers up to 1000. These constraints indicate that a solution that uses linear time relative to the array size is feasible, and we should also handle duplicate elements in each array gracefully.

Edge cases to consider include arrays that are identical, arrays where all elements of one are missing from the other, arrays with repeated values, and arrays with minimal length of 1.

## Approaches

The **brute-force approach** involves iterating through each element of `nums1` and checking whether it exists in `nums2`, collecting elements that do not appear. Similarly, we iterate through `nums2` to find elements not in `nums1`. This works correctly but has a time complexity of $O(n \cdot m)$, where $n$ and $m$ are the lengths of the arrays, because each membership check is linear. It is inefficient for large arrays.

The **optimal approach** leverages sets to store the elements of `nums1` and `nums2`. Checking membership in a set is $O(1)$, so we can iterate through each array once and compute the set differences. This reduces the overall complexity to linear time relative to the array sizes.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m) | O(1) | Check each element in one array against the other |
| Optimal | O(n + m) | O(n + m) | Use sets to compute distinct differences efficiently |

## Algorithm Walkthrough

1. Convert `nums1` to a set called `set1` and `nums2` to a set called `set2`. This ensures we are only working with distinct elements and allows for O(1) membership checks.
2. Compute the difference `set1 - set2` to find all elements present in `nums1` but absent in `nums2`.
3. Compute the difference `set2 - set1` to find all elements present in `nums2` but absent in `nums1`.
4. Convert both set differences to lists and return them as a two-element list: `[list(set1 - set2), list(set2 - set1)]`.

**Why it works:** Sets remove duplicates and allow fast membership checking. By computing set differences, we directly obtain all elements exclusive to each array, which satisfies the problem requirements.

## Python Solution

```python
from typing import List

class Solution:
    def findDifference(self, nums1: List[int], nums2: List[int]) -> List[List[int]]:
        set1 = set(nums1)
        set2 = set(nums2)
        diff1 = list(set1 - set2)
        diff2 = list(set2 - set1)
        return [diff1, diff2]
```

The implementation first converts the input arrays to sets to ensure uniqueness. It then calculates the differences using the set subtraction operator `-`, which produces another set containing only elements not present in the other set. Finally, the resulting sets are converted to lists and returned in a list of size 2, as required.

## Go Solution

```go
func findDifference(nums1 []int, nums2 []int) [][]int {
    set1 := make(map[int]struct{})
    set2 := make(map[int]struct{})

    for _, num := range nums1 {
        set1[num] = struct{}{}
    }
    for _, num := range nums2 {
        set2[num] = struct{}{}
    }

    var diff1, diff2 []int
    for num := range set1 {
        if _, exists := set2[num]; !exists {
            diff1 = append(diff1, num)
        }
    }
    for num := range set2 {
        if _, exists := set1[num]; !exists {
            diff2 = append(diff2, num)
        }
    }

    return [][]int{diff1, diff2}
}
```

In Go, sets are implemented using maps with empty structs as values. The solution constructs two maps for `nums1` and `nums2` and then iterates through the keys to check membership and compute differences. The results are collected in slices and returned as a two-element slice of slices.

## Worked Examples

**Example 1:**

`nums1 = [1,2,3]`, `nums2 = [2,4,6]`

1. Convert to sets: `set1 = {1,2,3}`, `set2 = {2,4,6}`
2. Difference `set1 - set2 = {1,3}`
3. Difference `set2 - set1 = {4,6}`
4. Return `[[1,3],[4,6]]`

**Example 2:**

`nums1 = [1,2,3,3]`, `nums2 = [1,1,2,2]`

1. Convert to sets: `set1 = {1,2,3}`, `set2 = {1,2}`
2. Difference `set1 - set2 = {3}`
3. Difference `set2 - set1 = {}`
4. Return `[[3], []]`

The complexity is efficient for arrays of length up to 1000, which is well within the problem constraints.

## Edge Cases

One important edge case is when one or both arrays are empty. The algorithm handles this correctly because converting an empty array to a set results in an empty set, and the set difference with another set produces the other set or an empty set accordingly.

Another edge case involves arrays where all elements overlap but may be repeated multiple times. By converting to a set, duplicates are removed, ensuring the output only contains distinct elements.

A third edge case is when both arrays are identical. In this case, the set difference will be empty for both directions, and the algorithm correctly returns `[[], []]`. This validates that the solution handles maximum overlap scenarios correctly.
