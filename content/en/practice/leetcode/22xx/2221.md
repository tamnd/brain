## Solution

## Problem Understanding

The problem asks us to compute the **triangular sum** of an array of digits. The input is a **0-indexed integer array** `nums`, where each element is a digit from 0 to 9. The process to compute the triangular sum repeatedly transforms the array by summing adjacent elements modulo 10 until only one element remains. This single element is the triangular sum.

In simpler terms, for an array `[a0, a1, a2, ..., an-1]`, we repeatedly create a new array where each element is `(nums[i] + nums[i+1]) % 10`, shrinking the array by one element each time. This continues until a single element remains.

The **constraints** tell us that the array can be up to length 1000 and all values are small digits (0-9). This means a brute-force simulation is feasible in terms of array size but may be improved using combinatorial insights.

Important **edge cases** include arrays of length 1 (the triangular sum is the number itself) and arrays with repeated numbers, zeros, or maximum length (1000 elements).

## Approaches

The **brute-force approach** is a direct simulation of the problem statement. We iteratively compute new arrays of length `n-1` until only one element remains. This works because the problem explicitly defines this iterative process, but it is not optimal: for an array of length `n`, it performs about `n*(n-1)/2` operations.

The **optimal approach** uses **combinatorial mathematics**. Each element in the final triangular sum is a weighted combination of the original elements. Specifically, for `nums[i]`, its contribution to the final sum is `nums[i] * C(n-1, i) % 10`, where `C(n-1, i)` is the binomial coefficient modulo 10. This insight comes from recognizing the triangular pattern as Pascal’s triangle, where each number’s coefficient in the last element corresponds to the binomial coefficients.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Iteratively simulates array reduction |
| Optimal | O(n^2) | O(1) | Uses combinatorial coefficients to compute the result directly modulo 10 |

The optimal approach is mathematically more elegant but for `n <= 1000`, both approaches are feasible.

## Algorithm Walkthrough

1. Initialize a variable `result = 0` to store the final triangular sum.
2. Compute the **binomial coefficients modulo 10** using a row-by-row method similar to Pascal's triangle. Start with the first coefficient `1` for the first element.
3. For each index `i` from 0 to `n-1`, update `result = (result + nums[i] * C(n-1, i)) % 10`.
4. After all elements are processed, `result` holds the triangular sum.
5. Return `result`.

**Why it works**: Each number’s coefficient in the final triangular sum corresponds exactly to its binomial coefficient from Pascal’s triangle. Summing these weighted values modulo 10 gives the correct final single element, which matches the iterative simulation.

## Python Solution

```python
from typing import List

class Solution:
    def triangularSum(self, nums: List[int]) -> int:
        n = len(nums)
        coeff = [1]  # Initial row of Pascal's triangle modulo 10
        for _ in range(1, n):
            new_coeff = [1]
            for j in range(1, len(coeff)):
                new_coeff.append((coeff[j-1] + coeff[j]) % 10)
            new_coeff.append(1)
            coeff = new_coeff

        result = 0
        for i in range(n):
            result = (result + nums[i] * coeff[i]) % 10
        return result
```

The Python solution builds the coefficients row by row, ensuring that each coefficient is computed modulo 10 to prevent overflow. After building the coefficients for the last row, it multiplies each original number by its coefficient, sums them modulo 10, and returns the result.

## Go Solution

```go
func triangularSum(nums []int) int {
    n := len(nums)
    coeff := []int{1}

    for i := 1; i < n; i++ {
        newCoeff := []int{1}
        for j := 1; j < len(coeff); j++ {
            newCoeff = append(newCoeff, (coeff[j-1]+coeff[j])%10)
        }
        newCoeff = append(newCoeff, 1)
        coeff = newCoeff
    }

    result := 0
    for i := 0; i < n; i++ {
        result = (result + nums[i]*coeff[i]) % 10
    }
    return result
}
```

The Go implementation mirrors the Python logic. Slices are used instead of arrays, and modulo 10 arithmetic is performed throughout to maintain small numbers. Go handles integer arithmetic safely here because all values remain within the 32-bit integer range.

## Edge Cases

**Single-element arrays**: The algorithm immediately returns the element, preventing unnecessary computations.

**Arrays with zeros**: Ensures modulo arithmetic is handled correctly; zeros do not contribute to the sum.

**Maximum-length arrays**: The algorithm handles large `n` by computing coefficients row-by-row modulo 10, keeping all intermediate results small and avoiding overflow.

**Alternating patterns**: Arrays like `[1,0,1,0,...]` verify that the modulo addition works correctly at each reduction step.
