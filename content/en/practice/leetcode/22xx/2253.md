## Solution

## Problem Understanding

The problem is asking to transform a table that has a fixed number of columns for stores into a more normalized form where each row contains a single product, a single store, and the price of that product at that store. In the original `Products` table, each row represents a product and contains multiple columns for different stores, where each column may have a price or be null if the product is not sold at that store. The task is to "unpivot" this table so that every row in the result corresponds to a single `(product_id, store, price)` combination, and any null prices should be omitted.

The input is a `Products` table with a variable number of store columns. The output should have exactly three columns: `product_id`, `store`, and `price`. Each combination of product and store with a non-null price should appear as a row in the output.

Constraints are that there will be at least one store and at most 30, which is small enough that iterating over columns dynamically is feasible. The main complexity arises from the dynamic nature of the store column names: they are not fixed and vary across test cases. Edge cases to be careful about include products with all null prices, a single store, or maximum number of stores (30).

## Approaches

The naive approach is to manually write a `UNION ALL` statement for each store column. For example, if there are 5 stores, you could write a query that selects each `(product_id, store, price)` pair individually, filtering out nulls, and then unions all of them together. This works but is not scalable because the store names and number of stores are dynamic and change for each test case. Therefore, hardcoding column names is not practical.

The key insight for an optimal solution is to use **dynamic SQL with `INFORMATION_SCHEMA.COLUMNS`** to discover all store columns for the given table, and then generate an unpivoting query dynamically. This allows the solution to handle any number of stores without manually editing the SQL. By iterating through the store columns and creating a `SELECT` with `UNION ALL` for each, while filtering out null prices, the table can be unpivoted dynamically. This method scales well for up to 30 store columns and avoids hardcoding column names.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m) | O(n * m) | Hardcode `UNION ALL` for each store column, works but not dynamic |
| Optimal | O(n * m) | O(n * m) | Dynamically generate unpivot query using store column metadata; scalable for up to 30 stores |

Here `n` is the number of products, and `m` is the number of stores.

## Algorithm Walkthrough

1. Query the metadata of the `Products` table using `INFORMATION_SCHEMA.COLUMNS` to retrieve all column names except `product_id`. These columns represent the store names.
2. For each store column, generate a `SELECT` statement that selects `product_id`, the store name as a string literal, and the column value as `price`, with a `WHERE` clause filtering out null prices.
3. Combine all these individual `SELECT` statements using `UNION ALL` to produce a single query.
4. Execute this dynamic SQL to produce the unpivoted table with three columns: `product_id`, `store`, and `price`.
5. Return the result of the executed query.

**Why it works:** By iterating through all store columns and filtering null prices, we guarantee that every `(product_id, store, price)` combination in the output corresponds exactly to a valid product availability. Using `UNION ALL` ensures no rows are lost, and dynamically querying column names ensures that the solution works regardless of the store names.

## Python Solution

```python
import sqlite3

def UnpivotProducts():
    conn = sqlite3.connect(':memory:')
    cursor = conn.cursor()

    # Retrieve all store columns dynamically
    cursor.execute("PRAGMA table_info(Products)")
    columns = [row[1] for row in cursor.fetchall() if row[1] != 'product_id']

    # Build dynamic SELECT statements
    selects = []
    for col in columns:
        selects.append(f"SELECT product_id, '{col}' AS store, {col} AS price FROM Products WHERE {col} IS NOT NULL")

    # Combine all selects using UNION ALL
    query = " UNION ALL ".join(selects)
    cursor.execute(query)
    result = cursor.fetchall()
    return result
```

The implementation first queries the table schema to dynamically determine all store columns. For each store, it constructs a `SELECT` query that extracts the product ID, store name, and price, filtering out null prices. These queries are combined using `UNION ALL` to produce a single result set. Finally, executing the query returns the unpivoted table.

## Go Solution

```go
package main

import (
    "database/sql"
    _ "github.com/mattn/go-sqlite3"
    "fmt"
)

func UnpivotProducts(db *sql.DB) ([][3]interface{}, error) {
    // Retrieve all store columns dynamically
    rows, err := db.Query("PRAGMA table_info(Products)")
    if err != nil {
        return nil, err
    }
    defer rows.Close()

    var columns []string
    for rows.Next() {
        var cid int
        var name, ctype string
        var notnull, dflt, pk interface{}
        if err := rows.Scan(&cid, &name, &ctype, &notnull, &dflt, &pk); err != nil {
            return nil, err
        }
        if name != "product_id" {
            columns = append(columns, name)
        }
    }

    // Build dynamic SELECT statements
    selects := ""
    for i, col := range columns {
        if i > 0 {
            selects += " UNION ALL "
        }
        selects += fmt.Sprintf("SELECT product_id, '%s' AS store, %s AS price FROM Products WHERE %s IS NOT NULL", col, col, col)
    }

    resultRows, err := db.Query(selects)
    if err != nil {
        return nil, err
    }
    defer resultRows.Close()

    var result [][3]interface{}
    for resultRows.Next() {
        var product_id int
        var store string
        var price int
        if err := resultRows.Scan(&product_id, &store, &price); err != nil {
            return nil, err
        }
        result = append(result, [3]interface{}{product_id, store, price})
    }

    return result, nil
}
```

The Go implementation is very similar to Python. It uses `PRAGMA table_info` to retrieve column names, constructs dynamic SQL with `UNION ALL`, and executes the query to return a slice of 3-element arrays representing `(product_id, store, price)`. Go requires explicit handling of `interface{}` and slices.

## Worked Examples

For the example input:

```
+------------+----------+--------+------+------+
| product_id | LC_Store | Nozama | Shop | Souq |
+------------+----------+--------+------+------+
| 1          | 100      | null   | 110  | null |
| 2          | null     | 200    | null | 190  |
| 3          | null     | null   | 1000 | 1900 |
```

The dynamic query will generate:

```sql
SELECT product_id, 'LC_Store' AS store, LC_Store AS price FROM Products WHERE LC_Store IS NOT NULL
UNION ALL
SELECT product_id, 'Nozama' AS store, Nozama AS price FROM Products WHERE Nozama IS NOT NULL
UNION ALL
SELECT product_id, 'Shop' AS store, Shop AS price FROM Products WHERE Shop IS NOT NULL
UNION ALL
SELECT product_id, 'Souq' AS store, Souq AS price FROM Products WHERE Souq IS NOT NULL
```

Executing this yields the output:

```
+------------+----------+-------+
| product_id | store    | price |
+------------+----------+-------+
| 1          | LC_Store | 100   |
| 1          | Shop     | 110   |
| 2          | Nozama   | 200   |
| 2          | Souq     | 190   |
| 3          | Shop     | 1000  |
| 3          | Souq     | 1900  |
```

## Complexity Analysis

| Measure | Complexity | Explanation |

| Time | O(n * m) | Need to scan all products and all store columns to filter non-null prices |

The solution scales linearly with the number of products and the number of stores, which is acceptable since stores are capped at 30.
