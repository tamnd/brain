## Solution

## Problem Understanding

The problem is asking us to maximize the profit from trading stocks given the **current price** of each stock (`present`), the **future price** a year from now (`future`), and a **budget** constraint. Each stock can be purchased at most once, and the total cost of purchased stocks cannot exceed the budget. The profit for a single stock is calculated as `future[i] - present[i]`. The goal is to select a subset of stocks that maximizes the total profit while staying within the budget.

The inputs are arrays of integers `present` and `future` with length `n` where `1 <= n <= 1000`, and an integer `budget` where `0 <= budget <= 1000`. Prices are also constrained to be between 0 and 100. These constraints suggest that a solution that is polynomial in both `n` and `budget` is feasible.

Important edge cases include stocks where the future price is **lower than or equal to** the present price, since buying such stocks would not yield profit. Another edge case is a budget too small to buy any profitable stock, in which case the maximum profit is 0.

## Approaches

The brute-force approach would be to consider **all possible subsets of stocks**, compute their total cost and total profit, and return the maximum profit among subsets whose cost does not exceed the budget. While this guarantees the correct answer, it is infeasible because the number of subsets is `2^n`, which is exponential and unacceptable for `n` up to 1000.

The key insight for a more efficient solution is to notice that this problem is a **variation of the 0/1 Knapsack problem**. Each stock can be treated as an item where the weight is the `present[i]` (cost to buy) and the value is `future[i] - present[i]` (profit). We can use dynamic programming to solve this efficiently.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Check all subsets of stocks, impractical for n=1000 |
| Dynamic Programming (0/1 Knapsack) | O(n * budget) | O(budget) | Treat profit as value and present[i] as weight, standard DP optimization |

## Algorithm Walkthrough

1. Compute the **profit for each stock** as `profit[i] = future[i] - present[i]`. Ignore stocks with `profit <= 0` since they do not contribute to maximum profit.
2. Initialize a **DP array `dp`** of size `budget + 1` where `dp[j]` represents the maximum profit achievable with budget `j`.
3. Iterate through each stock. For each stock, iterate **backwards through the DP array** from `budget` down to `present[i]`. Update `dp[j] = max(dp[j], dp[j - present[i]] + profit[i])`. This ensures that each stock is considered at most once.
4. After processing all stocks, the value `dp[budget]` will hold the maximum achievable profit.

Why it works: The DP approach maintains an invariant that `dp[j]` always contains the maximum profit achievable for that exact budget. By iterating backwards for each stock, we ensure that no stock is counted multiple times, which correctly models the 0/1 purchase restriction.

## Python Solution

```python
from typing import List

class Solution:
    def maximumProfit(self, present: List[int], future: List[int], budget: int) -> int:
        n = len(present)
        profit = [future[i] - present[i] for i in range(n) if future[i] > present[i]]
        costs = [present[i] for i in range(n) if future[i] > present[i]]
        
        dp = [0] * (budget + 1)
        
        for i in range(len(profit)):
            for j in range(budget, costs[i] - 1, -1):
                dp[j] = max(dp[j], dp[j - costs[i]] + profit[i])
        
        return dp[budget]
```

The implementation first filters out unprofitable stocks, computes the corresponding costs and profits, and then uses a 1D DP array to solve the knapsack problem. Iterating backwards ensures no stock is counted twice.

## Go Solution

```go
func maximumProfit(present []int, future []int, budget int) int {
    n := len(present)
    profits := []int{}
    costs := []int{}
    
    for i := 0; i < n; i++ {
        if future[i] > present[i] {
            profits = append(profits, future[i] - present[i])
            costs = append(costs, present[i])
        }
    }
    
    dp := make([]int, budget+1)
    
    for i := 0; i < len(profits); i++ {
        for j := budget; j >= costs[i]; j-- {
            if dp[j] < dp[j - costs[i]] + profits[i] {
                dp[j] = dp[j - costs[i]] + profits[i]
            }
        }
    }
    
    return dp[budget]
}
```

In Go, we dynamically append the profitable stocks to separate slices. The DP array is initialized with zeros. Backward iteration ensures we correctly implement 0/1 knapsack semantics.

## Worked Examples

**Example 1:**

`present = [5,4,6,2,3]`, `future = [8,5,4,3,5]`, `budget = 10`

Profits: `[3, 1, -2, 1, 2]` → filter positive: `[3, 1, 1, 2]`

Costs: `[5, 4, 2, 3]`

| Budget | DP Updates |
| --- | --- |
| 10 | Buy stock 0 (profit 3) → dp[10] = 3 |
| 10 | Consider stock 1 (profit 1, cost 4) → dp[10] = max(3, dp[6] + 1) = 3 |
| 10 | Stock 2 (profit 1, cost 2) → dp[10] = max(3, dp[8] + 1) = 4 |
| 10 | Stock 3 (profit 2, cost 3) → dp[10] = max(4, dp[7] + 2) = 6 |

Maximum profit = 6.

**Example 2:** `present = [2,2,5]`, `future = [3,4,10]`, `budget = 6`

Profits: `[1,2,5]`, Costs: `[2,2,5]`

Maximum profit = 5 (buy stock 2).

**Example 3:** `present = [3,3,12]`, `future = [0,3,15]`, `budget = 10`

Profits: `[0,0,3]` → only stock 2 feasible cost exceeds budget → profit = 0.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * budget) | Each stock iterates over the DP array of size budget |
| Space | O(budget) | Only a 1D DP array is used |

The time complexity is efficient for `n <= 1000` and `budget <= 1000`, resulting in at most 1,000,000 iterations.
