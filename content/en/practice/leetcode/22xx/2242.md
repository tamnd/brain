## Solution

## Problem Understanding

The problem presents an **undirected graph** with `n` nodes, where each node has an associated `score`. You are asked to find the **maximum sum of scores** for a node sequence of length **4** such that every consecutive pair of nodes in the sequence is connected by an edge. Each node in the sequence must be unique. Essentially, this is about finding a 4-node path (not necessarily simple in terms of graph theory terminology, but each node appears only once) that maximizes the sum of the node scores.

The inputs are two-fold: an array `scores` where `scores[i]` is the score for node `i`, and a list of undirected edges. The output is a single integer: the maximum sum of a valid 4-node sequence, or `-1` if no such sequence exists.

The constraints indicate that the number of nodes and edges can be up to `50,000`. This rules out naive brute-force approaches that would enumerate all possible 4-node sequences explicitly, because there are $O(n^4)$ possible sequences, which is computationally infeasible.

Important edge cases include graphs where no path of length 4 exists, graphs that are sparsely connected, or nodes with extremely high scores that are not connected to others.

## Approaches

### Brute-Force Approach

The brute-force method would iterate over all possible 4-node combinations `(a, b, c, d)` and check if the edges `(a, b)`, `(b, c)`, `(c, d)` exist. For each valid sequence, compute the score and track the maximum. While this guarantees correctness, the time complexity is $O(n^4)$, which is unacceptable for $n = 5 \times 10^4$. Additionally, checking edge existence requires either another loop over edges or a hash set lookup, making it even slower.

### Optimal Approach

The key observation is that we only care about **sequences of length 4**. Instead of checking all 4-node combinations, we can reduce the search space by considering each **edge `(b, c)` as the middle of a potential sequence**. Then, we look for the **top neighbors of `b` and `c` with the highest scores**, excluding each other, to serve as nodes `a` and `d`.

The steps are:

1. For every node, maintain a sorted list of its **top 3 neighbors by score**.
2. Iterate through each edge `(b, c)` and attempt to form sequences `(a, b, c, d)` where `a` is a top neighbor of `b` (excluding `c`) and `d` is a top neighbor of `c` (excluding `b`).
3. For each valid candidate combination, compute the sum and track the maximum.

This approach reduces complexity because for each edge, you only consider a **few top neighbors**, not all neighbors, keeping the number of combinations small.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^4) | O(n + m) | Enumerate all sequences of length 4 and validate edges |
| Optimal | O(m * k^2) | O(n + m) | Use top `k` neighbors for each node; k=3 is sufficient because we only need max sequences of length 4 |

## Algorithm Walkthrough

1. Create an adjacency list for the graph where each node maps to a list of connected nodes.
2. For each node, sort its neighbors in **descending order of their scores**, keeping only the top 3 neighbors. This ensures we consider only the nodes that maximize potential sequences.
3. Initialize a variable `max_score` to `-1` to track the best sequence sum.
4. Iterate through every edge `(b, c)` in the graph. For each edge:

1. Consider node `b`'s top neighbors as potential `a` candidates, excluding `c`.
2. Consider node `c`'s top neighbors as potential `d` candidates, excluding `b`.
3. For every combination of `(a, b, c, d)`, ensure all nodes are distinct.
4. Compute the sequence score and update `max_score` if it is higher.
5. After iterating all edges, return `max_score`.

**Why it works:** By focusing on edges as the middle of sequences and only considering the top neighbors, we guarantee that we explore all sequences of length 4 that could potentially have the maximum score. Any sequence not considered would have a lower sum because it involves neighbors with lower scores than the top 3.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def maximumScore(self, scores: List[int], edges: List[List[int]]) -> int:
        n = len(scores)
        graph = defaultdict(list)
        
        # Build adjacency list
        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)
        
        # Keep top 3 neighbors by score for each node
        top_neighbors = {}
        for node in range(n):
            neighbors = graph[node]
            neighbors.sort(key=lambda x: scores[x], reverse=True)
            top_neighbors[node] = neighbors[:3]
        
        max_score = -1
        
        # Consider each edge as middle of a sequence
        for b, c in edges:
            for a in top_neighbors.get(b, []):
                if a == c:
                    continue
                for d in top_neighbors.get(c, []):
                    if d == b or d == a:
                        continue
                    total = scores[a] + scores[b] + scores[c] + scores[d]
                    max_score = max(max_score, total)
        
        return max_score
```

This Python solution builds an adjacency list and maintains the top 3 neighbors for each node. Iterating through each edge as a potential middle of a sequence ensures that we efficiently compute only relevant candidate sequences. The `max_score` variable tracks the best sequence.

## Go Solution

```go
package main

func maximumScore(scores []int, edges [][]int) int {
    n := len(scores)
    graph := make([][]int, n)
    
    // Build adjacency list
    for _, e := range edges {
        u, v := e[0], e[1]
        graph[u] = append(graph[u], v)
        graph[v] = append(graph[v], u)
    }
    
    // Function to get top 3 neighbors by score
    topNeighbors := make([][]int, n)
    for i := 0; i < n; i++ {
        neighbors := graph[i]
        // Simple selection of top 3
        for j := 0; j < len(neighbors)-1; j++ {
            for k := j + 1; k < len(neighbors); k++ {
                if scores[neighbors[k]] > scores[neighbors[j]] {
                    neighbors[j], neighbors[k] = neighbors[k], neighbors[j]
                }
            }
        }
        if len(neighbors) > 3 {
            neighbors = neighbors[:3]
        }
        topNeighbors[i] = neighbors
    }
    
    maxScore := -1
    for _, e := range edges {
        b, c := e[0], e[1]
        for _, a := range topNeighbors[b] {
            if a == c {
                continue
            }
            for _, d := range topNeighbors[c] {
                if d == b || d == a {
                    continue
                }
                total := scores[a] + scores[b] + scores[c] + scores[d]
                if total > maxScore {
                    maxScore = total
                }
            }
        }
    }
    
    return maxScore
}
```

In Go, slices are used for adjacency lists and top neighbors. The top 3 neighbors are selected using a simple nested loop to sort by scores. The logic mirrors the Python implementation, iterating through edges and considering top neighbors to compute the maximum sequence sum.
