## Solution

## Problem Understanding

The problem asks us to compute the **sum of scores of built strings** when constructing a string `s` from the end to the start. You are building the string by prepending characters, so `s1` is the last character of `s`, `s2` is the last two characters, and so on, up to `sn`, which is the full string `s`. Each `si` has a **score**, defined as the length of the **longest common prefix** (LCP) between `si` and the final string `s`.

For example, if `s = "babab"`, the built strings are:

- `s1 = "b"` → LCP with `s = "babab"` is `"b"` → score = 1
- `s2 = "ab"` → LCP = `""` → score = 0
- `s3 = "bab"` → LCP = `"bab"` → score = 3
- `s4 = "abab"` → LCP = `""` → score = 0
- `s5 = "babab"` → LCP = `"babab"` → score = 5

The task is to **compute the sum of all scores** efficiently.

Constraints indicate that `s` can be up to 100,000 characters, which makes a brute-force comparison of every prefix with the entire string impractical, as that would result in O(n²) time complexity.

Key edge cases include:

- Strings of length 1, which trivially have a score equal to 1.
- Strings with repeated characters, which might create overlapping LCP patterns.
- Strings with no repeated patterns, where only `sn` contributes a non-zero score.

Understanding these ensures we design a solution that handles large inputs efficiently and avoids naive string comparisons.

## Approaches

### Brute Force Approach

The brute-force approach is straightforward: for each `si`, compare it with `s` character by character until the LCP ends. Sum all these scores. This is correct but inefficient, as each comparison can take O(n) and there are n strings, resulting in **O(n²)** time complexity. For large strings, this approach is far too slow and will not meet time limits.

### Optimal Approach

The key insight for an optimal solution comes from string algorithms, particularly the **Z-algorithm** (or Z-function). The Z-array for a string `s` stores for each position `i` the length of the longest substring starting at `i` that matches the prefix of `s`. In this problem, we can construct a Z-array for the reversed string or directly use it on `s` to efficiently find the LCP of each `si` with `s` in **O(n)** time.

The Z-algorithm works in linear time because it reuses previous matches to skip redundant comparisons, making it ideal for this problem.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Compare each si with s character by character |
| Optimal (Z-algorithm) | O(n) | O(n) | Compute LCPs using Z-array for efficient string matching |

## Algorithm Walkthrough

1. Compute the **Z-array** of the string `s`. Each `Z[i]` represents the length of the longest prefix of `s[i:]` that matches the prefix of `s`.
2. Initialize the sum of scores with `n` (the score of `sn` with itself, which is the full string).
3. For each index `i` from 1 to `n-1`, add `Z[i]` to the total sum. `Z[i]` effectively represents the LCP between the suffix starting at `i` (which corresponds to `si` in our prepending construction) and the full string `s`.
4. Return the total sum.

**Why it works:** The Z-array captures exactly the information needed to compute the LCP for each built string without repeated comparisons. The score of each `si` corresponds directly to a value in the Z-array, so summing these values gives the correct total.

## Python Solution

```python
class Solution:
    def sumScores(self, s: str) -> int:
        n = len(s)
        Z = [0] * n
        l, r = 0, 0
        for i in range(1, n):
            if i <= r:
                Z[i] = min(r - i + 1, Z[i - l])
            while i + Z[i] < n and s[Z[i]] == s[i + Z[i]]:
                Z[i] += 1
            if i + Z[i] - 1 > r:
                l, r = i, i + Z[i] - 1
        return sum(Z) + n
```

The Python code first initializes the Z-array and two pointers `l` and `r` that track the current segment matching the prefix. For each position `i`, it uses previous Z-values to skip comparisons, then extends the match explicitly. Finally, the sum of Z-values plus `n` (score of the full string) is returned.

## Go Solution

```go
func sumScores(s string) int64 {
    n := len(s)
    Z := make([]int, n)
    var l, r int
    for i := 1; i < n; i++ {
        if i <= r {
            Z[i] = min(r-i+1, Z[i-l])
        }
        for i+Z[i] < n && s[Z[i]] == s[i+Z[i]] {
            Z[i]++
        }
        if i+Z[i]-1 > r {
            l, r = i, i+Z[i]-1
        }
    }
    total := int64(n)
    for _, val := range Z {
        total += int64(val)
    }
    return total
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

The Go implementation mirrors the Python approach, with explicit handling of integer conversions to `int64` for the sum. The Z-array computation is identical, leveraging array indexing and simple min comparison.
