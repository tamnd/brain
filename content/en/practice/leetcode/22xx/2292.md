## Solution

## Problem Understanding

The problem requires identifying products that were ordered **three or more times** in **two consecutive years**. The input is an `Orders` table with columns `order_id`, `product_id`, `quantity`, and `purchase_date`. Each row represents an individual order. The output should be a list of `product_id`s satisfying the consecutive-year condition.

In simpler terms, for each product, we need to count the number of orders per year and find at least one pair of consecutive years where the count is at least three for both years. The table could include multiple products, multiple years, and varying numbers of orders, but the guarantee is that `order_id` is unique and dates are valid.

Important edge cases include products with exactly three orders in two consecutive years, products that meet the condition in non-consecutive years, and products with many orders in a single year but not across consecutive years.

## Approaches

A **brute-force approach** would iterate through each product, compute the yearly order counts, and then check all consecutive year pairs for counts greater than or equal to three. This approach works but can be slow for a large dataset because it requires sorting and multiple nested loops.

The **optimal approach** leverages SQL aggregation and window functions. First, group orders by `product_id` and year using `YEAR(purchase_date)` and count orders per year. Then, use a **self-join** or window function to check consecutive years for each product. The key insight is that the problem reduces to checking if `(year, year + 1)` pairs for the same product have order counts of at least three, which avoids scanning unnecessary rows or using multiple nested loops.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(P * Y) | O(P * Y) | Iterate products and years; check consecutive years individually |
| Optimal | O(N) | O(P * Y) | Aggregate orders by product/year, then filter using consecutive-year logic |

## Algorithm Walkthrough

1. Extract the **year** from the `purchase_date` for each order. This allows grouping orders by year.
2. **Aggregate orders** by `product_id` and year using `COUNT(*)` to find how many times each product was ordered in a given year.
3. **Filter years** where a product has at least three orders, since only these years are relevant for consecutive-year comparison.
4. Perform a **self-join** on the filtered results, joining a product's year `y` with the same product's year `y + 1`. This ensures we are checking consecutive years.
5. Select distinct `product_id`s from the join. Any product appearing in this join satisfies the condition of being ordered three or more times in two consecutive years.

**Why it works**: The algorithm guarantees correctness because we only consider years with three or more orders and explicitly join consecutive years. Any product not appearing in the join either does not have enough orders in consecutive years or only meets the condition in a single year.

## Python Solution

```python
class Solution:
    def productsWithThreeOrdersInTwoConsecutiveYears(self, orders: list[dict]) -> list[int]:
        from collections import defaultdict
        
        # Step 1: Aggregate order counts by product and year
        counts = defaultdict(lambda: defaultdict(int))
        for order in orders:
            product_id = order['product_id']
            year = int(order['purchase_date'][:4])
            counts[product_id][year] += 1
        
        result = set()
        
        # Step 2: Check for consecutive years with >=3 orders
        for product, yearly_counts in counts.items():
            years = sorted(yearly_counts.keys())
            for i in range(len(years) - 1):
                if years[i+1] == years[i] + 1 and yearly_counts[years[i]] >= 3 and yearly_counts[years[i+1]] >= 3:
                    result.add(product)
                    break
        
        return list(result)
```

The Python implementation first builds a nested dictionary mapping each product to yearly order counts. Then, it iterates through sorted years for each product and checks if consecutive years meet the threshold. Using a `set` ensures uniqueness in the output.

## Go Solution

```go
package main

import (
    "time"
    "sort"
)

func productsWithThreeOrdersInTwoConsecutiveYears(orders []map[string]interface{}) []int {
    type yearCount map[int]int
    counts := map[int]yearCount{}
    
    // Aggregate orders per product per year
    for _, order := range orders {
        productID := order["product_id"].(int)
        dateStr := order["purchase_date"].(string)
        t, _ := time.Parse("2006-01-02", dateStr)
        year := t.Year()
        if _, exists := counts[productID]; !exists {
            counts[productID] = make(yearCount)
        }
        counts[productID][year]++
    }
    
    result := []int{}
    
    // Check consecutive years
    for product, yCounts := range counts {
        years := []int{}
        for y := range yCounts {
            years = append(years, y)
        }
        sort.Ints(years)
        for i := 0; i < len(years)-1; i++ {
            if years[i+1] == years[i]+1 && yCounts[years[i]] >= 3 && yCounts[years[i+1]] >= 3 {
                result = append(result, product)
                break
            }
        }
    }
    
    return result
}
```

In Go, the logic mirrors Python but uses maps and slices. Sorting is required for consecutive year checks. Go requires explicit type conversions for parsing dates and accessing map values.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(N + P log Y) | N is the number of orders, P is the number of products, Y is the number of years per product. Counting is O(N), sorting years is O(P log Y) |
| Space | O(P * Y) | We store counts per product per year in a nested dictionary or map |

The algorithm scales linearly with the number of orders and logarithmically with the number of years per product.
