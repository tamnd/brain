## Solution

## Problem Understanding

The problem asks us to compute the "k-beauty" of a number. We are given an integer `num` and another integer `k`. We must examine every substring of length `k` in the decimal representation of `num`.

For each substring:

1. Convert the substring into an integer.
2. Check whether that integer is a divisor of `num`.
3. Count it if:

- the integer is not zero, and
- `num % value == 0`.

The final answer is the total number of such valid substrings.

For example, if `num = 240` and `k = 2`, the string form is `"240"`. The substrings of length 2 are:

- `"24"` → 24 divides 240
- `"40"` → 40 divides 240

Both satisfy the condition, so the answer is `2`.

One important detail is that leading zeros are allowed. This means substrings such as `"04"` are valid substrings and should be interpreted numerically as `4`.

The constraints are small:

- `1 <= num <= 10^9`
- `k` is at most the number of digits in `num`

Since `num` has at most 10 digits, the total number of substrings is very small. Even a straightforward scan across all substrings is efficient enough.

There are several edge cases that can cause bugs if not handled carefully:

- Substrings equal to `"0"` or `"00"` must not be used as divisors because division by zero is invalid.
- Leading zeros must still be converted correctly, for example `"04"` becomes `4`.
- When `k` equals the total number of digits, there is only one substring to examine.
- Repeated substrings should each be counted independently if they appear multiple times.

## Approaches

### Brute Force Approach

The brute-force solution generates every substring of length `k`, converts it into an integer, and checks whether it divides `num`.

The process works as follows:

1. Convert `num` into a string.
2. Iterate through every possible starting index.
3. Extract the substring of length `k`.
4. Convert the substring into an integer.
5. Skip the substring if the integer value is `0`.
6. Otherwise, check whether `num % value == 0`.
7. Count all valid substrings.

This approach is correct because it directly follows the problem definition. Every possible substring is examined exactly once, and every divisor condition is checked explicitly.

Even though this is called "brute force", it is already fast enough because the number of digits in `num` is very small.

### Optimal Approach

The optimal approach is essentially the same scanning process, but viewed through the lens of a sliding window.

Instead of thinking of the solution as repeatedly generating independent substrings, we observe that all substrings are contiguous windows of fixed length `k`.

The key insight is that we only need a single pass over the string representation of `num`. At each position:

- we examine one window of size `k`,
- convert it to an integer,
- and test divisibility.

Because the input size is tiny, there is no need for more advanced optimizations such as rolling hashes or incremental integer construction.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × k) | O(k) | Extracts and converts every substring independently |
| Optimal | O(n × k) | O(1) excluding substring storage | Single sliding window scan across the digit string |

Here, `n` represents the number of digits in `num`.

## Algorithm Walkthrough

1. Convert `num` into its string representation so that substrings can be extracted easily.
2. Initialize a counter variable `count = 0`. This will store the number of valid divisor substrings.
3. Iterate through every valid starting position for a substring of length `k`.

The last valid starting index is:

```
len(num_string) - k
```

because the substring must remain within bounds.
4. Extract the current substring using slicing.

For example:

```
substring = num_string[i:i+k]
```
5. Convert the substring into an integer.

This automatically handles leading zeros correctly:

- `"04"` becomes `4`
- `"00"` becomes `0`
6. If the converted integer is `0`, skip it.

Zero cannot divide any number, and attempting `num % 0` would cause a runtime error.
7. Otherwise, check whether the substring value divides `num`.

If:

```
num % value == 0
```

then increment the counter.
8. After all substrings have been processed, return the counter.

### Why it works

The algorithm works because it examines every substring of length `k` exactly once. For each substring, it directly verifies the two required conditions:

1. The substring value is nonzero.
2. The substring value divides `num`.

Since every possible candidate is checked and counted correctly, the final count is exactly the k-beauty defined in the problem.

## Python Solution

```python
class Solution:
    def divisorSubstrings(self, num: int, k: int) -> int:
        num_string = str(num)
        count = 0

        for i in range(len(num_string) - k + 1):
            substring_value = int(num_string[i:i + k])

            if substring_value != 0 and num % substring_value == 0:
                count += 1

        return count
```

The implementation begins by converting the integer into a string so that fixed-length substrings can be extracted using slicing.

The loop iterates over every valid starting index. For each position, the substring of length `k` is converted into an integer.

The condition:

```python
substring_value != 0
```

protects against division by zero.

The divisibility test:

```python
num % substring_value == 0
```

checks whether the substring value is a divisor of the original number.

If both conditions are satisfied, the counter is incremented.

Finally, the method returns the total count.

## Go Solution

```go
func divisorSubstrings(num int, k int) int {
    numString := fmt.Sprintf("%d", num)
    count := 0

    for i := 0; i <= len(numString)-k; i++ {
        substring := numString[i : i+k]

        value := 0
        for _, ch := range substring {
            value = value*10 + int(ch-'0')
        }

        if value != 0 && num%value == 0 {
            count++
        }
    }

    return count
}
```

The Go version follows the same logic as the Python solution.

One implementation difference is substring conversion. Python can directly use `int(substring)`, while Go requires manual parsing or the use of `strconv.Atoi`. Here, the conversion is performed manually to avoid additional error handling.

Since the constraints are very small, integer overflow is not a concern. The largest possible substring value still fits comfortably within Go's `int` type.

## Edge Cases

One important edge case is substrings that evaluate to zero, such as `"0"` or `"00"`. A naive implementation might attempt to compute `num % 0`, which would raise a runtime error. The solution avoids this by explicitly checking whether the substring value is nonzero before performing divisibility testing.

Another important case involves leading zeros. Substrings such as `"04"` are perfectly valid according to the problem statement and must be interpreted numerically as `4`. Some implementations incorrectly reject such substrings because they begin with zero. This solution handles them correctly because integer conversion naturally transforms `"04"` into `4`.

A third edge case occurs when `k` equals the full length of the number. In this scenario, there is exactly one substring to evaluate, namely the entire number itself. Since every nonzero integer divides itself, the answer will always be `1`. The implementation handles this naturally because the loop still executes exactly once.

Repeated substrings are another subtle case. For example, in `"7777"` with `k = 2`, the substring `"77"` appears three times. Each occurrence must be counted independently if it divides the number. The algorithm processes substrings by position rather than by distinct value, so repeated valid substrings are counted correctly.
