## Solution

## Problem Understanding

The problem asks us to compute the sum of the total strengths of all contiguous groups (subarrays) of wizards, where the **total strength** of a group is defined as the product of the **minimum strength** in the group and the **sum of all strengths** in that group.

In other words, for each contiguous subarray of `strength`, we calculate:

```
total_strength(subarray) = min(subarray) * sum(subarray)
```

Then, we sum these values for all possible contiguous subarrays and return the result modulo `10^9 + 7`.

The input is a 0-indexed integer array `strength` of size up to `10^5`, and each strength value can be as large as `10^9`. The output is a single integer representing the sum modulo `10^9 + 7`.

Important aspects to note:

- Since the array length can be very large, a naive solution that enumerates all subarrays and computes `min` and `sum` for each will be too slow.
- We need to handle very large numbers, so modular arithmetic is necessary to avoid integer overflow.
- Edge cases include arrays with all equal elements, strictly increasing or decreasing sequences, and arrays of length 1.

## Approaches

### Brute Force

The brute-force solution iterates over all subarrays. For each subarray, we compute the sum of its elements and its minimum element, then multiply these to get the total strength. Finally, we sum these total strengths for all subarrays.

While this approach is correct, it is too slow for large arrays because it requires $O(n^3)$ time in the worst case: $O(n^2)$ subarrays, and each subarray requires $O(n)$ to compute `sum` and `min`.

### Optimal Approach

The key insight is that the problem involves the **minimum element in subarrays**, which suggests using a **monotonic stack** to efficiently find the range where each element is the minimum. Once we know the range for which an element is the minimum, we can compute the sum of all subarray sums involving that element efficiently using **prefix sums** and **prefix-of-prefix sums**.

The steps are:

1. Precompute prefix sums of the array, `prefix_sum[i]`, and prefix sums of the prefix sums, `prefix_of_prefix[i]`.
2. Use a monotonic increasing stack to find, for each element, the next smaller element to the left (`left[i]`) and right (`right[i]`) - the boundaries of the subarrays where the element is the minimum.
3. For each element `strength[i]`, compute its contribution as the minimum in all subarrays spanning `[left[i]+1, right[i]-1]` using prefix sums efficiently.
4. Sum all contributions modulo $10^9 + 7$.

This reduces the time complexity to $O(n)$ and space complexity to $O(n)$, which is acceptable for $n \le 10^5$.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(1) | Enumerate all subarrays, compute sum and min each time |
| Optimal | O(n) | O(n) | Use monotonic stack + prefix sum + prefix-of-prefix sum for efficient contribution calculation |

## Algorithm Walkthrough

1. Compute the prefix sum array `prefix[i]` where `prefix[i]` is the sum of the first `i` elements. Also compute the prefix of prefix sums `prefix_of_prefix[i]`.
2. Initialize a monotonic increasing stack.
3. Iterate over the array. For each index `i`, while the top of the stack has a value greater than `strength[i]`, pop it and compute the left and right boundaries of subarrays where it is the minimum.
4. For each popped element `x` at index `idx`, calculate its total contribution using the formula derived from prefix-of-prefix sums:

```
contribution = strength[idx] * (sum of right subarray prefix sums * left span - sum of left subarray prefix sums * right span)
```

1. Push the current index `i` to the stack.
2. After processing all elements, pop any remaining elements from the stack and compute their contributions similarly.
3. Return the total sum modulo $10^9 + 7$.

**Why it works:** The algorithm ensures that for each element, we consider all contiguous subarrays where it is the minimum exactly once, using monotonic stack boundaries. The prefix-of-prefix sums allow us to compute sums over ranges in constant time, ensuring efficiency.

## Python Solution

```python
from typing import List

class Solution:
    def totalStrength(self, strength: List[int]) -> int:
        MOD = 10**9 + 7
        n = len(strength)

        # Prefix sum and prefix-of-prefix sum
        prefix = [0] * (n + 1)
        for i in range(n):
            prefix[i + 1] = (prefix[i] + strength[i]) % MOD

        prefix_of_prefix = [0] * (n + 2)
        for i in range(n + 1):
            prefix_of_prefix[i + 1] = (prefix_of_prefix[i] + prefix[i]) % MOD

        stack = []
        result = 0

        for i in range(n + 1):
            while stack and (i == n or strength[stack[-1]] > (strength[i] if i < n else 0)):
                idx = stack.pop()
                left = stack[-1] if stack else -1
                right = i

                left_count = idx - left
                right_count = right - idx

                total = (
                    (prefix_of_prefix[right + 1] - prefix_of_prefix[idx + 1]) * left_count
                    - (prefix_of_prefix[idx + 1] - prefix_of_prefix[left + 1]) * right_count
                ) % MOD

                result = (result + strength[idx] * total) % MOD

            stack.append(i)

        return result
```

This implementation first builds prefix sums and prefix-of-prefix sums for efficient range sum calculation. The monotonic stack identifies the left and right boundaries of the subarrays where each element is the minimum. Then it computes each element's contribution to the total strength using the prefix-of-prefix sums formula and sums them modulo $10^9 + 7$.

## Go Solution

```go
func totalStrength(strength []int) int {
    const MOD = 1_000_000_007
    n := len(strength)

    prefix := make([]int64, n+1)
    for i := 0; i < n; i++ {
        prefix[i+1] = (prefix[i] + int64(strength[i])) % MOD
    }

    prefixOfPrefix := make([]int64, n+2)
    for i := 0; i <= n; i++ {
        prefixOfPrefix[i+1] = (prefixOfPrefix[i] + prefix[i]) % MOD
    }

    stack := []int{}
    var result int64

    for i := 0; i <= n; i++ {
        cur := int64(0)
        if i < n {
            cur = int64(strength[i])
        }
        for len(stack) > 0 && (i == n || int64(strength[stack[len(stack)-1]]) > cur) {
            idx := stack[len(stack)-1]
            stack = stack[:len(stack)-1]
            left := -1
            if len(stack) > 0 {
                left = stack[len(stack)-1]
            }
            right := i

            leftCount := int64(idx - left)
            rightCount := int64(right - idx)

            total := ((prefixOfPrefix[right+1]-prefixOfPrefix[idx+1])*leftCount -
                (prefixOfPrefix[idx+1]-prefixOfPrefix[left+1])*rightCount) % MOD

            result = (result + int64(strength[idx])*total) % MOD
        }
        stack = append(stack, i)
    }

    if result < 0 {
        result += MOD
    }
    return int(result)
}
```

Go-specific differences include explicit handling of int64 to avoid overflow and slice manipulation for the stack. Modular arithmetic requires careful treatment of negative numbers, hence the final adjustment of `result` if it is negative.
