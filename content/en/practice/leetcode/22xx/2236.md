## Solution

## Problem Understanding

The problem asks us to determine whether the value of the root node of a binary tree is equal to the sum of the values of its left and right children. The input is a binary tree consisting **exactly of three nodes**: a root, a left child, and a right child. Each node contains an integer value that can range from -100 to 100. The expected output is a boolean value: `true` if the root value equals the sum of its children, and `false` otherwise.

Given the constraints, there are no null children, and we do not need to handle trees larger than three nodes. This simplifies the problem significantly, as we only need to access the root, its left child, and its right child. Edge cases to consider include negative numbers, zero values, and the root value being larger, smaller, or equal to the sum of its children.

## Approaches

The brute-force approach is straightforward because the tree size is minimal. One could traverse the tree, sum the left and right child values, and compare with the root value. Although a tree traversal is generally more involved for larger trees, here it is trivial due to the guaranteed structure. This approach works correctly and efficiently, but it is essentially the only reasonable method given the input constraints.

The optimal approach is identical conceptually but is direct: access the left and right children of the root, sum their values, and compare with the root value. There is no need for recursion, stacks, or complex data structures. The key insight is recognizing that the problem is constrained to exactly three nodes, which allows for a simple, direct comparison.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(1) | O(1) | Traverse nodes (trivial due to 3-node tree) and sum children |
| Optimal | O(1) | O(1) | Directly sum left and right child values and compare to root |

## Algorithm Walkthrough

1. Access the root node's value. This is the value we will compare against.
2. Access the left child of the root and retrieve its value.
3. Access the right child of the root and retrieve its value.
4. Compute the sum of the left and right child values.
5. Compare the root value to the computed sum.
6. Return `true` if the root value equals the sum, otherwise return `false`.

Why it works: Since the tree contains exactly three nodes, these steps cover all possible values. The comparison directly answers the question, and no further traversal or computation is needed.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def checkTree(self, root: Optional[TreeNode]) -> bool:
        left_val = root.left.val
        right_val = root.right.val
        return root.val == left_val + right_val
```

In this implementation, we directly access `root.left.val` and `root.right.val` to get the child values. We then sum these values and compare the result to `root.val`. If they are equal, the function returns `True`; otherwise, it returns `False`. This implementation follows the algorithm exactly and leverages the constraint that the tree has exactly three nodes.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func checkTree(root *TreeNode) bool {
    leftVal := root.Left.Val
    rightVal := root.Right.Val
    return root.Val == leftVal + rightVal
}
```

The Go implementation mirrors the Python version. We access `root.Left.Val` and `root.Right.Val`, sum them, and compare with `root.Val`. Go handles nil pointers explicitly, but since the problem guarantees three nodes, we do not need to check for nil children.

## Edge Cases

One edge case is when all nodes have value `0`. A naive implementation might overlook that zero values are valid, but our sum comparison handles it correctly. Another edge case is when the nodes contain negative numbers. The sum can still equal the root, as in `[-5, -2, -3]`, which our solution correctly handles. Finally, cases where the sum differs by one from the root, either positive or negative, test that the comparison is strict (`==`) rather than `>=` or `<=`. The implementation handles these naturally by performing an exact equality check.
