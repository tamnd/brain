## Solution

## Problem Understanding

This problem requires designing a ticket booking system for a concert hall with `n` rows and `m` seats per row. The system must support two operations: `gather` and `scatter`. In `gather`, a group of spectators wants to sit **together** in a single row, while in `scatter`, they only require enough seats in total, possibly across multiple rows. Both operations must respect a `maxRow` constraint: the group cannot be seated beyond this row. When multiple options exist, the system must always choose the **earliest row and lowest seat index**.

The input represents the concert hall configuration and a series of booking requests. The expected output is either the coordinates of allocated seats (`gather`) or a boolean indicating success (`scatter`). The constraints are tight: `n` can go up to 50,000 and `k` (group size) and `m` (row length) can be up to 1e9. This rules out naive row-by-row simulation and demands a data structure that can quickly query seat availability and update bookings.

Important edge cases include when `k` is larger than any row (`gather` impossible), when a scatter request exceeds total available seats, when `maxRow` is 0, and when multiple `gather` and `scatter` operations interact on the same rows.

## Approaches

The brute-force approach would be to maintain an array for each row, simulating all seats. For `gather`, iterate through rows up to `maxRow` to find `k` consecutive empty seats. For `scatter`, iterate through rows to accumulate at least `k` empty seats. This approach is correct but inefficient: it could require iterating over up to `n * m` seats per query, which is infeasible for the problem's scale.

The optimal solution leverages a **Segment Tree** to maintain **row-wise maximum available consecutive seats** for `gather` and **total available seats per row** for `scatter`. This allows `O(log n)` queries to find the first eligible row and to update seat availability efficiently. The key insight is that `gather` queries need the largest consecutive block per row, while `scatter` only cares about the total seats, so segment trees can store both types of information.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m) per query | O(n * m) | Simulates each seat individually, too slow for large inputs |
| Optimal | O(log n) per query | O(n) | Uses a segment tree to maintain row-wise maximum and total available seats |

## Algorithm Walkthrough

1. Initialize a segment tree with `n` nodes, each node storing `maxContiguousSeats` (for `gather`) and `totalSeats` (for `scatter`). Initially, every row has `m` seats available.
2. For `gather(k, maxRow)`, perform a **binary search on the segment tree** to find the earliest row ≤ `maxRow` that has at least `k` consecutive seats. If found, allocate the first `k` seats in that row, update both `maxContiguousSeats` and `totalSeats` in the segment tree, and return `[rowIndex, seatStart]`. If no row has sufficient seats, return `[]`.
3. For `scatter(k, maxRow)`, first query the segment tree for the **sum of total available seats** in rows `0..maxRow`. If the sum is less than `k`, return `false`. Otherwise, iterate from row `0` to `maxRow`, allocating seats greedily from the left until `k` seats are booked. Update the segment tree nodes accordingly after each allocation. Return `true`.
4. The segment tree updates propagate changes to maintain correct `maxContiguousSeats` and `totalSeats` values for parent nodes, ensuring subsequent queries work efficiently.

Why it works: The segment tree guarantees that queries for maximum contiguous seats and total seats in any range are performed in logarithmic time. Propagating updates ensures data integrity, so the system always respects row order and seat order while efficiently managing large `n` and `m`.

## Python Solution

```python
from typing import List

class SegmentTreeNode:
    def __init__(self, l, r):
        self.l = l
        self.r = r
        self.left = None
        self.right = None
        self.max_contiguous = 0
        self.total = 0

class BookMyShow:
    def __init__(self, n: int, m: int):
        self.n = n
        self.m = m
        self.available = [m] * n
        self.seg_tree = self.build(0, n - 1)

    def build(self, l, r):
        node = SegmentTreeNode(l, r)
        if l == r:
            node.max_contiguous = self.m
            node.total = self.m
        else:
            mid = (l + r) // 2
            node.left = self.build(l, mid)
            node.right = self.build(mid + 1, r)
            node.max_contiguous = max(node.left.max_contiguous, node.right.max_contiguous)
            node.total = node.left.total + node.right.total
        return node

    def gather(self, k: int, maxRow: int) -> List[int]:
        row = self._gather_query(self.seg_tree, k, maxRow)
        if row == -1:
            return []
        seat = self.m - self.available[row]
        self.available[row] -= k
        self._update(self.seg_tree, row)
        return [row, seat]

    def scatter(self, k: int, maxRow: int) -> bool:
        total = self._scatter_query(self.seg_tree, 0, maxRow)
        if total < k:
            return False
        row = 0
        while k > 0:
            take = min(self.available[row], k)
            self.available[row] -= take
            k -= take
            self._update(self.seg_tree, row)
            row += 1
        return True

    def _gather_query(self, node, k, maxRow):
        if node.l > maxRow or node.max_contiguous < k:
            return -1
        if node.l == node.r:
            return node.l
        left_res = self._gather_query(node.left, k, maxRow)
        if left_res != -1:
            return left_res
        return self._gather_query(node.right, k, maxRow)

    def _scatter_query(self, node, l, r):
        if node.r < l or node.l > r:
            return 0
        if l <= node.l and node.r <= r:
            return node.total
        return self._scatter_query(node.left, l, r) + self._scatter_query(node.right, l, r)

    def _update(self, node, idx):
        if node.l == node.r == idx:
            node.max_contiguous = self.available[idx]
            node.total = self.available[idx]
            return
        if idx <= node.left.r:
            self._update(node.left, idx)
        else:
            self._update(node.right, idx)
        node.max_contiguous = max(node.left.max_contiguous, node.right.max_contiguous)
        node.total = node.left.total + node.right.total
```

The Python solution maintains both maximum contiguous seats and total seats in a segment tree. Gather queries search for rows with enough contiguous seats, while scatter queries verify total seat availability and allocate seats greedily. Updates propagate to maintain accurate information across all parent nodes.
