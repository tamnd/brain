## Problem Restatement

We are given an array of positive integers `arr`.

We need to return the lexicographically largest permutation smaller than `arr` that can be obtained with at most one swap.

If not possible, return `arr`.

The official constraints state that `1 <= arr.length <= 10^4` and `1 <= arr[i] <= 10^4`.

## Input and Output

| Item | Meaning |
|---|---|
| Input | Array `arr` |
| Output | Previous permutation with at most one swap |

Function shape:

```python
def prevPermOpt1(arr: list[int]) -> list[int]:
    ...
```

## Examples

Example 1:

```python
arr = [3, 2, 1]
```

Previous: `[3, 1, 2]`. One swap of index 1 and 2.

Answer:

```python
[3, 1, 2]
```

Example 2:

```python
arr = [1, 1, 5]
```

Previous: `[1, 1, 5]` — already smallest, return `arr`. Wait: We can swap to get `[1, 5, 1]` or `[5, 1, 1]` — those are larger. Previous permutation of `[1, 1, 5]` is... sort descending would be `[5, 1, 1]`. Previous of `[1, 1, 5]` is `[1, 1, 5]`? No, `[1, 1, 5]` — any swap that makes it smaller: swap 5 with 1 at index 1 → `[1, 5, 1]` is larger... Swap 5 with 1 at index 0 → `[5, 1, 1]` is larger. So we can't make it smaller → return `arr`.

Wait that can't be right. Let me reconsider. `arr = [1, 1, 5]`. Sorted ascending is `[1, 1, 5]`. So `arr` is already in ascending order. There's no previous permutation except `arr` itself... actually the "previous" permutation means lexicographically smaller. `[1, 1, 5]` in ascending order is already the smallest permutation. So return `arr`.

Answer for `[1, 1, 5]`:

```python
[1, 1, 5]
```

## Key Insight

To get the previous permutation with one swap:
1. Find the rightmost position `i` where `arr[i] > arr[i+1]` (a descent). If none, already the smallest.
2. Find the largest `arr[j]` to the right of `i` that is less than `arr[i]`. Among equal values, pick the leftmost (to avoid duplicate swaps).
3. Swap `arr[i]` with `arr[j]`.

## Correctness

We want the largest permutation smaller than `arr`. The rightmost descent is the best place to make a swap (affects only the suffix). Swapping with the largest value smaller than `arr[i]` to its right gives the lexicographically largest decrease.

## Complexity

| Metric | Value | Why |
|---|---:|---|
| Time | `O(n)` | Two linear scans |
| Space | `O(1)` | In-place |

## Implementation

```python
class Solution:
    def prevPermOpt1(self, arr: list[int]) -> list[int]:
        n = len(arr)
        i = n - 2
        while i >= 0 and arr[i] <= arr[i + 1]:
            i -= 1

        if i < 0:
            return arr

        j = n - 1
        while arr[j] >= arr[i]:
            j -= 1

        while j > 0 and arr[j] == arr[j - 1]:
            j -= 1

        arr[i], arr[j] = arr[j], arr[i]
        return arr
```
