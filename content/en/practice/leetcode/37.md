# LeetCode 37 - Sudoku Solver

## Problem Understanding

This problem asks us to solve a partially completed Sudoku board by filling all empty cells with digits from `1` to `9`. The board is represented as a `9 x 9` matrix where each cell contains either a digit character (`'1'` through `'9'`) or the character `'.'`, which represents an empty space.

A valid Sudoku solution must satisfy three conditions simultaneously:

1. Every row must contain each digit from `1` to `9` exactly once.
2. Every column must contain each digit from `1` to `9` exactly once.
3. Every `3 x 3` sub-box must contain each digit from `1` to `9` exactly once.

The key detail is that the solution must modify the board in-place. We do not return a new board. Instead, we directly update the given matrix.

The constraints are fixed and relatively small:

- The board is always `9 x 9`
- Each position is either a digit or `'.'`
- The puzzle always has exactly one valid solution

Although the board size is constant, the search space is still extremely large. A naive approach that blindly tries all possibilities would be computationally infeasible. This naturally suggests using backtracking with pruning.

Several edge cases are important to consider:

- The board may already be nearly solved, so the algorithm should efficiently detect valid placements.
- Some cells may have very few valid candidates, making pruning especially valuable.
- Incorrect placement decisions can create contradictions much later in the search process, so proper backtracking is critical.
- Since the problem guarantees exactly one solution, we can stop immediately once a valid solution is found.

## Approaches

### Brute Force Approach

The brute force idea is to fill every empty cell with every possible digit from `1` to `9`, then check whether the completed board is valid.

This approach works conceptually because it eventually explores all possible configurations of the board. If a valid solution exists, it will eventually be found.

However, this strategy is far too slow. Suppose there are `k` empty cells. Each cell has up to 9 possibilities, so the total search space becomes approximately:

$$9^k$$

For a board with many empty cells, this number becomes enormous.

Additionally, validating the entire board after every assignment introduces further overhead.

### Optimal Approach, Backtracking with Constraint Tracking

The key insight is that we do not need to generate complete boards before checking validity.

Instead, we can build the solution incrementally and immediately reject invalid states. This is the essence of backtracking.

At every empty cell:

- Try placing digits `1` through `9`
- Check whether the placement violates Sudoku rules
- If valid, continue recursively
- If a contradiction is reached later, undo the placement and try another digit

To make validation efficient, we maintain three data structures:

- A set for digits already used in each row
- A set for digits already used in each column
- A set for digits already used in each `3 x 3` box

This allows validity checks in constant time instead of scanning rows and columns repeatedly.

The algorithm aggressively prunes invalid branches, making it dramatically faster than brute force.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(9^k × 81) | O(k) | Tries all possible board configurations and validates repeatedly |
| Optimal | O(9^k) worst case | O(k) | Uses backtracking with efficient constraint checking |

Here, `k` represents the number of empty cells.

Although the worst case is still exponential, pruning reduces the practical runtime dramatically.

## Algorithm Walkthrough

1. Initialize tracking structures for rows, columns, and boxes.

We create:

- `rows[9]`, where each entry stores digits already used in that row
- `cols[9]`, where each entry stores digits already used in that column
- `boxes[9]`, where each entry stores digits already used in that `3 x 3` box

These structures allow constant-time validity checks.
2. Scan the board once.

For every non-empty cell:

- Add the digit to its row set
- Add the digit to its column set
- Add the digit to its box set

At the same time, collect coordinates of all empty cells.
3. Start recursive backtracking.

The recursive function processes one empty cell at a time.
4. Select the current empty cell.

Suppose the current cell is `(row, col)`.

We compute the box index using:

$\text{boxIndex} = \left(\lfloor row/3 \rfloor \times 3\right) + \lfloor col/3 \rfloor$

This maps every position to one of the 9 sub-boxes.
5. Try digits `1` through `9`.

For each candidate digit:

- Check whether it already exists in:

- the current row
- the current column
- the current box

If the digit exists anywhere, skip it.
6. Place a valid digit.

If a digit is valid:

- Write it into the board
- Add it to the row set
- Add it to the column set
- Add it to the box set
7. Recurse to the next empty cell.

If the recursive call succeeds, the puzzle is solved.
8. Backtrack if necessary.

If recursion fails:

- Remove the digit from the board
- Remove it from the row set
- Remove it from the column set
- Remove it from the box set

This restores the previous state so another digit can be tried.
9. Finish when all empty cells are filled.

Once the recursion index reaches the number of empty cells, the board is completely solved.

## Python Solution

```
from typing import List

class Solution:
    def solveSudoku(self, board: List[List[str]]) -> None:
        """
        Do not return anything, modify board in-place instead.
        """

        rows = [set() for _ in range(9)]
        cols = [set() for _ in range(9)]
        boxes = [set() for _ in range(9)]

        empty_cells = []

        # Initialize tracking structures
        for row in range(9):
            for col in range(9):
                value = board[row][col]

                if value == ".":
                    empty_cells.append((row, col))
                else:
                    rows[row].add(value)
                    cols[col].add(value)

                    box_index = (row // 3) * 3 + (col // 3)
                    boxes[box_index].add(value)

        def backtrack(index: int) -> bool:
            # All cells filled successfully
            if index == len(empty_cells):
                return True

            row, col = empty_cells[index]
            box_index = (row // 3) * 3 + (col // 3)

            for digit in "123456789":
                if (
                    digit not in rows[row]
                    and digit not in cols[col]
                    and digit not in boxes[box_index]
                ):
                    # Place digit
                    board[row][col] = digit

                    rows[row].add(digit)
                    cols[col].add(digit)
                    boxes[box_index].add(digit)

                    # Continue recursion
                    if backtrack(index + 1):
                        return True

                    # Undo placement
                    board[row][col] = "."

                    rows[row].remove(digit)
                    cols[col].remove(digit)
                    boxes[box_index].remove(digit)

            return False

        backtrack(0)
```

The implementation begins by creating three arrays of sets to track digits used in rows, columns, and boxes. These sets allow constant-time membership checks, which is crucial for efficient pruning.

Next, the board is scanned once. Every existing digit is inserted into the corresponding tracking sets, while empty positions are stored in the `empty_cells` list.

The recursive `backtrack()` function solves the puzzle one empty cell at a time. The parameter `index` indicates which empty cell is currently being processed.

For each empty position, the algorithm attempts digits `1` through `9`. Before placing a digit, it checks whether the digit already exists in the corresponding row, column, or box.

If the digit is valid, the algorithm updates both the board and the tracking structures, then recursively processes the next empty cell.

If the recursive call eventually fails, the algorithm removes the digit and restores the previous state. This undo step is the core idea behind backtracking.

Because the problem guarantees exactly one solution, the function can immediately return `True` once a valid configuration is found.

## Go Solution

```
func solveSudoku(board [][]byte) {
	rows := make([]map[byte]bool, 9)
	cols := make([]map[byte]bool, 9)
	boxes := make([]map[byte]bool, 9)

	for i := 0; i < 9; i++ {
		rows[i] = make(map[byte]bool)
		cols[i] = make(map[byte]bool)
		boxes[i] = make(map[byte]bool)
	}

	type Cell struct {
		row int
		col int
	}

	var emptyCells []Cell

	// Initialize tracking structures
	for row := 0; row < 9; row++ {
		for col := 0; col < 9; col++ {
			value := board[row][col]

			if value == '.' {
				emptyCells = append(emptyCells, Cell{row, col})
			} else {
				rows[row][value] = true
				cols[col][value] = true

				boxIndex := (row/3)*3 + (col / 3)
				boxes[boxIndex][value] = true
			}
		}
	}

	var backtrack func(int) bool

	backtrack = func(index int) bool {
		// All cells filled successfully
		if index == len(emptyCells) {
			return true
		}

		row := emptyCells[index].row
		col := emptyCells[index].col

		boxIndex := (row/3)*3 + (col / 3)

		for digit := byte('1'); digit <= byte('9'); digit++ {
			if !rows[row][digit] &&
				!cols[col][digit] &&
				!boxes[boxIndex][digit] {

				// Place digit
				board[row][col] = digit

				rows[row][digit] = true
				cols[col][digit] = true
				boxes[boxIndex][digit] = true

				// Continue recursion
				if backtrack(index + 1) {
					return true
				}

				// Undo placement
				board[row][col] = '.'

				delete(rows[row], digit)
				delete(cols[col], digit)
				delete(boxes[boxIndex], digit)
			}
		}

		return false
	}

	backtrack(0)
}
```

The Go solution follows the same algorithmic structure as the Python version, but uses `map[byte]bool` instead of Python sets.

The board is represented as `[][]byte`, so digits are handled as byte characters such as `'1'` and `'9'`.

Go does not support nested functions with implicit recursion references in the same way Python does, so the recursive function is declared using a variable first:

```
var backtrack func(int) bool
```

Then the actual function body is assigned afterward.

State restoration during backtracking uses the built-in `delete()` function to remove digits from maps.

The worst-case runtime is exponential because Sudoku solving is fundamentally a constraint satisfaction problem. However, pruning invalid states early dramatically reduces the actual search space in practice.

The space complexity comes mainly from recursion depth and the bookkeeping sets used for rows, columns, and boxes.

## Edge Cases

### Nearly Complete Board

A board may already contain most digits with only one or two empty cells remaining. A naive solution might still perform unnecessary work.

This implementation handles such cases efficiently because the recursion depth remains very small, and validity checks are constant time.

### Deep Backtracking Chains

Some Sudoku puzzles require many speculative placements before discovering contradictions. Incorrect state restoration can easily corrupt the board.

This implementation carefully undoes every modification during backtracking:

- The board cell is reset to `'.'`
- The digit is removed from row tracking
- The digit is removed from column tracking
- The digit is removed from box tracking

This guarantees the algorithm always returns to a clean previous state.

### Digits Valid in Row but Invalid in Box

A common bug is checking only rows and columns while forgetting sub-box constraints.

This implementation explicitly computes the sub-box index for every placement:

$\text{boxIndex} = \left(\lfloor row/3 \rfloor \times 3\right) + \lfloor col/3 \rfloor$

Every candidate digit must pass all three checks simultaneously before placement.

### Board with Many Empty Cells

Some boards begin extremely sparse, leading to a very large search space.

The implementation remains efficient because invalid branches are pruned immediately instead of after constructing full candidate boards. Constant-time validity checking is especially important in these scenarios.
