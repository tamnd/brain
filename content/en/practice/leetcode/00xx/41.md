## Solution

## Problem Understanding

The problem asks us to find the smallest positive integer that does not appear in an unsorted integer array.

The important detail is that we only care about positive integers starting from `1`. Negative numbers, zero, and very large positive numbers only matter if they interfere with the placement of valid values.

For example:

- If the array contains `[1,2,3]`, then the answer is `4`
- If the array contains `[2,3,4]`, then the answer is `1`
- If the array contains `[1,3,4]`, then the answer is `2`

The challenge is not just correctness, but also efficiency. The problem explicitly requires:

- `O(n)` time complexity
- `O(1)` auxiliary space

This immediately rules out several straightforward solutions.

The input size can be as large as `10^5`, so quadratic solutions are too slow. The values themselves can range from very negative to very large positive integers, which means we cannot create a huge boolean array indexed by value.

A key observation is that for an array of length `n`, the smallest missing positive integer must lie in the range `[1, n + 1]`.

Why?

- If the array contains every number from `1` through `n`, then the answer is `n + 1`
- Otherwise, one of the numbers in that range is missing

For example:

- `n = 5`
- If the array contains `[1,2,3,4,5]`, answer is `6`
- Otherwise, answer is somewhere between `1` and `5`

This observation allows us to ignore:

- Negative numbers
- Zero
- Numbers larger than `n`

Important edge cases include:

- Arrays containing only negative numbers
- Arrays containing duplicates
- Arrays missing `1`
- Arrays already containing all values from `1` to `n`
- Arrays with large irrelevant numbers
- Arrays of length `1`

The problem guarantees that the array is non-empty.

## Approaches

### Brute Force Approach

A simple approach is to repeatedly check whether `1`, `2`, `3`, and so on exist in the array.

For each candidate number:

1. Scan the entire array
2. If the number is found, continue
3. Otherwise, return it

For example:

```
nums = [3,4,-1,1]

Check 1 -> found
Check 2 -> not found
Answer = 2
```

This approach is correct because the first positive integer not present in the array is exactly what the problem asks for.

However, the time complexity is too large.

If the array length is `n`, then:

- We may check up to `n + 1` candidates
- Each check scans the array in `O(n)`

Total complexity becomes `O(n^2)`.

We can improve lookup speed using a hash set:

- Insert all numbers into a set
- Check candidates from `1` upward

This reduces time complexity to `O(n)`, but requires `O(n)` extra space, which violates the problem constraint.

### Key Insight for the Optimal Solution

The critical observation is that each valid positive integer `x` in the range `[1, n]` belongs at index `x - 1`.

For example:

| Value | Correct Index |
| --- | --- |
| 1 | 0 |
| 2 | 1 |
| 3 | 2 |
| 4 | 3 |

This means we can rearrange the array itself to act like a hash table.

If every valid number is placed into its correct position:

- Index `0` should contain `1`
- Index `1` should contain `2`
- Index `2` should contain `3`

Then the first index where this property fails tells us the missing positive number.

This works because:

- We only care about numbers in `[1, n]`
- The array itself has exactly `n` positions
- We can swap elements in place

The algorithm repeatedly swaps numbers into their proper positions until every valid number is either:

- Correctly placed
- Duplicate
- Out of range

Then we scan the array to find the first mismatch.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Repeatedly scans array for each candidate number |
| Hash Set | O(n) | O(n) | Fast lookups but violates space requirement |
| Optimal | O(n) | O(1) | Uses index placement and in-place swapping |

## Algorithm Walkthrough

1. Let `n` be the length of the array.

We only care about values in the range `[1, n]`, because any missing positive must lie within `[1, n + 1]`.
2. Iterate through the array.

For each index `i`, repeatedly attempt to place `nums[i]` into its correct position.

The correct position for value `x` is index `x - 1`.
3. Before swapping, check whether the current number is valid.

A number should only be moved if:

- It is positive
- It is less than or equal to `n`
- It is not already in the correct position
4. Swap the current number with the number at its target position.

Example:

```
nums = [3,4,-1,1]

3 belongs at index 2

Swap nums[0] and nums[2]

[-1,4,3,1]
```
5. Continue swapping until the current position cannot be improved.

This handles chains naturally.

Example:

```
[2,3,1]

2 goes to index 1
[3,2,1]

3 goes to index 2
[1,2,3]
```
6. After rearrangement, scan the array from left to right.

At each index `i`:

- Expected value is `i + 1`
- If `nums[i] != i + 1`, then `i + 1` is missing
7. If all positions are correct, return `n + 1`.

Example:

```
[1,2,3]

All positions correct
Missing number = 4
```

## Python Solution

```
from typing import List

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)

        for i in range(n):
            while (
                1 <= nums[i] <= n
                and nums[i] != nums[nums[i] - 1]
            ):
                correct_index = nums[i] - 1
                nums[i], nums[correct_index] = (
                    nums[correct_index],
                    nums[i],
                )

        for i in range(n):
            if nums[i] != i + 1:
                return i + 1

        return n + 1
```

The implementation follows the exact placement strategy described earlier.

The first loop rearranges the array. For every index, we repeatedly swap the current value into its proper position.

The `while` loop is important because a single swap may bring another valid number into the current index, and that new number may also need repositioning.

This condition:

```
1 <= nums[i] <= n
```

ensures that we only process relevant numbers.

This condition:

```
nums[i] != nums[nums[i] - 1]
```

prevents infinite loops caused by duplicates.

After all valid numbers are placed correctly, the second loop searches for the first mismatch between index and value.

If every position is correct, then all numbers from `1` through `n` exist, so the answer must be `n + 1`.

## Go Solution

```
func firstMissingPositive(nums []int) int {
	n := len(nums)

	for i := 0; i < n; i++ {
		for nums[i] >= 1 &&
			nums[i] <= n &&
			nums[i] != nums[nums[i]-1] {

			correctIndex := nums[i] - 1

			nums[i], nums[correctIndex] =
				nums[correctIndex], nums[i]
		}
	}

	for i := 0; i < n; i++ {
		if nums[i] != i+1 {
			return i + 1
		}
	}

	return n + 1
}
```

The Go implementation mirrors the Python logic closely.

Slices in Go behave similarly to dynamic arrays, so we can modify elements in place exactly as needed.

Go does not require special handling for `nil` slices here because the problem guarantees at least one element.

There are no integer overflow concerns because we only compare values against array bounds and use indices within `[0, n-1]`.

The tuple-style swap syntax in Go:

```
nums[i], nums[correctIndex] =
    nums[correctIndex], nums[i]
```

makes in-place swapping concise and safe.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each number is moved at most once into its correct position |
| Space | O(1) | Only constant extra variables are used |

Although the algorithm contains nested loops, the total number of swaps across the entire execution is linear.

Each swap places at least one number into its correct position, so no element is moved endlessly. This guarantees overall `O(n)` time complexity.

The algorithm modifies the input array directly and uses only a few temporary variables, so auxiliary space remains constant.

## Edge Cases

### Array Missing `1`

Consider:

```
[2,3,4]
```

This is an important case because the smallest positive integer is immediately missing.

A naive implementation might incorrectly start searching after the minimum value in the array.

The placement algorithm handles this naturally because index `0` should contain `1`. During the verification phase, the mismatch is detected immediately, and the algorithm returns `1`.

### Arrays with Duplicates

Consider:

```
[1,1]
```

Duplicates can easily cause infinite swapping loops if not handled carefully.

Without this condition:

```
nums[i] != nums[nums[i] - 1]
```

the algorithm could repeatedly swap identical values forever.

The implementation checks whether the target position already contains the same value before swapping, preventing unnecessary operations and infinite loops.

### Arrays Containing Only Invalid Numbers

Consider:

```
[-5,-2,0]
```

None of these values belong in the range `[1, n]`.

A buggy implementation might attempt invalid indexing or unnecessary swaps.

The condition:

```
1 <= nums[i] <= n
```

ensures that only relevant positive values are processed.

Since no valid numbers exist, the verification phase immediately detects that index `0` does not contain `1`, so the answer is correctly returned as `1`.

### Array Already Containing All Numbers

Consider:

```
[1,2,3,4]
```

Every number is already in the correct position.

The rearrangement phase performs no swaps, and the verification phase finds no mismatches.

In this situation, the algorithm correctly returns `n + 1`, which is `5`.
