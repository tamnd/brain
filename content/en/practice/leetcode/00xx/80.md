## Solution

## Problem Understanding

This problem asks us to modify a **sorted array in-place** so that each distinct number appears **at most twice**. The key requirement is that the relative order of the remaining elements must stay the same, meaning we cannot reorder elements or sort again. Instead, we selectively overwrite elements in the existing array.

The input is an integer array `nums` that is already sorted in **non-decreasing order**. Because the array is sorted, duplicates naturally appear next to each other. This property is important because it allows us to detect excessive duplicates efficiently without additional data structures.

The expected output is an integer `k`, which represents the number of valid elements after removing extra duplicates. The first `k` positions of `nums` must contain the final result. Any elements after index `k - 1` are irrelevant and may contain any value.

For example, if:

```
nums = [1,1,1,2,2,3]
```

the result should become:

```
[1,1,2,2,3,_]
```

where `_` represents values that do not matter. The function should return `5`.

The constraints tell us several important things:

- The array length can be as large as `3 * 10^4`, meaning an inefficient quadratic solution may become problematic.
- The array is guaranteed to be sorted, which strongly suggests a two-pointer strategy.
- We must use **O(1)** extra memory, meaning we cannot build a separate result array.

A few edge cases are immediately worth considering:

- Arrays with fewer than two elements, since every element is automatically valid.
- Arrays where every number appears more than twice.
- Arrays with no duplicates at all.
- Arrays containing only one repeated number.
- Negative numbers, though sorting guarantees still hold.

The sorted property is the most important guarantee because it lets us reason locally about duplicates without needing hash maps or counting structures.

## Approaches

### Brute Force Approach

A straightforward way to solve this problem is to iterate through the array and count how many times each number appears. We could build a new temporary array that stores each number at most twice.

For each number, we would check whether it has already been added twice. If not, we append it to the result array. After processing all elements, we copy the result back into `nums`.

This approach works correctly because it explicitly enforces the "at most twice" rule. However, it violates the problem requirement of **O(1) extra memory**, since we allocate another array.

The time complexity is linear because we visit each element once, but the extra memory usage makes it unacceptable for this problem.

### Key Insight for the Optimal Solution

The critical observation is that the array is already sorted.

Since duplicates appear consecutively, we only need to decide whether the current number should be written into the valid portion of the array.

Because each number may appear **at most twice**, we only need to compare the current number against the number written **two positions earlier**.

Why two positions earlier?

If the current number matches the value at `write_index - 2`, then we already have two copies of that number in the valid section, so adding another would violate the rule.

If it does not match, we can safely include it.

This insight enables a clean **two-pointer approach**:

- One pointer reads through the array.
- One pointer writes valid elements back into the array.

We overwrite elements in-place while maintaining order and constant extra memory.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Uses an extra array to store valid elements |
| Optimal | O(n) | O(1) | Uses two pointers and modifies the array in-place |

## Algorithm Walkthrough

1. Handle the trivial case where the array length is less than or equal to `2`.

Since every number is allowed to appear twice, arrays of size `0`, `1`, or `2` already satisfy the requirement. We can immediately return the length.
2. Initialize a write pointer.

We start the write pointer at index `2`. The first two elements are always valid because each number may appear up to two times.
3. Iterate through the remaining elements.

Starting from index `2`, inspect each element one by one.
4. Compare the current element with the element two positions before the write pointer.

If:

```
nums[current] != nums[write_index - 2]
```

then we have not exceeded two occurrences yet, so the number is valid.
5. Write valid elements into the array.

Copy the current value into `nums[write_index]`, then increment the write pointer.
6. Skip invalid duplicates.

If the current number equals `nums[write_index - 2]`, it means two copies already exist in the valid section, so we ignore it.
7. Return the final write pointer.

After traversal, the write pointer represents the number of valid elements, which is `k`.

## Python Solution

```
from typing import List

class Solution:
    def removeDuplicates(self, nums: List[int]) -> int:
        n = len(nums)

        if n <= 2:
            return n

        write_index = 2

        for current_index in range(2, n):
            if nums[current_index] != nums[write_index - 2]:
                nums[write_index] = nums[current_index]
                write_index += 1

        return write_index
```

The implementation begins by handling arrays of length `2` or smaller. Since every element may appear up to twice, no modifications are needed.

The variable `write_index` tracks where the next valid element should be placed. We initialize it to `2` because the first two elements are always allowed.

We then iterate from index `2` onward. For each element, we compare it against the element at `write_index - 2`.

This comparison works because the valid portion of the array is always maintained at the front. If the current value differs from the value two positions behind, we know adding it would not exceed the allowed duplicate count.

When the element is valid, we copy it to the write position and advance `write_index`. Otherwise, we simply skip it.

At the end, `write_index` represents the number of valid elements.

## Go Solution

```
func removeDuplicates(nums []int) int {
	n := len(nums)

	if n <= 2 {
		return n
	}

	writeIndex := 2

	for currentIndex := 2; currentIndex < n; currentIndex++ {
		if nums[currentIndex] != nums[writeIndex-2] {
			nums[writeIndex] = nums[currentIndex]
			writeIndex++
		}
	}

	return writeIndex
}
```

The Go implementation follows the exact same logic as the Python version.

Go slices behave similarly to dynamic views into arrays, so modifying `nums` directly updates the original data structure in-place. No special handling for `nil` slices is needed because the constraints guarantee at least one element, though the implementation would still safely handle an empty slice.

Integer overflow is not a concern here because indices stay within array bounds and the maximum array size is relatively small.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element is processed exactly once |
| Space | O(1) | Only a few variables are used, no extra data structure is allocated |

The algorithm runs in linear time because we scan through the array once using a single loop. Each element is checked and possibly written exactly one time.

The space complexity is constant because we only use a few integer variables regardless of input size. All modifications happen directly inside the original array.

## Edge Cases

### Very Small Arrays

Consider:

```
[1]
```

or

```
[1, 1]
```

These arrays already satisfy the requirement because no number appears more than twice. A naive implementation might accidentally access invalid indices during comparison. Our implementation avoids this by immediately returning when `n <= 2`.

### All Elements Are the Same

Consider:

```
[5,5,5,5,5]
```

Only the first two `5`s should remain.

The comparison against `nums[write_index - 2]` ensures that once two copies are accepted, every additional `5` is skipped. The result becomes:

```
[5,5]
```

with `k = 2`.

### No Duplicates

Consider:

```
[1,2,3,4]
```

Since every element is unique, every comparison succeeds and all values are written into the valid portion. The array remains unchanged and `k = 4`.

### Mixed Duplicate Counts

Consider:

```
[1,1,2,2,2,3,3,3]
```

Some values appear exactly twice, while others appear more than twice. The algorithm correctly preserves:

```
[1,1,2,2,3,3]
```

because the comparison logic applies uniformly to every number, regardless of how many times it repeats.
