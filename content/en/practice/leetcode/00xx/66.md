## Solution

## Problem Understanding

The problem gives us a non-negative integer represented as an array of digits. Each element in the array contains a single decimal digit, and the digits are stored in left-to-right order from most significant to least significant.

For example:

- `[1,2,3]` represents the number `123`
- `[4,3,2,1]` represents the number `4321`
- `[9]` represents the number `9`

Our task is to add one to this integer and return the resulting number in the same array format.

The challenge is that we are not supposed to convert the array directly into a built-in integer type and then back again. Although the constraints are small enough that some languages could technically handle it, the intended solution is to simulate manual addition exactly the way humans perform arithmetic on paper.

The important detail is handling carries correctly. When adding one to a digit:

- If the digit is less than `9`, we can simply increment it and stop.
- If the digit is `9`, adding one turns it into `0` and produces a carry into the next digit to the left.

This carry propagation can continue through multiple digits.

For example:

- `[1,2,3]` becomes `[1,2,4]`
- `[1,2,9]` becomes `[1,3,0]`
- `[9,9,9]` becomes `[1,0,0,0]`

The constraints tell us:

- `digits.length <= 100`, so the input is very small.
- Every digit is between `0` and `9`.
- There are no leading zeroes, which guarantees the representation is valid.

The most important edge cases involve numbers ending in one or more `9`s, because they require carry propagation. The extreme case is when every digit is `9`, since adding one increases the total number of digits.

## Approaches

### Brute Force Approach

A straightforward approach is to first convert the digit array into an integer, add one, and then convert the result back into an array of digits.

For example:

1. Convert `[1,2,3]` into integer `123`
2. Add one to get `124`
3. Convert `124` back into `[1,2,4]`

This works because it directly performs the arithmetic operation requested by the problem.

However, this approach has several drawbacks:

- It relies on integer conversion.
- In many programming languages, very large integers can overflow.
- The problem is designed to test digit-by-digit arithmetic logic rather than integer parsing.

Even though the constraints here are small enough that overflow is not an issue in Python, this approach is generally considered less robust and less aligned with the intended solution.

### Optimal Approach

The key observation is that addition only affects digits from right to left.

When adding one:

- Start from the last digit.
- If the digit is less than `9`, increment it and return immediately.
- If the digit is `9`, set it to `0` and continue moving left because a carry is generated.

If we finish processing all digits and every digit became `0`, then the original number consisted entirely of `9`s. In that case, we need to add a new leading `1`.

For example:

```
999 + 1

Step 1: last 9 -> 0, carry
Step 2: middle 9 -> 0, carry
Step 3: first 9 -> 0, carry
Step 4: add leading 1

Result: 1000
```

This approach directly simulates elementary-school addition and avoids unnecessary conversions.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Converts array to integer and back |
| Optimal | O(n) | O(1) excluding output | Simulates carry propagation from right to left |

## Algorithm Walkthrough

1. Start iterating from the last index of the array because addition begins with the least significant digit.
2. Check the current digit.

- If the digit is less than `9`, increment it by one.
- Since no carry is produced, the operation is complete and we can immediately return the array.
3. If the digit is `9`, adding one would produce `10`.

- Set the current digit to `0`.
- Carry the extra `1` into the next digit to the left.
4. Continue moving left through the array while carries exist.
5. If the loop finishes, it means every digit was originally `9`.

- Example: `[9,9,9]`
- After processing: `[0,0,0]`
- We must add a new leading digit `1`.
6. Return a new array consisting of `[1]` followed by all zeros.

This algorithm is efficient because it only processes digits until the carry disappears. In many cases, the algorithm finishes after touching just one digit.

## Python Solution

```
from typing import List

class Solution:
    def plusOne(self, digits: List[int]) -> List[int]:
        # Traverse from the last digit to the first
        for index in range(len(digits) - 1, -1, -1):
            # If digit is less than 9, increment and return
            if digits[index] < 9:
                digits[index] += 1
                return digits

            # Otherwise, set current digit to 0 and continue carry
            digits[index] = 0

        # If all digits were 9, add leading 1
        return [1] + digits
```

The implementation begins by iterating backward through the array because addition starts from the least significant digit.

Inside the loop, the algorithm checks whether the current digit is smaller than `9`.

If it is, we simply increment the digit and return immediately. No further digits are affected because no carry is generated.

If the digit equals `9`, then adding one would overflow the digit position. We therefore set the digit to `0` and continue processing the next digit to the left.

If the loop completes without returning, every digit must have been `9`. At that point, the array contains only zeros, so we prepend a `1` to represent the new higher place value.

## Go Solution

```
func plusOne(digits []int) []int {
    // Traverse from the last digit to the first
    for i := len(digits) - 1; i >= 0; i-- {
        // If digit is less than 9, increment and return
        if digits[i] < 9 {
            digits[i]++
            return digits
        }

        // Otherwise, set digit to 0 and continue carry
        digits[i] = 0
    }

    // If all digits were 9, add leading 1
    result := make([]int, len(digits)+1)
    result[0] = 1

    return result
}
```

The Go implementation follows exactly the same logic as the Python solution.

One notable difference is how the new array is created when all digits are `9`. In Python, we can easily prepend using list concatenation. In Go, slices have fixed underlying storage, so we allocate a new slice with one extra position and place `1` at index `0`.

Integer overflow is not a concern because each element only stores a single digit between `0` and `9`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | In the worst case, every digit is processed once |
| Space | O(1) excluding output | Only constant extra variables are used |

The algorithm scans the array from right to left. In the worst case, such as `[9,9,9,9]`, every digit must be visited once, giving a linear time complexity of `O(n)`.

The algorithm modifies the input array in place, so it uses constant auxiliary space. The only exception is when a new array must be created for cases like `[9,9,9]`, but output space is typically not counted toward auxiliary space complexity.

## Edge Cases

### All Digits Are 9

An input like `[9,9,9]` is the most important edge case because every digit generates a carry.

A buggy implementation might incorrectly return `[0,0,0]` or fail to add the extra leading digit.

The algorithm handles this correctly by finishing the loop and then prepending `1`, producing `[1,0,0,0]`.

### Last Digit Is Not 9

An input like `[2,4,5]` is the simplest scenario.

Only the final digit changes, and no carry propagation occurs.

The implementation immediately increments the last digit and returns early, which is both correct and efficient.

### Carry Propagates Through Multiple Digits

An input like `[1,2,9,9]` requires carry propagation through more than one digit.

The algorithm processes digits from right to left:

- Last `9` becomes `0`
- Next `9` becomes `0`
- `2` becomes `3`

The final result is `[1,3,0,0]`.

This case is important because partial carry chains are easy places for off-by-one mistakes.

### Single Digit Input

An input like `[0]` or `[9]` tests the smallest valid array size.

The algorithm correctly handles both:

- `[0] -> [1]`
- `[9] -> [1,0]`

This demonstrates that the solution works even at the minimum constraint boundary.
