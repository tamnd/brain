## Solution

## Problem Understanding

This problem asks us to modify a singly linked list so that every pair of adjacent nodes is swapped. Importantly, we are **not allowed to change the values stored inside the nodes**. Instead, we must physically rearrange the links between nodes.

In other words, if the linked list is:

`1 -> 2 -> 3 -> 4`

we should swap every adjacent pair of nodes:

`2 -> 1 -> 4 -> 3`

The first two nodes swap positions, then the next two nodes swap positions, and so on.

The input is the head of a singly linked list. Each node contains an integer value and a pointer to the next node. The output should be the new head of the modified linked list after all valid adjacent swaps have been completed.

The problem guarantees that the number of nodes is between `0` and `100`, meaning the list can be empty or relatively small. Since the size is small, efficiency is not extremely critical, but we should still aim for an optimal linear-time solution.

A key detail is that **only adjacent pairs are swapped**. If there is an odd number of nodes, the final node remains in its original position because it has no partner to swap with.

Several edge cases matter immediately:

- An empty list (`[]`) should simply return `[]`.
- A single-node list (`[1]`) cannot be swapped and should remain unchanged.
- An odd-length list (`[1,2,3]`) leaves the final node untouched.
- Pointer manipulation mistakes can easily break linked list structure, accidentally creating cycles or disconnecting nodes.

Because linked lists depend entirely on pointer relationships, careful pointer updates are necessary.

## Approaches

### Brute Force Approach

A brute-force idea is to first copy all node values into an array, swap adjacent values inside the array, and then write the swapped values back into the linked list.

For example:

`[1,2,3,4]`

becomes:

`[2,1,4,3]`

after swapping adjacent array elements.

This works because arrays allow easy indexing and swapping. However, the problem explicitly forbids modifying node values. We are only allowed to change the node connections themselves.

Even though this approach would produce the correct final arrangement, it violates the problem constraints and therefore is not acceptable.

Its time complexity would still be linear because we traverse the list multiple times, and its space complexity would be linear due to the extra array.

### Optimal Approach, Iterative Pointer Manipulation

The key observation is that swapping two adjacent linked list nodes only requires adjusting a few pointers.

Suppose we have:

`A -> B -> C`

To swap `A` and `B`, we transform it into:

`B -> A -> C`

The challenge is reconnecting the swapped pair to the rest of the list. This becomes easier if we introduce a **dummy node** before the head.

A dummy node helps us avoid special handling for swapping the first pair, because after swapping, the head itself changes.

At each iteration:

1. Identify the two nodes to swap.
2. Save the node after the pair.
3. Reverse the pair connection.
4. Reconnect the swapped pair back into the list.
5. Move forward to process the next pair.

Since each node is visited once and only a constant number of pointers are updated, this gives us an efficient solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Uses an array to swap values, violates problem constraints |
| Optimal | O(n) | O(1) | Iteratively swaps node links in-place |

## Algorithm Walkthrough

1. Create a dummy node whose `next` points to the original head. This simplifies pointer handling, especially when swapping the first pair because the head may change.
2. Maintain a pointer called `previous` that represents the node immediately before the current pair being processed. Initially, this is the dummy node.
3. While there are at least two nodes available to swap, identify:

- `first_node = previous.next`
- `second_node = first_node.next`

These are the adjacent nodes we want to reverse.
4. Save the node after the pair:

`next_pair = second_node.next`

This prevents losing access to the remainder of the list.
5. Perform the swap by updating pointers:

- Point `second_node.next` to `first_node`
- Point `first_node.next` to `next_pair`
- Point `previous.next` to `second_node`

This completes the pair reversal.
6. Move `previous` forward to `first_node`.

After swapping, `first_node` becomes the second node in the pair, so it now sits immediately before the next pair.
7. Continue until fewer than two nodes remain.
8. Return `dummy.next`, which represents the new head of the swapped list.

## Python Solution

```
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class Solution:
    def swapPairs(self, head: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode(0)
        dummy.next = head

        previous = dummy

        while previous.next and previous.next.next:
            first_node = previous.next
            second_node = first_node.next
            next_pair = second_node.next

            second_node.next = first_node
            first_node.next = next_pair
            previous.next = second_node

            previous = first_node

        return dummy.next
```

### Implementation Explanation

The implementation begins by creating a dummy node. This node points to the original head and ensures we can uniformly handle swapping at the beginning of the list without special-case logic.

The `previous` pointer tracks the node immediately before the pair currently being swapped. During each iteration, we check whether two nodes exist by verifying both `previous.next` and `previous.next.next`.

Inside the loop, we identify the two nodes involved in the swap, `first_node` and `second_node`. We also store `next_pair`, which is the node after the current pair. This is important because once pointers are rearranged, we would otherwise lose access to the rest of the linked list.

The actual swap consists of three pointer updates. First, `second_node.next` points backward to `first_node`. Then `first_node.next` connects to the remaining list. Finally, `previous.next` reconnects the swapped pair to the already processed portion of the list.

After swapping, `previous` moves to `first_node`, which is now the second node in the pair. This prepares us to process the next adjacent pair.

Finally, `dummy.next` is returned because the head may have changed during the first swap.

## Go Solution

```
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func swapPairs(head *ListNode) *ListNode {
	dummy := &ListNode{Next: head}
	previous := dummy

	for previous.Next != nil && previous.Next.Next != nil {
		firstNode := previous.Next
		secondNode := firstNode.Next
		nextPair := secondNode.Next

		secondNode.Next = firstNode
		firstNode.Next = nextPair
		previous.Next = secondNode

		previous = firstNode
	}

	return dummy.Next
}
```

### Go-Specific Notes

The Go implementation follows the exact same pointer manipulation logic as the Python version.

Instead of `None`, Go uses `nil` for null pointers. Therefore, the loop condition checks:

```
previous.Next != nil && previous.Next.Next != nil
```

to ensure that at least two nodes exist for swapping.

The dummy node is created using a struct literal:

```
dummy := &ListNode{Next: head}
```

Since linked lists in Go use pointers extensively, no additional memory allocation beyond the dummy node is needed.

There are no concerns about integer overflow because node values are never modified or used in arithmetic operations.

`[2,1,4,3]`

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited once during traversal |
| Space | O(1) | Only a few pointer variables are used |

The algorithm runs in linear time because every node participates in at most one swap operation and is processed only once. No nested traversal occurs.

The space complexity is constant because we only maintain a handful of pointers, regardless of input size. The linked list is modified in-place, and no auxiliary data structures proportional to `n` are used.

## Edge Cases

### Empty List

An empty linked list contains no nodes to swap. A naive implementation might attempt to dereference a null pointer and crash. This implementation safely handles the case because the loop condition immediately fails when `previous.next` is `None` or `nil`.

### Single Node

A list with one node cannot form a pair. Some implementations incorrectly attempt to access `head.next`, causing errors. Here, the condition requiring two nodes prevents invalid access, so the node remains unchanged.

### Odd Number of Nodes

When the list has an odd number of elements, the final node has no adjacent partner and should remain in place. For example:

`[1,2,3] → [2,1,3]`

The algorithm naturally handles this because the loop stops once fewer than two nodes remain.

### Two Nodes Only

A two-node list is the smallest valid swappable case:

`[1,2] → [2,1]`

This case often exposes bugs in head pointer updates. The dummy node guarantees the head is updated correctly after the swap.

### Multiple Consecutive Swaps

For longer lists such as `[1,2,3,4,5,6]`, pointer reconnection becomes important. Incorrect implementations may disconnect portions of the list. By explicitly storing `next_pair` before swapping, this implementation always preserves access to the remaining nodes and reconnects them correctly.
