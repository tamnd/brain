## Solution

## Problem Understanding

The problem asks us to find all unique triplets in an integer array such that the sum of the three numbers is exactly zero.

More formally, we need to return every combination of three distinct indices `i`, `j`, and `k` where:

- `i != j`
- `i != k`
- `j != k`

and:

$a+b+c=0$

The input is an array of integers called `nums`. The output is a list of triplets, where each triplet contains three numbers from the array whose sum equals zero.

An important detail is that the result must not contain duplicate triplets. For example, if `[-1, 0, 1]` is already included, we cannot include another `[-1, 0, 1]` formed using different indices. The uniqueness requirement applies to values, not indices.

The constraints tell us a lot about the expected solution quality:

- The array length can be as large as 3000
- Values can range from `-100000` to `100000`

A naive cubic solution would require:

$O(n^3)$

operations, which becomes far too slow when `n = 3000`.

At `3000^3`, we would be performing billions of operations, which is not practical for LeetCode time limits. This strongly suggests we need a more efficient approach.

Several edge cases are especially important:

- Arrays containing many duplicates, such as `[0,0,0,0]`
- Arrays with no valid triplets
- Arrays where all numbers are positive or all numbers are negative
- Multiple repeated values that could generate duplicate answers
- Very small arrays near the minimum size of 3

The problem guarantees that the input length is at least 3, so we never need to handle empty arrays or arrays with fewer than three elements.

## Approaches

### Brute Force Approach

The most straightforward solution is to try every possible triplet.

We can use three nested loops:

- The first loop chooses the first number
- The second loop chooses the second number
- The third loop chooses the third number

For every combination, we check whether the sum equals zero.

If the sum is zero, we add the triplet to the result set. Since duplicate triplets are not allowed, we would also need a mechanism to avoid duplicates. One common method is:

- Sort each triplet before insertion
- Store triplets inside a set

This approach is correct because it exhaustively checks every possible combination of three elements. Since every triplet is considered, no valid answer can be missed.

However, the performance is poor. Three nested loops require cubic time complexity:

$O(n^3)$

With `n = 3000`, this becomes prohibitively expensive.

### Optimal Approach, Sorting + Two Pointers

The key insight is that after sorting the array, we can solve the remaining part of the problem using the two pointer technique.

Suppose we fix one number `nums[i]`. We now need to find two other numbers whose sum equals:

$-nums[i]$

Because the array is sorted, we can efficiently search for these two numbers using two pointers:

- One pointer starts immediately after `i`
- Another pointer starts at the end of the array

We then adjust the pointers depending on whether the current sum is too small or too large.

Sorting enables this optimization because the ordered structure lets us make directional decisions:

- If the sum is too small, move the left pointer rightward
- If the sum is too large, move the right pointer leftward

This reduces the search for each fixed number from linear pairs to a single linear scan.

The other major challenge is duplicate handling. Sorting also makes duplicates easy to detect because equal values become adjacent.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(k) | Checks every possible triplet |
| Optimal | O(n²) | O(1) excluding output | Uses sorting and two pointers |

## Algorithm Walkthrough

1. Sort the array.

Sorting is essential because the two pointer technique only works on ordered data. Sorting also makes duplicate removal straightforward.
2. Iterate through the array using index `i`.

At each step, treat `nums[i]` as the first element of the triplet.
3. Skip duplicate values for `i`.

If `nums[i]` is the same as the previous value, we skip it. Otherwise, we would generate duplicate triplets starting with the same number.
4. Initialize two pointers.

Set:

- `left = i + 1`
- `right = len(nums) - 1`

These pointers search for the remaining two values.
5. Compute the current sum.

For each pointer configuration:

$current=nums[i]+nums[left]+nums[right]$
6. If the sum equals zero, record the triplet.

Add the triplet to the result list.
7. Move both pointers inward.

After finding a valid triplet:

- Increment `left`
- Decrement `right`

This continues the search for additional valid combinations.
8. Skip duplicate values for both pointers.

After moving pointers, continue skipping equal adjacent values. This prevents duplicate triplets from being added.
9. If the sum is too small, move the left pointer.

Since the array is sorted, moving `left` rightward increases the sum.
10. If the sum is too large, move the right pointer.

Moving `right` leftward decreases the sum.
11. Continue until the pointers cross.

Once `left >= right`, all pairs for the current `i` have been checked.

## Python Solution

```
class Solution:
    def threeSum(self, nums: list[int]) -> list[list[int]]:
        nums.sort()
        result = []

        n = len(nums)

        for i in range(n - 2):
            # Skip duplicate first elements
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            left = i + 1
            right = n - 1

            while left < right:
                current_sum = nums[i] + nums[left] + nums[right]

                if current_sum == 0:
                    result.append([nums[i], nums[left], nums[right]])

                    left += 1
                    right -= 1

                    # Skip duplicate left values
                    while left < right and nums[left] == nums[left - 1]:
                        left += 1

                    # Skip duplicate right values
                    while left < right and nums[right] == nums[right + 1]:
                        right -= 1

                elif current_sum < 0:
                    left += 1

                else:
                    right -= 1

        return result
```

The implementation begins by sorting the array. This transforms the problem into one where the two pointer strategy becomes possible.

The outer loop selects the first value of the triplet. Since duplicate first values would produce duplicate triplets, the code skips repeated values using:

```
if i > 0 and nums[i] == nums[i - 1]:
    continue
```

For each fixed value, two pointers search the remaining portion of the array.

The `left` pointer moves rightward when the sum is too small because larger values are needed. The `right` pointer moves leftward when the sum is too large because smaller values are needed.

When a valid triplet is found, it is appended to the result list. The pointers then move inward, and duplicate values are skipped to ensure uniqueness.

The algorithm never revisits the same pair configuration unnecessarily, which is why the solution achieves quadratic time complexity.

## Go Solution

```
package main

import "sort"

func threeSum(nums []int) [][]int {
	sort.Ints(nums)

	result := [][]int{}
	n := len(nums)

	for i := 0; i < n-2; i++ {
		// Skip duplicate first elements
		if i > 0 && nums[i] == nums[i-1] {
			continue
		}

		left := i + 1
		right := n - 1

		for left < right {
			currentSum := nums[i] + nums[left] + nums[right]

			if currentSum == 0 {
				result = append(result, []int{
					nums[i],
					nums[left],
					nums[right],
				})

				left++
				right--

				// Skip duplicate left values
				for left < right && nums[left] == nums[left-1] {
					left++
				}

				// Skip duplicate right values
				for left < right && nums[right] == nums[right+1] {
					right--
				}

			} else if currentSum < 0 {
				left++
			} else {
				right--
			}
		}
	}

	return result
}
```

The Go implementation closely mirrors the Python version.

One notable difference is sorting. Go uses:

```
sort.Ints(nums)
```

Slices in Go are dynamically sized views over arrays, so appending triplets is straightforward using `append`.

Unlike some languages, integer overflow is not a concern here because the constraints are small enough that sums remain within Go's `int` range.

The function returns an empty slice when no triplets exist, which is fully compatible with LeetCode expectations.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | Outer loop runs n times, two pointers scan linearly |
| Space | O(1) excluding output | Only pointer variables are used |

The array sorting step requires:

$O(n\log n)$

time, but the dominant cost is the nested two pointer traversal.

For each index `i`, the `left` and `right` pointers together traverse the remaining portion of the array only once. This creates an overall quadratic runtime:

$O(n^2)$

The algorithm uses constant auxiliary space because no additional data structures proportional to input size are allocated.

## Edge Cases

### Arrays With Many Duplicates

An input such as:

```
[0,0,0,0,0]
```

can easily produce duplicate triplets if duplicate handling is implemented incorrectly.

The solution handles this safely in two places:

- Duplicate starting values for `i` are skipped
- Duplicate values for `left` and `right` are skipped after finding a valid triplet

This guarantees that only one `[0,0,0]` triplet appears in the result.

### Arrays With No Valid Triplets

Consider:

```
[1,2,3,4]
```

Every possible triplet sum is positive, so no valid answer exists.

The algorithm naturally handles this because pointer adjustments eventually terminate the search without appending anything to the result list.

An empty list is returned.

### Arrays Containing All Negative Numbers

For example:

```
[-5,-4,-3,-2]
```

All sums remain negative, so no triplet can equal zero.

The two pointer logic still works correctly:

- Sums smaller than zero move the `left` pointer rightward
- The search terminates once pointers cross

No special handling is required.

### Minimal Length Arrays

The smallest valid input size is 3.

Example:

```
[-1,0,1]
```

The algorithm still works correctly because the outer loop runs exactly once, and the two pointers immediately evaluate the only possible triplet.

### Duplicate Values Producing Different Valid Triplets

Consider:

```
[-2,0,0,2,2]
```

The valid triplet is:

```
[-2,0,2]
```

Without careful duplicate skipping, this could be added multiple times.

The implementation avoids duplicates by skipping repeated pointer values after each successful match.
