# LeetCode 59, Spiral Matrix II

## Problem Understanding

This problem asks us to generate an `n x n` square matrix and fill it with numbers from `1` to `n²` in spiral order.

The spiral pattern starts at the top-left corner and initially moves to the right. Once it reaches a boundary or a previously filled cell, the direction changes clockwise. The movement order is:

1. Left to right
2. Top to bottom
3. Right to left
4. Bottom to top

This process repeats layer by layer until every cell in the matrix has been filled.

For example, when `n = 3`, we need to create this structure:

```
1 2 3
8 9 4
7 6 5
```

The numbers increase sequentially while following the spiral traversal path.

The input consists of a single integer `n`, which represents both the number of rows and the number of columns in the matrix. The output is a two-dimensional array containing integers from `1` through `n²`.

The constraints are relatively small:

```
1 <= n <= 20
```

This tells us several important things:

- The matrix size is at most `20 x 20`, which contains only 400 cells.
- Even an `O(n²)` solution is easily fast enough.
- Memory usage is not a concern because the output itself already requires `O(n²)` space.
- The main challenge is correct simulation of spiral movement, not optimization for huge input sizes.

Several edge cases are important:

- `n = 1`, the smallest possible matrix, should return `[[1]]`.
- Odd-sized matrices have a single center element that must be filled correctly.
- Direction changes must happen exactly at boundaries or already-filled cells.
- Off-by-one errors are common when updating row and column boundaries.

A correct solution must carefully manage movement and avoid overwriting existing values.

## Approaches

### Brute Force Approach

A brute-force strategy would repeatedly scan the matrix and determine the next valid spiral position dynamically at every step. One possible implementation could attempt movement in the current direction and repeatedly check whether the next position is valid. If not, it rotates directions until a valid move is found.

This approach works because it explicitly simulates the spiral traversal one cell at a time. Eventually, every number from `1` to `n²` gets placed into the matrix.

However, a naive brute-force implementation can become inefficient if it repeatedly searches for the next available position or rescans rows and columns multiple times. Some poorly designed versions may repeatedly inspect already-visited cells, increasing unnecessary work.

Although the constraints are small enough that even inefficient solutions may pass, this approach is less elegant and harder to reason about.

### Optimal Approach

The key insight is that spiral traversal naturally forms shrinking rectangular boundaries.

Instead of repeatedly searching for where to move next, we can maintain four boundaries:

- `top`
- `bottom`
- `left`
- `right`

At each stage:

1. Fill the top row from left to right.
2. Move the top boundary downward.
3. Fill the right column from top to bottom.
4. Move the right boundary leftward.
5. Fill the bottom row from right to left.
6. Move the bottom boundary upward.
7. Fill the left column from bottom to top.
8. Move the left boundary rightward.

After completing one full cycle, the spiral moves inward by one layer.

This works because every spiral layer is simply the perimeter of the current rectangle defined by the boundaries.

The approach is efficient, easy to visualize, and guarantees that every cell is visited exactly once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) to O(n³) | O(n²) | Simulates movement with repeated validity checks or rescanning |
| Optimal | O(n²) | O(n²) | Uses shrinking boundaries to fill the matrix layer by layer |

## Algorithm Walkthrough

1. Create an `n x n` matrix initialized with zeros. This matrix will store the final spiral arrangement.
2. Initialize four boundaries:

- `top = 0`
- `bottom = n - 1`
- `left = 0`
- `right = n - 1`

These boundaries represent the current unfilled rectangle.
3. Initialize a variable `current_number = 1`. This tracks the next number to place into the matrix.
4. Fill the top row from `left` to `right`.

- Place `current_number` into each cell.
- Increment `current_number` after every insertion.
- Once the row is filled, increment `top` because that row is complete.
5. Fill the right column from `top` to `bottom`.

- Continue inserting increasing numbers.
- After finishing, decrement `right` because that column is complete.
6. Before filling the bottom row, verify that `top <= bottom`.

- This prevents duplicate processing in odd-sized matrices.
- Fill the bottom row from `right` to `left`.
- Decrement `bottom`.
7. Before filling the left column, verify that `left <= right`.

- Fill the left column from `bottom` to `top`.
- Increment `left`.
8. Repeat this process while the boundaries remain valid.
9. Once all layers are processed, return the completed matrix.

This method works because each iteration fills exactly one outer layer of the spiral, then shrinks the remaining area inward.

## Python Solution

```
from typing import List

class Solution:
    def generateMatrix(self, n: int) -> List[List[int]]:
        matrix = [[0] * n for _ in range(n)]

        top = 0
        bottom = n - 1
        left = 0
        right = n - 1

        current_number = 1

        while top <= bottom and left <= right:
            # Fill top row
            for col in range(left, right + 1):
                matrix[top][col] = current_number
                current_number += 1
            top += 1

            # Fill right column
            for row in range(top, bottom + 1):
                matrix[row][right] = current_number
                current_number += 1
            right -= 1

            # Fill bottom row
            if top <= bottom:
                for col in range(right, left - 1, -1):
                    matrix[bottom][col] = current_number
                    current_number += 1
                bottom -= 1

            # Fill left column
            if left <= right:
                for row in range(bottom, top - 1, -1):
                    matrix[row][left] = current_number
                    current_number += 1
                left += 1

        return matrix
```

The implementation begins by creating a matrix filled with zeros. This provides the structure that will later be overwritten with spiral values.

The four boundary variables define the currently active rectangle. Initially, they cover the entire matrix.

The `while` loop continues as long as there is still an unfilled region remaining. Each iteration processes one spiral layer.

The first `for` loop fills the top row from left to right. After this row is complete, the `top` boundary moves downward because that row no longer belongs to the remaining unfilled region.

The second loop fills the right column from top to bottom. The `right` boundary is then moved leftward.

The next two sections handle the bottom row and left column. These sections require boundary checks because odd-sized matrices eventually collapse into a single row or single column. Without the checks, cells could be overwritten.

Finally, after all layers are processed, the completed matrix is returned.

## Go Solution

```
func generateMatrix(n int) [][]int {
    matrix := make([][]int, n)

    for i := 0; i < n; i++ {
        matrix[i] = make([]int, n)
    }

    top := 0
    bottom := n - 1
    left := 0
    right := n - 1

    currentNumber := 1

    for top <= bottom && left <= right {
        // Fill top row
        for col := left; col <= right; col++ {
            matrix[top][col] = currentNumber
            currentNumber++
        }
        top++

        // Fill right column
        for row := top; row <= bottom; row++ {
            matrix[row][right] = currentNumber
            currentNumber++
        }
        right--

        // Fill bottom row
        if top <= bottom {
            for col := right; col >= left; col-- {
                matrix[bottom][col] = currentNumber
                currentNumber++
            }
            bottom--
        }

        // Fill left column
        if left <= right {
            for row := bottom; row >= top; row-- {
                matrix[row][left] = currentNumber
                currentNumber++
            }
            left++
        }
    }

    return matrix
}
```

The Go implementation follows the same boundary-based strategy as the Python version.

The primary difference is matrix initialization. In Go, we must explicitly allocate each row using `make`.

Go slices are used instead of Python lists, but conceptually they behave similarly for this problem.

Integer overflow is not a concern because the largest value is only `400` when `n = 20`.

Unlike Python, Go does not support negative step ranges directly, so reverse traversal uses standard decrementing `for` loops.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | Every cell in the matrix is visited exactly once |
| Space | O(n²) | The output matrix itself requires `n²` storage |

The algorithm processes each matrix cell exactly one time while assigning values from `1` through `n²`. No cell is revisited or recomputed, so the total runtime is proportional to the number of cells in the matrix.

The space complexity is `O(n²)` because the matrix itself must be stored and returned. Aside from the output matrix, only a few integer variables are used.

## Edge Cases

### Smallest Possible Matrix

When `n = 1`, the matrix contains only a single cell. Some spiral implementations accidentally assume multiple rows or columns exist and may attempt invalid traversal.

This implementation handles the case naturally because the boundaries all start at `0`, and the first top-row traversal fills the only cell correctly.

### Odd-Sized Matrices

Odd-sized matrices contain a single center element. Incorrect boundary management can cause the center cell to be skipped or overwritten multiple times.

The conditions:

```
if top <= bottom
```

and

```
if left <= right
```

prevent duplicate processing and ensure the center cell is filled exactly once.

### Direction Changes at Boundaries

A common source of bugs is moving outside the matrix after completing a row or column traversal.

This implementation avoids invalid movement by shrinking the boundaries after each directional pass. Every traversal stays within the currently valid rectangle, so no out-of-bounds access occurs.
