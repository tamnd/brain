## Solution

## Problem Understanding

This problem gives us the head of a singly linked list where the values are already sorted in ascending order. Because the list is sorted, any duplicate values will always appear next to each other. Our task is to remove those duplicates so that every value appears exactly once.

The important detail is that we are not creating a new sorted structure from scratch. Instead, we modify the existing linked list by adjusting pointers so that duplicate nodes are skipped. The final linked list must still remain sorted.

For example, if the input list is:

```
1 -> 1 -> 2
```

the two `1` nodes are duplicates. We keep the first occurrence and remove the second, producing:

```
1 -> 2
```

The input consists of the `head` pointer of a singly linked list. Each node contains an integer value and a pointer to the next node. The output should also be the head of a linked list, after duplicates have been removed.

The constraints are small, at most 300 nodes, but the goal is still to produce an efficient linked list solution. Since the list is already sorted, we can solve the problem in a single traversal without needing extra memory structures like hash sets.

Several edge cases are important to recognize early:

- The list may be empty, meaning `head = null` or `None`
- The list may contain only one node
- Every node may contain the same value
- There may be no duplicates at all
- Duplicates may appear multiple times consecutively

The problem guarantees that the list is sorted, and this guarantee is the key observation that makes the optimal solution simple and efficient.

## Approaches

### Brute Force Approach

A brute force solution could first copy all unique values into another data structure such as an array or hash set, then construct a completely new linked list containing only unique values.

For example, we could iterate through the list, store values we have already seen in a hash set, and append unseen values into a new list.

This works because the hash set allows us to identify duplicates efficiently. However, this approach uses additional memory and ignores the fact that the input list is already sorted.

Although the constraints are small, rebuilding the list is unnecessary. We can instead modify the original list directly.

### Optimal Approach

The key observation is that the linked list is already sorted.

Because of this, duplicate values always appear consecutively. That means we only need to compare each node with its next node.

If the current node and next node have the same value, we remove the duplicate by skipping the next node:

```
current.next = current.next.next
```

Otherwise, we move forward normally.

This approach works in one pass through the list and uses constant extra space.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Uses extra memory such as a hash set or new list |
| Optimal | O(n) | O(1) | Traverses once and removes duplicates in place |

## Algorithm Walkthrough

1. Start with a pointer called `current` that points to the head of the linked list.
2. Continue processing while `current` and `current.next` both exist. We need `current.next` because we compare adjacent nodes.
3. Compare the values of `current.val` and `current.next.val`.
4. If the values are equal, we found a duplicate. Instead of moving forward, remove the duplicate node by changing the pointer:

```
current.next = current.next.next
```

This skips the duplicate node entirely.
5. If the values are different, move the `current` pointer forward to the next node because this value is unique relative to the previous one.
6. Continue until the end of the list is reached.
7. Return the original `head` pointer because the list has been modified in place.

The reason this works is that sorting guarantees all duplicates are adjacent. Without sorting, comparing only neighboring nodes would not be enough.

## Python Solution

```
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class Solution:
    def deleteDuplicates(self, head: Optional[ListNode]) -> Optional[ListNode]:
        current = head
        
        while current and current.next:
            if current.val == current.next.val:
                current.next = current.next.next
            else:
                current = current.next
        
        return head
```

The implementation starts by assigning `current` to the head of the list. This pointer will traverse the linked list node by node.

The `while` loop continues as long as both `current` and `current.next` exist. We need access to the next node because the algorithm compares adjacent values.

Inside the loop, there are two cases:

If `current.val == current.next.val`, we found a duplicate. The duplicate node is removed by redirecting the `next` pointer:

```
current.next = current.next.next
```

Notice that we do not move `current` forward in this case. This is important because there may be multiple duplicates in a row.

If the values are different, we move forward normally:

```
current = current.next
```

At the end, the modified linked list is returned using the original `head`.

## Go Solution

```
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func deleteDuplicates(head *ListNode) *ListNode {
    current := head

    for current != nil && current.Next != nil {
        if current.Val == current.Next.Val {
            current.Next = current.Next.Next
        } else {
            current = current.Next
        }
    }

    return head
}
```

The Go implementation follows exactly the same logic as the Python solution.

The primary difference is handling `nil` pointers explicitly. Go uses `nil` instead of Python's `None`, and field access uses capitalized struct fields such as `Val` and `Next`.

Since the node values are small integers, integer overflow is not a concern here.

## Worked Examples

### Example 1

Input:

```
1 -> 1 -> 2
```

Initial state:

| Current Node | Next Node | Action |
| --- | --- | --- |
| 1 | 1 | Duplicate found |

Since both values are equal:

```
current.next = current.next.next
```

The list becomes:

```
1 -> 2
```

Now:

| Current Node | Next Node | Action |
| --- | --- | --- |
| 1 | 2 | Move forward |

Move `current` to node `2`.

Now `current.next` is `null`, so the loop stops.

Final output:

```
1 -> 2
```

### Example 2

Input:

```
1 -> 1 -> 2 -> 3 -> 3
```

Initial list traversal:

| Current Node | Next Node | Action | Resulting List |
| --- | --- | --- | --- |
| 1 | 1 | Remove duplicate | 1 -> 2 -> 3 -> 3 |
| 1 | 2 | Move forward | 1 -> 2 -> 3 -> 3 |
| 2 | 3 | Move forward | 1 -> 2 -> 3 -> 3 |
| 3 | 3 | Remove duplicate | 1 -> 2 -> 3 |

After removing the final duplicate, the traversal ends.

Final output:

```
1 -> 2 -> 3
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is processed at most once |
| Space | O(1) | No extra data structures are used |

The algorithm performs a single traversal through the linked list. Each comparison and pointer update takes constant time, so the total runtime grows linearly with the number of nodes.

The space complexity is constant because the algorithm only uses a few pointer variables and modifies the linked list in place.

## Edge Cases

### Empty List

The input may be an empty linked list where `head` is `None` or `nil`.

This can easily cause null pointer errors if the implementation assumes at least one node exists. The solution handles this safely because the loop condition checks:

```
while current and current.next:
```

If `head` is empty, the loop never runs and the function correctly returns the empty list.

### Single Node List

A list with only one node cannot contain duplicates.

For example:

```
1
```

In this case, `current.next` is already `None`, so the loop terminates immediately. The original list is returned unchanged.

### Multiple Consecutive Duplicates

A common bug is advancing the pointer too early after removing a duplicate.

Consider:

```
1 -> 1 -> 1 -> 1
```

If we moved `current` immediately after removing one duplicate, some duplicates would remain.

The implementation avoids this by keeping `current` in place after deletion:

```
current.next = current.next.next
```

This allows repeated duplicate checks against the same node until all duplicates are removed.

### No Duplicates

The list may already contain only unique values:

```
1 -> 2 -> 3 -> 4
```

In this case, every comparison fails the duplicate check, and the pointer simply moves forward normally. The list remains unchanged.

### Duplicates at the End

Duplicates appearing at the tail of the list can sometimes cause incorrect pointer handling.

For example:

```
1 -> 2 -> 3 -> 3
```

The implementation correctly removes the last duplicate because it only rewires pointers and does not rely on future nodes existing beyond the duplicate.
