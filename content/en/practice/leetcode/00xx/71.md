## Solution

## Problem Understanding

This problem asks us to take an absolute Unix-style file path and convert it into its canonical simplified form. The input is always a valid absolute path, which means it always begins with the `'/'` character.

In Unix file systems, paths can contain special directory references. A single period `'.'` means "stay in the current directory", so it does not change the path. A double period `'..'` means "move to the parent directory", so it removes the most recent valid directory from the path if one exists. Multiple consecutive slashes such as `'//'` should be treated as a single slash.

The tricky part is that not every sequence of periods has special meaning. Only `'.'` and `'..'` are special. Any other sequence such as `'...'`, `'....'`, or `'..hidden'` should be treated as a normal directory name.

The goal is to return the simplified canonical version of the path with the following guarantees:

- It starts with exactly one slash.
- Directory names are separated by exactly one slash.
- It does not end with a trailing slash unless the path is simply `'/'`.
- It contains no `'.'` or `'..'` navigation tokens.

The input length can be as large as 3000 characters. This is small enough for linear-time processing, but large enough that repeatedly rebuilding strings inefficiently could still be undesirable. The constraints strongly suggest that we should process the path in a single pass.

Several edge cases are important:

- Paths with many consecutive slashes.
- Attempts to move above the root directory.
- Paths ending with slashes.
- Paths containing directory names like `'...'` that should not be interpreted specially.
- Empty path components produced by splitting on `'/'`.

A naive implementation often fails because it incorrectly treats every period-containing token as special or mishandles root directory navigation.

## Approaches

### Brute Force Approach

A brute-force strategy would repeatedly scan and modify the path string until no more simplifications are possible.

For example, we could repeatedly:

1. Replace multiple slashes with a single slash.
2. Remove occurrences of `'/./'`.
3. Resolve parent directory references by locating patterns like `'/dir/../'` and removing both components.
4. Continue until the string no longer changes.

This approach eventually produces the correct canonical path because every transformation reduces the path toward its simplified form.

However, this strategy is inefficient because each replacement operation may require rebuilding large portions of the string. In the worst case, we repeatedly scan the entire path many times. Since strings are immutable in many languages, repeated modifications become expensive.

A path of length `n` could require many passes, leading to quadratic behavior.

### Optimal Approach

The key observation is that path simplification naturally behaves like stack processing.

Each directory token either:

- Gets added to the current path,
- Gets ignored,
- Or removes the previous directory.

This is exactly how a stack works.

We process the path component by component:

- Ignore empty strings and `'.'`.
- If the component is `'..'`, pop from the stack if possible.
- Otherwise, push the directory name onto the stack.

At the end, the stack contains the exact sequence of directories that should appear in the simplified path.

This works efficiently because every component is processed once, and every directory can only be pushed and popped one time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly rebuilds and rescans the string |
| Optimal | O(n) | O(n) | Uses a stack to process path components once |

## Algorithm Walkthrough

1. Split the input path using `'/'` as the delimiter.

This converts the path into individual components. Consecutive slashes produce empty strings, which we can ignore later.

Example:

```
"/home//foo/"
```

becomes:

```
["", "home", "", "foo", ""]
```
2. Create an empty stack.

The stack represents the current valid directory structure.
3. Iterate through each path component.

For every token:

- If the token is empty or `'.'`, ignore it.

- Empty tokens come from consecutive slashes.
- `'.'` means stay in the current directory.
- If the token is `'..'`:

- Remove the most recent directory from the stack if one exists.
- If the stack is already empty, do nothing because we cannot move above root.
- Otherwise:

- Treat the token as a valid directory name and push it onto the stack.
4. Construct the final path.

Join all stack elements using `'/'` and prepend a leading slash.

Example:

```
stack = ["home", "user", "Pictures"]
```

becomes:

```
"/home/user/Pictures"
```
5. Return the result.

If the stack is empty, the result naturally becomes `'/'`.

The stack is the ideal data structure because directory traversal behaves like undo operations. Entering a directory pushes it onto the path, and `'..'` removes the most recently entered directory.

## Python Solution

```
class Solution:
    def simplifyPath(self, path: str) -> str:
        stack: list[str] = []

        components = path.split('/')

        for component in components:
            if component == '' or component == '.':
                continue

            if component == '..':
                if stack:
                    stack.pop()
            else:
                stack.append(component)

        return '/' + '/'.join(stack)
```

The implementation begins by splitting the path using `'/'`. This produces all path components, including empty strings generated by consecutive slashes.

A stack stores the valid directory names encountered so far.

During iteration:

- Empty strings and `'.'` are ignored because they do not affect the final location.
- `'..'` removes the most recent directory if possible.
- All other strings are treated as valid directory names and appended to the stack.

Finally, the stack contents are joined with `'/'`, and a leading slash is added to produce the canonical absolute path.

The implementation is concise because the stack naturally models directory navigation.

## Go Solution

```
package main

import "strings"

func simplifyPath(path string) string {
	stack := []string{}

	components := strings.Split(path, "/")

	for _, component := range components {
		if component == "" || component == "." {
			continue
		}

		if component == ".." {
			if len(stack) > 0 {
				stack = stack[:len(stack)-1]
			}
		} else {
			stack = append(stack, component)
		}
	}

	return "/" + strings.Join(stack, "/")
}
```

The Go implementation follows the same logic as the Python version.

Instead of Python lists, Go uses slices as the stack. Removing the top element is done using slice truncation:

```
stack = stack[:len(stack)-1]
```

Go strings are immutable, so `strings.Join` efficiently constructs the final result path.

Unlike some languages, Go does not have built-in stack utilities, but slices are sufficient for implementing stack behavior cleanly and efficiently.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character and path component is processed once |
| Space | O(n) | The stack may store all directory names |

The algorithm is linear because every component is examined exactly once. Each directory name can only be pushed and popped one time, so stack operations remain constant time overall.

The space complexity is linear because, in the worst case, every component is a valid directory name and must be stored in the stack.

## Edge Cases

One important edge case is attempting to move above the root directory. For example:

```
"/../../a"
```

A buggy implementation might incorrectly remove nonexistent directories or produce malformed paths. Our solution checks whether the stack is empty before popping, so extra `'..'` operations at root are safely ignored.

Another tricky case involves multiple consecutive slashes:

```
"/home///foo//bar/"
```

Splitting on `'/'` produces many empty strings. If these are not ignored, the final path could contain extra slashes or empty directory names. The implementation explicitly skips empty components.

A third important case is distinguishing between special directory tokens and normal names containing periods:

```
"/.../....hidden"
```

Only `'.'` and `'..'` have special meaning. Tokens like `'...'` are treated as regular directory names and pushed onto the stack normally. This prevents accidental removal of valid directories.

A final edge case is paths that simplify entirely to the root directory:

```
"/././../"
```

After processing, the stack becomes empty. Joining an empty stack produces an empty string, and prepending `'/'` correctly returns the root path `'/'`.
