## Solution

## Problem Understanding

The problem asks us to find every unique combination of four numbers in the array whose sum equals a given target value. Each quadruplet must use four different indices from the array, and the final result must not contain duplicates.

The input consists of an integer array `nums` and an integer `target`. The goal is to return a list of quadruplets such that:

```
nums[a] + nums[b] + nums[c] + nums[d] == target
```

with all four indices being distinct.

The key challenge is uniqueness. Even if the same values appear multiple times in the array, we should only include each distinct quadruplet once. For example, if the array contains many `2`s and the quadruplet `[2,2,2,2]` is valid, it should appear only once in the answer.

The constraints tell us several important things:

- The array length can be as large as 200
- Values can range from `-10^9` to `10^9`
- The target can also be very large or very small

A brute force solution that checks every possible quadruplet would require checking:

```
O(n^4)
```

combinations, which becomes too slow as `n` grows.

The problem strongly suggests using sorting and the two pointer technique. Sorting helps us efficiently avoid duplicates and enables a linear scan for the remaining two numbers after fixing the first two.

Several edge cases are important:

- Arrays with fewer than 4 elements should return an empty list
- Arrays with many duplicate values can easily produce duplicate quadruplets
- Large positive and negative integers could overflow in some languages if handled carelessly
- Multiple valid quadruplets may exist
- The array may contain all identical numbers

## Approaches

### Brute Force Approach

The most straightforward solution is to try every possible combination of four indices.

We can use four nested loops:

- The first loop chooses the first number
- The second loop chooses the second number
- The third loop chooses the third number
- The fourth loop chooses the fourth number

For each quadruplet, we compute the sum and compare it with the target. If the sum matches, we add the quadruplet to the result.

Since duplicate quadruplets are possible, we would also need a mechanism such as a set to avoid inserting repeated combinations.

This approach is correct because it exhaustively checks every possible group of four numbers. However, it is inefficient because the total number of combinations grows very quickly.

For an array of size `n`, the complexity is:

```
O(n^4)
```

With `n = 200`, this becomes far too slow.

### Optimal Approach

The key insight is that this problem is an extension of the classic 2Sum and 3Sum problems.

After sorting the array:

- We fix the first number using one loop
- We fix the second number using another loop
- Then we solve a 2Sum problem on the remaining portion using two pointers

Because the array is sorted:

- We can move pointers intelligently based on the current sum
- We can efficiently skip duplicate values
- We avoid generating repeated quadruplets

The two pointer method reduces the search for the remaining two numbers from `O(n^2)` to `O(n)`.

Overall complexity becomes:

```
O(n^3)
```

which is efficient enough for the given constraints.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^4) | O(1) or O(k) | Checks every quadruplet directly |
| Optimal | O(n^3) | O(1) excluding output | Uses sorting and two pointers |

## Algorithm Walkthrough

1. Sort the array.

Sorting is essential because it allows us to:

- Use the two pointer technique
- Detect duplicates efficiently
- Move pointers based on whether the current sum is too large or too small
2. Iterate through the array with index `i`.

This index represents the first number of the quadruplet.

Since duplicate starting values would produce duplicate quadruplets, we skip repeated values for `i`.
3. Inside that loop, iterate again with index `j`.

This index represents the second number of the quadruplet.

Just like with `i`, we skip duplicate values for `j`.
4. Initialize two pointers.

After fixing `i` and `j`:

- `left = j + 1`
- `right = n - 1`

These pointers search for the remaining two numbers.
5. Compute the current sum.

At each step:

```
nums[i] + nums[j] + nums[left] + nums[right]
```

is compared with the target.
6. If the sum matches the target:

- Add the quadruplet to the result
- Move both pointers inward
- Skip duplicate values to avoid repeated quadruplets
7. If the sum is too small:

Move `left` forward to increase the sum.
8. If the sum is too large:

Move `right` backward to decrease the sum.
9. Continue until `left >= right`.
10. Return the result list after all iterations finish.

## Python Solution

```
from typing import List

class Solution:
    def fourSum(self, nums: List[int], target: int) -> List[List[int]]:
        nums.sort()
        n = len(nums)
        result = []

        for i in range(n - 3):
            # Skip duplicate first values
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            for j in range(i + 1, n - 2):
                # Skip duplicate second values
                if j > i + 1 and nums[j] == nums[j - 1]:
                    continue

                left = j + 1
                right = n - 1

                while left < right:
                    current_sum = (
                        nums[i]
                        + nums[j]
                        + nums[left]
                        + nums[right]
                    )

                    if current_sum == target:
                        result.append([
                            nums[i],
                            nums[j],
                            nums[left],
                            nums[right]
                        ])

                        left += 1
                        right -= 1

                        # Skip duplicate third values
                        while left < right and nums[left] == nums[left - 1]:
                            left += 1

                        # Skip duplicate fourth values
                        while left < right and nums[right] == nums[right + 1]:
                            right -= 1

                    elif current_sum < target:
                        left += 1

                    else:
                        right -= 1

        return result
```

The implementation begins by sorting the array. This enables efficient duplicate handling and makes the two pointer strategy possible.

The outer loop selects the first number. Before processing, the code checks whether the current value is the same as the previous one. If it is, the iteration is skipped to avoid duplicate quadruplets.

The second loop selects the second number using the same duplicate skipping logic.

Once two numbers are fixed, the remaining search becomes a standard two pointer problem. The `left` pointer starts immediately after `j`, while the `right` pointer starts at the end of the array.

At each iteration, the sum of the four selected numbers is computed.

If the sum equals the target:

- The quadruplet is added to the result
- Both pointers move inward
- Duplicate values are skipped

If the sum is too small, the left pointer moves rightward to increase the total.

If the sum is too large, the right pointer moves leftward to decrease the total.

Eventually all valid unique quadruplets are collected and returned.

## Go Solution

```
package main

import (
	"sort"
)

func fourSum(nums []int, target int) [][]int {
	sort.Ints(nums)

	n := len(nums)
	result := [][]int{}

	for i := 0; i < n-3; i++ {
		// Skip duplicate first values
		if i > 0 && nums[i] == nums[i-1] {
			continue
		}

		for j := i + 1; j < n-2; j++ {
			// Skip duplicate second values
			if j > i+1 && nums[j] == nums[j-1] {
				continue
			}

			left := j + 1
			right := n - 1

			for left < right {
				currentSum := nums[i] + nums[j] + nums[left] + nums[right]

				if currentSum == target {
					quadruplet := []int{
						nums[i],
						nums[j],
						nums[left],
						nums[right],
					}

					result = append(result, quadruplet)

					left++
					right--

					// Skip duplicate third values
					for left < right && nums[left] == nums[left-1] {
						left++
					}

					// Skip duplicate fourth values
					for left < right && nums[right] == nums[right+1] {
						right--
					}

				} else if currentSum < target {
					left++
				} else {
					right--
				}
			}
		}
	}

	return result
}
```

The Go implementation follows the same logic as the Python version. The main difference is the use of slices instead of Python lists.

Go integers are generally safe for this problem on 64 bit systems, but in some contexts developers may choose to cast to `int64` when dealing with very large values to avoid overflow concerns.

The result is built incrementally using `append`, and sorting is handled with `sort.Ints`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n^3) | Two nested loops plus a linear two pointer scan |
| Space | O(1) excluding output | Only pointer variables are used |

The array is first sorted in `O(n log n)` time, but this is dominated by the main `O(n^3)` search process.

The outer loop runs `n` times, the inner loop also runs up to `n` times, and the two pointer scan runs in linear time for each pair of fixed indices.

No extra data structures proportional to input size are required, so auxiliary space remains constant aside from the result storage.

## Edge Cases

One important edge case occurs when the array contains fewer than four elements. In this situation, no valid quadruplet can exist. The implementation handles this naturally because the outer loops never execute when the array is too small.

Another tricky case involves many duplicate values. For example:

```
[2,2,2,2,2]
```

Without careful duplicate handling, the same quadruplet could be added multiple times. The implementation avoids this by skipping repeated values for `i`, `j`, `left`, and `right`.

Large positive and negative values are another important consideration. Since values can be as large as `10^9`, adding four numbers could exceed the range of a 32 bit integer in some languages. Python handles large integers automatically. In Go, standard `int` is typically sufficient on 64 bit systems, but developers should remain aware of overflow risks in constrained environments.

A final edge case is when no quadruplet exists. For example:

```
nums = [1,2,3,4]
target = 100
```

The algorithm still runs correctly and simply returns an empty list because no combination satisfies the target sum.
