# LeetCode 79, Word Search

## Problem Understanding

This problem gives us a two dimensional grid of characters called `board` and a target string called `word`. The goal is to determine whether the word can be formed by moving through the grid one character at a time.

A valid path must satisfy several rules:

1. Each character in the word must match the character in the current cell.
2. Movement is only allowed in four directions, up, down, left, and right.
3. A single cell cannot be reused within the same word construction path.

The output should be:

- `true` if there exists at least one valid path that forms the entire word
- `false` otherwise

The board dimensions are relatively small, at most `6 x 6`, and the word length is at most `15`. These constraints strongly suggest that an exhaustive search with pruning is feasible. Since the word length is short, recursive backtracking becomes a natural fit.

The key challenge is that there can be many possible paths through the grid. A naive implementation might accidentally reuse cells, fail to backtrack correctly, or repeatedly explore impossible paths.

Several important edge cases must be considered:

- The board may contain repeated letters, which creates many branching possibilities.
- The word may require revisiting the same character value but not the same cell.
- The word might be longer than the total number of cells in the board, which makes success impossible immediately.
- The board might contain only one cell.
- Multiple starting positions may match the first character of the word.

The problem guarantees that all board entries and word characters are English letters, so no special character handling is needed.

## Approaches

### Brute Force Approach

The brute force idea is to try every possible path starting from every cell in the board.

For each cell, we recursively explore all four directions while attempting to match the next character in the word. At every step, we branch into up to four recursive calls.

This approach is correct because it explores every valid path that could possibly construct the word. If a solution exists, brute force will eventually find it.

However, the number of possible paths grows exponentially. Without pruning and proper visited tracking, the algorithm repeatedly explores invalid paths and may even revisit the same cells infinitely.

The main inefficiency comes from exploring paths that already fail character matching requirements.

### Optimal Approach, Backtracking with DFS

The key observation is that the problem naturally forms a recursive search tree.

At each step:

- We must match one character of the word.
- We can move in only four directions.
- Once a cell is used in the current path, it cannot be reused.

This is a classic depth first search with backtracking problem.

The algorithm works as follows:

1. Iterate through every cell in the grid.
2. If the current cell matches the first character of the word, start a DFS search.
3. During DFS:

- Check whether the current character matches.
- Mark the cell as visited.
- Explore neighboring cells recursively.
- Restore the cell afterward, this is the backtracking step.

Backtracking is important because a cell should only be considered visited within the current recursive path. Once we return from recursion, the cell becomes available again for other paths.

This pruning strategy dramatically reduces unnecessary exploration.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m * n * 4^L) | O(L) | Explores all possible paths recursively |
| Optimal | O(m * n * 4^L) | O(L) | DFS with pruning and backtracking avoids invalid reuse |

Here:

- `m` = number of rows
- `n` = number of columns
- `L` = length of the word

Although the asymptotic complexity appears the same, pruning and early termination make the optimized solution much faster in practice.

## Algorithm Walkthrough

1. First, compute the dimensions of the board.

We need the row and column counts so we can safely check boundaries during DFS traversal.
2. Define a recursive DFS helper function.

The helper function should track:

- Current row
- Current column
- Current index in the word

The index tells us which character we are currently trying to match.
3. Check the base case.

If the current index equals the length of the word, it means every character has already been matched successfully, so return `True`.
4. Validate the current position.

Before continuing recursion, verify:

- The coordinates are inside the board.
- The current cell matches the target character.
- The cell has not already been used in this path.

If any condition fails, return `False`.
5. Mark the current cell as visited.

A common optimization is to temporarily replace the character with a special marker such as `"#"`.

This avoids maintaining a separate visited matrix and keeps space usage minimal.
6. Explore all four directions recursively.

Attempt to continue matching the next character by moving:

- Up
- Down
- Left
- Right

If any recursive call returns `True`, the word exists.
7. Backtrack.

Restore the original character before returning.

This step is critical because other recursive paths may need to use this cell later.
8. Try every possible starting cell.

Since the word may begin anywhere in the board, iterate through all cells and launch DFS whenever the first character matches.
9. If no path succeeds, return `False`.

## Python Solution

```
from typing import List

class Solution:
    def exist(self, board: List[List[str]], word: str) -> bool:
        rows = len(board)
        cols = len(board[0])

        def dfs(row: int, col: int, index: int) -> bool:
            if index == len(word):
                return True

            if (
                row < 0
                or row >= rows
                or col < 0
                or col >= cols
                or board[row][col] != word[index]
            ):
                return False

            temp = board[row][col]
            board[row][col] = "#"

            found = (
                dfs(row + 1, col, index + 1)
                or dfs(row - 1, col, index + 1)
                or dfs(row, col + 1, index + 1)
                or dfs(row, col - 1, index + 1)
            )

            board[row][col] = temp

            return found

        for row in range(rows):
            for col in range(cols):
                if dfs(row, col, 0):
                    return True

        return False
```

The solution begins by storing the board dimensions in `rows` and `cols`. This avoids repeatedly calling `len()` during recursion.

The `dfs()` function performs the recursive search. Its parameters identify the current cell and the character index being matched.

The first condition checks whether the entire word has already been matched. If so, the search succeeds immediately.

The next condition handles invalid states:

- Out of bounds positions
- Character mismatches
- Previously visited cells

Instead of maintaining a separate visited matrix, the implementation temporarily replaces the current character with `"#"`. This marks the cell as unavailable for the current path.

The recursive calls explore all four neighboring directions. If any direction successfully completes the word, the search returns `True`.

After exploration finishes, the original character is restored. This restoration is the backtracking step that ensures correctness for future searches.

Finally, the outer loops try every board position as a possible starting point.

## Go Solution

```
func exist(board [][]byte, word string) bool {
    rows := len(board)
    cols := len(board[0])

    var dfs func(int, int, int) bool

    dfs = func(row int, col int, index int) bool {
        if index == len(word) {
            return true
        }

        if row < 0 ||
            row >= rows ||
            col < 0 ||
            col >= cols ||
            board[row][col] != word[index] {
            return false
        }

        temp := board[row][col]
        board[row][col] = '#'

        found := dfs(row+1, col, index+1) ||
            dfs(row-1, col, index+1) ||
            dfs(row, col+1, index+1) ||
            dfs(row, col-1, index+1)

        board[row][col] = temp

        return found
    }

    for row := 0; row < rows; row++ {
        for col := 0; col < cols; col++ {
            if dfs(row, col, 0) {
                return true
            }
        }
    }

    return false
}
```

The Go implementation follows the same algorithmic structure as the Python version.

One important difference is that the board uses `[][]byte` instead of strings for individual characters. Since Go strings are immutable, using bytes makes in place modification efficient.

The recursive DFS function is declared using a function variable so it can recursively reference itself.

Like the Python solution, the implementation marks visited cells using `'#'` and restores them afterward during backtracking.

No additional memory allocation for visited tracking is required.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(m * n * 4^L) | Each cell can branch into up to four recursive directions |
| Space | O(L) | Recursive call stack depth equals word length |

The algorithm tries every cell as a starting position, giving `m * n` possibilities.

For each recursive step, there are at most four movement choices. Since the recursion depth is bounded by the word length `L`, the worst case complexity becomes `O(m * n * 4^L)`.

The space complexity comes entirely from recursion depth. The deepest possible recursive chain equals the length of the word.

## Edge Cases

### Single Cell Board

Consider:

```
board = [["A"]]
word = "A"
```

A naive implementation might incorrectly fail boundary checks or mishandle recursion termination.

This solution correctly identifies that the first character matches and immediately reaches the successful base case after one recursive step.

### Word Longer Than Total Cells

Consider:

```
board = [["A","B"],["C","D"]]
word = "ABCDE"
```

The board contains only four cells, but the word length is five.

Even though the algorithm does not explicitly check this condition upfront, DFS naturally fails because cells cannot be reused.

The search exhausts all paths safely and returns `false`.

### Repeated Characters

Consider:

```
board =
[
  ["A","A"],
  ["A","A"]
]

word = "AAAA"
```

This case is tricky because many paths reuse the same letter value.

The implementation correctly distinguishes cells by position rather than character value. The temporary visited marking prevents illegal reuse of the same cell while still allowing traversal through different cells containing the same letter.

### Multiple Starting Points

Consider:

```
board =
[
  ["C","A","A"],
  ["A","A","A"],
  ["B","C","D"]
]

word = "AAB"
```

The first matching `A` may not lead to a solution.

The outer loops ensure every cell is considered as a potential starting point. Even if one DFS branch fails, the algorithm continues exploring alternative starting positions until all possibilities are exhausted.
