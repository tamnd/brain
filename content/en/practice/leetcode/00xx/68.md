## Solution

## Problem Understanding

The problem asks us to format a sequence of words into fully justified text, where every output line has exactly `maxWidth` characters.

We are given an array of strings called `words`, where each element represents a single word with no spaces inside it. We are also given an integer `maxWidth`, which specifies the exact width every output line must have.

The task is to arrange the words greedily. This means that for each line, we should place as many words as possible before moving on to the next line. After deciding which words belong to a line, we must insert spaces between them so that the final line length becomes exactly `maxWidth`.

The spacing rules are the most important part of the problem:

- For normal lines, spaces must be distributed as evenly as possible between words.
- If the spaces cannot be divided evenly, the leftmost gaps receive more spaces than the rightmost gaps.
- The last line is handled differently. It must be left-justified, meaning:

- Words are separated by a single space.
- Remaining spaces are added only at the end.
- A line containing only one word is also left-justified, because there are no multiple gaps where spaces can be distributed.

The output is an array of strings, where each string represents one justified line.

The constraints tell us several useful things:

- `words.length <= 300`, so the input size is relatively small.
- `maxWidth <= 100`, meaning each line is short.
- Every word length is guaranteed to be at most `maxWidth`, so every word can always fit on a line by itself.
- Since the input size is modest, we do not need highly advanced optimization techniques. A clean linear simulation is sufficient.

Several edge cases can easily cause bugs in naive implementations:

- A line containing only one word.
- The final line, which follows different formatting rules.
- Cases where extra spaces are not evenly divisible.
- Words whose combined length exactly equals `maxWidth`.
- Very short widths, such as `maxWidth = 1`.

A correct solution must carefully handle all these situations.

## Approaches

### Brute Force Approach

A brute-force approach would attempt to generate every possible grouping of words into lines, then check which groupings satisfy the width constraint and formatting requirements.

For example, we could recursively decide where each line should end. Once all possible partitions are generated, we could format each candidate and validate whether every line respects the width requirements.

This approach is correct because it explores every possible valid arrangement of line breaks. Eventually, it would find the correct formatting.

However, this is extremely inefficient. If there are `n` words, the number of possible ways to partition them into lines grows exponentially. Even though the constraints are not huge, exponential exploration is unnecessary and impractical.

The key observation is that the problem explicitly asks us to use a greedy strategy. Once we know that each line should contain as many words as possible, there is no need to explore alternative partitions.

### Optimal Greedy Simulation

The optimal solution processes the words from left to right and greedily packs each line with as many words as possible.

For every line:

1. Determine the maximum number of words that fit.
2. Compute how many spaces are needed.
3. Distribute spaces evenly across the gaps.
4. Handle special cases:

- last line
- single-word line

This works because the problem definition itself guarantees that greedy packing is the desired behavior. Once a line's words are fixed, the spacing distribution becomes deterministic.

Since each word is processed only once, the algorithm runs efficiently in linear time relative to the total number of words.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(2^n) | Explores all possible line partitions |
| Optimal | O(n) | O(n) | Greedy line construction and formatting |

## Algorithm Walkthrough

1. Initialize an empty result list that will store the justified lines.
2. Start iterating through the `words` array using an index called `index`. This index marks the beginning of the current line.
3. Determine how many words can fit into the current line.

Begin with the first word, then keep adding additional words while checking whether:

```
total_characters + spaces_between_words <= maxWidth
```

The spaces between words are important because every adjacent pair of words requires at least one separating space.
4. Once the maximum valid group of words is identified, calculate:

- `num_words`, the number of words in the line
- `total_word_length`, the combined length of all words
- `total_spaces`, computed as:

```
maxWidth - total_word_length
```
5. Decide whether this line should be left-justified. A line is left-justified if:

- it is the last line, or
- it contains only one word
6. For left-justified lines:

- Join words using a single space.
- Add remaining spaces at the end until the line length becomes `maxWidth`.
7. For fully justified lines:

- Compute the number of gaps:

```
gaps = num_words - 1
```

- Divide spaces evenly:

```
spaces_per_gap = total_spaces // gaps
extra_spaces = total_spaces % gaps
```

- The first `extra_spaces` gaps receive one additional space.
8. Build the justified line carefully:

- Append a word.
- Append the calculated number of spaces after it.
- Continue until all words are processed.
9. Add the completed line to the result list.
10. Move `index` to the start of the next line and repeat until all words are processed.

## Python Solution

```
from typing import List

class Solution:
    def fullJustify(self, words: List[str], maxWidth: int) -> List[str]:
        result = []
        index = 0
        n = len(words)

        while index < n:
            line_start = index
            line_length = len(words[index])
            index += 1

            # Greedily add as many words as possible
            while (
                index < n and
                line_length + 1 + len(words[index]) <= maxWidth
            ):
                line_length += 1 + len(words[index])
                index += 1

            line_words = words[line_start:index]
            num_words = len(line_words)

            total_word_length = sum(len(word) for word in line_words)
            total_spaces = maxWidth - total_word_length

            # Last line or single-word line
            if index == n or num_words == 1:
                line = " ".join(line_words)
                remaining_spaces = maxWidth - len(line)
                line += " " * remaining_spaces
                result.append(line)
                continue

            # Fully justify
            gaps = num_words - 1
            spaces_per_gap = total_spaces // gaps
            extra_spaces = total_spaces % gaps

            line = ""

            for i in range(gaps):
                line += line_words[i]

                spaces_to_add = spaces_per_gap

                if i < extra_spaces:
                    spaces_to_add += 1

                line += " " * spaces_to_add

            line += line_words[-1]

            result.append(line)

        return result
```

The implementation follows the greedy strategy directly.

The outer `while` loop processes one line at a time. Inside it, we greedily extend the current line by adding more words while the width constraint remains valid.

After identifying the words for a line, we compute the total number of spaces that must be distributed. The implementation then separates the logic into two cases.

For the last line and single-word lines, we simply join words with single spaces and pad the end with remaining spaces.

For fully justified lines, we compute how many spaces each gap should receive. Integer division gives the minimum spaces per gap, while the remainder tells us how many leftmost gaps receive one extra space.

The final loop constructs the justified string gap by gap until the line reaches exactly `maxWidth`.

## Go Solution

```
package main

import "strings"

func fullJustify(words []string, maxWidth int) []string {
	var result []string
	n := len(words)
	index := 0

	for index < n {
		lineStart := index
		lineLength := len(words[index])
		index++

		// Greedily add words
		for index < n &&
			lineLength+1+len(words[index]) <= maxWidth {

			lineLength += 1 + len(words[index])
			index++
		}

		lineWords := words[lineStart:index]
		numWords := len(lineWords)

		totalWordLength := 0
		for _, word := range lineWords {
			totalWordLength += len(word)
		}

		totalSpaces := maxWidth - totalWordLength

		// Last line or single-word line
		if index == n || numWords == 1 {
			line := strings.Join(lineWords, " ")
			remainingSpaces := maxWidth - len(line)
			line += strings.Repeat(" ", remainingSpaces)

			result = append(result, line)
			continue
		}

		// Fully justify
		gaps := numWords - 1
		spacesPerGap := totalSpaces / gaps
		extraSpaces := totalSpaces % gaps

		var builder strings.Builder

		for i := 0; i < gaps; i++ {
			builder.WriteString(lineWords[i])

			spacesToAdd := spacesPerGap
			if i < extraSpaces {
				spacesToAdd++
			}

			builder.WriteString(strings.Repeat(" ", spacesToAdd))
		}

		builder.WriteString(lineWords[gaps])

		result = append(result, builder.String())
	}

	return result
}
```

The Go implementation mirrors the Python logic closely.

One important difference is string construction. Since Go strings are immutable, repeated concatenation can become inefficient. To avoid unnecessary allocations, the solution uses `strings.Builder` when constructing fully justified lines.

The implementation also uses `strings.Join` and `strings.Repeat` to simplify left-justified formatting and space generation.

Integer overflow is not a concern because all constraints are very small.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each word is processed once while building lines |
| Space | O(n) | Output storage plus temporary line construction |

The algorithm processes the input array sequentially. Each word is added to exactly one line, and each line is formatted once. Although string construction requires additional operations, the total amount of processed text remains proportional to the output size.

The space complexity is dominated by the result array that stores all justified lines. Temporary variables and formatting structures use only linear additional space.

## Edge Cases

### Single Word in a Line

A line may contain only one word because the next word does not fit within `maxWidth`. In this situation, there are no gaps where spaces can be distributed.

For example:

```
words = ["acknowledgment"]
maxWidth = 16
```

The implementation detects `num_words == 1` and left-justifies the line by appending all remaining spaces at the end.

### Last Line Formatting

The last line follows different rules from normal lines. Instead of distributing spaces evenly, words must be separated by a single space.

For example:

```
"shall be        "
```

A common bug is accidentally fully justifying the final line. The implementation avoids this by checking:

```
if index == n
```

which indicates that all words have been consumed.

### Uneven Space Distribution

When spaces cannot be evenly divided, the leftmost gaps must receive extra spaces.

For example:

```
totalSpaces = 5
gaps = 2
```

Then:

```
spacesPerGap = 2
extraSpaces = 1
```

This means the first gap receives 3 spaces and the second gap receives 2 spaces.

The implementation handles this using:

```
if i < extra_spaces:
    spaces_to_add += 1
```

which guarantees that extra spaces are assigned from left to right exactly as required by the problem statement.
