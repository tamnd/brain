## Solution

## Problem Understanding

The problem asks us to traverse a two dimensional matrix in **spiral order** and return the elements in the sequence they are visited.

In simpler terms, imagine standing at the top left corner of the matrix and walking around its perimeter in a clockwise direction. You first move left to right across the top row, then top to bottom along the right column, then right to left across the bottom row, and finally bottom to top along the left column. After completing one outer layer, you repeat the same process for the inner remaining submatrix until every element has been visited.

The input is an `m x n` matrix, where:

- `m` is the number of rows
- `n` is the number of columns
- each cell contains an integer

The output is a one dimensional array containing all matrix values in spiral order.

For example, given:

```
[
  [1,2,3],
  [4,5,6],
  [7,8,9]
]
```

We traverse:

```
1 → 2 → 3
        ↓
4 ← 5   6
↑       ↓
7 ← 8 ← 9
```

Result:

```
[1,2,3,6,9,8,7,4,5]
```

The constraints are small:

- `1 <= m, n <= 10`
- maximum matrix size is only `10 x 10`

Even though the matrix is small, the goal is still to produce an efficient and clean solution. Since every element must appear exactly once in the output, any reasonable solution will likely need to visit all cells at least one time.

There are several important edge cases to think about:

- A matrix with only one row, such as `[[1,2,3]]`. The traversal becomes a simple left to right scan.
- A matrix with only one column, such as `[[1],[2],[3]]`. The traversal becomes top to bottom.
- Rectangular matrices where rows and columns differ, such as `3 x 4` or `4 x 2`. A careless implementation can revisit cells.
- Odd dimensions where a single center element remains, such as a `3 x 3` matrix. We must avoid duplicating the middle cell.
- Very narrow remaining boundaries during traversal, where top crosses bottom or left crosses right.

The problem guarantees that the matrix is non empty, meaning we never need to handle invalid or missing input.

## Approaches

### Brute Force Approach

A brute force strategy is to simulate movement cell by cell while maintaining a **visited matrix** and explicit movement directions.

We can maintain four movement directions:

1. Right
2. Down
3. Left
4. Up

We start at `(0,0)` and continue moving in the current direction until we either hit the matrix boundary or encounter a previously visited cell. When that happens, we rotate to the next direction.

To avoid revisiting cells, we maintain a boolean `visited` matrix of size `m x n`.

This approach works because every movement decision is validated against boundaries and visitation state. Eventually, all elements are visited exactly once.

However, the downside is additional memory usage for the visited matrix. The implementation is also more simulation heavy than necessary because we already know the traversal naturally shrinks inward layer by layer.

### Optimal Approach

The key insight is that spiral traversal naturally defines **four shrinking boundaries**:

- `top` → current top row
- `bottom` → current bottom row
- `left` → current left column
- `right` → current right column

Instead of tracking visited cells, we repeatedly process the outer layer:

1. Traverse left to right across the top row.
2. Traverse top to bottom on the right column.
3. Traverse right to left across the bottom row, if still valid.
4. Traverse bottom to top on the left column, if still valid.

After finishing one layer, we shrink the boundaries inward.

This works because each pass consumes one side of the remaining rectangle. By carefully checking whether boundaries are still valid before traversing bottom or left edges, we avoid duplicate visits.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m × n) | O(m × n) | Uses a visited matrix and directional simulation |
| Optimal | O(m × n) | O(1) excluding output | Uses shrinking boundaries, simpler and more memory efficient |

## Algorithm Walkthrough

The optimal solution uses four boundaries that define the current unexplored region.

1. **Initialize boundaries**

We begin with four pointers:

- `top = 0`
- `bottom = rows - 1`
- `left = 0`
- `right = cols - 1`

These represent the outermost rectangle of the matrix.
2. **Traverse the top row**

Move from `left` to `right` along row `top` and add values to the result.

After finishing, increment `top` because this row has already been processed.
3. **Traverse the right column**

Move from `top` to `bottom` along column `right`.

After finishing, decrement `right` because this column is complete.
4. **Check if rows remain**

Before traversing the bottom row, verify:

```
top <= bottom
```

This prevents revisiting rows in matrices with odd dimensions.
5. **Traverse the bottom row**

Move from `right` to `left` along row `bottom`.

After finishing, decrement `bottom`.
6. **Check if columns remain**

Before traversing the left column, verify:

```
left <= right
```

This prevents duplicate traversal when only one column remains.
7. **Traverse the left column**

Move from `bottom` to `top` along column `left`.

After finishing, increment `left`.
8. **Repeat**

Continue until the boundaries cross:

```
top > bottom
or
left > right
```

At every stage, we process exactly one outer layer and then shrink inward.

## Python Solution

```
from typing import List

class Solution:
    def spiralOrder(self, matrix: List[List[int]]) -> List[int]:
        rows = len(matrix)
        cols = len(matrix[0])

        top = 0
        bottom = rows - 1
        left = 0
        right = cols - 1

        result = []

        while top <= bottom and left <= right:
            # Traverse left to right
            for col in range(left, right + 1):
                result.append(matrix[top][col])
            top += 1

            # Traverse top to bottom
            for row in range(top, bottom + 1):
                result.append(matrix[row][right])
            right -= 1

            # Traverse right to left
            if top <= bottom:
                for col in range(right, left - 1, -1):
                    result.append(matrix[bottom][col])
                bottom -= 1

            # Traverse bottom to top
            if left <= right:
                for row in range(bottom, top - 1, -1):
                    result.append(matrix[row][left])
                left += 1

        return result
```

### Implementation Explanation

We first determine the matrix dimensions using `len(matrix)` and `len(matrix[0])`.

The four boundary variables, `top`, `bottom`, `left`, and `right`, define the current unvisited rectangle.

The `while` loop continues as long as there is still a valid submatrix to process.

The first loop traverses the current top row from left to right. Since the row has been fully consumed, we increment `top`.

The second loop traverses the rightmost column from top to bottom. We then decrement `right`.

Before traversing the bottom row, we check whether `top <= bottom`. This is important because the top row may already have crossed the bottom row in matrices with a single remaining row.

Similarly, before traversing the left column, we check `left <= right`. Without this check, narrow matrices could cause duplicate entries.

The result list gradually accumulates all matrix elements in spiral order until no valid region remains.

## Go Solution

```
func spiralOrder(matrix [][]int) []int {
	rows := len(matrix)
	cols := len(matrix[0])

	top := 0
	bottom := rows - 1
	left := 0
	right := cols - 1

	result := make([]int, 0, rows*cols)

	for top <= bottom && left <= right {
		// Traverse left to right
		for col := left; col <= right; col++ {
			result = append(result, matrix[top][col])
		}
		top++

		// Traverse top to bottom
		for row := top; row <= bottom; row++ {
			result = append(result, matrix[row][right])
		}
		right--

		// Traverse right to left
		if top <= bottom {
			for col := right; col >= left; col-- {
				result = append(result, matrix[bottom][col])
			}
			bottom--
		}

		// Traverse bottom to top
		if left <= right {
			for row := bottom; row >= top; row-- {
				result = append(result, matrix[row][left])
			}
			left++
		}
	}

	return result
}
```

### Go Specific Notes

The Go implementation closely mirrors the Python version.

Since the problem guarantees a non empty matrix, accessing `matrix[0]` is safe.

We initialize the result slice with a capacity of `rows * cols`:

```
result := make([]int, 0, rows*cols)
```

This reduces reallocations during `append`.

Unlike Python lists, Go slices benefit from preallocated capacity for performance, although the asymptotic complexity remains unchanged.

There are no integer overflow concerns because matrix dimensions are very small and values stay within `[-100, 100]`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(m × n) | Every matrix element is visited exactly once |
| Space | O(1) excluding output | Only boundary variables are used |

The time complexity is `O(m × n)` because every cell is appended to the result exactly one time. There are no repeated traversals of the same element.

The auxiliary space complexity is `O(1)` because we only maintain four boundary variables and loop counters. The output array itself is not counted toward extra space complexity since returning the result is required by the problem.

## Edge Cases

### Single Row Matrix

Consider:

```
[[1,2,3,4]]
```

A buggy implementation might try to traverse the bottom row again after finishing the top row, producing duplicates.

Our implementation avoids this by checking:

```
top <= bottom
```

before traversing the bottom boundary.

### Single Column Matrix

Consider:

```
[
 [1],
 [2],
 [3]
]
```

After traversing downward, there is no remaining left boundary to process. Without careful validation, values could be revisited.

The condition:

```
left <= right
```

ensures we only traverse the left column when it still exists.

### Odd Sized Matrix With Center Element

Consider:

```
[
 [1,2,3],
 [4,5,6],
 [7,8,9]
]
```

Eventually, only the center element `5` remains.

An incorrect implementation may revisit it multiple times while traversing shrinking boundaries. Our approach handles this safely because the boundary checks ensure the center is processed exactly once.

### Rectangular Matrix

Consider:

```
[
 [1,2,3,4],
 [5,6,7,8]
]
```

The number of rows and columns are different, which can expose indexing mistakes.

The boundary based approach naturally supports rectangular shapes because traversal always depends on the current valid boundaries rather than assuming a square matrix.
