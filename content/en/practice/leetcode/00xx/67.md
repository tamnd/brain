## Solution

## Problem Understanding

The problem gives us two strings, `a` and `b`, where each string represents a binary number. A binary number uses only the digits `'0'` and `'1'`. Our task is to add these two binary numbers together and return the result as another binary string.

For example, the binary number `"11"` represents the decimal value `3`, and `"1"` represents the decimal value `1`. Their sum is `4`, which in binary is `"100"`.

The important detail is that the inputs are strings, not integers. This means we should think carefully about how to perform the addition. One possibility is converting both strings into integers, adding them, and converting the result back into binary. While this works in many programming languages for small values, the constraints make this approach risky because the input length can be as large as `10^4` characters. A binary number with 10,000 digits is far too large for standard integer types.

The problem is essentially asking us to simulate the way humans perform addition by hand, except in base 2 instead of base 10. In binary addition:

- `0 + 0 = 0`
- `0 + 1 = 1`
- `1 + 1 = 10`, which means write down `0` and carry `1`
- `1 + 1 + 1 = 11`, which means write down `1` and carry `1`

Since addition starts from the least significant digit, we process the strings from right to left.

The constraints tell us several important things:

- The strings are guaranteed to contain only `'0'` and `'1'`
- There are no leading zeros except for the number `"0"` itself
- The strings can be very large, so we need an efficient linear-time solution
- We cannot rely on fixed-size integer arithmetic

There are several edge cases worth considering upfront:

- One string may be longer than the other
- The final addition may produce an extra carry, such as `"1" + "1" = "10"`
- One or both strings may be `"0"`
- All digits may produce continuous carries, such as `"1111" + "1" = "10000"`

A correct solution must handle all of these cases cleanly.

## Approaches

### Brute Force Approach

The most direct idea is to convert both binary strings into decimal integers, add them together, and then convert the result back into a binary string.

In Python, this could be done using:

```
result = bin(int(a, 2) + int(b, 2))[2:]
```

This works because Python supports arbitrarily large integers. However, in many languages this approach is not safe because the numbers may exceed standard integer limits. Even though Python can technically handle it, the problem is designed to test understanding of binary addition and string manipulation rather than built-in conversion functions.

This approach also hides the actual binary addition process and does not demonstrate the intended algorithmic reasoning.

### Optimal Approach

The key observation is that binary addition works exactly like decimal addition, except each digit can only be `0` or `1`.

Instead of converting the entire strings into integers, we can process the strings digit by digit from right to left. At each step:

- Read the current bit from each string
- Add them together with any carry from the previous step
- The current result digit becomes `total % 2`
- The new carry becomes `total // 2`

We continue until we finish both strings and there is no carry left.

This approach is efficient because:

- Each character is processed exactly once
- No large integer conversion is required
- It naturally handles different string lengths
- It correctly manages carries across positions

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Converts binary strings to integers and back |
| Optimal | O(n) | O(n) | Simulates binary addition directly using strings |

## Algorithm Walkthrough

1. Initialize two pointers, one at the end of `a` and one at the end of `b`. We start from the end because binary addition begins with the least significant digit.
2. Initialize a variable called `carry` to `0`. This stores overflow from the previous digit addition.
3. Create an empty list called `result`. We use a list because appending characters is efficient, while repeated string concatenation is expensive.
4. Continue looping while:

- there are still digits left in `a`, or
- there are still digits left in `b`, or
- there is still a carry remaining
5. For each iteration:

- Start with `total = carry`
- If the pointer for `a` is valid, add the numeric value of that digit to `total`
- If the pointer for `b` is valid, add the numeric value of that digit to `total`
6. Compute the current binary digit:

- `total % 2` gives the digit to place in the result
- Append this digit to the result list
7. Compute the new carry:

- `carry = total // 2`
- In binary addition, the carry can only be `0` or `1`
8. Move both pointers one position to the left.
9. After the loop finishes, the result list contains digits in reverse order because we processed from right to left.
10. Reverse the result list and join it into a string.

## Python Solution

```
class Solution:
    def addBinary(self, a: str, b: str) -> str:
        i = len(a) - 1
        j = len(b) - 1
        
        carry = 0
        result = []
        
        while i >= 0 or j >= 0 or carry:
            total = carry
            
            if i >= 0:
                total += int(a[i])
                i -= 1
            
            if j >= 0:
                total += int(b[j])
                j -= 1
            
            result.append(str(total % 2))
            carry = total // 2
        
        return "".join(reversed(result))
```

The implementation closely follows the algorithm described earlier.

The variables `i` and `j` track the current positions in strings `a` and `b`. Since addition begins from the least significant digit, both pointers start at the last index of their respective strings.

The `carry` variable stores overflow from the previous addition step. At every iteration, we begin by setting `total` equal to the current carry value.

If either pointer is still within bounds, we convert the current character into an integer and add it to `total`.

After calculating the total:

- `total % 2` determines the current binary digit
- `total // 2` determines the next carry

Each result digit is appended to the `result` list. Since digits are generated from right to left, the list ends up reversed. Finally, we reverse the list and join it into the final binary string.

## Go Solution

```
func addBinary(a string, b string) string {
	i := len(a) - 1
	j := len(b) - 1

	carry := 0
	result := make([]byte, 0)

	for i >= 0 || j >= 0 || carry > 0 {
		total := carry

		if i >= 0 {
			total += int(a[i] - '0')
			i--
		}

		if j >= 0 {
			total += int(b[j] - '0')
			j--
		}

		result = append(result, byte(total%2)+'0')
		carry = total / 2
	}

	left := 0
	right := len(result) - 1

	for left < right {
		result[left], result[right] = result[right], result[left]
		left++
		right--
	}

	return string(result)
}
```

The Go implementation follows the same logic as the Python solution, but there are a few language-specific details.

Go strings store bytes, so individual binary digits are accessed as characters. To convert a character like `'1'` into the integer `1`, we subtract `'0'`.

The result is stored in a `[]byte` slice for efficient appending. Since digits are collected in reverse order, we manually reverse the slice before converting it back into a string.

Unlike Python, Go does not support arbitrary precision integer conversion as conveniently, so the simulation approach is the preferred solution.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each digit is processed exactly once |
| Space | O(n) | The result string stores up to n + 1 digits |

Here, `n` represents the length of the longer input string.

The algorithm is linear because each character from both strings is visited only once. The result list may contain at most one extra digit due to a final carry, so the space complexity is also linear.

## Edge Cases

One important edge case occurs when the inputs produce an extra carry after the final digit addition. For example, `"1" + "1"` results in `"10"`. A common mistake is stopping the loop too early and forgetting to append the remaining carry. The condition `while i >= 0 or j >= 0 or carry` ensures the final carry is always processed.

Another important case is when the strings have different lengths, such as `"101"` and `"1"`. A naive implementation might attempt to access invalid indices or incorrectly align digits. This solution handles differing lengths by checking whether each pointer is still valid before reading a digit.

A third important edge case is when one or both inputs are `"0"`. For example, `"0" + "0"` should return `"0"`. Since the algorithm still processes the digits normally, it naturally produces the correct answer without requiring special handling.

A more subtle case involves long chains of carries, such as `"1111" + "1"`. Each addition step generates another carry, which propagates through the entire string. The implementation correctly updates the carry after every iteration, allowing carry propagation to continue as long as necessary.
