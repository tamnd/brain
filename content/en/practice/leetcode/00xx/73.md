## Solution

## Problem Understanding

The problem gives us a two dimensional matrix of integers with dimensions `m x n`. Our task is to modify the matrix in place so that if any cell contains the value `0`, then every element in that cell's entire row and entire column must also become `0`.

The phrase "in place" is extremely important. It means we are not supposed to create and return a completely new matrix. Instead, we must directly modify the given matrix object.

For example, consider this matrix:

```
1 1 1
1 0 1
1 1 1
```

The element at position `(1,1)` is `0`. Therefore:

- Entire row `1` becomes `0`
- Entire column `1` becomes `0`

Result:

```
1 0 1
0 0 0
1 0 1
```

One important detail is that updates should behave as if they happen simultaneously. If we immediately start converting cells to `0` while scanning the matrix, those newly created zeros should not trigger additional rows and columns to become zero. Only the original zeros matter.

The constraints tell us:

- `m` and `n` can each be as large as `200`
- Total cells can therefore reach `40,000`

This size is small enough for an `O(mn)` traversal, but inefficient repeated scans could become expensive.

The follow up section strongly hints that the interviewer expects an optimized space solution. A naive approach may use additional storage proportional to the matrix size, but the best solution uses only constant extra space.

Several edge cases are important:

- A zero may appear in the first row
- A zero may appear in the first column
- Multiple zeros may overlap in affected rows and columns
- The matrix may already contain all zeros
- The matrix may contain no zeros at all
- A single row or single column matrix can create boundary handling issues

These cases are particularly important because the optimal solution reuses the first row and first column as markers.

## Approaches

### Brute Force Approach

The most direct solution is to first identify every row and column that must become zero.

We can scan the matrix once and store:

- Every row index containing a zero
- Every column index containing a zero

After that, we scan the matrix again. If a cell belongs to a marked row or a marked column, we set it to `0`.

This approach is correct because it separates detection from modification. Newly written zeros do not affect future decisions.

The downside is extra memory usage. We need additional storage for rows and columns, typically using two sets or boolean arrays.

### Key Insight for the Optimal Solution

The crucial observation is that the matrix itself can store the marker information.

Instead of allocating separate row and column arrays:

- Use the first cell of each row to indicate whether that row should become zero
- Use the first cell of each column to indicate whether that column should become zero

For example:

```
matrix[i][0]
```

can represent whether row `i` should become zero.

Similarly:

```
matrix[0][j]
```

can represent whether column `j` should become zero.

This immediately creates one complication:

```
matrix[0][0]
```

belongs to both the first row and the first column.

To resolve this ambiguity, we keep an additional boolean variable to track whether the first column should become zero.

This technique allows us to achieve constant extra space because we reuse the input matrix itself for bookkeeping.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(mn) | O(m + n) | Uses extra row and column storage |
| Optimal | O(mn) | O(1) | Uses first row and first column as markers |

## Algorithm Walkthrough

1. First, determine whether the first column needs to become zero.

We cannot immediately use the first column as markers without preserving its original state. We scan every row and check whether `matrix[i][0] == 0`. If true, we remember that the first column must eventually become zero.
2. Traverse the matrix starting from column `1`.

We skip the first column temporarily because it is being used as metadata storage.

Whenever we find a zero at position `(i, j)`:

- Set `matrix[i][0] = 0`
- Set `matrix[0][j] = 0`

This marks the entire row and column for future zeroing.
3. Use the markers to update the inner matrix.

Traverse the matrix again, starting from row `1` and column `1`.

For every cell `(i, j)`:

- If `matrix[i][0] == 0`, the row must become zero
- If `matrix[0][j] == 0`, the column must become zero

Therefore, set `matrix[i][j] = 0`.
4. Handle the first row separately.

If `matrix[0][0] == 0`, it means the first row originally contained a zero or was marked because another zero appeared in that row.

In that case, set every element in the first row to `0`.
5. Handle the first column separately.

If our saved boolean variable indicates the first column should become zero, set every element in the first column to `0`.

This order is extremely important. If we zero the first row or first column too early, we destroy the marker information before using it.

## Python Solution

```
from typing import List

class Solution:
    def setZeroes(self, matrix: List[List[int]]) -> None:
        """
        Do not return anything, modify matrix in-place instead.
        """
        
        rows = len(matrix)
        cols = len(matrix[0])

        first_col_zero = False

        # Use first row and first column as markers
        for row in range(rows):
            if matrix[row][0] == 0:
                first_col_zero = True

            for col in range(1, cols):
                if matrix[row][col] == 0:
                    matrix[row][0] = 0
                    matrix[0][col] = 0

        # Update cells based on markers
        for row in range(1, rows):
            for col in range(1, cols):
                if matrix[row][0] == 0 or matrix[0][col] == 0:
                    matrix[row][col] = 0

        # Handle first row
        if matrix[0][0] == 0:
            for col in range(cols):
                matrix[0][col] = 0

        # Handle first column
        if first_col_zero:
            for row in range(rows):
                matrix[row][0] = 0
```

The implementation begins by storing matrix dimensions and initializing a boolean variable named `first_col_zero`. This variable preserves whether the first column originally contained a zero.

The first traversal performs two tasks simultaneously:

- Detect whether the first column should become zero
- Store row and column markers inside the matrix itself

Whenever a zero is found, the corresponding row marker and column marker are updated.

The second traversal processes only the inner matrix, excluding the first row and first column. At this stage, the markers already contain all required information.

Finally, the first row and first column are processed separately because they serve dual purposes as both data and marker storage.

The order of operations ensures marker information is not overwritten prematurely.

## Go Solution

```
func setZeroes(matrix [][]int) {
    rows := len(matrix)
    cols := len(matrix[0])

    firstColZero := false

    // Use first row and first column as markers
    for row := 0; row < rows; row++ {
        if matrix[row][0] == 0 {
            firstColZero = true
        }

        for col := 1; col < cols; col++ {
            if matrix[row][col] == 0 {
                matrix[row][0] = 0
                matrix[0][col] = 0
            }
        }
    }

    // Update cells based on markers
    for row := 1; row < rows; row++ {
        for col := 1; col < cols; col++ {
            if matrix[row][0] == 0 || matrix[0][col] == 0 {
                matrix[row][col] = 0
            }
        }
    }

    // Handle first row
    if matrix[0][0] == 0 {
        for col := 0; col < cols; col++ {
            matrix[0][col] = 0
        }
    }

    // Handle first column
    if firstColZero {
        for row := 0; row < rows; row++ {
            matrix[row][0] = 0
        }
    }
}
```

The Go implementation follows exactly the same logic as the Python version. Since Go slices are reference-like structures, modifying `matrix` inside the function directly updates the caller's matrix.

No special integer overflow handling is required because we only assign zeros and read existing values.

Go does not distinguish between lists and arrays the same way Python does. Here, `[][]int` represents a slice of slices, which behaves similarly to a dynamic two dimensional array.

## Worked Examples

### Example 1

Input:

```
[
  [1,1,1],
  [1,0,1],
  [1,1,1]
]
```

### First Pass, Set Markers

| Position | Value | Action |
| --- | --- | --- |
| (1,1) | 0 | Set matrix[1][0] = 0 and matrix[0][1] = 0 |

Matrix becomes:

```
[
  [1,0,1],
  [0,0,1],
  [1,1,1]
]
```

### Second Pass, Update Inner Cells

We inspect cells excluding first row and first column.

- Cell `(1,2)` becomes `0` because row marker is `0`
- Cell `(2,1)` becomes `0` because column marker is `0`

Matrix becomes:

```
[
  [1,0,1],
  [0,0,0],
  [1,0,1]
]
```

### Handle First Row

`matrix[0][0]` is not `0`, so first row remains unchanged.

### Handle First Column

`first_col_zero` is `False`, so first column remains unchanged.

Final result:

```
[
  [1,0,1],
  [0,0,0],
  [1,0,1]
]
```

## Example 2

Input:

```
[
  [0,1,2,0],
  [3,4,5,2],
  [1,3,1,5]
]
```

### First Pass, Set Markers

Initial zero at `(0,0)`:

- `first_col_zero = True`

Zero at `(0,3)`:

- Set `matrix[0][0] = 0`
- Set `matrix[0][3] = 0`

Marker state:

```
[
  [0,1,2,0],
  [3,4,5,2],
  [1,3,1,5]
]
```

### Second Pass

Column `0` and column `3` are marked.

Affected cells:

- `(1,3)` becomes `0`
- `(2,3)` becomes `0`

Matrix:

```
[
  [0,1,2,0],
  [3,4,5,0],
  [1,3,1,0]
]
```

### Handle First Row

Since `matrix[0][0] == 0`, entire first row becomes zero.

```
[
  [0,0,0,0],
  [3,4,5,0],
  [1,3,1,0]
]
```

### Handle First Column

Since `first_col_zero == True`, entire first column becomes zero.

```
[
  [0,0,0,0],
  [0,4,5,0],
  [0,3,1,0]
]
```

Final result:

```
[
  [0,0,0,0],
  [0,4,5,0],
  [0,3,1,0]
]
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(mn) | Every cell is visited a constant number of times |
| Space | O(1) | Only a single boolean variable is used outside the matrix |

The algorithm performs several matrix traversals, but each traversal touches every cell at most once. Since there are `m * n` cells, the total running time is linear relative to the matrix size.

The space complexity is constant because no auxiliary arrays, hash maps, or sets are allocated. The matrix itself stores the marker information.

## Edge Cases

### Matrix With No Zeroes

Consider:

```
[
  [1,2],
  [3,4]
]
```

No markers are created, and neither the first row nor first column is modified. The matrix remains unchanged.

This case is important because some incorrect implementations accidentally overwrite values during traversal.

### Matrix Where First Row Contains Zero

Consider:

```
[
  [1,0,3]
]
```

The first row itself is used as marker storage, so careless implementations may lose information.

This solution uses `matrix[0][0]` to determine whether the first row must become zero, ensuring correct handling.

### Matrix Where First Column Contains Zero

Consider:

```
[
  [1],
  [0],
  [3]
]
```

The first column cannot rely solely on `matrix[0][0]` because that cell already represents the first row.

The separate boolean variable `first_col_zero` preserves this information safely.

### Entire Matrix Already Zero

Consider:

```
[
  [0,0],
  [0,0]
]
```

Every row and column marker becomes zero immediately, and the algorithm correctly preserves the all-zero matrix without introducing errors.

### Single Cell Matrix

Consider:

```
[
  [0]
]
```

or

```
[
  [5]
]
```

Single element matrices often expose indexing mistakes. This implementation correctly handles both because all loops safely work even when `rows == 1` or `cols == 1`.
