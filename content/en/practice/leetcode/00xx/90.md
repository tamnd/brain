## Solution

## Problem Understanding

The problem asks us to generate every possible subset of a given integer array. A subset is any collection of elements chosen from the array, including the empty subset and the subset containing every element.

The challenge is that the input array may contain duplicate values. While generating subsets is already a classic combinatorial problem, duplicates introduce an additional constraint: the final result must not contain duplicate subsets.

For example, if the input is `[1,2,2]`, a naive subset generator might produce the subset `[2]` twice, once using the first `2` and once using the second `2`. The problem requires us to avoid this duplication and return only unique subsets.

The input consists of:

- An integer array `nums`
- The array length is between `1` and `10`
- Values range from `-10` to `10`

The small constraint on `nums.length` is important. Since every element can either be included or excluded, the total number of subsets is naturally exponential. With at most 10 elements, the total number of possible subsets is at most `2^10 = 1024`, which is manageable.

Several edge cases are important:

- Arrays containing all duplicates, such as `[2,2,2]`
- Arrays containing no duplicates
- Arrays containing negative numbers
- Arrays of length 1
- Duplicate values appearing consecutively after sorting

A naive implementation that blindly generates all subsets will often create duplicate results. The main difficulty is designing the generation process so that duplicate subsets are skipped efficiently.

## Approaches

### Brute Force Approach

The brute force method generates every possible subset using binary masks or recursion. Since each element can either be included or excluded, there are `2^n` possible combinations.

After generating all subsets, we could sort each subset and insert it into a set to eliminate duplicates.

For example:

- Generate all subsets
- Convert each subset into a tuple or string representation
- Store it in a hash set
- Convert the unique results back into a list

This works because sets automatically remove duplicate entries.

However, this approach is inefficient because duplicate subsets are generated first and removed later. When the array contains many repeated values, a large amount of unnecessary work is performed.

### Optimal Approach

The key observation is that duplicate subsets occur because duplicate numbers can be chosen in equivalent ways.

To avoid generating duplicates in the first place, we sort the array first. Sorting places equal elements next to each other, which allows us to detect duplicates during backtracking.

During recursion:

- We iterate through candidate elements
- If the current number is the same as the previous number at the same recursive depth, we skip it

This ensures that only the first occurrence at each decision level is processed.

For example, after sorting `[1,2,2]`:

- At a given recursive level, choosing the first `2` is allowed
- Choosing the second `2` immediately afterward at the same level would generate identical subsets, so we skip it

This pruning strategy prevents duplicate subsets from ever being created.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(2^n * n) | Generates all subsets, then removes duplicates using a set |
| Optimal | O(2^n * n) | O(2^n * n) | Uses sorting and backtracking to avoid duplicates during generation |

## Algorithm Walkthrough

1. Sort the input array.

Sorting is essential because duplicate values must appear next to each other. Without sorting, we cannot reliably detect duplicate choices during recursion.
2. Initialize the result list.

This list stores all unique subsets generated during backtracking.
3. Define a recursive backtracking function.

The function needs:

- The current starting index
- The current subset being built
4. Add the current subset to the result.

Every recursive state represents a valid subset, including the empty subset.
5. Iterate through the array starting from the current index.

Each iteration represents choosing a new element to include in the subset.
6. Skip duplicates at the same recursive level.

If:

- `i > start`
- `nums[i] == nums[i - 1]`

then skip the current element.

This condition means:

- We are at the same recursive depth
- The current value is identical to the previous value
- Choosing it would generate duplicate subsets
7. Include the current element.

Add the element to the current subset.
8. Recurse to explore deeper subsets.

Call the recursive function using `i + 1` as the next starting index.
9. Backtrack.

Remove the last added element so the next iteration can explore another possibility.
10. Continue until all recursive paths are explored.

Eventually, every unique subset is generated exactly once.

## Python Solution

```
from typing import List

class Solution:
    def subsetsWithDup(self, nums: List[int]) -> List[List[int]]:
        nums.sort()
        
        result = []
        subset = []
        
        def backtrack(start: int) -> None:
            result.append(subset[:])
            
            for i in range(start, len(nums)):
                if i > start and nums[i] == nums[i - 1]:
                    continue
                
                subset.append(nums[i])
                backtrack(i + 1)
                subset.pop()
        
        backtrack(0)
        
        return result
```

The implementation begins by sorting the input array. This is the foundation of the duplicate-skipping logic because equal values become adjacent.

The `result` list stores all unique subsets. The `subset` list represents the current subset being constructed during recursion.

The `backtrack` function takes a `start` index indicating where the next candidate selection should begin. At every recursive call, the current subset is copied into the result list because every state corresponds to a valid subset.

The loop iterates through remaining candidates. Before choosing an element, the code checks:

```
if i > start and nums[i] == nums[i - 1]:
```

This condition detects duplicate choices at the same recursive depth. If true, the element is skipped to prevent duplicate subset generation.

When an element is chosen, it is appended to `subset`, recursion explores deeper combinations, and then `pop()` removes it during backtracking so the next branch can be explored cleanly.

The recursion systematically explores every valid unique subset.

## Go Solution

```
package main

import "sort"

func subsetsWithDup(nums []int) [][]int {
	sort.Ints(nums)

	var result [][]int
	var subset []int

	var backtrack func(int)

	backtrack = func(start int) {
		current := make([]int, len(subset))
		copy(current, subset)

		result = append(result, current)

		for i := start; i < len(nums); i++ {
			if i > start && nums[i] == nums[i-1] {
				continue
			}

			subset = append(subset, nums[i])

			backtrack(i + 1)

			subset = subset[:len(subset)-1]
		}
	}

	backtrack(0)

	return result
}
```

The Go implementation follows the same backtracking strategy as the Python version.

One important difference is slice handling. In Go, slices share underlying memory, so we must explicitly create a copy before appending the current subset into `result`:

```
current := make([]int, len(subset))
copy(current, subset)
```

Without this copy, later modifications to `subset` would affect previously stored results.

Go slices naturally support dynamic resizing, making them well suited for recursive backtracking problems like this one.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(2^n * n) | There are up to 2^n subsets, and copying each subset may take O(n) time |
| Space | O(2^n * n) | Result storage dominates because all subsets must be stored |

The recursion tree explores every unique subset exactly once. In the worst case, when all elements are distinct, there are `2^n` subsets.

Each subset may require up to `O(n)` time to copy into the result list, producing a total complexity of `O(2^n * n)`.

The recursion depth itself is at most `O(n)`, but storing all subsets dominates overall memory usage.

## Edge Cases

### All Elements Are Duplicates

Consider:

```
[2,2,2]
```

A naive implementation would generate many duplicate subsets such as `[2]` multiple times.

The sorting and duplicate-skipping condition ensure that only one recursive branch is explored for identical values at the same level. The algorithm correctly produces:

```
[
    [],
    [2],
    [2,2],
    [2,2,2]
]
```

### Single Element Array

Consider:

```
[5]
```

The only valid subsets are the empty subset and the subset containing the single element.

The recursion handles this naturally because every recursive state is added to the result before further exploration.

### Negative Numbers

Consider:

```
[-1,-1,2]
```

Negative values do not affect the algorithm because the logic depends only on equality comparisons after sorting.

The sorted array becomes:

```
[-1,-1,2]
```

Duplicate skipping still works correctly, and all unique subsets are generated properly.

### No Duplicates Present

Consider:

```
[1,2,3]
```

In this case, the duplicate-skipping condition never triggers. The algorithm behaves like a standard subset backtracking solution and generates all `2^n` subsets correctly.
