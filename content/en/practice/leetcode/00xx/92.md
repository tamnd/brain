## Solution

## Problem Understanding

This problem asks us to reverse only a specific portion of a singly linked list. We are given the head of the list and two positions, `left` and `right`, using 1-based indexing. Our task is to reverse all nodes between these two positions, inclusive, while leaving the rest of the list unchanged.

For example, if the input list is:

```
1 -> 2 -> 3 -> 4 -> 5
```

and `left = 2`, `right = 4`, then the sublist:

```
2 -> 3 -> 4
```

must be reversed into:

```
4 -> 3 -> 2
```

The final list becomes:

```
1 -> 4 -> 3 -> 2 -> 5
```

The important detail is that we are reversing the actual node connections, not merely swapping values. This means pointer manipulation is required.

The constraints tell us that the list contains at most 500 nodes, which is small enough that even less efficient solutions could pass. However, the follow up specifically asks whether the operation can be done in one pass, which strongly suggests that the intended solution should manipulate pointers directly while traversing the list only once.

Several edge cases are important:

- The reversal may start at the head of the list, meaning `left = 1`.
- The reversal may involve only a single node, meaning `left == right`.
- The reversal may extend to the end of the list.
- The list may contain only one node.
- Incorrect pointer handling can easily disconnect parts of the list or create cycles.

Because linked lists do not allow random access, careful pointer tracking is essential.

## Approaches

### Brute Force Approach

A straightforward approach is to first copy all node values into an array. Once the values are stored, we can reverse the portion of the array between indices `left - 1` and `right - 1`. After modifying the array, we traverse the linked list again and overwrite each node’s value with the updated array values.

This works because reversing the selected section of the array produces the same visible ordering as reversing the actual linked list nodes.

However, this approach is not ideal because it does not truly reverse the linked list structure. Instead, it modifies node values. It also requires extra memory proportional to the size of the list.

### Optimal Approach

The key insight is that linked list reversal can be performed entirely through pointer manipulation.

Instead of copying values, we can directly reverse the connections between nodes in the target range. The idea is to:

1. Traverse to the node immediately before the reversal section.
2. Reverse the pointers between `left` and `right`.
3. Reconnect the reversed sublist back into the original list.

A dummy node is extremely useful here because it simplifies handling cases where the reversal starts at the head.

The optimal solution performs the reversal in one traversal and uses only constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Copies node values into an array and reverses values |
| Optimal | O(n) | O(1) | Reverses pointers directly in one pass |

## Algorithm Walkthrough

### Optimal Pointer Reversal Algorithm

1. Create a dummy node and point it to the head of the list.

This simplifies edge cases where the reversal starts at position 1. Without a dummy node, updating the head would require separate handling.
2. Move a pointer called `prev` to the node immediately before position `left`.

After this step:

- `prev` points to the node before the reversal section.
- `prev.next` is the first node to reverse.
3. Store the first node of the reversal section in a variable called `current`.

This node will eventually become the tail of the reversed section.
4. Reverse the sublist using iterative pointer manipulation.

For each node in the reversal range:

- Remove the node immediately after `current`
- Insert it right after `prev`

This technique is sometimes called the "head insertion" method.
5. Continue this process until all nodes between `left` and `right` have been reversed.
6. Return `dummy.next`.

This correctly handles both normal cases and cases where the head changed due to reversal.

The reason this works is that each iteration moves one node from the unreversed portion of the sublist to the front of the reversed portion.

## Python Solution

```
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

from typing import Optional

class Solution:
    def reverseBetween(
        self,
        head: Optional[ListNode],
        left: int,
        right: int
    ) -> Optional[ListNode]:

        if not head or left == right:
            return head

        dummy = ListNode(0)
        dummy.next = head

        prev = dummy

        # Move prev to the node before position left
        for _ in range(left - 1):
            prev = prev.next

        current = prev.next

        # Reverse nodes between left and right
        for _ in range(right - left):
            next_node = current.next

            current.next = next_node.next

            next_node.next = prev.next

            prev.next = next_node

        return dummy.next
```

The implementation begins by handling trivial cases. If the list is empty or the reversal range contains only one node, no work is necessary.

A dummy node is created and attached before the original head. This allows uniform handling of reversals starting at the head.

The `prev` pointer is advanced until it reaches the node immediately before the reversal section. The `current` pointer then marks the first node in the section being reversed.

The reversal loop repeatedly extracts the node after `current` and inserts it immediately after `prev`. This gradually builds the reversed portion at the front of the sublist.

At the end, `dummy.next` is returned because the head may have changed during reversal.

## Go Solution

```
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func reverseBetween(head *ListNode, left int, right int) *ListNode {
	if head == nil || left == right {
		return head
	}

	dummy := &ListNode{Next: head}

	prev := dummy

	// Move prev to the node before position left
	for i := 0; i < left-1; i++ {
		prev = prev.Next
	}

	current := prev.Next

	// Reverse nodes between left and right
	for i := 0; i < right-left; i++ {
		nextNode := current.Next

		current.Next = nextNode.Next

		nextNode.Next = prev.Next

		prev.Next = nextNode
	}

	return dummy.Next
}
```

The Go implementation follows exactly the same algorithmic structure as the Python version. The primary differences are language-specific syntax and pointer handling.

In Go, linked list nodes are manipulated explicitly using pointers. Since Go does not have `None`, it uses `nil` to represent the absence of a node.

The dummy node is allocated with:

```
dummy := &ListNode{Next: head}
```

Pointer updates work identically to the Python solution, but the syntax is more explicit due to Go's static typing.

## Worked Examples

### Example 1

Input:

```
head = [1,2,3,4,5]
left = 2
right = 4
```

Initial list:

```
dummy -> 1 -> 2 -> 3 -> 4 -> 5
```

After moving `prev`:

```
prev = 1
current = 2
```

We now reverse positions 2 through 4.

| Iteration | next_node | List State |
| --- | --- | --- |
| Start | - | 1 -> 2 -> 3 -> 4 -> 5 |
| 1 | 3 | 1 -> 3 -> 2 -> 4 -> 5 |
| 2 | 4 | 1 -> 4 -> 3 -> 2 -> 5 |

Final result:

```
1 -> 4 -> 3 -> 2 -> 5
```

### Example 2

Input:

```
head = [5]
left = 1
right = 1
```

Since `left == right`, no reversal is needed.

The function immediately returns:

```
5
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited at most once |
| Space | O(1) | Only a few pointers are used |

The algorithm runs in linear time because we traverse the list only once to locate the reversal region and perform the reversal. No nested traversal occurs.

The space complexity is constant because the algorithm modifies the list in place and uses only a fixed number of pointer variables regardless of input size.

## Edge Cases

### Reversal Starts at the Head

Consider:

```
head = [1,2,3]
left = 1
right = 2
```

Without a dummy node, reversing from the head becomes tricky because the head pointer itself changes. Many buggy implementations fail to update the head correctly.

The dummy node solves this cleanly because there is always a node before the reversal section.

### Single Node Reversal

Consider:

```
head = [1,2,3]
left = 2
right = 2
```

No actual reversal should happen because the range contains only one node.

The implementation handles this immediately with:

```
if left == right:
    return head
```

This avoids unnecessary pointer manipulation.

### Reversal Extends to the End

Consider:

```
head = [1,2,3,4]
left = 2
right = 4
```

The reversed section includes the tail of the list. Incorrect pointer updates can accidentally disconnect the list or lose the tail reference.

The implementation safely preserves connections by carefully updating `current.next` before moving nodes.

### Entire List Reversal

Consider:

```
head = [1,2,3,4]
left = 1
right = 4
```

This reverses the whole list. Some implementations fail because both the head and tail change positions.

Using the dummy node and head insertion method ensures the full reversal works without special-case logic.
