## Solution

## Problem Understanding

This problem asks us to generate **every possible subset** of a given array of unique integers. A subset is any collection of elements selected from the array, including the possibility of selecting no elements at all. Since we want _all possible subsets_, we are effectively being asked to compute the **power set** of the input array.

For example, if the input is `[1,2,3]`, the valid subsets include:

- The empty subset `[]`
- Single element subsets like `[1]`, `[2]`, `[3]`
- Multi element subsets like `[1,2]`, `[1,3]`, `[2,3]`
- The full subset `[1,2,3]`

The order of subsets in the output does not matter. Likewise, the order of elements inside each subset follows the order in which we build them, but any valid arrangement is accepted by LeetCode.

The input array `nums` contains **unique integers**, which is an important guarantee. Since there are no duplicates in the input, we do not need special handling to avoid repeated subsets. Every combination of inclusion and exclusion naturally produces a unique subset.

The constraints tell us:

- `1 <= nums.length <= 10`
- Each number is in the range `[-10, 10]`
- All values are unique

The key observation is that the array length is very small, at most `10`. Since every element can either be included or excluded, the total number of subsets is `2^n`. With `n = 10`, that means at most `1024` subsets, which is very manageable.

There are several edge cases worth considering upfront. If the array contains only one element, such as `[0]`, the answer should simply be `[[], [0]]`. Since elements are unique, duplicate subsets are impossible, simplifying the implementation. Another important case is ensuring the empty subset is included, because every power set always contains it. A naive implementation might accidentally omit it.

## Approaches

### Brute Force Approach

A brute-force solution would generate every possible combination of elements and check whether each combination forms a valid subset. One way to think about this is to examine every binary pattern from `0` to `2^n - 1`, where each bit determines whether an element is included.

For example, with `nums = [1,2,3]`:

- `000` → `[]`
- `001` → `[3]`
- `010` → `[2]`
- `011` → `[2,3]`
- `100` → `[1]`

This works because every element has exactly two choices: include it or exclude it.

The brute-force approach is correct because it systematically explores every possible selection pattern. However, it still requires generating all `2^n` subsets, and each subset may require up to `n` work to construct.

### Optimal Approach

The key insight is that this problem naturally fits **backtracking**.

At every position in the array, we have exactly two decisions:

1. Include the current number
2. Exclude the current number

By recursively exploring both possibilities, we build every subset exactly once.

Backtracking works especially well here because subset construction forms a decision tree. At each level, we decide whether to keep an element, then move to the next index. Once we finish one branch, we undo the choice and explore the alternative branch.

Since the total number of valid outputs is already `2^n`, no algorithm can fundamentally avoid generating all subsets. The goal is simply to generate them efficiently and clearly.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × 2^n) | O(n × 2^n) | Enumerates every inclusion pattern explicitly |
| Optimal | O(n × 2^n) | O(n × 2^n) | Uses backtracking to construct subsets incrementally |

## Algorithm Walkthrough

The optimal solution uses recursive backtracking.

1. Initialize an empty result list that will store all subsets.
2. Define a recursive helper function that takes the current index and the current subset being built.
3. At the start of each recursive call, add a copy of the current subset into the result. This is important because every intermediate state represents a valid subset.
4. Iterate through the remaining elements starting from the current index. For each element:

- Add the current element to the subset.
- Recursively continue building subsets from the next index.
- Remove the last element after recursion finishes.
5. Removing the element is the "backtracking" step. It restores the previous state so we can explore other possibilities without interference.
6. Continue this process until every recursive path has been explored.

For example, with `[1,2,3]`, the recursion tree explores:

- `[]`

- `[1]`

- `[1,2]`

- `[1,2,3]`
- `[1,3]`
- `[2]`

- `[2,3]`
- `[3]`

Every node in this tree corresponds to a valid subset.

## Python Solution

```
from typing import List

class Solution:
    def subsets(self, nums: List[int]) -> List[List[int]]:
        result = []
        current_subset = []

        def backtrack(start_index: int) -> None:
            result.append(current_subset[:])

            for index in range(start_index, len(nums)):
                current_subset.append(nums[index])
                backtrack(index + 1)
                current_subset.pop()

        backtrack(0)
        return result
```

The implementation begins by creating `result`, which stores all generated subsets, and `current_subset`, which tracks the subset currently being constructed.

The nested `backtrack` function takes a `start_index`. This index ensures that elements are only chosen from positions ahead of the current one, preventing duplicate combinations and preserving ordering.

At the beginning of every recursive call, we append a copy of the current subset using `current_subset[:]`. A copy is necessary because lists in Python are mutable. If we appended the original list reference, later modifications would affect previously stored subsets.

The `for` loop iterates through remaining elements. For each number, we include it in the subset using `append()`, recursively continue exploration, and then remove it with `pop()` after returning. This undo operation is the core of backtracking and allows reuse of the same list structure efficiently.

Finally, we start recursion with `backtrack(0)` and return the completed result.

## Go Solution

```
func subsets(nums []int) [][]int {
	result := [][]int{}
	currentSubset := []int{}

	var backtrack func(startIndex int)

	backtrack = func(startIndex int) {
		subsetCopy := make([]int, len(currentSubset))
		copy(subsetCopy, currentSubset)
		result = append(result, subsetCopy)

		for index := startIndex; index < len(nums); index++ {
			currentSubset = append(currentSubset, nums[index])
			backtrack(index + 1)
			currentSubset = currentSubset[:len(currentSubset)-1]
		}
	}

	backtrack(0)

	return result
}
```

The Go implementation follows the same recursive logic as the Python solution, but slice handling requires extra care.

Since slices reference underlying arrays, we must explicitly create a copy before appending to `result`. This is done using `make()` and `copy()`. Without copying, every stored subset would reference the same backing array and eventually contain incorrect values.

The backtracking step removes the last element using slice truncation:

```
currentSubset = currentSubset[:len(currentSubset)-1]
```

Unlike Python, there is no concern about integer overflow because the problem constraints are very small.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × 2^n) | There are `2^n` subsets, and copying each subset can take up to `O(n)` time |
| Space | O(n × 2^n) | The result stores all subsets, and total storage across subsets is proportional to `n × 2^n` |

The time complexity comes from generating every subset. Since there are `2^n` possible subsets and each subset copy may take `O(n)` time, the total complexity becomes `O(n × 2^n)`.

The space complexity is dominated by storing the final answer. Although recursion only requires `O(n)` stack space, the output itself contains `2^n` subsets, each potentially holding up to `n` elements.

## Edge Cases

### Single Element Array

If the input contains only one element, such as `[5]`, the correct answer must include both the empty subset and the single element subset.

A buggy implementation might forget to include the empty subset or mishandle recursion termination. This implementation handles it naturally because the empty subset is added before any recursive exploration begins.

### Negative Numbers

The problem allows values between `-10` and `10`, including negatives such as `[-1, -2]`.

Since the algorithm never depends on numeric comparisons or arithmetic, negative values are handled identically to positive ones. Elements are simply included or excluded.

### Maximum Input Size

When `nums.length = 10`, there are `2^10 = 1024` subsets.

A naive algorithm with unnecessary repeated computation could become inefficient. The backtracking solution remains efficient because it explores each valid subset exactly once, which is optimal for this problem.

### Empty Subset Requirement

The empty subset `[]` is always part of the power set.

This can be easy to forget if the implementation only appends subsets after selecting an element. In this solution, the empty subset is included immediately at the start of recursion when `current_subset` is initially empty, guaranteeing correctness.
