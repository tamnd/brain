## Solution

## Problem Understanding

The problem gives us the root of a binary tree and asks whether the tree satisfies the rules of a Binary Search Tree, commonly abbreviated as BST.

A BST is not just a tree where each node is larger than its left child and smaller than its right child. The rule applies to the entire subtree.

For every node:

- Every value in the left subtree must be strictly smaller than the node's value.
- Every value in the right subtree must be strictly greater than the node's value.
- Both subtrees must themselves also be valid BSTs.

The keyword "strictly" is very important. Duplicate values are not allowed in a valid BST.

The input is represented as a binary tree. Each node contains:

- an integer value
- a pointer to a left child
- a pointer to a right child

The output is a boolean:

- `true` if the tree satisfies BST rules
- `false` otherwise

The constraints tell us that the tree can contain up to `10^4` nodes. This is large enough that we should avoid inefficient repeated subtree scans. We need an algorithm that visits each node only once or close to once.

Another important constraint is the node value range:

```
-2^31 <= Node.val <= 2^31 - 1
```

This means we cannot rely on adding or subtracting `1` safely to create boundaries, because overflow issues could occur in some languages.

Several edge cases are important:

- A tree with only one node is always a valid BST.
- A node deep inside a subtree may violate BST rules even if its immediate parent relationship looks correct.
- Duplicate values invalidate the BST because the rules require strict inequality.
- Extremely unbalanced trees behave like linked lists, which affects recursion depth and space usage.

A common mistake is checking only:

```
left child < parent < right child
```

This is insufficient because descendants must also satisfy the full ancestor constraints.

For example:

```
      5
     / \
    1   4
       / \
      3   6
```

The node `4` is less than `5`, so the entire right subtree violates BST rules even though `3 < 4 < 6` locally appears valid.

## Approaches

### Brute Force Approach

A straightforward solution is to validate every subtree independently.

For each node:

- Find the maximum value in its left subtree.
- Find the minimum value in its right subtree.
- Verify:

- `max(left subtree) < current node`
- `min(right subtree) > current node`
- Recursively repeat for both subtrees.

This works because it directly checks the BST definition.

However, it is inefficient. Every node repeatedly scans large parts of its subtrees. In the worst case, such as a skewed tree, each node may traverse almost the entire remaining tree.

This produces quadratic time complexity.

### Key Insight for the Optimal Solution

The important observation is that BST validity depends on allowable value ranges.

Instead of checking only parent-child relationships, each node should know:

- the minimum value it is allowed to have
- the maximum value it is allowed to have

When traversing:

- The left child inherits the current lower bound, but its upper bound becomes the current node value.
- The right child inherits the current upper bound, but its lower bound becomes the current node value.

This propagates all ancestor constraints downward through the recursion.

For example:

```
      10
     /  \
    5   15
       /  \
      6   20
```

The node `6` is in the right subtree of `10`, so it must be greater than `10`. Even though `6 < 15` is locally correct, it violates the inherited lower bound from `10`.

This approach visits each node exactly once.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(h) | Repeatedly scans subtrees for min/max values |
| Optimal | O(n) | O(h) | Passes valid ranges during DFS traversal |

Here:

- `n` is the number of nodes
- `h` is the height of the tree

## Algorithm Walkthrough

1. Start a Depth-First Search from the root node.
2. For every node, maintain two boundaries:

- `lower_bound`
- `upper_bound`

The current node value must satisfy:

```
lower_bound < node.val < upper_bound
```
3. Initially:

- the root has no lower limit
- the root has no upper limit

Conceptually, this means:

```
(-infinity, +infinity)
```
4. When moving to the left child:

- the upper bound becomes the current node value
- the lower bound remains unchanged

Because every node in the left subtree must be smaller than the current node.
5. When moving to the right child:

- the lower bound becomes the current node value
- the upper bound remains unchanged

Because every node in the right subtree must be larger than the current node.
6. At each node:

- if the value violates the range, immediately return `False`
- otherwise continue recursively
7. If every node satisfies its valid range, return `True`.

This works because every recursive call carries all ancestor constraints downward through the tree.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def isValidBST(self, root: Optional[TreeNode]) -> bool:
        
        def validate(
            node: Optional[TreeNode],
            lower_bound: float,
            upper_bound: float
        ) -> bool:
            
            if not node:
                return True
            
            if not (lower_bound < node.val < upper_bound):
                return False
            
            return (
                validate(node.left, lower_bound, node.val)
                and
                validate(node.right, node.val, upper_bound)
            )
        
        return validate(root, float("-inf"), float("inf"))
```

The implementation uses a recursive helper function named `validate`.

The helper receives:

- the current node
- the minimum allowed value
- the maximum allowed value

The base case handles `None` nodes. An empty subtree is always valid.

Next, the function checks whether the current node value lies strictly inside the valid range. If not, the BST property has been violated, so the function immediately returns `False`.

If the current node is valid:

- the left subtree is checked with an updated upper bound
- the right subtree is checked with an updated lower bound

Both recursive calls must return `True` for the current subtree to be valid.

The recursion naturally propagates all ancestor constraints downward through the tree.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

import "math"

func isValidBST(root *TreeNode) bool {
    
    var validate func(node *TreeNode, lowerBound, upperBound int64) bool
    
    validate = func(node *TreeNode, lowerBound, upperBound int64) bool {
        if node == nil {
            return true
        }
        
        value := int64(node.Val)
        
        if value <= lowerBound || value >= upperBound {
            return false
        }
        
        return validate(node.Left, lowerBound, value) &&
            validate(node.Right, value, upperBound)
    }
    
    return validate(root, math.MinInt64, math.MaxInt64)
}
```

The Go implementation follows the same recursive logic as the Python solution.

One important difference is integer handling. Since node values may be near the limits of 32-bit integers, the implementation uses `int64` boundaries with:

- `math.MinInt64`
- `math.MaxInt64`

This avoids overflow problems and safely supports all valid input values.

Go also uses `nil` instead of Python's `None`.

## Worked Examples

### Example 1

```
Input: [2,1,3]
```

Tree:

```
    2
   / \
  1   3
```

Initial call:

```
validate(2, -inf, +inf)
```

| Current Node | Lower Bound | Upper Bound | Valid? |
| --- | --- | --- | --- |
| 2 | -inf | +inf | Yes |
| 1 | -inf | 2 | Yes |
| 3 | 2 | +inf | Yes |

Step-by-step traversal:

1. Node `2` is inside `(-inf, +inf)`
2. Move left:

- Node `1`
- Valid range: `(-inf, 2)`
3. Move right:

- Node `3`
- Valid range: `(2, +inf)`
4. All nodes satisfy BST constraints

Result:

```
True
```

```
Input: [5,1,4,null,null,3,6]
```

Tree:

      5
     / \
    1   4
       / \
      3   6
```

Initial call:

```
validate(5, -inf, +inf)
```

| Current Node | Lower Bound | Upper Bound | Valid? |
| --- | --- | --- | --- |
| 5 | -inf | +inf | Yes |
| 1 | -inf | 5 | Yes |
| 4 | 5 | +inf | No |

Step-by-step traversal:

1. Node `5` is valid
2. Left child `1` is valid because:

```
-inf < 1 < 5

```
5 < 4 < +inf
```
4. This condition fails immediately

Even though `4` is the right child of `5`, it is smaller than `5`, so the BST property is violated.

Result:

```
False
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited exactly once |
| Space | O(h) | Recursive call stack depends on tree height |

The algorithm performs a single DFS traversal through the tree. Every node is processed once, and each operation at a node takes constant time.

The space complexity comes from recursion depth:

- balanced tree: `O(log n)`
- skewed tree: `O(n)`

No additional data structures proportional to the number of nodes are used.

## Edge Cases

### Single Node Tree

A tree containing only one node is always a valid BST because there are no children that could violate ordering rules.

Example:

```
[10]
```

The algorithm handles this naturally:

- the node lies within `(-inf, +inf)`
- both recursive calls receive `None`
- both return `True`

### Duplicate Values

BST rules require strict inequality.

Example:

```
    2
   / \
  1   2
```

The right child equals the root, which is invalid.

The implementation correctly rejects this because it checks:

```
lower_bound < node.val < upper_bound
```

Equality is not allowed.

### Deep Subtree Violations

This is the most common source of incorrect solutions.

Example:

```
      10
     /  \
    5   15
       /  \
      6   20
```

The node `6` appears valid relative to parent `15`, but it violates the rule that every node in the right subtree of `10` must be greater than `10`.

The recursive boundary propagation correctly catches this:

- node `6` receives bounds `(10, 15)`
- `6` is not greater than `10`
- the algorithm returns `False`

### Extremely Skewed Trees

A tree may resemble a linked list:

```
1
 \
  2
   \
    3
```

or

```
    3
   /
  2
 /
1
```

The algorithm still works correctly because each node simply passes updated bounds downward.

The only impact is recursion depth, which becomes `O(n)` in the worst case.
