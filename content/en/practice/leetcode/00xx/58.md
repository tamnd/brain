## Solution

## Problem Understanding

This problem asks us to find the length of the final word in a string. A word is defined as a sequence of consecutive non-space characters. The string may contain leading spaces, trailing spaces, or multiple spaces between words.

The input is a single string `s`. The output should be an integer representing the number of characters in the last word that appears in the string.

For example, in the string `"Hello World"`, the last word is `"World"`, so the answer is `5`. In the string `"   fly me   to   the moon  "`, the trailing spaces should be ignored, and the last word is `"moon"`, so the answer is `4`.

The constraints tell us that:

- The string length is at least 1 and at most `10^4`
- The string contains only English letters and spaces
- There is always at least one valid word

These constraints are small enough that even simple solutions will run efficiently, but the problem is mainly testing careful handling of spaces.

Several edge cases are important:

- Strings with trailing spaces, such as `"hello   "`
- Strings with only one word, such as `"leetcode"`
- Strings with multiple spaces between words
- Strings with leading spaces

A naive implementation that simply splits on spaces without handling empty strings carefully may produce incorrect results.

## Approaches

### Brute Force Approach

One straightforward solution is to split the string into words using spaces, remove any empty strings, and then return the length of the final word.

For example:

```
"   fly me   to   the moon  "
```

After splitting by spaces, we might get:

```
["", "", "", "fly", "me", "", "", "to", "", "", "the", "moon", "", ""]
```

We can filter out empty strings and then take the last remaining word.

This approach is easy to understand and implement. However, it creates additional arrays and temporary strings, which increases space usage unnecessarily.

### Optimal Approach

A better solution is to scan the string from right to left.

The key observation is that we only care about the final word. We do not need to process the entire string into separate words.

The algorithm works in two phases:

1. Skip all trailing spaces from the end of the string.
2. Count characters until we reach another space or the beginning of the string.

This approach avoids creating extra arrays and uses constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Splits the string into multiple substrings |
| Optimal | O(n) | O(1) | Scans backward without extra storage |

## Algorithm Walkthrough

1. Start from the last index of the string.
2. Move left while the current character is a space. This step removes trailing spaces from consideration because trailing spaces are not part of the last word.
3. Once the first non-space character is found, begin counting characters. This position marks the end of the last word.
4. Continue moving left while the current character is not a space. Increment the counter for each character encountered.
5. Stop when either:

- The beginning of the string is reached, or
- A space is encountered
6. Return the counter value, which represents the length of the last word.

This algorithm is efficient because each character is visited at most once.

## Python Solution

```
class Solution:
    def lengthOfLastWord(self, s: str) -> int:
        index = len(s) - 1

        # Skip trailing spaces
        while index >= 0 and s[index] == ' ':
            index -= 1

        length = 0

        # Count characters in the last word
        while index >= 0 and s[index] != ' ':
            length += 1
            index -= 1

        return length
```

The implementation begins by setting a pointer at the end of the string.

The first `while` loop skips all trailing spaces. This is necessary because the last character of the string may not belong to the final word.

After trailing spaces are skipped, the second loop counts the number of consecutive non-space characters. These characters form the last word.

The loop stops once a space or the beginning of the string is reached. The accumulated count is then returned.

The implementation uses only two integer variables and does not allocate additional data structures.

## Go Solution

```
func lengthOfLastWord(s string) int {
    index := len(s) - 1

    // Skip trailing spaces
    for index >= 0 && s[index] == ' ' {
        index--
    }

    length := 0

    // Count characters in the last word
    for index >= 0 && s[index] != ' ' {
        length++
        index--
    }

    return length
}
```

The Go implementation follows exactly the same logic as the Python solution.

One difference is that Go strings are indexed as byte slices. Since the problem guarantees only English letters and spaces, byte indexing is completely safe here. If Unicode characters were involved, a rune-based solution would be necessary.

No special handling for empty strings is required because the constraints guarantee at least one word exists.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | In the worst case, the algorithm scans the entire string once |
| Space | O(1) | Only a few integer variables are used |

The algorithm is linear because each character is processed at most one time while moving from right to left. No additional arrays, stacks, or hash maps are created, so the extra memory usage remains constant.

## Edge Cases

### Trailing Spaces

A string such as `"hello   "` can cause bugs if trailing spaces are treated as part of the final word. The implementation handles this by first skipping all spaces from the end before counting characters.

### Single Word Input

A string such as `"leetcode"` contains only one word and no spaces. Some implementations may incorrectly expect at least one separating space. This solution works correctly because the counting loop continues until the index becomes negative.

### Multiple Spaces Between Words

A string such as `"a   b"` contains several consecutive spaces between words. A naive split-based solution might produce empty strings that need filtering. The backward scanning approach naturally ignores unnecessary spaces and directly identifies the last word.

### Leading Spaces

A string such as `"   hello"` begins with spaces. These spaces do not affect the result because the algorithm scans from the end of the string and only processes the final word.
