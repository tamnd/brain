## Solution

## Problem Understanding

The problem asks us to find the first position where the string `needle` appears inside another string called `haystack`. If `needle` does not exist anywhere inside `haystack`, we return `-1`.

In other words, we are searching for a substring inside a larger string. The result should be the starting index of the first match.

For example, if:

```
haystack = "sadbutsad"
needle = "sad"
```

the substring `"sad"` appears twice, once starting at index `0` and again starting at index `6`. Since the problem asks for the first occurrence, the answer is `0`.

The input consists of two lowercase English strings. The constraints state that both strings have lengths between `1` and `10^4`. This means the strings are not extremely large, but they are large enough that inefficient algorithms could become slow if implemented poorly.

A naive approach that checks every possible starting position and compares characters one by one is acceptable here because the maximum input size is relatively small. However, understanding more advanced string matching ideas is still valuable because similar problems often require more optimized algorithms.

There are several important edge cases to consider:

- The `needle` may be longer than the `haystack`. In that case, a match is impossible, so we should immediately return `-1`.
- The `needle` may appear multiple times. We must return only the first occurrence.
- The strings may contain repeated characters, which can sometimes expose bugs in matching logic.
- The match may occur at the very beginning or very end of the `haystack`.
- The `needle` may not appear at all.

The problem guarantees that both strings are non-empty and contain only lowercase English letters, so we do not need to handle `null`, Unicode complications, or empty strings.

## Approaches

### Brute Force Approach

The most direct solution is to try every possible starting position in `haystack` and check whether `needle` matches character by character from that position.

Suppose the current starting index is `i`. We compare:

```
haystack[i + 0] with needle[0]
haystack[i + 1] with needle[1]
haystack[i + 2] with needle[2]
...
```

If every character matches, we return `i`. If a mismatch occurs, we move to the next starting position and repeat the process.

This approach is correct because every possible alignment of `needle` inside `haystack` is examined exactly once. If a valid occurrence exists, we will eventually find it.

The downside is that repeated comparisons can become expensive. In the worst case, many characters may match before a mismatch occurs, causing repeated work.

For example:

```
haystack = "aaaaaaaaaa"
needle   = "aaaab"
```

Each starting position compares several characters before failing.

Even though the worst case is quadratic, the given constraints are small enough that this approach still performs well in practice.

### Optimal Approach

A more advanced solution uses the Knuth-Morris-Pratt algorithm, commonly called KMP.

The key insight is that when a mismatch occurs, we should avoid rechecking characters that we already know match. Instead of restarting the comparison from scratch, KMP uses information about the pattern itself to determine how far we can safely skip ahead.

KMP preprocesses the `needle` and builds an array called the Longest Prefix Suffix array, usually abbreviated as LPS.

The LPS array tells us:

```
For each position, what is the length of the longest proper prefix
that is also a suffix?
```

This allows the algorithm to continue matching efficiently after a mismatch instead of repeating unnecessary comparisons.

The result is a linear time algorithm.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × m) | O(1) | Checks every possible starting position |
| Optimal, KMP | O(n + m) | O(m) | Uses preprocessing to avoid redundant comparisons |

Here, `n` is the length of `haystack`, and `m` is the length of `needle`.

## Algorithm Walkthrough

### Step 1: Handle Impossible Cases

If the length of `needle` is greater than the length of `haystack`, a match cannot exist. We immediately return `-1`.

### Step 2: Build the LPS Array

We preprocess the `needle` to create the LPS array.

The LPS value at index `i` represents the length of the longest proper prefix of `needle[0:i+1]` that is also a suffix.

For example:

```
needle = "ababaca"
LPS    = [0,0,1,2,3,0,1]
```

This preprocessing allows us to reuse previously matched information efficiently.

### Step 3: Start Matching

We use two pointers:

- `i` for the current character in `haystack`
- `j` for the current character in `needle`

We compare:

```
haystack[i] and needle[j]
```

### Step 4: Handle Matching Characters

If the characters match:

- Increment both pointers.
- Continue matching the next characters.

If `j` reaches the length of `needle`, the entire pattern has been matched. The starting index is:

```
i - len(needle)
```

### Step 5: Handle Mismatches

If a mismatch occurs after some characters already matched:

- We do not restart from the beginning.
- Instead, we use the LPS array to determine where to continue.

Specifically:

```
j = lps[j - 1]
```

This reuses previous matching information.

If `j` is already `0`, we simply move `i` forward.

### Step 6: Finish Search

If we reach the end of `haystack` without fully matching `needle`, we return `-1`.

## Python Solution

```
class Solution:
    def strStr(self, haystack: str, needle: str) -> int:
        if len(needle) > len(haystack):
            return -1

        # Build LPS array
        lps = [0] * len(needle)
        length = 0
        i = 1

        while i < len(needle):
            if needle[i] == needle[length]:
                length += 1
                lps[i] = length
                i += 1
            else:
                if length != 0:
                    length = lps[length - 1]
                else:
                    lps[i] = 0
                    i += 1

        # KMP search
        haystack_index = 0
        needle_index = 0

        while haystack_index < len(haystack):
            if haystack[haystack_index] == needle[needle_index]:
                haystack_index += 1
                needle_index += 1

                if needle_index == len(needle):
                    return haystack_index - needle_index
            else:
                if needle_index != 0:
                    needle_index = lps[needle_index - 1]
                else:
                    haystack_index += 1

        return -1
```

The implementation begins with a quick impossibility check. If `needle` is longer than `haystack`, no match can exist.

Next, the code builds the LPS array. The variable `length` stores the current length of the longest prefix-suffix match. The pointer `i` scans through the `needle`.

When characters match, the prefix length increases, and the new value is stored in `lps[i]`.

When a mismatch occurs, we do not restart from zero immediately. Instead, we use the previously computed LPS value to determine the next possible prefix length.

After preprocessing, the actual KMP search begins.

The variables `haystack_index` and `needle_index` track positions in the two strings.

If characters match, both pointers move forward.

If all characters in `needle` match successfully, we return the starting position of the match.

When a mismatch occurs, the LPS array determines how far the `needle_index` should fall back. This prevents unnecessary repeated comparisons.

If the search completes without a full match, the function returns `-1`.

## Go Solution

```
func strStr(haystack string, needle string) int {
	if len(needle) > len(haystack) {
		return -1
	}

	// Build LPS array
	lps := make([]int, len(needle))

	length := 0
	i := 1

	for i < len(needle) {
		if needle[i] == needle[length] {
			length++
			lps[i] = length
			i++
		} else {
			if length != 0 {
				length = lps[length-1]
			} else {
				lps[i] = 0
				i++
			}
		}
	}

	// KMP search
	haystackIndex := 0
	needleIndex := 0

	for haystackIndex < len(haystack) {
		if haystack[haystackIndex] == needle[needleIndex] {
			haystackIndex++
			needleIndex++

			if needleIndex == len(needle) {
				return haystackIndex - needleIndex
			}
		} else {
			if needleIndex != 0 {
				needleIndex = lps[needleIndex-1]
			} else {
				haystackIndex++
			}
		}
	}

	return -1
}
```

The Go implementation follows the same logic as the Python version.

One important difference is that Go strings are indexed as byte sequences. Since the problem guarantees lowercase English letters only, byte indexing is completely safe here.

The LPS array is created using `make([]int, len(needle))`.

Go does not have Python-style dynamic lists or tuple unpacking, so the code updates variables explicitly.

There are no integer overflow concerns because the maximum string length is only `10^4`.

The implementation also avoids slice boundary issues by carefully checking loop conditions before accessing indices.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n + m) | Each character is processed at most a constant number of times |
| Space | O(m) | The LPS array stores one value per character in `needle` |

The preprocessing phase scans the `needle` once, which takes `O(m)` time. The search phase scans the `haystack` once, which takes `O(n)` time.

The important detail is that the pointers never move backward unnecessarily. The LPS array allows the algorithm to skip repeated comparisons, ensuring linear performance overall.

The only extra memory used is the LPS array, whose size equals the length of `needle`.

## Edge Cases

### Needle Longer Than Haystack

Consider:

```
haystack = "abc"
needle = "abcd"
```

A match is impossible because the substring we are searching for is larger than the entire string.

A naive implementation might still attempt comparisons and accidentally access invalid indices. This solution avoids that by immediately returning `-1` before any processing begins.

### Multiple Occurrences

Consider:

```
haystack = "aaaaa"
needle = "aa"
```

The substring `"aa"` appears multiple times, at indices `0`, `1`, `2`, and `3`.

The problem specifically asks for the first occurrence. Since the algorithm scans from left to right and returns immediately after the first complete match, it correctly returns `0`.

### Repeated Characters Causing Backtracking

Consider:

```
haystack = "aaaab"
needle = "aaab"
```

This kind of input is important because repeated characters can cause inefficient repeated comparisons in naive solutions.

KMP handles this efficiently using the LPS array. Instead of restarting completely after a mismatch, it reuses previous matching information and skips unnecessary work.

### Match at the End of the String

Consider:

```
haystack = "xyzabc"
needle = "abc"
```

Some implementations incorrectly stop searching too early.

This algorithm continues until the end of `haystack`, so it correctly detects matches that begin near the final characters.

### No Match Exists

Consider:

```
haystack = "abcdef"
needle = "gh"
```

The algorithm scans the entire string and never finds a complete match, so it correctly returns `-1`.
