## Solution

## Problem Understanding

The problem asks us to find the length of the longest contiguous subarray within a given array of integers where the product of all the elements in the subarray is positive. A subarray is a sequence of consecutive elements, and a positive product means the product is strictly greater than zero.

The input is an array `nums` of integers that can range from negative to positive numbers, including zero. The constraints tell us that the length of the array can be up to `10^5`, and individual numbers can be as large as `10^9` in magnitude. This means we need a solution that runs in linear time `O(n)` because a brute-force approach that checks all subarrays would be too slow.

Key observations include: zeros break subarrays because multiplying by zero makes the product zero, which is not positive. Negative numbers flip the sign of the product, so we need to carefully track the parity of the negative numbers. Edge cases include arrays with all negative numbers, arrays with all zeros, or arrays where the positive product subarray is at the edges.

## Approaches

The brute-force approach would iterate over all possible subarrays, compute their products, and track the maximum length of subarrays with positive products. This works because it explicitly checks each subarray, but it is too slow because it requires `O(n^2)` time for computing all subarray products and lengths.

The optimal approach relies on a key insight: we do not need to compute the actual product. Instead, we only need to track the count of negative numbers and manage the length of subarrays with positive and negative products. Using a greedy dynamic programming approach, we can maintain two values at each position: the length of the longest subarray ending at that position with a positive product, and the length with a negative product. If we see a zero, we reset both lengths because zero breaks the subarray. For positive numbers, the positive length increases and negative length also increases if there was already a negative subarray. For negative numbers, positive and negative lengths swap because the sign flips.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check all subarrays and compute product explicitly |
| Optimal | O(n) | O(1) | Track lengths of positive and negative product subarrays dynamically |

## Algorithm Walkthrough

1. Initialize two variables `pos_len` and `neg_len` to 0. `pos_len` stores the length of the longest subarray ending at the current index with a positive product. `neg_len` stores the length with a negative product.
2. Initialize `max_len` to 0 to keep track of the maximum positive product subarray length encountered.
3. Iterate through the array `nums` element by element.
4. If the current number is zero, reset `pos_len` and `neg_len` to 0 because zero breaks the subarray.
5. If the current number is positive, increment `pos_len` by 1 (extending the positive subarray), and increment `neg_len` by 1 if `neg_len` is already non-zero (extending a negative subarray). If `neg_len` was zero, it remains zero.
6. If the current number is negative, swap `pos_len` and `neg_len` because multiplying by a negative number flips the sign. Then increment `neg_len` by 1 (new negative subarray length) if it was zero, or increment `pos_len` by 1 if `neg_len` was non-zero.
7. Update `max_len` with the maximum of itself and `pos_len`.
8. Continue until the end of the array, and return `max_len`.

Why it works: This algorithm works because at every step, it keeps track of the lengths of subarrays that could potentially contribute to a positive product. Swapping the lengths on a negative number correctly models the change in sign of the product. Resetting on zero ensures subarrays are contiguous and valid.

## Python Solution

```python
from typing import List

class Solution:
    def getMaxLen(self, nums: List[int]) -> int:
        pos_len = 0
        neg_len = 0
        max_len = 0
        
        for num in nums:
            if num == 0:
                pos_len = 0
                neg_len = 0
            elif num > 0:
                pos_len += 1
                neg_len = neg_len + 1 if neg_len > 0 else 0
            else:
                pos_len, neg_len = (neg_len + 1 if neg_len > 0 else 0, pos_len + 1)
            
            max_len = max(max_len, pos_len)
        
        return max_len
```

The Python implementation follows the algorithm directly. We iterate over each number and update `pos_len` and `neg_len` according to the sign. On zeros, we reset lengths. On positives, we extend positive length and optionally extend negative length. On negatives, we swap lengths to reflect the sign change. The maximum positive length is tracked in `max_len`.

## Go Solution

```go
func getMaxLen(nums []int) int {
    posLen, negLen, maxLen := 0, 0, 0
    
    for _, num := range nums {
        if num == 0 {
            posLen, negLen = 0, 0
        } else if num > 0 {
            posLen++
            if negLen > 0 {
                negLen++
            }
        } else {
            newPos := 0
            if negLen > 0 {
                newPos = negLen + 1
            }
            negLen = posLen + 1
            posLen = newPos
        }
        if posLen > maxLen {
            maxLen = posLen
        }
    }
    
    return maxLen
}
```

The Go solution mirrors the Python approach. Slices are used to iterate, and variables are updated similarly. Go does not have `None`, so `0` is used to reset lengths. Integer overflow is not an issue since we are only counting lengths.
