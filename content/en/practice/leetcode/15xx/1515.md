## Solution

## Problem Understanding

The problem is asking us to determine the optimal location for a service center on a 2D plane such that the sum of Euclidean distances from this center to all given customer locations is minimized. Each customer location is represented as a coordinate `[xi, yi]` in the `positions` array. The output is the minimum sum of distances achievable by placing the service center at an ideal point `[xcentre, ycentre]`.

The constraints are small, with up to 50 customers and coordinates between 0 and 100. This suggests that while exhaustive methods might be theoretically possible, they may be inefficient due to the continuous nature of Euclidean distances. The problem guarantees that at least one customer exists, and coordinates are non-negative integers, simplifying bounds for our optimization.

Important edge cases include when all customers are at the same location, where the minimum distance is zero, or when there are only two customers, where the optimal location lies on the line segment connecting them. Naive approaches may struggle with the continuous optimization aspect because the exact point minimizing the sum is not guaranteed to be at an integer coordinate.

## Approaches

### Brute Force

A brute-force approach would discretize the plane into a fine grid (e.g., using steps of 0.01 or smaller) and evaluate the sum of distances to all customers for each candidate point. While this guarantees correctness, it is extremely slow because the plane is continuous and even a small step size results in millions of candidate points. For 50 customers, the repeated distance calculations make it impractical.

### Optimal Approach - Gradient Descent / Weiszfeld’s Algorithm

The key insight is that this is a classic **geometric median problem**, where the optimal point minimizes the sum of Euclidean distances. This can be solved efficiently using **Weiszfeld’s algorithm**, an iterative method similar to gradient descent. Starting from an initial guess (such as the centroid of all points), the algorithm updates the candidate position using a weighted average where weights are inversely proportional to the distance to each customer. Repeating this process converges to the geometric median with high precision.

We choose this approach because it directly targets the continuous optimization problem and converges rapidly for small inputs like this problem.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(grid_size^2 * n) | O(1) | Discretizes plane, evaluates all points, too slow |
| Optimal (Weiszfeld) | O(n * iterations) | O(1) | Iteratively updates candidate point using distances, converges quickly |

## Algorithm Walkthrough

1. **Initialization**: Compute the centroid of all customer positions as an initial guess for the service center. This is reasonable because the geometric median is often near the centroid.
2. **Iterative Update**: For each iteration, calculate the weighted average of all customer positions. The weight for a customer is `1 / distance(current_point, customer)`. Update the current candidate point to this new weighted average.
3. **Convergence Check**: Compute the Euclidean distance between the new candidate point and the previous one. If this change is smaller than a very small threshold (e.g., `1e-6`), stop the iteration as convergence is achieved.
4. **Distance Summation**: Once converged, compute the total sum of distances from the final point to all customers.

**Why it works**: Weiszfeld’s algorithm converges to a point that satisfies the geometric median condition: for any small perturbation, the sum of distances cannot be decreased. Each update moves the point closer to minimizing the weighted sum of distances.

## Python Solution

```python
from typing import List
import math

class Solution:
    def getMinDistSum(self, positions: List[List[int]]) -> float:
        # Start from the centroid of all points
        x = sum(pos[0] for pos in positions) / len(positions)
        y = sum(pos[1] for pos in positions) / len(positions)
        
        epsilon = 1e-6  # convergence threshold
        max_iter = 1000  # safeguard against infinite loops
        
        for _ in range(max_iter):
            num_x = num_y = denom = 0
            for px, py in positions:
                dist = math.hypot(px - x, py - y)
                if dist < 1e-10:
                    continue  # avoid division by zero
                weight = 1 / dist
                num_x += px * weight
                num_y += py * weight
                denom += weight
            new_x, new_y = num_x / denom, num_y / denom
            if abs(new_x - x) < epsilon and abs(new_y - y) < epsilon:
                break
            x, y = new_x, new_y
        
        # Compute total distance to all points
        return sum(math.hypot(px - x, py - y) for px, py in positions)
```

The Python solution starts from the centroid and iteratively updates the candidate point using Weiszfeld’s formula. It skips points extremely close to the candidate to avoid division by zero. Once convergence is detected, it sums distances from the final location to all customers.

## Go Solution

```go
import (
    "math"
)

func getMinDistSum(positions [][]int) float64 {
    n := len(positions)
    x, y := 0.0, 0.0
    for _, pos := range positions {
        x += float64(pos[0])
        y += float64(pos[1])
    }
    x /= float64(n)
    y /= float64(n)
    
    epsilon := 1e-6
    maxIter := 1000
    
    for i := 0; i < maxIter; i++ {
        numX, numY, denom := 0.0, 0.0, 0.0
        for _, pos := range positions {
            px, py := float64(pos[0]), float64(pos[1])
            dist := math.Hypot(px - x, py - y)
            if dist < 1e-10 {
                continue
            }
            weight := 1 / dist
            numX += px * weight
            numY += py * weight
            denom += weight
        }
        newX, newY := numX/denom, numY/denom
        if math.Abs(newX - x) < epsilon && math.Abs(newY - y) < epsilon {
            break
        }
        x, y = newX, newY
    }
    
    totalDist := 0.0
    for _, pos := range positions {
        totalDist += math.Hypot(float64(pos[0]) - x, float64(pos[1]) - y)
    }
    return totalDist
}
```

The Go implementation mirrors the Python version. Floating-point precision and division by zero are handled similarly. The centroid is computed using `float64` arithmetic to avoid integer truncation.
