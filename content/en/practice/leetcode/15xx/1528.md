## Solution

## Problem Understanding

In this problem, we are given two inputs:

- A string `s`
- An integer array `indices`

Both have the same length, and each value in `indices` represents the new position where the corresponding character from `s` should be placed.

More specifically, the character currently located at position `i` in the original string should move to position `indices[i]` in the final shuffled string.

For example, if:

```
s = "abc"
indices = [2, 1, 0]
```

then:

- `'a'` from index `0` moves to index `2`
- `'b'` from index `1` stays at index `1`
- `'c'` from index `2` moves to index `0`

The final result becomes `"cba"`.

The task is to reconstruct and return the shuffled string after all characters are moved to their target positions.

The constraints are very small:

- `1 <= n <= 100`
- All indices are unique
- Every index is valid, meaning it lies in the range `[0, n - 1]`

These guarantees are important because they tell us:

- Every position in the output string will be filled exactly once
- We never need to handle duplicate destinations
- We never need bounds checking for invalid positions
- A simple direct placement strategy is sufficient

An important detail is that strings are immutable in many languages, including Python. That means we cannot directly modify characters in a string. Instead, we typically build a mutable structure such as a list or array and then convert it back into a string.

Some edge cases that could cause bugs in weaker implementations include:

- A string of length `1`
- Characters that all remain in their original positions
- A complete reversal of the string
- Arbitrary permutations where characters move far from their original positions

The problem guarantees that the permutation is always valid, so we do not need to handle malformed input.

## Approaches

### Brute Force Approach

A straightforward brute-force solution is to repeatedly search for which character belongs at each position in the final string.

For every target position from `0` to `n - 1`, we scan through the `indices` array to find which original index maps to that position. Once found, we append the corresponding character from `s`.

Conceptually, this works because every destination index appears exactly once.

For example:

```
indices = [4,5,6,7,0,2,1,3]
```

To determine the character for final position `0`, we scan the entire array until we find the value `0`. That occurs at index `4`, so we place `s[4]`.

This process is repeated for every output position.

The method is correct, but inefficient because it performs a nested scan:

- Outer loop over all positions
- Inner loop searching the indices array

This results in quadratic time complexity.

### Optimal Approach

The key observation is that the problem directly tells us where every character should go.

Instead of searching for positions repeatedly, we can immediately place each character into its destination index.

We create an output array of size `n`, then iterate through the string once:

```
result[indices[i]] = s[i]
```

After all characters are placed, we join the array into the final string.

This works efficiently because:

- Every target index is unique
- Every position is filled exactly once
- We only need one traversal of the input

This reduces the time complexity from quadratic to linear.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly scans the indices array to find each destination character |
| Optimal | O(n) | O(n) | Places each character directly into its final position |

## Algorithm Walkthrough

1. Determine the length of the string.

We need an output structure capable of storing characters at arbitrary positions. Since strings are immutable, we use a mutable array or list.
2. Create a result array of size `n`.

Initially, this array can contain empty placeholders. Each position will later receive exactly one character.
3. Iterate through the original string and indices array simultaneously.

For every position `i`:

- The current character is `s[i]`
- Its target position is `indices[i]`
4. Place the character into its destination.

We assign:

```
result[indices[i]] = s[i]
```

This directly reconstructs the shuffled string.
5. Convert the result array back into a string.

After all characters are placed, joining the array produces the final answer.

### Why it works

The algorithm works because the `indices` array defines a valid permutation of positions from `0` to `n - 1`.

Since all values are unique:

- No two characters compete for the same destination
- Every output position is filled exactly once

By placing each character directly into its target location, the algorithm reconstructs the exact shuffled string described in the problem statement.

## Python Solution

```python
from typing import List

class Solution:
    def restoreString(self, s: str, indices: List[int]) -> str:
        n = len(s)
        
        result = [""] * n
        
        for i in range(n):
            result[indices[i]] = s[i]
        
        return "".join(result)
```

The implementation begins by determining the length of the input string. This length is used to create a result list with the same number of positions.

The line:

```python
result = [""] * n
```

creates a mutable list that can store characters at arbitrary indices.

Next, the algorithm iterates through every character in the original string. For each index `i`, it uses `indices[i]` as the destination position and places `s[i]` directly into that slot.

Finally, because the result is stored as a list of characters, `"".join(result)` converts it back into a valid string.

The implementation exactly matches the optimal algorithm described earlier and performs only one pass through the input.

## Go Solution

```go
func restoreString(s string, indices []int) string {
	n := len(s)

	result := make([]byte, n)

	for i := 0; i < n; i++ {
		result[indices[i]] = s[i]
	}

	return string(result)
}
```

The Go implementation follows the same logic as the Python version, but there are some language-specific details worth noting.

In Go, strings are immutable, so we use a `[]byte` slice to build the output efficiently.

The statement:

```go
result := make([]byte, n)
```

creates a byte slice of the required size.

Each character is placed directly into its destination index using:

```go
result[indices[i]] = s[i]
```

Finally, the byte slice is converted back into a string using:

```go
string(result)
```

Since the problem guarantees lowercase English letters, working with bytes is completely safe here.

## Edge Cases

### Single Character String

A string of length `1` is the smallest possible valid input. A buggy implementation might incorrectly assume multiple positions exist or mishandle initialization logic.

For example:

```
s = "a"
indices = [0]
```

The implementation handles this naturally because the result array is created with size `1`, and the single character is placed directly into position `0`.

### Characters Already in Correct Positions

An identity permutation means every character stays where it already is.

Example:

```
s = "abc"
indices = [0,1,2]
```

Some implementations unnecessarily swap characters or attempt in-place mutations that can introduce errors. The direct-placement approach avoids this completely because assigning each character to its own index simply recreates the original string.

### Duplicate Characters

Even though positions are unique, characters themselves may repeat.

Example:

```
s = "aaaa"
indices = [2,0,3,1]
```

A flawed solution might incorrectly rely on character uniqueness when reconstructing the string. Our implementation never depends on character values. It only uses index mappings, so repeated characters are handled correctly without ambiguity.
