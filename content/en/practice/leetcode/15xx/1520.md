## Solution

## Problem Understanding

The problem asks us to select as many non-overlapping substrings as possible from a given lowercase string `s`, while satisfying a strict containment rule.

For every substring we choose, if that substring contains a character `c`, then it must contain every occurrence of `c` that appears anywhere in the original string. In other words, we are not allowed to partially include occurrences of a character. Either all occurrences of a character belong to the substring, or none of them do.

The substrings must also be non-overlapping. If one substring occupies indices `[i, j]`, then another substring cannot share any index inside that range.

Among all valid solutions, we first maximize the number of substrings. If multiple solutions contain the same number of substrings, we choose the one with the minimum total length. The problem guarantees that this minimum-length optimal solution is unique.

The input is a single string `s` consisting only of lowercase English letters. The string length can be as large as `10^5`, which immediately tells us that quadratic or cubic brute-force enumeration approaches will not be efficient enough.

The key challenge is that choosing a substring affects which characters must also be included. A substring may need to expand repeatedly because newly included characters introduce additional occurrences outside the current range.

Several edge cases are important:

- A character that appears only once can potentially form a single-character substring.
- Some characters force very large intervals because their occurrences are spread across the string.
- Different valid intervals may overlap heavily, so greedy interval selection becomes important.
- A valid substring may expand multiple times before stabilizing.
- Some candidate intervals become invalid because expanding them requires including characters whose first occurrence lies before the current interval start.

The problem guarantees lowercase English letters only, which means there are at most 26 distinct characters. This observation is extremely important for designing an efficient solution.

## Approaches

### Brute Force Approach

A brute-force solution would enumerate every possible substring and test whether it satisfies the containment rule.

For each substring `s[i:j]`, we could:

1. Collect all characters inside the substring.
2. For every character inside it, verify that all occurrences of that character in the original string also lie inside `[i, j]`.
3. Store all valid substrings.
4. Try every combination of non-overlapping valid substrings to find the maximum count and minimum total length.

This approach is correct because it explicitly checks every possible candidate and every possible selection.

However, the complexity becomes enormous:

- There are `O(n^2)` substrings.
- Each substring validation may cost `O(n)`.
- Enumerating combinations of non-overlapping substrings becomes exponential.

This is far too slow for `n = 10^5`.

### Key Insight

The critical observation is that a valid substring is completely determined by character occurrence ranges.

For each character:

- `first[c]` = first occurrence index
- `last[c]` = last occurrence index

If a substring contains character `c`, then it must cover the entire interval `[first[c], last[c]]`.

This means every valid substring must eventually expand to include the complete ranges of all characters appearing inside it.

Another important insight is that only intervals starting at a character's first occurrence can become minimal valid substrings. Since there are only 26 lowercase letters, there are at most 26 candidate intervals.

Once we generate all minimal valid intervals, the problem becomes a classic interval scheduling problem:

- Select the maximum number of non-overlapping intervals.
- Among equal counts, prefer smaller total length.

This is solved greedily by sorting by ending position.

## Algorithm Walkthrough

1. Compute the first and last occurrence of every character.

We scan the string once and record:

- the earliest index where each character appears
- the latest index where each character appears

Since there are only 26 lowercase letters, these arrays remain very small.
2. Generate candidate minimal valid intervals.

We only attempt to build intervals starting from indices that are the first occurrence of some character.

Suppose we start from character `c` at index `start`.

Initially:

- `end = last[c]`

Then we scan from `start` to `end`.

For every character encountered:

- If its first occurrence is before `start`, then this interval can never become valid.
- Otherwise, we extend:

`end = max(end, last[current_character])`

This process continues until the scan reaches the final stabilized `end`.
3. Store every valid minimal interval.

Each valid interval represents the smallest substring that fully contains all occurrences of its included characters.
4. Greedily select non-overlapping intervals.

We sort candidate intervals by ending position.

Then we iterate through them:

- If the current interval does not overlap the previously selected one, we take it.
- If it overlaps but ends earlier than the previously chosen interval, we replace the previous one.

In practice, sorting by end position naturally produces the optimal greedy ordering.
5. Return the selected substrings.

For every chosen interval `[l, r]`, we return `s[l:r+1]`.

### Why it works

Every valid substring must fully contain the occurrence range of every character inside it. The interval expansion process constructs the smallest interval satisfying this condition for a given starting character.

Among all valid intervals, choosing intervals with the smallest ending positions greedily maximizes the number of non-overlapping intervals. When two solutions contain the same number of substrings, the greedy choice also minimizes total length because earlier-ending intervals are never longer than overlapping later-ending alternatives.

## Python Solution

```python
from typing import List

class Solution:
    def maxNumOfSubstrings(self, s: str) -> List[str]:
        n = len(s)

        first = [n] * 26
        last = [-1] * 26

        for i, ch in enumerate(s):
            idx = ord(ch) - ord('a')
            first[idx] = min(first[idx], i)
            last[idx] = i

        intervals = []

        for i, ch in enumerate(s):
            idx = ord(ch) - ord('a')

            if i != first[idx]:
                continue

            end = last[idx]
            j = i
            valid = True

            while j <= end:
                current_idx = ord(s[j]) - ord('a')

                if first[current_idx] < i:
                    valid = False
                    break

                end = max(end, last[current_idx])
                j += 1

            if valid:
                intervals.append((i, end))

        intervals.sort(key=lambda x: x[1])

        result = []
        prev_end = -1

        for start, end in intervals:
            if start > prev_end:
                result.append(s[start:end + 1])
                prev_end = end

        return result
```

The implementation begins by computing the first and last occurrence indices for each character. These arrays allow us to quickly determine the mandatory coverage range for any character.

Next, the algorithm attempts to build a valid interval starting from every character's first occurrence. This is essential because a minimal valid substring cannot begin in the middle of a character's occurrence range.

The interval expansion loop repeatedly extends the right boundary whenever a newly encountered character has occurrences further to the right. If a character inside the interval has its first occurrence before the interval start, the candidate becomes invalid immediately.

After collecting all valid intervals, the code sorts them by ending position. This prepares the intervals for greedy interval scheduling.

Finally, the algorithm selects non-overlapping intervals greedily. Every accepted interval contributes one substring to the optimal solution.

## Go Solution

```go
package main

import "sort"

func maxNumOfSubstrings(s string) []string {
	n := len(s)

	first := make([]int, 26)
	last := make([]int, 26)

	for i := 0; i < 26; i++ {
		first[i] = n
		last[i] = -1
	}

	for i := 0; i < n; i++ {
		idx := int(s[i] - 'a')

		if i < first[idx] {
			first[idx] = i
		}

		last[idx] = i
	}

	type Interval struct {
		start int
		end   int
	}

	intervals := []Interval{}

	for i := 0; i < n; i++ {
		idx := int(s[i] - 'a')

		if i != first[idx] {
			continue
		}

		end := last[idx]
		j := i
		valid := true

		for j <= end {
			currentIdx := int(s[j] - 'a')

			if first[currentIdx] < i {
				valid = false
				break
			}

			if last[currentIdx] > end {
				end = last[currentIdx]
			}

			j++
		}

		if valid {
			intervals = append(intervals, Interval{i, end})
		}
	}

	sort.Slice(intervals, func(i, j int) bool {
		return intervals[i].end < intervals[j].end
	})

	result := []string{}
	prevEnd := -1

	for _, interval := range intervals {
		if interval.start > prevEnd {
			result = append(result, s[interval.start:interval.end+1])
			prevEnd = interval.end
		}
	}

	return result
}
```

The Go implementation follows the same algorithmic structure as the Python version. Arrays are used for first and last occurrence tracking because the alphabet size is fixed at 26.

Go slices are used to store intervals and the final substrings. Since Go strings support slicing by byte indices for ASCII characters, substring extraction remains straightforward and efficient.

There are no integer overflow concerns because indices never exceed `10^5`.

## Edge Cases

One important edge case occurs when all characters are identical, such as `"aaaaa"`. A naive implementation might incorrectly try to split this into smaller substrings, but every substring containing `'a'` must include all occurrences of `'a'`. The implementation handles this correctly because the interval for `'a'` expands to the entire string immediately.

Another tricky case appears when intervals are nested, such as `"abac"`. The interval for `'a'` becomes `[0,2]`, but the interval for `'b'` is `[1,1]`. Choosing `"aba"` would reduce the total number of substrings. The greedy interval scheduling step correctly prefers the smaller interval `"b"` because it ends earlier and allows more substrings overall.

A third important edge case involves invalid interval expansion. Consider `"adefadda"`. Starting from `'d'` initially gives interval `[1,6]`, but scanning inside it encounters `'a'`, whose first occurrence is at index 0. This means the interval cannot remain valid unless it expands leftward, which is impossible because the start is fixed. The implementation detects this condition with:

```
if first[current_idx] < start:
```

and discards the interval immediately.

Another subtle case occurs when many characters force repeated interval expansion. In strings with chained dependencies, the interval boundary may grow multiple times while scanning. The implementation continuously updates:

```
end = max(end, last[current_idx])
```

which guarantees that all required character occurrences are eventually included before validation completes.
