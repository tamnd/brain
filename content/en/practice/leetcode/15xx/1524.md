## Solution

## Problem Understanding

The problem gives an integer array `arr` and asks us to count how many contiguous subarrays have an odd sum.

A subarray is a continuous section of the array. For example, if the array is `[1,2,3]`, the subarrays are:

- `[1]`
- `[1,2]`
- `[1,2,3]`
- `[2]`
- `[2,3]`
- `[3]`

For each subarray, we calculate the sum of its elements. If that sum is odd, it contributes to the final answer.

The result can become very large because an array of length `n` contains `n * (n + 1) / 2` total subarrays. Since `n` can be as large as `10^5`, the total number of subarrays can exceed standard integer limits in some languages. Because of this, the problem asks us to return the answer modulo `10^9 + 7`.

The constraints tell us several important things:

- `arr.length <= 10^5`, so any solution worse than approximately `O(n log n)` will likely time out.
- `arr[i] <= 100`, meaning values are small, but that does not help enough for brute force because the number of subarrays is still enormous.
- We need an algorithm that processes the array in one pass or close to it.

An important observation is that whether a sum is odd or even depends only on parity, not on the exact value. This strongly suggests using prefix sums and parity tracking.

There are several edge cases that can cause mistakes in naive implementations:

- Arrays containing only even numbers should return `0`.
- Arrays containing only odd numbers generate many odd sums.
- A single-element array must still be handled correctly.
- Very large arrays require an efficient approach because checking every subarray individually is too slow.
- Since the answer can exceed integer range, modulo arithmetic must be applied carefully.

## Approaches

### Brute Force Approach

The most direct approach is to generate every possible subarray and compute its sum.

For every starting index `i`, we extend the subarray one element at a time toward the right. While extending, we maintain the running sum. Whenever the running sum is odd, we increment the answer.

This approach is correct because it explicitly examines every contiguous subarray exactly once and checks whether its sum is odd.

However, the time complexity is too large. An array of length `n` contains `O(n^2)` subarrays. With `n = 10^5`, this becomes approximately `10^10` operations, which is far beyond acceptable limits.

### Optimal Prefix Sum Parity Approach

The key insight is that we do not actually care about the exact prefix sums. We only care whether they are odd or even.

Suppose:

- `prefixSum[i]` is the sum of elements from index `0` to `i`
- The sum of subarray `(l ... r)` is:

$$prefixSum[r] - prefixSum[l-1]$$

A subtraction results in an odd number only when the two values have different parity:

- even - odd = odd
- odd - even = odd

Therefore:

- If the current prefix sum is even, then any previous odd prefix sum creates an odd subarray.
- If the current prefix sum is odd, then any previous even prefix sum creates an odd subarray.

Instead of storing all prefix sums, we only store:

- how many even prefix sums we have seen
- how many odd prefix sums we have seen

This reduces the problem to a simple linear scan.

## Algorithm Walkthrough

1. Initialize counters for prefix sum parity.

We keep:

- `even_count = 1`
- `odd_count = 0`

We start with one even prefix sum because a prefix sum of `0` exists before processing any element.
2. Initialize variables.

We maintain:

- `prefix_sum = 0`
- `result = 0`
3. Iterate through the array.

For each number:

- Add it to `prefix_sum`
- Determine whether the new prefix sum is odd or even
4. Handle the even prefix sum case.

If the current prefix sum is even:

- Any previous odd prefix sum forms an odd subarray
- Add `odd_count` to the result
- Increment `even_count`
5. Handle the odd prefix sum case.

If the current prefix sum is odd:

- Any previous even prefix sum forms an odd subarray
- Add `even_count` to the result
- Increment `odd_count`
6. Apply modulo arithmetic.

Since the answer may become large, take modulo `10^9 + 7` after updates.
7. Return the final result.

### Why it works

The algorithm relies on the parity relationship between prefix sums. A subarray sum is odd exactly when the two surrounding prefix sums have opposite parity. By counting how many even and odd prefix sums have appeared so far, we can immediately determine how many odd-sum subarrays end at the current index. Every valid subarray is counted exactly once, so the result is correct.

## Python Solution

```python
from typing import List

class Solution:
    def numOfSubarrays(self, arr: List[int]) -> int:
        MOD = 10**9 + 7
        
        even_count = 1
        odd_count = 0
        
        prefix_sum = 0
        result = 0
        
        for num in arr:
            prefix_sum += num
            
            if prefix_sum % 2 == 0:
                result += odd_count
                even_count += 1
            else:
                result += even_count
                odd_count += 1
            
            result %= MOD
        
        return result
```

The implementation follows the algorithm directly.

We first initialize counters for even and odd prefix sums. The value `even_count = 1` represents the empty prefix sum before processing any elements.

As we iterate through the array, we continuously update the running prefix sum. At each step, we determine whether the prefix sum is odd or even.

If the prefix sum is even, then subtracting any previous odd prefix sum produces an odd result. Therefore, we add `odd_count` to the answer.

If the prefix sum is odd, then subtracting any previous even prefix sum produces an odd result. Therefore, we add `even_count` to the answer.

After processing the current element, we update the corresponding parity counter so future subarrays can use it.

Modulo arithmetic is applied after every update to ensure the result remains within limits.

## Go Solution

```go
func numOfSubarrays(arr []int) int {
    const MOD int = 1e9 + 7

    evenCount := 1
    oddCount := 0

    prefixSum := 0
    result := 0

    for _, num := range arr {
        prefixSum += num

        if prefixSum%2 == 0 {
            result += oddCount
            evenCount++
        } else {
            result += evenCount
            oddCount++
        }

        result %= MOD
    }

    return result
}
```

The Go implementation is nearly identical to the Python version.

Go uses explicit integer declarations, so we define `MOD` as an integer constant. The loop iterates through the slice using `range`.

Since Go integers are large enough for intermediate computations in this problem, no special overflow handling is needed beyond applying modulo arithmetic.

An empty slice is handled naturally because the loop simply does not execute, although the constraints guarantee at least one element.

## Edge Cases

### All Elements Are Even

An array like `[2,4,6,8]` never produces an odd sum because adding even numbers always results in an even number. A buggy implementation might accidentally count some subarrays due to incorrect prefix handling.

This implementation handles the case correctly because the prefix sum always remains even, meaning only `even_count` increases and no odd subarrays are added to the result.

### Single Element Arrays

Arrays of length `1` are important because they test whether the initialization logic is correct.

For `[1]`, the answer should be `1`.

For `[2]`, the answer should be `0`.

The initialization `even_count = 1` correctly represents the empty prefix sum before processing begins, which allows single odd elements to be counted properly.

### Large Arrays With Many Odd Numbers

Arrays containing many odd values generate a large number of odd subarrays. A brute force solution would become extremely slow.

For example:

```
[1,1,1,1,1,...]
```

The optimal solution still runs in linear time because it only tracks parity counts instead of enumerating every subarray.

Modulo arithmetic also ensures the result never exceeds allowed integer bounds.
