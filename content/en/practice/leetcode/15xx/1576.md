# LeetCode 1576 - Replace All ?'s to Avoid Consecutive Repeating Characters

## Problem Understanding

This problem asks us to replace every `'?'` character in a string with a lowercase English letter such that the final string contains no consecutive repeating characters. The challenge is that we are only allowed to modify the `'?'` characters, while all existing lowercase letters must remain unchanged.

In other words, we are given a partially completed string where some positions are unknown, represented by `'?'`. Our task is to fill in those unknown positions so that no two adjacent characters are identical. Since the problem guarantees that the original string contains no consecutive repeating characters except those involving `'?'`, we know that the existing fixed characters already satisfy the condition.

The input is a string `s` consisting only of lowercase English letters and `'?'`. The output should be a valid string where every `'?'` has been replaced with a lowercase letter while ensuring no adjacent characters are equal.

The constraints are relatively small:

- The string length is between `1` and `100`
- Only lowercase English letters and `'?'` appear in the input

These constraints tell us that performance is not a major issue. Even an inefficient solution would likely pass. However, we still want to design a clean and optimal algorithm.

Several edge cases are important to consider upfront. A string may consist entirely of `'?'`, such as `"???"`, which means we must carefully avoid creating repeated neighbors while filling characters. Another case is when `'?'` appears at the beginning or end of the string, meaning one side may not have a neighboring character. Consecutive `'?'` characters are also important because replacing one character affects the valid choices for the next one. Fortunately, the problem guarantees that a valid answer always exists.

## Approaches

### Brute Force Approach

A brute force solution would try all possible replacements for every `'?'`. Since each `'?'` can be replaced by any of the `26` lowercase English letters, we could generate every possible combination and check whether the resulting string contains consecutive repeating characters.

For example, if there are `k` question marks, we would explore up to `26^k` possible strings. After generating each candidate string, we would scan it to verify whether adjacent characters differ.

This approach is guaranteed to find a correct answer because it exhaustively checks every possible replacement. However, it becomes inefficient as the number of `'?'` grows. Even though the input size is small, exponential growth makes this method unnecessarily expensive.

### Optimal Greedy Approach

The key observation is that each replacement decision only depends on the neighboring characters. Since we only care about avoiding consecutive duplicates, a character at index `i` only needs to differ from:

- The character at `i - 1`
- The character at `i + 1`

This means we can process the string from left to right and greedily choose the first valid letter for every `'?'`.

Because only adjacent characters matter, we never need to reconsider earlier decisions. At each `'?'`, we simply test letters from `'a'` to `'z'` and choose the first one that does not match its left or right neighbor.

Since there are only `26` possible letters, this process is extremely efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(26^k × n) | O(26^k) | Tries every replacement combination for `k` question marks |
| Optimal Greedy | O(26 × n) | O(n) | Replaces each `'?'` by checking neighboring characters |

Here, `n` is the length of the string and `k` is the number of `'?'` characters.

## Algorithm Walkthrough

### Optimal Greedy Algorithm

1. Convert the string into a mutable structure, such as a list of characters, because strings are immutable in Python.
2. Iterate through the string from left to right using an index `i`.
3. For each position:

- If the character is not `'?'`, leave it unchanged and move to the next index.
- If the character is `'?'`, determine the neighboring characters.
4. Retrieve the left neighbor:

- If `i > 0`, use `characters[i - 1]`
- Otherwise, there is no left neighbor
5. Retrieve the right neighbor:

- If `i < n - 1`, use `characters[i + 1]`
- Otherwise, there is no right neighbor
6. Try letters from `'a'` to `'z'`.
7. Select the first letter that is different from both the left and right neighbors.
8. Replace the `'?'` with the chosen letter and continue processing the string.
9. After processing all characters, join the list back into a string and return it.

### Why it works

The algorithm works because the only constraint is that adjacent characters cannot be equal. When replacing a `'?'`, we only need to ensure it differs from its immediate neighbors. Since there are `26` lowercase letters but at most `2` neighboring characters to avoid, there will always be at least one valid choice. By greedily selecting the first valid character, we preserve correctness without affecting future decisions.

## Python Solution

```python
class Solution:
    def modifyString(self, s: str) -> str:
        characters = list(s)
        n = len(characters)

        for i in range(n):
            if characters[i] != '?':
                continue

            left_char = characters[i - 1] if i > 0 else ''
            right_char = characters[i + 1] if i < n - 1 else ''

            for replacement in 'abcdefghijklmnopqrstuvwxyz':
                if replacement != left_char and replacement != right_char:
                    characters[i] = replacement
                    break

        return ''.join(characters)
```

The implementation begins by converting the string into a list called `characters`, since Python strings cannot be modified in place. We then iterate through every index.

Whenever we encounter a non `'?'` character, we simply skip it because the problem forbids modifications to existing letters.

For each `'?'`, we examine its left and right neighbors. Boundary cases are handled naturally by using an empty string when a neighbor does not exist. This prevents unnecessary special-case logic.

We then iterate through the alphabet and choose the first letter that differs from both neighbors. Once found, we immediately assign it and stop searching. Since the newly assigned character becomes part of the string, subsequent `'?'` positions can safely use it as their left neighbor.

Finally, we join the modified character list back into a string and return the result.

## Go Solution

```go
func modifyString(s string) string {
    characters := []byte(s)
    n := len(characters)

    for i := 0; i < n; i++ {
        if characters[i] != '?' {
            continue
        }

        leftChar := byte(0)
        if i > 0 {
            leftChar = characters[i-1]
        }

        rightChar := byte(0)
        if i < n-1 {
            rightChar = characters[i+1]
        }

        for replacement := byte('a'); replacement <= 'z'; replacement++ {
            if replacement != leftChar && replacement != rightChar {
                characters[i] = replacement
                break
            }
        }
    }

    return string(characters)
}
```

The Go implementation follows the same greedy strategy as the Python version. Since Go strings are immutable, we convert the string into a `[]byte` slice so characters can be modified in place.

Instead of using empty strings for missing neighbors, we use `byte(0)` as a sentinel value. Since valid characters are lowercase English letters, this value will never conflict with actual input.

No special handling for integer overflow is necessary because the problem constraints are very small. The byte slice approach also avoids unnecessary string reconstruction during iteration.

## Edge Cases

### Entire String Consists of `'?'`

An input such as `"????"` can easily expose bugs in naive implementations because every replacement depends on previous decisions. If we always choose the same letter, we would create consecutive duplicates. The greedy approach avoids this by checking the immediate neighbors before each replacement, ensuring adjacent characters remain different.

### `'?'` at the Beginning or End

Strings like `"?ab"` or `"ab?"` have only one neighboring character for the replacement position. A naive implementation might accidentally access an invalid index. Our implementation safely handles boundaries by treating missing neighbors as empty values or sentinel bytes.

### Consecutive `'?'` Characters

Cases like `"a???b"` are tricky because replacing one character affects future choices. If handled incorrectly, earlier replacements could force conflicts later. The greedy strategy naturally handles this because each replacement only needs to avoid the immediate neighbors, and there are always enough unused letters available.

### No `'?'` Characters

A string such as `"leetcode"` already satisfies the condition and requires no modifications. The implementation correctly skips all characters and returns the original string unchanged.
