# LeetCode 1592 - Rearrange Spaces Between Words

## Problem Understanding

This problem gives us a string that contains lowercase English words and spaces. The spaces may appear at the beginning, between words, or at the end of the string. The goal is to redistribute all spaces so that the number of spaces between every pair of adjacent words is equal and as large as possible.

The important detail is that all spaces must still exist in the final string. We are not allowed to remove or add characters. The returned string must have exactly the same length as the original input.

To achieve this, we first count how many spaces exist in the input. Then we determine how many words exist. If there are multiple words, the spaces should be distributed evenly between adjacent words. Any leftover spaces that cannot be distributed equally must be appended at the end of the string.

For example, suppose we have 9 spaces and 4 words. Since there are 3 gaps between words, we place `9 // 3 = 3` spaces in each gap, with no leftovers.

If there is only one word, then there are no gaps between words. In that case, all spaces must be placed after the word.

The constraints are small:

- `1 <= text.length <= 100`

This tells us that performance is not a major concern. Even less efficient approaches would likely pass. However, we should still aim for a clean and optimal solution.

Several edge cases are important:

- A string containing only one word and many spaces
- Spaces at the beginning or end of the string
- Uneven space distribution that leaves extra spaces
- Multiple consecutive spaces between words
- Minimal input size, such as a single character word

A naive implementation may incorrectly lose spaces, miscount words, or fail when only one word exists.

## Approaches

### Brute Force Approach

A brute force solution could scan the string character by character and manually reconstruct words and spaces. We could repeatedly append spaces between words while tracking how many spaces remain.

One possible brute force strategy is:

1. Traverse the string and extract words manually.
2. Count spaces manually.
3. Repeatedly distribute spaces one at a time across gaps.
4. Construct the final string character by character.

This works because it explicitly simulates the redistribution process. However, it is unnecessarily complicated for such a simple problem. Repeatedly distributing spaces one at a time adds avoidable complexity and makes the implementation harder to read.

### Optimal Approach

The key observation is that we only need two pieces of information:

- The total number of spaces
- The total number of words

Once we know these values, the answer becomes straightforward arithmetic.

If there are `n` words:

- There are `n - 1` gaps between words
- Each gap gets `spaces // (n - 1)` spaces
- The leftover spaces are `spaces % (n - 1)`

If there is only one word:

- All spaces go at the end

Python's `split()` method is especially useful because it automatically extracts words and ignores repeated spaces.

This leads to a very clean linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Manually parses and redistributes spaces character by character |
| Optimal | O(n) | O(n) | Uses word extraction and arithmetic distribution |

## Algorithm Walkthrough

1. Count the total number of spaces in the input string.

We can do this by counting occurrences of the `' '` character. This gives us the total amount of space that must exist in the final answer.
2. Extract all words from the string.

Using `split()` is convenient because it automatically ignores extra spaces and returns only the words.

Example:

```
"  this   is  a sentence ".split()
```

becomes:

```
["this", "is", "a", "sentence"]
```
3. Determine how many gaps exist between words.

If there are `k` words, then there are `k - 1` gaps.
4. Handle the single-word edge case.

If there is only one word, we cannot place spaces between words because no gaps exist. Therefore, all spaces are appended after the word.
5. Compute spaces per gap and leftover spaces.

Use integer division:

```
spaces_per_gap = total_spaces // gaps
```

Use modulo:

```
extra_spaces = total_spaces % gaps
```
6. Construct the final string.

Join the words using the computed separator:

```
separator = " " * spaces_per_gap
```

Then append leftover spaces at the end.
7. Return the final string.

### Why it works

The algorithm works because it preserves two important properties:

- Every original space is reused exactly once
- The spaces between adjacent words are maximized while remaining equal

Integer division guarantees the largest equal spacing possible. Any remainder must be leftover spaces, and the problem explicitly requires those extra spaces to appear at the end.

## Python Solution

```python
class Solution:
    def reorderSpaces(self, text: str) -> str:
        total_spaces = text.count(" ")
        words = text.split()

        if len(words) == 1:
            return words[0] + (" " * total_spaces)

        gaps = len(words) - 1
        spaces_per_gap = total_spaces // gaps
        extra_spaces = total_spaces % gaps

        separator = " " * spaces_per_gap

        return separator.join(words) + (" " * extra_spaces)
```

The implementation starts by counting all spaces in the original string using `count(" ")`. This ensures we preserve the exact string length in the final answer.

Next, `split()` extracts all words while ignoring repeated spaces. This simplifies parsing significantly.

The implementation then checks whether only one word exists. This is important because dividing by zero gaps would otherwise cause an error.

For multiple words, the code computes:

- The number of gaps
- The spaces placed in each gap
- The leftover spaces

The separator string is created once and reused in the `join()` operation. Finally, leftover spaces are appended to the end of the string.

This implementation directly follows the algorithm described earlier and remains concise and readable.

## Go Solution

```go
package main

import (
	"strings"
)

func reorderSpaces(text string) string {
	totalSpaces := 0

	for _, ch := range text {
		if ch == ' ' {
			totalSpaces++
		}
	}

	words := strings.Fields(text)

	if len(words) == 1 {
		return words[0] + strings.Repeat(" ", totalSpaces)
	}

	gaps := len(words) - 1
	spacesPerGap := totalSpaces / gaps
	extraSpaces := totalSpaces % gaps

	separator := strings.Repeat(" ", spacesPerGap)

	result := strings.Join(words, separator)
	result += strings.Repeat(" ", extraSpaces)

	return result
}
```

The Go implementation closely mirrors the Python solution. Instead of Python's `split()`, Go uses `strings.Fields()`, which automatically removes repeated spaces and extracts words.

String repetition is handled with `strings.Repeat()`, and joining words is done with `strings.Join()`.

Since the constraints are small, standard string operations are sufficient and efficient.

## Edge Cases

### Single Word Input

A string containing only one word is the most important edge case. Since there are no adjacent word pairs, there are no gaps where spaces can be inserted. A naive implementation may attempt division by zero when computing spaces per gap.

The implementation avoids this by checking whether the word count equals one before performing any division. In that case, all spaces are appended after the word.

### Uneven Space Distribution

When the total number of spaces cannot be evenly divided among all gaps, leftover spaces must be handled correctly. A common bug is accidentally discarding leftover spaces or distributing them incorrectly.

The implementation uses the modulo operator to compute the exact remainder and appends those spaces at the end, exactly as required by the problem statement.

### Leading and Trailing Spaces

The input may contain spaces before the first word or after the last word. These spaces are not special and must simply be counted as part of the total.

Using `split()` ensures that only words are extracted while all spaces are still accounted for separately through `count(" ")`. This guarantees that no spaces are lost during parsing.
