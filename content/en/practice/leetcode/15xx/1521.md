# LeetCode 1521 - Find a Value of a Mysterious Function Closest to Target

## Problem Understanding

The problem defines a mysterious function `func(arr, l, r)` that computes the bitwise AND of all elements in the subarray from index `l` to index `r`, inclusive.

Formally:

$$func(arr, l, r) = arr[l] \,\&\, arr[l+1] \,\&\, \dots \,\&\, arr[r]$$

We are given:

- An integer array `arr`
- An integer `target`

Our task is to examine every possible subarray, compute its bitwise AND value, and find the minimum possible value of:

$$|func(arr, l, r) - target|$$

The output is the smallest absolute difference between any subarray AND result and the target.

The constraints are important:

- `arr.length` can be as large as `10^5`
- Each number is at most `10^6`

A naive approach that checks all subarrays would require examining roughly `n^2` subarrays, which is far too slow for `10^5` elements.

The key challenge is efficiently handling all possible subarray AND values without explicitly enumerating every subarray.

There are several important observations and edge cases:

- Bitwise AND is monotonic decreasing as more elements are added to a subarray. Once a bit becomes `0`, it can never become `1` again.
- Many different subarrays can produce the same AND result.
- The number of distinct AND values ending at a given index is actually quite small.
- Arrays with all identical numbers produce repeated AND values.
- Arrays with powers of two quickly collapse AND values to zero.
- If any subarray AND equals the target exactly, the answer is immediately `0`.

## Approaches

### Brute Force Approach

The most direct solution is to examine every possible subarray.

For each starting index `l`, we extend the ending index `r` one step at a time and maintain the running AND value:

$$current = current \,\&\, arr[r]$$

At every step we compute:

$$|current - target|$$

and update the minimum answer.

This approach is correct because it explicitly checks every subarray, so it cannot miss the optimal result.

However, the time complexity is too large. There are `O(n^2)` subarrays, and even with incremental AND computation, the algorithm still requires quadratic time.

With `n = 10^5`, this becomes infeasible.

### Key Insight for the Optimal Solution

The critical observation is that bitwise AND values shrink very quickly.

Suppose we fix an index `i` and consider all subarrays ending at `i`.

If we repeatedly apply bitwise AND:

$$x_1,\quad x_1 \& x_2,\quad x_1 \& x_2 \& x_3,\dots$$

bits only transition from `1` to `0`. A bit never returns to `1`.

Since integers here have at most about 20 significant bits, the number of distinct AND values that can appear for subarrays ending at a specific position is very small, typically at most around 20.

Instead of storing every subarray result, we only store the distinct AND values for subarrays ending at the current index.

For each new number:

- Start a new subarray containing only that number
- Extend every previous subarray by AND-ing with the new number
- Deduplicate the resulting AND values

This dramatically reduces the number of states we process.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Examines every subarray explicitly |
| Optimal | O(n log M) | O(log M) | Tracks only distinct AND values, where `M` is maximum element size |

Here `log M` corresponds to the number of bits in the integers, which is small.

## Algorithm Walkthrough

### Step 1: Maintain distinct AND values ending at each position

For every index `i`, we maintain a set of all distinct AND values of subarrays ending at `i`.

Call this set `current_values`.

### Step 2: Start a new subarray

At position `i`, the subarray consisting only of `arr[i]` has AND value:

$$arr[i]$$

So this value is always included.

### Step 3: Extend previous subarrays

Every subarray ending at `i-1` can be extended to `i`.

If a previous subarray had AND value `v`, then extending it gives:

$$v \,\&\, arr[i]$$

We compute these new AND values for every value in the previous set.

### Step 4: Deduplicate values

Many subarrays collapse to identical AND values.

We store results in a set so duplicates are removed automatically.

This keeps the number of stored states small.

### Step 5: Update the answer

For every distinct AND value generated at this step, compute:

$$|value - target|$$

Update the global minimum answer.

### Step 6: Move to the next index

The newly computed set becomes the previous set for the next iteration.

### Why it works

The algorithm works because every subarray ending at position `i` is either:

- A new subarray `[i, i]`
- An extension of a subarray ending at `i-1`

Thus, by extending all previous AND values and adding the single-element subarray, we generate all possible subarray AND values ending at `i`.

Deduplication does not lose correctness because identical AND values are interchangeable for future computations.

The monotonic nature of bitwise AND guarantees the number of distinct values remains small.

## Python Solution

```python
from typing import List

class Solution:
    def closestToTarget(self, arr: List[int], target: int) -> int:
        answer = float("inf")
        previous = set()

        for num in arr:
            current = {num}

            for value in previous:
                current.add(value & num)

            for value in current:
                answer = min(answer, abs(value - target))

            previous = current

        return answer
```

The implementation maintains a set called `previous`, which stores all distinct AND values for subarrays ending at the previous index.

For each new element `num`, we create a new set `current`.

First, we add `num` itself because a subarray of length one is always valid.

Next, we extend every previous subarray by computing:

```python
value & num
```

This produces all AND values for subarrays ending at the current index.

We then compute the absolute difference from the target for every generated value and update the best answer.

Finally, `current` becomes the new `previous` for the next iteration.

The solution is concise because the set automatically removes duplicate AND values.

## Go Solution

```go
package main

import "math"

func closestToTarget(arr []int, target int) int {
	answer := math.MaxInt32
	previous := map[int]struct{}{}

	for _, num := range arr {
		current := map[int]struct{}{}
		current[num] = struct{}{}

		for value := range previous {
			current[value&num] = struct{}{}
		}

		for value := range current {
			diff := abs(value - target)
			if diff < answer {
				answer = diff
			}
		}

		previous = current
	}

	return answer
}

func abs(x int) int {
	if x < 0 {
		return -x
	}
	return x
}
```

The Go implementation follows the same logic as the Python version.

Since Go does not have a built in set type, we use:

```go
map[int]struct{}
```

to simulate a set efficiently.

The `struct{}` value occupies zero bytes, making it the standard idiomatic choice for sets in Go.

Go integers are fully sufficient for this problem because the constraints stay well within 32 bit range.

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n log M) | Each position maintains only a small number of distinct AND values |
| Space | O(log M) | At most about one distinct value per bit transition |

The crucial property is that bitwise AND can only remove bits.

An integer with at most 20 bits can only experience a limited number of distinct transitions before stabilizing.

Therefore, although we process sets repeatedly, the total number of distinct AND values per iteration remains very small in practice and theoretically bounded by the number of bits.

## Edge Cases

### Single Element Array

When the array contains only one element, the only possible subarray is the element itself.

A buggy implementation might assume at least one extension step exists or mishandle initialization of the previous set.

This implementation handles the case naturally because every iteration starts with:

```
current = {num}
```

So single element subarrays are always considered correctly.

### AND Values Quickly Become Zero

Arrays containing disjoint bit patterns often produce zero very quickly.

For example:

```
[1, 2, 4, 8]
```

Any AND between different elements becomes zero.

A naive implementation may still continue exploring huge numbers of redundant states.

This solution handles it efficiently because duplicate zeros collapse into a single set entry.

### Many Duplicate AND Results

Different subarrays can produce the same AND value repeatedly.

For example:

```
[7, 7, 7, 7]
```

Every subarray AND equals `7`.

Without deduplication, the algorithm could unnecessarily process many identical states.

Using a set guarantees each distinct AND value is processed only once per position.

### Large Numbers Near Constraint Limits

Values can be as large as `10^6`.

The algorithm remains efficient because the number of distinct AND transitions depends on bit count, not numeric magnitude directly.

Even large values have only about 20 significant bits, so the state space stays small.
