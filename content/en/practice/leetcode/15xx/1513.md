# LeetCode 1513 - Number of Substrings With Only 1s

## Problem Understanding

The problem gives us a binary string `s`, meaning the string contains only the characters `'0'` and `'1'`. We must count how many substrings consist entirely of `'1'` characters.

A substring is any contiguous portion of the string. For example, in the string `"111"`, the valid substrings containing only `'1'` are:

- `"1"` at index 0
- `"1"` at index 1
- `"1"` at index 2
- `"11"` from indices 0 to 1
- `"11"` from indices 1 to 2
- `"111"` from indices 0 to 2

That gives a total of 6 substrings.

The answer can become very large because the string length can be as large as `10^5`. Because of this, the problem requires returning the result modulo `10^9 + 7`.

The important observation is that substrings made entirely of `'1'` characters appear in consecutive runs of `'1'`. Every contiguous block of ones contributes a predictable number of substrings.

For example:

- `"1"` contributes 1 substring
- `"11"` contributes 3 substrings
- `"111"` contributes 6 substrings
- `"1111"` contributes 10 substrings

A run of length `k` contributes:

$1+2+3+\cdots+k=\frac{k(k+1)}{2}$

The constraints tell us that an `O(n^2)` solution will be too slow because there can be up to `100000` characters. We need a linear time solution.

Several edge cases are important:

- Strings containing only zeros should return 0.
- Strings containing only ones should correctly compute the triangular number.
- Alternating patterns like `"10101"` should count isolated single-character substrings.
- Very large runs of ones require modulo arithmetic to avoid overflow.

## Approaches

### Brute Force Approach

A straightforward approach is to generate every possible substring and check whether it contains only `'1'`.

There are `O(n^2)` substrings in a string of length `n`. For each substring, checking whether every character is `'1'` may take another `O(n)` time in the worst case.

This produces a worst-case complexity of `O(n^3)` if implemented naively. Even with optimizations, enumerating all substrings still requires `O(n^2)` time, which is too slow for `n = 100000`.

The brute-force method is correct because it explicitly checks every possible substring, but it does unnecessary repeated work.

### Optimal Approach

The key insight is that valid substrings only depend on consecutive runs of `'1'`.

Suppose we encounter a consecutive block of ones of length `k`.

For example:

- Length 1 contributes 1 substring
- Length 2 contributes 3 substrings
- Length 3 contributes 6 substrings
- Length 4 contributes 10 substrings

Instead of generating substrings explicitly, we can count them incrementally while scanning the string.

As we move through the string:

- Maintain `current_ones`, the length of the current run of consecutive ones.
- If the current character is `'1'`, increment `current_ones`.
- Add `current_ones` to the answer because every extension of the run creates new valid substrings ending at the current position.
- If the current character is `'0'`, reset `current_ones` to 0.

This gives a linear time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) to O(n³) | O(1) | Enumerates all substrings and checks validity |
| Optimal | O(n) | O(1) | Counts contributions from consecutive runs of ones |

## Algorithm Walkthrough

1. Initialize a variable `result` to store the total number of valid substrings.
2. Initialize another variable `current_ones` to track the length of the current consecutive run of `'1'` characters.
3. Iterate through each character in the string from left to right.
4. If the current character is `'1'`, increment `current_ones` by 1. This means the current run of consecutive ones has grown.
5. Add `current_ones` to `result`. This works because every new `'1'` creates additional substrings ending at the current position:

- A run length of 1 adds 1 substring
- A run length of 2 adds 2 new substrings
- A run length of 3 adds 3 new substrings
6. If the current character is `'0'`, reset `current_ones` to 0 because the run of consecutive ones has been broken.
7. Apply modulo `10^9 + 7` after each addition to prevent overflow.
8. After processing the entire string, return `result`.

### Why it works

The algorithm relies on the fact that every substring consisting only of ones must belong to some consecutive run of ones.

If a run currently has length `k`, then exactly `k` valid substrings end at the current position:

- The last character alone
- The last two characters
- The last three characters
- And so on

By adding `current_ones` at each step, we count every valid substring exactly once.

## Python Solution

```python
class Solution:
    def numSub(self, s: str) -> int:
        MOD = 10**9 + 7
        
        result = 0
        current_ones = 0
        
        for char in s:
            if char == '1':
                current_ones += 1
                result += current_ones
            else:
                current_ones = 0
            
            result %= MOD
        
        return result
```

The implementation follows the algorithm directly.

The variable `current_ones` stores the current streak of consecutive `'1'` characters. Every time another `'1'` is encountered, the streak grows by one.

The critical insight is that when the streak length becomes `k`, exactly `k` new valid substrings end at the current index. These substrings are automatically counted by adding `current_ones` to `result`.

When a `'0'` appears, the streak is broken, so `current_ones` is reset to zero.

Modulo arithmetic is applied after every update to keep the result within bounds.

## Go Solution

```go
func numSub(s string) int {
    const MOD int = 1000000007

    result := 0
    currentOnes := 0

    for _, char := range s {
        if char == '1' {
            currentOnes++
            result += currentOnes
        } else {
            currentOnes = 0
        }

        result %= MOD
    }

    return result
}
```

The Go implementation mirrors the Python solution closely.

Go strings are iterated using `range`, which produces rune values. Since the string contains only ASCII characters `'0'` and `'1'`, direct rune comparison works correctly.

The modulo constant is stored as an integer, and all computations remain within integer limits because the modulo operation is applied continuously.

## Edge Cases

One important edge case is a string containing only zeros, such as `"00000"`. A buggy implementation might accidentally count invalid substrings or fail to reset the running streak correctly. In this implementation, every zero resets `current_ones` to zero, so no invalid substrings are counted.

Another important case is a string containing only ones, such as `"111111"`. This produces the maximum number of valid substrings for a given string length. The implementation handles this correctly by continuously extending the streak and accumulating the triangular number incrementally.

A third critical edge case is alternating characters like `"1010101"`. These inputs repeatedly start and stop runs of ones. Some implementations incorrectly merge runs across zeros. Here, every zero immediately resets the streak counter, ensuring each run is treated independently.

A final important consideration is very large inputs near the maximum length constraint. Since the number of substrings grows quadratically with the run length, the result can become extremely large. The modulo operation after each update ensures the value always remains within bounds.
