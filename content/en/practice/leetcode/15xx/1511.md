## Solution

## Problem Understanding

The problem asks us to identify customers who spent at least $100 in **both June 2020 and July 2020**. We are given three tables: `Customers`, `Product`, and `Orders`. The `Customers` table contains customer details such as `customer_id`, `name`, and `country`. The `Product` table contains product details including `product_id`, `description`, and `price`. The `Orders` table contains order records, specifying which customer bought which product, on which date, and in what quantity.

Our task is to calculate the total spending for each customer in June 2020 and July 2020, and only include those customers whose spending reaches at least $100 in **both months**. The output is a table containing the `customer_id` and `name` of these qualifying customers.

Key points include understanding that spending is calculated as `quantity * price` for each order, and the date filtering must precisely handle only June (`2020-06-01` to `2020-06-30`) and July (`2020-07-01` to `2020-07-31`) 2020. Edge cases include customers with spending just below $100, customers with no orders in one of the months, and multiple orders contributing to cumulative spending in a month.

## Approaches

The brute-force approach involves joining all three tables, iterating over each order, grouping them by customer and month, summing the total spending, and then checking whether the sum meets the $100 threshold. While correct, it can be inefficient if there are many orders because it requires scanning the full `Orders` table and calculating totals for every customer manually.

A better approach uses SQL aggregation and conditional filtering. We join `Orders` with `Product` to compute spending per order, filter orders to June and July 2020, then group by `customer_id` and month, summing the spending. After grouping, we filter for customers who have at least $100 in both months using a `HAVING` clause to ensure each qualifying customer has two qualifying months. This approach leverages SQL aggregation functions efficiently and avoids manual iteration.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n*m) | O(k) | Iterate through all orders and accumulate spending per customer per month |
| Optimal | O(n) | O(k) | Aggregate spending via SQL joins and group-by, filter using HAVING; scales well for large datasets |

## Algorithm Walkthrough

1. Join the `Orders` table with the `Product` table on `product_id` to compute the total amount spent for each order using `quantity * price`.
2. Filter the resulting rows to include only orders from June 2020 or July 2020. Use SQL date functions or string pattern matching to extract the year and month.
3. Group the filtered orders by `customer_id` and the month extracted from `order_date`.
4. Aggregate the total spending per customer per month using `SUM(quantity * price)`.
5. After aggregation, filter out groups where the total spending is less than $100 for a month.
6. For each customer, count the number of months in which they met the $100 spending threshold. Only keep customers who have two months (June and July) meeting the threshold.
7. Join the result with the `Customers` table to fetch the customer name.
8. Return the resulting `customer_id` and `name`.

**Why it works**: By grouping first by `customer_id` and month and then filtering for $100 spending, we guarantee that only customers who meet the spending requirement in **both months** are retained. This leverages SQL’s aggregate functions efficiently, avoiding manual iteration and ensuring correctness for large datasets.

## Python Solution

```python
# This problem is typically solved with SQL; here is a Python simulation using pandas for clarity.
import pandas as pd

def customer_order_frequency(customers: pd.DataFrame, products: pd.DataFrame, orders: pd.DataFrame) -> pd.DataFrame:
    # Merge orders with products to get price information
    merged = orders.merge(products, on='product_id')
    # Calculate total spending per order
    merged['total_spent'] = merged['quantity'] * merged['price']
    # Filter for June and July 2020
    merged['order_date'] = pd.to_datetime(merged['order_date'])
    merged = merged[(merged['order_date'].dt.year == 2020) & (merged['order_date'].dt.month.isin([6, 7]))]
    # Aggregate spending by customer and month
    merged['month'] = merged['order_date'].dt.month
    spending = merged.groupby(['customer_id', 'month'])['total_spent'].sum().reset_index()
    # Keep only months with spending >= 100
    spending = spending[spending['total_spent'] >= 100]
    # Count qualifying months per customer
    month_count = spending.groupby('customer_id')['month'].nunique().reset_index()
    qualified = month_count[month_count['month'] == 2]
    # Join with customer names
    result = qualified.merge(customers, on='customer_id')[['customer_id', 'name']]
    return result
```

**Explanation**: The code first joins `orders` and `products` to compute `total_spent`. It then filters for June and July 2020. Next, it aggregates spending by customer and month, filters for spending >= $100, counts qualifying months, and finally filters for customers with both months qualifying. The resulting customer list is joined with `customers` to retrieve names.

## Go Solution

```go
// SQL-centric solution using Go database/sql package
package main

import (
	"database/sql"
	"fmt"
	_ "github.com/go-sql-driver/mysql"
)

func CustomerOrderFrequency(db *sql.DB) ([][2]interface{}, error) {
	query := `
	SELECT c.customer_id, c.name
	FROM Customers c
	JOIN (
	    SELECT o.customer_id
	    FROM Orders o
	    JOIN Product p ON o.product_id = p.product_id
	    WHERE o.order_date BETWEEN '2020-06-01' AND '2020-07-31'
	    GROUP BY o.customer_id, MONTH(o.order_date)
	    HAVING SUM(o.quantity * p.price) >= 100
	    ) qualified ON c.customer_id = qualified.customer_id
	GROUP BY c.customer_id, c.name
	HAVING COUNT(DISTINCT MONTH(order_date)) = 2
	`
	rows, err := db.Query(query)
	if err != nil {
		return nil, err
	}
	defer rows.Close()
	var result [][2]interface{}
	for rows.Next() {
		var customer_id int
		var name string
		if err := rows.Scan(&customer_id, &name); err != nil {
			return nil, err
		}
		result = append(result, [2]interface{}{customer_id, name})
	}
	return result, nil
}
```

**Explanation**: This Go solution uses a direct SQL query. The subquery calculates per-customer, per-month spending and filters months with at least $100. The outer query selects customers with two qualifying months and fetches names. Go-specific handling involves scanning rows into variables and using slices to collect results.
