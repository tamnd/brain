## Problem Understanding

The problem asks us to take a date string in a human-readable format such as `"20th Oct 2052"` and convert it into a standardized format `"YYYY-MM-DD"`. The input consists of three parts: the day, which includes an ordinal suffix like `st`, `nd`, `rd`, or `th`; the month, given as a three-letter English abbreviation; and the year, which is a four-digit number between 1900 and 2100. The output should have the year first, followed by a two-digit month and a two-digit day, separated by hyphens.

The constraints make the problem simpler because we are guaranteed that the input is valid, so no need to handle invalid dates. Important edge considerations include single-digit days such as `"1st"`, `"2nd"`, or `"9th"` that must be padded with a leading zero in the output. Similarly, months must be converted from text to numbers and also formatted with two digits.

## Approaches

A brute-force approach would manually parse the string using string operations: remove the ordinal suffix from the day, convert the month abbreviation to a number via multiple `if-elif` conditions or a list index, and then format the result with string formatting. This works correctly, but the string parsing could be tedious and error-prone if done with ad-hoc slicing and concatenation.

The optimal approach leverages a mapping of month abbreviations to numeric strings and standard string operations. We split the input into three components, remove the non-numeric characters from the day, convert the month using a dictionary lookup, and pad both the day and month to ensure two digits. This is simple, robust, and runs in constant time since the input size is small and bounded.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(1) | O(1) | Manual string parsing using multiple conditionals |
| Optimal | O(1) | O(1) | Split string, dictionary lookup for month, strip day suffix, pad to 2 digits |

## Algorithm Walkthrough

1. Define a dictionary mapping month abbreviations `"Jan"` through `"Dec"` to their corresponding two-digit string representations `"01"` through `"12"`.
2. Split the input string by spaces to separate the day, month, and year.
3. Remove the last two characters from the day string, which correspond to the ordinal suffix (`"st"`, `"nd"`, `"rd"`, `"th"`).
4. Convert the resulting day string to an integer to remove any leading zeros, then format it back as a two-digit string using zero-padding.
5. Use the month dictionary to convert the month abbreviation to its two-digit numeric string.
6. Combine the year, month, and day strings in the `"YYYY-MM-DD"` format and return the result.

**Why it works:** By splitting the string and handling each component separately, we preserve the semantics of the date. The dictionary guarantees the correct month number, and formatting ensures the correct zero-padding. Since the input is always valid, this approach is both safe and efficient.

## Python Solution

```python
class Solution:
    def reformatDate(self, date: str) -> str:
        month_map = {
            "Jan": "01", "Feb": "02", "Mar": "03", "Apr": "04",
            "May": "05", "Jun": "06", "Jul": "07", "Aug": "08",
            "Sep": "09", "Oct": "10", "Nov": "11", "Dec": "12"
        }
        
        day_str, month_str, year_str = date.split()
        day = day_str[:-2]  # Remove the ordinal suffix
        day = day.zfill(2)  # Pad single digit day with zero
        
        month = month_map[month_str]
        
        return f"{year_str}-{month}-{day}"
```

The Python implementation first creates a dictionary for month mapping, then splits the input string into day, month, and year. It removes the last two characters of the day to strip the ordinal suffix, pads the day to two digits, converts the month using the dictionary, and finally constructs the formatted string.

## Go Solution

```go
import (
    "fmt"
    "strings"
)

func reformatDate(date string) string {
    monthMap := map[string]string{
        "Jan": "01", "Feb": "02", "Mar": "03", "Apr": "04",
        "May": "05", "Jun": "06", "Jul": "07", "Aug": "08",
        "Sep": "09", "Oct": "10", "Nov": "11", "Dec": "12",
    }

    parts := strings.Split(date, " ")
    day := parts[0][:len(parts[0])-2]
    if len(day) == 1 {
        day = "0" + day
    }
    month := monthMap[parts[1]]
    year := parts[2]

    return fmt.Sprintf("%s-%s-%s", year, month, day)
}
```

The Go implementation follows the same logic as the Python version. It uses a map for month conversion, slices the day string to remove the ordinal suffix, pads single-digit days, and formats the final string using `fmt.Sprintf`.

## Edge Cases

A key edge case is single-digit days such as `"1st"`, `"2nd"`, `"3rd"`. A naive implementation might forget to pad them with a leading zero, producing `"YYYY-MM-D"` instead of `"YYYY-MM-DD"`. Our implementation uses `zfill(2)` in Python or string concatenation in Go to ensure proper formatting.

Another edge case is months with single-digit numeric equivalents, like `"Jan"`, `"Feb"`, `"Mar"`. Direct integer conversion without zero-padding would yield `"YYYY-M-DD"`, which is invalid. Using a dictionary mapping to two-digit strings guarantees consistent formatting.

Lastly, the input could have the ordinal suffix `st`, `nd`, `rd`, or `th`. Simply attempting to parse the day as an integer without removing the suffix would raise an error. By slicing off the last two characters, we ensure that the day string can safely be converted to an integer or zero-padded string.
