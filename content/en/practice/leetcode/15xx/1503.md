# LeetCode 1503 - Last Moment Before All Ants Fall Out of a Plank

## Problem Understanding

The problem describes a wooden plank of length `n`, with ants standing at different positions on the plank. Every ant moves at a constant speed of `1` unit per second. Some ants move toward the left side of the plank, while others move toward the right side.

Whenever two ants moving in opposite directions meet, they instantly reverse directions and continue moving. If an ant reaches either end of the plank, it immediately falls off.

We are given:

- `n`, the plank length
- `left`, positions of ants currently moving left
- `right`, positions of ants currently moving right

The goal is to determine the exact moment when the final ant falls off the plank.

At first glance, the collision behavior seems complicated because ants can repeatedly meet and reverse directions. A naive simulation would need to carefully track every collision event.

However, the important observation is that when two ants collide and reverse directions, the overall system behaves exactly the same as if the ants simply passed through each other. Since all ants are identical and move at the same speed, swapping directions is indistinguishable from swapping identities.

This means we never actually need to simulate collisions.

Instead:

- A left-moving ant at position `x` falls off after `x` seconds
- A right-moving ant at position `x` falls off after `n - x` seconds

The answer is simply the maximum fall time among all ants.

The constraints are small enough that even simulation could work, but the intended solution is much simpler and more elegant:

- `1 <= n <= 10^4`
- Total number of ants is at most `n + 1`

The problem guarantees:

- All positions are unique
- No position appears in both arrays
- There is at least one ant

Important edge cases include:

- All ants moving left
- All ants moving right
- Ants starting at the edges `0` or `n`
- Empty `left` or empty `right`
- Only one ant on the plank

## Approaches

### Brute Force Simulation

A brute force solution would explicitly simulate the ants moving across the plank over time.

The implementation would need to:

1. Track every ant’s position and direction
2. Advance time continuously or in tiny increments
3. Detect collisions between ants
4. Reverse directions when collisions occur
5. Remove ants when they fall off

This approach is correct because it models the exact physical behavior described in the problem.

However, it is unnecessarily complicated. Collision detection becomes messy, especially when multiple ants collide simultaneously. Continuous movement also introduces precision issues if using floating point arithmetic.

Even though the constraints are not extremely large, this simulation is difficult to implement correctly and efficiently.

### Optimal Observation

The key insight is that ants are indistinguishable.

Suppose two ants collide:

- Ant A moving right
- Ant B moving left

After collision:

- Ant A moves left
- Ant B moves right

This is physically identical to the ants passing through each other and continuing unchanged. Since ants have identical speed and appearance, swapping directions produces the same observable result as swapping identities.

Therefore:

- We can completely ignore collisions
- Each ant simply walks straight until falling off

So:

- A left-moving ant at position `x` takes `x` seconds to fall
- A right-moving ant at position `x` takes `n - x` seconds to fall

The final answer is the maximum of all these times.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(T × A²) | O(A) | Simulates movement and collisions explicitly |
| Optimal | O(A) | O(1) | Uses collision equivalence observation |

Here:

- `A` is the number of ants
- `T` is the number of simulation steps

## Algorithm Walkthrough

1. Initialize a variable `last_moment` to `0`.
2. Iterate through every position in `left`.

A left-moving ant at position `x` needs exactly `x` seconds to fall off the plank because it moves directly toward position `0`.

Update:

```
last_moment = max(last_moment, x)
```
3. Iterate through every position in `right`.

A right-moving ant at position `x` needs exactly `n - x` seconds to fall off because it moves toward position `n`.

Update:

```
last_moment = max(last_moment, n - x)
```
4. Return `last_moment`.

### Why it works

The correctness comes from the observation that collisions are equivalent to ants passing through one another.

Whenever two ants collide and reverse directions, the resulting configuration is indistinguishable from them continuing straight through each other. Therefore, each ant’s effective travel time depends only on:

- Its starting position
- Its initial direction

Thus the last ant to fall is simply the ant with the maximum direct distance to its corresponding edge.

## Python Solution

```python
from typing import List

class Solution:
    def getLastMoment(self, n: int, left: List[int], right: List[int]) -> int:
        last_moment = 0

        for position in left:
            last_moment = max(last_moment, position)

        for position in right:
            last_moment = max(last_moment, n - position)

        return last_moment
```

The implementation directly follows the mathematical observation.

The variable `last_moment` stores the largest fall time seen so far.

For every left-moving ant:

- Time to fall = `position`

For every right-moving ant:

- Time to fall = `n - position`

The algorithm scans both arrays once and continuously updates the maximum value.

No collision handling is needed because the collision behavior has already been reduced to the simpler equivalent model.

## Go Solution

```go
func getLastMoment(n int, left []int, right []int) int {
    lastMoment := 0

    for _, position := range left {
        if position > lastMoment {
            lastMoment = position
        }
    }

    for _, position := range right {
        fallTime := n - position

        if fallTime > lastMoment {
            lastMoment = fallTime
        }
    }

    return lastMoment
}
```

The Go implementation mirrors the Python solution closely.

Some Go-specific details:

- Go uses slices instead of Python lists
- The `range` keyword iterates through slice values
- Integer arithmetic is safe here because constraints are small
- Empty slices are handled naturally without special logic

## Edge Cases

### Empty `left` or `right`

One of the arrays may be empty. A buggy implementation might assume both arrays contain ants and accidentally access invalid indices or skip processing entirely.

This solution handles empty arrays naturally because the loops simply execute zero times.

For example:

```
left = []
right = [0,1,2]
```

Only the right-moving ants are processed.

### Ant Already at an Edge

An ant can start at position `0` or `n`.

Such ants fall immediately, meaning their fall time is `0`.

Incorrect implementations sometimes mistakenly treat these ants as needing one extra second. This solution correctly computes:

- Left-moving ant at `0` → `0`
- Right-moving ant at `n` → `n - n = 0`

### Collision Misinterpretation

The most common conceptual bug is attempting to explicitly simulate collisions.

Many developers initially think collisions alter the total time. However, because ants are identical, reversing directions is equivalent to passing through each other.

This implementation avoids all collision logic entirely and relies on the mathematical equivalence proof, which guarantees correctness.
