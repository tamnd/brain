## Solution

## Problem Understanding

The problem is asking to find **special positions** in a binary matrix. A position `(i, j)` is considered special if `mat[i][j]` is `1` and **all other elements** in the same row `i` and column `j` are `0`. The input is a 2D binary matrix of size `m x n` where each element is either `0` or `1`. The expected output is an integer representing the total number of such special positions.

The constraints indicate that both the number of rows `m` and columns `n` are at most 100, which is small enough to allow solutions with quadratic time complexity. Each element being strictly `0` or `1` simplifies the checking logic. Edge cases to consider include matrices with all zeros, matrices with all ones in a row or column, and the smallest possible matrix size `1 x 1`.

The problem guarantees that the matrix is non-empty and only contains binary values, so we do not need to handle invalid inputs.

## Approaches

The **brute-force approach** would iterate over every cell in the matrix. For each cell that contains `1`, we would check its entire row and column to ensure all other elements are `0`. While this approach is straightforward and guarantees correctness, it is inefficient because checking a row and column for each `1` takes `O(m + n)` operations, leading to an overall time complexity of `O(m * n * (m + n))`.

The **optimal approach** is based on precomputing the sums of each row and column. If we know the sum of each row and the sum of each column, then a `1` is special if and only if the sum of its row and the sum of its column are both equal to `1`. This reduces the per-cell check to constant time after preprocessing, giving a much faster overall algorithm.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m * n * (m + n)) | O(1) | Check every 1 by scanning its row and column |
| Optimal | O(m * n) | O(m + n) | Precompute row and column sums, then check each 1 in constant time |

## Algorithm Walkthrough

1. Initialize two arrays `rowSum` and `colSum` of lengths `m` and `n` respectively to store the sum of each row and each column.
2. Iterate through the matrix once, adding each element to its corresponding row sum and column sum.
3. Initialize a variable `count` to zero. This will store the number of special positions.
4. Iterate through the matrix a second time. For each cell `(i, j)` that is `1`, check if `rowSum[i]` is `1` and `colSum[j]` is `1`. If both conditions are true, increment `count`.
5. Return `count` as the result.

**Why it works:** By summing each row and column upfront, we avoid repeated scanning. A `1` can only be special if it is the only `1` in its row and column, which is exactly what the precomputed sums allow us to check in constant time. This guarantees correctness while improving efficiency.

## Python Solution

```python
from typing import List

class Solution:
    def numSpecial(self, mat: List[List[int]]) -> int:
        m, n = len(mat), len(mat[0])
        rowSum = [0] * m
        colSum = [0] * n
        
        # Compute sums of rows and columns
        for i in range(m):
            for j in range(n):
                rowSum[i] += mat[i][j]
                colSum[j] += mat[i][j]
        
        count = 0
        # Check each 1 if it is a special position
        for i in range(m):
            for j in range(n):
                if mat[i][j] == 1 and rowSum[i] == 1 and colSum[j] == 1:
                    count += 1
        
        return count
```

**Explanation:** First, `rowSum` and `colSum` arrays are filled by iterating through the matrix once. Then, each `1` is checked in constant time against these sums. This avoids repeated scanning of rows and columns and correctly counts all special positions.

## Go Solution

```go
func numSpecial(mat [][]int) int {
    m, n := len(mat), len(mat[0])
    rowSum := make([]int, m)
    colSum := make([]int, n)

    // Compute sums of rows and columns
    for i := 0; i < m; i++ {
        for j := 0; j < n; j++ {
            rowSum[i] += mat[i][j]
            colSum[j] += mat[i][j]
        }
    }

    count := 0
    // Check each 1 if it is a special position
    for i := 0; i < m; i++ {
        for j := 0; j < n; j++ {
            if mat[i][j] == 1 && rowSum[i] == 1 && colSum[j] == 1 {
                count++
            }
        }
    }

    return count
}
```

**Go-specific notes:** Go uses slices for dynamic arrays, and integer overflow is not a concern since the constraints are small. The logic mirrors the Python solution exactly, with syntax differences in array creation and for-loops.

- `(2,0)` → rowSum=1, colSum=2 → not special

## Edge Cases

**Single element matrix:** A `1x1` matrix with `0` or `1` tests if the algorithm handles the smallest input. Our solution correctly counts `1` as special if it exists.

**Multiple ones in the same row or column:** Rows or columns with more than one `1` can trick a naive check. The precomputed sum ensures these are not counted as special.

**Sparse matrix:** Matrices with mostly zeros and a few ones test if the algorithm correctly identifies isolated ones. Precomputing sums ensures constant-time checks for each candidate `1`, handling sparse data efficiently.
