## Solution

## Problem Understanding

This problem asks us to analyze phone call data and determine which countries have an average call duration that is strictly greater than the global average call duration across all calls.

We are given three database tables:

- `Person`, which maps a person ID to a phone number
- `Country`, which maps a country code to a country name
- `Calls`, which stores phone calls between people along with call duration

The key detail is that a person's country is determined by the first three digits of their phone number. For example, the phone number `051-1234567` belongs to the country whose code is `051`.

The company wants to invest in countries where the average duration of calls involving people from that country exceeds the global average duration of all calls.

A subtle but extremely important detail is how a call contributes to a country's statistics. Every call contributes separately to both participants' countries. That means:

- If a call happens between two people in the same country, that duration contributes twice to that country's average, once for the caller and once for the callee.
- If a call happens between people in different countries, the duration contributes once to each country's statistics.

The output should contain the names of all countries whose average call duration is strictly greater than the global average.

The input guarantees that:

- `Person.id` is unique
- `Country.country_code` is unique
- `caller_id != callee_id`
- Phone numbers always follow the format `xxx-yyyyyyy`

Because this is a database problem, efficiency mainly depends on how we structure joins and aggregations rather than traditional algorithmic complexity.

Some important edge cases include:

- Calls between people from the same country
- Countries that never appear in any call
- Duplicate call rows in the `Calls` table
- Countries whose average equals the global average exactly, since the comparison must be strictly greater

A naive implementation can easily miscount calls by forgetting that each call contributes separately for both participants.

## Approaches

### Brute Force Approach

A brute force solution would examine every country individually and recompute all relevant calls for that country from scratch.

For each country:

1. Iterate through all calls.
2. Determine the caller's country.
3. Determine the callee's country.
4. If either participant belongs to the current country, include the duration.
5. Compute the country's average.
6. Separately compute the global average from all calls.

This works because it explicitly checks every possible relationship between calls and countries. However, it is inefficient because the same calls are repeatedly scanned for every country.

If there are `C` countries and `N` calls, this repeated scanning leads to unnecessary work.

### Optimal Approach

The key observation is that every call can be transformed into country-based participation rows exactly once.

Instead of processing each country separately, we can:

1. Convert each call into two participation records:

- one for the caller's country
- one for the callee's country
2. Aggregate durations by country
3. Compute the global average once
4. Compare each country's average against the global average

This transformation avoids rescanning the calls table multiple times and naturally handles both domestic and international calls.

The cleanest SQL implementation uses `UNION ALL` to create the participation rows.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(C × N) | O(1) | Repeatedly scans all calls for every country |
| Optimal | O(N) | O(N) | Processes each call once through aggregation |

## Algorithm Walkthrough

1. First, map every person to their country.

We extract the first three digits from `phone_number` and join them with the `Country` table using `country_code`.
2. Next, transform each phone call into country participation records.

Every call contributes once for the caller's country and once for the callee's country. We use `UNION ALL` so both contributions are preserved, even if they belong to the same country.
3. Compute the global average duration.

Since every participation row represents one country contribution, the global average can be computed directly from the transformed dataset.
4. Group the participation rows by country.

For each country:

- sum all durations
- count all participation rows
- compute the average duration
5. Compare each country's average against the global average.

Only countries whose average is strictly greater are included in the final result.

### Why it works

The algorithm works because every call contribution is represented exactly once per participating country.

A domestic call contributes twice to the same country because both participants belong to that country. An international call contributes once to each country. This precisely matches the problem definition.

Since all averages are computed from the same participation model, the comparison against the global average is mathematically correct.

## Python Solution

```python
# Write your MySQL query statement below

WITH person_country AS (
    SELECT
        p.id,
        c.name AS country
    FROM Person p
    JOIN Country c
        ON SUBSTRING(p.phone_number, 1, 3) = c.country_code
),
country_calls AS (
    SELECT
        pc.country,
        cl.duration
    FROM Calls cl
    JOIN person_country pc
        ON cl.caller_id = pc.id

    UNION ALL

    SELECT
        pc.country,
        cl.duration
    FROM Calls cl
    JOIN person_country pc
        ON cl.callee_id = pc.id
),
global_avg AS (
    SELECT AVG(duration) AS avg_duration
    FROM country_calls
)

SELECT
    cc.country
FROM country_calls cc
CROSS JOIN global_avg ga
GROUP BY cc.country, ga.avg_duration
HAVING AVG(cc.duration) > ga.avg_duration;
```

The solution begins by constructing a `person_country` common table expression. This maps each person ID to a country name by extracting the first three digits from the phone number and joining with the `Country` table.

The second common table expression, `country_calls`, expands every call into two rows. One row represents the caller's country participation, and the other represents the callee's country participation. Using `UNION ALL` is important because duplicate participation rows must not be removed.

The `global_avg` common table expression computes the average duration across all participation rows.

Finally, the query groups rows by country and filters only those countries whose average duration exceeds the global average.

## Edge Cases

### Calls Between People in the Same Country

Domestic calls are one of the easiest places to introduce bugs. A naive solution may accidentally count the duration only once because both participants belong to the same country.

The implementation handles this correctly by using `UNION ALL`. The caller contribution and callee contribution are inserted separately, so a domestic call contributes twice to the country's statistics.

### Countries With No Calls

Some countries may exist in the `Country` table but never appear in any call participation.

These countries should not appear in the result because no average duration can be computed for them. The query naturally excludes them because aggregation only happens over rows present in `country_calls`.

### Duplicate Rows in Calls

The `Calls` table explicitly allows duplicate rows. Removing duplicates would produce incorrect averages.

The implementation uses `UNION ALL` rather than `UNION`, ensuring all duplicate participation rows are preserved exactly as they appear in the input data.

### Countries With Average Equal to Global Average

The problem requires the country's average to be strictly greater than the global average.

Using `HAVING AVG(cc.duration) > ga.avg_duration` correctly excludes countries whose averages are exactly equal to the global average.
