## Solution

## Problem Understanding

This problem requires us to balance a string of parentheses under a non-standard rule: each left parenthesis `'('` must correspond to **two consecutive right parentheses `'))'`**. That is, a balanced string is not just about matching counts of `'('` and `')'`, but about matching the pattern that each `'('` pairs with `'))'` specifically. The string may already contain some balanced pairs or unmatched parentheses. We are allowed to **insert additional `'('` or `')'` anywhere** in the string to make it balanced, and the goal is to minimize the number of insertions.

The input is a string `s` of length up to `10^5` consisting only of `'('` and `')'`. The expected output is an integer indicating the minimum number of insertions needed to balance the string. The constraints indicate that the solution should run in **linear time** and **constant or linear space**; anything quadratic would be too slow.

Important edge cases include:

- Strings starting with `')'` that require adding a `'('` at the beginning.
- Strings ending with `'('` that need additional `'))'` at the end.
- Strings with odd numbers of consecutive `')'` which may require adding one `')'` to complete the pair.
- Empty string, which trivially requires no insertions.

## Approaches

### Brute Force Approach

A brute-force approach would try to generate all possible insertions of `'('` and `')'` to balance the string and track the minimum number of insertions. One could simulate pairing every `'('` with a `'))'` by scanning left to right and trying all combinations of insertions to fix unmatched parentheses. This approach is **correct in principle**, but it is **exponentially slow** because the number of insertion combinations grows rapidly with string length, making it infeasible for `s.length = 10^5`.

### Optimal Approach

The key insight is to process the string **linearly from left to right**, maintaining the number of **needed right parentheses** (`need`) and the number of insertions made (`insertions`). The rules for updating these variables are:

- For `'('`, we anticipate two `')'` to match it, so we increment `need` by 2.
- For `')'`, if `need > 0`, we decrement `need` by 1 (one closing matched). If `need == 0`, it means this `')'` is unmatched, so we need to insert a `'('` before it and increment `need` by 1 (because now one `'('` expects 2 `')'` but we just consumed one).

Additionally, if we encounter **two consecutive `')'`**, we can decrement `need` by 2, and if the count is odd, we handle the last single `')'` separately. This way, we track insertions on the fly and avoid any backtracking or extra storage beyond counters.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Try all possible insertions to balance string; infeasible for large n |
| Optimal | O(n) | O(1) | Single pass maintaining needed closing parentheses and counting insertions |

## Algorithm Walkthrough

1. Initialize `insertions = 0` to count total insertions and `need = 0` to track how many `')'` are required to balance `'('`.
2. Iterate through the string with index `i` from 0 to `len(s) - 1`.
3. If `s[i] == '('`, increment `need` by 2 because this `'('` expects two consecutive `')'`.
4. If `s[i] == ')'`:

- Check if the next character `s[i+1]` is also `')'` to handle consecutive closing parentheses.
- If `s[i+1] == ')'`, decrement `need` by 2 to satisfy the pair and increment `i` by 1 to skip the next `')'`.
- If `s[i+1] != ')'` or this is the last character, decrement `need` by 1 and increment `insertions` by 1 because we need an extra `')'` to form a proper pair.
5. If `need` ever goes below 0, it means there are unmatched `')'` that need a `'('` before them. Increment `insertions` by 1 and reset `need` to 1 (one `')'` still required for the newly inserted `'('`).
6. After finishing the iteration, any remaining `need > 0` indicates unmatched `'('` that need to be closed. Increment `insertions` by `need`.
7. Return `insertions`.

**Why it works:** This algorithm works because it maintains a correct invariant: `need` always represents how many `')'` are required to balance all `'('` seen so far. By incrementally updating `insertions` whenever `need` goes negative or a single `')'` appears, we guarantee the minimal number of insertions without overcounting.

## Python Solution

```python
class Solution:
    def minInsertions(self, s: str) -> int:
        insertions = 0
        need = 0
        i = 0
        n = len(s)
        
        while i < n:
            if s[i] == '(':
                need += 2
            else:  # s[i] == ')'
                if i + 1 < n and s[i + 1] == ')':
                    need -= 2
                    i += 1  # skip next ')'
                else:
                    need -= 1
                    insertions += 1  # insert one ')' to make a pair

            if need < 0:
                insertions += 1  # insert '(' before this ')'
                need = 1  # one ')' still needed for the new '('
            
            i += 1
        
        insertions += need  # insert remaining ')' if needed
        return insertions
```

**Implementation Explanation:** The Python code follows the algorithm directly. We track `need` to know how many `')'` are required. Consecutive `')'` are handled by checking the next character and skipping it. Whenever `need < 0`, we insert `'('` to fix an unmatched `')'`. Finally, any leftover `need` is handled by adding the required `')'`.

## Go Solution

```go
func minInsertions(s string) int {
    insertions := 0
    need := 0
    n := len(s)
    
    for i := 0; i < n; i++ {
        if s[i] == '(' {
            need += 2
        } else {
            if i+1 < n && s[i+1] == ')' {
                need -= 2
                i++ // skip next ')'
            } else {
                need -= 1
                insertions++ // insert one ')' to make a pair
            }
        }
        
        if need < 0 {
            insertions++ // insert '(' before this ')'
            need = 1
        }
    }
    
    insertions += need
    return insertions
}
```

**Go-specific notes:** The Go version mirrors the Python logic. We use integer variables to track insertions and need. Slices are indexed similarly, and the `for` loop is adjusted to increment `i` manually when skipping a `')'`.
