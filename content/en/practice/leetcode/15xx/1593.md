## Solution

## Problem Understanding

The problem asks us to split a given string `s` into the maximum number of **unique substrings**. In other words, we need to partition the string such that each substring appears **only once** in the final list, and we want the count of substrings to be as large as possible. The input is a string of length between 1 and 16 containing only lowercase English letters. The output is a single integer representing the maximum number of unique substrings we can achieve.

The constraints are very important. The string length being at most 16 indicates that a brute-force or backtracking approach is feasible because the total number of substring combinations is limited. This also hints that an exhaustive search with careful pruning is acceptable.

Important edge cases include:

1. All characters are the same, e.g., `"aaaa"`. Only one substring is possible.
2. All characters are unique, e.g., `"abcd"`. Each character could be a substring.
3. Strings with repeated patterns, e.g., `"abab"`. We must avoid counting the same substring multiple times.

The problem guarantees that the string contains only lowercase letters and has at least one character, so we do not need to handle empty strings or non-alphabetic input.

## Approaches

The **brute-force approach** is to generate every possible way to split the string into substrings, track which substrings have already been used, and compute the maximum count among valid splits. This can be implemented using recursion: for each position in the string, try splitting it at every possible next index and recursively process the remaining string, keeping track of used substrings in a set. This is correct because it explores every valid combination, but it is slow because the number of splits grows exponentially.

The **optimal approach** uses **backtracking with a hash set** to prune invalid splits early. Instead of generating all splits blindly, we maintain a set of substrings already used. At each step, we try all possible substrings starting from the current index. If a substring has already been used, we skip it. Otherwise, we add it to the set, recursively compute the maximum unique split count for the remaining string, and backtrack by removing the substring from the set. This ensures that we explore only valid unique combinations and still consider all possibilities, which is necessary to guarantee the maximum count.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Try all possible splits recursively, without pruning |
| Optimal (Backtracking + Set) | O(n * 2^n) | O(n) | Backtracking with pruning using a set of used substrings |

## Algorithm Walkthrough

1. Define a recursive function `backtrack(start, seen)` where `start` is the current index in the string and `seen` is a set of substrings already used.
2. If `start` reaches the end of the string, return `0` because no more substrings are left to count.
3. Initialize a variable `max_count` to 0 to track the maximum number of substrings from the current state.
4. Loop through the string from `start` to the end, extracting the substring `s[start:i+1]`.
5. If the substring is not in `seen`, add it to `seen` and recursively call `backtrack(i+1, seen)` to compute the number of substrings for the remaining string. Add 1 to the result for the current substring.
6. Update `max_count` if this new count is larger.
7. Remove the substring from `seen` to backtrack and try the next possibility.
8. After the loop, return `max_count`.

Why it works: The algorithm explores all valid partitions of the string while enforcing uniqueness through the `seen` set. The recursion guarantees that all possibilities are considered, and backtracking ensures that substrings are only counted once per path.

## Python Solution

```python
class Solution:
    def maxUniqueSplit(self, s: str) -> int:
        def backtrack(start: int, seen: set[str]) -> int:
            if start == len(s):
                return 0
            max_count = 0
            for i in range(start, len(s)):
                sub = s[start:i+1]
                if sub not in seen:
                    seen.add(sub)
                    max_count = max(max_count, 1 + backtrack(i+1, seen))
                    seen.remove(sub)
            return max_count
        
        return backtrack(0, set())
```

The `backtrack` function recursively explores all possible unique substring splits. At each index, it attempts all substrings starting at that index, only continuing if the substring has not been used. The set `seen` ensures uniqueness, and `max_count` accumulates the best possible result. After each recursive call, the substring is removed from the set to allow exploration of other combinations.

## Go Solution

```go
func maxUniqueSplit(s string) int {
    var backtrack func(start int, seen map[string]bool) int
    backtrack = func(start int, seen map[string]bool) int {
        if start == len(s) {
            return 0
        }
        maxCount := 0
        for i := start; i < len(s); i++ {
            sub := s[start:i+1]
            if !seen[sub] {
                seen[sub] = true
                count := 1 + backtrack(i+1, seen)
                if count > maxCount {
                    maxCount = count
                }
                delete(seen, sub)
            }
        }
        return maxCount
    }
    return backtrack(0, make(map[string]bool))
}
```

In Go, the `map[string]bool` is used instead of a set to track used substrings. The algorithm otherwise mirrors the Python version. The `delete(seen, sub)` step handles backtracking.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * 2^n) | Each index can split in multiple ways; the number of unique splits grows exponentially. The factor `n` comes from substring creation. |
| Space | O(n) | Maximum recursion depth is n, and `seen` set can store up to n substrings. |

Because `n` is at most 16, this approach is feasible in practice.

## Edge Cases

One important edge case is when all characters are identical, such as `"aaaa"`. Here, the algorithm must recognize that no further splitting is possible beyond the first character, and the correct answer is 1. Another edge case is a string with completely unique characters, such as `"abcdef"`, which allows splitting each character individually. A third edge case involves repeated patterns interspersed with unique characters, such as `"abac"`, where careful tracking of the `seen` set ensures that substrings like `"a"` are not reused. The backtracking algorithm naturally handles these cases because it always checks for uniqueness in the set before choosing a substring.
