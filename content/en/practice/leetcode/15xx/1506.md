## Solution

## Problem Understanding

The problem asks us to identify the root of an N-ary tree given an **unordered array of all nodes**. Each node contains a unique value and a list of children, but the nodes are presented in arbitrary order. The key is that the array contains every node exactly once, so the root is included, but the array does not explicitly indicate parent-child relationships beyond each node's own `children` list.

In other words, we must find the node that is **never listed as a child** of any other node, because by definition, the root of a tree has no parent. The input may have thousands of nodes, so an efficient approach is needed.

Constraints emphasize that the tree size can reach up to `5 * 10^4` nodes and all node values are unique. This rules out naive nested loops over all nodes to check parenthood because it would be too slow for large inputs.

Important edge cases include:

- A tree with **only one node**, which is trivially the root.
- Nodes appearing in **random order**, meaning the first element in the array is not necessarily the root.
- Nodes with **no children** at all, which must not be confused with the root.

## Approaches

A brute-force approach would compare each node against every other node to see if it appears as a child. This would guarantee correctness because the root is the only node never referenced as a child, but it requires `O(n^2)` time, which is impractical for large `n`.

The key observation for an optimal solution is based on the **sum of unique values** or **bit manipulation**. If we compute the sum of all node values and subtract the sum of all children values, we are left with the value of the root, since every non-root node appears exactly once as a child. Similarly, using XOR across all values also isolates the root's value. This approach achieves **linear time and constant space** relative to the number of nodes.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Compare each node to all children to find the one never listed as a child |
| Optimal | O(n) | O(1) | Use sum or XOR of all node values minus all children values to identify root |

## Algorithm Walkthrough

1. Initialize a variable, `total`, to zero. This will track the XOR (or sum) of all node values and children values.
2. Iterate through every node in the input array and **add its value** to `total`.
3. For each node, iterate through its children and **subtract their values** (or XOR) from `total`.
4. After processing all nodes, `total` will equal the value of the root node, since every other node has been added and subtracted exactly once.
5. Iterate through the node array again to **find the node with the value equal to `total`**.
6. Return that node as the root.

Why it works: every non-root node appears once as a child, so its value is cancelled out. The root never appears as a child, so its value remains.

## Python Solution

```python
"""
# Definition for a Node.
class Node:
    def __init__(self, val: Optional[int] = None, children: Optional[List['Node']] = None):
        self.val = val
        self.children = children if children is not None else []
"""

from typing import List, Optional

class Solution:
    def findRoot(self, tree: List['Node']) -> 'Node':
        xor_total = 0
        
        for node in tree:
            xor_total ^= node.val
            for child in node.children:
                xor_total ^= child.val
                
        for node in tree:
            if node.val == xor_total:
                return node
```

The Python code first computes the XOR of all node values and children values, leaving the root's value. It then searches the array for the node with that value and returns it.

## Go Solution

```go
/**
 * Definition for a Node.
 * type Node struct {
 *     Val int
 *     Children []*Node
 * }
 */

func findRoot(tree []*Node) *Node {
    xorTotal := 0
    
    for _, node := range tree {
        xorTotal ^= node.Val
        for _, child := range node.Children {
            xorTotal ^= child.Val
        }
    }
    
    for _, node := range tree {
        if node.Val == xorTotal {
            return node
        }
    }
    
    return nil
}
```

In Go, we use the same XOR logic. We iterate over all nodes and their children to isolate the root's value. Go handles slices of nodes, and we return `nil` if no root is found, although the problem guarantees one exists.

## Edge Cases

One important edge case is a **tree with only one node**. The algorithm handles this because the XOR of the single value is itself, so it correctly identifies it as the root.

Another edge case is a **tree with deep nesting but unordered nodes**. Because we do not rely on the order and use XOR or sum, the algorithm correctly identifies the root regardless of order.

A third edge case is a tree where some nodes have **no children**. These nodes are properly canceled in XOR sums, so they do not affect identification of the root. This ensures that leaf nodes do not mistakenly become the root.
