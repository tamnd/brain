## Solution

## Problem Understanding

The problem requires implementing a **bidirectional iterator** over a binary search tree (BST) that supports in-order traversal. In simple terms, a BST has the property that for any node, all values in its left subtree are smaller, and all values in its right subtree are larger. An in-order traversal of a BST yields values in **ascending order**.

The iterator should support four operations:

1. `next()` - Move the pointer one step forward in the in-order traversal and return the value.
2. `prev()` - Move the pointer one step backward in the in-order traversal and return the value.
3. `hasNext()` - Check if a forward step is possible.
4. `hasPrev()` - Check if a backward step is possible.

The pointer is initialized to a "virtual" position before the smallest element. This ensures that the first `next()` call returns the smallest value. The problem guarantees that any call to `next()` or `prev()` is valid, which simplifies boundary checks.

**Constraints**:

- Up to 100,000 nodes in the tree.
- Node values are non-negative integers up to 1,000,000.
- Up to 100,000 method calls.
- These constraints indicate that the solution must be efficient in both time and space.

**Edge cases** to consider include trees with a single node, trees that are completely unbalanced (like a linked list), and repeated calls to `prev()` or `next()` at the boundaries.

## Approaches

The **brute-force approach** is to perform a full in-order traversal of the BST during initialization, store the values in an array, and then use an index pointer to navigate forward and backward. This guarantees correct behavior, but it uses **O(n)** extra space and requires O(n) time upfront to traverse the tree.

The **optimal approach** can avoid precomputing all nodes if memory is constrained, by using a stack to simulate in-order traversal incrementally. However, supporting `prev()` efficiently requires either storing visited nodes or maintaining a double-ended structure like a list. Given the problem allows storing values, the simplest and most efficient approach for practical use is to precompute the in-order list and use an index pointer.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force / Precompute | O(n) initialization, O(1) per `next`/`prev` | O(n) | Traverse BST once and store in a list; pointer moves along list |
| Optimal / Stack Simulation | O(1) amortized `next`, O(n) worst-case `prev` | O(h) for stack | Avoid storing all values; harder to support efficient `prev` without additional storage |

For this problem, the **precompute approach** is straightforward, simple to implement, and meets the problem constraints.

## Algorithm Walkthrough

1. Perform an **in-order traversal** of the BST recursively or iteratively to generate a sorted list of node values.
2. Initialize a pointer `index` at `-1`, representing the virtual position before the smallest element.
3. `next()` increments the pointer by one and returns the value at `values[index]`.
4. `prev()` decrements the pointer by one and returns the value at `values[index]`.
5. `hasNext()` checks if `index + 1 < len(values)` to determine if a forward step exists.
6. `hasPrev()` checks if `index - 1 >= 0` to determine if a backward step exists.

**Why it works**: The list maintains the exact in-order sequence, and the index pointer accurately represents the iterator's current position. Incrementing or decrementing the pointer simulates moving forward or backward in the traversal.

## Python Solution

```python
from typing import Optional, List

# Definition for a binary tree node.
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

class BSTIterator:

    def __init__(self, root: Optional[TreeNode]):
        self.values: List[int] = []
        self.index: int = -1
        self._inorder(root)
    
    def _inorder(self, node: Optional[TreeNode]):
        if not node:
            return
        self._inorder(node.left)
        self.values.append(node.val)
        self._inorder(node.right)
    
    def hasNext(self) -> bool:
        return self.index + 1 < len(self.values)
    
    def next(self) -> int:
        self.index += 1
        return self.values[self.index]
    
    def hasPrev(self) -> bool:
        return self.index - 1 >= 0
    
    def prev(self) -> int:
        self.index -= 1
        return self.values[self.index]
```

**Implementation Explanation**: The constructor builds the in-order list using `_inorder`. The `index` pointer tracks the current position, and `next()` and `prev()` simply move the pointer and return the corresponding value. The `hasNext()` and `hasPrev()` methods are simple boundary checks.

## Go Solution

```go
type BSTIterator struct {
    values []int
    index  int
}

func Constructor(root *TreeNode) BSTIterator {
    it := BSTIterator{values: []int{}, index: -1}
    var inorder func(node *TreeNode)
    inorder = func(node *TreeNode) {
        if node == nil {
            return
        }
        inorder(node.Left)
        it.values = append(it.values, node.Val)
        inorder(node.Right)
    }
    inorder(root)
    return it
}

func (this *BSTIterator) HasNext() bool {
    return this.index+1 < len(this.values)
}

func (this *BSTIterator) Next() int {
    this.index++
    return this.values[this.index]
}

func (this *BSTIterator) HasPrev() bool {
    return this.index-1 >= 0
}

func (this *BSTIterator) Prev() int {
    this.index--
    return this.values[this.index]
}
```

**Go-specific notes**: Go slices are used for dynamic arrays, similar to Python lists. Nil checks handle absent children. Index boundaries ensure safe forward/backward movement.

1. **Single-node tree**: The iterator must correctly handle `next()` and `prev()` with a single value. Initialization at `-1` ensures `next()` returns the single node and `prev()` correctly identifies the boundary.
2. **Completely unbalanced tree**: A tree that resembles a linked list, either all left or all right. The iterator still performs correctly
