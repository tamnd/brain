## Solution

## Problem Understanding

The problem is asking us to clean and aggregate sales data from a `Sales` table where `product_name` may contain inconsistent capitalization and leading or trailing whitespace. Each row in the table represents a single sale with a unique `sale_id`, a `product_name`, and the `sale_date`.

The goal is to produce a report that groups sales by **cleaned product name** (all lowercase and trimmed of spaces) and **month** (formatted as `'YYYY-MM'`), counting the total number of sales for each product per month. The output must be sorted alphabetically by `product_name` and then chronologically by `sale_date`.

Important points include:

- `product_name` is case-insensitive, so `LCPhone`, `LcPhone`, and `lcphone` should all be treated as the same product.
- Leading and trailing spaces must be removed, otherwise identical products with extra spaces would be treated differently.
- Sales must be aggregated by month, not by day, so the `sale_date` should be truncated to `'YYYY-MM'`.
- The output is ordered first by `product_name` ascending, then by `sale_date` ascending.

Edge cases to consider:

- Products with the same name but different capitalization or extra spaces.
- Months with no sales (though these won't appear in the result table).
- Single product sold multiple times in a month.
- The `Sales` table is non-empty, but the problem does not specify maximum table size, so a solution must be efficient in grouping and sorting.

## Approaches

The brute-force approach would involve fetching all rows from the table, cleaning the product names in application code, grouping by month and product name, and then counting the occurrences. This works but is inefficient because it requires transferring all data to the application layer and manually handling aggregation.

The optimal solution leverages SQL string functions and aggregation. By using `LOWER()` to normalize case and `TRIM()` to remove spaces, and `DATE_FORMAT()` (or equivalent) to extract the month, we can group and count directly in the database. This reduces data movement and uses the database’s optimized aggregation mechanisms.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Fetch all data, clean and group in application code, slow for large tables |
| Optimal | O(n) | O(n) | Use SQL aggregation and string/date functions to clean, group, and count in database efficiently |

## Algorithm Walkthrough

1. Select `product_name` from the `Sales` table.
2. Apply `TRIM()` to remove leading and trailing spaces and `LOWER()` to normalize the case.
3. Format `sale_date` to `'YYYY-MM'` to aggregate by month.
4. Use `GROUP BY` on the cleaned `product_name` and the formatted month to count total sales per product per month.
5. Alias the aggregated count as `total`.
6. Use `ORDER BY` to sort the results alphabetically by `product_name` and then chronologically by `sale_date`.

**Why it works:** By cleaning the product name and extracting the month directly in SQL, we ensure that all identical products are grouped together, and monthly aggregation is accurate. SQL aggregation guarantees correct counts per group, and sorting ensures the desired output order.

## Python Solution

```python
# SQL solution implemented in Python for LeetCode submission
# LeetCode executes SQL directly, so the Python class just returns the query

class Solution:
    def fixNames(self) -> str:
        return """
        SELECT
            LOWER(TRIM(product_name)) AS product_name,
            DATE_FORMAT(sale_date, '%Y-%m') AS sale_date,
            COUNT(*) AS total
        FROM Sales
        GROUP BY product_name, sale_date
        ORDER BY product_name ASC, sale_date ASC;
        """
```

**Explanation:** The query selects the cleaned product name using `LOWER(TRIM(product_name))`, extracts the month from `sale_date` using `DATE_FORMAT`, counts the occurrences for each group, and orders the result by `product_name` and `sale_date`. This follows the algorithm steps exactly and avoids application-level processing.

## Edge Cases

**Products with only whitespace:** If a product name is just spaces, `TRIM()` converts it to an empty string. SQL still groups it correctly under an empty product name, avoiding errors.

**Case-insensitive duplicates:** Products with identical names but different capitalization must be treated as one. `LOWER()` ensures that aggregation counts all variations correctly.

**Single sale or month:** A month with only one sale should appear in the output with `total = 1`. The aggregation and `COUNT(*)` correctly handle this, preventing missing data or incorrect counts.
