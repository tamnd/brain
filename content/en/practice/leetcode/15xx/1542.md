**Topics:** Hash Table, String, Bit Manipulation

## Solution

## Problem Understanding

The problem asks us to find the length of the longest substring of a digit string that can be rearranged into a palindrome.

A palindrome is a string that reads the same forward and backward. When rearrangement is allowed, the exact order of characters inside the substring does not matter. What matters is the frequency of each digit.

A string can be rearranged into a palindrome if:

- Every character appears an even number of times, or
- Exactly one character appears an odd number of times

This property comes from how palindromes are structured. Every character on the left side must match a character on the right side. Only the center character, in the case of odd-length palindromes, may appear an odd number of times.

The input string contains only digits from `'0'` to `'9'`. Since there are only 10 possible characters, we can efficiently track whether each digit appears an even or odd number of times.

The goal is not to construct the palindrome. We only need the maximum length of a substring that satisfies the palindrome rearrangement condition.

The constraints are important:

- `1 <= s.length <= 10^5`
- Only digits are present

A quadratic solution that checks every substring would require checking roughly `10^10` substrings in the worst case, which is far too slow. We need a solution close to linear time.

Several edge cases are important:

- A single character is always awesome because a one-character string is already a palindrome.
- Strings where all digits are unique can only produce substrings of length `1`.
- Entire strings may already satisfy the palindrome condition.
- Long strings require efficient state tracking instead of repeated frequency counting.

## Approaches

### Brute Force Approach

The most direct solution is to examine every possible substring.

For each substring:

1. Count the frequency of each digit.
2. Determine how many digits have odd frequencies.
3. If at most one digit has an odd count, the substring is awesome.
4. Track the maximum valid length.

This approach is correct because it explicitly checks every candidate substring against the palindrome rearrangement condition.

However, it is too slow for the given constraints.

There are `O(n^2)` substrings. For each substring, counting frequencies requires up to `O(n)` time if done naively, or `O(10)` time with prefix frequency arrays. Even with optimization, the total complexity remains `O(n^2)`, which is too large for `n = 10^5`.

### Optimal Approach

The key observation is that we only care about whether counts are odd or even, not the exact counts.

This naturally suggests using a bitmask.

We maintain a 10-bit integer:

- Bit `i` represents whether digit `i` has appeared an odd number of times in the current prefix.
- `0` means even count.
- `1` means odd count.

As we scan the string:

- Encountering a digit flips its corresponding bit using XOR.
- The current mask represents the parity state of digit frequencies.

Now consider two prefix masks:

- `prefixMask[i]`
- `prefixMask[j]`

The substring between them has parity state:

```
prefixMask[i] XOR prefixMask[j]
```

A substring can form a palindrome if:

- All counts are even, meaning XOR result is `0`
- Exactly one count is odd, meaning XOR result has exactly one bit set

Therefore, for each position we need to check:

1. Whether the current mask has appeared before
2. Whether any mask differing by one bit has appeared before

Because there are only `2^10 = 1024` possible masks, this approach is extremely efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) or O(n) | Checks every substring explicitly |
| Optimal | O(n × 10) | O(1024) | Uses parity bitmask and earliest occurrence tracking |

## Algorithm Walkthrough

1. Initialize a hash map called `first_seen`.

This map stores the earliest index where each parity mask appears. We set:

```
first_seen[0] = -1
```

Mask `0` means all digit counts are even before processing any characters.
2. Initialize:

- `mask = 0`
- `max_length = 0`

The mask will track odd/even frequency parity while scanning the string.
3. Iterate through the string character by character.

For each digit:

- Convert it to an integer.
- Flip its corresponding bit using XOR.

Example:

```
mask ^= (1 << digit)
```
4. Check if the current mask has appeared before.

If it has, then the substring between the earlier index and current index has all even counts.

Compute:

```
current_length = i - first_seen[mask]
```

Update the maximum length.
5. Check masks differing by exactly one bit.

A valid palindrome rearrangement may contain exactly one odd count.

For every digit `0` through `9`:

- Flip one bit from the current mask.
- Check whether that modified mask has appeared before.

Example:

```
candidate = mask ^ (1 << bit)
```

If this candidate exists in `first_seen`, then the substring has exactly one odd-frequency digit.
6. Store the earliest occurrence of the current mask.

We only store the first occurrence because earlier positions produce longer substrings.
7. Continue until the entire string has been processed.
8. Return `max_length`.

### Why it works

The algorithm relies on prefix parity states.

For any substring, the parity of digit frequencies equals the XOR difference between two prefix masks.

A substring can be rearranged into a palindrome if its parity state contains:

- No odd counts, mask `0`
- Exactly one odd count, mask with one set bit

By storing the earliest occurrence of each parity mask, we can instantly determine the longest valid substring ending at the current position.

Because there are only 1024 possible masks, the solution runs efficiently even for very large inputs.

## Python Solution

```python
class Solution:
    def longestAwesome(self, s: str) -> int:
        first_seen = {0: -1}

        mask = 0
        max_length = 0

        for i, ch in enumerate(s):
            digit = int(ch)

            mask ^= (1 << digit)

            if mask in first_seen:
                max_length = max(max_length, i - first_seen[mask])

            for bit in range(10):
                candidate = mask ^ (1 << bit)

                if candidate in first_seen:
                    max_length = max(
                        max_length,
                        i - first_seen[candidate]
                    )

            if mask not in first_seen:
                first_seen[mask] = i

        return max_length
```

The implementation directly follows the algorithm described earlier.

The dictionary `first_seen` records the earliest index where each parity mask appears. Initializing `0: -1` handles substrings starting at index `0`.

The variable `mask` tracks the parity state of digit frequencies. Each digit flips its corresponding bit using XOR.

For every position, the algorithm first checks whether the exact same mask has been seen before. This corresponds to substrings where all digit counts are even.

Next, it checks all masks differing by one bit. Each such mask represents a substring with exactly one odd-frequency digit.

Finally, the algorithm stores the earliest occurrence of the current mask. Keeping the earliest index ensures maximum substring length.

## Go Solution

```go
func longestAwesome(s string) int {
    firstSeen := map[int]int{
        0: -1,
    }

    mask := 0
    maxLength := 0

    for i := 0; i < len(s); i++ {
        digit := int(s[i] - '0')

        mask ^= (1 << digit)

        if prevIndex, exists := firstSeen[mask]; exists {
            length := i - prevIndex
            if length > maxLength {
                maxLength = length
            }
        }

        for bit := 0; bit < 10; bit++ {
            candidate := mask ^ (1 << bit)

            if prevIndex, exists := firstSeen[candidate]; exists {
                length := i - prevIndex
                if length > maxLength {
                    maxLength = length
                }
            }
        }

        if _, exists := firstSeen[mask]; !exists {
            firstSeen[mask] = i
        }
    }

    return maxLength
}
```

The Go implementation mirrors the Python solution closely.

Go strings are byte arrays, so digits are accessed using `s[i]`. Conversion to integer digit values is done with:

```go
digit := int(s[i] - '0')
```

The hash map uses `map[int]int` to store earliest indices. Since Go does not provide a built-in `max` function for integers, comparisons are written manually.

Integer overflow is not a concern because masks use only 10 bits.

## Edge Cases

### Single Character Strings

A one-character substring is always a palindrome. This case is important because the minimum input length is `1`. A buggy implementation might incorrectly require all counts to be even.

The algorithm handles this naturally because any mask with exactly one set bit is considered valid.

### Entire String Is Awesome

Some inputs allow the entire string to be rearranged into a palindrome. For example:

```
"213123"
```

All digit counts are even. The algorithm correctly detects this when the current mask matches an earlier prefix mask.

The initialization:

```
first_seen[0] = -1
```

is essential for handling substrings that begin at index `0`.

### All Digits Unique

Inputs like:

```
"123456789"
```

contain only odd counts with no repetitions.

A naive implementation might incorrectly assume larger substrings are valid. The bitmask method correctly identifies that more than one odd count exists, so only length `1` substrings qualify.

### Repeated Same Digit

Inputs such as:

```
"000000"
```

can easily expose parity tracking bugs.

The mask repeatedly flips the same bit:

```
0 -> 1 -> 0 -> 1 -> 0
```

The algorithm correctly recognizes long valid substrings whenever the same mask reappears.

### Earliest Occurrence Requirement

If we overwrite previously stored indices for a mask, we may lose the longest possible substring.

For example, suppose the same mask appears at indices `2` and `10`. Keeping the earliest occurrence allows future substrings to maximize their length.

This is why the implementation only inserts a mask into `first_seen` if it has not appeared before.
