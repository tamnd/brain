## Solution

## Problem Understanding

The problem gives us an integer array `nums` and an integer `target`. We must find the maximum number of non-empty subarrays whose sum equals `target`, with the additional restriction that the chosen subarrays cannot overlap.

A subarray is a contiguous portion of the array. Two subarrays overlap if they share at least one index. Our goal is not to find all valid subarrays, but instead to select as many non-overlapping valid subarrays as possible.

For example, suppose we have:

```
nums = [1,1,1,1,1]
target = 2
```

There are multiple subarrays whose sum is `2`:

```
[1,1] at indices [0,1]
[1,1] at indices [1,2]
[1,1] at indices [2,3]
[1,1] at indices [3,4]
```

However, we cannot take overlapping ones. The best possible choice is:

```
[0,1] and [2,3]
```

So the answer is `2`.

The constraints are important:

- `nums.length` can be as large as `100000`
- Elements may be negative
- `target` can be zero

Because negative numbers are allowed, classic sliding window techniques do not work reliably. Sliding windows depend on monotonic behavior of sums, which disappears once negative values are introduced.

The large input size also tells us that quadratic solutions are too slow. We need something close to linear time.

Several edge cases are especially important:

- Arrays containing negative numbers
- Multiple overlapping valid subarrays
- `target = 0`
- Arrays where every element individually equals the target
- Long ranges whose sums equal the target
- Cases where greedily taking the earliest valid subarray matters

The problem guarantees at least one element in the array, so we never deal with empty input.

## Approaches

### Brute Force Approach

The most direct solution is to examine every possible subarray.

We can generate all subarrays by choosing a start index `i` and an end index `j`, then compute the sum of `nums[i:j+1]`. Whenever the sum equals `target`, we record that interval.

After collecting all valid intervals, we would need to determine the maximum number of non-overlapping intervals we can choose.

This brute force method is correct because it explicitly checks every possible subarray. However, it is far too slow.

There are `O(n^2)` subarrays in total. Even if prefix sums reduce sum computation to `O(1)`, we still must examine all pairs `(i, j)`.

With `n = 100000`, an `O(n^2)` solution is infeasible.

### Optimal Approach, Prefix Sum + Greedy

The key observation is that we do not actually need to store every valid subarray.

Instead, we process the array from left to right and greedily take the earliest valid subarray whenever possible.

Why does this work?

Suppose we find a valid subarray ending at the current position. Taking it immediately is always optimal because earlier finishing subarrays leave more room for future non-overlapping subarrays.

To efficiently detect whether a subarray ending at the current index sums to `target`, we use prefix sums.

If:

```
current_prefix_sum - previous_prefix_sum = target
```

then:

```
previous_prefix_sum = current_prefix_sum - target
```

So while scanning the array, we store previously seen prefix sums in a hash set.

Whenever we detect a valid subarray:

- We increment the answer
- We reset the prefix tracking structure

Resetting is important because once we commit to a subarray, future subarrays must start after it.

This produces a linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n²) | Checks every subarray and resolves overlaps afterward |
| Optimal | O(n) | O(n) | Uses prefix sums, hash set, and greedy interval selection |

## Algorithm Walkthrough

### Step 1: Initialize Tracking Variables

We maintain:

- `prefix_sum`, the running cumulative sum
- `seen_prefix`, a hash set storing prefix sums seen since the last chosen subarray
- `count`, the number of non-overlapping valid subarrays found

Initially:

```
prefix_sum = 0
count = 0
seen_prefix = {0}
```

The initial `0` allows detection of subarrays that start at index `0`.

### Step 2: Iterate Through the Array

We scan the array from left to right.

For each number:

```
prefix_sum += num
```

This gives the cumulative sum up to the current position.

### Step 3: Check Whether a Valid Subarray Exists

We want to know whether there exists a previous prefix sum such that:

```
prefix_sum - previous_prefix = target
```

Rearranging:

```
previous_prefix = prefix_sum - target
```

So we check:

```
if prefix_sum - target in seen_prefix
```

If true, then some subarray ending at the current index sums to `target`.

### Step 4: Greedily Take the Subarray

Once we find a valid subarray, we immediately count it.

```
count += 1
```

Then we reset:

```
seen_prefix = {0}
prefix_sum = 0
```

This reset ensures all future subarrays begin after the current one, preventing overlap.

Greedily taking the earliest ending valid subarray is optimal because it maximizes the remaining space for future subarrays.

### Step 5: Otherwise Store the Current Prefix Sum

If no valid subarray is found, we add the current prefix sum to the set:

```
seen_prefix.add(prefix_sum)
```

This allows future positions to form valid subarrays.

### Step 6: Return the Final Count

After processing the entire array, `count` contains the maximum number of non-overlapping target-sum subarrays.

### Why it works

The algorithm maintains the invariant that `seen_prefix` only contains prefix sums from the current active segment, meaning after the most recently selected subarray.

Whenever a valid subarray is found, choosing it immediately is optimal because it ends as early as possible. Any later-ending overlapping choice would leave less room for future subarrays and therefore cannot improve the total count.

Because every chosen interval is greedily minimized in ending position, the algorithm always maximizes the total number of non-overlapping subarrays.

## Python Solution

```python
from typing import List

class Solution:
    def maxNonOverlapping(self, nums: List[int], target: int) -> int:
        prefix_sum = 0
        count = 0
        
        seen_prefix = {0}
        
        for num in nums:
            prefix_sum += num
            
            if prefix_sum - target in seen_prefix:
                count += 1
                
                prefix_sum = 0
                seen_prefix = {0}
            else:
                seen_prefix.add(prefix_sum)
        
        return count
```

The implementation follows the algorithm directly.

We begin by initializing the running prefix sum, the answer counter, and a hash set containing `0`. The initial `0` is necessary so that subarrays starting from the beginning of the current segment can be detected.

As we iterate through `nums`, we continuously update the running sum.

At each step, we check whether:

```python
prefix_sum - target
```

already exists in the hash set. If it does, then a valid subarray ending at the current index has been found.

Once a valid subarray is discovered, we increment the answer and reset both the prefix sum and the hash set. This reset is the key greedy step that prevents future overlaps.

If no valid subarray exists at the current position, we store the current prefix sum for future checks.

The algorithm processes each element once, making it highly efficient for large inputs.

## Go Solution

```go
package main

func maxNonOverlapping(nums []int, target int) int {
    prefixSum := 0
    count := 0

    seenPrefix := map[int]bool{
        0: true,
    }

    for _, num := range nums {
        prefixSum += num

        if seenPrefix[prefixSum-target] {
            count++

            prefixSum = 0
            seenPrefix = map[int]bool{
                0: true,
            }
        } else {
            seenPrefix[prefixSum] = true
        }
    }

    return count
}
```

The Go implementation mirrors the Python solution closely.

Go does not have a built-in set type, so we use `map[int]bool` to simulate a hash set.

Resetting the state after finding a valid subarray is done by creating a fresh map containing only `0`.

Integer overflow is not a concern here because the maximum possible cumulative sum remains safely within Go's `int` range under the given constraints.

## Edge Cases

One important edge case is when `target = 0`. Zero-sum subarrays are especially tricky because repeated prefix sums indicate valid ranges. A careless implementation may accidentally count overlapping zero-sum intervals multiple times. The reset step in this solution prevents that problem by ensuring each chosen interval is finalized before searching for another.

Another important case involves negative numbers. Many array problems involving sums can be solved with sliding windows, but negative numbers break the monotonic properties required for sliding windows to work correctly. Using prefix sums and a hash set avoids this issue entirely because the method depends only on arithmetic relationships between prefix sums.

A third critical edge case occurs when many small valid subarrays exist back-to-back. For example:

```
nums = [2,2,2]
target = 2
```

The optimal answer is `3`. A non-greedy strategy might accidentally choose larger intervals and reduce the total count. This implementation immediately commits to the earliest finishing valid subarray, which guarantees the maximum number of non-overlapping selections.

Another subtle edge case is when the entire array itself forms one valid subarray. Since the hash set initially contains `0`, the algorithm correctly detects subarrays beginning at index `0`, ensuring whole-array solutions are handled naturally.
