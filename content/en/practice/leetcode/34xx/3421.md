## Solution

## Problem Understanding

This is a SQL database problem that asks us to identify student-subject pairs where the student's performance improved over time.

The `Scores` table contains exam results. Each row represents a student's score in a specific subject on a specific exam date. The primary key is `(student_id, subject, exam_date)`, which guarantees that a student can have at most one score for a given subject on a particular date.

A student is considered to have improved in a subject only if both of the following conditions are satisfied:

1. The student took at least two exams in the same subject.
2. The score on the latest exam date is strictly greater than the score on the earliest exam date.

For every qualifying `(student_id, subject)` pair, we must return:

- `student_id`
- `subject`
- `first_score` (score on the earliest exam date)
- `latest_score` (score on the latest exam date)

The results must be sorted by `student_id` and `subject` in ascending order.

The key observation is that intermediate exam scores do not matter. Even if a student's score fluctuates between exams, the only values that affect the answer are the score on the first exam date and the score on the latest exam date.

Some important edge cases include students with only one exam in a subject, students whose scores decreased over time, students whose first and latest scores are equal, and students with more than two exams where only the earliest and latest records matter.

## Approaches

### Brute Force Approach

A brute force solution would process each `(student_id, subject)` group independently. For every group, we could gather all exam records, sort them by date, identify the first and last entries, and then compare their scores.

This approach is correct because sorting guarantees that we can identify the earliest and latest exam dates. Once those dates are known, determining improvement is straightforward.

However, repeatedly sorting records for every student-subject group is more expensive than necessary. Since SQL provides powerful window functions that can directly identify the first and last rows within each partition, we can avoid explicit sorting logic at the application level.

### Key Insight

The problem only requires the earliest and latest exam for each `(student_id, subject)` combination.

Window functions are ideal here because they allow us to:

- Partition rows by `(student_id, subject)`
- Order rows by `exam_date`
- Assign row numbers from the beginning and end
- Identify the first and last records directly

Once we know the first and last records, we can join them together and compare their scores. We also naturally exclude subjects with only one exam because the first and last row would be the same record.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N log N) | O(N) | Sort each student-subject group to find first and last exams |
| Optimal | O(N log N) | O(N) | Uses window functions to identify earliest and latest records efficiently |

## Algorithm Walkthrough

### Optimal SQL Algorithm

1. Partition the table by `(student_id, subject)` and order rows by `exam_date` ascending. Assign a row number to each row. The row with number `1` represents the earliest exam.
2. Partition the table again by `(student_id, subject)` and order rows by `exam_date` descending. Assign another row number. The row with number `1` represents the latest exam.
3. Extract all rows whose ascending row number is `1`. These are the first exams for each student-subject pair.
4. Extract all rows whose descending row number is `1`. These are the latest exams for each student-subject pair.
5. Join the first-exam rows with the latest-exam rows using `(student_id, subject)`.
6. Filter out cases where the first and latest exam dates are identical. Such groups contain only one exam and therefore do not satisfy the problem requirements.
7. Keep only rows where `latest_score > first_score`.
8. Return the required columns and sort by `student_id` and `subject`.

### Why it works

For every `(student_id, subject)` partition, ordering by date ascending guarantees that row number `1` corresponds to the earliest exam. Ordering by date descending guarantees that row number `1` corresponds to the latest exam. Therefore, comparing these two rows exactly captures the problem definition. Since every partition is processed independently, all student-subject pairs are handled correctly.

```python
WITH ranked AS (
    SELECT
        student_id,
        subject,
        score,
        exam_date,
        ROW_NUMBER() OVER (
            PARTITION BY student_id, subject
            ORDER BY exam_date
        ) AS first_rn,
        ROW_NUMBER() OVER (
            PARTITION BY student_id, subject
            ORDER BY exam_date DESC
        ) AS last_rn
    FROM Scores
),
first_exam AS (
    SELECT
        student_id,
        subject,
        score AS first_score,
        exam_date AS first_date
    FROM ranked
    WHERE first_rn = 1
),
last_exam AS (
    SELECT
        student_id,
        subject,
        score AS latest_score,
        exam_date AS latest_date
    FROM ranked
    WHERE last_rn = 1
)
SELECT
    f.student_id,
    f.subject,
    f.first_score,
    l.latest_score
FROM first_exam f
JOIN last_exam l
    ON f.student_id = l.student_id
   AND f.subject = l.subject
WHERE l.latest_date > f.first_date
  AND l.latest_score > f.first_score
ORDER BY f.student_id, f.subject;
```

The query begins by assigning two row numbers to every record. One identifies the earliest exam within each student-subject group, and the other identifies the latest exam. Two common table expressions then isolate the first and latest records. After joining these records together, the query removes groups containing only one exam and keeps only those where the latest score exceeds the first score. Finally, the results are sorted according to the problem requirements.

## Edge Cases

### Student Has Only One Exam

A student may have taken only a single exam in a subject. Such a student does not satisfy the requirement of having exams on at least two different dates. A common bug is accidentally comparing the same row against itself and treating it as both first and latest. The solution avoids this by requiring the latest exam date to be later than the first exam date.

### Latest Score Equals First Score

The problem requires improvement, meaning the latest score must be strictly greater than the first score. Using `>=` instead of `>` would incorrectly include students whose performance remained unchanged. The query explicitly checks `latest_score > first_score`.

### More Than Two Exams

A student may have many exams in the same subject. The problem does not ask whether scores generally increased over time. It only compares the earliest and latest exams. For example, scores of `70 -> 95 -> 80` should still count as improvement because `80 > 70`. The window-function approach correctly selects only the first and last records.

### Intermediate Score Fluctuations

Scores may rise and fall throughout the sequence of exams. A naive solution might compare consecutive exams or calculate an average trend. The problem definition ignores all intermediate values. By extracting only the earliest and latest exams, the solution precisely matches the required behavior.

### Multiple Subjects Per Student

A student can take exams in several subjects. Improvement in one subject should not affect another subject. Partitioning by both `student_id` and `subject` guarantees complete independence between subject histories.
