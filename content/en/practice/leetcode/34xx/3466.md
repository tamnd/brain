## Solution

## Problem Understanding

This problem asks us to calculate the **maximum coins Mario can collect** while driving on a two-lane freeway. The freeway is represented by two integer arrays, `lane1` and `lane2`, where the value at index `i` corresponds to the coins Mario gains or loses at mile `i` in that lane. A positive number represents coins gained, while a negative number represents a toll or loss.

Mario **must start in lane 1**, can switch lanes **at most twice**, and must drive for at least one mile. He can enter at the beginning or skip some initial negative miles, and he can exit early if it maximizes his coins. The output is a single integer: the maximum coins he can collect under these constraints.

The constraints indicate that arrays can be as long as $10^5$ and values can be as extreme as $10^9$ in magnitude. This rules out any brute-force solution that tries all possible lane switch positions. Important edge cases include arrays of length 1, arrays with all negative numbers, and arrays where the optimal solution involves switching lanes immediately or at the end.

## Approaches

The **brute-force approach** would attempt every combination of lane switches up to two times and every possible starting and ending mile. For each candidate sequence of lane segments, we would sum the coins and track the maximum. This guarantees correctness because it considers all feasible sequences of switches, but it is **too slow**: with $n$ miles, there are $O(n^3)$ ways to choose switch positions, leading to a time complexity of $O(n^3)$, which is infeasible for $n$ up to $10^5$.

The key insight for an **optimal solution** is that the maximum coins for a sequence of at most two switches can be computed using **prefix sums** and **dynamic programming**. Instead of enumerating all switch positions, we maintain the maximum coins Mario can collect for each number of lane switches so far and update them as we iterate through the miles. We only need to track three states: the maximum coins with 0 switches (staying in lane 1), 1 switch (lane 1 → lane 2 or lane 2 → lane 1), and 2 switches. At each mile, we update these states based on whether Mario stays in the current lane or switches.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(1) | Try every possible sequence of lane switches and sum coins |
| Optimal | O(n) | O(1) | Track maximum coins for 0, 1, or 2 switches using dynamic programming |

## Algorithm Walkthrough

1. **Initialize states**: We track the maximum coins Mario can collect ending at the current mile with 0, 1, or 2 lane switches. Start with `dp0` for 0 switches in lane 1, `dp1` for 1 switch in lane 2, and `dp2` for 2 switches back in lane 1. Initialize all to negative infinity except `dp0` for the first mile in lane 1.
2. **Iterate through miles**: For each mile, update the maximum coins for each state. Staying in the same lane extends the current state, while switching lanes increases the switch count by 1. Specifically, `dp0` adds coins from lane1, `dp1` is the max of staying in lane2 or switching from dp0, and `dp2` is the max of staying in lane1 or switching from dp1.
3. **Track the global maximum**: Since Mario can exit anytime, maintain a variable `max_coins` that stores the largest value among `dp0`, `dp1`, and `dp2` at each mile.
4. **Return result**: After iterating through all miles, `max_coins` holds the maximum coins Mario can collect under the constraints.

**Why it works**: By maintaining the three states, we ensure that we correctly account for all possibilities of lane switching up to two times. The states respect the constraints, and at each mile, we update only using valid transitions, guaranteeing the optimal solution.

## Python Solution

```python
from typing import List

class Solution:
    def maxCoins(self, lane1: List[int], lane2: List[int]) -> int:
        n = len(lane1)
        dp0 = float('-inf')  # Max coins with 0 switches, in lane1
        dp1 = float('-inf')  # Max coins with 1 switch, in lane2
        dp2 = float('-inf')  # Max coins with 2 switches, back in lane1
        max_coins = float('-inf')
        
        for i in range(n):
            if i == 0:
                dp0 = lane1[0]
                dp1 = dp2 = float('-inf')
            else:
                new_dp0 = dp0 + lane1[i]
                new_dp1 = max(dp1 + lane2[i], dp0 + lane2[i])
                new_dp2 = max(dp2 + lane1[i], dp1 + lane1[i])
                dp0, dp1, dp2 = new_dp0, new_dp1, new_dp2
            
            max_coins = max(max_coins, dp0, dp1, dp2)
        
        return max_coins
```

The code iterates through each mile and updates three states corresponding to the number of lane switches. `dp0` always extends lane 1, `dp1` either continues in lane2 or switches from lane1, and `dp2` either continues in lane1 after one switch or switches from lane2. `max_coins` tracks the best result so far.

## Go Solution

```go
func maxCoins(lane1 []int, lane2 []int) int64 {
    n := len(lane1)
    dp0 := int64(lane1[0])
    dp1 := int64(-1 << 60)
    dp2 := int64(-1 << 60)
    maxCoins := dp0
    
    for i := 1; i < n; i++ {
        newDp0 := dp0 + int64(lane1[i])
        newDp1 := max(dp1+int64(lane2[i]), dp0+int64(lane2[i]))
        newDp2 := max(dp2+int64(lane1[i]), dp1+int64(lane1[i]))
        dp0, dp1, dp2 = newDp0, newDp1, newDp2
        maxCoins = max(maxCoins, dp0, dp1, dp2)
    }
    
    return maxCoins
}

func max(a, b int64, c ...int64) int64 {
    res := a
    if b > res {
        res = b
    }
    for _, v := range c {
        if v > res {
            res = v
        }
    }
    return res
}
```

Go handles large negative numbers and potential integer overflow by using `int64`. The logic mirrors the Python version, updating the three states for each mile.

## Complexity Analysis

| Measure | Complexity | Explanation |

The algorithm is highly efficient due to the small, fixed state space and avoids nested loops.
