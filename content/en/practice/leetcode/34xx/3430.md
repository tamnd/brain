# LeetCode 3430 - Maximum and Minimum Sums of at Most Size K Subarrays

## Problem Understanding

We are given an integer array `nums` and an integer `k`. For every contiguous subarray whose length is at most `k`, we compute:

- The minimum element of that subarray.
- The maximum element of that subarray.
- Their sum.

The final answer is the sum of these values across all valid subarrays.

In other words, for every subarray `nums[l:r]` where:

$$1 \le (r-l+1) \le k$$

we add:

$$\min(nums[l:r]) + \max(nums[l:r])$$

to the result.

The array length can be as large as `80,000`, which immediately rules out any solution that explicitly enumerates all subarrays. The total number of subarrays is approximately:

$$\frac{n(n+1)}{2}$$

which is more than 3 billion when `n = 80,000`.

The constraints also allow negative values, so we cannot rely on monotonicity of sums or any special property of positive numbers.

The key challenge is therefore:

- Count each element's contribution as a minimum.
- Count each element's contribution as a maximum.
- Restrict the counted subarrays to lengths at most `k`.

Important edge cases include arrays of length `1`, arrays with all equal values, strictly increasing or decreasing arrays, negative numbers, and situations where `k = 1` or `k = n`.

## Approaches

### Brute Force

A straightforward solution is to generate every subarray whose length is at most `k`.

For each starting position, extend the subarray up to length `k`, tracking the current minimum and maximum. Add their sum to the answer.

This approach is correct because every valid subarray is examined exactly once and its minimum and maximum are computed.

Unfortunately, the number of subarrays can be:

$$O(nk)$$

and when both `n` and `k` are `80,000`, this becomes approximately 6.4 billion operations, which is far too slow.

### Key Insight

Instead of iterating over subarrays, we count how many subarrays use a particular element as their minimum or maximum.

This is a classic monotonic stack technique.

For each index `i`:

- Determine the range of starts and ends where `nums[i]` is the minimum.
- Count only those subarrays whose length does not exceed `k`.
- Multiply that count by `nums[i]`.

Do the same for maximums.

The only extra difficulty compared to the standard monotonic stack contribution problem is the length restriction.

Without a length restriction, the number of subarrays where index `i` is the chosen minimum is:

$$L \times R$$

where:

- `L` = choices for the left boundary.
- `R` = choices for the right boundary.

With a maximum length `k`, we must count only pairs satisfying:

$$leftChoice + rightChoice - 1 \le k$$

This counting can still be done in constant time using arithmetic formulas.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(nk) | O(1) | Enumerates all valid subarrays |
| Optimal | O(n) | O(n) | Monotonic stacks plus contribution counting |

## Algorithm Walkthrough

### 1. Compute contribution ranges for minimums

For every index `i`, determine:

- The previous strictly smaller element.
- The next smaller-or-equal element.

This tie-breaking ensures every subarray minimum is assigned exactly once.

Let:

- `left = i - prevSmaller`
- `right = nextSmallerEqual - i`

Then every subarray in which `nums[i]` is the designated minimum corresponds to choosing:

- A left extension between `1` and `left`.
- A right extension between `1` and `right`.

### 2. Compute contribution ranges for maximums

Similarly determine:

- Previous strictly greater element.
- Next greater-or-equal element.

Again this guarantees unique ownership of every maximum.

Let:

- `left = i - prevGreater`
- `right = nextGreaterEqual - i`

### 3. Count valid pairs under the length constraint

Suppose:

- `a` is chosen from `[1, left]`
- `b` is chosen from `[1, right]`

The resulting subarray length is:

$$a+b-1$$

We need:

$$a+b-1 \le k$$

or:

$$a+b \le k+1$$

Therefore we must count pairs:

$$(a,b)$$

such that:

$$1 \le a \le left$$

$$1 \le b \le right$$

$$a+b \le k+1$$

### 4. Count these pairs in O(1)

For each valid `a`, the largest allowed `b` is:

$$\min(right,\; k+1-a)$$

Summing this directly would be too slow.

Observe that the values form arithmetic sequences. The sum can be split into:

- A region where `b = right`.
- A region where `b = k+1-a`.

Using arithmetic progression formulas, the total number of valid pairs can be computed in constant time.

### 5. Add minimum contributions

For every index:

- Compute the number of valid subarrays where it acts as minimum.
- Multiply by `nums[i]`.
- Add to the minimum total.

### 6. Add maximum contributions

Repeat the same process using the maximum boundaries.

### 7. Return the combined answer

The final result is:

$$\text{sumMin} + \text{sumMax}$$

### Why it works

The monotonic stack boundaries partition all subarrays uniquely among indices. Every valid subarray contributes exactly one designated minimum index and exactly one designated maximum index. The counting formula determines how many of those subarrays also satisfy the length constraint. Therefore every valid subarray contributes its minimum and maximum exactly once, producing the correct total.

## Python Solution

```python
from typing import List

class Solution:
    def minMaxSubarraySum(self, nums: List[int], k: int) -> int:
        n = len(nums)

        def count_pairs(left: int, right: int) -> int:
            limit = k + 1

            max_a = min(left, limit - 1)
            if max_a <= 0:
                return 0

            threshold = limit - right

            full_count = min(max_a, threshold)
            if full_count < 0:
                full_count = 0

            result = full_count * right

            start = full_count + 1
            if start <= max_a:
                cnt = max_a - start + 1

                first = limit - start
                last = limit - max_a

                result += cnt * (first + last) // 2

            return result

        def contribution_as_min() -> int:
            prev = [-1] * n
            stack = []

            for i in range(n):
                while stack and nums[stack[-1]] > nums[i]:
                    stack.pop()
                prev[i] = stack[-1] if stack else -1
                stack.append(i)

            nxt = [n] * n
            stack = []

            for i in range(n - 1, -1, -1):
                while stack and nums[stack[-1]] >= nums[i]:
                    stack.pop()
                nxt[i] = stack[-1] if stack else n
                stack.append(i)

            total = 0

            for i in range(n):
                left = i - prev[i]
                right = nxt[i] - i
                total += nums[i] * count_pairs(left, right)

            return total

        def contribution_as_max() -> int:
            prev = [-1] * n
            stack = []

            for i in range(n):
                while stack and nums[stack[-1]] < nums[i]:
                    stack.pop()
                prev[i] = stack[-1] if stack else -1
                stack.append(i)

            nxt = [n] * n
            stack = []

            for i in range(n - 1, -1, -1):
                while stack and nums[stack[-1]] <= nums[i]:
                    stack.pop()
                nxt[i] = stack[-1] if stack else n
                stack.append(i)

            total = 0

            for i in range(n):
                left = i - prev[i]
                right = nxt[i] - i
                total += nums[i] * count_pairs(left, right)

            return total

        return contribution_as_min() + contribution_as_max()
```

The implementation consists of three major pieces.

The `count_pairs` function computes how many valid subarrays correspond to a given `(left, right)` span while respecting the length limit `k`. Instead of iterating through all choices, it uses arithmetic progression formulas to compute the answer in constant time.

The `contribution_as_min` function builds monotonic increasing stacks to locate the previous smaller and next smaller-or-equal boundaries. These boundaries uniquely determine all subarrays for which a given index serves as the minimum.

The `contribution_as_max` function performs the symmetric operation using decreasing stacks. The final answer is the sum of both contribution totals.

## Go Solution

```go
func minMaxSubarraySum(nums []int, k int) int64 {
	n := len(nums)

	countPairs := func(left, right int) int64 {
		limit := k + 1

		maxA := left
		if maxA > limit-1 {
			maxA = limit - 1
		}

		if maxA <= 0 {
			return 0
		}

		threshold := limit - right

		fullCount := maxA
		if fullCount > threshold {
			fullCount = threshold
		}
		if fullCount < 0 {
			fullCount = 0
		}

		var result int64 = int64(fullCount * right)

		start := fullCount + 1

		if start <= maxA {
			cnt := maxA - start + 1

			first := limit - start
			last := limit - maxA

			result += int64(cnt*(first+last)) / 2
		}

		return result
	}

	calcMin := func() int64 {
		prev := make([]int, n)
		stack := make([]int, 0)

		for i := 0; i < n; i++ {
			for len(stack) > 0 && nums[stack[len(stack)-1]] > nums[i] {
				stack = stack[:len(stack)-1]
			}

			if len(stack) == 0 {
				prev[i] = -1
			} else {
				prev[i] = stack[len(stack)-1]
			}

			stack = append(stack, i)
		}

		next := make([]int, n)
		stack = stack[:0]

		for i := n - 1; i >= 0; i-- {
			for len(stack) > 0 && nums[stack[len(stack)-1]] >= nums[i] {
				stack = stack[:len(stack)-1]
			}

			if len(stack) == 0 {
				next[i] = n
			} else {
				next[i] = stack[len(stack)-1]
			}

			stack = append(stack, i)
		}

		var total int64

		for i := 0; i < n; i++ {
			left := i - prev[i]
			right := next[i] - i

			total += int64(nums[i]) * countPairs(left, right)
		}

		return total
	}

	calcMax := func() int64 {
		prev := make([]int, n)
		stack := make([]int, 0)

		for i := 0; i < n; i++ {
			for len(stack) > 0 && nums[stack[len(stack)-1]] < nums[i] {
				stack = stack[:len(stack)-1]
			}

			if len(stack) == 0 {
				prev[i] = -1
			} else {
				prev[i] = stack[len(stack)-1]
			}

			stack = append(stack, i)
		}

		next := make([]int, n)
		stack = stack[:0]

		for i := n - 1; i >= 0; i-- {
			for len(stack) > 0 && nums[stack[len(stack)-1]] <= nums[i] {
				stack = stack[:len(stack)-1]
			}

			if len(stack) == 0 {
				next[i] = n
			} else {
				next[i] = stack[len(stack)-1]
			}

			stack = append(stack, i)
		}

		var total int64

		for i := 0; i < n; i++ {
			left := i - prev[i]
			right := next[i] - i

			total += int64(nums[i]) * countPairs(left, right)
		}

		return total
	}

	return calcMin() + calcMax()
}
```

The Go implementation mirrors the Python version exactly. The main difference is that all accumulated values are stored in `int64` to avoid overflow. Since the answer can be much larger than 32-bit integer limits, every contribution multiplication is explicitly converted to `int64`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each index is pushed and popped from each monotonic stack at most once |
| Space | O(n) | Boundary arrays and stacks require linear storage |

The critical observation is that the counting of valid subarrays for each index is performed in constant time. Therefore, after the monotonic stack boundaries are computed, each element contributes only O(1) additional work. The overall running time remains linear.

## Edge Cases

### All Elements Are Equal

Arrays such as `[7,7,7,7]` are a common source of bugs because duplicate values can cause double counting. The implementation carefully uses asymmetric inequalities when computing previous and next boundaries. For minimums it uses `>` on one side and `>=` on the other, and for maximums it uses `<` on one side and `<=` on the other. This guarantees every subarray is assigned to exactly one index.

### k = 1

When `k` equals `1`, only single-element subarrays are valid. Every element is simultaneously the minimum and maximum of its own subarray. The counting formula naturally reduces to one valid pair per index, producing:

$$2 \times \sum nums[i]$$

which is exactly correct.

### k = n

When `k` equals the array length, every subarray is valid. The length restriction becomes inactive. The counting formula still works because all admissible `(left,right)` pairs satisfy the constraint, and the result matches the classic monotonic-stack contribution method for total subarray minimums and maximums.

### Large Negative and Positive Values

Values may be as large as `10^6` in magnitude, and there can be up to `80,000` elements. The final answer can therefore become much larger than a 32-bit integer. The Go solution uses `int64` throughout accumulation, and Python naturally supports arbitrary precision integers, ensuring correctness without overflow.

### Strictly Increasing or Strictly Decreasing Arrays

These inputs produce the deepest possible monotonic stack structures and often reveal boundary mistakes. Since each index is pushed and popped at most once, the algorithm remains linear and computes the correct ownership ranges for every minimum and maximum contribution.
