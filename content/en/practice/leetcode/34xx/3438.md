## Solution

## Problem Understanding

The problem requires us to find a **valid pair of adjacent digits** in a string `s` where each digit in the pair appears in the string **exactly as many times as its numeric value**, and the two digits are different. In other words, we are scanning a string of digits and looking for the first two consecutive characters such that: the first digit is not equal to the second, the total count of the first digit in the string equals its numeric value, and the total count of the second digit equals its numeric value. If no such pair exists, we return an empty string.

The input `s` is guaranteed to contain only digits `'1'` through `'9'`, and its length is at least 2 but at most 100. This constraint is small enough that counting all digit frequencies upfront is efficient. Edge cases include strings where no valid pair exists, strings where all digits are the same, or strings where the digits' frequency never matches their numeric values.

Key observations include that we only need to count the frequency of each digit once, and then we can check **adjacent pairs** efficiently in a single pass. This is because the validation condition depends on **global frequency**, but the pair itself is **local adjacency**.

## Approaches

The brute-force approach would examine **every possible adjacent pair** and count the occurrences of each digit on the fly for every pair. This works correctly but is inefficient because counting the entire string for each pair results in redundant computation, leading to $O(n^2)$ time complexity. Given `n <= 100`, this is tolerable, but we can do better.

The optimal approach leverages a **hash map or fixed-size frequency array** to count all digits once, then iterates through adjacent pairs to check the validity condition. By precomputing the frequencies, the check for each pair becomes $O(1)$, reducing the overall time complexity to $O(n)$.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check every adjacent pair and count digits in the string each time |
| Optimal | O(n) | O(1) | Count all digits once and validate each adjacent pair in a single pass |

## Algorithm Walkthrough

1. Initialize an array `count` of size 10 (or a hash map) to store the frequency of each digit `'1'` through `'9'`.
2. Iterate through the string `s` and increment the corresponding `count[digit]` for each character.
3. Iterate through all **adjacent pairs** in `s` from left to right.
4. For each pair `(s[i], s[i+1])`, check if the digits are different.
5. If they are different, check whether the frequency of `s[i]` equals its numeric value and the frequency of `s[i+1]` equals its numeric value.
6. If both conditions hold, return the string `s[i] + s[i+1]`.
7. If no valid pair is found after checking all pairs, return an empty string.

**Why it works**: By precomputing frequencies, we guarantee that the frequency condition is checked correctly for each digit. Iterating left to right ensures that the **first valid adjacent pair** is returned.

## Python Solution

```python
class Solution:
    def findValidPair(self, s: str) -> str:
        count = [0] * 10  # Initialize frequency array for digits 1-9

        for char in s:
            digit = int(char)
            count[digit] += 1

        for i in range(len(s) - 1):
            a, b = int(s[i]), int(s[i+1])
            if a != b and count[a] == a and count[b] == b:
                return s[i] + s[i+1]

        return ""
```

In the Python implementation, we first build a frequency array where `count[digit]` stores how many times that digit appears. Then, we scan all adjacent pairs and check the required conditions. If both digits in the pair satisfy the frequency condition and are different, we immediately return the pair. Otherwise, we return an empty string if no valid pair exists.

## Go Solution

```go
func findValidPair(s string) string {
    count := [10]int{}

    for _, ch := range s {
        digit := int(ch - '0')
        count[digit]++
    }

    for i := 0; i < len(s)-1; i++ {
        a := int(s[i] - '0')
        b := int(s[i+1] - '0')
        if a != b && count[a] == a && count[b] == b {
            return s[i:i+2]
        }
    }

    return ""
}
```

The Go implementation mirrors the Python solution. We use a fixed-size array to store frequencies. The type conversion `ch - '0'` converts a byte to an integer. Substrings are taken using `s[i:i+2]` to return the valid pair.

## Edge Cases

1. **String with all identical digits**: For example, `"11111"`. This case could cause a naive implementation that only checks adjacency without checking frequency to incorrectly return a pair. Our implementation handles this by requiring both `a != b` and correct frequency counts.
2. **Multiple valid pairs**: For instance, `"11122333"` contains `"12"` and `"23"` as valid pairs. The algorithm correctly returns the **first valid pair** because it checks pairs in left-to-right order.
3. **Digits with frequency not matching their numeric value**: For example, `"123456789"` where each digit appears once. Although all digits are present, none satisfy `count[digit] == digit` except `1`. The implementation ensures both digits in a pair satisfy the frequency condition, so no false positives occur.

This solution efficiently handles all these edge cases while maintaining linear time complexity and constant space.
