## Solution

## Problem Understanding

This problem asks us to find the longest path in a binary tree where adjacent nodes differ by exactly `1`. The sequence may either increase or decrease.

The important detail is that the path does not need to move strictly from parent to child in one direction. The path may follow a child-parent-child pattern. In other words, the valid path may pass through a node and extend into both subtrees.

For example, in the tree:

```
    2
   / \
  1   3
```

the valid path is:

```
1 -> 2 -> 3
```

This path is consecutive because every adjacent pair differs by `1`.

The input is the root of a binary tree. Each node contains an integer value and pointers to left and right children. The output is a single integer representing the maximum length of any valid consecutive path.

The constraints are important:

- The tree can contain up to `3 * 10^4` nodes.
- Node values can range from `-3 * 10^4` to `3 * 10^4`.

A tree with thirty thousand nodes is large enough that inefficient repeated traversals will likely time out. This strongly suggests we need a linear time solution, ideally visiting each node only once.

Several edge cases are important:

- A tree with only one node should return `1`.
- Trees where no consecutive relationship exists should also return `1`.
- The longest path may pass through the root, or through any internal node.
- Increasing and decreasing sequences must both be considered.
- The sequence may switch direction at the middle node, such as `1 -> 2 -> 3`.

A naive implementation often fails because it only considers paths moving downward from parent to child, while the actual answer may require combining paths from both children through a common parent.

## Approaches

### Brute Force Approach

A brute force solution would attempt to start a traversal from every node and explore all possible consecutive paths.

For each node, we could recursively search for increasing and decreasing chains extending outward. Since the path may pass through a parent and continue into another subtree, we would need to repeatedly recompute path information for overlapping subtrees.

This approach is correct because it eventually examines every possible valid path in the tree. However, it performs a large amount of redundant work. Each subtree may be traversed many times from different starting points.

In the worst case, such as a skewed tree, the time complexity becomes quadratic.

### Key Insight for the Optimal Solution

The critical observation is that every node can serve as the middle point of a valid consecutive sequence.

At each node, we want to know:

- The longest increasing consecutive path starting from this node downward.
- The longest decreasing consecutive path starting from this node downward.

Once we know these two values for the left and right children, we can combine them at the current node.

For example:

```
1 <- 2 -> 3
```

The node `2` connects a decreasing chain from the left and an increasing chain from the right.

Instead of recomputing information repeatedly, we compute these values once during a postorder DFS traversal.

This allows us to process the entire tree in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(h) | Repeatedly explores overlapping subtrees |
| Optimal | O(n) | O(h) | Single DFS traversal computes all needed information |

Here:

- `n` is the number of nodes.
- `h` is the height of the tree.

## Algorithm Walkthrough

### Optimal DFS Strategy

We perform a depth first traversal where each recursive call returns two values:

- `increasing`, the length of the longest increasing path starting at the current node.
- `decreasing`, the length of the longest decreasing path starting at the current node.

The traversal is postorder because we must process children before computing the current node's values.

### Step-by-Step Process

1. Start a DFS traversal from the root.
2. For each node, recursively compute results for the left and right children.
3. Initialize:

- `increasing = 1`
- `decreasing = 1`

Every node alone forms a valid path of length `1`.
4. Check the left child:

- If `left.val == node.val + 1`, then the current node can extend an increasing path through the left child.
- If `left.val == node.val - 1`, then the current node can extend a decreasing path through the left child.
5. Check the right child using the same logic.
6. After processing both children:

- `increasing` stores the longest increasing chain downward from the current node.
- `decreasing` stores the longest decreasing chain downward from the current node.
7. Combine the two directions:

- A valid full path through the current node has length:

```
increasing + decreasing - 1
```

We subtract `1` because the current node is counted twice.

1. Update the global maximum answer.
2. Return `(increasing, decreasing)` to the parent.

### Why it works

The algorithm works because every valid consecutive path has a highest point, or turning point, in the tree. When DFS processes a node, it already knows the best increasing and decreasing chains available from its children. Combining these chains produces the longest valid path passing through that node. Since every node is considered once as a potential center, the algorithm examines all possible valid paths.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional, Tuple

class Solution:
    def longestConsecutive(self, root: Optional[TreeNode]) -> int:
        self.best = 0

        def dfs(node: Optional[TreeNode]) -> Tuple[int, int]:
            if not node:
                return (0, 0)

            left_inc, left_dec = dfs(node.left)
            right_inc, right_dec = dfs(node.right)

            increasing = 1
            decreasing = 1

            if node.left:
                if node.left.val == node.val + 1:
                    increasing = max(increasing, left_inc + 1)
                elif node.left.val == node.val - 1:
                    decreasing = max(decreasing, left_dec + 1)

            if node.right:
                if node.right.val == node.val + 1:
                    increasing = max(increasing, right_inc + 1)
                elif node.right.val == node.val - 1:
                    decreasing = max(decreasing, right_dec + 1)

            self.best = max(self.best, increasing + decreasing - 1)

            return (increasing, decreasing)

        dfs(root)

        return self.best
```

The implementation follows the postorder DFS strategy directly.

The recursive helper function returns two integers:

- The longest increasing chain starting at the current node.
- The longest decreasing chain starting at the current node.

The base case returns `(0, 0)` for null nodes.

For every real node, we begin with both path lengths equal to `1` because the node itself forms a valid sequence.

We then examine both children. If a child value differs by exactly `1`, we extend either the increasing or decreasing chain appropriately.

The most important step is updating the global answer using:

```
increasing + decreasing - 1
```

This combines both directions into a single path passing through the current node.

Finally, the DFS returns the computed pair upward so the parent can continue building larger sequences.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

func longestConsecutive(root *TreeNode) int {
    best := 0

    var dfs func(node *TreeNode) (int, int)

    dfs = func(node *TreeNode) (int, int) {
        if node == nil {
            return 0, 0
        }

        leftInc, leftDec := dfs(node.Left)
        rightInc, rightDec := dfs(node.Right)

        increasing := 1
        decreasing := 1

        if node.Left != nil {
            if node.Left.Val == node.Val+1 {
                increasing = max(increasing, leftInc+1)
            } else if node.Left.Val == node.Val-1 {
                decreasing = max(decreasing, leftDec+1)
            }
        }

        if node.Right != nil {
            if node.Right.Val == node.Val+1 {
                increasing = max(increasing, rightInc+1)
            } else if node.Right.Val == node.Val-1 {
                decreasing = max(decreasing, rightDec+1)
            }
        }

        best = max(best, increasing+decreasing-1)

        return increasing, decreasing
    }

    dfs(root)

    return best
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

The Go implementation closely mirrors the Python version.

Go does not support returning tuples directly in the same way Python does, but multiple return values make this straightforward.

Nil pointers are explicitly checked before accessing child fields. Integer overflow is not a concern because path lengths never exceed the number of nodes.

## Worked Examples

### Example 1

Input:

```
    1
   / \
  2   3
```

Expected output:

```

```

### DFS Traversal State

| Node | Increasing | Decreasing | Combined Path | Global Best |
| --- | --- | --- | --- | --- |
| 2 | 1 | 1 | 1 | 1 |
| 3 | 1 | 1 | 1 | 1 |
| 1 | 2 | 1 | 2 | 2 |

Explanation:

- Node `2` has no children, so both values are `1`.
- Node `3` also produces `(1,1)`.
- At node `1`:

- Left child `2 = 1 + 1`, so increasing becomes `2`.
- Right child `3` is not consecutive with difference `1`.
- Longest path becomes `2`.

### Example 2

Input:

```
    2
   / \
  1   3
```

Expected output:

```
3
```

### DFS Traversal State

| Node | Increasing | Decreasing | Combined Path | Global Best |
| --- | --- | --- | --- | --- |
| 1 | 1 | 1 | 1 | 1 |
| 3 | 1 | 1 | 1 | 1 |
| 2 | 2 | 2 | 3 | 3 |

Explanation:

- Left child `1 = 2 - 1`, so decreasing becomes `2`.
- Right child `3 = 2 + 1`, so increasing becomes `2`.
- Combined path:

```
1 -> 2 -> 3
```

Length:

```
2 + 2 - 1 = 3
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is processed exactly once |
| Space | O(h) | Recursive call stack height equals tree height |

The DFS visits every node a single time and performs only constant work per node.

The auxiliary space comes from recursion depth. In a balanced tree this is `O(log n)`, while in the worst case of a completely skewed tree it becomes `O(n)`.

## Edge Cases

### Single Node Tree

A tree containing only one node is the smallest valid input. Many implementations accidentally return `0` because they initialize the answer incorrectly or fail to count the node itself as a valid path. This implementation initializes both increasing and decreasing lengths to `1`, ensuring every node contributes at least a path of length `1`.

### Longest Path Passes Through an Internal Node

The most common mistake is only considering downward paths. The actual longest sequence may combine a decreasing chain from one child and an increasing chain from another child. For example:

```
1 -> 2 -> 3
```

The implementation correctly handles this by combining:

```
increasing + decreasing - 1
```

at every node.

### Duplicate or Non-Consecutive Values

Nodes with equal values should not extend a sequence because the difference must be exactly `1`. Similarly, values differing by more than `1` must be ignored. The implementation explicitly checks:

```
child.val == node.val + 1
```

and

```
child.val == node.val - 1
```

Only exact matches extend a path.
