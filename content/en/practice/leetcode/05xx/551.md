## Solution

## Problem Understanding

The problem gives us a string `s` where each character represents a student's attendance on a particular day. The possible characters are:

- `'A'` for absent
- `'L'` for late
- `'P'` for present

We must determine whether the student qualifies for an attendance award. The student only qualifies if both conditions are satisfied:

1. The total number of absences (`'A'`) is strictly less than 2, meaning the student can have at most one absence.
2. The student never has 3 or more consecutive late days (`'L'`).

The input is a single string, and the output is a boolean value:

- `true` if the student qualifies for the award
- `false` otherwise

The constraints are small:

- `1 <= s.length <= 1000`
- Every character is guaranteed to be one of `'A'`, `'L'`, or `'P'`

Since the string length is at most 1000, even an `O(n^2)` solution would technically run within limits. However, this problem is naturally solvable in a single linear scan, so an `O(n)` solution is preferred and expected.

Several edge cases are important to consider:

- A string with exactly one absence should still return `true`.
- A string with two absences should immediately fail.
- Three consecutive `'L'` characters anywhere in the string should fail.
- Late streaks separated by another character should reset correctly.
- Very short strings such as `"A"` or `"LL"` should be handled properly.
- The input is guaranteed to contain only valid attendance characters, so no input validation is necessary.

## Approaches

A brute-force approach would examine the string multiple times. One pass could count the number of absences, while another pass checks every possible substring of length 3 to see whether it equals `"LLL"`.

This approach works because the award conditions are independent:

- Count all `'A'` characters
- Detect any occurrence of 3 consecutive `'L'` characters

Although correct, this solution performs redundant work because it traverses the string more than necessary.

The key insight for the optimal approach is that both conditions can be checked simultaneously during a single traversal of the string. While iterating through the characters, we can maintain:

- A running count of absences
- A running count of consecutive late days

Whenever we encounter:

- `'A'`, increment the absence count and reset the late streak
- `'L'`, increment the late streak
- `'P'`, reset the late streak

The moment the absence count reaches 2 or the late streak reaches 3, we can immediately return `false`.

This allows us to solve the problem in one pass with constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Multiple scans of the string |
| Optimal | O(n) | O(1) | Single-pass scan tracking absences and late streaks |

## Algorithm Walkthrough

1. Initialize two variables:

- `absence_count = 0`
- `late_streak = 0`

The first variable tracks how many times the student was absent. The second tracks the current number of consecutive late days.
2. Traverse the string character by character.

We process each attendance record exactly once.
3. If the current character is `'A'`:

- Increment `absence_count`
- Reset `late_streak` to `0`

This reset is necessary because an absence breaks any sequence of consecutive late days.
4. If the current character is `'L'`:

- Increment `late_streak`

Consecutive late days must be counted continuously.
5. If the current character is `'P'`:

- Reset `late_streak` to `0`

A present day also breaks a late streak.
6. After updating the counters, check the failure conditions:

- If `absence_count >= 2`, return `False`
- If `late_streak >= 3`, return `False`

We can terminate early because the student is already disqualified.
7. If the loop finishes without violating either rule, return `True`.

### Why it works

The algorithm maintains two invariants throughout the traversal:

- `absence_count` always equals the total number of `'A'` characters seen so far.
- `late_streak` always equals the current number of consecutive trailing `'L'` characters.

Because every character updates these values correctly, the algorithm detects violations immediately when they occur. If the scan completes without triggering either failure condition, the attendance record necessarily satisfies both award requirements.

## Python Solution

```
class Solution:
    def checkRecord(self, s: str) -> bool:
        absence_count = 0
        late_streak = 0

        for char in s:
            if char == 'A':
                absence_count += 1
                late_streak = 0
            elif char == 'L':
                late_streak += 1
            else:
                late_streak = 0

            if absence_count >= 2:
                return False

            if late_streak >= 3:
                return False

        return True
```

The implementation closely follows the algorithm described above.

The variable `absence_count` keeps track of how many absences have occurred so far. Each time the algorithm encounters `'A'`, this counter increases by one.

The variable `late_streak` tracks consecutive late days. Whenever the algorithm encounters `'L'`, the streak increases. Encountering either `'A'` or `'P'` resets the streak because the sequence of consecutive late days has been interrupted.

The two conditional checks inside the loop allow early termination. As soon as the student becomes ineligible, the function immediately returns `False` instead of processing unnecessary remaining characters.

If the loop completes successfully, neither condition was violated, so the function returns `True`.

## Go Solution

```
func checkRecord(s string) bool {
    absenceCount := 0
    lateStreak := 0

    for _, char := range s {
        if char == 'A' {
            absenceCount++
            lateStreak = 0
        } else if char == 'L' {
            lateStreak++
        } else {
            lateStreak = 0
        }

        if absenceCount >= 2 {
            return false
        }

        if lateStreak >= 3 {
            return false
        }
    }

    return true
}
```

The Go implementation mirrors the Python logic almost exactly.

One small language-specific detail is that iterating over a string with `range` in Go returns Unicode code points of type `rune`. Since the attendance characters are ASCII characters, direct comparisons like `char == 'A'` work correctly.

No additional handling is needed for empty strings because the problem guarantees the string length is at least 1.

## Edge Cases

One important edge case is a record containing exactly two absences, such as `"PAAP"`. A common bug is checking whether absences exceed 2 instead of checking whether they are greater than or equal to 2. The implementation correctly returns `False` as soon as `absence_count >= 2`.

Another important case is when three consecutive late days appear at the end of the string, such as `"PPALLL"`. Some implementations only check streaks when the streak ends, which can miss violations at the end of traversal. This implementation checks the streak immediately after incrementing it, ensuring the violation is detected instantly.

A third edge case involves interrupted late streaks, such as `"LLPLL"`. Although there are four total late days, there are never three consecutive late days. Incorrect implementations may accidentally track total late days instead of consecutive late days. This solution correctly resets `late_streak` whenever a non-`'L'` character appears, preventing false failures.
