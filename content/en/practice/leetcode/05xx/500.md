## Problem Understanding

This problem asks us to identify words from a list that can be typed using letters from **only one row of an American keyboard**. The keyboard has three distinct rows: the top row `"qwertyuiop"`, the middle row `"asdfghjkl"`, and the bottom row `"zxcvbnm"`. Each input word may use both uppercase and lowercase letters, but the case is **ignored**, so `"A"` is treated the same as `"a"`.

The input `words` is an array of strings where each string consists of only English letters. The goal is to return all words where every character belongs to a single row. For example, `"Dad"` is valid because all its letters belong to the second row, but `"Hello"` is invalid because it spans multiple rows.

Constraints tell us that the input size is small: at most 20 words, and each word is at most 100 characters. This means even a simple algorithm iterating over characters will perform efficiently, but we still want a clean solution. Important edge cases include single-character words, words that contain letters from multiple rows, and case-insensitivity. The problem guarantees all words are non-empty and contain only letters, so we do not need to handle numbers, symbols, or empty strings.

## Approaches

A brute-force approach is straightforward. For each word, we can iterate through every character and check which row it belongs to. If all characters belong to the same row, we include it in the result. This works correctly but requires checking each character against each row set, which is slightly inefficient.

The optimal approach leverages a **hash map** (or dictionary) that maps each letter to its corresponding row number. With this mapping, we can determine the row of each character in **O(1)** time. For each word, we just check if all characters map to the same row. This approach reduces repeated row membership checks and is conceptually simpler and faster in practice.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m * r) | O(r) | n = number of words, m = average word length, r = 3 rows; check each character against each row set |
| Optimal | O(n * m) | O(26) | Use a letter-to-row mapping to check row membership efficiently |

## Algorithm Walkthrough

1. Create a hash map (dictionary) that maps each lowercase letter to its row index (1, 2, or 3). For example, `'q' -> 1`, `'a' -> 2`, `'z' -> 3`.
2. Initialize an empty list `result` to store valid words.
3. Iterate over each word in the input list.
4. Convert the first character of the word to lowercase and find its row index using the hash map. This row index becomes the reference row for the word.
5. For each subsequent character in the word, convert it to lowercase and check if its row index matches the reference row. If any character belongs to a different row, break and skip this word.
6. If all characters belong to the same row, append the word to `result`.
7. After processing all words, return the `result` list.

**Why it works**: The algorithm works because it enforces a single-row constraint for each word. By using a hash map, we can check membership in constant time, ensuring every word in the result satisfies the problem condition.

## Python Solution

```
from typing import List

class Solution:
    def findWords(self, words: List[str]) -> List[str]:
        # Map each letter to its keyboard row
        row_map = {}
        for char in "qwertyuiop":
            row_map[char] = 1
        for char in "asdfghjkl":
            row_map[char] = 2
        for char in "zxcvbnm":
            row_map[char] = 3
        
        result = []
        
        for word in words:
            lower_word = word.lower()
            first_row = row_map[lower_word[0]]
            valid = True
            for ch in lower_word:
                if row_map[ch] != first_row:
                    valid = False
                    break
            if valid:
                result.append(word)
        
        return result
```

The Python implementation first builds a mapping from letters to rows for constant-time lookup. Then, for each word, it checks if all letters belong to the same row by comparing to the first letter's row. If the word is valid, it is added to the result.

## Go Solution

```
func findWords(words []string) []string {
    rowMap := make(map[rune]int)
    for _, ch := range "qwertyuiop" {
        rowMap[ch] = 1
    }
    for _, ch := range "asdfghjkl" {
        rowMap[ch] = 2
    }
    for _, ch := range "zxcvbnm" {
        rowMap[ch] = 3
    }
    
    result := []string{}
    
    for _, word := range words {
        firstRow := rowMap[rune(toLower(word[0]))]
        valid := true
        for _, ch := range word {
            if rowMap[rune(toLower(byte(ch)))] != firstRow {
                valid = false
                break
            }
        }
        if valid {
            result = append(result, word)
        }
    }
    return result
}

func toLower(ch byte) byte {
    if ch >= 'A' && ch <= 'Z' {
        return ch + 32
    }
    return ch
}
```

In Go, we use a map from rune to row index and handle case conversion manually because Go does not have built-in string `.lower()` for bytes. The algorithm otherwise mirrors the Python approach, using slices instead of lists.

## Edge Cases

One important edge case is **single-character words**. Since a single character trivially belongs to one row, the algorithm should include it automatically. The implementation handles this because the loop comparing letters never triggers a mismatch.

Another edge case is **mixed-case words**, such as `"AlAsKa"`. Without converting to lowercase, the algorithm could incorrectly identify row mismatches. We explicitly lowercase all letters before mapping them to rows.
