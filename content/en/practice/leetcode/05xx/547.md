## Solution

## Problem Understanding

The problem asks us to find the number of **provinces** in a network of cities. Each city may be directly connected to some other cities, as indicated by an `n x n` adjacency matrix `isConnected`. A value of `isConnected[i][j] = 1` indicates that city `i` and city `j` are directly connected, while `0` indicates no direct connection. Indirect connections are also relevant: if city `a` is connected to city `b` and city `b` is connected to city `c`, then city `a` is considered connected to city `c` as well. A **province** is a set of cities that are directly or indirectly connected to each other and not connected to any other cities outside this set.

The input is always a square matrix with size `n` where `1 <= n <= 200`. The matrix is symmetric because `isConnected[i][j] == isConnected[j][i]`, and the diagonal is always `1` because a city is always connected to itself. The expected output is a single integer representing the number of provinces.

Important considerations include isolated cities (which are provinces by themselves), fully connected groups, and ensuring we do not double-count connections. The constraints are modest enough to allow an `O(n^2)` approach.

## Approaches

### Brute Force

The brute-force approach treats each city as a starting point and tries to find all cities connected to it by iterating over the entire matrix repeatedly. One could mark visited cities and increment the province count each time a new unvisited city is found. This approach works correctly but becomes inefficient because it may repeatedly traverse already explored connections, leading to an `O(n^3)` time complexity.

### Optimal Approach

The key insight is that this problem can be modeled as **graph connectivity**, where cities are nodes and `1`s in the matrix indicate edges. Once modeled this way, finding provinces is equivalent to counting connected components in an undirected graph. Depth-First Search (DFS), Breadth-First Search (BFS), or Union-Find are all suitable for this problem.

Using DFS, we traverse each unvisited node, mark all reachable nodes as visited, and increment the province count. This guarantees that each connected component is counted exactly once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n) | Repeatedly searches for connected cities without using graph traversal efficiently |
| DFS / BFS / Union-Find | O(n^2) | O(n) | Efficient traversal; each edge is visited once, each node is processed once |

## Algorithm Walkthrough

1. Initialize a `visited` array of size `n` with all `False` values to track which cities have already been explored.
2. Initialize a `province_count` variable to zero.
3. Iterate through each city `i` from `0` to `n-1`. If `i` has not been visited, start a DFS from city `i`.
4. In the DFS, mark the current city as visited. For each city `j`, if `isConnected[i][j] == 1` and `j` is unvisited, recursively perform DFS on city `j`.
5. After DFS finishes for a starting city, increment the `province_count` by one.
6. Continue until all cities are visited.
7. Return the `province_count`.

**Why it works:** The DFS explores all cities connected to the starting city, marking them as visited. Since a province is a connected component, each DFS call accounts for exactly one province. The visited array ensures no double-counting occurs.

## Python Solution

```
from typing import List

class Solution:
    def findCircleNum(self, isConnected: List[List[int]]) -> int:
        def dfs(city: int):
            visited[city] = True
            for neighbor in range(n):
                if isConnected[city][neighbor] == 1 and not visited[neighbor]:
                    dfs(neighbor)
        
        n = len(isConnected)
        visited = [False] * n
        province_count = 0
        
        for city in range(n):
            if not visited[city]:
                dfs(city)
                province_count += 1
        
        return province_count
```

This implementation initializes the visited array and iterates through all cities. The inner DFS recursively marks all cities reachable from the current city. Once a DFS finishes, it signifies one complete province, so the counter is incremented.

## Go Solution

```
func findCircleNum(isConnected [][]int) int {
    n := len(isConnected)
    visited := make([]bool, n)
    
    var dfs func(int)
    dfs = func(city int) {
        visited[city] = true
        for neighbor := 0; neighbor < n; neighbor++ {
            if isConnected[city][neighbor] == 1 && !visited[neighbor] {
                dfs(neighbor)
            }
        }
    }
    
    provinceCount := 0
    for city := 0; city < n; city++ {
        if !visited[city] {
            dfs(city)
            provinceCount++
        }
    }
    
    return provinceCount
}
```

The Go version mirrors the Python approach with minor syntax differences. The DFS is defined as a function literal. A slice `visited` tracks visited cities. Otherwise, the logic is identical, ensuring each province is counted once.

**Example 1:**

```
Input: [[1,1,0],[1,1,0],[0,0,1]]
```

| Step | Current City | Visited Array | Action |
| --- | --- | --- | --- |
| 1 | 0 | [False, False, False] | Start DFS from 0 |
| 2 | 0 | [True, False, False] | Visit neighbor 1 |
| 3 | 1 | [True, True, False] | Neighbor 0 already visited |
| 4 | End DFS | [True, True, False] | Increment province count to 1 |
| 5 | 1 | Already visited | Skip |
| 6 | 2 | [True, True, False] | Start DFS from 2 |
| 7 | 2 | [True, True, True] | Increment province count to 2 |

Output: 2

**Example 2:**

```
Input: [[1,0,0],[0,1,0],[0,0,1]]
```

Each city is isolated, so each DFS increments the count separately. Output is 3.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n^2) | Each city checks all other cities, visiting each edge once |
| Space | O(n) | Visited array plus recursion stack up to n |

The algorithm efficiently traverses all connections once without repeated work, making `O(n^2)` feasible for `n <= 200`.

## Edge Cases

The first edge case is a **single city**, which is a province by itself. The algorithm handles it because DFS starts and immediately marks it visited.

The second edge case is **fully connected cities**, which should be counted as one province. The visited array prevents double-counting.

The third edge case is **disconnected subgraphs** with indirect connections. DFS ensures that even if cities are indirectly connected through multiple hops, they are counted as one province. Any isolated cities are handled naturally as DFS is triggered separately for unvisited cities.
