## Solution

## Problem Understanding

The problem gives an integer `n` and asks us to transform it into a string in a very specific way. Each decimal digit must first be converted into its lowercase English word representation. For example, the digit `4` becomes `"four"` and the digit `1` becomes `"one"`. These words are then concatenated in the same order as the digits appear in the original number.

After constructing the resulting string `s`, we must examine the frequency of every character that appears in it. The goal is to count how many distinct characters occur an odd number of times. A character contributes exactly once to the answer if its frequency is odd, regardless of how many times it actually appears.

For example, if `n = 41`, the digits become `"four"` and `"one"`, producing the string `"fourone"`. The character frequencies are:

- `f → 1`
- `o → 2`
- `u → 1`
- `r → 1`
- `n → 1`
- `e → 1`

Since five of these letters have odd frequencies, the answer is 5.

The constraint `1 <= n <= 10^9` tells us that the number contains at most ten digits. Therefore, even if we explicitly build the entire word string, its length remains very small. The problem is fundamentally a counting task, and efficiency is not a concern because the input size is tiny.

Several edge cases are worth noting. A number may contain repeated digits, which causes some words to appear multiple times. Different digit words can also share letters, such as `"one"` and `"zero"` both containing `'e'` and `'o'`. The input is guaranteed to be positive, so there is no need to handle negative numbers or a leading minus sign.

## Approaches

A straightforward brute-force approach is to convert every digit into its English word, concatenate all the words into one large string, and then count the frequency of every character with a hash map. After computing all frequencies, we iterate through the map and count how many characters have odd frequencies.

This approach is correct because every character in the final string is processed exactly once, so the frequency table accurately records how many times each character appears. Once the frequencies are known, determining whether each count is odd is immediate.

Because the maximum number of digits is only ten, the total string length is bounded by a small constant. Therefore, this method is already more than fast enough.

An alternative observation is that we do not actually need to store the concatenated string. As each digit word is generated, we can immediately update the character frequency table. This avoids constructing an intermediate string while producing the same frequency counts.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(L) | O(L) | Build the entire concatenated string and count characters afterward |
| Optimal | O(L) | O(1) | Update frequencies directly while processing digit words |

Here, `L` denotes the total number of letters across all digit words. Since there are at most ten digits, `L` is bounded by a small constant.

## Algorithm Walkthrough

1. Create an array containing the English word corresponding to each digit from `0` to `9`. This allows constant-time lookup for any digit.
2. Convert the number into its string representation so that each digit can be processed individually in the original order.
3. Maintain a hash map that stores the frequency of every character encountered so far. A hash map is suitable because characters are naturally used as keys and updates are constant time.
4. For each digit character, convert it into an integer and retrieve its corresponding English word.
5. Traverse every character in that word and increment its count in the frequency map.
6. After all digits have been processed, iterate over all frequency values in the map.
7. Count how many frequencies are odd, meaning the count modulo two equals one.
8. Return this count as the answer.

### Why it works

The algorithm maintains the invariant that after processing some prefix of the digits, the frequency map contains the exact number of occurrences of every character appearing in the corresponding concatenated words. Since every character from every digit word is added exactly once, the final map contains the true frequencies for the entire string. Therefore, counting the entries whose frequencies are odd produces the correct answer.

## Python Solution

```python
class Solution:
    def countOddLetters(self, n: int) -> int:
        digit_words = [
            "zero",
            "one",
            "two",
            "three",
            "four",
            "five",
            "six",
            "seven",
            "eight",
            "nine"
        ]

        frequency = {}

        for digit_char in str(n):
            word = digit_words[int(digit_char)]

            for ch in word:
                frequency[ch] = frequency.get(ch, 0) + 1

        answer = 0

        for count in frequency.values():
            if count % 2 == 1:
                answer += 1

        return answer
```

The implementation begins by storing the English representation of all ten digits in an array. This allows each digit to be translated with a simple indexing operation.

The number is converted into a string so that its digits can be traversed from left to right. For each digit, the corresponding word is obtained, and every character in that word updates the frequency map.

After all characters have been processed, the code iterates through the frequency values and counts those whose parity is odd. That count is returned as the final answer.

## Go Solution

```go
func countOddLetters(n int) int {
	digitWords := []string{
		"zero",
		"one",
		"two",
		"three",
		"four",
		"five",
		"six",
		"seven",
		"eight",
		"nine",
	}

	frequency := make(map[rune]int)

	for _, digitChar := range []byte(string(runeSliceFromInt(n))) {
		word := digitWords[digitChar-'0']

		for _, ch := range word {
			frequency[ch]++
		}
	}

	answer := 0

	for _, count := range frequency {
		if count%2 == 1 {
			answer++
		}
	}

	return answer
}

func runeSliceFromInt(n int) []rune {
	return []rune(fmt.Sprintf("%d", n))
}
```

A more idiomatic Go implementation can avoid helper structures and work directly with the string representation of the number.

```go
import "strconv"

func countOddLetters(n int) int {
	digitWords := []string{
		"zero",
		"one",
		"two",
		"three",
		"four",
		"five",
		"six",
		"seven",
		"eight",
		"nine",
	}

	frequency := make(map[rune]int)

	for _, digitChar := range strconv.Itoa(n) {
		word := digitWords[digitChar-'0']

		for _, ch := range word {
			frequency[ch]++
		}
	}

	answer := 0

	for _, count := range frequency {
		if count%2 == 1 {
			answer++
		}
	}

	return answer
}
```

The main difference from Python is that Go uses `strconv.Itoa` to convert the integer into a string. Character frequencies are stored in a `map[rune]int`, which plays the same role as the Python dictionary. Integer overflow is not an issue because the input size is extremely small.

## Edge Cases

A particularly important case is when the same digit appears multiple times. For example, `11` produces `"oneone"`. Every letter appears exactly twice, so the answer should be zero. An implementation that counts distinct letters instead of odd-frequency letters would fail on this case. The frequency map correctly accumulates counts and identifies that all frequencies are even.

Another important situation occurs when different digit words share letters. For example, `20` becomes `"twozero"`, where the letter `'o'` appears once in `"two"` and once in `"zero"`. A buggy implementation that treats each word independently might miss this interaction. Because the algorithm maintains one global frequency table, shared letters are counted correctly.

The maximum input value `10^9` contains ten digits and many repeated zeros. Even though the same word `"zero"` is processed repeatedly, the algorithm still runs efficiently because the total number of characters involved is very small. The memory usage remains constant since only letter frequencies are stored.
