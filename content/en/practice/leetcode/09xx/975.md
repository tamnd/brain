## Solution

## Problem Understanding

The problem is asking us to determine how many starting indices in an array allow a sequence of jumps to reach the last element of the array. A jump is defined differently depending on whether it is **odd-numbered** or **even-numbered**. An odd-numbered jump requires jumping forward to the smallest index where the value is greater than or equal to the current value. An even-numbered jump requires jumping forward to the smallest index where the value is less than or equal to the current value. If no such index exists, the jump cannot be made.

The input is a single integer array `arr` of length up to `2 * 10^4` with elements in the range `[0, 10^5)`. The output is a single integer representing the number of "good" starting indices from which we can reach the end of the array.

The constraints indicate that a naive brute-force simulation of all jump sequences is likely too slow, because iterating over all possible jumps from each starting index would take O(n²) or worse time.

Important edge cases include arrays with only one element (trivially good), arrays where all elements are identical, strictly increasing arrays, and strictly decreasing arrays, since these can test whether odd and even jumps behave correctly at boundaries.

## Approaches

### Brute Force Approach

A brute-force solution would simulate jumps from each starting index. For each index, we would repeatedly find the next odd or even jump according to the rules, checking every forward index to identify the correct destination. This guarantees correctness because it exactly follows the rules, but for each starting index we may need to scan the rest of the array repeatedly, resulting in O(n²) time in the worst case. This becomes impractical for the upper limit of n = 20,000.

### Optimal Approach

The key insight is that we can precompute, for each index, the next possible odd and even jumps using sorting and a monotonic stack. Specifically, if we sort indices by value ascending, we can determine the "next higher or equal" index for odd jumps. Sorting by value descending gives the "next lower or equal" index for even jumps. Once these jump targets are precomputed, we can use dynamic programming: maintain two boolean arrays `odd[i]` and `even[i]` indicating whether starting from index `i` with an odd or even jump can reach the end. We fill these arrays backwards from the last index, which trivially can reach the end. This approach reduces time complexity to O(n log n) due to sorting, and space complexity is O(n).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Simulate every jump from every index |
| Optimal | O(n log n) | O(n) | Precompute next jumps using sorting and monotonic stack, then use DP |

## Algorithm Walkthrough

1. **Initialization**: Create two boolean arrays `odd` and `even` of size n. These arrays indicate whether starting at index i with an odd or even jump can reach the end. Set `odd[n-1] = True` and `even[n-1] = True` because the last index can always reach itself.
2. **Compute next jump indices**: To know where an odd or even jump goes from each index, we need to precompute the "next greater or equal" and "next smaller or equal" indices.

- For odd jumps, sort indices of `arr` by value ascending and index ascending.
- For even jumps, sort indices by value descending and index ascending.
- Use a monotonic stack to determine, for each index in the sorted order, the next jump target. The stack maintains indices for which we haven't yet found the next jump.
3. **Dynamic programming**: Iterate backwards from index `n-2` to `0`. For each index:

- If `next_higher[i]` exists, set `odd[i] = even[next_higher[i]]`.
- If `next_lower[i]` exists, set `even[i] = odd[next_lower[i]]`.
4. **Count good starting indices**: All indices `i` where `odd[i]` is True are good, because the first jump from a starting index is always odd-numbered.

**Why it works**: By precomputing the next valid jumps and filling `odd` and `even` arrays in reverse, each entry reflects whether a sequence starting from that position can reach the end. The DP correctly propagates reachability from end to start, ensuring correctness.

## Python Solution

```
from typing import List

class Solution:
    def oddEvenJumps(self, arr: List[int]) -> int:
        n = len(arr)
        odd = [False] * n
        even = [False] * n
        odd[-1] = even[-1] = True

        def make_next_indices(sorted_indices: List[int]) -> List[int]:
            next_idx = [None] * n
            stack = []
            for i in sorted_indices:
                while stack and i > stack[-1]:
                    next_idx[stack.pop()] = i
                stack.append(i)
            return next_idx

        # Prepare indices sorted for odd and even jumps
        sorted_asc = sorted(range(n), key=lambda i: (arr[i], i))
        sorted_desc = sorted(range(n), key=lambda i: (-arr[i], i))

        next_higher = make_next_indices(sorted_asc)
        next_lower = make_next_indices(sorted_desc)

        # Fill DP arrays
        for i in range(n - 2, -1, -1):
            if next_higher[i] is not None:
                odd[i] = even[next_higher[i]]
            if next_lower[i] is not None:
                even[i] = odd[next_lower[i]]

        return sum(odd)
```

The Python solution first computes the next jump for each index using a monotonic stack. Then it fills the `odd` and `even` arrays backward, propagating reachability. Finally, it sums the `odd` array because starting jumps are odd-numbered.

## Go Solution

```
package main

func oddEvenJumps(arr []int) int {
    n := len(arr)
    odd := make([]bool, n)
    even := make([]bool, n)
    odd[n-1], even[n-1] = true, true

    type pair struct {
        idx, val int
    }

    makeNext := func(pairs []pair) []int {
        next := make([]int, n)
        for i := range next {
            next[i] = -1
        }
        stack := []int{}
        for _, p := range pairs {
            for len(stack) > 0 && p.idx > stack[len(stack)-1] {
                next[stack[len(stack)-1]] = p.idx
                stack = stack[:len(stack)-1]
            }
            stack = append(stack, p.idx)
        }
        return next
    }

    ascPairs := make([]pair, n)
    descPairs := make([]pair, n)
    for i := 0; i < n; i++ {
        ascPairs[i] = pair{i, arr[i]}
        descPairs[i] = pair{i, arr[i]}
    }

    // Sort ascPairs by value ascending then index
    sort.Slice(ascPairs, func(i, j int) bool {
        if ascPairs[i].val == ascPairs[j].val {
            return ascPairs[i].idx < ascPairs[j].idx
        }
        return ascPairs[i].val < ascPairs[j].val
    })

    // Sort descPairs by value descending then index
    sort.Slice(descPairs, func(i, j int) bool {
        if descPairs[i].val == descPairs[j].val {
            return descPairs[i].idx < descPairs[j].idx
        }
        return descPairs[i].val > descPairs[j].val
    })

    nextHigher := makeNext(ascPairs)
    nextLower := makeNext(descPairs)

    for i := n - 2; i >= 0; i-- {
        if nextHigher[i] != -1 {
            odd[i] = even[nextHigher[i]]
        }
        if nextLower[i] != -1 {
            even[i] = odd[nextLower[i]]
        }
    }

    count := 0
    for _, v := range odd {
        if v {
            count++
        }
    }
    return count
}
```

In Go, we use slices for boolean arrays and a custom struct `pair` for sorting indices. Nil values are replaced with -1 to indicate no next jump. Sorting uses `sort.Slice` with custom comparators.
