## Solution

## Problem Understanding

The problem gives us a list of unique points on a 2D coordinate plane. Each point is represented as `[x, y]`. We must determine the minimum possible area of an axis-aligned rectangle that can be formed using four of these points as corners.

An axis-aligned rectangle means its sides must be parallel to the X-axis and Y-axis. In other words, if we pick two diagonal corners `(x1, y1)` and `(x2, y2)`, then the other two corners must also exist:

- `(x1, y2)`
- `(x2, y1)`

The area of such a rectangle is:

$$|x2 - x1| \times |y2 - y1|$$

We must return the smallest rectangle area that can be formed. If no valid rectangle exists, we return `0`.

The constraints are important:

- There can be up to `500` points.
- Coordinates can be as large as `40000`.
- All points are unique.

Since `500` is relatively small, quadratic algorithms are acceptable, but cubic or quartic algorithms become risky. A brute-force solution that checks every possible group of four points would be extremely inefficient.

Several edge cases are important:

- There may be fewer than four points, making a rectangle impossible.
- Points may all lie on a single line.
- Multiple rectangles may exist, and we must track the minimum area.
- Rectangles can have very large coordinates, so we must compute area carefully.
- Some point pairs may share an x-coordinate or y-coordinate, which cannot form a diagonal of a rectangle.

The uniqueness guarantee simplifies the implementation because we do not need to handle duplicate points.

## Approaches

### Brute Force Approach

The most direct approach is to examine every possible combination of four points and determine whether they form a rectangle.

For each set of four points, we would verify:

1. There are exactly two distinct x-values.
2. There are exactly two distinct y-values.
3. All four corner combinations exist.

If those conditions hold, we compute the rectangle area and update the minimum.

This works because every rectangle consists of exactly four corners with two unique x-values and two unique y-values.

However, this approach is too slow. With `n` points, there are:

$$\binom{n}{4}$$

possible quadruples. For `n = 500`, this becomes impractically large.

### Optimal Approach

The key observation is that a rectangle can be uniquely identified by its diagonal corners.

Suppose we choose two points:

- `(x1, y1)`
- `(x2, y2)`

If:

- `x1 != x2`
- `y1 != y2`

then these points could form opposite corners of a rectangle.

The remaining two corners must be:

- `(x1, y2)`
- `(x2, y1)`

So instead of checking every group of four points, we can:

1. Store all points in a hash set for O(1) lookup.
2. Iterate through every pair of points.
3. Treat each pair as potential diagonal corners.
4. Check whether the other two required corners exist.

This reduces the problem to checking point existence efficiently.

Because there are only `O(n²)` point pairs, this approach is fast enough for the constraints.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n⁴) | O(1) | Checks every combination of four points |
| Optimal | O(n²) | O(n) | Uses hash set lookups to validate rectangles efficiently |

## Algorithm Walkthrough

1. Create a hash set containing every point.

We store points as tuples `(x, y)` because hash sets provide average O(1) lookup time. This allows us to quickly check whether a required corner exists.
2. Initialize the minimum area.

We start with infinity so that any valid rectangle area becomes the new minimum.
3. Iterate through every pair of points.

For each pair:

- `(x1, y1)`
- `(x2, y2)`

we consider them as potential diagonal corners of a rectangle.
4. Skip invalid diagonals.

If either:

- `x1 == x2`
- `y1 == y2`

then the points lie on the same vertical or horizontal line, so they cannot form opposite corners of a rectangle.
5. Compute the missing corners.

The rectangle would require:

- `(x1, y2)`
- `(x2, y1)`
6. Check whether both missing corners exist.

Using the hash set, we verify both points are present.

If they exist, then a valid rectangle has been formed.
7. Compute the area.

The area is:

$$|x2 - x1| \times |y2 - y1|$$

Update the minimum area if this rectangle is smaller.
8. Return the result.

If no rectangle was found, the minimum area remains infinity, so we return `0`.

### Why it works

A valid axis-aligned rectangle is completely determined by two diagonal corners with different x-values and y-values. If the remaining two corners exist, the rectangle must exist. By checking every pair of points as potential diagonals, we guarantee that every possible rectangle is considered exactly enough times to find the minimum area. The hash set ensures corner existence checks are efficient.

## Python Solution

```
from typing import List

class Solution:
    def minAreaRect(self, points: List[List[int]]) -> int:
        point_set = set()

        for x, y in points:
            point_set.add((x, y))

        min_area = float("inf")
        n = len(points)

        for i in range(n):
            x1, y1 = points[i]

            for j in range(i + 1, n):
                x2, y2 = points[j]

                # Cannot form a diagonal
                if x1 == x2 or y1 == y2:
                    continue

                # Check whether the other two corners exist
                if (x1, y2) in point_set and (x2, y1) in point_set:
                    area = abs(x2 - x1) * abs(y2 - y1)
                    min_area = min(min_area, area)

        return 0 if min_area == float("inf") else min_area
```

The implementation begins by converting all points into a hash set of tuples. This allows constant-time membership checks when validating rectangle corners.

The nested loops iterate through every pair of points exactly once. Using `j` starting at `i + 1` avoids duplicate pair checks.

The condition:

```
if x1 == x2 or y1 == y2:
```

filters out pairs that cannot possibly represent diagonal corners.

For valid diagonals, the algorithm computes the remaining two required corners and checks whether both exist in the hash set.

If they do, the rectangle area is computed and compared against the current minimum.

Finally, if no rectangle was discovered, the method returns `0`.

## Go Solution

```
package main

import "math"

func minAreaRect(points [][]int) int {
	pointSet := make(map[[2]int]bool)

	for _, point := range points {
		pointSet[[2]int{point[0], point[1]}] = true
	}

	minArea := math.MaxInt32
	n := len(points)

	for i := 0; i < n; i++ {
		x1, y1 := points[i][0], points[i][1]

		for j := i + 1; j < n; j++ {
			x2, y2 := points[j][0], points[j][1]

			// Cannot form a diagonal
			if x1 == x2 || y1 == y2 {
				continue
			}

			// Check the other two corners
			if pointSet[[2]int{x1, y2}] &&
				pointSet[[2]int{x2, y1}] {

				area := abs(x2-x1) * abs(y2-y1)

				if area < minArea {
					minArea = area
				}
			}
		}
	}

	if minArea == math.MaxInt32 {
		return 0
	}

	return minArea
}

func abs(x int) int {
	if x < 0 {
		return -x
	}
	return x
}
```

The Go implementation uses a `map[[2]int]bool` to represent the hash set of points. Arrays are comparable in Go, so `[2]int` works naturally as a map key.

Unlike Python, Go does not provide a built-in integer absolute value function for plain integers, so a helper `abs` function is implemented.

The logic otherwise mirrors the Python solution closely.

- `(1,3)`
- `(3,1)`

Both exist.

Area:

$$(3 - 1) \times (3 - 1) = 4$$

#### Pair: (1,3) and (3,1)

#### Pair: (1,1) and (2,2)

Required corners:

- `(1,2)`
- `(2,1)`

These do not exist.

No rectangle forms.

All remaining pairs are checked similarly.

Final answer:

| Time | O(n²) | Every pair of points is examined once |
| Space | O(n) | Hash set stores all points |

The nested loops generate all point pairs, producing `O(n²)` iterations. Each iteration performs constant-time hash lookups, so the total complexity remains quadratic.

The extra space comes from storing every point in the hash set for efficient existence checks.

## Edge Cases

One important edge case occurs when there are not enough points to form a rectangle. A rectangle requires four corners, so inputs with fewer than four points must return `0`. The implementation naturally handles this because no valid diagonal pair can produce all four required corners.

Another important case involves points lying entirely on a horizontal or vertical line. In these situations, many point pairs share either the same x-coordinate or the same y-coordinate. Such pairs cannot form rectangle diagonals. The implementation explicitly skips them using:

```
if x1 == x2 or y1 == y2:
```

This prevents invalid rectangles from being considered.

A third edge case involves multiple overlapping rectangles. Some naive solutions may stop after finding the first valid rectangle, but the problem requires the minimum area rectangle. The implementation continuously updates `min_area` whenever a smaller rectangle is found, ensuring the optimal answer is returned.

A final subtle edge case occurs when many extra points exist inside or outside valid rectangles. Interior points do not matter because the algorithm only checks whether the four corner points exist. This keeps the logic correct regardless of how cluttered the point set becomes.
