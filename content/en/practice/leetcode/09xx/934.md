## Solution

## Problem Understanding

This problem asks us to connect two separate islands in a binary matrix with the fewest number of 0-to-1 flips. The input is an `n x n` grid, where `1` represents land and `0` represents water. Each island is a 4-directionally connected cluster of `1`s, and the problem guarantees that there are exactly **two islands**. The output is a single integer: the smallest number of `0`s that must be changed to `1`s to connect the islands.

The constraints tell us the grid can be up to `100 x 100`, which rules out algorithms with cubic or worse time complexity. The problem also guarantees two islands, so we do not have to handle more complex configurations. Edge cases could include islands that are diagonally adjacent (requiring multiple flips to connect) or one island being entirely surrounded by water with no immediate neighbors.

Understanding this problem requires recognizing that the solution involves **graph traversal**, either Depth-First Search (DFS) or Breadth-First Search (BFS), since islands are connected components in the grid.

## Approaches

The brute-force approach would be to first identify all cells of the first island, then compute the distance to every cell of the second island, tracking the minimum Manhattan distance minus 1 (because we only flip water cells). This works because the shortest path between two points in a grid is related to Manhattan distance, but it requires comparing each cell in one island with every cell in the other. If each island has `O(n^2)` cells, this yields **O(n^4)** time complexity, which is infeasible for `n = 100`.

The key insight for an optimal solution is that we do not need to calculate distances between every pair of island cells. Instead, we can use BFS starting from one island and expand outward into the water until we reach the second island. This is more efficient because BFS naturally finds the shortest path in an unweighted grid, and we avoid redundant distance calculations.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^4) | O(n^2) | Compare every pair of cells between islands |
| Optimal | O(n^2) | O(n^2) | DFS to mark first island, BFS to expand until reaching second island |

## Algorithm Walkthrough

1. **Identify the first island**: Traverse the grid to find the first `1`. Use DFS to mark all connected `1`s as visited and collect their coordinates into a queue. This allows us to treat the first island as a starting frontier for BFS.
2. **Prepare for BFS**: Initialize a queue with all cells of the first island and maintain a `visited` set or modify the grid to prevent revisiting cells.
3. **BFS expansion**: Expand in layers from the first island. At each step, for each cell in the current layer, attempt to move up, down, left, and right. If the neighboring cell is water (`0`), add it to the next layer. If the neighboring cell is part of the second island (`1` but not visited), we have reached the other island.
4. **Count flips**: Each BFS level corresponds to flipping water cells to land. When a cell of the second island is reached, the current BFS level count is the minimum number of flips needed.
5. **Return result**: The BFS ensures that the first time we reach the second island is the shortest path because BFS explores all cells at the current distance before moving to a greater distance.

**Why it works**: The BFS guarantees the shortest number of water cells to flip because it explores cells in increasing order of distance from the first island. Once a cell of the second island is reached, any other path would require equal or more flips.

## Python Solution

```
from typing import List
from collections import deque

class Solution:
    def shortestBridge(self, grid: List[List[int]]) -> int:
        n = len(grid)
        directions = [(0,1),(1,0),(0,-1),(-1,0)]
        visited = [[False]*n for _ in range(n)]
        queue = deque()
        
        # DFS to mark first island
        def dfs(x, y):
            visited[x][y] = True
            queue.append((x, y))
            for dx, dy in directions:
                nx, ny = x+dx, y+dy
                if 0 <= nx < n and 0 <= ny < n and not visited[nx][ny] and grid[nx][ny] == 1:
                    dfs(nx, ny)
        
        # Find first island and mark it
        found = False
        for i in range(n):
            if found:
                break
            for j in range(n):
                if grid[i][j] == 1:
                    dfs(i, j)
                    found = True
                    break
        
        # BFS to expand from first island
        flips = 0
        while queue:
            for _ in range(len(queue)):
                x, y = queue.popleft()
                for dx, dy in directions:
                    nx, ny = x+dx, y+dy
                    if 0 <= nx < n and 0 <= ny < n and not visited[nx][ny]:
                        if grid[nx][ny] == 1:
                            return flips
                        visited[nx][ny] = True
                        queue.append((nx, ny))
            flips += 1
        return -1
```

**Explanation**: The DFS function marks the first island and populates the BFS queue. The BFS loop then expands outward, layer by layer, counting flips until a cell of the second island is reached, at which point the current number of flips is returned.

## Go Solution

```
package main

func shortestBridge(grid [][]int) int {
    n := len(grid)
    directions := [][2]int{{0,1},{1,0},{0,-1},{-1,0}}
    visited := make([][]bool, n)
    for i := range visited {
        visited[i] = make([]bool, n)
    }
    queue := [][2]int{}
    
    var dfs func(int,int)
    dfs = func(x, y int) {
        visited[x][y] = true
        queue = append(queue, [2]int{x, y})
        for _, d := range directions {
            nx, ny := x+d[0], y+d[1]
            if nx >= 0 && nx < n && ny >= 0 && ny < n && !visited[nx][ny] && grid[nx][ny] == 1 {
                dfs(nx, ny)
            }
        }
    }
    
    found := false
    for i := 0; i < n && !found; i++ {
        for j := 0; j < n && !found; j++ {
            if grid[i][j] == 1 {
                dfs(i, j)
                found = true
            }
        }
    }
    
    flips := 0
    for len(queue) > 0 {
        nextQueue := [][2]int{}
        for _, cell := range queue {
            x, y := cell[0], cell[1]
            for _, d := range directions {
                nx, ny := x+d[0], y+d[1]
                if nx >= 0 && nx < n && ny >= 0 && ny < n && !visited[nx][ny] {
                    if grid[nx][ny] == 1 {
                        return flips
                    }
                    visited[nx][ny] = true
                    nextQueue = append(nextQueue, [2]int{nx, ny})
                }
            }
        }
        queue = nextQueue
        flips++
    }
    return -1
}
```

**Go notes**: The implementation mirrors Python closely. We preallocate `visited` slices and manage the BFS queue explicitly with slices. Go requires careful bounds checking and slice appends, but the logic is otherwise identical.

## Worked Examples

### Example 1: `grid = [[0,1],[1,0]]`

**DFS** marks `(0,1)` and `(1,0)` is the first island? Actually `(0,1)` is first. Queue = [(0,1)]. BFS expands:

| BFS Layer | Queue | Flips |
| --- | --- | --- |
| 0 | [(0,1)] | 0 |
| 1 | [(1,1),(0,0)] | 1 |

Second island `(1,0)` reached at flips = 1. Output: 1.
