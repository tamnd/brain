## Solution

## Problem Understanding

The problem describes a network of `n` nodes represented as an `n x n` adjacency matrix `graph`. Each node can be connected to other nodes directly, and `graph[i][j] == 1` indicates that nodes `i` and `j` are connected. Some nodes are initially infected by malware, given in the `initial` list. Malware spreads through direct connections, so any node connected to an infected node becomes infected as well, continuing until no further infections occur.

The task is to remove exactly one node from the initially infected list `initial` to minimize the total number of infected nodes after the spread. If multiple nodes yield the same minimized infection, the smallest index node should be chosen.

The constraints provide important information: the network can be up to 300 nodes, `graph` is symmetric and always has 1s on the diagonal, and the `initial` list is unique. This means the problem is manageable with O(n²) algorithms because 300² = 90,000, but O(n³) brute-force approaches may become too slow.

Important edge cases include fully disconnected nodes, all nodes initially infected, or multiple nodes that, if removed, have the same effect. Additionally, nodes that are part of separate connected components need careful handling, as removing one node may not prevent the infection of others in different components.

## Approaches

The brute-force approach would simulate removing each node from `initial` and then perform a full graph traversal (DFS or BFS) to calculate the total infection for each removal. This ensures correctness because it directly evaluates the effect of each removal, but it is too slow for `n = 300` because it could lead to O(n³) operations: O(n) for each node removal and O(n²) for traversing the adjacency matrix for each simulation.

The key observation that allows a more optimal solution is that the network can be decomposed into connected components, and the effect of removing a node can be reasoned about using component sizes and the count of initially infected nodes in each component. If a component has exactly one infected node, removing that node prevents the entire component from being infected. If a component has multiple infected nodes, removing any single node does not reduce the spread in that component. This allows us to calculate the optimal removal in O(n²) time using DFS or BFS, without full simulations for each removal.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(n²) | Simulate removing each node and spread malware |
| Optimal | O(n²) | O(n) | Count infected nodes per connected component and apply the single-infection rule |

## Algorithm Walkthrough

1. Sort the `initial` list to ensure that ties return the smallest index node.
2. Use DFS or BFS to label all connected components in the graph. Store for each node the component it belongs to and the size of that component.
3. Count the number of initially infected nodes in each component.
4. Iterate through each node in `initial`. If removing that node would eliminate the only infection in its component, calculate the size of the component that would be saved.
5. Track the node that maximizes the number of saved nodes. If no node saves any nodes, return the smallest index node in `initial`.
6. Return the node that either maximizes the reduction of infection or the smallest node if no reduction occurs.

**Why it works:** The algorithm leverages the property that only components with a single infected node can benefit from removing that node. Components with multiple infected nodes cannot be protected by removing a single node. By mapping nodes to components and counting infections, we avoid redundant full simulations, ensuring correctness efficiently.

## Python Solution

```
from typing import List

class Solution:
    def minMalwareSpread(self, graph: List[List[int]], initial: List[int]) -> int:
        n = len(graph)
        
        # Step 1: Identify connected components
        component = [-1] * n
        def dfs(node: int, comp_id: int):
            component[node] = comp_id
            for neighbor, connected in enumerate(graph[node]):
                if connected and component[neighbor] == -1:
                    dfs(neighbor, comp_id)
        
        comp_id = 0
        for node in range(n):
            if component[node] == -1:
                dfs(node, comp_id)
                comp_id += 1
        
        # Step 2: Count component sizes and infected nodes per component
        comp_size = [0] * comp_id
        infected_count = [0] * comp_id
        for node in range(n):
            comp_size[component[node]] += 1
        for node in initial:
            infected_count[component[node]] += 1
        
        # Step 3: Evaluate the effect of removing each initial node
        initial.sort()
        best_node = initial[0]
        max_saved = -1
        for node in initial:
            c = component[node]
            if infected_count[c] == 1:  # only one infection in this component
                saved = comp_size[c]
                if saved > max_saved:
                    max_saved = saved
                    best_node = node
        
        return best_node
```

**Explanation:** First, we use DFS to label connected components. Then, we count component sizes and how many initial infections exist per component. We then iterate over sorted `initial` nodes to determine which removal maximizes the saved nodes, returning the optimal node or the smallest index if no removal helps.

## Go Solution

```
func minMalwareSpread(graph [][]int, initial []int) int {
    n := len(graph)
    component := make([]int, n)
    for i := range component {
        component[i] = -1
    }

    var dfs func(int, int)
    dfs = func(node, compID int) {
        component[node] = compID
        for neighbor, connected := range graph[node] {
            if connected == 1 && component[neighbor] == -1 {
                dfs(neighbor, compID)
            }
        }
    }

    compID := 0
    for node := 0; node < n; node++ {
        if component[node] == -1 {
            dfs(node, compID)
            compID++
        }
    }

    compSize := make([]int, compID)
    infectedCount := make([]int, compID)
    for _, c := range component {
        compSize[c]++
    }
    for _, node := range initial {
        infectedCount[component[node]]++
    }

    minNode := initial[0]
    maxSaved := -1
    // Sort initial
    for i := 0; i < len(initial)-1; i++ {
        for j := i + 1; j < len(initial); j++ {
            if initial[i] > initial[j] {
                initial[i], initial[j] = initial[j], initial[i]
            }
        }
    }

    for _, node := range initial {
        c := component[node]
        if infectedCount[c] == 1 {
            saved := compSize[c]
            if saved > maxSaved {
                maxSaved = saved
                minNode = node
            }
        }
    }
    return minNode
}
```

**Explanation:** The Go solution mirrors the Python version, with DFS labeling components, counting component sizes and infections, sorting `initial`, and selecting the optimal node. Differences include explicit slice initialization and manual sorting, which replaces Python's built-in sort.

## Worked Examples

**Example 1:** `graph = [[1,1,0],[1,1,0],[0,0,1]], initial = [0,1]`

1. Connected components: node 0 and 1 in comp 0, node 2 in comp 1.
2. Component sizes: comp 0 → 2, comp 1 → 1
3. Initial infections per component: comp 0 → 2, comp 1 → 0
4. No component has a single infection, so return smallest index `0`.

**Example 2:** `graph = [[1,0,0],[0,1,0],[0,0,1]], initial = [0,2]`

1. Connected components: each node in its own component.
2. Component sizes: all 1
3. Each initial node is the only infection in its component. Both removeable, return smallest index `0`.

**Example 3:** `graph = [[1,1,1],[1,1,1],[1,1,1]], initial = [1,2]`

1. All nodes in one component of size 3.
2. Initial infections per component: 2
3. No node can save the component entirely, return smallest index `1`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | DFS to find components traverses adjacency matrix once, sorting `initial` is O(n log n) which is dominated |
| Space | O(n) | Arrays to track components, component sizes, and infection counts |

The main cost comes from traversing the graph, which is quadratic in the number of nodes due to adjacency matrix representation.
