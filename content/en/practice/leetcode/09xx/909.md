## Solution

## Problem Understanding

The problem gives us an `n x n` game board representing a variation of the classic Snakes and Ladders game. Each square on the board is labeled from `1` to `n^2`, but the numbering follows a boustrophedon pattern. This means the direction alternates every row. The bottom row is labeled left to right, the next row above is labeled right to left, and so on.

The input matrix `board` contains either `-1` or another square number. A value of `-1` means the square is normal. Any other value represents a snake or ladder that immediately transports the player to another square.

The player always starts on square `1`. During each move, the player rolls a standard six-sided die and may move between `1` and `6` squares forward. If the destination square contains a snake or ladder, the player must follow it exactly once. The objective is to determine the minimum number of dice rolls required to reach square `n^2`.

The key challenge in this problem is that the board numbering is not stored directly in order. We must correctly convert between square numbers and matrix coordinates. Another important detail is that snakes and ladders are only applied once per move. If a ladder leads to another ladder, the second ladder is ignored.

The constraints tell us that `2 <= n <= 20`, so the maximum number of squares is only `400`. This is small enough that graph traversal algorithms such as Breadth-First Search are extremely practical.

Several edge cases are important:

- A square may contain a snake that sends the player backward significantly.
- Some boards may make the destination unreachable.
- The board numbering alternates direction every row, which is easy to implement incorrectly.
- Snakes and ladders are applied only once per move, not recursively.
- The destination square itself never contains a snake or ladder.

## Approaches

A brute-force approach would try every possible sequence of dice rolls recursively. From each square, we could explore up to six possible next positions, then continue exploring until reaching the destination. While this guarantees correctness because all paths are examined, it becomes extremely inefficient due to repeated exploration of the same states. The number of possible move sequences grows exponentially.

A much better observation is that this problem is fundamentally a shortest path problem on an unweighted graph. Each square represents a node, and each possible die roll creates an edge to another node. Since every move costs exactly one dice roll, Breadth-First Search naturally finds the minimum number of moves required to reach the destination.

The only nontrivial part is correctly translating square numbers into board coordinates because of the boustrophedon numbering pattern.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(6^(n²)) | O(n²) | Explores all possible move sequences recursively |
| Optimal BFS | O(n²) | O(n²) | Uses BFS to find the shortest number of dice rolls |

## Algorithm Walkthrough

1. Treat every square as a node in a graph.

From square `x`, the player can move to any square in the range `[x + 1, x + 6]`, provided the square does not exceed `n^2`.
2. Convert square numbers into matrix coordinates.

The board is not labeled in straightforward row-major order. We first determine how far the square is from the bottom of the board, then determine whether that row is traversed left-to-right or right-to-left.
3. Create a helper function for coordinate conversion.

For a square number:

- Convert it into a zero-based index.
- Determine the row from the bottom.
- Determine the column direction based on row parity.
- Return `(row, col)`.
4. Start Breadth-First Search from square `1`.

BFS is ideal because each edge corresponds to exactly one dice roll. The first time we reach a square is guaranteed to be the minimum number of moves.
5. Maintain a queue.

Each queue entry stores:

- Current square
- Number of moves used to reach it
6. Maintain a visited set.

Without a visited structure, the algorithm could revisit squares indefinitely because snakes can move backward.
7. Process all possible dice rolls.

For every current square:

- Try moves from `1` to `6`
- Compute the tentative next square
- Convert it to board coordinates
- If the board contains a snake or ladder, move to its destination
8. Stop when reaching `n^2`.

Since BFS explores states level by level, the first time we reach the final square is the minimum number of dice rolls.
9. Return `-1` if unreachable.

If the queue becomes empty before reaching the destination, no valid path exists.

### Why it works

Breadth-First Search always explores positions in increasing order of distance from the starting node. Since every dice roll counts as exactly one move, BFS guarantees that the first time we reach square `n^2`, we have used the minimum possible number of rolls. The visited set ensures every square is processed at most once, preventing infinite cycles caused by snakes.

## Python Solution

```
from collections import deque
from typing import List

class Solution:
    def snakesAndLadders(self, board: List[List[int]]) -> int:
        n = len(board)

        def get_position(square: int) -> tuple[int, int]:
            quotient, remainder = divmod(square - 1, n)

            row = n - 1 - quotient

            if quotient % 2 == 0:
                col = remainder
            else:
                col = n - 1 - remainder

            return row, col

        queue = deque([(1, 0)])
        visited = {1}

        while queue:
            current_square, moves = queue.popleft()

            if current_square == n * n:
                return moves

            for next_square in range(
                current_square + 1,
                min(current_square + 6, n * n) + 1
            ):
                row, col = get_position(next_square)

                if board[row][col] != -1:
                    destination = board[row][col]
                else:
                    destination = next_square

                if destination not in visited:
                    visited.add(destination)
                    queue.append((destination, moves + 1))

        return -1
```

The implementation begins by defining a helper function called `get_position`. This function converts a square number into the correct `(row, col)` coordinates on the board. Since the numbering alternates direction every row, we must carefully determine whether the row is left-to-right or right-to-left.

The BFS queue stores pairs of `(square, moves)`. The `moves` value represents how many dice rolls were needed to reach that square.

For every dequeued square, the algorithm simulates all six possible dice rolls. Each resulting square is checked for snakes or ladders. If one exists, the destination is updated immediately.

The visited set prevents repeated processing of the same square, which is especially important because snakes may create cycles.

The algorithm returns as soon as the final square is reached, ensuring the minimum number of moves.

## Go Solution

```
package main

import "container/list"

func snakesAndLadders(board [][]int) int {
	n := len(board)

	getPosition := func(square int) (int, int) {
		quotient := (square - 1) / n
		remainder := (square - 1) % n

		row := n - 1 - quotient

		var col int
		if quotient%2 == 0 {
			col = remainder
		} else {
			col = n - 1 - remainder
		}

		return row, col
	}

	type State struct {
		square int
		moves  int
	}

	queue := list.New()
	queue.PushBack(State{1, 0})

	visited := make(map[int]bool)
	visited[1] = true

	for queue.Len() > 0 {
		front := queue.Front()
		queue.Remove(front)

		current := front.Value.(State)

		if current.square == n*n {
			return current.moves
		}

		maxNext := current.square + 6
		if maxNext > n*n {
			maxNext = n * n
		}

		for nextSquare := current.square + 1; nextSquare <= maxNext; nextSquare++ {
			row, col := getPosition(nextSquare)

			destination := nextSquare
			if board[row][col] != -1 {
				destination = board[row][col]
			}

			if !visited[destination] {
				visited[destination] = true
				queue.PushBack(State{destination, current.moves + 1})
			}
		}
	}

	return -1
}
```

The Go implementation follows the exact same BFS logic as the Python solution. Since Go does not provide a built-in deque structure, the implementation uses `container/list` as the queue.

The visited structure is implemented using `map[int]bool`, which efficiently tracks already processed squares.

Go does not have tuple support like Python, so a small `State` struct is used to store the current square and move count together.

## Edge Cases

One important edge case occurs when a snake sends the player backward to a previously visited square. Without a visited set, the algorithm could repeatedly revisit the same states and potentially loop forever. The BFS implementation avoids this by marking every processed destination square as visited before enqueueing it.

Another subtle case involves the boustrophedon numbering pattern. Since every row alternates direction, a naive coordinate conversion can easily map squares incorrectly. The helper function carefully determines whether a row should be traversed left-to-right or right-to-left based on the row index counted from the bottom.

A third important edge case occurs when a ladder leads to another ladder. The problem explicitly states that only one snake or ladder may be taken per dice roll. The implementation handles this correctly because it checks the board exactly once after determining the immediate destination square. No recursive teleportation occurs.

A final edge case involves very small boards such as `2x2`. On these boards, the destination may be reachable in a single roll. The BFS structure naturally handles this because all reachable squares from the current position are explored immediately.
