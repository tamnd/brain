## Solution

## Problem Understanding

The problem gives us a binary tree and a target preorder traversal sequence called `voyage`. Our goal is to determine whether we can transform the tree so that its preorder traversal exactly matches `voyage` by flipping some nodes.

A flip operation on a node swaps its left and right children. Importantly, we are not allowed to arbitrarily rearrange the tree. The only operation available is swapping the two child pointers of a node.

The preorder traversal order is:

1. Visit the current node
2. Traverse the left subtree
3. Traverse the right subtree

Because preorder traversal always visits the left subtree before the right subtree, flipping a node changes the traversal order of its children.

The task is not only to determine whether matching the voyage is possible, but also to return the set of nodes that must be flipped. If multiple valid answers exist, any valid list is acceptable. If matching the voyage is impossible, we return `[-1]`.

The constraints are relatively small:

- `n <= 100`
- All node values are unique
- All voyage values are unique

The uniqueness guarantee is extremely important because it allows us to identify exactly which subtree should appear next during traversal. Without uniqueness, deciding whether to flip would become ambiguous.

Several edge cases are important:

- The root itself may not match the first value in `voyage`, which immediately makes the task impossible.
- A node may have only one child, so flipping does not meaningfully change traversal order.
- The traversal may diverge midway through recursion, even if the beginning matches.
- We must minimize flips, which means we should only flip when necessary.

The small input size means even slower approaches could technically pass, but the problem naturally admits a clean linear-time DFS solution.

## Approaches

### Brute Force Approach

A brute-force strategy would try every possible combination of flips across the tree.

For each node, we have two choices:

- Do not flip
- Flip the node

If the tree has `n` nodes, this creates up to `2^n` possible tree configurations. For each configuration, we could generate the preorder traversal and compare it against `voyage`.

This works because it exhaustively explores every possible flipped state of the tree. Eventually, if a valid configuration exists, brute force will discover it.

However, this approach is extremely inefficient. Even for `n = 100`, `2^100` possibilities are astronomically large and completely infeasible.

The main inefficiency is that the brute-force method ignores the strong structure imposed by preorder traversal. We do not actually need to try arbitrary flip combinations because preorder traversal tells us exactly which child must come next.

### Optimal DFS Greedy Approach

The key observation is that preorder traversal gives us a strict expected visitation order.

Suppose we are currently visiting node `x`.

- The next value in `voyage` tells us which child should be visited next.
- If the left child already matches the next expected value, we continue normally.
- If the right child matches instead, then we must flip this node.
- If neither child matches, the traversal is impossible.

This creates a natural greedy DFS strategy.

We maintain an index into `voyage` representing the next expected node value during traversal.

At each node:

- Verify the current node matches the expected voyage value.
- Advance the index.
- Decide whether a flip is necessary by inspecting the next expected value.
- Continue recursively.

Because every node is processed exactly once, this solution runs in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Tries every flip combination and validates traversal |
| Optimal | O(n) | O(n) | DFS traversal with greedy flip decisions |

## Algorithm Walkthrough

### Step 1: Maintain a Global Voyage Index

We keep an integer `index` that points to the current expected position in `voyage`.

Whenever we visit a node during DFS, its value must equal `voyage[index]`.

If it does not match, the traversal is impossible.

### Step 2: Perform Preorder DFS

We recursively traverse the tree in preorder order:

1. Visit current node
2. Traverse left subtree
3. Traverse right subtree

This mirrors the exact traversal pattern we are trying to match.

### Step 3: Validate Current Node

When entering a node:

- Compare `node.val` with `voyage[index]`
- If they differ, return failure immediately
- Otherwise increment `index`

This guarantees the traversal remains synchronized with `voyage`.

### Step 4: Determine Whether a Flip Is Needed

After consuming the current node, we inspect the next expected voyage value.

Normally, preorder visits the left child first.

So:

- If `node.left` exists and its value equals `voyage[index]`, continue normally.
- Otherwise, if `node.right` matches the next expected value, we must flip.

When flipping:

- Record the current node value in the answer list
- Traverse the right subtree before the left subtree

This simulates the flipped tree structure without physically modifying the tree.

### Step 5: Continue DFS Recursively

After deciding the traversal order:

- Recursively process the first subtree
- Then recursively process the second subtree

If any recursive call fails, propagate failure upward.

### Step 6: Return Result

If traversal completes successfully:

- Return the collected flip list

Otherwise:

- Return `[-1]`

### Why it works

The algorithm works because preorder traversal uniquely determines the next node that must appear. Since all node values are unique, the next value in `voyage` tells us exactly whether the left or right subtree must be visited next.

If the left child matches, flipping is unnecessary. If the right child matches, flipping is mandatory. If neither matches, no sequence of future flips can repair the mismatch.

Thus the greedy choice at every node is forced, which guarantees correctness.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def flipMatchVoyage(self, root: Optional[TreeNode], voyage: List[int]) -> List[int]:
        self.index = 0
        flips = []

        def dfs(node):
            if not node:
                return True

            if node.val != voyage[self.index]:
                return False

            self.index += 1

            if (
                node.left and
                self.index < len(voyage) and
                node.left.val != voyage[self.index]
            ):
                flips.append(node.val)

                return dfs(node.right) and dfs(node.left)

            return dfs(node.left) and dfs(node.right)

        return flips if dfs(root) else [-1]
```

The implementation closely follows the preorder traversal logic.

The variable `self.index` tracks our current position in the target voyage. Every time we visit a node, we verify that the node value matches the expected voyage value.

The recursive helper `dfs` returns `True` if the subtree can successfully match the remaining voyage sequence.

The critical logic appears in the flip decision:

```
if node.left and node.left.val != voyage[self.index]:
```

This condition means the left subtree does not contain the next expected node. Since preorder traversal visits the left subtree first, the only way to proceed is to flip the current node and visit the right subtree first.

Instead of physically swapping children, we simply reverse the DFS traversal order:

```
dfs(node.right)
dfs(node.left)
```

This avoids unnecessary tree mutations and keeps the implementation simple.

If any mismatch occurs, recursion immediately returns `False`, and the final answer becomes `[-1]`.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func flipMatchVoyage(root *TreeNode, voyage []int) []int {
    index := 0
    flips := []int{}

    var dfs func(node *TreeNode) bool

    dfs = func(node *TreeNode) bool {
        if node == nil {
            return true
        }

        if node.Val != voyage[index] {
            return false
        }

        index++

        if node.Left != nil &&
            index < len(voyage) &&
            node.Left.Val != voyage[index] {

            flips = append(flips, node.Val)

            return dfs(node.Right) && dfs(node.Left)
        }

        return dfs(node.Left) && dfs(node.Right)
    }

    if dfs(root) {
        return flips
    }

    return []int{-1}
}
```

The Go implementation mirrors the Python solution almost exactly.

The recursive DFS function is implemented as a closure so it can access `index` and `flips` from the outer scope.

Go uses `nil` checks instead of Python truthiness checks. Slice operations are straightforward because Go slices dynamically resize during `append`.

No integer overflow concerns exist because node values are small.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every node is visited exactly once |
| Space | O(n) | Recursive DFS call stack in worst case |

The DFS traversal processes each node exactly one time. Every operation inside each DFS call is constant time, so the total runtime is linear in the number of nodes.

The recursive call stack can grow to the height of the tree. In the worst case of a completely skewed tree, the height becomes `n`, giving `O(n)` auxiliary space usage.

## Edge Cases

One important edge case occurs when the root node itself does not match the first voyage value. Since preorder traversal always begins with the root, no amount of flipping can change this fact. The implementation immediately detects this during the first DFS comparison and returns `[-1]`.

Another subtle case occurs when a node has only one child. Flipping such a node does not meaningfully alter traversal order because there is no second subtree to swap with. The algorithm naturally handles this because it only attempts a flip when the left child exists and does not match the next expected voyage value.

A third important edge case involves mismatches deep inside the tree. The traversal may initially align with `voyage`, but eventually encounter a node whose value differs from the expected value. The recursive DFS propagates failure upward immediately, ensuring the algorithm stops as soon as impossibility is detected.

A final edge case is a completely skewed tree, where every node has only one child. In this scenario, recursion depth becomes `O(n)`. The implementation handles this correctly because the constraints are small enough that recursion depth is safe.
