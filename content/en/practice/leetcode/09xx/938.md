## Solution

## Problem Understanding

The problem gives us the root of a Binary Search Tree (BST) and two integers, `low` and `high`. Our goal is to compute the sum of all node values whose values fall within the inclusive range `[low, high]`.

A Binary Search Tree has an important property:

- Every node in the left subtree has a smaller value than the current node.
- Every node in the right subtree has a larger value than the current node.

This ordering property is the key to solving the problem efficiently.

The input tree is represented using the standard binary tree node structure. Each node contains:

- An integer value
- A pointer/reference to its left child
- A pointer/reference to its right child

The output is a single integer representing the total sum of all node values satisfying:

$$low \leq node.val \leq high$$

The constraints tell us several important things:

- The tree can contain up to `2 * 10^4` nodes, so the solution should ideally avoid unnecessary work.
- Node values are unique.
- Node values range up to `10^5`.
- The tree is guaranteed to be a valid BST.

A naive solution that visits every node will still pass because `2 * 10^4` is not extremely large, but the BST property allows us to prune unnecessary branches and improve efficiency in many cases.

Several edge cases are important:

- The tree may contain only one node.
- Every node may fall inside the range.
- No node may fall inside the range.
- The range boundaries may exactly match node values.
- The tree may be heavily skewed, making recursion depth close to the number of nodes.

Because the tree is guaranteed to be a BST and values are unique, we can safely use ordering comparisons to decide whether entire subtrees can be skipped.

## Approaches

### Brute Force Approach

The simplest approach is to perform a complete traversal of the tree, usually with Depth-First Search (DFS). For every node:

1. Check whether its value lies within `[low, high]`.
2. If it does, add it to the running sum.
3. Continue recursively exploring both left and right children.

This approach works because it eventually visits every node exactly once, ensuring no valid value is missed.

However, it ignores the BST structure entirely. Even when a subtree clearly cannot contain valid values, the algorithm still traverses it. For example, if the current node value is already smaller than `low`, then the entire left subtree must also be smaller than `low`, yet brute force still explores it unnecessarily.

### Optimal Approach Using BST Pruning

The key insight is that the BST ordering lets us eliminate entire subtrees from consideration.

Suppose we are currently at a node with value `x`.

- If `x < low`, then:

- Every value in the left subtree is also `< low`
- The left subtree can never contribute to the answer
- We only need to search the right subtree
- If `x > high`, then:

- Every value in the right subtree is also `> high`
- The right subtree can never contribute
- We only need to search the left subtree
- If `low <= x <= high`, then:

- The current node contributes to the sum
- Both subtrees may still contain valid nodes
- We recursively explore both sides

This pruning significantly reduces unnecessary traversal in many cases.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(h) | Visits every node regardless of BST ordering |
| Optimal | O(n) worst case, often better | O(h) | Uses BST property to skip irrelevant subtrees |

Here, `n` is the number of nodes and `h` is the tree height.

## Algorithm Walkthrough

1. Start a recursive DFS traversal from the root node.
2. If the current node is `None`, return `0` because there is no value to contribute.
3. Compare the current node value with `low`.

- If `node.val < low`, then all values in the left subtree are also too small because of BST ordering.
- Skip the left subtree entirely.
- Recursively search only the right subtree.
4. Compare the current node value with `high`.

- If `node.val > high`, then all values in the right subtree are also too large.
- Skip the right subtree entirely.
- Recursively search only the left subtree.
5. If the current node value lies within the range:

- Add the current node value to the result.
- Recursively compute the sums from both left and right subtrees.
- Return the combined total.
6. Continue recursively until all relevant nodes have been processed.

### Why it works

The algorithm relies on the BST invariant:

- Left subtree values are always smaller than the current node.
- Right subtree values are always larger than the current node.

Because of this ordering, whenever a node is outside the target range, we can mathematically prove that one entire subtree cannot contain any valid values. By pruning these subtrees, the algorithm avoids unnecessary work while still guaranteeing that every valid node is included exactly once.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def rangeSumBST(self, root: Optional[TreeNode], low: int, high: int) -> int:
        def dfs(node: Optional[TreeNode]) -> int:
            if not node:
                return 0

            if node.val < low:
                return dfs(node.right)

            if node.val > high:
                return dfs(node.left)

            return (
                node.val
                + dfs(node.left)
                + dfs(node.right)
            )

        return dfs(root)
```

The implementation uses a nested helper function `dfs` to perform recursive traversal.

The base case checks whether the current node is `None`. If so, the function returns `0` because an empty subtree contributes nothing to the sum.

The next two conditions implement BST pruning:

- If the node value is smaller than `low`, the entire left subtree is ignored.
- If the node value is larger than `high`, the entire right subtree is ignored.

Only when the node value lies inside the valid range do we:

- Add the node value to the total
- Recursively explore both children

The final answer is returned from the initial call `dfs(root)`.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func rangeSumBST(root *TreeNode, low int, high int) int {
    var dfs func(node *TreeNode) int

    dfs = func(node *TreeNode) int {
        if node == nil {
            return 0
        }

        if node.Val < low {
            return dfs(node.Right)
        }

        if node.Val > high {
            return dfs(node.Left)
        }

        return node.Val + dfs(node.Left) + dfs(node.Right)
    }

    return dfs(root)
}
```

The Go implementation follows the same recursive logic as the Python version.

The main difference is Go's explicit use of pointers and `nil` checks. The recursive helper function is declared using a function variable because recursive anonymous functions in Go require prior declaration.

Integer overflow is not a concern here because the maximum possible sum remains safely within Go's integer limits for standard platforms.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) worst case | Every node may need to be visited |
| Space | O(h) | Recursive call stack height equals tree height |

In the worst case, such as a completely skewed tree or a range covering all nodes, the algorithm visits every node, giving `O(n)` time complexity.

However, BST pruning often reduces the actual number of visited nodes significantly.

The space complexity comes from recursion depth. In a balanced BST, the height is `O(log n)`. In the worst case of a skewed tree, the height becomes `O(n)`.

## Edge Cases

### Single Node Tree

A tree containing only one node is the smallest possible input. This case can expose bugs where recursion assumes child nodes always exist. The implementation handles this safely because the recursive helper immediately returns `0` for `None` children.

### No Valid Nodes in Range

Sometimes every node value lies outside the target interval. A naive implementation could accidentally include invalid values or traverse unnecessary branches inefficiently. The pruning logic ensures that subtrees are skipped whenever their values cannot possibly contribute to the answer.

### Highly Skewed Tree

A BST may become completely unbalanced, resembling a linked list. In this case, recursion depth approaches `n`. The algorithm still produces the correct result because it processes nodes sequentially through recursive calls, although recursion depth becomes the worst-case space complexity.

### Range Boundaries Equal Node Values

The range is inclusive, meaning nodes equal to `low` or `high` must be counted. Off-by-one comparison mistakes are common here. The implementation correctly uses:

```
if node.val < low
```

and

```
if node.val > high
```

which ensures nodes equal to the boundaries are included in the sum.
