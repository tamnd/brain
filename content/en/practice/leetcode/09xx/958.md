## Solution

## Problem Understanding

The problem asks us to determine whether a given binary tree is a complete binary tree. A complete binary tree has a very specific structural property. Every level of the tree must be completely filled except possibly the last level, and all nodes on the last level must appear as far left as possible.

The input is the root node of a binary tree. Each node contains a value and references to its left and right children. The actual values stored in the nodes are irrelevant for this problem because completeness depends only on the structure of the tree.

The output is a boolean value:

- `true` if the tree satisfies the definition of a complete binary tree
- `false` otherwise

A complete binary tree allows the last level to be partially filled, but there cannot be any gaps between nodes on that level. For example, if a node is missing a left child but has a right child, the tree is immediately invalid because nodes are no longer positioned as far left as possible.

The constraints are relatively small, with at most 100 nodes. Even though this means many approaches would be fast enough in practice, the goal is still to design a clean and logically correct algorithm that scales properly and demonstrates understanding of tree traversal techniques.

Several edge cases are important:

- A tree with only one node is complete
- A node with a right child but no left child is never valid
- Once a missing child is encountered during level order traversal, every following node must also be missing children
- Sparse structures near the bottom of the tree can violate completeness even if the tree looks mostly balanced

## Approaches

### Brute Force Approach

A brute force approach is to assign array-style indices to every node, similar to how binary heaps are represented. If a node has index `i`, then:

- Left child gets index `2 * i`
- Right child gets index `2 * i + 1`

In a complete binary tree with `n` nodes, the maximum assigned index should also be `n`. If there are gaps in the structure, some indices will be skipped, causing the maximum index to become larger than the total number of nodes.

This approach works because complete binary trees correspond exactly to contiguous heap-style indexing without holes.

To implement this solution:

1. Count the total number of nodes
2. Perform another traversal assigning indices
3. Verify that no node receives an index greater than the total node count

This solution is correct, but it requires extra bookkeeping and two traversals of the tree.

### Optimal Approach

The optimal solution uses Breadth-First Search, also called level order traversal.

The key insight is that during level order traversal of a complete binary tree, once we encounter a `null` position, every subsequent position must also be `null`.

Why is this true?

In a complete binary tree, nodes are filled from left to right. Therefore, if we ever see an empty slot before seeing all remaining nodes, the structure contains a gap, which violates completeness.

The algorithm performs a BFS traversal using a queue:

- Add the root to the queue
- Process nodes level by level
- Once a `null` node is seen, set a flag
- If any later node is non-null after that flag is set, return `false`

This approach directly models the definition of completeness and is both elegant and efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Uses heap-style indexing and multiple traversals |
| Optimal | O(n) | O(n) | BFS traversal with null detection |

## Algorithm Walkthrough

### Optimal BFS Algorithm

1. Initialize a queue and insert the root node into it.

We use Breadth-First Search because completeness is fundamentally a level-by-level property. BFS naturally processes nodes in the exact order required to validate completeness.
2. Create a boolean flag called `seen_null` and initialize it to `False`.

This flag records whether we have already encountered a missing position in the tree.
3. Start processing nodes from the queue until the queue becomes empty.

Each iteration removes one node from the front of the queue.
4. If the current node is `None`, set `seen_null = True`.

This means we found an empty position in level order traversal. From this point onward, all remaining positions must also be empty for the tree to remain complete.
5. If the current node is not `None`:

- First check whether `seen_null` is already `True`
- If it is, return `False`

This situation means a real node appeared after a missing position, creating a structural gap.
6. If the current node is valid, add both its left and right children to the queue.

We enqueue children even if they are `None` because missing positions are important for detecting completeness violations.
7. If the traversal finishes without finding an invalid structure, return `True`.

### Why it works

The algorithm relies on a fundamental property of complete binary trees: during level order traversal, all nodes must appear consecutively without gaps. The first missing position marks the point where the tree must end structurally. If any later node exists after that point, the tree cannot be complete. Because BFS visits nodes exactly in left-to-right level order, this invariant guarantees correctness.

## Python Solution

```
from collections import deque
from typing import Optional

# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def isCompleteTree(self, root: Optional[TreeNode]) -> bool:
        queue = deque([root])
        seen_null = False

        while queue:
            current = queue.popleft()

            if current is None:
                seen_null = True
            else:
                if seen_null:
                    return False

                queue.append(current.left)
                queue.append(current.right)

        return True
```

The implementation begins by creating a queue initialized with the root node. The queue supports efficient FIFO operations, which makes it ideal for BFS traversal.

The `seen_null` flag tracks whether a missing node position has already been encountered. During traversal, every node is removed from the queue in level order.

If the current node is `None`, the algorithm marks that an empty position has been seen. If a later node is non-null after this point, the tree structure contains a gap, so the function immediately returns `False`.

For valid nodes, both children are added to the queue, including `None` children. This is important because missing positions themselves carry structural information.

If the traversal completes without detecting any violations, the tree satisfies the completeness property and the function returns `True`.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

func isCompleteTree(root *TreeNode) bool {
	queue := []*TreeNode{root}
	seenNil := false

	for len(queue) > 0 {
		current := queue[0]
		queue = queue[1:]

		if current == nil {
			seenNil = true
		} else {
			if seenNil {
				return false
			}

			queue = append(queue, current.Left)
			queue = append(queue, current.Right)
		}
	}

	return true
}
```

The Go implementation follows the same BFS strategy as the Python solution. The main difference is queue management. Since Go does not have a built-in deque structure, a slice is used as a queue by removing elements from the front.

Go uses `nil` instead of Python's `None`, but the logic remains identical. Children are appended to the queue regardless of whether they are `nil`, which preserves the structural ordering needed for correctness.

There are no integer overflow concerns because the algorithm does not use indexing arithmetic.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every node is processed exactly once |
| Space | O(n) | The queue may store an entire tree level |

The BFS traversal visits each node one time, giving linear time complexity. The queue can contain up to an entire level of the tree at once. In the worst case, this can be proportional to the number of nodes in the tree, resulting in linear auxiliary space usage.

## Edge Cases

A tree containing only a single node is an important boundary case. Many implementations accidentally assume children exist or fail to handle the smallest possible input correctly. In this implementation, the BFS queue begins with the root node, processes it normally, and terminates successfully, returning `True`.

A node that has a right child but no left child is always invalid in a complete binary tree. This is one of the most common structural mistakes. During BFS traversal, the missing left child produces a `None` entry first. When the later right child is encountered, the algorithm detects a non-null node after a null position and correctly returns `False`.

Another subtle case occurs when a gap appears in the middle of the last level. For example:

```
        1
      /   \
     2     3
    /     /
   4     6
```

Even though the tree may initially look balanced, the missing node between `4` and `6` violates the left-to-right filling requirement. Because the BFS traversal processes positions exactly in level order, the algorithm detects the missing slot before reaching node `6`, correctly identifying the tree as incomplete.
