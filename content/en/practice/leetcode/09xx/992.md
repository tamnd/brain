## Solution

## Problem Understanding

The problem asks us to count how many contiguous subarrays contain **exactly `k` distinct integers**.

We are given an integer array `nums` and an integer `k`. A subarray is any contiguous segment of the array, meaning elements must remain adjacent and in order. The goal is to count all valid subarrays where the number of unique values is exactly `k`.

For example, if `nums = [1,2,1,2,3]` and `k = 2`, we want every contiguous segment that contains exactly two distinct numbers. A subarray like `[1,2]` qualifies because it contains `{1,2}`. A subarray like `[1,2,1]` also qualifies because repeated elements do not increase the distinct count. However, `[1,2,1,2,3]` does not qualify because it contains three distinct numbers.

The constraints are important:

- `1 <= nums.length <= 2 * 10^4`
- `1 <= nums[i], k <= nums.length`

The maximum input size is `20,000`, which is large enough that an `O(n²)` or `O(n³)` solution will likely time out. This strongly suggests we need a linear or near linear solution.

Several edge cases are worth considering:

- When `k = 1`, we count subarrays consisting of a single repeated value.
- When `k` equals the number of unique elements in the array, long subarrays may qualify.
- Arrays with many repeated numbers can produce many overlapping valid subarrays.
- Arrays with entirely unique elements require careful counting because each expansion changes the distinct count immediately.
- The problem guarantees `k >= 1`, so we never need to handle invalid zero or negative values.

The main challenge is efficiently counting subarrays with **exactly** `k` distinct values without enumerating every possible subarray.

## Approaches

### Brute Force Approach

A straightforward approach is to examine every possible subarray.

We can generate all subarrays by fixing a starting index and extending the ending index one step at a time. For each subarray, we maintain a set or frequency map to track distinct integers. Whenever the number of distinct integers becomes exactly `k`, we increment the answer.

This method is correct because it explicitly checks every contiguous subarray and verifies whether it satisfies the requirement.

However, the problem is performance. There are `O(n²)` subarrays, and maintaining distinct counts may cost additional time. Even with incremental updates, the worst case becomes `O(n²)`, which is too slow for `n = 20,000`.

### Key Insight

The critical observation is that counting subarrays with **exactly `k` distinct integers** is difficult directly, but counting subarrays with **at most `k` distinct integers** is much easier.

We can use the following identity:

$$\text{Exactly}(k) = \text{AtMost}(k) - \text{AtMost}(k - 1)$$

Why does this work?

- `AtMost(k)` counts all subarrays containing `0` to `k` distinct numbers.
- `AtMost(k - 1)` counts all subarrays containing `0` to `k - 1` distinct numbers.
- Subtracting removes every subarray with fewer than `k` distinct values, leaving only those with exactly `k`.

The reason `AtMost(k)` is easy to compute is because it naturally fits a **sliding window** approach. We maintain a window that always contains at most `k` distinct integers, shrinking it whenever the limit is exceeded.

For every valid ending index, we can count how many valid starting indices exist.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Enumerates all subarrays and tracks distinct values |
| Optimal | O(n) | O(n) | Sliding window using `AtMost(k) - AtMost(k-1)` |

## Algorithm Walkthrough

We define a helper function `atMost(limit)` that returns the number of subarrays containing **at most `limit` distinct integers**.

### Step 1: Initialize Sliding Window State

We maintain:

- A frequency map to count occurrences of numbers inside the current window.
- A `left` pointer for the beginning of the window.
- A counter tracking how many distinct numbers currently exist.
- A running total of valid subarrays.

The window will always satisfy the condition:

> It contains at most `limit` distinct integers.

### Step 2: Expand the Window

Iterate through the array using a `right` pointer.

For every new element:

- Add it to the frequency map.
- If it appears for the first time, increase the distinct count.

This grows the current window.

### Step 3: Shrink the Window When Invalid

If the number of distinct integers exceeds `limit`, shrink the window from the left.

We repeatedly:

- Decrease the count of `nums[left]`.
- Remove it from the distinct count if its frequency becomes zero.
- Move `left` forward.

This continues until the window becomes valid again.

### Step 4: Count Valid Subarrays

Once the window is valid:

Every subarray ending at `right` and starting anywhere between `left` and `right` is valid.

The number of such subarrays is:

$$right - left + 1$$

We add this to the total.

### Step 5: Compute Exactly `k`

Finally:

1. Compute `atMost(k)`
2. Compute `atMost(k - 1)`
3. Return their difference

This gives the count of subarrays with exactly `k` distinct integers.

### Why it works

The sliding window maintains an invariant:

> The current window always contains at most `k` distinct integers.

For every position `right`, once the window becomes valid, every starting point between `left` and `right` forms a valid subarray ending at `right`. Since each element enters and leaves the window at most once, we count all valid subarrays exactly once in linear time.

The subtraction trick works because every subarray with exactly `k` distinct integers belongs to `AtMost(k)` but not `AtMost(k-1)`.

## Python Solution

```
from collections import defaultdict
from typing import List

class Solution:
    def subarraysWithKDistinct(self, nums: List[int], k: int) -> int:
        def at_most(limit: int) -> int:
            frequency = defaultdict(int)
            left = 0
            distinct_count = 0
            total_subarrays = 0

            for right in range(len(nums)):
                current_num = nums[right]

                if frequency[current_num] == 0:
                    distinct_count += 1

                frequency[current_num] += 1

                while distinct_count > limit:
                    left_num = nums[left]
                    frequency[left_num] -= 1

                    if frequency[left_num] == 0:
                        distinct_count -= 1

                    left += 1

                total_subarrays += right - left + 1

            return total_subarrays

        return at_most(k) - at_most(k - 1)
```

The implementation follows the algorithm directly.

The `at_most()` helper function counts subarrays with at most a given number of distinct integers. A `defaultdict(int)` is used as a frequency map because it automatically initializes unseen keys to zero.

As the `right` pointer expands, each number is added to the frequency map. If its previous frequency was zero, it means a new distinct value entered the window.

Whenever the distinct count exceeds the allowed limit, the `while` loop shrinks the window from the left until validity is restored.

Once the window is valid, every subarray ending at `right` and beginning between `left` and `right` is valid. Therefore, we add `right - left + 1` to the answer.

Finally, we apply:

$$\text{Exactly}(k) = \text{AtMost}(k) - \text{AtMost}(k - 1)$$

to obtain the final result.

## Go Solution

```
func subarraysWithKDistinct(nums []int, k int) int {
	atMost := func(limit int) int {
		frequency := make(map[int]int)
		left := 0
		distinctCount := 0
		totalSubarrays := 0

		for right := 0; right < len(nums); right++ {
			currentNum := nums[right]

			if frequency[currentNum] == 0 {
				distinctCount++
			}

			frequency[currentNum]++

			for distinctCount > limit {
				leftNum := nums[left]
				frequency[leftNum]--

				if frequency[leftNum] == 0 {
					distinctCount--
				}

				left++
			}

			totalSubarrays += right - left + 1
		}

		return totalSubarrays
	}

	return atMost(k) - atMost(k-1)
}
```

The Go implementation mirrors the Python solution closely.

Instead of `defaultdict`, Go uses `map[int]int` for frequency counting. Missing keys default to zero, so frequency checks behave naturally.

Go slices are reference backed, making traversal efficient without copying data. Integer overflow is not a concern because the maximum number of subarrays is roughly `n(n+1)/2`, which comfortably fits in Go's integer range for the given constraints.

No special handling for `nil` versus empty slices is needed because the constraints guarantee at least one element.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element enters and leaves the sliding window at most once |
| Space | O(n) | Frequency map may store up to all unique elements |

Although `at_most()` is called twice, each call runs in linear time. Since constants are ignored in asymptotic analysis:

$$O(n) + O(n) = O(n)$$

The space complexity comes from the frequency map, which can contain at most all unique numbers in the input.

## Edge Cases

### Case 1: All Elements Are the Same

Consider:

```
nums = [1,1,1,1], k = 1
```

Every subarray is valid because every subarray contains exactly one distinct integer. This case can expose bugs where duplicate values are mistakenly treated as new distinct elements.

The frequency map prevents this issue because the distinct count increases only when an element's frequency changes from `0` to `1`.

### Case 2: Every Element Is Unique

Consider:

```
nums = [1,2,3,4], k = 2
```

Every new element immediately increases the distinct count, causing frequent shrinking of the window. A buggy implementation may incorrectly count invalid windows.

The sliding window invariant guarantees correctness because the window is shrunk immediately whenever the number of distinct integers exceeds the allowed limit.

### Case 3: `k` Equals Array Diversity

Consider:

```
nums = [1,2,1,3], k = 3
```

Only windows containing all three distinct values should count. It is easy to accidentally overcount subarrays with fewer unique numbers.

The formula:

$$\text{Exactly}(k) = \text{AtMost}(k) - \text{AtMost}(k-1)$$

eliminates this risk by mathematically isolating subarrays containing exactly `k` distinct integers.
