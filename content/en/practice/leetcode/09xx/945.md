## Solution

## Problem Understanding

This problem is asking us to make all elements in an integer array `nums` unique using the fewest number of increment operations. Each increment operation increases a single element by 1. The input `nums` is an array of non-negative integers, potentially containing duplicates. The output should be the minimum number of increments needed so that no two elements in `nums` are equal.

Constraints provide important information about the problem scale. The array length can be up to 100,000, and each number can also be as large as 100,000. This indicates that any solution with quadratic time complexity (O(n²)) would be too slow, while linear or linearithmic solutions are feasible.

Important edge cases include arrays where all elements are already unique, arrays with all elements the same, or arrays that contain zeros and the maximum value 100,000. The problem guarantees that the answer will fit within a 32-bit integer, so we do not need to worry about integer overflow.

## Approaches

The brute-force approach is straightforward. We could iterate over the array and, for each element, check whether it has appeared before. If it has, we increment it until it becomes unique. While this method eventually gives the correct result, it is too slow for large inputs because each increment may require repeated scanning of the array to check for uniqueness, leading to potentially O(n²) time complexity.

The optimal approach uses sorting and a greedy strategy. If we sort the array first, all duplicates appear consecutively. We can then traverse the sorted array and ensure that each element is at least one greater than the previous element. If it is not, we increment it to that minimum required value and count the number of moves. This works because, after sorting, adjusting duplicates to the next available number locally guarantees global uniqueness with the minimum number of increments.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Increment each duplicate by scanning the array for uniqueness |
| Optimal | O(n log n) | O(1) or O(n) | Sort the array, then increment duplicates greedily |

## Algorithm Walkthrough

1. Sort the input array `nums` in ascending order. Sorting ensures that all duplicates are adjacent, making it easier to handle increments sequentially.
2. Initialize a variable `moves` to 0. This will accumulate the total number of increment operations needed.
3. Traverse the sorted array starting from the second element (index 1). For each element, check if it is less than or equal to the previous element.
4. If it is, calculate the number of increments needed to make it one greater than the previous element. Add this number to `moves`, and update the current element to its new value.
5. Continue this process until the end of the array. Each step ensures that the array remains sorted and unique.
6. Return `moves` as the result.

Why it works: After sorting, any duplicates are consecutive. Incrementing a number to be at least one greater than its predecessor guarantees uniqueness without skipping any smaller possible increments. This greedy strategy ensures the minimum total moves because we always use the smallest possible increment at each step.

## Python Solution

```
from typing import List

class Solution:
    def minIncrementForUnique(self, nums: List[int]) -> int:
        nums.sort()
        moves = 0
        for i in range(1, len(nums)):
            if nums[i] <= nums[i - 1]:
                increment = nums[i - 1] + 1 - nums[i]
                moves += increment
                nums[i] = nums[i - 1] + 1
        return moves
```

The Python implementation first sorts `nums`. Then it iterates from the second element, checking if it is less than or equal to the previous element. If it is, it calculates how much to increment it so that it is one greater than the previous element, updates the array in place, and accumulates the number of moves. Sorting ensures the greedy step works optimally.

## Go Solution

```
import "sort"

func minIncrementForUnique(nums []int) int {
    sort.Ints(nums)
    moves := 0
    for i := 1; i < len(nums); i++ {
        if nums[i] <= nums[i-1] {
            increment := nums[i-1] + 1 - nums[i]
            moves += increment
            nums[i] = nums[i-1] + 1
        }
    }
    return moves
}
```

In Go, we first sort the slice using `sort.Ints`. We then traverse the slice starting from index 1, incrementing elements as needed and accumulating the total moves. Go handles slices efficiently in place, similar to Python lists, so we do not need additional storage.

## Edge Cases

The first edge case is a single-element array. Since there are no duplicates, the algorithm must correctly return 0 moves. This tests the base case handling without unnecessary operations.

The second edge case is an array where all elements are identical. For example, `[0,0,0,0]` requires incrementing each duplicate incrementally. The algorithm handles this correctly because it always increments a duplicate to the next available number.

The third edge case is large numbers or arrays close to the maximum constraints. Using `[100000]*5` tests that the implementation handles high values without integer overflow. Sorting and greedy increments work for any integer value within the allowed range.

This covers both correctness for simple inputs and performance for large-scale inputs.
