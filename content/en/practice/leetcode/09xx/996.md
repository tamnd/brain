## Solution

## Problem Understanding

The problem asks us to count how many distinct permutations of an array are "squareful". A permutation is squareful if every pair of adjacent numbers adds up to a perfect square.

For example, consider the array `[1, 17, 8]`. The permutation `[1, 8, 17]` is valid because:

- `1 + 8 = 9`, which is a perfect square
- `8 + 17 = 25`, which is also a perfect square

Similarly, `[17, 8, 1]` is valid for the same reason in reverse order.

The input is an integer array `nums`, and the output is the number of distinct permutations that satisfy this adjacency condition.

A very important detail is that the array may contain duplicate values. Two permutations are considered different only if at least one position contains a different value. This means duplicate permutations must not be counted multiple times.

The constraints tell us several important things:

- `nums.length <= 12`, which is small enough for exponential algorithms
- Values can be as large as `10^9`, so we cannot use small fixed-size frequency arrays indexed by value
- Since permutations are involved, the natural search space is factorial, specifically `n!`
- A naive brute-force solution becomes infeasible near `n = 12`

The small length strongly suggests that techniques like backtracking, memoization, and bitmask dynamic programming are appropriate.

There are several important edge cases:

- Arrays with all duplicate values, such as `[2,2,2]`
- Arrays where no adjacent pair forms a perfect square
- Arrays where many permutations are duplicates because of repeated numbers
- Arrays of length `1`, which are always squareful because there are no adjacent pairs to violate the condition

The problem guarantees only that the array length is at least `1`, so we must correctly handle singleton arrays.

## Approaches

### Brute Force Approach

The most straightforward solution is to generate every possible permutation of the array, then check whether each permutation is squareful.

For every permutation:

1. Iterate through adjacent pairs
2. Compute their sum
3. Check whether the sum is a perfect square
4. Count the permutation if all adjacent sums satisfy the condition

This approach is correct because it explicitly examines every possible ordering and validates each one.

However, the complexity is extremely high. An array of length `n` has `n!` permutations. Even with pruning duplicate permutations, the search space grows too quickly.

For `n = 12`:

```
12! = 479,001,600
```

Checking nearly half a billion permutations is far too slow.

The brute-force approach also wastes work because it repeatedly checks the same adjacency relationships across many permutations.

### Key Insight

The crucial observation is that this problem can be modeled as a graph.

Think of each array index as a node. There is an edge between two indices if the sum of their values is a perfect square.

For example:

```
nums = [1,17,8]

1 + 8 = 9     -> edge
17 + 8 = 25   -> edge
1 + 17 = 18   -> no edge
```

This creates a graph where valid permutations correspond to valid paths that visit every node exactly once.

Since `n <= 12`, we can use:

- Backtracking to explore valid paths
- Bitmasking to track which indices are already used
- Duplicate skipping to avoid counting identical permutations multiple times

Instead of generating all permutations blindly, we only extend paths using valid squareful neighbors.

This dramatically reduces the search space.

## Algorithm Walkthrough

### Step 1: Sort the Array

We first sort the array.

Sorting helps us efficiently skip duplicate values during backtracking. Without sorting, duplicate elimination becomes much harder.

For example:

```
[2,2,2]
```

After sorting, identical values become adjacent, allowing us to detect duplicate choices.

### Step 2: Build the Adjacency Graph

We precompute which indices can connect to each other.

For every pair of indices `(i, j)`:

1. Compute `nums[i] + nums[j]`
2. Check whether the sum is a perfect square
3. If it is, add an edge between `i` and `j`

We store this in an adjacency list.

The perfect square check works by:

1. Computing `root = int(sqrt(x))`
2. Verifying `root * root == x`

This preprocessing avoids recomputing square checks repeatedly during DFS.

### Step 3: Use DFS with Bitmasking

We perform depth-first search to construct valid permutations.

The DFS state contains:

- `mask`, a bitmask representing which indices are already used
- `last`, the index of the most recently placed number

If all bits are set in the mask, we have constructed a complete valid permutation.

### Step 4: Try Extending the Current Path

From the current node:

1. Iterate through all valid neighbors
2. Skip neighbors already used in the bitmask
3. Skip duplicate choices when appropriate
4. Recurse with the updated mask

The duplicate skipping rule is critical:

```
if nums[next] == nums[next - 1] and not used_previous_duplicate:
    continue
```

This ensures identical values are only considered in one canonical order.

### Step 5: Memoize Results

Many DFS states repeat.

For example, reaching the same `(mask, last)` state through different paths leads to identical future possibilities.

We cache results using memoization:

```
memo[(mask, last)]
```

This transforms exponential recomputation into dynamic programming.

### Step 6: Sum All Valid Starting Points

We begin DFS from every valid starting index.

Again, duplicate starting values are skipped to avoid overcounting.

## Why it works

The algorithm explores every valid permutation exactly once.

The adjacency graph guarantees that only squareful transitions are considered. The bitmask ensures every index is used exactly once. Duplicate skipping guarantees identical permutations are not counted multiple times.

Memoization ensures each state is solved once, making the algorithm efficient enough for `n <= 12`.

## Python Solution

```
from typing import List
from functools import lru_cache
import math

class Solution:
    def numSquarefulPerms(self, nums: List[int]) -> int:
        nums.sort()
        n = len(nums)

        graph = [[] for _ in range(n)]

        def is_perfect_square(x: int) -> bool:
            root = int(math.isqrt(x))
            return root * root == x

        for i in range(n):
            for j in range(n):
                if i != j and is_perfect_square(nums[i] + nums[j]):
                    graph[i].append(j)

        @lru_cache(None)
        def dfs(mask: int, last: int) -> int:
            if mask == (1 << n) - 1:
                return 1

            total = 0

            for nei in graph[last]:
                if mask & (1 << nei):
                    continue

                if (
                    nei > 0
                    and nums[nei] == nums[nei - 1]
                    and not (mask & (1 << (nei - 1)))
                ):
                    continue

                total += dfs(mask | (1 << nei), nei)

            return total

        answer = 0

        for i in range(n):
            if i > 0 and nums[i] == nums[i - 1]:
                continue

            answer += dfs(1 << i, i)

        return answer
```

The implementation begins by sorting the array so duplicates can be skipped consistently.

The graph construction phase precomputes all valid transitions. This is much faster than recomputing square checks during every DFS call.

The `dfs(mask, last)` function represents the number of squareful permutations that can be formed starting from the current state.

The bitmask tracks which indices have already been used. Each bit corresponds to one index.

Memoization is handled with `@lru_cache(None)`, which prevents repeated computation of identical states.

The duplicate skipping condition is subtle but extremely important. Suppose we have identical values:

```
[1,1,8]
```

If we freely choose either `1` first, we would count identical permutations multiple times. The skip condition enforces a canonical ordering among duplicates.

Finally, the algorithm tries every distinct starting index and sums the results.

## Go Solution

```
package main

import (
	"math"
	"sort"
)

func numSquarefulPerms(nums []int) int {
	sort.Ints(nums)
	n := len(nums)

	graph := make([][]int, n)

	isPerfectSquare := func(x int) bool {
		root := int(math.Sqrt(float64(x)))
		return root*root == x
	}

	for i := 0; i < n; i++ {
		for j := 0; j < n; j++ {
			if i != j && isPerfectSquare(nums[i]+nums[j]) {
				graph[i] = append(graph[i], j)
			}
		}
	}

	memo := make(map[[2]int]int)

	var dfs func(mask int, last int) int

	dfs = func(mask int, last int) int {
		if mask == (1<<n)-1 {
			return 1
		}

		key := [2]int{mask, last}

		if val, exists := memo[key]; exists {
			return val
		}

		total := 0

		for _, nei := range graph[last] {
			if mask&(1<<nei) != 0 {
				continue
			}

			if nei > 0 &&
				nums[nei] == nums[nei-1] &&
				(mask&(1<<(nei-1))) == 0 {
				continue
			}

			total += dfs(mask|(1<<nei), nei)
		}

		memo[key] = total
		return total
	}

	answer := 0

	for i := 0; i < n; i++ {
		if i > 0 && nums[i] == nums[i-1] {
			continue
		}

		answer += dfs(1<<i, i)
	}

	return answer
}
```

The Go implementation follows the same overall strategy as the Python version.

Since Go does not have built-in memoization decorators like Python's `lru_cache`, we explicitly use a map with a composite key:

```
map[[2]int]int
```

The key stores:

- Current bitmask
- Current ending index

Go also requires explicit recursive function declarations using a variable assignment pattern:

```
var dfs func(mask int, last int) int
```

The square root computation uses `math.Sqrt`, which operates on `float64`, so explicit conversions are necessary.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n² × 2ⁿ) | Graph construction costs O(n²), DFS states are bounded by bitmask DP |
| Space | O(n × 2ⁿ) | Memoization table stores states for each mask and ending index |

The graph preprocessing examines every pair of indices, producing `O(n²)` work.

The DFS state is determined by:

- One of `2ⁿ` masks
- One of `n` ending positions

Therefore, the total number of unique states is:

```
O(n × 2ⁿ)
```

Each state may iterate through up to `n` neighbors, giving the final complexity:

```
O(n² × 2ⁿ)
```

This is efficient enough for `n <= 12`.

## Edge Cases

### Arrays with All Duplicate Values

Consider:

```
[2,2,2]
```

Every permutation looks identical. A naive permutation generator would count the same arrangement multiple times.

The implementation handles this by sorting the array and skipping duplicate choices unless the previous identical value has already been used in the current path. This guarantees each distinct permutation is counted exactly once.

### Arrays with No Valid Adjacent Pairs

Consider:

```
[3,6,7]
```

No pair sums to a perfect square.

In this case, the graph contains no edges. The DFS immediately fails to extend any path beyond length `1`, resulting in an answer of `0`.

The implementation naturally handles this because the adjacency list for each node is empty.

### Single Element Arrays

Consider:

```
[42]
```

There are no adjacent pairs at all.

By definition, the permutation is squareful because there is no violating pair.

The implementation correctly returns `1` because the DFS base case immediately succeeds when all elements are used.

### Arrays Containing Zero

Consider:

```
[0,0,0]
```

Since:

```
0 + 0 = 0
```

and `0` is a perfect square, all adjacencies are valid.

Some implementations incorrectly assume perfect squares start at `1`, which would fail here.

The implementation correctly uses integer square root logic, so `0` is properly recognized as a perfect square.
