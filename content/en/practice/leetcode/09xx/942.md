## Solution

## Problem Understanding

In this problem, we are given a string `s` consisting only of the characters `'I'` and `'D'`. The length of the string is `n`, and we must construct a permutation of the integers from `0` to `n`.

A permutation means every integer in the range appears exactly once.

The string describes the relationship between adjacent elements in the permutation:

- `'I'` means increasing, so `perm[i] < perm[i + 1]`
- `'D'` means decreasing, so `perm[i] > perm[i + 1]`

The goal is to return any valid permutation that satisfies all these relationships.

For example, if `s = "IDID"`, then:

- `perm[0] < perm[1]`
- `perm[1] > perm[2]`
- `perm[2] < perm[3]`
- `perm[3] > perm[4]`

The resulting permutation must contain exactly the numbers `[0, 1, 2, 3, 4]` in some order.

The constraints are important:

- `1 <= s.length <= 10^5`

This tells us the input can be very large, so exponential or quadratic solutions are not acceptable. We need a linear time solution.

There are several edge cases worth thinking about early:

- A string containing only `'I'`, such as `"III"`, should produce a strictly increasing sequence.
- A string containing only `'D'`, such as `"DDD"`, should produce a strictly decreasing sequence.
- Alternating patterns like `"IDIDID"` require careful assignment of values.
- Since the permutation must use every number exactly once, we cannot reuse or skip numbers.

The problem guarantees that the string only contains `'I'` and `'D'`, so input validation is unnecessary.

## Approaches

### Brute Force Approach

A brute force solution would generate every possible permutation of the integers from `0` to `n`, then check whether each permutation satisfies the pattern described by the string.

For each permutation, we would compare adjacent values:

- If `s[i] == 'I'`, verify `perm[i] < perm[i + 1]`
- If `s[i] == 'D'`, verify `perm[i] > perm[i + 1]`

The first valid permutation could then be returned.

This approach is correct because it exhaustively checks all possible permutations, guaranteeing that a valid one will eventually be found.

However, this method is far too slow. There are `(n + 1)!` permutations, which becomes enormous even for relatively small values of `n`. Since `n` can be as large as `100000`, brute force is completely infeasible.

### Optimal Greedy Two Pointer Approach

The key observation is that we do not need to try all permutations. Instead, we can construct one directly.

We maintain two numbers:

- `low`, the smallest unused number
- `high`, the largest unused number

Initially:

- `low = 0`
- `high = n`

Then we process the string from left to right:

- If we see `'I'`, we place the smallest available number and increment `low`
- If we see `'D'`, we place the largest available number and decrement `high`

At the end, exactly one number remains unused, and we append it.

Why does this work?

- When we encounter `'I'`, we want the current value to be smaller than the next one. Using the smallest available number guarantees that some larger number remains for the future.
- When we encounter `'D'`, we want the current value to be larger than the next one. Using the largest available number guarantees that some smaller number remains later.

This greedy strategy satisfies every constraint while using each number exactly once.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((n+1)!) | O(n) | Generates all permutations and checks each one |
| Optimal Greedy | O(n) | O(n) | Uses two pointers and constructs the answer directly |

## Algorithm Walkthrough

1. Initialize two pointers:

- `low = 0`
- `high = len(s)`

These represent the smallest and largest unused numbers available for the permutation.
2. Create an empty result array.

This array will store the permutation as we build it.
3. Iterate through each character in the string.

At every step, we decide whether to use the smallest or largest remaining number.
4. If the current character is `'I'`:

- Append `low` to the result
- Increment `low`

We choose the smallest available number because the next number must be larger.
5. If the current character is `'D'`:

- Append `high` to the result
- Decrement `high`

We choose the largest available number because the next number must be smaller.
6. After processing all characters, exactly one number remains unused.

At this point, `low == high`.
7. Append the remaining number to the result.

This completes the permutation.

### Why it works

The algorithm maintains a crucial invariant:

- All numbers smaller than `low` have already been used.
- All numbers larger than `high` have already been used.
- Every unused number lies within `[low, high]`.

When processing `'I'`, selecting the smallest remaining number guarantees that a larger unused number still exists later. Similarly, when processing `'D'`, selecting the largest remaining number guarantees that a smaller unused number remains later.

Because each step preserves the required inequality and every number is used exactly once, the final permutation is always valid.

## Python Solution

```
from typing import List

class Solution:
    def diStringMatch(self, s: str) -> List[int]:
        low = 0
        high = len(s)

        result = []

        for char in s:
            if char == 'I':
                result.append(low)
                low += 1
            else:
                result.append(high)
                high -= 1

        result.append(low)

        return result
```

The implementation follows the greedy strategy directly.

We start by initializing `low` and `high` to represent the smallest and largest available numbers. Since the permutation must contain values from `0` to `n`, the largest possible value is `len(s)`.

The loop processes each character in the string exactly once.

When the character is `'I'`, we append the smallest available number because the next value must be larger. After using that number, we increment `low`.

When the character is `'D'`, we append the largest available number because the next value must be smaller. After using that number, we decrement `high`.

After the loop finishes, only one number remains unused. At that moment, `low` and `high` are equal, so appending either one completes the permutation.

The solution is concise because the greedy logic naturally enforces all constraints.

## Go Solution

```
func diStringMatch(s string) []int {
    low := 0
    high := len(s)

    result := make([]int, 0, len(s)+1)

    for _, char := range s {
        if char == 'I' {
            result = append(result, low)
            low++
        } else {
            result = append(result, high)
            high--
        }
    }

    result = append(result, low)

    return result
}
```

The Go implementation mirrors the Python logic closely.

One notable difference is slice allocation. In Go, we preallocate the slice capacity using:

```
make([]int, 0, len(s)+1)
```

This avoids repeated reallocations during appends.

Go uses `rune` values when iterating over strings with `range`, so `char` is compared against `'I'` and `'D'` as rune literals.

There are no integer overflow concerns because the values are bounded by `10^5`.

## Edge Cases

### All Characters Are `'I'`

An input like `"IIII"` requires a strictly increasing permutation.

A naive implementation might accidentally reuse numbers or fail to maintain increasing order throughout the entire sequence. The greedy solution handles this naturally by always choosing the smallest remaining value. This produces:

```
[0, 1, 2, 3, 4]
```

which is guaranteed to be increasing.

### All Characters Are `'D'`

An input like `"DDDD"` requires a strictly decreasing permutation.

This can be tricky because every adjacent pair must satisfy the decreasing condition. The algorithm solves this by repeatedly selecting the largest remaining number first, producing:

```
[4, 3, 2, 1, 0]
```

Every comparison is valid.

### Alternating Patterns

Inputs such as `"IDIDID"` or `"DIDIDI"` are common sources of bugs because the required relationship changes at every step.

The two pointer approach works well here because it dynamically chooses either the smallest or largest remaining value depending on the current constraint. The remaining range always preserves future possibilities.

### Minimum Input Size

The smallest possible input length is `1`.

Examples:

- `"I"`
- `"D"`

These cases are important because the permutation contains only two numbers. Some implementations incorrectly assume larger arrays or mishandle the final append operation.

The current implementation correctly processes one character and then appends the single remaining value.
