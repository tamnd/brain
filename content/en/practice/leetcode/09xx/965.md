## Solution

## Problem Understanding

The problem asks us to determine whether a given binary tree is **uni-valued**, meaning that all nodes in the tree share the same value. The input is the root node of a binary tree, which can have up to 100 nodes, and each node’s value is an integer between 0 and 99. The output is a boolean: `true` if every node has the same value, and `false` otherwise.

In simpler terms, we traverse the tree and check that every node's value matches the root node’s value. If we ever encounter a node with a different value, we can immediately return `false`.

The constraints indicate that the tree is small (max 100 nodes), so even less efficient recursive or iterative approaches are feasible, but an optimal solution should avoid unnecessary traversals. The problem guarantees that the tree is non-empty, so we do not need to handle a `null` root as a special case.

Important edge cases include a tree with a single node, a tree where only one leaf differs in value, or a completely uniform tree. These help ensure that the algorithm correctly handles minimal input, subtle mismatches, and standard cases.

## Approaches

### Brute Force Approach

The brute force approach is simple: traverse every node in the tree, collect their values in a list or set, and then check if all values are identical. This works because if all values match, the set will have only one element. This approach guarantees correctness but is slightly wasteful in space because it stores all node values even though we could stop early if a mismatch occurs.

### Optimal Approach

The optimal solution leverages a single traversal with a simple comparison. We keep the root node value as a reference and traverse the tree using either depth-first search (DFS) or breadth-first search (BFS). At each node, we compare its value to the reference. If a mismatch occurs, we immediately return `false`. If we complete the traversal without mismatches, the tree is uni-valued. This approach is efficient because it does not store all node values and can terminate early.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Traverses the whole tree and stores values in a list or set |
| Optimal | O(n) | O(h) | Traverses the tree using DFS or BFS; early termination possible; h = tree height |

## Algorithm Walkthrough

1. Start by storing the value of the root node in a variable `target`. This is the value that every other node must match.
2. Define a recursive function `dfs(node)` that takes a tree node as input.
3. In `dfs(node)`, check if the node is `None`. If it is, return `true` because an empty subtree cannot violate the uni-value condition.
4. Compare the node's value to `target`. If it differs, return `false`.
5. Recursively call `dfs` on the left child and right child. If either recursive call returns `false`, propagate that result up.
6. Start the recursion with the root node and return its result.

**Why it works:** The invariant is that every subtree rooted at a node will only return `true` if all its descendants match the `target` value. Since recursion checks every node exactly once, the algorithm guarantees correctness and early termination as soon as a mismatch is found.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
from typing import Optional

class Solution:
    def isUnivalTree(self, root: Optional[TreeNode]) -> bool:
        target = root.val

        def dfs(node: Optional[TreeNode]) -> bool:
            if not node:
                return True
            if node.val != target:
                return False
            return dfs(node.left) and dfs(node.right)
        
        return dfs(root)
```

The `dfs` function checks each node against the target value, returning `false` immediately if a mismatch is found. If both left and right subtrees are uni-valued, it returns `true`. The recursion handles all nodes efficiently, avoiding unnecessary storage of values.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func isUnivalTree(root *TreeNode) bool {
    target := root.Val

    var dfs func(node *TreeNode) bool
    dfs = func(node *TreeNode) bool {
        if node == nil {
            return true
        }
        if node.Val != target {
            return false
        }
        return dfs(node.Left) && dfs(node.Right)
    }

    return dfs(root)
}
```

In Go, `nil` represents the absence of a child node, similar to `None` in Python. The recursive DFS logic mirrors the Python implementation, checking node values against the target and propagating any `false` result immediately.

## Worked Examples

### Example 1: `[1,1,1,1,1,null,1]`

| Node | Value | Left DFS | Right DFS | Return |
| --- | --- | --- | --- | --- |
| root | 1 | dfs(1) | dfs(1) | True |
| left | 1 | dfs(1) | dfs(1) | True |
| left.left | 1 | True | True | True |
| left.right | 1 | True | True | True |
| right | 1 | nil → True | dfs(1) | True |
| right.right | 1 | True | True | True |

All nodes match the target value 1, so the final result is `true`.

### Example 2: `[2,2,2,5,2]`

| Node | Value | Comparison | Result |
| --- | --- | --- | --- |
| root | 2 | 2 == 2 | continue |
| left | 2 | 2 == 2 | continue |
| left.left | 5 | 5 != 2 | return false |

The first mismatch occurs at node with value 5, so the algorithm terminates early and returns `false`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited once in DFS/BFS |
| Space | O(h) | Recursive call stack; h = height of the tree |

Even though the tree can have up to 100 nodes, the recursion stack will only reach the height of the tree. BFS using a queue would have the same O(n) space in the worst case.

## Edge Cases

A single-node tree is an edge case. The algorithm correctly returns `true` because there are no other nodes to compare.

A tree where only one leaf differs in value can be a subtle bug if the traversal does not check all paths. The recursive DFS handles this by checking every node, ensuring mismatches are caught anywhere in the tree.

A tree where all nodes except one branch have the same value tests early termination logic. The algorithm stops as soon as a mismatch is found, avoiding unnecessary checks and correctly returning `false`.
