## Solution

## Problem Understanding

The problem asks us to compute the sum of a non-negative integer represented in array form and a standard integer `k`, then return the result in the same array form. The **array-form** of an integer represents each digit of the number as an element in a list, in left-to-right order. For instance, the number `1321` is represented as `[1,3,2,1]`.

The inputs are `num`, a list of integers each in the range `[0,9]`, representing the digits of the number, and `k`, a standard integer. The output should also be a list of integers, representing the digits of the sum of the number and `k`.

The constraints give us several insights about the input size and characteristics. `num` can have up to `10^4` digits, meaning we cannot safely convert it into a single integer directly in languages with fixed integer sizes, though Python handles arbitrarily large integers. `k` is limited to `10^4`, so it is small relative to `num`. The array `num` does not have leading zeros except for zero itself, so we do not need to handle cases like `[0,1,2]`.

Important edge cases include when `num` has very few digits (like `[0]`), when `k` causes multiple carries that increase the number of digits, and when the sum of the last digit and `k` exceeds 10 repeatedly.

## Approaches

The **brute-force approach** is to convert the array-form `num` into an integer using arithmetic or string manipulation, add `k`, and then convert the result back to an array of digits. This works because the arithmetic addition is straightforward, but it is inefficient if the number of digits is very large and exceeds language-specific integer limits. Python handles it but other languages may not, and this approach also does not leverage the array structure efficiently.

The **optimal approach** is to simulate the addition as we would do by hand, adding `k` to `num` starting from the least significant digit (the rightmost). We can maintain a carry as we add each digit with the corresponding digit from `k`. We process `k` digit by digit using modulo and integer division. Once the sum is complete, we reverse the result to maintain the left-to-right order. This approach avoids converting the entire array into an integer and works efficiently regardless of the number of digits.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n + log k) | O(n + log k) | Convert array to integer, add, then convert back to array |
| Optimal | O(n + log k) | O(n + log k) | Add digit by digit from the right, handle carry, reverse result |

## Algorithm Walkthrough

1. Initialize an empty list `res` to store the resulting digits.
2. Start iterating `i` from the last index of `num` toward the first.
3. While `i >= 0` or `k > 0`, perform the following:

- Add `k % 10` to `num[i]` if `i >= 0`, else just use `k % 10`.
- Compute the digit to append to `res` as `(current_digit + carry) % 10`.
- Update `k` as `k // 10` to remove the processed digit and add any carry-over.
- Move to the next digit to the left by decrementing `i`.
4. If any carry remains after the loop, prepend it to `res`.
5. Reverse `res` to maintain left-to-right order, since we built it from least significant digit first.

**Why it works:** At every step, we maintain the invariant that `res` stores the sum of processed digits from the least significant end correctly. By using modulo and division to handle the addition of `k` digit by digit, we ensure each place value is computed correctly with proper carry-over, mirroring manual addition.

## Python Solution

```
from typing import List

class Solution:
    def addToArrayForm(self, num: List[int], k: int) -> List[int]:
        res = []
        i = len(num) - 1
        
        while i >= 0 or k > 0:
            if i >= 0:
                k += num[i]
            res.append(k % 10)
            k //= 10
            i -= 1
        
        return res[::-1]
```

**Explanation:** We initialize an empty list `res` to store digits. Iterating from the last index to the first, we add the current `num[i]` to `k`. We append the last digit of the sum (`k % 10`) to `res` and update `k` for the next iteration by integer division (`k //= 10`). Finally, we reverse the result to restore the correct order.

## Go Solution

```
func addToArrayForm(num []int, k int) []int {
    res := []int{}
    i := len(num) - 1
    
    for i >= 0 || k > 0 {
        if i >= 0 {
            k += num[i]
        }
        res = append(res, k % 10)
        k /= 10
        i--
    }
    
    // reverse res
    for l, r := 0, len(res)-1; l < r; l, r = l+1, r-1 {
        res[l], res[r] = res[r], res[l]
    }
    
    return res
}
```

**Go-specific notes:** Go does not have automatic reversal of slices, so we manually reverse `res` at the end. Appending to slices dynamically handles the storage. Integer overflow is not an issue here since `k` is small and the digits are managed individually.

## Worked Examples

**Example 1:** `num = [1,2,0,0], k = 34`

| i | k | res |
| --- | --- | --- |
| 3 | 34 + 0 = 34 | append 4 → res = [4], k = 3 |
| 2 | 3 + 0 = 3 | append 3 → res = [4,3], k = 0 |
| 1 | 0 + 2 = 2 | append 2 → res = [4,3,2], k = 0 |
| 0 | 0 + 1 = 1 | append 1 → res = [4,3,2,1], k = 0 |

Reverse res → `[1,2,3,4]`.

**Example 2:** `num = [2,7,4], k = 181`

| i | k | res |
| --- | --- | --- |
| 2 | 181 + 4 = 185 | append 5 → res = [5], k = 18 |
| 1 | 18 + 7 = 25 | append 5 → res = [5,5], k = 2 |
| 0 | 2 + 2 = 4 | append 4 → res = [5,5,4], k = 0 |

Reverse res → `[4,5,5]`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n + log k) | Each digit of `num` is processed once, plus each digit of `k` is processed |
| Space | O(n + log k) | The result array stores all digits of the sum |

The total time complexity combines the linear scan of `num` and the number of digits in `k`. Space complexity accounts for the result list which grows to the number of digits in the sum.
