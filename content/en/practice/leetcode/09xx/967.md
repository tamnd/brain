## Solution

## Problem Understanding

This problem asks us to generate all integers of a specific length `n` such that the absolute difference between every two consecutive digits is exactly `k`. In other words, for a number like `abc...`, the condition `abs(a-b) == k`, `abs(b-c) == k`, and so on, must hold for every pair of consecutive digits. The input integers are `n`, representing the number of digits, and `k`, representing the required difference between consecutive digits. The output is a list of integers satisfying this condition, and the order does not matter.

There are a few important constraints: numbers cannot have leading zeros, which eliminates sequences starting with `0` for `n > 1`. The value of `n` ranges from 2 to 9, so the algorithm must handle numbers with multiple digits efficiently. The difference `k` ranges from 0 to 9, and if `k` is 0, all digits in the number must be the same.

Edge cases include scenarios where `k = 0` (all digits are identical), `k = 9` (extreme jumps, only numbers like `10`, `92`, `81` are possible for `n=2`), or numbers starting at `1` or `9` where adding or subtracting `k` could produce an invalid digit.

## Approaches

The brute-force approach would generate every integer of length `n` and check if consecutive digits satisfy the difference `k`. For example, for `n=3`, this would involve generating numbers from `100` to `999` and checking each number. While this guarantees correctness, it is inefficient because it evaluates `9 * 10^(n-1)` numbers, which grows exponentially as `n` increases.

The optimal approach leverages **backtracking or breadth-first search**. Instead of generating all numbers, we can build numbers digit by digit. We start with a non-zero digit, then recursively append digits that satisfy the difference `k` with the last digit. This dramatically reduces the search space because invalid sequences are never expanded. BFS can also be used iteratively by maintaining a queue of partial numbers and extending them until they reach length `n`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(9 * 10^(n-1)) | O(1) | Generates all numbers and filters those with correct consecutive differences |
| Optimal | O(2 * 9 * 2^(n-1)) | O(2^(n-1)) | Builds numbers recursively/BFS using valid next digits only |

## Algorithm Walkthrough

1. Initialize an empty result list to store valid numbers.
2. Iterate over starting digits from 1 to 9 because numbers cannot have leading zeros.
3. Define a recursive helper function (or use BFS queue) that takes the current number and its length.
4. If the current number has reached length `n`, add it to the result list.
5. Otherwise, compute the next valid digits by adding and subtracting `k` from the last digit.
6. Only append next digits that are in the range 0 to 9.
7. Recurse with the new number and increment the length.
8. Return the final result list once all sequences have been explored.

Why it works: At every step, only digits that satisfy the consecutive difference requirement are considered. Since recursion or BFS ensures all possible sequences are generated, all valid numbers of length `n` are eventually produced, and invalid sequences are never formed.

## Python Solution

```
from typing import List

class Solution:
    def numsSameConsecDiff(self, n: int, k: int) -> List[int]:
        if n == 1:
            return [i for i in range(10)]
        
        result = []

        def dfs(num: int, length: int):
            if length == n:
                result.append(num)
                return
            last_digit = num % 10
            next_digits = set([last_digit + k, last_digit - k])
            for next_digit in next_digits:
                if 0 <= next_digit <= 9:
                    dfs(num * 10 + next_digit, length + 1)

        for start_digit in range(1, 10):
            dfs(start_digit, 1)

        return result
```

The Python implementation uses a depth-first search to construct numbers. The `dfs` function takes the current number and its length, then explores adding digits that satisfy the consecutive difference condition. Using a set avoids duplicates when `k=0`. The recursion stops when a number reaches length `n` and adds it to the result.

## Go Solution

```
func numsSameConsecDiff(n int, k int) []int {
    if n == 1 {
        res := make([]int, 10)
        for i := 0; i < 10; i++ {
            res[i] = i
        }
        return res
    }

    var result []int

    var dfs func(num, length int)
    dfs = func(num, length int) {
        if length == n {
            result = append(result, num)
            return
        }
        lastDigit := num % 10
        nextDigits := map[int]struct{}{lastDigit + k: {}, lastDigit - k: {}}
        for nextDigit := range nextDigits {
            if nextDigit >= 0 && nextDigit <= 9 {
                dfs(num*10+nextDigit, length+1)
            }
        }
    }

    for start := 1; start <= 9; start++ {
        dfs(start, 1)
    }

    return result
}
```

The Go version mirrors the Python DFS logic. It uses a map to avoid duplicate digits when `k=0`. Since Go requires pre-declaring slices for append operations, `result` is initialized as an empty slice. The recursion builds numbers digit by digit and appends valid numbers to the slice.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(2 * 9 * 2^(n-1)) | Each digit can branch into at most 2 valid digits, starting from 9 options |
| Space | O(2^(n-1)) | Recursion stack depth up to n, plus result storage for all valid numbers |

This complexity arises because each step can branch to two numbers (add/subtract `k`), but digits are constrained between 0 and 9. The result storage also scales with the number of valid sequences.
