## Solution

## Problem Understanding

In this problem, we are given a list of steel rods. Each rod has a certain length, and every rod can be used at most once. We want to build a billboard supported by two vertical steel supports, where both supports must have exactly the same total height.

The rods can be combined together by welding them, so each support may consist of multiple rods. The goal is to maximize the common height of the two supports.

Another way to think about the problem is this:

- We must divide the rods into three groups:

- rods used in the left support
- rods used in the right support
- rods not used at all
- The sum of the left support and the sum of the right support must be equal.
- Among all valid equal-height constructions, we want the largest possible height.

For example, with rods `[1,2,3,6]`, we can create:

- Left support: `1 + 2 + 3 = 6`
- Right support: `6`

Both supports have equal height `6`, so the answer is `6`.

The constraints are important:

- At most `20` rods
- Each rod length is at most `1000`
- Total sum of all rods is at most `5000`

These constraints immediately suggest that exhaustive subset enumeration may become expensive. Since every rod has three choices, left/right/unused, a naive brute-force approach explores `3^n` states. With `n = 20`, this becomes too large.

The total rod sum being limited to `5000` is the key observation. It suggests that dynamic programming based on differences between support heights may be feasible.

There are several important edge cases:

- It may be impossible to build two equal supports at all, such as `[1,2]`.
- Multiple different subsets may produce the same height difference, and we only want to keep the best one.
- Some rods may optimally be ignored entirely.
- The best solution does not necessarily use all rods.
- Very large rods may dominate smaller ones, creating many overlapping DP states.

## Approaches

### Brute Force

The brute-force approach tries every possible assignment of rods.

For each rod, we have three choices:

1. Put it in the left support
2. Put it in the right support
3. Ignore it

At the end, we check whether the two supports have equal height. If they do, we update the answer with that height.

This approach is correct because it explores every possible valid configuration. However, its complexity is exponential.

With `n` rods, there are `3^n` possible assignments. For `n = 20`, this is approximately:

$$3^{20} \approx 3.4 \text{ billion}$$

which is far too large.

### Key Insight

The crucial observation is that we do not need to track the exact composition of both supports independently.

Instead, we only need:

- the difference between the two support heights
- the maximum achievable smaller support height for that difference

Suppose:

- taller support height = `T`
- shorter support height = `S`
- difference = `T - S`

If we know the difference and the shorter height, then we implicitly know the taller height as well.

This dramatically reduces the state space.

We use dynamic programming where:

- `dp[diff] = maximum shorter support height achievable`

For every rod, we update states in three ways:

- ignore the rod
- add it to the taller side
- add it to the shorter side

Because the total sum is at most `5000`, the number of possible differences is manageable.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(3^n) | O(n) | Tries every left/right/unused assignment |
| Optimal Dynamic Programming | O(n * sum(rods)) | O(sum(rods)) | Tracks height differences efficiently |

## Algorithm Walkthrough

### Dynamic Programming State

We maintain a hash map:

```
dp[diff] = maximum smaller support height
```

Interpretation:

- `diff` represents the difference between the taller and shorter supports.
- The value represents the maximum possible shorter support height for that difference.

Initially:

```
dp = {0: 0}
```

This means:

- difference `0`
- both supports have height `0`

### Processing Each Rod

For every rod, we create updates from the current states.

We iterate over a snapshot of the current DP map because we must avoid modifying the map while iterating.

Suppose we are currently at:

```
diff = d
shorter = s
```

Then:

```
taller = s + d
```

Now we consider three choices.

### 1. Ignore the Rod

We do nothing.

The current state remains valid.

### 2. Add Rod to Taller Support

If we add rod `r` to the taller side:

```
new_diff = d + r
new_shorter = s
```

The shorter side does not change.

We update:

```
dp[new_diff] = max(dp[new_diff], s)
```

### 3. Add Rod to Shorter Support

If we add rod `r` to the shorter side, two cases arise.

If the shorter side remains shorter:

```
new_diff = d - r
```

If it overtakes the taller side:

```
new_diff = r - d
```

In both cases:

```
new_shorter = min(taller, shorter + r)
```

A compact formula is:

```
new_diff = abs(d - r)
new_shorter = s + min(d, r)
```

Why does this work?

- If `r <= d`, the shorter side gains `r`
- If `r > d`, the previous taller side becomes the new shorter side, so the gain is only `d`

We update:

```
dp[new_diff] = max(dp[new_diff], new_shorter)
```

### Final Answer

At the end:

```
dp[0]
```

represents the largest equal-height billboard.

### Why it works

The invariant is:

```
dp[diff] always stores the maximum achievable shorter support height
for that exact difference.
```

Every rod transition correctly models all valid choices:

- unused
- added to taller side
- added to shorter side

Because every possible construction can be represented as a sequence of these choices, and because we always preserve the best shorter height for each difference, the DP eventually computes the optimal equal-height configuration.

When `diff = 0`, both supports are equal, so the stored shorter height is exactly the answer.

## Python Solution

```
from typing import List

class Solution:
    def tallestBillboard(self, rods: List[int]) -> int:
        dp = {0: 0}

        for rod in rods:
            current = dp.copy()

            for diff, shorter in current.items():
                # Add rod to taller side
                new_diff = diff + rod
                dp[new_diff] = max(dp.get(new_diff, 0), shorter)

                # Add rod to shorter side
                new_diff = abs(diff - rod)
                new_shorter = shorter + min(diff, rod)

                dp[new_diff] = max(
                    dp.get(new_diff, 0),
                    new_shorter
                )

        return dp[0]
```

The implementation starts with a dictionary containing a single state:

```
{0: 0}
```

This represents two empty supports with equal height.

For each rod, we iterate over a snapshot of the current DP states. Using `current = dp.copy()` is essential because we must base transitions only on states that existed before processing the current rod.

The first transition adds the rod to the taller support. This increases the difference but leaves the shorter support unchanged.

The second transition adds the rod to the shorter support. The formula:

```
shorter + min(diff, rod)
```

correctly computes the new smaller support height after balancing.

At every step, we keep only the best possible shorter height for each difference. Inferior states are discarded automatically through the `max` operation.

Finally, `dp[0]` contains the maximum equal support height.

## Go Solution

```
package main

func max(a, b int) int {
	if a > b {
		return a
	}
	return b
}

func abs(x int) int {
	if x < 0 {
		return -x
	}
	return x
}

func tallestBillboard(rods []int) int {
	dp := map[int]int{
		0: 0,
	}

	for _, rod := range rods {
		current := make(map[int]int)

		for k, v := range dp {
			current[k] = v
		}

		for diff, shorter := range current {
			// Add rod to taller side
			newDiff := diff + rod
			dp[newDiff] = max(dp[newDiff], shorter)

			// Add rod to shorter side
			newDiff = abs(diff - rod)
			newShorter := shorter + min(diff, rod)

			dp[newDiff] = max(dp[newDiff], newShorter)
		}
	}

	return dp[0]
}

func min(a, b int) int {
	if a < b {
		return a
	}
	return b
}
```

The Go implementation follows the same dynamic programming logic as the Python solution.

One important difference is that Go maps return the zero value for missing keys. This works naturally for this problem because all stored heights are non-negative.

Since Go does not allow safe modification during map iteration in the same conceptual way as Python, we explicitly create a copy of the map before processing transitions for each rod.

Integer overflow is not a concern because the maximum total rod sum is only `5000`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * sum(rods)) | Each rod processes all reachable differences |
| Space | O(sum(rods)) | DP stores states for possible differences |

The maximum possible difference is bounded by the total rod sum, which is at most `5000`.

For each rod, we iterate through all currently reachable differences. Since there are at most `5000` differences, the overall complexity becomes:

$$O(n \times 5000)$$

which is efficient for the given constraints.

The space complexity is also bounded by the number of distinct differences.

## Edge Cases

One important edge case is when no equal supports can be formed. Inputs like `[1,2]` or `[5]` cannot produce two supports with the same height. A buggy implementation might accidentally return a non-zero difference state instead of requiring exact equality. This implementation correctly returns `dp[0]`, ensuring only perfectly balanced supports count.

Another important case is when rods should be ignored. For example, in `[1,2,3,6]`, using all rods is impossible if both supports must remain equal. The optimal solution ignores nothing here, but many other cases require discarding rods entirely. Since every rod has an implicit "unused" option, the DP naturally handles this situation.

A third tricky case occurs when adding a rod to the shorter side causes it to become taller than the previous taller side. This is where many incorrect implementations fail. The transition:

```
new_shorter = shorter + min(diff, rod)
```

carefully handles both possibilities, whether the shorter side remains shorter or overtakes the taller side. This guarantees the DP state remains consistent regardless of how the balance shifts.
