## Solution

## Problem Understanding

The problem asks us to determine if two nodes in a binary tree are **cousins**. In a binary tree, two nodes are cousins if they satisfy two conditions: they are at the same depth, and they have different parents. The input provides a tree `root` and two integers `x` and `y` representing the values of the nodes. The output is a boolean indicating whether these nodes meet the cousin criteria.

The tree has unique values, so each node can be uniquely identified by its value. The constraints limit the number of nodes to `[2, 100]` and the values to `[1, 100]`, ensuring the input is small enough to allow straightforward traversal techniques like depth-first search (DFS) or breadth-first search (BFS) without performance concerns.

Important edge cases include nodes that are direct siblings, nodes at different depths, and nodes located in opposite subtrees. The problem guarantees that `x` and `y` exist in the tree and are distinct, simplifying the solution because we do not need to handle missing values or identical nodes.

## Approaches

The brute-force approach is to traverse the tree and, for every node, check the depth of `x` and `y` and their respective parents. This can be done using DFS to record the parent and depth of each node. After traversal, we simply compare the depths and parent values to determine if they are cousins. This approach is correct but could involve unnecessary checks if implemented naively.

The optimal approach leverages a **single-pass DFS or BFS** to find both nodes while simultaneously recording their depths and parents. Once both nodes are found, we immediately compare their depths and parents. BFS is particularly useful here because it traverses level by level, allowing us to identify the nodes at the same depth quickly and terminate early once both nodes are found.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Traverse entire tree and record parent/depth for all nodes |
| Optimal (DFS or BFS) | O(n) | O(h) | Track parent and depth while traversing; early termination possible; h is tree height |

## Algorithm Walkthrough

1. Initialize variables to store the depth and parent for nodes `x` and `y`.
2. Use DFS (or BFS) to traverse the tree. At each node:

- If the node matches `x` or `y`, record its parent and depth.
- Continue traversal recursively for left and right children if using DFS, or enqueue children if using BFS.
3. After traversal, check the following conditions:

- Depth of `x` equals depth of `y`.
- Parent of `x` is not equal to parent of `y`.
4. If both conditions are satisfied, return `True`; otherwise, return `False`.

**Why it works:** The algorithm guarantees correctness because it explicitly tracks both the parent and depth of each target node. By directly comparing these two properties, it ensures the nodes are cousins according to the formal definition.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def isCousins(self, root: Optional[TreeNode], x: int, y: int) -> bool:
        parent = {}
        depth = {}
        
        def dfs(node: Optional[TreeNode], par: Optional[TreeNode], d: int):
            if not node:
                return
            if node.val == x or node.val == y:
                parent[node.val] = par
                depth[node.val] = d
            dfs(node.left, node, d + 1)
            dfs(node.right, node, d + 1)
        
        dfs(root, None, 0)
        return depth[x] == depth[y] and parent[x] != parent[y]
```

The DFS function records the parent and depth for both `x` and `y`. The dictionary `parent` maps each node value to its parent node, and `depth` stores the depth. After DFS, comparing `depth` and `parent` values directly tells us if they are cousins.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func isCousins(root *TreeNode, x int, y int) bool {
    parent := make(map[int]*TreeNode)
    depth := make(map[int]int)
    
    var dfs func(node *TreeNode, par *TreeNode, d int)
    dfs = func(node *TreeNode, par *TreeNode, d int) {
        if node == nil {
            return
        }
        if node.Val == x || node.Val == y {
            parent[node.Val] = par
            depth[node.Val] = d
        }
        dfs(node.Left, node, d+1)
        dfs(node.Right, node, d+1)
    }
    
    dfs(root, nil, 0)
    return depth[x] == depth[y] && parent[x] != parent[y]
}
```

In Go, `nil` is used instead of `None`, and maps are used for tracking depth and parent relationships. The DFS traversal logic is identical to Python, recording depth and parent and then comparing them after traversal.

## Worked Examples

**Example 1:** `root = [1,2,3,4], x = 4, y = 3`

DFS traversal finds `4` at depth 2 with parent `2` and `3` at depth 1 with parent `1`. Depths differ, so output is `False`.

**Example 2:** `root = [1,2,3,null,4,null,5], x = 5, y = 4`

DFS traversal finds `5` at depth 2 with parent `3` and `4` at depth 2 with parent `2`. Depths match, parents differ, so output is `True`.

**Example 3:** `root = [1,2,3,null,4], x = 2, y = 3`

DFS traversal finds `2` at depth 1 with parent `1` and `3` at depth 1 with parent `1`. Depths match, but parents are the same, so output is `False`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited once during DFS traversal |
| Space | O(h) | Recursion stack in DFS is proportional to tree height; maps store at most two entries |

The time complexity is linear because every node may need to be examined. Space complexity is dominated by recursion depth for DFS, which is the height of the tree. Maps only track two nodes, so they contribute constant extra space.

## Edge Cases

One important edge case is when the nodes are direct siblings, sharing the same parent. This could incorrectly be flagged as cousins if only depth is considered. Our implementation explicitly compares parent nodes to prevent this.

Another edge case is when one node is much deeper than the other. Our DFS correctly records depth during traversal and ensures that the comparison fails if depths differ.

A third edge case is when both nodes are leaves in opposite subtrees. Even though they are far apart in the tree, as long as their depths match and parents differ, they are cousins. The DFS and BFS approaches correctly track depth and parent regardless of position, handling this scenario accurately.
