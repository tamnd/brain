## Solution

## Problem Understanding

The problem gives us an array of unique strings called `words`. Our task is to construct the shortest possible string such that every word in the array appears somewhere inside the final result as a substring.

A substring means the characters appear contiguously. For example, `"leetcode"` is a substring of `"alexlovesleetcode"` because the exact sequence of characters appears without interruption.

The important detail is that the words can overlap. If the suffix of one word matches the prefix of another word, we can merge them and avoid repeating characters. For example:

- `"abc"` and `"bcd"` can become `"abcd"`
- The overlap is `"bc"`

The objective is therefore not simply concatenation, but optimal merging.

The constraints are critical for choosing the algorithm:

- `words.length <= 12`
- Each word length is at most `20`

The small number of words strongly suggests an exponential algorithm with memoization or bitmask dynamic programming is intended. A brute-force solution over all permutations is technically possible for `12` words, but `12!` is far too large to run efficiently.

The problem also guarantees that no word is already a substring of another word. This simplifies the logic because we never need to remove redundant words beforehand.

Several edge cases are important:

- Words may have no overlap at all, forcing full concatenation.
- Multiple overlaps may exist, and choosing the locally largest overlap is not always globally optimal.
- The optimal path may require sacrificing a large immediate overlap to create better total compression later.
- There may be multiple shortest answers, and any valid shortest superstring is accepted.

## Approaches

### Brute Force Approach

The most direct solution is to generate every possible ordering of the words. For each permutation, we repeatedly merge adjacent strings using the maximum possible overlap.

For example, if the permutation is:

```
["gcta", "ctaagt", "ttca"]
```

we would merge:

```
gcta + ctaagt -> gctaagt
gctaagt + ttca -> gctaagttca
```

After constructing the resulting superstring for every permutation, we return the shortest one.

This approach is correct because every valid superstring corresponds to some ordering of the words. By checking all permutations, we are guaranteed to eventually discover the optimal arrangement.

However, the runtime becomes prohibitively expensive. With `n` words, there are `n!` permutations. Even for `n = 12`, this exceeds hundreds of millions of possibilities.

The key inefficiency is that many permutations share partial states. For example, multiple orderings may produce the same subset of already-used words ending with the same last word.

### Key Insight

The crucial observation is that the problem resembles the Traveling Salesman Problem.

Instead of thinking about building strings directly, think of:

- each word as a node
- the overlap between two words as an edge weight

If word `A` overlaps with word `B` by `k` characters, then appending `B` after `A` costs only:

```
len(B) - k
```

The optimal solution becomes:

> Find the ordering of words that maximizes total overlap.

Because `n <= 12`, we can use bitmask dynamic programming.

The DP state is:

```
dp[mask][last]
```

which represents the maximum overlap achievable using the set of words in `mask` and ending at word `last`.

This reduces the factorial explosion into a manageable exponential complexity.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n! * n * L) | O(n * L) | Tries every ordering of words |
| Optimal Bitmask DP | O(n² * 2ⁿ + n² * L) | O(n * 2ⁿ) | Uses overlap graph and DP over subsets |

Here:

- `n` is the number of words
- `L` is the maximum word length

## Algorithm Walkthrough

### Step 1: Precompute Pairwise Overlaps

For every pair of words `(i, j)`, compute the largest suffix of `words[i]` that matches a prefix of `words[j]`.

For example:

```
words[i] = "catg"
words[j] = "atgcatc"
```

The overlap is `"atg"` of length `3`.

We store this in:

```
overlap[i][j]
```

This preprocessing allows us to instantly know how many characters can be reused when transitioning from one word to another.

### Step 2: Define the DP State

We create:

```
dp[mask][last]
```

where:

- `mask` represents which words have already been used
- `last` is the index of the final word in the current sequence
- the value stores the maximum total overlap achieved

We also maintain a `parent` table to reconstruct the path later.

### Step 3: Initialize Base Cases

If only one word is used:

```
dp[1 << i][i] = 0
```

No overlap exists yet because there is only a single word.

### Step 4: Transition Between States

For every state `(mask, last)`:

Try appending a new word `next_word` not yet in the mask.

The new overlap becomes:

```
dp[mask][last] + overlap[last][next_word]
```

If this value improves the best known result for the new state, update it.

### Step 5: Find the Best Ending Word

After filling the DP table, examine:

```
dp[(1 << n) - 1][i]
```

for all `i`.

This corresponds to using every word exactly once and ending at word `i`.

Choose the ending word with the maximum total overlap.

### Step 6: Reconstruct the Optimal Ordering

Using the `parent` table, backtrack from the best final word.

This reconstructs the order of words in the optimal superstring.

### Step 7: Build the Final Superstring

Start with the first word.

For every subsequent word, append only the non-overlapping portion.

If overlap is `k`:

```
append words[j][k:]
```

instead of the entire word.

This produces the shortest possible superstring.

### Why it works

The DP invariant is:

> `dp[mask][last]` always stores the maximum total overlap achievable for that exact subset and ending word.

Because every transition considers all possible previous endings, the DP explores every valid ordering implicitly. Maximizing overlap is equivalent to minimizing final string length, since every reused character shortens the result by one. Therefore, the reconstructed sequence produces an optimal shortest superstring.

## Python Solution

```
from typing import List

class Solution:
    def shortestSuperstring(self, words: List[str]) -> str:
        n = len(words)

        # overlap[i][j] = maximum overlap between
        # suffix of words[i] and prefix of words[j]
        overlap = [[0] * n for _ in range(n)]

        for i in range(n):
            for j in range(n):
                if i == j:
                    continue

                max_overlap = min(len(words[i]), len(words[j]))

                for k in range(max_overlap, 0, -1):
                    if words[i][-k:] == words[j][:k]:
                        overlap[i][j] = k
                        break

        size = 1 << n

        dp = [[0] * n for _ in range(size)]
        parent = [[-1] * n for _ in range(size)]

        for mask in range(size):
            for last in range(n):
                if not (mask & (1 << last)):
                    continue

                prev_mask = mask ^ (1 << last)

                if prev_mask == 0:
                    continue

                for prev in range(n):
                    if not (prev_mask & (1 << prev)):
                        continue

                    candidate = (
                        dp[prev_mask][prev]
                        + overlap[prev][last]
                    )

                    if candidate > dp[mask][last]:
                        dp[mask][last] = candidate
                        parent[mask][last] = prev

        full_mask = size - 1

        best_last = 0
        best_overlap = 0

        for i in range(n):
            if dp[full_mask][i] > best_overlap:
                best_overlap = dp[full_mask][i]
                best_last = i

        order = []

        mask = full_mask
        last = best_last

        while last != -1:
            order.append(last)

            next_last = parent[mask][last]
            mask ^= (1 << last)
            last = next_last

        order.reverse()

        superstring = words[order[0]]

        for i in range(1, len(order)):
            prev_word = order[i - 1]
            curr_word = order[i]

            k = overlap[prev_word][curr_word]

            superstring += words[curr_word][k:]

        return superstring
```

The implementation begins by computing all pairwise overlaps. This preprocessing phase is essential because the DP repeatedly needs overlap information. Without preprocessing, overlap checks would be recomputed many times.

The DP table stores the maximum overlap achievable for every subset and ending word. Since subsets are represented by bitmasks, we can efficiently determine whether a word has already been used.

The transition step iterates over all possible previous words and updates the best overlap value for the current state. The `parent` table remembers which previous word produced the optimal result.

Once the DP is complete, we identify the best final word and reconstruct the optimal ordering through backtracking.

Finally, the superstring is constructed incrementally. Instead of appending entire words, only the non-overlapping suffix is added, ensuring the shortest possible result.

## Go Solution

```
package main

func shortestSuperstring(words []string) string {
	n := len(words)

	overlap := make([][]int, n)
	for i := range overlap {
		overlap[i] = make([]int, n)
	}

	// Compute overlaps
	for i := 0; i < n; i++ {
		for j := 0; j < n; j++ {
			if i == j {
				continue
			}

			maxOverlap := min(len(words[i]), len(words[j]))

			for k := maxOverlap; k >= 1; k-- {
				if words[i][len(words[i])-k:] == words[j][:k] {
					overlap[i][j] = k
					break
				}
			}
		}
	}

	size := 1 << n

	dp := make([][]int, size)
	parent := make([][]int, size)

	for i := range dp {
		dp[i] = make([]int, n)
		parent[i] = make([]int, n)

		for j := range parent[i] {
			parent[i][j] = -1
		}
	}

	// Bitmask DP
	for mask := 0; mask < size; mask++ {
		for last := 0; last < n; last++ {
			if (mask & (1 << last)) == 0 {
				continue
			}

			prevMask := mask ^ (1 << last)

			if prevMask == 0 {
				continue
			}

			for prev := 0; prev < n; prev++ {
				if (prevMask & (1 << prev)) == 0 {
					continue
				}

				candidate := dp[prevMask][prev] + overlap[prev][last]

				if candidate > dp[mask][last] {
					dp[mask][last] = candidate
					parent[mask][last] = prev
				}
			}
		}
	}

	fullMask := size - 1

	bestLast := 0
	bestOverlap := 0

	for i := 0; i < n; i++ {
		if dp[fullMask][i] > bestOverlap {
			bestOverlap = dp[fullMask][i]
			bestLast = i
		}
	}

	// Reconstruct order
	order := []int{}

	mask := fullMask
	last := bestLast

	for last != -1 {
		order = append(order, last)

		nextLast := parent[mask][last]
		mask ^= (1 << last)
		last = nextLast
	}

	// Reverse order
	for i, j := 0, len(order)-1; i < j; i, j = i+1, j-1 {
		order[i], order[j] = order[j], order[i]
	}

	// Build result
	result := words[order[0]]

	for i := 1; i < len(order); i++ {
		prev := order[i-1]
		curr := order[i]

		k := overlap[prev][curr]

		result += words[curr][k:]
	}

	return result
}

func min(a, b int) int {
	if a < b {
		return a
	}
	return b
}
```

The Go implementation follows the same algorithmic structure as the Python version, but there are several language-specific considerations.

Go does not provide dynamic multidimensional arrays automatically, so the DP and overlap tables are explicitly allocated using nested slices.

The `parent` table must be initialized with `-1` manually because Go slices default to `0`.

String slicing in Go works directly on byte indices, which is safe here because the problem guarantees lowercase English letters only.

Unlike Python, Go requires an explicit helper function for `min`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n² * 2ⁿ + n² * L²) | DP transitions plus overlap computation |
| Space | O(n * 2ⁿ) | DP and parent tables |

The overlap preprocessing compares every pair of words. For each pair, we may test overlaps of different lengths, producing approximately `O(n² * L²)` complexity.

The dominant DP portion contains:

- `2ⁿ` subset states
- `n` possible ending words
- `n` possible transitions

giving:

```
O(n² * 2ⁿ)
```

Because `n <= 12`, this complexity is practical.

## Edge Cases

### No Overlaps Between Any Words

If none of the words share suffix-prefix matches, every overlap value becomes zero. A naive implementation might incorrectly assume overlaps always exist and accidentally omit characters.

The implementation handles this naturally because the overlap table defaults to `0`. In that situation, the DP simply chooses an arbitrary ordering and concatenates all words completely.

### Multiple Optimal Answers

Several different orderings may produce superstrings of the same minimum length. This can happen when overlap totals are tied.

The problem explicitly allows returning any shortest valid superstring. The DP implementation stores whichever optimal transition it encounters first, which is sufficient for correctness.

### Repeated Characters and Ambiguous Overlaps

Words with repeated patterns such as:

```
["aaaa", "aaab", "aaba"]
```

can easily break incorrect overlap calculations. A buggy implementation might stop at the first match instead of the largest match.

The preprocessing step checks overlap lengths from largest to smallest, ensuring the maximum valid overlap is always selected.

### Single Word Input

When the array contains only one word, the shortest superstring is simply the word itself.

The DP initialization handles this correctly because the reconstruction phase immediately returns that word without requiring transitions.

### Dense Overlap Graphs

In some inputs, every word overlaps with many others. Greedy algorithms often fail here because the locally largest overlap may not lead to the globally shortest superstring.

The bitmask DP avoids this pitfall by exploring all subset states and preserving the globally optimal overlap total.
