## Solution

## Problem Understanding

The problem asks us to count how many distinct non-empty subsequences can be formed from a given string `s`.

A subsequence is created by deleting zero or more characters while preserving the original relative order of the remaining characters. For example, `"ace"` is a subsequence of `"abcde"` because we can remove `'b'` and `'d'` while keeping the order intact.

The important detail is that we are counting only distinct subsequences. If multiple deletion patterns produce the same resulting string, it should only be counted once.

For example, in `"aba"`:

- Choosing the first `'a'` gives subsequence `"a"`
- Choosing the last `'a'` also gives subsequence `"a"`

These should count as only one distinct subsequence.

The input consists of:

- A single string `s`
- Length between `1` and `2000`
- Only lowercase English letters

The output should be:

- The total number of distinct non-empty subsequences
- Returned modulo `10^9 + 7`

The constraint `n <= 2000` is extremely important. The total number of subsequences of a string of length `n` is `2^n`, which becomes astronomically large even for moderate values of `n`. This immediately tells us that generating all subsequences explicitly is impossible.

Several edge cases are important:

- Strings with all unique characters, such as `"abc"`, maximize the number of subsequences
- Strings with repeated characters, such as `"aaa"`, create duplicate subsequences that must not be double counted
- Strings where repeated characters appear far apart, such as `"abca"`, require careful handling because duplicates can arise from multiple positions
- Very long strings require an efficient dynamic programming solution rather than brute force enumeration

## Approaches

### Brute Force Approach

The most straightforward approach is to generate every possible subsequence using recursion or bitmasking.

For each character, we have two choices:

- Include it in the subsequence
- Exclude it from the subsequence

This produces `2^n` total subsequences. We can insert each generated subsequence into a set to remove duplicates, then return the set size minus one to exclude the empty subsequence.

This approach is correct because it explicitly enumerates every possible subsequence and uses a set to ensure uniqueness.

However, it is far too slow and memory intensive for `n = 2000`.

Even for `n = 50`, the number of subsequences exceeds one quadrillion.

### Key Insight for the Optimal Solution

The key observation is that when we process a new character, every existing distinct subsequence can either:

- Ignore the new character
- Append the new character

If there are currently `x` distinct subsequences, adding a new character appears to double the count.

However, duplicates arise when the same character has appeared before.

Suppose we already processed a previous `'a'`. When we encounter another `'a'`, appending it to older subsequences may recreate subsequences that were already formed when processing the earlier `'a'`.

Therefore, we need a way to subtract the duplicated subsequences contributed by the previous occurrence of the same character.

This naturally leads to a dynamic programming solution.

We maintain:

- `total`, the number of distinct subsequences including the empty subsequence
- `last[c]`, the number of subsequences that existed before the previous occurrence of character `c`

When processing a character:

- Doubling creates all possible extensions
- Subtracting `last[c]` removes duplicates

This gives an efficient linear-time solution.

## Algorithm Walkthrough

### Optimal Dynamic Programming Algorithm

1. Initialize a variable `total = 1`.

This represents the number of distinct subsequences including the empty subsequence. Initially, only the empty subsequence exists.
2. Create an array `last` of size 26 initialized to `0`.

`last[i]` stores the value of `total` before the previous occurrence of character `i`.
3. Iterate through each character `ch` in the string.
4. Before updating anything, store the current value of `total` in a temporary variable `previous_total`.

We need this because after processing the current character, the old value will be needed for future duplicate correction.
5. Double the current number of subsequences.

Every existing subsequence can either:

- Stay unchanged
- Append the current character

So the count becomes:

$\text{newTotal} = 2 \cdot \text{total}$
6. Subtract the duplicated subsequences caused by previous occurrences of the same character.

If this character appeared before, some newly formed subsequences already existed.

The number of duplicated subsequences equals the value stored in `last[ch]`.

So we compute:

$\text{newTotal} = 2 \cdot \text{total} - \text{last}[ch]$
7. Take modulo `10^9 + 7`.

Since subtraction may produce a negative intermediate value, we add the modulus before taking `% MOD`.
8. Update `last[ch] = previous_total`.

This records the number of subsequences that existed before this occurrence of the character.
9. Continue until all characters are processed.
10. Subtract one from the final result to exclude the empty subsequence.

### Why it works

The algorithm maintains the invariant that `total` always equals the number of distinct subsequences including the empty subsequence after processing the current prefix of the string.

When processing a character, every existing subsequence can produce a new subsequence by appending the character. This doubles the count. However, if the character appeared previously, some of these new subsequences are duplicates of subsequences already created during the earlier occurrence. Subtracting `last[ch]` removes exactly those duplicates, ensuring each distinct subsequence is counted once.

## Python Solution

```
class Solution:
    def distinctSubseqII(self, s: str) -> int:
        MOD = 10**9 + 7
        
        total = 1
        last = [0] * 26
        
        for ch in s:
            index = ord(ch) - ord('a')
            
            previous_total = total
            
            total = (2 * total - last[index]) % MOD
            
            last[index] = previous_total
        
        return (total - 1) % MOD
```

The implementation follows the dynamic programming strategy directly.

`total` tracks the number of distinct subsequences including the empty subsequence. Starting from `1` simplifies the recurrence because the empty subsequence participates naturally in the doubling process.

The `last` array stores information for each lowercase letter. Since the input contains only lowercase English letters, a fixed-size array of length 26 is more efficient than a hash map.

For each character:

- We first save the old `total`
- We double the count because every subsequence can append the current character
- We subtract the duplicated subsequences contributed by the previous occurrence of the same character
- We store the old total in `last`

Finally, we subtract one to exclude the empty subsequence from the answer.

The modulo operation ensures values remain within bounds throughout computation.

## Go Solution

```
func distinctSubseqII(s string) int {
    const MOD int = 1e9 + 7

    total := 1
    last := make([]int, 26)

    for _, ch := range s {
        index := ch - 'a'

        previousTotal := total

        total = (2*total - last[index] + MOD) % MOD

        last[index] = previousTotal
    }

    return (total - 1 + MOD) % MOD
}
```

The Go implementation mirrors the Python logic closely.

One important difference is handling negative values during modulo arithmetic. In Go, `%` with negative numbers can remain negative, so we explicitly add `MOD` before taking `% MOD`.

The `last` structure is implemented as a fixed-size integer slice of length 26 for constant-time access.

Go uses rune iteration in the `for _, ch := range s` loop, but since the input contains only lowercase English letters, subtracting `'a'` works safely.
