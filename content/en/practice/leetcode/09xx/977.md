## Solution

## Problem Understanding

The problem gives us an integer array `nums` that is already sorted in non-decreasing order. This means the values increase from left to right, but negative numbers are allowed. We must return a new array containing the square of every number, and the resulting array must also be sorted in non-decreasing order.

At first glance, the task appears straightforward. We could square every number and then sort the resulting array. However, the follow-up explicitly asks for an `O(n)` solution, which means we should avoid sorting after squaring if possible.

The key detail is that squaring changes the ordering behavior of negative numbers. In the original sorted array, large negative values appear on the left side. After squaring, those same values can become very large positive numbers. For example:

```
[-7, -3, 2, 3, 11]
```

becomes:

```
[49, 9, 4, 9, 121]
```

The squared values are no longer sorted because `49` from `-7` is larger than several values to its right.

The constraints tell us several important things:

- The array length can be up to `10^4`, which is large enough that inefficient approaches should be avoided.
- Values range from `-10^4` to `10^4`, so squaring produces values up to `10^8`, which safely fits within standard integer ranges.
- The input array is guaranteed to already be sorted, which strongly hints that we should exploit this property instead of re-sorting from scratch.

Several edge cases are important:

- Arrays containing only negative numbers.
- Arrays containing only non-negative numbers.
- Arrays containing zeros.
- Arrays with duplicate values.
- Arrays with a single element.
- Large negative values whose squares dominate the result.

A naive implementation can fail to take advantage of the existing sorted order and unnecessarily spend extra time sorting.

## Approaches

### Brute Force Approach

The most direct solution is to square every element and then sort the resulting array.

The algorithm works as follows:

1. Iterate through the input array.
2. Square each number.
3. Store the squared values in a new array.
4. Sort the new array.
5. Return the sorted result.

This approach is correct because sorting guarantees the final array is in non-decreasing order regardless of how squaring changes the values.

However, sorting requires `O(n log n)` time. Since the original array is already sorted, this approach wastes useful structural information contained in the input.

### Optimal Two-Pointer Approach

The important observation is that the largest squared value must come from one of the two ends of the array.

Because the array is sorted:

- The most negative number appears on the left.
- The most positive number appears on the right.

When squared, both extremes can produce the largest value. For example:

```
[-9, -2, 0, 3]
```

becomes:

```
[81, 4, 0, 9]
```

The largest square comes from `-9`, which is on the left side.

This suggests a two-pointer strategy:

- One pointer starts at the beginning.
- One pointer starts at the end.
- Compare absolute values.
- Place the larger square into the result array from right to left.

Since we process each element exactly once, this achieves `O(n)` time complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(n) | Square all elements, then sort |
| Optimal | O(n) | O(n) | Use two pointers to fill result from largest to smallest |

## Algorithm Walkthrough

1. Create a result array of the same length as `nums`.

We need a separate array because the output must contain sorted squared values. Preallocating the array avoids repeated resizing operations.
2. Initialize two pointers.

Set `left = 0` and `right = len(nums) - 1`.

The left pointer represents the most negative remaining value, while the right pointer represents the largest positive remaining value.
3. Initialize a write position.

Set `position = len(nums) - 1`.

Since the largest square will always come from one of the ends, we place values into the result array from right to left.
4. Compare absolute values at both pointers.

Compute:

```
abs(nums[left])
abs(nums[right])
```

The larger absolute value produces the larger square.
5. Place the larger square into the result array.

If the left value has the larger absolute value:

- Square `nums[left]`
- Store it at `result[position]`
- Move `left` forward

Otherwise:

- Square `nums[right]`
- Store it at `result[position]`
- Move `right` backward
6. Move the write position leftward.

After placing one value, decrement `position`.
7. Continue until all elements are processed.

The loop stops when `left > right`.

### Why it works

The algorithm relies on the invariant that the largest remaining square always comes from either the leftmost or rightmost unprocessed element.

Because the array is sorted, all middle elements have absolute values less than or equal to at least one endpoint. By always selecting the larger absolute value and placing its square at the current largest available position, we guarantee the result array is built in sorted order from back to front.

## Python Solution

```
from typing import List

class Solution:
    def sortedSquares(self, nums: List[int]) -> List[int]:
        n = len(nums)
        result = [0] * n

        left = 0
        right = n - 1
        position = n - 1

        while left <= right:
            left_value = abs(nums[left])
            right_value = abs(nums[right])

            if left_value > right_value:
                result[position] = left_value * left_value
                left += 1
            else:
                result[position] = right_value * right_value
                right -= 1

            position -= 1

        return result
```

The implementation begins by allocating a result array with the same length as the input. This ensures we can directly place squared values into their final positions.

Two pointers are initialized at opposite ends of the array. The `left` pointer tracks the most negative remaining value, while the `right` pointer tracks the largest positive remaining value.

The algorithm repeatedly compares absolute values because the largest square always comes from the number with the greatest magnitude. The larger square is placed at the current `position` in the result array, which starts at the end and moves backward.

If the left side produces the larger square, the `left` pointer advances. Otherwise, the `right` pointer moves inward. This process continues until all elements are processed.

Because every element is visited exactly once, the algorithm runs in linear time.

## Go Solution

```
func sortedSquares(nums []int) []int {
	n := len(nums)
	result := make([]int, n)

	left := 0
	right := n - 1
	position := n - 1

	for left <= right {
		leftValue := nums[left]
		rightValue := nums[right]

		if leftValue < 0 {
			leftValue = -leftValue
		}

		if rightValue < 0 {
			rightValue = -rightValue
		}

		if leftValue > rightValue {
			result[position] = leftValue * leftValue
			left++
		} else {
			result[position] = rightValue * rightValue
			right--
		}

		position--
	}

	return result
}
```

The Go implementation follows the same logic as the Python version but uses Go slices and explicit integer handling.

Go does not provide a built-in `abs` function for integers in the standard library without type conversion, so the implementation manually converts negative values to positive using conditional checks.

The result slice is preallocated using `make([]int, n)`, which avoids dynamic resizing and improves efficiency.

Since Go slices are reference-backed structures, the function naturally returns the constructed slice without additional copying.

The algorithm performs a single pass through the array using two pointers. Each iteration places exactly one squared value into the result array, so the total number of operations grows linearly with the input size.

The additional space comes from the output array itself. Aside from that array, only a few integer variables are used.

## Edge Cases

### Arrays Containing Only Negative Numbers

An array such as:

```
[-5, -4, -2]
```

can be problematic because squaring reverses the relative ordering of the values. The smallest original value becomes the largest squared value.

A naive implementation that simply squares elements in place without sorting would produce:

```
[25, 16, 4]
```

which is not sorted.

The two-pointer approach handles this naturally because it always compares absolute values and places the largest square at the end of the result array first.

### Arrays Containing Only Non-Negative Numbers

For inputs like:

```
[1, 2, 3, 4]
```

the squared values remain in sorted order:

```
[1, 4, 9, 16]
```

Although this case is simpler, the algorithm still works correctly without requiring any special logic. The right pointer continually contributes the largest square, and the result array fills correctly from back to front.

### Equal Absolute Values

Inputs such as:

```
[-3, 3]
```

produce equal squares:

```
[9, 9]
```

This can introduce subtle pointer movement bugs if equality cases are not handled carefully.

The implementation resolves ties consistently by using the `else` branch when values are equal. One pointer moves inward each iteration, guaranteeing progress and preventing infinite loops.
