## Solution

## Problem Understanding

The problem asks us to build a spellchecker with very specific matching rules and precedence. We are given two arrays:

- `wordlist`, which contains the valid dictionary words
- `queries`, which contains words we need to correct

For every query, we must return the best matching word from the dictionary according to the following priority order:

1. Exact match, case-sensitive
2. Case-insensitive match
3. Vowel-error match, case-insensitive
4. No match, return an empty string

The important detail is that once a higher-priority match exists, lower-priority matches must be ignored.

A vowel-error match means that all vowels are considered interchangeable. The vowels are:

```
a, e, i, o, u
```

For example:

```
"KiTe" and "keti"
```

match because after lowercasing both words and replacing every vowel with a placeholder character, both become:

```
k*t*
```

The problem also requires that when multiple words could match, we always return the first matching word from `wordlist`.

The constraints are relatively small:

- Up to 5000 words in both `wordlist` and `queries`
- Maximum word length is 7

Even though the input size is not huge, a naive solution that compares every query against every word can still become inefficient because each comparison may involve multiple transformations and checks.

Several edge cases are important:

- Multiple dictionary words may normalize to the same lowercase form
- Multiple dictionary words may normalize to the same vowel-masked form
- Exact matches must override all other rules
- The first occurrence in `wordlist` must be preserved
- Words with different lengths can never vowel-match
- Queries may produce no valid match at all

The key challenge is implementing the precedence rules correctly while efficiently handling case-insensitive and vowel-insensitive matching.

## Approaches

### Brute Force Approach

The brute-force solution processes each query independently and compares it against every word in the dictionary.

For each query:

1. First scan the entire `wordlist` for an exact match
2. If none exists, scan again for a case-insensitive match
3. If still none exists, scan again for a vowel-error match

To check vowel-error matches, we lowercase both words and replace every vowel with a common placeholder such as `'*'`. If the transformed strings match, the words are considered equivalent under vowel substitution.

This approach is straightforward and directly follows the problem statement, so it is easy to verify for correctness. However, it performs repeated scans through the dictionary for every query.

If:

- `N = len(wordlist)`
- `Q = len(queries)`
- `L = maximum word length`

then the time complexity becomes:

```
O(Q * N * L)
```

Because every query may compare against every dictionary word.

While the constraints are moderate, this repeated work is unnecessary because many comparisons are duplicated across queries.

### Optimal Approach

The key insight is that the matching rules are based on normalized representations of words.

We can preprocess the dictionary into hash-based lookup structures:

- A set for exact matches
- A map from lowercase word to first matching dictionary word
- A map from vowel-normalized word to first matching dictionary word

This preprocessing allows every query to be answered in constant time lookups.

The vowel normalization works by converting all vowels to a single placeholder character. For example:

```
"KiTe" -> "k*t*"
"keti" -> "k*t*"
```

Both words normalize to the same representation.

Another important observation is that the problem requires returning the first matching word from the dictionary. Therefore, when building the maps, we only store the first occurrence for each normalized key.

This transforms the problem from repeated scanning into efficient hash table lookups.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(Q × N × L) | O(1) | Compare every query against every dictionary word |
| Optimal | O((N + Q) × L) | O(N × L) | Uses hash maps for constant-time lookups |

## Algorithm Walkthrough

1. Create a set containing all words from `wordlist`.

This allows constant-time exact match checks. Exact matches have the highest priority, so we always test this first.
2. Create a hash map for lowercase matches.

For every word in `wordlist`, compute:

```
lowercase_word = word.lower()
```

Store the first occurrence only:

```
lowercase_map[lowercase_word] = original_word
```

We use the first occurrence because the problem explicitly requires returning the earliest matching dictionary word.
3. Create a vowel-normalized hash map.

Define a helper function that:

- Lowercases the word
- Replaces every vowel with `'*'`

Example:

```
"YellOw" -> "y*ll*w"
```

Store the first occurrence for every normalized form.
4. Process each query independently.

For each query:

- First check whether it exists exactly in the original set
- If not, lowercase the query and check the lowercase map
- If still not found, normalize the query using vowel masking and check the vowel map
- If none match, return `""`
5. Append the selected result to the answer list.
6. Return the completed answer list after processing all queries.

### Why it works

The algorithm works because the lookup structures exactly mirror the precedence rules from the problem statement.

- Exact matches are checked first
- Case-insensitive matches are checked second
- Vowel-insensitive matches are checked third

Since we store only the first occurrence of each normalized representation, the algorithm also satisfies the requirement that the earliest matching dictionary word must be returned.

Hash maps guarantee efficient retrieval while preserving correctness.

## Python Solution

```
from typing import List

class Solution:
    def spellchecker(self, wordlist: List[str], queries: List[str]) -> List[str]:
        vowels = set("aeiou")

        def normalize(word: str) -> str:
            chars = []
            
            for ch in word.lower():
                if ch in vowels:
                    chars.append('*')
                else:
                    chars.append(ch)
            
            return ''.join(chars)

        exact_words = set(wordlist)

        lowercase_map = {}
        vowel_map = {}

        for word in wordlist:
            lower = word.lower()
            masked = normalize(word)

            if lower not in lowercase_map:
                lowercase_map[lower] = word

            if masked not in vowel_map:
                vowel_map[masked] = word

        answers = []

        for query in queries:
            if query in exact_words:
                answers.append(query)
                continue

            lower_query = query.lower()

            if lower_query in lowercase_map:
                answers.append(lowercase_map[lower_query])
                continue

            masked_query = normalize(query)

            if masked_query in vowel_map:
                answers.append(vowel_map[masked_query])
            else:
                answers.append("")

        return answers
```

The implementation begins by defining a normalization helper function. This function lowercases the word and replaces all vowels with `'*'`. Using a single placeholder ensures that all vowel substitutions collapse into the same canonical representation.

The `exact_words` set handles the highest-priority exact matches.

The `lowercase_map` stores the first dictionary word corresponding to every lowercase form. This handles capitalization errors.

The `vowel_map` stores the first dictionary word corresponding to every vowel-normalized representation. This handles vowel substitution errors.

When processing queries, the algorithm checks the structures in priority order:

1. Exact match
2. Lowercase match
3. Vowel-normalized match

The use of `continue` ensures that once a higher-priority match is found, lower-priority checks are skipped.

## Go Solution

```
package main

import "strings"

func spellchecker(wordlist []string, queries []string) []string {
	vowels := map[rune]bool{
		'a': true,
		'e': true,
		'i': true,
		'o': true,
		'u': true,
	}

	normalize := func(word string) string {
		word = strings.ToLower(word)

		chars := []rune(word)

		for i, ch := range chars {
			if vowels[ch] {
				chars[i] = '*'
			}
		}

		return string(chars)
	}

	exactWords := make(map[string]bool)

	lowercaseMap := make(map[string]string)
	vowelMap := make(map[string]string)

	for _, word := range wordlist {
		exactWords[word] = true

		lower := strings.ToLower(word)
		masked := normalize(word)

		if _, exists := lowercaseMap[lower]; !exists {
			lowercaseMap[lower] = word
		}

		if _, exists := vowelMap[masked]; !exists {
			vowelMap[masked] = word
		}
	}

	result := make([]string, 0, len(queries))

	for _, query := range queries {
		if exactWords[query] {
			result = append(result, query)
			continue
		}

		lowerQuery := strings.ToLower(query)

		if word, exists := lowercaseMap[lowerQuery]; exists {
			result = append(result, word)
			continue
		}

		maskedQuery := normalize(query)

		if word, exists := vowelMap[maskedQuery]; exists {
			result = append(result, word)
		} else {
			result = append(result, "")
		}
	}

	return result
}
```

The Go implementation follows the same algorithmic structure as the Python solution.

Instead of Python sets and dictionaries, Go uses maps:

- `map[string]bool` for exact matches
- `map[string]string` for normalized lookups

The normalization function operates on rune slices because strings in Go are immutable.

Go slices are used for building the result dynamically. Since the maximum result size is known in advance, the slice is initialized with sufficient capacity for efficiency.

There are no integer overflow concerns because the constraints are small and the algorithm only performs string operations and hash lookups.

The space usage comes from storing normalized forms in hash maps. Since each stored string can be length `L`, the total auxiliary storage is O(N × L).

## Edge Cases

One important edge case occurs when multiple dictionary words normalize to the same lowercase representation. For example:

```
["KiTe", "kite"]
```

Both lowercase to `"kite"`.

The problem requires returning the first matching word in the original dictionary order. A common mistake is overwriting entries in the hash map during preprocessing. This implementation avoids that by only inserting into the map if the key does not already exist.

Another tricky edge case involves vowel normalization collisions. Consider:

```
["YellOw", "yollow"]
```

Both normalize to:

```
y*ll*w
```

Again, only the first occurrence should be stored. If later words overwrite earlier ones, the output becomes incorrect.

A third edge case involves exact matches taking precedence over all other forms. Suppose:

```
wordlist = ["KiTe"]
query = "KiTe"
```

Even though lowercase and vowel-normalized matches also exist, the exact word itself must be returned immediately. The implementation handles this correctly by checking the exact-match set before any normalization logic.

Another subtle case is words with different lengths. For example:

```
"yellow"
"yllw"
```

A naive implementation that only compares vowel positions incorrectly might accidentally treat these as equivalent. Since normalization preserves word length, these strings normalize differently and therefore do not match.

Finally, single-character vowel words can behave unexpectedly. For example:

```
wordlist = ["A"]
query = "e"
```

Both normalize to:

```
*
```

So the query correctly resolves to `"A"`. The normalization function naturally handles this case without requiring special logic.
