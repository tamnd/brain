## Solution

## Problem Understanding

The problem asks us to find the largest possible island size in a binary `n x n` matrix after we are allowed to flip **at most one** `0` to `1`. In the matrix, `1` represents land and `0` represents water. An **island** is defined as a group of `1`s that are connected **4-directionally** (up, down, left, right). The goal is to determine the maximum number of connected `1`s after performing at most one flip.

The input is a square grid where `1 <= n <= 500`, so the matrix can have up to 250,000 cells. Each cell contains either `0` or `1`. The output is a single integer, the size of the largest island that can be formed after changing at most one `0` to `1`.

Important edge cases include:

- A grid entirely filled with `1`s, where flipping is unnecessary, and the result is `n*n`.
- A grid entirely filled with `0`s, where flipping one cell creates an island of size `1`.
- Multiple disconnected islands where flipping a single `0` connects multiple islands.

Constraints indicate we need an efficient algorithm; a naive O(n^4) solution will not scale to the upper bound of `n=500`.

## Approaches

The brute-force approach would be to try flipping each `0` to `1`, then compute the size of the largest island using BFS or DFS for each modified grid. While this guarantees correctness, it is inefficient: for each `0` (potentially O(n^2) zeros), we traverse the grid to compute island sizes (another O(n^2)), giving a total of O(n^4) complexity. This is too slow for n up to 500.

The key observation for an optimal solution is that instead of recomputing island sizes repeatedly, we can first label all existing islands with unique IDs and store their sizes in a hash map. Then, for each `0`, we examine its neighboring islands using their IDs, sum their sizes (making sure to avoid double-counting the same island), and add `1` for the flipped cell. This reduces repeated computation and allows us to solve the problem in O(n^2) time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^4) | O(n^2) | Flip each zero and compute island size repeatedly |
| Optimal | O(n^2) | O(n^2) | Label islands once, then for each zero check neighboring island sizes |

## Algorithm Walkthrough

1. Initialize a hash map `island_sizes` to store island IDs and their corresponding areas. Start island IDs from `2` to avoid confusion with `0` and `1`.
2. Traverse the grid using DFS or BFS. For each unvisited `1`, assign it a unique island ID and calculate the size of the island, storing it in `island_sizes`.
3. Keep track of the maximum island size found in the initial grid in case there are no `0`s to flip.
4. Iterate over all `0`s in the grid. For each `0`, examine its 4-directionally adjacent cells. Collect the unique island IDs among neighbors, sum their sizes from `island_sizes`, and add `1` for the flipped cell.
5. Update the global maximum with this potential new island size.
6. Return the maximum island size found.

**Why it works**: By labeling each island uniquely, we can efficiently calculate the effect of flipping any `0` without recomputing entire island sizes. The sum of adjacent unique islands plus the flipped cell gives the maximum possible island size for that flip.
