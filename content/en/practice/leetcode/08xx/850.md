## Solution

## Problem Understanding

The problem asks us to compute the **total area covered by a set of axis-aligned rectangles** on a 2D plane, counting overlapping areas only once. Each rectangle is defined by its bottom-left corner `(xi1, yi1)` and top-right corner `(xi2, yi2)`. The input is a list of rectangles, and the output should be a single integer representing the total area modulo `10^9 + 7`.

In other words, if two or more rectangles overlap, the overlapping portion should not be double-counted. This makes the problem non-trivial because naive summation of rectangle areas will overcount the overlapped regions.

The constraints tell us that there are **at most 200 rectangles**, but each coordinate can be as large as **10^9**, which rules out a naive grid-based solution. Rectangles are guaranteed to have **non-zero area**, so we do not need to handle degenerate rectangles.

Important edge cases include rectangles that **completely overlap**, **partially overlap**, **touch at edges or corners**, or a rectangle that covers the entire coordinate range. The modulo operation also suggests that the resulting area could exceed standard integer limits.

## Approaches

### Brute Force

The naive approach is to use a **2D grid** to mark all points covered by rectangles. We could iterate over each rectangle and mark all cells in a boolean matrix as covered. After processing all rectangles, summing the number of `True` cells gives the total area. While this approach works in theory, it is **impractical** for this problem because coordinates can go up to `10^9`, making a grid impossible to store in memory. Time complexity is also prohibitive, as we would need to iterate over potentially `10^18` points.

### Optimal Approach

A better approach leverages a **sweep line algorithm** with **coordinate compression** or a **segment tree**. The idea is to sweep along one axis (e.g., x-axis) and compute the area covered at each vertical slice. For each sweep line:

1. Sort all rectangle edges along the x-axis.
2. Maintain a structure to track active y-intervals (e.g., sorted list or segment tree).
3. Compute the total height covered by active y-intervals at the current x-position.
4. Multiply the covered height by the width of the current x-slice to get the area contribution.

This approach efficiently counts overlaps because the segment tree or interval merge ensures no double counting. With **at most 400 endpoints**, the operations remain efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(10^18) | O(10^18) | Not feasible for large coordinates, uses 2D grid |
| Sweep Line + Segment Tree | O(N log N) | O(N) | Efficiently counts overlapping rectangles without overcounting |

## Algorithm Walkthrough

1. **Extract edges and sort events**: For each rectangle `[x1, y1, x2, y2]`, generate two events: `x1` (rectangle starts) and `x2` (rectangle ends). Store y-interval `[y1, y2]` with each event. Sort events by x-coordinate.
2. **Coordinate compression**: Gather all unique y-coordinates and sort them. This allows us to use a **segment tree or list** with size at most `2 * N` rather than `10^9`.
3. **Initialize active intervals**: Use a data structure to maintain which y-intervals are active. As we sweep the x-axis from left to right, update active intervals whenever a rectangle starts or ends.
4. **Calculate covered height**: At each x-position, compute the total vertical length covered by active intervals. This can be done by merging intervals or querying a segment tree.
5. **Compute area incrementally**: For each slice between `x[i]` and `x[i+1]`, multiply the covered height by the width `(x[i+1] - x[i])` to get the area for this slice. Accumulate this into the total area modulo `10^9 + 7`.
6. **Update active intervals**: Add y-intervals for rectangle starts, remove y-intervals for rectangle ends.

**Why it works**: The algorithm ensures that at every x-position, we correctly compute the total height covered by rectangles. Multiplying by the horizontal distance ensures no overlap is counted twice. Coordinate compression guarantees that we handle large coordinates efficiently, and the sweep line ensures all overlaps are accounted for in order.

## Python Solution

```
from typing import List

class Solution:
    def rectangleArea(self, rectangles: List[List[int]]) -> int:
        MOD = 10**9 + 7

        # Collect all unique y-coordinates
        ys = sorted({y for x1, y1, x2, y2 in rectangles for y in (y1, y2)})
        y_i = {v: i for i, v in enumerate(ys)}

        # Prepare events: (x, type, y1, y2)
        events = []
        for x1, y1, x2, y2 in rectangles:
            events.append((x1, 1, y1, y2))  # rectangle starts
            events.append((x2, -1, y1, y2))  # rectangle ends
        events.sort()

        # Segment tree to store counts and total coverage
        N = len(ys)
        count = [0] * (4 * N)
        total = [0] * (4 * N)

        def update(node, l, r, ul, ur, val):
            if ur <= l or r <= ul:
                return
            if ul <= l and r <= ur:
                count[node] += val
            else:
                mid = (l + r) // 2
                update(node*2, l, mid, ul, ur, val)
                update(node*2+1, mid, r, ul, ur, val)
            if count[node]:
                total[node] = ys[r] - ys[l]
            else:
                total[node] = total[node*2] + total[node*2+1] if r - l > 1 else 0

        cur_x = events[0][0]
        res = 0
        for x, typ, y1, y2 in events:
            res += (x - cur_x) * total[1]
            res %= MOD
            update(1, 0, N-1, y_i[y1], y_i[y2], typ)
            cur_x = x

        return res
```

**Explanation**:

The Python solution implements a sweep line with a segment tree. We first compress y-coordinates to reduce memory usage. Events are sorted by x-coordinates, with type 1 for start and -1 for end. The segment tree tracks how many intervals cover each segment and computes the total covered length efficiently. The area is accumulated slice by slice along the x-axis.

## Go Solution

```
package main

import "sort"

func rectangleArea(rectangles [][]int) int {
    const MOD = 1e9 + 7

    ysMap := map[int]struct{}{}
    for _, r := range rectangles {
        ysMap[r[1]] = struct{}{}
        ysMap[r[3]] = struct{}{}
    }

    ys := make([]int, 0, len(ysMap))
    for y := range ysMap {
        ys = append(ys, y)
    }
    sort.Ints(ys)

    yIndex := map[int]int{}
    for i, y := range ys {
        yIndex[y] = i
    }

    type event struct{ x, typ, y1, y2 int }
    events := make([]event, 0, len(rectangles)*2)
    for _, r := range rectangles {
        events = append(events, event{r[0], 1, r[1], r[3]})
        events = append(events, event{r[2], -1, r[1], r[3]})
    }
    sort.Slice(events, func(i, j int) bool { return events[i].x < events[j].x })

    N := len(ys)
    count := make([]int, 4*N)
    total := make([]int, 4*N)

    var update func(node, l, r, ul, ur, val int)
    update = func(node, l, r, ul, ur, val int) {
        if ur <= l || r <= ul {
            return
        }
        if ul <= l && r <= ur {
            count[node] += val
        } else {
            mid := (l + r) / 2
            update(node*2, l, mid, ul, ur, val)
            update(node*2+1, mid, r, ul, ur, val)
        }
        if count[node] > 0 {
            total[node] = ys[r] - ys[l]
        } else if r-l > 1 {
            total[node] = total[node*2] + total[node*2+1]
        } else {
            total[node] = 0
        }
    }

    curX := events[0].x
    res := 0
    for _, e := range events {
        res += (e.x - curX) * total[1] % MOD
        res %= MOD
        update(1, 0, N-1, yIndex[e.y1], yIndex[e.y2], e.typ)
        curX = e.x
```
