## Solution

## Problem Understanding

The problem gives us an `n x n` binary matrix called `image`. Every element in the matrix is either `0` or `1`. We must perform two operations on every row of the matrix:

1. Flip the row horizontally
2. Invert every bit in the row

Flipping horizontally means reversing the order of elements in each row. For example:

```
[1, 0, 1, 0] -> [0, 1, 0, 1]
```

After reversing the row, we invert every value:

- `0` becomes `1`
- `1` becomes `0`

For example:

```
[0, 1, 0, 1] -> [1, 0, 1, 0]
```

The final transformed matrix must be returned.

The input size is relatively small because `n <= 20`. Even though the constraints are small enough that almost any reasonable solution will pass, the problem is designed to test understanding of array manipulation, two pointers, and bit operations.

An important observation is that both operations can be combined into a single pass. Instead of first reversing the row and then traversing again to invert values, we can swap mirrored elements and invert them at the same time.

There are several edge cases worth thinking about before implementation.

If a row has an odd number of elements, the middle element maps to itself during reversal. That value still needs to be inverted exactly once.

If two mirrored elements are different, such as `0` and `1`, reversing them and then inverting them produces the exact same pair again. This means no change is necessary for such pairs.

If two mirrored elements are equal, both must be flipped after the swap and inversion process.

The problem guarantees:

- The matrix is square
- Every element is binary
- The matrix has at least one row and one column

Because of these guarantees, we do not need to handle malformed input.

## Approaches

### Brute Force Approach

The straightforward approach is to perform the two required operations separately.

For each row:

1. Reverse the row
2. Traverse the reversed row again and invert every value

This solution is simple and directly follows the problem statement. Reversing a row takes linear time, and inverting every element also takes linear time. Since there are `n` rows and each row has `n` elements, the total work is proportional to `n²`.

This approach is completely acceptable for the given constraints, but it performs unnecessary work because every row is traversed twice.

### Optimal Approach

The key insight is that flipping and inversion can happen simultaneously.

Suppose we look at two mirrored positions in a row:

```
row[left], row[right]
```

After reversing:

- `row[left]` moves to `right`
- `row[right]` moves to `left`

After inversion:

- both values are toggled

Instead of doing these operations separately, we can process mirrored pairs directly.

There are two important cases:

- If the values are different, such as `(0,1)` or `(1,0)`, reversing and inverting leaves them unchanged.
- If the values are the same, such as `(0,0)` or `(1,1)`, both values must be toggled.

This allows us to process each row using two pointers in a single traversal.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) or O(n²) | Reverse each row, then invert each element |
| Optimal | O(n²) | O(1) | Combine reversal and inversion in one pass using two pointers |

## Algorithm Walkthrough

### Optimal Algorithm

1. Iterate through every row in the matrix.
2. For each row, initialize two pointers:

- `left = 0`
- `right = len(row) - 1`

These pointers represent mirrored positions in the row.
3. Continue processing while `left <= right`.

We use `<=` instead of `<` so that the middle element in odd-length rows is also processed.
4. Compare the values at `row[left]` and `row[right]`.

If the values are different:

- Reversing swaps them
- Inverting flips both
- The final result becomes identical to the original pair

Therefore, no work is needed.
5. If the values are equal:

- Both positions must be inverted
- We can toggle both values using XOR with `1`

For example:

```
0 ^ 1 = 1
1 ^ 1 = 0
```
6. Move the pointers inward:

- `left += 1`
- `right -= 1`
7. After processing all rows, return the modified matrix.

### Why it works

The algorithm works because horizontal flipping only swaps mirrored positions. Inversion then toggles every bit. For mirrored pairs that differ, the combined effect produces the original values again, so no modification is required. For mirrored pairs that are equal, both values must change. By processing pairs directly, we simulate both operations simultaneously while maintaining correctness for every position in the row.

## Python Solution

```
from typing import List

class Solution:
    def flipAndInvertImage(self, image: List[List[int]]) -> List[List[int]]:
        for row in image:
            left = 0
            right = len(row) - 1

            while left <= right:
                if row[left] == row[right]:
                    row[left] ^= 1
                    row[right] ^= 1

                left += 1
                right -= 1

        return image
```

The implementation iterates through every row of the matrix and uses two pointers to process mirrored positions.

The condition `row[left] == row[right]` is the key optimization. When both values are identical, they must be toggled after the combined flip-and-invert operation. XOR with `1` efficiently flips the binary value.

Using `while left <= right` ensures that odd-length rows correctly process the center element. When `left == right`, the same element is toggled exactly once.

The solution modifies the matrix in place, which keeps the space complexity constant.

## Go Solution

```
func flipAndInvertImage(image [][]int) [][]int {
    for _, row := range image {
        left := 0
        right := len(row) - 1

        for left <= right {
            if row[left] == row[right] {
                row[left] ^= 1
                row[right] ^= 1
            }

            left++
            right--
        }
    }

    return image
}
```

The Go implementation follows the same logic as the Python version.

Slices in Go are reference-like structures, so modifying `row` directly updates the original matrix. No additional copying is necessary.

There are no integer overflow concerns because all values are binary. XOR with `1` safely toggles between `0` and `1`.

The implementation also correctly handles odd-length rows because the loop condition uses `left <= right`.

## Edge Cases

One important edge case is a matrix containing only a single element, such as `[[0]]` or `[[1]]`. In this scenario, the element maps to itself during reversal, so the implementation must invert it exactly once. Using `left <= right` ensures the middle position is processed correctly.

Another important case is rows with odd lengths. The center element does not move during reversal, but it still must be inverted. A common bug is using `left < right`, which skips the center element entirely. The implementation avoids this issue by allowing the pointers to meet.

Rows where mirrored values differ are also subtle. For example, consider `[1,0]`. Reversing produces `[0,1]`, and inverting produces `[1,0]`, which is identical to the original row. A naive implementation might unnecessarily swap and toggle values. The optimized solution recognizes that no changes are needed when mirrored values differ.

A final edge case involves rows where all elements are identical, such as `[0,0,0,0]` or `[1,1,1,1]`. In these cases, every position must change after processing. The implementation correctly toggles both mirrored positions whenever the values match.
