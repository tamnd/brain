## Problem Understanding

This problem asks us to find the seat in a row of seats that maximizes the distance to the closest person already sitting. The input is an array `seats` where each element is either `0` (empty) or `1` (occupied). The goal is to determine the maximum possible distance between a new person sitting down and any other person.

The output is a single integer representing that maximum distance.

Key constraints include a minimum array length of 2 and a maximum of 20,000, meaning any solution must handle relatively large input efficiently. The problem guarantees there is at least one empty seat and at least one occupied seat, so we do not need to handle all-zero or all-one arrays.

Important edge cases involve empty seats at the start or end of the array, as these create scenarios where the distance is counted only in one direction. Another edge case is a row with a single empty seat surrounded by occupied seats, where the distance is limited to 1.

## Approaches

The **brute-force approach** involves checking every empty seat and computing the distance to the nearest occupied seat by scanning left and right from that position. This guarantees correctness because it explicitly calculates the distance for every candidate seat. However, this is too slow for the maximum input size, as scanning left and right for every empty seat in a 20,000-length array could require millions of operations in the worst case.

The **optimal approach** leverages the observation that the maximum distance always occurs either at the beginning of a row of empty seats, the end of a row of empty seats, or the midpoint of consecutive empty seats between two occupied seats. By scanning the array once and keeping track of stretches of empty seats, we can calculate the maximum distance in linear time without checking each seat individually.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Scan left and right for each empty seat |
| Optimal | O(n) | O(1) | Track consecutive zeros and handle edges efficiently |

## Algorithm Walkthrough

1. Initialize variables to track the maximum distance (`max_dist`), the index of the last occupied seat (`last_occupied`), and the length of the array (`n`).
2. Iterate through the `seats` array from left to right.
3. If a seat is occupied (`1`), check the number of consecutive zeros since the last occupied seat.
4. If this is the first occupied seat encountered, the stretch of empty seats at the beginning is only bounded on the right, so the distance is the length of the stretch itself.
5. If this is not the first occupied seat, the stretch of empty seats is between two occupied seats, so the maximum distance is half the length of the stretch, rounded down.
6. Update `last_occupied` to the current index.
7. After the iteration, check if there is a stretch of empty seats at the end of the row and update `max_dist` accordingly.
8. Return `max_dist`.

**Why it works**: This algorithm works because it systematically evaluates every potential seat where the maximum distance can occur: at the start, in the middle between two people, or at the end. By calculating distances using only stretches of empty seats, we guarantee that the maximum distance is found efficiently.

## Python Solution

```
from typing import List

class Solution:
    def maxDistToClosest(self, seats: List[int]) -> int:
        n = len(seats)
        max_dist = 0
        last_occupied = -1
        
        for i in range(n):
            if seats[i] == 1:
                if last_occupied == -1:
                    # Stretch at the start
                    max_dist = i
                else:
                    # Stretch between two occupied seats
                    max_dist = max(max_dist, (i - last_occupied) // 2)
                last_occupied = i
        
        # Stretch at the end
        if seats[-1] == 0:
            max_dist = max(max_dist, n - 1 - last_occupied)
        
        return max_dist
```

This code scans the array once. The first occupied seat determines if there is an initial stretch. For every occupied seat thereafter, it calculates the half-length of empty stretches between seats. Finally, it handles trailing zeros to ensure distances at the end are considered.

## Go Solution

```
func maxDistToClosest(seats []int) int {
    n := len(seats)
    maxDist := 0
    lastOccupied := -1
    
    for i := 0; i < n; i++ {
        if seats[i] == 1 {
            if lastOccupied == -1 {
                maxDist = i
            } else {
                maxDist = max(maxDist, (i - lastOccupied)/2)
            }
            lastOccupied = i
        }
    }
    
    // Stretch at the end
    if seats[n-1] == 0 {
        maxDist = max(maxDist, n-1-lastOccupied)
    }
    
    return maxDist
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

The Go implementation mirrors the Python version closely. It uses an explicit `max` helper function and handles slices instead of Python lists. Otherwise, the logic is identical, ensuring linear runtime and constant space.

## Edge Cases

One important edge case is an empty stretch at the beginning of the row. If the first seat(s) are empty, the distance to the closest person is the number of empty seats before the first occupied seat. The algorithm correctly handles this by initializing `last_occupied` to -1 and checking if this is the first occupied seat.

Another edge case is an empty stretch at the end. If the last seat(s) are empty, the distance is the number of empty seats after the last occupied seat. The final check `n - 1 - last_occupied` ensures this is correctly considered.

A third edge case is consecutive empty seats between two occupied seats. The maximum distance occurs at the midpoint, so integer division is used to handle both even and odd lengths correctly. This guarantees that Alex sits as far as possible from both neighbors.
