## Solution

## Problem Understanding

The problem gives us two binary square matrices, `img1` and `img2`, each of size `n x n`. Every cell contains either `0` or `1`. A value of `1` represents an active pixel, while `0` represents an empty pixel.

We are allowed to translate one image relative to the other. Translation means shifting the entire image left, right, up, or down by any number of cells. Rotation is not allowed, and diagonal movement is simply achieved through combinations of horizontal and vertical shifts.

After shifting one image, we place it on top of the other image and count how many positions contain `1` in both matrices simultaneously. That count is called the overlap. Our goal is to find the maximum overlap possible across all translations.

An important detail is that pixels shifted outside the matrix boundaries disappear. They are not wrapped around.

For example, if we shift an image one row downward, the bottom row falls off the matrix and is lost, while a new empty row effectively appears at the top.

The constraints are relatively small:

- `1 <= n <= 30`
- Each value is either `0` or `1`

Since `n` is at most `30`, the total number of cells is at most `900`. This is small enough that several brute-force style solutions are feasible, but we still want an efficient and clean implementation.

There are several important edge cases to keep in mind:

- Both matrices may contain only zeros, in which case the answer is `0`.
- A matrix may contain only a single `1`.
- The maximum overlap may occur after large shifts near the matrix boundary.
- Some translations may completely move all `1`s outside the matrix.
- Multiple different translations may produce the same overlap count.

The problem guarantees that both matrices are square and have identical dimensions, so we never need to handle malformed input.

## Approaches

### Brute Force Approach

The most direct solution is to try every possible translation.

If the matrix size is `n`, then the row shift can range from `-(n - 1)` to `+(n - 1)`, and the column shift can also range across the same interval. For every possible `(rowShift, colShift)` pair, we compare every cell in `img1` against the corresponding shifted position in `img2`.

For each translation:

1. Iterate through every cell `(r, c)` in `img1`
2. Compute the shifted position in `img2`
3. Check whether both positions contain `1`
4. Count overlaps
5. Track the maximum overlap found

This works because it explicitly evaluates every valid translation, guaranteeing that the best overlap is discovered.

However, the time complexity becomes expensive:

- There are approximately `(2n - 1)^2` translations
- For each translation, we scan all `n^2` cells

That leads to a total complexity of `O(n^4)`.

With `n <= 30`, this is technically acceptable, but we can do better conceptually.

### Optimal Approach, Translation Vector Counting

The key observation is that overlap only matters for cells containing `1`.

Suppose:

- `img1` has a `1` at `(r1, c1)`
- `img2` has a `1` at `(r2, c2)`

If we shift `img1` by:

```
(r2 - r1, c2 - c1)
```

then those two `1`s will align.

This means every pair of `1`s defines a translation vector that would cause them to overlap.

If many pairs of `1`s produce the same translation vector, then that translation aligns many pixels simultaneously.

So instead of checking every matrix position repeatedly, we:

1. Collect coordinates of all `1`s in both matrices
2. Compute translation vectors between every pair
3. Count how many times each vector appears
4. The most frequent vector gives the maximum overlap

This transforms the problem into a frequency counting problem using a hash map.

Since each matrix contains at most `900` cells, the number of `1`s is also bounded by `900`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^4) | O(1) | Try every translation and scan entire matrix |
| Optimal | O(k1 × k2) | O(k1 × k2) | Count translation vectors between active pixels |

Here:

- `k1` = number of `1`s in `img1`
- `k2` = number of `1`s in `img2`

In the worst case, both equal `n^2`, giving worst-case complexity `O(n^4)`, but in practice this approach is much faster because it only processes active pixels.

## Algorithm Walkthrough

1. Traverse `img1` and record coordinates of every cell containing `1`.

We store these coordinates in a list because only active pixels matter for overlap calculations.
2. Traverse `img2` and record coordinates of every cell containing `1`.

Again, we ignore zeros because they never contribute to overlap.
3. Create a hash map called `translation_count`.

The key is a translation vector `(dr, dc)`, representing how far `img1` must shift for one pair of pixels to align.

The value stores how many pixel pairs produce that translation.
4. For every active pixel `(r1, c1)` in `img1`:

- For every active pixel `(r2, c2)` in `img2`:

- Compute:

```
dr = r2 - r1
dc = c2 - c1
```
- Increment the count for `(dr, dc)` in the hash map
5. While updating the hash map, track the largest frequency seen.

That frequency represents the maximum number of overlapping `1`s achievable with a single translation.
6. Return the maximum frequency.

### Why it works

Every overlap between two active pixels corresponds to exactly one translation vector. If several pairs of active pixels share the same vector, then shifting the image by that vector aligns all of them simultaneously.

Therefore, the translation vector with the highest frequency produces the largest overlap.

## Python Solution

```
from typing import List
from collections import defaultdict

class Solution:
    def largestOverlap(self, img1: List[List[int]], img2: List[List[int]]) -> int:
        n = len(img1)

        ones_img1 = []
        ones_img2 = []

        # Collect coordinates of all 1s in img1
        for row in range(n):
            for col in range(n):
                if img1[row][col] == 1:
                    ones_img1.append((row, col))

        # Collect coordinates of all 1s in img2
        for row in range(n):
            for col in range(n):
                if img2[row][col] == 1:
                    ones_img2.append((row, col))

        translation_count = defaultdict(int)
        max_overlap = 0

        # Count translation vectors
        for r1, c1 in ones_img1:
            for r2, c2 in ones_img2:
                shift = (r2 - r1, c2 - c1)

                translation_count[shift] += 1

                max_overlap = max(
                    max_overlap,
                    translation_count[shift]
                )

        return max_overlap
```

The implementation begins by scanning both matrices and storing the coordinates of all active pixels. This preprocessing step reduces unnecessary work because zeros never contribute to overlap.

The `translation_count` hash map stores frequencies of translation vectors. Every pair of active pixels generates one vector.

For example:

```
img1 pixel: (1, 2)
img2 pixel: (3, 4)
```

produces translation:

```
(2, 2)
```

meaning `img1` must move down `2` rows and right `2` columns for those pixels to align.

As we process all pairs, repeated vectors accumulate counts. The vector with the largest count corresponds to the best alignment.

The solution continuously updates `max_overlap` so that no second traversal of the hash map is needed.

## Go Solution

```
package main

func largestOverlap(img1 [][]int, img2 [][]int) int {
	n := len(img1)

	type Point struct {
		r int
		c int
	}

	ones1 := []Point{}
	ones2 := []Point{}

	// Collect all 1s from img1
	for r := 0; r < n; r++ {
		for c := 0; c < n; c++ {
			if img1[r][c] == 1 {
				ones1 = append(ones1, Point{r, c})
			}
		}
	}

	// Collect all 1s from img2
	for r := 0; r < n; r++ {
		for c := 0; c < n; c++ {
			if img2[r][c] == 1 {
				ones2 = append(ones2, Point{r, c})
			}
		}
	}

	translationCount := make(map[[2]int]int)
	maxOverlap := 0

	// Count translation vectors
	for _, p1 := range ones1 {
		for _, p2 := range ones2 {
			shift := [2]int{
				p2.r - p1.r,
				p2.c - p1.c,
			}

			translationCount[shift]++

			if translationCount[shift] > maxOverlap {
				maxOverlap = translationCount[shift]
			}
		}
	}

	return maxOverlap
}
```

The Go implementation follows the same algorithmic structure as the Python version.

One notable difference is that Go does not have tuple keys like Python. Instead, we use a fixed-size array `[2]int` as the hash map key to represent translation vectors.

Slices are used to store coordinates of active pixels, and the map stores vector frequencies.

Integer overflow is not a concern because all values remain very small under the problem constraints.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(k1 × k2) | Compare every active pixel in `img1` with every active pixel in `img2` |
| Space | O(k1 × k2) | Hash map may store many unique translation vectors |

In the worst case, every cell contains `1`, so:

```
k1 = k2 = n²
```

This gives worst-case time complexity:

```
O(n^4)
```

However, the algorithm is usually much faster than brute force because it ignores all zero cells and avoids repeatedly scanning the entire matrix for every translation.

## Edge Cases

One important edge case occurs when both matrices contain only zeros. In this scenario, there are no active pixels to compare, so no translation vectors are generated. A buggy implementation might assume at least one vector exists and fail when accessing map values. This implementation handles the case naturally because the loops never execute and `max_overlap` remains `0`.

Another important case is when the optimal translation moves many pixels outside the matrix. Since translated pixels beyond the boundary are discarded, some implementations incorrectly attempt wraparound behavior. This solution avoids that problem entirely because it only reasons about relative coordinates of active pixels. Translation vectors are purely geometric and do not require explicitly simulating matrix movement.

A third edge case involves matrices with very sparse active pixels. For example, each matrix may contain only one `1`. In that case, exactly one translation vector exists, and the answer must be `1`. The hash map counting strategy handles sparse matrices efficiently because it only processes existing active pixels rather than scanning every matrix cell repeatedly.

Another subtle case is when multiple translation vectors produce the same overlap count. The problem only asks for the maximum overlap value, not the translation itself. This implementation correctly tracks only the largest frequency and does not depend on uniqueness of the optimal shift.
