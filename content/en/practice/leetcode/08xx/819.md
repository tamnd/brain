## Solution

## Problem Understanding

The problem asks us to find the most frequent word in a paragraph that is not part of a given list of banned words. The paragraph is a string consisting of words separated by spaces and may contain punctuation symbols such as `!?',;.`. Words are **case-insensitive**, meaning "Ball" and "ball" are treated as the same word. The returned word should be **lowercase**, regardless of how it appears in the paragraph.

The input consists of two elements: a string `paragraph` and a list of strings `banned`. The paragraph length is between 1 and 1000 characters, which is small enough for standard string processing. The `banned` list can contain up to 100 words, each with 1-10 lowercase letters. The problem guarantees that there is at least one word not in the banned list and that the solution is unique.

Important edge cases include paragraphs with only one word, paragraphs that are entirely punctuation, paragraphs with mixed casing, and empty banned lists. The solution must handle punctuation removal correctly and normalize casing to avoid incorrect counting.

## Approaches

A **brute-force approach** would first remove all punctuation from the paragraph, then split it into words, normalize each word to lowercase, and count the frequency of every word in a dictionary. After counting, it would iterate through the dictionary to find the most frequent word that is not in the banned list. While this approach is correct, it is slightly inefficient due to multiple passes over the data and repeated membership checks in the banned list, especially if implemented with a list lookup instead of a set.

The **optimal approach** builds on the brute-force idea but optimizes the banned word lookup using a set for O(1) access. It uses a single pass to normalize words, count frequencies, and skip banned words. This ensures linear time complexity relative to the length of the paragraph while keeping memory usage manageable. The key insight is that using a set for banned words and a dictionary for word counts avoids repeated linear scans for membership and counting.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m) | O(n) | Count all words and check banned words using list membership, where n = number of words, m = banned list length |
| Optimal | O(n) | O(n + k) | Count words using a dictionary and check banned words using a set, k = number of banned words |

## Algorithm Walkthrough

1. Convert all punctuation characters in the paragraph to spaces. This simplifies splitting words.
2. Convert the entire paragraph to lowercase to ensure case-insensitive counting.
3. Split the paragraph by spaces to get a list of words.
4. Create a set from the `banned` list for O(1) membership checking.
5. Initialize an empty dictionary to count the frequency of each non-banned word.
6. Iterate over each word in the list. If the word is non-empty and not in the banned set, increment its count in the dictionary.
7. Track the word with the maximum count while updating the dictionary to avoid a second pass.
8. Return the word with the highest count.

**Why it works:** By normalizing the text, removing punctuation, and using a hash map for counting, the algorithm guarantees that every word is counted exactly once, excluding banned words. Using a set ensures that checking if a word is banned is efficient, and the word with the highest count is guaranteed to be unique.

## Python Solution

```
from typing import List
import re
from collections import defaultdict

class Solution:
    def mostCommonWord(self, paragraph: str, banned: List[str]) -> str:
        # Convert banned list to a set for O(1) lookups
        banned_set = set(banned)
        
        # Replace punctuation with spaces and convert to lowercase
        normalized_paragraph = re.sub(r"[!?',;.\s]+", " ", paragraph).lower()
        
        # Split into words
        words = normalized_paragraph.split()
        
        # Count frequency of non-banned words
        word_count = defaultdict(int)
        max_count = 0
        most_common = ""
        for word in words:
            if word and word not in banned_set:
                word_count[word] += 1
                if word_count[word] > max_count:
                    max_count = word_count[word]
                    most_common = word
        
        return most_common
```

This implementation first converts the banned words to a set for efficient lookup. It uses a regular expression to replace punctuation with spaces and lowercases the paragraph. Words are then split, counted using a dictionary, and the most frequent non-banned word is returned in lowercase. Tracking the maximum while counting avoids a second pass over the dictionary.

## Go Solution

```
import (
    "strings"
    "unicode"
)

func mostCommonWord(paragraph string, banned []string) string {
    bannedSet := make(map[string]bool)
    for _, word := range banned {
        bannedSet[word] = true
    }

    // Normalize paragraph: convert to lowercase and replace punctuation with spaces
    var sb strings.Builder
    for _, ch := range paragraph {
        if unicode.IsLetter(ch) {
            sb.WriteRune(unicode.ToLower(ch))
        } else {
            sb.WriteRune(' ')
        }
    }

    words := strings.Fields(sb.String())
    wordCount := make(map[string]int)
    maxCount := 0
    mostCommon := ""
    for _, word := range words {
        if word != "" && !bannedSet[word] {
            wordCount[word]++
            if wordCount[word] > maxCount {
                maxCount = wordCount[word]
                mostCommon = word
            }
        }
    }

    return mostCommon
}
```

The Go version uses a `strings.Builder` to efficiently normalize the paragraph, replacing punctuation with spaces and converting letters to lowercase. A map tracks banned words and another map counts occurrences. `strings.Fields` splits words by whitespace. The tracking of the most common word occurs in the same loop to avoid a second pass.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | n = length of paragraph; each character is processed once and each word is counted once |
| Space | O(n + k) | n = number of unique words in paragraph, k = size of banned set; dictionaries and sets are used |

The complexity is linear because each step, from normalization to splitting and counting, scales with the paragraph length. Using a set for banned words ensures constant-time membership checks.
