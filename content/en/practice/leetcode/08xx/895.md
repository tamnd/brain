## Solution

## Problem Understanding

The problem asks us to design a special stack-like data structure called `FreqStack`. Unlike a normal stack, where `pop()` always removes the most recently inserted element, this structure removes the element with the highest frequency first.

If multiple elements share the same highest frequency, then we fall back to normal stack behavior among those tied elements. In other words, the most recently pushed element among the tied maximum-frequency elements should be removed.

This creates a hybrid behavior:

- Primary priority: highest frequency
- Secondary priority: most recent insertion

The structure supports two operations:

- `push(val)` inserts a value into the stack
- `pop()` removes and returns the most frequent value

The constraints are important:

- Up to `2 * 10^4` operations
- Values can be as large as `10^9`
- `pop()` is guaranteed to be called only when the structure is non-empty

These constraints tell us that we need near constant-time operations. A naive implementation that scans the entire stack on every `pop()` would become too slow.

The main challenge is maintaining both:

1. Frequency information
2. Recency order among equally frequent elements

A naive implementation can easily track one of these properties, but efficiently tracking both simultaneously requires a more careful design.

Some important edge cases include:

- Multiple values having identical frequencies
- One value becoming more frequent again after previous pops
- Repeated pushes of the same number
- Alternating push and pop operations
- Large numbers as values, which means array indexing is not appropriate and hash maps are necessary

The problem guarantees that `pop()` is always valid, so we never need to handle popping from an empty structure.

## Approaches

### Brute Force Approach

A straightforward solution would use a normal stack to store all inserted values. Every time `pop()` is called, we would:

1. Count the frequency of every element currently in the stack
2. Find the maximum frequency
3. Scan backward from the top of the stack
4. Remove the first element whose frequency equals the maximum

This approach works because scanning backward naturally respects the "closest to the top" tie-breaking rule.

However, this is inefficient. Every `pop()` may require traversing the entire stack multiple times:

- Once for frequency counting
- Once for backward scanning
- Potentially additional work for removal

With up to `20,000` operations, worst-case performance becomes too slow.

### Optimal Approach

The key insight is that we should organize elements by frequency instead of maintaining only insertion order.

We need two mappings:

1. A mapping from value → current frequency
2. A mapping from frequency → stack of values having that frequency

We also maintain the current maximum frequency.

The critical observation is this:

When a value's frequency increases to `f`, we append it to the stack corresponding to frequency `f`.

This automatically preserves recency ordering among elements with the same frequency.

When popping:

- We always pop from the stack corresponding to the current maximum frequency
- Since that stack preserves insertion order, the most recent tied element is returned automatically

This gives constant-time operations.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) per pop | O(n) | Recomputes frequencies repeatedly |
| Optimal | O(1) per operation | O(n) | Uses frequency groups and hash maps |

## Algorithm Walkthrough

1. Create a hash map called `frequency`.

This stores the current frequency of every value.

Example:

```
frequency[5] = 3
```

means the value `5` currently appears three times in the structure.
2. Create another hash map called `group`.

This maps a frequency to a stack of values that reached that frequency.

Example:

```
group[2] = [5, 7]
```

means:

- `5` reached frequency 2 earlier
- `7` reached frequency 2 later

Therefore, if both are tied at frequency 2, `7` should be popped first.
3. Maintain a variable called `max_frequency`.

This tracks the highest frequency currently present in the structure.
4. During `push(val)`:

- Increment the frequency of `val`
- Store the new frequency
- Push `val` into the stack associated with that frequency
- Update `max_frequency` if necessary

Example:

```
push(5)
```

If `5` previously had frequency 2:

- New frequency becomes 3
- Append `5` to `group[3]`
5. During `pop()`:

- Look at the stack corresponding to `max_frequency`
- Pop the most recent element from that stack
- Decrease its frequency in `frequency`
- If the stack for `max_frequency` becomes empty, decrement `max_frequency`
6. Return the popped value.

The reason this works is that each frequency group behaves like a stack ordered by recency.

### Why it works

The invariant is:

- `group[f]` always stores values in the order they reached frequency `f`
- The top of `group[max_frequency]` is therefore the most recent element among all elements with maximum frequency

Because every push inserts into exactly one frequency group, and every pop removes from the current highest-frequency group, the structure always returns the correct value according to both priority rules.

## Python Solution

```
from collections import defaultdict
from typing import Dict, List

class FreqStack:

    def __init__(self):
        self.frequency: Dict[int, int] = defaultdict(int)
        self.group: Dict[int, List[int]] = defaultdict(list)
        self.max_frequency = 0

    def push(self, val: int) -> None:
        new_frequency = self.frequency[val] + 1
        self.frequency[val] = new_frequency

        self.group[new_frequency].append(val)

        if new_frequency > self.max_frequency:
            self.max_frequency = new_frequency

    def pop(self) -> int:
        val = self.group[self.max_frequency].pop()

        self.frequency[val] -= 1

        if not self.group[self.max_frequency]:
            self.max_frequency -= 1

        return val

# Your FreqStack object will be instantiated and called as such:
# obj = FreqStack()
# obj.push(val)
# param_2 = obj.pop()
```

The implementation uses two `defaultdict` structures.

The `frequency` map tracks how many times each value currently exists in the structure. Every push increments this count, and every pop decrements it.

The `group` map organizes values by frequency. Each frequency level stores values in insertion order using a list, which behaves like a stack.

During `push()`, we first compute the new frequency for the value. Then we append the value into the stack corresponding to that frequency.

The `max_frequency` variable allows us to instantly know which frequency group should be popped from. Without it, we would need to search for the highest frequency every time.

During `pop()`, we directly access the stack associated with the maximum frequency. Since the stack preserves recency order, the popped element is automatically the correct one according to the problem rules.

If the frequency group becomes empty after removal, we decrease `max_frequency` because no elements remain at that frequency.

## Go Solution

```
type FreqStack struct {
	frequency   map[int]int
	group       map[int][]int
	maxFreq int
}

func Constructor() FreqStack {
	return FreqStack{
		frequency: make(map[int]int),
		group:     make(map[int][]int),
		maxFreq:   0,
	}
}

func (this *FreqStack) Push(val int) {
	newFreq := this.frequency[val] + 1
	this.frequency[val] = newFreq

	this.group[newFreq] = append(this.group[newFreq], val)

	if newFreq > this.maxFreq {
		this.maxFreq = newFreq
	}
}

func (this *FreqStack) Pop() int {
	stack := this.group[this.maxFreq]

	lastIndex := len(stack) - 1
	val := stack[lastIndex]

	this.group[this.maxFreq] = stack[:lastIndex]

	this.frequency[val]--

	if len(this.group[this.maxFreq]) == 0 {
		this.maxFreq--
	}

	return val
}

/**
 * Your FreqStack object will be instantiated and called as such:
 * obj := Constructor();
 * obj.Push(val);
 * param_2 := obj.Pop();
 */
```

The Go implementation mirrors the Python logic closely.

Since Go does not have a built-in stack type, slices are used as stacks. Appending adds elements to the top, and slicing removes the last element during pop operations.

Maps in Go return the zero value for missing keys, which simplifies frequency counting because missing entries naturally behave like frequency zero.

Unlike Python's `defaultdict`, Go requires explicit initialization of the maps in the constructor.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(1) | Every push and pop performs only hash map and stack operations |
| Space | O(n) | We store frequency data and grouped stacks for all inserted elements |

Each operation performs a constant number of dictionary lookups and stack operations.

No scanning or sorting occurs during either `push()` or `pop()`, which is why the solution achieves constant-time complexity.

The space complexity is linear because every pushed element is stored once inside a frequency group.

## Edge Cases

### Multiple values with identical maximum frequency

This is the most important edge case in the problem. A buggy implementation may correctly identify the maximum frequency but fail to return the most recently inserted element among tied candidates.

For example:

```
push(1)
push(2)
push(1)
push(2)
```

Both `1` and `2` have frequency `2`. The correct answer is `2` because it reached frequency `2` more recently.

The implementation handles this correctly because each frequency level maintains its own stack ordered by insertion time.

### Frequency decreases after popping

After an element is popped, its frequency decreases. This can change which frequency is currently the maximum.

For example:

```
push(5)
push(5)
pop()
```

After the pop, frequency `2` becomes empty, so `max_frequency` must decrease to `1`.

The implementation explicitly checks whether the maximum-frequency stack becomes empty after a pop and updates `max_frequency` accordingly.

### Repeated pushes of the same value

A value can appear many times consecutively:

```
push(7)
push(7)
push(7)
push(7)
```

A naive implementation might overwrite previous state incorrectly.

This implementation handles repeated pushes naturally because every push creates a new entry in the appropriate frequency stack. Each frequency transition is recorded independently, preserving the correct historical ordering.
