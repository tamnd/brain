## Solution

# Problem Understanding

In this problem, we are given a sentence containing words separated by single spaces. Our task is to transform every word into a modified language called "Goat Latin" by following a fixed set of rules.

The transformation depends on the first character of each word:

- If the word starts with a vowel, we simply append `"ma"` to the end.
- If the word starts with a consonant, we remove the first character, move it to the end of the word, and then append `"ma"`.

After applying one of those two transformations, we must also append a sequence of `'a'` characters. The number of `'a'` characters depends on the position of the word in the sentence, starting from `1`. The first word gets `"a"`, the second word gets `"aa"`, the third gets `"aaa"`, and so on.

The final result is all transformed words joined back together into a sentence separated by spaces.

For example:

- `"apple"` starts with a vowel, so it becomes `"applema"`.
- `"goat"` starts with a consonant, so we move `'g'` to the end and get `"oatgma"`.

Then we add the appropriate number of `'a'` characters based on the word index.

The constraints are very small:

- The sentence length is at most `150`.
- Words contain only English letters.
- Words are separated by exactly one space.
- There are no leading or trailing spaces.

These guarantees simplify parsing because we can safely use a normal string split operation without worrying about empty tokens.

Some important edge cases include:

- A sentence with only one word.
- Words that begin with uppercase vowels like `"I"` or `"Apple"`.
- Very short words consisting of a single character.
- Sentences where every word begins with a vowel or every word begins with a consonant.

A naive implementation could accidentally treat uppercase vowels as consonants, or incorrectly append the `'a'` suffix counts.

# Approaches

## Brute Force Approach

A brute force solution would repeatedly build strings character by character using inefficient concatenation. For every word, we could manually copy characters into a new string, repeatedly append suffixes, and rebuild the result one character at a time.

This works because the rules are deterministic and each word can be independently transformed. However, repeated immutable string concatenation can become inefficient because every concatenation may create a new string and copy existing content.

Although the constraints are small enough that even this method would pass, it is not ideal from an algorithmic perspective.

## Optimal Approach

The key observation is that each word can be processed independently in a single pass through the sentence.

We can:

1. Split the sentence into words.
2. Check whether the first character is a vowel.
3. Apply the corresponding transformation.
4. Append `"ma"` and the appropriate number of `'a'` characters.
5. Store each transformed word in a list.
6. Join everything together once at the end.

This approach is efficient because:

- Each word is visited exactly once.
- Each character is copied only a constant number of times.
- Using a list and joining once avoids repeated expensive string concatenation.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeated string concatenation can repeatedly copy data |
| Optimal | O(n) | O(n) | Processes each character once and joins efficiently |

# Algorithm Walkthrough

1. Create a set containing all vowels, both lowercase and uppercase.

We include uppercase vowels because the problem statement allows uppercase letters, and words like `"I"` should be treated as vowel-starting words.
2. Split the sentence into a list of words.

Since the input guarantees exactly one space between words and no leading or trailing spaces, a standard split operation works perfectly.
3. Create an empty list to store transformed words.

Building the final sentence incrementally in a list is more efficient than repeated string concatenation.
4. Iterate through the words using their index.

The index is important because the number of `'a'` characters depends on the word position.
5. For each word, determine whether the first character is a vowel.

- If it is a vowel:

- Keep the word unchanged.
- Append `"ma"`.
- If it is a consonant:

- Remove the first character.
- Move it to the end.
- Append `"ma"`.
6. Append `'a' * (index + 1)` to the transformed word.

The first word gets one `'a'`, so we use `index + 1`.
7. Store the transformed word in the result list.
8. Join all transformed words with spaces and return the final sentence.

## Why it works

Each word transformation exactly follows the Goat Latin rules defined in the problem statement. The algorithm processes every word independently and appends the correct number of `'a'` characters according to its position. Since every word is transformed according to the required specification and the final ordering is preserved, the resulting sentence is guaranteed to be correct.

# Python Solution

```
class Solution:
    def toGoatLatin(self, sentence: str) -> str:
        vowels = set("aeiouAEIOU")
        
        words = sentence.split()
        transformed_words = []
        
        for index, word in enumerate(words):
            if word[0] in vowels:
                transformed_word = word + "ma"
            else:
                transformed_word = word[1:] + word[0] + "ma"
            
            transformed_word += "a" * (index + 1)
            transformed_words.append(transformed_word)
        
        return " ".join(transformed_words)
```

The implementation begins by creating a vowel set containing both lowercase and uppercase vowels. Using a set allows constant time membership checks.

Next, the sentence is split into words using `split()`. Because the problem guarantees clean spacing, this produces exactly the desired tokens.

We then iterate through each word using `enumerate()`, which provides both the word and its position. The position is necessary for generating the correct number of `'a'` characters.

Inside the loop, we check whether the first character is a vowel. If it is, we append `"ma"` directly. Otherwise, we move the first character to the end before appending `"ma"`.

Afterward, we append `"a" * (index + 1)` to satisfy the positional suffix rule.

Finally, all transformed words are joined with spaces and returned as the final sentence.

# Go Solution

```
package main

import "strings"

func toGoatLatin(sentence string) string {
	vowels := map[byte]bool{
		'a': true, 'e': true, 'i': true, 'o': true, 'u': true,
		'A': true, 'E': true, 'I': true, 'O': true, 'U': true,
	}

	words := strings.Split(sentence, " ")
	result := make([]string, 0, len(words))

	for i, word := range words {
		var transformed string

		if vowels[word[0]] {
			transformed = word + "ma"
		} else {
			transformed = word[1:] + string(word[0]) + "ma"
		}

		transformed += strings.Repeat("a", i+1)
		result = append(result, transformed)
	}

	return strings.Join(result, " ")
}
```

The Go solution follows the same algorithmic structure as the Python version. One difference is that Go strings are byte slices, so accessing `word[0]` returns a byte. Since the input contains only English letters, byte indexing is completely safe here.

We use a `map[byte]bool` for vowel lookup and `strings.Repeat()` to generate the repeated `'a'` suffixes efficiently.

The result is accumulated in a string slice and joined once at the end using `strings.Join()`.

# Edge Cases

## Single Character Words

Words of length one can easily expose slicing bugs. For example, when transforming `"b"`, removing the first character leaves an empty substring. A careless implementation might accidentally produce invalid indexing behavior.

This implementation handles the case safely because `word[1:]` becomes an empty string, and appending `word[0]` still produces the correct result.

## Uppercase Vowels

The problem allows uppercase letters, so words like `"I"` and `"Apple"` must be treated as vowel-starting words. A common mistake is checking only lowercase vowels.

This implementation avoids that issue by including both lowercase and uppercase vowels in the vowel set.

## Large Numbers of Appended 'a' Characters

As word positions increase, the number of appended `'a'` characters also increases. An incorrect implementation might accidentally use zero-based indexing and append too few characters.

This implementation correctly uses `(index + 1)` so that the first word gets one `'a'`, the second gets two, and so forth.
