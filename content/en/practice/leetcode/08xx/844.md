## Solution

## Problem Understanding

The problem gives us two strings, `s` and `t`, that represent text typed into a text editor. The special character `'#'` acts as a backspace operation. Every time we encounter `'#'`, the most recently typed character is removed, if one exists.

Our task is to determine whether the final text produced from `s` is identical to the final text produced from `t`.

For example, consider:

```
s = "ab#c"
```

Typing proceeds like this:

- Type `'a'` → `"a"`
- Type `'b'` → `"ab"`
- `'#'` removes `'b'` → `"a"`
- Type `'c'` → `"ac"`

So the final result is `"ac"`.

We perform the same simulation for `t` and compare the final outputs.

The constraints are relatively small, with both strings having length at most `200`. This means even less efficient solutions are acceptable from a performance standpoint. However, the follow up specifically asks whether we can achieve `O(n)` time and `O(1)` extra space, which suggests that the intended optimal solution avoids explicitly constructing the final strings.

Several edge cases are important:

- Multiple consecutive backspaces, such as `"a###"`
- Backspaces applied to an already empty string
- Strings that become empty after processing
- Different raw strings that produce identical final text
- Strings with different lengths after processing

A naive implementation can easily fail if it does not correctly handle extra backspaces or if it compares characters before fully applying deletions.

## Approaches

### Brute Force Approach

The most direct solution is to simulate typing for both strings using a stack.

We iterate through each character:

- If the character is a normal letter, we push it onto the stack.
- If the character is `'#'`, we pop from the stack if the stack is not empty.

At the end, the stack contains the final edited text. We repeat this process for both strings and compare the results.

This approach is straightforward because stacks naturally model backspace behavior. The most recently typed character is exactly the character that should be removed first.

Although this solution is efficient enough for the given constraints, it uses additional memory proportional to the size of the strings.

### Optimal Approach

The key observation is that we do not actually need to build the final strings.

Instead, we can process both strings from right to left.

When scanning backward:

- A `'#'` means we should skip one future character.
- Multiple `'#'` characters accumulate multiple skips.
- We can dynamically determine the next valid character in each string.

This allows us to compare characters one at a time without constructing new strings or stacks.

The two pointer technique works perfectly here because backspaces only affect characters to their left. By traversing backward, we always know how many characters must be ignored.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n + m) | O(n + m) | Simulates typing using stacks |
| Optimal | O(n + m) | O(1) | Uses backward traversal with two pointers |

## Algorithm Walkthrough

### Optimal Two Pointer Algorithm

1. Initialize two pointers:

- `i` starts at the end of `s`
- `j` starts at the end of `t`
2. Create two skip counters:

- `skip_s`
- `skip_t`

These counters track how many characters should be ignored because of backspaces.
3. Process string `s` backward until a valid character is found:

- If `s[i]` is `'#'`, increment `skip_s` and move left.
- If `skip_s > 0`, decrement `skip_s` and move left.
- Otherwise, the current character is valid and ready for comparison.
4. Process string `t` the same way using `j` and `skip_t`.
5. After locating the next valid character in both strings:

- If both characters exist and are different, return `False`.
- If only one string still has a valid character, return `False`.
- Otherwise, continue moving both pointers left.
6. If the entire traversal finishes without mismatches, return `True`.

### Why it works

The algorithm maintains the invariant that each pointer always lands on the next valid character after all backspaces to its right have been applied.

Because we process the strings backward, every backspace operation is fully resolved before comparing characters. This guarantees that each comparison reflects the actual final editor state, making the result correct.

## Python Solution

```
class Solution:
    def backspaceCompare(self, s: str, t: str) -> bool:
        i = len(s) - 1
        j = len(t) - 1

        skip_s = 0
        skip_t = 0

        while i >= 0 or j >= 0:
            # Find next valid character in s
            while i >= 0:
                if s[i] == '#':
                    skip_s += 1
                    i -= 1
                elif skip_s > 0:
                    skip_s -= 1
                    i -= 1
                else:
                    break

            # Find next valid character in t
            while j >= 0:
                if t[j] == '#':
                    skip_t += 1
                    j -= 1
                elif skip_t > 0:
                    skip_t -= 1
                    j -= 1
                else:
                    break

            # Compare characters
            if i >= 0 and j >= 0:
                if s[i] != t[j]:
                    return False
            elif i >= 0 or j >= 0:
                return False

            i -= 1
            j -= 1

        return True
```

The implementation follows the exact two pointer strategy described earlier.

The variables `i` and `j` traverse the strings from right to left. The skip counters track how many characters must be ignored due to backspaces.

The inner `while` loops are responsible for locating the next usable character in each string. If a `'#'` is found, the skip counter increases. If a normal character is encountered while the skip counter is positive, that character is skipped because it would have been deleted.

Once both pointers reach valid characters, the algorithm compares them. Any mismatch immediately returns `False`.

If traversal completes successfully, both processed strings are identical, so the algorithm returns `True`.

## Go Solution

```
func backspaceCompare(s string, t string) bool {
	i := len(s) - 1
	j := len(t) - 1

	skipS := 0
	skipT := 0

	for i >= 0 || j >= 0 {

		// Find next valid character in s
		for i >= 0 {
			if s[i] == '#' {
				skipS++
				i--
			} else if skipS > 0 {
				skipS--
				i--
			} else {
				break
			}
		}

		// Find next valid character in t
		for j >= 0 {
			if t[j] == '#' {
				skipT++
				j--
			} else if skipT > 0 {
				skipT--
				j--
			} else {
				break
			}
		}

		// Compare characters
		if i >= 0 && j >= 0 {
			if s[i] != t[j] {
				return false
			}
		} else if i >= 0 || j >= 0 {
			return false
		}

		i--
		j--
	}

	return true
}
```

The Go implementation mirrors the Python logic closely.

One important difference is that Go strings are indexed as bytes. Since the problem guarantees only lowercase English letters and `'#'`, byte indexing is completely safe here.

Go also requires explicit variable declarations and uses `false` and `true` in lowercase. Other than syntax differences, the algorithmic behavior is identical.

## Worked Examples

### Example 1

```
s = "ab#c"
t = "ad#c"
```

Final processed strings should both become `"ac"`.

| Step | i | s[i] | skip_s | j | t[j] | skip_t | Action |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 3 | c | 0 | 3 | c | 0 | Compare c vs c |
| 2 | 2 | # | 0 | 2 | # | 0 | Increment skips |
| 3 | 1 | b | 1 | 1 | d | 1 | Skip characters |
| 4 | 0 | a | 0 | 0 | a | 0 | Compare a vs a |

Both valid characters match at every step, so the result is `True`.

### Example 2

```
s = "ab##"
t = "c#d#"
```

Both strings become empty.

| Step | Action |
| --- | --- |
| s: '#' encountered | skip_s = 1 |
| s: '#' encountered | skip_s = 2 |
| s: skip 'b' | skip_s = 1 |
| s: skip 'a' | skip_s = 0 |
| t: '#' encountered | skip_t = 1 |
| t: skip 'd' | skip_t = 0 |
| t: '#' encountered | skip_t = 1 |
| t: skip 'c' | skip_t = 0 |

No valid characters remain in either string, so the result is `True`.

### Example 3

```
s = "a#c"
t = "b"
```

Processed strings become:

```
s -> "c"
t -> "b"
```

| Step | Valid Character from s | Valid Character from t | Result |
| --- | --- | --- | --- |
| 1 | c | b | Mismatch |

Since the characters differ, the algorithm returns `False`.

| Measure | Complexity | Explanation |

The algorithm is linear because both pointers move strictly from right to left without revisiting characters. Every character is examined at most once.

The space complexity is constant because we do not allocate additional stacks or strings. Only a few integer variables are maintained throughout execution.

## Edge Cases

One important edge case occurs when backspaces exceed the number of available characters, such as `"a###"` or `"####"`. A buggy implementation might attempt to remove characters from an empty stack or produce invalid indices. The current implementation handles this safely by only decrementing skip counters and never accessing invalid positions.

Another tricky case involves consecutive backspaces, such as `"abc###"`. The algorithm must correctly chain deletions so that each backspace removes the latest remaining character. The backward traversal naturally supports this because skip counters accumulate deletions before valid characters are processed.

A third important edge case occurs when one processed string becomes empty while the other still contains valid characters. For example:

```
s = "a#"
t = "b"
```

After processing, `s` becomes empty while `t` becomes `"b"`. The implementation explicitly checks whether only one pointer still references a valid character and immediately returns `False` in that situation.

Finally, leading backspaces such as `"#a#c"` can cause issues for implementations that assume deletions always have preceding characters. Since the algorithm only skips characters when available, extra backspaces simply have no effect, which matches the problem specification exactly.
