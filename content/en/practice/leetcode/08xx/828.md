## Solution

## Problem Understanding

The problem asks us to compute the total number of unique characters across every possible substring of a given string.

A character is considered unique within a substring if it appears exactly once in that substring. For example, in the substring `"ABA"`, only `"B"` appears once, so its contribution is `1`.

We are given a string `s` consisting only of uppercase English letters. We must examine every substring `t` of `s`, compute `countUniqueChars(t)`, and sum all of those values together.

For example, if `s = "ABC"`, then every substring contains only distinct characters:

- `"A"` contributes `1`
- `"B"` contributes `1`
- `"C"` contributes `1`
- `"AB"` contributes `2`
- `"BC"` contributes `2`
- `"ABC"` contributes `3`

The total is `10`.

The key challenge is the input size. The string length can be as large as `10^5`. A naive solution that explicitly generates all substrings would be far too slow because the number of substrings is already `O(n^2)`.

The important realization is that we do not actually need to inspect every substring individually. Instead, we can determine how many substrings each character occurrence contributes to.

Several edge cases are important:

- Strings with all identical characters, such as `"AAAAA"`, where most substrings have no unique characters.
- Strings with all distinct characters, such as `"ABCDE"`, where every character contributes heavily.
- Characters appearing multiple times with large gaps between occurrences.
- Very short strings, especially length `1`.

The problem guarantees:

- `1 <= s.length <= 10^5`
- Characters are uppercase English letters only

Because the alphabet size is fixed at `26`, we can maintain occurrence information very efficiently.

## Approaches

### Brute Force Approach

The most direct approach is to generate every substring and count how many characters appear exactly once inside it.

For every starting index `i`, we expand the substring ending index `j`. For each substring `s[i:j+1]`, we maintain a frequency table and count how many characters currently have frequency `1`.

This approach is correct because it explicitly evaluates every substring exactly as the problem defines.

However, the time complexity is far too large. There are `O(n^2)` substrings, and updating or recomputing uniqueness information can cost additional time. Even with optimizations, the complexity remains too high for `n = 10^5`.

### Optimal Approach

The key insight is to reverse the perspective.

Instead of asking:

> For each substring, how many unique characters does it contain?

we ask:

> For each character occurrence, in how many substrings is this occurrence unique?

Suppose a character appears at index `i`.

This occurrence contributes to all substrings that:

- include index `i`
- do not include another occurrence of the same character

If we know:

- the previous occurrence of this character
- the next occurrence of this character

then we can determine exactly how many substrings treat this occurrence as unique.

Assume:

- `prev` is the previous occurrence index
- `next` is the next occurrence index

Then:

- the left boundary can be chosen from `(prev + 1)` through `i`
- the right boundary can be chosen from `i` through `(next - 1)`

Number of choices:

- left choices = `i - prev`
- right choices = `next - i`

Therefore, this occurrence contributes:

```
(i - prev) * (next - i)
```

to the final answer.

By summing this contribution for every character occurrence, we obtain the total answer in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) to O(n³) | O(1) to O(26) | Enumerates all substrings and counts unique characters directly |
| Optimal | O(n) | O(n) | Computes contribution of each character occurrence independently |

## Algorithm Walkthrough

1. Create a mapping from each character to the list of indices where it appears.

For example, for `"ABA"`:

```
A -> [0, 2]
B -> [1]
```

We need occurrence positions because each occurrence's contribution depends on its neighboring occurrences.
2. For each character, add sentinel boundaries.

Suppose the indices for a character are:

```
[2, 5, 9]
```

We transform this into:

```
[-1, 2, 5, 9, n]
```

where `n` is the string length.

The sentinel values simplify boundary calculations for the first and last occurrences.
3. Iterate through each real occurrence.

For an occurrence at position `curr`:

- `prev` is the previous occurrence
- `next` is the next occurrence

Compute:

```
left_choices = curr - prev
right_choices = next - curr
contribution = left_choices * right_choices
```
4. Add the contribution to the answer.

This contribution counts every substring where this occurrence is the only occurrence of that character.
5. Continue until all character occurrences are processed.
6. Return the accumulated total.

### Why it works

For a character occurrence at index `i`, a substring contributes this occurrence as unique only if the substring excludes every other occurrence of the same character.

The nearest conflicting occurrences are the previous and next occurrences. Any valid substring must start after the previous occurrence and end before the next occurrence.

The number of valid left boundaries is `i - prev`, and the number of valid right boundaries is `next - i`.

Multiplying these counts gives exactly the number of substrings where this occurrence is unique. Since every valid substring-character pair is counted exactly once, the total sum is correct.

## Python Solution

```
from collections import defaultdict
from typing import List

class Solution:
    def uniqueLetterString(self, s: str) -> int:
        positions = defaultdict(list)

        for index, char in enumerate(s):
            positions[char].append(index)

        total = 0
        n = len(s)

        for indices in positions.values():
            extended = [-1] + indices + [n]

            for i in range(1, len(extended) - 1):
                prev_index = extended[i - 1]
                current_index = extended[i]
                next_index = extended[i + 1]

                left_choices = current_index - prev_index
                right_choices = next_index - current_index

                total += left_choices * right_choices

        return total
```

The implementation begins by grouping indices for each character. This allows us to process every occurrence independently.

The `defaultdict(list)` structure is convenient because we can append indices without checking whether the character already exists in the map.

After collecting positions, we iterate through each character's occurrence list. Sentinel values `-1` and `n` are inserted so that the same formula works uniformly for all occurrences, including the first and last.

For every occurrence:

- `left_choices` counts how many starting positions are possible
- `right_choices` counts how many ending positions are possible

Their product equals the number of substrings where this occurrence is unique.

The total sum across all occurrences becomes the final answer.

## Go Solution

```
func uniqueLetterString(s string) int {
	positions := make(map[byte][]int)

	for i := 0; i < len(s); i++ {
		positions[s[i]] = append(positions[s[i]], i)
	}

	total := 0
	n := len(s)

	for _, indices := range positions {
		extended := make([]int, 0, len(indices)+2)
		extended = append(extended, -1)

		for _, index := range indices {
			extended = append(extended, index)
		}

		extended = append(extended, n)

		for i := 1; i < len(extended)-1; i++ {
			prevIndex := extended[i-1]
			currentIndex := extended[i]
			nextIndex := extended[i+1]

			leftChoices := currentIndex - prevIndex
			rightChoices := nextIndex - currentIndex

			total += leftChoices * rightChoices
		}
	}

	return total
}
```

The Go implementation follows the same logic as the Python version.

A `map[byte][]int` stores occurrence indices for each character. Since the string contains only uppercase English letters, indexing by byte is sufficient.

Slices are used to build the extended occurrence list containing sentinel boundaries. Integer arithmetic remains safe because the problem guarantees the answer fits in a 32-bit integer.

## Edge Cases

### Single Character String

A string such as `"A"` is the smallest possible input. There is only one substring, and the single character is unique within it. A buggy implementation might mishandle sentinel boundaries or produce zero contributions. The algorithm handles this correctly because the extended occurrence list becomes `[-1, 0, 1]`, producing exactly one valid substring.

### All Characters Identical

Consider `"AAAA"`.

Most substrings contain repeated `'A'` characters, so only the single-letter substrings contribute. Naive counting logic can easily overcount here. The contribution formula correctly limits each occurrence to substrings that exclude neighboring occurrences, resulting in exactly four contributing substrings.

### Characters Reappearing After Large Gaps

In a string like `"ABCADEAF"`, the same character may appear multiple times with many unrelated characters in between. Incorrect implementations may only consider immediate neighboring positions in the substring rather than globally nearest equal characters. This algorithm explicitly tracks the previous and next occurrence of the same character, guaranteeing correct contribution boundaries regardless of spacing.
