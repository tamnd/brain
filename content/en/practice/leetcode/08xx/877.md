## Problem Understanding

This problem involves a two-player game where Alice and Bob take turns removing piles of stones from either the beginning or the end of a row. Each pile contains a positive number of stones, and the total number of stones is **odd**, ensuring there is always a winner. Alice always moves first. The goal is to determine if Alice, playing optimally, will always win the game.

The input is a list of integers `piles`, representing the number of stones in each pile. The output is a boolean value: `true` if Alice wins and `false` if Bob wins. The constraints guarantee that `piles.length` is even, `2 <= piles.length <= 500`, each pile has at least one stone and at most 500 stones, and the total number of stones is odd.

Important observations are that because the length is even, the game is symmetric in the sense that Alice can always control whether she takes piles from even or odd indices. Edge cases to consider include very small inputs (length 2) and highly skewed piles (e.g., one pile vastly larger than the others). The problem guarantees that no tie is possible.

## Approaches

The brute-force approach would simulate every possible sequence of moves for Alice and Bob. For each turn, you would consider taking the left or right pile and recursively calculate the maximum stones Alice could obtain, assuming both play optimally. This approach correctly models the game but has an exponential time complexity because every move branches into two possibilities, leading to $O(2^n)$ time, which is infeasible for `n` up to 500.

The key insight is that this problem is a variant of the classic "take the ends of an array" game. Because Alice moves first and the number of piles is even, she can always choose either all even-indexed piles or all odd-indexed piles. By summing the stones at even indices and at odd indices, she can always guarantee selecting the sequence with more stones. This guarantees Alice's win and eliminates the need for dynamic programming or recursion. Thus, the optimal solution has $O(n)$ time and $O(1)$ space complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Recursively tries all sequences of moves; too slow for n = 500 |
| Optimal | O(n) | O(1) | Uses the parity insight of even/odd indices; Alice can always force a win |

## Algorithm Walkthrough

1. Calculate the sum of stones at even indices of the array. This represents the total Alice could collect if she chooses all piles at even positions.
2. Calculate the sum of stones at odd indices of the array. This represents the total Alice could collect if she chooses all piles at odd positions.
3. Compare the sums from steps 1 and 2. Alice will choose the sequence (even or odd indices) that maximizes her total stones.
4. Since Alice can always pick the optimal parity sequence and the total sum is odd, she will always win. Return `true`.

Why it works: The even-length array allows Alice to control the parity of piles she picks by choosing first. No matter how Bob responds, Alice can stick to the optimal parity (even or odd) and collect more stones than Bob. The total sum being odd ensures a strict win.

## Python Solution

```
from typing import List

class Solution:
    def stoneGame(self, piles: List[int]) -> bool:
        even_sum = sum(piles[i] for i in range(0, len(piles), 2))
        odd_sum = sum(piles[i] for i in range(1, len(piles), 2))
        return True  # Alice can always pick the parity with the larger sum
```

In this Python solution, we calculate the sums of stones at even and odd indices. Because the number of piles is even, Alice can always choose the parity (even or odd) that yields the higher sum, guaranteeing a win. We do not need to explicitly simulate the game turns because the parity strategy is sufficient.

## Worked Examples

**Example 1:** `piles = [5,3,4,5]`

Alice can choose parity: even indices sum = 5 + 4 = 9, odd indices sum = 3 + 5 = 8. Alice picks even indices (sum 9) and guarantees a win.

**Example 2:** `piles = [3,7,2,3]`

Even indices sum = 3 + 2 = 5, odd indices sum = 7 + 3 = 10. Alice picks odd indices (sum 10) and guarantees a win.

## Edge Cases

**Smallest array:** With only two piles, Alice picks the larger pile. This is trivial but must be handled correctly. Implementation returns `true` automatically because she always moves first and can select optimally.

**Equal piles:** When all piles have the same number of stones, Alice still wins because she moves first and the total number of stones is odd. Even if Bob mirrors moves, Alice will have the first advantage.

**Highly skewed distribution:** If one pile is much larger than the others, Alice will pick that pile first. The parity strategy naturally handles this because she can always stick to a sequence that maximizes her total stones, guaranteeing the win.
