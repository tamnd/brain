## Problem Understanding

The problem asks whether you can visit all rooms in a building if each room may contain keys to other rooms. You start in room `0`, which is unlocked, and the rooms are numbered from `0` to `n - 1`. Each room `i` contains a list of keys represented as integers in `rooms[i]`. A key with value `k` allows you to unlock room `k`. Your task is to determine whether, starting from room `0`, you can eventually enter all rooms.

The input `rooms` is a list of lists, where `rooms[i]` contains distinct integers representing the keys found in room `i`. The output is a boolean: `true` if all rooms can be visited, otherwise `false`.

The constraints tell us that there can be up to 1000 rooms, each containing up to 1000 keys, and the total number of keys is at most 3000. This ensures that the input is manageable but large enough that naive recursive approaches without bookkeeping could be inefficient.

Important edge cases include:

- Some rooms have no keys, meaning they cannot help unlock others.
- Keys may create cycles, e.g., room `1` has key to room `2` and room `2` has key back to room `1`.
- A room might be unreachable if its key is inside itself.
- The input guarantees keys are valid (i.e., `0 <= rooms[i][j] < n`) and unique per room, so no need to handle duplicates inside a room.

## Approaches

The brute-force approach would attempt to explore every possible order of visiting rooms, trying each key combination recursively. While this approach could give the correct answer, it is highly inefficient due to exponential growth in paths as the number of rooms increases. This approach essentially performs a combinatorial search over all key sequences.

The key insight is that this problem can be modeled as a graph traversal problem. Each room is a node, and keys represent directed edges to other rooms. Starting from room `0`, we simply need to determine whether we can reach all nodes in this directed graph. This is efficiently solved using Depth-First Search (DFS) or Breadth-First Search (BFS), which explores reachable nodes while keeping track of visited rooms to avoid revisiting.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n!) | O(n) | Recursively try all sequences of keys, impractical for n up to 1000 |
| Optimal (DFS/BFS) | O(n + k) | O(n) | Graph traversal; `k` is the total number of keys; visit each room once |

## Algorithm Walkthrough

1. Initialize a set `visited` to keep track of rooms that have already been entered.
2. Initialize a stack (for DFS) or queue (for BFS) with room `0` as the starting point.
3. While the stack/queue is not empty:

1. Pop a room `current` from the stack/queue.
2. If `current` has not been visited, mark it as visited.
3. Iterate over each key in `rooms[current]`. For each key `next_room`, if it has not been visited, add it to the stack/queue for future exploration.
4. After traversal, check if the size of `visited` is equal to `n`. If it is, return `true` as all rooms have been visited; otherwise, return `false`.

**Why it works:** The algorithm maintains the invariant that every room in `visited` has been reached starting from room `0`. Since every key is a valid edge in the graph, DFS or BFS will eventually visit all reachable rooms. The final comparison with `n` ensures that no room is left unvisited.

## Python Solution

```
from typing import List

class Solution:
    def canVisitAllRooms(self, rooms: List[List[int]]) -> bool:
        visited = set()
        stack = [0]
        
        while stack:
            current = stack.pop()
            if current not in visited:
                visited.add(current)
                for key in rooms[current]:
                    if key not in visited:
                        stack.append(key)
        
        return len(visited) == len(rooms)
```

The code initializes a set `visited` to track entered rooms and uses a stack for DFS traversal. For each room popped from the stack, it marks it as visited and adds all unvisited keys from that room to the stack. At the end, it checks if all rooms have been visited, matching our algorithm walkthrough.

## Go Solution

```
func canVisitAllRooms(rooms [][]int) bool {
    visited := make([]bool, len(rooms))
    stack := []int{0}
    
    for len(stack) > 0 {
        current := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        
        if !visited[current] {
            visited[current] = true
            for _, key := range rooms[current] {
                if !visited[key] {
                    stack = append(stack, key)
                }
            }
        }
    }
    
    for _, v := range visited {
        if !v {
            return false
        }
    }
    
    return true
}
```

In Go, we use a boolean slice `visited` instead of a set for efficiency. The stack is implemented using a slice. The logic mirrors the Python version: pop a room, mark it as visited, and add keys to the stack if unvisited. Finally, we iterate through `visited` to ensure all rooms were reached.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n + k) | Each room is visited at most once; `k` is total keys across all rooms |
| Space | O(n) | Visited set/boolean array stores all rooms; stack/queue stores at most n rooms |

The time complexity is linear in the number of rooms plus the total number of keys because we process each room once and iterate over each key once. Space is linear due to the visited tracking and stack/queue usage.

## Edge Cases

**Edge Case 1: No keys in starting room**

If room `0` contains no keys, no other rooms are reachable. This can easily happen with input like `[[ ], [1]]`. The algorithm correctly returns `False` because only room 0 is visited.

**Edge Case 2: Self-loop keys**

A room may contain a key to itself. For example, `[[0], [2]]`. The algorithm handles this correctly because visited rooms are checked before pushing keys onto the stack. This prevents infinite loops.
