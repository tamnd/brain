# LeetCode 825 - Friends Of Appropriate Ages

## Problem Understanding

This problem asks us to count how many friend requests are sent between people on a social media platform based on age restrictions.

We are given an array `ages`, where each value represents the age of one person. For every pair of people `(x, y)`, we need to determine whether person `x` sends a friend request to person `y`.

A request is rejected if any of these conditions are true:

1. `age[y] <= 0.5 * age[x] + 7`
2. `age[y] > age[x]`
3. `age[y] > 100 && age[x] < 100`

If none of these conditions apply, then `x` sends a friend request to `y`.

The important detail is that friend requests are directional. If `x` sends a request to `y`, it does not mean `y` sends one back. Also, nobody can send a request to themselves.

The task is to return the total number of valid friend requests.

The constraints are relatively small:

- `1 <= n <= 2 * 10^4`
- `1 <= ages[i] <= 120`

The age values are especially important because they are bounded between 1 and 120. This suggests that counting or frequency-based techniques may be useful.

Several edge cases are important:

- People younger than 15 can never send requests, because `0.5 * age + 7` is always too restrictive.
- Duplicate ages matter because people of the same age may mutually send requests.
- Requests are directional, so `(a -> b)` and `(b -> a)` are counted separately.
- Self requests are not allowed, even if the age rules otherwise permit them.

## Approaches

### Brute Force Approach

The most direct solution is to examine every ordered pair of people `(i, j)` where `i != j`.

For each pair:

1. Let person `i` be the sender.
2. Let person `j` be the receiver.
3. Check the three invalid conditions.
4. If none apply, increment the answer.

This approach is correct because it explicitly evaluates every possible request.

However, the time complexity is `O(n^2)`, since we compare every pair of people. With `n = 20,000`, this becomes approximately 400 million comparisons, which is inefficient.

### Optimal Approach

The key observation is that the validity conditions depend only on ages, not on individual identities.

If we sort the ages array, then valid receivers for a given sender age form a contiguous range.

For a person with age `A`:

- The receiver must satisfy:

`age[y] > 0.5 * A + 7`
- The receiver must also satisfy:

`age[y] <= A`

After sorting, all valid recipients appear in a continuous window.

We can therefore use binary search to find:

- The first valid age
- The first age greater than `A`

The number of valid recipients is the size of this range minus one, because the sender cannot send a request to themselves.

This reduces the complexity to `O(n log n)` because we sort once and perform binary searches for each element.

## Algorithm Walkthrough

### Optimal Algorithm

1. Sort the `ages` array.

Sorting allows us to use binary search and efficiently identify valid age ranges.
2. Initialize a variable `total_requests = 0`.

This will store the final answer.
3. Iterate through each person in the sorted array.

Suppose the current age is `age`.
4. Compute the minimum valid receiver age.

The rule says a request is invalid if:

```
age[y] <= 0.5 * age[x] + 7
```

Therefore, valid receivers must satisfy:

```
age[y] > 0.5 * age[x] + 7
```
5. Use binary search to find the left boundary.

Find the first index where:

```
ages[index] > 0.5 * age + 7
```
6. Use binary search to find the right boundary.

Find the first index where:

```
ages[index] > age
```

Every age before this index is less than or equal to `age`.
7. Compute the number of valid receivers.

The valid range is:

```
[left, right)
```

Its size is:

```
right - left
```
8. Subtract one to exclude the sender themselves.

Since the sender is inside the valid range:

```
valid_count = right - left - 1
```
9. Add the valid count to the total.
10. Return `total_requests`.

### Why it works

After sorting, every valid recipient for a sender age `A` lies inside a contiguous interval:

```
0.5 * A + 7 < age[y] <= A
```

Binary search correctly identifies the boundaries of this interval. Every person inside the interval is a valid receiver, and every person outside violates at least one rule. Subtracting one removes the sender themselves. Summing over all senders counts every valid directed request exactly once.

## Python Solution

```
from bisect import bisect_right
from typing import List

class Solution:
    def numFriendRequests(self, ages: List[int]) -> int:
        ages.sort()
        total_requests = 0

        for age in ages:
            if age < 15:
                continue

            lower_bound = 0.5 * age + 7

            left = bisect_right(ages, lower_bound)
            right = bisect_right(ages, age)

            total_requests += right - left - 1

        return total_requests
```

The implementation begins by sorting the array so that all ages are in nondecreasing order. This ordering is essential because binary search only works on sorted data.

For each age, we first skip ages below 15. Such people can never send valid requests because the inequality becomes impossible to satisfy.

We compute the lower age threshold using the formula from the problem statement. Then we use `bisect_right` twice:

- The first call finds the first age strictly greater than the lower bound.
- The second call finds the first age greater than the current sender age.

The difference between these indices gives the number of people in the valid range. We subtract one to exclude the sender themselves.

Finally, we accumulate the counts and return the total.

## Go Solution

```
package main

import (
	"sort"
)

func numFriendRequests(ages []int) int {
	sort.Ints(ages)

	totalRequests := 0
	n := len(ages)

	for _, age := range ages {
		if age < 15 {
			continue
		}

		lowerBound := float64(age)/2.0 + 7.0

		left := sort.Search(n, func(i int) bool {
			return float64(ages[i]) > lowerBound
		})

		right := sort.Search(n, func(i int) bool {
			return ages[i] > age
		})

		totalRequests += right - left - 1
	}

	return totalRequests
}
```

The Go solution follows the same algorithmic structure as the Python version.

Instead of Python's `bisect_right`, Go uses `sort.Search`, which performs binary search using a predicate function.

The lower bound calculation uses `float64` because the formula involves division by 2. The logic for determining valid intervals remains identical.

Go slices naturally handle empty arrays, so no additional nil handling is required.

## Edge Cases

One important edge case involves very young users, especially ages below 15. These users can never send valid requests because the inequality:

```
age[y] > 0.5 * age[x] + 7
```

becomes impossible to satisfy. A naive implementation might still attempt unnecessary searches or accidentally count invalid pairs. The solution handles this by skipping all ages below 15 immediately.

Another critical case is duplicate ages. Suppose several people all have age 21. Every person can send requests to every other person with the same age, but not to themselves. This creates many directed requests. Incorrect implementations often forget to subtract one for self exclusion. The algorithm correctly computes the full interval size and subtracts one for the sender.

A third tricky case involves the special rule about ages above 100. Many solutions overcomplicate this condition. In reality, the second rule:

```
age[y] > age[x]
```

already prevents younger users from sending requests to older users above 100. The sorted interval logic automatically handles this without requiring separate code. This keeps the implementation simpler and less error prone.
