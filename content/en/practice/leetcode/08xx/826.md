## Solution

## Problem Understanding

The problem gives us three arrays:

- `difficulty[i]` represents how hard a job is.
- `profit[i]` represents how much money that job pays.
- `worker[j]` represents the maximum difficulty that worker `j` can handle.

Each worker can perform at most one job, but jobs can be reused by multiple workers. This is an important detail because it means we are not matching workers to unique jobs. Instead, every worker independently chooses the most profitable job they are capable of completing.

The goal is to maximize the total profit earned by all workers combined.

For example, suppose a worker has ability `5`. That worker can complete any job whose difficulty is less than or equal to `5`. Among all such jobs, we should assign the one with the highest profit.

The constraints are important:

- `n` and `m` can both be up to `10^4`
- Difficulties, profits, and worker abilities can be up to `10^5`

A naive solution that checks every worker against every job would require `O(n * m)` operations. In the worst case, that becomes `10^8` comparisons, which is inefficient for LeetCode medium constraints.

There are several edge cases we need to think about:

- A worker may not be able to complete any job at all.
- Multiple jobs can have the same difficulty but different profits.
- A harder job may pay less than an easier job.
- Multiple workers may choose the same job, which is completely allowed.
- Workers can appear in any order, so sorting may help.

The key challenge is efficiently finding the best available profit for each worker ability.

## Approaches

### Brute Force Approach

The simplest solution is to evaluate every worker independently.

For each worker:

1. Iterate through all jobs.
2. Check whether the worker can perform the job.
3. Track the maximum profit among all valid jobs.
4. Add that maximum profit to the answer.

This works because each worker only cares about the highest-paying job they can complete.

However, the time complexity is too high. If there are `m` workers and `n` jobs, we perform `m * n` comparisons.

With the maximum constraints:

- `n = 10^4`
- `m = 10^4`

The algorithm may perform up to `10^8` operations, which is unnecessarily slow.

### Optimal Approach

The key observation is that workers with larger ability can do everything weaker workers can do, plus possibly more jobs.

If we sort both:

- jobs by difficulty
- workers by ability

then we can process them in increasing order using a two-pointer technique.

As we move through workers from weakest to strongest:

- we continuously add jobs that the current worker can perform
- we maintain the best profit seen so far

This works because once a job becomes available for a worker, it also remains available for all stronger workers.

Instead of repeatedly scanning all jobs for every worker, we scan the jobs list only once.

| --- | --- | --- | --- |
| Brute Force | O(n * m) | O(1) | Check every job for every worker |
| Optimal | O(n log n + m log m) | O(n) | Sort jobs and workers, then use two pointers |

## Algorithm Walkthrough

### Optimal Algorithm

1. Combine the `difficulty` and `profit` arrays into a single list of jobs.

Each job becomes a pair:

```
(difficulty, profit)
```

This makes it easy to sort jobs while preserving the relationship between difficulty and profit.
2. Sort the jobs by difficulty.

After sorting, jobs appear from easiest to hardest. This allows us to progressively unlock jobs as worker ability increases.
3. Sort the `worker` array.

Processing workers from weakest to strongest allows us to reuse information from previous workers.
4. Initialize:

- `job_index = 0`
- `best_profit = 0`
- `total_profit = 0`

`job_index` tracks which jobs have become available.

`best_profit` stores the maximum profit among all jobs the current worker can perform.
5. Iterate through each worker ability in sorted order.

For the current worker:

- While the worker can perform the current job:

- update `best_profit`
- move `job_index` forward

This step ensures we process every job exactly once.
6. Add `best_profit` to `total_profit`.

Since `best_profit` always represents the highest-paying accessible job, this gives the optimal assignment for the current worker.
7. Continue until all workers are processed.
8. Return `total_profit`.

### Why it works

The algorithm maintains an important invariant:

`best_profit` always equals the maximum profit among all jobs whose difficulty is less than or equal to the current worker's ability.

Because both jobs and workers are sorted, we never need to reconsider earlier jobs. Every job is processed once, and every worker receives the best available profit at that point.

This guarantees the optimal total profit.

## Python Solution

```
from typing import List

class Solution:
    def maxProfitAssignment(
        self,
        difficulty: List[int],
        profit: List[int],
        worker: List[int]
    ) -> int:
        
        jobs = sorted(zip(difficulty, profit))
        worker.sort()

        total_profit = 0
        best_profit = 0
        job_index = 0

        for ability in worker:

            while job_index < len(jobs) and jobs[job_index][0] <= ability:
                best_profit = max(best_profit, jobs[job_index][1])
                job_index += 1

            total_profit += best_profit

        return total_profit
```

The implementation begins by combining the difficulty and profit arrays using `zip`. This creates pairs where each difficulty remains associated with its corresponding profit.

The jobs are sorted by difficulty so we can process them incrementally as worker ability increases.

The worker array is also sorted. This allows the algorithm to use a monotonic traversal strategy where we never move backward through the jobs list.

The variable `job_index` acts as a pointer into the sorted jobs array. As worker ability increases, more jobs become available.

The variable `best_profit` tracks the maximum profit encountered among all jobs accessible to the current worker. Since workers are processed in increasing order, this value never decreases.

For each worker, the inner `while` loop advances through all newly available jobs and updates `best_profit`.
