# LeetCode 840, Magic Squares In Grid

## Problem Understanding

The problem asks us to count how many `3 x 3` subgrids inside a larger matrix are valid magic squares.

A `3 x 3` magic square must satisfy several strict conditions:

1. Every number must be distinct.
2. Every number must be between `1` and `9`.
3. Every row must have the same sum.
4. Every column must have the same sum.
5. Both diagonals must also have the same sum.

The input is a two dimensional integer matrix called `grid`, with dimensions `row x col`. We must examine every possible `3 x 3` section of this matrix and determine whether it forms a magic square.

The output is a single integer representing the total number of valid magic square subgrids.

The constraints are relatively small:

- `1 <= row, col <= 10`
- `0 <= grid[i][j] <= 15`

These constraints immediately suggest that checking every possible `3 x 3` subgrid is feasible. Even in the largest case, there are at most:

$$(10 - 2) \times (10 - 2) = 64$$

possible `3 x 3` windows.

This is a very small search space, so an exhaustive verification approach is completely acceptable.

Several edge cases are important:

- A grid smaller than `3 x 3` can never contain a magic square.
- Numbers outside the range `1` to `9` immediately invalidate a candidate square.
- Duplicate numbers invalidate the square even if sums match.
- Some subgrids may satisfy row and column sums but fail diagonal checks.
- Some subgrids may have all distinct values but inconsistent sums.

A naive implementation can easily forget one of these conditions, especially the uniqueness requirement.

## Approaches

### Brute Force Approach

The brute force approach checks every possible `3 x 3` subgrid independently.

For each subgrid:

1. Extract all 9 numbers.
2. Verify that all numbers are distinct.
3. Verify that every number is between `1` and `9`.
4. Compute all row sums.
5. Compute all column sums.
6. Compute both diagonal sums.
7. Check whether all sums are equal.

This approach is guaranteed to work because it directly implements the definition of a magic square.

Although this sounds expensive, the constraints are tiny. There are at most 64 candidate subgrids, and each verification examines only 9 cells.

### Key Insight for the Optimal Approach

The important observation is that a `3 x 3` magic square using numbers `1` through `9` has very rigid structure.

Instead of recomputing large amounts of information repeatedly, we can validate each candidate subgrid efficiently with a few targeted checks:

- Every value must be in `[1, 9]`
- Every value must appear exactly once
- All rows, columns, and diagonals must sum to the same value

Since the subgrid size is fixed at `3 x 3`, every validation is effectively constant time.

Thus, the optimal solution is still conceptually brute force over all windows, but with efficient validation logic.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(row × col × 9) | O(9) | Explicitly checks every condition for every subgrid |
| Optimal | O(row × col) | O(1) | Each `3 x 3` window is validated in constant time |

## Algorithm Walkthrough

1. First, determine whether the grid is large enough to contain a `3 x 3` subgrid.

If either dimension is smaller than 3, immediately return `0`.
2. Iterate through every possible top left corner of a `3 x 3` subgrid.

If the grid dimensions are `m x n`, then the valid starting positions are:

- rows: `0` through `m - 3`
- columns: `0` through `n - 3`
3. For each candidate subgrid, run a helper validation function.

This helper function determines whether the current `3 x 3` region forms a magic square.
4. Inside the validation function, first check value range and uniqueness.

Use a set to track seen numbers.

For every cell in the `3 x 3` region:

- If the number is less than 1 or greater than 9, return `False`
- If the number already exists in the set, return `False`
- Otherwise insert it into the set
5. Compute the target sum.

The first row sum is used as the reference value.
6. Check all three row sums.

Every row must equal the target sum.
7. Check all three column sums.

Every column must also equal the target sum.
8. Check both diagonals.

The main diagonal and anti diagonal must match the same target sum.
9. If all checks pass, the subgrid is a valid magic square.
10. Count every valid subgrid and return the final total.

### Why it works

The algorithm directly enforces every condition in the definition of a magic square.

The uniqueness and range checks guarantee that the subgrid contains exactly the numbers `1` through `9` without repetition. The row, column, and diagonal checks guarantee that every required sum is equal. Since every possible `3 x 3` subgrid is examined exactly once, the algorithm counts all valid magic squares and no invalid ones.

## Python Solution

```
from typing import List

class Solution:
    def numMagicSquaresInside(self, grid: List[List[int]]) -> int:
        rows = len(grid)
        cols = len(grid[0])

        if rows < 3 or cols < 3:
            return 0

        def is_magic_square(r: int, c: int) -> bool:
            seen = set()

            # Check range and uniqueness
            for i in range(r, r + 3):
                for j in range(c, c + 3):
                    value = grid[i][j]

                    if value < 1 or value > 9:
                        return False

                    if value in seen:
                        return False

                    seen.add(value)

            # Reference sum from first row
            target = sum(grid[r][c:c + 3])

            # Check rows
            for i in range(r, r + 3):
                if sum(grid[i][c:c + 3]) != target:
                    return False

            # Check columns
            for j in range(c, c + 3):
                column_sum = (
                    grid[r][j]
                    + grid[r + 1][j]
                    + grid[r + 2][j]
                )

                if column_sum != target:
                    return False

            # Check diagonals
            diagonal1 = (
                grid[r][c]
                + grid[r + 1][c + 1]
                + grid[r + 2][c + 2]
            )

            diagonal2 = (
                grid[r][c + 2]
                + grid[r + 1][c + 1]
                + grid[r + 2][c]
            )

            return diagonal1 == target and diagonal2 == target

        count = 0

        for r in range(rows - 2):
            for c in range(cols - 2):
                if is_magic_square(r, c):
                    count += 1

        return count
```

The implementation begins by checking whether the matrix is large enough to contain a `3 x 3` region. This prevents unnecessary work and avoids index errors.

The helper function `is_magic_square` performs all validation logic for a candidate subgrid. It first ensures that all values are within the required range and are distinct. A Python `set` is ideal here because membership checks are average `O(1)` time.

The first row sum becomes the reference target. Every subsequent row, column, and diagonal is compared against this target. If any mismatch occurs, the function immediately returns `False`.

The outer loops iterate through every valid top left position for a `3 x 3` window. Every valid magic square increments the counter.

## Go Solution

```
func numMagicSquaresInside(grid [][]int) int {
	rows := len(grid)
	cols := len(grid[0])

	if rows < 3 || cols < 3 {
		return 0
	}

	isMagicSquare := func(r, c int) bool {
		seen := make(map[int]bool)

		// Check range and uniqueness
		for i := r; i < r+3; i++ {
			for j := c; j < c+3; j++ {
				value := grid[i][j]

				if value < 1 || value > 9 {
					return false
				}

				if seen[value] {
					return false
				}

				seen[value] = true
			}
		}

		target := grid[r][c] + grid[r][c+1] + grid[r][c+2]

		// Check rows
		for i := r; i < r+3; i++ {
			rowSum := grid[i][c] + grid[i][c+1] + grid[i][c+2]

			if rowSum != target {
				return false
			}
		}

		// Check columns
		for j := c; j < c+3; j++ {
			colSum := grid[r][j] + grid[r+1][j] + grid[r+2][j]

			if colSum != target {
				return false
			}
		}

		// Check diagonals
		diag1 := grid[r][c] + grid[r+1][c+1] + grid[r+2][c+2]
		diag2 := grid[r][c+2] + grid[r+1][c+1] + grid[r+2][c]

		return diag1 == target && diag2 == target
	}

	count := 0

	for r := 0; r < rows-2; r++ {
		for c := 0; c < cols-2; c++ {
			if isMagicSquare(r, c) {
				count++
			}
		}
	}

	return count
}
```

The Go implementation follows exactly the same logic as the Python solution. The primary difference is the use of a `map[int]bool` instead of a Python set for uniqueness tracking.

Go slices are used naturally for matrix access. Since the constraints are small, there are no concerns about integer overflow.

## Edge Cases

### Grid Smaller Than `3 x 3`

If the grid has fewer than 3 rows or 3 columns, there cannot possibly be a valid magic square. This case is easy to overlook and can lead to index out of bounds errors during subgrid extraction.

The implementation handles this immediately with:

```
if rows < 3 or cols < 3:
    return 0
```

This guarantees safe iteration later.

### Duplicate Numbers Inside the Subgrid

A subgrid may have correct row and column sums while still violating the uniqueness requirement. For example:

```
2 2 6
9 5 1
4 3 8
```

Without explicit duplicate checking, this could incorrectly be accepted.

The implementation uses a set to ensure every value appears exactly once.

### Values Outside the Range `1` to `9`

The problem definition requires that a magic square use only numbers from `1` through `9`.

A subgrid containing values like `0`, `10`, or `15` must be rejected immediately, even if sums match.

The implementation validates the range during traversal:

```
if value < 1 or value > 9:
    return False
```

This prevents invalid candidates from passing later sum checks.
