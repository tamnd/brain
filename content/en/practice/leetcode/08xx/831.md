## Solution

## Problem Understanding

This problem asks us to take a string representing personal information and return a masked version that hides sensitive details while preserving enough information to identify the format.

The input string `s` can represent one of two possible formats:

1. An email address
2. A phone number

The problem guarantees that the input is always valid, meaning we do not need to handle malformed emails or invalid phone numbers.

For email addresses, the masking rules are very specific. The name portion before the `@` symbol must be converted to lowercase, and all characters between the first and last character of the name must be replaced with exactly five asterisks. The domain portion after the `@` must also be converted to lowercase.

For example:

```
LeetCode@LeetCode.com
```

becomes:

```
l*****e@leetcode.com
```

Even if the name portion has only two characters, we still insert five asterisks in the middle.

For phone numbers, the input may contain formatting characters such as:

- `+`
- `-`
- `(`
- `)`
- spaces

These characters must be ignored. After extracting only the digits, the last 10 digits represent the local phone number, while any preceding digits represent the country code.

The masking rules preserve only the last four digits of the local number. Everything else is hidden with asterisks.

Examples:

```
1234567890
```

becomes:

```
***-***-7890
```

and:

```
+86(10)12345678
```

after digit extraction becomes:

```
861012345678
```

which contains a 2 digit country code:

```
+**-***-***-5678
```

The constraints are small. The maximum string length is at most 40 characters for emails and 20 characters for phone numbers. Because the input size is tiny, efficiency is not a major concern. However, the goal is still to design a clean and correct linear-time solution.

Several edge cases are important:

- Email names with only two characters
- Phone numbers with no country code
- Phone numbers with the maximum 3 digit country code
- Mixed uppercase and lowercase email characters
- Phone numbers containing many formatting symbols

A naive implementation could easily mishandle lowercase conversion, country code calculation, or formatting placement.

## Approaches

### Brute Force Approach

A brute force solution would manually inspect every character and repeatedly reconstruct the output string character by character using many conditionals.

For email addresses, the algorithm could:

1. Scan the string to find the `@`
2. Build a lowercase version manually
3. Append the first character
4. Append five asterisks
5. Append the last character
6. Append the domain

For phone numbers, the brute force approach could:

1. Iterate through the string
2. Store digits in an array
3. Count formatting characters separately
4. Rebuild the masked output one segment at a time

This works because the constraints are small and the formatting rules are deterministic. However, the implementation becomes unnecessarily verbose and repetitive. It also increases the chance of formatting mistakes.

### Optimal Approach

The key observation is that the input type can be identified immediately:

- If the string contains `'@'`, it is an email
- Otherwise, it is a phone number

Once we know the type, the masking logic becomes straightforward.

For emails:

- Convert the entire string to lowercase
- Split at `'@'`
- Keep only the first and last character of the name
- Insert `"*****"` between them

For phone numbers:

- Extract only digits
- Determine the country code length as `len(digits) - 10`
- Construct the prefix dynamically using repeated `'*'`
- Append the masked local number format

This produces a very clean linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Manual reconstruction with many conditionals |
| Optimal | O(n) | O(n) | Simple parsing and direct formatting |

## Algorithm Walkthrough

1. Check whether the input contains the `'@'` character.

If it does, the input is an email address. Otherwise, it is a phone number. This works because the problem guarantees valid input formats.
2. Handle the email case.

Convert the entire email to lowercase first. This ensures both the name and domain satisfy the lowercase requirement automatically.
3. Split the email into two parts using `'@'`.

The left side is the name, and the right side is the domain.
4. Construct the masked email.

Keep:

- the first character of the name
- five asterisks
- the last character of the name

Then append `'@'` and the lowercase domain.
5. Handle the phone number case.

Extract only numeric digits from the string. Ignore all formatting characters such as spaces, parentheses, dashes, and plus signs.
6. Determine the country code length.

The local number always contains exactly 10 digits. Any extra leading digits belong to the country code.

```
country_length = total_digits - 10
```
7. Construct the masked phone number.

The local portion always has the format:

```
***-***-XXXX
```

where `XXXX` is the last four digits.
8. Add the country code prefix if needed.

If the country code length is greater than zero:

```
+<stars>-
```

where the number of stars equals the country code length.
9. Return the final masked result.

### Why it works

The algorithm works because the problem defines deterministic formatting rules for both input types. Every valid email contains exactly one `@`, making identification unambiguous. Every valid phone number contains 10 to 13 digits, where the last 10 digits always represent the local number. By directly applying the masking specification after parsing the input, the algorithm guarantees the correct output format.

## Python Solution

```
class Solution:
    def maskPII(self, s: str) -> str:
        if '@' in s:
            s = s.lower()
            name, domain = s.split('@')
            
            return f"{name[0]}*****{name[-1]}@{domain}"
        
        digits = ''.join(ch for ch in s if ch.isdigit())
        
        local_mask = "***-***-" + digits[-4:]
        country_length = len(digits) - 10
        
        if country_length == 0:
            return local_mask
        
        return "+" + "*" * country_length + "-" + local_mask
```

The implementation first checks whether the input contains `'@'`. This immediately determines which masking logic to apply.

For emails, the entire string is converted to lowercase before splitting. This simplifies the implementation because both the name and domain are required to be lowercase in the final answer.

The name masking logic is straightforward. The first and last characters are preserved, while the middle portion is replaced with exactly five asterisks.

For phone numbers, the implementation extracts digits using a generator expression with `isdigit()`. This cleanly removes all formatting characters without requiring manual checks for each symbol type.

The local masked format is always:

```
***-***-XXXX
```

where `XXXX` represents the last four digits.

The country code prefix is constructed dynamically using string multiplication:

```
"*" * country_length
```

This automatically handles country codes of length 1, 2, or 3.

## Go Solution

```
package main

import (
	"strings"
	"unicode"
)

func maskPII(s string) string {
	if strings.Contains(s, "@") {
		s = strings.ToLower(s)

		parts := strings.Split(s, "@")
		name := parts[0]
		domain := parts[1]

		return string(name[0]) + "*****" + string(name[len(name)-1]) + "@" + domain
	}

	digits := ""

	for _, ch := range s {
		if unicode.IsDigit(ch) {
			digits += string(ch)
		}
	}

	localMask := "***-***-" + digits[len(digits)-4:]
	countryLength := len(digits) - 10

	if countryLength == 0 {
		return localMask
	}

	return "+" + strings.Repeat("*", countryLength) + "-" + localMask
}
```

The Go implementation follows the same algorithmic structure as the Python solution.

One important difference is string handling. In Go, strings are immutable, so concatenation creates new strings. Because the input size is tiny, this is perfectly acceptable here.

The implementation uses:

- `strings.Contains` to detect emails
- `strings.ToLower` for lowercase conversion
- `unicode.IsDigit` for digit extraction
- `strings.Repeat` to generate the country code mask

Unlike Python, Go requires explicit string conversion when accessing individual bytes:

```
string(name[0])
```

This is necessary because indexing a string returns a byte rather than a string.

## Edge Cases

One important edge case is an email name with only two characters. A naive implementation might incorrectly remove all middle characters and produce something like `"ab@qq.com"` instead of `"a*****b@qq.com"`. The problem explicitly requires exactly five asterisks regardless of name length. The implementation handles this correctly by always inserting `"*****"` between the first and last characters.

Another important edge case is phone numbers containing multiple formatting symbols. Inputs may include spaces, parentheses, dashes, and plus signs in arbitrary combinations. A buggy implementation might accidentally preserve some separators or miscount digits. The solution avoids this by extracting only numeric characters using `isdigit()` in Python or `unicode.IsDigit()` in Go.

A third important edge case is the maximum possible country code length of three digits. The formatting rules change depending on whether the country code length is 0, 1, 2, or 3. Hardcoding every format separately can easily introduce mistakes. The implementation handles this dynamically by repeating the `'*'` character according to:

```
country_length = len(digits) - 10
```

This guarantees the correct formatting for all valid phone number lengths.
