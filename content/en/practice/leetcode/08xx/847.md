## Solution

## Problem Understanding

This problem gives us an undirected, connected graph with `n` nodes numbered from `0` to `n - 1`. The graph is represented as an adjacency list, where `graph[i]` contains all nodes directly connected to node `i`.

We must compute the length of the shortest path that visits every node in the graph at least once.

Several details are extremely important:

- We may start from any node.
- We may stop at any node.
- We are allowed to revisit nodes.
- We are allowed to reuse edges.

The output is not the actual path itself, only the minimum number of edges required to visit all nodes.

The graph is guaranteed to be connected, so every node is reachable from every other node.

The constraints are small:

- `1 <= n <= 12`

This immediately suggests that exponential solutions involving subsets or bitmasks are feasible. Since `2^12 = 4096`, storing information for all subsets of nodes is practical.

The graph size is also small enough that combining graph traversal with dynamic programming over subsets becomes an ideal strategy.

A few edge cases are important:

- A graph with only one node should return `0`, because we already start on the only node.
- Star graphs can force revisiting the center many times.
- Dense cyclic graphs may have many equivalent shortest paths.
- Since revisiting is allowed, the shortest solution is not necessarily a Hamiltonian path.
- Starting from the wrong node can produce a longer path, so the algorithm must consider all possible starting points.

## Approaches

### Brute Force Approach

A brute force solution would attempt to explore every possible path through the graph while tracking which nodes have been visited.

We could use DFS and recursively try every neighbor from every current position. At each step we would maintain:

- The current node
- The set of visited nodes
- The current path length

Whenever all nodes become visited, we update the minimum answer.

This approach is correct because it explores all possible traversal sequences. Eventually it finds the shortest valid one.

However, the number of possible paths grows exponentially. Since nodes and edges may be revisited, the search tree becomes enormous. Even with pruning, the complexity becomes impractical.

The graph may contain cycles, which means the same states are repeatedly explored.

### Key Insight

The crucial observation is that the true state of the problem is not only the current node, but also the set of visited nodes.

If we are currently at node `u` and have visited subset `mask`, then any future decisions depend only on:

- `u`
- `mask`

The exact path taken to reach that state no longer matters.

This transforms the problem into a shortest path problem on an expanded state graph:

- State = `(node, visited_mask)`

Since we need the shortest number of edges, Breadth First Search is ideal.

Another key optimization is multi-source BFS:

- We can start BFS simultaneously from every node.
- Each starting node begins with its own visited mask.

This guarantees that the first time we reach a state where all nodes are visited, the distance is minimal.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Exponential, roughly O(n!) or worse | Exponential | Explores all traversal possibilities |
| Optimal BFS + Bitmask | O(n × 2^n) | O(n × 2^n) | Uses BFS over compressed graph states |

## Algorithm Walkthrough

### Optimal BFS with Bitmasking

1. Define the target mask.

Since there are `n` nodes, we use a bitmask of length `n`.

A bit is `1` if the corresponding node has been visited.

The mask representing all nodes visited is:

```
(1 << n) - 1
```

For example, if `n = 4`:

```
1111 binary = 15
```
2. Initialize the BFS queue with all starting nodes.

Since we may start anywhere, every node is a valid starting point.

For node `i`:

- Current node = `i`
- Visited mask = `1 << i`
- Distance = `0`

This is called multi-source BFS.
3. Maintain a visited state table.

We must avoid revisiting identical states.

A state is uniquely identified by:

```
(current_node, visited_mask)
```

We store these states in a set or boolean table.
4. Perform BFS level by level.

Remove a state from the queue.

For every neighbor:

- Move to that neighbor
- Update the visited mask
- Create the new state

Example:

```
new_mask = mask | (1 << neighbor)
```
5. Check whether all nodes are visited.

If:

```
new_mask == full_mask
```

then we have found the shortest valid traversal.

Because BFS explores states in increasing order of path length, the first solution encountered is optimal.
6. Continue until the answer is found.

Since the graph is connected, BFS is guaranteed to eventually reach the full mask.

### Why it works

The algorithm works because BFS always explores states in order of increasing distance.

Each state completely captures all relevant information:

- Which nodes have already been visited
- Where we currently are

If two paths reach the same `(node, mask)` state, only the shorter one matters. BFS guarantees the first occurrence is already optimal.

Therefore, once BFS first reaches a state where every node has been visited, that path length is the global minimum.

## Python Solution

```
from collections import deque
from typing import List

class Solution:
    def shortestPathLength(self, graph: List[List[int]]) -> int:
        n = len(graph)

        if n == 1:
            return 0

        full_mask = (1 << n) - 1

        queue = deque()
        visited = set()

        # Multi-source BFS initialization
        for node in range(n):
            mask = 1 << node
            queue.append((node, mask, 0))
            visited.add((node, mask))

        while queue:
            current_node, mask, distance = queue.popleft()

            for neighbor in graph[current_node]:
                new_mask = mask | (1 << neighbor)

                if new_mask == full_mask:
                    return distance + 1

                state = (neighbor, new_mask)

                if state not in visited:
                    visited.add(state)
                    queue.append((neighbor, new_mask, distance + 1))

        return -1
```

The implementation begins by handling the smallest edge case. If the graph contains only one node, the answer is immediately `0`.

Next, we compute `full_mask`, which represents the state where every node has been visited.

The BFS queue stores triples:

- Current node
- Visited mask
- Current distance

The algorithm uses multi-source BFS by inserting every node into the queue initially. Each node starts with only itself marked as visited.

The `visited` set prevents repeated exploration of identical states. This is essential because graphs may contain cycles.

Inside the BFS loop, we examine all neighbors of the current node. Moving to a neighbor updates the visited mask using bitwise OR.

If the updated mask equals the full mask, then all nodes have been visited, and we immediately return the distance plus one.

Otherwise, if the state has not been seen before, we enqueue it for future processing.

Because BFS explores shortest distances first, the first valid solution returned is optimal.

## Go Solution

```
package main

import "container/list"

func shortestPathLength(graph [][]int) int {
	n := len(graph)

	if n == 1 {
		return 0
	}

	fullMask := (1 << n) - 1

	type State struct {
		node     int
		mask     int
		distance int
	}

	queue := list.New()

	visited := make(map[[2]int]bool)

	// Multi-source BFS initialization
	for node := 0; node < n; node++ {
		mask := 1 << node

		queue.PushBack(State{
			node:     node,
			mask:     mask,
			distance: 0,
		})

		visited[[2]int{node, mask}] = true
	}

	for queue.Len() > 0 {
		front := queue.Front()
		queue.Remove(front)

		current := front.Value.(State)

		for _, neighbor := range graph[current.node] {
			newMask := current.mask | (1 << neighbor)

			if newMask == fullMask {
				return current.distance + 1
			}

			stateKey := [2]int{neighbor, newMask}

			if !visited[stateKey] {
				visited[stateKey] = true

				queue.PushBack(State{
					node:     neighbor,
					mask:     newMask,
					distance: current.distance + 1,
				})
			}
		}
	}

	return -1
}
```

The Go implementation follows the same logic as the Python version.

Go does not provide a built in deque structure similar to Python's `collections.deque`, so we use `container/list` as the BFS queue.

The visited states are stored in a map using a two element integer array as the key:

```
[2]int{node, mask}
```

This provides efficient lookup for previously explored states.

Since the constraints are small, integer overflow is not a concern. The largest mask is:

```
2^12 - 1 = 4095
```

which easily fits within Go's standard integer type.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × 2^n) | There are at most `n * 2^n` unique states |
| Space | O(n × 2^n) | BFS queue and visited set store graph states |

The complexity comes from the number of distinct states.

For each node, we may pair it with every possible visited subset.

Since there are:

```
2^n
```

possible masks and `n` nodes, the total number of states is:

```
n * 2^n
```

Each state is processed once, and each transition examines graph edges.

Given the constraint `n <= 12`, this complexity is entirely manageable.

## Edge Cases

### Single Node Graph

A graph containing only one node is the smallest valid input.

```
graph = [[]]
```

Since we may start at any node, we are already visiting every node immediately. The correct answer is `0`.

This case can easily cause bugs if the implementation assumes at least one edge exists. The solution handles it explicitly before starting BFS.

### Graphs That Require Revisiting Nodes

Consider a star graph:

```
0 connected to 1, 2, 3
```

To visit all leaf nodes, we must repeatedly return to the center.

A naive algorithm searching for a Hamiltonian path would fail because no such path exists without revisiting nodes.

The BFS state approach naturally handles revisits because states are based on `(node, mask)`, not on whether a node was visited previously in the path.

### Cyclic Graphs

Graphs containing cycles can cause infinite loops in naive traversal algorithms.

For example:

```
0 - 1 - 2 - 0
```

Without proper state tracking, BFS or DFS may repeatedly traverse the same cycle forever.

The visited state table prevents this by ensuring each `(node, mask)` combination is processed only once.

### Multiple Optimal Starting Points

The shortest traversal may depend heavily on the starting node.

Instead of trying every start separately, the algorithm uses multi-source BFS. All starting states are inserted simultaneously at distance `0`.

This guarantees that the globally optimal starting position is automatically considered.
