## Solution

## Problem Understanding

The problem asks us to generate every distinct subsequence of the input array such that the subsequence is non-decreasing and contains at least two elements.

A subsequence is formed by deleting zero or more elements from the array without changing the order of the remaining elements. This means we are not allowed to rearrange elements. For example, from `[4,6,7,7]`, the subsequence `[4,7,7]` is valid because the elements appear in the same order as the original array.

A non-decreasing subsequence means every next element must be greater than or equal to the previous element. Sequences like `[4,6,7]` and `[7,7]` are valid, while `[7,6]` is not.

The input consists of an integer array `nums`, where:

- `1 <= nums.length <= 15`
- `-100 <= nums[i] <= 100`

The small maximum length of 15 is extremely important. Since subsequences grow exponentially with array size, this constraint strongly suggests that an exponential solution such as backtracking or subset generation is acceptable.

The output should contain all distinct non-decreasing subsequences of length at least two. Duplicate subsequences must not appear in the result, even if they can be formed through different index selections.

The most important challenge is handling duplicates correctly. Consider the array `[4,6,7,7]`. Without careful duplicate handling, the subsequence `[4,7]` could be generated multiple times by choosing different occurrences of `7`.

Several edge cases are important:

- Arrays containing repeated numbers such as `[4,4,4]`
- Strictly decreasing arrays such as `[5,4,3,2]`
- Arrays where every element is identical
- Arrays containing negative numbers
- Arrays of length 1, where no valid subsequence exists

The problem guarantees the input size is small enough that exploring all possible subsequences is feasible.

## Approaches

### Brute Force Approach

The brute force approach is to generate every possible subsequence of the array and then filter the valid ones.

For an array of length `n`, there are `2^n` possible subsequences. We can generate each subsequence using bit masking or recursion. After generating a subsequence, we check:

1. Whether its length is at least 2
2. Whether it is non-decreasing

If both conditions are satisfied, we insert it into a set to avoid duplicates.

This approach is correct because it examines every possible subsequence, so no valid answer can be missed.

However, it is inefficient because many generated subsequences are obviously invalid. For example, if a partial subsequence already decreases, continuing to build it is pointless. The brute force solution still explores all possibilities anyway.

### Optimal Backtracking Approach

The better approach uses backtracking with pruning.

Instead of generating all subsequences blindly, we build subsequences incrementally while maintaining the non-decreasing property at every step.

At each recursive level:

- We only choose numbers greater than or equal to the last chosen number
- We use a local hash set to avoid generating duplicate subsequences from repeated values at the same recursion depth

This pruning significantly reduces unnecessary exploration because invalid decreasing paths are never explored.

The key insight is that once a subsequence violates the non-decreasing condition, no extension of it can ever become valid again. Therefore, we can stop exploring that branch immediately.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n × n) | O(2^n × n) | Generates every subsequence, then validates each one |
| Optimal Backtracking | O(2^n × n) | O(2^n × n) | Prunes invalid branches early and avoids duplicates during generation |

Although both approaches are exponential in the worst case, the backtracking solution is much more efficient in practice because it avoids exploring many invalid paths.

## Algorithm Walkthrough

### Optimal Backtracking Algorithm

1. Initialize an empty result list to store all valid subsequences.
2. Define a recursive backtracking function that takes:

- The current index in the array
- The current subsequence being built
3. At the start of each recursive call, if the current subsequence has length at least 2, add a copy of it to the result list.
4. Create a local hash set called `used` for the current recursion depth. This set prevents duplicate numbers from being chosen multiple times at the same level.
5. Iterate through the array starting from the current index.
6. For each number:

- Skip it if it has already been used at the current recursion depth.
- Skip it if it would violate the non-decreasing property.
7. If the number is valid:

- Add it to the current subsequence.
- Mark it as used in the current recursion level.
- Recursively continue building the subsequence from the next index.
- Remove the number afterward to backtrack and explore other possibilities.
8. Continue until all recursive branches have been explored.

### Why it works

The algorithm maintains an important invariant: every subsequence being constructed is always non-decreasing.

Because we only append values that are greater than or equal to the last chosen value, invalid subsequences are never created.

The local `used` set ensures that identical numbers at the same recursion depth are explored only once, which prevents duplicate subsequences from appearing in the output.

Since every valid non-decreasing subsequence can be formed through some recursive path, and duplicates are eliminated systematically, the algorithm generates exactly the required answer set.

## Python Solution

```
from typing import List

class Solution:
    def findSubsequences(self, nums: List[int]) -> List[List[int]]:
        result = []
        current = []

        def backtrack(start: int) -> None:
            if len(current) >= 2:
                result.append(current[:])

            used = set()

            for i in range(start, len(nums)):
                if nums[i] in used:
                    continue

                if current and nums[i] < current[-1]:
                    continue

                used.add(nums[i])
                current.append(nums[i])

                backtrack(i + 1)

                current.pop()

        backtrack(0)

        return result
```

The solution uses recursive backtracking to construct subsequences incrementally.

The `current` list stores the subsequence currently being built. The `result` list stores all valid subsequences discovered during recursion.

The recursive function `backtrack(start)` begins exploring choices from index `start`.

Whenever the current subsequence length becomes at least 2, a copy is appended to the result. A copy is necessary because lists are mutable and the subsequence will continue changing during backtracking.

The `used` set is recreated at every recursion depth. This is crucial because duplicate elimination only applies among sibling branches. For example, in `[7,7]`, we should explore only one branch starting with `7` at a particular depth.

The condition:

```
if current and nums[i] < current[-1]:
```

ensures the subsequence remains non-decreasing.

After adding an element, recursion continues from the next index. Once recursion returns, the element is removed to restore the previous state before exploring another branch.

## Go Solution

```
func findSubsequences(nums []int) [][]int {
	var result [][]int
	var current []int

	var backtrack func(start int)

	backtrack = func(start int) {
		if len(current) >= 2 {
			temp := make([]int, len(current))
			copy(temp, current)
			result = append(result, temp)
		}

		used := make(map[int]bool)

		for i := start; i < len(nums); i++ {
			if used[nums[i]] {
				continue
			}

			if len(current) > 0 && nums[i] < current[len(current)-1] {
				continue
			}

			used[nums[i]] = true
			current = append(current, nums[i])

			backtrack(i + 1)

			current = current[:len(current)-1]
		}
	}

	backtrack(0)

	return result
}
```

The Go implementation follows the same recursive structure as the Python version.

One important difference is slice handling. In Go, slices reference shared underlying arrays, so we must explicitly create a copy before appending a subsequence to `result`.

This is done using:

```
temp := make([]int, len(current))
copy(temp, current)
```

The duplicate tracking structure is implemented using `map[int]bool`, which serves the same role as Python's `set`.

Go slices naturally handle dynamic resizing, so backtracking is implemented by slicing off the last element after recursion returns.

## Edge Cases

### Arrays With All Duplicate Values

An input like `[7,7,7,7]` is a common source of duplicate-generation bugs. Without careful handling, the same subsequence could be generated many times through different index combinations.

The implementation avoids this by using a local `used` set at each recursion depth. This ensures that once a value has been explored at a particular level, identical values at the same level are skipped.

### Strictly Decreasing Arrays

An array such as `[5,4,3,2,1]` contains no valid non-decreasing subsequences of length at least two.

A naive implementation might still generate many invalid paths before filtering them later. The backtracking solution avoids this inefficiency because it immediately skips any candidate smaller than the last chosen element.

As a result, no invalid subsequences are ever constructed.

### Arrays of Length One

If the input contains only one element, such as `[1]`, it is impossible to form a subsequence of length at least two.

The algorithm handles this naturally. The recursive search runs normally, but the condition:

```
if len(current) >= 2:
```

is never satisfied, so the result remains empty.

### Negative Numbers

The problem allows values between `-100` and `100`, which includes negative numbers.

The implementation does not rely on positivity or sorting assumptions. Comparisons are purely relational using `<` and `>=`, so negative values work exactly the same as positive values.
