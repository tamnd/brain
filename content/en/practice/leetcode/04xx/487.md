## Solution

## Problem Understanding

The problem gives us a binary array, meaning every element is either `0` or `1`. We want to determine the maximum number of consecutive `1`s that can appear if we are allowed to flip at most one `0` into a `1`.

The phrase "at most one" is important. It means we may choose to flip one zero, or we may choose not to flip anything if the array already contains all ones.

For example, consider:

```
[1,0,1,1,0]
```

If we flip the first `0`, the array effectively becomes:

```
[1,1,1,1,0]
```

This creates a consecutive sequence of four `1`s, which is the best possible answer.

The input size can be as large as `10^5`, which immediately tells us that quadratic solutions will likely be too slow. Any approach that repeatedly scans large portions of the array for every index will not scale efficiently. This strongly suggests that we should look for a linear time solution.

Several edge cases are important to think about before designing the algorithm.

If the array contains all `1`s, then the answer is simply the entire array length because no flip is needed.

If the array contains exactly one `0`, flipping it allows the whole array to become consecutive `1`s.

If the array contains many zeros close together, we must ensure we never count a window that requires flipping more than one zero.

Arrays of length `1` are also important because the smallest possible input may be either `[0]` or `[1]`.

## Approaches

### Brute Force Approach

A straightforward approach is to try flipping every zero individually and compute the longest consecutive sequence of ones after each flip.

For each index:

- If the element is `0`, temporarily flip it to `1`
- Scan the entire array to find the longest streak of consecutive `1`s
- Restore the original value
- Track the maximum result

We also need to consider the case where we do not flip anything.

This approach is correct because it explicitly tests every possible valid flip and computes the resulting maximum streak. Since one of those possibilities must be optimal, the algorithm eventually finds the correct answer.

However, this solution is inefficient. If the array has length `n`, then for each position we may scan the entire array again. This produces a time complexity of `O(n^2)`, which is too slow for `n = 10^5`.

### Optimal Sliding Window Approach

The key observation is that we do not actually need to try every flip independently.

Instead, we can think of the problem differently:

We want the longest contiguous subarray containing at most one zero.

Why does this work?

Because if a window contains at most one zero, then we can flip that zero and make the entire window all ones.

This transforms the problem into a classic sliding window problem.

We maintain a window using two pointers:

- `left` marks the beginning
- `right` expands the window

As we move `right` through the array:

- We count how many zeros exist inside the current window
- If the window contains more than one zero, we shrink it from the left until it becomes valid again

At every step, the window represents a valid segment where at most one zero would need to be flipped. Therefore, its length is a candidate answer.

This gives us a linear time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Try flipping every zero and rescan the array |
| Optimal | O(n) | O(1) | Sliding window maintaining at most one zero |

## Algorithm Walkthrough

1. Initialize two pointers, `left` and `right`, both starting at `0`. These pointers define the current sliding window.
2. Maintain a variable called `zero_count` to track how many zeros exist inside the current window. Initially, this value is `0`.
3. Maintain another variable called `max_length` to store the best valid window length seen so far.
4. Iterate through the array using `right` as the expanding pointer.
5. Whenever `nums[right]` is `0`, increment `zero_count` because the current window now contains another zero.
6. If `zero_count` becomes greater than `1`, the current window is invalid because we would need to flip more than one zero to make all elements equal to `1`.
7. To restore validity, repeatedly move `left` forward:

- If `nums[left]` is `0`, decrement `zero_count`
- Increment `left`

Continue shrinking until the window contains at most one zero again.
8. Once the window is valid, compute its length:

```
right - left + 1
```
9. Update `max_length` if the current valid window is larger than any previously seen window.
10. Continue until `right` reaches the end of the array.
11. Return `max_length`.

### Why it works

The sliding window always maintains the invariant that the current window contains at most one zero. Any such window can be transformed into all ones by flipping that single zero.

Because the algorithm examines every possible valid window exactly once, and because the window expands and contracts in a controlled manner, the maximum valid window length found during the process must be the optimal answer.

## Python Solution

```
from typing import List

class Solution:
    def findMaxConsecutiveOnes(self, nums: List[int]) -> int:
        left = 0
        zero_count = 0
        max_length = 0

        for right in range(len(nums)):
            if nums[right] == 0:
                zero_count += 1

            while zero_count > 1:
                if nums[left] == 0:
                    zero_count -= 1
                left += 1

            current_length = right - left + 1
            max_length = max(max_length, current_length)

        return max_length
```

The implementation follows the sliding window algorithm directly.

The variable `left` represents the start of the current window, while the loop variable `right` expands the window one element at a time.

Whenever we encounter a zero at the `right` pointer, we increment `zero_count`. This tells us how many flips would be required to convert the current window into all ones.

If the number of zeros exceeds one, the window becomes invalid. The `while` loop shrinks the window from the left side until only one zero remains.

After restoring validity, the current window length is computed and compared against `max_length`.

Since each pointer moves at most `n` times, the solution runs efficiently in linear time.

## Go Solution

```
func findMaxConsecutiveOnes(nums []int) int {
    left := 0
    zeroCount := 0
    maxLength := 0

    for right := 0; right < len(nums); right++ {
        if nums[right] == 0 {
            zeroCount++
        }

        for zeroCount > 1 {
            if nums[left] == 0 {
                zeroCount--
            }
            left++
        }

        currentLength := right - left + 1

        if currentLength > maxLength {
            maxLength = currentLength
        }
    }

    return maxLength
}
```

The Go implementation mirrors the Python logic closely.

Go uses explicit variable declarations and standard `for` loops instead of Python's `range`.

There are no special concerns about integer overflow because the maximum array length is only `10^5`, well within Go's integer range.

Slices in Go behave similarly to arrays for indexed access, so no additional handling is required.

## Edge Cases

One important edge case is an array containing all ones, such as:

```
[1,1,1,1]
```

A buggy implementation might incorrectly assume exactly one zero must be flipped. However, the problem allows flipping at most one zero. The sliding window naturally handles this because the window never becomes invalid, so the entire array length is returned.

Another important edge case is an array containing only zeros:

```
[0,0,0]
```

Even though no consecutive ones initially exist, we may flip one zero into a one. The maximum possible answer is therefore `1`. The sliding window correctly limits the window to at most one zero, preventing invalid larger windows.

A third important edge case involves consecutive zeros:

```
[1,1,0,0,1]
```

This case can easily break incorrect implementations because a valid window cannot contain both zeros simultaneously. The `while zero_count > 1` loop ensures the window shrinks immediately after the second zero enters, preserving the invariant that at most one zero exists inside the window.

Another subtle edge case is a single-element array:

```
[0]
```

The correct answer is `1` because we may flip the zero. The algorithm handles this naturally because the single-element window is always valid and its length becomes the answer.
