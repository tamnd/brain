## Solution

## Problem Understanding

The problem defines a special sequence called the magical string. This string contains only the characters `'1'` and `'2'`, and its defining property is that the lengths of its consecutive groups reproduce the string itself.

For example, the beginning of the magical string looks like this:

```
1221121221221121122...
```

If we split it into consecutive groups of identical characters, we get:

```
1 | 22 | 11 | 2 | 1 | 22 | 1 | 22 | 11 | 2 ...
```

Now count the size of each group:

```
1, 2, 2, 1, 1, 2, 1, 2, 2, 1 ...
```

That sequence is exactly the same as the original magical string.

The input `n` represents how many characters from the beginning of the magical string we should examine. The goal is to return how many `'1'` characters appear within the first `n` positions.

The constraints are important:

- `1 <= n <= 10^5`

This means the solution must efficiently generate the magical string up to length `n`. An exponential or quadratic solution would be risky at this scale. Since `10^5` is relatively moderate, a linear time solution is ideal.

There are several edge cases worth identifying immediately.

When `n = 1`, the answer must be `1` because the string begins with `"1"`.

Another tricky case occurs near the boundary where the generated string slightly exceeds length `n`. The algorithm must avoid counting extra characters beyond the first `n`.

A naive implementation can also accidentally generate incorrect alternating groups because the magical string is self-referential. Careful pointer management is required to ensure the generated counts and appended digits stay synchronized.

## Approaches

### Brute Force Approach

A brute force strategy would attempt to explicitly simulate the magical string construction in the most direct way possible.

We could repeatedly scan the string, compute consecutive group lengths, and verify or reconstruct the sequence character by character. Each time a new segment is needed, we would recompute grouping information from earlier parts of the string.

This approach is correct because it follows the definition literally. However, repeatedly scanning previously generated sections introduces significant redundant work.

If each extension of the string requires rescanning a growing prefix, the total work becomes quadratic in the worst case.

Since `n` can be as large as `100000`, an `O(n^2)` solution is unnecessarily expensive and may struggle under time limits.

### Optimal Approach

The key observation is that the magical string generates itself incrementally.

Instead of repeatedly recomputing group lengths, we can maintain:

- A growing array containing the magical string
- A pointer indicating which value tells us the size of the next group
- A current digit to append, alternating between `1` and `2`

The magical string starts with:

```
1 2 2
```

From this point onward:

- Each existing digit tells us how many times the next digit should appear
- After appending a group, we switch between `1` and `2`

This creates a natural two-pointer style process:

- One pointer reads group lengths
- One pointer extends the sequence

Because every character is generated exactly once, the algorithm runs in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly rescans and recomputes group lengths |
| Optimal | O(n) | O(n) | Generates the magical string incrementally using pointers |

## Algorithm Walkthrough

1. Handle the smallest edge case immediately.

If `n == 1`, return `1` because the magical string begins with `"1"`.
2. Initialize the magical string with the known starting pattern.

The magical string always begins as:

```
[1, 2, 2]
```

This seed is enough to generate the rest of the sequence.
3. Create a pointer called `head`.

This pointer reads values from the magical string itself. Each value indicates how many times the next digit should be appended.

Initially:

```
head = 2
```

because the first unread group size is the third element.
4. Track the next number to append.

After the initial sequence `[1,2,2]`, the next group should contain `1`s.

So initialize:

```
next_num = 1
```
5. Maintain a running count of how many `1`s appear.

Since the initial sequence `[1,2,2]` already contains one `1`, initialize:

```
ones_count = 1
```
6. Continue generating the magical string until its length reaches at least `n`.

At each iteration:

- Read the group size from `magical[head]`
- Append `next_num` that many times
- Count newly added `1`s if they fall within the first `n` positions
- Flip `next_num` between `1` and `2`
- Advance `head`
7. Alternating digits is easy.

Since the sequence contains only `1` and `2`, we can switch using:

```
next_num = 3 - next_num
```

If `next_num` is `1`, it becomes `2`.

If it is `2`, it becomes `1`.
8. Return the final count of `1`s.

### Why it works

The magical string is self-describing. Every digit in the sequence specifies the size of a future group. By maintaining a reading pointer into the already generated sequence, we always know exactly how many copies of the next digit to append.

The invariant is:

- The prefix already generated is always a valid magical string.
- `head` always points to the next unread group length.
- `next_num` always represents the correct digit for the next group.

Since each step preserves these properties, the algorithm correctly constructs the magical string and counts the number of `1`s in the first `n` characters.

## Python Solution

```
class Solution:
    def magicalString(self, n: int) -> int:
        if n == 1:
            return 1

        magical = [1, 2, 2]

        head = 2
        next_num = 1
        ones_count = 1

        while len(magical) < n:
            group_size = magical[head]

            for _ in range(group_size):
                magical.append(next_num)

                if next_num == 1 and len(magical) <= n:
                    ones_count += 1

            next_num = 3 - next_num
            head += 1

        return ones_count
```

The implementation begins by handling the smallest possible input. This avoids unnecessary processing and guarantees correctness for the base case.

The list `magical` stores the generated sequence. The initial seed `[1, 2, 2]` is critical because it already satisfies the magical property and provides enough information to continue generation.

The variable `head` acts as the reading pointer. Instead of recomputing group sizes, we directly read them from the sequence itself.

The variable `next_num` tracks which digit should be appended next. Because the sequence alternates groups of `1`s and `2`s, we can efficiently toggle using `3 - next_num`.

Inside the loop, the algorithm appends the correct number of digits according to the current group size. Each appended `1` contributes to the running count, but only if it belongs within the first `n` positions.

Finally, the function returns the accumulated count.

## Go Solution

```
func magicalString(n int) int {
    if n == 1 {
        return 1
    }

    magical := []int{1, 2, 2}

    head := 2
    nextNum := 1
    onesCount := 1

    for len(magical) < n {
        groupSize := magical[head]

        for i := 0; i < groupSize; i++ {
            magical = append(magical, nextNum)

            if nextNum == 1 && len(magical) <= n {
                onesCount++
            }
        }

        nextNum = 3 - nextNum
        head++
    }

    return onesCount
}
```

The Go implementation follows the same logic as the Python version, but uses slices instead of Python lists.

One important detail is slice growth. The `append` function automatically resizes the slice as needed.

Go integers are fully sufficient for this problem because the maximum value involved is only `10^5`.

Unlike Python, Go does not provide implicit dynamic typing, so all variables use explicit integer types.

## Edge Cases

One important edge case is `n = 1`. Many implementations initialize the magical string as `[1,2,2]`, which already exceeds the requested length. If the algorithm does not special-case this input, it may incorrectly return the count for the larger prefix instead of only the first character. The implementation handles this immediately by returning `1`.

Another subtle case occurs when the generated string exceeds length `n` during the final append operation. For example, if the current group size is `2` but only one more character belongs within the first `n` positions, counting both characters would produce an incorrect answer. The implementation avoids this by only increasing `ones_count` when `len(magical) <= n`.

A third common source of bugs is incorrect alternation between `1` and `2`. Since the magical string is self-referential, a single wrong digit corrupts the rest of the sequence. The implementation avoids complicated conditionals by using `next_num = 3 - next_num`, which guarantees perfect alternation between `1` and `2`.
