## Solution

## Problem Understanding

The problem asks us to count the number of **reverse pairs** in an array of integers `nums`. A reverse pair `(i, j)` is defined such that `i < j` and `nums[i] > 2 * nums[j]`. Essentially, for each element in the array, we are looking for all elements to its right that are less than half of its value.

The input is an integer array with length up to 50,000 and values ranging from `-2^31` to `2^31 - 1`. The output is a single integer representing the total number of reverse pairs. Given the size of the array, any naive O(n²) solution will likely be too slow, which signals the need for a more advanced algorithm.

Important edge cases include arrays with all equal elements, arrays sorted in ascending or descending order, arrays with negative numbers, and arrays with only one element. The problem guarantees that the array has at least one element and that integer multiplication will not overflow standard 32-bit arithmetic if implemented carefully.

## Approaches

The **brute-force approach** iterates through all possible pairs `(i, j)` with nested loops, checking the condition `nums[i] > 2 * nums[j]`. This approach is correct because it checks every valid pair explicitly. However, its time complexity is O(n²), which is prohibitively slow for arrays of length 50,000, as it could result in over 2.5 billion comparisons in the worst case.

The **optimal approach** leverages a modified **merge sort** algorithm. The key insight is that when the array is divided into two halves, all elements in the left half are before the right half in the original array. By sorting each half recursively and counting reverse pairs during the merge step, we can efficiently determine the number of valid pairs without checking each one individually. This works because if both halves are sorted, we can use a two-pointer approach to count the number of elements in the right half that satisfy `nums[i] > 2 * nums[j]` for each `nums[i]` in the left half.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Nested loops checking each pair |
| Merge Sort | O(n log n) | O(n) | Divide-and-conquer counting during merge |

## Algorithm Walkthrough

1. **Divide** the array into two halves recursively until each half contains one element. Single elements are trivially sorted.
2. **Conquer** by merging the halves back together in a sorted manner. Before merging, iterate through each element in the left half and count how many elements in the right half satisfy the reverse pair condition `nums[i] > 2 * nums[j]`.
3. **Merge** the two sorted halves into one sorted array. This ensures that each merge step works on sorted subarrays, which is critical for counting efficiently.
4. **Aggregate** the counts from each recursive merge step. The sum of these counts gives the total number of reverse pairs.

**Why it works:** The algorithm guarantees correctness because each reverse pair involves one element in the left half and one element in the right half during each merge step. By counting during the merge and recursively applying this to all subarrays, we ensure that every valid pair is counted exactly once. Sorting ensures that counting can be done in linear time for each merge step.

## Python Solution

```
from typing import List

class Solution:
    def reversePairs(self, nums: List[int]) -> int:
        def merge_sort(start: int, end: int) -> int:
            if start >= end:
                return 0
            
            mid = (start + end) // 2
            count = merge_sort(start, mid) + merge_sort(mid + 1, end)
            
            # Count reverse pairs
            j = mid + 1
            for i in range(start, mid + 1):
                while j <= end and nums[i] > 2 * nums[j]:
                    j += 1
                count += j - (mid + 1)
            
            # Merge step
            temp = []
            left, right = start, mid + 1
            while left <= mid and right <= end:
                if nums[left] <= nums[right]:
                    temp.append(nums[left])
                    left += 1
                else:
                    temp.append(nums[right])
                    right += 1
            while left <= mid:
                temp.append(nums[left])
                left += 1
            while right <= end:
                temp.append(nums[right])
                right += 1
            
            for i in range(len(temp)):
                nums[start + i] = temp[i]
            
            return count
        
        return merge_sort(0, len(nums) - 1)
```

The Python implementation defines a recursive `merge_sort` function that counts reverse pairs while sorting subarrays. The inner loop `while j <= end and nums[i] > 2 * nums[j]: j += 1` efficiently counts all reverse pairs for a given `i`. The merge step combines the sorted halves into one sorted array, ensuring correctness for subsequent counts.

## Go Solution

```
func reversePairs(nums []int) int {
    var mergeSort func(start, end int) int
    mergeSort = func(start, end int) int {
        if start >= end {
            return 0
        }
        mid := (start + end) / 2
        count := mergeSort(start, mid) + mergeSort(mid+1, end)
        
        // Count reverse pairs
        j := mid + 1
        for i := start; i <= mid; i++ {
            for j <= end && nums[i] > 2*nums[j] {
                j++
            }
            count += j - (mid + 1)
        }
        
        // Merge step
        temp := make([]int, 0, end-start+1)
        left, right := start, mid+1
        for left <= mid && right <= end {
            if nums[left] <= nums[right] {
                temp = append(temp, nums[left])
                left++
            } else {
                temp = append(temp, nums[right])
                right++
            }
        }
        for left <= mid {
            temp = append(temp, nums[left])
            left++
        }
        for right <= end {
            temp = append(temp, nums[right])
            right++
        }
        copy(nums[start:end+1], temp)
        return count
    }
    
    return mergeSort(0, len(nums)-1)
}
```

In Go, the algorithm is almost identical to Python. Go requires explicit slice management and the `copy` function to update the original slice. Overflow is avoided by using `int64` if necessary, though Go `int` is typically sufficient for this problem size.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n log n) | Each merge step counts in O(n) and there are log n levels of recursion |
| Space | O(n) | Temporary array is created during each merge step, proportional to the array size |

The divide-and-conquer approach efficiently reduces the quadratic brute-force complexity to O(n log n) by leveraging sorted subarrays.
