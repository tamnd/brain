## Solution

## Problem Understanding

This problem asks us to traverse a given `m x n` matrix in a diagonal zigzag order and return all elements in that specific order as a flat array. In other words, we do not simply traverse rows or columns sequentially. Instead, we start from the top-left element and move diagonally upward-right, then diagonally downward-left, alternating directions whenever we reach the matrix boundaries.

The input, `mat`, is a 2D list (or slice in Go) of integers with dimensions `m` rows and `n` columns. Each element can be any integer within the range `-10^5` to `10^5`. The output is a single 1D array of length `m * n`, containing all the matrix elements in the required diagonal order.

Key constraints and implications are:

1. `1 <= m, n <= 10^4` and `1 <= m * n <= 10^4`, which implies that although one dimension can be large, the total number of elements is capped at 10,000, so an `O(m * n)` solution is feasible.
2. Negative and positive numbers are allowed, so our solution must handle general integers.

Important edge cases include:

- A matrix with a single row or a single column. This can test whether boundary conditions are handled correctly.
- A matrix where `m` or `n` is `1`. This reduces diagonal movement to simple linear traversal.
- The smallest matrix of size `1x1`. The function must return that single element.

## Approaches

The brute-force approach would attempt to simulate every diagonal by collecting elements into temporary lists and reversing every other diagonal before appending to the result. While correct, this adds unnecessary overhead due to the use of extra storage and repeated list reversals, though it is acceptable for small matrices.

The optimal approach leverages the observation that the sum of the row and column indices of a matrix element uniquely identifies its diagonal. Specifically, elements with the same `i + j` value belong to the same diagonal. We can iterate through the matrix once, appending elements to the appropriate diagonal container. Then, for diagonals with even `i + j` sums, we reverse the elements to maintain the upward-right zigzag, while for odd sums, we leave them in place. This approach avoids nested traversals or repeated reversals, giving a clean linear solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m * n) | O(m * n) | Use temporary lists for each diagonal and reverse alternate diagonals before appending |
| Optimal | O(m * n) | O(1) extra (excluding output) | Traverse the matrix once, directly placing elements into the output array, switching directions on boundaries |

## Algorithm Walkthrough

1. Initialize `m` and `n` as the dimensions of the matrix. Create a result list `res` of length `m * n`. Set starting indices `row = 0` and `col = 0`. Use a boolean `going_up = True` to track traversal direction.
2. Iterate `k` from `0` to `m * n - 1` to fill each element in `res`. Assign `res[k] = mat[row][col]`.
3. If `going_up` is `True`, attempt to move diagonally upward-right: decrement `row` by 1, increment `col` by 1. If the next position exceeds the matrix boundaries, adjust `row` and `col` to the correct boundary position and switch `going_up` to `False`.
4. If `going_up` is `False`, move diagonally downward-left: increment `row` by 1, decrement `col` by 1. If the next position exceeds boundaries, adjust `row` and `col` appropriately and switch `going_up` to `True`.
5. Continue until all elements are added to `res`.

Why it works: At each step, the algorithm ensures that either `row` or `col` stays within valid bounds, while alternating the direction on each diagonal boundary. This guarantees that each element is visited exactly once and in the required zigzag order.

## Python Solution

```
from typing import List

class Solution:
    def findDiagonalOrder(self, mat: List[List[int]]) -> List[int]:
        if not mat or not mat[0]:
            return []

        m, n = len(mat), len(mat[0])
        res = [0] * (m * n)
        row, col = 0, 0
        going_up = True

        for k in range(m * n):
            res[k] = mat[row][col]

            if going_up:
                if col == n - 1:
                    row += 1
                    going_up = False
                elif row == 0:
                    col += 1
                    going_up = False
                else:
                    row -= 1
                    col += 1
            else:
                if row == m - 1:
                    col += 1
                    going_up = True
                elif col == 0:
                    row += 1
                    going_up = True
                else:
                    row += 1
                    col -= 1

        return res
```

The Python implementation follows the algorithm exactly. It initializes the result array and iterates over each element while adjusting the direction based on matrix boundaries. Special conditions handle when the traversal reaches the first row, last row, first column, or last column.

## Go Solution

```
func findDiagonalOrder(mat [][]int) []int {
    if len(mat) == 0 || len(mat[0]) == 0 {
        return []int{}
    }

    m, n := len(mat), len(mat[0])
    res := make([]int, m*n)
    row, col := 0, 0
    goingUp := true

    for k := 0; k < m*n; k++ {
        res[k] = mat[row][col]

        if goingUp {
            if col == n-1 {
                row++
                goingUp = false
            } else if row == 0 {
                col++
                goingUp = false
            } else {
                row--
                col++
            }
        } else {
            if row == m-1 {
                col++
                goingUp = true
            } else if col == 0 {
                row++
                goingUp = true
            } else {
                row++
                col--
            }
        }
    }

    return res
}
```

In Go, slices are initialized with `make`, and boolean variables use camelCase by convention. Edge cases are handled identically to Python, and integer overflow is not a concern since `m * n <= 10^4`.
