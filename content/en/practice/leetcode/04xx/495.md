## Solution

## Problem Understanding

The problem describes a sequence of poison attacks applied by Teemo to Ashe. Every attack starts a poison effect that lasts for exactly `duration` seconds. If a new attack happens while Ashe is still poisoned, the poison timer is reset instead of stacking independently.

More formally, an attack at time `t` poisons Ashe during the inclusive interval:

$$[t, t + duration - 1]$$

The input array `timeSeries` contains all attack times in non-decreasing order. The task is to compute the total number of seconds during which Ashe is poisoned.

The important detail is that overlapping poison intervals must not be counted twice. If one attack happens before the previous poison expires, the two intervals merge into one continuous poisoned range.

For example, if `duration = 2` and attacks occur at times `1` and `2`, then:

- Attack at `1` poisons during `[1, 2]`
- Attack at `2` poisons during `[2, 3]`

The union of these intervals is `[1, 3]`, which contains 3 seconds total, not 4.

The constraints are relatively small:

- `timeSeries.length <= 10^4`
- Attack times and duration can be as large as `10^7`
- The array is already sorted in non-decreasing order

These constraints suggest that an efficient linear scan is more than sufficient. Since the input is already sorted, we can process attacks in chronological order without additional sorting.

Several edge cases are important:

- A single attack
- Multiple attacks with no overlap
- Multiple attacks with complete overlap
- `duration = 0`
- Duplicate attack timestamps
- Consecutive attacks where one starts exactly when another begins to expire

The sorted input guarantee is especially useful because it allows us to compare only neighboring attacks to determine overlap.

## Approaches

### Brute Force Approach

A straightforward solution is to explicitly track every second during which Ashe is poisoned.

For each attack time `t`, we could iterate through all seconds from `t` to `t + duration - 1` and insert those seconds into a set. Since sets automatically avoid duplicates, overlapping poison durations would only be counted once.

This approach is correct because every poisoned second is represented explicitly, and overlapping intervals naturally collapse into a single entry in the set.

However, this method is inefficient when `duration` is large. If `duration = 10^7`, iterating through every poisoned second becomes prohibitively expensive. Even though the number of attacks is only `10^4`, the total number of processed seconds could become enormous.

### Optimal Approach

The key observation is that we do not need to track every poisoned second individually. Instead, we only need to determine how much new poison time each attack contributes.

Consider two consecutive attacks:

- Previous attack at `timeSeries[i - 1]`
- Current attack at `timeSeries[i]`

If the gap between them is greater than or equal to `duration`, then the previous poison effect fully expires before the next attack starts. In this case, the previous attack contributes the full `duration`.

If the gap is smaller than `duration`, then the intervals overlap. The previous attack contributes only the gap length because the remaining poisoned seconds are covered again by the next attack.

Mathematically:

$$\text{contribution} = \min(duration, current - previous)$$

After processing all adjacent pairs, we add the full `duration` for the final attack because nothing comes after it to overlap with it.

This transforms the problem into a simple linear scan.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × duration) | O(n × duration) | Explicitly stores every poisoned second |
| Optimal | O(n) | O(1) | Adds only newly contributed poison time |

## Algorithm Walkthrough

1. Initialize a variable `total_poisoned_time` to `0`.

This variable will accumulate the total number of poisoned seconds.
2. Iterate through all attack times except the last one.

We compare each attack with the next attack to determine whether their poison durations overlap.
3. Compute the gap between consecutive attacks.

Let:

$$gap = timeSeries[i + 1] - timeSeries[i]$$

This tells us how many seconds pass before the next attack occurs.
4. Add the effective contribution of the current attack.

The current attack contributes:

$$\min(duration, gap)$$

If the next attack happens after the poison expires, we add the full duration.

If the next attack happens before expiration, we add only the non-overlapping portion.
5. After the loop finishes, add the full `duration` for the last attack.

The final attack is never truncated by a future attack, so it always contributes its entire duration.
6. Return the accumulated total.

### Why it works

The algorithm works because every attack interval overlaps only with future intervals that begin before it expires. By processing attacks in chronological order and adding only the newly contributed time for each interval, we avoid double-counting overlapping sections.

For each attack except the last:

- Either it contributes its full duration
- Or it contributes only up to the next attack

This guarantees that every poisoned second is counted exactly once.

## Python Solution

```
from typing import List

class Solution:
    def findPoisonedDuration(self, timeSeries: List[int], duration: int) -> int:
        if not timeSeries or duration == 0:
            return 0

        total_poisoned_time = 0

        for i in range(len(timeSeries) - 1):
            current_attack = timeSeries[i]
            next_attack = timeSeries[i + 1]

            gap = next_attack - current_attack

            total_poisoned_time += min(duration, gap)

        total_poisoned_time += duration

        return total_poisoned_time
```

The implementation begins by handling edge cases. If the attack list is empty or the poison duration is zero, Ashe is never poisoned, so the answer is `0`.

The variable `total_poisoned_time` stores the running total.

The loop processes every attack except the final one. For each attack, the code computes the gap to the next attack. The contribution from the current attack is the smaller of:

- The full poison duration
- The time until the next attack

This exactly matches the overlap logic discussed earlier.

After the loop, the final attack contributes its full duration because there is no future attack to shorten it.

The algorithm uses only constant extra space and scans the array once.

## Go Solution

```
func findPoisonedDuration(timeSeries []int, duration int) int {
    if len(timeSeries) == 0 || duration == 0 {
        return 0
    }

    totalPoisonedTime := 0

    for i := 0; i < len(timeSeries)-1; i++ {
        gap := timeSeries[i+1] - timeSeries[i]

        if gap < duration {
            totalPoisonedTime += gap
        } else {
            totalPoisonedTime += duration
        }
    }

    totalPoisonedTime += duration

    return totalPoisonedTime
}
```

The Go implementation follows the same logic as the Python version.

Go does not provide a built-in `min` function for integers in older versions, so the comparison is implemented using an `if` statement.

Slices in Go naturally handle dynamic arrays, so no special handling is required beyond checking `len(timeSeries) == 0`.

The integer constraints are well within Go's standard `int` range, so overflow is not a concern here.

## Edge Cases

### Zero Duration

If `duration = 0`, then every attack poisons Ashe for zero seconds. A careless implementation might still add poison time for each attack. The solution explicitly checks for this case at the beginning and immediately returns `0`.

### Duplicate Attack Times

The input is non-decreasing, not strictly increasing. This means multiple attacks can occur at the same timestamp.

For example:

```
timeSeries = [1,1,1]
duration = 2
```

Each new attack resets the poison timer immediately, but no extra time is added before the reset. The algorithm handles this correctly because the gap between attacks is `0`, so:

```
min(duration, 0) = 0
```

Only the final duration contributes to the total.

### Fully Overlapping Intervals

Consider:

```
timeSeries = [1,2,3]
duration = 5
```

Each attack happens before the previous poison expires. A naive implementation might incorrectly add all durations independently and produce `15`.

The correct poisoned interval is:

```
[1,7]
```

which contains only 7 seconds.

The algorithm avoids double-counting by adding only the gap between attacks whenever overlap occurs.
