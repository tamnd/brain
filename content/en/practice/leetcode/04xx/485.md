## Solution

## Problem Understanding

The problem gives us a binary array named `nums`, meaning every element in the array is either `0` or `1`. Our goal is to determine the maximum number of consecutive `1`s that appear in the array.

In simpler terms, we want to scan through the array and find the longest uninterrupted sequence of `1`s. Whenever we encounter a `0`, the current sequence ends, and we begin counting again from the next position.

For example, in the array `[1,1,0,1,1,1]`, there are two groups of consecutive `1`s:

- The first group is `[1,1]`, which has length `2`
- The second group is `[1,1,1]`, which has length `3`

Since `3` is the larger value, the correct answer is `3`.

The constraints tell us that the array length can be as large as `10^5`. This is important because it means we should avoid unnecessarily expensive algorithms such as checking every possible subarray. A linear time solution is ideal here.

The problem also guarantees that every value is either `0` or `1`, which simplifies the logic significantly. We never need to handle other integers or invalid input values.

Several edge cases are important to think about before implementing the solution. An array containing only `1`s should return the full length of the array. An array containing only `0`s should return `0`. Arrays with alternating values such as `[1,0,1,0,1]` test whether the counter resets correctly after each `0`. Very small arrays of length `1` must also be handled correctly.

## Approaches

### Brute Force Approach

A brute force solution would examine every possible subarray and determine whether that subarray consists entirely of `1`s. For each starting index, we could expand forward until we encounter a `0`, tracking the maximum length found.

This approach works because it explicitly checks all possible consecutive segments of `1`s. However, it is inefficient because many overlapping segments are repeatedly examined.

In the worst case, such as an array containing all `1`s, the algorithm would repeatedly scan large portions of the array, resulting in quadratic time complexity.

Given that the array can contain up to `100000` elements, an `O(n^2)` solution is too slow.

### Optimal Approach

The key observation is that we do not need to examine every possible subarray. We only need to keep track of two values while scanning the array once:

- The current streak of consecutive `1`s
- The maximum streak seen so far

As we iterate through the array:

- If the current number is `1`, we increase the current streak
- If the current number is `0`, the streak is broken, so we reset the current streak to `0`
- After each step, we update the maximum streak if necessary

This works because consecutive sequences are naturally processed in order during a single traversal.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every possible consecutive segment |
| Optimal | O(n) | O(1) | Single pass tracking current and maximum streak |

## Algorithm Walkthrough

1. Initialize two variables:

- `current_streak` to track the number of consecutive `1`s currently being processed
- `max_streak` to store the largest streak found so far
2. Traverse the array from left to right.
3. For each number in the array:

- If the number is `1`, increment `current_streak` because the consecutive sequence continues.
- If the number is `0`, reset `current_streak` to `0` because the sequence has been broken.
4. After processing each element, compare `current_streak` with `max_streak`.

- If `current_streak` is larger, update `max_streak`.
5. Continue until the entire array has been processed.
6. Return `max_streak` as the final answer.

### Why it works

The algorithm maintains an important invariant: at every position in the array, `current_streak` represents the number of consecutive `1`s ending at the current index, and `max_streak` represents the largest such streak seen anywhere so far.

Because every element is processed exactly once and every streak is measured as it occurs, the maximum recorded value must be the correct answer.

## Python Solution

```
from typing import List

class Solution:
    def findMaxConsecutiveOnes(self, nums: List[int]) -> int:
        current_streak = 0
        max_streak = 0

        for num in nums:
            if num == 1:
                current_streak += 1
                max_streak = max(max_streak, current_streak)
            else:
                current_streak = 0

        return max_streak
```

The implementation starts by initializing two counters. `current_streak` tracks the current run of consecutive `1`s, while `max_streak` stores the best answer found so far.

The loop processes each number in the array exactly once. When a `1` is encountered, the current streak increases because the sequence continues uninterrupted. The algorithm immediately updates `max_streak` if the new streak is larger than the previous maximum.

When a `0` is encountered, the sequence is broken, so `current_streak` resets to `0`.

At the end of the traversal, `max_streak` contains the length of the longest consecutive sequence of `1`s.

## Go Solution

```
func findMaxConsecutiveOnes(nums []int) int {
    currentStreak := 0
    maxStreak := 0

    for _, num := range nums {
        if num == 1 {
            currentStreak++

            if currentStreak > maxStreak {
                maxStreak = currentStreak
            }
        } else {
            currentStreak = 0
        }
    }

    return maxStreak
}
```

The Go implementation follows the same logic as the Python solution. The main difference is syntax. Go uses `:=` for variable initialization and a `range` loop for iterating through slices.

There are no special concerns about integer overflow because the maximum possible streak is bounded by the array length, which is at most `100000`, well within Go's integer range.

The function also handles empty streaks naturally by resetting `currentStreak` to `0` whenever a `0` is encountered.

## Edge Cases

One important edge case is an array containing only `1`s, such as `[1,1,1,1]`. A buggy implementation might fail to update the maximum streak after the loop ends if it only updates on encountering a `0`. This implementation avoids that problem by updating `max_streak` immediately whenever a `1` is processed.

Another important case is an array containing only `0`s, such as `[0,0,0]`. Some implementations incorrectly initialize the answer to `1`, assuming at least one streak exists. This solution correctly initializes both counters to `0`, so the result remains `0`.

A third edge case is an alternating pattern like `[1,0,1,0,1]`. This tests whether the algorithm properly resets the current streak after each `0`. Since `current_streak` is reset immediately when a `0` is encountered, the implementation correctly returns `1`.

Very small arrays of length `1` are also important. Arrays like `[1]` and `[0]` ensure the algorithm works correctly without relying on multiple iterations or special assumptions about array size.
