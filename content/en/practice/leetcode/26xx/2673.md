## Solution

## Problem Understanding

The problem presents a perfect binary tree with `n` nodes, numbered from `1` to `n`, where the root is node `1`. Each node has a cost given in an array `cost`, where `cost[i]` corresponds to the cost of node `i + 1`. The tree is structured such that for a node `i`, its left child is `2 * i` and its right child is `2 * i + 1`. The objective is to ensure that all root-to-leaf paths in the tree have the same total cost. To achieve this, we can increment the cost of any node by `1` any number of times. The task is to compute the **minimum total number of increments** required to equalize all root-to-leaf path costs.

The input constraints indicate that `n + 1` is a power of two, which guarantees the tree is perfect. The cost array has length `n`, and each cost is between `1` and `10^4`. The problem size allows up to `10^5` nodes, so a naive solution that explores every path explicitly may be too slow.

Edge cases include trees with minimal nodes (e.g., `n = 3`) where paths may already be equal, and trees where all nodes already have identical costs, requiring zero increments.

The key challenge is efficiently propagating the necessary increments from the leaves up to the root without recalculating all paths multiple times.

## Approaches

A brute-force approach would involve calculating the cost of every root-to-leaf path, determining the maximum path cost, and then incrementing the nodes along paths that are below this maximum. While this works logically, it is computationally expensive. For a tree of `n` nodes, there are `n / 2` leaves, and each leaf path has roughly `O(log n)` length. Recomputing path sums for each leaf leads to a time complexity close to `O(n log n)` at best, but managing updates and increments can make it even slower, which is problematic for `n` up to `10^5`.

The key insight is that the problem can be solved **bottom-up** using recursion or a post-order traversal. At each node, we consider the costs of its left and right subtrees and calculate the increments needed to equalize them. Specifically, we propagate the maximum path cost from each child to the parent and add increments to balance the paths. This ensures we only visit each node once, leading to an `O(n)` solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(n log n) | Compute every root-to-leaf path separately and increment differences |
| Optimal | O(n) | O(n) | Post-order traversal, propagating maximum path sums and calculating increments recursively |

## Algorithm Walkthrough

1. Define a recursive function `dfs(node)` that returns two values: the maximum path sum from this node to any leaf and the total increments required in its subtree.
2. For a leaf node (nodes with indices greater than `n // 2`), the maximum path sum is its own cost, and no increments are required because it has no children.
3. For an internal node, recursively call `dfs` on its left and right children. Each call returns the child's maximum path sum and the increments needed in that subtree.
4. Calculate the difference between the left and right maximum path sums. Increment the subtree with the smaller sum to match the larger sum. The total increments at this node are the sum of the left and right increments plus this difference.
5. Return the sum of the node's own cost and the maximum of its children's path sums (this is the new maximum path sum from this node) and the total increments accumulated.
6. Call this function starting from the root node. The total increments returned by the root node is the answer.

Why it works: By processing the tree bottom-up, we ensure that at each internal node, all paths from that node's children are equalized. Propagating the maximum path sum upward guarantees that all root-to-leaf paths eventually have the same total cost. The recursive approach handles all nodes exactly once, ensuring efficiency.

## Python Solution

```python
from typing import List, Tuple

class Solution:
    def minIncrements(self, n: int, cost: List[int]) -> int:
        def dfs(node: int) -> Tuple[int, int]:
            if node > n:
                return 0, 0
            left_sum, left_inc = dfs(2 * node)
            right_sum, right_inc = dfs(2 * node + 1)
            total_inc = left_inc + right_inc + abs(left_sum - right_sum)
            max_sum = cost[node - 1] + max(left_sum, right_sum)
            return max_sum, total_inc

        return dfs(1)[1]
```

This solution defines a helper `dfs` function that handles both the leaf and internal nodes. For each node, it computes the maximum path sum for its subtree and the total increments required to equalize its left and right subtrees. The final answer is the total increments from the root.

## Go Solution

```go
func minIncrements(n int, cost []int) int {
    var dfs func(node int) (int, int)
    dfs = func(node int) (int, int) {
        if node > n {
            return 0, 0
        }
        leftSum, leftInc := dfs(2 * node)
        rightSum, rightInc := dfs(2 * node + 1)
        totalInc := leftInc + rightInc + abs(leftSum - rightSum)
        maxSum := cost[node-1] + max(leftSum, rightSum)
        return maxSum, totalInc
    }

    return dfs(1)
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

In Go, the approach is similar. A recursive function `dfs` returns the maximum path sum and total increments. Helper functions `abs` and `max` handle arithmetic operations cleanly. The slice `cost` is 0-indexed, so we access `cost[node-1]` for the node's cost.

## Worked Examples

**Example 1:** `n = 7, cost = [1,5,2,2,3,3,1]`

- Start DFS at leaf nodes: 4,5,6,7.
- For node 4: leaf, max_sum=2, inc=0. Node 5: leaf, max_sum=3, inc=0.
- Node 2: left_sum=2, right_sum=3, abs diff=1 → total_inc=1, max_sum=5+3=8? Actually need node cost=5 + max(2,3)=8. Correct total_inc=1.
- Similarly, node 3: left_sum=3, right_sum=1, diff=2 → total_inc=2, max_sum=2+3=5. Wait need correct calculation: node cost=2+max(3,1)=5, total_inc=0+0+2=2.
- Node 1: left_sum=8, right_sum=5, diff=3 → total_inc=1+2+3=6. max_sum=1+max(8,5)=9.

Result = 6 increments, which matches the expected output.

**Example 2:** `n = 3, cost = [5,3,3]`

- Leaf nodes 2 and 3: max_sum=3 each, inc=0.
- Node 1: left_sum=3, right_sum=3, diff=0 → total_inc=0. max_sum=5+3=8.

Result = 0 increments.
