## Solution

## Problem Understanding

The problem asks us to implement a **memoized version of a function `fn`**. Memoization is an optimization technique that caches results of expensive function calls so that repeated calls with the same inputs return the cached result instead of recomputing. The input is a function `fn` which can accept any number of arguments of any type, and a series of input arrays that are passed to `fn`. The output is a memoized function that tracks how many times the original function `fn` is actually called, returning the value along with a count.

Inputs are considered identical only if they are `===` in JavaScript terms, meaning that for objects and arrays, they must be the exact same reference in memory. This introduces an important distinction: two objects with identical structure are not the same unless they are the same reference. For primitive values like numbers or strings, equality is straightforward.

Constraints tell us the number of input sets can be up to `10^5`, and the total number of individual input arguments across all calls can also be up to `10^5`. This rules out solutions that recompute results for repeated calls without caching, as they would be too slow.

Edge cases include calls with objects or arrays that look identical but are different references, empty inputs, and repeated identical references.

## Approaches

The **brute-force approach** is to call the function `fn` every time, storing no results. While this always produces correct results, it is inefficient because repeated calls with the same inputs will recompute results, which is exactly what memoization is designed to prevent. With large input sizes (`10^5` calls), this approach becomes too slow.

The **optimal approach** uses a cache to store results. Since inputs can be of any type and we must check reference equality, a suitable data structure is a **nested `Map`** for each argument position. Each level of the map corresponds to one argument, mapping the argument object or primitive to another map for the next argument. Once the last argument is reached, the final map stores the result along with the number of times `fn` has been called for that unique input set.

This approach guarantees O(1) average access time per memoized call (amortized), and correctly handles both primitive and reference-based input comparisons.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) per call | O(1) | Calls `fn` every time, no caching |
| Optimal | O(n) total | O(n) | Uses nested Maps to cache calls for reference equality |

## Algorithm Walkthrough

1. **Initialize a root cache** using a `Map`. This root will hold mappings for the first argument.
2. **Track total calls** to the original function with a counter variable.
3. **Define the memoized function** that accepts arbitrary arguments.
4. **Traverse the nested maps** for each argument in order:

- Check if the current argument exists in the current level of the map.
- If it exists, move to the next level map.
- If it does not exist, create a new `Map` at that level for the argument.
5. **At the last argument**:

- If a cached value exists, return it along with the current call count.
- If not, call `fn` with the arguments, increment the call counter, store the result in the cache, and return the result with the updated call count.

**Why it works:** Each unique combination of argument references is mapped to a unique path in the nested maps. Repeated calls with the same references hit the existing path and return the cached value, while new references extend the map structure. This ensures no repeated calls are made for identical inputs.

## Python Solution

```python
from typing import Any, Callable

def memoize(fn: Callable) -> Callable:
    cache = {}
    calls = 0

    def memoized(*args: Any):
        nonlocal calls
        node = cache
        for arg in args[:-1]:
            if arg not in node:
                node[arg] = {}
            node = node[arg]
        last_arg = args[-1] if args else None
        if last_arg not in node:
            calls += 1
            node[last_arg] = {"val": fn(*args), "calls": calls}
        return node[last_arg]

    return memoized
```

The implementation initializes a nested dictionary (`cache`) and a counter `calls`. Each function call traverses the nested dictionary according to the argument order. For new inputs, the function is invoked, and the result is stored. For repeated references, the cached result is returned. The last argument is treated separately to store the function's result.

## Go Solution

```go
package main

type MemoResult struct {
    Val   any
    Calls int
}

func Memoize(fn func(...any) any) func(...any) MemoResult {
    cache := make(map[any]any)
    calls := 0

    var memoized func(args ...any) MemoResult
    memoized = func(args ...any) MemoResult {
        node := cache
        for i := 0; i < len(args)-1; i++ {
            arg := args[i]
            if _, exists := node[arg]; !exists {
                node[arg] = make(map[any]any)
            }
            node = node[arg].(map[any]any)
        }
        var lastArg any
        if len(args) > 0 {
            lastArg = args[len(args)-1]
        }
        if _, exists := node[lastArg]; !exists {
            calls++
            node[lastArg] = MemoResult{Val: fn(args...), Calls: calls}
        }
        return node[lastArg].(MemoResult)
    }

    return memoized
}
```

In Go, we use `map[any]any` for each level of the cache and wrap the result with a `MemoResult` struct. Type assertions are necessary when traversing the nested maps. This ensures Go correctly stores and retrieves the cached result.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each function call involves traversing the number of arguments, but each unique path is visited once. |
| Space | O(n) | Cache stores a nested map entry for each unique combination of argument references. |

The complexity depends primarily on the number of unique argument reference combinations, not the total number of calls.
