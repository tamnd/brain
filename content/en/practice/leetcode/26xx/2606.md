# LeetCode 2606, Find the Substring With Maximum Cost

## Problem Understanding

The problem gives us a string `s` and a custom character value mapping. Every lowercase English letter normally has a value equal to its alphabetical position, where `'a' = 1`, `'b' = 2`, and so on up to `'z' = 26`.

However, some characters may have overridden values. The string `chars` contains distinct characters, and `vals[i]` represents the custom value for `chars[i]`. Whenever a character appears in `chars`, we must use the overridden value instead of the default alphabetical value.

Our task is to find the substring of `s` with the maximum possible total cost. The cost of a substring is simply the sum of the values of all characters inside it.

An important detail is that the empty substring is allowed, and its value is `0`. This means the answer can never be negative. If every possible substring has a negative total value, we should return `0`.

For example, suppose:

```
s = "adaa"
chars = "d"
vals = [-1000]
```

The character `'d'` has value `-1000`, while `'a'` keeps its default value `1`.

The substring `"aa"` has total value:

```
1 + 1 = 2
```

Any substring containing `'d'` becomes extremely negative, so the maximum answer is `2`.

The constraints are important:

- `s.length` can be as large as `10^5`
- Only lowercase English letters appear
- At most 26 custom mappings exist

A quadratic solution that checks every substring would be too slow because there can be about `10^10` substrings in the worst case. We need an algorithm that runs in linear time.

Several edge cases are important:

- Every character may have negative value
- The best answer may be the empty substring
- A custom value may override a normally positive character with a negative value
- The optimal substring may consist of only one character
- The optimal substring may span almost the entire string

## Approaches

### Brute Force Approach

A straightforward solution is to generate every possible substring and compute its cost.

For every starting index `i`, we can extend the substring to every ending index `j` and keep track of the running sum. While this avoids recomputing sums from scratch, it still requires checking all substrings.

There are:

```
O(n^2)
```

possible substrings, where `n` is the length of the string.

This approach is correct because it explicitly evaluates every valid substring and chooses the maximum cost. However, with `n = 10^5`, this becomes far too slow.

### Optimal Approach

The key observation is that this problem is actually a variation of the classic maximum subarray problem.

Once every character is converted into its numeric value, the problem becomes:

> Find the contiguous subarray with the maximum sum.

This is exactly what Kadane's Algorithm solves efficiently in linear time.

The idea behind Kadane's Algorithm is simple:

- If the current running sum becomes negative, continuing that substring can only hurt future results.
- Therefore, whenever the running sum drops below zero, we discard it and start a new substring.

This allows us to process the string in one pass while always maintaining the best possible substring ending at the current position.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every substring |
| Optimal | O(n) | O(1) | Uses Kadane's Algorithm |

## Algorithm Walkthrough

1. Create a mapping of character values.

We first build a dictionary that stores the value of every character. By default, each letter gets its alphabetical position:

```
a -> 1
b -> 2
...
z -> 26
```

Then we overwrite any characters that appear in `chars` using the values from `vals`.
2. Initialize the running variables.

We maintain two variables:

- `current_sum`, the best substring sum ending at the current position
- `best_sum`, the maximum substring sum seen so far

Both start at `0` because the empty substring is allowed.
3. Iterate through the string.

For each character in `s`, retrieve its value from the mapping.
4. Update the running sum.

Add the current character value to `current_sum`.

If `current_sum` becomes negative, reset it to `0`.

This works because a negative prefix can never improve a future substring.
5. Update the global maximum.

After processing each character, update `best_sum` with the maximum of:

- the existing best answer
- the current running sum
6. Return the result.

At the end of the iteration, `best_sum` contains the maximum substring cost.

### Why it works

Kadane's Algorithm relies on the invariant that `current_sum` always represents the maximum possible substring sum ending at the current index.

If `current_sum` ever becomes negative, carrying it forward would only reduce the value of any future substring. Resetting it to zero ensures we always begin future substrings from the most advantageous point.

Since every substring ending at every position is considered implicitly, the algorithm guarantees that the global maximum substring sum is found.

## Python Solution

```python
from typing import List

class Solution:
    def maximumCostSubstring(self, s: str, chars: str, vals: List[int]) -> int:
        # Default values: a=1, b=2, ..., z=26
        char_values = {
            chr(ord('a') + i): i + 1
            for i in range(26)
        }

        # Override custom values
        for ch, val in zip(chars, vals):
            char_values[ch] = val

        best_sum = 0
        current_sum = 0

        for ch in s:
            current_sum += char_values[ch]

            # Reset if the running sum becomes negative
            if current_sum < 0:
                current_sum = 0

            best_sum = max(best_sum, current_sum)

        return best_sum
```

The implementation begins by constructing a dictionary containing the value for every lowercase English letter. Initially, each character receives its alphabetical position as the default value.

Next, the code applies the custom overrides from `chars` and `vals`. Since the problem guarantees that characters in `chars` are distinct, each assignment is straightforward.

The main loop processes the string once from left to right. For every character, its value is added to the running sum.

If the running sum becomes negative, it is reset to zero. This is the core Kadane's Algorithm optimization because a negative prefix cannot contribute positively to future substrings.

The variable `best_sum` continuously tracks the largest substring cost encountered during traversal.

Finally, the function returns `best_sum`.

## Go Solution

```go
func maximumCostSubstring(s string, chars string, vals []int) int {
	charValues := make(map[byte]int)

	// Default values: a=1, b=2, ..., z=26
	for i := 0; i < 26; i++ {
		charValues[byte('a'+i)] = i + 1
	}

	// Override custom values
	for i := 0; i < len(chars); i++ {
		charValues[chars[i]] = vals[i]
	}

	bestSum := 0
	currentSum := 0

	for i := 0; i < len(s); i++ {
		currentSum += charValues[s[i]]

		if currentSum < 0 {
			currentSum = 0
		}

		if currentSum > bestSum {
			bestSum = currentSum
		}
	}

	return bestSum
}
```

The Go implementation follows the same logic as the Python solution.

A `map[byte]int` is used because all characters are lowercase English letters and can be handled efficiently as bytes.

Unlike Python dictionaries, Go maps require explicit initialization using `make`.

Integer overflow is not a concern because the maximum possible substring sum is well within Go's integer range:

```go
26 * 10^5 = 2.6 * 10^6
```

which easily fits inside a standard integer type.

The algorithm performs a single pass through the string, so the runtime grows linearly with the input size.

The extra memory usage is constant because the character mapping contains only lowercase English letters. Even though a hash map is used, its size never exceeds 26 entries.

## Edge Cases

One important edge case occurs when every character has a negative value. In this situation, every non-empty substring also has a negative total cost. Since the empty substring is allowed and has value `0`, the correct answer must be `0`. The implementation handles this naturally because `best_sum` starts at `0` and the running sum resets whenever it becomes negative.

Another important case is when a normally valuable character receives a large negative override. For example, `'z'` normally contributes `26`, but it may instead become `-1000`. A naive implementation that assumes alphabetical values without properly applying overrides would produce incorrect results. The solution avoids this by building the complete character mapping before processing the string.

A third edge case involves frequent resets of the running sum. Consider a string where large negative values separate positive regions. The algorithm must correctly discard harmful prefixes and restart from the next character. Kadane's Algorithm handles this by resetting `current_sum` to zero whenever it becomes negative, ensuring future substrings are not penalized by earlier losses.

A final edge case occurs when the optimal substring contains exactly one character. For example:

```
s = "ab"
b = -100
```

The best answer is simply `"a"`. The implementation correctly handles this because every position is individually considered as a potential start of a new substring after resets.
