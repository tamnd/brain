## Solution

## Problem Understanding

The problem is asking us to maintain an array of length `n` where initially every element is uncolored, represented by `0`. We are then given a series of queries, each of which updates a specific index in the array with a new color. After each query, we need to determine the number of **adjacent elements** in the array that share the same color. The result is a list of integers, where each integer represents the count after each query.

The input consists of the array size `n` and a list of queries, each query specifying an index and a color. The output is an array of integers of the same length as `queries`.

Constraints tell us that both `n` and the number of queries can be as large as 100,000. Therefore, a naive solution that recomputes the adjacent counts across the whole array after every query will be too slow, because that would involve O(n * q) operations in the worst case.

Important edge cases include when the array has only one element (no adjacency is possible), when a query sets a color to an element that already has that color (the count may remain unchanged), and when updates occur at the ends of the array where only one neighbor exists.

## Approaches

### Brute Force

The brute-force approach would maintain the array `colors` and, after each query, iterate through the entire array to count the number of adjacent elements with the same color. This guarantees correctness, but each query would take O(n) time, leading to an overall complexity of O(n * q), which is too slow for the given constraints.

### Optimal Approach

The key insight for an optimal solution is that **only the neighbors of the updated index can change the count of adjacent same-colored pairs**. We do not need to check the entire array after each update; we only need to look at the left and right neighbors of the index being updated.

For each query:

1. Before updating the color, check if the left neighbor and/or the right neighbor currently forms a same-color pair with the current element. If so, decrease the count.
2. Update the element to the new color.
3. Check the left and right neighbors again. If they now form a same-color pair with the new color, increase the count.
4. Store the current count in the result.

This allows each query to be processed in O(1) time, resulting in an overall O(q) solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * q) | O(n) | Iterates through the entire array for each query |
| Optimal | O(q) | O(n) | Only considers neighbors of the updated index |

## Algorithm Walkthrough

1. Initialize an array `colors` of length `n` with all elements set to `0`.
2. Initialize a variable `same_count` to track the number of adjacent pairs with the same color, starting at `0`.
3. Initialize an empty array `answer` to store results after each query.
4. Iterate through each query `[index, color]`:

1. Check the left neighbor (index - 1) if it exists. If `colors[index] == colors[index - 1]` and `colors[index] != 0`, decrement `same_count`.
2. Check the right neighbor (index + 1) if it exists. If `colors[index] == colors[index + 1]` and `colors[index] != 0`, decrement `same_count`.
3. Update `colors[index]` to the new `color`.
4. Check the left neighbor again. If `colors[index] == colors[index - 1]`, increment `same_count`.
5. Check the right neighbor again. If `colors[index] == colors[index + 1]`, increment `same_count`.
6. Append `same_count` to `answer`.
5. Return `answer`.

**Why it works**: The only positions affected by a color change are the updated index and its immediate neighbors. By updating the count incrementally, we maintain a correct `same_count` after each query without scanning the entire array.

## Python Solution

```python
from typing import List

class Solution:
    def colorTheArray(self, n: int, queries: List[List[int]]) -> List[int]:
        colors = [0] * n
        same_count = 0
        answer = []

        for index, color in queries:
            if index > 0 and colors[index] != 0 and colors[index] == colors[index - 1]:
                same_count -= 1
            if index < n - 1 and colors[index] != 0 and colors[index] == colors[index + 1]:
                same_count -= 1

            colors[index] = color

            if index > 0 and colors[index] == colors[index - 1]:
                same_count += 1
            if index < n - 1 and colors[index] == colors[index + 1]:
                same_count += 1

            answer.append(same_count)

        return answer
```

The implementation first reduces the count if the current color was forming same-color pairs with neighbors. It then updates the color and increments the count for any new pairs formed. This ensures the answer is accurate for each query.

## Go Solution

```go
func colorTheArray(n int, queries [][]int) []int {
    colors := make([]int, n)
    answer := make([]int, 0, len(queries))
    sameCount := 0

    for _, q := range queries {
        index, color := q[0], q[1]

        if index > 0 && colors[index] != 0 && colors[index] == colors[index-1] {
            sameCount--
        }
        if index < n-1 && colors[index] != 0 && colors[index] == colors[index+1] {
            sameCount--
        }

        colors[index] = color

        if index > 0 && colors[index] == colors[index-1] {
            sameCount++
        }
        if index < n-1 && colors[index] == colors[index+1] {
            sameCount++
        }

        answer = append(answer, sameCount)
    }

    return answer
}
```

Go handles slices and zero initialization naturally. The logic is identical to Python, with careful use of slice bounds for neighbors.
