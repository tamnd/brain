## Solution

## Problem Understanding

The problem gives us a 0-indexed integer array `nums` and asks us to construct another array called the distinct difference array.

For every index `i`, we must calculate:

- The number of distinct elements in the prefix `nums[0...i]`
- The number of distinct elements in the suffix `nums[i+1...n-1]`

Then we compute:

$$diff[i] = (\text{distinct elements in prefix}) - (\text{distinct elements in suffix})$$

The result array must contain this value for every index.

The important detail is that the suffix starts at `i + 1`, not at `i`. This means the current element belongs only to the prefix portion.

For example, if:

```
nums = [1,2,3,4,5]
```

At index `2`:

- Prefix = `[1,2,3]`, distinct count = `3`
- Suffix = `[4,5]`, distinct count = `2`

So:

```
diff[2] = 3 - 2 = 1
```

The constraints are very small:

- `1 <= n <= 50`
- `1 <= nums[i] <= 50`

This tells us that even a brute-force solution would pass comfortably. However, the problem is still useful for practicing prefix and suffix counting techniques with hash sets.

Several edge cases are important:

- Arrays with only one element
- Arrays where all values are the same
- Arrays where all values are distinct
- Arrays with repeated values appearing in both prefix and suffix

A common mistake is accidentally including `nums[i]` in the suffix count. The suffix must begin strictly after index `i`.

## Approaches

### Brute Force Approach

The most direct solution is to compute the answer independently for every index.

For each position `i`:

1. Create a set containing all elements from `nums[0...i]`
2. Create another set containing all elements from `nums[i+1...n-1]`
3. Subtract the sizes of the two sets
4. Store the result

This works because sets automatically remove duplicates, so the size of the set equals the number of distinct elements.

Although this solution is simple and correct, it repeatedly scans large portions of the array for every index. Since we rebuild sets again and again, there is unnecessary repeated work.

With `n <= 50`, this is still fast enough, but we can do better conceptually.

### Optimal Approach

The key observation is that the number of distinct elements in prefixes and suffixes can be precomputed efficiently.

We can build:

- A `prefixDistinct` array where `prefixDistinct[i]` stores the number of distinct elements from `0` to `i`
- A `suffixDistinct` array where `suffixDistinct[i]` stores the number of distinct elements from `i` to `n-1`

Once these arrays are available, the answer for index `i` becomes:

```
prefixDistinct[i] - suffixDistinct[i + 1]
```

If `i` is the last index, the suffix is empty, so the suffix distinct count is `0`.

We use hash sets because they provide efficient insertion and membership checking while naturally handling duplicates.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Rebuilds prefix and suffix sets for every index |
| Optimal | O(n) | O(n) | Precomputes prefix and suffix distinct counts |

## Algorithm Walkthrough

### Step 1: Build the Prefix Distinct Count Array

Create an empty hash set called `seen`.

Traverse the array from left to right.

At each index:

- Add the current value to the set
- Store the current size of the set in `prefixDistinct[i]`

This works because the set always contains every unique value seen so far.

### Step 2: Build the Suffix Distinct Count Array

Clear the set and traverse the array from right to left.

At each index:

- Add the current value to the set
- Store the size of the set in `suffixDistinct[i]`

Now `suffixDistinct[i]` represents the number of distinct elements from index `i` to the end.

### Step 3: Construct the Answer

Create the result array.

For each index `i`:

- The prefix distinct count is `prefixDistinct[i]`
- The suffix distinct count is:

- `suffixDistinct[i + 1]` if `i + 1 < n`
- `0` otherwise

Compute:

```
result[i] = prefixDistinct[i] - suffixCount
```

### Why it works

The prefix array correctly stores the number of distinct elements up to each index because the set accumulates all unique values encountered during the left-to-right traversal.

Similarly, the suffix array correctly stores the number of distinct elements from each index to the end because the reverse traversal accumulates all unique values encountered from the right side.

For every position `i`, the required suffix excludes `nums[i]`, so we use `suffixDistinct[i + 1]`. Therefore, each result entry exactly matches the definition given in the problem statement.

## Python Solution

```python
from typing import List

class Solution:
    def distinctDifferenceArray(self, nums: List[int]) -> List[int]:
        n = len(nums)

        prefix_distinct = [0] * n
        suffix_distinct = [0] * n

        seen = set()

        # Build prefix distinct counts
        for i in range(n):
            seen.add(nums[i])
            prefix_distinct[i] = len(seen)

        seen.clear()

        # Build suffix distinct counts
        for i in range(n - 1, -1, -1):
            seen.add(nums[i])
            suffix_distinct[i] = len(seen)

        result = [0] * n

        # Construct answer
        for i in range(n):
            suffix_count = suffix_distinct[i + 1] if i + 1 < n else 0
            result[i] = prefix_distinct[i] - suffix_count

        return result
```

The implementation begins by allocating two arrays that will store prefix and suffix distinct counts.

The first traversal moves from left to right. The `seen` set keeps track of all unique numbers encountered so far. After inserting the current element, the size of the set becomes the distinct count for the current prefix.

The second traversal moves from right to left. This time, the set tracks distinct elements appearing in the suffixes.

Finally, the algorithm computes the answer for every position by subtracting the suffix distinct count from the prefix distinct count. The last index requires special handling because its suffix is empty.

## Go Solution

```go
func distinctDifferenceArray(nums []int) []int {
	n := len(nums)

	prefixDistinct := make([]int, n)
	suffixDistinct := make([]int, n)

	seen := make(map[int]bool)

	// Build prefix distinct counts
	for i := 0; i < n; i++ {
		seen[nums[i]] = true
		prefixDistinct[i] = len(seen)
	}

	// Clear map
	seen = make(map[int]bool)

	// Build suffix distinct counts
	for i := n - 1; i >= 0; i-- {
		seen[nums[i]] = true
		suffixDistinct[i] = len(seen)
	}

	result := make([]int, n)

	// Construct answer
	for i := 0; i < n; i++ {
		suffixCount := 0

		if i+1 < n {
			suffixCount = suffixDistinct[i+1]
		}

		result[i] = prefixDistinct[i] - suffixCount
	}

	return result
}
```

The Go solution follows exactly the same logic as the Python version.

Instead of Python sets, Go uses `map[int]bool` to simulate a hash set. The number of distinct elements is obtained using `len(seen)`.

Slices are dynamically allocated using `make`. Since the constraints are small, integer overflow is not a concern.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We traverse the array a constant number of times |
| Space | O(n) | Prefix array, suffix array, result array, and hash sets require linear space |

The algorithm performs three linear passes over the array:

- One pass for prefix distinct counts
- One pass for suffix distinct counts
- One pass to build the result

Each set insertion and lookup operation is average-case O(1), so the total runtime is linear.

The extra arrays and hash sets together require linear auxiliary memory.

## Edge Cases

### Single Element Array

If the array contains only one element, the prefix contains one distinct element and the suffix is empty.

For example:

```
nums = [7]
```

The result should be:

```
[1]
```

A buggy implementation might accidentally attempt to access `suffixDistinct[i + 1]` out of bounds. The implementation handles this safely by explicitly using `0` when `i` is the last index.

### All Elements Identical

Consider:

```
nums = [2,2,2,2]
```

The number of distinct elements in any non-empty subarray is always `1`.

This case is useful because it verifies that duplicate values are not counted multiple times. The hash set correctly guarantees uniqueness.

### All Elements Distinct

Consider:

```
nums = [1,2,3,4,5]
```

In this case, the prefix distinct count increases by one at every step, while the suffix distinct count decreases by one.

This validates that the algorithm correctly separates prefix and suffix ranges and does not accidentally include the current element in the suffix.

### Repeated Values on Both Sides

Consider:

```
nums = [1,2,1,2,1]
```

Some values appear in both the prefix and suffix simultaneously. This can expose logical errors where implementations incorrectly remove duplicates or mix ranges.

The algorithm avoids this problem because prefix and suffix distinct counts are computed independently using separate traversals.
