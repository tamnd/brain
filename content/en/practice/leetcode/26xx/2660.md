## Solution

## Problem Understanding

The problem presents a variation of scoring in a bowling game. You are given two integer arrays, `player1` and `player2`, each representing the number of pins hit in each turn of a bowling game. Every turn has exactly 10 pins. The scoring rule is not the standard one: if a player hits a **strike** (10 pins) in either of the previous two turns, the current turn's pin count is **doubled**; otherwise, the score is simply the number of pins hit.

The task is to compute the **total scores** of both players and determine the winner: return 1 if player 1 has a higher score, 2 if player 2 has a higher score, and 0 if the scores are equal.

The constraints indicate that each array has a length between 1 and 1000, and each pin count is between 0 and 10. This allows for a linear-time solution. Edge cases to consider include very short arrays, consecutive strikes, or no strikes at all.

## Approaches

A brute-force approach would involve iterating over the arrays and computing the score for each turn by checking all previous turns for strikes. This is correct but unnecessary because the problem only depends on the previous two turns.

The optimal approach leverages the observation that for each turn `i`, we only need to check `i-1` and `i-2` to determine whether the turn’s score should be doubled. This allows a simple linear scan through each player's array while maintaining the score calculation efficiently.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Checks all previous turns for strikes, unnecessary because only the last two matter |
| Optimal | O(n) | O(1) | Scans once per player, doubling based on the previous two turns only |

## Algorithm Walkthrough

1. Initialize a variable `score1` for player 1 and `score2` for player 2, both set to 0. These will accumulate total scores.
2. For each player, iterate over their array of pin counts. For turn `i`:

- If `i >= 1` and the previous turn had 10 pins, or `i >= 2` and two turns ago had 10 pins, multiply the current pins by 2 and add to the score.
- Otherwise, add the current pins as-is.
3. After computing both scores, compare `score1` and `score2`.

- If `score1 > score2`, return 1.
- If `score2 > score1`, return 2.
- Otherwise, return 0 for a draw.

Why it works: The algorithm directly implements the game’s scoring rule. The key invariant is that at each turn, the score depends only on the last two turns, so we do not miss any doubling opportunities or miscalculate any turn.

## Python Solution

```python
from typing import List

class Solution:
    def isWinner(self, player1: List[int], player2: List[int]) -> int:
        def compute_score(pins: List[int]) -> int:
            score = 0
            n = len(pins)
            for i in range(n):
                if (i >= 1 and pins[i - 1] == 10) or (i >= 2 and pins[i - 2] == 10):
                    score += 2 * pins[i]
                else:
                    score += pins[i]
            return score
        
        score1 = compute_score(player1)
        score2 = compute_score(player2)
        
        if score1 > score2:
            return 1
        elif score2 > score1:
            return 2
        else:
            return 0
```

The `compute_score` helper abstracts the scoring logic, iterating once per player array. It checks only the last two turns to determine if doubling is necessary, ensuring an O(n) runtime. The final comparison handles all three outcomes cleanly.

## Go Solution

```go
func isWinner(player1 []int, player2 []int) int {
    computeScore := func(pins []int) int {
        score := 0
        n := len(pins)
        for i := 0; i < n; i++ {
            if (i >= 1 && pins[i-1] == 10) || (i >= 2 && pins[i-2] == 10) {
                score += 2 * pins[i]
            } else {
                score += pins[i]
            }
        }
        return score
    }
    
    score1 := computeScore(player1)
    score2 := computeScore(player2)
    
    if score1 > score2 {
        return 1
    } else if score2 > score1 {
        return 2
    } else {
        return 0
    }
}
```

Go-specific notes: The solution uses an inline function for `computeScore`, which is a common Go pattern. Array bounds are explicitly checked (`i >= 1`, `i >= 2`) to prevent index out-of-range errors. Otherwise, the logic mirrors the Python implementation.

## Edge Cases

One important edge case is a **single-turn game**. In this case, there are no previous turns, so no doubling occurs. The solution handles this naturally by checking `i >= 1` and `i >= 2`.

Another edge case is **consecutive strikes**, where multiple turns in a row are 10. The doubling effect cascades correctly because each turn only looks at the previous two turns.

A third edge case is a **draw**, either with zero pins or sequences that lead to equal scores despite different pin distributions. The final comparison explicitly checks for equality and returns 0 in that case, avoiding incorrect winner assignment.
