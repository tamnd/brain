## Solution

## Problem Understanding

The problem requires implementing a generator for **multi-dimensional arrays** that yields integers in **inorder traversal**. In simpler terms, we have an array that may contain integers or other arrays nested arbitrarily deep. We need to traverse the array in a way that we first process elements from left to right, and whenever an element is itself an array, we recursively traverse it before moving on.

The input `arr` can be empty, partially empty (arrays with no elements), or deeply nested. The output is a generator that produces integers one by one without creating a flattened copy of the array. This means we need a **lazy evaluation approach**, which yields integers as we traverse rather than precomputing the whole result.

Constraints suggest that the total number of integers after flattening is up to `10^5` and nesting depth is also up to `10^5`. This makes recursion risky if implemented naively because Python has a recursion limit, and deep nested arrays may cause a stack overflow. Therefore, an **iterative approach using an explicit stack** is preferable for robustness.

Edge cases include an empty array, arrays containing only empty arrays, deeply nested single integers, and arrays with mixed integers and nested arrays.

## Approaches

The **brute-force approach** is to first flatten the array completely using recursion or built-in flattening methods. Once flattened, we can simply yield each element one by one. This guarantees correct results but has major drawbacks: it requires **O(n) extra space** to store the flattened array and can be slow if `n` is large.

The **optimal approach** uses an explicit **stack to simulate recursion**. The key observation is that we can traverse a nested array lazily by maintaining a stack of iterators. Whenever we encounter an integer, we yield it. Whenever we encounter a nested array, we push its iterator to the stack. This approach avoids creating a full flattened copy, processes elements on-demand, and handles deep nesting safely.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Flatten entire array then iterate, extra memory for flattened array |
| Optimal | O(n) | O(d) | Use stack of iterators to yield elements lazily, d is max nesting depth |

## Algorithm Walkthrough

1. Initialize a **stack** with the iterator of the input array. The stack keeps track of arrays currently being traversed and their positions.
2. Enter a loop that continues as long as the stack is not empty.
3. Peek at the top iterator from the stack and attempt to get the next element.
4. If the iterator is exhausted, pop it from the stack and continue.
5. If the element is an integer, yield it immediately.
6. If the element is a nested array, push its iterator onto the stack. This ensures we process the nested array before continuing with the remaining elements of the current array.
7. Repeat steps 2-6 until all elements have been processed.

**Why it works:** This algorithm maintains the invariant that the stack always represents the path from the outermost array down to the current element. By using iterators, we can pause and resume traversal at any level, yielding integers in **exact left-to-right order** as required. No additional storage for flattened elements is needed, and the stack depth never exceeds the maximum nesting depth.

## Python Solution

```python
from typing import List, Generator, Union

NestedArray = List[Union[int, 'NestedArray']]

class NestedIterator:
    def __init__(self, arr: NestedArray):
        self.stack = [iter(arr)]

    def __iter__(self) -> Generator[int, None, None]:
        return self

    def __next__(self) -> int:
        while self.stack:
            try:
                element = next(self.stack[-1])
                if isinstance(element, int):
                    return element
                elif isinstance(element, list):
                    self.stack.append(iter(element))
                else:
                    raise ValueError("Invalid element type")
            except StopIteration:
                self.stack.pop()
        raise StopIteration

def inorderTraversal(arr: NestedArray) -> Generator[int, None, None]:
    return NestedIterator(arr)
```

This implementation defines a `NestedIterator` class that encapsulates a stack of iterators. The `__next__` method retrieves the next integer, handling nested arrays by pushing their iterators to the stack. `inorderTraversal` simply returns an instance of this iterator.

## Go Solution

```go
package main

type NestedArray []interface{}

type NestedIterator struct {
	stack []interface{}
	index []int
	arr   []NestedArray
}

func Constructor(arr NestedArray) *NestedIterator {
	return &NestedIterator{
		stack: []interface{}{arr},
	}
}

func (it *NestedIterator) Next() int {
	for len(it.stack) > 0 {
		top := it.stack[len(it.stack)-1]
		it.stack = it.stack[:len(it.stack)-1]
		switch val := top.(type) {
		case int:
			return val
		case []interface{}:
			for i := len(val) - 1; i >= 0; i-- {
				it.stack = append(it.stack, val[i])
			}
		}
	}
	panic("No more elements")
}

func (it *NestedIterator) HasNext() bool {
	for len(it.stack) > 0 {
		top := it.stack[len(it.stack)-1]
		switch val := top.(type) {
		case int:
			return true
		case []interface{}:
			it.stack = it.stack[:len(it.stack)-1]
			for i := len(val) - 1; i >= 0; i-- {
				it.stack = append(it.stack, val[i])
			}
		}
	}
	return false
}
```

In Go, we simulate the generator by providing `Next()` and `HasNext()` methods. The stack stores interface{} elements to handle both integers and nested slices. The nested arrays are pushed in reverse order to ensure left-to-right traversal.

## Edge Cases

One critical edge case is **completely empty arrays** or arrays containing only empty arrays. These can cause naive recursion to yield None or fail. Our implementation handles this by popping exhausted iterators and continuing until the stack is empty.

Another edge case is **deeply nested single integers**. Recursive solutions may exceed the Python recursion limit. By using an explicit stack of iterators, our algorithm avoids recursion limits and safely traverses even 10^5 depth.

Finally, **mixed nesting with integers and empty arrays** requires careful traversal. We must skip over empty arrays and only yield integers, which is correctly handled by continuously attempting `next()` and popping exhausted iterators.
