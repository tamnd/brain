## Solution

## Problem Understanding

The problem is asking to split a given array `arr` into smaller subarrays of length `size`. Each subarray must preserve the original order of elements from `arr`. If the length of `arr` is not divisible evenly by `size`, the last subarray will contain the remaining elements, which will be fewer than `size`. The input array can be as small as empty or as large as 105 elements, and the chunk size is guaranteed to be at least 1 and at most `arr.length + 1`. The problem guarantees valid input, so we do not need to handle negative sizes or null arrays. Important edge cases include an empty input array, a chunk size larger than the array, and a chunk size of 1.

The expected output is a list of lists (or slice of slices in Go), where each inner list represents a chunk of the original array.

## Approaches

The brute-force approach is straightforward: we can iterate over the array, collect elements into a temporary subarray, and once it reaches the given `size`, we append it to the output list and start a new temporary subarray. At the end, any remaining elements are appended as the final chunk. This method works correctly but does not leverage any language-specific slicing features that make the solution more concise. In terms of time complexity, it is still O(n) since every element is visited once, but the implementation can be verbose.

The optimal approach uses array slicing directly. We iterate over `arr` with a step size equal to the chunk `size`, and for each step, we slice the array from the current index to the current index plus `size`. This directly creates each chunk without manually managing temporary subarrays. This approach is concise, readable, and efficient in both Python and Go.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Manually collects elements into temporary arrays and appends to output |
| Optimal | O(n) | O(n) | Uses array slicing to create chunks directly, simpler and more readable |

## Algorithm Walkthrough

1. Initialize an empty list `result` to store the chunks.
2. Iterate over `arr` using a loop with index `i` starting from 0 up to the length of the array, stepping by `size`.
3. For each index `i`, slice the array from `i` to `i + size`. This creates a subarray containing at most `size` elements.
4. Append the sliced subarray to `result`.
5. Continue until the end of the array is reached. If the remaining elements are fewer than `size`, slicing automatically captures only the remaining elements.
6. Return `result`.

Why it works: This algorithm works because slicing handles boundaries correctly and preserves the original order of elements. Iterating with a step of `size` guarantees non-overlapping chunks of the desired size.

## Python Solution

```python
from typing import List

class Solution:
    def chunkArray(self, arr: List[int], size: int) -> List[List[int]]:
        result = []
        for i in range(0, len(arr), size):
            result.append(arr[i:i+size])
        return result
```

The implementation initializes an empty list `result` to hold the final chunks. The `for` loop iterates over the array with a step equal to `size`, and the slicing operation `arr[i:i+size]` extracts each chunk. The slice operation automatically handles the last chunk when fewer than `size` elements remain. Finally, the list of chunks is returned.

## Go Solution

```go
func chunkArray(arr []int, size int) [][]int {
    result := [][]int{}
    for i := 0; i < len(arr); i += size {
        end := i + size
        if end > len(arr) {
            end = len(arr)
        }
        result = append(result, arr[i:end])
    }
    return result
}
```

In Go, we initialize `result` as a slice of slices of integers. The loop increments by `size`. Since Go slicing does not automatically handle overflows, we calculate `end` as the smaller of `i + size` and `len(arr)`. This ensures we do not exceed the array bounds. We append each slice to `result` and return it at the end.

## Edge Cases

The first important edge case is an **empty array**. If `arr` is empty, no chunks should be created, and the function must return an empty list. Both Python slicing and Go slicing handle this automatically.

The second edge case is when the **chunk size is greater than the array length**. In this case, the entire array becomes a single chunk. In Python, slicing automatically returns the whole array without error. In Go, we manually ensure the `end` index does not exceed the array length.

The third edge case is when the **chunk size is 1**. This tests that the algorithm can handle the smallest valid chunk size and returns each element as its own subarray. Iterating in steps of 1 ensures correctness.

All other edge cases, such as chunk size equal to array length or uneven last chunk, are naturally handled by the slicing logic and loop stepping.

This solution is robust, efficient, and handles all edge cases within the problem constraints.
