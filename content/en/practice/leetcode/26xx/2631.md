## Solution

## Problem Understanding

This problem asks us to extend JavaScript arrays with a custom method called `groupBy`. The method receives a callback function `fn`, and it must organize the array elements into groups based on the string key returned by that function.

For every element `arr[i]`, we call `fn(arr[i])`. The returned string becomes the key in the result object. All elements that produce the same key are collected together inside the same array.

The final output is therefore an object where:

- Each property name is a grouping key returned by `fn`
- Each property value is an array containing all matching elements
- The original ordering of elements inside each group must remain unchanged

The problem guarantees that the callback function always returns a string. This is important because object keys in JavaScript are strings, so we do not need to handle conversion ourselves.

For example, suppose we have:

```
[1, 2, 3, 4]
```

and:

```
fn = n => String(n % 2)
```

Then:

- `1 % 2 = 1`
- `2 % 2 = 0`
- `3 % 2 = 1`
- `4 % 2 = 0`

The grouped result becomes:

```
{
  "1": [1, 3],
  "0": [2, 4]
}
```

The array may contain any kind of values:

- Numbers
- Strings
- Objects
- Arrays
- Mixed types

The grouping logic depends entirely on the callback function.

The constraint `array.length <= 10^5` tells us the solution must be efficient. An algorithm with quadratic complexity, `O(n^2)`, would be too slow for large inputs. We should aim for a linear time solution.

Several edge cases are important:

- An empty array should return an empty object
- All elements may belong to the same group
- Every element may belong to a different group
- The array may contain objects or nested arrays
- Keys may repeat many times
- The grouping function may generate `"true"` and `"false"` as string keys

The problem guarantees valid input and valid string keys, so we do not need additional validation logic.

## Approaches

### Brute Force Approach

A brute force solution would process each element and search through previously created groups to determine whether the key already exists.

One possible implementation is:

1. Maintain a list of `(key, values)` pairs
2. For every element:

- Compute the key
- Linearly scan existing groups
- If the key exists, append the element
- Otherwise create a new group

This approach works because every element is eventually placed into exactly one matching group. However, searching through existing groups repeatedly becomes expensive.

If there are `n` elements and many unique keys, we may perform nearly `n` comparisons for every insertion. That leads to `O(n^2)` time complexity in the worst case.

With up to `10^5` elements, quadratic performance is too slow.

### Optimal Approach

The key observation is that grouping is fundamentally a hash map problem.

We need fast lookup by key. A hash map, or JavaScript object, allows average `O(1)` insertion and lookup.

Instead of searching through existing groups manually:

1. Compute the key
2. Check whether the key already exists in the object
3. If not, initialize an empty array
4. Append the current element to that array

This avoids repeated scanning and processes every element exactly once.

Because each lookup and insertion is constant time on average, the total runtime becomes linear.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly scans existing groups |
| Optimal | O(n) | O(n) | Uses hash map for constant time grouping |

## Algorithm Walkthrough

1. Create an empty object called `groups`.

This object will store the grouped arrays. Each property corresponds to one grouping key.
2. Iterate through every element in the array.

We must process each item exactly once because every element belongs to exactly one group.
3. Call the callback function on the current element.

```
const key = fn(item);
```

This determines which group the current element belongs to.
4. Check whether the key already exists in the object.

If it does not exist yet, initialize it with an empty array.

```
if (!groups[key]) {
    groups[key] = [];
}
```
5. Append the current element into its group.

```
groups[key].push(item);
```

This preserves the original ordering because elements are inserted in traversal order.
6. Continue until all elements are processed.
7. Return the final object.

### Why it works

The algorithm maintains the invariant that after processing the first `i` elements, every processed element has already been placed into exactly one correct group.

When a new element is processed, its grouping key is computed deterministically by `fn`. The algorithm appends the element directly into the corresponding bucket. Since every element is processed once and inserted exactly once, the final object contains all elements grouped correctly.

Because insertion order is preserved during traversal, the relative order inside each group also remains correct.

## Python Solution

```python
from collections import defaultdict
from typing import Any, Callable, Dict, List

class Solution:
    def groupBy(
        self,
        array: List[Any],
        fn: Callable[[Any], str]
    ) -> Dict[str, List[Any]]:
        
        groups = defaultdict(list)

        for item in array:
            key = fn(item)
            groups[key].append(item)

        return dict(groups)
```

The implementation uses Python's `defaultdict(list)` to simplify group creation. Instead of manually checking whether a key exists, `defaultdict` automatically initializes an empty list whenever a new key is accessed.

The loop processes each element exactly once. For every item:

1. The grouping key is computed using the callback function
2. The item is appended directly into the correct list

Finally, the `defaultdict` is converted into a normal dictionary before returning.

This implementation directly mirrors the optimal algorithm described earlier.

## Go Solution

```go
package main

func groupBy(array []interface{}, fn func(interface{}) string) map[string][]interface{} {
	groups := make(map[string][]interface{})

	for _, item := range array {
		key := fn(item)
		groups[key] = append(groups[key], item)
	}

	return groups
}
```

The Go implementation uses a map from strings to slices.

```go
map[string][]interface{}
```

Unlike Python's `defaultdict`, Go maps do not automatically initialize missing keys. However, Go's `append` works correctly even on a nil slice, so we can directly write:

```go
groups[key] = append(groups[key], item)
```

If the key does not exist yet, the zero value for the slice is `nil`, and `append` automatically creates the slice.

Go does not require special handling for integer overflow here because no arithmetic operations are involved.

## Edge Cases

### Empty Array

An empty input array is an important boundary case. A naive implementation might accidentally return `null`, `None`, or an uninitialized structure.

The correct behavior is to return an empty object because there are no elements to group. The implementation handles this naturally because the loop never executes, so the initially empty hash map is returned unchanged.

### Every Element Has a Unique Key

If every callback result is different, the algorithm creates a new group for every element.

This can expose bugs in implementations that incorrectly reuse arrays or overwrite existing groups. The current solution safely initializes a fresh list for every unseen key and stores each element independently.

### All Elements Belong to the Same Group

If all callback results are identical, the output contains exactly one key with all elements inside it.

Some incorrect implementations may accidentally replace the existing array instead of appending to it. The current implementation always appends to the existing group, ensuring all matching elements are preserved in order.

### Order Preservation Inside Groups

The problem explicitly requires that elements remain in their original order within each group.

A buggy implementation that sorts elements or inserts them incorrectly would violate this requirement. Because the algorithm processes the array from left to right and appends elements immediately, the original relative ordering is preserved automatically.
