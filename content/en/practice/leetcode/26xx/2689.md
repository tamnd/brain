## Solution

## Problem Understanding

The problem gives us a special binary tree called a Rope Tree. Instead of storing one large string directly, the string is split across leaf nodes, and internal nodes represent concatenations of their children.

Each node contains:

- `node.val`, a string
- `node.len`, the total length of the string represented by that subtree

There are two kinds of nodes:

- Leaf nodes contain actual string data in `node.val`, have no children, and always have `node.len = 0`
- Internal nodes contain an empty string in `node.val`, have at least one child, and their `node.len` equals the total length of the concatenated subtree

The recursive definition is:

- If the node is a leaf, then `S[node] = node.val`
- Otherwise, `S[node] = S[node.left] + S[node.right]`

Our goal is to return the `k`-th character of the final string represented by the root node.

An important detail is that `k` is 1-indexed. That means:

- `k = 1` refers to the first character
- `k = 2` refers to the second character
- and so on

The constraints are relatively small:

- At most `1000` nodes
- Each leaf string has length at most `50`
- Total rope length can reach `10^4`

Even though the limits are not enormous, the structure of a rope tree strongly suggests that we should avoid reconstructing the entire string if possible. Rope trees are specifically designed to support efficient substring and indexing operations without materializing the full concatenated string.

Several edge cases are important:

- The tree may consist of only a single leaf node
- Internal nodes may have only one child
- `k` may point to the very first or very last character
- Some internal nodes may have an empty left or right subtree
- The tree can be highly unbalanced

The problem guarantees that `k` is always valid, meaning it will always fall within the total length of `S[root]`.

## Approaches

### Brute Force Approach

The most straightforward solution is to recursively construct the entire string represented by the rope tree.

For every node:

- If it is a leaf node, return `node.val`
- Otherwise, recursively build the left subtree string and the right subtree string, then concatenate them

Once we have the full string, we simply return the character at index `k - 1`.

This approach is correct because it directly follows the recursive definition of the rope tree. Every internal node represents concatenation of its children, so recursively building the entire string reconstructs `S[root]`.

However, this method is inefficient because string concatenation repeatedly copies characters. Constructing the full string also requires storing the entire result in memory, even though we only need one character.

### Optimal Approach

The key insight is that we never actually need the full string.

Each internal node already stores enough information to decide whether the `k`-th character lies in the left subtree or the right subtree.

Suppose:

- The left subtree length is `leftLength`

Then:

- If `k <= leftLength`, the answer must be inside the left subtree
- Otherwise, the answer lies in the right subtree, and we search for the `(k - leftLength)`-th character there

This allows us to navigate directly toward the desired character without constructing unnecessary strings.

At a leaf node, we simply return `node.val[k - 1]`.

This transforms the problem into a guided DFS traversal.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n + L) | O(L) | Builds the entire concatenated string |
| Optimal | O(h) | O(h) | Traverses only one path from root to leaf |

Here:

- `n` is the number of nodes
- `L` is the total string length
- `h` is the tree height

## Algorithm Walkthrough

1. Start from the root node and the given `k`.
2. Check whether the current node is a leaf node. A leaf node has no children. If it is a leaf:

- The node directly contains a string
- Return the character at index `k - 1`
3. Otherwise, determine the total length represented by the left subtree.

- If the left child exists:

- If the left child is a leaf, its length is `len(left.val)`
- Otherwise, its length is `left.len`
- If the left child does not exist, the left length is `0`
4. Compare `k` with the left subtree length.

- If `k <= leftLength`, the target character lies entirely inside the left subtree
- Recursively continue into the left child using the same `k`
5. Otherwise:

- The target character belongs to the right subtree
- Subtract `leftLength` from `k`
- Recursively continue into the right child
6. Continue until reaching a leaf node.

### Why it works

At every internal node, the rope definition guarantees that the represented string equals:

`leftSubtreeString + rightSubtreeString`

Therefore, the first `leftLength` characters always belong to the left subtree. If `k` exceeds that range, the character must belong to the right subtree after adjusting the index by subtracting the left portion size.

Because each recursive step eliminates one entire subtree from consideration, the algorithm always follows the unique path leading to the target character.

## Python Solution

```python
# Definition for a rope tree node.
# class RopeTreeNode(object):
#     def __init__(self, len=0, val="", left=None, right=None):
#         self.len = len
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def getKthCharacter(self, root: Optional[object], k: int) -> str:
        def dfs(node: Optional[object], k: int) -> str:
            # Leaf node
            if node.left is None and node.right is None:
                return node.val[k - 1]

            left_length = 0

            if node.left:
                if node.left.left is None and node.left.right is None:
                    left_length = len(node.left.val)
                else:
                    left_length = node.left.len

            if k <= left_length:
                return dfs(node.left, k)

            return dfs(node.right, k - left_length)

        return dfs(root, k)
```

The implementation closely follows the recursive structure of the rope tree.

The helper function `dfs` searches for the `k`-th character inside the current subtree.

The first step checks whether the current node is a leaf node. If both children are `None`, the node directly stores the actual string data, so we return the character at index `k - 1`.

For internal nodes, we compute the length of the left subtree. Internal nodes already store their subtree length in `node.len`, but leaf nodes always have `len = 0`, so for leaves we must compute the length using `len(node.val)`.

Once we know the left subtree length, we determine which side contains the target character.

If `k` falls within the left subtree range, we recursively continue left. Otherwise, we skip all left subtree characters by subtracting `left_length` from `k` and continue right.

Because we only traverse one path from the root to a leaf, the solution remains efficient.

## Go Solution

```go
/**
 * Definition for a rope tree node.
 * type RopeTreeNode struct {
 *     len   int
 *     val   string
 *     left  *RopeTreeNode
 *     right *RopeTreeNode
 * }
 */

func getKthCharacter(root *RopeTreeNode, k int) byte {
	var dfs func(node *RopeTreeNode, k int) byte

	dfs = func(node *RopeTreeNode, k int) byte {
		// Leaf node
		if node.left == nil && node.right == nil {
			return node.val[k-1]
		}

		leftLength := 0

		if node.left != nil {
			if node.left.left == nil && node.left.right == nil {
				leftLength = len(node.left.val)
			} else {
				leftLength = node.left.len
			}
		}

		if k <= leftLength {
			return dfs(node.left, k)
		}

		return dfs(node.right, k-leftLength)
	}

	return dfs(root, k)
}
```

The Go implementation mirrors the Python solution closely.

One important Go-specific detail is that strings are indexed as bytes. Since the problem guarantees lowercase English letters only, returning `byte` is completely safe because each character occupies exactly one byte.

Go uses `nil` instead of Python's `None`, and recursive helper functions are typically declared using function variables when recursion is needed inside another function.

## Worked Examples

### Example 1

Input:

```
root = [10,4,"abcpoe","g","rta"]
k = 6
```

Constructed string:

```
"grtaabcpoe"
```

We want the 6th character.

| Step | Current Node | Left Length | k | Action |
| --- | --- | --- | --- | --- |
| 1 | Root | 4 | 6 | Go right, k = 6 - 4 = 2 |
| 2 | "abcpoe" leaf | - | 2 | Return val[1] = "b" |

Final answer:

```
"b"
```

| --- | --- | --- | --- | --- |
| 1 | Root | 6 | 3 | Go left |
| 2 | Left internal node | 3 | 3 | Go left |
| 3 | "abc" leaf | - | 3 | Return val[2] = "c" |

Final answer:

```
"c"
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(h) | We traverse only one root-to-leaf path |
| Space | O(h) | Recursive call stack depth equals tree height |

The algorithm never explores both subtrees simultaneously. At each internal node, it chooses exactly one direction based on the left subtree length.

In the best case, the tree is balanced and the height is logarithmic. In the worst case, the tree may become completely skewed, making the height equal to the number of nodes.

## Edge Cases

### Single Leaf Node

The entire rope tree may consist of only one leaf node with no internal structure. A buggy implementation might incorrectly assume internal nodes always exist and attempt subtree traversal.

This implementation handles the case immediately through the leaf-node check:

```
if node.left is None and node.right is None:
```

The character is returned directly from the leaf string.

### Highly Skewed Tree

The tree may degenerate into a linked-list-like structure where every internal node has only one meaningful subtree. Incorrect implementations may accidentally assume both children always exist.

This solution safely checks whether the left child exists before accessing its length and naturally follows whichever subtree contains the answer.

### k Located at Subtree Boundary

Boundary conditions are common sources of off-by-one errors.

Suppose the left subtree length is exactly `5`:

- `k = 5` should still go left
- `k = 6` should go right with adjusted index `1`

The implementation correctly uses:

```
if k <= left_length:
```

This guarantees that the last character of the left subtree is handled correctly.

### Internal Nodes with Empty Strings

Internal nodes always contain empty strings. A naive implementation might accidentally attempt to read characters from `node.val` at internal nodes.

This solution never accesses `node.val` unless the node is confirmed to be a leaf node, preventing incorrect indexing or empty-string access.
