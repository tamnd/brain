# LeetCode 2619 - Array Prototype Last

## Problem Understanding

This problem asks us to extend the built in JavaScript `Array` object by adding a new method named `last()`. Once implemented, every array should be able to call this method directly.

The method must return the final element of the array when at least one element exists. If the array is empty, the method should return `-1`.

The important detail is that we are not writing a standalone helper function. Instead, we are modifying the `Array.prototype`, which means all array instances inherit the method automatically.

For example, after implementing the prototype method:

```
const nums = [1, 2, 3];
nums.last(); // returns 3
```

If the array has no elements:

```
const nums = [];
nums.last(); // returns -1
```

The constraints are very small, since array length is at most 1000. Performance is not a concern here. The main challenge is understanding how JavaScript prototypes work and how methods become available on all arrays.

Several edge cases are important:

- Empty arrays must return `-1`
- Arrays can contain any valid JSON values, including `null`, objects, arrays, booleans, and numbers
- The last element itself may be falsy, such as `0`, `false`, or `null`, so we must not accidentally treat these as missing values
- The implementation should work for all arrays globally after prototype modification

## Approaches

### Brute Force Approach

A straightforward solution is to iterate through the entire array and keep updating a variable with the current value. After reaching the end, the variable will contain the last element.

This works correctly because the final visited element is the array's last element. However, it unnecessarily scans the entire array even though JavaScript arrays support direct indexing.

For an empty array, we would return `-1`.

Although this approach is correct, it performs extra work and ignores the fact that arrays provide constant time access to elements by index.

### Optimal Approach

The key observation is that the last element of an array is always located at index `length - 1`.

JavaScript arrays support direct random access, so we can retrieve the final element immediately without iteration.

The algorithm becomes:

- If the array length is `0`, return `-1`
- Otherwise, return `this[this.length - 1]`

This is both simpler and more efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Iterates through the entire array |
| Optimal | O(1) | O(1) | Directly accesses the final index |

## Algorithm Walkthrough

1. Extend `Array.prototype` by defining a new method called `last`.
2. Inside the method, check whether the array is empty by examining `this.length`.
3. If the length is `0`, return `-1`. This satisfies the problem requirement for empty arrays.
4. Otherwise, compute the last index as `this.length - 1`.
5. Return the element stored at that index.

The reason this works efficiently is that JavaScript arrays provide constant time indexed access.

### Why it works

An array of length `n` always stores its final element at index `n - 1`. Therefore, if the array is non empty, accessing `this[this.length - 1]` always retrieves the correct value. If the array is empty, no valid index exists, so returning `-1` satisfies the specification.

## Python Solution

LeetCode 2619 is a JavaScript specific problem because Python does not use JavaScript style prototypes. Therefore, there is no meaningful Python submission for the official problem.

For educational purposes, the equivalent behavior in Python can be implemented as a helper function.

```python
from typing import Any, List

class Solution:
    def last(self, arr: List[Any]) -> Any:
        if not arr:
            return -1
        
        return arr[-1]
```

This Python version mirrors the same logic:

- Return `-1` for empty lists
- Otherwise, return the final element using negative indexing

The implementation is concise because Python already supports direct access to the final element through `arr[-1]`.

## Go Solution

LeetCode 2619 is also JavaScript specific, so Go does not have an equivalent prototype system. However, the same functionality can be represented with a helper function.

```go
package main

func last(arr []interface{}) interface{} {
    if len(arr) == 0 {
        return -1
    }

    return arr[len(arr)-1]
}
```

In Go, slices do not support prototype extension like JavaScript arrays. Instead, we create a regular helper function.

The implementation checks whether the slice length is zero. If so, it returns `-1`. Otherwise, it accesses the final element using `len(arr) - 1`.

Unlike JavaScript, Go requires explicit handling of heterogeneous values through `interface{}`.

## JavaScript Solution

Since this is the actual language used by the LeetCode problem, this is the real submission solution.

```java
Array.prototype.last = function() {
    if (this.length === 0) {
        return -1;
    }

    return this[this.length - 1];
};
```

This implementation modifies `Array.prototype`, making the method available to every array instance automatically.

## Edge Cases

One important edge case is the empty array. Attempting to access index `-1` directly in JavaScript would return `undefined`, but the problem explicitly requires `-1`. The implementation handles this by checking `this.length === 0` before accessing any element.

Another subtle case involves falsy values such as `0`, `false`, or `null`. A buggy implementation might incorrectly treat these values as missing if it uses truthiness checks. This solution avoids that problem entirely because it only checks the array length, not the value itself.

A third important edge case is arrays containing complex JSON values such as objects or nested arrays. Since the implementation simply returns the stored element without modification, it correctly handles all valid JSON types.
