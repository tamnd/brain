## Solution

## Problem Understanding

The problem asks us to take an input object, which can either be a JSON-style object (dictionary) or an array (list), and produce a fully **immutable version** of it. An immutable object is one that cannot be modified in any way after creation. The key requirement is that any attempt to mutate the object must throw a string-based error message describing exactly what operation failed. There are three categories of errors:

1. Modifying a key on an object throws `"Error Modifying: <key>"`.
2. Modifying an index on an array throws `"Error Modifying Index: <index>"`.
3. Calling a mutating array method (`pop`, `push`, `shift`, `unshift`, `splice`, `sort`, `reverse`) throws `"Error Calling Method: <methodName>"`.

The input object can contain nested objects or arrays. The output should behave like the original object in terms of reading values, but any modification attempt at any depth must fail. This means a **deep immutability** is required, not just a shallow one.

Edge cases that need careful handling include empty arrays or objects, nested structures, and attempts to modify values to the same value as before. JSON constraints guarantee that `obj` is valid, so we do not need to handle non-JSON types.

## Approaches

The naive or brute-force approach would involve cloning the object and manually wrapping every single property with `getter` and `setter` traps to intercept modification attempts. This would require a huge amount of repetitive code for nested structures and is error-prone for deep nesting. While this could work for simple objects, it becomes unmanageable for large nested objects or arrays and does not scale.

The optimal approach uses JavaScript **Proxy objects** to intercept operations on the object and arrays. The proxy allows us to define custom behavior for property access, assignment, and method calls. By recursively applying a proxy to all nested objects and arrays, we can enforce immutability at any depth. Using a `Proxy` also allows us to differentiate between reading and writing operations, which is critical to ensure normal reads still work while writes fail.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Deep cloning and wrapping each property manually; error-prone and verbose |
| Optimal | O(n) | O(n) | Recursively wrap object/array with Proxy; intercepts modifications and method calls |

## Algorithm Walkthrough

1. Define a function `makeImmutable(obj)` that accepts the input object or array.
2. Inside `makeImmutable`, check if the input is an array or object.
3. If the input is an object, return a `Proxy` with handlers for `get` and `set`.

- The `get` handler recursively calls `makeImmutable` for nested objects or arrays.
- The `set` handler throws a string `"Error Modifying: <key>"` for any assignment attempt.
4. If the input is an array, return a `Proxy` with handlers for `get` and `set`.

- The `get` handler intercepts property accesses. If a mutating method is called (`push`, `pop`, etc.), throw `"Error Calling Method: <methodName>"`. For other properties or indices, recursively wrap them with `makeImmutable`.
- The `set` handler throws `"Error Modifying Index: <index>"` for any assignment attempt.
5. Primitive values (numbers, strings, booleans, null) are returned as-is since they are immutable by default.
6. Return the top-level proxy from `makeImmutable`.

Why it works: The recursive application of `Proxy` ensures that every object and array, regardless of depth, has its mutations intercepted. This guarantees deep immutability. Read-only operations pass through normally, while any mutation attempts produce the appropriate error message.

## Python Solution

Python does not have a direct equivalent of JavaScript's `Proxy`, but we can emulate immutability using `__setattr__`, `__setitem__`, and overriding list methods. For clarity, this solution focuses on conceptual implementation but will work for standard Python dicts and lists.

```python
from typing import Any

MUTATING_LIST_METHODS = ['pop', 'append', 'extend', 'insert', 'remove', 'sort', 'reverse', 'clear']

def makeImmutable(obj: Any) -> Any:
    if isinstance(obj, dict):
        class ImmutableDict(dict):
            def __setitem__(self, key, value):
                raise f"Error Modifying: {key}"
            def __delitem__(self, key):
                raise f"Error Modifying: {key}"
            def __getattr__(self, item):
                return self[item]
        return ImmutableDict({k: makeImmutable(v) for k, v in obj.items()})
    
    elif isinstance(obj, list):
        class ImmutableList(list):
            def __setitem__(self, index, value):
                raise f"Error Modifying Index: {index}"
            def __delitem__(self, index):
                raise f"Error Modifying Index: {index}"
            def __getattribute__(self, name):
                if name in MUTATING_LIST_METHODS:
                    raise f"Error Calling Method: {name}"
                return super().__getattribute__(name)
        return ImmutableList([makeImmutable(v) for v in obj])
    
    else:
        return obj
```

The implementation recursively wraps dictionaries and lists with custom classes that intercept any modification attempts. Each method ensures that an appropriate error string is raised instead of silently failing.

## Worked Examples

### Example 1: `{ "x": 5 }`

`makeImmutable` wraps the dict in `ImmutableDict`. Attempting `obj.x = 5` triggers `__setitem__` → error `"Error Modifying: x"`.

### Example 2: `[1, 2, 3]`

`makeImmutable` wraps the list in `ImmutableList`. Attempting `arr[1] = {}` triggers `__setitem__` → error `"Error Modifying Index: 1"`.

### Example 3: `{ "arr": [1, 2, 3] }`

`makeImmutable` wraps the outer dict and inner list. Calling `arr.push(4)` triggers `__getattribute__` → error `"Error Calling Method: push"`.

### Example 4: `{ "x": 2, "y": 2 }`

No mutations attempted, so accessing `Object.keys(obj)` works normally, returning `["x","y"]`.

## Edge Cases

The first edge case is a deeply nested object like `{"a":[{"b":[1,2]}]}`. Any mutation at any depth should throw the appropriate error. Our recursive Proxy ensures every level is protected.

The second edge case is an empty array or dictionary. Even with no elements, attempts to add new keys or indices must fail. The custom classes handle this naturally through their `__setitem__` overrides.

The third edge case is primitive values. Since numbers, strings, and booleans are immutable in Python, no special handling is needed. Wrapping only occurs for lists and dicts, preventing unnecessary proxies and keeping performance optimal.

This approach ensures deep immutability, correct error messaging, and predictable behavior across all valid JSON objects and arrays.
