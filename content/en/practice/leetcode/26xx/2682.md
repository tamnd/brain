## Solution

## Problem Understanding

The problem describes a game played by `n` friends sitting in a circle, numbered from `1` to `n` in clockwise order. The game starts with the first friend holding a ball, and the ball is passed around according to a growing step pattern: on the `i-th` turn, the ball moves `i * k` steps clockwise from the current holder. The game ends as soon as a friend receives the ball for a second time. The challenge is to identify the friends who **never received the ball** during the game and return them in ascending order.

The inputs are two integers: `n` representing the number of friends, and `k` representing the step increment. The constraints `1 <= k <= n <= 50` indicate that the input size is small, so solutions do not need to scale to very large `n`. Important edge cases include when `k` is equal to `n` (the ball may immediately return to the starting friend) and when `k = 1` (the ball moves incrementally, hitting everyone before repeating).

The output is an array of integers listing all friends who never received the ball, sorted in ascending order.

## Approaches

A **brute-force approach** simulates the game exactly as described. Starting from friend `1`, we repeatedly compute the next friend to receive the ball using `(current_position + step_size) % n`, keeping track of each friend who has received the ball in a set or array. We increment the step multiplier each turn until we detect a repeat. While correct and easy to implement, this approach is already efficient due to the small constraints, but we can still describe an **optimal observation**.

The key observation is that the number of steps is cumulative and modular arithmetic naturally handles circular movement. We only need to track which friends have received the ball. The optimal approach still uses a set to track visited friends and directly computes the next position using modulo arithmetic, terminating when a repeat is detected.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Simulates each pass until a friend receives the ball twice, tracking visited friends |
| Optimal | O(n) | O(n) | Same as brute force due to small constraints; uses modulo to handle circular movement efficiently |

## Algorithm Walkthrough

1. Initialize a set `visited` to store all friends who have received the ball. Add the first friend (`1`) to this set.
2. Initialize `current` to `0` to represent the 0-based index of the current friend.
3. Initialize a counter `i` to `1`, representing the turn number.
4. While the next friend has not been visited:

1. Compute the next friend using `(current + i * k) % n`.
2. Check if the next friend is already in `visited`. If yes, break the loop because the game ends.
3. Add the next friend to `visited`.
4. Update `current` to the next friend.
5. Increment `i` by 1 for the next turn.
5. After the loop, iterate through all friends `1` to `n`. If a friend is not in `visited`, add them to the `losers` array.
6. Return the `losers` array sorted in ascending order.

**Why it works:** Using modulo arithmetic ensures circular movement around the friends. The invariant is that the set `visited` always contains all friends who have received the ball, so those missing from it at the end are precisely the losers.

## Python Solution

```python
from typing import List

class Solution:
    def circularGameLosers(self, n: int, k: int) -> List[int]:
        visited = set()
        current = 0  # 0-based index
        visited.add(current)
        i = 1
        
        while True:
            next_friend = (current + i * k) % n
            if next_friend in visited:
                break
            visited.add(next_friend)
            current = next_friend
            i += 1
        
        losers = [friend + 1 for friend in range(n) if friend not in visited]
        return losers
```

The implementation uses a set to track all friends who have received the ball. Each iteration calculates the next friend using modular arithmetic, ensuring we correctly wrap around the circle. After the game ends, the list comprehension collects all friends not in the `visited` set and returns them in ascending order.

## Go Solution

```go
func circularGameLosers(n int, k int) []int {
    visited := make(map[int]bool)
    current := 0
    visited[current] = true
    i := 1

    for {
        next := (current + i*k) % n
        if visited[next] {
            break
        }
        visited[next] = true
        current = next
        i++
    }

    losers := []int{}
    for friend := 0; friend < n; friend++ {
        if !visited[friend] {
            losers = append(losers, friend+1)
        }
    }
    return losers
}
```

In Go, we use a map instead of a set to track visited friends. Slices are used to collect the losers, and the `+1` adjustment converts from 0-based indexing to 1-based friend numbers. The algorithm is structurally identical to the Python version.

## Edge Cases

**Single Friend (`n = 1`)**: Only one friend exists, and the game ends immediately. Our implementation correctly returns an empty list since there are no losers.

**Step Equals Number of Friends (`k = n`)**: The ball returns immediately to the first friend. The algorithm correctly identifies all other friends as losers.

**Small `k` (`k = 1`)**: Each pass moves only one friend ahead. This requires multiple passes to hit everyone, and the set ensures all visited friends are tracked accurately before stopping.

These edge cases cover the scenarios that could cause off-by-one errors or incorrect modulo calculations. The modulo arithmetic ensures circular movement is handled consistently.
