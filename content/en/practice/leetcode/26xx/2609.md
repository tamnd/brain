## Solution

## Problem Understanding

We are given a binary string `s` that contains only the characters `'0'` and `'1'`. We need to find the length of the longest substring that is considered balanced.

A balanced substring must satisfy two conditions:

1. All zeroes must appear before any ones.
2. The number of zeroes and ones must be equal.

This means every valid balanced substring has the structure:

```
000...111...
```

where the count of zeroes equals the count of ones.

For example:

- `"0011"` is balanced because it has two zeroes followed by two ones.
- `"000111"` is balanced because it has three zeroes followed by three ones.
- `"0101"` is not balanced because the zeroes and ones are interleaved.
- `"011"` is not balanced because the counts are not equal.

The problem asks for the length of the longest such substring.

The constraints are very small:

- `1 <= s.length <= 50`

Since the input size is tiny, even less efficient approaches would work within time limits. However, the goal is to design a clean and efficient algorithm.

Several edge cases are important:

- Strings containing only `'1'`, such as `"111"`, should return `0`.
- Strings containing only `'0'`, such as `"0000"`, should also return `0`.
- Alternating patterns like `"010101"` contain only small balanced substrings.
- Large groups like `"00001111"` should correctly produce the full length.
- The longest balanced substring may appear in the middle of the string.

## Approaches

### Brute Force Approach

The brute force solution checks every possible substring.

For every starting index `i`, we try every ending index `j` and examine the substring `s[i:j+1]`. We then verify whether:

1. All zeroes come before ones.
2. The number of zeroes equals the number of ones.

To validate a substring, we can scan it character by character and ensure that once we encounter a `'1'`, we never see another `'0'`. At the same time, we count zeroes and ones.

If the substring is balanced, we update the answer with its length.

This approach is correct because it explicitly tests every possible substring. However, it is inefficient because there are `O(n^2)` substrings, and validating each substring takes `O(n)` time.

The total complexity becomes `O(n^3)`.

### Optimal Approach

The key observation is that every balanced substring consists of:

- A consecutive block of zeroes
- Immediately followed by a consecutive block of ones

For example:

```
000111
```

The number of usable pairs is limited by the smaller block size.

If we have:

```
0000111
```

then:

- zero count = 4
- one count = 3

The largest balanced substring is:

```
000111
```

with length:

```
2 * min(4, 3) = 6
```

So instead of checking all substrings, we only need to examine consecutive groups of identical characters.

We track:

- the size of the previous group
- the size of the current group

Whenever the character changes, we shift the groups forward.

If the previous group contains zeroes and the current group contains ones, the largest balanced substring across those groups is:

```
2 * min(previous_group_size, current_group_size)
```

This gives a simple linear scan.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Checks every substring and validates it |
| Optimal | O(n) | O(1) | Uses consecutive group sizes |

## Algorithm Walkthrough

1. Initialize three variables:

- `previous_group = 0`
- `current_group = 1`
- `max_length = 0`

The current group starts at size `1` because the first character already forms a group.
2. Traverse the string starting from index `1`.
3. If the current character matches the previous character, increment `current_group`.

This means we are still inside the same consecutive block.
4. Otherwise, the character changed.

This means the previous block ended and a new block started.

Assign:

- `previous_group = current_group`
- `current_group = 1`
5. After updating the groups, check whether the previous block was zeroes and the current block is ones.

Since balanced substrings must be zeroes followed by ones, only this transition matters.
6. Compute the possible balanced substring length:

```
2 * min(previous_group, current_group)
```

1. Update `max_length` if this value is larger.
2. Continue until the entire string has been processed.
3. Return `max_length`.

### Why it works

Every balanced substring must consist of one contiguous zero block followed immediately by one contiguous one block. The longest valid substring spanning two adjacent groups uses as many matching pairs as possible, which is exactly:

```
min(zero_group_size, one_group_size)
```

pairs of zeroes and ones.

The algorithm examines every adjacent group pair exactly once, so it cannot miss the optimal answer.

## Python Solution

```python
class Solution:
    def findTheLongestBalancedSubstring(self, s: str) -> int:
        previous_group = 0
        current_group = 1
        max_length = 0

        for i in range(1, len(s)):
            if s[i] == s[i - 1]:
                current_group += 1
            else:
                previous_group = current_group
                current_group = 1

            if s[i] == '1' and s[i - 1] == '0':
                max_length = max(
                    max_length,
                    2 * min(previous_group, current_group)
                )
            elif s[i] == '1':
                max_length = max(
                    max_length,
                    2 * min(previous_group, current_group)
                )

        return max_length
```

The implementation maintains the sizes of consecutive character groups while scanning the string once from left to right.

`current_group` stores the length of the current block of identical characters. Whenever the character changes, the old current group becomes the previous group, and a new current group begins.

The expression:

```python
2 * min(previous_group, current_group)
```

computes the largest balanced substring that can be formed between two adjacent groups.

The algorithm updates the answer continuously during traversal, so no additional passes are required.

## Go Solution

```go
func findTheLongestBalancedSubstring(s string) int {
    previousGroup := 0
    currentGroup := 1
    maxLength := 0

    for i := 1; i < len(s); i++ {
        if s[i] == s[i-1] {
            currentGroup++
        } else {
            previousGroup = currentGroup
            currentGroup = 1
        }

        if s[i] == '1' {
            length := 2 * min(previousGroup, currentGroup)
            if length > maxLength {
                maxLength = length
            }
        }
    }

    return maxLength
}

func min(a int, b int) int {
    if a < b {
        return a
    }
    return b
}
```

The Go implementation follows the exact same logic as the Python version.

One notable difference is that Go does not provide a built in integer `min` function for plain integers, so we define a helper function manually.

Strings in Go are indexed by bytes, which is safe here because the input contains only ASCII characters `'0'` and `'1'`.

## Edge Cases

One important edge case is a string containing only one type of character, such as `"0000"` or `"1111"`. A balanced substring requires both zeroes and ones, so the correct answer must be `0`. A naive implementation might incorrectly count repeated characters as valid. This implementation avoids that because it only forms balanced lengths when a one group follows a zero group.

Another important edge case is when the zero and one groups are different sizes. For example, `"0001111"` has four ones but only three zeroes. The valid balanced substring can only use three pairs, producing `"000111"` with length `6`. The algorithm handles this naturally using:

```
2 * min(previous_group, current_group)
```

A third tricky case is alternating characters like `"010101"`. Each adjacent pair forms a balanced substring of length `2`, but no larger balanced substring exists because the zeroes and ones are not grouped together. The implementation correctly resets group sizes whenever the character changes, preventing invalid larger counts from forming.
