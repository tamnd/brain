## Solution

## Problem Understanding

This problem asks us to compute the arrival time of a train after it has been delayed. The input is two positive integers: `arrivalTime`, representing the initial arrival time in hours on a 24-hour clock, and `delayedTime`, representing the delay in hours. The task is to calculate the new arrival time in 24-hour format.

The key points are that the time should wrap around the 24-hour clock. For example, if the train is supposed to arrive at hour 23 and is delayed by 2 hours, the arrival time should be `(23 + 2) % 24 = 1`. The constraints are small, since `arrivalTime` is between 1 and 23, and `delayedTime` is between 1 and 24, so performance is not a concern. Important edge cases include situations where the sum equals exactly 24, or the delay pushes the time past midnight.

## Approaches

The simplest approach is to simulate the passage of each hour, incrementing the `arrivalTime` by 1 repeatedly until the total delay is consumed. This works correctly but is unnecessary because a mathematical observation allows a constant-time solution: since the clock wraps every 24 hours, the new arrival time is simply `(arrivalTime + delayedTime) % 24`. This formula automatically handles all wrapping and the edge case when the sum equals 24.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force Increment | O(delayedTime) | O(1) | Increment `arrivalTime` by 1 repeatedly `delayedTime` times |
| Optimal Modulo | O(1) | O(1) | Use `(arrivalTime + delayedTime) % 24` to handle wrapping |

## Algorithm Walkthrough

1. Take the initial arrival time, `arrivalTime`.
2. Take the delay time, `delayedTime`.
3. Compute the sum of `arrivalTime` and `delayedTime`.
4. Apply modulo 24 to the sum: `(arrivalTime + delayedTime) % 24`. This ensures that if the sum is 24 or more, it wraps around the 24-hour clock.
5. Return the result as the new arrival time.

Why it works: The modulo operation directly simulates the behavior of a 24-hour clock. Any sum of hours greater than 24 will automatically wrap around, producing the correct arrival time in 24-hour format.

## Python Solution

```python
class Solution:
    def findDelayedArrivalTime(self, arrivalTime: int, delayedTime: int) -> int:
        return (arrivalTime + delayedTime) % 24
```

This Python solution simply applies the modulo operation to compute the arrival time after delay. The addition handles the delay, and the modulo ensures wrapping in the 24-hour format. This implementation is concise, direct, and handles all edge cases automatically.

## Go Solution

## Worked Examples

**Example 1:** arrivalTime = 15, delayedTime = 5

Step 1: Compute `15 + 5 = 20`

Step 2: Apply modulo 24: `20 % 24 = 20`

Output: 20

**Example 2:** arrivalTime = 13, delayedTime = 11

Step 1: Compute `13 + 11 = 24`

Step 2: Apply modulo 24: `24 % 24 = 0`

Output: 0

**Example 3:** arrivalTime = 23, delayedTime = 2

Step 1: Compute `23 + 2 = 25`

Step 2: Apply modulo 24: `25 % 24 = 1`

Output: 1

## Edge Cases

An important edge case is when the sum of `arrivalTime` and `delayedTime` equals exactly 24. Without modulo, one might incorrectly return 24 instead of 0. Another edge case is when the delay pushes the time past midnight, e.g., arrivalTime = 23 and delayedTime = 2; using modulo ensures correct wrapping to 1. A third edge case is a full-day delay, such as arrivalTime = 1 and delayedTime = 24, where the modulo ensures the arrival time remains unchanged at 1. All of these edge cases are correctly handled by the modulo approach.
