## Solution

## Problem Understanding

The problem asks us to compute a special value called the punishment number for a given integer `n`.

For every integer `i` in the range `[1, n]`, we calculate `i * i`. Then we check whether the decimal representation of that square can be split into contiguous pieces whose integer values add up to `i`.

If such a partition exists, then `i` is considered valid, and we add `i * i` to the final answer.

The key detail is that the substrings must remain contiguous. We are not allowed to rearrange digits. We can only choose where to cut the string.

For example:

- `9 * 9 = 81`
- We can partition `"81"` into `"8"` and `"1"`
- `8 + 1 = 9`
- Therefore, `9` contributes `81` to the punishment number

Another example:

- `36 * 36 = 1296`
- We can partition `"1296"` into `"1"`, `"29"`, and `"6"`
- `1 + 29 + 6 = 36`
- Therefore, `36` contributes `1296`

The input consists of a single integer `n`, where:

- `1 <= n <= 1000`

This constraint is important because it tells us the search space is relatively small. Even a recursive backtracking solution is practical since the largest square is:

$$1000^2 = 1000000$$

This has only 7 digits, so the number of possible partitions remains manageable.

Several edge cases are important:

- Single digit numbers like `1`
- Squares containing zeros like `100`
- Cases where many partition combinations exist
- Cases where no valid partition exists
- Maximum constraint values like `n = 1000`

A naive implementation that generates all partitions inefficiently could still work because the input size is small, but we should still design the recursion carefully.

## Approaches

### Brute Force Approach

The brute force idea is straightforward.

For every integer `i` from `1` to `n`:

1. Compute `square = i * i`
2. Convert the square into a string
3. Generate every possible way to partition the string
4. Compute the sum of each partition
5. If any partition sum equals `i`, add `square` to the answer

A string of length `m` has `2^(m-1)` possible partition patterns because each gap between digits can either contain a cut or not.

For example, `"1296"` has these partition possibilities:

- `1 | 2 | 9 | 6`
- `1 | 2 | 96`
- `1 | 29 | 6`
- `12 | 9 | 6`
- `129 | 6`
- `12 | 96`
- `1 | 296`
- `1296`

This brute force method is correct because it checks every possible valid partitioning.

However, explicitly generating all partitions and constructing substring arrays repeatedly is inefficient and unnecessarily expensive.

### Optimal Approach

The better solution uses recursive backtracking.

Instead of generating all partitions first and evaluating them later, we build partitions incrementally while tracking the current accumulated sum.

At each position in the string:

- Extend the current substring
- Convert it into a number
- Add it to the running total
- Recursively process the remaining digits

If we reach the end of the string and the total equals `i`, then the square is valid.

The important optimization is pruning:

- If the running sum already exceeds `i`, we can stop immediately
- No future partition can reduce the sum

Because the maximum square length is very small, this recursive solution runs comfortably within limits.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × 2^m × m) | O(m) | Explicitly generates all partitions |
| Optimal | O(n × 2^m) | O(m) | Backtracking with pruning |

Here, `m` is the number of digits in `i²`, which is at most 7.

## Algorithm Walkthrough

### Step 1: Initialize the Answer

Create a variable `total` to store the punishment number.

We will iterate through every integer from `1` to `n`.

### Step 2: Compute the Square

For each integer `i`:

- Compute `square = i * i`
- Convert the square into a string so we can partition digits easily

### Step 3: Start Recursive Partition Checking

We define a recursive function with:

- Current index in the string
- Current accumulated sum

The recursion explores every valid partition.

### Step 4: Build Numbers Incrementally

From the current index:

- Extend the substring digit by digit
- Convert it into an integer
- Add it to the running sum
- Recurse on the remaining suffix

For example, with `"1296"`:

Starting at index `0`:

- `"1"`
- `"12"`
- `"129"`
- `"1296"`

Each becomes a candidate partition segment.

### Step 5: Apply Pruning

If the running sum exceeds `i`, stop exploring that path immediately.

This avoids unnecessary recursion.

### Step 6: Check Completion

When we reach the end of the string:

- If the sum equals `i`, return `True`
- Otherwise return `False`

### Step 7: Add Valid Squares

If recursion returns `True`, add `square` to the answer.

### Step 8: Return the Final Sum

After processing all integers, return the accumulated punishment number.

### Why it works

The recursion explores every possible contiguous partition of the square's digit string. Every partition corresponds to a unique sequence of cut positions. Since all possibilities are explored, no valid partition can be missed.

The pruning step is safe because all partition values are non-negative. Once the running sum exceeds `i`, no future additions can bring it back down.

Therefore, the algorithm correctly identifies exactly the integers that satisfy the condition.

## Python Solution

```python
class Solution:
    def punishmentNumber(self, n: int) -> int:
        def can_partition(num_str: str, target: int, index: int, current_sum: int) -> bool:
            if index == len(num_str):
                return current_sum == target

            if current_sum > target:
                return False

            value = 0

            for end in range(index, len(num_str)):
                value = value * 10 + int(num_str[end])

                if can_partition(
                    num_str,
                    target,
                    end + 1,
                    current_sum + value
                ):
                    return True

            return False

        total = 0

        for i in range(1, n + 1):
            square = i * i

            if can_partition(str(square), i, 0, 0):
                total += square

        return total
```

The implementation closely follows the recursive backtracking algorithm.

The helper function `can_partition` performs the partition search.

The parameters are:

- `num_str`, the string representation of the square
- `target`, the original number `i`
- `index`, the current digit position
- `current_sum`, the accumulated partition sum

The base case occurs when we reach the end of the string. At that point, we verify whether the accumulated sum equals the target.

The pruning condition checks whether the current sum already exceeds the target.

Inside the loop, we progressively build the next partition number using:

```python
value = value * 10 + int(num_str[end])
```

This avoids repeatedly slicing substrings and converting them to integers.

For each possible partition endpoint, recursion explores the remaining suffix.

The outer loop processes all integers from `1` through `n`, adding valid squares to the answer.

## Go Solution

```go
func punishmentNumber(n int) int {
    var canPartition func(string, int, int, int) bool

    canPartition = func(numStr string, target int, index int, currentSum int) bool {
        if index == len(numStr) {
            return currentSum == target
        }

        if currentSum > target {
            return false
        }

        value := 0

        for end := index; end < len(numStr); end++ {
            value = value*10 + int(numStr[end]-'0')

            if canPartition(numStr, target, end+1, currentSum+value) {
                return true
            }
        }

        return false
    }

    total := 0

    for i := 1; i <= n; i++ {
        square := i * i

        if canPartition(fmt.Sprintf("%d", square), i, 0, 0) {
            total += square
        }
    }

    return total
}
```

The Go implementation follows the same recursive structure as the Python version.

One important difference is digit conversion. In Go, characters are bytes, so digit extraction uses:

```go
int(numStr[end] - '0')
```

The recursive helper is declared as a function variable because Go requires recursive anonymous functions to be declared before assignment.

The solution uses `fmt.Sprintf` to convert integers into strings. In production code, `strconv.Itoa` would also work efficiently.

Go integers are fully sufficient because the maximum punishment number fits comfortably within 32-bit integer range.

At worst, each number explores roughly 128 recursive partition states, making the solution extremely efficient in practice.

## Edge Cases

One important edge case is numbers whose squares contain zeros, such as `10² = 100`. Incorrect implementations sometimes reject partitions with leading or trailing zeros. This solution handles them naturally because partitions are formed directly from contiguous substrings, and integer conversion correctly interprets `"0"` as zero.

Another important case is single-digit squares. For example, `1² = 1`. The recursion must correctly recognize that a string with no cuts is still a valid partition. The base case naturally handles this because the entire string can be treated as one segment.

A third edge case involves partitions that temporarily exceed the target value. For example, if the running sum becomes larger than `i`, continuing recursion would be wasteful. The pruning condition avoids unnecessary work while remaining correct because all future partition values are non-negative.

Finally, the maximum constraint `n = 1000` is important because it confirms recursion depth remains small. The largest square has only 7 digits, so recursive exploration remains efficient and safe within stack limits.
