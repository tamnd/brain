# LeetCode 2611 - Mice and Cheese

## Problem Understanding

The problem describes a scenario with two mice and several different cheeses. Every cheese must be eaten by exactly one mouse, and each mouse earns a different number of points depending on which cheese it eats.

For every cheese at index `i`:

- If mouse 1 eats it, the total score increases by `reward1[i]`
- If mouse 2 eats it, the total score increases by `reward2[i]`

The important restriction is that the first mouse must eat exactly `k` cheeses. All remaining cheeses are automatically assigned to the second mouse.

The goal is to maximize the total combined reward.

In other words, we need to decide which `k` cheeses should go to mouse 1 so that the total score becomes as large as possible.

The constraints are important:

- `n` can be as large as `10^5`
- A brute-force search over all combinations is impossible
- We need an algorithm close to `O(n log n)` or better

The problem guarantees:

- Both reward arrays have the same length
- All rewards are positive integers
- `k` is always valid, meaning `0 <= k <= n`

Several edge cases immediately stand out:

- `k = 0`, meaning mouse 1 eats nothing
- `k = n`, meaning mouse 1 eats everything
- Situations where both mice give identical rewards
- Cases where assigning a cheese to mouse 1 is actually much worse than assigning it to mouse 2

These cases are important because they help define the optimal greedy strategy.

## Approaches

### Brute Force Approach

A straightforward solution is to try every possible choice of `k` cheeses for mouse 1.

For every combination:

1. Assign the chosen cheeses to mouse 1
2. Assign the remaining cheeses to mouse 2
3. Compute the total reward
4. Keep the maximum result

This approach is correct because it explores every valid assignment.

However, the number of possible assignments is:

$$\binom{n}{k}$$

This grows exponentially large. With `n = 10^5`, brute force is completely infeasible.

### Optimal Greedy Observation

The key insight is to think about the problem differently.

Suppose every cheese is initially assigned to mouse 2. Then the starting score is:

$$\text{base} = \sum reward2[i]$$

Now consider what happens if we reassign a cheese from mouse 2 to mouse 1.

The score changes by:

$$reward1[i] - reward2[i]$$

This value represents the gain, or loss, from letting mouse 1 eat that cheese instead of mouse 2.

Therefore:

- A large positive difference means mouse 1 should definitely eat that cheese
- A negative difference means mouse 2 should probably keep it

Since mouse 1 must eat exactly `k` cheeses, the optimal strategy is:

- Compute all differences
- Select the `k` largest differences
- Add those gains to the base score

This transforms the problem into a sorting and greedy selection problem.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(C(n, k) × n) | O(k) | Tries every possible assignment |
| Optimal Greedy | O(n log n) | O(n) | Sorts reward differences and picks best gains |

## Algorithm Walkthrough

1. Start by assuming every cheese is eaten by mouse 2.

Compute:

```
total = sum(reward2)
```

This gives a valid baseline score.
2. For every cheese, calculate the advantage of assigning it to mouse 1 instead of mouse 2.

The gain is:

```
reward1[i] - reward2[i]
```

A positive value means mouse 1 is better for that cheese.
3. Store all gain values in an array.
4. Sort the gains in descending order.

The largest gains represent the cheeses that most improve the score when reassigned to mouse 1.
5. Select the first `k` gains.

Since mouse 1 must eat exactly `k` cheeses, we choose the `k` best opportunities.
6. Add those gains to the baseline total.
7. Return the final result.

### Why it works

The algorithm works because every cheese contributes independently to the total score.

The baseline assumes all cheeses belong to mouse 2. Reassigning a cheese to mouse 1 changes the score by exactly:

$$reward1[i] - reward2[i]$$

Therefore, maximizing the total score reduces to selecting the `k` largest independent gains. Since each choice does not affect the others, a greedy strategy is optimal.

## Python Solution

```python
from typing import List

class Solution:
    def miceAndCheese(self, reward1: List[int], reward2: List[int], k: int) -> int:
        total_score = sum(reward2)

        gains = []

        for r1, r2 in zip(reward1, reward2):
            gains.append(r1 - r2)

        gains.sort(reverse=True)

        for i in range(k):
            total_score += gains[i]

        return total_score
```

The implementation begins by computing the score obtained when every cheese is assigned to mouse 2.

Next, the code computes the gain for switching each cheese to mouse 1. These gains are stored in a list.

The gains are then sorted in descending order so that the best assignments appear first.

Since mouse 1 must eat exactly `k` cheeses, the algorithm adds the top `k` gains to the baseline score.

Finally, the maximum total score is returned.

## Go Solution

```go
package main

import (
	"sort"
)

func miceAndCheese(reward1 []int, reward2 []int, k int) int {
	totalScore := 0
	n := len(reward1)

	gains := make([]int, n)

	for i := 0; i < n; i++ {
		totalScore += reward2[i]
		gains[i] = reward1[i] - reward2[i]
	}

	sort.Sort(sort.Reverse(sort.IntSlice(gains)))

	for i := 0; i < k; i++ {
		totalScore += gains[i]
	}

	return totalScore
}
```

The Go implementation follows the same logic as the Python version.

The primary difference is that Go requires explicit slice allocation and uses the `sort` package for descending sorting.

Integer overflow is not a concern because the maximum possible score is well within Go's `int` range.

The algorithm performs a single pass to compute gains and then sorts them. Since sorting requires `O(n log n)` time, that becomes the overall complexity. The additional memory usage comes from storing the gains array.

## Edge Cases

One important edge case occurs when `k = 0`. In this scenario, mouse 1 cannot eat any cheese, so every cheese must remain assigned to mouse 2. A buggy implementation might still attempt to select gains or access invalid indices. This implementation handles the case naturally because the loop that adds gains runs zero times.

Another important case is `k = n`. Here, mouse 1 must eat every cheese. The algorithm correctly handles this by adding all gains to the baseline score. Mathematically, this transforms the total from the sum of `reward2` values into the sum of `reward1` values.

A subtle edge case appears when many gains are negative. Since mouse 1 must eat exactly `k` cheeses, the algorithm may be forced to choose cheeses that reduce the score. A common mistake is to select only positive gains, which would violate the requirement. This implementation correctly sorts all gains and selects exactly `k` of them, even if some are negative.

Another useful edge case is when both reward arrays are identical. In that situation, every gain equals zero, meaning any assignment produces the same score. The algorithm still works correctly because sorting zeros changes nothing and adding them does not affect the total.
