## Solution

## Problem Understanding

The problem gives us an integer array `nums`, where each element represents the strength of a hero. We must consider every possible non-empty subset of heroes. For each subset, we compute its power using this formula:

$$\text{power} = (\max(\text{subset}))^2 \times \min(\text{subset})$$

The goal is to return the sum of the powers of all possible non-empty subsets, modulo:

$$10^9 + 7$$

A subset can contain any number of heroes from `1` up to `n`, and the order does not matter.

For example, if the subset is `[2, 4, 1]`, then:

- Maximum value is `4`
- Minimum value is `1`

So the power becomes:

$$4^2 \times 1 = 16$$

The constraints are very important:

- `nums.length` can be as large as `10^5`
- Each value can be as large as `10^9`

These constraints immediately tell us that enumerating all subsets is impossible. An array of size `10^5` has:

$$2^{100000}$$

possible subsets, which is astronomically large.

This means we need a mathematical or combinational optimization instead of brute force subset generation.

Several edge cases are important:

- Arrays with only one element
- Arrays where all elements are identical
- Arrays with very large values
- Arrays already sorted or reverse sorted
- Cases where the minimum and maximum are the same element

The problem guarantees:

- The array is non-empty
- All values are positive integers

Because all values are positive, sorting and contribution counting become much easier.

## Approaches

### Brute Force Approach

The most direct solution is to generate every non-empty subset.

For each subset:

1. Find the minimum element
2. Find the maximum element
3. Compute:

$$\max^2 \times \min$$
4. Add the result to the answer

This approach is correct because it explicitly evaluates the definition of power for every possible subset.

However, this is far too slow.

An array with `n` elements has:

$$2^n - 1$$

non-empty subsets.

Even for `n = 30`, this becomes infeasible. Since the constraint is `10^5`, brute force cannot work.

### Key Insight for the Optimal Solution

The key observation is that instead of thinking about subsets directly, we can think about how each element contributes as the maximum value.

Suppose we sort the array:

$$a_0 \le a_1 \le a_2 \le \dots \le a_{n-1}$$

Now consider element `a[i]` as the maximum of a subset.

If `a[i]` is the maximum, then:

- Every chosen element must come from indices `0..i`
- The subset must include `a[i]`
- Any subset of earlier elements may also be included

Now the question becomes:

What is the total contribution of all possible minimum values for subsets whose maximum is `a[i]`?

If we can efficiently compute:

$$\sum (\text{minimum values of valid subsets})$$

then we can multiply by:

$$a[i]^2$$

to get the total contribution from subsets where `a[i]` is the maximum.

This transforms the problem from exponential subset enumeration into a linear dynamic programming accumulation after sorting.

## Algorithm Walkthrough

### Step 1: Sort the Array

We first sort `nums` in non-decreasing order.

After sorting:

```
a[0] <= a[1] <= ... <= a[n-1]
```

Sorting allows us to process elements in increasing order and guarantee that when we process `a[i]`, it is the maximum of the subsets we are considering.

### Step 2: Define Contribution for Each Maximum

For each element `a[i]`, we want to compute the total contribution of all subsets where:

- `a[i]` is the maximum
- The subset contains `a[i]`

The power formula becomes:

$$a[i]^2 \times (\text{minimum of subset})$$

So we need the sum of all possible minimum values.

### Step 3: Observe How Minimum Values Appear

Suppose we are processing `a[i]`.

Any subset ending at `a[i]` looks like:

```sql
{a[i]} + any subset from a[0..i-1]
```

If `a[j]` is the minimum of such a subset, then:

- `a[j]` must be included
- Any elements between `j+1` and `i-1` may or may not be included

The number of such subsets is:

$$2^{i-j-1}$$

Therefore the total minimum contribution becomes:

$$a[i] + \sum_{j=0}^{i-1} a[j] \times 2^{i-j-1}$$

The standalone `a[i]` corresponds to the subset containing only `a[i]`.

### Step 4: Maintain the Running Weighted Sum

We define:

$$prefix = \sum_{j=0}^{i-1} a[j] \times 2^{i-j-1}$$

Then the contribution of `a[i]` is:

$$a[i]^2 \times (a[i] + prefix)$$

Now we need an efficient recurrence for `prefix`.

Suppose we move from index `i` to `i+1`.

The recurrence becomes:

$$newPrefix = 2 \times prefix + a[i]$$

Why?

Because every previous subset choice can either include or exclude the new element, doubling the number of combinations.

### Step 5: Accumulate the Answer

For each element:

1. Compute:

$$contribution = a[i]^2 \times (a[i] + prefix)$$
2. Add it to the answer
3. Update:

$$prefix = 2 \times prefix + a[i]$$

Take modulo at every step.

### Why It Works

The algorithm works because every subset has a unique maximum element after sorting.

When processing `a[i]`, we count exactly all subsets whose maximum is `a[i]`.

The `prefix` variable compactly stores the weighted contribution of all possible minimum elements from previous positions. The doubling recurrence correctly reflects that each earlier subset can either include or exclude newly processed elements.

Thus every valid subset is counted exactly once, and its contribution is computed correctly.

## Python Solution

```python
from typing import List

class Solution:
    def sumOfPower(self, nums: List[int]) -> int:
        MOD = 10**9 + 7

        nums.sort()

        answer = 0
        prefix = 0

        for value in nums:
            contribution = value * value
            contribution %= MOD

            contribution *= (value + prefix)
            contribution %= MOD

            answer += contribution
            answer %= MOD

            prefix = (2 * prefix + value) % MOD

        return answer
```

The implementation begins by sorting the array so that every element can be treated as the maximum of subsets formed from earlier elements.

The variable `prefix` stores the weighted sum of all possible minimum contributions from previously processed elements.

For each number:

1. We compute `value²`
2. Multiply it by `(value + prefix)`
3. Add the result to the final answer

The expression:

```python
(value + prefix)
```

represents the sum of all possible minimum values among subsets where `value` is the maximum.

After processing the current value, we update:

```python
prefix = 2 * prefix + value
```

This recurrence efficiently accounts for all future subset combinations.

Modulo arithmetic is applied throughout to avoid overflow and satisfy the problem requirement.

## Go Solution

```go
package main

import "sort"

func sumOfPower(nums []int) int {
	const MOD int64 = 1e9 + 7

	sort.Ints(nums)

	var answer int64 = 0
	var prefix int64 = 0

	for _, value := range nums {
		v := int64(value)

		contribution := (v * v) % MOD
		contribution = (contribution * ((v + prefix) % MOD)) % MOD

		answer = (answer + contribution) % MOD

		prefix = (2*prefix + v) % MOD
	}

	return int(answer)
}
```

The Go implementation follows the same logic as the Python version.

The main difference is careful handling of integer overflow. Since values can become extremely large, all arithmetic is performed using `int64`.

The built-in `sort.Ints` function is used for sorting.

Modulo operations are applied after every multiplication and accumulation step to keep numbers within range.

## Edge Cases

### Single Element Array

If the array contains only one element, there is exactly one subset.

For example:

```
[5]
```

The only subset is `[5]`, whose power is:

$$5^2 \times 5 = 125$$

This case is important because there are no previous elements contributing to `prefix`. The implementation handles this naturally because `prefix` starts at `0`.

### All Elements Identical

Consider:

```
[1,1,1]
```

Every subset has minimum `1` and maximum `1`, so every subset contributes `1`.

Since there are:

$$2^3 - 1 = 7$$

non-empty subsets, the answer is `7`.

Duplicate values can sometimes cause double-counting bugs in contribution-based algorithms, but this implementation works correctly because subsets are distinguished by indices, not by values.

### Very Large Numbers

Values may be as large as:

$$10^9$$

Squaring them produces numbers around:

$$10^{18}$$

Without modulo arithmetic at every step, overflow could occur.

The implementation carefully applies modulo after multiplication and accumulation operations. The Go solution additionally uses `int64` to safely hold intermediate values before modulo reduction.
