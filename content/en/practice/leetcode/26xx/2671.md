## Solution

## Problem Understanding

The problem asks us to design a data structure that supports three operations efficiently:

1. Add a number into the structure.
2. Remove one occurrence of a number.
3. Check whether any number currently appears exactly `frequency` times.

This is not a traditional array or list problem where we process a fixed input once. Instead, we are maintaining a dynamic multiset of integers over many operations.

The key challenge is the `hasFrequency(frequency)` operation. A naive implementation could scan all stored numbers and check their counts every time, but that would be too slow when there are up to `2 * 10^5` operations.

We need a structure that updates quickly when numbers are added or removed, while also answering frequency existence queries efficiently.

Suppose the current data structure contains:

```
[3, 3, 5, 5, 5]
```

Then:

- `3` appears 2 times
- `5` appears 3 times

So:

- `hasFrequency(2)` returns `true`
- `hasFrequency(3)` returns `true`
- `hasFrequency(1)` returns `false`

The important detail is that `hasFrequency(frequency)` only asks whether at least one number has that frequency. It does not ask which number it is.

The constraints are large enough that we need near constant time operations:

- Up to `2 * 10^5` total operations
- Values and frequencies up to `10^5`

This strongly suggests using hash maps for constant average time lookups and updates.

Several edge cases are important:

- Deleting a number that does not exist
- Frequencies becoming zero after deletion
- Multiple numbers sharing the same frequency
- A frequency disappearing entirely after updates
- Repeated additions and deletions of the same number

A correct implementation must carefully maintain frequency information during every update.

## Approaches

### Brute Force Approach

The simplest solution is to maintain a hash map:

```
number -> count
```

For example:

```
{
    3: 2,
    5: 3
}
```

The `add` and `deleteOne` operations are straightforward because we only need to update the count for one number.

However, `hasFrequency(frequency)` becomes expensive. To answer it, we would need to iterate through all stored numbers and check whether any count equals the requested frequency.

For example:

```
for count in number_count.values():
    if count == frequency:
        return True
```

This works correctly, but in the worst case there may be `O(n)` distinct numbers. Since we can perform up to `2 * 10^5` queries, the total runtime becomes too large.

### Optimal Approach

The key insight is that we should track not only the count of each number, but also how many numbers currently have a given frequency.

We maintain two hash maps:

1. `number_count`

Tracks the frequency of each number.

```
number -> frequency
```

1. `frequency_count`

Tracks how many numbers currently have a particular frequency.

```
frequency -> how many numbers have this frequency
```

Example:

```
number_count = {
    3: 2,
    5: 3
}

frequency_count = {
    2: 1,
    3: 1
}
```

Now `hasFrequency(frequency)` becomes extremely efficient:

```
return frequency_count[frequency] > 0
```

The challenge is correctly updating both maps whenever frequencies change.

When a number changes from frequency `f` to `f + 1`:

- Decrease `frequency_count[f]`
- Increase `frequency_count[f + 1]`

Similarly, when deleting:

- Decrease `frequency_count[f]`
- Increase `frequency_count[f - 1]` if `f - 1 > 0`

This allows every operation to run in constant average time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) per query | O(n) | `hasFrequency` scans all counts |
| Optimal | O(1) average per operation | O(n) | Uses two hash maps |

## Algorithm Walkthrough

1. Initialize two hash maps.

The first map stores the frequency of each number.

```
number_count[number] = frequency
```

The second map stores how many numbers currently have each frequency.

```
frequency_count[frequency] = amount
```
2. Handle `add(number)`.

First retrieve the current frequency of the number.

```
old_freq = number_count[number]
```

If the number already had a frequency greater than zero, we decrease the count of numbers having that old frequency.

Then increase the number's frequency by one.

Finally, increase the count of numbers having the new frequency.
3. Handle `deleteOne(number)`.

If the number does not exist or its frequency is zero, do nothing.

Otherwise:

- Decrease the count of numbers having the old frequency
- Reduce the number's frequency by one
- If the new frequency is still positive, increase the count for that new frequency
4. Handle `hasFrequency(frequency)`.

Simply check whether:

```
frequency_count[frequency] > 0
```

If true, at least one number appears exactly that many times.

### Why it works

The core invariant is:

```
frequency_count[f]
=
the number of distinct values whose frequency equals f
```

Every add or delete operation carefully removes the number from its old frequency group and inserts it into its new frequency group.

Because this invariant is always maintained, checking whether any number has a given frequency becomes a direct lookup.

## Python Solution

```python
from collections import defaultdict

class FrequencyTracker:

    def __init__(self):
        self.number_count = defaultdict(int)
        self.frequency_count = defaultdict(int)

    def add(self, number: int) -> None:
        old_frequency = self.number_count[number]

        if old_frequency > 0:
            self.frequency_count[old_frequency] -= 1

        new_frequency = old_frequency + 1

        self.number_count[number] = new_frequency
        self.frequency_count[new_frequency] += 1

    def deleteOne(self, number: int) -> None:
        old_frequency = self.number_count[number]

        if old_frequency == 0:
            return

        self.frequency_count[old_frequency] -= 1

        new_frequency = old_frequency - 1
        self.number_count[number] = new_frequency

        if new_frequency > 0:
            self.frequency_count[new_frequency] += 1

    def hasFrequency(self, frequency: int) -> bool:
        return self.frequency_count[frequency] > 0
```

The implementation directly follows the algorithm described earlier.

The constructor initializes two `defaultdict(int)` objects so missing keys automatically behave like zero.

Inside `add`, we first determine the current frequency of the number. If the number already existed, we remove one entry from the old frequency bucket. Then we increment the number's frequency and add it into the new frequency bucket.

The `deleteOne` method mirrors this process in reverse. If the number does not exist, we immediately return. Otherwise, we remove the number from its old frequency bucket, decrease its frequency, and if the new frequency is still positive, place it into the smaller frequency bucket.

The `hasFrequency` method becomes extremely simple because the second hash map already tracks exactly how many numbers have each frequency.

## Go Solution

```go
type FrequencyTracker struct {
	numberCount    map[int]int
	frequencyCount map[int]int
}

func Constructor() FrequencyTracker {
	return FrequencyTracker{
		numberCount:    make(map[int]int),
		frequencyCount: make(map[int]int),
	}
}

func (this *FrequencyTracker) Add(number int) {
	oldFrequency := this.numberCount[number]

	if oldFrequency > 0 {
		this.frequencyCount[oldFrequency]--
	}

	newFrequency := oldFrequency + 1

	this.numberCount[number] = newFrequency
	this.frequencyCount[newFrequency]++
}

func (this *FrequencyTracker) DeleteOne(number int) {
	oldFrequency := this.numberCount[number]

	if oldFrequency == 0 {
		return
	}

	this.frequencyCount[oldFrequency]--

	newFrequency := oldFrequency - 1
	this.numberCount[number] = newFrequency

	if newFrequency > 0 {
		this.frequencyCount[newFrequency]++
	}
}

func (this *FrequencyTracker) HasFrequency(frequency int) bool {
	return this.frequencyCount[frequency] > 0
}

/**
 * Your FrequencyTracker object will be instantiated and called as such:
 * obj := Constructor();
 * obj.Add(number);
 * obj.DeleteOne(number);
 * param_3 := obj.HasFrequency(frequency);
 */
```

The Go implementation is structurally identical to the Python version.

Go maps return the zero value for missing keys, which works perfectly for frequency tracking because nonexistent entries naturally behave like frequency zero.

Unlike Python's `defaultdict`, Go does not require special handling for missing keys because integer maps default to `0`.

There are no integer overflow concerns because the maximum number of operations is only `2 * 10^5`.

## Edge Cases

### Deleting a Number That Does Not Exist

A very common bug is attempting to decrease frequencies for numbers that are not present. If we blindly decrement counters, we may create negative frequencies or corrupt the frequency tracking structure.

For example:

```
deleteOne(100)
```

when `100` was never added.

The implementation prevents this by checking:

```
if old_frequency == 0:
    return
```

This ensures invalid deletions are ignored safely.

### Frequency Becoming Zero

When a number's frequency decreases from `1` to `0`, the number should no longer contribute to any frequency bucket.

For example:

```
[5]
```

after deleting `5` becomes:

```
[]
```

A common mistake is accidentally incrementing `frequency_count[0]`.

The implementation avoids this with:

```
if new_frequency > 0:
    self.frequency_count[new_frequency] += 1
```

This ensures zero frequencies are never tracked.

### Multiple Numbers Sharing the Same Frequency

Suppose:

```
1 appears twice
2 appears twice
```

Then:

```
frequency_count[2] = 2
```

If one of those numbers changes frequency, we must decrement the count rather than remove the frequency entirely.

Incorrect implementations sometimes store only a boolean indicating whether a frequency exists. That fails when multiple numbers share the same frequency.

By storing counts instead of booleans, the implementation correctly handles overlapping frequencies.
