## Solution

## Problem Understanding

The problem asks us to compute a **prefix common array** `C` from two integer permutations `A` and `B` of length `n`. In simpler terms, for each index `i`, we want to know how many numbers from `1` to `n` have appeared in both arrays **up to index `i`**. A permutation is a sequence containing each integer from `1` to `n` exactly once, which guarantees there are no duplicates in either array.

The input consists of two arrays `A` and `B` of length `n`, both guaranteed to be permutations. The output is another array `C` of the same length where `C[i]` counts numbers that appear in both `A` and `B` up to index `i`.

Constraints tell us the arrays are small (`n <= 50`), so even a slightly inefficient algorithm will run in a reasonable time. Edge cases include the smallest arrays (`n = 1`) and arrays where all elements match late in the sequence.

The main challenge is efficiently tracking which numbers have appeared in both arrays without repeatedly scanning the prefix for every index.

## Approaches

The **brute-force approach** is straightforward. For each index `i`, we can iterate through the prefix `A[0..i]` and check for each element whether it also exists in the prefix `B[0..i]`. This is correct because it directly implements the problem definition, but it is inefficient: for each of the `n` indices, we do up to `O(n)` work, resulting in `O(n^2)` time.

The **optimal approach** leverages **sets** or **boolean arrays** to track which numbers have appeared in either array. As we iterate from `0` to `n-1`, we mark numbers seen in `A` and `B`. If a number has been seen in both arrays, it contributes to the count for the current index. This way, we only do constant-time work per number per index, giving an `O(n)` solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check each prefix pair directly |
| Optimal | O(n) | O(n) | Use sets or boolean arrays to track occurrences |

## Algorithm Walkthrough

1. Initialize a `seen` boolean array (or set) of size `n+1` to track numbers that have appeared in `A` or `B`.
2. Initialize another boolean array (or set) `counted` of the same size to track numbers that have been counted as common.
3. Initialize an empty array `C` to store the result.
4. Initialize a counter `common_count` to `0`.
5. Iterate through indices `i = 0` to `n-1`:

1. Mark `A[i]` and `B[i]` as seen.
2. If `A[i]` has now been seen in both arrays and not yet counted, increment `common_count` and mark it as counted.
3. Similarly, do the same for `B[i]`.
4. Append `common_count` to `C`.
6. Return `C`.

**Why it works**: Each number can only appear once in each permutation. By marking numbers as seen in both arrays and counting them only once, we maintain an invariant that `common_count` always reflects the total number of numbers seen in both arrays up to index `i`.

## Python Solution

```python
from typing import List

class Solution:
    def findThePrefixCommonArray(self, A: List[int], B: List[int]) -> List[int]:
        n = len(A)
        seen = [0] * (n + 1)
        counted = [0] * (n + 1)
        result = []
        common_count = 0
        
        for i in range(n):
            for num in (A[i], B[i]):
                if seen[num] == 1 and counted[num] == 0:
                    common_count += 1
                    counted[num] = 1
                seen[num] = 1
            result.append(common_count)
        return result
```

The Python implementation uses two arrays of size `n+1`. `seen` tracks numbers that have appeared, and `counted` ensures we increment the `common_count` only once per number. At each iteration, we consider both `A[i]` and `B[i]` to update counts and append the running total.

## Go Solution

```go
func findThePrefixCommonArray(A []int, B []int) []int {
    n := len(A)
    seen := make([]int, n+1)
    counted := make([]int, n+1)
    result := make([]int, n)
    commonCount := 0

    for i := 0; i < n; i++ {
        nums := []int{A[i], B[i]}
        for _, num := range nums {
            if seen[num] == 1 && counted[num] == 0 {
                commonCount++
                counted[num] = 1
            }
            seen[num] = 1
        }
        result[i] = commonCount
    }
    return result
}
```

The Go version mirrors the Python approach. Slices are used instead of arrays, and integer `0/1` flags represent boolean states. Slice allocation for `result` ensures we can directly assign the cumulative count at each step.
