**Difficulty:** 🔴 Hard  
**Topics:** String, Greedy

## Solution

## Problem Understanding

The problem asks us to generate the **lexicographically smallest beautiful string** that is strictly larger than a given string `s` of length `n`, using the first `k` letters of the English lowercase alphabet. A string is **beautiful** if it avoids palindromic substrings of length 2 or more. In other words, no two consecutive letters can be equal, and no three consecutive letters can form a palindrome.

The input `s` is already guaranteed to be a beautiful string, so we do not need to check it for validity. The task is to increment this string in a way that respects both the lexicographical order and the beauty constraints.

Constraints indicate that `n` can be as large as `10^5`, so any brute-force method that tries all strings would be far too slow. The integer `k` varies between 4 and 26, meaning we always have at least 4 letters to choose from, which is important for the greedy approach.

Important edge cases include: strings that are already the largest possible under `k`, strings that end in repeated sequences, and strings where we must modify a letter early to avoid palindromes later.

## Approaches

The **brute-force approach** would try to generate every string larger than `s` and check if it is beautiful. This guarantees correctness but is computationally infeasible because the number of candidates grows exponentially with `n`.

The **optimal approach** uses a **greedy, right-to-left strategy**. Starting from the end of the string, we attempt to increment the current character to the next valid letter under the beauty constraints. If this is possible, we fill the remaining positions to the right with the lexicographically smallest valid characters, ensuring the new string is beautiful. If a position cannot be incremented without violating the beauty constraints, we backtrack to the previous character.

The key insight is that the "beautiful" property is **local**: to avoid palindromes of length 2 or 3, each character depends only on the previous two characters. This makes greedy construction efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(k^n) | O(n) | Generate all larger strings and check beauty. Infeasible for n=10^5. |
| Optimal | O(n * k) | O(n) | Increment characters from right to left, greedily choosing the smallest valid letters. |

## Algorithm Walkthrough

1. Convert the input string `s` into a list of characters for easier modification.
2. Start from the last character and move backwards. For each character, try to increment it to the next valid letter (from current+1 up to `'a' + k - 1`) such that it does not form a palindrome with the previous one or two characters.
3. If a valid increment is found, fix the rest of the string from this position to the end using the lexicographically smallest letters that maintain the beautiful property.
4. If no valid increment exists at the current position, move one position to the left and repeat.
5. If we reach the first character and cannot increment, return an empty string because no larger beautiful string exists.

**Why it works:**

The algorithm guarantees the smallest lexicographically larger string by incrementing the rightmost character first, and filling subsequent characters with the minimal allowed values. The local palindrome checks ensure the string remains beautiful throughout.

## Python Solution

```python
class Solution:
    def smallestBeautifulString(self, s: str, k: int) -> str:
        n = len(s)
        s = list(s)
        
        def is_valid(char, i):
            if i >= 1 and s[i-1] == char:
                return False
            if i >= 2 and s[i-2] == char:
                return False
            return True
        
        for i in range(n - 1, -1, -1):
            for c in range(ord(s[i]) + 1, ord('a') + k):
                if is_valid(chr(c), i):
                    s[i] = chr(c)
                    # Fill the rest with the smallest valid letters
                    for j in range(i + 1, n):
                        for next_c in range(ord('a'), ord('a') + k):
                            if is_valid(chr(next_c), j):
                                s[j] = chr(next_c)
                                break
                    return ''.join(s)
        return ""
```

**Implementation walkthrough:**

The `is_valid` helper ensures that a candidate letter does not form a palindrome with the previous one or two letters. The outer loop iterates backwards to find the rightmost position where we can increment. Once a valid increment is found, the inner loop fills the remaining positions with the smallest possible letters that satisfy the beauty constraints. If no increment is possible, the function returns an empty string.

## Go Solution

```go
func smallestBeautifulString(s string, k int) string {
    n := len(s)
    arr := []byte(s)
    
    isValid := func(c byte, i int) bool {
        if i >= 1 && arr[i-1] == c {
            return false
        }
        if i >= 2 && arr[i-2] == c {
            return false
        }
        return true
    }
    
    for i := n - 1; i >= 0; i-- {
        for c := arr[i] + 1; c < byte('a'+k); c++ {
            if isValid(c, i) {
                arr[i] = c
                for j := i + 1; j < n; j++ {
                    for next := byte('a'); next < byte('a'+k); next++ {
                        if isValid(next, j) {
                            arr[j] = next
                            break
                        }
                    }
                }
                return string(arr)
            }
        }
    }
    return ""
}
```

**Go-specific notes:**

Go uses `byte` slices for strings to allow modification. The palindrome check is identical to Python. Loops are structured using byte arithmetic, which is efficient for ASCII characters.

## Worked Examples

**Example 1:** `s = "abcz", k = 26`

Start from last character `'z'` at index 3. Cannot increment `'z'`, so move to index 2 `'c'`. Increment to `'d'`, check validity: previous is `'b'`, no conflict. Fill remainder index 3 with smallest valid letter: `'a'` (cannot be `'d'` because previous is `'d'`, cannot be `'d'`). Result: `"abda"`.

**Example 2:** `s = "dc", k = 4`

Start from index 1 `'c'`. Increment attempts: `'d'` invalid (prev `'d'`), cannot increment further. Move to index 0 `'d'`. Increment attempts exceed `'d'` limit for k=4. No valid increment, return `""`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * k) | Each character may try up to k letters, iterating from right to left. |
| Space | O(n) | Converted string to list for mutability. |

The algorithm is efficient because each character is only revisited a small number of times, bounded by `k`, and `n` can be up to 10^5 comfortably.

## Edge Cases

**Edge Case 1:** Largest possible string.

Input like `"zzz"` with `k=26` cannot be incremented. Implementation correctly returns `""` by trying all positions and finding no valid increments.

**Edge Case 2:** Strings requiring mid-string change.

For `"abac"` with `k=4`, the change cannot occur at the end. The algorithm moves left until it finds a valid increment, then fills the rest with minimal letters, ensuring beauty.

**Edge Case 3:** Minimal k with small alphabet.

When `k=4`, the limited alphabet increases conflicts with palindrome constraints. The `is_valid` check ensures that no invalid letter is chosen, preventing accidental palindromes while still generating the lexicographically smallest valid string.
