## Solution

## Problem Understanding

The problem asks us to take a **circular linked list** of positive integers and split it into two separate circular linked lists. The first circular linked list should contain the first half of the original nodes, using `ceil(n/2)` where `n` is the total number of nodes, and the second list should contain the remaining nodes. The relative order of nodes must be preserved in both lists.

The input is a circular linked list, meaning that the last node points back to the first node. The output should be an array or slice containing exactly two circular linked lists.

The constraints indicate that the input can be fairly large (`n` up to `10^5`), so a solution must ideally traverse the list only a constant number of times. Each node’s value is within `[0, 10^9]`, which is mainly relevant for understanding that integer operations are safe in most languages.

Important edge cases include: when the list has only two nodes (smallest allowed), when the number of nodes is odd or even (affecting the first half calculation), and when all node values are identical.

## Approaches

A brute-force approach would first traverse the circular linked list to count the number of nodes. Then, we could traverse again, creating two new linked lists by allocating new nodes and appending them to the respective lists. This works correctly but is inefficient in space because it creates entirely new nodes instead of reusing the original list. Its time complexity is `O(n)` for traversal, but space complexity is also `O(n)` due to node duplication.

The optimal approach leverages the **two-pointer technique** or a single traversal with careful bookkeeping. First, we determine the total length of the list in a traversal. Then, we calculate the first half size as `ceil(n / 2)`. Next, we traverse the original list, detaching nodes as needed to form the first and second circular linked lists in place. This approach reuses the original nodes, preserving memory efficiency.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Creates new nodes for both halves, preserves order |
| Optimal | O(n) | O(1) | Splits original circular list in place, reuses nodes |

## Algorithm Walkthrough

1. **Calculate list length**: Initialize a counter and traverse the circular list until returning to the head, incrementing the counter at each step. This gives the total number of nodes, `n`.
2. **Compute first half size**: Use the formula `ceil(n / 2)`. In integer arithmetic, this can be expressed as `(n + 1) // 2`.
3. **Split the list**: Initialize two pointers: `first_head` pointing to the original head and `second_head` as `None`. Traverse the list again, keeping track of the node index. When reaching the node at index `first_half_size - 1`, mark it as the last node of the first circular list. Set its `next` to `first_head` to close the circle. The next node becomes the head of the second list.
4. **Complete the second circular list**: Continue traversal until reaching the original tail. Set its `next` pointer to `second_head` to close the second circle.
5. **Return result**: Package `first_head` and `second_head` in an array or slice and return.

**Why it works**: By traversing the list exactly twice and carefully updating `next` pointers, we maintain the original order, correctly split the list at the midpoint, and preserve circularity for both halves. All nodes are reused and no data is lost.

## Python Solution

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
from typing import Optional, List

class Solution:
    def splitCircularLinkedList(self, list: Optional[ListNode]) -> List[Optional[ListNode]]:
        if not list or not list.next:
            return [list, None]
        
        # Step 1: Determine the length of the circular list
        length = 1
        current = list.next
        while current != list:
            length += 1
            current = current.next
        
        first_half_size = (length + 1) // 2
        
        # Step 2: Split the list
        first_head = list
        current = list
        for _ in range(first_half_size - 1):
            current = current.next
        second_head = current.next
        current.next = first_head
        
        # Step 3: Complete the second circular list
        current = second_head
        while current.next != list:
            current = current.next
        current.next = second_head
        
        return [first_head, second_head]
```

**Implementation Explanation**: First, we count the nodes to compute the split point. Then we traverse to the last node of the first half and close the first circular list. Next, we traverse to the last node of the second half and close its circular list. Finally, we return the two heads.

## Go Solution

```go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func splitCircularLinkedList(list *ListNode) []*ListNode {
    if list == nil || list.Next == list {
        return []*ListNode{list, nil}
    }
    
    // Step 1: Determine the length of the circular list
    length := 1
    current := list.Next
    for current != list {
        length++
        current = current.Next
    }
    
    firstHalfSize := (length + 1) / 2
    
    // Step 2: Split the list
    firstHead := list
    current = list
    for i := 0; i < firstHalfSize-1; i++ {
        current = current.Next
    }
    secondHead := current.Next
    current.Next = firstHead
    
    // Step 3: Complete the second circular list
    current = secondHead
    for current.Next != list {
        current = current.Next
    }
    current.Next = secondHead
    
    return []*ListNode{firstHead, secondHead}
}
```

**Go Notes**: Nil handling ensures the edge case of a single-node circular list is handled correctly. The arithmetic `(length + 1)/2` handles odd/even splits. Go uses pointers, so all node manipulations are in-place, similar to Python.

## Worked Examples

### Example 1: `[1,5,7]`

Step through:

1. Count nodes: length = 3
2. First half size = ceil(3 / 2) = 2
3. Traverse to node 2 (`5`), set its `next` to head `1`
4. Remaining node `7` forms second circular list
5. Return heads `[1 -> 5 -> 1, 7 -> 7]`

### Example 2: `[2,6,1,5]`

Step through:

1. Count nodes: length = 4
2. First half size = ceil(4 / 2) = 2
3. Traverse to node 2 (`6`), set its `next` to head `2`
4. Remaining nodes `[1,5]` form second circular list
5. Return heads `[2 -> 6 -> 2, 1 -> 5 -> 1]`

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Two passes through the circular linked list: one for length, one for splitting |
| Space | O(1) | Only pointers are used, no new nodes are created |

The complexity is optimal because we traverse each node a constant number of times and do not allocate extra memory proportional to `n`.
