**Difficulty:** 🔴 Hard  
**Topics:** Array, Dynamic Programming, Tree, Depth-First Search, Graph Theory

## Solution

## Problem Understanding

The problem gives us an undirected tree with `n` nodes. Every node has a price, and we are also given several trips between pairs of nodes. Since the graph is a tree, there is exactly one simple path between any two nodes.

For each trip, the total cost is the sum of the prices of all nodes along the path from the start node to the end node. Before any trips begin, we are allowed to choose some nodes whose prices will be permanently halved. However, there is an important restriction: no two chosen nodes can be adjacent in the tree.

The goal is to minimize the total cost across all trips.

Another way to think about the problem is this:

1. Every trip contributes usage counts to the nodes along its path.
2. If a node appears many times across trips, reducing its price may save a lot of money.
3. But adjacent nodes cannot both be discounted, so we must carefully choose which nodes to halve.

The input consists of:

- `n`, the number of nodes
- `edges`, describing the tree structure
- `price`, where `price[i]` is the cost associated with node `i`
- `trips`, where each trip specifies a start and end node

The output is a single integer, the minimum possible total cost after optimally selecting non-adjacent discounted nodes.

The constraints are relatively small:

- `n <= 50`
- `trips.length <= 100`

These limits are important because they allow us to use repeated DFS traversals across the tree without performance concerns. A solution with roughly `O(n^2)` or `O(n * trips)` complexity is completely acceptable.

Several edge cases are important:

- A trip may start and end at the same node, meaning the path contains only one node.
- The optimal answer may involve halving no nodes at all.
- Since adjacent discounted nodes are forbidden, a greedy strategy of halving the most expensive nodes can fail.
- Some nodes may never appear in any trip, meaning discounting them provides no benefit.
- The tree can degenerate into a line, which often exposes bugs in parent handling during DFS.

## Approaches

### Brute Force Approach

The most direct approach is to try every valid subset of nodes that can be discounted.

A subset is valid if no two selected nodes are adjacent. For each valid subset:

1. Compute the discounted prices.
2. For every trip, find the path between the two nodes.
3. Sum all path costs.
4. Track the minimum total.

This approach is correct because it explicitly evaluates every legal configuration of discounted nodes.

However, it is too slow. There are `2^n` subsets of nodes, and even though many are invalid, the search space becomes enormous as `n` approaches 50.

Additionally, for each subset we would still need to evaluate all trips.

This leads to exponential complexity, which is infeasible.

### Key Insight

The crucial observation is that the contribution of each node to the final answer depends only on:

- how many times the node appears across all trips
- whether that node is discounted

Suppose node `u` appears `count[u]` times across all trip paths.

Then:

- if not discounted, it contributes

`count[u] * price[u]`
- if discounted, it contributes

`count[u] * (price[u] // 2)`

So the problem becomes:

> Choose a set of non-adjacent nodes to maximize the total savings.

This transforms the problem into a classic tree dynamic programming problem similar to the "House Robber on Trees" pattern.

We first compute how many times each node is used across all trips. Then we perform tree DP:

- `dp[u][0]` = minimum cost of subtree rooted at `u` when `u` is NOT discounted
- `dp[u][1]` = minimum cost of subtree rooted at `u` when `u` IS discounted

If `u` is discounted, none of its children may be discounted.

This produces an efficient polynomial-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * trips * n) | O(n) | Tries every valid discounted-node configuration |
| Optimal | O(trips * n + n) | O(n) | Counts node usage, then applies tree DP |

## Algorithm Walkthrough

1. Build the adjacency list for the tree.

Since the graph is a tree, an adjacency list is the most natural representation. It allows efficient DFS traversal in both directions.
2. Compute how many times each node is used across all trips.

For every trip `[start, end]`, run DFS to locate the unique path between the two nodes.

As the DFS backtracks along the successful path, increment the frequency counter for every node on that path.

At the end:

`count[u]` tells us how many trips include node `u`.
3. Root the tree arbitrarily.

The tree is undirected, but tree DP becomes much easier if we choose a root. We can safely root the tree at node `0`.
4. Define the DP states.

For every node `u`:

- `not_halved` = minimum cost for the subtree when `u` is not discounted
- `halved` = minimum cost for the subtree when `u` is discounted
5. Compute the base contribution of the current node.

If node `u` is not discounted:

```
count[u] * price[u]
```

If node `u` is discounted:

```
count[u] * (price[u] // 2)
```
6. Process each child recursively.

For every child `v`:

- If `u` is not discounted, child `v` may either be discounted or not:

```
min(child_not_halved, child_halved)
```
- If `u` is discounted, child `v` cannot be discounted:

```
child_not_halved
```
7. Combine results.

Add the child contributions into the current node's DP states.
8. Return the best result for the root.

Since the root may or may not be discounted:

```
min(root_not_halved, root_halved)
```

### Why it works

The DP works because of optimal substructure.

For every node, the only restriction imposed on its children is whether the current node is discounted. Once that decision is fixed, each subtree becomes independent.

The recurrence correctly enforces the adjacency constraint:

- discounted parent => children cannot be discounted
- non-discounted parent => children choose whichever option is cheaper

Because every subtree is solved optimally under both states, the final result is globally optimal.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def minimumTotalPrice(
        self,
        n: int,
        edges: List[List[int]],
        price: List[int],
        trips: List[List[int]]
    ) -> int:

        graph = defaultdict(list)

        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)

        count = [0] * n

        def find_path(curr: int, parent: int, target: int) -> bool:
            if curr == target:
                count[curr] += 1
                return True

            for neighbor in graph[curr]:
                if neighbor == parent:
                    continue

                if find_path(neighbor, curr, target):
                    count[curr] += 1
                    return True

            return False

        for start, end in trips:
            find_path(start, -1, end)

        def tree_dp(node: int, parent: int):
            not_halved = count[node] * price[node]
            halved = count[node] * (price[node] // 2)

            for neighbor in graph[node]:
                if neighbor == parent:
                    continue

                child_not_halved, child_halved = tree_dp(neighbor, node)

                not_halved += min(child_not_halved, child_halved)

                halved += child_not_halved

            return not_halved, halved

        result_not_halved, result_halved = tree_dp(0, -1)

        return min(result_not_halved, result_halved)
```

The implementation begins by constructing an adjacency list for the tree. Since the graph is undirected, every edge is inserted in both directions.

Next, the `count` array tracks how many times each node appears across all trips. The `find_path` DFS searches for the unique path between two nodes. Once the target is found, the recursion backtracks and increments the count for every node along the successful path.

After all trips are processed, the algorithm performs tree DP using the `tree_dp` function.

For each node:

- `not_halved` stores the minimum subtree cost if the current node is not discounted.
- `halved` stores the minimum subtree cost if the current node is discounted.

The recursive transitions enforce the adjacency restriction naturally.

Finally, the root may either be discounted or not, so the minimum of the two states is returned.

## Go Solution

```go
package main

func minimumTotalPrice(n int, edges [][]int, price []int, trips [][]int) int {
	graph := make([][]int, n)

	for _, edge := range edges {
		u, v := edge[0], edge[1]
		graph[u] = append(graph[u], v)
		graph[v] = append(graph[v], u)
	}

	count := make([]int, n)

	var findPath func(int, int, int) bool

	findPath = func(curr, parent, target int) bool {
		if curr == target {
			count[curr]++
			return true
		}

		for _, neighbor := range graph[curr] {
			if neighbor == parent {
				continue
			}

			if findPath(neighbor, curr, target) {
				count[curr]++
				return true
			}
		}

		return false
	}

	for _, trip := range trips {
		start, end := trip[0], trip[1]
		findPath(start, -1, end)
	}

	var treeDP func(int, int) (int, int)

	treeDP = func(node, parent int) (int, int) {
		notHalved := count[node] * price[node]
		halved := count[node] * (price[node] / 2)

		for _, neighbor := range graph[node] {
			if neighbor == parent {
				continue
			}

			childNotHalved, childHalved := treeDP(neighbor, node)

			if childNotHalved < childHalved {
				notHalved += childNotHalved
			} else {
				notHalved += childHalved
			}

			halved += childNotHalved
		}

		return notHalved, halved
	}

	resultNotHalved, resultHalved := treeDP(0, -1)

	if resultNotHalved < resultHalved {
		return resultNotHalved
	}

	return resultHalved
}
```

The Go implementation follows the same structure as the Python solution.

Instead of tuples, Go uses multiple return values for the DP states. The adjacency list is represented using slices of slices.

Integer division in Go naturally truncates toward zero, which matches the problem requirement because all prices are guaranteed to be even.

The recursive DFS logic is otherwise identical.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(trips * n + n) | Each trip performs DFS over the tree, followed by one tree DP traversal |
| Space | O(n) | Adjacency list, recursion stack, and count array |

The tree contains at most 50 nodes, so even repeated DFS traversals are inexpensive.

Each trip requires at most one full traversal of the tree, giving `O(trips * n)` time. The DP traversal itself is linear in the number of nodes.

The space usage is linear because the adjacency list, recursion stack, and auxiliary arrays all scale with `n`.

## Edge Cases

One important edge case occurs when a trip starts and ends at the same node. In this case, the path contains exactly one node, and no traversal through neighbors is needed. A buggy implementation may accidentally skip counting the node or double count it during recursion. The provided implementation handles this correctly because the DFS immediately recognizes the target node and increments its count once.

Another important case is a tree shaped like a straight line. These structures frequently expose mistakes involving parent tracking during DFS. Without proper parent checks, the recursion may revisit nodes indefinitely. The implementation avoids this by always passing the parent node and skipping it during neighbor traversal.

A third edge case involves nodes that never appear in any trip. Such nodes have a frequency count of zero, meaning their contribution is always zero regardless of whether they are discounted. A naive solution might still waste effort trying to optimize them. In this implementation, their DP contribution naturally becomes zero, so they do not affect the final answer.

A fourth subtle edge case occurs when the most frequently used nodes are adjacent. A greedy strategy might incorrectly try to discount both of them. The tree DP correctly enforces the adjacency restriction by allowing discounted children only when the parent is not discounted.
