# LeetCode 2678 - Number of Senior Citizens

## Problem Understanding

This problem gives us an array of strings named `details`. Each string has a fixed length of `15` characters and stores information about a passenger in a compressed format.

The structure of each string is always the same:

- Characters `0` through `9` represent the passenger's phone number
- Character `10` represents gender
- Characters `11` and `12` represent age
- Characters `13` and `14` represent seat number

Our task is to count how many passengers are strictly older than `60`.

The important observation is that the age is always stored in the same position inside every string. Since the age occupies indices `11` and `12`, we can directly extract those two characters and convert them into an integer.

For example:

```
"7868190130M7522"
            ^^
            age = 75
```

If the extracted age is greater than `60`, we increase our answer count.

The constraints are very small. The array contains at most `100` strings, and every string has exactly `15` characters. This tells us that even a straightforward linear scan will be more than efficient enough.

A few important edge cases are worth noting upfront:

- Age `60` should not be counted because the condition is strictly greater than `60`
- Leading zeros may appear in ages such as `"05"` or `"09"`, so we should parse the substring as an integer
- We do not need to validate input formatting because the problem guarantees every string follows the required structure

## Approaches

### Brute Force Approach

A brute force approach would manually parse every field of the string character by character. We could iterate through all 15 characters, reconstruct the age digits, convert them into a number, and then compare against `60`.

This works because the input format is fixed and deterministic. Every passenger record contains the age at the same positions.

However, this approach performs unnecessary work because we do not actually care about the phone number, gender, or seat information. Parsing the entire string is wasteful when only two characters matter.

### Optimal Approach

The key insight is that the age always appears at indices `11` and `12`.

Instead of processing the whole string, we directly slice the age substring:

```
detail[11:13]
```

We then convert it to an integer and check whether it exceeds `60`.

Since we only scan the array once and perform constant work for each passenger, this gives an optimal linear solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * 15) | O(1) | Parses every character manually |
| Optimal | O(n) | O(1) | Directly extracts the age substring |

Here, `n` is the number of passengers.

## Algorithm Walkthrough

1. Initialize a variable `senior_count` to `0`.
2. Iterate through each string `detail` in the `details` array.
3. Extract the age substring using indices `11` and `12`.

```
age = int(detail[11:13])
```

We use slicing because the problem guarantees the age is always stored in those exact positions.
4. Check whether the extracted age is greater than `60`.
5. If it is, increment `senior_count`.
6. After processing all passengers, return `senior_count`.

### Why it works

The algorithm works because every passenger record follows the exact same fixed-width format. The age always occupies positions `11` and `12`, so extracting those characters always gives the correct age. Since we examine every passenger exactly once and count only ages strictly greater than `60`, the final count is guaranteed to be correct.

## Python Solution

```python
from typing import List

class Solution:
    def countSeniors(self, details: List[str]) -> int:
        senior_count = 0

        for detail in details:
            age = int(detail[11:13])

            if age > 60:
                senior_count += 1

        return senior_count
```

The implementation follows the algorithm directly.

We first initialize `senior_count` to store the number of qualifying passengers.

Next, we iterate through every passenger record in `details`. Since the problem guarantees a fixed string structure, we safely extract the age using `detail[11:13]`.

The substring is converted into an integer so numerical comparison can be performed correctly.

Whenever the age exceeds `60`, we increment the answer counter.

Finally, after all records have been processed, we return the total count.

## Go Solution

```go
package main

import "strconv"

func countSeniors(details []string) int {
    seniorCount := 0

    for _, detail := range details {
        age, _ := strconv.Atoi(detail[11:13])

        if age > 60 {
            seniorCount++
        }
    }

    return seniorCount
}
```

The Go solution is almost identical to the Python version.

The main difference is that Go requires explicit string-to-integer conversion using `strconv.Atoi`.

The problem guarantees valid input, so the conversion error can safely be ignored.

Go slices strings similarly to Python, so `detail[11:13]` extracts the two-character age substring directly.

## Edge Cases

One important edge case is passengers whose age is exactly `60`. A careless implementation might use `>= 60` instead of `> 60`, which would incorrectly count those passengers as seniors. The implementation explicitly checks `age > 60`, so this case is handled correctly.

Another edge case is ages with leading zeros, such as `"05"` or `"09"`. If the implementation compared strings lexicographically instead of converting them to integers, the logic could become error-prone. By converting the substring to an integer first, the implementation handles all numeric values correctly.

A third important edge case is the smallest possible input size, where the array contains only one passenger. Some implementations accidentally assume multiple elements or initialize counters incorrectly. Since this solution simply iterates through the array regardless of length, it works correctly even for a single passenger record.
