## Solution

## Problem Understanding

The problem gives us two inputs:

- An array of strings called `words`
- A string called `s`

We need to determine whether `s` is the acronym formed by taking the first character from every word in `words`, in the same order.

For example, if:

```
words = ["alice", "bob", "charlie"]
```

then the acronym formed is:

```
"a" + "b" + "c" = "abc"
```

So if `s == "abc"`, we return `true`.

The important detail is that the characters must come from the first letter of each word, and they must appear in order. We are not allowed to skip words, rearrange them, or use any character other than the first character.

The constraints are very small:

- At most 100 words
- Each word has at most 10 characters
- `s` has at most 100 characters

These constraints tell us that efficiency is not a major concern. Even a straightforward solution will easily run within limits. Still, it is useful to think about cleaner and more optimal implementations.

There are several important edge cases to consider:

- If the number of words does not match the length of `s`, then `s` cannot possibly be the acronym.
- Multiple words may start with the same character.
- A single mismatched character should immediately make the answer `false`.
- The problem guarantees every word has at least one character, so accessing the first character is always safe.

## Approaches

### Brute Force Approach

A brute force solution explicitly constructs the acronym string.

We iterate through every word in `words`, take the first character, and append it to a temporary string. After processing all words, we compare the constructed string with `s`.

For example:

```
words = ["never","gonna","give","up","on","you"]
```

We build:

```
"n" -> "ng" -> "ngg" -> "nggu" -> "ngguo" -> "ngguoy"
```

Finally, we check whether the constructed acronym equals `s`.

This approach is correct because it directly follows the definition of an acronym given in the problem statement.

The downside is that it creates an additional string, although with the small constraints this is not a practical issue.

### Optimal Approach

The key observation is that we do not actually need to build the entire acronym string.

Instead, we can compare characters one by one while iterating through the words:

1. First check whether the number of words equals the length of `s`
2. For each index `i`

- Compare `words[i][0]` with `s[i]`
- If they differ, return `false` immediately
3. If all characters match, return `true`

This avoids constructing an unnecessary intermediate string and allows early termination as soon as a mismatch is found.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Builds the acronym string before comparing |
| Optimal | O(n) | O(1) | Compares characters directly without extra storage |

Here, `n` represents the number of words.

## Algorithm Walkthrough

1. First, compare the number of words with the length of `s`.

The acronym must contain exactly one character from each word. Therefore, if:

```
len(words) != len(s)
```

then it is impossible for `s` to be the correct acronym.
2. Iterate through every word using its index.

We process the words in order because the acronym must preserve the same ordering.
3. For each index `i`, compare:

```
words[i][0]
```

with:

```
s[i]
```

The first expression gives the first character of the current word, while the second gives the expected acronym character.
4. If the characters differ, immediately return `false`.

There is no need to continue because a single mismatch invalidates the acronym.
5. If the loop finishes without mismatches, return `true`.

This means every word contributed exactly the expected character.

### Why it works

The algorithm checks the exact condition required by the problem definition. Each position `i` verifies that the first character of `words[i]` matches the corresponding character in `s`. The length check guarantees there are neither missing nor extra characters. Since all required conditions are validated, the algorithm is correct.

## Python Solution

```python
from typing import List

class Solution:
    def isAcronym(self, words: List[str], s: str) -> bool:
        if len(words) != len(s):
            return False

        for index, word in enumerate(words):
            if word[0] != s[index]:
                return False

        return True
```

The implementation starts with the length check because the acronym must contain one character per word. If the lengths differ, we immediately return `False`.

Next, we iterate through the words using `enumerate`, which gives both the index and the current word. For every word, we compare its first character with the character at the same position in `s`.

If any comparison fails, the function returns `False` immediately. This early exit avoids unnecessary work.

If the loop completes successfully, every character matched correctly, so the function returns `True`.

## Go Solution

```go
func isAcronym(words []string, s string) bool {
    if len(words) != len(s) {
        return false
    }

    for i, word := range words {
        if word[0] != s[i] {
            return false
        }
    }

    return true
}
```

The Go implementation follows the same logic as the Python version.

Strings in Go can be indexed directly, so `word[0]` accesses the first byte of the string. Since the problem guarantees lowercase English letters only, byte indexing is completely safe here.

The solution uses constant extra memory and performs direct character comparisons without constructing an intermediate string.

## Edge Cases

### Length mismatch between `words` and `s`

One of the most important edge cases occurs when the number of words differs from the length of `s`.

For example:

```
words = ["an", "apple"]
s = "a"
```

The acronym generated from the words would contain two characters, not one. A naive implementation that only compares available positions could incorrectly return `true`. The solution avoids this bug by checking lengths before any iteration begins.

### Single character words

Words may contain only one character:

```
words = ["x", "y", "z"]
s = "xyz"
```

Some implementations accidentally assume words are longer than one character. The problem guarantees every word has at least one character, so accessing `word[0]` is always safe. The algorithm handles these inputs naturally.

### Early mismatch

Another important case is when a mismatch appears early in the sequence:

```
words = ["hello", "world"]
s = "ha"
```

The first character matches:

```
'h' == 'h'
```

but the second comparison fails:

```
'w' != 'a'
```

The implementation immediately returns `False` instead of continuing unnecessary work. This makes the solution both cleaner and more efficient.
