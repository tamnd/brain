## Solution

## Problem Understanding

The problem presents a **directed graph** with `n` nodes and `n` edges, forming a functional graph where each node has **exactly one outgoing edge**. You are given an array `edges` such that `edges[i]` represents the node that node `i` points to. Starting from any node, you are to traverse along the edges until you encounter a node you have already visited in that traversal, then count the number of **distinct nodes** visited in that path.

In simpler terms, for each node, you must determine the size of the cycle or path (including nodes leading into a cycle) that it ultimately reaches. The output is an array `answer` where `answer[i]` equals the number of unique nodes visited starting from node `i`.

The constraints tell us that `n` can be up to 100,000. This immediately rules out naive approaches that traverse each path fully for every node because the worst-case runtime would be O(n²). Importantly, the graph is guaranteed to have no self-loops (`edges[i] != i`), and every node has exactly one outgoing edge, so every connected component is either a cycle or a tree leading into a cycle.

Key edge cases to be aware of include **chains leading to cycles**, **disconnected cycles**, and **long linear paths**. A naive solution could double-count nodes or fail to terminate properly if cycles are not detected efficiently.

## Approaches

The **brute-force approach** would start from each node and follow its outgoing edges while tracking visited nodes until a cycle is detected. For each starting node, you could store visited nodes in a set and return the size. While correct, this method is inefficient because overlapping paths would be recalculated repeatedly, giving O(n²) time complexity in the worst case.

The **optimal approach** leverages **DFS with memoization**. Since each node has only one outgoing edge, you can perform a DFS and store the number of nodes reachable from each node once computed. If during DFS you encounter a node that is part of the current path, you detect a cycle. Nodes inside a cycle all have the same count equal to the cycle length, and nodes pointing into the cycle can sum the distance to the cycle plus the cycle length. This avoids recomputation and gives a linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Traverse from each node independently using a visited set |
| Optimal | O(n) | O(n) | DFS with memoization; cycle detection is integrated, each node processed once |

## Algorithm Walkthrough

1. Initialize an array `answer` of size `n` with zeros. This will store the final counts for each node.
2. Initialize a set `visited` to track nodes in the current DFS path to detect cycles.
3. For each node `i` from 0 to n-1:

1. If `answer[i]` is already computed, skip it.
2. Otherwise, start a DFS from node `i`.
4. In DFS:

1. If the current node is already in `visited`, a **cycle is detected**. Compute the cycle length by traversing from the node back to itself.
2. Mark nodes in the cycle with the cycle length in `answer`.
3. If the next node has `answer` already computed, propagate the reachable node count back along the DFS path.
5. Return the fully populated `answer` array after all nodes have been processed.

**Why it works**: Each node is processed only once in DFS. Nodes leading into cycles sum the distance to the cycle plus the cycle length. Nodes inside a cycle all share the cycle length. Memoization ensures that no node’s reachable count is recomputed, giving linear time complexity.

## Python Solution

```python
from typing import List

class Solution:
    def countVisitedNodes(self, edges: List[int]) -> List[int]:
        n = len(edges)
        answer = [0] * n
        visited = set()
        
        def dfs(node: int, path: List[int]) -> int:
            if answer[node]:
                return answer[node]
            if node in visited:
                # Cycle detected
                cycle_start = path.index(node)
                cycle_length = len(path) - cycle_start
                for i in range(cycle_start, len(path)):
                    answer[path[i]] = cycle_length
                for i in range(cycle_start):
                    answer[path[i]] = cycle_length + cycle_start - i
                return answer[node]
            
            visited.add(node)
            path.append(node)
            next_node = edges[node]
            result = dfs(next_node, path)
            visited.remove(node)
            path.pop()
            
            if not answer[node]:
                answer[node] = result
            return answer[node]
        
        for i in range(n):
            if not answer[i]:
                dfs(i, [])
        
        return answer
```

In this implementation, `dfs` recursively traverses the graph while tracking the current path. When a cycle is found, the cycle nodes are updated immediately. Nodes leading to the cycle are updated based on the distance to the cycle. Memoization prevents redundant work.

## Go Solution

```go
func countVisitedNodes(edges []int) []int {
    n := len(edges)
    answer := make([]int, n)
    visited := make(map[int]bool)
    
    var dfs func(node int, path []int) int
    dfs = func(node int, path []int) int {
        if answer[node] != 0 {
            return answer[node]
        }
        for i, v := range path {
            if v == node {
                cycleLen := len(path) - i
                for j := i; j < len(path); j++ {
                    answer[path[j]] = cycleLen
                }
                for j := 0; j < i; j++ {
                    answer[path[j]] = cycleLen + i - j
                }
                return answer[node]
            }
        }
        
        path = append(path, node)
        result := dfs(edges[node], path)
        path = path[:len(path)-1]
        if answer[node] == 0 {
            answer[node] = result
        }
        return answer[node]
    }
    
    for i := 0; i < n; i++ {
        if answer[i] == 0 {
            dfs(i, []int{})
        }
    }
    return answer
}
```

Go implementation mirrors the Python solution, using slices to track the path and a map for temporary visited detection. Unlike Python, slice append and slicing handles path management efficiently.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited at most once in DFS, and memoization avoids redundant computation. |
| Space | O(n) | For recursion stack/path tracking and the `answer` array. |

The DFS visits each node once. Even in cycles, nodes are updated immediately, so no extra traversals occur. The recursive call stack grows at most n deep.
