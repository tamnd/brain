## Solution

## Problem Understanding

The problem asks us to find a subsequence of a given 1-indexed integer array `prices` such that the subsequence is **linear**. Here, "linear" is defined in a very specific way: the difference in the values of consecutive elements in the subsequence must equal the difference in their indices. Formally, for a subsequence `indexes` of length `k`, we require that for every `j` from `2` to `k`, `prices[indexes[j]] - prices[indexes[j-1]] == indexes[j] - indexes[j-1]`. The score of a subsequence is the sum of the elements in that subsequence, and we are asked to find the **maximum possible score** of any linear subsequence.

The input is a 1-indexed array of integers, `prices`, where `prices[i]` represents the stock price on day `i`. The constraints indicate that the array can be large (`1 <= prices.length <= 10^5`) and the stock prices themselves can be up to `10^9`. This rules out brute-force solutions that examine all possible subsequences.

Important edge cases include arrays with a single element, arrays where all elements are the same, strictly increasing sequences, and sequences where no two consecutive elements satisfy the linear condition with a difference of 1.

The problem essentially reduces to finding subsequences where the difference between the value and the index is constant, because:

```
prices[index] - index = constant
```

This follows from the linearity condition:

```
prices[j] - prices[i] == j - i  =>  prices[j] - j == prices[i] - i
```

## Approaches

The brute-force approach would attempt to examine all possible subsequences, check whether they satisfy the linearity condition, and compute their scores. This method is clearly infeasible for large `n` because the number of subsequences grows exponentially, making the time complexity `O(2^n)`.

The key insight for an optimal approach is noticing the transformation `prices[i] - i`. For any linear subsequence, all elements must share the same value for `prices[i] - i`. This allows us to reduce the problem to grouping elements by this transformed value and summing each group. The maximum sum among these groups is the answer.

This is essentially a **hash map aggregation** problem: compute `prices[i] - i` for each index, sum values with the same key, and track the maximum sum.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Generate all subsequences and validate linearity, too slow |
| Optimal | O(n) | O(n) | Transform and group by `prices[i] - i` using a hash map |

## Algorithm Walkthrough

1. Initialize an empty hash map `group_sums` to store the sum of prices for each unique `prices[i] - i`.
2. Iterate through the array `prices` with 1-based indexing.
3. For each element at index `i`, compute the key `prices[i] - i`.
4. Add `prices[i]` to the cumulative sum stored in `group_sums` under that key.
5. Track the maximum sum across all keys as you update the map.
6. Return the maximum sum after processing all elements.

**Why it works**: By transforming the array with `prices[i] - i`, we exploit the linearity condition directly. All elements of any valid linear subsequence must share the same key, so summing by key ensures we capture the maximum possible sum for each candidate subsequence. This guarantees correctness and is efficient.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def maxScore(self, prices: List[int]) -> int:
        group_sums = defaultdict(int)
        max_sum = 0
        
        for i, price in enumerate(prices, start=1):
            key = price - i
            group_sums[key] += price
            max_sum = max(max_sum, group_sums[key])
        
        return max_sum
```

In this implementation, we use a `defaultdict` to simplify sum accumulation. The loop iterates through each price with 1-based indexing, computes the key for linearity, and updates the sum for that group. The maximum sum is tracked during iteration for efficiency.

## Go Solution

```go
func maxScore(prices []int) int64 {
    groupSums := make(map[int]int64)
    var maxSum int64 = 0
    
    for i, price := range prices {
        key := price - (i + 1)
        groupSums[key] += int64(price)
        if groupSums[key] > maxSum {
            maxSum = groupSums[key]
        }
    }
    
    return maxSum
}
```

The Go implementation is similar to Python, using a map to store sums. The main differences include explicit handling of `int64` to avoid integer overflow for large sums, and converting the 0-based index to 1-based by adding 1.

## Edge Cases

1. **Single Element Array**: If the array contains only one element, the maximum linear score is the element itself. This is handled naturally since the loop processes each element and `group_sums` will have a single entry.
2. **All Elements Equal**: When all elements are identical, `prices[i] - i` produces distinct keys for each index, so the optimal linear subsequence might only include individual elements. The algorithm handles this by summing elements per key and taking the maximum sum, which correctly accounts for this scenario.
3. **No Valid Multi-Element Linear Subsequence**: For arrays where differences between consecutive prices never match the differences in indices (e.g., strictly decreasing sequences with steps not equal to 1), each element forms its own group. The algorithm will return the maximum single element value, which is correct.

This approach robustly handles all the nuances of linear subsequences efficiently.
