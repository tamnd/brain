## Solution

## Problem Understanding

The problem presents a two-player game between Alice and Bob, each possessing a lexicographically sorted list of words, `a` and `b`. The game rules require players to take turns playing words that are **closely greater** than the last played word. Alice always starts with her lexicographically smallest word. A word `w` is closely greater than a word `z` if it is lexicographically greater and its first letter is either the same as `z`'s first letter or the next letter in the alphabet. The challenge is to determine if Alice can guarantee a win when both players play optimally.

The input arrays are already sorted lexicographically and contain distinct words. The combined length of all words does not exceed `10^6`, and each array can contain up to `10^5` words. This indicates that a brute-force exploration of all possible game sequences will be computationally infeasible. The problem requires understanding the relative positions of words and first-letter constraints, making a greedy or game-theoretic approach suitable.

Key edge cases include situations where the first letters of the players’ words prevent any valid move, arrays of minimal length (1), or scenarios where one player dominates the alphabetically earliest words.

## Approaches

The **brute-force approach** would recursively simulate every possible move for both players, checking if Alice can win. For each word played, we would explore all valid closely greater words the opponent could play, continuing until one player cannot move. While correct, this approach is computationally intractable due to the exponential number of game sequences, especially with arrays of size up to `10^5`.

The **optimal approach** leverages the lexicographic order and first-letter constraints. Observations:

1. Since the arrays are sorted, Alice always plays her smallest word first.
2. Bob can only respond with a word that starts with either the same first letter or the next letter and is lexicographically larger.
3. By maintaining pointers to the next playable word in each array, we can simulate the game efficiently without exploring all possibilities.
4. We only need to check the first letters of remaining words to decide if the opponent has a valid move, effectively reducing the problem to a **two-pointer simulation**.

This approach works because at each turn, only the lexicographically smallest valid word matters due to the sorted arrays and optimal play.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^(m+n)) | O(m+n) | Exponential recursion simulating all game paths |
| Optimal | O(m+n) | O(1) | Two-pointer simulation using lexicographical properties and first-letter constraints |

## Algorithm Walkthrough

1. Initialize two pointers `i` and `j` at 0 for Alice and Bob’s arrays.
2. Set `last_word` to Alice’s first word (`a[0]`) since she always starts with the smallest.
3. Increment Alice’s pointer `i` to point to her next available word.
4. Enter a loop alternating turns:

1. On Bob’s turn, search from `j` to find the first word in `b` that is **closely greater** than `last_word`. If none exist, Alice wins.
2. Update `last_word` to the word Bob played and increment `j`.
3. On Alice’s turn, search from `i` for the first word in `a` that is **closely greater** than `last_word`. If none exist, Bob wins.
4. Update `last_word` and increment `i`.
5. Continue until one player cannot move.
6. Return `True` if Alice wins, `False` otherwise.

**Why it works**: The two-pointer approach works because the arrays are sorted. Since optimal play always involves picking the smallest valid word, this simulation guarantees that no other sequence could provide a better outcome. Lexicographical order ensures no valid word before the pointer can be played.

## Python Solution

```python
from typing import List

class Solution:
    def canAliceWin(self, a: List[str], b: List[str]) -> bool:
        def is_closely_greater(w: str, z: str) -> bool:
            if w <= z:
                return False
            return ord(w[0]) == ord(z[0]) or ord(w[0]) == ord(z[0]) + 1
        
        i, j = 1, 0  # Alice starts with a[0]
        last_word = a[0]
        turn = 1  # 1 for Bob, -1 for Alice

        while True:
            if turn == 1:  # Bob's turn
                while j < len(b) and not is_closely_greater(b[j], last_word):
                    j += 1
                if j == len(b):
                    return True
                last_word = b[j]
                j += 1
                turn = -1
            else:  # Alice's turn
                while i < len(a) and not is_closely_greater(a[i], last_word):
                    i += 1
                if i == len(a):
                    return False
                last_word = a[i]
                i += 1
                turn = 1
```

This Python solution first defines a helper function `is_closely_greater` to encapsulate the game rules. It then uses two pointers to simulate the game turn by turn. Alice starts, and the loop alternates turns until one player has no valid move, returning the outcome.

func canAliceWin(a []string, b []string) bool {
    isCloselyGreater := func(w, z string) bool {
        if w <= z {
            return false
        }
        return w[0] == z[0] || w[0] == z[0]+1
    }

    i, j := 1, 0
    lastWord := a[0]
    turn := 1 // 1 for Bob, -1 for Alice

    for {
        if turn == 1 {
            for j < len(b) && !isCloselyGreater(b[j], lastWord) {
                j++
            }
            if j == len(b) {
                return true
            }
            lastWord = b[j]
            j++
            turn = -1
        } else {
            for i < len(a) && !isCloselyGreater(a[i], lastWord) {
                i++
            }
            if i == len(a) {
                return false
            }
            lastWord = a[i]
            i++
            turn = 1
        }
    }
}
