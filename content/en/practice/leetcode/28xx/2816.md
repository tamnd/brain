## Solution

## Problem Understanding

The problem asks us to take a singly linked list representing a non-negative integer, where each node contains a single digit in **most significant to least significant order**, and return a new linked list representing **twice that number**. In other words, the input linked list `[1,8,9]` represents the number 189, and doubling it gives 378, which should be returned as `[3,7,8]`.

The input constraints tell us that the linked list contains at least one node and at most 10,000 nodes, with each node value between 0 and 9. The number does not have leading zeros except for the special case of the number 0. This guarantees that we do not need to handle malformed input like `[0,0,1]`.

Important edge cases to keep in mind include when doubling causes **carry-over that increases the number of digits**, such as `[9,9,9]` doubling to `[1,9,9,8]`. Another potential edge case is a single-digit node `[0]` which remains `[0]` after doubling.

The challenge lies in handling carry propagation efficiently while maintaining the original linked list structure or creating a new list without reversing unnecessarily.

## Approaches

A **brute-force approach** would convert the linked list into an integer, double it, and then convert it back into a linked list. This is conceptually simple but problematic because Python supports large integers, but in languages like Go, integers could overflow with 10,000-digit numbers. Additionally, converting to an integer and back is O(n) in space and time for very large inputs, and it loses the elegance of a linked list manipulation.

A more **optimal approach** uses a stack to reverse-traverse the linked list without modifying it. By pushing all nodes onto a stack, we can process the least significant digit first (top of the stack) and propagate carry as we go. After processing, we reconstruct a new linked list in the correct order. This avoids integer conversion and handles arbitrarily large numbers efficiently.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Convert list to integer, double, convert back |
| Optimal | O(n) | O(n) | Use stack to traverse from least significant digit, handle carry |

## Algorithm Walkthrough

1. Initialize an empty stack and traverse the linked list from head to tail, pushing each node's value onto the stack. This allows us to process the number from least significant to most significant digit.
2. Initialize a variable `carry = 0` to track carry-over during doubling.
3. Initialize a pointer `new_head = None` to build the resulting linked list in reverse order.
4. While the stack is not empty, pop the top value `val`, double it, and add the current `carry`.
5. Compute `new_digit = (val * 2 + carry) % 10` and update `carry = (val * 2 + carry) // 10`.
6. Create a new node with `new_digit` and set its next pointer to `new_head`. Update `new_head` to this new node.
7. After the stack is empty, check if `carry > 0`. If so, create a final node with `carry` as its value and link it to `new_head`.
8. Return `new_head` as the resulting linked list.

**Why it works:** The stack allows us to process the number from least significant digit to most significant digit without reversing the original linked list. Doubling with carry propagation is performed correctly at each digit, ensuring the final linked list represents exactly twice the original number.

## Python Solution

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def doubleIt(self, head: Optional[ListNode]) -> Optional[ListNode]:
        stack = []
        current = head
        while current:
            stack.append(current.val)
            current = current.next
        
        carry = 0
        new_head = None
        
        while stack:
            val = stack.pop()
            total = val * 2 + carry
            new_digit = total % 10
            carry = total // 10
            node = ListNode(new_digit)
            node.next = new_head
            new_head = node
        
        if carry > 0:
            node = ListNode(carry)
            node.next = new_head
            new_head = node
        
        return new_head
```

In this implementation, we first traverse the input linked list to build a stack of its digits. Then, we process each digit from least significant to most significant, applying doubling and carry propagation. Finally, we construct a new linked list by prepending each new node to maintain the correct order. This ensures O(n) time and space complexity without integer conversion.

## Go Solution

```go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func doubleIt(head *ListNode) *ListNode {
    stack := []*ListNode{}
    current := head
    for current != nil {
        stack = append(stack, current)
        current = current.Next
    }
    
    var newHead *ListNode = nil
    carry := 0
    
    for len(stack) > 0 {
        n := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        total := n.Val*2 + carry
        newDigit := total % 10
        carry = total / 10
        node := &ListNode{Val: newDigit, Next: newHead}
        newHead = node
    }
    
    if carry > 0 {
        node := &ListNode{Val: carry, Next: newHead}
        newHead = node
    }
    
    return newHead
}
```

The Go version mirrors the Python solution. The main difference is that slices are used as a stack, and memory is explicitly allocated for each new `ListNode`. Handling `nil` pointers is equivalent to Python's `None`. Carry propagation logic and list reconstruction are identical.

## Complexity Analysis

| Time | O(n) | We traverse the list twice: once to build the stack, once to process doubling and carry propagation. |
| Space | O(n) | Stack stores all nodes, and new linked list requires O(n) additional space. |

The algorithm avoids integer conversion and handles arbitrarily large numbers efficiently while maintaining linear time and space complexity.
