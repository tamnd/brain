## Solution

## Problem Understanding

The problem asks us to calculate the "sum of remoteness" for all cells in a given `n x n` grid where each cell is either a positive integer or blocked with `-1`. The remoteness `R[i][j]` for a non-blocked cell `(i, j)` is defined as the sum of the values of all non-blocked cells that **cannot reach `(i, j)` via a path of adjacent non-blocked cells**. For blocked cells, the remoteness is 0.

In simpler terms, two non-blocked cells are considered connected if there exists a path of non-blocked cells between them. For each cell, we sum the values of all cells that are **not in the same connected component**. The task is to compute the total sum of `R[i][j]` across the entire grid.

The input is an `n x n` matrix with `1 <= n <= 300` and cell values between 1 and 10^6 or `-1` for blocked cells. Since `n` can be as large as 300, a naive approach that computes reachability from each cell individually will be too slow.

Important edge cases include:

1. The entire grid is blocked except one cell.
2. A single cell grid.
3. Multiple disconnected clusters of non-blocked cells.
4. Large grids with many blocked cells forming isolated components.

## Approaches

The brute-force approach would be to run a BFS/DFS from each non-blocked cell to determine which cells it can reach, then sum the values of unreachable cells. This is correct but too slow because for each of up to `n^2` cells, you might traverse almost the entire grid, giving `O(n^4)` complexity.

The key insight for optimization is to realize that remoteness depends on **connected components** of non-blocked cells. If we precompute the total sum of all non-blocked cells, and the sum of values in each connected component, we can calculate the remoteness for any cell `(i, j)` as the total sum of non-blocked cells **minus the sum of its own component**. This avoids repeated BFS/DFS traversals for every cell.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^4) | O(n^2) | Run BFS/DFS from each cell to determine unreachable cells. Too slow for n=300. |
| Optimal | O(n^2) | O(n^2) | Identify connected components once, compute their sums, then compute remoteness per cell using component sums. |

## Algorithm Walkthrough

1. Initialize a 2D `visited` array to track which cells have been processed and a `component_id` matrix to store which connected component each cell belongs to.
2. Traverse the grid. For each unvisited non-blocked cell, perform a BFS or DFS to find all connected non-blocked cells, assigning them a unique component id and calculating the sum of their values. Store these sums in a dictionary `component_sums`.
3. Calculate the total sum `total_sum` of all non-blocked cell values.
4. Traverse the grid again. For each non-blocked cell `(i, j)`, its remoteness `R[i][j]` is `total_sum - component_sums[component_id[i][j]]`. For blocked cells, `R[i][j] = 0`.
5. Sum all values of `R[i][j]` to get the final answer.

Why it works: The invariant is that all cells in the same connected component can reach each other. Therefore, the sum of unreachable cell values for a given cell is exactly the sum of values not in its component.

## Python Solution

```python
from typing import List
from collections import deque

class Solution:
    def sumRemoteness(self, grid: List[List[int]]) -> int:
        n = len(grid)
        visited = [[False] * n for _ in range(n)]
        component_id = [[-1] * n for _ in range(n)]
        component_sums = {}
        comp_counter = 0
        
        directions = [(0,1), (1,0), (0,-1), (-1,0)]
        
        for i in range(n):
            for j in range(n):
                if grid[i][j] != -1 and not visited[i][j]:
                    queue = deque([(i, j)])
                    visited[i][j] = True
                    component_id[i][j] = comp_counter
                    comp_sum = grid[i][j]
                    
                    while queue:
                        x, y = queue.popleft()
                        for dx, dy in directions:
                            nx, ny = x + dx, y + dy
                            if 0 <= nx < n and 0 <= ny < n:
                                if grid[nx][ny] != -1 and not visited[nx][ny]:
                                    visited[nx][ny] = True
                                    component_id[nx][ny] = comp_counter
                                    comp_sum += grid[nx][ny]
                                    queue.append((nx, ny))
                    
                    component_sums[comp_counter] = comp_sum
                    comp_counter += 1
        
        total_sum = sum(grid[i][j] for i in range(n) for j in range(n) if grid[i][j] != -1)
        result = 0
        for i in range(n):
            for j in range(n):
                if grid[i][j] != -1:
                    result += total_sum - component_sums[component_id[i][j]]
        return result
```

This implementation first identifies connected components using BFS, sums the values for each component, and finally computes the remoteness for each cell using the precomputed sums.

## Go Solution

```go
package main

func sumRemoteness(grid [][]int) int64 {
    n := len(grid)
    visited := make([][]bool, n)
    componentID := make([][]int, n)
    for i := range grid {
        visited[i] = make([]bool, n)
        componentID[i] = make([]int, n)
        for j := range grid[i] {
            componentID[i][j] = -1
        }
    }
    componentSums := make(map[int]int)
    compCounter := 0
    directions := [][2]int{{0,1},{1,0},{0,-1},{-1,0}}
    
    var queue [][2]int
    
    for i := 0; i < n; i++ {
        for j := 0; j < n; j++ {
            if grid[i][j] != -1 && !visited[i][j] {
                queue = queue[:0]
                queue = append(queue, [2]int{i, j})
                visited[i][j] = true
                componentID[i][j] = compCounter
                compSum := grid[i][j]
                
                for len(queue) > 0 {
                    x, y := queue[0][0], queue[0][1]
                    queue = queue[1:]
                    for _, d := range directions {
                        nx, ny := x + d[0], y + d[1]
                        if nx >= 0 && nx < n && ny >= 0 && ny < n {
                            if grid[nx][ny] != -1 && !visited[nx][ny] {
                                visited[nx][ny] = true
                                componentID[nx][ny] = compCounter
                                compSum += grid[nx][ny]
                                queue = append(queue, [2]int{nx, ny})
                            }
                        }
                    }
                }
                
                componentSums[compCounter] = compSum
                compCounter++
            }
        }
    }
    
    var totalSum int64 = 0
    for i := 0; i < n; i++ {
        for j := 0; j < n; j++ {
            if grid[i][j] != -1 {
                totalSum += int64(grid[i][j])
            }
        }
    }
    
    var result int64 = 0
    for i := 0; i < n; i++ {
        for j := 0; j < n; j++ {
            if grid[i][j] != -1 {
                result += totalSum - int64(componentSums[componentID[i][j]])
            }
        }
    }
    
    return result
}
```

The Go implementation mirrors the Python logic, using slices for the BFS queue and maps for component sums. `int64` is used to prevent overflow due to large grid values.
