## Solution

## Problem Understanding

The problem asks us to process a `DataFrame` of product information that contains three columns: `name`, `quantity`, and `price`. The `quantity` column may contain missing values represented as `None`. The task is to replace every missing value in the `quantity` column with `0` while keeping all other data intact. The expected output is a modified `DataFrame` with no missing values in `quantity`.

The input represents a structured table where each row corresponds to a product and columns represent attributes of the product. Missing data in `quantity` can occur due to incomplete records, and filling it with `0` is a standard data cleaning technique.

Edge cases include a `DataFrame` where all quantities are missing, none are missing, or the `DataFrame` is empty. The problem guarantees that the `quantity` column is integer typed with potential `None` values, so we do not have to handle unexpected data types. This is a small-scale problem without strict performance constraints, as filling missing values is a straightforward operation in data manipulation libraries.

## Approaches

The brute-force approach is to iterate over every row in the `DataFrame` manually and check if the `quantity` is `None`. If it is, we replace it with `0`. This works correctly because each row is examined individually. However, iterating manually over rows in Python can be inefficient for large datasets because it does not leverage vectorized operations provided by `pandas` or SQL-like systems.

The optimal approach leverages the built-in `fillna` method from the `pandas` library. This method is designed for efficiently filling missing values across entire columns or the entire `DataFrame`. It is vectorized, making it much faster than manual iteration, especially for large datasets. The key insight is that `fillna` handles the replacement of `None` (or `NaN`) values directly and safely, preserving the existing values without additional conditional logic.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Iterate row by row, check for None, replace with 0 |
| Optimal | O(n) | O(1) | Use `pandas.DataFrame.fillna()` for vectorized replacement |

## Algorithm Walkthrough

1. Import the `pandas` library, which provides the `DataFrame` class and `fillna` method.
2. Access the `quantity` column of the `DataFrame`.
3. Apply the `fillna(0)` method to the `quantity` column. This scans each value in the column and replaces any `None` with `0`.
4. Assign the result back to the `quantity` column to update the `DataFrame`.
5. Return the updated `DataFrame`.

Why it works: The algorithm works because `fillna` guarantees that all missing values in the specified column are replaced while leaving non-missing values unchanged. The operation is vectorized, meaning that it applies the replacement across all rows efficiently and consistently.

## Python Solution

```python
import pandas as pd

class Solution:
    def fillMissing(self, products: pd.DataFrame) -> pd.DataFrame:
        products['quantity'] = products['quantity'].fillna(0)
        return products
```

In this Python implementation, we first access the `quantity` column of the `products` DataFrame. The `fillna(0)` method replaces all missing values with `0`. Finally, we return the modified DataFrame. This corresponds directly to the optimal algorithm steps outlined above.

## Go Solution

```go
package main

type Product struct {
    Name     string
    Quantity *int
    Price    int
}

func fillMissing(products []Product) []Product {
    for i := range products {
        if products[i].Quantity == nil {
            zero := 0
            products[i].Quantity = &zero
        }
    }
    return products
}
```

In Go, we represent each product as a struct with `Quantity` being a pointer to `int` to allow `nil` values. The function iterates over the slice of products and checks if `Quantity` is `nil`. If it is, it creates a local variable `zero` and assigns its address to `Quantity`. This approach mirrors the Python solution but requires explicit pointer handling since Go does not have `None` or `NaN` for integers.

## Edge Cases

One edge case is a DataFrame with all quantities missing. This could potentially lead to errors if the code assumes at least one non-missing value. The implementation handles it correctly because `fillna` replaces all missing values regardless of count.

A second edge case is a DataFrame where no quantities are missing. The function must leave the data unchanged. Using `fillna` ensures that existing non-missing values remain intact.

A third edge case is an empty DataFrame with the correct columns but no rows. Iterating over an empty column or applying `fillna` should not cause errors, and the implementation safely returns an empty DataFrame, preserving column names.
