## Solution

## Problem Understanding

The problem is about maximizing the number of alloys a company can produce given a set of machines, metal stocks, costs, and a budget. Each machine has a required composition for each metal type, and you can choose **only one machine** to produce all alloys. You can use your existing stock of metals and purchase additional metals if necessary, but the total cost cannot exceed the budget.

The input provides:

- `n`, the number of different metal types.
- `k`, the number of machines available.
- `budget`, the maximum coins you can spend buying metals.
- `composition`, a 1-indexed 2D array where `composition[i][j]` represents the units of metal type `j` required by machine `i` to produce one alloy.
- `stock`, a 1-indexed array indicating current units of each metal.
- `cost`, a 1-indexed array specifying the cost of one unit of each metal.

The expected output is an integer representing the maximum number of alloys that can be produced using **one machine**, without exceeding the budget.

Constraints indicate that `n` and `k` are at most 100, budget can be very large (up to 10^8), and metal requirements and costs are moderate. This suggests that any solution with O(k * n * log(max_possible_alloys)) time complexity is feasible.

Edge cases include:

1. Stock is sufficient to produce many alloys without buying.
2. Some metals have zero stock but high cost.
3. The budget is zero.
4. Machine requirements vary drastically, making one machine much more cost-effective.

## Approaches

The **brute-force approach** would be to try every machine and attempt to produce increasing numbers of alloys until the budget is exceeded. For each machine and target number of alloys, you compute the total purchase cost of metals, which is linear in `n`. This approach is simple but inefficient, as it may require iterating through all possible numbers of alloys up to very high numbers, potentially 10^8, which is not feasible.

The **optimal approach** leverages **binary search**. The key insight is that for a given machine, the total cost to produce `x` alloys is monotonic: producing more alloys requires equal or more coins. Therefore, for each machine, we can binary search over the number of alloys `x` from 0 to a reasonable upper bound (e.g., based on budget and metal requirements) to find the maximum number producible within the budget. For each candidate `x`, we calculate the total cost using the formula:

```
cost_for_alloys = sum(max(0, x * composition[i][j] - stock[j]) * cost[j] for each metal type j)
```

We repeat this for all machines and take the maximum across all machines.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(k * n * max_possible_alloys) | O(1) | Incrementally checks each possible number of alloys per machine until budget exceeded |
| Optimal | O(k * n * log(max_alloys_possible)) | O(1) | Binary search per machine using monotonic cost property |

## Algorithm Walkthrough

1. Initialize `max_alloys` to 0 to store the result.
2. Iterate through each machine `i` from 0 to `k-1`.
3. For machine `i`, perform a binary search for the number of alloys `x` between `0` and a large upper bound (e.g., `budget // min(cost) + max(stock)`).
4. For a candidate `x`, calculate the total cost: for each metal type `j`, compute `needed = max(0, x * composition[i][j] - stock[j])` and multiply by `cost[j]`. Sum over all metals.
5. If the total cost ≤ budget, `x` is feasible, move the binary search lower bound up to try more alloys. Otherwise, move the upper bound down.
6. Record the largest feasible `x` for the current machine.
7. After checking all machines, return the maximum feasible `x` found.

**Why it works**: The cost function is monotonic with respect to the number of alloys. Binary search ensures we efficiently find the maximum number producible without exceeding the budget. Checking all machines guarantees we select the most cost-effective machine.

## Python Solution

```python
from typing import List

class Solution:
    def maxNumberOfAlloys(self, n: int, k: int, budget: int, composition: List[List[int]], stock: List[int], cost: List[int]) -> int:
        max_alloys = 0
        
        for i in range(k):
            left, right = 0, 10**12  # large upper bound for binary search
            while left <= right:
                mid = (left + right) // 2
                total_cost = 0
                for j in range(n):
                    need = max(0, mid * composition[i][j] - stock[j])
                    total_cost += need * cost[j]
                    if total_cost > budget:
                        break
                if total_cost <= budget:
                    left = mid + 1
                else:
                    right = mid - 1
            max_alloys = max(max_alloys, right)
        
        return max_alloys
```

The implementation iterates over each machine, using binary search to determine the maximum number of alloys feasible for that machine. The inner loop calculates the cost of producing `mid` alloys. If the cost exceeds the budget, binary search narrows the range. After checking all machines, the highest feasible number of alloys is returned.

## Go Solution

```go
func maxNumberOfAlloys(n int, k int, budget int, composition [][]int, stock []int, cost []int) int {
    maxAlloys := 0
    
    for i := 0; i < k; i++ {
        left, right := 0, int(1e12)
        for left <= right {
            mid := (left + right) / 2
            totalCost := 0
            for j := 0; j < n; j++ {
                need := mid*composition[i][j] - stock[j]
                if need < 0 {
                    need = 0
                }
                totalCost += need * cost[j]
                if totalCost > budget {
                    break
                }
            }
            if totalCost <= budget {
                left = mid + 1
            } else {
                right = mid - 1
            }
        }
        if right > maxAlloys {
            maxAlloys = right
        }
    }
    
    return maxAlloys
}
```

The Go implementation mirrors the Python logic. Integer arithmetic is handled carefully to avoid overflow; we use `int(1e12)` as a safe upper bound. Slice indexing is zero-based, similar to Python lists. Early breaking from the loop avoids unnecessary calculations if the cost already exceeds the budget.

## Worked Examples

### Example 1

`n=3, k=2, budget=15, composition=[[1,1,1],[1,1,10]], stock=[0,0,0], cost=[1,2,3]`

- Machine 0:

- Binary search `x`:

- mid=7 → cost=7_1+7_2+7*3=42 > budget → adjust right
- mid=3 → cost=3_1+3_2+3*3=18 > budget → adjust right
- mid=2 → cost=2_1+2_2+2*3=12 ≤ budget → feasible
- Maximum alloys for machine 0: 2
- Machine 1:

- mid=2 → cost=2_1+2_2+2*10=26 > budget → adjust right
- mid=1 → cost=1_1+1_2+1*10=13 ≤ budget → feasible
- Maximum alloys for machine 1: 1

Result: 2 alloys

- Machine 0: maximum 2 alloys (same as before)

### Example 3

Result: 2 alloys
