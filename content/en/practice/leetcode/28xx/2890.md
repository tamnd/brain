## Solution

## Problem Understanding

This problem asks us to reshape a pandas DataFrame from a wide format into a long format. The input DataFrame contains one row per product, and each quarter's sales are stored in separate columns such as `quarter_1`, `quarter_2`, `quarter_3`, and `quarter_4`.

The goal is to transform the structure so that each row represents exactly one product-quarter pair. Instead of having multiple quarter columns, we want two columns:

- `quarter`, which stores the quarter name
- `sales`, which stores the corresponding sales value

The `product` column remains unchanged because it identifies the product associated with each sales value.

For example, if a product has four quarterly sales values, the reshaped output will contain four rows for that product, one for each quarter.

The input represents a classic wide-to-long transformation problem. Wide format stores different categories as separate columns, while long format stores category names and values as rows. This type of reshaping is very common in data analysis and is directly supported by pandas through the `melt()` function.

The constraints are small because this is an Easy pandas problem. The main focus is understanding how reshaping works rather than optimizing for performance. The problem guarantees that the quarter columns exist and that the input DataFrame is valid.

Several edge cases are important to consider:

- A DataFrame with only one product should still correctly generate one row per quarter.
- A DataFrame with zero sales values should preserve those values exactly.
- If the DataFrame contains many products, the reshaping should still preserve the ordering and correctly associate products with quarters.
- The implementation must not lose any rows or mismatch sales values with the wrong quarter.

## Approaches

### Brute Force Approach

A brute force solution would manually iterate through every row of the DataFrame and then iterate through every quarter column. For each `(product, quarter)` pair, we could build a new dictionary or list representing the transformed row.

After processing all rows, we could create a new DataFrame from the accumulated records.

This approach works because it explicitly constructs every required output row. However, it is unnecessarily verbose and does not take advantage of pandas' built-in reshaping operations. Manual iteration over DataFrame rows is also slower and less idiomatic in pandas.

### Optimal Approach

The optimal solution uses pandas' `melt()` function.

The key observation is that this problem is exactly a wide-to-long transformation. Pandas already provides a specialized operation for this pattern.

The `melt()` function works by:

- Keeping one or more identifier columns fixed
- Converting the remaining columns into row entries
- Creating one column for the original column names
- Creating another column for the original values

In this problem:

- `product` is the identifier column
- The quarter columns are melted into rows
- The new column names become `quarter`
- The sales numbers become `sales`

This produces the desired output directly in a concise and efficient way.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × q) | O(n × q) | Manually iterates through products and quarters |
| Optimal | O(n × q) | O(n × q) | Uses pandas `melt()` for direct reshaping |

Here:

- `n` is the number of products
- `q` is the number of quarter columns

## Algorithm Walkthrough

1. Start with the original DataFrame containing one row per product and multiple quarter columns.
2. Identify the column that should remain fixed during reshaping. In this problem, the `product` column uniquely identifies each product, so it becomes the identifier column.
3. Call the pandas `melt()` function on the DataFrame.
4. Set `id_vars=["product"]` so that the `product` column remains unchanged in the output.
5. Set `var_name="quarter"` so that the names of the melted columns are stored in a new column called `quarter`.
6. Set `value_name="sales"` so that the actual sales numbers are stored in a new column called `sales`.
7. Return the reshaped DataFrame.

### Why it works

The `melt()` operation systematically converts every quarter column into rows while preserving the associated product value. Each original cell in the quarter columns becomes exactly one row in the output. Since pandas maintains the mapping between identifiers and values during the transformation, every sales value remains correctly associated with its product and quarter.

## Python Solution

```python
import pandas as pd

class Solution:
    def meltTable(self, report: pd.DataFrame) -> pd.DataFrame:
        return report.melt(
            id_vars=["product"],
            var_name="quarter",
            value_name="sales"
        )
```

The implementation is extremely concise because pandas already provides a dedicated reshaping function.

The `id_vars=["product"]` argument tells pandas that the `product` column should remain fixed. All other columns are automatically transformed into rows.

The `var_name="quarter"` argument specifies the name of the column that will contain the original quarter column names.

The `value_name="sales"` argument specifies the name of the column that will contain the corresponding sales values.

The returned DataFrame contains exactly the required structure.

## Go Solution

```go
package main

import "fmt"

type Row struct {
	Product string
	Quarter string
	Sales   int
}

func meltTable(report []map[string]interface{}) []Row {
	result := []Row{}

	quarters := []string{
		"quarter_1",
		"quarter_2",
		"quarter_3",
		"quarter_4",
	}

	for _, record := range report {
		product := record["product"].(string)

		for _, quarter := range quarters {
			sales := record[quarter].(int)

			result = append(result, Row{
				Product: product,
				Quarter: quarter,
				Sales:   sales,
			})
		}
	}

	return result
}

func main() {
	report := []map[string]interface{}{
		{
			"product":   "Umbrella",
			"quarter_1": 417,
			"quarter_2": 224,
			"quarter_3": 379,
			"quarter_4": 611,
		},
		{
			"product":   "SleepingBag",
			"quarter_1": 800,
			"quarter_2": 936,
			"quarter_3": 93,
			"quarter_4": 875,
		},
	}

	result := meltTable(report)

	for _, row := range result {
		fmt.Println(row)
	}
}
```

Unlike Python pandas, Go does not provide a built-in DataFrame reshaping operation. Therefore, the Go implementation manually iterates through the records and quarter fields.

Slices are used to store the output rows dynamically. Type assertions are required because the input records are stored as `map[string]interface{}`.

There are no integer overflow concerns here because the sales values are small standard integers.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × q) | Every product-quarter combination is processed once |
| Space | O(n × q) | The reshaped DataFrame stores one row per product-quarter pair |

The algorithm must create a new row for every quarter value associated with every product. Since there are `n` products and `q` quarter columns, the total number of output rows is `n × q`. Both the runtime and memory usage scale proportionally with this output size.

## Edge Cases

One important edge case is a DataFrame containing only a single product. A naive implementation might accidentally collapse dimensions or return only one row instead of one row per quarter. The current implementation handles this correctly because `melt()` independently processes every quarter column regardless of the number of products.

Another important case is when sales values are zero. Some incorrect implementations may mistakenly filter out falsy values or treat zero as missing data. The pandas `melt()` function preserves all original values exactly, including zeros.

A third edge case involves maintaining the correct association between products and quarters. During reshaping, it is easy to accidentally mismatch sales values with the wrong product if rows are processed incorrectly. The `melt()` operation guarantees that every value remains tied to its original identifier column, preventing such mismatches automatically.
