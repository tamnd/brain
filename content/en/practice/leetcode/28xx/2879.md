## Solution

## Problem Understanding

This problem provides a Pandas DataFrame named `employees` with four columns:

- `employee_id`, an integer identifier for each employee
- `name`, a string containing the employee's name
- `department`, a string describing the employee's department
- `salary`, an integer salary value

The task is extremely straightforward. We must return only the first three rows of the DataFrame.

In other words, given a table of employee records, we need to extract the rows at indices `0`, `1`, and `2`, assuming they exist. The output should preserve all columns and maintain the original order of rows.

The problem belongs to the Pandas category on LeetCode, so the expected solution uses standard Pandas DataFrame operations instead of manual iteration or custom data structures.

The constraints are minimal because this is an Easy problem focused on familiarity with DataFrame operations rather than algorithmic optimization. Still, it is useful to think about behavior in edge situations.

One important detail is that the problem asks for the "first 3 rows", not rows matching any condition. This means the operation depends purely on row order.

Potential edge cases include:

- The DataFrame contains exactly three rows
- The DataFrame contains fewer than three rows
- The DataFrame is empty
- The DataFrame contains many rows

Pandas handles all these situations naturally. Calling `.head(3)` on a DataFrame with fewer than three rows simply returns all available rows without errors.

## Approaches

### Brute Force Approach

A brute force solution would manually iterate through the DataFrame row by row, collect the first three rows into a new structure, and then construct another DataFrame from those rows.

Conceptually, the algorithm would:

1. Start with an empty list
2. Iterate through each row of the DataFrame
3. Stop after collecting three rows
4. Create a new DataFrame containing those rows

This works because it explicitly selects the first three entries in order. However, it is unnecessarily verbose for a problem that Pandas already solves with a built in operation.

Although the runtime is still effectively linear in the number of accessed rows, the approach introduces unnecessary complexity and additional memory usage.

### Optimal Approach

The optimal solution uses the Pandas `.head()` method:

```
employees.head(3)
```

The key observation is that Pandas already provides a built in API specifically designed for retrieving the first `n` rows of a DataFrame efficiently and clearly.

This approach is preferable because:

- It is concise
- It is idiomatic Pandas code
- It handles edge cases automatically
- It avoids manual iteration
- It preserves DataFrame structure and metadata

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(3) | O(3) | Manually iterates and copies first three rows |
| Optimal | O(3) | O(3) | Uses built in Pandas `.head(3)` method |

Since only three rows are requested, both approaches effectively operate in constant time relative to input size.

## Algorithm Walkthrough

### Optimal Algorithm

1. Receive the `employees` DataFrame as input.

The DataFrame already contains all employee information in tabular form, so no preprocessing is necessary.
2. Call the `.head(3)` method on the DataFrame.

The `.head(n)` method returns the first `n` rows while preserving all columns and row ordering.
3. Return the resulting DataFrame.

Pandas automatically handles situations where fewer than three rows exist.

### Why it works

The `.head(3)` operation is guaranteed to return the first three rows in the DataFrame's current order. Since the problem explicitly asks for the first three rows without modification or filtering, this directly satisfies the requirement.

## Python Solution

```python
import pandas as pd

def selectFirstRows(employees: pd.DataFrame) -> pd.DataFrame:
    return employees.head(3)
```

The implementation is intentionally minimal because Pandas already provides the exact operation needed.

The function receives a DataFrame named `employees`. Calling `employees.head(3)` extracts the first three rows while keeping the original structure intact.

The returned object is also a DataFrame, which matches the expected LeetCode output format.

## Go Solution

LeetCode Pandas problems are designed specifically for Python and Pandas, so there is no official Go submission format for this problem. However, the equivalent logic in Go can be demonstrated conceptually using slices.

```go
package main

import "fmt"

type Employee struct {
	EmployeeID int
	Name       string
	Department string
	Salary     int
}

func selectFirstRows(employees []Employee) []Employee {
	if len(employees) < 3 {
		return employees
	}

	return employees[:3]
}

func main() {
	employees := []Employee{
		{3, "Bob", "Operations", 48675},
		{90, "Alice", "Sales", 11096},
		{9, "Tatiana", "Engineering", 33805},
		{60, "Annabelle", "IT", 37678},
	}

	result := selectFirstRows(employees)

	fmt.Println(result)
}
```

The Go version uses slice operations instead of DataFrame methods.

A safety check is included to avoid slicing beyond the array bounds when fewer than three employees exist. Unlike Pandas `.head(3)`, Go slices require explicit bounds handling.

## Edge Cases

One important edge case occurs when the DataFrame contains fewer than three rows. A naive implementation using direct indexing could attempt to access rows that do not exist, causing errors. The `.head(3)` method avoids this problem because Pandas automatically returns all available rows when fewer than the requested number exist.

Another edge case is an empty DataFrame. Some manual iteration approaches may fail if they assume at least one row exists. In this implementation, `.head(3)` simply returns another empty DataFrame with the same columns, which is the correct behavior.

A third edge case involves very large DataFrames. A poor implementation might unnecessarily copy or iterate through the entire dataset before selecting rows. The Pandas `.head()` method is optimized to retrieve only the required rows, making it efficient even for large inputs.
