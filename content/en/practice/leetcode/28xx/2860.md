# LeetCode 2860 - Happy Students

## Problem Understanding

The problem gives us an array `nums` where `nums[i]` represents a condition for the `i-th` student. We want to choose some students to form a selected group such that every student in the class is happy.

A student can be happy in exactly one of two ways:

- If the student is selected, then the total number of selected students must be strictly greater than `nums[i]`.
- If the student is not selected, then the total number of selected students must be strictly less than `nums[i]`.

We are asked to count how many different valid group sizes and selections satisfy these rules for every student simultaneously.

The key observation is that happiness depends only on the total number of selected students, not on the identities of the selected students themselves. If the selected group size is `k`, then:

- Every selected student must satisfy `nums[i] < k`
- Every unselected student must satisfy `nums[i] > k`

This means no student can have `nums[i] == k`, because:

- If selected, they need `k > nums[i]`, which fails
- If unselected, they need `k < nums[i]`, which also fails

The constraints are important:

- `1 <= nums.length <= 10^5`
- `0 <= nums[i] < nums.length`

Since `n` can be as large as `100000`, exponential or quadratic approaches are too slow. We need something around `O(n log n)` or better.

Several edge cases are important:

- Selecting nobody, meaning `k = 0`
- Selecting everybody, meaning `k = n`
- Duplicate values in the array
- Cases where some `nums[i]` equals the candidate group size
- Arrays where no valid intermediate group size exists

Carefully handling these situations is necessary for correctness.

## Approaches

### Brute Force Approach

A brute force solution would try every possible subset of students. For each subset:

1. Count how many students are selected.
2. Check every student to determine whether they are happy.
3. Count the subset if all students are satisfied.

This approach is correct because it directly follows the problem definition. However, there are `2^n` possible subsets, which becomes impossible for `n = 100000`.

Even checking all possible group sizes and trying all combinations for each size would still be computationally infeasible.

### Key Insight for the Optimal Solution

The important observation is that only the number of selected students matters, not which specific students are selected.

Suppose exactly `k` students are selected.

For the configuration to work:

- Selected students must have values strictly smaller than `k`
- Unselected students must have values strictly larger than `k`

Therefore:

- Exactly `k` students must satisfy `nums[i] < k`
- The remaining `n - k` students must satisfy `nums[i] > k`

If we sort the array, we can efficiently determine whether a particular `k` works.

After sorting:

- The first `k` elements would need to be strictly smaller than `k`
- The remaining elements would need to be strictly larger than `k`

This translates into two conditions:

- `nums[k - 1] < k`
- `nums[k] > k`

with careful boundary handling for `k = 0` and `k = n`.

Sorting allows us to test every possible group size efficiently.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Tries every subset and validates happiness |
| Optimal | O(n log n) | O(1) or O(log n) | Sorts array and checks valid group sizes |

## Algorithm Walkthrough

### Step 1: Sort the Array

Sort `nums` in non-decreasing order.

Sorting groups students by their happiness thresholds, making it easy to determine how many students have values less than a candidate group size `k`.

For example:

```
nums = [6,0,3,3,6,7,2,7]
sorted = [0,2,3,3,6,6,7,7]
```

### Step 2: Check the Case Where Nobody Is Selected

If `k = 0`, then every student must be unselected and happy.

For an unselected student to be happy:

```
0 < nums[i]
```

Therefore, all values must be strictly positive.

In sorted order, this means:

```
nums[0] > 0
```

If true, increment the answer.

### Step 3: Check Every Possible Group Size From 1 to n - 1

For each candidate group size `k`:

- The first `k` students should have values strictly smaller than `k`
- The remaining students should have values strictly larger than `k`

Because the array is sorted, we only need to check the boundary:

```
nums[k - 1] < k < nums[k]
```

If both conditions hold, then exactly `k` students can be selected happily.

Increment the answer.

### Step 4: Check the Case Where Everyone Is Selected

If `k = n`, every student is selected.

Each selected student must satisfy:

```
n > nums[i]
```

The constraints already guarantee:

```
nums[i] < n
```

So selecting everyone is always valid.

Increment the answer.

### Why it works

After sorting, the first `k` elements are the only candidates that could belong to a selected group of size `k`, because they are the smallest thresholds.

If the largest among those selected students still satisfies `nums[k - 1] < k`, then all selected students are happy.

Similarly, if the smallest unselected student satisfies `nums[k] > k`, then all unselected students are happy.

These two boundary checks completely determine validity because of the sorted order.

## Python Solution

```python
from typing import List

class Solution:
    def countWays(self, nums: List[int]) -> int:
        nums.sort()
        n = len(nums)

        ways = 0

        # Case: select nobody
        if nums[0] > 0:
            ways += 1

        # Cases: select k students where 1 <= k < n
        for k in range(1, n):
            if nums[k - 1] < k and nums[k] > k:
                ways += 1

        # Case: select everybody
        ways += 1

        return ways
```

The implementation begins by sorting the array so that all threshold values are ordered from smallest to largest. This ordering allows us to reason about how many students can belong to a valid selected group.

The variable `ways` stores the number of valid group selections.

The first condition checks whether selecting zero students works. This is only possible when every student has a threshold greater than zero, which becomes `nums[0] > 0` after sorting.

Next, the loop iterates through all intermediate group sizes from `1` to `n - 1`. For each candidate size `k`, we verify:

- The largest selected threshold is smaller than `k`
- The smallest unselected threshold is larger than `k`

These conditions guarantee that every student is happy.

Finally, selecting all students is always valid because every `nums[i]` is strictly smaller than `n`.

## Go Solution

```go
package main

import "sort"

func countWays(nums []int) int {
	sort.Ints(nums)

	n := len(nums)
	ways := 0

	// Case: select nobody
	if nums[0] > 0 {
		ways++
	}

	// Cases: select k students where 1 <= k < n
	for k := 1; k < n; k++ {
		if nums[k-1] < k && nums[k] > k {
			ways++
		}
	}

	// Case: select everybody
	ways++

	return ways
}
```

The Go implementation follows the exact same logic as the Python version.

The built in `sort.Ints` function sorts the slice in ascending order. Go slices are dynamically sized views over arrays, so no extra copying is required.

There are no integer overflow concerns because all values remain within the constraints of standard Go integers.

The algorithm uses only constant extra memory aside from the sorting implementation.

## Edge Cases

### Selecting Nobody

When `k = 0`, every student is unselected. For everyone to be happy, every threshold must be greater than zero.

This case is easy to overlook because the general boundary formula uses `nums[k - 1]`, which does not exist for `k = 0`.

The implementation handles this separately using:

```
if nums[0] > 0:
```

### Selecting Everybody

When `k = n`, there are no unselected students. Every selected student only needs:

```
n > nums[i]
```

Since the constraints guarantee `nums[i] < n`, this case is always valid.

The implementation correctly adds one final valid configuration unconditionally.

### Duplicate Values Around the Boundary

Cases like:

```
[1,1]
```

or

```
[3,3,3]
```

can easily break incorrect solutions.

If any student has `nums[i] == k`, then a group of size `k` is impossible because neither the selected nor unselected condition can be satisfied.

The implementation handles this naturally through the strict inequalities:

```
nums[k - 1] < k and nums[k] > k
```

which automatically reject equal values at the boundary.
