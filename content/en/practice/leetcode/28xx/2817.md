## Solution

## Problem Understanding

The problem gives us an integer array `nums` and an integer `x`. We need to find two elements in the array whose indices are at least `x` positions apart, while minimizing the absolute difference between their values.

Formally, we want to find indices `i` and `j` such that:

$$|i - j| \ge x$$

and among all such valid pairs, minimize:

$$|nums[i] - nums[j]|$$

The output is the smallest possible absolute difference.

The array can contain up to `10^5` elements, and each value can be as large as `10^9`. These constraints immediately tell us that a quadratic solution will not work efficiently. A brute force approach would examine every valid pair, resulting in roughly `10^10` comparisons in the worst case, which is far too slow.

An important detail is that the constraint is on the distance between indices, not on the values themselves. This means that as we scan through the array, we need an efficient way to compare the current value against previously eligible values.

Several edge cases are important:

- If `x = 0`, then every pair of indices is valid, including the same index. Since `|nums[i] - nums[i]| = 0`, the answer is immediately `0`.
- Arrays may contain duplicate values. If two equal values satisfy the index distance constraint, the answer becomes `0`.
- The minimum difference may come from values that are not adjacent in the original array.
- Large input size means we must use a near linear or `O(n log n)` solution.

## Approaches

### Brute Force Approach

The most direct solution is to examine every valid pair of indices.

For every index `i`, we iterate through every index `j` such that `|i - j| >= x`. For each valid pair, we compute:

$$|nums[i] - nums[j]|$$

and keep track of the minimum value encountered.

This approach is correct because it explicitly checks all possible valid pairs, guaranteeing that the minimum difference will be found.

However, this method is too slow. There are `O(n^2)` possible pairs, and with `n = 10^5`, the number of comparisons becomes impractical.

### Optimal Approach

The key observation is that for a fixed element `nums[i]`, we only need to find the previously eligible number that is closest in value.

Suppose we are processing index `i`. Any index `j` is valid if:

$$i - j \ge x$$

which means:

$$j \le i - x$$

As we move from left to right, we can maintain a sorted collection of all values whose indices are far enough away from the current position.

For the current value `nums[i]`, the closest number in value inside the sorted structure must be either:

- the first number greater than or equal to `nums[i]`
- or the largest number smaller than `nums[i]`

This is a classic ordered set or binary search problem.

Using a balanced sorted structure allows:

- insertion in `O(log n)`
- nearest value lookup in `O(log n)`

Thus the entire algorithm runs in `O(n log n)` time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every valid pair explicitly |
| Optimal | O(n log n) | O(n) | Maintains a sorted structure for nearest-value lookup |

## Algorithm Walkthrough

1. Handle the special case where `x == 0`.

If `x` is zero, then the same index can be used for both elements. The absolute difference becomes zero immediately, so we return `0`.
2. Create an empty sorted structure.

This structure will store values from indices that are already far enough away from the current index.
3. Iterate through the array from left to right.

At position `i`, the index `i - x` becomes newly eligible because:

$$i - (i - x) = x$$

Therefore, before processing `nums[i]`, we insert `nums[i - x]` into the sorted structure.
4. Use binary search to locate the insertion position of `nums[i]`.

Since the structure is sorted, binary search tells us where the current value would fit.
5. Check the closest larger value.

If the insertion position exists inside the structure, that value is the smallest value greater than or equal to `nums[i]`. Compute the difference.
6. Check the closest smaller value.

If there is an element before the insertion position, that value is the largest value smaller than `nums[i]`. Compute the difference.
7. Update the global minimum difference.

The closest valid value for `nums[i]` must be one of those two neighbors in sorted order.
8. Continue until all indices are processed.
9. Return the minimum difference found.

### Why it works

The algorithm maintains the invariant that the sorted structure contains exactly the values whose indices are at least `x` positions behind the current index.

For every current value `nums[i]`, the smallest absolute difference can only come from the numerically closest values in sorted order. Binary search efficiently finds these neighbors.

Because every valid candidate is eventually inserted into the structure before it becomes eligible, and because we always compare against the closest possible values, the algorithm guarantees the optimal minimum difference.

## Python Solution

```python
from typing import List
from bisect import bisect_left, insort

class Solution:
    def minAbsoluteDifference(self, nums: List[int], x: int) -> int:
        if x == 0:
            return 0

        sorted_values = []
        answer = float("inf")

        for i in range(x, len(nums)):
            insort(sorted_values, nums[i - x])

            position = bisect_left(sorted_values, nums[i])

            if position < len(sorted_values):
                answer = min(
                    answer,
                    abs(sorted_values[position] - nums[i])
                )

            if position > 0:
                answer = min(
                    answer,
                    abs(sorted_values[position - 1] - nums[i])
                )

        return answer
```

The implementation begins by handling the special case where `x == 0`. In that situation, the answer is immediately `0`.

The `sorted_values` list stores all numbers whose indices are sufficiently far from the current position. Python's `bisect` module provides efficient binary search operations on sorted lists.

For each index `i`, we first insert `nums[i - x]` because that value has just become eligible according to the index constraint.

We then use `bisect_left` to find where `nums[i]` would be inserted in sorted order. The nearest possible values are:

- the value at `position`
- the value immediately before `position`

We compute differences with both candidates if they exist and update the answer.

The algorithm processes each element exactly once while maintaining sorted order throughout the scan.

## Go Solution

```go
package main

import (
	"math"
	"sort"
)

func minAbsoluteDifference(nums []int, x int) int {
	if x == 0 {
		return 0
	}

	sortedValues := []int{}
	answer := math.MaxInt32

	for i := x; i < len(nums); i++ {
		valueToInsert := nums[i-x]

		insertPos := sort.SearchInts(sortedValues, valueToInsert)

		sortedValues = append(sortedValues, 0)
		copy(sortedValues[insertPos+1:], sortedValues[insertPos:])
		sortedValues[insertPos] = valueToInsert

		position := sort.SearchInts(sortedValues, nums[i])

		if position < len(sortedValues) {
			diff := abs(sortedValues[position] - nums[i])
			if diff < answer {
				answer = diff
			}
		}

		if position > 0 {
			diff := abs(sortedValues[position-1] - nums[i])
			if diff < answer {
				answer = diff
			}
		}
	}

	return answer
}

func abs(x int) int {
	if x < 0 {
		return -x
	}
	return x
}
```

The Go version follows the same algorithmic idea as the Python solution. Since Go does not provide a built in ordered multiset, we maintain a manually sorted slice.

`sort.SearchInts` performs binary search to locate insertion positions and nearest neighbors.

Insertion into the sorted slice requires shifting elements to make room for the new value. This keeps the slice ordered throughout execution.

Integer overflow is not a concern here because the maximum difference between two values is less than `10^9`, which easily fits inside Go's `int` type on standard platforms.

## Edge Cases

One important edge case occurs when `x = 0`. In this situation, the same index may be paired with itself because the index difference requirement becomes:

$$|i - i| = 0$$

The minimum possible absolute difference is therefore always `0`. Without explicitly handling this case, the algorithm might perform unnecessary processing or even miss the immediate answer.

Another important case involves duplicate values. If two equal numbers appear at indices at least `x` apart, the correct answer is `0`. Some incorrect implementations only compare nearby elements in index order instead of nearby elements in value order, which can miss this optimal case. The ordered structure guarantees that equal values are detected immediately.

A third tricky case appears when the closest value lies just before or just after the binary search insertion point. If an implementation checks only one neighbor, it may miss the true minimum difference. The algorithm correctly checks both surrounding candidates after binary search, ensuring the closest numerical value is always considered.

A final edge case is very large arrays. Since `n` can reach `10^5`, quadratic solutions become infeasible. The `O(n log n)` ordered-search approach ensures the implementation remains efficient even at maximum input size.
