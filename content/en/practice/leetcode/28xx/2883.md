# LeetCode 2883, Drop Missing Data

## Problem Understanding

This problem gives us a Pandas DataFrame named `students` with three columns:

| Column | Type |
| --- | --- |
| `student_id` | int |
| `name` | object |
| `age` | int |

Some rows contain missing values in the `name` column. In Pandas, these missing values are represented as `None` or `NaN`.

The task is to remove every row where the `name` column is missing, then return the filtered DataFrame.

In simpler terms, we only want to keep students whose names are present. Any student record with an empty or missing name should be excluded from the final result.

The input is already structured as a Pandas DataFrame, so we do not need to parse raw input manually. The expected output is another DataFrame containing only valid rows.

The scale of the input is small enough that efficiency is not a major concern, but the solution should still use idiomatic Pandas operations. Since the problem is categorized as Easy, the focus is mainly on understanding how to filter missing values correctly.

One important detail is that only the `name` column matters. Even if other columns contain valid data, the row must still be removed if `name` is missing.

There are several edge cases worth considering:

- Every row could contain a missing name, resulting in an empty DataFrame.
- No rows may contain missing values, meaning the original DataFrame should remain unchanged.
- Multiple consecutive rows could contain missing values.
- The DataFrame could contain only one row.

A naive implementation that checks for empty strings instead of actual missing values would fail, because Pandas treats `None` and `NaN` differently from regular strings.

## Approaches

### Brute Force Approach

A straightforward solution is to iterate through every row manually and build a new collection containing only rows where `name` is not missing.

We could loop through the DataFrame row by row using methods such as `iterrows()`. For each row, we would check whether the `name` field is `None` or `NaN`. If the value is valid, we append the row to a result list. Finally, we reconstruct a new DataFrame from the collected rows.

This approach works because every row is explicitly examined, and rows with missing names are skipped.

However, this solution is not ideal in Pandas. Manual iteration is slower and less idiomatic than using vectorized DataFrame operations. Pandas is specifically optimized for column-wise filtering, so iterating row by row loses those performance advantages.

### Optimal Approach

The better solution is to use Pandas built in filtering with `dropna()`.

The key observation is that Pandas already provides efficient functionality for removing rows containing missing values. By specifying the `subset=["name"]` parameter, we tell Pandas to only inspect the `name` column when deciding which rows to remove.

This solution is concise, efficient, and easy to read.

The operation:

```
students.dropna(subset=["name"])
```

returns a new DataFrame containing only rows where `name` is present.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Iterates row by row and rebuilds the DataFrame manually |
| Optimal | O(n) | O(n) | Uses Pandas vectorized filtering with `dropna()` |

## Algorithm Walkthrough

1. Start with the input DataFrame `students`.
2. Inspect the `name` column for missing values. In Pandas, missing values may appear as `None` or `NaN`.
3. Use the `dropna()` method with `subset=["name"]`. This tells Pandas:

- Look only at the `name` column.
- Remove any row where `name` is missing.
4. Store the filtered DataFrame returned by `dropna()`.
5. Return the filtered DataFrame as the final answer.

### Why it works

The correctness comes from how `dropna(subset=["name"])` operates. The method examines every row and checks whether the `name` column contains a valid value. If the value is missing, the row is excluded. Otherwise, the row is preserved. Since every row is processed exactly once, the final DataFrame contains precisely the rows with non missing names.

## Python Solution

```python
import pandas as pd

class Solution:
    def dropMissingData(self, students: pd.DataFrame) -> pd.DataFrame:
        return students.dropna(subset=["name"])
```

The implementation is intentionally simple because Pandas already provides the exact functionality needed.

The method receives the `students` DataFrame as input. The call to `dropna(subset=["name"])` removes rows where the `name` column contains missing data. Since Pandas returns a new DataFrame, the original input is not modified unless explicitly requested.

Finally, the filtered DataFrame is returned.

## Go Solution

```go
package main

import "fmt"

type Student struct {
	StudentID int
	Name      *string
	Age       int
}

func dropMissingData(students []Student) []Student {
	result := []Student{}

	for _, student := range students {
		if student.Name != nil {
			result = append(result, student)
		}
	}

	return result
}

func main() {
	name1 := "Piper"
	name2 := "Georgia"
	name3 := "Willow"

	students := []Student{
		{StudentID: 32, Name: &name1, Age: 5},
		{StudentID: 217, Name: nil, Age: 19},
		{StudentID: 779, Name: &name2, Age: 20},
		{StudentID: 849, Name: &name3, Age: 14},
	}

	result := dropMissingData(students)

	fmt.Println(result)
}
```

Go does not have a built in DataFrame structure like Pandas, so we model the data using a `Student` struct and a slice of students.

The `Name` field is defined as a pointer to a string. A `nil` pointer represents missing data, similar to `None` in Python.

The function iterates through all students and appends only those with non nil names to the result slice.

Unlike Python Pandas, Go requires explicit iteration and manual filtering.

## Edge Cases

One important edge case occurs when every row contains a missing name. A careless implementation might crash or return `None` instead of an empty DataFrame. Using `dropna()` handles this naturally by returning an empty DataFrame with the same columns preserved.

Another edge case is when no rows contain missing values. Some implementations unnecessarily copy or modify data. The Pandas solution correctly returns all rows unchanged because none satisfy the removal condition.

A third important edge case involves a DataFrame with only one row. If that row contains a missing name, the result should be empty. If the name exists, the row should remain. The implementation handles both scenarios correctly because the filtering logic applies uniformly regardless of DataFrame size.

A subtle source of bugs is distinguishing missing values from empty strings. This problem specifically concerns missing values such as `None` or `NaN`. The `dropna()` method correctly targets true missing data rather than ordinary string values.
