## Solution

## Problem Understanding

The problem is asking us to split a given array of non-negative integers into **contiguous subarrays** in such a way that the **sum of their bitwise AND scores** is minimized, while simultaneously **maximizing the number of subarrays**. Each subarray's score is calculated as the result of the bitwise AND operation applied to all elements in that subarray.

In simpler terms, we want to break the array into as many pieces as possible such that each piece's AND value is as small as possible, ideally zero. The output is the **maximum number of subarrays** that satisfy this condition.

The input `nums` can be as large as 100,000 elements and each element can go up to 1,000,000. This tells us that any solution that tries **all possible splits** will be too slow, and we need a linear or near-linear solution. Key observations include:

1. A bitwise AND of any number with zero is zero. This suggests that the **minimal subarray score is 0**, and the problem reduces to counting how many subarrays we can split to achieve a score of 0.
2. Edge cases to consider include arrays of length 1, arrays that contain only zeros, and arrays where no two numbers AND to zero.

## Approaches

The **brute-force approach** would be to try every possible split of the array, calculate the AND for each subarray, and track the sum of all ANDs. After that, we would choose the split that maximizes the number of subarrays while minimizing the sum. This is clearly infeasible because the number of possible splits is exponential in the size of the array.

The **optimal approach** leverages a greedy strategy with bit manipulation. We can iterate through the array while maintaining a running AND of the current subarray. If at any point this running AND becomes zero, we know we can end the current subarray here. This is because extending the subarray further would only maintain a non-negative AND and cannot decrease the sum further. Then, we reset the running AND and start a new subarray. This method guarantees we maximize the number of subarrays with the minimal sum.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Try all possible splits and calculate AND sums, infeasible for n up to 10^5 |
| Optimal | O(n) | O(1) | Greedily split whenever running AND reaches 0, linear scan suffices |

## Algorithm Walkthrough

1. Initialize a variable `current_and` to store the running AND for the current subarray and set it to the first element or `-1` to represent "undefined".
2. Initialize a counter `subarrays_count` to zero.
3. Iterate through the array:

1. If `current_and` is undefined, set it to the current number.
2. Otherwise, update `current_and` to `current_and AND nums[i]`.
3. If `current_and` becomes zero, increment `subarrays_count` by 1, and reset `current_and` to undefined to start a new subarray.
4. After iterating through the array, return `subarrays_count`.

**Why it works:**

The greedy strategy works because the bitwise AND operation is **monotone in the sense that adding more numbers cannot increase the value of the AND**. Therefore, the earliest point where the running AND becomes zero is the best place to split: it guarantees the subarray has the minimal possible score (0) and allows us to start maximizing the number of subsequent subarrays.

## Python Solution

```python
from typing import List

class Solution:
    def maxSubarrays(self, nums: List[int]) -> int:
        subarrays_count = 0
        current_and = None
        
        for num in nums:
            if current_and is None:
                current_and = num
            else:
                current_and &= num
            if current_and == 0:
                subarrays_count += 1
                current_and = None
                
        return subarrays_count
```

**Implementation Explanation:**

We maintain a running `current_and` for the current subarray. Each time we include a new number, we update the AND. If the AND reaches 0, we know that extending the subarray cannot decrease the score further, so we end the subarray and increment `subarrays_count`. The use of `None` for `current_and` ensures we correctly start new subarrays.

## Go Solution

```go
func maxSubarrays(nums []int) int {
    subarraysCount := 0
    var currentAnd *int
    
    for _, num := range nums {
        if currentAnd == nil {
            currentAnd = new(int)
            *currentAnd = num
        } else {
            *currentAnd &= num
        }
        if *currentAnd == 0 {
            subarraysCount++
            currentAnd = nil
        }
    }
    
    return subarraysCount
}
```

**Go-specific Notes:**

In Go, we use a pointer for `currentAnd` to represent the "undefined" state, similar to `None` in Python. The logic otherwise follows the same greedy approach.

## Edge Cases

**Edge Case 1:** Array of all zeros. Each element can be its own subarray, yielding the maximum count. Our solution correctly splits at each zero.

**Edge Case 2:** Array with no zero AND anywhere. Only one subarray can be formed, and the algorithm does not create unnecessary splits because the running AND never reaches zero.

**Edge Case 3:** Array with a mix of zeros and non-zero numbers. The algorithm must correctly reset the running AND after every zero to maximize the number of subarrays without violating the minimal sum condition. This is handled by setting `current_and` to `None` or `nil` after each split.
