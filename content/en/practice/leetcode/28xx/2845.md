## Solution

## Problem Understanding

The problem asks us to count how many contiguous subarrays satisfy a very specific modular condition.

For every subarray `nums[l..r]`, we examine each element in that range and count how many indices satisfy:

```
nums[i] % modulo == k
```

Let this count be called `cnt`.

A subarray is considered interesting if:

```
cnt % modulo == k
```

The task is to return the total number of such subarrays.

In simpler terms, we are not directly interested in the values inside the subarray. Instead, each element contributes either:

- `1` if `nums[i] % modulo == k`
- `0` otherwise

Then for every subarray, we sum these contributions and check whether the result modulo `modulo` equals `k`.

The input consists of:

- `nums`, an integer array
- `modulo`, the modulus value
- `k`, the target remainder

The output is the number of interesting subarrays.

The constraints are important:

```
1 <= nums.length <= 10^5
```

This immediately tells us that an `O(n^2)` solution will likely be too slow because the number of subarrays can be approximately `5 * 10^9` in the worst case.

The values inside `nums` and the value of `modulo` can also be large, up to `10^9`, but this does not significantly affect complexity because modulo operations are constant time.

Several edge cases are important:

- Arrays where every element satisfies `nums[i] % modulo == k`
- Arrays where no element satisfies the condition
- Cases where `k = 0`
- Very large arrays requiring linear time
- Situations where prefix modulo values become negative if handled incorrectly

The problem guarantees:

- `k < modulo`
- The array is non-empty
- All numbers are positive

These guarantees simplify implementation details.

## Approaches

### Brute Force Approach

The most direct solution is to examine every possible subarray.

For each starting index `l`, we extend the subarray one element at a time toward the right. While extending, we maintain a running count of elements satisfying:

```
nums[i] % modulo == k
```

At each ending index `r`, we check whether:

```
cnt % modulo == k
```

If true, we increment the answer.

This approach is correct because it explicitly evaluates every subarray exactly as defined in the problem statement.

However, the complexity is too large. There are `O(n^2)` subarrays, and although we can update the count incrementally, we still must process all subarrays. With `n = 10^5`, this becomes infeasible.

### Optimal Approach

The key insight is that we only care about how many qualifying elements appear inside a subarray.

We transform the problem into a prefix sum problem.

Define:

```
good[i] = 1 if nums[i] % modulo == k
           0 otherwise
```

Now the problem becomes:

Find subarrays where:

```
(sum of good values) % modulo == k
```

Let:

```sql
prefix[i] = number of qualifying elements from index 0 to i
```

For a subarray `(l..r)`:

```
count = prefix[r] - prefix[l-1]
```

We need:

```
(prefix[r] - prefix[l-1]) % modulo == k
```

Rearranging:

```
prefix[l-1] % modulo == (prefix[r] - k) % modulo
```

This means that for every current prefix remainder, we need to know how many earlier prefix remainders equal the required target value.

A hash map allows us to count these efficiently in constant time.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every subarray directly |
| Optimal | O(n) | O(n) | Uses prefix sums with modular arithmetic and a hash map |

## Algorithm Walkthrough

1. Initialize a hash map called `frequency`.

This map stores how many times each prefix remainder has appeared so far.

We start with:

```
frequency[0] = 1
```

This represents the empty prefix before processing any elements.
2. Initialize:

- `prefix_count = 0`
- `answer = 0`

The variable `prefix_count` stores how many qualifying elements have appeared so far.
3. Iterate through each number in `nums`.
4. For the current number, check:

```
nums[i] % modulo == k
```

If true, increment `prefix_count`.
5. Compute the current prefix remainder:

```
current = prefix_count % modulo
```
6. Compute the remainder we need from a previous prefix:

```
target = (current - k + modulo) % modulo
```

We add `modulo` before taking `% modulo` to avoid negative values.
7. Any previous prefix with remainder `target` forms an interesting subarray ending at the current index.

Therefore:

```
answer += frequency[target]
```
8. Record the current remainder in the hash map:

```
frequency[current] += 1
```
9. Continue until all elements are processed.
10. Return `answer`.

### Why it works

The algorithm relies on the prefix sum identity:

```
subarray_count = prefix[r] - prefix[l-1]
```

We need:

```
(prefix[r] - prefix[l-1]) % modulo == k
```

Rearranging gives:

```
prefix[l-1] % modulo == (prefix[r] - k) % modulo
```

Therefore, for every current prefix remainder, the number of valid subarrays ending at the current position equals the number of earlier prefixes with the required target remainder.

The hash map stores exactly these counts, allowing us to compute the answer in linear time.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def countInterestingSubarrays(self, nums: List[int], modulo: int, k: int) -> int:
        frequency = defaultdict(int)
        frequency[0] = 1

        prefix_count = 0
        answer = 0

        for num in nums:
            if num % modulo == k:
                prefix_count += 1

            current_remainder = prefix_count % modulo
            target_remainder = (current_remainder - k + modulo) % modulo

            answer += frequency[target_remainder]

            frequency[current_remainder] += 1

        return answer
```

The implementation follows the prefix sum approach directly.

The `frequency` hash map stores how many times each prefix remainder has appeared. We initialize `frequency[0] = 1` because an empty prefix exists before processing the array.

As we iterate through the array, we determine whether the current element contributes to the qualifying count. If it does, we increment `prefix_count`.

The current prefix remainder is computed using modulo arithmetic. We then calculate the target remainder that earlier prefixes must have in order to form an interesting subarray.

The hash map immediately tells us how many such prefixes exist, so we add that count to the answer.

Finally, we record the current remainder so future positions can use it.

## Go Solution

```go
func countInterestingSubarrays(nums []int, modulo int, k int) int64 {
	frequency := make(map[int]int64)
	frequency[0] = 1

	prefixCount := 0
	var answer int64 = 0

	for _, num := range nums {
		if num%modulo == k {
			prefixCount++
		}

		currentRemainder := prefixCount % modulo
		targetRemainder := (currentRemainder - k + modulo) % modulo

		answer += frequency[targetRemainder]

		frequency[currentRemainder]++
	}

	return answer
}
```

The Go implementation mirrors the Python solution closely.

One important difference is that the answer uses `int64` because the number of subarrays can exceed the range of a standard 32-bit integer.

The hash map is declared as:

```go
map[int]int64
```

because it stores counts of prefix remainders.

Go does not provide a built-in defaultdict equivalent, but accessing a missing map key automatically returns the zero value, which works perfectly for frequency counting.

## Edge Cases

One important edge case occurs when `k = 0`. In this situation, we are looking for subarrays where the number of qualifying elements is divisible by `modulo`. This often creates many repeated prefix remainders. Incorrect handling of modulo arithmetic can easily produce negative values during target computation. The implementation avoids this problem using:

```
(current - k + modulo) % modulo
```

which guarantees a non-negative remainder.

Another important case occurs when no elements satisfy:

```
nums[i] % modulo == k
```

In this situation, the prefix count never increases. The algorithm must still correctly count subarrays when `k = 0`, because zero qualifying elements may itself satisfy the condition. The frequency map initialization with `frequency[0] = 1` ensures these subarrays are counted properly.

A third edge case occurs when every element qualifies. In that scenario, the prefix count increases at every step, and many valid subarrays can exist. The total number of subarrays may become very large, especially for arrays near the maximum constraint size. The Go solution uses `int64` to avoid overflow, and Python naturally handles arbitrarily large integers.

Another subtle edge case involves repeated prefix remainders. Multiple earlier prefixes may produce the same remainder, and every one of them can form a valid subarray with the current position. A naive implementation might only track whether a remainder exists, but this problem requires tracking the frequency of each remainder. The hash map correctly accumulates counts for all occurrences.
