## Solution

## Problem Understanding

The problem asks us to find the **maximum value of a triplet of indices** `(i, j, k)` in a 0-indexed array `nums` such that `i < j < k`. The value of a triplet is defined as `(nums[i] - nums[j]) * nums[k]`. If all possible triplets have negative values, we return `0`.

In simpler terms, you are looking for three elements in the array where the first comes before the second, and the second comes before the third. You subtract the second element from the first and multiply the result by the third element. The goal is to maximize this result.

The constraints indicate that `nums` has at least 3 elements and at most 100, and all values are positive integers. Since the array is relatively small, a brute-force solution is feasible but may not be elegant. Edge cases include arrays where all triplet results are negative or arrays where elements are strictly increasing or decreasing.

Key considerations:

- The subtraction `(nums[i] - nums[j])` can produce negative values, so careful choice of `i` and `j` is essential.
- Multiplying by `nums[k]` can amplify the result, so choosing the maximum possible `nums[k]` given `i` and `j` is beneficial.
- The problem guarantees at least 3 elements, so we do not need to handle smaller arrays.

## Approaches

**Brute Force:**

The naive approach is to iterate through all possible triplets `(i, j, k)` using three nested loops. For each triplet, compute `(nums[i] - nums[j]) * nums[k]` and track the maximum. This guarantees correctness because it checks all possibilities. However, its time complexity is O(n³), which is acceptable for `n ≤ 100` but not ideal in general.

**Optimal Approach:**

The key insight is that for a fixed `j` (middle element), the best `i` is the **maximum element before `j`** to maximize `nums[i] - nums[j]`. Similarly, the best `k` is the **maximum element after `j`** to maximize the product. By precomputing the prefix maximum for `i` candidates and the suffix maximum for `k` candidates, we can evaluate the maximum triplet value in a single pass over `j`. This reduces the complexity from O(n³) to O(n).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Checks all triplets exhaustively |
| Optimal | O(n) | O(n) | Uses prefix and suffix maximums to reduce redundant computations |

## Algorithm Walkthrough

1. Compute a prefix array `prefix_max` where `prefix_max[j]` stores the maximum value of `nums[i]` for all `i < j`. This allows us to quickly find the largest `nums[i]` before any `j`.
2. Compute a suffix array `suffix_max` where `suffix_max[j]` stores the maximum value of `nums[k]` for all `k > j`. This allows us to quickly find the largest `nums[k]` after any `j`.
3. Initialize a variable `max_value` to `0` to track the maximum triplet value found.
4. Iterate over `j` from `1` to `n-2` (since `i < j < k`). For each `j`, compute the candidate triplet value `(prefix_max[j] - nums[j]) * suffix_max[j]`.
5. If the computed value is greater than `max_value`, update `max_value`.
6. After the loop, return `max_value`.

**Why it works:**

By precomputing the best possible `i` and `k` for each `j`, we ensure that every triplet we evaluate is the optimal one with that middle index. The loop over `j` ensures all valid middle indices are considered, and the prefix/suffix arrays guarantee we always select the optimal `i` and `k`.

## Python Solution

```python
from typing import List

class Solution:
    def maximumTripletValue(self, nums: List[int]) -> int:
        n = len(nums)
        if n < 3:
            return 0
        
        prefix_max = [0] * n
        prefix_max[0] = nums[0]
        for i in range(1, n):
            prefix_max[i] = max(prefix_max[i-1], nums[i])
        
        suffix_max = [0] * n
        suffix_max[-1] = nums[-1]
        for i in range(n-2, -1, -1):
            suffix_max[i] = max(suffix_max[i+1], nums[i])
        
        max_value = 0
        for j in range(1, n-1):
            current_value = (prefix_max[j-1] - nums[j]) * suffix_max[j+1]
            max_value = max(max_value, current_value)
        
        return max_value
```

The Python solution constructs `prefix_max` and `suffix_max` arrays to store the best candidates for `i` and `k` around each `j`. We iterate over all valid `j` indices, compute the candidate triplet value, and track the maximum. This ensures O(n) time with clear logic.

## Go Solution

```go
func maximumTripletValue(nums []int) int64 {
    n := len(nums)
    if n < 3 {
        return 0
    }

    prefixMax := make([]int, n)
    prefixMax[0] = nums[0]
    for i := 1; i < n; i++ {
        if nums[i] > prefixMax[i-1] {
            prefixMax[i] = nums[i]
        } else {
            prefixMax[i] = prefixMax[i-1]
        }
    }

    suffixMax := make([]int, n)
    suffixMax[n-1] = nums[n-1]
    for i := n-2; i >= 0; i-- {
        if nums[i] > suffixMax[i+1] {
            suffixMax[i] = nums[i]
        } else {
            suffixMax[i] = suffixMax[i+1]
        }
    }

    var maxValue int64 = 0
    for j := 1; j < n-1; j++ {
        val := int64(prefixMax[j-1]-nums[j]) * int64(suffixMax[j+1])
        if val > maxValue {
            maxValue = val
        }
    }

    return maxValue
}
```

In Go, the main differences are explicit slice initialization and converting integers to `int64` when performing the multiplication to prevent overflow. The logic mirrors the Python solution.

## Worked Examples

**Example 1:** `nums = [12,6,1,2,7]`

`prefix_max = [12,12,12,12,12]`

`suffix_max = [7,7,7,7,7]`

Evaluate `j = 1` → `(12-6)*7 = 42`

Evaluate `j = 2` → `(12-1)*7 = 77` (maximum)

Evaluate `j = 3` → `(12-2)*7 = 70`

Return `77`.

**Example 2:** `nums = [1,10,3,4,19]`

`prefix_max = [1,10,10,10,19]`

`suffix_max = [19,19,19,19,19]`

Evaluate `j = 1` → `(1-10)*19 = -171`

Evaluate `j = 2` → `(10-3)*19 = 133` (maximum)

Evaluate `j = 3` → `(10-4)*19 = 114`

Return `133`.

**Example 3:** `nums = [1,2,3]`

Evaluate `j = 1` → `(1-2)*3 = -3`

Return `0` because all values are negative.
