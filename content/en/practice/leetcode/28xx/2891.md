## Solution

## Problem Understanding

This problem provides a Pandas DataFrame named `animals` with four columns:

| Column | Type | Meaning |
| --- | --- | --- |
| `name` | object | The animal's name |
| `species` | object | The animal's species |
| `age` | int | The animal's age |
| `weight` | int | The animal's weight in kilograms |

The task is to return a DataFrame containing only the `name` column for animals whose `weight` is strictly greater than `100`.

The output must also be sorted by `weight` in descending order, meaning the heaviest qualifying animal should appear first.

The important detail is that the solution should use method chaining and ideally be written in one line. In Pandas, method chaining means combining multiple DataFrame operations together without intermediate variables. Instead of performing filtering, sorting, and column selection separately, we connect them into a single expression.

For example, we need to:

1. Filter rows where `weight > 100`
2. Sort those rows by `weight` descending
3. Return only the `name` column

The input size is not explicitly constrained, but the operations used are standard DataFrame transformations. Pandas handles these efficiently for normal interview scale datasets.

Several edge cases are worth considering:

- If no animals weigh more than `100`, the result should be an empty DataFrame with the `name` column.
- If exactly one animal satisfies the condition, the result should contain a single row.
- Multiple animals may have the same weight, which is acceptable because the problem only specifies descending order, not tie breaking.
- Animals with weight exactly equal to `100` must not be included because the comparison is strictly greater than `100`.

## Approaches

### Brute Force Approach

A brute force solution would manually iterate through every row in the DataFrame, check whether the weight exceeds `100`, store qualifying rows in a temporary structure, sort the collected rows afterward, and finally construct a new DataFrame containing only the `name` column.

This approach is correct because it explicitly evaluates every animal and applies the required filtering and sorting conditions. However, it is unnecessarily verbose and does not take advantage of Pandas' optimized vectorized operations.

In addition, manual iteration over DataFrame rows is slower and less idiomatic in Pandas. Using loops such as `iterrows()` introduces overhead and reduces readability.

### Optimal Approach

The optimal solution uses Pandas vectorized operations and method chaining.

The key observation is that Pandas already provides efficient built in methods for filtering, sorting, and column selection. These operations can be chained together directly:

- Boolean indexing filters rows
- `sort_values()` orders the rows
- Double brackets select the required column as a DataFrame

This produces concise, readable, and efficient code in a single expression.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(n) | Manual iteration and sorting |
| Optimal | O(n log n) | O(n) | Uses efficient Pandas vectorized operations |

## Algorithm Walkthrough

1. Start with the `animals` DataFrame.
2. Apply a filter to keep only rows where the `weight` column is greater than `100`.

This uses boolean indexing:

```
animals[animals["weight"] > 100]
```

The comparison produces a boolean mask where each row is either `True` or `False`.
3. Sort the filtered rows by the `weight` column in descending order.

This uses:

```
.sort_values("weight", ascending=False)
```

Setting `ascending=False` ensures heavier animals appear first.
4. Select only the `name` column.

Using double brackets keeps the result as a DataFrame instead of converting it into a Series:

```
[["name"]]
```
5. Return the final DataFrame.

### Why it works

The algorithm works because each transformation preserves exactly the rows and ordering required by the problem statement.

The filtering step guarantees that every remaining animal has `weight > 100`. The sorting step guarantees descending order by weight. The final column selection guarantees that only the `name` column appears in the result.

Since these operations are applied sequentially and deterministically, the final DataFrame always satisfies the problem requirements.

## Edge Cases

One important edge case occurs when no animals satisfy the condition `weight > 100`. A naive implementation might accidentally return `None`, raise an error, or return incorrectly shaped data. This implementation correctly returns an empty DataFrame containing the `name` column because Pandas filtering naturally preserves the schema even when zero rows remain.

Another important edge case involves animals whose weight is exactly `100`. The problem explicitly requires weights to be strictly greater than `100`. A careless implementation using `>= 100` would incorrectly include these animals. The solution avoids this bug by using the exact comparison `> 100`.

A third edge case occurs when multiple animals have identical weights. Some implementations may unintentionally rely on unique sorting keys or produce unstable behavior. Pandas sorting handles duplicate weights correctly, and all qualifying rows remain present in the result.

A final edge case involves already sorted input. Some algorithms unnecessarily perform extra work or accidentally reverse correct ordering. Since this solution always applies an explicit descending sort, it produces the correct ordering regardless of the original input arrangement.
