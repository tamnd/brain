## Solution

## Problem Understanding

The problem asks us to "invert" an object or array. In practical terms, this means swapping keys and values.

If the input object contains:

```
{
  "a": "1",
  "b": "2"
}
```

then the output should become:

```
{
  "1": "a",
  "2": "b"
}
```

The important detail is that the original values become keys in the new object, and the original keys become values.

Arrays are treated the same way because, in JavaScript, arrays are also objects whose keys are indices represented as strings. For example:

```
["1", "2", "3"]
```

behaves like:

```
{
  "0": "1",
  "1": "2",
  "2": "3"
}
```

So the inverted result becomes:

```
{
  "1": "0",
  "2": "1",
  "3": "2"
}
```

The tricky part is handling duplicate values.

Suppose the input is:

```
{
  "a": "1",
  "b": "2",
  "c": "2"
}
```

Both `"b"` and `"c"` map to `"2"`. After inversion, `"2"` cannot map to only one key, otherwise information would be lost. Instead, the inverted object should store all matching keys inside an array:

```
{
  "1": "a",
  "2": ["b", "c"]
}
```

The constraints tell us that:

- Every value is guaranteed to be a string
- The input is a valid JSON object or array
- The serialized size can be as large as `10^5`

This means we need an efficient linear solution. Nested scans or repeated searching would become unnecessarily expensive for large inputs.

Several edge cases are important:

- Multiple keys sharing the same value
- Arrays instead of plain objects
- Inputs where every value is unique
- Inputs where every value is identical
- Single duplicate transitions, where a value first maps to a string and later must become an array

A naive implementation often fails when converting from a single string value into an array during duplicate handling.

## Approaches

### Brute Force Approach

A brute force approach would process each key-value pair and repeatedly search through the partially constructed inverted object to determine whether the value has already been used.

One possible implementation could maintain a list of processed entries and scan it every time a new element is added. If a matching inverted key is found, we either append to an array or convert a single string into an array.

This approach is correct because every original pair is eventually examined and inserted into the inverted structure. However, repeatedly scanning previously processed entries leads to unnecessary repeated work.

If there are `n` entries, and each insertion may scan up to `n` existing entries, the overall complexity becomes quadratic.

For the given constraints, `O(n^2)` is too slow.

### Optimal Hash Map Approach

The key observation is that inversion naturally fits a hash map.

We can iterate through the input exactly once:

- The original value becomes the new key
- The original key becomes the new value

A hash map allows constant time lookup to determine whether a value has already appeared.

When processing each pair:

- If the value has not appeared before, store the key directly
- If the value already exists:

- Convert the stored string into an array if needed
- Append the new key

This avoids repeated scanning and guarantees linear time complexity.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly scans existing entries |
| Optimal | O(n) | O(n) | Uses hash map lookups for constant time insertion |

## Algorithm Walkthrough

1. Create an empty hash map called `inverted`.

This structure will store the inverted relationships. The keys of this map are the original values, and the values are either:

- a single original key as a string
- an array of original keys if duplicates exist
2. Iterate through every key in the input object.

For arrays, the indices are automatically treated as string keys during iteration.
3. Read the current value associated with the key.

Suppose we are processing:

```
key = "b"
value = "2"
```
4. Check whether `value` already exists in `inverted`.

This determines whether we are seeing this value for the first time or handling a duplicate.
5. If the value does not exist yet, store the key directly.

Example:

```
inverted["2"] = "b"
```

We store a single string instead of an array initially because the problem expects arrays only when duplicates occur.
6. If the value already exists, handle duplicates carefully.

There are two possibilities:

- The existing entry is a string
- The existing entry is already an array
7. If the existing entry is a string, convert it into an array.

Example:

```
inverted["2"] = ["b"]
```

This preserves the original key while preparing for multiple keys.
8. Append the current key to the array.

Example:

```
inverted["2"].append("c")
```
9. Continue until all entries are processed.
10. Return the completed inverted object.

### Why it works

The algorithm maintains the invariant that after processing each key-value pair, the `inverted` map correctly represents all processed relationships.

For every original pair `(key, value)`:

- `value` becomes a key in the result
- All original keys associated with that value are preserved

Because each input entry is processed exactly once and duplicates are accumulated correctly, the final structure is the correct inversion of the original object.

## Python Solution

```python
from typing import Any, Dict, List, Union

class Solution:
    def invertObject(self, obj: Any) -> Dict[str, Union[str, List[str]]]:
        inverted: Dict[str, Union[str, List[str]]] = {}

        for key, value in enumerate(obj) if isinstance(obj, list) else obj.items():
            key = str(key)

            if value not in inverted:
                inverted[value] = key
            else:
                if isinstance(inverted[value], str):
                    inverted[value] = [inverted[value]]

                inverted[value].append(key)

        return inverted
```

The implementation starts by creating an empty dictionary called `inverted`.

The iteration logic supports both arrays and objects:

- For arrays, `enumerate(obj)` produces index-value pairs
- For objects, `obj.items()` produces key-value pairs

Since array indices must become string keys in the inverted object, we explicitly convert keys to strings.

The duplicate handling logic is the most important section:

- The first occurrence stores a plain string
- The second occurrence converts that string into a list
- Later duplicates are appended directly

This exactly matches the output format required by the problem.

## Go Solution

```go
package main

func invertObject(obj map[string]string) map[string]interface{} {
	inverted := make(map[string]interface{})

	for key, value := range obj {
		existing, found := inverted[value]

		if !found {
			inverted[value] = key
		} else {
			switch v := existing.(type) {
			case string:
				inverted[value] = []string{v, key}
			case []string:
				inverted[value] = append(v, key)
			}
		}
	}

	return inverted
}
```

The Go implementation differs slightly because Go is statically typed.

The output map uses:

```go
map[string]interface{}
```

because each value may be either:

- a `string`
- a `[]string`

Type assertions and a type switch are used to distinguish these cases safely.

Unlike Python, Go does not directly support arrays and objects with identical iteration behavior, so this implementation assumes a map input. If array support is needed, indices would need to be converted into string keys manually before processing.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each entry is processed exactly once |
| Space | O(n) | The inverted object may store all entries |

The algorithm performs a single pass through the input. Each lookup and insertion into the hash map takes average constant time. Therefore, total runtime grows linearly with the number of entries.

The extra space comes from the output structure itself and any arrays created for duplicate values.

## Edge Cases

One important edge case occurs when a value appears for the second time. Initially, the inverted object stores a single string value. When a duplicate appears, the implementation must correctly convert that string into an array before appending the new key. Forgetting this conversion often causes runtime errors or incorrect output structures. The implementation handles this explicitly using an `isinstance(..., str)` check.

Another important case is array input. Arrays in JavaScript behave like objects with numeric indices as keys. A careless implementation might treat arrays differently or forget to convert indices into strings. The solution correctly uses `enumerate` and converts indices with `str(key)`.

A third important edge case is when every value is identical. In this situation, the output contains only one key whose value is a large array of original keys. Some incorrect implementations overwrite earlier entries instead of accumulating them. This solution preserves all matching keys by appending duplicates into a list.
