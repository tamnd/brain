## Solution

## Problem Understanding

The problem is asking us to implement a function `deepFilter` that recursively filters an input object or array based on a predicate function `fn`. The input `obj` can be either a JSON object (dictionary) or a JSON array (list). The function `fn` takes a value and returns `true` if the value should be kept, or `false` if it should be removed. After filtering, any empty objects or arrays resulting from removed elements should themselves be removed. If the entire structure becomes empty after filtering, the function should return `undefined`.

In essence, the problem is a combination of two operations: recursive traversal of nested structures and conditional pruning of values. The constraints guarantee that the input is a valid JSON structure and that the stringified length of the input is at most 105, which is reasonable for a recursive solution. Key edge cases include nested empty arrays or objects, deeply nested structures, and primitive values that do not meet the predicate.

## Approaches

The brute-force approach is to iterate over each element or key in the object or array, apply the predicate `fn`, and recursively filter nested arrays or objects. After filtering, we would manually check if the resulting object or array is empty and replace it with `undefined`. This approach works correctly but is verbose and can be inefficient if we repeatedly create new objects for every recursive call.

The optimal approach is to use a clean recursive depth-first traversal that simultaneously filters elements and prunes empty structures. The key insight is that we do not need to separately check for emptiness at the top level; we can do it naturally by returning `undefined` from recursive calls when a substructure becomes empty. This ensures that empty arrays or objects automatically propagate upward and get removed without extra post-processing.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Recursively traverses all elements, creates new objects/arrays, checks for emptiness separately |
| Optimal | O(n) | O(n) | Recursively filters while simultaneously pruning empty structures |

## Algorithm Walkthrough

1. Define a helper function `deepFilterHelper` that takes an input value (object, array, or primitive) and returns the filtered version or `undefined` if the structure is empty.
2. If the input is an array, initialize a new array to hold filtered elements. Iterate over each element, recursively apply the helper, and add the result to the new array if it is not `undefined`. After iteration, if the new array is empty, return `undefined`; otherwise, return the array.
3. If the input is an object, initialize a new object to hold filtered key-value pairs. Iterate over each key, recursively apply the helper to its value, and add the key-value pair to the new object if the result is not `undefined`. After iteration, if the object has no keys, return `undefined`; otherwise, return the object.
4. If the input is a primitive value, apply the predicate `fn`. Return the value itself if `fn` returns `true`, otherwise return `undefined`.
5. Call `deepFilterHelper` on the top-level input and return the result.

**Why it works**: At every recursive level, the function ensures that empty substructures are replaced with `undefined`. This invariant guarantees that the final returned object or array contains only values satisfying `fn` and no empty nested structures.

## Python Solution

```python
from typing import Any, Callable, Union

class Solution:
    def deepFilter(self, obj: Any, fn: Callable[[Any], bool]) -> Any:
        def deepFilterHelper(value: Any) -> Any:
            if isinstance(value, list):
                filtered_list = []
                for item in value:
                    filtered_item = deepFilterHelper(item)
                    if filtered_item is not None:
                        filtered_list.append(filtered_item)
                return filtered_list if filtered_list else None
            elif isinstance(value, dict):
                filtered_dict = {}
                for k, v in value.items():
                    filtered_value = deepFilterHelper(v)
                    if filtered_value is not None:
                        filtered_dict[k] = filtered_value
                return filtered_dict if filtered_dict else None
            else:
                return value if fn(value) else None

        result = deepFilterHelper(obj)
        return result if result is not None else None
```

The implementation defines a recursive helper function that handles arrays, objects, and primitive values differently. Arrays and objects are recursively filtered, while primitive values are directly checked against the predicate. The helper returns `None` when a structure or value is removed, which naturally prunes empty objects and arrays. Finally, `None` is mapped to `undefined` in JSON/JavaScript terms, but in Python, it is `None`.

## Go Solution

```go
package main

func deepFilter(obj any, fn func(any) bool) any {
    var deepFilterHelper func(any) any
    deepFilterHelper = func(value any) any {
        switch v := value.(type) {
        case []any:
            var filtered []any
            for _, item := range v {
                filteredItem := deepFilterHelper(item)
                if filteredItem != nil {
                    filtered = append(filtered, filteredItem)
                }
            }
            if len(filtered) == 0 {
                return nil
            }
            return filtered
        case map[string]any:
            filtered := make(map[string]any)
            for key, val := range v {
                filteredVal := deepFilterHelper(val)
                if filteredVal != nil {
                    filtered[key] = filteredVal
                }
            }
            if len(filtered) == 0 {
                return nil
            }
            return filtered
        default:
            if fn(v) {
                return v
            }
            return nil
        }
    }

    result := deepFilterHelper(obj)
    return result
}
```

In Go, we use type assertions for `[]any` and `map[string]any` to differentiate between arrays, objects, and primitives. The handling of `nil` in Go corresponds to `undefined` in JavaScript or `None` in Python. Recursive filtering and empty pruning follow the same logic as the Python solution.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We visit each element exactly once in the recursive traversal. |
| Space | O(n) | Recursion stack can go as deep as the nesting level, plus new objects/arrays are created. |

The complexity is linear with respect to the total number of elements in the input JSON structure, since each element is visited once and no extra repeated work is done.
