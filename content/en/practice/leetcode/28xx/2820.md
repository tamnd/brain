## Solution

## Problem Understanding

The problem is asking us to calculate the winner of an election given a voting table where each voter may vote for one or more candidates, or abstain entirely. Each voter has exactly **one vote**, and if they vote for multiple candidates, their vote is equally split among those candidates. For example, a voter who votes for 2 candidates gives `0.5` vote to each. A voter who votes for `null` does not contribute any vote.

The input table `Votes` contains two columns: `voter` and `candidate`. Each row represents one candidate that a voter voted for, and `(voter, candidate)` is unique. The expected output is a table containing the candidate or candidates who received the **maximum total votes**, sorted in ascending order of candidate name.

Key points to note include handling voters with multiple candidates correctly, accounting for `null` votes as zero contribution, and ensuring the final output is sorted alphabetically in case of a tie. Edge cases that can trip a naive solution include voters with no votes, voters with only one candidate, and multiple candidates tying for the top votes.

## Approaches

A brute-force approach would involve iterating over each voter, counting how many candidates they voted for, calculating fractional votes for each candidate, summing these totals, and then identifying the candidate(s) with the maximum sum. While this is correct, it is somewhat cumbersome to implement directly in SQL because we need to aggregate votes carefully and handle `null`s.

The optimal approach leverages SQL’s grouping and aggregation capabilities. First, we calculate the number of candidates each voter voted for using `COUNT` and `GROUP BY` while ignoring `null`s. Then, for each vote, we compute the fractional contribution as `1.0 / number_of_candidates`. Finally, we aggregate the fractional votes per candidate, identify the maximum, and return candidates whose total votes match this maximum. Sorting ensures the output order is correct.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Iterates each vote and sums fractional contributions manually, then finds maximum. |
| Optimal | O(n) | O(n) | Uses SQL aggregation functions to compute fractional votes, sum per candidate, and filter maximum efficiently. |

## Algorithm Walkthrough

1. **Filter Null Votes:** We ignore `null` candidates as they do not contribute to the vote total. This ensures only actual votes are counted.
2. **Count Voter Choices:** For each voter, calculate how many candidates they voted for. This gives us the denominator for fractional votes.
3. **Compute Fractional Vote:** For each row in the table, compute `1.0 / number_of_candidates` as the contribution to that candidate.
4. **Aggregate Votes by Candidate:** Sum all fractional votes per candidate to get the total votes for each candidate.
5. **Identify Maximum Votes:** Determine the maximum total vote value across all candidates.
6. **Select Winners:** Return all candidates whose total votes equal the maximum, ordering them alphabetically.

Why it works: Each voter's single vote is correctly distributed across all their chosen candidates. Aggregating fractional votes ensures totals are correctly summed. Filtering for candidates with the maximum vote guarantees we get all winners, and sorting ensures the output meets the required order.

## Python Solution

```python
import sqlite3

def election_results(connection: sqlite3.Connection):
    cursor = connection.cursor()
    cursor.execute("""
        WITH voter_counts AS (
            SELECT voter, COUNT(candidate) AS num_votes
            FROM Votes
            WHERE candidate IS NOT NULL
            GROUP BY voter
        ),
        candidate_votes AS (
            SELECT v.candidate,
                   SUM(1.0 / vc.num_votes) AS total_votes
            FROM Votes v
            JOIN voter_counts vc
            ON v.voter = vc.voter
            WHERE v.candidate IS NOT NULL
            GROUP BY v.candidate
        ),
        max_votes AS (
            SELECT MAX(total_votes) AS max_vote
            FROM candidate_votes
        )
        SELECT c.candidate
        FROM candidate_votes c
        JOIN max_votes m
        ON c.total_votes = m.max_vote
        ORDER BY c.candidate ASC
    """)
    return cursor.fetchall()
```

This implementation first creates a CTE `voter_counts` to compute how many candidates each voter voted for. The next CTE, `candidate_votes`, calculates each candidate's total fractional votes. Then `max_votes` identifies the maximum vote value. Finally, we select candidates whose vote totals match this maximum and sort them alphabetically.

            FROM Votes v

            ON v.voter = vc.voter

            GROUP BY v.candidate
        ),
        max_votes AS (
            SELECT MAX(total_votes) AS max_vote
            FROM candidate_votes
        )
        SELECT c.candidate
        FROM candidate_votes c
        JOIN max_votes m

        ORDER BY c.candidate ASC

The Go implementation mirrors the Python logic. The main difference is handling database connections explicitly. The SQL query remains identical. The `Query` method returns a `*sql.Rows` result that can be iterated to extract candidate names.

## Edge Cases

One edge case is when all voters abstain by having `null` candidates. The implementation handles this by filtering `null` candidates, resulting in zero votes and an empty output.

Another edge case is a voter who votes for multiple candidates, creating fractional votes. If we do not calculate `1 / num_votes`, the totals would be incorrect. This is handled via the `voter_counts` CTE and division.

A third edge case is when multiple candidates tie with the exact maximum vote total. The solution correctly identifies all such candidates and sorts them alphabetically to meet output requirements. This ensures correctness even in cases of complex fractional vote distributions.
