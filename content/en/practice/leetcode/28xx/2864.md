## Solution

## Problem Understanding

The problem asks us to take a binary string `s` (consisting only of `'0'`s and `'1'`s) and rearrange its characters to form the **maximum odd binary number** possible. A binary number is odd if its **least significant bit (the last character)** is `'1'`. Therefore, the task is equivalent to moving exactly one `'1'` to the end of the string and arranging the remaining bits to form the largest possible number. Since binary numbers are larger when higher-order bits (leftmost) are `'1'`, the optimal strategy is to place all remaining `'1'`s at the start and the remaining `'0'`s in between.

The input guarantees at least one `'1'`, so we do not need to handle all-zero strings. The constraints are moderate (`1 <= s.length <= 100`), so a linear-time algorithm is acceptable. Edge cases include strings with a single `'1'`, strings that are already maximum, or strings with multiple `'0'`s at the start.

## Approaches

The **brute-force approach** would be to generate all permutations of the string `s`, convert them to integers, filter out even numbers, and then select the maximum. This is correct but impractical, since the number of permutations grows factorially with the length of `s` (O(n!)), which is far too slow for `n` up to 100.

The **key insight** for an optimal approach is that the **maximum binary number is obtained by placing all `'1'`s at the leftmost positions** and **all `'0'`s in between**, except for a single `'1'` at the end to make the number odd. This observation allows us to solve the problem in **O(n)** time by simply counting the number of `'1'`s and `'0'`s and reconstructing the string accordingly.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n!) | O(n!) | Generate all permutations, filter odd numbers, pick max |
| Optimal | O(n) | O(n) | Count '1's and '0's, reconstruct maximum odd number |

## Algorithm Walkthrough

1. **Count the digits**: Iterate through the input string `s` and count the number of `'1'`s and `'0'`s. Let `ones` be the count of `'1'`s and `zeros` be the count of `'0'`s.
2. **Reserve the last '1'**: To ensure the result is odd, one `'1'` must be placed at the end. Reduce `ones` by 1, since we will use this last `'1'` separately.
3. **Construct the maximum number**: Place all remaining `'1'`s at the beginning of the string, followed by all `'0'`s, and finally append the reserved last `'1'` at the end.
4. **Return the string**: The constructed string now represents the maximum odd binary number.

**Why it works:** The maximum value of a binary number is achieved when the highest-order bits are `'1'`. Since the last bit must be `'1'` for the number to be odd, we reserve one `'1'` at the end and put all remaining `'1'`s as leftmost bits. `'0'`s naturally occupy the middle positions. This guarantees the largest possible odd binary number.

## Python Solution

```python
class Solution:
    def maximumOddBinaryNumber(self, s: str) -> str:
        ones = s.count('1')
        zeros = s.count('0')
        
        # Reserve one '1' for the last position
        ones -= 1
        
        # Construct the maximum odd binary number
        return '1' * ones + '0' * zeros + '1'
```

The Python implementation counts `'1'`s and `'0'`s, reserves one `'1'` for the end, and then constructs the string in order. The use of string multiplication ensures that all bits are placed correctly, and the final result is returned.

## Go Solution

```go
func maximumOddBinaryNumber(s string) string {
    ones, zeros := 0, 0
    for _, ch := range s {
        if ch == '1' {
            ones++
        } else {
            zeros++
        }
    }

    // Reserve one '1' for the last position
    ones--

    result := make([]byte, 0, len(s))
    for i := 0; i < ones; i++ {
        result = append(result, '1')
    }
    for i := 0; i < zeros; i++ {
        result = append(result, '0')
    }
    result = append(result, '1')

    return string(result)
}
```

In Go, the implementation uses a slice of bytes for string construction, which is efficient. The logic mirrors Python: count digits, reserve one `'1'` at the end, append remaining `'1'`s, then `'0'`s, and finally the reserved `'1'`.

## Edge Cases

**Edge Case 1: Single '1'**

If the string contains only one `'1'`, it must go at the end. The algorithm handles this by reserving the `'1'` and placing it at the last position. No other rearrangement is needed.

**Edge Case 2: Leading zeros**

Strings that start with multiple zeros must maintain their count in the middle. The algorithm counts zeros separately and places them between the remaining `'1'`s and the last `'1'`.

**Edge Case 3: All ones**

For a string of only `'1'`s, one `'1'` is reserved for the end, and all remaining ones are placed at the start. This ensures the number is odd and maximized, preserving all other bits.
