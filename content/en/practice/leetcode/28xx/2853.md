## Solution

## Problem Understanding

The problem requires calculating the absolute difference between the **highest salaries** in two specific departments: **Engineering** and **Marketing**. The input is a table called `Salaries` with three columns: `emp_name`, `department`, and `salary`. Each row represents an employee's name, their department, and their salary. The combination of `emp_name` and `department` is unique, ensuring there are no duplicate entries for an employee in the same department.

The output should be a table with a single column, `salary_difference`, containing one row with the absolute difference between the maximum salary in Marketing and the maximum salary in Engineering. The problem guarantees that both departments exist and contain at least one employee, so we do not need to handle missing departments.

Important edge cases to consider include situations where all employees in one department have the same salary, or when the highest salaries are equal across departments, resulting in a difference of zero.

## Approaches

The **brute-force approach** involves iterating through the entire table twice: first to find the maximum salary in the Marketing department and then to find the maximum salary in the Engineering department. While this guarantees correctness, it requires scanning all rows multiple times, which may be inefficient for very large tables.

The **optimal approach** leverages SQL aggregation functions. We can compute the maximum salary for each department in a single pass using a `GROUP BY` clause. Once we have the two maximum values, we calculate their absolute difference using the `ABS()` function. This approach is simple, efficient, and leverages built-in SQL operations to avoid redundant scans.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Scan table twice to find max for each department separately |
| Optimal | O(n) | O(1) | Use `GROUP BY` and `MAX()` in SQL to find department maxima and compute difference |

## Algorithm Walkthrough

1. Group the `Salaries` table by the `department` column. This allows you to consider each department independently.
2. Use the `MAX()` aggregation function to find the highest salary within each department. This gives two values: one for Engineering and one for Marketing.
3. Extract the maximum salaries for Marketing and Engineering from the grouped results.
4. Calculate the absolute difference between these two values using the `ABS()` function. This ensures the result is always non-negative, regardless of which department has a higher maximum salary.
5. Return the result as a single-row table with the column `salary_difference`.

**Why it works:** SQL guarantees that aggregation functions like `MAX()` will scan all rows in a group and return the correct maximum. By isolating departments first, we ensure the maxima are department-specific. Calculating the absolute difference afterwards ensures correctness regardless of the relative size of the salaries.

## Python Solution

```python
# There is no class/method stub because this is a SQL problem, but if executing in Python via a database, the query would be:

import sqlite3

def highest_salary_difference(connection: sqlite3.Connection) -> int:
    query = """
    WITH dept_max AS (
        SELECT department, MAX(salary) AS max_salary
        FROM Salaries
        WHERE department IN ('Engineering', 'Marketing')
        GROUP BY department
    )
    SELECT ABS(
        (SELECT max_salary FROM dept_max WHERE department = 'Engineering') -
        (SELECT max_salary FROM dept_max WHERE department = 'Marketing')
    ) AS salary_difference;
    """
    cursor = connection.cursor()
    cursor.execute(query)
    result = cursor.fetchone()
    return result[0]
```

**Explanation:** The `WITH` clause creates a temporary table `dept_max` that contains the maximum salary for Engineering and Marketing. Then, two subqueries extract these maxima, and `ABS()` computes their absolute difference. The query executes in a single pass over the table, leveraging SQL efficiency.

## Go Solution

```go
package main

import (
    "database/sql"
    _ "github.com/mattn/go-sqlite3"
    "log"
)

func HighestSalaryDifference(db *sql.DB) int {
    query := `
    WITH dept_max AS (
        SELECT department, MAX(salary) AS max_salary
        FROM Salaries
        WHERE department IN ('Engineering', 'Marketing')
        GROUP BY department
    )
    SELECT ABS(
        (SELECT max_salary FROM dept_max WHERE department = 'Engineering') -
        (SELECT max_salary FROM dept_max WHERE department = 'Marketing')
    ) AS salary_difference;
    `
    var diff int
    err := db.QueryRow(query).Scan(&diff)
    if err != nil {
        log.Fatal(err)
    }
    return diff
}
```

**Go-specific notes:** In Go, `QueryRow` is used to fetch a single row result. The database driver handles conversion to `int`. Errors must be explicitly checked, which differs from Python where exceptions may propagate automatically.

## Edge Cases

1. **Equal Maximum Salaries:** If both Engineering and Marketing have the same maximum salary, the absolute difference must be zero. The use of `ABS()` in SQL ensures that this case is handled automatically.
2. **Single Employee Departments:** If a department only has one employee, that salary is trivially the maximum. The algorithm correctly identifies it without requiring special handling.
3. **Large Salary Values:** If salaries are extremely large, the algorithm must still compute the difference accurately. SQL `INT` or equivalent types typically handle large values, and `ABS()` ensures correctness even if the difference is negative before applying absolute value.

This approach guarantees correctness and efficiency across all guaranteed input conditions.
