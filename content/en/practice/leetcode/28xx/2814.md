## Solution

## Problem Understanding

The problem asks us to determine the minimum time it takes for a person to move from a starting cell `"S"` to a destination cell `"D"` in a 2D grid while avoiding flooded cells `"*"` and stone cells `"X"`. The grid evolves over time: at each second, every empty cell `"."` adjacent to a flooded cell also becomes flooded. The person can move to any adjacent (up, down, left, right) cell at each second, but cannot step onto flooded or stone cells. The destination cell `"D"` is guaranteed never to flood, and there is exactly one `"S"` and one `"D"` in the grid. The grid size can go up to 100x100, which is moderate, but brute-force simulation of every path is potentially expensive.

The key challenge is that the flooding spreads dynamically, so the algorithm must consider both the shortest path in space and time while avoiding cells that will be flooded when the person arrives.

Important edge cases include when the start is immediately adjacent to flooded cells, when the destination is blocked by stones, and when the flooding spreads faster than the person can move.

## Approaches

A brute-force approach would attempt to simulate every possible path from `"S"` to `"D"` while updating the flooded cells at each step. This approach would be correct but highly inefficient because the number of potential paths grows exponentially with the grid size, making it infeasible for a 100x100 grid.

The optimal approach uses a **multi-source BFS** strategy. First, precompute the earliest time each cell will be flooded using BFS starting from all initial flooded cells. Then, perform a BFS from `"S"` to `"D"`, but only move into a cell if the current time is less than the time it becomes flooded. This guarantees the shortest path in terms of time while accounting for dynamic flooding.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(4^(n*m)) | O(n*m) | Simulates every possible path with dynamic flooding; impractical |
| Optimal BFS | O(n*m) | O(n*m) | Precompute flood times, then BFS from `"S"` to `"D"` avoiding flooded cells |

## Algorithm Walkthrough

1. **Initialize Flood Times**: Create a 2D array `flood_time` of size `n x m` initialized with infinity. This will store the earliest second each cell becomes flooded.
2. **Multi-source BFS for Flooding**: Initialize a queue with all initially flooded cells `"*"` and set their flood time to 0. Perform BFS: for each cell, propagate the flood to adjacent empty cells if they have not been flooded yet, updating their flood time as the current time plus one.
3. **BFS for Movement**: Initialize a queue with the starting cell `"S"` and set the current time to 0. Perform BFS: at each step, move to adjacent cells if they are empty or `"D"` and the current time plus one is strictly less than the flood time of that cell. Keep track of visited cells to prevent revisiting.
4. **Check Destination**: If we reach the destination `"D"`, return the time. If the BFS ends without reaching `"D"`, return -1.

**Why it works**: By precomputing the flood times, the BFS for the person ensures we only step into cells that are safe at the current second. BFS guarantees the minimum number of steps since it explores all positions at the current time before moving to the next time, ensuring the shortest-time path is found.

## Python Solution

```python
from collections import deque
from typing import List

class Solution:
    def minimumSeconds(self, land: List[List[str]]) -> int:
        n, m = len(land), len(land[0])
        flood_time = [[float('inf')] * m for _ in range(n)]
        queue = deque()

        # Initialize flood BFS
        for i in range(n):
            for j in range(m):
                if land[i][j] == '*':
                    queue.append((i, j, 0))
                    flood_time[i][j] = 0

        directions = [(-1,0), (1,0), (0,-1), (0,1)]
        
        # BFS to calculate flood times
        while queue:
            x, y, t = queue.popleft()
            for dx, dy in directions:
                nx, ny = x + dx, y + dy
                if 0 <= nx < n and 0 <= ny < m:
                    if land[nx][ny] == '.' and t + 1 < flood_time[nx][ny]:
                        flood_time[nx][ny] = t + 1
                        queue.append((nx, ny, t + 1))
        
        # Find starting position
        for i in range(n):
            for j in range(m):
                if land[i][j] == 'S':
                    start = (i, j)
                if land[i][j] == 'D':
                    dest = (i, j)
        
        # BFS for movement
        visited = [[False] * m for _ in range(n)]
        queue = deque()
        queue.append((start[0], start[1], 0))
        visited[start[0]][start[1]] = True

        while queue:
            x, y, t = queue.popleft()
            if (x, y) == dest:
                return t
            for dx, dy in directions:
                nx, ny = x + dx, y + dy
                if 0 <= nx < n and 0 <= ny < m:
                    if not visited[nx][ny] and land[nx][ny] in ('.', 'D') and t + 1 < flood_time[nx][ny]:
                        visited[nx][ny] = True
                        queue.append((nx, ny, t + 1))
        
        return -1
```

The code first precomputes the time each cell will be flooded, then safely navigates from `"S"` to `"D"` using BFS. Visited cells prevent infinite loops and repeated processing.

## Go Solution

```go
package main

import "container/list"

func minimumSeconds(land [][]string) int {
    n, m := len(land), len(land[0])
    floodTime := make([][]int, n)
    for i := range floodTime {
        floodTime[i] = make([]int, m)
        for j := range floodTime[i] {
            floodTime[i][j] = 1 << 30
        }
    }

    type Point struct { x, y, t int }
    queue := list.New()

    var startX, startY, destX, destY int

    for i := 0; i < n; i++ {
        for j := 0; j < m; j++ {
            if land[i][j] == "*" {
                floodTime[i][j] = 0
                queue.PushBack(Point{i, j, 0})
            }
            if land[i][j] == "S" {
                startX, startY = i, j
            }
            if land[i][j] == "D" {
                destX, destY = i, j
            }
        }
    }

    dirs := [][2]int{{-1,0},{1,0},{0,-1},{0,1}}

    // BFS for flood
    for queue.Len() > 0 {
        front := queue.Remove(queue.Front()).(Point)
        for _, d := range dirs {
            nx, ny := front.x + d[0], front.y + d[1]
            if nx >= 0 && nx < n && ny >= 0 && ny < m {
                if land[nx][ny] == "." && front.t + 1 < floodTime[nx][ny] {
                    floodTime[nx][ny] = front.t + 1
                    queue.PushBack(Point{nx, ny, front.t + 1})
                }
            }
        }
    }

    visited := make([][]bool, n)
    for i := range visited {
        visited[i] = make([]bool, m)
    }

    queue.PushBack(Point{startX, startY, 0})
    visited[startX][startY] = true

    // BFS for movement
    for queue.Len() > 0 {
        front := queue.Remove(queue.Front()).(Point)
        if front.x == destX && front.y == destY {
            return front.t
        }
        for _, d := range dirs {
            nx, ny := front.x + d[0], front.y + d[1]
            if nx >= 0 && nx < n && ny >= 0 && ny < m {
                if !visited[nx][ny] && (land[nx][ny] == "." || land[nx][ny] == "D") && front.t + 1 < floodTime[nx][ny] {
                    visited[nx][ny] = true
                    queue.PushBack(Point{nx, ny, front.t + 1})
                }
            }
        }
    }

    return -1
}
```

In Go, we use the `container/list` package for BFS queues and define a `Point` struct for coordinates and time. Slice initialization handles `floodTime` and `visited` matrices.
