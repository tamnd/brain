## Solution

## Problem Understanding

The problem is asking us to remove duplicate rows in a DataFrame based on the `email` column. In simpler terms, if two or more rows have the same `email`, we need to keep only the first occurrence and discard all subsequent duplicates. The input is a tabular DataFrame representing customer information, with columns `customer_id`, `name`, and `email`. The expected output is a DataFrame with the same columns but without any duplicate emails, preserving the original order of rows for the first occurrence of each email.

The constraints implied by the problem suggest that the input size is small to moderate, typical for an "Easy" difficulty LeetCode problem. The problem guarantees that each row has valid values and that there is at least one column to identify duplicates (`email`). Important edge cases include an empty DataFrame, a DataFrame with all unique emails, and a DataFrame where all rows share the same email.

## Approaches

The brute-force approach is to iterate through each row of the DataFrame and compare the email with every other row that comes after it. If a duplicate is found, we remove the later row. While this would give the correct answer, it is inefficient because it requires nested loops, resulting in a time complexity of O(n²), which becomes slow as the number of rows increases.

The optimal approach leverages the observation that we can track seen emails using a hash set. By iterating over each row once and checking if the email has been encountered before, we can decide whether to keep or drop the row in O(1) time per row. This reduces the overall time complexity to O(n) while maintaining the order of first occurrences.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Iterates through every pair of rows to remove duplicates |
| Optimal | O(n) | O(n) | Uses a hash set to track seen emails and removes duplicates in one pass |

## Algorithm Walkthrough

1. Initialize an empty set `seen_emails` to track emails that have already been encountered.
2. Initialize an empty list `result_rows` to store the rows that will be kept in the final DataFrame.
3. Iterate over each row in the input DataFrame in order.
4. For each row, check if the `email` value exists in `seen_emails`.
5. If the email is not in the set, add it to `seen_emails` and append the row to `result_rows`.
6. If the email is already in the set, skip this row, as it is a duplicate.
7. After processing all rows, construct a new DataFrame from `result_rows`.
8. Return the resulting DataFrame as the output.

**Why it works**: The algorithm works because the set ensures that each email is only added once. By iterating in order, the first occurrence is kept, and duplicates are naturally ignored. The set provides constant-time checks and inserts, making the algorithm efficient.

## Python Solution

```python
import pandas as pd

class Solution:
    def dropDuplicateRows(self, customers: pd.DataFrame) -> pd.DataFrame:
        seen_emails = set()
        result_rows = []

        for _, row in customers.iterrows():
            email = row['email']
            if email not in seen_emails:
                seen_emails.add(email)
                result_rows.append(row)

        return pd.DataFrame(result_rows)
```

This implementation first creates a set `seen_emails` to track unique emails. Using `iterrows()`, it loops through each row of the DataFrame. If the email has not been seen, it is added to the set and the row is appended to `result_rows`. Finally, a new DataFrame is created from the list of rows, ensuring that the original order is preserved and duplicates are removed.

## Go Solution

```go
package main

import (
    "github.com/kniren/gota/dataframe"
)

func DropDuplicateRows(customers dataframe.DataFrame) dataframe.DataFrame {
    seenEmails := make(map[string]bool)
    var rows [][]string

    for i := 0; i < customers.Nrow(); i++ {
        email := customers.Elem(i, customers.ColIndex("email")).String()
        if !seenEmails[email] {
            seenEmails[email] = true
            var row []string
            for j := 0; j < customers.Ncol(); j++ {
                row = append(row, customers.Elem(i, j).String())
            }
            rows = append(rows, row)
        }
    }

    return dataframe.LoadRecords(rows, dataframe.DetectTypes(false))
}
```

In Go, we use a map `seenEmails` to track unique emails. We iterate through each row, check if the email has been encountered, and if not, append the row to a slice of slices. The final DataFrame is constructed using `LoadRecords`. Differences from Python include explicit type handling and row-by-row string extraction, because Go does not have pandas' `DataFrame` abstraction natively.

## Edge Cases

One important edge case is an empty DataFrame. If not handled correctly, iterating through rows could produce an error. The implementation works safely because iterating over zero rows results in an empty result.

Another edge case is a DataFrame where all emails are unique. A naive implementation might attempt unnecessary deletions or fail to preserve order. Using a set and appending only unseen emails ensures every row is preserved.

A third edge case is a DataFrame where all rows have the same email. The implementation correctly retains only the first row, as the set immediately flags all subsequent rows as duplicates. This ensures correctness regardless of the number of duplicates.
