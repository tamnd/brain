## Solution

## Problem Understanding

The problem gives us an integer array `nums`, and we need to find two numbers whose largest digit is the same. Among all such valid pairs, we must return the maximum possible sum.

For example, consider the number `2373`. Its digits are `2`, `3`, `7`, and `3`, so the largest digit is `7`. The problem asks us to group numbers according to this largest digit value.

If two numbers both have largest digit `7`, they form a valid pair. We then compute their sum and try to maximize it across all valid pairs.

The input size is relatively small:

- `2 <= nums.length <= 100`
- `1 <= nums[i] <= 10^4`

These constraints tell us several important things:

- Each number contains at most 5 digits.
- Even a quadratic solution is technically fast enough because `100^2 = 10,000`.
- However, the problem clearly encourages recognizing the grouping structure and solving it more efficiently.

A few edge cases are important:

- There may be no valid pair at all. In that case, we return `-1`.
- Multiple numbers may share the same largest digit, so we must choose the two largest among them.
- Numbers can contain repeated digits, but only the maximum digit matters.
- Single digit numbers are valid, and their largest digit is simply the number itself.

## Approaches

### Brute Force

The most direct solution is to check every possible pair of numbers.

For each pair `(nums[i], nums[j])`, we compute the largest digit of both numbers. If those largest digits match, we compute the sum and update the answer if it is larger than the current maximum.

This approach is correct because it explicitly examines every possible valid pair. Since no pair is skipped, the maximum valid sum will always be found.

However, this solution performs redundant work. The largest digit of the same number may be recomputed many times across different pair checks.

### Optimal Approach

The key observation is that we only care about grouping numbers by their largest digit.

There are only 10 possible largest digits, from `0` to `9`. For each digit group, we only need the two largest numbers because those produce the maximum pair sum for that group.

Instead of comparing all pairs, we can:

1. Compute the largest digit of each number.
2. Store the largest number seen so far for each digit group.
3. When another number with the same largest digit appears, compute a candidate sum immediately.

This reduces unnecessary comparisons and gives a cleaner linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n² × d) | O(1) | Checks every pair and recomputes largest digits |
| Optimal | O(n × d) | O(1) | Groups numbers by largest digit using a hash map |

Here, `d` is the number of digits in a number, which is at most 5.

## Algorithm Walkthrough

1. Create a hash map that stores the largest number seen for each possible largest digit.

The key is the largest digit, and the value is the largest number encountered so far with that digit.
2. Initialize the answer as `-1`.

This handles the case where no valid pair exists.
3. Iterate through each number in `nums`.

For every number, determine its largest digit by repeatedly extracting digits with modulo and division.
4. Check whether this largest digit already exists in the hash map.

If it does, we have found another number with the same largest digit, so a valid pair exists.
5. Compute the candidate sum.

Add the current number to the stored maximum number for that digit group, then update the global answer if this sum is larger.
6. Update the stored maximum for this digit group.

We always keep the largest number seen so far because larger numbers lead to larger future sums.
7. Continue until all numbers are processed.
8. Return the final answer.

### Why it works

For any largest-digit group, the maximum pair sum must come from the two largest numbers in that group. The algorithm maintains the largest number seen so far for each group. Whenever another number in the same group appears, it forms the best possible pair involving that new number and any earlier number. Since every number is processed exactly once, and every valid group comparison is considered, the algorithm correctly finds the global maximum pair sum.

## Python Solution

```python
from typing import List

class Solution:
    def maxSum(self, nums: List[int]) -> int:
        def largest_digit(num: int) -> int:
            maximum = 0
            
            while num > 0:
                maximum = max(maximum, num % 10)
                num //= 10
            
            return maximum

        best_by_digit = {}
        answer = -1

        for num in nums:
            digit = largest_digit(num)

            if digit in best_by_digit:
                answer = max(answer, num + best_by_digit[digit])

            best_by_digit[digit] = max(
                best_by_digit.get(digit, 0),
                num
            )

        return answer
```

The implementation begins with a helper function `largest_digit`, which extracts digits one by one using modulo (`% 10`) and integer division (`// 10`). It tracks the maximum digit encountered.

The dictionary `best_by_digit` stores the largest number seen for each largest-digit group.

As we iterate through `nums`, we first compute the current number's largest digit. If that digit already exists in the dictionary, we immediately form a candidate pair sum using the stored maximum number.

After checking the pair, we update the dictionary entry to ensure it always stores the largest number encountered for that digit group.

The variable `answer` tracks the maximum valid sum across all groups.

## Go Solution

```go
func maxSum(nums []int) int {
	largestDigit := func(num int) int {
		maximum := 0

		for num > 0 {
			digit := num % 10
			if digit > maximum {
				maximum = digit
			}
			num /= 10
		}

		return maximum
	}

	bestByDigit := make(map[int]int)
	answer := -1

	for _, num := range nums {
		digit := largestDigit(num)

		if value, exists := bestByDigit[digit]; exists {
			sum := value + num
			if sum > answer {
				answer = sum
			}
		}

		if current, exists := bestByDigit[digit]; !exists || num > current {
			bestByDigit[digit] = num
		}
	}

	return answer
}
```

The Go implementation follows the same logic as the Python version. A map is used instead of a Python dictionary.

Go does not provide a built-in `max` function for integers in older versions, so comparisons are handled explicitly with `if` statements.

Since the constraints are small and all values fit comfortably inside standard integers, there are no overflow concerns.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × d) | Each number is processed once, and digit extraction takes `d` operations |
| Space | O(1) | At most 10 digit groups are stored |

The algorithm processes each number exactly once. Computing the largest digit requires scanning the digits of the number, which takes at most 5 iterations because `nums[i] <= 10^4`.

The extra memory usage is constant because there are only 10 possible largest digits, from `0` through `9`.

## Edge Cases

One important edge case is when no valid pair exists. This happens when every number has a different largest digit. A careless implementation might incorrectly return `0` or another default value. The solution avoids this by initializing the answer to `-1` and only updating it when a valid pair is found.

Another edge case occurs when more than two numbers belong to the same largest-digit group. In this situation, we must ensure we use the two largest numbers, not merely the first two encountered. The implementation handles this by continuously storing the maximum number seen so far for each group.

A third edge case involves single digit numbers. Since the number itself is its largest digit, values like `7`, `8`, and `9` must still be processed correctly. The digit extraction loop naturally handles this case because the single digit is immediately identified as the maximum digit.

A subtle case appears when a smaller number is processed after a larger number in the same group. For example, with `[90, 81, 72]`, all numbers have largest digit `9`, `8`, and `7` respectively, so no pair exists. However, with `[98, 89, 79]`, all belong to the `9` group, and the algorithm must preserve `98` as the group's maximum even after seeing smaller numbers. The update logic correctly keeps only the largest value for future comparisons.
