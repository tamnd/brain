## Solution

## Problem Understanding

The problem gives us a circular array `nums`, meaning the first and last elements are considered adjacent. At every second, each index can simultaneously choose one of three values:

- Its current value
- The value from its left neighbor
- The value from its right neighbor

The goal is to determine the minimum number of seconds required so that every element in the array becomes equal.

The important detail is that updates happen simultaneously. This means every position chooses based on the values from the previous second, not partially updated values during the same operation.

We are not asked to produce the sequence of operations. We only need the minimum number of seconds.

The array length can be as large as `10^5`, which immediately tells us that quadratic simulations will be too slow. We need a solution close to linear time.

To understand the transformation process, think about how a value spreads through the array. If a position already contains some value `x`, then after one second, the adjacent positions may also become `x`. After two seconds, positions at distance two may become `x`, and so on. In other words, every occurrence of a value acts like a spreading source.

The key challenge is determining which value can spread to the entire circular array the fastest.

Several edge cases are important:

- If all elements are already equal, the answer is `0`.
- Since the array is circular, the distance between the last and first occurrence of a value must also be considered.
- A value may appear only once, which means it must spread around the entire circle.
- Multiple values may compete, and we want the one requiring the fewest seconds.

## Approaches

### Brute Force Approach

A direct simulation approach would repeatedly apply the operation second by second until all values become equal.

At each second, for every position, we would determine whether it can copy a neighboring value. We would continue simulating until the array becomes uniform.

This approach is correct because it exactly follows the rules of the problem. However, it is inefficient.

The issue is that each second requires scanning the entire array, and the number of seconds could itself be proportional to `n`. In the worst case, this leads to `O(n^2)` complexity, which is too slow for `n = 10^5`.

### Key Insight

Instead of simulating the process, we should think about how values spread.

Suppose we want the entire array to become value `x`.

If we know all positions where `x` initially appears, then every second, the influence of `x` expands outward by one step in both directions.

The slowest region to convert is the largest gap between consecutive occurrences of `x`.

For example:

```
x .... x
```

If the gap length is `g`, then both occurrences of `x` spread toward the middle simultaneously. The time required is:

```
g // 2
```

Because the array is circular, we must also consider the wraparound gap between the last occurrence and the first occurrence.

Therefore:

1. Collect all indices for each value.
2. Compute the maximum circular gap between consecutive occurrences.
3. The required seconds for that value is `max_gap // 2`.
4. Take the minimum across all values.

This transforms the problem into a clean gap analysis problem.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Simulates every second until the array becomes equal |
| Optimal | O(n) | O(n) | Uses index gaps between occurrences of each value |

## Algorithm Walkthrough

1. Create a hash map where the key is a value from the array and the value is a list of indices where that number appears.

For example:

```
nums = [2,1,3,3,2]
```

Produces:

```
2 -> [0,4]
1 -> [1]
3 -> [2,3]
```

We use a hash map because we need efficient grouping of indices by value.
2. Initialize the answer as a very large number.

We will compute the required time for every distinct value and keep the minimum.
3. For each value, examine the gaps between consecutive indices.

Suppose the indices are:

```
[i1, i2, i3]
```

The gaps are:

```
i2 - i1
i3 - i2
```
4. Handle the circular wraparound gap.

Since the array is circular, we must also include the distance from the last occurrence back to the first occurrence through the end of the array.

The wraparound gap is:

```
n - last_index + first_index
```
5. Find the maximum gap for the current value.

This represents the largest segment that must be filled by spreading.
6. Convert the gap into required seconds.

If two occurrences spread toward each other, the maximum time needed is:

```
max_gap // 2
```
7. Update the global minimum answer.

Repeat the process for every distinct value.
8. Return the minimum answer after processing all values.

### Why it works

Each occurrence of a value expands outward by one position per second. The slowest part of the array to convert is always the midpoint of the largest gap between consecutive occurrences of that value.

Because spreading happens simultaneously from both ends of a gap, only half the gap length determines the required time. By minimizing this maximum propagation distance across all values, we obtain the optimal answer.

## Python Solution

```python
from collections import defaultdict
from typing import List

class Solution:
    def minimumSeconds(self, nums: List[int]) -> int:
        n = len(nums)

        positions = defaultdict(list)

        for index, value in enumerate(nums):
            positions[value].append(index)

        answer = n

        for indices in positions.values():
            max_gap = 0

            for i in range(1, len(indices)):
                gap = indices[i] - indices[i - 1]
                max_gap = max(max_gap, gap)

            circular_gap = n - indices[-1] + indices[0]
            max_gap = max(max_gap, circular_gap)

            answer = min(answer, max_gap // 2)

        return answer
```

The implementation begins by grouping all indices for each distinct value using a `defaultdict(list)`.

Next, for every value, we compute the gaps between consecutive occurrences. These gaps represent regions that need to be filled by propagation.

The circular gap is especially important because the array wraps around. Without this step, the algorithm would incorrectly treat the array as linear.

After identifying the maximum gap, we divide it by two using integer division. This models the simultaneous spreading from both ends.

Finally, we keep the minimum required time across all values and return it.

## Go Solution

```go
package main

func minimumSeconds(nums []int) int {
	n := len(nums)

	positions := make(map[int][]int)

	for i, value := range nums {
		positions[value] = append(positions[value], i)
	}

	answer := n

	for _, indices := range positions {
		maxGap := 0

		for i := 1; i < len(indices); i++ {
			gap := indices[i] - indices[i-1]

			if gap > maxGap {
				maxGap = gap
			}
		}

		circularGap := n - indices[len(indices)-1] + indices[0]

		if circularGap > maxGap {
			maxGap = circularGap
		}

		if maxGap/2 < answer {
			answer = maxGap / 2
		}
	}

	return answer
}
```

The Go implementation follows the same logic as the Python version.

A `map[int][]int` stores all occurrence indices for each value. Since Go does not provide a built in `max` function for integers, comparisons are done manually with `if` statements.

Integer division in Go automatically truncates toward zero, which matches the required floor division behavior.

## Edge Cases

### All Elements Are Already Equal

Consider:

```
[5,5,5,5]
```

A naive implementation might still attempt unnecessary propagation steps. In this case, every gap between occurrences is `1`, so the computed answer becomes `1 // 2 = 0`.

The implementation naturally handles this without special casing.

### A Value Appears Only Once

Consider:

```
[1,2,3,4]
```

Each value appears only once. That means a single occurrence must spread around the entire circular array.

The circular gap becomes the full array length, and the answer correctly becomes:

```
n // 2
```

This is important because forgetting the circular interpretation would produce incorrect distances.

### Circular Wraparound Is Required

Consider:

```
[1,2,3,1]
```

The two `1`s are located at indices `0` and `3`. In a linear array, they appear far apart, but in a circular array they are adjacent.

If we ignored the circular gap calculation, we would incorrectly think the maximum gap is `3`. The wraparound computation correctly identifies the smaller circular distance and produces the optimal answer.
