## Solution

## Problem Understanding

The problem asks us to transform a given string `num`, representing a non-negative integer, into a number divisible by 25 using the minimum number of **deletions**. Deletions can remove any single digit at a time, and if we remove all digits, the resulting number is considered `0`, which is divisible by 25.

The input is a string `num` of length 1 to 100, containing digits `0-9` with no leading zeros. The output is the minimum number of deletions required to make the number **special**, where a special number is divisible by 25.

The problem relies on understanding divisibility rules for 25: any number divisible by 25 ends with one of the following pairs of digits: `"00"`, `"25"`, `"50"`, or `"75"`. Thus, the task reduces to rearranging digits (via deletions) to leave one of these pairs at the end, with as few deletions as possible.

Important edge cases include a single-digit input like `"0"` or `"5"`, strings already ending with a valid pair, or strings where forming a valid pair requires removing almost all digits.

## Approaches

The brute-force approach would be to generate all subsequences of `num`, check each resulting number for divisibility by 25, and track the minimum number of deletions. While correct, this approach is exponentially slow because the number of subsequences of length `n` is `2^n`. For `n = 100`, this is infeasible.

The key insight for an optimal solution is to **look for the last two digits that can form a number divisible by 25**. Instead of checking all subsequences, we can iterate from the end of the string and try to find the minimal deletions needed to produce each valid ending (`"00"`, `"25"`, `"50"`, `"75"`). For each pair, we scan backward for the second-to-last digit first, then for the last digit, counting all skipped digits as deletions. This gives an efficient way to compute the minimum deletions without enumerating all subsequences.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Generate all subsequences and check divisibility by 25 |
| Optimal | O(n) | O(1) | Iterate from the end and check each target pair, counting deletions |

## Algorithm Walkthrough

1. Define a list of valid ending pairs that make a number divisible by 25: `"00"`, `"25"`, `"50"`, `"75"`.
2. Initialize a variable `min_ops` to infinity. This will track the minimum deletions needed.
3. For each valid pair, scan the string from the end backward to find the last digit of the pair.
4. Once the last digit is found, continue scanning backward to find the first digit of the pair.
5. Count all digits skipped during the scan as deletions.
6. If both digits are found, update `min_ops` with the minimum between the current `min_ops` and the number of deletions needed.
7. Return `min_ops`.

**Why it works**: This method guarantees the minimum deletions because we scan from the end and always pick the closest possible digits to form the required ending pair, ensuring that any deletions are strictly necessary.

## Python Solution

```python
class Solution:
    def minimumOperations(self, num: str) -> int:
        targets = ["00", "25", "50", "75"]
        n = len(num)
        min_ops = float('inf')
        
        for pair in targets:
            last_digit = pair[1]
            first_digit = pair[0]
            found_last = False
            ops = 0
            
            # Scan from the end to find last_digit
            for i in range(n-1, -1, -1):
                if not found_last:
                    if num[i] == last_digit:
                        found_last = True
                    else:
                        ops += 1
                else:
                    if num[i] == first_digit:
                        min_ops = min(min_ops, ops)
                        break
                    else:
                        ops += 1
        
        return min_ops
```

The solution defines all four valid pairs. For each pair, it searches backward to locate the two digits forming the pair. Each skipped digit increments the deletion counter `ops`. Once both digits are found, we update `min_ops` to ensure we track the minimum deletions across all pairs.

## Go Solution

```go
import "math"

func minimumOperations(num string) int {
    targets := []string{"00", "25", "50", "75"}
    n := len(num)
    minOps := math.MaxInt32

    for _, pair := range targets {
        lastDigit := pair[1]
        firstDigit := pair[0]
        foundLast := false
        ops := 0

        for i := n - 1; i >= 0; i-- {
            if !foundLast {
                if num[i] == lastDigit {
                    foundLast = true
                } else {
                    ops++
                }
            } else {
                if num[i] == firstDigit {
                    if ops < minOps {
                        minOps = ops
                    }
                    break
                } else {
                    ops++
                }
            }
        }
    }
    return minOps
}
```

The Go version follows the same logic as Python, with minor language-specific differences such as `math.MaxInt32` for initial `minOps` and slice indexing.

| --- | --- | --- | --- | --- |
| Target `"00"` | None found | None found | 7 | inf |
| Target `"25"` | '5' at index 5 | '2' at index 2 | 2 | 2 |
| Target `"50"` | '0' at index 4 | '5' at index 3 | 3 | 2 |
| Target `"75"` | '5' at index 5 | '7' at index 6 | can't form | 2 |

Result: `2`

**Example 2: `"2908305"`**

Following the same backward scan, the minimum deletions required to form `"25"` is `3`.

**Example 3: `"10"`**

Target `"00"` or `"50"` can be formed by deleting one digit to get `"0"`. Result: `1`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | For each of the 4 valid ending pairs, we scan the string backward once, resulting in O(4n) = O(n) |
| Space | O(1) | Only counters and constants are used; no additional space scales with input |

The complexity is efficient because we avoid checking all subsequences, focusing only on the four target endings.

## Edge Cases

A crucial edge case is when the number is `"0"` or already divisible by 25. In these cases, the algorithm correctly returns `0` deletions since the string ends with a valid pair.

Another edge case is a single-digit number, for example `"5"`. The algorithm correctly identifies that the only way to make it divisible by 25 is deleting it, leaving `"0"`.

Finally, strings where the required digits to form a valid pair are scattered far apart, such as `"1111110"` with a target `"00"`. The algorithm correctly counts all skipped digits as deletions, ensuring it finds the true minimum.
