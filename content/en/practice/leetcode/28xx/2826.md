## Solution

## Problem Understanding

We are given an array `nums` where every element is either `1`, `2`, or `3`. In one operation, we may remove any element from the array. Our goal is to determine the minimum number of removals required so that the remaining array becomes non-decreasing.

A non-decreasing array means every element is less than or equal to the next one. Since the array contains only the values `1`, `2`, and `3`, every valid final array must look like:

- some number of `1`s,
- followed by some number of `2`s,
- followed by some number of `3`s.

Any of these groups may be empty.

Another way to think about the problem is this: instead of minimizing removals, we can maximize the number of elements we keep while preserving non-decreasing order. The answer then becomes:

$$\text{minimum removals} = n - \text{length of longest non-decreasing subsequence}$$

where `n` is the length of the array.

The constraints are small:

- `1 <= nums.length <= 100`
- `nums[i]` is always `1`, `2`, or `3`

Because the input size is only 100, even an `O(n^2)` dynamic programming solution is fast enough. However, the follow-up asks whether we can do better, ideally `O(n)`.

Several edge cases are important:

- Arrays already sorted, such as `[1,1,2,2,3]`
- Arrays in completely descending order, such as `[3,2,1]`
- Arrays containing only one distinct value
- Very short arrays of length 1
- Alternating patterns like `[3,1,3,1,3,1]`

A naive implementation can easily make mistakes when handling transitions between groups, especially when deciding whether to keep or remove a value that appears out of order.

## Approaches

### Brute Force Approach

The brute-force idea is to try every possible subsequence of the array and check whether it is non-decreasing. Among all valid subsequences, we keep the longest one.

For each element, we have two choices:

- include it,
- or exclude it.

That produces `2^n` possible subsequences. For each subsequence, we can verify whether it is non-decreasing in linear time.

This approach is guaranteed to find the optimal answer because it exhaustively explores every valid possibility. However, it is far too slow. Even for `n = 100`, `2^100` possibilities are completely infeasible.

### Optimal Approach

The key observation is that removing the minimum number of elements is equivalent to keeping the maximum number of elements that already form a non-decreasing subsequence.

This becomes a longest non-decreasing subsequence problem.

Since the values are restricted to only `1`, `2`, and `3`, we can solve the problem very efficiently with dynamic programming.

We define:

- `dp1` = longest valid subsequence ending with `1`
- `dp2` = longest valid subsequence ending with `2`
- `dp3` = longest valid subsequence ending with `3`

As we scan through the array:

- A `1` can only extend subsequences ending in `1`
- A `2` can extend subsequences ending in `1` or `2`
- A `3` can extend subsequences ending in `1`, `2`, or `3`

This produces an `O(n)` solution with constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Enumerates every subsequence |
| Optimal | O(n) | O(1) | Dynamic programming with three states |

## Algorithm Walkthrough

### Optimal Dynamic Programming Algorithm

1. Initialize three variables:

- `dp1 = 0`
- `dp2 = 0`
- `dp3 = 0`

These variables represent the longest valid non-decreasing subsequence ending with each value.
2. Traverse the array from left to right.
3. If the current number is `1`:

- We can only append it to sequences already ending in `1`.
- Update:

```
dp1 = dp1 + 1
```
4. If the current number is `2`:

- We may append it after either `1`s or `2`s.
- The best previous subsequence length is `max(dp1, dp2)`.
- Update:

```
dp2 = max(dp1, dp2) + 1
```
5. If the current number is `3`:

- We may append it after `1`s, `2`s, or `3`s.
- The best previous subsequence length is `max(dp1, dp2, dp3)`.
- Update:

```
dp3 = max(dp1, dp2, dp3) + 1
```
6. After processing all elements, the longest non-decreasing subsequence length is:

```
max(dp1, dp2, dp3)
```
7. The minimum removals required are:

```
len(nums) - longest_subsequence
```

### Why it works

The algorithm maintains the optimal subsequence length for every possible ending value. Since a non-decreasing sequence may only transition from smaller values to equal or larger values, each state only depends on valid predecessor states.

At every step, the DP values represent the best subsequence achievable using elements processed so far. Because every new element either extends a valid subsequence or starts a new one, the algorithm always preserves the optimal solution.

## Python Solution

```python
from typing import List

class Solution:
    def minimumOperations(self, nums: List[int]) -> int:
        dp1 = 0
        dp2 = 0
        dp3 = 0

        for num in nums:
            if num == 1:
                dp1 += 1

            elif num == 2:
                dp2 = max(dp1, dp2) + 1

            else:
                dp3 = max(dp1, dp2, dp3) + 1

        longest = max(dp1, dp2, dp3)

        return len(nums) - longest
```

The implementation directly follows the dynamic programming transitions described earlier.

The variables `dp1`, `dp2`, and `dp3` each store the length of the best non-decreasing subsequence ending in that value.

When processing a `1`, we can only extend sequences already ending in `1`, because placing `1` after `2` or `3` would violate non-decreasing order.

When processing a `2`, we can extend either a sequence ending in `1` or one ending in `2`.

When processing a `3`, we can extend any valid sequence because `3` is the largest value.

Finally, we compute the longest valid subsequence and subtract its length from the array size to obtain the minimum removals.

## Go Solution

```go
func minimumOperations(nums []int) int {
    dp1 := 0
    dp2 := 0
    dp3 := 0

    for _, num := range nums {
        if num == 1 {
            dp1++
        } else if num == 2 {
            if dp1 > dp2 {
                dp2 = dp1 + 1
            } else {
                dp2 = dp2 + 1
            }
        } else {
            best := dp1
            if dp2 > best {
                best = dp2
            }
            if dp3 > best {
                best = dp3
            }

            dp3 = best + 1
        }
    }

    longest := dp1
    if dp2 > longest {
        longest = dp2
    }
    if dp3 > longest {
        longest = dp3
    }

    return len(nums) - longest
}
```

The Go implementation mirrors the Python solution closely. Since Go does not provide a built-in `max` function for integers, comparisons are written explicitly using `if` statements.

No special handling for `nil` slices is required because the problem guarantees at least one element. Integer overflow is also not a concern because the maximum array length is only 100.

## Edge Cases

### Arrays Already Non-Decreasing

An input such as `[1,1,2,2,3,3]` is already valid. A buggy implementation might accidentally remove elements unnecessarily if the DP transitions are incorrect.

This implementation handles the case naturally because every new element extends an existing valid subsequence. The longest subsequence becomes the full array length, producing zero removals.

### Strictly Decreasing Arrays

Inputs like `[3,2,1]` are challenging because almost every element conflicts with later elements.

The algorithm correctly identifies that the longest non-decreasing subsequence has length `1`, meaning we must remove the other two elements.

### Arrays Containing Only One Value

Arrays such as `[2,2,2,2]` should require no removals because equal values are allowed in a non-decreasing sequence.

The implementation correctly treats equal values as valid extensions of the same DP state, so the entire array is preserved.

### Alternating Patterns

Arrays like `[2,1,2,1,2,1]` can expose incorrect transition logic.

The DP formulation handles this correctly because each state always represents the optimal subsequence achievable so far. Even when values alternate repeatedly, the algorithm consistently chooses the best subsequence instead of making greedy local decisions.
