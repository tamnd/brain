## Solution

## Problem Understanding

The problem gives us the head of a singly linked list where each node contains an integer value. Our task is to modify this linked list in place by inserting a new node between every pair of adjacent nodes.

The value of each inserted node must be the greatest common divisor, usually abbreviated as GCD, of the two neighboring node values.

For example, suppose the list is:

```
18 -> 6 -> 10 -> 3
```

We examine every adjacent pair:

- `gcd(18, 6) = 6`
- `gcd(6, 10) = 2`
- `gcd(10, 3) = 1`

After inserting these values between the original nodes, the list becomes:

```
18 -> 6 -> 6 -> 2 -> 10 -> 1 -> 3
```

The important detail is that we are not replacing nodes. We are inserting entirely new nodes between the existing ones.

The constraints are relatively small:

- The linked list contains between `1` and `5000` nodes.
- Each node value is between `1` and `1000`.

These limits tell us that we do not need advanced optimization techniques. A linear traversal is more than fast enough.

There are several important edge cases to keep in mind.

If the list contains only one node, there are no adjacent pairs, so nothing should be inserted.

If two adjacent values are equal, their GCD is the value itself.

If two adjacent numbers are coprime, their GCD is `1`.

Because this is a singly linked list, careless pointer manipulation can easily break the chain or create cycles. Correct handling of `next` pointers is the most important implementation detail.

## Approaches

### Brute Force Approach

A straightforward approach is to first copy all linked list values into an array. Once the values are stored in an array, we can iterate through adjacent pairs, compute the GCD for each pair, and build an entirely new linked list containing both the original values and the inserted GCD values.

This works because arrays make adjacent access simple. We can process the values in order and reconstruct the desired output
