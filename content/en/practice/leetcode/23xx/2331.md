## Solution

## Problem Understanding

The problem presents a **full binary tree** in which leaf nodes represent boolean values (`0` for `False` and `1` for `True`) and non-leaf nodes represent boolean operators (`2` for `OR` and `3` for `AND`). The goal is to **evaluate the tree** recursively by applying the boolean operation at each non-leaf node to the results of its two children.

In simpler terms, the input is a tree structure where each node either carries a value to be returned directly (for leaves) or a boolean operation to combine its children’s evaluations. The expected output is a single boolean representing the evaluation of the root node.

The constraints indicate that the tree has at most 1000 nodes, all nodes are either leaves or have exactly two children, and values fall in the range `[0, 3]`. This ensures the tree is manageable in size and depth, allowing recursive approaches without hitting stack overflow in most environments.

Important edge cases include: a tree with a single node (a leaf), a tree where all operations are the same (`AND` or `OR` only), and trees with mixed operations that test short-circuit behavior in evaluation.

## Approaches

The **brute-force approach** is essentially what the problem describes: recursively traverse the tree from the root. For each node, if it is a leaf, return its boolean value; otherwise, recursively evaluate its left and right children and apply the boolean operation defined by the current node. This approach is correct and efficient for the given constraints because each node is visited exactly once.

The **key insight** is that since the tree is a **full binary tree** and has at most 1000 nodes, a simple **recursive depth-first search (DFS)** is optimal. There is no need for memoization or iterative stack management because the recursion depth will not exceed the number of nodes, and each node is visited once. We can implement this with a clean, direct recursion that mirrors the problem statement.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Recursively evaluate each node; n is the number of nodes. |
| Optimal | O(n) | O(n) | DFS recursion evaluates each node once; recursion stack uses space proportional to tree height. |

## Algorithm Walkthrough

1. **Check if the node is a leaf.** If both `left` and `right` are `None`, return the boolean value of the node directly (`False` for `0`, `True` for `1`). This is the base case for recursion.
2. **Recursively evaluate the left child.** Call the same evaluation function on `root.left` and store the result.
3. **Recursively evaluate the right child.** Call the evaluation function on `root.right` and store the result.
4. **Apply the operation of the current node.** If the node’s value is `2`, return the logical `OR` of the left and right evaluations. If the node’s value is `3`, return the logical `AND` of the left and right evaluations.
5. **Return the result.** This result propagates upward, eventually evaluating the root node.

**Why it works**: By definition, a full binary tree ensures every non-leaf has exactly two children. The recursive evaluation ensures that each subtree is fully evaluated before applying the operation of its parent. Because leaf values are correctly interpreted as booleans and operations match standard logical semantics, the recursion preserves correctness and eventually evaluates the root.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def evaluateTree(self, root: Optional[TreeNode]) -> bool:
        if not root.left and not root.right:
            return bool(root.val)
        
        left_val = self.evaluateTree(root.left)
        right_val = self.evaluateTree(root.right)
        
        if root.val == 2:
            return left_val or right_val
        elif root.val == 3:
            return left_val and right_val
        else:
            raise ValueError("Invalid node value")
```

The Python solution follows the DFS recursive algorithm. First, it checks if the node is a leaf and returns its boolean value. Otherwise, it evaluates both children and applies the node’s operation using `or` or `and`. The code is straightforward, mirroring the natural structure of the tree.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func evaluateTree(root *TreeNode) bool {
    if root.Left == nil && root.Right == nil {
        return root.Val == 1
    }
    
    leftVal := evaluateTree(root.Left)
    rightVal := evaluateTree(root.Right)
    
    if root.Val == 2 {
        return leftVal || rightVal
    } else if root.Val == 3 {
        return leftVal && rightVal
    } else {
        panic("Invalid node value")
    }
}
```

The Go solution mirrors the Python logic. Leaf detection checks for `nil` children, boolean evaluation uses Go’s `||` and `&&`, and error handling uses `panic` for unexpected values. Go requires explicit boolean conversion from `int` (`root.Val == 1`).

## Edge Cases

**Single-node tree**: The simplest case, either `0` or `1`. Recursive check handles this by returning the node’s boolean value directly.

**All nodes are OR**: Tests whether multiple nested ORs correctly propagate truth values. DFS correctly evaluates left and right before applying OR.

**All nodes are AND**: Tests whether multiple nested ANDs correctly propagate false values. DFS ensures each subtree is fully evaluated, so even if one leaf is false, the AND chain will produce false.

**Invalid node values**: Although the problem guarantees valid values, the implementation includes a guard (`raise ValueError` in Python, `panic` in Go) to catch unexpected values, preventing silent incorrect results.
