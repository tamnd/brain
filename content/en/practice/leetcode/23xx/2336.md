## Solution

## Problem Understanding

The problem requires designing a data structure that represents an infinite set of positive integers starting from 1, with the ability to efficiently extract the smallest element and add elements back if they were previously removed. In simpler terms, we start with a set `{1, 2, 3, 4, ...}`. Calling `popSmallest()` should always remove and return the smallest number currently available in the set. If a number is removed, it can later be added back with `addBack(num)`, provided it is not already in the set.

The input consists of method calls to the `SmallestInfiniteSet` class and optional integer arguments for `addBack()`. The expected output corresponds to the return values of `popSmallest()` calls and `null` for constructor or `addBack()` calls.

Constraints indicate that the numbers are capped at 1000 for `addBack()` calls and that the total number of operations does not exceed 1000. This allows us to use data structures with time complexity logarithmic in the number of elements rather than worrying about actual infinity.

Edge cases include calling `addBack()` on a number that is already in the set (should do nothing), calling `popSmallest()` after adding numbers back, and sequentially popping numbers beyond the initially expected range.

## Approaches

A **brute-force approach** would involve maintaining a list of all numbers in the set, removing the smallest one via linear search, and adding numbers back by checking for duplicates. This approach works logically but is inefficient because removing the smallest element from a large list is O(n) each time, making it slow as the set grows.

The **optimal approach** leverages two key observations. First, numbers naturally increase in sequence unless explicitly added back. Second, we need a fast way to retrieve and remove the smallest number while also supporting insertion of previously removed numbers. Using a min-heap (priority queue) to track the numbers that were removed and added back is ideal because it allows O(log n) insertion and removal. A hash set is also maintained to avoid duplicate entries in the heap. This ensures the smallest number is always either the next sequential integer or the smallest number in the heap.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) per pop | O(n) | Maintain full list, linear search for smallest, linear insertion |
| Optimal | O(log n) per pop/addBack | O(n) | Use min-heap for added-back numbers, track next sequential number |

## Algorithm Walkthrough

1. Initialize the class with an integer `next_number = 1`, a min-heap `added_back_heap = []`, and a hash set `added_back_set = set()`.
2. When `popSmallest()` is called, check if `added_back_heap` is non-empty. If it is, pop the smallest element from the heap and remove it from the set. Return this element.
3. If the heap is empty, return `next_number` and increment `next_number` by 1 to represent removing it from the infinite sequence.
4. When `addBack(num)` is called, first check if `num` is smaller than `next_number` and not in `added_back_set`. If both conditions hold, insert it into `added_back_heap` and `added_back_set`.
5. The invariant is that `next_number` always tracks the smallest number that has never been popped and not explicitly added back, while the heap contains all previously popped numbers that have been added back and are candidates for future pops.

Why it works: This approach guarantees that every call to `popSmallest()` returns the actual smallest number in the current set because we either return the smallest element from the heap (previously removed and added back) or the next sequential number. The hash set ensures no duplicates are added to the heap.

## Python Solution

```python
import heapq

class SmallestInfiniteSet:

    def __init__(self):
        self.next_number = 1
        self.added_back_heap = []
        self.added_back_set = set()

    def popSmallest(self) -> int:
        if self.added_back_heap:
            smallest = heapq.heappop(self.added_back_heap)
            self.added_back_set.remove(smallest)
            return smallest
        else:
            current = self.next_number
            self.next_number += 1
            return current

    def addBack(self, num: int) -> None:
        if num < self.next_number and num not in self.added_back_set:
            heapq.heappush(self.added_back_heap, num)
            self.added_back_set.add(num)
```

The Python implementation initializes `next_number` to track the next number in the infinite sequence. A min-heap efficiently tracks added-back numbers, and a set ensures no duplicates are inserted into the heap. `popSmallest()` checks the heap first, falling back to the sequence if the heap is empty. `addBack()` ensures only relevant numbers are pushed to the heap.

## Go Solution

```go
package main

import "container/heap"

type IntHeap []int

func (h IntHeap) Len() int           { return len(h) }
func (h IntHeap) Less(i, j int) bool { return h[i] < h[j] }
func (h IntHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }
func (h *IntHeap) Push(x any)        { *h = append(*h, x.(int)) }
func (h *IntHeap) Pop() any {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[0 : n-1]
    return x
}

type SmallestInfiniteSet struct {
    nextNumber     int
    addedBackHeap  *IntHeap
    addedBackSet   map[int]bool
}

func Constructor() SmallestInfiniteSet {
    h := &IntHeap{}
    heap.Init(h)
    return SmallestInfiniteSet{
        nextNumber:    1,
        addedBackHeap: h,
        addedBackSet:  make(map[int]bool),
    }
}

func (this *SmallestInfiniteSet) PopSmallest() int {
    if this.addedBackHeap.Len() > 0 {
        smallest := heap.Pop(this.addedBackHeap).(int)
        delete(this.addedBackSet, smallest)
        return smallest
    }
    current := this.nextNumber
    this.nextNumber++
    return current
}

func (this *SmallestInfiniteSet) AddBack(num int) {
    if num < this.nextNumber && !this.addedBackSet[num] {
        heap.Push(this.addedBackHeap, num)
        this.addedBackSet[num] = true
    }
}
```

In Go, the heap implementation requires a type implementing `heap.Interface`. The logic mirrors Python, using `addedBackSet` to prevent duplicates. The main difference is explicit type conversions and heap initialization.
