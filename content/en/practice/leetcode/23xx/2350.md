## Solution

## Problem Understanding

The problem asks us to find the **length of the shortest sequence of rolls** that cannot be formed as a subsequence from the given `rolls` array. The input consists of an array `rolls` representing the outcomes of `n` dice rolls and an integer `k` representing the number of sides of the dice. Each value in `rolls` is between `1` and `k`. The expected output is a single integer, the minimum length of a dice sequence that is **impossible** to construct from the `rolls`.

The key challenge is that for sequences of length `len`, there are `k^len` possible sequences of dice rolls. Checking all sequences is infeasible due to combinatorial explosion, especially when `k` and `n` are large. The constraints indicate `n` and `k` can each be up to `10^5`, so any brute-force solution attempting to enumerate sequences or subsequences will time out.

Important edge cases include sequences where some numbers in `1..k` never appear, `rolls` being very small, or `k` being larger than the length of `rolls`. The problem guarantees `1 <= rolls[i] <= k`, so we do not need to handle invalid dice values.

The intuitive goal is to identify the **first missing combination** of rolls as the length of the shortest impossible sequence.

## Approaches

The **brute-force approach** would attempt to generate all possible sequences of increasing lengths and check if they appear in `rolls`. For length `len`, there are `k^len` sequences. Checking each sequence as a subsequence in `rolls` is computationally prohibitive since both the number of sequences and the length of `rolls` can be very large. This approach is correct in principle but completely impractical given the input constraints.

The **optimal approach** relies on a greedy observation: the shortest impossible sequence is determined by **coverage of all numbers from 1 to k** in successive blocks. If we iterate through `rolls` and count occurrences of unique numbers, every time we encounter a full set of all `k` numbers, we can increment the sequence length. Essentially, the shortest impossible sequence length is equal to the number of times we can form full sets of `1..k` plus one. This method works because each completed set guarantees that all sequences of the current length are possible; as soon as we cannot form a new complete set, sequences of that length become impossible.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(k^len * n) | O(k^len) | Enumerate all sequences of increasing lengths and check subsequences in `rolls`. Impractical for large n, k. |
| Optimal | O(n) | O(k) | Track seen numbers in a rolling set. Increment length when a full set of 1..k is found. Efficient for constraints. |

## Algorithm Walkthrough

1. Initialize a `set` called `seen` to track unique numbers encountered in the current block and a counter `length` set to 1, representing the current sequence length under consideration.
2. Iterate through each roll in `rolls`. For each number, add it to the `seen` set.
3. After each addition, check if the size of `seen` equals `k`. If true, this means a complete set of numbers `1..k` has been seen. Clear the `seen` set and increment the `length` counter because the next sequence will need to include an additional roll to form a new combination.
4. Continue this process until all rolls are processed.
5. Return `length` as the shortest impossible sequence length.

**Why it works:** Every time we complete a set of all `k` numbers, it guarantees that all sequences of the current length can be formed. As soon as we cannot complete a new set, the next sequence of this length is impossible. The greedy approach ensures we capture the first missing sequence without enumerating all possibilities.

## Python Solution

```python
from typing import List

class Solution:
    def shortestSequence(self, rolls: List[int], k: int) -> int:
        seen = set()
        length = 1
        
        for roll in rolls:
            seen.add(roll)
            if len(seen) == k:
                seen.clear()
                length += 1
        
        return length
```

**Explanation:** We initialize a `seen` set to track numbers in the current block. Iterating through `rolls`, we add each roll to `seen`. Whenever `seen` contains all `k` numbers, we clear it and increment `length`. After processing all rolls, `length` represents the minimum length of a sequence that cannot appear in `rolls`.

## Go Solution

```go
func shortestSequence(rolls []int, k int) int {
    seen := make(map[int]struct{})
    length := 1
    
    for _, roll := range rolls {
        seen[roll] = struct{}{}
        if len(seen) == k {
            seen = make(map[int]struct{})
            length++
        }
    }
    
    return length
}
```

**Explanation:** In Go, we use a map to mimic a set, storing each number encountered. Once all `k` numbers are seen, we reset the map and increment the sequence length. Go-specific details include the use of `struct{}` for memory-efficient set representation.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We iterate through each roll once, performing constant-time set operations. |

The solution scales linearly with the number of rolls and uses space proportional to the number of dice sides, which is efficient given the constraints.
