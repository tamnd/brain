# LeetCode 2319, Check if Matrix Is X-Matrix

## Problem Understanding

The problem gives us a square matrix `grid` of size `n x n`. We must determine whether this matrix satisfies the definition of an X-Matrix.

A matrix is considered an X-Matrix when two conditions are true at the same time:

1. Every element on the two diagonals must be non-zero.
2. Every element not on the diagonals must be exactly zero.

There are two diagonals in a square matrix:

- The primary diagonal, where `row == col`
- The secondary diagonal, where `row + col == n - 1`

Any cell belonging to either diagonal must contain a non-zero value. Every other position must contain `0`.

The input is a 2D integer array where:

- `grid[i][j]` represents the value at row `i` and column `j`
- `n` is between `3` and `100`
- Values range from `0` to `100000`

The constraints are small enough that we can examine every cell directly without worrying about performance issues.

An important detail is that, for odd-sized matrices, the center element belongs to both diagonals simultaneously. That center element still only needs to be non-zero.

Several edge cases can easily cause bugs in a naive implementation:

- Forgetting that the center element in odd-sized matrices belongs to both diagonals
- Accidentally checking only one diagonal
- Incorrectly classifying off-diagonal cells
- Returning true too early before checking the entire matrix
- Treating diagonal cells with value `0` as valid

The problem guarantees that the matrix is square, so we do not need to validate dimensions.

## Approaches

### Brute Force Approach

A brute-force solution could separately collect all diagonal positions into a set, then iterate through the matrix and verify whether each cell satisfies the required condition.

The algorithm would work like this:

1. Traverse the primary diagonal and store positions.
2. Traverse the secondary diagonal and store positions.
3. Iterate through every matrix cell.
4. If the cell belongs to the diagonal set, verify it is non-zero.
5. Otherwise, verify it is zero.

This approach is correct because every matrix position is explicitly validated against the X-Matrix rules.

However, it uses additional memory to store diagonal coordinates, which is unnecessary because diagonal membership can be determined mathematically in constant time.

### Optimal Approach

The key observation is that a cell belongs to a diagonal if:

- `i == j`, or
- `i + j == n - 1`

This means we do not need any extra data structure.

We can directly iterate through every cell once and immediately determine whether it should be:

- Non-zero, if it is on a diagonal
- Zero, otherwise

The moment we find a violation, we return `false`.

If the entire matrix is processed without violations, we return `true`.

This is optimal because every cell must be checked at least once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n²) | Uses extra storage for diagonal coordinates |
| Optimal | O(n²) | O(1) | Directly checks diagonal membership mathematically |

## Algorithm Walkthrough

1. Store the matrix size `n`.

We need the matrix dimension to identify the secondary diagonal using the condition `i + j == n - 1`.
2. Traverse every cell in the matrix using nested loops.

The outer loop iterates through rows, and the inner loop iterates through columns.
3. Determine whether the current cell belongs to a diagonal.

A cell is part of the X shape if either:

- `i == j`
- `i + j == n - 1`
4. If the cell is on a diagonal, verify that its value is non-zero.

If a diagonal cell contains `0`, the matrix immediately fails the X-Matrix definition, so return `false`.
5. If the cell is not on a diagonal, verify that its value is zero.

Any non-zero off-diagonal value violates the definition, so return `false`.
6. Continue checking until all cells are processed.

If no violations are found, return `true`.

### Why it works

The algorithm examines every matrix position exactly once and classifies each position into one of two categories:

- Diagonal positions, which must be non-zero
- Non-diagonal positions, which must be zero

Since every cell is validated against the exact X-Matrix definition, the algorithm returns `true` if and only if the matrix satisfies all required conditions.

## Python Solution

```python
from typing import List

class Solution:
    def checkXMatrix(self, grid: List[List[int]]) -> bool:
        n = len(grid)

        for i in range(n):
            for j in range(n):
                is_diagonal = (i == j) or (i + j == n - 1)

                if is_diagonal:
                    if grid[i][j] == 0:
                        return False
                else:
                    if grid[i][j] != 0:
                        return False

        return True
```

The implementation begins by storing the matrix size in `n`.

The nested loops iterate through every position `(i, j)` in the matrix. For each cell, the variable `is_diagonal` determines whether the position belongs to either diagonal.

If the current cell is part of a diagonal, the code checks whether the value equals `0`. Since diagonal cells must be non-zero, encountering `0` immediately returns `False`.

If the current cell is not on a diagonal, the code checks whether the value is non-zero. Since off-diagonal cells must be `0`, any non-zero value causes an immediate failure.

If the loops complete without returning `False`, then every cell satisfies the required conditions, so the function returns `True`.

## Go Solution

```go
func checkXMatrix(grid [][]int) bool {
    n := len(grid)

    for i := 0; i < n; i++ {
        for j := 0; j < n; j++ {
            isDiagonal := (i == j) || (i+j == n-1)

            if isDiagonal {
                if grid[i][j] == 0 {
                    return false
                }
            } else {
                if grid[i][j] != 0 {
                    return false
                }
            }
        }
    }

    return true
}
```

The Go implementation follows the exact same logic as the Python version.

Go slices are used to represent the 2D matrix. Since the problem guarantees a valid square matrix, no additional dimension checks are necessary.

There are no integer overflow concerns because the matrix values are small and we only perform comparisons and index arithmetic.

The solution uses constant extra space because no additional arrays, maps, or sets are created.

## Edge Cases

### Odd-Sized Matrix Center Cell

In matrices with odd dimensions, the center cell belongs to both diagonals simultaneously. A buggy implementation might accidentally process this incorrectly or require separate handling.

This solution avoids the issue naturally because the condition:

```
(i == j) or (i + j == n - 1)
```

correctly classifies the center cell as diagonal. The cell only needs to be non-zero once, which the implementation handles automatically.

### Off-Diagonal Non-Zero Values

A common mistake is to focus only on validating diagonal cells while forgetting to ensure all other cells are zero.

For example:

```
[
  [1, 9, 2],
  [0, 5, 0],
  [3, 0, 4]
]
```

The value `9` violates the X-Matrix definition even though all diagonal elements are valid.

The implementation explicitly checks every non-diagonal position and immediately returns `False` when encountering a non-zero value.

### Zero Values on Diagonals

Another easy bug is assuming diagonal membership alone is sufficient without checking the actual value.

For example:

```
[
  [1, 0, 2],
  [0, 0, 0],
  [3, 0, 4]
]
```

The center element belongs to both diagonals but is `0`, which invalidates the matrix.

The implementation directly checks every diagonal cell and immediately fails if any diagonal value equals `0`.
