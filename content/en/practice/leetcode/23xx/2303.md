## Solution

## Problem Understanding

The problem asks us to compute the total amount of tax a person owes given a set of tax brackets and their income. Each bracket defines an upper bound and a tax percentage. The brackets are **sorted** by their upper bounds, and the tax for a bracket only applies to the portion of income within that bracket.

For instance, if the first bracket has an upper bound of 3 and a 50% tax rate, only the first 3 dollars are taxed at 50%. Any income beyond 3 dollars will fall into the next bracket, taxed at that bracket’s rate. The key is that the tax is **progressive**: each portion of income is taxed at the rate corresponding to its bracket.

The inputs are `brackets`, a 2D array where each entry is `[upperi, percenti]`, and an integer `income`. The expected output is a floating-point number representing the total tax, accurate to within `10^-5`. Constraints ensure `income` and bracket upper bounds are reasonably small (≤1000), and the brackets are sorted and non-overlapping. Edge cases include zero income, income falling entirely within the first bracket, and a single bracket.

## Approaches

The **brute-force approach** would iterate through every single dollar of income, determining which bracket it belongs to, and summing the tax for each dollar. This is correct because it simulates the tax process precisely, but it is inefficient when income is large, because it could perform up to `income` iterations, which is unnecessary given that we can calculate the taxed portion of a bracket directly.

The **optimal approach** leverages the progressive structure of tax brackets. Instead of iterating dollar by dollar, we iterate bracket by bracket. For each bracket, we compute the taxed portion as the **minimum of the remaining income and the bracket range**, then multiply by the bracket’s percentage. We accumulate this until all income is accounted for. This approach runs in O(n) time relative to the number of brackets and uses constant additional space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(income) | O(1) | Iterates each dollar of income, summing tax individually |
| Optimal | O(n) | O(1) | Iterates each bracket, computing tax based on bracket ranges |

## Algorithm Walkthrough

1. Initialize a variable `tax` to 0.0 to accumulate total taxes.
2. Initialize a variable `prev_upper` to 0 to keep track of the lower bound of the current bracket.
3. Iterate over each bracket `[upper, percent]` in `brackets`.
4. For each bracket, calculate the taxable portion as `min(income, upper) - prev_upper`. This ensures we do not tax income above what the person earned.
5. Multiply the taxable portion by `percent / 100.0` to get the tax for this bracket, and add it to `tax`.
6. Update `prev_upper` to `upper` for the next iteration.
7. If the income is less than or equal to the current bracket's upper bound, break the loop as no further brackets are relevant.
8. Return the accumulated `tax`.

**Why it works:** The algorithm correctly computes the tax for each bracket independently, applying the progressive rule. Since each dollar is counted exactly once in the appropriate bracket, and the brackets cover all possible income ranges, the total tax is correct.

## Python Solution

```python
from typing import List

class Solution:
    def calculateTax(self, brackets: List[List[int]], income: int) -> float:
        tax = 0.0
        prev_upper = 0
        for upper, percent in brackets:
            if income <= prev_upper:
                break
            taxable_income = min(income, upper) - prev_upper
            tax += taxable_income * percent / 100.0
            prev_upper = upper
        return tax
```

This Python implementation follows the algorithm exactly. `tax` accumulates the total tax, `prev_upper` tracks the lower bound of each bracket, and `min(income, upper) - prev_upper` ensures we never overcount the taxable portion. The loop breaks once all income has been accounted for.

## Go Solution

```go
func calculateTax(brackets [][]int, income int) float64 {
    tax := 0.0
    prevUpper := 0
    for _, bracket := range brackets {
        upper, percent := bracket[0], bracket[1]
        if income <= prevUpper {
            break
        }
        taxable := min(income, upper) - prevUpper
        tax += float64(taxable) * float64(percent) / 100.0
        prevUpper = upper
    }
    return tax
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

In Go, we need to explicitly convert integers to `float64` when performing division and multiplication to avoid integer truncation. The `min` helper function simplifies the calculation of taxable income.
