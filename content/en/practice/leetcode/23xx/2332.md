## Solution

## Problem Understanding

This problem asks us to determine the latest possible arrival time at a bus station such that we are still able to board a bus. There are several important rules that define how boarding works.

We are given two arrays:

- `buses`, where each value represents the departure time of a bus
- `passengers`, where each value represents the arrival time of a passenger

Each bus can carry at most `capacity` passengers. A passenger can board a bus if:

1. The passenger arrives at or before the bus departure time.
2. The bus still has available seats.

Passengers always board in chronological order. Earlier arriving passengers get priority over later ones. If more passengers are waiting than the bus can hold, only the earliest `capacity` passengers board.

Our task is to determine the latest integer time we can arrive and still successfully board some bus. However, there is one extra restriction: our arrival time cannot be equal to any existing passenger arrival time.

The arrays are not sorted initially, so sorting will be necessary. The constraints are as large as `10^5`, which strongly suggests that quadratic solutions will not work. Any efficient solution will likely involve sorting combined with a linear scan.

Several edge cases are important:

- The last bus may still have empty seats, allowing us to arrive exactly at the bus departure time.
- The last bus may become full, forcing us to arrive before the last boarded passenger.
- Our desired arrival time might already be occupied by another passenger, meaning we must move backward until we find an unused time.
- Passengers may arrive very early, and buses may depart much later.
- There may be only one bus or one passenger.

The problem guarantees that all bus times are unique and all passenger times are unique, which simplifies collision handling.

## Approaches

### Brute Force Approach

A brute force solution would try candidate arrival times starting from the latest possible moment and simulate the entire boarding process for each candidate time.

For every possible arrival time:

1. Insert ourselves into the passenger list.
2. Sort passengers again.
3. Simulate boarding for every bus.
4. Check whether we successfully boarded.

This approach is correct because it directly models the problem statement exactly. However, it is far too slow. Bus and passenger times can be as large as `10^9`, so scanning backward through times is impractical. Even if we only checked nearby times, repeatedly simulating boarding would still be extremely expensive.

### Optimal Approach

The key observation is that we only care about the final bus and the final boarding state.

After sorting both arrays:

1. Simulate the boarding process once.
2. Determine who boards the final bus.
3. If the final bus still has room, we can potentially arrive at the bus departure time.
4. If the final bus is full, we must arrive before the last passenger who boarded it.
5. Since we cannot share a time with another passenger, move backward until we find an unused time.

The important insight is that the boarding order is completely deterministic after sorting. We do not need to test many candidate times. We only need to compute the latest theoretically valid time, then adjust downward if that time conflicts with an existing passenger arrival.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(T × (m log m + n + m)) | O(m) | Repeatedly simulates boarding for many candidate times |
| Optimal | O(n log n + m log m) | O(m) | Sort once, simulate once, then adjust answer |

Here, `T` represents the number of candidate times checked in the brute force approach.

## Algorithm Walkthrough

1. Sort both `buses` and `passengers`.

Sorting is necessary because passengers always board in chronological order, and buses depart in chronological order.
2. Use a pointer to track the next unboarded passenger.

This allows us to efficiently simulate boarding without repeatedly scanning the passenger array.
3. Process each bus one by one.

For every bus:

- Reset the number of seats used on this bus.
- While:

- there are remaining passengers,
- the passenger arrives before or at the bus departure time,
- and the bus is not full,

board the passenger and move the pointer forward.
4. After processing all buses, examine the final bus state.

Two situations are possible:

- The final bus is not full.

In this case, we can potentially arrive exactly at the departure time of the last bus.
- The final bus is full.

In this case, we must arrive before the last passenger who successfully boarded that bus.
5. Compute the candidate answer.

- If the last bus has room:

`candidate = last_bus_time`
- Otherwise:

`candidate = last_boarded_passenger_time - 1`
6. Avoid collisions with passenger arrival times.

Since we cannot arrive at the same time as another passenger, repeatedly decrement the candidate while it exists in the passenger set.
7. Return the resulting value.

### Why it works

The boarding process is deterministic after sorting. Every passenger either boards or does not board based solely on arrival order and remaining capacity. The latest feasible arrival time must therefore be either:

- the departure time of the final bus, if seats remain, or
- immediately before the last boarded passenger on the final bus, if the bus is full.

Any later arrival would either miss the bus or lose priority to another passenger. By decrementing past occupied passenger times, we ensure the final answer remains valid.

## Python Solution

```python
from typing import List

class Solution:
    def latestTimeCatchTheBus(self, buses: List[int], passengers: List[int], capacity: int) -> int:
        buses.sort()
        passengers.sort()

        passenger_index = 0
        boarded_on_last_bus = 0

        for bus_time in buses:
            boarded_on_last_bus = 0

            while (
                passenger_index < len(passengers)
                and passengers[passenger_index] <= bus_time
                and boarded_on_last_bus < capacity
            ):
                passenger_index += 1
                boarded_on_last_bus += 1

        passenger_set = set(passengers)

        if boarded_on_last_bus < capacity:
            candidate = buses[-1]
        else:
            candidate = passengers[passenger_index - 1] - 1

        while candidate in passenger_set:
            candidate -= 1

        return candidate
```

The implementation begins by sorting both arrays so that buses and passengers can be processed chronologically.

The variable `passenger_index` acts as a pointer into the passenger array. As passengers board buses, this pointer advances and never moves backward. This guarantees linear traversal after sorting.

For each bus, the algorithm boards as many eligible passengers as possible, stopping either when the bus reaches capacity or when no waiting passenger can catch the bus.

After simulating all buses, the algorithm checks whether the final bus became full. If it did not become full, arriving exactly at the last bus departure time is possible. Otherwise, we must arrive before the last boarded passenger.

Finally, the algorithm handles the restriction that our arrival time cannot equal another passenger's arrival time. A set enables constant time collision checks while decrementing the candidate answer.

## Go Solution

```go
package main

import "sort"

func latestTimeCatchTheBus(buses []int, passengers []int, capacity int) int {
	sort.Ints(buses)
	sort.Ints(passengers)

	passengerIndex := 0
	boardedOnLastBus := 0

	for _, busTime := range buses {
		boardedOnLastBus = 0

		for passengerIndex < len(passengers) &&
			passengers[passengerIndex] <= busTime &&
			boardedOnLastBus < capacity {

			passengerIndex++
			boardedOnLastBus++
		}
	}

	passengerSet := make(map[int]bool)

	for _, passenger := range passengers {
		passengerSet[passenger] = true
	}

	var candidate int

	if boardedOnLastBus < capacity {
		candidate = buses[len(buses)-1]
	} else {
		candidate = passengers[passengerIndex-1] - 1
	}

	for passengerSet[candidate] {
		candidate--
	}

	return candidate
}
```

The Go implementation follows the same logic as the Python version. The primary difference is the use of `sort.Ints()` for sorting and a `map[int]bool` to simulate a hash set for fast passenger time lookups.

Go slices behave similarly to dynamic arrays, so the passenger pointer logic remains identical. Integer overflow is not a concern because all values remain comfortably within Go's integer range.

## Edge Cases

One important edge case occurs when the final bus still has empty seats. In this situation, the latest theoretically valid arrival time is simply the departure time of the final bus itself. A buggy implementation might incorrectly force the arrival time to be before another passenger even when extra capacity exists. This implementation correctly allows arriving exactly at the bus departure time, provided that time is not already occupied.

Another tricky case occurs when the final bus is completely full. In that scenario, arriving at the bus departure time is impossible because there are already enough earlier passengers occupying all seats. The correct strategy is to arrive before the last passenger who successfully boarded. The implementation tracks exactly which passenger boarded last and computes the candidate time accordingly.

A third important edge case involves consecutive occupied passenger times. Suppose the candidate answer is already used by another passenger, and several earlier times are also occupied. A naive implementation might decrement only once and still return an invalid answer. This solution repeatedly decrements until it finds an unused time, guaranteeing correctness even when many adjacent passenger times exist.
