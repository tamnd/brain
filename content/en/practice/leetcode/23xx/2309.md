## Solution

## Problem Understanding

The problem asks us to find the **greatest English letter** in a given string `s` that occurs in both lowercase and uppercase forms. The input is a string composed entirely of English letters, and the output should be a single uppercase letter. If no letter exists in both forms, the output is an empty string.

The string length is constrained to be at most 1000, which allows us to use linear scans and hash-based storage without worrying about performance. The main challenge is not simply detecting letters, but ensuring we account for both uppercase and lowercase versions. Edge cases include strings with no repeating letters in different cases, strings with only lowercase or only uppercase letters, and strings where multiple letters satisfy the condition, in which case we must return the **lexicographically greatest** letter.

## Approaches

The **brute-force approach** would involve iterating through each letter from 'A' to 'Z', checking if both its uppercase and lowercase versions exist in the string. While correct, this approach involves repeated string scans, resulting in higher time complexity than necessary. Specifically, checking the presence of each character in the string for 26 letters results in O(26 * n) = O(n), but it involves repeated membership checks over the string, which could be slow if implemented naively without a hash structure.

The **optimal approach** uses **hash sets** to store observed lowercase and uppercase letters. By scanning the string once, we populate two sets. Then we iterate from 'Z' to 'A', checking whether both the uppercase and lowercase versions exist in the sets. This ensures we find the greatest letter efficiently without repeated scans over the string. The use of sets guarantees constant-time membership checking.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n*26) | O(1) | Iterates through all letters and checks existence in string repeatedly |
| Optimal | O(n) | O(26*2) = O(1) | Single scan to build sets, then iterate from 'Z' to 'A' to find greatest letter |

## Algorithm Walkthrough

1. Initialize two empty sets: `lowercase_seen` and `uppercase_seen`. These will store all letters seen in their respective cases.
2. Iterate through each character `c` in the string `s`. If `c` is lowercase, add it to `lowercase_seen`. If `c` is uppercase, add it to `uppercase_seen`.
3. Iterate from the letter 'Z' down to 'A'. For each letter `L`, check if `L` exists in `uppercase_seen` **and** if its lowercase equivalent exists in `lowercase_seen`.
4. If such a letter is found, return it immediately, because iterating from 'Z' to 'A' guarantees it is the greatest.
5. If no letter satisfies the condition, return an empty string.

**Why it works:** By storing all observed letters in hash sets, we ensure O(1) lookup for both cases. Iterating from 'Z' to 'A' guarantees that the first match we find is the lexicographically greatest. This approach efficiently avoids redundant scanning and ensures correctness.

## Python Solution

```python
class Solution:
    def greatestLetter(self, s: str) -> str:
        lowercase_seen = set()
        uppercase_seen = set()
        
        for c in s:
            if c.islower():
                lowercase_seen.add(c)
            else:
                uppercase_seen.add(c)
        
        for ascii_code in range(ord('Z'), ord('A') - 1, -1):
            upper_char = chr(ascii_code)
            if upper_char in uppercase_seen and upper_char.lower() in lowercase_seen:
                return upper_char
        
        return ""
```

In the implementation, we first populate `lowercase_seen` and `uppercase_seen` by scanning `s`. Then we iterate in reverse ASCII order from 'Z' to 'A', checking membership in both sets. Returning immediately on the first match ensures the greatest letter is returned. If no matches are found, an empty string is returned.

## Go Solution

```go
func greatestLetter(s string) string {
    lowercaseSeen := make(map[rune]bool)
    uppercaseSeen := make(map[rune]bool)
    
    for _, c := range s {
        if c >= 'a' && c <= 'z' {
            lowercaseSeen[c] = true
        } else if c >= 'A' && c <= 'Z' {
            uppercaseSeen[c] = true
        }
    }
    
    for c := 'Z'; c >= 'A'; c-- {
        if uppercaseSeen[c] && lowercaseSeen[c+'a'-'A'] {
            return string(c)
        }
    }
    
    return ""
}
```

In Go, we use `map[rune]bool` for sets. The iteration over `s` uses `range` to handle runes. To convert uppercase to lowercase, we add the ASCII offset `'a' - 'A'`. The logic mirrors the Python solution.

## Edge Cases

The first important edge case is a string with **no letters appearing in both cases**. Without proper checks, a naive solution might return a lowercase or uppercase letter incorrectly. The implementation handles this by explicitly checking the existence of both lowercase and uppercase forms.

The second edge case is **strings with multiple valid letters**. Lexicographic ordering is crucial. Iterating from 'Z' to 'A' guarantees that the first match is the greatest.

The third edge case is **minimal input strings**, such as `"a"` or `"A"` or `"aA"`. The implementation correctly returns an empty string for single characters and handles the smallest valid pair without errors.

The algorithm also naturally handles strings containing repeated letters, as set insertion automatically de-duplicates characters.
