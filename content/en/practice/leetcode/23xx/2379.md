# LeetCode 2379 - Minimum Recolors to Get K Consecutive Black Blocks

## Problem Understanding

The problem gives us a string called `blocks`, where each character represents the color of a block. A character `'B'` means the block is black, and `'W'` means the block is white.

We are also given an integer `k`. Our goal is to make sure that somewhere in the string there exists a group of exactly `k` consecutive black blocks. We are allowed to perform operations where a white block is recolored into a black block. Each recoloring costs one operation.

The task is to compute the minimum number of recoloring operations needed so that at least one substring of length `k` contains only black blocks.

Another way to think about the problem is this:

For every contiguous substring of length `k`, count how many white blocks it contains. Since every white block in that window would need to be recolored to black, the number of white blocks directly equals the number of required operations for that window. The answer is therefore the minimum number of white blocks among all windows of size `k`.

The constraints are very small:

- `1 <= n <= 100`
- `1 <= k <= n`

Since `n` is at most 100, even relatively inefficient solutions would work. However, this problem is designed to teach the sliding window technique, which gives an elegant linear-time solution.

Several edge cases are important:

- The string may already contain `k` consecutive black blocks, so the answer could be `0`.
- `k` may equal the entire string length, meaning we must evaluate the whole string at once.
- The string may contain only white blocks, requiring recoloring of all `k` positions.
- The smallest possible input has `n = 1`.

## Approaches

### Brute Force Approach

The brute-force method checks every substring of length `k`.

For each starting position:

1. Extract the substring of length `k`.
2. Count how many characters are `'W'`.
3. Track the minimum count across all windows.

This works because every white block inside a candidate window must eventually become black. Therefore, the number of white blocks is exactly the number of required operations.

Although this approach is correct, it repeatedly scans overlapping windows from scratch. If `n` were much larger, this repeated work would become inefficient.

The brute-force time complexity is `O(n * k)` because we examine up to `n` windows and count up to `k` characters in each one.

### Optimal Sliding Window Approach

The key observation is that consecutive windows overlap heavily.

Suppose we already know the number of white blocks in a window of size `k`. When moving the window one position to the right:

- One character leaves the window.
- One character enters the window.

Instead of recounting all `k` characters, we can update the white count incrementally:

- If the outgoing character was white, decrement the count.
- If the incoming character is white, increment the count.

This is the classic sliding window optimization.

The sliding window technique reduces the time complexity to `O(n)` because each character is processed only a constant number of times.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * k) | O(1) | Recounts white blocks for every window |
| Optimal Sliding Window | O(n) | O(1) | Maintains white count incrementally |

## Algorithm Walkthrough

1. Initialize a variable `white_count` to store the number of white blocks in the current window.
2. Process the first window of size `k` by counting how many characters are `'W'`. This gives the number of recolors needed for the first candidate substring.
3. Store this initial value as `minimum_operations`, because it is currently the best answer we have seen.
4. Start sliding the window from left to right.
5. For each step:

- Remove the leftmost character from the current window.
- Add the new rightmost character into the window.
- If the removed character was `'W'`, decrement `white_count`.
- If the added character was `'W'`, increment `white_count`.
6. After updating the window, compare `white_count` with `minimum_operations` and keep the smaller value.
7. Continue until all windows of size `k` have been processed.
8. Return `minimum_operations`.

### Why it works

At every step, the sliding window represents exactly one contiguous substring of length `k`.

The variable `white_count` always stores the number of white blocks inside the current window. Since every white block would need to be recolored, `white_count` is exactly the number of operations needed for that window.

By examining every possible window of size `k` and taking the minimum white count, we guarantee that the algorithm returns the minimum number of recoloring operations.

## Python Solution

```python
class Solution:
    def minimumRecolors(self, blocks: str, k: int) -> int:
        white_count = 0

        # Count whites in the first window
        for i in range(k):
            if blocks[i] == 'W':
                white_count += 1

        minimum_operations = white_count

        # Slide the window
        for right in range(k, len(blocks)):
            left = right - k

            # Remove left character
            if blocks[left] == 'W':
                white_count -= 1

            # Add right character
            if blocks[right] == 'W':
                white_count += 1

            minimum_operations = min(minimum_operations, white_count)

        return minimum_operations
```

The implementation begins by counting the number of white blocks in the first window of size `k`. This establishes the initial number of operations needed.

The variable `minimum_operations` stores the best answer found so far.

Next, the code slides the window one position at a time. Instead of recalculating the entire window, it updates the count incrementally by removing the contribution of the outgoing character and adding the contribution of the incoming character.

After every update, the algorithm checks whether the current window requires fewer recolors than any previous window.

Finally, the smallest observed value is returned.

## Go Solution

```go
func minimumRecolors(blocks string, k int) int {
    whiteCount := 0

    // Count whites in the first window
    for i := 0; i < k; i++ {
        if blocks[i] == 'W' {
            whiteCount++
        }
    }

    minimumOperations := whiteCount

    // Slide the window
    for right := k; right < len(blocks); right++ {
        left := right - k

        // Remove left character
        if blocks[left] == 'W' {
            whiteCount--
        }

        // Add right character
        if blocks[right] == 'W' {
            whiteCount++
        }

        if whiteCount < minimumOperations {
            minimumOperations = whiteCount
        }
    }

    return minimumOperations
}
```

The Go implementation follows the same logic as the Python version.

Strings in Go can be indexed directly because the input contains only ASCII characters (`'W'` and `'B'`). No special Unicode handling is necessary.

The algorithm uses integer variables to track the current white count and the minimum answer. Since the constraints are extremely small, integer overflow is not a concern.

## Edge Cases

### Window Size Equals the Entire String

When `k == n`, there is only one possible window, the entire string itself. A buggy implementation might incorrectly attempt to slide beyond valid bounds.

This implementation handles the case naturally because the sliding loop starts at index `k`. If `k == len(blocks)`, the loop never runs, and the algorithm simply returns the white count from the initial window.

### String Already Contains Consecutive Black Blocks

If a valid window already exists with all black blocks, the correct answer is `0`.

Some incorrect implementations continue recoloring unnecessarily or fail to update the minimum correctly. In this solution, any window with zero white blocks immediately updates `minimum_operations` to `0`, which is the smallest possible answer.

### All Blocks Are White

If every block is `'W'`, every position in a window must be recolored.

For example, with `"WWWWW"` and `k = 3`, every window contains exactly three white blocks. The implementation correctly counts whites in each window and returns `3`.

### Minimum Window Size

When `k = 1`, every single character forms its own window.

The algorithm still works correctly because the first window contains exactly one character, and sliding simply checks each individual block. The answer becomes `0` if any block is black, otherwise `1`.
