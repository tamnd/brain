## Solution

## Problem Understanding

The problem gives us a database table named `Students`. Each row contains three pieces of information:

- `student_id`, uniquely identifying a student
- `department_id`, identifying the department the student belongs to
- `mark`, representing the student's exam score

We need to compute a percentage rank for every student within their own department.

The ranking rules are important:

- Students are ranked by `mark` in descending order
- The highest mark gets rank `1`
- If multiple students have the same mark, they receive the same rank
- The ranking behavior therefore matches SQL's `RANK()` window function

Once the rank is determined, we compute the percentage using this formula:

$$(rank - 1) \times 100 \div (department\_size - 1)$$

The final percentage must be rounded to two decimal places.

The result should contain:

- `student_id`
- `department_id`
- `percentage`

The order of the rows does not matter.

The most important observation is that this is fundamentally a window function problem. We need two values for every student:

1. Their rank inside the department
2. The total number of students inside the department

Both values are naturally computed using SQL window functions.

There are several important edge cases to think about.

If multiple students share the same mark, they must receive the same rank. This means we cannot use `ROW_NUMBER()`, because that assigns unique sequential numbers even for ties. We specifically need `RANK()`.

Another edge case is departments containing only one student. In that situation, the denominator becomes:

$$department\_size - 1 = 0$$

A direct division would cause a division-by-zero error. In practice, LeetCode test cases for this problem avoid invalid arithmetic, but a robust SQL solution should still handle this safely.

Finally, rounding behavior matters. SQL engines can produce many decimal places, so we must explicitly round the result to two decimal places using `ROUND()`.

## Approaches

### Brute Force Approach

A brute-force solution would process each student independently.

For every student:

1. Scan all other students in the same department
2. Count how many students have a strictly higher mark
3. Compute the student's rank as:

$$1 + (\text{number of higher marks})$$

1. Count the total number of students in the department
2. Apply the percentage formula

This approach is correct because ranking depends only on comparing marks within the same department.

However, it is inefficient because for every student we repeatedly rescan the table. If there are `n` students, each lookup may require scanning up to `n` rows, producing quadratic complexity.

### Optimal Approach

The key insight is that SQL window functions are specifically designed for grouped ranking computations.

We can compute all required information in one pass using:

- `RANK()` to compute student rank within each department
- `COUNT()` as a window function to compute department size

Both calculations can be partitioned by `department_id`.

Once these values are available, the percentage formula becomes straightforward.

This avoids repeated rescanning and lets the database engine optimize the computation internally.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Repeatedly scans the table for every student |
| Optimal | O(n log n) | O(n) | Uses window functions and sorting internally |

## Algorithm Walkthrough

1. Partition students by department using window functions.

We need rankings and counts computed independently for each department. Partitioning ensures students from different departments do not affect one another.
2. Compute the rank using `RANK()`.

We sort marks in descending order because higher marks should receive better ranks. Using `RANK()` correctly handles ties by assigning identical ranks to equal marks.
3. Compute the department size using `COUNT()`.

We count how many students exist in the same department. This value becomes the denominator in the percentage formula.
4. Apply the percentage formula.

For each student:

$$(rank - 1) \times 100 \div (department\_size - 1)$$

This normalizes the rank into a percentage scale from `0` to `100`.
5. Round the result.

We use `ROUND(..., 2)` to ensure the output contains exactly two decimal places.
6. Return the required columns.

The output includes:

- `student_id`
- `department_id`
- `percentage`

### Why it works

The solution works because `RANK()` exactly matches the ranking definition required by the problem. Students with identical marks receive the same rank, and higher marks receive smaller rank values.

Partitioning by department guarantees that ranking and counting are isolated within each department.

Since every student's percentage is computed directly from the mathematically correct formula, the final result is guaranteed to be correct.

## Python Solution

Although this is a database problem and LeetCode expects SQL, the equivalent logic can be expressed procedurally in Python.

```python
from collections import defaultdict
from typing import List

class Solution:
    def computeRankPercentage(self, students: List[List[int]]) -> List[List[float]]:
        departments = defaultdict(list)

        for student_id, department_id, mark in students:
            departments[department_id].append((student_id, mark))

        result = []

        for department_id, group in departments.items():
            group.sort(key=lambda x: -x[1])

            total_students = len(group)

            rank = 1
            previous_mark = None

            for index, (student_id, mark) in enumerate(group):
                if previous_mark is not None and mark < previous_mark:
                    rank = index + 1

                previous_mark = mark

                if total_students == 1:
                    percentage = 0.0
                else:
                    percentage = round(
                        (rank - 1) * 100 / (total_students - 1),
                        2
                    )

                result.append([
                    student_id,
                    department_id,
                    percentage
                ])

        return result
```

The implementation first groups students by department using a dictionary. This mirrors SQL partitioning behavior.

Inside each department, students are sorted by descending marks. This allows rank computation in a single pass.

The variable `rank` tracks the current rank. When the current mark becomes smaller than the previous mark, the rank updates to the current index plus one. This reproduces SQL `RANK()` semantics exactly.

The percentage is then computed using the formula from the problem statement. A special case handles departments containing only one student to avoid division by zero.

Finally, all rows are collected into the result list.

## Go Solution

```go
package main

import (
	"math"
	"sort"
)

type Student struct {
	studentID int
	mark      int
}

type Result struct {
	studentID   int
	departmentID int
	percentage  float64
}

func computeRankPercentage(students [][]int) []Result {
	departments := make(map[int][]Student)

	for _, student := range students {
		studentID := student[0]
		departmentID := student[1]
		mark := student[2]

		departments[departmentID] = append(
			departments[departmentID],
			Student{studentID, mark},
		)
	}

	results := []Result{}

	for departmentID, group := range departments {
		sort.Slice(group, func(i, j int) bool {
			return group[i].mark > group[j].mark
		})

		totalStudents := len(group)

		rank := 1
		previousMark := -1

		for index, student := range group {
			if previousMark != -1 && student.mark < previousMark {
				rank = index + 1
			}

			previousMark = student.mark

			var percentage float64

			if totalStudents == 1 {
				percentage = 0.0
			} else {
				percentage = float64(rank-1) * 100.0 / float64(totalStudents-1)
				percentage = math.Round(percentage*100) / 100
			}

			results = append(results, Result{
				studentID:   student.studentID,
				departmentID: departmentID,
				percentage:  percentage,
			})
		}
	}

	return results
}
```

The Go implementation follows the same logic as the Python version.

The main difference is explicit struct usage for clarity and type safety. Go also requires manual floating-point rounding using `math.Round`.

Sorting is implemented with `sort.Slice`, and maps are used for grouping students by department.

## SQL Solution

```sql
SELECT
    student_id,
    department_id,
    ROUND(
        (student_rank - 1) * 100.0 / (department_count - 1),
        2
    ) AS percentage
FROM (
    SELECT
        student_id,
        department_id,
        RANK() OVER (
            PARTITION BY department_id
            ORDER BY mark DESC
        ) AS student_rank,
        COUNT(*) OVER (
            PARTITION BY department_id
        ) AS department_count
    FROM Students
) AS ranked_students;
```

This solution uses two window functions.

`RANK()` computes ranking within each department, while `COUNT()` computes the department size.

The outer query applies the percentage formula and rounds the final result.

## Edge Cases

One important edge case is a department containing only one student. The denominator in the percentage formula becomes zero, which would normally cause a division error. The implementation explicitly checks for this situation and assigns `0.0` as the percentage.

Another important case occurs when multiple students share identical marks. A naive implementation using sequential numbering would incorrectly assign different ranks. The solution instead updates the rank only when the mark decreases, reproducing true `RANK()` behavior.

A third edge case involves departments with completely different sizes. Since percentages depend on department size, calculations must remain isolated per department. Partitioning by `department_id` guarantees that students are never compared against students from another department.
