## Solution

## Problem Understanding

The problem gives us an array called `cookies`, where each element represents a bag containing some number of cookies. We also receive an integer `k`, which represents the number of children.

Every bag must be assigned to exactly one child. Bags cannot be split apart, so a child either receives the whole bag or none of it. After distributing all bags, each child will have some total number of cookies.

The key quantity we care about is called the **unfairness** of the distribution. This is defined as the maximum number of cookies received by any single child. In other words, if one child gets much more than everyone else, the unfairness becomes large.

Our goal is to minimize this unfairness.

For example, suppose the totals received by the children are:

```
[31, 30]
```

Then the unfairness is:

```
max(31, 30) = 31
```

We want to find the distribution where this maximum value is as small as possible.

The constraints are especially important here:

- `cookies.length <= 8`
- `k <= cookies.length`

The number of bags is very small. This strongly suggests that exponential algorithms such as backtracking or bitmask enumeration are acceptable. A brute force search over all possible assignments is theoretically possible because there are at most 8 bags, but we still need pruning or optimization to avoid unnecessary work.

Several edge cases are important:

- When `k == cookies.length`, every child can receive exactly one bag, so the answer becomes the maximum bag size.
- When `k == 1`, all bags must go to the same child, so the answer becomes the sum of all cookies.
- Large cookie values can create highly unbalanced distributions, making pruning especially important.
- Multiple identical bags can create duplicate states if symmetry is not handled carefully.

## Approaches

### Brute Force

The most straightforward solution is to try every possible assignment of bags to children.

For each bag, we choose one of the `k` children. Since there are `n` bags, this creates `k^n` possible assignments.

For every completed assignment, we compute the total cookies each child received, then calculate the unfairness as the maximum total among them. We keep the minimum unfairness across all assignments.

This approach is guaranteed to find the correct answer because it exhaustively checks every valid distribution. However, it becomes inefficient because the number of possibilities grows exponentially.

With `n = 8` and `k = 8`, the number of assignments becomes:

```
8^8 = 16,777,216
```

That is far too large for a naive exhaustive search.

### Optimal Approach, Backtracking with Pruning

The important observation is that we do not need to explore every possible distribution.

We can build the assignment incrementally using backtracking. At each step, we assign the current bag to one child and continue recursively.

The optimization comes from pruning unnecessary branches early.

The main pruning ideas are:

1. If the current maximum cookies assigned to any child is already greater than or equal to the best answer found so far, there is no reason to continue exploring this branch.
2. If two children currently have the same number of cookies, assigning the current bag to either child produces equivalent states. We only need to explore one of them.
3. Sorting bags in descending order first helps pruning happen earlier because large bags quickly expose bad distributions.

These optimizations dramatically reduce the search space while still guaranteeing correctness.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(k^n * n) | O(n) | Tries every assignment of bags to children |
| Optimal | O(k^n) worst case, much smaller in practice | O(k + n) | Backtracking with aggressive pruning and symmetry elimination |

## Algorithm Walkthrough

1. Sort the `cookies` array in descending order.

We place larger bags first because they affect unfairness more strongly. This helps bad distributions exceed the current best answer earlier, allowing pruning sooner.
2. Create an array called `children`.

This array has length `k`, where `children[i]` stores the current total cookies assigned to the `ith` child.
3. Initialize the answer.

We maintain a variable called `best_unfairness`, initially set to infinity. This stores the smallest unfairness found so far.
4. Start recursive backtracking from the first bag.

At each recursive call, we decide which child should receive the current bag.
5. Try assigning the current bag to every child.

For each child:

- Add the bag value to that child's total.
- Compute the new current unfairness.
- Recursively process the next bag.
- Remove the bag afterward to backtrack.
6. Prune bad branches early.

Before recursing further, if the current unfairness is already greater than or equal to `best_unfairness`, stop exploring this branch.
7. Avoid symmetric states.

Suppose two children currently have the same total cookies. Assigning the current bag to either child creates equivalent states.

To avoid duplicate exploration, maintain a set of totals already tried at this level of recursion.
8. When all bags are assigned, update the answer.

Once we reach the end of the array, compute the unfairness and minimize `best_unfairness`.

### Why it works

The algorithm systematically explores every distinct valid distribution of bags among children. Backtracking guarantees completeness because every possible assignment is considered unless it is provably worse than a previously found solution.

Pruning is safe because if the current unfairness already exceeds the best known answer, adding more bags can only keep it the same or increase it. Therefore, such branches can never produce a better solution.

Symmetry elimination is also safe because states with identical child totals are indistinguishable for future decisions.

## Python Solution

```python
from typing import List

class Solution:
    def distributeCookies(self, cookies: List[int], k: int) -> int:
        cookies.sort(reverse=True)

        children = [0] * k
        best_unfairness = float("inf")

        def backtrack(index: int) -> None:
            nonlocal best_unfairness

            if index == len(cookies):
                best_unfairness = min(best_unfairness, max(children))
                return

            seen = set()

            for child in range(k):
                if children[child] in seen:
                    continue

                seen.add(children[child])

                children[child] += cookies[index]

                current_unfairness = max(children)

                if current_unfairness < best_unfairness:
                    backtrack(index + 1)

                children[child] -= cookies[index]

                if children[child] == 0:
                    break

        backtrack(0)

        return best_unfairness
```

The implementation begins by sorting the cookie bags in descending order. This improves pruning efficiency because larger bags immediately influence the unfairness.

The `children` array tracks the running cookie totals for each child throughout the recursion.

The recursive `backtrack` function processes one bag at a time. The parameter `index` tells us which bag we are currently assigning.

When all bags have been assigned, the algorithm computes the unfairness using `max(children)` and updates the global best answer.

Inside each recursive call, the algorithm attempts to assign the current bag to every child. Before exploring a branch, it checks whether another child with the same current total has already been explored at this level. If so, that branch is skipped because it would produce an equivalent state.

After assigning a bag, the algorithm computes the current unfairness. If it is already greater than or equal to the best answer found so far, recursion stops immediately for that branch.

Finally, the algorithm removes the bag from the child to restore the previous state before exploring the next possibility.

The optimization:

```python
if children[child] == 0:
    break
```

avoids redundant exploration among empty children. If assigning a bag to one empty child did not lead to a better solution, assigning it to another empty child would produce the same state.

## Go Solution

```go
package main

import (
	"math"
	"sort"
)

func distributeCookies(cookies []int, k int) int {
	sort.Slice(cookies, func(i, j int) bool {
		return cookies[i] > cookies[j]
	})

	children := make([]int, k)
	bestUnfairness := math.MaxInt32

	var backtrack func(int)

	backtrack = func(index int) {
		if index == len(cookies) {
			currentMax := 0

			for _, total := range children {
				if total > currentMax {
					currentMax = total
				}
			}

			if currentMax < bestUnfairness {
				bestUnfairness = currentMax
			}

			return
		}

		seen := make(map[int]bool)

		for child := 0; child < k; child++ {
			if seen[children[child]] {
				continue
			}

			seen[children[child]] = true

			children[child] += cookies[index]

			currentMax := 0
			for _, total := range children {
				if total > currentMax {
					currentMax = total
				}
			}

			if currentMax < bestUnfairness {
				backtrack(index + 1)
			}

			children[child] -= cookies[index]

			if children[child] == 0 {
				break
			}
		}
	}

	backtrack(0)

	return bestUnfairness
}
```

The Go implementation follows the same algorithmic structure as the Python solution.

Because Go does not provide a built in `max` function for slices, we manually compute the maximum child total using a loop.

The `seen` structure is implemented using `map[int]bool` to eliminate symmetric states during recursion.

Go slices are reference types, so updates to `children` inside recursive calls affect the shared slice directly. Backtracking is therefore essential to restore previous values after recursion returns.

Integer overflow is not a concern because the maximum total cookies is:

```go
8 * 100000 = 800000
```

which easily fits within Go's `int`.

## Edge Cases

One important edge case occurs when the number of children equals the number of bags. In this scenario, every child can receive exactly one bag. The unfairness therefore becomes the maximum bag size. A buggy implementation might still perform unnecessary recursion or incorrectly combine bags. The current implementation handles this naturally because backtracking explores assignments correctly and pruning quickly terminates redundant states.

Another important case is when there is only one child. Since all bags must go to the same child, the answer is simply the sum of all cookies. Recursive algorithms sometimes fail here because they assume multiple distribution choices exist. In this implementation, the recursion still works correctly because there is only one possible branch at every step.

A third tricky case involves duplicate states caused by equal child totals. For example, if two children both currently have 0 cookies, assigning the current bag to either child creates identical future possibilities. Without symmetry pruning, the algorithm explores redundant branches repeatedly, causing major slowdowns. The `seen` set prevents this duplication by skipping equivalent states.

Another subtle case occurs when one bag is much larger than all others combined. In such situations, the unfairness is forced to be at least that large bag. Poor pruning strategies may still explore many impossible distributions. Sorting the bags in descending order ensures these difficult states are detected early, allowing aggressive pruning.
