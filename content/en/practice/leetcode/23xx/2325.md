## Solution

## Problem Understanding

This problem is asking us to decode a message using a substitution cipher that is defined by a given `key`. The `key` is a string containing lowercase letters and spaces, where the **first occurrence** of each letter determines the mapping from encoded letters to decoded letters. The mapping aligns these letters to the standard English alphabet, `'a'` to `'z'`. Every letter in the `message` is then substituted according to this mapping, and spaces remain unchanged.

The inputs are `key` and `message`. `key` has a length between 26 and 2000 characters and is guaranteed to contain every letter from `'a'` to `'z'` at least once, which ensures that a complete substitution table can always be created. `message` also has a length between 1 and 2000 characters and only contains lowercase letters and spaces.

Edge cases to consider include consecutive spaces in `key`, letters appearing multiple times (we only use the first appearance), and messages with multiple spaces. The constraints make this problem suitable for a linear-time solution because the maximum string length is moderate (2000 characters).

## Approaches

The brute-force approach would involve manually creating a table with 26 slots for letters and iterating over the `key` repeatedly to find the first occurrence of each letter, then iterating over the `message` to replace each character. This approach is unnecessarily complicated because we can track the first occurrence in a single pass.

The optimal approach uses a **hash map** (or dictionary in Python) to record the first occurrence of each character in the `key`. Once the mapping is complete, we iterate over the `message` and substitute each letter according to the mapping. Spaces are handled separately to remain unchanged. This method is efficient because it only requires a single pass through `key` and `message`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(26 * key.length + message.length) | O(26) | Repeated scanning of `key` to find first occurrences |
| Optimal | O(key.length + message.length) | O(26) | Single pass through `key` to build mapping, single pass through `message` to decode |

## Algorithm Walkthrough

1. Initialize an empty dictionary `mapping` to store the mapping from encoded characters to decoded characters.
2. Initialize a variable `next_char` to `'a'` to track which letter of the alphabet to assign next.
3. Iterate over each character `c` in `key`. If `c` is not a space and is not already in `mapping`, assign `mapping[c] = next_char` and increment `next_char` to the next letter.
4. After processing the entire `key`, `mapping` contains a complete substitution table where each letter in `key` maps to a unique letter in the English alphabet.
5. Initialize an empty list `decoded_chars` to build the decoded message.
6. Iterate over each character `c` in `message`. If `c` is a space, append a space to `decoded_chars`. Otherwise, append `mapping[c]`.
7. Join the `decoded_chars` list into a single string and return it as the decoded message.

Why it works: This works because the problem guarantees that `key` contains every letter at least once, so a complete mapping will always be formed. The mapping is built in the order of first appearances, which aligns with the problem statement, ensuring that every character in the `message` is correctly substituted.

## Python Solution

```python
class Solution:
    def decodeMessage(self, key: str, message: str) -> str:
        mapping = {}
        next_char = ord('a')
        
        for c in key:
            if c != ' ' and c not in mapping:
                mapping[c] = chr(next_char)
                next_char += 1
        
        decoded_chars = []
        for c in message:
            if c == ' ':
                decoded_chars.append(' ')
            else:
                decoded_chars.append(mapping[c])
        
        return ''.join(decoded_chars)
```

The Python implementation initializes a mapping dictionary and tracks the next letter to assign using `ord` and `chr`. It processes `key` in one pass to build the mapping, then decodes `message` in a second pass. Spaces are appended directly to preserve formatting.

## Go Solution

```go
func decodeMessage(key string, message string) string {
    mapping := make(map[rune]rune)
    nextChar := 'a'
    
    for _, c := range key {
        if c != ' ' {
            if _, exists := mapping[c]; !exists {
                mapping[c] = nextChar
                nextChar++
            }
        }
    }
    
    decoded := make([]rune, len(message))
    for i, c := range message {
        if c == ' ' {
            decoded[i] = ' '
        } else {
            decoded[i] = mapping[c]
        }
    }
    
    return string(decoded)
}
```

In Go, we use a `map[rune]rune` for the substitution table. `nextChar` is tracked as a rune. We build the mapping and decode `message` similarly to Python, taking care to preallocate the slice for the decoded message.

## Edge Cases

One important edge case is when `key` contains repeated letters with spaces interleaved. Only the first occurrence of each letter matters, so our mapping correctly skips duplicates and spaces. Another edge case is a `message` consisting entirely of spaces, which should remain unchanged; the implementation handles this by directly appending spaces without looking them up. A third edge case is the minimal `key` and `message` lengths; our solution handles this naturally because it iterates over every character in `key` and `message` without relying on their length. All these are guaranteed by the algorithm design.
