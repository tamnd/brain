## Solution

## Problem Understanding

This problem asks us to count the number of valid sequences of dice rolls under two strict constraints. You are given an integer `n`, representing the number of times a fair 6-sided dice is rolled. Each sequence of rolls must satisfy the following:

1. **Adjacent GCD Constraint**: Any two consecutive rolls in the sequence must have a greatest common divisor equal to `1`. This disallows pairs like `(2,4)` or `(3,6)` because their GCD is greater than 1.
2. **Repeat Distance Constraint**: Any repeated value in the sequence must have at least two other rolls in between. Formally, if the value at index `i` is the same as at index `j`, then `abs(i - j) > 2`. This prevents immediate repeats or repeats with just one intervening value, e.g., `(1,2,1)` is invalid.

The input `n` ranges from `1` to `10^4`, so a naive brute-force approach that enumerates all `6^n` sequences is completely infeasible.

The output is the total number of valid sequences modulo `10^9 + 7`. Two sequences are considered distinct if they differ at least at one roll.

Important edge cases include very small sequences (like `n = 1` or `n = 2`) and sequences that attempt invalid repeats or have adjacent numbers with GCD greater than 1.

## Approaches

### Brute Force

A naive brute-force solution would generate all sequences of length `n` using the numbers 1 through 6. For each sequence, we would check the two constraints: whether all adjacent pairs have a GCD of 1 and whether repeats occur at least 3 positions apart. This would give the correct answer, but its complexity is O(6^n), which is completely infeasible for `n` up to 10,000.

### Optimal Approach

The problem can be solved efficiently using **dynamic programming with memoization**. The key insight is that the constraints only depend on the last two dice rolls. Therefore, we can define a state as `(position, last roll, second-to-last roll)` and recursively count the valid sequences that can extend from this state.

1. `position` indicates the current roll index (0-based).
2. `last` is the value of the previous roll.
3. `second_last` is the value of the roll before that.

From this state, we try all valid next rolls (numbers 1 through 6) that satisfy both the GCD and repeat constraints. Using memoization, we avoid recomputation for the same `(position, last, second_last)` combination.

This reduces the complexity drastically because there are at most `n * 7 * 7` unique states (we use 0 as a placeholder for "no previous roll"), and for each state, we check at most 6 possible next rolls.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(6^n) | O(n) | Generate all sequences and check constraints, infeasible for n > 10 |
| Optimal | O(n * 6 * 6 * 6) ≈ O(n) | O(n * 7 * 7) ≈ O(n) | DP with memoization on last two rolls reduces state space drastically |

## Algorithm Walkthrough

1. Define a memoization table `dp[pos][last][second_last]` that stores the number of valid sequences starting at position `pos` with `last` and `second_last` rolls fixed. Initialize it with -1 to indicate uncomputed states.
2. Create a recursive helper function `dfs(pos, last, second_last)`:

- If `pos == n`, return 1 because we have formed a valid sequence of length `n`.
- If `dp[pos][last][second_last]` is not -1, return the memoized value.
3. Iterate through all possible next rolls `next_roll` from 1 to 6:

- Skip `next_roll` if `last` is not 0 and `gcd(last, next_roll) != 1`.
- Skip `next_roll` if `next_roll` equals `last` or `second_last` to satisfy the repeat-distance constraint.
4. Accumulate the count of valid sequences recursively and store it in the memoization table.
5. Return the total count modulo `10^9 + 7`.

**Why it works**: By only tracking the last two rolls, we ensure that all sequences obey both constraints. Memoization ensures each unique state is computed only once. The algorithm explores all valid sequences systematically without enumerating invalid ones.

## Python Solution

```python
from math import gcd
from functools import lru_cache

class Solution:
    def distinctSequences(self, n: int) -> int:
        MOD = 10**9 + 7
        
        @lru_cache(None)
        def dfs(pos: int, last: int, second_last: int) -> int:
            if pos == n:
                return 1
            total = 0
            for next_roll in range(1, 7):
                if last != 0 and gcd(last, next_roll) != 1:
                    continue
                if next_roll == last or next_roll == second_last:
                    continue
                total += dfs(pos + 1, next_roll, last)
                total %= MOD
            return total
        
        return dfs(0, 0, 0)
```

The Python implementation uses `lru_cache` for memoization. The `dfs` function checks both constraints efficiently. The base case ensures that sequences of length `n` are counted exactly once. The modulo operation is applied during accumulation to avoid overflow.

## Go Solution

```go
package main

func gcd(a, b int) int {
    for b != 0 {
        a, b = b, a%b
    }
    return a
}

func distinctSequences(n int) int {
    const MOD = 1_000_000_007
    memo := make([][][]int, n+1)
    for i := range memo {
        memo[i] = make([][]int, 7)
        for j := range memo[i] {
            memo[i][j] = make([]int, 7)
            for k := range memo[i][j] {
                memo[i][j][k] = -1
            }
        }
    }

    var dfs func(pos, last, secondLast int) int
    dfs = func(pos, last, secondLast int) int {
        if pos == n {
            return 1
        }
        if memo[pos][last][secondLast] != -1 {
            return memo[pos][last][secondLast]
        }
        total := 0
        for next := 1; next <= 6; next++ {
            if last != 0 && gcd(last, next) != 1 {
                continue
            }
            if next == last || next == secondLast {
                continue
            }
            total = (total + dfs(pos+1, next, last)) % MOD
        }
        memo[pos][last][secondLast] = total
        return total
    }

    return dfs(0, 0, 0)
}
```

The Go version explicitly manages a 3D slice for memoization instead of `lru_cache`. It follows the same logic as the Python version. The GCD helper function uses the Euclidean algorithm.

## Worked Examples

**Example 1: n = 4**

Start: `pos=0, last=0, second_last=0`

At `pos=0`, we can choose 1 through 6 as first roll. Suppose we pick 1:

`pos=1, last=1, second_last=0`

Next valid rolls must satisfy GCD=1 with last=1 and cannot repeat last or second_last. This yields rolls 2,3,4,5,6. Continue recursively until `pos=4`. Summing all valid sequences gives 184.

**Example 2: n = 2**

`pos=0, last=0, second_last=0` → pick 1-6.

At `pos=1`, last roll is 1, second_last=0 → next roll must have GCD=1 with 1, cannot be 1. Valid choices: 2,3,4,5,6. Count all valid sequences → total 22.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * 6 * 6 * 6) ≈ O(n) | There are n positions and at most 6x6x6 unique states for memoization |
| Space | O(n * 7 * 7) ≈ O(n) | Memoization table or recursion stack stores states for n positions with last two rolls |

The algorithm is efficient because the state space is independent of the exponential 6^n sequences and depends only on last two rolls and position.
