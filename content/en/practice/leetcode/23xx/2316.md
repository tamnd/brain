## Solution

## Problem Understanding

This problem asks us to determine how many pairs of nodes in an **undirected graph** are **unreachable** from each other. In simpler terms, for each pair of nodes `(u, v)`, we want to check if there is no path connecting `u` to `v`. If such a path does not exist, the pair is considered unreachable. The input provides `n`, the number of nodes, and `edges`, a list of undirected connections between nodes. Each edge `[a, b]` indicates that nodes `a` and `b` are directly connected.

The expected output is a single integer representing the total count of all unreachable node pairs. Because the graph is undirected, the pair `(u, v)` is considered the same as `(v, u)`, and no node is paired with itself.

The constraints tell us the graph can be very large: up to `10^5` nodes and `2*10^5` edges. This eliminates any brute-force solution that checks every pair of nodes directly, since that would be O(n²) in time.

Important edge cases include:

- No edges at all: all nodes are isolated, so every pair is unreachable.
- All nodes fully connected: no unreachable pairs.
- Graph with multiple disconnected components of varying sizes: the algorithm must account for pairs across different components.

## Approaches

The **brute-force approach** would be to iterate over every pair of nodes `(i, j)` and check if `i` can reach `j` via a graph traversal like BFS or DFS. This is correct because we directly check reachability. However, the time complexity is O(n² + m) per check, which is too slow for `n` up to `10^5`.

The **key insight** is to realize that nodes in the **same connected component** are all mutually reachable. Therefore, the number of unreachable pairs is determined by pairs of nodes in **different connected components**. If a component has size `c`, then each of its nodes forms unreachable pairs with all nodes outside the component, giving `c * (n - c)` unreachable pairs. Summing across all components and dividing by 2 (because each pair is counted twice) gives the correct total.

We can efficiently find component sizes using **DFS, BFS, or Union-Find**. Union-Find is particularly clean and efficient for this problem because we can quickly determine connected components and their sizes without explicitly traversing the entire graph repeatedly.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n² + m) | O(n + m) | Check reachability for every node pair using DFS/BFS |
| Optimal (Union-Find / Component Sizes) | O(n + m) | O(n) | Compute connected components sizes and calculate unreachable pairs using the formula |

## Algorithm Walkthrough

1. Initialize a **Union-Find** data structure for `n` nodes. Each node starts in its own component.
2. Iterate over the list of edges and **union** the nodes in each edge. This merges connected nodes into the same component.
3. After processing all edges, compute the **size of each connected component**. This can be done by iterating over all nodes and counting how many nodes have the same root.
4. Initialize a variable `total_pairs` to 0. For each component of size `c`, add `c * (n - c)` to `total_pairs`. This counts each node in the component paired with nodes outside the component.
5. Divide `total_pairs` by 2 because each pair is counted twice.
6. Return the result.

**Why it works:** The union-find ensures that all nodes in the same connected component are identified correctly. The formula `c * (n - c)` counts all pairs of nodes where one node is in this component and the other is outside. Dividing by 2 eliminates double-counting. This guarantees all unreachable pairs are counted exactly once.

## Python Solution

```python
from typing import List

class UnionFind:
    def __init__(self, n: int):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.size = [1] * n

    def find(self, x: int) -> int:
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x: int, y: int) -> None:
        root_x, root_y = self.find(x), self.find(y)
        if root_x == root_y:
            return
        if self.rank[root_x] < self.rank[root_y]:
            self.parent[root_x] = root_y
            self.size[root_y] += self.size[root_x]
        else:
            self.parent[root_y] = root_x
            self.size[root_x] += self.size[root_y]
            if self.rank[root_x] == self.rank[root_y]:
                self.rank[root_x] += 1

class Solution:
    def countPairs(self, n: int, edges: List[List[int]]) -> int:
        uf = UnionFind(n)
        for a, b in edges:
            uf.union(a, b)
        
        # Compute component sizes
        root_count = {}
        for i in range(n):
            root = uf.find(i)
            root_count[root] = root_count.get(root, 0) + 1
        
        # Count unreachable pairs
        total_pairs = 0
        for size in root_count.values():
            total_pairs += size * (n - size)
        return total_pairs // 2
```

**Explanation:** We first use a Union-Find structure to merge nodes connected by edges. Then, we calculate the size of each component by counting nodes with the same root. The final unreachable pairs are computed using the sum of `c * (n - c)` over all components divided by two to correct double counting.

## Go Solution

```go
func countPairs(n int, edges [][]int) int64 {
    parent := make([]int, n)
    size := make([]int, n)
    for i := 0; i < n; i++ {
        parent[i] = i
        size[i] = 1
    }

    var find func(int) int
    find = func(x int) int {
        if parent[x] != x {
            parent[x] = find(parent[x])
        }
        return parent[x]
    }

    union := func(x, y int) {
        rootX, rootY := find(x), find(y)
        if rootX == rootY {
            return
        }
        if size[rootX] < size[rootY] {
            parent[rootX] = rootY
            size[rootY] += size[rootX]
        } else {
            parent[rootY] = rootX
            size[rootX] += size[rootY]
        }
    }

    for _, edge := range edges {
        union(edge[0], edge[1])
    }

    rootCount := make(map[int]int)
    for i := 0; i < n; i++ {
        root := find(i)
        rootCount[root]++
    }

    var totalPairs int64 = 0
    for _, sz := range rootCount {
        totalPairs += int64(sz) * int64(n - sz)
    }

    return totalPairs / 2
}
```

**Go Notes:** We use slices for `parent` and `size` and a map for component counts. Go requires explicit type casting to `int64` to prevent integer overflow during multiplication, which is necessary given the constraints.

### Example 1: `n = 3, edges = [[0,1],[0,2],[1,2]]`

After union-find:

- All nodes are connected in one component of size 3.
- Unreachable pairs: `3 * (3-3)/2 = 0`.

### Example 2: `n = 7, edges = [[0,2],[0,5],[2,4],[1,6],[5,4]]`

After union-find:

- Component 1: {0,2,4,5} size 4
- Component 2: {1,6} size 2
- Component 3: {3} size 1

Pairs counted:

- Component 1: 4*(7-4)=12
- Component 2: 2*(7-2)=10
- Component 3: 1*(7-1)=6

Total = 28 / 2 = 14 pairs
