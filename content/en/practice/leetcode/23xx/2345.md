## Solution

## Problem Understanding

Each mountain is defined by a peak `(x, y)`. Because the mountain has slopes `+1` and `-1`, it forms an isosceles right triangle whose base touches the `x-axis`.

A mountain centered at `x` with height `y` covers the interval:

$$[x - y,\ x + y]$$

on the `x-axis`.

A peak `(x1, y1)` lies inside or on the border of another mountain `(x2, y2)` if and only if:

$$|x1 - x2| + y1 \le y2$$

Using the interval representation, mountain `A` completely contains mountain `B` if:

$$L_A \le L_B \quad \text{and} \quad R_B \le R_A$$

where:

$$L = x - y,\quad R = x + y$$

The problem asks us to count how many mountains are visible. A mountain is visible only if its peak is not inside or on the border of another mountain.

The input size can be as large as `10^5`, which immediately tells us that any `O(n^2)` solution will be too slow. We need something closer to `O(n log n)`.

There are several important edge cases:

- Completely overlapping mountains, such as `[[1,3],[1,3]]`
- Mountains that share the same left boundary
- Mountains nested inside larger mountains
- Mountains touching exactly on the border, because border inclusion still counts as hidden
- Multiple identical intervals, which require special handling because they hide each other

## Approaches

### Brute Force

The most direct solution is to compare every mountain against every other mountain.

For each mountain `i`, we check whether its peak lies inside or on another mountain `j`. Using the condition:

$$|x_i - x_j| + y_i \le y_j$$

if this is true for any other mountain, then mountain `i` is not visible.

This approach is correct because it explicitly checks the visibility condition for every pair of mountains.

However, with up to `10^5` mountains, this requires:

$$O(n^2)$$

comparisons, which is far too slow.

### Optimal Approach

The key observation is that each mountain can be represented as an interval:

$$[x-y,\ x+y]$$

A mountain becomes invisible if another mountain completely covers its interval.

This transforms the problem into an interval containment problem.

We sort mountains by:

1. Increasing left boundary
2. Decreasing right boundary for ties

The second rule is extremely important. If two mountains share the same left boundary, the larger one must appear first so that smaller contained mountains are correctly identified.

After sorting, we scan from left to right while maintaining the largest right endpoint seen so far.

- If the current mountain's right endpoint is less than or equal to the maximum right endpoint already seen, then it is hidden.
- Otherwise, it is visible.

We also must carefully handle duplicate mountains. If the exact same interval appears multiple times, none of them are visible because each lies on the border of another identical mountain.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Compare every mountain against every other mountain |
| Optimal | O(n log n) | O(n) | Sort intervals and scan using maximum right boundary |

## Algorithm Walkthrough

1. Convert each mountain into an interval.

For a peak `(x, y)`:

$$L = x - y,\quad R = x + y$$

This interval fully describes the mountain's coverage.
2. Sort the intervals.

Sort by:

- Left boundary ascending
- Right boundary descending for ties

This ensures that when two mountains start at the same point, the larger mountain comes first.
3. Detect duplicate intervals.

If the exact same interval appears more than once, none of those mountains are visible. We can count interval frequencies using a hash map.
4. Scan through the sorted intervals.

Maintain:

- `max_right`, the farthest right endpoint seen so far

For each interval:

- If `right <= max_right`, the mountain is contained inside another mountain and is invisible.
- Otherwise, it is potentially visible.
5. Exclude duplicates.

Even if an interval is not contained by a larger interval, it is still invisible if it appears multiple times.
6. Count visible mountains.

Increment the answer only for intervals that:

- Extend beyond `max_right`
- Appear exactly once

### Why it works

After sorting by increasing left boundary, every previously processed mountain starts no later than the current one. Therefore, if a previous mountain also has a right boundary greater than or equal to the current mountain's right boundary, then the previous mountain completely contains the current mountain.

Sorting equal left boundaries by decreasing right boundary guarantees that larger containing mountains appear before smaller contained ones.

Duplicate intervals are excluded because identical mountains contain each other exactly on their borders.

## Python Solution

```python
from typing import List
from collections import Counter

class Solution:
    def visibleMountains(self, peaks: List[List[int]]) -> int:
        intervals = []
        
        for x, y in peaks:
            left = x - y
            right = x + y
            intervals.append((left, right))
        
        frequency = Counter(intervals)
        
        intervals.sort(key=lambda interval: (interval[0], -interval[1]))
        
        visible_count = 0
        max_right = float('-inf')
        
        for left, right in intervals:
            if right > max_right:
                if frequency[(left, right)] == 1:
                    visible_count += 1
                
                max_right = right
        
        return visible_count
```

The implementation begins by converting every mountain into its interval representation. This transformation simplifies the geometric visibility condition into interval containment.

A `Counter` tracks how many times each interval appears. This is necessary because duplicate mountains are never visible.

The intervals are then sorted by increasing left endpoint and decreasing right endpoint. This ordering ensures correct containment detection during the linear scan.

During the scan, `max_right` stores the largest right endpoint encountered so far. If the current interval extends farther right than any previous interval, it is not contained inside another mountain. At that point, we check whether the interval is unique before counting it as visible.

If the interval does not extend beyond `max_right`, then it must already be covered by a previously processed mountain.

## Go Solution

```go
package main

import (
	"sort"
)

func visibleMountains(peaks [][]int) int {
	type Interval struct {
		left  int
		right int
	}

	intervals := make([]Interval, 0, len(peaks))
	frequency := make(map[Interval]int)

	for _, peak := range peaks {
		x := peak[0]
		y := peak[1]

		interval := Interval{
			left:  x - y,
			right: x + y,
		}

		intervals = append(intervals, interval)
		frequency[interval]++
	}

	sort.Slice(intervals, func(i, j int) bool {
		if intervals[i].left == intervals[j].left {
			return intervals[i].right > intervals[j].right
		}
		return intervals[i].left < intervals[j].left
	})

	visibleCount := 0
	maxRight := -1

	for _, interval := range intervals {
		if interval.right > maxRight {
			if frequency[interval] == 1 {
				visibleCount++
			}

			maxRight = interval.right
		}
	}

	return visibleCount
}
```

The Go implementation follows the same logic as the Python version. A custom `Interval` struct is used so intervals can serve as map keys for frequency counting.

Go slices are used for interval storage, and `sort.Slice` handles the custom sorting order.

Since all coordinates are positive and bounded by `10^5`, integer overflow is not a concern for standard Go `int` values.

## Edge Cases

### Duplicate Mountains

When two mountains have exactly the same peak, they completely overlap. Since the problem states that lying on another mountain's border still counts as hidden, neither mountain is visible.

This can easily cause bugs if duplicates are not explicitly detected. The implementation handles this using a frequency map and only counts intervals that appear exactly once.

### Same Left Boundary

Consider mountains with intervals:

```
[0,10]
[0,7]
```

If the smaller interval were processed first, it might incorrectly be considered visible before the larger interval appears.

Sorting by decreasing right boundary when left boundaries are equal ensures the larger mountain always appears first.

### Border Containment

A mountain is invisible even if its peak lies exactly on another mountain's edge.

For example:

```
[0,4]
[2,4]
```

The second interval is fully contained because its right boundary equals the larger mountain's right boundary.

Using the condition:

```
right <= max_right
```

correctly handles border containment cases.
