[LeetCode Problem 2313](https://leetcode.com/problems/minimum-flips-in-binary-tree-to-get-result/)

**Difficulty:** 🔴 Hard  
**Topics:** Dynamic Programming, Tree, Depth-First Search, Binary Tree

## Solution

## Problem Understanding

This problem gives us a binary tree that represents a boolean expression. Each node in the tree either represents a boolean value or a boolean operation.

Leaf nodes contain either:

- `0`, representing `false`
- `1`, representing `true`

Internal nodes represent operators:

- `2` = OR
- `3` = AND
- `4` = XOR
- `5` = NOT

The tree evaluates recursively. For example:

- An OR node evaluates to the OR of its two children.
- An AND node evaluates to the AND of its two children.
- A NOT node evaluates to the negation of its single child.

We are also given a target boolean value called `result`. Our task is to determine the minimum number of leaf flips required so that the entire expression evaluates to this target value.

A flip operation only applies to leaf nodes:

- `0` becomes `1`
- `1` becomes `0`

We need to compute the minimum number of flips necessary to force the root expression to evaluate to the desired result.

The constraints are extremely important here:

- Up to `10^5` nodes
- Tree recursion structure
- Binary boolean operators

A brute force solution that tries all combinations of flips is impossible because each leaf has two possible states, producing exponential complexity.

The key observation is that every subtree can independently answer two questions:

- What is the minimum cost to make this subtree evaluate to `false`?
- What is the minimum cost to make this subtree evaluate to `true`?

This naturally leads to a dynamic programming solution on trees.

Several edge cases are important:

- A tree with only one node
- NOT nodes with only one child
- Expressions already matching the target
- Expressions where multiple different flip combinations exist
- Deep trees that require efficient traversal
- XOR behavior, where equal inputs produce `false` and different inputs produce `true`

The problem guarantees that a solution always exists.

## Approaches

### Brute Force Approach

The brute force idea is to enumerate every possible subset of leaf nodes to flip.

Suppose the tree has `k` leaves. Each leaf can either:

- remain unchanged
- be flipped

That creates `2^k` possible configurations.

For every configuration:

1. Apply the flips
2. Recompute the entire tree evaluation
3. Check whether the result matches the target
4. Track the minimum number of flips

This approach is correct because it examines every possible valid state of the leaves. Eventually it will find the optimal one.

However, it is far too slow. In the worst case, the tree may contain nearly `10^5` leaves. Exponential complexity is completely infeasible.

### Optimal Dynamic Programming on Trees

The key insight is that each subtree only needs to know two values:

- minimum flips to make the subtree evaluate to `false`
- minimum flips to make the subtree evaluate to `true`

Once we know these values for the children, we can compute the values for the parent based on the operator type.

For example:

- For an OR node to become `true`, at least one child must be `true`
- For an AND node to become `true`, both children must be `true`
- For XOR, the children must differ

This transforms the problem into a bottom up tree DP.

We process the tree with DFS and return a pair:

```
(false_cost, true_cost)
```

for every node.

Since every node is processed once and each transition is constant time, the total complexity becomes linear.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^L × N) | O(L) | Tries every leaf flip combination |
| Optimal | O(N) | O(H) | Tree DP using DFS |

Here:

- `N` = number of nodes
- `L` = number of leaves
- `H` = tree height

## Algorithm Walkthrough

1. Define a DFS function that returns two values for every subtree:

- minimum flips to make subtree evaluate to `false`
- minimum flips to make subtree evaluate to `true`
2. Handle leaf nodes first.

If the leaf value is:

- `0`

- cost for `false` = `0`
- cost for `true` = `1`
- `1`

- cost for `true` = `0`
- cost for `false` = `1`

This works because flipping a leaf costs exactly one operation.
3. Recursively compute DP values for children.

For every internal node, compute:

```
left_false, left_true
right_false, right_true
```
4. Process OR nodes.

OR becomes `true` if at least one child is `true`.

OR becomes `false` only if both children are `false`.

Therefore:

```
false_cost = left_false + right_false

true_cost =
    min(
        left_true + right_true,
        left_true + right_false,
        left_false + right_true
    )
```
5. Process AND nodes.

AND becomes `true` only if both children are `true`.

AND becomes `false` if at least one child is `false`.

Therefore:

```
true_cost = left_true + right_true

false_cost =
    min(
        left_false + right_false,
        left_false + right_true,
        left_true + right_false
    )
```
6. Process XOR nodes.

XOR becomes `true` when children differ.

XOR becomes `false` when children are equal.

Therefore:

```
false_cost =
    min(
        left_false + right_false,
        left_true + right_true
    )

true_cost =
    min(
        left_false + right_true,
        left_true + right_false
    )
```
7. Process NOT nodes.

NOT simply swaps the meanings.

If the child can become:

```
child_false, child_true
```

then:

```
false_cost = child_true
true_cost = child_false
```
8. After DFS finishes, return:

- `true_cost` if target is `true`
- `false_cost` if target is `false`

### Why it works

For every subtree, the algorithm computes the optimal cost for both possible boolean outcomes. Since boolean operators combine child results independently, the optimal solution for a parent must be composed of optimal solutions for its children.

This creates an optimal substructure property, which makes dynamic programming valid. Because every operator transition exactly models boolean logic, the computed costs are guaranteed to be correct.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional, Tuple
from math import inf

class Solution:
    def minimumFlips(self, root: Optional[TreeNode], result: bool) -> int:
        
        def dfs(node: TreeNode) -> Tuple[int, int]:
            # Returns:
            # (cost_to_make_false, cost_to_make_true)
            
            # Leaf node
            if node.val == 0:
                return (0, 1)
            
            if node.val == 1:
                return (1, 0)
            
            # NOT node
            if node.val == 5:
                child = node.left if node.left else node.right
                child_false, child_true = dfs(child)
                return (child_true, child_false)
            
            left_false, left_true = dfs(node.left)
            right_false, right_true = dfs(node.right)
            
            # OR
            if node.val == 2:
                false_cost = left_false + right_false
                
                true_cost = min(
                    left_true + right_true,
                    left_true + right_false,
                    left_false + right_true
                )
                
                return (false_cost, true_cost)
            
            # AND
            if node.val == 3:
                true_cost = left_true + right_true
                
                false_cost = min(
                    left_false + right_false,
                    left_false + right_true,
                    left_true + right_false
                )
                
                return (false_cost, true_cost)
            
            # XOR
            false_cost = min(
                left_false + right_false,
                left_true + right_true
            )
            
            true_cost = min(
                left_false + right_true,
                left_true + right_false
            )
            
            return (false_cost, true_cost)
        
        false_cost, true_cost = dfs(root)
        return true_cost if result else false_cost
```

The implementation follows the tree DP strategy directly.

The `dfs()` function returns a tuple where:

- index `0` represents minimum flips for `false`
- index `1` represents minimum flips for `true`

Leaf nodes are the base case because their values are already fixed unless flipped.

For internal nodes, the solution recursively gathers the optimal costs from child subtrees. Each operator then combines those child costs according to boolean logic.

The OR, AND, and XOR transitions enumerate all valid child state combinations that produce the desired parent result. Since only a constant number of combinations exist, each node is processed in constant time.

The final answer comes from selecting either the `false` or `true` cost at the root based on the target result.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

func minimumFlips(root *TreeNode, result bool) int {

    var dfs func(node *TreeNode) (int, int)

    dfs = func(node *TreeNode) (int, int) {
        // Returns:
        // (costToFalse, costToTrue)

        // Leaf nodes
        if node.Val == 0 {
            return 0, 1
        }

        if node.Val == 1 {
            return 1, 0
        }

        // NOT node
        if node.Val == 5 {
            child := node.Left
            if child == nil {
                child = node.Right
            }

            childFalse, childTrue := dfs(child)
            return childTrue, childFalse
        }

        leftFalse, leftTrue := dfs(node.Left)
        rightFalse, rightTrue := dfs(node.Right)

        // OR
        if node.Val == 2 {
            falseCost := leftFalse + rightFalse

            trueCost := min(
                leftTrue + rightTrue,
                min(
                    leftTrue + rightFalse,
                    leftFalse + rightTrue,
                ),
            )

            return falseCost, trueCost
        }

        // AND
        if node.Val == 3 {
            trueCost := leftTrue + rightTrue

            falseCost := min(
                leftFalse + rightFalse,
                min(
                    leftFalse + rightTrue,
                    leftTrue + rightFalse,
                ),
            )

            return falseCost, trueCost
        }

        // XOR
        falseCost := min(
            leftFalse + rightFalse,
            leftTrue + rightTrue,
        )

        trueCost := min(
            leftFalse + rightTrue,
            leftTrue + rightFalse,
        )

        return falseCost, trueCost
    }

    falseCost, trueCost := dfs(root)

    if result {
        return trueCost
    }

    return falseCost
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

The Go implementation mirrors the Python logic almost exactly.

One important difference is handling `nil` children for NOT nodes. Since NOT nodes only have one child, the code explicitly checks whether the left child exists before falling back to the right child.

Go also requires a helper `min()` function because the language does not provide a built in integer minimum utility.

Since the maximum number of flips is at most the number of leaves, integer overflow is not a concern.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(N) | Every node is processed exactly once |
| Space | O(H) | DFS recursion stack height |

The DFS visits every node one time and performs only constant work at each node. Each operator computes a small fixed number of transitions.

The auxiliary space comes entirely from recursion depth. In the worst case of a completely skewed tree, the height can become `O(N)`. For balanced trees, it becomes `O(log N)`.

## Edge Cases

### Single Node Tree

A tree may contain only one leaf node. This is the smallest valid input and can easily expose incorrect base case handling.

For example:

```
root = [0]
result = false
```

The answer should be `0` because the tree already evaluates correctly.

The implementation handles this directly in the leaf node base case by returning:

```
(0,1)
```

for a `0` leaf.

### NOT Nodes With Only One Child

NOT nodes are different from all other operators because they are unary operators. A naive implementation may incorrectly assume both children exist and attempt to recurse into `None`.

The solution explicitly checks which child exists:

```
child = node.left if node.left else node.right
```

This guarantees correct traversal.

### Deep Skewed Trees

The tree may become highly unbalanced, producing recursion depth close to `10^5`.

Algorithmically, the solution still remains linear because every node is processed once. However, Python recursion depth may require increasing recursion limits in some environments.

LeetCode typically accepts recursive DFS solutions for this problem, but an iterative postorder traversal could also be implemented if necessary.

### Multiple Optimal Solutions

Some expressions may have several different ways to achieve the target with the same minimum number of flips.

For example:

```
1 XOR 1
```

To make this expression evaluate to `true`, either leaf can be flipped.

The DP solution naturally handles this by taking the minimum across all valid state transitions. It does not depend on which specific leaves are chosen.
