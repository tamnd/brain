## Solution

## Problem Understanding

This problem asks us to count the number of non-empty contiguous subarrays of a given array `nums` such that the **score** of each subarray is strictly less than a given integer `k`. The score is defined as the sum of the subarray multiplied by its length.

In simpler terms, for each subarray `nums[i..j]`, compute `score = sum(nums[i..j]) * (j - i + 1)`. We need to count how many of these scores are strictly less than `k`.

The input array `nums` consists of positive integers, with up to `10^5` elements. Each element can be as large as `10^5`, and `k` can be very large, up to `10^15`. This tells us that brute-force enumeration of all subarrays, which would be O(n²) or O(n³) depending on implementation, will be too slow. Efficient handling of sums and comparisons is required.

Important edge cases include:

- Subarrays of length 1, which are the simplest but must still satisfy the score condition.
- Subarrays where the sum is very large, possibly exceeding `k` after multiplication.
- Arrays where all elements are equal or arrays that are strictly increasing or decreasing, which may affect how sliding window techniques behave.

## Approaches

**Brute-Force Approach**

The naive solution would iterate over all possible subarrays of `nums`, calculate the sum and length of each, compute the score, and check if it is less than `k`. This is correct but inefficient. For an array of length `n`, there are `n*(n+1)/2` subarrays, and computing the sum for each can be O(n), leading to O(n³) in the worst case.

**Optimized Approach**

The key observation is that all elements are positive. This allows us to use a **sliding window** approach. For a window `nums[i..j]`, increasing `j` always increases the sum and the length. Hence, if the score exceeds `k`, extending the window further cannot decrease the score, so we can shrink the window from the left. This creates a two-pointer method where we efficiently find all valid subarrays ending at each position.

We can maintain a running sum of the current window and expand or shrink the window according to the score condition `(window_sum * window_length < k)`. Every time the window is valid, we can count all subarrays ending at the current `right` pointer efficiently using the window length.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Compute every subarray sum and check score |
| Sliding Window | O(n) | O(1) | Two-pointer expansion and shrinking, counting subarrays efficiently |

## Algorithm Walkthrough

1. Initialize two pointers `left = 0` and `right = 0` to represent the current window, a variable `window_sum = 0` to track the sum of the window, and `count = 0` to accumulate valid subarrays.
2. Iterate `right` from `0` to `n-1`. Add `nums[right]` to `window_sum`.
3. While the current window's score `(window_sum * (right - left + 1))` is greater than or equal to `k`, shrink the window by incrementing `left` and subtracting `nums[left]` from `window_sum`.
4. Once the window is valid, all subarrays ending at `right` and starting anywhere from `left` to `right` are valid. Add `(right - left + 1)` to `count`.
5. Continue until `right` reaches the end of the array.
6. Return `count`.

**Why it works**: The invariant is that `nums[left..right]` always represents the largest valid window ending at `right`. All smaller subarrays within it are also valid because the elements are positive, ensuring any subarray sum decreases when shrinking from the left.

## Python Solution

```python
from typing import List

class Solution:
    def countSubarrays(self, nums: List[int], k: int) -> int:
        n = len(nums)
        left = 0
        window_sum = 0
        count = 0
        
        for right in range(n):
            window_sum += nums[right]
            
            while left <= right and window_sum * (right - left + 1) >= k:
                window_sum -= nums[left]
                left += 1
            
            count += right - left + 1
        
        return count
```

The implementation maintains a running sum of the current window. The while loop ensures the score condition is met by shrinking from the left when necessary. Each time the window is valid, we add the number of valid subarrays ending at the current `right` pointer to `count`.

## Go Solution

```go
func countSubarrays(nums []int, k int64) int64 {
    var left int
    var windowSum int64
    var count int64
    
    for right := 0; right < len(nums); right++ {
        windowSum += int64(nums[right])
        
        for left <= right && windowSum*int64(right-left+1) >= k {
            windowSum -= int64(nums[left])
            left++
        }
        
        count += int64(right - left + 1)
    }
    
    return count
}
```

In Go, we explicitly convert integers to `int64` to handle potential overflows, as `nums[i]` and `k` can be very large. The algorithm logic mirrors the Python solution.

## Edge Cases

**Case 1: Single-element array**

An array with only one element is the smallest non-empty array possible. The algorithm must correctly count it if its score is less than `k`, or ignore it otherwise. This is handled by initializing the window pointers properly and checking the score once per element.

**Case 2: Very large `k` or large elements**

With `nums[i]` up to 10⁵ and `k` up to 10¹⁵, the product can easily overflow 32-bit integers. In Python, this is handled automatically; in Go, explicit `int64` types are required to avoid overflow.
