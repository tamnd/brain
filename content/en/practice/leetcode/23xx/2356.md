## Solution

## Problem Understanding

The problem provides a database table named `Teacher`. Each row represents a relationship between a teacher, a subject, and a department. Specifically, a row tells us that a teacher teaches a particular subject within a particular department.

The columns are:

| Column | Meaning |
| --- | --- |
| `teacher_id` | The unique identifier of a teacher |
| `subject_id` | The identifier of a subject |
| `dept_id` | The identifier of a department |

The table has a composite primary key of `(subject_id, dept_id)`, which guarantees that the same subject cannot appear multiple times within the same department. However, the same teacher may teach the same subject in multiple departments.

The task is to determine how many unique subjects each teacher teaches. The key word is "unique". If a teacher teaches the same subject in multiple departments, that subject should only be counted once for that teacher.

For example, if teacher `1` teaches subject `2` in both department `3` and department `4`, subject `2` should contribute only one count toward that teacher's total.

The expected output is a table with:

| Column | Meaning |
| --- | --- |
| `teacher_id` | The teacher identifier |
| `cnt` | Number of distinct subjects taught by that teacher |

The result can be returned in any order, which means sorting is not required.

An important detail is that the same teacher may appear many times in the table. A naive solution that simply counts rows per teacher would produce incorrect results because duplicate subjects across departments would be counted multiple times.

The problem guarantees that the table structure is valid and that each row represents a legitimate teaching assignment. Since this is labeled as an Easy database problem, the intended solution relies on SQL aggregation and distinct counting.

Important edge cases include:

- A teacher teaching the same subject in multiple departments
- A teacher teaching only one subject
- Multiple teachers with different numbers of subjects
- A table containing only one row
- Teachers appearing many times due to department variations

The main challenge is ensuring that duplicate subjects for the same teacher are counted only once.

## Approaches

### Brute Force Approach

A brute force strategy would manually group all rows by teacher and then inspect every subject taught by that teacher one by one.

Conceptually, this approach works like this:

1. Iterate through every row in the table.
2. For each teacher, store all encountered subjects in a list.
3. Before inserting a subject into the list, scan the existing list to check whether it already exists.
4. After processing all rows, count the number of stored subjects for each teacher.

This approach is correct because it explicitly prevents duplicate subject entries for each teacher.

However, this becomes inefficient when a teacher teaches many subjects. Every insertion may require scanning an existing list, which can lead to quadratic behavior in the worst case.

### Optimal Approach

The key observation is that SQL already provides built in support for distinct counting through `COUNT(DISTINCT column)`.

Instead of manually removing duplicates, we can:

1. Group rows by `teacher_id`
2. Count the distinct `subject_id` values inside each group

This works perfectly because the problem only cares about unique subjects per teacher, regardless of department.

Using SQL aggregation allows the database engine to efficiently handle grouping and duplicate elimination internally.

## Algorithm Walkthrough

### Optimal SQL Algorithm

1. Start by grouping all rows using `teacher_id`.

This ensures that all records belonging to the same teacher are processed together.
2. Inside each teacher group, examine the `subject_id` values.

A teacher may teach the same subject in multiple departments, so duplicates may exist.
3. Use `COUNT(DISTINCT subject_id)`.

The `DISTINCT` keyword removes duplicate subject identifiers before counting them.
4. Return the grouped result.

Each row in the output contains:

- The teacher identifier
- The number of unique subjects they teach

### Why it works

The correctness comes from the fact that `DISTINCT` guarantees that each subject is counted only once within a teacher's group. Since the grouping is performed by `teacher_id`, each teacher's subjects are isolated from all others. Therefore, the final count exactly matches the number of unique subjects taught by each teacher.

## Python Solution

```python
# Write your MySQL query statement below

SELECT
    teacher_id,
    COUNT(DISTINCT subject_id) AS cnt
FROM Teacher
GROUP BY teacher_id;
```

This solution uses SQL aggregation directly.

The `GROUP BY teacher_id` clause collects all rows belonging to the same teacher into a single group.

The `COUNT(DISTINCT subject_id)` expression removes duplicate subject identifiers before counting them. This is necessary because the same teacher may teach the same subject in multiple departments.

The alias `cnt` matches the required output format from the problem statement.

Because the problem allows returning results in any order, no `ORDER BY` clause is needed.

## Edge Cases

### A teacher teaches the same subject in multiple departments

This is the most important edge case in the problem. A naive solution using `COUNT(*)` would incorrectly count each row separately. For example, if a teacher teaches subject `2` in three departments, the answer should still count that subject only once.

The implementation handles this correctly using `COUNT(DISTINCT subject_id)`.

### A teacher teaches only one subject

A teacher may appear only once in the table or may teach only one unique subject. The solution still works because grouping creates a single group and the distinct count becomes `1`.

### Multiple teachers teach the same subject

Different teachers may teach the same subject ID. This should not affect the counts because counting is performed independently inside each `teacher_id` group.

The `GROUP BY teacher_id` clause guarantees complete separation between teachers during aggregation.
