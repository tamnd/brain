## Solution

## Problem Understanding

The problem gives us a strictly increasing array `nums` and a positive integer `diff`. We need to count how many triplets `(i, j, k)` satisfy all of the following conditions:

- `i < j < k`
- `nums[j] - nums[i] == diff`
- `nums[k] - nums[j] == diff`

In simpler terms, we are looking for three numbers in the array that form an arithmetic progression with common difference `diff`.

If we rewrite the conditions mathematically, then for some value `x`:

- `x` exists in the array
- `x + diff` exists in the array
- `x + 2 * diff` exists in the array

Because the array is strictly increasing, if these three values exist, their indices will automatically satisfy `i < j < k`.

The input consists of:

- `nums`, a sorted array with unique integers
- `diff`, the required difference between consecutive elements in the triplet

The output is a single integer representing how many arithmetic triplets exist.

The constraints are small:

- `nums.length <= 200`
- `nums[i] <= 200`

This means even cubic solutions would technically pass. However, the problem is designed to encourage recognizing a more efficient pattern using hashing or set lookup.

A very important guarantee is that `nums` is strictly increasing. This simplifies the problem significantly because we never need to worry about duplicate values or index ordering conflicts. If three required values exist, their indices are automatically valid.

Some important edge cases include:

- Arrays with no valid triplets
- Arrays where only one triplet exists
- Cases where multiple overlapping triplets exist
- Very small arrays of length `3`
- Large `diff` values that make triplets impossible

For example:

```
nums = [1, 2, 3], diff = 5
```

No triplet can exist because the required differences are too large.

Another important case is overlapping triplets:

```
nums = [1, 3, 5, 7, 9], diff = 2
```

This contains multiple valid triplets:

- `(1, 3, 5)`
- `(3, 5, 7)`
- `(5, 7, 9)`

Our algorithm must count all of them correctly.

## Approaches

### Brute Force Approach

The most direct solution is to try every possible triplet `(i, j, k)`.

We use three nested loops:

- The first loop chooses `i`
- The second loop chooses `j`
- The third loop chooses `k`

For every combination, we check whether:

```
nums[j] - nums[i] == diff
nums[k] - nums[j] == diff
```

If both conditions are true, we increment the answer.

This works because it explicitly examines every possible triplet and validates the arithmetic conditions directly.

However, the time complexity is `O(n^3)` because we test every triplet combination.

Even though the constraints are small enough for this to pass, it is inefficient and unnecessary.

### Optimal Approach

The key observation is that we do not actually need to search for indices explicitly.

Because the array is strictly increasing:

- if `x`
- `x + diff`
- `x + 2 * diff`

all exist in the array, then their indices must naturally satisfy the required ordering.

This transforms the problem into a fast membership lookup problem.

We can place all numbers into a hash set for `O(1)` average lookup time.

Then for every number `x` in `nums`, we simply check whether:

```
x + diff
x + 2 * diff
```

exist in the set.

If both exist, then we found one arithmetic triplet.

This reduces the time complexity to `O(n)`.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Checks every possible triplet explicitly |
| Optimal | O(n) | O(n) | Uses a hash set for constant-time lookups |

## Algorithm Walkthrough

### Optimal Hash Set Algorithm

1. Create a hash set containing every number in `nums`.

This allows constant-time membership checks. Instead of searching the array repeatedly, we can instantly determine whether a required number exists.
2. Initialize a variable `count = 0`.

This variable stores the total number of arithmetic triplets found.
3. Iterate through every number `num` in `nums`.

Treat `num` as the first element of a possible arithmetic triplet.
4. For the current number, compute:

```
second = num + diff
third = num + 2 * diff
```

These are the exact values required to complete the arithmetic progression.
5. Check whether both `second` and `third` exist in the hash set.

If both values exist, then we have found a valid arithmetic triplet.
6. Increment `count` whenever a valid triplet is found.
7. After processing all numbers, return `count`.

### Why it works

The algorithm works because the array is strictly increasing.

If `num`, `num + diff`, and `num + 2 * diff` all exist in the array, then their indices must appear in increasing order automatically.

Therefore, checking value existence is sufficient to guarantee a valid arithmetic triplet.

The hash set ensures that each existence check is efficient, making the entire solution linear in the size of the array.

## Python Solution

```python
from typing import List

class Solution:
    def arithmeticTriplets(self, nums: List[int], diff: int) -> int:
        numbers = set(nums)
        count = 0

        for num in nums:
            if num + diff in numbers and num + 2 * diff in numbers:
                count += 1

        return count
```

The implementation begins by converting `nums` into a set called `numbers`. This is the core optimization because set membership checks run in average `O(1)` time.

Next, we initialize `count` to track how many arithmetic triplets we discover.

We then iterate through every value in `nums`. For each number, we check whether the next two required numbers in the arithmetic sequence exist:

```python
num + diff
num + 2 * diff
```

If both are present in the set, we increment the answer.

Finally, we return the total count.

The implementation directly mirrors the algorithm described earlier, making it concise and efficient.

## Go Solution

```go
func arithmeticTriplets(nums []int, diff int) int {
    numbers := make(map[int]bool)

    for _, num := range nums {
        numbers[num] = true
    }

    count := 0

    for _, num := range nums {
        if numbers[num+diff] && numbers[num+2*diff] {
            count++
        }
    }

    return count
}
```

The Go implementation uses a `map[int]bool` to simulate a hash set.

Unlike Python, Go does not have a built-in set type, so a map is the standard approach for constant-time membership checks.

The rest of the logic is identical to the Python solution:

- Insert all numbers into the map
- Iterate through each number
- Check whether the next two arithmetic values exist
- Increment the count if they do

There are no integer overflow concerns because the constraints are very small.

## Edge Cases

### Very Small Arrays

The minimum array length is `3`, which means the array may contain exactly one possible triplet.

For example:

```
nums = [1,2,3]
diff = 1
```

A buggy implementation might accidentally skip checking the only valid triplet. Our solution correctly processes every element and identifies the sequence.

### No Valid Triplets

Some arrays contain no arithmetic progression with the required difference.

Example:

```
nums = [1,2,4]
diff = 1
```

Even though `1` and `2` differ by `1`, the required third value `3` does not exist.

Our implementation handles this naturally through set membership checks.

### Overlapping Triplets

Triplets may share elements.

Example:

```
nums = [1,3,5,7,9]
diff = 2
```

This contains:

- `(1,3,5)`
- `(3,5,7)`
- `(5,7,9)`

A flawed implementation might accidentally skip overlapping sequences or double count incorrectly.

Our approach checks every value independently as a potential starting point, ensuring all valid triplets are counted exactly once.

### Large Difference Values

When `diff` is very large relative to the array values, triplets may become impossible.

Example:

```
nums = [0,1,2,3]
diff = 10
```

The required values simply do not exist in the set.

The implementation still works efficiently because each lookup remains constant time regardless of whether the value exists.
