# LeetCode 2378 - Choose Edges to Maximize Score in a Tree

## Problem Understanding

This problem gives us a rooted weighted tree with `n` nodes labeled from `0` to `n - 1`. Each node except the root has exactly one parent, and every parent-child connection has an associated edge weight.

The input format is slightly unusual. Instead of giving a list of edges directly, the array `edges` stores parent information for each node:

- `edges[i][0]` is the parent of node `i`
- `edges[i][1]` is the weight of the edge between node `i` and its parent

For the root node `0`, the value is `[-1, -1]` because it has no parent.

The goal is to select some edges so that:

1. No two selected edges are adjacent
2. The total sum of selected edge weights is maximized

Two edges are adjacent if they share a node.

This means that if we choose an edge connecting parent `u` and child `v`, then:

- We cannot choose any other edge touching `u`
- We cannot choose any other edge touching `v`

The problem is essentially asking for a maximum weight matching in a tree.

The constraints are important:

- `n` can be as large as `10^5`
- Edge weights may be negative
- The graph is guaranteed to form a valid tree

These constraints immediately tell us that brute force enumeration is impossible. A tree with `10^5` nodes can have an enormous number of possible edge subsets.

Negative weights also matter. Since we are allowed to choose no edges at all, the answer can never be negative. If an edge decreases the score, we should simply skip it.

Several edge cases are important:

- A tree with only one node has no edges, so the answer is `0`
- A star-shaped tree allows at most one chosen edge because all edges share the root
- A chain-shaped tree behaves similarly to the classic house robber problem
- Negative edges should never be chosen unless they somehow enable a larger gain later, which they never do in this problem

The tree structure strongly suggests a depth-first search dynamic programming solution.

## Approaches

### Brute Force Approach

A brute force solution would consider every subset of edges and check whether the chosen edges are pairwise non-adjacent.

For each subset:

1. Verify that no two chosen edges share a node
2. Compute the total weight
3. Track the maximum valid sum

This approach is correct because it explicitly checks all possibilities, guaranteeing that the optimal solution is found.

However, a tree with `n` nodes has `n - 1` edges, which means there are `2^(n-1)` possible subsets. With `n` up to `100000`, this is completely infeasible.

Even for relatively small trees, the exponential growth becomes overwhelming.

### Optimal Dynamic Programming on Trees

The key observation is that this is a tree DP problem with local constraints.

For every node, we want to know:

- What is the best score obtainable in its subtree if the edge connecting it to its parent is not chosen?
- What is the best score obtainable if one child edge may still be chosen?

The adjacency restriction is local. Choosing an edge only prevents choosing edges touching the same endpoints.

This allows us to process the tree bottom-up using DFS.

For each node:

- We first compute the best possible score from all children assuming none of the edges from the current node are chosen
- Then we try selecting exactly one edge from the current node to one child
- If we choose edge `(u, v)`, then node `u` cannot use any other child edges, and node `v` cannot use its parent edge

This creates a clean DP transition.

Because every node and edge is processed only once, the solution runs in linear time.

| Approach | Time Complexity | Space Complexity | Notes |

| Brute Force | O(2^n * n) | O(n) | Enumerates all edge subsets |
| Optimal DFS + DP | O(n) | O(n) | Processes each node and edge once |

## Algorithm Walkthrough

### Dynamic Programming State

For every node `u`, we maintain two values:

- `dp0[u]` = maximum score in the subtree rooted at `u` when no edge connected to `u`'s parent is selected
- `dp1[u]` = maximum score in the subtree rooted at `u` when the edge to the parent is already selected

The meaning of `dp1` is important. If the parent edge is already chosen, then node `u` cannot participate in any other chosen edge.

### DFS Processing

We process the tree using postorder DFS so children are solved before parents.

### Step-by-step Algorithm

1. Build an adjacency list representation of the tree from the parent array.
2. Start a DFS traversal from the root node `0`.
3. For each node `u`, initialize:

- `base = 0`

This value represents the score obtained when we do not choose any edge connected to `u`.
4. Visit every child `v` of `u`.
5. Recursively compute:

- `child0 = dp0[v]`
- `child1 = dp1[v]`
6. Add `child0` to `base`.

This means that if we do not choose edge `(u, v)`, then child `v` is free to optimize its subtree normally.
7. After computing the baseline score, try selecting one child edge.
8. For every child `(v, weight)`:

- If we choose edge `(u, v)`, then:

- node `u` cannot choose any other child edges
- node `v` cannot connect to its children using adjacent edges
9. The gain from selecting edge `(u, v)` is:

$$weight + dp1[v] - dp0[v]$$

The subtraction removes the previously included `dp0[v]`, and replaces it with the restricted state `dp1[v]`.
10. Track the maximum possible improvement.
11. Set:

- `dp1[u] = base`
- `dp0[u] = base + best_extra_gain`

1. Return the final answer `dp0[0]`.

### Why it works

The algorithm works because the only interaction between selected edges occurs through shared nodes.

For any node `u`, at most one edge touching `u` can be selected. Therefore, after computing the best independent contribution from each child subtree, we only need to decide whether selecting one child edge improves the result.

The DFS guarantees that child subtrees are already optimally solved before processing the parent. Since the DP states fully capture whether a node is already matched to its parent, every valid configuration is considered exactly once.

This ensures the optimal answer is computed.

## Python Solution

```python
from typing import List
import sys

sys.setrecursionlimit(1 << 20)

class Solution:
    def maxScore(self, edges: List[List[int]]) -> int:
        n = len(edges)

        children = [[] for _ in range(n)]

        for node in range(1, n):
            parent, weight = edges[node]
            children[parent].append((node, weight))

        def dfs(node: int) -> tuple[int, int]:
            base_score = 0

            child_results = []

            for child, weight in children[node]:
                dp0_child, dp1_child = dfs(child)

                base_score += dp0_child
                child_results.append((child, weight, dp0_child, dp1_child))

            best_gain = 0

            for _, weight, dp0_child, dp1_child in child_results:
                gain = weight + dp1_child - dp0_child
                best_gain = max(best_gain, gain)

            dp1 = base_score
            dp0 = base_score + best_gain

            return dp0, dp1

        answer, _ = dfs(0)

        return answer
```

The implementation begins by converting the parent representation into a children adjacency list. This makes DFS traversal straightforward because each node can directly access its descendants.

The recursive `dfs` function returns two DP values:

- `dp0`, where the node is free to match with one child
- `dp1`, where the node is already matched with its parent

Inside the DFS, `base_score` accumulates the contribution from all child subtrees assuming no edge from the current node is chosen.

Next, the algorithm checks whether selecting one child edge improves the result. The gain formula:

$$weight + dp1[child] - dp0[child]$$

correctly transitions the child from unrestricted mode to restricted mode.

Finally:

- `dp1 = base_score`
- `dp0 = base_score + best_gain`

The root node has no parent, so the final answer is `dp0[0]`.

## Go Solution

```go
package main

func maxScore(edges [][]int) int64 {
	n := len(edges)

	type Edge struct {
		to     int
		weight int64
	}

	children := make([][]Edge, n)

	for node := 1; node < n; node++ {
		parent := edges[node][0]
		weight := int64(edges[node][1])

		children[parent] = append(children[parent], Edge{
			to:     node,
			weight: weight,
		})
	}

	var dfs func(int) (int64, int64)

	dfs = func(node int) (int64, int64) {
		var baseScore int64 = 0

		type ChildInfo struct {
			weight int64
			dp0    int64
			dp1    int64
		}

		childResults := []ChildInfo{}

		for _, edge := range children[node] {
			dp0Child, dp1Child := dfs(edge.to)

			baseScore += dp0Child

			childResults = append(childResults, ChildInfo{
				weight: edge.weight,
				dp0:    dp0Child,
				dp1:    dp1Child,
			})
		}

		var bestGain int64 = 0

		for _, child := range childResults {
			gain := child.weight + child.dp1 - child.dp0

			if gain > bestGain {
				bestGain = gain
			}
		}

		dp1 := baseScore
		dp0 := baseScore + bestGain

		return dp0, dp1
	}

	answer, _ := dfs(0)

	return answer
}
```

The Go implementation follows the same logic as the Python version but uses explicit structs for readability.

The return type is `int64` because edge weights can be as large as `10^6`, and summing many edges may exceed 32-bit integer limits.

Slices are used for adjacency lists and child DP storage. Since Go does not support tuple returns in the same style as Python, the DFS returns two `int64` values directly.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node and edge is processed once |
| Space | O(n) | Adjacency list and recursion stack |

The DFS visits every node exactly once. For each node, we iterate through its children a constant number of times.

Since a tree with `n` nodes contains exactly `n - 1` edges, the total amount of work is linear.

The adjacency list requires `O(n)` space, and the recursion stack may also reach `O(n)` depth in a degenerate chain-shaped tree.

## Edge Cases

A single-node tree is an important edge case because there are no edges to choose from. Some implementations incorrectly assume every node has a parent edge or fail when the adjacency list is empty. This implementation handles it naturally because the DFS simply returns `(0, 0)` for the root.

Negative edge weights are another subtle case. Since selecting no edges is allowed, the answer should never become negative. The implementation handles this by initializing `best_gain` to `0`. As a result, harmful edges are automatically ignored.

A star-shaped tree is especially important because every edge shares the root node. This means at most one edge can be selected. A buggy implementation might incorrectly allow multiple child edges simultaneously. The DP transition explicitly considers choosing only one child edge, which correctly enforces the adjacency rule.

A chain-shaped tree can create deep recursion and requires alternating edge selection. For example, selecting edges `(0,1)` and `(1,2)` together is invalid. The two-state DP precisely tracks whether a node is already connected to its parent, preventing illegal adjacent selections.
