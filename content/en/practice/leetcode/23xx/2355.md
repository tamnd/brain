## Solution

## Problem Understanding

The problem provides a 0-indexed array `books` where each element represents the number of books on a shelf. You are asked to select a contiguous subarray of shelves and take books in such a way that for every consecutive shelf in the chosen subarray, the number of books you take **strictly increases** as you move to the right. The goal is to maximize the total number of books taken.

The input array can be as long as $10^5$, and each element can be as large as $10^5$. This means a brute-force approach that tries all subarrays would be too slow. The output is a single integer, the maximum total number of books you can take under the constraints.

Important edge cases include shelves with zero books, all shelves with the same number of books, or arrays of length one. The problem guarantees that the array is non-empty and values are non-negative, so no negative handling is required.

## Approaches

### Brute-Force Approach

The brute-force approach would be to iterate through all possible contiguous subarrays and, for each subarray, determine the maximum number of books that can be taken following the strictly increasing rule. This involves checking every valid starting point and every valid ending point, and for each, computing the allowed sequence. While correct in principle, this approach is infeasible due to the $O(n^2)$ or $O(n^3)$ complexity for arrays of size up to $10^5$.

### Optimal Approach

The key observation is that the strictly increasing property means that at any shelf `i`, the maximum number of books you can take from shelf `i` is constrained by the minimum of `books[i]` and one more than the number of books you took from the previous shelf (going backward). This naturally suggests a dynamic programming approach where we traverse the array from left to right, calculating the maximum sum of books ending at each position. To do this efficiently, we can use a monotonic stack to maintain valid increasing sequences and their cumulative sums.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Try all subarrays, sum valid increasing sequences |
| Optimal | O(n) | O(n) | Use DP with a monotonic stack to track maximum books ending at each shelf |

## Algorithm Walkthrough

1. Initialize a stack to keep track of tuples `(index, total_books_taken)` where `index` is the shelf index and `total_books_taken` is the total books collected in a strictly increasing sequence ending at that index.
2. Iterate through the shelves from left to right.
3. For each shelf `i`, determine the maximum books you can take from it. If the previous shelf had more books than allowed for strictly increasing order, adjust the count.
4. While the stack is not empty and the last shelf in the stack has more books than allowed to maintain a strictly increasing sequence with the current shelf, pop elements from the stack.
5. Compute the cumulative sum of books for the current shelf including the valid increasing sequence before it and push `(i, total_books)` onto the stack.
6. Track the maximum total books found after processing each shelf.

**Why it works:** The stack maintains the property that for every element, the sequence ending at that element is valid and strictly increasing. By keeping cumulative sums, we can efficiently compute the total for each possible ending shelf and ensure the overall maximum is found.

## Python Solution

```python
from typing import List

class Solution:
    def maximumBooks(self, books: List[int]) -> int:
        n = len(books)
        dp = [0] * n
        stack = []
        max_books = 0

        for i in range(n):
            current_books = books[i]
            count = 1

            # Adjust stack to maintain strictly increasing sequences
            while stack and stack[-1][0] >= current_books:
                prev_books, prev_count, prev_sum = stack.pop()
                count += prev_count

            total = current_books * count
            if stack:
                total += stack[-1][2]

            stack.append((current_books, count, total))
            max_books = max(max_books, total)

        return max_books
```

**Implementation Walkthrough:** We use a stack to maintain sequences where each element represents `(books_taken, length_of_sequence, total_sum)`. For each shelf, we combine it with previous sequences if the previous books are larger than the current shelf. The total sum is updated accordingly, and `max_books` keeps track of the global maximum.

## Go Solution

```go
func maximumBooks(books []int) int64 {
    n := len(books)
    type item struct {
        val, count, total int64
    }

    stack := []item{}
    var maxBooks int64 = 0

    for i := 0; i < n; i++ {
        current := int64(books[i])
        count := int64(1)
        for len(stack) > 0 && stack[len(stack)-1].val >= current {
            top := stack[len(stack)-1]
            stack = stack[:len(stack)-1]
            count += top.count
        }

        total := current * count
        if len(stack) > 0 {
            total += stack[len(stack)-1].total
        }

        stack = append(stack, item{current, count, total})
        if total > maxBooks {
            maxBooks = total
        }
    }

    return maxBooks
}
```

**Go-specific notes:** Go handles integers explicitly, so we use `int64` to avoid overflow. Slices are used as the stack, and we manage the slice manually when popping elements.

## Edge Cases

The first edge case is when the array has only one shelf. In this scenario, the algorithm correctly returns the number of books on that shelf because there is no other shelf to constrain the strictly increasing condition.

The second edge case involves shelves with zero books. Zeros can block sequences, and the algorithm correctly handles them by effectively allowing sequences to start after zeros or take fewer books before zeros.

The third edge case occurs when multiple contiguous sequences can be formed, but only one maximizes the sum. The monotonic stack approach efficiently combines sequences while respecting the strictly increasing property to always find the maximal sum. This avoids manual enumeration of all possible sequences, which would be inefficient.
