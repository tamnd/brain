## Solution

## Problem Understanding

In this problem, we are given two database tables, `Employees` and `Logs`.

The `Employees` table stores the minimum number of hours each employee must work during October 2022 in order to receive their full salary. Each employee appears exactly once in this table.

The `Logs` table records work sessions. Every row represents one continuous working session for an employee, defined by a start timestamp (`in_time`) and an end timestamp (`out_time`). A session may cross midnight, which means `out_time` can belong to the next calendar day.

The key requirement is that each session duration must be rounded up to the next whole minute before summing all sessions together.

For example:

- A session lasting exactly 51 minutes counts as 51 minutes.
- A session lasting 51 minutes and 2 seconds counts as 52 minutes.
- A session lasting 4 hours and 1 second counts as 241 minutes.

After calculating the total worked minutes for every employee, we compare that total against the employee's required working hours.

Since `needed_hours` is stored in hours, we convert it into minutes:

$$\text{required minutes} = \text{needed\_hours} \times 60$$

An employee should appear in the output if:

$$\text{worked minutes} < \text{required minutes}$$

The result only needs to contain the `employee_id` values of employees who should receive deductions.

Several edge cases are important:

- Employees may have no logs at all. Their worked time should be treated as zero minutes.
- Sessions can cross midnight, so timestamp subtraction must work across dates.
- Sessions containing extra seconds must round upward correctly.
- Employees may work exactly the required amount, which means they should not be deducted.
- Multiple sessions must be summed together before comparison.

This is fundamentally an aggregation problem involving grouped duration calculations and conditional filtering.

## Approaches

### Brute Force Approach

A brute-force solution would process each employee independently and repeatedly scan the entire `Logs` table to find matching sessions.

For every employee:

1. Iterate through all log entries.
2. Select only the sessions belonging to that employee.
3. Compute each session duration in minutes.
4. Round each session upward.
5. Sum all rounded minutes.
6. Compare the total against the required hours.

This approach is correct because every employee's total working time is computed exactly from their sessions.

However, it is inefficient because if there are `E` employees and `L` log records, we repeatedly scan the logs table for every employee. That leads to:

$$O(E \times L)$$

time complexity.

When the number of employees and logs becomes large, repeated full-table scans become unnecessarily expensive.

### Optimal Approach

The key observation is that each log entry only needs to be processed once.

Instead of scanning the logs repeatedly, we can:

1. Group logs by employee using aggregation.
2. Compute rounded session durations immediately.
3. Sum all durations per employee.
4. Join the aggregated totals with the `Employees` table.
5. Filter employees whose total worked minutes are below the requirement.

SQL databases are optimized for grouped aggregation, making this approach much more efficient.

The important detail is the rounding rule. Since each session must be rounded upward individually, we cannot simply sum raw seconds and round at the end.

For each session:

$$\text{rounded minutes} =
\left\lceil
\frac{\text{TIMESTAMPDIFF(SECOND, in\_time, out\_time)}}{60}
\right\rceil$$

In MySQL, this can be implemented using:

```
CEIL(TIMESTAMPDIFF(SECOND, in_time, out_time) / 60)
```

After summing these rounded session durations, we compare against `needed_hours * 60`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(E × L) | O(1) | Repeatedly scans all logs for every employee |
| Optimal | O(E + L) | O(E) | Aggregates logs once using grouping |

## Algorithm Walkthrough

1. Start by calculating the worked duration for every session in the `Logs` table.

We subtract `in_time` from `out_time` using `TIMESTAMPDIFF(SECOND, in_time, out_time)` to get the session duration in seconds.
2. Convert each session duration into rounded-up minutes.

Since even one extra second should count as a full additional minute, we divide by 60 and apply `CEIL`.

For example:

- 480 seconds → 8 minutes
- 482 seconds → 9 minutes
3. Group all sessions by `employee_id`.

This allows us to compute the total worked minutes for each employee across all sessions.
4. Use a `LEFT JOIN` between `Employees` and the aggregated log totals.

This step is critical because employees with no sessions still need to appear in consideration. Without a `LEFT JOIN`, employees without logs would disappear from the result.
5. Replace missing totals with zero using `COALESCE`.

Employees without any logs should have worked minutes equal to zero.
6. Compare total worked minutes against required minutes.

The required minutes are:

```
needed_hours * 60
```
7. Return employees whose total worked minutes are smaller than the requirement.

### Why it works

The algorithm works because every session duration is individually rounded upward before aggregation, exactly matching the problem specification. Grouping guarantees that all sessions for the same employee are summed together. The `LEFT JOIN` ensures employees without logs are still included, and comparing against `needed_hours * 60` correctly identifies employees who failed to meet their required working time.

## Python Solution

```python
class Solution:
    def employeesWithDeductions(self, employees: 'pd.DataFrame', logs: 'pd.DataFrame') -> 'pd.DataFrame':
        import pandas as pd
        import numpy as np

        if logs.empty:
            return employees[['employee_id']]

        # Calculate session duration in seconds
        duration_seconds = (
            logs['out_time'] - logs['in_time']
        ).dt.total_seconds()

        # Round each session upward to minutes
        logs['worked_minutes'] = np.ceil(duration_seconds / 60)

        # Aggregate total worked minutes per employee
        worked_totals = (
            logs.groupby('employee_id', as_index=False)['worked_minutes']
            .sum()
        )

        # Merge with employees table
        merged = employees.merge(
            worked_totals,
            on='employee_id',
            how='left'
        )

        # Employees with no logs worked 0 minutes
        merged['worked_minutes'] = merged['worked_minutes'].fillna(0)

        # Filter employees below required hours
        result = merged[
            merged['worked_minutes'] < merged['needed_hours'] * 60
        ]

        return result[['employee_id']]
```

The implementation starts by handling the special case where the `Logs` table is empty. In that scenario, every employee failed to meet their requirement, so all employee IDs are returned.

Next, the solution computes each session duration in seconds by subtracting `in_time` from `out_time`. Pandas datetime subtraction produces a timedelta object, and `.dt.total_seconds()` converts it into numeric seconds.

The code then applies `np.ceil(duration_seconds / 60)` to round each session upward individually. This is important because rounding must occur per session, not after summing all sessions together.

After computing rounded session durations, the solution groups rows by `employee_id` and sums all worked minutes.

The aggregated totals are merged with the `Employees` table using a left join. Employees without logs receive `NaN` values, which are replaced with zero using `fillna(0)`.

Finally, the solution filters employees whose worked minutes are strictly less than their required minutes.

## Go Solution

```go
func employeesWithDeductions(employees Table, logs Table) Table {
    return `
SELECT e.employee_id
FROM Employees e
LEFT JOIN (
    SELECT
        employee_id,
        SUM(
            CEIL(
                TIMESTAMPDIFF(SECOND, in_time, out_time) / 60
            )
        ) AS worked_minutes
    FROM Logs
    GROUP BY employee_id
) l
ON e.employee_id = l.employee_id
WHERE COALESCE(l.worked_minutes, 0) < e.needed_hours * 60
`
}
```

The Go version for database problems on LeetCode is typically just the SQL query returned as a string.

The implementation uses a subquery to aggregate worked minutes per employee. `TIMESTAMPDIFF(SECOND, in_time, out_time)` computes raw session duration, while `CEIL(... / 60)` applies the required upward rounding.

The `LEFT JOIN` ensures employees without any logs remain included in the final comparison. `COALESCE` converts missing aggregated totals into zero so those employees are correctly marked for deductions.

## Edge Cases

### Employees With No Logs

An employee may never appear in the `Logs` table. A naive inner join would completely exclude such employees from consideration, producing incorrect results.

The implementation solves this by using a `LEFT JOIN` in SQL and a left merge in Pandas. Missing totals are converted to zero using `COALESCE` or `fillna(0)`.

### Sessions Crossing Midnight

Some sessions begin on one day and end after midnight on the next day. A simplistic implementation that only compares clock times could produce negative durations or incorrect calculations.

The solution correctly subtracts full datetime values, so crossing dates does not affect correctness.

### Per-Session Rounding

The most subtle edge case is that rounding must happen separately for every session.

Suppose an employee works:

- 30 minutes 1 second
- 29 minutes 1 second

Each session rounds upward individually:

- 31 minutes
- 30 minutes

Total:

$$61 \text{ minutes}$$

If we incorrectly summed raw seconds first and rounded only once at the end, we would get 60 minutes instead. The implementation avoids this bug by applying `CEIL` before aggregation.
