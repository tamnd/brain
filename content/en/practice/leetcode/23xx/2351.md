## Solution

## Problem Understanding

The problem asks us to find the first letter in a string that appears **twice**. In other words, we need to identify which letter repeats first based on the index of its **second occurrence**. For example, if `a` appears at indices 0 and 5, and `b` appears at indices 1 and 4, the second occurrence of `b` (index 4) comes before the second occurrence of `a` (index 5), so `b` would be returned.

The input is a string `s` consisting of lowercase English letters only. The constraints are relatively small: the string length is between 2 and 100, and there is always **at least one repeated letter**, which guarantees that we will always have a valid output.

Key points to understand are that we are only concerned with **the second occurrence** of letters, and ties are broken by which letter’s second occurrence comes first. Edge cases include strings where the repeated letter is at the very end or when multiple letters repeat quickly.

## Approaches

The **brute-force approach** would involve checking every letter against every other letter to find the first repeat. Essentially, for each letter, we could scan the rest of the string to see if it appears again. This is guaranteed to work but requires checking every pair, which results in $O(n^2)$ time complexity. Given our constraints, this is technically feasible, but it is unnecessary when we can do better.

The **optimal approach** uses a **hash set** to keep track of letters we have already seen. We iterate through the string, and for each letter, we check if it is already in the set. If it is, this is the first letter to appear twice, and we return it immediately. Otherwise, we add it to the set and continue. This approach leverages the fact that set membership checking is O(1) on average, giving us a linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Compare each letter with all letters after it |
| Optimal | O(n) | O(26) = O(1) | Track seen letters in a hash set for constant-time lookup |

## Algorithm Walkthrough

1. Initialize an empty hash set called `seen_letters`. This will store all letters we have encountered so far.
2. Iterate through each character `char` in the string `s`.
3. For each `char`, check if it exists in `seen_letters`.
4. If `char` exists in the set, return `char` immediately. This is guaranteed to be the first letter whose second occurrence appears earliest.
5. If `char` does not exist in the set, add it to `seen_letters`.
6. Continue the iteration until the repeated character is found.

**Why it works:** The set guarantees that we track every letter exactly once. The first time we encounter a letter already in the set, it must be the one whose second occurrence comes earliest, satisfying the problem condition.

## Python Solution

```python
class Solution:
    def repeatedCharacter(self, s: str) -> str:
        seen_letters = set()
        for char in s:
            if char in seen_letters:
                return char
            seen_letters.add(char)
        # Given constraints guarantee a repeated letter, so no need for a return here
```

The code initializes a set to store characters seen. As we iterate through `s`, if a character is already in the set, we immediately return it as the first repeating letter. Otherwise, we add it to the set for future checks.

## Go Solution

```go
func repeatedCharacter(s string) byte {
    seenLetters := make(map[rune]bool)
    for _, char := range s {
        if seenLetters[char] {
            return byte(char)
        }
        seenLetters[char] = true
    }
    return 0 // unreachable because input guarantees a repeated character
}
```

In Go, we use a `map[rune]bool` to mimic the behavior of a hash set. We iterate over the string as runes for Unicode safety, although here only lowercase letters are used. The first repeated rune is returned. Returning `0` is technically unreachable due to the problem constraints.

## Edge Cases

The first important edge case is when the repeated character occurs at the very beginning, for example `"aabb"`. A naive implementation that only counts occurrences might incorrectly pick the last repeated letter rather than the first to repeat. Our set-based approach immediately returns the first duplicate.

The second edge case is when the repeated character occurs at the very end of the string, such as `"abcdefggh"`. This checks that the algorithm does not assume repeats occur early and correctly iterates through the entire string.

The third edge case is a string composed of repeated instances of the same letter, like `"zzzzzz"`. It ensures that the algorithm handles multiple duplicates of the same letter and returns the first one correctly without confusion.

This solution handles all edge cases naturally because it checks letters sequentially and returns immediately upon the first repeat.
