## Solution

## Problem Understanding

The problem gives us exactly five poker cards. Each card has two properties:

- A rank, represented by the integer array `ranks`
- A suit, represented by the character array `suits`

The card at index `i` is defined by the pair `(ranks[i], suits[i])`.

We need to determine the strongest possible poker hand that can be formed from these five cards. The valid hand rankings are already simplified for us and ordered from strongest to weakest:

1. `"Flush"`, all five cards have the same suit
2. `"Three of a Kind"` , at least three cards share the same rank
3. `"Pair"` , at least two cards share the same rank
4. `"High Card"` , none of the above conditions are satisfied

The important detail is that we return only the best possible category. For example, if the hand contains both a pair and a three of a kind, we must return `"Three of a Kind"` because it ranks higher.

The constraints are very small:

- `ranks.length == suits.length == 5`
- Ranks are between `1` and `13`
- Suits are between `'a'` and `'d'`
- No duplicate card exists

Because there are always exactly five cards, the input size is constant. This means performance is not a major concern, but we still want a clean and logically correct solution.

Several edge cases are important:

- All suits identical, which should immediately produce `"Flush"`
- Multiple repeated ranks, such as four cards with the same rank
- Exactly one pair
- No repeated ranks and mixed suits, resulting in `"High Card"`
- Hands where more than one category is possible, where we must correctly choose the strongest one

For example, `[4,4,4,2,2]` contains both a pair and a three of a kind. The correct answer is `"Three of a Kind"`.

## Approaches

### Brute Force Approach

A brute force solution would explicitly examine every possible hand category separately using repeated scans through the arrays.

For example:

- Check whether all suits are equal by comparing every pair of suits
- Count how many times each rank appears using nested loops
- Search for ranks appearing at least three times
- Search again for ranks appearing at least two times

This approach works because the input size is tiny, so even repeated scanning is fast enough. However, the implementation becomes unnecessarily repetitive and less clean.

The nested loop counting approach would compare each rank against every other rank, leading to quadratic work.

### Optimal Approach

The better solution uses frequency counting.

The key observation is that the poker hand strength depends entirely on:

- Whether all suits are identical
- The maximum frequency of any rank

We can solve the problem efficiently by:

1. Checking whether all suits match
2. Counting occurrences of each rank using a hash map
3. Tracking the highest rank frequency

If all suits match, return `"Flush"` immediately because it is the strongest possible category.

Otherwise:

- If any rank appears at least 3 times, return `"Three of a Kind"`
- Else if any rank appears at least 2 times, return `"Pair"`
- Otherwise return `"High Card"`

This approach is simpler, cleaner, and naturally extends to larger inputs if needed.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Uses nested loops to count matching ranks |
| Optimal | O(n) | O(n) | Uses a hash map for frequency counting |

Since `n = 5`, both approaches are fast in practice, but the optimal solution is more elegant and scalable.

## Algorithm Walkthrough

1. First, check whether all suits are identical.

We compare every suit against the first suit. If every card has the same suit, then the hand is a `"Flush"`.

We do this first because `"Flush"` is the highest ranked hand in this problem. Once found, we can immediately return the answer.
2. Create a hash map to count rank frequencies.

The key is the rank value, and the value is the number of times that rank appears.

For example:

```
ranks = [4,4,2,4,4]
```

Produces:

```
{
    4: 4,
    2: 1
}
```
3. Track the maximum frequency while building the map.

Each time we increment a rank count, we update `max_frequency`.

This avoids needing another pass through the map later.
4. Determine the best hand based on the maximum frequency.

- If `max_frequency >= 3`, return `"Three of a Kind"`
- Else if `max_frequency == 2`, return `"Pair"`
- Else return `"High Card"`
5. Return the final answer.

### Why it works

The algorithm works because the problem defines poker hands entirely through two independent properties:

- Suit uniformity determines whether the hand is a flush
- Rank frequencies determine whether the hand contains repeated ranks

Checking for a flush first guarantees we return the strongest possible category. After that, the largest rank frequency uniquely determines whether the hand is a three of a kind, pair, or high card. Since every card is processed correctly and every category is checked in priority order, the algorithm always produces the correct answer.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def bestHand(self, ranks: List[int], suits: List[str]) -> str:
        # Check for Flush
        if all(suit == suits[0] for suit in suits):
            return "Flush"

        rank_count = defaultdict(int)
        max_frequency = 0

        # Count rank frequencies
        for rank in ranks:
            rank_count[rank] += 1
            max_frequency = max(max_frequency, rank_count[rank])

        # Determine best remaining hand
        if max_frequency >= 3:
            return "Three of a Kind"

        if max_frequency == 2:
            return "Pair"

        return "High Card"
```

The implementation begins by checking whether every suit matches the first suit. Python's `all()` function makes this concise and readable.

If the hand is not a flush, we create a `defaultdict(int)` to count occurrences of each rank. As we iterate through the `ranks` array, we increment the count for each rank and simultaneously maintain the highest frequency encountered.

After processing all cards, the maximum frequency directly determines the best remaining hand category:

- Frequency at least `3` means `"Three of a Kind"`
- Frequency exactly `2` means `"Pair"`
- Otherwise `"High Card"`

The solution closely follows the algorithm steps described earlier and keeps the logic simple and efficient.

## Go Solution

```go
func bestHand(ranks []int, suits []byte) string {
    // Check for Flush
    flush := true

    for i := 1; i < len(suits); i++ {
        if suits[i] != suits[0] {
            flush = false
            break
        }
    }

    if flush {
        return "Flush"
    }

    rankCount := make(map[int]int)
    maxFrequency := 0

    // Count rank frequencies
    for _, rank := range ranks {
        rankCount[rank]++

        if rankCount[rank] > maxFrequency {
            maxFrequency = rankCount[rank]
        }
    }

    // Determine best remaining hand
    if maxFrequency >= 3 {
        return "Three of a Kind"
    }

    if maxFrequency == 2 {
        return "Pair"
    }

    return "High Card"
}
```

The Go implementation follows the same logic as the Python version.

Instead of Python's `defaultdict`, Go uses a `map[int]int` for frequency counting. Missing keys automatically default to `0`, which makes counting straightforward.

The suit check uses a boolean variable `flush` and iterates through the `suits` slice starting from index `1`.

There are no integer overflow concerns because all counts remain extremely small. The solution also does not need special handling for nil slices because the problem guarantees exactly five cards.

## Edge Cases

### Flush With Repeated Ranks

A hand such as:

```
ranks = [8,8,2,3,4]
suits = ["c","c","c","c","c"]
```

contains both a pair and a flush. A buggy implementation might incorrectly return `"Pair"` if it checks rank frequencies first.

This implementation avoids that problem by checking for `"Flush"` before anything else. Since flush is the strongest category in this problem, it immediately returns the correct answer.

### Four of a Kind

The problem does not explicitly define `"Four of a Kind"` as a separate category.

For example:

```
[9,9,9,9,2]
```

A naive implementation might fail because it looks only for frequency exactly equal to `3`.

This solution correctly uses:

```
if max_frequency >= 3
```

which ensures that both three-of-a-kind and four-of-a-kind hands return `"Three of a Kind"`.

### Multiple Pairs

A hand like:

```
[5,5,9,9,2]
```

contains two separate pairs.

The problem does not define `"Two Pair"` as a category, so the correct answer is simply `"Pair"`.

The implementation handles this naturally because it only tracks the maximum frequency. The highest frequency is `2`, so the result becomes `"Pair"` automatically.
