**Difficulty:** 🔴 Hard  
**Topics:** Array, Depth-First Search, Union-Find, Graph Theory

## Solution

## Problem Understanding

The problem presents a system of multiplicative equations expressed as `Ai / Bi = values[i]`. Each equation links two variables with a given ratio. The task is to determine if any contradiction exists in the system. A contradiction occurs when two different chains of equations imply conflicting ratios between the same pair of variables. For example, if the equations imply `a / b = 2` and `a / b = 3` simultaneously, this is a contradiction.

The input consists of `equations`, a 2D array of strings representing variable pairs, and `values`, an array of positive floating-point numbers representing the ratio of the first variable to the second. The output is a boolean indicating whether any contradiction exists.

Constraints ensure that the number of equations is moderate (up to 100), variable names are short, and all ratios are positive with at most two decimal places. This allows the use of floating-point arithmetic without excessive precision concerns. Important edge cases include cycles in the equations (which may produce contradictions) and equations where a variable is indirectly linked to itself through a chain of ratios.

## Approaches

The brute-force approach would be to try to assign values to each variable iteratively and check all pairs for consistency. One could pick an initial value for a variable, propagate values through all equations, and check for contradictions. This approach is correct in principle, but it is complex to implement and has exponential time complexity in the worst case because all chains of equations must be evaluated, especially in cycles.

The optimal approach recognizes that the equations form a weighted undirected graph. Each variable is a node, and an edge between `Ai` and `Bi` has a weight equal to the ratio `Ai / Bi`. Using Union-Find (Disjoint Set Union) with path compression and weight tracking allows us to merge connected components while maintaining the implied ratios. When merging, if we encounter an existing connection between two variables, we can verify the ratio consistency. This efficiently detects contradictions.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N^2) | O(N) | Iteratively propagate values through equations; may evaluate many paths repeatedly |
| Optimal (Union-Find) | O(N α(N)) | O(N) | Maintains ratio relationships using weighted union-find with path compression; α(N) is the inverse Ackermann function |

## Algorithm Walkthrough

1. **Initialize Union-Find structures**: Maintain a parent map for each variable and a weight map representing the ratio between the variable and its parent.
2. **Define `find` function with path compression**: While finding the root of a variable, update its parent and adjust the weight to maintain the cumulative ratio to the root.
3. **Define `union` function**: When connecting two variables, find their roots. If roots differ, merge one root under the other and update weights to maintain the correct ratio relationship.
4. **Iterate through each equation**: For each `Ai / Bi = value`, check if `Ai` and `Bi` are already connected. If connected, verify that the implied ratio matches `value` within a tolerance of `1e-5`. If not, a contradiction exists.
5. **Merge otherwise**: If no contradiction is detected, merge the two variables using the `union` function, maintaining the ratio in the weight map.
6. **Return false** if all equations are processed without contradiction.

Why it works: Union-Find maintains connected components and efficiently tracks the relative ratios among variables. By updating weights during path compression and merging, we ensure that all implied ratios are consistent. Any detected inconsistency corresponds directly to a contradiction in the original system.

## Python Solution

```python
from typing import List

class Solution:
    def checkContradictions(self, equations: List[List[str]], values: List[float]) -> bool:
        parent = {}
        weight = {}

        def find(x: str) -> str:
            if x not in parent:
                parent[x] = x
                weight[x] = 1.0
                return x
            if parent[x] != x:
                orig_parent = parent[x]
                parent[x] = find(parent[x])
                weight[x] *= weight[orig_parent]
            return parent[x]

        def union(x: str, y: str, value: float):
            root_x = find(x)
            root_y = find(y)
            if root_x != root_y:
                parent[root_x] = root_y
                weight[root_x] = value * weight[y] / weight[x]

        for (a, b), val in zip(equations, values):
            if a not in parent:
                parent[a] = a
                weight[a] = 1.0
            if b not in parent:
                parent[b] = b
                weight[b] = 1.0
            root_a, root_b = find(a), find(b)
            if root_a == root_b:
                if abs(weight[a] / weight[b] - val) > 1e-5:
                    return True
            else:
                union(a, b, val)
        return False
```

The Python implementation uses a dictionary to store the parent and weight of each variable. The `find` function performs path compression while updating the weight to maintain the ratio from a variable to its root. The `union` function merges two trees while preserving the ratio consistency. During iteration, any detected discrepancy in ratios immediately returns `True` to indicate a contradiction.

## Go Solution

```go
package main

func checkContradictions(equations [][]string, values []float64) bool {
    parent := map[string]string{}
    weight := map[string]float64{}

    var find func(string) string
    find = func(x string) string {
        if _, ok := parent[x]; !ok {
            parent[x] = x
            weight[x] = 1.0
        }
        if parent[x] != x {
            orig := parent[x]
            parent[x] = find(parent[x])
            weight[x] *= weight[orig]
        }
        return parent[x]
    }

    union := func(x, y string, val float64) {
        rootX := find(x)
        rootY := find(y)
        if rootX != rootY {
            parent[rootX] = rootY
            weight[rootX] = val * weight[y] / weight[x]
        }
    }

    for i, eq := range equations {
        a, b := eq[0], eq[1]
        if _, ok := parent[a]; !ok {
            parent[a] = a
            weight[a] = 1.0
        }
        if _, ok := parent[b]; !ok {
            parent[b] = b
            weight[b] = 1.0
        }
        rootA, rootB := find(a), find(b)
        if rootA == rootB {
            if abs(weight[a]/weight[b]-values[i]) > 1e-5 {
                return true
            }
        } else {
            union(a, b, values[i])
        }
    }
    return false
}

func abs(x float64) float64 {
    if x < 0 {
        return -x
    }
    return x
}
```

The Go implementation mirrors the Python version closely, using maps for parents and weights. The `find` function performs path compression while updating the weights. The `union` function merges two sets while maintaining correct ratios. An `abs` helper is used for floating-point comparison.

## Worked Examples

### Example 1

Equations: `[["a","b"],["b","c"],["a","c"]]`, values: `[3,0.5,1.5]`

1. Initialize: `parent = {}`, `weight = {}`.
2. Process `"a/b = 3"`: both variables are new; assign parent to self, weight = 1.0. Merge `a` and `b`. `weight[a] = 3 * 1 / 1 = 3`.
3. Process `"b/c = 0.5"`: merge `b` and `c`. `weight[b] = 0.5 * 1 / 1 = 0.5`.
4. Process `"a/c = 1.5"`: `find(a)` → root `c` via `b`; `weight[a]/weight[c] = 1.5`, matches `value`. No contradiction.
5. Return `False`.

### Example 2

Equations: `[["le","et"],["le","code"],["code","et"]]`, values: `[2,5,0.5]`

1. `"le/et = 2"`: merge `le` and `et`; `weight[le] = 2`.
2. `"le/code = 5"`: merge `le` and `code`; `weight[le] = 5 * 1 / 2 = 2.5`.
3. `"code/et = 0.5"`: `find(code)` and `find(et)` both root to same component; calculated ratio `weight[code]/weight[et] = 0.4`. Contradiction with `0.5`.
4. Return `True`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(N α(N)) | Each `find` and `union` operation is almost constant time due to path compression; α(N) is inverse Ackermann function |
| Space | O(N) | Store parent and weight for each variable; N is the number of unique variables |

The algorithm is highly efficient for up to 100 equations and variable names of length up to 5.
