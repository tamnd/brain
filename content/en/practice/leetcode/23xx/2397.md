# LeetCode 2397 - Maximum Rows Covered by Columns

## Problem Understanding

This problem asks us to choose exactly `numSelect` columns from a binary matrix in a way that maximizes the number of covered rows.

The matrix contains only `0`s and `1`s. A row is considered covered if every column containing a `1` in that row has been selected. In other words, all required columns for that row must belong to our chosen set.

For example, if a row is:

```
[1, 0, 1, 0]
```

then columns `0` and `2` must both be selected for the row to count as covered.

Rows containing only `0`s are always covered because they do not require any selected columns.

The input consists of:

- A binary matrix `matrix` of size `m x n`
- An integer `numSelect`, representing exactly how many columns we must choose

The output is the maximum number of rows that can be covered by some valid selection of exactly `numSelect` distinct columns.

The constraints are extremely important:

- `1 <= m, n <= 12`

Since the number of columns is at most `12`, the total number of possible column subsets is manageable:

```
2^12 = 4096
```

This immediately suggests that bitmask enumeration is feasible. Even trying every subset of columns is computationally practical.

Several edge cases are important:

- Rows with all zeros should always count as covered
- `numSelect` may equal `n`, meaning every column is selected
- Multiple rows may require identical column sets
- Some rows may require more than `numSelect` columns, making them impossible to cover
- A naive implementation using repeated list scanning can become inefficient if not carefully structured

The problem guarantees that `numSelect` is always valid and between `1` and `n`.

## Approaches

### Brute Force Approach

The brute force solution generates every possible combination of exactly `numSelect` columns. For each combination, it checks every row to determine whether that row is covered.

To determine whether a row is covered, we scan the row and verify that every position containing `1` belongs to the selected column set.

This approach is correct because it exhaustively checks all valid selections of columns, guaranteeing that the best answer will eventually be found.

However, the implementation can become inefficient if we repeatedly scan rows and perform membership checks using lists or sets. Although the constraints are small enough that even brute force may pass, there is a more elegant and faster solution using bitmasks.

### Key Insight

The key observation is that both rows and selected columns can be represented as bitmasks.

For example:

```
Row: [1, 0, 1, 1]
Mask: 1101
```

Each bit represents whether a column is required.

Similarly, a selected set of columns can also be represented as a bitmask.

A row is covered if every required bit in the row exists in the selected mask. This can be checked efficiently using bit operations:

```
(rowMask & selectedMask) == rowMask
```

This condition means all required bits from the row are present in the selected columns.

Since `n <= 12`, enumerating all `2^n` masks is fast enough.

## Algorithm Walkthrough

1. Convert each row into a bitmask.

For every row, create an integer where the `j`th bit is set if `matrix[i][j] == 1`.

Example:

```
[0,1,1,0] -> 0110
```

This compact representation allows very fast subset checks later.
2. Initialize a variable to store the maximum number of covered rows.

We will update this answer while examining different column selections.
3. Enumerate every possible column subset.

Since there are `n` columns, there are `2^n` possible subsets. Each subset is represented as a bitmask from `0` to `(1 << n) - 1`.
4. Ignore subsets that do not contain exactly `numSelect` columns.

We only want masks whose number of set bits equals `numSelect`.
5. For each valid subset, count how many rows are covered.

A row is covered if:

```
(rowMask & selectedMask) == rowMask
```

This means every required column in the row exists inside the selected subset.
6. Update the maximum answer.

After counting covered rows for the current subset, compare it with the best answer found so far.
7. Return the final maximum.

### Why it works

Each row mask represents the exact set of columns required to cover that row. A selected column mask covers a row if it contains all bits from the row mask.

Because we enumerate every valid selection of exactly `numSelect` columns, we are guaranteed to examine the optimal selection. The subset check using bit operations is mathematically correct because bitwise AND preserves only shared bits.

Therefore, the algorithm always returns the maximum possible number of covered rows.

## Python Solution

```python
from typing import List

class Solution:
    def maximumRows(self, matrix: List[List[int]], numSelect: int) -> int:
        row_masks = []

        for row in matrix:
            mask = 0

            for col, value in enumerate(row):
                if value == 1:
                    mask |= (1 << col)

            row_masks.append(mask)

        n = len(matrix[0])
        max_covered = 0

        for selected_mask in range(1 << n):
            if selected_mask.bit_count() != numSelect:
                continue

            covered_rows = 0

            for row_mask in row_masks:
                if (row_mask & selected_mask) == row_mask:
                    covered_rows += 1

            max_covered = max(max_covered, covered_rows)

        return max_covered
```

The implementation begins by converting every row into a bitmask. This preprocessing step avoids repeatedly scanning rows during subset evaluation.

The outer loop enumerates every possible column subset using integers from `0` to `2^n - 1`.

The `bit_count()` method ensures we only consider subsets containing exactly `numSelect` selected columns.

For every valid subset, we iterate through all row masks and apply the subset condition:

```python
(row_mask & selected_mask) == row_mask
```

If true, the row is covered and contributes to the count.

Finally, we maintain the maximum number of covered rows across all valid subsets.

## Go Solution

```go
func maximumRows(matrix [][]int, numSelect int) int {
    m := len(matrix)
    n := len(matrix[0])

    rowMasks := make([]int, m)

    for i := 0; i < m; i++ {
        mask := 0

        for j := 0; j < n; j++ {
            if matrix[i][j] == 1 {
                mask |= (1 << j)
            }
        }

        rowMasks[i] = mask
    }

    maxCovered := 0

    for selectedMask := 0; selectedMask < (1 << n); selectedMask++ {
        if bitCount(selectedMask) != numSelect {
            continue
        }

        coveredRows := 0

        for _, rowMask := range rowMasks {
            if (rowMask & selectedMask) == rowMask {
                coveredRows++
            }
        }

        if coveredRows > maxCovered {
            maxCovered = coveredRows
        }
    }

    return maxCovered
}

func bitCount(x int) int {
    count := 0

    for x > 0 {
        count += x & 1
        x >>= 1
    }

    return count
}
```

The Go implementation follows the same logic as the Python version but uses explicit integer manipulation.

Go does not provide a built in `bit_count()` method for integers in the same way Python does, so we implement a helper function to count set bits manually.

Slices are used to store row masks efficiently, and bitwise operations work naturally with Go integers.

There are no overflow concerns because the maximum number of columns is only `12`, so all masks easily fit within standard integer types.

## Edge Cases

### Rows Containing Only Zeros

Rows with no `1`s are automatically covered regardless of which columns are selected. This can easily be overlooked in naive implementations.

The bitmask representation handles this naturally because such rows produce a mask value of `0`.

The condition:

```
(row_mask & selected_mask) == row_mask
```

always evaluates to true when `row_mask == 0`.

### Rows Requiring More Columns Than Available

A row may contain more `1`s than `numSelect`. Such rows can never be covered because it is impossible to select enough required columns.

The implementation does not need a special case for this. The subset check automatically fails for every invalid selection.

### Selecting All Columns

When `numSelect == n`, there is only one valid selection, all columns.

In this situation, every row must be covered because all required columns are included.

The algorithm handles this correctly because the full bitmask is included during enumeration and satisfies every row mask.
