## Solution

## Problem Understanding

The problem asks us to find the length of the longest subsequence of a string such that every pair of adjacent characters in the subsequence differs by at most `k` in alphabetical order.

A subsequence does not require characters to remain contiguous. We are allowed to delete characters from the original string while preserving the relative order of the remaining characters.

For example, if `s = "acfgbd"` and `k = 2`, the subsequence `"acbd"` is valid because:

- `|a - c| = 2`
- `|c - b| = 1`
- `|b - d| = 2`

All adjacent differences are less than or equal to `2`.

However, the full string `"acfgbd"` is not ideal because:

- `|c - f| = 3`

which exceeds `k`.

The input consists of:

- A lowercase English string `s`
- An integer `k`

The output is a single integer representing the maximum possible length of an ideal subsequence.

The constraints are very important:

- `1 <= s.length <= 10^5`
- Only lowercase English letters are used
- `0 <= k <= 25`

The string can be very large, so any exponential or quadratic solution will be too slow. However, there are only 26 lowercase letters, which strongly suggests that we should exploit the small alphabet size.

Several edge cases are important:

- When `k = 0`, adjacent characters in the subsequence must be identical, so the answer becomes the maximum frequency of any character.
- When `k = 25`, every character can follow every other character, so the entire string is always valid.
- Strings with repeated characters can produce long subsequences even when transitions between different letters are restricted.
- Very short strings, such as length `1`, should correctly return `1`.

## Approaches

### Brute Force Approach

A brute force solution would generate every possible subsequence of the string and check whether it satisfies the ideal condition.

A string of length `n` has `2^n` subsequences. For each subsequence, we would verify whether every adjacent pair differs by at most `k`.

This approach is correct because it exhaustively checks all valid possibilities and returns the maximum valid length.

However, it is completely impractical for `n = 10^5`. Even for much smaller values, exponential growth makes the solution unusable.

### Key Insight

The important observation is that the validity of extending a subsequence depends only on its last character.

Suppose we are processing a character `c`. To build the best ideal subsequence ending at `c`, we only need to know:

- The longest ideal subsequence ending with each possible previous character
- Which of those characters can legally transition into `c`

Since there are only 26 lowercase letters, we can maintain a DP array of size 26.

Let:

- `dp[i]` represent the length of the longest ideal subsequence ending with character `chr(i + 'a')`

When processing a new character `c`:

1. Convert it to an index `current`
2. Look at all characters within distance `k`
3. Find the maximum subsequence length among them
4. Extend that subsequence by appending `c`

This reduces the problem from exponential complexity to linear complexity.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Generates every subsequence and validates each one |
| Optimal Dynamic Programming | O(n * 26) | O(26) | Maintains best subsequence length for each ending character |

Because `26` is constant, the optimal solution is effectively `O(n)`.

## Algorithm Walkthrough

1. Create a DP array of size 26, initialized to zero.

Each position represents the longest ideal subsequence ending with that character.

For example:

- `dp[0]` corresponds to `'a'`
- `dp[1]` corresponds to `'b'`
- and so on.
2. Iterate through each character in the string.

For the current character `c`, compute:

```
current = ord(c) - ord('a')
```
3. Determine which previous characters can transition into `c`.

Any character whose alphabetical distance from `c` is at most `k` is valid.

The valid range is:

```
[current - k, current + k]
```

Clamp the range to stay within `[0, 25]`.
4. Find the best subsequence length among all valid previous characters.

Scan through the allowed range and compute:

```
best_previous = max(dp[i])
```
5. Extend the subsequence.

Since we append the current character:

```
new_length = best_previous + 1
```
6. Update the DP entry for the current character.

```
dp[current] = max(dp[current], new_length)
```

The `max` is important because there may already be a better subsequence ending with the same character.
7. After processing all characters, return the maximum value in `dp`.

### Why it works

The algorithm maintains the invariant that `dp[i]` always stores the length of the longest ideal subsequence ending with character `i`.

When processing a new character, every valid ideal subsequence that can be extended must end with a character within distance `k`. By selecting the best among those candidates and appending the current character, we correctly compute the optimal subsequence ending at the current character.

Since every character is processed exactly once and every transition is considered, the final maximum value is the length of the longest ideal subsequence.

## Python Solution

```python
class Solution:
    def longestIdealString(self, s: str, k: int) -> int:
        dp = [0] * 26

        for char in s:
            current = ord(char) - ord('a')

            left = max(0, current - k)
            right = min(25, current + k)

            best_previous = 0

            for i in range(left, right + 1):
                best_previous = max(best_previous, dp[i])

            dp[current] = max(dp[current], best_previous + 1)

        return max(dp)
```

The solution begins by creating a fixed-size DP array with 26 entries, one for each lowercase English letter.

As we iterate through the string, we convert the current character into an index between `0` and `25`. We then determine the valid range of characters that can precede it in an ideal subsequence.

The algorithm scans that range to find the best previously computed subsequence length. Once found, we extend it by one to include the current character.

The DP entry for the current character is updated only if the new subsequence is better than the previous one ending with the same character.

Finally, the maximum value in the DP array is returned because the longest ideal subsequence may end with any character.

## Go Solution

```go
func longestIdealString(s string, k int) int {
	dp := make([]int, 26)

	for _, char := range s {
		current := int(char - 'a')

		left := current - k
		if left < 0 {
			left = 0
		}

		right := current + k
		if right > 25 {
			right = 25
		}

		bestPrevious := 0

		for i := left; i <= right; i++ {
			if dp[i] > bestPrevious {
				bestPrevious = dp[i]
			}
		}

		if bestPrevious+1 > dp[current] {
			dp[current] = bestPrevious + 1
		}
	}

	answer := 0

	for _, value := range dp {
		if value > answer {
			answer = value
		}
	}

	return answer
}
```

The Go implementation follows exactly the same logic as the Python version.

A slice of length 26 is used for the DP table. Since Go does not provide built-in `max` functions for integers, comparisons are handled manually with `if` statements.

Go strings are iterated as runes using `range`, which safely handles character processing. Since the input contains only lowercase English letters, rune conversion is straightforward.

No special handling for empty strings is necessary because the constraints guarantee that the string length is at least `1`.

## Edge Cases

### Edge Case 1, `k = 0`

When `k` equals zero, adjacent characters in the subsequence must be exactly the same.

A naive implementation might accidentally allow transitions between different characters due to incorrect range calculations.

The DP solution handles this naturally because the valid range becomes exactly one character wide. For example, when processing `'c'`, only `dp['c']` is considered.

This effectively turns the problem into finding the maximum frequency of any character.

### Edge Case 2, `k = 25`

When `k` equals 25, every lowercase letter can transition to every other lowercase letter.

An incorrect implementation might still unnecessarily restrict transitions.

The algorithm correctly expands the valid range to the entire alphabet, allowing every character to extend the longest subsequence seen so far. As a result, the answer becomes the full string length.

### Edge Case 3, Repeated Characters with Better Later Paths

Consider:

```
s = "abac"
k = 1
```

The character `'a'` appears multiple times. A buggy implementation might overwrite DP values incorrectly and lose the best subsequence.

The solution avoids this by using:

```
dp[current] = max(dp[current], best_previous + 1)
```

This guarantees that we never replace a better subsequence ending with the same character by a worse one.

### Edge Case 4, Characters Near Alphabet Boundaries

Characters like `'a'` and `'z'` require careful range handling.

For example:

```
current = 0
left = current - k
```

Without bounds checking, the range could become negative and cause invalid indexing.

The implementation safely clamps the range:

```
left = max(0, current - k)
right = min(25, current + k)
```

This guarantees all accesses remain valid.
