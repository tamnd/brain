## Solution

## Problem Understanding

This problem gives a **directed graph** with `n` nodes, where each node has **exactly one outgoing edge**. The graph is represented by an array `edges` of length `n`, where `edges[i]` indicates the node that node `i` points to. The task is to determine the node with the **highest edge score**, where the edge score of a node `i` is the sum of labels of all nodes pointing to `i`. If multiple nodes have the same highest score, we return the node with the **smallest index**.

In other words, for every node in the graph, we need to calculate the sum of all node indices that point to it. The problem guarantees that the graph is simple in the sense that no node points to itself (`edges[i] != i`), and the array has up to 100,000 elements, which requires an efficient solution.

Important edge cases include multiple nodes having the same maximum edge score, nodes that are **not pointed to by any other node**, or very large graphs where naive solutions could be too slow.

## Approaches

The **brute-force approach** would involve iterating over all nodes and for each node, scanning the entire array to sum up which nodes point to it. This approach is correct but inefficient because it requires checking `n` elements for each of the `n` nodes, resulting in O(n²) time complexity, which is impractical for `n = 10^5`.

The **optimal approach** uses a **hash map or array to accumulate edge scores**. Since each node has exactly one outgoing edge, we can iterate through the `edges` array once, and for each node `i`, add its index to the edge score of `edges[i]`. This reduces the time complexity to O(n) with a single pass through the array, and uses O(n) additional space for the edge score accumulation.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | For each node, sum indices of nodes pointing to it |
| Optimal | O(n) | O(n) | Single pass accumulation using array or hash map |

## Algorithm Walkthrough

1. Initialize an array `score` of length `n` with all zeros. Each index `i` in `score` will hold the sum of indices of nodes pointing to node `i`.
2. Iterate over each node `i` from `0` to `n-1`.
3. For each node `i`, add `i` to `score[edges[i]]`. This accumulates the edge scores efficiently.
4. After computing all scores, initialize `max_score` to -1 and `result_node` to -1.
5. Iterate over all nodes `0` to `n-1` and for each node, if `score[i]` is greater than `max_score`, update `max_score` and `result_node`. If `score[i]` is equal to `max_score`, retain the smaller index.
6. Return `result_node` as the node with the highest edge score.

**Why it works:** By using an array to store scores and accumulating contributions of each node, we ensure that every node’s score is correctly computed. Iterating once guarantees O(n) complexity, and checking for the maximum score ensures we return the smallest index in case of ties.

## Python Solution

```python
from typing import List

class Solution:
    def edgeScore(self, edges: List[int]) -> int:
        n = len(edges)
        score = [0] * n
        
        for i, target in enumerate(edges):
            score[target] += i
        
        max_score = -1
        result_node = -1
        
        for i, s in enumerate(score):
            if s > max_score:
                max_score = s
                result_node = i
        
        return result_node
```

The Python solution first initializes a list to store edge scores. Then, it iterates over each node, adding its index to the corresponding target node’s score. Finally, it finds the node with the maximum score, ensuring the smallest index is returned in case of a tie.

## Go Solution

```go
func edgeScore(edges []int) int {
    n := len(edges)
    score := make([]int, n)
    
    for i, target := range edges {
        score[target] += i
    }
    
    maxScore := -1
    resultNode := -1
    
    for i, s := range score {
        if s > maxScore {
            maxScore = s
            resultNode = i
        }
    }
    
    return resultNode
}
```

The Go solution mirrors the Python implementation, using a slice to accumulate scores. Integer overflow is not a concern in Go for the given constraints, and slices are used efficiently for O(n) access.

## Edge Cases

One important edge case is when **multiple nodes share the same maximum edge score**. In this situation, the implementation correctly returns the node with the smallest index due to the iteration order in the final loop.

Another edge case is when **a node has no incoming edges**. The algorithm handles this naturally since the initial score is zero, and only nodes with incoming edges will have non-zero scores.

A third edge case is the **maximum input size**. The algorithm scales linearly, and since the score array is preallocated, there are no performance issues, even with `n = 10^5`. This ensures the solution is both correct and efficient for all allowed inputs.
