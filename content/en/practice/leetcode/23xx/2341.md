## Solution

## Problem Understanding

This problem asks us to determine how many pairs of equal integers can be formed from a given integer array `nums` and how many integers remain unpaired afterward. Specifically, in one operation, we can select **two equal numbers** from the array and remove them. We repeat this operation until no more such pairs exist. The expected output is a two-element array: `answer[0]` is the total number of pairs formed, and `answer[1]` is the number of leftover integers.

The input array `nums` is **0-indexed**, has at least one element, and each element is between 0 and 100 inclusive. The constraints are relatively small (`1 <= nums.length <= 100`), which allows straightforward counting strategies without worrying about performance issues for large arrays.

Important edge cases include arrays with only one element (cannot form any pairs), arrays where all elements are identical (maximizing pairs), and arrays with all unique elements (no pairs formed). The problem guarantees the array is non-empty and that all integers are within a fixed range.

## Approaches

### Brute-Force Approach

A brute-force approach would repeatedly search the array for any two equal integers, remove them, and increment a pair counter. This approach is correct because it directly follows the problem's operation definition. However, each removal operation may require scanning the array to find a matching pair, which could lead to quadratic time complexity `O(n^2)` due to nested iterations when repeatedly searching for pairs.

### Optimal Approach

The optimal approach uses a **hash map (dictionary in Python or map in Go)** to count the frequency of each number in the array. The key insight is that for any number `x` with frequency `f`, the number of pairs we can form from `x` is `f // 2`, and the leftover is `f % 2`. By iterating over the frequency counts once, we can compute the total pairs and leftovers efficiently. This reduces the time complexity to linear `O(n)` and uses `O(n)` space for the hash map.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Repeatedly search and remove pairs |
| Optimal | O(n) | O(n) | Count frequencies and calculate pairs |

## Algorithm Walkthrough

1. Initialize a hash map to count occurrences of each integer in `nums`. This helps track how many times each value appears.
2. Iterate over each number in `nums` and update the hash map with the frequency of that number.
3. Initialize two counters: `pairs` for the total number of pairs formed and `leftovers` for remaining numbers.
4. For each number's frequency `f` in the hash map, calculate `f // 2` and add this to `pairs` since each integer contributes to that many pairs.
5. Calculate `f % 2` and add this to `leftovers` to account for any integers that could not form a pair.
6. Return `[pairs, leftovers]`.

**Why it works:** The hash map ensures that each number is counted exactly once, and integer division accurately calculates the maximum number of pairs per number. The remainder gives the exact leftover count. This guarantees correctness by treating each number independently.

## Python Solution

```python
from typing import List
from collections import Counter

class Solution:
    def numberOfPairs(self, nums: List[int]) -> List[int]:
        frequency = Counter(nums)
        pairs = 0
        leftovers = 0
        
        for count in frequency.values():
            pairs += count // 2
            leftovers += count % 2
        
        return [pairs, leftovers]
```

In this Python implementation, we use `collections.Counter` to efficiently count the frequency of each number. We then iterate over the counts to compute the total pairs and leftover numbers. The use of `//` ensures integer division for pair calculation, and `%` correctly accounts for leftover elements.

## Go Solution

```go
func numberOfPairs(nums []int) []int {
    frequency := make(map[int]int)
    pairs := 0
    leftovers := 0

    for _, num := range nums {
        frequency[num]++
    }

    for _, count := range frequency {
        pairs += count / 2
        leftovers += count % 2
    }

    return []int{pairs, leftovers}
}
```

In Go, we use a `map[int]int` to count the frequencies of numbers. Integer division `/` is used for pair calculation and modulo `%` for leftovers, similar to the Python solution. Go's slice is used to return the final result. Edge cases like empty slices are inherently handled because the loop simply does not run.

## Worked Examples

### Example 1: `nums = [1,3,2,1,3,2,2]`

| Step | Frequency Map | Pairs | Leftovers |
| --- | --- | --- | --- |
| Initial | {} | 0 | 0 |
| Count nums | {1:2, 2:3, 3:2} | 0 | 0 |
| 1 → 2 count | pairs += 1, leftovers += 0 | 1 | 0 |
| 2 → 3 count | pairs += 1, leftovers += 1 | 2 | 1 |
| 3 → 2 count | pairs += 1, leftovers += 0 | 3 | 1 |

Result: `[3, 1]`

### Example 2: `nums = [1,1]`

Frequency: `{1:2}` → pairs = 1, leftovers = 0 → `[1, 0]`

### Example 3: `nums = [0]`

Frequency: `{0:1}` → pairs = 0, leftovers = 1 → `[0, 1]`

## Edge Cases

One important edge case is when the array has only **one element**, like `[0]`. This can easily be mishandled if the implementation assumes at least two elements exist for forming a pair. The hash map approach naturally handles it because `1 // 2 = 0` and `1 % 2 = 1`.

Another edge case is when all elements are **identical**, like `[2]*100`. A naive implementation that removes elements one by one could be inefficient. Using frequency counts, we directly calculate `100 // 2 = 50` pairs and `100 % 2 = 0` leftovers, avoiding repeated iterations.

A third edge case is when the array has **all unique elements**, like `range(1,101)`. No pairs can be formed, and all elements are leftovers. The frequency count ensures each element contributes `0` pairs and `1` leftover without needing nested loops. This guarantees correct results even in worst-case diversity scenarios.
