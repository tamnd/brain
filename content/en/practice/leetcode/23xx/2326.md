# LeetCode 2326, Spiral Matrix IV

## Problem Understanding

This problem combines two common data structures and traversal patterns, linked lists and spiral matrix traversal.

We are given:

- Two integers, `m` and `n`, representing the number of rows and columns in a matrix.
- The head of a singly linked list.

Our task is to create an `m x n` matrix and fill it in spiral order using values from the linked list.

The spiral traversal starts at the top-left corner and proceeds clockwise:

1. Move left to right across the top row.
2. Move top to bottom along the right column.
3. Move right to left across the bottom row.
4. Move bottom to top along the left column.
5. Repeat inward until the matrix is completely processed.

As we traverse the matrix in this spiral pattern, we place linked list values into the visited cells one by one.

If the linked list runs out of values before the matrix is completely filled, all remaining cells must contain `-1`.

The constraints are important:

- `1 <= m * n <= 10^5`
- The linked list length is at most `m * n`

This tells us:

- We must process the matrix efficiently.
- An `O(m * n)` solution is expected.
- Excessively revisiting cells or repeatedly searching for the next spiral position would be too slow.

The linked list guarantees at least one node, so we never deal with an entirely empty list. However, the list may still be shorter than the matrix capacity.

Several edge cases are important:

- A single row matrix.
- A single column matrix.
- A linked list shorter than the matrix size.
- A linked list exactly equal to the matrix size.
- Very rectangular matrices such as `1 x n` or `m x 1`.

A naive implementation can easily make mistakes with boundary updates or overwrite cells during spiral traversal.

## Approaches

### Brute Force Approach

A brute force strategy would simulate movement cell by cell while repeatedly checking whether the next position is valid.

We could maintain:

- The current direction.
- A visited matrix.
- The current coordinates.

At every step, we attempt to move forward. If the next cell is outside the matrix or already visited, we rotate to the next direction.

This approach works correctly because spiral traversal is fundamentally directional simulation. By marking visited cells, we avoid overwriting previously filled positions.

However, this implementation introduces unnecessary overhead:

- Extra memory for the visited matrix.
- Additional boundary and visitation checks on every move.
- More complicated control logic.

Although the asymptotic complexity is still acceptable, the implementation is less elegant and more error-prone.

### Optimal Approach

The better solution uses the standard spiral boundary technique.

Instead of tracking visited cells, we maintain four boundaries:

- `top`
- `bottom`
- `left`
- `right`

Each traversal layer fills:

1. Top row
2. Right column
3. Bottom row
4. Left column

After completing one side, we shrink the corresponding boundary inward.

This works because spiral traversal naturally consumes the matrix layer by layer.

The key insight is that every cell belongs to exactly one spiral layer, so we can systematically process layers without needing a visited structure.

This approach is cleaner, simpler, and uses only the output matrix as storage.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m × n) | O(m × n) | Uses visited matrix and directional simulation |
| Optimal | O(m × n) | O(1), excluding output | Uses shrinking spiral boundaries |

## Algorithm Walkthrough

1. Initialize an `m x n` matrix filled with `-1`.

We prefill every position with `-1` because some cells may never receive linked list values if the list is too short.
2. Initialize four spiral boundaries:

- `top = 0`
- `bottom = m - 1`
- `left = 0`
- `right = n - 1`

These boundaries define the current unfilled rectangular region.
3. Continue processing while:

- The linked list still has nodes.
- The boundaries are still valid.
4. Traverse the top row from left to right.

For each column between `left` and `right`:

- Place the current linked list value into the matrix.
- Advance the linked list pointer.

After finishing the row, increment `top`.
5. Traverse the right column from top to bottom.

For each row between `top` and `bottom`:

- Fill the matrix cell.
- Advance the linked list pointer.

After finishing the column, decrement `right`.
6. Traverse the bottom row from right to left.

This step only occurs if `top <= bottom`.

For each column between `right` and `left` in reverse order:

- Fill the matrix cell.
- Advance the linked list pointer.

After finishing the row, decrement `bottom`.
7. Traverse the left column from bottom to top.

This step only occurs if `left <= right`.

For each row between `bottom` and `top` in reverse order:

- Fill the matrix cell.
- Advance the linked list pointer.

After finishing the column, increment `left`.
8. Repeat until either:

- The linked list is exhausted, or
- The spiral region becomes invalid.

### Why it works

The algorithm maintains the invariant that all cells outside the current boundaries have already been filled exactly once.

Each traversal step fills one side of the current spiral layer, then shrinks the corresponding boundary inward. Since boundaries only move inward and never overlap incorrectly, every matrix position is processed at most once.

Because linked list values are consumed sequentially during traversal, the final matrix exactly matches clockwise spiral order.

## Python Solution

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

from typing import List, Optional

class Solution:
    def spiralMatrix(
        self,
        m: int,
        n: int,
        head: Optional[ListNode]
    ) -> List[List[int]]:
        
        matrix = [[-1] * n for _ in range(m)]

        top = 0
        bottom = m - 1
        left = 0
        right = n - 1

        current = head

        while current and top <= bottom and left <= right:

            # Traverse left to right
            for col in range(left, right + 1):
                if not current:
                    return matrix

                matrix[top][col] = current.val
                current = current.next

            top += 1

            # Traverse top to bottom
            for row in range(top, bottom + 1):
                if not current:
                    return matrix

                matrix[row][right] = current.val
                current = current.next

            right -= 1

            # Traverse right to left
            if top <= bottom:
                for col in range(right, left - 1, -1):
                    if not current:
                        return matrix

                    matrix[bottom][col] = current.val
                    current = current.next

                bottom -= 1

            # Traverse bottom to top
            if left <= right:
                for row in range(bottom, top - 1, -1):
                    if not current:
                        return matrix

                    matrix[row][left] = current.val
                    current = current.next

                left += 1

        return matrix
```

The implementation begins by creating the result matrix filled with `-1`. This immediately handles all unused cells automatically.

The four boundaries define the current spiral layer. During each iteration of the loop, the algorithm processes one complete outer ring of the remaining submatrix.

The linked list pointer `current` advances every time a matrix cell is filled. Before every assignment, we check whether the linked list still contains nodes. If it does not, we return the partially filled matrix immediately.

The conditional checks before traversing the bottom row and left column are essential. Without them, the algorithm could revisit rows or columns in matrices with odd dimensions or narrow remaining regions.

The overall structure mirrors the conceptual spiral traversal exactly, which makes the implementation both efficient and easy to verify.

## Go Solution

```go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func spiralMatrix(m int, n int, head *ListNode) [][]int {

    matrix := make([][]int, m)

    for i := 0; i < m; i++ {
        matrix[i] = make([]int, n)

        for j := 0; j < n; j++ {
            matrix[i][j] = -1
        }
    }

    top := 0
    bottom := m - 1
    left := 0
    right := n - 1

    current := head

    for current != nil && top <= bottom && left <= right {

        // Traverse left to right
        for col := left; col <= right; col++ {
            if current == nil {
                return matrix
            }

            matrix[top][col] = current.Val
            current = current.Next
        }

        top++

        // Traverse top to bottom
        for row := top; row <= bottom; row++ {
            if current == nil {
                return matrix
            }

            matrix[row][right] = current.Val
            current = current.Next
        }

        right--

        // Traverse right to left
        if top <= bottom {
            for col := right; col >= left; col-- {
                if current == nil {
                    return matrix
                }

                matrix[bottom][col] = current.Val
                current = current.Next
            }

            bottom--
        }

        // Traverse bottom to top
        if left <= right {
            for row := bottom; row >= top; row-- {
                if current == nil {
                    return matrix
                }

                matrix[row][left] = current.Val
                current = current.Next
            }

            left++
        }
    }

    return matrix
}
```

The Go implementation follows the same logic as the Python solution.

One notable difference is matrix initialization. In Go, integer slices default to `0`, so we explicitly fill every position with `-1`.

The linked list pointer uses `nil` checks instead of Python's truthiness checks. Slice indexing and boundary handling remain essentially identical.

Since all values fit comfortably within Go's `int` range, integer overflow is not a concern.

## Edge Cases

### Single Row Matrix

When `m = 1`, the spiral traversal effectively becomes a simple left-to-right traversal.

This case can easily cause bugs if the algorithm still attempts bottom-row or left-column traversals after processing the top row. Without careful boundary checks, cells may be revisited or overwritten.

The implementation avoids this issue by checking:

```
if top <= bottom
```

before processing the bottom row.

### Single Column Matrix

When `n = 1`, traversal becomes top-to-bottom only.

This creates a similar risk where the algorithm might incorrectly revisit the same column during the left-column traversal phase.

The condition:

```
if left <= right
```

prevents duplicate traversal and guarantees each cell is filled once.

### Linked List Shorter Than Matrix Capacity

The linked list may end before all matrix positions are filled.

A naive implementation might continue traversal and attempt to dereference a null pointer or access `current.val` after the list is exhausted.

This implementation checks whether the linked list pointer is still valid before every assignment. As soon as the list ends, the algorithm immediately returns the matrix, leaving all remaining cells as `-1`.

### Odd-Dimension Matrices

Matrices such as `3 x 3` or `5 x 5` contain a center cell that belongs to the innermost spiral layer.

Incorrect boundary updates can cause the center cell to be skipped or filled multiple times.

The inward-shrinking boundary technique guarantees that every layer is processed exactly once, including the final single-cell layer in the center.
