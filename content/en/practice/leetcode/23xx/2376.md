## Solution

## Problem Understanding

The problem asks us to count how many integers in the range `[1, n]` contain only distinct digits. A number is considered special if no digit appears more than once in its decimal representation.

For example, the number `123` is special because all digits are unique. The number `121` is not special because the digit `1` appears twice. Similarly, `11` is not special because both digits are the same.

The input is a single integer `n`, and we must return the number of special integers between `1` and `n`, inclusive.

The constraint is important:

```
1 <= n <= 2 * 10^9
```

This means `n` can be as large as two billion, which contains up to 10 digits. A naive approach that checks every number individually would require iterating through billions of integers, which is far too slow.

The key challenge is efficiently counting numbers with distinct digits without enumerating all of them.

Several edge cases are important:

- Small numbers such as `n = 1` or `n = 5`, where every number is automatically special.
- Numbers containing repeated digits, such as `11`, `100`, or `131`.
- Large numbers close to the limit, where efficiency becomes critical.
- Numbers with leading zero considerations. For example, `0012` is not treated as a valid separate integer from `12`.

The problem guarantees that `n` is positive, so we never need to handle zero or negative input.

## Approaches

### Brute Force Approach

The most direct solution is to iterate through every integer from `1` to `n` and check whether all digits are distinct.

For each number:

1. Convert the number into digits.
2. Use a set to track digits already seen.
3. If a digit repeats, the number is not special.
4. Otherwise, increment the answer.

This approach is correct because it explicitly validates every candidate number.

However, it is too slow for large inputs. If `n = 2 * 10^9`, we would need billions of iterations, and each iteration performs digit processing. This is computationally infeasible.

### Optimal Approach, Digit Dynamic Programming with Combinatorics

The key observation is that we do not need to generate every number individually.

Instead, we can count how many valid numbers can be formed digit by digit.

This is a classic Digit DP problem:

- We process digits from left to right.
- We keep track of which digits have already been used.
- At each position, we count how many smaller digits can be placed.
- Once a digit is chosen, the remaining positions can be filled using permutations of unused digits.

The important insight is that the number of digits is very small, at most 10. This allows us to efficiently explore all valid digit combinations.

We split the problem into two parts:

1. Count all special numbers with fewer digits than `n`.
2. Count special numbers with the same number of digits as `n`, while respecting the upper bound.

This reduces the complexity dramatically.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(log n) | Checks every number individually |
| Optimal | O((log n)^2) | O(log n) | Uses digit counting and permutations |

## Algorithm Walkthrough

### Step 1: Convert the Number into Digits

We first convert `n` into a string or digit array so we can process it from left to right.

For example:

```
n = 135
digits = ['1', '3', '5']
```

This allows us to compare prefixes against the limit.

### Step 2: Count All Valid Numbers with Fewer Digits

Any number with fewer digits than `n` is automatically less than `n`.

For each possible length `L` from `1` to `len(n) - 1`:

1. The first digit can be chosen from `1` through `9`.
2. Remaining digits can be chosen from unused digits.
3. We compute permutations.

For example, for length `3`:

- First digit: `9` choices
- Second digit: `9` choices
- Third digit: `8` choices

Total:

```
9 × 9 × 8
```

We compute this systematically.

### Step 3: Process Numbers with the Same Length

Now we count numbers with the same number of digits as `n`.

We process digits left to right.

At each position:

1. Try placing a smaller digit than the current digit of `n`.
2. Skip digits already used.
3. For every valid smaller digit:

- Count all possible ways to fill the remaining positions using unused digits.
4. Then place the actual digit from `n` and continue.

If the current digit was already used before, we stop immediately because any continuation would contain repeated digits.

### Step 4: Add One if `n` Itself is Special

If we successfully process all digits without repetition, then `n` itself is special, so we add one to the answer.

### Step 5: Return the Result

The accumulated total is the number of special integers in `[1, n]`.

### Why it works

The algorithm systematically counts every valid integer exactly once.

- Numbers with fewer digits are counted independently using permutations.
- Numbers with the same length are constructed digit by digit while respecting the upper bound.
- The used-digit set guarantees uniqueness.
- Prefix comparison guarantees we never exceed `n`.

Because every valid number belongs to exactly one counted category, the final answer is correct.

## Python Solution

```python
class Solution:
    def countSpecialNumbers(self, n: int) -> int:
        digits = list(map(int, str(n)))
        length = len(digits)

        def permutations(m: int, k: int) -> int:
            result = 1
            for i in range(k):
                result *= (m - i)
            return result

        answer = 0

        # Count numbers with fewer digits
        for digits_count in range(1, length):
            answer += 9 * permutations(9, digits_count - 1)

        used_digits = set()

        # Count numbers with same length
        for index in range(length):
            current_digit = digits[index]

            for digit in range(0 if index > 0 else 1, current_digit):
                if digit in used_digits:
                    continue

                remaining_positions = length - index - 1
                remaining_digits = 10 - (index + 1)

                answer += permutations(
                    remaining_digits,
                    remaining_positions
                )

            if current_digit in used_digits:
                return answer

            used_digits.add(current_digit)

        return answer + 1
```

The implementation follows the exact algorithm described earlier.

The `permutations()` helper computes:

```python
m × (m - 1) × (m - 2) × ...
```

This represents the number of ways to fill remaining positions using distinct digits.

The first loop counts all special numbers with fewer digits than `n`.

The second loop handles numbers with the same length. At each digit position, we try all smaller valid digits and count how many completions are possible.

The `used_digits` set ensures we never reuse digits.

If we encounter a repeated digit while processing `n` itself, we stop immediately because all continuations would be invalid.

Finally, if every digit is unique, we add one because `n` itself is special.

## Go Solution

```go
func countSpecialNumbers(n int) int {
	digits := []int{}
	temp := n

	for temp > 0 {
		digits = append([]int{temp % 10}, digits...)
		temp /= 10
	}

	length := len(digits)

	permutations := func(m int, k int) int {
		result := 1
		for i := 0; i < k; i++ {
			result *= (m - i)
		}
		return result
	}

	answer := 0

	// Count numbers with fewer digits
	for digitsCount := 1; digitsCount < length; digitsCount++ {
		answer += 9 * permutations(9, digitsCount-1)
	}

	used := make(map[int]bool)

	// Count numbers with same length
	for index := 0; index < length; index++ {
		currentDigit := digits[index]

		start := 0
		if index == 0 {
			start = 1
		}

		for digit := start; digit < currentDigit; digit++ {
			if used[digit] {
				continue
			}

			remainingPositions := length - index - 1
			remainingDigits := 10 - (index + 1)

			answer += permutations(
				remainingDigits,
				remainingPositions,
			)
		}

		if used[currentDigit] {
			return answer
		}

		used[currentDigit] = true
	}

	return answer + 1
}
```

The Go implementation mirrors the Python solution closely.

A `map[int]bool` is used instead of a Python set to track used digits.

Go does not provide built in permutation utilities, so the helper function computes permutations manually.

All calculations safely fit inside standard Go integers because the maximum possible count is below `10!`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O((log n)^2) | At most 10 digits, with nested digit processing |
| Space | O(log n) | Used digit tracking and digit storage |

The number of digits in `n` is at most 10 because `n <= 2 * 10^9`.

The outer loops operate over digit positions, and the inner work is bounded by 10 possible digits. Therefore, the runtime is effectively constant in practice, though formally expressed as `O((log n)^2)`.

The space usage is also tiny because we only store digit arrays and a used-digit set.

## Edge Cases

### Numbers with Repeated Prefix Digits

Inputs like `131` or `1223` are tricky because repetition may occur after several valid digits.

For example, when processing `131`, the algorithm correctly counts all valid numbers beginning with smaller prefixes before stopping once the second `1` repeats. This avoids overcounting invalid continuations.

### Numbers Containing Zero

Zero requires careful handling because leading zeros are not allowed.

For example, in `1023`, the digit `0` is perfectly valid after the first position. The implementation handles this by allowing digits from `0` onward only for non leading positions.

### Maximum Constraint Values

Large inputs near `2 * 10^9` would make brute force impossible.

The digit based counting approach only depends on the number of digits, not the magnitude itself. Since there are at most 10 digits, the implementation remains extremely fast even at the upper limit.
