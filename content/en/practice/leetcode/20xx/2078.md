## Solution

## Problem Understanding

This problem asks us to find the **maximum distance** between any two houses that are painted **different colors** along a street. We are given a **0-indexed array `colors`** of length `n`, where `colors[i]` represents the color of the `ith` house. The distance between two houses is measured as the absolute difference of their indices: `abs(i - j)`. The goal is to identify the pair of houses with different colors that are furthest apart.

The constraints provide important guidance. Since `2 <= n <= 100`, the input size is small, which means even a brute-force solution may work within time limits. Additionally, all colors are between `0` and `100`, and the problem guarantees that **at least two houses have different colors**, so we do not need to handle the degenerate case of all houses having the same color.

Important edge cases to consider include scenarios where the first and last houses are the ones with different colors, or when there are clusters of identical colors at either end of the array.

## Approaches

### Brute Force

The simplest approach is to consider **every possible pair of houses** `(i, j)` and check whether their colors are different. If they are, compute the distance `abs(i - j)` and track the maximum distance found.

While this approach works, it is **inefficient** because it requires checking all `n*(n-1)/2` pairs. For `n = 100`, this results in 4950 iterations, which is feasible for this problem but unnecessary.

### Optimal Approach

The key observation is that the **maximum distance** will always involve either the **first house** or the **last house** paired with some house of a different color. This is because choosing any two houses in the middle will always result in a smaller distance than one of the edges.

Therefore, the optimal strategy is:

1. Compare the **first house** with houses from the **end** backward until a house with a different color is found.
2. Compare the **last house** with houses from the **start** forward until a house with a different color is found.
3. Return the **larger distance** from the two comparisons.

This approach only requires **two linear scans**, resulting in O(n) time complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check all pairs of houses for different colors |
| Optimal | O(n) | O(1) | Scan from both ends for maximum distance with different colors |

## Algorithm Walkthrough

1. Initialize a variable `n` to the length of the array `colors`.
2. Initialize `max_dist` to 0, which will store the maximum distance found.
3. Check the distance from the **first house** to the furthest house with a different color:

- Start scanning from the last index backward.
- Stop at the first house where the color differs from `colors[0]`.
- Update `max_dist` to `n - 1 - index_found`.
4. Check the distance from the **last house** to the furthest house with a different color:

- Start scanning from index 0 forward.
- Stop at the first house where the color differs from `colors[n-1]`.
- Update `max_dist` to the maximum of its current value and `index_found`.
5. Return `max_dist`.

**Why it works:** By focusing on the edges, we guarantee that the distance is maximized because any internal pair will have a smaller index difference than at least one of the edge-to-different-color comparisons. This approach leverages the problem's linear geometry and reduces redundant comparisons.

## Python Solution

```python
from typing import List

class Solution:
    def maxDistance(self, colors: List[int]) -> int:
        n = len(colors)
        max_dist = 0
        
        # Check distance from first house
        for i in range(n-1, -1, -1):
            if colors[i] != colors[0]:
                max_dist = n - 1 - i if n - 1 - i > max_dist else max_dist
                break
        
        # Check distance from last house
        for i in range(n):
            if colors[i] != colors[-1]:
                max_dist = i if i > max_dist else max_dist
                break
        
        return max_dist
```

The Python implementation first scans backward from the end to find a house with a different color from the first house, then scans forward from the start to find a house with a different color from the last house. It updates `max_dist` with the larger distance of the two, and returns it.

## Go Solution

```go
func maxDistance(colors []int) int {
    n := len(colors)
    maxDist := 0

    // Check distance from first house
    for i := n - 1; i >= 0; i-- {
        if colors[i] != colors[0] {
            if n-1-i > maxDist {
                maxDist = n - 1 - i
            }
            break
        }
    }

    // Check distance from last house
    for i := 0; i < n; i++ {
        if colors[i] != colors[n-1] {
            if i > maxDist {
                maxDist = i
            }
            break
        }
    }

    return maxDist
}
```

The Go version mirrors the Python logic. It uses standard indexing for slices and integer comparisons, and handles edge cases without any special nil or empty slice checks since `n >= 2` is guaranteed.

## Worked Examples

**Example 1:** `colors = [1,1,1,6,1,1,1]`

Scan from the end backward: `colors[6] = 1` matches, `colors[5] = 1` matches, `colors[4] = 1` matches, `colors[3] = 6` differs → distance = `3 - 0 = 3`.

Scan from the start forward: `colors[0] = 1` matches, `colors[1] = 1`, ..., `colors[3] = 6` differs → distance = `6 - 3 = 3`. Maximum distance = 3.

**Example 2:** `colors = [1,8,3,8,3]`

Scan from the end backward: `colors[4] = 3` differs from `colors[0] = 1` → distance = `4 - 0 = 4`.

Scan from start forward: `colors[0] = 1`, `colors[1] = 8`, `colors[2] = 3`, `colors[3] = 8`, `colors[4] = 3` → distance = 4. Maximum distance = 4.

**Example 3:** `colors = [0,1]`

Scan from end backward: `colors[1] = 1` differs from `colors[0] = 0` → distance = 1.

Scan from start forward: `colors[0] = 0` differs from `colors[1] = 1` → distance = 1. Maximum distance = 1.

The reasoning is straightforward: we only need to track distances and indices, no additional data structures are required.
