## Solution

## Problem Understanding

This problem asks us to manipulate a singly linked list by reversing certain groups of nodes. The nodes are divided sequentially into groups where the group sizes follow the natural number sequence: the first group has 1 node, the second group has 2 nodes, the third group has 3 nodes, and so on. If the list ends before a group is fully formed, the final group simply contains the remaining nodes.

The key operation is **conditional reversal**: only reverse the nodes of a group if the group has an **even length**. After processing all groups, we return the modified head of the linked list.

For inputs, we are given the head of a linked list where each node has a value between 0 and 105, and the number of nodes can be as high as 100,000. This means our solution must be efficient and avoid unnecessary data structure overhead.

Edge cases to consider include:

- A single-node list (first group is 1, no reversal happens).
- A list where the last group is smaller than expected (we must reverse it if it is even).
- Groups that transition between odd and even lengths.

## Approaches

The **brute-force approach** would be to traverse the linked list and collect nodes in a list or array for each group, reverse the group if it is even, and then reconnect nodes. This works correctly but requires extra O(n) space, which is not necessary because we can manipulate the list in-place.

The **optimal approach** is to traverse the linked list iteratively, keeping track of the current group size and pointers to the previous and next nodes. For each group, we first determine its actual length by counting nodes. If the length is even, we reverse the nodes in-place; if not, we simply advance the pointer. This uses O(1) extra space and processes each node exactly once, giving O(n) time complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Collect nodes in an array to reverse, reconnect |
| Optimal | O(n) | O(1) | Iteratively reverse even-length groups in-place |

## Algorithm Walkthrough

1. Initialize `current` to the head of the list, `prev_group_end` to `None`, and `group_size` to 1.
2. While `current` is not `None`, count the number of nodes in the current group by iterating up to `group_size` nodes or until the end of the list. Store this count as `actual_size`.
3. If `actual_size` is even, reverse the group in-place. To reverse, keep track of `prev`, `node`, and `next_node` pointers and iterate through the group nodes, adjusting `next` pointers.
4. Connect the reversed (or untouched) group with `prev_group_end.next` and update `prev_group_end` to the end of the group.
5. Move `current` to the start of the next group. Increment `group_size` by 1.
6. Repeat until all nodes are processed.

**Why it works:** At each iteration, we correctly isolate a group, measure its actual length, and perform an in-place reversal only if the length is even. The linkage between groups is always maintained, ensuring the final list is correctly formed.

## Python Solution

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def reverseEvenLengthGroups(self, head: Optional[ListNode]) -> Optional[ListNode]:
        def reverse(start: ListNode, end: ListNode) -> ListNode:
            prev, curr = None, start
            while curr != end:
                nxt = curr.next
                curr.next = prev
                prev = curr
                curr = nxt
            return prev  # New head of the reversed segment

        dummy = ListNode(0, head)
        prev_group_end = dummy
        current = head
        group_size = 1

        while current:
            group_start = current
            count = 0
            node = current
            while node and count < group_size:
                node = node.next
                count += 1

            next_group_start = node

            if count % 2 == 0:
                new_group_head = reverse(group_start, next_group_start)
                prev_group_end.next = new_group_head
                group_start.next = next_group_start
                prev_group_end = group_start
            else:
                prev_group_end = group_start
                for _ in range(count - 1):
                    prev_group_end = prev_group_end.next

            current = next_group_start
            group_size += 1

        return dummy.next
```

The Python implementation uses a helper function `reverse` to reverse a segment between `start` and `end`. The main loop iterates through each group, counts its actual length, reverses if needed, and updates pointers to maintain the correct linkage. The `dummy` node simplifies edge cases at the head.

## Go Solution

```go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func reverseEvenLengthGroups(head *ListNode) *ListNode {
    dummy := &ListNode{Next: head}
    prevGroupEnd := dummy
    current := head
    groupSize := 1

    for current != nil {
        groupStart := current
        count := 0
        node := current
        for node != nil && count < groupSize {
            node = node.Next
            count++
        }
        nextGroupStart := node

        if count%2 == 0 {
            var prev *ListNode
            curr := groupStart
            for curr != nextGroupStart {
                nxt := curr.Next
                curr.Next = prev
                prev = curr
                curr = nxt
            }
            prevGroupEnd.Next = prev
            groupStart.Next = nextGroupStart
            prevGroupEnd = groupStart
        } else {
            prevGroupEnd = groupStart
            for i := 1; i < count; i++ {
                prevGroupEnd = prevGroupEnd.Next
            }
        }

        current = nextGroupStart
        groupSize++
    }

    return dummy.Next
}
```

The Go solution closely mirrors the Python version. The main differences are syntax-related: `nil` is used instead of `None`, and explicit pointer manipulation is required. We use a dummy node to simplify head changes and handle in-place reversal efficiently.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited once; group counting and reversal are linear in nodes processed. |
| Space | O(1) | No extra data structures are used; all operations are in-place except the dummy node. |

The reasoning is that each node contributes to at most one reversal and one group traversal, ensuring linear time. Using only a few pointers maintains constant extra space.
