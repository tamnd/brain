## Solution

## Problem Understanding

The problem asks us to find the smallest index `i` in a 0-indexed integer array `nums` such that the remainder when `i` is divided by 10 equals the value at that index, i.e., `i mod 10 == nums[i]`. If no such index exists, we return `-1`.

The input `nums` is an array of integers where each element is between `0` and `9`, inclusive. The length of `nums` is at least 1 and at most 100, which is relatively small. This means that a straightforward solution iterating through all indices is feasible and efficient.

Key edge cases to consider include an array where every element matches its modulo condition, an array where no element matches, and arrays with minimal and maximal lengths. The problem guarantees that `nums[i]` is always a single-digit integer, which ensures the modulo comparison is valid for every index.

## Approaches

### Brute Force

The brute-force solution involves iterating through the array from the first index to the last. At each index `i`, we calculate `i % 10` and compare it with `nums[i]`. The first index that satisfies this condition is returned. If the iteration completes without finding a valid index, we return `-1`.

Given the constraints of `nums.length <= 100`, this approach is acceptable, as it only requires a single pass through the array.

### Optimal Insight

The brute-force solution is already optimal for this problem. Because the array length is small and the operation per index is constant time, no further optimization is necessary. The key observation is that the modulo operation is very cheap, and we only need the first index that satisfies the condition, making a single-pass linear scan both correct and efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Iterate through the array and check i mod 10 == nums[i] |
| Optimal | O(n) | O(1) | Same as brute force; already efficient for given constraints |

## Algorithm Walkthrough

1. Initialize a loop variable `i` starting from `0` and iterate through each index of `nums` up to `len(nums) - 1`.
2. For each index `i`, calculate `i % 10`.
3. Compare the result of `i % 10` with `nums[i]`.
4. If they are equal, return `i` immediately since it is the smallest index satisfying the condition.
5. If the loop completes without finding a valid index, return `-1`.

**Why it works:** The algorithm checks every index in ascending order, ensuring that the first index meeting the condition is returned. The modulo operation is correctly defined for non-negative integers, and the loop guarantees we examine all possible candidates.

## Python Solution

```python
from typing import List

class Solution:
    def smallestEqual(self, nums: List[int]) -> int:
        for i, num in enumerate(nums):
            if i % 10 == num:
                return i
        return -1
```

This Python implementation uses `enumerate` to iterate over both the index and value simultaneously. The modulo check is applied directly, and the first matching index is returned. If no index satisfies the condition, the function returns `-1`.

## Go Solution

```go
func smallestEqual(nums []int) int {
    for i, num := range nums {
        if i % 10 == num {
            return i
        }
    }
    return -1
}
```

The Go version mirrors the Python solution. We use a range-based loop to iterate over indices and values. The modulo comparison is the same, and the first matching index is returned. The function handles empty slices implicitly since the loop will not execute, returning `-1`.

## Edge Cases

One important edge case is a single-element array. If the element equals `0`, it matches immediately. If it is any other number, the function must correctly return `-1`.

Another edge case involves arrays where multiple indices satisfy the modulo condition. The algorithm must return the smallest index, not the largest or any arbitrary matching index. The linear scan ensures the first match is selected.

A third edge case involves the last element of a large array. For example, if the last element at index 99 is `9`, it satisfies `99 % 10 == 9`. The algorithm correctly iterates through all indices and returns this value only if no smaller indices match first.
