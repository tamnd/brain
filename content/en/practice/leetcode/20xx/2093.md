[LeetCode Problem 2093](https://leetcode.com/problems/minimum-cost-to-reach-city-with-discounts/)

**Difficulty:** 🟡 Medium  
**Topics:** Graph Theory, Heap (Priority Queue), Shortest Path

## Solution

## Problem Understanding

This problem asks us to find the minimum cost to travel from city `0` to city `n-1` in a network of `n` cities connected by highways, where each highway has a toll cost. The twist is that you have a limited number of discounts that can halve the cost of traveling across a highway, but each highway can only use at most one discount.

The input consists of the number of cities `n`, a list of `highways` represented as `[city1, city2, toll]`, and the number of available `discounts`. The output is the minimum total cost to reach city `n-1` from city `0`, or `-1` if there is no possible path.

Key points to note are that `n` can be up to 1000, the number of highways can also be up to 1000, and the number of discounts can be up to 500. This suggests that a brute-force exploration of all paths is infeasible because the number of paths grows exponentially. The graph is undirected, and tolls are non-negative integers.

Important edge cases include situations where `discounts = 0`, where there is no path connecting city `0` to `n-1`, or when multiple discounts could optimally be applied to different paths.

## Approaches

The brute-force approach would try every path from city `0` to city `n-1`, applying discounts in every possible way. This guarantees the correct answer because it exhaustively checks all possibilities. However, with up to 1000 cities and 1000 edges, the number of paths grows exponentially, making this approach far too slow.

The key observation is that this problem can be reduced to a variant of Dijkstra's shortest path algorithm where the state of the node is not just the city but also the number of remaining discounts. Each state can be represented as `(current_city, discounts_left)`. We can use a priority queue to always expand the cheapest path next. When traversing a highway, we can either use a discount if available or pay full price. This ensures that we explore all optimal possibilities without redundant work.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^m) | O(n) | Try all paths with all discount applications; infeasible for large n, m |
| Optimal | O((n * d + m) * log(n * d)) | O(n * d + m) | Modified Dijkstra with state `(city, discounts_left)` using a min-heap |

Here `n` is the number of cities, `m` is the number of highways, and `d` is the number of discounts.

## Algorithm Walkthrough

1. Construct an adjacency list `graph` where each city maps to a list of tuples `(neighbor, toll)`. This allows efficient access to connected cities.
2. Initialize a 2D array `min_cost[city][discounts_left]` to keep track of the minimum cost to reach a city with a certain number of discounts remaining. Fill it with infinity initially.
3. Use a priority queue (min-heap) to store tuples `(current_total_cost, current_city, discounts_left)`. Start by pushing `(0, 0, discounts)` into the heap.
4. While the heap is not empty, pop the state with the smallest `current_total_cost`.
5. For each neighbor city connected by a highway, calculate two possible costs: `cost_without_discount` and, if `discounts_left > 0`, `cost_with_discount`.
6. If the new cost is smaller than `min_cost[neighbor][new_discounts_left]`, update `min_cost` and push the new state into the heap.
7. Continue until all reachable states are processed.
8. Return the smallest value in `min_cost[n-1][*]` across all possible remaining discounts. If all are infinity, return `-1`.

**Why it works:** By maintaining the state `(city, discounts_left)` and always expanding the cheapest current cost first, this ensures that we explore paths in increasing order of cost. This guarantees the first time we reach a state with minimal cost, it is optimal for that number of discounts remaining.

## Python Solution

```python
from typing import List
import heapq

class Solution:
    def minimumCost(self, n: int, highways: List[List[int]], discounts: int) -> int:
        graph = [[] for _ in range(n)]
        for u, v, toll in highways:
            graph[u].append((v, toll))
            graph[v].append((u, toll))

        min_cost = [[float('inf')] * (discounts + 1) for _ in range(n)]
        min_cost[0][discounts] = 0
        heap = [(0, 0, discounts)]  # (cost, city, discounts_left)

        while heap:
            cost, city, d_left = heapq.heappop(heap)
            if cost > min_cost[city][d_left]:
                continue
            for neighbor, toll in graph[city]:
                # Travel without discount
                if cost + toll < min_cost[neighbor][d_left]:
                    min_cost[neighbor][d_left] = cost + toll
                    heapq.heappush(heap, (cost + toll, neighbor, d_left))
                # Travel with discount if available
                if d_left > 0:
                    discounted_toll = toll // 2
                    if cost + discounted_toll < min_cost[neighbor][d_left - 1]:
                        min_cost[neighbor][d_left - 1] = cost + discounted_toll
                        heapq.heappush(heap, (cost + discounted_toll, neighbor, d_left - 1))

        result = min(min_cost[n-1])
        return result if result != float('inf') else -1
```

This Python solution constructs the graph using an adjacency list. It initializes the minimum cost table, then applies a Dijkstra-style traversal using a heap. It considers both traveling with and without a discount, updating the heap only when a cheaper path is found. The final result checks all possible discount usages at the destination.

## Go Solution

```go
package main

import (
    "container/heap"
    "math"
)

type State struct {
    cost int
    city int
    discountsLeft int
}

type MinHeap []State

func (h MinHeap) Len() int { return len(h) }
func (h MinHeap) Less(i, j int) bool { return h[i].cost < h[j].cost }
func (h MinHeap) Swap(i, j int) { h[i], h[j] = h[j], h[i] }
func (h *MinHeap) Push(x interface{}) { *h = append(*h, x.(State)) }
func (h *MinHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

func minimumCost(n int, highways [][]int, discounts int) int {
    graph := make([][][2]int, n)
    for _, h := range highways {
        u, v, toll := h[0], h[1], h[2]
        graph[u] = append(graph[u], [2]int{v, toll})
        graph[v] = append(graph[v], [2]int{u, toll})
    }

    inf := math.MaxInt64
    minCost := make([][]int, n)
    for i := 0; i < n; i++ {
        minCost[i] = make([]int, discounts+1)
        for j := 0; j <= discounts; j++ {
            minCost[i][j] = inf
        }
    }

    minCost[0][discounts] = 0
    h := &MinHeap{}
    heap.Init(h)
    heap.Push(h, State{0, 0, discounts})

    for h.Len() > 0 {
        s := heap.Pop(h).(State)
        if s.cost > minCost[s.city][s.discountsLeft] {
            continue
        }
        for _, edge := range graph[s.city] {
            neighbor, toll := edge[0], edge[1]
            // Without discount
            if s.cost + toll < minCost[neighbor][s.discountsLeft] {
                minCost[neighbor][s.discountsLeft] = s.cost + toll
                heap.Push(h, State{s.cost + toll, neighbor, s.discountsLeft})
            }
            // With discount
            if s.discountsLeft > 0 {
                discountedToll := toll / 2
                if s.cost + discountedToll < minCost[neighbor][s.discountsLeft-1] {
                    minCost[neighbor][s.discountsLeft-1] = s.cost + discountedToll
                    heap.Push(h, State{s.cost + discountedToll, neighbor, s.discountsLeft-1})
                }
            }
        }
    }

    result := inf
    for _, c := range minCost[n-1] {
        if c < result {
            result = c
        }
    }
    if result == inf {
        return -1
    }
    return result
}
```

The Go version mirrors the Python solution, using a min-heap implemented with `container/heap` and struct `State` to track `(cost, city, discountsLeft)`. It handles initialization, graph building, and heap updates similarly, with attention to Go slices and integer overflow.
