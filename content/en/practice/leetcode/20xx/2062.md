# LeetCode 2062 - Count Vowel Substrings of a String

## Problem Understanding

The problem asks us to count how many substrings of a given string satisfy two conditions simultaneously:

1. The substring contains only vowels.
2. The substring contains all five vowels, `'a'`, `'e'`, `'i'`, `'o'`, and `'u'`, at least once.

A substring must be contiguous, meaning the characters must appear next to each other in the original string.

The input is a single string `word` containing lowercase English letters. The output is an integer representing the total number of valid vowel substrings.

A key detail is that a substring becomes invalid immediately if it contains any consonant. This means valid substrings can only exist entirely inside continuous vowel segments.

The constraints are very small:

- `1 <= word.length <= 100`

Because the input size is tiny, even an `O(n^2)` or `O(n^3)` solution would pass comfortably. However, it is still valuable to design a clean and efficient algorithm.

There are several important edge cases to keep in mind:

- Strings with no vowels at all, such as `"bcdfg"`, should return `0`.
- Strings containing vowels but missing one or more required vowels, such as `"aeio"`, should also return `0`.
- Strings where every character is a vowel, such as `"aeiouu"`, can contain multiple overlapping valid substrings.
- Consonants split the string into separate regions, so substrings cannot cross over consonants.
- Repeated vowels can create many valid substrings ending at different positions.

Understanding these details helps avoid incorrect counting logic.

## Approaches

### Brute Force Approach

The most direct solution is to generate every possible substring and check whether it is a valid vowel substring.

For every starting index `i`, we generate every ending index `j`. For each substring `word[i:j+1]`, we verify:

1. Every character is a vowel.
2. All five vowels appear at least once.

To check this, we can scan the substring character by character, inserting vowels into a set. If we encounter a consonant, the substring is invalid. At the end, if the set contains exactly five vowels, we count the substring.

This approach is correct because it explicitly examines every possible substring and validates it according to the problem definition.

However, the runtime becomes inefficient because:

- There are `O(n^2)` substrings.
- Each substring validation can take `O(n)` time.

This leads to an `O(n^3)` solution.

Although `n <= 100` makes this acceptable, we can do better.

### Optimal Approach

The key observation is that valid substrings must consist entirely of vowels. Therefore, whenever we encounter a consonant, we know any substring crossing that character cannot be valid.

Instead of repeatedly rescanning substrings, we can expand substrings incrementally.

For each starting index:

- Extend the substring character by character.
- Stop immediately when a consonant appears.
- Maintain a set of vowels currently present.
- Whenever the set contains all five vowels, increment the answer.

This avoids repeatedly rescanning the same substring contents.

The resulting complexity becomes `O(n^2)`.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Generates every substring and rescans each one |
| Optimal | O(n²) | O(1) | Expands substrings incrementally while tracking vowels |

## Algorithm Walkthrough

### Optimal Algorithm

1. Create a set containing all vowels: `{'a', 'e', 'i', 'o', 'u'}`.
2. Initialize a variable `result` to store the number of valid vowel substrings.
3. Iterate through every possible starting index `start` in the string.
4. For each starting index, create an empty set called `seen_vowels`.
5. Extend the substring one character at a time using another index `end`.
6. At each step:

- If `word[end]` is not a vowel, stop extending this substring because any longer substring would also contain a consonant.
- Otherwise, add the vowel to `seen_vowels`.
7. After adding the vowel:

- If `seen_vowels` contains all five vowels, increment `result`.
8. Continue until all starting positions have been processed.
9. Return `result`.

### Why it works

The algorithm examines every vowel-only substring exactly once. For a fixed starting position, the substring expands continuously until a consonant appears. The `seen_vowels` set always contains exactly the vowels present in the current substring. Whenever the set size becomes five, the substring satisfies the problem conditions, so counting it is correct. Since every valid substring has a unique start and end index, every valid substring is counted exactly once.

## Python Solution

```python
class Solution:
    def countVowelSubstrings(self, word: str) -> int:
        vowels = {'a', 'e', 'i', 'o', 'u'}
        result = 0
        n = len(word)

        for start in range(n):
            seen_vowels = set()

            for end in range(start, n):
                current_char = word[end]

                if current_char not in vowels:
                    break

                seen_vowels.add(current_char)

                if len(seen_vowels) == 5:
                    result += 1

        return result
```

The implementation begins by defining the set of valid vowels. Using a set allows constant time membership checks.

The outer loop chooses every possible starting index for a substring. For each start position, we initialize an empty set to track which vowels have appeared so far.

The inner loop extends the substring one character at a time. If a consonant is encountered, the loop breaks immediately because no longer substring from the same start position can remain vowel-only.

Whenever a vowel is added to the set, we check whether all five vowels are present by checking `len(seen_vowels) == 5`. If true, the current substring is valid and contributes to the answer.

The algorithm naturally handles overlapping substrings because every `(start, end)` pair is considered independently.

## Go Solution

```go
package main

func countVowelSubstrings(word string) int {
    vowels := map[byte]bool{
        'a': true,
        'e': true,
        'i': true,
        'o': true,
        'u': true,
    }

    result := 0
    n := len(word)

    for start := 0; start < n; start++ {
        seenVowels := make(map[byte]bool)

        for end := start; end < n; end++ {
            currentChar := word[end]

            if !vowels[currentChar] {
                break
            }

            seenVowels[currentChar] = true

            if len(seenVowels) == 5 {
                result++
            }
        }
    }

    return result
}
```

The Go implementation follows the same logic as the Python version.

A `map[byte]bool` is used instead of a Python set. Since the string contains only lowercase English letters, indexing the string as bytes is completely safe.

The `seenVowels` map tracks distinct vowels currently present in the substring. The built-in `len()` function on the map tells us how many unique vowels have been seen so far.

No special handling for integer overflow is required because the maximum number of substrings for `n = 100` is small.

## Edge Cases

### Strings Without Any Valid Vowel Block

A string like `"bbbb"` contains no vowels at all. A careless implementation might still attempt unnecessary substring checks or fail to terminate correctly. In this implementation, the inner loop immediately breaks whenever a consonant is encountered, so these cases are handled efficiently and correctly.

### Strings Missing One Required Vowel

A string such as `"aeio"` contains only vowels but never includes `'u'`. This can be tricky because the substring satisfies the "only vowels" condition but not the "all five vowels" condition. The implementation avoids this bug by counting substrings only when the size of `seen_vowels` becomes exactly `5`.

### Overlapping Valid Substrings

In `"aeiouu"`, both `"aeiou"` and `"aeiouu"` are valid. Some implementations accidentally count only the shortest valid substring or stop after finding the first one. This solution continues extending the substring even after all five vowels are present, ensuring every valid extension is counted.

### Consonants Splitting the Search Space

A substring cannot cross a consonant boundary. For example, in `"aeiobuaeiou"`, the consonant `'b'` invalidates any substring spanning across it. The implementation correctly stops expanding the current substring as soon as a consonant appears, preventing invalid counts.
