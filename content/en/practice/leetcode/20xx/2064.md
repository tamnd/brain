## Solution

## Problem Understanding

This problem asks us to distribute a given set of products of different types to a fixed number of stores in a way that minimizes the **maximum number of products** any store receives. We are given `n`, the number of stores, and an array `quantities`, where `quantities[i]` represents how many items exist of product type `i`. Each store can receive at most **one product type**, but any number of units of that type. The goal is to split the products such that the store with the largest allocation has as few items as possible.

The constraints tell us that `1 <= m <= n <= 10^5` and `1 <= quantities[i] <= 10^5`, which means a brute-force enumeration of all possible distributions would be infeasible. The input guarantees that we always have at least as many stores as product types, which avoids impossible scenarios.

Edge cases include when there is only one store (`n = 1`), when there is only one product type, or when `quantities` contains very large numbers close to the constraint limits. These cases could stress integer calculations or naive loops.

## Approaches

The **brute-force approach** would attempt to enumerate all possible distributions of products to stores and track the maximum per configuration, returning the minimum across all configurations. While correct, this approach is completely impractical because the number of possible distributions grows combinatorially with `quantities` and `n`.

The key insight for an **optimal approach** is that we are essentially looking for a number `x` such that it is possible to distribute all products while **no store receives more than `x` items**. If we can check whether a given `x` is feasible efficiently, we can perform a **binary search** over the possible range of `x` (from 1 to `max(quantities)`). The feasibility check involves determining how many stores would be needed if each store can hold at most `x` items for each product type. If the total stores needed is less than or equal to `n`, then `x` is feasible; otherwise, it is too small.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((n+m)^m) | O(n) | Enumerates all possible distributions, infeasible for large n or m |
| Optimal | O(m log(max_quantity)) | O(1) | Binary search over the maximum allocation per store, using simple arithmetic to check feasibility |

## Algorithm Walkthrough

1. **Determine the search space**: The minimum possible `x` is 1 (each store receives at least one item), and the maximum possible `x` is `max(quantities)` (giving all items of one type to a single store).
2. **Binary search setup**: Initialize `low = 1` and `high = max(quantities)`. This will represent the current bounds for the minimal feasible `x`.
3. **Feasibility check**: For a candidate `mid` value, calculate the number of stores needed to distribute each product type if each store receives at most `mid` units. For product `q`, the number of stores required is `ceil(q / mid)`, which can be computed efficiently as `(q + mid - 1) // mid`.
4. **Adjust search bounds**: If the total stores required is less than or equal to `n`, then `mid` is feasible, and we try a smaller `x` by setting `high = mid`. Otherwise, we need a larger `x` and set `low = mid + 1`.
5. **Convergence**: The binary search continues until `low == high`, which will be the minimal feasible `x`.

**Why it works**: The feasibility function is monotonic - if `x` is feasible, any larger value is also feasible. This property guarantees that binary search will correctly converge to the minimal feasible `x`.

## Python Solution

```python
from typing import List

class Solution:
    def minimizedMaximum(self, n: int, quantities: List[int]) -> int:
        def can_distribute(max_per_store: int) -> bool:
            required_stores = 0
            for q in quantities:
                required_stores += (q + max_per_store - 1) // max_per_store
            return required_stores <= n

        low, high = 1, max(quantities)
        while low < high:
            mid = (low + high) // 2
            if can_distribute(mid):
                high = mid
            else:
                low = mid + 1
        return low
```

This Python implementation defines a helper function `can_distribute` that calculates how many stores are needed for a candidate `max_per_store`. The binary search loop then repeatedly adjusts `low` and `high` until it finds the smallest feasible maximum.

## Go Solution

```go
func minimizedMaximum(n int, quantities []int) int {
    canDistribute := func(maxPerStore int) bool {
        requiredStores := 0
        for _, q := range quantities {
            requiredStores += (q + maxPerStore - 1) / maxPerStore
        }
        return requiredStores <= n
    }

    low, high := 1, 0
    for _, q := range quantities {
        if q > high {
            high = q
        }
    }

    for low < high {
        mid := (low + high) / 2
        if canDistribute(mid) {
            high = mid
        } else {
            low = mid + 1
        }
    }
    return low
}
```

The Go solution mirrors the Python implementation. Go integer division automatically floors, so `(q + maxPerStore - 1) / maxPerStore` correctly computes the ceiling division.

## Worked Examples

**Example 1**: `n = 6, quantities = [11, 6]`

- Start binary search with `low = 1`, `high = 11`.
- Check `mid = 6`: stores needed = `(11+5)//6 + (6+5)//6 = 2 + 2 = 4 <= 6`, feasible → set `high = 6`.
- Check `mid = 3`: stores needed = `(11+2)//3 + (6+2)//3 = 4 + 2 = 6 <= 6`, feasible → set `high = 3`.
- Check `mid = 2`: stores needed = `(11+1)//2 + (6+1)//2 = 6 + 3 = 9 > 6`, not feasible → set `low = 3`.
- Converged: `low = high = 3`. Answer is `3`.

**Example 2**: `n = 7, quantities = [15, 10, 10]`

- `low = 1`, `high = 15`. Binary search converges on `mid = 5` after feasibility checks. Total stores needed = 7. Answer is `5`.

**Example 3**: `n = 1, quantities = [100000]`

- Only one store exists. Max allocation is `100000`.
