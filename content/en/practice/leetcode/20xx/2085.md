## Solution

## Problem Understanding

The problem gives us two arrays of strings, `words1` and `words2`. We need to count how many strings satisfy two conditions at the same time:

1. The string appears exactly once in `words1`
2. The string appears exactly once in `words2`

The final answer is the number of strings that meet both requirements.

For example, if a word appears twice in one array and once in the other, it does not count. Likewise, if it appears once in one array but not at all in the other, it also does not count.

The input arrays can each contain up to 1000 strings, and every string contains only lowercase English letters. Since the constraints are relatively small, many approaches would technically work, but we still want an efficient and clean solution.

The important detail is that we care about frequency, not just existence. A naive implementation that only checks whether a word exists in both arrays would produce incorrect results because duplicate occurrences matter.

Some important edge cases include:

- Words appearing multiple times in one array but once in the other
- Words appearing exactly once in one array and not appearing at all in the other
- Arrays with completely disjoint words
- Arrays where every word is duplicated
- Arrays of minimum size, such as length 1

The problem guarantees that the arrays are non-empty and all strings are lowercase English letters, so we do not need to handle invalid input formats.

## Approaches

### Brute Force Approach

A straightforward approach is to examine every unique word and manually count how many times it appears in both arrays.

For each word in `words1`, we could:

1. Count its occurrences in `words1`
2. Count its occurrences in `words2`
3. Check whether both counts equal 1

This works because we directly verify the problem condition for every candidate word.

However, counting occurrences repeatedly is inefficient. If we scan the arrays every time we want a frequency, the total runtime becomes quadratic.

For example, if there are `n` words in `words1` and `m` words in `words2`, then repeatedly scanning both arrays for every word leads to approximately `O(n * (n + m))` time complexity.

Although the constraints are only 1000, this approach is unnecessarily slow and repetitive.

### Optimal Approach

The key observation is that frequency counting is exactly what hash maps are designed for.

Instead of repeatedly scanning arrays, we can preprocess both arrays into frequency maps:

- `count1[word]` stores how many times `word` appears in `words1`
- `count2[word]` stores how many times `word` appears in `words2`

Once we have these maps, we simply iterate through one of them and count how many words have frequency exactly 1 in both maps.

This reduces repeated work and gives constant time frequency lookup.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × (n + m)) | O(1) | Repeatedly scans arrays to count occurrences |
| Optimal | O(n + m) | O(n + m) | Uses hash maps for efficient frequency counting |

## Algorithm Walkthrough

1. Create a hash map called `count1` to store frequencies of words in `words1`.

We iterate through `words1` and increment the count for each word. After this step, we know exactly how many times every word appears in the first array.
2. Create another hash map called `count2` for `words2`.

This gives us the frequencies for the second array.
3. Initialize a variable called `result` to `0`.

This variable will store the number of valid words.
4. Iterate through every word in `count1`.

We only need to examine unique words because the frequency map already compresses duplicates into a single entry.
5. For each word, check two conditions:

- `count1[word] == 1`
- `count2.get(word, 0) == 1`

The first condition ensures the word appears exactly once in `words1`.

The second condition ensures the word appears exactly once in `words2`. If the word does not exist in `words2`, we treat its frequency as `0`.
6. If both conditions are true, increment `result`.
7. Return `result` after the loop finishes.

### Why it works

The algorithm works because the hash maps store the exact frequency of every word in each array. A word should be counted if and only if its frequency equals 1 in both maps. Since every unique word is checked against these precise counts, the algorithm correctly identifies all valid words and excludes all invalid ones.

## Python Solution

```python
from typing import List
from collections import Counter

class Solution:
    def countWords(self, words1: List[str], words2: List[str]) -> int:
        count1 = Counter(words1)
        count2 = Counter(words2)

        result = 0

        for word in count1:
            if count1[word] == 1 and count2.get(word, 0) == 1:
                result += 1

        return result
```

The implementation begins by importing `Counter` from the `collections` module. `Counter` is a convenient hash map specialized for frequency counting.

We create `count1` and `count2` directly from the input arrays. Each counter maps a word to the number of times it appears.

Next, we initialize `result` to store the final answer.

We then iterate through every unique word in `count1`. For each word, we verify that:

- It appears exactly once in `words1`
- It appears exactly once in `words2`

The `get(word, 0)` call safely handles words that do not exist in `words2`.

Whenever both conditions are satisfied, we increment the answer.

Finally, we return the total count.

## Go Solution

```go
func countWords(words1 []string, words2 []string) int {
	count1 := make(map[string]int)
	count2 := make(map[string]int)

	for _, word := range words1 {
		count1[word]++
	}

	for _, word := range words2 {
		count2[word]++
	}

	result := 0

	for word, freq := range count1 {
		if freq == 1 && count2[word] == 1 {
			result++
		}
	}

	return result
}
```

The Go implementation follows the same logic as the Python solution but uses built-in maps instead of `Counter`.

In Go, accessing a missing key from a map returns the zero value for the type. Since the map stores integers, missing keys automatically return `0`. This means we do not need a separate existence check when evaluating `count2[word] == 1`.

There are no integer overflow concerns because the maximum array size is only 1000.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n + m) | Each array is scanned once to build frequency maps |
| Space | O(n + m) | Hash maps store frequencies for unique words |

The algorithm processes each word exactly once while building the frequency maps. Afterward, checking valid words only requires iterating through unique entries in the first map. Hash map operations are average-case constant time, so the overall runtime is linear relative to the input size.

The space complexity comes from storing word frequencies. In the worst case, every word is unique, so both maps together store `n + m` entries.

## Edge Cases

One important edge case occurs when a word appears multiple times in one array but exactly once in the other. A naive implementation that only checks for existence would incorrectly count the word. For example, `"is"` in the first sample appears twice in `words1` and once in `words2`. The frequency maps prevent this mistake because the algorithm explicitly checks for frequency equal to 1 in both arrays.

Another important edge case is when a word exists in only one array. For example, `"as"` appears once in `words1` but not at all in `words2`. The implementation handles this correctly because `count2.get(word, 0)` returns `0` for missing words, so the condition fails.

A third edge case is when all words are duplicated. Consider:

```
words1 = ["a", "a"]
words2 = ["a", "a"]
```

Even though the same word appears in both arrays, it should not count because it does not appear exactly once. The implementation correctly rejects it because both frequencies equal `2`.

A final edge case involves minimum-size arrays such as:

```
words1 = ["a"]
words2 = ["a"]
```

This verifies that the algorithm works correctly even at the smallest valid input size. Since both frequencies equal `1`, the implementation correctly returns `1`.
