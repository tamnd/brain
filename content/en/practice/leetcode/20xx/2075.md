## Solution

## Problem Understanding

This problem describes a custom string encoding process based on a matrix and diagonal traversal. We are given the final encoded string and the number of rows used during encoding, and our task is to reconstruct the original text.

The encoding process works like this:

1. A matrix with a fixed number of rows is created.
2. The original text is written diagonally, moving from top-left to bottom-right.
3. Empty cells are filled with spaces.
4. The encoded string is created by reading the matrix row by row.

The decoding process must reverse this transformation.

A key observation is that the encoded string already represents the matrix flattened row-wise. Since the number of rows is known, we can reconstruct the matrix dimensions immediately.

If:

```
len(encodedText) = rows * cols
```

then:

```
cols = len(encodedText) // rows
```

Once the matrix is reconstructed, the original text can be recovered by reading diagonals starting from every column in the first row.

For example:

```
encodedText = "ch   ie   pr"
rows = 3
```

The matrix becomes:

```
c h    
  i e  
    p r
```

Reading diagonally gives:

```
c -> i -> p
h -> e -> r
```

which reconstructs:

```
"cipher"
```

An important detail is that the original text never contains trailing spaces. During encoding, extra spaces may appear because empty matrix cells are padded with `' '`. Therefore, after reconstructing the decoded string, we must remove trailing spaces.

The constraints are important:

- `encodedText.length` can be as large as `10^6`
- `rows` can be up to `1000`

This immediately tells us that expensive matrix operations or repeated string concatenations could become too slow. We need an efficient linear-time solution.

Important edge cases include:

- `rows = 1`, where the encoded text is already the original text
- Empty strings
- Large amounts of padding spaces
- Cases where diagonals terminate early at matrix boundaries

The problem guarantees that the input is always valid and corresponds to exactly one original text.

## Approaches

### Brute Force Approach

A straightforward solution is to explicitly rebuild the entire matrix and then simulate the original diagonal writing process in reverse.

We first compute the number of columns:

```
cols = len(encodedText) // rows
```

Then we fill a 2D matrix row by row using the encoded string. After reconstructing the matrix, we traverse every diagonal beginning from the first row:

- Start at `(0, col)`
- Move repeatedly to `(row + 1, col + 1)`
- Append characters until we leave the matrix

Finally, we remove trailing spaces.

This solution is correct because it exactly reverses the encoding process. However, storing an explicit matrix introduces unnecessary memory overhead. Since the matrix can contain up to `10^6` characters, we should avoid duplicating data when possible.

### Optimal Approach

The key insight is that we do not actually need to build a separate matrix.

Because the encoded string is already stored row-wise, we can compute the matrix position directly using index arithmetic.

For a matrix cell:

```
matrix[row][col]
```

its corresponding index inside `encodedText` is:

```
row * cols + col
```

Instead of creating a matrix, we simulate diagonal traversal directly using these indices.

For every starting column:

- Begin at `(0, startCol)`
- Move diagonally:

- `(1, startCol + 1)`
- `(2, startCol + 2)`
- and so on

At each step:

```
index = row * cols + col
```

This avoids constructing a separate matrix while still visiting characters in exactly the correct order.

After collecting all characters, we trim trailing spaces.

This achieves optimal efficiency.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Explicitly reconstructs the matrix |
| Optimal | O(n) | O(n) | Uses direct index arithmetic without matrix storage |

## Algorithm Walkthrough

1. Compute the total length of the encoded text.

The encoded string represents a flattened matrix, so its total size equals:

```
rows * cols
```
2. Compute the number of columns.

Since the matrix was flattened row-wise:

```
cols = len(encodedText) // rows
```
3. Initialize a result container.

We use a list of characters because repeated string concatenation is inefficient in both Python and Go.
4. Start diagonal traversals from every column in the first row.

Every diagonal corresponds to one starting column.

For each `start_col` from `0` to `cols - 1`:

- Begin at row `0`
- Current column is `start_col`
5. Traverse diagonally downward.

While both row and column remain inside bounds:

```
row < rows
col < cols
```

compute the encoded string index:

```
index = row * cols + col
```

Append that character to the result.
6. Move diagonally.

Increment both row and column:

```
row += 1
col += 1
```
7. Remove trailing spaces.

The encoding process padded unused cells with spaces, so the reconstructed string may contain extra spaces at the end.

Use:

```
rstrip()
```

to remove them.
8. Return the final decoded string.

### Why it works

The encoding process writes the original text along diagonals and then reads the matrix row-wise. Since the encoded string preserves row-wise ordering, we can reconstruct any matrix cell directly through index arithmetic.

By traversing diagonals in the same order used during encoding, we recover characters in their original sequence. Removing trailing spaces restores the exact original text because the problem guarantees the original text itself never ends with spaces.

## Python Solution

```python
class Solution:
    def decodeCiphertext(self, encodedText: str, rows: int) -> str:
        if rows == 0:
            return ""

        cols = len(encodedText) // rows
        decoded_chars = []

        for start_col in range(cols):
            row = 0
            col = start_col

            while row < rows and col < cols:
                index = row * cols + col
                decoded_chars.append(encodedText[index])

                row += 1
                col += 1

        return "".join(decoded_chars).rstrip()
```

The implementation begins by computing the number of columns using the matrix size relationship. Instead of reconstructing a 2D matrix, the code directly accesses characters from the encoded string using calculated indices.

The outer loop iterates through every possible diagonal starting position in the first row. The inner loop walks diagonally downward by incrementing both row and column together.

Each matrix coordinate is converted into its flattened string index using:

```python
row * cols + col
```

Characters are collected into a list because appending to a list is efficient, while repeated string concatenation would be expensive for large inputs.

Finally, `"".join()` creates the decoded string, and `rstrip()` removes trailing spaces introduced by matrix padding.

## Go Solution

```go
func decodeCiphertext(encodedText string, rows int) string {
    if rows == 0 {
        return ""
    }

    cols := len(encodedText) / rows
    result := make([]byte, 0)

    for startCol := 0; startCol < cols; startCol++ {
        row := 0
        col := startCol

        for row < rows && col < cols {
            index := row*cols + col
            result = append(result, encodedText[index])

            row++
            col++
        }
    }

    decoded := string(result)

    for len(decoded) > 0 && decoded[len(decoded)-1] == ' ' {
        decoded = decoded[:len(decoded)-1]
    }

    return decoded
}
```

The Go implementation follows exactly the same logic as the Python version.

Instead of a Python list of characters, Go uses a byte slice for efficient appending. Since the input contains only lowercase English letters and spaces, byte-based operations are sufficient.

Go does not provide a built-in equivalent of Python's `rstrip()` for removing only trailing spaces efficiently in this context, so the implementation manually trims spaces from the end of the string.

No integer overflow concerns exist because the constraints remain well within Go's integer limits.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every character is visited exactly once |
| Space | O(n) | The decoded output must be stored |

The algorithm performs a single traversal across all matrix cells. Each character from `encodedText` is processed exactly once during diagonal traversal, giving linear time complexity.

The additional memory usage comes from storing the decoded characters before constructing the final string. Since the output itself can contain up to `n` characters, this space usage is optimal.

## Edge Cases

### Single Row

When `rows = 1`, the matrix contains only one row, so the encoded text is identical to the original text. A buggy implementation might still attempt diagonal traversal and accidentally skip characters or go out of bounds.

This implementation handles the case naturally because diagonal traversal from each column simply visits that single character.

### Empty String

The input may be an empty string. Some implementations incorrectly assume at least one character exists and attempt invalid indexing operations.

Here, `cols` becomes `0`, the traversal loops never execute, and the algorithm correctly returns an empty string.

### Trailing Padding Spaces

The encoding process fills unused matrix cells with spaces. Those spaces appear at the end of the decoded reconstruction but are not part of the original text.

A common mistake is either:

- forgetting to remove trailing spaces
- removing all spaces, including valid internal spaces

This implementation uses `rstrip()`, which removes only trailing spaces while preserving meaningful spaces inside the message.

### Large Inputs

The input size can reach `10^6` characters. Inefficient solutions using repeated string concatenation may degrade to quadratic complexity.

This implementation avoids that problem by storing characters in a list or byte slice and joining them only once at the end.
