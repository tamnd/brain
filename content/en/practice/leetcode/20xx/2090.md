## Solution

## Problem Understanding

This problem asks us to compute the k-radius average for each element in a given array. The input is a 0-indexed array `nums` of `n` integers and a non-negative integer `k`. The k-radius average for an element at index `i` is defined as the average of all elements from index `i - k` to `i + k` inclusive. If the subarray around `i` extends beyond the bounds of the array (either before index 0 or after index n-1), the average is `-1`.

The output is an array of the same length as `nums` where each element is the k-radius average or `-1` if the subarray is too small. Integer division is used, so fractional parts are truncated. The constraints indicate that the array can have up to `10^5` elements and individual values can also be up to `10^5`. This rules out naive O(n*k) solutions for large k and n.

Important edge cases include when `k = 0` (the average is just the element itself), when `k` is larger than `n` (all outputs are `-1`), and when `nums` has only one element.

## Approaches

The brute-force approach calculates the k-radius average for each element by summing all elements in the subarray `[i-k, i+k]` individually. While correct, this approach has a time complexity of O(n*k) and is too slow for large arrays. Each sum operation repeats many computations unnecessarily.

The optimal approach leverages a **sliding window sum**. Instead of summing the subarray from scratch for every index, we maintain a running sum of the current window of size `2*k + 1`. When moving from index `i` to `i+1`, we subtract the element leaving the window and add the element entering the window. This reduces the time complexity to O(n) while still using only O(n) additional space for the output array.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n*k) | O(n) | Computes subarray sum from scratch for each index, too slow for large n/k |
| Optimal | O(n) | O(n) | Uses sliding window sum to compute averages efficiently |

## Algorithm Walkthrough

1. Initialize an output array `avgs` of size `n` filled with `-1`. This handles the default case where the k-radius average cannot be computed.
2. Compute the required window size as `window_size = 2 * k + 1`.
3. If `window_size > n`, return `avgs` immediately because no k-radius averages are possible.
4. Initialize a running sum variable `window_sum` to store the sum of the current sliding window.
5. Loop over the first `window_size` elements and compute the initial `window_sum`.
6. Set `avgs[k]` as `window_sum // window_size`, the first index where a valid k-radius average exists.
7. Slide the window forward from index `k+1` to `n-k-1`:

a. Subtract the element leaving the window at index `i - k - 1`.

b. Add the element entering the window at index `i + k`.

c. Compute the average as `window_sum // window_size` and store it in `avgs[i]`.
8. Return the `avgs` array.

Why it works: The sliding window ensures that each subarray of length `2*k + 1` is summed exactly once per position, avoiding redundant computation. The initialization and bounds checks guarantee correctness at the edges.

## Python Solution

```python
from typing import List

class Solution:
    def getAverages(self, nums: List[int], k: int) -> List[int]:
        n = len(nums)
        avgs = [-1] * n
        window_size = 2 * k + 1
        if window_size > n:
            return avgs
        
        window_sum = sum(nums[:window_size])
        avgs[k] = window_sum // window_size
        
        for i in range(k + 1, n - k):
            window_sum -= nums[i - k - 1]
            window_sum += nums[i + k]
            avgs[i] = window_sum // window_size
        
        return avgs
```

The implementation first initializes the result array to handle invalid indices. The sliding window sum is computed for the initial valid window and then updated incrementally for the rest of the array. Integer division ensures the truncation requirement is met.

## Go Solution

```go
func getAverages(nums []int, k int) []int {
    n := len(nums)
    avgs := make([]int, n)
    for i := range avgs {
        avgs[i] = -1
    }
    windowSize := 2*k + 1
    if windowSize > n {
        return avgs
    }

    windowSum := 0
    for i := 0; i < windowSize; i++ {
        windowSum += nums[i]
    }
    avgs[k] = windowSum / windowSize

    for i := k + 1; i < n - k; i++ {
        windowSum -= nums[i - k - 1]
        windowSum += nums[i + k]
        avgs[i] = windowSum / windowSize
    }

    return avgs
}
```

In Go, we prefill the output slice with `-1` and use integer division, just like in Python. Index management is similar, with care taken to ensure the window boundaries do not exceed array bounds.

## Edge Cases

1. **k = 0**: This is a trivial case where each element's average is the element itself. A naive implementation may still attempt to form subarrays, but the sliding window handles this efficiently.
2. **k > n/2**: When the radius exceeds half the array size, the window never fits, and all elements should be `-1`. This case can expose off-by-one errors in indexing.
3. **Single-element arrays**: When `n = 1`, the only valid k-radius average occurs if `k = 0`. If `k > 0`, the result must be `-1`. Handling this ensures no out-of-bounds accesses occur.

This solution correctly addresses all these edge cases using the initialization of `-1` for invalid indices, bounds checking for the window
