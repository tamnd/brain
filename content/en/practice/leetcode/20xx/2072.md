## Problem Understanding

The problem describes a competition between two universities: New York University and California University. Each university has a table with student scores for an exam. The tables, `NewYork` and `California`, contain two columns: `student_id` and `score`, where `student_id` is unique within each table.

The goal is to determine which university has more **excellent students**, where an excellent student is defined as one with a score of 90 or higher. If both universities have the same number of excellent students, the result is a draw. The output should be a single row with a column `winner` containing either `"New York University"`, `"California University"`, or `"No Winner"`.

Key points to note include:

- Each table can have any number of students, but the problem assumes a comparable number of students in each competition.
- Scores are integers, so checking for `>= 90` is straightforward.
- Edge cases include empty tables, ties, or all students being excellent in both universities.
- The problem explicitly focuses on **counting excellent students** rather than average or total scores, which simplifies the logic.

The input is relational tables in SQL, and the expected output is a single row indicating the winner.

## Approaches

The simplest approach is to count the excellent students in each table separately and then compare the counts. This approach is correct because the definition of the winner depends solely on the count of students with scores 90 or higher.

A brute-force approach could involve joining the tables or iterating over all students in a procedural way to count excellent students, but this is unnecessary because the tables are independent and we only need a count per table. The optimal approach leverages SQL aggregation functions to directly compute the count of excellent students from each table and then uses conditional logic to determine the winner.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n + m) | O(1) | Count excellent students by iterating over each table in application code |
| Optimal | O(n + m) | O(1) | Use SQL `COUNT` with `CASE` or `WHERE` to compute counts directly, then compare |

Here, `n` and `m` represent the number of students in the `NewYork` and `California` tables, respectively.

## Algorithm Walkthrough

1. Compute the number of excellent students in the `NewYork` table using a SQL `COUNT` with a `WHERE` clause or `CASE` statement: `score >= 90`.
2. Similarly, compute the number of excellent students in the `California` table.
3. Compare the two counts:

- If `NewYork` has more excellent students, return `"New York University"`.
- If `California` has more excellent students, return `"California University"`.
- If the counts are equal, return `"No Winner"`.
4. Return the result as a single-row table with the column `winner`.

Why it works: By using SQL aggregation functions, we directly count the relevant students without needing to iterate manually. The conditional comparison guarantees that the correct winner is determined according to the problem rules.

## Python Solution

```python
class Solution:
    def winnerOfCompetition(self, NewYork: list[dict], California: list[dict]) -> list[dict]:
        ny_excellent = sum(1 for student in NewYork if student['score'] >= 90)
        ca_excellent = sum(1 for student in California if student['score'] >= 90)
        
        if ny_excellent > ca_excellent:
            winner = "New York University"
        elif ca_excellent > ny_excellent:
            winner = "California University"
        else:
            winner = "No Winner"
        
        return [{"winner": winner}]
```

In this Python implementation, we use list comprehensions to count the number of excellent students in each university. The comparison logic then assigns the correct winner string, and the result is returned as a list containing a dictionary to mimic a SQL-style single-row table.

## Go Solution

```go
package main

type Student struct {
    StudentId int
    Score     int
}

func winnerOfCompetition(NewYork []Student, California []Student) []map[string]string {
    nyExcellent, caExcellent := 0, 0

    for _, student := range NewYork {
        if student.Score >= 90 {
            nyExcellent++
        }
    }

    for _, student := range California {
        if student.Score >= 90 {
            caExcellent++
        }
    }

    var winner string
    if nyExcellent > caExcellent {
        winner = "New York University"
    } else if caExcellent > nyExcellent {
        winner = "California University"
    } else {
        winner = "No Winner"
    }

    return []map[string]string{{"winner": winner}}
}
```

In Go, we define a `Student` struct to represent each row. We then iterate over slices representing each university's students, count the excellent ones, compare the counts, and return a slice of maps to simulate a SQL-style single-row result.

## Edge Cases

One important edge case is **empty tables**. If both universities have no students, the excellent student count is zero for both, resulting in a draw. The implementation handles this because summing over an empty list returns zero.

A second edge case is **all students being excellent**. This can lead to ties if both universities have the same number of excellent students, and the code correctly identifies a draw by comparing counts.

A third edge case is **one university having all excellent and the other having none**. This tests that the comparison logic correctly identifies the winner when there is a stark difference. Our implementation counts and compares explicitly, ensuring the correct winner is returned.
