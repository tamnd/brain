## Solution

## Problem Understanding

The problem requires us to place food buckets optimally in a linear arrangement of hamster cages. The input string `hamsters` contains `'H'` representing a hamster and `'.'` representing an empty space where we can potentially place a food bucket. A hamster is considered fed if there is at least one food bucket directly adjacent to it, either to the left or right. The goal is to determine the minimum number of food buckets to place such that every hamster is fed, or return `-1` if it is impossible.

The constraints specify that the string can be as long as `10^5`, so any solution must be at worst linear time complexity. Edge cases that can trip up a naive solution include consecutive hamsters with no available empty space between them, or strings that start or end with a hamster, limiting bucket placement options. We also need to consider isolated hamsters and sequences of empty spaces that can optimally feed multiple hamsters with a single bucket.

## Approaches

### Brute Force

A brute-force approach would be to try all possible combinations of bucket placements in empty slots and check if all hamsters are fed. For each combination, we would iterate over the string and verify if each hamster has an adjacent bucket. While this guarantees correctness, it is computationally infeasible because there are $2^n$ possible placements for `n` empty positions, which is exponential and far too large for the input limit of $10^5$.

### Optimal Approach

The optimal approach is greedy. The key insight is that to minimize buckets, we should place a bucket **to the right of a hamster whenever possible** because it can potentially feed the next hamster in sequence as well. Specifically, we iterate over the string from left to right. When we encounter a hamster, we check if the next slot is empty, and place a bucket there if possible. If not, we check the previous slot for a bucket. If neither is possible, it is impossible to feed that hamster. This guarantees the minimum number of buckets because each bucket placement is made to cover as many subsequent hamsters as possible without placing unnecessary additional buckets.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Try all combinations of bucket placements and check feeding conditions |
| Optimal | O(n) | O(1) | Greedy placement: rightward bucket first, fallback to left, ensure minimal coverage |

## Algorithm Walkthrough

1. Initialize a counter `buckets` to zero, which will keep track of the number of buckets placed.
2. Iterate over the string `hamsters` from left to right using an index `i`.
3. For each character at index `i`:

1. If it is `'.'`, continue to the next index because no bucket is needed here unless dictated by a neighboring hamster.
2. If it is `'H'`:

1. Check if the next index `i + 1` is within bounds and is `'.'`. If so, place a bucket at `i + 1` and increment `buckets`. Skip the next index `i + 1` because the bucket placed there already feeds the current hamster and possibly the next one.
2. Otherwise, check if the previous index `i - 1` is within bounds and contains a bucket (from a prior placement). If so, the hamster is already fed.
3. If neither option is possible, return `-1` because the hamster cannot be fed.
4. After finishing the iteration, return the `buckets` count.

**Why it works**: The algorithm ensures that every hamster is fed either by a bucket placed to its right or left. By always attempting to place the bucket to the right, we maximize the likelihood of feeding future hamsters with the same bucket, minimizing the total number of buckets.

## Python Solution

```python
class Solution:
    def minimumBuckets(self, hamsters: str) -> int:
        n = len(hamsters)
        buckets = 0
        i = 0
        hamsters = list(hamsters)  # convert to list for mutability
        
        while i < n:
            if hamsters[i] == 'H':
                if i + 1 < n and hamsters[i + 1] == '.':
                    buckets += 1
                    hamsters[i + 1] = 'B'  # mark bucket
                    i += 2  # skip next index
                elif i - 1 >= 0 and hamsters[i - 1] == 'B':
                    i += 1
                else:
                    return -1
            else:
                i += 1
        return buckets
```

The implementation converts the string to a list for mutability, allowing us to mark where buckets are placed. The iteration logic checks each hamster and greedily places a bucket to the right whenever possible. If not feasible, it checks for an existing left-side bucket before declaring the problem unsolvable.

## Go Solution

```go
func minimumBuckets(hamsters string) int {
    n := len(hamsters)
    buckets := 0
    arr := []byte(hamsters)
    
    for i := 0; i < n; {
        if arr[i] == 'H' {
            if i+1 < n && arr[i+1] == '.' {
                buckets++
                arr[i+1] = 'B'
                i += 2
            } else if i-1 >= 0 && arr[i-1] == 'B' {
                i++
            } else {
                return -1
            }
        } else {
            i++
        }
    }
    return buckets
}
```

The Go implementation follows the same greedy logic. Strings are converted to byte slices for mutability. Index skipping is used to prevent placing redundant buckets.
