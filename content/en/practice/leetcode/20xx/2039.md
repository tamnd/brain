**Difficulty:** 🟡 Medium  
**Topics:** Array, Breadth-First Search, Graph Theory

## Solution

## Problem Understanding

The problem describes a network of `n` servers connected by bidirectional channels, with server `0` designated as the **master server** and the rest as **data servers**. Each data server initially sends a message to the master server at time `0`. The messages travel along the **shortest path** and the master server processes all messages **instantly**. Upon processing, the master server replies via the **reverse path** of the incoming message.

Each data server has a `patience[i]` parameter which dictates how often it resends a message if it has **not yet received a reply**. Starting from second 1, every server checks whether a reply has arrived. If it has not, the server resends the message every `patience[i]` seconds. Once the reply arrives, the server stops sending further messages.

The goal is to determine the earliest time when the network becomes **idle**, i.e., there are **no messages in transit**.

Key observations:

1. Messages travel along the **shortest path**; hence, we only need **distances from the master server** to each data server.
2. Replies travel back along the same path, so the **round-trip time** is `2 * distance`.
3. Resends happen **periodically** until the first reply arrives. The last message sent may define when the network becomes idle.
4. The network is guaranteed to be connected, so all servers can reach the master.

Important edge cases:

- `patience[i]` very large compared to message travel time (server sends only once).
- `patience[i]` very small (server may resend many times before reply arrives).
- Servers at distance 1 from master.
- Only two servers (`n=2`).

## Approaches

**Brute Force:** Simulate the network second by second. Maintain a queue of messages, track message arrival and resend times. At each second, send resends and process arrivals until no messages remain. This guarantees correctness but is infeasible for large `n` (`up to 10^5`) due to time complexity `O(Total Time * n)`.

**Optimal Approach:** The key insight is that **the only factor determining when the network becomes idle for each server** is the **round-trip time** and the **patience**. We can compute the shortest distance from the master server using **BFS** (breadth-first search). Once we know the distance `d` for a server, the **last message sent** occurs at time:

```
last_send_time = floor((2*d - 1) / patience[i]) * patience[i]
```

Then, the **reply for that last message arrives** at:

```
arrival_time = last_send_time + 2*d
```

The network becomes idle at `max(arrival_time) + 1`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(Total Time * n) | O(n) | Simulate second by second; too slow for large n |
| Optimal (BFS + Math) | O(n + e) | O(n + e) | BFS computes distances, then math calculates last message arrival |

## Algorithm Walkthrough

1. **Construct the graph:** Represent the network as an adjacency list for efficient traversal.
2. **Compute shortest distances:** Use BFS starting from the master server `0`. For each server, record the distance `d` (number of edges) from server `0`.
3. **Compute last message arrival for each data server:** For server `i`, the **round-trip time** is `2 * d`. If `2*d <= patience[i]`, the server only sends the first message. Otherwise, calculate:

```
last_send_time = ((2*d - 1) // patience[i]) * patience[i]
arrival_time = last_send_time + 2*d
```

This accounts for resends sent **just before the first reply**.

1. **Compute network idle time:** Track the maximum `arrival_time` among all data servers and add `1` to get the earliest second when the network becomes idle.

**Why it works:** BFS guarantees the shortest path to each server, which is necessary for computing the earliest possible reply. The formula for the last message accounts for the periodic resends and ensures we capture the final message that contributes to network activity. By taking the maximum of all arrival times, we ensure no server has pending messages.

## Python Solution

```python
from collections import deque
from typing import List

class Solution:
    def networkBecomesIdle(self, edges: List[List[int]], patience: List[int]) -> int:
        n = len(patience)
        # Build adjacency list
        graph = [[] for _ in range(n)]
        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)
        
        # BFS to find shortest distance from master server (0)
        dist = [0] * n
        visited = [False] * n
        queue = deque([0])
        visited[0] = True
        
        while queue:
            node = queue.popleft()
            for neighbor in graph[node]:
                if not visited[neighbor]:
                    visited[neighbor] = True
                    dist[neighbor] = dist[node] + 1
                    queue.append(neighbor)
        
        # Compute the last arrival time for each data server
        max_time = 0
        for i in range(1, n):
            d = dist[i]
            round_trip = 2 * d
            if round_trip <= patience[i]:
                last_arrival = round_trip
            else:
                last_send_time = ((round_trip - 1) // patience[i]) * patience[i]
                last_arrival = last_send_time + round_trip
            max_time = max(max_time, last_arrival)
        
        return max_time + 1
```

**Explanation:**

- We first construct a graph to represent the network.
- BFS computes the **shortest distance** from the master server to each data server efficiently.
- For each data server, we calculate the **last message sent** before the first reply and its **arrival time**.
- The network idle time is the **maximum arrival time plus 1**, which we return.

## Go Solution

```go
package main

func networkBecomesIdle(edges [][]int, patience []int) int {
    n := len(patience)
    graph := make([][]int, n)
    for _, e := range edges {
        u, v := e[0], e[1]
        graph[u] = append(graph[u], v)
        graph[v] = append(graph[v], u)
    }
    
    dist := make([]int, n)
    visited := make([]bool, n)
    queue := []int{0}
    visited[0] = true
    
    for len(queue) > 0 {
        node := queue[0]
        queue = queue[1:]
        for _, neighbor := range graph[node] {
            if !visited[neighbor] {
                visited[neighbor] = true
                dist[neighbor] = dist[node] + 1
                queue = append(queue, neighbor)
            }
        }
    }
    
    maxTime := 0
    for i := 1; i < n; i++ {
        d := dist[i]
        roundTrip := 2 * d
        var lastArrival int
        if roundTrip <= patience[i] {
            lastArrival = roundTrip
        } else {
            lastSend := ((roundTrip - 1) / patience[i]) * patience[i]
            lastArrival = lastSend + roundTrip
        }
        if lastArrival > maxTime {
            maxTime = lastArrival
        }
    }
    
    return maxTime + 1
}
```

**Go-specific notes:** We handle slices for adjacency lists and queues, use integer division carefully, and maintain a boolean `visited` slice to track BFS. The logic mirrors the Python solution.

## Worked Examples

**Example 1:** `edges = [[0,1],[1,2]], patience = [0,2,1]`

- BFS distances: `[0, 1, 2]`
- Server 1: round_trip = 2 → 2 ≤ patience[1]=2 → last_arrival = 2
- Server 2: round_trip = 4 → 4 > patience[2]=1 → last_send = ((4-1)//1)*1=3 → last_arrival = 3+4=7
- max(last_arrival) = 7 → idle_time = 7+1=8

**Example 2:** `edges = [[0,1],[0,2],[1,2]], patience = [0,10,10]`

- BFS distances: `[0,1,1]`
- Server 1: round_trip=2 ≤ patience[1]=10 → last_arrival=2
- Server 2: round_trip=2 ≤ patience[2]=10 → last_arrival=2
- max(last_arrival)=2 → idle_time=3
