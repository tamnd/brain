## Solution

## Problem Understanding

The problem gives us a sentence made up of tokens separated by single spaces. Each token is either a lowercase English word or a positive integer. Our task is to examine all numeric tokens in the order they appear and determine whether they are strictly increasing from left to right.

A sequence is strictly increasing if every number is larger than the number before it. Equal values are not allowed, and decreasing values are also invalid.

For example, in the sentence:

```
"1 box has 3 blue 4 red 6 green"
```

the numbers appear in this order:

```
1, 3, 4, 6
```

Since every number is larger than the previous one, the answer is `true`.

However, in:

```
"hello world 5 x 5"
```

the numbers are:

```
5, 5
```

The second `5` is not greater than the first, so the answer is `false`.

The constraints are small:

- The string length is at most 200.
- Numbers are less than 100.
- Tokens are already cleanly separated by single spaces.
- There are at least two numbers.

These guarantees simplify the implementation significantly. We do not need to worry about malformed input, extra spaces, negative numbers, or leading zeros.

The most important edge cases are:

- Consecutive equal numbers such as `5 5`
- A later number smaller than an earlier one such as `10 3`
- Sentences where numbers are mixed heavily with words
- Multi-digit numbers such as `12` and `51`
- Numbers appearing at the beginning or end of the sentence

Because the constraints are small, performance is not a concern, but we still want the cleanest and most efficient solution.

## Approaches

### Brute Force Approach

A brute-force solution would first extract every number from the sentence into a separate list. After building the list, we could iterate through adjacent pairs and verify that each number is strictly smaller than the next one.

This approach works because it explicitly stores the sequence of numbers in their original order and checks the increasing condition directly.

For example:

```
"1 box has 3 blue 4 red"
```

would produce:

```
[1, 3, 4]
```

Then we compare:

- `1 < 3`
- `3 < 4`

If all comparisons succeed, we return `true`.

Although this approach is perfectly acceptable for the given constraints, it uses extra space to store all numbers even though we only ever need to compare the current number with the previous one.

### Optimal Approach

The key observation is that we only need the previous number to validate the current one. We do not need to store the entire sequence.

We can process the sentence token by token:

1. Split the sentence into tokens.
2. Check whether each token is numeric.
3. Convert numeric tokens into integers.
4. Compare the current number with the previous number.
5. If the current number is not greater, return `false`.
6. Otherwise, update the previous number and continue.

This solution is more space efficient because it processes the sentence in a single pass while storing only one previous value.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Stores all numbers before checking |
| Optimal | O(n) | O(1) | Single pass with only previous number stored |

Here, `n` represents the length of the sentence or total number of tokens.

## Algorithm Walkthrough

1. Split the sentence into individual tokens using spaces.

Since the problem guarantees tokens are separated by single spaces with no leading or trailing spaces, `split()` gives us a clean list of tokens.
2. Initialize a variable called `previous_number`.

This variable stores the most recent number we have seen. We can initialize it to `0` because all numbers in the input are positive.
3. Iterate through every token in the sentence.

Each token may be either a word or a number.
4. Check whether the current token is numeric.

In Python, we can use `isdigit()`. If the token is not numeric, we simply skip it.
5. Convert the numeric token into an integer.

We need integer comparison because string comparison would produce incorrect ordering for multi-digit numbers.
6. Compare the current number with the previous number.

If the current number is less than or equal to the previous number, the sequence is not strictly increasing, so we immediately return `False`.
7. Update `previous_number`.

If the comparison succeeds, assign the current number to `previous_number` and continue scanning.
8. Finish the iteration.

If we process all tokens without finding any violation, return `True`.

### Why it works

The algorithm maintains the invariant that `previous_number` always contains the last valid number encountered in the sentence. Every new number is checked against this value. If any number fails the strictly increasing condition, we return `false` immediately. Since every numeric token is processed exactly once in left-to-right order, the algorithm correctly verifies the required property.

## Python Solution

```python
class Solution:
    def areNumbersAscending(self, s: str) -> bool:
        previous_number = 0

        for token in s.split():
            if token.isdigit():
                current_number = int(token)

                if current_number <= previous_number:
                    return False

                previous_number = current_number

        return True
```

The implementation begins by initializing `previous_number` to `0`. Since every number in the sentence is positive, this ensures the first numeric token will always pass the comparison.

The sentence is split into tokens using `split()`. We then iterate through each token one by one.

The `isdigit()` check determines whether a token is a number or a word. For numeric tokens, we convert them into integers using `int()`.

The critical comparison is:

```python
if current_number <= previous_number:
```

This verifies strict increase. If the condition fails, we return `False` immediately because the sentence no longer satisfies the requirement.

If the current number is valid, we update `previous_number` and continue processing the remaining tokens.

If the loop completes successfully, every numeric token appeared in strictly increasing order, so we return `True`.

## Go Solution

```go
package main

import (
	"strconv"
	"strings"
)

func areNumbersAscending(s string) bool {
	previousNumber := 0

	tokens := strings.Split(s, " ")

	for _, token := range tokens {
		number, err := strconv.Atoi(token)

		if err == nil {
			if number <= previousNumber {
				return false
			}

			previousNumber = number
		}
	}

	return true
}
```

The Go solution follows the same logic as the Python implementation. The primary difference is numeric detection.

Instead of using `isdigit()`, Go uses `strconv.Atoi()`. If conversion succeeds, the token is numeric. If an error occurs, the token is a word and should be ignored.

Go strings are immutable, but since we only read tokens and perform comparisons, this does not affect the implementation.

Integer overflow is not a concern because all numbers are guaranteed to be less than 100.

## Edge Cases

One important edge case is repeated numbers, such as:

```
"5 x 5"
```

A naive implementation might mistakenly allow non-decreasing sequences instead of strictly increasing ones. The condition:

```
current_number <= previous_number
```

correctly rejects equal values.

Another important edge case involves multi-digit numbers. String comparison would fail for inputs like:

```
"9 10"
```

because lexicographically `"10"` is smaller than `"9"`. By converting tokens to integers before comparison, the implementation performs proper numeric ordering.

A third edge case is when numbers appear sparsely among many words, such as:

```
"apple 1 banana cat dog 7 elephant 20"
```

The algorithm correctly ignores non-numeric tokens and only evaluates numbers. This prevents accidental comparisons involving words and ensures the logic remains focused solely on numeric tokens.
