## Solution

## Problem Understanding

The problem asks us to split a sentence into multiple rows while respecting a maximum row length `k`. The sentence contains words separated by single spaces, and we are only allowed to insert line breaks between words. A word cannot be broken apart, reordered, or omitted. Every word must appear exactly once and in the original order.

The objective is not simply to fit words into rows, but to do so with the **minimum possible cost**. For every row except the final one, we compute a penalty based on how much unused space remains in the row. If a row has length `n`, then its cost is:

$$(k - n)^2$$

The last row is special because its cost is completely ignored. This detail significantly changes the optimization strategy, because it can sometimes be beneficial to leave extra unused space in earlier rows if it improves the arrangement of later rows.

The input consists of a string `sentence` and an integer `k`. The sentence is guaranteed to contain lowercase English letters and spaces, with no leading or trailing spaces. Each word length is guaranteed to be at most `k`, meaning every individual word can always fit in a row by itself.

The output is a single integer representing the smallest achievable total cost after optimally splitting the sentence into rows.

The constraints are important:

- `sentence.length <= 5000`
- `k <= 5000`

A brute force strategy that tries every possible arrangement of line breaks is infeasible because the number of possible partitions grows exponentially with the number of words. Since the input size is relatively large, we need a more efficient optimization technique.

Several edge cases deserve attention immediately. A sentence with only one word should always return `0`, because the only row is also the last row, and the last row has no cost. Another important case occurs when a word exactly fills the row length `k`, producing zero cost for that row. We also need to carefully handle spacing between words, because spaces count toward row length. Finally, since the final row contributes no cost, an implementation that accidentally charges for it will produce incorrect answers.

## Approaches

### Brute Force Approach

A straightforward solution is to recursively try every valid way of splitting the sentence.

At each word index, we decide how many consecutive words to place in the current row, as long as the total length remains at most `k`. For each valid choice, we recursively compute the minimum cost for the remaining words. The row cost is added unless the chosen row becomes the final row.

This approach guarantees correctness because it explores every possible legal arrangement of line breaks and chooses the minimum total cost among them.

However, the number of possible partitions is exponential. If there are `n` words, every gap between adjacent words may or may not contain a line break, resulting in approximately `2^(n-1)` possibilities. Since `sentence.length` can be as large as `5000`, this is far too slow.

### Optimal Approach, Dynamic Programming

The key observation is that the problem contains **overlapping subproblems**.

Suppose we already know the minimum cost for formatting the suffix of words starting at index `j`. Then when considering words starting at index `i`, we can reuse that previously computed answer instead of recalculating it.

This naturally suggests **dynamic programming**.

We define:

`dp[i] = minimum cost to arrange words starting from index i`

To compute `dp[i]`, we try placing consecutive words `i...j` into the same row, stopping when the row exceeds length `k`. For every valid row:

- If `j` is the final word, cost is `0`
- Otherwise, cost is:

$$(k - rowLength)^2 + dp[j + 1]$$

We take the minimum across all valid choices.

Since every subproblem is solved once and reused, the complexity becomes polynomial.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Tries every possible row partition |
| Optimal | O(n²) | O(n) | Dynamic programming with memoization |

Here, `n` represents the number of words.

## Algorithm Walkthrough

1. Split the sentence into individual words.

We first parse the sentence using spaces. Since words cannot be rearranged or split, treating the sentence as an ordered list of words makes the problem easier to model.
2. Create a DP array.

Define:

`dp[i] = minimum cost for arranging words from index i onward`

We also define:

`dp[n] = 0`

where `n` is the total number of words. This represents the base case, an empty suffix requires no cost.
3. Process words from right to left.

We iterate backward so that when computing `dp[i]`, all future states `dp[j+1]` are already available.
4. Try every possible row starting at index `i`.

Begin with word `i` and progressively add words `i+1`, `i+2`, and so on.

Keep track of the current row length.

Remember that spaces count toward row length, so adding another word contributes:

`1 + len(next_word)`
5. Stop when row length exceeds `k`.

Once the row becomes too long, no larger extension can work either. We immediately stop exploring further words for this starting position.
6. Compute transition cost.

If the current row reaches the last word, its cost is `0`.

Otherwise:

$$cost = (k - rowLength)^2 + dp[j+1]$$

Update `dp[i]` with the minimum cost.
7. Return `dp[0]`.

This represents the minimum cost for arranging the entire sentence.

### Why it works

The dynamic programming recurrence is correct because every valid formatting of the sentence must begin with some legal first row. Once that first row is fixed, the remaining problem becomes independent and identical in structure, arranging the remaining suffix of words optimally. Since `dp[j+1]` always stores the minimum cost for the remaining words, combining it with the current row cost guarantees an optimal result. Taking the minimum across all valid row endings ensures no better arrangement is missed.

## Python Solution

```python
class Solution:
    def minimumCost(self, sentence: str, k: int) -> int:
        words = sentence.split()
        n = len(words)

        dp = [float("inf")] * (n + 1)
        dp[n] = 0

        for i in range(n - 1, -1, -1):
            current_length = 0

            for j in range(i, n):
                if j == i:
                    current_length = len(words[j])
                else:
                    current_length += 1 + len(words[j])

                if current_length > k:
                    break

                if j == n - 1:
                    cost = 0
                else:
                    remaining_spaces = k - current_length
                    cost = remaining_spaces * remaining_spaces + dp[j + 1]

                dp[i] = min(dp[i], cost)

        return dp[0]
```

The implementation starts by splitting the sentence into words so we can reason about row placement directly.

The `dp` array stores the minimum achievable cost for every suffix of the word list. Initializing all values to infinity ensures that any valid arrangement will improve the state. The base case `dp[n] = 0` represents an empty suffix.

The outer loop processes words from right to left so future results are already known. Inside the nested loop, we attempt to extend the current row one word at a time while tracking its total length.

Whenever the row exceeds `k`, we stop immediately because adding more words can only make the row longer.

If the current row includes the final word, no penalty is applied. Otherwise, we compute the unused space penalty and combine it with the optimal future solution stored in `dp[j + 1]`.

Finally, `dp[0]` gives the minimum cost for formatting the full sentence.

## Go Solution

```go
func minimumCost(sentence string, k int) int {
	words := strings.Split(sentence, " ")
	n := len(words)

	const INF = int(1e18)

	dp := make([]int, n+1)
	for i := range dp {
		dp[i] = INF
	}
	dp[n] = 0

	for i := n - 1; i >= 0; i-- {
		currentLength := 0

		for j := i; j < n; j++ {
			if j == i {
				currentLength = len(words[j])
			} else {
				currentLength += 1 + len(words[j])
			}

			if currentLength > k {
				break
			}

			var cost int
			if j == n-1 {
				cost = 0
			} else {
				remainingSpaces := k - currentLength
				cost = remainingSpaces*remainingSpaces + dp[j+1]
			}

			if cost < dp[i] {
				dp[i] = cost
			}
		}
	}

	return dp[0]
}
```

The Go implementation closely mirrors the Python version. Since Go lacks Python's `float("inf")`, we initialize with a very large integer constant. Strings are split using `strings.Split`, and the DP transition logic remains identical.

Integer overflow is not an issue here because the maximum possible squared cost is `5000² = 25,000,000`, and even summing many rows stays well within Go's integer limits.

## Edge Cases

A sentence containing only one word is an important edge case. Since the final row is excluded from cost calculations, the answer must always be `0`. A naive implementation might incorrectly apply the penalty formula and return `(k - wordLength)^2`. The implementation handles this by explicitly assigning cost `0` whenever the current row reaches the last word.

Another subtle edge case occurs when a row length exactly equals `k`. In this situation, the row cost should become `0`, because there is no unused space. Some implementations mistakenly reject rows of exact length by using `>= k` instead of `> k`. This solution correctly allows rows whose length equals `k`.

A third important case involves sentences where multiple arrangements are possible and greedily filling rows is not optimal. For example, packing as many words as possible into each row may create expensive earlier rows. Dynamic programming avoids this pitfall by evaluating every valid row ending and choosing the globally optimal configuration.

Finally, spaces between words can easily introduce off by one errors. Since every adjacent pair of words requires exactly one space, row length must include those spaces. The implementation carefully adds `1 + len(word)` whenever extending a row, ensuring precise length calculation.
