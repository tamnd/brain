## Solution

## Problem Understanding

This problem is asking us to identify candidates from an interview database who meet two criteria: they have at least two years of work experience, and the sum of their interview round scores exceeds 15. We are given two tables: `Candidates`, which contains candidate information including years of experience and a foreign key `interview_id`, and `Rounds`, which records the scores of each interview round linked to `interview_id`.

The expected output is a single column table with `candidate_id` of candidates meeting both criteria. The order of results does not matter. Important constraints and guarantees include: every candidate has a unique `candidate_id`, and each interview round score is associated with a valid `interview_id`.

Key edge cases include candidates with exactly 2 years of experience, candidates whose total scores are exactly 15, candidates with no rounds recorded, and candidates with multiple rounds summing to just above 15. These must be handled carefully to satisfy the strict inequality and minimum experience requirement.

## Approaches

A brute-force approach would be to join `Candidates` and `Rounds` tables without filtering first, compute the sum of scores for each candidate in memory or via SQL aggregation, then filter candidates based on experience and score. While correct, this approach is inefficient because it processes potentially unnecessary data for candidates who are clearly ineligible due to low experience, wasting computation on aggregation.

The optimal approach leverages SQL aggregation with filtering applied at the right stages. By first filtering candidates with at least two years of experience and then joining with the `Rounds` table to sum their scores, we minimize the number of rows processed. Aggregating with `GROUP BY` on `interview_id` or `candidate_id` allows us to directly compute the total score per candidate, and we can then apply the score threshold in the `HAVING` clause to efficiently select the desired candidates.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n*m) | O(n) | Join all candidates and rounds, compute sums, filter after aggregation |
| Optimal | O(n + m) | O(n) | Filter candidates first, join only relevant rows, aggregate scores, filter with HAVING |

## Algorithm Walkthrough

1. Filter the `Candidates` table to only include candidates with `years_of_exp >= 2`. This ensures we only process eligible candidates.
2. Join the filtered `Candidates` table with the `Rounds` table on `interview_id` to link candidate IDs with their respective interview scores.
3. Use SQL `GROUP BY candidate_id` to sum the scores for each candidate across all rounds.
4. Apply a `HAVING` clause to select only candidates whose summed score is strictly greater than 15.
5. Select the `candidate_id` column as the final result.

Why it works: By filtering early and aggregating scores with `GROUP BY` on `candidate_id`, the algorithm ensures that only candidates meeting both criteria-minimum experience and sufficient total score-are selected. The SQL engine efficiently computes these operations without unnecessary intermediate data.

## Python Solution

```python
import sqlite3
from typing import List, Tuple

def accepted_candidates(conn: sqlite3.Connection) -> List[Tuple[int]]:
    query = """
        SELECT c.candidate_id
        FROM Candidates c
        JOIN Rounds r ON c.interview_id = r.interview_id
        WHERE c.years_of_exp >= 2
        GROUP BY c.candidate_id
        HAVING SUM(r.score) > 15
    """
    cursor = conn.cursor()
    cursor.execute(query)
    return cursor.fetchall()
```

This Python implementation executes a SQL query directly on a SQLite database connection. We join `Candidates` with `Rounds`, filter candidates with sufficient experience, group by `candidate_id` to compute total scores, and use `HAVING` to enforce the threshold. The result is returned as a list of tuples, each containing a candidate ID.

## Go Solution

```go
package main

import (
    "database/sql"
)

func AcceptedCandidates(db *sql.DB) ([]int, error) {
    query := `
        SELECT c.candidate_id
        FROM Candidates c
        JOIN Rounds r ON c.interview_id = r.interview_id
        WHERE c.years_of_exp >= 2
        GROUP BY c.candidate_id
        HAVING SUM(r.score) > 15
    `
    rows, err := db.Query(query)
    if err != nil {
        return nil, err
    }
    defer rows.Close()

    var result []int
    for rows.Next() {
        var candidateID int
        if err := rows.Scan(&candidateID); err != nil {
            return nil, err
        }
        result = append(result, candidateID)
    }
    return result, nil
}
```

The Go implementation closely mirrors the Python solution. SQL aggregation handles filtering, grouping, and summing. Differences include explicit error handling, deferred row closing, and appending results to a slice. No special handling for nil values is needed because SQL aggregation will return results only for rows matching the join and filtering criteria.

## Edge Cases

One edge case is a candidate with exactly 2 years of experience. This could be mishandled if the condition uses `>` instead of `>=`. Our implementation correctly includes such candidates.

Another edge case is candidates whose total interview score sums to exactly 15. The `HAVING SUM(score) > 15` clause ensures that these candidates are excluded, satisfying the problem's strict inequality.

A third edge case involves candidates with no associated interview rounds. If a candidate has no rows in `Rounds`, their sum defaults to null or zero depending on SQL engine, and our join plus aggregation correctly excludes them because the sum does not exceed 15. This prevents false positives from appearing in the result.
