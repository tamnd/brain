## Solution

## Problem Understanding

The problem asks us to count how many substrings of a given string start and end with the same character.

A substring is any contiguous portion of the string. For every possible substring, we check whether the first and last characters are identical. If they are, that substring contributes to the final answer.

For example, in the string `"abcba"`:

- Every single character is a valid substring because the first and last characters are obviously the same.
- `"bcb"` is also valid because it starts and ends with `'b'`.
- `"abcba"` is valid because it starts and ends with `'a'`.

The input is a single string `s` containing only lowercase English letters. The length of the string can be as large as `10^5`, which immediately tells us that a naive quadratic or cubic approach may become too slow.

The expected output is the total number of valid substrings.

An important observation is that a substring is completely determined by its starting and ending indices. Therefore, for every pair of indices `(i, j)` where `i <= j`, the substring `s[i:j+1]` is valid if and only if:

```
s[i] == s[j]
```

The constraints also reveal an important detail:

- There are only 26 lowercase English letters.
- The string can be very large.
- We need an efficient counting strategy instead of generating all substrings explicitly.

Several edge cases are important:

- A string of length `1` always has exactly one valid substring.
- A string where all characters are the same produces many substrings, specifically every possible substring is valid.
- A string where all characters are distinct only contributes substrings of length `1`.
- Large inputs require an `O(n)` or near `O(n)` solution to avoid time limit exceeded errors.

## Approaches

### Brute Force Approach

The most direct approach is to generate every possible substring and check whether its first and last characters are equal.

We can use two nested loops:

- The outer loop selects the starting index.
- The inner loop selects the ending index.

For every substring `s[i:j+1]`, we check:

```
s[i] == s[j]
```

If the condition is true, we increment the answer.

This approach is correct because every substring is examined exactly once, and every valid substring is counted.

However, there are `O(n^2)` substrings in a string of length `n`. With `n = 10^5`, this becomes far too slow. A quadratic solution would require around `10^10` operations in the worst case.

### Optimal Approach

The key insight is that we do not actually need to generate substrings.

Suppose we process the string from left to right. When we encounter a character, every previous occurrence of the same character can form a valid substring ending at the current position.

For example, consider:

```
s = "abaca"
```

When we reach the final `'a'`:

- The previous `'a'` at index `0` forms `"abaca"`.
- The previous `'a'` at index `2` forms `"aca"`.
- The current `'a'` itself forms `"a"`.

So if a character has appeared `k` times before, then the current occurrence creates `k + 1` new valid substrings.

This means we only need to track how many times each character has appeared so far.

We can maintain a frequency map:

- `count[c]` = number of times character `c` has appeared previously.

For each character:

1. Add `count[c] + 1` to the answer.
2. Increment `count[c]`.

Since each character is processed once, the algorithm runs in linear time.

| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every substring individually |
| Optimal | O(n) | O(1) | Uses character frequency counting |

## Algorithm Walkthrough

1. Initialize a variable `answer` to `0`.

This variable stores the total number of valid substrings.
2. Create a frequency map to store how many times each character has appeared.

Since the string contains only lowercase English letters, we can use:

- a dictionary, or
- an array of size 26.
3. Traverse the string from left to right.

At each position, let the current character be `c`.
4. Determine how many new substrings end at the current position.

Every previous occurrence of `c` can pair with the current `c` to form a substring whose first and last characters match.

Additionally, the single-character substring consisting only of `c` is always valid.

Therefore:

```
new substrings = previous occurrences of c + 1
```
5. Add this value to the answer.
6. Update the frequency count for the current character.

This ensures future positions can use this occurrence.
7. Continue until the entire string has been processed.
8. Return the final answer.

### Why it works

The algorithm works because every valid substring is uniquely identified by its ending position and starting character.

When processing character `c` at index `i`, every earlier occurrence of `c` produces exactly one valid substring ending at `i`. The current character itself also contributes the single-character substring.

Since each valid substring is counted exactly once, the final total is correct.

## Python Solution

```python
class Solution:
    def numberOfSubstrings(self, s: str) -> int:
        char_count = {}
        answer = 0

        for char in s:
            previous_count = char_count.get(char, 0)

            answer += previous_count + 1

            char_count[char] = previous_count + 1

        return answer
```

The implementation follows the exact logic from the algorithm walkthrough.

The dictionary `char_count` keeps track of how many times each character has appeared so far.

For every character:

- `previous_count` tells us how many earlier occurrences exist.
- Each earlier occurrence creates one valid substring ending at the current position.
- The current character alone forms another valid substring.

So we add:

```python
previous_count + 1
```

to the answer.

After counting those substrings, we update the frequency map so future characters can use this occurrence.

Because the string is traversed exactly once, the solution runs efficiently even for the largest allowed input size.

## Go Solution

```go
func numberOfSubstrings(s string) int64 {
    counts := make([]int64, 26)
    var answer int64 = 0

    for _, ch := range s {
        index := ch - 'a'

        answer += counts[index] + 1

        counts[index]++
    }

    return answer
}
```

The Go solution uses a fixed-size array instead of a hash map because the input only contains lowercase English letters.

The array index is computed using:

```go
index := ch - 'a'
```

This converts characters like `'a'`, `'b'`, and `'c'` into indices `0`, `1`, and `2`.

The function returns `int64` because the total number of substrings can exceed the range of a 32-bit integer when the string length approaches `10^5`.

## Edge Cases

### Single Character String

A string of length one is the smallest possible input. It is easy to accidentally overlook that a single character is itself a valid substring.

For example:

```
s = "a"
```

The implementation handles this naturally because every character contributes at least one substring:

```
answer += previous_count + 1
```

When the character appears for the first time, `previous_count` is `0`, so the contribution becomes `1`.

### All Characters Are Unique

If every character appears only once, then no substring longer than one character can satisfy the condition.

Example:

```
s = "abcd"
```

A naive implementation might accidentally overcount by assuming repeated patterns exist.

The frequency-based solution correctly adds exactly one substring per character because every character has zero previous occurrences.

### All Characters Are the Same

This is the opposite extreme and produces the maximum number of valid substrings.

Example:

```
s = "aaaa"
```

Every possible substring is valid:

```
"a", "a", "a", "a"
"aa", "aa", "aa"
"aaa", "aaa"
"aaaa"
```

The implementation handles this efficiently because each new occurrence contributes one more valid substring than the previous occurrence:

- First `'a'` contributes `1`
- Second contributes `2`
- Third contributes `3`
- Fourth contributes `4`

Total:

```
1 + 2 + 3 + 4 = 10
```

This avoids explicitly generating all substrings while still counting them correctly.
