## Solution

## Problem Understanding

The problem asks us to compute the total number of vowels that appear in **all substrings** of a given string `word`. A substring is any contiguous sequence of characters within the string, and we are only counting the vowels `'a'`, `'e'`, `'i'`, `'o'`, and `'u'`. For example, for the string `"aba"`, its substrings are `"a"`, `"ab"`, `"aba"`, `"b"`, `"ba"`, and `"a"`. Each substring may contain zero, one, or multiple vowels, and we are tasked with summing up the count across all of them.

The input is a string of lowercase English letters with length up to `10^5`, which is large enough that a brute-force approach iterating over all substrings would be too slow. The output is an integer representing the total sum of vowels, which can be very large, so we should use a data type that can hold values bigger than a 32-bit signed integer.

Important edge considerations include strings without vowels, strings consisting entirely of vowels, and handling large strings efficiently without exceeding time constraints.

## Approaches

The **brute-force approach** would generate all possible substrings of `word`, count the vowels in each substring, and sum the results. While this approach is correct, it is prohibitively slow because a string of length `n` has `n*(n+1)/2` substrings, making the time complexity `O(n^3)` if we count vowels in each substring individually. This will not work for `n` up to `10^5`.

The **optimal approach** uses a combinatorial observation: instead of iterating through all substrings, we can compute how many substrings each vowel contributes to. For a vowel at index `i` in a string of length `n`, it appears in `(i + 1) * (n - i)` substrings. Here, `(i + 1)` is the number of choices for the starting index of a substring that includes this vowel, and `(n - i)` is the number of choices for the ending index. By summing this contribution for every vowel in the string, we get the total number of vowels across all substrings in **O(n)** time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(1) | Generate all substrings, count vowels individually |
| Optimal | O(n) | O(1) | Compute contribution of each vowel to all substrings using combinatorial counting |

## Algorithm Walkthrough

1. Initialize a variable `total` to zero. This will accumulate the total number of vowels across all substrings.
2. Define a set of vowels for quick lookup: `vowels = {'a', 'e', 'i', 'o', 'u'}`.
3. Iterate through the string `word` with index `i` and character `c`.
4. For each character, check if it is a vowel by seeing if `c` is in the `vowels` set.
5. If it is a vowel, calculate the number of substrings it appears in as `(i + 1) * (n - i)` and add this to `total`.
6. After the loop, return `total`.

**Why it works:** Every vowel contributes to multiple substrings depending on its position. The formula `(i + 1) * (n - i)` counts all possible substrings containing the vowel at index `i`. By summing this for all vowels, we correctly compute the total count without enumerating each substring explicitly. This ensures both correctness and efficiency.

## Python Solution

```python
class Solution:
    def countVowels(self, word: str) -> int:
        total = 0
        n = len(word)
        vowels = {'a', 'e', 'i', 'o', 'u'}
        
        for i, c in enumerate(word):
            if c in vowels:
                total += (i + 1) * (n - i)
        
        return total
```

This Python implementation follows the algorithm directly. We initialize a running total and define the set of vowels. Using `enumerate`, we track both the index and character. When the character is a vowel, we compute its contribution to all possible substrings using the combinatorial formula and add it to the total. The result is returned after processing all characters.

## Go Solution

```go
func countVowels(word string) int64 {
    var total int64 = 0
    n := int64(len(word))
    vowels := map[rune]bool{'a': true, 'e': true, 'i': true, 'o': true, 'u': true}

    for i, c := range word {
        if vowels[c] {
            total += int64(i+1) * (n - int64(i))
        }
    }

    return total
}
```

In Go, we use a `map[rune]bool` to store vowels for quick lookup. Since the result may exceed the 32-bit limit, we use `int64` for the total. The loop iterates over the string with both index and rune character. Each vowel's contribution is computed similarly to the Python solution, converted to `int64` as needed, and accumulated into `total`. The final sum is returned as an `int64`.

## Edge Cases

1. **Single-character strings**: The string may be a single vowel or consonant. The formula `(i+1)*(n-i)` correctly computes `1*1 = 1` for a vowel and `0` for a consonant, ensuring correct output.
2. **Strings with no vowels**: For example, `"bcdfg"`. The algorithm checks membership in the vowel set and skips contributions for consonants, returning zero without error.
3. **Strings of maximum length with all vowels**: A string like `"a"*10^5` tests the algorithm's efficiency and integer handling. Using `int64` in Go and Python's unbounded integer ensures that large sums do not overflow. The formula scales linearly with string length, so performance remains acceptable.
