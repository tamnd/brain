## Solution

## Problem Understanding

The problem asks us to find the `kth` distinct string in an array of strings, while preserving the original order of appearance.

A string is considered distinct if it appears exactly once in the array. Any string that appears two or more times is not distinct.

The input consists of:

- `arr`, an array of lowercase strings
- `k`, a positive integer

The output should be:

- The `kth` distinct string encountered when scanning the array from left to right
- An empty string `""` if fewer than `k` distinct strings exist

The key detail is that order matters. We are not sorting strings or selecting alphabetically. We must preserve the original sequence of the array.

For example, if:

```
arr = ["d","b","c","b","c","a"]
```

then:

- `"d"` appears once
- `"b"` appears twice
- `"c"` appears twice
- `"a"` appears once

The distinct strings, in order, are:

```
["d", "a"]
```

So the 2nd distinct string is `"a"`.

The constraints are relatively small:

- `arr.length <= 1000`
- `arr[i].length <= 5`

Even an `O(n^2)` solution would technically pass, but the problem is a perfect fit for a hash map counting approach, which gives linear time complexity and cleaner logic.

Several edge cases are important:

- The array may contain no distinct strings
- `k` may be larger than the number of distinct strings
- Every string may be distinct
- Duplicate strings may appear many times
- Distinct strings must be counted in original order, not sorted order

Because the array length is small and string lengths are tiny, hashing strings is very efficient here.

## Approaches

### Brute Force Approach

A straightforward solution is to examine every string and count how many times it appears in the array.

For each string `arr[i]`, we scan the entire array again and count its occurrences. If the count equals `1`, then it is distinct. We keep track of how many distinct strings we have seen so far. Once we reach the `kth` one, we return it.

This works because it directly follows the definition of a distinct string. However, it repeatedly scans the array for every element, causing unnecessary repeated work.

If the array contains `n` elements, then for every element we may scan all `n` elements again, leading to `O(n^2)` time complexity.

### Optimal Approach

The better solution uses a hash map to count frequencies.

The key observation is that determining whether a string is distinct depends only on how many times it appears in the array. A hash map allows us to compute all frequencies efficiently in one pass.

The algorithm works in two passes:

1. Count the frequency of every string using a hash map.
2. Traverse the array again in original order, selecting strings whose frequency is exactly `1`.

Every time we encounter a distinct string, we decrement `k`. When `k` becomes `0`, we return that string.

This approach is efficient because:

- Frequency lookup in a hash map is `O(1)` on average
- We only traverse the array twice
- Order is naturally preserved during the second traversal

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Recounts occurrences for every string |
| Optimal | O(n) | O(n) | Uses hash map frequency counting |

## Algorithm Walkthrough

### Optimal Hash Map Algorithm

1. Create a hash map called `frequency`.

This map stores how many times each string appears in the array. The key is the string itself, and the value is its occurrence count.
2. Traverse the array once to build frequency counts.

For every string in `arr`, increment its count in the hash map.

Example:

```
arr = ["d","b","c","b","c","a"]
```

After counting:

```
{
    "d": 1,
    "b": 2,
    "c": 2,
    "a": 1
}
```
3. Traverse the array a second time in original order.

This step is important because the problem requires preserving appearance order.
4. For each string, check whether its frequency equals `1`.

If it does, then the string is distinct.
5. Decrement `k` whenever a distinct string is found.

This tracks how many distinct strings we have encountered so far.
6. If `k` becomes `0`, return the current string.

This means we have reached the `kth` distinct string.
7. If the traversal finishes without reaching `k == 0`, return an empty string.

This means fewer than `k` distinct strings exist.

### Why it works

The frequency map correctly identifies which strings are distinct because a distinct string is defined as appearing exactly once.

The second traversal preserves the original ordering of the array. Since we decrement `k` only when encountering distinct strings, the moment `k` reaches zero corresponds exactly to the `kth` distinct string in appearance order.

Therefore, the algorithm always returns the correct answer.

## Python Solution

```python
from typing import List
from collections import Counter

class Solution:
    def kthDistinct(self, arr: List[str], k: int) -> str:
        frequency = Counter(arr)

        for string in arr:
            if frequency[string] == 1:
                k -= 1

                if k == 0:
                    return string

        return ""
```

The implementation begins by importing `Counter` from Python's `collections` module. `Counter` automatically builds a frequency map for all strings in the array.

The first line inside the method:

```python
frequency = Counter(arr)
```

creates a dictionary-like object where each string maps to its occurrence count.

Next, the code traverses the original array in order:

```python
for string in arr:
```

This preserves the original appearance ordering required by the problem.

Inside the loop, the condition:

```python
if frequency[string] == 1:
```

checks whether the current string is distinct.

Whenever a distinct string is found, `k` is decremented:

```python
k -= 1
```

Once `k` reaches zero, the current string is the desired answer.

If the loop completes without finding enough distinct strings, the function returns an empty string.

## Go Solution

```go
func kthDistinct(arr []string, k int) string {
    frequency := make(map[string]int)

    for _, str := range arr {
        frequency[str]++
    }

    for _, str := range arr {
        if frequency[str] == 1 {
            k--

            if k == 0 {
                return str
            }
        }
    }

    return ""
}
```

The Go implementation follows the same logic as the Python solution.

A `map[string]int` is used to store frequencies. Go maps automatically initialize missing integer values to `0`, so incrementing counts is straightforward.

The first loop builds frequency counts:

```go
frequency[str]++
```

The second loop preserves original ordering while checking for distinct strings.

Unlike Python, Go does not have a built in `Counter` type, so the frequency map is manually constructed.

Returning an empty string in Go is simply:

```go
return ""
```

No special handling for nil values is needed because the function always returns a string.

The hash map may store up to `n` unique strings in the worst case, giving `O(n)` auxiliary space complexity.

## Edge Cases

One important edge case occurs when there are no distinct strings at all. For example:

```
["a", "a", "b", "b"]
```

Every string appears more than once, so no valid distinct string exists. A buggy implementation might accidentally return the first unique encounter before seeing later duplicates. The frequency counting approach avoids this by computing all counts before searching for the answer.

Another important case is when `k` exceeds the number of distinct strings. For example:

```
arr = ["a", "b", "a"]
k = 2
```

There is only one distinct string, `"b"`. The algorithm safely finishes traversal and returns `""` because `k` never reaches zero.

A third edge case is when every string is distinct. For example:

```
["a", "b", "c", "d"]
```

In this situation, the algorithm should simply return the `kth` element of the array. The implementation handles this naturally because every frequency equals `1`.

Another subtle case involves preserving original order. Consider:

```
["x", "y", "x", "z"]
```

The distinct strings are `"y"` and `"z"`, not `"x"` and `"y"`. A solution that only tracks first occurrences without checking final frequencies would fail here. The two pass hash map solution correctly identifies distinct strings only after complete counting.
