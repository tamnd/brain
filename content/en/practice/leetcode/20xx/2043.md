## Solution

## Problem Understanding

The problem requires designing a simple banking system that can handle deposits, withdrawals, and transfers between accounts. We are given `n` accounts with initial balances stored in a 0-indexed array `balance`. Each account is identified by an integer number from `1` to `n`. The operations must follow two main rules: the account numbers involved must be valid (i.e., within the range `1` to `n`), and any withdrawal or transfer must not exceed the current balance of the source account.

The expected output for each operation is a boolean indicating whether the operation was successfully executed. For instance, a deposit will always succeed if the account number is valid, whereas a withdrawal or transfer can fail if the source account has insufficient funds or if any account number is invalid.

Constraints indicate that there could be up to `10^5` accounts and balances and amounts up to `10^12`. Each operation type may be called up to `10^4` times. This ensures that our solution must handle high numbers of operations efficiently in constant time per operation. Edge cases include invalid account numbers (less than 1 or greater than `n`) and operations where the amount exceeds the current balance.

## Approaches

The brute-force approach would simply iterate over the accounts for each operation to check validity and update balances. Since the accounts are stored in a 0-indexed array, any operation requires checking indices and performing simple arithmetic. This method is already efficient because each operation modifies a single element (or two in case of a transfer) in constant time, so iteration over all accounts is unnecessary.

The optimal approach leverages direct array indexing. By using `balance[account - 1]`, we can access any account in constant time. Each deposit, withdrawal, or transfer can be performed in O(1) time with simple conditional checks to validate the operation. There are no further optimizations needed because each operation does only a few arithmetic and comparison steps.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(1) per operation | O(n) | Directly access the array for each operation, already optimal |
| Optimal | O(1) per operation | O(n) | Uses direct array indexing, no loops required, handles constraints efficiently |

## Algorithm Walkthrough

1. Initialize the bank object by storing the given `balance` array as an instance variable. This allows constant-time access to each account.
2. For the `deposit` operation, first check if the account number is valid (between `1` and `n`). If valid, add the deposit amount to the corresponding index in the `balance` array and return `true`. Otherwise, return `false`.
3. For the `withdraw` operation, check if the account number is valid. Then verify if the account has enough balance for the withdrawal. If both checks pass, subtract the amount from the balance and return `true`. Otherwise, return `false`.
4. For the `transfer` operation, check the validity of both account numbers. Verify if the source account has sufficient funds. If both conditions are met, deduct the amount from the source account and add it to the destination account, then return `true`. Otherwise, return `false`.

**Why it works**: By maintaining balances in a direct-access array and performing validation checks for each operation, the algorithm guarantees correctness while operating in constant time per call. The invariant is that `balance[i]` always stores the current balance of account `i + 1`.

## Python Solution

```python
from typing import List

class Bank:

    def __init__(self, balance: List[int]):
        self.balance = balance
        self.n = len(balance)

    def transfer(self, account1: int, account2: int, money: int) -> bool:
        if 1 <= account1 <= self.n and 1 <= account2 <= self.n and self.balance[account1 - 1] >= money:
            self.balance[account1 - 1] -= money
            self.balance[account2 - 1] += money
            return True
        return False

    def deposit(self, account: int, money: int) -> bool:
        if 1 <= account <= self.n:
            self.balance[account - 1] += money
            return True
        return False

    def withdraw(self, account: int, money: int) -> bool:
        if 1 <= account <= self.n and self.balance[account - 1] >= money:
            self.balance[account - 1] -= money
            return True
        return False
```

The Python implementation follows the algorithm closely, using array indexing for constant-time access and conditional checks for operation validity. Deposits always succeed if the account is valid, while withdrawals and transfers also require sufficient funds.

## Go Solution

```go
type Bank struct {
    balance []int64
}

func Constructor(balance []int64) Bank {
    return Bank{balance: balance}
}

func (this *Bank) Transfer(account1 int, account2 int, money int64) bool {
    n := len(this.balance)
    if account1 < 1 || account1 > n || account2 < 1 || account2 > n || this.balance[account1-1] < money {
        return false
    }
    this.balance[account1-1] -= money
    this.balance[account2-1] += money
    return true
}

func (this *Bank) Deposit(account int, money int64) bool {
    n := len(this.balance)
    if account < 1 || account > n {
        return false
    }
    this.balance[account-1] += money
    return true
}

func (this *Bank) Withdraw(account int, money int64) bool {
    n := len(this.balance)
    if account < 1 || account > n || this.balance[account-1] < money {
        return false
    }
    this.balance[account-1] -= money
    return true
}
```

In Go, the solution is nearly identical. The main difference is the use of `int64` to handle large balances, and the `Bank` struct is initialized using a constructor function. Slice indexing allows constant-time access.
