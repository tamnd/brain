## Solution

## Problem Understanding

The problem is asking us to convert a range of IP addresses into a minimal set of CIDR blocks, ensuring that every IP in the range is covered **exactly once**, and no IP outside the range is included.

The input consists of a starting IP address in the form `"a.b.c.d"` and a number `n` indicating how many consecutive IPs need to be covered. The output is a list of CIDR strings, each representing a contiguous block of IPs.

Key constraints include:

- The IP is guaranteed to be valid and in IPv4 format.
- The number of IPs `n` is at most 1000, which allows for bitwise operations without performance issues.
- We must ensure **no extra IP addresses** outside the given range are included. This is the primary constraint that rules out simply covering the range with the largest possible block.

Important edge cases are:

- `n = 1`, meaning only a single IP needs coverage.
- IP addresses that are near the 32-bit boundary (`255.255.255.255`) and wraparound issues.
- When the starting IP is not aligned on a boundary that a large CIDR block would naturally cover.

The problem is essentially a **bit manipulation and greedy coverage problem**, where the goal is to take the starting IP and cover as many addresses as possible in powers of two without exceeding `n` or covering IPs outside the range.

## Approaches

### Brute-Force Approach

A naive approach would be to iterate over each IP in the range and assign it its own `/32` CIDR block. This approach is guaranteed to work but is extremely inefficient, especially for large `n`. Each IP would be processed individually, and we would end up with `n` CIDR blocks, which is not minimal. While correct, it does not leverage the properties of IP alignment or powers-of-two block sizes.

### Optimal Approach

The optimal approach relies on **bit manipulation and a greedy strategy**. The key observations are:

1. CIDR blocks cover ranges in powers of two. The suffix `/k` means the last `32-k` bits can vary, resulting in a block of size `2^(32-k)`.
2. At each step, the largest block we can take is determined by the **least significant bit set in the current IP** (this gives the maximal alignment) and the number of remaining IPs `n`.
3. By taking the maximal aligned block at each step, we greedily reduce `n` while ensuring no extra IPs are included.

The algorithm repeatedly calculates the largest valid block size, converts the current IP to CIDR, subtracts that block from `n`, and increments the IP, until all IPs are covered.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Each IP covered individually, not minimal. |
| Optimal | O(log n) | O(log n) | Greedy coverage using powers of two, converting IPs with bit manipulation. |

## Algorithm Walkthrough

1. Convert the starting IP string into a 32-bit integer. This allows for arithmetic and bit manipulation.
2. Initialize an empty list to store CIDR blocks.
3. While `n > 0`:

1. Identify the **largest power-of-two block** that the current IP can start with. This is determined by the lowest set bit in the integer representation of the IP (`ip & -ip`).
2. Compute the largest block size that **does not exceed the remaining IP count `n`**.
3. Determine the prefix length `k` from the block size using the formula `32 - log2(block size)`.
4. Convert the current IP integer back to standard dot notation and append the `/k` CIDR string to the result list.
5. Increment the IP integer by the block size, and decrement `n` by the same amount.
4. Return the list of CIDR blocks.

**Why it works:** Each step selects the largest possible aligned block while remaining within the constraints. This greedy choice guarantees the minimal number of blocks because any smaller block would require more CIDR strings to cover the same IP range.

## Python Solution

```
from typing import List
import math

class Solution:
    def ipToCIDR(self, ip: str, n: int) -> List[str]:
        def ip_to_int(ip_str: str) -> int:
            parts = list(map(int, ip_str.split('.')))
            res = 0
            for part in parts:
                res = res * 256 + part
            return res
        
        def int_to_ip(ip_int: int) -> str:
            return '.'.join(str((ip_int >> (8 * i)) % 256) for i in reversed(range(4)))
        
        result = []
        start = ip_to_int(ip)
        
        while n > 0:
            # Largest power of two divisible by current IP
            lsb = start & -start
            block_size = lsb
            # Limit block size by remaining IPs
            while block_size > n:
                block_size //= 2
            # Determine prefix length
            prefix_length = 32 - int(math.log2(block_size))
            result.append(f"{int_to_ip(start)}/{prefix_length}")
            start += block_size
            n -= block_size
        
        return result
```

**Explanation:**

We first convert the IP to an integer for easy arithmetic. The `lsb` calculation (`start & -start`) identifies the largest block that aligns with the current IP. The `while block_size > n` ensures we do not cover more than the remaining addresses. The prefix length is derived from the block size, then converted back to string format for output. This loop continues until all addresses are covered.

## Go Solution

```
import (
    "fmt"
    "math/bits"
    "strconv"
    "strings"
)

func ipToCIDR(ip string, n int) []string {
    ipToInt := func(ipStr string) uint32 {
        parts := strings.Split(ipStr, ".")
        var res uint32
        for _, part := range parts {
            num, _ := strconv.Atoi(part)
            res = res*256 + uint32(num)
        }
        return res
    }
    
    intToIP := func(ipInt uint32) string {
        return fmt.Sprintf("%d.%d.%d.%d", (ipInt>>24)&255, (ipInt>>16)&255, (ipInt>>8)&255, ipInt&255)
    }
    
    res := []string{}
    start := ipToInt(ip)
    
    for n > 0 {
        lsb := start & -start
        blockSize := lsb
        for int(blockSize) > n {
            blockSize >>= 1
        }
        prefixLength := 32 - bits.Len32(blockSize) + 1
        res = append(res, fmt.Sprintf("%s/%d", intToIP(start), prefixLength))
        start += blockSize
        n -= int(blockSize)
    }
    
    return res
}
```

**Go-specific notes:**

`bits.Len32` is used to compute the number of bits in the block size for prefix calculation. Integer overflow is not an issue due to `uint32`. Slices are used to store results. Conversion between string and integer uses `strconv.Atoi` and `fmt.Sprintf`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(log n) | Each iteration covers at least one IP, often a power-of-two block; total iterations bounded by log n. |
| Space | O(log n) | Number of CIDR blocks stored is at most log n for the range. |

The logarithmic behavior comes from greedily covering the largest aligned block at each step, effectively halving the remaining range when possible.
