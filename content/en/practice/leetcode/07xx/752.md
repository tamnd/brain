## Solution

## Problem Understanding

The problem presents a 4-wheel combination lock, where each wheel can display digits from `'0'` to `'9'`. Each wheel can be rotated forward or backward by one slot in a single move, and the wheels wrap around such that `'9'` can become `'0'` and `'0'` can become `'9'`. The lock starts at the combination `'0000'`, and the goal is to reach a given `target` combination in the minimum number of moves.

However, there are certain `deadends`-combinations that, if reached, prevent further movement of the lock. The challenge is to navigate from `'0000'` to `target` without ever passing through a deadend. The input consists of a list of deadends, and a string `target` representing the combination to unlock the lock. The expected output is the minimum number of moves required, or `-1` if it is impossible to reach the target without hitting a deadend.

The constraints provide that the number of deadends is at most 500 and each combination is always of length 4. The `target` is guaranteed not to be in the deadends. Important edge cases include the initial state `'0000'` being a deadend, or the target being just one move away. The problem scale is small enough that exhaustive search is feasible, but optimization is necessary to avoid redundant work.

## Approaches

### Brute Force

The brute-force approach would be to perform an uninformed search from `'0000'` to all possible 4-digit combinations, keeping track of the minimum number of moves to reach the target. This could be done recursively or iteratively, generating all 10,000 possible combinations. While this would eventually find a solution, it is inefficient because it explores many redundant states and does not efficiently avoid deadends or previously visited combinations.

### Optimal Approach

The optimal solution uses **Breadth-First Search (BFS)**, which is suitable because each move has the same cost, and we are asked for the minimum number of moves. BFS systematically explores all reachable states level by level, starting from `'0000'`. At each step, it generates all valid next states by incrementing or decrementing each wheel by one, while skipping any states that are in the `deadends` or already visited. This ensures the first time we reach the target, it is via the minimum number of moves.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(10^4) | O(10^4) | Explores all combinations without pruning, inefficient |
| BFS Optimal | O(10^4) | O(10^4) | Explores states level by level, avoids deadends and revisits |

## Algorithm Walkthrough

1. Convert the `deadends` list into a set for O(1) lookup.
2. Handle the edge case where the initial state `'0000'` is a deadend by returning `-1` immediately.
3. Initialize a queue with the tuple `('0000', 0)` representing the state and the number of moves.
4. Maintain a `visited` set to track combinations we have already processed to avoid revisiting.
5. While the queue is not empty, dequeue the front state. If it matches the `target`, return the number of moves.
6. For each wheel (0 to 3), generate two new states by incrementing or decrementing the digit. Wrap around using modulo logic to stay within `'0'` to `'9'`.
7. If a generated state is not a deadend and not visited, add it to the queue and mark it as visited.
8. If the queue empties without finding the target, return `-1` as it is unreachable.

**Why it works**: BFS guarantees that the first time we encounter the target, it is through the shortest sequence of moves because all states at distance `k` are explored before states at distance `k+1`. Using sets for `deadends` and `visited` ensures efficient pruning of invalid or redundant paths.

## Python Solution

```
from collections import deque
from typing import List

class Solution:
    def openLock(self, deadends: List[str], target: str) -> int:
        dead_set = set(deadends)
        if '0000' in dead_set:
            return -1
        
        queue = deque([('0000', 0)])
        visited = {'0000'}
        
        while queue:
            state, moves = queue.popleft()
            if state == target:
                return moves
            
            for i in range(4):
                digit = int(state[i])
                for delta in [-1, 1]:
                    new_digit = (digit + delta) % 10
                    new_state = state[:i] + str(new_digit) + state[i+1:]
                    if new_state not in dead_set and new_state not in visited:
                        visited.add(new_state)
                        queue.append((new_state, moves + 1))
        
        return -1
```

**Explanation**: We first convert the deadends into a set for fast lookup and check if the starting point `'0000'` is blocked. We use a deque to implement BFS, enqueueing new states along with their move count. Each wheel can turn forward or backward, generating neighboring states. We skip deadends and previously visited states. BFS ensures that the first time the target is reached is via the minimum moves.

## Go Solution

```
package main

import "fmt"

func openLock(deadends []string, target string) int {
    deadSet := make(map[string]bool)
    for _, d := range deadends {
        deadSet[d] = true
    }
    if deadSet["0000"] {
        return -1
    }
    
    type stateMove struct {
        state string
        moves int
    }
    
    queue := []stateMove{{"0000", 0}}
    visited := map[string]bool{"0000": true}
    
    for len(queue) > 0 {
        curr := queue[0]
        queue = queue[1:]
        
        if curr.state == target {
            return curr.moves
        }
        
        for i := 0; i < 4; i++ {
            digit := int(curr.state[i] - '0')
            for _, delta := range []int{-1, 1} {
                newDigit := (digit + delta + 10) % 10
                newState := curr.state[:i] + string('0'+newDigit) + curr.state[i+1:]
                if !deadSet[newState] && !visited[newState] {
                    visited[newState] = true
                    queue = append(queue, stateMove{newState, curr.moves + 1})
                }
            }
        }
    }
    
    return -1
}
```

**Explanation**: The Go version mirrors the Python logic. A `map` is used for `deadends` and `visited` to ensure O(1) lookups. BFS is implemented using a slice as a queue. We take care of Go string indexing and converting digits correctly. The modulo arithmetic ensures wrap-around of digits.
