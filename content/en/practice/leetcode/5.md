## Solution

## Problem Understanding

The problem asks us to find the longest substring of a given string that is also a palindrome.

A palindrome is a sequence of characters that reads the same forward and backward. For example, `"racecar"`, `"abba"`, and `"a"` are all palindromes. A substring is a contiguous portion of the string, which means the characters must appear next to each other in the original string.

The input is a single string `s`, containing only English letters and digits. The length of the string is between 1 and 1000 inclusive. The output should be the longest palindromic substring found anywhere inside the string.

For example, if the input is `"babad"`, there are two valid longest palindromic substrings: `"bab"` and `"aba"`. Either answer is accepted because both have length 3, which is the maximum possible for that input.

The constraint of at most 1000 characters is important because it rules out extremely expensive algorithms, but it still allows quadratic time solutions. An algorithm with cubic complexity may pass in some languages but is unnecessarily slow. A well-designed O(n²) solution is the standard expectation for this problem.

There are several edge cases that deserve attention:

- A string of length 1 is already a palindrome, so the answer is the string itself.
- A string where all characters are different, such as `"abcde"`, has no palindrome longer than one character.
- A string where all characters are the same, such as `"aaaaa"`, forms one large palindrome spanning the entire string.
- Even-length palindromes such as `"abba"` behave differently from odd-length palindromes such as `"aba"`, so the implementation must correctly handle both cases.
- Multiple longest palindromes may exist, and returning any valid one is acceptable.

## Approaches

### Brute Force Approach

The most direct solution is to generate every possible substring and check whether each substring is a palindrome.

A string of length `n` has O(n²) possible substrings. For each substring, we can check whether it is a palindrome by comparing characters from both ends inward. That palindrome check itself takes O(n) time in the worst case.

This leads to a total time complexity of O(n³).

The brute-force solution is correct because it exhaustively checks every substring and keeps track of the longest palindrome found. Since no substring is skipped, the final answer must be correct.

However, this approach becomes inefficient as the string grows larger. With a maximum input size of 1000 characters, O(n³) operations can become too slow.

### Optimal Approach, Expand Around Center

The key observation is that every palindrome has a center.

For odd-length palindromes like `"aba"`, the center is a single character. For even-length palindromes like `"abba"`, the center lies between two characters.

Instead of generating every substring, we can start from each possible center and expand outward as long as the characters on both sides match.

For example, in `"racecar"`:

- Start at center `'e'`
- Expand to `'cec'`
- Expand to `'aceca'`
- Expand to `'racecar'`

This works because palindromes are symmetric. Once the symmetry breaks, expansion stops immediately.

There are `2n - 1` possible centers:

- `n` odd-length centers
- `n - 1` even-length centers

Each expansion can take up to O(n) time in the worst case, giving a total complexity of O(n²). This is efficient enough for the given constraints and requires only constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Generates every substring and checks each one |
| Optimal | O(n²) | O(1) | Expands outward from every possible palindrome center |

## Algorithm Walkthrough

1. Initialize variables to track the longest palindrome found so far.

We store the starting index and ending index of the best palindrome discovered during processing. Initially, the longest palindrome can simply be the first character of the string.
2. Iterate through every character index in the string.

Each index can serve as the center of an odd-length palindrome. In addition, the gap between the current character and the next character can serve as the center of an even-length palindrome.
3. Expand around the odd-length center.

Set two pointers:

- `left = i`
- `right = i`

While both pointers remain inside the string and the characters match, continue expanding outward.

This detects palindromes like `"aba"` or `"racecar"`.
4. Expand around the even-length center.

Set:

- `left = i`
- `right = i + 1`

Again, expand outward while characters match.

This detects palindromes like `"abba"` or `"cbbd"`.
5. Compare the lengths of the discovered palindromes.

After each expansion, compute the palindrome length. If it is longer than the currently stored best palindrome, update the answer boundaries.
6. Continue until all centers have been processed.

Since every palindrome must have a center, checking all centers guarantees that the longest palindrome will be found.
7. Return the substring defined by the best boundaries.

In Python, slicing is straightforward. In Go, slicing with indices is also efficient because strings are immutable byte slices.

## Python Solution

```
class Solution:
    def longestPalindrome(self, s: str) -> str:
        if len(s) <= 1:
            return s

        start = 0
        end = 0

        def expand(left: int, right: int) -> tuple[int, int]:
            while left >= 0 and right < len(s) and s[left] == s[right]:
                left -= 1
                right += 1

            return left + 1, right - 1

        for i in range(len(s)):
            odd_left, odd_right = expand(i, i)

            if odd_right - odd_left > end - start:
                start, end = odd_left, odd_right

            even_left, even_right = expand(i, i + 1)

            if even_right - even_left > end - start:
                start, end = even_left, even_right

        return s[start:end + 1]
```

The implementation begins by handling the simplest case where the string length is 0 or 1. In such cases, the string itself is already the longest palindrome.

The variables `start` and `end` store the boundaries of the best palindrome found so far.

The helper function `expand` performs the core logic. It receives two indices representing the center of a potential palindrome. While the characters at both pointers match, it expands outward. Once expansion stops, the valid palindrome boundaries are returned.

Inside the main loop, each index is processed twice:

- Once for odd-length palindromes using `(i, i)`
- Once for even-length palindromes using `(i, i + 1)`

After each expansion, the resulting palindrome length is compared against the current best. If the new palindrome is longer, the answer boundaries are updated.

Finally, the substring from `start` to `end` inclusive is returned.

## Go Solution

```
func longestPalindrome(s string) string {
    if len(s) <= 1 {
        return s
    }

    start := 0
    end := 0

    expand := func(left int, right int) (int, int) {
        for left >= 0 && right < len(s) && s[left] == s[right] {
            left--
            right++
        }

        return left + 1, right - 1
    }

    for i := 0; i < len(s); i++ {
        oddLeft, oddRight := expand(i, i)

        if oddRight-oddLeft > end-start {
            start = oddLeft
            end = oddRight
        }

        evenLeft, evenRight := expand(i, i+1)

        if evenRight-evenLeft > end-start {
            start = evenLeft
            end = evenRight
        }
    }

    return s[start : end+1]
}
```

The Go implementation closely mirrors the Python version. The main difference is syntax and string handling.

Go strings are immutable byte slices. Since the problem guarantees only English letters and digits, indexing by byte is safe because each character occupies exactly one byte. If Unicode characters were involved, rune handling would be necessary.

The helper function is implemented as a closure assigned to `expand`. It returns the valid palindrome boundaries after expansion stops.

There are no concerns about integer overflow because the string length is at most 1000.

The slicing operation `s[start : end+1]` correctly extracts the final palindrome substring.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | Each center may expand across the string |
| Space | O(1) | Only a few pointer variables are used |

The algorithm examines every possible palindrome center. There are O(n) centers, and each expansion can take O(n) time in the worst case, such as when the string consists entirely of identical characters.

No additional data structures proportional to the input size are allocated. The algorithm only stores a few integer indices and temporary variables, so the space complexity remains constant.

## Edge Cases

A single-character string such as `"a"` is the smallest valid input. A naive implementation might incorrectly assume there must be at least two characters for a palindrome. This solution correctly handles the case immediately because every single character is itself a palindrome.

A string with no repeated characters, such as `"abcdef"`, can also cause issues in implementations that expect expansions to succeed frequently. In this case, every palindrome has length 1. The algorithm still works because every character is treated as an odd-length center, guaranteeing at least one valid palindrome.

Even-length palindromes are a common source of bugs. Many incorrect implementations only consider odd-length centers and therefore fail on inputs like `"abba"` or `"cbbd"`. This solution explicitly checks both `(i, i)` and `(i, i + 1)` centers, ensuring even-length palindromes are detected correctly.

Strings where all characters are identical, such as `"aaaaaa"`, represent the worst-case expansion scenario because every expansion continues until the boundaries of the string. The implementation still works correctly and remains within O(n²) complexity.

Boundary handling is another important edge case. During expansion, pointers may move beyond the valid string indices. The while-loop conditions ensure bounds are checked before any character access occurs, preventing out-of-range errors in both Python and Go.
