# LeetCode 3775 - Reverse Words With Same Vowel Count

## Problem Understanding

The problem gives us a string `s` containing lowercase English words separated by single spaces. We need to process the words based on the number of vowels contained in the first word.

The first step is to determine how many vowels appear in the first word. This vowel count becomes our reference value. After that, we examine every word that comes after the first one. If a word contains exactly the same number of vowels as the first word, we reverse that word. Otherwise, we leave it unchanged.

The first word itself is never modified. It is only used to determine the target vowel count.

For example, if the first word contains one vowel, then every subsequent word with exactly one vowel must be reversed. Words with zero vowels, two vowels, or any other vowel count remain unchanged.

The input consists of:

- Lowercase English letters and spaces.
- Exactly one space between adjacent words.
- No leading or trailing spaces.

The output is a new string formed by joining the transformed words with single spaces.

The constraint `1 <= s.length <= 10^5` is important. The input can be quite large, so the solution should process the string efficiently. Ideally, we should scan each character only a small number of times, leading to linear time complexity.

Several edge cases are worth considering:

- The string may contain only one word.
- Every word may have the same vowel count as the first word.
- No later word may match the first word's vowel count.
- Words may contain zero vowels.
- Very long inputs near the maximum length require an efficient linear solution.

## Approaches

### Brute Force Approach

A straightforward solution is to repeatedly compute vowel counts whenever needed.

We first split the string into words and determine the vowel count of the first word. Then, for each remaining word, we count its vowels by scanning all characters. If the count matches the reference count, we reverse the word.

This approach is already reasonably efficient because each word is processed independently. However, from a conceptual standpoint, it does not take advantage of any special observations beyond directly following the problem statement.

The correctness is immediate because every word is examined, its vowel count is computed exactly, and the reversal rule is applied whenever the counts match.

### Key Insight

The key observation is that each character only needs to be examined once while computing vowel counts. Since every word must be inspected at least once to determine its vowel count, any optimal solution will have linear complexity with respect to the total number of characters.

Therefore, the natural optimal strategy is:

1. Split the string into words.
2. Compute the vowel count of the first word.
3. Process every remaining word exactly once.
4. Reverse a word only when its vowel count matches the reference count.
5. Join all words back together.

This achieves linear time complexity and is optimal because every character contributes to the final decision.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Split words, count vowels for each word, reverse when needed |
| Optimal | O(n) | O(n) | Single pass over all words and characters, optimal for the input size |

Here, `n` denotes the length of the entire string.

## Algorithm Walkthrough

1. Split the input string into a list of words using spaces.
2. Create a vowel set containing `'a'`, `'e'`, `'i'`, `'o'`, and `'u'`. This allows constant-time vowel membership checks.
3. Count the vowels in the first word. This value becomes the target vowel count that all later words will be compared against.
4. Iterate through the words starting from index `1`, because the first word is never modified.
5. For each word, count how many characters belong to the vowel set.
6. Compare the current word's vowel count with the target count.
7. If the counts are equal, replace the word with its reversed version.
8. If the counts differ, leave the word unchanged.
9. After processing all words, join them back together using single spaces.
10. Return the resulting string.

### Why it works

The algorithm computes the exact vowel count of the first word and stores it as the reference value. Every subsequent word is independently checked against this reference. A word is reversed if and only if its vowel count equals the reference count, which matches the problem specification exactly. Since every word is processed once and the transformation rule is applied correctly, the final string is guaranteed to be correct.

## Python Solution

```python
class Solution:
    def reverseWords(self, s: str) -> str:
        vowels = {"a", "e", "i", "o", "u"}

        words = s.split()

        target_count = sum(1 for ch in words[0] if ch in vowels)

        for i in range(1, len(words)):
            current_count = sum(1 for ch in words[i] if ch in vowels)

            if current_count == target_count:
                words[i] = words[i][::-1]

        return " ".join(words)
```

The implementation begins by creating a set of vowels for efficient membership testing.

The string is then split into individual words. Since the problem guarantees single spaces and no leading or trailing spaces, `split()` produces exactly the desired list of words.

Next, the vowel count of the first word is computed and stored in `target_count`.

The algorithm then iterates through all remaining words. For each word, it counts the vowels and compares that count with `target_count`. When the counts match, Python's slicing syntax `[::-1]` is used to reverse the word.

Finally, all processed words are joined together with spaces and returned.

## Go Solution

```go
func reverseWords(s string) string {
	words := strings.Split(s, " ")

	isVowel := func(ch byte) bool {
		return ch == 'a' || ch == 'e' || ch == 'i' || ch == 'o' || ch == 'u'
	}

	countVowels := func(word string) int {
		count := 0
		for i := 0; i < len(word); i++ {
			if isVowel(word[i]) {
				count++
			}
		}
		return count
	}

	targetCount := countVowels(words[0])

	for i := 1; i < len(words); i++ {
		if countVowels(words[i]) == targetCount {
			bytes := []byte(words[i])

			for left, right := 0, len(bytes)-1; left < right; {
				bytes[left], bytes[right] = bytes[right], bytes[left]
				left++
				right--
			}

			words[i] = string(bytes)
		}
	}

	return strings.Join(words, " ")
}
```

The Go solution follows exactly the same algorithm. A helper function counts vowels, and another helper checks whether a character is a vowel.

Unlike Python, Go does not provide built-in string reversal syntax. Therefore, matching words are converted to a byte slice and reversed using the classic two-pointer technique.

Since the input contains only lowercase English letters, byte-based processing is completely safe and efficient.

## Edge Cases

### Single Word Input

The smallest valid input may contain only one word. In this situation there are no later words to examine or reverse. The algorithm naturally handles this because the processing loop begins at index `1`, which is never reached. The original word is returned unchanged.

### Words With Zero Vowels

A word can contain no vowels at all, such as `"rhythm"`. If the first word has zero vowels, then only later words with zero vowels should be reversed. The vowel-counting logic correctly returns zero, so comparisons remain valid without any special handling.

### Every Word Matches

Consider an input like `"u up under"`. The first word contains one vowel, and every later word also contains one vowel. In this case every subsequent word must be reversed. The algorithm checks each word independently and reverses all matching words, producing the correct result.

### No Word Matches

An input such as `"hello world code"` may contain no later words with the same vowel count as the first word. The algorithm still computes vowel counts for all words, but no reversals occur. Since words are modified only when counts match exactly, the original string structure is preserved correctly.

### Large Inputs Near the Constraint Limit

The total string length can be as large as `10^5`. Any solution that repeatedly rescans large portions of the string would risk inefficiency. This implementation processes each character only a constant number of times, giving O(n) time complexity and making it suitable for the maximum input size.
