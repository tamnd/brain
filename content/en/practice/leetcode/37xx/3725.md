# LeetCode 3725 - Count Ways to Choose Coprime Integers from Rows

## Problem Understanding

We are given an `m × n` matrix where every entry is a positive integer between `1` and `150`. We must choose exactly one number from each row. After making one choice per row, we compute the greatest common divisor (GCD) of all selected numbers.

Our goal is to count how many such selections produce a final GCD equal to `1`.

For example, if we select one value from each row and the resulting numbers are `[6, 10, 15]`, their GCD is `1`, so this selection contributes to the answer. If the selected numbers are `[6, 12, 18]`, their GCD is `6`, so this selection does not contribute.

The answer can become extremely large because there may be up to `150` choices in each of `150` rows. Therefore, we must return the result modulo `10^9 + 7`.

The constraints contain a very important observation:

- The matrix dimensions can be as large as `150 × 150`.
- Values themselves are also bounded by `150`.

A naive enumeration would need to examine up to `150^150` combinations, which is completely impossible. However, every value belongs to the very small range `[1, 150]`. This suggests that the state space should be based on possible GCD values rather than on the chosen numbers themselves.

Important edge cases include situations where every number is divisible by the same integer greater than `1`, making the answer zero. Another important case is when some row contains the value `1`, since any selection including that value immediately forces the running GCD to become `1`. Duplicate values inside rows must also be handled correctly because each occurrence represents a distinct choice.

## Approaches

### Brute Force

The most direct approach is to recursively choose one element from every row.

For each complete selection, compute the GCD of all chosen numbers and count the selection if the GCD equals `1`.

This approach is correct because it explicitly examines every possible valid selection. However, its complexity is astronomical.

If each row contains `n` values and there are `m` rows, then there are `n^m` possible selections. Under the maximum constraints this becomes `150^150`, which is far beyond practical computation.

### Key Insight

Instead of tracking which numbers were chosen, we track only the GCD of the chosen numbers so far.

Suppose we have already processed some rows and the current GCD equals `g`.

When we choose a value `x` from the next row, the new GCD becomes:

$$\gcd(g, x)$$

Since every matrix value is at most `150`, every possible GCD is also at most `150`.

Therefore, instead of an enormous state space, we have at most `150` possible GCD states.

We can perform dynamic programming where:

- State = current GCD after processing several rows.
- Transition = choose a value from the next row and update the GCD.

This reduces the problem from exponential in the number of rows to polynomial in the value range.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^m · m) | O(m) | Enumerates every selection |
| Optimal DP on GCD States | O(m · n · 150) | O(150) | Uses bounded GCD state space |

## Algorithm Walkthrough

### Step 1

Create a DP array where `dp[g]` represents the number of ways to process the rows seen so far and obtain a current GCD equal to `g`.

Since values are bounded by `150`, we only need indices `1..150`.

### Step 2

Initialize the DP using the first row.

For every value `x` in the first row, selecting that value alone produces a current GCD of `x`.

Therefore:

```
dp[x] += 1
```

### Step 3

Process rows one by one.

For each new row, create a fresh array `next_dp`.

### Step 4

For every existing GCD state `g` and every value `x` in the current row:

1. Choose value `x`.
2. Compute the new GCD:

```
new_g = gcd(g, x)
```
3. Add the number of ways represented by `dp[g]` into:

```
next_dp[new_g]
```

All updates are performed modulo `10^9 + 7`.

### Step 5

After processing the current row, replace:

```
dp = next_dp
```

This ensures that `dp[g]` always represents results after all processed rows.

### Step 6

After the final row is processed, the answer is:

```
dp[1]
```

because we only want selections whose overall GCD equals `1`.

### Why it works

The key invariant is that after processing any prefix of rows, `dp[g]` stores exactly the number of selections from those rows whose GCD is `g`.

Initially this is true for the first row. During a transition, every possible next choice is considered, and the GCD property

$$\gcd(a_1,a_2,\ldots,a_k,x)
=
\gcd(\gcd(a_1,a_2,\ldots,a_k),x)$$

guarantees that the new state is precisely `gcd(g, x)`. Therefore every valid selection contributes to exactly one state and no selection is missed or counted twice. By induction, the invariant remains true throughout the algorithm, and `dp[1]` after the last row is exactly the number of valid selections.

## Python Solution

```python
from typing import List
from math import gcd

class Solution:
    def countCoprime(self, mat: List[List[int]]) -> int:
        MOD = 10**9 + 7
        MAX_VAL = 150

        dp = [0] * (MAX_VAL + 1)

        for value in mat[0]:
            dp[value] = (dp[value] + 1) % MOD

        for row in mat[1:]:
            next_dp = [0] * (MAX_VAL + 1)

            for current_gcd in range(1, MAX_VAL + 1):
                ways = dp[current_gcd]
                if ways == 0:
                    continue

                for value in row:
                    new_gcd = gcd(current_gcd, value)
                    next_dp[new_gcd] = (
                        next_dp[new_gcd] + ways
                    ) % MOD

            dp = next_dp

        return dp[1]
```

### Implementation Explanation

The solution begins by creating a DP array indexed by possible GCD values. Since all numbers are at most `150`, every reachable GCD also lies in the same range.

The first row initializes the DP. Choosing a value from the first row creates exactly one partial selection whose GCD equals that value.

For every subsequent row, a new DP array is constructed. Every existing GCD state is combined with every value in the current row. The resulting state is computed using `math.gcd`.

The count of ways associated with the old state is added to the new state. All additions are performed modulo `10^9 + 7`.

After all rows have been processed, `dp[1]` contains the number of complete selections whose overall GCD equals `1`.

## Go Solution

```go
package main

func gcd(a, b int) int {
	for b != 0 {
		a, b = b, a%b
	}
	return a
}

func countCoprime(mat [][]int) int {
	const MOD = 1000000007
	const MAXV = 150

	dp := make([]int, MAXV+1)

	for _, v := range mat[0] {
		dp[v] = (dp[v] + 1) % MOD
	}

	for i := 1; i < len(mat); i++ {
		nextDP := make([]int, MAXV+1)

		for g := 1; g <= MAXV; g++ {
			if dp[g] == 0 {
				continue
			}

			for _, v := range mat[i] {
				ng := gcd(g, v)
				nextDP[ng] = (nextDP[ng] + dp[g]) % MOD
			}
		}

		dp = nextDP
	}

	return dp[1]
}
```

### Go Specific Notes

The Go implementation follows the same state transition logic as the Python version.

A custom iterative Euclidean algorithm is used for GCD computation. Integer overflow is not an issue because every update is reduced modulo `1,000,000,007`. Dynamic slices are used for the DP arrays, and a new slice is allocated for each row to avoid accidental reuse of previous states.

## Edge Cases

### Single Row Matrix

When there is only one row, choosing exactly one number means the final GCD is simply the chosen number itself. The answer becomes the number of entries equal to `1`. A solution that assumes at least two rows may incorrectly perform transitions that never occur. The DP initialization naturally handles this case because the algorithm immediately returns `dp[1]`.

### All Numbers Share a Common Divisor

Consider a matrix where every entry is even. No matter which values are selected, the final GCD must remain divisible by `2`. Therefore the answer is zero. The DP correctly reflects this because state `1` is never reached.

### Duplicate Values Within Rows

Rows may contain repeated numbers. Each matrix position represents a distinct choice, even if values are identical. For example, in `[[1,1],[2,2]]`, there are four valid selections, not one. The implementation processes every element occurrence individually, ensuring duplicates contribute the correct multiplicity to the count.

### Presence of the Value 1

Once a running GCD becomes `1`, all future GCD computations remain `1` because `gcd(1, x) = 1`. This could easily be mishandled by an incorrect optimization. The DP naturally captures this property because all transitions from state `1` remain in state `1`, accumulating the correct number of ways.

### Maximum Constraints

The matrix can contain up to `22,500` entries, but every value remains bounded by `150`. The algorithm leverages this small value range, ensuring the number of DP states never exceeds `150`, which keeps the solution efficient even at the largest input size.
