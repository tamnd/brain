# LeetCode 3774 - Absolute Difference Between Maximum and Minimum K Elements

## Problem Understanding

We are given an integer array `nums` and an integer `k`.

The task is to find two quantities:

1. The sum of the `k` largest elements in the array.
2. The sum of the `k` smallest elements in the array.

After computing these two sums, we return their absolute difference.

In other words, we want:

$$\left|(\text{sum of k largest}) - (\text{sum of k smallest})\right|$$

The input array can contain duplicate values, and duplicates must be treated as separate elements. For example, if the array contains `[2, 2, 4, 5]` and `k = 2`, then the two smallest elements are `2` and `2`, not `2` and `4`.

The constraints are very small:

- `1 <= nums.length <= 100`
- `1 <= nums[i] <= 100`
- `1 <= k <= n`

Since the array contains at most 100 elements, even relatively simple algorithms are efficient enough.

Several edge cases are important:

- When `n = 1`, the largest and smallest elements are the same.
- When `k = n`, both sums include all elements, so the answer is always `0`.
- Arrays may contain duplicate values.
- Arrays may already be sorted or completely unsorted.

The problem guarantees that `k` is always valid and does not exceed the array length.

## Approaches

### Brute Force Approach

A straightforward solution is to examine every element and determine whether it belongs among the `k` smallest or `k` largest values.

One way to do this is to repeatedly search for the next smallest and next largest elements. This would correctly identify the required elements, but it requires repeatedly scanning the array.

Although this approach produces the correct result, it performs unnecessary work because the relative ordering of the elements is what matters.

### Key Insight

The problem only asks for the smallest and largest elements.

If we sort the array in ascending order:

- The first `k` elements are exactly the `k` smallest elements.
- The last `k` elements are exactly the `k` largest elements.

After sorting, we simply compute:

- `sum(nums[:k])`
- `sum(nums[n-k:])`

and return their absolute difference.

Because the array size is at most 100, sorting is more than fast enough.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(nk) | O(1) | Repeatedly search for smallest and largest elements |
| Optimal | O(n log n) | O(n) or O(1)* | Sort array, then sum first and last k elements |

* Depending on the language and sorting implementation.

## Algorithm Walkthrough

1. Create a sorted version of the input array in ascending order.
2. Compute the sum of the first `k` elements. These are the `k` smallest values because the array is sorted.
3. Compute the sum of the last `k` elements. These are the `k` largest values because the array is sorted.
4. Subtract the two sums and take the absolute value.
5. Return the result.

### Why It Works

Sorting places all elements in nondecreasing order. Therefore, the first `k` positions contain exactly the `k` smallest elements, and the last `k` positions contain exactly the `k` largest elements. Since the problem only asks for the sums of those groups, computing the sums directly from the sorted array yields the correct answer.

## Python Solution

```python
from typing import List

class Solution:
    def absDifference(self, nums: List[int], k: int) -> int:
        nums.sort()

        smallest_sum = sum(nums[:k])
        largest_sum = sum(nums[-k:])

        return abs(largest_sum - smallest_sum)
```

### Implementation Explanation

The solution begins by sorting the array in ascending order.

After sorting:

- `nums[:k]` contains the `k` smallest elements.
- `nums[-k:]` contains the `k` largest elements.

We compute both sums using Python's built-in `sum()` function and return the absolute value of their difference.

The implementation is concise because sorting directly provides the ordering information required by the problem.

## Go Solution

```go
package main

import "sort"

func absDifference(nums []int, k int) int {
	sort.Ints(nums)

	smallestSum := 0
	for i := 0; i < k; i++ {
		smallestSum += nums[i]
	}

	largestSum := 0
	for i := len(nums) - k; i < len(nums); i++ {
		largestSum += nums[i]
	}

	diff := largestSum - smallestSum
	if diff < 0 {
		diff = -diff
	}

	return diff
}
```

### Go-Specific Notes

Go does not provide a built-in absolute value function for integers, so we manually convert negative results to positive values.

The array is sorted in place using `sort.Ints(nums)`. Since the constraints are small, there are no concerns about integer overflow.

## Edge Cases

### Single Element Array

When the array contains only one element, that element is simultaneously the smallest and largest value. Both sums are identical, so the answer must be zero. The implementation naturally handles this because both slices contain the same element.

### k Equals n

When `k` equals the array length, the `k` smallest elements and the `k` largest elements are both the entire array. Their sums are therefore equal, producing an answer of zero. No special handling is required.

### Duplicate Values

Arrays may contain repeated numbers. A common mistake is to treat the smallest or largest values as unique values rather than elements. Sorting preserves duplicates, and summing the first and last `k` positions correctly counts each occurrence separately.

### All Elements Equal

If every element has the same value, the smallest and largest groups have identical sums regardless of `k`. The algorithm correctly returns zero because both computed sums are equal after sorting.
