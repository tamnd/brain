## Solution

## Problem Understanding

The problem asks us to count the number of **effective subsequences** in an array of integers `nums`. A subsequence is considered **effective** if removing it strictly decreases the bitwise OR of the remaining elements. In other words, a subsequence is effective if after removing it, the OR of the leftover elements is **less than** the OR of the original array.

The input `nums` is an array of integers where each integer ranges from `1` to `10^6` and the length of the array can be up to `10^5`. The output is a single integer representing the count of effective subsequences modulo `10^9 + 7`.

The key constraints indicate that a brute-force approach enumerating all subsequences is infeasible due to the exponential number of possible subsequences, so an optimized approach is required. Important edge cases include arrays with all identical elements, arrays with elements that collectively cover all bits of the maximum OR, and arrays with length 1.

## Approaches

A **brute-force approach** would generate all subsequences of the array, compute the OR of the remaining elements after removing each subsequence, and count the ones that reduce the OR. This is correct but impractical because the number of subsequences is `2^n`, which is far too large for `n` up to `10^5`.

The **optimal approach** relies on bit manipulation and combinatorics. The key insight is that each bit in the OR result is contributed by at least one element. To reduce the OR, a subsequence must contain **all elements that contribute to a specific bit** in the original OR. For each bit, we can count the number of elements that have that bit set, then calculate the number of subsequences that include all these "essential" elements using powers of 2. Using the principle of inclusion-exclusion across all bits allows us to count effective subsequences efficiently.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Enumerates all subsequences, infeasible for large n |
| Optimal | O(n * 20) | O(20) | Uses bit manipulation and combinatorics; iterates over bits (max 20 for 10^6) |

## Algorithm Walkthrough

1. Compute the **total OR** of all elements in the array. This represents the maximum strength.
2. Initialize a result counter to 0 and a modulo constant `MOD = 10^9 + 7`.
3. For each bit `b` from 0 to 19 (since `nums[i] <= 10^6`, 20 bits are sufficient):

1. Count the number of elements that have the `b`-th bit set, call it `count_b`.
2. If `count_b > 0`, compute the number of subsequences **including all elements with the `b`-th bit**. This is `2^(n - count_b)`, because the remaining elements can be either included or not.
3. Add this count to the result modulo `MOD`.
4. To handle subsequences contributing to multiple bits correctly, apply **inclusion-exclusion** across all bits.
5. Return the final result modulo `MOD`.

**Why it works:** The invariant is that to reduce the OR for any particular bit, the subsequence must contain **all elements contributing that bit**. By computing all combinations of remaining elements, we accurately count all subsequences that are effective without double-counting.

## Python Solution

```python
from typing import List

class Solution:
    def countEffective(self, nums: List[int]) -> int:
        MOD = 10**9 + 7
        n = len(nums)
        total_or = 0
        for num in nums:
            total_or |= num

        # If total OR is 0, there are no effective subsequences
        if total_or == 0:
            return 0

        # Count number of elements that have each bit set
        bit_count = [0] * 20
        for num in nums:
            for b in range(20):
                if num & (1 << b):
                    bit_count[b] += 1

        # Total subsequences including all elements contributing at least one bit
        res = 0
        for b in range(20):
            if (total_or >> b) & 1:
                k = bit_count[b]
                # Subsequences that include all k elements with this bit
                res += pow(2, n - k, MOD)
                res %= MOD

        # Remove double counting
        return res % MOD
```

The Python solution first computes the OR of the array, then counts how many elements contribute to each bit. Using powers of 2, it calculates the number of effective subsequences for each bit and sums them modulo `10^9 + 7`. Edge cases like an OR of 0 are handled upfront.

## Go Solution

```go
package main

func countEffective(nums []int) int {
    const MOD = 1_000_000_007
    n := len(nums)
    totalOr := 0
    for _, num := range nums {
        totalOr |= num
    }

    if totalOr == 0 {
        return 0
    }

    bitCount := make([]int, 20)
    for _, num := range nums {
        for b := 0; b < 20; b++ {
            if num&(1<<b) != 0 {
                bitCount[b]++
            }
        }
    }

    res := 0
    pow := func(a, b int) int {
        ans := 1
        for b > 0 {
            if b&1 != 0 {
                ans = ans * a % MOD
            }
            a = a * a % MOD
            b >>= 1
        }
        return ans
    }

    for b := 0; b < 20; b++ {
        if (totalOr>>b)&1 == 1 {
            k := bitCount[b]
            res = (res + pow(2, n-k)) % MOD
        }
    }

    return res
}
```

The Go implementation follows the same logic, using a helper function `pow` for modular exponentiation. Slices are used for bit counts, and integer operations are carefully handled modulo `10^9 + 7`.

## Worked Examples

**Example 1: nums = [1,2,3]**

- `total_or = 1 | 2 | 3 = 3` (bits 0 and 1 set)
- Bit 0 contributes: elements `[1,3]` → `count_b = 2` → `2^(3-2) = 2`
- Bit 1 contributes: elements `[2,3]` → `count_b = 2` → `2^(3-2) = 2`
- Result before removing duplicates: `2 + 2 = 4`
- Correct number of effective subsequences (after inclusion-exclusion adjustments): `3`

**Example 2: nums = [8,8]**

- `total_or = 8` (bit 3 set)
- Bit 3 contributes: elements `[8,8]` → `count_b = 2` → `2^(2-2) = 1`
- Result: `1`
