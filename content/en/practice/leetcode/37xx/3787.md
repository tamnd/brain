**Difficulty:** 🟡 Medium  
**Topics:** Tree, Breadth-First Search, Graph Theory

## Solution

## Problem Understanding

This problem gives us an undirected tree with `n` nodes labeled from `0` to `n - 1`.

A tree is a connected graph with exactly `n - 1` edges and no cycles. That guarantee is important because it means there is exactly one simple path between any pair of nodes.

The goal is not to compute the diameter length itself. Instead, we must identify every node that can appear as an endpoint of at least one diameter path.

A diameter path is any longest simple path in the tree. Trees can have more than one diameter. Different longest paths may share some endpoints or may have different endpoints.

For every node:

- Output `'1'` if the node is an endpoint of at least one diameter.
- Output `'0'` otherwise.

The final answer is returned as a binary string of length `n`.

For example:

- In a straight chain `0 - 1 - 2`, only nodes `0` and `2` are endpoints of the unique diameter, so the answer is `"101"`.
- In a branching tree, multiple longest paths may exist, so several nodes may become valid diameter endpoints.

The constraints are large:

- `2 <= n <= 100000`
- Up to `100000` nodes

This immediately rules out algorithms that repeatedly traverse the tree from every node. We need something close to linear time.

Important edge cases include:

- Very small trees such as `n = 2`, where both nodes are automatically special.
- Star-shaped trees where many leaves participate in diameters.
- Trees with multiple centers and multiple diameter paths.
- Deep chain-like trees where recursion may overflow, so iterative traversal is safer.

## Approaches

### Brute Force

A direct approach is to compute distances from every node.

For each node:

1. Run BFS.
2. Compute distances to all other nodes.
3. Determine the maximum distance.
4. Record which pairs achieve the global diameter.
5. Mark all endpoints.

This works because BFS in a tree gives shortest path distances, and checking all node pairs eventually discovers every diameter.

The issue is performance.

Running BFS from every node costs `O(n)` each, resulting in `O(n²)` total time, which is impossible for `100000` nodes.

### Key Insight

A tree has a powerful property:

All diameter endpoints must be among the nodes that are farthest from one of the diameter centers.

Instead of examining every path, we can use tree DP.

For each node:

- `down[u]` = maximum distance from `u` downward into its subtree.
- `up[u]` = maximum distance from `u` upward through its parent.

Then:

```
eccentricity(u) = max(down[u], up[u])
```

The eccentricity of a node is its distance to the farthest node in the entire tree.

The diameter length is:

```
diameter = max(eccentricity(u))
```

A node is special if its eccentricity equals the diameter.

Why?

If the farthest node from `u` is at distance equal to the tree diameter, then `u` must be one endpoint of some diameter.

This transforms the problem into computing farthest distances for all nodes in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | BFS from every node |
| Optimal | O(n) | O(n) | Tree DP with rerooting |

## Algorithm Walkthrough

### Step 1: Build the adjacency list

Convert `edges` into an adjacency list.

Since the graph is undirected, insert both directions.

This allows traversing neighbors efficiently in `O(total edges)`.

### Step 2: Compute parent relationships and traversal order

Perform an iterative DFS from node `0`.

During traversal:

- Store `parent[u]`
- Record visitation order

This order later allows processing nodes bottom-up.

Recursive DFS is avoided because `n` can reach `100000`.

### Step 3: Compute downward distances

Process nodes in reverse traversal order.

For every node:

- Examine each child.
- Candidate distance:

```
1 + down[child]
```

Store the maximum.

After this pass:

```
down[u]
```

contains the longest path starting at `u` and moving into descendants.

### Step 4: Compute upward distances

Traverse nodes in original order.

For each node:

- Find the two largest child contributions.
- For each child:

- Either continue through the best sibling.
- Or move upward using `up[parent]`.

Formula:

```
up[child] =
1 + max(
    up[parent],
    best_other_child_distance
)
```

This gives each child access to paths outside its own subtree.

### Step 5: Compute eccentricity

For each node:

```
ecc[u] = max(down[u], up[u])
```

This is the farthest possible distance from that node.

### Step 6: Find diameter endpoints

Compute:

```
diameter = max(ecc)
```

Every node with:

```
ecc[u] == diameter
```

is special.

Build the answer string.

### Why it works

For every node, `down` captures longest paths staying inside its subtree and `up` captures longest paths leaving the subtree. Their maximum therefore equals the farthest distance from that node to any other node.

The diameter length is the largest eccentricity in the tree. A node whose eccentricity equals the diameter reaches some node through a longest possible path, making it an endpoint of a diameter.

## Python Solution

```python
from typing import List

class Solution:
    def findSpecialNodes(self, n: int, edges: List[List[int]]) -> str:
        graph = [[] for _ in range(n)]

        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)

        parent = [-1] * n
        order = []

        stack = [0]
        parent[0] = 0

        while stack:
            node = stack.pop()
            order.append(node)

            for nei in graph[node]:
                if parent[nei] == -1:
                    parent[nei] = node
                    stack.append(nei)

        down = [0] * n

        for node in reversed(order):
            best = 0

            for nei in graph[node]:
                if nei == parent[node]:
                    continue

                best = max(best, down[nei] + 1)

            down[node] = best

        up = [0] * n

        for node in order:
            first = -1
            second = -1

            for nei in graph[node]:
                if nei == parent[node]:
                    continue

                value = down[nei] + 1

                if value > first:
                    second = first
                    first = value
                elif value > second:
                    second = value

            for nei in graph[node]:
                if nei == parent[node]:
                    continue

                use = first

                if down[nei] + 1 == first:
                    use = second

                up[nei] = max(up[node] + 1, use + 1 if use != -1 else 1)

        eccentricity = [
            max(down[i], up[i])
            for i in range(n)
        ]

        diameter = max(eccentricity)

        result = []

        for value in eccentricity:
            result.append("1" if value == diameter else "0")

        return "".join(result)
```

The implementation begins by constructing the adjacency list.

An iterative DFS establishes parent relationships and creates an ordering of nodes. Processing this order in reverse computes `down`, which represents the deepest reachable descendant distance.

The second traversal performs rerooting DP. Each child inherits information either from the parent side (`up`) or from sibling subtrees.

Finally, eccentricity is computed for every node and compared with the diameter value to determine which nodes are special.

## Go Solution

```go
func findSpecialNodes(n int, edges [][]int) string {
	graph := make([][]int, n)

	for _, e := range edges {
		u, v := e[0], e[1]
		graph[u] = append(graph[u], v)
		graph[v] = append(graph[v], u)
	}

	parent := make([]int, n)

	for i := range parent {
		parent[i] = -1
	}

	order := []int{}
	stack := []int{0}
	parent[0] = 0

	for len(stack) > 0 {
		node := stack[len(stack)-1]
		stack = stack[:len(stack)-1]

		order = append(order, node)

		for _, nei := range graph[node] {
			if parent[nei] == -1 {
				parent[nei] = node
				stack = append(stack, nei)
			}
		}
	}

	down := make([]int, n)

	for i := len(order) - 1; i >= 0; i-- {
		node := order[i]

		for _, nei := range graph[node] {
			if nei == parent[node] {
				continue
			}

			if down[nei]+1 > down[node] {
				down[node] = down[nei] + 1
			}
		}
	}

	up := make([]int, n)

	for _, node := range order {
		first, second := -1, -1

		for _, nei := range graph[node] {
			if nei == parent[node] {
				continue
			}

			value := down[nei] + 1

			if value > first {
				second = first
				first = value
			} else if value > second {
				second = value
			}
		}

		for _, nei := range graph[node] {
			if nei == parent[node] {
				continue
			}

			use := first

			if down[nei]+1 == first {
				use = second
			}

			candidate := 1

			if use != -1 {
				candidate = use + 1
			}

			if up[node]+1 > candidate {
				up[nei] = up[node] + 1
			} else {
				up[nei] = candidate
			}
		}
	}

	ecc := make([]int, n)
	diameter := 0

	for i := 0; i < n; i++ {
		if down[i] > up[i] {
			ecc[i] = down[i]
		} else {
			ecc[i] = up[i]
		}

		if ecc[i] > diameter {
			diameter = ecc[i]
		}
	}

	ans := make([]byte, n)

	for i := 0; i < n; i++ {
		if ecc[i] == diameter {
			ans[i] = '1'
		} else {
			ans[i] = '0'
		}
	}

	return string(ans)
}
```

The Go implementation follows the same rerooting structure. Slices replace Python lists, and explicit maximum comparisons are used instead of helper functions. Iterative traversal avoids recursion depth limitations.

## Edge Cases

A tree with only two nodes is easy to mishandle because every node becomes an endpoint automatically. Since `down` and `up` both remain small, the implementation correctly assigns equal eccentricity to both nodes and returns `"11"`.

A star-shaped tree can expose incorrect sibling handling. Every leaf should become a diameter endpoint because any leaf-to-leaf path passes through the center and has maximum length. The rerooting step explicitly tracks the two largest child distances so each child can consider alternative branches.

A long chain of `100000` nodes can break recursive DFS due to stack overflow. The implementation avoids recursion entirely and uses iterative traversal with explicit stacks, ensuring it works safely within problem limits.
