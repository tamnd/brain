# LeetCode 3780 - Maximum Sum of Three Numbers Divisible by Three

## Problem Understanding

We are given an integer array `nums`, and we must select exactly three distinct elements from the array. Among all possible triplets, we are interested only in those whose sum is divisible by three.

A sum is divisible by three if its remainder when divided by three is zero. Therefore, the task is not simply to find the three largest numbers. Instead, we must find the triplet with the largest possible sum while also satisfying the divisibility condition.

If no valid triplet exists, we return `0`.

The constraints are important:

- `nums.length` can be as large as `100,000`.
- Each value can be as large as `100,000`.

Because of the large input size, any algorithm that examines every triplet explicitly is far too slow. We need something close to linear time.

An important observation is that divisibility by three depends only on each number's remainder modulo three. Every number belongs to one of three groups:

- Remainder `0`
- Remainder `1`
- Remainder `2`

Since we only choose exactly three numbers, the possible remainder combinations that produce a sum divisible by three are limited.

Potential edge cases include arrays where no valid triplet exists, arrays where all numbers belong to the same remainder class, and arrays where the optimal answer requires skipping some larger values in favor of a better remainder combination.

## Approaches

### Brute Force

The most direct solution is to examine every possible triplet.

For every combination of three indices `(i, j, k)`, we compute:

```
nums[i] + nums[j] + nums[k]
```

If the sum is divisible by three, we update the maximum answer.

This approach is guaranteed to find the correct answer because every possible triplet is checked. However, it requires evaluating all combinations of three elements.

For an array of size `n`, there are:

```
n choose 3
```

triplets, resulting in `O(n³)` time complexity, which is impossible for `n = 100,000`.

### Optimal Observation

Because only three numbers are selected, the only information that matters for divisibility is each number's remainder modulo three.

The valid remainder patterns are:

- `(0, 0, 0)`
- `(1, 1, 1)`
- `(2, 2, 2)`
- `(0, 1, 2)`

These are the only ways three remainders can add to a multiple of three.

To maximize the sum, we only need the largest numbers from each remainder class.

Specifically:

- Store the three largest numbers with remainder `0`.
- Store the three largest numbers with remainder `1`.
- Store the three largest numbers with remainder `2`.

Since only three elements can ever be chosen from a remainder class, keeping more than three values is unnecessary.

After processing the array, evaluate the four valid remainder patterns and return the largest achievable sum.

## Algorithm Walkthrough

1. Create three containers corresponding to remainder classes `0`, `1`, and `2`.
2. Iterate through the array once.
3. For each number:

- Compute `num % 3`.
- Insert the number into the corresponding remainder group.
- Keep only the three largest values in that group.
4. After processing all numbers, sort each remainder group in descending order.
5. Initialize the answer to `0`.
6. If the remainder-0 group contains at least three values, compute the sum of its three largest elements and update the answer.
7. If the remainder-1 group contains at least three values, compute the sum of its three largest elements and update the answer.
8. If the remainder-2 group contains at least three values, compute the sum of its three largest elements and update the answer.
9. If all three groups are non-empty, compute:

```
largest remainder-0 value
+ largest remainder-1 value
+ largest remainder-2 value
```

Update the answer if this sum is larger.
10. Return the final answer.

### Why it works

Any valid triplet must belong to one of the four remainder patterns:

```
(0,0,0)
(1,1,1)
(2,2,2)
(0,1,2)
```

No other combination can produce a sum divisible by three.

For each pattern, the maximum possible sum is obtained by taking the largest available numbers participating in that pattern. Since at most three numbers from any remainder class are ever needed, storing only the top three values from each class preserves all information necessary to construct the optimal answer.

Therefore, evaluating all four valid patterns guarantees that the maximum valid triplet sum is found.

## Python Solution

```python
from typing import List

class Solution:
    def maximumSum(self, nums: List[int]) -> int:
        groups = [[], [], []]

        for num in nums:
            r = num % 3
            groups[r].append(num)
            groups[r].sort(reverse=True)
            if len(groups[r]) > 3:
                groups[r].pop()

        ans = 0

        for r in range(3):
            if len(groups[r]) >= 3:
                ans = max(ans, sum(groups[r][:3]))

        if groups[0] and groups[1] and groups[2]:
            ans = max(ans, groups[0][0] + groups[1][0] + groups[2][0])

        return ans
```

### Implementation Explanation

The algorithm maintains three groups according to remainder modulo three.

As each number is processed, it is added to its corresponding group. Because only the three largest values in a group can ever participate in an optimal solution, each group is trimmed to size three.

After the scan is complete, the algorithm evaluates every valid remainder pattern.

The patterns `(0,0,0)`, `(1,1,1)`, and `(2,2,2)` require at least three elements in their respective groups. The pattern `(0,1,2)` requires at least one element from each group.

The maximum sum among these possibilities is returned.

## Go Solution

```go
func maximumSum(nums []int) int {
	groups := make([][]int, 3)

	for _, num := range nums {
		r := num % 3

		groups[r] = append(groups[r], num)

		for i := len(groups[r]) - 1; i > 0; i-- {
			if groups[r][i] > groups[r][i-1] {
				groups[r][i], groups[r][i-1] = groups[r][i-1], groups[r][i]
			} else {
				break
			}
		}

		if len(groups[r]) > 3 {
			groups[r] = groups[r][:3]
		}
	}

	ans := 0

	for r := 0; r < 3; r++ {
		if len(groups[r]) >= 3 {
			sum := groups[r][0] + groups[r][1] + groups[r][2]
			if sum > ans {
				ans = sum
			}
		}
	}

	if len(groups[0]) > 0 &&
		len(groups[1]) > 0 &&
		len(groups[2]) > 0 {

		sum := groups[0][0] + groups[1][0] + groups[2][0]
		if sum > ans {
			ans = sum
		}
	}

	return ans
}
```

### Go Specific Notes

The Go implementation follows the same logic as the Python version.

Instead of repeatedly sorting an entire group, it performs a small insertion step. Since each group is limited to at most three elements, maintaining sorted order costs constant time.

All arithmetic safely fits within Go's `int` type because the largest possible triplet sum is:

```
3 × 100000 = 300000
```

which is far below integer limits.

## Edge Cases

### No Valid Triplet Exists

An array such as `[2,1,5]` contains exactly three elements, but their sum is not divisible by three. Since there is only one possible triplet, the correct answer is `0`. The implementation naturally handles this because none of the valid remainder patterns can be formed.

### All Numbers Belong to the Same Remainder Class

Consider `[1,4,7]`. Every number has remainder `1`. A naive solution might incorrectly assume that a mixture of remainders is required. However, three numbers with remainder `1` produce a total remainder of `3`, which is divisible by three. The algorithm explicitly checks the `(1,1,1)` pattern.

### The Largest Numbers Do Not Form the Best Remainder Pattern

Suppose the largest three numbers do not yield a divisible sum. A greedy strategy that simply picks the three largest values would fail. The algorithm avoids this issue by organizing numbers according to remainder classes and evaluating every valid remainder pattern independently.

### Large Input Size

With up to `100,000` numbers, enumerating triplets is impossible. The implementation processes each value exactly once and stores at most three values per remainder class, ensuring that performance remains efficient even at maximum input size.

### Multiple Candidates Within the Same Remainder Class

A remainder class may contain thousands of numbers. Only the largest three can ever participate in an optimal triplet because a valid solution never needs more than three values from a single class. The implementation trims each group to size three, preserving all necessary information while keeping memory usage constant.
