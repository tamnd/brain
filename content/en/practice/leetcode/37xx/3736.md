## Solution

## Problem Understanding

The problem asks us to calculate the **maximum total calories burned** by jumping on a series of blocks with given heights, starting from the ground. Each jump between blocks burns calories equal to the square of the difference in height `(a - b)²`. The first jump is always from the ground (height 0) to some block. After the first jump, you must continue jumping from one block to another, visiting **each block exactly once**. The goal is to find the **optimal sequence** of jumps that maximizes the total calories burned.

The input `heights` is an array of positive integers representing the heights of the blocks. The output is a single integer representing the maximum total calories burned. The constraints indicate that the input size can be as large as `10^5` and heights can be up to `10^5`, which rules out any brute-force approach that tries all permutations of blocks, as that would require factorial time.

Important edge cases include: arrays of length 1 (only one jump), arrays where all heights are equal, and arrays with alternating high and low values.

## Approaches

The brute-force approach would consider **all permutations** of the blocks, calculate the calories burned for each sequence, and return the maximum. While this guarantees the correct answer, its complexity is O(n!), which is infeasible for n up to 100,000.

The key insight for an optimal solution is to **maximize the square of height differences at each step**. Since `(a - b)²` grows faster as the difference between `a` and `b` increases, the strategy is to **jump between the highest remaining block and the lowest remaining block alternately**, starting with the largest jump from the ground. Sorting the array allows us to easily pick these extremes. This greedy approach ensures that we always maximize `(a - b)²` at every step.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n!) | O(n) | Enumerate all permutations and compute total calories |
| Optimal (Greedy with Sorting) | O(n log n) | O(1) or O(n) | Sort the array and jump between largest and smallest remaining blocks |

## Algorithm Walkthrough

1. **Sort the array** `heights` in ascending order. This allows easy access to the smallest and largest remaining blocks.
2. **Initialize two pointers**, `left` at the start of the array and `right` at the end of the array. These pointers will track the smallest and largest remaining blocks.
3. **Initialize `calories` to 0** to accumulate the total calories burned.
4. **Choose the first jump**: jump from ground (0) to the block with the largest height, which is `heights[right]`. Add `(0 - heights[right])²` to `calories`. Decrement `right`.
5. **Iteratively jump**: at each step, choose the block that is farthest in height from the current block (either `heights[left]` or `heights[right]`). Add the square of the difference to `calories`, and move the corresponding pointer.
6. **Continue until all blocks are visited**. The sum accumulated in `calories` is the maximum possible.

**Why it works:** The invariant is that at each step we always select the block that maximizes the next jump's calorie burn. Since `(a - b)²` increases quadratically with the height difference, alternating between extremes ensures the sum of squares is maximized. Sorting ensures we can efficiently pick these extremes.

## Python Solution

```python
from typing import List

class Solution:
    def maxCaloriesBurnt(self, heights: List[int]) -> int:
        heights.sort()
        left, right = 0, len(heights) - 1
        calories = 0
        
        # First jump from ground to the tallest block
        current = heights[right]
        calories += (0 - current) ** 2
        right -= 1
        
        # Jump between extremes greedily
        while left <= right:
            # Choose the next block with maximum height difference
            if abs(current - heights[left]) > abs(current - heights[right]):
                calories += (current - heights[left]) ** 2
                current = heights[left]
                left += 1
            else:
                calories += (current - heights[right]) ** 2
                current = heights[right]
                right -= 1
        
        return calories
```

The code first sorts the heights to allow two-pointer access to the smallest and largest blocks. The first jump is always to the tallest block to maximize the initial calorie burn. Then, in a loop, we select the block with the largest height difference from the current position. We update the current block after each jump and adjust the pointers accordingly.

## Go Solution

```go
import "sort"

func maxCaloriesBurnt(heights []int) int64 {
    sort.Ints(heights)
    left, right := 0, len(heights)-1
    var calories int64
    
    // First jump from ground to tallest block
    current := heights[right]
    calories += int64(current * current)
    right--
    
    // Jump between extremes greedily
    for left <= right {
        if abs(current - heights[left]) > abs(current - heights[right]) {
            diff := current - heights[left]
            calories += int64(diff * diff)
            current = heights[left]
            left++
        } else {
            diff := current - heights[right]
            calories += int64(diff * diff)
            current = heights[right]
            right--
        }
    }
    
    return calories
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}
```

In Go, we sort the array and use two pointers, similar to the Python version. Since the result can be large, we use `int64` to store calories. The `abs` helper ensures we calculate absolute differences correctly.

## Worked Examples

**Example 1:** heights = [1,7,9]

1. Sort → [1,7,9], left=0, right=2
2. First jump: 0 → 9, calories=81, current=9, right=1
3. Next jump: max(|9-1|=8, |9-7|=2) → jump to 1, calories += 64 → total 145, current=1, left=1
4. Next jump: only 7 remains, calories += |1-7|² = 36 → total 181
5. Done

**Example 2:** heights = [5,2,4]

1. Sort → [2,4,5]
2. First jump: 0 → 5, calories=25
3. Next jump: max(|5-2|=3, |5-4|=1) → jump to 2, calories += 9 → total 34
4. Next jump: only 4 remains, calories += |2-4|² = 4 → total 38

## Edge Cases

1. **Single block:** When `heights` contains only one element, the function should calculate calories from the ground to that block. The algorithm handles this naturally because the initial jump calculation works for a single element.
2. **All blocks same height:** If all blocks are equal, every jump after the first will have `(a-b)² = 0`. The greedy approach will pick any sequence, and the sum of calories will correctly reflect only the first jump.
3. **Large height differences:** When the array contains extreme values like `[1, 100000]`, intermediate jumps are minimal, but the first jump dominates. Using `int64` in Go or Python integers ensures no overflow occurs. The algorithm correctly handles both small and large differences.
