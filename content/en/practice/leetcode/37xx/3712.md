# LeetCode 3712 - Sum of Elements With Frequency Divisible by K

## Problem Understanding

This problem gives us an integer array `nums` and an integer `k`. Our task is to find all values whose total frequency in the array is divisible by `k`, then compute the sum of all occurrences of those values.

A key detail is that we do not add each qualifying value only once. Instead, if a number qualifies, every occurrence of that number contributes to the final sum.

For example, if the number `3` appears four times and `4 % 2 == 0`, then all four occurrences are included in the sum. Their contribution is:

```
3 + 3 + 3 + 3 = 12
```

The input consists of:

- `nums`, an array of integers.
- `k`, a positive integer.

The output is a single integer representing the sum of all elements whose frequency is divisible by `k`.

The constraints are small:

- `1 <= nums.length <= 100`
- `1 <= nums[i] <= 100`
- `1 <= k <= 100`

Since the array length is at most 100, even less efficient approaches would work within the limits. However, we should still design a clean and efficient solution.

Some important edge cases include:

- No value has frequency divisible by `k`, the answer should be `0`.
- Every value has frequency divisible by `k`, the answer should be the sum of the entire array.
- `k = 1`, every frequency is divisible by 1, so every element contributes.
- Arrays containing only one distinct value.
- Arrays where frequencies are close to, but not exactly divisible by, `k`.

The problem guarantees valid inputs, so we never need to handle empty arrays or invalid values.

## Approaches

### Brute Force

A straightforward approach is to compute the frequency of every value by repeatedly scanning the array.

For each element in `nums`, we could count how many times it appears by traversing the entire array. If that frequency is divisible by `k`, we add the element to the answer.

This approach is correct because every element's frequency is determined exactly. However, counting frequencies separately for each element requires repeatedly scanning the array, resulting in quadratic time complexity.

Since there are up to `n` elements and each frequency computation may require another full traversal of `n` elements, the total cost becomes `O(n²)`.

### Optimal Approach

The key observation is that frequency information can be computed once and reused.

A hash map allows us to count the occurrences of every distinct value in a single pass. After obtaining all frequencies, we can iterate through the array again. For each element, we check whether its frequency is divisible by `k`.

If it is divisible, we add that occurrence to the answer. Otherwise, we ignore it.

This works because the frequency of each value is already known, making each lookup an `O(1)` operation on average.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Repeatedly counts frequencies by scanning the array |
| Optimal | O(n) | O(n) | Uses a hash map to store frequencies once |

## Algorithm Walkthrough

### Optimal Algorithm

1. Create a hash map called `frequency`.

This map will store how many times each distinct value appears in the array.
2. Traverse `nums` once and populate the frequency map.

For each number, increment its count in the map.
3. Initialize `answer = 0`.

This variable will store the final sum.
4. Traverse `nums` a second time.

For every element `num`, look up its frequency using the hash map.
5. Check whether `frequency[num] % k == 0`.

If the frequency is divisible by `k`, add `num` to `answer`.
6. Continue until all elements have been processed.
7. Return `answer`.

The hash map is the ideal data structure because it provides efficient frequency counting and constant-time lookups.

### Why it works

The frequency map correctly stores the total number of occurrences for every distinct value. During the second traversal, every occurrence of a value whose frequency is divisible by `k` is added to the answer. Values whose frequencies are not divisible by `k` contribute nothing.

Since every array element is examined exactly once during the summation phase and each inclusion decision is based on the correct total frequency of its value, the algorithm computes exactly the required sum.

## Python Solution

```python
from typing import List
from collections import Counter

class Solution:
    def sumDivisibleByK(self, nums: List[int], k: int) -> int:
        frequency = Counter(nums)

        answer = 0

        for num in nums:
            if frequency[num] % k == 0:
                answer += num

        return answer
```

The implementation begins by building a frequency table using `Counter`. This gives the total count of every distinct number in the array.

Next, we initialize `answer` to zero.

We then iterate through the original array rather than through the frequency map. This is important because the problem wants every qualifying occurrence included in the sum. For each element, we check whether its total frequency is divisible by `k`.

If the condition is satisfied, the current occurrence contributes its value to the running total.

Finally, after processing all elements, we return the accumulated sum.

## Go Solution

```go
func sumDivisibleByK(nums []int, k int) int {
	frequency := make(map[int]int)

	for _, num := range nums {
		frequency[num]++
	}

	answer := 0

	for _, num := range nums {
		if frequency[num]%k == 0 {
			answer += num
		}
	}

	return answer
}
```

The Go implementation follows exactly the same logic as the Python version.

A `map[int]int` is used to store frequencies. The first loop builds the frequency table, and the second loop computes the answer.

There are no special concerns regarding empty slices because the constraints guarantee at least one element. Integer overflow is also not an issue because the maximum possible sum is small. At worst, there are 100 elements with value 100, producing a sum of only 10,000.

## Edge Cases

### No Frequency Is Divisible by K

Consider:

```
nums = [1,2,3,4]
k = 2
```

Every value appears once, and `1 % 2 != 0`. A buggy implementation might accidentally include values or return an uninitialized result. Our solution correctly skips every element and returns `0`.

### K Equals 1

When `k = 1`, every positive integer frequency is divisible by 1.

```
nums = [1,2,3]
k = 1
```

All elements contribute to the answer, so the result is simply the sum of the entire array. The condition `frequency[num] % 1 == 0` naturally handles this case.

### Single Distinct Value

Consider:

```
nums = [4,4,4,4]
k = 2
```

The frequency of `4` is 4, which is divisible by 2. Every occurrence must be counted, producing:

```
4 + 4 + 4 + 4 = 16
```

Iterating over the original array rather than over distinct values ensures that each occurrence contributes exactly once.

### Frequency Close to Divisible but Not Divisible

Consider:

```
nums = [9,9,9,9]
k = 3
```

The frequency is 4, not 3 or 6. A common mistake is to check whether the frequency is at least `k` instead of divisible by `k`. Our implementation uses the exact condition:

```
frequency[num] % k == 0
```

so the value is correctly excluded.
