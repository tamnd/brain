# LeetCode 3784 - Minimum Deletion Cost to Make All Characters Equal

## Problem Understanding

We are given a string `s` and an array `cost` of the same length. The character `s[i]` can be deleted at a cost of `cost[i]`.

Our goal is to delete some characters so that the resulting string satisfies two conditions:

1. The resulting string is non-empty.
2. Every remaining character is identical.

The order of the remaining characters does not matter because we are only deleting characters, not rearranging them. Therefore, if we decide that the final string should contain only character `'a'`, then every character that is not `'a'` must be deleted, while every `'a'` can be kept.

The key observation is that the final string must consist entirely of one specific character. Since the string contains only lowercase English letters, there are at most 26 possible target characters to consider.

The input size can be as large as `10^5`, so any solution that repeatedly scans the entire string for every position or explores subsets of deletions would be too slow. We need a solution that runs in linear time.

Some important edge cases include:

- The string already consists of one repeated character, in which case the answer is `0`.
- The string contains only one character, in which case no deletion is needed.
- The cheapest solution may involve keeping multiple occurrences of the same character, not just one.
- Costs can be as large as `10^9`, so the total deletion cost may exceed 32-bit integer limits. A 64-bit integer type is required.

## Approaches

### Brute Force

A straightforward approach is to try every lowercase character as the final character of the string.

For a chosen character `c`, we compute the cost of deleting every character that is not equal to `c`. The minimum cost across all choices is the answer.

This already works because there are only 26 lowercase letters. However, if we think of the brute force approach more generally, we would repeatedly scan the entire string for every possible target character.

The method is correct because every valid final string must consist entirely of one character, so checking all possible target characters guarantees that we examine every possible final configuration.

The complexity is `O(26 * n)`, which is actually acceptable, but we can derive a cleaner and more direct linear solution.

### Key Insight

Instead of computing deletion costs separately for every possible target character, consider what happens if we keep all occurrences of one character.

Let:

- `totalCost` = sum of all deletion costs.
- `keep[c]` = total cost associated with positions containing character `c`.

If we decide that character `c` should remain, then:

- We keep all occurrences of `c`.
- We delete every other character.

The deletion cost becomes:

`totalCost - keep[c]`

Therefore, to minimize deletion cost, we want to maximize the total cost that remains undeleted.

In other words:

- Compute the total sum of all costs.
- Compute the accumulated cost for each character.
- Find the character with the largest accumulated cost.
- Delete everything else.

This transforms the problem into finding the maximum total cost contributed by any character.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(26n) | O(1) | Recompute deletion cost for each possible target character |
| Optimal | O(n) | O(1) | Track total cost per character and keep the most valuable character |

## Algorithm Walkthrough

1. Initialize a variable `total_cost` to store the sum of all deletion costs.
2. Create an array of size 26, where `char_cost[i]` stores the total cost contributed by the corresponding lowercase letter.
3. Traverse the string and cost array simultaneously.
4. For each position:

- Add `cost[i]` to `total_cost`.
- Determine the character index using `ord(s[i]) - ord('a')`.
- Add `cost[i]` to that character's accumulated total.
5. After processing the entire string, find the maximum value in `char_cost`.
6. This maximum value represents the total cost of all occurrences of the character we should keep.
7. Return `total_cost - max(char_cost)`.

### Why it works

Any valid final string must contain only one character type. If we choose character `c`, every occurrence of `c` can remain because keeping additional copies never increases deletion cost. Therefore, the deletion cost for choosing `c` is exactly the total cost of all other characters.

Since `total_cost` is fixed, minimizing

`total_cost - keep[c]`

is equivalent to maximizing `keep[c]`. The algorithm computes the total cost contributed by every character and keeps the character with the largest contribution, guaranteeing the minimum possible deletion cost.

## Python Solution

```python
from typing import List

class Solution:
    def minCost(self, s: str, cost: List[int]) -> int:
        total_cost = 0
        char_cost = [0] * 26

        for ch, c in zip(s, cost):
            total_cost += c
            char_cost[ord(ch) - ord('a')] += c

        return total_cost - max(char_cost)
```

The implementation follows the algorithm directly.

The variable `total_cost` stores the sum of all deletion costs. At the same time, `char_cost` accumulates the total cost associated with each letter.

After a single pass through the input, `char_cost[i]` tells us how much cost would remain undeleted if we kept every occurrence of that character. The best choice is the character with the largest accumulated value.

The final answer is the total cost minus the largest character contribution.

## Go Solution

```go
func minCost(s string, cost []int) int64 {
	var totalCost int64
	charCost := make([]int64, 26)

	for i := 0; i < len(s); i++ {
		c := int64(cost[i])

		totalCost += c
		charCost[s[i]-'a'] += c
	}

	var maxKeep int64
	for i := 0; i < 26; i++ {
		if charCost[i] > maxKeep {
			maxKeep = charCost[i]
		}
	}

	return totalCost - maxKeep
}
```

### Go-Specific Notes

The constraints allow individual costs up to `10^9` and the length up to `10^5`. The total cost can therefore reach `10^14`, which exceeds the range of a 32-bit integer.

For this reason, the Go solution uses `int64` for all accumulated sums and returns `int64`, matching the provided function signature.

The indexing operation `s[i] - 'a'` directly maps lowercase letters to values in the range `[0, 25]`.

## Edge Cases

### Single Character String

When `n = 1`, the string is already non-empty and consists entirely of one character. A common mistake is to assume at least one deletion is required. The algorithm correctly returns zero because the total cost and the maximum keepable cost are identical.

### All Characters Already Equal

For inputs such as `"zzzzz"`, every character belongs to the same group. The accumulated cost for that character equals the total cost, producing:

```
answer = total_cost - total_cost = 0
```

No special handling is required.

### Expensive Character Appears Only Once

Consider:

```
s = "aaaaab"
cost = [1,1,1,1,1,100]
```

A naive intuition might be to keep the character that appears most frequently. That would be incorrect. The objective depends on deletion cost, not frequency. The algorithm correctly keeps `'b'` because its total retained cost is larger than the combined cost of all `'a'` characters.

### Very Large Costs

The maximum possible total cost is:

```
10^5 × 10^9 = 10^14
```

This does not fit in a 32-bit integer. The Go solution uses `int64`, and Python integers automatically support arbitrary precision, ensuring correct results for the largest test cases.
