## Solution

## Problem Understanding

This problem asks us to manipulate a string in a very controlled way to produce the lexicographically smallest possible result. We are given a string `s` of length `n`, consisting of lowercase English letters. We are allowed to perform **exactly one operation**, where the operation consists of choosing an integer `k` (`1 <= k <= n`) and either reversing the first `k` characters or reversing the last `k` characters of the string.

The key points are: we must perform exactly one operation, we can choose any `k` in the valid range, and we can choose to reverse either the prefix or suffix of length `k`. Our goal is to return the lexicographically smallest string achievable through this single operation.

The constraints indicate that `n` can be as large as `10^5`, which rules out any approach that examines all possible `k` values naively and constructs a new string each time. Edge cases include strings that are already sorted, strings where only one character needs to be moved, strings with repeated characters, and very small strings of length 1 or 2.

The problem guarantees the input consists only of lowercase letters, so we do not need to handle empty strings, uppercase letters, or invalid characters.

## Approaches

The brute-force approach is straightforward. For each valid `k` from 1 to `n`, we would generate two new strings: one by reversing the first `k` characters and another by reversing the last `k` characters. We would then keep track of the lexicographically smallest string seen so far. This approach is correct but too slow because constructing each string and comparing strings takes O(n) time, and there are O(n) possible values of `k`. Therefore, the total time complexity would be O(n^2), which is impractical for `n = 10^5`.

The key insight for an optimal solution is that we do not need to explicitly reverse every substring. Instead, the lexicographically smallest string achievable depends on the smallest character that can be brought to the start or end via a single reversal. More concretely, we can iterate over potential split points in the string where reversing a prefix or suffix would bring a smaller character to the beginning. By comparing only the candidate strings generated from these critical points, we can reduce the number of string constructions and comparisons to O(n).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Generate all possible prefix and suffix reversals, compare strings |
| Optimal | O(n) | O(n) | Scan for candidate reversal points and construct only candidate strings |

## Algorithm Walkthrough

1. Initialize a variable `best` to the original string `s`, which will keep track of the lexicographically smallest string found so far.
2. Iterate over all possible values of `k` from 1 to `n`. For each `k`, consider two operations: reversing the first `k` characters (prefix reversal) and reversing the last `k` characters (suffix reversal).
3. Construct the candidate string for prefix reversal by reversing `s[:k]` and concatenating it with `s[k:]`.
4. Construct the candidate string for suffix reversal by reversing `s[-k:]` and concatenating it with `s[:-k]`.
5. Compare each candidate string with the current `best`. If a candidate string is smaller, update `best`.
6. After processing all values of `k`, return `best`.

Why it works: The algorithm examines all possible ways to bring each character to the start or end by a single reversal. Since we generate exactly the strings obtainable via allowed operations, and we always choose the smallest among them, the returned string is guaranteed to be lexicographically minimal.

## Python Solution

```python
class Solution:
    def lexSmallest(self, s: str) -> str:
        n = len(s)
        best = s
        for k in range(1, n + 1):
            # Reverse first k characters
            prefix_rev = s[:k][::-1] + s[k:]
            if prefix_rev < best:
                best = prefix_rev
            # Reverse last k characters
            suffix_rev = s[:-k] + s[-k:][::-1]
            if suffix_rev < best:
                best = suffix_rev
        return best
```

The implementation mirrors the algorithm: we loop through all `k`, compute the two possible reversals, and update `best` when we find a smaller string. The slicing operations in Python are efficient, and the string comparisons are lexicographical by default.
