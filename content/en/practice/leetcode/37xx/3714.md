# LeetCode 3714 - Longest Balanced Substring II

## Problem Understanding

We are given a string `s` consisting only of the characters `'a'`, `'b'`, and `'c'`.

A substring is considered **balanced** when every distinct character that appears inside that substring appears the same number of times.

The key phrase is **distinct characters**. Characters that do not appear in the substring are completely ignored.

For example:

- `"aaaa"` is balanced because the only distinct character is `'a'`, which appears 4 times.
- `"abba"` is balanced because `'a'` appears 2 times and `'b'` appears 2 times.
- `"abcabc"` is balanced because `'a'`, `'b'`, and `'c'` each appear 2 times.
- `"aab"` is not balanced because `'a'` appears 2 times while `'b'` appears 1 time.

The goal is to find the **maximum length** among all balanced substrings.

The constraint `s.length <= 100000` immediately rules out expensive quadratic solutions. We need an algorithm that is close to linear time.

An important observation is that the alphabet contains only **three possible characters**. This small alphabet size is the key that allows an efficient solution.

### Important Edge Cases

A few cases deserve special attention:

- A substring containing only one distinct character is always balanced.
- A balanced substring may contain exactly two distinct characters.
- A balanced substring may contain all three characters.
- The longest answer might be the entire string.
- The string may consist of a single character.
- Long runs such as `"aaaaaa"` must be handled efficiently.

## Approaches

### Brute Force

The most direct approach is to examine every possible substring.

For each starting position, we extend the ending position and maintain counts of `'a'`, `'b'`, and `'c'`. After each extension, we check whether all non-zero counts are equal.

Since there are `O(n²)` substrings, and each substring requires checking the counts, the overall complexity remains `O(n²)`.

This approach is correct because it explicitly evaluates every possible substring, but it is far too slow for `n = 100000`.

### Key Observation

Since only three characters exist, every balanced substring must belong to one of three categories:

1. Only one distinct character appears.
2. Exactly two distinct characters appear, and their counts are equal.
3. All three characters appear, and their counts are equal.

Each category can be expressed using prefix-count relationships.

Let:

- `A[i]` = number of `'a'` characters in the prefix
- `B[i]` = number of `'b'` characters in the prefix
- `C[i]` = number of `'c'` characters in the prefix

For a substring, counts are obtained by subtracting two prefix states.

For example, if a substring contains only `'a'` and `'b'` and their counts are equal:

- `count(a) = count(b)`
- `count(c) = 0`

Using prefix sums:

- `(A - B)` must remain unchanged
- `C` must remain unchanged

Therefore we only need to find the longest distance between equal states:

`(A - B, C)`

The same idea works for the other character pairs and for the three-character case.

Because hash maps can store the earliest occurrence of each state, we can find the longest valid substring in linear time.

## Algorithm Walkthrough

### Step 1

Compute running prefix counts for `'a'`, `'b'`, and `'c'`.

At every position we know how many times each character has appeared so far.

### Step 2

Handle the one-character case.

A substring containing only one distinct character is simply a consecutive run of identical characters.

Scan the string once and record the longest run length.

### Step 3

Create hash maps for the three possible two-character combinations.

For the pair `(a, b)` we store states:

`(A - B, C)`

If the same state appears at two prefix positions, then the substring between them satisfies:

- equal numbers of `'a'` and `'b'`
- zero occurrences of `'c'`

Similarly:

- `(A - C, B)` handles `(a, c)`
- `(B - C, A)` handles `(b, c)`

Each map stores the earliest index where a state first appeared.

### Step 4

Create another hash map for the three-character case.

Store states:

`(A - B, A - C)`

If two prefix positions have the same state, then:

- count(a) = count(b)
- count(a) = count(c)

Therefore all three counts are equal.

### Step 5

Initialize all maps with the empty prefix at index `-1`.

This allows substrings starting at position `0` to be considered naturally.

### Step 6

Process the string from left to right.

After updating prefix counts:

- compute the four state representations
- if a state has appeared before, update the answer using the distance to its earliest occurrence
- otherwise store the current index as the earliest occurrence

### Step 7

Return the maximum length found across all categories.

### Why it works

Every balanced substring must contain either one, two, or three distinct characters.

The longest one-character balanced substring is simply the longest run of identical characters.

For two-character balanced substrings, equality of counts and absence of the third character can be represented by a prefix state that remains unchanged. Equal prefix states therefore identify exactly the valid substrings.

For three-character balanced substrings, equal counts of all characters correspond exactly to equal values of `(A - B, A - C)`.

Since every possible balanced substring falls into one of these categories, and the algorithm finds the longest substring in each category, the final answer is correct.

## Python Solution

```python
class Solution:
    def longestBalanced(self, s: str) -> int:
        n = len(s)

        # Case 1: longest substring containing only one distinct character
        longest_run = 1
        current_run = 1

        for i in range(1, n):
            if s[i] == s[i - 1]:
                current_run += 1
            else:
                current_run = 1
            longest_run = max(longest_run, current_run)

        ans = longest_run

        a = b = c = 0

        ab_map = {(0, 0): -1}  # (A-B, C)
        ac_map = {(0, 0): -1}  # (A-C, B)
        bc_map = {(0, 0): -1}  # (B-C, A)
        abc_map = {(0, 0): -1}  # (A-B, A-C)

        for i, ch in enumerate(s):
            if ch == 'a':
                a += 1
            elif ch == 'b':
                b += 1
            else:
                c += 1

            state_ab = (a - b, c)
            if state_ab in ab_map:
                ans = max(ans, i - ab_map[state_ab])
            else:
                ab_map[state_ab] = i

            state_ac = (a - c, b)
            if state_ac in ac_map:
                ans = max(ans, i - ac_map[state_ac])
            else:
                ac_map[state_ac] = i

            state_bc = (b - c, a)
            if state_bc in bc_map:
                ans = max(ans, i - bc_map[state_bc])
            else:
                bc_map[state_bc] = i

            state_abc = (a - b, a - c)
            if state_abc in abc_map:
                ans = max(ans, i - abc_map[state_abc])
            else:
                abc_map[state_abc] = i

        return ans
```

### Implementation Explanation

The first loop computes the longest contiguous run of identical characters. This covers all balanced substrings that contain exactly one distinct character.

The variables `a`, `b`, and `c` maintain prefix counts while scanning the string.

Four hash maps are used:

- `ab_map` tracks states `(A-B, C)`
- `ac_map` tracks states `(A-C, B)`
- `bc_map` tracks states `(B-C, A)`
- `abc_map` tracks states `(A-B, A-C)`

Each map stores the earliest index where a state was seen.

Whenever the same state reappears, the substring between the two occurrences satisfies the required balance condition. The distance between the current index and the earliest occurrence gives a candidate answer.

The maximum length among all candidates is returned.

## Go Solution

```go
func longestBalanced(s string) int {
	n := len(s)

	longestRun := 1
	currentRun := 1

	for i := 1; i < n; i++ {
		if s[i] == s[i-1] {
			currentRun++
		} else {
			currentRun = 1
		}
		if currentRun > longestRun {
			longestRun = currentRun
		}
	}

	ans := longestRun

	a, b, c := 0, 0, 0

	abMap := map[[2]int]int{{0, 0}: -1}
	acMap := map[[2]int]int{{0, 0}: -1}
	bcMap := map[[2]int]int{{0, 0}: -1}
	abcMap := map[[2]int]int{{0, 0}: -1}

	for i := 0; i < n; i++ {
		switch s[i] {
		case 'a':
			a++
		case 'b':
			b++
		case 'c':
			c++
		}

		stateAB := [2]int{a - b, c}
		if pos, ok := abMap[stateAB]; ok {
			if i-pos > ans {
				ans = i - pos
			}
		} else {
			abMap[stateAB] = i
		}

		stateAC := [2]int{a - c, b}
		if pos, ok := acMap[stateAC]; ok {
			if i-pos > ans {
				ans = i - pos
			}
		} else {
			acMap[stateAC] = i
		}

		stateBC := [2]int{b - c, a}
		if pos, ok := bcMap[stateBC]; ok {
			if i-pos > ans {
				ans = i - pos
			}
		} else {
			bcMap[stateBC] = i
		}

		stateABC := [2]int{a - b, a - c}
		if pos, ok := abcMap[stateABC]; ok {
			if i-pos > ans {
				ans = i - pos
			}
		} else {
			abcMap[stateABC] = i
		}
	}

	return ans
}
```

### Go-Specific Notes

The Go solution uses fixed-size arrays of length two, `[2]int`, as hash map keys. Arrays are comparable in Go and therefore can be used directly as map keys.

The logic is otherwise identical to the Python implementation. No special handling for integer overflow is necessary because the maximum count is at most `100000`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Single pass through the string |
| Space | O(n) | Hash maps store prefix states |

The algorithm processes each character exactly once. Every hash map operation is expected `O(1)`, so the total running time is linear. The number of distinct prefix states can grow linearly with the input length, leading to `O(n)` auxiliary space.

## Edge Cases

### Entire String Contains One Character

Consider `"aaaaaa"`.

A common mistake is to focus only on equal-count relationships between different characters and forget that a substring with one distinct character is also balanced. The implementation explicitly computes the longest run of identical characters before processing the prefix-state maps, ensuring this case is handled correctly.

### Balanced Substring Starts at Index Zero

Consider `"abcabc"`.

Many prefix-sum solutions miss substrings beginning at position `0`. Initializing each hash map with the empty-prefix state at index `-1` guarantees that such substrings are detected naturally.

### Long Balanced Substring With Only Two Characters

Consider `"aaabbb"`.

The three-character equality condition does not apply because `'c'` never appears. The dedicated pair states `(A-B, C)`, `(A-C, B)`, and `(B-C, A)` ensure that balanced substrings involving exactly two characters are still found correctly.

### Multiple Balanced Categories Exist

Consider `"aaaabbbbcccc"`.

The string contains balanced substrings with one, two, and three distinct characters. The algorithm evaluates all categories independently and returns the maximum length among them, guaranteeing that no valid candidate is overlooked.
