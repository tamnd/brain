## Solution

## Problem Understanding

The problem gives us a string `s` and a dictionary of valid words called `wordDict`. We need to determine whether the entire string can be split into one or more dictionary words without leaving any unused characters.

The segmentation must cover the whole string. Partial matches are not enough. Each chosen substring must exist in the dictionary, and dictionary words may be reused any number of times.

For example, if `s = "applepenapple"` and the dictionary contains `"apple"` and `"pen"`, we can split the string into:

```
"apple" + "pen" + "apple"
```

Since every piece exists in the dictionary, the answer is `true`.

The important detail is that we are not asked to return the actual segmentation. We only need to determine whether at least one valid segmentation exists.

The constraints provide useful insight into the expected solution:

- The string length is at most 300
- The dictionary can contain up to 1000 words
- Each word length is at most 20

A brute force recursive search would generate many repeated subproblems and become extremely slow for longer strings. Since the maximum string length is only 300, dynamic programming becomes a natural fit because we can efficiently reuse results for overlapping subproblems.

Several edge cases are important:

- A string may have many possible segmentations
- A string may almost match but fail near the end
- Words can be reused multiple times
- Some dictionary words may overlap, such as `"cat"` and `"cats"`
- The entire string itself may already be a dictionary word

A correct solution must carefully handle all these situations.

## Approaches

### Brute Force Recursive Search

The most straightforward solution is to try every possible prefix of the string.

At each position, we check whether a substring starting at the current index exists in the dictionary. If it does, we recursively attempt to segment the remaining suffix.

For example:

```
leetcode
```

We first try:

```
"l"
"le"
"lee"
"leet"
```

When we find `"leet"` in the dictionary, we recursively process `"code"`.

This approach is correct because it explores every possible valid segmentation path. If any path successfully reaches the end of the string, the answer is `true`.

However, the problem is that the same suffix gets recomputed repeatedly.

For example, if multiple segmentations lead to the same remaining substring, the recursive solution will solve that same subproblem again and again. This creates exponential time complexity.

For larger inputs, especially strings with many overlapping prefixes, brute force becomes too slow.

### Dynamic Programming

The key observation is that this problem has overlapping subproblems.

Suppose we already know whether:

```
s[i:]
```

can be segmented successfully. Then whenever another path reaches index `i`, we can immediately reuse that result instead of recomputing it.

This naturally leads to dynamic programming.

We define:

```
dp[i] = whether s[0:i] can be segmented
```

If there exists some earlier position `j` such that:

- `dp[j]` is true
- `s[j:i]` exists in the dictionary

then `dp[i]` must also be true.

This transforms the exponential recursive search into a polynomial-time solution.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Tries every possible segmentation recursively |
| Optimal Dynamic Programming | O(n²) | O(n) | Reuses solutions to overlapping subproblems |

## Algorithm Walkthrough

### Dynamic Programming Strategy

1. Convert `wordDict` into a hash set.

A hash set allows constant-time lookup for dictionary words. Since we repeatedly check whether substrings exist in the dictionary, fast lookup is essential.
2. Create a DP array.

We define:

```
dp[i]
```

as whether the substring:

```
s[0:i]
```

can be segmented successfully.

The array size is `len(s) + 1`.
3. Initialize the base case.

Set:

```
dp[0] = true
```

This represents the empty string. An empty string is considered successfully segmented because it requires no words.
4. Iterate through all ending positions.

For every position `i` from `1` to `len(s)`:

- Try every possible split point `j`
- Check whether:

- `dp[j]` is true
- `s[j:i]` exists in the dictionary
5. Update the DP state.

If both conditions are true, then:

```
dp[i] = true
```

because the prefix up to `j` is valid, and the substring from `j` to `i` is also a valid dictionary word.
6. Continue until the end.

The final answer is:

```
dp[len(s)]
```

which represents whether the entire string can be segmented.

## Python Solution

```
from typing import List

class Solution:
    def wordBreak(self, s: str, wordDict: List[str]) -> bool:
        word_set = set(wordDict)
        n = len(s)

        dp = [False] * (n + 1)
        dp[0] = True

        for end in range(1, n + 1):
            for start in range(end):
                if dp[start] and s[start:end] in word_set:
                    dp[end] = True
                    break

        return dp[n]
```

The implementation begins by converting `wordDict` into a set. This is important because set membership checks are much faster than list lookups.

The variable `n` stores the length of the string for convenience.

The `dp` array contains boolean values. Each index represents whether the substring ending at that position can be segmented successfully.

The base case is:

```
dp[0] = True
```

This means the empty prefix is valid.

The outer loop iterates through all possible ending positions. For each ending position, the inner loop tries every possible starting position.

The condition:

```
dp[start] and s[start:end] in word_set
```

checks two things:

- The prefix before `start` is segmentable
- The current substring is a valid dictionary word

If both conditions hold, we mark `dp[end]` as `True`.

The `break` statement is an optimization. Once we find one valid segmentation, there is no need to continue checking other split points.

Finally, we return `dp[n]`, which tells us whether the entire string can be segmented.

## Go Solution

```
func wordBreak(s string, wordDict []string) bool {
	wordSet := make(map[string]bool)

	for _, word := range wordDict {
		wordSet[word] = true
	}

	n := len(s)

	dp := make([]bool, n+1)
	dp[0] = true

	for end := 1; end <= n; end++ {
		for start := 0; start < end; start++ {
			if dp[start] && wordSet[s[start:end]] {
				dp[end] = true
				break
			}
		}
	}

	return dp[n]
}
```

The Go solution follows the same algorithmic structure as the Python version.

Since Go does not have a built-in set type, we simulate one using:

```
map[string]bool
```

Substring slicing in Go uses:

```
s[start:end]
```

which behaves similarly to Python slicing.

The DP array is created using:

```
make([]bool, n+1)
```

All values default to `false`, so we only need to explicitly set:

```
dp[0] = true
```

No integer overflow concerns exist here because the problem constraints are small.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | Two nested loops iterate over substring boundaries |
| Space | O(n) | DP array stores one boolean per position |

The outer loop iterates through all ending positions, and the inner loop checks every possible split point before that position.

Although substring creation has some implementation-dependent cost, the constrained input size keeps the solution efficient.

The space usage comes primarily from the DP array, which stores one boolean value for each index in the string.

## Edge Cases

### Single Character String

Consider:

```
s = "a"
wordDict = ["a"]
```

A naive implementation might mishandle very small strings or incorrectly initialize the DP array.

This solution works correctly because `dp[0] = True`, and the substring `"a"` is checked normally during iteration.

### Repeated Word Usage

Consider:

```
s = "aaaaaaa"
wordDict = ["aaa", "aaaa"]
```

The problem explicitly allows reuse of dictionary words.

The DP solution naturally supports this because once a position becomes reachable, later positions can reuse the same dictionary entries without restriction.

### Overlapping Prefixes

Consider:

```
s = "catsanddog"
wordDict = ["cat", "cats", "and", "sand", "dog"]
```

Both `"cat"` and `"cats"` are valid prefixes.

A greedy solution that always chooses the longest or shortest prefix could fail. The DP approach avoids this issue because it explores all possible split points systematically.

### Impossible Final Segment

Consider:

```
s = "applepenappl"
wordDict = ["apple", "pen"]
```

Most of the string matches successfully, but the final portion cannot be segmented completely.

The DP solution correctly returns `False` because the final index never becomes reachable.

### Entire String Is One Word

Consider:

```
s = "leetcode"
wordDict = ["leetcode"]
```

Some implementations mistakenly require multiple segments.

This solution handles the case correctly because the substring from index `0` to `n` is checked directly against the dictionary.
