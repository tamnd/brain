# LeetCode 131, Palindrome Partitioning

## Problem Understanding

This problem asks us to split a string into multiple substrings such that every substring in the partition is a palindrome. A palindrome is a string that reads the same forward and backward.

The input is a single string `s`, containing only lowercase English letters. We must return all possible ways to partition the string so that every piece satisfies the palindrome condition.

For example, if the input is `"aab"`, there are two valid partitions:

- `["a", "a", "b"]`
- `["aa", "b"]`

Both are valid because every substring inside each partition is a palindrome.

The important detail is that we are not searching for the minimum number of partitions or a single valid answer. We must generate every possible valid partitioning.

The constraints are relatively small:

- `1 <= s.length <= 16`

This small maximum length strongly suggests that exponential algorithms are acceptable. Since every character position can potentially become a partition boundary, the total number of possible partitions grows exponentially. Because the input size is capped at 16, backtracking becomes a practical solution.

Several edge cases are important:

- A single character string such as `"a"` is always a palindrome, so the answer is simply `[["a"]]`.
- A string where all characters are the same, such as `"aaaa"`, produces many valid partitions because almost every substring is a palindrome.
- A string with no multi-character palindromes, such as `"abc"`, only allows partitions into individual characters.
- Empty substrings are never considered because partitions must consist of non-empty substrings.

## Approaches

### Brute Force Approach

The brute force solution generates every possible way to partition the string, then checks whether all substrings in each partition are palindromes.

For a string of length `n`, there are `n - 1` possible cut positions. Each cut position can either contain a partition boundary or not, producing `2^(n-1)` total partition combinations.

For each generated partitioning, we must check every substring to determine whether it is a palindrome. A palindrome check itself costs `O(k)` for a substring of length `k`.

This approach is correct because it exhaustively examines every possible partition configuration. However, it performs many repeated palindrome checks on the same substrings, making it inefficient.

### Optimal Approach

The better solution combines backtracking with dynamic programming style palindrome preprocessing.

The key insight is that during backtracking, we repeatedly ask the same question:

"Is substring `s[i:j]` a palindrome?"

Instead of recomputing this repeatedly, we precompute all palindrome information in advance using a 2D table.

We create a boolean table:

```
is_palindrome[start][end]
```

This table tells us whether the substring from `start` to `end` is a palindrome.

Once this table is built, each palindrome lookup becomes `O(1)`.

Then we use backtracking to explore all valid partitions:

- Start at index `0`
- Try every possible ending position
- If the substring is a palindrome, add it to the current partition
- Recursively continue from the next index
- Backtrack afterward to explore other possibilities

This avoids repeated palindrome computations and makes the solution significantly faster.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × 2^n × n) | O(n × 2^n) | Generates all partitions and repeatedly checks palindromes |
| Optimal | O(n^2 + n × 2^n) | O(n^2 + n × 2^n) | Uses precomputed palindrome table with backtracking |

## Algorithm Walkthrough

1. Create a 2D boolean table called `is_palindrome`.

The table stores whether every substring `s[i:j]` is a palindrome. This prevents repeated palindrome calculations during recursion.
2. Fill the palindrome table.

A substring is a palindrome if:

- The characters at both ends match
- The inner substring is also a palindrome

The recurrence relation is:

```
s[i] == s[j] AND (j - i <= 2 OR is_palindrome[i+1][j-1])
```

The condition `j - i <= 2` handles:

- Single characters
- Two-character substrings
- Three-character substrings
3. Initialize the result list.

This will store all valid palindrome partitions.
4. Start backtracking from index `0`.

At each position, try every possible substring starting from the current index.
5. Check whether the current substring is a palindrome.

Because of preprocessing, this lookup is `O(1)`.
6. If the substring is valid:

- Add it to the current partition
- Recursively continue from the next position
7. When the recursion reaches the end of the string:

- A complete valid partition has been formed
- Add a copy of the current partition to the result list
8. Backtrack by removing the last substring.

This restores the previous state so other partition possibilities can be explored.

## Python Solution

```
from typing import List

class Solution:
    def partition(self, s: str) -> List[List[str]]:
        n = len(s)

        is_palindrome = [[False] * n for _ in range(n)]

        for end in range(n):
            for start in range(end + 1):
                if s[start] == s[end]:
                    if end - start <= 2 or is_palindrome[start + 1][end - 1]:
                        is_palindrome[start][end] = True

        result = []
        current_partition = []

        def backtrack(start_index: int) -> None:
            if start_index == n:
                result.append(current_partition[:])
                return

            for end_index in range(start_index, n):
                if is_palindrome[start_index][end_index]:
                    current_partition.append(
                        s[start_index:end_index + 1]
                    )

                    backtrack(end_index + 1)

                    current_partition.pop()

        backtrack(0)

        return result
```

The implementation begins by constructing the palindrome lookup table. The nested loops iterate through every possible substring boundary.

The condition:

```
s[start] == s[end]
```

ensures the outer characters match.

The second condition:

```
end - start <= 2 or is_palindrome[start + 1][end - 1]
```

ensures the inner portion is also a palindrome.

After preprocessing, the algorithm moves into the recursive backtracking phase.

The `backtrack` function explores all valid partitions beginning at `start_index`.

Whenever a palindrome substring is found, it is temporarily added to `current_partition`, and recursion continues from the next position.

When the recursion reaches the end of the string, a complete valid partition has been constructed, so a copy is added to `result`.

The final `pop()` operation performs the backtracking step, restoring the previous state before trying another substring choice.

## Go Solution

```
func partition(s string) [][]string {
	n := len(s)

	isPalindrome := make([][]bool, n)
	for i := range isPalindrome {
		isPalindrome[i] = make([]bool, n)
	}

	for end := 0; end < n; end++ {
		for start := 0; start <= end; start++ {
			if s[start] == s[end] {
				if end-start <= 2 || isPalindrome[start+1][end-1] {
					isPalindrome[start][end] = true
				}
			}
		}
	}

	var result [][]string
	var currentPartition []string

	var backtrack func(int)

	backtrack = func(startIndex int) {
		if startIndex == n {
			partitionCopy := make([]string, len(currentPartition))
			copy(partitionCopy, currentPartition)

			result = append(result, partitionCopy)
			return
		}

		for endIndex := startIndex; endIndex < n; endIndex++ {
			if isPalindrome[startIndex][endIndex] {
				currentPartition = append(
					currentPartition,
					s[startIndex:endIndex+1],
				)

				backtrack(endIndex + 1)

				currentPartition = currentPartition[:len(currentPartition)-1]
			}
		}
	}

	backtrack(0)

	return result
}
```

The Go solution follows the same overall algorithm as the Python version.

One important Go-specific detail is slice copying. Since slices are references to underlying arrays, we must explicitly create a copy before appending the current partition to the result:

```
partitionCopy := make([]string, len(currentPartition))
copy(partitionCopy, currentPartition)
```

Without this step, future modifications during backtracking would alter previously stored results.

Go also uses explicit slice truncation for backtracking:

```
currentPartition = currentPartition[:len(currentPartition)-1]
```

Unlike Python, Go does not have built-in list popping.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n^2 + n × 2^n) | `O(n^2)` preprocessing plus exponential partition generation |
| Space | O(n^2 + n × 2^n) | Palindrome table plus recursion and stored results |

The `O(n^2)` preprocessing comes from filling the palindrome lookup table.

The backtracking phase explores all valid partition combinations. In the worst case, such as `"aaaaaa..."`, nearly every substring is a palindrome, causing the number of valid partitions to grow exponentially.

The recursion stack depth is at most `n`, and the result list itself may contain exponentially many partitions, which dominates the total space usage.

## Edge Cases

### Single Character String

A string such as `"a"` is the smallest valid input. A naive implementation might incorrectly treat this as a special case or fail to terminate recursion properly.

This implementation handles it naturally because every single character is marked as a palindrome during preprocessing. The backtracking function immediately forms one valid partition.

### No Multi-Character Palindromes

Consider:

```
"abc"
```

The only valid partition is:

```
["a", "b", "c"]
```

A buggy implementation might incorrectly assume larger substrings could still work or fail to prune invalid paths efficiently.

The palindrome lookup table prevents this issue because non-palindrome substrings are rejected immediately during recursion.

### All Characters Identical

Consider:

```
"aaaa"
```

This creates many valid partitions:

```
["a","a","a","a"]
["aa","a","a"]
["a","aa","a"]
["a","a","aa"]
["aa","aa"]
["aaa","a"]
["a","aaa"]
["aaaa"]
```

This case is important because it produces the maximum branching factor during recursion.

The implementation handles this correctly because backtracking systematically explores every valid palindrome substring while preserving correctness through state restoration after each recursive call.
