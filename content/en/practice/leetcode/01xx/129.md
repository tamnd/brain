## Solution

## Problem Understanding

This problem asks us to compute the sum of all numbers formed by root-to-leaf paths in a binary tree. Every node contains a single digit from `0` to `9`, and each path from the root to a leaf represents a decimal number created by concatenating the digits along that path.

For example, consider the path:

```
1 -> 2 -> 3
```

This path represents the number:

```
123
```

The task is to compute the sum of every such root-to-leaf number in the tree.

The input is the root node of a binary tree. Each node contains a single digit. The output is a single integer representing the total sum of all numbers formed by valid root-to-leaf paths.

The constraints tell us several important things:

- The tree contains at most `1000` nodes.
- The depth of the tree is at most `10`.
- Node values are digits from `0` to `9`.

The small maximum depth is particularly important because it means each constructed number has at most 10 digits. The problem also guarantees that the final answer fits inside a 32-bit integer.

A few important observations emerge immediately:

- We only care about complete root-to-leaf paths.
- Internal nodes do not contribute directly to the final answer unless they are part of a path leading to a leaf.
- Since each node is a digit, building the current number follows decimal rules:

```
new_number = current_number * 10 + node.val
```

Several edge cases are worth considering:

- A tree with only one node, where the answer is simply the value of that node.
- Trees containing zeros, such as `1 -> 0 -> 5`, which should correctly form `105`.
- Highly unbalanced trees, where recursion depth could become large in other problems, though the depth limit of 10 makes recursion safe here.

## Approaches

### Brute Force Approach

A straightforward brute-force solution is to explicitly construct every root-to-leaf path as a string or list of digits. Once a leaf is reached, we convert the collected digits into an integer and add it to the result.

For example:

```
4 -> 9 -> 5
```

could first be stored as:

```
["4", "9", "5"]
```

then converted into:

```
495
```

This works because every root-to-leaf path is visited exactly once, and every constructed number is correctly interpreted.

However, this approach performs unnecessary work:

- We repeatedly allocate memory for path storage.
- We repeatedly convert strings into integers.
- Copying path information at each recursive step increases overhead.

Although the constraints are small enough that this solution would pass, it is not the most elegant or efficient approach.

### Optimal Approach

The key insight is that we do not need to store the entire path explicitly.

At every node, we already know the number formed so far from the root to the parent. To append the current digit, we simply shift the existing number one decimal place to the left and add the new digit.

Mathematically:

```
new_number = previous_number * 10 + current_digit
```

This allows us to build the number incrementally during traversal.

A depth-first traversal is ideal because:

- We naturally move from root to leaf.
- Each recursive call carries the current partially formed number.
- Once we reach a leaf, the accumulated value is already the complete number for that path.

This eliminates the need for extra path storage or string conversions.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N * H) | O(H) | Stores and converts entire root-to-leaf paths |
| Optimal | O(N) | O(H) | Builds numbers incrementally during DFS |

Here:

- `N` is the number of nodes.
- `H` is the height of the tree.

## Algorithm Walkthrough

1. Start a depth-first traversal from the root node.
2. Maintain a variable representing the number formed from the root down to the current node.
3. At each node, update the current number using:

```
current = current * 10 + node.val
```

This appends the current digit to the end of the existing number.
4. Check whether the current node is a leaf node. A node is a leaf if both its left and right children are `None`.
5. If the node is a leaf, return the completed number because this path now represents a valid root-to-leaf number.
6. Otherwise, recursively continue traversing the left and right subtrees.
7. Sum the results returned from both recursive calls.
8. Return the final accumulated sum from the root call.

The recursion naturally handles traversal and backtracking. No explicit stack or additional path structure is required because the recursive call stack already tracks traversal state.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def sumNumbers(self, root: Optional[TreeNode]) -> int:
        
        def dfs(node: Optional[TreeNode], current_number: int) -> int:
            if not node:
                return 0
            
            current_number = current_number * 10 + node.val
            
            if not node.left and not node.right:
                return current_number
            
            left_sum = dfs(node.left, current_number)
            right_sum = dfs(node.right, current_number)
            
            return left_sum + right_sum
        
        return dfs(root, 0)
```

The implementation uses a helper function named `dfs` to perform recursive depth-first traversal.

The parameter `current_number` stores the value formed from the root down to the current node. At each recursive step, the current digit is appended using:

```
current_number = current_number * 10 + node.val
```

If the current node is a leaf node, the completed number is returned immediately.

For internal nodes, recursion continues into both subtrees. Each subtree independently computes the sum of all root-to-leaf numbers beneath it. The current call then combines those sums and returns the total upward.

The base case:

```
if not node:
    return 0
```

ensures that missing children contribute nothing to the final sum.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func sumNumbers(root *TreeNode) int {
    
    var dfs func(node *TreeNode, currentNumber int) int

    dfs = func(node *TreeNode, currentNumber int) int {
        if node == nil {
            return 0
        }

        currentNumber = currentNumber*10 + node.Val

        if node.Left == nil && node.Right == nil {
            return currentNumber
        }

        leftSum := dfs(node.Left, currentNumber)
        rightSum := dfs(node.Right, currentNumber)

        return leftSum + rightSum
    }

    return dfs(root, 0)
}
```

The Go solution follows the same recursive DFS strategy as the Python version.

A few Go-specific details are worth noting:

- Go uses `nil` instead of `None`.
- The recursive helper function is declared as a function variable so it can recursively reference itself.
- Integer handling is straightforward because the problem guarantees the result fits within a 32-bit integer.

The recursive structure and traversal logic remain identical.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(N) | Every node is visited exactly once |
| Space | O(H) | Recursive call stack stores at most tree height frames |

The algorithm performs a single DFS traversal through the tree. Each node contributes constant work:

- One arithmetic update
- One leaf check
- Recursive calls

Therefore, the total running time is linear in the number of nodes.

The space complexity comes from recursion depth. In the worst case, the recursion stack contains one frame per tree level. Since the tree height is `H`, the auxiliary space complexity is `O(H)`.

## Edge Cases

### Single Node Tree

Consider a tree containing only one node:

```
[7]
```

The root is also a leaf, so the number represented by the only path is simply `7`.

A common bug is forgetting that the root itself can be a leaf. This implementation correctly handles it because the leaf check occurs immediately after updating the current number.

### Nodes Containing Zero

Consider:

```
1 -> 0 -> 5
```

The correct number is `105`, not `15`.

A naive implementation that stores digits incorrectly or trims leading zeros improperly could produce the wrong result. This solution avoids such issues because the number is built mathematically using decimal expansion:

```
current = current * 10 + digit
```

### Highly Unbalanced Tree

Consider a tree shaped like a linked list:

```
1
 \
  2
   \
    3
```

The traversal still works correctly because DFS naturally follows the single path until the leaf is reached.

Although recursive depth can sometimes be dangerous in tree problems, the maximum depth here is only `10`, making recursion completely safe within problem constraints.
