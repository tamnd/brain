## Solution

## Problem Understanding

The problem asks us to find the length of the longest contiguous substring in a given string `s` that contains **at most two distinct characters**.

A substring is a continuous sequence of characters within the original string. The phrase "at most two distinct characters" means the substring may contain one unique character or two unique characters, but never three or more.

For example, in the string `"eceba"`, the substring `"ece"` contains only the characters `'e'` and `'c'`, which makes it valid. Its length is `3`, and no longer valid substring exists, so the answer is `3`.

The input is a single string `s`, composed only of English letters. The expected output is an integer representing the maximum length of a valid substring.

The constraints are important:

- `1 <= s.length <= 10^5`
- `s` contains only English letters

A string length of up to `100,000` means we must be careful about performance. Any solution with quadratic complexity, such as `O(n²)`, will likely be too slow in the worst case. This strongly suggests we should aim for a linear or near-linear solution.

Several edge cases are worth identifying early:

A string with only one distinct character, such as `"aaaaa"`, should return the full length because it already satisfies the condition.

A string with exactly two distinct characters, such as `"ababab"`, should also return the full length.

A string that quickly introduces a third distinct character, such as `"abc"`, requires shrinking the considered substring appropriately.

Very short strings, such as a single-character input `"a"` or a two-character input `"ab"`, should also work naturally without requiring special handling.

## Approaches

### Brute Force Approach

A straightforward way to solve this problem is to generate **every possible substring** and check whether it contains at most two distinct characters.

We can do this by choosing every starting index and every ending index, forming a substring for each pair. For each substring, we count how many distinct characters it contains. If the number of distinct characters is less than or equal to two, we update our maximum length.

This approach is correct because it exhaustively examines all candidate substrings. Since the answer must be one of those substrings, we are guaranteed to find it.

However, the performance is poor. There are `O(n²)` substrings in a string of length `n`, and checking distinct characters can take additional time. Even with optimizations, the total runtime becomes too expensive for `n = 10^5`.

We need a way to avoid repeatedly recomputing information for overlapping substrings.

### Optimal Approach, Sliding Window with Hash Map

The key insight is that we only care about **contiguous substrings**, and the condition depends on the **number of distinct characters currently inside the substring**.

Instead of checking every substring independently, we can maintain a moving window over the string.

We use two pointers:

- A `left` pointer marking the start of the current window
- A `right` pointer expanding the window character by character

We also maintain a hash map that tracks the frequency of characters currently inside the window.

As we expand the window, we may eventually exceed two distinct characters. When that happens, we repeatedly move the `left` pointer forward and remove characters from the frequency map until the window becomes valid again.

This works because every character enters and leaves the window at most once, giving us an efficient linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Checks every substring and counts distinct characters |
| Optimal | O(n) | O(1) | Sliding window with a hash map, at most three distinct characters tracked |

Although the optimal solution uses a hash map, the space complexity is effectively `O(1)` because the alphabet consists only of English letters, meaning the map size is bounded by a small constant.

## Algorithm Walkthrough

1. Initialize two pointers, `left` and `right`, to represent the boundaries of a sliding window. Initially, both start at the beginning of the string.
2. Create a hash map called `character_count` to store how many times each character appears inside the current window. This allows us to efficiently determine how many distinct characters are present.
3. Create a variable `max_length` to track the longest valid substring found so far.
4. Iterate through the string using the `right` pointer. For each character:

- Add the current character to `character_count`.
- Increase its frequency by one.
5. After adding a character, check whether the window now contains more than two distinct characters:

- If the hash map size exceeds `2`, the window is invalid.
- Move the `left` pointer forward.
- Decrease the count of the character leaving the window.
- If a character count becomes zero, remove it from the hash map entirely.
6. Continue shrinking until the window becomes valid again, meaning it contains at most two distinct characters.
7. Once the window is valid, compute its length using:

```
right - left + 1
```

Compare this with `max_length` and update if necessary.
8. Continue until the `right` pointer reaches the end of the string.
9. Return `max_length`, which now stores the length of the longest valid substring.

The reason a hash map is a good choice is that we need constant-time insertion, deletion, and lookup for character frequencies. Since characters may enter and leave the window frequently, efficiency is essential.

## Python Solution

```
class Solution:
    def lengthOfLongestSubstringTwoDistinct(self, s: str) -> int:
        left = 0
        max_length = 0
        character_count: dict[str, int] = {}

        for right in range(len(s)):
            current_char = s[right]
            character_count[current_char] = (
                character_count.get(current_char, 0) + 1
            )

            while len(character_count) > 2:
                left_char = s[left]
                character_count[left_char] -= 1

                if character_count[left_char] == 0:
                    del character_count[left_char]

                left += 1

            current_window_length = right - left + 1
            max_length = max(max_length, current_window_length)

        return max_length
```

The implementation begins by initializing the sliding window boundaries and the hash map for character frequencies.

The `right` pointer iterates through the string one character at a time. Each character is added to the frequency map.

Whenever the number of distinct characters exceeds two, the `while` loop shrinks the window from the left side. We decrement frequencies as characters leave the window and remove entries completely once their count reaches zero. Removing zero-frequency characters is important because the number of keys in the map represents the number of distinct characters currently in the window.

After restoring validity, we compute the current window length and update `max_length` if this window is larger than any previously seen valid window.

Because each character is added and removed at most once, the implementation runs efficiently in linear time.

## Go Solution

```
func lengthOfLongestSubstringTwoDistinct(s string) int {
	left := 0
	maxLength := 0
	characterCount := make(map[byte]int)

	for right := 0; right < len(s); right++ {
		currentChar := s[right]
		characterCount[currentChar]++

		for len(characterCount) > 2 {
			leftChar := s[left]
			characterCount[leftChar]--

			if characterCount[leftChar] == 0 {
				delete(characterCount, leftChar)
			}

			left++
		}

		currentWindowLength := right - left + 1
		if currentWindowLength > maxLength {
			maxLength = currentWindowLength
		}
	}

	return maxLength
}
```

The Go implementation follows the exact same logic as the Python version, but there are a few language-specific details.

Instead of a Python dictionary, Go uses `map[byte]int` to store character frequencies. Since the input consists only of English letters, accessing characters as `byte` values is perfectly safe and efficient.

Go does not have a built-in `max()` function for integers, so we update `maxLength` using a simple conditional check.

There are no concerns about integer overflow because the maximum string length is only `10^5`, which comfortably fits inside Go's integer range.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character enters and leaves the sliding window at most once |
| Space | O(1) | At most a constant number of characters are stored in the hash map |

The time complexity is linear because the `right` pointer moves through the string once, and the `left` pointer also advances at most once per character. No character is revisited unnecessarily.

The space complexity is constant because the hash map can only contain a limited number of English letters, and during execution it never stores more than three distinct characters at once before shrinking occurs.

## Edge Cases

### Single Character String

Consider the input `"a"`.

A naive implementation might accidentally fail when handling extremely short inputs or assume there are at least two characters. In this case, the sliding window immediately contains one valid character, and the algorithm correctly returns `1`.

### All Characters Are the Same

Consider `"aaaaaa"`.

This case can expose bugs where implementations unnecessarily shrink the window despite only having one distinct character. Since the window always remains valid, the algorithm simply expands to include the entire string and returns `6`.

### Exactly Two Distinct Characters

Consider `"abababab"`.

Some incorrect implementations mistakenly shrink the window when it reaches exactly two distinct characters instead of more than two. Here, the entire string remains valid throughout execution, so the algorithm correctly returns the full length.

### Frequent Introduction of a Third Character

Consider `"abcabcabc"`.

This pattern repeatedly introduces a third distinct character, forcing frequent shrinking of the window. A buggy implementation might shrink only once instead of repeatedly until validity is restored. The `while len(character_count) > 2` loop ensures the window is always repaired correctly before continuing.
