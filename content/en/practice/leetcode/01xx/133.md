# LeetCode 133 - Clone Graph

## Problem Understanding

This problem asks us to create a deep copy of an undirected connected graph.

A graph consists of nodes, where each node contains two things:

- An integer value, `val`
- A list of neighboring nodes, `neighbors`

The input provides a reference to one node in the graph, specifically the node with value `1`. From that node, every other node in the graph can be reached because the graph is guaranteed to be connected.

The goal is not to duplicate just the values. We must create entirely new node objects and rebuild all neighbor relationships so that the copied graph has the exact same structure as the original graph.

A deep copy means:

- Every node in the cloned graph must be a newly allocated node
- Neighbor relationships must point only to cloned nodes
- No node in the cloned graph should reference an original node

The graph is represented conceptually as an adjacency list. For example:

```
[[2,4],[1,3],[2,4],[1,3]]
```

means:

- Node 1 connects to nodes 2 and 4
- Node 2 connects to nodes 1 and 3
- Node 3 connects to nodes 2 and 4
- Node 4 connects to nodes 1 and 3

The output should be another graph with the exact same adjacency structure.

The constraints are relatively small:

- At most 100 nodes
- Node values are unique
- No self-loops
- No duplicate edges

Although the graph size is small, the main challenge is handling cycles correctly. Since graphs can contain cycles, a naive recursive copy can easily enter infinite recursion if we repeatedly revisit already cloned nodes.

Several important edge cases appear immediately:

- The input node can be `None`, representing an empty graph
- A node may have no neighbors
- The graph may contain cycles
- Multiple nodes may reference the same neighbor

These cases require careful bookkeeping to avoid creating duplicate cloned nodes or entering infinite loops.

## Approaches

### Brute Force Approach

A naive brute-force approach would recursively clone every node and all of its neighbors without tracking previously visited nodes.

For each node:

1. Create a copy
2. Recursively clone all neighbors
3. Attach cloned neighbors to the copied node

At first glance, this seems correct because every node eventually gets copied.

However, this approach fails for graphs with cycles.

Consider this simple graph:

```
1 <-> 2
```

If we start cloning node 1:

- Clone node 1
- Clone neighbor node 2
- While cloning node 2, clone neighbor node 1 again
- While cloning the new node 1, clone node 2 again

This process continues forever.

Even if we somehow stopped recursion, we could still incorrectly create multiple cloned versions of the same node, breaking the graph structure.

Because graphs naturally contain repeated references and cycles, we need a way to remember which nodes have already been cloned.

### Optimal Approach

The key insight is that every original node must map to exactly one cloned node.

This naturally suggests using a hash map:

```
original node -> cloned node
```

When we encounter a node:

- If it has already been cloned, return the existing clone
- Otherwise:

- Create a clone
- Store it in the hash map immediately
- Recursively clone neighbors

Storing the clone before processing neighbors is extremely important because it prevents infinite recursion in cyclic graphs.

This approach performs a graph traversal using either:

- Depth-First Search (DFS)
- Breadth-First Search (BFS)

DFS is simpler to implement recursively, so we use it here.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Infinite in cyclic graphs | Infinite in cyclic graphs | Repeatedly revisits nodes and cannot handle cycles |
| Optimal | O(V + E) | O(V) | Uses DFS with a hash map to clone each node exactly once |

Where:

- `V` is the number of vertices (nodes)
- `E` is the number of edges

## Algorithm Walkthrough

### DFS with Hash Map

1. First, check whether the input node is `None`.

If the graph is empty, return `None` immediately because there is nothing to clone.
2. Create a hash map called `cloned_nodes`.

This map stores:

```
original node -> cloned node
```

The purpose of this map is to ensure that every original node is cloned exactly once.
3. Define a recursive DFS function.

The function receives an original node and returns its cloned version.
4. Inside the DFS function, first check whether the current node already exists in the hash map.

If it does, return the existing clone immediately.

This prevents:

- Infinite recursion
- Duplicate cloned nodes
5. If the node has not been cloned yet:

- Create a new cloned node with the same value
- Store it in the hash map immediately

Storing first is critical because neighboring nodes may point back to this node due to cycles.
6. Traverse every neighbor of the original node.

For each neighbor:

- Recursively clone it using DFS
- Append the cloned neighbor to the current cloned node's neighbor list
7. After all neighbors are processed, return the cloned node.
8. Start DFS from the input node and return the final cloned graph root.

## Python Solution

```
"""
# Definition for a Node.
class Node:
    def __init__(self, val = 0, neighbors = None):
        self.val = val
        self.neighbors = neighbors if neighbors is not None else []
"""

from typing import Optional

class Solution:
    def cloneGraph(self, node: Optional['Node']) -> Optional['Node']:
        if not node:
            return None

        cloned_nodes = {}

        def dfs(current_node: 'Node') -> 'Node':
            if current_node in cloned_nodes:
                return cloned_nodes[current_node]

            clone = Node(current_node.val)
            cloned_nodes[current_node] = clone

            for neighbor in current_node.neighbors:
                clone.neighbors.append(dfs(neighbor))

            return clone

        return dfs(node)
```

### Python Implementation Explanation

The solution begins by handling the empty graph case:

```
if not node:
    return None
```

This prevents errors when the input graph does not contain any nodes.

Next, we create the hash map:

```
cloned_nodes = {}
```

This dictionary ensures that each original node maps to exactly one cloned node.

The recursive DFS function handles graph traversal and cloning simultaneously.

The first important step inside DFS is:

```
if current_node in cloned_nodes:
    return cloned_nodes[current_node]
```

This avoids revisiting already cloned nodes. It is the mechanism that safely handles cycles.

When a node is first encountered, we create its clone:

```
clone = Node(current_node.val)
```

Then we immediately store it:

```
cloned_nodes[current_node] = clone
```

This timing is critical because neighboring nodes may point back to this node.

Finally, we recursively process neighbors:

```
for neighbor in current_node.neighbors:
    clone.neighbors.append(dfs(neighbor))
```

Each neighbor is cloned recursively, and the cloned references are added to the cloned node's neighbor list.

The final result is returned from:

```
return dfs(node)
```

## Go Solution

```
/**
 * Definition for a Node.
 * type Node struct {
 *     Val int
 *     Neighbors []*Node
 * }
 */

func cloneGraph(node *Node) *Node {
    if node == nil {
        return nil
    }

    clonedNodes := make(map[*Node]*Node)

    var dfs func(*Node) *Node

    dfs = func(currentNode *Node) *Node {
        if clone, exists := clonedNodes[currentNode]; exists {
            return clone
        }

        clone := &Node{
            Val: currentNode.Val,
        }

        clonedNodes[currentNode] = clone

        for _, neighbor := range currentNode.Neighbors {
            clone.Neighbors = append(clone.Neighbors, dfs(neighbor))
        }

        return clone
    }

    return dfs(node)
}
```

### Go Implementation Differences

The Go implementation follows the same DFS strategy as the Python version.

The main difference is the use of pointers:

```
map[*Node]*Node
```

The hash map uses original node pointers as keys and cloned node pointers as values.

The empty graph check uses `nil` instead of `None`:

```
if node == nil
```

Neighbor lists are represented as slices:

```
[]*Node
```

Appending neighbors uses Go's built-in `append` function.

Because node values are small integers, integer overflow is not a concern in this problem.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(V + E) | Each node and edge is visited exactly once |
| Space | O(V) | Hash map and recursion stack store up to all nodes |

The DFS traversal visits every node exactly one time because already cloned nodes are immediately returned from the hash map.

Each edge is also processed once while traversing neighbor lists.

The auxiliary space comes from two sources:

- The hash map storing cloned nodes
- The recursion stack during DFS traversal

In the worst case, both can contain all `V` nodes.

## Edge Cases

### Empty Graph

The input may be `None`, representing an empty graph.

This case is easy to overlook because many graph problems assume at least one node exists. Without checking for `None`, the solution would attempt to access attributes of a null object and crash.

The implementation handles this safely with:

```
if not node:
    return None
```

and similarly in Go:

```
if node == nil
```

### Single Node Without Neighbors

A graph may contain exactly one node and no edges.

A buggy implementation might incorrectly initialize neighbors or accidentally return the original node instead of a clone.

The DFS correctly creates one new node and processes an empty neighbor list, producing a valid deep copy.

### Cyclic Graphs

Graphs frequently contain cycles.

For example:

```
1 <-> 2
```

Without a visited map, recursive traversal would continue forever.

The hash map prevents this by returning already created clones immediately:

```
if current_node in cloned_nodes:
    return cloned_nodes[current_node]
```

This guarantees termination and ensures that each original node corresponds to exactly one cloned node.

### Multiple References to the Same Node

A node may be reachable through multiple paths.

If we cloned nodes every time we encountered them, we would create duplicate copies and destroy the graph structure.

The hash map guarantees structural correctness because all references to the same original node reuse the same cloned instance.
