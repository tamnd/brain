## Solution

## Problem Understanding

This problem asks us to modify a two-dimensional grid in-place by identifying regions of `'O'` cells that are completely surrounded by `'X'` cells. Any such surrounded region must be converted into `'X'`.

The board is represented as an `m x n` matrix where every cell contains either `'X'` or `'O'`. Cells are considered connected only in four directions: up, down, left, and right. Diagonal connections do not count.

The important detail is the definition of a **surrounded region**. A group of connected `'O'` cells is considered surrounded only if none of those cells touch the border of the board. If even one cell in a connected region touches the edge, the entire region is safe and should remain unchanged.

For example, consider the following board:

```
X X X X
X O O X
X X O X
X O X X
```

The cluster in the middle is completely enclosed by `'X'`, so those `'O'` cells should be captured and changed to `'X'`. However, the `'O'` in the bottom row remains unchanged because it touches the border, which means it cannot be surrounded.

A naive interpretation might attempt to check every `'O'` region independently and determine whether it touches an edge. While correct, this can be inefficient because the same cells may be revisited many times.

The constraints tell us that:

- `1 <= m, n <= 200`
- The board contains only `'X'` and `'O'`

At worst, the board contains `200 × 200 = 40,000` cells. This size is manageable for graph traversal algorithms such as Depth-First Search (DFS) or Breadth-First Search (BFS), but inefficient repeated traversals may become costly.

Several edge cases are important to recognize immediately:

- A board with only one row or one column cannot contain surrounded regions because every cell touches the border.
- A board containing only `'X'` requires no changes.
- A board containing only `'O'` should remain unchanged because every region touches an edge.
- Regions that appear enclosed but connect indirectly to the boundary through a chain of `'O'` cells must not be captured.

The core challenge is correctly distinguishing safe `'O'` regions from surrounded ones.

## Approaches

### Brute Force Approach

A straightforward solution is to examine every `'O'` region independently.

For each unvisited `'O'`, we could perform a DFS or BFS to collect all connected `'O'` cells in that region. During traversal, we track whether any cell touches the board boundary. If the region never touches an edge, we convert every cell in the region into `'X'`. Otherwise, we leave it unchanged.

This method is correct because every connected component of `'O'` cells is analyzed separately, and the boundary condition determines whether the region is surrounded.

However, the problem with this approach is inefficiency. If implemented carelessly, the same cells may be revisited repeatedly while checking multiple regions. Maintaining temporary storage for each component also increases overhead.

### Key Insight for the Optimal Solution

Instead of asking:

> "Which `'O'` regions are surrounded?"

we can ask:

> "Which `'O'` regions are definitely **not** surrounded?"

Any `'O'` connected to the border can never be captured. Therefore:

1. Start from every border `'O'`.
2. Mark all connected `'O'` cells as **safe**.
3. After marking finishes:

- Any remaining `'O'` must be surrounded, so convert it to `'X'`.
- Convert temporary safe markers back to `'O'`.

This is much more efficient because every cell is processed at most once.

We typically mark safe cells using a temporary character such as `'T'` (for temporary or trusted).

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m × n) | O(m × n) | Explore every region separately and track whether it touches the border |
| Optimal | O(m × n) | O(m × n) | Start from border `'O'` cells and mark safe regions |

Although both approaches can achieve linear time in careful implementations, the optimal solution is conceptually cleaner and avoids repeated region validation.

## Algorithm Walkthrough

1. **Handle small or empty boards**

If the board is empty or too small to contain interior regions, we can return immediately. A board with fewer than 3 rows or columns cannot contain surrounded cells.
2. **Define a DFS helper function**

Create a DFS function that marks safe cells. When it visits an `'O'`, change it to `'T'`.

We use `'T'` as a temporary marker because:

- `'X'` means wall
- `'O'` means unprocessed region
- `'T'` means safe region connected to border

This allows us to distinguish protected cells from surrounded ones.
3. **Traverse all border cells**

Since border-connected regions are safe, inspect:

- First row
- Last row
- First column
- Last column

Whenever a border cell contains `'O'`, start DFS from it.
4. **Mark connected safe regions**

During DFS:

- Move up, down, left, and right.
- Ignore out-of-bounds positions.
- Ignore cells that are not `'O'`.

Every reachable `'O'` becomes `'T'`.
5. **Convert surrounded regions**

After all border-connected regions are marked:

- Any remaining `'O'` is surrounded.
- Convert those cells into `'X'`.
6. **Restore safe regions**

Finally, convert every `'T'` back into `'O'`.

These are border-connected regions and should remain unchanged.

By separating the problem into "safe" and "captured" cells, the algorithm becomes simpler and avoids complicated region validation logic.

## Python Solution

```
from typing import List

class Solution:
    def solve(self, board: List[List[str]]) -> None:
        """
        Do not return anything, modify board in-place instead.
        """
        if not board or not board[0]:
            return

        rows, cols = len(board), len(board[0])

        def dfs(row: int, col: int) -> None:
            if (
                row < 0
                or row >= rows
                or col < 0
                or col >= cols
                or board[row][col] != "O"
            ):
                return

            board[row][col] = "T"

            dfs(row + 1, col)
            dfs(row - 1, col)
            dfs(row, col + 1)
            dfs(row, col - 1)

        for row in range(rows):
            if board[row][0] == "O":
                dfs(row, 0)

            if board[row][cols - 1] == "O":
                dfs(row, cols - 1)

        for col in range(cols):
            if board[0][col] == "O":
                dfs(0, col)

            if board[rows - 1][col] == "O":
                dfs(rows - 1, col)

        for row in range(rows):
            for col in range(cols):
                if board[row][col] == "O":
                    board[row][col] = "X"
                elif board[row][col] == "T":
                    board[row][col] = "O"
```

The implementation begins by checking whether the board is empty. This prevents index errors.

The `dfs()` helper function performs recursive traversal. Whenever an `'O'` is found, it is immediately marked as `'T'`. Marking before recursion prevents infinite loops and duplicate visits.

Next, the algorithm scans the border of the board. Since border-connected regions cannot be surrounded, DFS is launched only from border `'O'` cells.

After traversal completes, every safe cell has been marked `'T'`. The final nested loop performs cleanup:

- Remaining `'O'` cells are surrounded, so they become `'X'`
- Temporary `'T'` markers are restored back to `'O'`

This guarantees in-place modification while avoiding extra memory for visited tracking.

## Go Solution

```
func solve(board [][]byte) {
	if len(board) == 0 || len(board[0]) == 0 {
		return
	}

	rows, cols := len(board), len(board[0])

	var dfs func(int, int)

	dfs = func(row, col int) {
		if row < 0 || row >= rows ||
			col < 0 || col >= cols ||
			board[row][col] != 'O' {
			return
		}

		board[row][col] = 'T'

		dfs(row+1, col)
		dfs(row-1, col)
		dfs(row, col+1)
		dfs(row, col-1)
	}

	for row := 0; row < rows; row++ {
		if board[row][0] == 'O' {
			dfs(row, 0)
		}

		if board[row][cols-1] == 'O' {
			dfs(row, cols-1)
		}
	}

	for col := 0; col < cols; col++ {
		if board[0][col] == 'O' {
			dfs(0, col)
		}

		if board[rows-1][col] == 'O' {
			dfs(rows-1, col)
		}
	}

	for row := 0; row < rows; row++ {
		for col := 0; col < cols; col++ {
			if board[row][col] == 'O' {
				board[row][col] = 'X'
			} else if board[row][col] == 'T' {
				board[row][col] = 'O'
			}
		}
	}
}
```

The Go solution follows the same algorithmic structure as the Python version. Since Go slices can be empty, we first check `len(board)` and `len(board[0])` to avoid index access errors.

Go uses `[][]byte` rather than strings, so character comparisons use single quotes such as `'O'`, `'X'`, and `'T'`.

The DFS helper is implemented using a function variable because recursive anonymous functions in Go require a declared variable before assignment.

No integer overflow concerns exist because board dimensions are at most `200 × 200`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(m × n) | Every cell is visited at most once during DFS and once during cleanup |
| Space | O(m × n) | Recursive DFS call stack may contain all cells in the worst case |

The time complexity is linear because each cell is processed a constant number of times. Border scanning touches only perimeter cells, DFS visits each `'O'` once, and the final pass scans the board once.

The space complexity comes from recursion depth. In the worst case, such as a board filled entirely with `'O'`, the recursive stack could grow to `O(m × n)`.

## Edge Cases

### Single Row or Single Column

A board with only one row or one column cannot contain surrounded regions because every cell lies on the border.

For example:

```
O O O
```

All cells are automatically safe. The border DFS marks everything, and the final pass restores them unchanged.

### Entire Board Filled with `'O'`

Consider:

```
O O O
O O O
O O O
```

A naive implementation might incorrectly capture the center cell. However, every cell connects to the border through adjacent `'O'` cells.

The DFS started from border cells reaches every position and marks them safe. Nothing gets converted.

### Interior Surrounded Region

Consider:

```
X X X
X O X
X X X
```

The center `'O'` is not connected to any border cell, so it remains unmarked during DFS.

During the final sweep, the algorithm converts it into `'X'`, correctly capturing the surrounded region.

### Indirect Border Connection

Consider:

```
X O X X
X O O X
X X O X
```

Although some `'O'` cells are deep inside the board, they connect indirectly to the border through adjacent `'O'` cells.

A buggy implementation that only checks immediate border neighbors might incorrectly capture them. Our DFS avoids this mistake by exploring the entire connected component from border `'O'` cells.
