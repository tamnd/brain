## Solution

## Problem Understanding

The problem asks us to divide a string `s` into contiguous substrings such that **every substring is a palindrome**, while minimizing the number of cuts used.

A palindrome is a string that reads the same forwards and backwards. For example, `"aa"`, `"aba"`, and `"racecar"` are palindromes, while `"ab"` and `"abc"` are not.

The input is a single string `s`, consisting only of lowercase English letters. The task is not to return the partition itself, but instead to return the **minimum number of cuts** required so that every resulting substring is a palindrome.

For example, if `s = "aab"`:

- One valid partition is `["a", "a", "b"]`, which requires 2 cuts.
- A better partition is `["aa", "b"]`, because `"aa"` is a palindrome, requiring only 1 cut.

Therefore, the answer is `1`.

The constraints are important:

- `1 <= s.length <= 2000`
- The string contains only lowercase English letters.

A length of 2000 is large enough that naive exponential solutions will not work. Since many substring operations and palindrome checks are involved, repeatedly recomputing whether a substring is a palindrome would be too expensive. This strongly suggests using **dynamic programming** to reuse previously computed information.

Several edge cases stand out immediately:

- A single character string, such as `"a"`, already forms a palindrome, so no cuts are needed.
- A fully palindromic string, such as `"racecar"`, requires `0` cuts.
- A string with no palindromic substrings longer than one character, such as `"abcd"`, requires the maximum number of cuts, which is `n - 1`.
- Strings with overlapping palindrome opportunities, such as `"aab"` or `"banana"`, can trick greedy strategies into making poor decisions.

The problem guarantees that the input string is non-empty and contains only lowercase English letters, so we do not need to handle null inputs or special character processing.

## Approaches

### Brute Force Approach

A straightforward way to solve this problem is to try **every possible partition** of the string and check whether each substring is a palindrome.

At every position, we can decide whether to place a cut or continue extending the current substring. This naturally creates a recursive backtracking solution. For each partition, we verify that every substring is palindromic and count how many cuts were used. The smallest valid count becomes the answer.

This approach is correct because it exhaustively explores every valid partition configuration, guaranteeing that the minimum cut count is found.

However, it is extremely inefficient. A string of length `n` has roughly `2^(n-1)` possible ways to partition it, because each position between characters may or may not contain a cut. Additionally, checking whether substrings are palindromes repeatedly adds extra cost.

With `n = 2000`, an exponential-time algorithm is completely infeasible.

### Key Insight for an Optimal Solution

The main observation is that this problem has **overlapping subproblems**.

If we already know the minimum cuts needed for prefixes of the string, we can reuse those results instead of recomputing them repeatedly.

Define:

- `dp[i]` = minimum cuts needed for substring `s[0:i+1]`

For each ending position `i`, we try every possible starting position `j`.

If `s[j:i+1]` is a palindrome:

- If `j == 0`, then the whole prefix is already a palindrome, so `dp[i] = 0`.
- Otherwise, we can cut before `j`, giving:

`dp[i] = min(dp[i], dp[j-1] + 1)`

The remaining challenge is checking whether `s[j:i+1]` is a palindrome efficiently.

Instead of recomputing palindrome checks repeatedly, we precompute a **palindrome table**:

- `is_palindrome[j][i] = True` if substring `s[j:i+1]` is a palindrome.

A substring is a palindrome if:

- The characters at both ends match.
- The inner substring is also a palindrome.

This gives:

`is_palindrome[j][i] = (s[j] == s[i]) and (i - j <= 2 or is_palindrome[j+1][i-1])`

This transforms repeated palindrome checks into constant-time lookups.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n × n) | O(n) | Explores every partition and repeatedly checks palindromes |
| Optimal Dynamic Programming | O(n²) | O(n²) | Precomputes palindromes and computes minimum cuts efficiently |

## Algorithm Walkthrough

1. **Initialize the DP array**

Create an array `dp` of size `n`.

`dp[i]` will store the minimum cuts needed for substring `s[0:i+1]`.

Initially, assume the worst case for every position:

`dp[i] = i`

This corresponds to cutting between every character.
2. **Create a palindrome table**

Create a 2D boolean table `is_palindrome` of size `n × n`.

`is_palindrome[start][end]` tells us whether substring `s[start:end+1]` is a palindrome.

This avoids repeated substring checking.
3. **Iterate through ending positions**

For each ending index `end` from `0` to `n-1`, we determine the best way to partition the prefix ending there.
4. **Try all possible starting positions**

For each `start` from `0` to `end`, check whether `s[start:end+1]` forms a palindrome.

A substring is palindromic if:

- `s[start] == s[end]`
- and either:

- the substring length is at most 3, or
- the inner substring is already known to be a palindrome.
5. **Update the palindrome table**

If the substring is palindromic, mark:

`is_palindrome[start][end] = True`
6. **Update minimum cuts**

If the palindrome starts at index `0`, no cut is required:

`dp[end] = 0`

Otherwise:

`dp[end] = min(dp[end], dp[start - 1] + 1)`

This means we place one cut before the palindrome and combine it with the best solution for the prefix.
7. **Return the result**

After processing the whole string, `dp[n - 1]` contains the minimum cuts needed for the entire string.

## Python Solution

```
class Solution:
    def minCut(self, s: str) -> int:
        n = len(s)

        dp = [i for i in range(n)]
        is_palindrome = [[False] * n for _ in range(n)]

        for end in range(n):
            for start in range(end + 1):
                if s[start] == s[end] and (
                    end - start <= 2
                    or is_palindrome[start + 1][end - 1]
                ):
                    is_palindrome[start][end] = True

                    if start == 0:
                        dp[end] = 0
                    else:
                        dp[end] = min(dp[end], dp[start - 1] + 1)

        return dp[-1]
```

The implementation begins by initializing the `dp` array with the worst-case scenario, where every character requires a cut. For example, index `i` initially assumes `i` cuts.

The `is_palindrome` table stores whether any substring is a palindrome. This is crucial because repeatedly checking substrings directly would increase complexity dramatically.

The outer loop iterates through every ending index. For each ending index, the inner loop tries all possible starting indices.

Whenever a palindrome is found, the algorithm updates `is_palindrome[start][end]` to `True`. Then it determines whether the palindrome starts at the beginning of the string.

If it does, no cuts are needed for that prefix. Otherwise, the algorithm combines the current palindrome with the best partitioning found before it by using:

`dp[start - 1] + 1`

Since every substring pair `(start, end)` is processed only once, the algorithm runs efficiently in quadratic time.

## Go Solution

```
func minCut(s string) int {
	n := len(s)

	dp := make([]int, n)
	for i := 0; i < n; i++ {
		dp[i] = i
	}

	isPalindrome := make([][]bool, n)
	for i := 0; i < n; i++ {
		isPalindrome[i] = make([]bool, n)
	}

	for end := 0; end < n; end++ {
		for start := 0; start <= end; start++ {
			if s[start] == s[end] &&
				(end-start <= 2 || isPalindrome[start+1][end-1]) {

				isPalindrome[start][end] = true

				if start == 0 {
					dp[end] = 0
				} else {
					candidate := dp[start-1] + 1
					if candidate < dp[end] {
						dp[end] = candidate
					}
				}
			}
		}
	}

	return dp[n-1]
}
```

The Go implementation follows exactly the same algorithmic structure as the Python version. Since Go does not have list comprehensions, the `dp` array is initialized with a loop.

The 2D palindrome table is represented using a slice of boolean slices. Go strings can be indexed directly because the problem guarantees lowercase English letters, meaning byte indexing is safe.

Integer overflow is not a concern because the maximum number of cuts is at most `n - 1`, which is only `1999`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | Every `(start, end)` pair is processed once |
| Space | O(n²) | The palindrome table stores results for all substring pairs |

The algorithm uses two nested loops over the string, resulting in `O(n²)` time complexity. The palindrome lookup table ensures that palindrome checks occur in constant time rather than recomputing substrings repeatedly.

The main space cost comes from the `n × n` boolean matrix used to cache palindrome information. The `dp` array only requires `O(n)` space, but the overall complexity remains `O(n²)`.

## Edge Cases

### Single Character String

A string like `"a"` is already a palindrome. A naive implementation might incorrectly assume at least one cut is required. This implementation correctly identifies that the substring starts at index `0`, so `dp[0]` becomes `0`.

### Entire String Is a Palindrome

Consider `"racecar"`.

A buggy implementation might unnecessarily split the string into smaller palindromes. Here, when the algorithm identifies that the substring from index `0` to `n-1` is a palindrome, it immediately sets the minimum cut count to `0`.

### No Multi-Character Palindromes

For `"abcd"`, every character is isolated because no longer palindrome exists.

The optimal partition becomes:

`["a", "b", "c", "d"]`

requiring `3` cuts. Since each single character is a palindrome, the algorithm gradually builds:

`dp = [0, 1, 2, 3]`

which is the correct answer.

### Overlapping Palindrome Choices

A string such as `"aab"` can expose flaws in greedy strategies.

A greedy method might prematurely cut after the first `"a"` and miss the better partition `"aa" | "b"`.

This implementation evaluates all palindrome possibilities ending at each index and chooses the minimum cut count dynamically, ensuring globally optimal results.
