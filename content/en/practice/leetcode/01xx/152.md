## Solution

## Problem Understanding

The problem gives an integer array `nums` and asks us to find the contiguous subarray whose product is the largest among all possible subarrays.

A subarray must consist of consecutive elements from the original array. We cannot skip elements. For example, in `[2, 3, -2, 4]`, the subarray `[2, 3]` is valid, but `[2, -2]` is not because the elements are not contiguous.

The result we return is the maximum product value itself, not the subarray.

The constraints are important:

- The array length can be as large as `2 * 10^4`
- Each element is between `-10` and `10`
- The final answer fits in a 32-bit integer

The array size tells us that an `O(n^2)` or worse solution may become too slow in practice, especially since there can be about 200 million subarrays in the worst case.

This problem is trickier than the classic "Maximum Sum Subarray" because multiplication behaves differently from addition. Negative numbers can completely change the situation:

- A large positive product multiplied by a negative number becomes negative
- A large negative product multiplied by another negative number becomes positive
- Zero resets the product completely

These behaviors create several important edge cases:

- Arrays containing zeros
- Arrays containing only negative numbers
- Arrays where the maximum product comes from an even number of negatives
- Arrays with a single element
- Arrays where the optimal subarray appears in the middle

A naive implementation can easily fail because the current maximum product is not always enough information to continue correctly. Sometimes the current minimum product becomes extremely valuable after multiplying by a negative number.

## Approaches

### Brute Force Approach

The brute force solution considers every possible subarray.

For each starting index, we expand the subarray one element at a time while maintaining the running product. Every time we extend the subarray, we update the maximum product seen so far.

This works because every contiguous subarray is evaluated exactly once, so the true maximum product must eventually be discovered.

However, there are `O(n^2)` possible subarrays. Even though we can compute products incrementally, the overall runtime is still quadratic.

With `n = 2 * 10^4`, this approach becomes too slow.

### Optimal Dynamic Programming Approach

The key observation is that multiplication with negative numbers can reverse the meaning of maximum and minimum values.

Suppose we currently have:

- A very large positive product
- A very large negative product

If the next number is negative:

- The positive product becomes negative
- The negative product becomes positive

This means we cannot track only the maximum product ending at the current position. We must also track the minimum product ending at the current position.

At every index, we maintain:

- `current_max`, the maximum product ending at this index
- `current_min`, the minimum product ending at this index

When we encounter a negative number, these two values can swap roles.

For every new number, we consider three possibilities:

1. Start a new subarray from the current number
2. Extend the previous maximum product subarray
3. Extend the previous minimum product subarray

This allows us to process the array in a single pass.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every possible subarray product |
| Optimal Dynamic Programming | O(n) | O(1) | Tracks both maximum and minimum products ending at each position |

## Algorithm Walkthrough

1. Initialize three variables using the first element of the array:

- `current_max`
- `current_min`
- `global_max`

We use the first element because every subarray must contain at least one element.
2. Iterate through the array starting from index `1`.
3. For each number, first store the previous `current_max` because we will need it when computing the new minimum.
4. Compute the new maximum product ending at the current position by taking the maximum of:

- The current number itself
- Current number multiplied by previous `current_max`
- Current number multiplied by previous `current_min`

This handles both starting a new subarray and extending previous ones.
5. Compute the new minimum product ending at the current position using the same three candidates, but take the minimum instead.

Tracking the minimum is essential because multiplying a large negative value by another negative value can produce the next maximum.
6. Update `global_max` if the new `current_max` is larger.
7. Continue until the end of the array.
8. Return `global_max`.

This algorithm works because at every position we preserve exactly the information needed to build the optimal solution for the next position.

## Python Solution

```
from typing import List

class Solution:
    def maxProduct(self, nums: List[int]) -> int:
        current_max = nums[0]
        current_min = nums[0]
        global_max = nums[0]

        for i in range(1, len(nums)):
            num = nums[i]

            previous_max = current_max
            previous_min = current_min

            current_max = max(
                num,
                num * previous_max,
                num * previous_min
            )

            current_min = min(
                num,
                num * previous_max,
                num * previous_min
            )

            global_max = max(global_max, current_max)

        return global_max
```

The implementation begins by initializing all tracking variables with the first element. This avoids special handling for empty products and ensures single-element arrays work correctly.

Inside the loop, we examine the current number and calculate the best and worst products ending at the current index.

The variable `previous_max` stores the old maximum before it gets updated. This is important because the minimum calculation still needs the previous values.

For every element, we evaluate three possibilities:

- Start fresh from the current element
- Extend the previous maximum product
- Extend the previous minimum product

The maximum of these becomes the new `current_max`, while the minimum becomes the new `current_min`.

Finally, `global_max` tracks the best product seen anywhere in the array.

## Go Solution

```
func maxProduct(nums []int) int {
	currentMax := nums[0]
	currentMin := nums[0]
	globalMax := nums[0]

	for i := 1; i < len(nums); i++ {
		num := nums[i]

		previousMax := currentMax
		previousMin := currentMin

		currentMax = max(
			num,
			max(num*previousMax, num*previousMin),
		)

		currentMin = min(
			num,
			min(num*previousMax, num*previousMin),
		)

		globalMax = max(globalMax, currentMax)
	}

	return globalMax
}

func max(a, b int) int {
	if a > b {
		return a
	}
	return b
}

func min(a, b int) int {
	if a < b {
		return a
	}
	return b
}
```

The Go implementation follows the same logic as the Python solution. Since Go does not provide built-in `max` and `min` functions for integers, helper functions are defined explicitly.

The problem guarantees that the answer fits within a 32-bit integer, so standard Go `int` arithmetic is sufficient for LeetCode.

Unlike Python, Go does not have dynamic integer resizing, but the constraints ensure overflow is not an issue here.

The algorithm performs a single pass through the array. At every index, only constant-time calculations are performed. No additional arrays or data structures proportional to the input size are needed.

## Edge Cases

### Single Element Array

An array such as `[5]` or `[-3]` is important because the answer must simply be that single value.

Some implementations incorrectly assume subarrays must contain multiple elements or initialize variables improperly. This solution handles the case naturally because all tracking variables start with `nums[0]`.

### Arrays Containing Zero

A zero completely resets multiplication because any product involving zero becomes zero.

For example:

```
[2, -3, 0, 4, -1]
```

After reaching zero, previous products should not continue affecting future calculations. The algorithm handles this automatically because starting a new subarray with the current number is always considered as one candidate.

### Even Number of Negative Values

Negative numbers are the main challenge in this problem.

For example:

```
[-2, 3, -4]
```

The full product is:

```
(-2) * 3 * (-4) = 24
```

A solution that tracks only the maximum product would fail here because intermediate products become negative. By tracking both `current_max` and `current_min`, the algorithm correctly captures the sign reversal effect caused by multiplying negatives.

### All Negative Numbers

Consider:

```
[-1, -2, -3]
```

The best subarray is `[-2, -3]` with product `6`.

The algorithm correctly handles this because the running minimum can later become the running maximum after multiplication by another negative number.
