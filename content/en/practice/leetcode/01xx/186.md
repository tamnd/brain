## Solution

## Problem Understanding

This problem asks us to reverse the order of words inside a character array, while modifying the array directly in-place.

The input is not a normal string. Instead, it is an array of characters. For example:

```
["t","h","e"," ","s","k","y"]
```

represents the string:

```
"the sky"
```

The goal is not to reverse the characters inside each word. Instead, we must reverse the order of the words themselves.

For example:

```
"the sky is blue"
```

should become:

```
"blue is sky the"
```

The important constraint is that the solution must work in-place. That means we cannot create another large array or use extra memory proportional to the input size. We must modify the original array directly.

The constraints tell us several important things:

- The input length can be as large as `10^5`, so the algorithm must be efficient.
- Words are separated by exactly one space.
- There are no leading or trailing spaces.
- Every word contains only letters or digits.

These guarantees simplify the implementation because we do not need to handle multiple consecutive spaces or trim whitespace.

Several edge cases are important:

- A single-word input such as `["a"]`
- Very long inputs near the constraint limit
- Words with different lengths
- Inputs where reversing the entire array could accidentally corrupt word boundaries if handled incorrectly

A naive implementation using string splitting would violate the in-place requirement because it allocates additional memory.

## Approaches

### Brute Force Approach

A straightforward solution is to convert the character array into a string, split it into words, reverse the word list, join the words again, and finally copy the result back into the original array.

For example:

```
"the sky is blue"
```

would become:

```
["the", "sky", "is", "blue"]
```

Then:

```
["blue", "is", "sky", "the"]
```

Finally:

```
"blue is sky the"
```

This approach is easy to understand and implement. It produces the correct answer because splitting isolates each word and reversing the list changes the word order correctly.

However, it requires extra memory proportional to the input size because splitting and joining create new strings and lists. Since the problem explicitly requires an in-place solution, this approach does not satisfy the constraints.

### Optimal Approach

The key insight is that we can reverse the words in-place using two reversals:

1. Reverse the entire character array.
2. Reverse each individual word.

Consider:

```
"the sky is blue"
```

After reversing the entire array:

```
"eulb si yks eht"
```

Now the words are in the correct order, but each word itself is reversed.

If we then reverse each word individually:

```
"blue is sky the"
```

we obtain the correct result.

This works because reversing twice restores the original character order within each word while preserving the reversed word sequence.

This approach only uses constant extra space and processes the array in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Uses split and join, violates in-place requirement |
| Optimal | O(n) | O(1) | Reverses entire array, then reverses each word |

## Algorithm Walkthrough

1. Reverse the entire character array.

This step moves the words into reverse order automatically. However, each word itself becomes reversed.

Example:

```
"the sky"
```

becomes:

```
"yks eht"
```
2. Traverse the array to locate each word.

Since words are separated by single spaces, we can identify word boundaries by scanning until we encounter a space or the end of the array.
3. Reverse each individual word.

Once a word boundary is identified, reverse only that portion of the array. This restores the correct character order for the word.

Example:

```
"yks"
```

becomes:

```
"sky"
```
4. Continue until all words are processed.

By the end, every word has been corrected while the overall word order remains reversed.
5. Use a helper function for reversing.

A reusable reverse function simplifies the implementation. It swaps characters from both ends toward the center.

## Python Solution

```
from typing import List

class Solution:
    def reverseWords(self, s: List[str]) -> None:
        """
        Do not return anything, modify s in-place instead.
        """

        def reverse(left: int, right: int) -> None:
            while left < right:
                s[left], s[right] = s[right], s[left]
                left += 1
                right -= 1

        n = len(s)

        # Step 1: Reverse the entire array
        reverse(0, n - 1)

        # Step 2: Reverse each word
        start = 0

        for end in range(n + 1):
            if end == n or s[end] == ' ':
                reverse(start, end - 1)
                start = end + 1
```

The implementation begins with a helper function named `reverse`. This function swaps characters between two indices until the pointers meet in the middle.

The first major step reverses the entire array. At this point, the word order becomes reversed automatically, but each word is backwards.

Next, the algorithm scans through the array using the `end` pointer. Whenever it encounters a space or reaches the end of the array, it knows a complete word has been found.

The word between `start` and `end - 1` is then reversed in-place. After reversing that word, the next word begins immediately after the space.

The algorithm processes every character exactly a constant number of times, making it highly efficient.

## Go Solution

```
func reverseWords(s []byte) {
	n := len(s)

	reverse := func(left int, right int) {
		for left < right {
			s[left], s[right] = s[right], s[left]
			left++
			right--
		}
	}

	// Step 1: Reverse the entire slice
	reverse(0, n-1)

	// Step 2: Reverse each word
	start := 0

	for end := 0; end <= n; end++ {
		if end == n || s[end] == ' ' {
			reverse(start, end-1)
			start = end + 1
		}
	}
}
```

The Go implementation closely mirrors the Python solution. Since Go strings are immutable, the function uses a `[]byte` slice so characters can be modified directly in-place.

The helper function is implemented as a closure inside `reverseWords`, which keeps the logic localized and clean.

The traversal logic is identical to the Python version. The condition:

```
end == n || s[end] == ' '
```

handles both normal word boundaries and the final word at the end of the slice.

No additional arrays or slices are allocated, so the implementation satisfies the constant space requirement.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is visited a constant number of times |
| Space | O(1) | All modifications are done in-place |

The algorithm performs two major passes over the array. The first reverses the entire array, and the second reverses each word individually. Even though there are multiple reversals, every character participates in only a constant number of swaps, so the total runtime remains linear.

The space complexity is constant because the algorithm only uses a few pointer variables and does not allocate additional arrays or strings.

## Edge Cases

### Single Word Input

An input such as:

```
["h","e","l","l","o"]
```

contains only one word. Reversing the entire array changes the word to `"olleh"`, and reversing the word again restores it to `"hello"`.

The algorithm naturally handles this case without requiring special logic.

### Very Long Input

The input size can reach `10^5` characters. Any solution using repeated string concatenation or additional large arrays could become inefficient or violate memory constraints.

The in-place two-reversal approach remains efficient because it processes the array in linear time and constant space regardless of input size.

### Words of Different Lengths

Inputs may contain words with very different lengths:

```
"i love programming"
```

The algorithm correctly identifies word boundaries using spaces rather than relying on fixed lengths. Each word is reversed independently based on its actual start and end positions.

### Final Word Handling

A common bug occurs when processing the last word because there is no trailing space after it.

The condition:

```
if end == n or s[end] == ' ':
```

ensures the final word is processed correctly when the scan reaches the end of the array.
