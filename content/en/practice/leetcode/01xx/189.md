## Solution

## Problem Understanding

The problem asks us to rotate an array to the right by `k` positions. Rotating to the right means that each element moves `k` positions forward, and elements that go past the end of the array wrap around to the beginning.

For example, if we rotate `[1,2,3,4,5,6,7]` by `3` steps:

- `5` moves to index `0`
- `6` moves to index `1`
- `7` moves to index `2`
- The remaining elements shift right accordingly

The final array becomes `[5,6,7,1,2,3,4]`.

The input consists of:

- `nums`, an integer array
- `k`, a non-negative integer representing how many times to rotate

The function must modify the original array in-place. This is important because the problem specifically says not to return a new array.

The constraints tell us several important things:

- The array can contain up to `10^5` elements, so inefficient solutions may time out.
- `k` can also be as large as `10^5`, meaning we cannot safely rotate one step at a time without considering performance.
- Since the array length is at least `1`, we never need to handle a completely empty array in LeetCode's official constraints.

Several edge cases are important:

- `k` may be larger than the array length. Rotating by `n` positions returns the same array, so we should use `k % n`.
- `k` may be `0`, meaning no rotation should occur.
- The array may contain only one element, in which case rotation changes nothing.
- Negative numbers are allowed, but rotation logic is independent of element values.

## Approaches

### Brute Force Approach

A straightforward solution is to rotate the array one step at a time, repeating the process `k` times.

For each rotation:

1. Store the last element.
2. Shift every other element one position to the right.
3. Place the stored element at index `0`.

This works because each individual operation correctly performs a single right rotation. Repeating it `k` times produces the final rotated array.

However, this approach is inefficient. If the array has `n` elements and we rotate `k` times, each rotation costs `O(n)` work, producing a total complexity of `O(n * k)`.

With constraints up to `10^5`, this can become far too slow.

### Optimal Approach, Reverse Method

The key observation is that rotation rearranges the array in a very structured way.

Consider:

```
[1,2,3,4,5,6,7]
```

Rotated by `3` becomes:

```
[5,6,7,1,2,3,4]
```

Notice that the final array consists of:

- The last `k` elements moved to the front
- The first `n-k` elements shifted to the back

The reverse technique exploits this structure efficiently.

The process is:

1. Reverse the entire array
2. Reverse the first `k` elements
3. Reverse the remaining `n-k` elements

Example:

```
Original:
[1,2,3,4,5,6,7]

Reverse all:
[7,6,5,4,3,2,1]

Reverse first 3:
[5,6,7,4,3,2,1]

Reverse remaining:
[5,6,7,1,2,3,4]
```

This achieves the rotation in-place with only constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |

| Optimal | O(n) | O(1) | Uses three reversals to rotate efficiently |

## Algorithm Walkthrough

### Optimal Reverse Algorithm

1. Compute the array length `n`.

We need the size because rotation depends on wrapping around the array.
2. Reduce `k` using modulo.

If `k > n`, rotating more than `n` times repeats the same pattern. For example, rotating length `7` by `10` is equivalent to rotating by `3`.

```
k = k % n
```
3. Reverse the entire array.

This brings the elements that should appear at the front into the correct region.

Example:

```
[1,2,3,4,5,6,7]
->
[7,6,5,4,3,2,1]
```
4. Reverse the first `k` elements.

These elements represent the rotated front section. Reversing restores their correct order.

```
[7,6,5,4,3,2,1]
->
[5,6,7,4,3,2,1]
```
5. Reverse the remaining `n-k` elements.

This restores the order of the remaining suffix.

```
[5,6,7,4,3,2,1]
->
[5,6,7,1,2,3,4]
```
6. The array is now correctly rotated in-place.

The reverse operation itself uses two pointers:

- One pointer starts at the left
- One pointer starts at the right
- Swap elements while moving inward

This gives `O(1)` extra space.

## Python Solution

```
from typing import List

class Solution:
    def rotate(self, nums: List[int], k: int) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """

        n = len(nums)
        k %= n

        def reverse(left: int, right: int) -> None:
            while left < right:
                nums[left], nums[right] = nums[right], nums[left]
                left += 1
                right -= 1

        reverse(0, n - 1)
        reverse(0, k - 1)
        reverse(k, n - 1)
```

The implementation begins by computing the array length and reducing `k` using modulo arithmetic. This ensures that unnecessary full rotations are ignored.

A helper function named `reverse` performs in-place reversal using the two-pointer technique. The left and right pointers move toward each other, swapping elements until the section is fully reversed.

The algorithm then performs the three reversal operations:

1. Reverse the entire array
2. Reverse the first `k` elements
3. Reverse the remaining elements

Each reversal contributes to transforming the array into its rotated configuration without allocating additional memory.

## Go Solution

```
func rotate(nums []int, k int) {
	n := len(nums)
	k %= n

	reverse := func(left int, right int) {
		for left < right {
			nums[left], nums[right] = nums[right], nums[left]
			left++
			right--
		}
	}

	reverse(0, n-1)
	reverse(0, k-1)
	reverse(k, n-1)
}
```

The Go implementation follows the same logic as the Python version.

A closure function named `reverse` performs the in-place reversal. Go slices behave similarly to references for underlying array storage, so modifications inside the helper affect the original array directly.

Unlike Python, Go does not require explicit type hints because the language is statically typed. The tuple-style swap syntax:

```
nums[left], nums[right] = nums[right], nums[left]
```

provides a concise way to exchange values.

Since the constraints guarantee at least one element, division-by-zero issues with modulo do not occur.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element is visited a constant number of times across reversals |
| Space | O(1) | Rotation is performed entirely in-place |

The algorithm performs three reversals. Each reversal processes at most `n` elements, so the total work remains linear.

No auxiliary arrays or extra data structures are allocated. Only a few pointer variables are used, which keeps space complexity constant.

## Edge Cases

### Case 1, k Larger Than Array Length

Consider:

```
nums = [1,2,3]
k = 5
```

Rotating by `5` is equivalent to rotating by `2` because every full rotation of length `3` returns the array to its original state.

The implementation handles this using:

```
k %= n
```

Without this step, unnecessary work would occur and some algorithms could behave incorrectly.

### Case 2, Single Element Array

Consider:

```
nums = [10]
k = 100
```

No matter how many times we rotate, the array remains unchanged.

After modulo reduction:

```
k = 100 % 1 = 0
```

The reversal operations safely execute without modifying the array incorrectly.

### Case 3, k Equals Zero

Consider:

```
nums = [1,2,3]
k = 0
```

A rotation of zero means the array should remain exactly the same.

The algorithm still works correctly:

1. Reverse all
2. Reverse first `0` elements, which does nothing
3. Reverse the remaining elements

The net effect restores the original ordering.

### Case 4, Negative Numbers in Array

Consider:

```
nums = [-5,-2,-1]
k = 1
```

The algorithm does not depend on the numeric values themselves. It only rearranges positions.

As a result, negative values are handled naturally without requiring any special logic.
