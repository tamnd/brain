## Solution

## Problem Understanding

The problem asks us to find the length of the shortest transformation sequence between two words. A valid transformation changes exactly one character at a time, and every intermediate word must exist in the given dictionary, `wordList`.

We are given three inputs:

- `beginWord`, the starting word
- `endWord`, the target word
- `wordList`, the dictionary of allowed intermediate words

The goal is not to return the actual sequence, but instead to return how many words appear in the shortest valid sequence, including both the starting and ending words.

For example, if the sequence is:

```
hit -> hot -> dot -> dog -> cog
```

then the answer is `5`, because there are five words in the path.

The key rule is that every adjacent pair of words must differ by exactly one letter. This means:

- `"hit"` can become `"hot"` because only `'i'` changes to `'o'`
- `"hot"` cannot become `"dog"` because multiple letters differ

The constraints tell us several important things about the input size and structure:

- Each word has length at most `10`
- There can be up to `5000` words
- All words contain only lowercase English letters
- All dictionary words are unique

A naive graph construction comparing every pair of words would already require millions of comparisons in the worst case. Therefore, we need a more efficient approach.

Several edge cases are important:

- If `endWord` is not in `wordList`, the answer must immediately be `0`
- The shortest sequence may not exist at all
- `beginWord` itself may not appear in `wordList`
- Multiple shortest paths may exist, but we only need the minimum length
- The dictionary can contain many unrelated words that should not affect correctness

This problem is fundamentally a shortest path problem on an implicit graph, where each word is a node and edges connect words differing by one character.

## Approaches

### Brute Force Approach

The brute force idea is to model every word as a node in a graph and explicitly connect all pairs of words that differ by one character.

To build this graph:

1. Compare every pair of words
2. Check whether they differ by exactly one letter
3. If they do, create an edge between them

After constructing the graph, we can run Breadth-First Search, BFS, from `beginWord` to find the shortest path to `endWord`.

This works because BFS always finds the shortest path in an unweighted graph.

However, the graph construction is expensive. If there are `N` words and each word has length `L`, comparing every pair costs:

```
O(N^2 * L)
```

With up to `5000` words, this becomes inefficient.

### Key Insight for an Optimal Solution

The important observation is that two words are neighbors if they differ by exactly one character.

Instead of comparing every pair of words, we can generate neighbors directly.

For a word of length `L`, each position can be replaced with any of the `26` lowercase letters. This means each word has at most:

```
26 * L
```

possible transformations.

For example, from `"hot"`:

- Replace first character:

- `"aot"`, `"bot"`, `"cot"`, ...
- Replace second character:

- `"hat"`, `"hbt"`, `"hct"`, ...
- Replace third character:

- `"hoa"`, `"hob"`, `"hoc"`, ...

Most generated words will not exist in the dictionary, but checking membership in a hash set is very fast.

This transforms the problem into an efficient BFS where neighbors are generated dynamically rather than precomputed.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N² × L) | O(N²) | Explicitly builds the entire graph |
| Optimal BFS with Hash Set | O(N × L × 26) | O(N) | Generates neighbors dynamically during BFS |

## Algorithm Walkthrough

1. Convert `wordList` into a hash set.

A hash set allows constant time lookup when checking whether a transformed word exists in the dictionary. This is critical because we will perform many membership checks.
2. Check whether `endWord` exists in the dictionary.

If it does not exist, no valid transformation sequence is possible, so we immediately return `0`.
3. Initialize a BFS queue.

BFS is the correct algorithm because every transformation has equal cost, exactly one step. BFS guarantees that the first time we reach `endWord`, we have found the shortest path.

Each queue entry stores:

- the current word
- the current transformation length
4. Start BFS from `beginWord`.

The initial path length is `1` because the sequence already includes the starting word.
5. Process words level by level.

For the current word:

- Convert it into a mutable character array
- For every position in the word:

- Try replacing that character with every lowercase letter
- Form the transformed word
6. Check whether the transformed word is valid.

If the transformed word exists in the dictionary:

- It is a valid next step
- Add it to the BFS queue
- Remove it from the dictionary to avoid revisiting

Removing visited words immediately is important because:

- It prevents cycles
- It avoids repeated processing
- It keeps the runtime efficient
7. Stop when `endWord` is found.

The first time BFS reaches `endWord`, return the current transformation length plus one.
8. If BFS finishes without reaching `endWord`, return `0`.

This means no valid transformation sequence exists.

## Python Solution

```
from collections import deque
from typing import List

class Solution:
    def ladderLength(self, beginWord: str, endWord: str, wordList: List[str]) -> int:
        word_set = set(wordList)

        if endWord not in word_set:
            return 0

        queue = deque([(beginWord, 1)])

        while queue:
            current_word, steps = queue.popleft()

            if current_word == endWord:
                return steps

            word_chars = list(current_word)

            for i in range(len(word_chars)):
                original_char = word_chars[i]

                for c in "abcdefghijklmnopqrstuvwxyz":
                    if c == original_char:
                        continue

                    word_chars[i] = c
                    next_word = "".join(word_chars)

                    if next_word in word_set:
                        queue.append((next_word, steps + 1))
                        word_set.remove(next_word)

                word_chars[i] = original_char

        return 0
```

The implementation begins by converting `wordList` into a set called `word_set`. This allows constant time lookups when checking whether a generated transformation exists in the dictionary.

The early return for `endWord` is an important optimization and correctness check. Since every intermediate word must belong to the dictionary, reaching a missing `endWord` is impossible.

The queue stores pairs of `(word, steps)`. BFS explores transformations in increasing order of path length, which guarantees that the first time we reach the target, the sequence is shortest.

For each word, we attempt all possible one letter transformations. The nested loops iterate through:

- every character position
- every lowercase replacement letter

Whenever a generated word exists in `word_set`, it becomes a valid BFS neighbor.

The word is removed immediately after insertion into the queue. This acts as the visited set and prevents duplicate processing.

Finally, if BFS exhausts all possibilities without finding `endWord`, the function returns `0`.

## Go Solution

```
package main

func ladderLength(beginWord string, endWord string, wordList []string) int {
	wordSet := make(map[string]bool)

	for _, word := range wordList {
		wordSet[word] = true
	}

	if !wordSet[endWord] {
		return 0
	}

	type Pair struct {
		word  string
		steps int
	}

	queue := []Pair{{beginWord, 1}}

	for len(queue) > 0 {
		current := queue[0]
		queue = queue[1:]

		if current.word == endWord {
			return current.steps
		}

		wordChars := []byte(current.word)

		for i := 0; i < len(wordChars); i++ {
			originalChar := wordChars[i]

			for c := byte('a'); c <= byte('z'); c++ {
				if c == originalChar {
					continue
				}

				wordChars[i] = c
				nextWord := string(wordChars)

				if wordSet[nextWord] {
					queue = append(queue, Pair{
						word:  nextWord,
						steps: current.steps + 1,
					})

					delete(wordSet, nextWord)
				}
			}

			wordChars[i] = originalChar
		}
	}

	return 0
}
```

The Go implementation follows the same BFS strategy as the Python version.

Instead of Python's `deque`, Go uses a slice as the queue. Queue removal is implemented using:

```
queue = queue[1:]
```

Go strings are immutable, so transformations are performed using a `[]byte` array. This allows efficient character replacement during neighbor generation.

The dictionary is implemented as `map[string]bool`, which provides constant time lookup and deletion.

There are no integer overflow concerns because the maximum transformation length is bounded by the number of words.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(N × L × 26) | For each word, generate all possible one letter transformations |
| Space | O(N) | Queue and hash set store at most all dictionary words |

Here:

- `N` is the number of words in `wordList`
- `L` is the length of each word

Each word is processed at most once because visited words are removed from the set immediately after discovery.

For every processed word:

- We iterate through all `L` positions
- For each position, we try `26` possible letters

Thus the total work is:

```
O(N × L × 26)
```

Since `26` is constant, the complexity is often simplified to:

```
O(N × L)
```

## Edge Cases

### `endWord` Does Not Exist in the Dictionary

This is the most important edge case. The problem requires every intermediate and final word to exist in `wordList`. If `endWord` is absent, no valid transformation sequence can exist.

A buggy implementation might still attempt BFS and waste time exploring impossible paths. The solution handles this immediately with:

```
if endWord not in word_set:
    return 0
```

### No Possible Transformation Path

Even if `endWord` exists, the graph may still be disconnected.

For example:

```
beginWord = "hit"
endWord = "cog"
wordList = ["hot","dot","tod","hog","cog"]
```

There may be no valid sequence connecting the start to the target.

The BFS naturally handles this case. If the queue becomes empty before reaching `endWord`, the algorithm returns `0`.

### Repeated Processing and Cycles

Without a visited mechanism, the BFS could revisit the same words repeatedly and potentially enter cycles.

For example:

```
hot -> dot -> hot
```

This would cause unnecessary work or infinite loops.

The implementation prevents this by removing words from the hash set immediately after adding them to the queue:

```
word_set.remove(next_word)
```

This guarantees that every word is processed at most once.

### `beginWord` Not in `wordList`

The problem explicitly states that `beginWord` does not need to appear in the dictionary.

A flawed implementation might incorrectly assume otherwise.

This solution starts BFS directly from `beginWord`, regardless of whether it appears in `wordList`, which correctly matches the problem requirements.
