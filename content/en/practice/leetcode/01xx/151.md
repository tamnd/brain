## Solution

## Problem Understanding

This problem asks us to reverse the order of words in a string while cleaning up spaces at the same time.

A **word** is any sequence of non-space characters. The input string may contain extra spaces in several places:

- Leading spaces at the beginning
- Trailing spaces at the end
- Multiple consecutive spaces between words

The goal is to return a new string where:

1. The words appear in reverse order.
2. Exactly one space separates adjacent words.
3. No leading or trailing spaces exist.

For example:

```
Input:  "  hello world  "
Output: "world hello"
```

The spaces at the beginning and end are removed, and the word order is reversed.

The input size is relatively small, up to `10^4` characters, which means an `O(n)` or `O(n log n)` solution is perfectly acceptable. Since the task mainly involves scanning and rearranging strings, we should aim for a linear-time solution.

An important detail is that the problem guarantees there is **at least one word** in the string. This means we never need to handle an entirely empty input after trimming.

There are several edge cases that can trip up a naive implementation:

- Strings with leading and trailing spaces:

```
"   hello   "
```
- Multiple spaces between words:

```
"a   good    example"
```
- A single word:

```
"hello"
```
- Strings containing numbers or mixed characters:

```
"123 abc"
```

A naive approach that simply reverses the characters in the string would produce incorrect results because the problem wants the **order of words reversed**, not the characters themselves.

## Approaches

### Brute Force Approach

A straightforward solution is to manually parse the string character by character.

We can scan through the input, identify words, clean unnecessary spaces, and store each extracted word into a list. After collecting all words, we reverse the list and join them using a single space.

This approach works because every word is extracted exactly once, and reversing the collected list guarantees the final ordering is correct.

Although this approach is already efficient for the problem constraints, we can still consider it somewhat brute force because it explicitly constructs intermediate structures and performs separate parsing, reversing, and reconstruction phases.

### Key Insight for an Optimal Solution

The key observation is that Python and Go already provide convenient string operations for tokenizing words and reconstructing strings efficiently.

If we split the string by whitespace, consecutive spaces are automatically ignored. Once we have the words, reversing their order becomes trivial.

For example:

```
"  hello   world  "
```

After splitting:

```
["hello", "world"]
```

Reverse:

```
["world", "hello"]
```

Join with a single space:

```
"world hello"
```

This gives us a clean and concise linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Manually parse characters, extract words, reverse, reconstruct |
| Optimal | O(n) | O(n) | Split into words, reverse order, join with one space |

## Algorithm Walkthrough

1. **Split the string into words**

Use whitespace splitting to extract only valid words. This automatically removes extra spaces, including leading, trailing, and repeated spaces between words.
2. **Reverse the word order**

Once we have a list of words, reverse the sequence so that the last word becomes first.
3. **Join words using a single space**

Reconstruct the result string using exactly one space between words. This guarantees the formatting requirement is satisfied.
4. **Return the final string**

The resulting string contains reversed words with no unnecessary spaces.

The reason this works is that whitespace splitting normalizes spacing automatically, allowing us to focus only on word ordering.

## Python Solution

```
class Solution:
    def reverseWords(self, s: str) -> str:
        words = s.split()
        reversed_words = words[::-1]
        return " ".join(reversed_words)
```

The implementation is intentionally concise because Python provides powerful built-in string operations.

The first line:

```
words = s.split()
```

splits the input by whitespace. Unlike `split(" ")`, calling `split()` without arguments automatically ignores multiple consecutive spaces and removes leading and trailing spaces.

Next:

```
reversed_words = words[::-1]
```

creates a reversed copy of the list.

Finally:

```
return " ".join(reversed_words)
```

joins all words using exactly one space, satisfying the formatting requirement.

## Go Solution

```
package main

import "strings"

func reverseWords(s string) string {
	words := strings.Fields(s)

	left, right := 0, len(words)-1
	for left < right {
		words[left], words[right] = words[right], words[left]
		left++
		right--
	}

	return strings.Join(words, " ")
}
```

In Go, `strings.Fields()` serves the same purpose as Python's `split()` without arguments. It splits the string on whitespace and automatically removes extra spaces.

Since Go does not support Python-style slicing for reversal, we reverse the slice in-place using two pointers:

```
left, right := 0, len(words)-1
```

We swap elements until the pointers meet.

Finally, `strings.Join(words, " ")` rebuilds the string with exactly one space between words.

Unlike Python, Go strings are immutable, so we manipulate a slice of strings instead of directly modifying the original string.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We scan the string to split words, reverse the list, and join the result |
| Space | O(n) | We store extracted words in an additional list |

The runtime is linear because every character is processed a constant number of times. Splitting scans the string once, reversing the word list takes linear time relative to the number of words, and joining also processes the characters once.

The space complexity is `O(n)` because we store all words separately before constructing the final output string.

## Edge Cases

### Single Word

A string containing only one word:

```
"hello"
```

can cause unnecessary logic errors if reversal assumes multiple elements exist.

Our implementation handles this naturally. Splitting produces:

```
["hello"]
```

Reversing a single-element list changes nothing, and joining returns the same word.

### Multiple Consecutive Spaces

Inputs like:

```
"a    good     example"
```

can break implementations that split only on a single space character.

For example:

```
s.split(" ")
```

would produce empty strings between repeated spaces.

Using:

```
s.split()
```

or:

```
strings.Fields()
```

avoids this problem because consecutive whitespace is treated as a single separator.

### Leading and Trailing Spaces

Strings such as:

```
"   hello world   "
```

can accidentally produce extra spaces in the output if not handled carefully.

Since whitespace splitting removes unnecessary spaces automatically, our implementation never includes leading or trailing spaces in the final answer.

### Mixed Alphanumeric Words

The input may contain digits:

```
"abc 123 xyz"
```

The problem defines a word as any sequence of non-space characters, so numbers are treated exactly like letters.

Our implementation does not rely on character type checks, which ensures all valid words are processed correctly.
