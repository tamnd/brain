## Solution

## Problem Understanding

The problem gives us three inputs:

- A sorted array `nums` containing unique integers
- A lower bound `lower`
- An upper bound `upper`

Every value in `nums` lies within the inclusive range `[lower, upper]`.

We need to find all numbers inside `[lower, upper]` that do not appear in `nums`. Instead of returning every missing number individually, we compress consecutive missing numbers into ranges.

Each returned range is represented as `[start, end]`.

For example:

- If only the number `2` is missing, the range is `[2,2]`
- If every number from `4` to `49` is missing, the range is `[4,49]`

The output must satisfy two conditions:

1. Every missing number is covered exactly once
2. No number from `nums` appears inside any returned range

The input array is already sorted and contains unique values. This is extremely important because it allows us to detect gaps efficiently by comparing neighboring values.

The constraints are relatively small:

- `nums.length <= 100`
- Values can be as small as `-10^9`
- Values can be as large as `10^9`

Although the array size is small enough that even inefficient approaches could pass, the problem is fundamentally about interval detection and gap processing. The intended solution uses a clean linear scan.

Several edge cases are important:

- `nums` may be empty
- There may be no missing numbers at all
- Missing ranges may appear before the first element
- Missing ranges may appear after the last element
- Consecutive values in `nums` create no missing range
- A missing range may contain exactly one number

A correct implementation must carefully handle all of these situations.

## Approaches

### Brute Force Approach

The brute force idea is straightforward. We iterate through every number from `lower` to `upper`, checking whether that number exists in `nums`.

To make membership checks efficient, we can place all values from `nums` into a hash set.

As we scan through the interval:

- If a number is missing, we start building a missing range
- If numbers continue to be missing consecutively, we extend the range
- Once we encounter a present number, we finalize the current missing range

This approach is correct because it explicitly examines every value in the interval and identifies all missing segments.

However, it becomes inefficient when the numeric range is large. Even though `nums.length` is at most `100`, the interval `[lower, upper]` can span billions of integers. Iterating over every integer is unnecessary.

### Optimal Approach

The key observation is that we do not need to examine every integer individually.

Since `nums` is sorted and unique, every missing range must occur between two consecutive known values.

For example:

```
nums = [0,1,3,50,75]
```

The missing ranges are determined by the gaps:

- Between `1` and `3`, number `2` is missing
- Between `3` and `50`, numbers `4` through `49` are missing
- Between `50` and `75`, numbers `51` through `74` are missing

We also must consider:

- The gap before the first element
- The gap after the last element

We can solve the problem with a single linear scan by tracking the smallest candidate value that could still be missing.

At each number:

- If there is a gap between the current candidate and the current number, that gap forms a missing range
- Then we move the candidate forward

This avoids iterating through huge numeric intervals and gives an efficient linear solution.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(upper - lower + 1) | O(n) | Checks every integer individually |
| Optimal | O(n) | O(1) excluding output | Detects gaps directly using sorted order |

## Algorithm Walkthrough

1. Initialize an empty result list to store missing ranges.
2. Create a variable called `next_missing` and set it to `lower`. This variable represents the next number that could potentially be missing.
3. Iterate through each number in `nums`.
4. For each number `num`, compare it with `next_missing`.
5. If `num` is greater than `next_missing`, then all values from `next_missing` to `num - 1` are missing. Add the range `[next_missing, num - 1]` to the result.
6. Move `next_missing` to `num + 1`. This works because `num` itself is present in the array and therefore cannot be missing.
7. Continue processing every element in `nums`.
8. After the loop finishes, there may still be a missing range after the final array element.
9. If `next_missing <= upper`, then all values from `next_missing` to `upper` are missing. Add this final range.
10. Return the completed result list.

This algorithm works because every possible missing number belongs to exactly one gap:

- Before the first element
- Between two elements
- After the last element

The sorted property guarantees that these gaps can be processed independently and in order.

## Python Solution

```
from typing import List

class Solution:
    def findMissingRanges(self, nums: List[int], lower: int, upper: int) -> List[List[int]]:
        missing_ranges = []
        
        next_missing = lower
        
        for num in nums:
            if num > next_missing:
                missing_ranges.append([next_missing, num - 1])
            
            next_missing = num + 1
        
        if next_missing <= upper:
            missing_ranges.append([next_missing, upper])
        
        return missing_ranges
```

The implementation starts by initializing the result list `missing_ranges`.

The variable `next_missing` tracks the smallest number that has not yet been accounted for. Initially, this is simply `lower`.

As we iterate through `nums`, we compare the current number with `next_missing`.

If the current number is larger than `next_missing`, there is a gap. That entire gap forms a missing range, from `next_missing` through `num - 1`.

After processing the gap, we update `next_missing` to `num + 1` because the current number exists in the array and therefore is not missing.

Once all numbers have been processed, we may still have remaining missing values after the final array element. The final `if` statement handles this trailing range.

The algorithm processes each array element exactly once and requires only constant extra space.

## Go Solution

```
func findMissingRanges(nums []int, lower int, upper int) [][]int {
    missingRanges := [][]int{}
    
    nextMissing := lower

    for _, num := range nums {
        if num > nextMissing {
            missingRanges = append(missingRanges, []int{nextMissing, num - 1})
        }

        nextMissing = num + 1
    }

    if nextMissing <= upper {
        missingRanges = append(missingRanges, []int{nextMissing, upper})
    }

    return missingRanges
}
```

The Go implementation follows the same logic as the Python solution.

Instead of Python lists, Go uses slices of slices, `[][]int`, to represent the collection of ranges.

The `append` function is used to add new ranges dynamically.

Go integers are safe for this problem because the constraints stay within the standard 32-bit integer range, and operations such as `num + 1` remain valid.

An empty result is naturally represented by an empty slice.

## Edge Cases

### Empty Input Array

If `nums` is empty, then every number in `[lower, upper]` is missing.

Example:

```
nums = []
lower = 1
upper = 5
```

The correct result is:

```
[[1,5]]
```

This case can easily break implementations that assume at least one array element exists. Our solution handles it naturally because the loop never executes, and the final check adds the entire range.

### No Missing Numbers

If every number in the interval exists in `nums`, the answer should be an empty list.

Example:

```
nums = [1,2,3]
lower = 1
upper = 3
```

There are no gaps anywhere.

The implementation correctly avoids adding ranges because the condition `num > next_missing` never becomes true, and the final check also fails.

### Single Missing Number

A missing range may contain exactly one number.

Example:

```
nums = [1,3]
lower = 1
upper = 3
```

The missing value is only `2`, so the answer must be:

```
[[2,2]]
```

Some incorrect implementations try to represent single values differently from ranges. This solution avoids special-case logic because every missing interval is uniformly represented as `[start, end]`.

### Missing Range Before the First Element

Example:

```
nums = [5,6]
lower = 1
upper = 10
```

Numbers `1` through `4` are missing before the first array element.

The algorithm handles this because `next_missing` starts at `lower`. When the first number `5` is processed, the gap `[1,4]` is detected immediately.

### Missing Range After the Last Element

Example:

```
nums = [1,2]
lower = 1
upper = 5
```

Numbers `3` through `5` are missing after the final array element.

This is why the final post-processing check is necessary. Without it, trailing missing ranges would never be added.
