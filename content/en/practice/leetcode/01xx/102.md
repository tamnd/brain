## Solution

## Problem Understanding

The problem asks us to traverse a binary tree level by level and return the values grouped by depth. This traversal order is commonly called "level order traversal" or "breadth-first traversal".

A binary tree consists of nodes where each node may have a left child and a right child. The input `root` represents the root node of the tree. If the tree is empty, `root` is `None` in Python or `nil` in Go.

The expected output is a two-dimensional array where:

- Each inner array represents one level of the tree.
- Values inside a level appear from left to right.
- Levels appear from top to bottom.

For example, consider this tree:

```
        3
       / \
      9  20
         / \
        15  7
```

The correct traversal is:

```
[
  [3],
  [9, 20],
  [15, 7]
]
```

The first level contains only the root. The second level contains the root's children. The third level contains the children of node `20`.

The constraints tell us that the tree contains at most 2000 nodes. This is small enough for standard traversal algorithms such as Breadth-First Search or Depth-First Search. Since every node must appear in the output exactly once, any correct solution will require at least linear time.

Several edge cases are important:

- The tree may be empty.
- The tree may contain only one node.
- The tree may be highly unbalanced, behaving almost like a linked list.
- Some nodes may have only one child.
- Node values may be negative.

A correct implementation must handle all of these cases without errors.

## Approaches

### Brute Force Approach

A brute-force approach would first compute the height of the tree, then process each level individually using a recursive helper function.

The algorithm would work like this:

1. Compute the maximum depth of the tree.
2. For every depth from `0` to `height - 1`, traverse the entire tree and collect nodes that belong to that depth.

This works because every level is explicitly searched and collected. However, it is inefficient because the tree is traversed repeatedly for every level.

For a skewed tree with height `n`, the algorithm may scan almost the entire tree for each level, resulting in quadratic time complexity.

### Optimal Approach

The key observation is that level order traversal naturally matches Breadth-First Search (BFS).

BFS explores nodes level by level using a queue:

- Start with the root in the queue.
- Remove nodes from the front of the queue.
- Add their children to the back of the queue.
- Process all nodes currently in the queue before moving to the next level.

The queue automatically preserves the correct left-to-right order within each level.

Since every node is inserted and removed exactly once, the traversal is efficient and linear.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(h) | Repeatedly traverses the tree for every level |
| Optimal | O(n) | O(n) | Uses BFS with a queue to process each node once |

## Algorithm Walkthrough

1. Initialize an empty result list that will store all levels.
2. Handle the empty tree case immediately. If the root is `None` or `nil`, return an empty list because there are no nodes to traverse.
3. Create a queue and place the root node inside it. The queue is the core data structure of BFS because it preserves traversal order level by level.
4. Start a loop that continues while the queue is not empty. Each iteration of this loop processes one entire level of the tree.
5. Determine the number of nodes currently in the queue. This number represents exactly how many nodes belong to the current level. This step is important because children added during processing belong to the next level and should not be processed immediately.
6. Create an empty list for the current level.
7. Process exactly `level_size` nodes:

1. Remove a node from the front of the queue.
2. Add its value to the current level list.
3. If the node has a left child, add it to the queue.
4. If the node has a right child, add it to the queue.
8. After processing all nodes for the current level, append the level list to the final result.
9. Continue until the queue becomes empty.
10. Return the completed result.

This approach guarantees that:

- Nodes are visited level by level.
- Left-to-right order is preserved.
- Every node is processed exactly once.

## Python Solution

```
from collections import deque
from typing import List, Optional

# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def levelOrder(self, root: Optional[TreeNode]) -> List[List[int]]:
        if not root:
            return []

        result = []
        queue = deque([root])

        while queue:
            level_size = len(queue)
            current_level = []

            for _ in range(level_size):
                node = queue.popleft()
                current_level.append(node.val)

                if node.left:
                    queue.append(node.left)

                if node.right:
                    queue.append(node.right)

            result.append(current_level)

        return result
```

The implementation begins by handling the empty tree case. Returning immediately prevents unnecessary processing and avoids queue operations on invalid input.

The `deque` data structure from Python's `collections` module is used because it provides efficient `popleft()` operations in O(1) time. Using a normal list would make removals from the front inefficient.

The queue is initialized with the root node. During each iteration of the outer `while` loop, the algorithm processes one complete level.

The variable `level_size` captures the number of nodes currently in the queue before processing begins. This is critical because the queue grows as child nodes are added. Without fixing the level size beforehand, nodes from different levels could become mixed together.

Inside the `for` loop:

- A node is removed from the queue.
- Its value is added to the current level list.
- Its children are appended to the queue for future processing.

After all nodes at the current level are processed, the completed level list is appended to the final result.

When the queue becomes empty, all nodes have been visited and the result is returned.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func levelOrder(root *TreeNode) [][]int {
    if root == nil {
        return [][]int{}
    }

    result := [][]int{}
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        levelSize := len(queue)
        currentLevel := []int{}

        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]

            currentLevel = append(currentLevel, node.Val)

            if node.Left != nil {
                queue = append(queue, node.Left)
            }

            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }

        result = append(result, currentLevel)
    }

    return result
}
```

The Go implementation follows the same BFS logic as the Python version.

Since Go does not have a built-in deque structure in the standard library, a slice is used as the queue. Removing the first element is done with:

```
queue = queue[1:]
```

The implementation carefully checks for `nil` children before appending them to the queue.

Unlike Python, Go distinguishes between `nil` pointers and empty slices explicitly. Returning `[][]int{}` ensures the result is an empty two-dimensional slice instead of `nil`.

Integer overflow is not a concern because node values are constrained between `-1000` and `1000`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every node is visited exactly once |
| Space | O(n) | The queue may store an entire level of the tree |

The time complexity is linear because each node enters the queue once and leaves the queue once. No node is revisited.

The space complexity is O(n) in the worst case. For a complete binary tree, the last level may contain roughly half of all nodes, causing the queue to temporarily hold O(n) nodes.

## Edge Cases

### Empty Tree

An empty tree is represented by `root = None` in Python or `root = nil` in Go. This case can easily cause errors if the algorithm assumes the root always exists.

The implementation handles this immediately at the start by returning an empty list before initializing the queue.

### Single Node Tree

A tree containing only one node is the smallest non-empty valid input.

Example:

```
    1
```

The algorithm correctly creates one level containing only the root value:

```
[[1]]
```

The queue processes exactly one node, and no children are added.

### Highly Skewed Tree

A skewed tree behaves like a linked list.

Example:

```
1
 \
  2
   \
    3
```

Each level contains exactly one node:

```
[[1], [2], [3]]
```

Some incorrect implementations accidentally mix levels together when the queue grows during iteration. Using `level_size` ensures nodes added during processing belong only to the next level.

### Nodes With Missing Children

Some nodes may have only one child.

Example:

```
    1
   /
  2
```

The implementation checks whether children exist before adding them to the queue:

```
if node.left:
    queue.append(node.left)
```

This prevents `None` or `nil` values from entering the queue and causing runtime errors later.
