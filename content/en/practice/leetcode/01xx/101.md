## Solution

## Problem Understanding

This problem asks us to determine whether a binary tree is **symmetric around its center**, meaning the left and right halves of the tree are mirror images of each other.

A binary tree is symmetric if, for every node on the left side of the root, there is a corresponding node on the right side with the **same value** and a **mirrored structure**. In other words, the left subtree should look exactly like the reflection of the right subtree.

The input is the `root` node of a binary tree. Each node contains an integer value and references to a left child and a right child. The expected output is a boolean:

- Return `true` if the tree is symmetric.
- Return `false` if it is not symmetric.

To better understand the requirement, consider this symmetric tree:

```
        1
      /   \
     2     2
    / \   / \
   3   4 4   3
```

The left subtree mirrors the right subtree:

- `2` matches `2`
- `3` matches `3`
- `4` matches `4`

Now consider this non-symmetric tree:

```
        1
      /   \
     2     2
      \     \
       3     3
```

Although the values may seem balanced, the structure is not mirrored. The left subtree has a right child, while the corresponding mirrored position on the right subtree should be a left child.

The constraints tell us that the number of nodes is at most `1000`. This is relatively small, which means an `O(n)` traversal is easily efficient enough. Since every node may need to be examined, we should expect that a linear solution is optimal.

Several edge cases are important to consider:

- A tree with only one node is symmetric by definition.
- Trees with missing children can easily break symmetry if mirrored positions do not match.
- Trees where values differ at mirrored positions are not symmetric even if the structure matches.
- Empty child references (`null` or `nil`) must be compared carefully because symmetry depends on both structure and value.

## Approaches

### Brute Force Approach

A brute force way to solve this problem is to explicitly create a **mirror copy** of one subtree and compare it with the other subtree.

The idea works as follows:

1. Generate the mirrored version of the left subtree.
2. Compare the mirrored tree with the right subtree node by node.
3. Return `true` if they are identical.

This approach is correct because symmetry is equivalent to saying:

> Left subtree = mirror(right subtree)

However, this method is inefficient because it requires constructing an entirely new mirrored tree, which uses additional memory and unnecessary traversal work. We end up traversing the tree multiple times, once to build the mirror and once again to compare.

Since the tree can already be traversed directly while checking symmetry, building an extra mirrored tree is wasteful.

### Optimal Approach

The key observation is that we do **not need to build a mirrored tree**. Instead, we can directly compare nodes in mirrored positions.

For two nodes to be mirror images:

1. Their values must be equal.
2. The left child of one node must match the right child of the other.
3. The right child of one node must match the left child of the other.

This naturally leads to a **recursive depth-first search** solution.

We define a helper function:

```
isMirror(left, right)
```

This function checks whether two nodes are mirror images.

At each step:

- If both nodes are `null`, they match.
- If exactly one node is `null`, symmetry fails.
- If values differ, symmetry fails.
- Otherwise, recursively compare:

- `left.left` with `right.right`
- `left.right` with `right.left`

The follow-up asks for an iterative solution as well. This can be implemented using a queue, where mirrored node pairs are processed level by level. The recursive version is simpler and more elegant, so we will implement that as the main solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Builds a mirrored tree and compares it |
| Optimal | O(n) | O(h) | Recursively compares mirrored nodes directly |

Here, `n` is the number of nodes and `h` is the height of the tree.

## Algorithm Walkthrough

The optimal recursive algorithm works by comparing mirrored node pairs.

1. **Start from the root’s children**

A tree is symmetric if its left subtree mirrors its right subtree. We begin by comparing:

```
root.left and root.right
```
2. **Handle the base case where both nodes are empty**

If both nodes are `null`, they are symmetric at this position because neither subtree contains a node.
3. **Handle mismatched emptiness**

If one node exists and the other does not, symmetry breaks immediately because the structure is different.
4. **Compare node values**

If both nodes exist but their values differ, the tree cannot be symmetric.
5. **Recursively compare mirrored children**

This is the most important step.

Instead of comparing corresponding children:

```
left.left with right.left
```

We compare mirrored positions:

```
left.left with right.right
left.right with right.left
```

This preserves mirror symmetry.
6. **Combine the recursive results**

Both mirrored comparisons must succeed for the current pair to be symmetric.
7. **Return the final result**

If every mirrored pair satisfies the symmetry condition, return `true`. Otherwise, return `false`.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def isSymmetric(self, root: Optional[TreeNode]) -> bool:
        def is_mirror(
            left_node: Optional[TreeNode],
            right_node: Optional[TreeNode]
        ) -> bool:
            if left_node is None and right_node is None:
                return True

            if left_node is None or right_node is None:
                return False

            if left_node.val != right_node.val:
                return False

            return (
                is_mirror(left_node.left, right_node.right)
                and is_mirror(left_node.right, right_node.left)
            )

        return is_mirror(root.left, root.right)
```

The implementation uses a nested helper function called `is_mirror()` to compare two nodes.

The first condition checks whether both nodes are `None`. If so, they are symmetric at this position.

The second condition detects structural mismatches. If only one node exists, symmetry is broken.

The third condition compares node values. Even if the structure matches, different values invalidate symmetry.

Finally, the recursive step compares mirrored children:

- Left child of the left subtree with right child of the right subtree
- Right child of the left subtree with left child of the right subtree

The use of recursion makes the implementation concise and closely aligned with the problem definition.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func isSymmetric(root *TreeNode) bool {
	var isMirror func(leftNode, rightNode *TreeNode) bool

	isMirror = func(leftNode, rightNode *TreeNode) bool {
		if leftNode == nil && rightNode == nil {
			return true
		}

		if leftNode == nil || rightNode == nil {
			return false
		}

		if leftNode.Val != rightNode.Val {
			return false
		}

		return isMirror(leftNode.Left, rightNode.Right) &&
			isMirror(leftNode.Right, rightNode.Left)
	}

	return isMirror(root.Left, root.Right)
}
```

The Go implementation follows the exact same logic as the Python solution. The main difference is that Go uses `nil` instead of `None` and explicitly declares the recursive helper function using a function variable.

Since node values are constrained to `[-100, 100]`, integer overflow is not a concern. Go pointers naturally represent absent child nodes through `nil`, making structural comparisons straightforward.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every node is visited at most once |
| Space | O(h) | Recursive call stack depends on tree height |

The algorithm performs a single traversal of the tree. Each node participates in one mirrored comparison, leading to linear time complexity.

The space complexity comes from recursion. In a balanced tree, the recursion depth is `O(log n)`. In the worst case, such as a skewed tree, the recursion depth becomes `O(n)`.

## Edge Cases

### Single Node Tree

A tree containing only one node is symmetric because there are no subtrees that could violate symmetry.

```
    1
```

The algorithm handles this naturally because both `root.left` and `root.right` are `None`, causing the helper function to immediately return `True`.

### Missing Child on One Side

This is one of the easiest cases to mishandle.

```
    1
   / \
  2   2
 /     
3
```

The mirrored position on the right side is missing. A naive implementation that only compares values may mistakenly accept this tree.

Our implementation explicitly checks:

```
if left_node is None or right_node is None
```

This guarantees structural mismatches are caught.

### Matching Structure but Different Values

Even if the tree shape is perfectly mirrored, differing values invalidate symmetry.

```
        1
      /   \
     2     2
    /       \
   3         4
```

The structure is mirrored, but `3 != 4`.

The algorithm detects this using:

```
if left_node.val != right_node.val
```

and immediately returns `False`.

### Deeply Nested Symmetric Tree

A tree may look symmetric at the top but fail deeper down.

```
        1
      /   \
     2     2
    / \   / \
   3   5 4   3
```

The mismatch occurs deeper in the tree where `5` does not mirror `4`.

Because the algorithm recursively checks every mirrored pair, deeper violations are reliably detected.
