## Solution

## Problem Understanding

The problem gives us the head of a singly linked list where all values are already sorted in ascending order. Our goal is to convert this linked list into a height balanced binary search tree.

A binary search tree, commonly called a BST, has the property that every node's left subtree contains smaller values and every node's right subtree contains larger values. Since the linked list is already sorted, the middle element naturally becomes a good root candidate because all elements before it are smaller and all elements after it are larger.

The tree must also be height balanced. A height balanced tree is one where the height difference between the left and right subtree of every node is at most one. This requirement prevents extremely skewed trees and ensures efficient operations.

The input is a singly linked list, which means we can only move forward through the list. Unlike arrays, we cannot directly access the middle element in constant time. This detail heavily influences the design of the algorithm.

The constraints allow up to `2 * 10^4` nodes. That size is large enough that inefficient repeated traversals can become expensive, so we should aim for an algorithm that avoids unnecessary work.

Several edge cases are important:

- The list may be empty, in which case we should return `None`.
- The list may contain only one node, which should become a single tree node.
- The list may contain two nodes, where careful subtree construction is needed to maintain balance.
- The linked list may contain negative numbers, but BST logic works the same regardless of value sign.
- Because the list is singly linked, splitting the list incorrectly can easily cause cycles or disconnected sublists.

## Approaches

### Brute Force Approach

A straightforward solution is to repeatedly find the middle node of the linked list and use it as the root of the current subtree.

The idea works as follows:

1. Use the slow and fast pointer technique to locate the middle node.
2. Create a tree node from the middle value.
3. Disconnect the left half of the linked list from the middle.
4. Recursively build the left subtree from the left half.
5. Recursively build the right subtree from the nodes after the middle.

This approach produces a valid height balanced BST because the middle element is chosen at every recursive step.

However, the problem is efficiency. Finding the middle node requires traversing part of the list every time. Since this happens recursively for many sublists, the total cost becomes `O(n log n)` in the average case.

While acceptable for the constraints, we can do better.

### Optimal Approach

The key insight is that the linked list is already sorted in the exact order needed for an inorder traversal of a BST.

For a BST:

- Inorder traversal of the left subtree gives smaller values
- Then comes the root
- Then comes the inorder traversal of the right subtree

This order exactly matches the sorted linked list.

Instead of repeatedly searching for middle nodes, we can simulate inorder tree construction while advancing through the linked list only once.

The algorithm works like this:

1. Count the total number of nodes in the linked list.
2. Recursively construct the BST using index boundaries.
3. Build the left subtree first.
4. Use the current linked list node as the root.
5. Advance the linked list pointer.
6. Build the right subtree.

Because every linked list node is visited exactly once, the algorithm runs in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(log n) | Repeatedly searches for middle nodes |
| Optimal | O(n) | O(log n) | Simulates inorder traversal using one pass through the list |

## Algorithm Walkthrough

1. First, traverse the linked list once to determine its length. We need the size so we know how many nodes belong to each subtree during recursive construction.
2. Store the head pointer in a shared variable. This pointer will move forward exactly once during the recursive process.
3. Define a recursive helper function that constructs a BST for a given index range `[left, right]`.
4. If `left > right`, there are no elements left for this subtree, so return `None`.
5. Compute the middle index using:

```
mid = (left + right) // 2
```

This middle position represents the root of the current subtree.
6. Recursively build the left subtree first. This is important because inorder traversal processes nodes in sorted order, and the linked list pointer must advance correctly.
7. After the left subtree is built, the current linked list node corresponds exactly to the root value for this subtree.
8. Create a tree node using the current linked list node's value.
9. Move the linked list pointer forward to the next node.
10. Recursively build the right subtree.
11. Attach the left and right subtrees to the root node and return the completed subtree.

Because the recursion mirrors inorder traversal, the linked list values are consumed in sorted order without ever needing random access.

## Python Solution

```
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def sortedListToBST(self, head: Optional[ListNode]) -> Optional[TreeNode]:
        def get_length(node: Optional[ListNode]) -> int:
            length = 0
            
            while node:
                length += 1
                node = node.next
            
            return length
        
        size = get_length(head)
        self.current = head
        
        def build_bst(left: int, right: int) -> Optional[TreeNode]:
            if left > right:
                return None
            
            mid = (left + right) // 2
            
            left_subtree = build_bst(left, mid - 1)
            
            root = TreeNode(self.current.val)
            root.left = left_subtree
            
            self.current = self.current.next
            
            root.right = build_bst(mid + 1, right)
            
            return root
        
        return build_bst(0, size - 1)
```

The implementation starts by calculating the length of the linked list. This allows us to treat the list conceptually like an indexed structure even though we still traverse it sequentially.

The variable `self.current` acts as a moving pointer through the linked list. During recursive construction, this pointer always refers to the node that should become the next BST node in inorder sequence.

The recursive function `build_bst` receives the current index range. It first constructs the left subtree because inorder traversal visits left children before the root.

Once the left subtree is complete, the current linked list node becomes the root. We then advance the pointer and recursively build the right subtree.

This design guarantees that every list node is processed exactly once.

## Go Solution

```
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */

/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

func sortedListToBST(head *ListNode) *TreeNode {
    size := getLength(head)
    current := head

    var buildBST func(left int, right int) *TreeNode

    buildBST = func(left int, right int) *TreeNode {
        if left > right {
            return nil
        }

        mid := (left + right) / 2

        leftSubtree := buildBST(left, mid-1)

        root := &TreeNode{
            Val: current.Val,
        }

        root.Left = leftSubtree

        current = current.Next

        root.Right = buildBST(mid+1, right)

        return root
    }

    return buildBST(0, size-1)
}

func getLength(node *ListNode) int {
    length := 0

    for node != nil {
        length++
        node = node.Next
    }

    return length
}
```

The Go implementation follows the same inorder simulation strategy as the Python version.

Instead of using a class variable like `self.current`, Go uses a closure over the `current` pointer. Because closures capture variables by reference, recursive calls can correctly update the shared linked list pointer.

Go uses `nil` instead of `None`, and tree nodes are allocated with address syntax using `&TreeNode{}`.

## Worked Examples

### Example 1

Input:

```
[-10, -3, 0, 5, 9]
```

The linked list length is 5.

We begin with:

```
buildBST(0, 4)
```

| Step | Range | Mid | Current List Node | Action |
| --- | --- | --- | --- | --- |
| 1 | (0,4) | 2 | -10 | Build left subtree |
| 2 | (0,1) | 0 | -10 | Build left subtree |
| 3 | (0,-1) | - | -10 | Return None |
| 4 | Root for (0,1) | 0 | -10 | Create node -10 |
| 5 | (1,1) | 1 | -3 | Build right subtree |
| 6 | Root for (1,1) | 1 | -3 | Create node -3 |
| 7 | Root for (0,4) | 2 | 0 | Create node 0 |
| 8 | (3,4) | 3 | 5 | Build right subtree |
| 9 | Root for (3,4) | 3 | 5 | Create node 5 |
| 10 | (4,4) | 4 | 9 | Create node 9 |

Final BST:

```
        0
      /   \
    -3     9
    /     /
 -10     5
```

This tree is height balanced and satisfies BST ordering.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every linked list node is visited exactly once |
| Space | O(log n) | Recursive call stack height for a balanced BST |

The time complexity is linear because the algorithm never rescans the linked list. Each node is consumed exactly once during inorder construction.

The space complexity comes from recursion depth. Since the constructed BST is balanced, the recursion depth remains logarithmic relative to the number of nodes.

## Edge Cases

### Empty Linked List

An empty list is the smallest possible input. A naive implementation might attempt to access `head.val` and crash with a null reference error.

This implementation safely handles the case because the calculated size becomes zero, and the recursive helper immediately returns `None` when `left > right`.

### Single Node List

For a list containing one value, the BST should contain exactly one node with no children.

The recursion correctly computes a single midpoint, creates one tree node, and both recursive subtree calls return `None`.

### Two Node List

Two node lists are subtle because there are multiple valid balanced BST shapes.

For example:

```
[1, 2]
```

The algorithm chooses the lower midpoint as the root during recursion. One valid result is:

```
    1
     \
      2
```

This structure remains height balanced because the subtree height difference is at most one.

### Large Linked Lists

With up to `2 * 10^4` nodes, inefficient repeated traversals can become expensive.

The inorder simulation avoids repeated middle searches and guarantees linear performance, making it efficient even for the largest allowed inputs.

### Negative Values

The linked list may contain negative numbers such as:

```
[-10, -3, 0, 5]
```

BST ordering logic depends only on relative ordering, not absolute value size, so negative values are handled naturally without any special cases.
