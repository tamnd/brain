## Solution

## Problem Understanding

This problem asks us to populate the `next` pointer for every node in a **perfect binary tree** so that each node points to its immediate neighbor on the same level. If a node is the rightmost node in its level, its `next` pointer should remain `NULL`.

A perfect binary tree has two important guarantees:

1. Every internal node has exactly **two children**.
2. All leaf nodes exist at the **same depth**.

These guarantees are critical because they allow us to connect nodes efficiently without additional data structures.

The input is the root of a binary tree where every node contains:

```
val   -> node value
left  -> left child
right -> right child
next  -> pointer to the next node on the same level
```

Initially, every `next` pointer is `NULL`. Our task is to modify the tree in place and return the root after establishing all the correct horizontal connections.

For example, given this tree:

```
        1
      /   \
     2     3
    / \   / \
   4   5 6   7
```

The desired connections become:

```
1 -> NULL
2 -> 3 -> NULL
4 -> 5 -> 6 -> 7 -> NULL
```

The constraints tell us the tree can contain up to `2^12 - 1 = 4095` nodes, which is relatively small. However, the follow-up requirement is more important than the raw size:

> Only constant extra space may be used.

This means solutions using queues for level-order traversal, while acceptable for correctness, are not considered optimal for the follow-up.

Several edge cases are worth considering immediately. An empty tree (`root = []`) should simply return `None`. A tree containing only one node should leave that node’s `next` pointer as `NULL`. Since the tree is guaranteed to be perfect, we do not need to handle irregular structures where children are missing.

## Approaches

### Brute Force Approach, Breadth-First Search Using a Queue

A straightforward solution is to perform a **level-order traversal** using a queue.

At each level, we process nodes one by one. While iterating through a level, we connect each node to the next node in the queue for that same level. The final node in the level gets `NULL`.

This works because BFS naturally groups nodes level by level, making it easy to establish horizontal links.

For example:

```
Level 0: [1]
Level 1: [2,3]
Level 2: [4,5,6,7]
```

We simply connect adjacent nodes within each level.

This approach is correct because every node is visited exactly once, and each level is processed independently. However, it uses a queue that can hold an entire level of nodes, requiring extra memory proportional to the width of the tree.

Since the follow-up explicitly requests constant extra space, this solution is not optimal.

### Key Insight for the Optimal Solution

The key observation is that **the tree is perfect**.

Because every node has both a left and right child, and every level is complete, we can use already-established `next` pointers to move horizontally through a level without a queue.

For any node:

1. Its left child should always connect to its right child.
2. Its right child should connect to the left child of its next sibling.

That means:

```
node.left.next = node.right
```

and

```
node.right.next = node.next.left
```

if `node.next` exists.

For example:

```
        1
      /   \
     2 ---> 3
    / \   / \
   4->5->6->7
```

Node `2` connects:

```
4 -> 5
```

Then, because `2.next = 3`:

```
5 -> 6
```

This observation allows us to connect nodes level by level while using only pointers, achieving constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | BFS using a queue for level traversal |
| Optimal | O(n) | O(1) | Uses existing `next` pointers to traverse levels |

## Algorithm Walkthrough

The optimal algorithm proceeds level by level using already-established `next` pointers.

1. **Handle the empty tree case.**

If `root` is `None`, return immediately because there are no nodes to connect.
2. **Start at the root level.**

Since the tree is perfect, if a node has a left child, it must also have a right child. We initialize a pointer called `leftmost` to track the first node of the current level.
3. **Traverse the current level horizontally.**

Use another pointer called `current` to move through the level using `next` pointers.
4. **Connect sibling nodes.**

For every node:

```
current.left.next = current.right
```

This connects the left child directly to the right child.
5. **Connect across parent boundaries.**

If the current node has a neighbor:

```
current.right.next = current.next.left
```

This bridges two adjacent subtrees.
6. **Move horizontally through the level.**

Advance:

```
current = current.next
```

Because previous levels are already connected, horizontal traversal becomes possible without extra memory.
7. **Move to the next level.**

After finishing a level:

```
leftmost = leftmost.left
```

Since the tree is perfect, the leftmost node always exists until we reach the leaves.
8. **Stop at leaf level.**

Leaf nodes have no children, so no more connections are necessary.

## Python Solution

```
"""
# Definition for a Node.
class Node:
    def __init__(
        self,
        val: int = 0,
        left: 'Node' = None,
        right: 'Node' = None,
        next: 'Node' = None
    ):
        self.val = val
        self.left = left
        self.right = right
        self.next = next
"""

from typing import Optional

class Solution:
    def connect(self, root: 'Optional[Node]') -> 'Optional[Node]':
        if not root:
            return root

        leftmost = root

        while leftmost.left:
            current = leftmost

            while current:
                current.left.next = current.right

                if current.next:
                    current.right.next = current.next.left

                current = current.next

            leftmost = leftmost.left

        return root
```

The implementation begins by handling the empty tree case. If `root` is `None`, we immediately return.

The variable `leftmost` tracks the first node at each level. Since the tree is perfect, moving to the next level simply means following the left child.

Inside the outer loop, `current` traverses horizontally using `next` pointers. For every node, we first connect its left child to its right child. Then, if a neighboring parent exists, we connect the right child to that neighbor’s left child.

Once all nodes in a level are processed, we move downward to the next level and repeat the process.

The algorithm modifies the tree in place and finally returns the original root.

## Go Solution

```
/**
 * Definition for a Node.
 * type Node struct {
 *     Val int
 *     Left *Node
 *     Right *Node
 *     Next *Node
 * }
 */

func connect(root *Node) *Node {
	if root == nil {
		return root
	}

	leftmost := root

	for leftmost.Left != nil {
		current := leftmost

		for current != nil {
			current.Left.Next = current.Right

			if current.Next != nil {
				current.Right.Next = current.Next.Left
			}

			current = current.Next
		}

		leftmost = leftmost.Left
	}

	return root
}
```

The Go implementation follows the exact same logic as the Python version. The main difference is handling `nil` instead of `None`.

Pointer manipulation is explicit in Go, so we directly access fields such as `current.Left.Next`. Since the problem constraints guarantee a perfect binary tree, we do not need defensive checks for missing children inside the loop.

There are no integer overflow concerns because no arithmetic is performed. The solution also avoids slices or queues, maintaining constant extra space.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every node is visited exactly once |
| Space | O(1) | Only a few pointer variables are used |

The time complexity is **O(n)** because each node participates in a constant amount of work. Every node is visited once while establishing connections.

The space complexity is **O(1)** because the algorithm does not allocate auxiliary data structures like queues or hash maps. Only a few pointers are maintained regardless of tree size. Although recursion would also be acceptable under the follow-up, this iterative solution avoids even recursive stack usage.

## Edge Cases

### Empty Tree

An empty input tree is the simplest edge case:

```
root = []
```

A naive implementation might try to access `root.left`, causing a null reference error. Our implementation explicitly checks:

```
if not root:
    return root
```

This guarantees safe execution.

### Single Node Tree

Consider:

```
    1
```

There are no neighboring nodes to connect.

Since `leftmost.left` does not exist, the outer loop never runs. The root is returned unchanged, with:

```
1 -> NULL
```

This behavior is correct.

### Rightmost Node at Each Level

The last node of every level should point to `NULL`.

For example:

```
2 -> 3 -> NULL
```

A buggy implementation might accidentally connect the final node to an invalid child. Our implementation prevents this with:

```
if current.next:
    current.right.next = current.next.left
```

When `current.next` is `None`, no cross-level connection is made, leaving the pointer correctly set to `NULL`.

### Deep Perfect Trees

For taller trees, repeatedly scanning each level must remain efficient.

Because we traverse horizontally using `next` pointers instead of repeatedly searching or rebuilding level structures, the algorithm remains linear even for the maximum constraint size. The perfect-tree guarantee ensures every traversal path is predictable and safe.
