## Solution

## Problem Understanding

This problem asks us to convert an Excel-style column title into its corresponding numerical column index.

Excel labels its columns using uppercase English letters:

```
A -> 1
B -> 2
...
Z -> 26
AA -> 27
AB -> 28
```

At first glance, this resembles a standard base-26 numbering system, but there is one important difference. In normal base-26 systems, digits range from `0` to `25`. In Excel column notation, the characters range from `A` to `Z`, representing values from `1` to `26`.

The input is a string called `columnTitle`, containing only uppercase English letters. The output should be an integer representing the corresponding Excel column number.

For example:

- `"A"` maps to `1`
- `"Z"` maps to `26`
- `"AA"` maps to `27`
- `"ZY"` maps to `701`

The constraints tell us that the input length is at most `7`, which means the input is extremely small. Even inefficient solutions would likely pass. However, the problem is designed to test understanding of positional numeral systems and string processing.

An important observation is that each character contributes to the final value according to its position, just like digits in decimal numbers:

```
ABC
= 1 * 26² + 2 * 26¹ + 3 * 26⁰
```

where:

```
A = 1
B = 2
C = 3
```

There are several edge cases worth considering:

- A single-character column such as `"A"` or `"Z"`
- Multi-character columns where carry-over occurs, such as `"AA"`
- Very large valid inputs such as `"FXSHRXW"`, which is the maximum allowed value
- Inputs consisting entirely of `'Z'`, since these maximize digit values

The problem guarantees that the input contains only valid uppercase letters, so we do not need to handle invalid characters or empty strings.

## Approaches

### Brute Force Approach

A brute-force strategy would simulate the entire Excel column generation process. We could start from `"A"` and repeatedly generate the next column label until we reach the target string, counting how many steps were taken.

For example:

```
A -> B -> C -> ... -> Z -> AA -> AB -> ...
```

This approach works because it directly models how Excel columns are ordered. Once the generated string matches the target, the current count is the answer.

However, this method is extremely inefficient. The number of generated strings grows very quickly. For large column titles, such as `"FXSHRXW"`, generating every intermediate column would be prohibitively slow.

Even though the input size is small, the numerical value represented by the column title can be very large. Therefore, brute force is not practical.

### Optimal Approach

The optimal solution treats the column title as a modified base-26 number.

The key insight is that each character contributes a positional value:

```
A = 1
B = 2
...
Z = 26
```

As we process each character from left to right:

1. Multiply the current result by `26`
2. Add the value of the current character

This is identical to how decimal numbers are processed:

```
result = result * 10 + digit
```

except here the base is `26`.

For example:

```
"AB"

Start: result = 0

'A':
result = 0 * 26 + 1 = 1

'B':
result = 1 * 26 + 2 = 28
```

This approach is efficient, simple, and mathematically correct.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N) where N is the target column number | O(1) | Simulates Excel column generation until the target is reached |
| Optimal | O(L) | O(1) | Processes each character once using base-26 conversion |

Here, `L` is the length of the string.

## Algorithm Walkthrough

1. Initialize a variable called `result` to `0`.

This variable will store the running numerical value of the column title as we process characters from left to right.
2. Iterate through each character in `columnTitle`.

We process characters in order because earlier characters represent higher positional values, just like digits in decimal notation.
3. Convert the current character into its alphabetical value.

We use:

```
ord(character) - ord('A') + 1
```

This produces:

```
A -> 1
B -> 2
...
Z -> 26
```
4. Multiply the current result by `26`.

This shifts all previously processed characters one position to the left in the base-26 number system.
5. Add the current character value to the result.

This incorporates the contribution of the current character into the total.
6. Continue until all characters are processed.
7. Return the final result.

This algorithm uses only a single integer variable and processes the string once, making it both time and space efficient.

## Python Solution

```
class Solution:
    def titleToNumber(self, columnTitle: str) -> int:
        result = 0

        for character in columnTitle:
            value = ord(character) - ord('A') + 1
            result = result * 26 + value

        return result
```

The implementation begins by initializing `result` to `0`. This variable acts as the accumulated numerical representation of the Excel column title.

The loop iterates through every character in the string. For each character, we calculate its alphabet position using ASCII arithmetic. Since `ord('A')` gives the ASCII value of `'A'`, subtracting it from the current character and adding `1` produces the correct mapping from `A` through `Z`.

Next, we multiply the existing result by `26`. This shifts previously processed digits to their correct positional weight, exactly like multiplying by `10` when processing decimal numbers.

Finally, we add the current character's value and continue processing until the string ends.

The algorithm is concise because the mathematical relationship naturally maps to iterative accumulation.

## Go Solution

```
func titleToNumber(columnTitle string) int {
    result := 0

    for _, character := range columnTitle {
        value := int(character-'A') + 1
        result = result*26 + value
    }

    return result
}
```

The Go implementation follows the exact same logic as the Python version.

One small difference is that Go iterates over strings using `range`, which returns Unicode code points as `rune` values. Since the input only contains uppercase English letters, subtracting `'A'` works correctly.

Go also requires explicit integer conversion:

```
int(character-'A')
```

Unlike Python, Go uses statically typed integers, but the problem constraints guarantee the result fits safely within a standard integer type.

## Edge Cases

### Single Character Input

A column title such as `"A"` or `"Z"` is the smallest possible input size. These cases can expose bugs where implementations incorrectly assume multiple characters exist.

The algorithm handles this naturally because the loop runs exactly once, producing the correct numerical value immediately.

### Multiple Carry Positions

Inputs such as `"AA"` or `"ZZ"` test whether the positional multiplication is implemented correctly.

For example:

```
"AA"
= 1 × 26 + 1
= 27
```

If the multiplication step is omitted or placed incorrectly, these cases fail. The implementation correctly multiplies the accumulated result by `26` before adding the next digit.

### Maximum Constraint Input

The largest allowed value is `"FXSHRXW"`.

This tests whether the implementation can correctly process long column titles and large resulting numbers.

Since the algorithm processes characters iteratively and uses only integer arithmetic, it handles the maximum input efficiently without additional memory usage.

### All High-Value Characters

Inputs like `"ZZZZZZZ"` maximize the contribution of each character.

These cases are useful for validating that character conversion works correctly for `'Z'`, which should map to `26`, not `25`.

The implementation explicitly adds `1` during conversion:

```
ord(character) - ord('A') + 1
```

which guarantees the correct mapping.
