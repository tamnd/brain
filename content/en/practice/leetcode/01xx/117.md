# LeetCode 117 - Populating Next Right Pointers in Each Node II

## Problem Understanding

This problem asks us to connect nodes at the same depth level in a binary tree using the `next` pointer that already exists inside each node.

Each node contains four fields:

```
val   -> node value
left  -> left child
right -> right child
next  -> pointer to the next node on the same level
```

The goal is to modify the tree so that every node's `next` pointer points to the node immediately to its right on the same level. If there is no node to the right, the `next` pointer should remain `NULL`.

Unlike LeetCode 116, this tree is not guaranteed to be perfect. Some nodes may have only one child, and levels may be uneven. That difference makes the problem more challenging because we cannot rely on simple left-to-right structural relationships.

For example, consider this tree:

```
        1
      /   \
     2     3
    / \     \
   4   5     7
```

After connecting the nodes level by level:

```
1 -> NULL
2 -> 3 -> NULL
4 -> 5 -> 7 -> NULL
```

The input is the root of the binary tree. The output is the same tree, but with correctly populated `next` pointers.

The constraints tell us the tree contains at most 6000 nodes, which is small enough for breadth-first traversal solutions. However, the follow-up requirement asks for constant extra space, which means we should avoid using a queue if possible.

Important edge cases include:

- An empty tree where `root` is `NULL`
- A tree with only one node
- Sparse trees where some children are missing
- Trees where nodes only have left children or only right children
- Levels with disconnected structural gaps that still need proper `next` connections

The problem guarantees the input is a valid binary tree, so we do not need to handle cycles or malformed structures.

## Approaches

### Brute Force Approach, Breadth-First Search Using a Queue

A straightforward solution is to perform level-order traversal using a queue.

We process the tree one level at a time. For every level:

1. Determine how many nodes belong to that level
2. Pop nodes from the queue one by one
3. Connect each node's `next` pointer to the following node in the queue for that level
4. Push children into the queue for the next level

This works because breadth-first traversal naturally visits nodes level by level from left to right.

The main drawback is the extra memory usage. In the worst case, the queue may contain an entire tree level, which requires `O(n)` space.

### Optimal Approach, Level Traversal Using Existing Next Pointers

The key observation is that once one level has been connected, we can traverse that level using the `next` pointers themselves.

Instead of using a queue, we build the next level while traversing the current level.

We maintain:

- A pointer to the current node in the current level
- A dummy node that helps build connections for the next level
- A tail pointer that always points to the end of the next level chain

As we move across the current level using `next`, we connect children into the next level chain.

This approach achieves constant extra space because it reuses the already established `next` pointers instead of storing nodes in a queue.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Uses BFS queue for level traversal |
| Optimal | O(n) | O(1) | Uses existing next pointers to traverse levels |

## Algorithm Walkthrough

### Optimal Constant Space Algorithm

1. Start with the root node as the current level's starting point.
2. Create a dummy node before processing each level. This dummy node acts as the temporary head of the next level.
3. Maintain a `tail` pointer that initially points to the dummy node. Every time we discover a child node, attach it to `tail.next`, then move `tail` forward.
4. Traverse the current level using the already established `next` pointers. This avoids needing a queue.
5. For each node in the current level:

- If the node has a left child, append it to the next level chain.
- If the node has a right child, append it to the next level chain.
6. After finishing the current level, the dummy node's `next` pointer contains the first node of the next level.
7. Move to the next level by setting the current pointer to `dummy.next`.
8. Repeat until there are no more levels.

This method works because each level is fully connected before we move downward to process the next one.

## Python Solution

```
"""
# Definition for a Node.
class Node:
    def __init__(self, val: int = 0, left: 'Node' = None, right: 'Node' = None, next: 'Node' = None):
        self.val = val
        self.left = left
        self.right = right
        self.next = next
"""

class Solution:
    def connect(self, root: 'Node') -> 'Node':
        if not root:
            return root

        current = root

        while current:
            dummy = Node(0)
            tail = dummy

            while current:
                if current.left:
                    tail.next = current.left
                    tail = tail.next

                if current.right:
                    tail.next = current.right
                    tail = tail.next

                current = current.next

            current = dummy.next

        return root
```

### Python Implementation Explanation

The algorithm begins by checking whether the tree is empty. If `root` is `None`, we immediately return because there is nothing to connect.

The variable `current` represents the current node in the current level.

Inside the outer loop, we create a fresh dummy node for constructing the next level's linked structure. The `tail` pointer tracks the last connected node in the next level.

The inner loop traverses the current level using `current.next`.

Whenever a left child exists, we append it to the next level chain:

```
tail.next = current.left
tail = tail.next
```

The same logic applies to the right child.

After processing both children, we move horizontally across the current level:

```
current = current.next
```

Once the level is complete, the next level starts at:

```
dummy.next
```

So we update:

```
current = dummy.next
```

Eventually, when there are no more levels, `current` becomes `None` and the loop terminates.

## Go Solution

```
/**
 * Definition for a Node.
 * type Node struct {
 *     Val int
 *     Left *Node
 *     Right *Node
 *     Next *Node
 * }
 */

func connect(root *Node) *Node {
	if root == nil {
		return root
	}

	current := root

	for current != nil {
		dummy := &Node{}
		tail := dummy

		for current != nil {
			if current.Left != nil {
				tail.Next = current.Left
				tail = tail.Next
			}

			if current.Right != nil {
				tail.Next = current.Right
				tail = tail.Next
			}

			current = current.Next
		}

		current = dummy.Next
	}

	return root
}
```

### Go Implementation Notes

The Go solution follows the same logic as the Python version.

The primary difference is Go's explicit pointer syntax. Instead of `None`, Go uses `nil`.

The dummy node is created using:

```
dummy := &Node{}
```

Pointer field access still uses dot notation in Go because the language automatically dereferences struct pointers.

No additional slices or queues are used, so the implementation satisfies the constant extra space requirement.

## Edge Cases

### Empty Tree

An empty tree is represented by `root = NULL`.

This case is important because attempting to traverse or access children would cause errors in many implementations.

The solution handles this immediately with:

```
if not root:
    return root
```

This guarantees safe execution.

### Single Node Tree

Consider:

```
1
```

There are no neighboring nodes at the same level.

The algorithm processes the root once, finds no children, and terminates correctly with:

```
1 -> NULL
```

No special-case logic is required because the general algorithm already handles it naturally.

### Sparse Tree with Missing Children

Consider:

```
        1
      /   \
     2     3
      \      \
       5      7
```

A naive implementation might incorrectly assume every node has both children.

The optimal solution explicitly checks:

```
if current.left:
```

and

```
if current.right:
```

before connecting nodes.

This ensures only valid children are linked, while still correctly connecting nodes across structural gaps:

```
5 -> 7
```

### Tree with Only Left Children

Consider:

```
1
|
2
|
3
```

Every level contains exactly one node.

The algorithm still works because each level traversal simply builds the next level with a single child node.

All `next` pointers remain `NULL`, which is correct.

### Tree with Only Right Children

Consider:

```
1
 \
  2
   \
    3
```

This is another skewed-tree scenario that can expose pointer traversal bugs.

Because the algorithm independently checks both left and right children, it correctly builds each next level even when only right children exist.
