# LeetCode 126 - Word Ladder II

**Difficulty:** Hard

**Topics:** Hash Table, String, Backtracking, Breadth-First Search

## Problem Understanding

This problem asks us to find **all shortest transformation sequences** from `beginWord` to `endWord`.

A transformation sequence is valid if every step changes **exactly one character**, and every intermediate word exists in the provided `wordList`. The `beginWord` itself does not need to be part of the dictionary, but every subsequent transformed word must exist in `wordList`.

The key requirement is that we are **not looking for just one valid path**, and we are **not looking for every possible path**. We specifically need **all shortest paths**.

For example:

```
hit -> hot -> dot -> dog -> cog
```

and

```
hit -> hot -> lot -> log -> cog
```

are both valid because each adjacent pair differs by one letter, every intermediate word exists in the dictionary, and both sequences have the minimum possible number of transformations.

The input consists of:

- `beginWord`: the starting word.
- `endWord`: the target word.
- `wordList`: a dictionary of allowed intermediate words.

The output should be a list of transformation sequences, where each sequence itself is a list of words.

The constraints tell us important things about the problem size:

- Word length is at most `5`, which means generating neighboring words by changing letters is relatively cheap.
- `wordList.length <= 500`, which is moderate in size.
- The total number of shortest sequences does not exceed `10^5`, meaning the output itself may become large.

A naive brute-force search through all possible transformations would quickly become infeasible because the number of paths grows exponentially. Since we specifically care about the **shortest** paths, this strongly suggests using **Breadth-First Search (BFS)** because BFS naturally explores nodes level by level and guarantees shortest distances in an unweighted graph.

Several edge cases matter immediately:

If `endWord` is not in `wordList`, there is no possible transformation, so the answer must be empty. Another tricky case is when multiple shortest paths converge into the same word. A naive visited strategy may accidentally remove valid shortest paths if it marks words too early. We must also avoid exploring paths longer than the shortest discovered solution.

## Approaches

### Brute Force Approach

A brute-force solution would try every possible transformation sequence using backtracking or DFS.

Starting from `beginWord`, we could recursively generate all words differing by one character and continue exploring until we either reach `endWord` or exhaust possibilities.

To avoid cycles, we would maintain a visited set for the current path.

This approach is correct because it eventually explores every valid transformation path. Once all paths are found, we could filter out only the shortest ones.

However, this method is extremely slow because the number of possible paths grows exponentially. Even with pruning, repeatedly exploring the same words across many branches leads to massive redundancy.

The core inefficiency is that DFS does not naturally prioritize shorter paths. We may spend significant time exploring deep paths before even discovering the shortest one.

### Optimal Approach

The key insight is that this problem can be viewed as an **unweighted graph shortest path problem**.

Each word represents a node. An edge exists between two words if they differ by exactly one letter.

Since all transformations have equal cost, **BFS guarantees shortest distances**.

However, a standard BFS that stops after finding `endWord` only gives one shortest path. We need **all shortest paths**.

To solve this, we use a two-phase strategy:

1. **BFS Phase**

Use BFS to build a graph of shortest transitions. Instead of storing only one parent for each word, we store **all possible parents** that reach the word at the shortest distance.
2. **DFS / Backtracking Phase**

Starting from `endWord`, reconstruct all shortest paths by traversing the parent relationships backward.

The subtle but important detail is how visited words are handled. During BFS, we delay removing visited words until an entire level has been processed. This ensures that multiple shortest routes arriving at the same word in the same BFS layer are preserved.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Exponential | O(N) | Explores all possible paths |
| Optimal | O(N × L × 26 + K) | O(N + K) | BFS for shortest graph, DFS for reconstruction |

Where:

- `N` = number of words
- `L` = word length
- `K` = total size of returned paths

## Algorithm Walkthrough

1. **Convert `wordList` into a hash set**

We place all dictionary words into a set for O(1) lookups. This allows us to efficiently check whether a transformed word exists.
2. **Check whether `endWord` exists**

If `endWord` is not present in the dictionary, immediately return `[]`. No valid transformation can ever reach it.
3. **Initialize BFS structures**

We use:

- A queue for BFS traversal
- A `parents` map to record shortest predecessors
- A `visited_this_level` set to delay removal of visited words
- A flag to stop BFS once the shortest level containing `endWord` is found

The `parents` map is crucial because a word may have **multiple shortest predecessors**.
4. **Run BFS level by level**

For every word in the current BFS layer:

- Try changing every character position.
- Replace the character with every letter from `'a'` to `'z'`.
- Generate a neighboring word.

If the generated word exists in the dictionary:

- Record the current word as a parent.
- Add the new word to the BFS queue.

We process the entire BFS level before removing words from the dictionary.
5. **Stop after shortest distance**

Once `endWord` is found in a BFS layer, we continue processing that layer but do not go deeper.

This guarantees that only shortest paths are included.
6. **Backtrack using DFS**

Starting from `endWord`, recursively move through parent links until reaching `beginWord`.

Since parents are stored in reverse direction, we build paths backward and reverse them at the end.
7. **Return all reconstructed paths**

Every reconstructed path represents a shortest transformation sequence.

## Python Solution

```
from collections import defaultdict, deque
from typing import List

class Solution:
    def findLadders(
        self,
        beginWord: str,
        endWord: str,
        wordList: List[str]
    ) -> List[List[str]]:

        word_set = set(wordList)

        if endWord not in word_set:
            return []

        parents = defaultdict(list)
        queue = deque([beginWord])

        found = False

        while queue and not found:
            level_visited = set()

            for _ in range(len(queue)):
                current_word = queue.popleft()
                word_chars = list(current_word)

                for i in range(len(word_chars)):
                    original_char = word_chars[i]

                    for char_code in range(26):
                        new_char = chr(ord('a') + char_code)

                        if new_char == original_char:
                            continue

                        word_chars[i] = new_char
                        next_word = ''.join(word_chars)

                        if next_word in word_set:
                            if next_word not in level_visited:
                                level_visited.add(next_word)
                                queue.append(next_word)

                            parents[next_word].append(current_word)

                            if next_word == endWord:
                                found = True

                    word_chars[i] = original_char

            word_set -= level_visited

        if not found:
            return []

        result = []
        path = [endWord]

        def backtrack(word: str) -> None:
            if word == beginWord:
                result.append(path[::-1])
                return

            for parent in parents[word]:
                path.append(parent)
                backtrack(parent)
                path.pop()

        backtrack(endWord)

        return result
```

### Implementation Explanation

The first step converts `wordList` into a set called `word_set`. This gives constant-time membership checks, which is critical because we generate many candidate words during BFS.

We immediately check whether `endWord` exists. If not, there is no solution.

The `parents` dictionary stores reverse edges. For example:

```
dog <- dot
log <- lot
cog <- dog
cog <- log
```

This reverse relationship allows us to reconstruct all shortest paths later.

The BFS loop processes one level at a time. We use `level_visited` to delay removal of words until the current level is finished. This is essential because multiple shortest paths may reach the same word in the same layer.

For every word, we try changing each character position into every lowercase letter. Since the word length is small, this operation is efficient.

Once `endWord` is found, we still finish the current level but stop afterward. This ensures all shortest parents are captured.

The DFS backtracking begins at `endWord` and recursively walks backward to `beginWord`. Since the path is built in reverse order, we reverse it before storing the result.

## Go Solution

```
package main

func findLadders(beginWord string, endWord string, wordList []string) [][]string {
	wordSet := make(map[string]bool)

	for _, word := range wordList {
		wordSet[word] = true
	}

	if !wordSet[endWord] {
		return [][]string{}
	}

	parents := make(map[string][]string)
	queue := []string{beginWord}
	found := false

	for len(queue) > 0 && !found {
		levelVisited := make(map[string]bool)
		levelSize := len(queue)

		for i := 0; i < levelSize; i++ {
			currentWord := queue[0]
			queue = queue[1:]

			wordChars := []byte(currentWord)

			for pos := 0; pos < len(wordChars); pos++ {
				originalChar := wordChars[pos]

				for ch := byte('a'); ch <= 'z'; ch++ {
					if ch == originalChar {
						continue
					}

					wordChars[pos] = ch
					nextWord := string(wordChars)

					if wordSet[nextWord] {
						if !levelVisited[nextWord] {
							levelVisited[nextWord] = true
							queue = append(queue, nextWord)
						}

						parents[nextWord] = append(
							parents[nextWord],
							currentWord,
						)

						if nextWord == endWord {
							found = true
						}
					}
				}

				wordChars[pos] = originalChar
			}
		}

		for word := range levelVisited {
			delete(wordSet, word)
		}
	}

	if !found {
		return [][]string{}
	}

	var result [][]string
	path := []string{endWord}

	var backtrack func(string)

	backtrack = func(word string) {
		if word == beginWord {
			reversed := make([]string, len(path))

			for i := range path {
				reversed[i] = path[len(path)-1-i]
			}

			result = append(result, reversed)
			return
		}

		for _, parent := range parents[word] {
			path = append(path, parent)
			backtrack(parent)
			path = path[:len(path)-1]
		}
	}

	backtrack(endWord)

	return result
}
```

### Go-Specific Implementation Notes

The Go solution mirrors the Python logic closely.

Instead of Python sets, Go uses `map[string]bool` to simulate set behavior.

Character mutation is done using `[]byte` rather than strings because Go strings are immutable.

When constructing final paths, we explicitly create a reversed copy since slices reference underlying arrays.

Go returns an empty slice `[][]string{}` when no solution exists, matching LeetCode expectations.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(N × L × 26 + K) | BFS explores words, generates neighbors, and DFS outputs paths |
| Space | O(N + K) | Parent graph, queue, visited states, and result storage |

Here:

- `N` is the number of words
- `L` is word length
- `K` is the total size of all returned shortest paths

During BFS, every word can be processed once. For each word, we try changing every character position (`L`) into all `26` letters. Therefore BFS costs `O(N × L × 26)`.

The DFS reconstruction cost depends on the total number of shortest paths returned. Since the problem guarantees the sum of shortest sequences does not exceed `10^5`, reconstruction remains manageable.

## Edge Cases

### `endWord` Does Not Exist

This is the most important early exit case.

Example:

```
beginWord = "hit"
endWord = "cog"
wordList = ["hot","dot"]
```

A buggy implementation might still attempt BFS unnecessarily. Our implementation immediately checks membership in `word_set` and returns `[]`.

### Multiple Shortest Paths to the Same Word

Example:

```
hit -> hot -> dot
hit -> hot -> lot
dot -> dog
lot -> dog
```

The word `"dog"` has multiple shortest predecessors.

A naive visited strategy that removes words immediately would only keep one parent. Our level-based visitation ensures all shortest parents are preserved.

### No Possible Transformation

Example:

```
beginWord = "aaa"
endWord = "bbb"
wordList = ["aac", "acc", "ccc"]
```

Even though `endWord` exists hypothetically, there may be no path connecting the words.

Our BFS eventually exhausts the queue without finding `endWord`, and the function correctly returns an empty list.

### `beginWord` Not in `wordList`

The problem explicitly states this may happen.

Example:

```
beginWord = "hit"
wordList = ["hot","dot"]
```

A buggy solution might assume the start word exists in the dictionary.

Our implementation begins BFS directly from `beginWord`, independent of whether it exists in `wordList`, so this case works naturally.
