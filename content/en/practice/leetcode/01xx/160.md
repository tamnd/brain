## Solution

## Problem Understanding

This problem asks us to determine whether two singly linked lists intersect, and if they do, return the exact node where the intersection begins.

The important detail is that intersection is defined by reference equality, not by node value equality. Two nodes intersect only if they are literally the same node in memory. This means that even if two nodes contain the same value, they are not considered intersecting unless both linked lists point to the exact same node object.

We are given two head pointers, `headA` and `headB`, representing two singly linked lists. The task is to return the first shared node between the lists. If the lists never share a node, we return `null` in Python or `nil` in Go.

Because the lists are singly linked and acyclic, once two lists intersect, every node afterward must also be shared. In other words, the lists form a Y-shaped structure after the intersection point.

The constraints tell us several important things:

- Each list can contain up to `3 * 10^4` nodes.
- A quadratic solution may still technically pass, but it is inefficient and not ideal.
- The follow-up explicitly asks for `O(m + n)` time and `O(1)` extra space.
- The lists must retain their original structure, so we cannot modify node connections.

Several edge cases are important:

- The lists may not intersect at all.
- One list may be significantly longer than the other.
- The intersection might occur at the head node itself.
- The intersection might occur at the last node only.
- The lists may contain identical values but no shared node references.

A naive implementation that compares node values instead of node references would produce incorrect results.

## Approaches

### Brute Force Approach

The most direct approach is to compare every node in list A with every node in list B.

We start from the first node in list A and traverse the entirety of list B to check whether any node reference matches. If no match is found, we move to the next node in list A and repeat the process.

This approach is correct because it exhaustively checks every possible node pair. If an intersection exists, eventually both pointers will point to the same node object.

However, this method is inefficient. If list A has `m` nodes and list B has `n` nodes, we perform up to `m * n` comparisons.

For the maximum constraints, this becomes unnecessarily expensive.

### Optimal Two Pointer Approach

The key insight is that the only difference between the two traversals is the unequal prefix length before the intersection point.

Suppose:

- List A length = `m`
- List B length = `n`
- Shared tail length = `k`

Then:

- Unique part of A = `m - k`
- Unique part of B = `n - k`

If we allow each pointer to traverse both lists completely, the total distance traveled becomes equal.

The trick is:

- Pointer `pointerA` starts at `headA`
- Pointer `pointerB` starts at `headB`
- When a pointer reaches the end of its list, redirect it to the other list's head

Eventually:

- If an intersection exists, the pointers meet at the intersection node
- If no intersection exists, both pointers become `null` simultaneously

This works because both pointers travel exactly `m + n` steps.

The algorithm achieves:

- `O(m + n)` time
- `O(1)` extra space

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m × n) | O(1) | Compare every node in A against every node in B |
| Optimal Two Pointers | O(m + n) | O(1) | Equalizes traversal distance by switching heads |

## Algorithm Walkthrough

1. Initialize two pointers, `pointerA` and `pointerB`, at the heads of the two lists.
2. Traverse both lists simultaneously.
3. At each iteration:

- Move each pointer to its next node.
- If a pointer reaches the end of a list, redirect it to the head of the opposite list.
4. Continue this process until the two pointers become equal.
5. There are only two possible outcomes:

- The pointers meet at the intersection node.
- Both pointers become `null`, meaning no intersection exists.
6. Return the node where the pointers meet.

The reason this works is that both pointers traverse the exact same total distance:

- Pointer A travels: `lengthA + lengthB`
- Pointer B travels: `lengthB + lengthA`

After the path lengths equalize, the pointers align at the intersection point.

## Python Solution

```
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, x):
#         self.val = x
#         self.next = None

from typing import Optional

class Solution:
    def getIntersectionNode(
        self,
        headA: ListNode,
        headB: ListNode
    ) -> Optional[ListNode]:

        pointerA = headA
        pointerB = headB

        while pointerA != pointerB:
            if pointerA is None:
                pointerA = headB
            else:
                pointerA = pointerA.next

            if pointerB is None:
                pointerB = headA
            else:
                pointerB = pointerB.next

        return pointerA
```

The implementation starts by assigning two pointers to the heads of the lists.

The loop continues until both pointers reference the same node. Importantly, the comparison checks object identity, not node values.

Whenever a pointer reaches the end of its current list, it is redirected to the other list's head. This equalizes traversal distances without explicitly calculating list lengths.

If the lists intersect, both pointers eventually align at the shared node. If the lists do not intersect, both pointers eventually become `None` at the same time, terminating the loop.

The final returned value is either the intersection node or `None`.

## Go Solution

```
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    pointerA := headA
    pointerB := headB

    for pointerA != pointerB {
        if pointerA == nil {
            pointerA = headB
        } else {
            pointerA = pointerA.Next
        }

        if pointerB == nil {
            pointerB = headA
        } else {
            pointerB = pointerB.Next
        }
    }

    return pointerA
}
```

The Go implementation follows the same logic as the Python version.

The primary language-specific difference is the use of `nil` instead of `None`.

Pointer comparisons in Go naturally compare memory references, which is exactly what this problem requires. No additional equality logic is necessary.

Since Go uses explicit struct pointers, moving through the linked list uses the `Next` field.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(m + n) | Each pointer traverses both lists at most once |
| Space | O(1) | Only two pointers are used |

Although the pointers may switch lists, each pointer visits at most `m + n` nodes. No auxiliary data structures are used, so the space usage remains constant.

## Edge Cases

One important edge case occurs when the lists do not intersect at all. A buggy implementation might enter an infinite loop or incorrectly match nodes with equal values. In this solution, both pointers eventually become `null` after traversing both lists completely, which safely terminates the loop.

Another edge case is when the intersection occurs at the very first node. In this scenario, `headA` and `headB` already reference the same node object. The loop condition immediately fails, and the algorithm correctly returns the shared head node.

A third important case is when one list is much longer than the other. A naive simultaneous traversal would become misaligned and fail to detect the intersection. The pointer-switching strategy automatically compensates for the length difference by ensuring both pointers travel the same total distance.

Another subtle case is when lists contain identical values but different node objects. For example:

```
A: 1 -> 2 -> 3
B: 1 -> 2 -> 3
```

Even though the values match, these are separate nodes in memory. Since the algorithm compares node references rather than values, it correctly returns `null`.
