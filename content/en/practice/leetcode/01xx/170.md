## Solution

## Problem Understanding

The problem asks us to design a reusable data structure that continuously receives integers through an `add()` operation and later answers queries through a `find()` operation.

The `find(value)` method must determine whether there are any two numbers currently stored in the data structure whose sum equals `value`.

This is different from the classic Two Sum problem because the numbers are not given all at once. Instead, they arrive incrementally over time. The data structure must support interleaved calls such as:

1. Add a few numbers
2. Query for a target sum
3. Add more numbers
4. Query again

The important detail is that the structure must remember previously added numbers and efficiently answer future sum queries.

For example:

```
add(1)
add(3)
add(5)
find(4)
```

The query returns `true` because `1 + 3 = 4`.

However:

```
find(7)
```

returns `false` because no pair among `{1, 3, 5}` sums to `7`.

The constraints are relatively small, at most `10^4` operations total, but the goal is still to design an efficient solution. Since `find()` may be called many times, repeatedly scanning all possible pairs would become unnecessarily expensive.

Several edge cases are important:

- Duplicate numbers matter. If the target is `6`, and only one `3` exists, the answer should be `false`. If two `3`s exist, the answer should be `true`.
- Negative numbers are allowed, so we cannot rely on sorting assumptions involving only positive values.
- The same number may be added many times.
- Queries may occur before enough numbers exist to form a pair.

The problem guarantees valid integer inputs within 32-bit signed integer range, so standard integer arithmetic is sufficient in both Python and Go.

## Approaches

### Brute Force Approach

The most straightforward solution is to store all inserted numbers in a list.

Whenever `find(value)` is called, we examine every possible pair of numbers and check whether their sum equals the target value.

For example, if the stored numbers are:

```
[1, 3, 5]
```

then `find(4)` checks:

```
1 + 3
1 + 5
3 + 5
```

until either a valid pair is found or all pairs are exhausted.

This approach is correct because it explicitly evaluates every possible pair, so it cannot miss a valid answer.

However, the time complexity is poor. If there are `n` stored numbers, checking all pairs requires `O(n²)` time for every `find()` call. With many operations, this becomes inefficient.

### Optimal Approach

The key observation is that during a `find(value)` query, for every number `x`, we only need to know whether `value - x` also exists.

This naturally suggests using a hash map.

We store:

```
number -> frequency
```

Then during `find(value)`:

- Iterate through each distinct number `x`
- Compute the complement:

```
complement = value - x
```

- If the complement exists, we may have a valid pair

There is one special case:

If:

```
x == complement
```

then we need at least two occurrences of that number.

For example:

```
find(6)
```

requires two `3`s, not just one.

Using a hash map gives constant time lookups, making the solution much more efficient.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | `O(n²)` per `find()` | `O(n)` | Checks every pair explicitly |
| Optimal | `O(1)` add, `O(n)` find | `O(n)` | Uses hash map frequencies for fast complement lookup |

## Algorithm Walkthrough

### Optimal Hash Map Algorithm

1. Initialize a hash map called `counts`.

The map stores how many times each number has been added.

Example:

```
{
    1: 1,
    3: 2,
    5: 1
}
```
2. When `add(number)` is called, increment the frequency of that number.

This allows us to efficiently handle duplicates.
3. When `find(value)` is called, iterate through every distinct number in the hash map.
4. For each number `current`, compute:

```
complement = value - current
```

This is the number needed to complete the target sum.
5. Check whether the complement exists in the hash map.

If it does not exist, continue searching.
6. If:

```
current != complement
```

then a valid pair exists immediately.

Example:

```
current = 1
complement = 3
```
7. If:

```
current == complement
```

then we need at least two copies of that number.

Example:

```
value = 6
current = 3
complement = 3
```

We must verify:

```
counts[3] >= 2
```
8. If no valid pair is found after checking all numbers, return `false`.

This design is efficient because hash map lookups are constant time on average, allowing the algorithm to avoid checking every pair explicitly.

## Python Solution

```
class TwoSum:

    def __init__(self):
        self.counts: dict[int, int] = {}

    def add(self, number: int) -> None:
        self.counts[number] = self.counts.get(number, 0) + 1

    def find(self, value: int) -> bool:
        for number in self.counts:
            complement = value - number

            if complement not in self.counts:
                continue

            if complement != number:
                return True

            if self.counts[number] >= 2:
                return True

        return False

# Your TwoSum object will be instantiated and called as such:
# obj = TwoSum()
# obj.add(number)
# param_2 = obj.find(value)
```

The implementation uses a dictionary called `counts` to store frequencies of inserted numbers.

Inside `add()`, we increment the frequency count for the given number. Using `get(number, 0)` ensures that unseen numbers start with frequency zero.

Inside `find()`, we iterate through all distinct numbers currently stored. For each number, we calculate the complement needed to reach the target sum.

If the complement does not exist, we skip that number.

If the complement exists and is different from the current number, we immediately return `True`.

If the complement equals the current number, we verify that the frequency is at least two. This correctly handles cases like needing two `3`s to form `6`.

If no valid pair is found after the loop completes, we return `False`.

## Go Solution

```
type TwoSum struct {
	counts map[int]int
}

func Constructor() TwoSum {
	return TwoSum{
		counts: make(map[int]int),
	}
}

func (this *TwoSum) Add(number int) {
	this.counts[number]++
}

func (this *TwoSum) Find(value int) bool {
	for number := range this.counts {
		complement := value - number

		count, exists := this.counts[complement]
		if !exists {
			continue
		}

		if complement != number {
			return true
		}

		if count >= 2 {
			return true
		}
	}

	return false
}

/**
 * Your TwoSum object will be instantiated and called as such:
 * obj := Constructor();
 * obj.Add(number);
 * param_2 := obj.Find(value);
 */
```

The Go implementation follows the same logic as the Python solution but uses a Go map:

```
map[int]int
```

to store frequencies.

Go maps return both the stored value and a boolean indicating whether the key exists. This is used during complement lookup:

```
count, exists := this.counts[complement]
```

Unlike Python dictionaries, Go requires explicit initialization with `make(map[int]int)`.

Integer overflow is not a concern because the problem guarantees values fit within 32-bit signed integer range, and Go's `int` type safely handles these operations on modern platforms.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | `O(1)` add, `O(n)` find | Hash map insertion is constant time, `find()` scans distinct numbers |
| Space | `O(n)` | Hash map stores frequencies of inserted numbers |

The `add()` operation is extremely efficient because it only updates a hash map entry.

The `find()` operation iterates through all distinct stored numbers. For each number, complement lookup is constant time on average due to the hash map.

The space complexity is linear because every distinct inserted number may require its own entry in the frequency map.

## Edge Cases

### Duplicate Numbers Needed for the Same Value

Consider:

```
add(3)
find(6)
```

A naive implementation might incorrectly return `true` because `6 - 3 = 3`.

However, only one `3` exists, so no valid pair can be formed.

Our implementation correctly checks:

```
counts[3] >= 2
```

before returning `true`.

### Negative Numbers

Consider:

```
add(-2)
add(7)
find(5)
```

The valid pair is:

```
-2 + 7 = 5
```

Since the solution relies only on hash map lookups and arithmetic subtraction, negative numbers work naturally without requiring any special handling.

### Query Before Enough Numbers Exist

Consider:

```
add(10)
find(20)
```

Only one number exists, so no pair can be formed.

The algorithm checks:

```
complement = 10
```

but since the frequency of `10` is only `1`, it correctly returns `false`.

### Multiple Duplicate Insertions

Consider:

```
add(4)
add(4)
find(8)
```

Now two copies of `4` exist.

The algorithm detects:

```
current = 4
complement = 4
counts[4] = 2
```

and correctly returns `true`.
