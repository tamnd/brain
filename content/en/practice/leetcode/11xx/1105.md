**Topics:** Array, Dynamic Programming

## Solution

## Problem Understanding

This problem asks us to arrange books on shelves while preserving the original order of the books. Each book has two properties:

- `thickness`, which consumes horizontal shelf space
- `height`, which contributes to the shelf height

Each shelf has a maximum width, `shelfWidth`. We may place consecutive books on the same shelf as long as the sum of their thicknesses does not exceed this limit.

The important detail is how shelf height is calculated. The height contributed by a shelf is not the sum of the heights of its books. Instead, it is the maximum height among all books placed on that shelf.

Our goal is to partition the sequence of books into shelves so that the total height of all shelves is minimized.

The order constraint is critical. We cannot rearrange books. We can only decide where to break the sequence into shelves.

For example, suppose we have:

```
books = [[1,1],[2,3],[2,3]]
shelfWidth = 4
```

We could place:

- Book 1 and Book 2 on the first shelf
- Book 3 on the second shelf

Or:

- Book 1 alone
- Book 2 and Book 3 together

Different groupings produce different total heights.

The constraints are also important:

- `books.length <= 1000`
- `shelfWidth <= 1000`

A brute-force recursive search over all possible shelf partitions would become extremely expensive because each book boundary could either continue the current shelf or start a new one. This leads to exponential growth.

However, `n = 1000` is small enough for an `O(n^2)` dynamic programming solution.

There are several edge cases worth identifying early:

- A single book may exactly fill the shelf width.
- Every book may need its own shelf.
- All books may fit on one shelf.
- Heights can vary dramatically, so adding a short book to a tall shelf may effectively cost nothing.
- A naive greedy strategy can fail because minimizing the current shelf height does not necessarily minimize the total future height.

## Approaches

### Brute Force Approach

The brute-force solution tries every possible way to partition the books into shelves.

At each book index, we decide:

- Continue placing books on the current shelf if width allows
- Or start a new shelf

This recursively explores every valid shelf arrangement.

The approach is correct because it exhaustively checks all possible partitions and returns the minimum total height among them.

However, it is far too slow. The number of ways to partition a sequence grows exponentially. In the worst case, every boundary between books can either be a shelf break or not, leading to approximately `2^(n-1)` possibilities.

With up to 1000 books, this is infeasible.

### Dynamic Programming Insight

The key observation is that the problem has overlapping subproblems.

Suppose we define:

```
dp[i] = minimum total height needed for the first i books
```

To compute `dp[i]`, we consider all possible ways the final shelf could be formed.

We look backward from book `i`:

- Put only book `i` on the last shelf
- Put books `i-1` and `i` on the last shelf
- Put books `i-2`, `i-1`, and `i` on the last shelf
- Continue while total width stays within `shelfWidth`

For each possible final shelf, we compute:

```
height before this shelf + height of this shelf
```

This gives the recurrence:

```
dp[i] = min(dp[j] + maxHeightOfBooks(j+1 ... i))
```

where books `j+1` through `i` form the final shelf.

This transforms the exponential brute-force search into a polynomial-time dynamic programming solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Tries every shelf partition recursively |
| Optimal Dynamic Programming | O(n^2) | O(n) | Builds solutions incrementally using previous states |

## Algorithm Walkthrough

### Step 1: Define the DP State

Create a DP array where:

```
dp[i] = minimum height needed to place the first i books
```

This means:

- `dp[0] = 0`, no books require zero height
- `dp[n]` will be the final answer

We use 1-based indexing for convenience in the DP array.

### Step 2: Process Books One by One

For each position `i`, we determine the best possible final shelf arrangement ending at book `i`.

Initially, assume the current book starts a new shelf by itself.

Then:

```
dp[i] = dp[i-1] + currentBookHeight
```

This is always valid.

### Step 3: Extend the Current Shelf Backward

Now try placing earlier books on the same shelf.

Move backward from book `i` toward the beginning:

- Keep adding thickness
- Track the maximum height on the shelf
- Stop once width exceeds `shelfWidth`

For every valid grouping:

```
books[j ... i]
```

compute:

```
candidate = dp[j-1] + currentShelfHeight
```

Take the minimum over all candidates.

### Step 4: Maintain Running Shelf Information

While scanning backward:

- `currentWidth` stores total thickness
- `currentHeight` stores maximum height

These are updated incrementally instead of recomputing them repeatedly.

### Step 5: Return the Final Answer

After processing all books:

```
return dp[n]
```

This represents the minimum total bookshelf height.

### Why it works

The algorithm works because every valid arrangement must end with some final shelf configuration. For each book position `i`, we enumerate all possible valid final shelves ending at `i`. For each such shelf, the books before it form an independent subproblem already solved by `dp[j-1]`.

Since we consider every valid final shelf and always choose the minimum total height, the DP recurrence guarantees the globally optimal solution.

## Python Solution

```
from typing import List

class Solution:
    def minHeightShelves(self, books: List[List[int]], shelfWidth: int) -> int:
        n = len(books)

        # dp[i] = minimum height for first i books
        dp = [float("inf")] * (n + 1)
        dp[0] = 0

        for i in range(1, n + 1):
            current_width = 0
            current_height = 0

            # Try placing books[j:i] on the same shelf
            for j in range(i, 0, -1):
                thickness, height = books[j - 1]

                current_width += thickness

                if current_width > shelfWidth:
                    break

                current_height = max(current_height, height)

                dp[i] = min(
                    dp[i],
                    dp[j - 1] + current_height
                )

        return dp[n]
```

The implementation directly follows the dynamic programming strategy.

The DP array stores the minimum height required for progressively larger prefixes of the book list. Initially, all values are set to infinity because we want to minimize them later.

For each ending position `i`, the algorithm scans backward to determine which books can fit together on the final shelf.

`current_width` tracks total thickness while expanding the shelf backward. If the width exceeds `shelfWidth`, the loop immediately stops because adding even more books would only increase the width further.

`current_height` tracks the tallest book on the current shelf. Since shelf height equals the maximum book height, this value determines the shelf contribution.

At every valid backward extension, the algorithm combines:

- The optimal solution for earlier books, `dp[j - 1]`
- The height of the current shelf

The minimum of all such possibilities becomes `dp[i]`.

Finally, `dp[n]` contains the optimal total height for all books.

## Go Solution

```
func minHeightShelves(books [][]int, shelfWidth int) int {
	n := len(books)

	dp := make([]int, n+1)

	const INF = int(1e9)

	for i := 1; i <= n; i++ {
		dp[i] = INF
	}

	dp[0] = 0

	for i := 1; i <= n; i++ {
		currentWidth := 0
		currentHeight := 0

		for j := i; j >= 1; j-- {
			thickness := books[j-1][0]
			height := books[j-1][1]

			currentWidth += thickness

			if currentWidth > shelfWidth {
				break
			}

			if height > currentHeight {
				currentHeight = height
			}

			candidate := dp[j-1] + currentHeight

			if candidate < dp[i] {
				dp[i] = candidate
			}
		}
	}

	return dp[n]
}
```

The Go implementation mirrors the Python solution closely.

Since Go does not have built-in infinity values for integers, a large constant `INF` is used to initialize the DP array.

Slices are used instead of Python lists, and explicit comparisons replace Python's `max` and `min` helper behavior.

Integer overflow is not a concern here because the maximum possible answer is bounded by:

```
1000 books * 1000 height = 1,000,000
```

which easily fits inside Go's `int`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n^2) | For each book, we may scan backward through all previous books |
| Space | O(n) | DP array stores one value per prefix |

The outer loop runs once for each book. The inner loop scans backward until the shelf width is exceeded. In the worst case, all books fit on a shelf, causing the inner loop to examine all previous books.

Therefore, the total time complexity is quadratic.

The algorithm only stores the DP array and a few scalar variables, so the space complexity is linear.

## Edge Cases

### Case 1: A Single Book Exactly Matches Shelf Width

Consider:

```
books = [[5,10]]
shelfWidth = 5
```

This can expose off-by-one errors in width calculations. A buggy implementation might incorrectly reject shelves whose width equals the limit instead of only rejecting widths greater than the limit.

The implementation handles this correctly because it only stops when:

```
current_width > shelfWidth
```

Equal width remains valid.

### Case 2: Every Book Requires Its Own Shelf

Consider:

```
books = [[3,1],[3,2],[3,3]]
shelfWidth = 3
```

No two books can share a shelf.

This tests whether the DP correctly falls back to the simplest valid arrangement. The implementation always considers the option where the current book starts a new shelf:

```
dp[i] = dp[i-1] + current_height
```

So even when no grouping is possible, the algorithm still works correctly.

### Case 3: A Very Tall Book Mixed With Short Books

Consider:

```
books = [[1,1],[1,10],[1,1]]
```

A naive implementation might mistakenly add heights instead of taking the maximum shelf height.

The algorithm correctly tracks:

```
current_height = max(current_height, height)
```

This ensures that shelf contribution is always determined by the tallest book only.

### Case 4: Greedy Packing Produces the Wrong Answer

Suppose we always try to pack as many books as possible onto each shelf. That strategy can fail because placing one tall book earlier may increase shelf height unnecessarily.

Dynamic programming avoids this mistake by evaluating every valid final shelf arrangement and choosing the globally minimal total height rather than making local greedy decisions.
