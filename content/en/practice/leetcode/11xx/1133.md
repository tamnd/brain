## Solution

## Problem Understanding

The problem gives us an integer array `nums` and asks us to find the largest number that appears exactly once in the array. If every number appears more than once, we must return `-1`.

In other words, we need to examine the frequency of every integer in the array. Among all integers whose frequency is `1`, we choose the maximum value.

For example, in the array `[5,7,3,9,4,9,8,3,1]`, the number `9` is the largest value overall, but it appears twice, so it does not qualify. The numbers that appear exactly once are `5`, `7`, `4`, `8`, and `1`. Among them, `8` is the largest, so the answer is `8`.

The constraints are small:

- `1 <= nums.length <= 2000`
- `0 <= nums[i] <= 1000`

These constraints tell us several important things:

- The array is relatively small, so even less efficient approaches may still pass.
- Values are limited to the range `0` through `1000`, which means frequency counting is very manageable.
- Since duplicates matter, we need some way to count occurrences efficiently.

There are several important edge cases to consider:

- Every number may appear multiple times, in which case the answer should be `-1`.
- The array may contain only one element. Since it appears once, it must be returned.
- The largest value might not be unique, so we cannot simply return the maximum element.
- The answer could be `0`, since `0` is within the valid input range.

## Approaches

### Brute Force Approach

A straightforward solution is to examine every number and count how many times it appears by scanning the entire array repeatedly.

For each element `nums[i]`, we iterate through the whole array and count its occurrences. If the count is exactly `1`, we compare it against the current best answer and keep the larger value.

This works because every element is checked thoroughly, and we only consider numbers that occur once.

However, this approach is inefficient because counting occurrences for each element requires another full traversal of the array.

If the array length is `n`, then:

- The outer loop runs `n` times.
- The inner counting loop also runs `n` times.

This leads to `O(n^2)` time complexity.

Although the constraints are small enough that this would technically pass, it is unnecessarily slow compared to a frequency counting solution.

### Optimal Approach

The key observation is that we repeatedly compute the same frequency information in the brute-force approach.

Instead of recounting occurrences every time, we can compute all frequencies once using a hash map.

The algorithm becomes:

1. Count how many times every number appears.
2. Iterate through the frequency map.
3. Among numbers with frequency `1`, keep the maximum.

A hash map is ideal here because:

- It provides average `O(1)` insertion and lookup.
- It allows us to efficiently track occurrence counts.

This reduces the total time complexity to `O(n)`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Repeatedly scans the array to count occurrences |
| Optimal | O(n) | O(n) | Uses a hash map to store frequencies efficiently |

## Algorithm Walkthrough

1. Create a hash map called `frequency`.

This map will store each number as the key and the number of occurrences as the value.
2. Traverse the input array once.

For every number in `nums`, increment its count in the hash map.

For example, after processing `[5,7,3,9,4,9]`, the map becomes:

```
{
    5: 1,
    7: 1,
    3: 1,
    9: 2,
    4: 1
}
```
3. Initialize a variable `largest_unique` to `-1`.

This acts as the default result in case no unique number exists.
4. Iterate through the frequency map.

For each `(number, count)` pair:

- If `count == 1`, the number is unique.
- Compare it with `largest_unique`.
- Update the answer if the current number is larger.
5. Return `largest_unique`.

If no unique number was found, the value remains `-1`.

### Why it works

The algorithm works because the frequency map correctly records how many times every number appears in the array. A number is considered valid only if its frequency equals `1`. By checking every unique candidate and keeping the maximum, we guarantee that the returned value is the largest number that appears exactly once.

## Python Solution

```python
from typing import List
from collections import Counter

class Solution:
    def largestUniqueNumber(self, nums: List[int]) -> int:
        frequency = Counter(nums)

        largest_unique = -1

        for number, count in frequency.items():
            if count == 1:
                largest_unique = max(largest_unique, number)

        return largest_unique
```

The implementation begins by using `Counter` from Python's `collections` module. This automatically builds a frequency map where each key is a number and each value is the number of times that number appears.

Next, the variable `largest_unique` is initialized to `-1`. This ensures that if no unique number exists, the correct default value is returned.

The loop over `frequency.items()` examines every distinct number and its count. Whenever a number appears exactly once, the algorithm compares it against the current best answer and stores the larger value.

Finally, the method returns the computed answer.

## Go Solution

```go
func largestUniqueNumber(nums []int) int {
    frequency := make(map[int]int)

    for _, num := range nums {
        frequency[num]++
    }

    largestUnique := -1

    for num, count := range frequency {
        if count == 1 && num > largestUnique {
            largestUnique = num
        }
    }

    return largestUnique
}
```

The Go implementation follows the same logic as the Python solution. A `map[int]int` is used to store frequencies.

Go does not have a built in `Counter` type like Python, so frequencies are updated manually using:

```go
frequency[num]++
```

The solution does not require special handling for nil slices because the problem guarantees at least one element in the input array.

Integer overflow is also not a concern because all values are small and fit easily within Go's standard integer type.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | One pass to count frequencies and one pass to evaluate unique numbers |
| Space | O(n) | The hash map may store up to n distinct elements |

The algorithm performs a linear scan to build the frequency map and another linear scan across the distinct keys in the map. Since hash map operations are average `O(1)`, the total running time is `O(n)`.

The space complexity is `O(n)` in the worst case, when every element in the array is distinct and must be stored in the frequency map.

## Edge Cases

One important edge case occurs when every number appears more than once. In this scenario, there is no valid answer, so the algorithm must return `-1`. A buggy implementation might accidentally return the largest number regardless of frequency. Initializing the answer variable to `-1` ensures the correct behavior.

Another important case is when the array contains exactly one element. Since that element appears once, it must automatically be the answer. The frequency map correctly stores a count of `1`, and the algorithm returns the value without requiring any special handling.

A third edge case involves the number `0`. Because `0` is within the allowed input range, it can legitimately be the largest unique number if all larger numbers are duplicated. Some implementations mistakenly treat `0` as a falsy or invalid value. This solution compares frequencies explicitly and handles `0` exactly like any other integer.
