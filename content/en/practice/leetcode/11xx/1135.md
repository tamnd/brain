**Difficulty:** 🟡 Medium  
**Topics:** Union-Find, Graph Theory, Heap (Priority Queue), Minimum Spanning Tree

## Solution

## Problem Understanding

The problem is asking us to connect `n` cities with a set of bidirectional roads while minimizing the total cost. Each road has a cost associated with it, and we are given these as an array of triples `[xi, yi, costi]`. The goal is to select a subset of these roads such that all cities are connected (meaning there is a path between any two cities) while minimizing the total cost. If it is impossible to connect all cities due to missing connections, we return `-1`.

In other words, we are asked to find a **Minimum Spanning Tree (MST)** of the graph where cities are nodes and roads are weighted edges. The MST property ensures that all cities are connected with the smallest total weight. The constraints indicate that `n` and the number of connections can go up to 10,000, which means we need an efficient algorithm, ideally O(E log E) or O(E α(n)).

Important edge cases include:

- A graph with isolated cities (no way to connect all cities). The algorithm should return `-1`.
- A graph with multiple roads between the same pair of cities. Only the cheapest should be considered.
- The graph is already minimally connected (like a tree). The algorithm should recognize this and return the sum of all costs.

## Approaches

The **brute-force approach** would consider all possible subsets of edges that connect `n` cities and compute the total cost for each. We would then select the subset with the minimum cost. While correct, this approach is completely infeasible because the number of subsets grows exponentially, roughly `O(2^E)`, making it impractical for `E = 10^4`.

The **key insight** is that this is a classical **Minimum Spanning Tree (MST)** problem. Two efficient algorithms are commonly used: **Kruskal's algorithm** and **Prim's algorithm**. Kruskal's algorithm sorts edges by cost and incrementally adds them using a **Union-Find (Disjoint Set Union) structure** to avoid cycles. Prim's algorithm grows the MST from a starting node using a **priority queue** to select the next minimum edge. Both guarantee minimal total cost.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^E) | O(E) | Checks all subsets of edges, infeasible for large graphs |
| Kruskal MST | O(E log E + E α(n)) | O(n) | Sort edges and use Union-Find to avoid cycles, very efficient |
| Prim MST | O(E log n) | O(n + E) | Use priority queue to grow MST, also efficient |

We will implement **Kruskal’s algorithm** because it is simpler to code in both Python and Go for this problem, given the edge list input.

## Algorithm Walkthrough

1. **Sort all connections by cost in ascending order**. This ensures we always consider the smallest-cost edge first, which is necessary for Kruskal's algorithm to produce a MST.
2. **Initialize a Union-Find data structure** for the `n` cities. Each city initially is its own parent, representing disjoint sets.
3. **Iterate through the sorted edges**. For each edge connecting cities `u` and `v`:

- Check if `u` and `v` are already connected using the Union-Find structure. If they are, skip this edge to avoid forming a cycle.
- If they are not connected, merge their sets (union operation) and add the cost of this edge to the total cost.
4. **Track the number of edges added**. For `n` cities, we need exactly `n-1` edges in the MST. If at the end, we have fewer than `n-1` edges, return `-1`.
5. **Return the total cost**. If all cities are connected, the sum of the chosen edges represents the minimum cost.

**Why it works**: Kruskal’s algorithm guarantees that we always choose the smallest available edge that does not form a cycle. By definition, this produces a Minimum Spanning Tree, which is exactly the minimal total cost required to connect all nodes.

## Python Solution

```
from typing import List

class UnionFind:
    def __init__(self, n: int):
        self.parent = list(range(n + 1))
        self.rank = [0] * (n + 1)
    
    def find(self, x: int) -> int:
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    
    def union(self, x: int, y: int) -> bool:
        rootX = self.find(x)
        rootY = self.find(y)
        if rootX == rootY:
            return False
        if self.rank[rootX] > self.rank[rootY]:
            self.parent[rootY] = rootX
        elif self.rank[rootX] < self.rank[rootY]:
            self.parent[rootX] = rootY
        else:
            self.parent[rootY] = rootX
            self.rank[rootX] += 1
        return True

class Solution:
    def minimumCost(self, n: int, connections: List[List[int]]) -> int:
        connections.sort(key=lambda x: x[2])
        uf = UnionFind(n)
        total_cost = 0
        edges_used = 0
        
        for u, v, cost in connections:
            if uf.union(u, v):
                total_cost += cost
                edges_used += 1
                if edges_used == n - 1:
                    return total_cost
        
        return -1
```

**Implementation walkthrough**: The `UnionFind` class efficiently manages connectivity between cities, using path compression in `find` and union by rank in `union`. Sorting the connections ensures the cheapest edges are considered first. Each successful union adds an edge to the MST and accumulates the cost. If we reach `n-1` edges, the MST is complete.

## Go Solution

```
package main

import (
    "sort"
)

type UnionFind struct {
    parent []int
    rank   []int
}

func NewUnionFind(n int) *UnionFind {
    uf := &UnionFind{
        parent: make([]int, n+1),
        rank:   make([]int, n+1),
    }
    for i := 0; i <= n; i++ {
        uf.parent[i] = i
    }
    return uf
}

func (uf *UnionFind) Find(x int) int {
    if uf.parent[x] != x {
        uf.parent[x] = uf.Find(uf.parent[x])
    }
    return uf.parent[x]
}

func (uf *UnionFind) Union(x, y int) bool {
    rootX := uf.Find(x)
    rootY := uf.Find(y)
    if rootX == rootY {
        return false
    }
    if uf.rank[rootX] > uf.rank[rootY] {
        uf.parent[rootY] = rootX
    } else if uf.rank[rootX] < uf.rank[rootY] {
        uf.parent[rootX] = rootY
    } else {
        uf.parent[rootY] = rootX
        uf.rank[rootX]++
    }
    return true
}

func minimumCost(n int, connections [][]int) int {
    sort.Slice(connections, func(i, j int) bool {
        return connections[i][2] < connections[j][2]
    })

    uf := NewUnionFind(n)
    totalCost := 0
    edgesUsed := 0

    for _, conn := range connections {
        if uf.Union(conn[0], conn[1]) {
            totalCost += conn[2]
            edgesUsed++
            if edgesUsed == n-1 {
                return totalCost
            }
        }
    }

    return -1
}
```

**Go-specific notes**: We use slices for `parent` and `rank`. The `sort.Slice` function sorts the connections by cost. The union-find logic mirrors the Python implementation, with no nil concerns since slices are always initialized.
