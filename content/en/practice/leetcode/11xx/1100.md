## Solution

## Problem Understanding

This problem asks us to count all substrings of length `k` in a given string `s` that contain no repeated characters. A substring is a contiguous sequence of characters in a string. For example, in `"havefunonleetcode"`, the substring `"havef"` is of length 5, and all characters are unique, so it counts toward the result.

The input `s` is a string of lowercase English letters with a length up to 10,000, and `k` is an integer representing the desired substring length, also up to 10,000. The output is a single integer, the number of valid substrings with all unique characters.

Important points from the constraints include that `k` can be larger than `s.length`, in which case the answer must be `0` because it is impossible to form a substring longer than the string itself. Another key detail is that `s` contains only lowercase letters, meaning we can use a fixed-size array or hash map of length 26 to track character occurrences efficiently.

Edge cases that could trip up a naive solution include `k > len(s)`, `k = 1`, and strings where all characters are identical.

## Approaches

The **brute-force approach** is straightforward: generate every possible substring of length `k` and check whether it contains all unique characters. This can be done by using a set or a frequency map for each substring. While this approach is correct, it is inefficient because for each of the `O(n)` possible substrings of length `k`, checking uniqueness takes `O(k)` time, leading to an overall complexity of `O(n*k)`. This becomes slow for the upper constraint of `n = 10,000`.

The **optimal approach** uses a sliding window combined with a hash set or a frequency counter. We maintain a window of size `k` and track character frequencies as we move the window one character at a time. If all characters in the window are unique, we count the substring. This approach avoids recomputing uniqueness from scratch for each window, reducing the time complexity to `O(n)`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n*k) | O(k) | Check each substring independently using a set or map |
| Optimal | O(n) | O(26) ≈ O(1) | Sliding window with character frequency or set for uniqueness |

## Algorithm Walkthrough

1. Check if `k` is larger than the string length `n`. If yes, return `0` because no substring of length `k` exists.
2. Initialize a frequency counter (hash map or array of size 26) to track characters in the current window of length `k`.
3. Initialize two pointers, `left` and `right`, representing the sliding window.
4. Expand the window by moving the `right` pointer. Update the frequency counter for the new character.
5. If the window size exceeds `k`, move the `left` pointer forward and decrement the frequency of the removed character.
6. After updating the window, check if all character counts in the window are `1`. If yes, increment the result counter.
7. Continue sliding the window until the `right` pointer reaches the end of the string.
8. Return the final count of substrings with all unique characters.

**Why it works:** By maintaining a sliding window and a frequency map, we ensure that each substring of length `k` is checked efficiently for uniqueness. The invariant is that the window always contains exactly `k` characters, and the frequency map accurately reflects the count of each character in the window.

## Python Solution

```
class Solution:
    def numKLenSubstrNoRepeats(self, s: str, k: int) -> int:
        if k > len(s):
            return 0
        
        count = 0
        freq = [0] * 26  # frequency array for lowercase letters
        unique = 0  # number of unique characters in current window
        
        for i in range(len(s)):
            idx = ord(s[i]) - ord('a')
            freq[idx] += 1
            if freq[idx] == 1:
                unique += 1
            elif freq[idx] == 2:
                unique -= 1
            
            if i >= k:
                left_idx = ord(s[i - k]) - ord('a')
                freq[left_idx] -= 1
                if freq[left_idx] == 1:
                    unique += 1
                elif freq[left_idx] == 0:
                    unique -= 1
            
            if i >= k - 1 and unique == k:
                count += 1
        
        return count
```

This Python implementation initializes a frequency array for lowercase letters. As the window slides, it updates the frequency counts and tracks the number of unique characters. Once the window size reaches `k`, it checks whether all characters are unique and increments the result if so.

## Go Solution

```
func numKLenSubstrNoRepeats(s string, k int) int {
    n := len(s)
    if k > n {
        return 0
    }
    
    count := 0
    freq := [26]int{}
    unique := 0
    
    for i := 0; i < n; i++ {
        idx := s[i] - 'a'
        freq[idx]++
        if freq[idx] == 1 {
            unique++
        } else if freq[idx] == 2 {
            unique--
        }
        
        if i >= k {
            leftIdx := s[i-k] - 'a'
            freq[leftIdx]--
            if freq[leftIdx] == 1 {
                unique++
            } else if freq[leftIdx] == 0 {
                unique--
            }
        }
        
        if i >= k-1 && unique == k {
            count++
        }
    }
    
    return count
}
```

The Go solution mirrors the Python logic. Go uses fixed-size arrays for the frequency map and indexes characters using `s[i] - 'a'`. Edge cases such as `k > len(s)` are handled upfront. There is no need to worry about integer overflow due to the problem constraints.
