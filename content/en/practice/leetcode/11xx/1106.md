## Solution

## Problem Understanding

The problem is asking us to evaluate a boolean expression that is provided as a string. The expression can represent a simple boolean value `'t'` or `'f'`, or it can be a composite expression using logical operators: NOT `'!'`, AND `'&'`, and OR `'|'`. Each composite expression has parentheses around its inner expressions and may contain multiple sub-expressions separated by commas. The goal is to parse this string and return the final boolean result (`True` or `False`).

The input is guaranteed to be a valid expression, which means we do not need to handle malformed input. The expression length can be up to `2 * 10^4`, so efficiency matters. Nested expressions, such as `"!(&(t,f))"` or `"&(|(f),!(t))"`, introduce recursion naturally. Important edge cases include expressions with a single literal, deeply nested expressions, and expressions with many sub-expressions.

## Approaches

The brute-force approach would involve parsing the expression character by character, manually keeping track of parentheses, and recursively evaluating sub-expressions. This would work correctly but can be cumbersome to implement. It may also use additional string copies or slicing repeatedly, which is inefficient for large inputs.

The optimal solution leverages a **stack-based approach with recursive evaluation**, treating the expression as a tree where each operator applies to its sub-expressions. Each time we encounter a closing parenthesis `')'`, we pop values from the stack until the corresponding operator is reached. Then we evaluate the sub-expression immediately, reducing memory usage and avoiding repeated parsing of the same substring. This approach is both clean and efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n^2) | Repeated slicing and parsing of strings for nested expressions |
| Optimal | O(n) | O(n) | Single pass with stack, evaluate each sub-expression exactly once |

## Algorithm Walkthrough

1. Initialize a stack to keep track of operators and values.
2. Iterate through each character of the expression:

- If the character is `'t'`, `'f'`, `'&'`, `'|'`, `'!'`, or `'('`, push it onto the stack.
- If the character is `','`, skip it as it only separates sub-expressions.
- If the character is `')'`, this marks the end of a sub-expression:

1. Pop elements from the stack until you reach the operator for this sub-expression.
2. Collect all boolean values encountered between the operator and `')'`.
3. Apply the operator (`!`, `&`, `|`) to the collected boolean values.
4. Push the result (`'t'` or `'f'`) back onto the stack.
3. After processing the entire expression, the stack contains a single value representing the result.
4. Convert this final value to `True` or `False` and return it.

Why it works: At any closing parenthesis, we evaluate the sub-expression fully and replace it with a single boolean value. This ensures that nested expressions are reduced layer by layer, preserving correctness.

## Python Solution

```
class Solution:
    def parseBoolExpr(self, expression: str) -> bool:
        stack = []
        for ch in expression:
            if ch in ('t', 'f', '!', '&', '|', '('):
                stack.append(ch)
            elif ch == ')':
                vals = []
                while stack and stack[-1] != '(':
                    vals.append(stack.pop())
                stack.pop()  # Remove the '('
                op = stack.pop()  # Operator before '('
                if op == '!':
                    stack.append('t' if vals[0] == 'f' else 'f')
                elif op == '&':
                    stack.append('t' if all(v == 't' for v in vals) else 'f')
                elif op == '|':
                    stack.append('t' if any(v == 't' for v in vals) else 'f')
        return stack[0] == 't'
```

In this implementation, we maintain a stack to store operators and operands. When encountering `')'`, we extract all values up to the corresponding `'('`, determine which operator applies, compute the result, and push it back to the stack. This avoids repeated string slicing or recursion.

## Go Solution

```
func parseBoolExpr(expression string) bool {
    stack := []rune{}
    for _, ch := range expression {
        if ch == 't' || ch == 'f' || ch == '!' || ch == '&' || ch == '|' || ch == '(' {
            stack = append(stack, ch)
        } else if ch == ')' {
            vals := []rune{}
            for stack[len(stack)-1] != '(' {
                vals = append(vals, stack[len(stack)-1])
                stack = stack[:len(stack)-1]
            }
            stack = stack[:len(stack)-1] // pop '('
            op := stack[len(stack)-1]
            stack = stack[:len(stack)-1] // pop operator
            var res rune
            switch op {
            case '!':
                if vals[0] == 'f' {
                    res = 't'
                } else {
                    res = 'f'
                }
            case '&':
                res = 't'
                for _, v := range vals {
                    if v == 'f' {
                        res = 'f'
                        break
                    }
                }
            case '|':
                res = 'f'
                for _, v := range vals {
                    if v == 't' {
                        res = 't'
                        break
                    }
                }
            }
            stack = append(stack, res)
        }
    }
    return stack[0] == 't'
}
```

The Go solution mirrors the Python stack approach, using a slice of `rune` instead of characters. The main differences are in slice management for popping elements and the use of a switch statement for operator evaluation.

## Worked Examples

**Example:** `"!(&(f,t))"`

| Step | Stack | Action |
| --- | --- | --- |
| 1 | `['!', '(', '&', '(']` | Push first characters |
| 2 | `['!', '(', '&', '(', 'f']` | Push `'f'` |
| 3 | `['!', '(', '&', '(', 'f', 't']` | Push `'t'` |
| 4 | `['!', '(', '&']` | Encounter `')'`, evaluate `'&(f,t)'` → `'f'` |
| 5 | `['!','(', 'f']` | Push result `'f'` |
| 6 | `['!']` | Encounter `')'`, evaluate `'!(f)'` → `'t'` |
| 7 | `['t']` | Final result `'t'` → `True` |

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is processed at most twice (once in iteration, once in popping) |
| Space | O(n) | Stack can grow up to the size of the expression in the worst case |

The algorithm achieves linear time and space because every character is pushed and popped exactly once.

## Edge Cases

1. **Single Literal Expressions:** The expression is just `'t'` or `'f'`. A naive implementation that expects parentheses would fail. Our stack-based approach correctly handles these by pushing the literal and returning its boolean value.
2. **Deeply Nested Expressions:** Expressions like `"!(&(|(f,t),!(f)))"` test recursion depth. By using a stack rather than recursive function calls, the implementation avoids exceeding Python or Go recursion limits and handles arbitrarily deep nesting efficiently.
3. **Expressions with Many Operands:** Cases like `"|(f,f,f,...,t)"` stress the algorithm's ability to handle long lists of sub-expressions. The stack approach efficiently aggregates values
