## Solution

## Problem Understanding

This problem asks us to find the **lowest common ancestor (LCA) of the deepest leaves** in a binary tree. In simpler terms, we are given a binary tree and need to identify the node that is the shared ancestor of all nodes at the maximum depth in the tree. The "lowest" in LCA means the ancestor closest to the leaves, so among all common ancestors of the deepest nodes, we want the one furthest from the root.

The input is the `root` of a binary tree, which is guaranteed to have at least one node and contains unique integer values in the range `0 <= Node.val <= 1000`. The output is a reference to the `TreeNode` representing the LCA of the deepest leaves. The deepest leaves are the nodes that are furthest from the root; their depth is maximal across the tree.

Important points to note are that there can be multiple deepest leaves at the same depth, and the tree is not necessarily balanced. Edge cases include trees with only one node (where the root itself is the deepest leaf) or linear trees where all nodes are along a single path. The problem guarantees uniqueness of node values and a moderate input size (`1 <= n <= 1000`), so we can focus on correctness and standard DFS/BFS approaches without worrying about extreme performance tuning.

## Approaches

The **brute-force approach** would involve two main steps: first, find the deepest depth of the tree by performing a BFS or DFS traversal, and second, for each node at that depth, repeatedly compute the LCA pairwise until a single LCA for all deepest leaves is found. While this works logically, it is inefficient because we traverse the tree multiple times and recompute LCAs unnecessarily. Its complexity is higher than needed and involves repeated traversal and storage of deepest nodes.

The **optimal approach** relies on a **bottom-up depth-first search (DFS)**. The key insight is that for each node, if its left and right subtrees are of the same maximum depth, the node itself is the LCA of the deepest nodes in those subtrees. If one subtree is deeper, the LCA lies entirely within that subtree. By returning both the depth and the candidate LCA from each recursive call, we can efficiently compute the answer in a single traversal.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Multiple traversals to find deepest leaves and compute LCA repeatedly |
| Optimal | O(n) | O(h) | Single DFS traversal returning depth and LCA; h = tree height |

## Algorithm Walkthrough

1. Define a recursive function `dfs(node)` that returns a tuple `(max_depth, lca_node)` for the subtree rooted at `node`.
2. If `node` is `None`, return `(-1, None)` because an empty subtree has depth `-1` and no LCA.
3. Recursively call `dfs(node.left)` and `dfs(node.right)` to obtain the maximum depths and LCAs of the left and right subtrees.
4. Compare the depths:

- If `left_depth == right_depth`, then `node` is the LCA of the deepest leaves in its subtree.
- If `left_depth > right_depth`, propagate the left LCA upward.
- If `right_depth > left_depth`, propagate the right LCA upward.
5. Return `(max(left_depth, right_depth) + 1, candidate_LCA)` for the current node.
6. Finally, call `dfs(root)` and return the LCA from the result.

**Why it works:** This works because the recursive function ensures that at each node, we know the depth of the deepest leaves in both subtrees. The first node where both sides have equal maximum depth must be the LCA of all deepest leaves in its subtree. By propagating the candidate LCA upward with depth information, we guarantee a correct, bottom-up identification of the LCA.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def lcaDeepestLeaves(self, root: Optional[TreeNode]) -> Optional[TreeNode]:
        def dfs(node: Optional[TreeNode]) -> tuple[int, Optional[TreeNode]]:
            if not node:
                return -1, None
            
            left_depth, left_lca = dfs(node.left)
            right_depth, right_lca = dfs(node.right)
            
            if left_depth == right_depth:
                return left_depth + 1, node
            elif left_depth > right_depth:
                return left_depth + 1, left_lca
            else:
                return right_depth + 1, right_lca
        
        return dfs(root)[1]
```

The `dfs` function computes the maximum depth and potential LCA of the deepest leaves in a bottom-up manner. If a node’s left and right subtrees have the same depth, that node is the LCA. Otherwise, the LCA propagates from the deeper subtree. The final result is extracted from the DFS call on the root.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func lcaDeepestLeaves(root *TreeNode) *TreeNode {
    var dfs func(node *TreeNode) (int, *TreeNode)
    dfs = func(node *TreeNode) (int, *TreeNode) {
        if node == nil {
            return -1, nil
        }
        
        leftDepth, leftLCA := dfs(node.Left)
        rightDepth, rightLCA := dfs(node.Right)
        
        if leftDepth == rightDepth {
            return leftDepth + 1, node
        } else if leftDepth > rightDepth {
            return leftDepth + 1, leftLCA
        } else {
            return rightDepth + 1, rightLCA
        }
    }
    
    _, lca := dfs(root)
    return lca
}
```

The Go implementation mirrors the Python solution. Nil pointers are handled explicitly. Tuples are replaced with multiple return values. Other than syntax differences, the logic is identical.

## Worked Examples

**Example 1:** `root = [3,5,1,6,2,0,8,null,null,7,4]`

- DFS returns `(depth, lca)` for each node:

- Leaves 7,4 → depth 0 → LCA is themselves
- Node 2 → left depth 1 (7), right depth 1 (4) → depths equal → LCA = 2
- Node 5 → left depth 1 (6), right depth 2 (2 subtree) → right deeper → LCA = 2
- Node 1 → both children depth 1 (0,8) → LCA = 1
- Root 3 → left depth 3 (2 subtree), right depth 2 → left deeper → LCA = 2
- Return 2

**Example 2:** `root = [1]`

- Single node tree → LCA is root itself.

**Example 3:** `root = [0,1,3,null,2]`

- Deepest leaf is 2 → LCA is 2 itself.

The optimal DFS ensures linear traversal, avoiding repeated work, and the space is proportional to the tree height, which is acceptable for n ≤ 1000.

## Edge Cases

1. **Single-node tree:** The root is both the only leaf and the LCA. Without handling this explicitly, some naive DFS implementations might attempt to access nonexistent children.
2. **Linear tree:** When all nodes form a single path, the deepest leaf is at the end, and the LCA is that leaf itself. Ensuring the algorithm returns the deepest leaf, not its parent, is critical
