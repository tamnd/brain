## Solution

## Problem Understanding

The problem asks us to compute the maximum average value among all possible subtrees in a binary tree. A subtree consists of a node together with all of its descendants. For every node in the tree, we can consider the subtree rooted at that node, calculate its average value, and then determine which subtree produces the largest average.

The input is the root node of a binary tree. Each node contains a non-negative integer value, and the tree may contain up to `10^4` nodes. The output is a floating point number representing the maximum subtree average. Since the problem accepts answers within `10^-5` of the correct value, standard floating point arithmetic is sufficient.

To calculate the average of a subtree, we need two pieces of information: the total sum of all node values in that subtree and the total number of nodes. The average is:

$$\text{average} = \frac{\text{subtree sum}}{\text{subtree node count}}$$

The constraints are important because they eliminate inefficient approaches. A naive solution that repeatedly traverses subtrees from scratch for every node could become quadratic in the worst case, which would be too slow for `10^4` nodes. This strongly suggests that we should compute subtree information in a single traversal.

Several edge cases are important to recognize early. A tree with only one node should return that node’s value as the answer. Highly unbalanced trees behave like linked lists, so recursive depth must be considered carefully. Trees with many equal values should still produce the correct average without relying on strict comparisons. Since node values are guaranteed to be non-negative, the maximum average will always be at least zero.

## Approaches

### Brute Force Approach

A brute force solution would examine every node as a potential subtree root. For each node, we could run a traversal to calculate the total sum and total number of nodes in that subtree, compute the average, and update the maximum average if necessary.

This approach is correct because it explicitly computes the average for every possible subtree. However, it is inefficient because many nodes are visited repeatedly. For example, in a skewed tree, the subtree rooted at the top includes almost the entire tree, the next subtree includes almost the entire remaining tree, and so on. This repeated work leads to quadratic time complexity in the worst case.

### Optimal Approach

The key observation is that each subtree only needs two values: its total sum and its node count. Once we know those values for the left and right children, we can immediately compute them for the current node.

This naturally leads to a postorder depth-first traversal. We process the left subtree first, then the right subtree, and finally the current node. At each node, we combine information from the children to determine:

- Total subtree sum
- Total subtree node count
- Current subtree average

Since every node contributes to these calculations exactly once, the entire computation becomes linear.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(h) | Recomputes subtree sums and counts repeatedly |
| Optimal | O(n) | O(h) | Single DFS traversal computes all subtree statistics |

Here, `n` is the number of nodes and `h` is the height of the tree.

## Algorithm Walkthrough

1. Start a depth-first traversal from the root using postorder traversal. Postorder is important because a node depends on information from both of its children before it can compute its own subtree statistics.
2. For each node, recursively process the left subtree and obtain:

- The total sum of values in the left subtree
- The number of nodes in the left subtree
3. Recursively process the right subtree and obtain:

- The total sum of values in the right subtree
- The number of nodes in the right subtree
4. Combine the child results with the current node’s value:

- Current subtree sum = left sum + right sum + current node value
- Current subtree count = left count + right count + 1
5. Compute the current subtree average:

$$\text{average} = \frac{\text{subtree sum}}{\text{subtree count}}$$
6. Compare this average against the global maximum average. If the current average is larger, update the global result.
7. Return the subtree sum and subtree count back to the parent node so the parent can compute its own subtree statistics.
8. Continue this process until the entire tree has been processed.

### Why it works

The algorithm works because every subtree rooted at a node is completely determined by its left subtree, right subtree, and the node itself. Postorder traversal guarantees that when we process a node, all information about its descendants has already been computed correctly. Since every subtree average is evaluated exactly once, and the global maximum is updated whenever a larger average is found, the final result is guaranteed to be the maximum subtree average in the tree.

## Python Solution

```
from typing import Optional

# Definition for a binary tree node.
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

class Solution:
    def maximumAverageSubtree(self, root: Optional[TreeNode]) -> float:
        self.max_average = 0.0

        def dfs(node: Optional[TreeNode]) -> tuple[int, int]:
            if not node:
                return (0, 0)

            left_sum, left_count = dfs(node.left)
            right_sum, right_count = dfs(node.right)

            subtree_sum = left_sum + right_sum + node.val
            subtree_count = left_count + right_count + 1

            current_average = subtree_sum / subtree_count
            self.max_average = max(self.max_average, current_average)

            return (subtree_sum, subtree_count)

        dfs(root)

        return self.max_average
```

The implementation follows the postorder DFS strategy described earlier. The recursive helper function returns two values for every subtree: the total sum and the total node count.

The base case handles null nodes by returning `(0, 0)`, which represents an empty subtree with no contribution to either sum or count.

For non-null nodes, the function recursively gathers information from the left and right children. These results are combined with the current node’s value to compute the total subtree sum and total subtree size.

Once the subtree statistics are available, the average is computed immediately. The global variable `self.max_average` stores the best average seen so far across all subtrees.

Finally, the function returns the computed sum and count so the parent node can use them in its own calculations.

## Go Solution

```
package main

/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

func maximumAverageSubtree(root *TreeNode) float64 {
	var maxAverage float64

	var dfs func(node *TreeNode) (int, int)

	dfs = func(node *TreeNode) (int, int) {
		if node == nil {
			return 0, 0
		}

		leftSum, leftCount := dfs(node.Left)
		rightSum, rightCount := dfs(node.Right)

		subtreeSum := leftSum + rightSum + node.Val
		subtreeCount := leftCount + rightCount + 1

		currentAverage := float64(subtreeSum) / float64(subtreeCount)

		if currentAverage > maxAverage {
			maxAverage = currentAverage
		}

		return subtreeSum, subtreeCount
	}

	dfs(root)

	return maxAverage
}
```

The Go implementation mirrors the Python solution closely. Since Go does not support returning tuples directly, the recursive helper returns two integers representing the subtree sum and subtree node count.

Floating point conversion is necessary when computing the average because integer division would truncate the result. Go’s explicit type conversion ensures accurate floating point arithmetic.

The recursion depth behavior is similar to Python. The space complexity still depends on the tree height because of the recursive call stack.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every node is visited exactly once |
| Space | O(h) | Recursive call stack depends on tree height |

The algorithm performs a single DFS traversal. Each node contributes constant work involving arithmetic operations and comparisons, so the total running time is linear in the number of nodes.

The space complexity comes from recursion. In a balanced tree, the recursion depth is `O(log n)`. In the worst case of a completely skewed tree, the recursion depth becomes `O(n)`.

## Edge Cases

A single-node tree is the smallest possible valid input. This case is important because the subtree consists only of the root itself. Some implementations mistakenly assume children always exist, which can lead to null pointer errors. The recursive base case correctly handles missing children by returning zero sums and counts.

Highly skewed trees are another important edge case. A tree that behaves like a linked list creates maximum recursion depth. Incorrect subtree aggregation logic often appears in these structures because each node only has one child. The implementation handles this naturally because null children contribute `(0, 0)` and the remaining subtree still computes correctly.

Trees where the maximum average belongs to a leaf node can also cause bugs. Some incorrect solutions assume larger subtrees produce better averages. In reality, a single high-valued node may exceed the average of every larger subtree. Since the algorithm evaluates every subtree individually, including single-node subtrees, it correctly identifies leaf nodes as the answer when appropriate.
