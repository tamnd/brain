## Solution

## Problem Understanding

The problem gives us an undirected tree with `n` nodes labeled from `0` to `n - 1`. A tree is a connected graph with no cycles, which guarantees that there is exactly one path between any pair of nodes.

We are allowed to choose any node as the root of the tree. Once a root is chosen, the height of the resulting rooted tree is defined as the number of edges in the longest downward path from the root to any leaf.

The goal is to find every possible root that produces the minimum possible tree height. These roots are called Minimum Height Trees, or MHTs.

The input consists of two parts:

- `n`, the number of nodes
- `edges`, an array of `n - 1` undirected edges

Because the graph is guaranteed to be a valid tree:

- The graph is connected
- There are no cycles
- There are exactly `n - 1` edges

The constraints are important:

- `n` can be as large as `2 * 10^4`
- A brute-force traversal from every node would likely be too slow
- We need something close to linear time

The output is a list of node labels that can serve as roots of minimum height trees.

An important property of trees is that a tree can have either:

- One center node
- Two center nodes

These center nodes are exactly the roots that produce minimum height trees.

Several edge cases are important:

- A tree with only one node
- A tree with two nodes
- A chain-shaped tree, where the answer may contain two middle nodes
- A star-shaped tree, where the center node is the answer
- Extremely unbalanced trees

The problem guarantees valid tree input, so we do not need to handle disconnected graphs or cycles.

## Approaches

### Brute-Force Approach

The most straightforward solution is to try every node as the root.

For each node:

1. Perform a BFS or DFS traversal
2. Compute the height of the rooted tree
3. Track the minimum height encountered
4. Return all nodes that achieve that minimum

This works because we explicitly evaluate every possible root and measure its resulting height.

However, this approach is too slow for the given constraints.

If we perform a BFS or DFS from every node:

- Each traversal costs `O(n)`
- We repeat this for all `n` nodes

The total complexity becomes `O(n^2)`.

With `n = 2 * 10^4`, this can become too expensive.

### Optimal Approach, Topological Leaf Trimming

The key insight is that the roots of minimum height trees are always located at the center of the tree.

Instead of testing every possible root, we can repeatedly remove leaf nodes.

A leaf is a node with only one connection.

This process works similarly to topological sorting:

- Remove all current leaves
- Update the degrees of neighboring nodes
- New leaves emerge
- Continue until only one or two nodes remain

Those remaining nodes are the centers of the tree, and therefore the roots of all minimum height trees.

This works because removing outer layers gradually shrinks the tree toward its center.

The process is analogous to peeling an onion layer by layer.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Run BFS or DFS from every node |
| Optimal | O(n) | O(n) | Repeatedly trim leaves until centers remain |

## Algorithm Walkthrough

1. Handle the single-node edge case.

If `n == 1`, the tree contains only one node and that node is trivially the root of the minimum height tree. Return `[0]`.
2. Build the adjacency list.

Since the graph is undirected, each edge connects two nodes bidirectionally. We store neighbors for every node so we can efficiently traverse and update the graph.
3. Track node degrees.

The degree of a node is the number of connected neighbors. Leaves are exactly the nodes with degree `1`.

We use an array called `degree` where `degree[i]` stores the number of neighbors of node `i`.
4. Initialize the first layer of leaves.

Iterate through all nodes and collect those whose degree is `1`.

These nodes form the outermost layer of the tree.
5. Repeatedly remove leaves layer by layer.

While more than two nodes remain:

1. Count how many leaves exist in the current layer.
2. Remove those leaves from the graph conceptually.
3. Decrease the degree of each neighboring node.
4. If a neighbor's degree becomes `1`, it becomes a new leaf and is added to the queue.

This process continuously strips away the outermost layer.
6. Return the remaining nodes.

Eventually, only one or two nodes remain.

These nodes are the tree centers and therefore the roots of the minimum height trees.

### Why it works

The longest paths in a tree always pass through the center of the tree.

Leaf nodes are guaranteed not to be optimal roots because they maximize distance to other nodes. By repeatedly removing leaves, we eliminate nodes that cannot possibly be centers.

Each round moves inward by one level from both ends of the tree. Eventually, only the central node or central pair remains. Those nodes minimize the maximum distance to all other nodes, which means they produce minimum height trees.

## Python Solution

```
from collections import deque
from typing import List

class Solution:
    def findMinHeightTrees(self, n: int, edges: List[List[int]]) -> List[int]:
        if n == 1:
            return [0]

        graph = [[] for _ in range(n)]
        degree = [0] * n

        # Build graph and degree count
        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)

            degree[u] += 1
            degree[v] += 1

        # Initialize leaves
        leaves = deque()

        for node in range(n):
            if degree[node] == 1:
                leaves.append(node)

        remaining_nodes = n

        # Trim leaves layer by layer
        while remaining_nodes > 2:
            leaf_count = len(leaves)
            remaining_nodes -= leaf_count

            for _ in range(leaf_count):
                leaf = leaves.popleft()

                for neighbor in graph[leaf]:
                    degree[neighbor] -= 1

                    if degree[neighbor] == 1:
                        leaves.append(neighbor)

        return list(leaves)
```

The implementation begins with the special case where the tree contains only one node. Since there are no edges, node `0` is automatically the answer.

Next, the adjacency list and degree array are constructed simultaneously. The adjacency list allows efficient neighbor traversal, while the degree array lets us quickly identify leaves.

A queue stores all current leaves. Using a queue is important because we process the tree level by level, similar to BFS.

The main loop continues until at most two nodes remain. At each iteration:

- We remove the current leaves
- Reduce the degrees of neighboring nodes
- Detect newly formed leaves

Finally, the remaining nodes in the queue are returned as the centers of the tree.

## Go Solution

```
func findMinHeightTrees(n int, edges [][]int) []int {
	if n == 1 {
		return []int{0}
	}

	graph := make([][]int, n)
	degree := make([]int, n)

	// Build graph and degree count
	for _, edge := range edges {
		u := edge[0]
		v := edge[1]

		graph[u] = append(graph[u], v)
		graph[v] = append(graph[v], u)

		degree[u]++
		degree[v]++
	}

	// Initialize leaves
	queue := []int{}

	for i := 0; i < n; i++ {
		if degree[i] == 1 {
			queue = append(queue, i)
		}
	}

	remainingNodes := n

	// Trim leaves layer by layer
	for remainingNodes > 2 {
		leafCount := len(queue)
		remainingNodes -= leafCount

		newQueue := []int{}

		for _, leaf := range queue {
			for _, neighbor := range graph[leaf] {
				degree[neighbor]--

				if degree[neighbor] == 1 {
					newQueue = append(newQueue, neighbor)
				}
			}
		}

		queue = newQueue
	}

	return queue
}
```

The Go implementation follows the same logic as the Python version.

One notable difference is queue handling. Instead of using a dedicated deque structure, the solution uses slices to represent the current layer of leaves and the next layer.

Go slices dynamically resize as needed, making them suitable for BFS-style processing.

There are no integer overflow concerns because the constraints are small enough for Go's standard `int` type.

## Edge Cases

### Single Node Tree

A tree with only one node is a special case because there are no edges and no leaves with degree `1` in the usual sense. Without explicit handling, the queue initialization logic would fail to identify the correct answer.

The implementation handles this immediately:

```
if n == 1:
    return [0]
```

This guarantees correctness for the smallest valid input.

### Two-Node Tree

When the tree contains exactly two connected nodes, both nodes are valid MHT roots because rooting at either node produces height `1`.

A buggy implementation might continue trimming until no nodes remain. This implementation stops once the number of remaining nodes is less than or equal to `2`, preserving both centers correctly.

### Long Chain-Shaped Tree

A chain is one of the most important stress cases because it produces the maximum possible tree height.

For example:

```
0 - 1 - 2 - 3 - 4 - 5
```

The correct answer is `[2, 3]`.

The layer-by-layer trimming process naturally converges toward the middle of the chain, ensuring the algorithm correctly identifies one or two center nodes depending on whether the chain length is odd or even.

### Star-Shaped Tree

In a star-shaped tree, one central node connects to every other node.

Example:

```
    1
    |
2 - 0 - 3
    |
    4
```

All outer nodes are leaves and are removed in the first iteration. The center node remains as the only valid MHT root.

This case validates that the algorithm correctly handles nodes with very high degree values.
