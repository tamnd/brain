## Solution

## Problem Understanding

The problem asks us to reverse only the vowels in a string while leaving every non-vowel character in its original position. A vowel is defined as one of the characters `'a'`, `'e'`, `'i'`, `'o'`, or `'u'`, and the problem explicitly states that both lowercase and uppercase vowels must be treated as vowels.

The input is a string `s` containing printable ASCII characters. The output should be a new string where the sequence of vowels appears in reverse order, but every consonant, digit, symbol, or whitespace character remains exactly where it originally appeared.

For example, if the input is `"IceCreAm"`, the vowels encountered from left to right are `['I', 'e', 'e', 'A']`. Reversing this sequence gives `['A', 'e', 'e', 'I']`. Replacing the original vowels with these reversed vowels produces `"AceCreIm"`.

The constraints are important because the string length can be as large as `3 * 10^5`. This tells us that we need an efficient solution. An algorithm with quadratic complexity, such as repeatedly rebuilding strings or scanning the string multiple times unnecessarily, could become too slow for large inputs.

There are several edge cases worth identifying early:

- A string with no vowels, such as `"rhythm"`, should remain unchanged.
- A string consisting entirely of vowels, such as `"aeiou"`, should become `"uoiea"`.
- Uppercase vowels must be handled correctly, so `"HELLO"` should reverse vowels as `"HOLLE"`.
- Strings of length `1` must work correctly whether the character is a vowel or not.
- Repeated vowels and adjacent vowels can expose pointer movement bugs if not handled carefully.

Because Python strings and Go strings are immutable, we also need a mutable representation during processing, typically a list of characters in Python or a byte slice in Go.

## Approaches

A brute-force approach would first collect all vowels from the string into a separate list. Then, reverse that list of vowels. Finally, scan the original string again and replace each vowel with the next reversed vowel.

This approach is correct because it preserves the positions of all non-vowel characters while explicitly reversing only the vowel sequence. However, depending on implementation details, repeated string concatenation can become inefficient. Building new strings character by character may introduce unnecessary overhead.

The key insight for the optimal solution is that we do not actually need to explicitly store and reverse all vowels. Instead, we can use two pointers, one starting from the left side of the string and one from the right side. The left pointer searches for the next vowel from the beginning, while the right pointer searches for the next vowel from the end. Once both pointers are positioned on vowels, we swap them and continue moving inward.

This works because reversing a sequence naturally pairs the first vowel with the last vowel, the second vowel with the second-to-last vowel, and so on. The two-pointer technique directly performs these swaps in a single traversal.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Collect vowels, reverse them, then rebuild the string |
| Optimal | O(n) | O(n) | Two pointers swap vowels in one pass |

Although both approaches use linear time, the two-pointer solution is more direct and avoids storing a separate reversed vowel list.

## Algorithm Walkthrough

1. Create a set containing all vowels, including uppercase and lowercase versions.

We use a set because membership checks are extremely fast, typically O(1). This allows us to quickly determine whether a character is a vowel.
2. Convert the string into a mutable sequence of characters.

Strings are immutable in both Python and Go, so we need a structure that allows character swapping.
3. Initialize two pointers.

The left pointer starts at index `0`, and the right pointer starts at the last index of the string.
4. Move the left pointer forward until it points to a vowel.

Any non-vowel character should remain in place, so we skip over consonants and symbols.
5. Move the right pointer backward until it points to a vowel.

This identifies the matching vowel from the opposite end of the string.
6. Swap the vowels at the two pointers.

This places the correct reversed vowel into each position.
7. Move both pointers inward.

After swapping, both positions are finalized, so we continue searching toward the center.
8. Repeat until the pointers cross.

Once the pointers meet or cross, all vowel swaps are complete.
9. Convert the mutable character sequence back into a string and return it.

This produces the final result.

### Why it works

The algorithm maintains the invariant that all vowels outside the current pointer range have already been placed into their correct reversed positions. Each swap matches the next vowel from the left side with the next vowel from the right side, exactly mirroring how reversal works. Since every vowel is processed once and non-vowel characters are never moved, the final string is guaranteed to contain the original non-vowel characters in their original positions and the vowels in reversed order.

## Python Solution

```
class Solution:
    def reverseVowels(self, s: str) -> str:
        vowels = set("aeiouAEIOU")
        
        characters = list(s)
        
        left = 0
        right = len(characters) - 1
        
        while left < right:
            while left < right and characters[left] not in vowels:
                left += 1
            
            while left < right and characters[right] not in vowels:
                right -= 1
            
            characters[left], characters[right] = (
                characters[right],
                characters[left],
            )
            
            left += 1
            right -= 1
        
        return "".join(characters)
```

The implementation begins by creating a set of vowels for constant-time membership checking. This is important because the algorithm performs many vowel checks while scanning the string.

The string is converted into a list named `characters` because Python strings cannot be modified directly. Lists allow efficient in-place swaps.

Two pointers, `left` and `right`, are initialized at opposite ends of the list. The outer `while` loop continues as long as the pointers have not crossed.

Inside the loop, the left pointer advances until it finds a vowel. Similarly, the right pointer moves backward until it finds a vowel. Once both pointers identify vowels, the algorithm swaps them.

After the swap, both pointers move inward because those positions are now finalized. Once the loop finishes, the modified character list is joined back into a string and returned.

## Go Solution

```
func reverseVowels(s string) string {
    vowels := map[byte]bool{
        'a': true, 'e': true, 'i': true, 'o': true, 'u': true,
        'A': true, 'E': true, 'I': true, 'O': true, 'U': true,
    }

    chars := []byte(s)

    left := 0
    right := len(chars) - 1

    for left < right {
        for left < right && !vowels[chars[left]] {
            left++
        }

        for left < right && !vowels[chars[right]] {
            right--
        }

        chars[left], chars[right] = chars[right], chars[left]

        left++
        right--
    }

    return string(chars)
}
```

The Go solution follows the same algorithmic structure as the Python implementation. Since Go strings are immutable, the string is converted into a byte slice using `[]byte(s)`.

A `map[byte]bool` is used for vowel lookup. This provides efficient membership testing similar to a Python set.

Because the input consists of printable ASCII characters, using a byte slice is safe and efficient. If Unicode handling were required, we would need to use rune slices instead.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is visited at most once by each pointer |
| Space | O(n) | Mutable character storage is required |

The time complexity is linear because both pointers only move inward, never backward. Each character is processed at most once by the left pointer and once by the right pointer.

The space complexity is O(n) because strings are immutable, so we must create a mutable copy of the characters before performing swaps.

## Edge Cases

A string with no vowels is an important edge case because the pointers may continuously skip characters without ever performing a swap. A buggy implementation might accidentally move pointers incorrectly or attempt invalid swaps. In this implementation, the inner loops safely advance until the pointers cross, and the original string is returned unchanged.

A string containing only vowels is another critical case because every character participates in the reversal. This fully exercises the swapping logic and pointer movement. The implementation correctly swaps matching vowels from opposite ends until the middle is reached.

Uppercase vowels can easily be overlooked if the vowel set contains only lowercase letters. Inputs such as `"HELLO"` or `"IceCreAm"` verify whether the algorithm correctly identifies uppercase vowels. This implementation explicitly includes both lowercase and uppercase vowels in the lookup set, ensuring correct handling.

Single-character strings are also important because the two pointers immediately start at the same position. The loop condition `left < right` prevents unnecessary processing and safely returns the original string.

Finally, strings containing symbols, spaces, or punctuation test whether the implementation properly ignores non-vowel characters. Since the algorithm only swaps characters found in the vowel set, all other characters remain untouched in their original positions.
