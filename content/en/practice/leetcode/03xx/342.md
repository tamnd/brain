## Solution

## Problem Understanding

The problem asks us to determine whether a given integer `n` is a power of four.

A number is considered a power of four if it can be written in the form:

$n = 4^x$

where `x` is a non-negative integer.

This means valid powers of four include:

- `1 = 4^0`
- `4 = 4^1`
- `16 = 4^2`
- `64 = 4^3`

and so on.

The input is a single integer `n`, and the expected output is a boolean value:

- Return `true` if `n` is a power of four
- Return `false` otherwise

The constraints allow values from `-2^31` to `2^31 - 1`, which means the input may be negative, zero, or positive. Since powers of four are always positive, any non-positive number must immediately return `false`.

The follow-up question asks whether the solution can be implemented without loops or recursion. This strongly suggests that a mathematical or bit manipulation approach is expected, rather than repeated division.

Several edge cases are important:

- `n <= 0` should always return `false`
- `n = 1` should return `true`, because `1 = 4^0`
- Numbers that are powers of two but not powers of four, such as `8` or `32`, should return `false`
- Very large integers near the 32-bit limit should still work correctly without overflow issues

## Approaches

### Brute Force Approach

The most direct approach is to repeatedly divide the number by `4` until it is no longer divisible by `4`.

The algorithm works as follows:

- If `n` is less than or equal to `0`, return `false`
- While `n` is divisible by `4`, divide it by `4`
- At the end, if the remaining value is exactly `1`, then the original number was a power of four

This works because every power of four can be reduced to `1` by repeatedly dividing by `4`.

For example:

- `64 -> 16 -> 4 -> 1`
- `20 -> 5`, stop because `5` is not divisible by `4`

Although this solution is already efficient for the given constraints, it still uses a loop. The follow-up encourages us to find a constant-time mathematical or bit-based solution.

### Optimal Bit Manipulation Approach

The key observation is that every power of four has two important properties in binary representation.

First, every power of four is also a power of two.

A power of two contains exactly one set bit in binary. For example:

| Number | Binary |
| --- | --- |
| 1 | `0001` |
| 4 | `0100` |
| 16 | `10000` |
| 64 | `1000000` |

A standard bit trick checks whether a number is a power of two:

$n \& (n-1) = 0$

This expression removes the lowest set bit. If the result becomes zero, then the number had exactly one set bit.

However, powers of two alone are not enough. Numbers like `8` and `32` are powers of two but not powers of four.

The second observation is that powers of four always place their single set bit in an even position:

| Number | Binary | Bit Position |
| --- | --- | --- |
| 1 | `0001` | 0 |
| 4 | `0100` | 2 |
| 16 | `10000` | 4 |
| 64 | `1000000` | 6 |

All these positions are even.

We can use the hexadecimal mask `0x55555555`, whose binary form contains `1`s only in even bit positions:

```
01010101010101010101010101010101
```

If the single set bit overlaps with this mask, the number is a power of four.

The final condition becomes:

$n > 0 \land (n \& (n-1)) = 0 \land (n \& 0x55555555) \neq 0$

## Algorithm Walkthrough

1. First, check whether `n` is positive.

Powers of four are always positive integers. If `n <= 0`, immediately return `false`.
2. Check whether `n` is a power of two.

Use the expression:

```
n & (n - 1)
```

A power of two has exactly one set bit. Subtracting `1` flips all bits after that set bit, so the bitwise AND becomes zero only for powers of two.
3. Check whether the set bit is in an even position.

Use the mask:

```
0x55555555
```

This mask keeps only bits in even positions. If the number's single set bit overlaps with this mask, then the number is a power of four.
4. Return `true` only if all conditions pass.

### Why it works

The algorithm relies on two mathematical properties.

Every power of four is also a power of two, so it must contain exactly one set bit. Additionally, powers of four place that set bit only at even bit positions. The mask `0x55555555` filters exactly those positions. Combining these checks guarantees that the number is a power of four and excludes all other integers.

## Python Solution

```
class Solution:
    def isPowerOfFour(self, n: int) -> bool:
        if n <= 0:
            return False

        is_power_of_two = (n & (n - 1)) == 0
        has_even_position_bit = (n & 0x55555555) != 0

        return is_power_of_two and has_even_position_bit
```

The implementation begins by rejecting all non-positive numbers, since powers of four must be positive.

Next, the code checks whether the number is a power of two. The expression `(n & (n - 1)) == 0` ensures that only one bit is set.

After that, the code verifies that the set bit appears in an even position using the mask `0x55555555`.

Finally, both conditions are combined with logical AND. Only numbers satisfying both properties return `True`.

## Go Solution

```
func isPowerOfFour(n int) bool {
    if n <= 0 {
        return false
    }

    isPowerOfTwo := (n & (n - 1)) == 0
    hasEvenPositionBit := (n & 0x55555555) != 0

    return isPowerOfTwo && hasEvenPositionBit
}
```

The Go implementation is nearly identical to the Python version. Go uses explicit boolean variables for readability, just like Python.

One important detail is that Go integers are fixed-width machine integers, while Python integers are arbitrary precision. However, since the problem constraints fit within 32-bit signed integers, the bit mask works safely in both languages.

## Edge Cases

One important edge case is when the input is `0`. A naive implementation that only checks divisibility by `4` might accidentally treat zero incorrectly because zero is divisible by many numbers. The implementation avoids this by immediately rejecting all non-positive numbers before performing any bit operations.

Another critical edge case is negative numbers such as `-4` or `-64`. Bitwise operations on signed integers can behave unexpectedly if positivity is not checked first. Since powers of four are always positive, the algorithm safely handles this by returning `false` immediately for any value less than or equal to zero.

A subtle edge case involves numbers that are powers of two but not powers of four, such as `8` or `32`. These numbers contain exactly one set bit, so they pass the power-of-two check. However, their set bit appears in an odd position. The mask `0x55555555` correctly filters them out because it only allows set bits in even positions.

Another edge case is `1`. Some implementations mistakenly assume powers must be greater than one. However, mathematically:

$1 = 4^0$

Therefore `1` is a valid power of four, and the implementation correctly returns `true`.
