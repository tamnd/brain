## Solution

## Problem Understanding

The problem requires designing a data structure that allows random selection of indices for a given target value in an array, where duplicates exist. Specifically, given an integer array `nums` that may contain repeated elements, we need to implement a class `Solution` with a constructor and a `pick` method. The `pick` method should randomly return the index of an element equal to the `target`. If there are multiple occurrences, each index must have equal probability of being selected.

The input `nums` is a list of integers, possibly with duplicates, and its length can reach up to 20,000 elements. The `target` is guaranteed to exist in `nums`. The output is a single index such that `nums[index] == target`, chosen uniformly at random. We may call `pick` up to 10,000 times, so efficiency in both time and space is important.

Edge cases include arrays with a single element, arrays where all elements are identical, and targets appearing either at the start, end, or multiple positions in the array. The problem guarantees that the target exists, so we do not need to handle cases where it does not.

## Approaches

The brute-force approach is straightforward: for each call to `pick`, iterate over the entire array, collect all indices where the element equals the target, then select a random index from that list. This guarantees uniform probability because every valid index is considered. However, this approach is inefficient if `pick` is called many times, as each call requires scanning the entire array, resulting in high time complexity.

The optimal approach leverages **Reservoir Sampling**, a probabilistic algorithm for selecting a random element from a stream of unknown or large size. The idea is to iterate through the array once, and for each occurrence of the target, decide probabilistically whether to replace the previously selected index. This ensures uniform probability without storing all indices, using constant space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) per pick | O(n) worst-case | Scans array each pick, stores indices temporarily |
| Reservoir Sampling | O(n) per pick | O(1) | Iterates once per pick, constant space, uniform probability |

## Algorithm Walkthrough

1. **Initialization**: Store the input array `nums` in the `Solution` class. This allows subsequent `pick` operations to access the original array without duplication.
2. **Pick Method**: Initialize two variables, `count` and `result`. `count` tracks the number of times the target has been seen, and `result` stores the currently chosen index.
3. **Iterate Over Array**: Loop through each index `i` in `nums`. If `nums[i]` equals the target, increment `count`.
4. **Reservoir Decision**: For each matching element, generate a random integer between 1 and `count`. If the result is 1, replace `result` with the current index. This guarantees each index is selected with probability 1/count.
5. **Return Result**: After the loop, return the selected index stored in `result`.

**Why it works**: Each occurrence of the target has a 1/count chance of being selected at each step, maintaining uniform probability. Reservoir Sampling ensures that, after iterating through all elements, every index of the target is equally likely.

## Python Solution

```
import random
from typing import List

class Solution:

    def __init__(self, nums: List[int]):
        self.nums = nums

    def pick(self, target: int) -> int:
        count = 0
        result = -1
        for i, num in enumerate(self.nums):
            if num == target:
                count += 1
                if random.randint(1, count) == 1:
                    result = i
        return result
```

In this implementation, the constructor stores the input array. In `pick`, we iterate through `nums`, counting occurrences of `target`. Using `random.randint(1, count) == 1` implements the reservoir sampling probability logic. The selected index is updated only probabilistically, ensuring uniform selection across all valid indices.

## Go Solution

```
package main

import (
	"math/rand"
	"time"
)

type Solution struct {
	nums []int
}

func Constructor(nums []int) Solution {
	rand.Seed(time.Now().UnixNano())
	return Solution{nums: nums}
}

func (this *Solution) Pick(target int) int {
	count := 0
	result := -1
	for i, num := range this.nums {
		if num == target {
			count++
			if rand.Intn(count) == 0 {
				result = i
			}
		}
	}
	return result
}
```

In Go, `rand.Intn(count)` generates a number between 0 and `count-1`. If the result is 0, we replace the previously chosen index. Seeding the random number generator ensures different results across runs. The rest mirrors the Python logic.

## Worked Examples

Example: `nums = [1, 2, 3, 3, 3]`, target = 3.

| i | num | count | random check | result |
| --- | --- | --- | --- | --- |
| 0 | 1 | - | - | -1 |
| 1 | 2 | - | - | -1 |
| 2 | 3 | 1 | rand=1 -> replace | 2 |
| 3 | 3 | 2 | rand=1? -> maybe replace | 2 or 3 |
| 4 | 3 | 3 | rand=1? -> maybe replace | 2, 3, or 4 |

The final index is uniformly selected from all occurrences of 3.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We iterate through the array once per `pick` call. |
| Space | O(1) | No additional storage proportional to input size is used. |

Although the brute-force approach may seem simpler, it requires O(n) extra space for storing indices. Reservoir sampling avoids this entirely.

## Edge Cases

One edge case is an array with a single element. This is trivial but tests if `pick` correctly handles arrays where the only index is the answer. Another is when all elements are identical; the algorithm must select any index uniformly, which is a common source of off-by-one errors. Finally, targets located at the beginning or end of the array could cause naive implementations that prematurely return a value to fail. Reservoir sampling avoids these pitfalls by considering all occurrences equally.

This approach handles all edge cases without extra conditional logic, ensuring correctness and uniform randomness.
