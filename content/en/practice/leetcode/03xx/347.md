## Solution

## Problem Understanding

The problem gives us an integer array `nums` and an integer `k`. Our task is to return the `k` elements that appear most frequently in the array.

The key detail is that we care about frequency, not the numerical value of the elements themselves. For example, in the array `[1,1,1,2,2,3]`, the number `1` appears three times, `2` appears twice, and `3` appears once. Since `k = 2`, we return the two most common elements, which are `[1,2]`.

The output can be returned in any order. That means `[2,1]` is also valid for the first example.

The constraints are important because they guide us toward an efficient solution:

- The array can contain up to `10^5` elements.
- A naive sorting-based approach may become too slow.
- The follow-up explicitly asks for a solution better than `O(n log n)`.

The problem also guarantees that the answer is unique. This means there will never be ambiguity about which elements belong in the top `k`.

Several edge cases are important to recognize early:

- The array may contain only one element.
- All elements may be unique.
- One element may dominate the frequency count.
- Negative numbers are allowed.
- `k` may equal the number of unique elements.

A correct solution must handle all of these cases efficiently.

## Approaches

### Brute Force Approach

The most direct approach is to count the frequency of every number using a hash map, then sort all unique elements by frequency in descending order.

The algorithm works like this:

1. Traverse the array and count occurrences of each number.
2. Convert the frequency map into a list of `(number, frequency)` pairs.
3. Sort the list by frequency descending.
4. Return the first `k` elements.

This approach is correct because sorting guarantees that the most frequent elements appear first.

However, sorting introduces an `O(m log m)` cost, where `m` is the number of unique elements. In the worst case, every element is unique, so `m = n`, giving a total complexity of `O(n log n)`.

The follow-up specifically asks for something better than this complexity.

### Optimal Approach, Bucket Sort

The key observation is that frequencies are bounded.

If the array length is `n`, then no element can appear more than `n` times. This means frequencies lie in the range `[1, n]`.

Instead of sorting elements by frequency, we can group elements into buckets where the bucket index represents the frequency.

For example:

- Bucket `1` contains all numbers appearing once.
- Bucket `2` contains all numbers appearing twice.
- Bucket `3` contains all numbers appearing three times.

After building these buckets, we iterate from the highest frequency bucket down to the lowest and collect elements until we have `k` results.

This avoids sorting entirely and achieves linear time complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | `O(n log n)` | `O(n)` | Count frequencies, then sort by frequency |
| Optimal | `O(n)` | `O(n)` | Use frequency buckets to avoid sorting |

## Algorithm Walkthrough

1. Create a frequency map using a hash table.

We traverse the array once and count how many times each number appears. A hash map is ideal because insertion and lookup are approximately `O(1)`.

Example:

```
nums = [1,1,1,2,2,3]

frequency = {
    1: 3,
    2: 2,
    3: 1
}
```
2. Create buckets indexed by frequency.

Since the maximum possible frequency is `len(nums)`, we create an array of empty lists with size `n + 1`.

```
buckets[frequency] = list of numbers with that frequency
```

After filling:

```
buckets[1] = [3]
buckets[2] = [2]
buckets[3] = [1]
```
3. Traverse the buckets in reverse order.

We start from the highest possible frequency and move downward.

This guarantees we encounter the most frequent elements first.
4. Collect elements until we have `k` results.

Each time we encounter a bucket containing numbers, we append them to the result list.

Once the result size reaches `k`, we stop immediately.
5. Return the result.

Since the problem allows any order, no additional sorting is needed.

### Why it works

The algorithm works because every number is placed into the bucket corresponding exactly to its frequency. Iterating from higher-frequency buckets to lower-frequency buckets guarantees that elements are processed in descending frequency order. Since we stop after collecting `k` elements, the returned list contains exactly the `k` most frequent numbers.

## Python Solution

```
from typing import List
from collections import defaultdict

class Solution:
    def topKFrequent(self, nums: List[int], k: int) -> List[int]:
        # Step 1: Count frequencies
        frequency_map = defaultdict(int)

        for num in nums:
            frequency_map[num] += 1

        # Step 2: Create buckets
        buckets = [[] for _ in range(len(nums) + 1)]

        for num, freq in frequency_map.items():
            buckets[freq].append(num)

        # Step 3: Collect top k frequent elements
        result = []

        for freq in range(len(buckets) - 1, 0, -1):
            for num in buckets[freq]:
                result.append(num)

                if len(result) == k:
                    return result

        return result
```

The implementation begins by constructing a frequency map using `defaultdict(int)`. This allows frequency counting without manually checking whether a key already exists.

Next, the code creates a list of buckets. The index of each bucket corresponds directly to a frequency count. Each bucket stores all numbers that appear that many times.

The algorithm then iterates through the buckets from highest frequency to lowest. Since higher frequencies are processed first, elements are appended to the result in descending frequency order.

As soon as the result contains `k` elements, the function returns immediately. This avoids unnecessary work.

## Go Solution

```
func topKFrequent(nums []int, k int) []int {
    // Step 1: Count frequencies
    frequencyMap := make(map[int]int)

    for _, num := range nums {
        frequencyMap[num]++
    }

    // Step 2: Create buckets
    buckets := make([][]int, len(nums)+1)

    for num, freq := range frequencyMap {
        buckets[freq] = append(buckets[freq], num)
    }

    // Step 3: Collect top k frequent elements
    result := []int{}

    for freq := len(buckets) - 1; freq >= 1; freq-- {
        for _, num := range buckets[freq] {
            result = append(result, num)

            if len(result) == k {
                return result
            }
        }
    }

    return result
}
```

The Go implementation follows the same algorithmic structure as the Python version.

A `map[int]int` is used for frequency counting. Buckets are represented using a slice of integer slices, where each index corresponds to a frequency.

Unlike Python lists, Go slices must be explicitly initialized using `make`. Appending to slices automatically handles dynamic resizing.

There are no integer overflow concerns because the maximum frequency is bounded by `10^5`, which easily fits within Go's integer range.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | `O(n)` | Frequency counting, bucket creation, and traversal are all linear |
| Space | `O(n)` | Frequency map and buckets may together store up to `n` elements |

The algorithm avoids sorting entirely. Every element is processed a constant number of times:

- Once while counting frequencies
- Once while inserting into buckets
- Once while traversing buckets

Because each operation is linear, the total runtime is `O(n)`.

The space complexity is also linear because both the frequency map and bucket structure can collectively store up to `n` unique elements.

## Edge Cases

### Single Element Array

A one-element array is the smallest valid input. A buggy implementation might incorrectly assume multiple unique values exist or mishandle bucket initialization.

For example:

```
nums = [1]
k = 1
```

The implementation handles this correctly because the frequency map contains one entry and bucket traversal immediately returns the single value.

### All Elements Unique

When every element appears exactly once, all numbers belong in the same frequency bucket.

Example:

```
nums = [1,2,3,4]
k = 2
```

A flawed implementation could accidentally rely on distinct frequency levels existing. Our bucket-based solution works correctly because bucket `1` simply contains all numbers, and we return the first `k` encountered.

### k Equals Number of Unique Elements

In this case, the algorithm must return every unique number.

Example:

```
nums = [4,4,6,6,7]
k = 3
```

Some implementations incorrectly stop early or assume `k` is smaller than the number of unique elements. Our solution continues traversing buckets until exactly `k` elements have been collected.

### Negative Numbers

The input allows negative integers, which can expose bugs in solutions that incorrectly use array indexing based on element values.

Example:

```
nums = [-1,-1,-2,-2,-2]
k = 1
```

Because the implementation uses a hash map keyed by integers, negative values work naturally without any special handling.

### Very Large Frequency Differences

One number may appear dramatically more often than all others.

Example:

```
nums = [9,9,9,9,9,1,2,3]
k = 1
```

The reverse bucket traversal ensures the highest-frequency element is encountered first, making this case efficient and correct.
