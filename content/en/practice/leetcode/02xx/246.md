## Solution

## Problem Understanding

The problem asks us to determine whether a given numeric string is **strobogrammatic**. A number is considered strobogrammatic if it appears exactly the same after being rotated **180 degrees**, meaning viewed upside down.

This is not the same as a palindrome. Instead of requiring the digits to remain identical, some digits transform into other valid digits after rotation. For example, `6` becomes `9`, `9` becomes `6`, and `8` remains `8`. A string is strobogrammatic only if every digit transforms correctly and the transformed result matches the original number when viewed from the opposite direction.

The input is a string `num`, representing an integer. We are asked to return a boolean value:

- `true` if the number remains valid and unchanged after a `180` degree rotation
- `false` otherwise

The constraints tell us several useful things:

- `1 <= num.length <= 50`, meaning the input size is very small.
- The string contains only digits (`0-9`).
- No leading zeros are present except for the number `"0"` itself.

Since the maximum length is only `50`, performance is not a major concern. Even an `O(n)` or `O(n²)` solution would be acceptable. However, this problem naturally lends itself to a very efficient linear-time approach.

The most important edge cases involve digits that are invalid under rotation. For example, `2`, `3`, `4`, `5`, and `7` do not map to any valid digit after rotation. Any occurrence of these immediately makes the number non-strobogrammatic. Another important edge case is odd-length strings, where the middle character must rotate to itself, meaning it can only be `0`, `1`, or `8`. Finally, pairs like `6` and `9` require careful handling because they swap positions rather than remaining identical.

## Approaches

### Brute Force Approach

A brute-force strategy would simulate the entire rotation process explicitly.

We could iterate through the string from right to left, rotate each digit using a mapping table, and build a new transformed string. Once the rotated string is complete, we compare it with the original input. If both match, the number is strobogrammatic.

For example:

- Input: `"69"`
- Reverse traversal: `"9"` → `"6"`, `"6"` → `"9"`
- Constructed rotated string: `"69"`
- Compare with original: equal, return `true`

This method is correct because it directly reproduces the rotated number and checks equality. However, it requires creating an additional string, which increases space usage unnecessarily.

### Optimal Approach, Two Pointers with Digit Mapping

The key observation is that we do not actually need to construct the rotated number.

When rotating a number `180` degrees, the left digit must correctly map to the corresponding right digit. This naturally suggests a **two-pointer** strategy:

- One pointer starts at the beginning (`left`)
- One pointer starts at the end (`right`)

At each step, we check whether the left digit maps to the right digit according to valid strobogrammatic pairs:

| Digit | Rotates To |
| --- | --- |
| `0` | `0` |
| `1` | `1` |
| `6` | `9` |
| `8` | `8` |
| `9` | `6` |

If any pair fails this condition, we immediately return `false`.

This works because rotation reverses the order of digits. The first digit must match the rotated version of the last digit, the second digit must match the rotated version of the second-to-last digit, and so on.

Instead of constructing a new string, we validate the structure in place, resulting in an efficient `O(n)` solution with constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | `O(n)` | `O(n)` | Builds a rotated string and compares it |
| Optimal | `O(n)` | `O(1)` | Uses two pointers and a digit mapping |

## Algorithm Walkthrough

1. Create a mapping of valid rotated digit pairs.

We need a quick way to determine how each digit transforms after rotation. A hash map is ideal because it provides constant-time lookup.

```
{
    '0': '0',
    '1': '1',
    '6': '9',
    '8': '8',
    '9': '6'
}
```
2. Initialize two pointers.

Set:

- `left = 0`
- `right = len(num) - 1`

These pointers represent mirrored positions that must correspond after rotation.
3. Compare digits at both ends.

At every iteration:

- Check whether `num[left]` exists in the mapping.
- Verify that `mapping[num[left]] == num[right]`.

If either condition fails, return `false`.
4. Move inward.

After a successful match:

- Increment `left`
- Decrement `right`

Continue until the pointers cross.
5. Return `true`.

If all mirrored digit pairs are valid, the number is strobogrammatic.

### Why it works

The algorithm maintains the invariant that every examined pair of characters correctly mirrors one another under `180` degree rotation. Since rotation reverses position order, validating each mirrored pair guarantees that the fully rotated number would match the original. If every pair satisfies the mapping, the number is strobogrammatic. If even one pair fails, the property is broken, making the answer `false`.

## Python Solution

```
class Solution:
    def isStrobogrammatic(self, num: str) -> bool:
        rotation_map = {
            '0': '0',
            '1': '1',
            '6': '9',
            '8': '8',
            '9': '6'
        }

        left = 0
        right = len(num) - 1

        while left <= right:
            if (
                num[left] not in rotation_map or
                rotation_map[num[left]] != num[right]
            ):
                return False

            left += 1
            right -= 1

        return True
```

The implementation begins by defining a dictionary called `rotation_map`, which stores every valid digit transformation. This structure allows constant-time lookup for rotated digits.

Two pointers are then initialized, one at the beginning and one at the end of the string. Since rotation reverses order, the left character must rotate into the right character.

The `while left <= right` condition ensures we process all mirrored pairs, including the middle digit in odd-length numbers. For example, in `"818"`, the center `1` must map to itself. Digits like `6` or `9` in the middle would fail because they rotate into different digits.

Inside the loop, we first check whether the left digit exists in the mapping. This prevents invalid digits such as `2`, `3`, `4`, `5`, and `7` from being processed. We then verify that the rotated digit matches the right-side digit. If not, we immediately return `False`.

If all comparisons succeed, we move both pointers inward and eventually return `True`.

## Go Solution

```
func isStrobogrammatic(num string) bool {
	rotationMap := map[byte]byte{
		'0': '0',
		'1': '1',
		'6': '9',
		'8': '8',
		'9': '6',
	}

	left := 0
	right := len(num) - 1

	for left <= right {
		rotatedDigit, exists := rotationMap[num[left]]

		if !exists || rotatedDigit != num[right] {
			return false
		}

		left++
		right--
	}

	return true
}
```

The Go implementation follows the same logic as the Python solution, but uses a `map[byte]byte` since strings in Go are indexed as bytes.

Instead of checking membership using `in` like Python, Go uses a two-value map lookup:

```
rotatedDigit, exists := rotationMap[num[left]]
```

The `exists` boolean ensures we handle invalid digits safely. If the digit is not present in the map, the function immediately returns `false`.

No additional memory proportional to input size is allocated, keeping space complexity constant.

## Edge Cases

### Odd-Length Numbers

Odd-length inputs require special handling because the center digit has no mirrored partner. The middle digit must rotate into itself, meaning only `0`, `1`, or `8` are valid.

For example, `"818"` is valid because `1` maps to itself. However, `"696"` is invalid because the center `9` rotates into `6`, which changes the number. The `left <= right` condition ensures the center digit is checked properly.

### Invalid Digits

Digits `2`, `3`, `4`, `5`, and `7` have no valid `180` degree rotation. A naive implementation might accidentally ignore this and produce incorrect results.

For example, `"25"` should immediately return `false`. The hash map membership check prevents invalid digits from being processed.

### Single Character Inputs

A single digit still must satisfy the strobogrammatic property by itself. This can be a source of bugs because `6` and `9` rotate into different digits and therefore fail.

Valid single-digit numbers are:

```
0, 1, 8
```

Invalid single-digit numbers include:

```
6, 9, 2, 3, 4, 5, 7
```

The implementation naturally handles this through the same two-pointer logic without requiring any special-case code.
