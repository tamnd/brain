## Solution

## Problem Understanding

This problem asks us to find the minimum distance between two specific words in a list of strings. The distance is defined as the absolute difference between the indices where the two target words appear in the array.

In other words, we are given an array `wordsDict`, along with two distinct words `word1` and `word2`. Both words are guaranteed to exist in the array, and we need to determine the smallest number of positions separating any occurrence of `word1` from any occurrence of `word2`.

For example, consider:

```
["practice", "makes", "perfect", "coding", "makes"]
```

If `word1 = "coding"` and `word2 = "practice"`, the indices are:

- `"practice"` at index `0`
- `"coding"` at index `3`

The distance is:

```
|3 - 0| = 3
```

Now consider:

```
word1 = "makes"
word2 = "coding"
```

The word `"makes"` appears multiple times, at indices `1` and `4`, while `"coding"` appears at index `3`. We must consider all possible pairs:

```
|1 - 3| = 2
|4 - 3| = 1
```

The shortest distance is `1`.

The constraints tell us important information about the scale of the problem:

- The array length can be as large as `3 * 10^4`, meaning inefficient nested-loop solutions may become expensive.
- Each word is short, at most 10 characters.
- `word1` and `word2` are guaranteed to exist in the input.
- `word1 != word2`, meaning we never have to compare a word with itself.

An important subtlety is that the target words may appear multiple times. A naive implementation might stop after finding the first occurrence of each word, but that could miss a shorter distance later in the array. The algorithm must continuously track occurrences and update the minimum distance whenever a better candidate appears.

## Approaches

### Brute Force Approach

A straightforward approach is to first collect every index where `word1` appears and every index where `word2` appears. After that, compare every possible pair of indices and compute their absolute difference.

For example:

```
wordsDict = ["practice", "makes", "perfect", "coding", "makes"]
word1 = "makes"
word2 = "coding"
```

We would build:

```
makes  -> [1, 4]
coding -> [3]
```

Then compute:

```
|1 - 3| = 2
|4 - 3| = 1
```

The minimum value is returned.

This method works because it explicitly checks every possible combination, guaranteeing the shortest distance is found. However, it becomes inefficient when the words appear many times. If each word appears `n/2` times, the comparison step becomes quadratic.

### Optimal Approach

The key observation is that we do not need to compare every pair of positions.

As we scan through the array from left to right, we only care about the _most recent occurrence_ of each target word. Whenever we encounter one of the words, we update its latest index and immediately compute the distance to the most recent occurrence of the other word.

Why does this work? Because the shortest distance involving the current word must come from the nearest previously seen occurrence of the other word. Older occurrences are always farther away than the latest one.

For example:

```
["practice", "makes", "perfect", "coding", "makes"]
```

When we encounter `"coding"` at index `3`, the latest `"practice"` is at index `0`, so distance is `3`.

Later, when we encounter `"makes"` at index `4`, the latest `"coding"` is at index `3`, giving distance `1`.

This single-pass strategy avoids redundant comparisons and runs in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Store all indices and compare every pair |
| Optimal | O(n) | O(1) | Single pass while tracking latest positions |

## Algorithm Walkthrough

1. Initialize two variables, `last_word1_index` and `last_word2_index`, to `-1`. These variables store the most recent positions where `word1` and `word2` were found. Using `-1` indicates the word has not been seen yet.
2. Initialize a variable `minimum_distance` to positive infinity. This allows us to safely minimize distances as we process the array.
3. Iterate through `wordsDict` using both the index and word value.
4. If the current word matches `word1`, update `last_word1_index` to the current position.
5. If the current word matches `word2`, update `last_word2_index` to the current position.
6. After updating indices, check whether both words have been seen at least once. This means both index variables are no longer `-1`.
7. If both words exist, compute the current distance:

```
abs(last_word1_index - last_word2_index)
```

Compare this value with `minimum_distance` and keep the smaller one.
8. Continue scanning until the end of the array.
9. Return `minimum_distance`.

### Why it works

At any point during traversal, the algorithm stores the latest seen position of each target word. When a new occurrence appears, the nearest possible pairing must involve the most recent occurrence of the other word, because any older occurrence would necessarily be farther away in index distance. By updating the answer every time a relevant word appears, the algorithm guarantees that the global minimum distance is found.

## Python Solution

```
from typing import List

class Solution:
    def shortestDistance(self, wordsDict: List[str], word1: str, word2: str) -> int:
        last_word1_index = -1
        last_word2_index = -1
        minimum_distance = float("inf")

        for index, word in enumerate(wordsDict):
            if word == word1:
                last_word1_index = index

            if word == word2:
                last_word2_index = index

            if last_word1_index != -1 and last_word2_index != -1:
                current_distance = abs(
                    last_word1_index - last_word2_index
                )
                minimum_distance = min(
                    minimum_distance,
                    current_distance
                )

        return minimum_distance
```

The implementation follows the single-pass strategy described earlier.

We begin by storing the latest known index of each target word using `last_word1_index` and `last_word2_index`. Both start at `-1`, indicating the word has not been encountered yet.

The loop iterates through `wordsDict` using `enumerate`, which gives both the current index and word value. Whenever we encounter `word1` or `word2`, we update the corresponding index variable.

After each update, we check whether both words have already appeared. If they have, we compute the current distance between their most recent occurrences and compare it with the best answer found so far.

The use of `float("inf")` ensures that the first valid distance will always replace the initial value.

Because the array is scanned exactly once and only a few variables are maintained, the solution is both time and memory efficient.

## Go Solution

```
import "math"

func shortestDistance(wordsDict []string, word1 string, word2 string) int {
	lastWord1Index := -1
	lastWord2Index := -1
	minimumDistance := math.MaxInt32

	for index, word := range wordsDict {
		if word == word1 {
			lastWord1Index = index
		}

		if word == word2 {
			lastWord2Index = index
		}

		if lastWord1Index != -1 && lastWord2Index != -1 {
			currentDistance := lastWord1Index - lastWord2Index
			if currentDistance < 0 {
				currentDistance = -currentDistance
			}

			if currentDistance < minimumDistance {
				minimumDistance = currentDistance
			}
		}
	}

	return minimumDistance
}
```

The Go implementation follows the same logic as the Python solution but adapts to Go language conventions.

Instead of `float("inf")`, Go uses `math.MaxInt32` as the initial large value for `minimumDistance`.

Go does not have a built-in `abs` function for integers, so the absolute difference is computed manually by checking whether the result is negative and negating it if necessary.

The solution uses a `range` loop over the slice, which provides both the index and word value efficiently. Since Go slices are dynamically sized views over arrays, no additional memory allocation is required.

## Edge Cases

### Multiple occurrences of the target words

A common source of bugs is assuming each word appears only once. For example:

```
["a", "x", "a", "b"]
```

If we stop after the first occurrence of `"a"` and `"b"`, we would return `3`. However, the second `"a"` produces a shorter distance of `1`. The implementation handles this by continuously updating the latest index and recalculating the minimum.

### Target words appearing next to each other

When the words are adjacent:

```
["hello", "world"]
```

the shortest distance is `1`, which is the smallest possible valid answer because the words must occupy different indices. The algorithm correctly detects this immediately after both words have been seen.

### Words appearing at opposite ends of the array

Consider:

```
["start", "middle", "end"]
```

Here the shortest distance is relatively large compared to the array size. A buggy implementation might incorrectly assume nearby occurrences exist or mishandle index arithmetic. Since this solution uses absolute index differences, it correctly computes the distance regardless of how far apart the words are.

### One word appears much more frequently than the other

For example:

```
["a", "x", "x", "x", "x", "b", "a"]
```

One target may appear many times while the other appears rarely. The algorithm remains efficient because it only stores the most recent occurrence of each word instead of comparing every pair of positions. This guarantees linear runtime even for large inputs.
