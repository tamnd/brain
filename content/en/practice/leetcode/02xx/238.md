## Solution

## Problem Understanding

The problem asks us to construct a new array where each position contains the product of every element in the original array except the element at the same index.

For example, if the input array is `[1,2,3,4]`, then:

- The result for index `0` should be `2 * 3 * 4 = 24`
- The result for index `1` should be `1 * 3 * 4 = 12`
- The result for index `2` should be `1 * 2 * 4 = 8`
- The result for index `3` should be `1 * 2 * 3 = 6`

So the final output becomes `[24,12,8,6]`.

A very important restriction is that we are not allowed to use division. A common shortcut would be to compute the total product of all numbers and divide by `nums[i]`, but the problem explicitly forbids that approach.

The constraints also matter significantly:

- `nums.length` can be as large as `10^5`
- An `O(n^2)` solution would be far too slow at this scale
- We need an `O(n)` solution
- The follow-up asks for `O(1)` extra space, excluding the output array

The problem guarantees that the product of any prefix or suffix fits inside a 32-bit integer. This means we do not need to worry about overflow handling beyond normal integer operations in Python or Go.

Several edge cases are especially important:

- Arrays containing zeros
- Arrays containing multiple zeros
- Negative numbers
- Very small arrays such as length `2`

Zeros are particularly tricky because division-based solutions break down completely when zeros are present. The optimal prefix/suffix method handles zeros naturally without requiring special-case logic.

## Approaches

### Brute Force Approach

The most straightforward solution is to compute the answer for each index independently.

For every position `i`, we iterate through the entire array and multiply every value except `nums[i]`.

For example, for index `2` in `[1,2,3,4]`, we compute:

```
1 * 2 * 4 = 8
```

We repeat this process for every index.

This approach is correct because each output value is explicitly calculated according to the problem definition. However, it is inefficient because for every element we scan the entire array again.

If the array has `n` elements:

- We perform `n` computations
- Each computation takes `O(n)` time
- Total complexity becomes `O(n^2)`

With `n` up to `100000`, this is too slow.

### Optimal Prefix and Suffix Product Approach

The key observation is that the product except self at index `i` can be separated into two independent parts:

```
(product of all elements to the left of i)
*
(product of all elements to the right of i)
```

For example, in:

```
[1,2,3,4]
```

At index `2`:

```
left product  = 1 * 2 = 2
right product = 4
answer = 2 * 4 = 8
```

Instead of recomputing these products repeatedly, we can build them incrementally.

We first compute prefix products:

- `prefix[i]` stores the product of all elements before index `i`

Then we compute suffix products while traversing from right to left:

- `suffix` stores the running product of elements to the right

By multiplying the left product and right product together, we get the correct answer for every index in linear time.

This eliminates redundant multiplication work and achieves `O(n)` complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Recomputes products for every index |
| Optimal | O(n) | O(1) excluding output | Uses prefix and suffix products efficiently |

## Algorithm Walkthrough

1. Create an output array initialized with `1`s.

We will reuse this array to store prefix products first, and later combine them with suffix products. Initializing with `1` works because `1` is the multiplicative identity.
2. Traverse the array from left to right while maintaining a running prefix product.

At each index:

- Store the current prefix product into `answer[i]`
- Multiply the prefix product by `nums[i]`

After this pass, `answer[i]` contains the product of all elements strictly to the left of index `i`.
3. Traverse the array from right to left while maintaining a running suffix product.

At each index:

- Multiply `answer[i]` by the current suffix product
- Update the suffix product by multiplying with `nums[i]`

Now `answer[i]` becomes:

```
left product * right product
```

which is exactly the desired result.
4. Return the output array.

### Why it works

The algorithm works because every element except `nums[i]` can be partitioned into two non-overlapping groups:

- Elements before `i`
- Elements after `i`

The first pass computes the product of all elements before each index. The second pass computes the product of all elements after each index. Multiplying these together gives the product of all elements except the current one.

The two passes together ensure every element contributes exactly once to every valid position and never contributes to itself.

## Python Solution

```
from typing import List

class Solution:
    def productExceptSelf(self, nums: List[int]) -> List[int]:
        n = len(nums)
        
        answer = [1] * n
        
        prefix_product = 1
        for i in range(n):
            answer[i] = prefix_product
            prefix_product *= nums[i]
        
        suffix_product = 1
        for i in range(n - 1, -1, -1):
            answer[i] *= suffix_product
            suffix_product *= nums[i]
        
        return answer
```

The implementation follows the exact algorithm described earlier.

The first section initializes the result array with `1`s. This array will eventually hold the final answers.

The first loop computes prefix products. Before multiplying by `nums[i]`, we store the current running product in `answer[i]`. This guarantees that the current element itself is excluded.

For example:

```
nums = [1,2,3,4]
```

After the prefix pass:

```
answer = [1,1,2,6]
```

Each position now contains the product of all elements to its left.

The second loop processes the array from right to left. The variable `suffix_product` keeps track of the product of all elements to the right.

We multiply the existing prefix product in `answer[i]` by the suffix product, producing the final result for that index.

This approach avoids extra arrays and satisfies the `O(1)` auxiliary space requirement.

## Go Solution

```
func productExceptSelf(nums []int) []int {
    n := len(nums)

    answer := make([]int, n)

    prefixProduct := 1
    for i := 0; i < n; i++ {
        answer[i] = prefixProduct
        prefixProduct *= nums[i]
    }

    suffixProduct := 1
    for i := n - 1; i >= 0; i-- {
        answer[i] *= suffixProduct
        suffixProduct *= nums[i]
    }

    return answer
}
```

The Go implementation mirrors the Python logic very closely.

A slice is allocated using `make([]int, n)`. Since Go initializes integer slices to zero by default, we explicitly overwrite each position during the prefix pass.

Unlike Python, Go uses fixed integer types. However, the problem guarantees that the results fit within a 32-bit integer, so normal `int` operations are sufficient.

Go slices are reference-backed structures, but since we only modify the local `answer` slice and return it directly, there are no aliasing concerns.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Two linear passes through the array |
| Space | O(1) | Only constant extra variables are used, excluding output |

The algorithm performs exactly two traversals of the array, one from left to right and one from right to left. Each traversal performs constant work per element, resulting in linear time complexity.

The only extra variables used are the running prefix and suffix products. Since the output array does not count toward auxiliary space according to the problem statement, the extra space complexity is `O(1)`.

## Edge Cases

One important edge case is when the array contains exactly one zero. In this situation, every position except the zero's index must become zero because every product includes that zero. The position corresponding to the zero should contain the product of all non-zero elements. Division-based solutions often fail here, but the prefix/suffix method naturally produces the correct result because the zero only affects products that actually include it.

Another critical edge case is when the array contains multiple zeros. If there are two or more zeros, then every product except self necessarily includes at least one zero, meaning the entire output array must be all zeros. The prefix/suffix approach handles this automatically without requiring any special branching logic.

Negative numbers are also a common source of mistakes. The sign of each result depends on how many negative numbers are included in the multiplication. Since the algorithm performs ordinary multiplication without shortcuts or division tricks, it naturally preserves the correct sign behavior for all combinations of positive and negative values.

A final important edge case is the minimum allowed array size of `2`. Some implementations accidentally assume larger arrays or make off-by-one mistakes during prefix or suffix traversal. Initializing prefix and suffix products to `1` ensures the logic still works correctly even when there are only two elements.
