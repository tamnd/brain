## Solution

## Problem Understanding

This problem asks us to compute the minimum distance between two words in a list of strings. The distance is defined as the absolute difference between the indices where the two words appear in the array.

We are given three inputs:

- `wordsDict`, an array of strings representing a sequence of words
- `word1`, the first target word
- `word2`, the second target word

The goal is to return the smallest index difference between occurrences of `word1` and `word2`.

The important twist in this problem is that `word1` and `word2` may be the same word. This changes the logic significantly compared to simpler versions of the problem, such as LeetCode 243 or 244.

If the words are different, the task is straightforward: find the closest pair of positions where the two words occur.

For example:

```
["practice", "makes", "perfect", "coding", "makes"]
```

If `word1 = "makes"` and `word2 = "coding"`:

- `"makes"` appears at indices `1` and `4`
- `"coding"` appears at index `3`

The distances are:

- `|1 - 3| = 2`
- `|4 - 3| = 1`

The shortest distance is `1`.

However, when `word1 == word2`, we are looking for the minimum distance between **two separate occurrences of the same word**.

For example:

```
["practice", "makes", "perfect", "coding", "makes"]
```

If both words are `"makes"`:

- occurrences are at indices `1` and `4`
- distance is `4 - 1 = 3`

The answer is `3`.

The constraints are important:

- `wordsDict.length <= 10^5`
- each word length is small
- both target words are guaranteed to exist

Since the input can contain up to 100,000 words, an inefficient nested loop approach will likely be too slow. We should aim for a linear time solution.

There are also several edge cases to consider upfront.

When `word1 == word2`, we must avoid accidentally comparing a word occurrence with itself, because the distance would incorrectly become `0`. We need two distinct positions.

Another tricky case is when the matching words appear many times throughout the array. A naive solution that checks every pair would become inefficient.

The problem also guarantees that both words exist in `wordsDict`, which means we do not need to handle missing targets.

## Approaches

### Brute Force Approach

The brute force method is to first collect every index where `word1` appears and every index where `word2` appears. Then, compare every possible pair of indices and compute their absolute difference.

If `word1 != word2`, we compare every occurrence of the first word against every occurrence of the second word.

If `word1 == word2`, we compare every pair of distinct occurrences of the same word and find the minimum distance.

This approach works because it explicitly checks every valid possibility, guaranteeing that the minimum distance is found.

However, it is inefficient. In the worst case, if both words appear many times, we may end up comparing nearly every pair of positions. For example, if the array contains repeated target words across most of the input, the number of comparisons can grow quadratically.

Since `wordsDict` can contain up to `10^5` words, this approach is too slow.

### Optimal Approach

The key insight is that we only need to remember the **most recent occurrence** of the relevant word while scanning the array from left to right.

Suppose we encounter an occurrence of one target word. The closest possible match can only come from the most recent occurrence of the other target word. Earlier occurrences are farther away because indices only increase as we move forward.

This observation allows us to process the array in one pass.

We maintain previous indices:

- If `word1 != word2`, we track the latest position of each word separately.
- If `word1 == word2`, we treat consecutive occurrences as candidates and compute distances between adjacent appearances.

Because we only scan once and store a few variables, this solution runs in linear time and constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Stores indices and compares all pairs |
| Optimal | O(n) | O(1) | Single pass using most recent indices |

## Algorithm Walkthrough

### Optimal Algorithm

1. Initialize a variable `minimum_distance` to positive infinity. This will keep track of the best answer found so far.
2. Determine whether `word1` and `word2` are the same word. This affects how we track indices.
3. If the words are different, maintain two variables:

- `last_word1_index`
- `last_word2_index`

These store the latest seen positions of each word.
4. Traverse `wordsDict` from left to right using both index and word.
5. If the current word matches `word1`, update `last_word1_index`.
6. If the current word matches `word2`, update `last_word2_index`.
7. After updating either index, if both positions are valid, compute the distance:

```
abs(last_word1_index - last_word2_index)
```

Update `minimum_distance` if this value is smaller.
8. If `word1 == word2`, use a different strategy:

- Maintain only one variable, `previous_index`
- Whenever the target word appears again, compute the distance from the previous occurrence
- Update the minimum distance
- Move `previous_index` forward
9. After finishing the traversal, return `minimum_distance`.

### Why it works

The algorithm works because, during a left to right scan, the nearest possible match for the current occurrence must be the most recently seen matching word. Any earlier occurrence would have a larger index difference.

When `word1 != word2`, keeping only the latest index of each word is sufficient to always test the closest candidate pair.

When `word1 == word2`, the shortest distance must occur between consecutive appearances of the same word. Non-consecutive occurrences are always farther apart than at least one adjacent pair. Therefore, checking consecutive matches guarantees correctness.

## Python Solution

```
from typing import List

class Solution:
    def shortestWordDistance(
        self,
        wordsDict: List[str],
        word1: str,
        word2: str
    ) -> int:
        minimum_distance = float("inf")

        # Case when both words are the same
        if word1 == word2:
            previous_index = -1

            for index, word in enumerate(wordsDict):
                if word == word1:
                    if previous_index != -1:
                        minimum_distance = min(
                            minimum_distance,
                            index - previous_index
                        )

                    previous_index = index

            return minimum_distance

        # Case when words are different
        last_word1_index = -1
        last_word2_index = -1

        for index, word in enumerate(wordsDict):
            if word == word1:
                last_word1_index = index

            if word == word2:
                last_word2_index = index

            if (
                last_word1_index != -1 and
                last_word2_index != -1
            ):
                minimum_distance = min(
                    minimum_distance,
                    abs(last_word1_index - last_word2_index)
                )

        return minimum_distance
```

The implementation separates the logic into two cases because the same-word scenario behaves differently from the different-word scenario.

When the words are identical, we maintain `previous_index`, which stores the last occurrence of the word. Every time we see the word again, we calculate the distance from the previous occurrence and update the answer. Since consecutive occurrences are sufficient for correctness, no additional tracking is necessary.

When the words are different, we maintain separate indices for each target word. As we scan through the list, we continuously update the most recent occurrence of each word. Once both have been seen, we calculate the distance and keep the minimum.

The use of sentinel values (`-1`) helps determine whether a word has already been encountered.

## Go Solution

```
import "math"

func shortestWordDistance(wordsDict []string, word1 string, word2 string) int {
	minDistance := math.MaxInt32

	// Case when both words are the same
	if word1 == word2 {
		prevIndex := -1

		for i, word := range wordsDict {
			if word == word1 {
				if prevIndex != -1 {
					distance := i - prevIndex
					if distance < minDistance {
						minDistance = distance
					}
				}

				prevIndex = i
			}
		}

		return minDistance
	}

	// Case when words are different
	lastWord1Index := -1
	lastWord2Index := -1

	for i, word := range wordsDict {
		if word == word1 {
			lastWord1Index = i
		}

		if word == word2 {
			lastWord2Index = i
		}

		if lastWord1Index != -1 && lastWord2Index != -1 {
			distance := lastWord1Index - lastWord2Index
			if distance < 0 {
				distance = -distance
			}

			if distance < minDistance {
				minDistance = distance
			}
		}
	}

	return minDistance
}
```

The Go implementation follows the same logic as the Python solution. Since Go does not provide a built in `abs` function for integers, the absolute difference is computed manually.

Instead of Python's `float("inf")`, the implementation uses `math.MaxInt32` as the initial maximum distance.

Go slices naturally handle empty and populated collections, and since the problem guarantees valid input, no special nil handling is required.

## Edge Cases

### When `word1 == word2`

This is the most important edge case in the problem. A naive implementation might accidentally compare a word occurrence with itself and return `0`, which is invalid because the problem requires two distinct occurrences.

The implementation handles this by tracking only the previous occurrence and calculating distances between consecutive appearances. This guarantees that two separate indices are used.

### Adjacent target words

If the target words appear next to each other, the shortest distance is `1`, which is the minimum possible valid answer.

For example:

```
["a", "b"]
```

A buggy implementation could overlook immediate neighbors if index updates happen in the wrong order. The solution updates indices during traversal and immediately computes distances, ensuring adjacent words are detected correctly.

### Many repeated occurrences

When words appear frequently, a brute force solution becomes expensive because it compares many index pairs.

For example:

```
["a", "x", "a", "x", "a", "b"]
```

The optimal implementation avoids unnecessary comparisons by storing only the latest relevant index. This works because earlier indices can never produce a smaller distance than the most recent occurrence.

### Closest match appears late

A solution that prematurely assumes the first valid pair is optimal would fail.

For example:

```
["a", "x", "x", "b", "a", "b"]
```

The first distance is `3`, but a later pair gives distance `1`.

The implementation continues scanning the entire array and updates the minimum whenever a smaller distance is found, ensuring the globally optimal answer is returned.
