# LeetCode 283, Move Zeroes

## Problem Understanding

The problem gives an integer array `nums` and asks us to move every `0` value to the end of the array while preserving the relative order of all non-zero elements.

The key detail is that the operation must be performed in-place. This means we cannot create another array and return it. Instead, we must directly modify the original array.

For example:

```
Input:  [0,1,0,3,12]
Output: [1,3,12,0,0]
```

The non-zero values `1`, `3`, and `12` remain in the same relative order as before. All zeros are simply shifted to the back.

The input is an integer array with length between `1` and `10^4`. Since the input size is moderate, an `O(n)` solution is ideal, though even `O(n^2)` might technically pass. However, the follow-up question specifically asks us to minimize operations, which strongly hints that an efficient linear-time in-place solution is expected.

Several important edge cases must be considered:

- Arrays that already contain no zeroes, such as `[1,2,3]`
- Arrays consisting entirely of zeroes, such as `[0,0,0]`
- Arrays with only one element
- Zeroes appearing at the beginning, middle, or end
- Multiple consecutive zeroes

A naive implementation can easily overwrite values incorrectly or fail to preserve the relative ordering of non-zero elements.

## Approaches

### Brute Force Approach

A straightforward approach is to create a temporary array containing all non-zero elements first, then append the required number of zeroes at the end.

For example:

```
[0,1,0,3,12]
```

First collect:

```
[1,3,12]
```

Then append two zeroes:

```
[1,3,12,0,0]
```

Finally, copy the result back into the original array.

This approach is easy to understand and guarantees correctness because it explicitly rebuilds the array in the required order. However, it uses extra memory proportional to the size of the array, which violates the spirit of the in-place requirement.

### Optimal Two Pointers Approach

The key observation is that we only care about maintaining the order of non-zero elements. We can process the array from left to right and keep track of the next position where a non-zero value should be placed.

We use two pointers:

- One pointer iterates through the array
- Another pointer tracks the position where the next non-zero element belongs

Whenever we encounter a non-zero value, we swap it into the correct position.

This works because:

- Every non-zero element is moved forward exactly when needed
- Relative order is preserved because elements are processed left to right
- Zeroes naturally accumulate at the end

This produces an in-place `O(n)` solution with `O(1)` extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Uses an additional array to rebuild the result |
| Optimal | O(n) | O(1) | Uses two pointers and modifies the array in-place |

## Algorithm Walkthrough

1. Initialize a pointer called `insert_position` to `0`.

This pointer represents the index where the next non-zero element should be placed.
2. Iterate through the array from left to right.

For each element, check whether it is non-zero.
3. If the current element is non-zero, swap it with the element at `insert_position`.

This moves the non-zero value into its correct compacted location near the front of the array.
4. Increment `insert_position`.

After placing a non-zero element, the next non-zero value should go into the following slot.
5. Continue until the entire array has been processed.

By the end:

- All non-zero elements appear at the front in original order
- All zeroes have been pushed to the back

### Why it works

The algorithm maintains an important invariant:

- All positions before `insert_position` contain the non-zero elements seen so far, in their original relative order.

As we scan from left to right, every non-zero element is placed into the next available slot. Since elements are processed in order, their relative ordering is preserved. Any skipped positions naturally become occupied by zeroes after swaps occur.

## Python Solution

```
from typing import List

class Solution:
    def moveZeroes(self, nums: List[int]) -> None:
        """
        Do not return anything, modify nums in-place instead.
        """
        
        insert_position = 0
        
        for current_index in range(len(nums)):
            if nums[current_index] != 0:
                nums[insert_position], nums[current_index] = (
                    nums[current_index],
                    nums[insert_position],
                )
                insert_position += 1
```

The implementation follows the two-pointer strategy directly.

The variable `insert_position` tracks where the next non-zero value should be placed. Initially, it starts at index `0`.

The loop iterates through every element using `current_index`.

Whenever a non-zero value is found, it is swapped into the correct location at `insert_position`. After the swap, `insert_position` is incremented because that slot is now occupied correctly.

If the current element is already in the correct location, the swap still works safely because swapping an element with itself has no effect.

The algorithm modifies the array directly and does not return anything, matching the problem requirements exactly.

## Go Solution

```
func moveZeroes(nums []int) {
	insertPosition := 0

	for currentIndex := 0; currentIndex < len(nums); currentIndex++ {
		if nums[currentIndex] != 0 {
			nums[insertPosition], nums[currentIndex] =
				nums[currentIndex], nums[insertPosition]

			insertPosition++
		}
	}
}
```

The Go implementation is nearly identical to the Python version.

Go slices are mutable references to underlying arrays, so modifications inside the function affect the original input directly. No additional memory allocation is required.

Unlike Python, Go does not require explicit type hints because the slice type `[]int` already specifies integer elements.

Integer overflow is not a concern here because the algorithm only performs swaps and index operations.

## Edge Cases

### Arrays With No Zeroes

An input like `[1,2,3]` can expose bugs in implementations that unnecessarily rearrange elements.

In this algorithm, every non-zero element swaps with itself because `insert_position` always equals the current index. The array remains unchanged, which is the correct behavior.

### Arrays Containing Only Zeroes

An input like `[0,0,0]` is important because no swaps should occur.

The algorithm handles this naturally. Since no non-zero values are found, `insert_position` never moves, and the array stays exactly the same.

### Consecutive Zeroes

Inputs such as `[1,0,0,2,3]` can be tricky because naive shifting approaches may repeatedly move elements inefficiently.

The two-pointer solution avoids repeated shifting entirely. It simply places each non-zero value into the next available position. This keeps the algorithm efficient while preserving ordering correctly.

### Single Element Arrays

Arrays like `[0]` or `[5]` are boundary cases because the loop executes only once.

The implementation handles both correctly:

- `[0]` remains unchanged
- `[5]` swaps with itself harmlessly

No special-case logic is required.
