## Solution

## Problem Understanding

The problem asks us to design a reusable data structure that can efficiently answer multiple shortest distance queries between words in a given list of strings.

We are given an array called `wordsDict`, where each element is a word. The array represents the order in which the words appear. The distance between two words is defined as the absolute difference between their indices in the array.

For example, in the array:

```
["practice", "makes", "perfect", "coding", "makes"]
```

the word `"coding"` appears at index `3` and `"practice"` appears at index `0`, so the distance is:

```
|3 - 0| = 3
```

The challenge is that the class must support many repeated queries. A naive solution that scans the entire array every time would work functionally, but it would become inefficient as the number of queries grows.

The constraints are important:

- The array length can be as large as `3 * 10^4`
- There can be up to `5000` calls to `shortest`
- Every queried word is guaranteed to exist
- `word1 != word2`

These constraints strongly suggest that preprocessing the input once is beneficial. Since the same dictionary is queried repeatedly, we should store useful information during initialization to avoid repeated scanning.

An important observation is that each word may appear multiple times. A naive implementation might stop after finding the first occurrence of each word, but that could miss a shorter pair later in the array. We must consider all occurrences.

Another subtle point is that the queried words are always different, which simplifies the logic because we never need to compare a word with itself.

## Approaches

### Brute Force Approach

The simplest approach is to process each query independently by scanning the entire array.

For every query:

- Traverse `wordsDict`
- Record all indices where `word1` appears
- Record all indices where `word2` appears
- Compare every pair of indices
- Return the minimum absolute difference

This approach is correct because it checks every possible occurrence pair, so it cannot miss the optimal answer.

However, it is inefficient because the same array is scanned repeatedly for every query. Since there may be thousands of queries, repeatedly traversing the full array becomes expensive.

If `word1` appears `m` times and `word2` appears `n` times, comparing every pair costs `O(m * n)` after the scan.

### Optimal Approach

The key insight is that the dictionary never changes after initialization.

Instead of scanning the entire array for every query, we can preprocess the dictionary once and store the positions of every word in a hash map.

For example:

```
"makes"   -> [1, 4]
"coding"  -> [3]
"practice"-> [0]
```

These index lists are naturally sorted because we build them while traversing the array from left to right.

Once we have two sorted lists of indices, we can use the two-pointer technique to efficiently find the minimum distance between them.

Why does this work?

Because both lists are sorted:

- If one index is smaller, moving the smaller pointer forward is the only way to potentially reduce the distance.
- Moving the larger pointer would only increase or maintain the gap.

This transforms the comparison from `O(m * n)` into `O(m + n)`.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N + m * n) per query | O(m + n) | Repeatedly scans the entire array |
| Optimal | O(N) preprocessing, O(m + n) per query | O(N) | Uses hash map and two pointers |

## Algorithm Walkthrough

1. During initialization, create a hash map where:

- The key is a word
- The value is a list of indices where that word appears
2. Traverse the input array once using index `i`.

- For each word, append `i` to its corresponding list.
- Since indices are added in ascending order, each list remains sorted automatically.
3. When `shortest(word1, word2)` is called:

- Retrieve the sorted index lists for both words.
- Example:

```
word1_indices = [1, 4]
word2_indices = [3]
```
4. Initialize two pointers:

- `i = 0`
- `j = 0`
5. Compare the current indices:

- Compute:

```
abs(word1_indices[i] - word2_indices[j])
```
- Update the minimum distance if needed.
6. Move the pointer pointing to the smaller index.

- If:

```
word1_indices[i] < word2_indices[j]
```

increment `i`
- Otherwise increment `j`
7. Continue until one pointer reaches the end of its list.
8. Return the minimum distance found.

### Why it works

The correctness relies on the sorted nature of the index lists.

At every step, the current pair represents the closest possible candidate involving those two current indices. If one index is smaller, the only way to potentially reduce the distance is to move that smaller index forward. Moving the larger index would only keep the difference the same or make it larger.

Because every potentially optimal pair is considered through this ordered traversal, the algorithm always finds the true minimum distance.

## Python Solution

```
from collections import defaultdict
from typing import List

class WordDistance:

    def __init__(self, wordsDict: List[str]):
        self.word_positions = defaultdict(list)

        for index, word in enumerate(wordsDict):
            self.word_positions[word].append(index)

    def shortest(self, word1: str, word2: str) -> int:
        positions1 = self.word_positions[word1]
        positions2 = self.word_positions[word2]

        pointer1 = 0
        pointer2 = 0

        minimum_distance = float("inf")

        while pointer1 < len(positions1) and pointer2 < len(positions2):
            index1 = positions1[pointer1]
            index2 = positions2[pointer2]

            minimum_distance = min(
                minimum_distance,
                abs(index1 - index2)
            )

            if index1 < index2:
                pointer1 += 1
            else:
                pointer2 += 1

        return minimum_distance

# Your WordDistance object will be instantiated and called as such:
# obj = WordDistance(wordsDict)
# param_1 = obj.shortest(word1,word2)
```

The implementation begins by preprocessing the input array inside the constructor. A `defaultdict(list)` is used because each word may appear multiple times, and we want to store all indices associated with that word.

As the array is traversed with `enumerate`, every index is appended to the appropriate list. Since indices are encountered in increasing order, the lists remain sorted automatically without additional work.

Inside the `shortest` method, we retrieve the two sorted index lists. The two-pointer technique is then used to efficiently search for the minimum distance.

The pointers always move forward, never backward. This guarantees that each index is processed at most once, which gives linear complexity relative to the sizes of the two lists.

The variable `minimum_distance` tracks the best answer found so far and is updated whenever a smaller difference is discovered.

## Go Solution

```
package main

import "math"

type WordDistance struct {
	wordPositions map[string][]int
}

func Constructor(wordsDict []string) WordDistance {
	wordPositions := make(map[string][]int)

	for index, word := range wordsDict {
		wordPositions[word] = append(wordPositions[word], index)
	}

	return WordDistance{
		wordPositions: wordPositions,
	}
}

func (this *WordDistance) Shortest(word1 string, word2 string) int {
	positions1 := this.wordPositions[word1]
	positions2 := this.wordPositions[word2]

	pointer1 := 0
	pointer2 := 0

	minimumDistance := math.MaxInt32

	for pointer1 < len(positions1) && pointer2 < len(positions2) {
		index1 := positions1[pointer1]
		index2 := positions2[pointer2]

		difference := abs(index1 - index2)

		if difference < minimumDistance {
			minimumDistance = difference
		}

		if index1 < index2 {
			pointer1++
		} else {
			pointer2++
		}
	}

	return minimumDistance
}

func abs(value int) int {
	if value < 0 {
		return -value
	}
	return value
}

/**
 * Your WordDistance object will be instantiated and called as such:
 * obj := Constructor(wordsDict);
 * param_1 := obj.Shortest(word1,word2);
 */
```

The Go implementation closely mirrors the Python version, but there are a few language-specific details.

Go does not provide a built-in absolute value function for integers, so a custom `abs` helper function is included.

The hash map uses the type:

```
map[string][]int
```

where each word maps to a slice of indices.

Instead of Python's `float("inf")`, Go uses `math.MaxInt32` as the initial large value for the minimum distance.

Slices in Go behave dynamically like Python lists, so appending indices is straightforward with `append`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(N) preprocessing, O(m + n) per query | Each pointer moves at most once through its list |
| Space | O(N) | All word indices are stored |

The preprocessing step traverses the entire dictionary once, so it costs `O(N)`.

For each query, the two-pointer traversal processes each index list linearly. Since each pointer only moves forward, the total work is proportional to the combined lengths of the two lists.

The space complexity is `O(N)` because every index from the original array is stored exactly once in the hash map.

## Edge Cases

One important edge case occurs when both queried words appear many times throughout the array. A buggy implementation might compare only the first occurrences and miss a smaller distance later. The two-pointer approach avoids this problem because it systematically examines candidate pairs across both sorted index lists.

Another important case is when one word appears only once while the other appears many times. Some implementations incorrectly assume balanced list sizes or rely on symmetric traversal logic. The algorithm here works correctly because the pointers advance independently and terminate safely when either list ends.

A subtle edge case happens when the shortest distance comes from later occurrences rather than earlier ones. For example:

```
["a", "x", "b", "x", "a", "b"]
```

The first `"a"` and `"b"` are distance `2`, but the later pair has distance `1`. A naive greedy implementation might incorrectly return the first valid pair. The two-pointer traversal continues through the lists until all meaningful comparisons are exhausted, guaranteeing the true minimum is found.

Another edge case involves adjacent words. Since the minimum possible valid distance is `1`, implementations must correctly handle neighboring indices without off-by-one errors. Because the solution directly computes absolute index differences, adjacent words naturally produce the correct answer.
