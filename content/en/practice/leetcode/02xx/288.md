## Solution

## Problem Understanding

This problem asks us to design a data structure that can determine whether a word’s abbreviation is unique within a given dictionary.

The abbreviation rule is straightforward:

- If the word length is greater than 2:

- Take the first character.
- Count the number of characters between the first and last character.
- Append the last character.
- If the word length is 2 or less:

- The abbreviation is the word itself.

For example:

- `"dog"` becomes `"d1g"`
- `"localization"` becomes `"l10n"`
- `"it"` stays `"it"`

The class must support two operations:

1. Initialization with a dictionary of words.
2. A query method `isUnique(word)` that checks whether the abbreviation of `word` is unique.

The uniqueness rule is subtle and very important:

A word is considered unique if:

- No dictionary word has the same abbreviation, or
- Every dictionary word with the same abbreviation is exactly the same word.

This means duplicate copies of the same word in the dictionary do not invalidate uniqueness.

For example:

- Dictionary: `["cake"]`
- Query: `"cake"`

The abbreviation `"c2e"` exists, but only for the exact same word `"cake"`, so the answer is `true`.

However:

- Dictionary: `["cake"]`
- Query: `"cane"`

Both abbreviate to `"c2e"`, but the words differ, so the answer is `false`.

The constraints are important:

- Up to `3 * 10^4` dictionary words
- Up to `5000` calls to `isUnique`

A brute-force scan through the entire dictionary for every query would be too slow in the worst case. We need preprocessing so each query can be answered efficiently.

Several edge cases are important:

- Duplicate words in the dictionary
- Very short words of length 1 or 2
- Multiple different words sharing the same abbreviation
- Queries for words not present in the dictionary
- Queries for words already in the dictionary

The problem guarantees all words contain only lowercase English letters, so we do not need to handle uppercase characters or special symbols.

## Approaches

### Brute Force Approach

The most direct solution is:

1. Store the entire dictionary.
2. For every `isUnique(word)` query:

- Compute the abbreviation of `word`.
- Scan every dictionary word.
- Compute each dictionary word’s abbreviation.
- If another word shares the same abbreviation and is not equal to `word`, return `false`.
- Otherwise return `true`.

This works because it directly follows the definition of uniqueness.

However, it is inefficient. If the dictionary contains `N` words, every query requires scanning all `N` words. With up to `5000` queries and `30000` dictionary entries, this becomes expensive.

### Optimal Approach

The key observation is that the only thing that matters is:

- The abbreviation
- Which words produce that abbreviation

We can preprocess the dictionary into a hash map:

```
abbreviation -> set of words
```

For example:

```
"d2r" -> {"deer", "door"}
"c2e" -> {"cake"}
```

Then `isUnique(word)` becomes very fast:

1. Compute the abbreviation.
2. Look up the abbreviation in the map.
3. If no entry exists, return `true`.
4. If the set contains only `word`, return `true`.
5. Otherwise return `false`.

Hash maps give average `O(1)` lookup time, making the solution efficient even for large inputs.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N) per query | O(N) | Scan entire dictionary for every query |
| Optimal | O(N) preprocessing, O(1) per query | O(N) | Hash map from abbreviation to word set |

## Algorithm Walkthrough

1. Create a helper function to generate abbreviations.

If the word length is less than or equal to 2, return the word itself. Otherwise construct:

```
first letter + middle count + last letter
```

For example:

```
"international" -> "i11l"
```
2. During initialization, build a hash map.

The key is the abbreviation.

The value is a set of all dictionary words that share that abbreviation.

Example:

```
dictionary = ["deer", "door", "cake"]

map = {
    "d2r": {"deer", "door"},
    "c2e": {"cake"}
}
```
3. When `isUnique(word)` is called, compute its abbreviation.
4. Check whether the abbreviation exists in the map.

If it does not exist, then no dictionary word shares the abbreviation, so return `true`.
5. If the abbreviation exists, retrieve the associated set.
6. The word is unique only if the set contains exactly one word and that word equals the query word.

Examples:

```
{"cake"} + query "cake" -> true
{"cake"} + query "cane" -> false
{"deer", "door"} + query "dear" -> false
```

### Why it works

The algorithm works because the hash map stores exactly the information required by the problem definition: all dictionary words grouped by abbreviation.

For a queried word, only words with the same abbreviation matter. If none exist, the abbreviation is unique. If the only matching word is the word itself, uniqueness still holds. Any different word with the same abbreviation violates uniqueness.

Therefore the algorithm always produces the correct result.

## Python Solution

```
from typing import List
from collections import defaultdict

class ValidWordAbbr:

    def __init__(self, dictionary: List[str]):
        self.abbr_map = defaultdict(set)

        for word in dictionary:
            abbreviation = self._get_abbreviation(word)
            self.abbr_map[abbreviation].add(word)

    def _get_abbreviation(self, word: str) -> str:
        if len(word) <= 2:
            return word

        return f"{word[0]}{len(word) - 2}{word[-1]}"

    def isUnique(self, word: str) -> bool:
        abbreviation = self._get_abbreviation(word)

        if abbreviation not in self.abbr_map:
            return True

        words = self.abbr_map[abbreviation]

        return len(words) == 1 and word in words

# Your ValidWordAbbr object will be instantiated and called as such:
# obj = ValidWordAbbr(dictionary)
# param_1 = obj.isUnique(word)
```

The implementation starts by creating a hash map named `abbr_map`. Each abbreviation maps to a set of dictionary words that produce it.

Using a set is important because the dictionary may contain duplicate words. We only care about distinct words sharing an abbreviation.

The helper method `_get_abbreviation` centralizes abbreviation generation logic. This avoids duplicated code and ensures consistency between initialization and query operations.

During initialization, every dictionary word is processed once and inserted into the correct abbreviation bucket.

The `isUnique` method first computes the abbreviation of the query word. If the abbreviation does not exist in the map, uniqueness is guaranteed.

Otherwise, we retrieve the associated set of words. The query is unique only if the set contains exactly one word and that word equals the query word.

## Go Solution

```
package main

import "strconv"

type ValidWordAbbr struct {
	abbrMap map[string]map[string]bool
}

func Constructor(dictionary []string) ValidWordAbbr {
	abbrMap := make(map[string]map[string]bool)

	for _, word := range dictionary {
		abbr := getAbbreviation(word)

		if _, exists := abbrMap[abbr]; !exists {
			abbrMap[abbr] = make(map[string]bool)
		}

		abbrMap[abbr][word] = true
	}

	return ValidWordAbbr{
		abbrMap: abbrMap,
	}
}

func getAbbreviation(word string) string {
	if len(word) <= 2 {
		return word
	}

	return string(word[0]) +
		strconv.Itoa(len(word)-2) +
		string(word[len(word)-1])
}

func (this *ValidWordAbbr) IsUnique(word string) bool {
	abbr := getAbbreviation(word)

	words, exists := this.abbrMap[abbr]

	if !exists {
		return true
	}

	return len(words) == 1 && words[word]
}

/**
 * Your ValidWordAbbr object will be instantiated and called as such:
 * obj := Constructor(dictionary);
 * param_1 := obj.IsUnique(word);
 */
```

The Go implementation mirrors the Python approach closely.

Since Go does not have a built in set type, a map of type `map[string]bool` is used to simulate a set.

The outer map stores:

```
abbreviation -> set of words
```

The abbreviation helper function uses `strconv.Itoa` to convert the middle character count into a string.

The logic for uniqueness checking is identical to the Python solution.

## Complexity Analysis

| Time | O(N + Q) | O(N) preprocessing, O(1) average per query |
| Space | O(N) | Hash map stores dictionary words grouped by abbreviation |

Initialization processes every dictionary word exactly once. Each query performs only constant time hash map operations on average, making the solution highly efficient for the problem constraints.

## Edge Cases

### Duplicate Words in the Dictionary

A common mistake is treating duplicate entries as separate conflicting words.

For example:

```
dictionary = ["dog", "dog"]
```

The abbreviation `"d1g"` should still be considered unique for `"dog"` because there is only one distinct word involved.

The implementation handles this correctly by storing words in a set rather than a list. Duplicate insertions have no effect.

### Words of Length 1 or 2

The problem defines that words with length less than or equal to 2 are their own abbreviations.

For example:

```
"it" -> "it"
"a" -> "a"
```

A buggy implementation might incorrectly produce abbreviations like `"i0t"`.

The helper function explicitly checks for short words and returns them unchanged.

### Multiple Different Words Sharing One Abbreviation

Cases like:

```
["deer", "door", "dear"]
```

all map to `"d2r"`.

The implementation must recognize that any query with abbreviation `"d2r"` is not unique unless the set contains only the queried word.

Using a hash map from abbreviation to a set of words ensures all collisions are tracked correctly.
