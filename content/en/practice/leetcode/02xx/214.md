## Solution

## Problem Understanding

The problem asks us to transform a given string `s` into a palindrome by **only adding characters to the front of the string**. Among all possible palindromes that can be formed this way, we must return the **shortest one**.

A palindrome is a string that reads the same forward and backward. Since we are only allowed to prepend characters, we cannot rearrange characters already in the string or insert them elsewhere.

Another way to think about the problem is this: we want to preserve as much of the original string as possible and only add the minimum necessary prefix. To achieve that, we need to identify the **longest prefix of `s` that is already a palindrome**. Once we know this prefix, everything after it is the "problematic suffix" that prevents the whole string from being a palindrome. We can reverse that suffix and place it in front.

For example:

- `s = "aacecaaa"`

- The longest palindromic prefix is `"aacecaa"`
- Remaining suffix is `"a"`
- Reverse suffix → `"a"`
- Result → `"aaacecaaa"`
- `s = "abcd"`

- Longest palindromic prefix is `"a"`
- Remaining suffix is `"bcd"`
- Reverse suffix → `"dcb"`
- Result → `"dcbabcd"`

The constraints are important:

- `0 <= s.length <= 5 * 10^4`
- Only lowercase English letters appear

A string length of `50,000` means quadratic solutions will likely time out. An `O(n²)` approach might work for small inputs but becomes impractical here. This strongly suggests we need a near linear solution, ideally `O(n)`.

Several edge cases stand out immediately:

- An empty string should return an empty string.
- A single character is already a palindrome.
- A string that is already a palindrome should remain unchanged.
- Strings with no large palindromic prefix, such as `"abcd"`, require prepending almost the entire reversed suffix.
- Repeated characters can sometimes trick naive palindrome checks into redundant work.

## Approaches

### Brute Force Approach

The most straightforward solution is to repeatedly check prefixes of the string, starting from the entire string and moving backward.

We attempt the following:

1. Check if the entire string is a palindrome.
2. If not, check whether `s[:n-1]` is a palindrome.
3. Continue shrinking the prefix until a palindromic prefix is found.
4. Reverse the remaining suffix and prepend it.

For example, with `"abcd"`:

- `"abcd"` → not palindrome
- `"abc"` → not palindrome
- `"ab"` → not palindrome
- `"a"` → palindrome

Then:

- suffix = `"bcd"`
- reverse suffix = `"dcb"`
- result = `"dcbabcd"`

This method is correct because it explicitly finds the **largest prefix that is a palindrome**, which guarantees minimal additions.

However, palindrome checking costs `O(n)` each time, and we may do it for nearly every prefix. This leads to `O(n²)` time complexity, which is too slow for `n = 50,000`.

### Key Insight for an Optimal Solution

The key observation is that we are really searching for the **longest palindromic prefix**.

A prefix is palindromic if it matches its reverse. Instead of repeatedly checking every prefix, we can efficiently detect the longest matching prefix using a **string matching technique similar to KMP (Knuth-Morris-Pratt)**.

We build:

```
s + "#" + reverse(s)
```

Why?

A palindrome prefix means the prefix of `s` matches a suffix of `reverse(s)`.

The separator `"#"` prevents accidental overlap between the original and reversed portions.

For example:

```
s = "aacecaaa"
reverse(s) = "aaacecaa"

combined =
"aacecaaa#aaacecaa"
```

We compute the **Longest Prefix Suffix (LPS)** array from KMP. The final value of the LPS array tells us the length of the **longest prefix of `s` that is also a suffix of `reverse(s)`**, which is exactly the longest palindromic prefix.

Once we know that length:

- Extract the non-palindromic suffix
- Reverse it
- Prepend it

This gives an `O(n)` solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Repeatedly checks shrinking prefixes for palindrome validity |
| Optimal, KMP String Matching | O(n) | O(n) | Uses LPS array to find longest palindromic prefix efficiently |

## Algorithm Walkthrough

1. **Handle trivial cases**

If the string length is `0` or `1`, return it immediately because it is already a palindrome.
2. **Reverse the string**

Create a reversed version of the string:

```
reversed_s = reverse(s)
```

This allows us to compare prefix relationships against reversed suffixes.
3. **Build a combined string**

Construct:

```
combined = s + "#" + reversed_s
```

The separator ensures we never incorrectly match characters across the boundary.
4. **Build the LPS array**

The LPS array stores:

```
lps[i] = length of the longest proper prefix
         that is also a suffix
```

We compute this using standard KMP preprocessing.
5. **Interpret the final LPS value**

The last element:

```
lps[-1]
```

tells us the length of the **longest palindromic prefix**.
6. **Find the remaining suffix**

Everything after that prefix is not part of the palindrome:

```
suffix = s[palindrome_length:]
```
7. **Prepend the reversed suffix**

Reverse the suffix and place it in front:

```
result = reverse(suffix) + s
```
8. **Return the result**

This guarantees the shortest palindrome because we preserve the largest palindromic prefix.

### Why it works

The critical invariant is that the final LPS value represents the **longest prefix of `s` matching a suffix of `reverse(s)`**. Since suffixes in the reversed string correspond to prefixes in the original string read backward, this match can only represent a palindromic prefix. Choosing the longest such prefix minimizes the number of added characters, guaranteeing the shortest palindrome.

## Python Solution

```
class Solution:
    def shortestPalindrome(self, s: str) -> str:
        if len(s) <= 1:
            return s

        reversed_s = s[::-1]
        combined = s + "#" + reversed_s

        lps = [0] * len(combined)
        length = 0

        for i in range(1, len(combined)):
            while length > 0 and combined[i] != combined[length]:
                length = lps[length - 1]

            if combined[i] == combined[length]:
                length += 1

            lps[i] = length

        palindrome_length = lps[-1]
        suffix = s[palindrome_length:]

        return suffix[::-1] + s
```

The implementation begins by handling trivial cases where the string is already a palindrome by definition, namely empty strings and single characters.

Next, we reverse the string and construct the combined string:

```
s + "#" + reversed_s
```

This transformation converts the palindrome detection problem into a prefix matching problem.

The `lps` array is then computed using KMP preprocessing. The variable `length` tracks the current longest prefix match. If characters mismatch, we fall back using previously computed LPS values instead of restarting from scratch, which is what enables linear time complexity.

After building the LPS array, the final value gives the length of the longest palindromic prefix. We slice off the remaining suffix, reverse it, and prepend it to the original string.

## Go Solution

```
func shortestPalindrome(s string) string {
	if len(s) <= 1 {
		return s
	}

	reversed := reverseString(s)
	combined := s + "#" + reversed

	lps := make([]int, len(combined))
	length := 0

	for i := 1; i < len(combined); i++ {
		for length > 0 && combined[i] != combined[length] {
			length = lps[length-1]
		}

		if combined[i] == combined[length] {
			length++
		}

		lps[i] = length
	}

	palindromeLength := lps[len(lps)-1]
	suffix := s[palindromeLength:]

	return reverseString(suffix) + s
}

func reverseString(s string) string {
	runes := []rune(s)

	left, right := 0, len(runes)-1
	for left < right {
		runes[left], runes[right] = runes[right], runes[left]
		left++
		right--
	}

	return string(runes)
}
```

The Go implementation follows the same logic as the Python version. Since strings are immutable in Go, a helper function is used to reverse strings. The implementation uses `[]rune` instead of raw bytes to ensure correctness for Unicode characters, even though this problem only contains lowercase English letters.

Slices are used naturally for substring operations:

```
suffix := s[palindromeLength:]
```

There are no concerns about integer overflow because all indices remain bounded by `5 * 10^4`.

## Edge Cases

### Empty String

An empty string can easily break implementations that assume at least one character exists. For example, indexing into the string or accessing the final LPS element without safeguards could fail. This implementation handles it immediately:

```
if len(s) <= 1:
    return s
```

The function simply returns an empty string.

### Already Palindromic Strings

A common source of inefficiency is unnecessarily modifying strings that are already palindromes. Examples include `"aba"` or `"abba"`. The KMP preprocessing naturally identifies the entire string as the longest palindromic prefix, so:

```
suffix = ""
```

No extra characters are added.

### Strings With No Significant Palindromic Prefix

Inputs like `"abcd"` are tricky because only the first character forms a palindrome. A naive implementation might incorrectly choose a shorter or invalid prefix. Here, the LPS computation correctly identifies:

```
palindrome_length = 1
```

Only the minimum required reversed suffix is prepended, producing the shortest valid palindrome.

### Repeated Characters

Repeated characters such as `"aaaa"` can sometimes confuse prefix matching logic and lead to incorrect overlaps. The separator character `"#"` prevents accidental cross-boundary matching between the original and reversed strings, ensuring correct KMP behavior.
