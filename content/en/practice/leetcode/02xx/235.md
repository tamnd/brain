## Solution

## Problem Understanding

The problem gives us the root of a Binary Search Tree (BST) and two nodes, `p` and `q`. Our task is to find their Lowest Common Ancestor (LCA).

A lowest common ancestor is the deepest node in the tree that has both `p` and `q` as descendants. The definition also allows a node to be a descendant of itself. This means if one of the target nodes is an ancestor of the other, then that node itself is the answer.

The key detail in this problem is that the input tree is specifically a Binary Search Tree. In a BST:

- Every node in the left subtree has a value smaller than the current node.
- Every node in the right subtree has a value larger than the current node.

This ordering property is what allows us to solve the problem efficiently.

The input consists of:

- `root`, the root node of the BST
- `p`, one target node
- `q`, another target node

The expected output is the tree node representing the lowest common ancestor.

The constraints tell us several important things:

- The tree can contain up to `10^5` nodes, which means inefficient approaches may become too slow.
- All node values are unique, which simplifies comparisons.
- Both `p` and `q` are guaranteed to exist in the tree.
- `p != q`, so we never need to handle identical target nodes.

Several edge cases are important:

- One node may be the ancestor of the other.
- The tree may be extremely skewed, behaving like a linked list.
- The LCA may be the root node.
- The two nodes may both lie entirely in the left subtree or entirely in the right subtree.

Because the nodes are guaranteed to exist, we do not need additional validation logic for missing nodes.

## Approaches

### Brute Force Approach

A straightforward way to solve the problem is to treat the tree like a normal binary tree and ignore the BST property.

One common brute-force strategy is:

1. Build parent pointers for every node using DFS or BFS.
2. Trace all ancestors of `p` and store them in a set.
3. Move upward from `q` until reaching a node already in the set.

The first common node encountered is the LCA.

This approach works because the first shared ancestor on the upward traversal must be the lowest common ancestor. However, it requires additional memory for parent mappings and ancestor tracking.

Although this approach is acceptable for general binary trees, it fails to take advantage of the BST ordering property. Since the BST property gives directional information, we can do much better.

### Optimal Approach

The important insight is that in a BST, node values determine traversal direction.

Suppose the current node has value `x`.

- If both `p.val` and `q.val` are smaller than `x`, then both nodes must lie in the left subtree.
- If both `p.val` and `q.val` are larger than `x`, then both nodes must lie in the right subtree.
- Otherwise, the current node lies between the two values, or matches one of them. In that situation, the current node is the split point, which means it is the LCA.

This observation allows us to walk down the tree exactly once without extra memory structures.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Uses parent mapping and ancestor tracking |
| Optimal | O(h) | O(1) | Uses BST ordering to navigate directly |

Here, `h` represents the height of the tree.

## Algorithm Walkthrough

1. Start at the root node of the BST.
2. Compare the values of `p` and `q` with the current node's value.
3. If both target values are smaller than the current node's value, move to the left child. This works because BST ordering guarantees both nodes must be located in the left subtree.
4. If both target values are larger than the current node's value, move to the right child. Again, BST ordering guarantees both nodes must be in the right subtree.
5. Otherwise, the current node is the first node where the paths to `p` and `q` diverge. This means:

- One node is on the left and the other is on the right, or
- One of the nodes is equal to the current node
6. Return the current node as the lowest common ancestor.

### Why it works

The algorithm relies on the BST invariant:

- Left subtree values are smaller than the current node.
- Right subtree values are larger than the current node.

As long as both target nodes lie on the same side of the current node, we can safely continue downward. The first node where they no longer lie on the same side must be the deepest node that contains both targets in its subtrees. Therefore, that node is exactly the lowest common ancestor.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, x):
#         self.val = x
#         self.left = None
#         self.right = None

class Solution:
    def lowestCommonAncestor(
        self,
        root: 'TreeNode',
        p: 'TreeNode',
        q: 'TreeNode'
    ) -> 'TreeNode':

        current = root

        while current:
            if p.val < current.val and q.val < current.val:
                current = current.left
            elif p.val > current.val and q.val > current.val:
                current = current.right
            else:
                return current
```

The implementation starts with a pointer called `current` initialized to the root node.

Inside the loop, the algorithm compares the values of `p` and `q` against the current node's value.

If both target nodes are smaller, the search moves left because both nodes must exist in the left subtree according to BST rules.

If both target nodes are larger, the search moves right for the same reason.

The remaining case is the important one. Either:

- The current node lies between `p` and `q`, or
- The current node matches one of the targets

In either situation, the current node is the lowest common ancestor, so the function immediately returns it.

The loop continues until the answer is found. Since the problem guarantees both nodes exist in the tree, a valid answer will always be returned.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val   int
 *     Left  *TreeNode
 *     Right *TreeNode
 * }
 */

func lowestCommonAncestor(root, p, q *TreeNode) *TreeNode {
	current := root

	for current != nil {
		if p.Val < current.Val && q.Val < current.Val {
			current = current.Left
		} else if p.Val > current.Val && q.Val > current.Val {
			current = current.Right
		} else {
			return current
		}
	}

	return nil
}
```

The Go implementation follows the same logic as the Python version.

The main difference is explicit handling of `nil` pointers instead of Python's `None`. Go also uses struct field access with capitalized names such as `Val`, `Left`, and `Right`.

The algorithm remains iterative, so no recursion stack is used.

## Worked Examples

### Example 1

Input:

```
root = [6,2,8,0,4,7,9,null,null,3,5]
p = 2
q = 8
```

Tree structure:

```
        6
      /   \
     2     8
    / \   / \
   0   4 7   9
      / \
     3   5
```

Step-by-step traversal:

| Current Node | p.val | q.val | Decision |
| --- | --- | --- | --- |
| 6 | 2 | 8 | One value smaller and one larger, return 6 |

The algorithm immediately finds that `2 < 6 < 8`, so node `6` is the split point and therefore the LCA.

Output:

```
6
```

### Example 2

Input:

```
root = [6,2,8,0,4,7,9,null,null,3,5]
p = 2
q = 4
```

Step-by-step traversal:

| Current Node | p.val | q.val | Decision |
| --- | --- | --- | --- |
| 6 | 2 | 4 | Both smaller, move left |
| 2 | 2 | 4 | Current node equals p, return 2 |

At node `2`, one target is the current node itself. Since a node can be its own descendant, node `2` is the LCA.

Output:

```
2
```

### Example 3

  1

Step-by-step traversal:

| Current Node | p.val | q.val | Decision |
| --- | --- | --- | --- |

```

```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(h) | Traverses at most one path from root to leaf |
| Space | O(1) | Uses only a few pointer variables |

The algorithm only moves downward through the tree once. At every step, one entire subtree is discarded based on BST ordering.

In a balanced BST, the height `h` is `O(log n)`, giving logarithmic performance. In the worst case, when the tree is completely skewed, the height becomes `O(n)`.

The iterative solution uses constant extra space because it stores only the current traversal pointer.

## Edge Cases

### One Node Is the Ancestor of the Other

This is one of the most common sources of bugs in LCA problems. Some implementations incorrectly continue traversing even after reaching one of the target nodes.

For example:

```
    2
   /
  1
```

If `p = 2` and `q = 1`, the correct answer is `2`.

The implementation handles this naturally because the algorithm stops whenever the current node lies between the two targets or matches one of them.

### Completely Skewed Tree

A BST may become fully unbalanced:

```
1
 \
  2
   \
    3
     \
      4
```

This structure behaves like a linked list. Recursive solutions may risk deep recursion stacks on very large inputs.

The iterative implementation avoids recursion entirely, so it safely handles trees with up to `10^5` nodes.

### Lowest Common Ancestor Is the Root

Sometimes the two target nodes are located in opposite subtrees of the root.

Example:

```
      6
     / \
    2   8
```

Here, `2` and `8` immediately split at the root.

The implementation correctly detects this because one value is smaller than the current node and the other is larger. As soon as this happens, the current node is returned as the LCA.
