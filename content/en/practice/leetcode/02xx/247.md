## Solution

## Problem Understanding

The problem asks us to generate all strobogrammatic numbers of a given length `n`.

A strobogrammatic number is a number that remains valid and appears identical when rotated `180` degrees. This property only works for specific digit mappings:

| Original Digit | Rotated Digit |
| --- | --- |
| 0 | 0 |
| 1 | 1 |
| 6 | 9 |
| 8 | 8 |
| 9 | 6 |

For example, `"69"` becomes `"96"` after rotation, which still forms a valid number. Similarly, `"88"` remains `"88"`.

The input is a single integer `n`, representing the exact length of the numbers we must generate. The output is a list of all valid strobogrammatic numbers with exactly `n` digits.

The constraints are relatively small, `1 <= n <= 14`, which strongly suggests that generating all valid combinations directly is feasible. However, brute force enumeration of every `n` digit number would still be extremely inefficient because the search space grows exponentially as `10^n`.

An important detail is that numbers cannot contain leading zeros unless the number itself has length `1`. For example:

- `"0"` is valid when `n = 1`
- `"00"` is not valid when `n = 2`

This restriction is one of the most common sources of bugs in implementations.

Several edge cases must be handled carefully:

- `n = 1`, where `"0"` is allowed
- Even versus odd length numbers
- The middle digit in odd length numbers, which can only be `0`, `1`, or `8`
- Preventing leading zeros for multi digit numbers

## Approaches

### Brute Force Approach

The brute force solution would generate every possible number with `n` digits and then check whether it is strobogrammatic.

To verify whether a number is strobogrammatic, we could scan from both ends toward the center and ensure that each digit pair matches one of the valid rotational mappings:

- `0 <-> 0`
- `1 <-> 1`
- `6 <-> 9`
- `8 <-> 8`
- `9 <-> 6`

For example, for `"619"`:

- `6` matches `9`
- `1` matches `1`

Therefore the number is valid.

Although this approach is conceptually simple, it is extremely inefficient. For length `n`, there are approximately `10^n` candidate numbers. Even with pruning of leading zeros, the search space becomes enormous for `n = 14`.

The brute force approach wastes time examining numbers that could never become strobogrammatic.

### Optimal Recursive Construction

The key observation is that strobogrammatic numbers are highly structured.

Instead of generating all numbers and checking validity afterward, we can construct only valid numbers from the start.

A strobogrammatic number must be symmetric under rotational mapping. This means we can build the number from the outside inward by placing valid digit pairs at mirrored positions.

The valid outer pairs are:

- `("0", "0")`
- `("1", "1")`
- `("6", "9")`
- `("8", "8")`
- `("9", "6")`

For odd length numbers, the center digit can only be:

- `"0"`
- `"1"`
- `"8"`

This naturally leads to a recursive solution:

- Build smaller strobogrammatic numbers
- Surround them with valid outer pairs
- Continue until reaching the desired length

This avoids invalid candidates entirely.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(10^n × n) | O(n) | Generates every possible number and validates each one |
| Optimal | O(5^(n/2)) | O(5^(n/2)) | Builds only valid strobogrammatic numbers recursively |

## Algorithm Walkthrough

1. Define all valid strobogrammatic digit pairs.

We store the five valid rotational mappings:

```
("0","0")
("1","1")
("6","9")
("8","8")
("9","6")
```

These pairs are the only valid choices for mirrored positions.
2. Use recursion to build smaller valid numbers first.

The recursive function will generate all strobogrammatic numbers of a smaller length and then expand them outward.

This works because if the inner portion is valid, adding a valid outer pair preserves the strobogrammatic property.
3. Define the base cases carefully.

There are two critical stopping conditions:

- If the remaining length is `0`, return `[""]`
- If the remaining length is `1`, return `["0", "1", "8"]`

The empty string base case allows even length constructions to terminate correctly.
4. Recursively generate inner numbers.

Suppose we want numbers of length `m`.

We first recursively generate all valid numbers of length `m - 2`.

These become the inner cores around which we place outer digit pairs.
5. Add valid outer digit pairs.

For each inner number:

- Add `"1"` and `"1"`
- Add `"6"` and `"9"`
- Add `"8"` and `"8"`
- Add `"9"` and `"6"`

We also add `"0"` and `"0"` unless we are constructing the outermost layer of the final answer.
6. Prevent leading zeros.

The pair `("0","0")` cannot be used when building the outermost layer because that would create invalid numbers like `"0110"`.

We detect this by comparing the current recursive length with the original target length.
7. Return all constructed results.

After recursively assembling all combinations, return the final list.

### Why it works

The algorithm works because every strobogrammatic number must consist entirely of valid mirrored digit pairs. The recursion guarantees that the inner portion is already valid before adding outer pairs. Since every possible valid pair is explored exactly once at every level, the algorithm generates all valid strobogrammatic numbers without duplicates or invalid constructions.

## Python Solution

```
from typing import List

class Solution:
    def findStrobogrammatic(self, n: int) -> List[str]:
        pairs = [
            ("0", "0"),
            ("1", "1"),
            ("6", "9"),
            ("8", "8"),
            ("9", "6")
        ]

        def build(length: int, total_length: int) -> List[str]:
            if length == 0:
                return [""]

            if length == 1:
                return ["0", "1", "8"]

            middle_numbers = build(length - 2, total_length)

            result = []

            for middle in middle_numbers:
                for left, right in pairs:
                    # Prevent leading zeros
                    if length == total_length and left == "0":
                        continue

                    result.append(left + middle + right)

            return result

        return build(n, n)
```

The implementation begins by defining the valid strobogrammatic digit pairs. These are the only digit combinations that remain valid after rotation.

The recursive helper function `build(length, total_length)` generates all valid strobogrammatic numbers of size `length`.

The first base case handles `length == 0`. Returning `[""]` may seem unusual, but it allows recursive concatenation to work naturally for even length numbers.

The second base case handles `length == 1`, where only `"0"`, `"1"`, and `"8"` are valid center digits.

For larger lengths, the function recursively generates all smaller strobogrammatic numbers of size `length - 2`. Each of these becomes the middle section of a larger number.

The nested loop appends every valid outer pair around each middle number. The condition preventing leading zeros is critical. It ensures that `"00"` is not used as the outermost pair unless the number length is exactly `1`.

Finally, the function returns the fully constructed list.

## Go Solution

```
func findStrobogrammatic(n int) []string {
	pairs := [][2]string{
		{"0", "0"},
		{"1", "1"},
		{"6", "9"},
		{"8", "8"},
		{"9", "6"},
	}

	var build func(int, int) []string

	build = func(length int, totalLength int) []string {
		if length == 0 {
			return []string{""}
		}

		if length == 1 {
			return []string{"0", "1", "8"}
		}

		middleNumbers := build(length-2, totalLength)

		result := []string{}

		for _, middle := range middleNumbers {
			for _, pair := range pairs {
				left := pair[0]
				right := pair[1]

				// Prevent leading zeros
				if length == totalLength && left == "0" {
					continue
				}

				result = append(result, left+middle+right)
			}
		}

		return result
	}

	return build(n, n)
}
```

The Go implementation follows the same recursive strategy as the Python version.

One difference is that Go requires explicit slice declarations and function variable definitions for recursion. The recursive helper is declared as a variable before assignment because anonymous recursive functions require prior declaration in Go.

The result container uses slices rather than dynamic lists. String concatenation works similarly to Python for these relatively small inputs.

There are no integer overflow concerns because the algorithm manipulates strings rather than numeric values.

## Edge Cases

### Single Digit Input

When `n = 1`, the valid outputs are only `"0"`, `"1"`, and `"8"`.

This case is special because `"0"` is normally forbidden as a leading digit. A naive implementation might incorrectly exclude it entirely. The implementation handles this with a dedicated base case for `length == 1`.

### Leading Zero Prevention

Numbers such as `"0110"` are invalid because multi digit numbers cannot begin with zero.

This is a very common bug in recursive solutions. Without explicit checking, adding the pair `("0","0")` to the outermost layer would generate invalid results.

The implementation prevents this by skipping the `"0"` pair when `length == total_length`.

### Odd Length Numbers

For odd length numbers, the center digit must map to itself after rotation.

Digits like `"6"` and `"9"` cannot appear in the center because they rotate into different digits. Only `"0"`, `"1"`, and `"8"` are valid.

The implementation handles this correctly through the `length == 1` base case, which returns only valid center digits.
