## Solution

## Problem Understanding

The problem gives us a string representing a mathematical expression containing non-negative integers, spaces, and the operators `+`, `-`, `*`, and `/`. We must evaluate the expression and return the resulting integer value.

The critical detail is that multiplication and division have higher precedence than addition and subtraction. This means we cannot simply evaluate operations from left to right without considering operator priority. For example, in the expression `"3+2*2"`, the multiplication must happen first, so the result is `3 + 4 = 7`, not `(3 + 2) * 2 = 10`.

The division operation must truncate toward zero. This matters especially for negative intermediate values. For example:

```
-3 / 2 = -1
```

because truncation toward zero removes the fractional part instead of flooring downward.

The input size can be as large as `3 * 10^5`, which immediately tells us that inefficient parsing strategies are dangerous. Any approach that repeatedly rebuilds strings or rescans the expression many times could become too slow. We need a solution that processes the string in linear time.

The problem also guarantees several important things:

- The expression is always valid.
- Only non-negative integers appear in the input.
- Operators are always one of `+`, `-`, `*`, or `/`.
- Intermediate and final results fit within 32-bit signed integers.

These guarantees simplify parsing because we do not need to handle malformed expressions or parentheses.

Several edge cases are important to think about before implementation:

- Expressions with spaces in arbitrary positions, such as `" 3+5 / 2 "`
- Multi-digit numbers, such as `"123+456"`
- Expressions ending with a number, since there is no trailing operator
- Division involving negative intermediate values
- Expressions containing only one number, such as `"42"`

A naive implementation often fails when handling operator precedence correctly or when processing the last number in the string.

## Approaches

### Brute Force Approach

A brute-force approach would first tokenize the entire expression and repeatedly scan for high-precedence operators. For example, we could:

1. Convert the string into a list of numbers and operators.
2. Repeatedly scan the list for `*` and `/`, compute them, and collapse the list.
3. After all multiplication and division operations are resolved, scan again for `+` and `-`.

This approach is correct because it explicitly enforces operator precedence in the same way a standard arithmetic parser would.

However, it is inefficient because each collapse operation may require shifting elements in a list or rebuilding arrays. In the worst case, repeated rescanning leads to quadratic time complexity.

With input sizes up to `300,000` characters, an `O(n^2)` solution is too slow.

### Optimal Approach

The key insight is that we do not need to store the entire expression or repeatedly rescan it. We can process the string in a single pass while maintaining enough state to respect operator precedence.

Addition and subtraction can be delayed because they have lower precedence. Multiplication and division, however, must be applied immediately to the most recent number.

A stack works perfectly for this:

- When we see `+`, we push the current number.
- When we see `-`, we push the negative of the current number.
- When we see `*` or `/`, we pop the previous number from the stack, compute the result immediately, and push the updated value back.

At the end, the stack contains values that should simply be added together.

This works because multiplication and division are resolved as soon as they appear, preserving arithmetic precedence naturally.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeated rescanning and collapsing of expression |
| Optimal | O(n) | O(n) | Single-pass parsing with immediate handling of `*` and `/` |

## Algorithm Walkthrough

1. Initialize a stack to store processed numbers.

The stack represents numbers that are already correctly adjusted for precedence. Addition and subtraction simply contribute positive or negative numbers into this stack.
2. Maintain three variables while scanning the string.

- `current_number` stores the number currently being parsed.
- `previous_operator` stores the operator that should be applied to the current number.
- The stack stores finalized intermediate values.
3. Iterate through each character in the string.

If the character is a digit, update `current_number` by multiplying the existing value by `10` and adding the new digit. This correctly handles multi-digit numbers.
4. When an operator or the end of the string is reached, process the current number.

At this point, we know the full number that came before the operator.
5. Apply the previous operator to the current number.

- If the previous operator is `+`, push the number onto the stack.
- If the previous operator is `-`, push the negative number.
- If the previous operator is `*`, pop the top stack value, multiply it with the current number, and push the result.
- If the previous operator is `/`, pop the top stack value, divide it by the current number using truncation toward zero, and push the result.
6. Update the previous operator.

After processing the current number, store the current operator so it can be applied to the next number.
7. Reset `current_number` to zero.

This prepares parsing for the next integer.
8. After the loop finishes, sum all values in the stack.

Since multiplication and division were already resolved immediately, the remaining values can simply be added together.

### Why it works

The algorithm maintains the invariant that the stack always contains values whose relative precedence has already been resolved. Multiplication and division are applied immediately because they bind more tightly than addition and subtraction. Addition and subtraction merely contribute signed values to the stack. Therefore, summing the stack at the end produces the correct final result.

## Python Solution

```
class Solution:
    def calculate(self, s: str) -> int:
        stack: list[int] = []
        current_number = 0
        previous_operator = "+"

        for index, char in enumerate(s):
            if char.isdigit():
                current_number = current_number * 10 + int(char)

            if char in "+-*/" or index == len(s) - 1:
                if previous_operator == "+":
                    stack.append(current_number)

                elif previous_operator == "-":
                    stack.append(-current_number)

                elif previous_operator == "*":
                    stack.append(stack.pop() * current_number)

                elif previous_operator == "/":
                    previous_value = stack.pop()
                    stack.append(int(previous_value / current_number))

                previous_operator = char
                current_number = 0

        return sum(stack)
```

The implementation follows the algorithm directly.

The stack stores intermediate values that are already adjusted for precedence. The variable `current_number` accumulates digits while parsing multi-digit integers. The variable `previous_operator` remembers which operation should be applied once the current number is complete.

The loop processes characters one at a time. Whenever a digit is encountered, the current number is extended. For example, parsing `"123"` happens as:

```
0 -> 1 -> 12 -> 123
```

When an operator or the end of the string is reached, the previous operator determines how the current number should interact with the stack.

Addition and subtraction simply push signed numbers. Multiplication and division immediately combine with the previous stack value, ensuring correct precedence.

The expression may end with a number rather than an operator, so the condition `index == len(s) - 1` ensures the final number is processed.

The division step uses:

```
int(previous_value / current_number)
```

This is important because Python's floor division operator `//` rounds downward for negative numbers, which does not match truncation toward zero. Converting the floating-point division result to `int` correctly truncates toward zero.

## Go Solution

```
package main

func calculate(s string) int {
	stack := []int{}
	currentNumber := 0
	previousOperator := byte('+')

	for i := 0; i < len(s); i++ {
		char := s[i]

		if char >= '0' && char <= '9' {
			currentNumber = currentNumber*10 + int(char-'0')
		}

		if char == '+' || char == '-' || char == '*' || char == '/' || i == len(s)-1 {

			if previousOperator == '+' {
				stack = append(stack, currentNumber)

			} else if previousOperator == '-' {
				stack = append(stack, -currentNumber)

			} else if previousOperator == '*' {
				lastValue := stack[len(stack)-1]
				stack = stack[:len(stack)-1]
				stack = append(stack, lastValue*currentNumber)

			} else if previousOperator == '/' {
				lastValue := stack[len(stack)-1]
				stack = stack[:len(stack)-1]
				stack = append(stack, lastValue/currentNumber)
			}

			previousOperator = char
			currentNumber = 0
		}
	}

	result := 0

	for _, value := range stack {
		result += value
	}

	return result
}
```

The Go implementation follows the same logic as the Python version, but there are several language-specific details worth noting.

Go integer division already truncates toward zero, so standard integer division behaves exactly as required by the problem. Unlike Python, no special conversion is necessary.

Slices are used as stacks. Appending pushes values, and removing the last element is done with:

```
stack = stack[:len(stack)-1]
```

Go also requires explicit digit conversion using:

```
int(char - '0')
```

because characters are stored as bytes.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is processed exactly once |
| Space | O(n) | The stack may store all numbers in the worst case |

The algorithm performs a single linear scan over the string. Each character contributes to at most one push or pop operation on the stack, so the total work is proportional to the input size.

The stack can grow to contain every number in the expression when only addition and subtraction are present, leading to linear auxiliary space usage.

## Edge Cases

One important edge case is expressions containing many spaces. Since spaces may appear anywhere between numbers and operators, a parser that assumes tightly packed input can fail. The implementation handles this naturally because spaces are ignored unless the loop reaches the final character.

Another critical edge case is multi-digit numbers. A naive parser might process digits independently and incorrectly treat `"123"` as separate values. The implementation correctly builds numbers incrementally using:

```
current_number = current_number * 10 + int(char)
```

which preserves positional value.

Division involving negative intermediate values is another common source of bugs. Python's `//` operator floors instead of truncating toward zero, which would produce incorrect results for negative values. Using:

```
int(a / b)
```

ensures truncation toward zero exactly matches the problem specification.

A final important edge case is expressions ending directly with a number. Since the last number is not followed by an operator, implementations that only process numbers upon encountering operators will miss the final value entirely. The condition:

```
index == len(s) - 1
```

guarantees the final number is processed correctly.
