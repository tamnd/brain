## Solution

## Problem Understanding

The problem asks us to count how many subtrees in a binary tree are "uni-value" subtrees. A uni-value subtree is a subtree where every node inside that subtree has the exact same value.

A subtree can consist of a single node. This is important because every leaf node automatically forms a valid uni-value subtree since there are no differing child values to violate the condition.

The input is the root node of a binary tree. Each node contains:

- An integer value
- A left child pointer
- A right child pointer

The output is a single integer representing the total number of uni-value subtrees in the entire tree.

For example, consider this tree:

```
        5
       / \
      1   5
     / \   \
    5   5   5
```

The four uni-value subtrees are:

- The left leaf node with value `5`
- The right leaf node with value `5`
- The other leaf node with value `5`
- The subtree rooted at the right child `5`

The subtree rooted at `1` is not uni-value because its children contain different values. The entire tree is also not uni-value.

The constraints tell us that the tree contains at most 1000 nodes. This is small enough that even an `O(n^2)` solution could potentially pass, but the problem is clearly designed for a more efficient depth-first traversal solution. Since this is a tree problem involving subtree properties, recursive DFS is a natural fit.

Several edge cases are important:

- An empty tree should return `0`
- A single-node tree should return `1`
- Trees where every node has the same value should count every subtree
- Trees with mixed values require careful validation of parent-child relationships
- Null children should not invalidate a uni-value subtree

## Approaches

### Brute Force Approach

The brute-force approach checks every node as a potential subtree root. For each node, we perform a traversal of its entire subtree to determine whether all nodes share the same value.

The process works like this:

1. Traverse every node in the tree
2. For each node:

- Record its value
- Traverse all descendants
- Verify that every descendant matches the root value
3. Increment the count if the subtree is uni-value

This approach is correct because every subtree is explicitly validated. However, it is inefficient because the same nodes are repeatedly traversed many times.

In the worst case, for a skewed tree with `n` nodes:

- The root validation visits `n` nodes
- The next node validation visits `n-1` nodes
- And so on

This leads to:

```
n + (n-1) + (n-2) + ... + 1 = O(n^2)
```

### Optimal DFS Approach

The key insight is that whether a subtree is uni-value depends entirely on its children.

A subtree rooted at node `X` is uni-value if:

1. The left subtree is uni-value
2. The right subtree is uni-value
3. If the left child exists, its value equals `X.val`
4. If the right child exists, its value equals `X.val`

This naturally suggests a bottom-up DFS traversal.

Instead of repeatedly validating subtrees from scratch, each recursive call returns whether the current subtree is uni-value. Parent nodes then use this information to determine their own validity.

This avoids redundant traversal because each node is processed exactly once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(h) | Repeatedly traverses the same subtrees |
| Optimal | O(n) | O(h) | Bottom-up DFS processes each node once |

Here, `h` is the height of the tree.

## Algorithm Walkthrough

### Optimal DFS Algorithm

1. Start a depth-first traversal from the root.

We use postorder traversal because we must determine whether the left and right subtrees are uni-value before deciding for the current node.
2. Define a recursive helper function.

The helper function returns a boolean indicating whether the subtree rooted at the current node is uni-value.
3. Handle the base case.

If the current node is `None`, return `True`.

An empty subtree is considered uni-value because it does not violate the condition.
4. Recursively process the left subtree.

Store whether the left subtree is uni-value.
5. Recursively process the right subtree.

Store whether the right subtree is uni-value.
6. If either subtree is not uni-value, return `False`.

The current subtree cannot be uni-value if one of its children already violates the property.
7. Compare child values with the current node value.

- If the left child exists and `left.val != current.val`, return `False`
- If the right child exists and `right.val != current.val`, return `False`
8. If all checks pass, increment the global counter.

This means the current subtree satisfies all uni-value conditions.
9. Return `True` to indicate the current subtree is uni-value.
10. After DFS completes, return the accumulated count.

### Why it works

The algorithm maintains the invariant that a node is considered uni-value only if both of its child subtrees are already verified as uni-value and all existing child values match the current node value.

Because the traversal processes children before parents, every subtree decision is based on fully validated information. Since each node is checked exactly once, the algorithm correctly counts all uni-value subtrees without duplication.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def countUnivalSubtrees(self, root: Optional[TreeNode]) -> int:
        self.count = 0

        def dfs(node: Optional[TreeNode]) -> bool:
            if not node:
                return True

            left_is_unival = dfs(node.left)
            right_is_unival = dfs(node.right)

            if not left_is_unival or not right_is_unival:
                return False

            if node.left and node.left.val != node.val:
                return False

            if node.right and node.right.val != node.val:
                return False

            self.count += 1
            return True

        dfs(root)
        return self.count
```

The implementation uses a nested DFS helper function to recursively determine whether each subtree is uni-value.

The variable `self.count` stores the total number of valid subtrees discovered during traversal.

The recursion begins with the base case. A `None` node returns `True`, allowing leaf nodes to be treated naturally as uni-value subtrees.

The algorithm then recursively evaluates the left and right subtrees before checking the current node. This is a classic postorder traversal pattern.

After both recursive calls complete, the function validates:

- Both child subtrees must already be uni-value
- Any existing child must have the same value as the current node

If these conditions hold, the subtree rooted at the current node is uni-value, so the count is incremented.

Finally, the helper returns `True` so parent nodes can use this information in their own validation.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func countUnivalSubtrees(root *TreeNode) int {
    count := 0

    var dfs func(node *TreeNode) bool

    dfs = func(node *TreeNode) bool {
        if node == nil {
            return true
        }

        leftIsUnival := dfs(node.Left)
        rightIsUnival := dfs(node.Right)

        if !leftIsUnival || !rightIsUnival {
            return false
        }

        if node.Left != nil && node.Left.Val != node.Val {
            return false
        }

        if node.Right != nil && node.Right.Val != node.Val {
            return false
        }

        count++
        return true
    }

    dfs(root)
    return count
}
```

The Go implementation closely mirrors the Python solution.

The main difference is that Go uses explicit pointer checks with `nil` instead of Python's `None`.

The recursive helper function is declared as a function variable:

```
var dfs func(node *TreeNode) bool
```

This allows the function to recursively reference itself.

The counter is maintained as a local variable captured by the closure.

Because node values are constrained to small integers, integer overflow is not a concern.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited exactly once |
| Space | O(h) | Recursive call stack height equals tree height |

The DFS traversal processes every node exactly one time. Each node performs only constant-time work consisting of recursive result checks and value comparisons.

The space complexity comes from recursion depth. In a balanced tree, the height is `O(log n)`. In the worst case of a completely skewed tree, the height becomes `O(n)`.

## Edge Cases

### Empty Tree

An empty tree is a common edge case that can cause null pointer errors in recursive implementations. Since there are no nodes, there are also no subtrees, so the correct answer is `0`.

The implementation handles this naturally because the DFS helper returns `True` for `None`, while the global count remains unchanged.

### Single Node Tree

A tree with exactly one node is always a uni-value subtree because there are no differing descendants.

Some incorrect implementations forget that leaf nodes qualify automatically. In this solution, leaf nodes succeed because both recursive child calls return `True`, and there are no child value mismatches.

### Parent With One Matching Child

Consider this tree:

```
    5
   /
  5
```

This entire subtree is still uni-value even though one child is missing.

A buggy implementation might incorrectly require both children to exist. This solution correctly treats missing children as valid because null subtrees return `True`.

### Mixed Internal Values

Consider:

```
      1
     / \
    1   1
   /
  2
```

The left subtree is not uni-value because of the `2`, which means the root subtree also cannot be uni-value.

The implementation correctly propagates failure upward because each recursive call returns whether its subtree satisfies the condition. Once a subtree returns `False`, all ancestors depending on it also fail validation.
