## Solution

## Problem Understanding

The problem asks us to split a binary string `s`, consisting only of `'0'` and `'1'` characters, into two non-empty substrings, a left part and a right part, and compute a score for each possible split. The score is defined as the number of zeros in the left substring plus the number of ones in the right substring. Our goal is to find the maximum score among all possible valid splits.

The input is a string of length between 2 and 500, which guarantees that there will always be at least one valid split. The output is a single integer representing the highest possible score obtained by any split.

Important edge cases include strings that consist entirely of zeros or ones, where the split might favor one side heavily, and strings where zeros and ones are evenly distributed. Since the string is relatively short (up to 500 characters), we need to ensure that our solution is efficient but do not need to optimize for extremely large inputs.

## Approaches

The brute-force approach is straightforward. For each possible split point between index `1` and `len(s)-1`, we can count the zeros in the left substring and ones in the right substring, sum them, and track the maximum score. This method is correct because it explicitly evaluates all valid splits, but it is inefficient because it recalculates counts repeatedly for overlapping parts of the string, giving it a time complexity of O(n²), which is unnecessary for strings of length up to 500.

The optimal approach uses a single pass of the string with running counts. We first calculate the total number of ones in the string. Then, as we iterate from left to right, we maintain a running count of zeros in the left substring. The number of ones in the right substring at any point can be calculated as `total_ones - ones_seen_so_far`. At each split, the score is `left_zeros + right_ones`, and we track the maximum score. This approach runs in O(n) time and uses O(1) additional space, making it ideal for the given constraints.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Evaluate all possible splits and count zeros and ones each time |
| Optimal | O(n) | O(1) | Maintain running counts of zeros and ones to compute score in a single pass |

## Algorithm Walkthrough

1. Compute the total number of ones in the string and store it in `total_ones`. This represents the initial number of ones in the right substring before any splits.
2. Initialize two counters: `left_zeros = 0` to track zeros in the left substring and `right_ones = total_ones` for the right substring.
3. Initialize `max_score` to 0, which will store the maximum score encountered.
4. Iterate through the string from index 0 to `len(s) - 2` (inclusive) to consider all valid splits. We stop before the last character because both substrings must be non-empty.
5. For each character, if it is `'0'`, increment `left_zeros`; if it is `'1'`, decrement `right_ones` because this one moves from the right substring to the left substring.
6. Compute the score as `left_zeros + right_ones` for the current split and update `max_score` if this score is greater than the current maximum.
7. After iterating through all valid splits, return `max_score`.

**Why it works:** The algorithm maintains accurate running counts of zeros in the left and ones in the right substring. By updating these counts incrementally at each split, we ensure that the computed score is always correct for that split. Because we evaluate all valid splits exactly once, we are guaranteed to find the maximum score.

## Python Solution

```python
class Solution:
    def maxScore(self, s: str) -> int:
        total_ones = s.count('1')
        left_zeros = 0
        right_ones = total_ones
        max_score = 0

        for i in range(len(s) - 1):
            if s[i] == '0':
                left_zeros += 1
            else:
                right_ones -= 1
            max_score = max(max_score, left_zeros + right_ones)

        return max_score
```

This Python implementation first counts all ones in the string to initialize the right substring. As we iterate, we maintain counts of zeros in the left substring and ones in the right substring, updating the maximum score at each step. The loop stops at `len(s) - 1` to ensure both substrings are non-empty.

## Go Solution

```go
func maxScore(s string) int {
    totalOnes := 0
    for _, ch := range s {
        if ch == '1' {
            totalOnes++
        }
    }

    leftZeros := 0
    rightOnes := totalOnes
    maxScore := 0

    for i := 0; i < len(s)-1; i++ {
        if s[i] == '0' {
            leftZeros++
        } else {
            rightOnes--
        }
        score := leftZeros + rightOnes
        if score > maxScore {
            maxScore = score
        }
    }

    return maxScore
}
```

The Go version mirrors the Python logic closely. We iterate using a standard for loop, using `range` for counting ones initially. Integer handling and slice indexing in Go is straightforward and aligns well with Python semantics for this problem.

The reasoning is that we only need to traverse the string twice, once for counting ones and once for the split evaluation, and we maintain counts with constant space.
