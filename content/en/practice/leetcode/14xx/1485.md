## Solution

## Problem Understanding

The problem asks us to create a deep copy of a binary tree where each node contains four fields: `val`, `left`, `right`, and an additional `random` pointer. The `random` pointer may point to any node in the tree, including itself, or it may be `null`.

A deep copy means the cloned tree must consist entirely of new nodes. None of the nodes in the cloned tree can share references with the original tree. Additionally, every relationship in the original tree must be preserved in the clone. That includes the standard binary tree structure through the `left` and `right` pointers, and also the arbitrary graph-like connections created by the `random` pointers.

The input is provided as a normal binary tree structure, but the serialization format includes an additional `random_index` value for each node. That index indicates which node the `random` pointer references. Internally, however, the function receives an actual tree of `Node` objects, not the serialized array representation.

The expected output is the root of a completely cloned tree using the `NodeCopy` class. Every node in the new tree should contain the same value and the same structural relationships as the original tree.

The constraints tell us that the tree contains at most 1000 nodes. This is small enough that an `O(n)` or even `O(n log n)` solution is easily acceptable. However, a naive repeated-search solution can degrade to `O(n^2)`, which is unnecessary and avoidable.

There are several important edge cases to consider:

- The tree may be empty, meaning `root == null`
- A node’s `random` pointer may point to itself
- Multiple nodes may share the same random target
- Random pointers may create cycles, even though the binary tree structure itself is acyclic
- Random pointers may be `null`

The presence of arbitrary random connections means we cannot safely clone nodes independently without tracking which original nodes have already been copied.

## Approaches

A brute-force solution would attempt to clone the tree structure first and then repeatedly search the original and copied trees to reconstruct the random pointer relationships. For every node, we could determine where its random pointer points in the original tree and then search for the corresponding node in the cloned tree.

This approach works because every node can eventually be matched by value and traversal order. However, it becomes inefficient because each random pointer reconstruction may require traversing large portions of the tree. If there are `n` nodes and each lookup costs `O(n)`, the total runtime becomes `O(n^2)`.

The key observation for a better solution is that each original node corresponds to exactly one cloned node. If we maintain a hash map from original nodes to cloned nodes, then any pointer relationship can be reconstructed in constant time.

This transforms the problem into a standard graph-copying pattern. Even though the main structure is a tree, the random pointers effectively create arbitrary graph edges. The hash map ensures we never duplicate nodes and always reconnect pointers correctly.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly searches for corresponding cloned nodes |
| Optimal | O(n) | O(n) | Uses a hash map from original nodes to cloned nodes |

## Algorithm Walkthrough

1. First, handle the empty tree case. If `root` is `None`, return `None` immediately because there is nothing to clone.
2. Create a hash map called `node_map`. This map stores the relationship between an original node and its cloned counterpart. The key is the original node, and the value is the cloned node.
3. Define a recursive DFS function that receives an original node and returns its clone.
4. Inside the DFS function, check whether the current node is `None`. If so, return `None`. This handles missing children and missing random pointers naturally.
5. Before creating a new clone, check whether the node already exists in the hash map. This is extremely important because random pointers may create cycles or repeated references. If the node has already been cloned, return the existing clone immediately.
6. Create a new `NodeCopy` instance using the current node’s value.
7. Store the mapping from the original node to the cloned node in the hash map immediately after creation. This early insertion prevents infinite recursion if random pointers form cycles.
8. Recursively clone the left child and assign it to the cloned node’s `left`.
9. Recursively clone the right child and assign it to the cloned node’s `right`.
10. Recursively clone the random target and assign it to the cloned node’s `random`.
11. Return the cloned node.
12. Start the DFS from the root and return the resulting cloned root.

### Why it works

The algorithm maintains a one-to-one mapping between original nodes and cloned nodes. Every time a node is encountered, either a new clone is created or an existing clone is reused. Because every edge (`left`, `right`, and `random`) is reconstructed using this mapping, the final cloned structure preserves all relationships exactly. The early insertion into the hash map guarantees termination even when random pointers create cycles.

## Python Solution

```python
# Definition for Node.
# class Node:
#     def __init__(self, val=0, left=None, right=None, random=None):
#         self.val = val
#         self.left = left
#         self.right = right
#         self.random = random

# Definition for NodeCopy.
# class NodeCopy:
#     def __init__(self, val=0, left=None, right=None, random=None):
#         self.val = val
#         self.left = left
#         self.right = right
#         self.random = random

from typing import Optional

class Solution:
    def copyRandomBinaryTree(
        self,
        root: 'Optional[Node]'
    ) -> 'Optional[NodeCopy]':

        node_map = {}

        def dfs(node):
            if node is None:
                return None

            if node in node_map:
                return node_map[node]

            cloned_node = NodeCopy(node.val)
            node_map[node] = cloned_node

            cloned_node.left = dfs(node.left)
            cloned_node.right = dfs(node.right)
            cloned_node.random = dfs(node.random)

            return cloned_node

        return dfs(root)
```

The implementation follows the recursive DFS strategy described earlier.

The `node_map` dictionary is the core data structure. It guarantees that every original node corresponds to exactly one cloned node. This prevents duplicate cloning and allows random pointers to be reconstructed efficiently.

The recursive `dfs` function handles all pointer types uniformly. Whether the pointer comes from `left`, `right`, or `random`, the same cloning logic applies.

The most important detail is that the clone is inserted into the map immediately after creation. Without this step, cyclic random references could cause infinite recursion.

Because the recursion returns the cloned version of any node it processes, assigning child and random pointers becomes very clean and concise.

## Go Solution

```go
/**
 * Definition for a Node.
 * type Node struct {
 *     Val int
 *     Left *Node
 *     Right *Node
 *     Random *Node
 * }
 */

type NodeCopy struct {
    Val    int
    Left   *NodeCopy
    Right  *NodeCopy
    Random *NodeCopy
}

func copyRandomBinaryTree(root *Node) *NodeCopy {
    nodeMap := make(map[*Node]*NodeCopy)

    var dfs func(*Node) *NodeCopy

    dfs = func(node *Node) *NodeCopy {
        if node == nil {
            return nil
        }

        if cloned, exists := nodeMap[node]; exists {
            return cloned
        }

        clonedNode := &NodeCopy{
            Val: node.Val,
        }

        nodeMap[node] = clonedNode

        clonedNode.Left = dfs(node.Left)
        clonedNode.Right = dfs(node.Right)
        clonedNode.Random = dfs(node.Random)

        return clonedNode
    }

    return dfs(root)
}
```

The Go implementation mirrors the Python solution closely. The primary difference is that Go uses explicit pointer types and a typed map declaration.

The hash map is declared as `map[*Node]*NodeCopy`, which maps original node pointers to cloned node pointers.

Go uses `nil` instead of `None`, and recursive closures require explicit declaration using `var dfs func(*Node) *NodeCopy` before assignment.

Since the constraints are small, recursion depth is safe for this problem.

## Worked Examples

### Example 1

Input:

```
[[1,null],null,[4,3],[7,0]]
```

Tree structure:

```
    1
     \
      4
       \
        7
```

Random relationships:

```
1.random -> null
4.random -> 7
7.random -> 1
```

#### Step-by-step traversal

| Step | Current Node | Action | node_map |
| --- | --- | --- | --- |
| 1 | 1 | Create clone of 1 | {1 → 1'} |
| 2 | 4 | Create clone of 4 | {1 → 1', 4 → 4'} |
| 3 | 7 | Create clone of 7 | {1 → 1', 4 → 4', 7 → 7'} |
| 4 | 1.random | null | unchanged |
| 5 | 4.random | points to existing 7' | unchanged |
| 6 | 7.random | points to existing 1' | unchanged |

Final cloned relationships:

```
1'.right = 4'
4'.right = 7'

1'.random = null
4'.random = 7'
7'.random = 1'
```

### Example 2

Input:

```
[[1,4],null,[1,0],null,[1,5],[1,5]]
```

This example demonstrates self-references and repeated random targets.

Suppose node A has:

```
A.random -> A
```

During DFS:

| Step | Current Node | Action |
| --- | --- | --- |
| 1 | A | Create A' |
| 2 | Store A → A' in map | Prevents recursion loop |
| 3 | Clone A.random | Returns existing A' |

The cloned node correctly points to itself.

### Example 3

Input:

```
[[1,6],[2,5],[3,4],[4,3],[5,2],[6,1],[7,0]]
```

This example forms a chain of random pointers in reverse order.

The important observation is that the DFS does not care about traversal direction. As soon as a node is cloned once, all future references reuse the existing clone from the hash map.

Even though the random pointers form many cross-links, each node is still cloned exactly once.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every node is visited and cloned exactly once |
| Space | O(n) | The hash map stores one entry per node, recursion stack may also reach O(n) |

The DFS processes each node only one time because the hash map prevents duplicate work. Every edge assignment is constant time.

The auxiliary space comes primarily from the hash map. In the worst case, the recursion depth may also become `O(n)` for a highly skewed tree.

## Edge Cases

A completely empty tree is the simplest edge case. A naive implementation might attempt to recurse immediately and fail with a null reference error. The solution handles this cleanly because the DFS function immediately returns `None` when the input node is `None`.

A node whose random pointer points to itself is another important case. Without inserting the clone into the hash map before recursively cloning neighbors, the recursion would loop forever. The implementation avoids this by storing the clone immediately after creation.

Random pointer cycles between multiple nodes are especially dangerous. For example, node A may randomly point to B while B randomly points back to A. Since the random relationships form a graph rather than a pure tree, cycle detection is essential. The hash map acts as both a memoization structure and a visited set, guaranteeing termination.

Another subtle edge case occurs when multiple nodes share the same random target. A flawed implementation might accidentally create duplicate cloned nodes for the same target. Because this solution always checks the hash map first, all references correctly point to the same cloned instance.
