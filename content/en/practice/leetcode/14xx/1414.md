## Solution

## Problem Understanding

The problem asks us to determine the minimum number of Fibonacci numbers whose sum equals a given integer `k`. The Fibonacci sequence starts with `1, 1, 2, 3, 5, 8, 13, ...` and each subsequent number is the sum of the previous two. The same Fibonacci number can be used multiple times if necessary, but in practice, using larger numbers first usually minimizes the count.

The input is a single integer `k` with constraints `1 <= k <= 10^9`. This tells us that `k` can be quite large, so any solution that relies on exhaustive enumeration of all possible combinations would be too slow. The expected output is a single integer representing the minimum number of Fibonacci numbers that sum to `k`.

Edge cases include very small values of `k` like `1` or `2`, large Fibonacci numbers near `10^9`, and values that can only be expressed as sums of multiple small Fibonacci numbers. The problem guarantees that a solution always exists, so we do not need to handle the case where `k` cannot be represented.

## Approaches

### Brute Force Approach

A brute force approach would try all combinations of Fibonacci numbers to sum up to `k`. This could be done using recursion or dynamic programming. For each Fibonacci number, we could either include it or skip it and recursively check if the sum can reach `k`. This approach is correct but inefficient because the number of combinations grows exponentially with `k`. Even memoization would be challenging given the large constraint `k <= 10^9`.

### Optimal Approach

The key insight is that Fibonacci numbers have a property similar to a greedy coin system. If we always subtract the largest Fibonacci number less than or equal to `k`, we reduce the problem size optimally. This is because Fibonacci numbers satisfy Zeckendorf's theorem, which states that every positive integer can be represented uniquely as the sum of non-consecutive Fibonacci numbers. Therefore, a greedy approach works:

1. Generate all Fibonacci numbers up to `k`.
2. While `k > 0`, subtract the largest Fibonacci number ≤ `k` and increment the count.

This approach is efficient and simple to implement.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Recursively try all combinations; infeasible for large `k`. |
| Optimal | O(log k) | O(log k) | Greedy subtraction of largest Fibonacci number; efficient and uses Zeckendorf's theorem. |

## Algorithm Walkthrough

1. Initialize a list to store Fibonacci numbers starting with `[1, 1]`.
2. Generate all Fibonacci numbers less than or equal to `k` by repeatedly adding the last two numbers until the next number exceeds `k`.
3. Initialize a counter `count = 0`.
4. While `k > 0`, do the following:

a. Find the largest Fibonacci number `fib` in the list that is ≤ `k`. This can be done by traversing from the end of the list.

b. Subtract `fib` from `k`.

c. Increment `count` by 1.
5. Return `count`.

**Why it works:** At each step, the algorithm chooses the largest Fibonacci number possible. Due to Zeckendorf's theorem, this greedy selection ensures the minimum number of Fibonacci numbers is used, and no solution is missed.

## Python Solution

```python
class Solution:
    def findMinFibonacciNumbers(self, k: int) -> int:
        fibs = [1, 1]
        while fibs[-1] + fibs[-2] <= k:
            fibs.append(fibs[-1] + fibs[-2])
        
        count = 0
        while k > 0:
            for i in range(len(fibs) - 1, -1, -1):
                if fibs[i] <= k:
                    k -= fibs[i]
                    count += 1
                    break
        return count
```

**Explanation:** We first generate all Fibonacci numbers up to `k`. Then, we iteratively subtract the largest possible Fibonacci number from `k` until `k` becomes zero, incrementing the counter each time. This implements the greedy approach described in the algorithm walkthrough.

## Go Solution

```go
func findMinFibonacciNumbers(k int) int {
    fibs := []int{1, 1}
    for fibs[len(fibs)-1]+fibs[len(fibs)-2] <= k {
        fibs = append(fibs, fibs[len(fibs)-1]+fibs[len(fibs)-2])
    }
    
    count := 0
    for k > 0 {
        for i := len(fibs) - 1; i >= 0; i-- {
            if fibs[i] <= k {
                k -= fibs[i]
                count++
                break
            }
        }
    }
    return count
}
```

**Go-specific notes:** We use a slice to store Fibonacci numbers, and append new numbers dynamically. The algorithm logic is identical to Python, with a `for` loop instead of `range` for backward traversal. Integer overflow is not a concern since the largest Fibonacci number needed is below 2^31 for `k <= 10^9`.

## Complexity Analysis

| Measure | Complexity | Explanation |

| Time | O(log k) | Fibonacci numbers grow exponentially, so there are roughly O(log k) numbers ≤ k. Each subtraction checks all numbers in reverse O(log k), giving O(log k) total. |
| Space | O(log k) | We store all Fibonacci numbers ≤ k, which is logarithmic in k. |

The reasoning relies on the exponential growth of Fibonacci numbers, which ensures both the list size and the number of iterations remain logarithmic relative to `k`.

## Edge Cases

**Edge Case 1: Smallest Input k=1**

This tests the algorithm’s handling of the minimal input. Since `1` is itself a Fibonacci number, the function should return `1` immediately. The implementation correctly identifies the largest Fibonacci number ≤ k as `1` and subtracts it.

**Edge Case 2: Fibonacci Number Equals k**

When `k` itself is a Fibonacci number, the minimum count is `1`. For example, `k=13`. The algorithm correctly selects `13` as the largest Fibonacci number and returns `1` without further iterations.

**Edge Case 3: Large k Near 10^9**

This tests the performance of the greedy approach. The algorithm precomputes Fibonacci numbers up to `k` and iteratively subtracts them, which remains efficient due to the exponential growth of Fibonacci numbers. Using this approach ensures both correctness and performance under the problem constraints.
