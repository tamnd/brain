# LeetCode 1455 - Check If a Word Occurs As a Prefix of Any Word in a Sentence

## Problem Understanding

This problem asks us to determine whether a given string, `searchWord`, appears as a prefix of any word inside a sentence. The sentence contains words separated by single spaces, and we must return the index of the first matching word using 1-based indexing.

A prefix means that the word starts with the given string. For example, `"burg"` is a prefix of `"burger"` because the first four characters of `"burger"` are `"burg"`.

The input consists of two strings:

- `sentence`, which contains lowercase English words separated by single spaces
- `searchWord`, which is the string we want to search for as a prefix

The expected output is:

- The position of the first word whose beginning matches `searchWord`
- `-1` if no such word exists

For example, if the sentence is `"i love eating burger"` and the search word is `"burg"`, then the answer is `4` because `"burger"` is the fourth word and starts with `"burg"`.

The constraints are very small:

- `sentence.length <= 100`
- `searchWord.length <= 10`

These limits tell us that efficiency is not a major concern. Even a straightforward solution will run comfortably within limits. Still, it is useful to think carefully about correctness and clean implementation.

Several edge cases are important:

- The matching word could be the very first word in the sentence.
- Multiple words may match, but we must return the first one.
- No word may match at all.
- `searchWord` may be longer than some candidate words.
- The sentence contains only lowercase letters and single spaces, so we do not need to handle punctuation or multiple consecutive spaces.

## Approaches

### Brute Force Approach

A brute-force solution checks every possible prefix manually.

First, split the sentence into words. Then, for each word, compare its characters one by one against `searchWord`.

For each word:

1. If the word is shorter than `searchWord`, it cannot contain the prefix.
2. Otherwise, compare each character from left to right.
3. If all characters match, return the current word index.

This approach is correct because it explicitly verifies whether `searchWord` matches the beginning of each word. The first successful match is immediately returned.

Although this approach is acceptable for the given constraints, it involves unnecessary manual character comparisons when most programming languages already provide optimized prefix-checking utilities.

### Optimal Approach

The better solution still iterates through the words one by one, but instead of manually comparing characters, it uses a built-in prefix operation such as:

- `startswith()` in Python
- `strings.HasPrefix()` in Go

The key insight is that the problem only requires checking whether a word begins with `searchWord`. Built-in prefix functions are concise, readable, and optimized internally.

As soon as we find the first matching word, we return its 1-based index.

- `m` is the length of `searchWord`

## Algorithm Walkthrough

### Optimal Algorithm

1. Split the sentence into individual words using spaces.

This gives us a list of words in their original order. Since the problem guarantees single spaces between words, splitting is straightforward and reliable.
2. Iterate through the words while keeping track of their positions.

Because the answer requires 1-based indexing, we either start counting from 1 directly or add 1 to the loop index.
3. For each word, check whether it starts with `searchWord`.

Using a built-in prefix function keeps the implementation concise and avoids manual character comparison logic.
4. If a match is found, immediately return the current index.

The problem specifically asks for the first matching word, so there is no reason to continue searching after the first valid prefix.
5. If the loop finishes without finding a match, return `-1`.

This means no word in the sentence starts with `searchWord`.

### Why it works

The algorithm examines every word in left-to-right order and checks exactly the property required by the problem: whether the word begins with `searchWord`.

Because the traversal follows the sentence order, the first successful match automatically corresponds to the smallest valid index. If no word satisfies the condition, returning `-1` is correct.

## Python Solution

```python
class Solution:
    def isPrefixOfWord(self, sentence: str, searchWord: str) -> int:
        words = sentence.split()

        for index, word in enumerate(words, start=1):
            if word.startswith(searchWord):
                return index

        return -1
```

The implementation begins by splitting the sentence into a list of words using `split()`.

Next, `enumerate(..., start=1)` is used so that the indices automatically match the problem's required 1-based indexing.

For each word, the code calls `startswith(searchWord)`. This directly checks whether the word begins with the target prefix.

If a match is found, the function immediately returns the corresponding index.

If the loop finishes without returning, no valid prefix exists in the sentence, so the function returns `-1`.

## Go Solution

```go
package main

import "strings"

func isPrefixOfWord(sentence string, searchWord string) int {
	words := strings.Split(sentence, " ")

	for i, word := range words {
		if strings.HasPrefix(word, searchWord) {
			return i + 1
		}
	}

	return -1
}
```

The Go solution follows the same logic as the Python version.

The sentence is split using `strings.Split(sentence, " ")`, which produces a slice of words.

Go uses 0-based indexing in the `range` loop, so we return `i + 1` to satisfy the problem's 1-based indexing requirement.

The built-in `strings.HasPrefix()` function performs the prefix comparison cleanly and efficiently.

There are no concerns about integer overflow because the input sizes are extremely small.

## Edge Cases

### Multiple Matching Words

A common mistake is continuing to search after finding a valid prefix and accidentally returning the last match instead of the first one.

For example:

```
sentence = "problem project program"
searchWord = "pro"
```

All three words match, but the correct answer is `1`.

The implementation avoids this bug by immediately returning as soon as the first prefix match is found.

### Search Word Longer Than Candidate Word

If the search word is longer than a sentence word, that word cannot possibly contain the prefix.

For example:

```
sentence = "hi there"
searchWord = "therefore"
```

A naive manual comparison could accidentally access invalid indices if not careful.

Using built-in prefix functions safely handles this case and correctly returns `-1`.

### Single Word Sentences

The sentence may contain only one word.

For example:

```
sentence = "apple"
searchWord = "app"
```

Some implementations incorrectly assume multiple words or mishandle indexing.

The current solution works naturally because splitting a single-word sentence still produces a valid one-element list, and enumeration starting at 1 correctly returns index `1`.
