## Solution

## Problem Understanding

This problem gives us access to a binary matrix where every row is sorted in non-decreasing order. That means each row consists of some number of `0`s followed by some number of `1`s. The task is to find the index of the leftmost column that contains at least one `1`.

The important restriction is that this is an interactive problem. We are not allowed to access the matrix directly. Instead, we can only use two methods from the `BinaryMatrix` interface:

- `get(row, col)` retrieves the value at a specific position.
- `dimensions()` returns the matrix size as `[rows, cols]`.

The challenge is not just correctness, but also efficiency. The judge limits us to at most `1000` calls to `get`, so a solution that scans every cell blindly may not scale well in larger matrices.

The input represents a matrix with:

- `rows` rows
- `cols` columns
- each element either `0` or `1`
- each row sorted from left to right

The output is:

- the smallest column index containing at least one `1`
- or `-1` if the matrix contains no `1`s at all

The constraints are relatively small, with both dimensions at most `100`, but the API call limit forces us to think carefully about how many matrix accesses we perform.

Several edge cases are important:

- A matrix containing only `0`s should return `-1`.
- A matrix containing only `1`s should return `0`.
- A single row or single column matrix can expose indexing mistakes.
- Rows may have different transition points from `0` to `1`, so we must search globally, not just rely on one row.

Because every row is sorted, we can exploit that structure to avoid unnecessary queries.

## Approaches

### Brute Force Approach

The most straightforward solution is to examine every cell in the matrix. We iterate row by row and column by column. Whenever we encounter a `1`, we record its column index and continue searching for a smaller one.

This approach is correct because it checks every possible location where a `1` could appear. By the end of the traversal, the smallest column index containing a `1` is guaranteed to be known.

However, this method is inefficient because it performs `rows × cols` calls to `get`. In the worst case, with a `100 × 100` matrix, this requires `10,000` API calls, which exceeds the problem limit of `1000`.

### Optimal Approach

The key observation is that each row is sorted. Once we encounter a `1` in a row, everything to its right must also be `1`. Similarly, everything before the first `1` must be `0`.

This allows us to use a staircase traversal starting from the top-right corner of the matrix.

The idea works as follows:

- Start at the top-right cell.
- If the current value is `1`, then this column could be the answer, but there might be another `1` further left, so move left.
- If the current value is `0`, then this row cannot contain a `1` in the current column or any column to the left, so move down.

Each move eliminates either an entire row or an entire column from consideration. Since we can only move left at most `cols` times and down at most `rows` times, the total number of API calls is at most `rows + cols`.

This is extremely efficient and safely stays under the API limit.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(rows × cols) | O(1) | Checks every cell directly |
| Optimal | O(rows + cols) | O(1) | Uses sorted row property with staircase traversal |

## Algorithm Walkthrough

1. Retrieve the matrix dimensions using `dimensions()`.

This gives us the number of rows and columns so we know the valid boundaries for traversal.
2. Start from the top-right corner of the matrix.

Initialize:

- `row = 0`
- `col = cols - 1`

The top-right position is ideal because it allows us to eliminate rows or columns efficiently based on the current value.
3. Maintain a variable `answer = -1`.

This stores the leftmost column containing a `1` found so far. If no `1` is ever found, the result remains `-1`.
4. While the current position is inside the matrix:

Continue while:

- `row < rows`
- `col >= 0`
5. Query the current cell using `get(row, col)`.

There are two cases:

- If the value is `1`:

- record `answer = col`
- move left with `col -= 1`

We move left because we want an even smaller column index.
- If the value is `0`:

- move down with `row += 1`

Since rows are sorted, everything left of this `0` is also `0`, so this row cannot help us further.
6. Continue until traversal leaves the matrix boundaries.
7. Return `answer`.

If no `1` was found, the value remains `-1`.

### Why it works

The algorithm maintains the invariant that all positions already excluded cannot contain a better answer than the current candidate.

When we move left after seeing a `1`, we search for a smaller valid column index. When we move down after seeing a `0`, we eliminate the current row because sorted order guarantees there are no `1`s to the left in that row.

Since every step removes either one row or one column from consideration, the algorithm eventually examines all relevant candidates without missing the true leftmost column containing a `1`.

## Python Solution

```python
# """
# This is BinaryMatrix's API interface.
# You should not implement it, or speculate about its implementation
# """
# class BinaryMatrix(object):
#     def get(self, row: int, col: int) -> int:
#     def dimensions(self) -> list[]:

class Solution:
    def leftMostColumnWithOne(self, binaryMatrix: 'BinaryMatrix') -> int:
        rows, cols = binaryMatrix.dimensions()

        row = 0
        col = cols - 1
        answer = -1

        while row < rows and col >= 0:
            if binaryMatrix.get(row, col) == 1:
                answer = col
                col -= 1
            else:
                row += 1

        return answer
```

The implementation begins by retrieving the matrix dimensions. This tells us how many rows and columns are available for traversal.

The algorithm initializes its search at the top-right corner. This is critical because it gives us directional information immediately. A `1` tells us to move left, while a `0` tells us to move down.

The `answer` variable tracks the smallest column index containing a `1` discovered so far. Each time we encounter a `1`, we update this variable and continue searching further left.

The loop continues until either:

- we move past the last row, or
- we move before the first column

At that point, all possible candidate positions have been eliminated, so the recorded answer is returned.

## Go Solution

```go
/**
 * // This is the BinaryMatrix's API interface.
 * // You should not implement it, or speculate about its implementation
 * type BinaryMatrix struct {
 *     Get func(int, int) int
 *     Dimensions func() []int
 * }
 */

func leftMostColumnWithOne(binaryMatrix BinaryMatrix) int {
    dimensions := binaryMatrix.Dimensions()
    rows := dimensions[0]
    cols := dimensions[1]

    row := 0
    col := cols - 1
    answer := -1

    for row < rows && col >= 0 {
        if binaryMatrix.Get(row, col) == 1 {
            answer = col
            col--
        } else {
            row++
        }
    }

    return answer
}
```

The Go implementation follows the exact same logic as the Python version. The primary difference is syntax and API invocation style.

In Go, the `Dimensions()` method returns a slice, so we extract the row and column counts using indexing. The traversal loop uses Go's standard `for` syntax instead of a `while` loop.

There are no integer overflow concerns here because matrix dimensions are very small. The solution also uses constant extra space, just like the Python version.

## Edge Cases

One important edge case is a matrix containing only `0`s. A naive implementation might accidentally return column `0` or fail to distinguish between "never found a `1`" and "found one at column zero." This implementation avoids that issue by initializing `answer` to `-1` and only updating it when a `1` is encountered.

Another critical edge case is a matrix containing only `1`s. In this situation, the algorithm repeatedly moves left until it exits the matrix. Some implementations incorrectly stop after finding the first `1`, but this solution continues searching for a smaller column index and correctly returns `0`.

Single-row and single-column matrices are also important because boundary logic can easily fail. In a single row, moving down immediately exits the matrix. In a single column, moving left immediately exits the matrix. The loop condition `row < rows and col >= 0` safely handles both situations without out-of-bounds access.

A more subtle case occurs when the leftmost `1` appears in a lower row instead of the top rows. The staircase traversal still works because moving down only happens after confirming that the current row cannot contain a better answer. This guarantees no valid candidate columns are skipped.
