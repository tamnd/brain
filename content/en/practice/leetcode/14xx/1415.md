## Solution

## Problem Understanding

The problem is asking us to generate strings consisting only of the letters `'a'`, `'b'`, and `'c'` such that no two consecutive letters are the same. These are called **happy strings**. Given two integers `n` and `k`, we need to consider all happy strings of length `n` sorted lexicographically and return the `k`-th string in that sorted list. If fewer than `k` happy strings exist for the given `n`, we return an empty string.

The input `n` represents the length of each happy string, which can range from 1 to 10, and `k` is the position in the lexicographically sorted list of happy strings, ranging from 1 to 100. The constraints are small, allowing us to explore solutions that generate all valid strings without performance issues.

Edge cases include when `k` is larger than the number of happy strings of length `n`, when `n` is 1 (the shortest possible string), and when the generated strings reach the maximum length of 10, which could potentially explode if not carefully handled.

The problem guarantees that the input `n` and `k` are positive integers and small enough to avoid performance issues with a backtracking solution.

## Approaches

The **brute-force approach** would be to generate all possible strings of length `n` using `'a'`, `'b'`, and `'c'`, filter out the strings that are not happy, and then sort the remaining strings lexicographically. This approach works correctly because generating all combinations and checking the happiness condition guarantees correctness. However, generating all 3^n strings and filtering them becomes inefficient as `n` grows because it explores unnecessary combinations that cannot be part of the final answer.

The **optimal approach** leverages backtracking with pruning. Instead of generating all strings, we only extend strings by adding characters that do not match the last character. This ensures that we only generate happy strings. We also track a counter to stop as soon as we reach the `k`-th happy string, avoiding unnecessary computation. This approach is much faster because it directly constructs valid strings and can terminate early once the target is found.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(3^n * n) | O(3^n * n) | Generate all strings and filter; slow and memory heavy |
| Optimal | O(k * n) | O(n) | Backtracking with early stopping; only generates valid strings until k-th |

## Algorithm Walkthrough

1. Initialize a list of characters `['a', 'b', 'c']` and a counter to track how many happy strings have been generated.
2. Define a backtracking helper function that takes the current string being built.
3. If the length of the current string equals `n`, increment the counter. If the counter equals `k`, store this string as the answer and stop further recursion.
4. For each character in `['a', 'b', 'c']`, check if it is different from the last character in the current string. If so, append it and recursively call the helper function.
5. Backtrack by removing the last character after recursion to try the next character.
6. If the helper function finishes without finding the `k`-th string, return an empty string.

**Why it works:** By only adding characters that differ from the previous character and exploring in lexicographical order, we generate happy strings in sorted order. Early termination ensures that we stop once the desired `k`-th string is found, keeping the algorithm efficient.

## Python Solution

```python
class Solution:
    def getHappyString(self, n: int, k: int) -> str:
        chars = ['a', 'b', 'c']
        result = []
        count = 0
        
        def backtrack(current: str):
            nonlocal count
            if len(current) == n:
                count += 1
                if count == k:
                    result.append(current)
                return
            for c in chars:
                if not current or current[-1] != c:
                    backtrack(current + c)
                    if result:
                        return
        
        backtrack("")
        return result[0] if result else ""
```

The Python implementation uses a recursive `backtrack` function. It builds strings character by character while maintaining the happy string property. Once the `k`-th string is found, the recursion stops immediately, and the string is returned. The `nonlocal count` allows us to keep track of how many strings have been generated across recursive calls. The result list stores the target string temporarily.

## Go Solution

```go
func getHappyString(n int, k int) string {
    chars := []byte{'a', 'b', 'c'}
    count := 0
    result := ""
    
    var backtrack func(current string)
    backtrack = func(current string) {
        if len(result) > 0 {
            return
        }
        if len(current) == n {
            count++
            if count == k {
                result = current
            }
            return
        }
        for _, c := range chars {
            if len(current) == 0 || current[len(current)-1] != byte(c) {
                backtrack(current + string(c))
            }
        }
    }
    
    backtrack("")
    return result
}
```

The Go implementation follows the same logic but uses closures for recursion. Go strings are immutable, so concatenation `current + string(c)` is safe. A string variable `result` stores the final answer, and recursion terminates early once it is found.
