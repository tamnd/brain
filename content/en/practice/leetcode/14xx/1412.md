## Solution

## Problem Understanding

This problem asks us to identify students who are "quiet" across all exams they participated in. A student is considered quiet if, in every exam they took, they were neither the highest scorer nor the lowest scorer.

We are given two tables. The `Student` table contains the mapping between a student's ID and their name. The `Exam` table records the score each student received in each exam. Every row in `Exam` represents one student's participation in one exam.

The key requirement is subtle but important. A student qualifies only if all of the following are true:

1. The student took at least one exam.
2. The student never achieved the maximum score in any exam.
3. The student never achieved the minimum score in any exam.

If a student is the top scorer even once, they are disqualified. If they are the bottom scorer even once, they are also disqualified.

An important detail is that ties count. If multiple students share the highest score in an exam, all of them are considered highest scorers and therefore are not quiet. The same applies to the lowest score.

The output should contain the `student_id` and `student_name` of all quiet students, ordered by `student_id`.

The input size is not explicitly stated in the problem description, but since this is a database problem labeled Hard, we should expect potentially large tables. That means inefficient repeated scans or nested queries could become expensive. The solution should minimize redundant work and rely on set-based SQL operations.

Several edge cases are important:

- A student who never took an exam must not appear in the result.
- An exam with only one participant makes that student both the highest and lowest scorer simultaneously.
- Multiple students can tie for highest or lowest score, and all tied students become disqualified.
- A student may be quiet in some exams but not others. One violation is enough to exclude them.

## Approaches

A brute-force solution would evaluate every student's participation one exam at a time. For each student and each exam they took, we could compute the minimum and maximum score of that exam and compare the student's score against them.

This approach is logically correct because it directly checks the problem definition. However, it is inefficient because the same exam statistics are recomputed repeatedly. If there are many students and many exam records, repeatedly scanning the `Exam` table becomes expensive.

The key observation for an optimal solution is that we do not actually need to evaluate every student independently. Instead, we can first determine all students who were ever extreme scorers, meaning they achieved either the minimum or maximum score in at least one exam. Once we know this disqualified set, the answer simply becomes all students who took at least one exam but are not in that set.

This transforms the problem into three clear steps:

1. Find exam-wise minimum and maximum scores.
2. Identify all students who matched those scores.
3. Return participating students who are not disqualified.

This is much more efficient because each exam is processed once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N²) | O(1) | Repeatedly scans exams for each student participation |
| Optimal | O(N) | O(N) | Precomputes extreme scorers and filters them efficiently |

Here, `N` represents the number of rows in the `Exam` table.

## Algorithm Walkthrough

1. First, compute the highest and lowest score for every exam.

We group the `Exam` table by `exam_id` and calculate:

- `MAX(score)`
- `MIN(score)`

This gives us the score boundaries for each exam.
2. Next, identify all students who were ever extreme scorers.

We join the original `Exam` table with the exam statistics computed above. For every row, if the student's score equals either the minimum or maximum score for that exam, that student is marked as disqualified.

We store these student IDs in a set-like result using `DISTINCT`.
3. Find all students who participated in at least one exam.

We need this because students who never took an exam must be excluded from the final answer.
4. Filter out disqualified students.

We take all participating students and remove anyone who appears in the disqualified set.
5. Join with the `Student` table.

Finally, we retrieve the student's name and order the result by `student_id`.

### Why it works

The algorithm works because every non-quiet student must appear in at least one exam where their score equals the exam minimum or maximum. By explicitly identifying all such students first, we completely characterize the set of disqualified students. Any participating student outside this set is guaranteed to have avoided both extremes in every exam, which exactly matches the definition of a quiet student.

## Python Solution

```python
# Write your MySQL query statement below

SELECT s.student_id, s.student_name
FROM Student s
JOIN (
    SELECT DISTINCT student_id
    FROM Exam
) participated
ON s.student_id = participated.student_id
WHERE s.student_id NOT IN (
    SELECT DISTINCT e.student_id
    FROM Exam e
    JOIN (
        SELECT
            exam_id,
            MAX(score) AS max_score,
            MIN(score) AS min_score
        FROM Exam
        GROUP BY exam_id
    ) stats
    ON e.exam_id = stats.exam_id
    WHERE e.score = stats.max_score
       OR e.score = stats.min_score
)
ORDER BY s.student_id;
```

The solution begins by identifying all students who participated in at least one exam. This is done using a subquery with `SELECT DISTINCT student_id FROM Exam`.

Next, the query computes the minimum and maximum score for every exam using a grouped aggregation. The inner derived table named `stats` contains one row per exam with its score boundaries.

We then join the original `Exam` table with these statistics. Whenever a student's score matches either the minimum or maximum value for that exam, that student becomes part of the disqualified set.

The outer query joins participating students with the `Student` table and removes every disqualified student using `NOT IN`.

Finally, the result is sorted by `student_id` as required.

## Edge Cases

One important edge case occurs when an exam has only one participant. In this scenario, the student's score is simultaneously the minimum and maximum score. A naive implementation might mistakenly treat the student as quiet because there are no other participants for comparison. This solution handles the case correctly because the score matches both `MIN(score)` and `MAX(score)`.

Another tricky case involves ties. Suppose three students all share the highest score in an exam. Every one of them must be excluded. A buggy implementation might only remove one student if it assumes unique maxima or minima. This solution avoids that issue because it compares scores directly against the computed exam boundaries, allowing all matching rows to be identified.

A third important edge case is students who never took any exams. Since they were never highest or lowest, it may seem tempting to include them. However, the problem explicitly states they must be excluded. The solution handles this by first building the participating student set and only considering students present in the `Exam` table.
