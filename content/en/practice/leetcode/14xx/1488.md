## Solution

## Problem Understanding

This problem involves managing a series of lakes over several days, where each lake can either receive rain and become full or be dried manually on days without rain. You are given an array `rains` where a positive number indicates rain on a specific lake, and zero indicates a dry day. The goal is to return an array `ans` that tells which lakes are dried on dry days while ensuring that no lake floods. Flooding occurs if it rains on a lake that is already full.

The input array `rains` has up to 100,000 elements, and the lake numbers can be very large, up to 10^9. This means a naive solution that checks every lake on each dry day is too slow. The output array must follow the rules: if it rains on a lake, mark it as `-1`; if it is a dry day, indicate which lake is dried. If avoiding flood is impossible, return an empty array.

Edge cases to be careful of include consecutive rains on the same lake without enough dry days in between, arrays with all zeros, arrays with all positive rains, and very large lake numbers that must be handled efficiently without constructing large arrays for each lake.

## Approaches

A brute-force approach would simulate each day by tracking all full lakes and, on each dry day, iterating over the full lakes to decide which one to dry. For every rain day, we would check if the lake is full and potentially cause a flood. While this approach is correct in principle, it is too slow because checking full lakes on each dry day can take O(n^2) time in the worst case.

The optimal approach uses a combination of a hash map and a sorted data structure, like a binary search tree or `SortedList`, to efficiently decide which lake to dry on dry days. The key insight is that we only need to dry a lake **before it rains again**. By keeping track of the last day each lake was filled and the available dry days, we can use binary search to find the first dry day after a lake was last filled and schedule the drying operation. This ensures no flood occurs while minimizing unnecessary checks.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Track full lakes and search linearly for each dry day |
| Optimal | O(n log n) | O(n) | Use hash map for last filled days and sorted structure for dry days to assign drying efficiently |

## Algorithm Walkthrough

1. Initialize a hash map `full_lakes` to record the last day each lake was filled.
2. Initialize a list `dry_days` to track indices of days with `0` in `rains`.
3. Initialize the result array `ans` with default values of `-1` for all rain days.
4. Iterate through each day `i` in `rains`. If it rains over a lake:

a. Check if the lake is in `full_lakes`. If it is, search for a dry day in `dry_days` **after the last fill day**.

b. If no suitable dry day exists, return an empty array because a flood is inevitable.

c. Otherwise, schedule drying on the earliest available dry day, remove that dry day from `dry_days`, and update `full_lakes` for this lake.
5. If the day is dry (`0`), add the index to `dry_days`. The lake to dry will be assigned later.
6. Return `ans` after processing all days.

**Why it works:** This algorithm ensures that a lake is only dried before it rains again, preventing flooding. By using a hash map and sorted list or binary search, we efficiently track lakes and dry days without unnecessary iterations.

## Python Solution

```python
from bisect import bisect_right
from typing import List

class Solution:
    def avoidFlood(self, rains: List[int]) -> List[int]:
        full_lakes = {}
        dry_days = []
        ans = [-1] * len(rains)
        
        for i, lake in enumerate(rains):
            if lake == 0:
                dry_days.append(i)
            else:
                if lake in full_lakes:
                    last_rain_day = full_lakes[lake]
                    idx = bisect_right(dry_days, last_rain_day)
                    if idx == len(dry_days):
                        return []
                    dry_day = dry_days[idx]
                    ans[dry_day] = lake
                    dry_days.pop(idx)
                full_lakes[lake] = i
        # Fill any remaining dry days with arbitrary lake number
        for i in dry_days:
            ans[i] = 1
        return ans
```

This implementation maintains `full_lakes` to track the last day each lake was filled and a sorted list of dry days. When rain occurs on a full lake, it uses `bisect_right` to find the first dry day after the lake was last filled. Remaining dry days are arbitrarily assigned to lake `1` to satisfy the output constraints.

## Go Solution

```go
package main

import (
	"sort"
)

func avoidFlood(rains []int) []int {
	fullLakes := make(map[int]int)
	dryDays := []int{}
	ans := make([]int, len(rains))
	for i := range ans {
		ans[i] = -1
	}

	for i, lake := range rains {
		if lake == 0 {
			dryDays = append(dryDays, i)
		} else {
			if lastDay, ok := fullLakes[lake]; ok {
				idx := sort.Search(len(dryDays), func(j int) bool {
					return dryDays[j] > lastDay
				})
				if idx == len(dryDays) {
					return []int{}
				}
				ans[dryDays[idx]] = lake
				dryDays = append(dryDays[:idx], dryDays[idx+1:]...)
			}
			fullLakes[lake] = i
		}
	}

	for _, day := range dryDays {
		ans[day] = 1
	}

	return ans
}
```

The Go version follows the same logic as Python. `sort.Search` is used for binary search over dry days. Slices are used carefully to remove used dry days, and remaining dry days are filled with lake `1`.

## Worked Examples

**Example 1:** `rains = [1,2,3,4]`

All days have rain. Each lake is filled once, no dry days exist, and no flood occurs. Output: `[-1,-1,-1,-1]`.

**Example 2:** `rains = [1,2,0,0,2,1]`

| Day | Rains | Full Lakes | Dry Days | Ans Update |
| --- | --- | --- | --- | --- |
| 0 | 1 | {1} | [] | -1 |
| 1 | 2 | {1,2} | [] | -1 |
| 2 | 0 | {1,2} | [2] |  |
| 3 | 0 | {1,2} | [2,3] |  |
| 4 | 2 | {1,2} | [2,3] | Dry lake 2 on day 2 → ans[2]=2 |
| 5 | 1 | {1,2} | [3] | Dry lake 1 on day 3 → ans[3]=1 |

Output: `[-1,-1,2,1,-1,-1]`.

**Example 3:** `rains = [1,2,0,1,2]`

| Day | Rains | Full Lakes | Dry Days | Ans Update |
| --- | --- | --- | --- | --- |
| 0 | 1 | {1} | [] | -1 |
| 1 | 2 | {1,2} | [] | -1 |
| 2 | 0 | {1,2} | [2] |  |
| 3 | 1 | {1,2} | [2] | Need to dry lake 1, dry day available? Yes, day 2 → ans[2]=1 |
| 4 | 2 | {2} | [] | Flood occurs → return [] |

Output: `[]`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n log n) | Each rain day may trigger binary search over dry days, total n days. |
| Space | O(n) | Hash map stores last filled day for each lake, dry_days list stores indices of zero days. |

The main performance gain comes from avoiding linear search over dry days and instead using binary search.
