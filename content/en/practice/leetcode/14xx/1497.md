## Solution

## Problem Understanding

The problem gives us an integer array `arr` whose length is guaranteed to be even, along with an integer `k`. We must determine whether it is possible to split the array into exactly `n / 2` pairs such that the sum of every pair is divisible by `k`.

In mathematical terms, for every chosen pair `(a, b)`, the following condition must hold:

$$(a + b) \bmod k = 0$$

The challenge is not to construct the pairs explicitly, but simply to determine whether such a pairing arrangement exists.

The input array may contain positive numbers, negative numbers, and zero. The values can be very large, up to `10^9` in magnitude, so we cannot rely on techniques that depend on small numeric ranges. The array length can be as large as `10^5`, which immediately tells us that expensive combinational or quadratic solutions will not be practical.

A key observation is that divisibility by `k` depends only on the remainder when dividing by `k`. If two numbers have remainders `r1` and `r2`, then:

$$(r1 + r2) \bmod k = 0$$

This means the two remainders must complement each other to sum to `k`, except for special cases like remainder `0`.

Several edge cases are important:

- Numbers can be negative, so remainder handling must normalize values into the range `[0, k - 1]`.
- Elements with remainder `0` must pair with other remainder `0` elements.
- If `k` is even, elements with remainder `k / 2` must pair with each other.
- Frequencies of complementary remainders must match exactly.
- Since the array length is even, we do not need to handle odd-length arrays.

These details are where many incorrect implementations fail.

## Approaches

### Brute Force Approach

A brute force solution would attempt to explicitly form valid pairs. One way is to repeatedly choose an unused element and search for another unused element whose sum with it is divisible by `k`.

For each element, we might scan the remaining array looking for a compatible partner. After finding one, both elements are marked as used. If every element can be paired successfully, we return `true`. Otherwise, we return `false`.

This method is correct because it directly checks the problem definition by constructing actual pairs. However, it is computationally expensive. In the worst case, each element may need to scan most of the remaining array.

With up to `10^5` elements, an `O(n^2)` solution is far too slow.

### Optimal Approach

The important insight is that divisibility depends only on remainders modulo `k`.

Suppose a number has remainder `r`. To form a sum divisible by `k`, it must pair with a number whose remainder is:

$$k - r$$

For example, if `k = 5`:

- remainder `1` pairs with remainder `4`
- remainder `2` pairs with remainder `3`

Some remainders are special:

- remainder `0` must pair with remainder `0`
- if `k` is even, remainder `k / 2` must pair with itself

Therefore, instead of constructing pairs directly, we only need to count how many times each remainder appears.

If the frequency conditions are satisfied, valid pairing is guaranteed.

This transforms the problem into a counting problem that can be solved in linear time using a hash map or frequency array.

## Algorithm Walkthrough

### Optimal Algorithm

1. Create a frequency map to count how many numbers produce each remainder when divided by `k`.

Since the array may contain negative numbers, normalize each remainder using:

$$remainder = ((num \% k) + k) \% k$$

This guarantees the remainder is always between `0` and `k - 1`.
2. Iterate through the array and populate the frequency map.

For example, if `k = 5` and a number produces remainder `2`, increment the count for remainder `2`.
3. Check the special case for remainder `0`.

Numbers with remainder `0` can only pair with other remainder `0` numbers. Therefore, the count of remainder `0` must be even.
4. Iterate through remainders from `1` to `k - 1`.

For each remainder `r`, compare its count with the count of its complementary remainder:

$$k - r$$

These counts must match exactly.
5. Handle the special midpoint case when `k` is even.

If `r = k / 2`, then numbers with this remainder must pair among themselves, so their count must also be even.
6. If all checks pass, return `true`.

Otherwise, return `false`.

### Why it works

Every valid pair must have remainders that sum to a multiple of `k`. Therefore, each remainder `r` requires a matching remainder `k - r`.

If the counts do not match, some elements will inevitably remain unpaired. Conversely, if all remainder frequencies satisfy the required conditions, we can always construct valid pairs greedily within each complementary remainder group.

This guarantees correctness.

## Python Solution

```python
from collections import Counter
from typing import List

class Solution:
    def canArrange(self, arr: List[int], k: int) -> bool:
        remainder_count = Counter()

        # Count normalized remainders
        for num in arr:
            remainder = ((num % k) + k) % k
            remainder_count[remainder] += 1

        # Remainder 0 must appear an even number of times
        if remainder_count[0] % 2 != 0:
            return False

        # Check complementary remainder frequencies
        for remainder in range(1, k):
            complement = k - remainder

            # Special case when remainder equals complement
            if remainder == complement:
                if remainder_count[remainder] % 2 != 0:
                    return False
            else:
                if remainder_count[remainder] != remainder_count[complement]:
                    return False

        return True
```

The implementation begins by creating a `Counter` to store remainder frequencies. Each number is converted into its normalized modulo value so negative integers are handled correctly.

After counting all remainders, the algorithm first checks the remainder `0` case. Since these values can only pair with each other, the count must be even.

The loop over remainders verifies the complementary pairing condition. For every remainder `r`, the count must equal the count of `k - r`.

The midpoint case occurs only when `k` is even. In this situation, remainder `k / 2` pairs only with itself, so its count must also be even.

If no condition fails, the function returns `True`.

## Go Solution

```go
package main

func canArrange(arr []int, k int) bool {
	remainderCount := make(map[int]int)

	// Count normalized remainders
	for _, num := range arr {
		remainder := ((num % k) + k) % k
		remainderCount[remainder]++
	}

	// Remainder 0 must appear an even number of times
	if remainderCount[0]%2 != 0 {
		return false
	}

	// Check complementary remainder frequencies
	for remainder := 1; remainder < k; remainder++ {
		complement := k - remainder

		// Special midpoint case
		if remainder == complement {
			if remainderCount[remainder]%2 != 0 {
				return false
			}
		} else {
			if remainderCount[remainder] != remainderCount[complement] {
				return false
			}
		}
	}

	return true
}
```

The Go implementation follows the same logic as the Python version. A `map[int]int` is used instead of `Counter` to store remainder frequencies.

Go's modulo operator can produce negative results for negative numbers, so the same normalization formula is required:

```go
((num % k) + k) % k
```

This guarantees remainders stay within the valid range.

The rest of the implementation directly mirrors the mathematical pairing logic.

## Edge Cases

### Negative Numbers

Negative integers are a common source of bugs because modulo behavior differs across programming languages. In Python, negative modulo already produces a non-negative result, but in Go and many other languages it does not.

For example:

```
-1 % 5 = -1   // in Go
```

Without normalization, remainder matching would fail. The implementation fixes this using:

```
((num % k) + k) % k
```

This guarantees every remainder falls within `[0, k - 1]`.

### Remainder Zero Elements

Numbers divisible by `k` produce remainder `0`. These numbers can only pair with each other because:

$$0 + 0 \equiv 0 \pmod{k}$$

If the count of remainder `0` elements is odd, at least one element will remain unpaired. The implementation explicitly checks for this condition before validating other remainder groups.

### Midpoint Remainder When k Is Even

When `k` is even, the remainder `k / 2` is special because:

$$\frac{k}{2} + \frac{k}{2} = k$$

So these elements can only pair with others having the same remainder. This means the count must be even.

For example, with `k = 4`, remainder `2` pairs only with remainder `2`.

The implementation handles this using the condition:

```
if remainder == complement:
```

and verifies the frequency is even.
