## Solution

## Problem Understanding

The problem asks us to identify **lonely nodes** in a binary tree. A lonely node is defined as a node that is the only child of its parent. Importantly, the root node is never considered lonely because it does not have a parent.

The input is the `root` of a binary tree, which may have anywhere from 1 to 1000 nodes. Each node has an integer value between 1 and 1,000,000. The output should be a list containing the values of all lonely nodes in the tree. The order of the nodes in the output does not matter.

Constraints tell us that the tree is moderately sized (up to 1000 nodes), so algorithms with linear time complexity, O(n), are acceptable. Recursive or iterative solutions using a traversal of the tree are both feasible.

Edge cases to consider include a tree with only a root node (no lonely nodes), a tree where every node except the root is lonely (a linear tree), or trees with multiple levels but only some lonely nodes scattered sparsely.

## Approaches

### Brute Force Approach

A brute-force approach would involve traversing the tree and, for each node, checking its children. If a node has only one child, that child is added to the result list. This can be done with a simple recursive traversal or breadth-first search. While this approach works correctly, it still has linear time complexity, O(n), because we must visit every node, and it does not improve on the optimal solution. However, it may be implemented without leveraging any extra insights about tree structure.

### Optimal Approach

The key insight is that a lonely node is defined purely by its parent having exactly one child. Therefore, during a depth-first or breadth-first traversal, we can simply check whether a node has a single child. If it does, we immediately add that child to the result list. This eliminates any unnecessary operations and is conceptually simpler because we only track the nodes that are lonely rather than all nodes.

Because each node is visited exactly once, the optimal approach achieves O(n) time complexity and O(h) space complexity for the recursion stack in DFS (where h is the height of the tree), or O(n) space for a BFS queue in the worst case if the tree is balanced.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Traverse tree and check each node; simple recursive or iterative solution |
| Optimal | O(n) | O(h) | Use DFS or BFS, directly check each node for single child, add child if lonely |

## Algorithm Walkthrough

1. Start at the root of the tree. The root is never lonely, so we do not add it to the result list.
2. Recursively traverse the tree using depth-first search (DFS). For each node, check if it has exactly one child.
3. If a node has only a left child, add the left child's value to the list of lonely nodes.
4. If a node has only a right child, add the right child's value to the list of lonely nodes.
5. Continue the recursion on both the left and right children if they exist.
6. Once all nodes are processed, return the result list.

**Why it works:** Every node is visited exactly once, and whenever a node is the sole child of its parent, it is added to the list. This guarantees that all lonely nodes are captured while no non-lonely nodes are included. The recursive traversal ensures the algorithm covers the entire tree.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
from typing import List, Optional

class Solution:
    def getLonelyNodes(self, root: Optional[TreeNode]) -> List[int]:
        lonely_nodes = []

        def dfs(node: Optional[TreeNode]):
            if not node:
                return
            if node.left and not node.right:
                lonely_nodes.append(node.left.val)
            if node.right and not node.left:
                lonely_nodes.append(node.right.val)
            dfs(node.left)
            dfs(node.right)

        dfs(root)
        return lonely_nodes
```

The implementation uses a helper function `dfs` to traverse the tree recursively. For each node, it checks if the node has only one child and appends that child to `lonely_nodes`. The recursion then continues on both left and right children. This ensures all nodes are examined, and only lonely nodes are added to the list.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func getLonelyNodes(root *TreeNode) []int {
    var lonelyNodes []int

    var dfs func(node *TreeNode)
    dfs = func(node *TreeNode) {
        if node == nil {
            return
        }
        if node.Left != nil && node.Right == nil {
            lonelyNodes = append(lonelyNodes, node.Left.Val)
        }
        if node.Right != nil && node.Left == nil {
            lonelyNodes = append(lonelyNodes, node.Right.Val)
        }
        dfs(node.Left)
        dfs(node.Right)
    }

    dfs(root)
    return lonelyNodes
}
```

The Go implementation mirrors the Python version, using a recursive DFS. Nil checks are used instead of Python's `None`, and slices are appended to store lonely nodes. The recursion structure and logic are otherwise identical.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited exactly once in the DFS traversal. |
| Space | O(h) | Maximum recursion depth is equal to the height of the tree. In the worst case (linear tree), h = n. |

Since every node must be checked to determine if it is lonely, linear time is the best achievable. Space is dominated by the recursion stack.
