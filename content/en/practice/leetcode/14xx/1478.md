**Topics:** Array, Math, Dynamic Programming, Sorting

## Solution

## Problem Understanding

The problem asks us to allocate `k` mailboxes along a street in such a way that the total distance between each house and its nearest mailbox is minimized. The input consists of an array `houses`, where `houses[i]` is the location of the `i-th` house, and an integer `k` representing the number of mailboxes we can place. The output should be a single integer representing the minimum total distance.

The key points to note are that house positions are unique integers, the array size is at most 100, and mailbox locations are not restricted to house positions but can be optimally placed anywhere. The constraints ensure that brute force placement of mailboxes across all positions is impractical, but dynamic programming is feasible.

Edge cases to be mindful of include `k` equal to 1 (all houses share a single mailbox) and `k` equal to `len(houses)` (each house gets its own mailbox, resulting in zero total distance).

## Approaches

### Brute Force Approach

The brute force approach would try all possible ways to place `k` mailboxes along the street. For each combination of mailbox positions, we would calculate the total distance from each house to its nearest mailbox. While this guarantees the correct answer, the number of possible placements grows combinatorially with the number of houses and `k`, making it infeasible for `n = 100`.

### Optimal Approach

The key insight is that for any segment of consecutive houses, the optimal placement of a single mailbox is at the median of that segment. This minimizes the sum of absolute distances. Using this, we can define a dynamic programming solution where `dp[i][k]` represents the minimum distance to cover the first `i` houses with `k` mailboxes.

We precompute the cost for placing a single mailbox over any segment `[i:j]` of houses using the median property. Then, we iterate over all partitions of the first `i` houses into previous segments and use `dp` to find the minimum distance efficiently.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((max_house)^k * n) | O(1) | Tries all possible mailbox placements |
| Optimal (DP + Median) | O(n^2 * k) | O(n^2 + n*k) | Uses precomputed segment costs and DP for efficiency |

## Algorithm Walkthrough

1. **Sort the Houses**: Sorting ensures the houses are in order along the street. This is critical because the optimal mailbox for consecutive houses is at their median.
2. **Precompute Segment Costs**: For each pair `(i, j)` representing a consecutive segment of houses, calculate the minimum total distance if one mailbox covers that segment. This distance is minimized by placing the mailbox at the median of that segment.
3. **Initialize DP Table**: Create a 2D array `dp[i][k]` where `dp[i][k]` represents the minimum total distance to cover the first `i` houses with `k` mailboxes. Initialize all values to infinity except `dp[0][0] = 0` as the base case.
4. **DP Transition**: For each number of mailboxes from 1 to `k` and for each house index `i`, iterate over all possible previous partitions `j < i`. Update `dp[i][k] = min(dp[i][k], dp[j][k-1] + cost[j+1][i])`, where `cost[j+1][i]` is the precomputed segment cost for placing one mailbox covering houses from `j+1` to `i`.
5. **Return Result**: The answer is `dp[n][k]`, the minimum distance to cover all `n` houses with `k` mailboxes.

**Why it works**: The invariant is that the optimal mailbox placement for any segment of consecutive houses is at the median. By building up the solution using dynamic programming, we ensure that every subproblem is solved optimally, guaranteeing the global minimum.

## Python Solution

```python
from typing import List

class Solution:
    def minDistance(self, houses: List[int], k: int) -> int:
        houses.sort()
        n = len(houses)
        
        # Precompute the cost to place one mailbox for any segment [i:j]
        cost = [[0] * n for _ in range(n)]
        for i in range(n):
            for j in range(i, n):
                median = houses[(i + j) // 2]
                for t in range(i, j + 1):
                    cost[i][j] += abs(houses[t] - median)
        
        # Initialize DP table
        dp = [[float('inf')] * (k + 1) for _ in range(n + 1)]
        dp[0][0] = 0
        
        # Fill DP table
        for i in range(1, n + 1):
            for m in range(1, k + 1):
                for j in range(i):
                    dp[i][m] = min(dp[i][m], dp[j][m-1] + cost[j][i-1])
        
        return dp[n][k]
```

**Implementation Walkthrough**:

First, we sort the houses to ensure order. The `cost` array precomputes distances for all segments to allow constant-time lookup in the DP step. The DP table `dp[i][m]` is filled by iterating over all possible previous partitions. Each update considers placing the `m-th` mailbox optimally for the segment `[j:i-1]`. The final result `dp[n][k]` gives the minimal total distance.

## Go Solution

```go
import "math"

func minDistance(houses []int, k int) int {
    n := len(houses)
    sort.Ints(houses)
    
    // Precompute cost for segments
    cost := make([][]int, n)
    for i := 0; i < n; i++ {
        cost[i] = make([]int, n)
        for j := i; j < n; j++ {
            median := houses[(i+j)/2]
            for t := i; t <= j; t++ {
                cost[i][j] += int(math.Abs(float64(houses[t] - median)))
            }
        }
    }
    
    // Initialize DP table
    dp := make([][]int, n+1)
    for i := range dp {
        dp[i] = make([]int, k+1)
        for j := range dp[i] {
            dp[i][j] = math.MaxInt32
        }
    }
    dp[0][0] = 0
    
    // Fill DP table
    for i := 1; i <= n; i++ {
        for m := 1; m <= k; m++ {
            for j := 0; j < i; j++ {
                if dp[j][m-1] != math.MaxInt32 {
                    dp[i][m] = min(dp[i][m], dp[j][m-1]+cost[j][i-1])
                }
            }
        }
    }
    
    return dp[n][k]
}

func min(a, b int) int {
    if a < b { return a }
    return b
}
```

**Go-specific Notes**: The Go implementation mirrors Python but uses `math.MaxInt32` for infinity. Slices are initialized for 2D arrays, and `math.Abs` converts integers to float64 for computation, then back to int.

## Worked Examples

### Example 1: `houses = [1,4,8,10,20], k = 3`

| Segment | Median | Cost |
| --- | --- | --- |
| [1,4] | 1 | 3 |
| [8,10] | 8 | 2 |
| [20] | 20 | 0 |

Optimal DP partitions: `[1,4]`, `[8,10]`, `[20]` → total cost = 5.

### Example 2: `houses = [2,3,5,12,18], k = 2`

| Segment | Median | Cost |
| --- | --- | --- |
| [2,3,5] | 3 | 3 |
| [12,18] | 15 | 6 |

Optimal DP partitions: `[2,3,5]`, `[12,18]` → total cost = 9.

## Complexity Analysis

| Measure | Complexity | Explanation |

| Space | O(n^2 + n*k) | Cost matrix O(n^2), DP table O(n*k) |

The DP approach scales efficiently for n ≤ 100, as required by constraints.
