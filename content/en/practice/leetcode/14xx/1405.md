## Solution

## Problem Understanding

The problem is asking us to construct the **longest possible string** consisting only of the characters `'a'`, `'b'`, and `'c'`, under two key restrictions. First, the string cannot contain three identical letters in a row. Second, the string cannot use more letters than the given counts `a`, `b`, and `c`.

The input consists of three non-negative integers, `a`, `b`, and `c`, which represent the maximum number of times `'a'`, `'b'`, and `'c'` can appear in the resulting string, respectively. The expected output is a string that is as long as possible while satisfying the "happy string" conditions. If it is impossible to construct any string that satisfies the rules (all counts are zero), the output is an empty string `""`.

The constraints `0 <= a, b, c <= 100` indicate that the input size is relatively small, so a solution with time complexity up to roughly `O((a+b+c)^2)` could still work. Importantly, the guarantee `a + b + c > 0` ensures that at least one character is available, so the output will never need to handle a completely empty input in terms of input validation.

Key edge cases to consider include when one character dominates the counts, such as `a = 7, b = 1, c = 0`, or when all counts are very low. Another edge case is when two counts are equal and larger than the third, which can influence the greedy choice of characters.

## Approaches

A brute-force approach would attempt to recursively generate all possible strings using `'a'`, `'b'`, and `'c'`, keeping track of counts and rejecting strings with three consecutive identical letters. While this approach is guaranteed to find the correct answer, it is infeasible because the number of combinations grows exponentially with the sum `a + b + c`. Specifically, for each position in the string, there are up to three choices, leading to `O(3^(a+b+c))` time complexity.

The key insight for an optimal solution is that we can **greedily always use the character with the largest remaining count**, as long as it does not create three consecutive identical letters. If adding that character would cause three in a row, we pick the next highest remaining character. This ensures the string is as long as possible while respecting the "no three consecutive identical letters" rule. A **max heap (priority queue)** is a natural choice to efficiently pick the character with the largest remaining count at each step.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(3^(a+b+c)) | O(a+b+c) | Recursively generates all valid strings; guaranteed correct but infeasible |
| Optimal | O(a+b+c) | O(1) | Greedy selection using a max heap to append the most abundant character without violating constraints |

## Algorithm Walkthrough

1. Initialize a max heap containing tuples of the form `(count, char)` for `'a'`, `'b'`, and `'c'` only if their count is greater than zero. This allows efficient selection of the most available character at each step.
2. Initialize an empty list `result` to construct the string.
3. While the heap is not empty, pop the character with the largest remaining count. Check if adding it would create three consecutive identical letters. If it does not, append the character once or twice (whichever keeps the count valid) and decrement the count accordingly. Push it back if the remaining count is still positive.
4. If adding the most abundant character would violate the three-in-a-row rule, pop the next character in the heap. Append it once, decrement its count, and push it back if positive. Push back the previous character as well.
5. Repeat steps 3 and 4 until the heap is empty or no further characters can be added without violating the consecutive constraint.
6. Join the list `result` into a string and return it.

**Why it works**: The algorithm maintains the invariant that at each step the character with the largest remaining count is considered first, which maximizes the total string length. The check for three consecutive identical letters ensures the string remains "happy." By using a heap, we always make the globally optimal greedy choice for the next character.

## Python Solution

```python
import heapq

class Solution:
    def longestDiverseString(self, a: int, b: int, c: int) -> str:
        max_heap = []
        for count, char in [(-a, 'a'), (-b, 'b'), (-c, 'c')]:
            if count != 0:
                heapq.heappush(max_heap, (count, char))
        
        result = []
        
        while max_heap:
            count1, char1 = heapq.heappop(max_heap)
            if len(result) >= 2 and result[-1] == result[-2] == char1:
                if not max_heap:
                    break
                count2, char2 = heapq.heappop(max_heap)
                result.append(char2)
                count2 += 1
                if count2 != 0:
                    heapq.heappush(max_heap, (count2, char2))
                heapq.heappush(max_heap, (count1, char1))
            else:
                result.append(char1)
                count1 += 1
                if count1 != 0:
                    heapq.heappush(max_heap, (count1, char1))
        
        return ''.join(result)
```

In this implementation, we first populate a max heap with negative counts so that Python's `heapq` behaves like a max heap. The loop ensures that at each iteration, we pick the character with the highest remaining count, only deferring it when it would create three consecutive identical letters. After appending a character, we push it back into the heap if there are remaining counts.

## Go Solution

```go
package main

import (
    "container/heap"
    "strings"
)

type CharCount struct {
    count int
    char  byte
}

type MaxHeap []CharCount

func (h MaxHeap) Len() int           { return len(h) }
func (h MaxHeap) Less(i, j int) bool { return h[i].count > h[j].count }
func (h MaxHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }

func (h *MaxHeap) Push(x any) {
    *h = append(*h, x.(CharCount))
}

func (h *MaxHeap) Pop() any {
    n := len(*h)
    x := (*h)[n-1]
    *h = (*h)[:n-1]
    return x
}

func longestDiverseString(a int, b int, c int) string {
    h := &MaxHeap{}
    if a > 0 {
        heap.Push(h, CharCount{a, 'a'})
    }
    if b > 0 {
        heap.Push(h, CharCount{b, 'b'})
    }
    if c > 0 {
        heap.Push(h, CharCount{c, 'c'})
    }
    heap.Init(h)

    var sb strings.Builder

    for h.Len() > 0 {
        first := heap.Pop(h).(CharCount)
        n := sb.Len()
        if n >= 2 && sb.String()[n-1] == first.char && sb.String()[n-2] == first.char {
            if h.Len() == 0 {
                break
            }
            second := heap.Pop(h).(CharCount)
            sb.WriteByte(second.char)
            second.count--
            if second.count > 0 {
                heap.Push(h, second)
            }
            heap.Push(h, first)
        } else {
            sb.WriteByte(first.char)
            first.count--
            if first.count > 0 {
                heap.Push(h, first)
            }
        }
    }

    return sb.String()
}
```

The Go solution mirrors the Python logic but uses a custom max heap type for `CharCount` structs. We use `strings.Builder` for efficient string concatenation and handle the edge case of consecutive letters identically to Python.
