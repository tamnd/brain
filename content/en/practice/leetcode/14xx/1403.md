## Solution

## Problem Understanding

The problem asks us to select a subsequence of an integer array `nums` such that the sum of the chosen elements is **strictly greater** than the sum of the elements not chosen. A subsequence is formed by removing zero or more elements without changing the order of the remaining elements. Among all valid subsequences, we are required to return the one with the **minimum number of elements**. If multiple subsequences satisfy the minimum size condition, we pick the one with the **largest total sum**. Finally, the result must be sorted in **non-increasing order**.

The input `nums` is a list of integers, where each integer is between 1 and 100, and the length of the list is between 1 and 500. This means the array is small enough that even sorting-based solutions will be efficient, and we do not need to consider extremely large input optimizations.

Important edge cases include arrays with all identical elements, arrays with only one element, and arrays where multiple minimal-size subsequences exist but differ in sum.

The problem guarantees that a unique solution exists, which allows us to focus purely on the greedy selection strategy without worrying about ties.

## Approaches

### Brute Force Approach

The brute-force approach would involve generating all possible subsequences of `nums`, computing their sums, and checking if the sum of a subsequence is strictly greater than the sum of the remaining elements. Among all valid subsequences, we would select the one with the smallest size and then the largest sum. While this approach is correct, it is computationally infeasible because an array of length `n` has `2^n` possible subsequences. For `n = 500`, this is astronomically large and impossible to compute within reasonable time.

### Optimal Approach

The key insight is that larger numbers contribute more to the sum. Therefore, to minimize the size of the subsequence while maximizing its sum, we should select the largest numbers first. We can implement this using a **greedy approach**:

1. Sort `nums` in **descending order**.
2. Iteratively add numbers from largest to smallest until the sum of selected numbers is strictly greater than the sum of remaining numbers.
3. Return the selected numbers as they are already in non-increasing order.

This approach works because choosing the largest numbers first ensures that we reach a sum greater than the remaining elements using the fewest elements possible, satisfying both the minimum size and maximum sum conditions.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Generate all subsequences and compute sums |
| Optimal | O(n log n) | O(n) | Sort array and select largest elements greedily |

## Algorithm Walkthrough

1. Compute the **total sum** of the array. This will help us track the sum of the remaining elements as we add elements to the subsequence.
2. Sort the array `nums` in **descending order**. This ensures that we always pick the largest elements first, which minimizes the number of elements required.
3. Initialize an empty list `subsequence` and a variable `current_sum = 0` to keep track of the sum of the selected elements.
4. Iterate through each element `num` in the sorted array. For each element:

- Add `num` to `subsequence`.
- Add `num` to `current_sum`.
- Subtract `num` from `remaining_sum` (which starts as the total sum of the array).
- Check if `current_sum` is **strictly greater** than `remaining_sum`. If true, break the loop because we have a valid subsequence.
5. Return `subsequence`. It is already sorted in non-increasing order.

**Why it works:** By choosing the largest numbers first, we guarantee that the sum of selected elements grows faster than the sum of unselected elements. This ensures the subsequence is minimal in size. Since we always pick the largest available numbers, we also maximize the total sum if multiple minimal-size subsequences exist.

## Python Solution

```python
from typing import List

class Solution:
    def minSubsequence(self, nums: List[int]) -> List[int]:
        total_sum = sum(nums)
        nums.sort(reverse=True)
        
        subsequence = []
        current_sum = 0
        
        for num in nums:
            subsequence.append(num)
            current_sum += num
            if current_sum > total_sum - current_sum:
                break
                
        return subsequence
```

The implementation first calculates the total sum and sorts the array in descending order. It iteratively adds elements to the subsequence until the selected sum surpasses the sum of remaining elements. The greedy strategy ensures both minimum size and maximum sum.

## Go Solution

```go
import "sort"

func minSubsequence(nums []int) []int {
    totalSum := 0
    for _, num := range nums {
        totalSum += num
    }

    sort.Slice(nums, func(i, j int) bool {
        return nums[i] > nums[j]
    })

    subsequence := []int{}
    currentSum := 0

    for _, num := range nums {
        subsequence = append(subsequence, num)
        currentSum += num
        if currentSum > totalSum - currentSum {
            break
        }
    }

    return subsequence
}
```

The Go solution mirrors the Python version. It handles slices and sorting carefully. Nil slices are not an issue because the input guarantees at least one element. Integer overflow is not a concern due to constraints (`nums[i] <= 100` and `len(nums) <= 500`).

## Edge Cases
