## Solution

## Problem Understanding

The problem asks us to find the length of the longest contiguous subarray of 1's in a binary array `nums` after removing exactly one element. The input is a list of integers containing only 0s and 1s. The output is a single integer representing the maximum possible length of consecutive 1's after deleting one element, whether it is a 0 or a 1. If the array contains all 0's, then the output should be 0, because deleting any 0 will not yield any 1's.

The constraints specify that the array length can be up to 100,000. This implies that a naive solution with nested loops, which has O(n²) complexity, will be too slow. We need an algorithm that runs in linear time or close to linear.

Important edge cases include arrays with all 1's (where we must still delete one element), arrays with all 0's (which should return 0), and arrays with alternating 1's and 0's. Single-element arrays are also a special case because deleting the only element leaves an empty array, which should return 0.

## Approaches

A brute-force approach involves iterating over each element, deleting it, and computing the length of the longest consecutive 1's in the remaining array. While this approach is correct, it is inefficient because computing the length of consecutive 1's for each deletion requires O(n) time, and repeating this for all n elements leads to O(n²) complexity.

The key insight for an optimal solution is to use a sliding window that allows for at most one zero in the current window. By maintaining a window where we can "skip" one zero, we can effectively calculate the maximum length of consecutive 1's after deleting a single element in linear time. This is a variant of the two-pointer or sliding window technique where the window dynamically adjusts based on the number of zeros it contains.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Delete each element and scan for longest 1's sequence |
| Optimal | O(n) | O(1) | Sliding window with at most one zero to simulate deletion |

## Algorithm Walkthrough

1. Initialize two pointers `left` and `right` at the beginning of the array. `left` represents the start of the current window, and `right` will iterate over each element.
2. Maintain a variable `zero_count` to track the number of zeros in the current window.
3. Iterate through the array with the `right` pointer. If `nums[right]` is 0, increment `zero_count`.
4. While `zero_count` exceeds 1, move the `left` pointer to the right, decrementing `zero_count` when a zero is passed. This ensures the window contains at most one zero.
5. At each step, calculate the current window size as `right - left`. We use `right - left` instead of `right - left + 1` because we simulate deletion by allowing one zero to be skipped.
6. Keep track of the maximum window size during the iteration.
7. Return the maximum window size found.

Why it works: The sliding window ensures that at any time, the window contains at most one zero. This models deleting exactly one element from the array. The window dynamically adjusts to maximize the number of consecutive 1's while accounting for the one deletion.

## Python Solution

```python
from typing import List

class Solution:
    def longestSubarray(self, nums: List[int]) -> int:
        left = 0
        max_len = 0
        zero_count = 0

        for right in range(len(nums)):
            if nums[right] == 0:
                zero_count += 1

            while zero_count > 1:
                if nums[left] == 0:
                    zero_count -= 1
                left += 1

            max_len = max(max_len, right - left)

        return max_len
```

The Python implementation initializes pointers and a counter for zeros. It iterates over the array with the `right` pointer. When more than one zero is encountered in the window, it increments `left` until the window contains at most one zero. The maximum length is computed as the difference between `right` and `left`, which models skipping a zero. This produces the optimal solution in O(n) time.

## Go Solution

```go
func longestSubarray(nums []int) int {
    left := 0
    maxLen := 0
    zeroCount := 0

    for right := 0; right < len(nums); right++ {
        if nums[right] == 0 {
            zeroCount++
        }

        for zeroCount > 1 {
            if nums[left] == 0 {
                zeroCount--
            }
            left++
        }

        if right-left > maxLen {
            maxLen = right - left
        }
    }

    return maxLen
}
```

The Go implementation mirrors the Python approach. We use `left` and `right` pointers with a `zeroCount` to track zeros in the window. The main difference is explicit type handling and avoiding nil checks, which are unnecessary since slices in Go are always non-nil. The logic and sliding window are otherwise identical.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element is visited at most twice (once by right, once by left pointer) |
| Space | O(1) | Only a few integer variables are used, no extra arrays |

The algorithm is linear because each element enters and leaves the sliding window at most once. The space is constant because no additional storage proportional to input size is required.
