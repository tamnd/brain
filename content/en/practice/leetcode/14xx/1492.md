## Solution

## Problem Understanding

The problem asks us to find the `kth` factor of a given integer `n`. A factor of `n` is an integer `i` such that `n % i == 0`. The factors must be considered in **ascending order**, and if there are fewer than `k` factors, we should return `-1`.

The inputs `n` and `k` are both positive integers. `n` represents the number we want to factorize, and `k` represents the position in the ordered list of factors we want to retrieve. The constraints `1 <= k <= n <= 1000` tell us that `n` is small, so a simple iteration approach will be efficient enough.

Important edge cases include when `k` is larger than the total number of factors of `n`, when `n` is prime (so it has only two factors, `1` and `n`), and when `n` is `1` (which has exactly one factor). The problem guarantees valid positive integers, so we do not need to handle `0` or negative numbers.

## Approaches

The simplest approach is a brute-force iteration over all integers from `1` to `n`. For each number `i`, check if `i` divides `n`. If it does, add it to a list or increment a counter. When the counter reaches `k`, we return `i`. This works because we are guaranteed to find factors in ascending order naturally by iterating from `1` upwards. However, this requires iterating up to `n`, which can be unnecessary when a faster approach exists.

The optimal approach uses the insight that factors come in pairs. For any factor `i` of `n`, there exists a corresponding factor `n // i`. We only need to iterate up to the square root of `n` to find all factor pairs. We maintain a list of factors less than or equal to the square root and a separate list for factors greater than the square root. By combining the two lists correctly, we can access the `kth` factor efficiently without traversing all integers up to `n`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Iterate through all numbers from 1 to n, counting factors |
| Optimal | O(sqrt(n)) | O(sqrt(n)) | Only iterate up to sqrt(n) and use factor pairs |

## Algorithm Walkthrough

1. Initialize an empty list `small_factors` to store factors less than or equal to the square root of `n`.
2. Initialize another empty list `large_factors` to store factors larger than the square root.
3. Iterate `i` from 1 to `int(sqrt(n))` inclusive.
4. If `n % i == 0`, append `i` to `small_factors`.
5. If `i` is not equal to `n // i`, append `n // i` to `large_factors`. This avoids double-counting the square root if `n` is a perfect square.
6. After iteration, check if `k` is less than or equal to the length of `small_factors`. If so, return `small_factors[k-1]`.
7. Otherwise, subtract the length of `small_factors` from `k` and check if `k` is within the length of `large_factors`. If so, return `large_factors[-k]` to maintain ascending order.
8. If `k` exceeds the total number of factors, return `-1`.

**Why it works:** This approach leverages the factor pairing property and ensures that all factors are counted in ascending order without iterating all numbers up to `n`. The separation into `small_factors` and `large_factors` preserves order and allows efficient indexing.

## Python Solution

```python
class Solution:
    def kthFactor(self, n: int, k: int) -> int:
        import math
        small_factors = []
        large_factors = []
        
        for i in range(1, int(math.isqrt(n)) + 1):
            if n % i == 0:
                small_factors.append(i)
                if i != n // i:
                    large_factors.append(n // i)
        
        if k <= len(small_factors):
            return small_factors[k-1]
        k -= len(small_factors)
        if k <= len(large_factors):
            return large_factors[-k]
        return -1
```

The Python implementation directly follows the optimal algorithm. The `math.isqrt(n)` efficiently computes the integer square root. The lists `small_factors` and `large_factors` store factors below and above the square root. The indexing logic ensures that we return the correct factor in ascending order.

## Go Solution

```go
import "math"

func kthFactor(n int, k int) int {
    smallFactors := []int{}
    largeFactors := []int{}
    
    sqrtN := int(math.Sqrt(float64(n)))
    
    for i := 1; i <= sqrtN; i++ {
        if n % i == 0 {
            smallFactors = append(smallFactors, i)
            if i != n / i {
                largeFactors = append(largeFactors, n / i)
            }
        }
    }
    
    if k <= len(smallFactors) {
        return smallFactors[k-1]
    }
    k -= len(smallFactors)
    if k <= len(largeFactors) {
        return largeFactors[len(largeFactors) - k]
    }
    return -1
}
```

The Go implementation mirrors the Python logic. We use slices instead of lists, and integer square root is computed using `math.Sqrt` with type conversion. Slice indexing follows the same logic to access the `kth` factor.
