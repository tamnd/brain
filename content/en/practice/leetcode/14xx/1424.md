## Solution

## Problem Understanding

The problem asks us to traverse a 2D jagged array `nums` diagonally and return all elements in the order of diagonals. In other words, elements are grouped by their “diagonal index,” which is defined as the sum of their row and column indices. For example, `nums[i][j]` belongs to diagonal `i + j`. Within each diagonal, elements must be output starting from the bottom-most row to the top-most row. This creates a zigzag-like diagonal traversal, not simply left-to-right row traversal.

The input is a jagged array, meaning each row can have a different number of elements. The constraints tell us that the total number of elements across all rows is at most `10^5`, so solutions that iterate over all elements a small number of times are feasible. Each element is guaranteed to be a positive integer, and there is at least one element in total. Edge cases involve rows of different lengths, a single row, or a single column, which must not break the traversal logic.

## Approaches

### Brute Force

The brute-force approach would involve simulating a 2D grid traversal using indices and trying to compute the order of diagonals dynamically. One could repeatedly check for the next element in the diagonal order, but this becomes messy and inefficient due to the jagged nature of the array. Keeping track of visited elements and jumping around would require extra bookkeeping, potentially leading to O(n^2) time complexity where n is the total number of elements, which is not acceptable for `sum(nums[i].length) = 10^5`.

### Optimal Approach

The key insight is that each element belongs to a diagonal identified by the sum of its row and column indices (`i + j`). If we use a hash map (or dictionary) to map each diagonal index to a list of elements, we can append elements as we iterate through `nums`. Since within a diagonal we need elements from the bottom-most row first, we can insert at the start of the list or append normally and reverse each list before combining all diagonals. Finally, iterating over the diagonals in increasing order of their index gives the correct traversal.

This approach leverages the property that diagonal indices are unique sums of coordinates and that the total number of elements is linear. Using a hash map allows grouping in O(n) time and ensures correct ordering without complicated index arithmetic.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Simulates diagonal traversal explicitly, inefficient for jagged arrays |
| Optimal | O(n) | O(n) | Groups elements by diagonal index in a hash map, then outputs in order |

## Algorithm Walkthrough

1. Initialize an empty hash map `diagonals` where the key is the diagonal index `i + j` and the value is a list of elements in that diagonal.
2. Iterate over each row `i` in `nums`.
3. Within each row, iterate over each column `j`.
4. Compute the diagonal index as `i + j`.
5. Insert the current element at the start of `diagonals[i + j]` to ensure bottom-to-top order within the diagonal.
6. After processing all elements, iterate over diagonal indices in increasing order.
7. Concatenate all lists of elements from each diagonal into a single result list.
8. Return the result list.

**Why it works:** By using `i + j` as the key, we group elements correctly by diagonals. Inserting elements at the start ensures the bottom-most element in a diagonal comes first. Iterating diagonals in order guarantees the correct global diagonal traversal.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def findDiagonalOrder(self, nums: List[List[int]]) -> List[int]:
        diagonals = defaultdict(list)
        
        for i, row in enumerate(nums):
            for j, val in enumerate(row):
                diagonals[i + j].insert(0, val)
        
        result = []
        for k in range(max(diagonals.keys()) + 1):
            result.extend(diagonals[k])
        
        return result
```

The Python implementation uses a `defaultdict` to automatically create lists for each diagonal. Each element is inserted at the beginning of its diagonal list to satisfy the bottom-to-top requirement. After filling the dictionary, the diagonals are concatenated in order.

## Go Solution

```go
func findDiagonalOrder(nums [][]int) []int {
    diagonals := make(map[int][]int)
    maxKey := 0
    
    for i, row := range nums {
        for j, val := range row {
            key := i + j
            diagonals[key] = append([]int{val}, diagonals[key]...)
            if key > maxKey {
                maxKey = key
            }
        }
    }
    
    result := []int{}
    for k := 0; k <= maxKey; k++ {
        result = append(result, diagonals[k]...)
    }
    
    return result
}
```

In Go, we use a map to store slices representing diagonals. Appending at the start of a slice is done by creating a new slice with the element first and the existing slice afterwards. `maxKey` tracks the highest diagonal index for iteration.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element is visited once and inserted at the start of a list. Concatenation also visits each element once. |
| Space | O(n) | Hash map stores each element once. Result array stores all elements. |

The linear complexity is justified because the total number of elements is `sum(nums[i].length)` and all operations are proportional to this count.
