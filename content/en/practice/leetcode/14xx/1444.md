## Solution

## Problem Understanding

This problem asks us to count how many valid ways we can divide a rectangular pizza into exactly `k` non empty pieces, where every piece must contain at least one apple.

The pizza is represented as a grid of characters:

- `'A'` means the cell contains an apple
- `'.'` means the cell is empty

A cut can be made either horizontally or vertically:

- A horizontal cut splits the pizza into an upper piece and a lower piece
- A vertical cut splits the pizza into a left piece and a right piece

After each cut:

- If the cut is horizontal, the upper piece is immediately given away
- If the cut is vertical, the left piece is immediately given away
- The remaining piece continues to be cut further

We must count all distinct sequences of cuts that produce exactly `k` pieces, where every resulting piece contains at least one apple.

The answer can become very large, so we return it modulo `10^9 + 7`.

The constraints are important:

- `rows, cols <= 50`
- `k <= 10`

A naive recursive exploration of every possible cutting sequence would grow exponentially and become infeasible. However, the relatively small value of `k` suggests that dynamic programming with memoization is likely intended.

An important detail is that after each cut, only one remaining subrectangle continues to be processed. This means each recursive state can be represented by:

- The top-left corner of the remaining pizza
- The number of cuts still needed

Several edge cases are important:

- A pizza with no apples in some regions
- `k = 1`, where we simply check whether the entire pizza contains at least one apple
- Thin pizzas with only one row or one column
- Situations where many cuts are geometrically possible but invalid because one side would contain no apples

Efficiently checking whether a subrectangle contains apples is the key optimization.

## Approaches

### Brute Force Approach

The brute force solution recursively tries every possible horizontal and vertical cut at every stage.

For a given subrectangle:

1. Try every horizontal cut position
2. Check whether the upper piece contains an apple
3. Recursively continue cutting the lower piece
4. Try every vertical cut position
5. Check whether the left piece contains an apple
6. Recursively continue cutting the right piece

This produces the correct answer because it explores every valid sequence of cuts.

However, this approach becomes extremely slow because:

- The same subproblems are recomputed many times
- Checking whether a subrectangle contains apples naively costs additional time
- The recursion tree grows exponentially

With up to `50 x 50` cells, brute force is completely impractical.

### Optimal Approach, Prefix Sum + Memoized Dynamic Programming

The key observation is that many recursive states repeat.

If we know:

- The current top-left corner `(row, col)`
- How many pieces still need to be formed

then the remaining subrectangle is uniquely determined.

This makes the problem ideal for memoization.

Another important optimization is fast apple counting. We frequently need to answer:

> Does this subrectangle contain at least one apple?

Using a 2D prefix sum allows us to answer this in `O(1)` time.

The final optimized solution combines:

- 2D prefix sums for constant time apple queries
- DFS recursion for exploring cuts
- Memoization for avoiding repeated computation

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Exponential | Exponential | Recomputes identical subproblems repeatedly |
| Optimal | O(k * rows * cols * (rows + cols)) | O(k * rows * cols) | Uses memoization and prefix sums |

## Algorithm Walkthrough

### Step 1, Build a Prefix Sum Matrix

We preprocess the pizza into a suffix-style prefix sum matrix.

For each position `(r, c)`, store:

- The number of apples in the subrectangle from `(r, c)` to the bottom-right corner

This allows us to determine whether any subrectangle contains apples in constant time.

If `apples(r1, c1)` is greater than zero, then the subrectangle starting at `(r1, c1)` contains at least one apple.

### Step 2, Define the DP State

We define:

```
dp(row, col, cutsRemaining)
```

This represents:

> The number of ways to cut the subrectangle starting at `(row, col)` into `cutsRemaining + 1` valid pieces.

We use memoization so each state is computed only once.

### Step 3, Handle the Base Case

If `cutsRemaining == 0`:

- No more cuts are needed
- The remaining subrectangle itself must contain at least one apple

If it does, return `1`, otherwise return `0`.

### Step 4, Try Horizontal Cuts

For every possible horizontal cut line:

- The upper portion is given away
- The lower portion continues recursively

We only allow the cut if the upper portion contains at least one apple.

If valid:

```
result += dp(nextRow, col, cutsRemaining - 1)
```

### Step 5, Try Vertical Cuts

Similarly, for every vertical cut:

- The left portion is given away
- The right portion continues recursively

We only proceed if the left portion contains an apple.

### Step 6, Store and Return the Result

Store the computed result in the memo table and return it.

All arithmetic is performed modulo `10^9 + 7`.

### Why it works

The algorithm works because every recursive state uniquely represents a remaining pizza subrectangle and the number of cuts still required.

For each state, we exhaustively explore every valid next cut. Since each recursive call represents a strictly smaller remaining pizza, the recursion eventually terminates.

Memoization guarantees that identical states are computed only once, and the prefix sum guarantees correct and efficient apple existence checks.

Therefore, the algorithm counts every valid cutting sequence exactly once.

## Python Solution

```python
from typing import List
from functools import lru_cache

class Solution:
    def ways(self, pizza: List[str], k: int) -> int:
        MOD = 10**9 + 7

        rows = len(pizza)
        cols = len(pizza[0])

        # suffix_sum[r][c] = number of apples
        # in rectangle (r,c) -> bottom-right
        suffix_sum = [[0] * (cols + 1) for _ in range(rows + 1)]

        for r in range(rows - 1, -1, -1):
            for c in range(cols - 1, -1, -1):
                suffix_sum[r][c] = (
                    suffix_sum[r + 1][c]
                    + suffix_sum[r][c + 1]
                    - suffix_sum[r + 1][c + 1]
                    + (1 if pizza[r][c] == 'A' else 0)
                )

        def has_apple(r1: int, c1: int, r2: int, c2: int) -> bool:
            apples = (
                suffix_sum[r1][c1]
                - suffix_sum[r2][c1]
                - suffix_sum[r1][c2]
                + suffix_sum[r2][c2]
            )
            return apples > 0

        @lru_cache(None)
        def dfs(row: int, col: int, cuts_remaining: int) -> int:
            # Remaining piece must contain an apple
            if suffix_sum[row][col] == 0:
                return 0

            # No more cuts needed
            if cuts_remaining == 0:
                return 1

            ways_count = 0

            # Horizontal cuts
            for next_row in range(row + 1, rows):
                if has_apple(row, col, next_row, cols):
                    ways_count += dfs(
                        next_row,
                        col,
                        cuts_remaining - 1
                    )

            # Vertical cuts
            for next_col in range(col + 1, cols):
                if has_apple(row, col, rows, next_col):
                    ways_count += dfs(
                        row,
                        next_col,
                        cuts_remaining - 1
                    )

            return ways_count % MOD

        return dfs(0, 0, k - 1)
```

The implementation begins by constructing a suffix sum matrix. Instead of storing cumulative sums from the top-left corner, this version stores apple counts from each position to the bottom-right corner. This makes subrectangle queries convenient for this problem because every recursive state represents a suffix-like rectangle.

The `has_apple()` helper computes whether a given subrectangle contains at least one apple using inclusion-exclusion arithmetic.

The recursive `dfs()` function is the core dynamic programming state. Each state is uniquely identified by:

- The current top row
- The current left column
- The remaining number of cuts

Before exploring cuts, the function first checks whether the current subrectangle contains any apples at all. If not, the state is invalid.

The horizontal and vertical loops try every possible cut line. A cut is only considered valid if the piece being given away contains at least one apple.

The `@lru_cache` decorator memoizes results automatically, preventing repeated recomputation of identical states.

## Go Solution

```go
package main

func ways(pizza []string, k int) int {
	const MOD = 1000000007

	rows := len(pizza)
	cols := len(pizza[0])

	// suffixSum[r][c] = apples from (r,c) to bottom-right
	suffixSum := make([][]int, rows+1)
	for i := range suffixSum {
		suffixSum[i] = make([]int, cols+1)
	}

	for r := rows - 1; r >= 0; r-- {
		for c := cols - 1; c >= 0; c-- {
			apple := 0
			if pizza[r][c] == 'A' {
				apple = 1
			}

			suffixSum[r][c] =
				suffixSum[r+1][c] +
					suffixSum[r][c+1] -
					suffixSum[r+1][c+1] +
					apple
		}
	}

	hasApple := func(r1, c1, r2, c2 int) bool {
		apples :=
			suffixSum[r1][c1] -
				suffixSum[r2][c1] -
				suffixSum[r1][c2] +
				suffixSum[r2][c2]

		return apples > 0
	}

	// memo[row][col][cuts]
	memo := make([][][]int, rows)
	for r := 0; r < rows; r++ {
		memo[r] = make([][]int, cols)
		for c := 0; c < cols; c++ {
			memo[r][c] = make([]int, k)
			for cuts := 0; cuts < k; cuts++ {
				memo[r][c][cuts] = -1
			}
		}
	}

	var dfs func(int, int, int) int

	dfs = func(row, col, cutsRemaining int) int {
		if suffixSum[row][col] == 0 {
			return 0
		}

		if cutsRemaining == 0 {
			return 1
		}

		if memo[row][col][cutsRemaining] != -1 {
			return memo[row][col][cutsRemaining]
		}

		result := 0

		// Horizontal cuts
		for nextRow := row + 1; nextRow < rows; nextRow++ {
			if hasApple(row, col, nextRow, cols) {
				result = (result + dfs(
					nextRow,
					col,
					cutsRemaining-1,
				)) % MOD
			}
		}

		// Vertical cuts
		for nextCol := col + 1; nextCol < cols; nextCol++ {
			if hasApple(row, col, rows, nextCol) {
				result = (result + dfs(
					row,
					nextCol,
					cutsRemaining-1,
				)) % MOD
			}
		}

		memo[row][col][cutsRemaining] = result
		return result
	}

	return dfs(0, 0, k-1)
}
```

The Go implementation follows the same algorithmic structure as the Python version, but explicit memoization arrays are used instead of `lru_cache`.

The memo table is initialized with `-1` to distinguish uncomputed states from computed states with value `0`.

Go requires explicit recursive function declarations using a function variable before assignment, which is why `dfs` is declared first and assigned afterward.

All arithmetic is kept within integer bounds using modulo operations during accumulation.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(k * rows * cols * (rows + cols)) | Each DP state tries all horizontal and vertical cuts |
| Space | O(k * rows * cols) | Memoization table plus recursion stack |

There are at most:

```
rows * cols * k
```

distinct states.

For each state, we may try:

- Up to `rows` horizontal cuts
- Up to `cols` vertical cuts

Therefore, the total runtime becomes:

```
O(k * rows * cols * (rows + cols))
```

The prefix sum matrix enables constant time apple checks, which is essential for achieving this complexity.

## Edge Cases

### Edge Case 1, Pizza Contains No Apples

A pizza like:

```
["...", "..."]
```

contains no apples at all.

A naive implementation might still attempt recursive cuts, producing invalid results. The solution prevents this immediately by checking:

```
if suffix_sum[row][col] == 0:
    return 0
```

This guarantees that any apple-free subrectangle is rejected instantly.

### Edge Case 2, k Equals 1

When `k = 1`, no cuts are required. The entire pizza itself becomes the only piece.

This is easy to mishandle if recursion assumes at least one cut occurs. The base case correctly handles this:

```
if cuts_remaining == 0:
    return 1
```

provided the current subrectangle contains at least one apple.

### Edge Case 3, Thin Pizzas

Single-row or single-column pizzas can only be cut in one direction.

For example:

```
["AAA"]
```

only allows vertical cuts.

A buggy implementation might incorrectly attempt invalid horizontal cuts or access out-of-bounds indices. The loops:

```
for next_row in range(row + 1, rows)
for next_col in range(col + 1, cols)
```

naturally avoid impossible cut directions.

### Edge Case 4, Geometrically Valid but Apple-Invalid Cuts

Some cuts split the pizza physically but leave one piece without apples.

For example:

```
["A..",
 "...",
 "..."]
```

A horizontal cut may create an empty lower region.

The `has_apple()` function ensures every discarded piece contains at least one apple before recursion continues. This prevents invalid partitions from contributing to the answer.
