## Solution

## Problem Understanding

The problem asks us to find **two non-overlapping sub-arrays** in a given array of positive integers, `arr`, such that the sum of each sub-array equals a given `target`. Among all possible valid pairs of sub-arrays, we must choose the pair that **minimizes the sum of their lengths**. If no such pair exists, we return `-1`.

The input is a list of integers where `1 <= arr.length <= 10^5` and each integer is between `1` and `1000`. The `target` sum can be very large, up to `10^8`. This indicates that brute-force approaches that check all sub-array combinations are likely too slow.

Edge cases include arrays where:

- No sub-array sums to `target`, which should return `-1`.
- Only one sub-array sums to `target`, which is also insufficient, returning `-1`.
- Multiple sub-arrays sum to `target`, but they overlap; we must ensure non-overlapping selection.
- The minimum sum of lengths requires skipping a shorter sub-array in favor of a better combination later.

## Approaches

**Brute Force:** The naive approach would be to iterate over all possible sub-arrays and calculate their sums. For each valid sub-array with sum equal to `target`, we store its length. Then, we check all pairs of these valid sub-arrays to find non-overlapping pairs and compute the minimal sum of lengths. This guarantees correctness but has **O(n^3)** complexity (O(n^2) for sub-arrays and O(n^2) for comparing pairs), which is too slow for `n = 10^5`.

**Optimal Approach:** We can exploit the **sliding window technique** combined with a **prefix minimum array** to solve this efficiently. Since all numbers are positive, a sliding window can efficiently find all sub-arrays that sum to `target`. We then track the **minimum length of any sub-array ending before index `i`**. For each new sub-array found, we check if there exists a previous non-overlapping sub-array to form a valid pair. This reduces the problem to **O(n)** time complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n^2) | Checks all sub-array sums and all pairs for overlap |
| Sliding Window + Prefix Minimum | O(n) | O(n) | Uses sliding window to find sub-arrays and prefix array to track minimum lengths efficiently |

## Algorithm Walkthrough

1. Initialize `left_min` array of size `n` to store the minimum length of any valid sub-array ending at or before index `i`. Fill it with infinity (`inf`). Initialize `min_len` as infinity to track the current minimum length, and `result` as infinity to store the minimum sum of lengths of two valid sub-arrays.
2. Use a sliding window with `start` and `end` pointers to find all sub-arrays summing to `target`. Initialize `curr_sum = 0` and `start = 0`.
3. Iterate through `arr` with the `end` pointer. Add `arr[end]` to `curr_sum`. While `curr_sum > target`, subtract `arr[start]` and increment `start`.
4. If `curr_sum == target`, calculate the length of this sub-array `length = end - start + 1`. Update `left_min[end] = min(left_min[end-1], length)` and update `min_len = min(min_len, length)`.
5. For each valid sub-array, check if there exists a previous non-overlapping sub-array: if `start > 0` and `left_min[start-1] != inf`, update `result = min(result, length + left_min[start-1])`.
6. After finishing the iteration, return `result` if it is less than infinity, otherwise return `-1`.

**Why it works:** The sliding window ensures that we find all valid sub-arrays in O(n) time, leveraging the fact that all numbers are positive. The prefix `left_min` array guarantees that for any sub-array starting at index `i`, we can efficiently find the shortest non-overlapping sub-array ending before `i`. This ensures correctness while maintaining minimal sum length.

## Python Solution

```python
from typing import List

class Solution:
    def minSumOfLengths(self, arr: List[int], target: int) -> int:
        n = len(arr)
        left_min = [float('inf')] * n
        start = 0
        curr_sum = 0
        min_len = float('inf')
        result = float('inf')
        
        for end in range(n):
            curr_sum += arr[end]
            while curr_sum > target:
                curr_sum -= arr[start]
                start += 1
            if curr_sum == target:
                length = end - start + 1
                if start > 0:
                    result = min(result, length + left_min[start - 1])
                min_len = min(min_len, length)
            left_min[end] = min(left_min[end - 1] if end > 0 else float('inf'), min_len)
        
        return result if result != float('inf') else -1
```

**Explanation:**

We maintain a sliding window with `start` and `end` to find sub-arrays summing to `target`. `left_min[end]` keeps track of the shortest sub-array length ending at or before `end`. Whenever we find a valid sub-array, we check if there is a previous non-overlapping sub-array (`left_min[start-1]`) to potentially update the minimum sum of lengths.

## Go Solution

```go
func minSumOfLengths(arr []int, target int) int {
    n := len(arr)
    leftMin := make([]int, n)
    for i := range leftMin {
        leftMin[i] = 1 << 30 // infinity
    }

    start := 0
    currSum := 0
    minLen := 1 << 30
    result := 1 << 30

    for end := 0; end < n; end++ {
        currSum += arr[end]
        for currSum > target {
            currSum -= arr[start]
            start++
        }
        if currSum == target {
            length := end - start + 1
            if start > 0 && leftMin[start-1] != 1<<30 {
                if length+leftMin[start-1] < result {
                    result = length + leftMin[start-1]
                }
            }
            if length < minLen {
                minLen = length
            }
        }
        if end > 0 {
            if leftMin[end-1] < minLen {
                leftMin[end] = leftMin[end-1]
            } else {
                leftMin[end] = minLen
            }
        } else {
            leftMin[end] = minLen
        }
    }

    if result == 1<<30 {
        return -1
    }
    return result
}
```

**Explanation:**

The Go version is almost identical in logic to Python. Since Go does not have `inf` for integers, we use a large constant (`1 << 30`). Slice indexing and updates are handled carefully to avoid out-of-range errors.

## Worked Examples

**Example 1:** `arr = [3,2,2,4,3], target = 3`

| end | start | curr_sum | length | left_min | result |
| --- | --- | --- | --- | --- | --- |
| 0 | 0 | 3 | 1 | 1 | inf |
| 1 | 1 | 2 | - | 1 | inf |
| 2 | 1 | 4 | - | 1 | inf |
| 3 | 3 | 4 | - | 1 | inf |
| 4 | 4 | 3 | 1 | 1 | 2 |

Return `2`.

**Example 2:** `arr = [7,3,4,7], target = 7`

Sliding window finds `[7]` at index 0 and `[7]` at index 3, non-overlapping, length sum = 2.

**Example 3:** `arr = [4,3,2,6,2,3,4], target = 6`

Only `[6]` at index 3 is found, no second sub-array exists. Return `-1`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element is added and removed at most once from the sliding window |
| Space | O(n) | `left_min` array stores prefix minimum lengths |

The linear time complexity is achieved due to the positive nature of array elements, allowing the sliding window to always move forward.
