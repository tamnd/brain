## Solution

## Problem Understanding

The problem gives us a binary tree where every node contains a digit from `1` to `9`. We need to count how many root-to-leaf paths are _pseudo-palindromic_.

A path is pseudo-palindromic if its values can be rearranged into a palindrome. A palindrome reads the same forward and backward, such as `[1,2,1]` or `[3,4,4,3]`.

The key property of a palindrome is frequency parity:

- For an even-length palindrome, every value must appear an even number of times.
- For an odd-length palindrome, exactly one value may appear an odd number of times.

This means that for any path, we only care about whether each digit appears an odd or even number of times. The exact order of the digits does not matter.

The input is a standard binary tree:

- Each node has a value from `1` to `9`
- We consider every path from the root to a leaf
- A leaf is a node with no children

The output is the total number of root-to-leaf paths that satisfy the pseudo-palindromic condition.

The constraints are important:

- The tree can contain up to `10^5` nodes
- A naive solution that repeatedly counts frequencies for every path could become too slow
- The digit range is very small, only `1` through `9`

That small digit range is the critical observation that enables an efficient bit manipulation solution.

Several edge cases are important:

- A tree with only one node is always pseudo-palindromic because a single character is trivially a palindrome.
- Paths with multiple odd frequencies should not be counted.
- Highly unbalanced trees can create recursion depth concerns in some languages.
- Since values are restricted to `1-9`, we can safely represent parity information inside a small integer bitmask.

## Approaches

### Brute Force Approach

A straightforward solution is to perform a DFS traversal from the root to every leaf while storing the full path. Whenever we reach a leaf, we compute the frequency of each digit along that path and check whether at most one digit has an odd count.

This works because the palindrome condition depends entirely on digit frequencies. If no more than one digit appears an odd number of times, then some permutation of the path can form a palindrome.

However, this approach is inefficient because frequency counting is repeated for every leaf path. In the worst case, the tree can contain many long root-to-leaf paths, causing repeated scanning of path arrays.

Although the digit range is small here, storing full paths and repeatedly recomputing counts is still unnecessary overhead.

### Optimal Approach

The key insight is that we do not need exact frequencies. We only need to know whether each digit count is odd or even.

This naturally suggests using a bitmask.

We use an integer where:

- Bit `1` represents digit `1`
- Bit `2` represents digit `2`
- ...
- Bit `9` represents digit `9`

As we traverse the tree:

- Visiting a digit toggles its corresponding bit using XOR
- A bit value of `1` means the digit has appeared an odd number of times
- A bit value of `0` means the digit has appeared an even number of times

At a leaf node, the path is pseudo-palindromic if the bitmask contains at most one set bit.

A classic bit trick checks this efficiently:

```
mask & (mask - 1) == 0
```

This expression is true when the number has zero or one set bits.

Because each node is processed exactly once, the algorithm runs in linear time.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N × H) | O(H) | Stores full paths and recomputes frequencies at leaves |
| Optimal | O(N) | O(H) | Uses DFS with parity bitmask tracking |

Here:

- `N` is the number of nodes
- `H` is the tree height

## Algorithm Walkthrough

1. Start a depth-first traversal from the root node.
2. Maintain a bitmask representing the parity of digit frequencies along the current root-to-node path.
3. When visiting a node with value `v`, toggle the corresponding bit:

```
mask ^= (1 << v)
```

If the digit was previously even, it becomes odd. If it was odd, it becomes even.
4. Continue DFS into the left and right children while passing along the updated mask.
5. When reaching a leaf node, check whether the current mask contains at most one set bit.

The condition:

```
mask & (mask - 1) == 0
```

works because:

- `0` set bits means all counts are even
- `1` set bit means exactly one odd count
- More than one set bit means multiple odd counts, which cannot form a palindrome
6. If the condition is true, increment the answer counter.
7. Continue traversal until all root-to-leaf paths have been processed.

### Why it works

The algorithm maintains the invariant that the bitmask always represents the parity state of digit frequencies along the current path.

Each bit indicates whether a digit count is odd or even. A path can form a palindrome if and only if at most one digit has an odd frequency. Therefore, checking whether the mask contains at most one set bit correctly determines whether the path is pseudo-palindromic.

Since every node is visited once and the mask is updated in constant time, the algorithm is both correct and efficient.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def pseudoPalindromicPaths(self, root: Optional[TreeNode]) -> int:
        
        def dfs(node: Optional[TreeNode], mask: int) -> int:
            if not node:
                return 0
            
            # Toggle the bit corresponding to node.val
            mask ^= (1 << node.val)
            
            # Leaf node
            if not node.left and not node.right:
                # Check if at most one bit is set
                if mask & (mask - 1) == 0:
                    return 1
                return 0
            
            return dfs(node.left, mask) + dfs(node.right, mask)
        
        return dfs(root, 0)
```

The implementation closely follows the algorithm described earlier.

The `dfs` function receives two parameters:

- The current node
- The current parity bitmask

At each node, the corresponding bit is toggled using XOR. This efficiently updates whether that digit has appeared an odd or even number of times.

The leaf node condition is important because only root-to-leaf paths count. When a leaf is reached, the bitmask is checked using the standard one-bit test:

```python
mask & (mask - 1) == 0
```

If true, the path is pseudo-palindromic and contributes `1` to the answer.

The recursive calls process both subtrees independently while carrying the updated parity state forward.

Because integers are immutable in Python, there is no need to manually backtrack the mask after recursion. Each recursive call receives its own copy of the integer value.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func pseudoPalindromicPaths(root *TreeNode) int {

    var dfs func(node *TreeNode, mask int) int

    dfs = func(node *TreeNode, mask int) int {
        if node == nil {
            return 0
        }

        // Toggle the bit corresponding to node.Val
        mask ^= (1 << node.Val)

        // Leaf node
        if node.Left == nil && node.Right == nil {
            if mask&(mask-1) == 0 {
                return 1
            }
            return 0
        }

        return dfs(node.Left, mask) + dfs(node.Right, mask)
    }

    return dfs(root, 0)
}
```

The Go implementation is nearly identical to the Python version in terms of logic.

Several Go-specific details are worth noting:

- Go uses `nil` instead of `None`
- Recursive helper functions are commonly declared using function variables
- Integer bit operations work naturally with Go's built-in integer types
- Like Python, the integer `mask` is passed by value, so recursive calls do not interfere with each other

Since the mask only needs bits `1-9`, integer overflow is never a concern.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(N) | Each node is visited exactly once |
| Space | O(H) | Recursive call stack stores at most tree height frames |

The DFS traversal processes every node one time, and each operation inside the traversal is constant time.

The bitmask update and palindrome check are both `O(1)` operations.

The space complexity comes from recursion depth. In the worst case of a completely skewed tree, the height can become `N`. In a balanced tree, the height is approximately `log N`.

## Edge Cases

### Single Node Tree

A tree containing only one node is an important boundary case. A single value always forms a palindrome because there is only one character. Some implementations accidentally reject this case if they incorrectly require an even number of nodes or multiple matching digits.

The bitmask solution handles this naturally because a single digit produces exactly one set bit, which satisfies the condition:

```
mask & (mask - 1) == 0
```

### Completely Skewed Tree

A highly unbalanced tree behaves like a linked list and can create recursion depth concerns. Algorithms that assume balanced trees may underestimate stack usage.

This implementation still works correctly because DFS naturally processes the single path. The space complexity becomes `O(N)` in the worst case due to recursion depth.

### Paths With Multiple Odd Frequencies

A common mistake is incorrectly allowing paths with two or more odd counts. For example, the path `[1,2,3]` cannot form a palindrome because all three digits appear an odd number of times.

The implementation avoids this bug by using the bit trick that strictly enforces at most one set bit in the parity mask. Any mask containing multiple set bits is rejected immediately.
