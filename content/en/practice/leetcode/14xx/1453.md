## Solution

## Problem Understanding

The problem is asking us to find the maximum number of darts that can lie within a single circular dartboard of a given radius `r` on a 2D plane. The input is an array of dart coordinates `darts`, where each entry `[xi, yi]` represents the location of a dart, and an integer `r` representing the radius of the dartboard. The expected output is an integer representing the maximum number of darts that can fit inside any circle of radius `r`.

The constraints indicate that there can be up to 100 darts, with coordinates ranging from `-10^4` to `10^4`, and the radius of the dartboard ranges from 1 to 5000. All darts are unique, so we do not need to handle duplicate points. Edge cases to be aware of include all darts being at the same point, the radius being extremely small, or the darts forming a sparse configuration where no circle can capture more than one or two darts.

This problem requires geometric reasoning because the circle can be placed anywhere, not necessarily at the location of a dart.

## Approaches

The brute-force approach would involve generating a candidate circle for every possible combination of two darts. For each pair, we calculate the potential circle(s) with radius `r` that pass through both points, then count how many darts lie within that circle. This works because any circle containing at least two darts can be uniquely determined using those two points as boundary points, except when all darts lie on a single point. However, checking all pairs scales as O(n²), and counting darts inside each candidate circle adds another factor of n, leading to O(n³) time complexity. While feasible for n ≤ 100, it is not efficient and can be optimized.

The key insight is that the maximum number of darts on a circle of radius `r` is determined by checking circles defined by each pair of points as potential boundary points. Using basic geometry, for two points within 2*r distance, there are at most two circle centers that can enclose both points. For each center, we simply count how many darts lie within radius `r`. This reduces the problem to iterating through pairs and checking distances, avoiding unnecessary guesses.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Check all circles passing through every pair of points and count darts inside |
| Optimal | O(n³) | O(1) | Use geometry to find circle centers for each dart pair within 2*r distance, then count darts inside |

## Algorithm Walkthrough

1. Iterate through each dart as the first point.
2. For every other dart, compute the distance between the two darts. If it exceeds 2*r, skip because a circle of radius r cannot contain both.
3. For a pair of darts within 2*r distance, calculate the midpoint and the vector perpendicular to the line segment connecting the two points.
4. Use the distance from midpoint to the circle center formula `h = sqrt(r² - (d/2)²)` to find the two potential centers.
5. For each candidate center, iterate through all darts and count how many lie within radius `r` using the Euclidean distance squared to avoid floating-point errors.
6. Track the maximum count encountered and return it.

Why it works: Any circle of radius `r` that contains at least two points must have its center lying on one of the two perpendicular bisectors at a distance `h` from the midpoint. By checking all pairs and candidate centers, we ensure that all configurations are considered, guaranteeing the maximum is found.

## Python Solution

```python
from typing import List
from math import sqrt

class Solution:
    def numPoints(self, darts: List[List[int]], r: int) -> int:
        def count_in_circle(cx: float, cy: float) -> int:
            count = 0
            r2 = r * r
            for x, y in darts:
                if (x - cx) ** 2 + (y - cy) ** 2 <= r2 + 1e-6:
                    count += 1
            return count

        n = len(darts)
        if n == 1:
            return 1
        
        max_count = 1
        for i in range(n):
            x1, y1 = darts[i]
            for j in range(i + 1, n):
                x2, y2 = darts[j]
                dx, dy = x2 - x1, y2 - y1
                d2 = dx * dx + dy * dy
                if d2 > (2 * r) ** 2:
                    continue
                mx, my = (x1 + x2) / 2, (y1 + y2) / 2
                q = sqrt(r * r - (d2 / 4))
                len_d = sqrt(d2)
                if len_d == 0:
                    continue
                offset_x = -dy / len_d * q
                offset_y = dx / len_d * q
                for cx, cy in [(mx + offset_x, my + offset_y), (mx - offset_x, my - offset_y)]:
                    max_count = max(max_count, count_in_circle(cx, cy))
        return max_count
```

The Python solution calculates candidate centers using vector math, iterates through all pairs, and counts darts inside. The `count_in_circle` helper ensures we account for floating-point precision with a small epsilon.

## Go Solution

```go
import (
    "math"
)

func numPoints(darts [][]int, r int) int {
    countInCircle := func(cx, cy float64) int {
        count := 0
        r2 := float64(r * r)
        for _, dart := range darts {
            dx := float64(dart[0]) - cx
            dy := float64(dart[1]) - cy
            if dx*dx+dy*dy <= r2+1e-6 {
                count++
            }
        }
        return count
    }

    n := len(darts)
    if n == 1 {
        return 1
    }

    maxCount := 1
    for i := 0; i < n; i++ {
        x1, y1 := darts[i][0], darts[i][1]
        for j := i + 1; j < n; j++ {
            x2, y2 := darts[j][0], darts[j][1]
            dx, dy := float64(x2-x1), float64(y2-y1)
            d2 := dx*dx + dy*dy
            if d2 > float64(4*r*r) {
                continue
            }
            mx, my := float64(x1+x2)/2, float64(y1+y2)/2
            q := math.Sqrt(float64(r*r) - d2/4)
            lenD := math.Sqrt(d2)
            if lenD == 0 {
                continue
            }
            offsetX := -dy / lenD * q
            offsetY := dx / lenD * q
            centers := [][2]float64{{mx + offsetX, my + offsetY}, {mx - offsetX, my - offsetY}}
            for _, c := range centers {
                maxCount = int(math.Max(float64(maxCount), float64(countInCircle(c[0], c[1]))))
            }
        }
    }
    return maxCount
}
```

The Go version mirrors the Python logic. Go requires explicit type conversions between integers and floats. Floating-point precision is handled similarly with `1e-6`.

## Worked Examples

Example 1: `darts = [[-2,0],[2,0],[0,2],[0,-2]], r = 2`

Iterate through pairs. The pair `[-2,0]` and `[2,0]` gives midpoint `(0,0)` and `q=0`. All points are within distance 2 of `(0,0)`. Max count = 4.

Example 2: `darts = [[-3,0],[3,0],[2,6],[5,4],[0,9],[7,8]], r = 5`

Consider pair `[-3,0]` and `[5,4]`. Candidate center at `(0,4)` covers `[-3,0],[3,0],[2,6],[5,4],[0,9]` => 5 darts.
