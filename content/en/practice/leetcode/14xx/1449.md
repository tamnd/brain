**Topics:** Array, Dynamic Programming

## Solution

## Problem Understanding

The problem asks us to construct the **largest possible integer** using digits 1 through 9 such that the **sum of their painting costs equals a given target**. Each digit has a specific painting cost given in a 0-indexed array `cost`, where `cost[i]` is the cost to paint digit `(i + 1)`. We must choose digits such that their total cost is exactly `target`. If no combination of digits satisfies this, we return `"0"`.

Inputs are:

- `cost`: an array of 9 integers representing the cost to paint digits 1 to 9.
- `target`: a positive integer representing the total allowable cost.

The output is a string representing the **largest integer** that can be formed with digits satisfying the total cost constraint.

Constraints tell us:

- `cost.length == 9`, so we have exactly 9 possible digits.
- `1 <= cost[i], target <= 5000`, which allows the target to be quite large. This means a brute-force solution that tries all combinations would be too slow.

Important edge cases include:

1. `target` smaller than any single digit cost, making it impossible to form any integer.
2. Multiple digits having the same cost but different values, where choosing the larger digit will maximize the integer.
3. `target` exactly equal to one digit's cost, which should return that digit.

## Approaches

**Brute Force Approach:**

A brute-force solution would try **every combination of digits** to see which total cost equals the target, then select the combination that forms the largest integer. We could attempt this with recursive backtracking by subtracting the cost of a chosen digit from the remaining target at each step. While this approach guarantees correctness, it has **exponential time complexity**, as there are 9 choices at each step, making it infeasible for large `target` values up to 5000.

**Optimal Approach:**

The problem can be solved using **dynamic programming**, similar to the unbounded knapsack problem. The key insight is to track, for each possible total cost from `0` to `target`, the **largest number (as a string)** we can form with that cost.

- Define `dp[t]` as the largest number (string) with total cost `t`.
- For each cost `c[i]` and corresponding digit `i + 1`, if `t - c[i] >= 0` and `dp[t - c[i]]` is not empty, then `dp[t]` can be updated as the **larger string** between `dp[t]` and `digit + dp[t - c[i]]`.
- We iterate over digits in **descending order** to ensure the largest number is formed.

This guarantees the result is optimal in terms of the largest integer.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(9^target) | O(target) | Recursive backtracking checking all combinations, too slow |
| Dynamic Programming (Optimal) | O(9 * target) | O(target) | Uses a 1D DP array, iterating over costs and building largest number |

## Algorithm Walkthrough

1. Initialize a DP array `dp` of size `target + 1`. Set all entries to `None` except `dp[0]` which is an empty string, representing zero cost.
2. Loop `t` from `1` to `target`. For each `t`, try adding each digit `d` from `9` down to `1`.
3. For each digit `d`, calculate the remaining cost `prev = t - cost[d-1]`. If `prev >= 0` and `dp[prev]` is not `None`, then a number can be formed.
4. Update `dp[t]` by comparing the current `dp[t]` and the string `str(d) + dp[prev]`. Keep the lexicographically larger string.
5. After filling the DP array, return `dp[target]` if it exists, otherwise return `"0"`.

**Why it works:**

The DP invariant is that `dp[t]` always stores the **largest integer possible** with cost `t`. By iterating digits in descending order, we prioritize larger digits for the highest places. This guarantees that we maximize the integer value while exactly matching the target cost.

## Python Solution

```python
from typing import List

class Solution:
    def largestNumber(self, cost: List[int], target: int) -> str:
        dp = [None] * (target + 1)
        dp[0] = ""  # Base case: cost 0 gives empty number
        
        for t in range(1, target + 1):
            for d in range(9, 0, -1):
                prev = t - cost[d - 1]
                if prev >= 0 and dp[prev] is not None:
                    candidate = str(d) + dp[prev]
                    if dp[t] is None or int(candidate) > int(dp[t]):
                        dp[t] = candidate
        
        return dp[target] if dp[target] is not None else "0"
```

**Explanation:**

- `dp[0]` is initialized as empty because zero cost corresponds to no digits.
- For each total cost `t`, we check if adding digit `d` leads to a valid number (`prev >= 0` and `dp[prev]` exists).
- `str(d) + dp[prev]` ensures the digit is added to the **most significant position** first.
- We keep the **largest integer** by comparing current and candidate numbers.

## Go Solution

```go
import "strconv"

func largestNumber(cost []int, target int) string {
    dp := make([]string, target+1)
    dp[0] = ""
    
    for t := 1; t <= target; t++ {
        for d := 9; d >= 1; d-- {
            prev := t - cost[d-1]
            if prev >= 0 && dp[prev] != "" {
                candidate := strconv.Itoa(d) + dp[prev]
                if dp[t] == "" || candidate > dp[t] {
                    dp[t] = candidate
                }
            }
        }
    }
    
    if dp[target] == "" {
        return "0"
    }
    return dp[target]
}
```

**Go-specific Notes:**

- `dp` is a slice of strings.
- `strconv.Itoa(d)` converts an integer to a string for concatenation.
- Go treats an empty string `""` as `None` in Python, simplifying initialization.

- No combination of digits sums to 5. Return `"0"`.

## Complexity Analysis

| Measure | Complexity | Explanation |

| Time | O(9 * target) | For each total cost from 1 to target, we iterate over 9 digits |
| Space | O(target) | DP array of size target + 1 stores strings |

The time complexity is acceptable because `target <= 5000`, resulting in roughly 45,000 operations in the worst case.
