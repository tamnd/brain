## Solution

## Problem Understanding

The problem is asking us to identify "active users" from a database of accounts and login records. An active user is defined as someone who has logged in for five or more consecutive days.

We are given two tables: `Accounts` and `Logins`. The `Accounts` table provides each user’s `id` and `name`, which serves as the unique identifier and the display name. The `Logins` table records each login event, with a possible duplication of multiple logins on the same day or repeated entries for the same user. The goal is to return the `id` and `name` of users who meet the consecutive login criterion, sorted by `id`.

The key points to note are that duplicate login dates for the same user should not count as separate days, only unique dates matter. Users may have gaps in their login history, so we must detect sequences of consecutive dates. The tables are relational, and the problem essentially tests the ability to work with SQL-style data processing using Python or Go.

Important edge cases include users with less than five unique login days, users with exactly five consecutive days, users with multiple sequences of consecutive logins, and users who have duplicate login records for a single day.

## Approaches

The brute-force approach is to first fetch all login dates for each user, sort them, and then iterate over the sorted list to detect consecutive sequences. For each user, we would check every subsequence of dates to see if there are five or more consecutive days. This works but is inefficient if the number of users and login records is large because it requires multiple iterations and sorting for each user.

The optimal approach leverages the concept of "gaps and islands" from SQL and array manipulation. We can remove duplicates, convert login dates to ordinal integers, and then group dates into sequences where consecutive days form an "island." By computing the length of each island, we can directly identify users who have at least five consecutive days. This approach avoids repeatedly iterating through individual days and takes advantage of vectorized or set-based operations, making it much more efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(U * D log D) | O(D) | For each user, sort D login dates and check for consecutive sequences |
| Optimal | O(U * D) | O(D) | Remove duplicates, map dates to integers, compute groups of consecutive days, and filter by length |

## Algorithm Walkthrough

1. Remove duplicates from the `Logins` table to ensure each login date per user is counted only once. This simplifies sequence detection.
2. Convert the `login_date` into an ordinal or integer representation so that consecutive dates can be compared mathematically.
3. For each user, sort their login dates in ascending order.
4. Calculate the difference between the date and its index position. Consecutive dates will have the same difference value, forming a "group" or "island."
5. Group login dates by the computed difference. Each group represents a sequence of consecutive login days.
6. Measure the size of each group. If a group has length greater than or equal to five, mark the user as active.
7. Join the resulting active user IDs with the `Accounts` table to fetch their names.
8. Return the final list of active users, ordered by `id`.

Why it works: By mapping dates to integers and comparing with the index, the difference remains constant for consecutive sequences. This invariant ensures that each group of equal differences corresponds to a valid consecutive sequence. Filtering by length guarantees that only sequences of five or more days are considered.

## Python Solution

```python
from typing import List
import pandas as pd

class Solution:
    def activeUsers(self, accounts: pd.DataFrame, logins: pd.DataFrame) -> pd.DataFrame:
        # Remove duplicates
        logins_unique = logins.drop_duplicates()
        
        # Convert login_date to datetime
        logins_unique['login_date'] = pd.to_datetime(logins_unique['login_date'])
        
        # Sort by user and date
        logins_sorted = logins_unique.sort_values(by=['id', 'login_date'])
        
        # Assign a sequence id to detect consecutive days
        logins_sorted['seq'] = (logins_sorted['login_date'] - pd.to_timedelta(logins_sorted.groupby('id').cumcount(), unit='d'))
        
        # Count consecutive sequences
        consecutive_counts = logins_sorted.groupby(['id', 'seq']).size().reset_index(name='count')
        
        # Filter users with at least 5 consecutive days
        active_user_ids = consecutive_counts[consecutive_counts['count'] >= 5]['id'].unique()
        
        # Join with Accounts to get names
        active_users = accounts[accounts['id'].isin(active_user_ids)].sort_values(by='id')
        
        return active_users
```

The Python implementation first removes duplicates, converts dates to a proper format, and sorts them. Using the cumulative count of rows per user, we create a sequence identifier for consecutive dates. Then we group by user and sequence to count the length of consecutive sequences, filter for sequences of at least five days, and finally join with the accounts table to get the names.

## Go Solution

```go
package main

import (
    "time"
    "sort"
)

type Account struct {
    ID   int
    Name string
}

type Login struct {
    ID        int
    LoginDate time.Time
}

func ActiveUsers(accounts []Account, logins []Login) []Account {
    // Map to store unique login dates per user
    userDates := make(map[int]map[string]struct{})
    for _, login := range logins {
        if _, ok := userDates[login.ID]; !ok {
            userDates[login.ID] = make(map[string]struct{})
        }
        dateStr := login.LoginDate.Format("2006-01-02")
        userDates[login.ID][dateStr] = struct{}{}
    }

    activeIDs := make(map[int]struct{})
    for userID, datesMap := range userDates {
        // Convert map to slice and sort
        var dates []time.Time
        for d := range datesMap {
            t, _ := time.Parse("2006-01-02", d)
            dates = append(dates, t)
        }
        sort.Slice(dates, func(i, j int) bool { return dates[i].Before(dates[j]) })

        // Check for consecutive days
        consec := 1
        for i := 1; i < len(dates); i++ {
            if dates[i].Sub(dates[i-1]) == 24*time.Hour {
                consec++
                if consec >= 5 {
                    activeIDs[userID] = struct{}{}
                    break
                }
            } else {
                consec = 1
            }
        }
    }

    // Collect active accounts
    var result []Account
    for _, acc := range accounts {
        if _, ok := activeIDs[acc.ID]; ok {
            result = append(result, acc)
        }
    }

    // Sort by ID
    sort.Slice(result, func(i, j int) bool { return result[i].ID < result[j].ID })
    return result
}
```

In Go, we use maps to remove duplicates and track user logins. Dates are converted and sorted, and we iterate to detect consecutive sequences. Active users are collected in a map and finally merged with the accounts slice, sorted by ID.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(U * D) | D is the number of login records per user, U is number of users; removing duplicates, sorting dates, and iterating once per user |
| Space | O(D) | Storing unique login dates per user, temporary maps for sequences |

The optimal algorithm scales linearly with the number of login records after deduplication. Sorting per user is bounded by the number of login days for that user.
