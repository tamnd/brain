## Solution

## Problem Understanding

The problem gives us an array of unique lowercase strings called `words`. Our task is to find every string that appears as a substring inside another string from the same array.

A substring is a contiguous sequence of characters inside another string. For example, `"as"` is a substring of `"mass"` because the characters `"a"` and `"s"` appear next to each other inside `"mass"`.

The input is an array of strings:

```
words = ["mass","as","hero","superhero"]
```

The expected output is a list containing all words that appear inside at least one other word in the array:

```
["as","hero"]
```

The order of the result does not matter.

The constraints are small:

- At most 100 words
- Each word has length at most 30
- All words are unique

These constraints are important because they tell us that even a straightforward comparison approach is acceptable. The maximum amount of work is relatively small. In the worst case, we compare every pair of strings, and each comparison only involves short strings.

Several edge cases are worth identifying immediately.

A word may be a substring of multiple other words. We should only include it once in the answer.

Some words may not appear inside any other word. Those should not be included.

A word should not be compared against itself. Since every string is trivially a substring of itself, failing to skip self-comparison would incorrectly include every word.

The problem guarantees that all strings are unique, which simplifies the logic because we do not need to worry about duplicate entries in the input.

## Approaches

### Brute Force Approach

The most direct solution is to compare every word against every other word.

For each word `words[i]`, we check all other words `words[j]`. If `words[i]` appears inside `words[j]`, then `words[i]` belongs in the answer.

In Python, substring checking is very convenient because we can use:

```
if small_word in large_word:
```

This brute-force strategy is correct because it explicitly examines every possible pair of words. If a substring relationship exists anywhere in the array, the algorithm will discover it.

The downside is that we repeatedly compare many pairs of strings. In general problems with very large inputs, this could become expensive. However, because the constraints here are tiny, this approach is actually perfectly acceptable.

### Key Insight for the Better Solution

The important observation is that we only care whether a word appears inside another word, not where it appears or how many times it appears.

Because the input size is small, the optimal practical solution is still based on pairwise comparison, but we can make it cleaner and slightly more efficient by stopping early once a match is found.

For each word:

- Compare it against every other word
- As soon as it is found as a substring, add it to the result
- Stop checking additional words for that candidate

This avoids unnecessary work after a successful match.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n² × m) | O(1) excluding output | Compare every pair of words fully |
| Optimal | O(n² × m) | O(1) excluding output | Early termination after first successful match |

Here:

- `n` is the number of words
- `m` is the average substring search cost

Since each word length is at most 30, the practical runtime is very small.

## Algorithm Walkthrough

1. Create an empty result list.

This list will store all words that are substrings of another word.
2. Iterate through each word in the array.

Treat the current word as the candidate substring we are trying to validate.
3. For the current word, compare it against every other word.

We need to determine whether the current word appears anywhere inside another string in the array.
4. Skip comparisons where both indices are the same.

A string is always a substring of itself, but the problem specifically asks for substrings of another word.
5. Use substring checking.

Check whether:

```
current_word in other_word
```

If this condition is true, then the current word satisfies the requirement.
6. Add the word to the result list immediately.

Once we know the word is a substring of another word, additional checks are unnecessary.
7. Break out of the inner loop.

This early exit prevents redundant comparisons.
8. Continue until all words have been processed.
9. Return the result list.

### Why it works

The algorithm checks every possible pair of distinct words. Therefore, if a word is a substring of any other word, the algorithm is guaranteed to discover it during one of these comparisons.

By skipping self-comparisons, we avoid false positives. By breaking early after finding a match, we maintain correctness while reducing unnecessary work.

## Python Solution

```python
from typing import List

class Solution:
    def stringMatching(self, words: List[str]) -> List[str]:
        result = []

        for i in range(len(words)):
            current_word = words[i]

            for j in range(len(words)):
                if i == j:
                    continue

                if current_word in words[j]:
                    result.append(current_word)
                    break

        return result
```

The implementation closely follows the algorithm described earlier.

The `result` list stores all valid substring words.

The outer loop selects each word as the candidate substring. The inner loop compares it against every other word in the array.

The condition:

```python
if i == j:
    continue
```

prevents comparing a word with itself.

The key operation is:

```python
if current_word in words[j]:
```

Python's `in` operator performs substring matching directly and efficiently.

Once a match is found, the word is appended to the result list and the inner loop exits immediately using `break`. This ensures that each matching word appears only once in the output.

Finally, the completed result list is returned.

## Go Solution

```go
func stringMatching(words []string) []string {
    result := []string{}

    for i := 0; i < len(words); i++ {
        currentWord := words[i]

        for j := 0; j < len(words); j++ {
            if i == j {
                continue
            }

            if contains(words[j], currentWord) {
                result = append(result, currentWord)
                break
            }
        }
    }

    return result
}

func contains(text string, pattern string) bool {
    return len(pattern) <= len(text) &&
        (text == pattern ||
            len(pattern) == 0 ||
            indexOf(text, pattern) != -1)
}

func indexOf(text string, pattern string) int {
    n := len(text)
    m := len(pattern)

    for i := 0; i <= n-m; i++ {
        match := true

        for j := 0; j < m; j++ {
            if text[i+j] != pattern[j] {
                match = false
                break
            }
        }

        if match {
            return i
        }
    }

    return -1
}
```

In Go, there is no built-in substring operator like Python's `in`. We therefore implement substring checking manually using a helper function.

The `indexOf` function performs a straightforward substring search by checking every possible starting position in the larger string.

The rest of the logic mirrors the Python solution closely:

- Outer loop selects the candidate substring
- Inner loop checks against all other words
- Self-comparisons are skipped
- Early termination occurs after the first successful match

Go slices are dynamically sized, so appending results works naturally using `append`.

Integer overflow is not a concern because the constraints are extremely small.

## Edge Cases

### Single Word Input

If the array contains only one word, there is no other string available for comparison. A careless implementation might accidentally compare the word with itself and incorrectly include it in the result.

The implementation avoids this issue using:

```
if i == j:
    continue
```

As a result, single-element inputs correctly return an empty list.

### A Word Appearing in Multiple Larger Words

A word may be a substring of several different strings. For example:

```
["a", "ab", "abc"]
```

The word `"a"` matches both `"ab"` and `"abc"`.

A buggy implementation might append `"a"` multiple times. Our implementation prevents duplicates by immediately breaking out of the inner loop after the first successful match.

### Substrings at Different Positions

A substring can appear at the beginning, middle, or end of another word. Some incorrect implementations accidentally only check prefixes or suffixes.

Examples include:

```
"test" in "testing"     # beginning
"sti" in "testing"      # middle
"ing" in "testing"      # end
```

Using the language's proper substring search operation guarantees that all positions are handled correctly.
