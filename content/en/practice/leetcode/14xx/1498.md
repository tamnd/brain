## Solution

## Problem Understanding

This problem asks us to count how many non-empty subsequences of the array satisfy a specific condition.

A subsequence is formed by choosing any subset of elements while preserving their relative order. Unlike subarrays, subsequences do not need to be contiguous. For example, from `[3,5,6,7]`, valid subsequences include `[3,6]`, `[5,7]`, and `[3,5,7]`.

For every chosen subsequence, we look at its minimum element and maximum element. The subsequence is considered valid if:

$$\text{minimum} + \text{maximum} \leq \text{target}$$

We must count all valid non-empty subsequences.

The answer can become extremely large because an array of length `n` can have up to:

$$2^n - 1$$

non-empty subsequences. Since `n` can be as large as `10^5`, the result must be returned modulo:

$$10^9 + 7$$

The constraints immediately rule out brute force enumeration. Even generating all subsequences would take exponential time, which is impossible for `10^5` elements.

Several important observations come from the constraints:

- The array length is very large, so the solution must be close to `O(n log n)` or better.
- Since only the minimum and maximum values matter, sorting is likely useful.
- Repeated values are allowed, so we must count subsequences carefully.
- Every non-empty subsequence must be considered separately, even if two subsequences contain the same values at different indices.

Important edge cases include:

- Arrays where no subsequence is valid.
- Arrays where every subsequence is valid.
- Arrays with duplicate values.
- Arrays of length `1`.
- Situations where only single-element subsequences satisfy the condition.
- Large inputs where modular arithmetic is required to avoid overflow.

## Approaches

### Brute Force Approach

The most direct solution is to generate every possible non-empty subsequence.

For each subsequence:

1. Find its minimum value.
2. Find its maximum value.
3. Check whether their sum is less than or equal to `target`.
4. Count it if valid.

This approach is correct because it explicitly checks every possible subsequence.

However, an array of length `n` has:

$$2^n - 1$$

non-empty subsequences.

For `n = 10^5`, this is astronomically large. Even for `n = 30`, brute force becomes impractical.

Therefore, we need a fundamentally different strategy.

### Key Insight Behind the Optimal Solution

The critical observation is that the condition depends only on the minimum and maximum values of the subsequence.

If the array is sorted:

```
nums[0] <= nums[1] <= ... <= nums[n-1]
```

then for a fixed left index `i`, representing the minimum element, we can determine the largest right index `j` such that:

$$nums[i] + nums[j] \leq target$$

Now consider all subsequences that:

- Must include `nums[i]`
- Only use elements between `i` and `j`

Since the array is sorted:

- `nums[i]` is guaranteed to be the minimum
- Any chosen element from `i` to `j` will not exceed `nums[j]`

Every subset of the elements between `i+1` and `j` can either be included or excluded independently.

If there are:

$$j - i$$

elements after `i`, then the number of valid subsequences is:

$$2^{(j-i)}$$

This transforms the problem from exponential enumeration into efficient counting.

To efficiently find valid `j` values, we use the two-pointer technique.

## Algorithm Walkthrough

### Optimal Algorithm

1. Sort the array in non-decreasing order.

Sorting allows us to reason about minimum and maximum values efficiently. Once sorted, if we choose `nums[left]` as the minimum, then any element to its right is greater than or equal to it.
2. Precompute powers of two modulo `10^9 + 7`.

We frequently need values like:

$$2^k$$

where `k` represents the number of optional elements between two pointers.

Precomputing these values avoids repeated exponentiation.
3. Initialize two pointers.

- `left = 0`
- `right = n - 1`

The `left` pointer represents the minimum value of a subsequence.

The `right` pointer represents the largest possible maximum value that still satisfies the condition.
4. While `left <= right`, check whether:

$$nums[left] + nums[right] \leq target$$
5. If the condition is valid:

Then every subsequence that:

- includes `nums[left]`
- chooses any subset of elements from `left+1` to `right`

is also valid.

The number of such subsequences is:

$$2^{(right-left)}$$

Add this value to the answer.

Then increment `left` because all subsequences using this minimum value have been counted.
6. Otherwise, if the sum exceeds the target:

The current maximum is too large.

Decrement `right` to try a smaller maximum value.
7. Continue until the pointers cross.
8. Return the final answer modulo `10^9 + 7`.

### Why It Works

The algorithm works because sorting creates a monotonic structure.

For a fixed minimum element `nums[left]`, if:

$$nums[left] + nums[right] \leq target$$

then every element between `left` and `right` is also compatible as a maximum because the array is sorted.

Thus every subset formed from the middle elements remains valid.

The two-pointer approach guarantees that every valid subsequence is counted exactly once, grouped by its minimum element.

## Python Solution

```python
from typing import List

class Solution:
    def numSubseq(self, nums: List[int], target: int) -> int:
        MOD = 10**9 + 7
        
        nums.sort()
        n = len(nums)
        
        power_of_two = [1] * n
        for i in range(1, n):
            power_of_two[i] = (power_of_two[i - 1] * 2) % MOD
        
        left = 0
        right = n - 1
        answer = 0
        
        while left <= right:
            if nums[left] + nums[right] <= target:
                answer += power_of_two[right - left]
                answer %= MOD
                left += 1
            else:
                right -= 1
        
        return answer
```

The implementation begins by sorting the array so that minimum and maximum values can be managed using pointers.

Next, the code precomputes powers of two modulo `10^9 + 7`. The value at index `k` represents:

$$2^k$$

which corresponds to the number of ways to include or exclude `k` optional elements.

The two-pointer section forms the core of the algorithm.

If `nums[left] + nums[right]` satisfies the target condition, then all subsequences formed by fixing `nums[left]` as the minimum and choosing any subset of the remaining elements up to `right` are valid.

The count of these subsequences is:

$$2^{(right-left)}$$

After counting them, the algorithm increments `left` because all subsequences with that minimum element have already been considered.

If the sum exceeds the target, then the maximum value is too large, so the algorithm decreases `right`.

The modulo operation is applied after each addition to prevent integer overflow and satisfy the problem requirements.

## Go Solution

```go
package main

import (
	"sort"
)

func numSubseq(nums []int, target int) int {
	const MOD int = 1e9 + 7

	sort.Ints(nums)

	n := len(nums)

	powerOfTwo := make([]int, n)
	powerOfTwo[0] = 1

	for i := 1; i < n; i++ {
		powerOfTwo[i] = (powerOfTwo[i-1] * 2) % MOD
	}

	left := 0
	right := n - 1
	answer := 0

	for left <= right {
		if nums[left]+nums[right] <= target {
			answer = (answer + powerOfTwo[right-left]) % MOD
			left++
		} else {
			right--
		}
	}

	return answer
}
```

The Go implementation follows the same logic as the Python version.

A few Go-specific details are worth noting:

- `sort.Ints(nums)` is used for sorting.
- A slice is used for storing powers of two.
- Modulo arithmetic is applied during accumulation to avoid overflow.
- Go does not have Python's arbitrary precision integers, so keeping values reduced modulo `10^9 + 7` is especially important.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n log n) | Sorting dominates the runtime |
| Space | O(n) | Extra space for precomputed powers |

The sorting step requires `O(n log n)` time.

The two-pointer traversal processes each element at most once from either side, so it runs in `O(n)` time.

Precomputing powers of two also takes `O(n)` time.

Therefore, the total complexity is dominated by sorting.

The extra space comes from storing the powers of two array.

## Edge Cases

### Single Element Arrays

An array containing only one element is easy to mishandle because the minimum and maximum are the same element.

For example:

```
nums = [5]
target = 10
```

The subsequence `[5]` is valid because:

```
5 + 5 <= 10
```

The implementation correctly handles this because the two pointers initially point to the same index, and the algorithm allows `left <= right`.

### No Valid Subsequences

Consider:

```
nums = [10,11,12]
target = 5
```

Even the smallest possible subsequence fails because:

```
10 + 10 > 5
```

The algorithm repeatedly decreases `right` until the pointers cross, never adding anything to the answer.

This correctly returns `0`.

### All Subsequences Valid

Consider:

```
nums = [1,1,1,1]
target = 2
```

Every subsequence satisfies the condition because:

```
1 + 1 <= 2
```

The algorithm efficiently counts all:

$$2^4 - 1 = 15$$

non-empty subsequences without explicitly generating them.

### Large Answers Requiring Modulo Arithmetic

For large arrays, the number of valid subsequences grows exponentially.

Without modulo arithmetic, integer overflow would occur in many programming languages.

The implementation applies modulo operations during every update step, ensuring correctness and preventing overflow issues.
