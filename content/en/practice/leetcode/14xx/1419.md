## Solution

## Problem Understanding

The problem asks us to determine the minimum number of frogs needed to produce a sequence of overlapping "croak" sounds. Each frog produces the sound `"croak"` in order: `c -> r -> o -> a -> k`. Multiple frogs can croak simultaneously, so the letters of different `"croak"` sequences can interleave. The input is a string `croakOfFrogs` consisting only of the characters `'c'`, `'r'`, `'o'`, `'a'`, `'k'`. The goal is to return the **minimum number of frogs that can produce this exact sequence**, or `-1` if the sequence is invalid.

The constraints tell us the input length can be up to 100,000 characters, so a solution must be linear or near-linear time. The input guarantees only valid croak letters, but it does not guarantee valid sequences of "croak". Important edge cases include inputs where letters are out of order, multiple frogs croak simultaneously, or the sequence ends mid-croak.

## Approaches

A brute-force approach might attempt to simulate each frog individually, tracking which frog is at which stage of the croak sequence. This would involve maintaining multiple sequences, updating them for each character, and possibly trying all combinations. While correct, it is too slow for large inputs because we could have thousands of active frogs, leading to O(n²) or worse behavior.

The key insight for an optimal solution is to **count the number of frogs at each stage** instead of simulating them individually. We can maintain a count for each letter `'c'`, `'r'`, `'o'`, `'a'`, `'k'` representing the number of frogs currently at that stage. For each character, we increment the count for that stage and decrement the previous stage. The number of frogs currently croaking is the sum of frogs in all stages except `'k'`, and the maximum number of concurrent croaking frogs gives the answer. If at any point a stage appears before its previous stage has enough frogs, the sequence is invalid.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Track each frog individually with separate states |
| Optimal | O(n) | O(1) | Count frogs at each stage and track maximum concurrent croaks |

## Algorithm Walkthrough

1. Initialize a counter for each stage `'c'`, `'r'`, `'o'`, `'a'`, `'k'`. Set `max_frogs = 0`.
2. Iterate through each character in `croakOfFrogs`.
3. For each character, check that there is at least one frog in the previous stage if it is not `'c'`. If not, return `-1`.
4. Increment the counter for the current stage.
5. If the character is not `'c'`, decrement the counter for the previous stage.
6. Calculate the total number of frogs currently croaking (sum of counts for `'c'` through `'a'`) and update `max_frogs` if this is larger.
7. After processing all characters, ensure all counts except `'k'` are zero, meaning all croaks are complete. If not, return `-1`.
8. Return `max_frogs`.

**Why it works:** At each step, the counters represent frogs in the middle of croaking. The invariants ensure no frog can skip a stage, and the maximum concurrent croaks correctly reflects the minimum number of frogs needed. The final check guarantees all croaks are complete.

## Python Solution

```python
class Solution:
    def minNumberOfFrogs(self, croakOfFrogs: str) -> int:
        stages = {'c': 0, 'r': 0, 'o': 0, 'a': 0, 'k': 0}
        prev = {'r': 'c', 'o': 'r', 'a': 'o', 'k': 'a'}
        max_frogs = 0
        
        for ch in croakOfFrogs:
            if ch not in stages:
                return -1
            if ch != 'c':
                if stages[prev[ch]] == 0:
                    return -1
                stages[prev[ch]] -= 1
            stages[ch] += 1
            max_frogs = max(max_frogs, stages['c'] + stages['r'] + stages['o'] + stages['a'])
        
        if stages['c'] + stages['r'] + stages['o'] + stages['a'] != 0:
            return -1
        
        return max_frogs
```

The code initializes counters for each stage of croaking. For each character, it validates that a frog is in the previous stage, then moves the frog to the current stage. The maximum sum of frogs in progress gives the answer. After iteration, any incomplete croaks result in `-1`.

## Go Solution

```go
func minNumberOfFrogs(croakOfFrogs string) int {
    stages := map[rune]int{'c': 0, 'r': 0, 'o': 0, 'a': 0, 'k': 0}
    prev := map[rune]rune{'r': 'c', 'o': 'r', 'a': 'o', 'k': 'a'}
    maxFrogs := 0
    
    for _, ch := range croakOfFrogs {
        if _, ok := stages[ch]; !ok {
            return -1
        }
        if ch != 'c' {
            if stages[prev[ch]] == 0 {
                return -1
            }
            stages[prev[ch]]--
        }
        stages[ch]++
        currFrogs := stages['c'] + stages['r'] + stages['o'] + stages['a']
        if currFrogs > maxFrogs {
            maxFrogs = currFrogs
        }
    }
    
    if stages['c']+stages['r']+stages['o']+stages['a'] != 0 {
        return -1
    }
    
    return maxFrogs
}
```

Go-specific differences include using `rune` for characters and maps for counts. The logic mirrors Python closely. The sum of intermediate stages gives current active frogs.
