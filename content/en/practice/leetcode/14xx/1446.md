## Solution

## Problem Understanding

The problem asks us to compute the "power" of a string. The power is defined as the length of the longest contiguous substring that contains only one distinct character.

In simpler terms, we need to scan the string and find the largest group of consecutive identical characters. The answer is the size of that group.

For example, in the string `"leetcode"`, the substring `"ee"` contains two consecutive `'e'` characters, and no other character appears consecutively more than twice. Therefore, the power is `2`.

The input consists of a single string `s` containing only lowercase English letters. The constraints specify that the length of the string is between `1` and `500`. This is a very small input size, which means even less efficient solutions would technically pass. However, the goal is still to design a clean and optimal algorithm.

A few important observations come from the constraints and problem definition:

- The substring must be contiguous. We cannot skip characters.
- Only one unique character may appear in the substring.
- The string is guaranteed to be non-empty, so there is always at least one valid substring.
- Since all characters are lowercase English letters, comparisons are straightforward character equality checks.

Several edge cases are important to think about:

- A string of length `1`, such as `"a"`, should return `1`.
- A string where all characters are identical, such as `"aaaaa"`, should return the full length of the string.
- A string where no adjacent characters match, such as `"abcde"`, should return `1`.
- Long runs appearing at the beginning or end of the string must be handled correctly.

## Approaches

### Brute Force Approach

A brute force solution would examine every possible substring of the input string and check whether that substring contains only one unique character.

The algorithm would work as follows:

1. Generate all substrings.
2. For each substring, verify whether every character matches the first character.
3. Track the maximum valid substring length.

This approach is correct because it explicitly checks every possible candidate substring, guaranteeing that the longest valid one is found.

However, it is inefficient. A string of length `n` has `O(n²)` substrings, and validating each substring may take up to `O(n)` time. This leads to a total complexity of `O(n³)`.

Even though the constraint is only `500`, this solution performs unnecessary repeated work.

### Optimal Approach

The key observation is that we do not need to examine every substring independently. The problem only asks for the longest sequence of consecutive identical characters.

Instead, we can scan the string once while maintaining:

- The current streak length of identical characters.
- The maximum streak length seen so far.

Whenever the current character matches the previous one, we extend the streak. Otherwise, we reset the streak to `1` because a new sequence begins.

This works because every valid substring consisting of one unique character is simply a consecutive run of identical characters.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Checks every substring and validates each one |
| Optimal | O(n) | O(1) | Single pass tracking consecutive character runs |

## Algorithm Walkthrough

### Optimal Sliding Scan Algorithm

1. Initialize two variables:

- `max_power` stores the largest streak found so far.
- `current_power` stores the current streak length.
2. Start both variables at `1` because the string is guaranteed to contain at least one character.
3. Iterate through the string starting from index `1`.
4. For each character:

- Compare it with the previous character.
- If they are equal, increment `current_power` because the streak continues.
- Otherwise, reset `current_power` to `1` because a new streak starts.
5. After updating the current streak, compare it with `max_power`.

- If `current_power` is larger, update `max_power`.
6. Continue until the end of the string.
7. Return `max_power`.

### Why it works

At every step, `current_power` represents the length of the current contiguous block of identical characters ending at the current index. Since every valid substring must be one of these contiguous blocks, tracking the largest block encountered guarantees that the final answer is correct.

## Python Solution

```python
class Solution:
    def maxPower(self, s: str) -> int:
        max_power = 1
        current_power = 1

        for index in range(1, len(s)):
            if s[index] == s[index - 1]:
                current_power += 1
            else:
                current_power = 1

            max_power = max(max_power, current_power)

        return max_power
```

The implementation begins by initializing both `max_power` and `current_power` to `1`. Since the string is non-empty, the minimum possible answer is always `1`.

The loop starts from index `1` because each character is compared with the previous character. If the characters match, the current streak grows by one. Otherwise, the streak resets because consecutive repetition has ended.

After processing each character, the algorithm updates `max_power` to ensure it always stores the largest streak encountered so far.

Finally, the method returns the maximum streak length.

## Go Solution

```go
func maxPower(s string) int {
    maxPower := 1
    currentPower := 1

    for i := 1; i < len(s); i++ {
        if s[i] == s[i-1] {
            currentPower++
        } else {
            currentPower = 1
        }

        if currentPower > maxPower {
            maxPower = currentPower
        }
    }

    return maxPower
}
```

The Go implementation follows the same logic as the Python version. Since strings in Go are indexable by byte and the problem guarantees lowercase English letters, direct byte comparison is completely safe.

No additional handling for empty strings is needed because the constraints guarantee that the input length is at least `1`.

## Edge Cases

A very important edge case is a string containing only one character, such as `"a"`. A naive implementation that starts iterating from index `1` without proper initialization could accidentally return `0`. This implementation avoids that problem by initializing both counters to `1`.

Another important case is when every character in the string is identical, such as `"aaaaaa"`. In this situation, the current streak continuously grows throughout the entire scan. The algorithm correctly increments `current_power` at every step and ultimately returns the full string length.

A third common source of bugs is when the longest streak appears at the beginning or end of the string. Some incorrect solutions update the maximum only when a streak ends, which may miss a run that reaches the final character. This implementation updates `max_power` during every iteration, ensuring that streaks at any position are handled correctly.
