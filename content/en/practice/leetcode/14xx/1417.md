## Solution

## Problem Understanding

The problem requires taking a string `s` that contains only lowercase letters and digits and rearranging its characters so that no two adjacent characters are of the same type. That is, a letter cannot be next to another letter, and a digit cannot be next to another digit. If such a rearrangement is impossible, the function should return an empty string.

The input string can have a length from 1 to 500, which is moderate and allows for algorithms that run in linear or linearithmic time. The string may be composed entirely of letters, entirely of digits, or a mix of both. The key challenge is determining whether a valid interleaving of letters and digits exists, and if so, constructing it efficiently.

Important edge cases include strings where all characters are letters or all characters are digits, strings where the difference in the count of letters versus digits is more than one (which makes interleaving impossible), and strings that are already alternating.

## Approaches

The brute-force approach would generate all permutations of the string and check for the alternating property. This guarantees correctness because it explores every possible arrangement. However, the factorial complexity `O(n!)` makes it completely infeasible for `n` up to 500.

A key observation enables a much more efficient solution. If the counts of letters and digits differ by more than 1, it is impossible to interleave them without having two adjacent characters of the same type. Otherwise, a valid rearrangement can always be constructed by alternating the characters, starting with the type that appears more frequently. This insight allows a linear-time algorithm that separates letters and digits, checks the counts, and merges them in alternating order.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n!) | O(n) | Generate all permutations and check alternating property |
| Optimal | O(n) | O(n) | Separate letters and digits, check counts, merge alternately |

## Algorithm Walkthrough

1. **Separate characters by type**: Iterate through the input string `s` and split it into two lists: one for letters and one for digits. This makes it easy to check counts and construct the output.
2. **Check feasibility**: If the difference in lengths of the letters and digits lists is greater than 1, return an empty string immediately because interleaving is impossible.
3. **Determine starting type**: If letters are more numerous, start the output with a letter. If digits are more numerous, start with a digit. If counts are equal, either type can start.
4. **Merge alternately**: Initialize an empty list for the result. Use two pointers to iterate through the letters and digits lists, appending one from each in turn, starting with the type chosen in the previous step.
5. **Return result**: Join the list of characters into a string and return it as the final answer.

Why it works: The invariant maintained is that letters and digits are appended alternately, and the check on the counts ensures that we never run out of one type before finishing the other. This guarantees the alternating pattern without violating the adjacency rule.

## Python Solution

```python
class Solution:
    def reformat(self, s: str) -> str:
        letters = [c for c in s if c.isalpha()]
        digits = [c for c in s if c.isdigit()]
        
        if abs(len(letters) - len(digits)) > 1:
            return ""
        
        if len(letters) > len(digits):
            longer, shorter = letters, digits
        else:
            longer, shorter = digits, letters
        
        result = []
        for i in range(len(shorter)):
            result.append(longer[i])
            result.append(shorter[i])
        if len(longer) > len(shorter):
            result.append(longer[-1])
        
        return "".join(result)
```

The implementation begins by separating the string into letters and digits. It checks the length difference to handle impossible cases efficiently. It identifies which list is longer to start the merging process, ensuring a valid alternating pattern. The final step joins the merged list into a single string.

## Go Solution

```go
func reformat(s string) string {
    letters := []byte{}
    digits := []byte{}
    
    for i := 0; i < len(s); i++ {
        if s[i] >= 'a' && s[i] <= 'z' {
            letters = append(letters, s[i])
        } else {
            digits = append(digits, s[i])
        }
    }
    
    if abs(len(letters)-len(digits)) > 1 {
        return ""
    }
    
    var longer, shorter []byte
    if len(letters) > len(digits) {
        longer, shorter = letters, digits
    } else {
        longer, shorter = digits, letters
    }
    
    result := make([]byte, 0, len(s))
    for i := 0; i < len(shorter); i++ {
        result = append(result, longer[i], shorter[i])
    }
    if len(longer) > len(shorter) {
        result = append(result, longer[len(longer)-1])
    }
    
    return string(result)
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}
```

The Go version mirrors the Python solution. Differences include using byte slices to represent characters and the need for an explicit `abs` function. Go handles slices efficiently, and the final conversion to string is explicit.

## Edge Cases

One important edge case is a string with only letters or only digits. If the implementation does not check counts, it may attempt to merge empty lists, producing index errors or invalid outputs. The solution handles this by checking the absolute difference and returning an empty string.

A second edge case is strings where the number of letters and digits differ by exactly 1. These must start with the type that has the extra element to avoid consecutive same-type characters. Our solution explicitly identifies the longer list and begins merging with it.

A third edge case is strings of length 1. Even though alternating is impossible, a single character is trivially valid, so the solution allows it by the separation and merge logic without triggering the feasibility check.
