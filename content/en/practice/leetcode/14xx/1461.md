## Solution

## Problem Understanding

The problem is asking us to determine if a given binary string `s` contains **all possible binary codes of length `k`** as substrings. In simpler terms, every combination of `k` bits, from `"0"*k` to `"1"*k`, must appear somewhere in `s`. The input consists of a string `s` containing only `'0'` and `'1'` characters and an integer `k` specifying the length of the binary codes we are checking for. The output is a boolean: `true` if all binary codes exist as substrings in `s`, and `false` otherwise.

The constraints indicate that `s` can be very large, up to `500,000` characters, while `k` is small (maximum 20). Since the number of distinct binary codes of length `k` is `2^k`, we know that for the largest `k = 20`, there could be over a million distinct codes. This implies that any solution iterating over all possible substrings without optimization would be inefficient for large inputs.

Important edge cases include situations where `k` is larger than the length of `s` (immediate `false`), `s` contains repeated patterns that might not cover all codes, or `k = 1`, where only `"0"` and `"1"` need to appear.

## Approaches

### Brute Force Approach

The brute-force approach involves generating **all possible binary codes of length `k`**, storing them in a set, and then checking every substring of `s` of length `k` to see if it matches a code in the set. Whenever a code is found, it is removed from the set. At the end, if the set is empty, all codes were found.

While this works logically, it is inefficient because for each substring, we need to construct a string and check membership in the set, which has a time complexity of `O(n * k)` where `n = len(s)`. For large strings and larger `k`, this becomes slow.

### Optimal Approach

The key insight is that we **do not need to generate the binary codes explicitly**. We can use a **rolling hash** to convert each substring of length `k` into an integer representing its binary value. This allows us to efficiently track which codes have appeared using a set or boolean array. Specifically, we can slide a window of size `k` across the string, updating the rolling hash in constant time. This reduces the time complexity to O(n), which is acceptable for large strings. Using a bitmask helps keep track of which codes are present without repeatedly converting strings to integers.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n*k) | O(2^k) | Generate all substrings and match against a set of codes |
| Optimal | O(n) | O(2^k) | Rolling hash with bitmask to track observed codes efficiently |

## Algorithm Walkthrough

1. Check if `k` is larger than the length of `s`. If it is, return `false` immediately because `s` cannot contain all codes of length `k`.
2. Initialize a set to store **all unique codes** found while iterating through `s`. Optionally, use a boolean array of size `2^k` for faster access.
3. Initialize a variable `hash_value` to 0. This will store the integer value of the current substring window.
4. Slide a window of length `k` over `s`. For each new character:

- Left-shift the current hash by 1 bit and add the integer value of the new bit.
- If the window size exceeds `k`, remove the leftmost bit using a bitmask operation `(hash_value &= (1 << k) - 1)`.
- Add the `hash_value` to the set of observed codes.
5. After iterating through `s`, check if the size of the set equals `2^k`. If it does, return `true`. Otherwise, return `false`.

**Why it works**: The rolling hash guarantees that each substring is converted to a unique integer between 0 and `2^k - 1`. By tracking all integers seen, we ensure that all possible codes are observed exactly once if they exist. This approach avoids redundant string operations and leverages efficient integer arithmetic.

## Python Solution

```python
class Solution:
    def hasAllCodes(self, s: str, k: int) -> bool:
        n = len(s)
        if k > n:
            return False

        required_codes = 1 << k  # 2^k total codes
        seen = set()
        hash_value = 0

        for i in range(n):
            # Shift left and add new bit
            hash_value = ((hash_value << 1) & ((1 << k) - 1)) | int(s[i])
            # Only start adding when we have the first full window
            if i >= k - 1:
                seen.add(hash_value)
                if len(seen) == required_codes:
                    return True

        return len(seen) == required_codes
```

The Python implementation creates a rolling hash for each window of size `k`, using bitmasking to remove bits that slide out of the window. The set `seen` keeps track of all unique codes encountered. If all `2^k` codes are seen, the function returns `true` early for efficiency.

## Go Solution

```go
func hasAllCodes(s string, k int) bool {
    n := len(s)
    if k > n {
        return false
    }

    requiredCodes := 1 << k
    seen := make(map[int]struct{})
    hashValue := 0

    for i := 0; i < n; i++ {
        hashValue = ((hashValue << 1) & ((1 << k) - 1)) | int(s[i]-'0')
        if i >= k-1 {
            seen[hashValue] = struct{}{}
            if len(seen) == requiredCodes {
                return true
            }
        }
    }

    return len(seen) == requiredCodes
}
```

In Go, the implementation is similar. The map `seen` uses an empty struct as the value to save memory. The key differences include explicit type conversion when reading a character from `s` and the use of `len(seen)` for checking set size.
