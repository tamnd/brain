## Solution

## Problem Understanding

This problem asks us to find the **longest contiguous subarray** of positive integers such that the **bitwise AND** of every pair of distinct elements in that subarray is `0`. In simpler terms, for any two numbers in this subarray, they do not share a `1` bit in the same position. A subarray of length `1` is trivially "nice" because there is no pair to compare.

The input is an array `nums` of positive integers, and the output is a single integer representing the length of the longest nice subarray. The constraints indicate that `nums` can have up to `10^5` elements, and each element can be up to `10^9`, which means a brute-force comparison of all subarrays is likely too slow.

Important edge cases include arrays where every element shares common bits (in which case the longest nice subarray may be of length `1`), arrays where all elements are powers of two (maximizing the chance for longer nice subarrays), and arrays with a single element.

## Approaches

The **brute-force approach** is straightforward. For each starting index in the array, we attempt to extend the subarray as far as possible to the right while maintaining the "nice" property. We can check every pair in the current subarray using nested loops and a bitwise AND. This approach is correct because it exhaustively examines all subarrays, but it is **too slow**. With a worst-case time complexity of O(n^2) to O(n^3) depending on implementation, it cannot handle the maximum input size of `10^5`.

The **optimal approach** uses a **sliding window with bitmasking**. Instead of checking all pairs repeatedly, we maintain a single integer `window_mask` representing the bitwise OR of all elements in the current window. If a new element has any common bit with `window_mask` (i.e., `window_mask & nums[right] != 0`), it violates the nice property, so we slide the left pointer to remove elements until it is safe to include the new element. This reduces the pairwise checks to a single bitwise operation per element, achieving **linear time complexity**.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check every subarray and all pairs using nested loops |
| Optimal | O(n) | O(1) | Sliding window with bitmask to track combined bits |

## Algorithm Walkthrough

1. Initialize two pointers `left = 0` and `right = 0`, representing the sliding window's bounds.
2. Initialize `window_mask = 0`, which will store the bitwise OR of all elements in the current window.
3. Initialize `max_length = 0` to store the length of the longest nice subarray found so far.
4. Iterate `right` over each index in `nums`:

1. While the new element `nums[right]` shares any bit with `window_mask` (i.e., `window_mask & nums[right] != 0`), remove elements from the left of the window:

1. Subtract the leftmost element's bits from `window_mask` using `window_mask ^= nums[left]`.
2. Increment `left` to shrink the window.
2. Add the current element to `window_mask` using `window_mask |= nums[right]`.
3. Update `max_length` with the current window size `right - left + 1`.
5. After the loop ends, return `max_length`.

**Why it works**: The `window_mask` invariant ensures that all elements in the current window have no overlapping bits. Removing elements from the left ensures we maintain the nice property. Since we process each element at most twice (once when added, once when removed), the algorithm is O(n).

## Python Solution

```python
from typing import List

class Solution:
    def longestNiceSubarray(self, nums: List[int]) -> int:
        left = 0
        window_mask = 0
        max_length = 0
        
        for right in range(len(nums)):
            while window_mask & nums[right]:
                window_mask ^= nums[left]
                left += 1
            window_mask |= nums[right]
            max_length = max(max_length, right - left + 1)
        
        return max_length
```

The code uses a sliding window approach. `window_mask` tracks which bits are currently present in the window. If adding a new number would introduce overlapping bits, we slide the left pointer and remove numbers from the mask until the new number can be safely included. We keep updating `max_length` for each valid window.

## Go Solution

```go
func longestNiceSubarray(nums []int) int {
    left := 0
    windowMask := 0
    maxLength := 0

    for right := 0; right < len(nums); right++ {
        for windowMask & nums[right] != 0 {
            windowMask ^= nums[left]
            left++
        }
        windowMask |= nums[right]
        if right-left+1 > maxLength {
            maxLength = right - left + 1
        }
    }

    return maxLength
}
```

The Go solution mirrors the Python implementation. We handle slices and integers natively. Go does not require explicit imports for basic types and bitwise operations. The approach ensures that each number is added and removed at most once, maintaining O(n) complexity.

## Edge Cases

One edge case is an array with **all elements equal**, e.g., `[7,7,7,7]`. Here, any two elements will always share bits, so the longest nice subarray is length `1`. Our implementation correctly handles this because the sliding window shrinks to size `1` whenever a conflict occurs.

Another edge case is an array of **powers of two**, e.g., `[1,2,4,8]`. Since each number has only one set bit, all numbers can coexist, and the algorithm correctly expands the window to include all elements, giving the maximum length.

A third edge case is an array with **alternating high-bit overlaps**, such as `[1,3,2,6]`. Our algorithm dynamically adjusts the left pointer whenever a conflict arises, ensuring we never include conflicting numbers while still maximizing the window size. This prevents subtle off-by-one or mask calculation errors.
