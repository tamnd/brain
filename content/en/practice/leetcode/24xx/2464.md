## Solution

## Problem Understanding

The problem asks us to split the array into the minimum possible number of contiguous subarrays such that every subarray satisfies one condition:

The greatest common divisor, or GCD, of the first element and the last element of that subarray must be greater than `1`.

A split is valid only if:

- every element belongs to exactly one subarray,
- the subarrays are contiguous,
- and every subarray individually satisfies the GCD condition.

We are asked to return the minimum number of subarrays needed to create such a valid split. If no valid splitting exists, we return `-1`.

Suppose we choose a subarray `nums[i:j+1]`. The only thing that matters for validity is:

```
gcd(nums[i], nums[j]) > 1
```

The values in the middle of the subarray do not affect validity directly.

An important observation is that a single-element subarray is always valid if the element itself is greater than `1`, because:

$\gcd(x,x)=x$

So:

- `[2]` is valid,
- `[5]` is valid,
- `[1]` is invalid because `gcd(1,1) = 1`.

The constraints are relatively small:

- `1 <= nums.length <= 1000`
- `1 <= nums[i] <= 10^5`

A quadratic dynamic programming solution is completely feasible because:

$1000^2 = 10^6$

Around one million pair checks is efficient enough.

Several edge cases are important:

- Arrays containing `1` are dangerous because `[1]` cannot form a valid subarray.
- If every element is prime and no adjacent connectivity exists through valid endpoints, the answer may be impossible.
- A single-element array containing `1` must return `-1`.
- A single-element array containing any number greater than `1` returns `1`.

## Approaches

### Brute Force Approach

The brute force idea is to generate every possible way to split the array and check whether all produced subarrays are valid.

For an array of length `n`, there are `2^(n-1)` possible splitting patterns because each gap between elements can either contain a cut or not contain a cut.

For every split configuration, we would:

1. Construct all subarrays.
2. Check whether each subarray satisfies:

$\gcd(first,last)>1$
3. Count the number of subarrays.
4. Keep the minimum valid answer.

This approach is correct because it exhaustively checks all possibilities. However, it is far too slow because exponential growth becomes impossible even for moderate `n`.

### Key Insight

The key insight is that this problem has optimal substructure.

If we know the minimum number of valid subarrays needed to cover the prefix ending at position `i`, then we can extend that solution using a valid subarray ending at a later position `j`.

Define:

- `dp[i]` = minimum number of valid subarrays needed to cover `nums[0:i]`

For every ending position `i`, we try every possible starting position `j`.

If:

$\gcd(nums[j],nums[i])>1$

then the subarray `nums[j:i+1]` is valid.

The transition becomes:

- if `j == 0`, then this entire prefix forms one valid subarray,
- otherwise:

$dp[i]=\min(dp[i],dp[j-1]+1)$

This transforms the problem into a classic dynamic programming interval transition.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Tries every possible split configuration |
| Optimal Dynamic Programming | O(n² * log M) | O(n) | Checks every valid subarray endpoint pair using GCD |

Here, `M` is the maximum value in `nums`, and the logarithmic factor comes from the GCD computation.

## Algorithm Walkthrough

1. Create a DP array of length `n`.

`dp[i]` will store the minimum number of valid subarrays needed to cover indices `0` through `i`.
2. Initialize all values to infinity.

This represents that we have not yet found a valid way to reach those positions.
3. Iterate through every possible ending index `end`.

For each position, we try to determine the best valid split ending there.
4. For each `end`, iterate through every possible starting index `start`.

This checks every subarray `nums[start:end+1]`.
5. Compute the GCD of the first and last elements.

If:

$\gcd(nums[start],nums[end])>1$

then the subarray is valid.
6. Handle the transition.

If `start == 0`, then the entire prefix forms one valid subarray:

```
dp[end] = 1
```

Otherwise, if `dp[start - 1]` is reachable:

```
dp[end] = min(dp[end], dp[start - 1] + 1)
```
7. After processing all states, check `dp[n - 1]`.

- If it is still infinity, return `-1`.
- Otherwise return the computed minimum.

### Why it works

The algorithm works because every valid solution must end with some valid final subarray.

Suppose the final subarray begins at index `start` and ends at index `end`.

Then:

- the prefix before `start` must already have been split optimally,
- and the last subarray must satisfy the GCD condition.

By considering every possible `start` for every `end`, the DP explores all valid decompositions. Since every transition builds from previously optimal prefixes, the final answer is globally optimal.

## Python Solution

```python
from typing import List
from math import gcd

class Solution:
    def validSubarraySplit(self, nums: List[int]) -> int:
        n = len(nums)
        INF = float("inf")

        dp = [INF] * n

        for end in range(n):
            for start in range(end + 1):
                if gcd(nums[start], nums[end]) > 1:
                    if start == 0:
                        dp[end] = 1
                    elif dp[start - 1] != INF:
                        dp[end] = min(dp[end], dp[start - 1] + 1)

        return -1 if dp[-1] == INF else dp[-1]
```

The implementation directly follows the DP formulation.

We first initialize the DP array with infinity values to represent unreachable states. Then we iterate over every possible subarray ending position.

For each pair `(start, end)`, we compute the GCD of the boundary elements. If the GCD is greater than `1`, then the subarray is valid and can potentially improve the solution.

When `start == 0`, the entire prefix is valid as a single subarray, so we assign `1`.

Otherwise, we extend the optimal solution for the previous prefix ending at `start - 1`.

Finally, we check whether the last position is reachable.

## Go Solution

```go
package main

func gcd(a, b int) int {
	for b != 0 {
		a, b = b, a%b
	}
	return a
}

func validSubarraySplit(nums []int) int {
	n := len(nums)
	const INF = int(1e9)

	dp := make([]int, n)

	for i := 0; i < n; i++ {
		dp[i] = INF
	}

	for end := 0; end < n; end++ {
		for start := 0; start <= end; start++ {
			if gcd(nums[start], nums[end]) > 1 {
				if start == 0 {
					dp[end] = 1
				} else if dp[start-1] != INF {
					if dp[start-1]+1 < dp[end] {
						dp[end] = dp[start-1] + 1
					}
				}
			}
		}
	}

	if dp[n-1] == INF {
		return -1
	}

	return dp[n-1]
}
```

The Go implementation mirrors the Python logic closely.

Since Go does not provide a built-in infinity constant for integers, we use a large sentinel value like `1e9`.

The GCD function is implemented manually using the Euclidean algorithm.

Slices are used instead of Python lists, but the transition logic remains identical.

| --- | --- | --- |
| Time | O(n² * log M) | We check every `(start, end)` pair and compute GCD |
| Space | O(n) | The DP array stores one value per index |

The algorithm uses two nested loops over the array, producing `O(n²)` subarray endpoint checks. Each GCD computation takes logarithmic time relative to the magnitude of the numbers due to the Euclidean algorithm.

The space usage is linear because we only maintain the DP array.

## Edge Cases

One important edge case is arrays containing `1`. Since:

$\gcd(1,1)=1$

a singleton subarray containing `1` is invalid. Additionally, `1` has no common divisor greater than `1` with any other number. This means any segment starting or ending with `1` immediately fails. The implementation naturally handles this because the GCD check never succeeds.

Another important edge case is when every element must become its own subarray. For example:

```
[2,3,5,7]
```

No pair of distinct elements shares a GCD greater than `1`, so no larger subarray is valid. However, each individual prime number forms a valid singleton subarray. The DP correctly computes this by repeatedly extending from previous valid prefixes.

A third edge case is when the entire array forms one valid segment even though middle elements are unrelated. For example:

```
[4,1,6]
```

The middle value does not matter because only the first and last elements determine validity. Since:

$\gcd(4,6)=2$

the whole array is valid. A common mistake is incorrectly checking all elements inside the segment instead of only the endpoints. The implementation avoids this by using only `nums[start]` and `nums[end]` in the GCD calculation.
