## Solution

## Problem Understanding

We are given a permutation of the integers from `0` to `n - 1`. Every number appears exactly once. The value `0` is special because it represents the empty space, while all other values represent movable items.

In one operation, we may move any item into the current empty space. After that move, the item's previous position becomes the new empty space. This means the empty slot effectively swaps positions with the chosen item.

The goal is to transform the array into one of two valid sorted configurations:

- `[0,1,2,3,...,n-1]`
- `[1,2,3,...,n-1,0]`

The first configuration places the empty space at the beginning, while the second places it at the end. We must return the minimum number of operations required to reach either configuration.

The constraints are important:

- `n` can be as large as `10^5`
- Every value is unique
- The array is always a permutation

These constraints immediately rule out exponential or quadratic search-based approaches. We need something close to linear time.

A key observation is that this problem is fundamentally about permutation cycles. Since every number has exactly one correct position in the target arrangement, the array defines a mapping from current positions to desired positions.

Several edge cases are especially important:

- The array may already be sorted in one of the two valid forms.
- The empty space may already be correctly positioned.
- The permutation may contain cycles that do not involve `0`.
- The optimal answer may differ depending on whether we target `0` at the front or at the back.

A correct solution must evaluate both target configurations and return the smaller answer.

## Approaches

### Brute Force Approach

A brute force solution would model the array states as nodes in a graph. From any state, we could generate all possible states obtained by moving one item into the empty space. Then we could perform BFS until we reach a sorted configuration.

This guarantees correctness because BFS explores states in increasing order of operations, so the first valid sorted state found would use the minimum number of moves.

However, this approach is completely infeasible for the given constraints. There are `n!` possible permutations, which becomes astronomically large even for modest `n`.

Even with aggressive pruning, the state space explosion makes brute force unusable.

### Optimal Approach

The efficient solution comes from analyzing permutation cycles.

For a chosen target arrangement, every index has exactly one value that belongs there. We can view the array as a permutation where each misplaced element points to its correct location.

The important insight is:

- Cycles containing `0` can be resolved directly.
- Cycles not containing `0` require an extra move to bring the empty space into the cycle.

This leads to a linear-time simulation approach.

We evaluate both target configurations:

1. `0` at the beginning
2. `0` at the end

For each target, we compute the minimum moves needed, then take the smaller result.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n!) | O(n!) | Explores permutation states with BFS |
| Optimal | O(n) | O(n) | Uses permutation cycle properties |

## Algorithm Walkthrough

### Target Configuration

Suppose we want the final array to match some target permutation.

For example:

```
nums   = [4,2,0,3,1]
target = [0,1,2,3,4]
```

Each value has exactly one correct destination.

We maintain:

- `arr`, a mutable copy of the array
- `position[value]`, the current index of every value

This allows constant-time swaps and updates.

### Numbered Steps

1. Build the target array.

We try two targets:

- `[0,1,2,...,n-1]`
- `[1,2,...,n-1,0]`
2. Create a position map.

For every value, store its current index.

This lets us instantly find where any item currently sits.
3. Repeatedly fix positions while the array is not sorted.

At each step, inspect the current location of `0`.
4. If `0` is not already in its target position:

Determine which value should occupy the current position of `0`.

Move that value into the empty space.

This fixes one position immediately.
5. If `0` is already in its target position but the array is still unsorted:

We are inside a cycle that does not involve `0`.

Pick any misplaced element and swap it with `0`.

This inserts the empty space into that cycle.
6. Continue until every position matches the target.
7. Repeat the process for both target configurations.
8. Return the smaller move count.

### Why it works

The algorithm works because every swap involving `0` either:

- Directly fixes one incorrect position, or
- Introduces `0` into a cycle that previously could not be resolved.

A cycle containing `k` elements:

- Costs `k - 1` moves if it already contains `0`
- Costs `k + 1` moves otherwise

The simulation achieves exactly these optimal costs, so the total number of operations is minimal.

## Python Solution

```python
from typing import List

class Solution:
    def sortArray(self, nums: List[int]) -> int:
        n = len(nums)

        def solve(target: List[int]) -> int:
            arr = nums[:]
            position = [0] * n

            for i, value in enumerate(arr):
                position[value] = i

            moves = 0

            while True:
                zero_index = position[0]

                if arr == target:
                    return moves

                # If 0 is not in its target position,
                # place the correct value into zero's position.
                if target[zero_index] != 0:
                    needed_value = target[zero_index]
                    needed_index = position[needed_value]

                    arr[zero_index], arr[needed_index] = (
                        arr[needed_index],
                        arr[zero_index],
                    )

                    position[0] = needed_index
                    position[needed_value] = zero_index

                    moves += 1

                else:
                    # 0 is already in the correct position.
                    # Bring 0 into another incorrect cycle.
                    misplaced_index = -1

                    for i in range(n):
                        if arr[i] != target[i]:
                            misplaced_index = i
                            break

                    arr[zero_index], arr[misplaced_index] = (
                        arr[misplaced_index],
                        arr[zero_index],
                    )

                    moved_value = arr[zero_index]

                    position[0] = misplaced_index
                    position[moved_value] = zero_index

                    moves += 1

        target_front = list(range(n))
        target_back = list(range(1, n)) + [0]

        return min(
            solve(target_front),
            solve(target_back),
        )
```

The implementation defines a helper function `solve()` that computes the minimum moves required for a specific target arrangement.

A copy of the original array is created because we need to evaluate two independent target configurations.

The `position` array maps each value to its current index. This is crucial because we repeatedly need to locate the correct item to move into the empty space. Without this structure, each lookup would cost linear time.

Inside the main loop, we first locate the current position of `0`.

If `0` is not where it belongs according to the target configuration, we move the correct value into that slot. This immediately fixes one target position.

If `0` is already correctly placed but the array is still unsorted, then some cycle does not involve `0`. We deliberately swap `0` with any misplaced element so that future swaps can resolve that cycle.

The process repeats until the array exactly matches the target.

Finally, we compute the answer for both valid target configurations and return the smaller result.

## Go Solution

```go
package main

func sortArray(nums []int) int {
	n := len(nums)

	solve := func(target []int) int {
		arr := make([]int, n)
		copy(arr, nums)

		position := make([]int, n)

		for i, value := range arr {
			position[value] = i
		}

		moves := 0

		for {
			if isEqual(arr, target) {
				return moves
			}

			zeroIndex := position[0]

			if target[zeroIndex] != 0 {
				neededValue := target[zeroIndex]
				neededIndex := position[neededValue]

				arr[zeroIndex], arr[neededIndex] =
					arr[neededIndex], arr[zeroIndex]

				position[0] = neededIndex
				position[neededValue] = zeroIndex

				moves++
			} else {
				misplacedIndex := -1

				for i := 0; i < n; i++ {
					if arr[i] != target[i] {
						misplacedIndex = i
						break
					}
				}

				arr[zeroIndex], arr[misplacedIndex] =
					arr[misplacedIndex], arr[zeroIndex]

				movedValue := arr[zeroIndex]

				position[0] = misplacedIndex
				position[movedValue] = zeroIndex

				moves++
			}
		}
	}

	targetFront := make([]int, n)
	for i := 0; i < n; i++ {
		targetFront[i] = i
	}

	targetBack := make([]int, n)
	for i := 0; i < n-1; i++ {
		targetBack[i] = i + 1
	}
	targetBack[n-1] = 0

	a := solve(targetFront)
	b := solve(targetBack)

	if a < b {
		return a
	}
	return b
}

func isEqual(a []int, b []int) bool {
	for i := 0; i < len(a); i++ {
		if a[i] != b[i] {
			return false
		}
	}
	return true
}
```

The Go implementation follows the same logic as the Python version. Since Go does not have built-in list equality comparison like Python, we use a helper function `isEqual()`.

Slices are copied explicitly using `copy()` so that each target configuration operates on an independent array state.

All operations use standard integer arithmetic, so overflow is not a concern under the given constraints.

Although the implementation contains loops, every element is moved only a constant number of times. The total number of swaps across the algorithm is linear in `n`.

The auxiliary space comes primarily from the position mapping and copied arrays used during simulation.

## Edge Cases

### Array Already Sorted

An important edge case occurs when the input already matches one of the two valid target configurations.

Examples include:

```
[0,1,2,3]
[1,2,3,0]
```

A buggy implementation might still perform unnecessary swaps. This solution avoids that by checking whether the current array already equals the target before attempting any operations.

### Cycles That Do Not Include `0`

Consider:

```
[0,2,1,3]
```

The elements `1` and `2` form a cycle that does not involve the empty space. A naive strategy that only fixes positions adjacent to `0` can get stuck.

The implementation handles this by deliberately swapping `0` with a misplaced element whenever `0` is already in its correct target position. This injects the empty space into the cycle so it can be resolved.

### Choosing the Wrong Final Configuration

Some arrays are cheaper to sort with `0` at the front, while others are cheaper with `0` at the end.

For example:

```
[1,2,3,4,0]
```

is already solved if `0` belongs at the end, but would require several moves if we forced `0` to the front.

The implementation correctly evaluates both valid targets independently and returns the minimum result.
