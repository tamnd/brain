## Problem Understanding

The problem asks us to compute, for each word in an array of strings, the sum of **scores of all its prefixes**. The **score of a prefix** is defined as the number of strings in the array that have that prefix.

For example, if the array is `["abc", "ab", "bc", "b"]`, the word `"abc"` has prefixes `"a"`, `"ab"`, `"abc"`. The score of `"a"` is 2 because `"abc"` and `"ab"` start with `"a"`. The score of `"ab"` is 2 because `"abc"` and `"ab"` start with `"ab"`. The score of `"abc"` is 1 because only `"abc"` starts with `"abc"`. Summing them gives `2 + 2 + 1 = 5`.

The input consists of up to 1000 words, each with up to 1000 characters. This gives a total character count of up to 1,000,000. A naive solution that checks every prefix against all words would have a quadratic runtime in terms of characters, which is too slow.

Important edge cases include single-character words, words that are prefixes of other words, and words that share no common prefixes. The input guarantees non-empty lowercase strings, so we do not need to handle empty strings or uppercase letters.

## Approaches

The **brute-force approach** is to generate all prefixes of each word, then for each prefix, iterate through all words and count how many start with that prefix. This works correctly but is extremely slow because for each word of length `L` and `n` words, it performs O(L × n) prefix comparisons. For large inputs, this is inefficient.

The **optimal approach** uses a **Trie** (prefix tree). Each node in the Trie represents a character and stores a **count** of how many words have passed through it. By inserting all words into the Trie and incrementing the counts along the path, we can later sum the counts along the path of each word to compute the total prefix scores in O(total_characters) time. The key insight is that prefix overlaps are naturally captured by the Trie, allowing us to avoid repeated scanning.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × L²) | O(1) extra | For each prefix of each word, check all words |
| Trie | O(n × L) | O(n × L) | Insert words in Trie with counts, sum along paths |

## Algorithm Walkthrough

1. **Initialize a Trie** with a root node. Each node contains a `count` and a dictionary of children nodes.
2. **Insert all words into the Trie**. For each word, start at the root and iterate through each character. For each character, move to the corresponding child node (creating it if it does not exist) and increment the node's `count`. The `count` will eventually represent the number of words that share that prefix.
3. **Compute prefix sums for each word**. For each word, start at the root and traverse the Trie according to its characters. At each node, add the node's `count` to a running sum. This sum will be the sum of scores of all prefixes of that word.
4. **Return the result array**, containing the prefix sum for each word in the order they appear in the input.

**Why it works**: Each Trie node’s `count` represents exactly how many words share the prefix up to that node. Summing counts along a word's path gives the total sum of its prefix scores. This ensures correctness by construction.

## Python Solution

```python
from typing import List

class TrieNode:
    def __init__(self):
        self.count = 0
        self.children = {}

class Solution:
    def sumPrefixScores(self, words: List[str]) -> List[int]:
        root = TrieNode()
        
        # Insert all words into the Trie
        for word in words:
            node = root
            for ch in word:
                if ch not in node.children:
                    node.children[ch] = TrieNode()
                node = node.children[ch]
                node.count += 1
        
        # Compute prefix scores for each word
        result = []
        for word in words:
            node = root
            total = 0
            for ch in word:
                node = node.children[ch]
                total += node.count
            result.append(total)
        
        return result
```

The implementation creates a `TrieNode` class with a `count` and `children` dictionary. It first builds the Trie with all words, incrementing counts at each node along the path. Then for each word, it traverses the Trie, summing the counts to get the total prefix score.

## Go Solution

```go
package main

func sumPrefixScores(words []string) []int {
    type TrieNode struct {
        count    int
        children map[rune]*TrieNode
    }
    
    root := &TrieNode{children: make(map[rune]*TrieNode)}
    
    // Insert all words into Trie
    for _, word := range words {
        node := root
        for _, ch := range word {
            if node.children[ch] == nil {
                node.children[ch] = &TrieNode{children: make(map[rune]*TrieNode)}
            }
            node = node.children[ch]
            node.count++
        }
    }
    
    // Compute prefix scores
    result := make([]int, len(words))
    for i, word := range words {
        node := root
        total := 0
        for _, ch := range word {
            node = node.children[ch]
            total += node.count
        }
        result[i] = total
    }
    
    return result
}
```

The Go version follows the same logic. The main differences are using a `map[rune]*TrieNode` for children and handling slices instead of lists. Initialization ensures no nil pointer errors.

## Edge Cases

1. **Single-character words**: Words like `"a"` or `"z"` test the handling of minimal length. The implementation correctly counts duplicates if they exist.
2. **Words that are prefixes of others**: Words like `"ab"`, `"abc"`, `"abcd"` ensure the Trie correctly accumulates counts along shared paths. Without a Trie, repeated counting could be inefficient.
3. **Non-overlapping prefixes**: Words like `"a"`, `"b"`, `"c"` ensure the algorithm does not mistakenly count prefixes across unrelated words. The Trie naturally separates distinct paths.

This solution handles all cases efficiently and correctly.
