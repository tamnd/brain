## Solution

## Problem Understanding

The problem gives us the root of a **perfect binary tree** and asks us to reverse the node values at every **odd level** of the tree.

A binary tree level is defined by the number of edges from the root. This means:

- Level `0` contains the root node.
- Level `1` contains the root’s children.
- Level `2` contains the grandchildren.
- Level `3` contains the next layer, and so on.

We only reverse values at **odd-numbered levels**, meaning level `1`, level `3`, level `5`, and so forth.

An important detail is that we are **not changing the tree structure**. We are only swapping the **values stored in nodes**. The parent-child relationships remain exactly the same.

For example, if level `1` contains values `[3, 5]`, after reversal it becomes `[5, 3]`. Similarly, if level `3` contains `[2,1,3,4,7,11,29,18]`, after reversal it becomes `[18,29,11,7,4,3,1,2]`.

The input is a perfect binary tree. This guarantee matters because every internal node has exactly two children, and all leaves are at the same depth. Since the tree is symmetric at every level, nodes on the left side always have corresponding mirror nodes on the right side.

The constraints indicate that the tree contains at most `2^14 = 16384` nodes. This is large enough that we want an efficient linear solution, but small enough that recursion depth is manageable.

There are several important edge cases:

A tree with only one node has no odd levels, so it should remain unchanged.

A tree with only two levels requires reversing exactly one pair of values.

Trees with repeated values can be tricky because swapping equal values appears to do nothing, but the algorithm still needs to correctly process positions.

Because the tree is guaranteed to be perfect, we never need to worry about missing children or irregular shapes. This simplifies traversal logic considerably.

## Approaches

### Brute Force Approach, Level Order Traversal with Array Reversal

A straightforward solution is to perform a **breadth-first search (BFS)** level by level.

For each level, we gather all nodes into an array. If the current level number is odd, we reverse the values by swapping the first node with the last node, the second with the second-last, and so on.

This works because BFS naturally processes nodes level by level. Since we explicitly collect nodes of each level, reversing becomes simple.

The algorithm proceeds as follows:

1. Use a queue for level order traversal.
2. For each level, collect nodes into a temporary list.
3. If the level index is odd, reverse the values.
4. Push children into the queue for the next level.

This approach is correct because every odd level is explicitly visited and reversed. However, it uses extra memory proportional to the width of the tree to store level nodes.

### Optimal Approach, Symmetric DFS

The key observation comes from the **perfect binary tree property**.

In a perfect tree, nodes at odd levels can be reversed by swapping values between **mirror nodes**.

For example:

```
        2
      /   \
     3     5
    / \   / \
   8 13 21 34
```

At level `1`, `3` and `5` are mirror nodes. Reversing the level simply means swapping them.

For deeper levels, mirror relationships continue:

```
left.left  <-> right.right
left.right <-> right.left
```

This means we can traverse the tree in **pairs of mirrored nodes**.

We perform DFS on two nodes simultaneously:

- `(left child, right child)`
- `(left.left, right.right)`
- `(left.right, right.left)`

Whenever we are at an odd level, we swap the node values.

This avoids explicitly storing levels and takes advantage of the tree symmetry.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force BFS | O(n) | O(n) | Uses queue and temporary arrays for levels |
| Optimal Symmetric DFS | O(n) | O(h) | Exploits perfect tree symmetry, only recursion stack |

Here, `n` is the number of nodes and `h` is the tree height.

## Algorithm Walkthrough

We use a recursive DFS that traverses mirrored node pairs.

### Step 1: Start from the root’s children

The root is level `0`, which is even, so we do not reverse it.

We begin recursion with:

```
(root.left, root.right, level = 1)
```

This pair represents mirrored nodes at level `1`.

### Step 2: Swap values at odd levels

Whenever the current level is odd:

```
if level % 2 == 1:
    swap(left.val, right.val)
```

We do this because reversing a level means swapping symmetric positions.

For example:

```
[3, 5]
```

becomes:

```
[5, 3]
```

### Step 3: Recurse into mirrored children

Because the tree is perfect, mirrored relationships are predictable.

We recurse into:

1. `left.left` and `right.right`
2. `left.right` and `right.left`

This preserves symmetry.

Example:

```
        2
      /   \
     A     B
    / \   / \
   C   D E   F
```

Mirror pairs are:

```
C <-> F
D <-> E
```

### Step 4: Stop at leaves

If either node becomes `None`, recursion stops.

Since the tree is perfect, this only happens after reaching leaf level.

### Why it works

The correctness comes from the invariant that every recursive call processes a pair of nodes occupying **mirror positions at the same level**.

At odd levels, swapping mirrored values exactly reproduces the effect of reversing the level array. Since every node belongs to exactly one mirrored pair, all odd levels are reversed correctly without affecting even levels or tree structure.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def reverseOddLevels(self, root: Optional[TreeNode]) -> Optional[TreeNode]:
        def dfs(left: Optional[TreeNode],
                right: Optional[TreeNode],
                level: int) -> None:
            
            if not left or not right:
                return

            if level % 2 == 1:
                left.val, right.val = right.val, left.val

            dfs(left.left, right.right, level + 1)
            dfs(left.right, right.left, level + 1)

        if not root:
            return root

        dfs(root.left, root.right, 1)

        return root
```

The implementation defines an inner recursive function `dfs()` that takes two mirrored nodes and their current level.

The base case checks whether either node is `None`. This prevents unnecessary recursion once leaf boundaries are reached.

When the level is odd, the values of the mirrored nodes are swapped. Since the tree structure remains unchanged, only `val` fields are modified.

The recursive calls move deeper into the tree while preserving mirror symmetry:

```python
left.left  <-> right.right
left.right <-> right.left
```

Finally, we begin recursion from the root’s children at level `1`, since the root itself belongs to level `0` and should not be reversed.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func reverseOddLevels(root *TreeNode) *TreeNode {
    var dfs func(left, right *TreeNode, level int)

    dfs = func(left, right *TreeNode, level int) {
        if left == nil || right == nil {
            return
        }

        if level%2 == 1 {
            left.Val, right.Val = right.Val, left.Val
        }

        dfs(left.Left, right.Right, level+1)
        dfs(left.Right, right.Left, level+1)
    }

    if root == nil {
        return root
    }

    dfs(root.Left, root.Right, 1)

    return root
}
```

The Go solution follows the exact same recursive logic as the Python version.

Instead of nested functions with closures, Go uses a function variable:

```go
var dfs func(...)
```

This allows recursive anonymous functions.

Go uses `nil` rather than `None`, and field access uses capitalized struct fields such as `Val`, `Left`, and `Right`.

Since node values are constrained to `10^5`, integer overflow is not a concern.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every node is visited exactly once |
| Space | O(h) | Recursion stack height equals tree height |

The DFS visits each node one time and performs only constant work per visit, making the time complexity linear.

The space complexity comes only from recursion depth. Since the tree is perfect, the height is:

```
h = O(log n)
```

Therefore, auxiliary space is effectively logarithmic.

# Helper functions for testing

class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def build_tree(values):
    if not values:
        return None

    nodes = [
        TreeNode(v) if v is not None else None
        for v in values
    ]

    for i in range(len(values)):
        if nodes[i]:
            left_idx = 2 * i + 1
            right_idx = 2 * i + 2

            if left_idx < len(values):
                nodes[i].left = nodes[left_idx]

            if right_idx < len(values):
                nodes[i].right = nodes[right_idx]

    return nodes[0]

def tree_to_list(root):
    result = []
    queue = [root]

    while queue:
        node = queue.pop(0)

        if node:
            result.append(node.val)
            queue.append(node.left)
            queue.append(node.right)

    return result

sol = Solution()

## Edge Cases

### Single Node Tree

A tree with only the root node has no odd levels. A naive implementation might attempt recursion on missing children and fail.

Our implementation handles this naturally because the recursive call begins at:

```
root.left, root.right
```

Both are `None`, so recursion exits immediately.

### Smallest Valid Odd Level

A tree like:

```
    1
   / \
  2   3
```

contains exactly one odd level.

This is important because off-by-one errors are common when counting levels. Since the root is level `0`, its children belong to level `1` and must be reversed. Our algorithm explicitly starts recursion at level `1`, preventing indexing mistakes.

### Deep Perfect Trees

A larger perfect tree may contain many odd levels.

A naive approach could unnecessarily reconstruct level arrays repeatedly. Our DFS solution avoids extra level storage and processes mirrored nodes directly, keeping memory usage efficient while still visiting every node exactly once.

### Repeated Values

If multiple nodes contain the same value, it may appear as though no reversal happened.

For example:

```
[1,5,5]
```

still requires a swap at level `1`.

Because our implementation swaps positions rather than checking values, correctness does not depend on values being distinct.
