## Solution

## Problem Understanding

This problem requires identifying customers whose total annual purchases increase strictly from the year of their first order to the year of their last order. The input is a table `Orders` containing `order_id`, `customer_id`, `order_date`, and `price`. Each row represents a single order made by a customer on a specific date.

The total purchases for a customer in a given year is the sum of all their order prices within that year. If the customer has no orders in a year between their first and last order, that year counts as zero. We want customers whose total purchases are strictly increasing year over year, meaning each year’s total is greater than the previous year.

Edge cases to note include customers who skip years, customers with the same purchase amount in consecutive years, and customers who only have a single year of orders. The problem guarantees that `order_id` is unique, but there are no constraints on the number of orders, so the solution should handle large datasets efficiently.

## Approaches

A brute-force approach would aggregate total purchases per customer for each year from their first to last order, then iterate through all years checking if the totals strictly increase. While correct, this approach involves building potentially large arrays for each customer and scanning them, which may be slow if there are many customers or years.

The optimal approach relies on SQL window functions and aggregation. We can first compute yearly totals for each customer. Then, by comparing each year’s total with the previous year using `LAG`, we can detect any non-increasing sequence. A customer qualifies only if all comparisons indicate a strictly increasing trend. This approach avoids unnecessary arrays and iterates just once per customer-year record.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(C * Y) | O(C * Y) | Compute totals for every year per customer, then check increasing order. May be slow if C or Y is large. |
| Optimal | O(N) | O(N) | Aggregate per customer-year, then use window functions to check strictly increasing totals. Scales well with large datasets. |

## Algorithm Walkthrough

1. **Compute the year of each order**: Extract the year from the `order_date` to group orders by customer and year.
2. **Aggregate yearly totals per customer**: Sum the `price` column for each customer and year to get their total purchases in that year.
3. **Compare consecutive years**: Use the SQL `LAG` function to retrieve the previous year’s total purchases for each customer.
4. **Identify non-increasing totals**: For each customer-year, check if the current year’s total is less than or equal to the previous year’s total. Mark these cases.
5. **Select qualifying customers**: Only include customers with no non-increasing totals from the first to last year.

This algorithm works because for a customer to have strictly increasing purchases, every consecutive year total must be greater than the previous. Aggregating per year and then using window functions ensures we check all necessary conditions without generating unnecessary intermediate structures.

## Python Solution

```python
import pandas as pd

def customers_with_increasing_purchases(orders: pd.DataFrame) -> pd.DataFrame:
    # Step 1: Extract year from order_date
    orders['year'] = orders['order_date'].dt.year
    
    # Step 2: Aggregate total purchases per customer per year
    yearly_totals = orders.groupby(['customer_id', 'year'], as_index=False)['price'].sum()
    
    # Step 3: Identify first and last year for each customer
    year_range = yearly_totals.groupby('customer_id').agg(first_year=('year', 'min'),
                                                         last_year=('year', 'max')).reset_index()
    
    # Step 4: Expand to include missing years with zero purchase
    all_years = []
    for _, row in year_range.iterrows():
        for y in range(row['first_year'], row['last_year'] + 1):
            all_years.append({'customer_id': row['customer_id'], 'year': y})
    all_years_df = pd.DataFrame(all_years)
    merged = all_years_df.merge(yearly_totals, on=['customer_id', 'year'], how='left').fillna(0)
    
    # Step 5: Check strictly increasing purchases using groupby and diff
    def is_strictly_increasing(df):
        return all(df['price'].diff().iloc[1:] > 0)
    
    result = merged.groupby('customer_id').filter(is_strictly_increasing)
    
    return result[['customer_id']].drop_duplicates().reset_index(drop=True)
```

This solution extracts the year, aggregates total purchases, fills missing years with zeros, and checks for strictly increasing totals using `diff` within each customer group. The `filter` function ensures only customers meeting the condition are included.

## Go Solution

```go
package main

import (
	"database/sql"
	"fmt"
)

func CustomersWithIncreasingPurchases(db *sql.DB) ([]int, error) {
	query := `
    WITH yearly_totals AS (
        SELECT customer_id, EXTRACT(YEAR FROM order_date) AS year, SUM(price) AS total
        FROM Orders
        GROUP BY customer_id, EXTRACT(YEAR FROM order_date)
    ),
    yearly_range AS (
        SELECT customer_id, MIN(year) AS first_year, MAX(year) AS last_year
        FROM yearly_totals
        GROUP BY customer_id
    ),
    customer_years AS (
        SELECT r.customer_id, y.year, COALESCE(t.total, 0) AS total
        FROM yearly_range r
        JOIN generate_series(r.first_year, r.last_year) AS y(year) ON true
        LEFT JOIN yearly_totals t
        ON t.customer_id = r.customer_id AND t.year = y.year
    ),
    diffs AS (
        SELECT customer_id, total - LAG(total) OVER(PARTITION BY customer_id ORDER BY year) AS diff
        FROM customer_years
    )
    SELECT DISTINCT customer_id
    FROM diffs
    WHERE diff <= 0;
    `
	rows, err := db.Query(query)
	if err != nil {
		return nil, err
	}
	defer rows.Close()

	var result []int
	for rows.Next() {
		var customerID int
		if err := rows.Scan(&customerID); err != nil {
			return nil, err
		}
		result = append(result, customerID)
	}
	return result, nil
}
```

In Go, we handle SQL operations directly against a database. The approach mirrors the Python logic, relying on CTEs and window functions. There is careful handling of `COALESCE` to replace nulls with zeros and `generate_series` to fill missing years.
