**Difficulty:** 🟡 Medium  
**Topics:** Array, Two Pointers, Heap (Priority Queue), Simulation

## Solution

## Problem Understanding

We are asked to simulate a hiring process for `k` workers from an array of `costs`. Each element of the array represents the cost to hire that worker. The twist is that in each hiring session we do not consider the entire array but only the first `candidates` workers or the last `candidates` workers, and we pick the worker with the smallest cost. If multiple workers have the same minimal cost, we choose the one with the smaller index. Once a worker is hired, they are removed from consideration. After exactly `k` rounds, we need to return the sum of all costs paid.

The constraints tell us that the number of workers can be up to `10^5` and each individual cost is up to `10^5`. The number of hiring rounds `k` can also be large, up to the size of the array. This means any algorithm that scans or sorts the array entirely for each hiring round would be too slow. We need something close to linearithmic or linear in `n` operations per session to meet the time limit. Edge cases arise when the first and last `candidates` windows overlap, for instance if `candidates` is more than half the array length, or if `k` equals the total number of workers, where all workers are eventually hired.

## Approaches

A brute-force approach would literally follow the hiring process as described. For each of the `k` rounds, we examine the first `candidates` workers and the last `candidates` workers that remain, compare their costs, select the minimal one, remove it from the array, and repeat. This is correct by definition but inefficient. In the worst case, examining `2*candidates` elements `k` times can reach `O(k*candidates)`, which could approach `O(n^2)` if `k` and `candidates` are near `n`. This is too slow for the maximum constraints.

The key insight to optimize is that we do not need to repeatedly scan the entire candidate windows. Instead, we can maintain two priority queues (min-heaps): one for the first `candidates` workers and one for the last `candidates` workers. At each round, we pop the smallest element from the two heaps. After hiring a worker, we refill the heap from the remaining unprocessed workers from the respective end of the array if any remain. This way, each worker is inserted and removed at most once from a heap, giving `O(log candidates)` per operation, and `O(k log candidates)` overall, which is efficient even for `n` up to `10^5`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(k * candidates) | O(n) | Too slow for k, candidates near n |
| Heap-based | O(k * log candidates) | O(candidates * 2) | Efficient and scalable |

## Algorithm Walkthrough

1. Initialize two min-heaps: one for the first `candidates` workers (`left_heap`) and one for the last `candidates` workers (`right_heap`). Also track the current pointers for the next unprocessed worker from each end (`left_ptr`, `right_ptr`).
2. Fill the `left_heap` with the first `candidates` workers and the `right_heap` with the last `candidates` workers. If the two windows overlap, ensure no worker is inserted twice by stopping at the mid-point.
3. Initialize `total_cost` to 0. This will accumulate the hiring costs.
4. For each of the `k` hiring rounds, compare the smallest elements in the two heaps. Pop the smallest cost worker. Add their cost to `total_cost`.
5. After hiring, if there are still unprocessed workers between the left and right pointers, push the next worker from the same side as the one just hired into the corresponding heap. Adjust `left_ptr` or `right_ptr` accordingly. Skip pushing if the heaps overlap or no unprocessed workers remain.
6. Repeat until `k` workers have been hired. Return `total_cost`.

Why it works: The invariant maintained is that each heap always contains the minimal cost workers from its end, up to the number of `candidates` allowed, and no worker appears in both heaps. By always picking the smallest among these, we correctly implement the rules given. The refill step guarantees that the heaps never miss any candidate that could be the minimal cost, so no selection is ever invalid.

## Python Solution

```python
import heapq
from typing import List

class Solution:
    def totalCost(self, costs: List[int], k: int, candidates: int) -> int:
        n = len(costs)
        total_cost = 0
        
        left_heap = []
        right_heap = []
        
        left_ptr = 0
        right_ptr = n - 1
        
        # Fill initial heaps
        for _ in range(candidates):
            if left_ptr > right_ptr:
                break
            heapq.heappush(left_heap, (costs[left_ptr], left_ptr))
            left_ptr += 1
        
        for _ in range(candidates):
            if right_ptr < left_ptr:
                break
            heapq.heappush(right_heap, (costs[right_ptr], right_ptr))
            right_ptr -= 1
        
        for _ in range(k):
            if left_heap and right_heap:
                if left_heap[0] <= right_heap[0]:
                    cost, idx = heapq.heappop(left_heap)
                    total_cost += cost
                    if left_ptr <= right_ptr:
                        heapq.heappush(left_heap, (costs[left_ptr], left_ptr))
                        left_ptr += 1
                else:
                    cost, idx = heapq.heappop(right_heap)
                    total_cost += cost
                    if left_ptr <= right_ptr:
                        heapq.heappush(right_heap, (costs[right_ptr], right_ptr))
                        right_ptr -= 1
            elif left_heap:
                cost, idx = heapq.heappop(left_heap)
                total_cost += cost
                if left_ptr <= right_ptr:
                    heapq.heappush(left_heap, (costs[left_ptr], left_ptr))
                    left_ptr += 1
            else:
                cost, idx = heapq.heappop(right_heap)
                total_cost += cost
                if left_ptr <= right_ptr:
                    heapq.heappush(right_heap, (costs[right_ptr], right_ptr))
                    right_ptr -= 1
        
        return total_cost
```

The implementation first sets up two heaps with the smallest `candidates` from each end, carefully avoiding overlap. In each hiring session, it pops the minimum from both heaps, updates the total, and refills the heap from the unprocessed range if possible. Boundary conditions for overlap are critical to avoid double counting.

## Go Solution

```go
package main

import (
	"container/heap"
)

type Item struct {
	cost int
	index int
}
type MinHeap []Item

func (h MinHeap) Len() int { return len(h) }
func (h MinHeap) Less(i, j int) bool {
	if h[i].cost == h[j].cost {
		return h[i].index < h[j].index
	}
	return h[i].cost < h[j].cost
}
func (h MinHeap) Swap(i, j int) { h[i], h[j] = h[j], h[i] }
func (h *MinHeap) Push(x any) { *h = append(*h, x.(Item)) }
func (h *MinHeap) Pop() any {
	old := *h
	n := len(old)
	item := old[n-1]
	*h = old[0 : n-1]
	return item
}

func totalCost(costs []int, k int, candidates int) int64 {
	n := len(costs)
	var total int64 = 0
	leftHeap := &MinHeap{}
	rightHeap := &MinHeap{}
	heap.Init(leftHeap)
	heap.Init(rightHeap)

	leftPtr, rightPtr := 0, n-1

	for i := 0; i < candidates && leftPtr <= rightPtr; i++ {
		heap.Push(leftHeap, Item{costs[leftPtr], leftPtr})
		leftPtr++
	}
	for i := 0; i < candidates && rightPtr >= leftPtr; i++ {
		heap.Push(rightHeap, Item{costs[rightPtr], rightPtr})
		rightPtr--
	}

	for hired := 0; hired < k; hired++ {
		var chosen Item
		if leftHeap.Len() > 0 && rightHeap.Len() > 0 {
			if (*leftHeap)[0].cost <= (*rightHeap)[0].cost {
				chosen = heap.Pop(leftHeap).(Item)
				if leftPtr <= rightPtr {
					heap.Push(leftHeap, Item{costs[leftPtr], leftPtr})
					leftPtr++
				}
			} else {
				chosen = heap.Pop(rightHeap).(Item)
				if leftPtr <= rightPtr {
					heap.Push(rightHeap, Item{costs[rightPtr], rightPtr})
					rightPtr--
				}
			}
		} else if leftHeap.Len() > 0 {
			chosen = heap.Pop(leftHeap).(Item)
			if leftPtr <= rightPtr {
				heap.Push(leftHeap, Item{costs[leftPtr], leftPtr})
				leftPtr++
			}
		} else {
			chosen = heap.Pop(rightHeap).(Item)
			if leftPtr <= rightPtr {
				heap.Push(rightHeap, Item{costs[rightPtr], rightPtr})
				rightPtr--
			}
		}
		total += int64(chosen.cost)
```
