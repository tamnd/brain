## Solution

## Problem Understanding

The problem requires simulating a robot that constructs a final string on paper from an initial input string `s` through two types of operations. The first operation allows the robot to take the first character of `s` and append it to an auxiliary string `t`. The second operation allows the robot to remove the last character from `t` and append it to the output string, which represents what is written on paper. The objective is to determine the sequence of operations that results in the lexicographically smallest string being written on the paper.

In other words, we are given an initial string `s` and a temporary storage `t`. We can either "push" characters from `s` to `t` or "pop" characters from `t` to the output. Our goal is to decide at each step whether to push or pop, so that the final output string is the smallest possible in lexicographical order.

The input `s` is a non-empty string of up to $10^5$ lowercase English letters. The constraints indicate that we must use an efficient algorithm because brute-force simulation of all possible operation sequences would be computationally infeasible. Edge cases include strings that are already sorted, strings where all characters are the same, or strings where the smallest character appears late in the string.

## Approaches

The brute-force approach is to recursively explore all sequences of push and pop operations. For every decision point, we can choose to move the first character of `s` to `t` or move the last character of `t` to the output. This method guarantees correctness because it evaluates all possible sequences and selects the lexicographically smallest result. However, it is far too slow for the given constraints because the number of possible sequences grows exponentially with the length of `s`.

The key observation that allows an optimal solution is that we can greedily decide whether to pop characters from `t` based on the smallest remaining character in `s`. Specifically, if the smallest character in `t` is less than or equal to the smallest character remaining in `s`, we can safely pop it to the output. Otherwise, we should continue pushing characters from `s` to `t`. To efficiently track the smallest remaining character in `s`, we can precompute a "suffix minimum" array that stores, for each index, the smallest character from that index to the end of `s`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Recursively explore all push/pop sequences, correct but infeasible |
| Optimal | O(n) | O(n) | Use stack for `t` and suffix minimum to decide greedy pops |

## Algorithm Walkthrough

1. Initialize an empty stack `t` to simulate the robot's auxiliary string.
2. Compute a suffix minimum array `min_suf` such that `min_suf[i]` stores the smallest character from `s[i:]`. This allows us to know the smallest remaining character in `s` at any step.
3. Initialize an empty string `output` to represent what the robot writes on paper.
4. Iterate over each character `c` in `s`. Push `c` onto the stack `t`.
5. After pushing a character, check if the top of the stack `t` is smaller than or equal to the smallest remaining character in `s`. If it is, repeatedly pop from `t` and append to `output` until this condition no longer holds or `t` is empty.
6. After processing all characters in `s`, pop any remaining characters from `t` to `output`.
7. Return the final `output`.

Why it works: The algorithm maintains the invariant that the output string is always built using the smallest possible character available either at the top of `t` or in the remaining `s`. By popping from `t` only when it is less than or equal to the smallest remaining character in `s`, we ensure that we never leave a smaller character in `t` that could have been placed earlier, thus guaranteeing the lexicographically smallest result.

## Python Solution

```python
class Solution:
    def robotWithString(self, s: str) -> str:
        n = len(s)
        t = []
        output = []

        # Precompute suffix minimum characters
        min_suf = [''] * n
        min_suf[-1] = s[-1]
        for i in range(n - 2, -1, -1):
            min_suf[i] = min(s[i], min_suf[i + 1])

        for i, c in enumerate(s):
            t.append(c)
            # Pop from t while top <= min remaining in s
            while t and (i == n - 1 or t[-1] <= min_suf[i + 1]):
                output.append(t.pop())

        return ''.join(output)
```

In the implementation, the suffix minimum array `min_suf` allows us to efficiently decide when to pop from `t`. For each character, we push it onto `t`, then check the top of `t` against the smallest remaining character in `s`. Popping continues as long as the top of `t` is smaller or equal, ensuring a greedy construction of the lexicographically smallest string.

## Go Solution

```go
func robotWithString(s string) string {
    n := len(s)
    t := []byte{}
    output := []byte{}

    // Precompute suffix minimum characters
    minSuf := make([]byte, n)
    minSuf[n-1] = s[n-1]
    for i := n - 2; i >= 0; i-- {
        if s[i] < minSuf[i+1] {
            minSuf[i] = s[i]
        } else {
            minSuf[i] = minSuf[i+1]
        }
    }

    for i := 0; i < n; i++ {
        t = append(t, s[i])
        for len(t) > 0 && (i == n-1 || t[len(t)-1] <= minSuf[i+1]) {
            output = append(output, t[len(t)-1])
            t = t[:len(t)-1]
        }
    }

    return string(output)
}
```

The Go version mirrors the Python logic but uses slices instead of lists. The suffix minimum is precomputed in a similar manner. Popping from `t` is done using slice manipulation, taking care to remove the last element correctly.

## Worked Examples

### Example 1: `s = "zza"`

| Step | s Remaining | t Stack | Output | Action |
| --- | --- | --- | --- | --- |
| 1 | "za" | ['z'] | "" | Push 'z' |
| 2 | "a" | ['z','z'] | "" | Push 'z' |
| 3 | "" | ['z','z','a'] | "" | Push 'a' |
| 4 | "" | ['z','z'] | "a" | Pop 'a' (top <= min remaining in s) |
| 5 | "" | ['z'] | "az" | Pop 'z' |
| 6 | "" | [] | "azz" | Pop 'z' |

Output is `"azz"`.

### Example 2: `s = "bac"`

| Step | s Remaining | t Stack | Output | Action |
| --- | --- | --- | --- | --- |
| 1 | "ac" | ['b'] | "" | Push 'b' |
| 2 | "c" | ['b','a'] | "" | Push 'a' |
| 3 | "" | ['b','a','c'] | "" | Push 'c' |
| 4 | "" | ['b','a'] | "c" | Pop 'c' |
| 5 | "" | ['b'] | "ca" | Pop 'a' |
| 6 | "" | [] | "cab" | Pop 'b' |

Output is `"abc"`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is pushed and popped at most once, resulting in linear time. |
| Space | O(n) | Stack `t` and suffix minimum array use linear space. |

The algorithm runs efficiently for the constraint $1 \le s.length \le 10^5$ because each operation is linear with respect to the string length. No recursion or backtracking is needed, avoiding exponential complexity.
