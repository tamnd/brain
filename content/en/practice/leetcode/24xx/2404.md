## Solution

## Problem Understanding

This problem asks us to find the even number that appears most frequently in a given list of integers. The input `nums` is an array of integers where each integer is non-negative and up to 100,000. We are required to return the most frequent even element. If multiple even numbers share the highest frequency, we must return the **smallest** of them. If the array contains no even numbers, we return `-1`.

The key points to note are that the array length is relatively small (up to 2000), so a solution that scans the array multiple times is feasible. The integers can be large, so solutions that rely on indexing an array by value may not be memory-efficient. Edge cases include arrays with all odd numbers, arrays with only one number, arrays where multiple even numbers tie in frequency, and arrays where the smallest even number appears less often than other higher-valued even numbers.

## Approaches

The brute-force approach is to iterate through all even numbers, count their occurrences, and track the one with the highest frequency. This works because we are guaranteed that the input size is modest, but it requires repeatedly scanning the array for each even number, which is inefficient.

The optimal approach is to use a hash map to count the frequency of each even number in a single pass. After building the frequency map, we scan the map to find the even number with the highest count. If there is a tie, the smallest number is chosen. This method ensures we only iterate through the array once and then iterate over the distinct even numbers, making it fast and memory-efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Count frequency of each even number by scanning the array multiple times |
| Optimal | O(n) | O(n) | Use a hash map to count occurrences, then find max frequency and smallest number |

## Algorithm Walkthrough

1. Initialize an empty hash map (or dictionary) to store frequencies of even numbers.
2. Iterate over each number in the array.
3. For each number, check if it is even. If it is, increment its count in the hash map.
4. After processing all numbers, initialize variables to track the most frequent even number and its frequency.
5. Iterate over the hash map:

- If the frequency of the current number is greater than the maximum frequency seen so far, update the most frequent number and frequency.
- If the frequency equals the maximum frequency, choose the smaller number.
6. If no even number was found (maximum frequency is still zero or the hash map is empty), return `-1`.
7. Otherwise, return the most frequent even number.

**Why it works:** The hash map ensures that each even number’s frequency is counted exactly once. By scanning the map afterward, we correctly identify the number with the highest frequency. The tie-breaker logic ensures the smallest number is returned in case of a frequency tie.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def mostFrequentEven(self, nums: List[int]) -> int:
        freq = defaultdict(int)
        for num in nums:
            if num % 2 == 0:
                freq[num] += 1
        
        max_count = 0
        result = -1
        
        for num in freq:
            if freq[num] > max_count or (freq[num] == max_count and num < result):
                max_count = freq[num]
                result = num
        
        return result
```

This implementation first counts the frequency of all even numbers using a `defaultdict`. Then it iterates over the dictionary to select the most frequent even number, using the tie-breaker logic for the smallest number. The use of `defaultdict` simplifies frequency counting because it automatically initializes missing keys to zero.

## Go Solution

```go
func mostFrequentEven(nums []int) int {
    freq := make(map[int]int)
    for _, num := range nums {
        if num % 2 == 0 {
            freq[num]++
        }
    }
    
    maxCount := 0
    result := -1
    
    for num, count := range freq {
        if count > maxCount || (count == maxCount && (result == -1 || num < result)) {
            maxCount = count
            result = num
        }
    }
    
    return result
}
```

In Go, we use a map to store frequencies, similar to Python. The main differences include explicit initialization and iteration using `range`. We also handle the initial case when `result` is `-1` to ensure the tie-breaker logic works.

## Edge Cases

The first edge case is an array with no even numbers. This is critical because returning `-1` requires detecting that no key in the frequency map exists. The algorithm handles this by initializing `result` to `-1` and only updating it for even numbers.

The second edge case involves a tie between multiple even numbers with the same frequency. A naive solution might return the last maximum instead of the smallest. The implementation explicitly checks `num < result` to correctly handle ties.

The third edge case is an array containing very large even numbers, such as `100000`. Since the solution uses a map rather than an array indexed by value, it avoids excessive memory usage while correctly counting frequencies.

Other minor edge cases include arrays of length one, arrays containing only zeros, and arrays where the smallest even number appears less frequently than a larger one. The solution handles all these through consistent frequency counting and tie-breaking logic.
