## Solution

## Problem Understanding

The problem requires finding the largest positive integer `k` in an array `nums` such that its negative counterpart `-k` also exists in the same array. The input array is guaranteed to have no zeros and can contain both positive and negative integers. The output is a single integer representing this largest `k`, or `-1` if no such number exists.

In simpler terms, we are asked to identify a pair of numbers of the form `(k, -k)` within the array and return the largest value of `k` among all such pairs. The array size constraint (`1 <= nums.length <= 1000`) indicates that we can consider solutions with quadratic time, but a more efficient solution is preferred. The constraints also specify that `nums[i] != 0`, simplifying the problem since we do not need to handle the zero edge case.

Important edge cases include arrays where there are no valid positive-negative pairs, arrays where multiple pairs exist and we must pick the largest positive value, and arrays where all numbers are either entirely positive or entirely negative.

## Approaches

The brute-force approach involves iterating over each positive number in the array and checking if its negative exists. While this works, it has a time complexity of O(n²) because for each number we may search the entire array. This is unnecessary given the input constraints and can be optimized.

A more efficient approach is to use a hash set to store all numbers in the array. This allows O(1) average-time lookups. We can then iterate over the array and, for each positive number, check if its negative exists in the set. This guarantees a linear scan with constant-time checks, reducing time complexity from O(n²) to O(n) while using O(n) extra space for the set.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Check each number against all others to find negative counterpart |
| Optimal | O(n) | O(n) | Use hash set to store elements and check for negative counterparts efficiently |

## Algorithm Walkthrough

1. Initialize an empty hash set called `seen` to store all numbers from the array.
2. Populate the `seen` set with all elements from `nums` to allow O(1) lookups.
3. Initialize a variable `max_k` to `-1` to store the largest valid `k` found.
4. Iterate through each number `num` in `nums`. For each `num`, check if it is positive and whether its negative `-num` exists in `seen`.
5. If both conditions are satisfied, update `max_k` to be the maximum of its current value and `num`.
6. After processing all numbers, return `max_k`. If no valid pair exists, `max_k` remains `-1`.

Why it works: By storing all numbers in a hash set, we ensure that every lookup for a negative counterpart is constant time. Iterating through all numbers ensures we examine all potential pairs. Updating the maximum during iteration guarantees we always retain the largest valid positive integer.

## Python Solution

```python
from typing import List

class Solution:
    def findMaxK(self, nums: List[int]) -> int:
        seen = set(nums)
        max_k = -1
        for num in nums:
            if num > 0 and -num in seen:
                max_k = max(max_k, num)
        return max_k
```

In this implementation, we first convert the list into a set for efficient lookups. The loop only considers positive numbers since only positive values are candidates for `k`. The `max` function keeps track of the largest number that satisfies the condition. Finally, we return `max_k`, which defaults to `-1` if no valid pairs exist.

## Go Solution

```go
func findMaxK(nums []int) int {
    seen := make(map[int]struct{})
    for _, num := range nums {
        seen[num] = struct{}{}
    }

    maxK := -1
    for _, num := range nums {
        if num > 0 {
            if _, exists := seen[-num]; exists {
                if num > maxK {
                    maxK = num
                }
            }
        }
    }
    return maxK
}
```

The Go implementation mirrors the Python logic. We use a map with empty structs to store elements for efficient lookups. The loop iterates through positive numbers and updates the maximum if the negative exists in the map. Go does not have a built-in set type, so a map is the standard alternative.
