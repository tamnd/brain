# LeetCode 2496 - Maximum Value of a String in an Array

## Problem Understanding

The problem gives us an array of alphanumeric strings called `strs`. Each string may contain lowercase English letters, digits, or both.

We must compute the "value" of every string according to the following rules:

- If the string contains only digits, its value is the integer represented by that string in base 10.
- Otherwise, its value is simply the length of the string.

Our goal is to return the maximum value among all strings in the array.

For example, the string `"123"` contains only digits, so its value is the integer `123`. However, the string `"ab12"` contains letters, so its value is its length, which is `4`.

The important distinction is that a string containing even a single letter is treated as a normal string whose value equals its length. Only purely numeric strings are converted to integers.

The constraints are small:

- At most `100` strings
- Each string has length at most `9`

These constraints tell us that efficiency is not a major concern. Even relatively straightforward solutions will run comfortably within limits. However, we still want a clean and correct implementation.

Several edge cases are important:

- Strings with leading zeros such as `"0001"` should convert to integer `1`.
- Strings containing both digits and letters such as `"a1"` should use length, not numeric conversion.
- A string like `"00000"` contains only digits, so its value is numeric `0`, not length `5`.
- Single character strings must also be handled correctly.
- The array always contains at least one string, so the maximum value always exists.

## Approaches

### Brute Force Approach

A brute force solution examines every string individually and determines whether it contains only digits.

For each string:

1. Check every character.
2. If all characters are digits, convert the string to an integer.
3. Otherwise, compute its length.
4. Track the maximum value seen so far.

This approach is guaranteed to work because the problem definition is applied independently to every string. After computing all values correctly, the largest one is the answer.

Although this is already efficient enough for the given constraints, we can still slightly simplify the logic using built-in language helpers such as `isdigit()` in Python.

### Optimal Approach

The key observation is that each string can be processed independently in a single pass.

We do not need sorting, additional data structures, or dynamic programming. The task reduces to:

- Determine whether a string is numeric-only.
- Compute its corresponding value.
- Maintain a running maximum.

Since each character is inspected at most once, the solution runs in linear time relative to the total number of characters.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × m) | O(1) | Manually checks every character in every string |
| Optimal | O(n × m) | O(1) | Uses direct digit checking and maintains running maximum |

Here:

- `n` is the number of strings
- `m` is the maximum string length

Because the strings are very short, both approaches are practical, but the optimal version is cleaner and easier to read.

## Algorithm Walkthrough

1. Initialize a variable `maximum_value` to `0`.

This variable stores the largest value encountered so far while processing the array.
2. Iterate through every string in `strs`.

We process one string at a time because the value of each string is independent of the others.
3. Determine whether the current string contains only digits.

In Python, we can use `s.isdigit()`.

In Go, we can iterate through characters and verify each one is between `'0'` and `'9'`.
4. If the string is numeric-only, convert it to an integer.

For example:

- `"3"` becomes `3`
- `"0001"` becomes `1`
- `"00000"` becomes `0`
5. Otherwise, compute the length of the string.

Mixed strings such as `"a1"` or alphabetic strings such as `"bob"` use length as their value.
6. Update the running maximum.

Compare the current value with `maximum_value` and keep the larger one.
7. After processing all strings, return `maximum_value`.

### Why it works

The algorithm works because the problem defines the value of each string independently. For every string, we exactly follow the rules:

- Numeric-only strings become integers.
- All other strings use their length.

Since we compute every value correctly and continuously track the largest one, the final result must be the maximum value in the array.

## Python Solution

```python
from typing import List

class Solution:
    def maximumValue(self, strs: List[str]) -> int:
        maximum_value = 0

        for current_string in strs:
            if current_string.isdigit():
                current_value = int(current_string)
            else:
                current_value = len(current_string)

            maximum_value = max(maximum_value, current_value)

        return maximum_value
```

The implementation starts by initializing `maximum_value` to `0`.

We then iterate through every string in the array. The method `isdigit()` checks whether every character in the string is a digit. If it returns `True`, we convert the string into an integer using `int()`.

If the string contains letters, we instead use `len(current_string)`.

After computing the current string's value, we compare it with the current maximum and update the answer if necessary.

Finally, once all strings are processed, we return the maximum value found.

## Go Solution

```go
package main

import (
	"strconv"
)

func maximumValue(strs []string) int {
	maximumValue := 0

	for _, s := range strs {
		isNumeric := true

		for _, ch := range s {
			if ch < '0' || ch > '9' {
				isNumeric = false
				break
			}
		}

		currentValue := 0

		if isNumeric {
			value, _ := strconv.Atoi(s)
			currentValue = value
		} else {
			currentValue = len(s)
		}

		if currentValue > maximumValue {
			maximumValue = currentValue
		}
	}

	return maximumValue
}
```

The Go implementation follows the same logic as the Python solution.

Since Go does not provide a direct equivalent of Python's `isdigit()` for strings, we manually iterate through each character and verify that every character lies between `'0'` and `'9'`.

If the string is numeric-only, we convert it using `strconv.Atoi`. Otherwise, we use `len(s)`.

The problem constraints guarantee valid inputs, so the conversion error from `Atoi` can safely be ignored.

## Edge Cases

### Strings With Leading Zeros

A string such as `"0001"` can easily cause mistakes if treated as a normal string length instead of a number. The correct numeric value is `1`, not `4`. Our implementation handles this correctly because numeric-only strings are converted using integer parsing.

### Strings Containing Both Letters and Digits

A string like `"a1"` may appear partially numeric, but the rules state that any string containing letters must use its length. The implementation correctly identifies that `"a1"` is not purely numeric and therefore uses `len("a1")`.

### Numeric Strings Representing Zero

The string `"00000"` is entirely numeric, so its value is the integer `0`. A common bug would be returning length `5` instead. Since the algorithm converts numeric-only strings directly to integers, it correctly evaluates this case as `0`.

### Single Element Arrays

The array may contain only one string. In that situation, the answer is simply the value of that string. Because the algorithm initializes a running maximum and processes every element exactly once, it handles single-element arrays naturally and correctly.
