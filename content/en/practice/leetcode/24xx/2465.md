## Solution

## Problem Understanding

The problem asks us to repeatedly remove the smallest and largest elements from a given array of even length, calculate the average of those two elements, and track how many **distinct averages** are produced throughout the process. The input array `nums` contains integers between 0 and 100, and the length of the array is guaranteed to be even, which ensures that we can always pair up the minimum and maximum elements without leftovers. The output is a single integer representing the number of unique averages obtained after emptying the array.

The constraints are important: the array length is at most 100, so algorithms with quadratic complexity may still work, but simpler linear or linearithmic solutions are preferred. The input guarantee of even length and integer values simplifies handling edge cases: we do not need to consider fractional indexing or empty removals until the array is fully processed.

Important edge cases include arrays where all numbers are identical, arrays already sorted in ascending or descending order, and arrays where extreme values dominate multiple occurrences. Our solution must correctly handle repeated values without double-counting averages.

## Approaches

The brute-force approach is straightforward: repeatedly scan the array to find the minimum and maximum values, remove them, compute the average, and insert it into a set to track distinct values. This guarantees correctness because it follows the problem definition exactly. However, finding the minimum and maximum repeatedly is `O(n)` per removal, and we perform `n/2` removals for an array of size `n`, resulting in `O(n^2)` time complexity. This is acceptable for `n <= 100` but inefficient.

The optimal approach relies on **sorting the array once** and then using a two-pointer technique. By sorting, the smallest and largest elements are at the two ends of the array. We can maintain two pointers, `left` starting at index 0 and `right` starting at the last index, and move them inward as we compute averages. This approach avoids repeated scanning and reduces the time complexity to `O(n log n)` due to the sorting step, with the two-pointer traversal itself being linear.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Repeatedly find min and max in unsorted array, store averages in a set |
| Optimal | O(n log n) | O(n) | Sort once and use two pointers to compute distinct averages |

## Algorithm Walkthrough

1. **Sort the array** `nums` in ascending order. Sorting ensures that the smallest and largest elements are at the boundaries of the array, simplifying the selection process.
2. **Initialize two pointers**, `left` at 0 and `right` at `len(nums) - 1`. These will track the current smallest and largest elements available.
3. **Initialize a set** `averages` to store the distinct averages. Using a set guarantees uniqueness automatically.
4. **Iterate while `left < right`**. In each iteration:

- Compute the average of `nums[left]` and `nums[right]`.
- Add the average to the set `averages`.
- Increment `left` and decrement `right` to move to the next smallest and largest numbers.
5. **Return the size of the set** `averages`. This count represents the number of distinct averages computed.

**Why it works:** Sorting preserves the relative ordering of numbers. By pairing the smallest and largest iteratively, we replicate the removal process described in the problem. Using a set ensures that only unique averages are counted. Since the array length is even, all elements are used exactly once, so all possible averages are considered.

## Python Solution

```python
from typing import List

class Solution:
    def distinctAverages(self, nums: List[int]) -> int:
        nums.sort()
        left, right = 0, len(nums) - 1
        averages = set()
        
        while left < right:
            avg = (nums[left] + nums[right]) / 2
            averages.add(avg)
            left += 1
            right -= 1
        
        return len(averages)
```

In this implementation, the array is first sorted. Two pointers, `left` and `right`, are initialized to the start and end of the array. In a loop, we calculate the average of the elements at the pointers, store it in a set, and move the pointers inward. Finally, the number of distinct averages is returned.

## Go Solution

```go
import "sort"

func distinctAverages(nums []int) int {
    sort.Ints(nums)
    left, right := 0, len(nums)-1
    averages := make(map[float64]struct{})
    
    for left < right {
        avg := float64(nums[left]+nums[right]) / 2
        averages[avg] = struct{}{}
        left++
        right--
    }
    
    return len(averages)
}
```

In Go, we sort the array using `sort.Ints`. A map with `float64` keys is used instead of a set, storing empty structs for memory efficiency. The two-pointer iteration is identical to Python, and the map length represents the number of distinct averages.

## Edge Cases

One important edge case is when all numbers are identical. In this scenario, every average calculated will be the same, so the result should always be 1. Our set-based approach automatically handles this without any additional checks.

Another edge case occurs when the array has two elements. This is the minimal valid input. The algorithm correctly computes the single average and returns 1, as intended.

A third edge case involves arrays with multiple duplicates of minimum and maximum values. For example, `[0,0,100,100]`. The algorithm pairs the smallest and largest values in order, but the set ensures duplicates do not inflate the count, correctly producing only one distinct average.
