**Topics:** Array, Queue, Sliding Window, Monotonic Queue

## Solution

## Problem Understanding

The problem asks us to count how many contiguous subarrays satisfy a very specific condition: the minimum value in the subarray must be exactly `minK`, and the maximum value in the subarray must be exactly `maxK`.

In other words, we are not looking for subarrays whose values merely fall between `minK` and `maxK`. Instead, a valid subarray must satisfy both of these requirements simultaneously:

- The smallest value present in the subarray must equal `minK`
- The largest value present in the subarray must equal `maxK`

This means that every valid subarray must contain at least one occurrence of `minK` and at least one occurrence of `maxK`. Additionally, no number in the subarray can fall outside the inclusive range `[minK, maxK]`, because any smaller value would reduce the minimum below `minK`, and any larger value would increase the maximum above `maxK`.

The input consists of:

- `nums`, an integer array
- `minK`, the required minimum value
- `maxK`, the required maximum value

The output is a single integer representing the total number of contiguous subarrays that satisfy the fixed-bound condition.

The constraints provide important clues about the expected solution:

- `nums.length` can be as large as `10^5`
- Values can be as large as `10^6`

A quadratic solution such as checking every subarray explicitly would be too slow because `O(n²)` on `10^5` elements would result in roughly `10^10` operations in the worst case. Therefore, we need a near linear-time approach, ideally `O(n)`.

Several edge cases immediately stand out:

When `minK == maxK`, every valid subarray must contain only that exact value. This changes the interpretation slightly because the minimum and maximum are identical.

Arrays containing invalid values, meaning values smaller than `minK` or larger than `maxK`, break possible subarrays. No valid subarray can cross such elements.

If `minK` or `maxK` never appears, then the answer must be `0` because no subarray can satisfy the fixed-bound requirement.

Repeated values matter because multiple valid subarrays may end at the same position.

## Approaches

### Brute Force Approach

A straightforward solution is to examine every possible subarray.

For each starting index `i`, we expand the ending index `j` and continuously track:

- The minimum value seen so far
- The maximum value seen so far

Whenever the current minimum equals `minK` and the current maximum equals `maxK`, we increment the answer.

This approach is correct because it explicitly evaluates every possible subarray and checks whether it satisfies the required conditions.

However, it is far too slow for the input constraints. Since there are `O(n²)` possible subarrays and each extension may require updating min/max values, the runtime becomes quadratic.

### Key Insight for an Optimal Solution

Instead of checking every subarray explicitly, we can determine how many valid subarrays end at each index.

For a subarray ending at index `i` to be valid:

1. It must contain at least one `minK`
2. It must contain at least one `maxK`
3. It cannot contain any invalid value outside `[minK, maxK]`

This suggests tracking three important indices while scanning from left to right:

- `last_min`, the most recent position of `minK`
- `last_max`, the most recent position of `maxK`
- `last_invalid`, the most recent position containing an invalid number

At each index, the earliest valid starting point depends on the smaller of `last_min` and `last_max`, because both values must appear.

If both required values have appeared after the most recent invalid element, then the number of valid subarrays ending at index `i` is:

```
min(last_min, last_max) - last_invalid
```

This transforms the problem into a single linear scan.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Check every subarray and track min/max |
| Optimal Sliding Window | O(n) | O(1) | Track last valid positions of key elements |

## Algorithm Walkthrough

1. Initialize tracking variables.

We maintain:

- `last_min = -1`, latest position of `minK`
- `last_max = -1`, latest position of `maxK`
- `last_invalid = -1`, latest invalid position
- `result = 0`, total valid subarrays

We use `-1` initially because no relevant positions have been seen.
2. Iterate through the array from left to right.

For each index `i`, inspect `nums[i]`.
3. Detect invalid elements.

If `nums[i] < minK` or `nums[i] > maxK`, this element invalidates any subarray crossing it.

Therefore, update:

```
last_invalid = i
```
4. Track occurrences of `minK`.

If the current number equals `minK`, update:

```
last_min = i
```
5. Track occurrences of `maxK`.

If the current number equals `maxK`, update:

```
last_max = i
```
6. Compute valid subarrays ending at the current index.

A valid subarray must include both `minK` and `maxK`.

The earliest guaranteed ending point where both exist is:

```
min(last_min, last_max)
```

Any starting position after `last_invalid` and before or equal to this value produces a valid subarray.

Therefore:

```
valid_count =
min(last_min, last_max) - last_invalid
```

If this becomes negative, treat it as zero:

```
result += max(0, valid_count)
```
7. Continue until the end of the array.
8. Return the accumulated result.

### Why it works

The key invariant is that `last_invalid` marks the latest position that no valid subarray may cross, while `last_min` and `last_max` record the most recent required values. For any ending index `i`, the earliest position that guarantees both bounds exist is `min(last_min, last_max)`. Every starting point between `last_invalid + 1` and that index forms a valid subarray. Since we count all valid endings exactly once, the algorithm is correct.

## Python Solution

```python
from typing import List

class Solution:
    def countSubarrays(self, nums: List[int], minK: int, maxK: int) -> int:
        last_min = -1
        last_max = -1
        last_invalid = -1

        total_subarrays = 0

        for index, value in enumerate(nums):
            if value < minK or value > maxK:
                last_invalid = index

            if value == minK:
                last_min = index

            if value == maxK:
                last_max = index

            valid_end = min(last_min, last_max)
            total_subarrays += max(0, valid_end - last_invalid)

        return total_subarrays
```

The implementation follows the algorithm directly.

We initialize three index trackers to `-1`. These variables remember the most recent occurrences of `minK`, `maxK`, and invalid values.

During the iteration, we first check whether the current element falls outside the valid range `[minK, maxK]`. If so, we update `last_invalid`, because no valid subarray may include that position.

Next, we check whether the element matches `minK` or `maxK`. If either condition holds, we update the corresponding position tracker.

The critical computation happens at every index. We determine the earliest index that guarantees both required values exist using:

```python
valid_end = min(last_min, last_max)
```

If this index lies after the latest invalid position, then every starting point between them produces a valid subarray ending at the current index.

The expression:

```python
max(0, valid_end - last_invalid)
```

ensures that negative contributions are ignored when one of the required values has not yet appeared.

## Go Solution

```go
func countSubarrays(nums []int, minK int, maxK int) int64 {
	lastMin := -1
	lastMax := -1
	lastInvalid := -1

	var total int64 = 0

	for i, value := range nums {
		if value < minK || value > maxK {
			lastInvalid = i
		}

		if value == minK {
			lastMin = i
		}

		if value == maxK {
			lastMax = i
		}

		validEnd := lastMin
		if lastMax < validEnd {
			validEnd = lastMax
		}

		if validEnd > lastInvalid {
			total += int64(validEnd - lastInvalid)
		}
	}

	return total
}
```

The Go implementation mirrors the Python logic closely.

One important difference is integer overflow handling. Since the number of subarrays can be very large, the function returns `int64`, which matches the required LeetCode signature.

Instead of using Python's built-in `min()` function, we compute the smaller value manually:

```go
validEnd := lastMin
if lastMax < validEnd {
    validEnd = lastMax
}
```

We also avoid negative additions by checking:

```go
if validEnd > lastInvalid
```

before updating the total.

## Edge Cases

### Case 1: `minK == maxK`

When the required minimum and maximum are equal, valid subarrays must contain only that exact value. A buggy implementation might incorrectly treat minimum and maximum tracking separately and overcount or undercount.

For example:

```
nums = [1,1,1]
minK = 1
maxK = 1
```

Every subarray is valid. Our implementation handles this naturally because `last_min` and `last_max` update together, and the formula still works correctly.

### Case 2: Invalid Numbers Split the Array

Values outside `[minK, maxK]` immediately invalidate any subarray crossing them.

For example:

```
nums = [1,5,8,1,5]
```

The `8` breaks the array into two independent regions. A naive implementation may accidentally count subarrays crossing the invalid value.

Our algorithm avoids this by updating:

```
last_invalid
```

which acts as a barrier that no valid subarray may cross.

### Case 3: One Required Value Never Appears

If `minK` or `maxK` never appears, the answer must be `0`.

For example:

```
nums = [5,5,5]
minK = 1
maxK = 5
```

Since `1` never appears, no valid subarray exists.

Our implementation handles this automatically because `last_min` remains `-1`, causing:

```
min(last_min, last_max) - last_invalid
```

to stay non-positive, meaning nothing is added to the result.

### Case 4: Multiple Overlapping Valid Subarrays

Many valid subarrays can end at the same index.

For example:

```
nums = [1,5,1,5]
```

Several different starting points produce valid endings. A naive sliding window may count only one subarray per ending position.

Our formula counts all possible valid starting positions at once:

```
min(last_min, last_max) - last_invalid
```

which guarantees every valid subarray is counted exactly once.
