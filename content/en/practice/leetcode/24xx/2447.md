## Solution

## Problem Understanding

The problem requires counting the number of **contiguous subarrays** in an integer array `nums` such that the **greatest common divisor (GCD)** of the subarray elements is equal to a given integer `k`. A subarray is any sequence of consecutive elements in the array. The GCD of a subarray is the largest integer that evenly divides every element in the subarray.

The input consists of an array `nums` of length up to 1000, where each element and `k` are integers up to $10^9$. The output is a single integer, representing the count of subarrays whose GCD is exactly `k`.

The constraints imply that a brute-force solution that checks every subarray for its GCD is feasible for small arrays but may be slow for arrays of length 1000 due to the quadratic number of subarrays. The problem guarantees positive integers, so we do not need to handle zero or negative values.

Important edge cases include arrays where `k` does not divide any element, arrays where all elements are equal to `k`, and arrays with a mix of multiples and non-multiples of `k`.

## Approaches

The **brute-force approach** iterates over all possible subarrays, computes the GCD for each subarray, and counts the ones equal to `k`. This approach works because it directly follows the problem definition: every contiguous subsequence is checked. However, it is inefficient since there are $O(n^2)$ subarrays, and computing the GCD for each subarray can take up to $O(\log(\text{max element}))$, leading to a total complexity of $O(n^3)$ in the naive implementation.

The **optimal approach** leverages two observations. First, any subarray that can have GCD `k` must contain elements divisible by `k`. Second, the GCD operation is associative and monotone: extending a subarray to the right by including a new element allows us to update the GCD incrementally. Therefore, we can iterate through the array, and for each start index, we expand the subarray while updating its GCD using the Euclidean algorithm. If the GCD becomes `k`, we count it, and if it drops below `k` or becomes 1, we can stop extending that subarray. This reduces unnecessary computation while still correctly counting all valid subarrays.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(1) | Checks every subarray and computes GCD from scratch |
| Optimal | O(n^2 log(max(nums))) | O(1) | Incrementally computes GCD while iterating over subarrays |

## Algorithm Walkthrough

1. Initialize a counter `count` to 0. This will store the number of valid subarrays.
2. Loop through each index `i` of `nums` as the starting index of a subarray.
3. Initialize `current_gcd` to 0, representing the GCD of the current subarray.
4. Loop through each index `j` starting from `i` to the end of `nums` to form subarrays starting at `i`.
5. Update `current_gcd` with the GCD of `current_gcd` and `nums[j]`. Use the Euclidean algorithm.
6. If `current_gcd` equals `k`, increment `count` by 1.
7. If `current_gcd` becomes less than `k`, break the inner loop early because extending the subarray further cannot produce GCD `k`.
8. Continue until all starting indices are processed.
9. Return `count` as the final answer.

**Why it works:** By incrementally computing the GCD for each subarray starting at `i`, we exploit the property that the GCD of a longer subarray cannot increase. Therefore, once the GCD drops below `k`, it cannot later equal `k`, making early termination safe. This guarantees correctness and efficiency.

## Python Solution

```python
from math import gcd
from typing import List

class Solution:
    def subarrayGCD(self, nums: List[int], k: int) -> int:
        count = 0
        n = len(nums)
        
        for i in range(n):
            current_gcd = 0
            for j in range(i, n):
                current_gcd = gcd(current_gcd, nums[j])
                if current_gcd == k:
                    count += 1
                elif current_gcd < k:
                    break
        return count
```

In this implementation, the outer loop sets the starting index of the subarray, while the inner loop extends the subarray to the right. `current_gcd` is updated incrementally using `math.gcd`. If the current GCD equals `k`, we count it. If it drops below `k`, the subarray cannot yield GCD `k` in further extensions, so we break early. This directly implements the algorithm steps described above.

## Go Solution

```go
package main

func gcd(a, b int) int {
    for b != 0 {
        a, b = b, a%b
    }
    return a
}

func subarrayGCD(nums []int, k int) int {
    count := 0
    n := len(nums)
    
    for i := 0; i < n; i++ {
        currentGCD := 0
        for j := i; j < n; j++ {
            currentGCD = gcd(currentGCD, nums[j])
            if currentGCD == k {
                count++
            } else if currentGCD < k {
                break
            }
        }
    }
    return count
}
```

In Go, we define a helper function `gcd` using the iterative Euclidean algorithm. We use standard slices and integer variables. Unlike Python, Go does not require importing a library for GCD, so the function is implemented manually. Otherwise, the logic mirrors the Python solution.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n^2 log(max(nums))) | For each subarray starting index, we compute GCD incrementally which is O(log(max(nums))), leading to O(n^2 log(max(nums))) overall. |
| Space | O(1) | Only constant extra space is used for counters and GCD computation. |

The complexity arises from checking all subarrays and performing GCD calculations incrementally. The early break reduces unnecessary iterations but does not affect the worst-case quadratic bound.
