# LeetCode 2413 - Smallest Even Multiple

## Problem Understanding

The problem gives us a positive integer `n` and asks for the smallest positive integer that is divisible by both `2` and `n`.

In other words, we need to find the least common multiple, often abbreviated as LCM, of the numbers `2` and `n`.

A number is considered a multiple of another number if it can be divided evenly without leaving a remainder. For example:

- `10` is a multiple of both `2` and `5`
- `6` is a multiple of both `2` and `6`

The input consists of a single integer:

- `n`, where `1 <= n <= 150`

The output should be:

- The smallest positive integer divisible by both `2` and `n`

Since the constraint is very small, performance is not a concern. However, the problem is designed to test understanding of divisibility and least common multiples.

There are a few important observations and edge cases:

- If `n` is already even, then `n` itself is divisible by `2`, so the answer is simply `n`
- If `n` is odd, then multiplying it by `2` produces the smallest even number divisible by both `2` and `n`
- The smallest possible input is `1`
- The largest possible input is `150`
- Because the numbers are tiny, integer overflow is not an issue in either Python or Go

## Approaches

### Brute Force Approach

A straightforward solution is to start from `1` and repeatedly check each integer until we find one divisible by both `2` and `n`.

For every number `current`, we would test:

- `current % 2 == 0`
- `current % n == 0`

The first number satisfying both conditions is the answer.

This approach is correct because it checks numbers in increasing order, guaranteeing that the first valid number is the smallest common multiple.

However, this method is inefficient compared to what is possible mathematically. Even though the constraints are tiny, repeatedly testing numbers is unnecessary.

### Optimal Approach

The key insight comes from understanding parity, whether a number is odd or even.

If `n` is even:

- `n` is already divisible by `2`
- Therefore, `n` itself is the smallest number divisible by both `2` and `n`

If `n` is odd:

- An odd number cannot be divisible by `2`
- Multiplying it by `2` creates the smallest even multiple of `n`

So the logic becomes:

- Return `n` if `n` is even
- Otherwise return `2 * n`

This solution works in constant time and uses no extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Checks integers one by one until finding a common multiple |
| Optimal | O(1) | O(1) | Uses parity observation to compute answer directly |

## Algorithm Walkthrough

1. Receive the integer `n`.
2. Check whether `n` is even by evaluating `n % 2 == 0`.
3. If `n` is even, return `n` directly because it is already divisible by both `2` and itself.
4. If `n` is odd, return `2 * n` because multiplying an odd number by `2` produces the smallest even multiple of that number.

### Why it works

The algorithm relies on a simple mathematical property.

Every even number is already divisible by `2`, so the least common multiple of `2` and an even `n` is simply `n`.

An odd number is not divisible by `2`, so the smallest number divisible by both `2` and `n` must include a factor of `2`. Multiplying `n` by `2` creates the smallest such number.

Therefore, the algorithm always returns the least common multiple of `2` and `n`.

## Python Solution

```python
class Solution:
    def smallestEvenMultiple(self, n: int) -> int:
        if n % 2 == 0:
            return n
        
        return n * 2
```

The implementation begins by checking whether `n` is even using the modulo operator.

If `n % 2 == 0`, the number is divisible by `2`, so the method immediately returns `n`.

Otherwise, the number is odd, meaning it cannot be divisible by `2`. The smallest even multiple is therefore obtained by multiplying `n` by `2`.

The implementation is concise because the mathematical observation eliminates the need for loops or additional data structures.

## Go Solution

```go
func smallestEvenMultiple(n int) int {
    if n%2 == 0 {
        return n
    }

    return n * 2
}
```

The Go implementation follows exactly the same logic as the Python version.

Go uses `%` for modulo operations just like Python. Since the input range is very small, integer overflow is not a concern. No additional variables, slices, or data structures are needed.

## Edge Cases

### Edge Case 1, `n = 1`

The number `1` is odd and not divisible by `2`. A careless implementation might incorrectly return `1`, forgetting that the result must also be even.

The algorithm handles this correctly by detecting that `1` is odd and returning `1 * 2 = 2`.

### Edge Case 2, `n` is already even

For inputs like `6`, `10`, or `150`, the input itself is already divisible by `2`.

A naive solution might unnecessarily compute larger multiples, but the algorithm immediately returns `n`, guaranteeing the smallest valid answer.

### Edge Case 3, largest odd input

The largest odd value within constraints is `149`.

This case verifies that multiplication logic still works near the upper boundary. The algorithm returns `298`, which is divisible by both `149` and `2`.

Since the computation is constant time and uses tiny integers, there are no performance or overflow concerns.
