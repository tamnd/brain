## Solution

## Problem Understanding

The problem asks us to identify the most popular video creators based on the total number of views for all their videos and then, for each of these creators, return the ID of their single most viewed video. The inputs consist of three parallel arrays: `creators`, `ids`, and `views`. Each index `i` represents a video created by `creators[i]`, with ID `ids[i]`, and a view count of `views[i]`. The output is a 2D array of strings where each element contains the creator's name and the ID of their most viewed video.

The constraints indicate that the input arrays can be quite large, up to 100,000 elements, which suggests that a naive approach with nested loops could be too slow. Creators and video IDs are short lowercase strings, and video IDs are not guaranteed to be unique, so different videos can have the same ID. A key aspect of the problem is handling ties in both total popularity and individual video views: if multiple creators have the same maximum popularity, all should be included; if multiple videos for a creator have the same highest view count, the lexicographically smallest ID should be chosen.

Important edge cases include a single creator with multiple videos, videos with identical view counts, and videos with identical IDs across different creators.

## Approaches

The brute-force approach would involve iterating over all videos to calculate the total popularity for each creator, then iterating again to find the most viewed video for each top creator. This involves scanning the entire list multiple times and can be inefficient for large input sizes.

The optimal approach uses hash maps to efficiently track the required information in a single pass. One map stores the total popularity per creator, while another map tracks the most viewed video for each creator along with its view count. When processing each video, we update both the total popularity and the most viewed video if necessary. At the end, we identify the maximum popularity value and collect all creators that match it along with their most viewed video IDs.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Iterates multiple times over all videos to calculate totals and find max per creator |
| Optimal | O(n) | O(n) | Uses hash maps to track total popularity and top video per creator in one pass |

## Algorithm Walkthrough

1. Initialize two hash maps: `total_views` to store total views per creator and `top_video` to store the highest viewed video ID and its view count per creator.
2. Iterate over all videos using their indices:

- Add the current video's views to the creator's total in `total_views`.
- Check if the current video has more views than the stored top video in `top_video`, or if the views are equal but the current video ID is lexicographically smaller. Update `top_video` accordingly.
3. After processing all videos, find the maximum total popularity among all creators.
4. Iterate over the creators in `total_views` and collect those whose total popularity equals the maximum. For each, fetch the corresponding top video ID from `top_video`.
5. Return the results as a 2D list of `[creator, top_video_id]`.

This works because the invariant maintained is that `top_video` always contains the video with the highest view count (and lexicographically smallest ID in case of ties) for each creator, while `total_views` correctly accumulates total popularity. By tracking maximum popularity separately, we avoid scanning multiple times.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def mostPopularCreator(self, creators: List[str], ids: List[str], views: List[int]) -> List[List[str]]:
        total_views = defaultdict(int)
        top_video = {}
        
        for i in range(len(creators)):
            creator = creators[i]
            vid_id = ids[i]
            view_count = views[i]
            
            total_views[creator] += view_count
            
            if creator not in top_video:
                top_video[creator] = (vid_id, view_count)
            else:
                current_id, current_views = top_video[creator]
                if view_count > current_views or (view_count == current_views and vid_id < current_id):
                    top_video[creator] = (vid_id, view_count)
        
        max_views = max(total_views.values())
        result = []
        for creator, total in total_views.items():
            if total == max_views:
                result.append([creator, top_video[creator][0]])
        return result
```

The code first sets up a default dictionary for counting total views and a regular dictionary to track the top video for each creator. It iterates once through all videos, updating both structures as needed. Finally, it calculates the maximum popularity and collects all creators who match this maximum, returning their top video IDs.

## Go Solution

```go
func mostPopularCreator(creators []string, ids []string, views []int) [][]string {
    totalViews := make(map[string]int)
    topVideo := make(map[string][2]interface{})
    
    for i := 0; i < len(creators); i++ {
        creator := creators[i]
        vidID := ids[i]
        viewCount := views[i]
        
        totalViews[creator] += viewCount
        
        if val, exists := topVideo[creator]; !exists {
            topVideo[creator] = [2]interface{}{vidID, viewCount}
        } else {
            currentID := val[0].(string)
            currentViews := val[1].(int)
            if viewCount > currentViews || (viewCount == currentViews && vidID < currentID) {
                topVideo[creator] = [2]interface{}{vidID, viewCount}
            }
        }
    }
    
    maxViews := 0
    for _, total := range totalViews {
        if total > maxViews {
            maxViews = total
        }
    }
    
    result := [][]string{}
    for creator, total := range totalViews {
        if total == maxViews {
            topID := topVideo[creator][0].(string)
            result = append(result, []string{creator, topID})
        }
    }
    
    return result
}
```

In Go, we use `map[string]int` for total views and `map[string][2]interface{}` for storing both the video ID and view count per creator. Type assertion is required to extract the stored values. The logic mirrors the Python solution, iterating once over all videos and collecting results at the end.

## Worked Examples

**Example 1:**

Input: `creators = ["alice","bob","alice","chris"], ids = ["one","two","three","four"], views = [5,10,5,4]`

Iteration states:

| Creator | Video | Views | Total Views | Top Video |
| --- | --- | --- | --- | --- |
| alice | one | 5 | 5 | one |
| bob | two | 10 | 10 | two |
| alice | three | 5 | 10 | one (tie, one < three lex) |
| chris | four | 4 | 4 | four |

Max total views = 10. Result: `[["alice","one"],["bob","two"]]`.

**Example 2:**

Input: `creators = ["alice","alice","alice"], ids = ["a","b","c"], views = [1,2,2]`

Iteration states:

| Creator | Video | Views | Total Views | Top Video |
| --- | --- | --- | --- | --- |
| alice | a | 1 | 1 | a |
| alice | b | 2 | 3 | b |
| alice | c | 2 | 5 | b (tie, b < c lex) |

Max total views = 5. Result: `[["alice","b"]]`.
