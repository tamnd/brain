# LeetCode 2423 - Remove Letter To Equalize Frequency

## Problem Understanding

The problem gives us a lowercase English string called `word`. We must remove exactly one character from the string, then determine whether all remaining characters that still appear in the string have the same frequency.

The important detail is that we are not allowed to skip the removal. Even if the original string already has equal frequencies, we still must delete one character and verify whether the remaining frequencies become equal afterward.

For example, consider `"abcc"`:

- Frequencies before removal:

- `a = 1`
- `b = 1`
- `c = 2`

If we remove one `'c'`, the frequencies become:

- `a = 1`
- `b = 1`
- `c = 1`

All present characters now occur exactly once, so the answer is `true`.

The input size is small:

- `2 <= word.length <= 100`

This constraint is extremely important because it means even relatively inefficient approaches are acceptable. We do not need advanced optimization techniques. A direct simulation approach works comfortably within limits.

There are several edge cases that can easily cause bugs:

- A string where all characters are identical, such as `"aaaa"`. Removing one character leaves `"aaa"`, which still has equal frequencies.
- A string where every character appears once, such as `"abc"`. Removing any character leaves two characters with frequency `1`.
- A string with two different frequencies where only one character can be adjusted correctly.
- Cases where removing one character entirely removes a letter from the string. We must ignore zero frequencies when checking equality.

The problem guarantees that the string contains only lowercase English letters, so the total number of distinct characters is at most 26.

## Approaches

### Brute Force Approach

The most direct solution is to try removing every possible character one by one.

For each index:

1. Remove the character at that position.
2. Recalculate the frequency of all remaining characters.
3. Check whether all non-zero frequencies are equal.
4. If any removal produces equal frequencies, return `true`.

If none of the removals work, return `false`.

This approach is guaranteed to work because it exhaustively checks every valid deletion possibility.

Since the maximum string length is only 100, this method is fast enough in practice.

### Key Insight for the Optimal Approach

The important observation is that there are only 26 lowercase English letters. Instead of rebuilding strings repeatedly, we can maintain a frequency counter and simulate deletions directly.

For each character:

1. Temporarily decrease its frequency by one.
2. Collect all remaining non-zero frequencies.
3. Check whether all collected frequencies are identical.
4. Restore the frequency afterward.

This avoids repeatedly constructing new strings and keeps the implementation cleaner and more efficient.

Even though both approaches are fast enough for the constraints, the frequency simulation approach is more elegant and scales better conceptually.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Rebuilds frequencies after every deletion |
| Optimal | O(26 × n) | O(26) | Uses frequency counting and simulated removal |

## Algorithm Walkthrough

1. Create a frequency map of all characters in the string.

We need to know how many times each letter appears. A hash map or counter structure is ideal because it provides constant-time updates and lookups.
2. Iterate through every character in the string.

Each iteration represents removing that specific character once.
3. Temporarily decrease the frequency of the current character.

This simulates deleting that character from the string.
4. Gather all non-zero frequencies.

Frequencies that become zero should be ignored because letters that no longer exist in the string are not considered when checking equality.
5. Check whether all remaining frequencies are equal.

This can be done by converting the list of frequencies into a set. If the set size is `1`, then all frequencies are identical.
6. If the frequencies are equal, return `true`.

We found a valid deletion that satisfies the condition.
7. Restore the removed character frequency.

This prepares the frequency map for the next simulation.
8. If all deletions fail, return `false`.

### Why It Works

The algorithm works because it explicitly tests every possible valid deletion. For each simulated removal, it verifies the exact condition required by the problem: every remaining character must have the same frequency. Since every possible deletion is examined, the algorithm cannot miss a valid solution.

## Python Solution

```python
from collections import Counter
from typing import List

class Solution:
    def equalFrequency(self, word: str) -> bool:
        frequency = Counter(word)

        for char in word:
            frequency[char] -= 1

            remaining_frequencies = [
                count
                for count in frequency.values()
                if count > 0
            ]

            if len(set(remaining_frequencies)) == 1:
                return True

            frequency[char] += 1

        return False
```

The implementation begins by creating a `Counter` object that stores the frequency of every character.

The loop iterates through each character in the string. During each iteration, the algorithm temporarily decreases the frequency of that character, simulating its removal.

The list comprehension extracts all positive frequencies. Zero values are excluded because letters removed completely from the string should not participate in the equality check.

The expression `len(set(remaining_frequencies)) == 1` verifies whether every remaining frequency is identical. If so, the function immediately returns `True`.

If the current deletion does not work, the character count is restored before moving to the next iteration.

If no deletion succeeds, the function returns `False`.

## Go Solution

```go
func equalFrequency(word string) bool {
	frequency := make(map[byte]int)

	for i := 0; i < len(word); i++ {
		frequency[word[i]]++
	}

	for i := 0; i < len(word); i++ {
		ch := word[i]
		frequency[ch]--

		same := true
		target := 0

		for _, count := range frequency {
			if count == 0 {
				continue
			}

			if target == 0 {
				target = count
			} else if count != target {
				same = false
				break
			}
		}

		if same {
			return true
		}

		frequency[ch]++
	}

	return false
}
```

The Go implementation follows the same logic as the Python solution. A map stores character frequencies, and each iteration simulates deleting one character.

Instead of using a set, the Go version keeps a `target` frequency and ensures all non-zero frequencies match it.

Go requires careful handling of integer types and map iteration, but otherwise the logic is identical. Since the maximum string length is only 100, integer overflow is not a concern.

## Edge Cases

One important edge case is when all characters are already identical, such as `"aaaa"`. A buggy implementation might incorrectly assume that removing one character breaks equality. In reality, after removing one `'a'`, the remaining string still has all characters occurring equally because only one character type exists. The implementation handles this naturally because it ignores zero frequencies and checks only remaining letters.

Another important case is when every character appears exactly once, such as `"abc"`. After removing any character, the remaining two characters both have frequency `1`. Some implementations mistakenly compare against the original frequencies instead of recalculating after deletion. This solution explicitly simulates the removal each time, so it correctly identifies the valid outcome.

A third tricky case occurs when removing a character completely eliminates that letter from the string. For example, in `"cccd"`, removing `'d'` leaves only `'c'` with frequency `3`. The implementation correctly ignores zero counts when checking equality. If zero frequencies were included, the algorithm would incorrectly conclude that the frequencies differ.
