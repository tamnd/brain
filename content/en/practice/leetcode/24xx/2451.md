## Solution

## Problem Understanding

In this problem, every string in the input array can be transformed into a "difference array". The difference array represents how much each consecutive character changes in alphabetical position.

For a string of length `n`, the difference array has length `n - 1`. Each value is computed as:

```
words[i][j+1] - words[i][j]
```

where characters are treated as their alphabet positions:

```
'a' = 0
'b' = 1
...
'z' = 25
```

For example:

```
"adc"
a -> 0
d -> 3
c -> 2

Differences:
3 - 0 = 3
2 - 3 = -1

Difference array = [3, -1]
```

The key observation is that all strings except one share exactly the same difference array. One string produces a different pattern, and we must return that string.

The input consists of:

- An array `words`
- All strings have equal length
- There are at least 3 strings
- Exactly one string has a unique difference array

The output is the single string whose difference array differs from all the others.

The constraints are small:

- Up to 100 strings
- Each string length is at most 20

This means even relatively straightforward approaches are fast enough. We do not need advanced optimization techniques.

An important detail is that difference values may be negative. For example:

```
"ba"
0 - 1 = -1
```

So we must preserve sign when storing differences.

Another important guarantee is that exactly one string is different. This means there will always be one clear outlier, and we do not need to handle ties or ambiguous cases.

Potential edge cases include:

- Strings of minimum length 2
- Difference arrays containing negative numbers
- The odd string appearing at the beginning of the array
- The odd string appearing at the end of the array
- Multiple repeated identical strings
- Large positive and negative jumps between characters

Because the problem guarantees a single valid answer, we do not need additional validation logic.

## Approaches

### Brute Force Approach

The brute force idea is to compute the difference array for every string, then compare each difference array against every other difference array to determine which one appears only once.

For each string:

1. Build its difference array
2. Compare it with all other difference arrays
3. Count how many times it matches
4. The one with count `1` is the answer

This works because the problem guarantees that all but one share the same pattern.

However, this approach performs repeated comparisons between arrays. If there are `m` words and each difference array has length `n - 1`, then comparing all pairs requires:

```
O(m^2 * n)
```

Although the constraints are small enough for this to pass, it performs unnecessary repeated work.

### Optimal Approach

The better approach is to use a hash map.

Instead of repeatedly comparing arrays, we can:

1. Convert each word into its difference array
2. Store the difference array as a hashable key
3. Count how many times each pattern appears
4. Return the word whose pattern appears exactly once

The important insight is that difference arrays uniquely identify the pattern of a word. Since only one word has a unique pattern, frequency counting immediately identifies the answer.

Because each difference array is short, we can convert it into a tuple in Python or a string key in Go.

This reduces the complexity to linear time with respect to the number of words.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m² × n) | O(m × n) | Compares every difference array against every other one |
| Optimal | O(m × n) | O(m × n) | Uses a hash map to count frequency of difference patterns |

Here:

- `m` = number of words
- `n` = length of each word

## Algorithm Walkthrough

1. Create a helper function that converts a word into its difference array.

For every adjacent pair of characters, compute:

```
ord(word[i + 1]) - ord(word[i])
```

This captures the relative movement between characters.
2. Iterate through all words and compute their difference patterns.

For example:

```
"adc" -> (3, -1)
"wzy" -> (3, -1)
"abc" -> (1, 1)
```
3. Store each pattern in a hash map.

The hash map tracks how many times each difference pattern appears.

Example:

```
{
    (3, -1): 2,
    (1, 1): 1
}
```
4. Also store a mapping from pattern to original word.

This allows us to retrieve the word later.
5. After processing all words, iterate through the hash map.

The pattern with frequency `1` corresponds to the odd string.
6. Return the word associated with that unique pattern.

### Why it works

All strings except one share the same difference array. Therefore, the shared pattern appears multiple times, while the odd pattern appears exactly once.

By counting frequencies of difference arrays, the algorithm directly identifies the unique pattern and therefore the unique word.

Because the problem guarantees exactly one odd string, this method always returns the correct answer.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def oddString(self, words: List[str]) -> str:
        def build_difference(word: str) -> tuple:
            differences = []

            for i in range(len(word) - 1):
                differences.append(ord(word[i + 1]) - ord(word[i]))

            return tuple(differences)

        pattern_count = defaultdict(int)
        pattern_to_word = {}

        for word in words:
            pattern = build_difference(word)

            pattern_count[pattern] += 1
            pattern_to_word[pattern] = word

        for pattern, count in pattern_count.items():
            if count == 1:
                return pattern_to_word[pattern]

        return ""
```

The implementation begins with a helper function named `build_difference`. This function converts a word into its difference pattern by computing the character distance between every adjacent pair.

The result is stored as a tuple because tuples are hashable and can be used as dictionary keys.

Two hash maps are maintained:

- `pattern_count` tracks how many times each pattern appears
- `pattern_to_word` remembers which word produced each pattern

As each word is processed, its pattern frequency increases.

After all words are processed, the algorithm scans the frequency map to find the pattern that appears exactly once. The associated word is returned immediately.

The final `return ""` statement is only included to satisfy function completeness. According to the problem constraints, a valid odd string always exists.

## Go Solution

```go
package main

import (
	"strconv"
	"strings"
)

func oddString(words []string) string {
	buildDifference := func(word string) string {
		var builder strings.Builder

		for i := 0; i < len(word)-1; i++ {
			diff := int(word[i+1]) - int(word[i])

			builder.WriteString(strconv.Itoa(diff))
			builder.WriteString(",")
		}

		return builder.String()
	}

	patternCount := make(map[string]int)
	patternToWord := make(map[string]string)

	for _, word := range words {
		pattern := buildDifference(word)

		patternCount[pattern]++
		patternToWord[pattern] = word
	}

	for pattern, count := range patternCount {
		if count == 1 {
			return patternToWord[pattern]
		}
	}

	return ""
}
```

The Go implementation follows the same overall logic as the Python version.

Since Go slices cannot be directly used as hash map keys, the difference array is converted into a string representation. A `strings.Builder` efficiently constructs the key.

For example:

```go
[3, -1]
```

becomes:

```go
"3,-1,"
```

Maps are then used to count frequencies and store the associated word.

There are no concerns about integer overflow because character differences are limited to the range `[-25, 25]`.

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(m × n) | Each word is processed once, and each difference array takes O(n) time to build |
| Space | O(m × n) | Hash maps store up to m difference patterns, each of length n - 1 |

Here:

- `m` is the number of words
- `n` is the length of each word

The algorithm is efficient because it avoids repeated pairwise comparisons. Each string contributes exactly one pass through its characters.

## Edge Cases

### Minimum Length Strings

The smallest allowed string length is 2, meaning each difference array contains exactly one value.

For example:

```
"ab" -> [1]
"bc" -> [1]
"az" -> [25]
```

A buggy implementation might incorrectly assume multiple differences exist. This solution handles the case naturally because the loop still executes correctly for length 2 strings.

### Negative Differences

Character differences may be negative.

For example:

```
"dc"
2 - 3 = -1
```

If differences were stored incorrectly or converted improperly, patterns could collide or become invalid. The implementation preserves signed integer values exactly, ensuring distinct patterns remain distinct.

### Odd String at the Beginning

The unique string is not guaranteed to appear near the end.

For example:

```
["ace", "abc", "bcd", "xyz"]
```

A naive implementation that assumes the first few strings are the majority pattern could fail. This solution avoids assumptions entirely by counting all patterns before deciding which one is unique.
