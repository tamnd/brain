## Solution

## Problem Understanding

The problem asks us to process a singly-linked list and remove nodes that have any node with a strictly greater value to their right. In other words, for each node in the list, if there exists a node later in the sequence with a larger value, the current node should be deleted. The input is a linked list `head` with at least one node and values in the range `[1, 10^5]`. The output is the head of the modified linked list containing only nodes that are not overshadowed by a larger node to their right.

The constraints indicate the list could have up to 100,000 nodes, so any naive solution that repeatedly scans the list for each node will likely exceed time limits. Edge cases include a list where all values are equal, a strictly decreasing or increasing list, and a list with a single node.

## Approaches

The brute-force approach is straightforward: for each node, traverse all nodes to its right to check if a larger value exists. If such a value exists, delete the current node. This guarantees correctness but is inefficient because for each node we may scan up to `n` nodes, resulting in O(n²) time complexity.

The key observation for an optimal solution is that we can reverse the list and process it from the end to the beginning. By traversing from the original list’s tail to head, we can maintain the maximum value seen so far. If the current node’s value is less than this maximum, it should be removed; otherwise, it becomes part of the result. This approach only requires one pass over the list after reversal, giving O(n) time complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Check each node against all nodes to its right |
| Optimal | O(n) | O(1) | Reverse list, keep track of max, rebuild list |

## Algorithm Walkthrough

1. Reverse the input linked list. This allows us to process nodes from the original tail to head.
2. Initialize a variable `max_value` to negative infinity and a pointer `new_head` to null. `max_value` keeps track of the largest node value encountered so far.
3. Traverse the reversed list. For each node, compare its value to `max_value`.
4. If the node value is greater than or equal to `max_value`, update `max_value` and prepend this node to the `new_head` list. Otherwise, skip the node.
5. After the traversal, `new_head` points to the resulting list with all smaller nodes removed.
6. Return `new_head` as the final head of the modified list.

**Why it works:** Reversing the list ensures that each node is compared with all nodes originally to its right. By keeping track of the maximum seen, we guarantee that only nodes not overshadowed by any larger value remain. Prepending nodes while traversing avoids the need for another reversal.

## Python Solution

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def removeNodes(self, head: Optional[ListNode]) -> Optional[ListNode]:
        prev = None
        current = head
        # Step 1: Reverse the list
        while current:
            nxt = current.next
            current.next = prev
            prev = current
            current = nxt
        
        # prev is now the head of reversed list
        new_head = None
        max_value = float('-inf')
        current = prev
        
        # Step 2: Filter nodes
        while current:
            if current.val >= max_value:
                max_value = current.val
                temp = current.next
                current.next = new_head
                new_head = current
                current = temp
            else:
                current = current.next
        
        return new_head
```

The Python code first reverses the input list using standard pointer manipulation. Then, it iterates through the reversed list, maintaining the maximum value seen so far. Nodes smaller than this max are skipped, while larger or equal nodes are prepended to the new list, ensuring the final list has the correct order and all required nodes.

## Go Solution

```go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func removeNodes(head *ListNode) *ListNode {
    var prev *ListNode
    current := head
    // Step 1: Reverse the list
    for current != nil {
        nxt := current.Next
        current.Next = prev
        prev = current
        current = nxt
    }
    
    newHead := (*ListNode)(nil)
    maxValue := 0
    current = prev
    first := true
    
    // Step 2: Filter nodes
    for current != nil {
        if first || current.Val >= maxValue {
            if first {
                maxValue = current.Val
                first = false
            } else {
                maxValue = current.Val
            }
            temp := current.Next
            current.Next = newHead
            newHead = current
            current = temp
        } else {
            current = current.Next
        }
    }
    
    return newHead
}
```

In Go, we handle `nil` pointers instead of Python `None`. The logic mirrors Python: reverse the list, then track the maximum value while rebuilding the list from nodes that should remain. A `first` flag is used to initialize `maxValue` safely.

## Worked Examples

**Example 1: head = [5,2,13,3,8]**

| Step | Node | Reversed List | max_value | new_head |
| --- | --- | --- | --- | --- |
| 0 | - | 8->3->13->2->5 | -inf | nil |
| 1 | 8 | 8->nil | 8 | 8 |
| 2 | 3 | 13->3->8->2->5 | 8 | 8 |
| 3 | 13 | 13->8 | 13 | 13->8 |
| 4 | 2 | 2 | 13 | 13->8 |
| 5 | 5 | 5 | 13 | 13->8 |

Output: [13,8]

**Example 2: head = [1,1,1,1]**

All nodes are equal. Traversing reversed list keeps every node since all are >= max_value at each step. Output: [1,1,1,1]

## Edge Cases

**Single Node:** The list contains only one node. It should be returned as-is. The implementation handles this automatically because the reverse operation and maximum comparison do not remove it.

**Strictly Increasing List:** Only the last node should remain. The algorithm correctly discards every node except the final maximum.

**Repeated Values:** When multiple consecutive nodes have the same value, none should be removed if no greater value exists to the right. The `>=` comparison ensures equality is preserved correctly.
