**Difficulty:** 🟡 Medium  
**Topics:** Array, Tree, Depth-First Search, Breadth-First Search, Graph Theory

## Solution

## Problem Understanding

The problem gives us an undirected tree with `n` nodes rooted at node `0`. Since the graph is a tree, there is exactly one simple path between any two nodes. Each node contains a gate associated with a value from the `amount` array. Positive values represent rewards that can be collected, while negative values represent costs that must be paid.

Two players move through the tree simultaneously:

- Alice starts at node `0` and moves downward toward some leaf node of her choosing.
- Bob starts at node `bob` and always moves upward toward node `0`.

Every second, both move exactly one edge unless they have already stopped. Alice stops once she reaches a leaf. Bob stops once he reaches node `0`.

Whenever a player reaches a node for the first time, they either collect or pay the gate value. If both arrive at the same node at the same time, they split the value equally. Since all values are guaranteed to be even, integer division is always safe.

The goal is to determine the maximum possible net income Alice can obtain by choosing the optimal leaf path.

The constraints are important:

- `n` can be as large as `10^5`
- The graph is guaranteed to be a valid tree
- Every node has at most one parent in the rooted interpretation
- A brute-force simulation of all possible Alice paths combined with repeated path computations would be too slow

These constraints strongly suggest that we need a linear or near-linear time solution.

Several edge cases are especially important:

- Bob may reach a node before Alice, making its reward unavailable.
- Alice and Bob may arrive simultaneously, requiring the amount to be halved.
- The optimal path may still produce a negative result if all choices are bad.
- The root itself may contain a negative value.
- A tree may degenerate into a chain, which tests recursion depth and traversal correctness.
- Alice may encounter nodes already opened by Bob, contributing zero profit.

## Approaches

### Brute Force Approach

A straightforward approach is to enumerate every possible root-to-leaf path for Alice.

For each leaf:

1. Compute Alice's arrival time at every node on the path.
2. Compute Bob's arrival time at every node by reconstructing his path to the root.
3. Simulate the interaction node by node:

- If Alice arrives first, she gets the full amount.
- If Bob arrives first, Alice gets nothing.
- If they arrive simultaneously, Alice gets half.
4. Track the maximum total income among all leaves.

This approach is correct because it explicitly simulates every possible valid strategy Alice can take.

However, it becomes inefficient because repeatedly reconstructing paths and arrival times for many leaves can lead to quadratic behavior in large trees. In the worst case, a skewed tree may contain `O(n)` leaves and each path computation may also cost `O(n)`.

With `n = 10^5`, this is too slow.

### Key Insight

The crucial observation is that Bob's movement is completely deterministic.

Bob always follows the unique path from his starting node back to the root. Therefore:

- We only need to compute Bob's arrival time for nodes on that single path.
- Any node not on Bob's route is never visited by Bob.

Once Bob's arrival times are known, the problem becomes a standard tree DFS for Alice:

- Traverse every root-to-leaf path once.
- At each node, determine Alice's profit contribution based on arrival timing.
- Keep the maximum leaf profit.

This transforms the problem into two linear traversals:

1. Compute Bob's timings.
2. DFS all Alice paths.

Because trees contain exactly `n - 1` edges, both traversals run in linear time.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Simulates every root-to-leaf path independently |
| Optimal | O(n) | O(n) | Precomputes Bob arrival times and performs one DFS |

## Algorithm Walkthrough

1. Build an adjacency list for the tree.

Since the graph is undirected and sparse, an adjacency list is the most efficient representation. Each node stores its neighboring nodes.
2. Run a DFS from the root to determine each node's parent and depth.

We need parent information so we can reconstruct Bob's unique path back to node `0`.
3. Compute Bob's arrival times.

Starting from `bob`, repeatedly move to the parent until reaching node `0`.

Maintain:

- `bob_time[node] = time Bob reaches node`

Nodes not visited by Bob can be initialized to infinity.
4. Start a DFS for Alice from node `0`.

During traversal:

- `time` represents Alice's arrival time.
- `income` represents cumulative profit.
5. At each node, determine Alice's reward contribution.

There are three cases:

- If `time < bob_time[node]`, Alice arrives first and gets the full amount.
- If `time == bob_time[node]`, they split the amount.
- If `time > bob_time[node]`, Bob already opened the gate and Alice gets `0`.
6. Detect leaf nodes.

A node is a leaf if:

- It is not the root, and
- It has only one neighbor in the undirected tree

When Alice reaches a leaf, update the global maximum profit.
7. Continue DFS until all root-to-leaf paths are explored.

### Why it works

Bob's movement is fixed and deterministic, so precomputing his arrival times fully captures his effect on the game. Alice's decision only affects which leaf path she follows. Since DFS explores every root-to-leaf path exactly once and computes the exact profit contribution at each node according to arrival timing, the algorithm evaluates every valid Alice strategy and returns the maximum achievable income.

## Python Solution

```python
from typing import List
from collections import defaultdict
import math

class Solution:
    def mostProfitablePath(self, edges: List[List[int]], bob: int, amount: List[int]) -> int:
        n = len(amount)

        graph = defaultdict(list)

        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)

        parent = [-1] * n
        depth = [0] * n

        def build_parent(node: int, prev: int) -> None:
            parent[node] = prev

            for neighbor in graph[node]:
                if neighbor == prev:
                    continue

                depth[neighbor] = depth[node] + 1
                build_parent(neighbor, node)

        build_parent(0, -1)

        bob_time = [math.inf] * n

        current = bob
        time = 0

        while current != -1:
            bob_time[current] = time
            current = parent[current]
            time += 1

        best_profit = -math.inf

        def dfs_alice(node: int, prev: int, time: int, income: int) -> None:
            nonlocal best_profit

            if time < bob_time[node]:
                income += amount[node]
            elif time == bob_time[node]:
                income += amount[node] // 2

            is_leaf = (node != 0 and len(graph[node]) == 1)

            if is_leaf:
                best_profit = max(best_profit, income)
                return

            for neighbor in graph[node]:
                if neighbor == prev:
                    continue

                dfs_alice(neighbor, node, time + 1, income)

        dfs_alice(0, -1, 0, 0)

        return best_profit
```

The implementation begins by constructing the adjacency list representation of the tree. This allows efficient traversal in both directions because the tree is undirected.

The `build_parent` DFS computes the parent of every node relative to the root. Once this information is available, Bob's path to the root can be reconstructed easily by repeatedly following parent pointers.

The `bob_time` array stores the exact second Bob arrives at each node. Nodes that Bob never visits remain initialized to infinity, meaning Alice always arrives first there.

The second DFS simulates Alice's traversal. At each node, the algorithm compares Alice's arrival time with Bob's arrival time to determine the correct profit contribution.

Leaf detection is handled carefully. In an undirected tree, a non-root node with only one neighbor is a leaf.

The algorithm keeps a global maximum profit and updates it whenever Alice reaches a leaf.

## Go Solution

```go
package main

import "math"

func mostProfitablePath(edges [][]int, bob int, amount []int) int {
	n := len(amount)

	graph := make([][]int, n)

	for _, edge := range edges {
		u := edge[0]
		v := edge[1]

		graph[u] = append(graph[u], v)
		graph[v] = append(graph[v], u)
	}

	parent := make([]int, n)

	var buildParent func(int, int)

	buildParent = func(node int, prev int) {
		parent[node] = prev

		for _, neighbor := range graph[node] {
			if neighbor == prev {
				continue
			}

			buildParent(neighbor, node)
		}
	}

	buildParent(0, -1)

	bobTime := make([]int, n)

	for i := 0; i < n; i++ {
		bobTime[i] = math.MaxInt32
	}

	current := bob
	time := 0

	for current != -1 {
		bobTime[current] = time
		current = parent[current]
		time++
	}

	bestProfit := math.MinInt32

	var dfsAlice func(int, int, int, int)

	dfsAlice = func(node int, prev int, time int, income int) {
		if time < bobTime[node] {
			income += amount[node]
		} else if time == bobTime[node] {
			income += amount[node] / 2
		}

		isLeaf := node != 0 && len(graph[node]) == 1

		if isLeaf {
			if income > bestProfit {
				bestProfit = income
			}
			return
		}

		for _, neighbor := range graph[node] {
			if neighbor == prev {
				continue
			}

			dfsAlice(neighbor, node, time+1, income)
		}
	}

	dfsAlice(0, -1, 0, 0)

	return bestProfit
}
```

The Go implementation closely mirrors the Python logic. Slices are used for adjacency lists and arrays such as `parent` and `bobTime`.

Go does not provide infinity for integers, so `math.MaxInt32` is used as a practical substitute for nodes Bob never reaches.

Recursive closures are declared using `var functionName func(...)` before assignment so that the function can recursively reference itself.

Integer division is safe because the problem guarantees all amounts are even.

## Edge Cases

### Simultaneous Arrival at a Node

One of the easiest mistakes is mishandling simultaneous arrival times. If Alice and Bob reach a node at the same second, the reward or cost must be split evenly.

The implementation explicitly checks:

```
elif time == bob_time[node]:
```

and adds `amount[node] // 2`.

Because all values are guaranteed to be even, integer division is always exact.

### Bob Opens a Gate Before Alice Arrives

If Bob reaches a node first, Alice should receive nothing from that node, even if the amount is positive.

A buggy implementation might accidentally add the amount anyway or subtract the cost twice.

The solution handles this correctly by only adding income when:

```
time < bob_time[node]
```

or

```
time == bob_time[node]
```

Otherwise, no contribution is added.

### Trees That Degenerate Into Chains

A tree may become a straight line of length `n`. In such cases:

- DFS depth becomes `O(n)`
- There is only one leaf path
- Timing calculations become especially sensitive

The algorithm still works correctly because every node is processed exactly once during each traversal. The logic for Bob's path also becomes particularly simple because every node lies on the same chain.
