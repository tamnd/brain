## Solution

## Problem Understanding

The problem requires us to sort each level of a binary tree in strictly increasing order by performing the minimum number of **swap operations between nodes at the same level**. Each node in the tree has a **unique value**, and the tree can be large, with up to $10^5$ nodes.

The input is the root of a binary tree, and the output is a single integer: the total minimum number of operations needed to sort all levels of the tree. Sorting must happen **level by level**, and swaps are only allowed **within the same level**, not across levels. The problem guarantees uniqueness of values, so we do not need to handle duplicates.

Edge cases to consider include trees that are already sorted at all levels, trees with only a single node, and highly skewed trees (all left or all right) which still have levels with a single node.

The key challenge is efficiently determining the **minimum number of swaps** required for each level while avoiding an $O(n^2)$ brute-force approach.

## Approaches

### Brute Force

A naive approach would be to traverse each level of the tree, sort the values of that level, and repeatedly swap elements until the level is sorted. This would involve checking every possible pair of swaps until the sorted order is achieved. While correct, this approach is very inefficient, roughly $O(k^2)$ per level (where $k$ is the number of nodes at that level), and could lead to $O(n^2)$ total time for large trees. It is too slow for the problem constraints.

### Optimal Approach

The key insight is that for **any array of unique numbers**, the minimum number of swaps to sort it can be computed using **cycle decomposition in a permutation**. Each level can be treated as an array. By mapping each element to its target index in the sorted array and counting the cycles, the number of swaps needed for that level is:

$$\text{swaps} = \text{sum of (cycle length - 1) for all cycles}$$

Thus, the algorithm is:

1. Traverse the tree level by level using BFS.
2. For each level, collect the node values.
3. Compute the minimum swaps needed to sort that array using cycle decomposition.
4. Accumulate the swap counts across all levels.

This reduces the problem to $O(n \log n)$ time per level for sorting and $O(k)$ for computing swaps, leading to an overall **efficient solution**.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Repeatedly swap elements on each level until sorted |
| Optimal | O(n log n) | O(n) | BFS per level + cycle decomposition for minimum swaps |

## Algorithm Walkthrough

1. **Initialize BFS**: Create a queue and add the root node.
2. **Process Levels**: While the queue is not empty, process nodes level by level.
3. **Collect Values**: For each level, store all node values in an array.
4. **Compute Minimum Swaps**: For the array of values, create a mapping from value to index in the sorted array. Use cycle detection:

- Initialize `visited` array.
- For each index `i`, if it has not been visited, traverse its cycle and count the cycle length.
- Add `cycle_length - 1` to the swap count.
5. **Add Children to Queue**: Add the left and right children of nodes at the current level to the queue.
6. **Accumulate Result**: Sum the swap counts from all levels.
7. **Return Total**: Return the accumulated total swaps as the answer.

**Why it works:** Sorting an array with minimum swaps is equivalent to breaking it into cycles and performing `cycle_length - 1` swaps for each cycle. Since each level is independent and swaps are only within a level, summing the swaps across levels gives the correct minimum total.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
from collections import deque
from typing import Optional

class Solution:
    def minimumOperations(self, root: Optional['TreeNode']) -> int:
        if not root:
            return 0
        
        total_swaps = 0
        queue = deque([root])
        
        while queue:
            level_size = len(queue)
            level_vals = []
            
            for _ in range(level_size):
                node = queue.popleft()
                level_vals.append(node.val)
                if node.left:
                    queue.append(node.left)
                if node.right:
                    queue.append(node.right)
            
            # Compute minimum swaps to sort level_vals
            sorted_vals = sorted(level_vals)
            index_map = {val: i for i, val in enumerate(sorted_vals)}
            visited = [False] * len(level_vals)
            
            for i in range(len(level_vals)):
                if visited[i] or index_map[level_vals[i]] == i:
                    continue
                cycle_size = 0
                j = i
                while not visited[j]:
                    visited[j] = True
                    j = index_map[level_vals[j]]
                    cycle_size += 1
                if cycle_size > 0:
                    total_swaps += cycle_size - 1
        
        return total_swaps
```

**Implementation Walkthrough:**

We perform BFS to process each level independently. `level_vals` stores the values at the current level. Using `sorted_vals` and `index_map`, we determine the final sorted positions. Cycle detection then counts the minimum swaps required. Finally, we sum swaps across all levels to return the answer.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func minimumOperations(root *TreeNode) int {
    if root == nil {
        return 0
    }

    totalSwaps := 0
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        levelSize := len(queue)
        levelVals := make([]int, levelSize)
        nextQueue := []*TreeNode{}

        for i := 0; i < levelSize; i++ {
            node := queue[i]
            levelVals[i] = node.Val
            if node.Left != nil {
                nextQueue = append(nextQueue, node.Left)
            }
            if node.Right != nil {
                nextQueue = append(nextQueue, node.Right)
            }
        }

        // Compute minimum swaps
        sortedVals := make([]int, levelSize)
        copy(sortedVals, levelVals)
        sort.Ints(sortedVals)
        indexMap := make(map[int]int)
        for i, val := range sortedVals {
            indexMap[val] = i
        }

        visited := make([]bool, levelSize)
        for i := 0; i < levelSize; i++ {
            if visited[i] || indexMap[levelVals[i]] == i {
                continue
            }
            cycleSize := 0
            j := i
            for !visited[j] {
                visited[j] = true
                j = indexMap[levelVals[j]]
                cycleSize++
            }
            if cycleSize > 0 {
                totalSwaps += cycleSize - 1
            }
        }
        queue = nextQueue
    }
    return totalSwaps
}
```

**Go Notes:**

We use slices for the queue and `levelVals`. Maps track indices for sorting. `visited` is a boolean slice. BFS logic is identical to Python, accounting for Go-specific nil checks and slice operations.
