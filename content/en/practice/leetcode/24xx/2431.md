## Solution

## Problem Understanding

The problem asks us to maximize the total tastiness of fruits we can purchase under a budget constraint, with the additional complexity of using a limited number of coupons that reduce the price of individual fruits by half (rounded down). We are given two arrays, `price` and `tastiness`, both of length `n`. `price[i]` is the cost of the `i`-th fruit, and `tastiness[i]` is its tastiness value. We are also given `maxAmount`, the total money we can spend, and `maxCoupons`, the maximum number of coupons we can use.

We are asked to select fruits such that the sum of their tastiness is maximized, the sum of their prices (after possible coupon discounts) does not exceed `maxAmount`, and each fruit can be purchased at most once, with a coupon applied at most once per fruit.

The constraints are moderate: `n` can be up to 100, prices and tastiness up to 1000, and `maxCoupons` up to 5. These constraints suggest that a solution with a dynamic programming approach based on `O(n * maxAmount * maxCoupons)` is feasible.

Edge cases to consider include fruits with zero price, tastiness values of zero, zero `maxAmount`, zero `maxCoupons`, and scenarios where using a coupon is not optimal due to budget constraints.

## Approaches

A brute-force approach would be to try all combinations of fruits with and without coupons, calculating the total cost and tastiness for each combination. This guarantees correctness but is computationally infeasible, since the number of combinations grows exponentially with `n` (specifically, `3^n` if we consider each fruit can be bought with, without, or not at all).

The key insight for an optimal solution is to use **dynamic programming**. We can define a DP state `dp[i][c][b]` as the maximum tastiness achievable using the first `i` fruits, with `c` coupons used, and spending `b` amount of money. The transition considers three choices for each fruit: skip it, buy it without a coupon, or buy it with a coupon if available. This approach efficiently explores all valid combinations within the given constraints.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(3^n) | O(n) | Exponentially explores all purchase/coupon options |
| Dynamic Programming | O(n * maxAmount * (maxCoupons + 1)) | O(maxAmount * (maxCoupons + 1)) | Uses DP table to track maximum tastiness efficiently |

## Algorithm Walkthrough

1. Initialize a DP array `dp` with dimensions `(maxCoupons + 1) x (maxAmount + 1)`, representing the maximum tastiness achievable with a given number of coupons used and budget spent. Initialize all entries to 0.
2. Iterate through each fruit in `price` and `tastiness`.
3. For each fruit, iterate over the number of coupons used in reverse from `maxCoupons` down to 0. This ensures we do not overwrite states needed for future transitions.
4. For each coupon count, iterate over budget from `maxAmount` down to 0.
5. Consider buying the fruit without a coupon if the budget allows: update `dp[c][b] = max(dp[c][b], dp[c][b - price[i]] + tastiness[i])`.
6. Consider buying the fruit with a coupon if at least one coupon remains and budget allows: update `dp[c][b] = max(dp[c][b], dp[c-1][b - price[i]//2] + tastiness[i])`.
7. Skip the fruit: `dp[c][b]` remains unchanged.
8. After processing all fruits, the answer is the maximum value in `dp[c][b]` for all `c` in `[0, maxCoupons]` and `b` in `[0, maxAmount]`.

This works because the DP array captures all possible combinations of budget and coupon usage up to each fruit, and taking the maximum ensures the tastiness is maximized.

## Python Solution

```python
from typing import List

class Solution:
    def maxTastiness(self, price: List[int], tastiness: List[int], maxAmount: int, maxCoupons: int) -> int:
        dp = [[0] * (maxAmount + 1) for _ in range(maxCoupons + 1)]
        
        for i in range(len(price)):
            for c in range(maxCoupons, -1, -1):
                for b in range(maxAmount, -1, -1):
                    # Buy without coupon
                    if b >= price[i]:
                        dp[c][b] = max(dp[c][b], dp[c][b - price[i]] + tastiness[i])
                    # Buy with coupon
                    if c > 0 and b >= price[i] // 2:
                        dp[c][b] = max(dp[c][b], dp[c - 1][b - price[i] // 2] + tastiness[i])
        
        return max(max(row) for row in dp)
```

In this implementation, the DP table is built iteratively. We iterate coupons and budgets in reverse to prevent overwriting values that are needed for subsequent calculations. We handle both coupon and non-coupon purchases by updating the DP table with the maximum tastiness possible for each state.

## Go Solution

```go
func maxTastiness(price []int, tastiness []int, maxAmount int, maxCoupons int) int {
    dp := make([][]int, maxCoupons+1)
    for i := range dp {
        dp[i] = make([]int, maxAmount+1)
    }

    for i := 0; i < len(price); i++ {
        for c := maxCoupons; c >= 0; c-- {
            for b := maxAmount; b >= 0; b-- {
                if b >= price[i] {
                    if dp[c][b] < dp[c][b-price[i]] + tastiness[i] {
                        dp[c][b] = dp[c][b-price[i]] + tastiness[i]
                    }
                }
                if c > 0 && b >= price[i]/2 {
                    if dp[c][b] < dp[c-1][b-price[i]/2] + tastiness[i] {
                        dp[c][b] = dp[c-1][b-price[i]/2] + tastiness[i]
                    }
                }
            }
        }
    }

    maxTast := 0
    for c := 0; c <= maxCoupons; c++ {
        for b := 0; b <= maxAmount; b++ {
            if dp[c][b] > maxTast {
                maxTast = dp[c][b]
            }
        }
    }
    return maxTast
}
```

The Go version mirrors the Python logic with explicit initialization of 2D slices. We iterate in reverse for coupons and budget to correctly maintain DP transitions. Integer division automatically rounds down, consistent with the coupon rule.

## Worked Examples

**Example 1:**

`price = [10,20,20], tastiness = [5,8,8], maxAmount = 20, maxCoupons = 1`

| Fruit | Coupons Used | Budget Left | Tastiness | Action |
| --- | --- | --- | --- | --- |
| 1 | 0 | 20 -> 10 | 5 | Buy without coupon |
| 2 | 1 | 10 -> 0 | 5 + 8 = 13 | Buy with coupon |
| 3 | 1 | 0 | 13 | Skip (cannot afford) |

`price = [10,15,7], tastiness = [5,8,20], maxAmount = 10, maxCoupons = 2`

| Fruit | Coupons Used | Budget Left | Tastiness | Action |
| --- | --- | --- | --- | --- |
| 1 | 0 | 10 | 0 | Skip |
| 2 | 1 | 10 -> 3 | 0 + 8 = 8 | Buy with coupon |
| 3 | 2 | 3 -> 0 | 8 + 20 = 28 | Buy with coupon |

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * maxAmount * (maxCoupons + 1)) | We iterate over all fruits, coupons, and budgets for DP updates |
| Space | O(maxAmount * (maxCoupons + 1)) | DP table stores max tastiness for each budget and coupon usage |

The complexity is feasible since `n <= 100`, `maxAmount <= 1000`, and `maxCoupons <= 5`.
