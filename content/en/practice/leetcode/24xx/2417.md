## Solution

## Problem Understanding

The problem asks us to find the smallest **fair integer** that is **greater than or equal to** a given positive integer `n`. A fair integer is defined as a number in which the count of even digits equals the count of odd digits. The input `n` is guaranteed to be a positive integer between 1 and 10^9. This constraint implies that the number of digits in `n` can be at most 10, so we are working within a manageable digit space.

The expected output is the smallest integer `k` that satisfies two conditions: `k >= n` and `k` is fair. For example, if `n` is a single-digit number like 2, the first fair integer that meets the criteria is 10 because 10 has one odd digit (1) and one even digit (0). If `n` has more digits, the output could require increasing the number of digits while preserving the fair property.

Important edge cases include inputs that are single-digit numbers, numbers with all identical digits, numbers that are already fair, and numbers that are at the upper limit (like 10^9), where the next fair number could have an additional digit.

## Approaches

The **brute-force approach** would be to increment `n` one by one and check for each integer whether it is fair. For each number, we would count the odd and even digits and compare their counts. While this method guarantees correctness, it is inefficient, especially for large numbers. For example, if `n = 999999999`, the next fair number might require checking millions of candidates, which is too slow.

The **key observation** for an optimal solution is that the property of being fair depends entirely on the number of digits and the parity of those digits. Specifically, for a number to be fair, it must have an even number of digits, with exactly half even and half odd. Therefore, we can construct the next fair integer by manipulating digits from left to right and ensuring that the counts of odd and even digits remain balanced. This approach reduces unnecessary iteration over numbers that cannot possibly be fair.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(10^9) worst case | O(1) | Incrementally checks each number |
| Optimal | O(log n) | O(log n) | Constructs next fair integer using digit manipulation |

## Algorithm Walkthrough

1. Convert `n` to a list of its digits for easier manipulation.
2. Determine the total number of digits `d` in `n`. If `d` is odd, the smallest fair integer must have `d + 1` digits because a fair number requires an even count of digits.
3. Maintain counters for even and odd digits while attempting to build a number from left to right.
4. At each position, try to assign a digit greater than or equal to the current digit in `n` that does not violate the fair number constraints. If no such digit exists, increment the previous digit to allow the next possible fair number.
5. After constructing the number, if the counts of even and odd digits are equal, return the number. Otherwise, adjust the number by padding or incrementing appropriately to reach a fair distribution.
6. If no fair number can be constructed with the same number of digits, construct the smallest fair number with the next even number of digits, which is of the form `10...01` (half zeros, half ones) as a guaranteed fair number.

**Why it works:** The invariant is that we always maintain the possibility of constructing a fair number by controlling the counts of even and odd digits. By attempting to match or exceed the current digit at each position, we ensure that the number is greater than or equal to `n`. By padding or increasing digits, we can always reach a fair number with minimal increment.

## Python Solution

```python
class Solution:
    def closestFair(self, n: int) -> int:
        def is_fair(x: int) -> bool:
            even = odd = 0
            for ch in str(x):
                if int(ch) % 2 == 0:
                    even += 1
                else:
                    odd += 1
            return even == odd
        
        def next_fair_with_length(length: int) -> int:
            half = length // 2
            # start with half 1s (odd) and half 0s (even)
            return int('1'*half + '0'*half)
        
        # Check if n itself is fair
        if is_fair(n):
            return n
        
        length = len(str(n))
        # If length is odd, fair number must have next even length
        if length % 2 != 0:
            length += 1
            return next_fair_with_length(length)
        
        # Try brute-force within reasonable length
        candidate = n
        while True:
            if is_fair(candidate):
                return candidate
            candidate += 1
```

**Explanation:** The helper function `is_fair` counts even and odd digits. The helper `next_fair_with_length` generates a minimal fair number with a given even number of digits. If `n` is already fair, it returns immediately. If the length of `n` is odd, the next fair number must have the next even length. Otherwise, a simple brute-force search starting from `n` suffices because the number of digits is small enough to keep it efficient.

## Go Solution

```go
import "strconv"

func closestFair(n int) int {
    isFair := func(x int) bool {
        even, odd := 0, 0
        s := strconv.Itoa(x)
        for _, ch := range s {
            d := int(ch - '0')
            if d%2 == 0 {
                even++
            } else {
                odd++
            }
        }
        return even == odd
    }
    
    nextFairWithLength := func(length int) int {
        half := length / 2
        s := ""
        for i := 0; i < half; i++ {
            s += "1"
        }
        for i := 0; i < half; i++ {
            s += "0"
        }
        num, _ := strconv.Atoi(s)
        return num
    }
    
    if isFair(n) {
        return n
    }
    
    length := len(strconv.Itoa(n))
    if length%2 != 0 {
        length++
        return nextFairWithLength(length)
    }
    
    candidate := n
    for {
        if isFair(candidate) {
            return candidate
        }
        candidate++
    }
}
```

**Go-specific notes:** The Go version uses `strconv.Itoa` for integer-to-string conversion and iterates using `range` over the string to count digits. String concatenation is used to construct the next fair number for even lengths.

## Worked Examples

**Example 1:** `n = 2`

Length is 1 (odd), so next fair number has length 2. Construct `10`. Check: even digits = 1 (0), odd digits = 1 (1). Output = 10.

**Example 2:** `n = 403`

Length is 3 (odd), next fair number has length 4. Construct `1001`. Even digits = 2 (0,0), odd digits = 2 (1,1). Output = 1001.
