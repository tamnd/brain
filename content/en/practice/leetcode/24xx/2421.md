**Topics:** Array, Hash Table, Tree, Union-Find, Graph Theory, Sorting

## Solution

## Problem Understanding

The problem presents a tree with `n` nodes numbered from `0` to `n - 1` and `n - 1` edges, which ensures the graph is connected and acyclic. Each node has an associated value given in the array `vals`. We are asked to count the number of **good paths**, defined as paths where the start and end nodes have the same value, and all nodes along the path do not exceed this value.

The inputs consist of `vals`, an array of node values, and `edges`, a list of undirected connections between nodes. The output is a single integer representing the number of distinct good paths. Importantly, a path and its reverse are considered identical, and a single node is trivially a good path.

Constraints indicate that the tree can be moderately large, up to 30,000 nodes, with node values ranging from 0 to 100,000. This eliminates naive brute-force approaches that would enumerate all possible paths, as the number of paths in a tree can be exponential in `n`.

Edge cases that could complicate a naive implementation include trees where all node values are equal, trees with only a single node, and cases with highly skewed value distributions where connections only allow very limited paths.

## Approaches

A brute-force approach would attempt to enumerate all paths in the tree and then check whether each path satisfies the good path conditions. While this would be correct, it is computationally infeasible due to the exponential number of paths. Traversing each path could require exploring combinatorially many sequences of nodes, which would easily exceed the time limit for `n = 30,000`.

The key insight for an optimal solution is to use **Union-Find (Disjoint Set Union)** combined with **sorting nodes by value**. Because the path must have all intermediate nodes with values not exceeding the endpoints, we can process nodes in increasing order of value. For each value `v`, we connect nodes whose neighbors have values less than or equal to `v` using Union-Find. Once all nodes of a specific value are connected in components, we can count the number of pairs of nodes with the same value in each component. This approach leverages the tree structure and value constraints to avoid enumerating paths explicitly.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) or worse | O(n) | Enumerate all paths and check the good path condition; infeasible for large `n` |
| Optimal | O(n log n) | O(n) | Process nodes by value, connect neighbors with Union-Find, count combinations in each component |

## Algorithm Walkthrough

1. Build an adjacency list representation of the tree from `edges` for efficient neighbor access.
2. Map node values to the list of nodes that have that value. This allows processing nodes in increasing value order.
3. Initialize a Union-Find data structure to manage connected components.
4. Sort the unique values from `vals` in ascending order.
5. For each value `v` in sorted order:

1. For each node with value `v`, iterate through its neighbors.
2. If a neighbor’s value is less than or equal to `v`, union the current node with the neighbor in the Union-Find structure.
3. After all unions, count nodes with value `v` in each connected component. Each component contributes `(count * (count - 1)) // 2` good paths plus `count` for single-node paths.
6. Sum contributions from all values to obtain the total number of good paths.

**Why it works:** By processing nodes in ascending order and only connecting nodes to neighbors with values less than or equal to the current value, we maintain the invariant that all nodes in a Union-Find component have values ≤ the component value. Counting nodes of the current value in each component ensures that only valid good paths are considered.

## Python Solution

```python
from collections import defaultdict
from typing import List

class UnionFind:
    def __init__(self, size: int):
        self.parent = list(range(size))
        self.rank = [0] * size

    def find(self, x: int) -> int:
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x: int, y: int) -> None:
        px, py = self.find(x), self.find(y)
        if px == py:
            return
        if self.rank[px] < self.rank[py]:
            self.parent[px] = py
        elif self.rank[px] > self.rank[py]:
            self.parent[py] = px
        else:
            self.parent[py] = px
            self.rank[px] += 1

class Solution:
    def numberOfGoodPaths(self, vals: List[int], edges: List[List[int]]) -> int:
        n = len(vals)
        adj = [[] for _ in range(n)]
        for u, v in edges:
            adj[u].append(v)
            adj[v].append(u)
        
        val_to_nodes = defaultdict(list)
        for i, val in enumerate(vals):
            val_to_nodes[val].append(i)
        
        uf = UnionFind(n)
        res = 0
        
        for val in sorted(val_to_nodes.keys()):
            for node in val_to_nodes[val]:
                for neighbor in adj[node]:
                    if vals[neighbor] <= val:
                        uf.union(node, neighbor)
            
            count = defaultdict(int)
            for node in val_to_nodes[val]:
                parent = uf.find(node)
                count[parent] += 1
            
            for c in count.values():
                res += c * (c + 1) // 2
        
        return res
```

The Python implementation starts by building an adjacency list for efficient neighbor queries. We map values to their corresponding nodes and initialize a Union-Find data structure. Iterating in sorted order ensures that when we union neighbors with values less than or equal to the current value, we maintain valid paths. The counting step uses a hash map to tally nodes per connected component, and the combination formula computes the number of good paths.

## Go Solution

```go
package main

func numberOfGoodPaths(vals []int, edges [][]int) int {
    n := len(vals)
    adj := make([][]int, n)
    for _, e := range edges {
        u, v := e[0], e[1]
        adj[u] = append(adj[u], v)
        adj[v] = append(adj[v], u)
    }

    valToNodes := make(map[int][]int)
    for i, val := range vals {
        valToNodes[val] = append(valToNodes[val], i)
    }

    parent := make([]int, n)
    rank := make([]int, n)
    for i := 0; i < n; i++ {
        parent[i] = i
    }

    var find func(int) int
    find = func(x int) int {
        if parent[x] != x {
            parent[x] = find(parent[x])
        }
        return parent[x]
    }

    union := func(x, y int) {
        px, py := find(x), find(y)
        if px == py {
            return
        }
        if rank[px] < rank[py] {
            parent[px] = py
        } else if rank[px] > rank[py] {
            parent[py] = px
        } else {
            parent[py] = px
            rank[px]++
        }
    }

    keys := make([]int, 0, len(valToNodes))
    for k := range valToNodes {
        keys = append(keys, k)
    }
    sort.Ints(keys)

    res := 0
    for _, val := range keys {
        for _, node := range valToNodes[val] {
            for _, nei := range adj[node] {
                if vals[nei] <= val {
                    union(node, nei)
                }
            }
        }
        count := make(map[int]int)
        for _, node := range valToNodes[val] {
            p := find(node)
            count[p]++
        }
        for _, c := range count {
            res += c * (c + 1) / 2
        }
    }
    return res
}
```

In Go, slices and maps are used instead of lists and dictionaries. The union-find operations are implemented similarly, with recursion for path compression. Sorting and counting operations mirror the Python version. Go requires explicit handling of integer division and initialization of slices for adjacency lists.

- Union neighbors: node 1 connects to 0 → union; node 4 connects to 2 → union
- Component includes nodes with value 3 → two nodes 1 and 4 in same component → contributes 3 good paths
