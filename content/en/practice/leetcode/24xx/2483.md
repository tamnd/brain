## Solution

## Problem Understanding

The problem asks us to determine the optimal closing time for a shop to minimize a penalty based on customer presence. The input is a string `customers` of length `n`, where each character is either `'Y'` or `'N'`. A `'Y'` at position `i` indicates that customers arrive at hour `i`, while `'N'` indicates no customers arrive. The shop can close at any hour `j` between `0` and `n` inclusive. The penalty for closing at hour `j` is calculated by adding one for every hour the shop is open with no customers (`'N'` before `j`) and one for every hour the shop is closed with customers (`'Y'` from `j` onward). The goal is to find the **earliest hour** with the minimum penalty.

The constraints specify that the length of `customers` can go up to `10^5`, which implies that any solution must run in linear time, as a brute-force approach calculating the penalty for every possible closing hour naively would be too slow.

Important edge cases include strings with all `'Y'`, all `'N'`, or alternating `'Y'` and `'N'`. These cases test whether the solution correctly handles extreme penalties and chooses the earliest closing time when multiple times yield the same penalty.

## Approaches

The brute-force approach would be to iterate through every possible closing hour `j` and calculate the penalty explicitly by summing the `'N'`s before `j` and `'Y'`s from `j` onward. While this gives the correct answer, it is inefficient because it requires O(n^2) time for strings of length `n`.

The optimal approach relies on using a **prefix sum of penalties**. We initialize the penalty assuming the shop closes at hour `0`, which means the shop is closed for all hours and each `'Y'` contributes one to the penalty. Then, as we iterate through the string, we update the penalty dynamically: if the current hour has `'N'`, the penalty decreases because the shop is now open during a no-customer hour; if the current hour has `'Y'`, the penalty increases as this `'Y'` is now before the closing hour. By tracking the minimum penalty and the corresponding hour, we can find the optimal closing time in O(n) time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Calculates penalty for each closing hour by iterating over the string every time |
| Optimal | O(n) | O(1) | Uses a running penalty counter and updates dynamically, tracking the minimum |

## Algorithm Walkthrough

1. Initialize `penalty` as the number of `'Y'` in the string, representing closing at hour `0`.
2. Set `min_penalty` equal to `penalty` and `best_hour` equal to `0`.
3. Iterate over each hour `i` in `customers`.
4. For each hour, if the character is `'Y'`, decrement the `penalty` by `1` because the hour moves from closed to open. If the character is `'N'`, increment the `penalty` by `1` because the hour moves from open to closed.
5. If the updated `penalty` is less than `min_penalty`, update `min_penalty` and set `best_hour` to `i + 1` (the next hour is the new closing time).
6. After iterating, return `best_hour`.

**Why it works**: The algorithm maintains a running penalty that accurately reflects the consequences of closing the shop at each hour. The invariant is that `penalty` always represents the penalty if the shop were to close **after the current hour**, allowing us to update the minimum efficiently in a single pass.

## Python Solution

```python
class Solution:
    def bestClosingTime(self, customers: str) -> int:
        penalty = customers.count('Y')  # penalty if closing at hour 0
        min_penalty = penalty
        best_hour = 0
        
        for i, c in enumerate(customers):
            if c == 'Y':
                penalty -= 1  # now open during a customer hour reduces penalty
            else:
                penalty += 1  # now open during a no-customer hour increases penalty
                
            if penalty < min_penalty:
                min_penalty = penalty
                best_hour = i + 1
        
        return best_hour
```

The implementation first calculates the initial penalty for closing at hour `0` by counting all `'Y'`s. Then, it iterates through the string, updating the penalty dynamically according to whether the hour has customers or not, tracking the minimum penalty and its corresponding closing time.

## Go Solution

```go
func bestClosingTime(customers string) int {
    penalty := 0
    for _, c := range customers {
        if c == 'Y' {
            penalty++
        }
    }
    minPenalty := penalty
    bestHour := 0
    
    for i, c := range customers {
        if c == 'Y' {
            penalty--
        } else {
            penalty++
        }
        if penalty < minPenalty {
            minPenalty = penalty
            bestHour = i + 1
        }
    }
    
    return bestHour
}
```

The Go implementation mirrors the Python version. The primary differences are syntax-related: using `range` for iteration, explicit type conversion is not needed here, and Go uses `int` for indexes. The logic for updating the penalty and tracking the best hour is identical.

## Edge Cases

One edge case is when the string contains all `'N'`. Here, the penalty is zero if the shop closes immediately, and the algorithm must correctly return `0`.

Another edge case is when the string contains all `'Y'`. The minimum penalty occurs when the shop closes at the very end, and the algorithm must return `n` rather than an earlier hour.

A third edge case involves alternating `'Y'` and `'N'` characters. This could cause naive implementations to miscount the penalty if they do not update it correctly at each hour. The dynamic penalty update in the optimal solution correctly handles this by incrementing or decrementing the penalty at each step, ensuring accurate tracking and selection of the earliest minimum.
