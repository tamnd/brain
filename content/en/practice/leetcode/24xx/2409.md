## Problem Understanding

The problem asks us to determine the total number of days that Alice and Bob are **both present in Rome at the same time**. Each person has a given range of dates during which they are in the city, represented as `"MM-DD"` strings. The ranges are inclusive, meaning that if Alice arrives on `"08-15"` and leaves on `"08-18"`, she is present for **four days**: August 15, 16, 17, and 18. The goal is to calculate the intersection of Alice's and Bob's date ranges, and return the number of overlapping days.

The input guarantees that all dates are valid and occur in the same non-leap year. This allows us to rely on a fixed number of days per month: `[31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31]`. Each person's arrival date is earlier than or equal to their leaving date, so we do not need to handle inverted ranges.

Important edge cases include situations where the two date ranges do not overlap at all, overlap by only one day, or overlap across months. The solution must handle all of these scenarios correctly.

## Approaches

A **brute-force approach** would enumerate every day of the year and mark whether Alice and Bob are present on each day. We could count the days where both are present. While this would give the correct answer, it is inefficient because it would require iterating over every day of the year even when the ranges are short. This results in a time complexity of **O(365)** and extra space of **O(365)** if we use an array to mark presence.

A more **optimal approach** leverages the fact that each person’s stay is a continuous date range. We only need to find the latest arrival date and the earliest leaving date between Alice and Bob. The number of overlapping days is simply the difference between these two dates plus one, clamped to zero if there is no overlap. This approach does not require enumerating individual days, and it works entirely with date arithmetic.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(365) | O(365) | Iterate through all days of the year, mark presence for each person, count overlap |
| Optimal | O(1) | O(1) | Compute max arrival and min leaving, then calculate overlap directly |

## Algorithm Walkthrough

1. Convert the `"MM-DD"` strings into **day-of-year integers**. This allows us to perform simple arithmetic on dates without dealing with months and days separately. For example, `"01-01"` becomes `1`, `"02-01"` becomes `32`, etc. Precompute the cumulative number of days before each month to facilitate this conversion.
2. Compute Alice’s arrival and leaving days as integers.
3. Compute Bob’s arrival and leaving days as integers.
4. Determine the overlapping interval by taking the **maximum** of the two arrival days as the start and the **minimum** of the two leaving days as the end.
5. If the start is greater than the end, there is no overlap, so return `0`. Otherwise, return `end - start + 1`, which accounts for inclusivity of both the start and end dates.

**Why it works:** By converting dates into day-of-year integers, we reduce the problem to simple numeric range intersection. Since both arrival-to-leave intervals are contiguous, finding the intersection as `max(start1, start2)` to `min(end1, end2)` guarantees correct counting. Clamping negative intervals to zero ensures correctness when there is no overlap.

## Python Solution

```python
class Solution:
    def countDaysTogether(self, arriveAlice: str, leaveAlice: str, arriveBob: str, leaveBob: str) -> int:
        days_in_month = [31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31]
        # Precompute cumulative days to get day-of-year easily
        cum_days = [0]
        for d in days_in_month:
            cum_days.append(cum_days[-1] + d)
        
        def to_day_of_year(date: str) -> int:
            month = int(date[:2])
            day = int(date[3:])
            return cum_days[month - 1] + day
        
        arriveA = to_day_of_year(arriveAlice)
        leaveA = to_day_of_year(leaveAlice)
        arriveB = to_day_of_year(arriveBob)
        leaveB = to_day_of_year(leaveBob)
        
        start = max(arriveA, arriveB)
        end = min(leaveA, leaveB)
        
        return max(0, end - start + 1)
```

The code first computes cumulative days per month to quickly convert `"MM-DD"` into a day-of-year integer. The helper function `to_day_of_year` is used for this conversion. After converting both Alice and Bob’s dates, the algorithm computes the overlapping interval and returns the count, or `0` if there is no overlap.

## Go Solution

```go
func countDaysTogether(arriveAlice string, leaveAlice string, arriveBob string, leaveBob string) int {
    daysInMonth := []int{31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31}
    cumDays := make([]int, 13)
    for i := 1; i <= 12; i++ {
        cumDays[i] = cumDays[i-1] + daysInMonth[i-1]
    }
    
    toDayOfYear := func(date string) int {
        month := int((date[0]-'0')*10 + (date[1]-'0'))
        day := int((date[3]-'0')*10 + (date[4]-'0'))
        return cumDays[month-1] + day
    }
    
    arriveA := toDayOfYear(arriveAlice)
    leaveA := toDayOfYear(leaveAlice)
    arriveB := toDayOfYear(arriveBob)
    leaveB := toDayOfYear(leaveBob)
    
    start := max(arriveA, arriveB)
    end := min(leaveA, leaveB)
    
    if start > end {
        return 0
    }
    return end - start + 1
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

The Go implementation mirrors the Python approach, using slices for cumulative days and helper functions for min and max. Date conversion is performed using byte arithmetic instead of string parsing functions.
