## Solution

## Problem Understanding

The problem asks us to find the longest contiguous subarray whose bitwise AND is as large as possible among all subarrays.

For every possible non-empty subarray, we can compute its bitwise AND. Among all those AND values, there exists a maximum value `k`. Once that maximum is determined, we only care about subarrays whose AND equals `k`. The final answer is the length of the longest such subarray.

Recall how bitwise AND behaves. When we AND multiple numbers together, bits can only stay the same or turn from `1` to `0`. Adding more elements to a subarray can never increase the AND value. It either stays the same or decreases.

The input is an integer array `nums` with up to `10^5` elements. Each value is between `1` and `10^6`. These constraints immediately suggest that any algorithm slower than roughly linear or near-linear time will likely time out. In particular, an `O(n^2)` solution becomes too expensive when `n = 100000`.

An important observation is that a single element subarray is allowed. Therefore, the maximum possible AND value can never exceed the maximum element in the array, because every subarray AND is bounded above by each of its elements.

Several edge cases are important:

- If the maximum value appears only once, the answer is `1`.
- If all elements are equal, the entire array is the answer.
- If the maximum value appears in multiple disconnected segments, we must return the longest consecutive segment.
- Arrays of length `1` should correctly return `1`.

## Approaches

### Brute Force Approach

A brute force solution would examine every possible subarray. For each subarray, we compute the bitwise AND and track:

1. The maximum AND value seen so far.
2. The longest subarray achieving that value.

We can generate all subarrays using two nested loops. For each starting index, we extend the subarray to the right while maintaining the running AND.

This approach is correct because it explicitly checks every valid subarray and therefore cannot miss the optimal answer.

However, there are `O(n^2)` subarrays. Even if we maintain the AND incrementally, the total runtime is still quadratic. With `n = 100000`, this is far too slow.

### Key Insight for the Optimal Solution

The crucial observation is that the maximum possible bitwise AND of any subarray is simply the maximum element in the array.

Why?

- A subarray AND can never exceed any individual element in that subarray.
- Since single-element subarrays are allowed, the maximum array element itself is always achievable.

Suppose the maximum value in the array is `max_value`.

For a subarray AND to equal `max_value`, every element in that subarray must also equal `max_value`. If we include any smaller number, some bit that exists in `max_value` will be cleared, producing a smaller AND result.

Therefore, the problem reduces to finding the longest consecutive run of the maximum element.

This transforms the problem into a simple linear scan.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every subarray and computes AND values |
| Optimal | O(n) | O(1) | Finds the longest consecutive segment of the maximum value |

## Algorithm Walkthrough

1. First, find the maximum value in the array using a linear scan or the built-in `max()` function.
2. Initialize two variables:

- `current_length`, representing the current streak of consecutive maximum values.
- `best_length`, representing the longest streak seen so far.
3. Traverse the array from left to right.
4. For each number:

- If the number equals the maximum value, increment `current_length`.
- Otherwise, reset `current_length` to `0`, because the streak has been broken.
5. After updating `current_length`, compare it with `best_length` and update `best_length` if needed.
6. After the traversal finishes, return `best_length`.

### Why it works

The maximum AND achievable by any subarray equals the maximum element in the array. Any subarray containing a smaller element would produce an AND smaller than that maximum. Therefore, the only valid subarrays are contiguous blocks consisting entirely of the maximum value. The algorithm simply finds the longest such block, which guarantees correctness.

## Python Solution

```python
from typing import List

class Solution:
    def longestSubarray(self, nums: List[int]) -> int:
        max_value = max(nums)

        best_length = 0
        current_length = 0

        for num in nums:
            if num == max_value:
                current_length += 1
                best_length = max(best_length, current_length)
            else:
                current_length = 0

        return best_length
```

The implementation begins by computing the largest element in the array. This value represents the maximum possible AND of any subarray.

Next, the algorithm scans through the array once. The variable `current_length` tracks the length of the current consecutive sequence of maximum values. Whenever a non-maximum value appears, the streak is reset because any subarray containing that value cannot achieve the maximum AND.

The variable `best_length` stores the longest streak encountered during the scan. At the end, it contains the answer.

The solution uses constant extra memory and processes each element exactly once.

## Go Solution

```go
func longestSubarray(nums []int) int {
    maxValue := nums[0]

    for _, num := range nums {
        if num > maxValue {
            maxValue = num
        }
    }

    bestLength := 0
    currentLength := 0

    for _, num := range nums {
        if num == maxValue {
            currentLength++

            if currentLength > bestLength {
                bestLength = currentLength
            }
        } else {
            currentLength = 0
        }
    }

    return bestLength
}
```

The Go implementation follows the same logic as the Python version. Since Go does not provide a built-in `max()` function for slices, we manually scan the array to determine the maximum value.

The solution uses integer variables for the streak counters and performs a single traversal after determining the maximum. No additional data structures are required.

## Edge Cases

One important edge case is an array containing only one element. Since single-element subarrays are allowed, that lone element automatically forms the maximum AND subarray. The implementation handles this naturally because the longest streak of the maximum value will be `1`.

Another important case is when the maximum value appears multiple times but never consecutively. For example, `[9,1,9,1,9]` contains three occurrences of the maximum value, but each is isolated. A buggy implementation might incorrectly count total occurrences instead of consecutive occurrences. The algorithm avoids this by resetting the streak whenever a non-maximum value appears.

A third important edge case occurs when every element equals the maximum value, such as `[7,7,7,7]`. In this situation, the entire array should be returned. Since the algorithm continuously increments the streak without resetting it, the final answer correctly becomes the full array length.

Another subtle case is when the longest streak appears at the beginning or end of the array. Some implementations incorrectly update the answer only after encountering a reset. This solution updates `best_length` during every successful extension of a streak, ensuring boundary streaks are handled correctly.
