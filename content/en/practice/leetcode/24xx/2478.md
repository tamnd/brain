# LeetCode 2478 - Number of Beautiful Partitions

## Problem Understanding

The problem gives us a string `s` containing only digits from `'1'` to `'9'`. We must divide this string into exactly `k` contiguous substrings such that every substring satisfies two conditions:

1. Its length is at least `minLength`.
2. It starts with a prime digit and ends with a non-prime digit.

The prime digits are specifically:

- `'2'`
- `'3'`
- `'5'`
- `'7'`

All other digits are considered non-prime.

The substrings must completely partition the original string without overlap or gaps. That means every character in `s` belongs to exactly one substring.

The output is the total number of valid ways to create such a partitioning. Since the number of possibilities can become extremely large, the result must be returned modulo:

```
10^9 + 7
```

The constraints are important:

- `s.length <= 1000`
- `k <= 1000`

These limits immediately tell us that brute force enumeration of all possible partitions is infeasible. The number of ways to split a string grows exponentially. We therefore need a dynamic programming solution.

Several edge cases are especially important:

- If the first character is not prime, no valid partition can exist because every substring must start with a prime digit.
- If the last character is prime, no valid partition can exist because every substring must end with a non-prime digit.
- If `k * minLength > len(s)`, it is impossible to create enough substrings while satisfying the minimum length requirement.
- Partition boundaries can only occur between a non-prime digit and a prime digit, because one substring must end with a non-prime digit and the next substring must start with a prime digit.

Understanding these structural constraints is the key to building an efficient solution.

## Approaches

### Brute Force Approach

A straightforward approach is to try every possible way to place `k - 1` cuts in the string. For each partitioning, we would verify:

- Every substring has length at least `minLength`
- Every substring starts with a prime digit
- Every substring ends with a non-prime digit

This approach is correct because it explicitly checks every possible partition configuration.

However, it is far too slow. A string of length `n` has exponentially many partition possibilities. Even for moderate values of `n`, the number of combinations becomes enormous.

Since `n` can reach `1000`, brute force is completely impractical.

### Optimal Dynamic Programming Approach

The key observation is that valid partitions can only begin at certain indices.

Suppose a substring ends at position `i - 1`. Then the next substring begins at position `i`. For this to be valid:

- `s[i - 1]` must be non-prime
- `s[i]` must be prime

This means only certain indices are legal partition boundaries.

We can define dynamic programming states based on:

- How many substrings have already been formed
- Where the current substring begins

To optimize transitions, we use prefix sums so that instead of iterating over every previous partition point repeatedly, we can aggregate valid counts efficiently.

This reduces the complexity from exponential time to polynomial time.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Exponential | O(k) | Tries every possible partition |
| Optimal DP + Prefix Sum | O(k * n) | O(n) | Efficiently counts valid partitions |

## Algorithm Walkthrough

### Step 1: Identify Prime Digits

We first create a helper function to determine whether a digit is prime.

The valid prime digits are:

```
2, 3, 5, 7
```

This helper will be used repeatedly throughout the algorithm.

### Step 2: Handle Impossible Cases Early

Before running dynamic programming, we check several conditions:

1. The first character must be prime.
2. The last character must be non-prime.
3. The string must be long enough to fit `k` substrings of length `minLength`.

If any of these conditions fail, the answer is immediately `0`.

### Step 3: Define Valid Partition Boundaries

A new substring can begin at index `i` only if:

- `s[i]` is prime
- `s[i - 1]` is non-prime

These are the only legal cut positions.

This dramatically reduces the number of valid transitions.

### Step 4: Define the DP State

Let:

```
dp[i]
```

represent the number of ways to partition the prefix ending before index `i`.

We iteratively build solutions for:

- 1 substring
- 2 substrings
- ...
- k substrings

For each layer, we compute a new DP array.

### Step 5: Use Prefix Sums to Optimize Transitions

Without optimization, for every position we would need to scan all previous valid cut positions.

That would lead to O(n²) transitions.

Instead, we maintain a running prefix sum of valid previous DP states.

Whenever an index can serve as the start of a new substring, we add its DP contribution into the prefix accumulator.

This allows each transition to be computed in O(1) time.

### Step 6: Build the DP Iteratively

For each partition count:

1. Maintain a running prefix sum.
2. Iterate through possible ending positions.
3. Update DP values only when:

- The substring length requirement is satisfied
- The substring ends with a non-prime digit

### Step 7: Return the Final Answer

After processing exactly `k` substrings, the answer is the number of ways to partition the entire string.

## Why it works

The algorithm works because every DP state represents a complete count of valid ways to partition a prefix of the string. Transitions only occur across legal partition boundaries, guaranteeing that every generated substring satisfies the start and end conditions. Prefix sums preserve correctness because they aggregate all previously valid partition counts without omitting or duplicating any configurations.

## Python Solution

```python
from typing import List

class Solution:
    def beautifulPartitions(self, s: str, k: int, minLength: int) -> int:
        MOD = 10**9 + 7
        n = len(s)

        prime_digits = {'2', '3', '5', '7'}

        def is_prime(ch: str) -> bool:
            return ch in prime_digits

        # Impossible cases
        if not is_prime(s[0]):
            return 0

        if is_prime(s[-1]):
            return 0

        if k * minLength > n:
            return 0

        # can_start[i] means index i can start a substring
        can_start = [False] * n
        can_start[0] = True

        for i in range(1, n):
            if not is_prime(s[i - 1]) and is_prime(s[i]):
                can_start[i] = True

        # dp[i] = ways to partition prefix ending before i
        dp = [0] * (n + 1)
        dp[0] = 1

        for part in range(1, k + 1):
            new_dp = [0] * (n + 1)

            prefix_sum = 0

            # minimum start index for current partition
            start_min = (part - 1) * minLength

            # ending positions
            for end in range(part * minLength, n + 1):

                start_index = end - minLength

                if can_start[start_index]:
                    prefix_sum = (prefix_sum + dp[start_index]) % MOD

                # substring must end with non-prime
                if not is_prime(s[end - 1]):
                    new_dp[end] = prefix_sum

            dp = new_dp

        return dp[n]
```

The implementation begins by checking impossible scenarios. This avoids unnecessary DP computation for cases that can never produce a valid answer.

The `can_start` array precomputes every index where a substring is allowed to begin. This is important because partition boundaries are highly restricted.

The DP array stores counts for prefixes. Instead of maintaining a full two-dimensional DP table, the solution only keeps the previous layer, which reduces memory usage.

For every partition count, the algorithm iterates through valid ending positions. The `prefix_sum` variable accumulates all valid DP states that can transition into the current substring.

Whenever a substring ending is valid, meaning its last digit is non-prime, the current prefix sum becomes the number of valid ways to reach that endpoint.

Finally, `dp[n]` contains the total number of valid beautiful partitions for the entire string.

## Go Solution

```go
package main

func beautifulPartitions(s string, k int, minLength int) int {
	const MOD = 1000000007

	n := len(s)

	isPrime := func(ch byte) bool {
		return ch == '2' || ch == '3' || ch == '5' || ch == '7'
	}

	// Impossible cases
	if !isPrime(s[0]) {
		return 0
	}

	if isPrime(s[n-1]) {
		return 0
	}

	if k*minLength > n {
		return 0
	}

	canStart := make([]bool, n)
	canStart[0] = true

	for i := 1; i < n; i++ {
		if !isPrime(s[i-1]) && isPrime(s[i]) {
			canStart[i] = true
		}
	}

	dp := make([]int, n+1)
	dp[0] = 1

	for part := 1; part <= k; part++ {
		newDP := make([]int, n+1)

		prefixSum := 0

		for end := part * minLength; end <= n; end++ {
			startIndex := end - minLength

			if canStart[startIndex] {
				prefixSum = (prefixSum + dp[startIndex]) % MOD
			}

			if !isPrime(s[end-1]) {
				newDP[end] = prefixSum
			}
		}

		dp = newDP
	}

	return dp[n]
}
```

The Go implementation follows the same logic as the Python version. Since Go does not have Python-style sets for constant membership checks, prime digit validation is implemented using direct comparisons.

Slices are used instead of arrays because the size depends on the input length. Integer overflow is avoided by applying modulo operations during every addition.

Go strings are byte indexed, which works perfectly here because the input contains only ASCII digits.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(k * n) | Each DP layer scans the string once |
| Space | O(n) | Only two DP arrays are stored |

The optimization comes from replacing nested transition loops with prefix sums. Without this improvement, the solution would require O(k * n²) time. Since each DP layer processes the string in linear time, the final complexity becomes O(k * n), which is efficient for `n <= 1000`.

## Edge Cases

One important edge case occurs when the string starts with a non-prime digit. Since every substring must begin with a prime digit, the very first substring would already violate the rules. The implementation handles this immediately with an early return.

Another critical edge case is when the string ends with a prime digit. Every substring must end with a non-prime digit, including the final substring. If the last character is prime, no valid partition exists. The solution checks this before starting dynamic programming.

A third important edge case is when the required total minimum length exceeds the size of the string. If we need `k` substrings and each must contain at least `minLength` characters, then the string must contain at least `k * minLength` characters. The implementation explicitly checks this condition and returns `0` if the string is too short.

Another subtle edge case involves partition boundaries. A naive implementation might allow cuts anywhere, but valid cuts can only occur where a non-prime digit is followed by a prime digit. The `can_start` array guarantees that the DP transitions only use legal partition starts, preventing invalid substring constructions.
