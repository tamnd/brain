## Solution

## Problem Understanding

The problem asks us to find the number of pairs `(i, j)` in two integer arrays `nums1` and `nums2` such that `i < j` and the inequality `nums1[i] - nums1[j] <= nums2[i] - nums2[j] + diff` holds. Essentially, for every pair of indices, we are comparing the differences of the elements in `nums1` to the differences in `nums2`, adjusted by the given `diff`.

The input arrays `nums1` and `nums2` are of equal length `n` with a size constraint up to `10^5`, meaning a brute-force solution that checks all pairs (which is `O(n^2)`) would be too slow. The arrays can contain negative numbers and `diff` itself can also be negative.

Key edge cases to be aware of include arrays where all elements are equal, cases where `diff` is very large or very negative, and very small arrays with size 2. The problem guarantees valid arrays of the same length with at least two elements.

## Approaches

The **brute-force approach** involves iterating over all pairs `(i, j)` with `i < j` and checking whether the inequality holds. This guarantees correctness because it literally checks every possibility, but its time complexity is `O(n^2)` which is infeasible for `n` up to `10^5`.

The **optimal approach** comes from observing that the inequality can be rewritten as:

```
nums1[i] - nums2[i] <= nums1[j] - nums2[j] + diff
```

Define a new array `arr[i] = nums1[i] - nums2[i]`. Then the inequality becomes:

```
arr[i] <= arr[j] + diff
```

We are now counting the number of pairs `(i, j)` such that `i < j` and `arr[i] <= arr[j] + diff`. This is a classic problem that can be solved using a modified **merge sort** to count the number of valid pairs efficiently in `O(n log n)` time, similar to counting inversions. Alternative approaches include using a **Binary Indexed Tree (Fenwick Tree)** or a **Segment Tree** for efficient prefix sum queries.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check all pairs directly |
| Merge Sort / BIT / Segment Tree | O(n log n) | O(n) | Count pairs efficiently by sorting or prefix sums |

## Algorithm Walkthrough

1. Transform the input arrays into a single array `arr[i] = nums1[i] - nums2[i]`. This reduces the problem to counting pairs `(i, j)` where `i < j` and `arr[i] <= arr[j] + diff`.
2. Use a **merge sort-based counting** approach:

- Divide the array into two halves recursively.
- Before merging, for each element in the left half, count how many elements in the right half satisfy `arr[i] <= arr[j] + diff`.
- Accumulate this count.
3. Merge the two halves while maintaining sorted order to allow efficient counting in subsequent recursive steps.
4. Return the total count from all recursive merges.

**Why it works**: Merge sort ensures that we are always comparing elements in the left half with those in the right half, which respects the `i < j` condition. Counting during the merge step ensures we efficiently account for all valid pairs without double counting.

## Python Solution

```python
from typing import List

class Solution:
    def numberOfPairs(self, nums1: List[int], nums2: List[int], diff: int) -> int:
        arr = [nums1[i] - nums2[i] for i in range(len(nums1))]
        
        def merge_count(nums: List[int]) -> int:
            if len(nums) <= 1:
                return 0
            mid = len(nums) // 2
            left = nums[:mid]
            right = nums[mid:]
            count = merge_count(left) + merge_count(right)
            
            j = 0
            for i in range(len(left)):
                while j < len(right) and left[i] > right[j] + diff:
                    j += 1
                count += len(right) - j
            
            # merge step
            l = r = 0
            sorted_nums = []
            while l < len(left) and r < len(right):
                if left[l] <= right[r]:
                    sorted_nums.append(left[l])
                    l += 1
                else:
                    sorted_nums.append(right[r])
                    r += 1
            sorted_nums.extend(left[l:])
            sorted_nums.extend(right[r:])
            
            nums[:] = sorted_nums
            return count
        
        return merge_count(arr)
```

The Python implementation first transforms the input arrays into a single array of differences. It then uses a recursive merge sort to count the number of valid pairs efficiently while merging sorted halves. During the merge, for each element in the left half, it counts how many elements in the right half satisfy the inequality and adds it to the total.

## Go Solution

```go
func numberOfPairs(nums1 []int, nums2 []int, diff int) int64 {
    n := len(nums1)
    arr := make([]int, n)
    for i := 0; i < n; i++ {
        arr[i] = nums1[i] - nums2[i]
    }

    var mergeCount func([]int) int64
    mergeCount = func(nums []int) int64 {
        if len(nums) <= 1 {
            return 0
        }
        mid := len(nums) / 2
        left := append([]int(nil), nums[:mid]...)
        right := append([]int(nil), nums[mid:]...)
        count := mergeCount(left) + mergeCount(right)
        
        j := 0
        for i := 0; i < len(left); i++ {
            for j < len(right) && left[i] > right[j]+diff {
                j++
            }
            count += int64(len(right) - j)
        }

        // merge step
        l, r := 0, 0
        sortedNums := make([]int, 0, len(nums))
        for l < len(left) && r < len(right) {
            if left[l] <= right[r] {
                sortedNums = append(sortedNums, left[l])
                l++
            } else {
                sortedNums = append(sortedNums, right[r])
                r++
            }
        }
        sortedNums = append(sortedNums, left[l:]...)
        sortedNums = append(sortedNums, right[r:]...)
        copy(nums, sortedNums)
        return count
    }

    return mergeCount(arr)
}
```

The Go implementation mirrors the Python version, with attention to Go slices and int64 for counting to prevent integer overflow. Copying sorted results back into the original slice maintains correct order for further recursive merges.

## Worked Examples

**Example 1:**

```
nums1 = [3,2,5], nums2 = [2,2,1], diff = 1
arr = [1,0,4]
```

Merge sort steps:

1. Split `[1,0,4]` into `[1]` and `[0,4]`.
2. Count pairs between `[1]` and `[0,4]`:

- `1 <= 0+1`? no, j++
- `1 <= 4+1`? yes, count += 1 → total 1
3. Merge `[0,4]` to `[0,4]`
4. Merge `[1]` and `[0,4]` → `[0,1,4]`, final count 3

**Example 2:**

```
nums1 = [3,-1], nums2 = [-2,2], diff = -1
arr = [5,-3]
```

- Split `[5,-3]` → `[5]` `[ -3 ]`
- Count pairs: `5 <= -3-1`? no → count 0
- Merge `[5,-3]` → `[-3,5]`
- Final count 0
