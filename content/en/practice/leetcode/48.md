## Solution

## Problem Understanding

The problem gives us an `n x n` matrix that represents an image. Each element in the matrix corresponds to a pixel or value in that image. We must rotate the image by 90 degrees clockwise.

The important requirement is that the rotation must be performed in-place. This means we are not allowed to allocate another separate `n x n` matrix to store the rotated result. Instead, we must directly modify the existing matrix.

To understand what rotation means, consider the following matrix:

```
1 2 3
4 5 6
7 8 9
```

After a 90 degree clockwise rotation, the matrix becomes:

```
7 4 1
8 5 2
9 6 3
```

Notice how:

- The first column becomes the first row in reverse order
- The second column becomes the second row in reverse order
- The third column becomes the third row in reverse order

The constraints tell us several important things:

- The matrix is always square, meaning the number of rows equals the number of columns
- `1 <= n <= 20`, so the matrix is relatively small
- Values may be negative or positive, but the actual values do not affect the algorithm because rotation depends only on positions

Even though the constraints are small enough that a brute force solution would work in practice, the problem explicitly requires an in-place transformation, which rules out approaches that allocate another matrix.

Important edge cases include:

- A `1 x 1` matrix, which should remain unchanged after rotation
- A `2 x 2` matrix, which is the smallest matrix where rotation actually changes positions
- Odd-sized matrices, where the center element stays fixed during rotation
- Even-sized matrices, where every element participates in swaps

Because the input is guaranteed to be square, we do not need to handle invalid dimensions.

## Approaches

### Brute Force Approach

The most straightforward solution is to create another matrix of the same size and place every element into its rotated position.

For a clockwise rotation:

```
matrix[row][col]
```

moves to:

```
matrix[col][n - 1 - row]
```

Using this observation, we can allocate a new matrix and fill it with rotated values.

For example:

```
Original:
1 2 3
4 5 6
7 8 9
```

Element `1` at `(0,0)` moves to `(0,2)`.

Element `2` at `(0,1)` moves to `(1,2)`.

This approach is easy to understand and guarantees correctness because every element is mapped directly to its rotated location.

However, it violates the in-place requirement because it uses an additional `n x n` matrix. The extra memory usage is `O(n²)`.

### Optimal Approach

The key insight is that a 90 degree clockwise rotation can be decomposed into two simpler operations:

1. Transpose the matrix
2. Reverse each row

A transpose swaps rows and columns:

```
matrix[row][col] -> matrix[col][row]
```

For example:

```
1 2 3
4 5 6
7 8 9
```

becomes:

```
1 4 7
2 5 8
3 6 9
```

Then reversing each row produces:

```
7 4 1
8 5 2
9 6 3
```

This exactly matches the required rotated matrix.

The major advantage is that both operations can be done in-place using only swaps, so the extra space complexity becomes `O(1)`.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n²) | Uses another matrix to store rotated values |
| Optimal | O(n²) | O(1) | Transpose followed by row reversal, fully in-place |

## Algorithm Walkthrough

### Optimal In-Place Rotation

1. First, transpose the matrix.

Transposing means converting rows into columns. For every position `(row, col)`, we swap it with `(col, row)`.

We only swap elements above the diagonal to avoid undoing swaps. Therefore, the inner loop starts from `row + 1`.
2. After transposition, reverse every row.

Reversing each row converts the transposed matrix into the final clockwise rotated matrix.
3. Perform all swaps directly inside the original matrix.

No extra matrix is allocated, which satisfies the in-place requirement.

### Why This Works

A clockwise rotation changes coordinates according to:

```
(row, col) -> (col, n - 1 - row)
```

Transposition changes:

```
(row, col) -> (col, row)
```

Row reversal changes:

```
(col, row) -> (col, n - 1 - row)
```

Combining these two transformations produces exactly the desired rotation.

## Python Solution

```
from typing import List

class Solution:
    def rotate(self, matrix: List[List[int]]) -> None:
        """
        Do not return anything, modify matrix in-place instead.
        """
        n = len(matrix)

        # Step 1: Transpose the matrix
        for row in range(n):
            for col in range(row + 1, n):
                matrix[row][col], matrix[col][row] = (
                    matrix[col][row],
                    matrix[row][col],
                )

        # Step 2: Reverse each row
        for row in matrix:
            row.reverse()
```

The implementation begins by determining the size of the matrix.

The first nested loop performs the transpose operation. The inner loop starts from `row + 1` because elements on the diagonal do not need swapping, and elements below the diagonal would duplicate earlier swaps.

For example:

```
Swap matrix[0][1] with matrix[1][0]
Swap matrix[0][2] with matrix[2][0]
```

After the transpose is complete, every row is reversed in-place using Python's built-in `reverse()` method.

The combination of these two operations produces the required 90 degree clockwise rotation.

## Go Solution

```
func rotate(matrix [][]int) {
	n := len(matrix)

	// Step 1: Transpose the matrix
	for row := 0; row < n; row++ {
		for col := row + 1; col < n; col++ {
			matrix[row][col], matrix[col][row] =
				matrix[col][row], matrix[row][col]
		}
	}

	// Step 2: Reverse each row
	for row := 0; row < n; row++ {
		left := 0
		right := n - 1

		for left < right {
			matrix[row][left], matrix[row][right] =
				matrix[row][right], matrix[row][left]

			left++
			right--
		}
	}
}
```

The Go implementation follows the exact same algorithmic structure as the Python solution.

One small difference is that Go slices do not provide a built-in reverse method like Python lists. Therefore, row reversal is implemented manually using a two-pointer approach.

The left pointer starts at the beginning of the row, and the right pointer starts at the end. Elements are swapped while the pointers move toward each other.

Since matrix values are within small integer bounds, integer overflow is not a concern.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | Every element is visited a constant number of times |
| Space | O(1) | Rotation is performed entirely in-place |

The transpose operation touches approximately half of the matrix elements because only the upper triangle is processed. The row reversal operation processes every row independently.

Together, these operations still scale proportionally to the total number of matrix elements, giving `O(n²)` time complexity.

No additional matrix or auxiliary data structure is allocated, so the extra space usage remains constant.

## Edge Cases

### Single Element Matrix

A matrix such as:

```
[[5]]
```

contains only one element. Rotating it changes nothing.

This case can sometimes expose incorrect loop boundaries. Our implementation handles it naturally because the transpose and reversal loops execute zero or minimal iterations.

### Even-Sized Matrices

Matrices like:

```
1 2
3 4
```

have no center element. Every value participates in swaps.

Incorrect index calculations can easily overwrite values in these cases. Since our implementation relies only on controlled pairwise swaps, all values are preserved correctly.

### Odd-Sized Matrices

Matrices with odd dimensions contain a center element that should remain fixed after rotation.

For example:

```
1 2 3
4 5 6
7 8 9
```

The value `5` remains in the center.

Our transpose and reverse operations naturally preserve the center element because it maps to itself.

### Negative Numbers

The matrix may contain negative values:

```
-1 -2
-3 -4
```

The algorithm depends only on positions, not on numeric properties. Therefore, negative numbers require no special handling and rotate correctly like any other values.
