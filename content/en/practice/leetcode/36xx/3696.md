# LeetCode 3696 - Maximum Distance Between Unequal Words in Array I

## Problem Understanding

We are given an array of strings called `words`. Our task is to find two different indices `i` and `j` such that the strings at those positions are not equal, meaning `words[i] != words[j]`.

For every valid pair, we compute its distance using the formula:

$$j - i + 1$$

The goal is to return the maximum such distance among all valid pairs.

In other words, we want the widest possible segment of the array whose two endpoints contain different words. If every word in the array is identical, then there is no valid pair and we return `0`.

The input consists of:

- A string array `words`
- Each string contains lowercase English letters
- The array length is between `1` and `100`

The constraints are very small. With at most 100 elements, even a quadratic solution is perfectly acceptable. However, it is still useful to think about whether a more efficient observation exists.

Several edge cases are important:

- An array with only one element cannot form a valid pair.
- An array where all words are identical should return `0`.
- The maximum-distance pair may not involve the first and last elements.
- Multiple identical words may appear at both ends of the array.
- The optimal pair could be formed by an interior element and an endpoint.

## Approaches

### Brute Force

The most direct solution is to examine every pair of indices `(i, j)` where `i < j`.

For each pair:

- Check whether `words[i] != words[j]`.
- If they are different, compute `j - i + 1`.
- Keep track of the largest distance seen so far.

Since there are at most 100 elements, the number of pairs is at most:

$$\frac{100 \cdot 99}{2} = 4950$$

This is very small, so the brute-force solution easily passes the constraints.

The approach is correct because it explicitly checks every possible valid pair and therefore cannot miss the optimal answer.

### Optimal Observation

The maximum distance is achieved by making the endpoints as far apart as possible.

Consider the first element and the last element:

- If they are different, the answer is immediately the entire array length.
- If they are the same, then any valid maximum-distance pair must involve one of the endpoints and some position containing a different word.

Why? Because the largest possible distance always tries to stretch as far left and right as possible. If the two ends contain the same word, we only need to find:

- The farthest position from the left end whose word differs from `words[0]`
- The farthest position from the right end whose word differs from `words[n - 1]`

The larger of these two distances is the answer.

This gives a linear-time solution.

## Algorithm Walkthrough

### Optimal Algorithm

1. Let `n` be the length of `words`.
2. Check whether the first and last words are different.

If `words[0] != words[n - 1]`, then the entire array forms a valid pair. The distance is:

`n`

Return this value immediately.
3. Otherwise, the first and last words are equal.

Initialize `answer = 0`.
4. Scan from left to right.

For every index `i`, check whether `words[i] != words[0]`.

Whenever this condition is true, the pair `(0, i)` is valid, and its distance is:

`i + 1`

Update `answer` with the maximum value seen.
5. Scan from right to left.

For every index `i`, check whether `words[i] != words[n - 1]`.

Whenever this condition is true, the pair `(i, n - 1)` is valid, and its distance is:

`n - i`

Update `answer` if this distance is larger.
6. Return `answer`.

### Why it works

If the first and last words are different, no larger distance than the entire array can exist, so the answer is `n`.

If the first and last words are equal, any optimal pair must exclude at least one endpoint because the two endpoints themselves cannot form a valid pair. To maximize distance, we keep one endpoint fixed and move the other endpoint inward until we find a different word. Examining both possibilities guarantees that we find the largest valid segment. Therefore, the algorithm always returns the maximum distance.

## Python Solution

```python
from typing import List

class Solution:
    def maxDistance(self, words: List[str]) -> int:
        n = len(words)

        if words[0] != words[n - 1]:
            return n

        answer = 0

        for i in range(n):
            if words[i] != words[0]:
                answer = max(answer, i + 1)

        for i in range(n - 1, -1, -1):
            if words[i] != words[n - 1]:
                answer = max(answer, n - i)

        return answer
```

The implementation begins by storing the array length in `n`.

If the first and last words differ, the entire array forms the maximum possible valid segment, so the function immediately returns `n`.

Otherwise, both endpoints contain the same word. The first loop searches for positions whose word differs from the left endpoint. Every such position creates a valid segment from index `0` to index `i`.

The second loop performs the symmetric operation from the right side. Every position containing a different word creates a valid segment ending at index `n - 1`.

The largest distance found during these scans is returned as the final answer.

## Go Solution

```go
func maxDistance(words []string) int {
	n := len(words)

	if words[0] != words[n-1] {
		return n
	}

	answer := 0

	for i := 0; i < n; i++ {
		if words[i] != words[0] {
			if i+1 > answer {
				answer = i + 1
			}
		}
	}

	for i := n - 1; i >= 0; i-- {
		if words[i] != words[n-1] {
			if n-i > answer {
				answer = n - i
			}
		}
	}

	return answer
}
```

The Go implementation follows exactly the same logic as the Python version. Since the problem constraints are small and distances are at most `100`, there are no integer overflow concerns. The solution uses only a few integer variables and scans the slice twice, resulting in constant extra space.

## Edge Cases

### Single Element Array

When the array contains only one word, there is no way to choose two distinct indices. A careless implementation might assume at least one pair exists and produce an incorrect result. The algorithm handles this naturally because neither scan finds a different word, so the answer remains `0`.

### All Words Are Identical

Arrays such as `["z", "z", "z"]` contain no valid pair. Every comparison fails the condition `words[i] != words[j]`. During both scans, no differing word is found, so the stored answer never changes from `0`.

### Different Word Appears Only Once

Consider:

```
["k", "k", "k", "k", "m", "k"]
```

The endpoints are equal, but one interior position contains a different word. A naive implementation might only compare the endpoints and incorrectly return `0`. The algorithm explicitly searches for words that differ from each endpoint, allowing it to discover the valid distance of `5`.

### Endpoints Already Form the Optimal Pair

Consider:

```
["a", "b", "c", "d"]
```

The first and last words are different. Since no distance can exceed the full array length, the algorithm immediately returns `4`. This avoids unnecessary scanning while still producing the optimal answer.
