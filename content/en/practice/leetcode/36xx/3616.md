# LeetCode 3616 - Number of Student Replacements

## Problem Understanding

The problem gives us an array `ranks` where students arrive one by one in the order specified by the array. Each value represents a student's rank, and a smaller rank number means a better student.

The first arriving student is automatically selected. After that, every new student is compared against the currently selected student. If the new student has a strictly better rank, meaning a smaller numerical value, that student replaces the current selection. Every such replacement must be counted.

Our goal is to return the total number of replacements that occur during the entire arrival sequence.

For example, if the ranks are `[4, 1, 2]`, the first student with rank `4` is selected. The second student has rank `1`, which is better than `4`, so a replacement occurs. The third student has rank `2`, which is worse than the current selection (`1`), so no replacement occurs. The answer is `1`.

The constraints tell us that the array can contain up to `100,000` elements. This means we should prefer a linear-time solution whenever possible. Since each rank is only compared against the current best selection, there is no need for expensive nested loops or complex data structures.

Several edge cases are important:

- A single student means no replacements can occur.
- Equal ranks do not cause replacements because the comparison is strictly better.
- Strictly increasing rank values never produce replacements.
- Strictly decreasing rank values produce a replacement at every step after the first.
- Large inputs require an efficient solution with minimal extra memory.

## Approaches

### Brute Force Approach

A straightforward way to think about the problem is to simulate the process exactly as described.

We keep track of the currently selected rank. For each arriving student, we compare their rank against the current selection. If the new rank is smaller, we increment the replacement count and update the current selection.

This approach already performs only one comparison per student and therefore runs in linear time.

Because the problem itself only requires maintaining the best rank seen so far, there is no meaningful slower brute-force approach that would naturally arise. Any implementation that repeatedly searches previous elements would be unnecessarily inefficient.

### Key Insight

A replacement happens exactly when we encounter a new minimum rank in the array.

The currently selected student is always the best-ranked student seen so far. Therefore:

- Start with the first rank as the current best.
- Whenever we find a rank smaller than the current best, a replacement occurs.
- Update the current best to this new minimum rank.
- Count how many times this happens.

In other words, the answer is simply the number of times a new prefix minimum appears after the first element.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force Simulation | O(n) | O(1) | Directly simulates the selection process |
| Optimal Prefix-Minimum Tracking | O(n) | O(1) | Counts how many times a new minimum rank appears |

## Algorithm Walkthrough

1. Initialize the current selected rank as the first element of the array.
2. Initialize a variable `replacements` to `0`. This will store the total number of replacements.
3. Iterate through the array starting from the second element.
4. For each rank, compare it with the current selected rank.
5. If the current rank is strictly smaller than the selected rank, a replacement occurs.

- Increment `replacements`.
- Update the selected rank to this new rank.
6. If the current rank is greater than or equal to the selected rank, do nothing because no replacement occurs.
7. Continue until all students have been processed.
8. Return the value of `replacements`.

### Why it works

The selected student is always the student with the smallest rank encountered so far. Therefore, a replacement occurs if and only if we encounter a rank that is smaller than every previous rank. By tracking the minimum rank seen so far and counting each time it decreases, we exactly count the number of replacements required by the problem statement.

## Python Solution

```python
from typing import List

class Solution:
    def totalReplacements(self, ranks: List[int]) -> int:
        current_best = ranks[0]
        replacements = 0

        for rank in ranks[1:]:
            if rank < current_best:
                replacements += 1
                current_best = rank

        return replacements
```

The implementation starts by treating the first student as the selected student, exactly as specified in the problem statement.

The variable `current_best` stores the rank of the currently selected student. The variable `replacements` stores the answer.

We then scan the remaining students. Whenever a student's rank is strictly smaller than `current_best`, we count a replacement and update `current_best` to reflect the new selection.

At the end of the traversal, `replacements` contains the total number of replacements that occurred.

## Go Solution

```go
func totalReplacements(ranks []int) int {
	currentBest := ranks[0]
	replacements := 0

	for i := 1; i < len(ranks); i++ {
		if ranks[i] < currentBest {
			replacements++
			currentBest = ranks[i]
		}
	}

	return replacements
}
```

The Go implementation follows exactly the same logic as the Python version. Since the constraints guarantee that the array contains at least one element, accessing `ranks[0]` is always safe.

No special handling for integer overflow is needed because the answer can never exceed `n - 1`, which is at most `99,999`. This easily fits within Go's `int` type.

## Edge Cases

### Single Student

When the array contains only one student, that student is automatically selected and there are no future arrivals that could trigger a replacement. The correct answer is therefore `0`. The implementation handles this naturally because the loop over `ranks[1:]` executes zero times.

### Equal Ranks

The problem requires a student to have a strictly better rank to cause a replacement. If the current selected rank is `2` and a new student also has rank `2`, no replacement occurs. A common mistake is using `<=` instead of `<`. The implementation correctly uses a strict comparison.

### Strictly Decreasing Ranks

For an input like `[5, 4, 3, 2, 1]`, every arriving student is better than the current selection. Therefore every student after the first causes a replacement. This produces the maximum possible answer of `n - 1`. The implementation correctly counts every new minimum rank encountered.

### Best Student Arrives First

For an input like `[1, 5, 3, 2]`, the first student already has the best possible rank among all arrivals. Since no later student has a smaller rank, no replacements occur. The algorithm correctly keeps `current_best = 1` throughout the entire scan and returns `0`.
