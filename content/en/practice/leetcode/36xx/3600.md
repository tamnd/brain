## Solution

## Problem Understanding

The problem asks us to construct a **spanning tree** from a weighted undirected graph with `n` nodes and a list of `edges`, each described by `[u, v, s, must]`. A spanning tree is a subset of edges that connects all nodes without forming a cycle and contains exactly `n - 1` edges. Each edge has a **strength** `s`, and some edges are mandatory (`must = 1`), meaning they **must** be included in the spanning tree and **cannot** be upgraded.

We can optionally perform up to `k` **upgrades** on edges with `must = 0`, doubling their strength. The **stability** of a spanning tree is defined as the **minimum strength** among the edges in the tree. Our goal is to maximize this stability.

The input size is large (`n, edges.length <= 10^5`), which implies that brute-force enumeration of all spanning trees is infeasible. Constraints ensure that each edge is unique, the graph may be disconnected, and the maximum strength and number of upgrades are bounded but can be substantial. Edge cases include cycles formed entirely by mandatory edges, disconnected graphs, and cases where upgrades must be carefully allocated to improve the minimum strength.

Key observations: the problem requires combining **graph connectivity**, **edge selection**, and **upgrade optimization**, all while maximizing the minimum edge strength in the tree.

## Approaches

A naive brute-force approach would attempt to enumerate all spanning trees and apply all combinations of upgrades. This is guaranteed to find the optimal solution because it tries all possibilities, but it is computationally impossible due to the combinatorial explosion, as the number of spanning trees in a graph grows exponentially with the number of edges.

The key insight is that **maximizing the minimum strength in a spanning tree** can be approached with a **binary search** over the possible stability values, combined with **greedy MST construction** and **Union-Find** to check connectivity. Specifically, for a candidate stability `x`, we can:

1. Include all mandatory edges. If any mandatory edge has strength < `x`, this candidate fails.
2. Identify optional edges that can reach at least strength `x` either directly or via a single upgrade.
3. Sort edges by preference: prefer unupgraded edges that meet `x` first, then upgraded edges.
4. Use Union-Find to greedily build a spanning tree with these edges, counting upgrades used.
5. If a tree is successfully formed with `≤ k` upgrades, `x` is achievable.

This reduces the problem to `O(log(max_strength) * (n + m) * α(n))`, where `α(n)` is the inverse Ackermann function from Union-Find operations.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^m * combinations of upgrades) | O(n + m) | Enumerate all spanning trees and upgrade combinations; infeasible for large inputs |
| Optimal | O((n + m) log(max_strength)) | O(n) | Binary search on stability + greedy MST + Union-Find for connectivity and upgrade tracking |

## Algorithm Walkthrough

1. **Define search space**: Set `low = 1` (minimum possible strength) and `high = max(strength of any edge) * 2` (maximum achievable with one upgrade).
2. **Binary search**: While `low <= high`, select `mid = (low + high) // 2` as a candidate stability.
3. **Check feasibility**:

a. Include all mandatory edges. If any have strength < `mid`, reject this candidate.

b. Separate optional edges into two groups: edges already ≥ `mid` and edges that could reach `mid` with one upgrade.

c. Sort these edges so that edges not requiring upgrades are preferred.

d. Use Union-Find to attempt building a spanning tree. Count upgrades used and reject if `> k`.
4. **Update search bounds**: If `mid` is feasible, set `best = mid` and try a higher stability (`low = mid + 1`). Otherwise, try a lower value (`high = mid - 1`).
5. **Return result**: After binary search, if a feasible stability was found, return it; otherwise, return `-1`.

**Why it works**: Binary search ensures we efficiently explore the maximum achievable minimum strength. Union-Find guarantees we maintain a valid spanning tree without cycles. Counting upgrades ensures we respect the `k` limit.

## Python Solution

```python
from typing import List

class UnionFind:
    def __init__(self, n: int):
        self.parent = list(range(n))
        self.rank = [0] * n
    
    def find(self, x: int) -> int:
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    
    def union(self, x: int, y: int) -> bool:
        xr, yr = self.find(x), self.find(y)
        if xr == yr:
            return False
        if self.rank[xr] < self.rank[yr]:
            self.parent[xr] = yr
        elif self.rank[xr] > self.rank[yr]:
            self.parent[yr] = xr
        else:
            self.parent[yr] = xr
            self.rank[xr] += 1
        return True

class Solution:
    def maxStability(self, n: int, edges: List[List[int]], k: int) -> int:
        max_strength = max(s for _, _, s, _ in edges)
        low, high, best = 1, max_strength * 2, -1
        
        def can_achieve(stability: int) -> bool:
            uf = UnionFind(n)
            upgrades_used = 0
            
            mandatory_edges = []
            optional_edges = []
            
            for u, v, s, must in edges:
                if must:
                    if s < stability:
                        return False
                    mandatory_edges.append((u, v))
                else:
                    if s >= stability:
                        optional_edges.append((u, v, 0))
                    elif s * 2 >= stability:
                        optional_edges.append((u, v, 1))
            
            for u, v in mandatory_edges:
                uf.union(u, v)
            
            optional_edges.sort(key=lambda x: x[2])
            
            for u, v, upgrade in optional_edges:
                if uf.union(u, v):
                    upgrades_used += upgrade
                    if upgrades_used > k:
                        return False
            
            roots = {uf.find(i) for i in range(n)}
            return len(roots) == 1
        
        while low <= high:
            mid = (low + high) // 2
            if can_achieve(mid):
                best = mid
                low = mid + 1
            else:
                high = mid - 1
        
        return best
```

**Explanation**: The `UnionFind` class efficiently checks connectivity and prevents cycles. We separate mandatory and optional edges, track possible upgrades, and use a greedy MST approach for optional edges. The binary search iteratively maximizes the minimum edge strength in the resulting spanning tree.
