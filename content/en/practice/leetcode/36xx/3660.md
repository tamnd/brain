## Solution

## Problem Understanding

The problem presents an array of integers `nums` and asks us to compute, for each index `i`, the maximum value that can be reached by performing a sequence of jumps according to specific rules. The rules are directional and value-dependent: if we jump forward (to a higher index), we can only go to an index `j` where `nums[j] < nums[i]`. Conversely, if we jump backward (to a lower index), we can only jump to an index `j` where `nums[j] > nums[i]`. The output is an array `ans` of the same length as `nums`, where each entry represents the maximum number reachable from that starting index following any valid sequence of jumps.

The input constraints indicate that the array can be quite large, up to 100,000 elements, and that each element can be as large as $10^9$. These constraints rule out any naive O(n²) solution since iterating all possible jumps for each index would be prohibitively expensive.

Important edge cases include arrays with all elements equal (where no jumps increase the value), strictly increasing or decreasing arrays (where jumps might be heavily constrained), and arrays with a single element (where the result is trivial).

## Approaches

The brute-force approach involves simulating all possible jump sequences starting from each index. For each starting index `i`, one would recursively or iteratively attempt all valid jumps and record the maximum value reached. This guarantees correctness but has a time complexity of O(n²) in the worst case because each index may attempt jumps to almost all other indices.

The key insight for an optimal solution is to observe the directional constraints and monotonicity in jump possibilities. Specifically, a forward jump is allowed only to a smaller value, and a backward jump only to a larger value. This structure allows us to use dynamic programming with a monotonic stack to efficiently propagate the maximum reachable value in the correct order. By sorting indices by value and processing jumps in increasing or decreasing order, we ensure that each maximum is computed only once per index.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Simulate all sequences of jumps for each index |
| Optimal | O(n log n) | O(n) | Use dynamic programming with monotonic stack propagation based on value order |

## Algorithm Walkthrough

1. Initialize an array `dp` of the same length as `nums`, where `dp[i]` will store the maximum reachable value starting at index `i`. Initially, `dp[i] = nums[i]` for all `i`.
2. Create two lists of indices: `indices_increasing` sorted by `nums[i]` in ascending order, and `indices_decreasing` sorted in descending order.
3. For forward jumps (to smaller values), iterate through `indices_increasing`. For each index `i`, check previous indices with larger values to see if a jump can update `dp[i]`. Use a stack to maintain candidate indices for which forward jumps are allowed.
4. For backward jumps (to larger values), iterate through `indices_decreasing`. For each index `i`, check next indices with smaller values to see if a jump can update `dp[i]`. Again, a stack can efficiently manage candidates.
5. After processing both directions, `dp[i]` will contain the maximum value reachable from index `i`. Return `dp` as `ans`.

Why it works: By processing indices in order of their values and using a monotonic stack, we ensure that for any index `i`, we have already computed the maximum reachable values for all indices that can jump to `i`. This guarantees that every valid jump sequence is implicitly considered, producing the correct maximum.

## Python Solution

```python
from typing import List

class Solution:
    def maxValue(self, nums: List[int]) -> List[int]:
        n = len(nums)
        dp = nums[:]  # Initially, the max reachable value from i is nums[i]

        # Forward jumps: i -> j where j > i and nums[j] < nums[i]
        inc_stack = []
        indices_inc = sorted(range(n), key=lambda x: nums[x])
        for i in indices_inc:
            while inc_stack and inc_stack[-1] < i and nums[inc_stack[-1]] > nums[i]:
                dp[i] = max(dp[i], dp[inc_stack.pop()])

            inc_stack.append(i)

        # Backward jumps: i -> j where j < i and nums[j] > nums[i]
        dec_stack = []
        indices_dec = sorted(range(n), key=lambda x: -nums[x])
        for i in reversed(indices_dec):
            while dec_stack and dec_stack[-1] > i and nums[dec_stack[-1]] < nums[i]:
                dp[i] = max(dp[i], dp[dec_stack.pop()])
            dec_stack.append(i)

        return dp
```

The Python implementation first copies `nums` into `dp` as each index can always reach itself. It then handles forward jumps by sorting indices by value in increasing order and using a stack to propagate the maximum values efficiently. Backward jumps are handled similarly but in decreasing order and iterating in reverse. The stack ensures each element is only considered once, avoiding redundant comparisons.

## Go Solution

```go
package main

func maxValue(nums []int) []int {
    n := len(nums)
    dp := make([]int, n)
    copy(dp, nums)

    type pair struct {
        val, idx int
    }

    indicesInc := make([]pair, n)
    for i, v := range nums {
        indicesInc[i] = pair{v, i}
    }
    sort.Slice(indicesInc, func(i, j int) bool { return indicesInc[i].val < indicesInc[j].val })

    incStack := []int{}
    for _, p := range indicesInc {
        i := p.idx
        for len(incStack) > 0 && incStack[len(incStack)-1] < i && nums[incStack[len(incStack)-1]] > nums[i] {
            dp[i] = max(dp[i], dp[incStack[len(incStack)-1]])
            incStack = incStack[:len(incStack)-1]
        }
        incStack = append(incStack, i)
    }

    indicesDec := make([]pair, n)
    for i, v := range nums {
        indicesDec[i] = pair{v, i}
    }
    sort.Slice(indicesDec, func(i, j int) bool { return indicesDec[i].val > indicesDec[j].val })

    decStack := []int{}
    for k := n - 1; k >= 0; k-- {
        i := indicesDec[k].idx
        for len(decStack) > 0 && decStack[len(decStack)-1] > i && nums[decStack[len(decStack)-1]] < nums[i] {
            dp[i] = max(dp[i], dp[decStack[len(decStack)-1]])
            decStack = decStack[:len(decStack)-1]
        }
        decStack = append(decStack, i)
    }

    return dp
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

The Go version mirrors the Python approach, using slices for the stacks and sorting pairs of value-index for orderings. The key difference is explicit handling of slices and copying arrays.

## Worked Examples

Example 1: nums = [2, 1, 3]

Forward jump pass:

indices_inc = [1,0,2] (nums sorted)

dp starts as [2,1,3]

Stack operations propagate maximums: index 1 sees nothing larger before it; index 0 sees nothing; index 2 sees nothing. dp = [2,1,3]

Backward jump pass:

indices_dec = [2,0,1]

Iterate in reverse: index 1 can jump to 0 (nums[0] > nums[1]) → dp[1] = max(1, 2) = 2

Index 0 and 2 see nothing

Final dp = [2,2,3]

Example 2: nums = [2,3,1]

Forward: indices_inc = [2,0,1] → dp updates: index 2 sees nothing; index 0 sees index 2 → dp[0] = max(2,1) = 2

Index 1 sees index 0? no (nums[0] < nums[1])

Backward: indices_dec = [1,0,2] iterating reversed: index 2 sees index 1 → dp[2] = max(1,3) = 3

Index 0 sees index 1 → dp[0] = max(2,3) = 3

Index 1 sees nothing

Final dp = [3,3,3]
