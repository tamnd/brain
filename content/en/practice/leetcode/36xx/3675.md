## Solution

## Problem Understanding

The problem gives a string `s` consisting of lowercase English letters and defines a single operation: choose a character `c` and replace every occurrence of `c` in the string with the next character in the alphabet, with wraparound so that `'z'` becomes `'a'`.

The goal is to transform the entire string into a string of only `'a'` characters using the minimum number of such operations.

Each operation is global to a character, meaning it affects all positions in the string that currently contain that character. This creates a coupling effect between characters that makes the problem non-trivial: changing one character can merge it into another, potentially reducing future work.

The input constraint `1 <= s.length <= 5 * 10^5` indicates that an `O(n log n)` or `O(n)` solution is required. The alphabet size is fixed at 26, which is a key structural hint.

Important edge cases include strings already composed entirely of `'a'`, strings containing a single repeated character like `"zzzzz"`, and strings containing scattered characters across the alphabet where optimal merging becomes essential.

## Approaches

### Brute Force Approach

A direct simulation approach would repeatedly scan the string, choose a character, and apply the transformation until all characters become `'a'`. At each step, one would try all possible character choices and simulate the resulting transformation to decide the best move.

While this is conceptually correct, it is far too slow because each operation is `O(n)`, and there can be up to `O(n)` operations in the worst case. This leads to an infeasible `O(n^2)` or worse runtime.

### Key Insight for Optimal Solution

The key observation is that the alphabet forms a cycle, and each character can be thought of as a point on this circular structure. The operation moves all occurrences of a character one step forward on this cycle.

Instead of simulating operations, we observe that what matters is the set of distinct characters present in the string. Each character must eventually be “advanced” until it becomes `'a'`, but intermediate characters can merge if they are aligned strategically.

This transforms the problem into selecting an optimal traversal around a circular arrangement of the distinct characters (including `'a'`) such that the total "movement cost" is minimized. The optimal strategy reduces to sorting the unique characters on a circular scale and minimizing the largest gap.

We compute all distinct character positions, treat them on a circle of size 26, and find a rotation where the cost is minimized, which is equivalent to subtracting the largest gap from the total circular distance.

### Complexity Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force Simulation | O(n²) | O(1) | Repeatedly simulates operations on full string |
| Optimal Circular Gap Method | O(n + 26 log 26) | O(1) | Uses fixed alphabet and set of unique characters |

## Algorithm Walkthrough

1. First, extract all distinct characters from the string and map them into integer positions from `0` to `25`, where `'a' = 0` and `'z' = 25`. This reduces the problem to a small fixed universe.
2. Ensure that `'a'` is included in the set of points. Even if it does not appear in the string, it acts as the final target state and anchors the circular structure.
3. Sort the set of unique character positions in increasing order. This gives a linear representation of their positions on the alphabet circle.
4. Extend the sorted list by appending the first element plus 26. This simulates circular wraparound so that gaps across the boundary are properly accounted for.
5. Compute the differences between consecutive elements in this extended list. These differences represent the gaps between adjacent characters on the circle.
6. Sum all computed gaps. This represents the total circular traversal distance if we traverse all characters in order.
7. Identify the maximum gap among these differences. This gap represents the optimal place to “break” the circle to minimize redundant traversal.
8. Subtract the maximum gap from the total sum. The resulting value is the minimum number of operations required.

### Why it works

The key invariant is that we are effectively selecting an optimal starting point on a circular ordering of characters. By removing the largest gap, we simulate starting traversal at that break point, ensuring we do not pay the cost of traversing that largest unused arc. This is a standard circular optimization technique where minimizing traversal over a cycle reduces to removing the largest edge in the cycle.

## Python Solution

```python
class Solution:
    def minOperations(self, s: str) -> int:
        # Convert characters to numeric positions
        unique = set(ord(c) - ord('a') for c in s)
        
        # Include 'a' as anchor point
        unique.add(0)
        
        arr = sorted(unique)
        
        # Build circular extension
        extended = arr + [arr[0] + 26]
        
        total = 0
        max_gap = 0
        
        for i in range(1, len(extended)):
            gap = extended[i] - extended[i - 1]
            total += gap
            max_gap = max(max_gap, gap)
        
        return total - max_gap
```

The implementation begins by mapping characters into a numeric domain so that circular arithmetic becomes straightforward. We ensure `'a'` is included to represent the final merged state. After sorting, we explicitly model the circular nature by appending the first element shifted by 26. We then compute all adjacent differences, accumulate their sum, and track the largest gap. The final answer is obtained by subtracting this largest gap from the total traversal cost.

## Go Solution

```go
func minOperations(s string) int {
    present := make([]bool, 26)

    for _, ch := range s {
        present[int(ch-'a')] = true
    }

    present[0] = true // ensure 'a' is included

    var arr []int
    for i := 0; i < 26; i++ {
        if present[i] {
            arr = append(arr, i)
        }
    }

    extended := make([]int, len(arr)+1)
    copy(extended, arr)
    extended[len(arr)] = arr[0] + 26

    total, maxGap := 0, 0

    for i := 1; i < len(extended); i++ {
        gap := extended[i] - extended[i-1]
        total += gap
        if gap > maxGap {
            maxGap = gap
        }
    }

    return total - maxGap
}
```

In Go, we use a fixed boolean array of size 26 to track presence of characters, which is efficient and avoids hashing overhead. We then construct a slice of active character positions, sort is unnecessary because we iterate in order. The rest of the logic mirrors the Python solution, including circular extension and gap computation. Integer overflow is not a concern since the maximum possible sum is bounded by 26.

## Worked Examples

Distinct characters are `{y, z, a}` which correspond to `{24, 25, 0}`.

Total = 26

Max gap = 24

Result = 26 - 24 = 2

Distinct set: `{a}` → `[0]`

Max gap = 26

This correctly reflects that no operations are needed.

## Edge Cases

One important edge case is when the string already consists entirely of `'a'`. In this situation, the algorithm must correctly return zero, which is handled naturally because the only gap becomes the full cycle and is removed.

Another edge case occurs when the string contains only one distinct non-`'a'` character such as `"zzzz"`. Even though multiple occurrences exist, they collapse into a single transformation requirement, and the circular gap method correctly yields a single operation.

A third edge case is when characters span almost the entire alphabet, such as `"abcdefghijklmnopqrstuvwxyz"`. Here, the largest gap becomes small, and the algorithm correctly identifies that only one large circular cut can be removed to minimize operations.
