## Problem Understanding

The problem asks us to calculate the sum of "beauty" values for all positive integers `g` based on an integer array `nums`. Here, **beauty** for a positive integer `g` is defined as the product of `g` and the number of **strictly increasing subsequences** of `nums` whose **greatest common divisor (GCD)** is exactly `g`. A subsequence is strictly increasing if each element is strictly greater than the previous one.

The input `nums` is an array of integers with length up to `10^4`, and each element can be as large as `7 * 10^4`. The expected output is a single integer, which is the sum of all beauty values modulo `10^9 + 7`.

Key edge cases include arrays with a single element, arrays where all elements are the same, arrays with prime numbers, and arrays where numbers have large common divisors. The problem guarantees all numbers are positive, which simplifies GCD calculations. A naive approach that generates all subsequences will fail due to the exponential number of subsequences.

## Approaches

A brute-force solution would generate all strictly increasing subsequences, calculate their GCDs, and accumulate beauty values. While correct, this is infeasible for `n = 10^4` because the number of subsequences grows exponentially, making time complexity `O(2^n * n)` and space complexity `O(2^n)` to store subsequences.

The optimal approach leverages dynamic programming combined with number theory. For each number in the array, we maintain a dictionary (or map) that tracks counts of subsequences ending at that number for every possible GCD. When processing a new number, we can extend existing subsequences, compute the new GCD, and update counts accordingly. This way, we avoid generating subsequences explicitly. By iterating through the array and updating GCD counts efficiently, we can compute the total beauty.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(2^n) | Generate all subsequences, compute GCD, sum beauty |
| Optimal | O(n * maxNum * log(maxNum)) | O(maxNum) | DP with GCD aggregation, using hash map or array for counts |

## Algorithm Walkthrough

1. Initialize a dictionary `gcd_count` to store counts of strictly increasing subsequences for each possible GCD.
2. Iterate through each number `num` in `nums`.
3. For each existing GCD `g` in `gcd_count`, if `num` is greater than the last element of the subsequence corresponding to `g`, compute the new GCD `g_new = gcd(g, num)`. Increment the count of subsequences with GCD `g_new` by the count of subsequences for `g`.
4. Include the single-element subsequence `[num]` by incrementing `gcd_count[num]` by 1.
5. After processing all numbers, iterate over `gcd_count` and compute the total beauty as `sum(g * count) % MOD`, where `MOD = 10^9 + 7`.
6. Return the total beauty.

Why it works: This algorithm guarantees that each strictly increasing subsequence is counted exactly once. By updating the dictionary with GCDs as new numbers are processed, the algorithm avoids enumerating subsequences explicitly while accurately tracking subsequences’ GCDs. Each subsequence ending with `num` is an extension of a previously counted subsequence or a new single-element subsequence.

## Python Solution

```python
from typing import List
from math import gcd
from collections import defaultdict

class Solution:
    def totalBeauty(self, nums: List[int]) -> int:
        MOD = 10**9 + 7
        gcd_count = defaultdict(int)
        
        for num in nums:
            new_counts = defaultdict(int)
            for g, count in gcd_count.items():
                g_new = gcd(g, num)
                new_counts[g_new] = (new_counts[g_new] + count) % MOD
            new_counts[num] = (new_counts[num] + 1) % MOD
            for g, count in new_counts.items():
                gcd_count[g] = (gcd_count[g] + count) % MOD
        
        total = sum(g * count for g, count in gcd_count.items()) % MOD
        return total
```

The Python implementation uses `defaultdict` to handle GCD counts dynamically. For each number, `new_counts` stores subsequences ending with that number, which are then merged into the main `gcd_count`. The single-element subsequence `[num]` is handled separately. The final sum multiplies each GCD by its count, modulo `10^9 + 7`.

## Go Solution

```go
package main

import (
    "fmt"
)

func gcd(a, b int) int {
    for b != 0 {
        a, b = b, a%b
    }
    return a
}

func totalBeauty(nums []int) int {
    MOD := int(1e9 + 7)
    gcdCount := make(map[int]int)

    for _, num := range nums {
        newCounts := make(map[int]int)
        for g, count := range gcdCount {
            gNew := gcd(g, num)
            newCounts[gNew] = (newCounts[gNew] + count) % MOD
        }
        newCounts[num] = (newCounts[num] + 1) % MOD
        for g, count := range newCounts {
            gcdCount[g] = (gcdCount[g] + count) % MOD
        }
    }

    total := 0
    for g, count := range gcdCount {
        total = (total + g*count) % MOD
    }
    return total
}
```

The Go implementation mirrors the Python logic but uses a map for GCD counts. Go requires a custom `gcd` function and explicit modulo operations to avoid integer overflow.

## Edge Cases

**Single-element array:** When `nums` contains only one number, the only strictly increasing subsequence is the number itself. The algorithm handles this because it always includes the single-element subsequence.

**Repeated elements:** Arrays like `[2,2,2]` could lead to incorrectly counting non-strictly increasing subsequences. The algorithm ensures strict increasing property implicitly by only extending subsequences when adding a new number.
