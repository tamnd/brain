## Solution

## Problem Understanding

This problem asks us to merge one linked list into another at a specific position. Specifically, we are given two singly linked lists: `list1` and `list2`, and two indices `a` and `b`. The task is to remove all nodes from `list1` starting at position `a` up to and including position `b`, and replace that segment with the entirety of `list2`. The final output should be a single linked list that preserves the order of nodes from `list1` before position `a`, followed by all nodes from `list2`, and then the remaining nodes from `list1` after position `b`.

The input guarantees that the indices `a` and `b` are valid and that both linked lists have at least one node. The constraints indicate that the lists can be fairly long (up to 10,000 nodes), so any solution must be efficient and operate in linear time with constant space.

Key edge cases include situations where the nodes to remove are at the very beginning or end of `list1`, where `list2` has only one node, or when `list2` is significantly shorter or longer than the removed segment. The problem guarantees that `a` is at least 1 and `b` is strictly less than `list1.length - 1`, so we do not have to handle empty `list1` or removing the first or last node directly.

## Approaches

The brute-force approach would involve iterating through `list1` to remove nodes `a` through `b` and then iteratively inserting `list2` node by node. While this works correctly, it may require repeated pointer updates and unnecessary traversal of `list2`, which is inefficient.

The optimal approach leverages direct pointer manipulation. The key observation is that linked lists allow insertion by just changing the `next` pointers. We only need to locate two nodes in `list1`: the node just before index `a` (`prevA`) and the node just after index `b` (`postB`). Once these are identified, we can connect `prevA.next` to the head of `list2` and the tail of `list2` to `postB`, effectively merging the lists in constant time relative to the size of `list2`. This approach traverses each list only once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n + m) | O(1) | Iterate and remove nodes manually, insert `list2` node by node |
| Optimal | O(n + m) | O(1) | Use pointer manipulation to splice `list2` directly into `list1` |

## Algorithm Walkthrough

1. Start at the head of `list1` and iterate to find the node just before index `a`. This node is `prevA`.
2. Continue iterating until you reach node `b`. The node after this is `postB`.
3. Iterate through `list2` to find its tail node. This will be the last node of `list2`.
4. Connect `prevA.next` to the head of `list2`. This attaches `list2` at the correct position.
5. Connect the tail of `list2` to `postB`. This reconnects the remainder of `list1` after the removed segment.
6. Return the head of `list1` as the merged list.

Why it works: By carefully adjusting only the necessary pointers, we replace exactly the nodes from index `a` to `b` in `list1` with the entirety of `list2`, preserving all other nodes. The traversal ensures that no nodes are lost and that the links maintain proper order.

## Python Solution

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def mergeInBetween(self, list1: ListNode, a: int, b: int, list2: ListNode) -> ListNode:
        prevA = list1
        for _ in range(a - 1):
            prevA = prevA.next
        
        postB = prevA
        for _ in range(b - a + 2):
            postB = postB.next
        
        tail2 = list2
        while tail2.next:
            tail2 = tail2.next
        
        prevA.next = list2
        tail2.next = postB
        
        return list1
```

The code first finds the node before index `a` using a simple loop and similarly locates the node after `b`. It then finds the tail of `list2` by iterating until `tail2.next` is `None`. Finally, it connects `prevA.next` to the head of `list2` and `tail2.next` to `postB`, completing the merge.

## Go Solution

```go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func mergeInBetween(list1 *ListNode, a int, b int, list2 *ListNode) *ListNode {
    prevA := list1
    for i := 0; i < a-1; i++ {
        prevA = prevA.Next
    }

    postB := prevA
    for i := 0; i < b-a+2; i++ {
        postB = postB.Next
    }

    tail2 := list2
    for tail2.Next != nil {
        tail2 = tail2.Next
    }

    prevA.Next = list2
    tail2.Next = postB

    return list1
}
```

In Go, we follow the same logic. We use `Next` instead of `next` and `nil` instead of `None`. Pointer manipulation in Go behaves the same as in Python, so the algorithm is nearly identical.

2. `postB` points to node with value `5` (index 5)

4. Connect `prevA.next = list2` → 13 → 1000000
5. Connect `tail2.next = postB` → 1000002 → 5

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n + m) | We traverse `list1` once to locate `prevA` and `postB` and traverse `list2` to find its tail |
| Space | O(1) | No additional data structures are used, just pointers |

The linear time complexity is sufficient for the constraints because `n` and `m` can each be up to 10,000. Space is constant since we only maintain a few pointer references.
