## Solution

## Problem Understanding

The problem requires sorting an array of integers based on the frequency of each element. Specifically, elements with lower frequencies should appear first in the sorted array. If two elements have the same frequency, the element with the **larger value** should appear first.

The input is an array of integers `nums` with length between 1 and 100, and element values ranging from -100 to 100. This constraint implies that the array is small, so even less-than-optimal sorting solutions will be efficient, but the problem still demands careful handling of both frequency counting and tie-breaking by value.

The output is the array sorted according to the rules described. Edge cases include arrays where all elements have the same frequency, arrays where all elements are identical, and arrays containing negative numbers, which must still respect the decreasing order tie-breaker.

## Approaches

The brute-force approach would involve first counting the frequency of each element and then repeatedly finding the element with the lowest frequency to append to a new array. This requires multiple passes through the array and nested operations, leading to higher time complexity, although it is feasible for small arrays.

The optimal approach leverages a hash map to count frequencies in one pass and then sorts the original array using a custom comparator based on frequency and value. Sorting is efficient with built-in algorithms, and using a single sort ensures correctness while keeping code concise.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Repeatedly scanning for lowest frequency element and building result |
| Optimal | O(n log n) | O(n) | Count frequencies with hash map, then sort using custom comparator |

## Algorithm Walkthrough

1. Initialize a hash map to count the frequency of each element in `nums`. Iterate through `nums` and increment the count for each element.
2. Use the built-in sorting function to sort `nums`. The sorting key should be a tuple `(frequency, -value)`, where `frequency` is the value from the hash map and `-value` ensures that in case of ties, larger numbers come first.
3. Return the sorted array.

**Why it works:**

The hash map ensures accurate frequency counting in O(n) time. The sorting key enforces the primary condition (increasing frequency) and secondary condition (decreasing value for ties). Since Python’s sorting is stable, elements with the same frequency and value order correctly, satisfying all problem constraints.

## Python Solution

```python
from typing import List
from collections import Counter

class Solution:
    def frequencySort(self, nums: List[int]) -> List[int]:
        # Step 1: Count frequencies
        freq = Counter(nums)
        # Step 2: Sort based on frequency, then value descending for tie-breaks
        nums.sort(key=lambda x: (freq[x], -x))
        return nums
```

In this implementation, `Counter` efficiently counts each element’s frequency. The `sort` function uses a lambda function as a key to first sort by frequency (`freq[x]`) and then by negative value (`-x`) to implement decreasing order for tie cases. The result is returned directly.

## Go Solution

```go
import (
    "sort"
)

func frequencySort(nums []int) []int {
    // Step 1: Count frequencies using a map
    freq := make(map[int]int)
    for _, num := range nums {
        freq[num]++
    }

    // Step 2: Sort using custom comparator
    sort.Slice(nums, func(i, j int) bool {
        if freq[nums[i]] == freq[nums[j]] {
            return nums[i] > nums[j] // higher number first if frequencies equal
        }
        return freq[nums[i]] < freq[nums[j]] // lower frequency first
    })
    
    return nums
}
```

In Go, a `map[int]int` is used to count frequencies. `sort.Slice` allows defining a custom comparator function, which first checks frequencies and breaks ties by value. Nil slices are handled correctly because the input constraints guarantee at least one element.

## Edge Cases

One important edge case is a single-element array. The function should return the array as-is, which is correctly handled since the sorting key works even with one element. Another edge case is when all elements have the same frequency; the array must be sorted in descending order, which is implemented via the negative value tie-break in Python and comparison in Go. A third edge case is negative numbers; the algorithm treats negative numbers correctly because the frequency map does not depend on sign, and the tie-break handles negative values consistently with descending order.
