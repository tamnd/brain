## Solution

## Problem Understanding

The problem gives us a list of points on a 2D plane, where each point is represented as `[xi, yi]`. We are asked to find the **widest vertical area** that contains no points strictly inside it.

A vertical area is defined as a strip between two vertical lines, meaning the area extends infinitely in the y-direction and only has a finite width along the x-axis. Since the area is vertical, only the **x-coordinates** matter. The y-coordinates are completely irrelevant to determining the width of the empty vertical region.

The phrase "no points are inside the area" means that if we choose two x-values as boundaries, there must not be any point whose x-coordinate lies strictly between them. Importantly, points located exactly on the boundary are allowed and are **not considered inside** the area.

Restating the problem in simpler terms, we want to find the **largest gap between x-coordinates of points**, after arranging them in sorted order.

For example, if the x-coordinates are:

```
[1, 3, 5, 8, 9]
```

The gaps are:

```
3 - 1 = 2
5 - 3 = 2
8 - 5 = 3
9 - 8 = 1
```

The widest vertical area is `3`.

### Understanding the Constraints

The number of points `n` can be as large as `10^5`, which immediately tells us that a quadratic solution, `O(n²)`, will likely be too slow.

The coordinate values can be as large as `10^9`, but this does not create computational difficulty because we only perform subtraction and sorting. There is no grid traversal or geometric simulation required.

The problem guarantees at least two points, so there will always be at least one vertical gap to evaluate.

### Important Edge Cases

One important edge case is when multiple points share the same x-coordinate. In that case, the width between them is `0`, and duplicates should not artificially increase the answer.

Another edge case is when only two points exist. Since there is exactly one gap, the answer is simply the difference between their x-values.

A naive implementation might also mistakenly involve y-coordinates in the calculation, but since the vertical strip extends infinitely along the y-axis, only x-values matter.

## Approaches

### Brute Force Approach

A straightforward idea would be to compare every pair of points and determine whether the region between their x-values contains another point.

For each pair of x-coordinates `(x1, x2)`, we could check all other points to see whether any point lies strictly between them. If not, compute the width and update the maximum answer.

This works because it explicitly checks every possible vertical region and verifies whether it is empty. However, it is extremely inefficient.

If there are `n` points, there are `O(n²)` pairs. For each pair, we may need another `O(n)` scan to verify emptiness. This results in an overall complexity of `O(n³)`, which is far too slow for `n = 10^5`.

Even an improved brute force that sorts and compares every pair would still be unnecessarily expensive.

### Key Insight for an Optimal Solution

The crucial observation is that the widest valid vertical area must occur **between two adjacent x-coordinates after sorting**.

Why?

Suppose we have sorted x-values:

```
[1, 3, 5, 8]
```

If we consider the interval from `1` to `8`, it is invalid because points exist at `3` and `5` inside that region.

This means any valid empty vertical area can only exist between **consecutive x-values**, since any larger interval would necessarily contain another point.

Therefore, the problem becomes:

1. Extract all x-coordinates.
2. Sort them.
3. Find the maximum difference between adjacent values.

This reduces the problem to a sorting problem.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Checks all pairs and verifies emptiness explicitly |
| Optimal | O(n log n) | O(n) | Sort x-coordinates and compute adjacent gaps |

## Algorithm Walkthrough

### Optimal Algorithm

1. Extract the x-coordinates from every point.

Since only x-values matter, ignore y-values completely. Create a list containing all `x` values.
2. Sort the x-coordinates in ascending order.

Sorting allows us to examine neighboring points efficiently. After sorting, any valid empty vertical area must exist between adjacent x-values.
3. Initialize a variable to track the maximum width.

Start with `max_width = 0`.
4. Iterate through adjacent x-coordinates.

For every index `i`, compute:

```
x[i] - x[i - 1]
```

This gives the width of the empty vertical area between consecutive points.
5. Update the maximum width.

If the current gap is larger than `max_width`, update it.
6. Return the maximum width after processing all gaps.

### Why it works

The correctness comes from the fact that any vertical region wider than the gap between adjacent sorted x-values must contain at least one point inside it.

After sorting, consecutive x-values define the only possible empty vertical strips. Therefore, checking all adjacent gaps guarantees that we find the widest valid vertical area.

## Python Solution

```python
from typing import List

class Solution:
    def maxWidthOfVerticalArea(self, points: List[List[int]]) -> int:
        # Extract x-coordinates
        x_coordinates = [point[0] for point in points]

        # Sort x-values
        x_coordinates.sort()

        max_width = 0

        # Find maximum adjacent difference
        for i in range(1, len(x_coordinates)):
            current_width = x_coordinates[i] - x_coordinates[i - 1]
            max_width = max(max_width, current_width)

        return max_width
```

The implementation begins by extracting all x-coordinates using a list comprehension. Since y-values are irrelevant, they are ignored entirely.

Next, the x-values are sorted in ascending order. This is the key step because it guarantees that every valid empty vertical area appears between neighboring x-values.

We then iterate through adjacent elements and compute their difference. Each difference represents the width of an empty vertical strip. The largest such difference is tracked in `max_width`.

Finally, the method returns the maximum gap found.

## Go Solution

```go
package main

import "sort"

func maxWidthOfVerticalArea(points [][]int) int {
	// Extract x-coordinates
	xCoordinates := make([]int, len(points))
	for i, point := range points {
		xCoordinates[i] = point[0]
	}

	// Sort x-values
	sort.Ints(xCoordinates)

	maxWidth := 0

	// Find maximum adjacent difference
	for i := 1; i < len(xCoordinates); i++ {
		currentWidth := xCoordinates[i] - xCoordinates[i-1]
		if currentWidth > maxWidth {
			maxWidth = currentWidth
		}
	}

	return maxWidth
}
```

The Go implementation follows exactly the same logic as the Python solution. Since Go does not have a built-in list sorting method like Python, we use `sort.Ints()` from the standard library.

We allocate a slice to store x-coordinates, populate it using a loop, sort it, and then compute adjacent differences.

Integer overflow is not a concern here because the maximum coordinate value is `10^9`, which comfortably fits inside Go's `int` type on LeetCode environments.

## Edge Cases

### Multiple Points With the Same X-Coordinate

A common source of bugs is duplicate x-values. Since points can share the same x-coordinate, the difference between adjacent sorted values may be zero.

For example:

```
[[2,3],[2,5],[2,7]]
```

The sorted x-values are:

```
[2,2,2]
```

All gaps are zero, so the answer should be `0`. The implementation handles this naturally because adjacent subtraction correctly yields zero.

### Only Two Points

When the input contains exactly two points, there is only one possible vertical gap.

For example:

```
[[1,1],[10,10]]
```

The answer is simply:

```
10 - 1 = 9
```

The implementation handles this correctly because the loop starts at index `1` and processes the single available gap.

### Unsorted Input

The input points may appear in arbitrary order. A naive implementation that computes gaps directly without sorting would fail.

For example:

```
[[10,0],[1,0],[5,0]]
```

Without sorting, computed differences would be meaningless. By sorting the x-values first, the implementation guarantees adjacent comparisons are valid and complete.
