## Solution

## Problem Understanding

This problem asks us to divide the array `nums` into exactly `k` subsets, where every subset has the same size and contains only distinct values. Among all valid ways to partition the array, we must minimize the total incompatibility.

The incompatibility of a subset is defined as:

$$\text{max(subset)} - \text{min(subset)}$$

The final answer is the sum of incompatibilities across all subsets.

The array length is always divisible by `k`, so each subset must contain:

$$\text{subsetSize} = \frac{n}{k}$$

elements.

The important restriction is that no subset may contain duplicate values. Duplicates are allowed globally across different subsets, but not inside the same subset.

For example, if `nums = [1,2,1,4]` and `k = 2`, then each subset must contain exactly 2 elements. A valid partition is:

- `[1,2]`
- `[1,4]`

The incompatibility is:

$$(2 - 1) + (4 - 1) = 1 + 3 = 4$$

The partition:

- `[1,1]`
- `[2,4]`

is invalid because the first subset contains duplicate elements.

The constraints are extremely important:

- `nums.length <= 16`
- Values are small
- We must partition the entire array exactly once

Since `n` is at most 16, exponential algorithms using bitmasking become feasible. A naive brute-force partitioning approach would still be far too large, but optimized subset dynamic programming is practical.

Several edge cases immediately stand out:

- If any number appears more than `k` times, the answer is automatically `-1`, because there are only `k` subsets available, and duplicates cannot coexist inside one subset.
- Arrays with all identical values are only solvable when subset size is 1.
- Small inputs such as `n = 1` must still work correctly.
- Multiple valid partitions may exist, so we must search for the minimum total incompatibility, not merely any valid arrangement.

## Approaches

### Brute Force Approach

A brute-force solution would attempt to generate every possible partition of the array into `k` equal-sized subsets, then check:

1. Whether each subset contains unique elements
2. The incompatibility of each subset
3. The minimum total among all valid partitions

This guarantees correctness because every possible arrangement is examined.

However, the number of possible partitions grows explosively. Even for `n = 16`, the number of ways to divide elements into groups becomes astronomically large.

For example, repeatedly choosing subset combinations creates a combinatorial explosion:

$$\binom{16}{4}\binom{12}{4}\binom{8}{4}\binom{4}{4}$$

and that is before handling permutations and duplicate states.

This approach is computationally infeasible.

### Key Insight

The key observation is that `n <= 16`, which strongly suggests using bitmask dynamic programming.

Instead of thinking about partitions directly, we think about:

- Which elements have already been used
- Which valid subset can be added next

We first precompute every valid subset of size `subsetSize`:

- The subset must contain unique values
- We compute its incompatibility once

Then we use DP over bitmasks.

Each DP state represents:

$$dp[mask] = \text{minimum incompatibility using exactly the elements in mask}$$

From a state `mask`, we try to add another valid subset that does not overlap with already-used elements.

This transforms the problem from exponential partition generation into optimized subset transitions.

| --- | --- | --- | --- |
| Brute Force | Exponential beyond practicality | Very large | Enumerates all partitions |
| Optimal Bitmask DP | $O(3^n)$ worst case, practical for $n \le 16$ | $O(2^n)$ | Uses subset precomputation and DP |

## Algorithm Walkthrough

### Step 1: Check Impossible Cases

Count the frequency of every number.

If any number appears more than `k` times, return `-1`.

Why? Because duplicates cannot appear in the same subset, and we only have `k` subsets available.

### Step 2: Determine Subset Size

Each subset must contain:

$$subsetSize = n / k$$

elements.

This size remains fixed throughout the algorithm.

### Step 3: Precompute Valid Subsets

We iterate through every bitmask from `0` to `(1 << n) - 1`.

For each mask:

1. Count how many elements it contains
2. Skip masks that are not exactly `subsetSize`
3. Check whether the selected elements contain duplicates
4. If valid:

- Compute minimum value
- Compute maximum value
- Store incompatibility

This preprocessing is crucial because it avoids recomputing subset validity repeatedly during DP.

We store:

```
subset_cost[mask] = incompatibility
```

only for valid subsets.

### Step 4: Initialize DP Array

We create:

```
dp[mask]
```

where:

- `mask` represents which indices are already used
- `dp[mask]` stores the minimum incompatibility achievable

Initialize:

```
dp[0] = 0
```

because using no elements has zero incompatibility.

All other states start as infinity.

### Step 5: Transition Between States

For every current state `mask`:

1. Skip unreachable states
2. Find all valid subsets that:

- Do not overlap with `mask`
3. Create:

```
new_mask = mask | subset
```

1. Update:

```
dp[new_mask] = min(
    dp[new_mask],
    dp[mask] + subset_cost[subset]
)
```

This gradually builds larger and larger valid partitions.

### Step 6: Return Final Answer

The final mask is:

```
(1 << n) - 1
```

which means all elements are used.

If this state remains infinity, return `-1`.

Otherwise return the stored minimum incompatibility.

## Why it works

The DP explores every valid way to partition the array into legal subsets. Each state uniquely represents which elements have already been assigned. Since every transition adds exactly one valid subset, every complete path corresponds to a legal partition.

Because the DP always keeps the minimum incompatibility for each mask, the final state necessarily contains the globally optimal answer.

## Python Solution

```python
from typing import List
from collections import Counter

class Solution:
    def minimumIncompatibility(self, nums: List[int], k: int) -> int:
        n = len(nums)
        subset_size = n // k
        
        freq = Counter(nums)
        if any(count > k for count in freq.values()):
            return -1
        
        subset_cost = {}
        
        for mask in range(1 << n):
            if mask.bit_count() != subset_size:
                continue
            
            seen = set()
            min_val = float('inf')
            max_val = float('-inf')
            valid = True
            
            for i in range(n):
                if mask & (1 << i):
                    if nums[i] in seen:
                        valid = False
                        break
                    
                    seen.add(nums[i])
                    min_val = min(min_val, nums[i])
                    max_val = max(max_val, nums[i])
            
            if valid:
                subset_cost[mask] = max_val - min_val
        
        INF = float('inf')
        dp = [INF] * (1 << n)
        dp[0] = 0
        
        for mask in range(1 << n):
            if dp[mask] == INF:
                continue
            
            remaining = ((1 << n) - 1) ^ mask
            
            submask = remaining
            
            while submask:
                if submask in subset_cost:
                    new_mask = mask | submask
                    dp[new_mask] = min(
                        dp[new_mask],
                        dp[mask] + subset_cost[submask]
                    )
                
                submask = (submask - 1) & remaining
        
        result = dp[(1 << n) - 1]
        return result if result != INF else -1
```

The implementation begins by checking whether the problem is impossible. If a value appears more than `k` times, there is no valid arrangement because duplicates cannot share a subset.

Next, the code precomputes all valid subsets of size `subset_size`. Each subset is represented as a bitmask. During validation, a set ensures that no duplicate values exist inside the subset.

For every valid subset, we compute:

```python
max_val - min_val
```

and store the result in `subset_cost`.

The DP array stores the minimum incompatibility for every used-element configuration. Each mask represents which indices have already been assigned to subsets.

The transition step iterates through all remaining submasks. Whenever a submask is a valid subset, we update the DP state by adding its incompatibility cost.

Finally, the answer is the DP value for the mask where all elements are used.

## Go Solution

```go
package main

import (
	"math"
)

func minimumIncompatibility(nums []int, k int) int {
	n := len(nums)
	subsetSize := n / k

	freq := make(map[int]int)
	for _, num := range nums {
		freq[num]++
		if freq[num] > k {
			return -1
		}
	}

	subsetCost := make(map[int]int)

	for mask := 0; mask < (1 << n); mask++ {
		if bitsCount(mask) != subsetSize {
			continue
		}

		seen := make(map[int]bool)
		minVal := math.MaxInt32
		maxVal := math.MinInt32
		valid := true

		for i := 0; i < n; i++ {
			if (mask & (1 << i)) != 0 {
				if seen[nums[i]] {
					valid = false
					break
				}

				seen[nums[i]] = true

				if nums[i] < minVal {
					minVal = nums[i]
				}

				if nums[i] > maxVal {
					maxVal = nums[i]
				}
			}
		}

		if valid {
			subsetCost[mask] = maxVal - minVal
		}
	}

	INF := math.MaxInt32
	dp := make([]int, 1<<n)

	for i := range dp {
		dp[i] = INF
	}

	dp[0] = 0

	for mask := 0; mask < (1 << n); mask++ {
		if dp[mask] == INF {
			continue
		}

		remaining := ((1 << n) - 1) ^ mask
		submask := remaining

		for submask > 0 {
			if cost, exists := subsetCost[submask]; exists {
				newMask := mask | submask

				if dp[mask]+cost < dp[newMask] {
					dp[newMask] = dp[mask] + cost
				}
			}

			submask = (submask - 1) & remaining
		}
	}

	result := dp[(1<<n)-1]

	if result == INF {
		return -1
	}

	return result
}

func bitsCount(x int) int {
	count := 0

	for x > 0 {
		count++
		x &= (x - 1)
	}

	return count
}
```

The Go implementation follows the same structure as the Python solution, but uses explicit maps and helper functions.

Since Go does not provide a built-in `bit_count()` method, we implement `bitsCount()` manually using Brian Kernighan's algorithm.

The DP array is initialized with `math.MaxInt32` to represent infinity. Maps are used for subset storage and frequency counting.

Integer overflow is not a concern because incompatibility values remain very small under the problem constraints.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | $O(3^n)$ | DP over masks and submasks |
| Space | $O(2^n)$ | DP table and subset storage |

The preprocessing step iterates through all subsets:

$$O(2^n \cdot n)$$

The DP transition uses submask enumeration. Across all masks, the total complexity becomes approximately:

$$O(3^n)$$

This is acceptable because `n <= 16`.

The space complexity comes primarily from:

- DP array of size `2^n`
- Valid subset storage

Both are manageable for the constraint limit.

## Edge Cases

One important edge case occurs when a number appears more than `k` times. Since each subset may contain that number at most once, having more than `k` copies makes a valid partition impossible. The implementation handles this immediately using a frequency counter and returns `-1` before any expensive computation begins.

Another critical edge case occurs when `k == n`. In this scenario, every subset contains exactly one element. The incompatibility of a single-element subset is always zero because the minimum and maximum are identical. The algorithm naturally handles this because every valid subset has cost `0`.

A third subtle edge case involves duplicate values distributed across subsets. For example:

```
[1,1,2,2]
```

with `k = 2`.

A greedy strategy might accidentally place duplicates together and fail. The bitmask DP avoids this issue entirely because it explores all valid subset combinations and guarantees the minimum achievable incompatibility.

Another potential source of bugs is handling repeated indices correctly. Two identical values at different positions must still be treated as distinct elements during partitioning. The implementation uses bitmasks over indices rather than values, which correctly preserves element identity while still enforcing uniqueness within each subset.
