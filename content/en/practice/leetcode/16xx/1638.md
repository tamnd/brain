## Solution

## Problem Understanding

The problem asks us to count how many pairs of substrings, one taken from `s` and one taken from `t`, differ by exactly one character.

A substring is any contiguous section of a string. For example, the string `"abc"` contains substrings such as `"a"`, `"ab"`, `"bc"`, and `"abc"`.

The key requirement is that the two chosen substrings must:

1. Have the same length.
2. Differ at exactly one position.
3. Match at every other position.

We are not actually modifying the strings. The phrase "replace a single character" simply describes the condition that one substring can become the other after changing exactly one character.

For example:

- `"abc"` and `"adc"` differ only at the middle character.
- `"abc"` and `"aec"` also differ at one character.
- `"abc"` and `"abd"` differ at one character.
- `"abc"` and `"xyz"` differ at three characters, so they are invalid.

The input consists of two lowercase English strings:

- `1 <= s.length, t.length <= 100`

Since both strings are small, an `O(n^3)` or even `O(n^4)` solution may still be acceptable, but we should still aim for a clean and efficient approach.

An important detail is that different occurrences count separately. Even if two substrings contain identical text, if they come from different positions, they count as distinct pairs.

Several edge cases are important:

- Strings of length 1.
- Completely identical strings, where only single-character substrings can contribute.
- Strings with all different characters.
- Repeated characters that create many overlapping valid substrings.
- Cases where the differing character appears at the beginning or end of the substring.

Understanding that we need "exactly one mismatch" is the central challenge of the problem.

## Approaches

### Brute Force Approach

The most direct approach is to generate every substring of `s` and every substring of `t`, then compare every pair.

For every substring pair:

1. Check whether the lengths are equal.
2. Count how many positions differ.
3. If exactly one position differs, increment the answer.

This approach is correct because it explicitly checks every possible valid pair.

However, it is inefficient.

A string of length `n` has `O(n^2)` substrings. Comparing all substring pairs gives:

- `O(n^2 * m^2)` substring pairs
- Each comparison may take `O(min(n, m))`

This leads to roughly `O(n^3 * m^3)` in the worst interpretation, or about `O(n^5)` when `n = m`.

Even though constraints are small, this approach is unnecessarily slow and complicated.

### Optimal Approach

The key observation is that we do not need to generate substrings explicitly.

Instead, we can treat every pair of indices `(i, j)` as a potential mismatch position:

- `s[i] != t[j]`

If these characters differ, then this mismatch can serve as the single differing position in a valid substring pair.

From this mismatch position, we expand outward:

- Count how many matching characters extend to the left.
- Count how many matching characters extend to the right.

Suppose:

- `left` matching positions exist to the left
- `right` matching positions exist to the right

Then the number of valid substrings centered on this mismatch is:

$$(left + 1) \times (right + 1)$$

This works because we can independently choose how far to extend left and right while keeping exactly one mismatch.

This transforms the problem into a much cleaner enumeration process.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³ × m³) | O(1) | Generates and compares all substring pairs |
| Optimal | O(n × m × min(n,m)) | O(1) | Expands around each mismatch position |

## Algorithm Walkthrough

### Optimal Expansion Around Mismatch

1. Iterate through every index `i` in `s`.

This index represents a possible mismatch location in `s`.
2. Iterate through every index `j` in `t`.

This index represents a possible mismatch location in `t`.
3. Check whether `s[i] != t[j]`.

We only care about positions where the characters differ, because we need exactly one mismatch.
4. Expand left from `(i, j)`.

Move pointers backward while characters continue matching.

Count how many matching pairs exist immediately to the left.

For example:

```
s = "abcde"
t = "axcde"
       ^
```

Starting at the mismatch `'b'` vs `'x'`, the left side has:

- `'a' == 'a'`

So `left = 1`.
5. Expand right from `(i, j)`.

Move pointers forward while characters continue matching.

Count how many matching pairs exist immediately to the right.

Continuing the example:

- `'c' == 'c'`
- `'d' == 'd'`
- `'e' == 'e'`

So `right = 3`.
6. Count all valid substrings using:

$$(left + 1) \times (right + 1)$$

Why?

- We may choose to extend 0 through `left` positions to the left.
- We may choose to extend 0 through `right` positions to the right.

Every combination produces a unique substring pair with exactly one mismatch.
7. Add this contribution to the answer.
8. Continue until all mismatch positions are processed.

### Why it works

Every valid substring pair has exactly one mismatching position. Our algorithm enumerates that mismatch directly.

For a fixed mismatch `(i, j)`, every valid extension must maintain equality everywhere else. The left and right expansions precisely measure the maximum valid matching region around the mismatch.

Since left and right choices are independent, multiplying the counts gives every possible valid substring centered at that mismatch, without duplicates or omissions.

## Python Solution

```python
class Solution:
    def countSubstrings(self, s: str, t: str) -> int:
        n = len(s)
        m = len(t)

        result = 0

        for i in range(n):
            for j in range(m):
                if s[i] == t[j]:
                    continue

                left = 0
                while (
                    i - left - 1 >= 0
                    and j - left - 1 >= 0
                    and s[i - left - 1] == t[j - left - 1]
                ):
                    left += 1

                right = 0
                while (
                    i + right + 1 < n
                    and j + right + 1 < m
                    and s[i + right + 1] == t[j + right + 1]
                ):
                    right += 1

                result += (left + 1) * (right + 1)

        return result
```

The implementation follows the algorithm exactly.

The outer nested loops enumerate every pair of positions `(i, j)`.

Whenever the characters differ, we treat that pair as the unique mismatch inside a valid substring pair.

The first `while` loop expands leftward and counts matching characters before the mismatch.

The second `while` loop expands rightward and counts matching characters after the mismatch.

Finally, `(left + 1) * (right + 1)` counts all valid substrings using this mismatch position.

The implementation uses only constant extra space and does not construct substrings explicitly, which keeps the solution efficient and clean.

## Go Solution

```go
func countSubstrings(s string, t string) int {
	n := len(s)
	m := len(t)

	result := 0

	for i := 0; i < n; i++ {
		for j := 0; j < m; j++ {

			if s[i] == t[j] {
				continue
			}

			left := 0
			for i-left-1 >= 0 &&
				j-left-1 >= 0 &&
				s[i-left-1] == t[j-left-1] {

				left++
			}

			right := 0
			for i+right+1 < n &&
				j+right+1 < m &&
				s[i+right+1] == t[j+right+1] {

				right++
			}

			result += (left + 1) * (right + 1)
		}
	}

	return result
}
```

The Go implementation mirrors the Python solution closely.

Strings in Go can be indexed directly because the problem guarantees lowercase English letters, which are single-byte ASCII characters. Therefore, byte indexing is safe here.

No special handling for empty strings is required because the constraints guarantee lengths of at least 1.

The result easily fits within Go's `int` type because the maximum number of substring pairs is relatively small for strings of length 100.

## Edge Cases

### Completely Identical Strings

If `s` and `t` are identical, it is easy to accidentally count substrings with zero mismatches.

For example:

```
s = "abc"
t = "abc"
```

The problem requires exactly one differing character, not zero or one.

Our implementation handles this correctly because it only starts expansion from positions where:

```
s[i] != t[j]
```

If characters match, the algorithm skips immediately.

### Single Character Strings

When both strings have length 1, expansion logic can easily go out of bounds.

Example:

```
s = "a"
t = "b"
```

There is exactly one valid substring pair.

The boundary checks inside both expansion loops prevent invalid indexing and correctly compute:

```
(left + 1) * (right + 1) = 1
```

### Repeated Characters

Repeated characters can create many overlapping valid substrings, which often causes undercounting or double counting.

Example:

```
s = "aaa"
t = "aab"
```

Many substring pairs share the same mismatch position but differ in length.

Our algorithm handles this naturally because every mismatch position independently counts all valid left and right expansions. Each unique substring pair is counted exactly once according to its unique mismatch location.
