## Solution

## Problem Understanding

The problem asks us to find the length of the longest substring that exists **between two identical characters** in a given string `s`. Specifically, for any character that appears more than once, we want the distance between its first and last occurrence, **excluding** the characters themselves. If no character appears at least twice, the answer is `-1`.

The input is a single string `s` containing only lowercase English letters, and its length ranges from 1 to 300. This means the string is relatively short, so even an approach with O(n²) complexity may be acceptable, but an O(n) solution is preferred for efficiency.

Key edge cases to consider are strings with no repeating characters (should return -1), strings where the first and last characters are the same (the substring could be empty), and strings where multiple characters repeat but at different distances.

## Approaches

**Brute-Force Approach**

The brute-force approach involves iterating over all pairs of characters in the string. For each pair `(i, j)` such that `s[i] == s[j]` and `i < j`, we calculate the substring length `j - i - 1` and track the maximum. This guarantees correctness because it checks every possible valid substring. The downside is that this approach has a time complexity of O(n²), which is inefficient for larger strings.

**Optimal Approach**

The optimal approach leverages a hash table (dictionary or map) to store the first occurrence of each character. As we iterate through the string, if we encounter a character that already exists in the hash table, we calculate the substring length as `current_index - first_occurrence_index - 1` and update the maximum. This approach is O(n) in time because each character is processed once, and O(n) in space for the hash table.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every pair of equal characters |
| Optimal | O(n) | O(n) | Uses hash table to track first occurrence |

## Algorithm Walkthrough

1. Initialize a hash table (dictionary or map) to store the **first occurrence** index of each character.
2. Initialize a variable `max_length` to `-1` to track the maximum substring length.
3. Iterate over the string using the index `i` and character `c`.
4. If `c` is **already in the hash table**, calculate the length of the substring between the first occurrence and current index: `i - first_index - 1`. Update `max_length` if this length is greater.
5. If `c` is **not in the hash table**, store its index as the first occurrence.
6. After completing the iteration, return `max_length`.

**Why it works**

The hash table ensures that for each character, we always use the **earliest occurrence** as the start of the substring, maximizing the possible substring length between equal characters. Each character is only considered once for insertion and potentially once for length calculation, ensuring correctness and efficiency.

## Python Solution

```python
class Solution:
    def maxLengthBetweenEqualCharacters(self, s: str) -> int:
        first_occurrence = {}
        max_length = -1
        
        for i, c in enumerate(s):
            if c in first_occurrence:
                max_length = max(max_length, i - first_occurrence[c] - 1)
            else:
                first_occurrence[c] = i
        
        return max_length
```

**Implementation Explanation**

We start by creating a dictionary `first_occurrence` to remember the first index of each character. `max_length` is initialized to -1 to handle the case when no characters repeat. As we iterate, if the character is already in the dictionary, we compute the length between its first occurrence and the current index, updating `max_length` if it is larger. Otherwise, we store its index. Finally, we return the maximum length found.

## Go Solution

```go
func maxLengthBetweenEqualCharacters(s string) int {
    firstOccurrence := make(map[rune]int)
    maxLength := -1

    for i, c := range s {
        if firstIndex, exists := firstOccurrence[c]; exists {
            length := i - firstIndex - 1
            if length > maxLength {
                maxLength = length
            }
        } else {
            firstOccurrence[c] = i
        }
    }

    return maxLength
}
```

**Go Implementation Notes**

In Go, we use a map with `rune` keys to account for characters. The logic is identical to Python. We explicitly check if a character exists in the map, calculate the substring length, and update `maxLength` if necessary. Go does not have negative indexing issues, so handling edge cases is straightforward.

No repeats; `max_length` stays -1. Return -1.

## Edge Cases

**Single character string**: For `s = "a"`, there are no repeating characters. Our hash table never finds a duplicate, and `max_length` remains -1. This handles the minimum input edge case.

**Adjacent repeated characters**: For `s = "aa"`, the two 'a's are next to each other, producing a substring length of `0`. The algorithm correctly calculates `i - first_index - 1 = 1 - 0 - 1 = 0`.

**Multiple repeating characters with different distances**: For `s = "abcaabc"`, the character 'a' appears at indices 0, 3, and 4. The algorithm only needs to consider the first occurrence at index 0 to maximize the substring length between equal characters. Subsequent duplicates produce shorter lengths, and `max_length` correctly remains at 3. This ensures that the algorithm correctly maximizes the distance between the first and last occurrence of a character.
