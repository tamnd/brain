## Solution

## Problem Understanding

In this problem, we are given an `m x n` matrix of integers, and we must assign a rank to every cell. The rank represents the relative ordering of values within rows and columns while satisfying several strict constraints.

The key rule is that comparisons only matter between elements that share either the same row or the same column. If two cells are connected through a row or column relationship, then their assigned ranks must preserve the ordering of their values.

More specifically:

- If two comparable cells have different values, the larger value must receive a strictly larger rank.
- If two comparable cells have the same value, they must receive the same rank.
- Ranks start from `1`.
- Among all valid assignments, we must choose the smallest possible ranks.

The result is another matrix of the same size where each position contains the computed rank.

The difficulty comes from the fact that row and column dependencies interact globally. A rank assigned in one row can affect another column, which then affects another row, and so on. This creates a graph-like dependency structure rather than a simple local comparison problem.

The constraints are important:

- `m, n <= 500`
- Total cells can reach `250,000`
- Values may be negative and can range up to `10^9`

These limits immediately rule out extremely expensive graph algorithms or repeated recomputation across the whole matrix. We need an approach that efficiently processes relationships while avoiding quadratic or cubic behavior.

Several edge cases are especially important:

- Multiple identical values in different rows and columns.
- Entire rows or columns containing the same number.
- Negative values intermixed with positive values.
- Duplicate values that form connected components through row and column links.
- Large matrices where naive propagation would time out.

The problem guarantees that the final answer is unique, which means the constraints define a single minimal valid ranking.

## Approaches

### Brute Force Approach

A naive approach would repeatedly scan the matrix and try to update ranks until all constraints are satisfied.

One way to imagine this is:

1. Start every cell with rank `1`.
2. Compare every pair of cells sharing a row or column.
3. If constraints are violated, increase ranks accordingly.
4. Repeat until no changes occur.

This eventually converges because ranks only increase and the constraints are finite. However, the number of comparisons is enormous.

For every cell, we may compare against all other cells in the same row and column. Repeating this process many times leads to extremely poor performance.

Another brute-force variation would build a full dependency graph and repeatedly perform relaxations similar to Bellman-Ford style propagation. This is also too expensive for matrices up to `250,000` cells.

The brute-force idea is conceptually correct because it enforces all ordering constraints, but it is computationally infeasible.

### Key Insight

The crucial observation is that ranks only depend on relative ordering between distinct values. Therefore, we should process matrix values in ascending order.

When processing a particular value:

- All smaller values already have finalized ranks.
- Equal values connected by rows or columns must share the same rank.
- The new rank for a group depends only on the maximum rank already used in its rows and columns.

This naturally suggests:

1. Sort cells by value.
2. Process equal values together.
3. Use Union-Find to group equal-valued cells connected through rows or columns.
4. Compute ranks for entire connected groups simultaneously.

Union-Find is essential because equal values may form indirect connections. For example:

```
5 . 5
. 5 .
```

All three `5`s must share the same rank even though some are not directly adjacent.

The algorithm maintains:

- `row_rank[r]`, the highest rank assigned so far in row `r`
- `col_rank[c]`, the highest rank assigned so far in column `c`

For a connected component, the next valid rank is:

```
max(row_rank[r], col_rank[c]) + 1
```

taken across all cells in the component.

This gives the smallest valid rank while preserving all ordering constraints.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((mn)^2) or worse | O(mn) | Repeatedly propagates constraints |
| Optimal | O(mn log(mn)) | O(mn) | Sort values and process connected equal-value groups with Union-Find |

## Algorithm Walkthrough

### Step 1: Group Cells by Value

We first collect all matrix positions by their value.

For example:

```
value_to_cells[7] = [(0,0), (1,2)]
```

This allows us to process values in ascending order.

We sort the distinct values because smaller values must be assigned ranks before larger values.

### Step 2: Maintain Row and Column Rank State

We keep two arrays:

```
row_max[m]
col_max[n]
```

These track the largest rank already assigned in each row and column.

Initially, all values are `0` because no cells have been assigned yet.

### Step 3: Process One Value at a Time

For all cells with the same value:

- We temporarily build connected components.
- Cells are connected if they share a row or column.

This is important because equal values connected through row or column relationships must receive identical ranks.

### Step 4: Build Union-Find Structure

For the current value group:

- Every cell becomes a node.
- Cells in the same row are unioned.
- Cells in the same column are unioned.

We use hash maps:

```
row_groups
col_groups
```

to efficiently connect cells sharing rows or columns.

After union operations, every connected component represents cells that must share the same rank.

### Step 5: Compute Rank for Each Component

For every component:

1. Inspect all cells inside it.
2. Find the maximum existing rank among their rows and columns.

```
base_rank = max(row_max[r], col_max[c])
```

1. The component rank becomes:

```
base_rank + 1
```

This guarantees the smallest valid rank satisfying all previous constraints.

### Step 6: Assign Ranks

All cells in the same component receive the same rank.

We write these values into the answer matrix.

### Step 7: Update Row and Column Maximums

After assigning ranks:

```
row_max[r] = rank
col_max[c] = rank
```

This ensures future larger values build upon correct previous rankings.

### Step 8: Continue for Remaining Values

Because values are processed in ascending order, all dependencies are already resolved when a value is processed.

### Why it works

The algorithm maintains the invariant that all processed values already satisfy the ranking constraints with minimal ranks.

Processing values in sorted order ensures smaller values are finalized before larger values. Union-Find guarantees that equal values connected through rows or columns receive identical ranks. Choosing:

```
max(existing row/column rank) + 1
```

produces the smallest valid rank for each component. Therefore, every assignment is both valid and minimal, which uniquely determines the correct answer.

## Python Solution

```python
from typing import List
from collections import defaultdict

class UnionFind:
    def __init__(self, size: int):
        self.parent = list(range(size))
    
    def find(self, x: int) -> int:
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]
    
    def union(self, x: int, y: int) -> None:
        root_x = self.find(x)
        root_y = self.find(y)
        
        if root_x != root_y:
            self.parent[root_y] = root_x

class Solution:
    def matrixRankTransform(self, matrix: List[List[int]]) -> List[List[int]]:
        rows = len(matrix)
        cols = len(matrix[0])
        
        value_to_cells = defaultdict(list)
        
        for r in range(rows):
            for c in range(cols):
                value_to_cells[matrix[r][c]].append((r, c))
        
        answer = [[0] * cols for _ in range(rows)]
        
        row_max_rank = [0] * rows
        col_max_rank = [0] * cols
        
        for value in sorted(value_to_cells.keys()):
            cells = value_to_cells[value]
            size = len(cells)
            
            uf = UnionFind(size)
            
            row_map = {}
            col_map = {}
            
            for index, (r, c) in enumerate(cells):
                if r in row_map:
                    uf.union(index, row_map[r])
                else:
                    row_map[r] = index
                
                if c in col_map:
                    uf.union(index, col_map[c])
                else:
                    col_map[c] = index
            
            groups = defaultdict(list)
            
            for index in range(size):
                root = uf.find(index)
                groups[root].append(index)
            
            component_rank = {}
            
            for root, indices in groups.items():
                max_rank = 0
                
                for idx in indices:
                    r, c = cells[idx]
                    max_rank = max(
                        max_rank,
                        row_max_rank[r],
                        col_max_rank[c]
                    )
                
                component_rank[root] = max_rank + 1
            
            for root, indices in groups.items():
                rank = component_rank[root]
                
                for idx in indices:
                    r, c = cells[idx]
                    
                    answer[r][c] = rank
                    
                    row_max_rank[r] = rank
                    col_max_rank[c] = rank
        
        return answer
```

The implementation begins by grouping all matrix cells by their values using a hash map. This allows the algorithm to process equal values together.

The `UnionFind` class manages connected components among equal-valued cells. Path compression is used in `find` to keep operations efficient.

For each value:

1. A fresh Union-Find structure is created.
2. Cells sharing rows or columns are unioned.
3. Components are extracted.
4. Each component computes its required rank from previously assigned row and column ranks.
5. The rank is assigned to all component members.
6. Row and column maximum trackers are updated.

The separation between processing phases is important. We first compute ranks for all components before updating future state, ensuring equal-valued cells are handled consistently.

## Go Solution

```go
package main

import "sort"

type UnionFind struct {
	parent []int
}

func NewUnionFind(size int) *UnionFind {
	parent := make([]int, size)
	for i := 0; i < size; i++ {
		parent[i] = i
	}
	return &UnionFind{parent: parent}
}

func (uf *UnionFind) Find(x int) int {
	if uf.parent[x] != x {
		uf.parent[x] = uf.Find(uf.parent[x])
	}
	return uf.parent[x]
}

func (uf *UnionFind) Union(x, y int) {
	rootX := uf.Find(x)
	rootY := uf.Find(y)

	if rootX != rootY {
		uf.parent[rootY] = rootX
	}
}

func matrixRankTransform(matrix [][]int) [][]int {
	rows := len(matrix)
	cols := len(matrix[0])

	valueToCells := make(map[int][][]int)

	for r := 0; r < rows; r++ {
		for c := 0; c < cols; c++ {
			valueToCells[matrix[r][c]] = append(
				valueToCells[matrix[r][c]],
				[]int{r, c},
			)
		}
	}

	values := make([]int, 0, len(valueToCells))
	for value := range valueToCells {
		values = append(values, value)
	}

	sort.Ints(values)

	answer := make([][]int, rows)
	for i := range answer {
		answer[i] = make([]int, cols)
	}

	rowMaxRank := make([]int, rows)
	colMaxRank := make([]int, cols)

	for _, value := range values {
		cells := valueToCells[value]
		size := len(cells)

		uf := NewUnionFind(size)

		rowMap := make(map[int]int)
		colMap := make(map[int]int)

		for i, cell := range cells {
			r := cell[0]
			c := cell[1]

			if prev, exists := rowMap[r]; exists {
				uf.Union(i, prev)
			} else {
				rowMap[r] = i
			}

			if prev, exists := colMap[c]; exists {
				uf.Union(i, prev)
			} else {
				colMap[c] = i
			}
		}

		groups := make(map[int][]int)

		for i := 0; i < size; i++ {
			root := uf.Find(i)
			groups[root] = append(groups[root], i)
		}

		componentRank := make(map[int]int)

		for root, indices := range groups {
			maxRank := 0

			for _, idx := range indices {
				r := cells[idx][0]
				c := cells[idx][1]

				if rowMaxRank[r] > maxRank {
					maxRank = rowMaxRank[r]
				}

				if colMaxRank[c] > maxRank {
					maxRank = colMaxRank[c]
				}
			}

			componentRank[root] = maxRank + 1
		}

		for root, indices := range groups {
			rank := componentRank[root]

			for _, idx := range indices {
				r := cells[idx][0]
				c := cells[idx][1]

				answer[r][c] = rank

				rowMaxRank[r] = rank
				colMaxRank[c] = rank
			}
		}
	}

	return answer
}
```

The Go implementation mirrors the Python logic closely. Slices are used instead of Python lists, and maps replace dictionaries.

One important difference is that Go does not have built-in tuples, so cell coordinates are stored as `[]int{r, c}` pairs.

Path compression is implemented recursively in the `Find` method exactly as in Python.

Go integers are sufficient because ranks never exceed `m * n`, which is safely within integer limits.

from typing import List

class Solution:
    def matrixRankTransform(self, matrix: List[List[int]]) -> List[List[int]]:
        from collections import defaultdict

        class UnionFind:
            def __init__(self, size):
                self.parent = list(range(size))

            def find(self, x):
                if self.parent[x] != x:
                    self.parent[x] = self.find(self.parent[x])
                return self.parent[x]

            def union(self, x, y):
                px = self.find(x)
                py = self.find(y)

                if px != py:
                    self.parent[py] = px

        rows = len(matrix)
        cols = len(matrix[0])

        value_to_cells = defaultdict(list)

        for r in range(rows):
            for c in range(cols):

        answer = [[0] * cols for _ in range(rows)]

        row_max = [0] * rows
        col_max = [0] * cols

        for value in sorted(value_to_cells):
            cells = value_to_cells[value]

            uf = UnionFind(len(cells))

            row_map = {}
            col_map = {}

            for i, (r, c) in enumerate(cells):
                if r in row_map:
                    uf.union(i, row_map[r])
                else:
                    row_map[r] = i

                else:
                    col_map[c] = i

                for idx in indices:
                    r, c = cells[idx]

                    answer[r][c] = rank
                    row_max[r] = rank
                    col_max[c] = rank

        return answer

## Edge Cases

### All Values Equal

A matrix where every value is identical is deceptively tricky. Since equal values in the same row or column must have identical ranks, the entire matrix collapses into one connected component.

A buggy implementation might assign increasing ranks while scanning rows or columns independently. The Union-Find grouping prevents this by merging all connected equal-valued cells before assigning ranks.

### Indirect Connections Between Equal Values

Consider:

```
1 . 1
. 1 .
```

The corner `1`s are not directly connected, but both connect through the center `1`. Therefore, all three must share the same rank.

Without Union-Find, it is easy to miss these transitive relationships. The connected component construction guarantees that all indirectly connected equal values receive identical ranks.

### Negative Numbers and Large Ranges

The matrix may contain values from `-10^9` to `10^9`.

A naive implementation relying on array indexing by value would fail or consume enormous memory. Using a hash map keyed by value avoids this issue entirely and handles arbitrary integer ranges efficiently.
