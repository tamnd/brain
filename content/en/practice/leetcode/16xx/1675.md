## Solution

## Problem Understanding

The problem asks us to minimize the **deviation** in an array of positive integers. The **deviation** is defined as the difference between the maximum and minimum values in the array. We can perform two types of operations any number of times on any element: if the element is even, we can divide it by 2; if the element is odd, we can multiply it by 2. The goal is to perform these operations optimally to make the maximum and minimum elements as close as possible.

The input is an array of integers `nums` with size `n`, constrained such that `2 <= n <= 50,000` and each number satisfies `1 <= nums[i] <= 10^9`. This tells us that brute-force approaches that attempt all sequences of operations are infeasible, because the number of potential states grows exponentially.

Edge cases that could trip up naive implementations include arrays where all elements are initially odd or all even, arrays with repeated elements, and arrays where the maximum element is extremely large compared to the minimum. The problem guarantees at least two elements, so we do not need to handle empty arrays.

## Approaches

The **brute-force approach** would consider all possible sequences of operations on every element: multiply odd numbers by 2 until they become even, divide even numbers by 2 until they become odd, and check all resulting arrays for deviation. While this would eventually produce the correct answer, the number of possible states for each element is proportional to the number of times you can halve or double it, which can be up to 30-odd operations for numbers up to `10^9`. With `n` up to 50,000, this approach is computationally infeasible.

The **key insight** is that we can **normalize all numbers to even numbers** first. Any odd number can only be doubled once, while even numbers can be repeatedly halved until they become odd. If we maintain a **max-heap** of the current maximum values, we can greedily reduce the largest number until it cannot be reduced further, while keeping track of the minimum seen so far. At each step, we update the deviation. This works because only reducing the current maximum can potentially reduce the deviation, and stopping when all numbers are minimal ensures we consider all reachable configurations.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((log(max(nums)))^n) | O(n) | Explore all operation sequences on each number, exponential in n, infeasible |
| Optimal | O(n log n) | O(n) | Use max-heap to greedily reduce maximum element, track minimum element |

## Algorithm Walkthrough

1. **Normalize numbers to even**: Iterate through `nums` and for each number, if it is odd, multiply it by 2. This ensures every number can only be reduced by dividing by 2.
2. **Initialize heap and min tracking**: Push the negative of each number into a max-heap (Python's `heapq` is a min-heap, so we use negation). Track the minimum value seen in the array.
3. **Iteratively reduce maximum**: Pop the largest element from the heap, update the deviation as `max_val - min_val`. If the maximum is even, divide it by 2, push it back into the heap, and update the minimum if necessary.
4. **Stop condition**: If the maximum element is odd (cannot reduce further), stop the loop because no further reduction is possible.
5. **Return deviation**: The smallest deviation observed during the process is the result.

**Why it works**: By always reducing the maximum element while keeping track of the minimum, we maintain the invariant that the deviation is non-increasing. Because all numbers are initially normalized to their highest reachable state, the process ensures that we explore all reachable deviations.

## Python Solution

```python
from typing import List
import heapq

class Solution:
    def minimumDeviation(self, nums: List[int]) -> int:
        # Step 1: Normalize numbers to even
        max_heap = []
        min_val = float('inf')
        for num in nums:
            if num % 2 == 1:
                num *= 2
            min_val = min(min_val, num)
            heapq.heappush(max_heap, -num)  # use negative for max-heap

        # Step 2: Iteratively reduce maximum
        deviation = float('inf')
        while True:
            max_val = -heapq.heappop(max_heap)
            deviation = min(deviation, max_val - min_val)
            if max_val % 2 == 0:
                max_val //= 2
                min_val = min(min_val, max_val)
                heapq.heappush(max_heap, -max_val)
            else:
                break

        return deviation
```

**Explanation**: The solution first transforms all numbers into even numbers. Then it uses a max-heap to efficiently get the current maximum. At each step, it calculates the deviation and reduces the maximum if possible. The process repeats until the maximum cannot be reduced further, ensuring the minimal deviation is found.

## Go Solution

```go
package main

import (
    "container/heap"
    "math"
)

type IntHeap []int

func (h IntHeap) Len() int { return len(h) }
func (h IntHeap) Less(i, j int) bool { return h[i] > h[j] } // max-heap
func (h IntHeap) Swap(i, j int) { h[i], h[j] = h[j], h[i] }
func (h *IntHeap) Push(x interface{}) { *h = append(*h, x.(int)) }
func (h *IntHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[0 : n-1]
    return x
}

func minimumDeviation(nums []int) int {
    h := &IntHeap{}
    minVal := math.MaxInt32
    for _, num := range nums {
        if num%2 == 1 {
            num *= 2
        }
        if num < minVal {
            minVal = num
        }
        heap.Push(h, num)
    }
    heap.Init(h)

    deviation := math.MaxInt32
    for {
        maxVal := heap.Pop(h).(int)
        if maxVal - minVal < deviation {
            deviation = maxVal - minVal
        }
        if maxVal%2 == 0 {
            maxVal /= 2
            if maxVal < minVal {
                minVal = maxVal
            }
            heap.Push(h, maxVal)
        } else {
            break
        }
    }

    return deviation
}
```

**Go-specific notes**: Go requires implementing the `heap.Interface`. We define a max-heap by inverting the comparison. Overflow is not a concern with `int` since the problem constraint is below `2^31-1`. Empty slices are handled by the loop naturally.

## Worked Examples

**Example 1: nums = [1,2,3,4]**

Initial array normalized: `[2,2,6,4]` (1→2, 2 stays, 3→6, 4 stays)

Heap: `[6,4,2,2]`, min_val = 2

Iteration 1: max_val=6, deviation=6-2=4, 6 even → 3, min_val=2, push 3 → heap `[4,3,2,2]`

Iteration 2: max_val=4, deviation=4-2=2, 4 even → 2, min_val=2, push 2 → heap `[3,2,2,2]`

Iteration 3: max_val=3, deviation=3-2=1, 3 odd → stop

Result = 1

**Example 2: nums = [4,1,5,20,3]**

Normalize: `[4,2,10,20,6]`

Heap: `[20,10,6,4,2]`, min_val=2

Iteration 1: max_val=20 → 10, min_val=2 → heap `[10,10,6,4,2]`, deviation=18 → 8

Iteration 2: max_val=10 → 5, min_val=2 → heap `[10,5,6,4,2]`, deviation=8 → 8

Iteration 3: max_val=10 → 5, min_val=2 → heap `[6,5,5,4,2]`, deviation=6-2=4

Iteration 4: max_val=6 → 3, min_val=2 → heap `[5,5,3,4,2]`, deviation=5-2=3

Iteration 5: max_val=5 odd → stop

Result = 3

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n log n) | Each element is pushed/popped at most log(max_val) times. Heap operations are log n. |
| Space | O(n) | Heap stores all n elements. |

The greedy approach is efficient because we only ever reduce the largest element, and each reduction reduces the heap size logarithmically.
