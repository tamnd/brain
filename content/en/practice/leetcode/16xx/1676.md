## Solution

## Problem Understanding

This problem asks us to find the Lowest Common Ancestor, abbreviated as LCA, of multiple nodes in a binary tree.

In the classic LCA problem, we are usually given two nodes and asked to find the deepest node in the tree that is an ancestor of both. This problem generalizes that idea to an arbitrary number of nodes.

The input consists of:

- `root`, the root node of a binary tree
- `nodes`, an array containing references to several nodes in that tree

The goal is to return the single node that is the lowest ancestor shared by every node in the `nodes` array.

The phrase "lowest" is important. Multiple ancestors may contain all target nodes in their subtree, but we want the deepest such node. Another important detail is that a node is considered a descendant of itself. That means if the array contains only one node, that node is automatically its own LCA.

The tree has up to `10^4` nodes, which means we need an algorithm that is close to linear time. A quadratic solution could become too slow at the upper limit.

The problem also guarantees several useful properties:

- Every node value is unique
- Every node in `nodes` exists in the tree
- All target nodes are distinct

These guarantees simplify the implementation because we do not need to handle duplicate targets or missing nodes.

Several edge cases are important:

- The `nodes` array may contain only one node
- One target node may be an ancestor of all the others
- All target nodes may lie entirely within one subtree
- The LCA may be the root itself
- The tree may be highly unbalanced, effectively behaving like a linked list

A naive implementation can easily become inefficient if it repeatedly searches subtrees for each target node independently.

## Approaches

### Brute Force Approach

A straightforward approach is to compute the ancestor chain for every target node.

For each node in `nodes`, we can:

1. Build a parent map for the entire tree
2. Walk upward from the node to the root
3. Store all ancestors

After collecting ancestor paths for every target node, we can compare them to find the deepest common ancestor.

This method is correct because every ancestor relationship is explicitly tracked. The deepest shared ancestor among all paths is exactly the LCA.

However, this approach is not ideal because:

- It requires additional preprocessing with parent pointers
- It stores ancestor paths for every target node
- It may repeatedly traverse long chains in skewed trees

Although still manageable for `10^4` nodes, it is more complicated and less elegant than necessary.

### Optimal Approach

The key observation is that the standard recursive LCA logic for two nodes naturally extends to multiple nodes.

Suppose we recursively explore the tree. At each node:

- If the current node itself is one of the target nodes, return it
- Recursively search the left subtree
- Recursively search the right subtree

Then:

- If both left and right recursive calls return non-null results, the current node must be the LCA
- Otherwise, propagate the non-null result upward

The only modification we need is an efficient way to check whether the current node belongs to the target set. A hash set gives us `O(1)` membership checking.

This works because the first node where target nodes split across different branches is precisely the lowest common ancestor.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N + KH) | O(N + KH) | Builds parent map and ancestor paths for all target nodes |
| Optimal | O(N) | O(H + K) | Single DFS traversal using recursive LCA logic |

Where:

- `N` is the number of tree nodes
- `K` is the number of target nodes
- `H` is the height of the tree

## Algorithm Walkthrough

1. Convert the `nodes` array into a hash set.

We do this because we frequently need to check whether the current node is one of the target nodes. A hash set allows constant time lookup.
2. Start a depth first search from the root.

The recursive DFS function will return one of three things:

- `None` if no target node exists in the subtree
- A target node if exactly one relevant node is found
- The LCA if multiple targets are discovered beneath the current node
3. At each recursive call, first handle the base cases.

If the current node is `None`, return `None`.

If the current node belongs to the target set, return the current node immediately.

Returning immediately works because a target node can itself be the LCA.
4. Recursively search the left subtree.

Store the result in `left`.
5. Recursively search the right subtree.

Store the result in `right`.
6. Determine how to combine the results.

- If both `left` and `right` are non-null, it means target nodes were found in both subtrees. Therefore, the current node is their lowest common ancestor.
- If only one side is non-null, propagate that node upward.
- If both sides are null, return null.
7. The DFS result from the root is the final answer.

### Why it works

The recursion maintains an important invariant:

- Every DFS call returns the deepest node that represents all target information discovered within that subtree.

If targets exist in both left and right subtrees of a node, then that node is the first point where all targets merge. Since DFS explores bottom up, the first such merge encountered is guaranteed to be the lowest common ancestor.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, x):
#         self.val = x
#         self.left = None
#         self.right = None

from typing import List, Optional

class Solution:
    def lowestCommonAncestor(
        self,
        root: 'TreeNode',
        nodes: 'List[TreeNode]'
    ) -> 'TreeNode':

        target_nodes = set(nodes)

        def dfs(current: Optional['TreeNode']) -> Optional['TreeNode']:
            if current is None:
                return None

            if current in target_nodes:
                return current

            left_result = dfs(current.left)
            right_result = dfs(current.right)

            if left_result and right_result:
                return current

            return left_result if left_result else right_result

        return dfs(root)
```

The implementation begins by converting the input `nodes` list into a hash set called `target_nodes`. This allows efficient membership testing during traversal.

The recursive `dfs` function performs the core logic.

The first base case checks whether the current node is `None`. If so, there is nothing useful in this subtree.

The second base case checks whether the current node itself is one of the targets. If true, the node is returned upward immediately. This is important because a target node may also be the LCA.

The recursion then explores both subtrees.

After the recursive calls complete:

- If both left and right return non-null values, targets were discovered in both branches, so the current node becomes the LCA.
- Otherwise, whichever side contains a valid result is propagated upward.

Finally, the DFS result from the root is returned.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

func lowestCommonAncestor(root *TreeNode, nodes []*TreeNode) *TreeNode {
	targetSet := make(map[*TreeNode]bool)

	for _, node := range nodes {
		targetSet[node] = true
	}

	var dfs func(node *TreeNode) *TreeNode

	dfs = func(node *TreeNode) *TreeNode {
		if node == nil {
			return nil
		}

		if targetSet[node] {
			return node
		}

		leftResult := dfs(node.Left)
		rightResult := dfs(node.Right)

		if leftResult != nil && rightResult != nil {
			return node
		}

		if leftResult != nil {
			return leftResult
		}

		return rightResult
	}

	return dfs(root)
}
```

The Go implementation follows exactly the same recursive logic as the Python version.

Instead of Python's `set`, Go uses a `map[*TreeNode]bool` to simulate hash set behavior.

Go requires explicit `nil` handling, so subtree checks use comparisons like `leftResult != nil`.

Function variables are declared using `var dfs func(...)` before assignment so the recursive closure can reference itself.

## Worked Examples

### Example 1

```
Input:
root = [3,5,1,6,2,0,8,null,null,7,4]
nodes = [4,7]
```

The relevant subtree is:

```
        2
       / \
      7   4
```

We start DFS from the root.

| Current Node | Left Result | Right Result | Return Value |
| --- | --- | --- | --- |
| 7 | None | None | 7 |
| 4 | None | None | 4 |
| 2 | 7 | 4 | 2 |

At node `2`, both left and right subtrees contain target nodes, so `2` becomes the LCA.

Final answer:

```
2
```

### Example 2

```
Input:
nodes = [1]
```

The DFS eventually reaches node `1`.

Since `1` belongs to the target set, it is returned immediately.

| Current Node | Action | Return |
| --- | --- | --- |
| 1 | Node is target | 1 |

Final answer:

```
1
```

### Example 3

```
Input:
nodes = [7,6,2,4]
```

Relevant structure:

```
          5
         / \
        6   2
           / \
          7   4
```

Traversal proceeds bottom up.

| Current Node | Left Result | Right Result | Return Value |
| --- | --- | --- | --- |
| 6 | None | None | 6 |
| 7 | None | None | 7 |
| 4 | None | None | 4 |
| 2 | 7 | 4 | 2 |
| 5 | 6 | 2 | 5 |

At node `5`, targets exist in both left and right branches, so `5` is the LCA.

Final answer:

```
5
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(N) | Every tree node is visited at most once |
| Space | O(H + K) | Recursive call stack plus target hash set |

The DFS traverses the tree exactly once, so the runtime is linear in the number of nodes.

The recursion depth depends on the tree height `H`. In the worst case of a skewed tree, this can become `O(N)`. The hash set storing target nodes requires `O(K)` extra memory.

## Edge Cases

### Single Target Node

If the `nodes` array contains only one node, the answer should be that node itself.

This can easily be mishandled if the implementation assumes at least two targets must exist before identifying an LCA. Our solution handles this naturally because the DFS immediately returns the node once it is found in the target set.

### One Target Is an Ancestor of Others

Suppose one target node already contains every other target inside its subtree.

For example:

```
    5
   / \
  6   2
     / \
    7   4
```

Targets: `[5,7,4]`

A buggy implementation might continue searching upward and incorrectly return a higher ancestor. Our implementation correctly returns node `5` because target nodes are returned immediately during DFS, allowing ancestor targets to propagate upward properly.

### Highly Unbalanced Tree

A skewed tree behaves like a linked list.

```
1
 \
  2
   \
    3
     \
      4
```

Recursive solutions must still work correctly when the height becomes `O(N)`.

Our algorithm still visits each node only once, so correctness is preserved. The only impact is that recursion depth also becomes linear in the worst case.
