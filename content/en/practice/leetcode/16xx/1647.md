## Solution

## Problem Understanding

The problem asks us to make a string **good** by ensuring that no two characters in the string share the same frequency. In other words, the number of occurrences of each character must be unique. The input is a string `s` consisting only of lowercase English letters, and the output is the **minimum number of deletions** required to reach this condition.

For example, in the string `"aaabbbcc"`, `'a'` appears 3 times, `'b'` 3 times, and `'c'` 2 times. Since `'a'` and `'b'` share the same frequency, the string is not good. Deleting one `'b'` (or `'a'`) and one `'c'` results in `"aaabbc"`, which satisfies the uniqueness condition.

The constraints indicate that the string length can be up to $10^5$, which rules out brute-force solutions that attempt every possible deletion. Only solutions that work in linear or near-linear time are feasible. Important edge cases include strings where all characters are the same, strings where all frequencies are initially unique, or strings where many characters share the same frequency.

## Approaches

### Brute Force Approach

The brute-force method would be to repeatedly check all character frequencies for duplicates and remove one character from a duplicate frequency until all frequencies are unique. This would involve counting frequencies, sorting them, detecting duplicates, and reducing frequencies step by step. While correct in principle, this approach is inefficient because checking and updating frequencies repeatedly is $O(n^2)$ in the worst case, which is too slow for large strings.

### Optimal Approach

The key insight is that we do not need to simulate deletions one by one. We can:

1. Count the frequency of each character using a hash map.
2. Sort the frequencies in descending order.
3. Iterate through the frequencies, and if a frequency is not smaller than the previous one, reduce it to the next lower unused frequency (or zero) and accumulate deletions.

This works because we are always reducing a frequency to ensure it is unique, and the optimal reduction is always to the next smaller integer not yet used. We ignore frequencies that reach zero since removing all characters of a type is allowed.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Repeatedly reduce duplicate frequencies |
| Optimal | O(n + k log k) | O(k) | Count frequencies and greedily adjust to unique values (k ≤ 26) |

## Algorithm Walkthrough

1. Initialize a frequency map to count occurrences of each character in the string.
2. Convert the frequency map values into a list and sort it in descending order to prioritize handling larger frequencies first.
3. Initialize a set to store used frequencies to enforce uniqueness.
4. Initialize a counter `deletions` to zero.
5. Iterate through each frequency `f` in the sorted list:

1. While `f` is greater than zero and already exists in the used set:

- Reduce `f` by 1.
- Increment `deletions` by 1.
2. Add the resulting `f` to the used set (if greater than zero).
6. Return the total `deletions` as the minimum number of deletions required.

**Why it works**: By always reducing frequencies from largest to smallest, we minimize the number of deletions. This greedy approach ensures that each character frequency is made unique with the smallest possible reduction, which guarantees the minimum deletions.

## Python Solution

```python
class Solution:
    def minDeletions(self, s: str) -> int:
        from collections import Counter
        
        freq_counter = Counter(s)
        freq_list = sorted(freq_counter.values(), reverse=True)
        
        used_frequencies = set()
        deletions = 0
        
        for f in freq_list:
            while f > 0 and f in used_frequencies:
                f -= 1
                deletions += 1
            if f > 0:
                used_frequencies.add(f)
        
        return deletions
```

The Python code uses `Counter` to tally character frequencies efficiently. Sorting frequencies in descending order allows the algorithm to handle higher counts first, reducing conflicts greedily. The `used_frequencies` set tracks which frequencies have been assigned, and the inner loop ensures uniqueness by decrementing duplicates.

## Go Solution

```go
func minDeletions(s string) int {
    freq := make([]int, 26)
    for _, ch := range s {
        freq[ch-'a']++
    }
    
    // Filter out zeros and sort descending
    var freqs []int
    for _, f := range freq {
        if f > 0 {
            freqs = append(freqs, f)
        }
    }
    
    sort.Slice(freqs, func(i, j int) bool {
        return freqs[i] > freqs[j]
    })
    
    used := make(map[int]bool)
    deletions := 0
    
    for _, f := range freqs {
        for f > 0 && used[f] {
            f--
            deletions++
        }
        if f > 0 {
            used[f] = true
        }
    }
    
    return deletions
}
```

In Go, we preallocate an array for character counts, filter out zeros, and sort. The logic mirrors Python, using a map for used frequencies. Edge cases such as empty frequencies are handled by filtering. Go-specific considerations include explicit slice creation and sorting with a comparator.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n + k log k) | Counting characters is O(n), sorting frequencies is O(k log k) where k ≤ 26, effectively O(n) |
| Space | O(k) | Storing frequencies and used set/map, k ≤ 26 |

Even though sorting appears to be a factor, `k` is bounded by 26, so sorting is effectively constant. The solution scales linearly with string length.
