## Solution

## Problem Understanding

The problem is asking us to find the maximum number of times a given string `word` can consecutively repeat and still appear as a substring within another string `sequence`. Formally, for a positive integer `k`, `word` is considered `k`-repeating if the string obtained by concatenating `word` `k` times appears in `sequence`. Our goal is to determine the largest such `k`.

The input consists of two strings: `sequence` and `word`. The output is a single integer representing the maximum repeating count of `word` in `sequence`. The constraints indicate that both strings have a length between 1 and 100 characters, and only contain lowercase English letters. This is important because it allows us to use simple string matching techniques without worrying about large-scale performance issues.

Important edge cases to consider include when `word` is longer than `sequence` (so the maximum repeating value is automatically `0`), when `word` does not appear at all in `sequence`, and when `word` can overlap with itself multiple times in `sequence`.

## Approaches

### Brute Force Approach

The brute force approach involves checking every possible value of `k` by repeatedly concatenating `word` and checking if it exists in `sequence`. Start with `k = 1` and keep incrementing `k` until the concatenated string is no longer a substring of `sequence`. This approach is guaranteed to work because it directly tests the definition of `k`-repeating, but it is inefficient since repeated string concatenation and substring search could become costly for larger strings.

### Optimal Approach

The optimal approach leverages the observation that the maximum possible `k` cannot exceed `len(sequence) // len(word)` because a concatenated string longer than `sequence` cannot be a substring. Instead of concatenating a new string each time, we can check the substring of `sequence` in-place. Starting from each index in `sequence`, attempt to match consecutive occurrences of `word` directly by comparing slices. Keep track of the maximum number of consecutive matches. This avoids repeated string creation and reduces overhead, resulting in a simple linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m * k) | O(k * m) | Concatenates `word` repeatedly and checks substring each time, slow for larger `k` |
| Optimal | O(n * m) | O(1) | Iterates through `sequence` and counts consecutive matches of `word` without creating new strings |

Here, `n = len(sequence)` and `m = len(word)`.

## Algorithm Walkthrough

1. Initialize a variable `max_k` to `0` to store the maximum repeating count.
2. Determine the maximum possible `k` using `len(sequence) // len(word)`.
3. Iterate through the `sequence` using a starting index `i`.
4. For each starting index `i`, initialize a counter `current_k` to `0`.
5. While the substring from `i` to `i + len(word)` equals `word`, increment `current_k` and move `i` forward by `len(word)` to check for the next consecutive occurrence.
6. Update `max_k` if `current_k` is greater than the previous maximum.
7. Continue this process until the end of `sequence`.
8. Return `max_k`.

**Why it works:** This algorithm correctly identifies consecutive occurrences of `word` by examining each potential starting position and counting matches. The invariant is that at each index, we either start a new count of consecutive matches or skip ahead if a match has already been considered. This ensures all possibilities are covered, and the maximum is found.

## Python Solution

```python
class Solution:
    def maxRepeating(self, sequence: str, word: str) -> int:
        max_k = 0
        n, m = len(sequence), len(word)
        
        for i in range(n):
            current_k = 0
            while i + m <= n and sequence[i:i+m] == word:
                current_k += 1
                i += m
            max_k = max(max_k, current_k)
        
        return max_k
```

The implementation first initializes `max_k` to track the maximum repeating count. We iterate through each index in `sequence`, and for each index, we check if the substring starting at that index matches `word`. If it does, we increment `current_k` and move the index forward by the length of `word` to check for consecutive matches. After each index, we update `max_k` if `current_k` is higher than the current maximum.

## Go Solution

```go
func maxRepeating(sequence string, word string) int {
    maxK := 0
    n, m := len(sequence), len(word)

    for i := 0; i < n; i++ {
        currentK := 0
        for i+m <= n && sequence[i:i+m] == word {
            currentK++
            i += m
        }
        if currentK > maxK {
            maxK = currentK
        }
    }
    return maxK
}
```

In Go, the logic is almost identical to Python. We use slices of the string to check for equality with `word` and track `maxK` for the maximum repeating count. The for-loop increments the index `i` manually whenever a match occurs to handle consecutive matches.

## Edge Cases

**Word longer than sequence:** If `word` has a length greater than `sequence`, it can never be a substring. The algorithm handles this automatically because the while-loop condition fails, resulting in `0`.

**Overlapping words:** When `word` overlaps with itself in `sequence` (like `"aa"` in `"aaaaa"`), the algorithm correctly counts consecutive occurrences without double-counting by moving the index forward by the length of `word`.

**No occurrence of word:** If `word` does not appear in `sequence`, the algorithm returns `0` because `current_k` never increments, and `max_k` remains `0`. This is handled naturally by the loop structure.
