## Solution

## Problem Understanding

This problem asks us to find the lowest common ancestor, often abbreviated as LCA, of two nodes in a binary tree. Unlike the classic LCA problem where we are usually given the root node, here each node contains a pointer to its parent. That additional parent reference changes the way we can approach the problem.

The input consists of two nodes, `p` and `q`, that both exist somewhere inside the same binary tree. Every node contains four fields:

```
val
left
right
parent
```

The `parent` pointer allows us to move upward in the tree from any node toward the root.

The goal is to return the lowest node in the tree that has both `p` and `q` as descendants. A node is allowed to be a descendant of itself, which is extremely important. If one node lies directly on the path from the other node to the root, then that node itself is the answer.

For example:

- If `p = 5` and `q = 1`, their first shared ancestor is `3`.
- If `p = 5` and `q = 4`, then `5` itself is the answer because `4` is inside the subtree rooted at `5`.

The constraints allow up to `10^5` nodes, which means we should avoid solutions that repeatedly traverse large parts of the tree. Since every node value is unique, there is no ambiguity when comparing nodes.

Several edge cases are especially important:

- One node may be the ancestor of the other.
- The tree may be highly unbalanced, essentially behaving like a linked list.
- The lowest common ancestor may be the root.
- The depths of `p` and `q` may differ significantly.

The problem guarantees that both nodes exist in the tree and that `p != q`, so we do not need to handle missing nodes or identical inputs.

## Approaches

### Brute Force Approach

A straightforward approach is to store all ancestors of one node in a hash set, then walk upward from the other node until we find a matching ancestor.

The algorithm works as follows:

1. Start from node `p`.
2. Move upward using the `parent` pointer and store every visited node in a hash set.
3. Start from node `q`.
4. Move upward until we encounter a node already present in the set.
5. Return that node.

This approach is correct because the first shared ancestor encountered while walking upward from `q` must be the lowest common ancestor.

The solution is efficient enough for the constraints, but it uses additional memory proportional to the height of the tree.

### Optimal Approach

A more elegant solution uses the same idea as the classic intersection-of-two-linked-lists problem.

Each node can move upward through parent pointers until reaching `None`. If one pointer reaches the end first, we redirect it to the other starting node.

Eventually:

- Both pointers traverse the same total distance.
- They either meet at the LCA or simultaneously become `None`.

Since the problem guarantees a valid LCA, the pointers will meet at the correct node.

The key insight is that redirecting the pointers compensates for differing depths. After switching starting positions, both pointers effectively align relative to the common ancestor.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(h) | O(h) | Store ancestors of one node in a hash set |
| Optimal | O(h) | O(1) | Two-pointer technique using parent links |
|  |  |  |  |

Here, `h` represents the height of the tree.

## Algorithm Walkthrough

### Optimal Two-Pointer Algorithm

1. Initialize two pointers:

- `pointer_a = p`
- `pointer_b = q`

These pointers will move upward toward the root.
2. Move both pointers one step at a time.

At each iteration:

- If a pointer is not `None`, move it to its parent.
- If a pointer becomes `None`, redirect it to the other starting node.
3. Continue until the two pointers become equal.

Because both pointers traverse:

- their own path to the root
- then the other node's path to the root

they will eventually synchronize.
4. Return the meeting node.

The meeting point is guaranteed to be the lowest common ancestor.

### Why it works

Suppose the distance from `p` to the LCA is `a`, and the distance from `q` to the LCA is `b`.

Without adjustment, the deeper node would reach the LCA later. By redirecting each pointer to the other starting node after reaching the root, both pointers traverse exactly:

```
distance(p to root) + distance(q to root)
```

Therefore, after enough steps, they align at the first common node, which is precisely the lowest common ancestor.

## Python Solution

```python
"""
# Definition for a Node.
class Node:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None
        self.parent = None
"""

from typing import Optional

class Solution:
    def lowestCommonAncestor(self, p: 'Node', q: 'Node') -> 'Node':
        pointer_a = p
        pointer_b = q

        while pointer_a != pointer_b:
            if pointer_a is None:
                pointer_a = q
            else:
                pointer_a = pointer_a.parent

            if pointer_b is None:
                pointer_b = p
            else:
                pointer_b = pointer_b.parent

        return pointer_a
```

The implementation closely follows the two-pointer algorithm.

We initialize two traversal pointers at `p` and `q`. During each iteration, both pointers move upward through parent references.

When a pointer reaches the top of the tree and becomes `None`, we redirect it to the opposite starting node. This equalizes the total traversal distance between the two pointers.

The loop terminates only when both pointers reference the same node. Since the tree guarantees a valid common ancestor, the meeting point must be the LCA.

The solution uses constant extra memory because no auxiliary data structures are required.

## Go Solution

```go
/**
 * Definition for Node.
 * type Node struct {
 *     Val int
 *     Left *Node
 *     Right *Node
 *     Parent *Node
 * }
 */

func lowestCommonAncestor(p *Node, q *Node) *Node {
    pointerA := p
    pointerB := q

    for pointerA != pointerB {
        if pointerA == nil {
            pointerA = q
        } else {
            pointerA = pointerA.Parent
        }

        if pointerB == nil {
            pointerB = p
        } else {
            pointerB = pointerB.Parent
        }
    }

    return pointerA
}
```

The Go implementation mirrors the Python logic almost exactly.

The primary language-specific difference is the use of `nil` instead of `None`. Pointer comparisons in Go are straightforward because nodes are referenced directly by memory address.

No additional slices, maps, or recursion are needed, so the implementation remains memory efficient.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(h) | Each pointer traverses at most two root paths |
| Space | O(1) | Only two pointers are used |
|  |  |  |

The height of the tree is denoted by `h`.

Each pointer moves upward through the tree at most twice. Even if the depths differ significantly, each pointer traverses no more than:

```sql
distance to root from p + distance to root from q
```

No recursion, hash sets, or additional containers are used, so the extra space remains constant.

## Edge Cases

### One node is the ancestor of the other

This is the most important edge case in the problem. Many incorrect implementations continue searching upward even after reaching an ancestor node.

For example:

```
5
 \
  4
```

The LCA of `5` and `4` is `5`, not the parent of `5`.

The two-pointer approach handles this naturally because the pointers eventually align exactly at node `5`.

### Highly unbalanced trees

The tree may resemble a linked list with depth close to `10^5`.

Recursive solutions could risk stack overflow in some languages. The iterative two-pointer solution avoids recursion entirely, making it safe for extremely deep trees.

### Different node depths

One node may be much deeper than the other.

For example:

```
1
 \
  2
   \
    3
     \
      4
```

If `p = 2` and `q = 4`, the pointers initially move at different speeds relative to the root. Redirecting each pointer to the opposite starting node compensates for the depth difference and guarantees synchronization at the correct ancestor.
