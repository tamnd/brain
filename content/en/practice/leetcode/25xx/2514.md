## Solution

## Problem Understanding

The problem asks us to count how many distinct sentence anagrams can be formed from a given string `s`.

The string consists of one or more lowercase words separated by single spaces. An anagram sentence is formed by independently permuting the characters inside each word, while keeping the word positions fixed.

For example, if the input is:

```
"too hot"
```

then:

- `"oot toh"` is valid because:

- `"oot"` is a permutation of `"too"`
- `"toh"` is a permutation of `"hot"`

But:

```
"hot too"
```

is not valid because the words changed positions.

The important detail is that we only count distinct permutations. If a word contains repeated characters, many permutations become duplicates.

For example:

```
"aa"
```

has only one unique arrangement:

```
"aa"
```

not two.

Mathematically, for a word of length `n`:

- Total permutations = `n!`
- If characters repeat, divide by the factorial of each repeated frequency

So for `"too"`:

- Length = 3
- Character counts:

- `t = 1`
- `o = 2`

Distinct permutations:

$\frac{3!}{2!}=3$

The words are independent from each other, so the total number of sentence anagrams is the product of the counts for each word.

The constraints are large:

- `s.length <= 10^5`

This immediately tells us:

- We cannot generate permutations explicitly
- We need an efficient combinatorics solution
- Factorials and modular arithmetic are required

The answer must be returned modulo:

```
10^9 + 7
```

which is a standard large prime used for modular arithmetic.

Several edge cases are important:

- Words with all identical characters
- Single-character words
- Very long words
- Many words
- Large factorial values requiring modular inverses

The input guarantees:

- Only lowercase English letters and spaces
- Exactly one space between consecutive words
- No leading or trailing spaces

These guarantees simplify parsing.

## Approaches

### Brute Force Approach

A brute force solution would generate every permutation for every word, then combine them into all possible sentence variations.

For example, for `"too hot"`:

- Generate all permutations of `"too"`
- Generate all permutations of `"hot"`
- Combine every pair

This works conceptually because it directly enumerates every valid anagram sentence.

However, the number of permutations grows factorially.

A word of length `10` already has:

$10!=3628800$

possible permutations before removing duplicates.

Since the total input length can reach `100000`, explicit generation is completely infeasible.

The brute force approach becomes exponentially slow and impossible to run within limits.

### Optimal Approach

The key observation is that we never need to generate the permutations themselves.

We only need the count.

For a word of length `n` with repeated character frequencies:

- `f1, f2, f3, ...`

the number of distinct permutations is:

$\frac{n!}{f_1!f_2!f_3!\cdots}$

Since words are independent, the final answer is simply the product of these counts across all words.

Because division under modulo arithmetic is not directly allowed, we use modular inverses:

$a^{-1}\equiv a^{MOD-2}\pmod{MOD}$

This works because `10^9 + 7` is prime, allowing us to use Fermat's Little Theorem.

To make the algorithm efficient:

- Precompute factorials up to `100000`
- Precompute inverse factorials
- Process each word in linear time

This gives an efficient overall solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n!) | O(n!) | Explicitly generates permutations |
| Optimal | O(n) | O(n) | Uses combinatorics and modular arithmetic |

## Algorithm Walkthrough

1. Split the string into words.

Each word can be processed independently because permutations in one word do not affect another word.
2. Precompute factorials from `0` to `len(s)`.

We need factorial values repeatedly for permutation counting. Precomputing them avoids recomputation.
3. Precompute inverse factorials.

Since division modulo `MOD` is implemented as multiplication by modular inverse, inverse factorials allow fast computation of:

```
1 / (f1! * f2! * ...)
```
4. Initialize the answer as `1`.

Since the final result is a product of each word's contribution, we start multiplicatively.
5. For each word:

- Count character frequencies using a hash map
- Let `n` be the word length
- Start with `factorial[n]`
- Multiply by inverse factorials of repeated frequencies

This computes:

$\frac{n!}{f_1!f_2!\cdots f_k!}$
6. Multiply the word's count into the global answer.

Since each word permutation choice is independent, multiplication applies.
7. Return the final answer modulo `10^9 + 7`.

### Why it works

The algorithm works because the number of distinct permutations of a multiset is given by the multinomial formula:

$\frac{n!}{f_1!f_2!\cdots f_k!}$

Each word contributes independently to the total number of valid sentence anagrams. By multiplying the counts for all words, we count every valid combined arrangement exactly once.

Modular inverses correctly simulate division under modular arithmetic because the modulus is prime.

## Python Solution

```python
from collections import Counter
from typing import List

class Solution:
    def countAnagrams(self, s: str) -> int:
        MOD = 10**9 + 7
        
        words = s.split()
        max_len = len(s)

        factorial = [1] * (max_len + 1)
        for i in range(1, max_len + 1):
            factorial[i] = factorial[i - 1] * i % MOD

        inv_factorial = [1] * (max_len + 1)
        inv_factorial[max_len] = pow(factorial[max_len], MOD - 2, MOD)

        for i in range(max_len, 0, -1):
            inv_factorial[i - 1] = inv_factorial[i] * i % MOD

        answer = 1

        for word in words:
            count = Counter(word)

            ways = factorial[len(word)]

            for freq in count.values():
                ways = ways * inv_factorial[freq] % MOD

            answer = answer * ways % MOD

        return answer
```

The implementation begins by splitting the sentence into individual words.

Next, factorials are precomputed up to the maximum possible needed value, which is at most the length of the string. This allows constant-time factorial lookup later.

Inverse factorials are then computed using Fermat's Little Theorem. Instead of recomputing modular inverses repeatedly, the solution efficiently fills the entire inverse factorial table in reverse order.

For each word:

- `Counter` stores character frequencies
- `factorial[len(word)]` computes total permutations
- Each repeated frequency divides the total count through multiplication by an inverse factorial

Finally, the result for every word is multiplied into the global answer.

All operations are performed modulo `10^9 + 7`.

## Go Solution

```go
package main

import (
	"strings"
)

func countAnagrams(s string) int {
	const MOD int64 = 1_000_000_007

	words := strings.Split(s, " ")
	maxLen := len(s)

	factorial := make([]int64, maxLen+1)
	factorial[0] = 1

	for i := 1; i <= maxLen; i++ {
		factorial[i] = factorial[i-1] * int64(i) % MOD
	}

	invFactorial := make([]int64, maxLen+1)
	invFactorial[maxLen] = modPow(factorial[maxLen], MOD-2, MOD)

	for i := maxLen; i >= 1; i-- {
		invFactorial[i-1] = invFactorial[i] * int64(i) % MOD
	}

	answer := int64(1)

	for _, word := range words {
		freq := make(map[rune]int)

		for _, ch := range word {
			freq[ch]++
		}

		ways := factorial[len(word)]

		for _, count := range freq {
			ways = ways * invFactorial[count] % MOD
		}

		answer = answer * ways % MOD
	}

	return int(answer)
}

func modPow(base, exp, mod int64) int64 {
	result := int64(1)

	for exp > 0 {
		if exp&1 == 1 {
			result = result * base % mod
		}

		base = base * base % mod
		exp >>= 1
	}

	return result
}
```

The Go implementation follows the same mathematical logic as the Python version.

Several Go-specific details are important:

- `int64` is used to avoid integer overflow during multiplication
- `map[rune]int` stores character frequencies
- A custom fast exponentiation function computes modular inverses efficiently
- `strings.Split` separates the sentence into words

Unlike Python, Go does not provide built-in modular exponentiation, so binary exponentiation must be implemented manually.

## Edge Cases

### Words With All Identical Characters

A word like:

```
"aaaaa"
```

can easily cause mistakes if duplicate permutations are not handled correctly.

A naive implementation might incorrectly compute:

```
5! = 120
```

even though every arrangement is identical.

The implementation correctly divides by:

$5!$

resulting in exactly `1`.

### Very Long Inputs

The string length can reach `100000`.

Generating permutations explicitly would be impossible due to factorial growth.

The implementation avoids construction entirely and only computes counts mathematically, keeping runtime linear.

### Multiple Independent Words

Inputs such as:

```
"abc def ghi"
```

require multiplying independent permutation counts.

A common bug is accidentally merging frequency counts across words.

The implementation processes each word separately, ensuring independence is preserved correctly.
