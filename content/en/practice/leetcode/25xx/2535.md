## Solution

## Problem Understanding

The problem gives us an array of positive integers called `nums`. We need to compute two different values from this array.

The first value is the element sum, which is simply the sum of every number in the array.

The second value is the digit sum, which is obtained by looking at every digit inside every number and summing those digits together. For example, the number `15` contributes `1 + 5 = 6` to the digit sum.

Our goal is to return the absolute difference between these two sums. The absolute difference means the result must always be non negative, so we compute:

$$| \text{element sum} - \text{digit sum} |$$

The input size is relatively small. The array can contain up to `2000` elements, and each value is at most `2000`. This tells us that even a straightforward solution will easily fit within time limits. Since the numbers themselves are small, extracting digits using division and modulo operations is very efficient.

A few important observations help clarify the problem:

- Single digit numbers contribute the same value to both sums. For example, `7` adds `7` to the element sum and also `7` to the digit sum.
- Multi digit numbers contribute more to the element sum than to the digit sum. For example, `15` contributes `15` versus `1 + 5 = 6`.
- The problem guarantees all numbers are positive integers, so we never need to handle negative values or zero length arrays.

Some edge cases worth considering are:

- Arrays containing only single digit numbers, where the answer becomes `0`.
- Arrays containing repeated large numbers such as `2000`, which test repeated digit extraction.
- Arrays of length `1`, where the logic should still work correctly.

## Approaches

### Brute Force Approach

The most direct solution is to separately compute:

1. The sum of all elements.
2. The sum of all digits in all elements.

To calculate the digit sum, we can convert each number into a string and iterate through its characters, or repeatedly use modulo and division operations to extract digits one by one.

This approach is guaranteed to work because every digit from every number is processed exactly once, ensuring the digit sum is correct.

Although this is called a brute force approach, it is already efficient enough for the given constraints. Since each number has at most four digits, the total amount of work remains very small.

### Optimal Approach

The key insight is that we do not need separate passes for the element sum and digit sum. We can compute both values simultaneously while iterating through the array once.

For each number:

- Add the number itself to the element sum.
- Extract each digit and add it to the digit sum.

At the end, return the absolute difference.

This avoids unnecessary traversals and keeps the implementation clean and efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × d) | O(1) | Computes element sum and digit sum separately |
| Optimal | O(n × d) | O(1) | Computes both sums in one traversal |

Here, `n` is the number of elements and `d` is the number of digits per number. Since each number is at most `2000`, `d` is at most `4`.

## Algorithm Walkthrough

1. Initialize two variables:

- `element_sum = 0`
- `digit_sum = 0`

These variables will store the two quantities required by the problem.
2. Iterate through every number in the array.

For each number:

- Add the number directly to `element_sum`.
- Store the number in a temporary variable so we can safely extract digits from it.
3. Extract digits from the current number.

While the temporary number is greater than `0`:

- Use `% 10` to get the last digit.
- Add that digit to `digit_sum`.
- Use integer division `// 10` to remove the last digit.

This continues until all digits have been processed.
4. After processing all numbers, compute the absolute difference.

Return:

$$|element\_sum - digit\_sum|$$

### Why it works

The algorithm processes every array element exactly once. For each number, every digit is extracted exactly once using modulo and division operations. Therefore:

- `element_sum` equals the sum of all numbers in the array.
- `digit_sum` equals the sum of all digits appearing in those numbers.

Since the final step returns the absolute difference between these two correctly computed values, the algorithm always produces the correct answer.

## Python Solution

```python
from typing import List

class Solution:
    def differenceOfSum(self, nums: List[int]) -> int:
        element_sum = 0
        digit_sum = 0

        for number in nums:
            element_sum += number

            current = number

            while current > 0:
                digit_sum += current % 10
                current //= 10

        return abs(element_sum - digit_sum)
```

The implementation follows the algorithm directly.

The variables `element_sum` and `digit_sum` track the two required totals. As we iterate through the array, each number is immediately added to `element_sum`.

To compute the digit sum, we use a temporary variable called `current`. This prevents us from modifying the original number. The loop repeatedly extracts the last digit using `% 10`, adds it to `digit_sum`, and removes the digit using integer division.

Finally, the function returns the absolute difference using Python's built in `abs()` function.

## Go Solution

```go
package main

import "math"

func differenceOfSum(nums []int) int {
	elementSum := 0
	digitSum := 0

	for _, number := range nums {
		elementSum += number

		current := number

		for current > 0 {
			digitSum += current % 10
			current /= 10
		}
	}

	return int(math.Abs(float64(elementSum - digitSum)))
}
```

The Go implementation is structurally identical to the Python version.

One notable difference is that Go does not have a built in integer absolute value function in the standard library. The solution therefore uses `math.Abs`, which operates on `float64`, so we convert the value before and after the call.

Go slices naturally handle both empty and non empty inputs, although this problem guarantees at least one element.

## Edge Cases

One important edge case is when all numbers are single digit values. In this situation, every number contributes the same amount to both the element sum and the digit sum. A buggy implementation might accidentally overcount or incorrectly process digits, but this solution handles it naturally because extracting digits from a single digit number simply returns that number itself.

Another important edge case involves numbers containing zeros, such as `2000` or `1000`. A careless implementation using string manipulation or incorrect loops could skip zeros improperly. In this solution, zeros are naturally handled because `% 10` correctly extracts them and adds `0` to the digit sum, preserving correctness.

A third edge case is the smallest possible input size, where the array contains only one element. Some implementations incorrectly assume multiple iterations or rely on pre initialized accumulations. This algorithm works correctly regardless of array size because each number is processed independently and the sums are initialized safely before iteration begins.
