# LeetCode 2506 - Count Pairs Of Similar Strings

## Problem Understanding

The problem gives us an array of strings called `words`. We need to count how many pairs of indices `(i, j)` satisfy two conditions:

1. `i < j`
2. `words[i]` and `words[j]` are similar

Two strings are considered similar if they contain exactly the same set of distinct characters. The number of times each character appears does not matter. Only the presence or absence of characters matters.

For example:

- `"aba"` and `"aabb"` are similar because both contain only `'a'` and `'b'`
- `"abc"` and `"cba"` are similar because both contain `'a'`, `'b'`, and `'c'`
- `"abc"` and `"abd"` are not similar because one contains `'c'` while the other contains `'d'`

The input size is relatively small:

- Up to 100 words
- Each word length is at most 100
- Only lowercase English letters appear

These constraints tell us that even an `O(n²)` solution is acceptable because `100² = 10,000`, which is small. However, we still want a clean and efficient approach.

An important observation is that repeated characters inside a word do not matter. For example:

- `"aaaa"` and `"a"` should be treated the same
- `"ab"` and `"baba"` should also be treated the same

This means every word can be reduced to a representation of its unique characters.

Some important edge cases include:

- Multiple identical words
- Words containing only one unique character
- Words with characters in different orders
- Words with many repeated letters
- No valid pairs at all

The problem guarantees that all strings contain only lowercase English letters, which makes bit manipulation especially convenient because there are only 26 possible characters.

## Approaches

### Brute Force Approach

The most direct solution is to compare every pair of words.

For each pair `(i, j)`:

1. Build a set of unique characters for `words[i]`
2. Build a set of unique characters for `words[j]`
3. Compare the two sets
4. If they are equal, increment the answer

This works because sets naturally ignore duplicate characters, which matches the definition of similarity.

However, this approach repeatedly rebuilds character sets for the same words. If there are `n` words and each word has length `m`, then each comparison may cost `O(m)` time.

Since there are `O(n²)` pairs, the total complexity becomes `O(n² × m)`.

Even though this passes the constraints, we can improve the implementation by preprocessing each word once.

### Optimal Approach

The key insight is that similarity depends only on the set of characters in a word.

Since there are only 26 lowercase English letters, we can represent each word as a 26-bit integer mask.

For example:

- `"ab"` becomes binary `11`
- `"ac"` becomes binary `101`
- `"abc"` becomes binary `111`

Each bit represents whether a character exists in the word.

To build the mask:

- Bit 0 represents `'a'`
- Bit 1 represents `'b'`
- Bit 2 represents `'c'`
- and so on

If two words produce the same bitmask, then they contain exactly the same set of characters and are therefore similar.

After converting each word into a mask, we count how many times each mask appears using a hash map.

If a mask appears `k` times, then it contributes:

$$\frac{k(k-1)}{2}$$

similar pairs.

This avoids repeated comparisons and makes the solution cleaner and more scalable.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n² × m) | O(m) | Compare every pair using character sets |
| Optimal | O(n × m) | O(n) | Convert each word into a bitmask and count frequencies |

Where:

- `n` = number of words
- `m` = average word length

## Algorithm Walkthrough

### Step 1: Create a frequency map

We use a hash map where:

- Key = bitmask representing a set of characters
- Value = number of words with that mask

This allows us to group similar words together efficiently.

### Step 2: Convert each word into a bitmask

For every character in the word:

1. Compute its position from `'a'`
2. Set the corresponding bit in the mask using bitwise OR

For example, for `"bac"`:

- `'b'` sets bit 1
- `'a'` sets bit 0
- `'c'` sets bit 2

Final mask:

$$111_2$$

Repeated characters do not matter because setting the same bit multiple times has no effect.

### Step 3: Count how many times each mask appears

After generating the mask for a word, increment its frequency in the hash map.

Example:

| Word | Mask | Frequency After Insert |
| --- | --- | --- |
| `"ab"` | `3` | 1 |
| `"ba"` | `3` | 2 |
| `"aabb"` | `3` | 3 |

### Step 4: Compute the number of pairs

For each mask frequency `k`:

- Any two words among those `k` words form a valid pair
- Number of pairs is:

$$\frac{k(k-1)}{2}$$

Add this value to the answer.

### Why it works

Every word is transformed into a canonical representation, its character bitmask. Two words are similar if and only if they produce the same mask. The hash map groups all similar words together, and the combination formula correctly counts all distinct pairs within each group.

## Python Solution

```python
from collections import defaultdict
from typing import List

class Solution:
    def similarPairs(self, words: List[str]) -> int:
        mask_count = defaultdict(int)

        for word in words:
            mask = 0

            for char in word:
                bit_position = ord(char) - ord('a')
                mask |= 1 << bit_position

            mask_count[mask] += 1

        pairs = 0

        for frequency in mask_count.values():
            pairs += frequency * (frequency - 1) // 2

        return pairs
```

The implementation starts by creating a dictionary called `mask_count`. This dictionary stores how many words correspond to each bitmask.

For every word, we initialize `mask = 0`. As we iterate through the characters, we compute the bit position using:

```python
ord(char) - ord('a')
```

We then set the corresponding bit using:

```python
mask |= 1 << bit_position
```

This operation ensures that each character contributes only once to the final mask, regardless of repetition.

After processing the word, we increment its count in the hash map.

Finally, we iterate through all frequencies in the dictionary. If a mask appears `k` times, we add:

```python
k * (k - 1) // 2
```

to the result, which counts all unique pairs.

## Go Solution

```go
func similarPairs(words []string) int {
	maskCount := make(map[int]int)

	for _, word := range words {
		mask := 0

		for _, ch := range word {
			bitPosition := int(ch - 'a')
			mask |= 1 << bitPosition
		}

		maskCount[mask]++
	}

	pairs := 0

	for _, frequency := range maskCount {
		pairs += frequency * (frequency - 1) / 2
	}

	return pairs
}
```

The Go solution follows the same logic as the Python version. A `map[int]int` is used instead of a Python dictionary.

Go characters are processed as runes during iteration, so we convert the difference `ch - 'a'` into an integer before shifting bits.

Since the constraints are small, integer overflow is not a concern. The maximum number of pairs is:

$$\frac{100 \times 99}{2} = 4950$$

which easily fits inside a standard Go `int`.

### Step-by-step processing

| Word | Unique Characters | Bitmask | Frequency Map |
| --- | --- | --- | --- |
| `"aba"` | `{a,b}` | `3` | `{3:1}` |
| `"aabb"` | `{a,b}` | `3` | `{3:2}` |
| `"abcd"` | `{a,b,c,d}` | `15` | `{3:2, 15:1}` |
| `"bac"` | `{a,b,c}` | `7` | `{3:2, 15:1, 7:1}` |
| `"aabc"` | `{a,b,c}` | `7` | `{3:2, 15:1, 7:2}` |

Now compute pairs:

- Mask `3` appears 2 times → `2 × 1 / 2 = 1`
- Mask `15` appears 1 time → `0`
- Mask `7` appears 2 times → `1`

Total:

```
1 + 1 = 2
```

Answer:

```
2
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × m) | Each character of each word is processed once |
| Space | O(n) | Hash map stores at most one entry per unique mask |

The algorithm scans every character exactly once while building masks. Since each word contributes a single integer key into the hash map, the additional memory usage is proportional to the number of distinct masks.

Because there are only 26 lowercase letters, mask operations are constant time.

## Edge Cases

One important edge case is words containing repeated characters. For example, `"aaaa"` and `"a"` should be considered similar. A naive implementation that compares frequency counts instead of character presence could incorrectly treat them as different. The bitmask approach handles this naturally because setting the same bit multiple times has no effect.

Another important edge case is words with identical character sets but different ordering, such as `"abc"` and `"cba"`. Since the algorithm ignores ordering and only records whether each character exists, both words produce the same mask and are correctly grouped together.

A third edge case is when no valid pairs exist. For example, `["ab", "cd", "ef"]` produces three completely different masks. The frequency of every mask is `1`, and the combination formula correctly produces zero pairs for each group.

A final edge case involves the maximum character variety. A word may contain all 26 lowercase letters. The implementation safely handles this because a standard integer easily stores 26 bits, and every character maps cleanly into the bitmask representation.
