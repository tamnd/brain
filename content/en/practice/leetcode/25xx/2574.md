# LeetCode 2574, Left and Right Sum Differences

## Problem Understanding

The problem gives us an integer array `nums` and asks us to compute a new array called `answer`.

For every index `i`, we must calculate two values:

- `leftSum[i]`, which is the sum of all elements strictly to the left of index `i`
- `rightSum[i]`, which is the sum of all elements strictly to the right of index `i`

The final value at each position is:

$$answer[i] = |leftSum[i] - rightSum[i]|$$

This means we take the absolute difference between the sum of numbers before the current index and the sum of numbers after the current index.

For example, if:

```
nums = [10,4,8,3]
```

Then for index `2`:

- Elements to the left are `[10,4]`, whose sum is `14`
- Elements to the right are `[3]`, whose sum is `3`

So:

$$answer[2] = |14 - 3| = 11$$

The constraints are relatively small:

- `1 <= nums.length <= 1000`
- `1 <= nums[i] <= 10^5`

An array size of 1000 is small enough that even a quadratic solution would technically pass, but the problem is clearly designed to teach prefix sum techniques and efficient cumulative calculations.

There are several important edge cases to consider.

If the array contains only one element, then both the left and right sums are zero because there are no elements on either side. The result must therefore be `[0]`.

The first element always has a left sum of zero, because nothing exists before it.

The last element always has a right sum of zero, because nothing exists after it.

Another important detail is that we must use the absolute difference. Without the absolute value, negative answers could appear.

## Approaches

### Brute Force Approach

The most direct solution is to compute the left and right sums separately for every index.

For each position `i`:

- Iterate from `0` to `i - 1` to compute the left sum
- Iterate from `i + 1` to `n - 1` to compute the right sum
- Store the absolute difference

This works correctly because it follows the definition exactly. However, for every index we repeatedly recompute sums that overlap heavily with previous computations.

For an array of size `n`, each index may require scanning nearly the entire array, resulting in:

$$O(n^2)$$

time complexity.

Even though the constraints are small enough for this approach to pass, it is inefficient and unnecessary.

### Optimal Prefix Sum Approach

The key observation is that left and right sums overlap heavily between adjacent indices.

Suppose we already know:

- the total sum of the array
- the running prefix sum while iterating

Then:

- `leftSum` can be maintained incrementally
- `rightSum` can be derived from the total sum

At index `i`:

$$rightSum = totalSum - leftSum - nums[i]$$

This works because:

- `totalSum` contains every element
- subtracting `leftSum` removes everything before `i`
- subtracting `nums[i]` removes the current element itself
- what remains is the sum of elements to the right

This allows us to compute every answer in a single pass.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Recomputes left and right sums for every index |
| Optimal | O(n) | O(1) extra space | Uses running prefix sum and total sum |

## Algorithm Walkthrough

### Optimal Algorithm

1. Compute the total sum of the array.

This allows us to derive right-side sums efficiently without repeatedly scanning the array.
2. Initialize a variable `left_sum = 0`.

This variable stores the cumulative sum of all elements before the current index.
3. Create an empty result array.

We will append each computed absolute difference into this array.
4. Iterate through the array from left to right.

For each index `i`, calculate the right-side sum using:

$$rightSum = totalSum - leftSum - nums[i]$$

This formula removes:

- everything already included in `left_sum`
- the current element itself

The remaining value is exactly the sum of elements to the right.
5. Compute the absolute difference.

Use:

$$abs(leftSum - rightSum)$$

Append the result to the answer array.
6. Update `left_sum`.

After processing index `i`, add `nums[i]` to `left_sum` so it becomes correct for the next iteration.
7. Return the completed answer array.

### Why it works

At every iteration, `left_sum` always represents the sum of elements strictly before the current index. Since `total_sum` contains the sum of the entire array, subtracting both `left_sum` and the current element leaves only the elements strictly after the current index. Therefore, the algorithm computes exactly the required values for every position.

## Python Solution

```python
from typing import List

class Solution:
    def leftRightDifference(self, nums: List[int]) -> List[int]:
        total_sum = sum(nums)
        left_sum = 0
        answer = []

        for num in nums:
            right_sum = total_sum - left_sum - num
            answer.append(abs(left_sum - right_sum))
            left_sum += num

        return answer
```

The implementation starts by computing the total sum of the array using Python's built-in `sum()` function.

The variable `left_sum` is initialized to zero because there are no elements to the left of the first index.

During each iteration:

- `right_sum` is derived from the total sum
- the absolute difference is appended to `answer`
- the current number is added into `left_sum`

The order is important. We must compute the answer before updating `left_sum`, otherwise the current element would incorrectly be included in the left-side sum.

Finally, the completed answer array is returned.

## Go Solution

```go
package main

import "math"

func leftRightDifference(nums []int) []int {
	totalSum := 0

	for _, num := range nums {
		totalSum += num
	}

	leftSum := 0
	answer := make([]int, 0, len(nums))

	for _, num := range nums {
		rightSum := totalSum - leftSum - num
		diff := int(math.Abs(float64(leftSum - rightSum)))

		answer = append(answer, diff)

		leftSum += num
	}

	return answer
}
```

The Go implementation follows the exact same logic as the Python solution.

One difference is that Go does not have a built-in absolute value function for integers in the standard library. The `math.Abs()` function operates on `float64`, so the value must be converted before and after calling it.

The solution uses slices instead of dynamic Python lists. The result slice is initialized with a capacity equal to the input size to avoid unnecessary reallocations.

Integer overflow is not a concern here because the maximum possible total sum is:

$$1000 \times 10^5 = 10^8$$

which safely fits inside Go's `int` type.

## Edge Cases

### Single Element Array

An array containing only one element is the smallest valid input.

```
nums = [1]
```

There are no elements on either side of the only index, so both left and right sums must be zero.

A common mistake is forgetting that empty ranges should contribute zero. This implementation handles the case naturally because:

- `left_sum` starts at `0`
- `right_sum = total_sum - left_sum - num = 1 - 0 - 1 = 0`

The result becomes `[0]`.

### First and Last Elements

The first index has no left-side elements, and the last index has no right-side elements.

Incorrect implementations sometimes accidentally include the current element itself in one of the sums.

This solution avoids that problem by using:

$$rightSum = totalSum - leftSum - nums[i]$$

The current value is explicitly removed from the right-side calculation, ensuring correctness at both boundaries.

### Arrays with Repeated Values

Arrays like:

```
[5,5,5]
```

can expose mistakes where indices are confused with values.

The algorithm never depends on element uniqueness. It only tracks cumulative sums, so repeated numbers are handled exactly the same as distinct values.

The middle index correctly produces:

```
|5 - 5| = 0
```

because the sums on both sides are equal.
