## Solution

## Problem Understanding

The problem asks us to compute the **divisibility array** for a given string of digits `word` with respect to a positive integer `m`. Specifically, for each prefix of the string, we want to know if the numeric value of that prefix is divisible by `m`. We define `div[i] = 1` if the prefix `word[0..i]` interpreted as an integer is divisible by `m`, otherwise `div[i] = 0`. The output is an array of length equal to `word`.

The input `word` is a string representing digits only, which could be very long, up to `10^5` characters. The integer `m` can also be quite large, up to `10^9`. These constraints suggest that directly converting each prefix to an integer for divisibility testing is not efficient because the numbers can be too large to handle naively and doing repeated string-to-integer conversions would be costly.

Important edge cases include strings with leading zeros, strings that are entirely zeros, and small values of `m` like `1`. The problem guarantees non-empty strings and valid integers `m`, which simplifies handling invalid inputs.

## Approaches

The **brute-force approach** would iterate through each prefix of `word`, convert it into an integer, and check divisibility by `m`. While this is conceptually straightforward and guarantees correctness, it is too slow for long strings due to repeated large integer conversions and high computational overhead.

The **key insight for an optimal solution** is to use the property of modular arithmetic: if we know the modulo of a prefix up to `i-1`, we can compute the modulo of the prefix up to `i` efficiently without converting the entire substring to an integer. Specifically, if `num_mod` is the value of the prefix modulo `m`, the next prefix modulo can be computed as:

```
num_mod = (num_mod * 10 + int(word[i])) % m
```

This allows us to incrementally compute divisibility in O(n) time, avoiding large integer arithmetic.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Converts each prefix string to an integer and checks divisibility |
| Optimal | O(n) | O(n) | Uses modular arithmetic to compute each prefix modulo efficiently |

## Algorithm Walkthrough

1. Initialize an empty list `div` to store the divisibility array.
2. Initialize a variable `prefix_mod` to 0. This variable will hold the modulo `m` of the prefix `word[0..i]`.
3. Iterate through each character `c` in the string `word` using its index `i`.
4. Convert the character `c` to its integer value and update `prefix_mod` using `prefix_mod = (prefix_mod * 10 + int(c)) % m`.
5. If `prefix_mod` equals 0, append 1 to `div` indicating the prefix is divisible by `m`. Otherwise, append 0.
6. Continue until all characters are processed.
7. Return the completed divisibility array `div`.

**Why it works:** The algorithm maintains an invariant: `prefix_mod` is always equal to the numeric value of `word[0..i]` modulo `m`. Modular arithmetic ensures that `(a * 10 + b) % m` correctly computes the modulo of the extended prefix, avoiding integer overflow and excessive computation. Thus, at each step, we correctly determine divisibility.

## Python Solution

```python
from typing import List

class Solution:
    def divisibilityArray(self, word: str, m: int) -> List[int]:
        div = []
        prefix_mod = 0
        for c in word:
            prefix_mod = (prefix_mod * 10 + int(c)) % m
            div.append(1 if prefix_mod == 0 else 0)
        return div
```

The implementation initializes the divisibility list and `prefix_mod` to zero. As we iterate through the digits, we update `prefix_mod` using modular arithmetic. Each computed modulo is checked for zero, and the result is appended to the `div` list. Finally, the function returns the full divisibility array.

## Go Solution

```go
func divisibilityArray(word string, m int) []int {
    div := make([]int, len(word))
    prefixMod := 0
    for i, c := range word {
        prefixMod = (prefixMod*10 + int(c-'0')) % m
        if prefixMod == 0 {
            div[i] = 1
        } else {
            div[i] = 0
        }
    }
    return div
}
```

In Go, we pre-allocate the `div` slice with the length of `word`. The conversion from character to integer is done using `c-'0'`, and modular arithmetic is applied similarly. There is no need for dynamic append because we know the array size in advance.
