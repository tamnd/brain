## Solution

## Problem Understanding

The problem asks us to take a positive integer `num` and split its digits into two non-negative integers `num1` and `num2` such that the sum of `num1` and `num2` is minimized. Importantly, the digits of `num1` and `num2` must together be a permutation of the digits in `num`. This means that each digit in `num` must appear exactly once across `num1` and `num2`, but the order in which digits appear in `num1` or `num2` does not matter, and leading zeros are allowed.

The input `num` is guaranteed to be at least 10 and at most $10^9$, so the number of digits ranges from 2 to 9. This ensures that the integer fits into standard 32-bit arithmetic types and is small enough that operations on the digits themselves are feasible. The output is the minimum sum of the two numbers formed by splitting the digits.

Edge cases to consider include numbers where digits are all identical, numbers with two digits only, numbers with zeros, and numbers whose digits, if not ordered carefully, could lead to larger sums than necessary. The problem guarantees no leading zeros in the original `num`, which simplifies input parsing.

## Approaches

The brute-force approach would generate all possible splits of the digits of `num` into two sets and then compute the sum of the corresponding numbers for each split. This would require iterating over all subsets of digits, converting each subset into a number, and computing the sum. While this approach guarantees correctness, it is computationally expensive. For a number with `d` digits, there are $2^d$ subsets, which becomes infeasible as `d` approaches 9.

The key insight for an optimal approach is that, to minimize the sum of two numbers formed from the digits, we should try to balance the digits as evenly as possible between the two numbers. Sorting the digits in ascending order and alternately assigning them to `num1` and `num2` ensures that both numbers grow slowly in magnitude, minimizing their sum. This approach leverages the fact that smaller digits in more significant positions reduce the number value more than small digits in less significant positions.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^d * d) | O(d) | Generate all subsets of digits, compute sums, track minimum. Exponential and impractical for d=9. |
| Optimal | O(d log d) | O(d) | Sort digits, distribute alternately to two numbers. Linear scan after sorting. Efficient and simple. |

## Algorithm Walkthrough

1. Convert the integer `num` into a list of its digits for easy manipulation.
2. Sort the digits in ascending order. This ensures that smaller digits are used earlier when constructing `num1` and `num2`.
3. Initialize two integers, `num1` and `num2`, to zero. These will store the split numbers.
4. Iterate over the sorted digits. Alternately append each digit to `num1` and `num2` by multiplying the current number by 10 and adding the new digit. This ensures that smaller digits occupy more significant positions when possible.
5. After all digits are assigned, return the sum of `num1` and `num2`.

Why it works: Alternating digits ensures that neither number accumulates disproportionately large digits in higher positions. Sorting guarantees that smaller digits are prioritized in the higher positions of both numbers. This strategy minimizes the sum across all possible permutations.

## Python Solution

```python
class Solution:
    def splitNum(self, num: int) -> int:
        digits = sorted([int(d) for d in str(num)])
        num1, num2 = 0, 0
        for i, digit in enumerate(digits):
            if i % 2 == 0:
                num1 = num1 * 10 + digit
            else:
                num2 = num2 * 10 + digit
        return num1 + num2
```

The Python solution converts `num` to a list of digits, sorts them, and iteratively constructs `num1` and `num2` by alternating digits. The use of modulo ensures that digits are distributed evenly, and the multiplication by 10 correctly shifts existing digits for place value.

## Go Solution

```go
import "sort"

func splitNum(num int) int {
    digits := []int{}
    for num > 0 {
        digits = append(digits, num % 10)
        num /= 10
    }
    sort.Ints(digits)
    num1, num2 := 0, 0
    for i, digit := range digits {
        if i % 2 == 0 {
            num1 = num1 * 10 + digit
        } else {
            num2 = num2 * 10 + digit
        }
    }
    return num1 + num2
}
```

In Go, we extract digits using modulo and integer division, sort the slice, and construct `num1` and `num2` similarly to Python. Since Go does not allow negative indices or dynamic slicing in the same way, we use simple integer arithmetic and slices.

## Worked Examples

For `num = 4325`, the sorted digits are `[2, 3, 4, 5]`. Assigning alternately, `num1 = 2*10 + 4 = 24` and `num2 = 3*10 + 5 = 35`. The sum is `24 + 35 = 59`.

For `num = 687`, sorted digits `[6, 7, 8]`. Alternating: `num1 = 6*10 + 8 = 68`, `num2 = 7`. Sum is `68 + 7 = 75`.

## Edge Cases

A critical edge case is when the number contains zeros. For example, `num = 109` must handle zero correctly without misplacing it in `num1` or `num2`, and the algorithm accommodates this because zeros are included in the sorted digits and do not change the alternating assignment logic.

Another edge case occurs with numbers that have all identical digits, like `1111`. Here, the algorithm still alternates digits correctly, ensuring the sum is minimized (e.g., 11 + 11 = 22).

A final edge case involves numbers with an odd number of digits. For `num = 687`, the algorithm alternates digits such that one number has one extra digit, but because the smallest digits are placed in higher positions of both numbers, the sum remains minimal. This shows that the alternating assignment works regardless of parity of digit count.
