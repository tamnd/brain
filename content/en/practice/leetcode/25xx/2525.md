## Solution

## Problem Understanding

This problem asks us to classify a box into one of four categories based on its dimensions and mass.

We are given four integers:

- `length`
- `width`
- `height`
- `mass`

The classification depends on two independent properties:

1. Whether the box is considered `"Bulky"`
2. Whether the box is considered `"Heavy"`

A box is `"Bulky"` if at least one of the following conditions is true:

- Any dimension is greater than or equal to `10^4`
- The volume is greater than or equal to `10^9`

The volume is calculated as:

$$\text{volume} = \text{length} \times \text{width} \times \text{height}$$

A box is `"Heavy"` if:

$$\text{mass} \ge 100$$

Once these two boolean properties are determined, we return:

- `"Both"` if the box is both bulky and heavy
- `"Bulky"` if only bulky
- `"Heavy"` if only heavy
- `"Neither"` if neither condition is true

The constraints are very small:

- Dimensions are at most `10^5`
- Mass is at most `10^3`

This tells us that performance is not a concern. We only perform a few arithmetic operations and comparisons. Even the largest possible volume is:

$$10^5 \times 10^5 \times 10^5 = 10^{15}$$

This fits safely inside Python integers. In Go, it also fits safely inside a 64-bit integer, although using plain `int` is still fine on LeetCode's environment.

An important detail is that the comparisons use "greater than or equal to". Boundary values matter. For example:

- A dimension exactly equal to `10000` makes the box bulky
- A volume exactly equal to `1000000000` also makes it bulky
- A mass exactly equal to `100` makes it heavy

These edge values are common sources of off-by-one mistakes.

## Approaches

### Brute Force Approach

A brute-force mindset would treat each classification possibility independently and manually check every combination of states.

For example, we could:

- First compute whether the box satisfies each bulky rule
- Then separately compute whether it satisfies the heavy rule
- Then explicitly test all four possible outcomes:

- bulky and heavy
- bulky only
- heavy only
- neither

This approach is already fast because there is only a constant amount of work. There is no iteration over input data.

The reason this works is that the problem has only a fixed number of conditions. Every possible category can be derived directly from the input values.

However, the implementation can become unnecessarily verbose if we repeatedly recompute conditions or structure the logic inefficiently.

### Optimal Approach

The key insight is that the problem naturally decomposes into two boolean properties:

- `is_bulky`
- `is_heavy`

Once these two values are computed, the final answer becomes a straightforward mapping.

This leads to a clean and maintainable solution:

1. Compute volume
2. Determine whether the box is bulky
3. Determine whether the box is heavy
4. Return the appropriate category based on the two boolean flags

Since all operations are constant time arithmetic and comparisons, the solution runs in constant time and constant space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(1) | O(1) | Explicitly checks all classification combinations |
| Optimal | O(1) | O(1) | Uses two boolean flags and direct classification |

## Algorithm Walkthrough

1. Compute the volume of the box using:

$$\text{volume} = \text{length} \times \text{width} \times \text{height}$$

We need this because one of the bulky conditions depends on volume.
2. Determine whether the box is bulky.

The box is bulky if:

- Any dimension is at least `10000`
- Or the volume is at least `1000000000`

We combine these conditions into a single boolean variable named `is_bulky`.
3. Determine whether the box is heavy.

The box is heavy if:

$$\text{mass} \ge 100$$

Store this result in a boolean variable named `is_heavy`.
4. Determine the final category.

There are four possible combinations:

- If both `is_bulky` and `is_heavy` are true, return `"Both"`
- If only `is_bulky` is true, return `"Bulky"`
- If only `is_heavy` is true, return `"Heavy"`
- Otherwise return `"Neither"`

### Why it works

The algorithm works because the problem definition itself is based entirely on two independent logical properties: bulky and heavy. Every valid output corresponds exactly to one combination of these two boolean states. By computing these states directly from the problem rules, the algorithm guarantees the correct classification.

## Python Solution

```python
class Solution:
    def categorizeBox(self, length: int, width: int, height: int, mass: int) -> str:
        volume = length * width * height

        is_bulky = (
            length >= 10000 or
            width >= 10000 or
            height >= 10000 or
            volume >= 1000000000
        )

        is_heavy = mass >= 100

        if is_bulky and is_heavy:
            return "Both"

        if is_bulky:
            return "Bulky"

        if is_heavy:
            return "Heavy"

        return "Neither"
```

The implementation follows the algorithm directly.

First, the code computes the volume using simple multiplication. This is necessary because one of the bulky conditions depends on the total volume.

Next, the variable `is_bulky` combines all bulky criteria into a single boolean expression. Using a boolean variable improves readability and avoids repeated condition checks later in the code.

Similarly, `is_heavy` stores whether the mass threshold is met.

The final section performs the classification. The `"Both"` case must be checked first because it is the most specific condition. After that, we handle the remaining single-property cases before finally returning `"Neither"`.

The logic is concise because the problem naturally maps to boolean state combinations.

## Go Solution

```go
func categorizeBox(length int, width int, height int, mass int) string {
    volume := length * width * height

    isBulky := length >= 10000 ||
        width >= 10000 ||
        height >= 10000 ||
        volume >= 1000000000

    isHeavy := mass >= 100

    if isBulky && isHeavy {
        return "Both"
    }

    if isBulky {
        return "Bulky"
    }

    if isHeavy {
        return "Heavy"
    }

    return "Neither"
}
```

The Go implementation is almost identical to the Python version.

One small consideration in Go is integer overflow. The maximum volume can reach `10^15`, which fits safely inside a 64-bit integer. On LeetCode's Go environment, `int` is large enough for this range, so no special handling is required.

The boolean logic and classification structure remain the same.

## Edge Cases

One important edge case is when a dimension is exactly `10000`. Many implementations accidentally use `>` instead of `>=`, which would incorrectly classify the box as not bulky. The implementation explicitly uses `>=` for all threshold checks, so this boundary case is handled correctly.

Another important case is when the volume is exactly `1000000000`. Since the problem states that the volume must be greater than or equal to this value, equality must count as bulky. The implementation checks `volume >= 1000000000`, ensuring correct behavior.

A third important edge case occurs when the box satisfies both bulky and heavy conditions simultaneously. If the implementation checks `"Bulky"` or `"Heavy"` first and returns immediately, it could miss the `"Both"` category entirely. This implementation avoids that bug by checking `is_bulky and is_heavy` before the individual cases.

A final subtle edge case involves very large dimensions. The maximum possible volume is `10^15`, which exceeds 32-bit integer range. Python handles this automatically because integers are arbitrary precision. In Go, the LeetCode environment uses sufficiently large integers, so the multiplication remains safe.
