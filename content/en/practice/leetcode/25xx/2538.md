## Solution

## Problem Understanding

This problem involves analyzing a tree, an undirected, connected acyclic graph. Each node has an associated price, and the goal is to determine the **maximum possible difference between the largest and smallest path price sums** when the tree is rooted at any node.

Formally, for a chosen root node, each path that starts at the root has a **price sum**, the sum of the node prices along that path. The **cost** for that root is the difference between the maximum and minimum price sums over all paths starting at that root. We need to find the maximum possible cost considering all possible choices of the root.

The input consists of the number of nodes `n` (up to $10^5$), a list of `edges` defining the tree, and a `price` array for node values. The constraints guarantee the structure is a valid tree and that node prices are positive integers. Edge cases to consider include single-node trees, trees with uniform prices, and highly skewed trees (like chains).

A naive approach that computes all paths for every possible root is infeasible due to the large input size. Efficient computation requires a tree-based traversal strategy like depth-first search (DFS) to calculate path sums recursively.

## Approaches

### Brute Force

A brute-force method would involve iterating over each node as a root, performing a traversal to compute all paths from that root, recording the sums, and then calculating the difference between the maximum and minimum sums. This approach guarantees correctness but is extremely inefficient. For a tree of $n$ nodes, there are $O(n^2)$ paths, and computing all paths for all roots leads to $O(n^3)$ complexity, which is completely infeasible for $n$ up to $10^5$.

### Optimal Approach

The key observation is that the **maximum path sum from a node depends only on the largest path sums from its children**, and similarly, the minimum path sum depends on the smallest path sums from its children. Using **depth-first search**, we can compute for each node the maximum and minimum price sum of any path starting from it downwards. Once these values are computed, the **maximum cost for a root** is simply the difference between its computed maximum and minimum path sums. This DFS approach allows us to calculate the values in a single traversal for all nodes.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n^2) | Compute all path sums for every root explicitly |
| Optimal | O(n) | O(n) | Single DFS traversal storing max and min path sums per node |

## Algorithm Walkthrough

1. **Graph Construction:** Convert the edge list into an adjacency list for efficient traversal.
2. **DFS Function Definition:** Define a recursive DFS function that returns two values for a given node: `max_sum` and `min_sum`, representing the maximum and minimum path price sums starting at that node.
3. **Recursive Traversal:** For each child node (excluding the parent to avoid cycles), recursively call the DFS function to compute its `max_sum` and `min_sum`.
4. **Aggregate Results:** Update the current node's `max_sum` as the node's own price plus the maximum `max_sum` of its children. Similarly, update the `min_sum` as the node's own price plus the minimum `min_sum` of its children.
5. **Update Global Maximum Cost:** At each node, calculate the difference between `max_sum` and `min_sum`, updating a global variable `max_cost` if this difference exceeds the current maximum.
6. **Return DFS Values:** Return the `max_sum` and `min_sum` for the current node to its parent call, allowing the parent to continue aggregation.

**Why it works:** The algorithm works because in a tree, each path starting at a node can be fully characterized by choosing a child subtree to continue the path. By recursively propagating the maximum and minimum path sums upward, every potential root's cost is correctly computed. No path is counted twice due to the tree's acyclic property.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def maxOutput(self, n: int, edges: List[List[int]], price: List[int]) -> int:
        graph = defaultdict(list)
        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)
        
        max_cost = 0
        
        def dfs(node: int, parent: int) -> (int, int):
            nonlocal max_cost
            current_max = price[node]
            current_min = price[node]
            for child in graph[node]:
                if child == parent:
                    continue
                child_max, child_min = dfs(child, node)
                current_max = max(current_max, price[node] + child_max)
                current_min = min(current_min, price[node] + child_min)
            max_cost = max(max_cost, current_max - current_min)
            return current_max, current_min
        
        dfs(0, -1)
        return max_cost
```

The Python solution constructs the tree using a defaultdict, performs a single DFS, and keeps track of the maximum cost globally. The DFS avoids revisiting the parent to prevent cycles and computes both the maximum and minimum path sums for each node. Finally, the maximum cost across all nodes is returned.

## Go Solution

```go
package main

func maxOutput(n int, edges [][]int, price []int) int64 {
    graph := make([][]int, n)
    for _, e := range edges {
        u, v := e[0], e[1]
        graph[u] = append(graph[u], v)
        graph[v] = append(graph[v], u)
    }

    var maxCost int64 = 0

    var dfs func(node, parent int) (int64, int64)
    dfs = func(node, parent int) (int64, int64) {
        currentMax := int64(price[node])
        currentMin := int64(price[node])
        for _, child := range graph[node] {
            if child == parent {
                continue
            }
            childMax, childMin := dfs(child, node)
            if price[node]+int(childMax) > int(currentMax) {
                currentMax = price[node] + childMax
            }
            if price[node]+int(childMin) < int(currentMin) {
                currentMin = price[node] + childMin
            }
        }
        if currentMax-currentMin > maxCost {
            maxCost = currentMax - currentMin
        }
        return currentMax, currentMin
    }

    dfs(0, -1)
    return maxCost
}
```

The Go solution mirrors the Python logic, using slices for adjacency lists and `int64` to handle potential overflow. Recursive DFS returns both maximum and minimum path sums and updates a global variable `maxCost`.
