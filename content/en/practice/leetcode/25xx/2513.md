## Solution

## Problem Understanding

This problem asks us to construct two arrays of positive integers, `arr1` and `arr2`, such that each array satisfies a specific divisibility constraint, each contains a specified number of **distinct** integers, and no integer is shared between the arrays. Specifically, `arr1` must have `uniqueCnt1` integers none of which are divisible by `divisor1`, and `arr2` must have `uniqueCnt2` integers none of which are divisible by `divisor2`. The objective is to **minimize the maximum integer** that appears in either array.

In simpler terms, we are picking integers starting from 1 upwards and distributing them between the two arrays while respecting the divisibility rules and the uniqueness constraints. The "minimum possible maximum integer" is the smallest number such that we can construct both arrays with the given rules using numbers only up to that value.

The input consists of four positive integers: `divisor1`, `divisor2`, `uniqueCnt1`, and `uniqueCnt2`. The output is a single integer representing the minimum possible maximum number. Constraints indicate that the counts can be very large, up to nearly a billion, and the divisors can go up to 100,000, so a naive iteration over numbers would be too slow.

Important edge cases include when `divisor1` and `divisor2` are the same, when one of the counts is much larger than the other, or when a number is divisible by both divisors (we must avoid assigning such numbers to both arrays).

## Approaches

### Brute Force

A brute-force approach would be to iterate through the positive integers sequentially and keep assigning each number to `arr1` or `arr2` if it satisfies the respective divisibility constraint and is not already in the other array. We would stop when both arrays have reached the required sizes and return the maximum number assigned. This approach is correct because it simulates the problem directly.

However, it is too slow. The counts `uniqueCnt1` and `uniqueCnt2` can each be as large as 10^9, so iterating one by one is infeasible. Additionally, checking membership in arrays would require extra memory or additional time, making this approach unrealistic.

### Optimal Approach

The key insight for an optimal solution is that we do **not need to know the exact numbers** in the arrays; we only need to know **how many numbers up to a certain value are valid for each array**. We can use the inclusion-exclusion principle:

- For any number `x`, the count of integers not divisible by `divisor1` up to `x` is `x - x // divisor1`.
- Similarly, for `divisor2`, it is `x - x // divisor2`.
- Numbers divisible by both divisors are double-counted if we naively sum, so we subtract `x // lcm(divisor1, divisor2)` for numbers divisible by both.

Using these formulas, we can determine whether a given maximum candidate `x` is sufficient to satisfy the required counts. This makes the problem suitable for **binary search**: we search over possible maximum values until we find the smallest one that can accommodate both arrays.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(uniqueCnt1 + uniqueCnt2) | O(uniqueCnt1 + uniqueCnt2) | Simulates the arrays directly, too slow for large inputs |
| Optimal | O(log((uniqueCnt1 + uniqueCnt2) * max(divisor1, divisor2))) | O(1) | Uses binary search and arithmetic counting, memory efficient |

## Algorithm Walkthrough

1. Compute the **least common multiple (LCM)** of `divisor1` and `divisor2`. This is necessary for applying inclusion-exclusion to avoid double-counting numbers divisible by both divisors.
2. Define a **helper function** `enough(x)` that checks whether it is possible to select the required numbers with `x` as the maximum value. Count numbers available for each array:

- `cnt1 = x - x // divisor1` counts numbers not divisible by `divisor1`.
- `cnt2 = x - x // divisor2` counts numbers not divisible by `divisor2`.
- `both = x // lcm(divisor1, divisor2)` counts numbers divisible by both divisors.
- The numbers that can be shared (need to be assigned carefully) are `cnt1 + cnt2 - (x - both) = cnt1 + cnt2 - x + both`.
3. For the check, subtract the overlap numbers if necessary to ensure both counts meet `uniqueCnt1` and `uniqueCnt2`.
4. Use **binary search** between `1` and `uniqueCnt1 + uniqueCnt2` times the maximum of the divisors. For each candidate middle value `mid`, check if `enough(mid)` is true:

- If yes, reduce the search space to smaller numbers.
- If no, increase the search space.
5. After the binary search terminates, return the smallest `x` that satisfies the conditions.

Why it works: the helper function correctly counts how many numbers are available for each array up to a maximum candidate value. Binary search ensures we find the minimal such maximum efficiently.

## Python Solution

```python
import math

class Solution:
    def minimizeSet(self, divisor1: int, divisor2: int, uniqueCnt1: int, uniqueCnt2: int) -> int:
        def lcm(a: int, b: int) -> int:
            return a * b // math.gcd(a, b)
        
        def enough(x: int) -> bool:
            both = x // lcm(divisor1, divisor2)
            cnt1 = x - x // divisor1
            cnt2 = x - x // divisor2
            return cnt1 + cnt2 - both >= uniqueCnt1 + uniqueCnt2 and cnt1 >= uniqueCnt1 and cnt2 >= uniqueCnt2
        
        left, right = 1, (uniqueCnt1 + uniqueCnt2) * max(divisor1, divisor2)
        result = right
        while left <= right:
            mid = (left + right) // 2
            if enough(mid):
                result = mid
                right = mid - 1
            else:
                left = mid + 1
        return result
```

The Python solution defines an `lcm` function for computing the least common multiple, then a helper `enough` that checks whether a candidate maximum satisfies the constraints using inclusion-exclusion. Binary search finds the minimal such value efficiently.

## Go Solution

```go
import "math"

func gcd(a, b int) int {
    for b != 0 {
        a, b = b, a%b
    }
    return a
}

func lcm(a, b int) int {
    return a / gcd(a, b) * b
}

func minimizeSet(divisor1 int, divisor2 int, uniqueCnt1 int, uniqueCnt2 int) int {
    enough := func(x int) bool {
        both := x / lcm(divisor1, divisor2)
        cnt1 := x - x/divisor1
        cnt2 := x - x/divisor2
        return cnt1+cnt2-both >= uniqueCnt1+uniqueCnt2 && cnt1 >= uniqueCnt1 && cnt2 >= uniqueCnt2
    }

    left, right := 1, (uniqueCnt1+uniqueCnt2)*max(divisor1, divisor2)
    result := right
    for left <= right {
        mid := (left + right) / 2
        if enough(mid) {
            result = mid
            right = mid - 1
        } else {
            left = mid + 1
        }
    }
    return result
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

In Go, we define `gcd` and `lcm` functions explicitly and use integer arithmetic carefully to avoid overflow. The binary search logic is identical to Python, and Go handles large integers adequately because the problem constraints fit in 64-bit integers.
