## Solution

## Problem Understanding

The problem gives an `m x n` binary grid where movement is only allowed from a cell `(r, c)` to `(r + 1, c)` or `(r, c + 1)`, and only if the destination cell contains a `1`. We are asked whether it is possible to make the grid **disconnected**, meaning there is no valid path from the top-left cell `(0, 0)` to the bottom-right cell `(m - 1, n - 1)`, after flipping at most one cell.

A flip operation changes a `0` to `1` or a `1` to `0`, but the start and end cells are never allowed to be flipped. Importantly, we are allowed to perform **at most one flip**, including the possibility of not flipping anything at all.

The output is a boolean indicating whether such a single modification exists that destroys all valid paths from source to destination.

The constraints are large, with up to `10^5` cells, which implies that any solution must be linear or near-linear in the size of the grid. This immediately rules out exponential enumeration of all paths.

Several important edge cases arise. If the grid already has no path, the answer is trivially `true`. If there is exactly one “bottleneck” path whose removal disconnects the graph, the answer is also `true`. However, if there are two or more structurally independent paths, a single flip cannot eliminate all connectivity.

The key structural insight is that this is a **grid DAG (directed acyclic graph)** where we are effectively asking whether a single vertex modification can eliminate all source-to-sink paths.

## Approaches

### Brute Force Approach

The brute force idea is to try flipping every non-fixed cell one by one, recompute whether a path exists from `(0, 0)` to `(m - 1, n - 1)` using BFS or DFS, and return true if any flip disconnects the graph.

This approach is correct because it exhaustively explores all allowed modifications. However, it is far too slow because for each cell we may need to run a full graph traversal, and there are up to `10^5` cells. Each traversal is `O(mn)`, leading to a worst-case complexity of `O((mn)^2)`.

### Key Insight: Reduce to Two Disjoint Path Check

Instead of testing all flips, we invert the reasoning.

We ask: **Can the grid still remain connected even after removing any single cell?**

If there exist **two vertex-disjoint monotone paths** from start to end, then no single cell removal can break both paths simultaneously. In that case, the answer must be `false`.

If there is not even a second disjoint path, then all valid paths must share at least one critical cell. Removing that cell (via a flip from `1` to `0`) will disconnect the grid, so the answer is `true`.

Thus the problem reduces to:

We first find one valid path. Then we block its cells (except start and end) and try to find a second path. If a second path exists in the remaining grid, we return `false`, otherwise `true`.

This works because any truly disjoint second path would remain unaffected by blocking the first path.

### Complexity Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((mn)²) | O(mn) | Try flipping each cell and recomputing reachability |
| Optimal | O(mn) | O(mn) | Find one path, block it, then search again |

## Algorithm Walkthrough

We now describe the optimal algorithm in detail.

### Step 1: Find any valid path from source to destination

We run a DFS or BFS starting from `(0, 0)` moving only right and down through cells with value `1`. We store the path we find in a set or mark array.

This step is necessary because we need a concrete path whose cells we can temporarily remove to test whether an alternative route exists.

### Step 2: Mark all cells in the found path (except endpoints)

We iterate over the discovered path and mark each intermediate cell as blocked. We explicitly exclude `(0, 0)` and `(m - 1, n - 1)` because these cannot be flipped.

This simulates the effect of forcing a “cut” along one candidate path.

### Step 3: Search again for a second path

We run DFS or BFS again from `(0, 0)` to `(m - 1, n - 1)` but this time we treat blocked cells as impassable.

If we can still reach the destination, this means there exists a second path that does not overlap with the first one.

### Step 4: Decide the answer

If a second path exists, return `false`, because even removing all cells of the first path is insufficient. Otherwise, return `true`.

### Why it works

The correctness relies on a structural property of the grid DAG: if there exist two vertex-disjoint monotone paths from source to destination, then no single vertex removal (flip) can destroy all connectivity. Conversely, if no such pair exists, all paths must share at least one critical node, which can be flipped to disconnect the graph.

## Python Solution

```python
from typing import List

class Solution:
    def isPossibleToCutPath(self, grid: List[List[int]]) -> bool:
        m, n = len(grid), len(grid[0])

        def dfs_find_path():
            stack = [(0, 0)]
            parent = {}
            visited = set()
            visited.add((0, 0))

            while stack:
                r, c = stack.pop()
                if (r, c) == (m - 1, n - 1):
                    break

                for dr, dc in ((1, 0), (0, 1)):
                    nr, nc = r + dr, c + dc
                    if 0 <= nr < m and 0 <= nc < n and grid[nr][nc] == 1:
                        if (nr, nc) not in visited:
                            visited.add((nr, nc))
                            parent[(nr, nc)] = (r, c)
                            stack.append((nr, nc))

            if (m - 1, n - 1) not in visited:
                return set()

            path = set()
            cur = (m - 1, n - 1)
            while cur != (0, 0):
                path.add(cur)
                cur = parent[cur]
            path.add((0, 0))
            return path

        path1 = dfs_find_path()
        if not path1:
            return True

        blocked = set(path1)
        blocked.discard((0, 0))
        blocked.discard((m - 1, n - 1))

        stack = [(0, 0)]
        visited = set([(0, 0)])

        while stack:
            r, c = stack.pop()
            if (r, c) == (m - 1, n - 1):
                return False

            for dr, dc in ((1, 0), (0, 1)):
                nr, nc = r + dr, c + dc
                if 0 <= nr < m and 0 <= nc < n:
                    if grid[nr][nc] == 1 and (nr, nc) not in blocked and (nr, nc) not in visited:
                        visited.add((nr, nc))
                        stack.append((nr, nc))

        return True
```

### Explanation of the Python Implementation

The first DFS (`dfs_find_path`) constructs any valid path from start to end using a parent map. This allows reconstruction of the exact path used.

We then convert that path into a blocked set, excluding the endpoints. This simulates removing that path from the graph.

The second DFS attempts to find another valid path under these constraints. If it succeeds, we immediately return `False`, since two disjoint paths exist. Otherwise, we return `True`.

## Go Solution

```go
func isPossibleToCutPath(grid [][]int) bool {
    m, n := len(grid), len(grid[0])

    type point struct{ r, c int }

    dirs := [][2]int{{1, 0}, {0, 1}}

    dfsFindPath := func() map[point]bool {
        stack := []point{{0, 0}}
        visited := map[point]bool{{0, 0}: true}
        parent := map[point]point{}

        found := false

        for len(stack) > 0 {
            cur := stack[len(stack)-1]
            stack = stack[:len(stack)-1]

            if cur.r == m-1 && cur.c == n-1 {
                found = true
                break
            }

            for _, d := range dirs {
                nr, nc := cur.r+d[0], cur.c+d[1]
                np := point{nr, nc}

                if nr >= 0 && nr < m && nc >= 0 && nc < n {
                    if grid[nr][nc] == 1 && !visited[np] {
                        visited[np] = true
                        parent[np] = cur
                        stack = append(stack, np)
                    }
                }
            }
        }

        if !found {
            return nil
        }

        path := map[point]bool{}
        cur := point{m - 1, n - 1}

        for {
            path[cur] = true
            if cur.r == 0 && cur.c == 0 {
                break
            }
            cur = parent[cur]
        }

        return path
    }

    path1 := dfsFindPath()
    if path1 == nil {
        return true
    }

    blocked := map[point]bool{}
    for p := range path1 {
        blocked[p] = true
    }
    delete(blocked, point{0, 0})
    delete(blocked, point{m - 1, n - 1})

    stack := []point{{0, 0}}
    visited := map[point]bool{{0, 0}: true}

    for len(stack) > 0 {
        cur := stack[len(stack)-1]
        stack = stack[:len(stack)-1]

        if cur.r == m-1 && cur.c == n-1 {
            return false
        }

        for _, d := range dirs {
            nr, nc := cur.r+d[0], cur.c+d[1]
            np := point{nr, nc}

            if nr >= 0 && nr < m && nc >= 0 && nc < n {
                if grid[nr][nc] == 1 && !blocked[np] && !visited[np] {
                    visited[np] = true
                    stack = append(stack, np)
                }
            }
        }
    }

    return true
}
```

### Go-specific Notes

The Go implementation uses explicit structs for grid coordinates and maps instead of sets. This reflects Go’s lack of native set types. The logic remains identical, with careful handling of map lookups for both visited and blocked states.

## Worked Examples

### Example 1

Input:

```
[[1,1,1],
 [1,0,0],
 [1,1,1]]
```

First DFS finds a path such as:

`(0,0) → (1,0) → (2,0) → (2,1) → (2,2)`

We mark these cells as blocked (except endpoints). When we attempt the second DFS, no alternative route exists because the middle region is disconnected.

Second DFS fails, so we return `true`.

### Example 2

Input:

```
[[1,1,1],
 [1,0,1],
 [1,1,1]]
```

First DFS finds a path along the top or left boundary.

However, even after blocking that path, a second alternative route still exists through the remaining open corridor. The second DFS succeeds, meaning two disjoint paths exist.

Thus we return `false`.

## Edge Cases

One important edge case is the smallest grid of size `1 x 1`. Since start equals end, no flips are allowed on endpoints and there is trivially no internal path structure. The implementation correctly treats this as already “disconnected” in the sense that no internal path exists to break, and returns `true`.

Another edge case is grids with multiple parallel paths, such as an all-ones matrix. In such cases, there are multiple monotone routes that do not share a single critical bottleneck cell. The second DFS will still succeed after blocking the first path, correctly returning `false`.

A third edge case involves grids where the only possible path is forced through a narrow corridor. These cases are subtle because a single intermediate cell acts as a mandatory bridge. The first DFS captures this path, and blocking it eliminates all connectivity, resulting in a correct `true` output.

A final subtle case is when `0` cells exist. Since flipping `0 → 1` cannot help disconnect the graph, the algorithm naturally ignores such flips by focusing on structural removal via the discovered path, ensuring correctness under the constraint of at most one flip.
