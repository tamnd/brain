## Solution

## Problem Understanding

The problem asks us to count the number of **non-empty subsets** of a given array `nums` where no two integers have an absolute difference equal to a given integer `k`. In simpler terms, we want to generate all possible subsets of `nums` (except the empty subset) and discard those that contain any two numbers that differ by exactly `k`. The remaining subsets are called **beautiful**.

The input `nums` is an array of positive integers, with length up to 18. The integer `k` is also positive and at most 1000. The constraints indicate that `nums` is small enough to allow enumeration of all subsets using a backtracking or bitmask approach, since the total number of subsets is `2^n` and `2^18 ≈ 260,000`, which is feasible.

Important edge cases include arrays with repeated numbers, arrays with a single element, and arrays where `k` is larger than the maximum difference between numbers. The problem guarantees all numbers are positive, so we do not need to handle zero or negative numbers.

## Approaches

A **brute-force approach** would enumerate all subsets of `nums`, check each subset for the forbidden difference condition, and count only those subsets that are beautiful. This works correctly but requires `O(2^n * n^2)` time because for each subset we might need to compare all pairs of elements to check if any difference equals `k`. This is too slow for `n = 18` in the worst case.

A **better approach** is to use **backtracking with pruning**. We can construct subsets element by element, and at each step, check if adding the current number would violate the beautiful subset condition. To efficiently check violations, we can maintain a frequency map of numbers in the current subset. When adding a number `x`, if the subset already contains `x - k` or `x + k`, we cannot include `x`. Otherwise, we include it and continue. This reduces unnecessary exploration and ensures we only generate valid subsets.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n^2) | O(n) | Generate all subsets and check each pair for forbidden difference |
| Backtracking / Optimal | O(2^n) | O(n + max(nums)) | Generate subsets recursively and use a frequency map to enforce constraints |

## Algorithm Walkthrough

1. Sort `nums` to help manage the ordering of elements. Sorting is optional for correctness but can make the implementation cleaner.
2. Initialize a global counter `count` to 0 and a dictionary `freq` to track how many times each number appears in the current subset.
3. Define a recursive function `backtrack(index)` that considers elements from `nums[index:]`.
4. At each index, attempt **two choices**: include `nums[index]` in the current subset or skip it.
5. If including `nums[index]`, check `freq` for the presence of `nums[index] - k` or `nums[index] + k`. If either exists, skip this branch.
6. If included, increment the frequency of `nums[index]` in `freq` and recurse to the next index.
7. If skipped, recurse without changing `freq`.
8. Every time a non-empty subset is constructed (at the leaf of recursion), increment `count`.
9. Return `count` after exploring all possibilities.

**Why it works:** The backtracking generates all possible subsets recursively, while the frequency map ensures we only include elements that do not violate the "beautiful" condition. This guarantees we count all valid subsets exactly once.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def beautifulSubsets(self, nums: List[int], k: int) -> int:
        count = 0
        freq = defaultdict(int)

        def backtrack(index: int):
            nonlocal count
            if index == len(nums):
                if sum(freq.values()) > 0:
                    count += 1
                return
            # Choice 1: Skip the current number
            backtrack(index + 1)
            # Choice 2: Include the current number if no conflict
            if freq[nums[index] - k] == 0 and freq[nums[index] + k] == 0:
                freq[nums[index]] += 1
                backtrack(index + 1)
                freq[nums[index]] -= 1

        backtrack(0)
        return count
```

The Python code uses `defaultdict` to simplify frequency checks. At each step, we decide whether to include the current number or skip it, maintaining the beautiful subset constraint. The `sum(freq.values()) > 0` ensures we only count non-empty subsets.

## Go Solution

```go
package main

func beautifulSubsets(nums []int, k int) int {
    count := 0
    freq := make(map[int]int)
    var backtrack func(index int)
    backtrack = func(index int) {
        if index == len(nums) {
            if len(freq) > 0 {
                count++
            }
            return
        }
        // Skip current number
        backtrack(index + 1)
        // Include current number if no conflict
        if freq[nums[index]-k] == 0 && freq[nums[index]+k] == 0 {
            freq[nums[index]]++
            backtrack(index + 1)
            freq[nums[index]]--
            if freq[nums[index]] == 0 {
                delete(freq, nums[index])
            }
        }
    }
    backtrack(0)
    return count
}
```

The Go version handles the map slightly differently since Go maps cannot have zero values for missing keys implicitly. We also delete keys when their count drops to zero to accurately check subset non-emptiness.

## Worked Examples

### Example 1

`nums = [2,4,6], k = 2`

Step through backtracking:

| Step | Current Subset | Decision | Reason |
| --- | --- | --- | --- |
| 0 | [] | include 2 | no conflict |
| 1 | [2] | include 4? | 4-2=2 conflict → skip |
| 2 | [2] | include 6 | 6-2=4 → freq[4]=0, include → [2,6] |
| 3 | [] | include 4 | [4] |
| 4 | [] | include 6 | [6] |

Beautiful subsets: `[2]`, `[6]`, `[4]`, `[2,6]` → total 4

### Example 2

`nums = [1], k = 1`

Step through backtracking:

Include 1 → `[1]` → beautiful, count=1

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(2^n) | Each element has two choices: include or skip. |
| Space | O(n + max(nums)) | Recursion stack O(n) and frequency map O(max(nums)) |

The exponential time complexity is acceptable because `n <= 18`. Space is dominated by recursion depth and the map used to check constraints.

## Edge Cases

**Single-element array:** Only one subset exists, which is non-empty and automatically beautiful. Both Python and Go implementations correctly return 1.

**All numbers identical:** If `k=0`, including more than one identical number violates the condition. The algorithm uses the frequency map to prevent adding conflicting numbers multiple times.

**Large `k` value:** When `k` is larger than the maximum difference in the array, no numbers conflict. The algorithm correctly counts all non-empty subsets.

**Empty subset exclusion:** The problem requires non-empty subsets. Both implementations use either the sum of frequencies (Python) or map length (Go) to ensure empty subsets are not counted.
