## Solution

## Problem Understanding

The problem gives us a numeric string `s` and an integer `k`. We must split the string into contiguous substrings such that every character belongs to exactly one substring, and the integer value represented by each substring is less than or equal to `k`.

Our goal is not just to find any valid partition, but specifically the partition that uses the fewest substrings possible.

For example, if `s = "165462"` and `k = 60`, we can split the string as:

```
"16" | "54" | "6" | "2"
```

Each substring represents a value less than or equal to `60`, so this is valid. The answer is `4` because no valid partition exists with fewer substrings.

The constraints are important:

- The string length can be as large as `10^5`
- `k` can be as large as `10^9`
- Digits are only from `'1'` to `'9'`

The input size immediately tells us that exponential or quadratic dynamic programming solutions will likely be too slow. We need something close to linear time.

Another important observation is that there are no `'0'` digits. Every digit is between `'1'` and `'9'`. This simplifies parsing because we never need to worry about leading zeros.

Several edge cases are important:

- A single digit may already be greater than `k`. In that case, no partition is possible and we return `-1`.
- Very large strings require an efficient streaming-style solution.
- Greedy decisions could appear risky at first because partitioning problems often require dynamic programming, so we need to carefully justify why greediness works here.
- If `k` is very large, the optimal answer may simply be `1`.

## Approaches

### Brute Force Approach

A brute-force solution would try every possible way to partition the string.

For a string of length `n`, there are `2^(n-1)` possible partition configurations because every gap between adjacent characters can either contain a split or not contain a split.

For each partition, we would:

1. Build all substrings
2. Convert them to integers
3. Check whether every substring value is at most `k`
4. Count the number of substrings
5. Track the minimum valid count

This approach is correct because it exhaustively checks all possibilities, guaranteeing that the minimum valid partition is found.

However, it is completely impractical for `n = 10^5`. Even for moderate string lengths, the number of partitions becomes astronomically large.

### Key Insight for the Optimal Solution

The crucial observation is that we should always make each substring as large as possible before starting a new partition.

Suppose we are building the current substring from left to right. If appending the next digit still keeps the value less than or equal to `k`, then delaying the split is always beneficial because:

- Larger substrings reduce the total number of partitions
- Splitting earlier cannot help create fewer partitions later

This means a greedy strategy works:

- Keep extending the current substring while its value remains at most `k`
- As soon as adding another digit would exceed `k`, start a new substring

This produces the minimum number of partitions because every partition is maximally extended.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Tries every possible partition |
| Optimal Greedy | O(n) | O(1) | Builds the largest valid substring at every step |

## Algorithm Walkthrough

1. Initialize a counter `partitions = 1` because we start with one substring being built.
2. Maintain a variable `current_value` representing the integer value of the current substring.
3. Iterate through the string character by character.
4. Convert the current digit character into its numeric value.
5. Before extending the current substring, check whether the digit itself is larger than `k`. If so, return `-1` immediately because no valid partition can exist.
6. Compute the new candidate value:

```
new_value = current_value * 10 + digit
```

This simulates appending the digit to the current substring.
7. If `new_value <= k`, continue extending the current substring because making substrings larger helps minimize the total partition count.
8. Otherwise, start a new partition:

- Increment `partitions`
- Reset `current_value` to the current digit
9. Continue until all digits are processed.
10. Return the total number of partitions.

### Why it works

The greedy choice is optimal because every substring is extended as much as possible before splitting.

Suppose we split earlier even though adding another digit would still keep the value at most `k`. That earlier split would create an additional substring unnecessarily. Since future digits are unaffected by delaying the split, extending the current substring can never increase the number of partitions needed later.

Therefore, the strategy of always taking the longest valid substring at each step minimizes the total number of substrings.

## Python Solution

```python
class Solution:
    def minimumPartition(self, s: str, k: int) -> int:
        partitions = 1
        current_value = 0

        for char in s:
            digit = int(char)

            if digit > k:
                return -1

            new_value = current_value * 10 + digit

            if new_value <= k:
                current_value = new_value
            else:
                partitions += 1
                current_value = digit

        return partitions
```

The implementation directly follows the greedy algorithm.

The variable `partitions` tracks how many substrings we have created so far. We begin at `1` because the first substring starts immediately.

The variable `current_value` stores the numeric value of the substring currently being built.

For every digit:

- We first check whether the digit itself exceeds `k`
- We attempt to append the digit to the current substring
- If the resulting number remains valid, we continue growing the substring
- Otherwise, we create a new partition and restart the current substring with this digit

Because each digit is processed exactly once, the solution runs in linear time.

## Go Solution

```go
func minimumPartition(s string, k int) int {
    partitions := 1
    currentValue := 0

    for _, ch := range s {
        digit := int(ch - '0')

        if digit > k {
            return -1
        }

        newValue := currentValue*10 + digit

        if newValue <= k {
            currentValue = newValue
        } else {
            partitions++
            currentValue = digit
        }
    }

    return partitions
}
```

The Go implementation mirrors the Python version very closely.

One small difference is character handling. In Go, iterating over a string with `range` yields runes, so we convert the rune digit into an integer using:

```go
digit := int(ch - '0')
```

Go integer arithmetic is safe here because the maximum possible value is bounded by `k <= 10^9`, which easily fits inside a standard `int`.

## Edge Cases

One important edge case occurs when a single digit is larger than `k`. Since every substring must contain at least one digit, a digit larger than `k` makes the entire problem impossible immediately. The implementation handles this by checking every digit individually before attempting to extend the current substring.

Another subtle edge case occurs when the entire string value is already less than or equal to `k`. A buggy implementation might still create unnecessary partitions. The greedy approach correctly keeps extending the current substring until the end of the string, producing exactly one partition.

A third important case involves repeated small digits where multiple grouping choices exist. For example, `"11111"` with `k = 11` could tempt an incorrect algorithm into suboptimal splits. The greedy strategy guarantees optimality because it always maximizes the current substring before splitting, ensuring the minimum number of partitions overall.

A final edge case involves values very close to the boundary. For example, when the current substring value is `99` and `k = 100`, appending another digit may barely exceed the limit. The implementation carefully checks the candidate value before committing to the extension, preventing invalid substrings from being formed.
