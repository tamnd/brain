## Solution

## Problem Understanding

The problem asks us to count how many digits in a given integer `num` evenly divide the number itself. In other words, for each digit `d` in `num`, we check if `num % d == 0`. The input is a single positive integer `num` with a value between 1 and 10^9. The output is a single integer representing the count of digits that are divisors of `num`.

Key constraints tell us that `num` does **not** contain the digit `0`, which simplifies the problem because division by zero does not need to be handled. Edge cases to keep in mind include single-digit numbers and numbers where all digits are the same. For instance, if `num = 1`, the answer is 1, and if `num = 111`, the answer is 3 since each 1 divides 111.

## Approaches

A brute-force approach would convert `num` to a string or list of digits, iterate over each digit, convert it back to an integer, and check if it divides `num`. This approach works correctly and efficiently given the constraints because `num` has at most 9 digits.

There is no need for a more complex approach, as iterating through the digits of a number is already optimal for the given input size.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force / Optimal | O(log10(num)) | O(log10(num)) | Convert number to digits and check divisibility for each digit |

## Algorithm Walkthrough

1. Convert the number `num` to a string to extract individual digits.
2. Initialize a counter to 0 to track how many digits divide `num`.
3. Iterate over each character in the string representation of `num`.
4. Convert the character back to an integer `digit`.
5. Check if `num % digit == 0`. If true, increment the counter.
6. After iterating over all digits, return the counter.

**Why it works:** Every digit of the number is checked exactly once for divisibility, and since the problem guarantees that no digit is zero, there are no division errors. Therefore, this procedure correctly counts all digits that divide the number.

## Python Solution

```python
class Solution:
    def countDigits(self, num: int) -> int:
        count = 0
        for ch in str(num):
            digit = int(ch)
            if num % digit == 0:
                count += 1
        return count
```

This implementation converts the integer to a string to easily iterate through its digits. Each digit is converted back to an integer for the modulo operation, and a counter tracks how many digits divide the number. Finally, the counter is returned.

## Go Solution

```go
func countDigits(num int) int {
    count := 0
    original := num
    for num > 0 {
        digit := num % 10
        if original % digit == 0 {
            count++
        }
        num /= 10
    }
    return count
}
```

In Go, the solution iterates through the digits by repeatedly taking the remainder when dividing by 10. Each digit is checked for divisibility against the original number. The loop continues until all digits are processed. No strings are needed, making it slightly more memory efficient than the Python version.

## Complexity Analysis

| Measure | Complexity | Explanation |

## Edge Cases

One important edge case is when `num` is a single-digit number. The algorithm correctly returns 1 because a number divides itself. Another edge case occurs when all digits are the same, such as 111 or 2222. Here, the algorithm counts each digit separately, correctly accounting for repetitions. A third edge case is a number where only some digits divide it, such as 13579. The algorithm handles this naturally because it evaluates each digit individually. In all these cases, the problem constraints (no zero digits) ensure there are no divisions by zero, making the implementation robust.
