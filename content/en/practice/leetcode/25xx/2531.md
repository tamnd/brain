# LeetCode 2531 - Make Number of Distinct Characters Equal

## Problem Understanding

The problem gives us two lowercase English strings, `word1` and `word2`. We are allowed to perform exactly one swap operation. In this operation, we choose one character from `word1` and one character from `word2`, then exchange them.

After performing this single swap, we must determine whether the number of distinct characters in both strings becomes equal.

A distinct character is a character that appears at least once in the string. For example, the string `"abcc"` has three distinct characters: `'a'`, `'b'`, and `'c'`.

The key detail is that we must perform exactly one move. Even if the two strings already have the same number of distinct characters, we still need to execute one swap and verify that the equality still holds afterward.

The constraints are important:

- Each string length can be as large as `10^5`
- Only lowercase English letters are used

Although the strings can be very large, the alphabet size is fixed at only 26 characters. This is the crucial observation that makes an efficient solution possible. Instead of thinking in terms of string length, we can focus on character frequencies.

Several edge cases are important:

- Swapping identical characters changes nothing
- Removing the last occurrence of a character decreases distinct count
- Introducing a previously absent character increases distinct count
- One string may initially contain only one distinct character
- The strings may already have equal distinct counts before swapping

A naive implementation that rebuilds strings after every swap would be far too slow for the input size.

## Approaches

### Brute Force Approach

The brute-force solution tries every possible swap.

For every index `i` in `word1` and every index `j` in `word2`, we:

1. Swap `word1[i]` and `word2[j]`
2. Recompute the number of distinct characters in both strings
3. Check whether the counts are equal
4. Restore the original strings

This approach is correct because it explicitly tests every valid move.

However, it is too slow. If both strings have length `10^5`, then there are up to:

$$10^5 \times 10^5 = 10^{10}$$

possible swaps.

Even computing distinct counts efficiently would not save this approach.

### Optimal Approach

The key observation is that only lowercase English letters exist, meaning there are at most 26 unique characters in each string.

Instead of considering swaps between indices, we only need to consider swaps between character types.

For example, if `'a'` exists in `word1` and `'c'` exists in `word2`, then swapping any occurrence of those characters produces the same effect on distinct counts.

This reduces the problem dramatically:

- At most 26 candidate characters from `word1`
- At most 26 candidate characters from `word2`

So we only need to test:

$$26 \times 26 = 676$$

possible character swaps.

For each candidate swap, we simulate how the distinct counts change without actually rebuilding strings.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × m) | O(1) | Tries every index pair swap |
| Optimal | O(26 × 26) | O(26) | Simulates swaps using frequency counts |

## Algorithm Walkthrough

1. Create frequency arrays of size 26 for both strings.

Each index corresponds to a lowercase English letter. For example, index `0` stores the frequency of `'a'`, index `1` stores the frequency of `'b'`, and so on.
2. Count the number of distinct characters in each string.

A character contributes to the distinct count if its frequency is greater than zero.
3. Iterate through all possible character pairs.

For every character `c1` from `word1` and every character `c2` from `word2`, determine whether both characters actually exist in their respective strings. If not, skip the pair.
4. Simulate the swap.

We do not actually modify the strings. Instead, we calculate how the distinct counts would change.

There are several cases:

- If `c1 == c2`, the swap changes nothing
- If removing `c1` from `word1` eliminates its last occurrence, distinct count decreases
- If adding `c2` to `word1` introduces a new character, distinct count increases
- The same logic applies symmetrically to `word2`
5. Compare the resulting distinct counts.

If the updated distinct counts are equal, return `True`.
6. If no swap works, return `False`.

### Why it works

The algorithm works because the only thing that matters is how frequencies change after a swap. Swapping two occurrences of the same character type always produces the same distinct-character effect regardless of position.

Since the alphabet is limited to 26 letters, checking all character-type swaps guarantees that every meaningful swap is examined.

## Python Solution

```python
class Solution:
    def isItPossible(self, word1: str, word2: str) -> bool:
        freq1 = [0] * 26
        freq2 = [0] * 26

        for ch in word1:
            freq1[ord(ch) - ord('a')] += 1

        for ch in word2:
            freq2[ord(ch) - ord('a')] += 1

        distinct1 = sum(1 for count in freq1 if count > 0)
        distinct2 = sum(1 for count in freq2 if count > 0)

        for i in range(26):
            if freq1[i] == 0:
                continue

            for j in range(26):
                if freq2[j] == 0:
                    continue

                new_distinct1 = distinct1
                new_distinct2 = distinct2

                if i != j:
                    if freq1[i] == 1:
                        new_distinct1 -= 1

                    if freq1[j] == 0:
                        new_distinct1 += 1

                    if freq2[j] == 1:
                        new_distinct2 -= 1

                    if freq2[i] == 0:
                        new_distinct2 += 1

                if new_distinct1 == new_distinct2:
                    return True

        return False
```

The implementation begins by building frequency arrays for both strings. Since the alphabet contains only lowercase English letters, arrays of length 26 are sufficient and more efficient than hash maps.

Next, the code computes the current number of distinct characters in each string. This is done by counting how many frequencies are greater than zero.

The nested loops iterate through all possible character swaps. Instead of swapping actual characters, the algorithm updates temporary distinct counts based on how frequencies would change.

When the swapped characters are different, several transitions may occur:

- A character may disappear entirely from one string
- A new character may appear in one string

These transitions determine whether the distinct count increases or decreases.

As soon as a valid swap produces equal distinct counts, the function returns `True`. If every possibility is exhausted, the function returns `False`.

## Go Solution

```go
func isItPossible(word1 string, word2 string) bool {
	freq1 := make([]int, 26)
	freq2 := make([]int, 26)

	for _, ch := range word1 {
		freq1[ch-'a']++
	}

	for _, ch := range word2 {
		freq2[ch-'a']++
	}

	distinct1 := 0
	distinct2 := 0

	for i := 0; i < 26; i++ {
		if freq1[i] > 0 {
			distinct1++
		}

		if freq2[i] > 0 {
			distinct2++
		}
	}

	for i := 0; i < 26; i++ {
		if freq1[i] == 0 {
			continue
		}

		for j := 0; j < 26; j++ {
			if freq2[j] == 0 {
				continue
			}

			newDistinct1 := distinct1
			newDistinct2 := distinct2

			if i != j {
				if freq1[i] == 1 {
					newDistinct1--
				}

				if freq1[j] == 0 {
					newDistinct1++
				}

				if freq2[j] == 1 {
					newDistinct2--
				}

				if freq2[i] == 0 {
					newDistinct2++
				}
			}

			if newDistinct1 == newDistinct2 {
				return true
			}
		}
	}

	return false
}
```

The Go implementation closely mirrors the Python version. Arrays of length 26 are used instead of maps because the alphabet size is fixed.

Go strings are iterated using `range`, which yields Unicode runes. Since the problem guarantees lowercase English letters, subtracting `'a'` safely converts characters into array indices.

No overflow concerns exist because frequencies never exceed `10^5`, which easily fits within Go's integer range.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(26 × 26) | Every character pair is checked once |
| Space | O(26) | Two frequency arrays of fixed size |

The complexity is effectively constant because the alphabet size never changes. Even though the input strings may contain up to `10^5` characters, the expensive part of the algorithm depends only on the 26 lowercase English letters.

Building the frequency arrays takes linear time with respect to input size, so the complete complexity is:

$$O(n + m + 26^2)$$

which simplifies to:

$$O(n + m)$$

## Edge Cases

One important edge case occurs when the swapped characters are identical. For example, swapping `'a'` from one string with `'a'` from the other changes absolutely nothing. A buggy implementation might accidentally modify distinct counts anyway. The solution handles this safely by skipping all distinct-count adjustments when `i == j`.

Another important case happens when removing the last occurrence of a character. Suppose `word1` contains exactly one `'c'`, and that `'c'` is swapped away. The distinct count must decrease because `'c'` disappears completely from the string. The implementation correctly checks `freq1[i] == 1` before decrementing the distinct count.

A third subtle case occurs when introducing a brand-new character into a string. If `word1` previously contained no `'x'`, and a swap adds `'x'`, then the distinct count increases by one. The implementation detects this using `freq1[j] == 0`.

A final edge case involves strings that already have equal distinct counts. Since the problem requires exactly one move, we cannot simply return `True` immediately. The algorithm still evaluates all possible swaps to verify whether at least one valid move preserves equality.
