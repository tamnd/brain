## Solution

## Problem Understanding

The problem gives us a string `s` containing only the characters `'a'`, `'b'`, and `'c'`, along with a non-negative integer `k`.

At each minute, we are allowed to remove exactly one character, either from the left end of the string or from the right end of the string. The removed characters are considered "taken".

Our goal is to take at least `k` occurrences of each of the three characters. In other words, after removing characters from both ends, the removed set must contain:

- at least `k` `'a'` characters
- at least `k` `'b'` characters
- at least `k` `'c'` characters

We need to compute the minimum number of minutes required to achieve this. Since one character is removed per minute, the answer is simply the minimum number of removed characters.

If it is impossible to collect `k` of every character because the original string does not contain enough of some character, we return `-1`.

The constraints are important:

- `s.length` can be as large as `10^5`
- only three distinct characters exist
- `k` may be `0`

Because the input size is large, any algorithm worse than roughly linear time will likely time out. A quadratic approach would be too slow for the upper bound.

Several edge cases are especially important:

- If `k == 0`, we already satisfy the requirement without removing anything, so the answer is `0`.
- If the string does not contain at least `k` copies of `'a'`, `'b'`, or `'c'`, the answer is immediately `-1`.
- The optimal strategy may involve taking characters from both ends, not just one side.
- Sometimes the entire string must be removed.

A key observation is that instead of thinking about what we remove, it is often easier to think about what we keep.

## Approaches

### Brute Force Approach

A straightforward brute-force solution would try every possible combination of characters taken from the left and right.

Suppose we take `i` characters from the left and `j` characters from the right. We can check whether the removed characters contain at least `k` of each character. Among all valid combinations, we keep the minimum value of `i + j`.

There are `O(n)` possibilities for the left side and `O(n)` possibilities for the right side, resulting in `O(n^2)` total combinations. For each combination, counting characters may require additional work unless optimized with prefix sums.

Even with preprocessing, the total complexity remains quadratic, which is too slow for `n = 10^5`.

### Optimal Sliding Window Approach

The key insight is to reverse the perspective.

Instead of directly deciding which characters to remove, we determine which contiguous substring to keep in the middle.

Since we may only remove characters from the left and right ends, the remaining characters must form one contiguous middle substring.

If we keep a substring, then everything outside that substring is removed. The removed characters must contain at least `k` of each character.

Suppose the total count of a character is:

- `totalA`
- `totalB`
- `totalC`

If we must remove at least `k` of each character, then the kept substring may contain at most:

- `totalA - k` `'a'`
- `totalB - k` `'b'`
- `totalC - k` `'c'`

Now the problem becomes:

Find the longest contiguous substring whose character counts do not exceed these limits.

Why longest? Because:

```
removed characters = total length - kept substring length
```

To minimize removed characters, we maximize the kept substring.

This becomes a classic sliding window problem.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) or O(n) | Try every left/right removal combination |
| Optimal | O(n) | O(1) | Sliding window to maximize valid middle substring |

## Algorithm Walkthrough

1. Count the total number of `'a'`, `'b'`, and `'c'` in the string.

We first determine whether a solution is even possible. If any character appears fewer than `k` times, we immediately return `-1`.
2. Handle the special case where `k == 0`.

If we do not need any characters, no removals are necessary, so return `0`.
3. Compute the maximum allowed counts inside the kept substring.

Since we must remove at least `k` of each character, the middle substring may contain at most:

```
total_count[char] - k
```

for each character.
4. Use a sliding window to find the longest valid substring.

We maintain two pointers:

- `left`
- `right`

The window represents the substring we keep.

We also maintain counts of characters currently inside the window.
5. Expand the window by moving `right`.

For each new character, add it to the window count.
6. Shrink the window if it becomes invalid.

If any character count inside the window exceeds its allowed limit, move `left` forward until the window becomes valid again.
7. Track the maximum valid window length.

Every valid window represents a possible middle substring that can remain after removals.
8. Compute the answer.

If the longest valid kept substring has length `max_window`, then:

```
answer = len(s) - max_window
```

### Why it works

The algorithm works because every valid sequence of removals from the ends corresponds to exactly one contiguous middle substring that remains.

The removed characters must contain at least `k` of each character. Equivalently, the kept substring may contain at most `total_count - k` of each character.

The sliding window efficiently finds the longest substring satisfying these limits. Since minimizing removals is equivalent to maximizing the kept substring, the algorithm produces the optimal answer.

## Python Solution

```python
class Solution:
    def takeCharacters(self, s: str, k: int) -> int:
        if k == 0:
            return 0

        total = [0, 0, 0]

        for ch in s:
            total[ord(ch) - ord('a')] += 1

        # Impossible case
        for count in total:
            if count < k:
                return -1

        # Maximum allowed counts inside the kept window
        limit = [count - k for count in total]

        window = [0, 0, 0]
        left = 0
        max_window = 0

        for right in range(len(s)):
            idx = ord(s[right]) - ord('a')
            window[idx] += 1

            # Shrink while invalid
            while window[idx] > limit[idx]:
                left_idx = ord(s[left]) - ord('a')
                window[left_idx] -= 1
                left += 1

            max_window = max(max_window, right - left + 1)

        return len(s) - max_window
```

The implementation starts by handling the special case where `k` is zero. In that situation, no removals are necessary.

Next, the code counts the total occurrences of each character in the string. This allows us to quickly detect impossible cases where some character appears fewer than `k` times.

The `limit` array stores the maximum number of each character allowed inside the kept substring. For example, if the string contains five `'a'` characters and we must remove at least two of them, then the middle substring may contain at most three `'a'` characters.

The sliding window is implemented using two pointers. The `right` pointer expands the window, while the `left` pointer shrinks it whenever the window becomes invalid.

A window becomes invalid if the count of some character exceeds its allowed limit. Since only the newly added character can violate the condition, checking `window[idx] > limit[idx]` is sufficient.

Whenever the window is valid, we update `max_window` with the largest valid substring length seen so far.

Finally, the minimum number of removals equals the total string length minus the length of the longest valid kept substring.

## Go Solution

```go
func takeCharacters(s string, k int) int {
    if k == 0 {
        return 0
    }

    total := [3]int{}

    for _, ch := range s {
        total[ch-'a']++
    }

    // Impossible case
    for _, count := range total {
        if count < k {
            return -1
        }
    }

    limit := [3]int{}
    for i := 0; i < 3; i++ {
        limit[i] = total[i] - k
    }

    window := [3]int{}
    left := 0
    maxWindow := 0

    for right := 0; right < len(s); right++ {
        idx := s[right] - 'a'
        window[idx]++

        // Shrink while invalid
        for window[idx] > limit[idx] {
            leftIdx := s[left] - 'a'
            window[leftIdx]--
            left++
        }

        currentLength := right - left + 1
        if currentLength > maxWindow {
            maxWindow = currentLength
        }
    }

    return len(s) - maxWindow
}
```

The Go implementation follows the exact same logic as the Python version.

Fixed-size arrays of length three are used instead of maps because the character set is limited to `'a'`, `'b'`, and `'c'`. This reduces overhead and keeps the implementation efficient.

Go strings are byte-indexable, so character access uses `s[right] - 'a'` directly.

No special integer overflow handling is required because the maximum string length is only `10^5`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each pointer moves at most `n` times |
| Space | O(1) | Only fixed-size counting arrays are used |

The sliding window processes each character at most twice:

- once when expanding the right pointer
- once when shrinking the left pointer

Therefore, the total runtime is linear.

The space usage remains constant because only a few arrays of size three are maintained regardless of input size.

## Edge Cases

One important edge case occurs when `k == 0`. A careless implementation might still attempt to run the sliding window logic unnecessarily. In reality, the answer is immediately `0` because no characters need to be collected. The implementation handles this with an early return.

Another critical edge case happens when some character appears fewer than `k` times in the entire string. Since we can only remove characters that already exist, it becomes impossible to collect enough copies of that character. The implementation detects this before running the sliding window and returns `-1`.

A third tricky case occurs when the optimal solution requires removing the entire string. For example, with `"abcabc"` and `k = 2`, we need all occurrences of every character, so nothing may remain in the middle. The algorithm handles this naturally because the maximum valid window length becomes `0`, producing the correct answer equal to the full string length.

Another subtle scenario appears when the valid middle substring can contain only one type of character or must avoid certain characters completely. The sliding window correctly shrinks whenever any character count exceeds its allowed limit, ensuring the window always satisfies the required constraints.
