## Solution

## Problem Understanding

The problem asks us to count how many subsequences of a given string are "good". A subsequence is formed by deleting zero or more characters from the original string while preserving the relative order of the remaining characters.

A subsequence is considered good if it satisfies two conditions:

1. It is not empty.
2. Every character that appears in the subsequence appears the same number of times.

For example:

- `"ab"` is good because both `'a'` and `'b'` appear once.
- `"aabb"` is good because both characters appear twice.
- `"aab"` is not good because `'a'` appears twice while `'b'` appears once.

The input is a string `s` consisting only of lowercase English letters. The length can be as large as `10^4`, which immediately tells us that generating all subsequences explicitly is impossible. Since a string of length `n` has `2^n` subsequences, even `n = 50` would already be too large for brute force.

The output should be the number of good subsequences modulo `10^9 + 7`.

An important detail is that subsequences are counted based on positions, not just resulting strings. If the same subsequence string can be formed in multiple ways, each formation counts separately. For example, in `"aabb"`, there are multiple ways to form `"ab"`.

Several edge cases are important:

- A string with all distinct characters, such as `"abcd"`, makes every non-empty subsequence good.
- A string with all identical characters, such as `"aaaa"`, only allows subsequences consisting solely of `'a'`, but every non-empty one is still good.
- Strings with uneven frequencies require careful combinatorial counting.
- The answer can become extremely large, so modular arithmetic is necessary throughout the computation.

## Approaches

### Brute Force Approach

The most direct approach is to generate every possible subsequence and test whether it is good.

For each subsequence:

1. Count the frequency of every character.
2. Ignore empty subsequences.
3. Check whether all non-zero frequencies are equal.
4. If yes, increment the answer.

This approach is correct because it examines every subsequence exactly once and directly verifies the definition of a good subsequence.

However, it is far too slow. A string of length `n` has `2^n` subsequences. With `n = 10^4`, exhaustive generation is completely infeasible.

### Key Insight for the Optimal Approach

A good subsequence has a very special structure:

- Every chosen character appears exactly `k` times for some positive integer `k`.

Instead of thinking about subsequences individually, we can think combinatorially.

Suppose:

- `'a'` appears `fa` times,
- `'b'` appears `fb` times,
- and so on.

For a fixed frequency `k`:

- A character can either:

- be excluded entirely, or
- contribute exactly `k` copies.

If a character appears `f` times in the original string and `f >= k`, then the number of ways to choose exactly `k` occurrences is:

$$\binom{f}{k}$$

For each character independently, we therefore have:

$$1 + \binom{f}{k}$$

The `1` means we skip the character entirely.

Multiplying these choices across all characters gives:

$$\prod (1 + \binom{f}{k})$$

This includes the case where every character is skipped, so we subtract `1`.

Thus, for every possible frequency `k`, the number of good subsequences is:

$$\left(\prod (1 + \binom{f}{k})\right) - 1$$

Summing over all valid `k` gives the final answer.

This transforms an exponential problem into a polynomial-time combinatorial counting problem.

| Optimal | O(26 * n) | O(n) | Uses combinatorics and frequency counting |

## Algorithm Walkthrough

1. Count the frequency of every character in the string.

Since the string contains only lowercase English letters, we only need an array of size 26. For example, for `"aabb"`:

- `'a' -> 2`
- `'b' -> 2`
2. Precompute factorials and inverse factorials.

We need to compute combinations efficiently:

$$\binom{n}{r}$$

Since the modulus is prime, we use Fermat's Little Theorem to compute modular inverses.
3. Iterate over every possible target frequency `k`.

A good subsequence requires every chosen character to appear exactly `k` times. The maximum possible `k` is the maximum character frequency in the string.
4. For each character frequency `f`, determine whether the character can participate.

If `f < k`, then this character cannot appear in a good subsequence with frequency `k`.

If `f >= k`, then we have:

$$\binom{f}{k}$$

ways to choose exactly `k` occurrences.
5. Build the total number of subsequences for this `k`.

For each character:

- choose nothing, contributing `1`
- or choose exactly `k` occurrences

Therefore:

$$current = current \times (1 + \binom{f}{k})$$
6. Subtract one to remove the empty subsequence.

The multiplication includes the possibility that every character is skipped.
7. Add the result for this `k` to the global answer.
8. Return the answer modulo `10^9 + 7`.

### Why it works

For a fixed frequency `k`, every good subsequence must choose either exactly `k` copies of a character or none at all. There is no other valid possibility because all included characters must share the same frequency.

The choices for different characters are independent, so multiplication correctly counts all combinations. Subtracting one removes the invalid empty subsequence. Summing over all possible `k` counts every good subsequence exactly once because each good subsequence has exactly one common frequency value.

## Python Solution

```python
from collections import Counter
from typing import List

MOD = 10**9 + 7

class Solution:
    def countGoodSubsequences(self, s: str) -> int:
        frequencies = Counter(s)
        max_frequency = max(frequencies.values())

        factorial = [1] * (max_frequency + 1)
        inverse_factorial = [1] * (max_frequency + 1)

        for i in range(1, max_frequency + 1):
            factorial[i] = factorial[i - 1] * i % MOD

        inverse_factorial[max_frequency] = pow(
            factorial[max_frequency],
            MOD - 2,
            MOD
        )

        for i in range(max_frequency, 0, -1):
            inverse_factorial[i - 1] = (
                inverse_factorial[i] * i
            ) % MOD

        def combination(n: int, r: int) -> int:
            if r < 0 or r > n:
                return 0

            return (
                factorial[n]
                * inverse_factorial[r]
                % MOD
                * inverse_factorial[n - r]
                % MOD
            )

        answer = 0

        for k in range(1, max_frequency + 1):
            ways = 1

            for frequency in frequencies.values():
                if frequency >= k:
                    ways = (
                        ways
                        * (1 + combination(frequency, k))
                    ) % MOD

            ways = (ways - 1) % MOD
            answer = (answer + ways) % MOD

        return answer
```

The implementation begins by counting character frequencies using `Counter`. This gives us the number of available occurrences for each letter.

Next, factorials and inverse factorials are precomputed. This allows combinations to be evaluated in constant time using:

$$\binom{n}{r} = \frac{n!}{r!(n-r)!}$$

under modular arithmetic.

The outer loop iterates through every possible common frequency `k`. For each `k`, the algorithm computes how many valid subsequences can be formed.

Inside the loop, every character contributes either:

- one option to skip it,
- or `C(frequency, k)` ways to include it.

These independent choices are multiplied together.

Finally, subtracting one removes the empty subsequence, and the result is accumulated into the global answer.

## Go Solution

```go
package main

const MOD int = 1_000_000_007

func modPow(base, exp int) int {
	result := 1

	for exp > 0 {
		if exp&1 == 1 {
			result = result * base % MOD
		}

		base = base * base % MOD
		exp >>= 1
	}

	return result
}

func countGoodSubsequences(s string) int {
	frequencies := make([]int, 26)

	for _, ch := range s {
		frequencies[ch-'a']++
	}

	maxFrequency := 0

	for _, frequency := range frequencies {
		if frequency > maxFrequency {
			maxFrequency = frequency
		}
	}

	factorial := make([]int, maxFrequency+1)
	inverseFactorial := make([]int, maxFrequency+1)

	factorial[0] = 1

	for i := 1; i <= maxFrequency; i++ {
		factorial[i] = factorial[i-1] * i % MOD
	}

	inverseFactorial[maxFrequency] =
		modPow(factorial[maxFrequency], MOD-2)

	for i := maxFrequency; i >= 1; i-- {
		inverseFactorial[i-1] =
			inverseFactorial[i] * i % MOD
	}

	combination := func(n, r int) int {
		if r < 0 || r > n {
			return 0
		}

		return factorial[n] *
			inverseFactorial[r] % MOD *
			inverseFactorial[n-r] % MOD
	}

	answer := 0

	for k := 1; k <= maxFrequency; k++ {
		ways := 1

		for _, frequency := range frequencies {
			if frequency >= k {
				ways = ways *
					(1 + combination(frequency, k)) % MOD
			}
		}

		ways = (ways - 1 + MOD) % MOD
		answer = (answer + ways) % MOD
	}

	return answer
}
```

The Go implementation follows the same mathematical strategy as the Python version.

A fixed-size integer slice of length 26 stores character frequencies efficiently. Since Go does not provide built-in modular exponentiation for this use case, a custom `modPow` function implements fast binary exponentiation.

One important Go-specific detail is handling negative modulo values safely. After subtracting one from `ways`, we add `MOD` before taking `% MOD` again:

```go
ways = (ways - 1 + MOD) % MOD
```

This prevents negative intermediate values.

### For k = 1

For `'a'`:

$$1 + \binom{2}{1} = 1 + 2 = 3$$

For `'b'`:

$$1 + \binom{2}{1} = 3$$

Total:

$$3 \times 3 = 9$$

Remove empty subsequence:

$$9 - 1 = 8$$

These correspond to:

```
a, a, b, b, ab, ab, ab, ab
```

### For k = 2

For `'a'`:

$$1 + \binom{2}{2} = 2$$

For `'b'`:

$$1 + \binom{2}{2} = 2$$

Total:

$$2 \times 2 = 4$$

Remove empty subsequence:

$$4 - 1 = 3$$

These correspond to:

```
aa, bb, aabb
```

Final answer:

$$8 + 3 = 11$$

$$16 - 1 = 15$$

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(26 * n) | Precomputation plus iteration across frequencies |
| Space | O(n) | Factorial and inverse factorial arrays |

The factorial preprocessing requires linear time up to the maximum character frequency, which is at most `n`. The main counting loop iterates over every possible frequency value and every lowercase letter. Since there are only 26 letters, this portion is effectively linear in practice.

The space complexity comes primarily from the factorial and inverse factorial arrays.

## Edge Cases

One important edge case is a string containing only one unique character, such as `"aaaa"`. Every non-empty subsequence is automatically good because all included characters have the same frequency by definition. A buggy implementation might overcomplicate the counting logic or accidentally exclude valid subsequences. This implementation handles the case naturally because the product formula becomes:

$$1 + \binom{4}{k}$$

for each valid `k`.

Another important case is when all characters are distinct, such as `"abcd"`. Here, the only possible common frequency is `1`. Every non-empty subsequence is therefore good. The algorithm correctly computes:

$$(1+1)^4 - 1$$

which equals `15`.

A more subtle edge case occurs when frequencies differ significantly, such as `"aaab"`. The character `'b'` cannot participate when `k = 2` or `k = 3`. A naive implementation might incorrectly include invalid combinations. This solution explicitly checks:

```
if frequency >= k:
```

before allowing a character to contribute to the count.

Finally, modular arithmetic is critical because the number of subsequences grows exponentially. Without modular reductions during multiplication, integer overflow would occur in many languages. Both implementations apply modulo operations at every multiplication step to keep values bounded safely.
