## Solution

## Problem Understanding

The problem presents a large, theoretically infinite grid where you start at the coordinate `(1, 1)` and aim to reach a target point `(targetX, targetY)`. Movement on this grid is restricted to four operations: subtracting the other coordinate from the current one, or doubling one of the coordinates. Formally, from point `(x, y)`, the allowed moves are `(x, y - x)`, `(x - y, y)`, `(2 * x, y)`, and `(x, 2 * y)`.

The input consists of two integers, `targetX` and `targetY`, representing the coordinates of the destination. The output is a boolean indicating whether it is possible to reach `(targetX, targetY)` from `(1, 1)` using any finite sequence of moves.

The constraints tell us that both coordinates can be as large as $10^9$, so a brute-force search exploring all paths from `(1,1)` to the target is infeasible. Edge cases that could trip up naive implementations include small targets `(1,1)` (already at the target), cases where the target is a power-of-two multiple of 1 in either dimension, and targets where `targetX` and `targetY` are coprime or share specific multiplicative relationships.

The key insight is to consider the problem **in reverse**, working from the target backwards to `(1,1)`. By inverting the operations, we can repeatedly reduce `targetX` and `targetY` towards 1, which is computationally efficient even for large numbers.

## Approaches

A brute-force approach would attempt to explore all sequences of valid moves from `(1,1)` to `(targetX, targetY)` using recursion or BFS. While this method is correct in principle, it is extremely slow because the number of possible paths grows exponentially and `targetX` and `targetY` can reach $10^9$. Memory usage would also be high if using BFS with a visited set.

The optimal approach leverages **reverse reasoning**. Starting from `(targetX, targetY)`, we apply the inverse of the allowed moves whenever possible:

1. If `targetX > targetY` and `targetX` is even, it may have been doubled, so we divide by 2.
2. If `targetY > targetX` and `targetY` is even, it may have been doubled, so we divide by 2.
3. If `targetX > targetY`, it may have been reached by `(x - y, y)` → we subtract `targetY` from `targetX`.
4. If `targetY > targetX`, it may have been reached by `(x, y - x)` → we subtract `targetX` from `targetY`.

This reverse process repeats until we either reach `(1,1)` or a state where no further reduction is possible without producing non-positive numbers.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(4^N) | O(N) | Explore all sequences recursively; infeasible for large N |
| Optimal | O(log(max(targetX, targetY))) | O(1) | Reverse operations until reaching (1,1) or failing |

## Algorithm Walkthrough

1. Initialize `x = targetX` and `y = targetY`.
2. While both `x > 1` and `y > 1`:

1. If `x > y` and `x` is even, divide `x` by 2.
2. Else if `y > x` and `y` is even, divide `y` by 2.
3. Else if `x > y`, subtract `y` from `x`.
4. Else if `y > x`, subtract `x` from `y`.
5. Else, break, since `x == y` but not 1, so the target is unreachable.
3. If after the loop either `x == 1` or `y == 1`, check that the other coordinate can be reduced to 1 by repeatedly subtracting or halving. If possible, return `True`.
4. Otherwise, return `False`.

**Why it works**: The reverse operations preserve reachability because each forward move has a unique reverse operation or set of operations that reduce the coordinates. By progressively reducing the target using these operations, we are guaranteed to identify whether `(1,1)` can reach the target without exploring exponential paths.

## Python Solution

```python
class Solution:
    def isReachable(self, targetX: int, targetY: int) -> bool:
        x, y = targetX, targetY
        while x > 1 and y > 1:
            if x > y:
                if x % 2 == 0:
                    x //= 2
                else:
                    x -= y
            elif y > x:
                if y % 2 == 0:
                    y //= 2
                else:
                    y -= x
            else:
                return False  # x == y != 1, impossible to reduce further
        # After loop, one of x or y is 1
        if x == 1:
            return (y - 1) % x == 0 or y >= 1
        if y == 1:
            return (x - 1) % y == 0 or x >= 1
        return x == 1 and y == 1
```

The Python implementation follows the algorithm exactly. We reduce `x` or `y` iteratively, handling the halving and subtraction cases carefully. If we reach a point where `x == y` but not 1, we know it is impossible to reach `(1,1)`. Once either coordinate reaches 1, the other can always be reduced by halving or subtraction to reach 1 as well.

## Go Solution

```go
func isReachable(targetX int, targetY int) bool {
    x, y := targetX, targetY
    for x > 1 && y > 1 {
        if x > y {
            if x%2 == 0 {
                x /= 2
            } else {
                x -= y
            }
        } else if y > x {
            if y%2 == 0 {
                y /= 2
            } else {
                y -= x
            }
        } else {
            return false
        }
    }
    if x == 1 {
        return true
    }
    if y == 1 {
        return true
    }
    return x == 1 && y == 1
}
```

The Go version mirrors the Python logic, using integer division and modulo to handle halving and subtraction. Go does not need explicit type casting for these operations, and the loop and conditionals are straightforward.
