## Solution

## Problem Understanding

The problem asks us to split an array `nums` at some index `i` such that the product of the elements on the left side (`nums[0]` to `nums[i]`) and the product of the elements on the right side (`nums[i+1]` to `nums[n-1]`) are **coprime**. Two numbers are coprime if their greatest common divisor (gcd) is `1`. The task is to find the **smallest valid split index** or return `-1` if no such index exists.

The input is an integer array of length `n`, with elements ranging from `1` to `10^6`, and `n` itself can be as large as `10^4`. This implies that directly computing the products for every split is impractical because the numbers would quickly exceed typical integer limits. Therefore, a solution must rely on **factorization or number-theoretic properties** rather than literal product computation.

Important edge cases include arrays with repeated prime factors, arrays where all elements are 1, or arrays where all elements share a common prime factor. The problem guarantees positive integers, which avoids zero or negative product complications.

## Approaches

**Brute Force Approach:** The naive solution would be to try every possible split index `i`, compute the product of the left and right subarrays, and check if their gcd is 1. While correct, this approach is infeasible due to integer overflow and the combinatorial growth of the product size. Computing products for arrays of length up to `10^4` with numbers up to `10^6` will exceed standard integer sizes and is inefficient.

**Optimal Approach:** The key observation is that instead of computing the products directly, we can track **prime factors**. Two products are coprime if and only if they share no common prime factors. By maintaining a map from each prime factor to the last index where it appears in the array, we can determine the earliest index where all prime factors to the left do not appear to the right. This avoids actual product calculation, uses factorization for numbers up to `10^6`, and allows a linear pass to find the valid split index.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2 * log(maxNum)) | O(1) | Compute products for all splits and check gcd |
| Optimal | O(n * sqrt(maxNum)) | O(n * log(maxNum)) | Track prime factors and last occurrence to check coprimality |

## Algorithm Walkthrough

1. Initialize a dictionary `last_index` that maps each prime factor to the last index in the array where it occurs.
2. Iterate through each element in the array, factorizing it into its prime components.
3. For each prime factor of an element at index `i`, update `last_index[p] = i`.
4. Initialize two variables: `current_max = 0` to track the farthest index any prime factor has appeared so far, and `i = 0` for the loop.
5. Iterate through the array from left to right:

- For each index `i`, update `current_max` to be the maximum of itself and the last index of each prime factor of `nums[i]`.
- If `current_max == i`, all prime factors of the left subarray do not appear in the right subarray, making the split valid.
6. If no split is found after completing the loop, return `-1`.

**Why it works:** The invariant maintained is that `current_max` represents the farthest index to which the current prime factors extend. If `current_max == i`, no prime factor from the left extends into the right, guaranteeing coprimality.

## Python Solution

```python
from typing import List
from collections import defaultdict
import math

class Solution:
    def findValidSplit(self, nums: List[int]) -> int:
        def prime_factors(x: int) -> set:
            factors = set()
            d = 2
            while d * d <= x:
                while x % d == 0:
                    factors.add(d)
                    x //= d
                d += 1
            if x > 1:
                factors.add(x)
            return factors

        last_index = {}
        for i, num in enumerate(nums):
            for p in prime_factors(num):
                last_index[p] = i

        current_max = 0
        for i, num in enumerate(nums[:-1]):
            for p in prime_factors(num):
                current_max = max(current_max, last_index[p])
            if current_max == i:
                return i
        return -1
```

**Explanation:** The `prime_factors` function extracts all unique prime factors of a number. We first record the last index each prime appears at. Then, iterating left to right, we track the farthest index (`current_max`) any left prime extends to. If at any point `current_max == i`, all primes in the left partition are isolated from the right, confirming coprimality.

## Go Solution

```go
package main

func primeFactors(x int) map[int]struct{} {
    factors := make(map[int]struct{})
    for d := 2; d*d <= x; d++ {
        for x%d == 0 {
            factors[d] = struct{}{}
            x /= d
        }
    }
    if x > 1 {
        factors[x] = struct{}{}
    }
    return factors
}

func findValidSplit(nums []int) int {
    lastIndex := make(map[int]int)
    for i, num := range nums {
        for p := range primeFactors(num) {
            lastIndex[p] = i
        }
    }

    currentMax := 0
    for i := 0; i < len(nums)-1; i++ {
        for p := range primeFactors(nums[i]) {
            if lastIndex[p] > currentMax {
                currentMax = lastIndex[p]
            }
        }
        if currentMax == i {
            return i
        }
    }
    return -1
}
```

**Go Notes:** Go uses a map with empty structs to store prime factors efficiently. Unlike Python, we explicitly loop over the map keys to update `currentMax`. Otherwise, the logic mirrors the Python approach.
