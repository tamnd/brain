## Solution

## Problem Understanding

The problem asks us to maximize the number of indices we can "mark" in an array `nums` following a specific rule. Initially, all indices are unmarked. We can repeatedly select two different unmarked indices `i` and `j` such that `2 * nums[i] <= nums[j]` and then mark both indices. The goal is to find the maximum number of indices we can mark after performing this operation any number of times.

The input is a 0-indexed integer array `nums` of length up to `10^5` where each element is between `1` and `10^9`. The output is a single integer representing the maximum number of indices that can be marked according to the operation rule.

Constraints indicate that a brute-force approach that tries all pairs repeatedly will be too slow. The array can be large, so we need an efficient approach, ideally near `O(n log n)`.

Important edge cases include arrays where no operation is possible (all elements are too close in value), arrays where all operations are possible, and arrays with the smallest or largest allowed values.

## Approaches

A brute-force approach would repeatedly iterate through all pairs of unmarked indices and check if `2 * nums[i] <= nums[j]`. If valid, mark them and continue. While this guarantees correctness, it is far too slow for large arrays because it could require `O(n^2)` operations per pass and multiple passes through the array, giving a worst-case complexity around `O(n^3)`.

The key observation for an optimal approach is that sorting the array simplifies the pairing process. If the array is sorted, we can attempt to pair the smallest unmarked elements with the largest unmarked elements that satisfy the condition. This naturally leads to a two-pointer strategy: one pointer starting from the smallest element and another from the midpoint or the larger half of the array, moving forward greedily. Sorting guarantees we always pick the smallest valid `i` and the smallest valid `j` that can pair with it, maximizing the number of pairs.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n) | Try all pairs repeatedly, mark if condition satisfied |
| Optimal | O(n log n) | O(1) | Sort + two-pointer greedy approach to maximize marked indices |

## Algorithm Walkthrough

1. Sort the array `nums` in non-decreasing order. This ensures that for any `i < j`, `nums[i] <= nums[j]` and helps efficiently check the condition `2 * nums[i] <= nums[j]`.
2. Split the sorted array conceptually into two halves. The first half contains smaller elements which will serve as candidate `i` values, and the second half contains larger elements which will serve as candidate `j` values. This ensures `2 * nums[i] <= nums[j]` has a chance to hold.
3. Initialize two pointers, `left` starting at index `0` of the first half, and `right` starting at the start of the second half. Initialize a counter `count` for the number of marked indices.
4. Iterate while `left` is within the first half and `right` is within the second half. For each pair `(left, right)`:

- If `2 * nums[left] <= nums[right]`, increment `count` by `2` (since both indices are marked), then move both pointers forward.
- If the condition fails, move `right` forward to find a suitable `j` that can pair with the current `left`.
5. Once either pointer reaches the end of its respective half, stop. Return `count` as the maximum number of marked indices.

Why it works: Sorting ensures that each `i` is paired with the smallest possible valid `j`. This greedy pairing maximizes the number of operations since using a larger `j` unnecessarily might block future valid pairs. The two-pointer strategy guarantees that no valid pair is skipped, and every selected pair satisfies the problem condition.

## Python Solution

```python
from typing import List

class Solution:
    def maxNumOfMarkedIndices(self, nums: List[int]) -> int:
        nums.sort()
        n = len(nums)
        left = 0
        right = n // 2
        count = 0
        
        while left < n // 2 and right < n:
            if 2 * nums[left] <= nums[right]:
                count += 2
                left += 1
                right += 1
            else:
                right += 1
        
        return count
```

In this implementation, we first sort the array, then use two pointers to find valid pairs. The `left` pointer scans the smaller half and the `right` pointer scans the larger half, incrementing the counter whenever a valid pair is found. If the pair is invalid, only `right` moves forward to find a larger element that may satisfy the condition.

## Go Solution

```go
import "sort"

func maxNumOfMarkedIndices(nums []int) int {
    sort.Ints(nums)
    n := len(nums)
    left, right := 0, n/2
    count := 0

    for left < n/2 && right < n {
        if 2*nums[left] <= nums[right] {
            count += 2
            left++
            right++
        } else {
            right++
        }
    }

    return count
}
```

In Go, the approach mirrors Python. We sort the slice, use integer indices for pointers, and increment a counter whenever we find a valid pair. The main difference is the explicit use of `sort.Ints` and no need for type hints.

## Worked Examples

Example: `nums = [3,5,2,4]`

1. Sort: `[2,3,4,5]`
2. Pointers: `left=0 (2), right=2 (4)`
3. Check `2*2 <= 4`: valid, count=2, move both pointers -> `left=1, right=3`
4. Check `2*3 <= 5`: false, move right -> `right=4` (end)
5. Return `count = 2`

Example: `nums = [9,2,5,4]`

1. Sort: `[2,4,5,9]`
2. Pointers: `left=0 (2), right=2 (5)`
3. Check `2*2 <= 5`: valid, count=2, move both -> `left=1, right=3`
4. Check `2*4 <= 9`: valid, count=4, move both -> `left=2, right=4` (end)
5. Return `count = 4`

Example: `nums = [7,6,8]`

1. Sort: `[6,7,8]`
2. Pointers: `left=0 (6), right=1 (7)`
3. Check `2*6 <= 7`: false, move right -> `right=2`
4. Check `2*6 <= 8`: false, move right -> `right=3` (end)
5. Return `count = 0`

## Edge Cases

The first edge case is a single-element array. Since no pair can be formed, the algorithm correctly returns 0. The second edge case is two elements with the same value. Even though there are two elements, the doubling condition fails (`2*nums[i] <= nums[j]`), so the algorithm correctly returns 0. A third edge case is a very large array where all elements in the first half can pair with the second half. The algorithm handles this efficiently because sorting and two-pointer scanning scales linearly after sorting, avoiding memory or performance issues. All edge cases are naturally handled by the sorted two-pointer strategy.
