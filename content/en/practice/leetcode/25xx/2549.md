# LeetCode 2549 - Count Distinct Numbers on Board

## Problem Understanding

The problem gives us a starting integer `n` that is initially written on a board. Every day, we look at every number currently on the board. For each number `x`, we search for all integers `i` such that:

```
1 <= i <= n
x % i == 1
```

Whenever such an integer `i` exists, we add it to the board. Numbers never disappear, and duplicates do not matter because the question asks for the number of distinct integers present after a very large number of days, specifically `10^9` days.

The key detail is that the process stabilizes long before a billion days. Once no new numbers can be added, the board stops changing permanently.

The input is a single integer `n`, where:

```
1 <= n <= 100
```

This is a very small constraint, which means even simulation approaches are feasible. However, the problem is actually based on discovering a mathematical pattern rather than implementing a heavy simulation.

The expected output is the total number of distinct integers that will eventually appear on the board.

An important edge case is when `n = 1`. In that situation, no integer `i` satisfies:

```
1 % i == 1
```

because `1 % 1 = 0`. Therefore, no new numbers are ever added, and the answer remains `1`.

Another important observation is that all generated numbers are always less than or equal to `n`, because the problem explicitly restricts `i` to the range `[1, n]`.

## Approaches

### Brute Force Simulation

A direct approach is to simulate the process exactly as described.

We can maintain a set containing all numbers currently on the board. For every number already in the set, we test every integer `i` from `1` to `n`. If `x % i == 1`, we add `i` to the set.

We repeat this process until no new numbers are added.

This works because the set eventually stabilizes. Since there are only `n` possible integers that can ever appear, the process can only expand a finite number of times.

Although this solution is completely acceptable for `n <= 100`, it is more complicated than necessary because the problem has a very simple mathematical pattern.

### Mathematical Observation

The key insight is that for every integer `x > 1`, the number `x - 1` satisfies:

```
x % (x - 1) == 1
```

For example:

```
5 % 4 == 1
4 % 3 == 1
3 % 2 == 1
```

This means that starting from `n`, we can always generate `n - 1`, then `n - 2`, and so on, all the way down to `2`.

However, the number `1` can never be generated because:

```
x % 1 == 0
```

for every integer `x`.

Therefore:

- If `n == 1`, the board only contains `{1}`, so the answer is `1`
- Otherwise, every integer from `2` through `n` eventually appears, giving exactly `n - 1` distinct numbers

## Algorithm Walkthrough

### Optimal Algorithm

1. Check whether `n` equals `1`.

This is the only special case in the problem. When `n = 1`, no new numbers can ever be generated because `1 % 1 = 0`.
2. If `n == 1`, return `1`.

The board permanently contains only the number `1`.
3. Otherwise, return `n - 1`.

For every number `x > 1`, we can generate `x - 1` because:

```
x % (x - 1) == 1
```

Starting from `n`, this chain continues until we reach `2`.
4. Since `1` can never be generated, the final board contains:

```
{2, 3, 4, ..., n}
```

The size of this set is `n - 1`.

### Why it works

The process guarantees that every integer greater than `1` can generate the integer immediately below it. This creates a chain from `n` down to `2`. At the same time, the integer `1` is impossible to generate because modulo by `1` always produces `0`, never `1`. Therefore, the final set contains exactly all integers from `2` to `n` when `n > 1`, and only `{1}` when `n = 1`.

## Python Solution

```python
class Solution:
    def distinctIntegers(self, n: int) -> int:
        if n == 1:
            return 1
        
        return n - 1
```

The implementation directly follows the mathematical observation.

The first condition handles the only exceptional case in the problem. When `n` equals `1`, no additional integers can be added to the board.

For every other value of `n`, the answer is simply `n - 1` because all integers from `2` through `n` eventually appear.

The implementation does not require simulation, sets, loops, or recursion because the entire process collapses into a simple formula.

## Edge Cases

The most important edge case is `n = 1`. This case behaves differently from every other input because modulo by `1` always produces `0`. Since the condition requires the remainder to equal `1`, no new integers can ever be added. A careless implementation that always returns `n - 1` would incorrectly return `0` here.

Another important edge case is `n = 2`. This is the smallest input where the chain-generation logic applies. The board starts with `{2}`, and since `2 % 1 = 0`, the number `1` is still impossible to generate. The final answer is therefore `1`, representing the set `{2}`.

A third important edge case involves the maximum constraint, `n = 100`. Although the constraints are small, some brute-force implementations might repeatedly simulate unnecessary days or fail to stop once the board stabilizes. The mathematical solution avoids all such inefficiencies because it computes the answer directly in constant time.
