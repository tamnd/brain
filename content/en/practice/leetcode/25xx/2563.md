## Solution

## Problem Understanding

The problem is asking us to count the number of pairs of indices `(i, j)` in a 0-indexed array `nums` such that `i < j` and the sum of the elements at those indices lies within a given inclusive range `[lower, upper]`. In other words, for every possible pair of elements in the array where the first element precedes the second, we need to check if their sum satisfies the range constraint. The input consists of the array `nums` of length `n` and the integers `lower` and `upper`, which define the allowed sum range. The output is a single integer representing the total number of fair pairs.

The constraints provide significant insight into the solution approach. With `nums` length up to `10^5` and values ranging from `-10^9` to `10^9`, a naive brute-force solution that checks all pairs in `O(n^2)` time will be far too slow. This indicates the need for an optimized algorithm, likely involving sorting, two pointers, or binary search to efficiently count qualifying pairs.

Important edge cases include arrays with minimum size `1` where no pairs exist, arrays with identical elements, or ranges where no sum can satisfy the constraint. Negative numbers and large ranges could also challenge a naive solution, particularly in handling integer overflow or array indices correctly.

## Approaches

The brute-force approach iterates over all possible pairs `(i, j)` in nested loops. For each pair, we calculate the sum and check if it falls within `[lower, upper]`. While this approach is simple and correct, its time complexity is `O(n^2)` and will not complete in reasonable time for the upper constraint of `n = 10^5`.

The optimal approach relies on sorting the array first. After sorting, for each element `nums[i]`, we want to count how many elements `nums[j]` with `j > i` produce sums within the range `[lower, upper]`. Because the array is sorted, for each `i`, we can efficiently find the range of valid `j` using binary search to identify the first index where the sum reaches `lower` and the first index where the sum exceeds `upper`. The difference gives the number of valid pairs for that `i`. This reduces the problem to `O(n log n)` complexity because of the sort and subsequent binary searches.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Checks all pairs individually, too slow for large n |
| Optimal | O(n log n) | O(1) | Sorts array and uses binary search for efficient counting |

## Algorithm Walkthrough

1. **Sort the array** `nums` in ascending order. Sorting enables binary search on subsequent elements to count valid sums quickly.
2. **Initialize a counter** `count = 0` to accumulate the number of fair pairs.
3. **Iterate over each index** `i` from `0` to `n-2`. For each element `nums[i]`:

1. Calculate `lower_target = lower - nums[i]` and `upper_target = upper - nums[i]`.
2. Use **binary search** to find the smallest index `left` greater than `i` where `nums[left] >= lower_target`.
3. Use **binary search** to find the smallest index `right` greater than `i` where `nums[right] > upper_target`.
4. The number of valid pairs with `i` as the first element is `right - left`. Add this to `count`.
4. **Return `count`** as the total number of fair pairs.

Why it works: Sorting ensures that for any `i`, all `nums[j]` with `j > i` are in ascending order. This allows us to efficiently find the range of valid `j` using binary search without checking each one individually. The difference `right - left` directly counts all indices `j` producing sums within `[lower, upper]`.

## Python Solution

```python
from bisect import bisect_left, bisect_right
from typing import List

class Solution:
    def countFairPairs(self, nums: List[int], lower: int, upper: int) -> int:
        nums.sort()
        count = 0
        n = len(nums)
        
        for i in range(n - 1):
            left = bisect_left(nums, lower - nums[i], i + 1)
            right = bisect_right(nums, upper - nums[i], i + 1)
            count += right - left
        
        return count
```

The Python solution first sorts `nums`. For each element `nums[i]`, `bisect_left` finds the first index after `i` that meets the lower bound, and `bisect_right` finds the first index that exceeds the upper bound. Subtracting these indices gives the number of valid pairs. Accumulating over all `i` yields the total number of fair pairs.

## Go Solution

```go
import "sort"

func countFairPairs(nums []int, lower int, upper int) int64 {
    sort.Ints(nums)
    var count int64 = 0
    n := len(nums)
    
    for i := 0; i < n-1; i++ {
        left := sort.Search(n, func(j int) bool {
            return j > i && nums[j] >= lower - nums[i]
        })
        right := sort.Search(n, func(j int) bool {
            return j > i && nums[j] > upper - nums[i]
        })
        count += int64(right - left)
    }
    
    return count
}
```

In Go, `sort.Search` performs binary search. The anonymous function ensures that indices are greater than `i`. The difference `right - left` gives the number of valid pairs, cast to `int64` to handle large counts without overflow.
