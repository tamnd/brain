## Solution

## Problem Understanding

The problem asks us to determine how many unique prime numbers appear in the prime factorization of the product of all numbers in the array.

Suppose we are given an array `nums`. If we multiply every element together, we get one large number. That large number can be broken down into its prime factors. Our goal is not to count how many times each prime appears, but only how many different prime values exist.

For example, if the product factorizes into:

```
2^5 * 3^2 * 5 * 7
```

then the distinct prime factors are `{2, 3, 5, 7}`, so the answer is `4`.

The input consists of:

- An integer array `nums`
- Each element is between `2` and `1000`
- The array length can be as large as `10^4`

The constraints are important because multiplying all numbers together directly could create an astronomically large integer. Even though Python supports arbitrarily large integers, constructing such a huge product is unnecessary and inefficient. Instead, we should exploit the mathematical property that the distinct prime factors of the product are simply the union of the prime factors of each individual number.

A few important observations and edge cases emerge immediately:

- Multiple numbers may share the same prime factors, but duplicates should only be counted once.
- Numbers can themselves already be prime.
- Numbers may contain repeated prime factors, such as `12 = 2^2 * 3`, but repeated occurrences of the same prime should not increase the answer.
- The smallest possible array contains one element.
- Arrays where every number is a power of the same prime should return `1`.
- Arrays with many composite numbers may still contain only a small set of distinct primes.

## Approaches

### Brute Force Approach

A straightforward approach would be:

1. Multiply all numbers in the array together.
2. Compute the prime factorization of the final product.
3. Count the distinct primes.

This works because the prime factorization of the product contains exactly the primes contributed by all numbers in the array.

However, this approach becomes problematic because the product can become extremely large. Consider an array of length `10^4` where each number is close to `1000`. The resulting product would contain tens of thousands of digits. Factoring such a massive number is inefficient and unnecessary.

Even if the language supports big integers, factorizing the gigantic product is much slower than factorizing each small number individually.

### Optimal Approach

The key insight is that we never actually need the full product.

If:

```
a = p1 * p2
b = p2 * p3
```

then:

```
a * b = p1 * p2^2 * p3
```

The distinct prime factors of the product are simply the union of the prime factors from each number.

Therefore, instead of building the huge product, we can:

1. Factorize each number independently.
2. Insert every discovered prime factor into a set.
3. Return the size of the set.

This works efficiently because every number is at most `1000`, making prime factorization very cheap.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Extremely large, impractical | Very large | Builds gigantic product before factorization |
| Optimal | O(n * sqrt(m)) | O(k) | Factorizes each number independently |

Here:

- `n` is the number of elements
- `m` is the maximum value in `nums`
- `k` is the number of distinct prime factors collected

Since `m <= 1000`, the square root is tiny, making the solution very efficient.

## Algorithm Walkthrough

1. Create an empty set called `prime_factors`.

A set is ideal because it automatically removes duplicates. If the same prime appears multiple times across different numbers, it will still only exist once in the set.
2. Iterate through every number in `nums`.

Each number will be factorized independently.
3. For the current number, start checking divisors from `2`.

Every composite number can be broken into prime factors using trial division.
4. While the divisor squared is less than or equal to the current number:

If the divisor evenly divides the number:

- Add the divisor to the set
- Continuously divide the number by that divisor until it no longer divides evenly

Repeated division removes all copies of the same prime factor efficiently.
5. Increment the divisor and continue searching.

This ensures we discover all prime factors.
6. After the loop finishes, check whether the remaining number is greater than `1`.

If it is, the remaining value itself must be a prime factor, so add it to the set.
7. After processing all numbers, return the size of the set.

### Why it works

Every integer greater than `1` has a unique prime factorization. By factorizing every number individually and storing all discovered primes in a set, we collect exactly the primes that appear in the product of the entire array.

Because multiplication combines prime exponents but does not introduce new primes, the union of all individual prime factors is exactly the set of distinct prime factors of the final product.

## Python Solution

```python
from typing import List

class Solution:
    def distinctPrimeFactors(self, nums: List[int]) -> int:
        prime_factors = set()

        for number in nums:
            divisor = 2

            while divisor * divisor <= number:
                if number % divisor == 0:
                    prime_factors.add(divisor)

                    while number % divisor == 0:
                        number //= divisor

                divisor += 1

            if number > 1:
                prime_factors.add(number)

        return len(prime_factors)
```

The implementation begins by creating a set called `prime_factors`, which stores every distinct prime encountered during factorization.

The outer loop iterates through each number in the input array. For every number, we attempt trial division starting from `2`.

The condition:

```python
while divisor * divisor <= number
```

works because if a number has a factor larger than its square root, the complementary factor must be smaller than the square root. Therefore, all non-prime factors are discoverable within this range.

Whenever a divisor evenly divides the number, we add it to the set and repeatedly divide the number until that factor is completely removed. This avoids processing the same prime multiple times.

After the loop, if the remaining number is greater than `1`, it must itself be prime, so we add it to the set.

Finally, the answer is simply the number of unique primes stored in the set.

## Go Solution

```go
func distinctPrimeFactors(nums []int) int {
    primeFactors := make(map[int]bool)

    for _, number := range nums {
        divisor := 2

        for divisor*divisor <= number {
            if number%divisor == 0 {
                primeFactors[divisor] = true

                for number%divisor == 0 {
                    number /= divisor
                }
            }

            divisor++
        }

        if number > 1 {
            primeFactors[number] = true
        }
    }

    return len(primeFactors)
}
```

The Go implementation follows the exact same logic as the Python version.

Since Go does not have a built in set type, we simulate one using:

```go
map[int]bool
```

The keys represent the distinct prime factors. The boolean values are irrelevant, we only care about key existence.

Integer division in Go naturally truncates toward zero, which works perfectly for repeated factor removal.

No overflow concerns exist because we never multiply the array elements together. We only factorize numbers individually, and every value remains within the problem constraints.

## Edge Cases

One important edge case occurs when all numbers are powers of the same prime, such as:

```
[2, 4, 8, 16]
```

A buggy implementation might incorrectly count repeated appearances of the same prime multiple times. Using a set guarantees that each prime factor is stored only once, so the correct answer remains `1`.

Another important case involves numbers with repeated internal factors, such as:

```
12 = 2^2 * 3
```

If the implementation only divides once after discovering a factor, it may leave extra copies behind and incorrectly process them later. The inner `while` loop repeatedly removes the factor until it no longer divides evenly, ensuring clean factorization.

A third important case involves prime numbers themselves, such as `997`. Trial division will never find a smaller divisor, so after the loop finishes, the remaining number is still greater than `1`. The final check:

```
if number > 1:
```

correctly recognizes the remaining value as a prime factor and adds it to the set.

Another subtle edge case occurs when multiple numbers share overlapping factors, such as:

```
[6, 10, 15]
```

The primes `{2, 3, 5}` appear across different numbers in different combinations. Since the algorithm independently factorizes every number and inserts factors into a shared set, all distinct primes are collected correctly without duplication.
