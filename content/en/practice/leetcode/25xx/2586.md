## Solution

## Problem Understanding

The problem gives us a 0-indexed array of strings called `words`, along with two integers, `left` and `right`. We need to count how many strings within the inclusive index range `[left, right]` are considered vowel strings.

A string qualifies as a vowel string if both of the following conditions are true:

- The first character is a vowel
- The last character is also a vowel

The valid vowels are:

```
a, e, i, o, u
```

For each word between indices `left` and `right`, we examine the first and last characters and determine whether both belong to the vowel set. If they do, we include that word in the count.

The input constraints are small:

- There can be at most 1000 words
- Each word has length at most 10

These constraints tell us that even a straightforward linear scan is efficient enough. There is no need for advanced optimization techniques such as prefix sums or dynamic programming because the total amount of work is already tiny.

The problem also guarantees several useful properties:

- Every word contains at least one character
- All characters are lowercase English letters
- `left` and `right` are valid indices
- `left <= right`

These guarantees simplify the implementation because we never need to handle empty strings or invalid ranges.

Several edge cases are important to think about:

- A single-character word like `"u"` should count because the first and last character are the same vowel.
- Words that start with a vowel but end with a consonant should not count.
- Words that end with a vowel but start with a consonant should not count.
- The range may contain only one element when `left == right`.

A correct solution must handle all of these cases consistently.

## Approaches

### Brute Force Approach

The most direct solution is to iterate through every word in the range `[left, right]` and check whether the first and last characters are vowels.

For each word:

- Extract the first character
- Extract the last character
- Check whether both are in the vowel set
- If both are vowels, increment the answer

This approach is correct because the problem definition is completely local to each word. The validity of one word does not depend on any other word.

Although this is called a brute-force approach, it is actually perfectly acceptable for the given constraints because there are at most 1000 words.

### Key Insight for the Optimal Approach

The key observation is that each word can be evaluated independently in constant time.

Checking whether a character is a vowel is fast if we store the vowels in a set. A set provides average-case `O(1)` membership checks.

Since we only need to scan the specified range once, the overall algorithm becomes linear in the number of words examined.

There is no need for preprocessing or extra data structures beyond a small vowel set.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Scan every word in the range and check first and last characters |
| Optimal | O(n) | O(1) | Uses a vowel set for constant-time vowel checks |

Here, `n` represents the number of words between `left` and `right`.

## Algorithm Walkthrough

1. Create a set containing all vowels: `'a'`, `'e'`, `'i'`, `'o'`, and `'u'`.

A set is used because membership checks are very fast. We will repeatedly ask whether a character is a vowel, so constant-time lookup is ideal.
2. Initialize a counter variable to `0`.

This variable keeps track of how many valid vowel strings we find.
3. Iterate through the indices from `left` to `right`, inclusive.

We only care about the specified portion of the array, so there is no need to examine words outside this range.
4. For each word, extract the first and last characters.

Since every word has length at least 1, both operations are always safe.
5. Check whether both characters exist in the vowel set.

If both checks succeed, the word satisfies the problem definition.
6. Increment the counter whenever a valid vowel string is found.
7. After processing the entire range, return the counter.

### Why it works

The algorithm checks exactly the condition given in the problem statement for every word in the required range. A word is counted if and only if its first and last characters are vowels. Since every relevant word is examined once and evaluated correctly, the final counter equals the number of vowel strings in the range.

## Python Solution

```python
from typing import List

class Solution:
    def vowelStrings(self, words: List[str], left: int, right: int) -> int:
        vowels = {"a", "e", "i", "o", "u"}
        count = 0

        for index in range(left, right + 1):
            word = words[index]

            if word[0] in vowels and word[-1] in vowels:
                count += 1

        return count
```

The implementation starts by creating a set of vowels. Using a set makes the membership checks efficient and clean.

The variable `count` stores the running total of valid vowel strings.

The loop iterates from `left` to `right`, inclusive. For each word, the code accesses:

- `word[0]` for the first character
- `word[-1]` for the last character

If both characters are present in the vowel set, the counter increases by one.

Finally, the function returns the total count after all words in the range have been processed.

## Go Solution

```go
func vowelStrings(words []string, left int, right int) int {
    vowels := map[byte]bool{
        'a': true,
        'e': true,
        'i': true,
        'o': true,
        'u': true,
    }

    count := 0

    for i := left; i <= right; i++ {
        word := words[i]

        if vowels[word[0]] && vowels[word[len(word)-1]] {
            count++
        }
    }

    return count
}
```

The Go implementation follows the same logic as the Python version.

Instead of a Python set, Go uses a `map[byte]bool` to represent the vowel lookup table. Since the strings contain only lowercase English letters, accessing bytes directly is safe and efficient.

The expression `word[len(word)-1]` retrieves the last character of the string.

No special handling is needed for empty strings because the constraints guarantee that every word has length at least 1.

## Edge Cases

One important edge case is a single-character word such as `"a"` or `"u"`. In these cases, the first and last character are the same character. A buggy implementation might incorrectly assume the word must contain at least two characters. This solution handles the case naturally because `word[0]` and `word[-1]` both refer to the same vowel character.

Another important edge case occurs when `left == right`. This means the range contains exactly one word. Some implementations accidentally use incorrect loop bounds and skip processing entirely. The current implementation uses an inclusive loop range, so the single word is correctly evaluated.

A third edge case involves words that satisfy only one of the two conditions. For example:

- `"ab"` starts with a vowel but ends with a consonant
- `"ba"` starts with a consonant but ends with a vowel

A common bug is using logical OR instead of logical AND. The implementation explicitly requires both conditions to be true before incrementing the count, which guarantees correctness.
