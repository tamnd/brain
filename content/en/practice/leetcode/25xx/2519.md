## Solution

## Problem Understanding

The problem asks us to find the number of **k-big indices** in a given array `nums`. An index `i` is considered k-big if there are at least `k` elements to the **left** of `i` that are **strictly smaller than `nums[i]`** and at least `k` elements to the **right** of `i` that are **strictly smaller than `nums[i]`**.

The input consists of a 0-indexed array of integers `nums` of length up to $10^5$ and an integer `k`. The output is a single integer representing the number of indices that satisfy the k-big condition.

The constraints are significant: the array can be very large, so a naive solution that checks every pair of indices would be too slow. Also, `k` can be as large as the length of the array, which may result in no valid indices.

Important edge cases include arrays where all elements are equal, arrays of length less than `k` (which cannot have any k-big indices), and arrays with strictly increasing or decreasing sequences.

## Approaches

The **brute-force approach** would involve iterating over each index `i` in `nums` and counting the number of elements less than `nums[i]` to its left and right. If both counts are at least `k`, we increment the result. This approach works correctly but requires $O(n^2)$ time in the worst case, which is too slow for $n = 10^5$.

The **optimal approach** relies on using a **Binary Indexed Tree (Fenwick Tree)** or **Segment Tree** to efficiently maintain prefix counts of values. The key insight is that for each index `i`, we want to know how many values less than `nums[i]` appear to the left and right. By compressing the array values and using a Fenwick Tree, we can query and update counts in $O(\log n)$ time, reducing the overall complexity to $O(n \log n)$.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check every index by counting elements on both sides |
| Optimal | O(n log n) | O(n) | Use value compression + Fenwick Tree to count elements efficiently |

## Algorithm Walkthrough

1. **Coordinate Compression**: To use a Fenwick Tree efficiently, map all numbers in `nums` to a range `[1, n]` while preserving their order. This avoids working with large arbitrary numbers.
2. **Initialize Fenwick Trees**: Create two Fenwick Trees - one for counting elements to the left of `i` and another for counting elements to the right.
3. **Right-Side Counts**: Iterate through the array once to populate a right-side frequency array or Fenwick Tree for all elements.
4. **Iterate Through Array**: For each index `i`, compute:

- The number of elements to the left smaller than `nums[i]` by querying the left Fenwick Tree.
- The number of elements to the right smaller than `nums[i]` by querying the right Fenwick Tree (decrement current element from right tree before query).

If both counts are ≥ k, increment the answer.
5. **Update Left Tree**: After processing index `i`, increment the count of `nums[i]` in the left Fenwick Tree to include it for future queries.
6. **Return Result**: After iterating all indices, return the total count of k-big indices.

**Why it works**: At each step, the Fenwick Trees efficiently maintain cumulative counts of values less than any target. The left tree contains counts of all values before `i` and the right tree counts all values after `i`. This guarantees correct counts in $O(\log n)$ time.

## Python Solution

```python
from typing import List
from bisect import bisect_left

class FenwickTree:
    def __init__(self, size: int):
        self.tree = [0] * (size + 1)
        self.size = size

    def update(self, index: int, value: int):
        while index <= self.size:
            self.tree[index] += value
            index += index & -index

    def query(self, index: int) -> int:
        result = 0
        while index > 0:
            result += self.tree[index]
            index -= index & -index
        return result

class Solution:
    def kBigIndices(self, nums: List[int], k: int) -> int:
        n = len(nums)
        sorted_unique = sorted(set(nums))
        ranks = {num: i + 1 for i, num in enumerate(sorted_unique)}

        left_tree = FenwickTree(len(ranks))
        right_tree = FenwickTree(len(ranks))

        for num in nums:
            right_tree.update(ranks[num], 1)

        result = 0
        for num in nums:
            rank = ranks[num]
            right_tree.update(rank, -1)

            left_count = left_tree.query(rank - 1)
            right_count = right_tree.query(rank - 1)

            if left_count >= k and right_count >= k:
                result += 1

            left_tree.update(rank, 1)

        return result
```

**Explanation**: We first compress the values in `nums` to smaller ranks suitable for a Fenwick Tree. The right Fenwick Tree is initialized with counts of all numbers. During iteration, we temporarily remove the current element from the right tree, query both trees for counts of smaller numbers, check the k-big condition, and then update the left tree.

## Go Solution

```go
package main

func kBigIndices(nums []int, k int) int {
    n := len(nums)
    sorted := make([]int, n)
    copy(sorted, nums)
    sort.Ints(sorted)

    rankMap := make(map[int]int)
    r := 1
    for _, val := range sorted {
        if _, exists := rankMap[val]; !exists {
            rankMap[val] = r
            r++
        }
    }

    leftTree := make([]int, r+1)
    rightTree := make([]int, r+1)

    update := func(tree []int, index int, val int) {
        for index < len(tree) {
            tree[index] += val
            index += index & -index
        }
    }

    query := func(tree []int, index int) int {
        res := 0
        for index > 0 {
            res += tree[index]
            index -= index & -index
        }
        return res
    }

    for _, num := range nums {
        update(rightTree, rankMap[num], 1)
    }

    result := 0
    for _, num := range nums {
        r := rankMap[num]
        update(rightTree, r, -1)

        leftCount := query(leftTree, r-1)
        rightCount := query(rightTree, r-1)

        if leftCount >= k && rightCount >= k {
            result++
        }

        update(leftTree, r, 1)
    }

    return result
}
```

**Go-Specific Notes**: Slice-based Fenwick Trees are used with helper functions for update and query. The rank mapping handles large numbers safely. Go requires explicit slice bounds checks, so `len(tree)` ensures no out-of-bounds errors.
