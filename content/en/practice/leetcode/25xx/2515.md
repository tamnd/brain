## Solution

## Problem Understanding

The problem asks us to find the shortest distance from a given `startIndex` to a target string in a **circular array** of words. A circular array means that after the last element, the next element wraps around to the first element, and similarly, before the first element, the previous element wraps around to the last element.

The input consists of three parts: a list of words `words`, a string `target`, and an integer `startIndex` indicating the starting position. The expected output is the minimum number of steps needed to reach `target` from `startIndex`, moving either left or right along the circular array. If the `target` is not present in `words`, the output should be `-1`.

The constraints indicate that `words` has at most 100 elements, and each word has at most 100 characters, meaning a brute-force solution will be acceptable in terms of efficiency. The constraints also guarantee valid input indices, lowercase English letters, and a non-empty array.

Important edge cases include situations where `target` occurs at the `startIndex` itself, when it appears multiple times in the array, and when it does not exist at all. Circular wrap-around behavior is another edge case that must be handled carefully.

## Approaches

The brute-force approach involves iterating through the array in both directions (left and right) from the `startIndex` and computing the distance to every occurrence of the `target`. We keep track of the minimum distance found. This works correctly but can be slightly redundant since we may compute distances multiple times.

The optimal approach relies on the observation that we can compute circular distances mathematically. For each index `i` where `words[i] == target`, the circular distance from `startIndex` can be computed as `min(abs(i - startIndex), n - abs(i - startIndex))`. This directly captures both forward and backward movements in a single calculation, and iterating through all `n` elements once is sufficient because `n <= 100`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Iterate left and right from startIndex for each target occurrence |
| Optimal | O(n) | O(1) | Compute circular distance for each occurrence directly |

## Algorithm Walkthrough

1. Initialize `n` as the length of `words`.
2. Initialize `min_distance` to infinity as a placeholder for the shortest distance.
3. Iterate through the array with index `i` from `0` to `n - 1`.
4. If `words[i] == target`, compute the circular distance from `startIndex` as `min(abs(i - startIndex), n - abs(i - startIndex))`.
5. Update `min_distance` if the computed distance is smaller than the current `min_distance`.
6. After iterating through the array, check if `min_distance` is still infinity. If so, return `-1` since `target` does not exist.
7. Otherwise, return `min_distance`.

Why it works: The algorithm correctly computes the minimum distance in both directions using the circular distance formula, ensuring we always find the shortest path. The iteration over the array ensures that all occurrences of `target` are considered.

## Python Solution

```python
from typing import List

class Solution:
    def closestTarget(self, words: List[str], target: str, startIndex: int) -> int:
        n = len(words)
        min_distance = float('inf')
        
        for i in range(n):
            if words[i] == target:
                distance = min(abs(i - startIndex), n - abs(i - startIndex))
                min_distance = min(min_distance, distance)
        
        return -1 if min_distance == float('inf') else min_distance
```

The implementation first computes the array length `n` and initializes `min_distance`. It loops over each index to check if the word matches the `target`. When it does, it calculates the circular distance using the formula and updates the minimum distance. Finally, it returns `-1` if no match was found, otherwise the shortest distance.

## Go Solution

```go
import "math"

func closestTarget(words []string, target string, startIndex int) int {
    n := len(words)
    minDistance := math.MaxInt32
    
    for i := 0; i < n; i++ {
        if words[i] == target {
            diff := i - startIndex
            if diff < 0 {
                diff = -diff
            }
            distance := diff
            if n - diff < distance {
                distance = n - diff
            }
            if distance < minDistance {
                minDistance = distance
            }
        }
    }
    
    if minDistance == math.MaxInt32 {
        return -1
    }
    return minDistance
}
```

The Go version uses `math.MaxInt32` as a placeholder for infinity. It iterates through the array, checks each word, calculates circular distance, and updates the minimum. The logic is identical to the Python version, adapted to Go's type system and integer handling.

## Edge Cases

One important edge case is when the target is at the `startIndex` itself. In this case, the shortest distance should be `0`. The implementation correctly computes `abs(i - startIndex) = 0` and updates `min_distance`.

Another edge case is when the target appears multiple times, and the shortest path involves wrapping around the circular array. The algorithm handles this by computing `min(abs(i - startIndex), n - abs(i - startIndex))` for each occurrence, ensuring that the circular path is considered.

A third edge case is when the target does not exist at all. Without a proper check, one might incorrectly return a large distance. The implementation uses `min_distance` initialized to infinity and returns `-1` if no match is found, correctly handling this scenario.
