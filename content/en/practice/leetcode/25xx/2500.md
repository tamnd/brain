# LeetCode 2500 - Delete Greatest Value in Each Row

## Problem Understanding

The problem gives us an `m x n` matrix called `grid`, where every value is a positive integer. We repeatedly perform an operation until the matrix becomes empty.

In each operation:

1. We remove the largest value from every row.
2. Among all removed values, we take the largest one and add it to the answer.

Since one element is removed from every row during each operation, the number of columns decreases by one after every round. After exactly `n` operations, all elements will be removed.

The goal is to compute the total accumulated sum.

For example, consider:

```
grid = [[1,2,4],
        [3,3,1]]
```

In the first round:

- Remove `4` from the first row
- Remove `3` from the second row
- Maximum removed value is `4`

Answer becomes `4`.

In the second round:

- Remove `2`
- Remove `3`
- Maximum removed value is `3`

Answer becomes `7`.

In the final round:

- Remove `1`
- Remove `1`
- Maximum removed value is `1`

Final answer becomes `8`.

The constraints are small:

- `1 <= m, n <= 50`
- `1 <= grid[i][j] <= 100`

This tells us that even moderately inefficient approaches may still pass, but we should still design a clean and optimal solution.

An important observation is that we are always removing the current maximum element from each row. If we sort each row beforehand, then the largest remaining element in a row is always at the end. This turns the repeated deletion process into a much simpler column-wise comparison problem.

Some important edge cases include:

- A matrix with only one element
- Rows containing duplicate maximum values
- Rows already sorted in increasing or decreasing order
- Multiple rows having the same removed maximum during a round

The problem guarantees that all rows have the same number of columns and all values are positive integers, so we never need to handle malformed input.

## Approaches

### Brute Force Approach

A straightforward simulation would repeatedly:

1. Find the maximum element in every row
2. Remove it
3. Track the largest removed value
4. Add that value to the answer

This correctly follows the problem statement exactly.

However, removing elements dynamically from arbitrary positions inside rows can become inefficient. Each round requires scanning every row to locate the maximum, and removing from lists may involve shifting elements.

Although the constraints are small enough for this to pass, the implementation becomes unnecessarily complicated.

### Key Insight

The key observation is that the order in which elements are removed from a row is completely determined by sorting.

If we sort every row in ascending order:

```
[1,2,4]
```

then the deletion order becomes:

```
4, 2, 1
```

which corresponds to iterating from right to left.

Now consider all rows sorted. During each operation, we are effectively comparing elements from the same column position after sorting.

For example:

```
[1,2,4]
[1,3,3]
```

Column-wise maximums are:

```
max(1,1) = 1
max(2,3) = 3
max(4,3) = 4
```

Adding them gives:

```
1 + 3 + 4 = 8
```

This avoids repeated deletion simulation entirely.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m × n²) | O(1) | Repeatedly scans rows and removes maximum values |
| Optimal | O(m × n log n) | O(1) extra | Sort rows once, then process columns |

## Algorithm Walkthrough

1. Sort every row in ascending order.

Sorting ensures that elements are arranged from smallest to largest. After sorting, the largest element in each row appears at the last column, the second largest at the second-to-last column, and so on.
2. Initialize the answer variable to `0`.

This variable stores the cumulative result.
3. Iterate through every column index from left to right.

Since rows are sorted, elements in the same column represent values removed during the same operation.
4. For each column, compute the maximum value across all rows.

This simulates the problem rule where we add the largest deleted value from the current round.
5. Add that maximum value to the answer.
6. After processing all columns, return the answer.

### Why it works

Sorting each row preserves the relative order in which elements would be removed during repeated maximum deletions. The largest values align in the last column, second largest values align in the second-to-last column, and so forth. Therefore, processing columns after sorting exactly simulates the original operations. Taking the maximum value from each column matches the rule of adding the largest removed element in every round.

## Python Solution

```python
from typing import List

class Solution:
    def deleteGreatestValue(self, grid: List[List[int]]) -> int:
        # Sort every row
        for row in grid:
            row.sort()

        answer = 0
        cols = len(grid[0])

        # Process each column
        for col in range(cols):
            current_max = 0

            for row in grid:
                current_max = max(current_max, row[col])

            answer += current_max

        return answer
```

The implementation begins by sorting every row in ascending order. This transforms the repeated deletion process into a simple column traversal problem.

The variable `answer` stores the final accumulated result.

We then iterate through each column index. For every column, we scan all rows and determine the largest value appearing in that column. This corresponds to the largest deleted element during that operation.

Finally, we add the column maximum to the answer and return the total.

## Go Solution

```go
package main

import "sort"

func deleteGreatestValue(grid [][]int) int {
    // Sort each row
    for i := 0; i < len(grid); i++ {
        sort.Ints(grid[i])
    }

    rows := len(grid)
    cols := len(grid[0])

    answer := 0

    // Process each column
    for col := 0; col < cols; col++ {
        currentMax := 0

        for row := 0; row < rows; row++ {
            if grid[row][col] > currentMax {
                currentMax = grid[row][col]
            }
        }

        answer += currentMax
    }

    return answer
}
```

The Go implementation follows the same logic as the Python version. The primary difference is that Go uses `sort.Ints()` to sort slices in place.

Since the constraints are small and all values are positive integers, standard `int` arithmetic is completely safe. Go slices naturally handle the matrix representation efficiently.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(m × n log n) | Each of the `m` rows is sorted independently |
| Space | O(1) extra | Sorting is done in place apart from sorting internals |

The dominant operation is sorting each row. Each row contains `n` elements, so sorting costs `O(n log n)` per row. Since there are `m` rows, the total complexity becomes `O(m × n log n)`.

The column traversal afterward costs only `O(m × n)`, which is smaller than the sorting cost.

## Edge Cases

### Single Element Matrix

A matrix containing only one element is the smallest valid input:

```
[[10]]
```

A buggy implementation might incorrectly assume multiple rounds or multiple rows exist. Our solution handles this naturally because sorting a single-element row changes nothing, and the column traversal runs exactly once.

### Duplicate Maximum Values

Rows may contain repeated largest values:

```
[[3,3,1]]
```

The problem allows removing any occurrence of the maximum value. Sorting eliminates ambiguity because identical values remain grouped together, and the exact removal position no longer matters.

### Single Column Matrices

Consider:

```
[[1],
 [5],
 [3]]
```

There is only one operation. A careless implementation could incorrectly attempt multiple deletion rounds. Our solution processes exactly one column, computes the maximum value `5`, and returns it correctly.

### Rows Already in Descending Order

Input rows may already be sorted in reverse order:

```
[[9,8,7]]
```

If an implementation assumes rows are already ascending, the logic breaks. Our algorithm explicitly sorts every row before processing, guaranteeing correctness regardless of initial ordering.
