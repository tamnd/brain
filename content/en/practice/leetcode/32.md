## Solution

## Problem Understanding

The problem gives a string containing only two characters, `'('` and `')'`. We must determine the length of the longest contiguous substring that forms a valid parentheses sequence.

A valid parentheses sequence is one where every opening parenthesis has a matching closing parenthesis in the correct order. Examples of valid sequences include `"()"`, `"(())"`, and `"()()"`. Examples of invalid sequences include `"(()"`, `")("`, and `"())("`.

The important detail is that we are looking for a substring, which means the characters must appear contiguously in the original string. We are not allowed to skip characters or rearrange them.

The input size can be as large as `3 * 10^4`, which means inefficient approaches will not pass. Any solution that repeatedly scans large portions of the string will likely become too slow. This strongly suggests that we need a linear time solution, or something very close to it.

Several edge cases are especially important:

- The string may be empty, in which case the answer is `0`.
- The string may contain only `'('` or only `')'`, which means no valid substring exists.
- A valid substring may appear in the middle of invalid characters.
- Nested parentheses such as `"(())"` must be handled correctly.
- Adjacent valid groups such as `"()()"` must also be recognized as a single longer valid substring.

A naive implementation can easily fail when valid regions are separated by unmatched parentheses, or when nested structures must be combined into a larger valid length.

## Approaches

### Brute Force Approach

The brute force solution checks every possible substring and determines whether it forms a valid parentheses sequence.

For a string of length `n`, there are `O(n^2)` possible substrings. For each substring, we can verify validity using a counter or stack. While scanning the substring:

- Increment for `'('`
- Decrement for `')'`
- If the counter ever becomes negative, the substring is invalid
- At the end, the counter must be zero

This works because a valid parentheses sequence must never close more parentheses than it has opened, and all opened parentheses must eventually close.

Although this approach is correct, it is far too slow for the constraints. Checking all substrings already costs `O(n^2)`, and validating each substring costs another `O(n)`, leading to `O(n^3)` time complexity in the worst case.

With `n = 30000`, such a solution is completely impractical.

### Optimal Stack-Based Approach

The key observation is that valid parentheses always form around matching pairs. Instead of checking every substring independently, we can process the string once while tracking unmatched positions.

A stack is especially useful because parentheses naturally follow a nested structure. When we encounter `'('`, we store its index. When we encounter `')'`, we attempt to match it with the most recent unmatched `'('`.

The crucial insight is that indices allow us to compute valid substring lengths efficiently.

We maintain:

- A stack containing indices
- A sentinel value `-1` at the bottom of the stack

The sentinel acts as the starting boundary for future valid substrings.

When we encounter:

- `'('`, we push its index
- `')'`, we pop from the stack

After popping:

- If the stack becomes empty, this `')'` cannot be matched, so we push its index as a new boundary
- Otherwise, the current valid substring length is:

current_index - stack[-1]

This works because the top of the stack represents the index immediately before the start of the current valid region.

This approach processes each character once, producing an efficient `O(n)` solution.

## Algorithm Walkthrough

1. Initialize a stack with the value `-1`.

The `-1` acts as a sentinel boundary. It helps compute lengths correctly when a valid substring starts at index `0`.
2. Initialize a variable `max_length` to `0`.

This variable stores the length of the longest valid substring found so far.
3. Traverse the string character by character using index `i`.

We need indices because substring lengths are computed from positions.
4. If the current character is `'('`, push its index onto the stack.

This index represents an unmatched opening parenthesis that may later pair with a closing parenthesis.
5. If the current character is `')'`, pop one index from the stack.

This attempts to match the current closing parenthesis with the most recent unmatched opening parenthesis.
6. After popping, check whether the stack is empty.

If it is empty, there is no valid opening parenthesis available for matching. This means the current `')'` becomes a new invalid boundary. Push its index onto the stack.
7. If the stack is not empty, compute the current valid substring length.

The length is:

current_index - stack[-1]

The top of the stack now represents the index immediately before the valid substring begins.
8. Update `max_length` if the current valid length is larger.
9. Continue until the end of the string.
10. Return `max_length`.

## Python Solution

```
class Solution:
    def longestValidParentheses(self, s: str) -> int:
        stack = [-1]
        max_length = 0

        for index, char in enumerate(s):
            if char == '(':
                stack.append(index)
            else:
                stack.pop()

                if not stack:
                    stack.append(index)
                else:
                    current_length = index - stack[-1]
                    max_length = max(max_length, current_length)

        return max_length
```

The implementation begins by initializing the stack with `-1`. This sentinel value simplifies length calculations and avoids special handling for substrings beginning at index `0`.

As we iterate through the string, every `'('` index is pushed onto the stack. This records unmatched opening parentheses.

When a `')'` appears, we pop from the stack to match it against the most recent `'('`.

If the stack becomes empty after popping, the current `')'` has no matching `'('`. We therefore push its index as a boundary marker. Future valid substrings must start after this point.

If the stack is still non-empty, the current substring is valid. The length is computed using the difference between the current index and the top element of the stack.

The algorithm updates `max_length` whenever a longer valid substring is discovered.

Because every character is processed exactly once, the implementation runs efficiently in linear time.

## Go Solution

```
func longestValidParentheses(s string) int {
    stack := []int{-1}
    maxLength := 0

    for i, char := range s {
        if char == '(' {
            stack = append(stack, i)
        } else {
            stack = stack[:len(stack)-1]

            if len(stack) == 0 {
                stack = append(stack, i)
            } else {
                currentLength := i - stack[len(stack)-1]

                if currentLength > maxLength {
                    maxLength = currentLength
                }
            }
        }
    }

    return maxLength
}
```

The Go implementation follows the same logic as the Python solution.

Slices are used to simulate stack behavior. Appending pushes elements onto the stack, and slicing removes the top element.

Go strings are UTF-8 encoded, but since the input contains only `'('` and `')'`, iterating with `range` works safely and efficiently.

Unlike Python, Go does not provide a built-in `max` function for integers, so we manually compare and update `maxLength`.

The solution handles empty strings naturally because the loop simply never executes, leaving `maxLength` equal to `0`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is pushed and popped at most once |
| Space | O(n) | The stack may store all indices in the worst case |

The algorithm is linear because every index enters and leaves the stack at most one time. No nested rescanning occurs.

The space complexity is also linear because the stack can grow to size `n` in cases such as `"(((((("`.

## Edge Cases

### Empty String

An empty string contains no parentheses, so no valid substring exists.

This case can easily cause issues in algorithms that assume at least one character exists. The implementation handles it naturally because the traversal loop never runs, and the initial value `0` is returned.

### All Opening Parentheses

Consider:

```
"(((("
```

Every character is pushed onto the stack, but no closing parenthesis ever appears to form a match.

A buggy implementation might incorrectly count unmatched opening parentheses. Our algorithm only updates the answer when a successful match occurs after processing a `')'`, so the final result correctly remains `0`.

### All Closing Parentheses

Consider:

```
"))))"
```

Each closing parenthesis immediately invalidates the current region because there is no matching opening parenthesis.

The algorithm handles this by resetting the boundary whenever the stack becomes empty. Each unmatched `')'` becomes the new reference point for future calculations.

### Nested Parentheses

Consider:

```
"(())"
```

Nested structures are important because the algorithm must combine multiple matching layers into one longer substring.

The stack naturally supports nested matching because the most recent unmatched `'('` is always matched first. This produces the correct answer `4`.

### Multiple Adjacent Valid Groups

Consider:

```
"()()"
```

The algorithm must recognize that adjacent valid groups together form a larger valid substring.

Because the stack stores boundaries rather than only individual matches, the computed length expands across adjacent valid groups correctly, producing the answer `4`.
