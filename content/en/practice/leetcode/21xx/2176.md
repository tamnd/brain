## Solution

## Problem Understanding

The problem gives us a 0-indexed integer array `nums` and an integer `k`. We need to count how many index pairs `(i, j)` satisfy all of the following conditions:

1. `0 <= i < j < n`
2. `nums[i] == nums[j]`
3. `(i * j)` is divisible by `k`

The important detail is that the comparison is performed on indices, not on values. Two elements are considered a valid pair only if:

- Their values are equal.
- The product of their indices is divisible by `k`.

We are not asked to return the pairs themselves, only the total number of valid pairs.

For example, if `nums[i]` and `nums[j]` are equal, we still must check whether `(i * j) % k == 0`. Equal values alone are not enough.

The constraints are small:

- `1 <= nums.length <= 100`
- `1 <= nums[i], k <= 100`

Since the array length is at most 100, even an `O(n^2)` solution is completely acceptable. A quadratic solution would perform at most about 10,000 pair checks, which is very small.

Several edge cases are important:

- Arrays with no repeated values should return `0`.
- Arrays where many values repeat should still correctly enforce the divisibility condition.
- When `k = 1`, every product is divisible by `1`, so the problem reduces to counting equal-value pairs.
- Index `0` deserves special attention because `0 * anything = 0`, and `0` is divisible by every positive integer. Any equal-value pair involving index `0` automatically satisfies the divisibility condition.

## Approaches

### Brute Force Approach

The most direct solution is to examine every possible pair `(i, j)` where `i < j`.

For each pair:

1. Check whether `nums[i] == nums[j]`.
2. If they are equal, check whether `(i * j) % k == 0`.
3. If both conditions are true, increment the answer.

This approach is guaranteed to work because it explicitly evaluates every possible candidate pair. No valid pair can be missed, and no invalid pair can be counted.

The downside is that we compare all pairs, which requires nested loops. The time complexity is `O(n^2)`.

However, because `n <= 100`, this is still efficient enough.

### Optimized Observation

A more refined observation is that only equal values matter. If two values differ, there is no reason to even test divisibility.

We can group indices by their corresponding value using a hash map:

- The key is the number in `nums`.
- The value is a list of indices where that number appears.

Then, for each group of equal values, we only compare indices inside that group.

This avoids unnecessary comparisons between different values.

Although the worst-case time complexity is still `O(n^2)`, in practice it performs fewer checks and makes the intent of the solution clearer.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every pair directly |
| Optimal | O(n²) | O(n) | Groups equal values first, reducing unnecessary comparisons |

## Algorithm Walkthrough

### Optimal Algorithm

1. Create a hash map where each key is a number from `nums`, and the corresponding value is a list of indices where that number appears.
2. Traverse the array once using `enumerate`.

For each index `i` and value `num`:

- Append `i` to the list associated with `num`.

This preprocessing step groups together all equal values.
3. Initialize a variable `count = 0`.

This variable will store the total number of valid pairs.
4. Iterate through each list of indices in the hash map.

Each list contains positions where the same value occurs.
5. For each list of indices, compare every pair of indices `(i, j)` inside that list.

Since all indices in the list correspond to equal values, we only need to test the divisibility condition.
6. For every pair:

- Compute `indices[i] * indices[j]`
- Check whether the product is divisible by `k`

If `(indices[i] * indices[j]) % k == 0`, increment `count`.
7. After processing all groups, return `count`.

### Why it works

The algorithm works because every valid pair must contain equal values. By grouping indices according to value, we ensure that every pair we test already satisfies `nums[i] == nums[j]`.

Inside each group, we examine all possible index pairs exactly once. Therefore:

- Every valid pair is counted.
- No invalid pair is counted.
- No pair is counted multiple times.

This guarantees correctness.

## Python Solution

```python
from collections import defaultdict
from typing import List

class Solution:
    def countPairs(self, nums: List[int], k: int) -> int:
        value_to_indices = defaultdict(list)

        for index, value in enumerate(nums):
            value_to_indices[value].append(index)

        count = 0

        for indices in value_to_indices.values():
            length = len(indices)

            for i in range(length):
                for j in range(i + 1, length):
                    if (indices[i] * indices[j]) % k == 0:
                        count += 1

        return count
```

The implementation begins by creating a `defaultdict(list)`. This data structure automatically creates an empty list whenever a new key is accessed, which makes it convenient for grouping indices.

The first loop traverses the array and records the index of each value. After this loop, all equal values are grouped together.

Next, the algorithm iterates through every list of indices. Since all indices in a single list correspond to equal numbers, we only need to test the divisibility condition.

The nested loops generate every unique pair of indices within the group. The condition `(indices[i] * indices[j]) % k == 0` directly checks whether the product is divisible by `k`.

Whenever the condition is satisfied, the counter increases.

Finally, the method returns the total number of valid pairs.

## Go Solution

```go
func countPairs(nums []int, k int) int {
	valueToIndices := make(map[int][]int)

	for index, value := range nums {
		valueToIndices[value] = append(valueToIndices[value], index)
	}

	count := 0

	for _, indices := range valueToIndices {
		length := len(indices)

		for i := 0; i < length; i++ {
			for j := i + 1; j < length; j++ {
				if (indices[i]*indices[j])%k == 0 {
					count++
				}
			}
		}
	}

	return count
}
```

The Go solution follows the same logic as the Python version.

A `map[int][]int` is used instead of Python's `defaultdict(list)`. When appending to a non-existent key in Go, the zero value for a slice is `nil`, and `append` works correctly on it.

Integer overflow is not a concern because indices are at most `99`, so the largest possible product is `99 * 99 = 9801`.

The rest of the implementation mirrors the Python approach closely.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | In the worst case, all elements are equal, requiring comparison of all index pairs |
| Space | O(n) | The hash map stores all indices |

The preprocessing step that builds the hash map takes `O(n)` time. The dominant cost comes from comparing index pairs within each group.

In the worst case, every element is identical, so we compare all possible pairs, resulting in `O(n²)` time complexity.

The extra space usage comes from storing indices in the hash map, which requires `O(n)` additional memory.

## Edge Cases

### Arrays With No Duplicate Values

If every value appears exactly once, no valid pair can exist because the condition `nums[i] == nums[j]` can never be satisfied.

A naive implementation might still waste time checking divisibility for every pair. The grouping approach naturally handles this because each group contains only one index, so no comparisons are performed.

### Pairs Involving Index 0

Since `0 * j = 0`, every pair involving index `0` automatically satisfies the divisibility requirement for any positive `k`.

This can easily be overlooked when reasoning about divisibility. The implementation correctly handles this because it directly evaluates `(i * j) % k == 0`, and `0 % k` is always `0`.

### Large Values of k

When `k` is large relative to possible index products, many pairs fail the divisibility test.

For example, with `n = 5`, the largest possible product is `3 * 4 = 12`. If `k = 100`, almost no pairs qualify unless one index is `0`.

The implementation handles this safely because it performs the exact divisibility check for every candidate pair without relying on assumptions about the size of `k`.
