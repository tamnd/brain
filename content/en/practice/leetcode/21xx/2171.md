## Solution

## Problem Understanding

The problem presents an array of positive integers `beans`, each representing a bag containing some number of magic beans. You are allowed to remove any number of beans from each bag (including zero) with the goal that every remaining **non-empty bag** contains the same number of beans. Once beans are removed, they cannot be redistributed. Your task is to determine the **minimum total number of beans that must be removed** to achieve this state.

The input array `beans` can contain up to $10^5$ elements, and each element can be as large as $10^5$. The output is a single integer representing the minimal total beans removed.

Key observations include the fact that the solution requires either reducing larger bags to match smaller bags or potentially emptying smaller bags entirely. The constraints suggest that an $O(n \log n)$ solution is feasible, but a naive $O(n^2)$ approach iterating through every possible target number of beans would be too slow.

Important edge cases include arrays where all elements are already equal, arrays with a single bag, and arrays with large numbers of bags with widely varying quantities of beans.

## Approaches

The **brute-force approach** involves considering every possible number of beans in a bag as a potential target value. For each target, you would calculate the total number of beans to remove by either reducing larger bags or emptying smaller bags. This approach is correct but too slow because the number of potential target values could be as large as $10^5$, leading to $O(n^2)$ complexity.

The **optimal approach** leverages **sorting and prefix sums**. By sorting the array in ascending order, we can consider each unique bean count as a potential target. For a given target at index `i`, all bags before index `i` must be emptied, and all bags from index `i` to the end must be reduced to the target value. Using a prefix sum of the sorted array allows computing the total beans removed in $O(1)$ per target after sorting.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Try every possible target value and calculate removals |
| Optimal | O(n log n) | O(n) | Sort and use prefix sums to compute removals efficiently |

## Algorithm Walkthrough

1. **Sort the array** `beans` in ascending order. Sorting allows us to consider potential target values in increasing order and guarantees that all bags to the left of a target will need to be emptied, while bags to the right will only need reduction.
2. **Compute the prefix sum** array of the sorted beans. This enables efficient calculation of the total beans in any prefix or suffix of the array.
3. **Initialize a variable** `min_removal` to track the minimum beans removed across all potential targets.
4. **Iterate over the sorted array** using index `i` as the candidate target bean count. For index `i`:

- Beans in bags before `i` are entirely removed: `prefix_sum[i]`.
- Beans in bags from `i` onward are reduced to the target: `(n - i) * beans[i]`. The beans removed from these bags are `suffix_sum[i] - (n - i) * beans[i]`, which can be derived using the total sum of the array and prefix sums.
- The total removal for this target is `prefix_sum[i] + (total_sum - prefix_sum[i] - (n - i) * beans[i])`.
5. **Update `min_removal`** if the calculated removal is smaller than the current value.
6. **Return `min_removal`** after iterating through all indices.

This works because at each step, the algorithm calculates the minimal removal required if the target bean count is set to `beans[i]`. Sorting ensures all lower values are handled optimally by emptying the bags and higher values are handled optimally by reduction.

## Python Solution

```python
from typing import List

class Solution:
    def minimumRemoval(self, beans: List[int]) -> int:
        beans.sort()
        total_sum = sum(beans)
        n = len(beans)
        min_removal = float('inf')
        
        prefix_sum = 0
        for i in range(n):
            # Beans to remove from the left (all emptied)
            left_removal = prefix_sum
            # Beans to remove from the right (reduce to beans[i])
            right_removal = total_sum - prefix_sum - (n - i) * beans[i]
            min_removal = min(min_removal, left_removal + right_removal)
            prefix_sum += beans[i]
        
        return min_removal
```

In this implementation, `beans.sort()` ensures the candidate targets are in ascending order. `prefix_sum` accumulates the total beans of all processed bags. At each index, `left_removal` accounts for bags emptied before the target, and `right_removal` accounts for reduction of remaining bags to the target. The sum of these two gives total beans removed for that target.

## Go Solution

```go
import "sort"

func minimumRemoval(beans []int) int64 {
    sort.Ints(beans)
    n := len(beans)
    var totalSum int64
    for _, b := range beans {
        totalSum += int64(b)
    }

    var minRemoval int64 = totalSum
    var prefixSum int64 = 0
    for i := 0; i < n; i++ {
        leftRemoval := prefixSum
        rightRemoval := totalSum - prefixSum - int64(n-i)*int64(beans[i])
        if leftRemoval+rightRemoval < minRemoval {
            minRemoval = leftRemoval + rightRemoval
        }
        prefixSum += int64(beans[i])
    }
    return minRemoval
}
```

The Go implementation mirrors the Python solution. `int64` is used to avoid integer overflow since sums could exceed `int`. Sorting and prefix sum logic are identical, and the algorithm iterates over all candidate targets to compute minimal removal.
