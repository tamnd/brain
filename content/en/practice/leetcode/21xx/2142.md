## Solution

## Problem Understanding

This problem involves two tables, `Buses` and `Passengers`, that record the arrival times of buses and passengers at a station. Each bus has a unique `bus_id` and arrives at a specific `arrival_time`. Similarly, each passenger has a unique `passenger_id` and arrives at a specific `arrival_time`. The goal is to determine how many passengers board each bus, given the rule that a passenger boards the first bus that arrives at or after their arrival time, and each passenger can only board once. The output is a table of `bus_id` and the corresponding count of passengers, sorted by `bus_id` in ascending order.

Important constraints are: no two buses arrive at the same time, and each passenger can only take a bus once. Edge cases include situations where no passengers board a bus, all passengers arrive before the first bus, or all passengers arrive after the last bus.

The challenge lies in efficiently matching passengers to buses based on arrival times, rather than performing a naive pairwise comparison which could be slow if the tables are large.

## Approaches

The brute-force approach would compare every passenger with every bus, checking if the passenger arrived before the bus and had not yet boarded another bus. This guarantees correctness but has a time complexity of $O(B \times P)$, where $B$ is the number of buses and $P$ is the number of passengers, which becomes inefficient for large datasets.

The key insight for an optimal solution is that we can sort both buses and passengers by arrival times. Once sorted, we can use a pointer to traverse passengers and another for buses, incrementing counts as we assign passengers to buses. This ensures each passenger is considered exactly once and each bus only processes relevant passengers, reducing time complexity dramatically.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(B * P) | O(1) | Compare each passenger with each bus and count matches |
| Optimal | O(B log B + P log P) | O(B) | Sort both tables and use a two-pointer traversal to assign passengers to buses |

## Algorithm Walkthrough

1. **Sort buses and passengers** by `arrival_time` in ascending order. Sorting ensures we can traverse them sequentially and assign passengers to the earliest available bus.
2. **Initialize a pointer for passengers** at the first passenger in the sorted list. Also, initialize a dictionary to store passenger counts for each bus.
3. **Traverse the sorted buses** one by one. For each bus, start counting passengers from the current passenger pointer.
4. **Assign passengers to the current bus** if their arrival time is less than or equal to the bus’s arrival time. For each assignment, increment the bus’s passenger count and move the passenger pointer to the next passenger.
5. **Store the passenger count** for the current bus in the dictionary or temporary table.
6. After processing all buses, **return the passenger counts** ordered by `bus_id` in ascending order.

**Why it works:** Sorting ensures that each passenger is considered for the earliest bus they can take, and the two-pointer approach guarantees each passenger is assigned to exactly one bus, while each bus only receives passengers that arrived before or at its arrival time.

## Python Solution

```python
from typing import List
import pandas as pd

class Solution:
    def countPassengers(self, Buses: pd.DataFrame, Passengers: pd.DataFrame) -> pd.DataFrame:
        buses_sorted = Buses.sort_values('arrival_time')
        passengers_sorted = Passengers.sort_values('arrival_time')
        
        passenger_index = 0
        bus_passengers = {}
        passengers_list = passengers_sorted['arrival_time'].tolist()
        num_passengers = len(passengers_list)
        
        for _, bus_row in buses_sorted.iterrows():
            bus_id = bus_row['bus_id']
            bus_time = bus_row['arrival_time']
            count = 0
            
            while passenger_index < num_passengers and passengers_list[passenger_index] <= bus_time:
                count += 1
                passenger_index += 1
            
            bus_passengers[bus_id] = count
        
        result = pd.DataFrame({
            'bus_id': list(bus_passengers.keys()),
            'passengers_cnt': list(bus_passengers.values())
        }).sort_values('bus_id')
        
        return result
```

The Python implementation first sorts both dataframes by `arrival_time`. It then iterates over buses, using a pointer to assign passengers sequentially. The dictionary `bus_passengers` temporarily stores the passenger count for each bus, which is finally converted into a DataFrame sorted by `bus_id`.

## Go Solution

```go
package main

import (
	"sort"
)

type Bus struct {
	BusID       int
	ArrivalTime int
}

type Passenger struct {
	PassengerID int
	ArrivalTime int
}

type Result struct {
	BusID          int
	PassengersCnt  int
}

func countPassengers(buses []Bus, passengers []Passenger) []Result {
	sort.Slice(buses, func(i, j int) bool {
		return buses[i].ArrivalTime < buses[j].ArrivalTime
	})
	sort.Slice(passengers, func(i, j int) bool {
		return passengers[i].ArrivalTime < passengers[j].ArrivalTime
	})

	passengerIndex := 0
	numPassengers := len(passengers)
	results := make([]Result, 0, len(buses))

	for _, bus := range buses {
		count := 0
		for passengerIndex < numPassengers && passengers[passengerIndex].ArrivalTime <= bus.ArrivalTime {
			count++
			passengerIndex++
		}
		results = append(results, Result{BusID: bus.BusID, PassengersCnt: count})
	}

	sort.Slice(results, func(i, j int) bool {
		return results[i].BusID < results[j].BusID
	})

	return results
}
```

In Go, slices are sorted using `sort.Slice`. The two-pointer approach is implemented by incrementing `passengerIndex` as passengers are assigned. The result slice is sorted by `BusID` before returning. Go handles empty slices gracefully, and integer overflow is not an issue here given problem constraints.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(B log B + P log P) | Sorting both buses and passengers dominates the runtime. The traversal is O(B + P). |
| Space | O(B) | Stores passenger count for each bus and temporary arrays from sorting. |

Sorting ensures a sequential assignment, which avoids nested iteration. Space is linear in the number of buses to store results.
