## Solution

## Problem Understanding

The problem gives us an integer array `nums` and an integer `k`. We must count how many index pairs `(i, j)` satisfy two conditions:

- `i < j`
- `nums[i] * nums[j]` is divisible by `k`

In other words, for every pair of different positions in the array, we check whether the product of the two numbers contains all the prime factors necessary to divide evenly by `k`.

The input array can contain up to `10^5` elements, and both the array values and `k` can also be as large as `10^5`. These constraints immediately tell us that checking every possible pair directly will not work efficiently. A brute-force solution would require examining approximately `n^2 / 2` pairs, which becomes far too slow when `n = 100000`.

The important observation is that divisibility depends on factor composition, not on the exact product value itself. Instead of multiplying every pair and testing divisibility directly, we can reason about how much of `k` each number contributes.

Several edge cases are important:

- If `k = 1`, every product is divisible by `1`, so the answer is simply the number of pairs.
- If no number contributes any useful factors of `k`, the answer may be `0`.
- Duplicate numbers can create many valid pairs, so counting efficiently matters.
- Large values require careful handling to avoid quadratic behavior.
- The result can exceed 32-bit integer range, especially when many pairs are valid, so the Go solution must use `int64`.

## Approaches

### Brute Force

The most direct solution is to examine every pair `(i, j)` where `i < j`. For each pair, compute:

```
nums[i] * nums[j]
```

and check whether:

```
(nums[i] * nums[j]) % k == 0
```

If the condition holds, increment the answer.

This approach is correct because it directly follows the problem statement. Every possible pair is checked exactly once.

However, the array can contain up to `100000` elements. The number of pairs is:

```
100000 * 99999 / 2 ≈ 5 * 10^9
```

which is far too large to process within time limits.

### Optimal Approach

The key insight is that we do not care about the exact values of the numbers. We only care about how their factors relate to `k`.

Suppose we define:

```
g = gcd(nums[i], k)
```

This value tells us which factors of `k` are already provided by `nums[i]`.

For another number `nums[j]`, let:

```
h = gcd(nums[j], k)
```

The pair is valid if:

```
(g * h) % k == 0
```

Why does this work?

Because `g` and `h` represent exactly the parts of each number that overlap with `k`. If their combined factors contain all prime factors required by `k`, then the original product will also be divisible by `k`.

This transforms the problem into counting compatible GCD values rather than checking full products.

We process the array from left to right while maintaining a frequency map of previously seen GCD values.

For each new number:

1. Compute its gcd with `k`
2. Compare it against all previously seen gcd values
3. Count how many satisfy the divisibility condition
4. Store the current gcd in the map

Since the number of divisors of `k` is relatively small, this approach is efficient enough.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks every pair directly |
| Optimal | O(n × d) | O(d) | Uses gcd properties, where `d` is the number of distinct gcd values |

## Algorithm Walkthrough

1. Initialize a hash map called `gcd_count`.

This map stores how many previous numbers produced each gcd value with `k`.

For example:

```
gcd_count[2] = 5
```

means we have already seen 5 numbers whose gcd with `k` equals 2.
2. Initialize the answer variable to `0`.

This will accumulate the number of valid pairs.
3. Iterate through each number in `nums`.

For every number `num`, compute:

```
current_gcd = gcd(num, k)
```

This extracts only the factors relevant to divisibility by `k`.
4. Compare the current gcd against all previously seen gcd values.

For each stored gcd value `prev_gcd`:

```
if (current_gcd * prev_gcd) % k == 0
```

then every number associated with `prev_gcd` forms a valid pair with the current number.
5. Add the matching frequency to the answer.

If:

```
gcd_count[prev_gcd] = count
```

then we add `count` valid pairs.
6. Store the current gcd in the map.

Increment:

```
gcd_count[current_gcd]
```

so future numbers can pair with it.
7. Continue until all numbers have been processed.
8. Return the final answer.

### Why it works

For any number `a`, the value `gcd(a, k)` captures exactly the factors of `k` contributed by `a`.

Suppose:

```
g1 = gcd(a, k)
g2 = gcd(b, k)
```

If:

```
(g1 * g2) % k == 0
```

then together they contain all prime factors required by `k`, meaning:

```
(a * b) % k == 0
```

Therefore, counting compatible gcd pairs is equivalent to counting valid array pairs.

## Python Solution

```python
from typing import List
from collections import defaultdict
from math import gcd

class Solution:
    def countPairs(self, nums: List[int], k: int) -> int:
        gcd_count = defaultdict(int)
        answer = 0

        for num in nums:
            current_gcd = gcd(num, k)

            for prev_gcd, frequency in gcd_count.items():
                if (current_gcd * prev_gcd) % k == 0:
                    answer += frequency

            gcd_count[current_gcd] += 1

        return answer
```

The implementation follows the algorithm directly.

We first create a `defaultdict(int)` so that missing gcd values automatically start with frequency `0`.

For every number in the array, we compute `gcd(num, k)`. This reduces the problem from handling arbitrary integers to handling only divisor relationships with `k`.

We then iterate through all previously seen gcd values stored in the hash map. Whenever the product of the two gcd values is divisible by `k`, all numbers associated with that gcd form valid pairs with the current number.

Finally, we insert the current gcd into the map so future elements can pair with it.

Because we only store gcd values relative to `k`, the number of distinct keys remains manageable.

## Go Solution

```go
package main

import "math"

func gcd(a int, b int) int {
	for b != 0 {
		a, b = b, a%b
	}
	return a
}

func countPairs(nums []int, k int) int64 {
	gcdCount := make(map[int]int64)
	var answer int64 = 0

	for _, num := range nums {
		currentGCD := gcd(num, k)

		for prevGCD, frequency := range gcdCount {
			if (currentGCD*prevGCD)%k == 0 {
				answer += frequency
			}
		}

		gcdCount[currentGCD]++
	}

	return answer
}

func main() {
	_ = math.MaxInt
}
```

The Go implementation mirrors the Python solution closely.

The main difference is that the answer uses `int64`, because the number of valid pairs can exceed the 32-bit integer range.

Go does not provide a built-in gcd function in older standard library versions, so we implement Euclid's algorithm manually.

The frequency map uses:

```go
map[int]int64
```

so counts remain compatible with the `int64` answer.

## Worked Examples

### Example 1

```
nums = [1,2,3,4,5]
k = 2
```

We process numbers one by one.

| Step | num | gcd(num,2) | Existing gcd map | Valid additions | Total |
| --- | --- | --- | --- | --- | --- |
| 1 | 1 | 1 | {} | 0 | 0 |
| 2 | 2 | 2 | {1:1} | 1 | 1 |
| 3 | 3 | 1 | {1:1,2:1} | 1 | 2 |
| 4 | 4 | 2 | {1:2,2:1} | 3 | 5 |
| 5 | 5 | 1 | {1:2,2:2} | 2 | 7 |

Detailed reasoning:

- `1` contributes gcd `1`
- `2` contributes gcd `2`

- `(2 * 1) % 2 == 0`, valid pair count increases by 1
- `3` contributes gcd `1`

- `(1 * 2) % 2 == 0`, another valid pair
- `4` contributes gcd `2`

- Pairs with all previous gcd values
- `5` contributes gcd `1`

- Only pairs with previous gcd `2`

Final answer:

```
7
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × d) | `d` is the number of distinct gcd values |
| Space | O(d) | Stores frequencies of gcd values |

The crucial observation is that every stored gcd must be a divisor of `k`. The number of divisors of `k` is relatively small compared to `n`, especially since `k <= 100000`.

Therefore, although we iterate through the map for every number, the map size remains bounded by the divisor count of `k`, making the solution efficient in practice.

## Edge Cases

### Case 1: `k = 1`

When `k` equals `1`, every integer is divisible by `1`. Therefore, every possible pair is valid.

This can easily expose bugs if the implementation incorrectly assumes divisibility checks are meaningful for all values of `k`. In this solution, the gcd of every number with `1` is also `1`, and:

```
(1 * 1) % 1 == 0
```

so all pairs are counted naturally.

### Case 2: No Valid Pairs

Consider:

```
nums = [1,1,1]
k = 10
```

Every gcd value equals `1`, and:

```
(1 * 1) % 10 != 0
```

A naive implementation might accidentally count pairs due to incorrect factor logic. This solution correctly rejects all pairs because the gcd compatibility condition fails.

### Case 3: Large Duplicate Counts

Suppose:

```
nums = [100000] * 100000
k = 100000
```

Almost every pair is valid, producing a very large answer.

This case is dangerous because the number of pairs exceeds 32-bit integer limits. The Go implementation uses `int64` to avoid overflow, while Python integers automatically expand as needed.

### Case 4: Complementary Factors

Consider:

```
nums = [6,10,15]
k = 30
```

No single number is divisible by `30`, but pairwise products are:

```
6 * 10 = 60
6 * 15 = 90
10 * 15 = 150
```

All divisible by `30`.

This demonstrates why checking only whether individual numbers are divisible by `k` would fail. The gcd-based approach correctly captures complementary factor contributions across pairs.
