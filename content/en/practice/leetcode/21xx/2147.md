**Difficulty:** 🔴 Hard  
**Topics:** Math, String, Dynamic Programming

## Solution

## Problem Understanding

The corridor is represented as a string containing two kinds of characters:

- `'S'` represents a seat
- `'P'` represents a plant

The corridor is already enclosed by dividers at both ends. We may optionally place additional dividers between adjacent characters. The goal is to split the corridor into sections such that every section contains exactly two seats.

Plants do not affect validity. A section may contain any number of plants, including zero. The only requirement is that each section contains exactly two `'S'` characters.

The task is to count how many different valid ways exist to place dividers. Two arrangements are considered different if at least one divider position differs.

The answer can become very large, so we return it modulo:

$10^9 + 7$

The constraint `n <= 10^5` is extremely important. It tells us that exponential or quadratic solutions are not acceptable. We need a linear time solution.

There are several important observations and edge cases:

- If the total number of seats is odd, it is impossible to divide the corridor into groups of exactly two seats.
- If there are zero seats, there is also no valid division.
- If there are exactly two seats total, there is exactly one valid configuration, which is to place no additional dividers.
- Plants between seat pairs create multiple divider choices, and this becomes the key combinational insight.

For example, consider:

```
SSPPSS
```

The first section must contain the first two seats, and the second section must contain the next two seats. The divider separating the sections can be placed in any position between those seat groups. Every plant between the groups increases the number of choices.

## Approaches

### Brute Force Approach

A brute force solution would try every possible combination of divider placements.

There are `n - 1` possible divider positions, and each position can either contain a divider or not. That means there are:

$2^{n-1}$

possible divider configurations.

For each configuration, we could scan the corridor and verify whether every section contains exactly two seats.

This approach is correct because it exhaustively checks every possible divider arrangement. However, it is completely infeasible for `n = 10^5`. Even for modest values of `n`, the number of configurations becomes astronomically large.

### Key Insight

The important observation is that the divider locations are not independent.

Once we group seats into consecutive pairs, the only places where we have freedom are between one completed seat pair and the next pair.

Suppose we have:

```
SSPPPSS
```

The first pair of seats is the first two `'S'` characters. The second pair is the next two `'S'` characters.

Between these seat groups, there are several valid divider positions:

```
SS|PPPSS
SSP|PPSS
SSPP|PSS
SSPPP|SS
```

If there are `k` plants between two seat groups, then there are:

$k+1$

possible divider placements.

The total number of valid arrangements becomes the product of all these choices.

This leads to a linear time greedy counting solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Tries every divider configuration |
| Optimal | O(n) | O(1) | Counts divider choices between seat pairs |

## Algorithm Walkthrough

1. First, count the total number of seats in the corridor.

If the number of seats is zero or odd, immediately return `0`. A valid partition requires every section to contain exactly two seats, so the total seat count must be a positive even number.
2. Initialize the answer as `1`.

We will multiply the number of choices between consecutive seat groups.
3. Traverse the corridor from left to right while counting seats.

Every time we complete a group of two seats, we begin looking for the next seat group.
4. After finishing one seat pair, count how many plants appear before the next seat.

For example:

```
SSPPPSS
  ^^^
```

There are three plants between the seat groups.
5. The divider can be placed in any gap among these plants.

If there are `k` plants, then there are `k + 1` divider positions.

Multiply the running answer by this value.
6. Continue processing until the entire corridor has been scanned.
7. Return the result modulo:

$10^9 + 7$

### Why it works

Every valid section boundary must occur between two consecutive seat pairs. The plants between those seat groups determine how many legal divider positions exist.

These choices are independent. Choosing one divider position does not affect the valid choices for other seat pair boundaries.

Therefore, the total number of valid arrangements is the product of the number of choices at every boundary.

## Python Solution

```python
class Solution:
    def numberOfWays(self, corridor: str) -> int:
        MOD = 10**9 + 7

        total_seats = corridor.count('S')

        if total_seats == 0 or total_seats % 2 == 1:
            return 0

        seat_count = 0
        plants_between = 0
        ways = 1

        for char in corridor:
            if char == 'S':
                seat_count += 1

                # Start of a new seat pair after previous completed pair
                if seat_count > 2 and seat_count % 2 == 1:
                    ways = (ways * (plants_between + 1)) % MOD

                plants_between = 0
            else:
                # Count plants only after completing a seat pair
                if seat_count % 2 == 0 and seat_count > 0:
                    plants_between += 1

        return ways
```

The implementation begins by counting the total number of seats. This early validation is important because no arrangement can work unless the total number of seats is both positive and even.

The variable `seat_count` tracks how many seats have been encountered while scanning the corridor.

The variable `plants_between` counts plants that occur after completing one seat pair and before encountering the next seat. These plants determine how many divider positions are available.

Whenever we encounter the first seat of a new pair after already completing a previous pair, we multiply the answer by:

$plants\_between + 1$

This corresponds exactly to the number of valid divider locations between the seat groups.

The algorithm processes the string once, uses constant extra memory, and maintains the result modulo `10^9 + 7`.

## Go Solution

```go
func numberOfWays(corridor string) int {
    const MOD int = 1_000_000_007

    totalSeats := 0
    for _, ch := range corridor {
        if ch == 'S' {
            totalSeats++
        }
    }

    if totalSeats == 0 || totalSeats%2 == 1 {
        return 0
    }

    seatCount := 0
    plantsBetween := 0
    ways := 1

    for _, ch := range corridor {
        if ch == 'S' {
            seatCount++

            if seatCount > 2 && seatCount%2 == 1 {
                ways = (ways * (plantsBetween + 1)) % MOD
            }

            plantsBetween = 0
        } else {
            if seatCount > 0 && seatCount%2 == 0 {
                plantsBetween++
            }
        }
    }

    return ways
}
```

The Go implementation follows exactly the same logic as the Python version.

One important detail is that Go does not have Python's arbitrary precision integers. However, the modulo operation is applied after every multiplication, so the values remain safely bounded.

The solution uses integer variables only, and no additional slices or maps are needed.

## Edge Cases

One important edge case occurs when the corridor contains no seats at all, such as `"PPPP"`. A naive implementation might incorrectly return `1` because no dividers are needed. However, every section must contain exactly two seats, and zero seats cannot satisfy this condition. The implementation explicitly checks for `total_seats == 0` and returns `0`.

Another important case is when the total number of seats is odd. For example:

```
"SSS"
```

No matter how dividers are placed, one section would eventually contain only one seat. The implementation handles this immediately with the parity check:

```
if total_seats % 2 == 1:
    return 0
```

A subtle edge case appears when consecutive seat groups have no plants between them, such as:

```
"SSSS"
```

There is exactly one valid divider location, between the second and third seat. The implementation correctly computes:

$0 + 1 = 1$

possible divider placement.

Another tricky scenario involves long stretches of plants:

```
"SSPPPPPPSS"
```

There are seven valid divider positions between the seat groups. A buggy implementation might accidentally count plants before the first pair or after the final pair. This solution only counts plants that occur after completing a seat pair and before the next seat group begins, which guarantees correctness.
