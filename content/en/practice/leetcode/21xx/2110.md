## Solution

## Problem Understanding

This problem asks us to count all contiguous periods in a stock price history where each subsequent day in the period decreases **exactly by 1** compared to the previous day. The input is an array `prices` where `prices[i]` is the stock price on day `i`. A period of length one (a single day) is also considered a smooth descent period, because there is no previous day to violate the rule. The output is a single integer representing the total number of smooth descent periods in the array.

Constraints tell us that the array can be very large, up to 100,000 elements, and each price can be up to 100,000. This means a naive solution that checks every possible subarray is likely too slow. Edge cases include arrays with only one element, arrays with all identical prices, or arrays that are strictly decreasing or increasing by values other than 1. The problem guarantees that the input array has at least one element and all prices are positive integers.

## Approaches

The brute-force approach would examine every contiguous subarray and check if it forms a smooth descent period. This guarantees correctness because it considers all possible periods. However, for an array of length `n`, there are `O(n^2)` subarrays, making this approach impractical for `n` up to 100,000.

The optimal approach relies on the key observation that if you know the length of a descending segment ending at a particular day, you can directly count all smooth descent periods ending at that day using a simple arithmetic series formula. Specifically, if a segment of length `k` ends at day `i`, it contributes `k` new periods ending at `i`. You can keep a running count as you iterate through the array, which reduces the time complexity to linear, `O(n)`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check every contiguous subarray, slow for large n |
| Optimal | O(n) | O(1) | Use running length of smooth descent periods and sum arithmetic series incrementally |

## Algorithm Walkthrough

1. Initialize a variable `total_periods` to 0 to store the cumulative count of smooth descent periods.
2. Initialize a variable `current_length` to 1 because a single day always forms a descent period.
3. Iterate through the array from the second element to the end.
4. At each day `i`, check if `prices[i-1] - prices[i]` equals 1. If true, increment `current_length` by 1. This extends the current smooth descent segment.
5. If the difference is not 1, reset `current_length` to 1 because a new descent period starts at the current day.
6. Add `current_length` to `total_periods` at each step. This counts all descent periods ending at the current day, including those of length 1, 2, ..., `current_length`.
7. After the loop, `total_periods` contains the total number of smooth descent periods.

**Why it works**: The invariant is that `current_length` always represents the length of the descent segment ending at the current day. By adding it to `total_periods` at each step, we count all valid subarrays ending at that day without double counting or missing any. This guarantees correctness.

## Python Solution

```python
from typing import List

class Solution:
    def getDescentPeriods(self, prices: List[int]) -> int:
        total_periods = 0
        current_length = 1
        
        for i in range(1, len(prices)):
            if prices[i-1] - prices[i] == 1:
                current_length += 1
            else:
                current_length = 1
            total_periods += current_length
        
        return total_periods
```

**Implementation Explanation**: We initialize `total_periods` and `current_length`. Iterating from the second element, we extend or reset the descent segment based on the exact difference of 1. At each step, we add `current_length` to `total_periods` to count all periods ending at the current day. This directly implements the arithmetic series counting logic described in the algorithm walkthrough.

## Go Solution

```go
func getDescentPeriods(prices []int) int64 {
    var totalPeriods int64 = 1
    currentLength := int64(1)
    
    for i := 1; i < len(prices); i++ {
        if prices[i-1]-prices[i] == 1 {
            currentLength++
        } else {
            currentLength = 1
        }
        totalPeriods += currentLength
    }
    
    return totalPeriods
}
```

**Go-specific Notes**: We use `int64` to handle potentially large totals. Slices in Go are 0-indexed like arrays in Python. We initialize `totalPeriods` to 1 for the first day, since a single-element period is always counted.

## Edge Cases

The first important edge case is an array with **all identical prices**, such as `[7,7,7,7]`. Here, no smooth descent periods longer than one exist, so the implementation must correctly reset `current_length` to 1 for every day to avoid counting invalid segments.

The second edge case is a **strictly decreasing sequence**, for example `[5,4,3,2,1]`. Every day extends the previous period, and the sum of all subarrays of lengths 1 to n must be correctly calculated using the running `current_length`.

The third edge case is the **minimum input size**, which is a single-element array `[x]`. The implementation must handle this without indexing errors and return 1 because a single day counts as a descent period by definition.

These cases ensure that both segment resets and extensions, as well as boundary conditions, are correctly handled by the solution.
