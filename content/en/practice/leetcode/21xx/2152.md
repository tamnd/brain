## Solution

## Problem Understanding

The problem asks us to find the minimum number of straight lines required to cover all given points on a 2D plane. Each point is represented by its `(x, y)` coordinates in the input array `points`. A line can pass through any number of points as long as they are collinear. The goal is to determine the smallest set of such lines that collectively cover every point at least once.

The constraints tell us that the number of points is small, at most 10, and all points are unique with coordinates between -100 and 100. This small input size is important because it allows us to consider combinatorial or exhaustive search techniques without worrying about performance. The key edge cases are scenarios where points are all collinear (only one line is needed), points are in such a configuration that many lines are necessary, or when there are only one or two points.

## Approaches

The brute-force approach would be to try every possible combination of lines connecting points and count the minimum number required to cover all points. Since the number of ways to choose lines grows exponentially with the number of points, this approach becomes infeasible for more than a few points, even though `points.length <= 10` makes it theoretically possible.

The key insight for an optimal approach is that, because the total number of points is small, we can use **bitmask dynamic programming**. We can represent the set of uncovered points as a bitmask. For every subset of points that are collinear, we can try covering them with one line and recursively compute the minimum number of additional lines needed for the remaining points. Memoization allows us to avoid recomputing the same state multiple times.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n^3) | O(n) | Enumerate all subsets and lines, check collinearity by choosing any two points |
| Optimal (Bitmask DP) | O(n^2 * 2^n) | O(2^n) | Precompute all collinear subsets and use bitmask DP to minimize number of lines |

## Algorithm Walkthrough

1. Precompute all subsets of points that are collinear. For each pair of points `(i, j)`, compute the line they define and find all other points that lie on the same line.
2. Encode each subset of collinear points as a bitmask. This allows us to represent which points are covered by a particular line efficiently.
3. Use a recursive dynamic programming function `dp(mask)` where `mask` represents the set of points already covered. If `mask` equals `(1 << n) - 1`, all points are covered, and the function returns 0.
4. For each line subset represented as a bitmask `line_mask`, if it covers any uncovered points, recursively compute `1 + dp(mask | line_mask)`. Take the minimum over all valid `line_mask`.
5. Memoize the results of `dp(mask)` to avoid redundant computation.

Why it works: The invariant is that the recursive function always computes the minimum number of lines needed for the uncovered points represented by `mask`. By considering all possible lines at each step and taking the minimum, the algorithm guarantees an optimal solution.

## Python Solution

```python
from typing import List
from functools import lru_cache
from itertools import combinations

class Solution:
    def minimumLines(self, points: List[List[int]]) -> int:
        n = len(points)
        
        if n <= 2:
            return 1
        
        # Function to check collinearity using cross multiplication
        def collinear(p1, p2, p3):
            return (p2[1] - p1[1]) * (p3[0] - p1[0]) == (p3[1] - p1[1]) * (p2[0] - p1[0])
        
        # Precompute all collinear subsets
        line_masks = []
        for i in range(n):
            for j in range(i + 1, n):
                mask = (1 << i) | (1 << j)
                for k in range(n):
                    if k != i and k != j and collinear(points[i], points[j], points[k]):
                        mask |= (1 << k)
                line_masks.append(mask)
        
        full_mask = (1 << n) - 1
        
        @lru_cache(None)
        def dp(mask):
            if mask == full_mask:
                return 0
            ans = float('inf')
            for line_mask in line_masks:
                if line_mask & mask != line_mask:  # covers at least one new point
                    ans = min(ans, 1 + dp(mask | line_mask))
            return ans
        
        return dp(0)
```

The code first handles the trivial case of one or two points, which always require one line. The `collinear` function uses cross multiplication to avoid floating-point errors when checking if three points lie on the same line. All line subsets are precomputed and encoded as bitmasks. The dynamic programming function uses memoization and bitmask manipulation to efficiently compute the minimum number of lines needed.

## Go Solution

```go
package main

func minimumLines(points [][]int) int {
    n := len(points)
    if n <= 2 {
        return 1
    }

    collinear := func(p1, p2, p3 []int) bool {
        return (p2[1]-p1[1])*(p3[0]-p1[0]) == (p3[1]-p1[1])*(p2[0]-p1[0])
    }

    lineMasks := []int{}
    for i := 0; i < n; i++ {
        for j := i + 1; j < n; j++ {
            mask := (1 << i) | (1 << j)
            for k := 0; k < n; k++ {
                if k != i && k != j && collinear(points[i], points[j], points[k]) {
                    mask |= (1 << k)
                }
            }
            lineMasks = append(lineMasks, mask)
        }
    }

    fullMask := (1 << n) - 1
    memo := map[int]int{}

    var dp func(mask int) int
    dp = func(mask int) int {
        if mask == fullMask {
            return 0
        }
        if val, exists := memo[mask]; exists {
            return val
        }
        ans := n
        for _, lineMask := range lineMasks {
            if lineMask&mask != lineMask {
                ansCandidate := 1 + dp(mask|lineMask)
                if ansCandidate < ans {
                    ans = ansCandidate
                }
            }
        }
        memo[mask] = ans
        return ans
    }

    return dp(0)
}
```

The Go implementation mirrors the Python version closely. Bitmask operations and recursion are used in the same way. Go maps are used for memoization instead of `lru_cache`. Integer operations are safe since the maximum number of points is small.

## Worked Examples

### Example 1: `[[0,1],[2,3],[4,5],[4,3]]`

Initial bitmask: `0000` (no points covered)

Line masks computed:

- Line through `[0,1],[2,3],[4,5]` → `0111`
- Line through `[2,3],[4,3]` → `1100`

Recursive computation:

1. Start with mask `0000`
2. Choose line `0111` → mask becomes `0111`, dp(0111) = 1 + dp(0111 | 1100) = 2
3. Minimum lines = 2

### Example 2: `[ [0,2],[-2,-2],[1,4] ]`

All points collinear, one line mask: `111`

dp(0) → choose `111` → dp(111) = 0

Answer = 1
