## Solution

## Problem Understanding

The problem describes a scenario at a bus station where buses and passengers arrive at specific times. Each bus has a limited capacity, and passengers can only board a bus if they have already arrived at the station and have not taken another bus yet. The input consists of two tables: `Buses` and `Passengers`. The `Buses` table contains the unique bus identifier (`bus_id`), the arrival time of the bus (`arrival_time`), and its capacity (`capacity`). The `Passengers` table contains the unique passenger identifier (`passenger_id`) and the passenger's arrival time (`arrival_time`).

The goal is to determine, for each bus, how many passengers it collects, respecting the bus capacities and the order of arrival. The result should be a table with `bus_id` and `passengers_cnt`, sorted by `bus_id` in ascending order.

Key constraints include that no two buses arrive at the same time, all bus capacities are positive integers, and each passenger and bus has a unique identifier. Edge cases to consider include situations where a bus arrives before any passengers, where a bus's capacity is smaller than the number of waiting passengers, or where multiple passengers arrive at the same time.

## Approaches

A brute-force approach involves iterating over each bus and, for each bus, scanning all passengers to find those who arrived before the bus and have not yet boarded any bus. You would select passengers in the order of their arrival until the bus reaches its capacity. While this guarantees correct results, it is inefficient because it repeatedly scans the passengers for every bus, leading to a potential O(B * P) time complexity, where B is the number of buses and P is the number of passengers.

The key insight for an optimal solution is that both passengers and buses can be sorted by arrival time. Once sorted, a single pass can be used to assign passengers to buses, maintaining a pointer or index for the next unassigned passenger. This approach avoids redundant scans of the passengers for each bus, reducing the complexity significantly.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(B * P) | O(P) | For each bus, scan all passengers to assign them |
| Optimal | O(B log B + P log P) | O(P) | Sort buses and passengers by arrival time, then assign passengers in one pass |

## Algorithm Walkthrough

1. Sort the `Buses` table by `arrival_time` to process buses in chronological order.
2. Sort the `Passengers` table by `arrival_time` to ensure we can pick passengers in the order they arrive.
3. Initialize a variable `passenger_index` to 0 to keep track of the next unassigned passenger.
4. Initialize a results list to store each bus’s passenger count.
5. Iterate over the sorted buses. For each bus:

1. Count how many passengers have arrived by the bus’s arrival time and have not yet been assigned to another bus.
2. Assign passengers to the bus until either the bus reaches its capacity or there are no more eligible passengers.
3. Update `passenger_index` to point to the first unassigned passenger after the current bus.
4. Append the bus's passenger count and `bus_id` to the results list.
6. Return the results sorted by `bus_id` in ascending order.

Why it works: Sorting both tables by arrival time ensures that passengers are always assigned to the earliest bus they are eligible for, respecting both arrival order and bus capacity constraints. The invariant maintained is that `passenger_index` always points to the first unassigned passenger.

## Python Solution

```python
import pandas as pd

class Solution:
    def countPassengers(self, buses: pd.DataFrame, passengers: pd.DataFrame) -> pd.DataFrame:
        buses_sorted = buses.sort_values('arrival_time').reset_index(drop=True)
        passengers_sorted = passengers.sort_values('arrival_time').reset_index(drop=True)
        passenger_index = 0
        n_passengers = len(passengers_sorted)
        results = []

        for _, bus in buses_sorted.iterrows():
            bus_time = bus['arrival_time']
            capacity = bus['capacity']
            count = 0

            while passenger_index < n_passengers and passengers_sorted.loc[passenger_index, 'arrival_time'] <= bus_time and count < capacity:
                count += 1
                passenger_index += 1

            results.append({'bus_id': bus['bus_id'], 'passengers_cnt': count})

        return pd.DataFrame(results).sort_values('bus_id').reset_index(drop=True)
```

The Python implementation follows the algorithm steps directly. Sorting ensures passengers are considered in chronological order. The `passenger_index` prevents revisiting passengers already assigned to previous buses, and the `while` loop respects both the bus capacity and the arrival time condition. Finally, the results are sorted by `bus_id` for output compliance.

## Go Solution

```go
package main

import (
	"sort"
)

type Bus struct {
	BusID       int
	ArrivalTime int
	Capacity    int
}

type Passenger struct {
	PassengerID int
	ArrivalTime int
}

type Result struct {
	BusID         int
	PassengersCnt int
}

func countPassengers(buses []Bus, passengers []Passenger) []Result {
	sort.Slice(buses, func(i, j int) bool {
		return buses[i].ArrivalTime < buses[j].ArrivalTime
	})
	sort.Slice(passengers, func(i, j int) bool {
		return passengers[i].ArrivalTime < passengers[j].ArrivalTime
	})

	passengerIndex := 0
	nPassengers := len(passengers)
	results := make([]Result, 0, len(buses))

	for _, bus := range buses {
		count := 0
		for passengerIndex < nPassengers && passengers[passengerIndex].ArrivalTime <= bus.ArrivalTime && count < bus.Capacity {
			count++
			passengerIndex++
		}
		results = append(results, Result{BusID: bus.BusID, PassengersCnt: count})
	}

	sort.Slice(results, func(i, j int) bool {
		return results[i].BusID < results[j].BusID
	})

	return results
}
```

In Go, we use slices for buses and passengers. Sorting uses `sort.Slice` with a custom comparator. The `passengerIndex` tracks the first unassigned passenger. Results are stored in a slice of structs and finally sorted by `BusID` before returning.
