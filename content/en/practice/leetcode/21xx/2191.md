## Problem Understanding

This problem requires sorting an array of integers `nums` based not on their natural values, but on a **custom digit mapping** defined by another array `mapping` of length 10. Each digit `i` from 0 to 9 is mapped to a different digit `mapping[i]`. The **mapped value** of a number is the result of replacing each of its digits according to this mapping and interpreting the result as a regular integer.

The input `mapping` guarantees that every digit maps uniquely to another digit, so there are no collisions in mapping. The input array `nums` can contain up to 30,000 numbers, each up to just under one billion. The output should be the original numbers sorted in non-decreasing order **according to their mapped values**, but the numbers themselves are not changed, so we are only using the mapped values as a sorting key. Elements with identical mapped values should preserve their original relative order, meaning this is a **stable sort** problem.

Important edge cases include numbers that map to values with leading zeros (for example, mapping `1 -> 0`) and multiple numbers mapping to the same value. The problem guarantees `mapping` length is exactly 10, all values are unique, and `nums` are non-negative integers, simplifying input validation.

## Approaches

### Brute Force

The brute-force approach directly computes the mapped value for each number, then sorts the numbers based on these mapped values. This works correctly because it respects the problem's requirement: sorting by mapped values while maintaining stability. However, a naive implementation may involve converting numbers to strings repeatedly or performing repeated digit extraction, which can be inefficient for large arrays of large numbers.

### Optimal Approach

The optimal approach also computes mapped values, but we optimize the conversion process by avoiding repeated string operations. Instead, we transform each number by extracting digits iteratively and building the mapped value directly using arithmetic. After computing all mapped values, we sort the original numbers using Python's built-in `sorted` or Go's `sort.SliceStable` with the mapped values as the key. This guarantees stability and efficiency because sorting operations are performed in O(n log n) time, and mapped value computation is linear in the number of digits.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * d + n log n) | O(n) | Compute mapped value as string for each number, then sort using standard sort. `d` is max digits in a number. |
| Optimal | O(n * d + n log n) | O(n) | Compute mapped value using arithmetic instead of string conversions; stable sort preserves original order. |

## Algorithm Walkthrough

1. Initialize an empty list of tuples where each tuple contains the mapped value and the original number.
2. For each number in `nums`, calculate its mapped value by iterating through its digits from least significant to most significant. Replace each digit with its mapping and reconstruct the mapped number as an integer.
3. Append the tuple `(mapped_value, original_number)` to the list.
4. Sort the list of tuples based on the first element, `mapped_value`, using a stable sorting method. Stability ensures numbers with identical mapped values remain in their original relative order.
5. Extract the second element of each tuple in the sorted list to produce the final sorted array of original numbers.
6. Return the resulting array.

**Why it works:** The algorithm guarantees correctness because it maps numbers consistently according to the mapping array and then sorts stably by mapped values. By keeping the original numbers in the tuples, the original data is preserved, and sorting by the mapped value enforces the desired order.

## Python Solution

```python
from typing import List

class Solution:
    def sortJumbled(self, mapping: List[int], nums: List[int]) -> List[int]:
        def get_mapped_value(num: int) -> int:
            if num == 0:
                return mapping[0]
            mapped = 0
            place = 1
            while num > 0:
                digit = num % 10
                mapped += mapping[digit] * place
                num //= 10
                place *= 10
            return mapped
        
        # Pair each number with its mapped value
        mapped_pairs = [(get_mapped_value(num), num) for num in nums]
        # Stable sort based on mapped value
        mapped_pairs.sort(key=lambda x: x[0])
        # Extract sorted numbers
        return [num for _, num in mapped_pairs]
```

**Explanation:** We define a helper function `get_mapped_value` that calculates the mapped integer without converting to a string. Each number is paired with its mapped value. Sorting is done using `sort` with a key on the mapped value, which is stable. Finally, we extract the original numbers in the new order.

## Go Solution

```go
import "sort"

func sortJumbled(mapping []int, nums []int) []int {
    getMappedValue := func(num int) int {
        if num == 0 {
            return mapping[0]
        }
        mapped := 0
        place := 1
        for num > 0 {
            digit := num % 10
            mapped += mapping[digit] * place
            num /= 10
            place *= 10
        }
        return mapped
    }

    type pair struct {
        mapped int
        original int
    }

    pairs := make([]pair, len(nums))
    for i, num := range nums {
        pairs[i] = pair{getMappedValue(num), num}
    }

    sort.SliceStable(pairs, func(i, j int) bool {
        return pairs[i].mapped < pairs[j].mapped
    })

    result := make([]int, len(nums))
    for i, p := range pairs {
        result[i] = p.original
    }
    return result
}
```

**Explanation:** The Go version mirrors the Python approach. We define a closure for mapped value computation, create a slice of structs to pair mapped and original numbers, and then perform a stable sort using `sort.SliceStable`. The result slice extracts the original numbers in the sorted order. Edge cases like zero are handled directly.
