## Solution

## Problem Understanding

The problem asks us to simulate two gardeners, Alice and Bob, watering a row of plants simultaneously from opposite ends. Each plant requires a specific amount of water, and both gardeners have their own watering can with a fixed capacity. Alice starts from the left and Bob starts from the right. They water plants in order, refilling only when necessary, and we are asked to compute the total number of times either gardener needs to refill.

The input consists of:

- `plants`: an array of integers, where `plants[i]` indicates the water requirement for the `i`-th plant.
- `capacityA` and `capacityB`: the maximum capacities of Alice's and Bob's watering cans.

The output is a single integer representing the total number of refills.

Constraints provide guidance on the scale:

- Up to `10^5` plants.
- Water requirement and can capacities can be as large as `10^9`.

Key points include:

- Simultaneous watering from both ends.
- Refill is instantaneous but only happens if the current water is insufficient for the next plant.
- If Alice and Bob meet at the same plant, the one with more remaining water waters it, with Alice winning ties.

Important edge cases:

- Only one plant in the array.
- Water requirements equal to can capacity.
- Large arrays near `10^5` to ensure time efficiency.

## Approaches

The brute-force approach would be to simulate the watering process explicitly for every plant, checking at each step whether Alice or Bob needs to refill, and counting each refill. This works correctly but is essentially O(n) time and O(1) space. Given that n ≤ 10^5, this is already efficient, but care must be taken with the midpoint case when both meet.

The optimal approach is effectively the same simulation but carefully handling the “meeting in the middle” condition. The insight is that we only need two pointers (`i` from the left, `j` from the right) and two counters for remaining water in each can. Each step moves one pointer closer, with simple arithmetic checks for refills. This reduces the risk of off-by-one errors and ensures O(n) time complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force Simulation | O(n) | O(1) | Walk through each plant from both ends, track refills; straightforward implementation |
| Two-Pointer Simulation | O(n) | O(1) | Same as brute-force but uses two pointers to efficiently handle simultaneous watering and meeting at the same plant |

## Algorithm Walkthrough

1. Initialize two pointers `i = 0` for Alice and `j = n - 1` for Bob.
2. Initialize counters for remaining water in each can, `waterA = capacityA` and `waterB = capacityB`, and a `refills` counter to zero.
3. Loop while `i <= j`:

1. If `i == j`, it means Alice and Bob meet at the same plant. Water it with the person who has more remaining water (Alice in case of tie). If that person does not have enough water, increment `refills`.
2. Otherwise, process Alice: if `waterA < plants[i]`, increment `refills` and refill; subtract `plants[i]` from `waterA`. Then increment `i`.
3. Process Bob: if `waterB < plants[j]`, increment `refills` and refill; subtract `plants[j]` from `waterB`. Then decrement `j`.
4. Continue until all plants are watered.
5. Return the total `refills`.

Why it works: At every step, the algorithm ensures that each plant is watered by the correct gardener according to the rules, tracking refills precisely. The two-pointer mechanism guarantees that each plant is visited exactly once.

## Python Solution

```python
from typing import List

class Solution:
    def minimumRefill(self, plants: List[int], capacityA: int, capacityB: int) -> int:
        i, j = 0, len(plants) - 1
        waterA, waterB = capacityA, capacityB
        refills = 0
        
        while i <= j:
            if i == j:
                # Both meet at the same plant
                if max(waterA, waterB) < plants[i]:
                    refills += 1
                break
            
            # Alice waters from left
            if waterA < plants[i]:
                refills += 1
                waterA = capacityA
            waterA -= plants[i]
            i += 1
            
            # Bob waters from right
            if waterB < plants[j]:
                refills += 1
                waterB = capacityB
            waterB -= plants[j]
            j -= 1
        
        return refills
```

The implementation initializes pointers and water levels for both Alice and Bob. In each iteration, it checks whether each gardener has enough water, refills if necessary, subtracts the plant's requirement, and moves the pointer. The special case when they meet ensures the person with more water or Alice in case of a tie waters the plant, incrementing `refills` if needed.

## Go Solution

```go
func minimumRefill(plants []int, capacityA int, capacityB int) int {
    i, j := 0, len(plants)-1
    waterA, waterB := capacityA, capacityB
    refills := 0
    
    for i <= j {
        if i == j {
            if max(waterA, waterB) < plants[i] {
                refills++
            }
            break
        }
        
        if waterA < plants[i] {
            refills++
            waterA = capacityA
        }
        waterA -= plants[i]
        i++
        
        if waterB < plants[j] {
            refills++
            waterB = capacityB
        }
        waterB -= plants[j]
        j--
    }
    
    return refills
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

Go-specific notes: Go requires a helper function to compute the maximum between two integers. Slices are indexed the same way as arrays in Python. Integer arithmetic is sufficient as capacities and requirements are within `int` limits.
