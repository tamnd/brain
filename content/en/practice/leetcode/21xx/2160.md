## Solution

## Problem Understanding

The problem requires taking a four-digit positive integer `num` and splitting its digits into two separate numbers, `new1` and `new2`, such that **all digits of `num` are used exactly once**. The goal is to find a combination that **minimizes the sum** of `new1` and `new2`. Leading zeros are allowed in the newly formed numbers, which simplifies the problem because we do not need to worry about invalid integer representations like `03` being treated differently from `3`.

The input, `num`, is guaranteed to be a four-digit number, meaning it will always be in the range `[1000, 9999]`. The output is a single integer, which is the minimum sum obtainable from any valid splitting of the digits.

Important edge cases include numbers with zeros, repeated digits, or digits in descending order. For example, `4009` contains zeros, and `2932` contains repeated digits, which could impact how we distribute the digits to achieve the minimum sum.

The constraints are small, and the number of digits is fixed, which allows us to use sorting and greedy approaches without worrying about performance on larger inputs.

## Approaches

### Brute Force

A brute-force approach would involve generating all permutations of the four digits of `num` and then trying all ways to split them into two numbers. For each permutation, we could try splits like `[1,3]`, `[2,2]`, or `[3,1]` digits in `new1` and `new2` and compute their sum. The minimum sum among all possible splits would be returned.

This method is guaranteed to find the correct answer because it exhaustively explores all combinations. However, it is unnecessarily complex given that there are only four digits, leading to 24 permutations and multiple splits per permutation. While manageable here, the brute-force approach is conceptually overkill.

### Optimal

The key insight is that **smaller digits should be placed in higher value positions in the numbers we form** to minimize their sum. Since we only have four digits, we can sort them in ascending order and distribute them greedily:

- Let `a, b, c, d` be the sorted digits (`a <= b <= c <= d`).
- Form the two numbers as `new1 = 10 * a + c` and `new2 = 10 * b + d`.
- This guarantees the smallest possible sum because the smallest digits `a` and `b` contribute to the tens place, and the larger digits `c` and `d` contribute to the units place in each number.

This approach is simple, requires only sorting and arithmetic, and runs in constant time because the input size is fixed.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(4! * 3) = O(24) | O(4) | Generates all permutations of 4 digits and splits |
| Optimal | O(4 log 4) = O(1) | O(4) | Sort digits and greedily assign to tens and units places |

## Algorithm Walkthrough

1. Convert the integer `num` into a list of its digits. This allows us to manipulate the individual digits easily.
2. Sort the list of digits in ascending order. This ensures that smaller digits appear first.
3. Assign digits to form `new1` and `new2` by taking the first smallest digit for the tens place of `new1`, the second smallest digit for the tens place of `new2`, the third smallest for the units place of `new1`, and the largest for the units place of `new2`.
4. Compute the sum of `new1` and `new2` and return it.

**Why it works:** Sorting the digits guarantees that the smallest digits occupy the higher-weight positions in the numbers formed. Placing smaller digits in the tens place of the resulting two-digit numbers and larger digits in the units place ensures the sum is minimized. Since all digits are used exactly once, the approach is valid for all inputs within the constraints.

## Python Solution

```python
class Solution:
    def minimumSum(self, num: int) -> int:
        digits = sorted([int(d) for d in str(num)])
        new1 = digits[0] * 10 + digits[2]
        new2 = digits[1] * 10 + digits[3]
        return new1 + new2
```

The Python implementation first converts `num` into a list of digits using a list comprehension and `str(num)`. Sorting the digits ensures we know which are the smallest. We then form `new1` and `new2` using the strategy described above, calculating each as `tens * 10 + units`. Finally, the sum is returned.

## Go Solution

```go
import "sort"

func minimumSum(num int) int {
    digits := []int{
        num / 1000,
        (num / 100) % 10,
        (num / 10) % 10,
        num % 10,
    }
    sort.Ints(digits)
    new1 := digits[0]*10 + digits[2]
    new2 := digits[1]*10 + digits[3]
    return new1 + new2
}
```

In Go, we manually extract each digit using integer division and modulo operations. The digits are stored in a slice and sorted using `sort.Ints`. We then form `new1` and `new2` similarly to Python, compute the sum, and return it. Since Go does not allow list comprehensions, the digit extraction is more verbose but straightforward.

## Worked Examples

### Example 1: `num = 2932`

1. Extract digits: `[2, 9, 3, 2]`
2. Sort digits: `[2, 2, 3, 9]`
3. Form numbers: `new1 = 2*10 + 3 = 23`, `new2 = 2*10 + 9 = 29`
4. Sum: `23 + 29 = 52`

### Example 2: `num = 4009`

1. Extract digits: `[4, 0, 0, 9]`
2. Sort digits: `[0, 0, 4, 9]`
3. Form numbers: `new1 = 0*10 + 4 = 4`, `new2 = 0*10 + 9 = 9`
4. Sum: `4 + 9 = 13`

## Edge Cases

**Case 1: Number contains zeros**

For `4009`, zeros must not be ignored. Leading zeros are allowed, so `0` can occupy the tens place, forming `04` as a valid number. The algorithm correctly sorts and assigns digits to minimize the sum.

**Case 2: Repeated digits**

For `2932` or `2112`, repeated digits exist. Sorting ensures repeated digits are correctly assigned to the tens and units places without violating the "use all digits" requirement.

**Case 3: Largest or smallest four-digit numbers**

For `1000` and `9999`, the edge digits test the ability of the algorithm to handle extremes. The algorithm correctly forms numbers like `10` + `0` = `10` or `99` + `99` = `198` without additional checks.
