## Solution

## Problem Understanding

The problem gives you a binary array `nums` of length `n` and asks you to consider **every possible division point** `i` from `0` to `n`. For each division, the array is split into `numsleft` (elements before index `i`) and `numsright` (elements from index `i` onwards). The **division score** for index `i` is defined as the sum of the number of zeros in `numsleft` and the number of ones in `numsright`.

Your task is to find **all indices `i`** that achieve the **maximum division score**. The output can be in any order.

The constraints (`1 <= n <= 10^5` and each element is `0` or `1`) tell us that a **naive O(n^2) solution** that counts zeros and ones for each split will be too slow. We need a **linear-time solution**.

Important edge cases include arrays of all zeros, all ones, and very short arrays (length 1), because they test empty subarrays at division points `0` and `n`.

## Approaches

### Brute Force

A straightforward solution is to try every division `i` from `0` to `n`. For each division, count the number of zeros in `numsleft` and the number of ones in `numsright`, sum them, and keep track of the maximum score. After checking all divisions, return all indices with that maximum score.

This approach is correct because it directly implements the problem definition, but it requires **recounting zeros and ones for each division**, leading to a time complexity of **O(n^2)**, which is infeasible for `n = 10^5`.

### Optimal Approach

The key observation is that we can compute the division score in **linear time** using **prefix sums**:

- Let `zeros_left` track the number of zeros in `numsleft`.
- Let `ones_right` track the number of ones in `numsright`.

We start with `zeros_left = 0` and `ones_right = total number of ones in nums`. For each index `i` from `0` to `n`:

- The current score = `zeros_left + ones_right`.
- Update `zeros_left` and `ones_right` as we move the division forward:

- If `nums[i] == 0`, increment `zeros_left`.
- If `nums[i] == 1`, decrement `ones_right`.

This allows computing each division score in **O(1)** per index, giving an overall **O(n)** solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check each division, count zeros and ones each time |
| Optimal | O(n) | O(1) | Use running counts (prefix sums) for zeros and ones |

## Algorithm Walkthrough

1. Count the total number of ones in the array. This will be our initial `ones_right` for the division at index `0`.
2. Initialize `zeros_left = 0`.
3. Initialize variables to track the maximum score found `max_score = -1` and a list `result = []` to store indices with that score.
4. Iterate through each index `i` from `0` to `n` (inclusive):

- Compute `score = zeros_left + ones_right`.
- If `score > max_score`, update `max_score = score` and reset `result = [i]`.
- If `score == max_score`, append `i` to `result`.
- If `i < n`, update `zeros_left` and `ones_right` based on `nums[i]`:

- If `nums[i] == 0`, `zeros_left += 1`.
- If `nums[i] == 1`, `ones_right -= 1`.
5. Return `result`.

**Why it works:** At each step, `zeros_left` and `ones_right` accurately reflect the number of zeros and ones in the left and right partitions, respectively. By iterating through all possible division indices, we guarantee that we consider every possible score, and tracking the maximum ensures correctness.

## Python Solution

```python
from typing import List

class Solution:
    def maxScoreIndices(self, nums: List[int]) -> List[int]:
        total_ones = sum(nums)
        zeros_left = 0
        ones_right = total_ones
        max_score = -1
        result = []

        for i in range(len(nums) + 1):
            score = zeros_left + ones_right
            if score > max_score:
                max_score = score
                result = [i]
            elif score == max_score:
                result.append(i)

            if i < len(nums):
                if nums[i] == 0:
                    zeros_left += 1
                else:
                    ones_right -= 1

        return result
```

**Implementation Explanation:** We first count all ones for the initial right partition. For each division, we calculate the score, update the maximum if necessary, and then adjust the running counts for the next index. This approach avoids recalculating sums for each division, making it linear.

## Go Solution

```go
func maxScoreIndices(nums []int) []int {
    totalOnes := 0
    for _, num := range nums {
        if num == 1 {
            totalOnes++
        }
    }

    zerosLeft := 0
    onesRight := totalOnes
    maxScore := -1
    result := []int{}

    for i := 0; i <= len(nums); i++ {
        score := zerosLeft + onesRight
        if score > maxScore {
            maxScore = score
            result = []int{i}
        } else if score == maxScore {
            result = append(result, i)
        }

        if i < len(nums) {
            if nums[i] == 0 {
                zerosLeft++
            } else {
                onesRight--
            }
        }
    }

    return result
}
```

**Go Notes:** Similar logic applies. The slice `result` dynamically tracks indices. Go requires explicit handling for appending to slices. We iterate to `len(nums)` to consider the division after the last element.
