## Solution

## Problem Understanding

This problem asks us to determine the maximum number of **good people** in a group, given statements they make about each other. Each person either always tells the truth (good person) or may lie (bad person). We are given a square matrix `statements` where `statements[i][j]` represents what person `i` says about person `j`: `0` for bad, `1` for good, and `2` for no statement. The goal is to determine the largest possible number of good people consistent with all statements made by the good people.

The input is constrained such that `2 <= n <= 15`. This upper bound suggests that enumerating all subsets of people is feasible because `2^15` is about 32,768. Each subset represents a possible assignment of good and bad people. Edge cases to watch include when everyone is bad, everyone is good, or statements are contradictory, which could reduce the number of good people.

The problem guarantees that no person makes a statement about themselves, so `statements[i][i] == 2`. This simplifies our verification logic, since we never need to check self-references.

## Approaches

The **brute-force approach** is to consider all possible assignments of good and bad for the `n` people. For each assignment, we verify whether it is consistent with the statements made by the people who are marked good. If a statement from a good person conflicts with our current assignment, that configuration is invalid. After checking all assignments, we return the maximum number of good people found in any valid configuration.

The **key insight** that improves the brute-force approach is that we can represent each assignment as a bitmask, where the i-th bit is `1` if person `i` is good and `0` if person `i` is bad. Using bit manipulation, we can efficiently iterate over all subsets of people, check consistency with statements, and count the number of good people. This reduces overhead and allows the solution to remain performant even for the upper bound `n = 15`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n^2) | O(n^2) | Enumerate all 2^n assignments, validate each against statements |
| Optimal (Bitmask) | O(2^n * n^2) | O(1) | Use bitmask to encode assignments efficiently, same validation |

Even though the asymptotic complexity is the same, the bitmask representation is faster and more memory-efficient than nested loops with arrays.

## Algorithm Walkthrough

1. **Iterate over all subsets**: Represent each possible combination of good and bad people as an integer `mask` of `n` bits. Each bit indicates whether the person is good (1) or bad (0).
2. **Check validity of the assignment**: For each person `i` marked as good in the mask, iterate over all their statements about others `j`. If `statements[i][j]` is `0`, then person `j` must be bad in the mask. If `statements[i][j]` is `1`, then person `j` must be good in the mask. If any statement contradicts the current mask, discard this assignment.
3. **Count good people**: If the mask is valid, count the number of `1`s (good people) in the mask using a bit count operation.
4. **Track the maximum**: Keep a variable `max_good` to track the maximum number of good people across all valid assignments.
5. **Return the result**: After iterating all masks, return `max_good`.

**Why it works**: Every possible combination of good and bad people is considered, and only consistent assignments are counted. This exhaustive search guarantees correctness. The bitmask ensures efficient enumeration and validation.

## Python Solution

```python
from typing import List

class Solution:
    def maximumGood(self, statements: List[List[int]]) -> int:
        n = len(statements)
        max_good = 0

        for mask in range(1 << n):
            valid = True
            for i in range(n):
                if not (mask & (1 << i)):
                    continue
                for j in range(n):
                    if statements[i][j] == 2:
                        continue
                    if statements[i][j] != ((mask >> j) & 1):
                        valid = False
                        break
                if not valid:
                    break
            if valid:
                max_good = max(max_good, bin(mask).count('1'))
        
        return max_good
```

The Python code iterates over all `2^n` assignments using bitmasks. For each assignment, it validates statements from the good people and counts the number of good people if valid. The use of `bin(mask).count('1')` allows efficient counting of bits set to `1`.

## Go Solution

```go
func maximumGood(statements [][]int) int {
    n := len(statements)
    maxGood := 0

    for mask := 0; mask < (1 << n); mask++ {
        valid := true
        for i := 0; i < n && valid; i++ {
            if mask&(1<<i) == 0 {
                continue
            }
            for j := 0; j < n; j++ {
                if statements[i][j] == 2 {
                    continue
                }
                if statements[i][j] != ((mask >> j) & 1) {
                    valid = false
                    break
                }
            }
        }
        if valid {
            count := 0
            for x := mask; x > 0; x &= x - 1 {
                count++
            }
            if count > maxGood {
                maxGood = count
            }
        }
    }

    return maxGood
}
```

The Go version mirrors the Python implementation. It uses bit operations to check if a person is good and iterates through statements for validity. Counting set bits is done using Brian Kernighan’s algorithm.

The `n` is small (`<= 15`), so 32,768 iterations are feasible. The validation loop checks all statements of good people, giving `n^2` operations in the worst case.
