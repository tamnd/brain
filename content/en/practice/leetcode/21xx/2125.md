## Solution

## Problem Understanding

The problem presents a binary matrix representing a bank floor plan. Each row of the matrix is a string of `'0'`s and `'1'`s, where `'1'` indicates a security device, and `'0'` indicates an empty cell. The goal is to count the total number of **laser beams** formed between security devices under two strict conditions.

The first condition is that the two devices must be on **different rows**. The second condition is that all rows **between these two rows** must be completely empty (contain only `'0'`s). A laser beam exists for **every pair of devices** that satisfy these conditions. Laser beams are independent, meaning no interference occurs between multiple beams.

The input constraints are: the number of rows `m` and columns `n` are each up to 500. Therefore, the matrix can be as large as 500x500, making a naive O((m*n)^2) approach potentially too slow.

Important edge cases include rows that are completely empty, consecutive rows with devices, and the smallest possible matrices with either a single row or a single device.

The expected output is the **total number of beams**, which is an integer.

## Approaches

### Brute Force

A brute-force solution would iterate over all pairs of rows `(r1, r2)` where `r1 < r2`. For each pair, it would check if all rows between `r1` and `r2` are empty. If so, it would count the number of devices in both rows and compute the product as the number of beams connecting the two rows. Summing these products over all valid pairs gives the total number of beams. This approach is correct but inefficient. In the worst case, it requires checking all pairs of rows and inspecting each intermediate row, leading to **O(m^2 * n)** time complexity.

### Optimal Approach

The key observation is that **only consecutive non-empty rows contribute to laser beams**. If a row is empty, it effectively separates beams. Therefore, we can traverse the bank row by row, count the number of devices in each non-empty row, and multiply the count with the count from the previous non-empty row to compute beams. Accumulating these products gives the total number of beams. This reduces the complexity to **O(m * n)** since each cell is visited once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m^2 * n) | O(1) | Check all pairs of rows and all intermediate rows for emptiness |
| Optimal | O(m * n) | O(1) | Count devices in each row and multiply consecutive non-empty rows |

## Algorithm Walkthrough

1. Initialize a variable `prev_count` to 0. This will store the number of devices in the last non-empty row.
2. Initialize `total_beams` to 0. This will accumulate the total number of laser beams.
3. Iterate over each row of the bank:

1. Count the number of `'1'`s in the current row, call this `curr_count`.
2. If `curr_count > 0` (i.e., the row is non-empty):

1. Multiply `curr_count` by `prev_count` and add the result to `total_beams`. This accounts for all beams between this row and the previous non-empty row.
2. Update `prev_count` to `curr_count`.
3. If `curr_count == 0`, do nothing; skip to the next row.
4. After processing all rows, return `total_beams`.

**Why it works:** By only multiplying counts of consecutive non-empty rows, the algorithm automatically satisfies the condition that all rows between two contributing rows must be empty. This ensures we count every beam exactly once and never count invalid beams.

## Python Solution

```python
from typing import List

class Solution:
    def numberOfBeams(self, bank: List[str]) -> int:
        prev_count = 0
        total_beams = 0
        
        for row in bank:
            curr_count = row.count('1')
            if curr_count > 0:
                total_beams += prev_count * curr_count
                prev_count = curr_count
        
        return total_beams
```

The implementation iterates over each row, counts the devices, and multiplies by the previous non-empty row's count to accumulate beams. Rows with zero devices are skipped, ensuring empty rows do not contribute or interfere with counting.

## Go Solution

```go
func numberOfBeams(bank []string) int {
    prevCount := 0
    totalBeams := 0
    
    for _, row := range bank {
        currCount := 0
        for _, ch := range row {
            if ch == '1' {
                currCount++
            }
        }
        if currCount > 0 {
            totalBeams += prevCount * currCount
            prevCount = currCount
        }
    }
    
    return totalBeams
}
```

In Go, we iterate over each string and count `'1'` characters using a simple loop. The logic mirrors the Python version, using integer multiplication to accumulate the number of beams.

## Worked Examples

**Example 1:** `bank = ["011001","000000","010100","001000"]`

| Row | Count of '1's | Prev Count | Total Beams Calculation | Total Beams |
| --- | --- | --- | --- | --- |
| 0 | 3 | 0 | 0 * 3 = 0 | 0 |
| 1 | 0 | 3 | skip | 0 |
| 2 | 3 | 3 | 3 * 3 = 9 | 9 |
| 3 | 1 | 3 | 3 * 1 = 3 | 12 |

After careful inspection, only consecutive non-empty rows are considered. Row 1 is empty, so beams are counted between row 0 and row 2 (3_3=9), then row 2 and row 3 (3_1=3). Total beams = 9 + 3 = 12.

**Example 2:** `bank = ["000","111","000"]`

| Row | Count of '1's | Prev Count | Total Beams Calculation | Total Beams |
| --- | --- | --- | --- | --- |
| 0 | 0 | 0 | skip | 0 |
| 1 | 3 | 0 | 0 * 3 = 0 | 0 |
| 2 | 0 | 3 | skip | 0 |

Total beams = 0.

## Edge Cases

**Empty Rows Only:** If the bank contains only empty rows, the algorithm correctly returns 0, as no beams can form.

**Single Row with Devices:** If the bank has only one row with devices, no beams can exist because beams require two different rows.

**Consecutive Non-Empty Rows:** The algorithm multiplies counts only for consecutive non-empty rows. Any intermediate empty rows automatically reset the previous count logic. This ensures correctness without explicitly checking every intermediate row.

**Large Sparse Matrices:** For matrices with hundreds of rows but mostly empty, the O(m_n) scan is efficient and avoids O(m^2_n) overhead. Each row is scanned once, ensuring performance at maximum input size.
