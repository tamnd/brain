## Solution

## Problem Understanding

The problem gives us an array of strings where every string contains only unique lowercase letters. Instead of treating the strings as ordered sequences of characters, we should think of each string as a set of letters.

Two strings are considered connected if we can transform one set of letters into the other using exactly one operation:

1. Add one letter
2. Remove one letter
3. Replace one letter with another letter, including replacing a letter with itself

Because every character appears at most once in a word, each word can be represented very naturally as a 26-bit bitmask. Each bit corresponds to one lowercase letter. For example:

- `"a"` becomes `000...0001`
- `"ab"` becomes `000...0011`
- `"cde"` becomes bits for `c`, `d`, and `e`

The key observation is that the actual order of letters does not matter, only which letters exist.

The goal is to partition all words into connected groups. If one word can reach another through a chain of valid connections, they belong to the same group. We must return:

- The number of connected groups
- The size of the largest group

The constraints are important:

- Up to `2 * 10^4` words
- Each word length is at most 26
- Only lowercase English letters
- No duplicate letters within a word

A naive graph construction comparing every pair of words would require checking roughly `n^2` pairs, which becomes too expensive when `n = 20000`.

Several edge cases are important:

- Multiple identical words
- Words differing by exactly one added or removed letter
- Replacement operations between words of the same length
- Completely isolated words
- Chains of connectivity where words are not directly connected but belong to the same component transitively

For example, `"a"` and `"abc"` are not directly connected, but if `"ab"` exists, then all three belong to the same group.

## Approaches

### Brute Force Approach

The brute force solution treats every word as a node in a graph.

For every pair of words, we check whether they are directly connected. Two words are connected if their letter sets differ by exactly one add/remove operation or exactly one replacement operation.

Using bitmasks, we can compare two words efficiently:

- If the XOR of masks has exactly one bit set, then one add/remove operation connects them.
- If the XOR has exactly two bits set and the words have equal length, then one replacement connects them.

After building the graph, we run DFS, BFS, or Union-Find to compute connected components.

This approach is correct because it explicitly checks every possible edge between words. However, the time complexity is too large.

With `n = 20000`, pairwise comparison requires about:

$$\frac{n(n-1)}{2} \approx 2 \times 10^8$$

comparisons in the worst case, which is not feasible.

### Key Insight

The important observation is that every word has at most 26 letters. Instead of comparing every pair of words, we can generate all possible neighboring masks directly.

Each word mask can transform into another valid mask by:

- Toggling one bit, add/remove
- Removing one bit and adding another, replacement

Since there are only 26 letters, the total number of possible one-step transformations from a mask is manageable.

We use Union-Find to dynamically connect masks that differ by one valid operation.

The replacement operation has a particularly elegant optimization:

If two masks become identical after deleting one character from each, then they differ by exactly one replacement.

Example:

- `"ab"` → remove `a` → `"b"`
- `"cb"` → remove `c` → `"b"`

Since both reduce to the same intermediate mask, they are connected by replacement.

This lets us avoid explicitly checking all 26 × 26 replacements.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Compare every pair of words |
| Optimal | O(n × 26²) | O(n) | Generate neighbors directly with bitmask operations |

## Algorithm Walkthrough

1. Convert every word into a bitmask.

Each character corresponds to one bit among 26 bits. For example:

$$\text{"abc"} \rightarrow 111_2$$

We store the mask for every word index.
2. Initialize a Union-Find structure.

Each word initially belongs to its own group.
3. Merge identical masks.

Multiple words may produce the same bitmask. Since they represent identical letter sets, they must belong to the same component immediately.

We keep a hash map:

```
mask -> representative index
```
4. Handle add/remove operations.

For each mask, toggle every bit:

- If the bit exists, removing it creates a candidate neighbor
- If the bit does not exist, adding it creates a candidate neighbor

If the resulting mask already exists in our map, union the two indices.
5. Handle replacement operations.

For every set bit in the current mask:

- Remove that bit
- Use the reduced mask as a key

If another word previously generated the same reduced mask, then the two original masks differ by exactly one replacement.

Example:

```
"ab" removing 'a' -> "b"
"cb" removing 'c' -> "b"
```

Therefore `"ab"` and `"cb"` are connected.
6. Count component sizes.

After all unions are complete:

- Count distinct roots
- Track the size of each component
- Return:

- number of groups
- largest group size

### Why it works

Union-Find maintains the invariant that two words belong to the same set if there exists a sequence of valid transformations connecting them.

The algorithm enumerates every possible one-operation transformation:

- Toggle one bit for add/remove
- Match reduced masks for replacement

Since every legal edge is considered exactly once, all connected components are correctly formed.

## Python Solution

```python
from typing import List
from collections import defaultdict

class UnionFind:
    def __init__(self, n: int):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x: int) -> int:
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]
            x = self.parent[x]
        return x

    def union(self, a: int, b: int) -> None:
        root_a = self.find(a)
        root_b = self.find(b)

        if root_a == root_b:
            return

        if self.size[root_a] < self.size[root_b]:
            root_a, root_b = root_b, root_a

        self.parent[root_b] = root_a
        self.size[root_a] += self.size[root_b]

class Solution:
    def groupStrings(self, words: List[str]) -> List[int]:
        n = len(words)
        uf = UnionFind(n)

        masks = []

        for word in words:
            mask = 0
            for ch in word:
                mask |= 1 << (ord(ch) - ord('a'))
            masks.append(mask)

        mask_to_index = {}

        for i, mask in enumerate(masks):
            if mask in mask_to_index:
                uf.union(i, mask_to_index[mask])
            else:
                mask_to_index[mask] = i

        deleted_mask_map = {}

        for i, mask in enumerate(masks):

            for bit in range(26):
                neighbor = mask ^ (1 << bit)

                if neighbor in mask_to_index:
                    uf.union(i, mask_to_index[neighbor])

            for bit in range(26):
                if mask & (1 << bit):
                    deleted = mask ^ (1 << bit)

                    if deleted in deleted_mask_map:
                        uf.union(i, deleted_mask_map[deleted])
                    else:
                        deleted_mask_map[deleted] = i

        component_sizes = defaultdict(int)

        for i in range(n):
            root = uf.find(i)
            component_sizes[root] += 1

        number_of_groups = len(component_sizes)
        largest_group_size = max(component_sizes.values())

        return [number_of_groups, largest_group_size]
```

The implementation starts by converting every word into its compact 26-bit representation. This allows all connectivity operations to become simple bit manipulations.

The `UnionFind` class efficiently maintains connected components. Path compression and union by size ensure near-constant amortized complexity.

The `mask_to_index` dictionary maps each exact mask to a representative index. This immediately merges duplicate words.

Next, for each mask, the code toggles every bit. This covers all add and remove operations. If the transformed mask exists in the dictionary, the corresponding indices are united.

The replacement operation uses the `deleted_mask_map`. For every character removed from a word, we record the resulting reduced mask. If another word already produced the same reduced mask, the two words differ by exactly one replacement, so they are merged.

Finally, we compute connected component sizes by grouping indices according to their Union-Find roots.

## Go Solution

```go
package main

type UnionFind struct {
	parent []int
	size   []int
}

func NewUnionFind(n int) *UnionFind {
	parent := make([]int, n)
	size := make([]int, n)

	for i := 0; i < n; i++ {
		parent[i] = i
		size[i] = 1
	}

	return &UnionFind{
		parent: parent,
		size:   size,
	}
}

func (uf *UnionFind) Find(x int) int {
	for uf.parent[x] != x {
		uf.parent[x] = uf.parent[uf.parent[x]]
		x = uf.parent[x]
	}
	return x
}

func (uf *UnionFind) Union(a, b int) {
	rootA := uf.Find(a)
	rootB := uf.Find(b)

	if rootA == rootB {
		return
	}

	if uf.size[rootA] < uf.size[rootB] {
		rootA, rootB = rootB, rootA
	}

	uf.parent[rootB] = rootA
	uf.size[rootA] += uf.size[rootB]
}

func groupStrings(words []string) []int {
	n := len(words)

	uf := NewUnionFind(n)

	masks := make([]int, n)

	for i, word := range words {
		mask := 0

		for _, ch := range word {
			mask |= 1 << (ch - 'a')
		}

		masks[i] = mask
	}

	maskToIndex := make(map[int]int)

	for i, mask := range masks {
		if idx, exists := maskToIndex[mask]; exists {
			uf.Union(i, idx)
		} else {
			maskToIndex[mask] = i
		}
	}

	deletedMaskMap := make(map[int]int)

	for i, mask := range masks {

		for bit := 0; bit < 26; bit++ {
			neighbor := mask ^ (1 << bit)

			if idx, exists := maskToIndex[neighbor]; exists {
				uf.Union(i, idx)
			}
		}

		for bit := 0; bit < 26; bit++ {
			if mask&(1<<bit) != 0 {
				deleted := mask ^ (1 << bit)

				if idx, exists := deletedMaskMap[deleted]; exists {
					uf.Union(i, idx)
				} else {
					deletedMaskMap[deleted] = i
				}
			}
		}
	}

	componentSizes := make(map[int]int)

	for i := 0; i < n; i++ {
		root := uf.Find(i)
		componentSizes[root]++
	}

	numberOfGroups := len(componentSizes)

	largestGroupSize := 0

	for _, size := range componentSizes {
		if size > largestGroupSize {
			largestGroupSize = size
		}
	}

	return []int{numberOfGroups, largestGroupSize}
}
```

The Go implementation follows the same logic as the Python version. Maps are used instead of Python dictionaries, and slices replace Python lists.

Go does not provide built in Union-Find utilities, so the disjoint set structure is implemented manually. Integer overflow is not a concern because masks fit comfortably inside a 32-bit integer.

words = ["a","b","ab","cde"]

| "ab" | 3 |

### Step 2, Add/Remove Connections

For `"a"`:

- Toggle `b` bit:

- `1 ^ 2 = 3`
- `"ab"` exists
- Union("a", "ab")
- Toggle `a` bit:

- `1 ^ 1 = 0`
- No word

For `"b"`:

- Toggle `a` bit:

- `2 ^ 1 = 3`
- `"ab"` exists
- Union("b", "ab")

### Step 3, Replacement Connections

For `"a"`:

- Remove `a`
- Reduced mask = `0`

For `"b"`:

- Remove `b`
- Reduced mask = `0`
- Match found
- Union("a", "b")

### Final Components

| Group | Words |
| --- | --- |
| 1 | ["a","b","ab"] |
| 2 | ["cde"] |

Answer:

```
[2,3]
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × 26) | Each word performs a constant number of 26-bit operations |
| Space | O(n) | Hash maps and Union-Find structures store one entry per word |

Although there are two loops over 26 bits, the factor is constant. Therefore the complexity simplifies to linear in the number of words.

The algorithm is efficient enough for `2 * 10^4` words.

## Edge Cases

One important edge case is duplicate words. Since words are treated as sets of letters, two identical strings represent the exact same node configuration. A buggy implementation might accidentally count them as separate groups. The solution handles this by immediately unioning indices that share the same mask through the `mask_to_index` dictionary.

Another tricky case involves replacement operations. Two words of the same length may differ by exactly one character, such as `"ab"` and `"cb"`. Detecting this efficiently is difficult with naive comparisons. The implementation solves this elegantly by removing one bit at a time and matching reduced masks. If two words reduce to the same intermediate mask, they differ by one replacement.

A third important edge case is transitive connectivity. Words may not be directly connected but still belong to the same component through intermediate transformations. For example:

```
"a" -> "ab" -> "abc"
```

Here `"a"` and `"abc"` are not directly connected, but they must belong to the same group. Union-Find naturally handles this by merging connected components incrementally, ensuring the final groups reflect full transitive closure.
