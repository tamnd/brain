## Solution

## Problem Understanding

The problem presents a 0-indexed integer array `nums` of length `3 * n` and asks us to remove a subsequence of exactly `n` elements. After removing these elements, the remaining `2 * n` elements are split into two equal halves: the first half consisting of the first `n` elements and the second half consisting of the next `n` elements. We are asked to minimize the difference between the sums of these two halves, specifically `sumfirst - sumsecond`.

The input `nums` represents the array from which elements can be removed. The output is a single integer representing the minimum possible difference between the sum of the first `n` elements and the sum of the second `n` elements after removing exactly `n` elements.

Constraints tell us that `n` can be as large as `10^5`, meaning `nums` can contain up to `3 * 10^5` elements. Each element is positive and does not exceed `10^5`. This guarantees that a brute-force removal of all possible subsequences is infeasible. The problem also guarantees that a solution always exists, so we do not need to handle cases where no valid subsequence exists.

Important edge cases include arrays where all elements are equal, arrays with strictly increasing or decreasing values, and scenarios where the optimal subsequence to remove involves elements from both halves of the remaining array. Handling these correctly ensures the algorithm does not oversimplify the problem.

## Approaches

The brute-force approach would generate all combinations of `n` elements to remove, compute the sums of the two halves of the remaining array for each removal, and return the minimum difference. While this is correct in principle, the number of combinations is `C(3n, n)`, which is astronomically large for the given constraints. This approach is therefore computationally infeasible.

The key insight for an optimal solution is that we only need to track the minimum sum of `n` elements from the first `2 * n` elements and the maximum sum of `n` elements from the last `2 * n` elements. Using two heaps (priority queues) allows us to maintain these sums efficiently. Specifically, we can traverse the first `2 * n` elements left to right using a max-heap to track the smallest `n` elements, and traverse the last `2 * n` elements right to left using a min-heap to track the largest `n` elements. Subtracting the maximum sum of the second half from the minimum sum of the first half gives a candidate for the minimal difference. Iterating through the split positions and maintaining these sums efficiently yields the optimal solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(C(3n, n) * n) | O(1) | Compute all possible subsequences, too slow for n up to 10^5 |
| Optimal (Heap-based) | O(n log n) | O(n) | Use two heaps to maintain sums of n smallest/largest elements |

## Algorithm Walkthrough

1. Compute `n` as `len(nums) // 3`. Initialize two arrays `left_min_sums` and `right_max_sums` of length `n+1` to store cumulative sums.
2. Traverse the first `2 * n` elements from left to right. Maintain a max-heap of size `n` to keep track of the smallest `n` elements. For each position, record the sum of the smallest `n` elements in `left_min_sums`.
3. Traverse the last `2 * n` elements from right to left. Maintain a min-heap of size `n` to keep track of the largest `n` elements. For each position, record the sum of the largest `n` elements in `right_max_sums`.
4. Iterate over possible split points `i` between `n` and `2 * n`. Compute the difference `left_min_sums[i - n] - right_max_sums[i]` for each split and track the minimal difference.
5. Return the minimal difference found.

Why it works: By maintaining the smallest sum from the left `n` elements and the largest sum from the right `n` elements at every split position, we ensure we explore all configurations of removing `n` elements and partitioning the remaining elements into two halves, thereby guaranteeing that the minimal difference is found efficiently.

## Python Solution

```python
from typing import List
import heapq

class Solution:
    def minimumDifference(self, nums: List[int]) -> int:
        n = len(nums) // 3
        
        left_heap, left_sum = [], 0
        left_min_sums = [0] * (n + 1)
        for i in range(2 * n):
            heapq.heappush(left_heap, -nums[i])
            left_sum += nums[i]
            if len(left_heap) > n:
                left_sum += heapq.heappop(left_heap)
            if i >= n - 1:
                left_min_sums[i - n + 1] = left_sum
        
        right_heap, right_sum = [], 0
        right_max_sums = [0] * (n + 1)
        for i in range(3 * n - 1, n - 1, -1):
            heapq.heappush(right_heap, nums[i])
            right_sum += nums[i]
            if len(right_heap) > n:
                right_sum -= heapq.heappop(right_heap)
            if i <= 2 * n:
                right_max_sums[i - n] = right_sum
        
        min_diff = float('inf')
        for i in range(n + 1):
            min_diff = min(min_diff, left_min_sums[i] - right_max_sums[i])
        
        return min_diff
```

The Python solution uses two heaps to efficiently track sums of `n` smallest elements from the left and `n` largest elements from the right. The arrays `left_min_sums` and `right_max_sums` store cumulative sums at each split point. Finally, iterating through possible splits gives the minimal difference.

## Go Solution

```go
package main

import (
    "container/heap"
    "math"
)

type IntHeap []int

func (h IntHeap) Len() int           { return len(h) }
func (h IntHeap) Less(i, j int) bool { return h[i] < h[j] }
func (h IntHeap) Swap(i, j int)      { h[i], h[j] = h[j], h[i] }

func (h *IntHeap) Push(x interface{}) { *h = append(*h, x.(int)) }
func (h *IntHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[0 : n-1]
    return x
}

func minimumDifference(nums []int) int64 {
    n := len(nums) / 3
    
    leftHeap := &IntHeap{}
    heap.Init(leftHeap)
    leftSum := int64(0)
    leftMinSums := make([]int64, n+1)
    
    for i := 0; i < 2*n; i++ {
        heap.Push(leftHeap, -nums[i])
        leftSum += int64(nums[i])
        if leftHeap.Len() > n {
            leftSum += int64(heap.Pop(leftHeap).(int))
        }
        if i >= n-1 {
            leftMinSums[i-n+1] = leftSum
        }
    }
    
    rightHeap := &IntHeap{}
    heap.Init(rightHeap)
    rightSum := int64(0)
    rightMaxSums := make([]int64, n+1)
    
    for i := 3*n - 1; i >= n; i-- {
        heap.Push(rightHeap, nums[i])
        rightSum += int64(nums[i])
        if rightHeap.Len() > n {
            rightSum -= int64(heap.Pop(rightHeap).(int))
        }
        if i <= 2*n {
            rightMaxSums[i-n] = rightSum
        }
    }
    
    minDiff := int64(math.MaxInt64)
    for i := 0; i <= n; i++ {
        diff := leftMinSums[i] - rightMaxSums[i]
        if diff < minDiff {
            minDiff = diff
        }
    }
    
    return minDiff
}
```

In Go, we implement a min-heap using the `container/heap` package. The logic mirrors the Python solution, but we must handle integer types carefully to avoid overflow since sums can be large. We use `int64` for all sums.

`n = 1`. Removing each element:

- Remove 1 → remaining [3,2], sumfirst = 3, sumsecond = 2, diff = 1
- Remove 2 → remaining [3,1], sumfirst = 3, sumsecond = 1, diff = 2

Minimum difference is -1.
