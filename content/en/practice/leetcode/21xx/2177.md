## Problem Understanding

The problem asks us to find three consecutive integers whose sum equals a given number `num`. In other words, we need to identify integers `x`, `x+1`, and `x+2` such that `x + (x + 1) + (x + 2) = num`. If no such integers exist, we return an empty array.

The input `num` is a non-negative integer, which can be as large as `10^15`. The output must either be a list of three consecutive integers sorted in increasing order or an empty array if no solution exists. This problem involves basic arithmetic and properties of consecutive numbers, but the constraint `num <= 10^15` implies that naive brute-force iteration over all possibilities is inefficient. Edge cases to consider include very small numbers, numbers too small to form three consecutive integers (like `0`, `1`, `2`), and numbers that are not divisible in a way that allows three consecutive numbers to sum to them.

## Approaches

The **brute-force approach** would iterate over all integers `x` from 0 up to `num/3` and check whether `x + (x+1) + (x+2) == num`. This works because we are testing all possible starting points for three consecutive numbers, but for large `num`, it is extremely inefficient and impractical.

The **optimal approach** comes from the observation that the sum of three consecutive integers starting at `x` is `x + (x + 1) + (x + 2) = 3x + 3 = 3(x + 1)`. From this, we can solve for `x`:

```
num = 3(x + 1)
x = (num / 3) - 1
```

This gives us a direct formula. If `x` is an integer, then `[x, x+1, x+2]` is the solution. Otherwise, no solution exists, and we return an empty array. This approach is O(1) in both time and space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(num) | O(1) | Iterates through all candidate starting integers to check sum |
| Optimal | O(1) | O(1) | Uses arithmetic formula to compute the starting integer directly |

## Algorithm Walkthrough

1. Observe that the sum of three consecutive integers starting at `x` is `3x + 3 = 3(x + 1)`.
2. Solve the equation `num = 3(x + 1)` for `x`:

```
x = num / 3 - 1
```
3. Check if `x` is an integer. This can be done by checking whether `num % 3 == 0`.
4. If `num` is divisible by 3, compute the consecutive integers `[x, x+1, x+2]` and return them.
5. If `num` is not divisible by 3, return an empty array, since no three consecutive integers can sum to `num`.

**Why it works**: The formula guarantees correctness because the sum of three consecutive integers always has the form `3(x + 1)`. This provides a necessary and sufficient condition: if `num` is divisible by 3, there exists exactly one valid starting integer. If not, no valid consecutive integers exist.

## Python Solution

```python
from typing import List

class Solution:
    def sumOfThree(self, num: int) -> List[int]:
        if num % 3 != 0:
            return []
        x = num // 3 - 1
        return [x, x + 1, x + 2]
```

In this implementation, we first check if `num` is divisible by 3. If it is not, we immediately return an empty list. Otherwise, we compute `x` as `num // 3 - 1` and construct the list of consecutive integers `[x, x+1, x+2]`.

## Edge Cases

One important edge case is `num = 0`. Here, `x = 0 // 3 - 1 = -1`, so the solution is `[-1, 0, 1]`. This can trip up naive implementations that assume `x` is always non-negative.

Another edge case is very small numbers like `1` or `2`. These cannot form three consecutive integers, and the divisibility check correctly returns an empty array without further computation.

A third edge case involves extremely large numbers, up to `10^15`. Using integer arithmetic without loops ensures that no overflow occurs and the solution remains efficient and correct. This demonstrates the robustness of the formula-based approach.
