## Solution

## Problem Understanding

The problem asks us to implement a **Bitset** class, which is a compact data structure to store bits efficiently and allows various operations on them. Conceptually, a Bitset is similar to an array of boolean values, but the key challenge is to perform operations efficiently, especially `flip`, `all`, `one`, and `count`, without iterating through all bits each time.

The input represents operations on the Bitset, such as setting a bit (`fix`), clearing a bit (`unfix`), flipping all bits (`flip`), and querying the bitset state (`all`, `one`, `count`, `toString`). The output is the result of query operations or `null` for update operations. The constraints indicate that the Bitset size can be up to `10^5` and up to `10^5` operations can be called. This rules out naive solutions that iterate over the entire bitset for every query, especially for `flip`, `all`, `one`, and `count`.

Important edge cases include flipping multiple times, fixing or unfixing the same index repeatedly, and checking queries on fully flipped or empty bitsets.

## Approaches

The brute-force approach is straightforward: use an array of bits initialized to `0`. `fix` and `unfix` simply update the bit at the specified index. `flip` iterates through the entire array to invert each bit. Queries like `all`, `one`, and `count` iterate through the array to compute the result. While this works for small sizes, it becomes too slow for large sizes (`10^5`) due to repeated iterations in `flip` and query methods.

The key observation for an optimal solution is that `flip` does not need to modify each bit individually. Instead, we can maintain a **flip state flag** that indicates whether the Bitset is currently inverted. Each bit operation (`fix`/`unfix`) and query can then interpret the bit according to this flag. Additionally, maintaining a **count of bits set to 1** allows `all`, `one`, and `count` queries to execute in O(1) time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(size) per flip/query | O(size) | Updates and flips iterate through all bits |
| Optimal | O(1) per operation | O(size) | Uses flip flag and count to handle flips and queries efficiently |

## Algorithm Walkthrough

1. Initialize a list `bits` of size `size` with all zeros, a counter `ones` for the number of bits set to 1, and a boolean `flipped` initially `False`.
2. For `fix(idx)`, check if the effective bit is `0` (considering `flipped`). If yes, set it to `1` and increment `ones`.
3. For `unfix(idx)`, check if the effective bit is `1`. If yes, set it to `0` and decrement `ones`.
4. For `flip()`, toggle the `flipped` flag and update `ones` to `size - ones` because the number of ones and zeros is swapped.
5. For `all()`, check if `ones == size`. Return `True` if all bits are 1, `False` otherwise.
6. For `one()`, check if `ones > 0`. Return `True` if at least one bit is 1, `False` otherwise.
7. For `count()`, return `ones`.
8. For `toString()`, construct the string by iterating over `bits` and applying the `flipped` flag to each bit.

**Why it works:** The invariant is that `bits[i]` represents the bit in its **original state**, and the `flipped` flag represents a global inversion. Any operation interprets the bit with respect to this flag, so `flip`, `fix`, and `unfix` are all consistent. The `ones` counter is always accurate relative to the current effective bit values.

## Python Solution

```python
class Bitset:

    def __init__(self, size: int):
        self.size = size
        self.bits = [0] * size
        self.ones = 0
        self.flipped = False

    def fix(self, idx: int) -> None:
        if not self.flipped:
            if self.bits[idx] == 0:
                self.bits[idx] = 1
                self.ones += 1
        else:
            if self.bits[idx] == 1:
                self.bits[idx] = 0
                self.ones += 1

    def unfix(self, idx: int) -> None:
        if not self.flipped:
            if self.bits[idx] == 1:
                self.bits[idx] = 0
                self.ones -= 1
        else:
            if self.bits[idx] == 0:
                self.bits[idx] = 1
                self.ones -= 1

    def flip(self) -> None:
        self.flipped = not self.flipped
        self.ones = self.size - self.ones

    def all(self) -> bool:
        return self.ones == self.size

    def one(self) -> bool:
        return self.ones > 0

    def count(self) -> int:
        return self.ones

    def toString(self) -> str:
        if not self.flipped:
            return ''.join(str(b) for b in self.bits)
        else:
            return ''.join(str(1 - b) for b in self.bits)
```

The implementation uses a `flipped` flag to handle bulk flips in O(1) time. The `ones` counter tracks the effective number of 1s. `fix` and `unfix` adjust the counter carefully depending on the `flipped` state. `toString` dynamically interprets bits according to the flip state.

## Go Solution

```go
type Bitset struct {
	bits    []int
	size    int
	ones    int
	flipped bool
}

func Constructor(size int) Bitset {
	return Bitset{
		bits: make([]int, size),
		size: size,
		ones: 0,
	}
}

func (this *Bitset) Fix(idx int) {
	if !this.flipped {
		if this.bits[idx] == 0 {
			this.bits[idx] = 1
			this.ones++
		}
	} else {
		if this.bits[idx] == 1 {
			this.bits[idx] = 0
			this.ones++
		}
	}
}

func (this *Bitset) Unfix(idx int) {
	if !this.flipped {
		if this.bits[idx] == 1 {
			this.bits[idx] = 0
			this.ones--
		}
	} else {
		if this.bits[idx] == 0 {
			this.bits[idx] = 1
			this.ones--
		}
	}
}

func (this *Bitset) Flip() {
	this.flipped = !this.flipped
	this.ones = this.size - this.ones
}

func (this *Bitset) All() bool {
	return this.ones == this.size
}

func (this *Bitset) One() bool {
	return this.ones > 0
}

func (this *Bitset) Count() int {
	return this.ones
}

func (this *Bitset) ToString() string {
	res := make([]byte, this.size)
	if !this.flipped {
		for i, b := range this.bits {
			if b == 1 {
				res[i] = '1'
			} else {
				res[i] = '0'
			}
		}
	} else {
		for i, b := range this.bits {
			if b == 0 {
				res[i] = '1'
			} else {
				res[i] = '0'
			}
		}
	}
	return string(res)
}
```

In Go, the implementation mirrors the Python version closely. Slices are used for the bit array, and `byte` is used to construct the string efficiently. The logic for flip, fix, and unfix is identical.
