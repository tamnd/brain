**Topics:** Array, Breadth-First Search, Sorting, Heap (Priority Queue), Matrix

## Solution

## Problem Understanding

The problem presents a **2D grid** representing a shop layout where each cell can be a wall (`0`), an empty space (`1`), or an item with a positive integer price. You are given a **start position** and a **price range** `[low, high]`. Your task is to find the **positions of the `k` highest-ranked items** within that price range according to a specific **ranking rule**:

1. Items closer to the starting point (shorter distance) are ranked higher.
2. Among items at the same distance, items with lower prices are ranked higher.
3. If both distance and price are equal, the item with the smaller row index is ranked higher.
4. If all above are equal, the item with the smaller column index is ranked higher.

The **input constraints** indicate that the grid can be as large as `10^5` cells, so any solution that iterates multiple times over the entire grid or uses inefficient searches could be too slow. The start position is guaranteed to be on a non-wall cell, and you always need to return up to `k` items if they exist, even if fewer are reachable.

Important **edge cases** include: the starting cell itself may contain an item in the price range, unreachable items surrounded by walls, or cases where there are fewer than `k` valid items. The grid may also contain large integer values for prices, but Python handles this natively without overflow concerns.

## Approaches

The **brute-force approach** would be to iterate over the entire grid, record every item within the price range, and then compute the distance from the starting point using a BFS or DFS for each item individually. After that, sort all items according to the ranking rules and return the top `k`. While this is correct, it is extremely inefficient because computing BFS for each item individually results in an `O(m * n * (m * n))` time complexity in the worst case.

The **optimal approach** leverages a **single BFS traversal** from the starting position. Since BFS explores nodes in increasing order of distance, you naturally compute the shortest path to each reachable cell. During the BFS, you collect all cells that contain items within the price range. Once BFS is complete, you sort the collected items based on price, row, and column, and return the top `k` entries. This approach is efficient because BFS ensures minimal traversal, and sorting is performed only on reachable items, not the entire grid.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((m*n)^2) | O(m*n) | BFS/DFS per item, inefficient for large grids |
| Optimal | O(m_n + r_log(r)) | O(m*n + r) | Single BFS, sort only reachable items `r` in price range |

## Algorithm Walkthrough

1. Initialize a queue for BFS starting at the `start` cell and a visited matrix to track explored cells.
2. Define BFS directions (up, down, left, right) to move between adjacent cells.
3. While the queue is not empty:

1. Dequeue the current cell and compute its distance from the start.
2. If the cell contains an item within the price range, store a tuple `(distance, price, row, col)` in a results list.
3. For each of the four directions, check if the neighboring cell is within bounds, is not a wall, and has not been visited. If so, mark it as visited and enqueue it with distance incremented by 1.
4. After BFS, sort the results list by `(distance, price, row, col)` to satisfy the ranking criteria.
5. Return the positions of the first `k` items from the sorted list. If fewer than `k` items exist, return all of them.

**Why it works:** BFS guarantees that the first time you reach a cell, it is through the shortest path, satisfying the distance ranking. Collecting all valid items during traversal ensures no reachable item is missed. Sorting afterward applies the secondary ranking criteria.

## Python Solution

```python
from collections import deque
from typing import List

class Solution:
    def highestRankedKItems(self, grid: List[List[int]], pricing: List[int], start: List[int], k: int) -> List[List[int]]:
        m, n = len(grid), len(grid[0])
        low, high = pricing
        sr, sc = start
        
        visited = [[False] * n for _ in range(m)]
        queue = deque([(sr, sc, 0)])
        visited[sr][sc] = True
        
        items = []
        
        directions = [(-1,0), (1,0), (0,-1), (0,1)]
        
        while queue:
            r, c, dist = queue.popleft()
            
            if low <= grid[r][c] <= high:
                items.append((dist, grid[r][c], r, c))
            
            for dr, dc in directions:
                nr, nc = r + dr, c + dc
                if 0 <= nr < m and 0 <= nc < n and not visited[nr][nc] and grid[nr][nc] != 0:
                    visited[nr][nc] = True
                    queue.append((nr, nc, dist + 1))
        
        items.sort()
        return [[r, c] for _, _, r, c in items[:k]]
```

**Implementation Walkthrough:** The BFS queue stores tuples of `(row, col, distance)`. The visited matrix ensures no cell is processed twice. Each valid item is appended to `items` during BFS, and final sorting applies all ranking rules automatically. Only the positions of the top `k` items are returned.

## Go Solution

```go
package main

func highestRankedKItems(grid [][]int, pricing []int, start []int, k int) [][]int {
    m, n := len(grid), len(grid[0])
    low, high := pricing[0], pricing[1]
    sr, sc := start[0], start[1]
    
    visited := make([][]bool, m)
    for i := 0; i < m; i++ {
        visited[i] = make([]bool, n)
    }
    visited[sr][sc] = true
    
    type Cell struct {
        r, c, dist int
    }
    
    queue := []Cell{{sr, sc, 0}}
    items := []Cell{}
    
    directions := [][2]int{{-1,0},{1,0},{0,-1},{0,1}}
    
    for len(queue) > 0 {
        cell := queue[0]
        queue = queue[1:]
        
        if grid[cell.r][cell.c] >= low && grid[cell.r][cell.c] <= high {
            items = append(items, cell)
        }
        
        for _, d := range directions {
            nr, nc := cell.r + d[0], cell.c + d[1]
            if nr >= 0 && nr < m && nc >= 0 && nc < n && !visited[nr][nc] && grid[nr][nc] != 0 {
                visited[nr][nc] = true
                queue = append(queue, Cell{nr, nc, cell.dist + 1})
            }
        }
    }
    
    // Sorting
    sort.Slice(items, func(i, j int) bool {
        if items[i].dist != items[j].dist {
            return items[i].dist < items[j].dist
        }
        if grid[items[i].r][items[i].c] != grid[items[j].r][items[j].c] {
            return grid[items[i].r][items[i].c] < grid[items[j].r][items[j].c]
        }
        if items[i].r != items[j].r {
            return items[i].r < items[j].r
        }
        return items[i].c < items[j].c
    })
    
    res := [][]int{}
    for i := 0; i < k && i < len(items); i++ {
        res = append(res, []int{items[i].r, items[i].c})
    }
    return res
}
```

**Go-specific Notes:** Go uses slices for queues and sort.Slice for custom sorting. The `Cell` struct encapsulates distance and coordinates. A visited 2D slice prevents revisiting cells. The sort function references `grid` directly to access prices.
