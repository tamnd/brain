## Solution

## Problem Understanding

The problem gives us a positive integer `num`, and asks us to count how many integers from `1` through `num` have an even digit sum.

The digit sum of a number is calculated by adding together all of its digits. For example:

- The digit sum of `24` is `2 + 4 = 6`
- The digit sum of `123` is `1 + 2 + 3 = 6`
- The digit sum of `19` is `1 + 9 = 10`

For each integer in the range `[1, num]`, we must determine whether its digit sum is even. If it is, we include it in the count.

The constraints are small:

- `1 <= num <= 1000`

This tells us that even a straightforward brute-force simulation will easily run within time limits. Since there are at most 1000 numbers to inspect, we can simply compute the digit sum for every number and count how many satisfy the condition.

There are still some important edge cases to think about:

- Very small inputs such as `num = 1`, where there may be no valid numbers.
- Numbers whose digit sums alternate parity frequently, such as around transitions like `9 -> 10`.
- Numbers containing multiple digits, where carrying changes the digit sum significantly.
- The upper boundary `num = 1000`, which ensures our implementation correctly handles four-digit numbers.

The problem guarantees that the input is always a positive integer, so we do not need to handle zero, negative numbers, or invalid input.

## Approaches

### Brute Force Approach

The most direct solution is to iterate through every number from `1` to `num`.

For each number:

1. Compute its digit sum.
2. Check whether the digit sum is even.
3. If it is even, increment the answer.

To compute the digit sum, we repeatedly extract the last digit using `% 10` and remove it using integer division `// 10`.

This approach is correct because it explicitly checks every candidate number and applies the exact condition required by the problem.

Even though this is called a brute-force solution, it is completely acceptable for the given constraints because `num` is at most `1000`.

### Optimal Observation

There is an interesting mathematical observation:

Among consecutive integers, digit sums tend to alternate parity. Roughly half of the numbers up to `num` have even digit sums.

A mathematical shortcut exists:

- If the digit sum of `num` is even, the answer is `num // 2`
- Otherwise, the answer is `(num - 1) // 2`

However, while this formula is elegant, the brute-force simulation is already efficient enough and often easier to understand.

For interview clarity and maintainability, many solutions still use direct simulation.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(num × d) | O(1) | Checks every number and computes its digit sum |
| Optimal | O(d) | O(1) | Uses parity observation about digit sums |

Here, `d` represents the number of digits in `num`.

## Algorithm Walkthrough

We will walk through the optimal mathematical solution.

### Step 1: Compute the digit sum of `num`

We first calculate the sum of all digits in `num`.

For example, if `num = 30`:

- `3 + 0 = 3`

### Step 2: Determine whether the digit sum is even

The parity of the digit sum determines the final count formula.

If the digit sum is even:

- Exactly `num // 2` numbers have even digit sums.

If the digit sum is odd:

- Exactly `(num - 1) // 2` numbers have even digit sums.

### Step 3: Return the computed count

We apply the correct formula and return the result.

### Why it works

Digit sum parity alternates in a predictable pattern across consecutive integers. Most neighboring integers flip the parity of the digit sum because adding `1` changes the total digit sum by `1`.

Although carries sometimes modify multiple digits, the global pattern still guarantees that exactly half the numbers up to a certain boundary have even digit sums, adjusted by the parity of the final number. This leads directly to the mathematical formula used in the optimal solution.

## Python Solution

```python
class Solution:
    def countEven(self, num: int) -> int:
        digit_sum = 0
        current = num

        while current > 0:
            digit_sum += current % 10
            current //= 10

        if digit_sum % 2 == 0:
            return num // 2

        return (num - 1) // 2
```

The implementation begins by computing the digit sum of `num`. We repeatedly extract the last digit using `% 10` and remove it using integer division `// 10`.

Once the digit sum is computed, we check its parity using `% 2`.

If the digit sum is even, we return `num // 2`. Otherwise, we return `(num - 1) // 2`.

The implementation uses only a few integer variables, so the space usage remains constant.

## Go Solution

```go
func countEven(num int) int {
    digitSum := 0
    current := num

    for current > 0 {
        digitSum += current % 10
        current /= 10
    }

    if digitSum%2 == 0 {
        return num / 2
    }

    return (num - 1) / 2
}
```

The Go implementation follows the same logic as the Python version.

Go uses standard integer division with `/` for integers, so no special handling is needed. Since the constraints are very small, integer overflow is not a concern. The solution uses only primitive integer variables and does not require slices, arrays, or additional data structures.

## Edge Cases

### Smallest Input Value

When `num = 1`, there are no positive integers less than or equal to `1` whose digit sum is even.

This case is important because some implementations accidentally include `0` in the count or incorrectly assume at least one valid answer exists. Our implementation correctly computes the digit sum of `1` as odd and returns `(1 - 1) // 2 = 0`.

### Carry Transitions

Numbers like `9`, `10`, `99`, and `100` are tricky because incrementing the number changes multiple digits at once.

For example:

- `9` has digit sum `9`
- `10` has digit sum `1`

A naive parity assumption could fail around these boundaries. The mathematical formula still works because it relies on the overall parity distribution rather than local transitions.

### Maximum Constraint

The largest allowed input is `1000`.

Its digit sum is:

```
1 + 0 + 0 + 0 = 1
```

Since the digit sum is odd, the answer becomes:

```
(1000 - 1) // 2 = 499
```

This case confirms that the implementation handles four-digit numbers correctly and remains efficient even at the upper bound.
