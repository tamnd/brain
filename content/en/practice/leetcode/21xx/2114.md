## Solution

## Problem Understanding

The problem asks us to determine the **maximum number of words in a single sentence** from an array of sentences. Each sentence is a string where words are separated by a single space, and there are no leading or trailing spaces. The input is an array of strings `sentences`, with each string representing a sentence, and the output should be a single integer representing the maximum word count found in any sentence.

Key points to note include the constraints: there can be at most 100 sentences, and each sentence has a length of at most 100 characters. Every sentence is properly formatted with no extra spaces, which simplifies parsing since splitting on spaces will correctly identify the words. The edge cases to consider include a sentence with only one word, multiple sentences with the same maximum number of words, and the smallest input of a single sentence with a single word.

## Approaches

### Brute Force

The brute-force approach would involve iterating over each sentence, splitting the sentence into words, counting the words, and keeping track of the maximum count. While this approach is straightforward and correct, it requires splitting every sentence into words, which creates additional arrays in memory for each sentence. Despite this, given the constraints (`sentences.length <= 100` and `sentences[i].length <= 100`), it is still efficient enough.

### Optimal Approach

The optimal approach does not need a more complex algorithm since the problem constraints are small. We can efficiently compute the number of words in a sentence by counting spaces and adding one. This avoids creating temporary arrays for the words, reducing memory overhead. The main idea is to iterate through each sentence, count the spaces, compute `words = spaces + 1`, and maintain the maximum count found.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m) | O(m) | Split each sentence into words and track max, n = number of sentences, m = max sentence length |
| Optimal | O(n * m) | O(1) | Count spaces directly without creating arrays |

## Algorithm Walkthrough

1. Initialize a variable `max_words` to `0` to keep track of the maximum number of words found.
2. Iterate through each sentence in the input array.
3. For each sentence, count the number of spaces. Since words are separated by single spaces, the total number of words is `spaces + 1`.
4. Compare this count with `max_words` and update `max_words` if the current sentence has more words.
5. After iterating through all sentences, return `max_words`.

**Why it works:** Counting spaces and adding one correctly computes the number of words because there are no leading or trailing spaces, and words are separated by exactly one space. Tracking the maximum as we go ensures that we return the largest count across all sentences.

## Python Solution

```python
from typing import List

class Solution:
    def mostWordsFound(self, sentences: List[str]) -> int:
        max_words = 0
        for sentence in sentences:
            word_count = sentence.count(' ') + 1
            if word_count > max_words:
                max_words = word_count
        return max_words
```

The Python implementation iterates over each sentence, counts the spaces using the built-in `count` method, computes the word count, and updates the maximum accordingly. This avoids unnecessary creation of intermediate lists and uses only a single integer variable for tracking.

## Go Solution

```go
func mostWordsFound(sentences []string) int {
    maxWords := 0
    for _, sentence := range sentences {
        wordCount := 1
        for i := 0; i < len(sentence); i++ {
            if sentence[i] == ' ' {
                wordCount++
            }
        }
        if wordCount > maxWords {
            maxWords = wordCount
        }
    }
    return maxWords
}
```

In Go, we manually iterate through each character of the sentence to count spaces and compute the word count. This avoids creating a slice from splitting the string, which is slightly more memory-efficient. Tracking the maximum uses a simple integer variable.

The time complexity is dominated by the iteration through each character in all sentences. No extra data structures proportional to input size are created, making space complexity constant.

## Edge Cases

The first edge case is a sentence with only one word. A naive implementation that counts spaces and does not add one would incorrectly return zero. Our implementation correctly accounts for this by adding one to the space count.

The second edge case is multiple sentences sharing the same maximum word count. The algorithm correctly returns the maximum value found, regardless of how many sentences share it, because it only tracks the largest count and not the sentence index.

The third edge case is a sentence at the maximum allowed length, potentially filled with words separated by spaces. The implementation efficiently counts spaces without creating extra lists, so it handles long sentences without excessive memory usage or slowdowns.
