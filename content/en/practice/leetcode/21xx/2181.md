## Solution

## Problem Understanding

This problem provides a singly linked list in which integers are separated by nodes with value `0`. The list **always starts and ends with a 0**, and there are no consecutive zeros inside the list, except for the edges. The task is to **collapse all nodes between each pair of zeros into a single node whose value is the sum of those nodes**, and then remove all zeros from the list.

In other words, if the linked list looks like `0 → 3 → 1 → 0 → 4 → 5 → 2 → 0`, the nodes between the first pair of zeros (3 and 1) are summed into `4`, and the nodes between the second pair of zeros (4, 5, 2) are summed into `11`. The resulting list is `4 → 11`.

The constraints tell us that the number of nodes is up to `2 * 10^5`, meaning an algorithm must be **linear time** and cannot rely on creating large auxiliary structures like arrays of sums unless it is O(n). Values of nodes are up to 1000, which is small enough to safely sum without worrying about integer overflow in Python. The guarantees about zeros simplify traversal because we do not need to check for consecutive zeros or empty sublists.

Important edge cases include lists with **single nodes between zeros** (like `0 → 1 → 0`) and lists where all nodes are minimal (`0 → 0`) or maximal in size.

## Approaches

The **brute-force approach** is to traverse the linked list and for each segment between zeros, collect values into an array or list. Then construct a new linked list using the sums of these segments. This works because we explicitly gather all numbers to sum them, but it requires **extra space proportional to the number of nodes** and an additional traversal to construct the new list.

The **optimal approach** leverages a single pass through the list, maintaining a running sum. Whenever we reach a node with value `0`, if the sum is non-zero, we **create a new node with the sum and append it to the new list**, then reset the sum. This requires **O(1) extra space** besides the new list nodes and only traverses the input once, making it both time and space efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Collect values in an array, sum segments, build a new list |
| Optimal | O(n) | O(1) | Single pass, maintain running sum, construct output list on the fly |

## Algorithm Walkthrough

1. **Initialize a dummy node** to serve as the head of the new list. This simplifies appending nodes without checking if the head exists.
2. **Create a pointer `current`** pointing to the dummy node. This will track the end of the new list as we append summed nodes.
3. **Initialize a variable `sum_segment`** to 0, which will store the sum of values between zeros.
4. **Traverse the input list** with a pointer `head`.
5. For each node, **check if its value is zero**:

- If it is zero and `sum_segment > 0`, **create a new node with value `sum_segment`**, attach it to `current.next`, advance `current`, and reset `sum_segment` to 0.
- If it is not zero, **add the node's value to `sum_segment`**.
6. **Continue until the end** of the input list.
7. **Return `dummy.next`** as the head of the new modified list.

**Why it works:** The running sum ensures that we correctly accumulate all values between zeros. Creating a new node only when a zero is encountered guarantees that each segment between zeros produces exactly one node. Using a dummy node simplifies handling the head pointer.

## Python Solution

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def mergeNodes(self, head: Optional[ListNode]) -> Optional[ListNode]:
        dummy = ListNode(0)
        current = dummy
        sum_segment = 0
        node = head.next  # skip the initial zero

        while node:
            if node.val == 0:
                if sum_segment > 0:
                    current.next = ListNode(sum_segment)
                    current = current.next
                    sum_segment = 0
            else:
                sum_segment += node.val
            node = node.next

        return dummy.next
```

In this implementation, we skip the initial zero (`head.next`) because it does not contribute to any sum. During traversal, we accumulate values until we encounter a zero. At that point, we create a new node if the sum is positive, ensuring that segments with no elements (which shouldn't exist per constraints) are ignored. Finally, `dummy.next` points to the head of the new list.

## Go Solution

```go
/**
 * Definition for singly-linked list.
 * type ListNode struct {
 *     Val int
 *     Next *ListNode
 * }
 */
func mergeNodes(head *ListNode) *ListNode {
    dummy := &ListNode{0, nil}
    current := dummy
    sumSegment := 0
    node := head.Next // skip initial zero

    for node != nil {
        if node.Val == 0 {
            if sumSegment > 0 {
                current.Next = &ListNode{Val: sumSegment}
                current = current.Next
                sumSegment = 0
            }
        } else {
            sumSegment += node.Val
        }
        node = node.Next
    }
    return dummy.Next
}
```

Go requires explicit handling of pointers, but the logic mirrors the Python implementation. `nil` represents the end of the list, and we allocate new `ListNode` structs for summed segments.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We traverse each node exactly once, summing values and occasionally creating new nodes |
| Space | O(1) | Only a few pointers and integer sums are stored; the new list uses output space but no extra data structures |

The time complexity is linear, and space complexity is constant relative to input size, which meets the problem's constraints for large inputs.
