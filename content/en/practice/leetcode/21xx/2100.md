## Solution

## Problem Understanding

The problem asks us to identify the days that are good for robbing a bank based on the number of guards on duty. We are given a 0-indexed array `security` where `security[i]` represents the number of guards present on the `i-th` day. A day `i` is considered a good day to rob the bank if the following conditions are satisfied:

1. There are at least `time` days before and after day `i`.
2. The number of guards for the `time` days before day `i` is non-increasing. This means that for every day `j` in the interval `[i-time, i]`, `security[j] >= security[j+1]`.
3. The number of guards for the `time` days after day `i` is non-decreasing. This means that for every day `k` in the interval `[i, i+time]`, `security[k] <= security[k+1]`.

The input `security` can have up to `10^5` elements and `time` can also be as large as `10^5`. Therefore, naive solutions that scan all `time` days before and after each potential day would be too slow. The output is a list of all good days, and the order does not matter.

Important edge cases include when `time` is `0`, in which every day qualifies as a good day, and when the length of `security` is less than `2 * time + 1`, where no day can satisfy the requirement.

## Approaches

### Brute Force

A brute-force solution would iterate over each day `i` in the array and check the non-increasing condition for the `time` days before `i` and the non-decreasing condition for the `time` days after `i`. If both conditions are satisfied, we add day `i` to the result list. This approach is correct but inefficient because it requires checking up to `2*time` elements for each day, resulting in a time complexity of O(n * time). Given the constraints, this would be too slow.

### Optimal Approach

The key observation is that we can preprocess the array to compute how many consecutive non-increasing days end at each index and how many consecutive non-decreasing days start at each index. Specifically, we can create two auxiliary arrays:

- `non_increasing[i]` stores the number of consecutive non-increasing days ending at index `i`.
- `non_decreasing[i]` stores the number of consecutive non-decreasing days starting at index `i`.

With these arrays, we can determine in constant time if day `i` satisfies the conditions: `non_increasing[i] >= time` and `non_decreasing[i] >= time`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * time) | O(1) | Checks each day individually, too slow for large inputs |
| Optimal | O(n) | O(n) | Precompute non-increasing and non-decreasing streaks, then select valid days |

## Algorithm Walkthrough

1. Initialize two arrays `non_increasing` and `non_decreasing` of size `n` where `n` is the length of `security`.
2. Traverse the `security` array from left to right. For each day `i > 0`, if `security[i] <= security[i-1]`, increment the non-increasing streak: `non_increasing[i] = non_increasing[i-1] + 1`. Otherwise, reset it to `0`.
3. Traverse the `security` array from right to left. For each day `i < n-1`, if `security[i] <= security[i+1]`, increment the non-decreasing streak: `non_decreasing[i] = non_decreasing[i+1] + 1`. Otherwise, reset it to `0`.
4. Iterate over all days `i` from `0` to `n-1`. If `non_increasing[i] >= time` and `non_decreasing[i] >= time`, add `i` to the result list.
5. Return the result list.

**Why it works**: The `non_increasing` array ensures that for each day, we know exactly how many previous consecutive non-increasing days exist, and `non_decreasing` does the same for the following days. By checking that both are at least `time`, we guarantee that the conditions of the problem are satisfied.

## Python Solution

```python
from typing import List

class Solution:
    def goodDaysToRobBank(self, security: List[int], time: int) -> List[int]:
        n = len(security)
        if time == 0:
            return list(range(n))
        
        non_increasing = [0] * n
        non_decreasing = [0] * n
        
        for i in range(1, n):
            if security[i] <= security[i-1]:
                non_increasing[i] = non_increasing[i-1] + 1
        
        for i in range(n-2, -1, -1):
            if security[i] <= security[i+1]:
                non_decreasing[i] = non_decreasing[i+1] + 1
        
        result = []
        for i in range(n):
            if non_increasing[i] >= time and non_decreasing[i] >= time:
                result.append(i)
        
        return result
```

**Explanation**: The solution first handles the edge case where `time` is `0`. It then builds two arrays to track non-increasing and non-decreasing streaks. Finally, it iterates over the days to collect all good days.

## Go Solution

```go
func goodDaysToRobBank(security []int, time int) []int {
    n := len(security)
    if time == 0 {
        result := make([]int, n)
        for i := 0; i < n; i++ {
            result[i] = i
        }
        return result
    }

    nonIncreasing := make([]int, n)
    nonDecreasing := make([]int, n)

    for i := 1; i < n; i++ {
        if security[i] <= security[i-1] {
            nonIncreasing[i] = nonIncreasing[i-1] + 1
        }
    }

    for i := n-2; i >= 0; i-- {
        if security[i] <= security[i+1] {
            nonDecreasing[i] = nonDecreasing[i+1] + 1
        }
    }

    result := []int{}
    for i := 0; i < n; i++ {
        if nonIncreasing[i] >= time && nonDecreasing[i] >= time {
            result = append(result, i)
        }
    }

    return result
}
```

**Go-specific notes**: Go requires explicit slice initialization for the result and the auxiliary arrays. The iteration logic mirrors Python, and integer overflow is not a concern since the constraints are within the 32-bit integer range.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We traverse the array three times: left-to-right, right-to-left, and once more to collect results. |
| Space | O(n) | Two auxiliary arrays of size n are used to store streaks. |

This linear time and space solution scales efficiently to the problem constraints.
