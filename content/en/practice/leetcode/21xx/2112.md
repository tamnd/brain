## Solution

## Problem Understanding

The problem provides a database table named `Flights`. Each row represents a direct connection between two airports and the number of flights that traveled along that route.

The table contains three columns:

| Column | Meaning |
| --- | --- |
| `departure_airport` | The airport where flights started |
| `arrival_airport` | The airport where flights ended |
| `flights_count` | Number of flights on that route |

The primary key is `(departure_airport, arrival_airport)`, which guarantees that every airport pair appears at most once.

The task is to determine which airport has the highest total traffic. Traffic for an airport is defined as:

- All flights departing from the airport
- Plus all flights arriving at the airport

In other words, every route contributes its `flights_count` value to two airports:

- The departure airport
- The arrival airport

After computing the total traffic for every airport, we must return all airport IDs whose traffic equals the maximum traffic value.

The result may contain one airport or multiple airports if several are tied for the highest traffic.

The problem is fundamentally an aggregation problem. Every row contributes values to multiple entities, and we must compute totals efficiently.

Several important edge cases must be considered:

- Multiple airports may tie for maximum traffic
- Some airports may appear only as departure airports
- Some airports may appear only as arrival airports
- The same airport may participate in many routes
- There may be only one row in the table
- Airports with zero participation never appear in the input, so we do not need to generate missing airport IDs

Because each flight contributes to exactly two airports, the most natural solution is to aggregate traffic counts using grouping operations.

## Approaches

### Brute Force Approach

A brute force solution would first collect all unique airport IDs from both the departure and arrival columns.

Then, for every airport, we would scan the entire `Flights` table and sum:

- All `flights_count` values where the airport appears as `departure_airport`
- All `flights_count` values where the airport appears as `arrival_airport`

After computing totals for all airports, we would determine the maximum value and return all matching airports.

This approach is correct because it explicitly checks every row for every airport, ensuring that all traffic contributions are counted.

However, this method is inefficient because the table is scanned repeatedly. If there are `N` flights and `M` unique airports, the complexity becomes `O(M × N)`. In the worst case, `M` can be proportional to `N`, leading to quadratic behavior.

### Optimal Approach

The key observation is that every row contributes traffic to exactly two airports:

- The departure airport gains `flights_count`
- The arrival airport also gains `flights_count`

Instead of recomputing totals repeatedly, we can aggregate all contributions in a single pass.

In SQL, this is naturally handled by:

1. Transforming each flight row into two records:

- One for departures
- One for arrivals
2. Combining them with `UNION ALL`
3. Grouping by airport ID
4. Summing all traffic values
5. Finding the maximum total
6. Returning airports whose total equals that maximum

This reduces the work to essentially linear aggregation over the rows.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(M × N) | O(M) | Repeatedly scans all flights for every airport |
| Optimal | O(N) | O(M) | Single aggregation pass using grouping |

Here:

- `N` = number of rows in `Flights`
- `M` = number of unique airports

## Algorithm Walkthrough

### Optimal SQL Aggregation Algorithm

1. First, create a temporary dataset representing all airport traffic contributions.

For every row in `Flights`:

- Add one record for the departure airport
- Add another record for the arrival airport

Both records carry the same `flights_count`.
2. Combine all contribution records using `UNION ALL`.

`UNION ALL` is important because duplicate traffic values must not be removed. Every flight contribution matters independently.
3. Group the combined dataset by airport ID.

This allows us to compute total traffic per airport.
4. Use `SUM(flights_count)` to calculate each airport's total traffic.
5. Find the maximum traffic value across all airports.
6. Return all airport IDs whose traffic equals the maximum value.

### Why it works

Every flight contributes exactly once to the departure airport and exactly once to the arrival airport. By explicitly generating both contributions and grouping by airport ID, we guarantee that every airport receives the correct total traffic count. Since we later filter by the global maximum, the returned airports are precisely those with the highest traffic.

## Python Solution

```python
# Write your MySQL query statement below

WITH airport_traffic AS (
    SELECT 
        departure_airport AS airport_id,
        flights_count
    FROM Flights
    
    UNION ALL
    
    SELECT 
        arrival_airport AS airport_id,
        flights_count
    FROM Flights
),
traffic_totals AS (
    SELECT
        airport_id,
        SUM(flights_count) AS total_traffic
    FROM airport_traffic
    GROUP BY airport_id
)

SELECT airport_id
FROM traffic_totals
WHERE total_traffic = (
    SELECT MAX(total_traffic)
    FROM traffic_totals
);
```

The solution begins by constructing the `airport_traffic` common table expression.

This CTE converts every flight row into two traffic contributions:

- One contribution for the departure airport
- One contribution for the arrival airport

`UNION ALL` is used instead of `UNION` because duplicate rows must remain intact. Multiple flights with the same counts are still separate contributions.

Next, the `traffic_totals` CTE groups all rows by `airport_id` and computes the total traffic using `SUM(flights_count)`.

Finally, the outer query identifies the maximum traffic value and returns every airport whose total equals that maximum.

This approach is concise, efficient, and directly models the problem definition.

## Edge Cases

### Multiple Airports Tie for Maximum Traffic

A common mistake is returning only one airport after finding the maximum traffic value. The problem explicitly requires returning all airports tied for the maximum.

The implementation handles this correctly by filtering with:

```sql
WHERE total_traffic = (
    SELECT MAX(total_traffic)
    FROM traffic_totals
)
```

This condition naturally returns every matching airport.

### Airports Appearing Only in One Column

Some airports may appear only as departure airports or only as arrival airports.

For example:

| departure_airport | arrival_airport | flights_count |
| --- | --- | --- |
| 5 | 6 | 7 |

Airport `5` appears only in departures, while airport `6` appears only in arrivals.

The solution handles this correctly because both columns are independently projected into the combined dataset before aggregation.

### Duplicate Traffic Counts

Using `UNION` instead of `UNION ALL` would introduce a subtle bug.

`UNION` removes duplicates, which could accidentally discard legitimate traffic contributions if multiple rows produce identical projected values.

Using `UNION ALL` preserves every contribution exactly once, ensuring correct totals.
