## Solution

## Problem Understanding

The problem asks us to simulate a chain reaction of detonating bombs placed on a 2D plane. Each bomb has a specific location and a circular range that defines which other bombs it can detonate directly. The goal is to determine the maximum number of bombs that can be detonated if we are allowed to manually detonate only one bomb.

The input is a list of bombs, where each bomb is represented by three integers `[xi, yi, ri]`. Here, `(xi, yi)` is the bomb's location on the 2D plane, and `ri` is the radius of its effect. When a bomb detonates, it can detonate any other bomb whose center lies within or on its circular range. Detonations can cascade: a bomb triggered by another can further trigger others.

The output is a single integer representing the largest number of bombs that can be detonated from any single initial choice.

Constraints tell us the input can be up to 100 bombs with coordinates and ranges up to 100,000. This indicates that a solution with a time complexity significantly worse than `O(n^2)` may be too slow, but `O(n^2)` approaches are likely feasible. Edge cases include bombs with minimal or maximal ranges, bombs that are all isolated, or bombs that are tightly clustered so that a single detonation triggers a large cascade. The problem guarantees that each bomb has a valid location and radius, so we do not need to handle invalid input.

## Approaches

A **brute-force approach** would consider each bomb as a starting point and recursively or iteratively check which bombs it can trigger. For each bomb, you would check all others to see if they fall within the circular range and continue detonating recursively. While correct, this approach has a worst-case complexity of `O(n^3)` if using naive recursion, because for each bomb you might examine all others and propagate recursively. This is acceptable for very small inputs but becomes inefficient for 100 bombs.

The **optimal approach** is to model the problem as a **directed graph**. Each bomb is a node, and there is an edge from bomb `i` to bomb `j` if bomb `j` lies within bomb `i`’s range. Once the graph is built, the problem reduces to finding the largest number of reachable nodes starting from any node, which can be efficiently done using Depth-First Search (DFS) or Breadth-First Search (BFS). Constructing the graph takes `O(n^2)` time, and each DFS/BFS traversal is `O(n)` in the worst case. Thus, the total time complexity is `O(n^3)` in the worst-case scenario, which is acceptable for n ≤ 100.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n) | Recursively checks all chains without precomputing relationships |
| Graph + DFS/BFS | O(n^3) | O(n^2) | Precomputes reachable edges for efficiency and avoids redundant calculations |

## Algorithm Walkthrough

1. **Graph Construction**: Iterate over each pair of bombs `(i, j)` and check if bomb `j` lies in the range of bomb `i`. If it does, add a directed edge from `i` to `j` in an adjacency list. This represents all bombs that `i` can directly detonate.
2. **DFS Traversal**: Define a recursive DFS function that, given a starting bomb, counts all bombs that can be reached (detonated) from it. Use a `visited` set to avoid counting the same bomb multiple times and to prevent infinite recursion.
3. **Evaluate Maximum**: For each bomb as a starting point, perform DFS and record the number of bombs detonated. Keep track of the maximum count across all starting bombs.
4. **Return Result**: After evaluating all bombs, return the maximum number of bombs detonated.

**Why it works**: By representing detonations as directed edges in a graph, we capture the full cascade effect of detonations. DFS ensures all reachable bombs are counted exactly once, giving the correct maximum.

## Python Solution

```python
from typing import List

class Solution:
    def maximumDetonation(self, bombs: List[List[int]]) -> int:
        n = len(bombs)
        graph = [[] for _ in range(n)]
        
        # Step 1: Build the directed graph
        for i in range(n):
            xi, yi, ri = bombs[i]
            for j in range(n):
                if i != j:
                    xj, yj, _ = bombs[j]
                    if (xi - xj) ** 2 + (yi - yj) ** 2 <= ri ** 2:
                        graph[i].append(j)
        
        # Step 2: DFS function
        def dfs(node, visited):
            visited.add(node)
            count = 1
            for nei in graph[node]:
                if nei not in visited:
                    count += dfs(nei, visited)
            return count
        
        # Step 3: Evaluate maximum detonation
        max_detonated = 0
        for i in range(n):
            max_detonated = max(max_detonated, dfs(i, set()))
        
        return max_detonated
```

The code first constructs a graph where each node points to bombs within its radius. The DFS function recursively explores each bomb's neighbors, counting the total number of bombs detonated. Finally, we iterate through each bomb as a potential starting point to find the maximum.

## Go Solution

```go
func maximumDetonation(bombs [][]int) int {
    n := len(bombs)
    graph := make([][]int, n)
    
    // Step 1: Build the directed graph
    for i := 0; i < n; i++ {
        xi, yi, ri := bombs[i][0], bombs[i][1], bombs[i][2]
        for j := 0; j < n; j++ {
            if i != j {
                xj, yj := bombs[j][0], bombs[j][1]
                dx, dy := xi-xj, yi-yj
                if dx*dx+dy*dy <= ri*ri {
                    graph[i] = append(graph[i], j)
                }
            }
        }
    }
    
    // Step 2: DFS function
    var dfs func(int, map[int]bool) int
    dfs = func(node int, visited map[int]bool) int {
        visited[node] = true
        count := 1
        for _, nei := range graph[node] {
            if !visited[nei] {
                count += dfs(nei, visited)
            }
        }
        return count
    }
    
    // Step 3: Evaluate maximum detonation
    maxDetonated := 0
    for i := 0; i < n; i++ {
        visited := make(map[int]bool)
        maxDetonated = max(maxDetonated, dfs(i, visited))
    }
    
    return maxDetonated
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

In Go, we use slices for adjacency lists and maps for visited tracking. Integer arithmetic is safe given the constraints, and we perform the same DFS-based traversal to count detonations.
