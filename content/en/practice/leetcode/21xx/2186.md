## Solution

## Problem Understanding

The problem gives us two strings, `s` and `t`, consisting only of lowercase English letters. In one operation, we are allowed to append any single character to either string. Our goal is to determine the minimum number of append operations required so that the two resulting strings become anagrams of each other.

Two strings are anagrams if they contain exactly the same characters with the same frequencies, regardless of order. For example, `"listen"` and `"silent"` are anagrams because both contain the same letters the same number of times.

A very important detail is that we are only allowed to append characters. We cannot delete characters, replace characters, or reorder characters directly. Since appending is unrestricted, the final anagram will effectively contain every character that appears in either string, with enough copies to satisfy the larger frequency requirement for each character.

The constraints are large:

- `1 <= s.length, t.length <= 2 * 10^5`
- Only lowercase English letters are used

These constraints immediately rule out expensive algorithms such as generating permutations or repeatedly scanning strings. We need a solution that runs in linear time.

The lowercase English alphabet contains only 26 characters, which is an important observation. It means we can efficiently count character frequencies using either a hash map or a fixed-size array.

Several edge cases are important to consider:

- The strings may already be anagrams, in which case the answer is `0`.
- One string may contain many repeated characters while the other contains none of them.
- One string may be much longer than the other.
- All characters may be completely different.
- Very large input sizes require avoiding nested loops or repeated string reconstruction.

The problem guarantees that both strings are non-empty and contain only lowercase English letters, so we do not need to handle invalid characters or empty input arrays.

## Approaches

### Brute Force Approach

A brute-force strategy would attempt to directly construct matching anagrams by repeatedly comparing the strings and appending missing characters one at a time.

One possible implementation would repeatedly:

1. Count character frequencies in both strings.
2. Find characters missing from one string.
3. Append them.
4. Repeat until the strings become anagrams.

This approach is correct because eventually both strings will contain matching frequencies for every character. However, repeatedly recomputing frequencies after every append operation is extremely inefficient.

If the strings are large, repeatedly scanning them leads to quadratic behavior. Since the input size can reach `2 * 10^5`, this would be far too slow.

The key inefficiency is that we do not actually need to simulate the append operations. We only need to know how many characters are missing between the two frequency distributions.

### Optimal Approach

The crucial observation is that for every character, the difference in frequency between `s` and `t` tells us exactly how many append operations are required.

For example:

- If `s` contains 5 copies of `'a'`
- And `t` contains 2 copies of `'a'`

Then we must append 3 `'a'` characters to `t`.

Likewise, if `t` has more copies of another character, we append the difference to `s`.

Therefore, for every character:

```
required steps = abs(freq_s - freq_t)
```

The final answer is simply the sum of these absolute differences across all 26 letters.

Because the alphabet size is constant, we can compute this efficiently in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly recomputes frequencies and simulates appends |
| Optimal | O(n + m) | O(1) | Counts frequency differences once and sums absolute values |

## Algorithm Walkthrough

1. Create a frequency counter for the characters in `s`.

We need to know how many times each character appears. Since the input only contains lowercase English letters, we can use an array of size 26 where each index corresponds to a letter from `'a'` to `'z'`.
2. Traverse string `s` and increment counts.

For every character in `s`, compute its array index using:

```
ord(char) - ord('a')
```

Increment the corresponding frequency.
3. Traverse string `t` and decrement counts.

Instead of creating a second array, we can reuse the same array. For every character in `t`, decrement the corresponding frequency.

After this step:

- Positive values mean `s` has extra copies.
- Negative values mean `t` has extra copies.
- Zero means frequencies already match.
4. Compute the total number of required append operations.

Iterate through all 26 entries and sum their absolute values.

Each absolute difference directly represents how many characters must be appended to balance that letter's frequency.
5. Return the total.

This total is the minimum number of operations because every mismatch must be corrected exactly once.

### Why it works

The algorithm works because anagrams require identical frequency counts for every character. For each letter, the absolute difference between the two frequencies represents the exact number of missing characters that must be appended to one side. Since character frequencies are independent of one another, summing all absolute differences gives the minimum total operations required.

## Python Solution

```python
class Solution:
    def minSteps(self, s: str, t: str) -> int:
        frequency = [0] * 26

        for char in s:
            frequency[ord(char) - ord('a')] += 1

        for char in t:
            frequency[ord(char) - ord('a')] -= 1

        steps = 0

        for count in frequency:
            steps += abs(count)

        return steps
```

The implementation starts by creating a fixed-size array named `frequency` with 26 elements. Each index corresponds to a lowercase English letter.

The first loop processes string `s` and increments the count for each character. The second loop processes string `t` and decrements the count for each character. By combining both operations into a single array, we efficiently track the frequency differences between the two strings.

After both traversals, each array element represents the imbalance for that letter. Positive values indicate extra copies in `s`, while negative values indicate extra copies in `t`.

The final loop sums the absolute values of all frequency differences. This directly computes the minimum number of append operations required.

The solution is efficient because every character is processed exactly once.

## Go Solution

```go
func minSteps(s string, t string) int {
    frequency := make([]int, 26)

    for _, char := range s {
        frequency[char-'a']++
    }

    for _, char := range t {
        frequency[char-'a']--
    }

    steps := 0

    for _, count := range frequency {
        if count < 0 {
            steps -= count
        } else {
            steps += count
        }
    }

    return steps
}
```

The Go implementation follows the same logic as the Python version. A slice of length 26 is used for frequency counting.

One small difference is that Go does not have a built-in `abs()` function for integers in the standard library without importing additional packages, so the code manually checks whether the count is negative.

Since the alphabet size is fixed, memory usage remains constant.

## Edge Cases

One important edge case occurs when the strings are already anagrams. In this scenario, every character frequency matches exactly, so all entries in the frequency array become zero. The algorithm correctly returns `0` because the sum of all absolute differences is zero.

Another important case is when the strings contain completely different characters. For example, `"abc"` and `"def"` share no letters at all. Every character contributes a frequency mismatch of `1`, resulting in a total answer of `6`. A naive implementation might incorrectly try to replace characters, but replacements are not allowed. Our frequency-difference method correctly models only append operations.

A third edge case involves highly unbalanced frequencies, such as `"aaaaa"` and `"a"`. Here, the algorithm must recognize that only the missing copies need to be appended. The frequency array stores a difference of `4` for `'a'`, and the absolute value calculation correctly returns `4`.

Another subtle case is when the strings have the same letters in different orders, such as `"listen"` and `"silent"`. Since anagrams depend only on frequencies and not positions, the algorithm correctly ignores ordering entirely and returns `0`.

Finally, very large inputs are important because inefficient solutions can time out. Since this implementation processes each character once and uses only a fixed-size array, it handles the maximum constraint efficiently.
