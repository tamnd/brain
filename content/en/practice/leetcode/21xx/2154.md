## Solution

## Problem Understanding

The problem is asking us to repeatedly search for a number in an array and, if found, double it, continuing this process until the number is no longer present in the array. Specifically, you are given an array of integers `nums` and an initial integer `original`. The task is to determine the final value of `original` after applying the repeated doubling process.

The input `nums` represents a collection of numbers where each number can potentially be used to continue the doubling chain. The integer `original` is the starting value we will try to find in the array. The expected output is a single integer, which is the last value of `original` after it can no longer be found in `nums`.

The constraints tell us that the array has at most 1000 elements and that the numbers themselves are also in the range [1, 1000]. This means the problem can be solved with simple data structures such as arrays or hash sets without worrying about performance limitations. Important edge cases include `original` not being in `nums` at all, `original` appearing multiple times, or doubling beyond the maximum element in the array.

## Approaches

The brute-force approach is straightforward: we repeatedly scan the array for `original`. If we find it, we multiply it by 2 and repeat. This guarantees correctness because we explicitly check each number in the array at every step, but it is inefficient. For an array of size `n`, if `original` doubles `k` times, this approach has time complexity O(n * k), which is acceptable here but unnecessary.

The optimal approach uses a hash set to store the numbers in `nums`. By converting the array into a set, we reduce each lookup of `original` from O(n) to O(1). The process is then simply checking membership in the set and doubling `original` until it is no longer found. This observation leverages the fact that we only need fast lookups and do not need the array to remain ordered.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * k) | O(1) | Scan array for `original` repeatedly, multiply if found |
| Optimal | O(n + k) | O(n) | Use a hash set for O(1) membership checks, multiply until not found |

## Algorithm Walkthrough

1. Convert the input array `nums` into a hash set. This allows for O(1) membership checks for any number.
2. Initialize the current number as `original`.
3. While the current number exists in the hash set, multiply it by 2.
4. Once the current number is not in the hash set, return it as the final value.

Why it works: The algorithm works because it directly implements the process described in the problem. At each step, the invariant is that the current number has been found in the set exactly once per iteration. Doubling continues until the number is not in the set, guaranteeing that the last returned value cannot appear in the array.

## Python Solution

```python
from typing import List

class Solution:
    def findFinalValue(self, nums: List[int], original: int) -> int:
        nums_set = set(nums)
        while original in nums_set:
            original *= 2
        return original
```

In this Python implementation, we first convert `nums` to a set to allow O(1) membership checks. Then we repeatedly check if `original` is in `nums_set`, doubling it if it is. When it is no longer present, we return `original`. This matches the algorithm walkthrough exactly.

## Go Solution

```go
func findFinalValue(nums []int, original int) int {
    numsSet := make(map[int]struct{})
    for _, num := range nums {
        numsSet[num] = struct{}{}
    }
    for {
        if _, found := numsSet[original]; !found {
            break
        }
        original *= 2
    }
    return original
}
```

In Go, we use a map with empty struct values to emulate a hash set. We iterate over `nums` to populate `numsSet`. The main loop continues to double `original` while it is found in the map. Once it is not found, we return the current value.

The time complexity is dominated by the set creation and the number of doublings. Space complexity is O(n) due to the hash set.

## Edge Cases

One important edge case is when `nums` is empty. In this scenario, `original` can never be found, so the correct output is `original` itself. The implementation handles this correctly because the while loop immediately exits.

Another edge case is when `original` is larger than any element in `nums`. Since `original` is not in the array, the function should return `original` unchanged. Our solution handles this naturally due to the set membership check.

A third edge case occurs when `original` undergoes multiple doublings that exceed the largest value in `nums`. For example, if `nums = [1,2,4,8]` and `original = 1`, it doubles to 2, then 4, then 8, then 16, which is not in the array. The implementation correctly returns 16. This tests both correctness of the doubling loop and the termination condition.
