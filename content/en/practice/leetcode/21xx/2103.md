## Solution

## Problem Understanding

The problem gives us a string called `rings` that encodes information about colored rings placed on rods. Every two characters in the string represent one ring:

- The first character is the color of the ring:

- `'R'` for red
- `'G'` for green
- `'B'` for blue
- The second character is the rod number, from `'0'` to `'9'`

Since rods are labeled only from `0` to `9`, there are exactly 10 possible rods.

For example:

```
"B0R1G0"
```

means:

- A blue ring is placed on rod 0
- A red ring is placed on rod 1
- A green ring is placed on rod 0

Our task is to determine how many rods contain all three colors, red, green, and blue.

The important observation is that multiple rings of the same color may appear on the same rod. We only care whether a rod has at least one ring of each color.

The constraints are very small:

- `1 <= n <= 100`
- `rings.length == 2 * n`

This means there are at most 100 rings, so performance is not a major concern. Even relatively inefficient solutions would work. However, the problem is designed to test clean use of hash tables or sets.

Several edge cases are important:

- A rod may contain repeated colors, such as multiple red rings
- Some rods may contain only one or two colors
- The input may contain only a single ring
- The rings are not guaranteed to appear in any sorted order
- A rod should only be counted once, even if it contains many rings of all colors

Because the input format is guaranteed to be valid, we do not need to handle malformed strings or invalid colors.

## Approaches

### Brute Force Approach

A straightforward solution is to examine every rod from `0` to `9` and scan the entire input string to determine whether that rod contains red, green, and blue rings.

For each rod:

1. Initialize three boolean variables:

- hasRed
- hasGreen
- hasBlue
2. Traverse the `rings` string two characters at a time
3. Whenever the current ring belongs to the rod being checked, update the corresponding color flag
4. After scanning the whole string, count the rod if all three flags are true

This works because we explicitly verify the presence of all colors for every rod.

However, this solution repeatedly scans the entire input for every rod. Since there are 10 rods and up to 100 rings, the total work becomes proportional to `10 * n`.

Although this is still acceptable for the given constraints, we can do better with a cleaner and more direct approach.

### Optimal Approach

The key insight is that we only need to track which colors appear on each rod.

Instead of repeatedly scanning the input, we process the string once and build a data structure that records the colors associated with each rod.

A natural choice is:

- A hash map where:

- key = rod number
- value = set of colors seen on that rod

As we process each ring:

- Extract the color
- Extract the rod
- Add the color to the set for that rod

At the end, any rod whose set size is `3` contains all three colors.

This approach is efficient because:

- Each ring is processed exactly once
- Set insertion is constant time
- Checking the final set sizes is simple

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(10 × n) | O(1) | Repeatedly scans the entire input for each rod |
| Optimal | O(n) | O(10) | Tracks colors per rod using sets |

## Algorithm Walkthrough

1. Create a hash map where each rod maps to a set of colors seen on that rod.

We use sets because duplicate colors do not matter. If a rod already has a red ring, adding another red ring changes nothing.
2. Traverse the string two characters at a time.

Since every pair represents one ring, we process indices:

```
0, 2, 4, 6, ...
```
3. For each pair:

- The current character is the color
- The next character is the rod number

Example:

```
"B0"
```

means blue on rod 0.
4. Insert the color into the set corresponding to that rod.

After processing all rings, each rod's set contains the unique colors placed on it.
5. Iterate through all rod sets and count how many contain exactly three colors.

A set size of `3` means the rod contains:

- Red
- Green
- Blue
6. Return the final count.

### Why it works

The algorithm maintains the invariant that, after processing each ring, the set for every rod contains exactly the distinct colors placed on that rod so far.

Because every ring is processed once and inserted into the appropriate set, the final sets accurately represent all colors present on each rod. Therefore, counting rods whose set size equals `3` correctly counts rods containing all three colors.

## Python Solution

```python
class Solution:
    def countPoints(self, rings: str) -> int:
        rod_colors = {}

        for i in range(0, len(rings), 2):
            color = rings[i]
            rod = rings[i + 1]

            if rod not in rod_colors:
                rod_colors[rod] = set()

            rod_colors[rod].add(color)

        count = 0

        for colors in rod_colors.values():
            if len(colors) == 3:
                count += 1

        return count
```

The solution begins by creating a dictionary called `rod_colors`. Each key is a rod number, and each value is a set storing the colors found on that rod.

The loop processes the string in increments of two because every pair of characters represents one ring. The first character is extracted as the color, and the second as the rod.

If the rod does not yet exist in the dictionary, a new empty set is created for it. The current color is then inserted into the set.

Using a set is important because duplicate colors should not increase the count of unique colors on a rod.

After processing all rings, the algorithm iterates through every set in the dictionary. Any rod whose set size equals `3` contains all three colors, so the answer counter is incremented.

Finally, the total count is returned.

## Go Solution

```go
func countPoints(rings string) int {
    rodColors := make(map[byte]map[byte]bool)

    for i := 0; i < len(rings); i += 2 {
        color := rings[i]
        rod := rings[i+1]

        if rodColors[rod] == nil {
            rodColors[rod] = make(map[byte]bool)
        }

        rodColors[rod][color] = true
    }

    count := 0

    for _, colors := range rodColors {
        if len(colors) == 3 {
            count++
        }
    }

    return count
}
```

The Go implementation uses a nested map structure because Go does not have a built in set type.

The outer map associates each rod with another map representing the set of colors seen on that rod. The inner map stores colors as keys with boolean values.

Before inserting a color, the code checks whether the rod already has an initialized inner map. If not, it creates one.

The rest of the logic matches the Python solution closely.

There are no concerns about integer overflow because the input size is extremely small. Empty maps behave safely in Go once initialized properly.

## Edge Cases

One important edge case is when the input contains only a single ring, such as `"G4"`. A naive implementation might incorrectly assume every rod eventually receives multiple colors. Our solution handles this correctly because the set for rod `4` contains only one element, so its size is not `3`.

Another important case is repeated colors on the same rod. For example:

```
"R0R0R0G0B0"
```

If we used a list instead of a set, duplicate red rings could incorrectly affect the logic. Using sets guarantees that each color is counted only once per rod.

A third important edge case occurs when colors are distributed across multiple rods instead of concentrated on one rod. For example:

```
"R0G1B2"
```

All three colors exist in the input, but no single rod contains all three. Our implementation correctly separates colors by rod using the dictionary structure, so no rod is counted incorrectly.

Another subtle case is when multiple rods are valid simultaneously. For example:

```
"R0G0B0R1G1B1"
```

The algorithm independently tracks colors for every rod, allowing it to correctly count multiple complete rods without interference between them.
