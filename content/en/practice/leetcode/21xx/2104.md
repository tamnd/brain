## Solution

## Problem Understanding

The problem asks us to calculate the sum of ranges of all subarrays of a given integer array `nums`. A subarray is defined as a contiguous non-empty segment of the array, and the **range** of a subarray is the difference between its largest and smallest elements. The input is an array of integers where `1 <= nums.length <= 1000` and each element lies in the range `-10^9` to `10^9`. The output is a single integer representing the sum of the ranges of all possible subarrays.

The brute-force interpretation is straightforward: generate all possible subarrays, find the maximum and minimum values for each, compute the difference, and accumulate the sum. However, this naive approach can be very slow because the number of subarrays grows quadratically with the size of the array, and finding max/min for each subarray adds further overhead.

Important constraints and edge considerations include arrays of length 1, arrays with all identical elements, and arrays containing negative numbers. The problem also hints at a follow-up: achieving an **O(n)** solution, which suggests using a monotonic stack or similar technique to efficiently count the contribution of each element as a minimum and maximum.

## Approaches

### Brute-Force Approach

The brute-force approach involves iterating over all possible subarrays using nested loops. For each subarray, we calculate the maximum and minimum values and compute their difference to get the range. We sum up these ranges to obtain the final result.

While this approach is simple and guarantees correctness, it has a time complexity of **O(n³)** if we recompute max and min for each subarray or **O(n²)** if we track max/min incrementally. For `n` up to 1000, this can result in up to a million operations, which is inefficient.

### Optimal Approach

The optimal approach leverages the insight that we can compute the contribution of each element to all subarray ranges where it acts as the maximum or minimum. Specifically:

- For each element, determine how many subarrays it is the **maximum** in and how many subarrays it is the **minimum** in.
- Use **monotonic stacks** to efficiently calculate the nearest previous and next elements that are larger or smaller, which allows computing these counts.
- The contribution of an element is `(count_as_max * value) - (count_as_min * value)`. Summing these contributions yields the total sum of all subarray ranges.

This approach reduces the problem from examining every subarray individually to evaluating each element’s contribution, achieving **O(n)** time with **O(n)** additional space for the stacks.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Iterate over all subarrays and calculate max-min for each |
| Optimal (Monotonic Stack) | O(n) | O(n) | Use monotonic stacks to compute each element's contribution as max and min |

## Algorithm Walkthrough

1. Initialize `n` as the length of `nums`.
2. Create arrays `next_greater`, `prev_greater`, `next_smaller`, `prev_smaller` to store indices for each element representing the nearest greater or smaller elements to the left and right.
3. Use a **monotonic increasing stack** to fill `next_smaller` and `prev_smaller`, computing for each element how many subarrays it serves as the minimum.
4. Use a **monotonic decreasing stack** to fill `next_greater` and `prev_greater`, computing for each element how many subarrays it serves as the maximum.
5. For each element at index `i`, compute its contribution as `nums[i] * (i - prev_greater[i]) * (next_greater[i] - i)` for max and `nums[i] * (i - prev_smaller[i]) * (next_smaller[i] - i)` for min.
6. Sum the contributions of all elements as `(sum_max - sum_min)` to obtain the final sum of all subarray ranges.

**Why it works:** Each element’s total contribution as a maximum or minimum can be counted without explicitly generating all subarrays. By counting the number of subarrays that include the element as max or min using the distance to the previous and next greater or smaller elements, we capture the effect of that element on all relevant subarrays, which guarantees correctness.

## Python Solution

```python
from typing import List

class Solution:
    def subArrayRanges(self, nums: List[int]) -> int:
        n = len(nums)
        next_greater = [n] * n
        prev_greater = [-1] * n
        next_smaller = [n] * n
        prev_smaller = [-1] * n

        # Monotonic stack for next/prev smaller
        stack = []
        for i in range(n):
            while stack and nums[i] < nums[stack[-1]]:
                idx = stack.pop()
                next_smaller[idx] = i
            if stack:
                prev_smaller[i] = stack[-1]
            stack.append(i)

        # Clear stack for next/prev greater
        stack = []
        for i in range(n):
            while stack and nums[i] > nums[stack[-1]]:
                idx = stack.pop()
                next_greater[idx] = i
            if stack:
                prev_greater[i] = stack[-1]
            stack.append(i)

        total = 0
        for i in range(n):
            max_count = (i - prev_greater[i]) * (next_greater[i] - i)
            min_count = (i - prev_smaller[i]) * (next_smaller[i] - i)
            total += nums[i] * (max_count - min_count)
        
        return total
```

The Python implementation uses two monotonic stacks, one for increasing elements (to track minimums) and one for decreasing elements (to track maximums). For each element, we calculate how many subarrays it is a maximum or minimum in using the distance to the nearest greater or smaller elements.

## Go Solution

```go
func subArrayRanges(nums []int) int64 {
    n := len(nums)
    nextGreater := make([]int, n)
    prevGreater := make([]int, n)
    nextSmaller := make([]int, n)
    prevSmaller := make([]int, n)
    for i := 0; i < n; i++ {
        nextGreater[i] = n
        nextSmaller[i] = n
        prevGreater[i] = -1
        prevSmaller[i] = -1
    }

    stack := []int{}
    for i := 0; i < n; i++ {
        for len(stack) > 0 && nums[i] < nums[stack[len(stack)-1]] {
            idx := stack[len(stack)-1]
            stack = stack[:len(stack)-1]
            nextSmaller[idx] = i
        }
        if len(stack) > 0 {
            prevSmaller[i] = stack[len(stack)-1]
        }
        stack = append(stack, i)
    }

    stack = []int{}
    for i := 0; i < n; i++ {
        for len(stack) > 0 && nums[i] > nums[stack[len(stack)-1]] {
            idx := stack[len(stack)-1]
            stack = stack[:len(stack)-1]
            nextGreater[idx] = i
        }
        if len(stack) > 0 {
            prevGreater[i] = stack[len(stack)-1]
        }
        stack = append(stack, i)
    }

    var total int64 = 0
    for i := 0; i < n; i++ {
        maxCount := int64((i - prevGreater[i]) * (nextGreater[i] - i))
        minCount := int64((i - prevSmaller[i]) * (nextSmaller[i] - i))
        total += int64(nums[i]) * (maxCount - minCount)
    }
    return total
}
```

In Go, the handling of slices and integer overflow is important. All counts are converted to `int64` to prevent overflow since element values and counts can produce large sums.
