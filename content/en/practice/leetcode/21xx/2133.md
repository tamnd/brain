## Solution

## Problem Understanding

The problem asks us to determine whether a given `n x n` integer matrix is **valid**, meaning that every row and every column contains all integers from `1` to `n` inclusive, with no duplicates or omissions. The input is a square matrix `matrix`, where each element is an integer in the range `[1, n]`. The expected output is a boolean value: `true` if the matrix is valid according to the above rules, and `false` otherwise.

The constraints `1 <= n <= 100` indicate that the matrix can be relatively small to moderately large, so we need an approach that avoids excessive nested loops with high time complexity. The constraint `1 <= matrix[i][j] <= n` ensures that all entries are within the valid range, meaning we do not need to handle negative numbers or zeros. Important edge cases include matrices where a row or column is missing a number, contains a duplicate, or has minimal size (like `1x1`), which is trivially valid.

## Approaches

A brute-force approach would involve iterating through each row and column, and for each, checking whether all integers from `1` to `n` appear. This can be done with nested loops and auxiliary storage like arrays or sets. While correct, the approach is slightly cumbersome and may perform unnecessary operations, especially in checking columns repeatedly if done inefficiently.

The key insight for an optimal approach is that we only need to track **presence of numbers** in each row and column. Using a set to track elements seen so far allows O(1) insertion and lookup. We can iterate once over the matrix, updating the sets for the current row and column simultaneously. After processing a row or column, if the set size is not `n`, we immediately know the matrix is invalid. This eliminates redundant checks and guarantees a solution in linear time relative to the matrix size.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n) | Check every number for each row and column separately, using loops |
| Optimal | O(n^2) | O(n) | Use sets to track seen numbers in each row and column simultaneously |

## Algorithm Walkthrough

1. Determine the size of the matrix `n`.
2. Iterate over each row index `i` from `0` to `n-1`.
3. Initialize two empty sets, `row_set` and `col_set`, for the current row and column.
4. Iterate over each column index `j` from `0` to `n-1`.
5. Add `matrix[i][j]` to `row_set` and `matrix[j][i]` to `col_set`. This tracks elements for the row and the corresponding column simultaneously.
6. After processing all columns for row `i`, check if the size of `row_set` or `col_set` is not equal to `n`. If either is not `n`, return `false` because a number is missing or duplicated.
7. If all rows and columns pass the check, return `true`.

**Why it works:** The algorithm maintains the invariant that after each row iteration, `row_set` contains all unique numbers in that row and `col_set` contains all unique numbers in the corresponding column. If any row or column is missing a number or has a duplicate, the set size will be less than `n`, triggering an immediate return of `false`. Thus, the algorithm correctly identifies valid matrices.

## Python Solution

```python
from typing import List

class Solution:
    def checkValid(self, matrix: List[List[int]]) -> bool:
        n = len(matrix)
        for i in range(n):
            row_set = set()
            col_set = set()
            for j in range(n):
                row_set.add(matrix[i][j])
                col_set.add(matrix[j][i])
            if len(row_set) != n or len(col_set) != n:
                return False
        return True
```

The Python solution iterates over each row and column simultaneously. Sets are used to store unique numbers, allowing for O(1) average-time checks and insertions. If any row or column does not have exactly `n` unique numbers, the function immediately returns `False`. Otherwise, the function completes the iteration and returns `True`.

## Go Solution

```go
func checkValid(matrix [][]int) bool {
    n := len(matrix)
    for i := 0; i < n; i++ {
        rowSet := make(map[int]bool)
        colSet := make(map[int]bool)
        for j := 0; j < n; j++ {
            rowSet[matrix[i][j]] = true
            colSet[matrix[j][i]] = true
        }
        if len(rowSet) != n || len(colSet) != n {
            return false
        }
    }
    return true
}
```

In Go, we use a `map[int]bool` as a set equivalent. The iteration logic is identical to the Python version. We use `len()` on the map to determine the number of unique elements in each row and column. Maps in Go handle insertion and lookup in amortized O(1) time, ensuring efficient performance.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n^2) | Iterate over each of the n rows and n columns simultaneously |
| Space | O(n) | Sets or maps store up to n unique numbers per row and column |

The time complexity is O(n^2) because we touch each matrix element once. Space is O(n) since we only store a single row and column at a time.

## Edge Cases

One important edge case is the smallest matrix size, `1x1`. Since it contains only the number 1, it is always valid. A second edge case is a matrix where all rows or all columns are identical, which can appear valid at first glance, but fails because the repeated column or row will be missing numbers. A third edge case involves a matrix with shuffled numbers but still contains exactly one of each number per row and column. The implementation correctly handles this because it checks sets of size `n` instead of relying on order.
