## Solution

## Problem Understanding

The problem gives us two inputs:

- An array of strings called `words`
- A string called `pref`

Our goal is to count how many strings in `words` begin with the string `pref`.

A prefix is defined as a leading contiguous substring. This means the prefix must appear at the very beginning of the string. For example:

- `"at"` is a prefix of `"attention"`
- `"at"` is a prefix of `"attend"`
- `"at"` is not a prefix of `"practice"`

The output should be a single integer representing the number of words that start with the given prefix.

The constraints are small:

- At most 100 words
- Each word length is at most 100
- The prefix length is at most 100

These limits tell us that even a straightforward solution will run comfortably within time limits. We do not need advanced data structures like tries or suffix arrays because the input size is tiny.

An important detail is that every string contains only lowercase English letters, which simplifies character comparisons.

Several edge cases are worth considering:

- The prefix may not match any word
- The prefix may match every word
- Some words may be shorter than the prefix
- The prefix may be exactly equal to a word
- The array may contain only one word

The problem guarantees that all strings are non-empty, so we never need to handle empty strings.

## Approaches

### Brute Force Approach

The brute-force solution checks every word character by character against the prefix.

For each word:

1. Compare the first character of the word with the first character of `pref`
2. Continue comparing characters one by one
3. If all characters in `pref` match, count the word
4. Otherwise, move to the next word

This approach is correct because a string has a prefix exactly when all characters at the beginning match the prefix characters in order.

Although this solution is simple, manually comparing characters introduces extra implementation complexity. The time complexity is still acceptable because the constraints are very small.

### Optimal Approach

Most programming languages already provide efficient built-in prefix checking operations.

In Python, we can use `startswith()`.

In Go, we can use `strings.HasPrefix()`.

The key insight is that we do not need to manually compare characters because the language runtime already implements prefix checking efficiently and clearly.

We simply iterate through the list and count how many words satisfy the prefix condition.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × m) | O(1) | Manually compares characters of each word with the prefix |
| Optimal | O(n × m) | O(1) | Uses built-in prefix checking functions |

Here:

- `n` is the number of words
- `m` is the length of the prefix

## Algorithm Walkthrough

### Optimal Algorithm

1. Initialize a counter variable to `0`.

This variable keeps track of how many words start with the prefix.
2. Iterate through every word in the `words` array.

We must examine every word because any word could potentially match the prefix.
3. For each word, check whether it starts with `pref`.

Instead of manually comparing characters, we use the language's built-in prefix checking method:

- Python: `word.startswith(pref)`
- Go: `strings.HasPrefix(word, pref)`
4. If the word starts with the prefix, increment the counter.

This means we found one valid match.
5. After processing all words, return the counter.

The final counter value represents the number of matching words.

### Why it works

A word contains `pref` as a prefix if and only if its starting characters exactly match all characters in `pref`.

The algorithm checks this condition independently for every word in the array. Since every word is examined exactly once and every valid match increments the counter exactly once, the final count is guaranteed to be correct.

## Python Solution

```python
from typing import List

class Solution:
    def prefixCount(self, words: List[str], pref: str) -> int:
        count = 0

        for word in words:
            if word.startswith(pref):
                count += 1

        return count
```

The implementation begins by initializing `count` to zero.

The loop iterates through every word in the input array. For each word, the built-in `startswith()` method checks whether the word begins with `pref`.

If the condition is true, the counter increases by one.

After all words are processed, the method returns the total number of matches.

This implementation directly follows the algorithm described earlier and keeps the code concise and readable.

## Go Solution

```go
package main

import "strings"

func prefixCount(words []string, pref string) int {
    count := 0

    for _, word := range words {
        if strings.HasPrefix(word, pref) {
            count++
        }
    }

    return count
}
```

The Go implementation is nearly identical in logic to the Python version.

The main difference is that Go uses the `strings.HasPrefix()` function from the standard library instead of a string method.

Go slices naturally handle iteration with the `range` keyword, making traversal concise and efficient.

No special handling for `nil` is required because the problem guarantees valid input arrays.

## Edge Cases

### Prefix Longer Than the Word

A common source of bugs occurs when the prefix is longer than the current word.

For example:

```
word = "ab"
pref = "abc"
```

A naive implementation that blindly indexes characters could produce an out-of-bounds error.

The built-in prefix checking functions safely handle this case by immediately returning `False`.

### Exact Match Between Word and Prefix

The prefix may be identical to the full word.

For example:

```
word = "apple"
pref = "apple"
```

This should count as a valid prefix because a string is always a prefix of itself.

Both implementations correctly treat exact matches as valid.

### Prefix Appearing Inside the Word

Another common mistake is checking whether the prefix exists anywhere inside the word instead of specifically at the beginning.

For example:

```
word = "yellow"
pref = "ell"
```

This should not count because `"ell"` starts at index 1, not index 0.

Using `startswith()` and `HasPrefix()` guarantees that only leading matches are accepted.
