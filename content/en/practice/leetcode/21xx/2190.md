## Solution

## Problem Understanding

The problem gives us an integer array `nums` and a specific integer `key`. We need to look through the array and examine every occurrence of `key`. Whenever we find `key` at index `i`, we inspect the next element, `nums[i + 1]`. That next value is considered a "target" that follows the key.

Our task is to determine which target value appears most frequently immediately after the key.

In simpler terms, we are counting adjacent pairs where the first number equals `key`. Among all numbers that appear directly after the key, we return the one with the highest frequency.

For example, if:

```
nums = [1,100,200,1,100]
key = 1
```

then:

- At index `0`, `nums[0] == 1`, so we count `nums[1] == 100`
- At index `3`, `nums[3] == 1`, so we count `nums[4] == 100`

The number `100` appears twice after the key, so the answer is `100`.

The constraints are relatively small:

- `nums.length` is at most `1000`
- Each number is between `1` and `1000`

Because the array is small, even less efficient solutions could pass. However, this problem naturally admits a very clean linear scan solution.

There are several important observations and edge cases:

- The key may appear many times consecutively.
- Multiple different targets may follow the key.
- The target with maximum frequency is guaranteed to be unique, so we never need tie-breaking logic.
- The key is guaranteed to exist in the array.
- We only examine indices up to `nums.length - 2`, because we always look at `i + 1`.

A common implementation mistake is iterating all the way to the last index and then accessing `nums[i + 1]`, which would cause an out-of-bounds error.

## Approaches

### Brute Force Approach

The brute-force idea is to consider every unique number in the array as a possible target. For each candidate target, we scan the entire array again and count how many times that target immediately follows the key.

For example:

1. Pick a candidate target.
2. Traverse the array.
3. Every time `nums[i] == key`, check whether `nums[i + 1]` equals the candidate target.
4. Keep the maximum count.

This works because it explicitly computes the exact frequency for every possible target. However, it repeatedly scans the array for each unique number, which is unnecessary work.

If there are `n` elements and up to `n` distinct values, this can take `O(n^2)` time.

### Optimal Approach

The key observation is that we do not need separate scans for each target.

Instead, we can process the array once:

- Whenever we encounter `nums[i] == key`, we immediately look at `nums[i + 1]`.
- We maintain a hash map that counts how many times each target follows the key.
- While updating counts, we track the target with the largest frequency.

This avoids repeated work and reduces the complexity to linear time.

A hash map is ideal because:

- We need frequency counting.
- Insertions and lookups are `O(1)` on average.
- The target values are arbitrary integers.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) or O(n) | Repeatedly scans the array for every possible target |
| Optimal | O(n) | O(n) | Single pass with hash map frequency counting |

## Algorithm Walkthrough

1. Create a hash map called `frequency`.

This map stores how many times each target value appears immediately after the key.
2. Initialize two variables:

- `best_target`, the current most frequent target
- `best_count`, the highest frequency seen so far
3. Traverse the array from index `0` to `len(nums) - 2`.

We stop at `len(nums) - 2` because we always access `nums[i + 1]`.
4. For each index `i`, check whether `nums[i] == key`.

If not, continue to the next index.
5. If the current element equals the key:

- Let `target = nums[i + 1]`
- Increment `frequency[target]`
6. After updating the frequency:

- Compare the updated count with `best_count`
- If the new count is larger:

- Update `best_count`
- Update `best_target`
7. After the loop finishes, return `best_target`.

### Why it works

The algorithm examines every valid adjacent pair where the first element equals `key`. Every such pair contributes exactly once to the frequency map. Therefore, the map accurately records how many times each target follows the key.

Because we continuously maintain the target with the highest frequency, the final answer is guaranteed to be the most frequent target. The problem guarantees uniqueness, so there is no ambiguity.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def mostFrequent(self, nums: List[int], key: int) -> int:
        frequency = defaultdict(int)

        best_target = -1
        best_count = 0

        for i in range(len(nums) - 1):
            if nums[i] == key:
                target = nums[i + 1]
                frequency[target] += 1

                if frequency[target] > best_count:
                    best_count = frequency[target]
                    best_target = target

        return best_target
```

The implementation closely follows the algorithm described earlier.

We first import `defaultdict` so that frequency counting becomes simple and concise. Missing keys automatically start with a value of `0`.

The loop iterates only until `len(nums) - 2`, ensuring that `nums[i + 1]` is always valid.

Whenever the current value equals `key`, we examine the next value and increment its count in the hash map.

After updating the count, we immediately compare it against the current best frequency. If the new frequency is larger, we update both `best_count` and `best_target`.

At the end of the traversal, `best_target` contains the number that appeared most frequently after the key.

## Go Solution

```go
func mostFrequent(nums []int, key int) int {
	frequency := make(map[int]int)

	bestTarget := -1
	bestCount := 0

	for i := 0; i < len(nums)-1; i++ {
		if nums[i] == key {
			target := nums[i+1]
			frequency[target]++

			if frequency[target] > bestCount {
				bestCount = frequency[target]
				bestTarget = target
			}
		}
	}

	return bestTarget
}
```

The Go implementation uses a `map[int]int` for frequency counting, which serves the same purpose as Python's dictionary.

Go maps return the zero value for missing keys, so incrementing `frequency[target]` works naturally without extra checks.

The loop bounds are identical to the Python solution, preventing out-of-range access when reading `nums[i+1]`.

There are no integer overflow concerns because the maximum count is at most `1000`, well within Go's integer range.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We scan the array exactly once |
| Space | O(n) | In the worst case, the hash map stores many distinct targets |

The algorithm performs a single linear traversal of the array. Each hash map update and lookup takes constant average time, so the total runtime is proportional to the number of elements.

The extra memory comes from storing target frequencies. In the worst case, every target after the key could be distinct, requiring up to `O(n)` space.

## Edge Cases

One important edge case occurs when the key appears consecutively multiple times. For example:

```
nums = [2,2,2,2]
```

A careless implementation might skip overlapping pairs. However, our loop checks every index independently, so all adjacent occurrences are counted correctly.

Another edge case is the smallest possible array size:

```
nums = [1,2]
```

Since the loop runs only to `len(nums) - 2`, we safely examine exactly one pair without accessing beyond the array boundary.

A third important case occurs when the key appears only once in the array. In this situation, there is only one possible target. The algorithm still works naturally because the frequency map records a single count, and that target becomes the answer immediately.

Finally, there is the case where the key is followed by itself repeatedly:

```
nums = [7,7,7,7]
```

Some implementations incorrectly assume the target must differ from the key. Our solution imposes no such restriction, so it correctly counts `7` as both the key and the target.
