## Solution

## Problem Understanding

This problem gives us a binary matrix, meaning every cell contains either `0` or `1`. The goal is to remove all `1`s using the minimum number of operations.

An operation works in a very specific way. You are allowed to choose a cell `(i, j)` only if `grid[i][j] == 1`. Once you choose that cell, every value in row `i` becomes `0`, and every value in column `j` also becomes `0`.

The important detail is that the operation affects both the entire row and the entire column simultaneously. This means one carefully chosen operation may eliminate many `1`s at once.

The input size is small:

- `1 <= m, n <= 15`
- `1 <= m * n <= 15`

The second constraint is the most important one. Even though each dimension can be as large as `15`, the total number of cells is at most `15`. That means the entire matrix contains at most `15` bits of information.

This strongly suggests that we should represent the grid as a bitmask. A state can be compressed into an integer where each bit represents whether a cell currently contains a `1`.

The expected output is the minimum number of operations required to reach the all-zero state.

Several edge cases are important:

- The grid may already contain only zeros, in which case the answer is `0`.
- A single operation may wipe out the entire grid if the chosen row and column together cover all remaining `1`s.
- Some grids require careful ordering of operations because choosing the wrong cell first may leave isolated `1`s behind.
- Since operations can only start from cells currently equal to `1`, we cannot arbitrarily clear rows and columns.

Because the state space is small, shortest-path style search techniques become feasible.

## Approaches

### Brute Force Approach

A straightforward brute force solution tries every possible sequence of operations recursively.

At each step:

1. Find all cells containing `1`.
2. For each such cell:

- Create a new grid after clearing its row and column.
- Recursively compute the minimum operations needed from that new state.
3. Return the smallest result among all choices.

This works because it explores every valid operation sequence, so eventually it finds the optimal answer.

However, this approach suffers from massive repeated computation. The same grid configuration can appear through many different paths. Without memoization, the recursion tree grows exponentially.

Even with memoization using the raw matrix representation, repeatedly copying matrices and recomputing transitions becomes inefficient.

### Key Insight

The total number of cells is at most `15`, which is extremely small for bitmask-based state compression.

We can encode the entire grid into a single integer:

- Bit `k` is `1` if the corresponding cell currently contains a `1`
- Bit `k` is `0` otherwise

Now every possible grid configuration becomes a state in a graph.

From any state:

- Each set bit represents a valid operation
- Applying the operation creates another state

Since we want the minimum number of operations, this becomes a shortest path problem on an unweighted graph.

Breadth-First Search is ideal here because BFS guarantees that the first time we reach the all-zero state, we used the minimum number of operations.

The number of states is at most:

```
2^15 = 32768
```

which is completely manageable.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((m*n)!) in worst case | O(recursion depth) | Explores many repeated states |
| Optimal BFS + Bitmask | O(2^(m*n) * m * n) | O(2^(m*n)) | Efficient state compression and shortest path search |

## Algorithm Walkthrough

### Step 1: Encode the Grid as a Bitmask

We map every cell to a unique bit position.

For example, in a `2 x 3` grid:

```
(0,0) -> bit 0
(0,1) -> bit 1
(0,2) -> bit 2
(1,0) -> bit 3
(1,1) -> bit 4
(1,2) -> bit 5
```

If a cell contains `1`, we set that bit in the mask.

This converts the entire matrix into a single integer state.

### Step 2: Precompute Operation Masks

For every cell `(r, c)`, we precompute which bits would be cleared if we perform an operation there.

That operation removes:

- Every cell in row `r`
- Every cell in column `c`

We build a bitmask called `remove_mask[r][c]`.

This preprocessing avoids recomputing row and column coverage during BFS.

### Step 3: Initialize BFS

We start BFS from the initial state.

The BFS queue stores:

- Current bitmask state
- Number of operations used so far

We also maintain a visited set so each state is processed only once.

### Step 4: Process States Level by Level

For the current state:

1. If the state is `0`, all cells are cleared.
2. Otherwise, iterate through every cell.
3. If that cell's bit is set:

- It is a valid operation
- Apply the operation

The next state becomes:

```
next_state = current_state & ~remove_mask[r][c]
```

This clears all bits affected by the operation.

### Step 5: Push New States into BFS

If the new state has not been visited:

- Mark it visited
- Push it into the queue with distance + 1

Because BFS explores states in increasing operation count order, the first time we reach state `0` is guaranteed to be optimal.

### Why it works

Each BFS node represents a unique grid configuration. Every edge corresponds to exactly one valid operation. Since all operations have equal cost, BFS explores states in order of increasing operations used. Therefore, the first time the algorithm reaches the all-zero state, it has found the minimum possible number of operations.

## Python Solution

```python
from collections import deque
from typing import List

class Solution:
    def removeOnes(self, grid: List[List[int]]) -> int:
        rows = len(grid)
        cols = len(grid[0])

        # Convert grid into bitmask state
        start_state = 0

        for r in range(rows):
            for c in range(cols):
                if grid[r][c] == 1:
                    bit = r * cols + c
                    start_state |= (1 << bit)

        # Already empty
        if start_state == 0:
            return 0

        # Precompute removal masks
        remove_masks = [[0] * cols for _ in range(rows)]

        for r in range(rows):
            for c in range(cols):
                mask = 0

                # Entire row
                for j in range(cols):
                    bit = r * cols + j
                    mask |= (1 << bit)

                # Entire column
                for i in range(rows):
                    bit = i * cols + c
                    mask |= (1 << bit)

                remove_masks[r][c] = mask

        queue = deque([(start_state, 0)])
        visited = {start_state}

        while queue:
            state, steps = queue.popleft()

            if state == 0:
                return steps

            for r in range(rows):
                for c in range(cols):
                    bit = r * cols + c

                    # Can only operate on cells containing 1
                    if state & (1 << bit):
                        next_state = state & ~remove_masks[r][c]

                        if next_state not in visited:
                            visited.add(next_state)
                            queue.append((next_state, steps + 1))

        return -1
```

The implementation begins by converting the matrix into a compact integer bitmask. Each bit corresponds to one cell in the grid.

Next, the code precomputes the row-column removal effect for every possible operation. This allows transitions between states to be computed in constant time during BFS.

The BFS queue stores pairs of `(state, steps)`. Every time we pop a state, we try all valid operations by checking which bits are currently set.

The transition:

```python
next_state = state & ~remove_masks[r][c]
```

removes all affected cells in a single bitwise operation.

The visited set prevents repeated processing of identical states, which keeps the search efficient.

Because BFS processes states in increasing distance order, the first time state `0` is reached is the optimal answer.

## Go Solution

```go
package main

import "container/list"

func removeOnes(grid [][]int) int {
	rows := len(grid)
	cols := len(grid[0])

	startState := 0

	for r := 0; r < rows; r++ {
		for c := 0; c < cols; c++ {
			if grid[r][c] == 1 {
				bit := r*cols + c
				startState |= (1 << bit)
			}
		}
	}

	if startState == 0 {
		return 0
	}

	// Precompute removal masks
	removeMasks := make([][]int, rows)

	for r := 0; r < rows; r++ {
		removeMasks[r] = make([]int, cols)

		for c := 0; c < cols; c++ {
			mask := 0

			// Entire row
			for j := 0; j < cols; j++ {
				bit := r*cols + j
				mask |= (1 << bit)
			}

			// Entire column
			for i := 0; i < rows; i++ {
				bit := i*cols + c
				mask |= (1 << bit)
			}

			removeMasks[r][c] = mask
		}
	}

	type State struct {
		mask  int
		steps int
	}

	queue := list.New()
	queue.PushBack(State{startState, 0})

	visited := map[int]bool{
		startState: true,
	}

	for queue.Len() > 0 {
		front := queue.Front()
		queue.Remove(front)

		current := front.Value.(State)

		if current.mask == 0 {
			return current.steps
		}

		for r := 0; r < rows; r++ {
			for c := 0; c < cols; c++ {
				bit := r*cols + c

				if (current.mask & (1 << bit)) != 0 {
					nextState := current.mask & ^removeMasks[r][c]

					if !visited[nextState] {
						visited[nextState] = true
						queue.PushBack(State{
							mask:  nextState,
							steps: current.steps + 1,
						})
					}
				}
			}
		}
	}

	return -1
}
```

The Go implementation follows the same algorithmic structure as the Python version.

A `map[int]bool` is used for the visited states because Go does not provide a built-in hash set type.

The BFS queue uses `container/list`, which efficiently supports push and pop operations from both ends.

Bitwise operations behave similarly to Python, although Go requires explicit use of `^` for bitwise NOT masking.

Since the maximum number of bits is only `15`, integer overflow is never a concern.

## Edge Cases

### Grid Already Contains Only Zeros

A common source of bugs is forgetting that no operations may be necessary.

For example:

```
[[0,0],
 [0,0]]
```

In this situation, the initial bitmask is already `0`. The implementation checks this immediately and returns `0` before starting BFS.

Without this check, some implementations accidentally perform unnecessary processing or even fail because no valid operations exist.

### Single Cell Grid

The smallest possible grids are:

```
[[1]]
```

and

```
[[0]]
```

These cases test whether the bitmask indexing and BFS transitions work correctly when only one bit exists.

The implementation handles both naturally because the bit position calculation works for any matrix size.

### Multiple Disconnected Ones

Consider:

```
[[1,0],
 [0,1]]
```

The two `1`s do not share a row or column. One operation cannot remove both simultaneously.

A greedy algorithm might incorrectly assume every operation should maximize immediate removals, but optimality requires exploring all possible state transitions.

BFS handles this correctly because it systematically explores all reachable configurations and guarantees the minimum number of operations.
