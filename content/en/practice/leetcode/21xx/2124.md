## Solution

## Problem Understanding

The problem asks us to determine whether all occurrences of the character `'a'` in a string appear **before** any occurrences of `'b'`. The string `s` consists exclusively of the characters `'a'` and `'b'`. The output should be `true` if every `'a'` comes before every `'b'`, and `false` otherwise.

For example, in the string `"aaabbb"`, all `'a'` characters occur at the start, followed by all `'b'` characters, so the output is `true`. In `"abab"`, an `'a'` appears after a `'b'`, so the output is `false`. Strings without any `'a'` or without any `'b'` are trivially valid and should return `true`.

The constraints tell us that the string length is at most 100 characters, which is very small, so even a simple linear scan is efficient. Edge cases to consider include strings of length 1, strings with only `'a'` or only `'b'`, and strings where a single `'a'` occurs after a `'b'`. The problem guarantees that the string contains only `'a'` and `'b'`, so no other character handling is needed.

## Approaches

The brute-force approach would involve checking **all pairs of indices** `(i, j)` where `i < j` to see if `s[i] == 'b'` and `s[j] == 'a'`. If such a pair exists, we would return `false`. Otherwise, we return `true`. This method works because any violation of the order will be detected by at least one such pair. However, it requires checking O(n²) pairs, which is unnecessary even for a string of length 100.

The key insight for an optimal solution is that the order is only violated if an `'a'` appears **after** a `'b'`. We can track whether we have already seen a `'b'` while scanning from left to right. If we encounter an `'a'` after seeing any `'b'`, we can immediately return `false`. If the scan finishes without encountering such a case, we return `true`. This reduces the algorithm to a simple linear scan with O(n) time complexity and O(1) space complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Check all pairs of indices for an `'a'` after `'b'` violation |
| Optimal | O(n) | O(1) | Single left-to-right scan checking if an `'a'` appears after a `'b'` |

## Algorithm Walkthrough

1. Initialize a boolean flag `seen_b` to `false`. This will indicate whether we have encountered any `'b'` so far in the scan.
2. Iterate over each character `c` in the string `s`.
3. If the current character `c` is `'b'`, set `seen_b` to `true`. This marks that any subsequent `'a'` would violate the ordering.
4. If the current character `c` is `'a'` and `seen_b` is `true`, return `false` immediately. This is the violation case: an `'a'` appears after a `'b'`.
5. If the loop completes without returning `false`, return `true`. This indicates that all `'a'` characters appear before all `'b'` characters.

**Why it works**: The invariant maintained during the scan is that `seen_b` accurately represents whether we have encountered any `'b'` so far. The first `'a'` occurring after `seen_b` becomes `true` directly violates the required ordering. Since the scan examines each character exactly once, the algorithm is guaranteed to detect any violation efficiently.

## Python Solution

```python
class Solution:
    def checkString(self, s: str) -> bool:
        seen_b = False
        for c in s:
            if c == 'b':
                seen_b = True
            elif c == 'a' and seen_b:
                return False
        return True
```

The implementation initializes a flag `seen_b` to track if any `'b'` has been encountered. It scans each character in the string, updating the flag when `'b'` is seen. If an `'a'` occurs after the flag is set, the function returns `false` because it violates the required ordering. If the loop completes, no violation occurred, and `true` is returned.

## Go Solution

```go
func checkString(s string) bool {
    seenB := false
    for i := 0; i < len(s); i++ {
        if s[i] == 'b' {
            seenB = true
        } else if s[i] == 'a' && seenB {
            return false
        }
    }
    return true
}
```

In Go, we use a `for` loop over the indices of the string to check each character. The boolean flag `seenB` tracks whether we have seen any `'b'`. If an `'a'` appears after `seenB` is `true`, we return `false`. Otherwise, we return `true` at the end. The logic mirrors the Python version exactly, and no special handling is needed for string indexing in Go.
