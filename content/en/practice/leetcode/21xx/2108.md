## Solution

## Problem Understanding

The problem gives us an array of strings called `words`, and we must return the **first string that is a palindrome**. A palindrome is a string that reads the same from left to right and right to left. If no word in the array is palindromic, we return an empty string `""`.

In simpler terms, we scan the array from left to right and check each word one by one. The moment we encounter a palindrome, we immediately return it. Since the requirement is to return the **first** palindromic string, we do not continue searching after finding a valid answer.

The input consists of an array of lowercase English strings. The output is a single string, either the first palindrome found or an empty string if none exists.

The constraints are relatively small:

- `1 <= words.length <= 100`
- `1 <= words[i].length <= 100`

This means the maximum total amount of work is very manageable. Even a straightforward approach would work efficiently because there are at most `100` words, each with at most `100` characters.

There are several important edge cases to keep in mind. A word with a single character is always a palindrome, since it reads the same in both directions. The first word in the array may already be palindromic, meaning we should return immediately. Multiple palindromes may exist, but we must return only the first one encountered. Finally, there may be no palindromes at all, in which case the correct result is `""`.

## Approaches

### Brute Force Approach

A brute-force approach would examine every word in the array and determine whether it is a palindrome by explicitly reversing the string and comparing it with the original.

For each word:

1. Reverse the string.
2. Compare the reversed string with the original.
3. If they match, return the word immediately.
4. If no matches are found after checking all words, return `""`.

This approach is correct because a palindrome is defined precisely as a string equal to its reverse. However, reversing a string creates a new copy of the string, which uses additional memory. While this is completely acceptable under the given constraints, we can improve the implementation slightly.

### Optimal Approach

The key observation is that we do not actually need to create a reversed string to determine whether a word is palindromic. Instead, we can use the **two pointers technique**.

For each word, place one pointer at the beginning and another at the end. Compare characters moving inward:

- If the characters differ at any point, the word is not a palindrome.
- If all corresponding characters match, the word is a palindrome.

This avoids allocating extra memory for reversed strings and checks the palindrome property directly.

Since we stop as soon as we find the first palindrome, the algorithm is efficient and straightforward.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × m) | O(m) | Reverse each word and compare with original |
| Optimal | O(n × m) | O(1) | Use two pointers to check palindromes in place |

Here, `n` is the number of words and `m` is the maximum word length.

## Algorithm Walkthrough

1. Iterate through each word in the `words` array from left to right. We process words in order because the problem specifically asks for the **first** palindromic string.
2. For the current word, initialize two pointers:

- `left = 0`, pointing to the first character.
- `right = len(word) - 1`, pointing to the last character.
3. Compare characters at the `left` and `right` positions.

- If they are different, the word is not a palindrome, so stop checking this word and move to the next one.
- If they are the same, move the pointers inward:

- Increment `left`
- Decrement `right`
4. Continue until `left >= right`.

- If all comparisons matched, the word is a palindrome.
- Return this word immediately because it is the first valid answer.
5. If all words are processed and no palindrome is found, return an empty string `""`.

### Why it works

The algorithm relies on the defining property of a palindrome: characters at symmetric positions must match. By comparing the first and last characters, then the second and second-last, and so on, we verify exactly this property. If any mismatch occurs, the word cannot be a palindrome. Since we examine words in order and return immediately upon finding a valid palindrome, the returned result is guaranteed to be the **first palindromic string**.

## Python Solution

```python
from typing import List

class Solution:
    def firstPalindrome(self, words: List[str]) -> str:
        def is_palindrome(word: str) -> bool:
            left = 0
            right = len(word) - 1

            while left < right:
                if word[left] != word[right]:
                    return False
                left += 1
                right -= 1

            return True

        for word in words:
            if is_palindrome(word):
                return word

        return ""
```

The implementation begins with a helper function `is_palindrome`, which uses the two pointers technique. One pointer starts at the beginning of the string and the other starts at the end. The function compares matching positions while moving inward. If any mismatch occurs, it immediately returns `False`. If the loop finishes without mismatches, the word is palindromic and the function returns `True`.

The main logic iterates through each word in `words`. For every word, we call `is_palindrome`. As soon as a palindrome is found, it is returned immediately. If the loop completes without finding any palindrome, the function returns an empty string.

## Go Solution

```go
func firstPalindrome(words []string) string {
	isPalindrome := func(word string) bool {
		left := 0
		right := len(word) - 1

		for left < right {
			if word[left] != word[right] {
				return false
			}
			left++
			right--
		}

		return true
	}

	for _, word := range words {
		if isPalindrome(word) {
			return word
		}
	}

	return ""
}
```

The Go implementation follows the same logic as the Python version. A helper function checks whether a word is palindromic using two pointers.

One small Go-specific detail is that strings are indexed as bytes. This works perfectly here because the problem guarantees only lowercase English letters, meaning every character is a single byte. If Unicode characters were involved, rune handling would be necessary.

Since slices in Go are safe to iterate over, there is no special handling needed for `nil` versus empty input. The problem constraints guarantee at least one word.

## Worked Examples

### Example 1

**Input:**

```
words = ["abc","car","ada","racecar","cool"]
```

We process words from left to right.

| Step | Current Word | Character Comparison | Palindrome? |
| --- | --- | --- | --- |
| 1 | `"abc"` | `'a' != 'c'` | No |
| 2 | `"car"` | `'c' != 'r'` | No |
| 3 | `"ada"` | `'a' == 'a'`, middle `'d'` | Yes |

At `"ada"`, all symmetric characters match, so we immediately return:

```
"ada"
```

We do not continue to `"racecar"` because the problem asks for the **first** palindrome.

### Example 2

**Input:**

```
words = ["notapalindrome","racecar"]
```

| Step | Current Word | Character Comparison | Palindrome? |
| --- | --- | --- | --- |
| 1 | `"notapalindrome"` | `'n' != 'e'` | No |
| 2 | `"racecar"` | `'r' == 'r'`, `'a' == 'a'`, `'c' == 'c'` | Yes |

The algorithm returns:

```
"racecar"
```

### Example 3

**Input:**

```
words = ["def","ghi"]
```

| Step | Current Word | Character Comparison | Palindrome? |
| --- | --- | --- | --- |
| 1 | `"def"` | `'d' != 'f'` | No |
| 2 | `"ghi"` | `'g' != 'i'` | No |

No palindrome exists, so the result is:

```
""
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × m) | Each word may require checking every character once |
| Space | O(1) | Only a few pointer variables are used |

The algorithm processes each word at most once. For every word, the palindrome check compares characters from both ends toward the center, requiring at most `m` comparisons where `m` is the word length. Across all words, this results in `O(n × m)` time complexity. Since we only use a constant amount of extra memory for pointers and loop variables, the space complexity is `O(1)`.

## Edge Cases

A very important edge case is when the **first word is already a palindrome**. A buggy implementation might continue searching unnecessarily or accidentally overwrite the result with a later palindrome. Since the requirement is to return the first palindrome, the implementation immediately returns once a valid word is found.

Another important case is **single-character strings**. A word like `"a"` should always count as a palindrome because it reads the same forward and backward. The two-pointer implementation handles this naturally because `left` and `right` start at the same position, so the loop never runs and the function correctly returns `True`.

A third edge case is **when no palindrome exists in the array**. Some implementations may accidentally return `None`, an uninitialized variable, or the last processed word. This implementation explicitly returns `""` after checking all words, ensuring the correct result.

A final subtle case involves **multiple palindromic strings**. For example, `["abc", "level", "radar"]` contains more than one palindrome. The implementation correctly returns `"level"` because it scans left to right and exits immediately after finding the first valid match.
