## Solution

## Problem Understanding

The problem asks us to compute the **longest winning streak** for each player based on the `Matches` table. Each row in this table represents a match played by a player, including the `player_id`, `match_day`, and the `result`, which can be `'Win'`, `'Draw'`, or `'Lose'`. The winning streak is defined as the number of consecutive wins uninterrupted by draws or losses. Our goal is to return a table showing, for each player, the maximum length of such consecutive wins.

The input guarantees that `(player_id, match_day)` is unique, so we do not need to handle duplicate matches for a single day. The output is a table with `player_id` and `longest_streak`.

Key edge cases to consider include players with no wins at all, players with only a single win, or streaks separated by draws or losses. The table can have many players, and each player may have a variable number of matches. This means any solution must handle multiple groups efficiently.

The follow-up question extends the concept to the longest streak without losing, meaning wins and draws both count towards the streak. This generalizes the problem slightly but keeps the same fundamental approach.

## Approaches

### Brute Force Approach

A brute force approach would iterate over each player and their matches in chronological order. For each player, we could maintain a counter for consecutive wins, resetting it to zero whenever a draw or loss occurs. We would track the maximum counter encountered. While correct, this approach becomes cumbersome if implemented in SQL using correlated subqueries because each match might need to check the previous match for continuity. This can lead to O(n²) complexity for large tables, making it inefficient for datasets with many players and matches.

### Optimal Approach

The key observation for an efficient solution is that **we can use window functions to assign a grouping for consecutive wins**. By marking non-winning rows and using the difference between row numbers over a player partition and ordering by match day, we can create "islands" of consecutive wins. This approach leverages the property that the difference between the row number and a sequential index is constant within consecutive wins. Once these groups are identified, we can count the size of each group and take the maximum for each player. This avoids nested loops and runs efficiently in O(n) time relative to the number of matches.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Iterate over matches per player and reset streaks manually |
| Optimal | O(n) | O(n) | Use window functions to group consecutive wins and aggregate |

## Algorithm Walkthrough

1. Select all matches and add a column `is_win` which is 1 if the result is `'Win'` and 0 otherwise. This simplifies streak calculations.
2. Assign a row number over `(player_id ORDER BY match_day)` to get a sequential index for each player's matches.
3. For winning matches (`is_win = 1`), compute a `group_id` as `row_number - SUM(is_win) OVER (PARTITION BY player_id ORDER BY match_day ROWS UNBOUNDED PRECEDING)`. This leverages the property that the difference is constant for consecutive wins.
4. Group by `player_id` and `group_id` to count the length of each winning streak.
5. Take the maximum streak per player. If a player has no wins, default the streak to 0 using a `COALESCE`.
6. Return `player_id` and `longest_streak`.

**Why it works**: The grouping technique ensures that consecutive wins form continuous segments with the same `group_id`. Counting within these segments gives the exact length of each streak. Aggregating these per player yields the maximum winning streak.

## Python Solution

```python
import pandas as pd

def longest_winning_streak(matches: pd.DataFrame) -> pd.DataFrame:
    # Convert result to numeric for win tracking
    matches['is_win'] = (matches['result'] == 'Win').astype(int)
    
    # Sort by player and match_day
    matches = matches.sort_values(['player_id', 'match_day'])
    
    # Compute row number per player
    matches['row_num'] = matches.groupby('player_id').cumcount() + 1
    
    # Compute cumulative wins
    matches['cum_win'] = matches.groupby('player_id')['is_win'].cumsum()
    
    # Only consider wins
    wins = matches[matches['is_win'] == 1].copy()
    
    # Assign group ID for consecutive wins
    wins['group_id'] = wins['row_num'] - wins['cum_win']
    
    # Compute streak length per group
    streaks = wins.groupby(['player_id', 'group_id']).size().reset_index(name='streak_length')
    
    # Get max streak per player
    max_streak = streaks.groupby('player_id')['streak_length'].max().reset_index()
    
    # Include players with zero wins
    all_players = matches[['player_id']].drop_duplicates()
    result = all_players.merge(max_streak, on='player_id', how='left').fillna(0)
    result['streak_length'] = result['streak_length'].astype(int)
    result.rename(columns={'streak_length': 'longest_streak'}, inplace=True)
    
    return result
```

**Explanation**: First, we mark wins and compute cumulative sums to identify streak boundaries. The difference between `row_num` and `cum_win` remains constant for consecutive wins, giving us a group identifier. Counting per group and taking the maximum gives the longest streak per player. Players without wins are handled by a left join with the unique player list.

## Go Solution

```go
package main

import (
    "database/sql"
    _ "github.com/go-sql-driver/mysql"
    "fmt"
)

func longestWinningStreak(db *sql.DB) (*sql.Rows, error) {
    query := `
    WITH win_flags AS (
        SELECT player_id, match_day,
               CASE WHEN result = 'Win' THEN 1 ELSE 0 END AS is_win
        FROM Matches
    ),
    numbered AS (
        SELECT player_id, match_day, is_win,
               ROW_NUMBER() OVER (PARTITION BY player_id ORDER BY match_day) AS row_num,
               SUM(is_win) OVER (PARTITION BY player_id ORDER BY match_day ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) AS cum_win
        FROM win_flags
    ),
    win_groups AS (
        SELECT player_id, (row_num - cum_win) AS group_id
        FROM numbered
        WHERE is_win = 1
    ),
    streaks AS (
        SELECT player_id, group_id, COUNT(*) AS streak_length
        FROM win_groups
        GROUP BY player_id, group_id
    ),
    max_streak AS (
        SELECT player_id, COALESCE(MAX(streak_length), 0) AS longest_streak
        FROM streaks
        GROUP BY player_id
    )
    SELECT p.player_id, COALESCE(m.longest_streak, 0) AS longest_streak
    FROM (SELECT DISTINCT player_id FROM Matches) p
    LEFT JOIN max_streak m ON p.player_id = m.player_id;
    `
    return db.Query(query)
}
```

**Go Notes**: SQL is used directly as Go primarily acts as a client. The `COALESCE` ensures players with no wins are handled. The window functions behave the same as in the Python pandas approach. Go code uses the `database/sql` package and expects a proper `sql.DB` connection.
