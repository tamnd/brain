## Problem Understanding

The problem asks us to determine the **minimum number of moves** required to transform a given string `s` into a palindrome. A move is defined as swapping **two adjacent characters** in the string. The input string consists only of lowercase English letters, and the problem guarantees that a palindrome is always possible.

In simpler terms, we are allowed to "bubble" characters into position by adjacent swaps until the string reads the same forwards and backwards. The input `s` is of length up to 2000, so any solution that is significantly worse than O(n^2) could be too slow. The output is a single integer representing the minimum swaps needed.

The constraints tell us that we do not need to handle strings that cannot form a palindrome. Edge cases that may trip up a naive implementation include strings with repeated characters, strings of length 1, and strings where one character occurs an odd number of times (which will naturally go in the middle of the palindrome).

## Approaches

The **brute-force approach** would try all possible permutations of the string and check which ones are palindromes. For each palindrome found, we could count the minimum adjacent swaps required to reach it. This method is correct but infeasible because the number of permutations is O(n!) and checking each one is also costly.

The **key insight** for a better solution is that we can solve this problem greedily from the outside in. A palindrome has symmetry: characters at positions `i` and `n-1-i` must match. We can pair matching characters starting from the leftmost character, find its counterpart as close as possible from the right, and "bubble" it to the correct position using adjacent swaps. This process can be repeated until the entire string is a palindrome.

To efficiently handle repeated characters and avoid repeatedly swapping through long ranges, a **Binary Indexed Tree (Fenwick Tree)** can be used to track the number of shifts that have already been applied to positions. This allows us to compute the effective distance a character needs to travel in O(log n) time instead of O(n).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n! * n) | O(n) | Enumerates all permutations and counts swaps, not feasible |
| Optimal | O(n log n) | O(n) | Uses greedy matching with a BIT to efficiently track swaps |

## Algorithm Walkthrough

1. Initialize a list of character positions for all letters in the string. This allows fast lookup of the next occurrence of a character we want to match.
2. Initialize a Binary Indexed Tree (Fenwick Tree) of size `n` to track the cumulative number of swaps that have moved characters past each index.
3. Start a two-pointer approach with `i = 0` on the left and `j = n-1` on the right. We will match `s[i]` with a character from the right half.
4. If `s[i]` already matches `s[j]`, move both pointers inward. If not, locate the nearest matching character to `s[i]` in the right portion. Use the BIT to calculate how many swaps it will take to bubble it into position `j`.
5. Perform the swap logically by updating the BIT to reflect that the character has moved past its previous positions.
6. If a character does not have a pair (only occurs an odd number of times), place it in the middle of the palindrome when `i == j`.
7. Continue the process until `i >= j`, summing all swaps along the way.

**Why it works:** The algorithm maintains the invariant that all characters outside the current `i` and `j` pointers are already in palindrome order. By always moving the nearest matching character to the correct mirrored position, it ensures minimal adjacent swaps. The BIT guarantees that we account for shifts caused by previous swaps.

## Python Solution

```python
class BIT:
    def __init__(self, n: int):
        self.tree = [0] * (n + 1)
        self.n = n

    def update(self, i: int, delta: int):
        i += 1
        while i <= self.n:
            self.tree[i] += delta
            i += i & -i

    def query(self, i: int) -> int:
        i += 1
        res = 0
        while i > 0:
            res += self.tree[i]
            i -= i & -i
        return res

class Solution:
    def minMovesToMakePalindrome(self, s: str) -> int:
        from collections import defaultdict, deque

        n = len(s)
        pos = defaultdict(deque)
        for i, ch in enumerate(s):
            pos[ch].append(i)

        bit = BIT(n)
        res = 0
        i, j = 0, n - 1
        used = [False] * n

        while i < j:
            if s[i] == s[j]:
                i += 1
                j -= 1
                continue

            # Find the matching character for s[i] from the right
            right_index = None
            for k in reversed(pos[s[i]]):
                if not used[k] and k >= i:
                    right_index = k
                    break

            if right_index is None:
                # s[i] is the middle character in an odd-length palindrome
                mid = i
                used[mid] = True
                i += 1
                continue

            # Count swaps to bring it to position j
            swaps = right_index - j + bit.query(right_index) - bit.query(j)
            res += swaps

            # Mark this character as used and update BIT
            used[right_index] = True
            bit.update(j, 1)

            i += 1
            j -= 1

        return res
```

**Implementation walkthrough:** First, we build a mapping from characters to their positions for quick access. We initialize a BIT to track shifts caused by previous swaps. Using two pointers, we greedily match the left character with the nearest unused right character. Swaps are counted using the BIT to adjust for prior moves. If a character has no pair, it is the middle character in the palindrome.

## Go Solution

```go
package main

func minMovesToMakePalindrome(s string) int {
	n := len(s)
	type BIT struct {
		tree []int
		n    int
	}
	bit := BIT{tree: make([]int, n+1), n: n}

	update := func(i, delta int) {
		i++
		for i <= bit.n {
			bit.tree[i] += delta
			i += i & -i
		}
	}

	query := func(i int) int {
		i++
		res := 0
		for i > 0 {
			res += bit.tree[i]
			i -= i & -i
		}
		return res
	}

	pos := make(map[byte][]int)
	for i := 0; i < n; i++ {
		ch := s[i]
		pos[ch] = append(pos[ch], i)
	}

	used := make([]bool, n)
	res := 0
	i, j := 0, n-1

	for i < j {
		if s[i] == s[j] {
			i++
			j--
			continue
		}

		var rightIndex int = -1
		for k := len(pos[s[i]]) - 1; k >= 0; k-- {
			if !used[pos[s[i]][k]] && pos[s[i]][k] >= i {
				rightIndex = pos[s[i]][k]
				break
			}
		}

		if rightIndex == -1 {
			used[i] = true
			i++
			continue
		}

		swaps := rightIndex - j + query(rightIndex) - query(j)
		res += swaps
		used[rightIndex] = true
		update(j, 1)

		i++
		j--
	}

	return res
}
```

**Go-specific notes:** The Go version closely mirrors the Python solution. We handle slices explicitly instead of Python’s deque. The BIT implementation uses zero-based indexing with +1 adjustment internally. Boolean arrays track used indices.
