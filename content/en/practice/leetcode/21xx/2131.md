## Solution

## Problem Understanding

The problem provides an array of two-letter lowercase strings, and the task is to form the **longest palindrome** by concatenating some of these strings in any order. Each string can be used **at most once**, and the palindrome property requires the resulting string to read the same forward and backward.

The input, `words`, is an array of strings where each string is guaranteed to be **exactly two lowercase letters**. The expected output is the **length** of the longest palindrome that can be constructed from these strings.

Important constraints are that `words` can have up to `10^5` elements, so any approach that tries all permutations would be too slow. Strings are very short (length 2), so operations like reversing a string or comparing strings are inexpensive.

Edge cases to consider include arrays where all strings are palindromic themselves (like `"aa"`, `"cc"`), arrays where there are no reversible pairs, and arrays with multiple duplicates of the same string.

## Approaches

### Brute Force

A brute-force approach would be to generate all possible subsets of the array and check if their concatenation forms a palindrome. For each subset, one would check **all permutations**, because the order affects whether the resulting string is a palindrome. This guarantees correctness because it examines every possibility. However, the time complexity would be astronomical-on the order of O(n! * n) for checking permutations and palindrome properties, which is infeasible for `n = 10^5`.

### Optimal Approach

The key observation is that we only need to consider two types of strings: **palindromic strings** (`"aa"`, `"cc"`) and **reversible pairs** (`"ab"` and `"ba"`).

1. **Reversible pairs**: If a word `"xy"` exists and its reverse `"yx"` also exists, they can form a palindrome segment like `"xy" + ... + "yx"`. Each pair contributes exactly 4 characters to the palindrome length.
2. **Palindromic strings**: Words where both letters are the same can sit in the middle of a palindrome (like `"aa"`) or pair with another identical word to form a mirrored segment. Only one unpaired palindromic word can be placed at the center; the others must be used in pairs.

Using a **hash map (dictionary)** to count occurrences allows us to efficiently compute the maximum palindrome length.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n! * n) | O(n) | Generates all permutations and checks for palindrome |
| Optimal | O(n) | O(n) | Counts pairs using hash map, handles palindromic center efficiently |

## Algorithm Walkthrough

1. Initialize a hash map `count` to store the frequency of each word in `words`. Initialize `length` to 0 to track the palindrome length, and `used_center` as False.
2. Iterate over each word in `words`. For each word, compute its reverse.
3. If the word is **palindromic** (both letters are the same), increment a separate counter for palindromic words. Later, use as many as possible in pairs, and if one remains, place it in the center.
4. If the word is **not palindromic**, check if its reverse exists in the hash map with a non-zero count. If so, add 4 to `length` (since `"ab" + "ba"` is a palindrome), and decrement the counts of both the word and its reverse.
5. After processing all words, check if there is any **remaining palindromic word** to place in the center. If so, add 2 to `length`.
6. Return the computed `length`.

**Why it works**: This approach ensures that all reversible pairs contribute to the palindrome symmetrically, and at most one unpaired palindromic word is used in the center. The hash map guarantees each word is counted at most once, satisfying the “use at most once” constraint.

## Python Solution

```python
from collections import Counter
from typing import List

class Solution:
    def longestPalindrome(self, words: List[str]) -> int:
        count = Counter(words)
        length = 0
        used_center = False
        
        for word in count:
            reverse_word = word[::-1]
            if word[0] == word[1]:  # Palindromic word
                pairs = count[word] // 2
                length += pairs * 4
                count[word] -= pairs * 2
            elif reverse_word in count:
                pairs = min(count[word], count[reverse_word])
                length += pairs * 4
                count[word] -= pairs
                count[reverse_word] -= pairs
        
        # Check for center palindrome
        for word in count:
            if word[0] == word[1] and count[word] > 0:
                length += 2
                break
        
        return length
```

This implementation first counts occurrences of each word using `Counter`. It handles reversible pairs symmetrically and uses as many palindromic pairs as possible. At the end, it places a single unpaired palindromic word in the center if available.

## Go Solution

```go
func longestPalindrome(words []string) int {
    count := make(map[string]int)
    for _, word := range words {
        count[word]++
    }
    
    length := 0
    usedCenter := false
    
    for word, c := range count {
        reverseWord := string([]byte{word[1], word[0]})
        if word[0] == word[1] {
            pairs := c / 2
            length += pairs * 4
            count[word] -= pairs * 2
        } else if count[reverseWord] > 0 {
            pairs := min(c, count[reverseWord])
            length += pairs * 4
            count[word] -= pairs
            count[reverseWord] -= pairs
        }
    }
    
    for word, c := range count {
        if word[0] == word[1] && c > 0 {
            length += 2
            break
        }
    }
    
    return length
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

The Go version mirrors the Python logic. Map operations handle counts efficiently. Go requires explicit type conversions for string reversals and an explicit `min` function.

This complexity is optimal for the input constraints, and the algorithm scales linearly with the number of words.
