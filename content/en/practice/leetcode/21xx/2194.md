# LeetCode 2194 - Cells in a Range on an Excel Sheet

## Problem Understanding

This problem asks us to generate every Excel cell contained within a rectangular range. The input string describes the top left and bottom right corners of the range using Excel style coordinates.

Each cell consists of two parts:

- A column letter, such as `A`, `B`, or `Z`
- A row number, such as `1`, `2`, or `9`

The input format is always:

```
"<col1><row1>:<col2><row2>"
```

For example:

```
"K1:L2"
```

means the rectangle starts at cell `K1` and ends at cell `L2`.

The output should contain every cell inside this rectangle, ordered first by column and then by row. That means we fully process one column before moving to the next column.

For the range `"K1:L2"`, the valid cells are:

- Column `K`, rows `1` through `2`
- Column `L`, rows `1` through `2`

So the answer becomes:

```
["K1", "K2", "L1", "L2"]
```

The constraints are extremely small:

- The input length is always exactly `5`
- Columns range only from `A` to `Z`
- Rows range only from `1` to `9`

This tells us the search space is tiny. Even the largest possible rectangle contains only:

```
26 columns × 9 rows = 234 cells
```

Because the limits are so small, efficiency is not a concern. The main challenge is correctly parsing the input and generating cells in the required order.

An important guarantee is that:

```
c1 <= c2
r1 <= r2
```

So the range is always valid and already sorted from top left to bottom right. We never need to swap bounds or validate malformed ranges.

Potential edge cases include:

- A range consisting of a single cell
- A range with only one row
- A range with only one column
- The full possible range from `A1` to `Z9`

The problem guarantees all inputs are valid uppercase Excel coordinates, so we do not need defensive parsing logic.

## Approaches

### Brute Force Approach

The most direct solution is to iterate through every possible Excel cell and check whether it lies inside the requested rectangle.

We could loop through all columns from `A` to `Z` and all rows from `1` to `9`, then verify whether each cell satisfies:

```
col1 <= col <= col2
row1 <= row <= row2
```

If it does, we append it to the result.

This works because the search space is very small. However, it performs unnecessary work because most cells may not belong to the requested range.

For example, if the input is `"A1:B2"`, we only need 4 cells, but the brute force method still scans all 234 possible cells.

### Optimal Approach

A better observation is that we already know the exact boundaries of the rectangle.

Instead of checking every possible cell, we can directly generate only the required cells by:

1. Iterating columns from `col1` to `col2`
2. Iterating rows from `row1` to `row2`
3. Constructing each cell string immediately

This avoids unnecessary checks and naturally produces the required ordering.

The solution becomes a straightforward nested loop.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(26 × 9) | O(k) | Scans all possible cells and filters valid ones |
| Optimal | O(k) | O(k) | Directly generates only required cells |

Here, `k` represents the number of cells in the output.

## Algorithm Walkthrough

1. Parse the input string to extract the starting and ending columns and rows.

The string format is fixed, so:

- `s[0]` is the starting column
- `s[1]` is the starting row
- `s[3]` is the ending column
- `s[4]` is the ending row
2. Convert the row characters into integers.

Since rows are stored as characters like `'1'` or `'9'`, we convert them into integers so we can iterate numerically.
3. Create an empty result list.

This list will store all generated cell names.
4. Iterate through columns from the starting column to the ending column.

Characters can be converted into ASCII values using `ord()` in Python or byte arithmetic in Go.
5. For each column, iterate through rows from the starting row to the ending row.

This ordering is important because the problem requires sorting first by column and then by row.
6. Construct each cell string.

Concatenate the current column character with the row number.

Example:

```
column = 'K'
row = 2
cell = "K2"
```
7. Append the generated cell to the result list.
8. Return the final list after all loops finish.

### Why it works

The algorithm systematically enumerates every coordinate inside the rectangular boundary defined by the input range.

The outer loop guarantees columns appear in non decreasing order. The inner loop guarantees rows appear in non decreasing order within each column.

Because every valid column and row combination is generated exactly once, the result contains all required cells in the correct order.

## Python Solution

```python
from typing import List

class Solution:
    def cellsInRange(self, s: str) -> List[str]:
        start_col = s[0]
        start_row = int(s[1])

        end_col = s[3]
        end_row = int(s[4])

        result: List[str] = []

        for col_code in range(ord(start_col), ord(end_col) + 1):
            current_col = chr(col_code)

            for row in range(start_row, end_row + 1):
                result.append(current_col + str(row))

        return result
```

The implementation starts by extracting the two columns and two rows from the input string. Because the format is fixed, direct indexing is sufficient and avoids unnecessary parsing logic.

The row characters are converted into integers so they can be used inside numeric loops.

The outer loop iterates through the column range using ASCII values. `ord()` converts a character into its integer representation, and `chr()` converts it back into a character.

The inner loop iterates through the row range. For every `(column, row)` pair, we build the Excel coordinate string and append it to the result.

Finally, the result list is returned.

## Go Solution

```go
func cellsInRange(s string) []string {
    startCol := s[0]
    startRow := int(s[1] - '0')

    endCol := s[3]
    endRow := int(s[4] - '0')

    result := []string{}

    for col := startCol; col <= endCol; col++ {
        for row := startRow; row <= endRow; row++ {
            cell := string(col) + string(rune('0'+row))
            result = append(result, cell)
        }
    }

    return result
}
```

The Go implementation follows the same logic as the Python solution.

One notable difference is character handling. Go strings are byte based, so column letters can be processed directly as bytes.

Rows are single digit numbers according to the constraints, so converting them back into characters is simple:

```go
rune('0' + row)
```

Unlike Python, Go requires explicit conversions between bytes, runes, and strings.

The result slice is initialized as an empty slice and expanded dynamically using `append`.

## Edge Cases

### Single Cell Range

A range like `"D4:D4"` represents exactly one cell. This case can expose off by one errors in loop boundaries.

The implementation handles this correctly because both loops use inclusive ranges:

```
range(start, end + 1)
```

When the start and end are equal, the loop still executes exactly once.

### Single Row Range

A range such as `"A1:F1"` contains multiple columns but only one row.

Some incorrect implementations accidentally swap loop order or produce row major ordering instead of column major ordering.

This implementation correctly iterates columns first and rows second, which matches the required output order.

### Single Column Range

A range like `"C3:C5"` contains only one column but multiple rows.

This case verifies that the outer column loop still functions correctly when there is only one iteration. The inner row loop then generates all rows in order.

### Maximum Valid Range

The largest possible input is effectively something like `"A1:Z9"`.

Even though this generates the most cells possible under the constraints, the total number of cells is still only 234, which is extremely small.

The algorithm handles this efficiently because it performs only one operation per generated cell.
