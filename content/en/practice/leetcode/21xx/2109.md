## Solution

## Problem Understanding

The problem asks us to modify a string `s` by inserting spaces at specific positions given by an array `spaces`. Each element in `spaces` represents an index in the original string `s`, and a space should be inserted **before** the character at that index. For example, if `s = "LeetcodeHelpsMeLearn"` and `spaces = [8,13,15]`, a space is inserted before the characters at indices 8, 13, and 15, producing `"Leetcode Helps Me Learn"`.

The input consists of a string `s` of length up to 300,000 and a strictly increasing list of indices `spaces` of length up to 300,000. The fact that `spaces` is strictly increasing simplifies our algorithm because we do not need to handle duplicate or unordered positions.

The expected output is the string `s` with spaces inserted at the correct positions. Edge cases include inserting a space before the first character or having `spaces` cover consecutive characters. The problem guarantees valid indices within the string, so we do not need to handle out-of-bounds errors.

The constraints tell us that the solution must handle very large strings efficiently. A naive approach that repeatedly inserts into a string would be too slow because string concatenation is O(n) per insertion in many languages.

## Approaches

A brute-force approach would iterate through the `spaces` array and insert a space at the given index each time. In Python or Go, inserting into a string is O(n) because strings are immutable and require rebuilding. With `m` spaces and string length `n`, this approach has a worst-case time complexity of O(m * n), which is not feasible for `n, m ~ 3 * 10^5`.

The optimal approach uses a two-pointer technique or a single pass with a preallocated list or slice. Since the indices in `spaces` are sorted, we can traverse `s` from start to finish while keeping track of the next index where a space should be inserted. By building the result incrementally in a list or slice, we avoid repeated O(n) insertions, achieving linear time complexity O(n + m).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m * n) | O(n) | Inserts spaces directly into the string at each index, costly for large n and m |
| Optimal | O(n + m) | O(n + m) | Uses a preallocated list or slice and single pass through string and spaces array |

## Algorithm Walkthrough

1. Initialize an empty list or slice `result` to store characters, including inserted spaces. This avoids expensive repeated string concatenation.
2. Maintain a pointer `space_idx` to track the next index in the `spaces` array where a space should be inserted.
3. Iterate through the string `s` using an index `i` from 0 to `len(s) - 1`.
4. Before appending `s[i]` to `result`, check if `i` equals `spaces[space_idx]`. If it does, append a space to `result` and increment `space_idx` to point to the next space position.
5. Append the current character `s[i]` to `result`.
6. After iterating through the string, convert `result` back to a string and return it.

**Why it works**: Because the `spaces` array is strictly increasing, we can scan both the string and the `spaces` array in a single pass. Each character is processed exactly once, and each space is inserted exactly where needed. This guarantees correctness and achieves linear time complexity.

## Python Solution

```python
from typing import List

class Solution:
    def addSpaces(self, s: str, spaces: List[int]) -> str:
        result = []
        space_idx = 0
        n = len(spaces)
        
        for i in range(len(s)):
            if space_idx < n and i == spaces[space_idx]:
                result.append(' ')
                space_idx += 1
            result.append(s[i])
        
        return ''.join(result)
```

In this Python implementation, we initialize a list `result` to store the string with inserted spaces. `space_idx` tracks our current position in `spaces`. For each character in `s`, we check if the current index matches `spaces[space_idx]`. If it does, we append a space and increment `space_idx`. Finally, we join the list into a string. This efficiently handles large inputs because list appends are O(1).

## Go Solution

```go
func addSpaces(s string, spaces []int) string {
    result := make([]byte, 0, len(s)+len(spaces))
    spaceIdx := 0
    n := len(spaces)
    
    for i := 0; i < len(s); i++ {
        if spaceIdx < n && i == spaces[spaceIdx] {
            result = append(result, ' ')
            spaceIdx++
        }
        result = append(result, s[i])
    }
    
    return string(result)
}
```

In Go, we use a byte slice `result` with preallocated capacity `len(s) + len(spaces)` to minimize reallocations. `spaceIdx` tracks the next space to insert. We iterate through `s`, appending a space when needed and then the current character. Converting the byte slice back to a string produces the final result.

## Edge Cases

One important edge case is inserting a space at the very first character (`spaces[0] == 0`). Naive implementations that prepend a space might mishandle this if they do not check the first character separately. Our algorithm handles it naturally because it checks each index before appending the character.

Another edge case is when spaces occur consecutively, for example `spaces = [2,3,4]`. This could cause off-by-one errors in some implementations, but our algorithm works because we increment `space_idx` each time a space is inserted, keeping alignment correct.
