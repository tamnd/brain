## Solution

## Problem Understanding

This problem asks us to distribute numbers into slots in a way that maximizes a scoring function based on the bitwise AND operation.

We are given:

- An integer array `nums`
- A number of slots, `numSlots`
- Each slot is numbered from `1` to `numSlots`
- Every slot can contain at most two numbers

For every number placed into a slot, we earn:

```
number AND slot_number
```

The final score is the sum of all these contributions.

The goal is to place every number into some slot so that the total AND sum is as large as possible.

The constraints are extremely important for identifying the correct approach:

- `numSlots <= 9`
- Each slot can hold at most two numbers
- Therefore, total capacity is at most `18`
- `n <= 18`

A naive assignment search would be enormous because each number could potentially go into many slots. However, the very small upper bound on total capacity strongly suggests using state compression dynamic programming with bitmasks.

The bitwise AND operation also matters conceptually. A number contributes more when placed into a slot whose binary representation shares many set bits with the number. Since slot numbers are small, trying all possible assignments intelligently through DP becomes feasible.

There are several important edge cases:

- A slot may remain empty, which means we do not need to fill every slot
- Some numbers may contribute `0` to many slots
- `numSlots` can be much larger than necessary, creating many unused positions
- Multiple numbers may compete for the same high-value slot
- Since each slot only allows two numbers, capacity tracking is essential

The problem guarantees:

- Every number can be placed because `2 * numSlots >= n`
- All values are positive integers
- Slot numbers are small enough for bitmask DP to work efficiently

## Approaches

### Brute Force Approach

The brute-force approach tries every possible assignment of numbers into slots.

For each number, we attempt placing it into every slot that still has remaining capacity. Once all numbers are assigned, we compute the resulting AND sum and keep the maximum.

This approach is correct because it explores every valid placement configuration. Eventually, it must encounter the optimal arrangement.

However, the runtime becomes prohibitively large. Suppose there are `n` numbers and `numSlots` slots. Each number may branch into several choices, producing an exponential search tree. Even with pruning from slot capacities, the total number of states grows far too quickly.

With up to `18` placements possible, brute force is not practical.

### Optimal Approach, Bitmask Dynamic Programming

The key observation is that each slot can contain at most two numbers.

Instead of thinking in terms of slots directly, we can model the problem as having `2 * numSlots` individual positions:

- Position `0` and `1` belong to slot `1`
- Position `2` and `3` belong to slot `2`
- Position `4` and `5` belong to slot `3`
- and so on

Now the problem becomes:

- Assign each number to one unused position
- The slot number for a position is `(position // 2) + 1`

This transformation is powerful because it lets us represent used positions with a bitmask.

We define:

- A bitmask of size `2 * numSlots`
- A set bit means that position is already occupied

The number of set bits tells us how many numbers have already been placed, which directly identifies the next number to assign.

Dynamic programming then explores all mask states efficiently.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Exponential, roughly O(numSlots^n) | O(n) recursion stack | Tries every assignment recursively |
| Optimal | O((2^(2_numSlots)) * (2_numSlots)) | O(2^(2*numSlots)) | Uses bitmask DP over slot positions |

## Algorithm Walkthrough

### Step 1: Represent Each Slot as Two Positions

Each slot can contain at most two numbers.

Instead of tracking capacities separately, we create exactly two positions per slot.

For example, if `numSlots = 3`, we create:

| Position | Slot |
| --- | --- |
| 0 | 1 |
| 1 | 1 |
| 2 | 2 |
| 3 | 2 |
| 4 | 3 |
| 5 | 3 |

This converts the problem into assigning numbers into distinct positions.

### Step 2: Define the DP State

We use a bitmask where:

- Bit `i` is `1` if position `i` is already occupied
- Bit `i` is `0` otherwise

If there are `k` set bits in the mask, then exactly `k` numbers have already been assigned.

Therefore:

```
current_number_index = popcount(mask)
```

This is important because we do not need another DP dimension for the index.

### Step 3: Define the Transition

For the current number:

```
nums[popcount(mask)]
```

we try placing it into every unused position.

For each unused position:

- Determine its slot number
- Compute the contribution:

```
nums[index] & slot_number
```

- Create the next mask by setting that bit
- Recursively compute the best future score

### Step 4: Memoize Results

Many assignment paths reach the same mask.

Without memoization, we would recompute identical subproblems repeatedly.

We store:

```sql
dp[mask] = maximum AND sum achievable from this state
```

This reduces the complexity dramatically.

### Step 5: Base Case

When all numbers are assigned:

```
popcount(mask) == len(nums)
```

there are no remaining decisions, so the score contribution is `0`.

### Why it works

The DP state uniquely captures which slot positions are already occupied. Since the next number to place is determined entirely by how many positions are filled, every future decision depends only on the current mask.

The recursion explores every valid placement exactly once per state, and memoization guarantees optimal substructure. Therefore, the DP computes the maximum achievable AND sum.

## Python Solution

```python
from typing import List
from functools import lru_cache

class Solution:
    def maximumANDSum(self, nums: List[int], numSlots: int) -> int:
        total_positions = 2 * numSlots
        n = len(nums)

        @lru_cache(None)
        def dp(mask: int) -> int:
            used_count = mask.bit_count()

            if used_count == n:
                return 0

            best = 0
            current_num = nums[used_count]

            for position in range(total_positions):
                if (mask & (1 << position)) == 0:
                    slot_number = (position // 2) + 1

                    score = (
                        (current_num & slot_number)
                        + dp(mask | (1 << position))
                    )

                    best = max(best, score)

            return best

        return dp(0)
```

The implementation closely follows the DP formulation.

We first compute the total number of available positions, which is twice the number of slots.

The recursive function `dp(mask)` represents the maximum AND sum achievable starting from the current occupancy state.

The number of assigned numbers is determined using:

```python
mask.bit_count()
```

This tells us which number from `nums` should be placed next.

For every unused position, we:

- Determine the corresponding slot
- Compute the immediate contribution
- Recursively evaluate the remaining placements
- Keep the best result

Memoization through `lru_cache` ensures every mask state is computed only once.

The recursion terminates once all numbers are assigned.

## Go Solution

```go
package main

import "math/bits"

func maximumANDSum(nums []int, numSlots int) int {
	totalPositions := 2 * numSlots
	n := len(nums)

	dp := make(map[int]int)

	var dfs func(mask int) int

	dfs = func(mask int) int {
		usedCount := bits.OnesCount(uint(mask))

		if usedCount == n {
			return 0
		}

		if value, exists := dp[mask]; exists {
			return value
		}

		best := 0
		currentNum := nums[usedCount]

		for position := 0; position < totalPositions; position++ {
			if (mask & (1 << position)) == 0 {
				slotNumber := (position / 2) + 1

				score := (currentNum & slotNumber) +
					dfs(mask|(1<<position))

				if score > best {
					best = score
				}
			}
		}

		dp[mask] = best
		return best
	}

	return dfs(0)
}
```

The Go implementation mirrors the Python solution almost exactly.

Since Go does not provide built-in memoization decorators, we explicitly use a map:

```go
map[int]int
```

to cache DP states.

The standard library function:

```go
bits.OnesCount()
```

efficiently computes the number of set bits in the mask.

Go integers are fully sufficient here because the mask size never exceeds 18 bits.

## Edge Cases

### Single Number Input

When there is only one number, the algorithm must still correctly evaluate all possible slots and choose the best one.

A buggy implementation might assume every slot must be filled sequentially or fail to handle sparse usage correctly. Our DP naturally handles this because it explores every available position independently.

### More Slots Than Numbers

In cases like:

```
nums = [1,3,10,4,7,1]
numSlots = 9
```

many slots remain unused.

Some implementations incorrectly force all slots to participate or track occupancy inefficiently. Our position-based bitmask representation avoids this issue entirely because unused positions simply remain unset.

### Contributions Equal to Zero

Some numbers may produce zero with certain slots.

Example:

```
2 & 1 = 0
```

A greedy algorithm could accidentally place numbers into poor slots too early. The DP evaluates all future consequences before committing to a choice, ensuring globally optimal placement even when many assignments yield zero immediate gain.

### Full Capacity Usage

When:

```
n = 2 * numSlots
```

every slot position must be used.

This is the densest possible configuration and maximizes the number of DP states explored. The memoized bitmask DP still handles it efficiently because the total number of states remains bounded by `2^18`.
