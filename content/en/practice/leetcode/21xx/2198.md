# LeetCode 2198, Number of Single Divisor Triplets

## Problem Understanding

The problem gives us an array `nums` containing positive integers. We must count how many ordered triplets of distinct indices `(i, j, k)` satisfy a special divisibility condition.

For a triplet to qualify:

- The indices must all be different.
- Let the sum be:

```
nums[i] + nums[j] + nums[k]
```

- This sum must be divisible by exactly one of the three numbers.

That means:

- The sum is divisible by one of the values.
- The sum is not divisible by the other two values.

An important detail is that the problem counts ordered triplets, not combinations. This means `(i, j, k)` and `(j, i, k)` are considered different if the indices appear in a different order.

For example:

```
nums = [1,2,2]
```

The triplet using indices `(0,1,2)` corresponds to values `[1,2,2]`.

The sum is:

```
1 + 2 + 2 = 5
```

`5` is divisible only by `1`, so this triplet is valid.

Since all permutations of the three distinct indices count separately, there are `3! = 6` ordered triplets.

The constraints are especially important:

- `nums.length <= 100000`
- `nums[i] <= 100`

The array can be extremely large, so iterating over all index triplets directly is impossible. A naive `O(n^3)` solution would time out immediately for large inputs.

However, the values themselves are very small, only from `1` to `100`. This strongly suggests that we should count frequencies of values instead of iterating over indices directly.

Several edge cases are important:

- Arrays with all equal numbers.
- Arrays containing many duplicates.
- Triplets where the sum is divisible by more than one element.
- Triplets where the sum is divisible by none of the elements.
- Cases where values repeat, because distinct indices are required even when values are equal.

The key challenge is correctly handling multiplicities while still counting ordered triplets.

## Approaches

### Brute Force Approach

The most direct solution is to try every possible triplet of distinct indices.

For every `(i, j, k)`:

1. Compute the sum.
2. Check divisibility against each of the three numbers.
3. Count how many divide the sum.
4. If exactly one divides the sum, increment the answer.

This approach is easy to reason about and obviously correct because it explicitly checks every valid candidate.

However, the complexity is far too large.

If `n = 100000`, then:

```
O(n^3) = 10^15
```

operations are completely infeasible.

### Key Insight for the Optimal Solution

The crucial observation is that the values are very small.

Although the array length can reach `100000`, every number lies between `1` and `100`.

Instead of iterating over indices, we can iterate over values.

We first count how many times each value appears. Then we enumerate all possible value triplets `(a, b, c)` where each value is between `1` and `100`.

There are at most:

```
100 * 100 * 100 = 1,000,000
```

possible value triplets, which is manageable.

For each value triplet:

1. Check whether the divisibility condition holds.
2. Determine how many index triplets can produce these values.
3. Add the correct number of ordered permutations.

This transforms the problem from depending on `n` to depending on the small value range.

| --- | --- | --- | --- |

## Algorithm Walkthrough

### Step 1, Count Frequencies

Create a frequency array or hash map storing how many times each value appears.

Since values range only from `1` to `100`, an array of size `101` is sufficient.

For example:

```
nums = [4,6,7,3,2]
```

produces:

```
freq[2] = 1
freq[3] = 1
freq[4] = 1
freq[6] = 1
freq[7] = 1
```

This allows us to work with counts instead of raw indices.

### Step 2, Enumerate All Value Triplets

Try every possible value combination:

```sql
a from 1 to 100
b from 1 to 100
c from 1 to 100
```

Skip any triplet where one of the values does not exist in the array.

### Step 3, Check the Divisibility Condition

Compute:

```
total = a + b + c
```

Count how many of these are true:

```
total % a == 0
total % b == 0
total % c == 0
```

If exactly one condition is true, then this value triplet is valid.

### Step 4, Compute How Many Index Triplets Exist

Now we determine how many distinct index triplets correspond to `(a, b, c)`.

There are three cases.

#### Case 1, All Values Different

If:

```
a != b != c
```

then the number of ordered triplets is:

```
freq[a] * freq[b] * freq[c]
```

because each position independently chooses an index.

### Case 2, Two Values Equal

Suppose:

```
a == b != c
```

Then we must choose two distinct indices for value `a`.

The number of ways is:

```
freq[a] * (freq[a] - 1)
```

because order matters.

Then multiply by:

```
freq[c]
```

The same logic applies for the other equality patterns.

### Case 3, All Values Equal

If:

```
a == b == c
```

then we need three distinct indices.

The number of ordered selections is:

```
freq[a] * (freq[a] - 1) * (freq[a] - 2)
```

### Step 5, Add to the Answer

Accumulate all valid ordered triplet counts.

### Why it works

The algorithm works because every valid ordered index triplet corresponds to exactly one ordered value triplet `(a, b, c)`.

The divisibility check correctly determines whether the triplet satisfies the problem condition. The combinational counting formulas correctly count how many distinct ordered index selections produce those values.

Since all possible value triplets are examined and counted exactly once, the final answer is correct.

## Python Solution

```python
from typing import List

class Solution:
    def singleDivisorTriplet(self, nums: List[int]) -> int:
        freq = [0] * 101
        
        for num in nums:
            freq[num] += 1
        
        answer = 0
        
        for a in range(1, 101):
            if freq[a] == 0:
                continue
            
            for b in range(1, 101):
                if freq[b] == 0:
                    continue
                
                for c in range(1, 101):
                    if freq[c] == 0:
                        continue
                    
                    total = a + b + c
                    
                    divisible_count = 0
                    
                    if total % a == 0:
                        divisible_count += 1
                    
                    if total % b == 0:
                        divisible_count += 1
                    
                    if total % c == 0:
                        divisible_count += 1
                    
                    if divisible_count != 1:
                        continue
                    
                    if a == b == c:
                        if freq[a] >= 3:
                            answer += (
                                freq[a]
                                * (freq[a] - 1)
                                * (freq[a] - 2)
                            )
                    
                    elif a == b:
                        if freq[a] >= 2:
                            answer += (
                                freq[a]
                                * (freq[a] - 1)
                                * freq[c]
                            )
                    
                    elif a == c:
                        if freq[a] >= 2:
                            answer += (
                                freq[a]
                                * (freq[a] - 1)
                                * freq[b]
                            )
                    
                    elif b == c:
                        if freq[b] >= 2:
                            answer += (
                                freq[b]
                                * (freq[b] - 1)
                                * freq[a]
                            )
                    
                    else:
                        answer += freq[a] * freq[b] * freq[c]
        
        return answer
```

The implementation begins by building a frequency array. This converts the large input array into compact counts over the small value range.

The triple nested loops enumerate every ordered value triplet `(a, b, c)`. Because values are limited to `100`, this remains efficient.

For each triplet, the code computes the sum and checks divisibility against all three values. Only triplets where exactly one divisor condition holds are considered valid.

The next section handles the combinational counting carefully. The formulas differ depending on whether values repeat. Since indices must be distinct, repeated values require permutations without replacement.

Finally, all valid ordered counts are accumulated into `answer`.

## Go Solution

```go
func singleDivisorTriplet(nums []int) int64 {
    freq := make([]int64, 101)

    for _, num := range nums {
        freq[num]++
    }

    var answer int64 = 0

    for a := 1; a <= 100; a++ {
        if freq[a] == 0 {
            continue
        }

        for b := 1; b <= 100; b++ {
            if freq[b] == 0 {
                continue
            }

            for c := 1; c <= 100; c++ {
                if freq[c] == 0 {
                    continue
                }

                total := a + b + c

                divisibleCount := 0

                if total%a == 0 {
                    divisibleCount++
                }

                if total%b == 0 {
                    divisibleCount++
                }

                if total%c == 0 {
                    divisibleCount++
                }

                if divisibleCount != 1 {
                    continue
                }

                if a == b && b == c {
                    if freq[a] >= 3 {
                        answer += freq[a] *
                            (freq[a] - 1) *
                            (freq[a] - 2)
                    }

                } else if a == b {
                    if freq[a] >= 2 {
                        answer += freq[a] *
                            (freq[a] - 1) *
                            freq[c]
                    }

                } else if a == c {
                    if freq[a] >= 2 {
                        answer += freq[a] *
                            (freq[a] - 1) *
                            freq[b]
                    }

                } else if b == c {
                    if freq[b] >= 2 {
                        answer += freq[b] *
                            (freq[b] - 1) *
                            freq[a]
                    }

                } else {
                    answer += freq[a] * freq[b] * freq[c]
                }
            }
        }
    }

    return answer
}
```

The Go implementation follows the same logic as the Python version. The main difference is that the return type is `int64`, so the frequency array and accumulator are also stored as `int64`.

This avoids overflow because the number of ordered triplets can become large when the array contains many repeated values.

Go slices are initialized with zero values automatically, so the frequency array starts correctly populated with zeros.

## Edge Cases

### All Values Equal

Consider:

```
[1,1,1]
```

A common mistake is to think divisibility by `1` counts only once. However, the problem checks divisibility against each of the three elements individually. Since all three elements divide the sum, the triplet is invalid.

The implementation correctly counts how many of the three divisibility checks succeed and requires the count to equal exactly one.

### Two Values Equal

Consider:

```
[1,2,2]
```

Handling duplicates incorrectly can easily produce undercounting or overcounting. The challenge is that indices must still be distinct even when values are the same.

The implementation uses:

```
freq[x] * (freq[x] - 1)
```

for selecting two ordered distinct indices with the same value. This correctly accounts for permutations.

### Large Input Size

With:

```
nums.length = 100000
```

a naive cubic solution is impossible.

The optimized approach avoids iterating over indices entirely. Instead, it depends only on the bounded value range `[1,100]`, making the runtime effectively constant relative to the maximum input size.
