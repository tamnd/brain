## Solution

## Problem Understanding

The problem asks us to find the minimum total time to complete a race consisting of `numLaps` laps, given multiple tire types with specific performance characteristics and a fixed tire change time. Each tire has two properties, `fi` and `ri`. The time for the `xth` consecutive lap on the same tire is `fi * ri^(x-1)`. This means tires get progressively slower if used consecutively because `ri` is greater than or equal to 2. After any lap, we may either continue with the same tire or change to a fresh tire of any type by waiting `changeTime` seconds. We can start with any tire type.

The input consists of a 2D array `tires` of length up to 100,000, each entry with two integers, `fi` and `ri`, which can be as large as 100,000. The number of laps `numLaps` is up to 1000, which is small relative to the tire types. The expected output is a single integer representing the minimum total time to finish all laps.

Edge cases to consider include very large `fi` and `ri`, very high `changeTime`, very small `numLaps` (like 1), and cases where using a single tire consecutively might be worse than changing tires after each lap. The problem guarantees at least one tire and at least one lap, which simplifies boundary checks.

## Approaches

A brute-force approach would simulate every possible sequence of tire usages over `numLaps` laps. For each lap, we would try continuing with the current tire or changing to any other tire. While this guarantees the correct answer, it is computationally infeasible because the number of sequences grows exponentially with the number of laps and tire types.

The key insight for an optimal solution is that we only need to consider **streaks of consecutive laps using a single tire without changing**, because using a tire too many times consecutively increases the lap time exponentially due to `ri`. Thus, for each tire type, we can precompute the minimum time required to complete `k` consecutive laps before it becomes more optimal to change tires. Once we know the minimum times for consecutive laps, the problem reduces to a **dynamic programming problem**, where `dp[i]` is the minimum time to complete `i` laps. For each `i`, we try all possible streak lengths from 1 to a precomputed maximum `maxStreak`, combining the minimum streak times and `changeTime` as necessary.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(len(tires)^numLaps) | O(numLaps) | Tries all sequences of tire changes; infeasible for constraints |
| Optimal | O(numLaps * maxStreak + len(tires) * maxStreak) | O(numLaps + maxStreak) | Precompute minimal streaks per tire, then use DP for overall minimum time |

## Algorithm Walkthrough

1. **Precompute minimal streak times for each tire:** Iterate over each tire type `(fi, ri)` and calculate the cumulative time for consecutive laps until adding another lap becomes slower than changing the tire plus `changeTime`. Keep track of the minimum cumulative times for each streak length.
2. **Determine `maxStreak`:** This is the largest number of consecutive laps that is ever beneficial to run on the same tire without changing. Any streak beyond this length is always worse than changing the tire.
3. **Dynamic Programming array initialization:** Create a `dp` array of size `numLaps + 1`, where `dp[i]` represents the minimum time to finish `i` laps. Initialize `dp[0]` to 0, representing no laps completed.
4. **Fill DP array:** For each lap count `i` from 1 to `numLaps`, iterate over possible streak lengths `k` from 1 to `min(i, maxStreak)`. Update `dp[i]` as the minimum of `dp[i]` and `dp[i-k] + minTimeForK[k] + changeTime` (add `changeTime` only if `i != k` since the first streak does not require a prior change).
5. **Return the result:** `dp[numLaps]` will contain the minimum total time to complete all laps.

Why it works: The algorithm guarantees the minimal time because it considers every beneficial streak length and uses dynamic programming to accumulate the minimal total times, ensuring we always pick the globally optimal combination of streaks and tire changes.

## Python Solution

```python
from typing import List
import math

class Solution:
    def minimumFinishTime(self, tires: List[List[int]], changeTime: int, numLaps: int) -> int:
        # Step 1: Precompute minimal streak times per tire
        maxStreak = 0
        minTimeForK = [math.inf] * (numLaps + 1)
        for f, r in tires:
            time = f
            streak = 1
            total = f
            while streak <= numLaps and time <= f + changeTime:
                if total < minTimeForK[streak]:
                    minTimeForK[streak] = total
                streak += 1
                time *= r
                total += time
            maxStreak = max(maxStreak, streak - 1)
        
        # Step 2: Dynamic programming over lap counts
        dp = [math.inf] * (numLaps + 1)
        dp[0] = 0
        for i in range(1, numLaps + 1):
            for k in range(1, min(i, maxStreak) + 1):
                if i == k:
                    dp[i] = min(dp[i], minTimeForK[k])
                else:
                    dp[i] = min(dp[i], dp[i - k] + changeTime + minTimeForK[k])
        
        return dp[numLaps]
```

The implementation first precomputes the minimal cumulative lap times for each tire without changing, stopping when adding another lap exceeds the cost of changing. Then, dynamic programming builds the minimum total time to complete all laps, considering all valid streak lengths.

## Go Solution

```go
import (
    "math"
)

func minimumFinishTime(tires [][]int, changeTime int, numLaps int) int {
    maxStreak := 0
    minTimeForK := make([]int, numLaps+1)
    for i := range minTimeForK {
        minTimeForK[i] = math.MaxInt32
    }

    for _, tire := range tires {
        f, r := tire[0], tire[1]
        time, total, streak := f, f, 1
        for streak <= numLaps && time <= f+changeTime {
            if total < minTimeForK[streak] {
                minTimeForK[streak] = total
            }
            streak++
            time *= r
            total += time
        }
        if streak-1 > maxStreak {
            maxStreak = streak - 1
        }
    }

    dp := make([]int, numLaps+1)
    for i := range dp {
        dp[i] = math.MaxInt32
    }
    dp[0] = 0

    for i := 1; i <= numLaps; i++ {
        for k := 1; k <= i && k <= maxStreak; k++ {
            if i == k {
                if minTimeForK[k] < dp[i] {
                    dp[i] = minTimeForK[k]
                }
            } else {
                if dp[i-k]+changeTime+minTimeForK[k] < dp[i] {
                    dp[i] = dp[i-k] + changeTime + minTimeForK[k]
                }
            }
        }
    }

    return dp[numLaps]
}
```

In Go, slices are used for dynamic arrays, and `math.MaxInt32` represents infinity. The logic mirrors the Python solution with attention to Go syntax.

`minTimeForK` = [inf, 2, 8, inf, inf], `maxStreak=2`.
2. DP:

`dp[1] = 2`

`dp[2] = 8`

`dp[3] = dp[1]+5+minTimeForK[2]=2+5+8=15`

`dp[4] = dp[2]+5+minTimeForK[2]=8+5+8=21`

Result: 21
