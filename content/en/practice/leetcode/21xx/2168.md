## Solution

## Problem Understanding

The problem asks us to count the number of **unique substrings** of a given digit string `s` such that **every digit in the substring occurs the same number of times**. A substring is any contiguous portion of the string. The input string consists only of digits `0` through `9`, and its length can be up to 1000. The output is a single integer, representing the count of such substrings, where duplicates are counted only once.

In other words, for each substring, we must check if all digits that appear in it occur with equal frequency. For instance, in `"1212"`, the substring `"12"` has both digits `1` and `2` appearing once, which is valid. A substring `"121"` is invalid if any digit appears more times than others.

Important constraints and observations include that `s` can be moderately long (up to 1000), so naive solutions iterating over all substrings and checking their frequencies could be expensive. We need a method that avoids counting duplicates and efficiently checks frequency equality. Edge cases to consider include single-character strings, strings with repeated digits, and strings where all characters are unique.

## Approaches

### Brute Force Approach

The brute-force approach iterates over all possible substrings of `s`, counts the frequency of each digit in the substring, and checks whether all digits appear the same number of times. If they do, the substring is added to a set to ensure uniqueness.

This approach is correct because it explicitly examines every substring and verifies the condition, but it is inefficient: there are O(n²) substrings and checking the frequency takes O(n) for each, giving a total complexity of O(n³), which is too slow for `n = 1000`.

### Optimal Approach

The key insight for a faster solution is that we can use a **rolling frequency count and hash-based uniqueness check**. By treating the frequency of digits as a vector of counts, we can generate a unique hash for each substring. Since the maximum frequency of any digit in a substring cannot exceed the substring length, we only need to iterate over substrings starting at each index and maintain a count array of length 10. We expand the substring one character at a time, updating the count array, and at each step, we check if the nonzero counts are all equal. This avoids recomputing the frequency from scratch for each substring and ensures uniqueness using a hash set. This reduces the practical runtime to O(n² * 10) = O(n²), which is acceptable for n ≤ 1000.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(n²) | Iterate all substrings, check frequency each time |
| Optimal | O(n²) | O(n²) | Maintain running counts and check equality, store unique substrings in a set |

## Algorithm Walkthrough

1. Initialize a set `unique_substrings` to store substrings that meet the condition. Using a set ensures we count each substring only once.
2. Iterate over all possible starting indices `i` of the substring.
3. For each starting index, initialize an array `count` of size 10 to zero, representing the frequency of each digit from `0` to `9`.
4. Expand the substring one character at a time by increasing the ending index `j`. For each new character `s[j]`, increment its corresponding count in `count`.
5. Extract the nonzero counts from `count` and check if all nonzero counts are equal. If they are, add the substring `s[i:j+1]` to `unique_substrings`.
6. Continue expanding the substring until the end of the string.
7. After iterating all possible substrings, return the size of `unique_substrings`.

**Why it works:** The algorithm maintains an accurate frequency count of digits as substrings are expanded. By checking that all nonzero counts are equal at each expansion, it guarantees that only valid substrings are counted. Using a set ensures uniqueness, so duplicates are not overcounted.

## Python Solution

```python
class Solution:
    def equalDigitFrequency(self, s: str) -> int:
        unique_substrings = set()
        
        for i in range(len(s)):
            count = [0] * 10
            for j in range(i, len(s)):
                digit = int(s[j])
                count[digit] += 1
                nonzero_counts = [c for c in count if c > 0]
                if len(set(nonzero_counts)) == 1:
                    unique_substrings.add(s[i:j+1])
        
        return len(unique_substrings)
```

In this implementation, we use a nested loop to explore all substrings starting from each index. `count` keeps track of digit occurrences efficiently, and `set(nonzero_counts)` allows a quick check if all digits in the substring appear equally. Adding the substring to a set guarantees uniqueness. The approach directly follows the algorithm walkthrough.

## Go Solution

```go
func equalDigitFrequency(s string) int {
    uniqueSubstrings := make(map[string]struct{})
    
    for i := 0; i < len(s); i++ {
        count := [10]int{}
        for j := i; j < len(s); j++ {
            digit := s[j] - '0'
            count[digit]++
            
            freq := 0
            valid := true
            for _, c := range count {
                if c > 0 {
                    if freq == 0 {
                        freq = c
                    } else if c != freq {
                        valid = false
                        break
                    }
                }
            }
            if valid {
                uniqueSubstrings[s[i:j+1]] = struct{}{}
            }
        }
    }
    
    return len(uniqueSubstrings)
}
```

In Go, we use a map to store unique substrings. The logic is the same as Python, with attention to Go-specific details such as using a fixed-size array for counts and handling string-to-digit conversion using `s[j] - '0'`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | Outer loop iterates over start index, inner loop iterates over end index. For each substring, checking counts is O(10) = O(1). |
| Space | O(n²) | In worst case, all substrings are unique and stored in the set. |

The time complexity is acceptable because `n ≤ 1000` and 10 operations per substring are negligible. Space complexity depends on the number of unique substrings.
