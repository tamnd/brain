## Solution

## Problem Understanding

The problem is asking us to find the **largest square** that can fit inside the **intersection of at least two rectangles** on a 2D plane. We are given two lists: `bottomLeft` and `topRight`, where each entry corresponds to a rectangle with its bottom-left and top-right coordinates. Each rectangle’s edges are parallel to the x and y axes, meaning they are axis-aligned.

The expected output is the **area** of the largest square that can fit inside any overlapping region formed by at least two rectangles. If no two rectangles intersect, the answer is `0`.

The constraints indicate that the number of rectangles `n` can go up to 1000, and coordinates can be very large (up to $10^7$). This implies that naive grid-based methods will be inefficient, so the solution should rely on geometry rather than brute-force enumeration of points. Also, rectangles can have varying sizes, and their intersections may be very small or non-existent.

Important edge cases include:

- No two rectangles intersect, which should return `0`.
- Multiple overlapping regions, where the largest square might not be at the center of any rectangle.
- Rectangles with minimal overlap, such as touching at a line or a point.
- Large coordinate values, requiring care to avoid overflow or precision errors.

## Approaches

### Brute Force

A naive brute-force approach would attempt to check **all pairs of rectangles**, compute their intersection, and then determine the maximum square that fits inside that intersected rectangle. For each rectangle pair, the intersection rectangle can be computed using:

```
intersection_left = max(bottomLeft[i][0], bottomLeft[j][0])
intersection_bottom = max(bottomLeft[i][1], bottomLeft[j][1])
intersection_right = min(topRight[i][0], topRight[j][0])
intersection_top = min(topRight[i][1], topRight[j][1])
```

The side of the largest square that fits is:

```
side_length = min(intersection_right - intersection_left, intersection_top - intersection_bottom)
```

Then, the area is `side_length^2`. Keep track of the maximum area over all pairs.

While correct, this approach has a **time complexity of O(n^2)**, which is acceptable for $n \leq 1000$, but further optimizations may be possible. Space complexity is O(1) as we are just keeping track of the maximum.

### Optimal Insight

The key insight is that **the maximum square side that can fit inside intersections of rectangles is determined entirely by overlapping coordinates**, and we do not need to consider all subsets beyond pairs since the problem asks for intersections of **at least two rectangles**. We can still use the pairwise intersection method efficiently because n is moderate.

This leads to an algorithm that loops over all pairs, computes the intersection, and updates the maximum square area. Since each rectangle is axis-aligned, the math is straightforward.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check every pair of rectangles and compute intersection |
| Optimal | O(n^2) | O(1) | Same as brute force but optimal for n <= 1000; no grid or extra structures needed |

## Algorithm Walkthrough

1. Initialize `max_side` to 0 to keep track of the maximum square side found.
2. Loop over each rectangle `i` from 0 to n-2.
3. Inside that loop, loop over each rectangle `j` from `i+1` to n-1.
4. Compute the intersection of rectangles `i` and `j`:

- Left = `max(bottomLeft[i][0], bottomLeft[j][0])`
- Bottom = `max(bottomLeft[i][1], bottomLeft[j][1])`
- Right = `min(topRight[i][0], topRight[j][0])`
- Top = `min(topRight[i][1], topRight[j][1])`
5. If `Right > Left` and `Top > Bottom`, there is a valid intersection rectangle.
6. Compute the side of the largest square inside this intersection as `min(Right - Left, Top - Bottom)`.
7. Update `max_side` if this side is larger than the current maximum.
8. After checking all pairs, return `max_side * max_side` as the area.

**Why it works:** By iterating over all pairs, we guarantee that we consider every possible intersection between two rectangles. The largest square that fits in any intersection must fit entirely within the intersecting rectangle, so taking the minimum of width and height ensures the square fits. This method covers all valid intersections and thus correctly finds the maximum area.

## Python Solution

```python
from typing import List

class Solution:
    def largestSquareArea(self, bottomLeft: List[List[int]], topRight: List[List[int]]) -> int:
        n = len(bottomLeft)
        max_side = 0
        
        for i in range(n - 1):
            for j in range(i + 1, n):
                left = max(bottomLeft[i][0], bottomLeft[j][0])
                bottom = max(bottomLeft[i][1], bottomLeft[j][1])
                right = min(topRight[i][0], topRight[j][0])
                top = min(topRight[i][1], topRight[j][1])
                
                if right > left and top > bottom:
                    side = min(right - left, top - bottom)
                    max_side = max(max_side, side)
        
        return max_side * max_side
```

This Python solution loops over all rectangle pairs, computes the intersection rectangle coordinates, checks if the intersection is valid, and then computes the largest square that fits. It keeps the maximum square side seen so far and finally returns the area.

## Go Solution

```go
func largestSquareArea(bottomLeft [][]int, topRight [][]int) int64 {
    n := len(bottomLeft)
    var maxSide int64 = 0
    
    for i := 0; i < n-1; i++ {
        for j := i + 1; j < n; j++ {
            left := max(bottomLeft[i][0], bottomLeft[j][0])
            bottom := max(bottomLeft[i][1], bottomLeft[j][1])
            right := min(topRight[i][0], topRight[j][0])
            top := min(topRight[i][1], topRight[j][1])
            
            if right > left && top > bottom {
                side := int64(min(right-left, top-bottom))
                if side > maxSide {
                    maxSide = side
                }
            }
        }
    }
    
    return maxSide * maxSide
}

func max(a, b int) int {
    if a > b { return a }
    return b
}

func min(a, b int) int {
    if a < b { return a }
    return b
}
```

In Go, we explicitly cast to `int64` when computing the area to avoid integer overflow, especially given that coordinates can go up to $10^7$. The rest follows the same logic as the Python implementation.
