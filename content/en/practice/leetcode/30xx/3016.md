## Solution

## Problem Understanding

The problem asks us to minimize the total number of key presses required to type a given string `word` on a telephone keypad. Each key on the keypad can be mapped to any number of **distinct letters**, and every letter must appear on exactly one key. Typing a letter requires pressing the key a certain number of times equal to the letter's position on that key. For instance, the first letter assigned to a key takes 1 push, the second letter takes 2 pushes, and so on.

The input is a string of lowercase English letters, and the output is the minimum total number of key presses needed after optimally remapping letters to keys. The constraints tell us that `word` can be up to $10^5$ characters long, which means a brute-force approach that examines all key-to-letter mappings is infeasible. We also need to handle repeated letters efficiently because their frequency affects the total cost.

Important edge cases include words where all letters are identical, where letters appear only once, or where the number of distinct letters exceeds the total number of key slots available (8 keys times the number of letters per key).

## Approaches

A brute-force approach would try all possible mappings of letters to the keys. For each mapping, we would compute the total key presses needed. This guarantees correctness but is computationally impossible because there are 26 letters and only 8 keys, leading to an enormous number of permutations.

The key insight for an optimal solution is that letters with higher frequency should occupy positions that require fewer key presses. Since each key can hold multiple letters, the minimum cost is achieved by assigning the most frequent letters to the first positions across the keys, the next most frequent letters to the second positions, and so on. Sorting letters by frequency and greedily assigning them to key positions ensures the minimum total number of pushes.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(8^26) | O(26) | Try all letter-to-key assignments, infeasible for large input |
| Optimal | O(n + 26 log 26) | O(26) | Count letter frequency, sort descending, assign greedily to key positions |

## Algorithm Walkthrough

1. Count the frequency of each letter in `word` using a hash map. This helps identify which letters contribute most to the total push count.
2. Sort the letters by their frequency in descending order. The most frequent letters should be assigned to positions that require fewer key presses.
3. Calculate key positions in order: positions on keys are numbered such that the first letter on a key costs 1 push, the second costs 2, etc. For 8 keys, the positions repeat every 8 letters.
4. For each letter in sorted order, compute the push cost as `(position_index // 8) + 1` multiplied by its frequency. Add this to the running total.
5. Return the total cost.

**Why it works:** By assigning the highest-frequency letters to the lowest-cost positions, we minimize the contribution of frequent letters to the total push count. This greedy approach is guaranteed to be optimal because each decision reduces the maximum cost possible at that point.

## Python Solution

```python
from collections import Counter

class Solution:
    def minimumPushes(self, word: str) -> int:
        freq = Counter(word)
        sorted_counts = sorted(freq.values(), reverse=True)
        total_pushes = 0
        
        for idx, count in enumerate(sorted_counts):
            key_position = (idx // 8) + 1
            total_pushes += count * key_position
        
        return total_pushes
```

**Explanation:** First, we count how many times each letter appears. Sorting the counts ensures the most frequent letters are processed first. Using integer division `(idx // 8)` computes which position on a key the letter is assigned to, adding 1 to get the 1-based push count. Multiplying by the letter's frequency gives the total pushes contributed by that letter. Summing over all letters produces the minimum total.

## Go Solution

```go
package main

import (
	"fmt"
	"sort"
)

func minimumPushes(word string) int {
	freq := make(map[rune]int)
	for _, ch := range word {
		freq[ch]++
	}

	counts := []int{}
	for _, v := range freq {
		counts = append(counts, v)
	}

	sort.Sort(sort.Reverse(sort.IntSlice(counts)))

	totalPushes := 0
	for idx, count := range counts {
		keyPosition := (idx / 8) + 1
		totalPushes += count * keyPosition
	}

	return totalPushes
}
```

**Go-specific notes:** We use a `map[rune]int` for counting frequencies, which is idiomatic in Go. Sorting requires converting the map values to a slice. Integer division works the same as in Python, and the slice indexing ensures correct assignment of positions.

## Edge Cases

One important edge case is when the word contains only a single letter repeated many times, for example `"aaaaa"`. The algorithm correctly assigns this letter to the first position on the first key, giving the total pushes as the frequency itself.

Another edge case is when the number of distinct letters exceeds 8. Since each key can hold multiple letters, the code correctly assigns letters in groups of 8 per "row" of key positions. This ensures that the push count increments appropriately when letters overflow to the second position on keys.

A third edge case is a word containing all 26 letters exactly once. The algorithm sorts them and assigns the first 8 letters to position 1, the next 8 to position 2, and so on. This correctly calculates the minimum pushes even when the letters exceed a single position per key, demonstrating that the greedy assignment scales properly.
