# LeetCode 3012 - Minimize Length of Array Using Operations

## Problem Understanding

The problem gives us an array of positive integers and allows us to repeatedly perform a special operation on any two distinct elements.

For two selected values `a` and `b`, we:

1. Compute `a % b`
2. Append the result to the array
3. Remove both original elements

This means that every operation reduces the total array length by exactly one, because two elements disappear and one new element is added.

Our goal is to minimize the final length of the array after applying any number of operations.

The important detail is that modulo operations can generate smaller numbers, especially zero. Since all numbers are positive initially, modulo behavior becomes the key mathematical observation.

The constraints are large:

- Up to `10^5` elements
- Values up to `10^9`

This immediately rules out brute-force simulation of all possible operations. The number of operation sequences grows exponentially.

The problem guarantees that all numbers are positive initially. However, modulo operations may generate zero, and zero behaves differently because:

- `x % x = 0`
- `0 % x = 0`

This makes zero extremely important in reducing the array.

The core challenge is understanding what values can ultimately remain after repeated modulo operations.

Several edge cases are important:

- Arrays where all numbers are identical
- Arrays where the minimum value appears multiple times
- Arrays where some element is not divisible by the minimum
- Single-element arrays
- Arrays that can generate `1`, since `1` can reduce everything else aggressively

The key mathematical insight is that modulo operations preserve the greatest common divisor structure of the array.

## Approaches

### Brute Force Approach

A brute-force solution would attempt every possible sequence of operations.

At each step, we could:

1. Pick every pair `(i, j)`
2. Compute `nums[i] % nums[j]`
3. Create the next array state
4. Recursively continue

Eventually we would track the minimum achievable array length.

This approach is correct because it explores every valid operation sequence. However, it is completely impractical.

For an array of size `n`, there are approximately `O(n^2)` choices at every step, and the recursion depth can also reach `O(n)`. The total number of states becomes enormous.

Even memoization would not save this approach because the number of possible arrays generated by modulo operations is extremely large.

### Key Insight for the Optimal Solution

The crucial observation is that modulo operations can only produce values smaller than the divisor.

Let `m` be the minimum element in the array.

There are two fundamentally different situations:

1. Some element is not divisible by `m`
2. Every element is divisible by `m`

If some number `x` is not divisible by `m`, then:

```
x % m != 0
```

and the result becomes a value strictly smaller than `m`.

But `m` was supposed to be the minimum element. Therefore, we can create an even smaller number.

By repeatedly applying modulo operations, we can eventually generate `1` or some very small gcd-related value, which allows the array to collapse to length `1`.

On the other hand, if every number is divisible by `m`, then no operation can ever create a positive number smaller than `m`.

The only way to create smaller values is:

```
m % m = 0
```

This means the minimum positive value remains `m`.

Suppose `m` appears `k` times.

Each operation involving two `m` values produces one zero:

```
m % m = 0
```

Two minimum values become one zero, reducing the count of minimum elements by two.

Eventually:

- If `k` is even, we end with `k / 2` zeros
- If `k` is odd, one `m` survives

The minimum achievable length becomes:

```
ceil(k / 2)
```

where `k` is the frequency of the minimum value.

## Algorithm Walkthrough

1. Find the minimum value in the array.

The minimum value determines the entire behavior of the process because modulo operations against the minimum either produce zero or produce something even smaller.
2. Count how many times the minimum value appears.

Let this count be `min_count`.
3. Check whether every element is divisible by the minimum value.

Iterate through the array and test:

```
nums[i] % min_value == 0
```
4. If any element is not divisible by the minimum value, return `1`.

This means we can generate a value smaller than the minimum, eventually allowing the array to collapse to a single element.
5. Otherwise, all elements are divisible by the minimum.

In this case, the only reducible structure comes from pairing minimum elements together:

```
min_value % min_value = 0
```
6. Return:

```
(min_count + 1) // 2
```

This computes the ceiling of `min_count / 2`.

### Why it works

If any element is not divisible by the minimum value, modulo operations can generate a strictly smaller positive number. Repeating this process eventually produces a value capable of reducing the entire array to one element.

If every number is divisible by the minimum, no smaller positive value can ever be created. The minimum value becomes invariant. The only reduction possible is converting pairs of minimum values into zeros. Therefore the final size depends entirely on how many minimum elements exist.

## Python Solution

```python
from typing import List

class Solution:
    def minimumArrayLength(self, nums: List[int]) -> int:
        min_value = min(nums)
        min_count = nums.count(min_value)

        for num in nums:
            if num % min_value != 0:
                return 1

        return (min_count + 1) // 2
```

The implementation begins by finding the smallest value in the array. This value controls the behavior of all modulo operations.

Next, the code counts how many times this minimum value appears. This count becomes important if every element is divisible by the minimum.

The loop checks whether any element violates divisibility by the minimum value. The moment we find such an element, we know a smaller positive value can be generated, meaning the entire array can eventually collapse to length `1`.

If the loop completes successfully, every element is divisible by the minimum. In that scenario, the only meaningful operation is pairing minimum values together to create zeros. The formula `(min_count + 1) // 2` computes the smallest achievable length.

## Go Solution

```go
func minimumArrayLength(nums []int) int {
    minValue := nums[0]

    for _, num := range nums {
        if num < minValue {
            minValue = num
        }
    }

    minCount := 0

    for _, num := range nums {
        if num == minValue {
            minCount++
        }

        if num%minValue != 0 {
            return 1
        }
    }

    return (minCount + 1) / 2
}
```

The Go implementation follows the same logic as the Python version.

Since Go does not provide built-in helpers like `min()` or `count()`, we manually compute both values using loops.

Integer division in Go naturally truncates toward zero, so `(minCount + 1) / 2` correctly computes the ceiling of half the count.

The constraints fit comfortably inside Go's standard `int` type because values are at most `10^9`.

## Edge Cases

### Single Element Array

If the array contains only one element, no operation can be performed because the operation requires two distinct indices.

For example:

```
[7]
```

The answer must remain `1`.

The implementation handles this naturally because the minimum count is `1`, and:

```
(1 + 1) // 2 = 1
```

### All Elements Equal

Consider:

```
[5,5,5,5]
```

Every operation between equal values produces zero:

```
5 % 5 = 0
```

The only thing we can do is repeatedly convert pairs into zeros. The implementation correctly computes the answer using the frequency formula.

### Presence of a Non-Divisible Element

Consider:

```
[6,10,15]
```

The minimum is `6`, but:

```
10 % 6 = 4
```

This creates a smaller positive number. Once smaller values appear, the process can continue reducing until only one element remains.

The divisibility check detects this immediately and returns `1`.

### Large Integer Values

The constraints allow numbers up to `10^9`.

For example:

```
[1000000000, 1000000000]
```

The algorithm only performs modulo operations and comparisons, both of which are constant-time arithmetic operations. No overflow occurs in either Python or Go.
