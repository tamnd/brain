# LeetCode 3008, Find Beautiful Indices in the Given Array II

## Problem Understanding

The problem gives us a string `s` and two pattern strings, `a` and `b`. We need to find every index `i` where the substring starting at `i` matches `a`, but with an additional condition.

An index `i` is considered beautiful if there exists another index `j` where the substring starting at `j` matches `b`, and the distance between `i` and `j` is at most `k`.

In other words:

- Find all occurrences of `a` inside `s`
- Find all occurrences of `b` inside `s`
- For each occurrence of `a`, determine whether at least one occurrence of `b` is within distance `k`

The output must contain all such indices in sorted order.

The constraints are very large:

- `s.length` can be up to `5 * 10^5`
- `a.length` and `b.length` can also be up to `5 * 10^5`

These limits immediately rule out expensive substring comparisons at every position. A naive solution that repeatedly slices strings or checks all pairs of matches would be far too slow.

Several edge cases are important:

- `a` and `b` may overlap
- `a` may equal `b`
- There may be many repeated matches
- One or both patterns may appear zero times
- The valid `b` occurrence may appear before or after the `a` occurrence
- Very small strings and very large strings both need to work efficiently

The key challenge is efficiently finding all pattern occurrences and efficiently checking nearby matches.

## Approaches

### Brute Force Approach

The most direct solution is:

1. Iterate through every possible index in `s`
2. Check whether `a` starts there
3. If it does, scan every possible index again looking for `b`
4. For each `b` occurrence, check whether `|i - j| <= k`

This works logically because it explicitly verifies the exact definition of a beautiful index.

However, it is much too slow.

Suppose:

- `n = len(s)`
- There are many occurrences of `a`
- There are many occurrences of `b`

The nested scanning can become quadratic.

Additionally, repeatedly comparing substrings character by character is expensive for large inputs.

### Optimal Approach

The main optimization comes from separating the problem into two parts:

1. Efficiently find all occurrences of `a` and `b`
2. Efficiently determine whether an occurrence of `b` exists within distance `k`

To find all occurrences efficiently, we use the Knuth-Morris-Pratt algorithm, commonly called KMP.

KMP allows us to find every occurrence of a pattern in linear time:

- `O(n + m)` where `m` is the pattern length

After we obtain:

- `positionsA`
- `positionsB`

both arrays are naturally sorted because KMP scans left to right.

Now the problem becomes:

For each index in `positionsA`, determine whether there exists an element in `positionsB` within the range:

```
[i - k, i + k]
```

Because `positionsB` is sorted, we can use binary search to efficiently check this condition.

For every `i`:

- Find the first `b` position that is at least `i - k`
- Check whether it is also at most `i + k`

This reduces the problem to efficient searching in sorted arrays.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Repeated substring scans and pair checking |
| Optimal | O(n + \|a\| + \|b\| + A log B) | O(A + B) | KMP for matching and binary search for range checking |

Here:

- `A` is the number of occurrences of `a`
- `B` is the number of occurrences of `b`

## Algorithm Walkthrough

### Step 1: Build the KMP Prefix Table

For a pattern string, compute the Longest Prefix Suffix array, commonly called the LPS array.

The LPS array tells us:

- after a mismatch,
- how much of the previous match can still be reused.

This avoids rechecking characters unnecessarily.

### Step 2: Use KMP to Find All Occurrences of `a`

Scan through `s` while maintaining:

- current index in `s`
- current matched length in `a`

Whenever a full match is found:

- record the starting index
- continue searching using the LPS information

Store all match positions in `positionsA`.

### Step 3: Use KMP to Find All Occurrences of `b`

Repeat the same process for pattern `b`.

Store all match positions in `positionsB`.

Both arrays will automatically be sorted because KMP processes the string from left to right.

### Step 4: Process Each Occurrence of `a`

For every index `i` in `positionsA`:

1. Compute the valid range:

```
[i - k, i + k]
```
2. Use binary search on `positionsB` to find the first value greater than or equal to `i - k`
3. If that value also satisfies:

```
value <= i + k
```

then `i` is beautiful
4. Add `i` to the answer

### Step 5: Return the Result

The indices are processed in sorted order, so the final answer is already sorted.

### Why it works

KMP guarantees that every occurrence of `a` and `b` is found exactly once in linear time.

Because all occurrences are stored in sorted order, binary search correctly finds the nearest candidate occurrence of `b`.

If the first `b` occurrence greater than or equal to `i - k` is still within `i + k`, then a valid nearby `b` exists. Otherwise, no valid occurrence exists in the allowed range.

Therefore, every returned index satisfies the definition of beautiful, and every beautiful index is included.

## Python Solution

```python
from typing import List
from bisect import bisect_left

class Solution:
    def beautifulIndices(self, s: str, a: str, b: str, k: int) -> List[int]:
        
        def build_lps(pattern: str) -> List[int]:
            lps = [0] * len(pattern)
            length = 0
            i = 1

            while i < len(pattern):
                if pattern[i] == pattern[length]:
                    length += 1
                    lps[i] = length
                    i += 1
                else:
                    if length != 0:
                        length = lps[length - 1]
                    else:
                        lps[i] = 0
                        i += 1

            return lps

        def kmp_search(text: str, pattern: str) -> List[int]:
            lps = build_lps(pattern)

            matches = []
            text_index = 0
            pattern_index = 0

            while text_index < len(text):
                if text[text_index] == pattern[pattern_index]:
                    text_index += 1
                    pattern_index += 1

                    if pattern_index == len(pattern):
                        matches.append(text_index - pattern_index)
                        pattern_index = lps[pattern_index - 1]
                else:
                    if pattern_index != 0:
                        pattern_index = lps[pattern_index - 1]
                    else:
                        text_index += 1

            return matches

        positions_a = kmp_search(s, a)
        positions_b = kmp_search(s, b)

        result = []

        for index_a in positions_a:
            left_bound = index_a - k
            right_bound = index_a + k

            pos = bisect_left(positions_b, left_bound)

            if pos < len(positions_b) and positions_b[pos] <= right_bound:
                result.append(index_a)

        return result
```

The implementation begins with the `build_lps` function, which constructs the prefix table used by KMP. This preprocessing allows the search phase to skip unnecessary comparisons after mismatches.

The `kmp_search` function performs the actual pattern matching. It scans the text while maintaining how many characters of the pattern currently match. Whenever the full pattern is matched, the starting index is stored.

We run KMP twice:

- once for `a`
- once for `b`

This produces two sorted arrays of occurrence indices.

The final loop processes each occurrence of `a`. Using `bisect_left`, we efficiently locate the first `b` occurrence that could satisfy the distance condition. If that occurrence falls within the valid range, the index is beautiful.

Because binary search is logarithmic, the solution remains efficient even for very large inputs.

## Go Solution

```go
package main

import "sort"

func beautifulIndices(s string, a string, b string, k int) []int {

	buildLPS := func(pattern string) []int {
		lps := make([]int, len(pattern))

		length := 0
		i := 1

		for i < len(pattern) {
			if pattern[i] == pattern[length] {
				length++
				lps[i] = length
				i++
			} else {
				if length != 0 {
					length = lps[length-1]
				} else {
					lps[i] = 0
					i++
				}
			}
		}

		return lps
	}

	kmpSearch := func(text string, pattern string) []int {
		lps := buildLPS(pattern)

		var matches []int

		textIndex := 0
		patternIndex := 0

		for textIndex < len(text) {
			if text[textIndex] == pattern[patternIndex] {
				textIndex++
				patternIndex++

				if patternIndex == len(pattern) {
					matches = append(matches, textIndex-patternIndex)
					patternIndex = lps[patternIndex-1]
				}
			} else {
				if patternIndex != 0 {
					patternIndex = lps[patternIndex-1]
				} else {
					textIndex++
				}
			}
		}

		return matches
	}

	positionsA := kmpSearch(s, a)
	positionsB := kmpSearch(s, b)

	var result []int

	for _, indexA := range positionsA {
		leftBound := indexA - k
		rightBound := indexA + k

		pos := sort.SearchInts(positionsB, leftBound)

		if pos < len(positionsB) && positionsB[pos] <= rightBound {
			result = append(result, indexA)
		}
	}

	return result
}
```

The Go implementation closely follows the Python version. The main difference is the use of `sort.SearchInts` for binary search instead of Python's `bisect_left`.

Go slices naturally handle dynamic append operations efficiently, so they work well for storing match positions.

Integer overflow is not a concern because indices remain well within Go's standard integer range.

If no beautiful indices exist, the function returns an empty slice.

## Edge Cases

### Overlapping Pattern Matches

Patterns can overlap heavily, such as:

```
s = "aaaaa"
a = "aa"
```

The valid matches are:

```
[0, 1, 2, 3]
```

A naive implementation that skips ahead after finding a match would miss overlapping occurrences. KMP handles this correctly by resetting the pattern index using the LPS table instead of restarting from zero.

### No Occurrences of One Pattern

If either `a` or `b` never appears in `s`, the answer must be empty.

For example:

```
s = "abcdef"
a = "ab"
b = "gh"
```

The binary search phase naturally handles this because `positionsB` becomes empty, and no valid range check succeeds.

### Large Inputs With Many Matches

Consider strings like:

```
s = "aaaaaaaaaa..."
```

with repeated small patterns.

This can create hundreds of thousands of overlapping matches. A quadratic approach would time out badly.

The implementation remains efficient because:

- KMP processes the string linearly
- Binary search is logarithmic
- No substring copying occurs

This keeps the solution scalable even at maximum constraints.
