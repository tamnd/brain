# LeetCode 3039 - Apply Operations to Make String Empty

## Problem Understanding

We are given a lowercase English string `s`. An operation consists of scanning through the alphabet from `'a'` to `'z'` and removing the first occurrence of each character that currently exists in the string.

This process is repeated until the string becomes empty.

The task is not to return the final empty string. Instead, we must return the string that exists immediately before the final operation is performed.

Consider the example:

```
s = "aabcbbca"

Round 1 removes:
a, b, c

Remaining: "abbca"

Round 2 removes:
a, b, c

Remaining: "ba"

Round 3 removes:
b, a

Remaining: ""

Answer = "ba"
```

The important observation is that every operation removes at most one occurrence of each distinct character. Therefore, if a character appears `k` times in the original string, it survives for exactly `k` rounds before disappearing.

The constraints are significant:

```
1 <= s.length <= 5 * 10^5
```

A string length of five hundred thousand means that any simulation involving repeated string rebuilding would be far too slow. We need a solution that runs in linear time.

Several edge cases are worth considering immediately:

- A string containing only one character, such as `"a"`.
- A string where every character appears exactly once, such as `"abcd"`.
- A string where one character appears many more times than others.
- Characters with equal maximum frequencies.
- Very large strings near the maximum constraint.

The problem guarantees that the string contains only lowercase English letters, so there are at most 26 distinct character types.

## Approaches

### Brute Force

A direct simulation would repeatedly perform the operation exactly as described.

For each round:

1. Find the first occurrence of every character.
2. Remove those characters.
3. Build the new string.
4. Continue until the string becomes empty.

This approach is correct because it follows the problem statement literally.

However, it is extremely inefficient. If the string length is `n`, we may perform many rounds, and each round requires scanning and rebuilding large portions of the string. In the worst case, this can become quadratic.

With `n = 500000`, such a solution is infeasible.

### Key Insight

Instead of simulating removals, think about how many rounds each character survives.

Suppose character `'x'` appears `freq[x]` times.

Every operation removes exactly one occurrence of `'x'`, so after:

```
freq[x] rounds
```

all occurrences of `'x'` are gone.

Therefore, the last operation occurs during round:

```
maxFreq = maximum character frequency
```

Any character whose frequency is less than `maxFreq` disappears earlier and cannot appear in the string immediately before the last operation.

Only characters whose frequency equals `maxFreq` survive until the final round.

Now consider which occurrence survives until that final round.

For a character with frequency `maxFreq`, one occurrence is removed each round. Immediately before the final round, exactly one occurrence remains, namely its last occurrence in the original string.

Therefore:

1. Find the maximum frequency.
2. Identify all characters having that frequency.
3. Take their last occurrence positions in the original string.
4. Sort those positions by their order in the string.
5. Build the answer using the characters at those positions.

This completely avoids simulation.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Simulates every removal round |
| Optimal | O(n) | O(1) | Count frequencies and keep last positions |

## Algorithm Walkthrough

1. Traverse the string once and count the frequency of every character.

Since there are only 26 lowercase letters, an array of size 26 is sufficient.
2. During the same traversal, record the last position where each character appears.

Whenever a character is encountered again, overwrite its stored position.
3. Compute the maximum frequency among all characters.

This tells us how many rounds the process lasts.
4. Iterate through all 26 characters and select those whose frequency equals the maximum frequency.

These are exactly the characters that survive until the final round.
5. For each selected character, collect its last occurrence index.

The surviving character immediately before the final operation is the last remaining occurrence of that character.
6. Sort the collected indices.

The answer string must preserve the original left-to-right order.
7. Build the result by reading characters from the original string at those sorted positions.
8. Return the constructed string.

### Why it works

A character appearing `f` times loses exactly one occurrence per round, so it survives for exactly `f` rounds. The final round is determined by the largest frequency in the string. Any character with smaller frequency disappears earlier. For characters whose frequency equals the maximum frequency, exactly one occurrence remains immediately before the final round, namely the last occurrence in the original string. Collecting those last occurrences in their original order reconstructs the string present immediately before the final operation.

## Python Solution

```python
class Solution:
    def lastNonEmptyString(self, s: str) -> str:
        freq = [0] * 26
        last_pos = [-1] * 26

        for i, ch in enumerate(s):
            idx = ord(ch) - ord('a')
            freq[idx] += 1
            last_pos[idx] = i

        max_freq = max(freq)

        positions = []
        for idx in range(26):
            if freq[idx] == max_freq:
                positions.append(last_pos[idx])

        positions.sort()

        return "".join(s[pos] for pos in positions)
```

### Implementation Explanation

The `freq` array stores how many times each character appears. The `last_pos` array stores the most recent index of each character while scanning the string.

After the scan, `max_freq` identifies the frequency of the characters that survive until the last round.

We then examine all 26 letters and collect the last occurrence position of every character whose frequency equals `max_freq`.

These positions correspond exactly to the characters that remain immediately before the final operation. Sorting them restores their original ordering in the string.

Finally, we build the answer by reading the characters located at those positions.

## Go Solution

```go
func lastNonEmptyString(s string) string {
	freq := make([]int, 26)
	lastPos := make([]int, 26)

	for i := 0; i < 26; i++ {
		lastPos[i] = -1
	}

	for i := 0; i < len(s); i++ {
		idx := int(s[i] - 'a')
		freq[idx]++
		lastPos[idx] = i
	}

	maxFreq := 0
	for _, f := range freq {
		if f > maxFreq {
			maxFreq = f
		}
	}

	positions := make([]int, 0)
	for i := 0; i < 26; i++ {
		if freq[i] == maxFreq {
			positions = append(positions, lastPos[i])
		}
	}

	sort.Ints(positions)

	result := make([]byte, 0, len(positions))
	for _, pos := range positions {
		result = append(result, s[pos])
	}

	return string(result)
}
```

### Go Specific Notes

The Go implementation uses fixed-size slices for frequencies and last positions because there are only 26 lowercase letters.

The positions slice is sorted using `sort.Ints`, after which the answer is built using a byte slice. No integer overflow concerns exist because the maximum string length is only `5 * 10^5`.


when submitting.

## Edge Cases

### Single Character String

A string such as `"a"` can be easy to mishandle if the solution assumes multiple rounds exist. Here, the first operation is also the last operation, so the answer must be the original string itself. The algorithm handles this naturally because the maximum frequency is `1`, and the only last occurrence is returned.

### Only One Distinct Character

For a string like `"aaaaaa"`, every round removes exactly one `'a'`. Immediately before the final round, only one `'a'` remains. Since the last occurrence position is selected, the algorithm correctly returns `"a"`.

### Multiple Characters Sharing the Maximum Frequency

Consider `"aabb"`. Both `'a'` and `'b'` appear twice, so both survive until the final round. The answer consists of their last occurrences. After collecting positions and sorting them, the algorithm returns `"ab"` in the correct left-to-right order.

### Highly Unbalanced Frequencies

For a string like `"zzzzxy"`, the characters `'x'` and `'y'` disappear after the first round, while `'z'` survives several more rounds. Only `'z'` has the maximum frequency, so only its last occurrence is included in the answer. The algorithm correctly returns `"z"`.

### Maximum Input Size

With strings approaching length `500000`, repeated simulation would be too slow. The implemented solution scans the string only once and performs constant-size extra work, ensuring it remains efficient even at the largest allowed input size.
