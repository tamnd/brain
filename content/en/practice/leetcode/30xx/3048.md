## Solution

## Problem Understanding

This problem asks us to determine the earliest second at which **all indices** of an array `nums` can be marked under specific operations. You are given two arrays: `nums`, representing counts at each index, and `changeIndices`, which indicates which index you can attempt to mark at each second. Initially, no index is marked, and each index can only be marked if its `nums[i]` value reaches zero. On each second `s`, you can either decrement **any** element of `nums` by 1, attempt to mark `changeIndices[s]` if its value is zero, or do nothing.

The goal is to find the minimum second by which **all indices** can be marked, or `-1` if it is impossible.

Important points in understanding the problem include:

- The arrays are **1-indexed**, so any access to `changeIndices[s]` refers directly to the corresponding index in `nums`.
- `nums[i]` can be as large as $10^9$, which precludes simulating each decrement naively.
- `changeIndices` length `m` is at most 2000, meaning the number of seconds is manageable for iterative approaches.
- Some indices might never appear in `changeIndices`, making it impossible to mark them.

Edge cases to watch for include indices already zero, indices never appearing in `changeIndices`, and large values of `nums[i]` requiring many decrements.

## Approaches

A brute-force approach would simulate each second, trying all decrement options optimally to minimize the time to reach zero for each index and then marking when possible. This is correct logically but inefficient because each `nums[i]` can be very large (up to $10^9$), making per-second simulation infeasible.

The optimal approach relies on **binary search over time**. The key insight is that for a given second `t`, we can determine if it is possible to mark all indices **by greedily using earlier seconds to decrement** the necessary indices so that at the marking time, each index in `changeIndices` can be marked. If we can check feasibility in $O(n + m)$ time for a candidate `t`, we can use binary search over the interval `[1, m]` to find the minimal `t`.

The steps for feasibility checking are:

1. Count how many times each index appears in the first `t` seconds of `changeIndices`.
2. For each index, determine if its total decrements available in those `t` seconds are at least its `nums[i]`.
3. If every index can be reduced to zero before its last marking attempt, `t` is feasible.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m * sum(nums)) | O(n) | Simulates each second decrement, too slow for large nums |
| Optimal | O(n log m) | O(n) | Uses binary search over seconds with greedy feasibility check |

## Algorithm Walkthrough

1. Initialize `left` to 1 and `right` to `m` (number of seconds). `ans` is initially -1.
2. Perform binary search: for each middle value `mid = (left + right) // 2`, check if it is possible to mark all indices within the first `mid` seconds.
3. Feasibility check:

1. Count the occurrences of each index in `changeIndices[1..mid]` using a dictionary or array.
2. Sort `nums` in descending order to prioritize indices requiring more decrements first.
3. Greedily use available decrement seconds before the last marking attempt for each index. For each index `i`, if `nums[i]` is greater than the number of seconds where it can be decremented, the configuration is infeasible.
4. If `mid` is feasible, update `ans = mid` and move `right = mid - 1`. Otherwise, move `left = mid + 1`.
5. Continue until `left > right`. Return `ans`.

**Why it works:** The algorithm works because the binary search efficiently narrows down the minimal second where marking all indices is feasible. The greedy decrement allocation ensures that each index reaches zero just before it is marked, which is optimal.

## Python Solution

```python
from typing import List

class Solution:
    def earliestSecondToMarkIndices(self, nums: List[int], changeIndices: List[int]) -> int:
        n = len(nums)
        m = len(changeIndices)

        def can_mark_by(second: int) -> bool:
            counts = [0] * n
            for i in range(second):
                idx = changeIndices[i] - 1
                counts[idx] += 1

            needed = []
            for i in range(n):
                if counts[i] == 0 and nums[i] > 0:
                    return False
                needed.append(nums[i])

            available = 0
            for i in range(n):
                if needed[i] > 0:
                    available += counts[i]
                    if available < needed[i]:
                        return False
            return True

        left, right = 1, m
        ans = -1
        while left <= right:
            mid = (left + right) // 2
            if can_mark_by(mid):
                ans = mid
                right = mid - 1
            else:
                left = mid + 1
        return ans
```

In this Python implementation, `can_mark_by` checks if all indices can be marked in the first `mid` seconds by counting occurrences in `changeIndices` and ensuring sufficient decrement opportunities. The binary search finds the minimal feasible second efficiently.

## Go Solution

```go
func earliestSecondToMarkIndices(nums []int, changeIndices []int) int {
    n, m := len(nums), len(changeIndices)

    canMarkBy := func(second int) bool {
        counts := make([]int, n)
        for i := 0; i < second; i++ {
            idx := changeIndices[i] - 1
            counts[idx]++
        }

        for i := 0; i < n; i++ {
            if counts[i] == 0 && nums[i] > 0 {
                return false
            }
        }

        available := 0
        for i := 0; i < n; i++ {
            if nums[i] > 0 {
                available += counts[i]
                if available < nums[i] {
                    return false
                }
            }
        }
        return true
    }

    left, right := 1, m
    ans := -1
    for left <= right {
        mid := (left + right) / 2
        if canMarkBy(mid) {
            ans = mid
            right = mid - 1
        } else {
            left = mid + 1
        }
    }
    return ans
}
```

In Go, the approach is analogous, with `make([]int, n)` used for counting occurrences and slices for arrays. Overflow is not an issue because integers are small enough.
