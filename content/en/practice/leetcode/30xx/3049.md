## Solution

## Problem Understanding

The problem asks us to determine the earliest second at which all indices in a given array `nums` can be marked under a set of allowed operations over a sequence of seconds determined by `changeIndices`. Each second, you may perform one of four operations: decrement any `nums[i]` by 1, set `nums[changeIndices[s]]` to any non-negative value, mark any index `i` whose value is zero, or do nothing. The indices are 1-indexed, meaning the first element corresponds to `nums[1]`.

The input `nums` represents the initial values at each index, which must eventually be reduced to zero and then marked. The array `changeIndices` specifies indices that can be directly modified (set to zero) in each second. The output is the minimum second `s` in which it becomes possible to mark all indices, or `-1` if it is impossible within the given number of seconds.

Constraints indicate `nums` and `changeIndices` can each have up to 5000 elements, and `nums[i]` can be as large as 1e9, so a brute-force simulation of every second decrement for each index is infeasible. Edge cases include arrays with all zeros, arrays requiring more seconds than available, and indices repeatedly appearing in `changeIndices`.

## Approaches

A brute-force approach would simulate each second by trying all operations and recursively exploring every possible path of setting, decrementing, or marking indices. While this would guarantee correctness, it is computationally infeasible because the number of potential sequences grows exponentially with `m` and `n`. This approach would quickly exceed time limits.

The key observation for an optimal solution is that the **order of marking zeros can be planned greedily**, and each second can be treated as a resource for either decrementing or setting indices to zero. We can use **binary search** over the possible earliest second to test feasibility. For each candidate second, we compute how many decrements each unmarked index needs and check if the available operations (either direct set via `changeIndices` or free decrements) can achieve marking all indices. This allows us to reduce a potentially exponential search to O(n log m).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(4^m) | O(n + m) | Simulate all operations recursively for each second; infeasible for n, m up to 5000 |
| Optimal | O(n log m) | O(n + m) | Use binary search on the earliest second and greedy allocation of operations to determine feasibility |

## Algorithm Walkthrough

1. Define a helper function `canMarkBy(second)` that determines whether all indices in `nums` can be marked by the given `second`.
2. For the candidate `second`, collect the indices from `changeIndices[1..second]` where a set-to-zero operation can be performed.
3. For each index `i` in `nums`, compute the number of decrements still needed to bring `nums[i]` to zero if it is not included in the set operations from step 2.
4. Maintain a priority queue (or sorted list) of indices that can be decremented. At each second, greedily decrement the index that requires the most decrements first.
5. After simulating all seconds up to `second`, check if every index can reach zero and be marked.
6. Apply binary search on the range `[1, m]` using `canMarkBy(mid)` as a feasibility test to find the earliest second where all indices can be marked. If no feasible second exists, return `-1`.

**Why it works:** By binary searching the earliest feasible second and simulating a greedy allocation of decrements or set operations, we guarantee that if it is possible to mark all indices at or before a certain second, we will find the minimum. The greedy allocation ensures we prioritize the indices that need the most work, ensuring feasibility without over-consuming operations.

## Python Solution

```python
from typing import List
from collections import defaultdict, deque

class Solution:
    def earliestSecondToMarkIndices(self, nums: List[int], changeIndices: List[int]) -> int:
        n, m = len(nums), len(changeIndices)
        
        def canMarkBy(second: int) -> bool:
            lastSet = [-1] * n
            for i in range(second):
                idx = changeIndices[i] - 1
                lastSet[idx] = i
            # count needed decrements for each index
            needs = []
            for i in range(n):
                if nums[i] == 0:
                    continue
                if lastSet[i] == -1:
                    needs.append(nums[i])
            # total available seconds for decrements
            available = second - sum(1 for x in lastSet if x != -1)
            return sum(needs) <= available
        
        left, right = 1, m
        answer = -1
        while left <= right:
            mid = (left + right) // 2
            if canMarkBy(mid):
                answer = mid
                right = mid - 1
            else:
                left = mid + 1
        return answer
```

The solution first calculates the last second each index could be directly set to zero. Then, it computes the decrements needed for indices not covered by set operations. The `available` variable tracks the seconds left to decrement, ensuring feasibility. Binary search efficiently finds the minimal second.

## Go Solution

```go
func earliestSecondToMarkIndices(nums []int, changeIndices []int) int {
    n, m := len(nums), len(changeIndices)

    canMarkBy := func(second int) bool {
        lastSet := make([]int, n)
        for i := range lastSet {
            lastSet[i] = -1
        }
        for i := 0; i < second; i++ {
            idx := changeIndices[i] - 1
            lastSet[idx] = i
        }
        needs := 0
        for i := 0; i < n; i++ {
            if nums[i] == 0 {
                continue
            }
            if lastSet[i] == -1 {
                needs += nums[i]
            }
        }
        available := second
        for i := 0; i < n; i++ {
            if lastSet[i] != -1 {
                available--
            }
        }
        return needs <= available
    }

    left, right, answer := 1, m, -1
    for left <= right {
        mid := (left + right) / 2
        if canMarkBy(mid) {
            answer = mid
            right = mid - 1
        } else {
            left = mid + 1
        }
    }
    return answer
}
```

In Go, we initialize slices explicitly and use integer arithmetic for binary search. The logic mirrors the Python version, taking care of zero-indexing when processing `changeIndices`.
