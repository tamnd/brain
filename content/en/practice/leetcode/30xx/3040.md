**Difficulty:** 🟡 Medium  
**Topics:** Array, Dynamic Programming, Memoization

## Solution

## Problem Understanding

The problem presents an array `nums` of integers and defines an operation that can remove either the first two elements, the last two elements, or the first and last elements. Each operation has a **score**, which is the sum of the two removed elements. The task is to find the **maximum number of operations** that can be performed such that **all operations have the same score**.

In simpler terms, we are allowed to pair elements from the start, end, or both ends of the array to remove them, and we want to see how many operations we can perform if every removed pair adds up to the same sum.

The input constraints, `2 <= nums.length <= 2000` and `1 <= nums[i] <= 1000`, indicate that the array can be moderately large and that the integers are small and positive. This allows solutions with O(n²) complexity to be feasible.

Important edge cases include arrays where all elements are equal, arrays of minimal length (2 elements), and arrays where multiple sums are possible but only one maximizes the number of operations. The problem guarantees at least two elements, so we never have an empty array initially.

## Approaches

### Brute Force

A brute-force approach would attempt all possible combinations of the operations and track the score for each sequence. Specifically, we would recursively try removing the first two, last two, or first and last elements and count how many operations produce the same sum.

While this guarantees correctness, it is extremely slow because for each pair choice, we create new subarrays and recursively explore all possibilities. The number of recursive calls grows exponentially, making this infeasible for arrays of length up to 2000.

### Optimal Approach

The key insight is that the problem can be reduced to a **dynamic programming** problem with memoization. We can define a function `dp(i, j, score)` which returns the maximum number of operations in the subarray `nums[i:j+1]` such that every operation has the same `score`.

The recursive cases are straightforward: if the first two elements sum to `score`, we recurse on `dp(i+2, j, score)`. Similarly, we check the last two elements and the first and last elements. We memoize results for `(i, j)` and each candidate `score` to avoid recomputation.

To avoid testing every score, we first generate all possible pair sums in the array because the score must come from one of these sums. Then we compute the maximum number of operations for each possible score.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(3^n) | O(n) | Recursively try all operation sequences, very slow |
| Optimal | O(n² * S) | O(n² * S) | Dynamic programming with memoization, S is the number of distinct possible sums |

## Algorithm Walkthrough

1. Generate all possible sums of two elements in the array. These are the candidate scores because an operation’s score must match the sum of some pair of elements.
2. Define a recursive function `dp(l, r, score)` that returns the maximum number of operations in the subarray `nums[l:r+1]` where all operations have sum `score`.
3. Base case: if `l >= r`, return 0 because fewer than 2 elements remain.
4. Recursive cases:

- If `nums[l] + nums[l+1] == score`, recursively call `dp(l+2, r, score)` and add 1.
- If `nums[r] + nums[r-1] == score`, recursively call `dp(l, r-2, score)` and add 1.
- If `nums[l] + nums[r] == score`, recursively call `dp(l+1, r-1, score)` and add 1.
5. Memoize results for `(l, r)` to avoid recomputation.
6. Compute `dp(0, n-1, score)` for each candidate sum and return the maximum.

**Why it works:** The recursive function explores all valid sequences of operations for a given score and counts the maximum possible operations. Memoization ensures that subproblems are not recomputed, leading to an efficient solution.

## Python Solution

```python
from typing import List
from functools import lru_cache

class Solution:
    def maxOperations(self, nums: List[int]) -> int:
        n = len(nums)
        possible_scores = set()
        for i in range(n):
            for j in range(i + 1, n):
                possible_scores.add(nums[i] + nums[j])

        @lru_cache(None)
        def dp(l: int, r: int, score: int) -> int:
            if l >= r:
                return 0
            res = 0
            if nums[l] + nums[l + 1] == score:
                res = max(res, 1 + dp(l + 2, r, score))
            if nums[r] + nums[r - 1] == score:
                res = max(res, 1 + dp(l, r - 2, score))
            if nums[l] + nums[r] == score:
                res = max(res, 1 + dp(l + 1, r - 1, score))
            return res

        max_ops = 0
        for score in possible_scores:
            max_ops = max(max_ops, dp(0, n - 1, score))
        return max_ops
```

**Explanation:** We first collect all candidate sums and then define a memoized recursive function `dp(l, r, score)` that computes the maximum number of operations for a specific score. For each score, we compute `dp(0, n-1, score)` and track the maximum across all candidate sums. This ensures correctness and efficiency.

## Go Solution

```go
func maxOperations(nums []int) int {
    n := len(nums)
    possibleScores := make(map[int]struct{})
    for i := 0; i < n; i++ {
        for j := i + 1; j < n; j++ {
            possibleScores[nums[i]+nums[j]] = struct{}{}
        }
    }

    memo := make(map[[3]int]int)
    var dp func(l, r, score int) int
    dp = func(l, r, score int) int {
        if l >= r {
            return 0
        }
        key := [3]int{l, r, score}
        if val, ok := memo[key]; ok {
            return val
        }
        res := 0
        if nums[l]+nums[l+1] == score {
            res = max(res, 1+dp(l+2, r, score))
        }
        if nums[r]+nums[r-1] == score {
            res = max(res, 1+dp(l, r-2, score))
        }
        if nums[l]+nums[r] == score {
            res = max(res, 1+dp(l+1, r-1, score))
        }
        memo[key] = res
        return res
    }

    maxOps := 0
    for score := range possibleScores {
        maxOps = max(maxOps, dp(0, n-1, score))
    }
    return maxOps
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

**Go-specific notes:** Go requires explicit handling of maps for memoization and integer arrays as keys. Nil slices are not a concern because we always use indices. The logic mirrors the Python solution.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n² * S) | For each candidate sum S, we explore subarrays of length n with memoization. |
| Space | O(n² * S) | Memoization cache stores results for each (l,r,score) combination. |

The O(n²) factor comes from the potential number of subarray states, and S is the number of distinct pair sums, which is at most O(n²) but usually much smaller in practice.
