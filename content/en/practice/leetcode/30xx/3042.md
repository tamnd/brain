# LeetCode 3042 - Count Prefix and Suffix Pairs I

## Problem Understanding

The problem gives us an array of strings called `words`. We need to count how many index pairs `(i, j)` satisfy two conditions:

1. `i < j`
2. `words[i]` is both a prefix and a suffix of `words[j]`

A string is a prefix of another string if it appears at the beginning. A string is a suffix if it appears at the end.

For example:

- `"ab"` is a prefix of `"abcd"`
- `"cd"` is a suffix of `"abcd"`
- `"aba"` is both a prefix and suffix of `"ababa"`

The task is not asking us to return the pairs themselves. We only need the total count.

The input constraints are very small:

- At most 50 words
- Each word length is at most 10

These constraints are important because they tell us that even relatively inefficient solutions will still run comfortably within time limits. A quadratic solution over all pairs is completely acceptable here.

A few important observations:

- The same string can be both a prefix and suffix of itself.
- The prefix and suffix must match the entire `words[i]`.
- We only consider pairs where `i < j`, so ordering matters.
- Words can have different lengths.
- If `words[i]` is longer than `words[j]`, it can never be both a prefix and suffix.

Some edge cases that can cause mistakes:

- Very short strings like `"a"`
- Identical strings
- Strings longer than the candidate target
- Cases where a string is only a prefix or only a suffix, but not both
- Single-character repeated strings like `"aa"` and `"aaaa"`

Because the constraints are tiny, correctness and clarity are more important than advanced optimization.

## Approaches

### Brute Force Approach

The most direct solution is to check every possible pair `(i, j)` where `i < j`.

For each pair:

1. Take `words[i]` as the candidate prefix/suffix string.
2. Take `words[j]` as the target string.
3. Check whether:

- `words[j]` starts with `words[i]`
- `words[j]` ends with `words[i]`
4. If both are true, increment the answer.

This approach is straightforward and guarantees correctness because every valid pair is explicitly checked.

The downside is that we examine all pairs. If the constraints were very large, this quadratic comparison would become too slow. However, for this problem, the limits are small enough that this solution is already optimal in practice.

### Key Insight

The crucial observation is that the problem only asks whether one string is simultaneously:

- a prefix of another string
- a suffix of another string

Modern string libraries already provide efficient operations for these checks:

- `startswith()` and `endswith()` in Python
- slicing comparisons in Go

Since the input size is tiny, there is no need for advanced techniques like Trie structures, rolling hash, or KMP preprocessing. A clean pairwise comparison solution is sufficient and easy to reason about.

## Algorithm Walkthrough

### Step 1: Initialize the Answer Counter

Create a variable `count` initialized to `0`.

This variable stores the number of valid `(i, j)` pairs.

### Step 2: Iterate Over All Pairs

Use two nested loops:

- The outer loop chooses index `i`
- The inner loop chooses index `j` such that `j > i`

This guarantees we only consider valid ordered pairs.

### Step 3: Extract the Two Strings

For each pair:

- `small = words[i]`
- `large = words[j]`

We want to determine whether `small` is both a prefix and suffix of `large`.

### Step 4: Check Prefix Condition

Verify whether `large` starts with `small`.

In Python, this is:

```
large.startswith(small)
```

### Step 5: Check Suffix Condition

Verify whether `large` ends with `small`.

In Python, this is:

```
large.endswith(small)
```

### Step 6: Count Valid Pairs

If both conditions are true, increment `count`.

### Step 7: Return the Result

After all pairs have been processed, return `count`.

### Why it works

The algorithm explicitly evaluates every valid pair `(i, j)` with `i < j`. For each pair, it directly verifies the exact definition required by the problem:

- `words[i]` must match the beginning of `words[j]`
- `words[i]` must match the end of `words[j]`

Because every pair is checked exactly once and only valid pairs are counted, the final answer is guaranteed to be correct.

## Python Solution

```python
from typing import List

class Solution:
    def countPrefixSuffixPairs(self, words: List[str]) -> int:
        count = 0
        n = len(words)

        for i in range(n):
            for j in range(i + 1, n):
                prefix_suffix = words[i]
                target = words[j]

                if target.startswith(prefix_suffix) and target.endswith(prefix_suffix):
                    count += 1

        return count
```

The implementation closely follows the algorithm described earlier.

The variable `count` stores the number of valid pairs discovered so far.

The nested loops iterate through all possible ordered pairs where `i < j`. This guarantees we never double count and never examine invalid index orderings.

For every pair, the code uses Python's built-in string methods:

- `startswith()`
- `endswith()`

These methods make the implementation concise and easy to read. Since the maximum string length is only 10, these operations are effectively constant time for this problem.

If both conditions evaluate to `True`, the pair satisfies the problem requirements, so the answer counter is incremented.

Finally, the total count is returned.

## Go Solution

```go
func countPrefixSuffixPairs(words []string) int {
	count := 0
	n := len(words)

	for i := 0; i < n; i++ {
		for j := i + 1; j < n; j++ {
			prefixSuffix := words[i]
			target := words[j]

			if len(prefixSuffix) > len(target) {
				continue
			}

			prefixMatch := target[:len(prefixSuffix)] == prefixSuffix
			suffixMatch := target[len(target)-len(prefixSuffix):] == prefixSuffix

			if prefixMatch && suffixMatch {
				count++
			}
		}
	}

	return count
}
```

The Go version follows the same logic as the Python implementation.

Instead of built-in `startswith()` and `endswith()` methods, Go uses string slicing:

- `target[:len(prefixSuffix)]` checks the prefix
- `target[len(target)-len(prefixSuffix):]` checks the suffix

An additional length check is included before slicing to avoid invalid slice boundaries when the candidate string is longer than the target string.

Go strings are immutable UTF-8 byte sequences, but since the problem guarantees lowercase English letters only, byte indexing is perfectly safe.

## Edge Cases

### Single Element Array

If the array contains only one word, there are no valid pairs because the condition requires `i < j`.

A naive implementation might accidentally compare a word with itself. The nested loop structure prevents this because the inner loop always starts at `i + 1`.

### Candidate String Longer Than Target

A string cannot be a prefix or suffix of another string if it is longer than the target.

For example:

```
words[i] = "abcd"
words[j] = "ab"
```

The Python implementation handles this naturally because `startswith()` and `endswith()` immediately return `False`. The Go implementation explicitly checks lengths before slicing to avoid runtime panics.

### Strings That Match Only One Side

Some strings may match only as a prefix or only as a suffix.

Example:

```
words[i] = "ab"
words[j] = "abcd"
```

Here, `"ab"` is a prefix but not a suffix.

A buggy implementation might incorrectly count partial matches. The solution avoids this by requiring both conditions simultaneously with a logical `and`.

### Identical Strings

Two equal strings should count as a valid pair because a string is trivially both its own prefix and suffix.

Example:

```
["abc", "abc"]
```

The implementation correctly handles this because both prefix and suffix checks evaluate to `True`.
