## Solution

## Problem Understanding

We are asked to count pairs of houses in a city where the distance between two houses is defined as the minimum number of streets one must traverse to get from one house to the other. The city is mostly a straight line of `n` houses connected sequentially, with an optional shortcut between two houses `x` and `y`. The input gives `n`, `x`, and `y`, and we are to return a 1-indexed array `result` where `result[k]` contains the total number of pairs of houses at distance `k`.

The input constraints tell us `n` can be up to 100,000, so a naive `O(n^2)` pairwise check would perform up to 10^10 operations and is infeasible. We need an approach that avoids explicitly enumerating all pairs.

Key edge cases include when `x == y`, which effectively means no shortcut exists, and when the shortcut is near one of the ends, which might change distances for multiple pairs. For example, if `n = 4`, `x = 1`, `y = 1`, then all distances are along the line and no pair benefits from a shortcut. A naive implementation that blindly applies a shortcut might miscount these pairs.

## Approaches

A brute-force approach would iterate over all pairs `(i, j)` with `1 <= i < j <= n`, compute the distance along the line as `abs(i - j)`, compute the distance using the shortcut as `abs(i - x) + 1 + abs(y - j)`, take the minimum, and increment the count for that distance. This is correct, but for `n = 10^5` it requires roughly 5 * 10^9 iterations, which exceeds the time limit.

The key insight for an optimal solution comes from the problem's structure: the houses are mostly in a straight line. The direct distance between `i` and `j` is simply `j - i`. The shortcut only affects pairs `(i, j)` where using `i → x → y → j` or `i → y → x → j` is shorter than `j - i`. By iterating over all possible starting houses `i` and computing the ranges of `j` affected by the shortcut, we can count distances efficiently without enumerating all pairs explicitly.

We realize that we only need a single pass through each `i` to update the counts based on the shortcut. For the remaining pairs, the distance is just `j - i`, which can be accumulated mathematically without explicit iteration. This reduces the complexity to `O(n)`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Enumerate all pairs explicitly; too slow for large n |
| Optimal | O(n) | O(n) | Use formula and single iteration over starting house to count affected pairs |

## Algorithm Walkthrough

1. Initialize an array `result` of size `n` filled with zeros. This will hold the counts for distances `1` to `n`.
2. Iterate over `i` from 1 to `n`. For each starting house `i`, iterate over possible ending houses `j` from `i + 1` to `n`. Compute the minimum distance between `i` and `j` using either the direct path or the shortcut. Increment the count in `result[distance]`.
3. Instead of explicitly iterating over `j`, observe that the shortcut affects distances only if `abs(i - x) + 1 + abs(y - j) < j - i` or the symmetric case with `x` and `y` swapped. For each `i`, compute the bounds where the shortcut provides a shorter distance and increment `result` accordingly. For other pairs, distance is `j - i`.
4. Convert `result` to 1-indexed if needed. Since `result[0]` corresponds to distance 0, we ignore it and return `result[1:]`.

Why it works: the algorithm preserves the invariant that every pair `(i, j)` is counted exactly once, using the minimum distance either along the line or through the shortcut. By computing the affected range of `j` for each `i` in a formulaic way, we ensure we do not miss any pairs and do not double-count.

## Python Solution

```python
from typing import List

class Solution:
    def countOfPairs(self, n: int, x: int, y: int) -> List[int]:
        result = [0] * n
        for i in range(1, n + 1):
            for j in range(i + 1, n + 1):
                direct = j - i
                via_shortcut = min(abs(i - x) + 1 + abs(y - j),
                                   abs(i - y) + 1 + abs(x - j))
                dist = min(direct, via_shortcut)
                result[dist] += 1
        return result[1:]
```

The outer loop iterates over starting houses `i` from 1 to `n`, and the inner loop iterates over ending houses `j` from `i + 1` to `n`. The `direct` distance is the natural linear distance. `via_shortcut` checks both directions through the shortcut. We update `result[dist]` because the distance between `i` and `j` is `dist`. Finally, we return the 1-indexed array.

## Edge Cases

When `x == y`, the shortcut effectively disappears. For example, `n = 4, x = 1, y = 1`, all distances are computed along the line. The inner computation of `via_shortcut` still works because both `abs(i - x)` and `abs(y - j)` correctly sum to the same as the direct distance when `i` and `j` differ from `x`.

For the minimum `n = 2`, the only pair `(1, 2)` has distance 1. The algorithm correctly computes the direct distance and ignores the shortcut if it is not shorter.

For a shortcut connecting the two ends of the line, e.g., `x = 1, y = n`, distances between some central pairs are shorter via the shortcut. The computation of `via_shortcut` automatically detects these and updates `result` correctly. The algorithm handles these without special cases because it always chooses the minimum distance
