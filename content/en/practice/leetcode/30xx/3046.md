## Solution

## Problem Understanding

The problem gives us an integer array `nums` whose length is always even. We need to divide the elements into two separate arrays, `nums1` and `nums2`, such that both arrays have the same size, specifically `nums.length / 2`.

The important requirement is that each resulting array must contain only distinct elements. In other words:

- No duplicate values are allowed inside `nums1`
- No duplicate values are allowed inside `nums2`

However, the same value is allowed to appear once in each array. For example, if the value `1` appears twice in the original array, we can place one copy in `nums1` and the other in `nums2`.

The task is to determine whether such a split is possible. We return `true` if we can construct the two arrays while satisfying all conditions, otherwise we return `false`.

The constraints are small:

- `nums.length <= 100`
- `nums[i] <= 100`

This tells us that efficiency is not a major concern, and even relatively simple approaches would work comfortably within limits. Still, we should aim for the cleanest and most direct solution.

The key observation comes from understanding what makes the split impossible. Suppose a number appears three times. Since each array can contain that number at most once, we only have two valid positions available total, one in `nums1` and one in `nums2`. A third occurrence has nowhere to go without creating a duplicate in one of the arrays.

Therefore:

- Any number appearing more than twice immediately makes the answer `false`
- If every number appears at most twice, the answer is always `true`

Some important edge cases include:

- Arrays where every element is identical, such as `[1,1,1,1]`
- Arrays with all unique elements
- Arrays where some numbers appear exactly twice
- The smallest possible input size, such as `[1,2]`

The problem guarantees the array length is even, so we never need to worry about uneven splits.

## Approaches

### Brute Force Approach

A brute force solution would attempt to generate all possible ways to split the array into two groups of equal size. For every possible partition, we would check whether both groups contain only distinct elements.

For an array of size `n`, this means choosing `n/2` elements for the first array, while the remaining elements go into the second array. The number of such combinations is extremely large:

$$\binom{n}{n/2}$$

For each partition, we would also need to verify uniqueness inside both arrays.

This approach is correct because it explicitly checks every valid split configuration. If any partition satisfies the conditions, we return `true`.

However, this becomes computationally expensive as `n` grows. Even though the constraints are small here, this approach is unnecessary because the problem has a much simpler property.

### Optimal Approach

The optimal solution relies on counting frequencies.

Each number can appear at most:

- Once in `nums1`
- Once in `nums2`

That means a value can appear at most twice overall.

Therefore, the entire problem reduces to checking whether any number occurs more than two times.

We can use a hash map, also called a frequency table, to count occurrences of each number. As soon as we find a count greater than `2`, we return `false`. If we finish processing the array without exceeding this limit, we return `true`.

This works because:

- Two copies can be distributed across the two arrays
- One copy is trivially valid
- Three or more copies cannot fit without creating duplicates

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Tries all possible splits and checks validity |
| Optimal | O(n) | O(n) | Counts frequencies and ensures no value appears more than twice |

## Algorithm Walkthrough

1. Create a hash map called `frequency` to store how many times each number appears.
2. Iterate through every number in `nums`.
3. For each number:

- Increase its count in the hash map.
- Immediately check whether the count exceeds `2`.
4. If any number appears more than twice:

- Return `False`
- This is because we only have two arrays, and each array can contain that value at most once.
5. If the loop completes without finding any count greater than `2`:

- Return `True`
- This guarantees every value can be distributed across the two arrays without duplicates.

### Why it works

The algorithm relies on a simple invariant:

- Each distinct value can occupy at most two valid positions overall, one in each array.

If a value appears more than twice, at least one array must contain duplicates, making the split impossible.

If every value appears at most twice, we can always distribute occurrences safely:

- One occurrence goes into `nums1`
- The second occurrence goes into `nums2`

Thus, the condition is both necessary and sufficient.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def isPossibleToSplit(self, nums: List[int]) -> bool:
        frequency = defaultdict(int)

        for num in nums:
            frequency[num] += 1

            if frequency[num] > 2:
                return False

        return True
```

The implementation uses `defaultdict(int)` so that missing keys automatically start at `0`.

We iterate through the array once and increment the count for each number. After incrementing, we immediately check whether the frequency exceeds `2`.

This early termination is useful because we do not need to process the remaining elements once the split becomes impossible.

If the loop finishes successfully, all values appeared at most twice, so we return `True`.

## Go Solution

```go
func isPossibleToSplit(nums []int) bool {
    frequency := make(map[int]int)

    for _, num := range nums {
        frequency[num]++

        if frequency[num] > 2 {
            return false
        }
    }

    return true
}
```

The Go version uses a `map[int]int` to store frequencies.

Go maps return the zero value for missing keys, so incrementing works naturally without additional checks.

There are no integer overflow concerns because the maximum frequency is bounded by the array size, which is at most `100`.

The logic matches the Python implementation exactly.

- One `1` can go into `nums1`

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We scan the array once |
| Space | O(n) | The hash map may store up to all distinct values |

The algorithm performs a single pass through the input array. Each hash map insertion and lookup takes constant average time.

The auxiliary space depends on the number of distinct elements. In the worst case, all elements are unique, so the map stores `n` entries.

## Edge Cases

One important edge case is when all elements are identical, such as `[1,1,1,1]`. A naive implementation might incorrectly assume duplicates can always be distributed evenly between the two arrays. However, each array may only contain one copy of a value. Once frequency exceeds `2`, the split becomes impossible. The implementation handles this by immediately returning `False` when a count exceeds `2`.

Another important case is when all elements are unique. For example, `[1,2,3,4]` is always valid because no duplicates exist anywhere. Some implementations may overcomplicate the partitioning process, but the frequency-based solution naturally handles this situation since every count remains `1`.

A third edge case is when values appear exactly twice, such as `[1,1,2,2,3,3]`. This is the maximum allowable duplication. The implementation correctly treats frequency `2` as valid because one occurrence can be placed in each array.

Finally, the smallest possible arrays deserve attention. Inputs like `[1,2]` and `[5,5]` verify that the implementation handles minimal sizes correctly. Since the algorithm only depends on frequency counts, it works consistently even for these tiny inputs.
