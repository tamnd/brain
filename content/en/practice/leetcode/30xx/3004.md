## Solution

## Problem Understanding

We are given a tree with `n` nodes labeled from `0` to `n - 1`. The tree is rooted at node `0`. The input array `edges` describes the connections between nodes, and the array `colors` assigns a color value to each node.

The goal is to find a subtree in which every node has exactly the same color. Among all such valid subtrees, we must return the maximum possible subtree size.

A subtree rooted at node `v` contains `v` itself and all of its descendants in the rooted tree.

The important detail is that the subtree must be completely monochromatic. It is not enough for the root and some children to match. Every node reachable below that root must share the same color.

For example, suppose node `2` has color `3`, and all nodes inside its subtree also have color `3`. Then the subtree rooted at `2` is valid. If even one descendant has a different color, then that subtree is invalid.

The constraints are important:

- `n` can be as large as `5 * 10^4`
- The graph is guaranteed to be a tree
- Colors can be any integer up to `10^5`

Because the tree can contain up to fifty thousand nodes, any algorithm worse than roughly linear time will likely time out. In particular, repeatedly traversing large subtrees for every node would become too expensive.

There are several edge cases worth identifying early:

- A tree with only one node. The answer must be `1`.
- A tree where all nodes have the same color. The answer is the size of the entire tree.
- A tree where every node has a different color. Every valid subtree has size `1`.
- Deep trees that resemble linked lists. Recursive DFS implementations must be careful about recursion depth in Python.
- Internal nodes whose immediate children match their color, but deeper descendants do not. A naive local check would fail here because the entire subtree must be uniform.

## Approaches

### Brute Force Approach

The brute force solution considers every node as a possible subtree root.

For each node:

1. Traverse the entire subtree rooted at that node.
2. Check whether every node inside the subtree has the same color as the root.
3. Count the subtree size if valid.
4. Keep track of the maximum valid size.

This approach is correct because it explicitly verifies every possible subtree independently.

However, it is too slow for the given constraints. In the worst case, each subtree traversal may visit nearly all nodes. If we do this for every node, the complexity becomes quadratic.

For a tree with `n = 5 * 10^4`, an `O(n^2)` solution is not feasible.

### Optimal Approach

The key observation is that subtree validity can be determined bottom up.

Suppose we perform a DFS traversal from the leaves upward.

For each node, we want to know:

- The size of its subtree
- Whether the entire subtree is monochromatic
- If monochromatic, what that color is

A subtree rooted at node `u` is monochromatic if and only if:

1. Every child subtree is monochromatic
2. Every child subtree has the same color as node `u`

This means we can combine child results efficiently during DFS.

Instead of repeatedly traversing the same subtrees, each node is processed exactly once.

This leads to a linear time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | `O(n^2)` | `O(n)` | Recomputes subtree information repeatedly |
| Optimal | `O(n)` | `O(n)` | Single DFS computes all subtree properties bottom up |

## Algorithm Walkthrough

1. Build an adjacency list for the tree.

Since the input only gives edges, we first convert them into a graph representation. An adjacency list is ideal because trees are sparse and we need efficient neighbor traversal.
2. Perform a DFS starting from the root node `0`.

The DFS processes children before their parent. This postorder behavior is important because a parent's validity depends on information from all child subtrees.
3. For each node, initialize:

- `subtree_size = 1`
- `is_valid = True`

Every node by itself forms a monochromatic subtree of size `1`.
4. Visit every child of the current node.

For each child, recursively obtain:

- Whether the child subtree is monochromatic
- The size of the child subtree
5. Accumulate subtree sizes.

Add the child subtree size into the current subtree size.
6. Validate subtree color consistency.

If either of the following is true:

- The child subtree is not monochromatic
- The child node color differs from the current node color

then the current subtree cannot be monochromatic.
7. After processing all children:

- If the subtree is valid, update the global answer using the subtree size.
- Return the validity and subtree size upward to the parent.
8. Continue until the DFS completes.

The final answer is the largest valid subtree size encountered during traversal.

### Why it works

The algorithm relies on a recursive invariant:

> A subtree is monochromatic if and only if all of its child subtrees are monochromatic and all child subtree colors match the root color.

Because DFS processes children before parents, every node receives complete and correct information about its descendants before determining its own validity.

Thus every subtree is evaluated exactly once, and the maximum valid size is computed correctly.

## Python Solution

```python
from typing import List
import sys

class Solution:
    def maximumSubtreeSize(self, edges: List[List[int]], colors: List[int]) -> int:
        n = len(colors)

        graph = [[] for _ in range(n)]

        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)

        sys.setrecursionlimit(100000)

        answer = 1

        def dfs(node: int, parent: int):
            nonlocal answer

            subtree_size = 1
            is_valid = True

            for neighbor in graph[node]:
                if neighbor == parent:
                    continue

                child_valid, child_size = dfs(neighbor, node)

                subtree_size += child_size

                if not child_valid or colors[neighbor] != colors[node]:
                    is_valid = False

            if is_valid:
                answer = max(answer, subtree_size)

            return is_valid, subtree_size

        dfs(0, -1)

        return answer
```

The implementation begins by constructing an adjacency list because the input edges are undirected.

The DFS function returns two pieces of information:

- Whether the subtree rooted at the current node is monochromatic
- The total size of that subtree

Each node starts as a valid monochromatic subtree of size `1`.

As child subtrees are processed, their sizes are accumulated. If any child subtree is invalid, or if the child color differs from the current node color, the current subtree becomes invalid.

Whenever a subtree remains valid after processing all children, the global answer is updated.

The recursion limit is increased because the tree may become extremely deep, especially in linked-list shaped inputs.

## Go Solution

```go
package main

func maximumSubtreeSize(edges [][]int, colors []int) int {
	n := len(colors)

	graph := make([][]int, n)

	for _, edge := range edges {
		u := edge[0]
		v := edge[1]

		graph[u] = append(graph[u], v)
		graph[v] = append(graph[v], u)
	}

	answer := 1

	var dfs func(int, int) (bool, int)

	dfs = func(node int, parent int) (bool, int) {
		subtreeSize := 1
		isValid := true

		for _, neighbor := range graph[node] {
			if neighbor == parent {
				continue
			}

			childValid, childSize := dfs(neighbor, node)

			subtreeSize += childSize

			if !childValid || colors[neighbor] != colors[node] {
				isValid = false
			}
		}

		if isValid && subtreeSize > answer {
			answer = subtreeSize
		}

		return isValid, subtreeSize
	}

	dfs(0, -1)

	return answer
}
```

The Go implementation follows the same logic as the Python version.

Because Go slices are dynamic, the adjacency list is naturally represented as `[][]int`.

The DFS function returns two values:

- `bool` for subtree validity
- `int` for subtree size

Unlike Python, Go does not require recursion limit adjustments. However, extremely deep recursion could still theoretically risk stack growth, though it is usually acceptable for this constraint range.

## Edge Cases

### Single Node Tree

A tree containing only one node is always a valid monochromatic subtree because there are no descendants that could violate the condition.

This case can expose bugs in implementations that assume at least one edge exists. The current implementation handles it naturally because the DFS starts with subtree size `1` and never enters the child loop.

### Deep Skewed Tree

A tree shaped like a linked list can produce recursion depth close to `n`.

In Python, this may exceed the default recursion limit. The implementation explicitly increases the recursion limit using `sys.setrecursionlimit(100000)` so that deep DFS traversals remain safe.

### Internal Color Mismatch

A subtree may appear valid locally while actually containing a deeper mismatch.

For example:

```
0(1)
|
1(1)
|
2(2)
```

Node `1` matches node `0`, but descendant node `2` breaks the monochromatic property.

A naive implementation that checks only immediate children would incorrectly accept the root subtree. The DFS solution avoids this by propagating subtree validity upward from descendants. Once node `2` invalidates node `1`, node `0` also becomes invalid automatically.
