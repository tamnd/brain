# LeetCode 3032 - Count Numbers With Unique Digits II

## Problem Understanding

This problem asks us to count how many integers in the inclusive range `[a, b]` contain only unique digits. A number has unique digits if no digit appears more than once in its decimal representation.

For example, the number `123` has unique digits because each digit appears exactly once. The number `121` does not have unique digits because the digit `1` appears twice.

The input consists of two positive integers, `a` and `b`. We must examine every number from `a` through `b` and determine whether its digits are all distinct. The final answer is the number of integers that satisfy this property.

The constraints are very small:

- `1 <= a <= b <= 1000`

This immediately tells us that performance is not a major concern. The largest possible range contains at most `1000` numbers. Even a straightforward brute-force solution will run comfortably within time limits.

Still, it is important to reason carefully about digit handling. Some edge cases can easily cause bugs:

- Numbers with repeated zeros, such as `1000`
- Numbers with repeated middle digits, such as `232`
- Single digit numbers, which are always valid
- Ranges where every number is valid
- Ranges where many numbers are invalid

The problem guarantees that all inputs are positive integers and that `a <= b`, so we never need to handle invalid ranges or negative numbers.

## Approaches

### Brute Force Approach

The most direct solution is to iterate through every number from `a` to `b`. For each number, we inspect its digits and determine whether any digit appears more than once.

One simple way to do this is:

1. Convert the number into a string.
2. Put all digits into a set.
3. Compare the size of the set with the length of the string.

If both sizes are equal, every digit appeared exactly once, so the number is valid.

This approach is correct because a set automatically removes duplicates. Therefore, if duplicates exist, the set size becomes smaller than the total number of digits.

Although brute force is sometimes too slow for large ranges, here the maximum range size is only `1000`, so this method is completely acceptable.

### Optimal Observation

Because the constraints are so small, the brute-force solution is already effectively optimal. There is no need for advanced digit dynamic programming or combinatorics.

The key insight is that checking uniqueness of digits is extremely cheap. Each number contains at most four digits because `b <= 1000`.

That means the total work is approximately:

- At most `1000` numbers
- Each requiring at most `4` digit checks

This results in only a few thousand operations overall.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * d) | O(d) | Check every number using a set of digits |
| Optimal | O(n * d) | O(d) | Same practical approach, constraints are already tiny |

Here:

- `n = b - a + 1`
- `d = number of digits`, at most `4`

## Algorithm Walkthrough

1. Initialize a counter variable `count` to `0`.
2. Iterate through every integer `num` in the range `[a, b]`.
3. Convert the number into a string so each digit can be processed individually.
4. Create a set from the digits of the string.
5. Compare the size of the set with the length of the string.

- If the sizes are equal, every digit is unique.
- Otherwise, at least one digit repeats.
6. If the number has unique digits, increment `count`.
7. After processing all numbers, return `count`.

### Why it works

The algorithm works because sets store only distinct values. If a digit appears multiple times, the set removes duplicates automatically. Therefore:

- Unique digits, set size equals string length
- Repeated digits, set size becomes smaller

Since every number in the range is checked exactly once, the final count is correct.

## Python Solution

```python
class Solution:
    def numberCount(self, a: int, b: int) -> int:
        def has_unique_digits(num: int) -> bool:
            digits = str(num)
            return len(digits) == len(set(digits))

        count = 0

        for num in range(a, b + 1):
            if has_unique_digits(num):
                count += 1

        return count
```

The implementation begins with a helper function called `has_unique_digits`. This function converts the number into a string and creates a set from its digits.

Because sets remove duplicates, comparing the lengths tells us whether all digits are distinct.

The main loop iterates from `a` through `b`, inclusive. Every valid number increments the counter.

Finally, the function returns the total count.

The implementation is intentionally simple because the constraints are very small. Readability is more important than micro-optimizations here.

## Go Solution

```go
func numberCount(a int, b int) int {
    hasUniqueDigits := func(num int) bool {
        seen := make(map[rune]bool)
        
        for _, ch := range []rune(string(runeSlice(num))) {
            if seen[ch] {
                return false
            }
            seen[ch] = true
        }
        
        return true
    }

    count := 0

    for num := a; num <= b; num++ {
        if hasUniqueDigits(num) {
            count++
        }
    }

    return count
}

func runeSlice(num int) []rune {
    str := ""
    
    if num == 0 {
        return []rune{'0'}
    }

    for num > 0 {
        digit := num % 10
        str = string(rune('0'+digit)) + str
        num /= 10
    }

    return []rune(str)
}
```

A cleaner and more idiomatic Go version uses `strconv.Itoa`, which is preferable in production code:

```go
import "strconv"

func numberCount(a int, b int) int {
    hasUniqueDigits := func(num int) bool {
        digits := strconv.Itoa(num)
        seen := make(map[rune]bool)

        for _, ch := range digits {
            if seen[ch] {
                return false
            }
            seen[ch] = true
        }

        return true
    }

    count := 0

    for num := a; num <= b; num++ {
        if hasUniqueDigits(num) {
            count++
        }
    }

    return count
}
```

The Go implementation follows the same logic as the Python version.

A map is used to track which digits have already appeared. As soon as a repeated digit is found, the function immediately returns `false`.

Go does not have a built-in set type, so a `map[rune]bool` is the standard substitute.

Integer overflow is not a concern because all values are at most `1000`.

## Edge Cases

One important edge case is a range containing only a single number. For example, `[11, 11]` should return `0`, while `[7, 7]` should return `1`. Some implementations accidentally assume multiple values exist in the range. This solution handles the case naturally because the loop still executes exactly once.

Another important case involves repeated zeros, such as `100` or `1000`. It is easy to overlook zeros when checking uniqueness. Since the implementation converts the entire number into a string and inserts every digit into a set, repeated zeros are detected correctly.

A third edge case occurs when transitioning between digit lengths, such as the range `[98, 102]`. Some buggy implementations incorrectly handle leading zeros or digit extraction across boundaries. This implementation avoids those issues entirely because it processes the exact decimal representation produced by the language runtime.

Finally, numbers with repeated non-adjacent digits, such as `121` or `909`, must also be rejected. The set-based approach catches duplicates regardless of position, ensuring these cases are handled correctly.
