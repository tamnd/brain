## Solution

## Problem Understanding

The problem asks us to find the longest **special substring** of a string `s` that occurs **at least three times**. A special substring is defined as a substring made up of the same character repeated consecutively. For example, `"aaa"` is special because it consists of only `'a'`, whereas `"abc"` is not special. The input string `s` consists only of lowercase English letters, and its length can be up to 500,000 characters, which is important because it rules out naive quadratic solutions.

The output should be a single integer representing the length of the longest special substring that appears three or more times, or `-1` if no such substring exists. Edge cases include strings with no repeated characters, strings where only one character occurs multiple times but never in a long enough sequence, and strings where the whole string is a repetition of a single character.

The key challenge is efficiently counting consecutive identical characters while keeping track of repeated patterns of different lengths.

## Approaches

A **brute-force** approach would enumerate every possible substring, check if it is special, and then count occurrences. This involves generating all `O(n^2)` substrings and checking each against the string for repeated appearances. While correct, this approach is infeasible for the constraint `n <= 5 * 10^5` due to quadratic time complexity.

The **optimal approach** leverages the observation that a special substring is fully determined by a **character and its consecutive run length**. We can first group the string into runs of identical characters. For example, `"aaabb"` becomes `[('a',3),('b',2)]`. Then, to check if a special substring of length `k` occurs at least three times, we simply check whether there are at least three runs (or overlapping positions within runs) of that character where the run length is at least `k`. Using **binary search** on possible lengths of substrings (from `1` to the max run length) allows us to efficiently find the maximum valid length. This reduces the problem to `O(n log n)`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(1) | Check all substrings and count occurrences. Too slow for large `n`. |
| Optimal | O(n log n) | O(n) | Use run-length encoding and binary search on substring length. |

## Algorithm Walkthrough

1. **Run-Length Encoding**: Traverse the string and group consecutive identical characters as `(char, run_length)` pairs. This helps us quickly check lengths of special substrings without scanning the string repeatedly.
2. **Binary Search on Lengths**: Define a search range from `1` to the maximum run length in the string. Use binary search to efficiently find the largest substring length `k` that occurs at least three times.
3. **Check Occurrences**: For each candidate length `k`, iterate through the run-length groups. Count how many positions can generate a special substring of length `k` (i.e., each run of length `r >= k` contributes `r - k + 1` valid substrings). Sum contributions across all runs for the same character. If any character reaches at least three occurrences, the length `k` is valid.
4. **Adjust Binary Search**: If `k` is valid, try a larger length. If not, try smaller lengths. Continue until convergence.
5. **Return Result**: If a valid length was found, return it. Otherwise, return `-1`.

**Why it works**: Run-length encoding reduces the problem to counting substrings in contiguous blocks. Binary search ensures we efficiently find the maximum length rather than testing each possibility linearly. The check guarantees that any candidate length has at least three occurrences, satisfying the problem constraints.

## Python Solution

```python
class Solution:
    def maximumLength(self, s: str) -> int:
        from collections import defaultdict
        
        n = len(s)
        if n < 3:
            return -1
        
        # Step 1: Run-length encoding
        runs = []
        prev = s[0]
        length = 1
        for c in s[1:]:
            if c == prev:
                length += 1
            else:
                runs.append((prev, length))
                prev = c
                length = 1
        runs.append((prev, length))
        
        max_run = max(length for _, length in runs)
        
        # Step 2: Binary search on substring length
        left, right = 1, max_run
        ans = -1
        
        while left <= right:
            mid = (left + right) // 2
            count_map = defaultdict(int)
            
            for char, run_len in runs:
                if run_len >= mid:
                    count_map[char] += run_len - mid + 1
            
            if any(v >= 3 for v in count_map.values()):
                ans = mid
                left = mid + 1
            else:
                right = mid - 1
        
        return ans
```

The Python code first builds the run-length encoding of the string. It then uses binary search to efficiently test candidate substring lengths, counting how many occurrences each character contributes. The `defaultdict` stores the cumulative count for each character. The algorithm updates the search bounds based on whether a valid substring length was found, ultimately returning the longest valid length or `-1`.

## Go Solution

```go
func maximumLength(s string) int {
    n := len(s)
    if n < 3 {
        return -1
    }
    
    // Step 1: Run-length encoding
    type Run struct {
        char  byte
        count int
    }
    runs := []Run{}
    prev := s[0]
    length := 1
    for i := 1; i < n; i++ {
        if s[i] == prev {
            length++
        } else {
            runs = append(runs, Run{prev, length})
            prev = s[i]
            length = 1
        }
    }
    runs = append(runs, Run{prev, length})
    
    maxRun := 0
    for _, r := range runs {
        if r.count > maxRun {
            maxRun = r.count
        }
    }
    
    // Step 2: Binary search
    left, right := 1, maxRun
    ans := -1
    
    for left <= right {
        mid := (left + right) / 2
        countMap := make(map[byte]int)
        for _, r := range runs {
            if r.count >= mid {
                countMap[r.char] += r.count - mid + 1
            }
        }
        valid := false
        for _, v := range countMap {
            if v >= 3 {
                valid = true
                break
            }
        }
        if valid {
            ans = mid
            left = mid + 1
        } else {
            right = mid - 1
        }
    }
    
    return ans
}
```

The Go implementation mirrors the Python version, with slices replacing Python lists and maps replacing `defaultdict`. Edge cases such as empty runs are handled implicitly, and integer arithmetic is safe within the given constraints.

## Worked Examples

### Example 1: `"aaaa"`

Run-length encoding: `[('a',4)]`

Max run = 4

Binary search steps:

| mid | count_map | valid? |
| --- | --- | --- |
| 2 | {'a':3} | Yes |
| 3 | {'a':2} | No |

Longest valid length = 2

### Example 2: `"abcdef"`

Run-length encoding: `[('a',1), ('b',1), ('c',1), ('d',1), ('e',1), ('f',1)]`

No run has length >= 1 with 3 occurrences → return -1

### Example 3: `"abcaba"`

Run-length encoding: `[('a',1),('b',1),('c',1),('a',1),('b',1),('a',1)]`

For length 1, count of 'a' = 3 → valid → return 1

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n log n) | Run-length encoding is O(n). Binary search tries log(max_run) lengths, each checked in O(n). |
| Space | O(n) | Store run-length encoding of up to n runs. Map of character counts is O(26) = O(1). |

The log factor comes from the binary search on possible substring lengths, and n from scanning all runs for each length.
