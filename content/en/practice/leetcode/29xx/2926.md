## Solution

## Problem Understanding

This problem asks us to find the maximum sum of a **balanced subsequence** from a given array of integers, `nums`. A subsequence is any subset of elements taken in order from the original array. A subsequence is defined as **balanced** if, for each consecutive pair of indices `(i, j)` in the subsequence, the difference in values `nums[j] - nums[i]` is at least as large as the difference in their indices `j - i`. A subsequence with a single element is automatically considered balanced.

The input is a 0-indexed integer array with length up to `10^5` and integer values in the range `[-10^9, 10^9]`. The output is a single integer representing the largest sum obtainable from any balanced subsequence.

Important points include: the array can contain negative numbers, subsequences do not need to be contiguous, and the constraints make a naive O(2^n) approach infeasible. Edge cases that can trip naive implementations include arrays with all negative numbers, arrays with only one element, and arrays where skipping some elements drastically increases the sum while maintaining balance.

## Approaches

### Brute Force

A brute-force approach would generate all possible subsequences of the array and check if each subsequence satisfies the balanced condition. For each valid balanced subsequence, we compute its sum and track the maximum sum encountered. This approach is correct but impractical, as generating all subsequences has O(2^n) time complexity, which is far too slow for n up to `10^5`.

### Optimal Approach

The key insight is that the balanced condition can be rewritten in a form that allows dynamic programming. For a subsequence ending at index `i`, we only need to consider previous indices `j < i` where `nums[i] - nums[j] >= i - j`. This inequality can be transformed into `nums[i] - i >= nums[j] - j`.

Define `dp[i]` as the maximum sum of a balanced subsequence ending at index `i`. Then:

```
dp[i] = nums[i] + max(dp[j]) for all j < i with nums[j] - j <= nums[i] - i
```

This can be efficiently implemented using a data structure that supports fast range maximum queries, such as a **Segment Tree**, **Binary Indexed Tree**, or a sorted map, after transforming `nums[j] - j` to keys. Using this approach reduces the complexity from O(2^n) to O(n log n).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Generate all subsequences and check balance |
| Optimal | O(n log n) | O(n) | Dynamic programming with segment tree / BIT to maintain maximum dp values |

## Algorithm Walkthrough

1. Preprocess the array by computing `key[i] = nums[i] - i` for each index `i`. This transforms the balanced condition into a simpler form: for subsequence ending at `i`, consider all `j < i` where `key[j] <= key[i]`.
2. Coordinate-compress the `key` values to map them into a smaller index range. This allows us to use a segment tree or BIT efficiently.
3. Initialize a segment tree (or BIT) that supports range maximum queries and point updates.
4. Iterate over each element `i` from left to right. For each `i`:

1. Query the segment tree for the maximum `dp[j]` among all `j` with `key[j] <= key[i]`.
2. Set `dp[i] = nums[i] + max_previous_dp` (if no previous, `dp[i] = nums[i]`).
3. Update the segment tree at the position corresponding to `key[i]` with `dp[i]`.
5. After processing all elements, the answer is `max(dp)`.

**Why it works:** The segment tree efficiently maintains the maximum sum of all balanced subsequences ending at indices with a particular `key`. By always querying the maximum `dp[j]` for valid predecessors, we ensure the subsequence remains balanced while achieving the maximum sum.

## Python Solution

```python
from typing import List
import bisect

class SegmentTree:
    def __init__(self, size: int):
        self.N = 1
        while self.N < size:
            self.N *= 2
        self.tree = [float('-inf')] * (2 * self.N)
    
    def update(self, idx: int, value: int):
        idx += self.N
        self.tree[idx] = max(self.tree[idx], value)
        while idx > 1:
            idx //= 2
            self.tree[idx] = max(self.tree[2*idx], self.tree[2*idx+1])
    
    def query(self, l: int, r: int) -> int:
        l += self.N
        r += self.N
        res = float('-inf')
        while l <= r:
            if l % 2 == 1:
                res = max(res, self.tree[l])
                l += 1
            if r % 2 == 0:
                res = max(res, self.tree[r])
                r -= 1
            l //= 2
            r //= 2
        return res

class Solution:
    def maxBalancedSubsequenceSum(self, nums: List[int]) -> int:
        n = len(nums)
        keys = [nums[i] - i for i in range(n)]
        unique_keys = sorted(set(keys))
        idx_map = {key: idx for idx, key in enumerate(unique_keys)}
        
        seg = SegmentTree(len(unique_keys))
        max_sum = float('-inf')
        
        for i in range(n):
            key_idx = idx_map[keys[i]]
            best_prev = seg.query(0, key_idx)
            dp_i = nums[i] + (best_prev if best_prev != float('-inf') else 0)
            seg.update(key_idx, dp_i)
            max_sum = max(max_sum, dp_i)
        
        return max_sum
```

**Explanation:** We first transform `nums[i] - i` into `keys` to simplify the balanced condition. We coordinate-compress these keys to allow efficient use in a segment tree. We iterate over `nums`, querying the maximum previous `dp` value that satisfies the balanced condition, then update the segment tree with the new `dp` value. The final result is the maximum value in `dp`.

## Go Solution

```go
package main

import "math"

type SegmentTree struct {
    N int
    tree []int64
}

func NewSegmentTree(size int) *SegmentTree {
    N := 1
    for N < size {
        N *= 2
    }
    tree := make([]int64, 2*N)
    for i := range tree {
        tree[i] = math.MinInt64
    }
    return &SegmentTree{N, tree}
}

func (st *SegmentTree) Update(idx int, value int64) {
    idx += st.N
    if st.tree[idx] < value {
        st.tree[idx] = value
    }
    for idx > 1 {
        idx /= 2
        st.tree[idx] = max(st.tree[2*idx], st.tree[2*idx+1])
    }
}

func (st *SegmentTree) Query(l, r int) int64 {
    res := int64(math.MinInt64)
    l += st.N
    r += st.N
    for l <= r {
        if l%2 == 1 {
            res = max(res, st.tree[l])
            l++
        }
        if r%2 == 0 {
            res = max(res, st.tree[r])
            r--
        }
        l /= 2
        r /= 2
    }
    return res
}

func max(a, b int64) int64 {
    if a > b {
        return a
    }
    return b
}

func maxBalancedSubsequenceSum(nums []int) int64 {
    n := len(nums)
    keys := make([]int, n)
    for i := 0; i < n; i++ {
        keys[i] = nums[i] - i
    }

    keySet := make(map[int]struct{})
    for _, k := range keys {
        keySet[k] = struct{}{}
    }
    uniqueKeys := make([]int, 0, len(keySet))
    for k := range keySet {
        uniqueKeys = append(uniqueKeys, k)
    }
    sort.Ints(uniqueKeys)
    idxMap := make(map[int]int)
    for i, k := range uniqueKeys {
        idxMap[k] = i
    }

    seg := NewSegmentTree(len(uniqueKeys))
    maxSum := int64(math.MinInt64)
    for i := 0; i < n; i++ {
        keyIdx := idxMap[keys[i]]
        bestPrev := seg.Query(0, keyIdx)
        var dpI int64
        if bestPrev == math.MinInt64 {
            dpI = int64(nums[i])
        } else {
            dpI = int64(nums[i]) + bestPrev
        }
        seg.Update(keyIdx, dpI)
        if dpI > maxSum {
            maxSum = dpI
        }
    }
    return maxSum
}
```
