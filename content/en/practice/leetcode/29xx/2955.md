## Solution

## Problem Understanding

We are given a string `s` and multiple queries. Each query specifies a substring `s[l..r]`, and for every query we must count how many substrings inside that range have the same starting and ending character.

A substring is considered "same-end" if:

```
substring[0] == substring[len(substring) - 1]
```

For example, in the substring `"abcaab"`:

- `"a"` is valid because the first and last character are both `'a'`
- `"bcaa"` is not valid because `'b' != 'a'`
- `"aa"` is valid because both ends are `'a'`

The important detail is that we are counting all substrings inside the queried interval, not just the query substring itself.

Suppose we look at a query `[l, r]`. Every valid substring is determined by choosing two indices `i` and `j` such that:

```
l <= i <= j <= r
s[i] == s[j]
```

This works because once the first and last characters match, the substring `s[i..j]` is automatically a same-end substring.

The constraints are large:

- `s.length <= 3 * 10^4`
- `queries.length <= 3 * 10^4`

A naive solution that checks every substring for every query would be far too slow. Since a substring range can contain `O(n^2)` substrings, we need something much more efficient.

There are several important edge cases to keep in mind:

- A query with a single character, such as `[3,3]`, always has exactly one valid substring.
- A substring where all characters are distinct only contributes single-character substrings.
- A substring where all characters are identical produces the maximum possible number of same-end substrings.
- Multiple queries may overlap heavily, so preprocessing is extremely valuable.

## Approaches

### Brute Force Approach

The most direct solution is to process each query independently.

For a query `[l, r]`, we enumerate every possible substring `s[i..j]` where:

```
l <= i <= j <= r
```

For each substring, we check whether:

```
s[i] == s[j]
```

If they match, we increment the answer.

This approach is correct because every substring is explicitly examined exactly once. However, the complexity is too large.

For a substring of length `k`, there are:

```
k * (k + 1) / 2
```

possible substrings. In the worst case, `k` can be `3 * 10^4`, which makes a single query already too expensive.

With up to `3 * 10^4` queries, this approach becomes completely infeasible.

### Optimal Approach

The key observation is that a substring is valid if and only if its first and last characters are equal.

For any character `c`, suppose it appears `freq` times inside the query range. Then:

- Every single occurrence contributes one valid length-1 substring.
- Every pair of occurrences forms a valid longer substring.

The number of valid substrings contributed by character `c` is therefore:

$\frac{freq \cdot (freq + 1)}{2}$

So instead of enumerating substrings, we only need to know how many times each character appears inside the query interval.

Since the string contains only lowercase English letters, there are only 26 possible characters. We can preprocess prefix frequencies for every character.

Let:

```
prefix[i][c]
```

represent how many times character `c` appears in `s[0..i-1]`.

Then the frequency of character `c` inside query `[l, r]` is:

```
prefix[r + 1][c] - prefix[l][c]
```

We compute this for all 26 characters and sum:

$\sum_{c='a'}^{'z'} \frac{freq_c \cdot (freq_c + 1)}{2}$

This gives an efficient solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(q * n²) | O(1) | Enumerates every substring for every query |
| Optimal | O(n * 26 + q * 26) | O(n * 26) | Uses prefix frequency counts |

## Algorithm Walkthrough

1. Create a prefix frequency table.

We build a 2D array `prefix` where:

```
prefix[i][c]
```

stores how many times character `c` appears in the first `i` characters of the string.

The table has `n + 1` rows so that range calculations become simple.
2. Copy frequencies from the previous position.

For each index `i`, we start by copying all counts from `prefix[i]` into `prefix[i + 1]`.

This ensures the next row contains all frequencies seen so far.
3. Increment the current character count.

If `s[i]` is `'c'`, we increment the corresponding frequency in `prefix[i + 1]`.
4. Process each query independently.

For query `[l, r]`, we compute the frequency of every character inside the interval using:

```
freq = prefix[r + 1][c] - prefix[l][c]
```
5. Count valid substrings contributed by each character.

If a character appears `freq` times, then the number of substrings whose ends are that character equals:

$\frac{freq(freq+1)}{2}$

We add this contribution to the answer.
6. Store the result.

After processing all 26 characters, we append the total to the result array.

### Why it works

A substring is uniquely identified by choosing its starting and ending indices. A substring is same-end exactly when the characters at those two positions are equal.

For a fixed character appearing `freq` times, every choice of:

- one occurrence with itself
- or two distinct occurrences

forms exactly one valid substring.

The number of such choices is:

```
freq + C(freq, 2)
```

which simplifies to:

$\frac{freq(freq+1)}{2}$

Summing across all 26 characters counts every valid substring exactly once.

## Python Solution

```python
from typing import List

class Solution:
    def sameEndSubstringCount(self, s: str, queries: List[List[int]]) -> List[int]:
        n = len(s)

        # prefix[i][c] = frequency of character c
        # in s[0:i]
        prefix = [[0] * 26 for _ in range(n + 1)]

        for i, ch in enumerate(s):
            for c in range(26):
                prefix[i + 1][c] = prefix[i][c]

            index = ord(ch) - ord('a')
            prefix[i + 1][index] += 1

        answers = []

        for left, right in queries:
            total = 0

            for c in range(26):
                freq = prefix[right + 1][c] - prefix[left][c]
                total += freq * (freq + 1) // 2

            answers.append(total)

        return answers
```

The implementation begins by constructing the prefix frequency table. Each row represents the cumulative character counts up to a particular index.

The extra row at index `0` represents an empty prefix. This allows range frequencies to be computed cleanly using subtraction.

For each query, we iterate through all 26 lowercase letters. The frequency of each character inside the interval is obtained in constant time using the prefix table.

Once the frequency is known, we compute how many same-end substrings that character contributes. The formula:

```python
freq * (freq + 1) // 2
```

counts all valid start-end position pairs for that character.

Finally, we append the computed total to the answer list.

## Go Solution

```go
func sameEndSubstringCount(s string, queries [][]int) []int {
	n := len(s)

	prefix := make([][]int, n+1)
	for i := range prefix {
		prefix[i] = make([]int, 26)
	}

	for i := 0; i < n; i++ {
		for c := 0; c < 26; c++ {
			prefix[i+1][c] = prefix[i][c]
		}

		index := int(s[i] - 'a')
		prefix[i+1][index]++
	}

	result := make([]int, 0, len(queries))

	for _, query := range queries {
		left := query[0]
		right := query[1]

		total := 0

		for c := 0; c < 26; c++ {
			freq := prefix[right+1][c] - prefix[left][c]
			total += freq * (freq + 1) / 2
		}

		result = append(result, total)
	}

	return result
}
```

The Go implementation follows the exact same logic as the Python version.

The main difference is that Go requires explicit slice allocation for the 2D prefix array. Integer arithmetic is safe here because the maximum number of substrings is well within Go's `int` range for the given constraints.

Slices are dynamically sized, so the result array is initialized with capacity equal to the number of queries for efficiency.

## Edge Cases

### Single Character Queries

A query like `[i, i]` contains exactly one substring, namely the single character itself. Since every one-character substring trivially has the same start and end character, the answer must always be `1`.

This can be a source of bugs if the implementation assumes substrings must have length greater than one. The formula handles this naturally because a frequency of `1` contributes:

```
1 * (1 + 1) / 2 = 1
```

### All Characters Distinct

Consider a string like `"abcdef"`.

No substring longer than one character can have matching ends because every character appears only once. Therefore, the answer equals the substring length.

A buggy implementation might accidentally count invalid pairs or mis-handle frequencies. The prefix-frequency method works correctly because every character frequency is exactly `1`, so each contributes only one substring.

### All Characters Identical

For a substring like `"aaaa"`, every possible substring is valid.

The total number of substrings is:

$\frac{n(n+1)}{2}$

This is an important stress case because it produces the maximum possible answer.

The implementation handles this correctly because the character `'a'` appears `n` times, contributing exactly the total number of possible substrings.
