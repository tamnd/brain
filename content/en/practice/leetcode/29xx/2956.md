# LeetCode 2956 - Find Common Elements Between Two Arrays

## Problem Understanding

The problem gives us two integer arrays, `nums1` and `nums2`. We must determine how many elements from each array also appear in the other array.

More specifically, we need to calculate two values:

- `answer1` is the number of indices `i` where `nums1[i]` exists somewhere inside `nums2`.
- `answer2` is the number of indices `i` where `nums2[i]` exists somewhere inside `nums1`.

The important detail is that we are counting indices, not distinct values. If a value appears multiple times in one array and also exists in the other array, every matching occurrence contributes to the answer.

For example, if `nums1 = [2,3,2]` and `nums2 = [1,2]`, the value `2` appears twice in `nums1`, and since `2` exists in `nums2`, both occurrences count toward `answer1`.

The input constraints are very small:

- `1 <= n, m <= 100`
- `1 <= nums1[i], nums2[i] <= 100`

These constraints tell us several things:

- The arrays are short, so even quadratic solutions are acceptable.
- The values are also small, which means frequency arrays or hash sets are easy to use.
- Performance is not a major concern, but we should still aim for clean and efficient logic.

Some important edge cases include:

- Arrays with no common elements.
- Arrays where every element matches.
- Arrays containing many duplicates.
- Arrays of size 1.
- Arrays where one value repeats many times.

The problem guarantees that both arrays contain at least one element, so we never need to handle empty arrays.

## Approaches

### Brute Force Approach

The simplest solution is to check every element of one array against every element of the other array.

For each value in `nums1`, we scan through `nums2` to see whether the value exists. If we find a match, we increment `answer1`.

We then repeat the same process in reverse for `nums2`.

This works because we directly verify whether each value exists in the opposite array. However, this requires nested loops.

If `n` is the length of `nums1` and `m` is the length of `nums2`, the total work becomes:

- `O(n * m)` for computing `answer1`
- `O(m * n)` for computing `answer2`

Overall complexity is `O(n * m)`.

Although this is perfectly acceptable for the given constraints, we can do better conceptually.

### Optimal Approach Using Hash Sets

The key observation is that we only care whether a value exists in the other array. We do not care how many times it appears there.

A hash set provides constant time membership checking on average.

We can:

1. Convert `nums1` into a set.
2. Convert `nums2` into a set.
3. Count how many elements in `nums1` appear in the set built from `nums2`.
4. Count how many elements in `nums2` appear in the set built from `nums1`.

This avoids repeated scanning and makes the logic simpler and more efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × m) | O(1) | Compare every element against the other array |
| Optimal | O(n + m) | O(n + m) | Uses hash sets for constant time membership checks |

## Algorithm Walkthrough

1. Create a hash set from `nums1`.

This allows us to quickly check whether a value exists in `nums1`. Hash sets provide average `O(1)` lookup time.
2. Create a hash set from `nums2`.

Similarly, this allows fast existence checks for values from `nums2`.
3. Initialize `answer1` to `0`.

This variable will count how many elements in `nums1` appear in `nums2`.
4. Iterate through every element in `nums1`.

For each value:

- Check whether it exists in the `nums2` set.
- If it does, increment `answer1`.

We iterate through the original array instead of the set because duplicates must count multiple times.
5. Initialize `answer2` to `0`.

This variable will count how many elements in `nums2` appear in `nums1`.
6. Iterate through every element in `nums2`.

For each value:

- Check whether it exists in the `nums1` set.
- If it does, increment `answer2`.
7. Return `[answer1, answer2]`.

### Why it works

The algorithm works because the sets contain exactly the distinct values present in each array. For every element in one array, we correctly determine whether the same value exists in the other array through constant time membership checks.

Since we iterate through the original arrays, duplicates are counted correctly. Every index whose value exists in the opposite array contributes exactly once to the answer.

## Python Solution

```python
from typing import List

class Solution:
    def findIntersectionValues(self, nums1: List[int], nums2: List[int]) -> List[int]:
        nums1_set = set(nums1)
        nums2_set = set(nums2)

        answer1 = 0
        for value in nums1:
            if value in nums2_set:
                answer1 += 1

        answer2 = 0
        for value in nums2:
            if value in nums1_set:
                answer2 += 1

        return [answer1, answer2]
```

The implementation begins by converting both arrays into sets. This preprocessing step allows efficient membership checks throughout the algorithm.

Next, the code iterates through `nums1`. For every value, it checks whether the value exists in `nums2_set`. If so, it increments `answer1`.

The same logic is then applied in reverse for `nums2`.

Finally, the method returns the two counts as a list.

An important detail is that we iterate over the original arrays instead of the sets. This ensures duplicate occurrences are counted correctly, which matches the problem requirements.

## Go Solution

```go
func findIntersectionValues(nums1 []int, nums2 []int) []int {
    nums1Set := make(map[int]bool)
    nums2Set := make(map[int]bool)

    for _, value := range nums1 {
        nums1Set[value] = true
    }

    for _, value := range nums2 {
        nums2Set[value] = true
    }

    answer1 := 0
    for _, value := range nums1 {
        if nums2Set[value] {
            answer1++
        }
    }

    answer2 := 0
    for _, value := range nums2 {
        if nums1Set[value] {
            answer2++
        }
    }

    return []int{answer1, answer2}
}
```

The Go implementation uses `map[int]bool` to simulate hash sets because Go does not have a built in set type.

The logic is otherwise identical to the Python version. We first populate both maps, then iterate through each original array and count elements whose values exist in the opposite map.

Go slices naturally handle dynamic array behavior, so returning `[]int{answer1, answer2}` is straightforward.

Integer overflow is not a concern because the maximum array size is only 100.

## Edge Cases

One important edge case is when there are no common elements between the arrays. A careless implementation might accidentally count partial matches or fail to reset counters properly. In our implementation, membership checks are explicit, so values that do not exist in the opposite set simply do not increment the answer.

Another important case involves duplicates. Since the problem counts indices rather than distinct values, repeated numbers must be counted multiple times. A common mistake is iterating over the sets instead of the original arrays, which would incorrectly remove duplicates from consideration. Our implementation correctly iterates through the original arrays while only using sets for fast lookup.

A third edge case is very small input sizes, especially arrays of length 1. Some implementations accidentally assume multiple elements exist or mishandle initialization logic. Our solution works naturally for single element arrays because the loops and set construction operate correctly regardless of array size.

A fourth edge case occurs when every element matches. In this situation, the result should equal the size of each array. Since every membership check succeeds, the counters correctly increase for every index visited.
