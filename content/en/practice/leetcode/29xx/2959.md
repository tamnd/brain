# LeetCode 2959 - Number of Possible Sets of Closing Branches

## Problem Understanding

This problem asks us to count how many different subsets of branches can be closed while still satisfying a distance constraint among all remaining active branches.

We are given `n` branches labeled from `0` to `n - 1`, and an undirected weighted graph represented by `roads`. Each road connects two branches and has a travel distance. Initially, the graph is fully connected, meaning every branch can reach every other branch through some sequence of roads.

We may choose any subset of branches to close, including closing none of them or even closing all of them. Once a branch is closed, all roads connected to it disappear as well. After removing the closed branches and their roads, the remaining active branches must satisfy this condition:

For every pair of active branches, the shortest path distance between them must be at most `maxDistance`.

The goal is to count how many different closure sets satisfy this property.

A useful way to think about the problem is to instead enumerate the set of branches that remain active. Every subset of active branches corresponds to exactly one subset of closed branches. For each active subset, we must verify whether all pairwise shortest path distances inside that induced subgraph are at most `maxDistance`.

The constraints are extremely important here:

- `n <= 10`
- `roads.length <= 1000`

The small value of `n` immediately suggests that exponential enumeration over subsets is feasible. Since there are only `2^10 = 1024` subsets, we can examine every possible active branch configuration.

The graph may contain multiple roads between the same pair of nodes, so we must always keep the minimum edge weight between two branches when building the graph.

Several edge cases are important:

- The empty active set is always valid because there are no branches violating the distance condition.
- A single active branch is always valid because no pairwise distances exist.
- Multiple edges between the same branches can change the shortest path significantly.
- Removing branches may disconnect the graph, which invalidates the subset unless the disconnected nodes are absent.
- Shortest paths are only allowed to use active branches. We cannot travel through closed branches.

## Approaches

### Brute Force Approach

The most direct solution is to enumerate every subset of active branches. For each subset:

1. Construct the induced graph containing only active branches.
2. Compute all-pairs shortest paths.
3. Verify that every pair of active branches has distance at most `maxDistance`.

If the condition holds, increment the answer.

This approach is correct because it explicitly checks every possible configuration and validates the exact requirement from the problem statement.

A naive implementation could recompute shortest paths inefficiently for every subset using repeated Dijkstra runs or BFS-like traversals. Since there are up to `1024` subsets and up to `1000` roads, careless implementations can become unnecessarily slow.

### Key Insight

The critical observation is that `n` is extremely small.

With only at most 10 nodes, Floyd-Warshall becomes very attractive. Floyd-Warshall computes all-pairs shortest paths in `O(n^3)`, which is only `1000` operations when `n = 10`.

Since there are only `2^n` subsets, we can:

1. Enumerate every active subset using bitmasks.
2. Build a distance matrix for that subset.
3. Run Floyd-Warshall restricted to active nodes.
4. Check whether all pairwise distances are within the limit.

The total complexity becomes manageable because:

- At most `1024` subsets exist.
- Each subset costs `O(n^3)`.

That gives roughly one million operations overall, which is easily fast enough.

## Algorithm Walkthrough

### Step 1: Enumerate Every Active Subset

We iterate through all bitmasks from `0` to `(1 << n) - 1`.

Each bitmask represents the set of active branches:

- Bit `i = 1` means branch `i` is active.
- Bit `i = 0` means branch `i` is closed.

This works because every subset of branches can be uniquely represented by a bitmask.

### Step 2: Initialize the Distance Matrix

For the current subset, create an `n x n` distance matrix.

Initialize:

- `dist[i][i] = 0`
- All other distances as infinity.

This matrix will store shortest distances among active nodes.

### Step 3: Add Valid Roads

For every road `[u, v, w]`:

- Only consider it if both `u` and `v` are active in the current subset.
- Update:

- `dist[u][v]`
- `dist[v][u]`

If multiple roads exist between the same pair, keep the minimum weight.

This step constructs the induced subgraph of active branches.

### Step 4: Run Floyd-Warshall

For every intermediate node `k`:

- Only process it if `k` is active.

Then relax all paths:

```
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

Again, only active nodes participate.

After this process, `dist[i][j]` contains the shortest possible distance between active branches `i` and `j`.

### Step 5: Validate the Subset

Check every pair of active nodes.

If any shortest distance exceeds `maxDistance`, this subset is invalid.

Otherwise, count it as valid.

### Step 6: Return the Final Count

After checking all subsets, return the number of valid configurations.

### Why It Works

The algorithm works because it exhaustively evaluates every possible set of active branches. For each subset, Floyd-Warshall correctly computes the shortest distances using only active nodes. The validation step directly checks the condition required by the problem statement: every pair of active branches must be reachable within `maxDistance`.

Since every subset is checked exactly once and shortest paths are computed correctly, the final count is guaranteed to be correct.

## Python Solution

```python
from typing import List

class Solution:
    def numberOfSets(self, n: int, maxDistance: int, roads: List[List[int]]) -> int:
        INF = 10**15
        total_valid = 0

        for mask in range(1 << n):
            dist = [[INF] * n for _ in range(n)]

            for i in range(n):
                if (mask >> i) & 1:
                    dist[i][i] = 0

            for u, v, w in roads:
                if ((mask >> u) & 1) and ((mask >> v) & 1):
                    if w < dist[u][v]:
                        dist[u][v] = w
                        dist[v][u] = w

            for k in range(n):
                if not ((mask >> k) & 1):
                    continue

                for i in range(n):
                    if not ((mask >> i) & 1):
                        continue

                    for j in range(n):
                        if not ((mask >> j) & 1):
                            continue

                        new_distance = dist[i][k] + dist[k][j]

                        if new_distance < dist[i][j]:
                            dist[i][j] = new_distance

            valid = True

            for i in range(n):
                if not ((mask >> i) & 1):
                    continue

                for j in range(n):
                    if not ((mask >> j) & 1):
                        continue

                    if dist[i][j] > maxDistance:
                        valid = False
                        break

                if not valid:
                    break

            if valid:
                total_valid += 1

        return total_valid
```

The implementation follows the algorithm exactly.

The outer loop enumerates every subset using a bitmask. For each subset, a fresh distance matrix is created and initialized with infinity. Active nodes receive self-distance `0`.

The roads are processed next. A road is added only if both endpoints remain active in the current subset. Since multiple roads may exist, the implementation stores the minimum edge weight.

The Floyd-Warshall section computes shortest paths among active nodes only. Skipping inactive nodes avoids unnecessary work and guarantees that closed branches are never used as intermediate vertices.

Finally, the validation loop checks whether every active pair satisfies the distance requirement. If so, the subset contributes to the answer.

## Go Solution

```go
package main

func numberOfSets(n int, maxDistance int, roads [][]int) int {
	const INF = int(1e15)

	totalValid := 0

	for mask := 0; mask < (1 << n); mask++ {
		dist := make([][]int, n)

		for i := 0; i < n; i++ {
			dist[i] = make([]int, n)

			for j := 0; j < n; j++ {
				dist[i][j] = INF
			}

			if (mask>>i)&1 == 1 {
				dist[i][i] = 0
			}
		}

		for _, road := range roads {
			u, v, w := road[0], road[1], road[2]

			if ((mask>>u)&1 == 1) && ((mask>>v)&1 == 1) {
				if w < dist[u][v] {
					dist[u][v] = w
					dist[v][u] = w
				}
			}
		}

		for k := 0; k < n; k++ {
			if ((mask >> k) & 1) == 0 {
				continue
			}

			for i := 0; i < n; i++ {
				if ((mask >> i) & 1) == 0 {
					continue
				}

				for j := 0; j < n; j++ {
					if ((mask >> j) & 1) == 0 {
						continue
					}

					if dist[i][k]+dist[k][j] < dist[i][j] {
						dist[i][j] = dist[i][k] + dist[k][j]
					}
				}
			}
		}

		valid := true

		for i := 0; i < n && valid; i++ {
			if ((mask >> i) & 1) == 0 {
				continue
			}

			for j := 0; j < n; j++ {
				if ((mask >> j) & 1) == 0 {
					continue
				}

				if dist[i][j] > maxDistance {
					valid = false
					break
				}
			}
		}

		if valid {
			totalValid++
		}
	}

	return totalValid
}
```

The Go implementation mirrors the Python logic closely.

Slices are used for the distance matrix. Since Go integers are fixed-width, a sufficiently large constant is used for infinity. The implementation carefully avoids using inactive nodes during Floyd-Warshall relaxation.

Bitmask operations are identical to the Python version, using shifts and bitwise AND operations.

## Edge Cases

### Empty Active Set

The subset where all branches are closed is always valid. There are no active branches, so no pairwise distances exist that could violate the constraint. A common bug is accidentally rejecting this subset because the graph appears disconnected. This implementation correctly accepts it because no invalid distances are found during validation.

### Multiple Roads Between the Same Nodes

The graph may contain several roads connecting the same pair of branches. Using the wrong edge can produce incorrect shortest paths. The implementation always stores the minimum direct edge weight before running Floyd-Warshall, ensuring the optimal edge is used.

### Closed Nodes Cannot Be Traversed

One subtle issue is that shortest paths must only use active branches. A naive implementation might compute shortest paths in the original graph and accidentally allow routes through closed branches. This solution avoids that problem by completely excluding inactive nodes from both graph construction and Floyd-Warshall transitions.

### Disconnected Active Subgraphs

Closing certain branches may disconnect the remaining graph. In that situation, some shortest distances remain infinity, which exceeds `maxDistance`. The implementation naturally handles this because unreachable pairs fail the validation check automatically.
