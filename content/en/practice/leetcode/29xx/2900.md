## Solution

## Problem Understanding

The problem gives us two arrays of the same length:

- `words`, where each element is a distinct string
- `groups`, where each element is either `0` or `1`

The goal is to select a subsequence of `words` such that adjacent selected words come from different group values. In other words, if two consecutive selected words are at indices `i` and `j`, then `groups[i] != groups[j]`.

A subsequence does not need to use contiguous elements. We are allowed to skip elements while preserving the original order.

The task is to return the longest possible alternating subsequence. If multiple valid answers exist, returning any one of them is acceptable.

For example, if:

```
words  = ["a", "b", "c", "d"]
groups = [1,   0,   1,   1]
```

then `["a", "b", "c"]` is valid because the corresponding group sequence is:

```
1 -> 0 -> 1
```

Every adjacent pair differs.

However, `["a", "c", "d"]` is invalid because:

```
1 -> 1 -> 1
```

Adjacent equal group values are not allowed.

The constraints are very small:

- `n <= 100`
- `groups[i]` is binary
- all words are distinct

Because the input size is tiny, even exponential solutions could technically pass in some environments. However, the problem is designed to reward recognizing a very simple greedy observation.

Several edge cases are important:

- All group values are identical, meaning the answer can only contain one word.
- The groups already alternate perfectly, meaning the entire array should be returned.
- Multiple valid longest subsequences may exist, so the implementation must not assume uniqueness.
- Since subsequences preserve order, we cannot rearrange elements.

## Approaches

### Brute Force Approach

A brute force solution would generate every possible subsequence of `words` and check whether it satisfies the alternating condition.

For each subsequence:

1. Extract the corresponding group values.
2. Verify that every adjacent pair differs.
3. Track the longest valid subsequence found.

Since there are `2^n` possible subsequences, this approach becomes exponentially expensive.

Even though `n <= 100` is relatively small, `2^100` is astronomically large, so brute force is not practical.

The brute force solution is correct because it checks every possible subsequence and therefore cannot miss the optimal answer. The issue is purely performance.

### Optimal Greedy Approach

The key observation is that we should always take a word whenever its group differs from the previously selected group's value.

Because the condition only depends on adjacent selected elements, there is never a disadvantage to selecting the earliest possible valid word.

Suppose the previous selected group is `0`. The next selected element must have group `1`. As soon as we encounter such an element, taking it immediately cannot reduce the future answer length because any later `1` could simply be replaced by this earlier one.

This means we can greedily build the answer in a single left-to-right pass.

We maintain:

- the current answer subsequence
- the group value of the last selected element

Whenever the current group differs from the previous selected group, we append the word.

This produces the maximum possible alternating subsequence.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n × n) | O(n) | Generates and validates every subsequence |
| Optimal | O(n) | O(n) | Greedily selects words whenever the group changes |

## Algorithm Walkthrough

1. Initialize an empty result list.

This list will store the words that form the alternating subsequence.
2. Add the first word to the result.

The first element can always be included because there is no previous element to compare against.
3. Store the group value of the last selected word.

This allows us to quickly determine whether the next candidate can be appended.
4. Iterate through the remaining indices from left to right.

For each index `i`, compare `groups[i]` with the last selected group.
5. If the current group differs from the previous selected group, append the current word.

This maintains the alternating property.
6. Update the last selected group after appending.

Future comparisons must use the newest selected element.
7. Continue until all elements are processed.
8. Return the resulting subsequence.

### Why it works

The greedy strategy works because the only requirement is that adjacent selected groups differ. Whenever we encounter a valid next group, selecting it immediately is always optimal.

Choosing an earlier valid element never reduces future possibilities because subsequences only require order preservation. Any future alternating continuation that works after a later valid element will also work after the earlier one.

Therefore, greedily taking every valid transition produces the longest possible alternating subsequence.

## Python Solution

```python
from typing import List

class Solution:
    def getLongestSubsequence(self, words: List[str], groups: List[int]) -> List[str]:
        result = [words[0]]
        last_group = groups[0]

        for i in range(1, len(words)):
            if groups[i] != last_group:
                result.append(words[i])
                last_group = groups[i]

        return result
```

The implementation begins by placing the first word into the answer because any valid subsequence can always include one element.

The variable `last_group` tracks the group value associated with the most recently selected word.

We then scan through the arrays from left to right. Whenever the current group differs from `last_group`, the current word can safely extend the alternating subsequence. We append it and update `last_group`.

Since every element is processed once and each operation is constant time, the algorithm runs efficiently in linear time.

## Go Solution

```go
func getLongestSubsequence(words []string, groups []int) []string {
    result := []string{words[0]}
    lastGroup := groups[0]

    for i := 1; i < len(words); i++ {
        if groups[i] != lastGroup {
            result = append(result, words[i])
            lastGroup = groups[i]
        }
    }

    return result
}
```

The Go implementation follows exactly the same logic as the Python version.

The main difference is slice handling. In Go, `result` is initialized as a slice containing the first word, and new elements are added using `append`.

No special integer overflow handling is needed because the constraints are extremely small.

## Worked Examples

### Example 1

```
words  = ["e", "a", "b"]
groups = [0,   0,   1]
```

We initialize:

```
result = ["e"]
last_group = 0
```

Now iterate through the remaining elements.

| i | words[i] | groups[i] | last_group | Action | result |
| --- | --- | --- | --- | --- | --- |
| 1 | "a" | 0 | 0 | Skip, same group | ["e"] |
| 2 | "b" | 1 | 0 | Append | ["e", "b"] |

Final answer:

```
["e", "b"]
```

### Example 2

```
words  = ["a", "b", "c", "d"]
groups = [1,   0,   1,   1]
```

Initial state:

```
result = ["a"]
last_group = 1
```

Process remaining elements.

| i | words[i] | groups[i] | last_group | Action | result |
| --- | --- | --- | --- | --- | --- |
| 1 | "b" | 0 | 1 | Append | ["a", "b"] |
| 2 | "c" | 1 | 0 | Append | ["a", "b", "c"] |
| 3 | "d" | 1 | 1 | Skip | ["a", "b", "c"] |

Final answer:

```
["a", "b", "c"]
```

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element is visited exactly once |
| Space | O(n) | The output subsequence may contain all words |

The algorithm performs a single pass through the input arrays. Each iteration only performs a constant amount of work, consisting of one comparison and possibly one append operation.

The result array stores the selected subsequence. In the best alternating case, all words are included, so the space complexity is linear.

## Edge Cases

One important edge case occurs when all group values are identical. For example:

```
groups = [0, 0, 0, 0]
```

In this situation, no two adjacent selected elements can coexist in the subsequence because every pair violates the alternating requirement. The correct answer therefore contains exactly one word. The implementation handles this naturally because it only appends when the group changes.

Another important case is when the sequence already alternates perfectly:

```
groups = [0, 1, 0, 1, 0]
```

Here, the optimal answer is the entire `words` array. Since every adjacent group differs, the algorithm appends every word during iteration.

A subtle edge case involves long runs of repeated group values:

```
groups = [1, 1, 1, 0, 0, 1]
```

A naive implementation might incorrectly attempt to keep multiple equal-group elements. The greedy solution avoids this by always comparing against the last selected group rather than the immediately previous array element. This guarantees that the resulting subsequence remains valid.

Another edge case is the smallest possible input size:

```
n = 1
```

The algorithm initializes the answer with the first word and never enters the loop, correctly returning a single-element subsequence.
