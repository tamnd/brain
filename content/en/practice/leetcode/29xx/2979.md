## Solution

## Problem Understanding

This problem is asking us to determine the largest price of an item that **cannot** be bought using an unlimited supply of coins with denominations `primeOne` and `primeTwo`. Both coins are **distinct primes**, which simplifies some number-theoretic reasoning. Essentially, we are asked to find the **Frobenius number** for two prime numbers: the largest integer that cannot be expressed as a non-negative linear combination of `primeOne` and `primeTwo`.

Formally, for two coprime integers `a` and `b` (which is guaranteed since they are distinct primes), the largest number that **cannot** be written as `x*a + y*b` for non-negative integers `x` and `y` is given by:

```
F = a*b - a - b
```

The input represents two distinct prime integers in the range `[2, 10^4)`. Because the product `primeOne * primeTwo < 10^5`, the numbers are small enough that this formula is valid without integer overflow concerns. The output is a single integer: the largest unrepresentable price.

The key edge cases are when the primes are very small (like 2 and 3) or when one prime is much smaller than the other (like 2 and 97). The formula works in all these cases because it is derived from the theory of the coin problem for two coprime integers.

## Approaches

The naive approach is to enumerate all possible combinations of `primeOne` and `primeTwo` until we find the largest number that cannot be represented. We could try all integers from 1 to `primeOne * primeTwo` and check for a combination that sums to it. This works because, for two coprime integers, all numbers beyond `primeOne * primeTwo - primeOne - primeTwo` can be expressed as a combination. However, this enumeration is unnecessary.

The optimal approach leverages the **Frobenius number formula**: for two coprime integers `a` and `b`, the largest number that cannot be represented as `x*a + y*b` is `a*b - a - b`. Since both numbers are primes, they are automatically coprime, and we can return the result directly.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(a*b) | O(1) | Enumerates all combinations to find the largest unrepresentable number |
| Optimal | O(1) | O(1) | Uses the Frobenius formula `a*b - a - b` for two coprime integers |

## Algorithm Walkthrough

1. Receive the input primes `primeOne` and `primeTwo`.
2. Recognize that these numbers are coprime because they are distinct primes.
3. Apply the Frobenius number formula for two coprime integers: `F = primeOne * primeTwo - primeOne - primeTwo`.
4. Return `F` as the result.

**Why it works:** For any two coprime positive integers, the Frobenius number formula guarantees that all integers greater than `F` can be represented as a linear combination of the two numbers, and `F` itself is the largest integer that cannot be represented. Since the primes are distinct, the coprime condition holds, ensuring correctness.

## Python Solution

```python
class Solution:
    def mostExpensiveItem(self, primeOne: int, primeTwo: int) -> int:
        # Since primeOne and primeTwo are distinct primes, they are coprime
        # Frobenius number formula for two coprime integers a and b: a*b - a - b
        return primeOne * primeTwo - primeOne - primeTwo
```

The Python implementation is straightforward. We calculate the product of the two primes, subtract each prime, and return the result. This directly uses the mathematical property of coprime integers, which avoids any loops or additional data structures.

2. Result: 3.

1. Apply formula: `5*7 - 5 - 7 = 35 - 12 = 23`.
2. Result: 23.

## Edge Cases

The first edge case is when the primes are consecutive, like `2` and `3`. This could cause confusion if one assumes only large gaps are needed for unrepresentable numbers, but the formula still works correctly. The second is when the primes are very close to the upper limit, such as `97` and `101`; this tests that the calculation does not overflow or miscompute. The third edge case is ensuring that the order of primes does not affect the outcome; the multiplication is commutative, and the formula is symmetric, so swapping inputs still returns the same result. All these cases are handled naturally by the Frobenius formula.
