# LeetCode 2914, Minimum Number of Changes to Make Binary String Beautiful

## Problem Understanding

The problem gives us a binary string `s` consisting only of the characters `'0'` and `'1'`. The length of the string is guaranteed to be even.

A string is considered beautiful if we can divide it into one or more substrings where:

- Every substring has even length.
- Every substring contains only one type of character, either all `'0'` or all `'1'`.

We are allowed to change any character in the string. Each change converts one character into either `'0'` or `'1'`. The goal is to determine the minimum number of changes needed so that the final string becomes beautiful.

The key detail is understanding what kinds of partitions are allowed. Since every valid substring must have even length and contain identical characters, the simplest valid building block is a pair of equal characters:

- `"00"`
- `"11"`

Any longer valid substring can also be viewed as multiple valid pairs grouped together. For example:

- `"0000"` can be partitioned as `"00|00"`
- `"111111"` can be partitioned as `"11|11|11"`

This observation simplifies the problem significantly.

The constraint `s.length <= 10^5` tells us that the solution must be efficient. Quadratic or exponential approaches would be far too slow. We need a linear-time solution.

There are several important edge cases to consider:

- A string that is already beautiful, such as `"0000"` or `"1111"`.
- A string where every adjacent pair differs, such as `"1010"`.
- The smallest possible input length, which is `2`.
- Alternating patterns where many changes may be required.
- Large inputs near `10^5`, where efficiency matters.

The problem guarantees that the string length is always even, so we never need to handle incomplete pairs.

## Approaches

### Brute Force Approach

A brute-force strategy would try every possible way to partition the string into even-length substrings. For each partition, we would calculate how many character changes are required to make every substring consist of identical characters.

For example, given `"1001"`, we could consider:

- `"10|01"`
- `"1001"`

For each substring, we would compute the minimum flips needed to make all characters equal.

This approach is correct because it explores every valid partition and chooses the minimum total cost. However, it is computationally expensive because the number of possible partitions grows exponentially with the string length.

With `n` up to `10^5`, this approach is completely impractical.

### Optimal Approach

The key insight is that we never actually need to consider larger substrings explicitly.

Since every valid substring must have even length and uniform characters, we can always split the string into pairs of length `2`. Therefore, every pair of adjacent characters must eventually become either:

- `"00"`
- `"11"`

For each pair:

- If both characters are already equal, no changes are needed.
- If the characters differ, exactly one change is needed.

This means the answer is simply the number of adjacent pairs that contain different characters.

For example:

- `"10"` requires `1` change.
- `"00"` requires `0` changes.
- `"01"` requires `1` change.

We process the string two characters at a time and count mismatched pairs.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Tries all valid partitions |
| Optimal | O(n) | O(1) | Counts mismatched adjacent pairs |

## Algorithm Walkthrough

1. Initialize a variable `changes` to `0`.
2. Traverse the string two characters at a time using indices `0, 2, 4, ...`.
3. For each pair `(s[i], s[i + 1])`, check whether the two characters are equal.
4. If the characters are different, increment `changes` by `1`.
5. Continue until all pairs have been processed.
6. Return `changes` as the final answer.

### Why it works

Every beautiful string can be partitioned into even-length uniform substrings. Any such substring can further be divided into pairs of equal characters. Therefore, ensuring that every adjacent pair contains identical characters is both necessary and sufficient.

Each mismatched pair requires at least one modification, and one modification is always enough to fix it. Since pairs are independent of each other, summing the required changes for each pair gives the global minimum.

## Python Solution

```python
class Solution:
    def minChanges(self, s: str) -> int:
        changes: int = 0

        for i in range(0, len(s), 2):
            if s[i] != s[i + 1]:
                changes += 1

        return changes
```

The implementation follows the exact logic from the algorithm walkthrough.

The variable `changes` stores the running total of required modifications.

The loop increments by `2` because we process the string pair by pair. Since the input length is guaranteed to be even, accessing `s[i + 1]` is always safe.

For each pair:

- If the characters match, the pair is already valid.
- If they differ, one character must be flipped, so we increment the answer.

At the end, the method returns the total number of mismatched pairs.

## Go Solution

```go
func minChanges(s string) int {
    changes := 0

    for i := 0; i < len(s); i += 2 {
        if s[i] != s[i+1] {
            changes++
        }
    }

    return changes
}
```

The Go implementation is almost identical to the Python version.

Strings in Go are indexed using bytes, which works perfectly here because the input contains only ASCII characters `'0'` and `'1'`.

No extra memory allocation is needed, and the algorithm processes the string in a single pass.

## Edge Cases

### Minimum Length Input

The smallest allowed string length is `2`. A naive implementation might accidentally assume larger substrings exist or incorrectly handle indexing.

For example:

```
"10"
```

The implementation safely handles this because the loop processes exactly one pair and stops.

### Already Beautiful Strings

Strings like `"0000"` or `"11110000"` already satisfy the conditions without any changes.

A buggy implementation might overcomplicate partitioning and accidentally count unnecessary modifications. Our approach correctly identifies that every adjacent pair already contains identical characters, so the answer remains `0`.

### Completely Alternating Strings

Inputs such as:

```
"01010101"
```

can be tricky because every pair is invalid.

Each pair requires exactly one change:

- `"01"` -> one change
- `"01"` -> one change
- `"01"` -> one change
- `"01"` -> one change

The algorithm correctly counts all four mismatched pairs independently.

### Large Inputs

The maximum input size is `10^5`. Any solution involving dynamic programming over partitions or recursive backtracking would be too slow.

This implementation uses a single linear pass with constant extra memory, so it scales efficiently even for the largest allowed inputs.
