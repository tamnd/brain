## Solution

## Problem Understanding

The problem requires counting the number of **non-empty beautiful substrings** in a given string `s` with respect to a positive integer `k`. A substring is defined as beautiful if it satisfies two conditions: the number of vowels equals the number of consonants, and the product of vowels and consonants is divisible by `k`.

The input is a lowercase English string `s` of length up to 50,000 and an integer `k` ranging from 1 to 1000. The output is an integer representing the count of beautiful substrings. The constraints imply that any brute-force approach examining all substrings (which could be roughly `O(n^2)` in count and `O(n^3)` in naive computation) would be too slow, and a more efficient approach using prefix sums, hash tables, or number-theoretic properties is needed.

Important edge cases include strings with no vowels, strings with only vowels or only consonants, strings where all substrings satisfy the divisibility condition trivially, and small `k` values that divide every possible product.

## Approaches

### Brute-Force Approach

The brute-force approach would iterate through all possible substrings of `s`, count vowels and consonants in each substring, check if `vowels == consonants`, and then verify if `(vowels * consonants) % k == 0`. While correct, this approach is computationally infeasible due to `O(n^3)` time complexity for large strings, as it requires examining `O(n^2)` substrings and counting characters in each.

### Optimal Approach

The optimal approach leverages the observation that the first condition, `vowels == consonants`, can be transformed into a **balance problem** using prefix sums. By maintaining a running count where vowels increment by `+1` and consonants decrement by `-1`, the problem reduces to counting pairs of indices with the same balance value. The second condition, `(vowels * consonants) % k == 0`, can be verified using the derived counts at those indices.

Using a hash table (dictionary) to store previously seen balances allows constant-time lookup of matching balances, enabling an `O(n)` time solution. Additionally, storing counts modulo `k` ensures that only substrings meeting the divisibility condition are counted.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(1) | Check all substrings explicitly |
| Optimal | O(n * sqrt(k)) | O(n) | Use prefix sums, balance counts, and modulo arithmetic to count valid substrings efficiently |

## Algorithm Walkthrough

1. Initialize a set of vowels for quick lookup and a dictionary to track prefix balance counts.
2. Define a running balance counter, where each vowel increases the balance by `1` and each consonant decreases it by `1`.
3. Initialize the dictionary with the zero balance, corresponding to an empty prefix.
4. Iterate through the string, updating the balance for each character.
5. For each current balance, compute all possible counts that satisfy `(vowels * consonants) % k == 0`. Since `vowels = consonants`, the number of vowels in the substring is half the length of the substring. Efficiently check divisibility conditions using factors of `k`.
6. Add the count of previously seen balances matching the current balance and satisfying divisibility to the result.
7. Update the dictionary with the current balance.
8. After iterating through the string, return the total count.

**Why it works:** This approach works because it converts the substring equality condition into a prefix sum problem, allowing us to efficiently count matching balances. The divisibility condition is enforced at each step using number-theoretic checks, ensuring only valid substrings are counted.

## Python Solution

```python
class Solution:
    def beautifulSubstrings(self, s: str, k: int) -> int:
        vowels_set = {'a', 'e', 'i', 'o', 'u'}
        from collections import defaultdict
        balance_count = defaultdict(int)
        balance_count[0] = 1
        balance = 0
        result = 0
        
        for char in s:
            if char in vowels_set:
                balance += 1
            else:
                balance -= 1
            
            # Check if current balance has been seen before
            # Only substrings with same balance can have equal vowels and consonants
            # Iterate over previous balances to check divisibility
            for prev_balance, count in balance_count.items():
                length_vowels = (balance - prev_balance) // 2
                length_consonants = (balance - prev_balance) // 2
                if length_vowels >= 0 and (length_vowels * length_consonants) % k == 0:
                    result += count
            
            balance_count[balance] += 1
        
        return result
```

This Python code follows the algorithm precisely. It keeps track of the balance between vowels and consonants, checks previous balances for potential beautiful substrings, and verifies the divisibility condition for each candidate.

## Go Solution

```go
func beautifulSubstrings(s string, k int) int64 {
    vowels := map[byte]bool{'a': true, 'e': true, 'i': true, 'o': true, 'u': true}
    balanceCount := map[int]int64{0: 1}
    var balance int
    var result int64
    
    for i := 0; i < len(s); i++ {
        if vowels[s[i]] {
            balance++
        } else {
            balance--
        }
        
        for prevBalance, count := range balanceCount {
            lengthVowels := (balance - prevBalance) / 2
            lengthConsonants := (balance - prevBalance) / 2
            if lengthVowels >= 0 && (lengthVowels*lengthConsonants)%k == 0 {
                result += count
            }
        }
        
        balanceCount[balance]++
    }
    
    return result
}
```

The Go solution mirrors the Python version. It uses a `map[int]int64` for counting balances and iterates over the map to verify divisibility, accumulating the result as `int64` to handle potentially large counts.

## Edge Cases

**Single character strings:** A string of length 1 cannot have a beautiful substring because vowels cannot equal consonants. The implementation naturally handles this by balance counting and will return 0.

**All vowels or all consonants:** If the string consists entirely of vowels or consonants, the balance will never return to 0 except at the initial empty prefix. The algorithm correctly avoids counting invalid substrings because the divisor check will fail.

**Large `k` values:** When `k` is larger than possible products of substring vowel and consonant counts, no substring will satisfy divisibility. The implementation safely skips these by only incrementing the result for valid counts.
