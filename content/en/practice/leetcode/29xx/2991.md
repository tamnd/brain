## Solution

## Problem Understanding

The problem asks us to generate a leaderboard of wineries **grouped by country**, based on the **total points** accumulated by each winery. The input table `Wineries` contains individual wine ratings (`points`) along with the winery name and country. A single winery can have multiple entries, and we must sum these points to get its total score.

The expected output is a table that lists the **top three wineries per country**. If a country has fewer than three wineries, we fill the missing ranks with the placeholders `'No second winery'` or `'No third winery'`. Additionally, if multiple wineries have the same total points, we break ties by alphabetical order of the winery name. Finally, the result should be ordered by `country` in ascending order.

Key points to note are that a winery can appear multiple times in the input, countries may have fewer than three wineries, and ties must be resolved consistently using the winery name. The input size is not explicitly bounded, but we should assume standard SQL table sizes up to tens of thousands of rows, meaning performance matters but extreme optimizations are not strictly necessary.

Important edge cases include:

- A country with only one or two wineries.
- Wineries with identical total points.
- Wineries with special characters or spaces in the name.

## Approaches

The brute-force approach would involve computing the total points for each winery, then iterating country by country to rank wineries. This could involve multiple nested queries or repeated joins, which is inefficient for larger tables.

The optimal approach leverages **window functions** in SQL, specifically `SUM()` to aggregate total points per winery and `ROW_NUMBER()` or `RANK()` to assign rankings within each country. Using `ROW_NUMBER()` allows us to rank wineries in order of descending total points and alphabetical order for ties. Finally, a **pivoting approach** converts rows into columns for the top three wineries, filling missing positions with the appropriate placeholders.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Aggregate points then manually iterate to find top 3 per country |
| Optimal | O(n log n) | O(n) | Uses SUM aggregation, ROW_NUMBER for ranking, and pivoting for output |

## Algorithm Walkthrough

1. **Aggregate total points per winery**: Use `GROUP BY country, winery` with `SUM(points)` to compute each winery's total score.
2. **Rank wineries within each country**: Apply `ROW_NUMBER()` ordered by descending total points and ascending winery name. This assigns ranks 1, 2, 3, etc., for each country.
3. **Pivot ranks into columns**: Convert rows into columns for `top_winery`, `second_winery`, and `third_winery` using conditional aggregation. Concatenate the winery name with its total points in the format `WineryName (Points)`.
4. **Fill missing values**: If a rank does not exist for a country, replace it with the appropriate placeholder `'No second winery'` or `'No third winery'`.
5. **Order by country**: Finally, order the output by the `country` column in ascending order.

**Why it works**: By aggregating first, we ensure that repeated entries for a winery are summed correctly. Window functions guarantee consistent ranking per country, while the pivot ensures the top three ranks are presented as columns with placeholders for missing values.

## Python Solution

```python
import sqlite3

def top_three_wineries(conn: sqlite3.Connection):
    query = """
    WITH winery_totals AS (
        SELECT
            country,
            winery,
            SUM(points) AS total_points
        FROM Wineries
        GROUP BY country, winery
    ),
    ranked_wineries AS (
        SELECT
            country,
            winery,
            total_points,
            ROW_NUMBER() OVER (
                PARTITION BY country
                ORDER BY total_points DESC, winery ASC
            ) AS rn
        FROM winery_totals
    )
    SELECT
        country,
        COALESCE(MAX(CASE WHEN rn = 1 THEN winery || ' (' || total_points || ')' END), 'No top winery') AS top_winery,
        COALESCE(MAX(CASE WHEN rn = 2 THEN winery || ' (' || total_points || ')' END), 'No second winery') AS second_winery,
        COALESCE(MAX(CASE WHEN rn = 3 THEN winery || ' (' || total_points || ')' END), 'No third winery') AS third_winery
    FROM ranked_wineries
    GROUP BY country
    ORDER BY country ASC;
    """
    return conn.execute(query).fetchall()
```

The Python implementation simply executes the SQL query using a database connection. The `WITH` clause computes total points and ranks the wineries. Conditional aggregation converts rows into columns, and `COALESCE` ensures missing ranks are filled with placeholders.

## Go Solution

```go
package main

import (
    "database/sql"
    _ "github.com/mattn/go-sqlite3"
    "fmt"

func TopThreeWineries(db *sql.DB) ([][4]string, error) {
    query := `
    WITH winery_totals AS (
        SELECT
            country,
            winery,
            SUM(points) AS total_points
        FROM Wineries
        GROUP BY country, winery
    ),
    ranked_wineries AS (
        SELECT
            country,
            winery,
            total_points,
            ROW_NUMBER() OVER (
                PARTITION BY country
                ORDER BY total_points DESC, winery ASC
            ) AS rn
        FROM winery_totals
    )
    SELECT
        country,
        COALESCE(MAX(CASE WHEN rn = 1 THEN winery || ' (' || total_points || ')' END), 'No top winery') AS top_winery,
        COALESCE(MAX(CASE WHEN rn = 2 THEN winery || ' (' || total_points || ')' END), 'No second winery') AS second_winery,
        COALESCE(MAX(CASE WHEN rn = 3 THEN winery || ' (' || total_points || ')' END), 'No third winery') AS third_winery
    FROM ranked_wineries
    GROUP BY country
    ORDER BY country ASC;
    `

    rows, err := db.Query(query)
    if err != nil {
        return nil, err
    }
    defer rows.Close()

    var result [][4]string
    for rows.Next() {
        var country, top, second, third string
        if err := rows.Scan(&country, &top, &second, &third); err != nil {
            return nil, err
        }
        result = append(result, [4]string{country, top, second, third})
    }
    return result, nil
}
```

In Go, we execute the same SQL query. Rows are scanned into a slice of string arrays. The main difference is Go's need to explicitly handle errors and manage slices instead of returning a plain list.
