## Solution

## Problem Understanding

The problem asks us to identify **symmetric coordinates** from a table of integer points `(X, Y)`. Two coordinates `(X1, Y1)` and `(X2, Y2)` are symmetric if swapping their components results in the other coordinate, i.e., `X1 == Y2` and `X2 == Y1`. The task is to return all unique symmetric coordinates, but only those with `X <= Y`, to avoid duplicates in mirrored form. The output must be sorted by `X` and then `Y` in ascending order.

The input is a table `Coordinates` with two integer columns `X` and `Y`. The table may contain duplicates, which means the same coordinate may appear multiple times. The output should eliminate duplicates and only list valid symmetric coordinates.

Important edge cases include:

1. Coordinates that are identical `(X == Y)` because they are trivially symmetric.
2. Coordinates that appear multiple times.
3. Coordinates whose symmetric partner appears in reverse order.

The problem guarantees that all coordinates are integers and that the table may contain duplicates, but it does not guarantee any order in the input.

## Approaches

### Brute-Force Approach

The brute-force approach is to compare every pair of coordinates to see if they satisfy the symmetric condition. Specifically, for each coordinate `(X1, Y1)`, we check if there exists another coordinate `(X2, Y2)` such that `X1 == Y2` and `X2 == Y1`. If so, we add the coordinate `(min(X1, X2), max(Y1, Y2))` to a set to avoid duplicates and enforce the `X <= Y` rule.

While this approach guarantees correctness, it is inefficient. If there are `n` coordinates, this approach requires checking every pair, leading to `O(n^2)` time complexity. For large datasets, this is too slow.

### Optimal Approach

The key observation is that symmetric coordinates can be identified using **self-joins** or by checking for the existence of a "flipped" coordinate efficiently. Using a hash-based approach, we can:

1. Convert all coordinates into a set of tuples `(X, Y)` for `O(1)` existence checks.
2. For each coordinate `(X, Y)`, check if `(Y, X)` exists in the set.
3. Include `(X, Y)` only if `X <= Y` to enforce uniqueness.
4. Sort the resulting set by `X` and `Y` for the final output.

This approach avoids the nested iteration of brute force and reduces time complexity to `O(n log n)` due to the final sort, with `O(n)` space for the set.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Compare all pairs of coordinates |
| Optimal | O(n log n) | O(n) | Use a set for quick symmetric lookup, then sort results |

## Algorithm Walkthrough

1. Retrieve all coordinates from the table and convert them into a set to allow `O(1)` lookups.
2. Initialize an empty set to store the symmetric coordinates that satisfy `X <= Y`.
3. Iterate over each coordinate `(X, Y)` in the set.
4. For each `(X, Y)`, check if `(Y, X)` exists in the set.
5. If `(Y, X)` exists and `X <= Y`, add `(X, Y)` to the result set.
6. After checking all coordinates, convert the result set into a list and sort it first by `X`, then by `Y`.
7. Return the sorted list as the final output table.

**Why it works**: The set ensures that symmetric partners can be checked in constant time. Restricting `X <= Y` guarantees that mirrored duplicates are avoided. Sorting ensures the required output order.

## Python Solution

```python
import pandas as pd

class Solution:
    def symmetricCoordinates(self, coordinates: pd.DataFrame) -> pd.DataFrame:
        coords_set = set(tuple(row) for row in coordinates.values)
        result_set = set()

        for x, y in coords_set:
            if (y, x) in coords_set and x <= y:
                result_set.add((x, y))

        result_list = sorted(result_set)
        return pd.DataFrame(result_list, columns=['x', 'y'])
```

This implementation converts the DataFrame into a set of tuples for O(1) lookups. It iterates over the set to identify symmetric pairs and uses the `X <= Y` rule to avoid duplicates. Finally, it sorts the results and returns them as a DataFrame with the required columns.

## Go Solution

```go
package main

import (
    "sort"
)

type Coordinate struct {
    X int
    Y int
}

func symmetricCoordinates(coordinates []Coordinate) []Coordinate {
    coordMap := make(map[[2]int]struct{})
    resultMap := make(map[[2]int]struct{})

    for _, c := range coordinates {
        key := [2]int{c.X, c.Y}
        coordMap[key] = struct{}{}
    }

    for key := range coordMap {
        flipped := [2]int{key[1], key[0]}
        if _, exists := coordMap[flipped]; exists && key[0] <= key[1] {
            resultMap[key] = struct{}{}
        }
    }

    result := make([]Coordinate, 0, len(resultMap))
    for key := range resultMap {
        result = append(result, Coordinate{X: key[0], Y: key[1]})
    }

    sort.Slice(result, func(i, j int) bool {
        if result[i].X == result[j].X {
            return result[i].Y < result[j].Y
        }
        return result[i].X < result[j].X
    })

    return result
}
```

In Go, we use maps to replicate set behavior, ensuring constant-time checks for symmetric partners. The result is collected in a separate map, then converted to a slice and sorted by `X` and `Y`.

## Edge Cases

1. **Empty table**: The algorithm correctly returns an empty DataFrame since no coordinates exist. This avoids errors related to iterating over an empty set.
2. **Coordinates symmetric to themselves**: Coordinates like `(5, 5)` are symmetric by definition. The `X <= Y` rule ensures they are included exactly once, even if they appear multiple times in the table.
3. **Multiple mirrored coordinates**: For a pair `(X, Y)` and `(Y, X)`, the algorithm ensures only the coordinate satisfying `X <= Y` is included. This avoids duplicate mirrored entries
