## Solution

## Problem Understanding

This problem asks us to count the number of contiguous **blocks of equal numbers** in a very large array, `nums`, where all occurrences of a value are adjacent. Essentially, the array can be thought of as being **run-length encoded** implicitly: each distinct number forms a continuous block, and the array is a sequence of these blocks.

The array is too large to store in memory (`nums.length` can be up to 10^15), so we cannot iterate over it in a standard way. Instead, we are given a `BigArray` interface that provides two operations: `at(index)` to access an element at a given index and `size()` to get the array length. The goal is to efficiently count how many contiguous segments of equal numbers exist in `nums`.

The expected output is an integer representing the total number of these maximal blocks. A naive approach of checking each element in sequence would be too slow due to the array size, so an **efficient solution must use binary search** to jump across blocks.

Important constraints and observations:

- All occurrences of a value are adjacent, so any transition from one number to another signals the start of a new block.
- The array may be extremely large, so **O(n) iteration is infeasible**.
- `nums[i]` is within `1 <= nums[i] <= 10^9`, but this is mostly relevant for equality checks.
- Edge cases include arrays where all numbers are the same, all numbers are distinct, or only one element exists.

## Approaches

### Brute Force Approach

A brute-force approach would iterate from index 0 to `nums.size() - 1`, checking if the current number is different from the previous number. Every time a difference is detected, a new block counter is incremented. This is simple and correct but **O(n)** in time, which is impractical for `n` up to 10^15.

### Optimal Approach

The key insight is that **blocks of equal numbers are contiguous**, so we can use **binary search to skip entire blocks** rather than checking each element. For each block starting at index `i`, we can perform a binary search to find the last index `j` of the same value. We then start the next search at `j + 1`. By doing this iteratively, we can count the number of blocks in `O(log n)` per block, making it feasible for extremely large arrays.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Iterate over all elements and count changes. Too slow for large n. |
| Optimal | O(b log n) | O(1) | Use binary search to jump across each block of equal numbers. `b` is the number of blocks. |

## Algorithm Walkthrough

1. Initialize `count` to 0 and `index` to 0. `count` will track the number of blocks.
2. While `index` is less than `nums.size()`, do the following:

1. Retrieve the value at `nums.at(index)` and call it `current_val`.
2. Perform a binary search between `index` and `nums.size() - 1` to find the last index `last_index` where the value equals `current_val`.

- Set `left = index` and `right = nums.size() - 1`.
- While `left < right`, calculate `mid = (left + right + 1) // 2`.
- If `nums.at(mid) == current_val`, move `left` to `mid`.
- Otherwise, move `right` to `mid - 1`.
- After the loop, `left` is the last index of the current block.
3. Increment `count` by 1 for this block.
4. Update `index = last_index + 1` to move to the next block.
3. Return `count` after iterating through all blocks.

**Why it works**: The invariant is that `index` always points to the start of a new block. Binary search finds the end of the current block efficiently without visiting every element. This guarantees that every block is counted exactly once.

## Python Solution

```python
# Definition for BigArray.
# class BigArray:
#     def at(self, index: int) -> int:
#         pass
#     def size(self) -> int:
#         pass

class Solution(object):
    def countBlocks(self, nums: 'BigArray') -> int:
        n = nums.size()
        count = 0
        index = 0
        
        while index < n:
            current_val = nums.at(index)
            left, right = index, n - 1
            
            # Binary search to find the last index of current_val
            while left < right:
                mid = (left + right + 1) // 2
                if nums.at(mid) == current_val:
                    left = mid
                else:
                    right = mid - 1
            
            count += 1
            index = left + 1  # Move to next block
        
        return count
```

**Explanation**: We start from the first element and use binary search to find the last occurrence of each value. Once the block is identified, `index` is updated to the next block's start. The loop repeats until the end of the array.

## Go Solution

```go
/**
 * Definition for BigArray.
 * type BigArray interface {
 *     At(int64) int
 *     Size() int64
 * }
 */
func countBlocks(nums BigArray) int {
    n := nums.Size()
    count := 0
    var index int64 = 0
    
    for index < n {
        currentVal := nums.At(index)
        left, right := index, n-1
        
        for left < right {
            mid := (left + right + 1) / 2
            if nums.At(mid) == currentVal {
                left = mid
            } else {
                right = mid - 1
            }
        }
        
        count++
        index = left + 1
    }
    
    return count
}
```

**Go-specific notes**: Go uses `int64` for indices because `nums.Size()` can be very large. The binary search logic is identical to Python.

## Worked Examples

### Example 1: `[3,3,3,3,3]`

| index | current_val | left | right | count |
| --- | --- | --- | --- | --- |
| 0 | 3 | 0 | 4 | 0 -> 1 |

All elements are the same. Binary search finds last index 4. `count = 1`.

### Example 2: `[1,1,1,3,9,9,9,2,10,10]`

| index | current_val | left | right | count |
| --- | --- | --- | --- | --- |
| 0 | 1 | 0 | 9 | 0 -> 1 |
| 3 | 3 | 3 | 9 | 1 -> 2 |
| 4 | 9 | 4 | 9 | 2 -> 3 |
| 7 | 2 | 7 | 9 | 3 -> 4 |
| 8 | 10 | 8 | 9 | 4 -> 5 |

Binary search finds end of each block; total `count = 5`.

### Example 3: `[1,2,3,4,5,6,7]`

Binary search will find each block is size 1, so `count = 7`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(b log n) | `b` is the number of blocks. Binary search takes O(log n) per block. |
| Space | O(1) | Only a few integer variables are used. |

The complexity is feasible for arrays of size up to 10^15 because `b` is typically much smaller than `n`.
