## Solution

## Problem Understanding

The problem asks us to count all **incremovable subarrays** of a given array `nums`. An **incremovable subarray** is defined as a contiguous non-empty segment of the array which, if removed, makes the remaining array **strictly increasing**. A strictly increasing array means every element is strictly smaller than the next one.

For input, we are given a **0-indexed array of positive integers** with lengths ranging from 1 to 50 and values from 1 to 50. The output is a single integer representing the total number of subarrays that satisfy the incremovable property.

Key points to understand include that removing the subarray could leave elements on both sides of the array, and these elements must form a strictly increasing sequence. Single-element subarrays are valid candidates if their removal leads to a strictly increasing array. Edge cases involve arrays that are already strictly increasing, arrays with duplicates, and arrays with only decreasing elements.

Constraints are small (`n <= 50`), so an `O(n^3)` brute-force solution is feasible but not elegant. However, we must handle all contiguous subarrays correctly, including those at the edges and the full array itself.

## Approaches

The brute-force approach is straightforward: iterate over all possible subarrays, remove each one, and check whether the resulting array is strictly increasing. This works because the array size is small, but checking all subarrays and then verifying the strictly increasing property can be computationally heavy.

The key insight for a better solution is that we only need to check **the endpoints of the subarray and the internal order of the remaining array**. By precomputing prefix and suffix information about strictly increasing segments, we can efficiently verify whether removing a subarray preserves a strictly increasing sequence. This reduces repeated computation and ensures correctness while keeping time complexity reasonable.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n) | Check every subarray removal and validate strictly increasing |
| Optimal | O(n^2) | O(n) | Use prefix and suffix increasing checks to validate subarray removal |

## Algorithm Walkthrough

1. Compute a **prefix array** `inc_left` such that `inc_left[i]` is `True` if `nums[0:i+1]` is strictly increasing.
2. Compute a **suffix array** `inc_right` such that `inc_right[i]` is `True` if `nums[i:n]` is strictly increasing.
3. Iterate over all possible subarrays defined by start `i` and end `j`.
4. For each subarray, check if removing it results in a strictly increasing array:

- Check if the prefix before `i` (`inc_left[i-1]`) is strictly increasing if `i > 0`.
- Check if the suffix after `j` (`inc_right[j+1]`) is strictly increasing if `j < n-1`.
- If both exist, ensure that the last element before the subarray is less than the first element after the subarray.
5. Count all subarrays that satisfy these conditions.

**Why it works:** This algorithm works because it verifies all necessary conditions for the remaining array to be strictly increasing. The prefix and suffix arrays efficiently capture increasing sequences, and checking boundary elements ensures global order is maintained. Since all subarrays are considered, no valid cases are missed.

## Python Solution

```python
from typing import List

class Solution:
    def incremovableSubarrayCount(self, nums: List[int]) -> int:
        n = len(nums)
        if n == 1:
            return 1
        
        # Compute prefix strictly increasing
        inc_left = [True] * n
        for i in range(1, n):
            inc_left[i] = inc_left[i-1] and nums[i-1] < nums[i]
        
        # Compute suffix strictly increasing
        inc_right = [True] * n
        for i in range(n-2, -1, -1):
            inc_right[i] = inc_right[i+1] and nums[i] < nums[i+1]
        
        count = 0
        # Check all subarrays
        for i in range(n):
            for j in range(i, n):
                left_ok = (i == 0) or inc_left[i-1]
                right_ok = (j == n-1) or inc_right[j+1]
                boundary_ok = (i == 0 or j == n-1 or nums[i-1] < nums[j+1])
                if left_ok and right_ok and boundary_ok:
                    count += 1
        return count
```

The implementation first precomputes whether prefixes and suffixes of the array are strictly increasing. Then, it iterates over all subarrays, checking whether removing each one preserves a strictly increasing sequence using these precomputed values, including boundary conditions.

## Go Solution

```go
func incremovableSubarrayCount(nums []int) int {
    n := len(nums)
    if n == 1 {
        return 1
    }
    
    incLeft := make([]bool, n)
    for i := range incLeft {
        incLeft[i] = true
    }
    for i := 1; i < n; i++ {
        incLeft[i] = incLeft[i-1] && nums[i-1] < nums[i]
    }
    
    incRight := make([]bool, n)
    for i := range incRight {
        incRight[i] = true
    }
    for i := n-2; i >= 0; i-- {
        incRight[i] = incRight[i+1] && nums[i] < nums[i+1]
    }
    
    count := 0
    for i := 0; i < n; i++ {
        for j := i; j < n; j++ {
            leftOk := i == 0 || incLeft[i-1]
            rightOk := j == n-1 || incRight[j+1]
            boundaryOk := i == 0 || j == n-1 || nums[i-1] < nums[j+1]
            if leftOk && rightOk && boundaryOk {
                count++
            }
        }
    }
    return count
}
```

Go implementation is very similar to Python. We handle slices carefully and use boolean arrays to store prefix and suffix increasing sequences. Nil slices are not an issue since input size is guaranteed at least 1.
