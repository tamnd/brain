## Solution

## Problem Understanding

The problem asks us to find all **non-empty substrings** of a given string `s` that satisfy two conditions simultaneously. First, the number of vowels and consonants in the substring must be equal. Second, the product of the number of vowels and consonants must be divisible by a given integer `k`. Vowels are defined as the letters `'a'`, `'e'`, `'i'`, `'o'`, `'u'`, and all other lowercase letters are consonants.

The input is a string `s` of length at most 1000, consisting of lowercase English letters, and a positive integer `k` up to 1000. The expected output is a single integer representing the total count of **beautiful substrings**. Because the string length is moderate (≤ 1000), naive solutions that consider all substrings might still be feasible but may not scale efficiently if the approach is overly redundant.

Edge cases include strings with no vowels, strings with no consonants, strings of length 1, and strings where all characters are the same. Another subtle case is when `k` is larger than the possible product of vowels and consonants for short substrings, which would automatically disqualify those substrings.

## Approaches

### Brute Force

The brute-force approach involves iterating over all possible substrings of `s`. For each substring, we count the vowels and consonants, then check the two conditions: equality of counts and divisibility by `k`. If both conditions hold, we increment the count of beautiful substrings.

While this approach guarantees correctness, it is inefficient because it examines $O(n^2)$ substrings, and for each substring, counting vowels and consonants requires up to $O(n)$ operations. This results in an overall time complexity of $O(n^3)$, which is too slow for $n = 1000$.

### Optimal Approach

The key observation for a more efficient solution is that we only care about the **difference between vowels and consonants** in a substring. Define a `balance` as `vowels - consonants`. A substring has an equal number of vowels and consonants if its `balance` at the start and end differ by 0. This reduces our problem to tracking prefix balances, similar to prefix sum techniques used in subarray sum problems.

We also need to consider divisibility by `k`. Since the product `vowels * consonants` can be rewritten as `((total_length / 2)^2)` when vowels == consonants, we only need to check whether `(length / 2)^2 % k == 0` for substrings where vowels == consonants. This allows us to efficiently filter valid substrings once the balance condition is satisfied.

By using a nested loop over start and end indices but leveraging prefix balances and simple arithmetic for the divisibility check, we reduce unnecessary recomputation.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(1) | Count vowels and consonants for each substring |
| Prefix Balance + Length Check | O(n^2) | O(n) | Track running balance and length to check conditions efficiently |

## Algorithm Walkthrough

1. Precompute a `prefix_balance` array where each index `i` stores `vowels - consonants` in the substring `s[0:i]`. Initialize `prefix_balance[0] = 0`.
2. Iterate over all possible substring start indices `i` from `0` to `n-1`.
3. For each start `i`, iterate over end indices `j` from `i` to `n-1`.
4. Compute the balance for substring `s[i:j+1]` using `prefix_balance[j+1] - prefix_balance[i]`.
5. If the balance is zero, compute the number of vowels and consonants as `length // 2`.
6. Check if `(vowels * consonants) % k == 0`. If true, increment the count.
7. Return the final count.

**Why it works:** The prefix balance array ensures that we can check the equality of vowels and consonants in constant time for any substring. The divisibility check using substring length avoids recounting vowels and consonants repeatedly, guaranteeing correctness and efficiency.

## Python Solution

```python
class Solution:
    def beautifulSubstrings(self, s: str, k: int) -> int:
        n = len(s)
        vowels_set = {'a', 'e', 'i', 'o', 'u'}
        prefix_balance = [0] * (n + 1)
        
        # Compute prefix balance: vowels - consonants
        for i in range(n):
            prefix_balance[i + 1] = prefix_balance[i] + (1 if s[i] in vowels_set else -1)
        
        count = 0
        for i in range(n):
            for j in range(i, n):
                # Balance of substring s[i:j+1]
                balance = prefix_balance[j + 1] - prefix_balance[i]
                if balance == 0:
                    length = j - i + 1
                    vowels = length // 2
                    consonants = length // 2
                    if (vowels * consonants) % k == 0:
                        count += 1
        return count
```

This Python implementation first computes the prefix balance for all positions in `s`. Then, for every substring, it checks if the balance is zero and if the product of half its length squared is divisible by `k`. This avoids repeated counting of vowels and consonants for every substring.

## Go Solution

```go
func beautifulSubstrings(s string, k int) int {
    n := len(s)
    vowels := map[byte]bool{'a': true, 'e': true, 'i': true, 'o': true, 'u': true}
    prefixBalance := make([]int, n+1)

    for i := 0; i < n; i++ {
        if vowels[s[i]] {
            prefixBalance[i+1] = prefixBalance[i] + 1
        } else {
            prefixBalance[i+1] = prefixBalance[i] - 1
        }
    }

    count := 0
    for i := 0; i < n; i++ {
        for j := i; j < n; j++ {
            balance := prefixBalance[j+1] - prefixBalance[i]
            if balance == 0 {
                length := j - i + 1
                v := length / 2
                if (v*v)%k == 0 {
                    count++
                }
            }
        }
    }

    return count
}
```

The Go version mirrors the Python solution, using a map to track vowels and an integer slice for prefix balances. Integer division and modulo arithmetic are used to check the conditions efficiently.

The algorithm avoids the O(n^3) brute-force vowel/consonant count by using prefix sums, making it feasible for n ≤ 1000.
