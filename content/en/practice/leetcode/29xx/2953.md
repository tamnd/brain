## Problem Understanding

The problem requires counting **complete substrings** within a given string `word`. A substring is considered complete if **every character in it appears exactly `k` times** and **adjacent characters differ in alphabetical order by at most 2**. The input `word` is a string of lowercase letters, and `k` is an integer between 1 and the length of `word`.

The output is a single integer representing how many substrings meet these two conditions.

In other words, the algorithm must consider **all possible contiguous substrings**, count character frequencies, and ensure that the difference between each consecutive character is no more than 2 in the alphabet. The constraints tell us that `word` can be up to length `10^5`, so **brute-force approaches that examine every substring naively will likely be too slow**.

Important edge cases include:

- Substrings with repeated characters that meet `k` but fail the adjacent difference condition.
- Substrings with lengths less than `k` multiplied by the number of distinct characters.
- Strings where `k` equals the string length.
- Strings with only one type of character.

## Approaches

The **brute-force approach** is to generate all substrings of `word`, count the frequency of each character for each substring, and check if every character appears exactly `k` times. Then check that each adjacent character in the substring satisfies the absolute difference condition. This approach is correct but inefficient, because generating all substrings and checking each condition is **O(n^3)** in the worst case.

The **optimal approach** uses a **sliding window** and **hash maps for frequency counting**. The key insight is that for a substring to be complete:

1. Its length must be divisible by `k`.
2. We only need to check substrings where all distinct characters appear exactly `k` times.
3. The adjacent difference condition allows us to break early if any two consecutive characters differ by more than 2.

We can slide a window, expanding until the adjacent difference is violated, and maintain a hash map to track character counts. When the counts meet the `k` condition for all characters in the window, we increment our result. This avoids recomputation for overlapping substrings.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(26) | Generate all substrings, count frequencies, and check adjacency differences |
| Optimal | O(n^2) | O(26) | Use sliding window and hash map; break early on adjacency violation |

## Algorithm Walkthrough

1. Initialize a result counter `res` to 0.
2. Iterate over each possible starting index `i` in the string `word`.
3. Initialize an empty frequency map `freq` and set `max_len` to the maximum valid substring starting at `i`.
4. Expand the substring from index `i` to `j` while checking the absolute difference of consecutive characters.
5. For each new character added at index `j`, increment its count in `freq`.
6. Check if all counts in `freq` equal `k`. If they do, increment `res`.
7. If the difference condition is violated at any point, break the inner loop and move to the next starting index.
8. Return `res` after processing all possible starting indices.

**Why it works**: The algorithm ensures that every substring counted meets both the frequency and adjacency conditions. The sliding window prevents unnecessary examination of invalid substrings, and the frequency hash map guarantees that counts are exact.

## Python Solution

```python
class Solution:
    def countCompleteSubstrings(self, word: str, k: int) -> int:
        res = 0
        n = len(word)
        
        for i in range(n):
            freq = {}
            valid = True
            for j in range(i, n):
                if j > i and abs(ord(word[j]) - ord(word[j - 1])) > 2:
                    break
                
                freq[word[j]] = freq.get(word[j], 0) + 1
                
                if all(count == k for count in freq.values()):
                    res += 1
        
        return res
```

The Python implementation iterates over each possible starting position and expands a substring until the adjacency condition fails. The frequency dictionary counts occurrences of characters, and the `all(count == k for count in freq.values())` check ensures only complete substrings are counted. Breaking early optimizes performance.

## Go Solution

```go
func countCompleteSubstrings(word string, k int) int {
    res := 0
    n := len(word)

    for i := 0; i < n; i++ {
        freq := make(map[rune]int)
        valid := true

        for j := i; j < n; j++ {
            if j > i && abs(int(word[j])-int(word[j-1])) > 2 {
                break
            }
            freq[rune(word[j])]++
            
            complete := true
            for _, count := range freq {
                if count != k {
                    complete = false
                    break
                }
            }
            if complete {
                res++
            }
        }
    }
    return res
}

func abs(a int) int {
    if a < 0 {
        return -a
    }
    return a
}
```

The Go implementation is analogous to Python. A `map[rune]int` is used to count character frequencies. A helper function `abs` checks adjacency differences. The iteration and breaking conditions are identical.

## Edge Cases

**Single character string**: When the string length is 1, any substring of length 1 is complete if `k = 1`. The implementation handles this because the loop iterates over the string and counts the character correctly.

**Adjacent difference violation**: Substrings with characters that differ by more than 2 alphabetically should not be counted. The `abs(ord(word[j]) - ord(word[j-1])) > 2` check ensures these are excluded immediately.

**Multiple overlapping substrings**: Substrings may overlap and still count multiple times if they independently satisfy the conditions. The algorithm counts each valid substring as it slides, so no duplicates are missed.
