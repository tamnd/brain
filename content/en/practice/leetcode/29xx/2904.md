## Solution

## Problem Understanding

The problem asks us to identify a specific substring from a binary string `s` that meets two criteria: it is **beautiful**, meaning it contains exactly `k` ones, and it is the **shortest possible** substring satisfying this condition. If multiple substrings have this minimal length, we must return the **lexicographically smallest** one. A lexicographically smaller string is one that, when compared character by character from left to right, has the first differing character smaller than the other string.

The input consists of a binary string `s` with length up to 100 characters and an integer `k` representing the exact number of ones a beautiful substring must contain. The constraints indicate that we can afford solutions up to roughly O(n²) without efficiency concerns, but we still want an approach that is optimal. Edge cases include strings with no ones, strings with all ones, `k` larger than the number of ones in the string, or multiple substrings of the same minimal length.

Understanding these requirements, the output should either be the desired substring or an empty string if no substring contains exactly `k` ones.

## Approaches

The **brute-force approach** involves generating every possible substring of `s`, counting the number of ones in each, and tracking those that contain exactly `k` ones. After identifying all such substrings, we select the shortest ones and then choose the lexicographically smallest among them. While this approach is correct, it is inefficient because generating and checking all substrings requires O(n³) time in the worst case, due to substring extraction and counting.

The **optimal approach** leverages a **sliding window technique**. By using a two-pointer window to maintain the count of ones in a substring dynamically, we can expand the window to include characters and contract it when we exceed `k` ones. This allows us to efficiently find all substrings with exactly `k` ones. While doing so, we track the **shortest length** and maintain a variable storing the lexicographically smallest substring of that length. This reduces the overall complexity to O(n²) in the worst case, which is acceptable given the input constraints.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(n) | Generate all substrings, count ones, select shortest lexicographically smallest |
| Optimal | O(n²) | O(1) | Sliding window maintains ones count, track minimal length and smallest substring |

## Algorithm Walkthrough

1. Initialize two pointers `left` and `right` at the start of the string and a variable `count_ones` to track the number of ones in the current window.
2. Iterate `right` through the string from index 0 to n-1, expanding the window to include `s[right]`. Increment `count_ones` if `s[right]` is '1'.
3. Once `count_ones` exceeds `k`, move `left` forward while decrementing `count_ones` for each '1' removed from the window until `count_ones` equals `k`.
4. Each time `count_ones` equals `k`, check if the current window length is smaller than the minimal length tracked. If so, update the minimal length and record the substring. If the length is equal to the minimal length, update the substring only if it is lexicographically smaller.
5. Continue until the end of the string.
6. Return the stored substring if found, otherwise return an empty string.

**Why it works**: The sliding window ensures that all contiguous substrings with exactly `k` ones are examined efficiently. By updating the minimal length and comparing lexicographical order as we go, we guarantee the correct solution without rechecking all substrings repeatedly.

## Python Solution

```python
class Solution:
    def shortestBeautifulSubstring(self, s: str, k: int) -> str:
        n = len(s)
        min_len = float('inf')
        result = ""
        left = 0
        count_ones = 0

        for right in range(n):
            if s[right] == '1':
                count_ones += 1

            while count_ones > k:
                if s[left] == '1':
                    count_ones -= 1
                left += 1

            if count_ones == k:
                current_len = right - left + 1
                current_sub = s[left:right + 1]
                if current_len < min_len or (current_len == min_len and current_sub < result):
                    min_len = current_len
                    result = current_sub

        return result
```

The code maintains a sliding window with pointers `left` and `right` and dynamically counts ones. When exactly `k` ones are inside the window, it checks if this window produces a shorter or lexicographically smaller substring. This directly follows the algorithm walkthrough above.

## Go Solution

```go
func shortestBeautifulSubstring(s string, k int) string {
    n := len(s)
    minLen := n + 1
    result := ""
    left, countOnes := 0, 0

    for right := 0; right < n; right++ {
        if s[right] == '1' {
            countOnes++
        }

        for countOnes > k {
            if s[left] == '1' {
                countOnes--
            }
            left++
        }

        if countOnes == k {
            currentLen := right - left + 1
            currentSub := s[left : right+1]
            if currentLen < minLen || (currentLen == minLen && currentSub < result) {
                minLen = currentLen
                result = currentSub
            }
        }
    }

    return result
}
```

In Go, string slicing `s[left:right+1]` works similarly to Python. We track minimal length and lexicographical order identically. There is no need for extra space besides simple variables.
