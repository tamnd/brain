**Difficulty:** 🔴 Hard  
**Topics:** Two Pointers, String, Dynamic Programming

## Solution

## Problem Understanding

We are given a string `s` and an integer `k`. The goal is to split the string into exactly `k` non-empty substrings. Each substring must be transformed into a semi-palindrome, and we want the total number of character changes across all substrings to be as small as possible.

A semi-palindrome is more complicated than a normal palindrome. For a substring of length `m`, we choose a divisor `d` such that:

- `d` divides `m`
- `1 <= d < m`

Once `d` is chosen, we group characters by indices that differ by `d`.

For example, if the substring length is `6` and `d = 3`, the groups are:

- indices `0, 3`
- indices `1, 4`
- indices `2, 5`

Each group individually must form a palindrome.

Another way to view this is that for each remainder modulo `d`, we collect characters spaced `d` apart, and every such sequence must be palindromic.

The input size is relatively small:

- `2 <= n <= 200`
- `1 <= k <= n / 2`

A length of `200` strongly suggests that an `O(n^3)` or `O(n^4)` dynamic programming solution may still be acceptable, but exponential partition enumeration is impossible.

An important observation is that substrings of length `1` can never be semi-palindromes, because there is no valid divisor `d < length`. Fortunately, the constraint `k <= n / 2` guarantees that we can partition the string into substrings of length at least `2`.

Several edge cases are important:

- Very short substrings, especially length `2`
- Substrings with many valid divisors
- Cases where the optimal partition is not obvious
- Strings already partitionable into semi-palindromes with zero changes
- Situations where different divisors produce different modification costs

A naive implementation can easily recompute costs repeatedly, which becomes too slow.

## Approaches

### Brute Force Approach

The brute force strategy tries every possible way to partition the string into `k` substrings. For each substring, it tries every valid divisor `d`, checks how many character modifications are needed to make every modulo group palindromic, and then chooses the minimum cost for that substring.

This approach is correct because it exhaustively explores every valid partition and every valid divisor configuration.

However, it is far too slow.

The number of ways to partition a string of length `n` into `k` substrings is combinatorial. Additionally, for every substring, we must test all divisors and compute palindrome conversion costs repeatedly.

Without memoization or preprocessing, the complexity grows exponentially.

### Key Insight

The critical observation is that the cost of converting any substring into a semi-palindrome is independent from the partitioning process.

This allows us to separate the problem into two phases:

1. Precompute the minimum transformation cost for every substring.
2. Use dynamic programming to find the optimal partition into `k` substrings.

For a fixed substring and divisor `d`, the problem becomes:

- Split indices into `d` modulo groups
- Each group must become a palindrome
- The minimum changes for a palindrome can be computed using two pointers

Once substring costs are precomputed, the remaining problem becomes a classic partition DP.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Exponential | Exponential | Enumerates all partitions and divisor choices |
| Optimal | O(n^3 + k * n^2) | O(n^2 + k * n) | Precompute substring costs, then run partition DP |

## Algorithm Walkthrough

### Step 1: Precompute All Valid Divisors

For every possible substring length `len`, compute all divisors `d` such that:

- `d < len`
- `len % d == 0`

We will later use these divisors to test semi-palindrome structures efficiently.

### Step 2: Compute Semi-palindrome Cost for Every Substring

Create a 2D array:

```
cost[i][j]
```

This stores the minimum number of changes needed to transform substring `s[i:j+1]` into a semi-palindrome.

For every substring:

1. Compute its length.
2. Iterate through every valid divisor `d`.
3. For each modulo class `r` in `[0, d-1]`:

- Collect characters:

```
r, r+d, r+2d, ...
```
- Use two pointers to count mismatches.
4. Sum mismatches across all groups.
5. Take the minimum across all divisors.

### Step 3: Define DP State

Let:

```
dp[p][i]
```

represent the minimum cost to partition the first `i` characters into `p` semi-palindromic substrings.

### Step 4: Initialize Base Cases

Set:

```
dp[0][0] = 0
```

All other states begin as infinity.

### Step 5: Transition

For every partition count `p`:

Try every ending position `i`.

Then try every previous cut position `j`.

The last substring is:

```
s[j:i]
```

Its cost is already precomputed.

Transition:

```
dp[p][i] = min(
    dp[p][i],
    dp[p-1][j] + cost[j][i-1]
)
```

### Step 6: Return Final Answer

The answer is:

```
dp[k][n]
```

where `n` is the string length.

### Why it works

The algorithm works because of optimal substructure.

For every partition ending at position `i`, the last substring contributes independently to the total cost. Since we precompute the optimal semi-palindrome conversion cost for every substring, the dynamic programming step only needs to choose the best previous partition boundary.

Because all possible partitions are explored and every substring cost is minimized independently, the final answer is globally optimal.

## Python Solution

```python
from typing import List
import math

class Solution:
    def minimumChanges(self, s: str, k: int) -> int:
        n = len(s)

        divisors = [[] for _ in range(n + 1)]

        for length in range(2, n + 1):
            for d in range(1, length):
                if length % d == 0:
                    divisors[length].append(d)

        INF = 10**9

        cost = [[INF] * n for _ in range(n)]

        for left in range(n):
            for right in range(left + 1, n):
                length = right - left + 1

                best = INF

                for d in divisors[length]:
                    current = 0

                    for r in range(d):
                        positions = []

                        idx = left + r

                        while idx <= right:
                            positions.append(idx)
                            idx += d

                        l = 0
                        rr = len(positions) - 1

                        while l < rr:
                            if s[positions[l]] != s[positions[rr]]:
                                current += 1

                            l += 1
                            rr -= 1

                    best = min(best, current)

                cost[left][right] = best

        dp = [[INF] * (n + 1) for _ in range(k + 1)]
        dp[0][0] = 0

        for parts in range(1, k + 1):
            for end in range(2 * parts, n + 1):
                for start in range(2 * (parts - 1), end - 1):
                    dp[parts][end] = min(
                        dp[parts][end],
                        dp[parts - 1][start] + cost[start][end - 1]
                    )

        return dp[k][n]
```

The implementation begins by precomputing valid divisors for every possible substring length. This avoids repeatedly recomputing divisibility information later.

The `cost` matrix is the most important preprocessing step. For every substring, we test every valid divisor and calculate how many mirrored mismatches exist inside each modulo group. Every mismatch requires one character replacement.

The DP table stores optimal partitioning results. Since every substring must have length at least `2`, the loops enforce valid partition boundaries using `2 * parts`.

The final answer is the minimum cost to partition the entire string into exactly `k` semi-palindromic substrings.

## Go Solution

```go
package main

import "math"

func minimumChanges(s string, k int) int {
	n := len(s)
	const INF = int(1e9)

	divisors := make([][]int, n+1)

	for length := 2; length <= n; length++ {
		for d := 1; d < length; d++ {
			if length%d == 0 {
				divisors[length] = append(divisors[length], d)
			}
		}
	}

	cost := make([][]int, n)
	for i := range cost {
		cost[i] = make([]int, n)
		for j := range cost[i] {
			cost[i][j] = INF
		}
	}

	for left := 0; left < n; left++ {
		for right := left + 1; right < n; right++ {
			length := right - left + 1

			best := INF

			for _, d := range divisors[length] {
				current := 0

				for r := 0; r < d; r++ {
					positions := []int{}

					for idx := left + r; idx <= right; idx += d {
						positions = append(positions, idx)
					}

					l := 0
					rr := len(positions) - 1

					for l < rr {
						if s[positions[l]] != s[positions[rr]] {
							current++
						}

						l++
						rr--
					}
				}

				if current < best {
					best = current
				}
			}

			cost[left][right] = best
		}
	}

	dp := make([][]int, k+1)

	for i := range dp {
		dp[i] = make([]int, n+1)

		for j := range dp[i] {
			dp[i][j] = INF
		}
	}

	dp[0][0] = 0

	for parts := 1; parts <= k; parts++ {
		for end := 2 * parts; end <= n; end++ {
			for start := 2 * (parts - 1); start <= end-2; start++ {
				dp[parts][end] = int(math.Min(
					float64(dp[parts][end]),
					float64(dp[parts-1][start]+cost[start][end-1]),
				))
			}
		}
	}

	return dp[k][n]
}
```

The Go implementation closely mirrors the Python version. Slices are used for both the divisor lists and the DP tables. Since Go does not provide a built-in infinity value for integers, a large constant is used instead.

One small difference is that `math.Min` operates on floating point values, so integer conversions are required during transitions.

## Edge Cases

One important edge case is substrings of length `2`. These only have one valid divisor, `d = 1`. The algorithm handles this naturally because the divisor preprocessing includes only proper divisors. The palindrome mismatch calculation correctly computes whether the two characters differ.

Another tricky case is when a substring is not a normal palindrome but is still a semi-palindrome under a larger divisor. For example, `"abcabc"` is not a palindrome with `d = 1`, but becomes valid with `d = 3`. A naive solution that checks only ordinary palindromes would fail. The implementation explicitly tests every valid divisor, ensuring the optimal structure is found.

A third important case involves partition validity. Since every semi-palindrome substring must have length at least `2`, invalid partitions containing length `1` substrings must never be considered. The DP transition bounds enforce this by ensuring enough remaining characters exist for every remaining partition.
