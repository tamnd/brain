# LeetCode 2934 - Minimum Operations to Maximize Last Elements in Arrays

## Problem Understanding

The problem gives us two integer arrays, `nums1` and `nums2`, both of the same length `n`. We are allowed to perform operations where we choose an index `i` and swap `nums1[i]` with `nums2[i]`.

Our goal is to make both arrays satisfy a very specific condition:

- The last element of `nums1` must be the maximum value in `nums1`
- The last element of `nums2` must be the maximum value in `nums2`

We want to return the minimum number of swaps required to achieve this. If it is impossible, we return `-1`.

The important detail is that swaps happen only between corresponding positions. We cannot arbitrarily rearrange elements. At each index, we only choose between two configurations:

- Keep `(nums1[i], nums2[i])` as is
- Swap them to `(nums2[i], nums1[i])`

The constraints are relatively small, `n <= 1000`, which means an `O(n)` or `O(n^2)` solution is acceptable. However, brute force over all swap combinations would still be exponential and infeasible.

Several edge cases are important:

- Arrays of length `1`, where the condition is automatically satisfied
- Cases where the last elements themselves may need to be swapped
- Situations where no configuration can make both last elements maximal
- Cases where multiple swap combinations are valid, requiring us to find the minimum number of swaps

The key observation is that only the final values at index `n - 1` matter as target maximums.

## Approaches

### Brute Force Approach

A brute force solution would try every possible combination of swaps. Since each index has two choices, swap or do not swap, there are `2^n` total configurations.

For every configuration, we would:

1. Apply the chosen swaps
2. Compute the maximum values in both arrays
3. Check whether the last elements are equal to those maxima
4. Track the minimum number of swaps among valid configurations

This approach is correct because it explores every possible state. However, its time complexity is exponential, which becomes infeasible even for moderate values of `n`.

### Optimal Approach

The critical insight is that the only values that matter as final maxima are the values placed at index `n - 1`.

There are only two possibilities:

1. Do not swap the last pair
2. Swap the last pair

Once the final target values are fixed, every earlier index must satisfy:

```
nums1[i] <= target1
nums2[i] <= target2
```

If an index violates this condition, we may swap that pair. If neither configuration works, the scenario is impossible.

This reduces the problem to checking two candidate configurations in linear time.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(n) | Tries every swap combination |
| Optimal | O(n) | O(1) | Checks only two possible target configurations |

## Algorithm Walkthrough

### Step 1: Consider the two possible final configurations

The last elements determine the maximum allowed values in each array.

There are only two possibilities:

1. Keep the last pair unchanged
2. Swap the last pair

For each possibility, we define:

- `target1`, the desired final maximum for `nums1`
- `target2`, the desired final maximum for `nums2`

### Step 2: Validate every earlier index

For every index `i < n - 1`, we check whether the current pair already fits:

```
nums1[i] <= target1
nums2[i] <= target2
```

If yes, we do not need a swap.

Otherwise, we check whether swapping works:

```
nums2[i] <= target1
nums1[i] <= target2
```

If yes, we count one swap.

If neither configuration works, this entire scenario is impossible.

### Step 3: Count swaps

For each valid scenario, we count:

- Swaps needed for earlier indices
- Plus one extra swap if the last pair itself was swapped

### Step 4: Return the minimum valid answer

We compute the result for both scenarios and return the smaller valid value.

If both scenarios are impossible, return `-1`.

### Why it works

The last elements define the maximum permissible values for each array. Every earlier position is independent because swaps only affect one index at a time.

Once the targets are fixed, every index has only two possible states, original or swapped. Therefore, greedily choosing the valid configuration with fewer swaps is always optimal.

Since there are only two possible target configurations, checking both guarantees correctness.

## Python Solution

```python
from typing import List

class Solution:
    def minOperations(self, nums1: List[int], nums2: List[int]) -> int:
        n = len(nums1)

        def check(target1: int, target2: int, swap_last: int) -> int:
            swaps = swap_last

            for i in range(n - 1):
                a = nums1[i]
                b = nums2[i]

                # No swap needed
                if a <= target1 and b <= target2:
                    continue

                # Swap works
                if b <= target1 and a <= target2:
                    swaps += 1
                    continue

                # Impossible
                return float('inf')

            return swaps

        # Scenario 1: keep last pair unchanged
        option1 = check(nums1[-1], nums2[-1], 0)

        # Scenario 2: swap last pair
        option2 = check(nums2[-1], nums1[-1], 1)

        answer = min(option1, option2)

        return -1 if answer == float('inf') else answer
```

The solution defines a helper function `check` that evaluates one target configuration.

The parameters `target1` and `target2` represent the required maximum values for the two arrays. The parameter `swap_last` indicates whether the final pair was swapped.

Inside the loop, we examine each earlier index independently. If the pair already fits within the target bounds, we leave it unchanged. Otherwise, we test whether swapping fixes the issue. If neither configuration is valid, we immediately mark the scenario as impossible.

Finally, we compute the answer for both possible final configurations and return the minimum valid result.

## Go Solution

```go
package main

import "math"

func minOperations(nums1 []int, nums2 []int) int {
	n := len(nums1)

	check := func(target1 int, target2 int, swapLast int) int {
		swaps := swapLast

		for i := 0; i < n-1; i++ {
			a := nums1[i]
			b := nums2[i]

			// No swap needed
			if a <= target1 && b <= target2 {
				continue
			}

			// Swap works
			if b <= target1 && a <= target2 {
				swaps++
				continue
			}

			// Impossible
			return math.MaxInt32
		}

		return swaps
	}

	option1 := check(nums1[n-1], nums2[n-1], 0)
	option2 := check(nums2[n-1], nums1[n-1], 1)

	answer := option1
	if option2 < answer {
		answer = option2
	}

	if answer == math.MaxInt32 {
		return -1
	}

	return answer
}
```

The Go implementation closely mirrors the Python solution. Since Go does not have `float('inf')`, we use `math.MaxInt32` as a sentinel impossible value.

The helper function is implemented as a closure so it can directly access `nums1`, `nums2`, and `n`.

All operations remain constant space because we only store counters and temporary variables.

## Edge Cases

### Arrays of Length One

When `n = 1`, the last element is automatically the maximum because it is the only element in each array. A buggy implementation might still attempt unnecessary swaps or comparisons. This solution handles the case naturally because the loop over `n - 1` elements becomes empty, and the answer correctly becomes `0`.

### Impossible Configurations

Some inputs cannot satisfy the constraints regardless of swaps. For example:

```
nums1 = [1,5,4]
nums2 = [2,5,3]
```

The pair `(5,5)` exceeds every possible target configuration. The implementation explicitly checks both original and swapped states for every index. If neither works, the helper immediately returns an impossible sentinel value.

### Swapping the Last Pair

A common mistake is assuming the last pair should never be swapped. In reality, swapping the final elements may drastically reduce the number of earlier swaps required.

The solution explicitly evaluates both possibilities:

- Keep the last pair
- Swap the last pair

This guarantees the minimum valid answer is always found.
