## Solution

## Problem Understanding

This problem involves analyzing a database table `Sessions` that records user sessions on a streaming platform. Each row contains a `user_id`, the start and end timestamps of a session, a unique `session_id`, and the `session_type`, which can be either `Viewer` or `Streamer`. The task is to identify users whose **first session** was as a viewer and count how many subsequent streaming sessions they had. The result should be a table containing `user_id` and `sessions_count`, sorted by `sessions_count` in descending order, and in case of ties, by `user_id` in descending order.

The input is guaranteed to have unique `session_id`s, but users may have multiple sessions of either type. Important edge cases include users who never streamed after viewing, users whose first session was as a streamer, and users who only have one session as a viewer.

The output requires filtering based on the **first session** only, counting subsequent streaming sessions, and sorting correctly.

## Approaches

### Brute Force Approach

A naive solution would iterate over each user, sort their sessions by `session_start`, check if the first session is a `Viewer`, then count the number of `Streamer` sessions. While correct, this approach requires sorting per user and scanning all sessions, making it inefficient for large datasets.

### Optimal Approach

The key insight is to identify the first session per user using a window function (`ROW_NUMBER`) or an aggregate (`MIN(session_start)`), filter users whose first session is `Viewer`, and then join or filter their streaming sessions to count them. This allows a single scan and aggregation, reducing the complexity compared to repeated sorting for each user.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N log N) | O(N) | Sort sessions per user, filter first session, count streams |
| Optimal | O(N) | O(N) | Use window functions or group by + join to filter first session viewers and count streams |

## Algorithm Walkthrough

1. Identify the first session for each user. This can be done using `ROW_NUMBER` over `PARTITION BY user_id ORDER BY session_start` or using `MIN(session_start)` aggregation.
2. Filter users whose first session type is `Viewer`.
3. Join or filter the sessions table to include only these users.
4. Count all `Streamer` sessions per user.
5. Order the results by `sessions_count` descending and then by `user_id` descending.

Why it works: By isolating the first session, we ensure only users starting as viewers are included. Counting streaming sessions afterward guarantees correctness since we only consider subsequent sessions. Aggregating per user ensures accurate counts, and the ordering satisfies the problem constraints.

## Python Solution

```python
import pandas as pd

def viewers_turned_streamers(sessions: pd.DataFrame) -> pd.DataFrame:
    # Step 1: Identify first session per user
    first_sessions = sessions.sort_values(['user_id', 'session_start']).groupby('user_id', as_index=False).first()
    
    # Step 2: Filter users whose first session is a Viewer
    first_viewers = first_sessions[first_sessions['session_type'] == 'Viewer']['user_id']
    
    # Step 3: Filter sessions for those users and count streaming sessions
    streamer_counts = sessions[
        (sessions['user_id'].isin(first_viewers)) &
        (sessions['session_type'] == 'Streamer')
    ].groupby('user_id', as_index=False).size().rename(columns={'size': 'sessions_count'})
    
    # Step 4: Sort by sessions_count and user_id in descending order
    result = streamer_counts.sort_values(['sessions_count', 'user_id'], ascending=[False, False])
    return result
```

Explanation: First, we sort sessions to get each user's earliest session and then filter those who started as viewers. We count only the streaming sessions for these users using a groupby and `size()`. Finally, the result is sorted as required.

## Go Solution

```go
package main

import (
    "database/sql"
    _ "github.com/go-sql-driver/mysql"
    "fmt"
)

func viewersTurnedStreamers(db *sql.DB) (*sql.Rows, error) {
    query := `
    WITH first_sessions AS (
        SELECT user_id, session_type, MIN(session_start) AS first_start
        FROM Sessions
        GROUP BY user_id, session_type
    ),
    first_viewers AS (
        SELECT user_id
        FROM first_sessions
        WHERE session_type = 'Viewer'
    )
    SELECT s.user_id, COUNT(*) AS sessions_count
    FROM Sessions s
    JOIN first_viewers f ON s.user_id = f.user_id
    WHERE s.session_type = 'Streamer'
    GROUP BY s.user_id
    ORDER BY sessions_count DESC, user_id DESC;
    `
    return db.Query(query)
}
```

Go-specific notes: SQL is used directly because Go does not provide native dataframe-like operations. `sql.DB.Query` returns a `*sql.Rows` object that must be scanned in the calling code.

## Edge Cases

One edge case is a user whose first session is Viewer but has no subsequent Streamer sessions. The implementation correctly filters out such users by counting only streaming sessions after identifying first session type.

Another edge case is a user with multiple sessions starting with Streamer. These users are excluded entirely because their first session is not Viewer.

Finally, handling multiple users with the same number of streaming sessions ensures the sorting handles ties by `user_id` descending, which the algorithm does by explicitly specifying the secondary sort key.
