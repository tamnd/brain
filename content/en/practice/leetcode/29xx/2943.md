# LeetCode 2943 - Maximize Area of Square Hole in Grid

## Problem Understanding

The problem describes a rectangular grid made from horizontal and vertical bars. There are `n + 2` horizontal bars and `m + 2` vertical bars. These bars partition the space into unit cells of size `1 x 1`.

Some bars are removable. The arrays `hBars` and `vBars` specify which horizontal and vertical bars may be removed. After removing selected bars, adjacent cells merge together into a larger hole.

The goal is to create the largest possible square-shaped hole and return its area.

A key detail is understanding how removing consecutive bars affects the hole size. Suppose we remove consecutive horizontal bars such as `[2, 3, 4]`. This merges multiple adjacent rows into one larger vertical span. If we remove `k` consecutive bars, the resulting span becomes `k + 1` cells tall. The same logic applies to vertical bars.

Since the hole must be square-shaped, the side length is determined by the smaller of:

- the maximum merged height
- the maximum merged width

The final answer is:

$$\text{side}^2$$

The constraints are very important:

- `n` and `m` can be as large as `10^9`
- however, `hBars.length` and `vBars.length` are at most `100`

This means we cannot simulate the entire grid. Fortunately, the removable bar lists are tiny, so the solution only needs to analyze those arrays.

Several edge cases are important:

- No consecutive removable bars exist.
- All removable bars form one long consecutive sequence.
- Horizontal and vertical maximum spans differ greatly.
- The grid dimensions themselves are huge, but removable arrays remain small.

The problem guarantees that all removable bar indices are distinct, which simplifies processing because we do not need to handle duplicates.

## Approaches

### Brute Force Approach

A brute force solution would attempt every subset of removable horizontal bars and every subset of removable vertical bars.

For each configuration:

1. Remove selected bars.
2. Compute the largest merged horizontal span.
3. Compute the largest merged vertical span.
4. Determine the largest square side.
5. Track the maximum area.

This approach is correct because it exhaustively explores all possible removal combinations.

However, it is computationally infeasible. If there are `h` removable horizontal bars and `v` removable vertical bars, then:

- horizontal subsets = `2^h`
- vertical subsets = `2^v`

The total combinations become:

$$2^h \times 2^v$$

With up to 100 removable bars, this is astronomically large.

### Optimal Approach

The key observation is that only consecutive removable bars matter.

If removable bars are consecutive, removing them creates a larger continuous opening. For example:

- removing `[2]` creates a span of `2`
- removing `[2,3]` creates a span of `3`
- removing `[2,3,4]` creates a span of `4`

Therefore:

$$\text{maximum span} = \text{longest consecutive removable sequence} + 1$$

The problem reduces to:

1. Find the longest consecutive sequence in `hBars`
2. Find the longest consecutive sequence in `vBars`
3. Compute:

$$\text{side} = \min(hSpan, vSpan)$$
4. Return:

$$\text{side}^2$$

Sorting the arrays makes consecutive detection straightforward.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^h × 2^v) | O(1) | Tries every possible removal combination |
| Optimal | O(h log h + v log v) | O(1) | Finds longest consecutive sequences after sorting |

## Algorithm Walkthrough

1. Sort the `hBars` array.

Sorting allows consecutive removable bars to appear next to each other, making sequence detection easy.
2. Scan the sorted horizontal bars to find the longest consecutive sequence.

Maintain:

- `current`, the current consecutive streak length
- `best`, the maximum streak found

If:

```
bars[i] == bars[i-1] + 1
```

then the sequence continues.

Otherwise, reset the streak.
3. Convert the streak length into a merged span.

If we remove `k` consecutive bars, the merged opening spans `k + 1` cells.
4. Repeat the same process for `vBars`.
5. The largest square side length is:

$$\min(hSpan, vSpan)$$

because a square requires equal width and height.
6. Return:

$$\text{side}^2$$

### Why it works

Removing consecutive bars merges adjacent cells into a larger continuous opening. A sequence of `k` removed bars always creates a span of `k + 1`.

The largest possible square is constrained by the smaller dimension between the maximum horizontal span and maximum vertical span. Therefore, computing the longest consecutive removable sequences independently for rows and columns guarantees the optimal square area.

## Python Solution

```python
from typing import List

class Solution:
    def maximizeSquareHoleArea(
        self,
        n: int,
        m: int,
        hBars: List[int],
        vBars: List[int]
    ) -> int:

        def max_span(bars: List[int]) -> int:
            bars.sort()

            longest = 1
            current = 1

            for i in range(1, len(bars)):
                if bars[i] == bars[i - 1] + 1:
                    current += 1
                else:
                    current = 1

                longest = max(longest, current)

            return longest + 1

        horizontal_span = max_span(hBars)
        vertical_span = max_span(vBars)

        side = min(horizontal_span, vertical_span)

        return side * side
```

The solution defines a helper function `max_span` that computes the maximum merged span from a removable bar array.

The array is first sorted so consecutive bars appear adjacent. The algorithm then scans the array while tracking the current streak length and the best streak seen so far.

Whenever two neighboring values differ by exactly one, the consecutive sequence continues. Otherwise, the streak resets.

After computing the longest consecutive removable sequence, the function adds one because removing `k` consecutive bars creates a span of `k + 1`.

The main function computes the maximum horizontal span and vertical span independently, then uses the smaller one as the square side length.

Finally, it returns the square area.

## Go Solution

```go
package main

import (
	"sort"
)

func maximizeSquareHoleArea(n int, m int, hBars []int, vBars []int) int {

	maxSpan := func(bars []int) int {
		sort.Ints(bars)

		longest := 1
		current := 1

		for i := 1; i < len(bars); i++ {
			if bars[i] == bars[i-1]+1 {
				current++
			} else {
				current = 1
			}

			if current > longest {
				longest = current
			}
		}

		return longest + 1
	}

	horizontalSpan := maxSpan(hBars)
	verticalSpan := maxSpan(vBars)

	side := horizontalSpan
	if verticalSpan < side {
		side = verticalSpan
	}

	return side * side
}
```

The Go implementation closely mirrors the Python version.

The main difference is explicit use of `sort.Ints` for sorting slices. Go also requires manual minimum comparisons instead of Python's built-in `min`.

Integer overflow is not a concern because the maximum possible side length is at most `101`, so the area easily fits inside a standard integer.

## Edge Cases

### No Consecutive Removable Bars

If removable bars are isolated, such as `[2,4,6]`, no large merged opening can form. Every removable bar only increases the span to `2`.

This case can easily cause off-by-one mistakes if the implementation forgets that removing one bar already creates a span of `2`. The solution handles this correctly by initializing the longest streak to `1` and returning `longest + 1`.

### Entire Array Forms One Consecutive Sequence

If every removable bar is consecutive, such as `[2,3,4,5,6]`, the algorithm must correctly accumulate the full streak length.

The scan logic continuously increments `current` while values remain consecutive, ensuring the largest possible span is discovered.

### Unsorted Input Arrays

The removable bar arrays are not guaranteed to be sorted. Without sorting, consecutive removable bars may not appear adjacent, causing incorrect streak detection.

The implementation explicitly sorts both arrays before scanning, guaranteeing correct identification of consecutive sequences.
