## Solution

## Problem Understanding

The problem asks us to rearrange a given array `nums` of positive integers into the lexicographically smallest possible array by performing swaps under a specific constraint. Each swap is allowed only between elements whose absolute difference does not exceed a given `limit`.

In other words, if `|nums[i] - nums[j]| <= limit`, you can swap `nums[i]` and `nums[j]`. We can perform these swaps any number of times and in any order. The goal is not to minimize the sum or maximize a pattern, but to make the array **lexicographically smallest**, which means at the first index where two arrays differ, the smaller element should appear first.

The input guarantees that the array length can be up to `10^5`, each element can be as large as `10^9`, and the limit can also be as large as `10^9`. This means any brute-force swapping approach will be far too slow because iterating over all possible swaps is exponential in time. We must exploit the structure of the array and the swap condition efficiently.

Edge cases include arrays where no swaps are possible (all differences exceed `limit`), arrays that are already sorted, or arrays where multiple swaps within local groups are required to achieve the smallest lexicographic order.

## Approaches

The **brute-force approach** is to repeatedly iterate over the array and swap any pair of elements that satisfy the condition and reduce the lexicographic order. You continue doing this until no swaps improve the array. While this is correct in principle, it is extremely inefficient because it can take exponential time as the array size grows, especially when `nums.length` is up to `10^5`.

The **optimal approach** leverages the observation that the swap condition effectively defines **connected groups** of elements. If two elements are part of the same connected group where consecutive differences are all `<= limit`, they can be rearranged freely within that group. Once the groups are identified, the solution is to sort each group independently and place the sorted elements back in their original positions. This guarantees the lexicographically smallest arrangement, because elements cannot move across different groups.

To implement this, we can use a **Union-Find (Disjoint Set Union)** data structure or a simple **connected component detection** by iterating through consecutive elements.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) worst case | O(1) | Iteratively swaps any eligible pairs until no improvement, too slow for large `n`. |
| Optimal | O(n log n) | O(n) | Identify connected groups, sort each group, and reconstruct array. Efficient for constraints. |

## Algorithm Walkthrough

1. Initialize a Union-Find structure (or an array `group_id`) to track which elements are connected via the swap condition.
2. Iterate through `nums` from left to right. For each pair of consecutive elements `nums[i]` and `nums[i+1]`, check if `|nums[i] - nums[i+1]| <= limit`. If true, union them into the same group.
3. After processing all consecutive elements, each element now belongs to a group representing a connected component of swappable elements.
4. Create a mapping from group ID to the list of elements in that group. This allows us to sort elements **within the group independently**.
5. For each group, sort the elements in ascending order.
6. Reconstruct the array by iterating through the original indices, replacing each element with the next smallest element from its group.

**Why it works**: Elements within the same group can be freely swapped with each other while respecting the `limit` constraint, so sorting each group independently guarantees the lexicographically smallest ordering without violating any constraints. Elements in different groups cannot be swapped, so their relative positions remain constrained, which is consistent with the rules.

## Python Solution

```python
from typing import List
from collections import defaultdict
import heapq

class Solution:
    def lexicographicallySmallestArray(self, nums: List[int], limit: int) -> List[int]:
        n = len(nums)
        parent = list(range(n))

        def find(x):
            while parent[x] != x:
                parent[x] = parent[parent[x]]
                x = parent[x]
            return x

        def union(x, y):
            px, py = find(x), find(y)
            if px != py:
                parent[py] = px

        # Build connected components based on the swap condition
        for i in range(n - 1):
            if abs(nums[i] - nums[i + 1]) <= limit:
                union(i, i + 1)

        groups = defaultdict(list)
        for i in range(n):
            root = find(i)
            heapq.heappush(groups[root], nums[i])

        # Reconstruct the array with sorted elements within each group
        result = [0] * n
        for i in range(n):
            root = find(i)
            result[i] = heapq.heappop(groups[root])
        return result
```

**Explanation**: We use Union-Find to identify connected components where swaps are allowed. Each component is stored in a min-heap so that extracting elements gives the smallest remaining number efficiently. Finally, we rebuild the array by taking the smallest element from the appropriate group for each index.

## Go Solution

```go
package main

func lexicographicallySmallestArray(nums []int, limit int) []int {
    n := len(nums)
    parent := make([]int, n)
    for i := 0; i < n; i++ {
        parent[i] = i
    }

    var find func(int) int
    find = func(x int) int {
        if parent[x] != x {
            parent[x] = find(parent[x])
        }
        return parent[x]
    }

    union := func(x, y int) {
        px, py := find(x), find(y)
        if px != py {
            parent[py] = px
        }
    }

    // Build connected components
    for i := 0; i < n-1; i++ {
        if abs(nums[i]-nums[i+1]) <= limit {
            union(i, i+1)
        }
    }

    groups := make(map[int][]int)
    for i := 0; i < n; i++ {
        root := find(i)
        groups[root] = append(groups[root], nums[i])
    }

    // Sort each group
    for _, v := range groups {
        sort.Ints(v)
    }

    // Reconstruct the array
    idx := make(map[int]int)
    res := make([]int, n)
    for i := 0; i < n; i++ {
        root := find(i)
        res[i] = groups[root][idx[root]]
        idx[root]++
    }
    return res
}

func abs(a int) int {
    if a < 0 {
        return -a
    }
    return a
}
```

**Go-specific notes**: Go does not have a built-in heap for small datasets, so we sort the slices of each group. We use a map to track how many elements have been used per group to reconstruct the array efficiently.

## Worked Examples

**Example 1:** `nums = [1,5,3,9,8], limit = 2`

Step through Union-Find:

- Union(1,2) because |5-3|=2 <= 2 → group {1,5,3}
- Union(3,4) because |9-8|=1 <= 2 → group {9,8}

Group 1 elements: [1,5,3] → sorted → [1,3,5]

Group 2 elements: [9,8] → sorted → [8,9]

Reconstruct: [1,3,5,8,9]

**Example 2:** `nums = [1,7,6,18,2,1], limit = 3`

Connected groups by consecutive swaps → group1: [7,6], group2: [1,2,1], group3: [18]

Sorting groups and reconstructing → [1,6,7,18,1,2]

**Example 3:** `nums = [1,7,28,19,10], limit = 3`

No consecutive elements satisfy the condition → no swaps possible → result = [1,7,28,19,10]

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n log n) | Iterating through array is O(n), sorting elements within each connected group is O(k log k) and sum(k) = n. |
| Space | O(n) | Union-Find array + groups storage + result array. |

Sorting is the dominant factor. Union-Find operations are nearly O(1) amortized due to path compression.
