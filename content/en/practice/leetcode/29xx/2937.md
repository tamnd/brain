## Solution

## Problem Understanding

The problem gives us three strings, `s1`, `s2`, and `s3`. In one operation, we are allowed to pick exactly one of the strings and delete its rightmost character. The important restriction is that we are never allowed to completely empty a string. Every string must always contain at least one character.

Our goal is to make all three strings identical using the minimum number of deletions. If there is no way to make them equal under the given rules, we must return `-1`.

Since we can only remove characters from the end of a string, we cannot rearrange characters or delete characters from the middle. This means the final equal string must be a common prefix of all three original strings. Any characters after that prefix must be deleted.

For example, if the strings are:

```
s1 = "abc"
s2 = "abb"
s3 = "ab"
```

the longest common prefix is `"ab"`. We can delete:

- `'c'` from `"abc"`
- `'b'` from `"abb"`

After two operations, all strings become `"ab"`.

The constraints are very small:

- Each string length is between 1 and 100
- Only lowercase English letters are used

Because the input size is tiny, efficiency is not a major concern. Even less optimal solutions would still run quickly. However, we should still design a clean and logically correct algorithm.

Several edge cases are important:

- The first characters may already differ, making equality impossible.
- The strings may already be equal, requiring zero operations.
- One string may already be the shortest valid common prefix.
- The only common prefix may be empty, but empty strings are not allowed.

The restriction against empty strings is extremely important. Even though the empty string is technically a common prefix of every string, it is not a valid final answer here.

## Approaches

### Brute Force Approach

A brute force strategy would try every possible prefix length for the three strings and check whether all strings can be reduced to that prefix.

Since deletions can only happen from the right side, every possible final string must be a prefix of each original string. We could generate all prefixes of the shortest string and test whether each prefix is shared by all three strings.

For each valid common prefix, we compute how many deletions are needed:

```
(len(s1) - prefix_length)
+ (len(s2) - prefix_length)
+ (len(s3) - prefix_length)
```

Then we choose the minimum.

This approach is correct because it explicitly examines every possible achievable final string. However, it does unnecessary repeated work because many prefixes are checked independently.

### Optimal Approach

The key observation is that the best final string is always the longest common prefix of the three strings.

Why is this true?

Every deletion shortens a string. Therefore, keeping more characters always reduces the number of operations. Among all common prefixes, the longest one minimizes deletions.

So the problem becomes:

1. Find the length of the longest common prefix among all three strings.
2. If the common prefix length is zero, return `-1`.
3. Otherwise, compute how many characters must be removed from each string.

This gives a very clean linear solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Checks many possible prefixes repeatedly |
| Optimal | O(n) | O(1) | Finds the longest common prefix directly |

Here, `n` represents the length of the shortest string.

## Algorithm Walkthrough

### Optimal Algorithm

1. Compute the minimum length among the three strings.

We only need to compare characters up to the shortest string length because no common prefix can be longer than the shortest string.
2. Initialize a counter for the common prefix length.

This variable tracks how many consecutive characters match across all three strings.
3. Iterate through each index from left to right.

At each position:

- Compare `s1[i]`, `s2[i]`, and `s3[i]`
- If all three characters match, increase the common prefix length
- Otherwise, stop immediately

We stop at the first mismatch because prefixes must be continuous from the beginning.
4. Check whether the common prefix length is zero.

If there is no shared prefix character, the strings can never become equal because we are only allowed to delete characters from the end. Returning `-1` is required.
5. Compute the total deletions.

For each string:

- Characters after the common prefix must be removed
- The number of deletions is `len(string) - common_prefix_length`

Add the deletions from all three strings together.
6. Return the total.

### Why it works

The algorithm works because deleting characters from the right can only preserve prefixes. Therefore, any achievable final string must be a common prefix of all three inputs. Among all common prefixes, the longest one minimizes deletions because it preserves the maximum number of characters. The algorithm directly finds this longest common prefix and computes the exact number of required deletions.

## Python Solution

```python
class Solution:
    def findMinimumOperations(self, s1: str, s2: str, s3: str) -> int:
        min_length = min(len(s1), len(s2), len(s3))
        
        common_prefix_length = 0
        
        for i in range(min_length):
            if s1[i] == s2[i] == s3[i]:
                common_prefix_length += 1
            else:
                break
        
        if common_prefix_length == 0:
            return -1
        
        operations = (
            len(s1) - common_prefix_length
            + len(s2) - common_prefix_length
            + len(s3) - common_prefix_length
        )
        
        return operations
```

The implementation begins by finding the smallest string length because the common prefix cannot exceed that boundary.

Next, the loop scans all three strings simultaneously from left to right. As long as the characters match at the same index, the common prefix length grows. The moment a mismatch appears, the loop stops because prefixes must remain continuous.

After determining the common prefix length, the code handles the impossible case. If no common prefix exists, the answer is `-1`.

Finally, the solution computes how many characters each string must delete in order to shrink down to the common prefix. The sum of these deletions is returned.

## Go Solution

```go
func findMinimumOperations(s1 string, s2 string, s3 string) int {
    minLength := len(s1)
    
    if len(s2) < minLength {
        minLength = len(s2)
    }
    
    if len(s3) < minLength {
        minLength = len(s3)
    }
    
    commonPrefixLength := 0
    
    for i := 0; i < minLength; i++ {
        if s1[i] == s2[i] && s2[i] == s3[i] {
            commonPrefixLength++
        } else {
            break
        }
    }
    
    if commonPrefixLength == 0 {
        return -1
    }
    
    operations := 
        (len(s1) - commonPrefixLength) +
        (len(s2) - commonPrefixLength) +
        (len(s3) - commonPrefixLength)
    
    return operations
}
```

The Go implementation follows the same logic as the Python version. Since strings in Go can be indexed directly by byte, character comparisons are straightforward because the input only contains lowercase English letters.

No special handling for empty strings is required because the problem guarantees every string has length at least one.

## Edge Cases

### No Common Prefix

If the first characters of the strings differ, there is no valid solution. Since deletions only occur at the end, the starting character can never change. A buggy implementation might mistakenly allow reducing everything to an empty string, but the problem explicitly forbids empty strings. The implementation correctly returns `-1` when the common prefix length is zero.

### Strings Already Equal

When all three strings are already identical, no operations are necessary. Some implementations accidentally continue deleting characters unnecessarily after finding equality. This solution correctly computes zero deletions because the common prefix length equals every string length.

### One String Is Already the Target Prefix

Consider:

```
s1 = "abcd"
s2 = "ab"
s3 = "abc"
```

The optimal final string is `"ab"`, which already equals `s2`. A naive implementation might incorrectly try to extend beyond the shortest string. This implementation avoids that issue by limiting comparisons to the minimum string length.

### Common Prefix Stops Early

For inputs like:

```
s1 = "abx"
s2 = "aby"
s3 = "abz"
```

the common prefix ends immediately after `"ab"`. The algorithm stops at the first mismatch and does not continue scanning unnecessarily. This guarantees correctness because prefixes must be continuous from the beginning of the strings.
