## Solution

## Problem Understanding

The problem provides a table `Loans` that contains information about different loans taken by users. Each row includes a `loan_id`, a `user_id`, and a `loan_type`. The goal is to find all `user_id`s that have taken **at least one loan of type "Refinance"** and **at least one loan of type "Mortgage"**. The result must include only distinct `user_id`s and should be sorted in ascending order.

In other words, we need to identify users who simultaneously have both a Refinance loan and a Mortgage loan. Users who have only one type or none should be excluded.

Important considerations include the fact that a single user may have multiple loans of the same type or of other types. The input is guaranteed to have unique `loan_id`s, but `user_id`s may repeat across rows. The edge cases involve users with loans of only one type, no loans at all, or multiple loans of the same type.

## Approaches

The **brute-force approach** would be to iterate through all users, collect their loan types in a list, and then check if both "Refinance" and "Mortgage" exist for each user. While this approach works correctly, it is inefficient for large datasets because it requires scanning all rows multiple times and using extra memory for temporary lists.

The **optimal approach** leverages SQL aggregation. By grouping the table by `user_id` and using conditional aggregation, we can count whether each user has at least one "Refinance" and at least one "Mortgage". This method is efficient because it performs a single pass over the table and directly filters the users satisfying both conditions. Using `COUNT(CASE WHEN ...)` or `SUM(CASE WHEN ...)` allows the query to evaluate the presence of each loan type without materializing large intermediate lists.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) per user lookup | O(n) | Iterate through all loans and check each user's loan types |
| Optimal | O(n) | O(u) | Group by `user_id` and count conditional loan types |

## Algorithm Walkthrough

1. Start with the `Loans` table as input.
2. Group all rows by `user_id` so that we can analyze loans per user.
3. For each group, count the number of loans of type "Refinance" and the number of loans of type "Mortgage" using conditional aggregation.
4. Filter the groups where both counts are at least one. This ensures that only users who have at least one of each type are included.
5. Order the resulting user IDs in ascending order to satisfy the output requirement.
6. Return the final list of user IDs.

**Why it works:** Grouping by `user_id` ensures that all loans of a single user are considered together. The conditional counts verify the presence of both required loan types. Filtering ensures correctness, and ordering provides the expected result format.

## Python Solution

```python
import pandas as pd

def loan_types(loans: pd.DataFrame) -> pd.DataFrame:
    grouped = loans.groupby("user_id").agg(
        refinance_count=pd.NamedAgg(column="loan_type", aggfunc=lambda x: (x == "Refinance").sum()),
        mortgage_count=pd.NamedAgg(column="loan_type", aggfunc=lambda x: (x == "Mortgage").sum())
    ).reset_index()
    
    result = grouped[
        (grouped["refinance_count"] > 0) & (grouped["mortgage_count"] > 0)
    ][["user_id"]].sort_values("user_id").reset_index(drop=True)
    
    return result
```

In this Python solution, we use `pandas` to mimic SQL-style operations. We group the loans by `user_id` and calculate conditional counts for "Refinance" and "Mortgage". We then filter the groups meeting both conditions and sort the results.

## Go Solution

```go
package main

import (
    "database/sql"
    _ "github.com/go-sql-driver/mysql"
    "fmt"
)

func LoanTypes(db *sql.DB) ([]int, error) {
    query := `
        SELECT user_id
        FROM Loans
        GROUP BY user_id
        HAVING SUM(CASE WHEN loan_type = 'Refinance' THEN 1 ELSE 0 END) > 0
           AND SUM(CASE WHEN loan_type = 'Mortgage' THEN 1 ELSE 0 END) > 0
        ORDER BY user_id ASC;
    `
    rows, err := db.Query(query)
    if err != nil {
        return nil, err
    }
    defer rows.Close()

    var result []int
    for rows.Next() {
        var userID int
        if err := rows.Scan(&userID); err != nil {
            return nil, err
        }
        result = append(result, userID)
    }
    return result, nil
}
```

The Go solution mirrors the SQL aggregation logic. Conditional sums inside `HAVING` filter users meeting both loan type requirements. Results are appended to a slice and returned.

## Edge Cases

One important edge case is a user who has multiple loans but only of one type. The algorithm handles this because the conditional counts for the missing loan type will be zero, and the filter excludes them.

A second edge case is an empty table. The algorithm correctly returns an empty result because grouping an empty DataFrame produces no rows.

A third edge case is users with many loans of the required types and other types intermixed. The algorithm correctly sums only the relevant types, ignoring others, so the presence of additional loan types does not affect correctness.
