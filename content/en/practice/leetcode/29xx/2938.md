## Solution

## Problem Understanding

The problem asks us to rearrange a binary string representing balls on a table, where `'0'` is a white ball and `'1'` is a black ball, such that all white balls appear to the left and all black balls appear to the right. We are allowed to swap only **adjacent** balls, and the goal is to find the **minimum number of swaps** needed to achieve this arrangement.

The input `s` is a string of length `n` up to 100,000, which means a brute-force simulation of all swaps would be inefficient. The output is an integer representing the minimum number of steps required.

Important observations include that we only care about the relative order of the `0`s and `1`s. Each `1` that is to the left of a `0` will need to "jump over" the zeros to reach its correct position. This suggests a counting-based approach rather than actually performing swaps.

Edge cases to note are when the string is already sorted (all `0`s to the left and all `1`s to the right) or when the string consists entirely of one color.

## Approaches

### Brute Force

A brute-force approach would simulate the process by repeatedly finding adjacent pairs `10` and swapping them until the string is sorted. This approach guarantees correctness but is extremely slow because each swap only moves a black ball one position to the right. For large `n`, this could lead to **O(n²)** time complexity, which is not feasible for `n` up to 10⁵.

### Optimal Approach

The optimal approach relies on counting the number of swaps each black ball needs to reach its correct position. We iterate through the string from left to right and track how many white balls (`0`s) we have seen so far. Every time we encounter a black ball (`1`), it will eventually need to move past all the white balls encountered so far. Therefore, we increment a counter by the number of white balls seen. This gives us the total minimum swaps needed in **O(n)** time and **O(1)** space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Simulate all adjacent swaps until sorted |
| Optimal | O(n) | O(1) | Count the number of zeros before each one to compute swaps |

## Algorithm Walkthrough

1. Initialize a counter `zero_count` to `0` to track the number of white balls (`0`s) encountered so far.
2. Initialize a counter `swaps` to `0` to accumulate the total number of required swaps.
3. Iterate through each character `c` in the string `s`.
4. If `c` is `'0'`, increment `zero_count` by `1` since we have another white ball that black balls may need to pass.
5. If `c` is `'1'`, increment `swaps` by `zero_count` because this black ball will need to move past all white balls seen so far.
6. After processing the entire string, return `swaps` as the minimum number of steps required.

**Why it works:** Each black ball needs to move past every white ball that appears before it. By counting these white balls as we scan the string, we directly calculate the exact number of swaps needed. This ensures correctness because each counted swap corresponds to a single necessary adjacent swap in any valid sorting sequence.

## Python Solution

```python
class Solution:
    def minimumSteps(self, s: str) -> int:
        zero_count = 0
        swaps = 0
        for c in s:
            if c == '0':
                zero_count += 1
            else:  # c == '1'
                swaps += zero_count
        return swaps
```

The implementation iterates through the string once. The `zero_count` variable keeps track of how many white balls are before each black ball, and `swaps` accumulates the total number of required moves. This directly follows the algorithm walkthrough and avoids the need for nested loops.

## Go Solution

```go
func minimumSteps(s string) int64 {
    var zeroCount int64 = 0
    var swaps int64 = 0
    for i := 0; i < len(s); i++ {
        if s[i] == '0' {
            zeroCount++
        } else {
            swaps += zeroCount
        }
    }
    return swaps
}
```

In Go, we use `int64` to handle large counts safely for the upper limit of `n`. The logic mirrors the Python solution exactly. We iterate over the string using indexing and update counters similarly.

## Edge Cases

**Single-character strings:** A string with only one ball cannot require any swaps, so the algorithm must correctly return 0. The implementation handles this automatically as the counters remain zero or only increment appropriately.

**Already sorted strings:** Strings like `"00001111"` require no swaps. The algorithm correctly returns zero because no black balls have zeros before them in this scenario.

**Alternating patterns:** Strings like `"101010"` or `"010101"` stress the counting logic because multiple black balls must pass multiple white balls. The cumulative sum approach accurately calculates the total swaps for all black balls, avoiding the need for simulation or complex index tracking.

This method is efficient, simple, and guarantees the minimum number of adjacent swaps required.
