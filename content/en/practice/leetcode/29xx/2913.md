## Solution

## Problem Understanding

The problem asks us to examine every possible non-empty subarray of the input array `nums`. For each subarray, we compute how many distinct elements it contains, square that count, and add the result to a running total.

A subarray is contiguous, which means we only consider segments formed by consecutive indices. For example, if `nums = [1,2,1]`, the subarray `[1,2]` is valid, but `[1,1]` formed by skipping the middle element is not.

The key operation for each subarray is computing its distinct count. Suppose a subarray contains the values `[1,2,2,3]`. The distinct elements are `{1,2,3}`, so the distinct count is `3`, and its contribution to the final answer is:

$3^2 = 9$

The input constraints are very small:

- `1 <= nums.length <= 100`
- `1 <= nums[i] <= 100`

These constraints are important because they tell us that even algorithms with quadratic or cubic behavior may still be acceptable. Since the array length is only at most `100`, we do not need advanced data structures or highly optimized techniques.

Some important edge cases naturally arise:

- Arrays where all elements are the same, because every subarray has distinct count `1`
- Arrays where all elements are unique, because distinct counts grow with subarray size
- Single-element arrays, which produce exactly one subarray
- Repeated patterns such as `[1,2,1,2]`, where the distinct count changes dynamically as the subarray expands

The problem guarantees that the array is non-empty, so we never need to handle an empty input.

## Approaches

### Brute Force Approach

The most direct solution is to generate every possible subarray and explicitly compute its distinct count from scratch.

For every starting index `i`, we try every ending index `j >= i`. Once we have the subarray `nums[i:j+1]`, we place all its elements into a set to count distinct values. Then we square the size of the set and add it to the answer.

This approach is correct because it literally follows the problem definition. Every subarray is examined exactly once, and the distinct count is computed accurately.

However, it performs unnecessary repeated work. Consider overlapping subarrays like:

- `[1]`
- `[1,2]`
- `[1,2,1]`

The distinct elements for earlier subarrays are recomputed repeatedly. Building a fresh set for every subarray leads to inefficient repeated scanning.

If there are `O(n^2)` subarrays and each distinct computation takes `O(n)` time, the total complexity becomes `O(n^3)`.

### Improved Incremental Hash Set Approach

A better approach uses the observation that when extending a subarray one element at a time, we do not need to recompute distinct elements from scratch.

For a fixed starting index `i`, we gradually expand the ending index `j`. We maintain a hash set containing all distinct elements currently present in the subarray `nums[i..j]`.

When we move from `j` to `j + 1`, we simply insert the new element into the set. The size of the set immediately tells us the distinct count.

This eliminates repeated scanning and reduces the total complexity to `O(n^2)`.

Because the constraints are small, this solution is more than fast enough and is the intended optimal approach.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(n) | Rebuilds distinct set for every subarray |
| Optimal | O(n^2) | O(n) | Reuses a growing hash set while expanding subarrays |

## Algorithm Walkthrough

1. Initialize a variable `answer` to `0`. This variable stores the final sum of squared distinct counts.
2. Iterate through every possible starting index `start` from `0` to `n - 1`.
3. For each starting index, create an empty hash set called `distinct_values`. This set will store the unique elements present in the current subarray.
4. Expand the subarray one element at a time by iterating `end` from `start` to `n - 1`.
5. Insert `nums[end]` into the hash set. Since sets automatically avoid duplicates, the set always contains exactly the distinct elements in the current subarray.
6. Compute the number of distinct elements using `len(distinct_values)`.
7. Square the distinct count and add it to `answer`.

If the distinct count is `k`, the contribution is:

$k^2$

1. Continue expanding until all subarrays starting at `start` have been processed.
2. After all starting indices are processed, return `answer`.

### Why it works

For every starting index, the algorithm incrementally builds all subarrays extending to the right. The hash set always contains exactly the distinct elements in the current subarray because every newly added element is inserted once, and duplicates do not affect the set.

Since every subarray is processed exactly once and its distinct count is computed correctly, the final accumulated sum is correct.

## Python Solution

```python
from typing import List

class Solution:
    def sumCounts(self, nums: List[int]) -> int:
        n = len(nums)
        answer = 0

        for start in range(n):
            distinct_values = set()

            for end in range(start, n):
                distinct_values.add(nums[end])

                distinct_count = len(distinct_values)
                answer += distinct_count * distinct_count

        return answer
```

The implementation directly follows the algorithm described earlier.

The outer loop chooses the starting index of the subarray. For each starting position, we create a fresh set called `distinct_values`.

The inner loop expands the subarray toward the right. Each newly included value is added to the set. Because Python sets automatically maintain uniqueness, the set size always equals the number of distinct elements in the current subarray.

After updating the set, we compute the squared distinct count and add it to the running total.

The implementation is concise because Python's built-in set structure provides constant average time insertion and lookup.

## Go Solution

```go
func sumCounts(nums []int) int {
	n := len(nums)
	answer := 0

	for start := 0; start < n; start++ {
		distinctValues := make(map[int]bool)

		for end := start; end < n; end++ {
			distinctValues[nums[end]] = true

			distinctCount := len(distinctValues)
			answer += distinctCount * distinctCount
		}
	}

	return answer
}
```

The Go implementation mirrors the Python logic closely.

Instead of Python's `set`, Go uses a `map[int]bool` to simulate a hash set. The keys represent distinct elements currently present in the subarray.

Go's `len(map)` returns the number of keys, which gives the distinct count directly.

Integer overflow is not a concern here because the maximum possible answer is small under the given constraints.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n^2) | There are O(n^2) subarrays, and each set insertion is O(1) average time |
| Space | O(n) | The hash set can contain at most n distinct elements |

The algorithm uses two nested loops to enumerate all subarrays. For each subarray extension, we perform constant average time hash set insertion and size retrieval.

The auxiliary space comes from the hash set used to track distinct elements for the current starting index.

## Edge Cases

A single-element array is the smallest valid input. There is only one subarray, and its distinct count is always `1`. Implementations sometimes accidentally mishandle very small inputs due to loop boundaries, but this solution naturally handles the case because both loops execute exactly once.

Arrays with all duplicate values are another important edge case. In such arrays, every subarray has distinct count `1`. A buggy implementation might accidentally overcount duplicates if it uses a list instead of a proper set. This implementation avoids the issue because the hash set stores each value only once.

Arrays with all unique elements are also significant because the distinct count grows continuously as the subarray expands. This tests whether the algorithm correctly updates the distinct count incrementally. Since each newly added value increases the set size by one, the implementation handles this naturally.

Alternating repetition patterns such as `[1,2,1,2]` are particularly useful for testing correctness. The distinct count sometimes increases and sometimes remains unchanged depending on whether the new element already exists in the set. Using a hash set guarantees that duplicates do not incorrectly inflate the distinct count.
