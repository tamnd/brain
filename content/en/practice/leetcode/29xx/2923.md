## Solution

## Problem Understanding

This problem is asking us to determine the **champion team** in a tournament where the outcomes between any two teams are predefined in a 2D boolean matrix `grid`. Each element `grid[i][j]` indicates if team `i` is stronger than team `j` (1 means `i` is stronger, 0 means `j` is stronger). A champion team is one that no other team is stronger than.

The input is a square matrix `grid` of size `n x n`, where `n` is the number of teams. Each team has a well-defined strength relationship with every other team. The diagonal elements `grid[i][i]` are always 0 because a team cannot compete with itself. Importantly, the matrix satisfies a **transitive property**: if team `a` is stronger than team `b` and team `b` is stronger than team `c`, then team `a` is stronger than team `c`. This guarantees there will always be a unique champion.

The output is simply the index of the champion team, which must be between `0` and `n-1`.

Edge cases to consider include the minimum size `n = 2`, cases where the champion is the first or last team, and matrices where many teams are indirectly weaker than the champion.

## Approaches

### Brute Force

A brute-force approach would check for each team whether there exists any other team stronger than it. For team `i`, we would iterate through all `j != i` and check if `grid[j][i] == 1`. If no team is stronger than `i`, then `i` is the champion. This is correct because it directly follows the definition of the champion, but it requires checking every pair of teams, leading to an O(n²) time complexity.

### Optimal

The key observation is that because the matrix is **transitive**, we can find the champion in a single pass by using a **candidate elimination strategy**. We start with the first team as a candidate. Then for each subsequent team `i`, if the candidate is weaker than `i` (`grid[candidate][i] == 0`), we replace the candidate with `i`. By the end, the candidate will be the champion. This works because any team weaker than a stronger team cannot be the champion due to transitivity. This reduces the time complexity to O(n).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Check each team against all others |
| Optimal | O(n) | O(1) | Candidate elimination using transitivity |

## Algorithm Walkthrough

1. Initialize `champion_candidate` as 0, assuming the first team might be the champion.
2. Iterate through each team `i` from 1 to `n-1`.
3. For each team, check if the current candidate is weaker than `i` by evaluating `grid[champion_candidate][i]`.
4. If the candidate is weaker (`grid[champion_candidate][i] == 0`), replace `champion_candidate` with `i`.
5. After checking all teams, `champion_candidate` is the champion because no remaining team can be stronger than it due to transitivity.
6. Return `champion_candidate`.

**Why it works:** The candidate elimination maintains the invariant that the current candidate is always a team that could be the champion. Because of the transitive property, once a stronger team is found, all previous weaker candidates are automatically eliminated.

## Python Solution

```python
from typing import List

class Solution:
    def findChampion(self, grid: List[List[int]]) -> int:
        n = len(grid)
        champion_candidate = 0
        
        for i in range(1, n):
            if grid[champion_candidate][i] == 0:
                champion_candidate = i
        
        return champion_candidate
```

**Explanation:** We start with the first team as the candidate. We iterate through all other teams. If the current candidate loses to a team, we update the candidate. After one full pass, the candidate cannot be beaten by any other team due to the transitive property, ensuring it is the champion.

## Go Solution

```go
func findChampion(grid [][]int) int {
    n := len(grid)
    championCandidate := 0

    for i := 1; i < n; i++ {
        if grid[championCandidate][i] == 0 {
            championCandidate = i
        }
    }

    return championCandidate
}
```

**Go-specific notes:** The Go version mirrors the Python logic. We use `len(grid)` to determine `n`, integer comparison for the 0/1 values in the grid, and a simple for loop to iterate through candidates.

## Worked Examples

**Example 1:** `grid = [[0,1],[0,0]]`

Start with `champion_candidate = 0`. Compare with team 1: `grid[0][1] = 1` → candidate stays 0. Return 0.

**Example 2:** `grid = [[0,0,1],[1,0,1],[0,0,0]]`

Start with candidate 0. Compare with team 1: `grid[0][1] = 0` → candidate becomes 1. Compare with team 2: `grid[1][2] = 1` → candidate stays 1. Return 1.

| 2 | 1 | 1 beats 2 |

## Edge Cases

**Edge Case 1:** Minimal matrix size `n = 2`. Only two teams exist, so the comparison is direct. The candidate elimination still works correctly because only one comparison occurs.

**Edge Case 2:** Champion is the last team. If all previous candidates lose to the final team, the algorithm correctly updates the candidate to the last team. This ensures no off-by-one errors.

**Edge Case 3:** Chain dominance. When a team dominates indirectly through transitive wins, the algorithm still identifies the champion in one pass without needing to check all pairwise relations. This guarantees correctness while maintaining O(n) time.
