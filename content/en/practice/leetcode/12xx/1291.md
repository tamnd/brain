## Solution

## Problem Understanding

The problem asks us to generate all integers within a given range `[low, high]` that have **sequential digits**. An integer has sequential digits if each digit is exactly one greater than the previous digit. For example, `123` is valid because `1 → 2 → 3` increments by one at each step, while `135` is not valid because `1 → 3` jumps by two.

The input consists of two integers `low` and `high` where `10 <= low <= high <= 10^9`. The output is a sorted list of integers with sequential digits that lie within this range, inclusive. Sorting is important because the sequence naturally grows, but we must ensure the final result is in ascending order.

Constraints imply that the numbers can have anywhere from 2 to 9 digits, since a 10-digit sequential number would start with `1` and end with `10`, which is invalid. Edge cases include scenarios where the lower or upper bound is itself a sequential number, or when no sequential numbers exist in the range.

## Approaches

A **brute-force approach** would iterate through every number between `low` and `high`, checking for the sequential property. For each number, we would convert it to a string or repeatedly extract digits to verify if each digit increments by exactly one. While this guarantees correctness, it is inefficient because the range can be very large, up to 1 billion numbers. Checking each number individually would be computationally expensive and unnecessary.

The **optimal approach** leverages the observation that sequential numbers are rare and predictable. They can only be constructed from starting digits `1` through `9` and are limited by length. Instead of iterating over all numbers, we generate sequential numbers by concatenating digits in order. For example, starting from `1`, we can generate `12, 123, 1234, ...` until we reach `high`. We repeat this for starting digits `2` through `9`. This drastically reduces unnecessary computation, as we only generate valid candidates.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(high - low) | O(1) | Check each number in the range for sequential property |
| Optimal | O(1) | O(1) | Generate sequential numbers directly using digit concatenation, independent of range size |

## Algorithm Walkthrough

1. Identify the lengths of numbers we need to consider based on the number of digits in `low` and `high`. This allows us to avoid generating sequential numbers that are obviously too small or too large.
2. For each possible starting digit `start` from `1` to `9`, begin building sequential numbers by concatenating digits in increasing order.
3. For each number generated, check if it lies within the inclusive range `[low, high]`. If it does, add it to the result list.
4. Continue extending each sequential number until it would exceed 9 as a digit, as any digit beyond 9 is invalid.
5. After generating all candidates, return the result list. By construction, it will already be sorted.

**Why it works**: This method guarantees that all numbers with sequential digits are generated, because every sequential number must start with a digit `1` through `9` and increment by 1 at each step. By checking the range as we generate them, we avoid unnecessary numbers and ensure correctness.

## Python Solution

```python
from typing import List

class Solution:
    def sequentialDigits(self, low: int, high: int) -> List[int]:
        result = []
        for start in range(1, 10):
            num = start
            next_digit = start
            while num <= high and next_digit < 10:
                if num >= low:
                    result.append(num)
                next_digit += 1
                num = num * 10 + next_digit
        return sorted(result)
```

This Python implementation first initializes an empty list `result`. For each starting digit from 1 to 9, it builds sequential numbers using a loop, appending numbers that fall within the desired range. The number is constructed by multiplying the current number by 10 and adding the next digit. Sorting at the end ensures the final list is in ascending order.

## Go Solution

```go
func sequentialDigits(low int, high int) []int {
    result := []int{}
    for start := 1; start <= 9; start++ {
        num := start
        nextDigit := start
        for num <= high && nextDigit < 10 {
            if num >= low {
                result = append(result, num)
            }
            nextDigit++
            num = num*10 + nextDigit
        }
    }
    return result
}
```

The Go implementation mirrors the Python version. The key differences are the explicit use of `append` for slices and integer types. Empty slices are initialized instead of lists. Sorting is unnecessary since numbers are generated in ascending order.

| 4 | 2 | 2 | 2 | num < low, continue |
| 5 | 2 | 23 | 3 | num < low, continue |

Result: `[123, 234]`

Sequential numbers generated: 1234, 2345, 3456, 4567, 5678, 6789, 12345. Each number is checked against the range and appended accordingly. Final result: `[1234,2345,3456,4567,5678,6789,12345]`.

## Edge Cases

**Lower bound at minimum**: When `low` is `10`, the algorithm correctly starts generating 2-digit sequential numbers beginning with `1` and stops when numbers exceed `high`. This prevents returning numbers smaller than `low`.

**Upper bound at maximum**: When `high` is `10^9`, the algorithm safely stops generating numbers once the next digit would exceed 9. This avoids integer overflow and ensures numbers like `123456789` are correctly handled.

**No valid sequential numbers**: If `low` and `high` define a range that does not contain any sequential numbers, such as `[90, 100]`, the algorithm returns an empty list. The check `if num >= low` ensures that numbers outside the range are not appended.

This implementation comprehensively handles all edge cases efficiently and correctly.
