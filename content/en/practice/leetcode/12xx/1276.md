## Solution

## Problem Understanding

The problem gives us two integers, `tomatoSlices` and `cheeseSlices`, which represent the total number of tomato slices and cheese slices available for making burgers.

There are only two possible burger types:

- A jumbo burger uses 4 tomato slices and 1 cheese slice.
- A small burger uses 2 tomato slices and 1 cheese slice.

We need to determine how many jumbo burgers and how many small burgers can be made such that:

- Every tomato slice is used
- Every cheese slice is used
- No ingredients remain unused

The output should be a list in the format:

```
[total_jumbo, total_small]
```

If no valid combination exists, we must return an empty list `[]`.

The important detail is that both burger counts must be non-negative integers. Fractional burgers or negative burger counts are invalid.

The constraints allow values up to `10^7`, which immediately tells us that inefficient enumeration approaches could become too slow. Since the problem involves only two variables and two equations, it strongly hints that a mathematical solution exists.

We can model the problem as a system of equations:

Let:

- `j` = number of jumbo burgers
- `s` = number of small burgers

Then:

```
4j + 2s = tomatoSlices
j + s = cheeseSlices
```

The challenge is to solve these equations while ensuring:

- `j >= 0`
- `s >= 0`
- both are integers

Several edge cases are important:

- `tomatoSlices` may be odd, which immediately makes the problem impossible because both burger types consume an even number of tomato slices.
- There may be too few tomato slices to support all cheese slices.
- There may be too many tomato slices for the available cheese slices.
- Both inputs may be zero, which should return `[0, 0]`.

## Approaches

### Brute Force Approach

A straightforward approach is to try every possible number of jumbo burgers.

Suppose we try all values of `j` from `0` to `cheeseSlices`. For each value:

- Compute how many tomato slices remain after making `j` jumbo burgers.
- Check whether the remaining tomato slices can be perfectly divided into small burgers.
- Verify whether the total cheese slices are also used exactly.

This works because the search space includes every possible valid number of jumbo burgers. If a valid solution exists, we will eventually find it.

However, this approach is inefficient for large inputs. Since `cheeseSlices` can be as large as `10^7`, iterating through all possibilities is unnecessary and wasteful.

### Optimal Mathematical Approach

The key observation is that the problem forms a system of linear equations.

We have:

```
4j + 2s = tomatoSlices
j + s = cheeseSlices
```

We can solve these equations algebraically.

From the second equation:

```
s = cheeseSlices - j
```

Substitute into the first equation:

```
4j + 2(cheeseSlices - j) = tomatoSlices
```

Simplify:

```
4j + 2cheeseSlices - 2j = tomatoSlices
2j = tomatoSlices - 2cheeseSlices
j = (tomatoSlices - 2cheeseSlices) / 2
```

Then:

```
s = cheeseSlices - j
```

This gives a direct constant-time solution.

We only need to validate:

- `tomatoSlices` is even
- `j` and `s` are non-negative
- both are integers

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(cheeseSlices) | O(1) | Tries every possible jumbo burger count |
| Optimal | O(1) | O(1) | Uses algebra to directly compute the answer |

## Algorithm Walkthrough

1. First, check whether `tomatoSlices` is even.

Both burger types use an even number of tomato slices. Therefore, any valid total must also be even. If `tomatoSlices` is odd, we can immediately return `[]`.
2. Compute the number of jumbo burgers.

Using the derived formula:

```
jumbo = (tomatoSlices - 2 * cheeseSlices) / 2
```

This comes directly from solving the linear equations.
3. Compute the number of small burgers.

Since each burger uses exactly one cheese slice:

```
small = cheeseSlices - jumbo
```
4. Validate the solution.

We must ensure:

- `jumbo >= 0`
- `small >= 0`

If either value is negative, the ingredient counts are impossible.
5. Return the result.

If all checks pass:

```
[jumbo, small]
```

Otherwise return `[]`.

### Why it works

The algorithm works because the burger requirements form a complete system of linear equations with two variables and two constraints. Solving the equations guarantees that any computed solution exactly uses all tomato and cheese slices. The validation checks ensure the solution also satisfies the requirement that burger counts must be non-negative integers.

## Python Solution

```python
from typing import List

class Solution:
    def numOfBurgers(self, tomatoSlices: int, cheeseSlices: int) -> List[int]:
        # Total tomato slices must be even
        if tomatoSlices % 2 != 0:
            return []

        jumbo = (tomatoSlices - 2 * cheeseSlices) // 2
        small = cheeseSlices - jumbo

        # Validate that both burger counts are non-negative
        if jumbo < 0 or small < 0:
            return []

        # Verify exact usage of ingredients
        if 4 * jumbo + 2 * small != tomatoSlices:
            return []

        return [jumbo, small]
```

The implementation begins by checking whether the tomato slice count is even. This is a necessary condition because every burger consumes an even number of tomato slices.

Next, the code computes the number of jumbo burgers using the algebraic formula derived earlier. Once the jumbo count is known, the small burger count is determined from the cheese equation.

After computing both values, the implementation validates that neither count is negative. Negative burgers are impossible and indicate that the ingredient totals cannot produce a valid combination.

Finally, the implementation performs a verification step to ensure the computed burger counts exactly reproduce the original tomato slice count. While the algebra already guarantees correctness in valid cases, this extra check improves robustness and clarity.

## Go Solution

```go
func numOfBurgers(tomatoSlices int, cheeseSlices int) []int {
    // Tomato slices must be even
    if tomatoSlices%2 != 0 {
        return []int{}
    }

    jumbo := (tomatoSlices - 2*cheeseSlices) / 2
    small := cheeseSlices - jumbo

    // Validate non-negative counts
    if jumbo < 0 || small < 0 {
        return []int{}
    }

    // Verify exact ingredient usage
    if 4*jumbo+2*small != tomatoSlices {
        return []int{}
    }

    return []int{jumbo, small}
}
```

The Go implementation follows the same mathematical derivation as the Python solution.

One small language-specific detail is that Go returns an empty slice using `[]int{}` rather than Python's `[]`.

Integer division is safe here because we first verify that `tomatoSlices` is even. Since all values remain within the problem constraints, integer overflow is not a concern in Go's standard integer type.

## Edge Cases

One important edge case occurs when `tomatoSlices` is odd. Since both burger types consume an even number of tomato slices, any odd total can never be formed. A naive implementation that skips this check may accidentally produce fractional burger counts or invalid integer division results. The implementation handles this immediately with an early return.

Another important edge case is when the computed number of jumbo burgers becomes negative. This happens when there are too many cheese slices relative to tomato slices. For example, `tomatoSlices = 4` and `cheeseSlices = 17` would mathematically produce `-15` jumbo burgers. The implementation explicitly checks for negative values and returns an empty list.

A third edge case occurs when tomato slices greatly exceed what the available cheese slices can support. Since each burger consumes exactly one cheese slice, the maximum tomato usage is achieved when every burger is jumbo. If even that maximum is insufficient, no solution exists. For example, `tomatoSlices = 10` and `cheeseSlices = 1` cannot work because one burger can consume at most 4 tomato slices. The validation logic correctly detects this through the final non-negative and equation checks.

The `(0, 0)` input is another subtle boundary case. Some implementations incorrectly return an empty list because they assume at least one burger must exist. However, zero burgers perfectly satisfies the requirement of using all ingredients without leftovers. The implementation correctly returns `[0, 0]`.
