## Solution

## Problem Understanding

The problem asks us to find the **maximum length** of a string that can be formed by concatenating a subsequence of strings from the given array `arr`, with the requirement that all characters in the resulting string are **unique**. In other words, we want the longest string formed from some subset of `arr` where no letter repeats.

The input is an array of strings `arr`, each string containing only lowercase English letters. The output is a single integer representing the maximum possible length of a valid concatenated string.

The constraints tell us that `arr` can contain up to 16 strings, and each string can have up to 26 characters. Since the total number of possible subsequences is `2^n` where `n` is the number of strings, the maximum number of subsequences is `2^16 = 65536`, which is feasible for a backtracking approach.

Important edge cases include:

- Strings in `arr` that themselves contain duplicate characters, which cannot be used in any valid concatenation.
- Strings that together would cause character duplication.
- Single-character arrays or arrays with a single string containing all letters of the alphabet.

## Approaches

The **brute-force approach** is to generate all possible subsequences of the array `arr`, concatenate the strings for each subsequence, and check if the resulting string contains only unique characters. This guarantees correctness but is inefficient because generating all subsequences is `O(2^n)` and checking uniqueness could cost `O(k)` per concatenated string, where `k` is the total length of the subsequence. For large arrays with long strings, this can be slow.

The **optimal approach** leverages **backtracking with bit masking** to efficiently represent sets of characters. Each string is first converted to a bitmask representing the characters it contains. If a string has duplicate characters, it is discarded. Then, using backtracking, we explore the inclusion or exclusion of each string. By using bitwise operations, we can check in `O(1)` time whether adding a string would introduce duplicate characters, making the solution both elegant and efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * k) | O(k) | Generates all subsequences, checks uniqueness per concatenation |
| Optimal (Backtracking + Bitmask) | O(2^n * 26) | O(n) | Uses bitmasking to quickly check character conflicts |

## Algorithm Walkthrough

1. **Preprocess strings into bitmasks**: For each string in `arr`, check if it contains duplicate characters. If it does, discard it. Otherwise, convert it into a 26-bit integer mask where each bit represents a character 'a' to 'z'.
2. **Initialize backtracking**: Start with an empty bitmask representing no characters used and an index at the first string.
3. **Recursive exploration**: At each step, consider including the current string if its mask does not conflict with the accumulated mask (bitwise AND is zero). Recursively call the function with the updated mask and index. Also, explore the path of skipping the current string.
4. **Update maximum length**: Each time a new mask is created by including a string, compute its length (number of 1-bits) and update a global maximum if it is larger.
5. **Return result**: After exploring all subsequences, return the maximum length found.

**Why it works**: The bitmask ensures that no duplicate characters are included in the concatenation. Backtracking systematically explores all subsequences. Because each combination is considered and uniqueness is enforced, the algorithm guarantees the maximum possible length is found.

## Python Solution

```python
from typing import List

class Solution:
    def maxLength(self, arr: List[str]) -> int:
        def to_bitmask(s: str) -> int:
            mask = 0
            for c in s:
                bit = 1 << (ord(c) - ord('a'))
                if mask & bit:
                    return 0  # duplicate character in string
                mask |= bit
            return mask

        masks = [to_bitmask(s) for s in arr if to_bitmask(s) > 0]
        max_len = 0

        def backtrack(index: int, current_mask: int):
            nonlocal max_len
            max_len = max(max_len, bin(current_mask).count('1'))
            for i in range(index, len(masks)):
                if current_mask & masks[i] == 0:
                    backtrack(i + 1, current_mask | masks[i])

        backtrack(0, 0)
        return max_len
```

**Explanation**:

We first define `to_bitmask` to convert strings into integer masks. Strings with duplicates return 0 and are ignored. The `masks` list stores all valid strings as bitmasks. The `backtrack` function explores subsequences recursively, using bitwise operations to ensure character uniqueness. We update `max_len` whenever a longer valid concatenation is found.

## Go Solution

```go
func maxLength(arr []string) int {
    toBitmask := func(s string) int {
        mask := 0
        for _, c := range s {
            bit := 1 << (c - 'a')
            if mask&bit != 0 {
                return 0
            }
            mask |= bit
        }
        return mask
    }

    masks := []int{}
    for _, s := range arr {
        if m := toBitmask(s); m != 0 {
            masks = append(masks, m)
        }
    }

    maxLen := 0
    var backtrack func(index int, currentMask int)
    backtrack = func(index int, currentMask int) {
        count := 0
        temp := currentMask
        for temp > 0 {
            count += temp & 1
            temp >>= 1
        }
        if count > maxLen {
            maxLen = count
        }
        for i := index; i < len(masks); i++ {
            if currentMask&masks[i] == 0 {
                backtrack(i+1, currentMask|masks[i])
            }
        }
    }

    backtrack(0, 0)
    return maxLen
}
```

**Explanation**:

The Go implementation mirrors the Python approach. `toBitmask` converts strings to integer masks, skipping duplicates. We use a recursive closure `backtrack` to explore subsequences. Go requires manually counting 1-bits in the mask using a loop. Otherwise, the logic is the same.

## Worked Examples

**Example 1: arr = ["un","iq","ue"]**

| Step | Current Mask | Included Strings | Max Length |
| --- | --- | --- | --- |
| Start | 0b0 | [] | 0 |
| Include "un" | 0b11110 | ["un"] | 2 |
| Include "iq" | 0b1111110 | ["un","iq"] | 4 |
| Skip "ue" | 0b1111110 | ["un","iq"] | 4 |
| Include "ue" alone | 0b1111110 | ["ue"] | 2 |
| Final max | - | - | 4 |

**Example 2: arr = ["cha","r","act","ers"]**

Exploring subsequences like "cha" + "ers" gives 6, which is the maximum.

**Example 3: arr = ["abcdefghijklmnopqrstuvwxyz"]**

Only one string is valid, length 26.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(2^n * 26) | Each subsequence is explored, checking uniqueness via bitmask in O(26) |
| Space | O(n) | Recursion depth up to n and storing n masks |

The recursion depth is at most the number of strings. Bitmasks are integers of fixed size, so space is dominated by recursion.
