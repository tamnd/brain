## Solution

## Problem Understanding

This problem asks us to design a system for controlling a traffic light at a two-road intersection, ensuring that cars can pass safely without deadlock. Road A allows cars to travel North-South (direction 1) and South-North (direction 2), while Road B allows cars to travel West-East (direction 3) and East-West (direction 4). At any moment, only one road can have a green light; the other road must have a red light, meaning no two cars from different roads can be crossing the intersection simultaneously.

The input is a stream of cars arriving at the intersection. Each car has a unique ID (`carId`), a road ID (`roadId`), and a direction. The system provides two functions: `turnGreen()`, which switches the light for a road, and `crossCar()`, which allows the car to cross. The task is to implement `carArrived()` such that it ensures no collisions and no deadlocks while allowing cars to cross as efficiently as possible.

Constraints provide that there will be at most 20 cars, all arrival times are non-decreasing, and cars are unique. This ensures that we do not need to optimize for very high throughput, but we must carefully synchronize access to the intersection to prevent deadlocks.

Important edge cases include multiple cars arriving at the same time, cars arriving on a road with a red light, and sequences where cars on one road continuously arrive while cars on the other road are waiting. The solution must guarantee fairness and avoid starvation.

## Approaches

The brute-force approach would simulate each car independently and try to cross without coordination, only checking if the light is green. Whenever a car arrives on a road with a red light, it waits. This naive approach works because cars wait if their road is red, but it does not manage switching the light efficiently. Moreover, without synchronization, multiple cars from different roads could attempt to cross simultaneously, leading to an unsafe state or deadlock. This approach could work for the small input sizes but is conceptually incomplete because it ignores coordination logic.

The optimal approach uses a concurrency control mechanism. We maintain the current green light state and a lock or condition variable for cars waiting on each road. When a car arrives, if its road is green, it crosses immediately. If the road is red, it signals the system to turn the light green for its road when the intersection is free. This approach guarantees that at most one road has a green light at any time, avoids deadlocks, and ensures fairness.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Simulate cars individually, naive waiting on red light |
| Optimal | O(n) | O(n) | Maintain green light state and queues for waiting cars, ensures deadlock-free operation |

## Algorithm Walkthrough

1. Initialize a variable to track the current green road, starting with Road A as green.
2. Maintain a queue or counter of cars waiting on each road. This allows us to know if a road has cars that need to cross.
3. When a car arrives, check if the road it is on is green. If it is green, immediately call `crossCar()`.
4. If the road is red, add the car to the waiting queue. If no car is currently crossing, call `turnGreen()` to switch the light to the car's road.
5. After turning the light green for a road, allow all cars waiting on that road to cross sequentially.
6. Once the queue is empty, the system can switch the light if cars arrive on the other road.
7. Ensure that only one road has a green light at any time, and prevent calling `turnGreen()` unnecessarily.

**Why it works:** The invariant maintained is that only one road can have a green light at any moment. Cars on the green road can cross immediately, and cars on the red road wait. When the red road has cars and the green road queue is empty, the system switches the light. This guarantees no collisions and no deadlocks, while allowing efficient flow.

## Python Solution

```python
from threading import Lock, Condition

class TrafficLight:
    def __init__(self):
        self.lock = Lock()
        self.road_green = 1  # 1 for Road A, 2 for Road B
        self.waiting = {1: 0, 2: 0}  # count of cars waiting on each road
        self.cond = Condition(self.lock)

    def carArrived(
        self,
        carId: int,
        roadId: int,
        direction: int,
        turnGreen: 'Callable[[], None]',
        crossCar: 'Callable[[], None]'
    ) -> None:
        with self.lock:
            if roadId != self.road_green:
                self.waiting[roadId] += 1
                while roadId != self.road_green:
                    if self.waiting[self.road_green] == 0:
                        turnGreen()
                        self.road_green = roadId
                    self.cond.wait()
                self.waiting[roadId] -= 1

            crossCar()
            self.cond.notify_all()
```

**Explanation:** The lock ensures mutual exclusion when accessing shared state. The condition variable manages cars waiting on a red light. When a car arrives on a red road, it increments the waiting count and waits until its road becomes green. If no cars are crossing on the current green road, it switches the light using `turnGreen()`. After crossing, the car notifies other waiting cars.

## Go Solution

```go
import "sync"

type TrafficLight struct {
    mu       sync.Mutex
    cond     *sync.Cond
    roadGreen int
    waiting  map[int]int
}

func Constructor() TrafficLight {
    tl := TrafficLight{
        roadGreen: 1,
        waiting:   map[int]int{1: 0, 2: 0},
    }
    tl.cond = sync.NewCond(&tl.mu)
    return tl
}

func (this *TrafficLight) CarArrived(
    carId int,
    roadId int,
    direction int,
    turnGreen func(),
    crossCar func(),
) {
    this.mu.Lock()
    defer this.mu.Unlock()

    if roadId != this.roadGreen {
        this.waiting[roadId]++
        for roadId != this.roadGreen {
            if this.waiting[this.roadGreen] == 0 {
                turnGreen()
                this.roadGreen = roadId
            }
            this.cond.Wait()
        }
        this.waiting[roadId]--
    }

    crossCar()
    this.cond.Broadcast()
}
```

**Go-specific notes:** The Go version uses a `sync.Cond` and `sync.Mutex` to coordinate access. The logic mirrors the Python version. `Broadcast()` is used instead of `notify_all()`.
