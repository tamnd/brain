[LeetCode Problem 1254](https://leetcode.com/problems/number-of-closed-islands/)

**Difficulty:** 🟡 Medium  
**Topics:** Array, Depth-First Search, Breadth-First Search, Union-Find, Matrix

## Solution

## Problem Understanding

The problem gives us a 2D grid where:

- `0` represents land
- `1` represents water

An island is a group of horizontally or vertically connected land cells. Diagonal connections do not count.

A closed island is an island that is completely surrounded by water on all four sides. The important detail is that none of the land cells in the island can touch the boundary of the grid. If even one cell of the island touches the outer border, then water could theoretically "escape" to the outside, so the island is not closed.

We need to return the total number of closed islands in the grid.

For example, consider a land region connected to the top edge of the grid. Even if most of the region is surrounded by water, it is still not closed because part of it touches the boundary.

The grid dimensions are at most `100 x 100`, which means the total number of cells is at most `10,000`. This size is small enough for graph traversal algorithms like Depth-First Search (DFS) or Breadth-First Search (BFS). Since each cell can be visited once, an `O(m * n)` solution is efficient enough.

Several edge cases are important:

- A land cell on the boundary can never belong to a closed island.
- Multiple disconnected islands may exist.
- The entire grid may be water.
- The entire grid may be land.
- Narrow corridors of land may connect an otherwise enclosed island to the boundary.
- A single-cell island surrounded by water should count as a closed island.

The problem guarantees valid input dimensions and that every cell contains either `0` or `1`.

## Approaches

### Brute Force Approach

A naive solution would examine every land cell and independently determine whether the island containing that cell is closed.

For each unvisited land cell, we could perform a DFS or BFS traversal across the entire connected component. During traversal, if any cell touches the boundary, we mark the island as open. Otherwise, it is closed.

This approach is correct because it fully explores each island and directly checks the defining property of a closed island.

However, a truly brute-force implementation might repeatedly revisit the same islands from different starting points if visitation tracking is not handled carefully. In the worst case, this could lead to repeated traversals of large regions, making the algorithm inefficient.

### Optimal Approach

The key insight is that any island connected to the boundary can never be closed.

Instead of checking every island independently from scratch, we can eliminate all boundary-connected land first.

The algorithm works in two phases:

1. Traverse all boundary cells.
2. Whenever a boundary land cell is found, flood-fill the entire connected component and mark it as visited or convert it to water.

After this cleanup step, every remaining land component must be completely enclosed by water.

We then scan the grid again. Every remaining unvisited land cell represents one closed island. We run DFS to mark the entire island and increment the answer.

This works because we remove all invalid islands before counting.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O((m * n)^2) | O(m * n) | May repeatedly traverse the same regions |
| Optimal | O(m * n) | O(m * n) | Each cell is processed at most once |

## Algorithm Walkthrough

1. Determine the dimensions of the grid.

We store the number of rows and columns because they are used repeatedly during traversal and boundary checks.
2. Define a DFS helper function.

The DFS function will:

- Stop if the coordinates are outside the grid.
- Stop if the current cell is water.
- Mark the current land cell as visited by converting it from `0` to `1`.
- Recursively visit the four neighboring cells.

We modify the grid in place to avoid maintaining a separate visited matrix.
3. Remove all boundary-connected islands.

We iterate through:

- The first and last row
- The first and last column

Whenever we encounter land (`0`), we run DFS from that cell.

This step floods all islands that touch the border, turning them into water.
4. Scan the interior of the grid.

After boundary cleanup, any remaining land must belong to a closed island.

We iterate through every cell:

- If the cell is land (`0`), we found a closed island.
- Increment the answer.
- Run DFS to mark the entire island as visited.
5. Return the total count.

By the time traversal finishes, every closed island has been counted exactly once.

### Why it works

The core invariant is:

> After removing all boundary-connected land regions, every remaining land component is completely enclosed by water.

Any island touching the boundary is invalid and removed during preprocessing. Since DFS explores all connected land cells, no remaining island can have a path to the border. Therefore, every remaining island is closed, and counting connected components gives the correct answer.

## Python Solution

```python
from typing import List

class Solution:
    def closedIsland(self, grid: List[List[int]]) -> int:
        rows = len(grid)
        cols = len(grid[0])

        def dfs(r: int, c: int) -> None:
            if r < 0 or r >= rows or c < 0 or c >= cols:
                return

            if grid[r][c] == 1:
                return

            grid[r][c] = 1

            dfs(r + 1, c)
            dfs(r - 1, c)
            dfs(r, c + 1)
            dfs(r, c - 1)

        # Remove boundary-connected islands
        for row in range(rows):
            if grid[row][0] == 0:
                dfs(row, 0)

            if grid[row][cols - 1] == 0:
                dfs(row, cols - 1)

        for col in range(cols):
            if grid[0][col] == 0:
                dfs(0, col)

            if grid[rows - 1][col] == 0:
                dfs(rows - 1, col)

        closed_islands = 0

        # Count remaining islands
        for row in range(rows):
            for col in range(cols):
                if grid[row][col] == 0:
                    closed_islands += 1
                    dfs(row, col)

        return closed_islands
```

The implementation starts by computing the grid dimensions. These values are reused throughout traversal operations.

The `dfs` helper function performs recursive flood fill. It immediately stops when:

- The coordinates go out of bounds
- The current cell is already water

When a land cell is found, it is converted into water by assigning `1`. This simultaneously marks the cell as visited and prevents revisiting it later.

The next section removes all islands touching the boundary. We traverse every boundary row and column and launch DFS whenever land is encountered. This eliminates all non-closed islands from consideration.

After cleanup, the grid contains only closed islands. The final nested loop scans every cell. Whenever land is found, we increment the result and perform DFS to consume the entire connected component.

Each island is visited exactly once, which guarantees linear time complexity.

## Go Solution

```go
func closedIsland(grid [][]int) int {
    rows := len(grid)
    cols := len(grid[0])

    var dfs func(int, int)

    dfs = func(r int, c int) {
        if r < 0 || r >= rows || c < 0 || c >= cols {
            return
        }

        if grid[r][c] == 1 {
            return
        }

        grid[r][c] = 1

        dfs(r+1, c)
        dfs(r-1, c)
        dfs(r, c+1)
        dfs(r, c-1)
    }

    // Remove boundary-connected islands
    for r := 0; r < rows; r++ {
        if grid[r][0] == 0 {
            dfs(r, 0)
        }

        if grid[r][cols-1] == 0 {
            dfs(r, cols-1)
        }
    }

    for c := 0; c < cols; c++ {
        if grid[0][c] == 0 {
            dfs(0, c)
        }

        if grid[rows-1][c] == 0 {
            dfs(rows-1, c)
        }
    }

    closedIslands := 0

    // Count remaining islands
    for r := 0; r < rows; r++ {
        for c := 0; c < cols; c++ {
            if grid[r][c] == 0 {
                closedIslands++
                dfs(r, c)
            }
        }
    }

    return closedIslands
}
```

The Go implementation follows the same logic as the Python version.

One notable difference is that Go does not support nested named functions directly, so the DFS function is declared as a variable first and then assigned a closure.

The grid is modified in place exactly as in Python, which avoids extra memory for visitation tracking.

Go slices are reference-like structures, so modifications inside DFS correctly update the original grid.

Integer overflow is not a concern because the grid dimensions are very small.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(m * n) | Every cell is visited at most once |
| Space | O(m * n) | Recursive DFS stack in worst case |

The algorithm performs two full scans of the grid, but each DFS permanently marks visited cells. Because each cell changes from land to water at most once, the total amount of work remains proportional to the number of cells.

The recursive DFS stack can grow to the size of an entire island in the worst case, which may require `O(m * n)` auxiliary space.

## Edge Cases

### Entire Grid is Land

If every cell is land, then every island touches the boundary. A naive implementation might incorrectly count the large connected component as closed.

The preprocessing step prevents this issue. All boundary cells trigger DFS, which floods the entire grid and removes all land before counting begins.

### Single-Cell Closed Island

A single land cell surrounded by water is still a valid closed island. Some implementations accidentally require an island to contain multiple cells.

The DFS-based approach treats every connected component equally, regardless of size. If the cell does not connect to the boundary, it is counted correctly.

### Narrow Path to Boundary

An island may appear enclosed except for a thin corridor connecting it to the edge.

For example:

```
[
 [1,0,1],
 [1,0,1],
 [1,0,1]
]
```

The center region is not closed because it connects to boundary land.

The boundary flood-fill correctly removes the entire connected component, including the narrow corridor, ensuring it is not counted.
