# LeetCode 1247 - Minimum Swaps to Make Strings Equal

## Problem Understanding

The problem gives us two strings, `s1` and `s2`, that have the same length and contain only the characters `'x'` and `'y'`. We are allowed to perform swap operations, but there is an important restriction: each swap must exchange one character from `s1` with one character from `s2`.

We are not allowed to swap characters within the same string.

The goal is to make both strings identical using the minimum number of swaps. If it is impossible to make the strings equal, we must return `-1`.

To understand the problem clearly, consider what it means for the strings to become equal. At every index `i`, we eventually need:

```
s1[i] == s2[i]
```

Any position where the characters already match does not require any work. The only important positions are mismatches.

Since the strings only contain `'x'` and `'y'`, there are only two possible mismatch types:

```
s1[i] = 'x', s2[i] = 'y'   -> "xy" mismatch
s1[i] = 'y', s2[i] = 'x'   -> "yx" mismatch
```

The constraints are relatively small:

- `1 <= length <= 1000`
- only two possible characters exist

This means we do not need sophisticated data structures or advanced optimization techniques. A linear scan is sufficient.

Several edge cases are important:

- The strings may already be equal.
- There may be an odd number of mismatches, making the task impossible.
- One mismatch type may appear more often than the other.
- A single `"xy"` and a single `"yx"` mismatch require two swaps, not one, which is easy to overlook.

Understanding how mismatch pairs behave is the key insight for solving this problem efficiently.

## Approaches

### Brute Force Approach

A brute force solution would try all possible swaps between the two strings and search for the minimum number of operations needed to make the strings equal.

At each step, we could:

1. Find all pairs `(i, j)` where we swap `s1[i]` with `s2[j]`
2. Generate the new string states
3. Continue searching until the strings become equal

This naturally leads to a graph search or BFS problem, where each state represents a pair of strings.

Although this approach eventually finds the optimal answer, it becomes extremely inefficient because the number of possible states grows exponentially. Even with pruning, the branching factor is large enough to make this impractical for strings of length up to 1000.

The brute force method is therefore far too slow.

### Optimal Greedy Observation

The key insight is that we never need to simulate swaps directly.

Only mismatched positions matter, and there are only two mismatch types:

```
xy mismatch
yx mismatch
```

Now observe the following:

- Two `"xy"` mismatches can be fixed in one swap.
- Two `"yx"` mismatches can also be fixed in one swap.
- One `"xy"` and one `"yx"` mismatch together require two swaps.

This completely determines the optimal answer.

Suppose:

```
xy_count = number of "xy" mismatches
yx_count = number of "yx" mismatches
```

Then:

- Every pair of `"xy"` mismatches contributes:

```
xy_count // 2 swaps
```

- Every pair of `"yx"` mismatches contributes:

```
yx_count // 2 swaps
```

- If one `"xy"` and one `"yx"` remain, they require exactly `2` more swaps.

If the total number of mismatches is odd, the transformation is impossible.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Exponential | Exponential | Explores all possible swap sequences |
| Optimal Greedy | O(n) | O(1) | Counts mismatch patterns directly |

## Algorithm Walkthrough

1. Initialize two counters:

- `xy_count` for positions where `s1[i] = 'x'` and `s2[i] = 'y'`
- `yx_count` for positions where `s1[i] = 'y'` and `s2[i] = 'x'`
2. Traverse both strings simultaneously.
3. For each index:

- If the characters match, do nothing.
- If the pair is `"xy"`, increment `xy_count`.
- If the pair is `"yx"`, increment `yx_count`.
4. After scanning the strings, check whether the total number of mismatches is odd:

```
(xy_count + yx_count) % 2 == 1
```

If it is odd, return `-1`.

1. Compute swaps contributed by same-type pairs:

```
xy_count // 2
yx_count // 2
```

Each pair requires one swap.

1. Check whether one `"xy"` and one `"yx"` mismatch remain:

```
xy_count % 2 == 1
```

If so, add `2` more swaps.

1. Return the final answer.

### Why it works

Each swap changes the mismatch structure in a predictable way. Two mismatches of the same type can always be resolved with one swap because the characters can be exchanged directly.

However, a leftover `"xy"` and `"yx"` mismatch cannot be fixed in a single operation because no direct swap simultaneously corrects both positions. This configuration always requires exactly two swaps.

Since every mismatch belongs to one of these two categories, and every optimal solution must resolve them in these minimal groups, the greedy counting method always produces the minimum number of swaps.

## Python Solution

```python
class Solution:
    def minimumSwap(self, s1: str, s2: str) -> int:
        xy_count = 0
        yx_count = 0

        for char1, char2 in zip(s1, s2):
            if char1 == 'x' and char2 == 'y':
                xy_count += 1
            elif char1 == 'y' and char2 == 'x':
                yx_count += 1

        total_mismatches = xy_count + yx_count

        if total_mismatches % 2 == 1:
            return -1

        swaps = (xy_count // 2) + (yx_count // 2)

        if xy_count % 2 == 1:
            swaps += 2

        return swaps
```

The implementation begins by counting the two mismatch types while scanning both strings together using `zip`.

Positions where the characters already match are ignored because they do not affect the answer.

After counting mismatches, the code checks whether the total mismatch count is odd. If it is, no sequence of swaps can make the strings equal, so the function immediately returns `-1`.

Next, the algorithm counts how many complete pairs of `"xy"` and `"yx"` mismatches exist. Each pair contributes one swap.

Finally, if both counters still have one leftover mismatch, two additional swaps are needed. This is detected by checking whether `xy_count` is odd.

The implementation follows the exact logic derived in the algorithm walkthrough and runs in linear time.

## Go Solution

```go
func minimumSwap(s1 string, s2 string) int {
    xyCount := 0
    yxCount := 0

    for i := 0; i < len(s1); i++ {
        if s1[i] == 'x' && s2[i] == 'y' {
            xyCount++
        } else if s1[i] == 'y' && s2[i] == 'x' {
            yxCount++
        }
    }

    totalMismatches := xyCount + yxCount

    if totalMismatches%2 == 1 {
        return -1
    }

    swaps := (xyCount / 2) + (yxCount / 2)

    if xyCount%2 == 1 {
        swaps += 2
    }

    return swaps
}
```

The Go implementation mirrors the Python logic closely.

Since Go strings are byte arrays internally, indexing with `s1[i]` and `s2[i]` works efficiently because the input only contains ASCII characters `'x'` and `'y'`.

Integer division in Go naturally performs floor division, so expressions like `xyCount / 2` behave exactly as required.

No additional slices or maps are needed, which keeps the space complexity constant.

## Edge Cases

One important edge case occurs when the strings are already equal. In this situation, there are no mismatches at all, so both counters remain zero. The algorithm correctly returns `0` because no swaps are required.

Another critical case happens when the total number of mismatches is odd. Since every swap affects two positions, mismatches must always be resolved in pairs. An odd mismatch count can never be fully eliminated. The implementation explicitly checks this condition and returns `-1`.

A subtle edge case involves exactly one `"xy"` mismatch and one `"yx"` mismatch. A naive implementation may incorrectly assume these can be fixed with one swap. In reality, they always require two swaps. The algorithm handles this by adding `2` when both counters have one leftover mismatch after pairing same-type mismatches.

A final important case is when all mismatches are of the same type, such as:

```
s1 = "xxxx"
s2 = "yyyy"
```

Every pair of mismatches can be solved independently with one swap, so the answer becomes `mismatch_count / 2`. The implementation naturally handles this through integer division.
