## Problem Understanding

The problem asks us to implement a **multi-threaded web crawler** that starts from a given URL `startUrl` and explores all links reachable under the **same hostname**. We can access links on a page using the `HtmlParser.getUrls(url)` method, which returns all URLs on that page. Our crawler must ensure it **does not visit the same URL twice** and should return all reachable URLs under the same hostname as `startUrl` in any order.

The input `startUrl` is a string representing a valid HTTP URL. The `HtmlParser` is an API interface simulating HTTP requests. URLs may contain multiple paths, but the hostname is the key for filtering which URLs to crawl. The hostname is the part between `http://` and the first `/` or end of string. For example, in `http://news.yahoo.com/news/topics`, the hostname is `news.yahoo.com`.

Constraints are important for scaling:

1. URLs array length ≤ 1000.
2. URL string length ≤ 300.
3. `getUrls(url)` is a blocking call but guaranteed to complete quickly (15ms), making single-threaded DFS/BFS potentially too slow.
4. No duplicates in the URL library.
5. Hostnames follow strict rules: 1-63 characters, only letters, digits, and `-`, cannot start or end with `-`.

Important edge cases:

- `startUrl` has no outgoing links.
- All URLs lead outside the hostname.
- URLs contain deep paths but share the same hostname.

The goal is to **efficiently crawl** URLs using **multi-threading**, respecting hostname constraints, and avoiding repeated visits.

## Approaches

### Brute Force Approach

A naive approach is to perform **single-threaded BFS or DFS** starting from `startUrl`. Keep a set of visited URLs, and for each URL, call `getUrls(url)` to get neighbors, adding them to the queue if the hostname matches. This guarantees correctness but will **time out** for large datasets since HTTP requests are blocking, and visiting 1000 URLs sequentially multiplies latency.

### Optimized Approach

The optimized approach uses **multi-threading**. Instead of processing URLs sequentially, we can spawn threads for each URL that has not been visited. We maintain a **thread-safe visited set** (e.g., with locks in Python or sync.Map in Go) to avoid duplicate visits. Threads concurrently fetch URLs using `getUrls` and recursively submit new URLs for crawling.

Key points:

1. Use **concurrent queue or task submission** to manage URLs.
2. Protect the visited set from race conditions.
3. Only enqueue URLs with the same hostname.

This approach reduces waiting time because HTTP requests are blocking, and multiple threads can request URLs concurrently, improving overall runtime.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force (Single-threaded BFS/DFS) | O(N + E) | O(N) | Visits every URL sequentially; may exceed time limits due to blocking calls |
| Optimal (Multi-threaded BFS/DFS) | O(N + E) | O(N) | Concurrently visits URLs; reduces total waiting time; requires thread-safe visited set |

## Algorithm Walkthrough

1. Parse the **hostname** from `startUrl` by splitting the URL string at `//` and `/`.
2. Initialize a **thread-safe set** for visited URLs and a **queue** (or thread pool executor) to manage URLs to visit.
3. Define a **crawl function** that takes a URL:

- If the URL is already visited, return immediately.
- Otherwise, mark it as visited.
- Call `htmlParser.getUrls(url)` to retrieve linked URLs.
- For each linked URL, if its hostname matches, submit it for crawling.
4. Start the first crawl with `startUrl`.
5. Wait for all threads/tasks to finish.
6. Return all visited URLs.

**Why it works:** The algorithm ensures each URL is visited **exactly once**, filtering only by the hostname. Threading allows multiple URLs to be fetched simultaneously, preventing sequential blocking delays. The visited set ensures no duplicates and prevents infinite recursion.

## Python Solution

```python
from urllib.parse import urlparse
from concurrent.futures import ThreadPoolExecutor
from typing import List

class Solution:
    def crawl(self, startUrl: str, htmlParser: 'HtmlParser') -> List[str]:
        visited = set()
        hostname = urlparse(startUrl).netloc

        def dfs(url):
            if url in visited:
                return
            visited.add(url)
            for next_url in htmlParser.getUrls(url):
                if urlparse(next_url).netloc == hostname:
                    dfs(next_url)

        # Using ThreadPoolExecutor for multi-threading
        with ThreadPoolExecutor(max_workers=8) as executor:
            future = executor.submit(dfs, startUrl)
            future.result()  # wait for the initial crawl to finish

        return list(visited)
```

**Explanation:**

1. `urlparse` extracts the hostname.
2. `visited` ensures we only crawl each URL once.
3. `dfs` recursively visits URLs under the same hostname.
4. `ThreadPoolExecutor` allows multi-threaded execution; we submit the initial crawl.
5. Multi-threading improves performance for blocking `getUrls` calls.

## Go Solution

```go
package main

import (
    "net/url"
    "sync"
)

type HtmlParser struct{}

func (hp *HtmlParser) getUrls(s string) []string { return []string{} }

func crawl(startUrl string, htmlParser *HtmlParser) []string {
    hostname, _ := url.Parse(startUrl)
    host := hostname.Host

    visited := sync.Map{}
    var wg sync.WaitGroup

    var dfs func(string)
    dfs = func(current string) {
        defer wg.Done()
        if _, ok := visited.LoadOrStore(current, true); ok {
            return
        }
        for _, next := range htmlParser.getUrls(current) {
            parsed, err := url.Parse(next)
            if err != nil {
                continue
            }
            if parsed.Host == host {
                wg.Add(1)
                go dfs(next)
            }
        }
    }

    wg.Add(1)
    go dfs(startUrl)
    wg.Wait()

    result := []string{}
    visited.Range(func(key, value any) bool {
        result = append(result, key.(string))
        return true
    })
    return result
}
```

**Explanation:**

- `sync.Map` provides thread-safe visited storage.
- `sync.WaitGroup` waits for all goroutines to finish.
- Recursive `dfs` is executed as goroutines for each new URL.
- Hostname filtering ensures only URLs under the same hostname are crawled.
