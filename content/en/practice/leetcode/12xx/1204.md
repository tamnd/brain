## Solution

## Problem Understanding

The problem is asking us to determine which person is the **last to successfully board a bus** without exceeding a given weight limit of 1000 kilograms. We are given a database table `Queue` representing people waiting to board the bus, along with their unique `person_id`, `person_name`, `weight`, and the `turn` in which they attempt to board. The `turn` column dictates the boarding order, with `turn=1` for the first person and `turn=n` for the last.

The key challenge is **accumulating the total weight as each person boards**, and stopping when adding the next person's weight would exceed the bus capacity. The output should be the `person_name` of the last person who successfully boarded without surpassing the limit.

Constraints ensure that the first person in line will always be able to board, so we never have to handle the case of an empty bus from the start. Additionally, each person boards individually in the order of `turn`. The weight limit is strict, meaning as soon as the cumulative weight would exceed 1000 kg, no further people can board.

Important edge cases to consider include the scenario where exactly the bus limit is reached, the next person slightly exceeds it, and situations with only one person in the queue. The problem guarantees no negative weights and unique turns, simplifying order handling.

## Approaches

### Brute Force

A brute-force approach involves **sorting the queue by `turn`**, iterating over each person, summing their weights cumulatively, and checking after each addition if the sum exceeds 1000 kg. The last person whose addition does not exceed the limit is returned. While this approach works for small input sizes, it requires scanning the entire table row by row, which may not be efficient for very large datasets in SQL if the table is huge.

### Optimal Approach

The key insight is that **only the cumulative sum of weights in `turn` order matters**, and SQL provides functions like `SUM() OVER (ORDER BY turn)` to calculate running totals efficiently. Using this approach, we compute a running total of weight for each person, and simply select the last person where the cumulative sum is less than or equal to the bus limit. This avoids manually iterating and makes the solution fully declarative and optimized for SQL engines.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(n) | Sort by turn and iterate to accumulate weight, returning last person under limit |
| Optimal | O(n) | O(n) | Use SQL window function to calculate cumulative sum and filter |

## Algorithm Walkthrough

1. **Order the queue by `turn`**. This ensures we process boarding in the correct sequence. The `turn` column guarantees a natural order.
2. **Calculate the cumulative weight** of people as they board. This uses a running sum, which tells us the total weight on the bus after each person attempts to board.
3. **Identify all persons whose cumulative weight is within the bus limit**. Any person whose cumulative weight exceeds 1000 kg cannot board.
4. **Return the `person_name` of the last person who fits** within the weight limit. Using `ORDER BY turn DESC` ensures we pick the latest person who successfully boards.

**Why it works:** The invariant is that the cumulative sum always reflects the total weight after each person boards. By checking against the bus limit at each step, we are guaranteed to correctly identify the last person who fits without exceeding the limit. SQL window functions ensure correctness without explicit loops.

## Python Solution

```python
import sqlite3
from typing import Optional

def last_person_to_fit(conn: sqlite3.Connection) -> Optional[str]:
    query = """
    WITH CumulativeWeight AS (
        SELECT person_name, SUM(weight) OVER (ORDER BY turn) AS total_weight, turn
        FROM Queue
    )
    SELECT person_name
    FROM CumulativeWeight
    WHERE total_weight <= 1000
    ORDER BY turn DESC
    LIMIT 1;
    """
    cur = conn.cursor()
    cur.execute(query)
    result = cur.fetchone()
    return result[0] if result else None
```

The implementation defines a common table expression `CumulativeWeight` to compute the running sum of weights ordered by `turn`. We then filter rows where the cumulative weight does not exceed 1000 kg, order them by `turn` descending, and return the first result. This directly implements the algorithm steps efficiently in SQL.

## Go Solution

```go
package main

import (
    "database/sql"
    _ "github.com/mattn/go-sqlite3"
    "log"

func LastPersonToFit(db *sql.DB) (string, error) {
    query := `
    WITH CumulativeWeight AS (
        SELECT person_name, SUM(weight) OVER (ORDER BY turn) AS total_weight, turn
        FROM Queue
    )
    SELECT person_name
    FROM CumulativeWeight
    WHERE total_weight <= 1000
    ORDER BY turn DESC
    LIMIT 1;
    `
    var name string
    err := db.QueryRow(query).Scan(&name)
    if err != nil {
        return "", err
    }
    return name, nil
}
```

## Edge Cases

One edge case is a **single person whose weight is exactly 1000**. This is valid since the first person is guaranteed not to exceed the limit. Our implementation handles this because the cumulative sum includes the first person and passes the `<= 1000` filter.

Another edge case is **the cumulative sum reaching exactly 1000 at some middle person**. This could be a source of off-by-one errors if the implementation uses `<` instead of `<=`. The use of `<= 1000` ensures correctness, including the person who brings the total exactly to 1000.

A third edge case is **people with small weights following someone who reaches near the limit**, where subsequent people cannot board. The algorithm correctly stops including people once the cumulative sum exceeds the limit, and always returns the last person who fits, handling the early termination correctly.
