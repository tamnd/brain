## Solution

## Problem Understanding

This problem is an interactive divide and conquer problem where we are not given direct access to the ship coordinates. Instead, we can only query the hidden grid through the API:

```
Sea.hasShips(topRight, bottomLeft)
```

This function tells us whether there is at least one ship inside the rectangle bounded by `bottomLeft` and `topRight`, inclusive.

The goal is to count exactly how many ships exist inside a given rectangle.

The important constraint is that there are at most 10 ships inside the queried region. Another critical constraint is that we are limited to at most 400 API calls. Because the coordinate range can go up to 1000 by 1000, iterating through every point in the rectangle would be extremely inefficient.

The rectangle is defined by two points:

- `bottomLeft = [x1, y1]`
- `topRight = [x2, y2]`

Both boundaries are inclusive. Ships exist only at integer coordinates, and each coordinate can contain at most one ship.

The output should be the total number of ships located within the rectangle.

Several edge cases are important:

- The rectangle may contain no ships at all.
- The rectangle may collapse into a single point during recursion.
- Some recursive regions may become invalid, such as when `bottomLeft.x > topRight.x`.
- Ships may lie exactly on rectangle boundaries, and those must still be counted because the boundaries are inclusive.
- Since there are at most 10 ships, an efficient solution should aggressively prune empty regions instead of exploring every coordinate.

## Approaches

### Brute Force Approach

The most straightforward approach is to check every integer coordinate inside the rectangle individually.

For every point `(x, y)` in the rectangle, we could call:

```
hasShips(Point(x, y), Point(x, y))
```

If the API returns `true`, then there is a ship at that exact coordinate.

This approach is correct because every coordinate is examined independently, and the problem guarantees at most one ship per point.

However, this method is far too slow. The coordinate range can be as large as `1000 x 1000`, meaning up to one million API calls. Since the judge allows only 400 calls, this solution immediately fails.

### Optimal Divide and Conquer Approach

The key observation is that `hasShips()` can eliminate large empty regions with a single query.

Instead of checking individual cells, we recursively divide the current rectangle into smaller subrectangles.

For each rectangle:

- If the rectangle is invalid, return 0.
- If `hasShips()` returns `false`, the entire region is empty, so stop exploring it.
- If the rectangle represents a single point and `hasShips()` is `true`, then exactly one ship exists there.
- Otherwise, split the rectangle into four smaller quadrants and recursively count ships in each.

This works efficiently because most regions are discarded immediately after one API call. Since there are at most 10 ships, the recursion tree remains relatively small.

The divide and conquer structure allows us to zoom into only the regions that actually contain ships.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(W × H) | O(1) | Checks every coordinate individually |
| Optimal | O(S × log W × log H) approximately | O(log W + log H) | Recursively subdivides only regions containing ships |

Here:

- `W` is rectangle width
- `H` is rectangle height
- `S` is the number of ships, at most 10

## Algorithm Walkthrough

1. Start with the full rectangle defined by `bottomLeft` and `topRight`.
2. First check whether the rectangle is valid. If:

- `bottomLeft.x > topRight.x`, or
- `bottomLeft.y > topRight.y`

then the rectangle does not exist, so return `0`.
3. Call `hasShips(topRight, bottomLeft)`.

This is the pruning step. If the API returns `false`, there are no ships anywhere inside this rectangle, so we immediately return `0` without further recursion.
4. If the rectangle represents a single point:

- `bottomLeft.x == topRight.x`
- `bottomLeft.y == topRight.y`

then this rectangle contains exactly one coordinate.

Since we already know `hasShips()` returned `true`, there must be exactly one ship here. Return `1`.
5. Otherwise, split the rectangle into four quadrants using the midpoint:

```
midX = (left + right) // 2
midY = (bottom + top) // 2
```
6. Recursively search the four subrectangles:

- Bottom-left quadrant
- Bottom-right quadrant
- Top-left quadrant
- Top-right quadrant
7. Sum the results from all four recursive calls.
8. Return the total ship count.

### Why it works

The algorithm maintains the invariant that every recursive call operates on a rectangle that could potentially contain ships.

Whenever `hasShips()` returns `false`, we know with certainty that no ship exists in that region, so pruning is safe.

Whenever recursion reaches a single point with `hasShips() == true`, exactly one ship must exist there because each coordinate contains at most one ship.

Since every valid ship-containing region is eventually subdivided down to individual points, all ships are counted exactly once.

## Python Solution

```python
# """
# This is Sea's API interface.
# You should not implement it, or speculate about its implementation
# """
# class Sea:
#     def hasShips(self, topRight: 'Point', bottomLeft: 'Point') -> bool:
#
# class Point:
#     def __init__(self, x: int, y: int):
#         self.x = x
#         self.y = y

class Solution:
    def countShips(
        self,
        sea: 'Sea',
        topRight: 'Point',
        bottomLeft: 'Point'
    ) -> int:

        def dfs(tr: 'Point', bl: 'Point') -> int:
            # Invalid rectangle
            if bl.x > tr.x or bl.y > tr.y:
                return 0

            # No ships in this region
            if not sea.hasShips(tr, bl):
                return 0

            # Single point containing a ship
            if tr.x == bl.x and tr.y == bl.y:
                return 1

            mid_x = (bl.x + tr.x) // 2
            mid_y = (bl.y + tr.y) // 2

            total = 0

            # Bottom-left
            total += dfs(
                Point(mid_x, mid_y),
                Point(bl.x, bl.y)
            )

            # Bottom-right
            total += dfs(
                Point(tr.x, mid_y),
                Point(mid_x + 1, bl.y)
            )

            # Top-left
            total += dfs(
                Point(mid_x, tr.y),
                Point(bl.x, mid_y + 1)
            )

            # Top-right
            total += dfs(
                Point(tr.x, tr.y),
                Point(mid_x + 1, mid_y + 1)
            )

            return total

        return dfs(topRight, bottomLeft)
```

The implementation follows the divide and conquer strategy directly.

The helper function `dfs()` recursively explores a rectangle.

The first condition handles invalid rectangles. These can appear naturally when splitting regions near boundaries.

The second condition performs the critical pruning step. If the API says there are no ships inside the rectangle, recursion stops immediately.

The third condition handles the base case where the rectangle has shrunk to a single coordinate. Since `hasShips()` already confirmed the presence of a ship, we return `1`.

The midpoint calculation divides the rectangle into four smaller quadrants. Using four-way subdivision is natural in two-dimensional geometry and ensures every coordinate belongs to exactly one subrectangle.

Finally, the recursive counts are summed and returned.

## Go Solution

```go
/**
 * // This is Sea's API interface.
 * // You should not implement it, or speculate about its implementation
 * type Sea struct {
 *     func hasShips(topRight, bottomLeft []int) bool {}
 * }
 */

func countShips(sea Sea, topRight, bottomLeft []int) int {
    var dfs func(tr, bl []int) int

    dfs = func(tr, bl []int) int {
        // Invalid rectangle
        if bl[0] > tr[0] || bl[1] > tr[1] {
            return 0
        }

        // No ships in this region
        if !sea.hasShips(tr, bl) {
            return 0
        }

        // Single point containing a ship
        if tr[0] == bl[0] && tr[1] == bl[1] {
            return 1
        }

        midX := (bl[0] + tr[0]) / 2
        midY := (bl[1] + tr[1]) / 2

        total := 0

        // Bottom-left
        total += dfs(
            []int{midX, midY},
            []int{bl[0], bl[1]},
        )

        // Bottom-right
        total += dfs(
            []int{tr[0], midY},
            []int{midX + 1, bl[1]},
        )

        // Top-left
        total += dfs(
            []int{midX, tr[1]},
            []int{bl[0], midY + 1},
        )

        // Top-right
        total += dfs(
            []int{tr[0], tr[1]},
            []int{midX + 1, midY + 1},
        )

        return total
    }

    return dfs(topRight, bottomLeft)
}
```

The Go implementation mirrors the Python logic closely.

Slices are used instead of custom `Point` objects because the Go stub represents coordinates as integer slices.

The recursive helper is implemented as a closure assigned to `dfs`.

There are no integer overflow concerns because coordinates are bounded by 1000.

The invalid rectangle checks are especially important in Go because recursive subdivision may generate ranges where the lower bound exceeds the upper bound.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(S × log W × log H) approximately | Only regions containing ships continue recursively |
| Space | O(log W + log H) | Recursion depth from repeatedly subdividing the rectangle |

The exact number of API calls depends on how the ships are distributed. However, because there are at most 10 ships, the recursion tree remains small.

Each recursive level halves the search dimensions, giving logarithmic depth in both width and height.

Most importantly, empty regions terminate immediately after a single `hasShips()` query, which keeps the total API calls well below the 400-call limit.

## Edge Cases

One important edge case occurs when a recursive subdivision creates an invalid rectangle. For example, after splitting a rectangle, one quadrant may have `bottomLeft.x > topRight.x`. Without checking for invalid ranges, recursion could continue indefinitely or generate incorrect API calls. The implementation explicitly checks rectangle validity at the start of every recursive call and immediately returns `0`.

Another important case is a single-point rectangle. Once recursion narrows the search down to one coordinate, the algorithm must stop subdividing further. If `hasShips()` returns `true` for that point, exactly one ship exists there because the problem guarantees at most one ship per coordinate. The implementation correctly returns `1` in this situation.

A third subtle edge case involves ships located exactly on rectangle boundaries. Since the API treats boundaries as inclusive, failing to partition rectangles carefully could either double count or miss ships on the midpoint lines. The implementation avoids overlap by using:

- `midX` in the left regions
- `midX + 1` in the right regions
- `midY` in the lower regions
- `midY + 1` in the upper regions

This guarantees every coordinate belongs to exactly one subrectangle.

Another important scenario is a very large empty rectangle. A naive solution would still explore many subdivisions. In contrast, this implementation immediately terminates after a single `hasShips()` call returns `false`, which is the key optimization that keeps the API usage under the limit.
