## Solution

## Problem Understanding

The problem gives us a positive integer `n` and asks us to compute two separate values using its digits:

1. The product of all digits
2. The sum of all digits

After computing both values, we return:

```
product of digits - sum of digits
```

For example, if `n = 234`, the digits are `2`, `3`, and `4`.

The product is:

```
2 * 3 * 4 = 24
```

The sum is:

```
2 + 3 + 4 = 9
```

The final answer is:

```
24 - 9 = 15
```

The input is guaranteed to satisfy:

```
1 <= n <= 10^5
```

This constraint tells us several important things. First, the number is always positive, so we never need to handle negative integers. Second, the maximum value has at most 6 digits, which means any reasonable digit-processing approach will be efficient enough.

The most natural way to solve the problem is to repeatedly extract digits using modulo and integer division operations:

- `digit = n % 10` gives the last digit
- `n //= 10` removes the last digit

Important edge cases include numbers containing zeros, because multiplying by zero immediately makes the product zero. Single-digit numbers are also important because the product and sum are both equal to the digit itself, producing a result of zero. The problem guarantees that the input is never zero itself, so we do not need special handling for an empty digit sequence.

## Approaches

A brute-force approach would first convert the integer into a string, iterate through every character, convert each character back into an integer digit, store all digits in a list, and then perform separate passes to compute the product and the sum.

This works correctly because every digit is explicitly extracted and processed. However, it uses extra memory to store the digits and performs more work than necessary because the digits are traversed multiple times.

A better solution processes the digits directly using mathematical operations. Instead of storing digits, we extract one digit at a time using modulo and integer division. While traversing the digits, we simultaneously update both the running product and the running sum.

The key insight is that we never actually need to keep the digits after processing them. Each digit contributes independently to the final product and sum, so we can process digits one at a time in constant space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(d) | O(d) | Converts number to string and stores digits |
| Optimal | O(d) | O(1) | Processes digits directly using math operations |

Here, `d` represents the number of digits in `n`.

## Algorithm Walkthrough

1. Initialize two variables:

- `product = 1`
- `digit_sum = 0`

The product starts at `1` because multiplying by `1` does not change the value. The sum starts at `0` because it is the additive identity.
2. Repeatedly process digits while `n > 0`.

Each iteration extracts the last digit of the current number.
3. Extract the last digit using:

```
digit = n % 10
```

The modulo operator gives the remainder after division by 10, which is exactly the final digit.
4. Update the running product:

```
product *= digit
```

Every digit contributes multiplicatively to the final product.
5. Update the running sum:

```
digit_sum += digit
```

Every digit also contributes additively to the final sum.
6. Remove the processed digit using integer division:

```
n //= 10
```

This shifts the number right by one decimal place.
7. Continue until all digits have been processed.
8. Return:

```
product - digit_sum
```

### Why it works

At every iteration, exactly one digit is extracted and incorporated into both the product and the sum. Integer division removes the processed digit, ensuring that every digit is processed once and only once. Since multiplication and addition are both associative operations, the order of processing does not affect correctness. After all digits are processed, the accumulated values exactly match the required product and sum.

## Python Solution

```python
class Solution:
    def subtractProductAndSum(self, n: int) -> int:
        product = 1
        digit_sum = 0

        while n > 0:
            digit = n % 10

            product *= digit
            digit_sum += digit

            n //= 10

        return product - digit_sum
```

The implementation follows the algorithm directly.

The variables `product` and `digit_sum` maintain the running calculations for all processed digits. The `while` loop continues until every digit has been removed from the number.

Inside the loop, `n % 10` extracts the current last digit. That digit is immediately used to update both running values. Then `n //= 10` removes the processed digit so the next iteration can continue with the remaining digits.

Finally, the method returns the difference between the computed product and sum.

## Go Solution

```go
func subtractProductAndSum(n int) int {
    product := 1
    digitSum := 0

    for n > 0 {
        digit := n % 10

        product *= digit
        digitSum += digit

        n /= 10
    }

    return product - digitSum
}
```

The Go implementation is nearly identical to the Python version. The main syntactic differences are variable declarations using `:=` and the use of Go's `for` loop in place of Python's `while` loop.

Integer operations behave safely here because the constraints are very small. Even the maximum possible product of digits remains well within Go's integer range.

## Edge Cases

A very important edge case is a single-digit number such as `5`. In this situation, the product and the sum are both equal to the same digit. A buggy implementation might incorrectly initialize the product to `0`, which would always produce an incorrect result. Initializing the product to `1` ensures correctness.

Another critical edge case is numbers containing zeros, such as `101` or `100000`. Since multiplication by zero immediately forces the product to zero, implementations that accidentally skip zeros or terminate early would fail. This implementation processes every digit individually, including zeros, so the product becomes zero exactly when it should.

A third edge case is the maximum constraint range, such as `99999`. Although the input is still small, this case ensures that repeated multiplication and addition behave correctly across multiple iterations. The implementation handles this naturally because each digit is processed independently and all intermediate values remain within normal integer limits.
