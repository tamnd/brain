# LeetCode 1252 - Cells with Odd Values in a Matrix

## Problem Understanding

The problem gives us an `m x n` matrix where every cell initially contains `0`. We are also given a list called `indices`, where each element is a pair `[ri, ci]`. For every pair, we must perform two operations simultaneously. First, increment every value in row `ri` by `1`. Second, increment every value in column `ci` by `1`.

After processing all operations, we must count how many cells in the matrix contain odd values.

In simpler terms, every operation affects one entire row and one entire column. Since cells at the intersection of the selected row and column are incremented twice during the same operation, their parity may change differently than other cells. The final goal is not to return the matrix itself, but only the total number of cells whose final value is odd.

The constraints are relatively small. Both `m` and `n` are at most `50`, and the number of operations is at most `100`. This means even a direct matrix simulation would be fast enough in practice. However, the follow-up asks for a more efficient solution using only `O(m + n)` extra space, which suggests there is an important mathematical observation about parity that avoids building the full matrix.

One important edge case occurs when the same row or column is incremented multiple times. Since we only care about whether values are odd or even, repeated increments matter only in terms of parity. Another edge case is when all increments cancel out and every cell ends up even. We also need to correctly handle very small matrices, such as `1 x 1`.

## Approaches

### Brute Force Simulation

The most direct approach is to explicitly build the matrix and simulate every operation exactly as described.

We initialize an `m x n` matrix filled with zeros. For every pair `[ri, ci]` in `indices`, we iterate through all columns in row `ri` and increment those cells by `1`. Then we iterate through all rows in column `ci` and increment those cells by `1`.

After processing all operations, we traverse the entire matrix and count how many cells contain odd values.

This method is straightforward and guarantees correctness because it directly follows the problem statement. However, it performs unnecessary work. Every operation touches an entire row and column, even though the only information we ultimately care about is whether each cell is odd or even.

### Optimal Observation

The key insight is that the exact numeric values do not matter. Only parity matters.

A cell `(r, c)` is incremented once every time row `r` is selected, and once every time column `c` is selected. Therefore, the final value at a cell depends entirely on:

- how many times its row was incremented
- how many times its column was incremented

The final value is:

```
row_increments[r] + col_increments[c]
```

A cell is odd if and only if the sum above is odd.

This means:

- odd row + even column = odd
- even row + odd column = odd

So a cell becomes odd exactly when one of its row or column counts is odd, and the other is even.

Instead of constructing the matrix, we only need to track the parity of each row and column.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(indices.length × (m + n) + m × n) | O(m × n) | Explicitly simulates the matrix |
| Optimal | O(indices.length + m + n) | O(m + n) | Tracks only row and column parity |

## Algorithm Walkthrough

1. Create two arrays:

- `row_parity` of size `m`
- `col_parity` of size `n`

These arrays track how many times each row and column has been incremented. Since we only care about odd or even counts, we can store parity directly.
2. Process every operation `[ri, ci]`.

For each operation:

- Toggle the parity of row `ri`
- Toggle the parity of column `ci`

Toggling can be done with XOR:

```
parity ^= 1
```

This works because:

- even + 1 becomes odd
- odd + 1 becomes even
3. Count how many rows have odd parity.

Let this count be `odd_rows`.
4. Count how many columns have odd parity.

Let this count be `odd_cols`.
5. Compute the total number of odd cells.

A cell is odd if:

- its row is odd and its column is even
- its row is even and its column is odd

Therefore:

```
odd_rows * (n - odd_cols)
+
(m - odd_rows) * odd_cols
```
6. Return the computed result.

### Why it works

Each cell's final value equals the number of times its row was incremented plus the number of times its column was incremented. A sum is odd only when exactly one operand is odd. By counting how many rows and columns have odd parity, we can mathematically determine how many cells end up odd without ever constructing the matrix.

## Python Solution

```python
from typing import List

class Solution:
    def oddCells(self, m: int, n: int, indices: List[List[int]]) -> int:
        row_parity = [0] * m
        col_parity = [0] * n

        for row, col in indices:
            row_parity[row] ^= 1
            col_parity[col] ^= 1

        odd_rows = sum(row_parity)
        odd_cols = sum(col_parity)

        return odd_rows * (n - odd_cols) + (m - odd_rows) * odd_cols
```

The implementation begins by creating two parity arrays. Each position stores either `0` for even or `1` for odd.

As we process each operation, we toggle the parity using XOR. This is more efficient than storing full increment counts because only odd versus even matters.

After processing all operations, we count how many rows and columns are odd. Once we know these counts, we apply the mathematical formula directly instead of iterating through the matrix.

The formula works because:

- every odd row combined with an even column creates an odd cell
- every even row combined with an odd column also creates an odd cell

## Go Solution

```go
func oddCells(m int, n int, indices [][]int) int {
    rowParity := make([]int, m)
    colParity := make([]int, n)

    for _, index := range indices {
        row := index[0]
        col := index[1]

        rowParity[row] ^= 1
        colParity[col] ^= 1
    }

    oddRows := 0
    for _, value := range rowParity {
        oddRows += value
    }

    oddCols := 0
    for _, value := range colParity {
        oddCols += value
    }

    return oddRows*(n-oddCols) + (m-oddRows)*oddCols
}
```

The Go implementation follows the same logic as the Python version. Slices are used for the row and column parity arrays. Since all calculations remain small under the given constraints, integer overflow is not a concern. Go arrays are initialized with zero values automatically, which conveniently represents even parity.

- `odd_cols = 0`

6

## Edge Cases

One important edge case occurs when the same row and column pair appears multiple times. A naive implementation might incorrectly assume increments always increase odd counts. In reality, parity flips each time. Performing the same operation twice returns affected cells to even parity. The implementation handles this correctly by using XOR toggling.

Another important edge case is a `1 x 1` matrix. In this scenario, incrementing row `0` and column `0` affects the same single cell twice during each operation. Since the cell increases by `2`, it always remains even. The algorithm correctly captures this because both the row and column parity become odd, producing an even sum.

A third edge case involves matrices where every row and every column end up with odd parity. In this situation, every cell becomes even because odd plus odd equals even. The mathematical formula naturally handles this case through:

```
odd_rows * even_cols + even_rows * odd_cols
```

If there are no even rows or columns, the result becomes zero automatically.
