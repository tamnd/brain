## Solution

## Problem Understanding

The problem asks us to implement a web crawler that starts from a given URL (`startUrl`) and retrieves all URLs that belong to the same hostname as the starting URL. The `HtmlParser` interface is provided to fetch the URLs from a given webpage. The input `startUrl` is guaranteed to be one of the URLs in the website, and all URLs use the HTTP protocol with no ports specified. The output should be a list of all URLs that share the same hostname, and the order does not matter.

Key points to note include: the crawler must **avoid revisiting URLs**, only explore URLs under the **same hostname**, and treat URLs with or without a trailing slash as distinct. Given the constraints (`urls.length` up to 1000), the problem can be solved efficiently with either **DFS** or **BFS** traversal while maintaining a `visited` set to prevent cycles.

Important edge cases include: the start URL having no links, all links leading to external hostnames, URLs differing only by a trailing slash, and a graph with cycles.

## Approaches

A **brute-force approach** would involve crawling all URLs returned by `HtmlParser.getUrls` recursively without filtering by hostname until every URL in the entire set is visited. This guarantees completeness but is unnecessary and inefficient because we would explore external hostnames that will never be part of the final result. For `n` URLs, if the web is densely connected, this could lead to repeatedly fetching irrelevant URLs.

The **optimal approach** leverages the hostname filter. By extracting the hostname from `startUrl`, we can crawl only URLs that match this hostname. We use a `visited` set to ensure we do not revisit URLs. Either **DFS** or **BFS** can be applied here; BFS may be slightly better for breadth exploration, but DFS is also straightforward to implement recursively. The critical insight is that by filtering URLs by hostname and maintaining a visited set, we ensure that the crawler does not fetch irrelevant URLs or enter infinite loops caused by cycles.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n + m) | O(n) | Recursively crawl all URLs, including external hostnames; may fetch irrelevant URLs |
| Optimal | O(n + m) | O(n) | Only crawl URLs with the same hostname; visited set prevents revisiting URLs |

Here, `n` is the number of URLs, and `m` is the total number of edges (links between pages).

## Algorithm Walkthrough

1. Extract the hostname from `startUrl` using string splitting. The hostname is the substring after `http://` and before the next `/` or end of string.
2. Initialize a `visited` set to store URLs that have already been crawled.
3. Define a recursive DFS function that takes a URL:

1. Add the current URL to `visited`.
2. Fetch all URLs from `HtmlParser.getUrls(url)`.
3. For each URL:

1. Check if the hostname matches the start URL's hostname.
2. If it matches and it is not in `visited`, recursively crawl this URL.
4. Call the DFS function starting with `startUrl`.
5. After traversal, return all URLs in `visited` as the result.

**Why it works:** The algorithm maintains a visited set to avoid cycles, and the hostname filter ensures only relevant URLs are crawled. Recursively visiting each unvisited matching URL guarantees that all URLs under the same hostname are captured.

## Python Solution

```python
from typing import List
from urllib.parse import urlparse

class Solution:
    def crawl(self, startUrl: str, htmlParser: 'HtmlParser') -> List[str]:
        visited = set()
        hostname = urlparse(startUrl).hostname

        def dfs(url: str):
            if url in visited:
                return
            visited.add(url)
            for next_url in htmlParser.getUrls(url):
                if urlparse(next_url).hostname == hostname:
                    dfs(next_url)

        dfs(startUrl)
        return list(visited)
```

The solution begins by extracting the hostname using `urlparse`, which handles URL parsing reliably. The `dfs` function recursively explores URLs, adding them to the `visited` set and only following links under the same hostname. Finally, the `visited` set is converted to a list to return all crawled URLs.

## Go Solution

```go
package main

import (
    "net/url"
)

type HtmlParser struct {
    GetUrls func(string) []string
}

func crawl(startUrl string, htmlParser HtmlParser) []string {
    visited := make(map[string]bool)
    parsedUrl, _ := url.Parse(startUrl)
    hostname := parsedUrl.Host

    var dfs func(string)
    dfs = func(u string) {
        if visited[u] {
            return
        }
        visited[u] = true
        for _, nextUrl := range htmlParser.GetUrls(u) {
            parsedNext, err := url.Parse(nextUrl)
            if err == nil && parsedNext.Host == hostname {
                dfs(nextUrl)
            }
        }
    }

    dfs(startUrl)

    result := make([]string, 0, len(visited))
    for url := range visited {
        result = append(result, url)
    }
    return result
}
```

In Go, we use the `net/url` package to extract the hostname. The `visited` map prevents revisiting URLs. The DFS recursion works similarly to Python, and finally, we convert the map keys to a slice for the result.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n + m) | Each URL is visited once, and each edge (link) is processed once in DFS |
| Space | O(n) | `visited` set stores each URL, recursion stack can go up to n in depth |

The time complexity is linear with respect to the number of URLs and links. Space complexity is dominated by the visited set and recursion stack.

## Edge Cases

1. **Single page with no links:** The crawler should return only the `startUrl`. The visited set handles this automatically by adding the start URL and making no further calls.
2. **URLs differing only by trailing slash:** Two URLs like `http://example.com` and `http://example.com/` are considered distinct. By adding URLs to a set or map exactly as returned by `getUrls`, the crawler treats them as separate.
3. **Cyclic links:** A page may link back to a previously visited page. Without a visited set, this would cause infinite recursion. The implementation marks URLs as visited immediately upon first visit, preventing revisiting.
4. **All links external:** If `startUrl` points to a page with links only to external hostnames, the crawler returns only the start URL. The hostname filter ensures no external URLs are explored.

This design ensures correctness, handles cycles, distinguishes URL variants, and avoids unnecessary network calls.
