## Solution

## Problem Understanding

This problem asks us to find a common time slot between two people’s schedules that can accommodate a meeting of a given duration. We are provided with two lists of availability intervals, `slots1` and `slots2`, where each interval is represented as `[start, end]` with inclusive endpoints. The task is to return the **earliest possible interval** of length `duration` that exists in both schedules. If no such interval exists, we return an empty array.

The inputs `slots1` and `slots2` are arrays of non-overlapping intervals for each person. This guarantees that within a single person's schedule, no intervals intersect, simplifying the logic since we do not need to merge overlapping slots. The duration constraint ensures that we only consider intervals where `end - start >= duration`.

Key points to note:

1. The intervals within each person’s schedule may not be sorted.
2. Intervals can be very large (`up to 10^9`) and there can be up to `10^4` intervals per person.
3. A naive approach comparing all pairs of intervals would have `O(n * m)` time complexity, which could be too slow for large inputs.

Important edge cases to consider include intervals that are exactly equal to the meeting duration, intervals that touch but do not overlap enough for the meeting, and cases where no overlap exists at all.

## Approaches

### Brute Force

The brute-force approach involves iterating over every pair of intervals, one from `slots1` and one from `slots2`. For each pair, we compute the overlapping interval by taking the maximum of the start times and the minimum of the end times. If the overlap is at least `duration`, we store it. After checking all pairs, we return the earliest interval.

While correct, this approach has a time complexity of `O(n * m)` where `n` and `m` are the lengths of the input arrays. With `n` and `m` up to `10^4`, this could result in `10^8` comparisons and is too slow.

### Optimal Approach

The key observation is that if we **sort both arrays by start times**, we can efficiently find overlaps using a **two-pointer technique**. After sorting, we initialize two pointers, one for each array. At each step, we calculate the overlap between the current intervals. If it satisfies the required duration, we return it. Otherwise, we advance the pointer of the interval that ends first. This ensures we do not miss any potential overlapping interval while scanning each list only once.

This approach is efficient because sorting is `O(n log n + m log m)` and scanning is `O(n + m)`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m) | O(1) | Compare all pairs of intervals |
| Optimal | O(n log n + m log m) | O(1) | Sort both arrays, then use two pointers |

## Algorithm Walkthrough

1. Sort `slots1` and `slots2` in ascending order by their start times. This ensures that we can process intervals chronologically and efficiently find overlaps.
2. Initialize two pointers `i` and `j` to `0`, pointing to the first interval of each list.
3. Loop while both pointers are within bounds of their respective arrays:

1. Let `start_overlap = max(slots1[i][0], slots2[j][0])` and `end_overlap = min(slots1[i][1], slots2[j][1])`.
2. If `end_overlap - start_overlap >= duration`, return `[start_overlap, start_overlap + duration]`.
3. Otherwise, move the pointer corresponding to the interval that ends first. This is because any potential overlap with the other interval must come after the smaller end time.
4. If the loop ends without finding a valid interval, return an empty array.

**Why it works:** By always advancing the pointer of the interval that ends first, we guarantee that we consider all possible overlaps without missing any valid slot. Sorting ensures chronological processing, so the first valid overlap found is the earliest.

## Python Solution

```python
from typing import List

class Solution:
    def minAvailableDuration(self, slots1: List[List[int]], slots2: List[List[int]], duration: int) -> List[int]:
        slots1.sort(key=lambda x: x[0])
        slots2.sort(key=lambda x: x[0])
        
        i, j = 0, 0
        
        while i < len(slots1) and j < len(slots2):
            start_overlap = max(slots1[i][0], slots2[j][0])
            end_overlap = min(slots1[i][1], slots2[j][1])
            
            if end_overlap - start_overlap >= duration:
                return [start_overlap, start_overlap + duration]
            
            if slots1[i][1] < slots2[j][1]:
                i += 1
            else:
                j += 1
        
        return []
```

The implementation first sorts the two slot arrays. It then uses two pointers to efficiently scan for overlapping intervals. When an overlap satisfying the duration is found, it immediately returns the earliest such interval. Otherwise, it advances the pointer for the interval that ends first to maximize future overlap opportunities.

## Go Solution

```go
import "sort"

func minAvailableDuration(slots1 [][]int, slots2 [][]int, duration int) []int {
    sort.Slice(slots1, func(i, j int) bool { return slots1[i][0] < slots1[j][0] })
    sort.Slice(slots2, func(i, j int) bool { return slots2[i][0] < slots2[j][0] })
    
    i, j := 0, 0
    
    for i < len(slots1) && j < len(slots2) {
        startOverlap := max(slots1[i][0], slots2[j][0])
        endOverlap := min(slots1[i][1], slots2[j][1])
        
        if endOverlap - startOverlap >= duration {
            return []int{startOverlap, startOverlap + duration}
        }
        
        if slots1[i][1] < slots2[j][1] {
            i++
        } else {
            j++
        }
    }
    
    return []int{}
}

func max(a, b int) int {
    if a > b { return a }
    return b
}

func min(a, b int) int {
    if a < b { return a }
    return b
}
```

The Go solution mirrors the Python implementation. It uses `sort.Slice` to sort the intervals and utility functions for `max` and `min`. Go handles empty slices by returning `[]int{}` which is equivalent to Python’s empty list.

## Worked Examples

**Example 1:**

```
slots1 = [[10,50],[60,120],[140,210]]
slots2 = [[0,15],[60,70]]
duration = 8
```

After sorting (though already sorted):

```
slots1 = [[10,50],[60,120],[140,210]]
slots2 = [[0,15],[60,70]]
i=0, j=0
```

1. Compare [10,50] and [0,15]: start_overlap = max(10,0)=10, end_overlap=min(50,15)=15. Overlap length=5 < 8. Move j (slot2 ends first) → j=1
2. Compare [10,50] and [60,70]: start_overlap=max(10,60)=60, end_overlap=min(50,70)=50. Overlap length=-10 → not valid. Move i (slot1 ends first) → i=1
3. Compare [60,120] and [60,70]: start_overlap=max(60,60)=60, end_overlap=min(120,70)=70. Overlap length=10 ≥ 8 → return [60,68]

**Example 2:**

```
slots1 = [[10,50],[60,120],[140,210]]
slots2 = [[0,15],[60,70]]
duration = 12
```

Following the same steps, all overlaps are either 5 or 10 units long, none satisfy duration=12 → return []

The complexity is dominated by the sorting step. The two-pointer scan is linear and does not require additional space.
