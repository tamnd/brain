## Solution

## Problem Understanding

This problem asks us to implement a probabilistic data structure called a skiplist. A skiplist behaves similarly to a balanced binary search tree, but internally it is built using multiple layers of linked lists instead of tree rotations and balancing logic.

The data structure must support four operations:

- Initialization of an empty skiplist
- Searching for a value
- Inserting a value
- Removing a value

All operations are expected to run in average `O(log n)` time.

The skiplist contains several levels. The bottom level stores every element in sorted order. Higher levels act as shortcuts that allow us to skip over many nodes at once. Each node may appear in multiple levels. The higher the level, the fewer nodes exist there.

For example, instead of scanning:

```
1 -> 2 -> 3 -> 4 -> 5 -> 6 -> 7
```

we may jump through upper levels like:

```
1 --------> 5 --------> 7
```

then descend to lower levels only when needed.

The input consists of method calls performed on the `Skiplist` object. Each operation modifies or queries the structure. The output depends on the operation:

- `search(target)` returns `True` if the value exists
- `erase(num)` returns `True` if one occurrence was removed
- `add(num)` has no return value

The constraints are important:

- Up to `5 * 10^4` operations
- Values range from `0` to `2 * 10^4`

A naive linked list implementation would degrade to `O(n)` per operation, which could become too slow for tens of thousands of operations. The problem specifically expects a logarithmic average-time design.

There are several edge cases that can easily cause bugs:

- Duplicate values are allowed
- Removing one occurrence should not remove all duplicates
- Searching for missing values must terminate correctly
- Random level generation must stay within bounds
- The skiplist may become empty after deletions
- Insertions at the beginning or end must work correctly

A correct implementation must preserve sorted order across every level while maintaining consistent forward pointers.

## Approaches

### Brute Force Approach

The simplest implementation would use a sorted linked list.

For insertion, we traverse the list until we find the correct position and insert the node. For searching, we scan sequentially until we either find the target or pass its possible position. For deletion, we again scan sequentially and reconnect pointers if the value is found.

This approach is correct because the list always remains sorted. However, every operation potentially scans the entire list.

With up to `5 * 10^4` operations, worst-case linear scans become inefficient.

### Optimal Approach, Skiplist with Randomized Levels

The key insight behind a skiplist is that we can accelerate linked list traversal by adding shortcut layers.

Instead of storing all nodes in a single level, some nodes are promoted to higher levels randomly. Each higher level contains fewer nodes. During search, we move horizontally while the next node is still smaller than the target. When we can no longer move right, we move down one level.

This creates behavior similar to binary search trees:

- Horizontal movement skips many nodes
- Vertical movement narrows the search region

Randomized promotion ensures the structure remains balanced on average without requiring rotations or complex rebalancing logic.

The expected complexity becomes:

- Search: `O(log n)`
- Insert: `O(log n)`
- Delete: `O(log n)`

while keeping implementation simpler than balanced trees.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) per operation | O(n) | Sorted singly linked list |
| Optimal | O(log n) average per operation | O(n) average | Multi-level probabilistic linked structure |

## Algorithm Walkthrough

### Data Structure Design

Each node stores:

- Its value
- A list of forward pointers, one for each level

For example:

```
Node(10)
forward[0] -> next node in level 0
forward[1] -> next node in level 1
forward[2] -> next node in level 2
```

We also maintain:

- A head sentinel node
- A maximum allowed level
- The current highest active level

### Random Level Generation

When inserting a node, we randomly determine how many levels it occupies.

We repeatedly flip a virtual coin:

- Heads, promote to next level
- Tails, stop

This creates exponentially fewer nodes at higher levels.

### Search Operation

1. Start at the highest active level from the head node.
2. Move right while the next node value is smaller than the target.
3. Once moving right would overshoot, move down one level.
4. Repeat until reaching level 0.
5. Check whether the next node equals the target.

This strategy skips large portions of the structure efficiently.

### Insert Operation

1. Traverse the structure similarly to search.
2. At every level, record the node immediately before the insertion position.
3. Generate a random level for the new node.
4. If the new level exceeds the current maximum level, update the skiplist level.
5. Insert the new node by reconnecting forward pointers level by level.

The recorded predecessor nodes make insertion efficient.

### Delete Operation

1. Traverse the structure and record predecessor nodes.
2. Check whether the target exists at level 0.
3. If absent, return `False`.
4. Otherwise, reconnect pointers to skip the node.
5. Remove empty top levels if necessary.

Only one occurrence is removed.

### Why it works

The skiplist maintains sorted order at every level. Each upper level acts as a sparse index into the lower levels. During traversal, moving right never skips over a possible target because we only move while the next value is still smaller than the target.

Insertions and deletions update all affected forward pointers consistently, so the structure remains valid after every operation. Random level assignment keeps the expected height logarithmic, which guarantees average `O(log n)` performance.

## Python Solution

```python
import random
from typing import List, Optional

class Node:
    def __init__(self, val: int, level: int):
        self.val = val
        self.forward: List[Optional["Node"]] = [None] * level

class Skiplist:

    MAX_LEVEL = 16
    P_FACTOR = 0.5

    def __init__(self):
        self.head = Node(-1, self.MAX_LEVEL)
        self.level = 1

    def random_level(self) -> int:
        level = 1

        while (
            random.random() < self.P_FACTOR
            and level < self.MAX_LEVEL
        ):
            level += 1

        return level

    def search(self, target: int) -> bool:
        current = self.head

        for i in range(self.level - 1, -1, -1):
            while (
                current.forward[i]
                and current.forward[i].val < target
            ):
                current = current.forward[i]

        current = current.forward[0]

        return current is not None and current.val == target

    def add(self, num: int) -> None:
        update = [None] * self.MAX_LEVEL
        current = self.head

        for i in range(self.level - 1, -1, -1):
            while (
                current.forward[i]
                and current.forward[i].val < num
            ):
                current = current.forward[i]

            update[i] = current

        new_level = self.random_level()

        if new_level > self.level:
            for i in range(self.level, new_level):
                update[i] = self.head

            self.level = new_level

        new_node = Node(num, new_level)

        for i in range(new_level):
            new_node.forward[i] = update[i].forward[i]
            update[i].forward[i] = new_node

    def erase(self, num: int) -> bool:
        update = [None] * self.MAX_LEVEL
        current = self.head

        for i in range(self.level - 1, -1, -1):
            while (
                current.forward[i]
                and current.forward[i].val < num
            ):
                current = current.forward[i]

            update[i] = current

        current = current.forward[0]

        if current is None or current.val != num:
            return False

        for i in range(self.level):
            if update[i].forward[i] != current:
                continue

            update[i].forward[i] = current.forward[i]

        while (
            self.level > 1
            and self.head.forward[self.level - 1] is None
        ):
            self.level -= 1

        return True
```

The implementation begins with a `Node` class. Each node stores a value and an array of forward pointers. The array size equals the node's level height.

The skiplist itself maintains a sentinel head node with the maximum number of levels. The sentinel simplifies insertion and deletion near the beginning of the structure because we never need special handling for the first real node.

The `random_level` method determines how many levels a new node receives. Each level promotion occurs with probability `0.5`, creating the balanced probabilistic structure.

The `search` method starts from the topmost active level and moves horizontally while possible. Once further movement would exceed the target, it descends one level. This process continues until level 0 is reached.

The `add` method first records predecessor nodes inside the `update` array. After generating the node level, it reconnects forward pointers level by level.

The `erase` method also uses the `update` array. If the target node exists, all references to that node are bypassed. Finally, unused upper levels are removed by decreasing the current active level count.

## Go Solution

```go
package main

import (
	"math/rand"
)

const (
	MAX_LEVEL = 16
	P_FACTOR  = 0.5
)

type Node struct {
	val     int
	forward []*Node
}

func NewNode(val int, level int) *Node {
	return &Node{
		val:     val,
		forward: make([]*Node, level),
	}
}

type Skiplist struct {
	head  *Node
	level int
}

func Constructor() Skiplist {
	return Skiplist{
		head:  NewNode(-1, MAX_LEVEL),
		level: 1,
	}
}

func (this *Skiplist) randomLevel() int {
	level := 1

	for rand.Float64() < P_FACTOR && level < MAX_LEVEL {
		level++
	}

	return level
}

func (this *Skiplist) Search(target int) bool {
	current := this.head

	for i := this.level - 1; i >= 0; i-- {
		for current.forward[i] != nil &&
			current.forward[i].val < target {
			current = current.forward[i]
		}
	}

	current = current.forward[0]

	return current != nil && current.val == target
}

func (this *Skiplist) Add(num int) {
	update := make([]*Node, MAX_LEVEL)
	current := this.head

	for i := this.level - 1; i >= 0; i-- {
		for current.forward[i] != nil &&
			current.forward[i].val < num {
			current = current.forward[i]
		}

		update[i] = current
	}

	newLevel := this.randomLevel()

	if newLevel > this.level {
		for i := this.level; i < newLevel; i++ {
			update[i] = this.head
		}

		this.level = newLevel
	}

	newNode := NewNode(num, newLevel)

	for i := 0; i < newLevel; i++ {
		newNode.forward[i] = update[i].forward[i]
		update[i].forward[i] = newNode
	}
}

func (this *Skiplist) Erase(num int) bool {
	update := make([]*Node, MAX_LEVEL)
	current := this.head

	for i := this.level - 1; i >= 0; i-- {
		for current.forward[i] != nil &&
			current.forward[i].val < num {
			current = current.forward[i]
		}

		update[i] = current
	}

	current = current.forward[0]

	if current == nil || current.val != num {
		return false
	}

	for i := 0; i < this.level; i++ {
		if update[i].forward[i] != current {
			continue
		}

		update[i].forward[i] = current.forward[i]
	}

	for this.level > 1 &&
		this.head.forward[this.level-1] == nil {
		this.level--
	}

	return true
}
```

The Go implementation closely mirrors the Python version, but there are some language-specific details.

Go uses slices for the forward pointer arrays. Since Go does not support nullable references in the same way as Python, we use `nil` pointer checks explicitly.

The `randomLevel` function uses `rand.Float64()` to simulate coin flips. Struct initialization is handled using constructors such as `NewNode`.

Because Go arrays have fixed size while slices are dynamic, the forward pointers are implemented using slices allocated with `make`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(log n) average | Search, insertion, and deletion traverse logarithmic expected levels |
| Space | O(n) average | Nodes store multiple forward pointers probabilistically |

The expected height of the skiplist is logarithmic because each level contains roughly half as many nodes as the level below it. Traversal therefore visits only a logarithmic number of nodes on average.

Although a node may contain multiple forward pointers, the expected total number of pointers across all nodes remains linear.

## Edge Cases

### Duplicate Values

One of the most important edge cases is handling duplicates correctly. A naive implementation may accidentally overwrite existing nodes or remove all matching values during deletion.

This implementation inserts duplicates normally into the sorted structure. During deletion, only the first matching occurrence encountered at level 0 is removed. Remaining duplicates stay intact.

### Empty Skiplist Operations

Searching or deleting in an empty skiplist can easily cause null pointer errors if forward pointers are accessed without checks.

The implementation safely checks whether forward pointers exist before dereferencing them. The sentinel head node also simplifies handling empty structures because traversal always begins from a valid node.

### Removing the Last Node

Deleting the final remaining node can leave higher levels empty. If the skiplist level counter is not updated properly, future operations may traverse invalid empty levels.

After every deletion, the implementation repeatedly decreases the active level count while the highest level contains no nodes. This guarantees traversal only visits meaningful levels.

### Inserting Smallest or Largest Values

Values inserted at the beginning or end of the structure can expose pointer update bugs.

Using the sentinel head node eliminates special-case logic for front insertions. Tail insertions naturally work because forward pointers may simply become `None` or `nil`.

### Random Height Distribution

If node heights are not generated carefully, the skiplist may become unbalanced and degrade toward linear performance.

The probabilistic promotion strategy ensures exponentially fewer nodes appear at higher levels, maintaining expected logarithmic behavior.
