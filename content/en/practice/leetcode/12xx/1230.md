## Solution

## Problem Understanding

The problem asks us to calculate the probability of obtaining exactly `target` heads when tossing a list of biased coins once. Each coin has its own probability `prob[i]` of landing heads. The input consists of an array of probabilities and a target integer, and the output is a single floating-point number representing the probability that exactly `target` coins show heads.

In other words, if we imagine flipping all coins one by one, the problem is equivalent to computing the probability of all sequences of heads and tails where the total number of heads is exactly `target`.

The constraints tell us that the number of coins can be up to 1000, and each probability is between 0 and 1. The target is non-negative and cannot exceed the number of coins. These constraints imply that brute-force enumeration of all possible sequences, which would involve `2^n` combinations, is computationally infeasible.

Important edge cases include having only one coin, having a target of zero, or having probabilities of 0 or 1. These cases test if the implementation correctly handles guaranteed outcomes or impossible outcomes without introducing floating-point errors.

## Approaches

The brute-force approach is to generate all possible sequences of coin flips, count how many sequences have exactly `target` heads, and sum their probabilities. This is correct because each sequence has a probability equal to the product of its individual coin probabilities. However, this is exponential in time (`O(2^n)`) and clearly infeasible for `n` up to 1000.

The key insight for an optimal solution is dynamic programming. Let `dp[i][j]` represent the probability of getting exactly `j` heads using the first `i` coins. For the `i`-th coin, it can either contribute to heads or not. Therefore, the recurrence is:

```
dp[i][j] = dp[i-1][j] * (1 - prob[i-1]) + dp[i-1][j-1] * prob[i-1]
```

This recurrence allows us to build the solution incrementally. We only need a 1D array of size `target + 1` if we update in reverse order, reducing the space complexity to `O(target)` instead of `O(n*target)`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Enumerate all possible coin sequences and sum probabilities |
| Dynamic Programming | O(n * target) | O(target) | Use recurrence relation to build probabilities incrementally |

## Algorithm Walkthrough

1. Initialize a 1D array `dp` of length `target + 1` with all zeros. Set `dp[0] = 1.0` because the probability of having 0 heads with 0 coins is 1.
2. Iterate through each coin in the `prob` array.
3. For each coin, iterate through the possible number of heads `j` from `target` down to 1. We iterate in reverse to avoid overwriting values that are needed for smaller `j`.
4. Update `dp[j]` using the formula `dp[j] = dp[j] * (1 - p) + dp[j-1] * p`, where `p` is the probability of the current coin landing heads. This accounts for the two possibilities: the coin is tails or heads.
5. Update `dp[0]` separately as `dp[0] = dp[0] * (1 - p)` because if we want 0 heads, the current coin must be tails.
6. After processing all coins, `dp[target]` contains the probability of getting exactly `target` heads.

**Why it works**: The DP array maintains the probability of obtaining each possible number of heads after each coin. By iterating in reverse for `j`, we ensure that each update uses the previous state before the current coin, preserving the correctness of the recurrence. The final `dp[target]` correctly aggregates all combinations of heads and tails that sum to the target.

## Python Solution

```python
from typing import List

class Solution:
    def probabilityOfHeads(self, prob: List[float], target: int) -> float:
        dp = [0.0] * (target + 1)
        dp[0] = 1.0
        
        for p in prob:
            for j in range(target, 0, -1):
                dp[j] = dp[j] * (1 - p) + dp[j - 1] * p
            dp[0] = dp[0] * (1 - p)
        
        return dp[target]
```

This Python implementation initializes the DP array to store probabilities for each possible number of heads. The outer loop iterates over each coin, and the inner loop updates probabilities from `target` down to 1 to prevent overwriting needed values. The update for `dp[0]` handles the case where the current coin contributes no heads.

## Go Solution

```go
func probabilityOfHeads(prob []float64, target int) float64 {
    dp := make([]float64, target+1)
    dp[0] = 1.0
    
    for _, p := range prob {
        for j := target; j >= 1; j-- {
            dp[j] = dp[j]*(1-p) + dp[j-1]*p
        }
        dp[0] = dp[0] * (1 - p)
    }
    
    return dp[target]
}
```

In Go, we create a slice `dp` of length `target + 1` initialized to 0, with `dp[0]` set to 1. The loops and recurrence are implemented similarly to Python. Go handles slices efficiently, and we avoid integer overflow because all values are floating-point numbers.

## Worked Examples

**Example 1:** `prob = [0.4], target = 1`

- Initialize `dp = [1.0, 0.0]`
- Process coin 0 (`p = 0.4`):

- `dp[1] = dp[1]*0.6 + dp[0]*0.4 = 0.0*0.6 + 1.0*0.4 = 0.4`
- `dp[0] = dp[0]*0.6 = 1.0*0.6 = 0.6`
- Result: `dp[1] = 0.4`

**Example 2:** `prob = [0.5,0.5,0.5,0.5,0.5], target = 0`

- Initialize `dp = [1.0] + [0.0]*0 = [1.0]`
- Each coin multiplies `dp[0]` by 0.5:

- After 5 coins: `dp[0] = 1.0 * 0.5^5 = 0.03125`
- Result: `dp[0] = 0.03125`

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * target) | For each of the n coins, we update dp array of size target+1 |
| Space | O(target) | Only a 1D array of size target+1 is needed for dynamic programming |

The time complexity is efficient given the constraints (`n` up to 1000, `target` up to `n`). Space complexity is linear in `target` due to the optimized 1D DP array.
