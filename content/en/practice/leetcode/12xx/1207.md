## Problem Understanding

The problem asks us to determine whether the number of times each integer appears in an array is unique. In other words, if we count how many times each distinct number occurs in the array, no two numbers should share the same occurrence count. The input is an array of integers, `arr`, with up to 1000 elements, and the elements themselves can range from -1000 to 1000. The output is a boolean: `true` if the counts are unique, `false` otherwise.

Constraints are relatively small, so algorithms with linear or near-linear time complexity are feasible. Important edge cases include arrays where all elements are the same, arrays where each element is distinct, arrays with negative numbers, and arrays where multiple elements share the same frequency. The problem guarantees that `arr` has at least one element.

## Approaches

The brute-force approach is to count the occurrences of each number by iterating through the array multiple times or by building a nested loop check. While this can produce correct results, it is inefficient and potentially O(n²) in time complexity, making it unnecessary given the constraints.

The optimal approach leverages hash maps (or dictionaries) to count occurrences efficiently. First, we iterate through the array to compute the count of each number. Then, we store these counts in a set to detect duplicates. If the size of the set is the same as the number of unique numbers, all occurrences are unique; otherwise, they are not.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Count occurrences via nested iteration, slow for larger arrays |
| Optimal | O(n) | O(n) | Use hash map to count occurrences and set to check uniqueness |

## Algorithm Walkthrough

1. Initialize an empty hash map (dictionary) to store the count of each number.
2. Iterate through each number in the input array.
3. For each number, increment its count in the hash map.
4. After processing the array, initialize an empty set to store occurrence counts.
5. Iterate through the values in the hash map (the counts of each number).
6. If the count is already in the set, return `false` immediately because it indicates a duplicate occurrence count.
7. Otherwise, add the count to the set.
8. If all counts are added without duplicates, return `true`.

Why it works: Each number's occurrence is counted exactly once and stored in the set. The set naturally enforces uniqueness, so checking its contents guarantees no duplicate counts exist. This ensures correctness in a straightforward and efficient manner.

## Python Solution

```python
from typing import List

class Solution:
    def uniqueOccurrences(self, arr: List[int]) -> bool:
        count_map = {}
        for num in arr:
            count_map[num] = count_map.get(num, 0) + 1
        
        seen_counts = set()
        for count in count_map.values():
            if count in seen_counts:
                return False
            seen_counts.add(count)
        
        return True
```

In this implementation, we first build `count_map` to store the frequency of each number. The `get` method provides a default of 0 for unseen numbers. Next, we iterate over the occurrence counts, using `seen_counts` to detect duplicates. If a duplicate exists, the function returns `false`; otherwise, it returns `true`.

## Go Solution

```go
func uniqueOccurrences(arr []int) bool {
    countMap := make(map[int]int)
    for _, num := range arr {
        countMap[num]++
    }

    seenCounts := make(map[int]bool)
    for _, count := range countMap {
        if seenCounts[count] {
            return false
        }
        seenCounts[count] = true
    }

    return true
}
```

In Go, we use a `map[int]int` to count occurrences and a `map[int]bool` to track whether a count has already been seen. Go does not have a built-in set, so the boolean map substitutes. The logic mirrors the Python solution.

## Edge Cases

One edge case is an array with only one element. Here, the occurrence count is 1, which is trivially unique, and the solution correctly returns true. Another edge case is an array where multiple numbers have the same frequency, such as `[1,2,3,4]`. Even though each number occurs once, all counts are equal, so the function must return false. The implementation correctly detects this via the set. A third edge case is an array with negative numbers and zeros. Since the problem allows negative numbers, our solution treats them identically to positive numbers, and counting works without modification.
