## Solution

## Problem Understanding

The problem asks us to convert a decimal number into a special hexadecimal representation called Hexspeak. The input is provided as a string `num`, where the string represents a decimal integer. Our task is to transform this decimal number into uppercase hexadecimal format, then apply two character substitutions:

- Replace every `'0'` with `'O'`
- Replace every `'1'` with `'I'`

After performing these substitutions, the resulting string is only considered valid if every character belongs to the set:

```
{'A', 'B', 'C', 'D', 'E', 'F', 'I', 'O'}
```

This means that hexadecimal digits like `2`, `3`, `4`, `5`, `6`, `7`, `8`, and `9` are not allowed in the final representation. If any such character appears, we must return `"ERROR"`.

For example, the decimal number `257` converts to hexadecimal `101`. Replacing `1 -> I` and `0 -> O` gives `"IOI"`, which contains only valid characters, so the answer is `"IOI"`.

On the other hand, the decimal number `3` converts to hexadecimal `"3"`. Since `'3'` is not one of the allowed Hexspeak characters, the result is invalid and we return `"ERROR"`.

The constraints are small. The input length is at most 12 digits, and the numeric value is at most `10^12`. This means even straightforward conversion approaches are efficient enough. We do not need advanced optimization techniques or special numeric representations.

Several edge cases are important:

- Numbers whose hexadecimal representation contains invalid digits such as `2` through `9`
- Numbers that produce only `0` and `1`, which must become `O` and `I`
- Very large numbers near `10^12`
- Numbers whose hexadecimal representation already contains letters `A` through `F`
- Single digit numbers like `1` or `15`

The problem guarantees that the input is a valid positive decimal integer with no leading zeros, so we do not need to validate the input format ourselves.

## Approaches

### Brute Force Approach

A brute force solution would manually compute the hexadecimal representation digit by digit using repeated division by 16. At each step, we would extract the remainder, map it to the appropriate hexadecimal character, build the hexadecimal string, and finally perform the Hexspeak validation and substitutions.

This approach works because hexadecimal conversion through repeated division is mathematically correct. Each remainder represents one hexadecimal digit, and processing the remainders from least significant to most significant reconstructs the full hexadecimal representation.

However, implementing manual base conversion is unnecessarily verbose for this problem. Modern programming languages already provide efficient and reliable hexadecimal conversion utilities. Writing the conversion logic ourselves increases code complexity and creates more opportunities for bugs.

### Optimal Approach

The key observation is that the difficult part of the problem is not hexadecimal conversion itself. The real task is validating whether the hexadecimal representation contains only allowed characters after substitutions.

Instead of manually computing hexadecimal digits, we can:

1. Convert the decimal number directly to uppercase hexadecimal
2. Scan each character
3. Replace:

- `'0' -> 'O'`
- `'1' -> 'I'`
4. Reject immediately if any character is between `'2'` and `'9'`
5. Keep hexadecimal letters `A-F` unchanged

This solution is both simpler and cleaner because it leverages built in language functionality for hexadecimal conversion.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(log n) | O(log n) | Manual repeated division and conversion |
| Optimal | O(log n) | O(log n) | Uses built in hexadecimal conversion and validation |

## Algorithm Walkthrough

1. Convert the decimal string into an integer.

The input arrives as a string because the number may be large. We first parse it into an integer type so we can perform hexadecimal conversion.
2. Convert the integer into an uppercase hexadecimal string.

In Python, we can use `hex(n)[2:].upper()`.

In Go, we can use `strconv.FormatInt(n, 16)` and then convert to uppercase.

We remove the `"0x"` prefix because the problem expects only the hexadecimal digits.
3. Create a result container.

Since we may need to replace characters while scanning, we build the final answer incrementally using a list or string builder.
4. Iterate through every hexadecimal character.

For each character:

- If the character is `'0'`, append `'O'`
- If the character is `'1'`, append `'I'`
- If the character is one of `'A'` through `'F'`, append it unchanged
- Otherwise, return `"ERROR"` immediately

This validation step ensures the final representation only contains valid Hexspeak characters.
5. Join and return the final string.

If the scan completes successfully, the constructed string is guaranteed to be valid Hexspeak.

### Why it works

The algorithm works because every hexadecimal digit must fall into exactly one of four categories:

- `'0'`, which becomes `'O'`
- `'1'`, which becomes `'I'`
- `'A'` through `'F'`, which are already valid
- Invalid digits `'2'` through `'9'`

If any invalid digit appears, no valid Hexspeak representation exists. Otherwise, replacing `0` and `1` produces a string containing only allowed characters. Since hexadecimal conversion itself is correct, the final result must also be correct.

## Python Solution

```python
class Solution:
    def toHexspeak(self, num: str) -> str:
        value = int(num)
        hex_string = hex(value)[2:].upper()

        result = []

        for ch in hex_string:
            if ch == '0':
                result.append('O')
            elif ch == '1':
                result.append('I')
            elif 'A' <= ch <= 'F':
                result.append(ch)
            else:
                return "ERROR"

        return ''.join(result)
```

The implementation begins by converting the decimal string into an integer using `int(num)`. We then convert the integer into hexadecimal form using Python's built in `hex()` function. Since `hex()` produces strings like `"0x101"`, we slice away the first two characters and convert the remaining digits to uppercase.

The `result` list is used because repeatedly concatenating strings is less efficient. Lists allow efficient appends, and we join everything together at the end.

During iteration, we classify each hexadecimal digit into one of the valid categories. Digits `0` and `1` are replaced with `O` and `I`, respectively. Letters `A` through `F` remain unchanged. Any numeric digit from `2` through `9` immediately invalidates the representation, so we return `"ERROR"`.

Finally, if no invalid digit was found, we join the characters into the final Hexspeak string.

## Go Solution

```go
package main

import (
	"strconv"
	"strings"
)

func toHexspeak(num string) string {
	value, _ := strconv.ParseInt(num, 10, 64)

	hexString := strings.ToUpper(strconv.FormatInt(value, 16))

	var result strings.Builder

	for _, ch := range hexString {
		if ch == '0' {
			result.WriteRune('O')
		} else if ch == '1' {
			result.WriteRune('I')
		} else if ch >= 'A' && ch <= 'F' {
			result.WriteRune(ch)
		} else {
			return "ERROR"
		}
	}

	return result.String()
}
```

The Go solution follows the same algorithmic structure as the Python version. The primary difference is that Go requires explicit integer parsing using `strconv.ParseInt`.

Hexadecimal conversion is performed using `strconv.FormatInt(value, 16)`, which generates lowercase hexadecimal output, so we convert it to uppercase using `strings.ToUpper`.

Instead of a list, Go uses `strings.Builder`, which efficiently constructs strings without repeated memory allocations.

Since the input constraint is at most `10^12`, a signed 64 bit integer safely holds all possible values.

## Edge Cases

One important edge case occurs when the hexadecimal representation contains invalid numeric digits such as `2` through `9`. For example, decimal `3` becomes hexadecimal `"3"`. A careless implementation might simply replace `0` and `1` while leaving all other digits unchanged, which would incorrectly produce invalid Hexspeak. Our implementation explicitly checks whether each character is either `0`, `1`, or `A-F`. Any other character immediately returns `"ERROR"`.

Another important edge case is numbers whose hexadecimal representation consists entirely of zeros and ones. For example, `256` converts to hexadecimal `"100"`. The algorithm must correctly replace every occurrence of `1` and `0`, producing `"IOO"`. Since the implementation processes characters one at a time and applies replacements independently, repeated substitutions work correctly.

A third edge case involves large inputs near the upper constraint limit. Since the input can be as large as `10^12`, implementations using small integer types could overflow. Both the Python and Go solutions avoid this problem. Python integers support arbitrary precision automatically, and the Go solution uses a 64 bit signed integer, which safely accommodates all allowed inputs.
