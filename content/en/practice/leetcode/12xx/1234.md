**Difficulty:** 🟡 Medium  
**Topics:** String, Sliding Window

## Solution

## Problem Understanding

The problem gives us a string `s` of length `n` containing only the characters `'Q'`, `'W'`, `'E'`, and `'R'`. A string is considered **balanced** if each of the four characters occurs exactly `n / 4` times. Our task is to determine the minimum length of a contiguous substring that we can replace with any other string of the same length so that the resulting string becomes balanced. If the string is already balanced, we should return `0`.

The input string has constraints that simplify the problem. The length `n` is always a multiple of 4, which means a perfectly balanced string is always possible. The string contains only four specific characters, which lets us use a fixed-size counting structure instead of a generic hash map. Important edge cases include already balanced strings (should return `0`) and strings where a single character is extremely overrepresented (requiring replacement of multiple consecutive characters).

The problem can be interpreted as: "Given a string, find the shortest window that can be modified to remove any excess characters such that the remainder of the string plus the modified window produces a balanced string."

## Approaches

### Brute Force

The brute-force approach considers every possible substring of `s` and replaces it with hypothetical characters that would make the string balanced. For each substring, we compute the character counts of the remaining string and check if it is possible to achieve balance by replacing the substring. If it is, we update the minimum length.

This approach is correct because it exhaustively explores all possible substrings and validates if they can yield a balanced string. However, it is extremely inefficient. For a string of length `n`, there are O(n²) substrings, and for each substring, we need O(n) to compute the counts, leading to O(n³) time complexity, which is impractical for `n` up to 10⁵.

### Optimal Approach

The key observation is that we only need to **remove or replace excess characters** to make the string balanced. We can compute how many extra occurrences each character has compared to `n / 4`. If we maintain a sliding window representing the substring we will replace, the characters inside the window reduce the excess count. We can expand the window to cover all excess characters and shrink it from the left while still covering the excess. The goal is to minimize the window length.

This is a classic **sliding window problem** because we are looking for the smallest substring that satisfies a certain property (covering all excess characters). The counts of `'Q'`, `'W'`, `'E'`, and `'R'` outside the window should not exceed `n / 4`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Check every substring to see if replacing it can balance the string |
| Optimal | O(n) | O(1) | Sliding window approach; counts excess characters and finds minimal window |

## Algorithm Walkthrough

1. Compute `n` as the length of `s` and calculate `target = n / 4`. This is the ideal count for each character in a balanced string.
2. Count occurrences of each character in the string. If all counts are equal to `target`, return `0` immediately.
3. Determine the excess of each character: `excess[c] = max(0, count[c] - target)`. Only characters with excess need to be reduced by replacement.
4. Initialize two pointers `left` and `right` for the sliding window and set `min_len = n`.
5. Expand the right pointer: for each character `s[right]`, decrement its excess count if it is part of the excess.
6. Once all excess counts are zero or less (meaning the window covers enough excess characters to allow balance), try to shrink the window from the left to minimize the length. Move the left pointer and restore excess counts for characters removed from the window.
7. Continue expanding the window and shrinking it whenever possible until the right pointer reaches the end of the string.
8. Return the smallest window length found.

**Why it works:** The sliding window ensures that at any moment, we are considering the minimal substring that can account for all excess characters. By adjusting the window dynamically and tracking the excess counts, we guarantee that the window length we compute is the minimal possible.

## Python Solution

```python
class Solution:
    def balancedString(self, s: str) -> int:
        from collections import Counter

        n = len(s)
        target = n // 4
        count = Counter(s)
        excess = {c: max(0, count[c] - target) for c in 'QWER'}

        if all(v == 0 for v in excess.values()):
            return 0

        min_len = n
        left = 0

        for right in range(n):
            if s[right] in excess:
                excess[s[right]] -= 1

            while all(v <= 0 for v in excess.values()):
                min_len = min(min_len, right - left + 1)
                if s[left] in excess:
                    excess[s[left]] += 1
                left += 1

        return min_len
```

The Python implementation first calculates excess counts for each character. The sliding window expands with `right` and shrinks from `left` while the excess is covered. We update `min_len` whenever the window is valid. Using `Counter` and a dictionary keeps the logic clean and efficient.

## Go Solution

```go
func balancedString(s string) int {
    n := len(s)
    target := n / 4
    count := map[byte]int{'Q': 0, 'W': 0, 'E': 0, 'R': 0}
    for i := 0; i < n; i++ {
        count[s[i]]++
    }

    excess := map[byte]int{}
    for k, v := range count {
        if v > target {
            excess[k] = v - target
        } else {
            excess[k] = 0
        }
    }

    allZero := func() bool {
        for _, v := range excess {
            if v > 0 {
                return false
            }
        }
        return true
    }

    if allZero() {
        return 0
    }

    minLen := n
    left := 0

    for right := 0; right < n; right++ {
        if _, ok := excess[s[right]]; ok {
            excess[s[right]]--
        }

        for allZero() {
            if right-left+1 < minLen {
                minLen = right - left + 1
            }
            if _, ok := excess[s[left]]; ok {
                excess[s[left]]++
            }
            left++
        }
    }

    return minLen
}
```

In Go, we use `map[byte]int` instead of Python’s `Counter`. The `allZero` function checks if the excess counts are all zero or negative, ensuring the window covers all necessary characters. The sliding window logic mirrors the Python version.

## Worked Examples

### Example 1: `"QWER"`

Initial counts: Q=1, W=1, E=1, R=1. Target = 1. Excess = {Q:0, W:0, E:0, R:0}. Already balanced → return 0.

### Example 2: `"QQWE"`

Counts: Q=2, W=1, E=1, R=0. Target=1. Excess = {Q:1, W:0, E:0, R:0}.

Sliding window examines substrings:

| left | right | excess | min_len |
| --- | --- | --- | --- |
| 0 | 0 | Q=0 | 1 |

Shortest window covering all excess is `"Q"` → return 1.

### Example 3: `"QQQW"`

Counts: Q=3, W=1, E=0, R=0. Target=1. Excess = {Q:2, W:0, E:0, R:0}.

Sliding window expands until window covers 2 Qs. Smallest such window length = 2 → return 2.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is processed at most twice by the sliding window |
| Space | O(1) | Only fixed-size dictionary of 4 characters |

The algorithm is linear because each pointer moves from 0 to n at most once. Extra space is constant because the problem only deals with four characters.
