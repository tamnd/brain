## Solution

## Problem Understanding

This problem models the movement and attack rules of queens in chess. You are given the positions of several black queens on a standard `8 x 8` chessboard and the position of a single white king. Your task is to determine which queens can directly attack the king.

A queen in chess can move horizontally, vertically, and diagonally. That means a queen can attack the king if both pieces lie on the same row, the same column, or the same diagonal. However, queens cannot attack through other queens. If another queen blocks the path between a queen and the king, the farther queen cannot attack the king directly.

The input consists of:

- `queens`, a list of coordinates where each coordinate represents the position of a black queen.
- `king`, a coordinate representing the white king's position.

Each coordinate is represented as `[row, column]`.

The output should contain all queens that can directly attack the king. The order of the returned queens does not matter.

The constraints are small:

- The board size is fixed at `8 x 8`.
- There are at most `63` queens.
- All positions are unique.

These constraints tell us that even less efficient approaches would still run quickly in practice. However, the problem is designed to test spatial reasoning and efficient directional scanning.

Several edge cases are important:

- Multiple queens may lie in the same direction from the king, but only the closest one can attack.
- Queens may appear on all eight possible attack directions.
- Some queens may align with the king but still be blocked.
- The king may be near the edge or corner of the board.
- There may be only one queen on the board.

The uniqueness guarantee simplifies the implementation because we never need to handle duplicate queen positions.

## Approaches

### Brute Force Approach

A straightforward approach is to examine every queen and determine whether it shares a row, column, or diagonal with the king. If it does, we then simulate movement from the queen toward the king and check whether another queen blocks the path.

For each queen:

1. Check whether it can theoretically attack the king based on alignment.
2. Move step by step toward the king.
3. If another queen is encountered first, the attack is blocked.
4. Otherwise, the queen can attack the king.

This approach is correct because it explicitly validates the attack path for every queen. However, it performs unnecessary repeated scans across the board. Many queens may examine overlapping paths.

Although the board is tiny, the repeated directional simulation is less elegant and scales poorly conceptually.

### Optimal Approach

The key insight is that only the closest queen in each of the eight directions around the king can attack.

There are exactly eight possible directions:

- Up
- Down
- Left
- Right
- Top-left
- Top-right
- Bottom-left
- Bottom-right

Starting from the king, we can walk outward one square at a time in each direction. The first queen encountered in that direction is the only queen capable of attacking from that line. Any queens farther away are blocked.

This eliminates unnecessary repeated checking and directly models how chess movement works.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(Q × 8) | O(Q) | Check every queen and simulate attack paths |
| Optimal | O(Q + 8 × 8) | O(Q) | Scan outward from king in all eight directions |

Since the board size is fixed, the optimal solution effectively runs in constant time.

## Algorithm Walkthrough

1. Store all queen positions in a hash set.

We need fast lookup to determine whether a square contains a queen. A hash set allows O(1) average-time membership checks.
2. Define the eight movement directions.

Each direction is represented as a pair `(dx, dy)`:

- `(-1, 0)` for up
- `(1, 0)` for down
- `(0, -1)` for left
- `(0, 1)` for right
- `(-1, -1)` for top-left
- `(-1, 1)` for top-right
- `(1, -1)` for bottom-left
- `(1, 1)` for bottom-right
3. For each direction, begin scanning from the king's position.

Move one step at a time using the direction vector.
4. Continue moving while the position remains inside the board.

The board boundaries are from `0` to `7`.
5. At each square, check whether a queen exists there.

Since queen positions are stored in a hash set, this lookup is efficient.
6. If a queen is found, add it to the answer and stop scanning in that direction.

This queen is the closest queen in that direction, so it blocks all others behind it.
7. Repeat for all eight directions.
8. Return the collected attacking queens.

### Why it works

The algorithm works because queens attack in straight lines. In any direction from the king, only the nearest queen can attack. Any queen farther away is blocked by the first queen encountered. By scanning outward from the king and stopping at the first queen in each direction, we guarantee that every returned queen has a clear path to the king and that no valid attacking queen is missed.

## Python Solution

```python
from typing import List

class Solution:
    def queensAttacktheKing(self, queens: List[List[int]], king: List[int]) -> List[List[int]]:
        queen_positions = set()

        for row, col in queens:
            queen_positions.add((row, col))

        directions = [
            (-1, 0),   # up
            (1, 0),    # down
            (0, -1),   # left
            (0, 1),    # right
            (-1, -1),  # top-left
            (-1, 1),   # top-right
            (1, -1),   # bottom-left
            (1, 1)     # bottom-right
        ]

        result = []

        for dx, dy in directions:
            row, col = king

            while True:
                row += dx
                col += dy

                if not (0 <= row < 8 and 0 <= col < 8):
                    break

                if (row, col) in queen_positions:
                    result.append([row, col])
                    break

        return result
```

The implementation begins by converting the list of queens into a set of coordinate tuples. This allows constant-time lookup when checking whether a square contains a queen.

The `directions` list encodes the eight possible movement directions a queen can attack from. Each pair represents how the row and column change during movement.

For every direction, the algorithm starts from the king's position and repeatedly moves one square farther in that direction. If the new position leaves the board, scanning stops. If a queen is encountered, it is added to the result immediately and scanning in that direction ends.

This matches the chess rule that the nearest queen blocks all others behind it.

## Go Solution

```go
func queensAttacktheKing(queens [][]int, king []int) [][]int {
    queenPositions := make(map[[2]int]bool)

    for _, queen := range queens {
        position := [2]int{queen[0], queen[1]}
        queenPositions[position] = true
    }

    directions := [][]int{
        {-1, 0},  // up
        {1, 0},   // down
        {0, -1},  // left
        {0, 1},   // right
        {-1, -1}, // top-left
        {-1, 1},  // top-right
        {1, -1},  // bottom-left
        {1, 1},   // bottom-right
    }

    result := [][]int{}

    for _, direction := range directions {
        dx := direction[0]
        dy := direction[1]

        row := king[0]
        col := king[1]

        for {
            row += dx
            col += dy

            if row < 0 || row >= 8 || col < 0 || col >= 8 {
                break
            }

            position := [2]int{row, col}

            if queenPositions[position] {
                result = append(result, []int{row, col})
                break
            }
        }
    }

    return result
}
```

The Go implementation follows the same logic as the Python version. Since Go does not have a built-in hash set type, a `map[[2]int]bool` is used to represent queen positions.

Arrays of fixed size can be used as map keys in Go, which makes `[2]int` a convenient coordinate representation.

The result is built using slices, and directional scanning proceeds exactly as in the Python solution.

There are no integer overflow concerns because all coordinates remain within the range `0` to `7`.

| Measure | Complexity | Explanation |

The board size is fixed at `8 x 8`, so directional scanning examines at most `8 × 8 = 64` positions. Therefore, the runtime is effectively linear in the number of queens. The extra memory usage comes entirely from storing queen coordinates in a hash set.

## Edge Cases

One important edge case occurs when multiple queens lie along the same attack direction. For example, if two queens are on the same row to the right of the king, only the closer queen can attack because it blocks the farther queen. A naive implementation might incorrectly include both queens. The directional scanning approach avoids this issue naturally because it stops immediately after finding the first queen.

Another important case is when the king is located near the edge or corner of the board. Some directions immediately go out of bounds. Incorrect boundary handling could cause index errors or infinite loops. The implementation explicitly checks whether coordinates remain between `0` and `7` before continuing the scan.

A third edge case is when no queens can attack the king. Some implementations accidentally return unrelated aligned queens or fail to handle an empty answer properly. This implementation only adds queens when they are the first visible queen in a direction, so if none satisfy that condition, the result remains an empty list.

A final subtle case involves diagonal alignment. A queen attacks diagonally only when the absolute row difference equals the absolute column difference. By scanning actual board directions instead of relying on arithmetic checks alone, the implementation correctly handles all diagonal cases without requiring special formulas.
