## Solution

## Problem Understanding

The problem asks us to generate a monthly summary report of transactions and chargebacks per country. We have two tables: `Transactions` and `Chargebacks`. The `Transactions` table contains transaction data including an `id`, `country`, `state` (approved or declined), `amount`, and `trans_date`. The `Chargebacks` table contains records of chargebacks linked to transactions by `trans_id` and their own `trans_date`.

Our task is to compute for each month and country: the count and total amount of approved transactions, and the count and total amount of chargebacks. The month is extracted from `trans_date` in the format `YYYY-MM`. We must omit rows where all four values (`approved_count`, `approved_amount`, `chargeback_count`, `chargeback_amount`) are zero. The result can be returned in any order.

Important constraints and observations include: `Transactions.id` is unique, `Chargebacks.trans_id` always refers to an existing transaction, and the dataset could include months without any approved transactions but with chargebacks, or vice versa. Edge cases could include months with only declined transactions, chargebacks referencing declined transactions, or months with multiple countries.

## Approaches

A brute-force approach would involve iterating over all transactions month by month, filtering approved transactions for counting and summing, and separately iterating over chargebacks to compute their counts and sums. This would involve multiple nested loops or repeated aggregation queries, which can be inefficient on large datasets.

The optimal approach leverages SQL aggregation functions and joins. We first aggregate `Transactions` by month and country, summing approved transactions and counting them. Separately, we join `Chargebacks` with `Transactions` to obtain their `country` and `month` and then aggregate chargebacks similarly. Finally, we perform a full outer join on month and country to combine both aggregates, using `COALESCE` to handle nulls and filtering out rows with all zeros. This approach avoids multiple passes and leverages the database engine's efficient grouping and joining capabilities.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(T * C) | O(T + C) | Iterate over all transactions and chargebacks for each month-country pair |
| Optimal | O(T + C) | O(M) | Aggregate transactions and chargebacks separately and join by month-country |

Here `T` is the number of transactions, `C` is the number of chargebacks, and `M` is the number of distinct month-country pairs.

## Algorithm Walkthrough

1. Extract the month in `YYYY-MM` format from `trans_date` in `Transactions` and `Chargebacks`.
2. Filter `Transactions` to only include approved ones, then group by month and country. Compute `approved_count` as the number of transactions and `approved_amount` as the sum of `amount`.
3. Join `Chargebacks` with `Transactions` on `trans_id` to obtain the associated country. Extract the month from `Chargebacks.trans_date` and group by month and country. Compute `chargeback_count` as the number of chargebacks and `chargeback_amount` as the sum of the original transaction amounts.
4. Perform a full outer join between the two aggregates on month and country. Use `COALESCE` to replace nulls with zeroes.
5. Filter out rows where all four values are zero.
6. Return the result with columns: `month`, `country`, `approved_count`, `approved_amount`, `chargeback_count`, `chargeback_amount`.

Why it works: Each aggregation computes totals per month-country independently, and the join ensures that any month-country with either approved transactions or chargebacks is included. Using `COALESCE` guarantees numeric zeroes instead of nulls, and filtering zeros ensures only meaningful rows are returned.

## Python Solution

```python
import pandas as pd

def monthly_transactions(transactions: pd.DataFrame, chargebacks: pd.DataFrame) -> pd.DataFrame:
    transactions['month'] = transactions['trans_date'].dt.to_period('M').astype(str)
    approved = transactions[transactions['state'] == 'approved']
    approved_agg = approved.groupby(['month', 'country'], as_index=False).agg(
        approved_count=('id', 'count'),
        approved_amount=('amount', 'sum')
    )
    
    chargebacks = chargebacks.merge(transactions[['id', 'country', 'amount']], left_on='trans_id', right_on='id', how='left')
    chargebacks['month'] = chargebacks['trans_date'].dt.to_period('M').astype(str)
    chargeback_agg = chargebacks.groupby(['month', 'country'], as_index=False).agg(
        chargeback_count=('trans_id', 'count'),
        chargeback_amount=('amount', 'sum')
    )
    
    result = pd.merge(approved_agg, chargeback_agg, on=['month', 'country'], how='outer')
    result = result.fillna(0)
    result = result.astype({'approved_count': int, 'approved_amount': int, 'chargeback_count': int, 'chargeback_amount': int})
    result = result[(result['approved_count'] + result['approved_amount'] + result['chargeback_count'] + result['chargeback_amount']) > 0]
    return result
```

The Python implementation first extracts months, aggregates approved transactions, aggregates chargebacks with transaction amounts, merges the two aggregates, replaces nulls with zeros, converts counts and sums to integers, and filters out all-zero rows.

## Go Solution

```go
package main

import (
    "database/sql"
    _ "github.com/go-sql-driver/mysql"
    "fmt"
)

func MonthlyTransactions(db *sql.DB) (*sql.Rows, error) {
    query := `
    WITH approved AS (
        SELECT 
            DATE_FORMAT(trans_date, '%Y-%m') AS month,
            country,
            COUNT(*) AS approved_count,
            SUM(amount) AS approved_amount
        FROM Transactions
        WHERE state = 'approved'
        GROUP BY month, country
    ),
    chargebacks_agg AS (
        SELECT 
            DATE_FORMAT(c.trans_date, '%Y-%m') AS month,
            t.country,
            COUNT(*) AS chargeback_count,
            SUM(t.amount) AS chargeback_amount
        FROM Chargebacks c
        JOIN Transactions t ON c.trans_id = t.id
        GROUP BY month, t.country
    )
    SELECT 
        COALESCE(a.month, c.month) AS month,
        COALESCE(a.country, c.country) AS country,
        COALESCE(approved_count, 0) AS approved_count,
        COALESCE(approved_amount, 0) AS approved_amount,
        COALESCE(chargeback_count, 0) AS chargeback_count,
        COALESCE(chargeback_amount, 0) AS chargeback_amount
    FROM approved a
    FULL OUTER JOIN chargebacks_agg c
    ON a.month = c.month AND a.country = c.country
    WHERE COALESCE(approved_count, 0) + COALESCE(approved_amount, 0) + COALESCE(chargeback_count, 0) + COALESCE(chargeback_amount, 0) > 0
    `
    return db.Query(query)
}
```

The Go solution uses a SQL query with Common Table Expressions (CTEs) to perform the aggregation, join, and filtering. The main difference is leveraging the SQL engine for all operations instead of pandas DataFrames. `COALESCE` ensures no nulls, and `FULL OUTER JOIN` includes all month-country combinations.
