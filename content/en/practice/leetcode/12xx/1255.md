**Difficulty:** 🔴 Hard  
**Topics:** Array, Hash Table, String, Dynamic Programming, Backtracking, Bit Manipulation, Counting, Bitmask

## Solution

## Problem Understanding

This problem asks us to choose a subset of words that can be formed using a limited pool of letters, while maximizing the total score of the chosen words.

Each word consists of lowercase English letters. We are also given a multiset of available letters, meaning letters may repeat and each occurrence can only be used once. Additionally, every character from `'a'` to `'z'` has an associated score value.

The goal is to determine the maximum total score obtainable by selecting some combination of words such that:

- Every selected word can be constructed from the available letters
- No letter is used more times than it appears in `letters`
- Each word can be chosen at most once
- We are allowed to skip words
- We are not required to use all letters

The important observation is that the number of words is very small:

```
1 <= words.length <= 14
```

This is the key constraint that shapes the solution. Since there are at most 14 words, the total number of subsets is:

```
2^14 = 16384
```

That is small enough to explore exhaustively with backtracking or bitmask enumeration.

At the same time, individual words can be up to length 15, and the total number of available letters can reach 100. These constraints tell us that checking letter frequencies is inexpensive compared to subset enumeration.

Several edge cases are important:

- A word may require letters that do not exist in the available pool
- Multiple valid words may compete for the same limited letters
- The optimal answer may involve skipping a high-scoring word in favor of several medium-scoring words
- Some letters may have score 0, meaning a word can consume resources without increasing score
- It may be impossible to form any word at all, in which case the answer is 0

The problem guarantees that all characters are lowercase English letters, which allows us to use fixed-size arrays of length 26 for frequency counting.

## Approaches

### Brute Force Approach

The most direct approach is to generate every possible subset of words.

For each subset:

1. Count how many times each letter is needed
2. Compare those counts against the available letters
3. If the subset is valid, compute its total score
4. Track the maximum score encountered

This approach is correct because every possible combination of words is examined, so the optimal solution cannot be missed.

However, the naive implementation can become inefficient if we repeatedly recompute character frequencies and scores from scratch for every subset. Since each subset may require scanning all words and characters again, unnecessary repeated work accumulates.

Still, because the number of words is at most 14, even exponential exploration is feasible.

### Optimal Approach

The key insight is that the search space is small enough for backtracking.

At every word index, we only have two choices:

- Skip the current word
- Take the current word, if enough letters remain

This naturally forms a binary decision tree.

We can preprocess:

- The frequency count of each word
- The score of each word

Then during recursion:

- We attempt to include the current word
- We verify whether enough letters remain
- If valid, we temporarily consume those letters
- We recurse to the next word
- We backtrack by restoring the consumed letters

This avoids repeatedly recomputing counts and scores from scratch.

The recursion efficiently explores all valid combinations while maintaining the current state incrementally.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n × n × 26) | O(26) | Rebuilds counts for every subset |
| Optimal | O(2^n × 26) | O(26 + n) | Backtracking with incremental updates |

## Algorithm Walkthrough

1. First, count the frequency of every available letter.

Since all characters are lowercase English letters, we use an array of length 26 where index `0` represents `'a'`, index `1` represents `'b'`, and so on.
2. Precompute the character frequency for every word.

For each word, create another length-26 array storing how many times each character appears in that word.

This preprocessing avoids recalculating frequencies during recursion.
3. Precompute the score of every word.

For each character in a word, add its corresponding value from the `score` array.

This lets us instantly know the contribution of including a word.
4. Define a recursive backtracking function.

The function takes:

- The current word index
- The remaining available letters

It returns the maximum score obtainable from the remaining words.
5. At each recursive step, consider skipping the current word.

This branch simply moves to the next index without changing available letters.
6. Then attempt to include the current word.

To do this:

- Check whether every required character count is available
- If any required count exceeds availability, the word cannot be chosen
7. If the word is valid:

- Subtract its character counts from the remaining letters
- Add the word's score
- Recurse to the next word
- Restore the removed counts afterward

This restoration step is the essence of backtracking.
8. Return the maximum score between:

- Skipping the word
- Taking the word
9. Start recursion from index 0 with the original letter counts.

### Why it works

The algorithm explores every possible subset of words through binary decisions at each index. Every valid subset corresponds to exactly one path in the recursion tree.

Because we only allow inclusion when enough letters remain, every constructed solution is valid. Since all valid subsets are explored and the maximum score is tracked, the algorithm is guaranteed to return the optimal answer.

## Python Solution

```python
from typing import List

class Solution:
    def maxScoreWords(self, words: List[str], letters: List[str], score: List[int]) -> int:
        available = [0] * 26
        
        for ch in letters:
            available[ord(ch) - ord('a')] += 1
        
        word_counts = []
        word_scores = []
        
        for word in words:
            count = [0] * 26
            total_score = 0
            
            for ch in word:
                idx = ord(ch) - ord('a')
                count[idx] += 1
                total_score += score[idx]
            
            word_counts.append(count)
            word_scores.append(total_score)
        
        n = len(words)
        
        def backtrack(index: int) -> int:
            if index == n:
                return 0
            
            best = backtrack(index + 1)
            
            can_take = True
            
            for i in range(26):
                if word_counts[index][i] > available[i]:
                    can_take = False
                    break
            
            if can_take:
                for i in range(26):
                    available[i] -= word_counts[index][i]
                
                best = max(
                    best,
                    word_scores[index] + backtrack(index + 1)
                )
                
                for i in range(26):
                    available[i] += word_counts[index][i]
            
            return best
        
        return backtrack(0)
```

The implementation begins by constructing the frequency array for the available letters. This array represents the mutable state of remaining resources during recursion.

Next, the solution preprocesses every word into two structures:

- `word_counts`, storing character frequencies
- `word_scores`, storing total word scores

This preprocessing is important because recursive calls happen many times, and recomputing these values repeatedly would waste time.

The `backtrack` function explores all decisions from the current index onward.

The first recursive branch skips the current word. This guarantees that every subset configuration is considered.

The second branch attempts to include the word. Before inclusion, the algorithm verifies that enough letters remain. If valid, the character counts are temporarily subtracted from `available`.

After recursion finishes, the removed letters are restored. This restoration ensures sibling recursive branches see the correct state.

Finally, the function returns the best score obtainable from the current decision point.

## Go Solution

```go
func maxScoreWords(words []string, letters []byte, score []int) int {
	available := make([]int, 26)

	for _, ch := range letters {
		available[ch-'a']++
	}

	wordCounts := make([][]int, len(words))
	wordScores := make([]int, len(words))

	for i, word := range words {
		count := make([]int, 26)
		totalScore := 0

		for _, ch := range word {
			idx := ch - 'a'
			count[idx]++
			totalScore += score[idx]
		}

		wordCounts[i] = count
		wordScores[i] = totalScore
	}

	var backtrack func(int) int

	backtrack = func(index int) int {
		if index == len(words) {
			return 0
		}

		best := backtrack(index + 1)

		canTake := true

		for i := 0; i < 26; i++ {
			if wordCounts[index][i] > available[i] {
				canTake = false
				break
			}
		}

		if canTake {
			for i := 0; i < 26; i++ {
				available[i] -= wordCounts[index][i]
			}

			scoreWithWord := wordScores[index] + backtrack(index+1)

			if scoreWithWord > best {
				best = scoreWithWord
			}

			for i := 0; i < 26; i++ {
				available[i] += wordCounts[index][i]
			}
		}

		return best
	}

	return backtrack(0)
}
```

The Go implementation closely mirrors the Python solution, but uses slices instead of Python lists.

Character arithmetic is slightly different because Go characters are bytes or runes. Subtracting `'a'` converts characters into indices from 0 to 25.

The recursive function is declared as a function variable so it can recursively reference itself.

Go slices are mutable references, so the `available` slice is modified in place during recursion and restored afterward during backtracking.

Integer overflow is not a concern because the maximum possible score is small relative to Go's integer range.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(2^n × 26) | Each recursive state checks up to 26 letters |
| Space | O(26 + n) | Frequency arrays and recursion stack |

The algorithm explores all subsets of the `n` words. Since each recursive level branches into two possibilities, the total number of states is `2^n`.

For every state, validating or applying a word requires scanning at most 26 characters because the alphabet size is fixed.

The space complexity is small because the algorithm only stores:

- Fixed-size frequency arrays
- Precomputed word data
- Recursive call stack of depth `n`

Since `n <= 14`, recursion depth remains safe.

## Edge Cases

One important edge case occurs when no words can be formed at all. A naive implementation might incorrectly assume at least one valid word exists, but the correct answer in this situation is 0. The backtracking solution naturally handles this because the "skip" branch is always available, allowing the recursion to return 0 when no inclusion branch is valid.

Another tricky case happens when multiple words compete for the same scarce letters. For example, one large high-scoring word may prevent several medium-scoring words from being selected together. Greedy strategies often fail here because the locally best choice is not necessarily globally optimal. The recursive subset exploration guarantees that every valid combination is considered.

Repeated letters are another common source of bugs. A word like `"leetcode"` may require several copies of the same character. Implementations that only check letter existence instead of frequency counts will produce incorrect results. Using fixed-size frequency arrays ensures exact usage accounting for every character.

A subtle edge case involves letters with score 0. Some words may consume valuable letters while contributing nothing to the final score. Since the algorithm evaluates all combinations based on total score, these words are naturally excluded unless they somehow enable a better combination, which in this problem they cannot because words are independent.

Finally, the recursion must properly restore state after exploring a branch. Forgetting to backtrack letter counts would corrupt subsequent recursive calls and produce incorrect results. The implementation carefully subtracts counts before recursion and restores them immediately afterward.
