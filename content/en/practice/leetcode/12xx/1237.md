## Solution

## Problem Understanding

The problem asks us to find all positive integer pairs `(x, y)` such that a hidden function `f(x, y)` equals a given integer `z`. The function `f(x, y)` is provided through an interface `CustomFunction`, and its formula is unknown. The only guaranteed property is that `f` is monotonically increasing with respect to both `x` and `y`, meaning `f(x + 1, y) > f(x, y)` and `f(x, y + 1) > f(x, y)` for all positive integers `x` and `y`.

Inputs include the function instance and the target value `z`. Output is a list of pairs `[x, y]` where `f(x, y) == z`. All pairs have positive integers and are guaranteed to exist within `1 <= x, y <= 1000`. The constraints also guarantee that the resulting values will fit within a 32-bit signed integer.

Key edge cases include situations where `z` is very small, such as `z = 1`, which may only allow a few solutions near `(1,1)`. Similarly, if `z` is very large, only pairs with larger `x` and `y` can satisfy the equality, but the upper bound of 1000 keeps the search space finite. The monotonicity property is critical for designing an efficient algorithm.

## Approaches

The brute-force approach iterates over all possible pairs `(x, y)` in the range `[1, 1000]`. For each pair, it calls `f(x, y)` and checks if it equals `z`. While this guarantees correctness, it results in up to 1,000,000 calls, which is inefficient and unnecessary given the monotonicity property.

The optimal approach uses a two-pointer strategy. Start with `x = 1` and `y = 1000` (the maximum allowed). Because the function is increasing in both directions, if `f(x, y) > z`, decreasing `y` may help; if `f(x, y) < z`, increasing `x` may help. This approach allows systematically exploring all pairs without testing every combination.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N^2) | O(1) | Iterate through all x and y from 1 to 1000 |
| Two-Pointer Optimal | O(N) | O(1) | Use monotonicity to adjust pointers and avoid unnecessary calls |

## Algorithm Walkthrough

1. Initialize an empty list `result` to store valid pairs.
2. Set `x = 1` (smallest possible positive integer) and `y = 1000` (largest possible positive integer).
3. While `x <= 1000` and `y >= 1`, perform the following:

1. Compute `current = customfunction.f(x, y)`.
2. If `current == z`, append `[x, y]` to `result`, then increment `x` and decrement `y` to explore new possible pairs.
3. If `current < z`, increment `x` because increasing `x` increases the function due to monotonicity.
4. If `current > z`, decrement `y` because decreasing `y` reduces the function value.
4. Return the `result` list containing all valid pairs.

**Why it works:** The monotonicity property guarantees that increasing `x` or decreasing `y` will consistently move towards `z`. Each step eliminates impossible regions, ensuring all valid pairs are found without missing any.

## Python Solution

```python
from typing import List

class Solution:
    def findSolution(self, customfunction: 'CustomFunction', z: int) -> List[List[int]]:
        result = []
        x, y = 1, 1000
        
        while x <= 1000 and y >= 1:
            current = customfunction.f(x, y)
            if current == z:
                result.append([x, y])
                x += 1
                y -= 1
            elif current < z:
                x += 1
            else:
                y -= 1
        
        return result
```

**Explanation:** We maintain two pointers `x` and `y` at opposite ends of the range. By checking `current` against `z`, we adjust the pointers based on the monotonicity. Valid pairs are added immediately to the result. This ensures all pairs are found efficiently.

## Go Solution

```go
func findSolution(customFunction func(int, int) int, z int) [][]int {
    result := [][]int{}
    x, y := 1, 1000

    for x <= 1000 && y >= 1 {
        current := customFunction(x, y)
        if current == z {
            result = append(result, []int{x, y})
            x++
            y--
        } else if current < z {
            x++
        } else {
            y--
        }
    }

    return result
}
```

**Explanation:** The Go implementation mirrors the Python logic. A slice of slices is used to store results. Integer overflow is not a concern due to constraints. The two-pointer approach allows linear exploration instead of exhaustive search.
