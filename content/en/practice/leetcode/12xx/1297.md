## Solution

## Problem Understanding

The problem asks us to analyze a string `s` and determine the maximum number of times any substring appears while satisfying certain constraints. Specifically, a substring must have no more than `maxLetters` unique characters, and its length must be between `minSize` and `maxSize` inclusive. We are asked to return the count of the most frequently occurring valid substring.

In other words, we are scanning through `s` to find substrings that are constrained by size and diversity of letters, counting how many times each appears, and returning the highest count. The input string can be up to 100,000 characters long, so naive approaches that consider all possible substrings of all lengths up to `maxSize` are likely too slow.

Constraints provide important guidance: `minSize` and `maxSize` are capped at 26 or the string length, whichever is smaller, which is a hint that the substring sizes are small enough to consider hashing or sliding window techniques. The string only contains lowercase letters, making it feasible to track character frequencies with arrays or hash maps efficiently.

Edge cases to consider include strings with all identical letters, strings shorter than `minSize`, `maxLetters` equal to 1, and overlapping substrings. The problem guarantees at least one substring exists if `s.length >= minSize`.

## Approaches

### Brute Force

A brute-force approach would generate all possible substrings of lengths ranging from `minSize` to `maxSize`, count their unique characters, and maintain a frequency map for valid substrings. After checking all substrings, we would return the maximum count. This approach works because it exhaustively checks every valid candidate, ensuring correctness.

However, this method is too slow. For a string of length `n`, there are approximately $O(n^2)$ substrings, and for each, counting unique characters is $O(minSize)$ to $O(maxSize)$. With `n` up to $10^5$, this results in a runtime far exceeding reasonable limits.

### Optimal Approach

The key insight is that we do not need to consider substrings longer than `minSize`. If a longer substring appears frequently, its `minSize` portion will appear at least as many times. Therefore, it is sufficient to only consider substrings of length `minSize`.

Using a sliding window of size `minSize`, we can efficiently check each substring and count its occurrences. A hash map (dictionary) tracks the frequency of each valid substring, and a simple character frequency array ensures we respect the `maxLetters` constraint. This reduces time complexity significantly to $O(n \cdot minSize)$, which is feasible for `n` up to $10^5$.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * maxSize^2) | O(n * maxSize) | Generates all substrings and counts unique letters |
| Optimal | O(n * minSize) | O(n * minSize) | Sliding window + hash map counting only substrings of length `minSize` |

## Algorithm Walkthrough

1. Initialize a hash map `freq` to store the frequency of valid substrings and an array `char_count` to track character occurrences in the current window.
2. Set up a sliding window of size `minSize` and a variable `unique_count` to track the number of unique characters in the current window.
3. Move the window across the string one character at a time:

1. Add the new character entering the window to `char_count`. If it is a new character (count goes from 0 to 1), increment `unique_count`.
2. Remove the character leaving the window from `char_count`. If its count drops to 0, decrement `unique_count`.
3. If `unique_count` <= `maxLetters`, add the current substring to `freq` and increment its count.
4. Track the maximum frequency seen in `freq`.
5. Return the maximum frequency.

**Why it works**: By focusing on substrings of length `minSize`, we guarantee that any substring longer than `minSize` cannot have a higher frequency without its `minSize` portion also appearing at least as often. The sliding window ensures we only process each character once, maintaining correct counts of unique characters in constant time.

## Python Solution

```python
class Solution:
    def maxFreq(self, s: str, maxLetters: int, minSize: int, maxSize: int) -> int:
        from collections import defaultdict
        
        freq = defaultdict(int)
        char_count = [0] * 26
        unique_count = 0
        max_freq = 0
        
        for i in range(len(s)):
            # Add new character
            idx_in = ord(s[i]) - ord('a')
            char_count[idx_in] += 1
            if char_count[idx_in] == 1:
                unique_count += 1
            
            # Remove old character when window exceeds minSize
            if i >= minSize:
                idx_out = ord(s[i - minSize]) - ord('a')
                char_count[idx_out] -= 1
                if char_count[idx_out] == 0:
                    unique_count -= 1
            
            # Check window frequency
            if i >= minSize - 1:
                if unique_count <= maxLetters:
                    substr = s[i - minSize + 1:i + 1]
                    freq[substr] += 1
                    max_freq = max(max_freq, freq[substr])
        
        return max_freq
```

The code sets up a sliding window to track substrings of `minSize`. Each time the window moves, it updates character counts and checks the unique letters condition. Valid substrings are counted in a hash map, and the highest frequency is tracked throughout.

## Go Solution

```go
func maxFreq(s string, maxLetters int, minSize int, maxSize int) int {
    freq := make(map[string]int)
    charCount := make([]int, 26)
    uniqueCount := 0
    maxFreq := 0
    
    for i := 0; i < len(s); i++ {
        idxIn := s[i] - 'a'
        charCount[idxIn]++
        if charCount[idxIn] == 1 {
            uniqueCount++
        }
        
        if i >= minSize {
            idxOut := s[i - minSize] - 'a'
            charCount[idxOut]--
            if charCount[idxOut] == 0 {
                uniqueCount--
            }
        }
        
        if i >= minSize - 1 && uniqueCount <= maxLetters {
            substr := s[i - minSize + 1:i + 1]
            freq[substr]++
            if freq[substr] > maxFreq {
                maxFreq = freq[substr]
            }
        }
    }
    
    return maxFreq
}
```

In Go, we use a fixed-size array for character counts and a map for substring frequencies. The logic mirrors the Python solution, with care for zero-based indices and slice extraction.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * minSize) | We iterate over each character once and slice substrings of length `minSize`. |
| Space | O(n * minSize) | The hash map may store up to n substrings of length minSize. |

The algorithm scales linearly with the string length for small `minSize`. Space complexity is dominated by storing the substrings themselves, which are small relative to `n`.
