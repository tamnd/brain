## Solution

## Problem Understanding

The problem asks us to process a list of intervals, where each interval is represented as a pair `[li, ri)` denoting the half-open interval starting at `li` and ending just before `ri`. Our task is to remove intervals that are "covered" by another interval in the list. An interval `[a, b)` is covered by `[c, d)` if `c <= a` and `b <= d`, meaning `[c, d)` fully contains `[a, b)`.

The input is an array of unique intervals, with constraints that each interval's start and end are non-negative and `li < ri`. The number of intervals can be up to 1000, and interval values can go up to `10^5`. The expected output is simply an integer representing how many intervals remain after removing all covered intervals.

Edge cases that could trip up a naive implementation include intervals that exactly overlap, intervals that share endpoints, or intervals that are subsets of multiple other intervals. The uniqueness guarantee removes concerns about duplicate intervals but does not prevent multiple intervals from covering the same one.

## Approaches

A brute-force solution would compare every interval against every other interval to check if it is covered. For each interval `[i]`, we would iterate through all other intervals `[j]` and check if `[j]` covers `[i]`. If it does, we discard `[i]`. This approach is correct but inefficient, with a worst-case time complexity of `O(n^2)`.

A more optimal solution leverages sorting. The key observation is that if we sort intervals first by their start point in ascending order and then by their end point in descending order, we can efficiently detect coverage in a single pass. The descending sort on the end ensures that if two intervals start at the same point, the longer one comes first, preventing a shorter interval from being incorrectly counted as non-covered. Then, we iterate through the sorted intervals, tracking the maximum end seen so far. If the current interval’s end is less than or equal to the maximum end, it is covered; otherwise, it is not covered and we update the maximum end.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Compare each interval against all others to check coverage |
| Optimal | O(n log n) | O(1) | Sort intervals and track the maximum end in a single pass |

## Algorithm Walkthrough

1. Sort the intervals by start in ascending order. If two intervals have the same start, sort by end in descending order. This guarantees that any interval that could cover another comes before it in the list.
2. Initialize a variable `max_end` to zero. This variable will track the largest end value of intervals seen so far that are not covered.
3. Initialize a counter `remaining` to zero. This will count intervals that are not covered.
4. Iterate through each interval `[start, end]` in the sorted list.
5. For each interval, compare its `end` with `max_end`.

- If `end` is less than or equal to `max_end`, it is covered by a previous interval, so skip it.
- Otherwise, it is not covered. Increment `remaining` and update `max_end` to `end`.
6. Return `remaining` as the number of intervals left after removing covered intervals.

**Why it works:** Sorting ensures that potential covering intervals always appear before intervals that could be covered. Tracking `max_end` guarantees that any interval with a smaller or equal end is covered by a previously seen interval, making the single pass sufficient for correctness.

## Python Solution

```python
from typing import List

class Solution:
    def removeCoveredIntervals(self, intervals: List[List[int]]) -> int:
        # Step 1: Sort intervals by start ascending, end descending
        intervals.sort(key=lambda x: (x[0], -x[1]))
        
        max_end = 0
        remaining = 0
        
        # Step 2: Iterate through sorted intervals
        for start, end in intervals:
            # Step 3: Check if current interval is covered
            if end > max_end:
                remaining += 1
                max_end = end
        
        return remaining
```

The code first sorts intervals to ensure that longer intervals come before shorter ones when starting points match. It then iterates through, using `max_end` to track the farthest endpoint seen. If an interval extends beyond `max_end`, it is not covered, so we increment `remaining` and update `max_end`. This matches the algorithm walkthrough exactly.

## Go Solution

```go
import "sort"

func removeCoveredIntervals(intervals [][]int) int {
    // Step 1: Sort intervals by start ascending, end descending
    sort.Slice(intervals, func(i, j int) bool {
        if intervals[i][0] == intervals[j][0] {
            return intervals[i][1] > intervals[j][1]
        }
        return intervals[i][0] < intervals[j][0]
    })

    maxEnd := 0
    remaining := 0

    // Step 2: Iterate through sorted intervals
    for _, interval := range intervals {
        start, end := interval[0], interval[1]
        if end > maxEnd {
            remaining++
            maxEnd = end
        }
    }

    return remaining
}
```

In Go, `sort.Slice` allows sorting based on a custom comparator, similar to Python’s lambda sorting. We track `maxEnd` as an integer and update it similarly. Slice iteration works the same way as Python lists, making the Go solution almost a direct translation of the Python logic.

## Edge Cases

The first edge case is when the list contains only one interval. This is a simple scenario, but a naive algorithm that assumes at least two intervals could fail. Our implementation correctly handles it by initializing `remaining` to 0 and incrementing once during iteration.

The second edge case is when multiple intervals start at the same point. Sorting by end descending ensures that the longest interval is processed first, preventing shorter intervals from being counted as non-covered.

The third edge case is when an interval is fully contained within multiple intervals. Because we track `max_end`, once an interval is covered by any previous interval, it is ignored. This guarantees correctness regardless of how many intervals cover it.
