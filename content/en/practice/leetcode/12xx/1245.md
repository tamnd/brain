## Solution

## Problem Understanding

The problem asks us to find the **diameter** of a tree, which is defined as the number of edges in the longest path between any two nodes. The input is a list of edges representing an undirected tree of `n` nodes, where each edge connects two nodes `[ai, bi]`. Our output is a single integer representing the length of the longest path in the tree.

The constraints clarify that the input is guaranteed to be a tree: `n = edges.length + 1`, each node label is unique within `[0, n - 1]`, and no edge connects a node to itself. The problem size can be up to 10,000 nodes, meaning naive solutions with quadratic complexity will be too slow. Important edge cases include a tree with only one node (diameter 0), a linear tree (like a linked list), or trees with branches of very uneven length.

The main insight is that the longest path in a tree always passes through the two farthest nodes. This allows us to compute the diameter efficiently using tree traversal.

## Approaches

A **brute-force approach** would consider each pair of nodes, compute the distance between them, and return the maximum distance. This is correct because it explicitly considers all paths. However, its time complexity is `O(n^2)` or worse, which is too slow for `n = 10^4`.

The **optimal approach** uses a key property of trees: the diameter can be found by performing **two depth-first searches (DFS)** or breadth-first searches (BFS). First, start from any node and find the farthest node `A`. Then, start from `A` and find the farthest node `B` from `A`. The path from `A` to `B` is the diameter. This works because in a tree, there is a unique simple path between any two nodes, and the longest path will always stretch between two leaf nodes that are farthest apart.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Compute distance between all node pairs |
| Optimal | O(n) | O(n) | Two DFS/BFS traversals to find diameter |

## Algorithm Walkthrough

1. Convert the list of edges into an adjacency list representation of the tree. This makes traversal efficient.
2. Start a DFS (or BFS) from any arbitrary node, commonly node 0. Track the farthest node reached during traversal and its distance.
3. Let this farthest node be `A`. Start a second DFS (or BFS) from `A` to find the node farthest from it, called `B`.
4. The distance from `A` to `B` is the tree diameter.
5. Return this distance.

The reasoning behind why this works is that the first DFS ensures that `A` is one endpoint of the longest path. The second DFS from `A` guarantees we reach the other endpoint `B` of the longest path. Since a tree has a unique path between any two nodes, this procedure correctly identifies the diameter.

## Python Solution

```python
from typing import List
from collections import defaultdict, deque

class Solution:
    def treeDiameter(self, edges: List[List[int]]) -> int:
        if not edges:
            return 0
        
        # Build adjacency list
        graph = defaultdict(list)
        for u, v in edges:
            graph[u].append(v)
            graph[v].append(u)
        
        # Helper function for BFS
        def bfs(start: int) -> (int, int):
            visited = set()
            queue = deque([(start, 0)])
            visited.add(start)
            farthest_node, max_dist = start, 0
            
            while queue:
                node, dist = queue.popleft()
                if dist > max_dist:
                    max_dist = dist
                    farthest_node = node
                for neighbor in graph[node]:
                    if neighbor not in visited:
                        visited.add(neighbor)
                        queue.append((neighbor, dist + 1))
            return farthest_node, max_dist
        
        # First BFS to find one endpoint
        far_node, _ = bfs(0)
        # Second BFS to find the diameter
        _, diameter = bfs(far_node)
        return diameter
```

The Python implementation first builds an adjacency list from the edge list for fast traversal. The `bfs` helper function returns both the farthest node and its distance from the start. Running BFS twice ensures we find the longest path. We handle the edge case of an empty tree by returning 0 immediately.

## Go Solution

```go
package main

func treeDiameter(edges [][]int) int {
    if len(edges) == 0 {
        return 0
    }

    // Build adjacency list
    n := len(edges) + 1
    graph := make([][]int, n)
    for _, e := range edges {
        u, v := e[0], e[1]
        graph[u] = append(graph[u], v)
        graph[v] = append(graph[v], u)
    }

    // BFS helper
    bfs := func(start int) (int, int) {
        visited := make([]bool, n)
        queue := []struct{ node, dist int }{{start, 0}}
        visited[start] = true
        farthestNode, maxDist := start, 0

        for len(queue) > 0 {
            curr := queue[0]
            queue = queue[1:]
            if curr.dist > maxDist {
                maxDist = curr.dist
                farthestNode = curr.node
            }
            for _, neighbor := range graph[curr.node] {
                if !visited[neighbor] {
                    visited[neighbor] = true
                    queue = append(queue, struct{ node, dist int }{neighbor, curr.dist + 1})
                }
            }
        }
        return farthestNode, maxDist
    }

    farNode, _ := bfs(0)
    _, diameter := bfs(farNode)
    return diameter
}
```

In Go, we use slices for adjacency lists and a simple struct to store the BFS queue elements. Unlike Python, we explicitly track visited nodes with a boolean slice instead of a set.

## Worked Examples

**Example 1:** `edges = [[0,1],[0,2]]`

1. Build adjacency list: `{0: [1,2], 1:[0], 2:[0]}`
2. BFS from node 0 finds farthest node 1 (distance 1)
3. BFS from node 1 finds farthest node 2 (distance 2)
4. Diameter is 2

**Example 2:** `edges = [[0,1],[1,2],[2,3],[1,4],[4,5]]`

1. Build adjacency list: `{0:[1],1:[0,2,4],2:[1,3],3:[2],4:[1,5],5:[4]}`
2. BFS from node 0 finds farthest node 3 (distance 3)
3. BFS from node 3 finds farthest node 5 (distance 4)
4. Diameter is 4

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Two BFS traversals, each visiting all nodes once |
| Space | O(n) | Adjacency list and BFS queue/store visited nodes |

Each BFS takes O(n) time and O(n) space. Building the adjacency list is also O(n), so the total complexity is linear in the number of nodes.

## Edge Cases

**Single node tree:** A tree with no edges has diameter 0. BFS from any node returns distance 0, which the implementation correctly handles.

**Two-node tree:** A tree with one edge has diameter 1. The BFS procedure naturally finds this since the farthest node from one endpoint is the other node.

**Linear tree (linked list):** The longest path is from one end to the other. The two-pass BFS ensures the diameter is correctly identified, even if the tree is skewed with many nodes on a single branch. This approach avoids recursion depth issues that could arise with DFS in Python for very deep trees.
