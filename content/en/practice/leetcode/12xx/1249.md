## Solution

## Problem Understanding

The problem gives a string containing three possible types of characters:

- Opening parentheses, `'('`
- Closing parentheses, `')'`
- Lowercase English letters

The goal is to remove the minimum number of parentheses so that the remaining string becomes a valid parentheses string.

A valid parentheses string follows the standard balancing rules:

- Every opening parenthesis must eventually be matched by a closing parenthesis.
- No closing parenthesis may appear before its corresponding opening parenthesis.
- Letters are always allowed and do not affect validity.

The important detail is that we are only allowed to remove parentheses, not letters. We may remove characters from any positions in the string, and we can return any valid result as long as the number of removed parentheses is minimal.

For example:

- `"a)b(c)d"` becomes `"ab(c)d"` because the first `')'` has no matching `'('`.
- `"))(("` becomes `""` because none of the parentheses can be matched.

The constraints are important:

- `1 <= s.length <= 10^5`

This immediately tells us that expensive algorithms are not feasible. Any brute-force strategy that explores many combinations of removals will become far too slow. The solution should ideally run in linear time, or close to it.

Several edge cases are especially important:

- Strings with only invalid closing parentheses, such as `")))"`.
- Strings with only unmatched opening parentheses, such as `"((("`.
- Strings that are already valid.
- Strings containing many letters intermixed with parentheses.
- Nested parentheses structures.
- Multiple valid answers.

The problem guarantees only lowercase letters and parentheses, so we do not need to handle any other characters.

## Approaches

### Brute Force Approach

A brute-force solution would try removing different combinations of parentheses and checking whether the resulting string is valid.

One way to do this is:

1. Generate all possible strings formed by removing parentheses.
2. Check each generated string for validity.
3. Keep the valid string that removes the fewest characters.

To validate a string, we can scan left to right:

- Increment a counter for `'('`
- Decrement for `')'`
- If the counter ever becomes negative, the string is invalid
- At the end, the counter must be zero

This approach is correct because it explores every possible removal combination, guaranteeing that one of the minimal valid strings will be found.

However, it is computationally infeasible. If the string contains many parentheses, the number of subsets grows exponentially. With length up to `10^5`, this approach is completely impractical.

### Optimal Approach

The key observation is that invalid parentheses can be identified during a single scan.

There are only two kinds of invalid parentheses:

- A closing parenthesis `')'` that appears before a matching `'('`
- An opening parenthesis `'('` that never finds a matching `')'`

We can handle these efficiently using a stack or index tracking.

The idea is:

1. Scan the string left to right.
2. Track indices of unmatched `'('`.
3. Whenever we encounter `')'`:

- If there is an unmatched `'('`, match them.
- Otherwise, mark this `')'` for removal.
4. After the scan, any remaining unmatched `'('` must also be removed.

This works because matching parentheses greedily from left to right guarantees that every valid pair is preserved whenever possible.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(2^n) | Generates all removal combinations |
| Optimal | O(n) | O(n) | Single pass with stack and removal tracking |

## Algorithm Walkthrough

1. Convert the string into a mutable structure or prepare a set of indices to remove.

Strings are immutable in Python, so modifying characters directly is inconvenient. Instead, we store indices that should be removed and rebuild the final string afterward.
2. Create a stack to track unmatched opening parentheses.

The stack stores indices of `'('` characters that have not yet been matched with a closing parenthesis.
3. Scan the string from left to right.

At each character:

- If the character is `'('`, push its index onto the stack.
- If the character is `')'`:

- If the stack is not empty, pop one opening parenthesis index, because we found a valid match.
- Otherwise, this closing parenthesis is invalid, so mark its index for removal.
- If the character is a letter, ignore it.
4. After the scan completes, any indices still in the stack represent unmatched `'('`.

These opening parentheses never found matching closing parentheses, so they must also be removed.
5. Build the final string.

Iterate through the original string and append only characters whose indices are not marked for removal.
6. Return the resulting valid string.

### Why it works

The algorithm maintains the invariant that every closing parenthesis is matched with the nearest available unmatched opening parenthesis to its left.

Any closing parenthesis encountered without an available opening parenthesis can never become valid later, so removing it is necessary. Similarly, any opening parenthesis left unmatched after the full scan can never be balanced and must also be removed.

Because we only remove parentheses that are provably impossible to match, the number of removals is minimal.

## Python Solution

```python
class Solution:
    def minRemoveToMakeValid(self, s: str) -> str:
        stack = []
        remove_indices = set()

        for index, char in enumerate(s):
            if char == '(':
                stack.append(index)
            elif char == ')':
                if stack:
                    stack.pop()
                else:
                    remove_indices.add(index)

        while stack:
            remove_indices.add(stack.pop())

        result = []

        for index, char in enumerate(s):
            if index not in remove_indices:
                result.append(char)

        return ''.join(result)
```

The implementation closely follows the algorithm described earlier.

The `stack` stores indices of unmatched opening parentheses. Whenever we encounter `'('`, we push its index. Whenever we encounter `')'`, we attempt to match it with the most recent unmatched `'('`.

If no opening parenthesis exists, the closing parenthesis is invalid and its index is added to `remove_indices`.

After the first scan finishes, any indices remaining in `stack` correspond to unmatched opening parentheses. These are also added to `remove_indices`.

Finally, we rebuild the string by skipping all indices marked for removal.

The reconstruction step uses a list because repeatedly concatenating strings is inefficient in Python. Appending to a list and using `''.join()` is the standard efficient approach.

## Go Solution

```go
func minRemoveToMakeValid(s string) string {
    stack := []int{}
    remove := make(map[int]bool)

    for i, ch := range s {
        if ch == '(' {
            stack = append(stack, i)
        } else if ch == ')' {
            if len(stack) > 0 {
                stack = stack[:len(stack)-1]
            } else {
                remove[i] = true
            }
        }
    }

    for len(stack) > 0 {
        index := stack[len(stack)-1]
        stack = stack[:len(stack)-1]
        remove[index] = true
    }

    result := make([]rune, 0, len(s))

    for i, ch := range s {
        if !remove[i] {
            result = append(result, ch)
        }
    }

    return string(result)
}
```

The Go implementation follows the same logic as the Python version.

The stack is implemented using a slice of integers. We push indices using `append` and pop by reslicing.

A `map[int]bool` is used to track indices that should be removed.

One Go-specific detail is the use of `[]rune` when building the result. Iterating over a string in Go yields runes, so using a rune slice ensures proper character handling.

Unlike Python, Go does not have a built-in mutable string builder using list append semantics, so rune slices are the natural equivalent.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is processed at most twice |
| Space | O(n) | Stack and removal set may store up to n indices |

The algorithm performs a single left-to-right scan of the string and then one reconstruction pass. Each operation on the stack and set is constant time on average.

In the worst case, all parentheses may be unmatched, causing both the stack and removal set to grow to size `n`.

## Edge Cases

One important edge case is a string containing only closing parentheses, such as `"))))"`. A naive implementation might incorrectly attempt to match them or produce negative balance counts. In this solution, every unmatched `')'` is immediately marked for removal because the stack is empty, resulting in the correct empty string.

Another critical edge case is a string containing only opening parentheses, such as `"(((("`. These parentheses cannot be matched because the string ends before any closing parentheses appear. The implementation handles this cleanly by leaving all indices in the stack after the scan and removing them during the cleanup phase.

A third important case involves interleaved letters and parentheses, such as `"a)b(c)d"`. The algorithm must remove only invalid parentheses while preserving all letters and the relative ordering of characters. Because the implementation only marks parentheses indices for removal and reconstructs the string afterward, all non-parenthesis characters remain untouched.

A more subtle edge case is already valid nested input like `"((a)(b))"`. Some incorrect implementations accidentally remove matched parentheses during cleanup. Here, matched opening parentheses are removed from the stack immediately when paired, ensuring only genuinely unmatched parentheses are discarded.

Finally, strings with multiple valid answers must also be handled correctly. For example, `"()())()"` can produce several valid outputs. The problem allows returning any valid minimal-removal result, and this greedy matching approach naturally produces one such valid answer.
