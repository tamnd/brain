## Solution

## Problem Understanding

The problem asks us to determine, from a given list of integers `nums`, how many numbers contain an **even number of digits**. In simpler terms, for each integer in the array, we need to count its digits and check if that count is divisible by 2. The input is an array of integers, where each integer ranges from 1 to 100,000, and the array length ranges from 1 to 500. The output is a single integer representing the count of numbers in the array that have an even number of digits.

The constraints tell us that the input size is relatively small, so even solutions that check each number individually are feasible. However, the numbers themselves can be up to 5 digits, so counting digits efficiently matters. Important edge cases include numbers with exactly one digit (which are odd), numbers at the upper boundary (like 100,000), and arrays with all numbers having the same number of digits.

## Approaches

A **brute-force approach** would involve converting each integer to a string, calculating the length of that string, and checking if the length is even. This is straightforward, guaranteed to be correct, and easy to implement. Since we are iterating over each element and performing a constant-time string conversion, the time complexity is proportional to the number of elements. Given the constraints, this approach is sufficient.

The **optimal approach** leverages a mathematical property: instead of converting numbers to strings, we can count digits by repeatedly dividing the number by 10 until it becomes zero. Each division corresponds to removing one digit. We increment a counter for each division, and finally, we check if the count is even. This approach avoids string conversion and works purely with integer arithmetic. Given the maximum number is 100,000, this is also extremely fast.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * d) | O(1) | Convert each number to a string, check length parity |
| Optimal | O(n * log10(max_num)) | O(1) | Count digits using integer division, avoid string conversion |

## Algorithm Walkthrough

1. Initialize a counter `even_digit_count` to zero. This will track how many numbers have an even number of digits.
2. Iterate through each number `num` in the input array `nums`.
3. For each number, initialize a local variable `digit_count` to zero. This will count the digits of `num`.
4. Use a loop to divide `num` by 10 repeatedly until `num` becomes zero. After each division, increment `digit_count` by one.
5. After counting the digits, check if `digit_count % 2 == 0`. If true, increment `even_digit_count` by one.
6. After iterating through all numbers, return `even_digit_count`.

**Why it works**: This algorithm works because each integer has a well-defined number of digits. By counting digits through repeated division by 10, we correctly measure the length of each number without relying on string conversion. Checking parity with modulo 2 ensures we only count numbers with even digits.

## Python Solution

```python
from typing import List

class Solution:
    def findNumbers(self, nums: List[int]) -> int:
        even_digit_count = 0
        for num in nums:
            digit_count = 0
            current = num
            while current > 0:
                current //= 10
                digit_count += 1
            if digit_count % 2 == 0:
                even_digit_count += 1
        return even_digit_count
```

In this implementation, we initialize a counter `even_digit_count` and iterate through the array. For each number, we count its digits using integer division by 10 in a loop. Once the digit count is determined, we check if it is even, and if so, increment our counter. Finally, we return the total count of numbers with even digits.

## Go Solution

```go
func findNumbers(nums []int) int {
    evenDigitCount := 0
    for _, num := range nums {
        digitCount := 0
        current := num
        for current > 0 {
            current /= 10
            digitCount++
        }
        if digitCount % 2 == 0 {
            evenDigitCount++
        }
    }
    return evenDigitCount
}
```

The Go implementation mirrors the Python approach. We initialize a counter `evenDigitCount`, iterate through the slice `nums`, count digits using integer division, and increment the counter if the number of digits is even. Go requires explicit type management and uses slices rather than lists, but the logic is identical.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n * log10(max_num)) | Each number is divided by 10 repeatedly to count digits; max number of divisions per number is proportional to the number of digits, log10(max_num) |
| Space | O(1) | Only a few integer counters are used; no extra data structures |

The reasoning behind the time complexity is that for each number, the number of iterations to count digits is at most proportional to the number of digits, which is log10 of the number. Given n numbers, this yields O(n * log10(max_num)).

## Edge Cases

One important edge case is when the input array contains only single-digit numbers. These are all odd-digit numbers, so the algorithm should correctly return 0. Another edge case is the maximum allowed integer, 100,000, which has six digits. The algorithm must handle this without integer overflow or errors in counting. A third edge case is an array where all numbers have an even number of digits. The solution must accurately count each one to ensure the total matches the array length. By using integer division and a loop for counting digits, the implementation naturally handles all these cases correctly.
