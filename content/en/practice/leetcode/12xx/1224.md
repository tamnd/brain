## Solution

## Problem Understanding

This problem is asking us to find the longest prefix of an array `nums` such that by removing **exactly one element** from that prefix, the frequency of all remaining numbers becomes equal. In other words, after removing one number, every number left in the prefix should appear the same number of times.

The input is a list of positive integers. The output is an integer representing the length of the longest valid prefix. The problem guarantees that `nums` has at least two elements and can be up to 100,000 elements long, which rules out any brute-force solution that checks every prefix naively. All elements are between 1 and 100,000, so hashing by value or frequency is feasible.

Important edge cases include arrays where one number dominates, arrays where all numbers are unique, and arrays that are already valid without removal. It is also critical to handle cases where the removal of one element results in all remaining elements having the same frequency of 1, or removing the last element leaves the array empty.

## Approaches

### Brute Force

A naive brute-force approach would be to iterate through every possible prefix of the array. For each prefix, we could check every single element removal and verify if the remaining numbers have equal frequency. This is correct because it exhaustively checks all possibilities, but it is computationally infeasible for large arrays, with a worst-case time complexity of `O(n^2)` or `O(n^3)` depending on the frequency verification approach.

### Optimal Approach

The key insight is that we do not need to check every removal explicitly. Instead, we can track the frequency of each number dynamically while iterating through the array. We maintain two hash maps: one that counts how many times each number appears (`num_count`), and another that counts how many numbers have a given frequency (`freq_count`). At each step, we check whether the current prefix satisfies one of the conditions that allows the frequencies to be equal after removing one element:

1. All numbers appear once.
2. All numbers have the same frequency except one number, which is either one more than the others or occurs exactly once.

This approach reduces the problem to `O(n)` time complexity because each number is processed only once, and frequency checks are performed in constant time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) or O(n^3) | O(n) | Check each prefix and simulate removing each element |
| Optimal | O(n) | O(n) | Use hash maps to track counts and validate prefix in constant time per element |

## Algorithm Walkthrough

1. Initialize two hash maps: `num_count` for counting each number's occurrences and `freq_count` for counting how many numbers have a certain frequency. Initialize `max_len` to 0.
2. Iterate through `nums`, updating the count of the current number in `num_count`.
3. Adjust `freq_count` to reflect the new frequency of the current number. If a number's frequency increases, decrement the previous frequency count and increment the new frequency count.
4. At each step, determine the maximum frequency `max_freq` in the current prefix.
5. Check if the prefix satisfies any of these valid conditions:

- `max_freq` is 1 (all numbers occur once)
- One number has frequency `max_freq` and all others have `max_freq - 1`
- One number occurs once and all others have the same frequency
6. If any condition is satisfied, update `max_len` to the current prefix length.
7. After iterating through the array, return `max_len`.

**Why it works:** The algorithm works because it maintains the exact count of occurrences and the number of numbers with a specific count. By checking only the necessary frequency patterns, it ensures we only consider prefixes where removing one element can normalize all frequencies, guaranteeing correctness in linear time.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def maxEqualFreq(self, nums: List[int]) -> int:
        num_count = defaultdict(int)
        freq_count = defaultdict(int)
        max_len = 0
        max_freq = 0
        
        for i, num in enumerate(nums):
            if num_count[num] > 0:
                freq_count[num_count[num]] -= 1
            num_count[num] += 1
            freq_count[num_count[num]] += 1
            max_freq = max(max_freq, num_count[num])
            
            # Total number of elements processed
            prefix_len = i + 1
            
            # Condition 1: All numbers occur once
            cond1 = max_freq == 1
            
            # Condition 2: One number has max_freq, rest have max_freq-1
            cond2 = freq_count[max_freq] * max_freq + freq_count[max_freq - 1] * (max_freq - 1) == prefix_len and freq_count[max_freq] == 1
            
            # Condition 3: One number occurs once, rest have max_freq
            cond3 = freq_count[1] == 1 and freq_count[max_freq] * max_freq + 1 == prefix_len
            
            if cond1 or cond2 or cond3:
                max_len = prefix_len
        
        return max_len
```

**Explanation:** We track both `num_count` and `freq_count` to efficiently verify whether removing one element can equalize the frequencies. The three conditions cover all scenarios where a prefix can be made uniform after removing one element.

## Go Solution

```go
func maxEqualFreq(nums []int) int {
    numCount := make(map[int]int)
    freqCount := make(map[int]int)
    maxLen := 0
    maxFreq := 0

    for i, num := range nums {
        if numCount[num] > 0 {
            freqCount[numCount[num]]--
        }
        numCount[num]++
        freqCount[numCount[num]]++
        if numCount[num] > maxFreq {
            maxFreq = numCount[num]
        }

        prefixLen := i + 1

        cond1 := maxFreq == 1
        cond2 := freqCount[maxFreq]*maxFreq+freqCount[maxFreq-1]*(maxFreq-1) == prefixLen && freqCount[maxFreq] == 1
        cond3 := freqCount[1] == 1 && freqCount[maxFreq]*maxFreq+1 == prefixLen

        if cond1 || cond2 || cond3 {
            maxLen = prefixLen
        }
    }

    return maxLen
}
```

**Go notes:** We use maps to mimic Python's defaultdict. Care is taken to decrement the previous frequency count before incrementing the new frequency. The rest of the logic mirrors the Python version.
