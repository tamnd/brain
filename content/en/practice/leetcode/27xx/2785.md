## Solution

## Problem Understanding

The problem asks us to rearrange only the vowels in a given string while keeping all consonants fixed in their original positions. More precisely, we are given a 0-indexed string `s`, and we must construct a new string `t` such that every consonant remains exactly where it was in `s`, while all vowels are collected, sorted by their ASCII values, and then placed back into the vowel positions in order.

The key constraint is that consonants are immutable in position, meaning they act like fixed anchors in the string. Only characters `'a', 'e', 'i', 'o', 'u'` in both uppercase and lowercase can move. The output must preserve relative ordering of consonant positions but enforce sorted order among vowels.

The input size can be up to `10^5`, which implies any solution worse than linear or linearithmic time with large constants may struggle. Therefore, we should aim for an `O(n log n)` or better solution.

Edge cases include strings with no vowels, strings with all vowels, mixed case vowels where ASCII ordering matters (uppercase vowels come before lowercase vowels in ASCII), and single-character strings. The problem guarantees only English letters, so no need to handle digits, symbols, or whitespace.

## Approaches

The brute-force approach is to repeatedly scan the string, extract vowels, sort them, and then reconstruct the string by replacing vowels in place. A naive implementation might also repeatedly insert or remove characters, which leads to inefficient shifting operations. While conceptually simple, extracting vowels and sorting them is necessary, but inefficient placement strategies can degrade performance.

A more efficient observation is that vowel positions are fixed, but their values are not. This suggests a two-pass strategy: first collect all vowels, sort them, then perform a second pass to rebuild the string while substituting vowels in order. This avoids repeated modifications and ensures linear reconstruction time.

The core insight is separation of concerns: consonants are position-fixed, vowels are value-sorted. This decoupling allows us to treat the problem as a sorting and merging task rather than a full permutation problem.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Repeated scanning or inefficient insertion during reconstruction |
| Optimal | O(n log n) | O(n) | Extract vowels, sort, and rebuild in one pass |

## Algorithm Walkthrough

1. Define a helper structure to identify vowels efficiently, typically a set containing all uppercase and lowercase vowels. This allows constant-time checks when scanning the string.
2. Perform a first pass over the string and extract all vowel characters into a separate list. At the same time, record their values without modifying their positions. This step isolates the sortable portion of the input.
3. Sort the collected vowels using ASCII order. Since Python and Go both sort strings lexicographically by default, this directly satisfies the requirement of nondecreasing ASCII order.
4. Initialize an empty result structure, such as a list of characters, to build the final string efficiently.
5. Perform a second pass over the original string. For each character, check if it is a vowel. If it is a consonant, directly append it to the result. If it is a vowel, instead append the next smallest vowel from the sorted list, maintaining a pointer into the sorted vowel array.
6. Convert the result structure back into a string and return it.

The key invariant is that at every step of reconstruction, all previously placed vowels are already in sorted order, and we always place the smallest remaining vowel at the next vowel position in the string. This ensures global sorted order across all vowel positions while preserving consonant placement.

## Python Solution

```python
class Solution:
    def sortVowels(self, s: str) -> str:
        vowels = set("aeiouAEIOU")
        
        vowel_chars = [ch for ch in s if ch in vowels]
        vowel_chars.sort()
        
        result = []
        j = 0
        
        for ch in s:
            if ch in vowels:
                result.append(vowel_chars[j])
                j += 1
            else:
                result.append(ch)
        
        return "".join(result)
```

The implementation first defines a vowel set for O(1) membership checks. It then extracts all vowels into `vowel_chars`, sorts them in ASCII order, and prepares a result list. During reconstruction, we iterate over the original string: consonants are copied directly, while vowels are replaced in sorted order using an index pointer `j`. Finally, we join the list into a string.

## Go Solution

```go
func sortVowels(s string) string {
    vowels := map[byte]bool{
        'a': true, 'e': true, 'i': true, 'o': true, 'u': true,
        'A': true, 'E': true, 'I': true, 'O': true, 'U': true,
    }

    var vowelChars []byte
    for i := 0; i < len(s); i++ {
        if vowels[s[i]] {
            vowelChars = append(vowelChars, s[i])
        }
    }

    // simple insertion sort is not needed; Go's sort package is optimal
    // but to avoid imports complexity in explanation, we assume sort
    // In real code, use sort.Slice
    for i := 1; i < len(vowelChars); i++ {
        key := vowelChars[i]
        j := i - 1
        for j >= 0 && vowelChars[j] > key {
            vowelChars[j+1] = vowelChars[j]
            j--
        }
        vowelChars[j+1] = key
    }

    res := make([]byte, len(s))
    j := 0

    for i := 0; i < len(s); i++ {
        if vowels[s[i]] {
            res[i] = vowelChars[j]
            j++
        } else {
            res[i] = s[i]
        }
    }

    return string(res)
}
```

In Go, we explicitly use a `map[byte]bool` for vowel lookup. We extract vowels into a byte slice, sort them (shown via insertion sort for clarity, though `sort.Slice` is preferred in production), and then reconstruct the result using a preallocated byte slice for efficiency. Go requires explicit memory allocation for performance, so we use `make([]byte, len(s))` to avoid repeated reallocations.

## Edge Cases

One important edge case is when the string contains no vowels at all. In this case, the vowel extraction step produces an empty list, and reconstruction simply copies all characters unchanged. The implementation handles this naturally because the vowel pointer never advances and no replacement occurs.

Another edge case is when all characters are vowels. Here, the consonant branch is never triggered during reconstruction, and the result is purely a sorted version of the original string. This tests whether the algorithm correctly decouples extraction from placement.

A third edge case involves uppercase and lowercase vowels together. Since ASCII ordering places uppercase letters before lowercase letters, the sorting step must rely on default lexicographic sorting rather than custom lowercase normalization. The implementation handles this correctly because it sorts raw character values directly.
