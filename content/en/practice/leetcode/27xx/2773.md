# LeetCode 2773 - Height of Special Binary Tree

## Problem Understanding

The problem gives us the root of a binary tree with a very unusual property involving its leaf nodes. Normally, leaf nodes in a binary tree have no children. In this special tree, however, the leaves are connected to each other in a circular doubly linked structure through their `left` and `right` pointers.

Suppose the leaves in sorted order are:

$$b_1 < b_2 < \dots < b_k$$

Then:

- `b_i.right` points to `b_{i+1}` for all `i < k`
- `b_k.right` points back to `b_1`
- `b_i.left` points to `b_{i-1}` for all `i > 1`
- `b_1.left` points back to `b_k`

These additional connections mean the structure is no longer a pure tree if we blindly traverse all pointers. The leaf nodes form a cycle.

The task is to compute the height of the original binary tree. The height is defined as the number of edges in the longest path from the root to any node in the actual tree structure.

The important detail is that the circular leaf connections are not part of the original tree hierarchy. They are artificial links added afterward. We must ignore them when computing the height.

The input guarantees:

- Every node value is unique.
- The number of nodes is at most `10^4`.
- The structure is valid.
- The special leaf links always exist exactly as described.

The constraint size suggests we should aim for an `O(n)` traversal.

A naive traversal can easily enter an infinite loop because the leaves point to each other cyclically. Correct handling of leaf nodes is the key challenge.

Some important edge cases include:

- A tree with only one leaf, where the leaf may point to itself.
- A skewed tree with maximum depth.
- Trees where internal nodes have only one child.
- Trees where leaf connections create cycles that would trap a standard DFS.

The problem guarantees that the underlying original structure is still a valid binary tree, so if we avoid traversing leaf-to-leaf links, the answer is straightforward.

## Approaches

### Brute Force Approach

A brute-force solution would perform a graph traversal while maintaining a visited set to avoid infinite loops caused by the cyclic leaf connections.

We could treat every pointer as a graph edge and recursively explore all reachable nodes while tracking visited nodes. During traversal, we compute the maximum depth reached from the root.

This works because the visited set prevents revisiting nodes in the leaf cycle.

However, this approach is unnecessarily complicated for the actual structure of the problem. The special leaf links are not part of the original tree height definition, so traversing them at all is wasted work.

It also introduces extra memory overhead for visited tracking and makes reasoning about correctness harder.

### Optimal Approach

The key observation is that every true leaf in the original tree can be identified because both its children are also leaves participating in the circular structure.

In the original binary tree, a leaf should terminate traversal. Therefore, when we reach a node whose children are both non-null and mutually part of the leaf cycle, we should stop recursion there.

A much simpler insight exists:

- Internal tree nodes always connect downward normally.
- Leaf nodes are exactly the nodes whose children point to other leaves.
- We should never continue DFS from a leaf's artificial neighbors.

A node is a true leaf of the original tree if:

```
node.left is not None AND node.left.val > node.val
OR
node.right is not None AND node.right.val < node.val
```

But an even cleaner observation is this:

In the original tree, a node is considered a leaf if both children exist and violate normal tree structure ordering caused by cyclic connections.

Instead of trying to detect graph cycles globally, we simply detect whether the current node is a special leaf and terminate recursion there.

This allows a clean DFS in linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Uses graph traversal with visited set to avoid cycles |
| Optimal | O(n) | O(h) | DFS stops at special leaves, where `h` is tree height |

## Algorithm Walkthrough

### Step 1: Define a DFS Function

We create a recursive depth-first search function that computes the height of the subtree rooted at the current node.

The function returns the maximum depth reachable beneath that node.

### Step 2: Detect Special Leaves

In a normal binary tree, a leaf has no children.

In this special tree, leaf nodes instead point to neighboring leaves. Therefore, we must identify when a node should logically be treated as a leaf.

A node is a special leaf if:

- Its left child exists and has a greater value than itself, or
- Its right child exists and has a smaller value than itself

These relationships occur because the leaf connections form cyclic neighbor links rather than parent-child tree relationships.

When this happens, we stop recursion immediately.

### Step 3: Recurse on Valid Tree Children

If the current node is not a special leaf:

- Recursively compute the left subtree height.
- Recursively compute the right subtree height.

We ignore null children by returning `0` for missing paths.

### Step 4: Combine Results

The height rooted at the current node is:

$$1 + \max(\text{leftHeight}, \text{rightHeight})$$

because moving from the current node to a child adds one edge.

### Step 5: Adjust Final Answer

The recursive function counts nodes in the path.

The problem defines height as the number of edges.

Therefore, we subtract `1` from the final DFS result.

## Why it works

The algorithm works because the only dangerous edges are the artificial leaf-cycle connections. By detecting when a node is actually a leaf in the original tree, we terminate traversal before entering the cycle.

Every legitimate tree edge is traversed exactly once, and every artificial cycle edge is ignored. Therefore, the recursion explores exactly the original binary tree structure, producing the correct height.

## Python Solution

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def heightOfTree(self, root: Optional[TreeNode]) -> int:
        def dfs(node: Optional[TreeNode]) -> int:
            if not node:
                return 0

            # Detect special leaf nodes
            if (
                (node.left and node.left.val > node.val) or
                (node.right and node.right.val < node.val)
            ):
                return 1

            left_height = dfs(node.left)
            right_height = dfs(node.right)

            return 1 + max(left_height, right_height)

        return dfs(root) - 1
```

The implementation starts with a recursive DFS helper function.

The base case handles null nodes by returning `0`.

Next, we detect whether the current node is a special leaf. If the node participates in the artificial cyclic structure, we stop recursion immediately and treat it as a terminal node in the original tree.

Otherwise, we recursively compute heights for the left and right subtrees.

The recursive function returns the number of nodes in the longest path. Since the problem defines height using edges, the final result subtracts one.

This implementation visits each node once and never enters the cyclic leaf structure.

## Go Solution

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

func heightOfTree(root *TreeNode) int {
    var dfs func(node *TreeNode) int

    dfs = func(node *TreeNode) int {
        if node == nil {
            return 0
        }

        // Detect special leaf nodes
        if (node.Left != nil && node.Left.Val > node.Val) ||
            (node.Right != nil && node.Right.Val < node.Val) {
            return 1
        }

        leftHeight := dfs(node.Left)
        rightHeight := dfs(node.Right)

        if leftHeight > rightHeight {
            return 1 + leftHeight
        }

        return 1 + rightHeight
    }

    return dfs(root) - 1
}
```

The Go implementation follows the same recursive structure as the Python version.

Go requires explicit nil checks before accessing child fields.

Instead of Python's built-in `max`, we manually compare subtree heights.

The recursion depth is bounded by the tree height, which is acceptable for the problem constraints.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited once |
| Space | O(h) | Recursive call stack stores at most tree height frames |

The DFS traverses each valid tree node exactly once. No repeated computation occurs.

The space complexity comes from recursion depth. In the worst case of a completely skewed tree, the recursion stack may contain `n` frames. In a balanced tree, the depth is `O(log n)`.

## Edge Cases

### Single Leaf Node

A tree may contain only one leaf node. In this case, the leaf's left and right pointers may point back to itself.

A naive DFS would recurse infinitely because the leaf references itself cyclically. The implementation correctly identifies the node as a special leaf and terminates recursion immediately.

### Completely Skewed Tree

A tree may degenerate into a linked list shape where every node has only one child.

This is important because recursion depth becomes `O(n)` in the worst case. The implementation still works correctly because each node is processed once and recursion follows the single valid path.

### Artificial Cyclic Connections

The leaf nodes form a circular structure. A standard DFS that blindly traverses all children would enter an infinite loop.

The implementation avoids this entirely by detecting special leaves before traversing their children. This guarantees that the DFS remains within the original binary tree structure only.
