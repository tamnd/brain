## Solution

## Problem Understanding

The problem is asking us to implement a cancellable interval function, similar to `setInterval` in JavaScript, but with the ability to stop it at a specified time. Specifically, you are given a function `fn`, an array of arguments `args`, and an interval `t` in milliseconds. You must call `fn` immediately with `args`, then continue to call it every `t` milliseconds until a cancel function `cancelFn` is invoked at a specified `cancelTimeMs`.

The input consists of a function and its arguments, which can vary in length between 1 and 10. The interval `t` is always between 30 and 100 milliseconds, and the cancel time is between 10 and 500 milliseconds. The expected output is a function (`cancelFn`) that, when called, stops the repeated calls.

Edge cases include very small intervals or cancel times that may mean `fn` is called only once, or very large `cancelTimeMs` relative to `t`, which would result in multiple calls. The problem guarantees that `args` is a valid JSON array, so there is no need to handle invalid input types.

## Approaches

The brute-force approach would be to manually track time and repeatedly call `fn` in a loop with a `sleep` or `setTimeout` mechanism. While correct, this is unnecessary since JavaScript/Go provides built-in interval timers. This naive approach may also be less precise due to cumulative timer drift.

The optimal approach is to immediately call `fn` once, then use a repeating timer (like `setInterval` in JS or a goroutine with `time.Ticker` in Go) to call `fn` every `t` milliseconds. When the returned cancel function is invoked, it stops the interval. The key insight is that the timer abstraction handles repeated execution efficiently and precisely, avoiding manual looping and sleep logic.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(cancelTimeMs / t) | O(1) | Loop with sleep calling `fn` manually |
| Optimal | O(cancelTimeMs / t) | O(1) | Use interval timer, immediate call, cancel stops it |

## Algorithm Walkthrough

1. Define a function `cancellable` that takes `fn`, `args`, and `t` as parameters.
2. Call `fn` immediately with `args` to ensure the first call occurs at time 0.
3. Set up an interval timer that calls `fn` every `t` milliseconds.
4. Return a function `cancelFn` that, when invoked, stops the interval timer. In JavaScript, this is done with `clearInterval`; in Go, with `ticker.Stop()`.
5. Ensure that no calls occur after `cancelFn` is invoked, respecting the `cancelTimeMs`.

**Why it works:** The invariant is that the interval timer triggers `fn` exactly every `t` milliseconds after the first call. By cancelling the timer at `cancelTimeMs`, we guarantee no further calls occur, preserving correctness.

## Python Solution

```python
from typing import Callable, List
import threading
import time

def cancellable(fn: Callable, args: List, t: int) -> Callable:
    # Immediately call the function
    fn(*args)
    
    # Create a stop event
    stop_event = threading.Event()
    
    # Define the interval loop
    def run():
        while not stop_event.wait(t / 1000):
            fn(*args)
    
    # Start the thread
    thread = threading.Thread(target=run)
    thread.daemon = True
    thread.start()
    
    # Define cancel function
    def cancelFn():
        stop_event.set()
    
    return cancelFn
```

The Python implementation uses a background thread with a `threading.Event` to manage cancellation. The interval loop waits for `t` milliseconds between calls, calling `fn` repeatedly until `stop_event` is set.

## Go Solution

```go
package main

import (
	"time"
)

func Cancellable(fn func(...interface{}), args []interface{}, t int) func() {
	// Call immediately
	fn(args...)

	ticker := time.NewTicker(time.Duration(t) * time.Millisecond)
	done := make(chan bool)

	// Goroutine to run interval
	go func() {
		for {
			select {
			case <-ticker.C:
				fn(args...)
			case <-done:
				ticker.Stop()
				return
			}
		}
	}()

	// Cancel function
	return func() {
		done <- true
	}
}
```

In Go, a `time.Ticker` manages repeated calls. A `done` channel is used to signal cancellation. Unlike Python, Go uses channels and goroutines instead of threads and events, but the logic remains equivalent.

## Edge Cases

One edge case is when `cancelTimeMs` is less than `t`. In this case, `fn` should only be called once immediately, and no interval calls occur. The implementation handles this because the interval timer is cancelled after the first call if the cancel function is invoked early.

A second edge case is a very small `t` relative to `cancelTimeMs`. This results in a large number of calls. Both Python threads and Go goroutines efficiently handle this scenario without accumulating memory because each call is independent.

A third edge case is when `args` contains the maximum allowed number of arguments (10). Our implementation correctly unpacks and passes all arguments to `fn` in both Python and Go, ensuring it works for any valid argument length.
