## Solution

## Problem Understanding

The problem gives us two arrays, `keysArr` and `valuesArr`, both of the same length. Each element in `keysArr` represents a key, and the element at the same index in `valuesArr` represents the corresponding value. Our goal is to construct a new object where every key-value pair comes from matching indices in these two arrays.

For example, if `keysArr[i] = "x"` and `valuesArr[i] = 10`, then the resulting object should contain:

```
"x": 10
```

There is one important rule regarding duplicate keys. Before insertion, every key must first be converted into a string using JavaScript's `String()` conversion. If two different elements become the same string after conversion, only the first occurrence should be kept. Any later duplicate occurrences must be ignored.

This detail matters because values such as `"1"` and `1` are different as raw values, but both become `"1"` after string conversion. Similarly, `false` becomes `"false"`.

The input arrays are guaranteed to be valid JSON arrays, and their lengths are equal. The constraints allow serialized input sizes up to `5 * 10^5`, which means the solution must be efficient. A quadratic solution could become too slow for large inputs, so we should aim for a linear-time approach.

Several edge cases are important:

- The arrays may be empty, producing an empty object.
- Keys may already be strings or may need conversion from numbers, booleans, or other JSON-compatible types.
- Multiple keys may become identical after string conversion.
- Duplicate keys should preserve the value from the first occurrence only.

The problem is fundamentally about controlled insertion into a hash map while respecting first-occurrence precedence.

## Approaches

### Brute Force Approach

A straightforward approach is to process the arrays from left to right and, for each key, check whether that key has already appeared earlier in the array.

The algorithm works as follows:

1. Convert the current key to a string.
2. Scan all previous elements to see whether the same converted key already exists.
3. If the key has not appeared before, insert it into the result object.
4. Otherwise, skip it.

This approach is correct because every insertion is validated against all earlier keys, ensuring only the first occurrence is kept.

However, the problem is performance. For every element, we may scan all previous elements. If there are `n` elements, this leads to:

```
1 + 2 + 3 + ... + (n - 1)
```

operations in the worst case, which is `O(n^2)` time complexity.

Given the large constraint size, a quadratic algorithm is not efficient enough.

### Optimal Approach

The key observation is that we only need a fast way to determine whether a key has already been inserted.

A hash map, or dictionary, provides constant average-time lookup and insertion. Since JavaScript objects naturally behave like hash maps, we can use the result object itself to track which keys have already been processed.

The optimized algorithm works like this:

1. Iterate through both arrays once.
2. Convert the current key to a string.
3. Check whether the key already exists in the result object.
4. If it does not exist, insert the key-value pair.
5. Otherwise, skip it.

This avoids repeated scanning and reduces the total time complexity to linear time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly scans previous keys for duplicates |
| Optimal | O(n) | O(n) | Uses hash map lookups to detect duplicates efficiently |

## Algorithm Walkthrough

1. Initialize an empty object called `result`.

This object will store the final key-value pairs. It also acts as a fast lookup structure for checking whether a key has already appeared.
2. Iterate through the arrays from index `0` to `n - 1`.

Since both arrays have equal length, each index represents one key-value pair.
3. Convert the current key into a string.

The problem explicitly requires all keys to be processed through `String()`. This ensures values like `1` and `"1"` become identical keys.
4. Check whether the converted key already exists in the result object.

If the key is already present, it means an earlier occurrence has already been inserted, so the current pair must be ignored.
5. Insert the key-value pair if the key is new.

Store:

```
result[stringKey] = valuesArr[i]
```
6. Continue processing until all elements are handled.
7. Return the completed object.

### Why it works

The algorithm processes keys from left to right, so the first occurrence of each converted key is encountered before any duplicates. Because insertion only happens when the key is absent, the first value is preserved permanently and later duplicates are ignored. The hash map guarantees efficient lookup for every key, ensuring correctness and optimal performance.

## Python Solution

```python
from typing import List, Any, Dict

class Solution:
    def createObject(self, keysArr: List[Any], valuesArr: List[Any]) -> Dict[str, Any]:
        result = {}

        for index in range(len(keysArr)):
            key = str(keysArr[index])

            if key not in result:
                result[key] = valuesArr[index]

        return result
```

The implementation begins by creating an empty dictionary called `result`. This dictionary stores the final object that will be returned.

The loop iterates through every index in the arrays. At each iteration, the current key is converted into a string using Python's `str()` function, which mirrors JavaScript's `String()` behavior for the purposes of this problem.

The condition:

```python
if key not in result:
```

ensures that only the first occurrence of a key is inserted. If the key already exists, the algorithm skips the insertion entirely.

Finally, the completed dictionary is returned.

## Go Solution

```go
package main

import "fmt"

func createObject(keysArr []interface{}, valuesArr []interface{}) map[string]interface{} {
	result := make(map[string]interface{})

	for i := 0; i < len(keysArr); i++ {
		key := fmt.Sprintf("%v", keysArr[i])

		if _, exists := result[key]; !exists {
			result[key] = valuesArr[i]
		}
	}

	return result
}
```

The Go implementation follows the same logic as the Python solution. Since Go does not have automatic string conversion equivalent to JavaScript's `String()`, `fmt.Sprintf("%v", value)` is used to generate a string representation of each key.

The result object is represented using:

```go
map[string]interface{}
```

This allows storage of arbitrary JSON-compatible values.

Go maps naturally provide constant average-time lookup and insertion, making them ideal for this problem.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element is processed exactly once |
| Space | O(n) | The result object may store up to n unique keys |

The algorithm performs a single pass through the arrays. Each lookup and insertion into the hash map takes constant average time, so the total runtime is linear.

The extra space comes from the output object itself. In the worst case, all keys are unique, requiring storage for all `n` key-value pairs.

## Edge Cases

One important edge case occurs when different raw values become identical after string conversion. For example, `"1"` and `1` are different values initially, but both convert to `"1"`. A naive implementation that compares raw values instead of converted strings would incorrectly insert duplicates. The implementation avoids this by converting every key before checking existence.

Another important case is empty input arrays. If both arrays are empty, the algorithm should return an empty object immediately. Some implementations accidentally assume at least one element exists and may fail with index-related errors. Since the loop simply performs zero iterations, the implementation naturally handles this safely.

A third edge case involves repeated duplicate keys appearing many times. For example:

```
keysArr = ["a", "a", "a", "a"]
```

A buggy solution might overwrite earlier values repeatedly, leaving the last value instead of the first. The implementation explicitly checks whether a key already exists before insertion, guaranteeing that the first value is preserved permanently.

Another subtle edge case is mixed-type input containing booleans, numbers, and null-like values. Since all keys are normalized into strings, the implementation consistently handles all supported JSON-compatible key types without special-case logic.
