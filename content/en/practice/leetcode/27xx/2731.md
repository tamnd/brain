## Solution

## Problem Understanding

The problem is asking us to calculate the sum of distances between all pairs of robots after a given number of seconds `d`. Each robot starts at a unique position on a number line as given in the array `nums`, and each robot has an initial movement direction specified in the string `s` where `'L'` means left (toward negative coordinates) and `'R'` means right (toward positive coordinates). Robots move one unit per second and if two robots collide, they instantaneously reverse directions.

The input `nums` represents the starting positions of the robots, and `s` provides the initial direction of movement for each robot. The output is a single integer representing the sum of distances between all distinct robot pairs after `d` seconds, modulo $10^9 + 7$.

The constraints tell us that there can be up to $10^5$ robots and that both positions and time can be very large, which implies that a naive simulation of each second would be too slow. Additionally, the robots' positions are guaranteed to be unique initially, which simplifies collision handling to pairwise swaps rather than more complex multi-robot overlaps.

Important edge cases include scenarios where all robots move in the same direction (no collisions occur), `d` is zero (sum of initial distances), and robots starting at negative coordinates.

## Approaches

The brute-force approach is to simulate the movement of each robot for each second up to `d` and handle collisions explicitly. After `d` seconds, compute the distance between every pair. This approach works correctly but is infeasible because the maximum `d` can reach $10^9$, leading to unacceptable runtime.

The optimal approach leverages a key observation from physics: in one-dimensional elastic collisions of identical objects, the final positions after a certain amount of time can be computed as if the collisions never happened. This means we can simulate each robot moving `d` units in their initial direction without handling collisions explicitly, because collisions merely swap robots' identities. Sorting these final positions allows us to efficiently compute the sum of pairwise distances using a prefix sum approach. The main insight is that after sorting, the sum of distances formula can be applied in linear time using cumulative sums.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force Simulation | O(n * d) | O(n) | Simulate each second and collisions explicitly |
| Optimal (Prefix Sum + Final Positions) | O(n log n) | O(n) | Compute positions ignoring collisions, sort, and sum distances efficiently |

## Algorithm Walkthrough

1. Initialize an array `final_positions` to store the positions of robots after moving `d` seconds. Compute each robot's final position as `nums[i] + d` if moving right, or `nums[i] - d` if moving left.
2. Sort the `final_positions` array in ascending order. Sorting allows us to efficiently compute pairwise distances.
3. Initialize variables `prefix_sum` and `total_distance` to zero. `prefix_sum` will store the cumulative sum of positions as we iterate through `final_positions`.
4. Iterate through the sorted `final_positions`. For each robot at index `i` with position `pos`, compute the contribution to `total_distance` as `(pos * i - prefix_sum)` and add it to `total_distance`. This formula works because the sum of distances between the current robot and all previous robots is `i * pos - sum_of_previous_positions`.
5. Update `prefix_sum` by adding `pos` for the next iteration.
6. Return `total_distance` modulo $10^9 + 7$.

Why it works: Sorting ensures that we always compute distances in one direction, avoiding double-counting and eliminating the need for explicit collision simulation. Using prefix sums allows us to compute the sum of distances to all previous robots in constant time per robot.

## Python Solution

```python
from typing import List

class Solution:
    def sumDistance(self, nums: List[int], s: str, d: int) -> int:
        MOD = 10**9 + 7
        n = len(nums)
        final_positions = [nums[i] + d if s[i] == 'R' else nums[i] - d for i in range(n)]
        final_positions.sort()
        
        total_distance = 0
        prefix_sum = 0
        
        for i, pos in enumerate(final_positions):
            total_distance = (total_distance + pos * i - prefix_sum) % MOD
            prefix_sum = (prefix_sum + pos) % MOD
        
        return total_distance
```

The code first calculates the final positions based on initial directions and the number of seconds `d`. It then sorts the positions and uses a prefix sum strategy to calculate the sum of pairwise distances efficiently. Modular arithmetic ensures that we stay within the integer limits.

## Go Solution

```go
import "sort"

func sumDistance(nums []int, s string, d int) int {
    const MOD int = 1e9 + 7
    n := len(nums)
    finalPositions := make([]int, n)
    
    for i := 0; i < n; i++ {
        if s[i] == 'R' {
            finalPositions[i] = nums[i] + d
        } else {
            finalPositions[i] = nums[i] - d
        }
    }
    
    sort.Ints(finalPositions)
    
    totalDistance := 0
    prefixSum := 0
    
    for i := 0; i < n; i++ {
        totalDistance = (totalDistance + finalPositions[i]*i - prefixSum) % MOD
        prefixSum = (prefixSum + finalPositions[i]) % MOD
    }
    
    return (totalDistance + MOD) % MOD
}
```

Go-specific notes: Go does not handle negative modulo in the same way as Python, so we add `MOD` and take modulo again to ensure a non-negative result. Arrays and slices are used instead of lists, and `sort.Ints` is used to sort the positions.
