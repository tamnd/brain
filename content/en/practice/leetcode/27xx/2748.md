## Solution

## Problem Understanding

The problem gives us an integer array `nums`, and we must count how many index pairs `(i, j)` satisfy two conditions:

1. `i < j`
2. The first digit of `nums[i]` and the last digit of `nums[j]` are coprime

Two numbers are coprime if their greatest common divisor, `gcd(a, b)`, equals `1`.

For every valid pair of indices, we extract:

- The first digit from the left side of `nums[i]`
- The last digit from the right side of `nums[j]`

Then we check whether those two digits are coprime.

For example, if:

```
nums[i] = 532
```

its first digit is `5`.

If:

```
nums[j] = 147
```

its last digit is `7`.

Since `gcd(5, 7) = 1`, this pair is beautiful.

The array length is at most `100`, which is very small. Each number is at most `9999`, so extracting digits is extremely cheap. The constraint:

```
nums[i] % 10 != 0
```

guarantees that the last digit is never `0`, which simplifies gcd checks slightly because we only deal with digits `1` through `9`.

An important observation is that we are not comparing the whole numbers. Only two single digits matter:

- the leading digit
- the trailing digit

Potential edge cases include:

- Numbers with only one digit, where the first and last digit are the same
- Arrays where no pairs are coprime
- Arrays where every pair is coprime
- Repeated values
- Digits like `1`, since `gcd(1, x) = 1` for every `x`

Because the constraints are small, even a quadratic solution is acceptable.

## Approaches

### Brute Force Approach

The most direct solution is to examine every pair `(i, j)` where `i < j`.

For each pair:

1. Extract the first digit of `nums[i]`
2. Extract the last digit of `nums[j]`
3. Compute their gcd
4. If the gcd equals `1`, increment the answer

This works because the definition of a beautiful pair directly maps to checking every possible pair.

To extract the first digit, we repeatedly divide by `10` until the number becomes a single digit.

To extract the last digit, we simply use:

```
num % 10
```

The total number of pairs is:

```
n * (n - 1) / 2
```

With `n <= 100`, this is at most `4950` pairs, which is perfectly manageable.

### Optimized Observation

Although the brute-force solution is already fast enough, we can slightly optimize the implementation by observing that only digits `1` through `9` matter.

Instead of recomputing the first digit repeatedly for earlier elements, we can process the array from left to right while maintaining counts of previously seen leading digits.

For each current number:

1. Compute its last digit
2. Check all previously seen first digits
3. If a stored first digit is coprime with the current last digit, add its frequency to the answer
4. Insert the current number's first digit into the frequency table

Since there are only `9` possible digits, the inner loop is constant sized.

This reduces repeated work and gives a cleaner counting-based solution.

## Algorithm Walkthrough

We will describe the optimized counting solution.

1. Create a frequency array or hash map to store how many times each leading digit has appeared so far.

This works because when processing index `j`, only indices before it can form valid pairs.
2. Initialize `answer = 0`.

This variable stores the total number of beautiful pairs.
3. Iterate through each number in `nums` from left to right.

At every step, the current number acts as the second element of the pair.
4. Compute the last digit of the current number.

This is simply:

```
last_digit = num % 10
```
5. Compare the current last digit with every previously seen first digit.

Since first digits can only be from `1` to `9`, we loop through them and compute:

```
gcd(first_digit, last_digit)
```

If the gcd equals `1`, then every previously seen occurrence of that first digit forms a valid beautiful pair.
6. Add the matching frequencies to the answer.

For example, if leading digit `3` has appeared `4` times before, and `gcd(3, last_digit) == 1`, then we add `4`.
7. Extract the first digit of the current number.

Repeatedly divide by `10` until the number becomes a single digit.
8. Store this first digit in the frequency structure.

This allows future numbers to pair with it.
9. Continue until all numbers are processed.
10. Return the final answer.

### Why it works

When processing index `j`, the frequency table contains exactly the first digits of all numbers at indices `< j`. Every valid earlier index is therefore represented.

For each possible first digit, we count how many earlier numbers have that digit. If it is coprime with the current last digit, every one of those earlier numbers forms a beautiful pair with the current number.

Since every pair is counted exactly once, the algorithm is correct.

## Python Solution

```python
from typing import List
from math import gcd

class Solution:
    def countBeautifulPairs(self, nums: List[int]) -> int:
        def first_digit(num: int) -> int:
            while num >= 10:
                num //= 10
            return num

        first_digit_count = [0] * 10
        beautiful_pairs = 0

        for num in nums:
            last_digit = num % 10

            for digit in range(1, 10):
                if gcd(digit, last_digit) == 1:
                    beautiful_pairs += first_digit_count[digit]

            leading = first_digit(num)
            first_digit_count[leading] += 1

        return beautiful_pairs
```

The solution starts by defining a helper function `first_digit()` that repeatedly removes the last digit until only the leading digit remains.

The array `first_digit_count` stores how many times each leading digit has appeared so far. Index `d` represents digit `d`.

As we iterate through `nums`, the current number acts as the right side of the pair. We extract its last digit using modulo `% 10`.

We then check every possible leading digit from `1` to `9`. If the gcd between that digit and the current last digit equals `1`, then all previously seen numbers with that leading digit form valid beautiful pairs.

After counting those pairs, we extract the current number's leading digit and add it to the frequency table so future elements can use it.

The algorithm processes each number exactly once and performs only constant-sized work per iteration.

## Go Solution

```go
package main

func gcd(a, b int) int {
	for b != 0 {
		a, b = b, a%b
	}
	return a
}

func firstDigit(num int) int {
	for num >= 10 {
		num /= 10
	}
	return num
}

func countBeautifulPairs(nums []int) int {
	firstDigitCount := make([]int, 10)
	beautifulPairs := 0

	for _, num := range nums {
		lastDigit := num % 10

		for digit := 1; digit <= 9; digit++ {
			if gcd(digit, lastDigit) == 1 {
				beautifulPairs += firstDigitCount[digit]
			}
		}

		leading := firstDigit(num)
		firstDigitCount[leading]++
	}

	return beautifulPairs
}
```

The Go implementation follows the same logic as the Python solution.

Since Go does not provide a built-in gcd function for integers in older standard usage, we implement Euclid's algorithm manually.

The frequency table is implemented as a slice of length `10`.

Go integers are more than sufficient because the maximum number of pairs is only `4950`, so overflow is impossible.

## Edge Cases

One important edge case is arrays containing the digit `1` as a leading digit. Since `gcd(1, x) = 1` for every positive integer `x`, these numbers form beautiful pairs with every possible trailing digit. A buggy implementation might incorrectly special case `1` or mishandle gcd computation. Our implementation correctly uses the standard gcd function, so these cases naturally work.

Another important edge case is single-digit numbers. For example, if `nums[i] = 7`, then both the first digit and last digit are `7`. Some implementations mistakenly assume multi-digit extraction logic is always needed. Our helper function repeatedly divides only while the number is at least `10`, so single-digit numbers are returned unchanged.

A third edge case involves repeated non-coprime digits, such as `[8, 8, 8, 8]`. Every pair has gcd `8`, so the answer should be `0`. Incorrect implementations may accidentally count all repeated values or forget the gcd condition. Our algorithm explicitly checks `gcd(digit, last_digit) == 1` before adding counts, preventing invalid pairs from being counted.

Another subtle case is when the leading and trailing digits are equal but still coprime, such as digit `1`. For example, `(1,1)` is coprime because `gcd(1,1)=1`. Some programmers incorrectly assume equal digits cannot be coprime. Using the mathematical gcd definition ensures correctness.
