# LeetCode 2799 - Count Complete Subarrays in an Array

## Problem Understanding

This problem asks us to count how many contiguous subarrays are **complete**.

A subarray is considered complete when the number of distinct values inside that subarray is exactly the same as the number of distinct values in the entire array.

For example, consider:

```
nums = [1,3,1,2,2]
```

The distinct values in the entire array are:

```
{1, 2, 3}
```

There are 3 distinct values overall.

A subarray is complete only if it also contains all three distinct values. For instance:

```
[1,3,1,2]
```

contains `{1,2,3}`, so it is complete.

However:

```
[1,3]
```

contains only `{1,3}`, so it is not complete.

The input consists of an array of positive integers. We need to examine all possible contiguous subarrays and count how many satisfy the completeness condition.

The constraints are:

```
1 <= nums.length <= 1000
1 <= nums[i] <= 2000
```

The array length is relatively small. An O(n²) solution is acceptable because:

```
1000² = 1,000,000
```

which is manageable.

Important edge cases include arrays where every element is identical, arrays where every element is unique, and arrays where the complete condition is only satisfied near the end of a window. These cases often expose mistakes in distinct-element counting logic.

## Approaches

### Brute Force

A straightforward approach is to generate every possible subarray.

First, compute the total number of distinct values in the entire array. Then, for every starting index, expand the subarray one element at a time while maintaining a set of distinct values inside the current subarray.

Whenever the size of the set becomes equal to the total distinct count of the whole array, the current subarray is complete.

This approach is correct because every possible subarray is examined exactly once.

The time complexity is O(n²), since there are O(n²) subarrays and we can maintain the distinct elements incrementally while extending each subarray.

### Key Insight

Instead of checking every subarray individually, we can count complete subarrays using a sliding window.

Suppose the whole array contains `k` distinct values.

For each right boundary, we maintain a window containing all `k` distinct values. Once the window becomes complete, every larger window ending at the same or a later position will also remain complete until a required value is removed.

This monotonic property makes sliding window applicable.

The key observation is:

- Find the smallest left boundary that still keeps all `k` distinct values.
- If a window `[left, right]` is complete, then every subarray starting at any index from `0` through `left` and ending at `right` is also complete.
- Therefore we can count many subarrays at once instead of checking them individually.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Enumerates all subarrays and tracks distinct values |
| Optimal | O(n) | O(n) | Sliding window counts multiple complete subarrays simultaneously |

## Algorithm Walkthrough

### Optimal Sliding Window

1. Compute the number of distinct values in the entire array.

Let:

```
targetDistinct = len(set(nums))
```

Every complete subarray must contain exactly this many distinct values.
2. Initialize a sliding window.

Use two pointers:

```
left = 0
right = 0
```

Also maintain a frequency map storing how many times each value appears inside the current window.
3. Expand the right boundary.

For each position `right`, add `nums[right]` to the frequency map.
4. Check whether the current window is complete.

A window is complete when:

```
len(frequencyMap) == targetDistinct
```

because the map keys represent the distinct values currently present.
5. Shrink the window while it remains complete.

While the window still contains all required distinct values:

- Count the current window as valid.
- Remove `nums[left]` from the window.
- Move `left` forward.
- If a frequency becomes zero, remove that key from the map.
6. Count valid subarrays.

Every time the window is complete, all starting positions before the current `left` are also valid.

Therefore:

```
answer += left
```

after shrinking.
7. Continue until every element has been processed.
8. Return the accumulated answer.

### Why it works

The sliding window always maintains a contiguous range. Whenever the window contains all distinct values from the original array, moving the left pointer forward identifies the smallest valid window ending at the current right position.

Once that smallest valid window is known, every earlier starting position also produces a complete subarray ending at the same right boundary. Therefore counting `left` valid starting positions correctly counts all complete subarrays exactly once.

## Python Solution

```python
from collections import defaultdict
from typing import List

class Solution:
    def countCompleteSubarrays(self, nums: List[int]) -> int:
        target_distinct = len(set(nums))

        freq = defaultdict(int)
        left = 0
        answer = 0

        for right in range(len(nums)):
            freq[nums[right]] += 1

            while len(freq) == target_distinct:
                freq[nums[left]] -= 1

                if freq[nums[left]] == 0:
                    del freq[nums[left]]

                left += 1

            answer += left

        return answer
```

The implementation begins by determining how many distinct values exist in the entire array. This value never changes and serves as the target condition for completeness.

A frequency map tracks the contents of the current sliding window. As the right pointer expands, elements are added to the map.

Whenever the number of distinct values in the window matches the target, the window is complete. At that point we repeatedly shrink from the left until the window is no longer complete. This process finds the smallest invalid position immediately after all complete windows ending at the current right index.

The variable `left` then represents how many starting positions produce complete subarrays ending at the current right boundary. Adding `left` to the answer counts all of them in constant time.

## Go Solution

```go
func countCompleteSubarrays(nums []int) int {
	targetDistinct := 0
	distinctSet := make(map[int]struct{})

	for _, num := range nums {
		distinctSet[num] = struct{}{}
	}

	targetDistinct = len(distinctSet)

	freq := make(map[int]int)
	left := 0
	answer := 0

	for right := 0; right < len(nums); right++ {
		freq[nums[right]]++

		for len(freq) == targetDistinct {
			freq[nums[left]]--

			if freq[nums[left]] == 0 {
				delete(freq, nums[left])
			}

			left++
		}

		answer += left
	}

	return answer
}
```

The Go implementation follows exactly the same sliding window logic as the Python solution. A `map[int]int` stores frequencies, and keys are deleted when their count reaches zero so that `len(freq)` accurately reflects the number of distinct values currently in the window.

Since the maximum number of subarrays for `n = 1000` is only 500,500, a standard Go `int` is more than sufficient and overflow is not a concern.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element enters and leaves the sliding window at most once |
| Space | O(n) | Frequency map may store all distinct values |

The sliding window guarantees that both pointers move only forward. The right pointer traverses the array once, and the left pointer also advances at most `n` times across the entire execution. Therefore the total work is linear. The auxiliary space is proportional to the number of distinct values stored in the frequency map.

## Edge Cases

### All Elements Are Identical

Consider:

```
[5,5,5,5]
```

The entire array contains only one distinct value. Therefore every non-empty subarray is complete. A common bug is forgetting that windows of length one are already valid. The sliding window handles this naturally because the target distinct count is one.

### Every Element Is Unique

Consider:

```
[1,2,3,4]
```

The array contains four distinct values, so a complete subarray must contain all four values. The only valid subarray is the entire array. The algorithm correctly identifies this because the frequency map reaches the target size only when the right pointer reaches the last element.

### Duplicates Near Window Boundaries

Consider:

```
[1,2,1,2]
```

When shrinking the window, removing one occurrence of a value does not necessarily remove that distinct value from the window. A common mistake is decrementing a count without deleting the key when the count reaches zero. The implementation explicitly removes keys whose frequency becomes zero, ensuring that the number of distinct values is always tracked correctly.

### Single Element Array

Consider:

```
[42]
```

The only subarray is `[42]`, which contains the same distinct values as the entire array. The algorithm immediately reaches the target distinct count, counts the valid subarray, and returns `1`.

### Large Number of Repeated Values

Consider:

```
[7,7,7,7,7,...]
```

A naive solution may still enumerate all subarrays. The sliding window efficiently counts them in linear time because each pointer moves only forward, regardless of how many complete subarrays exist.
