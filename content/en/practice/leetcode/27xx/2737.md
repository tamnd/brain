## Solution

## Problem Understanding

This problem asks us to compute the shortest distance from a given source node `s` to any node in a set of `marked` nodes in a directed, weighted graph. The graph is represented as a list of edges, where each edge contains the start node, end node, and the weight of the edge. The nodes are 0-indexed and the weights are positive integers.

The input consists of `n`, the number of nodes, `edges`, a 2D array defining the directed weighted edges, `s`, the starting node, and `marked`, a list of target nodes. The output should be the smallest distance from `s` to any node in `marked`. If no path exists to any marked node, the output should be `-1`.

Constraints indicate that the graph is relatively small (`n <= 500`), but the number of edges can be large (`edges.length <= 10^4`). Edge weights can be as high as `10^6`, and there may be multiple edges between the same pair of nodes but no self-loops. This suggests that a standard shortest-path algorithm such as Dijkstra's algorithm is appropriate, as Bellman-Ford may be unnecessarily slow given the positive edge weights.

Important edge cases include when `s` has no outgoing edges, when the marked nodes are unreachable, and when multiple paths exist with different weights.

## Approaches

The brute-force approach would be to perform a Depth-First Search (DFS) or Breadth-First Search (BFS) from `s` to every marked node separately, keeping track of distances. While this is correct, it is inefficient because it may traverse the same paths multiple times, resulting in O(n * edges) complexity in the worst case. For large graphs, this becomes infeasible.

The optimal approach is to use Dijkstra's algorithm starting from `s`. Dijkstra's algorithm efficiently computes the shortest distance from a source node to all reachable nodes using a min-heap (priority queue). Once all distances are computed, we can simply check the distances of all marked nodes and return the minimum. This approach leverages the property of Dijkstra's algorithm that nodes are processed in order of increasing distance, ensuring correctness.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * E) | O(n) | DFS/BFS from `s` to each marked node separately |
| Optimal | O((n + E) log n) | O(n + E) | Dijkstra's algorithm with priority queue, avoids redundant traversals |

## Algorithm Walkthrough

1. Initialize an adjacency list `graph` from the `edges` array. Each node points to a list of `(neighbor, weight)` pairs. This allows for O(1) lookup of outgoing edges from any node.
2. Initialize a distance array `dist` of size `n`, filled with infinity, representing the current known shortest distance from `s` to each node. Set `dist[s] = 0`.
3. Initialize a min-heap (priority queue) `heap` and push `(0, s)` into it. This stores nodes to process along with their current distance from `s`.
4. While the heap is not empty, pop `(current_distance, node)` from the heap. If `current_distance` is greater than the known `dist[node]`, skip this node because a shorter path has already been found.
5. For each neighbor of the current node, calculate the new distance `new_dist = current_distance + weight`. If `new_dist < dist[neighbor]`, update `dist[neighbor] = new_dist` and push `(new_dist, neighbor)` into the heap.
6. After processing all nodes, iterate through all `marked` nodes and find the minimum distance among them. If none are reachable, return `-1`; otherwise, return the minimum distance.

Why it works: Dijkstra's algorithm guarantees that the first time a node is popped from the priority queue, the shortest path to it has been found. By tracking distances for all nodes and checking only the marked nodes at the end, we ensure correctness while avoiding unnecessary computations.

## Python Solution

```python
from typing import List
import heapq

class Solution:
    def minimumDistance(self, n: int, edges: List[List[int]], s: int, marked: List[int]) -> int:
        graph = [[] for _ in range(n)]
        for u, v, w in edges:
            graph[u].append((v, w))
        
        dist = [float('inf')] * n
        dist[s] = 0
        heap = [(0, s)]
        
        while heap:
            current_distance, node = heapq.heappop(heap)
            if current_distance > dist[node]:
                continue
            for neighbor, weight in graph[node]:
                new_dist = current_distance + weight
                if new_dist < dist[neighbor]:
                    dist[neighbor] = new_dist
                    heapq.heappush(heap, (new_dist, neighbor))
        
        min_distance = min((dist[m] for m in marked if dist[m] != float('inf')), default=-1)
        return min_distance
```

The Python implementation initializes the adjacency list and distance array, then runs Dijkstra's algorithm with a priority queue. After all reachable nodes are processed, it selects the minimum distance among the marked nodes using a generator expression, returning `-1` if none are reachable.

## Go Solution

```go
package main

import (
    "container/heap"
    "math"
)

type Item struct {
    node, dist int
}

type PriorityQueue []Item

func (pq PriorityQueue) Len() int { return len(pq) }
func (pq PriorityQueue) Less(i, j int) bool { return pq[i].dist < pq[j].dist }
func (pq PriorityQueue) Swap(i, j int) { pq[i], pq[j] = pq[j], pq[i] }

func (pq *PriorityQueue) Push(x any) { *pq = append(*pq, x.(Item)) }
func (pq *PriorityQueue) Pop() any {
    n := len(*pq)
    item := (*pq)[n-1]
    *pq = (*pq)[:n-1]
    return item
}

func minimumDistance(n int, edges [][]int, s int, marked []int) int {
    graph := make([][]Item, n)
    for _, e := range edges {
        u, v, w := e[0], e[1], e[2]
        graph[u] = append(graph[u], Item{node: v, dist: w})
    }
    
    dist := make([]int, n)
    for i := range dist { dist[i] = math.MaxInt64 }
    dist[s] = 0
    
    pq := &PriorityQueue{{node: s, dist: 0}}
    heap.Init(pq)
    
    for pq.Len() > 0 {
        item := heap.Pop(pq).(Item)
        node, d := item.node, item.dist
        if d > dist[node] { continue }
        for _, neighbor := range graph[node] {
            newDist := d + neighbor.dist
            if newDist < dist[neighbor.node] {
                dist[neighbor.node] = newDist
                heap.Push(pq, Item{node: neighbor.node, dist: newDist})
            }
        }
    }
    
    minDistance := math.MaxInt64
    for _, m := range marked {
        if dist[m] < minDistance {
            minDistance = dist[m]
        }
    }
    if minDistance == math.MaxInt64 {
        return -1
    }
    return minDistance
}
```

The Go solution mirrors the Python implementation but uses a `PriorityQueue` type implementing the heap interface. Special care is taken with integer infinity and heap operations, which are different from Python’s built-in `heapq`.
