# LeetCode 2765 - Longest Alternating Subarray

## Problem Understanding

The problem gives us a 0-indexed integer array `nums`, and asks us to find the length of the longest contiguous subarray that follows a very specific alternating pattern.

An alternating subarray must satisfy several conditions:

- Its length must be at least `2`.
- The second element must be exactly one greater than the first element.
- After that, the differences between consecutive elements must alternate between `-1` and `+1`.

This means the subarray must look like this:

```
[a, a+1, a, a+1, a, a+1, ...]
```

For example:

```
[3,4,3,4]
```

is valid because:

- `4 - 3 = 1`
- `3 - 4 = -1`
- `4 - 3 = 1`

The differences alternate correctly.

However:

```
[2,3,5]
```

is not valid because:

- `3 - 2 = 1`
- `5 - 3 = 2`

The alternating pattern breaks.

The input array length is at most `100`, which is relatively small. This means even quadratic solutions are acceptable. However, we still want a clean and efficient implementation.

The output should be the maximum length among all valid alternating subarrays. If no valid alternating subarray exists, we return `-1`.

Several edge cases are important:

- Arrays where no adjacent pair differs by `1`
- Arrays where the pattern starts correctly but breaks later
- Arrays where the entire array alternates correctly
- Arrays with repeated values
- Very small arrays of length `2`

Because the array length is small, we do not need sophisticated data structures. The challenge is mainly implementing the alternating difference logic correctly.

## Approaches

### Brute Force Approach

The brute-force solution considers every possible subarray and checks whether it satisfies the alternating pattern definition.

For every starting index `i`, we try every ending index `j`. For each subarray `nums[i:j+1]`, we verify:

1. The subarray length is at least `2`
2. The first difference is `+1`
3. Every subsequent difference alternates between `-1` and `+1`

This approach is guaranteed to find the correct answer because it exhaustively checks every possible subarray.

However, validating each subarray takes linear time, and there are already `O(n^2)` subarrays. This leads to cubic complexity in the worst case.

Although `n <= 100` makes this technically feasible, it is unnecessarily inefficient.

### Optimal Approach

The key observation is that once we choose a starting point, we can extend the subarray greedily while the alternating pattern remains valid.

Suppose we start at index `i`.

The expected differences are:

```
+1, -1, +1, -1, ...
```

Instead of repeatedly rebuilding and validating subarrays, we can scan forward and stop immediately when the pattern breaks.

This reduces repeated work significantly.

Since every starting index performs at most one forward scan, the total complexity becomes quadratic.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Checks every subarray and validates each one fully |
| Optimal | O(n²) | O(1) | Expands valid alternating sequences greedily |

## Algorithm Walkthrough

### Optimal Algorithm

1. Initialize a variable `max_length` to `-1`.

We use `-1` because the problem explicitly requires returning `-1` when no valid alternating subarray exists.
2. Iterate through every possible starting index `start`.

Each index can potentially begin an alternating subarray.
3. For each starting index, check whether the first required condition holds.

Specifically:

```
nums[start + 1] - nums[start] == 1
```

If this is not true, then no valid alternating subarray can begin at `start`.
4. If the first difference is valid, initialize:

- `current_length = 2`
- `expected_difference = -1`

We already consumed the first required `+1` difference, so the next difference must be `-1`.
5. Continue extending the subarray forward.

For each next index:

- Compute the actual difference
- Compare it against `expected_difference`
6. If the difference matches:

- Increase `current_length`
- Flip `expected_difference` by multiplying by `-1`

This alternates between `+1` and `-1`.
7. If the difference does not match:

- Stop extending this subarray
- Move to the next starting index
8. Update `max_length` whenever a longer valid subarray is found.
9. After all starting indices are processed, return `max_length`.

### Why it works

The algorithm works because every valid alternating subarray must begin with a `+1` difference and then strictly alternate between `-1` and `+1`.

For every possible starting position, the algorithm greedily extends the subarray exactly as long as the required pattern remains valid. The moment the pattern breaks, no longer extension from that starting point can ever become valid again.

Since all starting positions are explored and every maximal valid extension is checked, the algorithm is guaranteed to find the longest alternating subarray.

## Python Solution

```python
from typing import List

class Solution:
    def alternatingSubarray(self, nums: List[int]) -> int:
        n = len(nums)
        max_length = -1

        for start in range(n - 1):
            if nums[start + 1] - nums[start] != 1:
                continue

            current_length = 2
            max_length = max(max_length, current_length)

            expected_difference = -1

            for end in range(start + 2, n):
                actual_difference = nums[end] - nums[end - 1]

                if actual_difference != expected_difference:
                    break

                current_length += 1
                max_length = max(max_length, current_length)

                expected_difference *= -1

        return max_length
```

### Python Implementation Explanation

The implementation begins by storing the array length and initializing `max_length` to `-1`.

The outer loop tries every possible starting index. Since a valid alternating subarray must contain at least two elements, the loop only goes to `n - 2`.

The first condition checked is whether the first difference equals `1`. If not, we skip immediately because the subarray cannot be valid.

Once a valid start is found, the algorithm initializes:

- `current_length = 2`
- `expected_difference = -1`

The inner loop extends the subarray one element at a time. For every new pair of adjacent elements, the code computes the actual difference and compares it with the expected one.

If the difference matches:

- The current subarray remains valid
- The length increases
- The expected difference flips sign

If the difference does not match, the alternating property is broken, so the loop terminates.

Throughout the process, `max_length` tracks the longest valid subarray discovered.

The Go implementation follows the same logic as the Python solution.

The function returns `-1` when no valid alternating subarray exists, exactly matching the problem requirements.

## Edge Cases

One important edge case occurs when no valid alternating subarray exists at all. For example:

```
[1,1,1]
```

or

```
[5,7,9]
```

In these cases, no adjacent pair differs by exactly `1`, so the algorithm never enters the extension phase. The implementation correctly returns `-1` because `max_length` remains unchanged.

Another important case is when the entire array forms a valid alternating sequence. For example:

```
[2,3,2,3,2,3]
```

A buggy implementation might stop too early or incorrectly alternate expected differences. This solution carefully flips the expected difference after every successful extension, allowing the full sequence to be counted correctly.

A third important edge case occurs when the pattern begins correctly but breaks midway. Consider:

```
[4,5,4,6]
```

The sequence starts with valid differences:

```
+1, -1
```

but then:

```
6 - 4 = 2
```

which breaks the pattern. The implementation immediately terminates the current expansion when the difference no longer matches the expected value, preventing invalid subarrays from being counted.
