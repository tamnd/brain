## Solution

## Problem Understanding

The problem requires transforming a nested JavaScript-style object or array such that any `undefined` values are replaced with `null`. In JavaScript, `undefined` represents a missing value, while `null` explicitly denotes an intentional absence of value. The distinction is significant when serializing objects using `JSON.stringify()`, since `undefined` is omitted, potentially leading to data loss.

The input `obj` can be a deeply nested structure composed of dictionaries (objects) and lists (arrays), potentially containing other nested objects, arrays, or primitive values like numbers, strings, `null`, or `undefined`. The output should preserve the structure of `obj` but replace every occurrence of `undefined` with `null`.

The constraints tell us that `obj` is always valid JSON (or JavaScript-like object) and that its serialized string length is manageable, up to 105 characters. This indicates that recursion or iterative deep traversal is feasible. Important edge cases include empty arrays or objects, nested arrays with `undefined` values, and mixed types within arrays or objects.

## Approaches

The brute-force approach would involve iterating over every key and index in the structure, checking the value, and explicitly replacing `undefined` with `null`. This could be implemented recursively. While this guarantees correctness, a naive implementation might be inefficient if it repeatedly reconstructs arrays or objects rather than modifying them in place.

The optimal approach leverages recursive traversal of the nested structure, directly mutating arrays and dictionaries as we visit them. The key insight is that JSON structures are inherently tree-like, so a depth-first traversal allows us to process each element exactly once, replacing `undefined` values with `null` without extra copying or reconstruction. This ensures O(n) time complexity, where n is the number of elements in the nested structure, and O(d) space complexity for recursion, where d is the depth of the nesting.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Recursively copies the structure while replacing undefined values; works but may involve extra memory |
| Optimal | O(n) | O(d) | In-place recursive traversal replacing undefined with null; efficient for deep nested structures |

## Algorithm Walkthrough

1. Define a recursive helper function `replace_undefined` that accepts a value `v`.
2. If `v` is `undefined`, return `null`. This handles the direct replacement at any level.
3. If `v` is a dictionary (object), iterate over each key-value pair. Recursively call `replace_undefined` on the value and assign the result back to the dictionary at that key. This preserves the object structure.
4. If `v` is a list (array), iterate over each element. Recursively call `replace_undefined` on each element and assign the result back to the array at that index. This preserves the array structure.
5. For any other primitive value (number, string, `null`), return it as is.
6. Call `replace_undefined` on the top-level `obj` and return the result.

Why it works: Each element is visited exactly once. Undefined values are replaced immediately, while all other values are preserved, ensuring that the structure remains unchanged. Since JSON-like structures are trees, the recursive depth-first approach guarantees that no value is skipped.

## Python Solution

```python
from typing import Any

class Solution:
    def undefinedToNull(self, obj: Any) -> Any:
        def replace_undefined(v: Any) -> Any:
            if v is None:
                return None
            if v == "undefined":
                return None
            if isinstance(v, dict):
                for key in v:
                    v[key] = replace_undefined(v[key])
                return v
            if isinstance(v, list):
                for i in range(len(v)):
                    v[i] = replace_undefined(v[i])
                return v
            return v
        
        return replace_undefined(obj)
```

In the Python solution, the function recursively traverses the input object. Dictionaries are iterated key by key, and lists are iterated index by index. The function ensures that any `"undefined"` values (represented as a string for Python compatibility) are replaced with `None`, which corresponds to `null` in JSON. Other types remain unchanged.

## Go Solution

```go
package main

func undefinedToNull(obj any) any {
	switch v := obj.(type) {
	case map[string]any:
		for key, val := range v {
			v[key] = undefinedToNull(val)
		}
		return v
	case []any:
		for i := range v {
			v[i] = undefinedToNull(v[i])
		}
		return v
	case string:
		if v == "undefined" {
			return nil
		}
		return v
	default:
		return v
	}
}
```

The Go solution mirrors the Python implementation using type assertions. Maps are traversed key by key, and slices are traversed index by index. The string `"undefined"` is converted to `nil`. Primitive types and `nil` are returned unchanged.

The time complexity is linear because every element is checked exactly once. The space complexity depends on the maximum depth of the nested object or array, as each recursive call adds to the stack.

## Edge Cases

One important edge case is an empty object or array. A naive implementation that assumes at least one key or element could fail or raise an exception. Our implementation correctly returns an empty structure without modification.

Another edge case is deeply nested structures, where recursion depth could be high. The algorithm handles this naturally, as each recursive call replaces undefined values in place, and Python or Go's recursion limits are sufficient for the problem constraints.

Finally, inputs that contain no `undefined` values at all should be returned unchanged. This ensures that our solution does not inadvertently modify valid data, which is important when preserving the integrity of JSON data.
