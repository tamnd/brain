## Solution

## Problem Understanding

The problem is asking to select a **non-empty subset** of numbers from a given array `nums` such that the **product** of all numbers in that subset is maximized. In other words, we need to form a group of students whose "strength" - defined as the multiplication of their scores - is the largest possible.

The input `nums` is an integer array where each element represents a score. The scores can be negative, zero, or positive, and the length of the array is small, between 1 and 13. The output is a single integer, the maximal product achievable from any non-empty subset.

The constraints suggest that a brute-force solution is feasible because the maximum number of subsets is `2^13 - 1 = 8191`, which is small enough to enumerate. However, we can optimize without generating all subsets. Key edge cases include:

- All numbers are negative. Choosing all may result in a negative product, but excluding the smallest magnitude number may yield a positive product.
- Zeros in the array. Multiplying by zero kills the product.
- Single element array. The product is just that element.

Understanding these nuances is critical to avoiding off-by-one or sign errors in subset selection.

## Approaches

The **brute-force approach** would enumerate all possible non-empty subsets of `nums`, compute their products, and return the maximum. This guarantees correctness but is inefficient because computing all subsets takes exponential time.

The **optimal approach** leverages the properties of negative and positive numbers. The key insight is that to maximize a product:

1. Positive numbers should always be included.
2. An even count of negative numbers is desirable because their product is positive.
3. If there is an odd count of negative numbers, exclude the one with the smallest absolute value.
4. Zeros are only useful if all other numbers are negative and excluding them would leave an empty subset.

This approach reduces the problem to counting negatives, identifying the smallest negative, and multiplying the suitable elements.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n * n) | O(1) | Enumerate all subsets and calculate products. |
| Optimal | O(n log n) | O(1) | Count negatives, handle odd/even negatives, ignore zeros unless needed. |

## Algorithm Walkthrough

1. Separate the numbers into positive numbers, negative numbers, and count zeros.
2. If there are no positive numbers and at most one negative number, return the maximum number, which might be negative or zero.
3. If the number of negative numbers is odd, remove the one with the smallest absolute value to maximize the product.
4. Multiply all remaining positive and negative numbers together. Ignore zeros, as multiplying by zero would reduce the product unless no other numbers exist.
5. Return the computed product.

**Why it works:** The product of an even number of negative numbers is positive, maximizing the result. Positive numbers always increase the product, while zeros are only used to avoid an empty subset if all numbers are negative and cannot form a positive product.

## Python Solution

```python
from typing import List

class Solution:
    def maxStrength(self, nums: List[int]) -> int:
        if len(nums) == 1:
            return nums[0]

        positives = [x for x in nums if x > 0]
        negatives = [x for x in nums if x < 0]

        if not positives and len(negatives) == 1:
            return negatives[0]

        if len(negatives) % 2 == 1:
            negatives.remove(max(negatives))

        product = 1
        for x in positives + negatives:
            product *= x

        if product == 1 and not positives and negatives == []:
            return 0

        return product
```

This implementation separates the array into positives and negatives. It ensures that an odd negative is removed to make the product positive. It multiplies all remaining numbers and handles the special cases where the subset would otherwise be empty or only contain negatives.

## Go Solution

```go
func maxStrength(nums []int) int64 {
    var positives []int64
    var negatives []int64

    for _, x := range nums {
        if x > 0 {
            positives = append(positives, int64(x))
        } else if x < 0 {
            negatives = append(negatives, int64(x))
        }
    }

    if len(positives) == 0 && len(negatives) == 1 {
        return negatives[0]
    }

    if len(negatives)%2 == 1 {
        maxNeg := negatives[0]
        idx := 0
        for i, v := range negatives {
            if v > maxNeg {
                maxNeg = v
                idx = i
            }
        }
        negatives = append(negatives[:idx], negatives[idx+1:]...)
    }

    product := int64(1)
    if len(positives) == 0 && len(negatives) == 0 {
        return 0
    }
    for _, x := range positives {
        product *= x
    }
    for _, x := range negatives {
        product *= x
    }
    return product
}
```

In Go, we use slices instead of separate lists. Typecasting to `int64` ensures we do not overflow the 32-bit integer range. Removing the largest negative when the count is odd is done via slicing.

## Edge Cases

**Edge Case 1: Single element array**

If the array contains only one element, the maximum strength is simply that element. This handles arrays like `[0]`, `[1]`, or `[-1]`.

**Edge Case 2: All negatives with odd count**

When all numbers are negative and their count is odd, including all would result in a negative product. The algorithm correctly removes the negative with the smallest absolute value to maximize the product.

**Edge Case 3: Presence of zeros**

Zeros can reduce the product to zero. The algorithm ignores zeros unless no positive or negative numbers are available, ensuring the result is never incorrectly zero when a positive product exists.
