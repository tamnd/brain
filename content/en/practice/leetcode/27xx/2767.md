## Solution

## Problem Understanding

The problem gives us a binary string `s`, and we must split it into one or more contiguous substrings such that every substring is considered "beautiful".

A substring is beautiful if it satisfies two conditions:

1. It does not start with `'0'`, meaning no leading zeros are allowed.
2. When interpreted as a binary number, its value is a power of `5`.

The goal is to return the minimum number of beautiful substrings needed to partition the entire string. If no valid partition exists, we return `-1`.

For example, the binary string `"101"` equals decimal `5`, which is a power of `5`, so it is beautiful. The string `"1"` equals decimal `1`, which is `5^0`, so it is also beautiful. However, `"0101"` is invalid because it contains a leading zero.

The constraints are very small:

- `1 <= s.length <= 15`

This is an important observation. Since the string length is at most 15, exponential style solutions such as backtracking or dynamic programming over subsets become feasible. We do not need sophisticated optimizations for large inputs.

Several edge cases are important:

- Strings beginning with `'0'` are problematic because no beautiful substring may contain leading zeros.
- Some strings cannot be partitioned at all, such as `"0"`.
- The optimal solution may require many small partitions instead of fewer large ones.
- A substring may represent a large power of `5`, so we must correctly recognize all valid binary powers within length 15.

Because the input size is small, we can precompute every power of `5` representable within 15 binary digits and use that information during partitioning.

## Approaches

### Brute Force Approach

The brute force idea is to try every possible partition of the string.

For a string of length `n`, there are `2^(n-1)` possible ways to place cuts between characters. For each partition, we check every substring:

- Does it start with `'0'`?
- Does its binary value equal a power of `5`?

Among all valid partitions, we keep the minimum number of substrings.

This approach is correct because it exhaustively explores every possible partition. If a valid partition exists, brute force will eventually find it.

However, this method repeats the same work many times. Different partition paths often re-evaluate the same suffix repeatedly. Although `n <= 15` is small enough that brute force could pass, it is inefficient and not scalable.

### Optimal Approach, Dynamic Programming with Backtracking

The key insight is that the problem has overlapping subproblems.

Suppose we already know the minimum number of beautiful substrings needed to partition the suffix starting at index `i`. Then we can reuse that result whenever another recursive path reaches the same position.

This naturally leads to dynamic programming.

We first precompute all binary strings that represent powers of `5` and fit within 15 bits. Then we use DFS with memoization:

- At index `i`, try every substring `s[i:j+1]`.
- If the substring is beautiful, recursively solve the remaining suffix starting at `j+1`.
- Take the minimum result among all valid choices.

Memoization ensures each index is solved only once.

Because the string length is tiny, this solution is both simple and efficient.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n × n) | O(n) | Tries every possible partition |
| Optimal | O(n²) | O(n) | DFS with memoization and precomputed powers of 5 |

## Algorithm Walkthrough

1. Precompute all powers of `5` whose binary representation length is at most 15.

Since the input string length is at most 15, no valid substring can exceed 15 bits. We repeatedly multiply by `5` and store the binary representations in a hash set for fast lookup.
2. Define a recursive function `dfs(index)`.

This function returns the minimum number of beautiful substrings needed to partition `s[index:]`.
3. Handle the base case.

If `index == len(s)`, we have successfully partitioned the entire string, so we return `0`.
4. Reject substrings with leading zeros.

If `s[index] == '0'`, then no valid beautiful substring can start here. Return infinity to indicate failure.
5. Try all possible ending positions.

For every `end` from `index` to `len(s)-1`, construct the substring:

```
s[index:end+1]
```

If this substring exists in the precomputed power-of-5 set, recursively solve the remaining suffix.
6. Compute the minimum answer.

If the recursive call succeeds, update:

```
answer = min(answer, 1 + dfs(end + 1))
```

The `1` represents the current beautiful substring.
7. Memoize results.

Store the computed result for each index so repeated computations are avoided.
8. Return the final answer.

If the result is infinity, return `-1`. Otherwise return the computed minimum.

### Why it works

At every position, the algorithm considers every possible beautiful substring that could start there. For each valid choice, it recursively computes the optimal solution for the remaining suffix. Since every partition must begin with some valid first substring, taking the minimum over all such choices guarantees the globally optimal answer. Memoization ensures each suffix is solved exactly once.

## Python Solution

```python
from functools import lru_cache
from math import inf

class Solution:
    def minimumBeautifulSubstrings(self, s: str) -> int:
        power_of_five = set()

        value = 1
        while len(bin(value)) - 2 <= 15:
            power_of_five.add(bin(value)[2:])
            value *= 5

        n = len(s)

        @lru_cache(None)
        def dfs(index: int) -> int:
            if index == n:
                return 0

            if s[index] == '0':
                return inf

            best = inf

            for end in range(index, n):
                substring = s[index:end + 1]

                if substring in power_of_five:
                    next_result = dfs(end + 1)

                    if next_result != inf:
                        best = min(best, 1 + next_result)

            return best

        answer = dfs(0)

        return -1 if answer == inf else answer
```

The implementation begins by precomputing every binary representation of powers of `5` that can fit within 15 bits. Using a set allows constant time membership checks later.

The recursive `dfs(index)` function solves the problem for the suffix beginning at `index`. Memoization with `@lru_cache` ensures that once a suffix has been solved, its result is reused immediately.

The base case occurs when `index == n`, meaning the entire string has been partitioned successfully.

If the current character is `'0'`, the function immediately returns infinity because leading zeros are forbidden.

The loop tries every possible substring beginning at the current index. Whenever a substring is found in the `power_of_five` set, the algorithm recursively solves the remaining suffix and updates the best answer.

Finally, if no valid partition exists, the result remains infinity and we return `-1`.

## Go Solution

```go
package main

import (
	"math"
)

func minimumBeautifulSubstrings(s string) int {
	powerOfFive := make(map[string]bool)

	value := 1
	for len(toBinary(value)) <= 15 {
		powerOfFive[toBinary(value)] = true
		value *= 5
	}

	n := len(s)

	memo := make([]int, n)
	for i := range memo {
		memo[i] = -2
	}

	var dfs func(int) int

	dfs = func(index int) int {
		if index == n {
			return 0
		}

		if s[index] == '0' {
			return math.MaxInt32
		}

		if memo[index] != -2 {
			return memo[index]
		}

		best := math.MaxInt32

		for end := index; end < n; end++ {
			substring := s[index : end+1]

			if powerOfFive[substring] {
				nextResult := dfs(end + 1)

				if nextResult != math.MaxInt32 {
					if 1+nextResult < best {
						best = 1 + nextResult
					}
				}
			}
		}

		memo[index] = best
		return best
	}

	answer := dfs(0)

	if answer == math.MaxInt32 {
		return -1
	}

	return answer
}

func toBinary(num int) string {
	if num == 0 {
		return "0"
	}

	result := ""

	for num > 0 {
		if num%2 == 0 {
			result = "0" + result
		} else {
			result = "1" + result
		}

		num /= 2
	}

	return result
}
```

The Go solution follows the same logic as the Python version. Since Go does not provide built in memoization decorators, we use a slice named `memo` initialized with `-2` to indicate uncomputed states.

Instead of Python's `inf`, the implementation uses `math.MaxInt32` as a sentinel value representing an impossible partition.

Go also requires a helper function to convert integers into binary strings because there is no direct equivalent of Python's `bin()`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n²) | There are `n` states, each trying up to `n` substrings |
| Space | O(n) | Memoization recursion stack and cache |

The algorithm computes each `dfs(index)` state once because of memoization. For every state, we iterate over all possible ending positions, producing at most `n` substring checks. Since `n <= 15`, this is extremely efficient in practice.

## Edge Cases

### Strings Beginning With Zero

Inputs such as `"0"` or `"01"` are invalid because beautiful substrings cannot contain leading zeros. A common bug is accidentally allowing `"0"` as a valid binary representation. The implementation prevents this immediately by rejecting any recursive state where the current character is `'0'`.

### Entire String Is One Power of Five

Some inputs, such as `"101"` or `"11001"`, should return `1` because the entire string already represents a power of `5`. The algorithm correctly handles this because it always tries every possible ending index, including the final character of the string.

### No Valid Partition Exists

Certain strings cannot be decomposed into valid beautiful substrings at all. For example, `"1001110001"` has no valid partition. The DFS uses infinity as a sentinel value for impossible states. If all recursive paths fail, the final answer remains infinity and the algorithm returns `-1`.

### Multiple Possible Partitions

Some strings can be partitioned in different valid ways. The challenge is finding the minimum number of substrings. For example, `"101101"` can be split as `"101" + "101"`. The recursive search evaluates every valid first substring and chooses the minimum total count, guaranteeing optimality.
