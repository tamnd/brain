## Solution

## Problem Understanding

The problem asks us to construct a new array `nums3` by choosing, at each index `i`, either `nums1[i]` or `nums2[i]`. The goal is to maximize the length of the **longest non-decreasing subarray** in `nums3`. A subarray is a contiguous sequence, so the problem focuses on maximizing contiguous segments where each element is greater than or equal to the previous element.

The inputs are two integer arrays of equal length `n` (up to 10^5), with each element between 1 and 10^9. The output is a single integer representing the maximum length of a non-decreasing subarray that can be achieved.

Important observations from the constraints:

- `n` can be as large as 10^5, so any solution that is worse than O(n) or O(n log n) will likely time out.
- Element values can be very large, so approaches depending on value ranges (like counting sort or array-indexed DP) are impractical.
- Arrays are non-empty, so we always have at least one element to form a subarray.

Potential edge cases include:

- Both arrays are identical.
- Arrays have strictly decreasing sequences in both.
- Arrays have alternating large and small numbers.
- Arrays of length 1.

The challenge is not just picking the largest elements but carefully deciding at each index which choice allows the longest contiguous non-decreasing subarray.

## Approaches

### Brute Force Approach

A naive brute-force solution would try every possible combination of selecting elements from `nums1` or `nums2` for `nums3`. For each combination, we would compute the length of the longest non-decreasing subarray. This is correct but infeasible because there are 2^n possible combinations, which is astronomical for n = 10^5.

### Optimal Approach

The key insight is that we do not need to try all combinations. At each index, the choice of which number to take depends **only on the previous index** and the current potential numbers. This allows a **dynamic programming approach**.

We maintain two state variables:

1. `dp1` = length of the longest non-decreasing subarray ending at index `i` if we pick `nums1[i]`.
2. `dp2` = length of the longest non-decreasing subarray ending at index `i` if we pick `nums2[i]`.

For each index `i > 0`, we can update:

- `dp1[i]` = 1 + max(dp1[i-1] if nums1[i] >= nums1[i-1], dp2[i-1] if nums1[i] >= nums2[i-1])
- `dp2[i]` = 1 + max(dp1[i-1] if nums2[i] >= nums1[i-1], dp2[i-1] if nums2[i] >= nums2[i-1])

The result is the maximum value of `dp1[i]` and `dp2[i]` over all indices. This solution only requires **O(n)** time and **O(1)** space if we maintain just the previous values instead of full arrays.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Try all combinations of nums1 and nums2 |
| Optimal | O(n) | O(1) | DP tracking longest subarray ending with nums1[i] or nums2[i] |

## Algorithm Walkthrough

1. Initialize `prev1` and `prev2` to 1, representing the longest non-decreasing subarray ending at index 0 using `nums1[0]` and `nums2[0]` respectively.
2. Initialize `max_len` to 1.
3. Iterate `i` from 1 to n-1:

1. For each index, calculate the new length if we pick `nums1[i]` by checking whether it can extend the subarray ending with `nums1[i-1]` or `nums2[i-1]`.
2. Similarly, calculate the new length if we pick `nums2[i]`.
3. Update `prev1` and `prev2` to these new values.
4. Update `max_len` as the maximum of `max_len`, `prev1`, and `prev2`.
4. Return `max_len`.

**Why it works:** The algorithm correctly propagates the longest non-decreasing subarray lengths for each possible choice at every index. By considering both previous options, it ensures we do not miss any valid contiguous sequence.

## Python Solution

```python
from typing import List

class Solution:
    def maxNonDecreasingLength(self, nums1: List[int], nums2: List[int]) -> int:
        n = len(nums1)
        prev1, prev2 = 1, 1
        max_len = 1
        
        for i in range(1, n):
            temp1 = temp2 = 1
            if nums1[i] >= nums1[i-1]:
                temp1 = max(temp1, prev1 + 1)
            if nums1[i] >= nums2[i-1]:
                temp1 = max(temp1, prev2 + 1)
            if nums2[i] >= nums1[i-1]:
                temp2 = max(temp2, prev1 + 1)
            if nums2[i] >= nums2[i-1]:
                temp2 = max(temp2, prev2 + 1)
            prev1, prev2 = temp1, temp2
            max_len = max(max_len, prev1, prev2)
        
        return max_len
```

**Implementation walkthrough:** We maintain `prev1` and `prev2` to track the length of the longest non-decreasing subarray ending at the previous index. At each iteration, we compute temporary lengths `temp1` and `temp2` for the current index based on both previous choices, then update the previous state variables. The maximum length is continuously tracked.

## Go Solution

```go
func maxNonDecreasingLength(nums1 []int, nums2 []int) int {
    n := len(nums1)
    prev1, prev2 := 1, 1
    maxLen := 1

    for i := 1; i < n; i++ {
        temp1, temp2 := 1, 1
        if nums1[i] >= nums1[i-1] {
            temp1 = max(temp1, prev1+1)
        }
        if nums1[i] >= nums2[i-1] {
            temp1 = max(temp1, prev2+1)
        }
        if nums2[i] >= nums1[i-1] {
            temp2 = max(temp2, prev1+1)
        }
        if nums2[i] >= nums2[i-1] {
            temp2 = max(temp2, prev2+1)
        }
        prev1, prev2 = temp1, temp2
        maxLen = max(maxLen, prev1, prev2)
    }

    return maxLen
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

**Go-specific notes:** The Go version is nearly identical. We define a helper function `max` because Go does not have a built-in generic max function for integers. We track `prev1` and `prev2` in the same way.
