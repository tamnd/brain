# LeetCode 2796 - Repeat String

## Problem Understanding

This problem asks us to enhance strings with a custom `replicate(x)` method that returns the original string repeated exactly `x` times. The important restriction is that we should not use the built-in `string.repeat` functionality.

Given a string and an integer `times`, we must construct a new string consisting of the original string concatenated to itself exactly `times` times.

For example, if the string is `"hello"` and `times = 2`, the result should be `"hellohello"`. If the string is `"code"` and `times = 3`, the result should be `"codecodecode"`.

The constraints tell us that:

- `times` can be as large as `100,000`
- The original string length can be as large as `1,000`

This means the final output itself can contain up to 100 million characters in a theoretical worst case. Since the output size is large, any solution must at least account for generating that output.

The follow up introduces an artificial assumption: string concatenation is considered an `O(1)` operation. Under this assumption, we can analyze algorithms based solely on the number of concatenation operations performed. This opens the door to a logarithmic solution using exponentiation by squaring.

Some important edge cases include:

- Repeating a string exactly once.
- Repeating a single-character string many times.
- Very large values of `times`.
- Strings containing arbitrary characters.
- The problem guarantees `times >= 1`, so we never need to handle zero or negative repetitions.

## Approaches

### Brute Force

The most straightforward solution is to repeatedly append the original string to a result variable.

We start with an empty string. Then we perform `times` iterations, appending the original string during each iteration.

For example:

- `"code"`, `times = 3`
- Result starts as `""`
- After first append: `"code"`
- After second append: `"codecode"`
- After third append: `"codecodecode"`

This approach is clearly correct because each iteration adds exactly one copy of the original string, and after `times` iterations we have exactly `times` copies.

However, it requires `times` concatenation operations. Under the follow up assumption where each concatenation costs `O(1)`, the runtime is `O(times)`.

### Optimal Approach

The key observation is that string repetition resembles integer exponentiation.

Instead of adding one copy at a time, we can repeatedly double larger and larger blocks:

- `s`
- `2s`
- `4s`
- `8s`
- ...

This is exactly the same idea as binary exponentiation.

We maintain:

- `current`, representing the current power-of-two block.
- `result`, representing the accumulated answer.

For every set bit in the binary representation of `times`, we add the corresponding block to the answer.

For example:

- `times = 13`
- Binary representation: `1101₂`
- `13 = 8 + 4 + 1`

Therefore:

```
13 * s = 8s + 4s + 1s
```

We build the answer using only logarithmically many doubling operations.

Under the follow up assumption that concatenation is `O(1)`, this requires only `O(log times)` operations.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Append one copy at a time |
| Optimal | O(log n) | O(n) | Uses binary exponentiation style doubling |

Here, `n` refers to `times` under the follow up analysis.

## Algorithm Walkthrough

### Optimal Algorithm

1. Initialize an empty string called `result`.
2. Initialize a variable called `current` with the original string.
3. While `times` is greater than zero, inspect its least significant bit.
4. If the least significant bit is `1`, add `current` to `result`. This means the corresponding power of two contributes to the final repetition count.
5. Double `current` by concatenating it with itself. After this operation, `current` represents the next power-of-two block.
6. Divide `times` by two using integer division, or equivalently perform a right shift.
7. Continue until all bits of `times` have been processed.
8. Return `result`.

### Why it works

The algorithm relies on the binary representation of `times`. Every positive integer can be expressed as a sum of powers of two. The variable `current` stores the string repeated a power-of-two number of times, while the bits of `times` determine which of those blocks should be included in the final answer. Since every required repetition count is represented exactly once in binary, the constructed string contains exactly `times` copies of the original string.

## Python Solution

```python
class String:
    def __init__(self, value: str):
        self.value = value

    def replicate(self, times: int) -> str:
        result = ""
        current = self.value

        while times > 0:
            if times & 1:
                result += current

            current += current
            times >>= 1

        return result
```

The implementation begins by storing the original string in `current`. The variable `result` accumulates the final answer.

During each iteration, the algorithm examines the lowest bit of `times`. If that bit is set, the current power-of-two block contributes to the answer and is appended to `result`.

Next, `current` is doubled by concatenating it with itself. This transforms a block representing `2^k` copies into one representing `2^(k+1)` copies.

Finally, `times` is shifted right by one position so the next bit can be processed. When all bits have been consumed, `result` contains exactly the required number of repetitions.

## Go Solution

```go
type String struct {
	value string
}

func (s String) Replicate(times int) string {
	result := ""
	current := s.value

	for times > 0 {
		if times&1 == 1 {
			result += current
		}

		current += current
		times >>= 1
	}

	return result
}
```

The Go implementation follows exactly the same binary exponentiation strategy as the Python version.

Strings in Go are immutable, just as they are in Python. The algorithm repeatedly creates larger strings through concatenation. The bitwise operations `&` and `>>` are used to inspect and process the binary representation of `times`.

There are no special concerns regarding integer overflow because the constraint on `times` is relatively small.

## Complexity Analysis

### Follow Up Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(log n) | One iteration per bit of `times` |
| Space | O(n) | Output string storage |

Under the follow up assumption that every string concatenation is an `O(1)` operation, the algorithm performs only logarithmically many iterations. Each iteration processes one bit of the repetition count, so the runtime becomes `O(log n)` where `n = times`.

In a real programming language, string concatenation is not actually constant time because new strings must be allocated and copied. Nevertheless, under the problem's stated assumption, the binary exponentiation solution achieves the desired logarithmic complexity.

## Edge Cases

### Repetition Count Equals One

When `times = 1`, the algorithm should return the original string unchanged. This is a common source of off-by-one errors in repetition algorithms. The binary exponentiation approach handles it naturally because the first bit is set, causing exactly one copy of the string to be added to the result.

### Very Large Repetition Counts

The value of `times` may be as large as `100,000`. A naive implementation performs one append operation per repetition. The optimal solution instead processes only the bits of `times`, requiring logarithmically many iterations. This makes it much more efficient under the follow up complexity model.

### Single Character Strings

Strings of length one often reveal mistakes in concatenation logic because visual inspection becomes difficult when many copies are produced. The algorithm treats single-character strings exactly the same as longer strings, so `"a"` repeated five times correctly becomes `"aaaaa"`.

### Power-of-Two Repetition Counts

Values such as `2`, `4`, `8`, and `16` contain exactly one set bit in binary. In these cases, only one accumulated block is added to the answer. This verifies that the doubling logic correctly constructs larger blocks and that no unnecessary copies are introduced.

### Odd Repetition Counts

Values such as `3`, `5`, and `7` require combining multiple power-of-two blocks. These cases verify that the algorithm correctly interprets the binary representation of `times` and includes every required component exactly once.
