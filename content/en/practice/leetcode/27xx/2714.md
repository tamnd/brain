## Solution

## Problem Understanding

The problem asks us to find the **shortest path** between two nodes `s` and `d` in an **undirected weighted graph**, but with a special condition: we are allowed to **set the weight of at most `k` edges to zero** along the path to potentially shorten it.

The graph is given as a list of edges, each with a start node `ui`, an end node `vi`, and a positive weight `wi`. The graph is connected, has no self-loops or repeated edges, and `n` ranges from 2 to 500. The maximum number of edges is limited either by the complete graph formula `n*(n-1)/2` or by 10,000, whichever is smaller.

The task is to compute the minimum possible path length from `s` to `d` **after strategically zeroing out up to `k` edge weights**. Edge cases that are important to note include when `k` is 0 (no edge can be zeroed), when `k` is greater than the number of edges in a shortest path (all edges could potentially be zeroed), or when the graph is dense enough that multiple paths exist with similar weights.

## Approaches

A brute-force solution would attempt **every combination of up to `k` edges to zero** and compute shortest paths each time. This would involve iterating over all subsets of edges of size `0..k` and performing a shortest path calculation (like Dijkstra) for each subset. Even for modest `n`, this approach is infeasible due to combinatorial explosion, as the number of subsets of edges grows exponentially:

$$\sum_{i=0}^{k} \binom{m}{i} \quad \text{where } m \text{ is the number of edges.}$$

The key insight for an optimal solution is to **treat the number of zeroed edges as a state in a modified Dijkstra algorithm**. Instead of a single distance per node, we maintain a distance for each node with `0..k` zeroed edges. This allows us to explore paths efficiently while dynamically choosing whether to zero the next edge or not.

Effectively, the algorithm keeps track of `dist[node][used_hops]`, representing the shortest distance to reach `node` after using `used_hops` free edges. We then use a **priority queue** to always expand the current shortest path, similar to Dijkstra.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(C(m,k) * (m + n log n)) | O(n + m) | Try every combination of edges to zero; infeasible for large graphs |
| Optimal | O(m * k * log(n * k)) | O(n * k + m) | Modified Dijkstra tracking number of zeroed edges as state |

## Algorithm Walkthrough

1. **Graph Representation**: Build an adjacency list `adj` from the `edges` array. Each node stores a list of tuples `(neighbor, weight)`.
2. **Distance Table**: Initialize a 2D array `dist[n][k+1]` to infinity. Here, `dist[u][h]` stores the shortest distance to reach node `u` after using `h` zeroed edges. Set `dist[s][0] = 0`.
3. **Priority Queue Initialization**: Use a min-heap priority queue `pq` that stores tuples `(current_distance, node, used_hops)`. Push `(0, s, 0)` to start.
4. **Modified Dijkstra Expansion**: While the priority queue is not empty:

4.1 Pop `(cur_dist, u, used_hops)` from the queue.

4.2 For each neighbor `v` of `u` with edge weight `w`:

4.3 **Option 1** - Do not zero the edge: if `cur_dist + w < dist[v][used_hops]`, update `dist[v][used_hops] = cur_dist + w` and push `(dist[v][used_hops], v, used_hops)`.

4.4 **Option 2** - Zero the edge: if `used_hops < k` and `cur_dist < dist[v][used_hops + 1]`, update `dist[v][used_hops + 1] = cur_dist` and push `(dist[v][used_hops + 1], v, used_hops + 1)`.
5. **Final Answer**: The shortest distance from `s` to `d` is `min(dist[d])`, considering all possible numbers of used zero edges.

**Why it works**: The algorithm maintains the invariant that `dist[u][h]` is always the shortest distance to `u` using `h` free edges. By using a priority queue, we ensure we expand the shortest possible paths first, and by considering both "use zero edge" and "do not use" options at each edge, we explore all optimal combinations efficiently without redundant computations.

## Python Solution

```python
from typing import List
import heapq

class Solution:
    def shortestPathWithHops(self, n: int, edges: List[List[int]], s: int, d: int, k: int) -> int:
        adj = [[] for _ in range(n)]
        for u, v, w in edges:
            adj[u].append((v, w))
            adj[v].append((u, w))
        
        dist = [[float('inf')] * (k + 1) for _ in range(n)]
        dist[s][0] = 0
        pq = [(0, s, 0)]  # (distance, node, used_hops)
        
        while pq:
            cur_dist, u, used_hops = heapq.heappop(pq)
            
            if cur_dist > dist[u][used_hops]:
                continue
            
            for v, w in adj[u]:
                # Option 1: do not zero the edge
                if cur_dist + w < dist[v][used_hops]:
                    dist[v][used_hops] = cur_dist + w
                    heapq.heappush(pq, (dist[v][used_hops], v, used_hops))
                
                # Option 2: zero the edge if possible
                if used_hops < k and cur_dist < dist[v][used_hops + 1]:
                    dist[v][used_hops + 1] = cur_dist
                    heapq.heappush(pq, (dist[v][used_hops + 1], v, used_hops + 1))
        
        return min(dist[d])
```

import (
	"container/heap"
	"math"
)

type Item struct {
	node, usedHops int
	dist           int
}

type PriorityQueue []Item

func (pq PriorityQueue) Len() int { return len(pq) }
func (pq PriorityQueue) Less(i, j int) bool { return pq[i].dist < pq[j].dist }
func (pq PriorityQueue) Swap(i, j int) { pq[i], pq[j] = pq[j], pq[i] }
func (pq *PriorityQueue) Push(x interface{}) { *pq = append(*pq, x.(Item)) }
func (pq *PriorityQueue) Pop() interface{} {
	old := *pq
	n := len(old)
	item := old[n-1]
	*pq = old[:n-1]
	return item
}

func shortestPathWithHops(n int, edges [][]int, s int, d int, k int) int {
	adj := make([][][2]int, n)
	for _, e := range edges {
		u, v, w := e[0], e[1], e[2]
		adj[u] = append(adj[u], [2]int{v, w})
		adj[v] = append(adj[v], [2]int{u, w})
	}

	dist := make([][]int, n)
	for i := range dist {
		dist[i] = make([]int, k+1)
		for j := 0; j <= k; j++ {
			dist[i][j] = math.MaxInt64
		}
	}
	dist[s][0] = 0

	pq := &PriorityQueue{}
	heap.Init(pq)
	heap.Push(pq, Item{s, 0, 0})

	for pq.Len() > 0 {
		cur := heap.Pop(pq).(Item)
		u, usedHops, curDist := cur.node, cur.usedHops, cur.dist

			continue
		}

		for _, next := range adj[u] {
			v, w := next[0], next[1]
			// Option 1: do not zero the edge
