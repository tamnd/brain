## Solution

## Problem Understanding

This problem asks us to process a list of strings and split each string using a given separator character. After splitting, we must collect all non-empty pieces into a single result array while preserving their original order.

The input consists of two parts:

- `words`, an array of strings
- `separator`, a single character

For every string in `words`, we split it wherever the separator character appears. The separator itself is removed from the output pieces. Some splits may produce empty strings, and those empty strings must not appear in the final answer.

For example, if we split `"one.two.three"` using `"."`, we get:

```
["one", "two", "three"]
```

However, splitting `"$easy$"` using `"$"` produces:

```
["", "easy", ""]
```

The empty strings at the beginning and end must be ignored, so only `"easy"` remains.

The problem also guarantees that the relative order of all resulting substrings must remain unchanged. This means that substrings produced earlier must appear earlier in the final array.

The constraints are very small:

- At most 100 words
- Each word has length at most 20

Because the total input size is tiny, performance is not a major concern. Even straightforward string processing solutions will easily fit within time limits. Still, we should aim for a clean and efficient implementation.

Several edge cases are important:

- Strings may begin or end with the separator
- Multiple separators may appear consecutively
- A string may contain only separators
- A string may contain no separator at all

A correct solution must properly exclude empty substrings in all these cases.

## Approaches

### Brute Force Approach

The brute force approach manually scans every character in every word. We maintain a temporary string builder that accumulates characters until we encounter the separator.

Whenever the separator is found, we check whether the current accumulated substring is non-empty. If it is, we append it to the answer array. Then we reset the temporary builder and continue scanning.

After finishing the word, we must also append the final accumulated substring if it is non-empty.

This approach is correct because it explicitly constructs every substring between separator characters while skipping empty results. Since every character is examined exactly once, the algorithm faithfully reproduces the intended splitting behavior.

Although this approach is completely acceptable for the given constraints, it is more verbose and requires careful handling of boundary cases such as consecutive separators.

### Optimal Approach

A cleaner solution uses the built-in string splitting operation.

Most programming languages provide a `split()` function that divides a string around occurrences of a separator. After splitting, we simply iterate through the produced pieces and append only the non-empty ones to the result array.

The key observation is that the language runtime already implements efficient string splitting logic. Instead of manually reconstructing substrings character by character, we can delegate that work to the standard library and focus only on filtering empty strings.

This solution is concise, readable, and still linear in the size of the input.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(N) | O(N) | Manually scans characters and builds substrings |
| Optimal | O(N) | O(N) | Uses built-in split and filters empty strings |

Here, `N` represents the total number of characters across all strings.

## Algorithm Walkthrough

1. Initialize an empty result array that will store all valid substrings.
2. Iterate through each word in the `words` array. We process one string at a time so the output order naturally matches the input order.
3. Split the current word using the separator character. This produces a list of substrings. Some of these substrings may be empty because of consecutive separators or separators at the boundaries.
4. Iterate through every substring produced by the split operation.
5. Check whether the substring is non-empty. If its length is greater than zero, append it to the result array. If it is empty, skip it.
6. Continue until all words have been processed.
7. Return the final result array.

### Why it works

The split operation correctly separates each word into all segments between separator characters. Every valid substring appears exactly once in the produced list. By filtering out empty strings, we satisfy the requirement that empty results must not be included. Since words and substrings are processed in their original order, the final output preserves the required ordering.

## Python Solution

```python
from typing import List

class Solution:
    def splitWordsBySeparator(self, words: List[str], separator: str) -> List[str]:
        result = []

        for word in words:
            parts = word.split(separator)

            for part in parts:
                if part:
                    result.append(part)

        return result
```

The solution begins by creating an empty `result` list that will hold all valid substrings.

For each word, we call Python's built-in `split()` method. This divides the string wherever the separator appears. The result may contain empty strings when separators appear consecutively or at the boundaries.

We then iterate through every produced substring. The condition `if part:` checks whether the substring is non-empty. Non-empty substrings are appended to the result list.

Finally, after processing all words, we return the accumulated result.

This implementation directly mirrors the algorithm steps described earlier and handles all edge cases automatically through the combination of `split()` and empty-string filtering.

## Go Solution

```go
package main

import "strings"

func splitWordsBySeparator(words []string, separator byte) []string {
    result := []string{}

    sep := string(separator)

    for _, word := range words {
        parts := strings.Split(word, sep)

        for _, part := range parts {
            if part != "" {
                result = append(result, part)
            }
        }
    }

    return result
}
```

The Go solution follows the same logic as the Python implementation. The main difference is that Go's `strings.Split` expects a string separator rather than a byte, so we convert the separator using `string(separator)`.

Go slices dynamically grow as elements are appended, making them a natural choice for the result container. Empty substrings are filtered using `if part != ""`.

Unlike some languages, Go distinguishes between `nil` slices and empty slices, but both behave correctly here. Returning an empty slice when no valid substrings exist satisfies the problem requirements.

## Edge Cases

One important edge case occurs when separators appear consecutively, such as `"a..b...c"`. A naive implementation might accidentally include empty strings between adjacent separators. In this solution, the filtering condition `if part:` ensures that only non-empty substrings are added to the result.

Another critical case is when the separator appears at the beginning or end of the string. For example, splitting `".abc."` by `"."` produces `["", "abc", ""]`. Without careful filtering, the output would incorrectly contain empty strings. The implementation correctly removes them.

A third edge case happens when a string contains only separators, such as `"|||"`. Splitting produces only empty strings. The correct answer should therefore be an empty array. Since the algorithm ignores all empty substrings, the result remains empty exactly as required.

A final edge case is when a word contains no separator at all. For example, splitting `"hello"` using `"."` produces `["hello"]`. The algorithm naturally keeps the entire string unchanged because it is already a valid non-empty substring.
