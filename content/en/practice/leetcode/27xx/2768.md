## Solution

## Problem Understanding

The problem asks us to analyze an `m x n` grid where some cells are colored black, and the rest are white. A **block** is defined as any `2 x 2` submatrix of the grid. For each such block, we need to count how many black cells it contains. The task is to produce an array of size 5, where each index `i` corresponds to the number of blocks that contain exactly `i` black cells.

The inputs are:

- `m` and `n`, the dimensions of the grid.
- `coordinates`, a list of `[x, y]` pairs, representing black cells. Cells not listed are white by default.

The output is:

- An array of length 5, with the count of blocks containing 0, 1, 2, 3, and 4 black cells, respectively.

Constraints give us an important clue about feasible solutions: while the grid itself can be extremely large (`m` and `n` up to 10^5), the number of black cells is relatively small (`<= 10^4`). This suggests that iterating over the entire grid is too slow, and we need a solution that only processes blocks affected by black cells.

Edge cases include:

- No black cells (`coordinates` is empty), where all blocks should count as 0 black cells.
- Black cells along the edges or corners of the grid, which may not contribute to all potential blocks.
- Maximum grid sizes, ensuring the algorithm scales efficiently.

## Approaches

### Brute Force

A naive approach is to build the entire `m x n` grid, mark black cells, and then iterate over all possible `2 x 2` blocks. For each block, count the number of black cells and update the result array. While correct, this approach is inefficient because the number of blocks is roughly `(m-1)*(n-1)` which can reach `10^10` for large grids - far beyond feasible limits.

### Optimal Approach

The key observation is that only blocks containing black cells need to be explicitly counted. Each black cell affects at most 4 `2 x 2` blocks: one starting at its own coordinates, and one each to its top, left, and top-left (if valid). By iterating over the black cells and incrementing counts in a dictionary keyed by the block's top-left coordinates, we efficiently track how many black cells each block has. Finally, we compute the number of blocks with zero black cells by subtracting the sum of non-zero counts from the total number of blocks `(m-1)*(n-1)`.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m * n) | O(m * n) | Build the full grid and count black cells in every block |
| Optimal | O(B) | O(B) | B = number of black cells; only process affected blocks, use hash map |

## Algorithm Walkthrough

1. Initialize a dictionary `block_count` to keep track of black cells for each block's top-left corner.
2. Iterate through each black cell `[x, y]`:

1. For the black cell, determine the 4 possible top-left corners of blocks it could affect: `[x, y]`, `[x-1, y]`, `[x, y-1]`, `[x-1, y-1]`.
2. For each candidate block, check if its top-left coordinates are valid (within `[0, m-2]` and `[0, n-2]`).
3. Increment the count of black cells in `block_count` for that block.
3. Initialize a result array `arr` of size 5, filled with zeros.
4. Iterate over the `block_count` dictionary:

1. For each block, increment `arr[count]` where `count` is the number of black cells in that block.
5. Calculate blocks with zero black cells as `(m-1)*(n-1) - sum(arr[1:])` and assign it to `arr[0]`.
6. Return `arr`.

**Why it works:** Each black cell affects at most 4 blocks, and we track the exact count per block. By handling only these affected blocks, we avoid processing the entire grid. The dictionary ensures we count each block once.

## Python Solution

```python
from typing import List
from collections import defaultdict

class Solution:
    def countBlackBlocks(self, m: int, n: int, coordinates: List[List[int]]) -> List[int]:
        block_count = defaultdict(int)
        
        for x, y in coordinates:
            for dx in (0, -1):
                for dy in (0, -1):
                    nx, ny = x + dx, y + dy
                    if 0 <= nx < m - 1 and 0 <= ny < n - 1:
                        block_count[(nx, ny)] += 1
        
        arr = [0] * 5
        for count in block_count.values():
            arr[count] += 1
        
        arr[0] = (m - 1) * (n - 1) - sum(arr[1:])
        return arr
```

The implementation iterates only over black cells and calculates their effect on neighboring blocks. It uses a `defaultdict` for counting, ensuring smooth incrementation without key existence checks.

## Go Solution

```go
func countBlackBlocks(m int, n int, coordinates [][]int) []int64 {
    type pair struct{ x, y int }
    blockCount := make(map[pair]int)
    
    for _, coord := range coordinates {
        x, y := coord[0], coord[1]
        for dx := 0; dx <= 1; dx++ {
            for dy := 0; dy <= 1; dy++ {
                nx, ny := x-dx, y-dy
                if nx >= 0 && nx < m-1 && ny >= 0 && ny < n-1 {
                    p := pair{nx, ny}
                    blockCount[p]++
                }
            }
        }
    }
    
    arr := make([]int64, 5)
    for _, count := range blockCount {
        arr[count]++
    }
    arr[0] = int64((m-1)*(n-1)) - (arr[1] + arr[2] + arr[3] + arr[4])
    return arr
}
```

Go-specific notes: `map` is used instead of `defaultdict`. A struct `pair` represents the block coordinates. Slices store the result with type `int64` to handle potentially large counts safely.

## Worked Examples

### Example 1

Input: `m=3, n=3, coordinates=[[0,0]]`

- Black cell `[0,0]` affects blocks `[0,0]` only (top-left is valid). Count = 1.
- Total blocks = 4 (`[0,0],[0,1],[1,0],[1,1]`)
- Block counts: `[1]` for `[0,0]`, others 0
- Result: `[3,1,0,0,0]`

### Example 2

Input: `m=3, n=3, coordinates=[[0,0],[1,1],[0,2]]`

- `[0,0]` → affects `[0,0]`
- `[1,1]` → affects `[0,0]`, `[0,1]`, `[1,0]`, `[1,1]`
- `[0,2]` → affects `[0,1]`
- Counted blocks: `[0,0]:2`, `[0,1]:2`, `[1,0]:1`, `[1,1]:1`
- Result: `[0,2,2,0,0]`

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(B) | Iterate over each black cell and its up to 4 affected blocks; B = number of black cells. |
| Space | O(B) | Dictionary stores at most 4B blocks. |

Because the algorithm only processes blocks influenced by black cells, it is extremely efficient compared to iterating over the entire grid.
