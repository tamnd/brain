# LeetCode 2711 - Difference of Number of Distinct Values on Diagonals

## Problem Understanding

The input is an `m × n` matrix called `grid`. For every cell `(r, c)`, we must examine the diagonal that passes through that cell, specifically the diagonal that runs from the top-left toward the bottom-right.

For a given cell, we split its diagonal into three parts:

- The cells strictly above and to the left of `(r, c)`.
- The cell `(r, c)` itself.
- The cells strictly below and to the right of `(r, c)`.

The problem asks us to count how many **distinct values** appear in the left-above portion and how many distinct values appear in the right-below portion.

If:

- `leftAbove[r][c]` is the number of distinct values before the cell on its diagonal.
- `rightBelow[r][c]` is the number of distinct values after the cell on its diagonal.

Then:

$$answer[r][c] = |leftAbove[r][c] - rightBelow[r][c]|$$

The output is another matrix of the same size where every position contains this absolute difference.

The constraints are:

- `1 <= m, n <= 50`
- `1 <= grid[i][j] <= 50`

The matrix is quite small. At most there are only `2500` cells. This means even moderately inefficient solutions may pass, but we should still look for a clean and efficient approach.

Several edge cases are important:

- A `1 × 1` matrix has no cells before or after the only element on its diagonal, so the answer is `0`.
- Cells on the top row or left column have no left-above cells.
- Cells on the bottom row or right column have no right-below cells.
- A diagonal may contain many repeated values. Since we count **distinct** values, duplicates should contribute only once.

## Approaches

### Brute Force

For every cell `(r, c)`, we can explicitly walk along its diagonal in both directions.

To compute `leftAbove[r][c]`, start from `(r-1, c-1)` and continue moving up-left until leaving the matrix. Insert each value into a set and count the final size.

To compute `rightBelow[r][c]`, start from `(r+1, c+1)` and continue moving down-right until leaving the matrix. Insert each value into another set and count the final size.

The answer for that cell is the absolute difference between the two set sizes.

This method is straightforward and obviously correct because it directly follows the problem definition. However, every cell may scan an entire diagonal, leading to unnecessary repeated work.

### Key Insight

All cells on the same diagonal share the same sequence of values.

Instead of recomputing distinct counts separately for every cell, we can process each diagonal once.

For a diagonal containing values:

$$[v_0, v_1, v_2, \ldots, v_{k-1}]$$

we need, for each position:

- Number of distinct values before it.
- Number of distinct values after it.

This can be obtained with two passes:

- A forward pass maintains a set of values already seen.
- A backward pass maintains a set of values seen from the end.

Since every diagonal is processed only once, we avoid repeatedly traversing the same cells.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(mn × min(m,n)) | O(min(m,n)) | Recomputes diagonal information for every cell |
| Optimal | O(mn) | O(min(m,n)) | Processes each diagonal exactly once |

## Algorithm Walkthrough

### Optimal Diagonal Processing

1. Create an answer matrix of size `m × n` initialized with zeros.
2. Enumerate every diagonal. A diagonal starts either from:

- A cell in the first row.
- A cell in the first column, excluding `(0,0)` because that diagonal was already included.
3. For a chosen starting point, collect all coordinates belonging to that diagonal by repeatedly moving one step down and one step right.
4. Let the diagonal contain `k` cells.
5. Create an array `leftDistinct` of length `k`.
6. Perform a forward traversal of the diagonal:

- Maintain a hash set `seen`.
- Before processing position `i`, store `len(seen)` into `leftDistinct[i]`.
- Insert the current value into `seen`.
7. Create an array `rightDistinct` of length `k`.
8. Perform a backward traversal:

- Clear the hash set.
- Before processing position `i`, store `len(seen)` into `rightDistinct[i]`.
- Insert the current value into `seen`.
9. For every position `i` on the diagonal:

- Compute:

$$|leftDistinct[i] - rightDistinct[i]|$$
- Store the result in the corresponding location of the answer matrix.
10. Continue until every diagonal has been processed.

### Why it works

For a particular diagonal position, the forward pass stores exactly the number of distinct values that appear earlier on the diagonal because the set contains every value encountered before reaching that position. Similarly, the backward pass stores exactly the number of distinct values that appear later on the diagonal because the reverse traversal has already seen all cells to the right and below. Taking the absolute difference therefore matches the problem definition exactly for every cell.

## Python Solution

```python
from typing import List

class Solution:
    def differenceOfDistinctValues(self, grid: List[List[int]]) -> List[List[int]]:
        m, n = len(grid), len(grid[0])
        answer = [[0] * n for _ in range(m)]

        def process_diagonal(start_row: int, start_col: int) -> None:
            cells = []
            r, c = start_row, start_col

            while r < m and c < n:
                cells.append((r, c))
                r += 1
                c += 1

            k = len(cells)

            left_distinct = [0] * k
            seen = set()

            for i, (r, c) in enumerate(cells):
                left_distinct[i] = len(seen)
                seen.add(grid[r][c])

            right_distinct = [0] * k
            seen.clear()

            for i in range(k - 1, -1, -1):
                r, c = cells[i]
                right_distinct[i] = len(seen)
                seen.add(grid[r][c])

            for i, (r, c) in enumerate(cells):
                answer[r][c] = abs(
                    left_distinct[i] - right_distinct[i]
                )

        for col in range(n):
            process_diagonal(0, col)

        for row in range(1, m):
            process_diagonal(row, 0)

        return answer
```

The solution begins by creating the result matrix. A helper function processes one diagonal at a time.

The diagonal coordinates are first collected into a list. This makes it easy to perform both forward and backward traversals.

The forward pass computes how many distinct values appear before each position. The backward pass computes how many distinct values appear after each position.

Finally, the absolute difference between the two counts is written into the answer matrix.

Every diagonal is processed independently, and every cell participates in exactly one forward pass and one backward pass.

## Go Solution

```go
func differenceOfDistinctValues(grid [][]int) [][]int {
	m := len(grid)
	n := len(grid[0])

	answer := make([][]int, m)
	for i := range answer {
		answer[i] = make([]int, n)
	}

	processDiagonal := func(startRow, startCol int) {
		type Cell struct {
			r int
			c int
		}

		var cells []Cell

		r, c := startRow, startCol
		for r < m && c < n {
			cells = append(cells, Cell{r, c})
			r++
			c++
		}

		k := len(cells)

		leftDistinct := make([]int, k)
		seen := make(map[int]struct{})

		for i, cell := range cells {
			leftDistinct[i] = len(seen)
			seen[grid[cell.r][cell.c]] = struct{}{}
		}

		rightDistinct := make([]int, k)
		seen = make(map[int]struct{})

		for i := k - 1; i >= 0; i-- {
			cell := cells[i]
			rightDistinct[i] = len(seen)
			seen[grid[cell.r][cell.c]] = struct{}{}
		}

		for i, cell := range cells {
			diff := leftDistinct[i] - rightDistinct[i]
			if diff < 0 {
				diff = -diff
			}
			answer[cell.r][cell.c] = diff
		}
	}

	for col := 0; col < n; col++ {
		processDiagonal(0, col)
	}

	for row := 1; row < m; row++ {
		processDiagonal(row, 0)
	}

	return answer
}
```

The Go implementation follows exactly the same logic as the Python version. Since Go does not provide a built-in set type, a `map[int]struct{}` is used to represent a hash set. Absolute values are computed manually because the standard library's integer absolute function is not necessary here.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(mn) | Every cell is visited a constant number of times |
| Space | O(min(m,n)) | Largest diagonal length determines auxiliary storage |

Each cell belongs to exactly one diagonal. During processing, each cell is visited once in the forward pass, once in the backward pass, and once when writing the answer. Therefore the total work is linear in the number of cells. The temporary arrays and sets only need enough space for a single diagonal, whose maximum length is `min(m, n)`.

## Edge Cases

### Single Cell Matrix

When the matrix contains only one element, there are no cells before or after it on the diagonal. Both distinct counts are zero, so the answer must be zero. The implementation naturally handles this because both forward and backward passes store `0`.

### Single Row or Single Column

In a single row or single column matrix, every diagonal contains exactly one cell. A common bug is assuming that diagonals always contain multiple elements. The algorithm correctly creates a diagonal of length one, resulting in zero distinct values on both sides.

### Repeated Values Along a Diagonal

A diagonal such as `[1, 1, 1, 1]` should contribute only one distinct value, not four. Using hash sets ensures duplicates are automatically ignored. This is essential because the problem asks for the number of distinct values, not the number of cells.

### Rectangular Matrices

The matrix is not guaranteed to be square. Some diagonals may be significantly shorter than others. By collecting coordinates until either the row or column boundary is reached, the implementation correctly handles any `m × n` dimensions.

### Boundary Cells

Cells on the top row or left column have no left-above portion, while cells on the bottom row or right column have no right-below portion. The forward and backward passes naturally produce a distinct count of zero for the missing side, avoiding special-case logic.
