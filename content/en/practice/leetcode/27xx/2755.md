## Solution

## Problem Understanding

This problem requires implementing a **deep merge** function that takes two JSON-like values, `obj1` and `obj2`, and returns a merged value according to specific rules. The inputs can be primitive values (numbers, strings, booleans, null), arrays, or objects (dictionaries in Python). The merging rules vary depending on the types of the inputs:

If both values are objects, the resulting object contains all keys from both. If a key exists in both, their associated values are recursively merged. If a key exists in only one object, it is added as-is.

If both values are arrays, the resulting array has the length of the longer array. Each index is treated like a key, recursively merging values if both arrays contain a value at that index.

If the two values are of different types, or neither is an array or object, the result is `obj2`.

The input values are guaranteed to be valid JSON and can be deeply nested, so the solution must handle recursion carefully. Edge cases include arrays and objects of unequal length, `null` values, primitive types overriding objects, and empty arrays or objects. The constraints also indicate that input can be large (up to roughly half a million characters), so the solution must be efficient in both time and space.

## Approaches

The **brute-force approach** is straightforward recursion. For any two inputs, check their types. If both are objects, iterate over all keys and recursively merge the corresponding values. If both are arrays, iterate over indices up to the length of the longer array and recursively merge values. Otherwise, return `obj2`. While correct, this approach may perform unnecessary copying and recursion and could hit stack limits for very deep inputs.

The **optimal approach** still uses recursion but avoids redundant data copying by constructing new objects and arrays as needed. It handles type mismatches efficiently and treats indices and keys uniformly. The main insight is that deep merge can be abstracted to a recursive function that handles objects and arrays in a unified manner, and primitive values are simply replaced. This ensures correctness while maintaining clarity and efficiency.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n + m) | O(d) | Recursively traverses all elements and keys; `d` is recursion depth |
| Optimal | O(n + m) | O(d) | Same complexity, but avoids unnecessary copying and handles type mismatches cleanly |

## Algorithm Walkthrough

1. Define a recursive function `deepMerge(obj1, obj2)`. This function will merge any two JSON-compatible values.
2. Check the types of `obj1` and `obj2`. If both are dictionaries (objects), initialize a new dictionary to hold the result. Iterate over all unique keys from both objects.
3. For each key present in both objects, recursively merge their values. For keys present in only one object, copy the value directly into the result dictionary.
4. If both `obj1` and `obj2` are lists (arrays), determine the length of the longer list. Initialize a new list of that length. Iterate through each index, recursively merging values if both arrays have a value at that index; otherwise, take the existing value.
5. If `obj1` and `obj2` are of different types or neither is an object or array, return `obj2`.
6. Return the resulting merged object or array.

**Why it works:**

The algorithm works because at each level of recursion, it enforces the deep merge rules specified in the problem. Objects are merged key by key, arrays index by index, and primitive values are overridden by `obj2`. Since recursion handles all nested structures, the merge is applied consistently at all depths.

## Python Solution

```python
from typing import Any

class Solution:
    def deepMerge(self, obj1: Any, obj2: Any) -> Any:
        if isinstance(obj1, dict) and isinstance(obj2, dict):
            result = {}
            keys = set(obj1.keys()).union(obj2.keys())
            for key in keys:
                if key in obj1 and key in obj2:
                    result[key] = self.deepMerge(obj1[key], obj2[key])
                elif key in obj1:
                    result[key] = obj1[key]
                else:
                    result[key] = obj2[key]
            return result
        elif isinstance(obj1, list) and isinstance(obj2, list):
            max_len = max(len(obj1), len(obj2))
            result = []
            for i in range(max_len):
                if i < len(obj1) and i < len(obj2):
                    result.append(self.deepMerge(obj1[i], obj2[i]))
                elif i < len(obj1):
                    result.append(obj1[i])
                else:
                    result.append(obj2[i])
            return result
        else:
            return obj2
```

The code first checks if both inputs are dictionaries. If so, it merges them key by key. For arrays, it merges index by index, using the longer array length. For all other types or type mismatches, it simply returns `obj2`. This mirrors the algorithm precisely.

## Go Solution

```go
package main

func deepMerge(obj1, obj2 any) any {
	switch v1 := obj1.(type) {
	case map[string]any:
		if v2, ok := obj2.(map[string]any); ok {
			result := make(map[string]any)
			for k := range v1 {
				if val, exists := v2[k]; exists {
					result[k] = deepMerge(v1[k], val)
				} else {
					result[k] = v1[k]
				}
			}
			for k, val := range v2 {
				if _, exists := v1[k]; !exists {
					result[k] = val
				}
			}
			return result
		}
	case []any:
		if v2, ok := obj2.([]any); ok {
			maxLen := len(v1)
			if len(v2) > maxLen {
				maxLen = len(v2)
			}
			result := make([]any, maxLen)
			for i := 0; i < maxLen; i++ {
				if i < len(v1) && i < len(v2) {
					result[i] = deepMerge(v1[i], v2[i])
				} else if i < len(v1) {
					result[i] = v1[i]
				} else {
					result[i] = v2[i]
				}
			}
			return result
		}
	}
	return obj2
}
```

In Go, type assertions are used to determine if values are `map[string]any` (objects) or `[]any` (arrays). Nil handling is implicit: missing keys or indices are managed using bounds checks. The algorithm otherwise mirrors the Python version.

## Worked Examples

**Example 1:**

Input: `obj1 = {"a": 1, "c": 3}`, `obj2 = {"a": 2, "b": 2}`

Step by step:

- Merge dictionaries. Keys: {"a", "b", "c"}
- Key "a": both exist → `deepMerge(1, 2)` = 2
- Key "b": only in `obj2` → 2
- Key "c": only in `obj1` → 3

Result: `{"a": 2, "c": 3, "b": 2}`

**Example 2:**

Input: `obj1 = [{}, 2, 3]`, `obj2 = [[], 5]`

Step by step:

- Merge arrays of length 3
- Index 0: `{}` vs `[]` → different types → take `[]`
- Index 1: `2` vs `5` → primitive → take `5`
- Index 2: `3` vs missing → take `3`

Result: `[[], 5, 3]`

**Example 3:**

Input: `obj1 = {"a": 1, "b": {"c": [1, [2, 7], 5], "d": 2}}`, `obj2 = {"a": 1, "b": {"c": [6, [6], [9]], "e": 3}}`

Step by step:

- Merge top-level objects.
- Key "a": both 1 → 1
- Key "b": merge nested objects

- Key "c": merge arrays

- Index 0: 1 vs 6 → 6
- Index 1: [2, 7] vs [6] → merge arrays → [6, 7]
- Index 2: 5 vs [9] → merge → [9]
- Key "d": only in obj1 → 2
- Key "e": only in obj2 → 3

Result: `{"a": 1, "b": {"c": [6, [6, 7], [9]], "d": 2, "e": 3}}`

**Example 4:**

Input: `obj1 = true, obj2 = null` → different types → return `null`
