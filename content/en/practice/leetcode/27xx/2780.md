# LeetCode 2780 - Minimum Index of a Valid Split

## Problem Understanding

The problem gives us an integer array `nums` that is guaranteed to contain exactly one dominant element. An element is considered dominant if it appears more than half the time in the array.

We need to split the array into two non-empty parts at some index `i`:

- Left subarray: `nums[0...i]`
- Right subarray: `nums[i+1...n-1]`

The split is only valid if both subarrays have the same dominant element.

The goal is to return the smallest valid split index. If no such split exists, we return `-1`.

The important detail is that the original array already has exactly one dominant element. This immediately tells us something very powerful:

- Any valid split must use the same dominant element as the original array.
- No other value can suddenly become dominant in both halves simultaneously.

This observation dramatically simplifies the problem because we only need to track the dominant element of the full array.

The array length can be as large as `10^5`, so any algorithm worse than linear or near-linear time will likely be too slow. A naive approach that repeatedly scans subarrays would become inefficient very quickly.

There are several important edge cases to consider:

- The dominant element may only barely exceed half the array size.
- A valid split might exist very late in the array.
- One side of the split may contain only one element.
- Some arrays may not allow any valid split even though the full array has a dominant element.
- The dominant element count must satisfy the strict condition `count * 2 > length`, not `>=`.

Understanding the strict inequality is essential because many incorrect implementations accidentally allow equality.

## Approaches

### Brute Force Approach

The brute-force solution tries every possible split index.

For each index `i`:

1. Construct the left and right subarrays.
2. Count frequencies in the left subarray.
3. Count frequencies in the right subarray.
4. Determine whether both sides have the same dominant element.
5. Return the first valid split.

This works because it directly checks every possible split according to the problem definition.

However, it is inefficient. There are `O(n)` possible split positions, and for each split we may spend `O(n)` time counting frequencies. This leads to `O(n^2)` time complexity, which is too slow for `n = 10^5`.

### Optimal Approach

The key insight is that the dominant element of any valid split must be the same as the dominant element of the entire array.

Why is this true?

Suppose the full array has dominant element `x`. Since `x` appears more than half the time overall, no other value can dominate both halves simultaneously.

Therefore, we only need to:

1. Find the dominant element of the full array.
2. Count how many times it appears overall.
3. Iterate through possible split positions while maintaining:

- Count of dominant element in the left side
- Count of dominant element in the right side
4. Check whether the dominant condition holds on both sides.

This reduces the problem to a simple linear scan.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Recomputes frequencies for every split |
| Optimal | O(n) | O(1) | Tracks only the global dominant element |

## Algorithm Walkthrough

### Step 1: Find the dominant element

Use the Boyer-Moore Voting Algorithm to identify the dominant element in linear time and constant space.

The algorithm works because the dominant element appears more than half the time, so it survives all cancellations.

### Step 2: Count total occurrences

After finding the dominant element, count how many times it appears in the entire array.

This value will help determine how many copies remain in the right subarray during traversal.

### Step 3: Iterate through split positions

Traverse indices from `0` to `n - 2`.

We stop at `n - 2` because both subarrays must be non-empty.

Maintain:

- `left_count`, the number of dominant elements seen so far
- `right_count = total_count - left_count`

### Step 4: Compute subarray lengths

For split index `i`:

- Left length = `i + 1`
- Right length = `n - i - 1`

### Step 5: Check dominance conditions

The dominant element must satisfy:

- `left_count * 2 > left_length`
- `right_count * 2 > right_length`

If both are true, return the current index immediately because we want the minimum valid split.

### Step 6: Return -1 if no split exists

If traversal finishes without finding a valid split, return `-1`.

### Why it works

The correctness relies on the fact that the array has exactly one dominant element. Any valid split must preserve that same dominant value in both halves.

By maintaining running counts of this element in the left and right subarrays, we can efficiently verify the dominance condition at every split position without recomputing frequencies from scratch.

Since every split is checked exactly once, and dominance is verified mathematically using counts, the algorithm always returns the smallest valid split if one exists.

## Python Solution

```python
from typing import List

class Solution:
    def minimumIndex(self, nums: List[int]) -> int:
        # Step 1: Find dominant element using Boyer-Moore
        candidate = None
        balance = 0
        
        for num in nums:
            if balance == 0:
                candidate = num
                balance = 1
            elif num == candidate:
                balance += 1
            else:
                balance -= 1
        
        # Step 2: Count total occurrences
        total_count = nums.count(candidate)
        
        # Step 3: Check each split
        left_count = 0
        n = len(nums)
        
        for i in range(n - 1):
            if nums[i] == candidate:
                left_count += 1
            
            right_count = total_count - left_count
            
            left_length = i + 1
            right_length = n - i - 1
            
            left_dominant = left_count * 2 > left_length
            right_dominant = right_count * 2 > right_length
            
            if left_dominant and right_dominant:
                return i
        
        return -1
```

The implementation begins with the Boyer-Moore Voting Algorithm. The `candidate` variable stores the current possible dominant element, while `balance` tracks relative frequency.

Whenever the balance reaches zero, we reset the candidate because previous values have effectively canceled out.

After identifying the candidate, we count its total occurrences in the array.

Next, we iterate through all valid split positions while maintaining `left_count`, which tracks how many times the dominant element appears in the left subarray.

The right-side count is computed dynamically using:

```python
right_count = total_count - left_count
```

For every split, we verify the dominant condition independently for both halves using strict majority checks.

The first valid index is returned immediately because the problem asks for the minimum valid split.

## Go Solution

```go
func minimumIndex(nums []int) int {
    // Step 1: Find dominant element using Boyer-Moore
    candidate := 0
    balance := 0

    for _, num := range nums {
        if balance == 0 {
            candidate = num
            balance = 1
        } else if num == candidate {
            balance++
        } else {
            balance--
        }
    }

    // Step 2: Count total occurrences
    totalCount := 0
    for _, num := range nums {
        if num == candidate {
            totalCount++
        }
    }

    // Step 3: Check each split
    leftCount := 0
    n := len(nums)

    for i := 0; i < n-1; i++ {
        if nums[i] == candidate {
            leftCount++
        }

        rightCount := totalCount - leftCount

        leftLength := i + 1
        rightLength := n - i - 1

        leftDominant := leftCount*2 > leftLength
        rightDominant := rightCount*2 > rightLength

        if leftDominant && rightDominant {
            return i
        }
    }

    return -1
}
```

The Go implementation follows exactly the same logic as the Python version.

One difference is that Go does not provide a built-in frequency count method like Python's `count`, so we explicitly loop through the array to count occurrences.

Go integers are sufficient because the constraints are small enough that overflow is not a concern for multiplication by `2`.

Slices are used naturally for array handling, and all operations remain constant extra space.

## Edge Cases

### Very Small Arrays

Arrays of length `2` are important because there is only one possible split.

For example:

```
[1,1]
```

Splitting at index `0` gives `[1]` and `[1]`, both of which have the same dominant element.

The implementation correctly handles this because the loop checks all split positions from `0` to `n - 2`.

### Dominance by a Narrow Margin

A dominant element may appear only slightly more than half the time.

For example:

```
[1,2,1,2,1,1]
```

The dominant element `1` appears four times in six positions.

Incorrect implementations sometimes use `>=` instead of `>` when checking dominance. This would incorrectly classify tied frequencies as dominant.

The solution uses:

```
count * 2 > length
```

which precisely matches the problem definition.

### No Valid Split Exists

Even though the overall array has a dominant element, no split may preserve dominance on both sides.

For example:

```
[3,3,3,3,7,2,2]
```

The left side easily maintains dominance, but the right side never contains enough `3`s.

The implementation checks both halves independently at every split position and correctly returns `-1` if no valid split is found.
