## Solution

## Problem Understanding

The problem is asking us to determine the status of flight tickets for passengers based on the order in which they booked and the capacity of each flight. We are given two tables: `Flights`, which contains each flight's `flight_id` and its `capacity`, and `Passengers`, which contains each passenger's `passenger_id`, the `flight_id` they booked, and the `booking_time`. Each passenger can only book one ticket per flight, and each flight has a limited number of seats.

The expected output is a table showing each `passenger_id` and the corresponding ticket `Status`, which can be either `Confirmed` if the passenger successfully secured a seat or `Waitlist` if the flight was already full when they booked. The output must be sorted by `passenger_id` in ascending order.

Constraints and guarantees provide important information: `passenger_id` and `booking_time` are distinct, meaning we do not have to handle ties in booking time. Each `flight_id` is also unique in the `Flights` table, which allows us to look up flight capacities efficiently.

Important edge cases include flights with capacity exactly equal to the number of passengers, flights with zero passengers, and passengers who booked at the exact boundary of a flight reaching full capacity.

## Approaches

A brute-force approach would involve iterating over each flight and checking for each passenger whether they are within the flight's capacity based on booking time. This approach would work but could be slow if the number of passengers and flights is large, as it would involve repeated filtering and sorting for each flight.

The key insight for an optimal solution is to leverage SQL window functions. By partitioning passengers by `flight_id` and ordering by `booking_time`, we can assign a sequence number to each booking. Comparing this sequence number with the flight capacity immediately tells us whether a passenger is confirmed or waitlisted. This avoids repeated scans and allows the database engine to optimize the operation efficiently.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(P * log P) | O(P) | Sort passengers per flight and compare with capacity for each flight, P = total passengers |
| Optimal | O(P log P) | O(P) | Use window function to assign row numbers within each flight and compare with capacity |

## Algorithm Walkthrough

1. Join the `Passengers` table with the `Flights` table on `flight_id` to have access to both the passenger's booking time and the flight capacity in one table. This allows us to make a capacity check for each passenger.
2. Use a window function `ROW_NUMBER()` partitioned by `flight_id` and ordered by `booking_time`. This assigns each passenger a sequential number representing their order of booking on the flight.
3. Compare the row number of each passenger with the `capacity` of the flight. If the row number is less than or equal to the capacity, mark the passenger as `Confirmed`. Otherwise, mark them as `Waitlist`.
4. Select `passenger_id` and the computed `Status` and order the results by `passenger_id` in ascending order.

**Why it works:**

The `ROW_NUMBER()` function guarantees that passengers are numbered in strict booking order. By comparing this number with the flight capacity, we correctly determine whether each passenger's booking fits within the available seats. This logic handles all edge cases, including flights filled exactly to capacity and passengers who arrive after the flight is full.

## Python Solution

```python
import pandas as pd

def flight_ticket_status(flights: pd.DataFrame, passengers: pd.DataFrame) -> pd.DataFrame:
    # Merge passengers with flight capacity
    merged = passengers.merge(flights, on='flight_id')
    
    # Sort passengers by flight_id and booking_time
    merged.sort_values(by=['flight_id', 'booking_time'], inplace=True)
    
    # Assign sequential booking numbers per flight
    merged['row_number'] = merged.groupby('flight_id').cumcount() + 1
    
    # Determine status
    merged['Status'] = merged.apply(
        lambda x: 'Confirmed' if x['row_number'] <= x['capacity'] else 'Waitlist', axis=1
    )
    
    # Return only required columns ordered by passenger_id
    result = merged[['passenger_id', 'Status']].sort_values(by='passenger_id')
    
    return result
```

This Python solution first merges the passengers with flight capacities so we can compare bookings directly with available seats. Sorting by flight and booking time ensures we maintain the correct booking order. Using `cumcount()` provides a sequential row number per flight, which is then compared against the flight capacity to determine the ticket status. Finally, we select the required columns and order by `passenger_id` as requested.

## Go Solution

```go
package main

import (
    "sort"
    "time"
)

type Flight struct {
    FlightID int
    Capacity int
}

type Passenger struct {
    PassengerID int
    FlightID    int
    BookingTime time.Time
}

type PassengerStatus struct {
    PassengerID int
    Status      string
}

func FlightTicketStatus(flights []Flight, passengers []Passenger) []PassengerStatus {
    // Build a map from flight_id to capacity
    flightCap := make(map[int]int)
    for _, f := range flights {
        flightCap[f.FlightID] = f.Capacity
    }

    // Group passengers by flight
    flightPassengers := make(map[int][]Passenger)
    for _, p := range passengers {
        flightPassengers[p.FlightID] = append(flightPassengers[p.FlightID], p)
    }

    var result []PassengerStatus

    for flightID, ps := range flightPassengers {
        // Sort passengers by booking_time
        sort.Slice(ps, func(i, j int) bool {
            return ps[i].BookingTime.Before(ps[j].BookingTime)
        })

        capacity := flightCap[flightID]
        for i, p := range ps {
            status := "Waitlist"
            if i < capacity {
                status = "Confirmed"
            }
            result = append(result, PassengerStatus{
                PassengerID: p.PassengerID,
                Status:      status,
            })
        }
    }

    // Sort final result by passenger_id
    sort.Slice(result, func(i, j int) bool {
        return result[i].PassengerID < result[j].PassengerID
    })

    return result
}
```

The Go implementation handles grouping passengers by flight using a map, then sorts them by booking time. It determines the status by comparing the index to the flight capacity, similar to the Python solution. The final result is sorted by `PassengerID`.
