# LeetCode 2739 - Total Distance Traveled

## Problem Understanding

This problem models a truck with two fuel tanks:

- `mainTank` contains the fuel that is directly used by the engine.
- `additionalTank` contains reserve fuel that can be transferred into the main tank under specific conditions.

The truck travels `10 km` for every `1 liter` of fuel consumed.

The special rule is that every time exactly `5 liters` are consumed from the main tank, the truck immediately transfers `1 liter` from the additional tank into the main tank, provided the additional tank still contains at least one liter.

The transfer is not continuous. It only occurs after each completed block of 5 liters consumed from the main tank.

The goal is to determine the maximum total distance the truck can travel before all usable fuel is exhausted.

The input consists of two integers:

- `mainTank`, the amount of fuel initially available in the main tank.
- `additionalTank`, the amount of fuel available in the reserve tank.

The output is a single integer representing the maximum distance traveled in kilometers.

The constraints are very small:

- `1 <= mainTank <= 100`
- `1 <= additionalTank <= 100`

Because the inputs are tiny, even a direct simulation is easily fast enough. However, understanding the transfer process correctly is important.

Some important edge cases include:

- The main tank starts with less than 5 liters, so no transfer can ever occur.
- The additional tank runs out before all possible transfers happen.
- The main tank contains enough fuel to trigger multiple transfers.
- The final fuel consumption ends exactly at a transfer boundary.

## Approaches

### Brute Force Simulation

The most straightforward solution is to simulate the process exactly as described.

Whenever the main tank contains at least 5 liters and the additional tank still has fuel available:

1. Consume 5 liters.
2. Add the corresponding distance.
3. Transfer 1 liter from the additional tank into the main tank.

Eventually, either:

- The main tank contains fewer than 5 liters, or
- The additional tank becomes empty.

At that point, all remaining fuel in the main tank can be consumed directly.

This approach follows the problem statement literally, making correctness easy to verify.

### Key Insight

Each transfer effectively gives the truck one extra liter of usable fuel.

A transfer occurs only after consuming a block of 5 liters from the main tank.

Therefore, instead of simulating fuel consumption one liter at a time, we can repeatedly process fuel in chunks of 5 liters.

Every successful transfer:

- Consumes 5 liters from the main tank.
- Adds 1 liter back from the additional tank.
- Produces 50 km of travel.

After no more transfers are possible, the remaining fuel in the main tank contributes directly to the final distance.

This yields a simple and efficient simulation.

| Optimal | O(min(mainTank, additionalTank)) | O(1) | Process transfers in 5-liter chunks |

## Algorithm Walkthrough

### Optimal Algorithm

1. Initialize a variable `distance` to `0`.
2. While the main tank contains at least `5` liters and the additional tank still contains fuel:

- Consume `5` liters from the main tank.
- Add `50` kilometers to the distance because 5 liters produce 50 km.
- Transfer `1` liter from the additional tank to the main tank.
- Decrease `additionalTank` by `1`.
- Increase the main tank by `1` because of the transfer.
3. When the loop ends, either the main tank has fewer than 5 liters remaining or the additional tank is empty.
4. All remaining fuel in the main tank can still be consumed normally.
5. Add `mainTank * 10` kilometers to the distance.
6. Return the final distance.

### Why it works

The algorithm exactly mirrors the rule given in the problem statement. Every time 5 liters are consumed from the main tank, it immediately performs the required transfer if reserve fuel exists. The loop continues until no further transfers are possible. After that point, the remaining fuel contributes directly to the traveled distance. Since every fuel transfer and every liter consumed are accounted for exactly once, the computed distance is the maximum possible distance the truck can travel.

## Python Solution

```python
class Solution:
    def distanceTraveled(self, mainTank: int, additionalTank: int) -> int:
        distance = 0

        while mainTank >= 5 and additionalTank > 0:
            mainTank -= 5
            distance += 50

            mainTank += 1
            additionalTank -= 1

        distance += mainTank * 10
        return distance
```

The implementation starts by tracking the accumulated distance.

The `while` loop handles every transfer event. Each iteration represents consuming exactly 5 liters from the main tank. This contributes 50 kilometers to the answer. If reserve fuel is available, one liter is transferred into the main tank and removed from the additional tank.

Once no further transfer can occur, the loop terminates. Any remaining fuel in the main tank is converted directly into distance using the truck's mileage of 10 km per liter.

The returned value is the total distance traveled.

## Go Solution

```go
func distanceTraveled(mainTank int, additionalTank int) int {
    distance := 0

    for mainTank >= 5 && additionalTank > 0 {
        mainTank -= 5
        distance += 50

        mainTank++
        additionalTank--
    }

    distance += mainTank * 10
    return distance
}
```

The Go implementation follows exactly the same logic as the Python version. Since all values are small integers, there are no concerns about integer overflow. No additional data structures are required, so the solution uses only constant extra memory.

## Edge Cases

### Main Tank Starts Below Five Liters

When `mainTank < 5`, the truck can never consume a full block of five liters, so no transfer is ever triggered. A buggy implementation might incorrectly transfer fuel anyway. The solution avoids this because the loop only executes when `mainTank >= 5`.

### Additional Tank Runs Out Early

Suppose `mainTank = 100` and `additionalTank = 1`. After the first transfer, the reserve tank becomes empty. No further transfers are allowed, even though the main tank still contains plenty of fuel. The loop condition explicitly checks `additionalTank > 0`, ensuring transfers stop immediately when the reserve tank is exhausted.

### Exact Transfer Boundary

Consider `mainTank = 5` and `additionalTank = 1`. After consuming exactly 5 liters, a transfer occurs and the main tank receives 1 liter. That final liter must still contribute to the traveled distance. The implementation correctly adds the transferred fuel back into the main tank before the loop terminates, ensuring the last 10 kilometers are counted.

### Large Reserve Fuel with Small Main Tank

For inputs such as `mainTank = 5` and `additionalTank = 100`, only one transfer can ever occur. The truck cannot repeatedly transfer fuel without first consuming another block of 5 liters. The simulation naturally enforces this rule because transfers only happen after successful consumption of 5 liters from the main tank.
