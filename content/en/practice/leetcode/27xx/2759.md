# LeetCode 2759 - Convert JSON String to Object

## Problem Understanding

This problem asks us to implement our own JSON parser without using the built in `JSON.parse` function. We are given a valid JSON string and must convert it into the corresponding JavaScript style object representation.

The input is a string named `str`. This string can represent any valid JSON value. That includes primitive values such as numbers, booleans, strings, and `null`, as well as composite structures such as arrays and objects.

The output should be the fully parsed structure represented by the JSON text. For example:

- `"true"` becomes the boolean `true`
- `"[1,2,3]"` becomes an array `[1,2,3]`
- `'{"a":2}'` becomes an object `{"a":2}`

The important detail is that the parser must recursively interpret nested structures. Arrays may contain objects, objects may contain arrays, and values can be nested arbitrarily deep.

The constraints tell us that:

- The JSON string is always valid
- The string length can be as large as `10^5`
- Escape sequences and invisible characters are not present

These guarantees simplify the implementation significantly. Since the input is always valid, we do not need extensive error handling. Since there are no escaped quotes inside strings, parsing string boundaries becomes much easier.

A naive implementation could still run into problems if it repeatedly slices large substrings or rescans characters many times. With an input size up to `100000`, we need a linear time parser that processes each character only once or a constant number of times.

Several edge cases are important:

- A single primitive value such as `"true"` or `"5"`
- Empty arrays like `"[]"`
- Empty objects like `"{}"`
- Deeply nested objects and arrays
- Strings that contain words like `"false"` which must remain strings, not booleans

The problem guarantees all keys are valid JSON strings and every JSON structure is syntactically correct.

## Approaches

### Brute Force Approach

A brute force solution would attempt to recursively split the string into smaller substrings every time we encounter commas, braces, or brackets.

For example, to parse an array, we could search for commas at the top level, split the string into pieces, and recursively parse each piece. Similarly, for objects we could repeatedly split key value pairs.

This approach works conceptually because JSON is recursively defined. Arrays contain values, objects contain key value pairs, and primitive values terminate recursion.

However, repeatedly creating substrings is expensive. Each substring operation may take linear time because a new string must be allocated. In deeply nested structures or large arrays, the total complexity can become quadratic.

The brute force approach is therefore too inefficient for strings of length up to `10^5`.

### Optimal Approach

The key observation is that JSON can be parsed sequentially from left to right using a pointer index.

Instead of repeatedly splitting strings, we maintain a single global index that advances as we consume characters. Every parsing function reads characters directly from the original string and updates the index appropriately.

This works well because JSON grammar is naturally recursive:

- If we see `{`, we parse an object
- If we see `[`, we parse an array
- If we see `"`, we parse a string
- Otherwise we parse a primitive value

By processing characters only once, we achieve linear time complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n²) | Repeated substring creation and rescanning |
| Optimal | O(n) | O(n) | Single pass recursive descent parser |

## Algorithm Walkthrough

1. Initialize a global index pointer at position `0`.
2. Create a recursive parsing function called `parseValue`. This function determines what type of JSON value starts at the current index.
3. If the current character is `{`, begin parsing an object.

- Advance past `{`
- Repeatedly parse key value pairs
- Parse the key as a string
- Skip the colon
- Parse the value recursively
- Store the pair in a hash map
- Continue until `}` is reached
4. If the current character is `[`, begin parsing an array.

- Advance past `[`
- Repeatedly parse values recursively
- Append each parsed value to a list
- Continue until `]` is reached
5. If the current character is `"`, parse a string.

- Skip the opening quote
- Read characters until the next quote
- Return the accumulated string
6. Otherwise parse a primitive value.

- Read characters until a delimiter such as `,`, `]`, or `}`
- Convert the token appropriately:

- `"true"` becomes `True`
- `"false"` becomes `False`
- `"null"` becomes `None`
- Numeric text becomes an integer or float
7. Continue recursion until the entire structure is parsed.

### Why it works

The algorithm works because JSON grammar is recursive and deterministic. Every JSON value starts with a character that uniquely identifies its type. By always consuming exactly the characters belonging to the current value and recursively parsing nested structures, the parser maintains the invariant that the index always points to the next unread character. Since every character is processed in order, the final parsed structure exactly matches the JSON representation.

## Python Solution

```python
from typing import Any

class Solution:
    def jsonParse(self, str: str) -> Any:
        index = 0
        n = len(str)

        def parse_value():
            nonlocal index

            if str[index] == '{':
                return parse_object()

            if str[index] == '[':
                return parse_array()

            if str[index] == '"':
                return parse_string()

            return parse_primitive()

        def parse_object():
            nonlocal index

            obj = {}
            index += 1  # skip '{'

            if str[index] == '}':
                index += 1
                return obj

            while True:
                key = parse_string()

                index += 1  # skip ':'

                value = parse_value()
                obj[key] = value

                if str[index] == '}':
                    index += 1
                    break

                index += 1  # skip ','

            return obj

        def parse_array():
            nonlocal index

            arr = []
            index += 1  # skip '['

            if str[index] == ']':
                index += 1
                return arr

            while True:
                arr.append(parse_value())

                if str[index] == ']':
                    index += 1
                    break

                index += 1  # skip ','

            return arr

        def parse_string():
            nonlocal index

            index += 1  # skip opening quote
            start = index

            while str[index] != '"':
                index += 1

            result = str[start:index]
            index += 1  # skip closing quote

            return result

        def parse_primitive():
            nonlocal index

            start = index

            while (
                index < n
                and str[index] not in [',', ']', '}']
            ):
                index += 1

            token = str[start:index]

            if token == "true":
                return True

            if token == "false":
                return False

            if token == "null":
                return None

            if '.' in token:
                return float(token)

            return int(token)

        return parse_value()
```

The implementation uses recursive descent parsing. The `parse_value` function acts as the dispatcher that decides which parser to invoke based on the current character.

The `parse_object` function constructs a dictionary by repeatedly parsing string keys and recursively parsing their associated values. Since the input is guaranteed valid, we can safely skip expected separators like `:` and `,`.

The `parse_array` function repeatedly parses elements until the closing bracket is reached.

The `parse_string` function extracts characters between quotation marks. The problem guarantees there are no escaped quotes, which simplifies the implementation.

The `parse_primitive` function handles booleans, null values, integers, and floating point numbers by reading characters until a structural delimiter is encountered.

The global `index` variable ensures the parser always knows its current position without repeatedly slicing substrings.

## Go Solution

```go
package main

import (
	"strconv"
)

type Solution struct{}

func (s *Solution) jsonParse(str string) interface{} {
	index := 0
	n := len(str)

	var parseValue func() interface{}
	var parseObject func() map[string]interface{}
	var parseArray func() []interface{}
	var parseString func() string
	var parsePrimitive func() interface{}

	parseValue = func() interface{} {
		if str[index] == '{' {
			return parseObject()
		}

		if str[index] == '[' {
			return parseArray()
		}

		if str[index] == '"' {
			return parseString()
		}

		return parsePrimitive()
	}

	parseObject = func() map[string]interface{} {
		obj := map[string]interface{}{}

		index++ // skip '{'

		if str[index] == '}' {
			index++
			return obj
		}

		for {
			key := parseString()

			index++ // skip ':'

			value := parseValue()
			obj[key] = value

			if str[index] == '}' {
				index++
				break
			}

			index++ // skip ','
		}

		return obj
	}

	parseArray = func() []interface{} {
		arr := []interface{}{}

		index++ // skip '['

		if str[index] == ']' {
			index++
			return arr
		}

		for {
			arr = append(arr, parseValue())

			if str[index] == ']' {
				index++
				break
			}

			index++ // skip ','
		}

		return arr
	}

	parseString = func() string {
		index++ // skip opening quote
		start := index

		for str[index] != '"' {
			index++
		}

		result := str[start:index]
		index++ // skip closing quote

		return result
	}

	parsePrimitive = func() interface{} {
		start := index

		for index < n &&
			str[index] != ',' &&
			str[index] != ']' &&
			str[index] != '}' {
			index++
		}

		token := str[start:index]

		if token == "true" {
			return true
		}

		if token == "false" {
			return false
		}

		if token == "null" {
			return nil
		}

		if value, err := strconv.Atoi(token); err == nil {
			return value
		}

		value, _ := strconv.ParseFloat(token, 64)
		return value
	}

	return parseValue()
}
```

The Go version follows the same recursive descent structure as the Python solution. Since Go is statically typed, the parser uses `interface{}` to represent arbitrary JSON values.

Objects are stored as `map[string]interface{}` and arrays are stored as `[]interface{}`.

Primitive parsing differs slightly because Go requires explicit numeric conversion using the `strconv` package. The implementation first attempts integer parsing, then falls back to floating point parsing.

Go also distinguishes `nil` from empty collections, so JSON `null` correctly maps to `nil`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each character is processed at most once |
| Space | O(n) | Output structure and recursion stack may require linear space |

The parser advances through the string sequentially using a single index pointer. No repeated rescanning or substring decomposition occurs, so the runtime is linear in the size of the input.

The space complexity comes primarily from storing the parsed output. Deeply nested structures also contribute recursion stack usage proportional to nesting depth.

## Edge Cases

One important edge case is empty containers such as `{}` and `[]`. A naive parser might incorrectly expect at least one element or key value pair and fail immediately after opening the structure. This implementation explicitly checks whether the next character is the closing delimiter immediately after consuming the opening brace or bracket.

Another important edge case is strings that resemble primitive values. For example, `"false"` is a string, not a boolean. The parser distinguishes this correctly because quoted values are always handled by `parse_string` before primitive parsing logic is considered.

Deep nesting is another common source of bugs. Inputs such as `{"a":[{"b":[1]}]}` require recursive parsing to correctly preserve structure boundaries. Since every parser function advances the shared index consistently and delegates nested structures recursively, deeply nested JSON is handled naturally and correctly.

A final edge case is numeric parsing. Values like `12` and `12.5` must become different numeric types. The implementation checks for a decimal point to distinguish integers from floating point numbers in Python, while the Go version attempts integer parsing first before falling back to float parsing.
