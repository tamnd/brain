## Solution

## Problem Understanding

The problem asks us to count the number of substrings in a given string `s` that contain **no repeating characters**. A substring is defined as a contiguous sequence of characters, and a "special" substring is one where each character appears **at most once**. The input is a string of lowercase English letters with length up to $10^5$, which is large enough to make naive $O(n^3)$ solutions impractical. The output is a single integer representing the total count of these special substrings.

The constraints indicate that we must handle large strings efficiently, and that all characters are lowercase, which allows us to use arrays or hash maps of size 26 to track character counts efficiently. Important edge cases include strings with all identical characters, strings with all unique characters, and the minimum length strings of size 1 or 2. The problem guarantees that the string is non-empty.

## Approaches

The **brute-force approach** would be to generate all possible substrings and check each substring for duplicates. This is guaranteed to work correctly but is too slow because generating all substrings takes $O(n^2)$ and checking each for uniqueness adds up to $O(n^3)$ in the worst case.

The **optimal approach** relies on the **sliding window technique**. The key observation is that substrings with no repeated characters can be counted by maintaining a window that expands until a repeat occurs, then shrinking it from the left until the repeat is removed. Each time we expand the window, all substrings ending at the current character and starting anywhere within the window are guaranteed to be special. This allows counting substrings in $O(n)$ time.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^3) | O(1) | Generate all substrings and check uniqueness |
| Optimal | O(n) | O(26) ~ O(1) | Sliding window with character count or last seen index |

## Algorithm Walkthrough

1. Initialize two pointers `left` and `right` to define the window of characters being considered. Set `left = 0` initially.
2. Maintain a map or array `last_seen` that tracks the last index at which each character was seen.
3. Iterate `right` from 0 to `len(s) - 1`.
4. For the current character `s[right]`, if it has been seen before and its last seen index is within the current window (`>= left`), move `left` to `last_seen[s[right]] + 1` to remove the duplicate from the window.
5. Update `last_seen[s[right]]` to the current index `right`.
6. The number of special substrings ending at index `right` is `right - left + 1`. Add this to a running total count.
7. After the loop completes, return the total count.

**Why it works:** Every time the window is expanded to the right without duplicates, all substrings ending at `right` starting from `left` to `right` are guaranteed to have no repeats. This sliding window invariant ensures all counted substrings are special.

## Python Solution

```python
class Solution:
    def numberOfSpecialSubstrings(self, s: str) -> int:
        last_seen = {}
        left = 0
        count = 0
        
        for right, char in enumerate(s):
            if char in last_seen and last_seen[char] >= left:
                left = last_seen[char] + 1
            last_seen[char] = right
            count += right - left + 1
        
        return count
```

The implementation uses a dictionary `last_seen` to track the last occurrence of each character. The `left` pointer ensures no repeated characters are in the window. For each character, we count all substrings ending at that index that are special and sum them to the total count.

## Go Solution

```go
func numberOfSpecialSubstrings(s string) int {
    lastSeen := make(map[byte]int)
    left := 0
    count := 0
    
    for right := 0; right < len(s); right++ {
        char := s[right]
        if idx, ok := lastSeen[char]; ok && idx >= left {
            left = idx + 1
        }
        lastSeen[char] = right
        count += right - left + 1
    }
    
    return count
}
```

The Go solution uses a `map[byte]int` to track the last seen indices of characters. The `left` pointer is adjusted similarly, and substrings ending at each `right` index are counted in the same manner. Go handles indexing with `byte` type since the string contains ASCII characters.

## Edge Cases

A critical edge case is a string with all identical characters, e.g., `"aaaaa"`. Here, the algorithm must correctly adjust `left` each time a repeat occurs to avoid overcounting substrings. The sliding window handles this by moving `left` past the last seen index.

Another edge case is a string where all characters are unique, e.g., `"abcdef"`. The algorithm should count every possible substring correctly, and the window naturally expands from start to end, counting `right - left + 1` substrings at each step.

A third edge case is alternating patterns, e.g., `"abababab"`, where repeats occur periodically. The sliding window correctly shrinks `left` each time a character repeats within the current window, ensuring that only valid substrings are counted, and duplicates are never included.

This solution handles all these edge cases efficiently and correctly.
