# LeetCode 2709 - Greatest Common Divisor Traversal

## Problem Understanding

The problem gives us an array `nums`, where each position in the array can be viewed as a node in a graph. Two indices `i` and `j` are directly connected if the greatest common divisor of `nums[i]` and `nums[j]` is greater than `1`.

The question is not asking whether every pair of values has a direct connection. Instead, it asks whether every pair of indices can reach each other through a sequence of valid traversals. In graph terms, we must determine whether the graph formed by these connections is fully connected.

For example, in `nums = [2,3,6]`, indices `0` and `1` are not directly connected because `gcd(2,3)=1`. However, both are connected to index `2`, so there exists a path between them. Therefore, all indices belong to a single connected component.

The input size is large:

- `nums.length` can be as large as `100,000`
- `nums[i]` can be as large as `100,000`

These constraints immediately rule out approaches that explicitly examine every pair of elements. A naive graph construction would require checking approximately `10^10` pairs in the worst case, which is impossible within time limits.

Several edge cases are particularly important:

- A single-element array is always traversable because there are no pairs that need connectivity.
- Any occurrence of `1` is problematic. Since `gcd(1, x) = 1` for every integer `x`, a node containing `1` cannot connect to any other node.
- Duplicate numbers should be handled correctly.
- Numbers may not share factors directly but may still become connected through intermediate values.
- Large arrays require an approach that avoids pairwise GCD computation.

## Approaches

### Brute Force

A straightforward solution is to build the entire graph explicitly.

For every pair of indices `(i, j)`, compute `gcd(nums[i], nums[j])`. If the GCD is greater than `1`, add an edge between the two nodes. After constructing the graph, perform a DFS or BFS from one node and check whether all nodes are reachable.

This approach is correct because it directly models the traversal rules given in the problem statement. If every node is reachable from every other node, the graph is connected.

Unfortunately, the graph construction requires examining every pair of elements, resulting in quadratic complexity. With up to `100,000` elements, this is far too slow.

### Key Insight

The crucial observation is that connectivity depends on shared prime factors.

If two numbers share a prime factor, then their GCD is greater than `1`, meaning they should belong to the same connected component.

Instead of connecting indices directly, we can connect numbers through their prime factors.

Consider:

```
[6, 10, 15]
```

Prime factorizations:

```
6  = 2 × 3
10 = 2 × 5
15 = 3 × 5
```

Although not every pair shares the same prime factor, all numbers become connected through the factor graph.

This suggests a Union-Find solution:

- Each array index starts in its own component.
- Factor each number into its distinct prime factors.
- For each prime factor, remember the first index where it appeared.
- Whenever another number contains the same prime factor, union the two indices.

After processing all numbers, the graph is connected if and only if all indices belong to the same Union-Find component.

Because `nums[i] ≤ 100000`, prime factorization can be performed efficiently.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n² log M) | O(n²) | Check every pair and build graph explicitly |
| Optimal | O(n log log M) | O(n + M) | Union indices through shared prime factors |

Here, `n` is the array length and `M = max(nums)`.

## Algorithm Walkthrough

### 1. Handle trivial cases

If the array contains only one element, return `true` immediately.

A single node is trivially connected.

### 2. Detect impossible cases involving 1

If the array length is greater than one and any element equals `1`, return `false`.

A value of `1` has no prime factors and cannot connect to any other number because every GCD involving `1` equals `1`.

### 3. Initialize Union-Find

Create a Union-Find structure where each index initially belongs to its own set.

The Union-Find data structure efficiently merges connected components and determines whether two indices belong to the same component.

### 4. Track prime-factor ownership

Maintain a hash map:

```
prime_factor -> first index containing it
```

When a prime factor is encountered for the first time, store the current index.

When the same factor appears again, union the current index with the stored index.

### 5. Factor every number

For each number:

- Extract its distinct prime factors.
- Ignore repeated occurrences of the same factor within the same number.
- For each distinct prime factor:

- If the factor has appeared before, union the two indices.
- Otherwise record the current index.

### 6. Verify connectivity

After processing all numbers, find the root of index `0`.

Check whether every index has the same root.

If all indices share one root, return `true`; otherwise return `false`.

### Why it works

Every valid graph edge exists because two numbers share at least one prime factor. The algorithm unions exactly those indices that share a prime factor. Therefore, the Union-Find structure represents the same connected components as the original graph.

If two numbers can be connected through a chain of shared factors, repeated union operations place them in the same set. Consequently, all indices are traversable if and only if the Union-Find structure contains a single connected component.

## Python Solution

```python
from typing import List

class Solution:
    def canTraverseAllPairs(self, nums: List[int]) -> bool:
        n = len(nums)

        if n == 1:
            return True

        if 1 in nums:
            return False

        parent = list(range(n))
        rank = [0] * n

        def find(x: int) -> int:
            if parent[x] != x:
                parent[x] = find(parent[x])
            return parent[x]

        def union(a: int, b: int) -> None:
            root_a = find(a)
            root_b = find(b)

            if root_a == root_b:
                return

            if rank[root_a] < rank[root_b]:
                parent[root_a] = root_b
            elif rank[root_a] > rank[root_b]:
                parent[root_b] = root_a
            else:
                parent[root_b] = root_a
                rank[root_a] += 1

        factor_owner = {}

        for index, value in enumerate(nums):
            x = value
            factor = 2

            while factor * factor <= x:
                if x % factor == 0:
                    if factor in factor_owner:
                        union(index, factor_owner[factor])
                    else:
                        factor_owner[factor] = index

                    while x % factor == 0:
                        x //= factor

                factor += 1

            if x > 1:
                if x in factor_owner:
                    union(index, factor_owner[x])
                else:
                    factor_owner[x] = index

        root = find(0)

        for i in range(1, n):
            if find(i) != root:
                return False

        return True
```

The implementation begins by handling special cases. A single-element array is immediately connected, while any array containing `1` and having more than one element is impossible to connect.

The Union-Find structure is implemented using path compression and union by rank. These optimizations make set operations nearly constant time.

The `factor_owner` dictionary records the first index associated with each prime factor. During factorization, every distinct prime factor either becomes registered or triggers a union with the previously registered index.

After all unions are complete, the algorithm checks whether every index belongs to the same connected component as index `0`. If so, the graph is connected.

## Go Solution

```go
func canTraverseAllPairs(nums []int) bool {
	n := len(nums)

	if n == 1 {
		return true
	}

	for _, x := range nums {
		if x == 1 {
			return false
		}
	}

	parent := make([]int, n)
	rank := make([]int, n)

	for i := 0; i < n; i++ {
		parent[i] = i
	}

	var find func(int) int
	find = func(x int) int {
		if parent[x] != x {
			parent[x] = find(parent[x])
		}
		return parent[x]
	}

	union := func(a, b int) {
		ra := find(a)
		rb := find(b)

		if ra == rb {
			return
		}

		if rank[ra] < rank[rb] {
			parent[ra] = rb
		} else if rank[ra] > rank[rb] {
			parent[rb] = ra
		} else {
			parent[rb] = ra
			rank[ra]++
		}
	}

	factorOwner := make(map[int]int)

	for idx, value := range nums {
		x := value

		for factor := 2; factor*factor <= x; factor++ {
			if x%factor == 0 {
				if owner, exists := factorOwner[factor]; exists {
					union(idx, owner)
				} else {
					factorOwner[factor] = idx
				}

				for x%factor == 0 {
					x /= factor
				}
			}
		}

		if x > 1 {
			if owner, exists := factorOwner[x]; exists {
				union(idx, owner)
			} else {
				factorOwner[x] = idx
			}
		}
	}

	root := find(0)

	for i := 1; i < n; i++ {
		if find(i) != root {
			return false
		}
	}

	return true
}
```

The Go implementation follows exactly the same logic as the Python version. Maps are used to store factor ownership, slices replace Python lists, and recursive path compression is implemented through a closure. Integer overflow is not a concern because all values remain within the problem constraints.

The dominant work comes from prime factor processing. Each distinct prime factor participates in a small number of union operations, and path compression ensures almost constant amortized cost per Union-Find operation. Given `M ≤ 100000`, this easily fits within the problem constraints.

## Edge Cases

### Arrays Containing 1

The number `1` has no prime factors. Since `gcd(1, x) = 1` for every integer `x`, it cannot connect to any other node. If the array contains more than one element and one of them is `1`, full connectivity is impossible. The implementation detects this immediately and returns `false`.

### Single-Element Arrays

An array with only one index contains no pairs that need to be connected. By definition, the traversal requirement is already satisfied. The implementation returns `true` before performing any Union-Find work.

### Connectivity Through Intermediate Numbers

A common mistake is checking only direct GCD relationships. For example:

```
[2, 3, 6]
```

Indices `0` and `1` have GCD equal to `1`, yet they are connected through index `2`. The Union-Find approach correctly captures transitive connectivity because shared prime factors repeatedly merge components.

### Numbers With Repeated Prime Factors

Values such as `12 = 2² × 3` or `72 = 2³ × 3²` contain repeated prime factors. A buggy implementation might process the same factor multiple times. The factorization step removes all occurrences of a prime factor after processing it once, ensuring each distinct prime factor contributes only a single union operation.

### Multiple Disconnected Components

Arrays such as:

```
[2, 4, 9, 27]
```

form two separate groups, one based on prime factor `2` and one based on prime factor `3`. The final root comparison detects that not all indices belong to the same Union-Find component and correctly returns `false`.
