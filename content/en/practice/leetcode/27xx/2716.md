# LeetCode 2716 - Minimize String Length

## Problem Understanding

The problem gives us a string `s` consisting of lowercase English letters. We are allowed to repeatedly perform two types of operations involving equal characters.

For any position `i`, if the character at that position is `c`, we may delete the closest occurrence of `c` to the left of `i`, if one exists. Similarly, we may delete the closest occurrence of `c` to the right of `i`, if one exists.

The goal is to perform these operations as many times as beneficial in order to minimize the final length of the string.

The key question is not how to simulate every possible sequence of deletions, but rather what the shortest possible string looks like after all valid operations have been used.

The input is a single string `s`, and the output is an integer representing the minimum achievable length.

The constraints are very small:

- `1 <= s.length <= 100`
- `s` contains only lowercase English letters

Although the input size is small, the challenge is recognizing the underlying pattern rather than performing an exhaustive search.

Important edge cases include strings containing only one character, strings where all characters are identical, and strings where every character is distinct. These cases help reveal the fundamental property of the allowed operations.

## Approaches

### Brute Force

A brute-force approach would attempt to simulate every possible operation sequence.

At each step, we would examine all indices and determine whether a valid deletion can be performed to the left or right. For every possible operation, we would generate a new string and recursively continue searching. Eventually, we would track the minimum string length obtainable among all possible states.

This approach is correct because it explores every valid sequence of operations. However, many different operation orders are possible, and the number of resulting states grows exponentially. Even though the input size is only 100, this approach is unnecessarily complicated and inefficient.

### Key Insight

Consider any character that appears multiple times.

Suppose a character `'a'` appears several times. As long as at least two occurrences remain, one occurrence can be used to delete another occurrence. Repeating this process allows us to eliminate duplicates until only one `'a'` remains.

However, once only a single occurrence remains, there is no second `'a'` available to perform further deletions. Therefore, at least one copy of every distinct character must remain in the final string.

This means:

- Every duplicate occurrence can eventually be removed.
- One occurrence of each distinct character must remain.
- The minimum possible length equals the number of distinct characters in the string.

Therefore, the problem reduces to counting unique characters.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Exponential | Exponential | Explores all possible deletion sequences |
| Optimal | O(n) | O(k) | Count distinct characters using a hash set |

Where:

- `n` is the length of the string.
- `k` is the number of distinct characters, at most 26.

## Algorithm Walkthrough

### Optimal Algorithm

1. Create an empty hash set.
2. Iterate through every character in the string.
3. Insert each character into the hash set.
4. Since a set automatically stores only unique values, duplicate characters are ignored.
5. After processing the entire string, the size of the set equals the number of distinct characters.
6. Return the size of the set.

The hash set is the natural data structure here because we only care about whether a character has appeared before, not how many times it appears.

### Why it works

Every duplicate occurrence of a character can eventually be removed by using another occurrence of the same character as the reference point for a deletion operation. This process can continue until exactly one copy of that character remains.

A character cannot disappear completely because deleting an occurrence always requires another occurrence of the same character to exist. Therefore, each distinct character contributes exactly one character to the final minimized string.

As a result, the minimum possible length is precisely the number of distinct characters in the original string.

## Python Solution

```python
class Solution:
    def minimizedStringLength(self, s: str) -> int:
        return len(set(s))
```

The implementation directly follows the insight discussed above.

The expression `set(s)` creates a hash set containing every distinct character in the string. Duplicate characters are automatically discarded.

The size of this set is obtained using `len()`, which gives the number of unique characters. Since each distinct character must appear exactly once in the minimized string, this value is the answer.

## Go Solution

```go
func minimizedStringLength(s string) int {
    seen := make(map[rune]bool)

    for _, ch := range s {
        seen[ch] = true
    }

    return len(seen)
}
```

The Go solution uses a map as a hash set.

Each character is inserted into the map with a boolean value. Since map keys are unique, duplicate insertions simply overwrite the existing entry and do not increase the map size.

After processing all characters, `len(seen)` gives the number of distinct characters.

Unlike Python's built-in `set`, Go does not provide a dedicated set type, so a map is the standard alternative.

## Edge Cases

### Single Character String

A string such as `"a"` contains only one occurrence of one character. Since deletions require another occurrence of the same character, no operation can be performed. The answer is `1`.

The implementation handles this naturally because `set("a")` contains one element.

### All Characters Identical

Consider `"aaaaaa"`.

Repeated deletions can remove every duplicate occurrence, but one `'a'` must remain because deleting an `'a'` always requires another `'a'` to exist. The final length is therefore `1`.

The implementation correctly returns `len(set("aaaaaa")) = 1`.

### All Characters Distinct

Consider `"abcdef"`.

No character appears more than once, so no deletion operation is ever possible. The string cannot be shortened.

The implementation returns `6`, which matches the original length because every character is unique.

### Characters Appearing Many Times

Consider `"abcabcabcabc"`.

Each character appears multiple times, but all duplicates can eventually be removed until exactly one `'a'`, one `'b'`, and one `'c'` remain.

The implementation correctly returns `3`, the number of distinct characters, regardless of how many duplicate occurrences originally existed.
