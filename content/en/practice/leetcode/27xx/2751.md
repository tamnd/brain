## Solution

## Problem Understanding

The problem asks us to simulate collisions between robots moving on a one-dimensional line. Each robot has a **unique starting position**, a **health value**, and a **movement direction**: 'L' for left and 'R' for right. All robots move simultaneously at the same speed. When two robots collide (i.e., occupy the same position at the same time), the robot with lower health is removed, and the surviving robot loses 1 health. If their health is equal, both are removed. The goal is to compute the healths of the robots that survive, in the same order as the input. If no robot survives, we return an empty array.

The input consists of three arrays: `positions` (distinct integers, 1-indexed meaning positions start at 1), `healths` (integer health values), and a string `directions` of 'L' and 'R'. The constraints (`n <= 10^5` and health/positions up to `10^9`) indicate that a naive pairwise simulation of all positions over time would be too slow. We need an efficient approach.

Important edge cases include:

1. All robots moving in the same direction - no collisions occur.
2. Collisions occur between robots of equal health - both are removed.
3. Multiple sequential collisions - a single robot may survive multiple interactions.

## Approaches

The brute-force approach would simulate every time step and check for collisions between all robot pairs. While correct, this would be O(n²) in the worst case and infeasible for n up to 10^5.

A key observation for an optimal solution is that collisions only occur when a robot moving **right** meets a robot moving **left**. Robots moving in the same direction never collide. By sorting the robots based on positions, we can use a **stack** to keep track of robots moving right. When a robot moving left is encountered, we check for collisions with robots in the stack, resolving them according to the health rules.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Simulate every time step and resolve collisions explicitly |
| Optimal | O(n log n) | O(n) | Sort robots by position and simulate collisions using a stack |

## Algorithm Walkthrough

1. **Pair and Sort Robots**: Create a list of tuples containing `(position, index, health, direction)` and sort them by position. Sorting ensures that we process collisions in the correct spatial order.
2. **Initialize Stack for Right-Moving Robots**: Create an empty stack that will store robots moving to the right. Each entry holds `(index, health)`.
3. **Process Each Robot in Order**: Iterate through the sorted robots.

a. If the robot is moving right, push it onto the stack.

b. If the robot is moving left, simulate collisions with the robots in the stack (right-moving robots).

i. Pop a right-moving robot from the stack and compare health.

ii. If the right-moving robot has higher health, decrease its health by 1, discard the left-moving robot, and push the updated right-moving robot back onto the stack.

iii. If the left-moving robot has higher health, decrease its health by 1 and continue colliding with the next right-moving robot on the stack.

iv. If healths are equal, remove both robots and continue.
4. **Collect Survivors**: After processing all robots, the stack contains remaining right-moving robots. Any left-moving robots that survived collisions should also be collected. Sort survivors back by their original input order.
5. **Return Final Healths**: Produce an array of surviving robots' healths in the same order as the input.

**Why it works**: By using a stack, we efficiently simulate only relevant collisions (right vs left). The sorted order ensures we handle collisions in the correct sequence. Each robot interacts with the stack at most once per collision, giving O(n) processing after sorting.

## Python Solution

```python
from typing import List

class Solution:
    def survivedRobotsHealths(self, positions: List[int], healths: List[int], directions: str) -> List[int]:
        robots = sorted([(pos, i, healths[i], directions[i]) for i, pos in enumerate(positions)])
        stack = []  # stores (index, health) for right-moving robots
        survived = {}
        
        for pos, idx, health, dirc in robots:
            if dirc == 'R':
                stack.append([idx, health])
            else:
                while stack and health > 0:
                    r_idx, r_health = stack[-1]
                    if r_health < health:
                        stack.pop()
                        health -= 1
                    elif r_health == health:
                        stack.pop()
                        health = 0
                    else:
                        stack[-1][1] -= 1
                        health = 0
                if health > 0:
                    survived[idx] = health
        
        for idx, health in stack:
            survived[idx] = health
        
        return [survived[i] for i in range(len(positions)) if i in survived]
```

The code first sorts robots by position, then uses a stack to simulate right-moving robots encountering left-moving robots. After resolving all collisions, it collects surviving robots in original order.

## Go Solution

```go
func survivedRobotsHealths(positions []int, healths []int, directions string) []int {
    type Robot struct {
        pos   int
        idx   int
        health int
        dir   byte
    }
    
    n := len(positions)
    robots := make([]Robot, n)
    for i := 0; i < n; i++ {
        robots[i] = Robot{positions[i], i, healths[i], directions[i]}
    }
    
    sort.Slice(robots, func(i, j int) bool {
        return robots[i].pos < robots[j].pos
    })
    
    stack := [][]int{} // stores [idx, health] for right-moving robots
    survived := map[int]int{}
    
    for _, r := range robots {
        if r.dir == 'R' {
            stack = append(stack, []int{r.idx, r.health})
        } else {
            h := r.health
            for len(stack) > 0 && h > 0 {
                top := stack[len(stack)-1]
                if top[1] < h {
                    stack = stack[:len(stack)-1]
                    h--
                } else if top[1] == h {
                    stack = stack[:len(stack)-1]
                    h = 0
                } else {
                    stack[len(stack)-1][1]--
                    h = 0
                }
            }
            if h > 0 {
                survived[r.idx] = h
            }
        }
    }
    
    for _, s := range stack {
        survived[s[0]] = s[1]
    }
    
    res := []int{}
    for i := 0; i < n; i++ {
        if h, ok := survived[i]; ok {
            res = append(res, h)
        }
    }
    return res
}
```

The Go version mirrors the Python logic. We use slices for the stack and a map to track survivors. Sorting and stack operations handle collisions efficiently.

## Worked Examples

**Example 1: positions = [5,4,3,2,1], healths = [2,17,9,15,10], directions = "RRRRR"**

All robots move right. The stack accumulates all robots. No collisions occur. Return healths in input order: `[2,17,9,15,10]`.

**Example 2: positions = [3,5,2,6], healths = [10,10,15,12], directions = "RLRL"**

Sorted positions: `[(2,2,15,'R'),(3,0,10,'R'),(5,1,10,'L'),(6,3,12,'L')]`

Stepwise:

- Robot at pos 2 (R) → stack: [(2,15)]
- Robot at pos 3 (R) → stack: [(2,15),(0,10)]
- Robot at pos 5 (L) collides:

- Compare with top (0,10): health equal? 10 vs 10 → remove both
- Next top: (2,15) vs 10 → 15 > 10 → 15-1=14, left robot removed
- Robot at pos 6 (L) → no stack? yes → survives 12? Actually stack empty? Only robot 2 remains with 14 → 14

Return `[14]`.

**Example 3: positions = [1,2,5,6], healths = [10,10,11,11], directions = "RLRL"**

All robots collide pairwise; both equal healths removed. Return `[]`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n log n) | Sorting robots by position takes O(n log n), processing collisions is O(n) |
| Space | O(n) | Stack for right-moving robots and map of survivors |

The dominant factor is sorting, and each robot is pushed/popped from the stack at most once.
