## Solution

## Problem Understanding

The problem requires breaking a given string `s` into substrings such that each substring is present in a given `dictionary`. Any characters that cannot be grouped into dictionary words are considered **extra characters**. The goal is to **minimize the number of extra characters** after optimally breaking the string.

The input string `s` is a 0-indexed sequence of lowercase letters, and the `dictionary` is a list of distinct lowercase words. Each word can have length up to 50, and the dictionary can contain up to 50 words. These constraints indicate that the input size is small enough to consider solutions with quadratic or cubic time complexity relative to the string length.

Key points to note include that words in the dictionary may overlap in `s`, the break must use **non-overlapping substrings**, and every unused character contributes to the extra character count. Edge cases could include strings where no dictionary word matches, strings fully matched by dictionary words, and strings with multiple overlapping potential matches.

## Approaches

A **brute-force approach** would be to try every possible way to break `s` into substrings and compute the leftover characters. This guarantees correctness because it considers all partitions, but the number of partitions grows exponentially with the string length, making it infeasible for even moderate lengths of `s`.

The **optimal approach** leverages **dynamic programming**. The key insight is that the problem exhibits optimal substructure: the minimum extra characters for the prefix `s[0:i]` depends only on the minimum extra characters for shorter prefixes `s[0:j]` combined with whether `s[j:i]` is a dictionary word. By storing intermediate results in a DP array, we avoid redundant computations and achieve polynomial time complexity. Using a **set for the dictionary** allows constant-time membership checks for candidate substrings.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Explore all partitions of `s` |
| Optimal DP | O(n^2 * L) | O(n + D) | n = len(s), L = max word length, D = dictionary size |

## Algorithm Walkthrough

1. Convert `dictionary` into a `set` to allow O(1) lookup of any substring.
2. Initialize a DP array `dp` of length `n+1` (where `n = len(s)`) with `dp[0] = 0`. Here, `dp[i]` will represent the **minimum extra characters** for the prefix `s[0:i]`.
3. Iterate over `i` from 1 to `n`. For each `i`, initialize `dp[i] = dp[i-1] + 1` assuming the `i-1`-th character is extra.
4. For each `j` from 0 to `i-1`, check if the substring `s[j:i]` exists in the dictionary. If it does, update `dp[i] = min(dp[i], dp[j])`.
5. After processing all `i`, the final result is `dp[n]`, representing the minimum extra characters for the full string.

**Why it works:** This works because at each step `i`, we either consider the last character as extra or use any valid dictionary word ending at `i`. By storing intermediate minimums in the DP array, we ensure that every prefix is optimally solved, which guarantees correctness by induction on the string length.

## Python Solution

```python
from typing import List

class Solution:
    def minExtraChar(self, s: str, dictionary: List[str]) -> int:
        n = len(s)
        word_set = set(dictionary)
        dp = [0] * (n + 1)
        
        for i in range(1, n + 1):
            dp[i] = dp[i - 1] + 1  # assume current character is extra
            for j in range(i):
                if s[j:i] in word_set:
                    dp[i] = min(dp[i], dp[j])
        
        return dp[n]
```

The Python implementation converts the dictionary into a set for fast lookup. It initializes the DP array and iterates over each prefix length `i`. For each prefix, it assumes the last character is extra and then checks all substrings ending at `i` to see if they are in the dictionary, updating the DP value accordingly. The final answer is stored in `dp[n]`.

## Go Solution

```go
func minExtraChar(s string, dictionary []string) int {
    n := len(s)
    wordSet := make(map[string]struct{})
    for _, word := range dictionary {
        wordSet[word] = struct{}{}
    }
    
    dp := make([]int, n+1)
    
    for i := 1; i <= n; i++ {
        dp[i] = dp[i-1] + 1
        for j := 0; j < i; j++ {
            if _, exists := wordSet[s[j:i]]; exists {
                if dp[j] < dp[i] {
                    dp[i] = dp[j]
                }
            }
        }
    }
    
    return dp[n]
}
```

The Go implementation is conceptually identical to Python. A `map` is used for dictionary lookup. The DP array is built iteratively with the same logic. Go-specific details include using `struct{}` for a set and explicitly checking `exists` in the map.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n^2 * L) | n = len(s), L = max word length in dictionary. For each i, we check all j and substring lookups take O(L). |
| Space | O(n + D) | n for DP array, D = size of dictionary for set/map storage. |

The nested loop over string indices dominates the runtime, but since n <= 50, this approach is efficient in practice.

## Edge Cases

One edge case is when the string contains no dictionary words. In this situation, every character is considered extra, and the DP array accumulates a simple count. Another edge case is when the string can be fully segmented using the dictionary. This requires checking that the DP update correctly sets `dp[i] = dp[j]` whenever a match is found, avoiding counting extra characters unnecessarily. Finally, overlapping dictionary words could create multiple valid partitions; the DP approach handles this because it always chooses the partition with the **minimum extra characters**, ensuring the global optimum even when multiple matches are possible.
