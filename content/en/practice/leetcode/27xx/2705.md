## Solution

## Problem Understanding

This problem requires creating a **compact version** of a given JSON object or array. Essentially, we must remove any key-value pairs or array elements where the value is **falsy**. In JavaScript terms, a falsy value is one that evaluates to `false` in a Boolean context, such as `null`, `0`, `false`, `""`, `undefined`, or `NaN`. Importantly, the compaction must be applied **recursively**, so nested objects or arrays must also have their falsy elements removed. Arrays are treated like objects with numeric keys, so the same rules apply.

The input `obj` is guaranteed to be valid JSON, meaning we do not need to handle circular references or non-serializable values. The constraint `2 <= JSON.stringify(obj).length <= 10^6` indicates that the input could be large, so we need a solution that scales efficiently.

Edge cases include empty arrays or objects, arrays or objects that contain only falsy values, deeply nested structures, and mixed-type arrays. The solution must correctly handle all nested layers while preserving non-falsy values.

## Approaches

A **brute-force approach** would involve iterating through the object or array and checking each value. If the value is falsy, we remove it; if the value is an object or array, we recursively apply the same operation. While correct, this approach may unnecessarily copy objects or arrays multiple times, creating overhead for large, deeply nested structures.

The **optimal approach** leverages **recursive traversal with in-place construction**. For arrays, we filter out falsy elements after recursively compacting nested arrays or objects. For objects, we iterate over each key, recursively compact its value, and include it in the result only if it is truthy. This guarantees that all falsy values, including those in nested structures, are removed efficiently.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Recursively copy and filter each element, correct but may use extra memory |
| Optimal | O(n) | O(n) | Recursively traverse and rebuild only truthy values, minimizing unnecessary copying |

## Algorithm Walkthrough

1. **Identify type**: Check if the current value is an array, object, or primitive.
2. **Process arrays**: If the value is an array, initialize an empty array for the result. Recursively process each element, and if the returned element is truthy, append it to the result.
3. **Process objects**: If the value is an object, initialize an empty object for the result. Iterate over each key, recursively compact the corresponding value, and add it to the result if truthy.
4. **Handle primitives**: If the value is a primitive (number, string, boolean, null), return it if it is truthy; otherwise, return a falsy indicator (e.g., `None` in Python) so it can be filtered by the parent.
5. **Return result**: After processing, return the compacted object or array.

**Why it works**: By recursively compacting nested structures and only including truthy values at each level, the algorithm guarantees that no falsy values remain anywhere in the final object. Arrays and objects are handled correctly according to their type, and the recursion naturally traverses all nested elements.

## Python Solution

```python
class Solution:
    def compactObject(self, obj: object) -> object:
        if isinstance(obj, list):
            return [self.compactObject(item) for item in obj if self.is_truthy(self.compactObject(item))]
        elif isinstance(obj, dict):
            return {k: v for k, v in ((key, self.compactObject(value)) for key, value in obj.items()) if self.is_truthy(v)}
        else:
            return obj

    def is_truthy(self, value: object) -> bool:
        return bool(value)
```

**Implementation walkthrough**: The `compactObject` method first determines whether `obj` is a list or dict. For lists, it recursively compacts each element and includes it in the new list only if truthy. For dictionaries, it recursively compacts each value and includes it only if truthy. The helper `is_truthy` method wraps Python’s `bool()` to make the code cleaner. Primitive values are returned as-is if truthy, ensuring they are preserved in the final compacted structure.

## Go Solution

```go
func compactObject(obj any) any {
    switch val := obj.(type) {
    case []any:
        result := []any{}
        for _, item := range val {
            compacted := compactObject(item)
            if isTruthy(compacted) {
                result = append(result, compacted)
            }
        }
        return result
    case map[string]any:
        result := map[string]any{}
        for k, v := range val {
            compacted := compactObject(v)
            if isTruthy(compacted) {
                result[k] = compacted
            }
        }
        return result
    default:
        return obj
    }
}

func isTruthy(value any) bool {
    switch v := value.(type) {
    case nil:
        return false
    case bool:
        return v
    case string:
        return v != ""
    case float64:
        return v != 0
    default:
        return true
    }
}
```

**Go-specific notes**: Go does not have a built-in `bool()` like Python, so we explicitly check common JSON types for truthiness. Nil, zero floats, empty strings, and false booleans are treated as falsy. Maps and slices are recursively compacted. The algorithm preserves structure and correctness across nested layers.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every element is visited once, recursively traversing all nested arrays and objects. |
| Space | O(n) | The recursion stack and new compacted structures require space proportional to the input size. |

The algorithm is linear in the size of the input JSON structure because each element or key-value pair is visited exactly once. Memory usage is linear due to the recursion stack and building the result structures.
