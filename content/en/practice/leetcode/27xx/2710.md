## Problem Understanding

The problem asks us to remove all trailing zeros from a string that represents a positive integer. The input is a string `num` consisting only of digits and guaranteed not to have leading zeros. Trailing zeros are consecutive zeros at the end of the string. For example, in `"51230100"`, the trailing zeros are the last two digits, `"00"`, so removing them results in `"512301"`.

The expected output is the same number without trailing zeros, still represented as a string. This is important because the input may be very large, up to 1000 digits, so we cannot assume it fits into a standard integer type in many languages.

Important constraints and guarantees include:

- `1 <= num.length <= 1000`: the string can be quite long, but linear operations are acceptable.
- `num` consists only of digits: we do not need to handle non-numeric characters.
- `num` has no leading zeros: we will not encounter inputs like `"000123"`.

Key edge cases include:

- The number has no trailing zeros, e.g., `"123"`.
- The number consists entirely of zeros, e.g., `"0"`.
- The number has one trailing zero, e.g., `"10"`.

## Approaches

The brute-force approach involves iterating from the end of the string and repeatedly removing the last character until a non-zero digit is encountered. While simple and correct, this can be slightly verbose in implementation if done manually character by character.

The key observation for an optimal solution is that most languages provide built-in string methods that can remove trailing characters efficiently. In Python, `rstrip('0')` directly removes trailing zeros, and in Go, a similar approach can be implemented using slicing and a loop to find the last non-zero index. This eliminates unnecessary string copying and gives a clear, readable solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Iterate from the end, remove trailing zeros one by one |
| Optimal | O(n) | O(1) | Use string methods to remove trailing zeros in one pass |

## Algorithm Walkthrough

1. Start with the input string `num`.
2. Identify the position of the last non-zero character by iterating from the end of the string toward the start.
3. Slice the string from the beginning up to (and including) this last non-zero character. This automatically removes all trailing zeros.
4. Return the resulting string.

Why it works: This algorithm works because the invariant we maintain is that all characters after the last non-zero digit are zeros. By finding the last non-zero character and slicing the string to that point, we guarantee that all trailing zeros are removed while preserving all other digits.

## Python Solution

```python
class Solution:
    def removeTrailingZeros(self, num: str) -> str:
        # Find the index of the last non-zero character
        last_non_zero = len(num) - 1
        while last_non_zero >= 0 and num[last_non_zero] == '0':
            last_non_zero -= 1
        # Slice the string up to the last non-zero character
        return num[:last_non_zero + 1]
```

The implementation first initializes a pointer `last_non_zero` at the end of the string. It moves backward while the character is `'0'`. Once a non-zero character is found, slicing the string up to this index effectively removes all trailing zeros. This directly follows the algorithm steps above.

## Go Solution

```go
func removeTrailingZeros(num string) string {
    lastNonZero := len(num) - 1
    for lastNonZero >= 0 && num[lastNonZero] == '0' {
        lastNonZero--
    }
    return num[:lastNonZero+1]
}
```

In Go, we similarly start from the end of the string and move backward until we find the last non-zero character. Slicing in Go is exclusive at the upper bound, so `num[:lastNonZero+1]` captures all characters up to and including the last non-zero digit.

## Edge Cases

The first important edge case is when the input is a single digit `"0"`. Removing trailing zeros here results in an empty string, which is technically correct if we follow the algorithm, but some implementations may want `"0"` instead depending on the interpretation. Our implementation correctly returns an empty string.

The second edge case is when the input has no trailing zeros, like `"123"`. The implementation must ensure that the string is returned unchanged. By scanning from the end and stopping at the first non-zero, we preserve all characters.

The third edge case is numbers with a mix of zeros in the middle, like `"100100100"`. The zeros in the middle should not be removed, only trailing zeros at the end. By specifically slicing up to the last non-zero character, the algorithm correctly handles this situation.

This approach is robust for very long strings up to 1000 characters, because the linear scan is efficient and we do not create unnecessary copies of the string.
