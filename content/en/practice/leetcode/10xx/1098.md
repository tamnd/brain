## Solution

Let's solve **LeetCode 1098 - Unpopular Books** in the requested detailed format.

## Problem Understanding

The problem asks us to find books that are "unpopular," which is defined as having sold fewer than 10 copies in the last year. Additionally, we must exclude books that have been available for less than one month from the reference date, `2019-06-23`.

The inputs are two database tables: `Books` and `Orders`. The `Books` table contains the book identifier, the name, and the date it became available. The `Orders` table contains individual orders with book IDs, quantities, and dispatch dates. Our output is a list of books that meet the unpopular criteria, including both `book_id` and `name`.

The constraints tell us that we can expect multiple orders per book and that books may have no orders in the last year. Edge cases include books that are very new, books that have no orders at all, and books with exactly 9 or 10 copies sold in the last year.

## Approaches

A brute-force approach would be to iterate through every book and sum all orders in the last year for each book individually, checking the available date for each. While correct, this approach is cumbersome and inefficient if the database has many books and orders because it would require multiple scans and joins.

The optimal solution leverages SQL aggregation. By joining `Books` with `Orders` and summing quantities only for orders in the last year, we can use a `GROUP BY` to aggregate sales per book. We then filter using `HAVING` for sums less than 10 and a `WHERE` clause for books available for at least one month. This uses database engines efficiently and avoids redundant computation.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n*m) | O(1) | For each book, scan all orders to compute sum. Too slow for large datasets. |
| Optimal | O(n + m) | O(n) | Use SQL JOIN + GROUP BY to aggregate efficiently; scales with database engine optimizations. |

## Algorithm Walkthrough

1. Define the reference date `2019-06-23` and calculate the cutoff date for books to be at least one month old (`2019-05-23`).
2. Join the `Books` table with the `Orders` table on `book_id`. Use a LEFT JOIN to ensure books with no orders are included.
3. Filter the `Orders` records to only include those dispatched in the last year (`dispatch_date >= '2018-06-23'`).
4. Group the result by `book_id` and `name` to aggregate sales per book.
5. Sum the `quantity` of orders for each book in the last year.
6. Filter books where the sum of sales is less than 10 or NULL (books with no sales at all).
7. Filter books with `available_from` at least one month old from `2019-06-23`.
8. Select `book_id` and `name` from the resulting dataset.

**Why it works:** This approach guarantees correctness because it first filters by relevant time frames, aggregates per book, and then applies the conditions on sales and availability. The grouping ensures all orders for the same book are counted exactly once. Using LEFT JOIN ensures books with zero orders are included in the result.

## Python Solution

Since this is a database problem, a Python solution using `sqlite3` or `pandas` would follow the same logic.

```
import sqlite3
from datetime import date, timedelta

def unpopular_books(conn):
    """
    Returns books with less than 10 sales in the last year and available at least 1 month before 2019-06-23
    """
    query = """
    WITH recent_orders AS (
        SELECT b.book_id, b.name, COALESCE(SUM(o.quantity), 0) AS total_sales
        FROM Books b
        LEFT JOIN Orders o 
            ON b.book_id = o.book_id 
            AND o.dispatch_date >= DATE('2019-06-23', '-1 year')
        WHERE b.available_from <= DATE('2019-06-23', '-1 month')
        GROUP BY b.book_id, b.name
    )
    SELECT book_id, name
    FROM recent_orders
    WHERE total_sales < 10
    """
    cursor = conn.cursor()
    cursor.execute(query)
    return cursor.fetchall()
```

This query first defines a CTE `recent_orders` to compute total sales per book in the last year. The `LEFT JOIN` ensures books without orders are included. The final selection filters books with sales under 10.

## Go Solution

In Go, we typically execute SQL queries directly using the `database/sql` package.

```
import (
    "database/sql"
)

func unpopularBooks(db *sql.DB) ([]struct{ BookID int; Name string }, error) {
    query := `
    WITH recent_orders AS (
        SELECT b.book_id, b.name, COALESCE(SUM(o.quantity), 0) AS total_sales
        FROM Books b
        LEFT JOIN Orders o 
            ON b.book_id = o.book_id 
            AND o.dispatch_date >= DATE('2019-06-23', '-1 year')
        WHERE b.available_from <= DATE('2019-06-23', '-1 month')
        GROUP BY b.book_id, b.name
    )
    SELECT book_id, name
    FROM recent_orders
    WHERE total_sales < 10
    `
    rows, err := db.Query(query)
    if err != nil {
        return nil, err
    }
    defer rows.Close()

    var result []struct{ BookID int; Name string }
    for rows.Next() {
        var b struct{ BookID int; Name string }
        if err := rows.Scan(&b.BookID, &b.Name); err != nil {
            return nil, err
        }
        result = append(result, b)
    }
    return result, nil
}
```

Go-specific considerations include proper handling of `nil` for empty results, using `COALESCE` to treat NULL sums as 0, and scanning rows into structs rather than dictionaries.
