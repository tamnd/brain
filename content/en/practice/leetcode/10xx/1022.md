## Solution

## Problem Understanding

The problem asks us to compute the sum of all numbers formed by root-to-leaf paths in a binary tree where each node contains either `0` or `1`. Each path from the root to a leaf represents a binary number, with the root node as the most significant bit and the leaf node as the least significant bit. The output is the sum of all these binary numbers interpreted as decimal integers.

The input is a binary tree of up to 1000 nodes, each node having a value `0` or `1`. We are guaranteed that the tree is non-empty. A leaf is a node with no left or right children. The constraints indicate that even a brute-force traversal of all paths is feasible, but there is room for optimization by computing the binary number incrementally during traversal.

Important edge cases include a single-node tree, a tree where all nodes are `0`, or a tree skewed to one side. The problem guarantees that all values are `0` or `1` and the sum fits in a 32-bit integer, so we do not need to handle invalid inputs or overflow.

## Approaches

A brute-force approach would involve generating all root-to-leaf paths as strings or arrays, converting each to a binary integer, and summing them. This works correctly but incurs additional space and conversion costs. Specifically, storing each path and then converting to decimal is redundant since the value can be computed incrementally during traversal.

The optimal approach uses depth-first search (DFS) recursively or iteratively while maintaining a running value of the current path interpreted as a binary number. As we traverse down the tree, we shift the current number left by one bit and add the current node’s value. When a leaf is reached, we add the running number to the total sum. This avoids constructing strings and reduces space usage to the recursion stack.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * h) | O(n * h) | Generate all root-to-leaf paths, convert each to int, sum them |
| Optimal | O(n) | O(h) | DFS traversal, accumulate binary number on the fly, h = tree height |

## Algorithm Walkthrough

1. Initialize a sum variable to store the total sum of all root-to-leaf numbers.
2. Define a recursive DFS function that accepts a node and the current accumulated binary value.
3. At each node, update the current value as `current = current * 2 + node.val`. This shifts the previous value left by one bit and adds the current node.
4. If the node is a leaf (no left or right children), add the current value to the total sum.
5. If the node has a left child, recursively call DFS on the left child with the updated value.
6. If the node has a right child, recursively call DFS on the right child with the updated value.
7. Start the DFS from the root with an initial value of 0.
8. Return the total sum after DFS completes.

Why it works: At each node, the current accumulated value correctly represents the binary number formed from the root to that node. By propagating this value down to the leaves, we ensure that each root-to-leaf path is converted into a decimal integer exactly once, giving the correct sum.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

from typing import Optional

class Solution:
    def sumRootToLeaf(self, root: Optional[TreeNode]) -> int:
        total_sum = 0
        
        def dfs(node: Optional[TreeNode], current: int):
            nonlocal total_sum
            if not node:
                return
            current = (current << 1) | node.val  # Shift left and add node.val
            if not node.left and not node.right:  # Leaf node
                total_sum += current
                return
            dfs(node.left, current)
            dfs(node.right, current)
        
        dfs(root, 0)
        return total_sum
```

The Python implementation defines a `dfs` helper function that recursively traverses the tree. The `current` value is updated using bit manipulation (`current << 1 | node.val`) for efficiency. At each leaf, the value is added to `total_sum`. Using `nonlocal` allows the helper to modify the variable defined in the outer function scope.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func sumRootToLeaf(root *TreeNode) int {
    totalSum := 0
    
    var dfs func(node *TreeNode, current int)
    dfs = func(node *TreeNode, current int) {
        if node == nil {
            return
        }
        current = (current << 1) | node.Val
        if node.Left == nil && node.Right == nil {
            totalSum += current
            return
        }
        dfs(node.Left, current)
        dfs(node.Right, current)
    }
    
    dfs(root, 0)
    return totalSum
}
```

The Go implementation mirrors the Python logic. It uses an inner function `dfs` with a closure over `totalSum`. The binary accumulation uses the same bit-shifting technique. Go handles nil pointers explicitly, unlike Python's `None`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited exactly once during DFS |
| Space | O(h) | Recursion stack height, h = tree height, worst case O(n) for skewed tree |

The optimal approach uses a single DFS traversal and maintains an integer accumulator. Bit-shifting operations are O(1) and do not affect asymptotic time complexity.

## Edge Cases

One edge case is a single-node tree. If the node value is `0` or `1`, the algorithm should return the node value itself, which is correctly handled by the DFS reaching the leaf.

Another edge case is a completely skewed tree, either left or right. In this scenario, the recursion depth reaches the number of nodes, but the algorithm still correctly accumulates the binary value along the single path.

A third edge case is a tree where all nodes are `0`. Each path from root to leaf produces the binary number `0`, so the sum should be `0`. The DFS correctly adds `0` at each leaf without special casing.
