## Solution

## Problem Understanding

In this problem, we are given a string `s` consisting only of lowercase English letters. We need to determine the length of the longest substring that appears at least twice in the string.

A substring is a contiguous sequence of characters. The repeated occurrences may overlap. For example, in the string `"aaaa"`, the substring `"aaa"` appears twice:

- positions `[0..2]`
- positions `[1..3]`

Since overlapping is allowed, the answer for `"aaaa"` is `3`.

The problem does not ask us to return the substring itself. We only need the maximum possible length among all repeating substrings.

For example:

- `"abcd"` contains no repeated substring, so the answer is `0`
- `"abbaba"` contains repeated substrings `"ab"` and `"ba"`, both of length `2`
- `"aabcaabdaab"` contains `"aab"` three times, so the answer is `3`

The constraints are important:

- `1 <= s.length <= 2000`

A length of 2000 is small enough that quadratic algorithms are acceptable, but cubic algorithms become too slow. This observation strongly influences which approaches are practical.

Several edge cases are important:

- Strings with no repeated characters, such as `"abcdef"`
- Strings where all characters are the same, such as `"aaaaaa"`
- Overlapping repetitions
- Very short strings like `"a"`
- Repetitions occurring many times throughout the string

A naive solution that explicitly compares all substrings can easily become too slow because the number of substrings grows quadratically.

## Approaches

### Brute Force Approach

The most direct approach is to generate every possible substring and count how many times it appears.

For every starting index `i` and ending index `j`, we can extract the substring `s[i:j]`. Then we search the remainder of the string to see whether this substring appears again.

This works because it exhaustively checks every possible substring, so it cannot miss the correct answer.

However, the time complexity is extremely poor:

- There are `O(n^2)` substrings
- Comparing or searching substrings costs up to `O(n)`
- Overall complexity becomes `O(n^3)` or worse

With `n = 2000`, cubic behavior is too expensive.

### Optimal Approach, Binary Search + Rolling Hash

The key insight is that instead of searching directly for the longest repeating substring, we can ask a simpler question:

> "Does there exist a repeating substring of length `L`?"

If we can answer this efficiently, we can binary search on the length.

This works because the property is monotonic:

- If a repeating substring of length `L` exists, then repeating substrings of all smaller lengths also exist
- If no repeating substring of length `L` exists, then no longer repeating substring can exist either

So the answer space is ordered, making binary search possible.

To efficiently check whether a substring length `L` repeats, we use a rolling hash:

- Compute hash values for all substrings of length `L`
- Store hashes in a set
- If the same hash appears again, a repeated substring exists

The rolling hash allows us to move from one substring to the next in constant time instead of recomputing the hash from scratch.

This reduces the complexity dramatically.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(1) | Explicitly compares many substrings |
| Optimal | O(n log n) | O(n) | Binary search combined with rolling hash |

## Algorithm Walkthrough

### Step 1, Convert Characters to Numbers

Rolling hash works more conveniently with integers.

We map:

- `'a' -> 0`
- `'b' -> 1`
- ...
- `'z' -> 25`

This allows efficient mathematical hash calculations.

### Step 2, Binary Search on Substring Length

We search for the largest valid substring length.

The search range is:

- minimum length = `1`
- maximum length = `n - 1`

We repeatedly test a midpoint length `mid`.

If a repeated substring of length `mid` exists:

- store `mid` as a candidate answer
- search for a larger valid length

Otherwise:

- search smaller lengths

### Step 3, Compute Initial Rolling Hash

For a fixed length `L`, compute the hash of the first substring:

```
s[0:L]
```

We use a polynomial rolling hash:

$$hash = (hash \times base + value) \mod mod$$

Typical choices:

- `base = 26`
- large modulus such as `2^63 - 1`

### Step 4, Slide the Window

Move the substring window one character at a time.

When shifting from:

```
s[i:i+L]
```

to

```
s[i+1:i+L+1]
```

we:

1. remove the contribution of the outgoing character
2. multiply by the base
3. add the incoming character

This update happens in constant time.

### Step 5, Detect Duplicates

Store every hash in a set.

If a newly computed hash already exists in the set, then two substrings of length `L` are equal with very high probability.

That means a repeating substring of length `L` exists.

### Step 6, Continue Binary Search

Binary search continues until the search range is exhausted.

The largest successful length is the answer.

### Why it works

The binary search is valid because substring repetition is monotonic with respect to length. If a substring of length `L` repeats, then its prefixes of smaller lengths also repeat.

The rolling hash guarantees efficient substring comparison. Every substring of length `L` is represented by a hash value computed in constant time during window sliding. Detecting duplicate hashes identifies repeated substrings efficiently.

Together, these properties ensure the algorithm correctly finds the maximum repeating substring length.

## Python Solution

```
class Solution:
    def longestRepeatingSubstring(self, s: str) -> int:
        n = len(s)
        nums = [ord(c) - ord('a') for c in s]

        base = 26
        mod = (1 << 63) - 1

        def has_repeat(length: int) -> bool:
            if length == 0:
                return True

            current_hash = 0

            for i in range(length):
                current_hash = (current_hash * base + nums[i]) % mod

            seen = {current_hash}

            highest_base = pow(base, length - 1, mod)

            for start in range(1, n - length + 1):
                outgoing = nums[start - 1]
                incoming = nums[start + length - 1]

                current_hash = (
                    (current_hash - outgoing * highest_base) * base
                    + incoming
                ) % mod

                if current_hash in seen:
                    return True

                seen.add(current_hash)

            return False

        left = 1
        right = n - 1
        answer = 0

        while left <= right:
            mid = (left + right) // 2

            if has_repeat(mid):
                answer = mid
                left = mid + 1
            else:
                right = mid - 1

        return answer
```

The implementation begins by converting each character into an integer representation. This simplifies rolling hash calculations.

The helper function `has_repeat(length)` determines whether any substring of the given length appears more than once.

First, the hash of the initial substring is computed. Then a sliding window moves across the string while updating the hash in constant time.

A set called `seen` stores all previously encountered hashes. If a hash appears again, the function immediately returns `True`.

The outer binary search determines the maximum valid length. Whenever a repeated substring exists for the current midpoint, we try searching larger lengths. Otherwise, we search smaller lengths.

The final answer stores the largest successful substring length.

## Go Solution

```
func longestRepeatingSubstring(s string) int {
	n := len(s)

	nums := make([]int64, n)
	for i := 0; i < n; i++ {
		nums[i] = int64(s[i] - 'a')
	}

	const base int64 = 26
	const mod int64 = (1 << 63) - 1

	hasRepeat := func(length int) bool {
		if length == 0 {
			return true
		}

		var hash int64 = 0

		for i := 0; i < length; i++ {
			hash = (hash*base + nums[i]) % mod
		}

		seen := map[int64]bool{
			hash: true,
		}

		var highestBase int64 = 1
		for i := 0; i < length-1; i++ {
			highestBase = (highestBase * base) % mod
		}

		for start := 1; start <= n-length; start++ {
			outgoing := nums[start-1]
			incoming := nums[start+length-1]

			hash = ((hash-outgoing*highestBase)*base + incoming) % mod

			if hash < 0 {
				hash += mod
			}

			if seen[hash] {
				return true
			}

			seen[hash] = true
		}

		return false
	}

	left := 1
	right := n - 1
	answer := 0

	for left <= right {
		mid := left + (right-left)/2

		if hasRepeat(mid) {
			answer = mid
			left = mid + 1
		} else {
			right = mid - 1
		}
	}

	return answer
}
```

The Go implementation follows the same algorithmic structure as the Python version.

One important difference is handling negative modulo results. In Go, `%` may produce negative values, so after updating the rolling hash we explicitly add `mod` if necessary.

The hash set is implemented using `map[int64]bool`.

Because Go does not have Python's built in `pow(..., mod)` functionality for modular exponentiation, the power of the base is computed manually with a loop.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n log n) | Binary search performs O(log n) checks, each check scans the string once |
| Space | O(n) | Hash set stores substring hashes |

The helper function processes all substrings of a fixed length in linear time using rolling hash updates. Binary search repeats this process logarithmically many times, resulting in overall `O(n log n)` complexity.

The extra memory usage comes from storing hashes of previously seen substrings.

## Edge Cases

### Single Character String

A string like `"a"` cannot contain a repeating substring because there is only one substring of length `1`.

This case can easily produce index errors in careless implementations, especially during binary search bounds setup. Our implementation handles it correctly because the binary search range becomes invalid immediately:

```
left = 1
right = 0
```

So the loop never executes and the answer remains `0`.

### Overlapping Repetitions

Strings such as `"aaaa"` are tricky because valid repeated substrings overlap.

For example:

```
"aaa"
```

appears at:

- indices `[0..2]`
- indices `[1..3]`

Some incorrect solutions mistakenly forbid overlaps. Our rolling hash approach naturally allows them because every substring window is processed independently.

### No Repeating Substrings

Inputs like `"abcdef"` contain no repeated substrings at any length.

A buggy implementation might incorrectly treat hash collisions as guaranteed matches. While rolling hash collisions are theoretically possible, using a very large modulus makes collisions extremely unlikely in practice for the given constraints.

The implementation correctly returns `0` when no duplicate hashes are found during any binary search step.
