## Solution

## Problem Understanding

The problem asks us to count how many integers in the range `[1, n]` contain at least one repeated digit.

A repeated digit means that some digit appears more than once in the decimal representation of the number. For example:

- `11` has repeated digit `1`
- `100` has repeated digit `0`
- `123` has no repeated digits

Instead of directly counting numbers with repeated digits, the key observation is that it is much easier to count numbers with all distinct digits. Once we know how many numbers from `1` to `n` have unique digits, we can subtract that value from `n`.

Mathematically:

$$\text{answer} = n - \text{count of numbers with unique digits}$$

The input consists of a single integer `n`, where:

$$1 \le n \le 10^9$$

This constraint is important because it immediately rules out brute force iteration over every number up to `n` in an efficient interview style solution. A naive approach that checks every number individually would require up to one billion iterations, which is too slow.

Since the upper bound is `10^9`, the number of digits is at most 10. This strongly suggests a combinatorial or digit dynamic programming solution.

Several edge cases are important:

- Very small numbers such as `1`, where no repeated digits exist.
- Powers of 10 such as `100`, where repeated zeros appear.
- Numbers like `999` or `1000`, where digit boundaries change.
- Numbers with repeated digits early in the scan, such as `1223`.

The problem guarantees that `n` is positive, so we do not need to handle zero or negative inputs.

## Approaches

### Brute Force Approach

The most direct solution is to iterate through every integer from `1` to `n` and check whether it contains repeated digits.

For each number:

1. Convert the number to a string.
2. Insert digits into a set.
3. If a digit already exists in the set, the number contains duplicates.

This approach is correct because every number is explicitly checked.

However, the time complexity is far too large. If `n = 10^9`, iterating over one billion numbers is infeasible.

### Optimal Approach

The key insight is that counting numbers with unique digits is much easier than directly counting numbers with repeated digits.

We use combinatorics together with digit-by-digit processing.

The strategy is:

1. Count all unique-digit numbers with fewer digits than `n`.
2. Count unique-digit numbers with the same number of digits as `n`.
3. Subtract the total unique-digit count from `n`.

This works because:

$$\text{Repeated-digit numbers} =
n - \text{Unique-digit numbers}$$

The digit-by-digit process resembles Digit DP, but because the digit count is very small, we can implement it with permutations directly.

### Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(log n) | Checks every number individually |
| Optimal | O((\log n)^2) | O(\log n) | Uses combinatorics and digit processing |

## Algorithm Walkthrough

### Step 1: Convert `n + 1` Into Digits

We process `n + 1` instead of `n`.

This simplifies the logic because we can count unique-digit numbers strictly less than `n + 1`.

For example:

- If `n = 100`
- We process `101`
- Then all valid numbers counted are automatically `<= 100`

Let:

```
digits = list(str(n + 1))
```

### Step 2: Count Unique-Digit Numbers With Fewer Digits

Suppose `n = 5432`.

Before considering 4 digit numbers, we count all valid:

- 1 digit numbers
- 2 digit numbers
- 3 digit numbers

For a number with length `k`:

- First digit cannot be zero
- Remaining digits must be distinct

The count is:

$$9 \times 9 \times 8 \times 7 \cdots$$

For example:

- 1 digit: `9`
- 2 digits: `9 × 9`
- 3 digits: `9 × 9 × 8`

We accumulate these counts.

### Step 3: Process Digits Left to Right

Now we count unique-digit numbers having the same length as `n + 1`.

We scan digit by digit.

At position `i`:

- Try placing a smaller digit than the current digit
- Ensure it has not already been used
- Count all permutations for remaining positions

We maintain a `used` set to track chosen digits.

### Step 4: Count Remaining Permutations

Suppose we fixed some prefix.

The remaining positions can be filled using unused digits.

If there are `m` available digits and `r` remaining positions:

$$P(m, r) = m \times (m-1) \times \cdots$$

This gives the number of valid completions.

### Step 5: Stop When Duplicate Appears

If the current digit already exists in `used`, then the prefix already contains repeated digits.

No further valid unique-digit numbers can be formed, so we stop.

### Step 6: Compute Final Answer

We now know:

```
unique_count = numbers with distinct digits
```

Therefore:

```
answer = n - unique_count
```

### Why it works

The algorithm exhaustively counts every number with distinct digits exactly once.

- Numbers with fewer digits are counted combinatorially.
- Numbers with the same digit length are counted lexicographically.
- The `used` set guarantees no digit repeats.
- Permutation counting ensures all valid suffixes are included.

Since every number is either unique-digit or repeated-digit, subtracting gives the exact answer.

## Python Solution

```
class Solution:
    def numDupDigitsAtMostN(self, n: int) -> int:
        digits = list(map(int, str(n + 1)))
        length = len(digits)

        def permutations(m: int, k: int) -> int:
            result = 1
            for i in range(k):
                result *= (m - i)
            return result

        unique_count = 0

        # Count numbers with fewer digits
        for digits_count in range(1, length):
            count = 9
            available = 9

            for _ in range(digits_count - 1):
                count *= available
                available -= 1

            unique_count += count

        # Count numbers with same length
        used = set()

        for i in range(length):
            current_digit = digits[i]

            for digit in range(0 if i > 0 else 1, current_digit):
                if digit in used:
                    continue

                remaining_positions = length - i - 1
                available_digits = 10 - (i + 1)

                unique_count += permutations(
                    available_digits,
                    remaining_positions
                )

            if current_digit in used:
                break

            used.add(current_digit)

        return n - unique_count
```

The implementation follows the exact algorithm described earlier.

The `permutations` helper computes:

$$P(m, k)$$

which represents the number of ways to fill remaining positions.

The first loop counts all valid numbers with fewer digits than `n + 1`.

The second loop processes numbers with the same length digit by digit. At each position, we try placing smaller unused digits and count all valid completions.

The `used` set guarantees that digits remain unique.

If a repeated digit appears, the loop terminates immediately because no longer prefixes can produce unique-digit numbers.

Finally, subtracting `unique_count` from `n` gives the number of integers containing repeated digits.

## Go Solution

```
package main

func numDupDigitsAtMostN(n int) int {
	digits := []int{}
	temp := n + 1

	for temp > 0 {
		digits = append([]int{temp % 10}, digits...)
		temp /= 10
	}

	length := len(digits)

	permutations := func(m int, k int) int {
		result := 1
		for i := 0; i < k; i++ {
			result *= (m - i)
		}
		return result
	}

	uniqueCount := 0

	// Count numbers with fewer digits
	for digitsCount := 1; digitsCount < length; digitsCount++ {
		count := 9
		available := 9

		for i := 0; i < digitsCount-1; i++ {
			count *= available
			available--
		}

		uniqueCount += count
	}

	used := make(map[int]bool)

	// Count numbers with same length
	for i := 0; i < length; i++ {
		currentDigit := digits[i]

		start := 0
		if i == 0 {
			start = 1
		}

		for digit := start; digit < currentDigit; digit++ {
			if used[digit] {
				continue
			}

			remainingPositions := length - i - 1
			availableDigits := 10 - (i + 1)

			uniqueCount += permutations(
				availableDigits,
				remainingPositions,
			)
		}

		if used[currentDigit] {
			break
		}

		used[currentDigit] = true
	}

	return n - uniqueCount
}
```

The Go implementation mirrors the Python logic closely.

One notable difference is that Go does not have a built in set type, so a `map[int]bool` is used to track used digits.

The permutation helper is implemented as an anonymous function.

Integer overflow is not a concern because the maximum value remains comfortably within Go's `int` range for this problem.

## Edge Cases

One important edge case is very small values such as `n = 1`. A buggy implementation might incorrectly count zero or mishandle the transition between `n` and `n + 1`. This implementation avoids that issue because the combinatorial counting starts from positive leading digits only.

Another important case is numbers containing repeated zeros, such as `100` or `1000`. Many implementations accidentally treat leading zeros as valid digits or fail to detect repeated internal zeros. This solution prevents leading zeros in the first digit position while still correctly allowing zeros elsewhere and tracking them in the `used` set.

A third tricky case occurs when repetition appears during digit scanning, such as `1223`. Once the second `2` is encountered, all remaining extensions become invalid for unique-digit counting. The algorithm immediately stops processing further positions, ensuring no invalid numbers are counted.

A final subtle case involves boundaries like `999` or `1000`, where the number of digits changes. The separate counting of shorter lengths guarantees these transitions are handled correctly and avoids off-by-one errors.
