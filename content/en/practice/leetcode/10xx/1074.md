## Solution

## Problem Understanding

This problem asks us to count the number of submatrices in a given 2D matrix whose sum equals a given target value. In other words, for a matrix of size `m x n`, we want all rectangular regions defined by top-left `(x1, y1)` and bottom-right `(x2, y2)` coordinates where the sum of elements inside that rectangle is exactly equal to `target`. Each submatrix must contain at least one element, and different submatrices are considered distinct if any of the coordinates differ.

The input is a 2D integer matrix where each element can be negative, zero, or positive, and a target integer. The constraints indicate that the matrix can be as large as 100x100, meaning a brute-force solution that examines all possible submatrices explicitly will be too slow. The constraints also allow negative numbers, so any solution relying on non-negative assumptions (like sliding window sum tricks) will fail.

Important edge cases include a matrix with all zeros, a matrix with a single element, a target that is outside the possible sum range, and cases with negative numbers that can cancel out positives to achieve the target.

## Approaches

The **brute-force approach** is straightforward: iterate over all possible submatrices, compute their sums, and count the ones equal to the target. To enumerate all submatrices, we need four nested loops for the top-left and bottom-right coordinates. Then, for each submatrix, summing its elements requires another two nested loops. This approach is correct but extremely slow because the time complexity is `O(m^2 * n^2 * m * n)` in the worst case, which is infeasible for `m = n = 100`.

The **optimal approach** leverages the concept of **prefix sums** and **hash maps**. First, we compute row-wise prefix sums for faster sum calculations across columns. Then, we fix two rows (`r1` and `r2`) and reduce the 2D problem into a 1D problem: finding subarrays in the row-sum array whose sum equals the target. This is equivalent to the "Subarray Sum Equals K" problem. Using a hash map to track cumulative sums allows us to count valid subarrays in linear time for the reduced 1D array. This reduces the overall complexity significantly.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(m^2 * n^2 * m * n) | O(1) | Enumerate all submatrices and sum them explicitly |
| Optimal | O(m^2 * n) | O(n) | Convert 2D to 1D for fixed row pairs, then use prefix sums and hash map |

## Algorithm Walkthrough

1. **Compute row-wise prefix sums**: For each row in the matrix, compute cumulative sums along columns. This allows quick calculation of sums between two columns in O(1) time.
2. **Enumerate row pairs**: Loop over all pairs of rows `(r1, r2)` where `r1 <= r2`. This represents the top and bottom boundaries of potential submatrices.
3. **Compute the compressed 1D array**: For the fixed row pair, compute a 1D array `row_sum` where each element is the sum of the column between rows `r1` and `r2`. Essentially, `row_sum[c] = sum(matrix[r1..r2][c])`.
4. **Use hash map to count subarrays**: Initialize a hash map `sum_count` with `{0:1}`. Iterate over `row_sum` while maintaining a running sum `curr_sum`. For each index, check if `curr_sum - target` exists in `sum_count`. If it does, it means a valid subarray ending at the current column has been found. Increment the count accordingly and update `sum_count` with `curr_sum`.
5. **Accumulate counts**: Sum up counts for all row pairs. After iterating over all pairs, this gives the total number of submatrices whose sum equals the target.

**Why it works**: By fixing the row boundaries and compressing the columns into a 1D array, every contiguous subarray sum corresponds exactly to a submatrix sum in the original 2D matrix. Using prefix sums ensures each sum calculation is O(1), and the hash map guarantees all valid subarrays are counted efficiently.

## Python Solution

```
from typing import List
from collections import defaultdict

class Solution:
    def numSubmatrixSumTarget(self, matrix: List[List[int]], target: int) -> int:
        rows, cols = len(matrix), len(matrix[0])
        count = 0
        
        # Precompute prefix sums for each row
        for r in range(rows):
            for c in range(1, cols):
                matrix[r][c] += matrix[r][c-1]
        
        # Enumerate all pairs of columns
        for c1 in range(cols):
            for c2 in range(c1, cols):
                sum_count = defaultdict(int)
                sum_count[0] = 1
                curr_sum = 0
                for r in range(rows):
                    # Sum of the submatrix from row r, columns c1..c2
                    row_sum = matrix[r][c2] - (matrix[r][c1-1] if c1 > 0 else 0)
                    curr_sum += row_sum
                    count += sum_count.get(curr_sum - target, 0)
                    sum_count[curr_sum] += 1
        return count
```

**Explanation**: We first compute row-wise prefix sums to allow O(1) sum queries across columns. We then fix two columns `(c1, c2)` and compute cumulative sums for rows between these columns. A hash map `sum_count` tracks prefix sums to efficiently count subarrays equal to `target`. This transforms the 2D problem into multiple 1D subarray sum problems.

## Go Solution

```
func numSubmatrixSumTarget(matrix [][]int, target int) int {
    rows, cols := len(matrix), len(matrix[0])
    count := 0

    // Precompute prefix sums for each row
    for r := 0; r < rows; r++ {
        for c := 1; c < cols; c++ {
            matrix[r][c] += matrix[r][c-1]
        }
    }

    for c1 := 0; c1 < cols; c1++ {
        for c2 := c1; c2 < cols; c2++ {
            sumCount := map[int]int{0: 1}
            currSum := 0
            for r := 0; r < rows; r++ {
                rowSum := matrix[r][c2]
                if c1 > 0 {
                    rowSum -= matrix[r][c1-1]
                }
                currSum += rowSum
                if val, exists := sumCount[currSum - target]; exists {
                    count += val
                }
                sumCount[currSum]++
            }
        }
    }
    return count
}
```

**Go-specific notes**: We use a map for the prefix sum hash table, initialize it with `0:1` to handle subarrays starting at the first row, and handle column prefix subtraction carefully. Slice indexing is safe since constraints guarantee non-empty matrices.

## Worked Examples

**Example 1**: `matrix = [[0,1,0],[1,1,1],[0,1,0]], target = 0`

Consider column pairs `(c1, c2)`:

- For `c1=0, c2=0`, row sums = `[0,1,0]`. Prefix sums: `[0,0,0]` → count = 2
- For `c1=0, c2=1`, row sums = `[1,2,1]` → count = 0
- For `c1=0, c2=2`, row sums = `[1,3,1]` → count = 0
- For `c1=1, c2=1`, row sums = `[1,1,1]` → count = 0
- For `c1=2, c2=2`, row sums = `[0,1,0]` → count = 2

Total = 4, which matches the expected output.

**Example 2**: `matrix = [[1,-1],[-1,1]], target = 0`

Column pairs `(0,0)` → `[1,-1]` → count = 1

Column pairs `(0,1)` → `[0,0]` → count = 3

Column pairs `(1,1)` → `[-1,1]` → count = 1

Total = 5

**Example 3**: `matrix = [[904]], target = 0` → only element 904 ≠ 0 → count = 0

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(cols^2 * rows) | Two nested loops over columns (O(cols^2)) and a single pass over rows for each pair (O(rows)) |
| Space | O(rows) | Hash map for prefix sums at most has `rows + 1` entries at a time |

The solution is efficient for `m,n <= 100`, giving at most 100 * 100 * 100 = 10^6 operations.
