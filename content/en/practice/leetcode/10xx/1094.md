## Solution

## Problem Understanding

The problem describes a car traveling only in one direction, east. Each trip specifies three values:

- `numPassengers`, the number of people entering the car
- `from`, the location where those passengers are picked up
- `to`, the location where those passengers are dropped off

The car starts empty and has a fixed seating capacity. We must determine whether it is possible to complete all trips without ever exceeding the car's capacity at any point along the route.

The important detail is that the car only moves east. This means passenger movement happens along a one dimensional timeline of positions. Once we move past a location, we never revisit it. Because of this property, the problem becomes similar to processing events in sorted order.

For example, consider the trip `[2,1,5]`. This means:

- At location `1`, two passengers enter the car
- Those passengers remain inside while traveling through locations `2`, `3`, and `4`
- At location `5`, they leave the car

If another trip overlaps during that interval, both groups of passengers must fit within the car simultaneously.

The input constraints provide important clues:

- `trips.length <= 1000`
- `0 <= from < to <= 1000`

The locations are relatively small, bounded by `1000`. This strongly suggests that a prefix sum or difference array approach may work efficiently.

The problem guarantees that every trip has valid pickup and drop off positions where `from < to`. We never need to handle malformed intervals.

Several edge cases are important:

- Multiple trips may start at the same location
- Multiple trips may end at the same location
- A drop off and pickup may occur at the same position
- Capacity may be exceeded only temporarily
- A trip may completely contain another trip
- Trips may not overlap at all

A naive implementation can easily make mistakes when handling simultaneous drop offs and pickups. For example, if passengers leave at position `5` and others enter at position `5`, the passengers leaving should free seats before the new passengers enter.

## Approaches

### Brute Force Approach

The most straightforward solution is to simulate the number of passengers at every location on the road.

Since locations range from `0` to `1000`, we can imagine an array representing each kilometer. For every trip:

- Add passengers to every position from `from` up to `to - 1`
- Check whether any position exceeds the capacity

For example, the trip `[3,2,7]` would increase passenger counts for positions `2`, `3`, `4`, `5`, and `6`.

This works because each position directly represents how many passengers are inside the car while traveling through that segment of the route.

However, this approach repeatedly updates large ranges. Although the constraints are small enough that it technically passes, it is inefficient compared to a more elegant solution.

### Optimal Approach, Difference Array and Prefix Sum

The key insight is that passenger counts only change at pickup and drop off locations.

Instead of updating every position within a trip range, we can record only the changes:

- At `from`, passengers enter the car
- At `to`, passengers leave the car

This is exactly the idea behind a difference array.

For each trip `[numPassengers, from, to]`:

- Add `numPassengers` at `from`
- Subtract `numPassengers` at `to`

After processing all trips, we compute a running prefix sum across locations. The running sum represents the number of passengers currently in the car.

If the running sum ever exceeds `capacity`, the trips are impossible.

This approach is efficient because each trip contributes only two updates.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n * m) | O(m) | Updates every position inside each trip range |
| Optimal | O(n + m) | O(m) | Uses difference array and prefix sum |
|  |  |  | `n = number of trips`, `m = maximum location` |

## Algorithm Walkthrough

1. Create a difference array large enough to cover all possible locations.

Since locations are at most `1000`, we can safely create an array of size `1001` or slightly larger. Each index represents a location on the road.
2. Process each trip and record passenger changes.

For a trip `[numPassengers, start, end]`:

- Add `numPassengers` at `start`
- Subtract `numPassengers` at `end`

This means passengers enter at the pickup location and leave at the drop off location.
3. Compute the running passenger count using a prefix sum.

Traverse the difference array from left to right.

Maintain a variable called `currentPassengers`.

At each position:

- Add the value from the difference array
- The updated running sum represents how many passengers are currently inside the car
4. Check the capacity constraint during traversal.

After updating the running sum:

- If `currentPassengers > capacity`, immediately return `False`

This means the car becomes overloaded at that location.
5. Finish traversal successfully.

If the entire route is processed without exceeding capacity, return `True`.

### Why it works

The difference array correctly records all passenger changes at their exact locations. The prefix sum reconstructs the actual number of passengers inside the car at every point along the route.

Because every passenger contributes a positive change at pickup and a negative change at drop off, the running sum always matches the true occupancy of the car. Therefore, checking whether the running sum exceeds capacity guarantees correctness.

## Python Solution

```
from typing import List

class Solution:
    def carPooling(self, trips: List[List[int]], capacity: int) -> bool:
        difference = [0] * 1001

        for passengers, start, end in trips:
            difference[start] += passengers
            difference[end] -= passengers

        current_passengers = 0

        for change in difference:
            current_passengers += change

            if current_passengers > capacity:
                return False

        return True
```

The implementation begins by creating a difference array called `difference`. Each index corresponds to a road location, and each value stores how the passenger count changes at that point.

For every trip:

- Passengers are added at the pickup location
- The same number of passengers are removed at the drop off location

This avoids updating every position in the interval.

Next, the code computes a running prefix sum using `current_passengers`. As we iterate through the array:

- Positive values increase occupancy
- Negative values decrease occupancy

At each location, we verify that the occupancy does not exceed the capacity. The moment it does, we immediately return `False`.

If the loop completes successfully, all trips fit within the car's capacity, so the method returns `True`.

## Go Solution

```
func carPooling(trips [][]int, capacity int) bool {
    difference := make([]int, 1001)

    for _, trip := range trips {
        passengers := trip[0]
        start := trip[1]
        end := trip[2]

        difference[start] += passengers
        difference[end] -= passengers
    }

    currentPassengers := 0

    for _, change := range difference {
        currentPassengers += change

        if currentPassengers > capacity {
            return false
        }
    }

    return true
}
```

The Go implementation follows the same logic as the Python version. A slice is used for the difference array, and a running sum tracks current occupancy.

Go integers are sufficient because the maximum possible passenger total is well within integer limits. No special overflow handling is required.

Unlike Python, Go requires explicit slice allocation with `make`, and variable declarations use static typing.

## Edge Cases

One important edge case occurs when passengers are dropped off and picked up at the same location. For example:

```
[2,1,5]
[3,5,7]
```

At location `5`, the first group leaves exactly when the second group enters. A buggy implementation might temporarily count both groups simultaneously. The difference array approach handles this correctly because the drop off contributes a negative change at the same index where the pickup contributes a positive change.

Another important edge case is when a single trip alone exceeds the car's capacity. For example:

```
trips = [[6,2,8]]
capacity = 5
```

The algorithm correctly detects this during prefix sum computation because the running passenger count immediately becomes `6`.

A third important edge case involves many overlapping trips whose combined passenger count exceeds capacity only in a small interval. For example:

```
[[2,1,10],[3,4,6],[4,5,7]]
```

The car may be valid for most of the route but overloaded briefly between locations `5` and `6`. The prefix sum traversal checks every location in order, ensuring temporary overloads are detected accurately.

A final edge case involves the maximum possible location values near `1000`. Since the implementation allocates a difference array of size `1001`, every valid location index is safely accessible without out of bounds errors.
