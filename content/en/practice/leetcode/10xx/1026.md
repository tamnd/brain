**Difficulty:** 🟡 Medium  
**Topics:** Tree, Depth-First Search, Binary Tree

## Solution

## Problem Understanding

The problem asks us to find the maximum difference in value between a node in a binary tree and any of its descendants, where a node is considered an ancestor of its child nodes and all nodes in its subtree. In other words, for any pair of nodes `(a, b)` such that `a` is an ancestor of `b`, we want to maximize the absolute difference `|a.val - b.val|`.

The input is the `root` of a binary tree, where each node has a value between `0` and `10^5`. The tree can have up to 5000 nodes, so any solution that scales worse than O(n) could be inefficient. The output is a single integer representing the maximum absolute difference.

Key edge cases to note include:

1. Trees that are highly unbalanced, such as a linear chain, which could cause naive recursion to be inefficient if not handled carefully.
2. Trees where the minimum and maximum differences occur at leaf nodes far from the root.
3. Trees where multiple nodes have the same value, which may result in zero differences.

The problem guarantees at least two nodes, so we do not need to handle an empty tree.

## Approaches

### Brute Force Approach

The naive approach is to consider every node as a potential ancestor and compare it to every descendant in its subtree. For each node, we traverse all nodes in its subtree and compute the absolute difference. We track the maximum difference across all ancestor-descendant pairs.

While this guarantees correctness, the time complexity is O(n²) in the worst case because each node might visit all other nodes in its subtree. With `n` up to 5000, this becomes inefficient.

### Optimal Approach

The key observation is that to compute the maximum difference for a node, we only need the minimum and maximum values in its subtree. Using depth-first search (DFS), we can propagate the minimum and maximum values from children up to the current node. At each node, the potential maximum difference is either `|node.val - min_in_subtree|` or `|node.val - max_in_subtree|`. By computing this recursively, we can find the global maximum in a single pass.

This reduces the time complexity to O(n) since each node is visited once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(h) | Compare each node with all descendants recursively; `h` is the height of the tree |
| Optimal | O(n) | O(h) | DFS to propagate min and max values from children; only one pass needed |

## Algorithm Walkthrough

1. Start a DFS traversal from the root. For each node, we will compute two values: the minimum and maximum values seen along the path from this node to any of its descendants.
2. For a leaf node, return its value as both the minimum and maximum.
3. For an internal node, recursively compute the minimum and maximum values for the left and right subtrees.
4. Compute the potential differences between the current node's value and the subtree's min and max values. Update a global variable tracking the maximum difference.
5. Return the updated minimum and maximum values for this node's subtree to the parent node.
6. After the DFS traversal completes, return the global maximum difference.

**Why it works:** At every node, we maintain the invariant that we know the minimum and maximum values in its subtree. The maximum absolute difference between the current node and any descendant is thus captured by `|node.val - subtree_min|` or `|node.val - subtree_max|`. Because DFS visits all nodes, we are guaranteed to find the true global maximum difference.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def maxAncestorDiff(self, root: Optional[TreeNode]) -> int:
        self.max_diff = 0
        
        def dfs(node: Optional[TreeNode]) -> tuple[int, int]:
            if not node:
                return float('inf'), float('-inf')
            
            if not node.left and not node.right:
                return node.val, node.val
            
            left_min, left_max = dfs(node.left)
            right_min, right_max = dfs(node.right)
            
            subtree_min = min(left_min, right_min, node.val)
            subtree_max = max(left_max, right_max, node.val)
            
            self.max_diff = max(self.max_diff,
                                abs(node.val - subtree_min),
                                abs(node.val - subtree_max))
            
            return subtree_min, subtree_max
        
        dfs(root)
        return self.max_diff
```

**Explanation:** The `dfs` function recursively computes the minimum and maximum values of the subtree rooted at each node. For each node, we update `self.max_diff` with the maximum difference between the node value and the subtree's min and max. Leaf nodes return their own value as min and max. The traversal ensures that every ancestor-descendant pair is considered exactly once.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func maxAncestorDiff(root *TreeNode) int {
    maxDiff := 0
    
    var dfs func(node *TreeNode) (int, int)
    dfs = func(node *TreeNode) (int, int) {
        if node == nil {
            return 1<<31 - 1, -1 << 31
        }
        if node.Left == nil && node.Right == nil {
            return node.Val, node.Val
        }
        
        leftMin, leftMax := dfs(node.Left)
        rightMin, rightMax := dfs(node.Right)
        
        subtreeMin := min(leftMin, rightMin, node.Val)
        subtreeMax := max(leftMax, rightMax, node.Val)
        
        maxDiff = max(maxDiff, abs(node.Val - subtreeMin), abs(node.Val - subtreeMax))
        
        return subtreeMin, subtreeMax
    }
    
    dfs(root)
    return maxDiff
}

func min(a, b, c int) int {
    if a < b {
        if a < c {
            return a
        }
        return c
    }
    if b < c {
        return b
    }
    return c
}

func max(a, b, c int) int {
    if a > b {
        if a > c {
            return a
        }
        return c
    }
    if b > c {
        return b
    }
    return c
}

func abs(a int) int {
    if a < 0 {
        return -a
    }
    return a
}
```

**Go-specific notes:** We handle `nil` nodes by returning extreme integer values. The helper functions `min`, `max`, and `abs` are used since Go does not provide built-in variadic min/max functions. Otherwise, the DFS logic mirrors the Python solution.

## Worked Examples

### Example 1: `[8,3,10,1,6,null,14,null,null,4,7,13]`

1. DFS starts at 8.
2. Traverse left subtree 3 → 1, 6 → 4, 7.
3. Subtree min/max for 1 → min=1, max=1; for 6 → min=4, max=7.
4. At node 3, min=1, max=7 → max_diff = max(0, |3-1|, |3-7|) = 4.
5. Continue right subtree 10 → 14 → 13.
6. Compute min/max values up to root 8: min=1, max=14 → max_diff = max(4, |8-1|, |8-14|) = 7.

### Example 2: `[1,null,2,null,0,3]`

1. DFS starts at 1 → 2 → 0 → 3.
2. Leaf 3: min/max = 3/3.
3. Node 0: min=0, max=3 → max_diff = max(0, |0-0|, |0-3|) = 3.
4. Node 2: min=0, max=3 → max_diff remains 3.
5. Node 1: min=0, max=3 → max_diff remains 3.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | DFS visits each node once; constant work per node. |
| Space | O(h) | Recursion stack up to height `h` of tree; O(n) in worst-case skewed tree, O(log n) for balanced tree. |
