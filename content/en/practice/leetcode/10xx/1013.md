## Solution

## Problem Understanding

The problem asks us to determine if a given array of integers can be split into **three non-empty contiguous subarrays** such that each part has the same sum. The array may contain positive, negative, or zero values. The output is a boolean: `true` if such a partition exists, otherwise `false`.

The key aspects to note are:

1. The three parts must be **non-empty**, meaning the smallest array that can possibly satisfy the condition has length 3.
2. We are looking for **contiguous partitions**, not just subsets.
3. The array can contain **negative numbers**, which means simple heuristics like "look for large numbers" may fail.
4. Constraints indicate the array can be fairly large (`up to 50,000` elements), so an algorithm with `O(n^2)` complexity may be too slow.

Edge cases to consider upfront include arrays where:

- All elements are zeros, which should always return `true`.
- No partition is possible due to the sum not being divisible by three.
- The array has exactly three elements, which trivially forms three parts if the sums match.

## Approaches

### Brute Force

A brute-force approach would iterate over all possible `i` and `j` indices to check every partition into three parts. For each pair `(i, j)` where `0 <= i < j <= len(arr)-1`, we compute the sum of the first, second, and third segments and compare them. If they are equal, we return `true`.

This approach is **correct**, but for large arrays, it is too slow because it requires computing sums repeatedly for all combinations, resulting in roughly `O(n^2)` time complexity.

### Optimal Approach

The key insight is that if the **total sum** of the array is divisible by 3, then each part must sum to `total_sum / 3`. We can then iterate through the array while maintaining a running sum. Each time the running sum reaches `target_sum`, we can mark that a part has been found. To ensure three non-empty parts, we only need to find **two partitions** during iteration. The remainder of the array automatically forms the third part.

This approach works in **O(n)** time and **O(1)** space, as it requires only a running sum and a counter for partitions found.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check all possible splits by iterating indices |
| Optimal | O(n) | O(1) | Use running sum and divide total by 3 |

## Algorithm Walkthrough

1. Compute the total sum of the array.
2. Check if the total sum is divisible by 3. If not, return `false` immediately.
3. Compute the target sum for each part: `target_sum = total_sum / 3`.
4. Initialize a `running_sum` to 0 and a `partitions_found` counter to 0.
5. Iterate through the array from start to the second-to-last element (to guarantee non-empty last part).
6. Add the current element to `running_sum`.
7. If `running_sum` equals `target_sum`, increment `partitions_found` and reset `running_sum` to 0.
8. After iteration, check if at least two partitions were found. If yes, return `true`; otherwise, return `false`.

**Why it works**: The algorithm guarantees that two valid partitions with `target_sum` exist, which ensures the remaining elements also sum to `target_sum`. This approach respects the non-empty constraint because the last element is never included in the first two partitions.

## Python Solution

```
from typing import List

class Solution:
    def canThreePartsEqualSum(self, arr: List[int]) -> bool:
        total_sum = sum(arr)
        if total_sum % 3 != 0:
            return False
        
        target_sum = total_sum // 3
        running_sum = 0
        partitions_found = 0
        
        for i in range(len(arr) - 1):
            running_sum += arr[i]
            if running_sum == target_sum:
                partitions_found += 1
                running_sum = 0
                if partitions_found == 2:
                    return True
        
        return False
```

The code computes the total sum and immediately checks divisibility by 3. The loop accumulates a running sum, increments the partition counter upon reaching the target, and returns early once two partitions are found.

## Go Solution

```
func canThreePartsEqualSum(arr []int) bool {
    totalSum := 0
    for _, num := range arr {
        totalSum += num
    }
    if totalSum % 3 != 0 {
        return false
    }

    targetSum := totalSum / 3
    runningSum := 0
    partitionsFound := 0

    for i := 0; i < len(arr)-1; i++ {
        runningSum += arr[i]
        if runningSum == targetSum {
            partitionsFound++
            runningSum = 0
            if partitionsFound == 2 {
                return true
            }
        }
    }

    return false
}
```

Go implementation mirrors Python closely. Note that slices are used instead of arrays. The loop stops before the last element to ensure the last partition is non-empty.
