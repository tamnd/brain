## Solution

## Problem Understanding

The problem gives us an array of lowercase English strings called `words`. We must find every character that appears in all strings, including duplicate occurrences.

The important detail is that duplicates matter. If a character appears multiple times in every word, it must appear that many times in the output. For example, in:

```
["bella","label","roller"]
```

the character `'l'` appears at least twice in every word, so the result includes `"l"` twice.

Another way to think about the problem is this: for every letter from `'a'` to `'z'`, determine the minimum number of times that letter appears across all words. That minimum frequency is exactly how many times the letter belongs in the answer.

The input size is small:

- There can be at most 100 words.
- Each word can have length at most 100.
- Only lowercase English letters are used.

These constraints tell us several useful things:

- The alphabet size is fixed at 26 characters.
- We do not need highly advanced optimization techniques.
- Counting character frequencies is very practical and efficient.

There are several edge cases worth noticing immediately. If there is only one word, every character in that word belongs in the answer because all characters trivially appear in every string. If no character appears in all words, the answer is an empty array. Another subtle case is duplicate handling, because a naive set-based solution would incorrectly remove repeated characters.

## Approaches

A brute-force approach would examine each character occurrence from the first word and check whether it exists in every other word while carefully tracking used occurrences. One possible implementation would repeatedly scan every string and remove matched characters as they are consumed.

This works because every output character must originate from the first word. By testing each occurrence individually, we can verify whether it exists across all strings. However, repeated rescanning and character removal make the implementation cumbersome and less efficient.

The key insight for the optimal solution is that this problem is fundamentally about frequency intersection. For each character, the number of times it can appear in the final answer is limited by the smallest frequency among all words.

For example:

```
bella   -> l appears 2 times
label   -> l appears 2 times
roller  -> l appears 2 times
```

The minimum frequency is 2, so `"l"` appears twice in the answer.

Since there are only 26 lowercase letters, we can efficiently maintain frequency counts and repeatedly take minimum values across all words.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n × m²) | O(m) | Repeatedly scans and consumes characters across words |
| Optimal | O(n × m) | O(1) | Uses frequency counting and minimum intersections |

Here:

- `n` is the number of words
- `m` is the average word length

## Algorithm Walkthrough

### Optimal Frequency Counting Algorithm

1. Create a frequency array of size 26 using the first word.

We begin by counting how many times each character appears in the first word. This array represents the current maximum possible answer frequencies.

For example:

```
"bella"
a -> 1
b -> 1
e -> 1
l -> 2
```
2. Process each remaining word one at a time.

For every new word, create another frequency array of size 26 that counts its characters.
3. Update the global frequency array using minimum values.

For every character from `'a'` to `'z'`, take:

```
global_count[i] = min(global_count[i], current_count[i])
```

This step is critical because a character can only appear in the final answer as many times as it appears in every word.
4. Continue until all words are processed.

After processing every word, the global frequency array stores the exact number of times each character belongs in the answer.
5. Build the result array.

Iterate through all 26 characters. If a character frequency is greater than zero, append that character to the result that many times.

For example:

```
l -> 2
```

means:

```
["l", "l"]
```

### Why it works

The algorithm maintains an invariant:

```
global_count[c]
```

always represents the minimum frequency of character `c` across all words processed so far.

Since the final answer requires characters that appear in every string, the minimum frequency across all strings is exactly the number of valid occurrences. By repeatedly taking minimum values, we guarantee correctness.

## Python Solution

```
from typing import List

class Solution:
    def commonChars(self, words: List[str]) -> List[str]:
        # Frequency count from the first word
        min_frequency = [0] * 26
        
        for char in words[0]:
            min_frequency[ord(char) - ord('a')] += 1
        
        # Process remaining words
        for word in words[1:]:
            current_frequency = [0] * 26
            
            for char in word:
                current_frequency[ord(char) - ord('a')] += 1
            
            # Keep minimum frequencies
            for i in range(26):
                min_frequency[i] = min(
                    min_frequency[i],
                    current_frequency[i]
                )
        
        # Build result
        result = []
        
        for i in range(26):
            result.extend(
                [chr(i + ord('a'))] * min_frequency[i]
            )
        
        return result
```

The implementation begins by creating a frequency array for the first word. This array acts as the running intersection of character counts.

For every additional word, we create a fresh frequency array and count its characters. We then update the running frequency array by taking the minimum value for each letter.

The final loop converts frequencies back into actual string characters. If the frequency for `'l'` is 2, we append `"l"` twice to the result.

The implementation uses arrays instead of hash maps because the alphabet is fixed to lowercase English letters. This keeps the solution simple and efficient.

## Go Solution

```
func commonChars(words []string) []string {
	minFrequency := make([]int, 26)

	// Count characters from first word
	for _, ch := range words[0] {
		minFrequency[ch-'a']++
	}

	// Process remaining words
	for _, word := range words[1:] {
		currentFrequency := make([]int, 26)

		for _, ch := range word {
			currentFrequency[ch-'a']++
		}

		// Keep minimum frequencies
		for i := 0; i < 26; i++ {
			if currentFrequency[i] < minFrequency[i] {
				minFrequency[i] = currentFrequency[i]
			}
		}
	}

	// Build result
	result := []string{}

	for i := 0; i < 26; i++ {
		for minFrequency[i] > 0 {
			result = append(result, string(rune(i+'a')))
			minFrequency[i]--
		}
	}

	return result
}
```

The Go implementation follows the exact same logic as the Python solution. The primary difference is that Go uses slices and rune arithmetic for character indexing.

Since Go strings are UTF-8 encoded, iterating with `range` produces runes. Because the problem guarantees lowercase English letters, rune subtraction like `ch - 'a'` is safe and straightforward.

The result is constructed using a `[]string` slice because the expected output format is an array of strings.

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × m) | Each character of each word is processed once |
| Space | O(1) | Frequency arrays always have fixed size 26 |

The time complexity is linear in the total number of characters across all words. Every character contributes to exactly one frequency count operation.

The space complexity is constant because the algorithm only stores fixed-size arrays of length 26, regardless of input size.

## Edge Cases

One important edge case is when there is only a single word in the input. In this scenario, every character in the word must appear in the answer because there are no other strings to restrict the frequencies. The implementation naturally handles this because the initial frequency array is created from the first word, and no further minimum operations occur.

Another critical edge case is when no characters are shared across all words. A buggy implementation might accidentally keep stale counts from earlier words. By taking the minimum frequency for every character at every step, the algorithm correctly reduces all non-common characters to zero.

Duplicate handling is also a major source of bugs. A set-based approach would incorrectly remove repeated occurrences. For example:

```
["aab", "aac"]
```

should produce:

```
["a", "a"]
```

not just:

```
["a"]
```

The frequency-based solution correctly tracks counts instead of mere existence, so duplicates are preserved exactly as required.

A final subtle case occurs when a character appears many times in one word but only a few times in another. The algorithm correctly limits the answer using the minimum frequency. For example:

```
["aaaa", "aa", "aaa"]
```

produces only two `"a"` characters because the second word contains only two occurrences.
