## Solution

## Problem Understanding

The problem presents an array `values` where each element represents the value of a sightseeing spot. You are asked to find the maximum score of a pair of sightseeing spots `(i, j)` such that `i < j`. The score of the pair is calculated as `values[i] + values[j] + i - j`, which combines the sum of the spot values while penalizing the distance between them.

In simpler terms, you want to choose two spots such that their combined value minus the distance between them is maximized. The constraints indicate that `values` can be up to 50,000 elements long, with each value between 1 and 1000, so any solution must be linear or near-linear; quadratic solutions will likely be too slow.

Important edge cases include arrays of length 2 (minimum allowed), arrays where values are increasing or decreasing, and arrays with uniform values. The problem guarantees at least two elements, so you do not need to handle empty arrays.

## Approaches

The brute-force approach involves examining all possible pairs `(i, j)` with `i < j`, calculating the score for each, and keeping track of the maximum. While this guarantees correctness, it is inefficient because it requires checking roughly `n*(n-1)/2` pairs, which is O(n²) time complexity. For `n` up to 50,000, this is impractical.

The optimal approach relies on algebraic manipulation. Notice that the score formula can be rewritten as `values[i] + i + values[j] - j`. This separates the terms involving `i` and `j`, allowing you to maintain a running maximum of `values[i] + i` while iterating over `j`. At each step, the maximum score using the current `j` can be computed in O(1) time. This approach is linear and uses constant extra space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Check all pairs `(i, j)` and compute score |
| Optimal | O(n) | O(1) | Maintain max `values[i] + i` while iterating over `j` |

## Algorithm Walkthrough

1. Initialize a variable `max_i_plus_value` to `values[0] + 0`. This keeps track of the maximum `values[i] + i` seen so far.
2. Initialize `max_score` to a very small number (or negative infinity) to store the best score found.
3. Iterate over the array starting from index `1` to `n-1`:

1. Compute the score for the current `j` as `max_i_plus_value + values[j] - j`.
2. Update `max_score` if the computed score is higher.
3. Update `max_i_plus_value` as the maximum of itself and `values[j] + j`.
4. After finishing the iteration, return `max_score`.

Why it works: By keeping the running maximum of `values[i] + i`, the algorithm guarantees that for each `j`, you are pairing it with the best possible `i < j`. This maintains the invariant that `max_i_plus_value` always contains the optimal left-side candidate for any future `j`.

## Python Solution

```
from typing import List

class Solution:
    def maxScoreSightseeingPair(self, values: List[int]) -> int:
        max_i_plus_value = values[0] + 0
        max_score = float('-inf')
        
        for j in range(1, len(values)):
            max_score = max(max_score, max_i_plus_value + values[j] - j)
            max_i_plus_value = max(max_i_plus_value, values[j] + j)
        
        return max_score
```

In this implementation, `max_i_plus_value` tracks the best left-side value plus index, and `max_score` tracks the highest pair score encountered. For each `j`, the algorithm computes the potential score using the best `i` seen so far, then updates the running maximum for future iterations.

```
func maxScoreSightseeingPair(values []int) int {
    maxIPlusValue := values[0] + 0

        maxIPlusValue = max(maxIPlusValue, values[j] + j)
    }
    
    return maxScore
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

In Go, the same principle applies. A helper function `max` is used to compute the maximum of two integers. The initial `maxScore` can be seeded with the first possible pair score to handle small arrays correctly.

## Edge Cases

One important edge case is the minimal array length of two elements. Here, the algorithm must handle `i=0` and `j=1` correctly and not attempt to access an invalid index. Another edge case is when all values are the same; in this situation, the correct pair is determined solely by their distance, so maintaining the running maximum ensures the algorithm picks the leftmost index. A third edge case is when the best pair includes the first element and a distant element; this tests that the algorithm correctly updates `max_i_plus_value` over time and does not miss potential distant pairs. In all cases, the algorithm handles these correctly by maintaining the maximum `values[i] + i` and updating it iteratively.
