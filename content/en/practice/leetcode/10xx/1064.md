## Solution

## Problem Understanding

The problem asks us to find a "fixed point" in a sorted array of distinct integers. A fixed point is defined as an index `i` such that the value at that index equals the index itself, `arr[i] == i`. We are asked to return the **smallest** such index if it exists, otherwise return `-1`.

The input is an array `arr` of length up to 10,000, sorted in ascending order, and containing integers that can range from `-10^9` to `10^9`. These constraints indicate that a simple linear scan will work, but a more efficient logarithmic approach is feasible due to the sorted nature of the array.

Important edge cases include arrays where all elements are negative (no fixed point possible), arrays where the fixed point is at the first element (`arr[0] = 0`), and arrays where the fixed point is at the last element. Since the array is sorted and distinct, there can be **at most one fixed point in any index**, and no repeated values need to be considered.

## Approaches

The **brute-force approach** iterates over the array from left to right, checking at each index `i` whether `arr[i] == i`. This works correctly because we visit every index in order, so the first match is guaranteed to be the smallest index. However, this has a linear time complexity of O(n) and does not leverage the sorted property of the array.

The **optimal approach** uses binary search. The key observation is that if `arr[mid] < mid`, any fixed point must be to the right of `mid`, because the array increases strictly and cannot "catch up" to the index on the left. Conversely, if `arr[mid] > mid`, any fixed point must be to the left. This allows us to reduce the search space by half each step, achieving O(log n) time complexity.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Iterate through all indices checking `arr[i] == i` |
| Optimal | O(log n) | O(1) | Binary search leveraging the sorted, distinct property |

## Algorithm Walkthrough

1. Initialize two pointers, `left = 0` and `right = len(arr) - 1`. These pointers define the search space.
2. While `left <= right`, compute the midpoint `mid = (left + right) // 2`.
3. Check if `arr[mid] == mid`. If true, record `mid` as a candidate for the smallest fixed point and continue searching to the left by setting `right = mid - 1` to find a smaller index.
4. If `arr[mid] < mid`, move the search to the right half by setting `left = mid + 1`.
5. If `arr[mid] > mid`, move the search to the left half by setting `right = mid - 1`.
6. After the loop ends, if a candidate was found, return it. Otherwise, return `-1`.

**Why it works:** Because the array is sorted and contains distinct integers, each comparison allows us to safely discard half of the search space. The invariant maintained is that any fixed point must lie within `[left, right]`. By always moving towards the side that could contain a fixed point, we guarantee finding the smallest index.

## Python Solution

```
from typing import List

class Solution:
    def fixedPoint(self, arr: List[int]) -> int:
        left, right = 0, len(arr) - 1
        result = -1
        while left <= right:
            mid = (left + right) // 2
            if arr[mid] == mid:
                result = mid
                right = mid - 1  # search left half for smaller index
            elif arr[mid] < mid:
                left = mid + 1
            else:
                right = mid - 1
        return result
```

The implementation sets up a binary search. The variable `result` tracks the smallest fixed point found so far. Each iteration narrows the search space based on the comparison between `arr[mid]` and `mid`. If a fixed point is found, the search continues to the left to ensure the smallest index is returned.

## Go Solution

```
func fixedPoint(arr []int) int {
    left, right := 0, len(arr)-1
    result := -1
    for left <= right {
        mid := (left + right) / 2
        if arr[mid] == mid {
            result = mid
            right = mid - 1 // search left half for smaller index
        } else if arr[mid] < mid {
            left = mid + 1
        } else {
            right = mid - 1
        }
    }
    return result
}
```

In Go, we use integer division for `mid`. The logic is the same as Python, with `result` storing the smallest fixed point. Empty arrays are handled naturally since `left <= right` will be false initially.

## Edge Cases

**All negative numbers:** If the array contains only negative numbers, `arr[i] < i` for all indices. The binary search correctly moves `left` to `mid + 1` until `left > right` and returns `-1`.

**Single element array:** For `arr = [0]`, the algorithm immediately finds `arr[0] == 0` and returns `0`. For `arr = [1]` or `arr = [-1]`, it correctly returns `-1` after checking the single element.

**Fixed point at the end of the array:** When the last element matches its index, the algorithm continues searching left when it finds a match, ensuring that if an earlier fixed point existed, it would be found. If the last element is the only fixed point, it is returned correctly.
