## Solution

## Problem Understanding

This problem asks us to assign one of four flower types to `n` gardens such that no two adjacent gardens (gardens connected by a path) have the same flower type. The input consists of an integer `n` representing the number of gardens, and an array `paths`, where each element `[xi, yi]` describes a bidirectional connection between garden `xi` and garden `yi`.

The output should be an array of length `n`, where the `i`-th element represents the flower type assigned to garden `(i+1)`. The flower types are integers `1`, `2`, `3`, or `4`.

Key constraints are that each garden has at most three paths connected to it and the problem guarantees that a valid solution always exists. These constraints are crucial because they imply that even with only four flower types, a valid coloring is always possible, since no garden has more neighbors than available flower types.

Edge cases that might trip a naive implementation include a garden with no paths (it can take any flower), a fully connected small cycle (all gardens connected to each other), or disconnected groups of gardens. The guarantee of at most three connections per garden ensures we will never run into a situation where we need more than four flowers, simplifying our solution.

## Approaches

The **brute-force approach** would attempt to try every possible assignment of flowers to gardens and check whether each assignment satisfies the adjacency condition. This approach is correct because it exhaustively explores all combinations, but it is infeasible because the number of combinations grows exponentially, `4^n`, which is impossible for `n` up to 10^4.

The **optimal approach** leverages the key constraint that each garden has at most 3 neighbors. This guarantees that for each garden, there is always at least one flower type that is not used by its neighbors. We can iterate over gardens in any order, and for each garden, pick a flower type that is not yet used by its neighbors. This greedy approach works because no garden has more than three neighbors, so at least one flower type is always available. The implementation uses an adjacency list to track garden connections, and a simple array to store flower assignments.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(4^n) | O(n) | Tries all combinations; infeasible for large n |
| Optimal | O(n + m) | O(n + m) | Greedy assignment using adjacency list, always feasible due to ≤3 neighbors |

## Algorithm Walkthrough

1. Initialize an adjacency list to represent the gardens and their connections. This allows quick lookup of neighbors for any garden.
2. Create an array `flowers` of length `n`, initialized to zero, representing the flower assigned to each garden.
3. Iterate over each garden `i` from `1` to `n`.
4. For the current garden, collect all flower types already assigned to its neighbors.
5. Select the first flower type from `1` to `4` that is not used by any neighbor. Assign it to the current garden.
6. Continue until all gardens have been assigned a flower.

**Why it works:** Because each garden has at most three neighbors, there will always be at least one flower type available that does not conflict with its neighbors. Iterating through gardens in any order guarantees a valid assignment is found in linear time relative to the number of gardens and paths.

## Python Solution

```
from typing import List

class Solution:
    def gardenNoAdj(self, n: int, paths: List[List[int]]) -> List[int]:
        # Step 1: Build the adjacency list
        graph = [[] for _ in range(n)]
        for x, y in paths:
            graph[x-1].append(y-1)
            graph[y-1].append(x-1)
        
        # Step 2: Prepare the flowers array
        flowers = [0] * n
        
        # Step 3: Assign flowers greedily
        for garden in range(n):
            used = set(flowers[neighbor] for neighbor in graph[garden] if flowers[neighbor] != 0)
            for flower_type in range(1, 5):
                if flower_type not in used:
                    flowers[garden] = flower_type
                    break
        
        return flowers
```

**Explanation:** The adjacency list `graph` maps each garden to its neighbors. We iterate over each garden and collect flower types already used by its neighbors in a set called `used`. Then we assign the first available flower type from `1` to `4`. The algorithm guarantees a valid assignment because at most three neighbors exist.

## Go Solution

```
func gardenNoAdj(n int, paths [][]int) []int {
    // Step 1: Build adjacency list
    graph := make([][]int, n)
    for _, path := range paths {
        x, y := path[0]-1, path[1]-1
        graph[x] = append(graph[x], y)
        graph[y] = append(graph[y], x)
    }

    // Step 2: Prepare flowers array
    flowers := make([]int, n)

    // Step 3: Assign flowers greedily
    for garden := 0; garden < n; garden++ {
        used := make(map[int]bool)
        for _, neighbor := range graph[garden] {
            if flowers[neighbor] != 0 {
                used[flowers[neighbor]] = true
            }
        }
        for flower := 1; flower <= 4; flower++ {
            if !used[flower] {
                flowers[garden] = flower
                break
            }
        }
    }

    return flowers
}
```

**Go-specific notes:** We use slices instead of lists, and maps to track used flower types. The zero value of the `flowers` slice represents unassigned gardens, similar to Python.

The algorithm runs efficiently because each garden only has up to 3 neighbors, making the inner loop small. Using an adjacency list avoids O(n^2) operations.
