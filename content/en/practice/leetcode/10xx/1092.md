# Shortest Common Supersequence

## Problem Understanding

The problem gives us two strings, `str1` and `str2`, and asks us to construct the shortest possible string that contains both original strings as subsequences.

A subsequence does not require characters to remain contiguous. The only requirement is that the relative order of characters must stay the same. For example, `"abc"` is a subsequence of `"axbxc"` because we can delete the extra characters and still preserve the order `a -> b -> c`.

The goal is not simply to concatenate the two strings. Instead, we want to overlap matching portions whenever possible so that the resulting string is as short as possible.

For example:

- `str1 = "abac"`
- `str2 = "cab"`

A valid answer is `"cabac"`.

Inside `"cabac"`:

- `"abac"` appears as a subsequence
- `"cab"` also appears as a subsequence

The important detail is that we are minimizing the total length of the constructed string.

The constraints are:

- `1 <= str1.length, str2.length <= 1000`
- Both strings contain lowercase English letters

A length of 1000 is large enough that exponential solutions are impossible. We need a polynomial time algorithm, most likely dynamic programming.

Several edge cases are important:

- Both strings may already be identical
- One string may already be a subsequence of the other
- The strings may share no characters at all
- Multiple shortest answers may exist, any one is acceptable
- Repeated characters can create ambiguous matching choices

These cases make greedy approaches unreliable, because local matching decisions may not produce the globally shortest result.

## Approaches

### Brute Force Approach

A brute force solution would attempt to generate all possible supersequences of the two strings and then select the shortest valid one.

One way to think about this is:

- Try every possible way of interleaving characters from `str1` and `str2`
- Check whether both strings appear as subsequences
- Keep the shortest valid result

This works because every valid shortest common supersequence must be some interleaving of the two strings while preserving order inside each string.

However, the number of interleavings grows exponentially. For strings of length `m` and `n`, there are roughly:

$$\binom{m+n}{m}$$

possible interleavings.

With lengths up to 1000, this is completely infeasible.

### Key Insight

The key observation is that overlapping matching characters reduces the total length.

This immediately connects the problem to the Longest Common Subsequence problem.

If two strings share a common subsequence, we only need to include those shared characters once in the final supersequence.

Suppose:

- `LCS(str1, str2)` has length `k`

Then:

- Total length of shortest common supersequence becomes:

$$m + n - k$$

because the shared subsequence is counted only once.

So the optimal strategy is:

1. Compute the Longest Common Subsequence
2. Merge the two strings around that shared subsequence

This reduces the problem to a classic dynamic programming formulation.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | Exponential | Exponential | Tries all interleavings of both strings |
| Optimal Dynamic Programming | O(m × n) | O(m × n) | Uses LCS to construct the shortest supersequence |

## Algorithm Walkthrough

### Step 1: Build the LCS DP Table

We create a 2D DP table where:

```
dp[i][j]
```

represents the length of the Longest Common Subsequence between:

- `str1[0:i]`
- `str2[0:j]`

The transitions are:

- If characters match:

$$dp[i][j] = 1 + dp[i-1][j-1]$$

- Otherwise:

$$dp[i][j] = \max(dp[i-1][j], dp[i][j-1])$$

This table tells us the maximum overlap between prefixes of the two strings.

### Step 2: Reconstruct the Shortest Common Supersequence

After computing the LCS table, we walk backward from `dp[m][n]`.

At each step:

1. If characters match:

- Add the character once
- Move diagonally
2. If characters differ:

- Move in the direction of the larger DP value
- Add the corresponding character to the answer

This process ensures:

- Every character from both strings appears
- Shared characters are included only once

### Step 3: Add Remaining Characters

If one string still has leftover characters after the other is exhausted:

- Append all remaining characters

These remaining characters cannot overlap with anything else.

### Step 4: Reverse the Result

Since reconstruction proceeds backward from the end of the strings, the result is built in reverse order.

Reverse the collected characters to obtain the final answer.

## Why it works

The DP table guarantees that we identify the maximum possible overlap between the two strings. During reconstruction, matching characters are included only once, while non matching characters are added in the necessary order. Since the overlap is maximized through the LCS, the total constructed string length is minimized. Therefore, the algorithm always produces a valid shortest common supersequence.

## Python Solution

```
class Solution:
    def shortestCommonSupersequence(self, str1: str, str2: str) -> str:
        m = len(str1)
        n = len(str2)

        # Build LCS table
        dp = [[0] * (n + 1) for _ in range(m + 1)]

        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if str1[i - 1] == str2[j - 1]:
                    dp[i][j] = 1 + dp[i - 1][j - 1]
                else:
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

        # Reconstruct shortest common supersequence
        i = m
        j = n
        result = []

        while i > 0 and j > 0:
            if str1[i - 1] == str2[j - 1]:
                result.append(str1[i - 1])
                i -= 1
                j -= 1
            elif dp[i - 1][j] >= dp[i][j - 1]:
                result.append(str1[i - 1])
                i -= 1
            else:
                result.append(str2[j - 1])
                j -= 1

        # Add remaining characters
        while i > 0:
            result.append(str1[i - 1])
            i -= 1

        while j > 0:
            result.append(str2[j - 1])
            j -= 1

        return "".join(reversed(result))
```

The implementation starts by building the classic LCS dynamic programming table. Each cell stores the best overlap length between prefixes of the two strings.

After the table is complete, the algorithm reconstructs the shortest common supersequence by traversing backward through the table.

When characters match, the character is added only once because both strings can share it. When characters differ, the algorithm follows the direction with the larger LCS value because that direction preserves a larger future overlap.

Finally, any leftover characters are appended, and the accumulated result is reversed because reconstruction happened from back to front.

## Go Solution

```
func shortestCommonSupersequence(str1 string, str2 string) string {
    m := len(str1)
    n := len(str2)

    // Build LCS table
    dp := make([][]int, m+1)
    for i := range dp {
        dp[i] = make([]int, n+1)
    }

    for i := 1; i <= m; i++ {
        for j := 1; j <= n; j++ {
            if str1[i-1] == str2[j-1] {
                dp[i][j] = 1 + dp[i-1][j-1]
            } else {
                if dp[i-1][j] > dp[i][j-1] {
                    dp[i][j] = dp[i-1][j]
                } else {
                    dp[i][j] = dp[i][j-1]
                }
            }
        }
    }

    // Reconstruct answer
    i := m
    j := n
    result := []byte{}

    for i > 0 && j > 0 {
        if str1[i-1] == str2[j-1] {
            result = append(result, str1[i-1])
            i--
            j--
        } else if dp[i-1][j] >= dp[i][j-1] {
            result = append(result, str1[i-1])
            i--
        } else {
            result = append(result, str2[j-1])
            j--
        }
    }

    // Add remaining characters
    for i > 0 {
        result = append(result, str1[i-1])
        i--
    }

    for j > 0 {
        result = append(result, str2[j-1])
        j--
    }

    // Reverse result
    left := 0
    right := len(result) - 1

    for left < right {
        result[left], result[right] = result[right], result[left]
        left++
        right--
    }

    return string(result)
}
```

The Go implementation follows the same logic as the Python version, but there are a few language specific details.

The DP table is built using slices of slices. The result is stored as a `[]byte` because strings in Go are immutable, and appending characters to a byte slice is efficient.

Since reconstruction happens backward, the byte slice must be reversed manually before converting it back into a string.

Go does not support Python style negative indexing or built in reverse operations, so the reversal uses a standard two pointer swap loop.

Start from `(4,3)`.

| i | j | str1[i-1] | str2[j-1] | Action | Result |
| --- | --- | --- | --- | --- | --- |
| 4 | 3 | c | b | take c from str1 | c |
| 3 | 3 | a | b | take a from str1 | ca |
| 2 | 3 | b | b | match | cab |
| 1 | 2 | a | a | match | caba |
| 0 | 1 | - | c | remaining from str2 | cabac |

The LCS length is `8`.

During reconstruction:

- Every step uses the matching branch
- Each character is added once

Final result:

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(m × n) | Filling the DP table and reconstructing the answer |
| Space | O(m × n) | DP table storage |

The dominant cost comes from building the LCS table. Since every pair of indices `(i, j)` is processed exactly once, the runtime is quadratic in the lengths of the two strings.

The reconstruction step is linear because each pointer moves only backward through the strings once.

## Edge Cases

### Identical Strings

If both strings are exactly the same, the shortest common supersequence is simply the string itself. A buggy implementation might duplicate characters unnecessarily during reconstruction.

This implementation handles the case correctly because matching characters are added only once while traversing diagonally through the DP table.

### No Common Characters

Consider:

```
str1 = "abc"
str2 = "def"
```

The LCS length is zero, meaning no overlap exists. The shortest common supersequence must therefore contain every character from both strings.

The algorithm correctly handles this by always taking characters from one side or the other until both strings are exhausted.

### One String Already Contains the Other

Consider:

```
str1 = "abcde"
str2 = "ace"
```

Here, `"ace"` is already a subsequence of `"abcde"`.

A poor implementation might still insert duplicate characters and produce a longer result.

The DP guided reconstruction avoids duplication because matching characters are merged naturally through the LCS traversal, producing `"abcde"` directly.

### Repeated Characters

Repeated characters create multiple valid LCS paths.

Example:

```
str1 = "aaab"
str2 = "abaa"
```

Different reconstruction paths may produce different shortest supersequences of the same minimum length.

The problem allows any valid shortest answer, so this implementation remains correct even when multiple optimal results exist.
