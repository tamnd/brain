## Solution

## Problem Understanding

The problem is asking us to find the **smallest positive integer** composed entirely of the digit `1` that is divisible by a given integer `k`. The input is a single integer `k`, where `1 <= k <= 10^5`. The output is the **length** of this number, not the number itself. If no such number exists, we return `-1`.

Restating the problem: we are looking for a number like `1`, `11`, `111`, `1111`, etc., and we want the **shortest** such number divisible by `k`. Since these numbers grow very quickly, they can exceed the storage limit of standard integers. This implies that we cannot construct the full number directly in code but need to rely on modular arithmetic to check divisibility.

Important edge cases include `k = 1`, where the answer is trivially `1`, and even numbers like `k = 2` or numbers divisible by 5, because a number consisting only of `1`s can never be divisible by `2` or `5`.

The problem guarantees that `k` is positive, so we do not need to handle `0` or negative numbers.

## Approaches

### Brute Force

A naive brute-force approach would generate numbers composed of `1`s sequentially and check if each is divisible by `k`. We would start with `1`, then `11`, `111`, and so on, converting them to integers and checking `num % k == 0`. This approach is correct but **impractical**, because the number quickly grows beyond the capacity of any integer type, and converting large strings to integers repeatedly is expensive. This would result in exponential time complexity with respect to the number of digits.

### Optimal Approach

The key insight is that we do not need the full number. Instead, we can **use the remainder modulo `k`**. For example, if we have a number `n` with remainder `r` modulo `k`, appending another `1` at the end is equivalent to `r * 10 + 1`. Using this property, we can build the number **digit by digit**, only tracking the remainder modulo `k`. If a remainder repeats, it means we have entered a cycle, and no divisible number exists, so we return `-1`.

This approach is efficient because we only need at most `k` iterations. Each iteration uses simple arithmetic to update the remainder and check if it reaches zero.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(10^length) | O(length) | Construct full numbers, slow and memory-heavy |
| Optimal | O(k) | O(1) | Only track remainder modulo `k`, stops if cycle detected |

## Algorithm Walkthrough

1. **Handle trivial case**: if `k` is divisible by 2 or 5, return `-1` because a number of all `1`s cannot be divisible by 2 or 5.
2. **Initialize remainder**: set `remainder = 0` and `length = 0`.
3. **Iterate until solution found**: loop from `1` to `k` (inclusive) to ensure we do not repeat remainders indefinitely.
4. **Update remainder**: at each step, compute `remainder = (remainder * 10 + 1) % k`. Increment `length`.
5. **Check divisibility**: if `remainder` becomes `0`, return `length`.
6. **Cycle detection**: if we finish `k` iterations without `remainder == 0`, return `-1`.

**Why it works**: The remainder modulo `k` after adding a digit is always between `0` and `k-1`. If a remainder repeats, the sequence will cycle indefinitely, so no solution exists. Since there are only `k` possible remainders, we are guaranteed to either find a divisible number within `k` iterations or detect that it does not exist.

## Python Solution

```
class Solution:
    def smallestRepunitDivByK(self, k: int) -> int:
        if k % 2 == 0 or k % 5 == 0:
            return -1
        
        remainder = 0
        for length in range(1, k + 1):
            remainder = (remainder * 10 + 1) % k
            if remainder == 0:
                return length
        
        return -1
```

In this implementation, we first handle the edge cases of `k` divisible by 2 or 5. We then iterate up to `k` times, each time updating the remainder using modular arithmetic. If the remainder becomes zero, the current length is the answer. If no remainder reaches zero, we return `-1`.

## Go Solution

```
func smallestRepunitDivByK(k int) int {
    if k % 2 == 0 || k % 5 == 0 {
        return -1
    }

    remainder := 0
    for length := 1; length <= k; length++ {
        remainder = (remainder*10 + 1) % k
        if remainder == 0 {
            return length
        }
    }
    return -1
}
```

In Go, the logic is identical. We use `int` types for the remainder, and handle the edge case of divisibility by 2 or 5 upfront. The loop iterates up to `k` times to either find a valid number or determine impossibility.

## Edge Cases

**Divisible by 2 or 5**: Numbers made entirely of `1`s cannot be divisible by 2 or 5. The code handles this immediately with a conditional check. This prevents unnecessary computation.

**Prime numbers requiring many digits**: For primes like 7, 13, or 23, the smallest number of `1`s divisible by the prime can be as large as `k` digits. The loop correctly iterates up to `k` times to guarantee finding the answer or detecting impossibility.

**Maximum input value**: When `k = 100000`, constructing the number directly would be infeasible due to memory and integer overflow. Using modular arithmetic allows handling even the largest input efficiently without storing the number.
