## Solution

## Problem Understanding

The problem asks us to count the number of pairs of songs whose total duration is divisible by 60. Each element in the input array `time` represents the duration of a single song in seconds. We are looking for all pairs of indices `(i, j)` such that `i < j` and `(time[i] + time[j]) % 60 == 0`. The output is a single integer, representing the total number of such pairs.

The input constraints are important: the length of `time` can be up to 60,000, and each song duration is between 1 and 500 seconds. This tells us that a naive approach that checks all pairs would require `O(n^2)` operations, which could be up to roughly 3.6 billion iterations and is too slow for large inputs. Therefore, we need an optimized approach.

Important edge cases include: songs with durations that are already multiples of 60, pairs that sum to exactly 60, and arrays with repeated durations.

## Approaches

The brute-force approach is straightforward: iterate over every possible pair of songs `(i, j)` and check whether their sum is divisible by 60. While this guarantees correctness, it requires `O(n^2)` time because we examine every pair, which is infeasible for `n = 60,000`.

The key observation for an optimal solution is to use modular arithmetic. If we compute `time[i] % 60` for each song, the problem reduces to finding two numbers whose remainders sum to 60 (or sum to 0 if both are multiples of 60). We can use a hash map (or an array of size 60) to count occurrences of each remainder and then efficiently compute the number of valid pairs. This reduces the complexity to `O(n)` time with `O(60)` space, since there are only 60 possible remainders.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Check all pairs `(i, j)` for divisibility by 60 |
| Optimal | O(n) | O(60) | Use remainder counts to find complementary pairs efficiently |

## Algorithm Walkthrough

1. Initialize an array `remainders` of size 60 with all zeros. Each index `i` will count the number of songs with `time % 60 == i`.
2. Initialize a variable `count` to zero to track the number of valid pairs.
3. Iterate through each song duration in the input array `time`.
4. Compute the remainder `r = time[i] % 60`.
5. Determine the complementary remainder `comp = (60 - r) % 60`. This is the remainder needed from a previous song to form a sum divisible by 60.
6. Increment `count` by the number of previous songs that had remainder `comp`.
7. Increment the count of remainder `r` in the `remainders` array.
8. After processing all songs, `count` contains the total number of pairs divisible by 60.

**Why it works:** By counting the remainders and using modular arithmetic, every time we encounter a new song, we know exactly how many previous songs can pair with it to make a total divisible by 60. This ensures all valid pairs are counted exactly once.

## Python Solution

```
from typing import List

class Solution:
    def numPairsDivisibleBy60(self, time: List[int]) -> int:
        remainders = [0] * 60
        count = 0
        for t in time:
            r = t % 60
            comp = (60 - r) % 60
            count += remainders[comp]
            remainders[r] += 1
        return count
```

In the Python code, `remainders` keeps track of how many times each possible remainder has appeared. For each song, we calculate the remainder and its complementary remainder. The count is updated based on how many previous songs could pair with the current one. Finally, we update the remainder count for the current song.

## Go Solution

```
func numPairsDivisibleBy60(time []int) int {
    remainders := make([]int, 60)
    count := 0
    for _, t := range time {
        r := t % 60
        comp := (60 - r) % 60
        count += remainders[comp]
        remainders[r]++
    }
    return count
}
```

The Go implementation is similar to Python. We use a slice of size 60 to store remainder counts. The loop iterates over the input array, calculating the remainder and complementary remainder, updating the pair count, and incrementing the remainder count. Go handles slices efficiently, and integer overflow is not a concern for the given constraints.

## Complexity Analysis

The time complexity is linear because we loop through the array once. Space complexity is constant because the `remainders` array size does not depend on the input size.

## Edge Cases

One important edge case is when the array contains only one element. Since a pair requires at least two elements, the output must be zero. Another edge case is when all elements are multiples of 60. In this case, every pair of songs forms a valid sum, and the code correctly counts all combinations using the remainder counting array. A third edge case is when the array contains repeated remainders but not multiples of 60. The algorithm correctly pairs complementary remainders using modular arithmetic, ensuring that no valid pair is missed and no invalid pair is counted.
