## Solution

## Problem Understanding

The problem gives us a string `text` and a list of unique words called `words`. Our task is to find every substring inside `text` that exactly matches one of the strings in `words`.

For every valid match, we must return the starting index `i` and ending index `j` such that the substring `text[i:j+1]` exists in the dictionary of words. The result must be sorted first by the starting index, and if two pairs share the same starting index, then by the ending index.

For example, if `text = "ababa"` and `words = ["aba", "ab"]`, then:

- `"ab"` appears at indices `[0,1]`
- `"aba"` appears at `[0,2]`
- `"ab"` appears again at `[2,3]`
- `"aba"` appears again at `[2,4]`

The important detail is that overlapping matches are allowed. A substring used in one answer does not block another substring from using the same characters.

The constraints are relatively small:

- `text.length <= 100`
- `words.length <= 20`
- `words[i].length <= 50`

These limits tell us that even moderately inefficient solutions can still pass. However, the problem is categorized under Trie and String matching, so there is value in discussing both a straightforward brute-force solution and a more optimized approach.

Several edge cases are important:

- Multiple words can start at the same index.
- Matches may overlap heavily.
- Some words may be prefixes of other words.
- A character in `text` may begin no valid word at all.
- The same word can appear multiple times throughout the string.
- The input guarantees all words are unique, which simplifies lookup logic.

## Approaches

### Brute Force Approach

The simplest solution is to examine every possible substring of `text` and check whether that substring exists in `words`.

We can store all words in a hash set for constant-time membership checks. Then, for every starting index `i`, we generate every ending index `j`, build the substring `text[i:j+1]`, and check whether it exists in the set.

This works because every possible substring is considered exactly once. If a substring belongs to `words`, we record its indices.

However, this approach performs unnecessary work. Most substrings will never match any word, and repeatedly constructing substrings increases runtime overhead.

Even though the constraints are small enough for this solution to pass, it is not the most efficient or elegant approach.

### Optimal Approach Using a Trie

The key observation is that we only care about substrings that could form one of the words in `words`.

Instead of generating all substrings, we can build a Trie from the dictionary words. A Trie allows us to efficiently check prefixes character by character.

For every starting position in `text`, we walk forward through the string while simultaneously traversing the Trie:

- If the current character does not exist in the Trie path, we stop immediately.
- If we reach a Trie node marked as a complete word, we record the current index pair.

This avoids exploring substrings that cannot possibly match any word.

The Trie is especially useful because many words may share prefixes. Rather than repeatedly comparing entire strings, we reuse prefix information efficiently.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(m) | Generates every substring and checks membership |
| Optimal Trie Solution | O(n × L) | O(total word length) | Stops early when prefixes fail |

Here:

- `n` is the length of `text`
- `L` is the maximum word length
- `m` is the number of words

## Algorithm Walkthrough

### Optimal Trie-Based Algorithm

1. Create a Trie data structure.

Each Trie node stores:

- A mapping from characters to child nodes
- A boolean indicating whether the path forms a complete word

The Trie is useful because it allows prefix matching while scanning the text.
2. Insert every word from `words` into the Trie.

For each character in a word:

- Move to the corresponding child node
- Create a new node if necessary

After inserting the final character, mark the node as a complete word.
3. Initialize an empty result list.

This list will store all valid index pairs.
4. Iterate through every starting index `start` in `text`.

We attempt to find all words beginning at this position.
5. Begin traversing the Trie from the root.

Starting from `text[start]`, move forward character by character.
6. For each character position `end`:

- If the current character does not exist in the Trie node's children, stop immediately.
- Otherwise move deeper into the Trie.

This early termination is the main optimization.
7. Whenever a Trie node is marked as a complete word:

Add `[start, end]` to the result list.

This means the substring `text[start:end+1]` exists in `words`.
8. Continue scanning until either:

- The Trie path fails
- The end of the string is reached
9. Return the result list.

Because we scan starting indices in increasing order and ending indices in increasing order, the output is already sorted correctly.

### Why it works

The Trie guarantees that every explored substring is a valid prefix of at least one dictionary word. Whenever traversal reaches a node marked as a complete word, we know the scanned substring exactly matches a word in `words`.

Since we start traversal from every index in `text`, every possible valid match is discovered. Early termination ensures no impossible substrings are explored.

## Python Solution

```
from typing import List

class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_word = False

class Solution:
    def indexPairs(self, text: str, words: List[str]) -> List[List[int]]:
        root = TrieNode()

        # Build Trie
        for word in words:
            node = root

            for char in word:
                if char not in node.children:
                    node.children[char] = TrieNode()

                node = node.children[char]

            node.is_word = True

        result = []

        # Search from every starting position
        for start in range(len(text)):
            node = root

            for end in range(start, len(text)):
                char = text[end]

                if char not in node.children:
                    break

                node = node.children[char]

                if node.is_word:
                    result.append([start, end])

        return result
```

The implementation begins by defining a `TrieNode` class. Each node contains a dictionary of child nodes and a boolean flag indicating whether the current path forms a complete word.

The Trie is built by inserting every word character by character. Shared prefixes automatically reuse existing nodes, which is one of the major advantages of the Trie structure.

After construction, the algorithm scans the string from every possible starting index. For each starting position, traversal begins at the Trie root and moves forward through the text.

If a character cannot continue the current Trie path, traversal stops immediately because no longer substring can possibly match.

Whenever traversal reaches a node marked as `is_word`, the current substring matches one of the dictionary words, so the index pair is appended to the result.

The output order is naturally sorted because the outer loop processes starting indices in ascending order and the inner loop processes ending indices in ascending order.

## Go Solution

```
package main

type TrieNode struct {
	children map[byte]*TrieNode
	isWord   bool
}

func newTrieNode() *TrieNode {
	return &TrieNode{
		children: make(map[byte]*TrieNode),
	}
}

func indexPairs(text string, words []string) [][]int {
	root := newTrieNode()

	// Build Trie
	for _, word := range words {
		node := root

		for i := 0; i < len(word); i++ {
			char := word[i]

			if _, exists := node.children[char]; !exists {
				node.children[char] = newTrieNode()
			}

			node = node.children[char]
		}

		node.isWord = true
	}

	result := [][]int{}

	// Search from every starting index
	for start := 0; start < len(text); start++ {
		node := root

		for end := start; end < len(text); end++ {
			char := text[end]

			nextNode, exists := node.children[char]

			if !exists {
				break
			}

			node = nextNode

			if node.isWord {
				result = append(result, []int{start, end})
			}
		}
	}

	return result
}
```

The Go implementation follows the same logic as the Python solution but uses explicit struct definitions and maps.

Instead of Python dictionaries, Go uses `map[byte]*TrieNode` for child pointers. Since the input only contains lowercase English letters, indexing by `byte` is sufficient and efficient.

Slices are used for the result list, and appending index pairs works naturally with `append`.

Unlike Python, Go requires explicit memory allocation for Trie nodes using helper constructors.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n × L) | For each starting position, traversal continues at most the maximum word length |
| Space | O(total word length) | Trie stores every character from all words |

The Trie allows early termination whenever a prefix fails. Since traversal from each starting index cannot exceed the longest word length, the runtime becomes proportional to `n × L` instead of checking all possible substrings.

The space complexity comes from storing all characters of every word in the Trie. Shared prefixes reduce duplicate storage.

## Edge Cases

One important edge case occurs when matches overlap. For example, in `"ababa"` with words `["ab", "aba"]`, the substring `"aba"` appears at both `[0,2]` and `[2,4]`. A flawed implementation might skip overlapping regions after finding a match. This solution correctly handles overlaps because it independently starts Trie traversal from every index in the string.

Another important case is when several words share prefixes. Consider `["a", "aa", "aaa"]`. A naive implementation may stop after finding the shortest word and fail to continue exploring longer matches. The Trie solution continues traversal even after discovering a valid word, ensuring all longer matches are also recorded.

A third edge case involves prefixes that almost match but fail near the end. For example, if the text contains `"abcd"` and the dictionary contains `"abce"`, traversal should terminate immediately once the mismatch occurs at the final character. The Trie naturally handles this by stopping whenever a character path does not exist.

Finally, single-character words and very short texts can expose indexing bugs. Because the implementation carefully uses inclusive end indices and scans character by character, cases like `"a"` with `["a"]` correctly return `[[0,0]]`.
