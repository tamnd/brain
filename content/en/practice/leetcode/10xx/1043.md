## Solution

## Problem Understanding

The problem asks us to partition an array of integers into contiguous subarrays, where each subarray can have a maximum length of `k`. Once a subarray is determined, all elements in that subarray are changed to the maximum value found in the subarray. The goal is to find a partitioning that maximizes the sum of the resulting array.

The input `arr` is an array of integers with length between 1 and 500, and each element is non-negative up to 10^9. The integer `k` determines the maximum allowed size of each contiguous subarray, and it is always at least 1 and at most the length of the array.

The expected output is a single integer representing the largest sum obtainable after the array has been partitioned and values replaced. The problem guarantees that this sum fits within a 32-bit integer.

Important edge cases include:

1. Arrays with length 1, where partitioning is trivial.
2. Arrays where all elements are equal or increasing/decreasing, which could create multiple valid partitions.
3. Cases where `k` equals the length of the array, meaning the entire array could be one subarray.

Understanding these cases is critical for avoiding off-by-one errors and ensuring correct subarray boundaries.

## Approaches

### Brute-Force Approach

The brute-force approach is to try every possible partition of the array. For each subarray of length up to `k`, calculate the sum after setting its values to the maximum in that subarray. Recursively compute the sum for the remaining elements and return the maximum sum across all possible partitions.

This method is correct because it explores all potential partitions, ensuring that the maximum sum is found. However, it is extremely slow because the number of possible partitions grows exponentially with the length of the array, which is impractical for `arr.length` up to 500.

### Optimal Approach

The key observation is that the problem exhibits **overlapping subproblems** and **optimal substructure**, which makes it a perfect candidate for **dynamic programming**. Instead of trying every combination, we can compute the maximum sum for the first `i` elements by considering the last partition of size `1` to `k`. For each ending index `i`, we keep track of the maximum element in the last partition and calculate the sum accordingly. This reduces the problem to a linear scan with nested checks up to `k`, giving a much faster solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(k^n) | O(n) | Recursively tries all partitions, exponential time |
| Optimal | O(n*k) | O(n) | Dynamic programming using max for last partition, efficient |

## Algorithm Walkthrough

1. Initialize a DP array `dp` where `dp[i]` represents the maximum sum achievable using the first `i` elements of `arr`.
2. Iterate over the array from index `0` to `n-1`.
3. For each position `i`, consider all possible partition sizes `j` from `1` to `k`, ensuring `i-j+1 >= 0`.
4. For each partition, find the maximum element in the subarray `arr[i-j+1:i+1]`.
5. Update `dp[i]` as the maximum between its current value and `dp[i-j] + max_in_partition * j` (or just `max_in_partition * j` if the partition starts at index 0).
6. Return `dp[n-1]` as the result.

**Why it works:** At each index `i`, `dp[i]` contains the best sum achievable for all partitions ending at `i`. By considering every possible partition length up to `k`, we ensure that no potential maximum sum is missed. The DP recurrence guarantees correctness because each step builds optimally on the previous solutions.

## Python Solution

```
from typing import List

class Solution:
    def maxSumAfterPartitioning(self, arr: List[int], k: int) -> int:
        n = len(arr)
        dp = [0] * n
        
        for i in range(n):
            current_max = 0
            for j in range(1, min(k, i + 1) + 1):
                current_max = max(current_max, arr[i - j + 1])
                if i - j >= 0:
                    dp[i] = max(dp[i], dp[i - j] + current_max * j)
                else:
                    dp[i] = max(dp[i], current_max * j)
        
        return dp[-1]
```

This code initializes the DP array and iterates over each index. For each index, it checks all valid partition sizes up to `k`. `current_max` keeps track of the largest value in the current partition, allowing us to calculate the sum efficiently. The `dp` array accumulates the maximum sums incrementally.

## Go Solution

```
func maxSumAfterPartitioning(arr []int, k int) int {
    n := len(arr)
    dp := make([]int, n)
    
    for i := 0; i < n; i++ {
        currentMax := 0
        for j := 1; j <= k && i-j+1 >= 0; j++ {
            if arr[i-j+1] > currentMax {
                currentMax = arr[i-j+1]
            }
            if i-j >= 0 {
                if dp[i-j]+currentMax*j > dp[i] {
                    dp[i] = dp[i-j] + currentMax*j
                }
            } else {
                if currentMax*j > dp[i] {
                    dp[i] = currentMax * j
                }
            }
        }
    }
    
    return dp[n-1]
}
```

The Go solution mirrors the Python solution closely. We use a slice `dp` to store the maximum sums. Indexing and slice bounds are carefully handled to avoid out-of-range errors. Multiplications are done safely, and Go automatically handles 32-bit integer limits.
