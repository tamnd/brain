## Solution

## Problem Understanding

The problem is asking us to find the largest sum of two distinct elements in an array such that the sum is strictly less than a given integer `k`. The array `nums` contains integers, and the condition `i < j` ensures that we only consider pairs of elements at different positions. The output is a single integer, which is either the maximum valid sum or `-1` if no such pair exists.

The input represents a typical integer array of length between 1 and 100, with values from 1 to 1000. The value of `k` ranges from 1 to 2000, so all sums will comfortably fit within standard integer types in both Python and Go.

Important edge cases include situations where all sums exceed `k` (requiring a return value of `-1`), arrays with only one element (no valid pairs exist), and arrays where multiple sums are valid but only the largest one matters.

## Approaches

The brute-force approach involves checking every possible pair `(i, j)` in the array. We calculate `nums[i] + nums[j]` for each pair and track the maximum sum that is less than `k`. This method is guaranteed to find the correct answer because it exhaustively explores all possibilities. However, it is inefficient, with a time complexity of O(n²), which is acceptable for n ≤ 100 but not ideal.

The optimal approach leverages sorting and the two-pointer technique. By first sorting the array, we can start with one pointer at the beginning and another at the end. We evaluate the sum of the two elements. If the sum is greater than or equal to `k`, we move the right pointer left to decrease the sum. If the sum is less than `k`, we record it as a potential maximum and move the left pointer right to try a larger sum. Sorting enables us to efficiently explore all valid sums in O(n log n) time for sorting plus O(n) for the two-pointer search.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(1) | Check every pair explicitly |
| Optimal | O(n log n) | O(1) | Sort array and use two-pointer search |

## Algorithm Walkthrough

1. **Sort the array** in ascending order. Sorting allows us to reason about sums: larger numbers are at the end, smaller at the beginning, so we can adjust pointers strategically.
2. **Initialize two pointers**: `left` at index 0 and `right` at the last index of the array.
3. **Initialize a variable `max_sum`** to -1. This will store the maximum sum found that is less than `k`.
4. **Iterate while `left < right`**:

- Calculate `current_sum = nums[left] + nums[right]`.
- If `current_sum` is greater than or equal to `k`, decrement the `right` pointer to reduce the sum.
- If `current_sum` is less than `k`, update `max_sum` if `current_sum` is larger than the current `max_sum`, then increment the `left` pointer to try a larger sum.
5. **Return `max_sum`** once the iteration ends.

**Why it works:** Sorting ensures that moving the left pointer right increases sums and moving the right pointer left decreases sums. This guarantees that we explore all possible sums efficiently without missing the maximum valid sum.

## Python Solution

```
from typing import List

class Solution:
    def twoSumLessThanK(self, nums: List[int], k: int) -> int:
        nums.sort()
        left, right = 0, len(nums) - 1
        max_sum = -1
        
        while left < right:
            current_sum = nums[left] + nums[right]
            if current_sum < k:
                max_sum = max(max_sum, current_sum)
                left += 1
            else:
                right -= 1
        
        return max_sum
```

The Python code first sorts the array, then uses two pointers to efficiently find the maximum sum less than `k`. If the current sum is valid, we potentially update `max_sum` and move the left pointer to explore larger sums. If the sum is too large, the right pointer moves left to reduce it.

## Go Solution

```
import "sort"

func twoSumLessThanK(nums []int, k int) int {
    sort.Ints(nums)
    left, right := 0, len(nums)-1
    maxSum := -1
    
    for left < right {
        currentSum := nums[left] + nums[right]
        if currentSum < k {
            if currentSum > maxSum {
                maxSum = currentSum
            }
            left++
        } else {
            right--
        }
    }
    
    return maxSum
}
```

In Go, we explicitly sort the slice using `sort.Ints`. The two-pointer approach is identical to Python. Go requires explicit handling of integer comparisons and updates but the logic is equivalent.
