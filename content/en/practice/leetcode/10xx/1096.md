## Solution

## Problem Understanding

This problem asks us to evaluate a string expression that represents a set of words, defined by a nested grammar using braces `{}`, commas `,`, and lowercase letters. The rules of the grammar define three types of expressions: a single letter as a singleton set, comma-separated expressions as a union of sets, and concatenations of expressions as Cartesian products. Our task is to compute the set of all unique words generated by the expression and return them in sorted order.

The input is a string `expression` of length up to 60 characters containing lowercase letters, `{`, `}`, and `,`. The output is a sorted list of distinct strings that the expression can produce. Constraints ensure that the expression is valid, so we do not need to handle malformed inputs. Edge cases include deeply nested braces, repeated letters, or repeated subexpressions that should collapse into a unique set.

Understanding the problem carefully, we need to handle both union operations (from commas) and concatenation operations (implicit) while respecting brace nesting. Naively parsing character by character without considering precedence or nested structure can lead to incorrect results.

## Approaches

A brute-force approach would attempt to generate all possible expansions recursively without carefully managing concatenations and unions. This would involve enumerating all combinations for nested braces at each level, leading to exponential work as the number of nested expressions grows. While correct for small examples, it becomes inefficient for deeply nested expressions or multiple concatenations.

The optimal approach uses a **stack-based parsing method** combined with set operations. The main insight is to treat concatenation and union operations distinctly and process them in a structured way using stacks. We maintain a stack for current sets, and whenever we encounter a `{`, we start a new expression context. When encountering `}`, we combine all collected expressions using union. Commas separate expressions inside the same brace level, and implicit concatenation is handled by taking the Cartesian product of consecutive sets.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(2^n) | Recursive expansion of all possible combinations without structured parsing; can explode with nested braces. |
| Optimal | O(n * k^2) | O(n * k) | Stack-based parsing with set union and Cartesian products; n = length of expression, k = average number of words per subexpression. |

## Algorithm Walkthrough

1. Initialize a stack to store intermediate sets and a current working set as a list containing an empty string.
2. Iterate through each character in the expression.
3. If the character is `{`, push the current working set and a marker onto the stack, then reset the current working set to an empty list. This starts a new nested expression.
4. If the character is `}`, process all elements accumulated for this brace level by repeatedly popping sets from the stack and computing the Cartesian product with the current set. Finally, merge the result with the previous context.
5. If the character is `,`, treat it as a union operator within the current brace level. Merge the current working set with a saved accumulator for the current level, then reset the current working set.
6. If the character is a letter, treat it as a singleton set. Concatenate it with every string in the current working set to handle implicit concatenation.
7. After parsing the entire string, merge any remaining sets on the stack to form the final set of words.
8. Sort the resulting set and return as a list.

**Why it works:** The algorithm maintains the invariant that `current_set` always contains all possible expansions for the current context. Nested braces are handled correctly by the stack, and implicit concatenation is handled by Cartesian products. The final sorting ensures the output is in lexicographical order.

## Python Solution

```
from typing import List

class Solution:
    def braceExpansionII(self, expression: str) -> List[str]:
        def cartesian_product(set1, set2):
            if not set1:
                return set2
            if not set2:
                return set1
            return {a + b for a in set1 for b in set2}
        
        stack = []
        current_set = {""}
        i = 0
        while i < len(expression):
            char = expression[i]
            if char == '{':
                stack.append(current_set)
                stack.append('{')
                current_set = {""}
            elif char == '}':
                temp = set()
                while stack and stack[-1] != '{':
                    temp = temp.union(stack.pop())
                stack.pop()  # pop the '{'
                prev_set = stack.pop()
                current_set = cartesian_product(prev_set, current_set.union(temp))
            elif char == ',':
                stack.append(current_set)
                current_set = {""}
            else:
                current_set = {s + char for s in current_set}
            i += 1

        result = set()
        while stack:
            result = result.union(stack.pop())
        result = result.union(current_set)
        return sorted(result)
```

**Implementation Walkthrough:** We define a helper `cartesian_product` function to combine two sets of strings with concatenation. The main loop iterates through the expression, managing braces using a stack and handling unions and concatenations. After parsing, we union all remaining sets from the stack with the current set to produce the final unique words. Sorting at the end ensures lexicographical order.

## Go Solution

```
package main

import (
	"sort"
)

func braceExpansionII(expression string) []string {
	var stack []map[string]struct{}
	currentSet := map[string]struct{}{ "": {} }
	cartesianProduct := func(set1, set2 map[string]struct{}) map[string]struct{} {
		if len(set1) == 0 {
			return set2
		}
		if len(set2) == 0 {
			return set1
		}
		result := make(map[string]struct{})
		for a := range set1 {
			for b := range set2 {
				result[a+b] = struct{}{}
			}
		}
		return result
	}

	for i := 0; i < len(expression); i++ {
		char := expression[i]
		switch char {
		case '{':
			stack = append(stack, currentSet)
			stack = append(stack, map[string]struct{}{"{": {}})
			currentSet = map[string]struct{}{ "": {} }
		case '}':
			temp := make(map[string]struct{})
			for len(stack) > 0 {
				top := stack[len(stack)-1]
				if _, ok := top["{"]; ok {
					stack = stack[:len(stack)-1]
					break
				}
				for k := range top {
					temp[k] = struct{}{}
				}
				stack = stack[:len(stack)-1]
			}
			prevSet := stack[len(stack)-1]
			stack = stack[:len(stack)-1]
			for k := range cartesianProduct(prevSet, currentSet) {
				temp[k] = struct{}{}
			}
			currentSet = temp
		case ',':
			stack = append(stack, currentSet)
			currentSet = map[string]struct{}{ "": {} }
		default:
			nextSet := make(map[string]struct{})
			for s := range currentSet {
				nextSet[s+string(char)] = struct{}{}
			}
			currentSet = nextSet
		}
	}

	resultSet := make(map[string]struct{})
	for len(stack) > 0 {
		for k := range stack[len(stack)-1] {
			resultSet[k] = struct{}{}
		}
		stack = stack[:len(stack)-1]
	}
	for k := range currentSet {
		resultSet[k] = struct{}{}
	}

	result := make([]string, 0, len(resultSet))
	for k := range resultSet {
		if k != "" {
			result = append(result, k)
		}
	}
	sort.Strings(result)
	return result
}
```

**Go Notes:** Go requires explicit handling of sets using maps. We represent a set of strings as `map[string]struct{}`. The Cartesian product and union operations are implemented similarly to Python. We also handle empty string initialization carefully to allow concatenation.
