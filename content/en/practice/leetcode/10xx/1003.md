## Solution

## Problem Understanding

The problem asks us to determine if a given string `s` can be generated by repeatedly inserting the substring `"abc"` into an initially empty string. In other words, we start with an empty string `t = ""` and perform zero or more insertions of `"abc"` at any position. The string `s` is **valid** if it can be obtained through such a sequence of insertions.

The input `s` is a string of length up to 20,000 consisting only of the characters `'a'`, `'b'`, and `'c'`. The expected output is a boolean value: `true` if the string is valid according to the rules, and `false` otherwise.

Key points include that `"abc"` must appear in the correct order and cannot be rearranged. Strings with characters out of order, extra characters, or missing characters cannot be valid. Edge cases that could trip up a naive implementation include strings like `"a"`, `"ab"`, `"acb"`, or strings with multiple nested `"abc"` sequences like `"aabcbc"` or `"abcabcababcc"`. The constraints indicate that we need an algorithm that is linear or near-linear in time because the string length can be as large as 20,000.

## Approaches

A **brute-force approach** would attempt to generate all possible strings by repeatedly inserting `"abc"` at every possible position, then check if any of these strings equal `s`. While this guarantees correctness because it explores every possibility, it is highly inefficient. The number of possible insertions grows exponentially with the string length, making it completely impractical for the upper bound of 20,000 characters.

The **key insight** for an optimal solution is to recognize that `"abc"` must appear as a contiguous sequence in `s`. Therefore, we can process the string sequentially and use a **stack** to track characters as we attempt to form valid `"abc"` sequences. Each time we encounter a `'c'`, we check if the two preceding characters on the stack are `'a'` and `'b'`. If so, we remove the three characters from the stack as a single valid `"abc"` block. If not, the string cannot be formed by repeated `"abc"` insertions. At the end, the stack should be empty for the string to be valid.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(3^n) | O(n) | Generates all possible insertions of "abc", exponential and infeasible |
| Optimal | O(n) | O(n) | Uses a stack to track valid "abc" sequences while iterating once through the string |

## Algorithm Walkthrough

1. Initialize an empty stack to keep track of characters as we scan the string.
2. Iterate through each character in the string `s`.
3. For each character, push it onto the stack.
4. After pushing a character, check if the top three characters on the stack form the sequence `"abc"`.
5. If they do, pop these three characters from the stack. This simulates removing a valid `"abc"` insertion.
6. Continue this process until all characters in `s` are processed.
7. After processing the string, check if the stack is empty. If it is, return `true`, indicating that the string is valid. If not, return `false`.

**Why it works:** The algorithm works because a valid string is made entirely of concatenated `"abc"` sequences. By using a stack, we maintain an invariant that any sequence that cannot form a valid `"abc"` will remain on the stack, and the string will ultimately be invalid if the stack is non-empty. Each valid `"abc"` sequence is removed immediately, preventing interference with future sequences.

## Python Solution

```
class Solution:
    def isValid(self, s: str) -> bool:
        stack = []
        for char in s:
            stack.append(char)
            if len(stack) >= 3 and stack[-3:] == ['a', 'b', 'c']:
                stack.pop()
                stack.pop()
                stack.pop()
        return not stack
```

This implementation iterates through the string once, pushing each character onto the stack. Whenever the top three characters form `"abc"`, they are removed, effectively "consuming" a valid insertion. If at the end the stack is empty, all characters were part of valid `"abc"` sequences.

## Go Solution

```
func isValid(s string) bool {
    stack := []byte{}
    for i := 0; i < len(s); i++ {
        stack = append(stack, s[i])
        n := len(stack)
        if n >= 3 && stack[n-3] == 'a' && stack[n-2] == 'b' && stack[n-1] == 'c' {
            stack = stack[:n-3]
        }
    }
    return len(stack) == 0
}
```

In Go, we use a byte slice as the stack. We append each character and check the top three characters to see if they form `"abc"`. Removing three elements is done using slice truncation. The algorithm is nearly identical to Python but handles Go-specific types and slice operations.

## Worked Examples

### Example 1: `s = "aabcbc"`

| Step | Stack | Action |
| --- | --- | --- |
| a | ['a'] | push 'a' |
| a | ['a','a'] | push 'a' |
| b | ['a','a','b'] | push 'b' |
| c | ['a','a','b','c'] | top three are 'a','b','c', pop them → ['a'] |
| b | ['a','b'] | push 'b' |
| c | ['a','b','c'] | top three are 'a','b','c', pop them → [] |

Stack is empty → valid string → return `true`.

### Example 2: `s = "abcabcababcc"`

| Step | Stack | Action |
| --- | --- | --- |
| a b c | [] → ['a','b','c'] → pop → [] | first "abc" |
| a b c | [] → ['a','b','c'] → pop → [] | second "abc" |
| a b a b c c | sequential pushes → remove each "abc" when formed | final stack empty → valid |

### Example 3: `s = "abccba"`

Stack never fully empties because the order of characters prevents forming consecutive `"abc"` sequences. → return `false`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | We iterate through the string once and each character is pushed/popped at most once. |
| Space | O(n) | In the worst case, no "abc" is formed and the stack stores all characters. |

The linear time complexity comes from single traversal and constant-time stack operations. Space is linear in the worst case, but often much smaller if many "abc" sequences are removed.

## Edge Cases

One important edge case is a string that contains only a single `"abc"`. This tests if the implementation correctly handles the simplest valid input. The stack-based approach works because it pushes each character and immediately removes the `"abc"` when completed, leaving an empty stack.

A second edge case is strings that start or end with incomplete `"abc"` sequences, such as `"ab"` or `"a"`. These could be incorrectly accepted if we only check for the presence of `"abc"` anywhere. By checking the stack at the end, we ensure incomplete sequences lead to `false`.

A third edge case is deeply nested or overlapping sequences, such as `"aaabcbcc"`. This tests if the algorithm correctly handles consecutive removals of `"abc"` and maintains the proper order. The stack approach ensures that only contiguous `"abc"` sequences are removed and any leftover characters are detected.
