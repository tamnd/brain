## Solution

## Problem Understanding

The problem gives us a string called `text` and two target words, `first` and `second`. We need to search through the text and find every occurrence where the words appear consecutively in the order:

```
first second third
```

Whenever we find such a sequence, we must collect the value of `third` and return all matching third words in an array.

The important detail is that the words must appear immediately next to each other in the exact order. We are not searching for words that appear somewhere later in the sentence. The pattern must be consecutive.

For example, suppose the input is:

```
text = "alice is a good girl she is a good student"
first = "a"
second = "good"
```

The text contains these word triples:

```
a good girl
a good student
```

In both cases, `"a"` is immediately followed by `"good"`, so we collect the words after `"good"`, which are `"girl"` and `"student"`.

The constraints are small. The length of the text is at most 1000 characters, and every word is separated by exactly one space. This tells us we do not need advanced string matching algorithms like KMP or suffix arrays. A straightforward scan through the words is completely sufficient.

The input also guarantees several useful properties:

- There are no leading or trailing spaces.
- Words are separated by exactly one space.
- All characters are lowercase English letters and spaces.

These guarantees simplify parsing because splitting the string by spaces produces a clean array of words without empty entries.

Several edge cases are important:

- The pattern may appear multiple times.
- Matching sequences may overlap.
- The text may contain fewer than three words.
- There may be no valid matches at all.
- The same third word may appear multiple times, and duplicates should be preserved.

A naive implementation could accidentally skip overlapping matches or access indices beyond the end of the array. Careful iteration bounds solve these issues.

## Approaches

### Brute Force Approach

The brute force solution checks every possible group of three consecutive words in the text.

First, split the text into an array of words. Then, for every index `i`, examine:

```
words[i], words[i + 1], words[i + 2]
```

If `words[i] == first` and `words[i + 1] == second`, then append `words[i + 2]` to the answer.

This approach is correct because every valid occurrence of the pattern must appear as some consecutive triple in the word array. By checking every possible triple exactly once, we guarantee that all matches are found.

Although this is called the brute force solution, it is already efficient enough for the problem constraints because the input size is small.

### Optimal Approach

The optimal approach is essentially the same scanning idea, but framed more cleanly as a sliding window over the word list.

The key insight is that we only care about consecutive groups of three words. There is no need for nested loops or substring generation. Once the text is split into words, we can process the array in a single linear pass.

At each position:

- Compare the current word with `first`
- Compare the next word with `second`
- If both match, collect the third word

This gives an `O(n)` solution, where `n` is the number of words.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(n) | Checks every consecutive triple after splitting the text |
| Optimal | O(n) | O(n) | Single linear scan through the word array |

## Algorithm Walkthrough

1. Split the input string into an array of words.

We use the built in split operation because the problem guarantees exactly one space between words. This converts the problem into array traversal rather than raw string manipulation.
2. Create an empty result list.

This list stores every valid third word we discover.
3. Iterate through the word array from index `0` to `len(words) - 3`.

We stop at `len(words) - 3` because each check needs three consecutive words:

```
words[i]
words[i + 1]
words[i + 2]
```
4. For each position, compare the first two words with the target pattern.

Specifically, check:

```
words[i] == first
words[i + 1] == second
```
5. If both conditions are true, append the third word.

The third word is:

```
words[i + 2]
```
6. Continue scanning until all triples have been processed.
7. Return the result list.

### Why it works

The algorithm examines every possible consecutive group of three words exactly once. Any valid occurrence of `"first second third"` must appear in one of these triples. Whenever the first two words match the required pattern, the algorithm records the third word. Since no triples are skipped or repeated, the final result contains exactly all valid third words.

## Python Solution

```
from typing import List

class Solution:
    def findOcurrences(self, text: str, first: str, second: str) -> List[str]:
        words = text.split()
        result = []

        for i in range(len(words) - 2):
            if words[i] == first and words[i + 1] == second:
                result.append(words[i + 2])

        return result
```

The implementation begins by splitting the input text into individual words using `split()`. Because the problem guarantees proper spacing, this produces a clean list of tokens.

Next, an empty list called `result` is created to store matching third words.

The loop iterates through all valid starting positions for a three word sequence. The range ends at `len(words) - 2` because we must safely access `i + 2`.

Inside the loop, the algorithm compares the current pair of words with `first` and `second`. If both match, the next word is appended to the answer list.

Finally, the result list is returned.

## Go Solution

```
package main

import "strings"

func findOcurrences(text string, first string, second string) []string {
	words := strings.Split(text, " ")
	result := []string{}

	for i := 0; i < len(words)-2; i++ {
		if words[i] == first && words[i+1] == second {
			result = append(result, words[i+2])
		}
	}

	return result
}
```

The Go implementation follows the same logic as the Python version. The text is split using `strings.Split`, producing a slice of words.

An empty slice is initialized for storing results. The loop scans every consecutive triple, and matching third words are appended using Go's built in `append` function.

Unlike Python lists, Go slices are dynamically resized arrays. Returning an empty slice is perfectly valid when no matches exist.

Integer overflow is not a concern because the input size is extremely small.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each word triple is checked once |
| Space | O(n) | Splitting the text creates an array of words |

The algorithm performs a single pass through the word list. Each iteration does constant time comparisons, so the total runtime grows linearly with the number of words.

The space complexity is linear because splitting the text stores all words in memory. The output list itself can also contain up to `O(n)` words in the worst case.

## Edge Cases

One important edge case occurs when the text contains fewer than three words. In this situation, no valid `"first second third"` pattern can exist. A careless implementation might attempt to access indices beyond the array bounds. The loop condition `range(len(words) - 2)` prevents this problem because the loop never executes when there are not enough words.

Another important case involves overlapping matches. Consider:

```
"a a a a"
```

with:

```
first = "a"
second = "a"
```

There are two valid matches:

```
a a a
a a a
```

A buggy implementation might skip the second match after finding the first one. Since this solution advances one index at a time, all overlapping triples are examined correctly.

A third edge case happens when no valid pattern exists anywhere in the text. The algorithm should return an empty array, not `None` or an error. Because the result list starts empty and only receives values when matches occur, the implementation naturally handles this case correctly.

Another subtle case involves repeated third words. The problem does not require uniqueness. If the same third word appears multiple times in matching triples, all occurrences must be included. The implementation appends every match independently, preserving duplicates exactly as required.
