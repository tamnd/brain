## Solution

## Problem Understanding

The problem asks us to count the number of non-empty contiguous subarrays of a given array `nums` where the leftmost element of the subarray is **not larger than any other element** in the subarray. In other words, for a valid subarray starting at index `i` and ending at index `j`, `nums[i] <= nums[k]` must hold for every index `k` between `i` and `j`.

The input `nums` is an integer array with constraints up to 50,000 elements, and each element can be as large as 100,000. This means that brute-force solutions that check every possible subarray individually would be too slow, because there could be roughly $O(n^2)$ subarrays to evaluate.

Edge cases that are important include arrays that are strictly decreasing, strictly increasing, or all elements equal. For example, in a strictly decreasing array, each element forms a valid subarray of length 1, but cannot extend beyond that. Conversely, in an array where all elements are equal, every contiguous subarray is valid. The problem guarantees at least one element in the array, so we do not need to handle empty arrays.

## Approaches

A naive brute-force approach would iterate through every possible subarray. For each subarray starting at index `i`, it would check every subsequent element to see if it is greater than or equal to `nums[i]`. If any element is smaller, the subarray is invalid. This approach is correct but inefficient because it requires $O(n^2)$ subarray checks and potentially another $O(n)$ to validate each subarray, resulting in $O(n^3)$ worst-case time complexity.

The key observation that allows an optimal solution is that the problem is inherently suited for a **monotonic stack**. Specifically, if we iterate through the array from right to left, we can use a stack to maintain elements in **non-decreasing order**. For each element `nums[i]`, we can calculate the number of valid subarrays starting at `i` by checking how many elements to the right are greater than or equal to `nums[i]` until a smaller element is encountered. This reduces the complexity to linear time because each element is pushed and popped from the stack at most once.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Checks each possible subarray individually |
| Optimal | O(n) | O(n) | Uses monotonic stack to count valid subarrays efficiently |

## Algorithm Walkthrough

1. Initialize a stack `stack` to keep track of elements and the count of valid subarrays starting at that element. Initialize a variable `total` to accumulate the total count of valid subarrays.
2. Iterate through the array `nums` from right to left. This ensures that for each element, we can look at elements to its right efficiently.
3. For the current element `nums[i]`, initialize a counter `count = 1` because the subarray consisting of just `nums[i]` is always valid.
4. While the stack is not empty and the top element of the stack is greater than or equal to `nums[i]`, pop the top element from the stack and add its count to `count`. This works because all these elements form valid subarrays with `nums[i]` as the leftmost element.
5. Push a tuple `(nums[i], count)` onto the stack.
6. Add `count` to `total`.
7. After iterating through all elements, return `total`.

**Why it works:** By maintaining a monotonic non-decreasing stack, we efficiently track contiguous sequences where the leftmost element is the smallest. The stack invariant ensures that every popped element corresponds to a valid subarray extension with the current element as the leftmost value. Each element is processed only once for push and pop, ensuring linear time.

## Python Solution

```
from typing import List

class Solution:
    def validSubarrays(self, nums: List[int]) -> int:
        stack = []
        total = 0
        
        for num in reversed(nums):
            count = 1
            while stack and stack[-1][0] >= num:
                count += stack.pop()[1]
            stack.append((num, count))
            total += count
        
        return total
```

The Python implementation uses a list as a stack where each element is a tuple `(value, count)`. We iterate through `nums` in reverse. For each element, we compute how many valid subarrays start with it using the stack. The `while` loop merges counts from elements greater than or equal to the current element. Finally, the sum of all counts is returned.

## Go Solution

```
func validSubarrays(nums []int) int {
	type pair struct {
		val   int
		count int
	}
	stack := []pair{}
	total := 0

	for i := len(nums) - 1; i >= 0; i-- {
		num := nums[i]
		count := 1
		for len(stack) > 0 && stack[len(stack)-1].val >= num {
			count += stack[len(stack)-1].count
			stack = stack[:len(stack)-1]
		}
		stack = append(stack, pair{num, count})
		total += count
	}
	return total
}
```

In Go, we define a `pair` struct to hold the value and count. The stack is implemented using a slice. We iterate backward over `nums` and use the same logic as Python. Pop operations are implemented by slicing, and push by `append`.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each element is pushed and popped from the stack at most once |
| Space | O(n) | The stack stores up to n elements in the worst case |

Because we process each element once and only maintain a stack of elements, the algorithm scales linearly with the size of `nums`.
