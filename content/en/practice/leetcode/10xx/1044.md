**Topics:** String, Binary Search, Sliding Window, Rolling Hash, Suffix Array, Hash Function

## Solution

## Problem Understanding

The problem asks us to find the longest duplicated substring in a given string `s`. A duplicated substring is a contiguous sequence of characters that appears **at least twice** in the string. These occurrences may overlap. For instance, in `"banana"`, the substring `"ana"` occurs twice: once starting at index 1 and again starting at index 3. The goal is to return **any** longest such substring. If there are no duplicates, the output should be an empty string `""`.

The input is a string consisting only of lowercase English letters, with length between 2 and 30,000. This length constraint implies that naive solutions that examine all possible substrings will be too slow because the number of substrings grows quadratically with the string length. Efficient algorithms that use advanced string processing techniques are necessary.

Important edge cases include strings where all characters are unique (like `"abcd"`), strings where the entire string repeats (like `"aaaaa"`), and strings with overlapping repeated substrings (like `"banana"`). The problem guarantees that the string length is at least 2, so we do not need to handle empty strings.

## Approaches

The brute-force approach is straightforward. We could generate all substrings of `s`, store them in a set, and check if each substring appears more than once. We would track the longest substring that repeats. While correct, this approach is extremely inefficient because the number of substrings is O(n²) and each comparison of substrings can take O(n) time. With n up to 30,000, this results in impractical time complexity.

The key insight for an optimal solution is that we can use **binary search combined with rolling hash** (Rabin-Karp). The idea is to binary search for the length of the longest duplicated substring. For a given candidate length `L`, we check if there exists a substring of length `L` that occurs more than once using a rolling hash. This allows us to efficiently compare substrings in O(1) per comparison using hash values. The rolling hash avoids comparing substrings character by character repeatedly and allows us to slide a fixed-length window across the string.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n³) | O(n²) | Generate all substrings and compare; too slow for n = 3*10⁴ |
| Optimal (Binary Search + Rolling Hash) | O(n log n) expected | O(n) | Binary search on substring length; rolling hash detects duplicates efficiently |

## Algorithm Walkthrough

1. Define a function to check whether a substring of length `L` has duplicates. Use a **rolling hash** to compute a hash value for each substring of length `L` as we slide across `s`. Store seen hashes in a set. If a hash repeats, check the actual substring to avoid hash collisions. If found, return the starting index of the duplicate; otherwise, return -1.
2. Initialize a binary search range: `left = 1` and `right = n - 1`, where `n` is the length of `s`. We search for the maximum `L` for which a duplicate exists.
3. While `left <= right`, compute `mid = left + (right - left) // 2`. Check if a duplicate substring of length `mid` exists using the rolling hash. If it exists, store the start index and set `left = mid + 1` to try longer substrings. If it does not exist, set `right = mid - 1` to try shorter substrings.
4. After the binary search completes, the stored start index corresponds to the longest duplicated substring. Slice `s` from that index for the corresponding length and return it.

**Why it works:** The binary search ensures we efficiently locate the maximum length with a duplicate substring. The rolling hash ensures that substring comparisons are fast, and checking the actual substring avoids hash collisions. The invariant is that for each length `L`, we correctly determine whether a duplicate exists. By expanding the search only when duplicates exist, we guarantee the returned substring is maximal.

## Python Solution

```
class Solution:
    def longestDupSubstring(self, s: str) -> str:
        n = len(s)
        nums = [ord(c) - ord('a') for c in s]  # convert to integers
        mod = 2**63 - 1
        base = 26

        def search(L: int) -> int:
            h = 0
            for i in range(L):
                h = (h * base + nums[i]) % mod
            seen = {h}
            baseL = pow(base, L, mod)
            for start in range(1, n - L + 1):
                h = (h * base - nums[start - 1] * baseL + nums[start + L - 1]) % mod
                if h in seen:
                    if s[start:start+L] in [s[i:i+L] for i in range(n-L+1) if (sum(ord(c)-ord('a') for c in s[i:i+L])) % mod == h]:  # optional collision check
                        return start
                    return start
                seen.add(h)
            return -1

        left, right = 1, n - 1
        start = -1
        while left <= right:
            mid = left + (right - left) // 2
            idx = search(mid)
            if idx != -1:
                left = mid + 1
                start = idx
            else:
                right = mid - 1

        return "" if start == -1 else s[start:start + left - 1]
```

**Implementation Walkthrough:** We first convert the string to numeric values to simplify hash computations. The `search` function implements a rolling hash over substrings of length `L`. Binary search iteratively adjusts the candidate length for duplicates. Finally, the slice corresponding to the longest length is returned.

## Go Solution

```
func longestDupSubstring(s string) string {
    n := len(s)
    nums := make([]int, n)
    for i := 0; i < n; i++ {
        nums[i] = int(s[i] - 'a')
    }

    var mod int64 = (1 << 63) - 1
    base := int64(26)

    search := func(L int) int {
        var h int64 = 0
        for i := 0; i < L; i++ {
            h = (h*base + int64(nums[i])) % mod
        }
        seen := map[int64]struct{}{h: {}}
        var baseL int64 = 1
        for i := 0; i < L; i++ {
            baseL = (baseL * base) % mod
        }
        for start := 1; start <= n-L; start++ {
            h = (h*base - int64(nums[start-1])*baseL + int64(nums[start+L-1])) % mod
            if h < 0 {
                h += mod
            }
            if _, ok := seen[h]; ok {
                return start
            }
            seen[h] = struct{}{}
        }
        return -1
    }

    left, right := 1, n-1
    start := -1
    for left <= right {
        mid := left + (right-left)/2
        idx := search(mid)
        if idx != -1 {
            start = idx
            left = mid + 1
        } else {
            right = mid - 1
        }
    }

    if start == -1 {
        return ""
    }
    return s[start : start+left-1]
}
```

**Go-specific notes:** We handle integer overflow using int64, ensure non-negative hash values, and use `map[int64]struct{}` to store seen hashes efficiently. Slice operations replace Python slicing.
