## Solution

## Problem Understanding

This problem asks us to **prune a binary tree** based on the sum of node values along root-to-leaf paths. Specifically, we need to remove nodes that are **insufficient**, meaning that every path from the root to a leaf that passes through that node has a sum strictly less than a given `limit`. The output should be the pruned tree, where all insufficient nodes have been removed.

The input is a binary tree represented by its root node and an integer `limit`. Each node has an integer value, and leaf nodes are defined as nodes without children. The constraints tell us the tree can have up to 5000 nodes and node values can range from -10^5 to 10^5, while `limit` can be between -10^9 and 10^9. This is a moderate-sized input, so we can afford a depth-first search (DFS) solution without worrying about time or stack limits.

Edge cases that could trip up a naive solution include trees with negative values, trees with a single path, trees where all nodes are insufficient, and cases where pruning a node makes its parent a new leaf that is also insufficient. The problem guarantees the tree has at least one node, so we do not need to handle an empty tree input.

## Approaches

The **brute-force approach** would be to generate all root-to-leaf paths, calculate their sums, and then traverse the tree again to remove nodes that are insufficient. This approach works because it directly follows the problem definition: you check each path sum and prune nodes accordingly. However, it is inefficient because it requires storing all paths, leading to high space complexity and redundant traversal of the tree.

The **optimal approach** relies on **post-order DFS traversal**. The key insight is that whether a node is sufficient depends entirely on its children. By recursively calculating the maximum path sum from a node to any leaf, we can prune insufficient nodes during the traversal. If a node is a leaf, it is pruned if its value is less than the limit minus the sum accumulated from the root. If a node has children, it is pruned if all children are pruned. This approach ensures we only traverse each node once, making it efficient in both time and space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n^2) | Generates all root-to-leaf paths explicitly |
| Optimal | O(n) | O(h) | Uses post-order DFS, pruning during recursion; h is tree height |

## Algorithm Walkthrough

1. Define a recursive DFS function that takes a node and the current sum along the path from the root.
2. If the node is `None`, return `None` immediately, as there is nothing to prune.
3. Recursively process the left and right children, passing the updated sum (`current_sum + node.val`).
4. After recursion, check the returned left and right children. If a child is pruned (returns `None`), remove the reference to that child.
5. If the node is a leaf (both left and right are `None`) and the accumulated path sum plus the node's value is less than the limit, return `None` to prune this node.
6. Otherwise, return the node itself as it is sufficient.

**Why it works:** The recursion ensures that we process children before parents (post-order). This guarantees that when we evaluate a node, we know whether its children are sufficient. The pruning invariant is: a node remains only if at least one path from this node to a leaf meets the limit requirement.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
from typing import Optional

class Solution:
    def sufficientSubset(self, root: Optional[TreeNode], limit: int) -> Optional[TreeNode]:
        def dfs(node: Optional[TreeNode], current_sum: int) -> Optional[TreeNode]:
            if not node:
                return None
            
            # Recursively prune left and right subtrees
            node.left = dfs(node.left, current_sum + node.val)
            node.right = dfs(node.right, current_sum + node.val)
            
            # If node becomes a leaf and its path sum is insufficient, prune it
            if not node.left and not node.right:
                if current_sum + node.val < limit:
                    return None
            
            return node
        
        return dfs(root, 0)
```

The `dfs` function handles pruning recursively. We first traverse the left and right children, updating their references to `None` if they are insufficient. Then, we check if the current node is a leaf and whether its accumulated path sum meets the limit. If it does not, we prune it. The recursion naturally bubbles up pruned nodes so that parents can be evaluated correctly.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func sufficientSubset(root *TreeNode, limit int) *TreeNode {
    var dfs func(node *TreeNode, currentSum int) *TreeNode
    dfs = func(node *TreeNode, currentSum int) *TreeNode {
        if node == nil {
            return nil
        }

        node.Left = dfs(node.Left, currentSum + node.Val)
        node.Right = dfs(node.Right, currentSum + node.Val)

        if node.Left == nil && node.Right == nil {
            if currentSum + node.Val < limit {
                return nil
            }
        }
        return node
    }
    return dfs(root, 0)
}
```

The Go solution mirrors the Python solution. Go requires explicit nil checks instead of Python's `None`. The recursion handles pruning and updates the left and right pointers in place. Integer overflow is not an issue here due to Go's `int` range being sufficient.

## Worked Examples

**Example 1:** `root = [1,2,3,4,-99,-99,7,8,9,-99,-99,12,13,-99,14], limit = 1`

| Node | Path sum check | Action |
| --- | --- | --- |
| 8 | 1+2+4+8=15 >=1 | keep |
| 9 | 1+2+4+9=16 >=1 | keep |
| -99 | 1+2+4-99=-92 <1 | prune |
| ... | ... | ... |

The DFS traverses each node, pruning nodes like -99 that fail the path sum check. The final tree is `[1,2,3,4,null,null,7,8,9,null,14]`.

**Example 2:** `root = [5,4,8,11,null,17,4,7,1,null,null,5,3], limit = 22`

All paths are checked recursively. Nodes like `1` and `3` are pruned because all root-to-leaf paths through them are below 22.

**Example 3:** `root = [1,2,-3,-5,null,4,null], limit = -1`

Negative values are handled correctly. Node `2` is pruned since its only paths sum below -1.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Each node is visited exactly once in the DFS |
| Space | O(h) | The recursion stack can go as deep as the height of the tree, h |

The DFS ensures a linear traversal of all nodes. Space usage is proportional to the tree height, which can be O(log n) for balanced trees or O(n) for skewed trees.
