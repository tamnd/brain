## Solution

## Problem Understanding

This problem asks us to assign bikes to workers in a way that minimizes the total Manhattan distance across all assignments.

We are given two arrays:

- `workers`, where each element represents the `(x, y)` position of a worker
- `bikes`, where each element represents the `(x, y)` position of a bike

Each worker must receive exactly one unique bike. A bike cannot be assigned to more than one worker.

The cost of assigning a bike to a worker is the Manhattan distance:

$|x_1 - x_2| + |y_1 - y_2|$

The goal is to minimize the total sum of distances across all worker-bike assignments.

The constraints are extremely important:

- `n <= m <= 10`
- At most 10 workers and 10 bikes

This immediately tells us that exponential algorithms may actually be feasible. A brute force permutation approach would still be too expensive in the worst case, but a carefully optimized dynamic programming solution using bitmasks works well because the state space remains manageable.

A few important observations:

- Every worker must get a different bike
- There may be more bikes than workers
- Coordinates are always non-negative
- All worker and bike positions are unique
- Since `m <= 10`, we can represent bike usage with a bitmask integer

Important edge cases include:

- Only one worker and one bike
- More bikes than workers, where some bikes remain unused
- Multiple assignments producing the same minimum cost
- Large coordinate values, producing large Manhattan distances
- Cases where a greedy local choice does not lead to the global optimum

A naive greedy approach can easily fail because assigning the closest bike to one worker may block a much better assignment for another worker later.

## Approaches

### Brute Force Approach

The brute force solution tries every possible assignment of bikes to workers.

For each worker, we choose one unused bike and recursively continue assigning bikes to the remaining workers. Eventually, we compute the total distance for that assignment and keep track of the minimum.

This guarantees correctness because every valid assignment is explored.

However, the number of assignments grows very quickly.

If there are `m` bikes and `n` workers, the number of possible assignments is:

$P(m,n)=\frac{m!}{(m-n)!}$

When `m = n = 10`, this becomes:

$10! = 3,628,800$

That is far too many states for repeated recursive computation without optimization.

### Optimal Approach, Dynamic Programming with Bitmasking

The key observation is that the state of the problem can be represented entirely by:

- Which worker we are currently assigning
- Which bikes have already been used

Since `m <= 10`, we can represent used bikes with a bitmask.

For example:

- `00101` means bikes 0 and 2 are already assigned
- `11111` means all bikes are assigned

This allows us to use memoized recursion or dynamic programming.

At every state:

- We know how many workers have already been assigned by counting the set bits in the mask
- We try assigning each unused bike to the current worker
- We recursively compute the remaining cost
- We store the result for the current mask so it is computed only once

This dramatically reduces repeated computation.

## Approach Comparison

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(P(m,n)) | O(n) | Tries every possible assignment |
| Optimal | O(m × 2^m) | O(2^m) | DP with memoization and bitmask states |

## Algorithm Walkthrough

1. Precompute all worker-bike Manhattan distances.

We create a matrix where `distance[i][j]` stores the distance between worker `i` and bike `j`.

This avoids recalculating distances repeatedly during recursion.
2. Use a bitmask to represent assigned bikes.

If bit `j` is set in the mask, then bike `j` is already assigned.

Example:

- `mask = 0101`
- Bikes 0 and 2 are used
3. Determine which worker to assign next.

The number of assigned workers equals the number of set bits in the mask.

So:

$worker = \text{popcount}(mask)$
4. Define the recursive DP function.

`dp(mask)` returns the minimum additional cost needed after assigning bikes represented by `mask`.
5. Handle the base case.

If all workers are assigned, return `0`.
6. Try every unused bike.

For each bike not already used:

- Assign it to the current worker
- Add the distance cost
- Recurse with the updated mask
7. Take the minimum result.

Among all possible bike assignments for the current worker, keep the smallest total cost.
8. Memoize results.

Store computed results for each mask so the same state is never recomputed.

### Why it works

The algorithm works because every valid assignment corresponds to exactly one sequence of mask states.

At each state, the algorithm considers every possible unused bike for the current worker. Since all possibilities are explored and the minimum is taken, the optimal assignment is guaranteed to be found.

Memoization ensures that each unique mask state is solved only once.

## Python Solution

```
from typing import List
from functools import lru_cache

class Solution:
    def assignBikes(self, workers: List[List[int]], bikes: List[List[int]]) -> int:
        num_workers = len(workers)
        num_bikes = len(bikes)

        # Precompute Manhattan distances
        distances = [
            [
                abs(workers[i][0] - bikes[j][0]) +
                abs(workers[i][1] - bikes[j][1])
                for j in range(num_bikes)
            ]
            for i in range(num_workers)
        ]

        @lru_cache(None)
        def dp(mask: int) -> int:
            worker_index = mask.bit_count()

            # All workers assigned
            if worker_index == num_workers:
                return 0

            minimum_cost = float("inf")

            # Try every unused bike
            for bike_index in range(num_bikes):
                if (mask & (1 << bike_index)) == 0:
                    new_mask = mask | (1 << bike_index)

                    total_cost = (
                        distances[worker_index][bike_index] +
                        dp(new_mask)
                    )

                    minimum_cost = min(minimum_cost, total_cost)

            return minimum_cost

        return dp(0)
```

The implementation begins by computing a distance matrix. This converts repeated Manhattan distance calculations into simple table lookups.

The recursive function `dp(mask)` represents the minimum remaining cost after assigning the bikes indicated by `mask`.

The current worker is determined using `mask.bit_count()`. Since one bike is assigned per worker, the number of assigned bikes equals the number of assigned workers.

For every unused bike, the algorithm:

- Assigns the bike to the current worker
- Recursively solves the smaller subproblem
- Keeps the minimum total cost

The `lru_cache` decorator memoizes states automatically, ensuring each bitmask is solved once.

## Go Solution

```
package main

import "math"

func assignBikes(workers [][]int, bikes [][]int) int {
	numWorkers := len(workers)
	numBikes := len(bikes)

	// Precompute distances
	distances := make([][]int, numWorkers)

	for i := 0; i < numWorkers; i++ {
		distances[i] = make([]int, numBikes)

		for j := 0; j < numBikes; j++ {
			distances[i][j] =
				abs(workers[i][0]-bikes[j][0]) +
					abs(workers[i][1]-bikes[j][1])
		}
	}

	memo := make(map[int]int)

	var dp func(int) int

	dp = func(mask int) int {
		if value, exists := memo[mask]; exists {
			return value
		}

		workerIndex := bitCount(mask)

		if workerIndex == numWorkers {
			return 0
		}

		minimumCost := math.MaxInt32

		for bikeIndex := 0; bikeIndex < numBikes; bikeIndex++ {
			if (mask & (1 << bikeIndex)) == 0 {
				newMask := mask | (1 << bikeIndex)

				totalCost :=
					distances[workerIndex][bikeIndex] +
						dp(newMask)

				if totalCost < minimumCost {
					minimumCost = totalCost
				}
			}
		}

		memo[mask] = minimumCost
		return minimumCost
	}

	return dp(0)
}

func abs(x int) int {
	if x < 0 {
		return -x
	}
	return x
}

func bitCount(x int) int {
	count := 0

	for x > 0 {
		count += x & 1
		x >>= 1
	}

	return count
}
```

The Go implementation follows the same logic as the Python version.

A `map[int]int` is used for memoization instead of Python's `lru_cache`.

Go does not provide a built-in `bit_count()` method for integers, so we implement a helper function manually.

Since coordinate values are small, integer overflow is not a concern.

    def assignBikes(self, workers: List[List[int]], bikes: List[List[int]]) -> int:
        from functools import lru_cache

        n = len(workers)
        m = len(bikes)

        distances = [
            [
                abs(workers[i][0] - bikes[j][0]) +
                abs(workers[i][1] - bikes[j][1])
                for j in range(m)

            for i in range(n)

                    best = min(
                        best,
                        distances[worker][bike] +
                        dp(mask | (1 << bike))

            return best

        return dp(0)

## Edge Cases

One important edge case is when there is only one worker and one bike. A recursive or DP solution can sometimes mishandle the base case if termination conditions are incorrect. In this implementation, once the number of assigned workers equals `n`, the recursion immediately returns `0`, ensuring the smallest input works correctly.

Another important edge case occurs when there are more bikes than workers. A naive implementation might incorrectly assume every bike must be used. Our bitmask DP avoids this problem naturally because it only tracks assigned bikes, and recursion stops once all workers have been assigned. Unused bikes remain irrelevant.

A third tricky edge case involves greedy traps. Consider a situation where assigning the closest bike to the current worker prevents a much better assignment later. Greedy approaches often fail here because local optimality does not guarantee global optimality. The DP solution avoids this entirely by exploring every valid assignment combination and selecting the global minimum.

Large coordinate values are another potential source of bugs. Manhattan distances can become large, especially near the upper coordinate limit of `999`. Both implementations use standard integer arithmetic, which safely handles the maximum possible total distance within problem constraints.

Finally, multiple assignments may produce the same minimum cost. The algorithm does not rely on uniqueness of the answer. Since it computes the minimum numeric value across all valid assignments, ties are handled automatically and correctly.
