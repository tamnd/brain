## Problem Understanding

This problem asks us to determine the minimum and maximum number of moves needed to make a set of stones consecutive on the X-axis. You are given an array `stones`, where each element represents a unique stone's position. An endpoint stone is defined as the stone with the smallest or largest position. The only allowed move is to pick up an endpoint stone and place it somewhere so that it is no longer an endpoint stone. The goal is to arrange the stones so that they occupy consecutive positions, minimizing and maximizing the number of moves.

The expected output is an integer array of length 2: `answer[0]` gives the minimum moves and `answer[1]` gives the maximum moves. The constraints tell us that `stones` can have up to 10,000 elements and positions can be as large as 10^9. This means naive simulation of all moves is infeasible. We must reason about gaps and endpoints mathematically.

Edge cases to note include when stones are almost consecutive (only one or two moves needed), when stones are widely spaced, and when stones form a consecutive block with one end far away. The problem guarantees that all stone positions are unique, so we do not need to handle duplicates.

## Approaches

A brute-force approach would attempt to simulate moving each endpoint stone in every possible valid way until the stones become consecutive. While this would eventually give the correct answer, the number of possible states grows exponentially with the number of stones and the maximum distance between endpoints. This is too slow for arrays with 10,000 elements or large position ranges.

The key insight for an optimal approach is to focus on **gaps** between stones and use a **sliding window** to calculate the minimum moves, while simple arithmetic can determine the maximum moves. Sorting the stones array first makes it easier to compute these gaps. The maximum moves are determined by moving endpoint stones toward the opposite end while leaving gaps as large as possible. The minimum moves are calculated by finding the largest block of consecutive stones that can fit within a window of size equal to the number of stones, then counting how many stones are outside this block.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(n) | Simulate every possible move of endpoints until stones are consecutive. Too slow for large n. |
| Optimal | O(n log n) | O(1) | Sort stones, use sliding window to find minimum moves, compute maximum moves using gaps at the endpoints. |

## Algorithm Walkthrough

1. Sort the `stones` array in ascending order. This allows us to reason about gaps and consecutive positions easily.
2. Initialize variables for `n` (number of stones), `max_moves`, and `min_moves`.
3. Calculate `max_moves` by considering the two extreme scenarios: moving all stones toward the leftmost or rightmost end. The formula is:

```
max_moves = max(stones[-1] - stones[1] - n + 2, stones[-2] - stones[0] - n + 2)
```

This subtracts the number of stones already in place from the largest gap, giving the number of moves needed to fill it.
4. Use a sliding window to determine the minimum moves. For each stone `stones[i]` as the start of a window, expand the window to include as many consecutive stones as possible within a range of size `n`. Count how many stones are outside this window, which gives candidate minimum moves. Handle the special case when there is exactly one empty position inside the window at the endpoint, which requires 2 moves.
5. Return `[min_moves, max_moves]`.

**Why it works:** Sorting ensures all gaps are accounted for in order. The sliding window captures the largest consecutive block of stones, minimizing the number of moves needed to fill gaps outside this block. Maximum moves are determined by endpoint gaps, ensuring no illegal moves are considered.

## Python Solution

```
from typing import List

class Solution:
    def numMovesStonesII(self, stones: List[int]) -> List[int]:
        stones.sort()
        n = len(stones)
        
        # Maximum moves
        max_moves = max(stones[-1] - stones[1] - n + 2, stones[-2] - stones[0] - n + 2)
        
        # Minimum moves using sliding window
        min_moves = n
        left = 0
        for right in range(n):
            while stones[right] - stones[left] + 1 > n:
                left += 1
            current_window_size = right - left + 1
            if current_window_size == n - 1 and stones[right] - stones[left] + 1 == n - 1:
                min_moves = min(min_moves, 2)  # Special case: one empty slot at endpoint
            else:
                min_moves = min(min_moves, n - current_window_size)
        
        return [min_moves, max_moves]
```

This Python implementation sorts the stones, calculates the maximum moves using endpoint gaps, and uses a sliding window to determine the minimum moves efficiently. It handles the special case when only one gap exists at an endpoint, which requires two moves.

## Go Solution

```
import "sort"

func numMovesStonesII(stones []int) []int {
    sort.Ints(stones)
    n := len(stones)
    
    maxMoves := max(stones[n-1]-stones[1]-n+2, stones[n-2]-stones[0]-n+2)
    
    minMoves := n
    left := 0
    for right := 0; right < n; right++ {
        for stones[right]-stones[left]+1 > n {
            left++
        }
        currentWindowSize := right - left + 1
        if currentWindowSize == n-1 && stones[right]-stones[left]+1 == n-1 {
            minMoves = min(minMoves, 2)
        } else {
            minMoves = min(minMoves, n-currentWindowSize)
        }
    }
    
    return []int{minMoves, maxMoves}
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

## Worked Examples

**Example 1:** `stones = [7,4,9]`

Sorted stones: `[4,7,9]`

Maximum moves: `max(9-7-3+2, 7-4-3+2) = max(1, 2) = 2`

Sliding window for minimum moves: `[4,7]` and `[7,9]` each need 1 move, so `min_moves = 1`

Output: `[1,2]`

**Example 2:** `stones = [6,5,4,3,10]`

Sorted stones: `[3,4,5,6,10]`

Maximum moves: `max(10-4-5+2, 6-3-5+2) = max(3, 0) = 3`

Sliding window: best consecutive block `[3,4,5,6]` leaves 1 stone outside, but one endpoint gap requires 2 moves, so `min_moves = 2`

Output: `[2,3]`
