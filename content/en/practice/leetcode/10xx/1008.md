## Solution

## Problem Understanding

The problem gives us the preorder traversal of a Binary Search Tree, and asks us to reconstruct the original BST.

A preorder traversal visits nodes in this order:

1. Visit the current node
2. Traverse the left subtree
3. Traverse the right subtree

For example, if the preorder traversal is:

```
[8,5,1,7,10,12]
```

then `8` must be the root because preorder always visits the root first.

Since this is a Binary Search Tree, every value smaller than `8` belongs somewhere in the left subtree, and every value greater than `8` belongs somewhere in the right subtree.

The challenge is reconstructing the tree efficiently without explicitly sorting or repeatedly splitting arrays in expensive ways.

The input array contains unique integers, which simplifies the BST construction because we never need to handle duplicate placement rules. The constraints are relatively small, at most 100 nodes, but the goal is still to produce an optimal and clean solution.

The important observation is that preorder traversal naturally preserves subtree boundaries. Once we know the valid numeric range for a subtree, we can determine whether the next preorder value belongs there.

Several edge cases are important:

- A single-element array should produce a tree with only one node.
- A strictly increasing preorder array creates a completely right-skewed BST.
- A strictly decreasing preorder array creates a completely left-skewed BST.
- Naive implementations that repeatedly scan subarrays can become inefficient.
- Because the problem guarantees the preorder traversal is valid, we never need to handle invalid BST configurations.

## Approaches

### Brute Force Approach

A straightforward solution is to recursively construct the tree by splitting the preorder array into left and right subtree portions.

The first value in the current preorder segment becomes the root. Then we scan forward until we find the first value greater than the root. Everything before that index belongs to the left subtree, and everything after belongs to the right subtree.

For example:

```
[8,5,1,7,10,12]
```

- Root = 8
- Left subtree preorder = [5,1,7]
- Right subtree preorder = [10,12]

We recursively repeat the same process for each subtree.

This approach works because preorder traversal always lists all left subtree nodes before right subtree nodes in a BST.

However, each recursive call may scan part of the array to find the split point. In the worst case, such as a skewed tree, this leads to quadratic time complexity.

### Optimal Approach

The key insight is that preorder traversal already gives nodes in the exact order they should be created.

Instead of repeatedly searching for subtree boundaries, we can maintain a valid numeric range for each recursive subtree.

When constructing a subtree:

- Any node in the left subtree must be smaller than the parent value.
- Any node in the right subtree must be greater than the parent value.
- If the next preorder value falls outside the allowed range, it does not belong to the current subtree.

We process the preorder array exactly once using a shared index pointer.

This avoids repeated scanning and gives an optimal linear-time solution.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n²) | O(n) | Repeatedly scans array segments to split left and right subtrees |
| Optimal | O(n) | O(n) | Uses preorder ordering and BST bounds to build tree in one pass |

## Algorithm Walkthrough

### Optimal Recursive Bounds Algorithm

1. Start with a global index pointing to the first element in the preorder array.

The preorder traversal always visits the root first, so the current index tells us which node should be created next.
2. Define a recursive function that accepts a lower bound and upper bound.

These bounds represent the valid range of values that may appear in the current subtree.
3. If the index has reached the end of the array, return `None`.

This means all nodes have already been processed.
4. Look at the current preorder value without consuming it yet.

If the value falls outside the valid range, it does not belong in this subtree, so return `None`.
5. Create a node using the current preorder value.

Advance the index because this value is now consumed.
6. Recursively construct the left subtree.

The left subtree must contain values strictly smaller than the current node value, so the new range becomes:

```
(lower_bound, current_value)
```
7. Recursively construct the right subtree.

The right subtree must contain values strictly greater than the current node value, so the new range becomes:

```
(current_value, upper_bound)
```
8. Return the constructed node.

### Why it works

The algorithm works because preorder traversal guarantees that nodes appear in root-left-right order. At every step, the next unprocessed value either belongs to the current subtree or does not. The lower and upper bounds precisely encode BST validity rules, ensuring nodes are attached only where they are allowed to exist. Since every node is processed exactly once, the reconstruction is both correct and efficient.

## Python Solution

```
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right

class Solution:
    def bstFromPreorder(self, preorder: List[int]) -> Optional[TreeNode]:
        index = 0

        def build(lower: int, upper: int) -> Optional[TreeNode]:
            nonlocal index

            if index >= len(preorder):
                return None

            value = preorder[index]

            if value < lower or value > upper:
                return None

            index += 1

            node = TreeNode(value)

            node.left = build(lower, value)
            node.right = build(value, upper)

            return node

        return build(float("-inf"), float("inf"))
```

The implementation uses a shared `index` variable to track which preorder element should be processed next.

The nested `build()` function constructs a subtree constrained by numeric bounds. The bounds enforce BST validity rules naturally during recursion.

Before creating a node, the function checks whether the current preorder value fits within the valid range. If it does not, the value belongs to a different subtree, so the function immediately returns `None`.

Once a node is created, the index advances exactly once. The recursive calls then construct the left and right subtrees using tighter bounds derived from the current node value.

This implementation is efficient because every preorder element is processed only once.

## Go Solution

```
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func bstFromPreorder(preorder []int) *TreeNode {
    index := 0

    var build func(int, int) *TreeNode

    build = func(lower int, upper int) *TreeNode {
        if index >= len(preorder) {
            return nil
        }

        value := preorder[index]

        if value < lower || value > upper {
            return nil
        }

        index++

        node := &TreeNode{Val: value}

        node.Left = build(lower, value)
        node.Right = build(value, upper)

        return node
    }

    return build(-1<<31, 1<<31-1)
}
```

The Go implementation follows the same recursive bounds strategy as the Python version.

Instead of Python's `nonlocal`, Go closures naturally capture the `index` variable from the outer scope.

Go uses `nil` instead of `None` for missing child pointers. Since the constraints guarantee values between `1` and `1000`, using large integer bounds is completely safe.

Slices in Go are lightweight references, so no extra copying occurs during recursion.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(n) | Every preorder value is processed exactly once |
| Space | O(n) | Recursive call stack can grow to height n in skewed trees |

The algorithm achieves linear time complexity because each node is created once and never revisited. There are no repeated scans or subtree copying operations.

The space complexity comes primarily from recursion depth. In a balanced BST, recursion depth is `O(log n)`, but in the worst case of a completely skewed tree, recursion depth becomes `O(n)`.

## Edge Cases

A single-element preorder array is the smallest valid input. This case is important because recursive solutions can accidentally overrun the array or incorrectly attempt subtree construction. The implementation handles this naturally because once the root node is created, the index reaches the end of the array and all recursive calls immediately return `None`.

A strictly increasing preorder traversal creates a completely right-skewed BST. This is a common worst-case scenario because every node becomes the right child of the previous node. Incorrect implementations may accidentally place nodes in left subtrees if bounds are not enforced properly. The bounds-based recursion correctly rejects left subtree placements and builds the chain entirely on the right side.

A strictly decreasing preorder traversal creates a completely left-skewed BST. This tests whether the algorithm correctly narrows upper bounds during recursive calls. If bounds are handled incorrectly, nodes may end up attached to invalid positions. The implementation ensures each new node fits only within the valid left subtree range.

Another subtle edge case occurs when transitioning from a deep left subtree back to a right subtree, such as:

```
[8,5,1,7,10,12]
```

After constructing node `1`, the next value `7` no longer belongs within the bounds of node `1`. The algorithm correctly stops recursion for that subtree and backtracks until it finds the valid parent subtree range. This behavior is exactly why the bounds technique works so effectively.
