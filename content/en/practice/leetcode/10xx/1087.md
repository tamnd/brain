## Solution

## Problem Understanding

The problem is asking us to generate all possible words from a compressed string representation that uses curly braces to indicate multiple options for a single position in the word. Each letter in the string may be a single character or a set of characters enclosed in braces. For example, the string `"{a,b}c{d,e}f"` represents all combinations where the first character can be `'a'` or `'b'`, the second character is always `'c'`, the third character can be `'d'` or `'e'`, and the last character is `'f'`. The expected output is a list of all valid words sorted in lexicographical order.

The input string `s` is guaranteed to be well-formed, meaning there are no nested braces, all options inside braces are unique, and only lowercase letters, commas, and braces are present. The constraints (`1 <= s.length <= 50`) indicate that while the string is small, a naive combinatorial approach must handle multiple branching efficiently.

Important edge cases include strings without any braces, strings where all characters are inside braces, strings with consecutive braces, and strings that include only a single letter. The guarantee of valid input simplifies parsing since we do not need to handle malformed strings or nested braces.

## Approaches

A brute-force approach would involve generating every possible combination by iterating through each character or brace, recursively forming words by picking each option at every step. While this will produce the correct result, it is not efficient because the number of combinations can grow exponentially with the number of options. For example, a string like `"{a,b,c,d,e,f,g,h,i,j}"` repeated multiple times would generate millions of combinations.

The key insight for a more optimal solution is to first parse the string into a list of lists, where each inner list contains all possible characters for that position. Then, we can use backtracking or an iterative breadth-first search to generate all combinations. Sorting each brace's options upfront ensures the final output is lexicographically ordered without needing a separate sorting step.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(2^n) | O(2^n) | Recursively generate all combinations without preprocessing; inefficient for multiple braces |
| Optimal | O(k^m) | O(k^m) | Parse string into options first, then generate combinations using backtracking; `k` is max options per brace, `m` is number of positions |

## Algorithm Walkthrough

1. Initialize an empty list `groups` to store options for each position.
2. Iterate through the string `s` using an index `i`. If the character is a letter, wrap it in a list and append to `groups`. If the character is an opening brace `{`, extract all characters until the closing brace `}`, split them by commas, sort them, and append the sorted list to `groups`.
3. Define a backtracking function that takes the current index in `groups` and the current path (partial word). If the index equals the length of `groups`, append the joined path to the result list.
4. Otherwise, iterate over all characters in the current group, append each to the path, recursively call backtracking for the next index, and remove the last character after recursion to backtrack.
5. Initialize an empty list `result` and call the backtracking function starting at index 0 with an empty path.
6. Return the `result` list.

This approach works because by parsing the string first into groups of options and using systematic backtracking, we explore every valid combination exactly once. Sorting the groups ensures lexicographical order naturally.

## Python Solution

```
from typing import List

class Solution:
    def expand(self, s: str) -> List[str]:
        groups = []
        i = 0
        while i < len(s):
            if s[i] == '{':
                j = i + 1
                while s[j] != '}':
                    j += 1
                options = s[i+1:j].split(',')
                groups.append(sorted(options))
                i = j + 1
            else:
                groups.append([s[i]])
                i += 1
        
        result = []
        
        def backtrack(index: int, path: List[str]):
            if index == len(groups):
                result.append(''.join(path))
                return
            for char in groups[index]:
                path.append(char)
                backtrack(index + 1, path)
                path.pop()
        
        backtrack(0, [])
        return result
```

The implementation first converts the string into structured options, which makes the backtracking simple and clean. The `backtrack` function constructs each word character by character, exploring every possible choice. Sorting the options in each group ensures the final output is in lexicographical order without extra work.

## Go Solution

```
package main
import (
    "sort"
    "strings"
)

func expand(s string) []string {
    var groups [][]string
    n := len(s)
    for i := 0; i < n; {
        if s[i] == '{' {
            j := i + 1
            for s[j] != '}' {
                j++
            }
            options := strings.Split(s[i+1:j], ",")
            sort.Strings(options)
            groups = append(groups, options)
            i = j + 1
        } else {
            groups = append(groups, []string{string(s[i])})
            i++
        }
    }

    var result []string
    var backtrack func(int, []string)
    backtrack = func(index int, path []string) {
        if index == len(groups) {
            result = append(result, strings.Join(path, ""))
            return
        }
        for _, char := range groups[index] {
            path = append(path, char)
            backtrack(index+1, path)
            path = path[:len(path)-1]
        }
    }
    backtrack(0, []string{})
    return result
}
```

In Go, slices replace Python lists for the path, and `strings.Join` replaces `''.join()` in Python. Sorting is done with `sort.Strings`. The recursive logic is identical, with slice manipulation handling backtracking.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(k^m) | `k` is the maximum options per group, `m` is the number of groups. Backtracking generates all combinations. |
| Space | O(k^m) | Space used to store all results. The recursion stack also takes O(m) space. |

The complexity is driven by the combinatorial nature of the problem. Parsing and sorting are linear with respect to input size and minor compared to backtracking.
