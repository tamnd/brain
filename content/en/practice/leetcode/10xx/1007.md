## Problem Understanding

This problem is asking us to determine the minimum number of rotations required so that either the top row or the bottom row of a series of dominoes contains the same value across all positions. Each domino has two numbers, represented by `tops[i]` for the top half and `bottoms[i]` for the bottom half. A rotation swaps these two numbers for a given domino.

The input consists of two integer arrays of equal length representing the top and bottom values of the dominoes. The output is a single integer representing the minimum number of rotations needed to make one row uniform, or `-1` if it is impossible.

The constraints indicate that the number of dominoes can be up to 20,000, and the numbers on each domino range from 1 to 6. This implies that a solution that iterates through every possible combination of rotations would be too slow. We need a solution that works efficiently with linear or near-linear time complexity.

Important edge cases include dominoes where all elements are already the same, dominoes where no single number can fill a row even with rotations, and cases where a solution is possible but requires selecting the correct target number to minimize rotations.

## Approaches

The brute-force approach would attempt to make every number from 1 to 6 the uniform value for the top row and bottom row. For each number, we would count the rotations needed and check if making all elements equal is possible. This involves iterating through the entire array for each possible target number, which is feasible in terms of correctness but can be inefficient.

The key observation for the optimal solution is that we only need to check at most two candidate numbers: the number at the first domino’s top or bottom. If a solution exists, it must involve one of these numbers because every domino must have the target number either on top or bottom to be rotatable into position. By focusing on these candidates, we avoid unnecessary iterations and reduce the time complexity to O(n).

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(6 * n) | O(1) | Check all numbers from 1 to 6 as potential targets |
| Optimal | O(n) | O(1) | Check only tops[0] and bottoms[0] as potential targets |

## Algorithm Walkthrough

1. Identify the candidate values: `tops[0]` and `bottoms[0]`. These are the only values that could potentially fill a row because every domino must include the target number in either top or bottom.
2. Define a helper function `check(target)` that computes the minimum rotations needed to make every domino show `target` on the top or bottom. For each domino, if `target` is not on either side, return `-1`. Otherwise, increment the rotation count depending on whether a swap is needed.
3. Call `check` on `tops[0]`. If it returns a non-negative number, this is a valid rotation count.
4. If `tops[0]` and `bottoms[0]` are different, call `check` on `bottoms[0]` and select the minimum non-negative rotation count from the two candidates.
5. Return `-1` if neither candidate produces a valid rotation count.

**Why it works**: The invariant here is that for a solution to exist, the uniform value must appear in every domino, either on the top or bottom. By checking only the first domino’s top and bottom values, we guarantee that any feasible uniform row will match one of these two numbers. The algorithm is linear because it examines each domino exactly once per candidate.

## Python Solution

```
from typing import List

class Solution:
    def minDominoRotations(self, tops: List[int], bottoms: List[int]) -> int:
        def check(target: int) -> int:
            rotations_top, rotations_bottom = 0, 0
            for t, b in zip(tops, bottoms):
                if t != target and b != target:
                    return -1
                elif t != target:
                    rotations_top += 1
                elif b != target:
                    rotations_bottom += 1
            return min(rotations_top, rotations_bottom)
        
        rotations = check(tops[0])
        if tops[0] != bottoms[0]:
            rotations_b = check(bottoms[0])
            if rotations == -1:
                rotations = rotations_b
            elif rotations_b != -1:
                rotations = min(rotations, rotations_b)
        return rotations
```

The implementation defines a helper function `check` to count the rotations needed for a given target. It then evaluates both possible targets and returns the minimum valid rotation count. Special attention is given to the case when the first domino’s top and bottom are equal to avoid redundant checks.

## Go Solution

```
func minDominoRotations(tops []int, bottoms []int) int {
    check := func(target int) int {
        rotationsTop, rotationsBottom := 0, 0
        for i := 0; i < len(tops); i++ {
            if tops[i] != target && bottoms[i] != target {
                return -1
            } else if tops[i] != target {
                rotationsTop++
            } else if bottoms[i] != target {
                rotationsBottom++
            }
        }
        if rotationsTop < rotationsBottom {
            return rotationsTop
        }
        return rotationsBottom
    }

    rotations := check(tops[0])
    if tops[0] != bottoms[0] {
        rotationsB := check(bottoms[0])
        if rotations == -1 {
            rotations = rotationsB
        } else if rotationsB != -1 && rotationsB < rotations {
            rotations = rotationsB
        }
    }
    return rotations
}
```

In Go, the logic mirrors the Python version. The `check` function calculates the rotations for each candidate. We handle comparison carefully, and no slices or pointers are required beyond normal array iteration.
