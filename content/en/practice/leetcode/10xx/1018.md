## Problem Understanding

The problem asks us to evaluate a binary array `nums` where each element represents a bit in a binary number. For each index `i`, we construct the number `xi` using the subarray `nums[0..i]` interpreted as a binary number. We then determine whether this number is divisible by 5, returning a boolean array of the same length where each element corresponds to the divisibility of `xi`.

The input `nums` is guaranteed to be non-empty, with each element either `0` or `1`. This means we do not need to handle invalid inputs. The array can be very large, up to `10^5` elements, which indicates that a naive solution that repeatedly converts subarrays to integers will be inefficient due to the potential size of the binary numbers. The output is an array of booleans of the same length as `nums`.

Important edge cases include arrays containing all zeros (where every prefix is divisible by 5), arrays with alternating 0s and 1s, and single-element arrays. Handling large numbers efficiently is crucial because directly converting binary strings to integers may lead to very large numbers that exceed standard integer sizes.

## Approaches

The brute-force approach would convert each subarray `nums[0..i]` into its integer value and then check divisibility by 5. This is correct but inefficient, because forming the integer from each prefix repeatedly requires `O(n^2)` operations for an array of length `n`. For the maximum constraint of `10^5`, this approach is computationally infeasible.

The key insight for an optimal solution is that we can calculate each prefix incrementally using modular arithmetic. When constructing `xi` from `xi-1`, appending the new bit `nums[i]` is equivalent to multiplying the previous number by 2 and adding `nums[i]`. Taking modulo 5 at each step ensures that the intermediate values remain small, eliminating the need to work with large integers.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(n) | Construct each prefix fully and check divisibility |
| Optimal | O(n) | O(n) | Incrementally build prefix modulo 5 and store boolean results |

## Algorithm Walkthrough

1. Initialize an empty result array `answer` to store the boolean results for each prefix.
2. Initialize a variable `prefix_mod` to 0. This will track the current prefix modulo 5.
3. Iterate through each element `bit` in `nums`:

1. Update `prefix_mod` using the formula `prefix_mod = (prefix_mod * 2 + bit) % 5`. This incrementally constructs the binary number modulo 5.
2. Append `True` to `answer` if `prefix_mod` is 0, otherwise append `False`.
4. Return the `answer` array containing the divisibility results.

Why it works: The key invariant is that `(prefix_mod * 2 + bit) % 5` correctly updates the remainder of the prefix number modulo 5. Because modulo operations are distributive over addition and multiplication, we do not lose any correctness while avoiding large integer computations. This ensures the algorithm produces accurate results for all prefixes.

## Python Solution

```
from typing import List

class Solution:
    def prefixesDivBy5(self, nums: List[int]) -> List[bool]:
        answer: List[bool] = []
        prefix_mod: int = 0
        for bit in nums:
            prefix_mod = (prefix_mod * 2 + bit) % 5
            answer.append(prefix_mod == 0)
        return answer
```

This implementation follows the algorithm exactly. `prefix_mod` tracks the prefix number modulo 5, ensuring it never grows too large. For each bit, the calculation `(prefix_mod * 2 + bit) % 5` efficiently updates the prefix remainder. The boolean result is appended to the result list for each prefix.

## Go Solution

```
func prefixesDivBy5(nums []int) []bool {
    answer := make([]bool, len(nums))
    prefixMod := 0
    for i, bit := range nums {
        prefixMod = (prefixMod*2 + bit) % 5
        answer[i] = prefixMod == 0
    }
    return answer
}
```

The Go version mirrors the Python logic. We preallocate the `answer` slice for efficiency. Since Go handles integers safely within the input constraints, we do not need additional handling for large numbers. Slice indexing is used to store results directly.

## Edge Cases

One important edge case is an array consisting entirely of zeros. Every prefix in this case is divisible by 5, and the implementation correctly returns `True` for all elements, demonstrating that the modulo logic works when the prefix remainder is initially zero.

Another edge case is a single-element array. If the element is 0, the output should be `True`; if 1, it should be `False`. The algorithm handles this naturally because the loop correctly computes the modulo for a single iteration.

A third edge case involves large arrays where the prefix binary number could exceed standard integer limits if computed directly. Using the modulo operation at every step avoids large number computations entirely, ensuring correctness and efficiency even for arrays of length `10^5`.

This approach also correctly handles alternating patterns of 0s and 1s, verifying that the incremental modulo calculation works for all possible binary sequences.
