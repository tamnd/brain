## Solution

## Problem Understanding

The problem asks us to find the lexicographically largest permutation smaller than the given array `arr`, using exactly one swap of two elements. In other words, we want to make a single swap that decreases the array in lexicographic order, but among all possible one-swap reductions, we must choose the one that results in the largest possible array.

The input array consists of positive integers, and it is not necessarily sorted or unique. The expected output is an array of the same length with either one swap applied or unchanged if no smaller permutation is possible. The constraints `1 <= arr.length <= 10^4` and `1 <= arr[i] <= 10^4` indicate that the solution must be efficient, ideally linear or near-linear in time.

Edge cases include arrays that are already in non-increasing order (no smaller permutation possible), arrays with duplicate elements (we cannot swap two identical elements to change the array), and arrays with length 1 (no swaps possible).

## Approaches

The brute-force approach is to try all possible pairs of indices `(i, j)` where `i < j` and swap `arr[i]` and `arr[j]` if it produces a smaller array. After checking all valid swaps, we would return the largest array that is smaller than the original. While this is correct, it requires `O(n^2)` swaps and comparisons, which is too slow for `n = 10^4`.

The optimal approach leverages the observation that a single swap to reduce the array should involve the **first position from the right where the array decreases**. Once we find that index, we should swap it with the **largest value to its right that is smaller than it**, which ensures that the resulting permutation is the largest possible among all smaller permutations. If duplicates exist, we choose the **rightmost occurrence** of the candidate to avoid unnecessary reductions.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n^2) | O(1) | Try every possible swap and choose the best smaller permutation |
| Optimal | O(n) | O(1) | Find first decrease from right and swap with largest smaller element on right |

## Algorithm Walkthrough

1. Traverse the array from right to left and find the first index `i` such that `arr[i] > arr[i+1]`. This is the pivot where we can make the array smaller.
2. If no such index exists, the array is non-decreasing from left to right and cannot be made smaller with one swap, so return the array as is.
3. To find the best element to swap with `arr[i]`, traverse the array to the right of `i` and look for the **largest element smaller than `arr[i]`**. If duplicates exist, choose the **rightmost occurrence** to maximize the lexicographic value.
4. Swap `arr[i]` with the identified element.
5. Return the modified array.

**Why it works:** By choosing the first decreasing element from the right, we ensure we are making the smallest necessary change. Swapping with the largest smaller element to its right guarantees that among all arrays smaller than the original, the result is lexicographically the largest. Handling duplicates by taking the rightmost ensures we do not unnecessarily decrease the array further.

## Python Solution

```
from typing import List

class Solution:
    def prevPermOpt1(self, arr: List[int]) -> List[int]:
        n = len(arr)
        i = n - 2
        # Step 1: Find first decrease from right
        while i >= 0 and arr[i] <= arr[i + 1]:
            i -= 1
        if i < 0:
            return arr  # Already smallest permutation
        
        # Step 2: Find largest element smaller than arr[i] to the right
        j = n - 1
        while arr[j] >= arr[i] or (j > 0 and arr[j] == arr[j-1]):
            j -= 1
        
        # Step 3: Swap
        arr[i], arr[j] = arr[j], arr[i]
        return arr
```

The Python implementation first identifies the pivot where the array can be decreased. It then finds the rightmost candidate for swapping to maximize the resulting array, handling duplicates carefully. Finally, it performs the swap and returns the result.

## Go Solution

```
func prevPermOpt1(arr []int) []int {
    n := len(arr)
    i := n - 2
    // Step 1: Find first decrease from right
    for i >= 0 && arr[i] <= arr[i+1] {
        i--
    }
    if i < 0 {
        return arr // Already smallest permutation
    }
    
    // Step 2: Find largest element smaller than arr[i] to the right
    j := n - 1
    for arr[j] >= arr[i] || (j > 0 && arr[j] == arr[j-1]) {
        j--
    }
    
    // Step 3: Swap
    arr[i], arr[j] = arr[j], arr[i]
    return arr
}
```

The Go implementation mirrors the Python logic, handling slices directly. Edge cases like empty or single-element slices are naturally handled by the loop conditions. Integer overflow is not an issue due to the constraints.

## Worked Examples

### Example 1: `[3,2,1]`

1. Scan from right: `2 > 1` → `i = 1`.
2. Candidate to swap: largest number smaller than `2` to its right is `1`.
3. Swap `arr[1]` and `arr[2]`: `[3,1,2]`.

### Example 2: `[1,1,5]`

1. Scan from right: no `arr[i] > arr[i+1]` → array cannot be decreased.
2. Return `[1,1,5]`.

### Example 3: `[1,9,4,6,7]`

1. Scan from right: `9 > 4` → `i = 1`.
2. Candidate to swap: largest number smaller than `9` to its right is `7`.
3. Swap `arr[1]` and `arr[4]`: `[1,7,4,6,9]`.

The algorithm is efficient for `n` up to 10^4 due to its linear time complexity and minimal space usage.

## Edge Cases

The first edge case is a single-element array. Since swaps require two elements, the array cannot be modified. The implementation handles this by naturally returning the input without any swaps.

The second edge case is an array of identical elements. No swap can make the array smaller, and duplicates must not cause unnecessary swaps. The rightmost duplicate handling ensures we do not attempt a redundant swap.

The third edge case involves multiple duplicates with a pivot. For instance, `[1,3,1,3]` requires swapping the first `3` with the last `1` to maximize the resulting array while still decreasing it. The algorithm correctly identifies the pivot and the rightmost candidate to swap. This ensures correct handling of lexicographic order even with repeated elements.
