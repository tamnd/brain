## Solution

## Problem Understanding

The problem requires calculating the **top five average score** for each student given a list of `[ID, score]` pairs. Each pair represents a score earned by a student with a unique ID. The input, `items`, is a list of such pairs and may contain multiple scores for the same student. The output must be a list of `[ID, topFiveAverage]` pairs, where `topFiveAverage` is computed by summing the student’s five highest scores and dividing by 5 using integer division, discarding any fractional part. The final list must be sorted by student ID in increasing order.

The constraints clarify the problem scale and guarantee: there are at most 1000 scores and IDs range from 1 to 1000. Every student has at least five scores, which removes the need to handle cases where fewer than five scores exist. Edge cases that may affect naive implementations include students with exactly five scores, students with multiple identical scores, and the requirement to use integer division rather than floating-point division.

## Approaches

### Brute Force Approach

A naive approach would first collect all scores for each student into a list. Then, for each student, we could sort their scores in descending order and take the sum of the first five scores, dividing by five using integer division. While correct, this method involves sorting potentially large lists of scores for every student, which can be inefficient if a student has many scores, especially since sorting is $O(n \log n)$.

### Optimal Approach

The key insight for an optimal solution is that we do not need to sort all scores for a student-only the top five matter. We can maintain a **min-heap of size 5** for each student. As we iterate over scores, we add scores to the heap and, if the heap exceeds size 5, we remove the smallest score. This ensures that after processing all scores, each student’s heap contains exactly their top five scores. This avoids unnecessary full sorting and reduces overhead.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(n) | Collect scores, sort all for each student, then compute average |
| Optimal | O(n log 5) ≈ O(n) | O(m*5) | Maintain a min-heap per student to track top five scores |

Where $n$ is the number of scores in `items` and $m$ is the number of unique students.

## Algorithm Walkthrough

1. Initialize a hash map (`defaultdict`) where the key is the student ID and the value is a min-heap storing at most five scores.
2. Iterate over each `[ID, score]` pair in `items`. For each student ID, push the score onto the min-heap.
3. After pushing, check if the heap size exceeds five. If it does, remove the smallest score using `heappop`. This ensures that the heap always contains the top five scores.
4. Initialize an empty result list.
5. Iterate over all student IDs in sorted order.
6. For each student, calculate the sum of the scores in the min-heap and divide by 5 using integer division.
7. Append `[ID, topFiveAverage]` to the result list.
8. Return the result list.

**Why it works:** The invariant maintained is that the min-heap for each student never exceeds five elements, and it always contains the top five scores. Removing the smallest when the heap exceeds size five ensures only the highest scores remain. Sorting IDs guarantees the output order requirement.

## Python Solution

```
from typing import List
from collections import defaultdict
import heapq

class Solution:
    def highFive(self, items: List[List[int]]) -> List[List[int]]:
        student_scores = defaultdict(list)
        
        for student_id, score in items:
            heapq.heappush(student_scores[student_id], score)
            if len(student_scores[student_id]) > 5:
                heapq.heappop(student_scores[student_id])
        
        result = []
        for student_id in sorted(student_scores.keys()):
            top_five_sum = sum(student_scores[student_id])
            result.append([student_id, top_five_sum // 5])
        
        return result
```

**Explanation:** We use `defaultdict(list)` to group scores per student. Each score is pushed to a min-heap, which keeps only the top five by popping the smallest when necessary. Finally, we iterate over sorted IDs, compute the sum of each heap, perform integer division, and construct the result.

## Go Solution

```
package main

import (
    "container/heap"
    "sort"
)

type IntHeap []int

func (h IntHeap) Len() int { return len(h) }
func (h IntHeap) Less(i, j int) bool { return h[i] < h[j] }
func (h IntHeap) Swap(i, j int) { h[i], h[j] = h[j], h[i] }
func (h *IntHeap) Push(x interface{}) { *h = append(*h, x.(int)) }
func (h *IntHeap) Pop() interface{} {
    old := *h
    n := len(old)
    x := old[n-1]
    *h = old[:n-1]
    return x
}

func highFive(items [][]int) [][]int {
    studentScores := make(map[int]*IntHeap)
    
    for _, item := range items {
        id, score := item[0], item[1]
        if _, exists := studentScores[id]; !exists {
            h := &IntHeap{}
            heap.Init(h)
            studentScores[id] = h
        }
        heap.Push(studentScores[id], score)
        if studentScores[id].Len() > 5 {
            heap.Pop(studentScores[id])
        }
    }
    
    ids := make([]int, 0, len(studentScores))
    for id := range studentScores {
        ids = append(ids, id)
    }
    sort.Ints(ids)
    
    result := make([][]int, 0, len(ids))
    for _, id := range ids {
        sum := 0
        for _, score := range *studentScores[id] {
            sum += score
        }
        result = append(result, []int{id, sum / 5})
    }
    
    return result
}
```

**Explanation:** The Go version mirrors Python but uses a custom `IntHeap` type to implement the min-heap. Each student ID maps to a pointer to a heap. Sorting IDs and integer division logic follow the same pattern.
