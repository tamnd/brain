## Solution

## Problem Understanding

The problem asks us to count **confusing numbers** between `1` and `n`, inclusive. A confusing number is a number that, when rotated 180 degrees, produces a **different number**, and all of its digits must be valid after rotation. Valid digits are `0, 1, 6, 8, 9`, and the rotation mapping is `0->0, 1->1, 6->9, 8->8, 9->6`. Digits `2, 3, 4, 5, 7` are invalid. The output should be the total number of numbers in the range `[1, n]` that satisfy this property.

For example, the number `6` becomes `9` when rotated. Since `6 != 9`, it is a confusing number. However, `11` becomes `11` after rotation, so it is **not** confusing because the number does not change.

The input constraint is `1 <= n <= 10^9`, which implies that we cannot iterate through all numbers from `1` to `n` directly because it would be too slow for large `n`. We must instead use a more efficient approach. Edge cases include numbers containing invalid digits (which cannot be used) and numbers that become the same after rotation.

Important edge cases to consider:

1. `n` is very small, e.g., `n = 1` or `n = 5`. The answer should correctly exclude numbers that cannot rotate.
2. Numbers that rotate into themselves, like `11` or `88`, should not be counted.
3. Numbers with zeros, especially leading zeros after rotation, e.g., `10` rotates to `01` which is `1`.

## Approaches

The brute-force approach is to iterate through every number from `1` to `n`, rotate it, and check if the rotated number differs from the original. While simple and correct, it is far too slow because `n` can be up to `10^9`. This approach requires converting every number to a string and performing digit-by-digit rotation, giving a time complexity of roughly `O(n * log n)`.

The key observation for a better solution is that **we only need to generate numbers using the valid digits**, and then check if they are confusing. This can be done with **backtracking**: start from an empty number and recursively append digits `0,1,6,8,9`, skipping numbers greater than `n`. Each number can be checked efficiently for being confusing. This avoids iterating over invalid numbers entirely and drastically reduces the search space.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n log n) | O(log n) | Iterates all numbers from 1 to n, rotates each, checks for confusion |
| Optimal (Backtracking) | O(5^d) | O(d) | Generates numbers using valid digits recursively, only explores numbers <= n |

`d` is the number of digits in `n`. Since we only consider valid digits at each position, the recursion tree has a branching factor of at most 5.

## Algorithm Walkthrough

1. Define a mapping of valid digits to their rotation: `0->0, 1->1, 6->9, 8->8, 9->6`.
2. Initialize a counter to track the number of confusing numbers.
3. Write a recursive backtracking function that builds numbers digit by digit:

- If the current number exceeds `n`, return immediately.
- If the current number is confusing (rotated number differs from the original), increment the counter.
- For each valid digit, append it to the current number and recursively continue.
4. Start the recursion from an empty number or each valid starting digit (cannot start with `0`).
5. Return the final counter value.

**Why it works:** The recursion generates all valid numbers up to `n` using only valid digits. By checking each number for the confusing property, we count exactly all confusing numbers without wasting time on invalid numbers. Leading zeros are naturally handled because we avoid starting a number with zero.

## Python Solution

```
class Solution:
    def confusingNumberII(self, n: int) -> int:
        self.mapping = {'0':'0', '1':'1', '6':'9', '8':'8', '9':'6'}
        self.count = 0

        def is_confusing(num: int) -> bool:
            original = str(num)
            rotated = ''.join(self.mapping[d] for d in reversed(original))
            return rotated != original

        def backtrack(num: int):
            if num > n:
                return
            if num != 0 and is_confusing(num):
                self.count += 1
            for d in '01689':
                if num == 0 and d == '0':
                    continue
                next_num = num * 10 + int(d)
                backtrack(next_num)

        backtrack(0)
        return self.count
```

**Explanation:** We define a mapping for rotation and a recursive function `backtrack` to generate all numbers using valid digits. `is_confusing` checks if the rotated number differs from the original. The recursion ensures we only explore numbers `<= n`, and we skip leading zeros. Whenever a confusing number is found, we increment the counter.

## Go Solution

```
func confusingNumberII(n int) int {
    mapping := map[byte]byte{'0':'0','1':'1','6':'9','8':'8','9':'6'}
    count := 0

    var isConfusing func(int) bool
    isConfusing = func(num int) bool {
        original := []byte(fmt.Sprintf("%d", num))
        rotated := make([]byte, len(original))
        for i, j := 0, len(original)-1; i <= j; i, j = i+1, j-1 {
            rotated[i], rotated[j] = mapping[original[j]], mapping[original[i]]
        }
        return string(rotated) != string(original)
    }

    var backtrack func(int)
    backtrack = func(num int) {
        if num > n {
            return
        }
        if num != 0 && isConfusing(num) {
            count++
        }
        for _, d := range []byte{'0','1','6','8','9'} {
            if num == 0 && d == '0' {
                continue
            }
            nextNum := num*10 + int(d-'0')
            backtrack(nextNum)
        }
    }

    backtrack(0)
    return count
}
```

**Go Notes:** We use `byte` slices to manipulate digits and a recursive backtracking function. Leading zeros are skipped, and `isConfusing` compares the rotated string with the original.

## Complexity Analysis

| Measure | Complexity | Explanation |
| --- | --- | --- |
| Time | O(5^d) | We explore all numbers with valid digits up to the number of digits `d` in `n`. Each step involves rotation checking, which is O(d) but dominated by the branching factor. |
| Space | O(d) | The recursion stack grows up to the number of digits in `n`. |

The algorithm is efficient because it prunes invalid numbers early and only explores numbers constructed from `0,1,6,8,9`.
