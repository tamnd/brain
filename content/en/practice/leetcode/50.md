# LeetCode 50, Pow(x, n)

## Problem Statement

The problem asks us to implement a function that computes `x` raised to the power `n`, written mathematically as `xⁿ`.

We are given two inputs:

- `x`, a floating point number that represents the base
- `n`, an integer that represents the exponent

The goal is to return the value of `x` multiplied by itself `n` times.

For example:

- `2^10 = 1024`
- `2.1^3 = 2.1 × 2.1 × 2.1 = 9.261`
- `2^-2 = 1 / (2^2) = 1/4 = 0.25`

The challenge comes from the size of `n`. The exponent can be as small as `-2^31` and as large as `2^31 - 1`. A naive solution that multiplies `x` repeatedly would become far too slow for very large exponents.

The constraints also tell us several important things:

- The exponent is always an integer.
- The base can be negative or fractional.
- Negative exponents must be handled correctly.
- `x` can be zero only when `n > 0`, so cases like `0^-1` do not occur.
- The final answer is guaranteed to stay within a manageable numeric range.

Several edge cases are especially important:

- `n = 0`, because any nonzero number raised to the zero power equals `1`
- Negative exponents, because they require taking the reciprocal
- Very large exponents, where brute force becomes infeasible
- `n = -2^31`, because negating this value directly can cause overflow in some languages
- Negative bases, where the sign of the result depends on whether the exponent is even or odd

## Approaches

### Brute Force Approach

The simplest approach is to multiply `x` by itself `n` times.

If `n` is positive:

```
result = x * x * x * ... * x
```

If `n` is negative:

```
result = 1 / (x * x * x * ... * x)
```

This approach is straightforward and produces the correct result because it directly follows the mathematical definition of exponentiation.

However, the problem is efficiency. If `n` is close to `2^31`, we would need billions of multiplications, which is far too slow.

### Optimal Approach, Binary Exponentiation

The key insight is that powers can be broken into smaller subproblems.

Consider:

```
x^10
```

Instead of multiplying `x` ten times, we can compute:

```
x^10 = (x^5)^2
x^5 = x * (x^2)^2
x^2 = x * x
```

More generally:

- If `n` is even:

```
x^n = (x^(n/2))^2
```

- If `n` is odd:

```
x^n = x * x^(n-1)
```

This reduces the exponent by half at each step, which dramatically improves performance.

This technique is called binary exponentiation, or exponentiation by squaring.

| Approach | Time Complexity | Space Complexity | Notes |
| --- | --- | --- | --- |
| Brute Force | O(n) | O(1) | Multiplies the base repeatedly |
| Optimal | O(log n) | O(1) | Uses exponentiation by squaring |

## Algorithm Walkthrough

1. Start by checking whether the exponent is negative.

Negative exponents follow the mathematical rule:

$x^{-n}=\frac{1}{x^n}$

So if `n` is negative, we invert the base:

```
x = 1 / x
```

Then convert `n` to a positive value.
2. Initialize two variables:

- `result = 1`
- `current_power = x`

The `result` variable stores the accumulated answer.
3. Process the exponent one bit at a time.

Binary exponentiation works because every integer can be represented in binary form.

For example:

```
13 = 1101₂
```

Which means:

```
13 = 8 + 4 + 1
```

Therefore:

```
x^13 = x^8 * x^4 * x^1
```
4. While `n > 0`:

- If the current bit is `1`:

Multiply `result` by `current_power`.
- Square `current_power`.

This moves from:

```
x -> x² -> x⁴ -> x⁸
```
- Divide `n` by `2` using integer division.
5. Continue until all bits of `n` have been processed.
6. Return `result`.

This algorithm is efficient because each iteration cuts the exponent in half.

## Python Solution

```
class Solution:
    def myPow(self, x: float, n: int) -> float:
        if n < 0:
            x = 1 / x
            n = -n

        result = 1.0
        current_power = x

        while n > 0:
            if n % 2 == 1:
                result *= current_power

            current_power *= current_power
            n //= 2

        return result
```

The implementation begins by handling negative exponents. Instead of treating them separately throughout the algorithm, we convert the problem into an equivalent positive exponent problem by taking the reciprocal of the base.

The `result` variable starts at `1.0` because multiplying by `1` does not change the answer. The `current_power` variable represents the current power of `x` being considered.

Inside the loop, we check whether the current exponent is odd. If it is, the current power contributes to the final answer, so we multiply it into `result`.

Then we square `current_power`, which advances us from:

```
x -> x² -> x⁴ -> x⁸
```

Finally, we halve the exponent using integer division.

The loop ends once all bits of the exponent have been processed.

## Go Solution

```
func myPow(x float64, n int) float64 {
    exponent := n

    if exponent < 0 {
        x = 1 / x
        exponent = -exponent
    }

    result := 1.0
    currentPower := x

    for exponent > 0 {
        if exponent%2 == 1 {
            result *= currentPower
        }

        currentPower *= currentPower
        exponent /= 2
    }

    return result
}
```

The Go implementation follows the same logic as the Python version.

One important detail in Go is integer overflow handling. In some environments, negating the minimum integer value can overflow because its positive counterpart cannot be represented. LeetCode's Go environment typically handles this safely with the provided constraints, but in production code it is common to convert the exponent to a larger integer type such as `int64`.

The rest of the implementation mirrors the Python solution closely.

## Complexity Analysis

| --- | --- | --- |

The time complexity is logarithmic because each iteration reduces the exponent by a factor of two. For example, an exponent of one billion would require only around thirty iterations.

The space complexity is constant because the algorithm uses only a fixed number of variables regardless of input size.

## Edge Cases

### Exponent Equals Zero

Any nonzero number raised to the power `0` equals `1`.

For example:

$x^0=1$

A common bug is forgetting to handle this case explicitly. In this implementation, the loop never executes when `n = 0`, so `result` remains `1.0`, which is correct.

### Negative Exponents

Negative exponents require taking the reciprocal of the base.

For example:

```
2^-3 = 1 / 2^3 = 1/8
```

A naive implementation might repeatedly divide instead of multiply, which complicates the logic. This solution simplifies the problem by converting it into a positive exponent calculation upfront.

### Very Large Exponents

A brute force solution would time out for exponents near `2^31`.

For example:

```
x = 1.0001
n = 2147483647
```

The binary exponentiation approach handles this efficiently because it performs only logarithmic work.

### Minimum Integer Exponent

The value:

```
n = -2^31
```

can be problematic in some languages because:

```
abs(-2^31)
```

cannot fit into a signed 32 bit integer.

Python handles large integers automatically, so the Python solution is safe. In lower level languages, extra care may be required to avoid overflow.

### Negative Bases

Negative bases produce different signs depending on whether the exponent is even or odd.

For example:

```
(-2)^3 = -8
(-2)^4 = 16
```

The implementation naturally handles this because multiplication preserves the correct sign behavior automatically.
