[Project Euler Problem 483](https://projecteuler.net/problem=483)

## Solution

Let the order of a permutation $P$ be

$$f(P)=\operatorname{lcm}(\text{cycle lengths of }P).$$

We need

$$g(n)=\frac1{n!}\sum_{P\in S_n} f(P)^2.$$

For a cycle type with $m_c$ cycles of length $c$,

$$\sum_{c\ge1} c\,m_c=n,$$

and the number of permutations with that cycle type is

$$\frac{n!}{\prod_c c^{m_c}m_c!}.$$

Therefore

$$g(n)=
\sum_{\sum c m_c=n}
\frac{
\operatorname{lcm}\{c:m_c>0\}^2
}{
\prod_c c^{m_c}m_c!
}.$$

The direct partition enumeration is impossible for $n=350$, so we use dynamic programming over:

- total used length,
- current lcm of chosen cycle lengths.

The crucial optimization is to process cycle lengths grouped by their **largest prime factor**.

After all cycle lengths containing a prime $p$ have been processed, the exponent of $p$ in the current lcm is final forever. Since the final answer uses the square of the lcm, we can immediately absorb that factor into the DP weight and remove $p$ from the state key. This compression keeps the state space small enough.

The resulting algorithm reproduces:

- $g(3)=31/6$,
- $g(5)=2081/120$,
- $g(20)\approx 5.106136147\times 10^3$,

exactly as required.

A correct Python implementation is:

```
from math import gcd
from decimal import Decimal, getcontext
from collections import defaultdict

N = 350
getcontext().prec = 80

# largest prime factor sieve
lpf = list(range(N + 1))
for p in range(2, N + 1):
    if lpf[p] == p:
        for k in range(p, N + 1, p):
            lpf[k] = p

# group cycle lengths by largest prime factor
groups = defaultdict(list)
for c in range(2, N + 1):
    groups[lpf[c]].append(c)

# dp[t][l] = accumulated weight
dp = [defaultdict(Decimal) for _ in range(N + 1)]

# fixed points only
fact = Decimal(1)
dp[0][1] = Decimal(1)
for t in range(1, N + 1):
    fact *= t
    dp[t][1] = Decimal(1) / fact

# process primes descending
for p in sorted(groups.keys(), reverse=True):

    # add cycle lengths whose largest prime factor is p
    for c in groups[p]:

        coeff = [Decimal(1)]
        cur = Decimal(1)

        for m in range(1, N // c + 1):
            cur /= Decimal(c * m)
            coeff.append(cur)

        for t in range(N - c, -1, -1):

            items = list(dp[t].items())
            if not items:
                continue

            for m in range(1, (N - t) // c + 1):

                wmul = coeff[m]
                target = dp[t + m * c]

                for l, w in items:
                    nl = l * c // gcd(l, c)
                    target[nl] += w * wmul

    # compress finished prime p
    for t in range(N + 1):

        nd = defaultdict(Decimal)

        for l, w in dp[t].items():

            a = 0
            while l % p == 0:
                l //= p
                a += 1

            if a:
                w *= Decimal(p ** (2 * a))

            nd[l] += w

        dp[t] = nd

# final expectation
ans = sum(Decimal(l * l) * w for l, w in dp[N].items())

print(ans)
```

Running this gives

$$g(350)=4.9934015671416996024\ldots\times 10^{22}.$$

Rounded to $10$ significant digits:

**Answer: 4.993401567e22**
