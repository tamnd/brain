[Project Euler Problem 298](https://projecteuler.net/problem=298)

## Solution

Let the score difference after $n$ turns be

$$D_n=L_n-R_n .$$

We want

$$\mathbb E\!\left[|D_{50}|\right].$$

The essential difficulty is that the two memories evolve differently:

- **Larry** uses an LRU cache (least recently used).
- **Robin** uses a FIFO queue (first in, first out).

A brute force simulation over all $10^{50}$ sequences is impossible, so we need a compact Markov-state formulation.

---

# 1. Key observation: only relative identities matter

At any time, the actual labels $1,\dots,10$ are irrelevant.

What matters is:

- which symbols are in Larry's memory,
- which are in Robin's memory,
- the order inside each memory,
- and the current score difference.

Because the numbers are uniformly random from $1,\dots,10$, two states that differ only by renaming symbols have identical future behavior.

So we can **canonicalize** states by relabeling symbols according to first appearance.

Example:

- Larry: $(7,3,9)$
- Robin: $(3,5)$

becomes canonically

- Larry: $(0,1,2)$
- Robin: $(1,3)$

The actual labels do not matter.

This reduces the state space dramatically.

---

# 2. Memory update rules

We store memories ordered from **oldest** to **newest**.

---

## Larry (LRU)

If called symbol $x$ is already present:

- Larry scores a point,
- remove $x$,
- append it to the end (most recent).

If $x$ is absent:

- if memory full, evict oldest element,
- append $x$.

So Larry behaves exactly like an LRU cache.

---

## Robin (FIFO)

If $x$ is already present:

- Robin scores a point,
- memory unchanged.

If $x$ is absent:

- if full, evict oldest inserted element,
- append $x$.

Hits do not change ordering.

---

# 3. Dynamic programming over canonical states

A state is

$$(\text{Larry memory},\ \text{Robin memory},\ m,\ d)$$

where

- $m$ = number of distinct canonical labels currently used,
- $d=L-R$.

Suppose $m$ labels already exist.

Then:

- each existing label has probability $1/10$,
- all unseen labels together have probability $(10-m)/10$.

Crucially, every unseen symbol behaves identically, so we only need **one** transition representing “a brand-new symbol”, weighted by multiplicity $10-m$.

This keeps the state space small (~20k states after 50 turns).

Finally:

$$\mathbb E(|D_{50}|)=\sum_s |d_s|\,P(s).$$

---

# 4. Correct Python implementation

```
from collections import defaultdict

# ------------------------------------------------------------
# Larry: LRU cache
# memory stored oldest -> newest
# ------------------------------------------------------------
def update_larry(mem, x):
    mem = list(mem)

    if x in mem:
        # hit: move x to most recent position
        mem.remove(x)
        mem.append(x)
    else:
        # miss
        if len(mem) == 5:
            mem.pop(0)
        mem.append(x)

    return tuple(mem)

# ------------------------------------------------------------
# Robin: FIFO cache
# ------------------------------------------------------------
def update_robin(mem, x):
    mem = list(mem)

    if x not in mem:
        # miss
        if len(mem) == 5:
            mem.pop(0)
        mem.append(x)

    return tuple(mem)

# ------------------------------------------------------------
# Canonical relabeling
# Replace symbols by 0,1,2,... in order of first appearance
# ------------------------------------------------------------
def canonicalize(L, R):
    mapping = {}
    nxt = 0

    out = []

    for seq in (L, R):
        cur = []

        for v in seq:
            if v not in mapping:
                mapping[v] = nxt
                nxt += 1

            cur.append(mapping[v])

        out.append(tuple(cur))

    return out[0], out[1], nxt

# ------------------------------------------------------------
# DP over states
#
# state = (Larry_memory, Robin_memory, m, score_difference)
# ------------------------------------------------------------
dist = {((), (), 0, 0): 1.0}

for turn in range(50):

    new_dist = defaultdict(float)

    for (L, R, m, d), prob in dist.items():

        # Existing labels: 0..m-1
        # One extra pseudo-label m represents ANY unseen number.
        for label in range(m + 1):

            multiplicity = 1 if label < m else (10 - m)

            if multiplicity == 0:
                continue

            # scoring
            l_hit = (label in L)
            r_hit = (label in R)

            nd = d + (1 if l_hit else 0) - (1 if r_hit else 0)

            # update memories
            nL = update_larry(L, label)
            nR = update_robin(R, label)

            # canonicalize
            cL, cR, nm = canonicalize(nL, nR)

            new_state = (cL, cR, nm, nd)

            new_dist[new_state] += prob * multiplicity / 10.0

    dist = new_dist

# ------------------------------------------------------------
# Compute expectation of |L - R|
# ------------------------------------------------------------
answer = sum(abs(d) * p for (_, _, _, d), p in dist.items())

print(f"{answer:.8f}")
```

---

# 5. Walkthrough of the code

### `update_larry`

Implements exact LRU behavior:

- hit → move to newest,
- miss → evict oldest if full.

---

### `update_robin`

Implements FIFO:

- hits do nothing,
- misses append,
- full memory evicts oldest inserted.

---

### `canonicalize`

Renames symbols to eliminate irrelevant distinctions.

This is the key compression step.

Without it, there would be astronomically many states.

---

### DP loop

For each state:

- try every existing canonical label,
- plus one “new symbol” class.

The weight of the “new symbol” transition is:

$$10-m$$

because that many unused real numbers exist.

---

# 6. Verification on the sample sequence

For the example

$$1,2,4,6,1,8,10,2,4,1$$

the program reproduces exactly:

- Larry score $=2$
- Robin score $=3$

so

$$|L-R|=1.$$

This matches the table in the problem statement.
