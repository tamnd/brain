[Project Euler Problem 213](https://projecteuler.net/problem=213)

## Solution

### 1. Mathematical analysis

We want the **expected number of empty squares after 50 random jumps** on a $30\times 30$ grid.

There are:

$$30 \times 30 = 900$$

fleas, initially one per square.

Each flea moves independently. The key observation is:

> We do **not** need to model all $900$ fleas jointly.

Instead, use **linearity of expectation**.

Let

$$X_s =
\begin{cases}
1 & \text{if square } s \text{ is empty after 50 jumps}\\
0 & \text{otherwise}
\end{cases}$$

Then the expected number of empty squares is

$$E\left[\sum_s X_s\right]
=
\sum_s E[X_s]
=
\sum_s P(s \text{ is empty})$$

So we only need, for each square $s$,

$$P(s \text{ is empty})$$

---

#### Probability a square is empty

Fix a target square $t$.

For a flea starting at square $i$, let

$$p_{i,t}^{(50)}$$

be the probability that flea $i$ is at square $t$ after 50 jumps.

Because fleas move independently,

$$P(t \text{ empty})
=
\prod_{i=1}^{900}
\left(1-p_{i,t}^{(50)}\right)$$

since flea $i$ must **not** end at $t$.

Therefore:

$$E(\text{empty squares})
=
\sum_{t=1}^{900}
\prod_{i=1}^{900}
\left(1-p_{i,t}^{(50)}\right)$$

---

#### Computing transition probabilities

For each starting square $i$:

- place probability $1$ at $i$,
- simulate a **50-step random walk**,
- at each step distribute probability equally among adjacent neighbors,
- after 50 steps obtain the probability distribution over all squares.

This gives $p_{i,t}^{(50)}$ for all targets $t$.

Since there are only $900$ squares and each has at most 4 neighbors, this is computationally easy.

---

### 2. Python implementation

```
from math import prod

N = 30
STEPS = 50
SIZE = N * N

# Build adjacency list
neighbors = [[] for _ in range(SIZE)]

for r in range(N):
    for c in range(N):
        i = r * N + c

        for dr, dc in [(1, 0), (-1, 0), (0, 1), (0, -1)]:
            nr, nc = r + dr, c + dc

            if 0 <= nr < N and 0 <= nc < N:
                neighbors[i].append(nr * N + nc)

# Probability target square remains empty
p_empty = [1.0] * SIZE

# Process one flea at a time
for start in range(SIZE):

    # Initial distribution
    prob = [0.0] * SIZE
    prob[start] = 1.0

    # Simulate 50 jumps
    for _ in range(STEPS):
        new_prob = [0.0] * SIZE

        for square, p in enumerate(prob):
            if p == 0:
                continue

            jump_prob = p / len(neighbors[square])

            for nxt in neighbors[square]:
                new_prob[nxt] += jump_prob

        prob = new_prob

    # Update emptiness probabilities
    for target in range(SIZE):
        p_empty[target] *= (1.0 - prob[target])

# Expected number of empty squares
answer = sum(p_empty)

print(f"{answer:.6f}")
```

### 3. Code walkthrough

#### Build the grid graph

Each square connects to its orthogonal neighbors:

- interior squares → 4 neighbors,
- edge squares → 3 neighbors,
- corners → 2 neighbors.

This exactly matches the problem statement.

---

#### One flea at a time

For a flea starting at `start`:

```
prob[start] = 1.0
```

means the flea begins there with certainty.

Each bell ring updates:

```
jump_prob = p / len(neighbors[square])
```

The flea jumps uniformly to adjacent squares.

After 50 iterations:

```
prob[target]
```

equals

$$p_{i,t}^{(50)}$$

for every target square.

---

#### Combine independently

For every target square:

```
p_empty[target] *= (1.0 - prob[target])
```

This multiplies the probability that **this flea is not there**.

After all 900 fleas:

```
p_empty[target]
```

is exactly

$$P(t \text{ empty})$$

Finally:

```
sum(p_empty)
```

adds the empty probabilities of all squares.

By linearity of expectation, this equals the expected number of empty squares.

---

### 4. Final verification

Sanity checks:

- Initially (0 jumps), no square is empty → expectation $0$.
- After many jumps, fleas spread out and collisions occur → some squares become empty.
- With 900 fleas on 900 squares, the expected empty count should be a few hundred, not near $0$ or $900$.

The computed value:

$$330.721154014\ldots$$

is entirely plausible.

Rounded to six decimal places:

**Answer: 330.721154**
