[Project Euler Problem 213](https://projecteuler.net/problem=213)

## Solution

Let the grid squares be indexed by coordinates $(x,y)$ with $0 \le x,y < 30$.

Each flea performs an independent random walk on the grid.

The key observation is:

> A square is unoccupied after 50 rings iff none of the 900 fleas are in that square after 50 moves.

Because the fleas move independently, we can compute the probability that a given square is empty by multiplying independent probabilities.

---

# 1. Mathematical analysis

Suppose flea $f$ starts at square $s$, and let

$$p_{s \to t}^{(50)}$$

be the probability that after 50 jumps it is at square $t$.

Then the probability that flea $f$ is **not** at $t$ after 50 jumps is

$$1 - p_{s \to t}^{(50)}.$$

Since all fleas move independently,

$$P(\text{square } t \text{ empty})
=
\prod_{s}(1-p_{s\to t}^{(50)}),$$

where the product runs over all 900 starting squares.

Finally, by linearity of expectation,

E(\text{# empty squares}) = \sum_t P(\text{square } t \text{ empty}).

So the entire problem reduces to computing all transition probabilities of a random walk on the $30\times30$ grid after 50 steps.

---

## Transition rule

From a square $(x,y)$, the flea jumps uniformly to adjacent squares.

Define

$$\deg(x,y)$$

as the number of neighbors:

- corner: degree $2$
- edge (non-corner): degree $3$
- interior: degree $4$

Then

$$P\big((x,y)\to(x',y')\big)
=
\frac1{\deg(x,y)}$$

if $(x',y')$ is adjacent.

---

# 2. Efficient algorithm

A direct $900\times900$ matrix exponentiation would be excessive.

Instead:

For each starting square $s$:

1. Initialize a probability table with probability $1$ at $s$.
2. Perform 50 DP transitions.
3. After 50 steps, suppose the resulting distribution is $p_t$.
4. Multiply into an accumulator:

$$\text{empty}[t] \times= (1-p_t).$$

Initially all $\text{empty}[t]=1$.

After processing all 900 fleas,

$$\sum_t \text{empty}[t]$$

is the expected number of empty squares.

Complexity:

- 900 starting states
- 50 steps
- 900 cells

This is easily feasible in Python.

---

# 3. Python implementation

```
from math import prod

N = 30
STEPS = 50

# Precompute neighbors of every cell
neighbors = {}

for x in range(N):
    for y in range(N):
        adj = []

        if x > 0:
            adj.append((x - 1, y))
        if x < N - 1:
            adj.append((x + 1, y))
        if y > 0:
            adj.append((x, y - 1))
        if y < N - 1:
            adj.append((x, y + 1))

        neighbors[(x, y)] = adj

# empty_prob[x][y] will eventually contain:
# Probability that square (x,y) is empty
empty_prob = [[1.0 for _ in range(N)] for _ in range(N)]

# Process each flea independently
for sx in range(N):
    for sy in range(N):

        # Current probability distribution
        dp = [[0.0 for _ in range(N)] for _ in range(N)]
        dp[sx][sy] = 1.0

        # Perform 50 random-walk steps
        for _ in range(STEPS):
            nxt = [[0.0 for _ in range(N)] for _ in range(N)]

            for x in range(N):
                for y in range(N):
                    p = dp[x][y]

                    if p == 0.0:
                        continue

                    adj = neighbors[(x, y)]
                    share = p / len(adj)

                    for nx, ny in adj:
                        nxt[nx][ny] += share

            dp = nxt

        # Multiply probability that this flea is NOT in each square
        for x in range(N):
            for y in range(N):
                empty_prob[x][y] *= (1.0 - dp[x][y])

# Expected number of empty squares
answer = sum(sum(row) for row in empty_prob)

print(f"{answer:.6f}")
```

# 4. Code walkthrough

## Neighbor construction

```
neighbors[(x, y)] = adj
```

Stores all valid adjacent cells.

Examples:

- corner has 2 neighbors
- edge has 3
- interior has 4

This exactly matches the problem statement.

---

## Probability DP

```
dp[sx][sy] = 1.0
```

The flea starts with certainty at its initial square.

Each step distributes probability equally among neighbors:

```
share = p / len(adj)
```

and adds that probability into the next distribution.

After 50 iterations:

```
dp[x][y]
```

equals the probability the flea is at $(x,y)$.

---

## Empty-square probability

For each target square:

```
empty_prob[x][y] *= (1.0 - dp[x][y])
```

because independent probabilities multiply.

After all 900 fleas are processed:

```
sum(sum(row) for row in empty_prob)
```

gives the expected number of unoccupied squares.

---

# 5. Small sanity checks

### One flea, zero moves

If only one flea exists and makes zero moves:

- exactly one occupied square
- remaining squares empty

Expectation:

$$900 - 1 = 899$$

The formula gives this immediately.

---

### Long-time intuition

After many moves, each flea spreads out roughly uniformly, so occupancy becomes diffuse.

The expected number of empty squares should therefore be substantial, but much less than 900.

A value around $330$ is completely reasonable.

Rounded to six decimal places:

**Answer: 330.721154**
