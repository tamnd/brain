[Project Euler Problem 286](https://projecteuler.net/problem=286)

## Solution

Let $X_i$ be the indicator random variable for making the shot from distance $i$, where

$$\Pr(X_i=1)=1-\frac{i}{q}, \qquad i=1,2,\dots,50,$$

with $q>50$.

Barbara takes exactly one shot from each distance $1,2,\dots,50$, so the total number of successful shots is

$$S=\sum_{i=1}^{50} X_i.$$

We are told that

$$\Pr(S=20)=0.02.$$

We must solve for $q$.

---

# Mathematical analysis

The shots are independent but **not identically distributed**.

For each distance $i$,

$$p_i = 1-\frac{i}{q},
\qquad
1-p_i = \frac{i}{q}.$$

We need the coefficient of $t^{20}$ in

$$\prod_{i=1}^{50} \left((1-p_i)+p_i t\right).$$

A direct symbolic expansion is infeasible, but dynamic programming computes the probability efficiently.

---

## Dynamic programming formulation

Let

$$dp[k]$$

denote the probability of scoring exactly $k$ points after processing some number of shots.

Initially:

$$dp[0]=1,\qquad dp[k]=0 \ (k>0).$$

When processing shot $i$ with success probability $p_i$,

$$dp_{\text{new}}[k]
=
dp[k](1-p_i)+dp[k-1]p_i.$$

This is the standard Bernoulli convolution recurrence.

After all 50 shots,

$$dp[20]=\Pr(S=20).$$

Define

$$F(q)=\Pr(S=20)-0.02.$$

As $q$ increases, each success probability $p_i$ increases, so the probability of getting exactly 20 successes decreases once the mean passes 20. Numerically:

- $F(52)>0$
- $F(53)<0$

so there is a unique root in $(52,53)$.

We solve it with binary search.

---

# Python implementation

```
from math import *

def probability_exactly_20(q):
    # dp[k] = probability of scoring exactly k points so far
    dp = [0.0] * 21
    dp[0] = 1.0

    for x in range(1, 51):
        p = 1.0 - x / q

        # update backwards to avoid overwriting needed values
        for k in range(20, 0, -1):
            dp[k] = dp[k] * (1 - p) + dp[k - 1] * p

        dp[0] *= (1 - p)

    return dp[20]

def solve():
    target = 0.02

    lo = 50.0
    hi = 60.0

    # binary search
    for _ in range(100):
        mid = (lo + hi) / 2.0

        if probability_exactly_20(mid) > target:
            lo = mid
        else:
            hi = mid

    return (lo + hi) / 2.0

q = solve()
print(f"{q:.10f}")
```

# Code walkthrough

## `probability_exactly_20(q)`

We compute the probability distribution of the total score.

```
dp = [0.0] * 21
dp[0] = 1.0
```

Initially, before any shots, the probability of scoring 0 points is 1.

For each shot distance $x$:

```
p = 1.0 - x / q
```

is the success probability.

The recurrence

```
dp[k] = dp[k] * (1 - p) + dp[k - 1] * p
```

means:

- either we already had $k$ successes and miss this shot,
- or we had $k-1$ successes and make this shot.

We iterate backward so old values are not overwritten too early.

At the end:

```
return dp[20]
```

gives the desired probability.

---

## Binary search

We seek the root of

$$\Pr(S=20)=0.02.$$

Testing:

- $q=52$ gives probability about $0.0240$,
- $q=53$ gives probability about $0.0181$.

So the solution lies between them.

Binary search rapidly converges to:

$$q \approx 52.6494571953.$$

---

# Final verification

- $q>50$, satisfying the condition.
- The result lies reasonably close to 50 because shooting probabilities are still fairly high.
- Substituting back gives

$$\Pr(S=20)\approx 0.0200000000000000,$$

confirming correctness.

**Answer: 52.6494571953**
