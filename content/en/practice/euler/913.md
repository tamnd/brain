[Project Euler Problem 913](https://projecteuler.net/problem=913)

## Solution

**Answer:** `2101925115560555020`

Let $N=nm$.

Number the entries $0,1,\dots,N-1$.

In row-major order, entry $i$ is at

$$(r,c)=\left(\left\lfloor \frac{i}{m}\right\rfloor,\ i\bmod m\right).$$

In column-major order, the same entry moves to

$$(r',c')=\left(i\bmod n,\ \left\lfloor \frac{i}{n}\right\rfloor\right),$$

whose row-major index is

$$p(i)=(i\bmod n)m+\left\lfloor\frac{i}{n}\right\rfloor.$$

For $0<i<N-1$,

$$p(i)\equiv im \pmod{N-1}.$$

Thus the permutation is multiplication by $m$ modulo $N-1$, with two fixed points $0$ and $N-1$.

For any permutation, the minimal number of swaps needed to realize it is

$$N-(\text{number of cycles}).$$

The cycle count of multiplication by $m$ modulo $L=N-1$ is

$$1+\sum_{d\mid L}\frac{\varphi(d)}{\operatorname{ord}_d(m)},$$

so

$$S(n,m)=nm-\left(1+\sum_{d\mid (nm-1)}
\frac{\varphi(d)}{\operatorname{ord}_d(m)}\right).$$

Applying this to $(n^4,m^4)$ and summing over

$$2\le n\le m\le 100$$

gives the following Python computation:

```
import math
from functools import lru_cache
from sympy import factorint

@lru_cache(None)
def factor_cached(n):
    return factorint(n)

@lru_cache(None)
def order_mod_prime_power(a, p, e):
    mod = p**e
    lam = (p - 1) * p**(e - 1)

    fac = factor_cached(lam)

    ordv = lam
    for q, eq in fac.items():
        for _ in range(eq):
            if ordv % q == 0 and pow(a, ordv // q, mod) == 1:
                ordv //= q
            else:
                break
    return ordv

def gen_divs(items, idx=0, cur=1, phi=1, fac=()):
    if idx == len(items):
        yield cur, phi, fac
    else:
        p, e = items[idx]

        yield from gen_divs(items, idx + 1, cur, phi, fac)

        pe = 1
        for k in range(1, e + 1):
            pe *= p
            yield from gen_divs(
                items,
                idx + 1,
                cur * pe,
                phi * (p - 1) * p**(k - 1),
                fac + ((p, k),)
            )

def order_from_fac(a, fac):
    o = 1
    for p, e in fac:
        oo = order_mod_prime_power(a % (p**e), p, e)
        o = o * oo // math.gcd(o, oo)
    return o

ans = 0

for n in range(2, 101):
    a = n**4

    for m in range(n, 101):
        b = m**4

        N = a * b
        L = N - 1

        items = tuple(factor_cached(L).items())

        cycles = 2  # fixed points 0 and N-1

        for d, phi, fac in gen_divs(items):
            if d != 1:
                cycles += phi // order_from_fac(b, fac)

        ans += N - cycles

print(ans)
```

The computation returns:

$$2101925115560555020$$

This is consistent with the given check value

$$\sum_{2\le n\le m\le100} S(n,m)=12578833.$$

**Answer: 2101925115560555020**
