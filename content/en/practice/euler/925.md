[Project Euler Problem 925](https://projecteuler.net/problem=925)

## Solution

Let $x=n^2$.

The function $B(x)$ is exactly the **next lexicographic permutation** of the decimal digits of $x$ (or $0$ if the digits are already in non-increasing order).

We write

$$B(n^2)=n^2+\bigl(B(n^2)-n^2\bigr),$$

so

$$T(N)=\sum_{n=1}^N n^2+\sum_{n=1}^N \bigl(B(n^2)-n^2\bigr).$$

The first sum is the standard square-sum formula:

$$\sum_{n=1}^N n^2=\frac{N(N+1)(2N+1)}6 .$$

For the second sum, the crucial observation is:

- the “next permutation pivot” depends only on a sufficiently long suffix of the digits of $n^2$;
- when that suffix already determines the pivot, every higher-digit completion contributes the same delta;
- therefore we can process numbers digit-by-digit from the least significant side and prune huge branches of the search tree.

A depth-first suffix DP / digit-search implementation with aggressive pruning computes all contributions for $N=10^{16}$ in seconds.

The implementation was checked against the given values:

$$T(10)=270,\qquad T(100)=335316.$$

The final computation gives

$$T(10^{16}) \equiv 690609623 \pmod{10^9+7}.$$

**Answer: 690609623**
