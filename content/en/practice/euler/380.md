[Project Euler Problem 380](https://projecteuler.net/problem=380)

## Solution

A maze on an $m\times n$ rectangular grid with exactly one path between any two cells is precisely a spanning tree of the $m\times n$ grid graph.

So the problem is asking for the number of spanning trees of the grid graph $G_{m,n}$.

Let $C(m,n)$ denote this number.

---

# 1. Mathematical analysis

The $m\times n$ grid graph has:

- $mn$ vertices,
- edges between horizontally and vertically adjacent cells.

By the Matrix–Tree Theorem, the number of spanning trees of a graph equals any cofactor of its Laplacian matrix.

For rectangular grids, the Laplacian eigenvalues are known explicitly.

For the $m\times n$ grid graph, the nonzero Laplacian eigenvalues are

$$\lambda_{a,b}
=
4
-
2\cos\left(\frac{a\pi}{m}\right)
-
2\cos\left(\frac{b\pi}{n}\right),$$

where

$$0\le a\le m-1,\qquad
0\le b\le n-1,$$

excluding $(a,b)=(0,0)$.

The Matrix–Tree Theorem gives:

$$C(m,n)
=
\frac1{mn}
\prod_{\substack{0\le a<m\\0\le b<n\\(a,b)\ne(0,0)}}
\left(
4
-
2\cos\frac{a\pi}{m}
-
2\cos\frac{b\pi}{n}
\right).$$

This is the classical spanning-tree formula for grid graphs.

---

## Check against the examples

### $2\times2$

$$C(2,2)=4.$$

Correct.

### $3\times4$

Evaluating the product gives

$$C(3,4)=2415.$$

Correct.

So the formula matches the problem statement.

# 2. Python implementation

```
import math

def maze_count(m, n):
    # We accumulate logarithms to avoid gigantic intermediate numbers.
    log10_value = 0.0

    for a in range(m):
        for b in range(n):

            # Skip the zero eigenvalue
            if a == 0 and b == 0:
                continue

            term = (
                4
                - 2 * math.cos(math.pi * a / m)
                - 2 * math.cos(math.pi * b / n)
            )

            log10_value += math.log10(term)

    # Matrix-Tree theorem contributes a factor 1/(mn)
    log10_value -= math.log10(m * n)

    exponent = int(math.floor(log10_value))
    mantissa = 10 ** (log10_value - exponent)

    return mantissa, exponent

# Small checks
print(maze_count(2, 2))   # should correspond to 4
print(maze_count(3, 4))   # should correspond to 2415

# Required computation
mantissa, exponent = maze_count(100, 500)

print(f"{mantissa:.4f}e{exponent}")
```

---

# 3. Code walkthrough

### Accumulating logarithms

The actual number $C(100,500)$ has over 25,000 digits, so direct multiplication is impractical.

Instead we use:

$$\log_{10}(xy)=\log_{10}x+\log_{10}y.$$

So we compute

$$\log_{10} C(m,n)
=
\sum \log_{10}(\lambda_{a,b})
-
\log_{10}(mn).$$

---

### Eigenvalue computation

For every pair $(a,b)\neq(0,0)$:

```
term = (
    4
    - 2 * cos(pi*a/m)
    - 2 * cos(pi*b/n)
)
```

This is the Laplacian eigenvalue.

---

### Recovering scientific notation

If

$$x = \log_{10} C,$$

then:

- exponent $= \lfloor x \rfloor$,
- mantissa $= 10^{x-\lfloor x\rfloor}$.

So

$$C = \text{mantissa} \times 10^{\text{exponent}}.$$

---

# 4. Final verification

The computation reproduces exactly:

- $C(2,2)=4$,
- $C(3,4)=2415$,

and for $9\times12$ gives approximately

$$2.5720\times10^{46},$$

matching the statement.

Running the same computation for $100\times500$ gives:

$$6.3202\times10^{25093}.$$

The magnitude is entirely reasonable since the number of spanning trees of a large grid grows exponentially in the number of vertices.
