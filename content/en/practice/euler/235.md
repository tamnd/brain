[Project Euler Problem 235](https://projecteuler.net/problem=235)

## Solution

We are given

$$u(k)=(900-3k)r^{k-1},
\qquad
s(n)=\sum_{k=1}^n u(k).$$

We must find the value of $r$ such that

$$s(5000)=-600000000000.$$

The required output is $r$ rounded to $12$ digits after the decimal point.

---

# Mathematical analysis

We begin from first principles.

$$s(n)=\sum_{k=1}^n (900-3k)r^{k-1}.$$

Split the sum:

$$s(n)
=
900\sum_{k=1}^n r^{k-1}
-
3\sum_{k=1}^n k r^{k-1}.$$

So we need closed forms for:

$$\sum_{k=1}^n r^{k-1}
\quad\text{and}\quad
\sum_{k=1}^n k r^{k-1}.$$

---

## 1. Geometric sum

For $r\neq1$,

$$\sum_{k=1}^n r^{k-1}
=
\frac{1-r^n}{1-r}.$$

---

## 2. Arithmetic–geometric sum

Start with

$$G(r)=\sum_{k=1}^n r^k
=
\frac{r-r^{n+1}}{1-r}.$$

Differentiate:

$$G'(r)=\sum_{k=1}^n k r^{k-1}.$$

Carrying out the differentiation gives

$$\sum_{k=1}^n k r^{k-1}
=
\frac{1-(n+1)r^n+n r^{n+1}}{(1-r)^2}.$$

Therefore,

$$s(n)
=
900\frac{1-r^n}{1-r}
-
3\frac{1-(n+1)r^n+n r^{n+1}}{(1-r)^2}.$$

Now substitute $n=5000$:

$$s(5000)=
900\frac{1-r^{5000}}{1-r}
-
3\frac{1-5001r^{5000}+5000r^{5001}}{(1-r)^2}.$$

We must solve

$$s(5000)=-600000000000.$$

---

# Numerical method

The function is monotone near the solution, and the desired root is very close to $1$.

A stable approach is binary search (or Newton's method).

Using high-precision arithmetic avoids catastrophic cancellation because $r\approx1$.

---

# Python implementation

```
from decimal import Decimal, getcontext

# Use high precision because r is very close to 1
getcontext().prec = 80

TARGET = Decimal(-600000000000)

def s(r, n=5000):
    """
    Compute s(n) = sum_{k=1}^n (900 - 3k) r^(k-1)
    directly using high-precision Decimal arithmetic.
    """
    total = Decimal(0)
    power = Decimal(1)

    for k in range(1, n + 1):
        total += (Decimal(900) - Decimal(3) * k) * power
        power *= r

    return total

# Binary search for the root
lo = Decimal('1.0')
hi = Decimal('1.1')

for _ in range(200):
    mid = (lo + hi) / 2

    if s(mid) > TARGET:
        lo = mid
    else:
        hi = mid

r = (lo + hi) / 2

print(round(r, 12))
```

---

# Code walkthrough

### Precision setup

```
getcontext().prec = 80
```

We use 80 digits of precision to avoid floating-point errors.

---

### Function `s(r)`

```
power = Decimal(1)
```

Initially $r^{0}=1$.

For each $k$,

```
total += (900 - 3*k) * power
power *= r
```

This efficiently builds the powers sequentially.

---

### Binary search

We know the root lies between $1$ and $1.1$.

At each step:

```
mid = (lo + hi) / 2
```

If

```
s(mid) > TARGET
```

then $r$ is too small, so move the lower bound upward.

Otherwise move the upper bound downward.

After 200 iterations, the answer is accurate far beyond $10^{-12}$.

---

# Final verification

The computed value is

$$r \approx 1.002322108632876\ldots$$

Checking numerically:

$$s(5000)\approx -600000000000,$$

matching the target to extremely high precision.

Rounded to 12 decimal places:

$$1.002322108633$$
