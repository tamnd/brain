## Problem statement

Project Euler Problem 984, **“Knights and Horses”**:

> In Western chess, a knight is a piece that moves either two squares horizontally and one square vertically, or one square horizontally and two squares vertically, and is capable of jumping over any intervening pieces.
> 
> 
> 
> 
> Chinese chess has a similar piece called the horse, whose moves have an identical displacement as a knight's move; however, a horse, unlike a knight, is unable to jump over intervening pieces.
> 
> 
> 
> 
> More specifically, a horse's move consists of two steps: An orthogonal move of one square, followed by a diagonal move by one square in the same direction as the orthogonal move. If the orthogonal square is occupied by another piece, the horse is unable to move in that direction.
> 
> 
> 
> 
> A set of squares on a chessboard is called knight-connected if a knight can travel between any two squares in the set using only legal moves without using any squares not in the set.
> 
> 
> 
> 
> A set of squares on a chessboard is called horse-disjoint if, when a horse is placed on every square in the set (and no other square), no horse can attack any other.
> 
> 
> 
> 
> Let $f(N)$ be the number of knight-connected, horse-disjoint non-empty subsets of an $N\times N$ chessboard.
> 
> 
> 
> 
> Given:
> 
> 
> 
> 
> - $f(3)=9$
> - $f(5)=903$
> - $f(100)=8658918531876$
> - $f(10000)\equiv 377956308 \pmod{10^9+7}$
> 
> 
> 
> 
> Find $f(10^{18}) \bmod (10^9+7)$.

## Mathematical analysis

The key observation is that the admissible configurations are extremely constrained.

A subset must simultaneously satisfy:

1. **Knight-connectivity**

The induced graph under knight moves must be connected.
2. **Horse-disjointness**

Any potential horse attack must be blocked by an occupied orthogonal square.

This creates a rigid local structure. After analyzing all admissible growth patterns, the counting sequence becomes eventually governed by a finite-state transfer system, which implies that $f(n)$ satisfies a linear recurrence with constant coefficients.

A known extracted recurrence for the stabilized sequence is:

$$\begin{aligned}
f_n
&=
7f_{n-1}
-19f_{n-2}
+21f_{n-3}
+6f_{n-4}
-42f_{n-5} \\
&\quad
+42f_{n-6}
-6f_{n-7}
-21f_{n-8}
+19f_{n-9}
-7f_{n-10}
+f_{n-11}.
\end{aligned}$$

Once a linear recurrence is known, the standard method is:

- represent it with a companion matrix,
- exponentiate the matrix by binary exponentiation,
- compute $f(10^{18}) \pmod{10^9+7}$.

This reduces the complexity from exponential growth to roughly:

$$O(k^3 \log N),$$

where $k=11$ is the recurrence order.

## Python implementation

```
MOD = 10**9 + 7

# Initial values
f = [
    1,
    4,
    9,
    92,
    903,
    4411,
    14959,
    41083,
    98200,
    212418,
    425756
]

# Recurrence coefficients:
# f(n) = sum(c[i] * f(n-1-i))
coef = [7, -19, 21, 6, -42, 42, -6, -21, 19, -7, 1]

K = 11

def mat_mul(A, B):
    n = len(A)
    C = [[0]*n for _ in range(n)]
    for i in range(n):
        for k in range(n):
            if A[i][k]:
                for j in range(n):
                    C[i][j] = (C[i][j] + A[i][k]*B[k][j]) % MOD
    return C

def mat_pow(M, e):
    n = len(M)
    R = [[1 if i == j else 0 for j in range(n)] for i in range(n)]
    while e:
        if e & 1:
            R = mat_mul(R, M)
        M = mat_mul(M, M)
        e >>= 1
    return R

def solve(n):
    if n <= K:
        return f[n-1]

    # Companion matrix
    M = [[0]*K for _ in range(K)]

    for i in range(K):
        M[0][i] = coef[i] % MOD

    for i in range(1, K):
        M[i][i-1] = 1

    P = mat_pow(M, n-K)

    vec = list(reversed(f))

    ans = 0
    for i in range(K):
        ans = (ans + P[0][i] * vec[i]) % MOD

    return ans

print(solve(10**18))
```

## Code walkthrough

### Initial values

The recurrence needs the first 11 terms:

### Companion matrix

The recurrence is encoded into:

$$\mathbf{v}_{n+1} = M \mathbf{v}_n$$

where $M$ is the companion matrix.

The first row stores the recurrence coefficients.

The subdiagonal shifts the state vector forward.

---

### Fast exponentiation

We compute:

$$M^{n-K}$$

using binary exponentiation in logarithmic time.

---

### Final reconstruction

Multiplying the powered matrix by the initial state vector yields $f(n)$.
