[Project Euler Problem 961](https://projecteuler.net/problem=961)

## Solution

**Answer:** `166666666689036288`

Let a position be represented only by whether each digit is zero or nonzero.

- Removing a nonzero digit changes a `1` to deletion.
- Removing a zero digit deletes only that zero.
- Leading zeros are stripped automatically.

Therefore the actual values `1..9` do not matter — only the zero/nonzero pattern matters.

Define a binary string pattern:

- `1` = nonzero digit
- `0` = zero digit

For example:

- `105  -> 101`
- `1001 -> 1001`

For each pattern, every realization contributes $9^{k}$ integers, where $k$ is the number of `1`s in the pattern.

We can recursively compute whether a pattern is winning:

- A pattern is winning iff it has a move to a losing pattern.
- The empty pattern is losing.

Since $10^{18}$ only involves lengths up to 18, there are only

$$2^{18}=262144$$

patterns, so brute force over patterns is tiny.

Python code:

```
from functools import cache

@cache
def winning(pattern):
    # empty pattern = no nonzero digits left
    if pattern == "":
        return False

    # try deleting each digit
    for i in range(len(pattern)):
        nxt = (pattern[:i] + pattern[i+1:]).lstrip('0')
        if not winning(nxt):
            return True

    return False

ans = 0

# lengths 1..18
for length in range(1, 19):

    # canonical patterns must start with 1
    for mask in range(1 << (length - 1), 1 << length):

        pattern = bin(mask)[2:]

        if winning(pattern):
            ans += 9 ** pattern.count('1')

print(ans)
```

This reproduces the given checks:

- $W(100)=18$
- $W(10^4)=1656$

and computes:

$$W(10^{18}) = 166666666689036288$$

**Answer: 166666666689036288**
