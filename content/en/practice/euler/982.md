## Solution

**Answer:** `4.381944`

Project Euler Problem 982, [The Third Dice problem statement](https://projecteuler.net/problem%3D982), states:

> Alice and Bob play the following game with two six-sided dice (numbered 1 to 6):
> 
> 
> 
> 
> 1. Alice rolls both dice; she can see the rolled values but Bob cannot
> 2. Alice chooses one of the dice and reveals it to Bob
> 3. Bob chooses one of the dice: either the one he can see, or the one he cannot
> 4. Alice pays Bob the value shown on Bob's chosen dice
> 
> 
> 
> 
> …
> 
> 
> 
> 
> With these strategies the expected payment from Alice to Bob is $145/36\approx4.027778$.
> 
> 
> 
> 
> To make the game more interesting, they introduce a third six-sided die:
> 
> 
> 
> 
> 1. Alice rolls three dice; she can see the rolled values but Bob cannot
> 2. Alice chooses two of the dice and reveals both to Bob
> 3. Bob chooses one of the three dice: either one of the two visible dice, or the one hidden die
> 4. Alice pays Bob the value shown on Bob's chosen die
> 
> 
> 
> 
> Supposing they settle on a pair of strategies that form a Nash equilibrium, find the expected payment from Alice to Bob, and give your answer rounded to six digits after the decimal point.

---

# Mathematical analysis

This is a finite zero-sum Bayesian signaling game.

- Alice knows all three dice.
- She chooses which die to hide.
- Bob observes the two revealed dice and then chooses:

- one of the visible dice, or
- the hidden die.

Because Bob can always pick the larger visible die, the only meaningful decision is:

> “Take the hidden die” vs “take the largest visible die”.

So for every revealed pair $(a,b)$ with $a\le b$, Bob’s payoff is

$$\max\!\left(b,\; \mathbb E[\text{hidden}\mid (a,b)]\right).$$

Alice’s strategy determines the conditional expectation of the hidden die.

---

# Linear programming formulation

There are:

$$6^3 = 216$$

equally likely states.

For each state $t=(d_1,d_2,d_3)$, Alice may hide die $1,2,$ or $3$.

Define variables:

$$x_{t,h}$$

= probability that in state $t$, Alice hides die $h$.

These satisfy:

$$\sum_h x_{t,h}=\frac1{216}$$

because every state occurs with probability $1/216$.

---

For every revealed signal $s=(a,b)$, introduce a variable

$$z_s$$

representing Bob’s equilibrium payoff after seeing $s$.

Bob can guarantee at least:

1. the visible maximum $b$,
2. or the expected hidden value.

Therefore:

$$z_s \ge b\cdot P(s),$$

and

$$z_s \ge \sum x_{t,h}\cdot (\text{hidden value}),$$

where the sums run over all $(t,h)$ producing signal $s$.

Alice minimizes:

$$\sum_s z_s.$$

This is a standard minimax LP.

# Python implementation

A compact implementation using linear programming is:

```
from itertools import product
from scipy.optimize import linprog

def solve(num_dice):
    values = range(1, 7)

    states = list(product(values, repeat=num_dice))
    p_state = 1 / len(states)

    # all possible revealed sorted tuples
    signals = sorted(set(
        tuple(sorted(t[i] for i in range(num_dice) if i != h))
        for t in states
        for h in range(num_dice)
    ))

    sig_index = {s:i for i,s in enumerate(signals)}

    num_x = len(states) * num_dice
    num_z = len(signals)
    nvars = num_x + num_z

    def x_index(s, h):
        return s * num_dice + h

    def z_index(i):
        return num_x + i

    Aeq, beq = [], []
    Aub, bub = [], []

    # probability constraints
    for s in range(len(states)):
        row = [0.0] * nvars
        for h in range(num_dice):
            row[x_index(s, h)] = 1.0
        Aeq.append(row)
        beq.append(p_state)

    # Bob payoff constraints
    for sig in signals:
        idx = sig_index[sig]
        visible_best = max(sig)

        row_vis = [0.0] * nvars
        row_hid = [0.0] * nvars

        row_vis[z_index(idx)] = -1.0
        row_hid[z_index(idx)] = -1.0

        for s, t in enumerate(states):
            for h in range(num_dice):
                revealed = tuple(sorted(
                    t[i] for i in range(num_dice) if i != h
                ))

                if revealed == sig:
                    row_vis[x_index(s, h)] += visible_best
                    row_hid[x_index(s, h)] += t[h]

        Aub.append(row_vis)
        bub.append(0.0)

        Aub.append(row_hid)
        bub.append(0.0)

    # minimize sum z_s
    c = [0.0] * nvars
    for i in range(num_z):
        c[z_index(i)] = 1.0

    res = linprog(
        c,
        A_ub=Aub,
        b_ub=bub,
        A_eq=Aeq,
        b_eq=beq,
        bounds=[(0, None)] * nvars,
        method="highs"
    )

    return res.fun

print(solve(2))  # 4.027777777777778
print(solve(3))  # 4.381944444444445
```

# Verification

The LP reproduces the known two-dice equilibrium value:

$$\frac{145}{36}=4.027777\ldots$$

matching the statement exactly.

For the three-dice game it yields:

$$4.381944444444445$$

which equals

$$\frac{631}{144}.$$

Rounded to six decimal places:

$$4.381944$$
