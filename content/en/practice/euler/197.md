[Project Euler Problem 197](https://projecteuler.net/problem=197)

## Solution

Let

$$f(x)=\left\lfloor 2^{30.403243784-x^2}\right\rfloor \cdot 10^{-9},$$

and define

$$u_0=-1,\qquad u_{n+1}=f(u_n).$$

We must compute

$$u_n+u_{n+1}\quad \text{for } n=10^{12},$$

to $9$ digits after the decimal point.

---

# 1. Mathematical analysis

The recurrence is deterministic but highly nonlinear because of the floor function.

A direct simulation up to $10^{12}$ iterations is impossible, so we must understand the long-term behavior of the sequence.

---

## Step 1: Observe the dynamics

Starting from $u_0=-1$,

$$u_1=f(-1)$$

is already positive, because the exponent $30.403243784-(-1)^2$ is large.

If we iterate numerically a few times, the sequence quickly settles into an alternating pattern:

$$u_{2k}\to a,\qquad u_{2k+1}\to b,$$

with

$$a\neq b.$$

That means the sequence approaches a stable 2-cycle.

Therefore,

$$u_n+u_{n+1}\to a+b.$$

Since $10^{12}$ is enormous, the transient behavior is completely gone, so the required answer is simply the limiting value of this alternating pair.

---

## Step 2: Why a 2-cycle appears

The map $f$ is approximately continuous except for tiny jumps caused by the floor.

Experimentally,

$$f(f(x)) \approx x$$

near the attractor, but $f(x)\neq x$. Hence the sequence alternates between two nearby values.

So after sufficiently many iterations:

$$u_{n+2}\approx u_n.$$

The convergence is extremely fast, meaning only a few hundred iterations are enough to reach machine precision.

---

## Step 3: Numerical stabilization

We iterate:

$$u_{k+1}=f(u_k)$$

until

$$|u_{k+2}-u_k|$$

is below machine precision.

At that point,

$$u_k+u_{k+1}$$

has stabilized to the required 9 decimal places.

The stabilized value is

$$1.710637717\ldots$$

---

# 2. Python implementation

```
import math

def f(x):
    return math.floor(2 ** (30.403243784 - x * x)) * 1e-9

u = -1.0

# Iterate enough times for convergence to the 2-cycle
for _ in range(1000):
    u = f(u)

answer = u + f(u)

print("{:.9f}".format(answer))
```

---

```

```

If $u$ is one element of the 2-cycle, then $f(u)$ is the other. Their sum is exactly what the problem asks for.

# 4. Final verification

The sequence values remain around $0.6$–$1.1$, so a sum near $1.7$ is reasonable.

The value stabilizes extremely rapidly; additional iterations do not change the first 9 decimal digits.

Thus the final value is:

$$1.710637717$$
