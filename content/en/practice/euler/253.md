[Project Euler Problem 253](https://projecteuler.net/problem=253)

## Solution

Let the caterpillar have $n$ pieces.

We place the pieces in a uniformly random order. After each placement, the occupied positions form several contiguous blocks (“segments”).

Define:

- $S_t$ = number of segments after $t$ pieces have been placed,
- $M = \max_t S_t$.

We want $\mathbb E[M]$ for $n=40$.

---

# 1. Key insight

Instead of tracking the occupied pieces, track the **empty intervals** that remain.

Every empty interval has one of three types:

- **Type 0**: the initial interval (no occupied neighbours),
- **Type 1**: interval touching one occupied segment,
- **Type 2**: interval between two occupied segments.

When we place a piece inside an interval, the interval splits into smaller intervals, and the segment count changes in a very structured way.

---

# 2. Segment-change rules

Suppose an interval has length $L$.

## Type 0 interval

Initially we have one interval of type 0 and length $n$.

Placing any piece creates a new segment:

$$\Delta S = +1$$

and splits into two type-1 intervals.

---

## Type 1 interval

This interval touches one existing segment.

- placing adjacent to the segment:

$$\Delta S = 0$$
- placing elsewhere:

$$\Delta S = +1$$

The interval splits into:

- one type-2 interval (between old and new segment),
- one type-1 interval (toward the edge).

---

## Type 2 interval

This interval lies between two existing segments.

- if $L=1$:

placing the only cell merges two segments,

$$\Delta S = -1$$
- placing at an endpoint:

$$\Delta S = 0$$
- placing strictly inside:

$$\Delta S = +1$$

Both resulting intervals are type 2.

---

# 3. Dynamic programming

We compute:

$$F(\text{state}, s, k)$$

= number of ways to complete the process starting from:

- current interval multiset = `state`,
- current segment count = $s$,
- while never exceeding $k$ segments.

Then:

$$\Pr(M \le k)
=
\frac{F(\text{initial},0,k)}{n!}$$

and finally

$$\mathbb E[M]
=
\sum_{m\ge1}\Pr(M\ge m)
=
\sum_{m\ge1}\left(1-\Pr(M\le m-1)\right).$$

---

# 4. Correct Python implementation

```
from functools import lru_cache
from math import factorial

# ------------------------------------------------------------
# Canonical representation of interval states
# ------------------------------------------------------------

def canon(intervals):
    """
    Sort intervals so equivalent states share the same cache key.

    Each interval is represented as:
        (type, length)

    type:
        0 = initial interval
        1 = one occupied neighbour
        2 = two occupied neighbours
    """
    return tuple(sorted((t, l) for t, l in intervals if l > 0))

# ------------------------------------------------------------
# DP:
# count ways to finish without ever exceeding k segments
# ------------------------------------------------------------

@lru_cache(None)
def count_leq(state, segments, k):

    remaining = sum(length for _, length in state)

    # all pieces placed
    if remaining == 0:
        return 1

    total = 0
    intervals = list(state)

    for idx, (typ, L) in enumerate(intervals):

        base = intervals[:idx] + intervals[idx + 1:]

        for pos in range(1, L + 1):

            new_segments = segments
            new_state = base.copy()

            # ------------------------------------------------
            # Type 0
            # ------------------------------------------------
            if typ == 0:

                new_segments += 1

                left = pos - 1
                right = L - pos

                if left:
                    new_state.append((1, left))
                if right:
                    new_state.append((1, right))

            # ------------------------------------------------
            # Type 1
            # ------------------------------------------------
            elif typ == 1:

                # only pos=1 touches existing segment
                if pos != 1:
                    new_segments += 1

                left = pos - 1
                right = L - pos

                if left:
                    new_state.append((2, left))
                if right:
                    new_state.append((1, right))

            # ------------------------------------------------
            # Type 2
            # ------------------------------------------------
            else:

                if L == 1:
                    new_segments -= 1
                elif pos != 1 and pos != L:
                    new_segments += 1

                left = pos - 1
                right = L - pos

                if left:
                    new_state.append((2, left))
                if right:
                    new_state.append((2, right))

            # prune if maximum exceeded
            if new_segments <= k:
                total += count_leq(
                    canon(new_state),
                    new_segments,
                    k
                )

    return total

# ------------------------------------------------------------
# Expected value of M
# ------------------------------------------------------------

def expected_max_segments(n):

    total_permutations = factorial(n)

    initial = canon([(0, n)])

    # P(M <= k)
    cumulative = []

    for k in range(n + 1):
        c = count_leq(initial, 0, k)
        cumulative.append(c / total_permutations)

    # E[M] = sum P(M >= m)
    ans = 0.0

    for m in range(1, n + 1):
        ans += 1.0 - cumulative[m - 1]

    return ans

# ------------------------------------------------------------
# Verification for n = 10
# ------------------------------------------------------------

print(round(expected_max_segments(10), 6))
# 3.400732

# ------------------------------------------------------------
# Final answer for n = 40
# ------------------------------------------------------------

print(round(expected_max_segments(40), 6))
```
