[Project Euler Problem 253](https://projecteuler.net/problem=253)

## Solution

Let the caterpillar have $n$ pieces.

The pieces are placed in a uniformly random order.

We track the number of connected occupied segments during the construction process, and define

$$M=\max(\text{number of segments seen during the process}).$$

We need the expected value of $M$ for $n=40$.

---

# Key observation: reverse the process

Instead of **adding** pieces, it is much easier to think about **removing** pieces.

Start with the fully assembled caterpillar (one segment of length $n$).

Remove pieces uniformly at random.

The distribution is identical to the forward process because a random insertion order reversed is a random deletion order.

So we begin with one segment of length $n$, and repeatedly delete one occupied square.

---

# Effect of deleting a piece

Suppose we delete a piece from a segment of length $L$.

There are three possibilities:

- delete an endpoint

$\Rightarrow$ segment count unchanged
- delete the unique piece of a segment of length $1$

$\Rightarrow$ segment count decreases by $1$
- delete an interior piece

$\Rightarrow$ the segment splits into two segments

$\Rightarrow$ segment count increases by $1$

Thus the entire state is determined only by the multiset of segment lengths.

Example:

$$(7,4,2)$$

means three occupied segments of lengths $7,4,2$.

---

# Dynamic programming formulation

Let a state $S$ be a sorted tuple of positive segment lengths.

Define

$$P(S,m)$$

to be the probability that, starting from state $S$, the process never reaches $m$ or more segments.

If the current number of segments is already at least $m$, then

$$P(S,m)=0.$$

If the state is empty:

$$P(\varnothing,m)=1.$$

Otherwise:

$$P(S,m)=\sum_{\text{transitions }S\to T}
\Pr(S\to T)\,P(T,m).$$

Finally,

$$\Pr(M \ge m)=1-P((n),m),$$

because the initial state is one segment of length $n$.

Using the standard identity

$$\mathbb E[M]=\sum_{m\ge1}\Pr(M\ge m),$$

we compute the expectation.

# Correct Python implementation

```
from functools import cache
from collections import defaultdict

def transitions(state):
    """
    Return all possible next states and their multiplicities.
    state is a sorted tuple of segment lengths.
    """
    total = sum(state)
    nxt = defaultdict(int)

    for i, L in enumerate(state):

        rest = list(state)
        rest.pop(i)

        # deleting the only piece
        if L == 1:
            ns = tuple(sorted(rest))
            nxt[ns] += 1

        else:
            # delete left or right endpoint
            ns = tuple(sorted(rest + [L - 1]))
            nxt[ns] += 2

            # delete an interior piece
            # split into a and b with a+b=L-1
            for a in range(1, L - 1):
                b = L - 1 - a
                ns = tuple(sorted(rest + [a, b]))
                nxt[ns] += 1

    return total, nxt

@cache
def all_transitions(state):
    return transitions(state)

@cache
def P(state, m):
    """
    Probability that the process never reaches
    m or more segments starting from 'state'.
    """

    # current number of segments
    if len(state) >= m:
        return 0.0

    # empty state
    if not state:
        return 1.0

    total, nxt = all_transitions(state)

    ans = 0.0

    for ns, count in nxt.items():
        ans += (count / total) * P(ns, m)

    return ans

def expected_M(n):
    E = 0.0

    for m in range(1, n + 1):
        E += 1 - P((n,), m)

    return E

print(expected_M(10))
print(expected_M(40))
```
