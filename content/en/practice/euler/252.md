[Project Euler Problem 252](https://projecteuler.net/problem=252)

## Solution

Let the generated points be $P_0,P_1,\dots,P_{499}$.

We seek the maximum-area **empty convex polygon** whose vertices are chosen from these points, where “empty” means that no given point lies strictly inside the polygon (points on the boundary are allowed).

---

# Mathematical analysis

## 1. Geometry of an empty convex polygon

Suppose a convex polygon is written in counterclockwise order as

$$(v_0,v_1,\dots,v_{m-1}).$$

Choose the unique lowest vertex $v_0$ (breaking ties by $x$-coordinate).

Then every other vertex lies strictly above $v_0$, and the polygon can be triangulated into the fan

$$(v_0,v_1,v_2),\;
(v_0,v_2,v_3),\;
\dots,\;
(v_0,v_{m-2},v_{m-1}).$$

The polygon area is therefore

$$\text{Area} =
\sum \text{Area}(v_0,v_i,v_{i+1}).$$

So if every triangle in this fan is empty and the chain remains convex, then the whole polygon is an empty convex polygon.

This converts the problem into a dynamic programming problem on triangles.

---

## 2. Testing whether a triangle is empty

For directed edge $AB$, define:

$$L(A,B)=\{P : P \text{ lies strictly to the left of } AB\}.$$

A point lies strictly inside triangle $ABC$ (with $ABC$ counterclockwise) iff it lies left of all three directed edges:

$$P \in L(A,B)\cap L(B,C)\cap L(C,A).$$

Hence triangle $ABC$ is empty iff

$$L(A,B)\cap L(B,C)\cap L(C,A)=\varnothing.$$

With only 500 points, we can store each $L(A,B)$ as a bitset, making triangle-emptiness checks extremely fast.

---

## 3. Dynamic programming

Fix the lowest vertex $s$.

Sort all points above $s$ by polar angle around $s$:

$$p_1,p_2,\dots,p_k.$$

Define:

$$dp[i][j]$$

to be the maximum doubled area of an empty convex chain ending with edge $p_i p_j$.

Transition:

- Start with empty triangle $(s,p_i,p_j)$.
- Extend from $p_h,p_i$ to $p_i,p_j$ whenever:

- the turn is left:

$$\operatorname{cross}(p_h,p_i,p_j)>0,$$
- triangle $(s,p_i,p_j)$ is empty.

Then

$$dp[i][j]
=
\max\left(
\text{area}(s,p_i,p_j),
\;
dp[h][i]+\text{area}(s,p_i,p_j)
\right).$$

Taking the maximum over all $s,i,j$ yields the optimal polygon.

The algorithm runs comfortably within limits.

# Python implementation

```
from math import atan2

# ------------------------------------------------------------
# Generate the 500 points
# ------------------------------------------------------------

N = 500

S = 290797
T = []

for _ in range(2 * N):
    S = (S * S) % 50515093
    T.append((S % 2000) - 1000)

pts = [(T[2 * i], T[2 * i + 1]) for i in range(N)]

# ------------------------------------------------------------
# Geometry helpers
# ------------------------------------------------------------

def cross(a, b, c):
    return (
        (b[0] - a[0]) * (c[1] - a[1])
        - (b[1] - a[1]) * (c[0] - a[0])
    )

# ------------------------------------------------------------
# Precompute left-of-edge bitsets
# left[i][j] = bitset of points strictly left of i->j
# ------------------------------------------------------------

left = [[0] * N for _ in range(N)]

for i in range(N):
    for j in range(N):
        if i == j:
            continue

        mask = 0

        for k in range(N):
            if cross(pts[i], pts[j], pts[k]) > 0:
                mask |= (1 << k)

        left[i][j] = mask

# ------------------------------------------------------------
# Empty triangle test
# ------------------------------------------------------------

def empty_triangle(a, b, c):
    if cross(pts[a], pts[b], pts[c]) <= 0:
        return False

    return (
        left[a][b]
        & left[b][c]
        & left[c][a]
    ) == 0

# ------------------------------------------------------------
# Dynamic programming
# ------------------------------------------------------------

best = 0  # doubled area

for s in range(N):

    # Only points strictly above s can appear
    # in a polygon where s is the lowest vertex.
    order = [
        i for i in range(N)
        if i != s and (
            pts[i][1] > pts[s][1]
            or (
                pts[i][1] == pts[s][1]
                and pts[i][0] > pts[s][0]
            )
        )
    ]

    order.sort(
        key=lambda i: atan2(
            pts[i][1] - pts[s][1],
            pts[i][0] - pts[s][0]
        )
    )

    m = len(order)

    dp = [[0] * m for _ in range(m)]

    for j in range(m):

        for k in range(j + 1, m):

            a = order[j]
            b = order[k]

            tri_area = cross(pts[s], pts[a], pts[b])

            if not empty_triangle(s, a, b):
                continue

            val = tri_area

            for i in range(j):

                c = order[i]

                # maintain convexity
                if (
                    dp[i][j]
                    and cross(pts[c], pts[a], pts[b]) > 0
                ):
                    val = max(
                        val,
                        dp[i][j] + tri_area
                    )

            dp[j][k] = val
            best = max(best, val)

# ------------------------------------------------------------
# Final answer
# ------------------------------------------------------------

if best % 2 == 0:
    print(f"{best // 2}.0")
else:
    print(f"{best // 2}.5")
```

# Code walkthrough

## Point generation

The problem defines:

$$S_{n+1}=S_n^2 \bmod 50515093,
\qquad
T_n=(S_n \bmod 2000)-1000.$$

We generate 1000 values $T_n$, then pair them into 500 planar points.

---

## Bitset preprocessing

For every directed edge $i\to j$, we compute a bitset containing all points strictly left of that edge.

This allows triangle emptiness to be checked using only three bitwise AND operations.

---

## Dynamic programming

For each lowest vertex $s$:

1. Sort all valid points by angle around $s$.
2. Build convex chains in angular order.
3. Add area triangle-by-triangle.

The DP stores **doubled areas** so everything remains integer arithmetic.
