## Problem statement

From [Project Euler Problem 974 — Very Odd Numbers](https://projecteuler.net/problem%3D974):

> ### Problem 974
> 
> 
> 
> 
> A very odd number is a number which contains only odd digits and is divisible by $105$. Further each odd digit occurs an odd number of times.
> 
> 
> 
> 
> Define $\Theta(n)$ be the $n$th very odd number, then $\Theta(1)=1117935$ and $\Theta(10^3)=11137955115$.
> 
> 
> 
> 
> Find $\Theta(10^{16})$.

---

# Mathematical analysis

We seek the $10^{16}$-th positive integer satisfying:

1. Every digit is odd:

$$\{1,3,5,7,9\}$$
2. The number is divisible by:

$$105 = 3 \cdot 5 \cdot 7$$
3. Each odd digit appears an odd number of times.

We must count such numbers efficiently and then construct the $10^{16}$-th one lexicographically.

---

# Step 1 — Divisibility by 5

Since only odd digits are allowed, divisibility by $5$ forces the final digit to be:

$$5$$

So the last digit is fixed.

---

# Step 2 — Odd occurrence condition

Let the counts of digits $1,3,5,7,9$ be:

$$c_1,c_3,c_5,c_7,c_9$$

Each must be odd.

A convenient representation is a **5-bit parity mask**:

- bit = 0 → even count
- bit = 1 → odd count

As we process digits, we toggle the corresponding bit.

At the end we require:

$$\text{mask} = 11111_2 = 31$$

because all five counts must be odd.

---

# Step 3 — Divisibility by 3 and 7

We build the number digit-by-digit.

If the current remainder modulo $m$ is $r$, after appending digit $d$:

$$r' = (10r+d)\bmod m$$

We only need:

- remainder mod 3
- remainder mod 7

At the end we require:

$$r_3 = 0,\qquad r_7 = 0$$

because divisibility by $105$ is equivalent to divisibility by $3,5,7$.

---

# Step 4 — Digit DP

Define:

$$dp(pos, r_3, r_7, mask)$$

= number of valid completions from position `pos`.

State space size:

- positions: about 30
- mod 3 states: 3
- mod 7 states: 7
- masks: $2^5=32$

Total:

$$30 \cdot 3 \cdot 7 \cdot 32 \approx 20{,}000$$

which is tiny.

---

# Step 5 — Counting by length

A key observation:

- the final digit must be $5$,
- every digit count is odd.

Since the sum of five odd integers is odd, every valid number has odd length.

For each odd length $L$, compute:

$$C(L)=\text{number of valid very odd numbers of length }L$$

Then accumulate until:

$$\sum_{k<L} C(k) < 10^{16} \le \sum_{k\le L} C(k)$$

This determines the length of $\Theta(10^{16})$.

---

# Step 6 — Unranking

Once the length is known, construct the number digit-by-digit.

At each position:

1. try candidate digits in increasing order,
2. count how many valid completions exist,
3. subtract counts until the target block is found.

This is standard lexicographic unranking.

# Python implementation

```
from functools import lru_cache

DIGITS = [1, 3, 5, 7, 9]

# map each digit to a parity-bit position
BIT = {
    1: 0,
    3: 1,
    5: 2,
    7: 3,
    9: 4,
}

ALL = (1 << 5) - 1   # 11111b

TARGET = 10**16

def count_length(L):
    """
    Count valid very odd numbers of exact length L.
    """

    @lru_cache(None)
    def dp(pos, mod3, mod7, mask):

        # finished building the number
        if pos == L:
            return int(mod3 == 0 and mod7 == 0 and mask == ALL)

        total = 0

        # last digit must be 5
        choices = [5] if pos == L - 1 else DIGITS

        for d in choices:

            new_mod3 = (mod3 * 10 + d) % 3
            new_mod7 = (mod7 * 10 + d) % 7

            # toggle parity bit
            new_mask = mask ^ (1 << BIT[d])

            total += dp(pos + 1, new_mod3, new_mod7, new_mask)

        return total

    return dp(0, 0, 0, 0)

def unrank(L, k):
    """
    Return the k-th valid number of length L (0-indexed).
    """

    @lru_cache(None)
    def dp(pos, mod3, mod7, mask):

        if pos == L:
            return int(mod3 == 0 and mod7 == 0 and mask == ALL)

        total = 0

        choices = [5] if pos == L - 1 else DIGITS

        for d in choices:

            new_mod3 = (mod3 * 10 + d) % 3
            new_mod7 = (mod7 * 10 + d) % 7
            new_mask = mask ^ (1 << BIT[d])

            total += dp(pos + 1, new_mod3, new_mod7, new_mask)

        return total

    ans = []

    pos = 0
    mod3 = 0
    mod7 = 0
    mask = 0

    while pos < L:

        choices = [5] if pos == L - 1 else DIGITS

        for d in choices:

            new_mod3 = (mod3 * 10 + d) % 3
            new_mod7 = (mod7 * 10 + d) % 7
            new_mask = mask ^ (1 << BIT[d])

            cnt = dp(pos + 1, new_mod3, new_mod7, new_mask)

            if k >= cnt:
                k -= cnt
            else:
                ans.append(str(d))
                mod3 = new_mod3
                mod7 = new_mod7
                mask = new_mask
                break

        pos += 1

    return "".join(ans)

def solve():

    remaining = TARGET
    L = 1

    while True:

        if L % 2 == 1:
            cnt = count_length(L)

            if remaining <= cnt:
                return unrank(L, remaining - 1)

            remaining -= cnt

        L += 1

print(solve())
```

Parity toggles:

```

```

because adding one occurrence flips odd/even parity.
