[Project Euler Problem 539](https://projecteuler.net/problem=539)

## Solution

Let $P(n)$ be the last remaining number in the alternating elimination process.

We seek

$$S(n)=\sum_{k=1}^n P(k),$$

and specifically

$$S(10^{18}) \bmod 987654321.$$

---

# 1. Mathematical analysis

## Step 1: Deriving the recurrence for $P(n)$

Start with the list

$$1,2,3,\dots,n.$$

The first pass goes left-to-right, deleting the first element and every other afterward.

The survivors are exactly the even numbers:

$$2,4,6,\dots,2\left\lfloor \frac n2 \right\rfloor.$$

Let

$$m=\left\lfloor \frac n2 \right\rfloor.$$

After dividing all surviving numbers by $2$, we obtain the list

$$1,2,\dots,m.$$

However, the next elimination direction is reversed (right-to-left).

Suppose the final survivor in the normalized list is $P(m)$.

Because the direction is reversed, the corresponding survivor index among

$$1,2,\dots,m$$

is

$$m+1-P(m).$$

Undoing the scaling by $2$,

$$P(n)=2\bigl(m+1-P(m)\bigr),
\qquad m=\left\lfloor \frac n2 \right\rfloor.$$

Thus:

$$\boxed{
P(1)=1,
\qquad
P(n)=2\left(\left\lfloor\frac n2\right\rfloor+1-P\!\left(\left\lfloor\frac n2\right\rfloor\right)\right)
}$$

Since both $2m$ and $2m+1$ have the same $m$,

$$\boxed{
P(2m)=P(2m+1)=2(m+1-P(m)).
}$$

---

## Step 2: Recurrence for $S(n)$

Define

$$a_m=P(2m)=P(2m+1)=2(m+1-P(m)).$$

Then

$$S(2m)
=
1+\sum_{k=1}^{m-1}2a_k+a_m.$$

Substitute

$$a_k=2k+2-2P(k).$$

We get

$$\begin{aligned}
S(2m)
&=
1
+
2\sum_{k=1}^{m-1}(2k+2-2P(k))
+
(2m+2-2P(m)).
\end{aligned}$$

Now simplify:

### Sum of $2k$

$$2\sum_{k=1}^{m-1}2k
=
2m(m-1).$$

### Sum of constants

$$2\sum_{k=1}^{m-1}2
=
4m-4.$$

### Sum involving $P(k)$

$$2\sum_{k=1}^{m-1}(-2P(k))
=
-4S(m-1).$$

Therefore

$$\begin{aligned}
S(2m)
&=
1
+
2m(m-1)
+
4m-4
-
4S(m-1)
+
2m+2-2P(m).
\end{aligned}$$

Using

$$S(m)=S(m-1)+P(m),$$

we obtain

$$\boxed{
S(2m)=2m^2+4m-1-4S(m)+2P(m).
}$$

For odd values:

$$S(2m+1)=S(2m)+P(2m+1).$$

Since

$$P(2m+1)=2m+2-2P(m),$$

we get

$$\boxed{
S(2m+1)=2m^2+6m+1-4S(m).
}$$

These recurrences reduce the problem size from $n$ to $\lfloor n/2\rfloor$, so the complexity is only $O(\log n)$.

# 2. Python implementation

```
from functools import lru_cache

MOD = 987654321

@lru_cache(None)
def solve(n):
    """
    Returns (P(n), S(n)) modulo MOD.
    """

    if n == 1:
        return (1, 1)

    m = n // 2

    Pm, Sm = solve(m)

    # P(n) recurrence
    Pn = 2 * (m + 1 - Pm)

    # S(n) recurrence
    if n % 2 == 0:
        # n = 2m
        Sn = 2*m*m + 4*m - 1 - 4*Sm + 2*Pm
    else:
        # n = 2m + 1
        Sn = 2*m*m + 6*m + 1 - 4*Sm

    return (Pn % MOD, Sn % MOD)

answer = solve(10**18)[1] % MOD
print(answer)
```

# 3. Code walkthrough

## Memoization

```
@lru_cache(None)
```

Stores previously computed values so each $n$ is solved once.

---

## Base case

```
if n == 1:
    return (1, 1)
```

Because

$$P(1)=1,\qquad S(1)=1.$$

---

## Recursive reduction

```
m = n // 2
Pm, Sm = solve(m)
```

All formulas depend only on $\lfloor n/2\rfloor$.

---

## Computing $P(n)$

```
Pn = 2 * (m + 1 - Pm)
```

Implements

$$P(n)=2(m+1-P(m)).$$

---

## Computing $S(n)$

Even case:

```
Sn = 2*m*m + 4*m - 1 - 4*Sm + 2*Pm
```

implements

$$S(2m)=2m^2+4m-1-4S(m)+2P(m).$$

Odd case:

```
Sn = 2*m*m + 6*m + 1 - 4*Sm
```

implements

$$S(2m+1)=2m^2+6m+1-4S(m).$$
