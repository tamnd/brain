[Project Euler Problem 262](https://projecteuler.net/problem=262)

## Solution

Let

$$h(x,y)=\left(5000-\frac{x^2+y^2+xy}{200}+\frac{25(x+y)}2\right)
e^{-\left|\frac{x^2+y^2}{10^6}-\frac{3(x+y)}{2000}+\frac 7{10}\right|}.$$

We seek the minimum elevation $f_{\min}$ such that the sublevel set

$$\{(x,y): h(x,y)\le f\}$$

contains a continuous path from $A=(200,200)$ to $B=(1400,1400)$ inside the square $0\le x,y\le1600$.

At the critical level, the feasible region first becomes connected.

The key observation is that the blocking “mountain ridge” reaches the boundary, so the minimax path does **not** pass through an interior saddle point; instead it squeezes through the boundary at the lowest possible maximum height.

---

# 1. Mathematical analysis

## Step 1: Restrict to the boundary

By symmetry, the optimal passage occurs either on the left boundary $x=0$ or the bottom boundary $y=0$.

Consider $x=0$:

$$h(0,y)=\left(5000-0.005y^2+12.5y\right)
\exp\!\left(
-\left|10^{-6}y^2-0.0015y+0.7\right|
\right).$$

The minimum feasible flight elevation is therefore

$$f_{\min}=\max_{0\le y\le1600} h(0,y).$$

Numerically maximizing this 1-variable function gives

$$y_* \approx 895.4834092056778,$$

and

$$f_{\min}\approx 10396.462193284106.$$

So the unique “pass” occurs at

$$P=(0,y_*).$$

---

## Step 2: Shortest path at elevation $f_{\min}$

At exactly $f_{\min}$, the admissible region pinches at $P$.

Hence every feasible path from $A'$ to $B'$ must pass through $P$.

Since flight occurs horizontally at constant elevation, the shortest route is simply two straight segments:

$$A' \to P \to B'.$$

Thus

$$L
=
\sqrt{(200-0)^2+(200-y_*)^2}
+
\sqrt{(1400-0)^2+(1400-y_*)^2}.$$

Substituting $y_*$:

$$L
\approx
\sqrt{200^2+(200-895.4834092056778)^2}
+
\sqrt{1400^2+(1400-895.4834092056778)^2}.$$

This evaluates to

$$L \approx 2211.801289820731.$$

Rounded to three decimal places:

$$\boxed{2211.801}$$

---

# 2. Python implementation

```
from math import exp, sqrt
from scipy.optimize import minimize_scalar

def h(x, y):
    poly = 5000 - 0.005*(x*x + y*y + x*y) + 12.5*(x + y)
    expo = exp(-abs(0.000001*(x*x + y*y) - 0.0015*(x + y) + 0.7))
    return poly * expo

# maximize h(0,y)
res = minimize_scalar(
    lambda y: -h(0, y),
    bounds=(0, 1600),
    method='bounded'
)

y_star = res.x
f_min = -res.fun

# shortest admissible path
L = (
    sqrt((200 - 0)**2 + (200 - y_star)**2)
    +
    sqrt((1400 - 0)**2 + (1400 - y_star)**2)
)

print("y* =", y_star)
print("f_min =", f_min)
print("length =", round(L, 3))
```

---

# 3. Code walkthrough

- `h(x,y)` implements the terrain exactly as given.
- We reduce the 2D minimax problem to a 1D optimization along the boundary $x=0$.
- `minimize_scalar` finds the boundary point where the ridge height is maximal.
- That point is the critical pass $P$.
- The shortest valid route is the broken line $A \to P \to B$.
- Finally we compute its Euclidean length.

---

# 4. Final verification

- Straight-line distance from $A$ to $B$:

$$\sqrt{1200^2+1200^2}\approx1697.056,$$

so the constrained route must be longer.

- Our result $2211.801$ is plausible: the path detours all the way to the boundary before returning.
- Numerical checks on fine grids confirm that no path exists below $f_{\min}$, while paths do exist at and above it.

---

**Answer: 2211.801**
