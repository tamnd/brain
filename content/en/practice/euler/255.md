[Project Euler Problem 255](https://projecteuler.net/problem=255)

## Solution

Let

$$F_x(n)=\left\lfloor \frac{x+\lceil n/x\rceil}{2}\right\rfloor .$$

Starting from the prescribed initial value $x_0$, the algorithm repeatedly replaces

$$x_{k+1}=F_{x_k}(n)$$

until $x_{k+1}=x_k$.

We must compute the average number of iterations for all $14$-digit integers:

$$10^{13}\le n<10^{14}.$$

For $14$-digit numbers, the starting value is

$$x_0=7\cdot 10^{6}=7{,}000{,}000.$$

---

# 1. Mathematical analysis

## Step 1: Characterize when a value is fixed

Suppose $x$ is already stable:

$$x=F_x(n).$$

Then

$$x=\left\lfloor \frac{x+\lceil n/x\rceil}{2}\right\rfloor .$$

This means

$$x \le \frac{x+\lceil n/x\rceil}{2}<x+1.$$

Multiply by $2$:

$$2x \le x+\lceil n/x\rceil <2x+2.$$

Hence

$$x\le \lceil n/x\rceil <x+2.$$

Since the middle quantity is an integer:

$$\lceil n/x\rceil \in \{x,x+1\}.$$

Using the definition of ceiling:

- $\lceil n/x\rceil=x$ iff $(x-1)x<n\le x^2$,
- $\lceil n/x\rceil=x+1$ iff $x^2<n\le x(x+1)$.

Combining:

$$x^2-x<n\le x^2+x.$$

So the fixed point is exactly the rounded square root.

---

## Step 2: Determine all $n$ mapping from $x$ to $y$

Suppose one iteration sends $x\mapsto y$:

$$y=\left\lfloor \frac{x+\lceil n/x\rceil}{2}\right\rfloor .$$

Then

$$2y\le x+\lceil n/x\rceil <2y+2.$$

Therefore

$$\lceil n/x\rceil \in \{2y-x,\;2y-x+1\}.$$

Let

$$t=2y-x.$$

Then:

- if $\lceil n/x\rceil=t$, we have $(t-1)x<n\le tx$,
- if $\lceil n/x\rceil=t+1$, we have $tx<n\le (t+1)x$.

Combining:

$$(t-1)x<n\le (t+1)x.$$

Substitute $t=2y-x$:

$$(2y-x-1)x<n\le (2y-x+1)x.$$

Thus every transition $x\to y$ corresponds to one contiguous interval of $n$.

This is the key observation:

the whole computation can be done by recursively subdividing intervals, not by iterating over all $9\times 10^{13}$ integers.

---

# 2. Efficient algorithm

We recursively maintain states:

$$(x,[L,U])$$

meaning:

> for every $n\in[L,U]$, the current iterate equals $x$.

For each possible next value $y$, we intersect the interval

$$(2y-x-1)x<n\le(2y-x+1)x$$

with $[L,U]$.

Every surviving subinterval contributes one iteration for each integer it contains.

If $y=x$, those numbers terminate.

Otherwise recurse.

The total number of generated states is only a few tens of millions, which is feasible in optimized code.

---

# 3. Python implementation

```
from functools import cache

A = 10**13
B = 10**14 - 1

# Initial value for 14-digit numbers
X0 = 7 * 10**6

@cache
def total_iterations(x, a, b):
    """
    Returns the total number of iterations required
    for all n in [a, b], assuming the current iterate is x.
    """
    total = 0

    # Smallest and largest possible next values
    ymin = (x + ((a + x - 1) // x)) // 2
    ymax = (x + ((b + x - 1) // x)) // 2

    for y in range(ymin, ymax + 1):

        t = 2 * y - x

        # Interval producing x -> y
        L = max(a, (t - 1) * x + 1)
        U = min(b, (t + 1) * x)

        if L <= U:
            count = U - L + 1

            # One iteration performed for every n here
            total += count

            # Continue only if not fixed yet
            if y != x:
                total += total_iterations(y, L, U)

    return total

total = total_iterations(X0, A, B)

average = total / (B - A + 1)

print(f"{average:.10f}")
```
