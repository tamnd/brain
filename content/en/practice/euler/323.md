[Project Euler Problem 323](https://projecteuler.net/problem=323)

## Solution

Let $N$ be the first index such that $x_N = 2^{32}-1$, i.e. all 32 bits have become $1$.

### Step 1: Model one bit

Focus on a single bit position.

Each random 32-bit integer $y_i$ has that bit equal to:

- $1$ with probability $1/2$,
- $0$ with probability $1/2$.

Since $x_i$ is formed by repeated bitwise ORs, once a bit becomes $1$, it stays $1$ forever.

After $n$ random numbers have been OR'ed (i.e. in $x_n$):

- the probability a given bit is **still 0** is

$$\left(\frac12\right)^n = 2^{-n},$$

because it must have been 0 in every one of the $n$ samples.

Thus the probability a given bit is **1** by time $n$ is

$$1 - 2^{-n}.$$

### Step 2: All 32 bits

The 32 bit positions are independent.

Therefore,

$$P(x_n = 2^{32}-1)
=
(1-2^{-n})^{32}.$$

This is exactly

$$P(N \le n).$$

So

$$P(N > n)
=
1 - (1-2^{-n})^{32}.$$

### Step 3: Expected value identity

For a nonnegative integer-valued random variable,

$$E[N]
=
\sum_{n=0}^{\infty} P(N>n).$$

Hence

$$E[N]
=
\sum_{n=0}^{\infty}
\left[
1-(1-2^{-n})^{32}
\right].$$

This converges very quickly because the tail decreases exponentially.

### Python implementation

```
from decimal import Decimal, getcontext

# high precision to avoid rounding issues
getcontext().prec = 50

expected = Decimal(0)

for n in range(200):
    p_not_finished = Decimal(1) - (
        Decimal(1) - (Decimal(2) ** Decimal(-n))
    ) ** 32

    expected += p_not_finished

    # stop once terms are negligible
    if p_not_finished < Decimal("1e-40"):
        break

print(expected)
print(round(float(expected), 10))
```

### Code walkthrough

- `Decimal` is used for high precision arithmetic.
- For each $n$:

- compute $2^{-n}$,
- compute $(1-2^{-n})^{32}$, the probability all bits are already set,
- subtract from 1 to get $P(N>n)$,
- add to the running total.
- We stop once the contribution becomes astronomically tiny.

### Small sanity check

For a 1-bit version:

$$P(N\le n)=1-2^{-n},$$

so

$$E[N]
=
\sum_{n\ge0}2^{-n}
=2.$$

That matches the expected waiting time for the first success of a fair coin, confirming the model is consistent.

For 32 bits, the expectation should be only somewhat larger than $5$, since after $n=5$,

$$P(\text{a bit missing}) \approx 2^{-5}=\frac1{32},$$

so roughly one bit is still missing on average. A value near $6$ is reasonable.

Numerically,

$$E[N]
=
6.355175845103938\ldots$$

Rounded to 10 digits after the decimal point:

**Answer: 6.3551758451**
