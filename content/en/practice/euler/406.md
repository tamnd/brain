[Project Euler Problem 406](https://projecteuler.net/problem=406)

## Solution

Let

$$C(n,a,b)$$

be the minimum worst-case cost to identify a hidden number in $\{1,\dots,n\}$, where:

- guessing too low costs $a$,
- guessing too high costs $b$,
- correct guess costs $0$.

The key observation is that this is an **optimal decision tree** problem with asymmetric branch costs.

Define $M(t)$ = maximum number of integers that can be distinguished with worst-case budget $t$.

If we ask a question first, then:

- the “lower” branch has remaining budget $t-a$,
- the “higher” branch has remaining budget $t-b$,
- and the current pivot contributes one number.

Hence:

$$M(t)=1+M(t-a)+M(t-b)$$

with boundary condition

$$M(t)=0 \quad (t<0).$$

Then:

$$C(n,a,b)=\min\{t:M(t)\ge n\}.$$

This recurrence reproduces the given examples exactly:

- $C(5,2,3)=5$
- $C(500,\sqrt2,\sqrt3)=13.22073197\ldots$
- $C(20000,5,7)=82$
- $C(2000000,\sqrt5,\sqrt7)=49.63755955\ldots$

For the target problem, compute

$$\sum_{k=1}^{30}
C\!\left(10^{12},\sqrt{k},\sqrt{F_k}\right)$$

where $F_k$ are Fibonacci numbers.

A memoized implementation of the recurrence combined with binary search for the minimal $t$ gives the total:

$$36813.127572068\ldots$$

Rounded to 8 decimal places:

**Answer: 36813.12757207**
