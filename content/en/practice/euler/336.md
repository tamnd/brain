[Project Euler Problem 336](https://projecteuler.net/problem=336)

## Solution

Let the carriages be represented by a permutation of $A,B,C,\dots$.

A single turntable operation chooses a cut position and reverses the suffix after that cut.

If the current arrangement is

$$x_0x_1\cdots x_{k-1}\;x_kx_{k+1}\cdots x_{n-1},$$

then one operation transforms it into

$$x_0x_1\cdots x_{k-1}\;x_{n-1}\cdots x_{k+1}x_k.$$

Equivalently, it is a **suffix reversal**.

---

# 1. Mathematical analysis

## Simon’s deterministic strategy

Simon fixes the carriages from left to right.

Suppose we are currently trying to place carriage $A_i$ into position $i$.

There are two possibilities:

### Case 1 — $A_i$ is already at the end

Then one suffix reversal puts it into the correct place.

### Case 2 — $A_i$ is somewhere in the middle

Simon needs two reversals:

1. Reverse the suffix beginning at $A_i$, bringing $A_i$ to the end.
2. Reverse the suffix beginning at position $i$, bringing $A_i$ into its final position.

So:

- middle position  $\Rightarrow$ 2 moves,
- end position     $\Rightarrow$ 1 move.

---

## Maximum possible number of moves

For $n$ carriages:

- The final carriage never needs work.
- The penultimate carriage can require at most 1 move.
- Every earlier carriage can require at most 2 moves.

Hence the absolute maximum is

$$2(n-2)+1 = 2n-3.$$

These are the **maximix arrangements**.

For $n=4$:

$$2\cdot 4 - 3 = 5,$$

matching the statement.

---

# 2. Constructing all maximix arrangements

Instead of simulating Simon forward, it is much cleaner to work **backwards** from the sorted train.

Start from

$$ABC\cdots$$

and undo Simon’s operations.

---

## Reverse construction

Suppose we are undoing the stage that fixed position $i$.

Forward Simon would do:

1. reverse at some $j>i$,
2. reverse at $i$.

Undoing this means:

1. reverse at $i$,
2. reverse at $j$.

For a maximix arrangement:

- for positions $0,1,\dots,n-3$,

the target carriage must **not** have been at the end,

so $j\neq n-1$.

Thus:

$$j \in \{i+1,i+2,\dots,n-2\}.$$

For the penultimate carriage ($i=n-2$), only the single reversal at $i$ occurs.

Therefore the number of maximix arrangements is

$$(n-2)!,$$

since the choices are

$$(n-2)(n-3)\cdots 1.$$

For $n=6$:

$$4! = 24,$$

exactly as stated.

# 3. Python implementation

```
from typing import List

def maximix_arrangements(n: int) -> List[str]:
    """
    Generate all maximix arrangements for n carriages.
    """
    start = list(chr(ord('A') + i) for i in range(n))
    result = []

    def dfs(state: List[str], i: int) -> None:
        """
        Undo Simon's process from right to left.
        """

        # Finished all stages
        if i < 0:
            result.append("".join(state))
            return

        # First undo the reversal at position i
        s1 = state[:]
        s1[i:] = reversed(s1[i:])

        # Last stage only needed one move
        if i == n - 2:
            dfs(s1, i - 1)
            return

        # Earlier stages must use TWO moves,
        # therefore the target carriage must NOT
        # have been at the end.
        for j in range(i + 1, n - 1):
            s2 = s1[:]
            s2[j:] = reversed(s2[j:])
            dfs(s2, i - 1)

    dfs(start, n - 2)

    return sorted(result)

# Problem asks for the 2011th lexicographic arrangement
arrangements = maximix_arrangements(11)

answer = arrangements[2010]   # 0-indexed

print(answer)
```

# 4. Code walkthrough

## Initial state

```
start = ['A', 'B', 'C', ...]
```

We begin from the fully sorted train.

---

## Recursive function

```
dfs(state, i)
```

means:

> “Undo Simon’s fixing of positions $i,i+1,\dots$.”

---

## Undo the mandatory reversal at $i$

```
s1[i:] = reversed(s1[i:])
```

This reverses the suffix starting at $i$.

---

## Last stage

```
if i == n - 2:
```

The penultimate carriage only required one move in the forward process, so there is no second reversal.

---

## Earlier stages

```
for j in range(i + 1, n - 1):
```

We choose the position of the first reversal.

Crucially:

```
j < n - 1
```

ensures the target carriage was **not already at the end**, forcing Simon to use two moves and therefore achieving the maximal total.

---

## Lexicographic order

Finally:

```
sorted(result)
```

puts all maximix arrangements into lexicographic order.
