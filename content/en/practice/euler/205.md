[Project Euler Problem 205](https://projecteuler.net/problem=205)

## Solution

We compute the probability exactly by counting all outcomes, not by simulation.

## 1. Mathematical analysis

Peter rolls **nine 4-sided dice** (values 1–4).

Colin rolls **six 6-sided dice** (values 1–6).

We want:

$$P(\text{Peter total} > \text{Colin total})$$

### Step 1: Count total outcomes

Peter has:

$$4^9 = 262144$$

possible outcomes.

Colin has:

$$6^6 = 46656$$

possible outcomes.

Total equally likely paired outcomes:

$$4^9 \cdot 6^6$$

---

### Step 2: Distribution of sums

Let:

- $P(s)$ = number of ways Peter can total $s$
- $C(t)$ = number of ways Colin can total $t$

We can build these distributions using dynamic programming (convolution of dice).

For one die:

Peter:

$$[1,1,1,1]$$

Colin:

$$[1,1,1,1,1,1]$$

Each additional die convolves with the previous distribution.

For example, with two 4-sided dice:

$$2:1,\,
3:2,\,
4:3,\,
5:4,\,
6:3,\,
7:2,\,
8:1$$

We continue until:

- Peter: 9 dice
- Colin: 6 dice

---

### Step 3: Count Peter wins

For every Peter sum $s$,

Peter wins whenever Colin’s total is **strictly smaller**.

So:

$$\text{wins}
=
\sum_s
P(s)
\left(
\sum_{t<s} C(t)
\right)$$

Then:

$$\Pr(\text{Peter wins})
=
\frac{\text{wins}}
{4^9\cdot 6^6}$$

---

## 2. Python implementation

```
from collections import Counter

# Build Peter's distribution (9d4)
peter = Counter({0: 1})

for _ in range(9):
    new_dist = Counter()

    for total, count in peter.items():
        for die in range(1, 5):
            new_dist[total + die] += count

    peter = new_dist

# Build Colin's distribution (6d6)
colin = Counter({0: 1})

for _ in range(6):
    new_dist = Counter()

    for total, count in colin.items():
        for die in range(1, 7):
            new_dist[total + die] += count

    colin = new_dist

# Count Peter wins
wins = 0

for p_sum, p_count in peter.items():
    colin_less = sum(
        count for c_sum, count in colin.items()
        if c_sum < p_sum
    )

    wins += p_count * colin_less

# Total outcomes
total_outcomes = (4 ** 9) * (6 ** 6)

probability = wins / total_outcomes

print(round(probability, 7))
```

---

## 3. Code walkthrough

### Building distributions

We begin with:

```
Counter({0: 1})
```

meaning there is exactly one way to obtain sum 0 before rolling any dice.

For each die:

```
for die in range(1, 5):
```

(or 1–6 for Colin), we add the die result to every existing sum.

This is exactly the discrete convolution of probability distributions.

---

### Counting wins

For every Peter total:

```
if c_sum < p_sum
```

we count Colin outcomes that lose.

Multiply:

```
p_count * colin_less
```

because every Peter configuration combines with every losing Colin configuration.

---

### Small sanity check

With one die each:

- Peter: d4
- Colin: d6

Peter wins in:

$$(2+1+0+0)=6$$

out of:

$$4\cdot6=24$$

so probability:

$$0.25$$

The algorithm reproduces such toy cases correctly.

---

## 4. Final verification

Computed probability:

$$0.5731440767829801$$

This makes sense:

- Peter has **more dice** (9 vs 6), giving lower variance and a mean of:

$$9 \cdot 2.5 = 22.5$$

- Colin’s mean:

$$6 \cdot 3.5 = 21$$

Peter should be favored, but not overwhelmingly — about **57%** is reasonable.

Rounded to seven decimal places:

**Answer: 0.5731441**
