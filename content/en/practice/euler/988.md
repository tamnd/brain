Here is the exact statement of Project Euler Problem 988:

> **Non-attacking Frogs**
> 
> 
> 
> 
> Frogs can be placed on the real number line at integer locations. Given coprime positive integers $(a,b)$, each frog has the ability to make jumps of distances $a$ or $b$ in the positive direction.
> 
> 
> 
> 
> Two frogs placed at $m$ and $n$, $m<n$, are attacking if the frog at $m$ can hop to $n$ with some series of jumps.
> 
> 
> 
> 
> A non-attacking configuration is a placement of any number of frogs such that:
> 
> 
> 
> 
> - one frog is placed at $0$;
> - all other frogs are placed at distinct positive integers;
> - no two frogs are attacking.
> 
> 
> 
> 
> Define $F(a,b)$ to be sum of the integer locations of every frog, summing over all non-attacking configurations.
> 
> 
> 
> 
> It is given that $F(3,5)=23$ and $F(5,13)=16336$.
> 
> 
> 
> 
> Find $F(19,53)$.

# Mathematical analysis

A frog at position $x$ can reach $y>x$ iff

$$y-x = ua + vb$$

for some nonnegative integers $u,v$.

So the forbidden differences are exactly the elements of the numerical semigroup

$$S=\{ua+vb : u,v\ge 0\}.$$

Therefore, if two frogs are both present, their difference must **not** belong to $S$.

---

## 1. The gaps of the semigroup

For coprime $a,b$, every sufficiently large integer belongs to $S$. The positive integers not in $S$ are called the **gaps**.

A classical fact is:

$$n \notin S
\iff
n = ab-ai-bj$$

for unique integers

$$1\le i\le b-1,\qquad 1\le j\le a-1,$$

with $ai+bj<ab$.

Thus every allowed frog position corresponds to a lattice point $(i,j)$.

The weight of that point is

$$w(i,j)=ab-ai-bj.$$

---

## 2. When are two gaps compatible?

Take two gaps

$$x=ab-ai_1-bj_1,
\qquad
y=ab-ai_2-bj_2.$$

Then

$$x-y=a(i_2-i_1)+b(j_2-j_1).$$

This belongs to $S$ exactly when both coefficients have the same sign.

Hence two gaps are compatible iff

$$(i_1-i_2)(j_1-j_2)<0,$$

i.e. one coordinate increases while the other decreases.

So a non-attacking configuration corresponds exactly to a set of lattice points with:

- increasing $i$,
- decreasing $j$.

Equivalently: at most one point in each row, and chosen column indices strictly increasing as rows decrease.

---

# Dynamic programming

For each row $j$, the allowed columns are

$$1 \le i \le \left\lfloor\frac{ab-bj-1}{a}\right\rfloor .$$

Define a DP:

$$dp(j,p)$$

where:

- rows $1,\dots,j$ remain,
- previously chosen column is $p$,
- future columns must be strictly larger than $p$.

The DP returns:

- number of configurations,
- total weight sum.

Transition:

- skip row $j$,
- or choose any admissible $i>p$.

If we choose $(i,j)$, we add weight

$$ab-ai-bj.$$

---

# Python implementation

```
from functools import lru_cache

def F(a, b):

    # allowed columns in each row
    rows = {}

    for j in range(1, a):
        mx = (a * b - b * j - 1) // a
        rows[j] = list(range(1, mx + 1))

    @lru_cache(None)
    def dp(j, prev_i):

        # no rows left
        if j == 0:
            return (1, 0)

        total_count = 0
        total_sum = 0

        # option 1: choose nothing from this row
        c, s = dp(j - 1, prev_i)
        total_count += c
        total_sum += s

        # option 2: choose one point from this row
        for i in rows[j]:

            if i > prev_i:

                c, s = dp(j - 1, i)

                weight = a * b - a * i - b * j

                total_count += c
                total_sum += s + c * weight

        return total_count, total_sum

    return dp(a - 1, 0)[1]

print(F(3, 5))    # 23
print(F(5, 13))   # 16336
print(F(19, 53))
```
