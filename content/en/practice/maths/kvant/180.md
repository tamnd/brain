## Problem

Two players play the following game. One of them thinks of a natural number $n$, and the other asks questions of the form “is it true that $n \ge x$” ($x$ he may choose at his own discretion) and receives answers “yes” or “no”. To each possible strategy $T$ of the second player we associate a function $f_T(n)$ equal to the number of questions (until the number is guessed), if the number $n$ was chosen.

Let, for example, strategy $T$ consist in asking first the questions: “is it true that $n \ge 10$?”, “is it true that $n \ge 20$?”, … until for some question “is it true that $n \ge 10 (k+1)$?” the answer “no” is given, and then asking questions “is it true that $n \ge 10k+1$?”, “that $n \ge 10k+2$?”, and so on. Then $f_T(n) = \dfrac{n-a}{10} + a + 2$, where $a$ is the last digit of the number $n$, i.e. $f_T(n)$ grows approximately like $\dfrac{n}{10}$.

1. Propose a strategy for which the function $f_T(n)$ grows as slowly as possible.
2. Comparing two strategies, it is convenient to introduce, instead of the function $f_T(n)$, the function $f_T'(n) = \underset{1 \leq k \leq n}{\max} f_T(k)$ — it shows how many questions are needed to guess any number not exceeding $n$. Give a lower bound for $f_T'(n)$ for an arbitrary strategy $T$.

Ya. M. Bardzin

## Exploration

Each question has the form “is $n \ge x$”, so every query splits the set of natural numbers into two contiguous parts. After $m$ questions, the strategy produces a binary decision tree of depth $m$, where each leaf corresponds to a set of possible values of $n$ consistent with the answers received.

To determine $n$ uniquely, each leaf must contain exactly one integer. Hence, after $m$ questions, at most $2^m$ different values of $n$ can be distinguished.

This immediately suggests that any strategy must have growth at least logarithmic in $n$. The remaining issue is whether this bound is tight and how to construct a strategy that achieves it up to a constant factor.

A naive linear strategy asks $1,2,3,\dots$, giving linear growth. A better idea is to first locate an interval containing $n$ efficiently, then perform a refined search inside it. Since the only operation is threshold comparison, the natural refinement tool is binary search once a finite interval is established.

The main subtlety is that the domain is unbounded, so the interval must be discovered first without spending too many queries. Doubling the tested threshold appears promising because it produces an exponential-size covering in logarithmic time.

The key point likely to govern optimality is the decision-tree bound: each query has only two outcomes, so the number of distinguishable numbers after $m$ steps cannot exceed $2^m$.

## Problem Understanding

The problem studies strategies for identifying an unknown natural number $n$ using adaptive questions of the form $n \ge x$. Each strategy $T$ induces a function $f_T(n)$ equal to the number of questions needed to determine $n$.

The first task asks for a strategy that minimizes the growth of $f_T(n)$ as $n$ increases. The second task introduces the worst-case function $f_T'(n)=\max_{1 \le k \le n} f_T(k)$ and asks for a universal lower bound valid for every strategy.

This is a classification problem of type B combined with a lower-bound extremal estimate: one must construct an efficient strategy and prove that no strategy can asymptotically do better than logarithmic growth.

The natural expectation is that the optimal growth is proportional to $\log_2 n$, since each question yields one bit of information.

## Proof Architecture

The first component is a construction lemma describing a strategy that first finds an exponential upper bound for $n$ and then performs binary search inside the obtained interval.

The second component is a decision-tree lemma stating that any strategy using $m$ questions distinguishes at most $2^m$ different numbers.

The third component converts this combinatorial bound into the inequality $f_T'(n) \ge \lceil \log_2 n \rceil$ by comparing the number of distinguishable outcomes with the size of the set ${1,2,\dots,n}$.

The most delicate point is ensuring that the binary search stage is valid after the exponential search stage, since correctness requires that the interval endpoints are deterministically established from the answers.

The lower-bound lemma is structurally rigid and depends on the injectivity of the mapping from numbers to answer sequences.

## Solution

A strategy achieving slow growth proceeds in two phases.

In the first phase, queries are asked with exponentially increasing thresholds $x=1,2,4,8,\dots,2^t$. This continues until the first time a “no” answer appears at some $2^t$. At that moment one knows that $2^{t-1} \le n < 2^t$, since the previous answer at $2^{t-1}$ must have been “yes”.

In the second phase, one performs binary search inside the known interval $[2^{t-1},2^t)$. At each step one asks whether $n$ is at least the midpoint of the current interval, thereby halving the remaining set of candidates. Since the interval contains at most $2^{t-1}$ integers, this phase requires at most $t-1$ questions to isolate $n$ exactly.

The total number of questions is therefore at most $t + (t-1) + O(1)$, where $t$ is the smallest integer with $2^t > n$. Hence $t = \lceil \log_2 n \rceil$, and the growth of $f_T(n)$ is linear in $\log_2 n$.

For the lower bound, fix an arbitrary strategy $T$ and consider all numbers $1,2,\dots,n$. After $m$ questions, each number produces a sequence of answers of length $m$, consisting of “yes” or “no”. This defines a mapping from ${1,\dots,n}$ into the set of binary strings of length $m$.

Distinct numbers must produce distinct answer sequences, since otherwise the strategy would not distinguish them after $m$ questions. Therefore, the number of possible values $n$ cannot exceed the number of binary strings of length $m$, which equals $2^m$.

Thus for $f_T'(n)=m$ we must have $n \le 2^m$, which implies $m \ge \log_2 n$. Since $m$ is an integer, this yields

$$f_T'(n) \ge \lceil \log_2 n \rceil.$$

This completes the derivation of the universal lower bound.

∎

## Verification of Key Steps

The correctness of the exponential-search phase relies on the fact that the first index $t$ with answer “no” at $2^t$ guarantees $n<2^t$ while the previous query ensures $n \ge 2^{t-1}$. This uses only monotonicity of the predicate $n \ge x$, which holds for all natural numbers.

In the binary search phase, each query partitions a known finite interval into two contiguous subintervals. Since the predicate is monotone, each answer preserves the invariant that $n$ lies in the remaining interval, and the interval size decreases by at least a factor of two at each step, ensuring termination in logarithmic time.

For the lower bound, the critical point is injectivity of the mapping from numbers to answer sequences. If two distinct numbers produced identical sequences of answers, no strategy could distinguish them, contradicting the requirement that the process must identify $n$ exactly. This forces the upper bound of $2^m$ distinguishable numbers after $m$ questions.

## Alternative Approaches

An alternative construction avoids the explicit exponential phase by directly maintaining an interval whose endpoints are powers of two determined dynamically, effectively embedding both phases into a single adaptive binary search on an expanding domain. This yields the same logarithmic complexity but is less transparent because it mixes interval discovery and refinement.

Another viewpoint is to interpret the process as building a decision tree for an ordered infinite set, where the optimal tree is balanced in the sense of minimizing maximum depth over the first $n$ leaves. The lower bound then becomes a direct consequence of entropy considerations, since each question contributes at most one bit of information.
