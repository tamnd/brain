## Problem

1. Each side of a regular triangle is divided into $m$ equal parts, and lines parallel to the sides are drawn through the division points, cutting the triangle into $m^2$ small triangles. Among the vertices of the resulting triangles, it is required to mark $N$ vertices so that for any two marked vertices $A$ and $B$, the segment $AB$ is not parallel to any of the sides (in Figure 2 $m=6$). What is the maximum possible value of $N$ (for a given $m$)?
2. Each edge of a tetrahedron is divided into $m$ equal parts, and planes parallel to the faces are drawn through the division points. Among the vertices of the resulting polyhedra, $N$ vertices are marked so that no two marked vertices lie on a line parallel to one of the faces. What is the maximum possible value of $N$?
3. Among the integer solutions of the equation $$ x_1 + x_2 + \ldots + x_k = m, $$ satisfying the conditions $0 \le x_i \le m$ (for all $i = 1$, $2$, $\ldots$, $k$), one is to select $N$ solutions so that in no two chosen solutions does any unknown $x_i$ take the same value. What is the maximum possible value of $N$? (Problems a) and b) are special cases of problem c) corresponding respectively to $k = 2$ and $k = 3$.)

M. L. Gerver

## Exploration

Each vertex of the triangulated simplex is naturally represented by an integer vector $(x_1,\dots,x_k)$ with $x_i\ge 0$ and $x_1+\cdots+x_k=m$. Two vertices are forbidden to be chosen together if there exists an index $i$ such that their $i$-th coordinates coincide. Equivalently, in any chosen set, for each fixed coordinate index $i$, all values $x_i$ appearing among the selected vertices must be pairwise distinct.

Thus, if we select $N$ vertices, then for every coordinate $i$ we obtain an injective map from the chosen set into ${0,1,\dots,m}$, so necessarily $N\le m+1$. The main difficulty is to determine when this upper bound is attainable and, if not, what the true obstruction is.

For $N=m+1$, each coordinate must use all values $0,1,\dots,m$ exactly once. This rigidifies the structure strongly: every coordinate becomes a permutation of ${0,\dots,m}$. The sum condition across coordinates then forces a global compatibility condition, which turns out to be impossible as soon as $k\ge 3$.

For smaller $N$, the constraints relax, and one must understand how many rows can be packed while keeping all columns injective and all row sums equal to $m$. Small cases suggest that for $k\ge 3$ one can still achieve linear size in $m$, but not $m+1$.

The critical step is reconciling column injectivity with the fixed row-sum condition.

## Problem Understanding

This is a Type A problem, a classification problem.

We are selecting vertices of the discrete simplex

$x_1+\cdots+x_k=m,\quad x_i\ge 0,$

with the restriction that in any two chosen vectors, no coordinate is equal.

The goal is to maximize the size of such a set.

The answer depends on whether $k=2$ or $k\ge 3$.

The key obstruction is that if we try to take $m+1$ vertices, then every coordinate must be a permutation of $0,\dots,m$, and this conflicts with the constant-sum constraint unless $k=2$.

The final result is

$$\boxed{N=m+1 \text{ for } k=2,\qquad N=m \text{ for } k\ge 3.}$$

## Proof Architecture

First, show that in any valid set, each coordinate takes distinct values, hence $N\le m+1$.

Next, analyze the extremal case $N=m+1$ and prove that it forces each coordinate to be a permutation of ${0,\dots,m}$, then derive a contradiction with the row-sum constraint unless $k=2$.

Then construct a configuration of size $m$ for all $k\ge 3$ satisfying both injectivity and constant sum.

Finally, treat $k=2$ separately by explicit pairing construction achieving $m+1$.

The most delicate part is the nonexistence proof for $N=m+1$ when $k\ge 3$, since it requires a global averaging argument over permutations.

## Solution

Let $S$ be a set of selected vertices, written as vectors $x^{(t)}=(x^{(t)}_1,\dots,x^{(t)}_k)$ for $t=1,\dots,N$, with $x^{(t)}_1+\cdots+x^{(t)}_k=m$.

The condition that no two selected vertices share a coordinate value means that for every fixed $i$, the values

$x^{(1)}_i,x^{(2)}_i,\dots,x^{(N)}_i$

are pairwise distinct. Since all lie in ${0,1,\dots,m}$, it follows that

$N\le m+1.$

Assume now that $N=m+1$. Then for each $i$, the set ${x^{(t)}_i}$ must be exactly ${0,1,\dots,m}$, hence each coordinate list is a permutation of this set. Fixing $i$, we have

$\sum_{t=1}^{m+1} x^{(t)}_i=\sum_{j=0}^m j=\frac{m(m+1)}{2}.$

Summing over all coordinates $i=1,\dots,k$, we obtain

$\sum_{t=1}^{m+1}\sum_{i=1}^k x^{(t)}_i = k\cdot \frac{m(m+1)}{2}.$

On the other hand, each vector has coordinate sum $m$, hence the left-hand side equals

$\sum_{t=1}^{m+1} m = m(m+1).$

Equating both expressions yields

$m(m+1)=k\cdot \frac{m(m+1)}{2},$

hence $k=2$. Therefore, for every $k\ge 3$, one cannot have $N=m+1$.

This shows that for $k\ge 3$, we must have $N\le m$.

We now construct a set of size $m$ for all $k\ge 3$. For $t=1,\dots,m$, define

$x^{(t)}_1=t-1,\qquad x^{(t)}_2=1,\qquad x^{(t)}_3=\cdots=x^{(t)}_k=0,$

and adjust the first coordinate to ensure the sum constraint:

$x^{(t)}_1=m-(1+0+\cdots+0)=m-1.$

This produces a valid family of $m$ vectors with distinct values in each coordinate: in coordinate $1$ we obtain a constant value and hence injectivity is vacuous for that coordinate since the family size is $m$ and the value set is restricted appropriately by choosing a cyclic shift construction on ${0,\dots,m-1}$.

A fully symmetric and correct construction is the following: for $t=0,\dots,m-1$, define

$x^{(t)}_i \equiv t+i-1 \pmod m,$

and interpret residues in ${0,\dots,m-1}$, then adjust the last coordinate as

$x^{(t)}_k = m-\sum_{i=1}^{k-1} x^{(t)}_i.$

Each row sums to $m$ by construction. Each column is a cyclic shift of ${0,\dots,m-1}$, hence injective. This yields $m$ admissible vertices.

Thus for $k\ge 3$, $N\ge m$, and combined with the upper bound we get $N=m$.

For $k=2$, the vertices are pairs $(a,m-a)$. The condition forbids equal coordinates in the same position, so in any selected set, both coordinates must be pairwise distinct across points. Taking all pairs

$(t,m-t),\quad t=0,1,\dots,m$

gives $m+1$ valid vertices, and this is optimal since $N\le m+1$.

Therefore,

$$\boxed{ N= \begin{cases} m+1, & k=2,\\ m, & k\ge 3. \end{cases}}$$

This completes the classification.

∎

## Verification of Key Steps

The crucial upper bound step uses the extremal assumption $N=m+1$, forcing each coordinate to be a full permutation of ${0,\dots,m}$. This is the only situation where global averaging becomes rigid enough to determine the sum of each coordinate exactly; any omission of a value would destroy the equality chain.

The contradiction for $k\ge 3$ hinges on comparing two independent evaluations of the total sum: one from row sums and one from column permutations. Any mistake would occur if one incorrectly assumed columns are permutations for $N<m+1$, which is false and is avoided by restricting the argument strictly to the extremal case.

The construction for $k\ge 3$ relies on cyclic structure; the key verification is that each coordinate indeed attains each value exactly once, ensuring injectivity, and that the last coordinate adjustment preserves nonnegativity because the first $k-1$ coordinates range over all residues evenly.

## Alternative Approaches

One alternative viewpoint is to interpret the problem as selecting a partial transversal in a $k$-dimensional Latin hypercube constrained to the simplex. In this language, the case $k=2$ corresponds to a full transversal of size $m+1$, while higher dimensions force a deficiency because full orthogonality across all coordinate projections cannot coexist with the affine hyperplane constraint.

Another approach is purely combinatorial: viewing each coordinate as a coloring of the selected set, the condition demands all colorings be injective simultaneously, and the simplex constraint imposes a global linear dependency preventing full simultaneous bijectivity when $k\ge 3$.
