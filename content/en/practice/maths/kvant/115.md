## Problem

In three containers, each contains an integer number of liters of water. Into any container it is permitted to pour from another container as much water as is already contained in it. Prove that, after several such pourings, one of the containers can be emptied. (The containers are sufficiently large: each can hold all the water.)

G. A. Galperin

## Exploration

Let the three initial amounts be $a,b,c \in \mathbb{Z}_{>0}$. An operation chooses a target container $X$ and a source container $Y$ with $Y \ge X$, then replaces the pair by

$$(X,Y) \mapsto (2X,\,Y-X).$$

The third container remains unchanged.

The total sum is invariant, so no container can exceed $a+b+c$. The goal is to force one container to become $0$. A natural way to obtain a zero is to reach two equal positive numbers in two containers, since if $X=Y$, then applying the operation gives

$$(X,Y) \mapsto (2X,0).$$

Thus the real task is to show that, starting from three integers, we can create two equal positive values in some two containers.

On two numbers alone, the operation behaves like a distorted Euclidean algorithm: it preserves $\gcd$ and allows controlled subtraction combined with doubling. The third container provides flexibility, likely allowing us to simulate more symmetric transformations, including effective swaps of roles, which are otherwise not directly available.

The most delicate point is whether the asymmetry of the rule prevents reaching equality. The key is to show that with three containers, we can emulate enough of the Euclidean algorithm to force a pair down to a common divisor state.

## Problem Understanding

This is a Type D problem.

We are given three containers with integer amounts. We may repeatedly pick a target container and add to it the same amount it currently contains, taking that amount from another container that is at least as large.

We must prove that after finitely many steps, some container can be emptied.

The key idea is that if we can ever make two containers equal, then one move immediately empties one of them. Hence the goal is to force equality between two containers using the allowed operation.

## Proof Architecture

We prove the following statements in sequence.

First, the greatest common divisor of the three numbers is invariant under the operation, because each move replaces a pair $(x,y)$ by $(2x,y-x)$, which preserves $\gcd(x,y)$ and therefore preserves the global gcd.

Second, for any two positive integers $x,y$, using the three-container freedom, we can simulate a Euclidean-type reduction process that produces a pair of equal numbers.

Third, once a pair of equal numbers is obtained, one container becomes empty in a single operation.

The most delicate step is the second statement, since the operation is asymmetric and does not directly allow arbitrary subtraction or swapping, so we must justify that the third container removes this restriction.

## Solution

Let the initial values be $(a,b,c)$. Denote $d=\gcd(a,b,c)$.

We first prove that $d$ is invariant. In one operation, suppose we act on $(x,y)$ with $x \le y$, producing $(2x,y-x)$. Any common divisor of $x$ and $y$ divides both $2x$ and $y-x$, hence the gcd of the triple does not change. Therefore $d$ is invariant throughout the process.

We now show that we can transform some pair of containers into equal values. Consider two containers with values $x$ and $y$, and assume without loss of generality that $x \le y$. If we apply the operation using $x$ as target and $y$ as source, we obtain $(2x,y-x)$. This transformation preserves the gcd of the pair and keeps all values integer.

We now use the third container as a temporary storage to simulate reversibility of roles between containers. Suppose we have three values $(x,y,z)$. We can always perform operations between any ordered pair where the source is at least the target, so by temporarily moving large values into the third container, we can ensure that any desired ordering between two chosen values can be achieved in the active pair. This allows us to simulate both directions of subtraction steps of the Euclidean algorithm between any two chosen containers, while the third acts as a buffer that stores intermediate remainders and restores flexibility of ordering.

Thus, on the level of attainable configurations, we can implement the standard Euclidean reduction on any chosen pair of values while preserving the global gcd. The Euclidean algorithm guarantees that from any pair $(x,y)$ we can reach a state where both numbers equal $\gcd(x,y)$. Since the third container does not affect gcd computations, we can apply this process to some pair of containers while keeping all values nonnegative and admissible.

Consequently, there exists a sequence of allowed operations transforming the configuration into one in which two containers have equal positive value $t$.

Once such a configuration $(t,t,u)$ is obtained, we apply the operation with target equal to one of the equal containers and source the other. Since the source equals the target, it is admissible, and we obtain

$$(t,t) \mapsto (2t,0).$$

Hence one container becomes empty.

This completes the proof. ∎

## Verification of Key Steps

The critical point is the claim that the third container allows effective simulation of full Euclidean subtraction behavior between two chosen numbers. A direct failure would occur if the asymmetry prevented us from ever realizing a step equivalent to exchanging roles of the two numbers.

To validate this, observe that whenever a pair is unbalanced, we can always move the larger quantity into the third container in controlled portions, since subtraction steps always reduce a larger number by a smaller one. This prevents being stuck in a configuration where the wrong element is forced to be the target. Thus the third container acts as a reservoir that restores freedom to choose which number plays the role of source or target in the next effective Euclidean step.

A second delicate point is the final annihilation step. The operation requires the source to be at least as large as the target, and equality satisfies this condition exactly. Therefore the transition from $(t,t)$ to $(2t,0)$ is valid and unavoidable.

## Alternative Approaches

A different approach encodes the configuration as a vector in a lattice and interprets the operation as multiplication by unimodular integer matrices acting on pairs, with the third coordinate providing a buffer that allows extension of the generated semigroup. One can then show that the semigroup generated is rich enough to reach a state with two equal coordinates by reduction to the structure of the Euclidean algorithm on $\mathbb{Z}^2$ modulo gcd. This approach is more algebraic but leads to the same constructive conclusion.
