## Problem

Four ones and five zeros are written on a circle in an arbitrary order. Then, in each gap between two equal numbers, a 1 is written, and between two different digits, a 0 is written; the original digits are then erased. Prove that no matter how many times this process is repeated, one never obtains a configuration of nine ones.

E. B. Dynkin, S. A. Molchanov, A. L. Rosenthal, A. K. Tolpygo

Mathematical Problems. 3rd ed. — Moscow: Nauka, 1971.

## Exploration

Let the digits on the circle be $a_1,\dots,a_9\in{0,1}$, with indices taken modulo $9$. The rule produces a new configuration $b_i$ where $b_i=1$ if $a_i=a_{i+1}$ and $b_i=0$ if $a_i\ne a_{i+1}$.

Since the digits are binary, this can be written as

$$b_i=1-(a_i\oplus a_{i+1}),$$

where $\oplus$ denotes addition modulo $2$.

The target configuration is nine ones. That means every adjacent pair is equal. On a circle this occurs only for the constant configurations $000000000$ and $111111111$. Thus obtaining nine ones at some stage means that the preceding stage had all adjacent pairs equal, hence was already constant.

Can a constant configuration ever appear? The initial configuration has four ones and five zeros, so it is not constant.

A quantity that might be preserved is the parity of the number of ones. Let

$$s=a_1+\cdots+a_9 \pmod 2.$$

Then

$$\sum b_i = \sum (1+a_i+a_{i+1}) \pmod 2,$$

because over $\mathbf F_2$ we have $1-(x+y)=1+x+y$.

Since there are $9$ terms,

$$\sum b_i \equiv 9+2\sum a_i \equiv 1 \pmod 2.$$

Thus every image has an odd number of ones, regardless of the previous configuration.

The initial configuration has four ones, an even number. After one step the number of ones is odd. After another step it is again odd, and so on. Hence every configuration from the first step onward has odd parity.

A constant configuration on nine positions is either $000000000$ with $0$ ones or $111111111$ with $9$ ones. Both correspond to oddity? No: $0$ is even, $9$ is odd. So parity alone excludes only all zeros. We still need to exclude all ones.

Observe that if all ones ever appear, then the preceding configuration must be constant. Among constant configurations, only $111111111$ has odd parity. Thus if all ones appear at some time, then all ones must already have appeared one step earlier. Repeating this argument indefinitely forces all ones to have been present from the first step onward. But the first produced configuration has odd parity, so this is not yet impossible. A sharper invariant is needed.

Consider instead the parity of the number of zeros. Since there are $9$ positions, odd parity of ones means even parity of zeros. The configuration $111111111$ has zero zeros, indeed even. So parity alone is insufficient.

Try counting transitions. Let

$$T(a)=\#\{i:a_i\ne a_{i+1}\}.$$

Then $b_i=0$ exactly at transitions, so the number of zeros in the new configuration equals $T(a)$.

On a circle, every time one enters a block of ones one must later leave it. Hence $T(a)$ is always even. Therefore every produced configuration contains an even number of zeros.

If a configuration consists of nine ones, it contains zero zeros, also even, so this still does not exclude it.

A better observation: nine ones arise from a constant predecessor. The only constant configuration reachable after the first step would have to contain an even number of zeros. The constant configuration $000000000$ has nine zeros, odd, so it is impossible after the first step. The constant configuration $111111111$ would require its predecessor already to be $111111111$. Thus once the process has started from a nonconstant configuration, reaching all ones would imply that all ones had already occurred at every earlier step, eventually contradicting the initial state.

This backward-invariance argument appears sufficient.

The delicate point is proving rigorously that the only predecessor of $111111111$ is $111111111$ itself.

## Problem Understanding

We have nine binary digits arranged on a circle, initially consisting of four ones and five zeros. At each step, every position is replaced by $1$ if the two neighboring digits defining that gap are equal and by $0$ if they are different. We must prove that repeated application of this rule never yields the configuration consisting entirely of ones.

This is a Type B problem. The goal is to prove a statement for all iterations of the process.

The core difficulty is to identify a property of the transformation that prevents the system from entering the state $111111111$. The crucial fact is that a configuration of nine ones can arise only from itself.
