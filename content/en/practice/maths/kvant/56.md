## Problem

Four ones and five zeros are written on a circle in arbitrary order. Then, in each interval between two equal numbers, a one is written, and between two different digits, a zero is written; the original digits are then erased. Prove that no matter how many times this process is repeated, one can never obtain a configuration consisting of nine ones.

E. B. Dynkin, S. A. Molchanov, A. L. Rosenthal, A. K. Tolpygo

Mathematical Problems. 3rd ed. – Moscow: Nauka, 1971.

## Exploration

Let the digits on the circle be $a_1,\dots,a_9\in{0,1}$, with indices taken modulo $9$. The rule says that the new digit between $a_i$ and $a_{i+1}$ is $1$ if they are equal and $0$ if they are different.

Over the field $\mathbb F_2$, equality of two bits is expressed by

$$1+a_i+a_{i+1},$$

since this equals $1$ when $a_i=a_{i+1}$ and $0$ otherwise. Thus the transformation is

$$b_i=1+a_i+a_{i+1}.$$

The target configuration is nine ones. A natural idea is to search for an invariant modulo $2$. The initial configuration contains four ones and five zeros, hence the sum of all digits is

$$4\equiv 0 \pmod 2.$$

Compute the parity of the sum after one step:

$$\sum b_i = \sum (1+a_i+a_{i+1}) = 9+2\sum a_i.$$

Modulo $2$ this is

$$\sum b_i\equiv 1.$$

The parity flips. Repeating once more,

$$\sum c_i = 9+2\sum b_i,$$

so again

$$\sum c_i\equiv 1.$$

The parity does not alternate; it becomes permanently odd after the first step.

This suggests that every configuration after the first iteration has an odd number of ones. The configuration of nine ones also has odd parity, so parity alone does not exclude it.

Another quantity is the sum modulo $2$ of the digits themselves. Since after the first step it is always $1$, perhaps the map preserves this value thereafter. Indeed, if

$$S(a)=\sum a_i \pmod 2,$$

then

$$S(T(a)) = 9+2S(a) \equiv 1.$$

Hence every image of the transformation has parity $1$.

To obtain nine ones at some stage, that stage must be an image of the preceding stage. Thus the crucial question is whether the all-ones configuration lies in the image of the transformation.

Suppose

$$b_i=1$$

for all $i$. Then

$$1+a_i+a_{i+1}=1,$$

hence

$$a_i=a_{i+1}$$

for every $i$. Around the circle this forces all $a_i$ to be equal. Therefore the only preimages of nine ones are the configurations of nine zeros and of nine ones.

Apply the transformation to these two configurations. Both are fixed:

$$T(0,\dots,0)=(1,\dots,1),\qquad T(1,\dots,1)=(1,\dots,1).$$

Thus nine ones can occur only if at the previous step all digits were equal.

Now check whether a configuration with all digits equal can ever arise. If all digits are zero, the parity of the number of ones is $0$, contradicting the fact that every image has parity $1$. If all digits are one, then the parity is $9\equiv1$, so parity does not exclude it. A better observation is that the only preimages of nine ones are again constant configurations. Tracing backward repeatedly, any occurrence of nine ones would force all earlier configurations to be constant. Since the initial configuration contains both zeros and ones, this is impossible.

The step most likely to hide an error is the backward implication from an occurrence of nine ones to all preceding configurations being constant. This must be proved carefully.

## Problem Understanding

We have nine positions on a circle, initially containing four ones and five zeros in an arbitrary arrangement. At each step, every adjacent pair is replaced by a digit: $1$ if the two digits are equal and $0$ if they are different. The process is repeated indefinitely.

This is a Type B problem. We must prove that a configuration consisting entirely of ones can never appear.

The core difficulty is to identify a property that allows information to be propagated backward from a hypothetical all-ones configuration. The key fact is that a configuration of nine ones has only two preimages, namely the two constant configurations.

## Proof Architecture

Lemma 1. If a configuration is mapped to nine ones, then every digit in the original configuration is the same. This follows because each output digit equals $1$ exactly when the corresponding adjacent input digits are equal.

Lemma 2. The only preimages of the configuration of nine ones are the configurations of nine zeros and of nine ones. This is a reformulation of Lemma 1.

Lemma 3. If a configuration is constant, then its image is nine ones. This is checked directly from the rule.

Lemma 4. Any configuration that eventually maps to nine ones must itself be constant. Apply Lemma 2 repeatedly backward.

The hardest direction is Lemma 4, because one must justify carefully why repeated backward application forces every earlier configuration to be constant.

## Solution

Let $T$ denote the transformation described in the problem.

Assume, seeking a contradiction, that after some number of iterations a configuration consisting of nine ones appears. Let this happen for the first time at step $m$.

Consider a configuration

$$a=(a_1,\dots,a_9),$$

where indices are taken modulo $9$. Its image

$$T(a)=(b_1,\dots,b_9)$$

is defined by the rule

$$b_i= \begin{cases} 1,& a_i=a_{i+1},\\ 0,& a_i\ne a_{i+1}. \end{cases}$$

Suppose that

$$T(a)=(1,1,\dots,1).$$

Then for every $i$ we have $b_i=1$, hence

$$a_i=a_{i+1}.$$

From

$$a_1=a_2,\quad a_2=a_3,\quad \dots,\quad a_9=a_1,$$

it follows that

$$a_1=a_2=\cdots=a_9.$$

Thus every preimage of the configuration of nine ones is constant. Since the digits are only $0$ and $1$, the only possibilities are

$$(0,0,\dots,0)$$

and

$$(1,1,\dots,1).$$

Conversely, each of these two constant configurations maps to nine ones, because every adjacent pair consists of equal digits.

Therefore the configuration of nine ones has exactly two preimages, and both are constant.

Now suppose that the configuration of nine ones occurs at step $m$. By the preceding paragraph, the configuration at step $m-1$ must be constant. A constant configuration maps to nine ones, so the configuration at step $m-1$ is either nine zeros or nine ones.

In either case, the configuration at step $m-2$ must again be a preimage of nine ones. Hence it is also constant. Repeating this argument, we obtain that every earlier configuration, all the way back to step $0$, is constant.

The initial configuration, however, contains four ones and five zeros. It is not constant.

This contradiction shows that no iteration can ever produce the configuration consisting of nine ones.

This completes the proof.

∎

## Verification of Key Steps

The first delicate step is the determination of all preimages of nine ones. If every output digit equals $1$, then every adjacent pair of input digits must be equal. The chain

$$a_1=a_2=a_3=\cdots=a_9=a_1$$

forces all nine digits to coincide. No nonconstant configuration satisfies these equalities.

The second delicate step is the backward iteration. From the fact that step $m$ is nine ones, one may conclude only that step $m-1$ is constant. It is necessary to continue the argument. Since every constant configuration maps to nine ones, step $m-1$ itself is a preimage of nine ones. Applying the same reasoning again shows that step $m-2$ is constant, and so on. This inductive backward propagation reaches the initial configuration.

A common error would be to prove only that the immediate predecessor of nine ones is constant and then stop. The contradiction arises only after propagating this information all the way to the initial state.

## Alternative Approaches

Represent the digits by elements of $\mathbb F_2$. If $a_i$ is the digit at position $i$, then the transformation is

$$b_i=1+a_i+a_{i+1}.$$

The configuration of nine ones is the constant vector $\mathbf 1$. Solving

$$1+a_i+a_{i+1}=1$$

gives

$$a_i=a_{i+1}$$

for all $i$, hence the kernel of $T-\mathbf 1$ consists exactly of the two constant vectors. Thus every preimage of $\mathbf 1$ is constant. Iterating this observation backward yields the same contradiction as in the main proof.

The direct combinatorial proof is preferable because it uses only the definition of the operation and avoids introducing algebraic notation. The crucial structure, namely that an all-ones output forces equality of every neighboring pair in the preceding configuration, becomes completely transparent.
