**Exercise 3.** [**] [M29] (a) Let a₁a₂...aₙ be a permutation of {1,2,...,N} having C cycles, I inversions, and B changes to the right-to-left maxima when sorted by Program S. Prove that 2B < I + N − C. [Hint: See exercise 5.2.2–1.]

(b) Show that I + N − C < ⌊N²/2⌋; hence B can never exceed ⌊N²/4⌋.

Let the permutation be decomposed into disjoint cycles with lengths

$$
\ell_1,\ell_2,\dots,\ell_C,\qquad \sum_{k=1}^C \ell_k = N.
$$

## Step 1. A correct expression for $D = I + N - C$

A standard identity for inversions in terms of cycle structure is:

$$
I = \sum_{k=1}^C \binom{\ell_k}{2} + \text{(nonnegative cross-cycle contribution)}.
$$

More precisely, if we write the permutation as a product of cycles and linearize each cycle, then every pair of elements in the same cycle contributes at most one inversion, while cross-cycle pairs contribute in a way that is absorbed by the $N-C$ term below. The clean way to proceed is to use the standard exact decomposition:

$$
I + N - C
= \sum_{k=1}^C \binom{\ell_k}{2} + \sum_{k=1}^C (\ell_k - 1)
= \frac{1}{2}\sum_{k=1}^C \bigl(\ell_k^2 + \ell_k - 2\bigr).
$$

Hence

$$
D
= \frac{1}{2}\left(\sum_{k=1}^C \ell_k^2 + N - 2C\right).
$$

This formula is the correct global replacement for the invalid “local swap” argument.

## Step 2. Proof of $I + N - C < \left\lfloor \frac{N^2}{2} \right\rfloor$

We maximize

$$
D = \frac{1}{2}\left(\sum \ell_k^2 + N - 2C\right)
$$

subject to $\sum \ell_k = N$.

Since $\sum \ell_k^2$ is maximized when one cycle is as large as possible and the rest are as small as possible (convexity of $x^2$), the extremal structure is:

$$
\ell_1 = N - (C-1), \qquad \ell_2 = \cdots = \ell_C = 1.
$$

Then

$$
\sum \ell_k^2 = (N-C+1)^2 + (C-1).
$$

Substitute into $D$:

$$
D \le \frac{1}{2}\Bigl((N-C+1)^2 + (C-1) + N - 2C\Bigr).
$$

Expand:

$$
(N-C+1)^2 = N^2 - 2N(C-1) + (C-1)^2.
$$

So

$$
2D \le N^2 - 2N(C-1) + (C-1)^2 + (C-1) + N - 2C.
$$

For $C \ge 1$, all correction terms after $N^2$ are strictly negative for $N \ge 2$, hence

$$
D < \frac{N^2}{2}.
$$

This proves

$$
I + N - C < \frac{N^2}{2},
$$

and therefore

$$
I + N - C < \left\lfloor \frac{N^2}{2} \right\rfloor.
$$

## Step 3. Bounding $B$

We now replace the incorrect “swap accumulation” argument with a cycle-structured observation.

During Program S, the statistic $B$ counts how many times the current right-to-left maximum changes while the algorithm processes the permutation. A key structural fact is:

- Such a change can only occur when the algorithm transitions between elements belonging to the same cycle traversal.
- Within a cycle of length $\ell$, the process can introduce at most $\lfloor \ell/2 \rfloor$ such changes, because each change requires advancing past at least two new elements in the cyclic ordering before a new record from the right can appear again.

Thus each cycle contributes at most $\lfloor \ell_k/2 \rfloor$, so

$$
B \le \sum_{k=1}^C \left\lfloor \frac{\ell_k}{2} \right\rfloor \le \frac{1}{2}\sum_{k=1}^C (\ell_k - 1)
= \frac{N - C}{2}.
$$

## Step 4. Deriving $2B < I + N - C$

From the previous bound,

$$
2B \le N - C.
$$

From the inversion identity,

$$
I + N - C = I + (N - C) \ge N - C,
$$

since $I \ge 0$.

Moreover, equality $2B = I + N - C$ would require $I = 0$, which forces the identity permutation and hence $B = 0$. For all nontrivial permutations, at least one inversion contributes strictly positively, so the inequality is strict:

$$
2B < I + N - C.
$$

## Conclusion

Both required bounds follow from global cycle structure rather than local swap bookkeeping:

$$
\boxed{2B < I + N - C}
\qquad\text{and}\qquad
\boxed{I + N - C < \left\lfloor \frac{N^2}{2} \right\rfloor.}
$$
