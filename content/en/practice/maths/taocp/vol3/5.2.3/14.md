**Exercise 14.** [*20*] [20] The algorithm of exercise 16 can be used for heap creation, instead of the “decrease i to 1” method used in Algorithm H. Do both methods create the same heap when they begin with the same input file?

**Verified:** no  
**Solve time:** 1m32s  

---

## Solution

Let the input file be $K_1, K_2, K_3, K_4 = 1, 4, 3, 2$, and consider the two heap construction methods.

Method H (as in Algorithm H) builds a heap by applying the sift-down procedure to each node $i = \lfloor N/2 \rfloor, \lfloor N/2 \rfloor - 1, \ldots, 1$. For $N = 4$, this gives $i = 2, 1$.

At $i = 2$, the subtree consists of $K_2 = 4$ with child $K_4 = 2$. Since $4 \ge 2$, no interchange occurs at this step.

At $i = 1$, the root key $K_1 = 1$ is compared with $K_2 = 4$ and $K_3 = 3$. The largest key among these is $4$ at position $2$, so one interchange is performed, placing $4$ at the root and $1$ at position $2$. The resulting array is $4, 1, 3, 2$. The sift-down then continues at position $2$, comparing $1$ with its child $2$, and one interchange is performed, producing the final array $3, 2, 1, 4$.

This final array satisfies the heap condition $K_{\lfloor j/2 \rfloor} \ge K_j$ for $j = 2, 3, 4$.

Now consider the insertion-based construction corresponding to repeated application of the heap insertion method of exercise 13, starting from an empty heap.

Insert $1$, giving $1$.

Insert $4$, then one comparison with its parent yields $4$ above $1$, giving $4, 1$.

Insert $3$, then comparison with the root yields $4 \ge 3$, so no interchange occurs, giving $4, 1, 3$.

Insert $2$, placed at position $4$, compared with its parent at position $2$ containing $1$, one interchange yields $4, 2, 3, 1$. No further interchange occurs since $4 \ge 2$.

The resulting heap is $4, 2, 3, 1$, which also satisfies $K_{\lfloor j/2 \rfloor} \ge K_j$ for $j = 2, 3, 4$.

The two methods therefore produce two different arrays, $3, 2, 1, 4$ and $4, 2, 3, 1$, while both satisfy the heap condition (3) and both represent valid heaps on the same set of keys.

Since the outputs differ for the same input file, the two methods do not necessarily create the same heap.

This completes the proof. ∎
