**Exercise 13.** [*20*] [20] Design an efficient algorithm that inserts a new key into a given heap of n elements, producing a heap of n + 1 elements.

## Solution

Let $K_1, K_2, \ldots, K_n$ satisfy the heap condition

$K_{\lfloor j/2 \rfloor} \ge K_j \quad (1 < j \le n).$

A new key $x$ is to be inserted so that the resulting sequence of length $n+1$ again satisfies the heap condition.

The insertion proceeds by placing $x$ at the end of the array and then moving it upward until the heap condition is restored.

Let $j \leftarrow n+1$ and place $x$ in location $K_j$ conceptually. The parent of position $j$ is $\lfloor j/2 \rfloor$.

Define the following procedure.

Step I1. Set $j \leftarrow n+1$.

Step I2. If $j = 1$, go to Step I5.

Step I3. If $K_{\lfloor j/2 \rfloor} \ge x$, go to Step I5.

Step I4. Set $K_j \leftarrow K_{\lfloor j/2 \rfloor}$, then set $j \leftarrow \lfloor j/2 \rfloor$, and return to Step I2.

Step I5. Set $K_j \leftarrow x$.

This constructs a heap of size $n+1$.

Correctness follows from an invariant maintained during the upward movement of $x$. At any stage, all nodes not on the path from the current index $j$ to the root satisfy the heap condition, since their relative positions are unchanged from the original heap. The only potential violations lie along the path from $j$ upward.

At Step I4, replacing $K_{\lfloor j/2 \rfloor}$ by moving it down to position $j$ preserves the heap condition at position $j$ because the value moved down is no smaller than any of its children in the original heap, and all children of $j$ lie in a subtree unchanged by previous steps. The index then moves to $\lfloor j/2 \rfloor$, and the element $x$ is compared only with ancestors on the unique path to the root.

The process terminates either when $j = 1$ or when $K_{\lfloor j/2 \rfloor} \ge x$. In the first case, $x$ is placed at the root, and the heap condition holds because every descendant is no larger than the original root values displaced along the path. In the second case, placing $x$ at position $j$ ensures $K_{\lfloor j/2 \rfloor} \ge x$, and all lower heap relations remain valid by construction.

The number of iterations of Step I4 equals the number of edges on the path from node $n+1$ to the root. In a complete binary tree with $n+1$ nodes, this path length is $\lfloor \log_2(n+1) \rfloor$. Each iteration performs one comparison and one assignment, so the procedure requires at most $\lfloor \log_2(n+1) \rfloor$ comparisons and the same number of data movements.

No algorithm can in general do better in the worst case, since a key inserted at a leaf of a complete binary heap may need to move to the root, and each move can only correct the heap condition one level at a time along the unique parent chain. Thus any comparison-based insertion procedure must examine at least one element on each level of this path in the worst case.

This completes the proof. ∎
