**Exercise 11.** [*10*] [10] Show that simple queues are special cases of priority queues. (Explain how keys can be assigned to the elements so that a largest-in-first-out procedure is equivalent to first-in-first-out.) Is a stack also a special case of a priority queue?

**Verified:** yes  
**Solve time:** 1m29s  

---

Let the elements of the queue be inserted in chronological order $1,2,\dots$, where the $t$-th inserted element receives key $K=t$ mapped to $K'=-t$. Since a priority queue removes an element of largest key, and $-t_1>-t_2$ whenever $t_1<t_2$, the element inserted first always has highest priority among those present. Therefore each deletion removes the earliest inserted element, which is exactly first-in-first-out behavior. Thus a simple queue is obtained as a priority queue under the key assignment $K'=-t$ (or equivalently any strictly decreasing function of insertion time).

A stack is also a special case of a priority queue. If the $t$-th inserted element is assigned key $K=t$, then the most recently inserted element has the largest key among all present elements. Since a priority queue deletes a maximum-key element, each deletion removes the most recent insertion, which is last-in-first-out behavior. This establishes the required correspondence. ∎
