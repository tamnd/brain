**Exercise 12.** [*M22*] [M22] (B. A. Chartres.) Design a high-speed algorithm that builds a table of the prime numbers < N, making use of a priority queue to avoid division operations. [Hint: Let the smallest key in the priority queue be the least odd nonprime number greater than the last odd number considered as a prime candidate. Try to minimize the number of elements in the queue.]

## Solution

Let the odd integers to be tested be

$3, 5, 7, \ldots < N,$

and let a priority queue $Q$ store records of the form $(x, d)$, where $x$ is an odd composite value and $d$ is an increment (always of the form $2p$ for some prime $p$). The key used in the queue is the first component $x$.

At every stage of the algorithm, $Q$ contains exactly one record for each prime $p$ for which a multiple of $p$ has been generated but not yet fully advanced beyond the current scanning position. The record for $p$ has the form $(x_p, 2p)$, where $x_p$ is the least odd multiple of $p$ such that $x_p \ge p^2$ and $x_p$ exceeds all odd numbers already classified with respect to $p$.

Let $c$ denote the current odd candidate under consideration. The algorithm maintains the invariant that every odd integer $< c$ has already been classified as either prime or composite, and that all composite odd integers $< c$ appear as keys in $Q$.

Initialization sets $c \leftarrow 3$ and $Q \leftarrow \emptyset$.

At each step, the algorithm compares $c$ with the minimum key of $Q$, when $Q$ is nonempty. If $Q$ is empty, $c$ is prime. If $Q$ is nonempty, let $(x, d)$ be a record whose key $x$ is minimum in $Q$.

If $x < c$, then $x$ is the smallest composite not yet eliminated at its position. The record $(x, d)$ is removed from $Q$, replaced by $(x + d, d)$, and reinserted. This preserves the invariant that all multiples of the underlying prime are represented in increasing order.

If $x = c$, then $c$ is composite. The record $(x, d)$ is removed from $Q$, replaced by $(x + d, d)$, and reinserted. The value $c$ is not output as prime.

If $x > c$, then no composite has $c$ as a multiple of any previously discovered prime. In this case $c$ is prime. A new record $(c^2, 2c)$ is inserted into $Q$, since $c^2$ is the least odd multiple of $c$ not previously handled, and all smaller multiples of $c$ are either even or have smaller prime factors. The value $c$ is output as a prime.

After processing $c$, the next candidate becomes $c \leftarrow c + 2$.

Correctness follows from the invariant that every composite odd integer has a least prime factor $p$, and the record corresponding to $p$ generates exactly the sequence $p^2, p^2+2p, p^2+4p, \ldots$, so every such composite eventually appears as some key in $Q$. Conversely, only such generated multiples enter $Q$, so no non-composite is ever inserted.

Since each composite is generated exactly once per prime factorization through its least prime factor stream, and each prime is inserted exactly once, the queue size is bounded by the number of primes $\le \sqrt{N}$.

The algorithm outputs exactly the primes $< N$.

This completes the proof. ∎
