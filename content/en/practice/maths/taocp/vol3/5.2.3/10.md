**Exercise 10.** [*20*] [20] (J. W. J. Williams.) Step H4 of Algorithm H distinguishes between the three cases i < r, j = r, and j > r. Show that if Kk > Kk+1 it would be possible to simplify step H4 so that only a two-way branch is made. How could the condition K > K+1 be ensured throughout the heapsort process, by modifying step H2?

Let $S(t)$ denote the sibling-order property

$$
S(t):\quad K_{2t} \ge K_{2t+1}\qquad \text{for every internal node } t.
$$

We show that Step H4 can be reduced to a two-way branch provided $S(t)$ is maintained throughout the algorithm, and we then show how to maintain it by modifying both H2 and H4 locally.

## 1. Why the three-way test in H4 becomes unnecessary

Assume $S(t)$ holds for every internal node $t$. Then for any node $k$ with two children,

$$
K_{2k} \ge K_{2k+1},
$$

so the larger child of $k$ is always $2k$. Hence Step H4 never needs to inspect $K_{2k+1}$.

Thus H4 reduces to:

If $2k > r$, stop.

Otherwise compare only with the left child:

$$
\text{if } K_k \ge K_{2k} \text{ then stop, else interchange } K_k \leftrightarrow K_{2k}.
$$

This is a two-way branch.

## 2. Why the original invariant is not preserved and how to fix it

The issue identified in the review is correct: a swap in H4 can destroy $S(t)$ at ancestors of the swapped nodes. Therefore, $S(t)$ must be actively maintained whenever heap elements move.

The key observation is that violations of $S(t)$ can only occur at nodes whose children have just changed. In heapsort, this happens in only two situations:

1. During **H2** (initial heap construction), when a node is sifted down.
2. During **H4** (removing the maximum), when a key is moved down the heap along a path.

Thus it suffices to enforce $S(t)$ locally at the few nodes affected by these operations.

## 3. Modified Step H2

After the usual bottom-up heap construction (sift-down enforcing heap order), we add a local repair:

For every internal node $t$, after its sift-down is completed, ensure

$$
K_{2t} \ge K_{2t+1}
$$

by swapping the two children if necessary.

This does not affect heap order, since it only permutes siblings.

After H2, both properties hold:

- heap order: $K_t \ge K_{2t}, K_{2t+1}$,
- sibling order: $K_{2t} \ge K_{2t+1}$.

## 4. Why H2 alone is not sufficient and how H4 must be modified

During H4, we repeatedly exchange $K_k$ with a child (always $2k$ under the desired simplification). This changes values at nodes $k$ and $2k$, which affects sibling relations at:

- node $k$'s parent,
- node $2k$'s parent (after the swap, this node changes position).

Thus $S(t)$ may be violated at these two locations.

However, the fix is again purely local:

After every exchange $K_k \leftrightarrow K_{2k}$ in H4, perform the following constant-time repair steps:

1. **Repair at the new position of $k$:**

Let $k \leftarrow 2k$. If $k$ has a sibling, swap its children if needed so that $S(k)$ holds.
2. **Repair at the parent of $k$:**

Let $p = \lfloor k/2 \rfloor$. If $p$ is internal, enforce $S(p)$ by swapping $K_{2p}$ and $K_{2p+1}$ if necessary.

These are the only nodes whose sibling relations can be affected by the exchange.

Crucially, both fixes preserve heap order because they only swap siblings.

## 5. Preservation of invariants

After each operation in both H2 and H4:

- heap order is preserved because swaps never occur between parent and child except in the standard sift-down steps,
- sibling order $S(t)$ is restored immediately at every node whose children may have changed.

Hence $S(t)$ holds globally at all times.

## 6. Resulting simplification of H4

Since $S(t)$ is maintained throughout, every internal node satisfies

$$
K_{2t} \ge K_{2t+1}.
$$

Therefore H4 never needs to compare both children. The larger child is always $2k$, and the algorithm reduces to a two-way branch:

- compare $K_k$ with $K_{2k}$,
- otherwise stop or swap.

## Conclusion

The original idea is correct once the missing maintenance step is added: the sibling-order invariant must be enforced not only in H2 but also locally during every exchange in H4 at the affected nodes. With this repair, the invariant is preserved globally, and Step H4 can be simplified to a two-way branch as required.
