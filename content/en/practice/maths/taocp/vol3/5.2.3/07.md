Let the structure in Fig. 23 be a tournament tree used for repeated selection of the largest key. Each internal node stores the larger of its two children. On each output step, the root is output, replaced by $-\infty$, and this value is “repaired” downward by repeatedly moving the larger child upward. This is equivalent to deleting the current maximum and re-establishing the tournament invariant.

### 1. Which keys remain after 14 outputs

After each output, exactly one key is removed from the structure, namely the current maximum among the remaining keys. After 14 outputs from a set of 16 keys, exactly 2 original keys remain unremoved.

Since each step removes the largest remaining key, the two survivors must be the two smallest keys of the original set. From the data of the exercise (as in the previous correct identification within the tree of Fig. 23), these keys are

$$
61 \quad \text{and} \quad 87.
$$

Thus every other original key has been replaced by $-\infty$ somewhere in the structure.

### 2. Structural invariant of the tournament tree

At every node $v$, the stored value equals the maximum of all values in the subtree rooted at $v$, after replacements by $-\infty$.

Therefore, after all deletions:

- Any node whose subtree contains neither 61 nor 87 stores $-\infty$.
- Any node whose subtree contains 61 but not 87 stores $61$.
- Any node whose subtree contains 87 stores $87$, since $87 > 61$.

### 3. Tree configuration

Let $P_{87}$ be the unique root-to-leaf path in Fig. 23 that leads to the leaf position originally containing 87, and let $P_{61}$ be the corresponding path for 61.

Let $L$ be the least common ancestor of these two leaves.

Then the final configuration is determined exactly as follows:

#### (a) Nodes on the 87 path

Every node on $P_{87}$, from the leaf containing 87 up to the root, contains

$$
87.
$$

This includes the root itself, since 87 is the maximum of the two remaining keys.

#### (b) Nodes on the 61-only branch

Every node on $P_{61}$ that is strictly below $L$ (that is, in the portion of the tree whose subtree contains 61 but not 87) contains

$$
61.
$$

This is because all comparisons in those subtrees involve either 61 or $-\infty$, and the maximum is always 61 along that chain.

#### (c) All remaining nodes

Every node whose subtree contains neither remaining key contains

$$
-\infty.
$$

### 4. Final structure description

The final tree consists of:

- A single root-to-leaf chain carrying the value 87 (the maximal surviving key).
- A second root-to-leaf chain carrying the value 61 down to its original leaf position, branching off at the least common ancestor of the two surviving leaves.
- All other nodes, which no longer lie on a path containing a surviving key, contain $-\infty$.

Equivalently, the tree is obtained by marking exactly those nodes that lie on the union of the two original root-to-leaf paths to 61 and 87, assigning 87 to every node whose subtree contains 87, assigning 61 to nodes whose subtree contains 61 but not 87, and assigning $-\infty$ everywhere else.

This completely specifies the configuration of the tree after 14 outputs. ∎
