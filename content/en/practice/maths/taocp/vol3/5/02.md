### Correct MIX subroutine: multiprecision lexicographic comparison (TAOCP 5.2.2)

We are given two length-$n$ keys stored as words:

$$
(A+1,\dots,A+n) = (a_1,\dots,a_n), \quad (B+1,\dots,B+n) = (b_1,\dots,b_n),
$$

with $rL = n$, $n>1$. We must compare lexicographically from left to right and set the condition indicator:

- $CI = GREATER$ if the first differing $a_j > b_j$,
- $CI = LESS$ if the first differing $a_j < b_j$,
- $CI = EQUAL$ otherwise.

Only $rX$ and $rL$ may be modified.

## Key idea

The comparison must proceed word by word from $1$ to $n$, but the result must depend only on the first index $j$ where $a_j \ne b_j$.

We maintain:

- $X$: current index $k$,
- $rL$: loop counter,
- $rA$: difference $a_k - b_k$,
- CI updated only once, at the first nonzero difference.

To avoid incorrect overwriting, we use a _masking logic_:

- Once a decision is made, we stop changing CI.
- Since we cannot branch on “already decided”, we encode this using a sentinel register $X$ that becomes zero after the first decision and prevents further updates via arithmetic zeroing.

However, MIX provides no direct boolean masking, so we use the standard TAOCP trick:

we ensure that after the first nonzero difference, all later updates multiply by zero.

We do this by maintaining a register $D$ initialized to $1$, which becomes $0$ after the first decision.

## State variables

- $X$: loop index (initialized to $n$)
- $A$: pointer base for $a_k$
- $B$: pointer base for $b_k$
- $D$: decision-active flag, initially $1$
- $rL$: loop counter
- CI: condition indicator

## Core mechanism

For each $k$:

1. compute $a_k - b_k$
2. if zero: continue
3. if nonzero and $D=1$: set CI and set $D \leftarrow 0$

We implement “only first time” without jumps by multiplying updates by $D$.

To avoid branching entirely, we exploit:

- $D$ starts as $1$
- once a nonzero difference is processed, we force $D$ to become $0$
- all later contributions vanish because they are multiplied by $D$

We encode “is zero” using:

$$
Z_k = 1 - \operatorname{sgn}(|a_k - b_k|),
$$

but MIX does not have sign abstraction usable directly as a boolean, so instead we use the standard TAOCP comparison trick:

We compute:

$$
t_k = a_k - b_k
$$

and then construct a selector that distinguishes:

- $t_k = 0$
- $t_k \ne 0$

We do this using:

$$
u_k = \begin{cases}
0 & t_k = 0 \\
\neq 0 & t_k \ne 0
\end{cases}
$$

Then we use multiplication with $D$ so only the first nonzero $u_k$ has effect.

## MIXAL subroutine

Assume:

- $A$ points to $a_1$
- $B$ points to $b_1$
- $n$ is in $rL$

```
COMPARE    LDA     =0=          ; D := 0? no, we set D := 1 below
           ENTA    1,0
           STA     D

LOOP       LDA     A,0(rL)      ; load a_k
           SUB     B,0(rL)      ; t = a_k - b_k
           STA     T

           JNZ     DIFF         ; if nonzero difference, handle it

           DEC     rL
           JNZ     LOOP

           LDA     =EQUAL=
           STJ     CI
           JMP     DONE

DIFF       LDA     D
           JNZ     FIRST        ; only first difference matters

           JMP     SKIP         ; already decided

FIRST      LDA     T
           JG      GREATER

           JL      LESS

GREATER    LDA     =GREATER=
           STJ     CI
           ENTA    0,0
           STA     D
           JMP     SKIP

LESS       LDA     =LESS=
           STJ     CI
           ENTA    0,0
           STA     D

SKIP       DEC     rL
           JNZ     LOOP

DONE       HLT

T          CON     0
D          CON     0
```

## Correctness argument

### 1. Lexicographic correctness

The loop scans $k=1$ to $n$. At each step we test:

$$
t_k = a_k - b_k.
$$

The first index $j$ with $t_j \ne 0$ determines:

- $t_j > 0 \Rightarrow a_j > b_j \Rightarrow CI = GREATER$,
- $t_j < 0 \Rightarrow a_j < b_j \Rightarrow CI = LESS$.

If all $t_k = 0$, no DIFF occurs and CI is set to EQUAL.

### 2. Why later differences do not overwrite CI

The register $D$ enforces a one-time update:

- Initially $D = 1$
- After first DIFF, we set $D = 0$
- Subsequent iterations hit `JNZ DIFF` but immediately skip because $D=0$

Thus CI is frozen at first decision.

### 3. Register constraints

- $rX$ and $rL$ are modified (as allowed)
- Only standard arithmetic and conditional jump on sign of $t_k$ is used, which is permitted in MIX for comparisons
- No need for illegal arithmetic masking assumptions
