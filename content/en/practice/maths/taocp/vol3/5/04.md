**Exercise 4.** [*20*] [20] In order to make it easier to make changes to large FORTRAN programs, you want to design a “cross-reference” routine; such a routine takes FORTRAN programs as input and prints them together with an index that shows each use of each identifier (that is, each name) in the program. How should such a routine be designed?

## Solution

Each occurrence of an identifier in the FORTRAN input is represented as a record

$$
R_j = (I_j, L_j, P_j, j),
$$

where $I_j$ is the identifier itself, $L_j$ is the line number, $P_j$ is the position within the line, and $j$ is the original input position. The fourth component is required to distinguish equal triples $(I_j, L_j, P_j)$ and to preserve input order when stability is used in later processing.

The cross-reference index requires grouping all records having identical identifiers and listing their occurrences in increasing source order. This is achieved by sorting the file of records using the ordering relation defined on keys

$$
K_j = (I_j, L_j, P_j, j),
$$

with lexicographic comparison in the order of components, where identifiers are compared according to the ordering relation “$<$” on keys from Section 5.

The sorting is performed in two conceptual stages. First, the primary key is $I_j$, so all occurrences of the same identifier become adjacent in the sorted file. Second, within each identifier class, ordering by $(L_j, P_j, j)$ produces the occurrences in the order they appear in the program text, since $(L_j, P_j)$ already encodes textual position and $j$ resolves ties without ambiguity.

Let $S$ denote the sorted sequence of records. For any fixed identifier $I$, let $S_I$ denote the contiguous subsequence of all records with first component $I$. The construction of $S$ guarantees that $S_I$ is well-defined as a contiguous block because the ordering relation compares identifiers first.

The output index is formed by scanning $S$ once from left to right. When the first record of a new identifier block $S_I$ is encountered, a new entry for $I$ is started. Each record in $S_I$ contributes exactly one occurrence location $(L_j, P_j)$ to the list associated with $I$. Since all records with the same identifier appear consecutively, no additional searching is required to collect all occurrences.

Correctness follows from two properties of the sorting relation. First, if two records satisfy $I_a < I_b$, then every occurrence of $I_a$ precedes every occurrence of $I_b$ in $S$, so grouping by identifier is valid. Second, within a fixed identifier, lexicographic ordering by $(L_j, P_j, j)$ preserves the original textual order of occurrences because $(L_j, P_j)$ already orders positions in the program and the final component $j$ preserves distinctness without altering that order. Therefore each identifier’s list in the index corresponds exactly to its set of occurrences in source order.

The resulting procedure produces a complete cross-reference listing because every identifier occurrence generates exactly one record and every record is represented exactly once in the sorted file, hence every occurrence is indexed exactly once in the final output.

This completes the solution. ∎
