**Exercise 1.** [**] [15] Mr. B. C. Dull (a MIX programmer) wanted to know if the number stored in location A is greater than, less than, or equal to the number stored in location B. So he wrote `LDA A; SUB B` and tested whether register A was positive, negative, or zero. What serious mistake did he make, and what should he have done instead?

**Verified:** yes  
**Solve time:** 58s  

---

After `LDA A; SUB B`, register A contains the word $A - B$, and Mr. Dull infers the relation between $A$ and $B$ by inspecting its sign. This inference is invalid in MIX arithmetic because subtraction is performed in fixed-word representation, so overflow can occur when $A - B$ is not representable; in that case the sign of the stored result does not reflect the true ordering of the mathematical values of $A$ and $B$. The original value of $A$ is also destroyed, so no reliable comparison state remains.

The correct method is to use the built-in comparison operation, which evaluates $A$ against $B$ without forming their difference. One should write `CMPA B`, then branch according to the resulting condition code (less, equal, or greater). This avoids overflow and preserves both operands. ∎
