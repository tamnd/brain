[Section 5.2: Internal Sorting](../)

**Exercise 1.** [**] [15] Would Algorithm C still work if _i_ varies from 2 up to _N_ in step C2, instead of from _N_ down to 2? What if _j_ varies from 1 up to _i_ − 1 in step C3?

**Verified:** yes  
**Solve time:** 59s  

---

Yes. Algorithm C still works if $i$ varies from $2$ up to $N$ in step C2. The correctness argument depends only on the fact that each unordered pair of indices ${i,j}$ with $i\ne j$ is compared exactly once. For every such pair, step C4 increases the count of the larger key by $1$. Hence, after all comparisons have been performed, $\mathrm{COUNT}[k]$ equals the number of keys less than $K_k$, regardless of the order in which the pairs are processed. Reversing the order of the values of $i$ changes only the sequence of comparisons, not the set of comparisons.

The same reasoning shows that Algorithm C still works if $j$ varies from $1$ up to $i-1$ in step C3. For each fixed $i$, the comparisons $(i,1),(i,2),\ldots,(i,i-1)$ are performed instead of the reverse sequence. Every pair $(i,j)$ with $1\le j<i\le N$ is still compared exactly once, and step C4 contributes the same increment to the same counter for each pair. Therefore the final values of $\mathrm{COUNT}[1],\ldots,\mathrm{COUNT}[N]$ are unchanged, and the algorithm remains correct. $\boxed{\text{Both modifications preserve correctness.}}$
