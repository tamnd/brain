**Exercise 4.** [**] [16] Write a MIX program that “finishes” the sorting begun by Program C; your program should transfer the keys to locations OUTPUT+1 through OUTPUT+N, in ascending order. How much time does your program require?

The correct finishing step must follow the actual invariant of Program C in TAOCP counting sort.

After Program C terminates, the array `COUNT` satisfies:

$$
\text{COUNT}[k] = \#\{\text{keys } \le k\}.
$$

This is indexed by key value $k$, not by record index $i$. The output position of an element is therefore determined by its key.

## Correct reconstruction of the finishing step

Each input record is a key $K_i$. For each key we must:

1. Use $COUNT[K_i]$ to determine the current final position.
2. Decrement $COUNT[K_i]$ so duplicates go to distinct positions.
3. Store the key into the output array at that position.

Formally, each occurrence of key $k$ is placed at:

$$
COUNT[k] \leftarrow COUNT[k] - 1,\quad
\text{OUTPUT}[COUNT[k]] \leftarrow k.
$$

This ordering is essential. The decrement must happen before storing.

## MIX program

Assume:

- `INPUT+1 ... INPUT+N` contains the keys.
- `COUNT` is indexed by key value.
- `OUTPUT+1 ... OUTPUT+N` is the result array.
- Registers: $X_1$ loop index $i$, $X_2$ key $k$, $X_3$ position $j$, $X_4$ temporary storage for key.

```
        ENT1    N

LOOP    LDA     INPUT,1      A ← INPUT[i]
        STA     X4           save key k in X4

        LDX     X4,0         X2 ← k

        LDA     COUNT,X2     A ← COUNT[k]
        SUB     =1=          A ← COUNT[k] - 1
        STA     COUNT,X2     COUNT[k] ← COUNT[k] - 1
        STA     X3           j ← new COUNT[k]

        LDA     X4           reload key k
        STA     OUTPUT,3     OUTPUT[j] ← k

        DEC1    1
        J1P     LOOP
        HLT
```

### Addressing explanation

- `INPUT,1` means $INPUT[i]$
- `COUNT,X2` means $COUNT[k]$
- `OUTPUT,X3` means $OUTPUT[j]$

This is standard MIX indexed addressing as used in TAOCP.

## Correctness argument

At the start of each iteration, `COUNT[k]` equals the number of keys $\le k$ remaining to be placed.

When we execute:

$$
COUNT[k] \leftarrow COUNT[k] - 1,
$$

we obtain the correct free slot for the next occurrence of $k$. Storing into `OUTPUT[COUNT[k]]` places the key in its final sorted position.

Distinct occurrences of the same key receive distinct positions because `COUNT[k]` is strictly decreased after each placement.

All keys are processed exactly once, so every input element is placed exactly once in sorted order.

Thus `OUTPUT+1 ... OUTPUT+N` contains the keys in nondecreasing order.

## Time analysis

Each iteration performs a constant number of MIX instructions:

- one load of `INPUT[i]`
- a constant number of arithmetic and indexed memory operations
- one store to `OUTPUT`
- loop control

Therefore the total running time is:

$$
T(N) = \Theta(N).
$$

This correctly implements the finishing phase of Program C in MIX and fixes the indexing and update errors in the original solution.
