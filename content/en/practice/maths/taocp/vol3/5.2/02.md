[Section 5.2: Internal Sorting](../)

**Exercise 2.** [**] [21] Show that Algorithm C works properly when equal keys are present. If $K_i = K_j$ and $j < i$, does $R_i$ come before or after $R_j$ in the final ordering?

**Verified:** yes  
**Solve time:** 3m08s  

---

Let $A_i = \{k < i : K_k \le K_i\}$, so that

$$
\mathrm{COUNT}[i] = |A_i|.
$$

Similarly,

$$
A_j = \{k < j : K_k \le K_j\}, \quad \mathrm{COUNT}[j] = |A_j|.
$$

Fix indices $j < i$ with $K_i = K_j$. We compare $A_i$ and $A_j$ directly.

Split the set defining $A_i$ into two disjoint parts:

$$
A_i = \{k < j : K_k \le K_i\} \;\cup\; \{j \le k < i : K_k \le K_i\}.
$$

Because $K_i = K_j$, for every $k < j$,

$$
K_k \le K_i \iff K_k \le K_j,
$$

so

$$
\{k < j : K_k \le K_i\} = \{k < j : K_k \le K_j\} = A_j.
$$

Now consider the second part $\{j \le k < i : K_k \le K_i\}$. This set contains at least the index $k = j$, since $j < i$ and $K_j = K_i$, hence $K_j \le K_i$. Therefore,

$$
j \in A_i \quad \text{but} \quad j \notin A_j.
$$

All other elements in $\{j < k < i : K_k \le K_i\}$ are additional possible contributions, but the single element $j$ already guarantees a strict increase.

Thus,

$$
A_j \subset A_i \quad \text{and} \quad A_i \neq A_j,
$$

so

$$
\mathrm{COUNT}[i] = |A_i| \ge |A_j| + 1 = \mathrm{COUNT}[j] + 1.
$$

Hence,

$$
\mathrm{COUNT}[i] > \mathrm{COUNT}[j].
$$

Since Algorithm C places each record $R_i$ in position $\mathrm{COUNT}[i] + 1$, it follows that

$$
\mathrm{COUNT}[i] + 1 > \mathrm{COUNT}[j] + 1,
$$

so $R_i$ is placed strictly after $R_j$.

Therefore, when $K_i = K_j$ and $j < i$, Algorithm C preserves stability: $R_j$ precedes $R_i$ in the final ordering, and equal keys appear in increasing index order.
