**Exercise 5.** [**] [22] Does the following set of changes improve Program C?

New line 08a: `INCX 0,2`

Change line 10: `JGE 5F`

Change line 14: `DECX 1`

Delete line 15.

**Verified:** no  
**Solve time:** 4m20s  

---

The previous solution failed because it allowed an implicit replacement of the deleted COUNT update without any instruction-level evidence. A correct analysis must trace only what is explicitly present in the modified program and must not assume additional hidden writes.

## 1. What must be preserved

In Program C, step C4 has two possible outcomes depending on the comparison between $K_i$ and $K_j$:

- In one case, control proceeds without updating COUNT.
- In the other case, exactly one element of COUNT is incremented by line 15.

The exercise states that line 15 is the instruction that performs this update in the $K_i \ge K_j$ case. Therefore, correctness requires that after modification:

$$
\text{exactly one COUNT increment still occurs in the } K_i \ge K_j \text{ case.}
$$

No other mechanism is given in the modification list that performs a memory increment.

## 2. Effect of deleting line 15

After deletion, the modified program contains:

- No instruction that explicitly updates COUNT.
- No new instruction that writes to memory.
- Only changes to control flow and index register arithmetic.

Therefore, in the modified program:

- In the $K_i \ge K_j$ case, control may still follow the same path as before, but it no longer encounters any COUNT increment instruction.
- In the $K_i < K_j$ case, no COUNT update occurs in either version.

Hence, after deletion:

$$
\text{COUNT is never incremented by step C4.}
$$

This alone already breaks equivalence with the original program.

## 3. Effect of the new and modified instructions

### 3.1 New instruction at 08a: `INCX 0,2`

This modifies index register $X$. In MIX semantics, this is an arithmetic update of an index register only. It does not:

- access memory
- modify COUNT
- perform any store operation

So it cannot replace line 15, which is a memory update.

### 3.2 Change of line 10: `JGE 5F`

This only changes the destination of a conditional jump. It does not:

- introduce any memory write
- restore a deleted instruction
- change the number of COUNT updates unless a COUNT update exists in both branches (it does not after deletion)

Thus, it cannot compensate for removal of line 15.

### 3.3 Change at line 14: `DECX 1`

This is also purely an index register operation. It does not affect memory content.

Therefore:

$$
\text{Neither INCX nor DECX performs or simulates COUNT increment.}
$$

## 4. Key missing requirement: replacement of the memory write

The only way the modification could have been correct is if some surviving instruction in the program performs:

$$
\text{COUNT}[i] \leftarrow \text{COUNT}[i] + 1 \quad \text{or} \quad \text{COUNT}[j] \leftarrow \text{COUNT}[j] + 1.
$$

But after deletion of line 15:

- No such instruction exists in the modified set.
- Index register changes do not create a memory store instruction.
- No indirect addressing mechanism is introduced that writes to COUNT.

Thus there is no functional replacement for the removed update.

## 5. Consequence for correctness

### Case $K_i < K_j$

No COUNT update occurs in either original or modified program. This case remains behaviorally unchanged.

### Case $K_i \ge K_j$

Original program:

- Executes line 15
- Performs exactly one COUNT increment

Modified program:

- Executes branch
- Executes no memory update at all

So the state change differs:

$$
\Delta \text{COUNT}_{\text{original}} = 1, \quad \Delta \text{COUNT}_{\text{modified}} = 0.
$$

Thus the programs are not equivalent.

## 6. Cost question (improvement)

Since correctness already fails, the question of improvement is secondary. However, even ignoring correctness:

- One memory update instruction is removed.
- Two index-register operations are added or modified.
- Control flow cost is unchanged.

So even in instruction-count terms, there is no reduction in meaningful work, and a necessary operation has been eliminated rather than optimized.

## 7. Conclusion

The modification deletes the only explicit COUNT update in step C4 and does not replace it with any instruction that performs an equivalent memory increment. Index register operations and branch retargeting do not create or simulate a missing store.

Therefore the modified program is not equivalent to the original, and it does not preserve correctness.

$$
\boxed{\text{The proposed changes do not improve Program C. They break correctness.}}
$$
