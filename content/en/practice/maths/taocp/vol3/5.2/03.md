[Section 5.2: Internal Sorting](../)

**Exercise 3.** [**] [21] Would Algorithm C still work properly if the test in step C4 were changed from “$K_i < K_j$” to “$K_i \leq K_j$”?

**Verified:** yes  
**Solve time:** 1m33s  

---

No. With the original test, when $K_i=K_j$ and $j<i$, step C4 takes the “otherwise” branch, increasing $\mathrm{COUNT}[i]$. Thus among equal keys the later record receives the larger count, and the final ordering is stable: $R_j$ precedes $R_i$.

If the test is changed to $K_i\le K_j$, then for $K_i=K_j$ the algorithm increases $\mathrm{COUNT}[j]$ instead. For every pair of equal keys with $j<i$, the earlier record receives the larger count. Consequently, if several records have the same key, their relative order is reversed. For example, with two equal keys $K_1=K_2$, the modified algorithm yields $\mathrm{COUNT}[1]=1$ and $\mathrm{COUNT}[2]=0$, so $R_2$ is placed before $R_1$.

The counts are still all distinct, and for unequal keys the same record receives the increment as before. Hence the records are still arranged in nondecreasing order of key; only the treatment of equal keys changes. Therefore Algorithm C still sorts correctly, but it is no longer stable. Equal keys appear in the reverse of their original order. $\boxed{\text{Yes, but the order of equal keys is reversed.}}$
