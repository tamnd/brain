**Exercise 10.** [*22*] [22] If $K_i > K_j$, when we begin step D3, Algorithm D specifies a lot of actions that accomplish nothing. Show how to modify Program D so that this redundant computation can be avoided, and discuss the merits of such a modification.

## Solution

Let $h = h_s$ be fixed during a pass of Algorithm D. In step D3 the algorithm initializes

$i = j - h,\quad K = K_j,\quad R = R_j.$

If $K_i > K_j$, then step D4 fails immediately, and the control enters step D5, which performs a sequence of moves before any further comparison with other elements of the $h$-ordered subsequence. The initialization of $K$ and $R$ together with the immediate comparison therefore does not contribute to the final placement of $R_j$ in the case where $R_j$ must move at least one position.

The redundancy can be removed by first testing whether $R_j$ already lies in its correct position with respect to the previous element in its $h$-chain.

### Modified Algorithm D

Insert a new preliminary step between D2 and D3.

**D2′. [Early test.]**

Set $i = j - h$. If $K_j \ge K_i$, then go to step D6.

Then modify the loop of D2 so that steps D3–D6 are executed only when $K_j < K_{j-h}$.

After this change, steps D3–D6 remain unchanged.

### Correctness of the modification

If $K_j \ge K_{j-h}$, then the pair $(R_{j-h}, R_j)$ already satisfies the required order condition for the current $h$-sorted file, since the elements at distance $h$ are locally ordered at position $j$. In this situation, the inner insertion loop of steps D3–D6 would perform no shifts that affect positions before $j$, and $R_j$ remains at index $j$.

If $K_j < K_{j-h}$, the condition in D2′ fails, and the algorithm proceeds to D3. In this case the original procedure is required, since at least one comparison and one move are needed to insert $R_j$ into the $h$-ordered prefix.

Thus every execution path of the modified algorithm produces the same final ordering as Algorithm D.

### Effect on the computation

For each index $j$ in a fixed $h$-pass, the original algorithm always performs step D3 and executes at least one comparison in D4. In the modified version, the case $K_j \ge K_{j-h}$ avoids all actions in D3–D6 entirely, since no assignments $K \leftarrow K_j$, $R \leftarrow R_j$, no inner-loop initialization, and no move operations are executed.

The modification replaces the unconditional entry into the inner insertion procedure by a single comparison with the preceding element in the $h$-chain. This removes all work associated with D3–D5 whenever the current element is already in correct local order.

### Merits

The improvement eliminates a full inner insertion pass whenever the element is locally ordered with respect to its immediate predecessor in the $h$-subsequence. The cost of the modification is one additional comparison per $j$ in D2′, while saving the initialization work of D3 together with all subsequent comparisons and moves in D4–D5 in the successful cases.

The asymptotic behavior of shellsort remains unchanged, since the number of passes and inversion structure are unaffected. The gain appears as a reduction in constant factors, most significant when the file becomes nearly $h$-ordered during later passes, where many indices satisfy $K_j \ge K_{j-h}$.
