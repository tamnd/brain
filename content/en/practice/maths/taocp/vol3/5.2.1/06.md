**Exercise 6.** [*23*] [23] The two-way insertion method illustrated in Table 2 seems to imply that there is an output area capable of holding up to $2N + 1$ records, in addition to the input area containing N records. Show that two-way insertion can be done using only enough space for $N + 1$ records, including both input and output.

The original attempt fails because it conflates output storage with unread input storage. A correct solution must explicitly maintain access to the remaining input while two output sequences grow in opposite directions.

We now construct a valid in-place simulation of two-way insertion using exactly $N+1$ cells.

## Correct model and storage layout

Let the array be $A[1 \ldots N+1]$. Initially:

- The input records occupy $A[1], A[2], \ldots, A[N]$,
- $A[N+1]$ is unused.

We introduce three pointers:

- $i$: the next unread input position,
- $L$: the last position used by the left-growing output,
- $R$: the first position used by the right-growing output.

Initially:

$$
i = 1, \quad L = 0, \quad R = N+1.
$$

We interpret:

- Unread input is in $A[i \ldots N]$,
- Left output occupies $A[1 \ldots L]$,
- Right output occupies $A[R \ldots N+1]$.

The key invariant is:

$$
L < i < R.
$$

This ensures:

- outputs never overwrite unread input,
- unread input is always accessible,
- the two output regions remain disjoint.

## Step of two-way insertion

At each step, we process the current input record $A[i]$ according to the two-way insertion rule (same comparison criterion as in the standard algorithm). Then:

### Case 1: record goes to the left output

We increment $L$ and store the record in $A[L]$, then advance $i$:

$$
L \leftarrow L + 1, \quad i \leftarrow i + 1.
$$

### Case 2: record goes to the right output

We decrement $R$ and store the record in $A[R]$, then advance $i$:

$$
R \leftarrow R - 1, \quad i \leftarrow i + 1.
$$

## Why overwriting input is safe

A potential problem is that a write into $A[L]$ or $A[R]$ might destroy unread input. The invariant $L < i < R$ prevents this:

- Left writes always occur at $L+1 \le i-1$,
- Right writes always occur at $R \ge i+1$.

Thus, neither side writes into unread input.

The only remaining issue is maintaining the invariant when $L+1 = i$ or $i = R$, i.e., when unread input becomes adjacent to an output block.

## Restoring separation when boundaries meet

If $L+1 = i$, the next left insertion would collide with unread input. Similarly, if $i = R$, a right insertion would collide.

In either case, we restore spacing using the single free cell guaranteed by $N+1$ total capacity.

Assume $L+1 = i$. Then there is exactly one free cell at position $R-1$ (since $L < i < R$). We shift the unread input block one position to the right:

For

$$
k = R-1, R-2, \ldots, i
$$

do:

$$
A[k+1] \leftarrow A[k].
$$

Then set:

$$
i \leftarrow i+1, \quad R \leftarrow R+1.
$$

This restores:

$$
L < i < R.
$$

The symmetric operation applies if $i = R$, shifting the unread input left using the free cell at $L+1$.

Each such shift occurs only when the gap closes, and it uses the single available free cell created by the $N+1$ capacity.

## Space bound

At all times, the array contains:

- $N$ data records total (distributed among unread input and outputs),
- exactly one unused cell guaranteed by the invariant $L < i < R$.

Thus the storage requirement is:

$$
N + 1 \text{ cells}.
$$

No separate input buffer is needed beyond the array itself, and no output area ever exceeds the available space because input is always preserved by enforcing the separation invariant.

## Correctness

1. Every input record is read exactly once via $i$.
2. Each record is placed exactly once into either the left or right output, preserving the two-way insertion rule.
3. Unread input is never overwritten due to $L < i < R$.
4. The separation invariant is always restorable using the single spare cell.
5. Therefore the simulation is faithful to the original two-way insertion method.

## Conclusion

Two-way insertion can be implemented in-place with a single array of size $N+1$ by maintaining a read pointer and two output boundaries, together with a dynamic separation invariant that is restored using the single spare cell whenever necessary. This correctly accounts for unread input while supporting both growing output sequences.
