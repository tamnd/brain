[Section 5.2.1: Sorting by Insertion](../)

**Exercise 2.** [**] [11] Would Algorithm S still sort numbers correctly if the relation “kK > Kk,” in step S3 were replaced by “Kk > k;”?

**Verified:** yes  
**Solve time:** 1m21s  

---

The comparison in step S3 determines the direction of the invariant maintained during insertion. In the original algorithm, the test “$K < K_i$” (equivalently the given “$K_i > K$”) ensures that all elements larger than the current key are shifted one position to the right, so that after termination of the inner loop every index $i$ to the left of the insertion position satisfies $K_i \le K$.

If the relation is replaced by its reversal, the shifting condition becomes that elements are moved precisely when $K_i < K$. In that case the inner loop moves all elements strictly smaller than the key to the right, and the insertion position is the first index $i$ such that $K_i \ge K$. After completion of the pass for index $j$, the segment $K_1,\dots,K_j$ satisfies $K_1 \ge K_2 \ge \cdots \ge K_j$.

Thus each prefix is arranged in nonincreasing order, and the final array satisfies $K_1 \ge K_2 \ge \cdots \ge K_N$. This is a correct ordering with respect to the reversed ordering relation, but it does not produce the ascending order $K_1 \le \cdots \le K_N$ required by Algorithm S. Therefore the modified algorithm does not sort correctly in the sense specified. ∎
