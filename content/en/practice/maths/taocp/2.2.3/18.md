**Exercise 18.** [*20*] What, if anything, is the significance of the values of `QLINK[0]`, `QLINK[1]`, ..., `QLINK[n]` when Algorithm `T` terminates?

**Verified:** no  
**Solve time:** 36s  

## Solution

Algorithm `T` is the topological sorting algorithm of Section 2.2.3. Its input is a directed graph represented by successor lists, where `QLINK` serves simultaneously as a queue mechanism and as temporary storage for the counts of predecessor nodes.

At the beginning of the algorithm, `QLINK[k]` records the number of immediate predecessors of node $k$, for $1 \le k \le n$. During execution, this information is repeatedly decreased. Whenever the count for node $k$ becomes zero, node $k$ is inserted into the queue of eligible vertices by assigning `QLINK[R] \leftarrow k`, where `R` is the current rear of the queue. Thus `QLINK` ceases to represent predecessor counts and becomes the linking structure of the queue.

When Algorithm `T` terminates successfully, every node has entered and left the queue exactly once. The output sequence is therefore a topological ordering of all $n$ nodes. Let this ordering be

$$
x_{i_1}, x_{i_2}, \ldots, x_{i_n}.
$$

During the final stage of the computation, the queue consists of a chain

$$
0 \to i_1 \to i_2 \to \cdots \to i_n \to 0,
$$

where `0` acts as the queue header. Since nodes are removed from the front but never disconnected, the links remain intact after termination.

Consequently, when Algorithm `T` terminates successfully:

- `QLINK[0]` contains the first element of the topological ordering;
- `QLINK[i_j] = i_{j+1}` for $1 \le j < n$;
- `QLINK[i_n] = 0$.

Equivalently, the array `QLINK[0], QLINK[1], \ldots, QLINK[n]` contains a linked-list representation of the topological ordering produced by the algorithm.

If Algorithm `T` terminates unsuccessfully because a cycle is present, some nodes never attain predecessor count zero. Their `QLINK` entries retain positive values, namely the number of predecessors remaining after all removable vertices have been deleted. These are precisely the vertices belonging to, or dependent upon, directed cycles.

Thus the final values of `QLINK` have a definite significance: in the acyclic case they encode the output ordering as a linked list, while in the cyclic case they identify the unresolved dependency structure.

$$
\boxed{\text{`QLINK' stores the output order as a linked list, unless a cycle prevents completion.}}
$$
