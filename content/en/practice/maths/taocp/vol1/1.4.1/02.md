**Exercise 2.** [*10*] Suggest code to substitute for (6) without using the `JSJ` instruction.

The correct replacement must reproduce the _defining semantics of_ `JSJ X` in MIX:

$$
\texttt{JSJ X: } J \leftarrow \text{address of next instruction}, \quad \text{then } \text{go to } X.
$$

Thus, any correct substitute must explicitly:

1. compute or encode the return address (next instruction),
2. place it into the J-register,
3. transfer control to `X`.

## Correct replacement of instruction (6)

Let the instruction (6) be located at address $k$, so that the next instruction has address $k+1$.

Then `JSJ MAXN` is replaced by:

```
        ENJ k+1
        JMP MAXN
```

## Correctness argument

### 1. Correct reconstruction of the J-register effect

In MIX, `ENJ v` loads the J-register with the constant value $v$. Therefore:

```
ENJ k+1
```

sets

$$
J \leftarrow k+1,
$$

which is exactly the return address that `JSJ MAXN` would have stored, since `JSJ` always stores the address of the next instruction.

This directly fixes the critical misunderstanding in the proposed solution: the return address is not derived from any preexisting J-register value, but is determined by the call site. The replacement explicitly encodes that value.

### 2. Correct control transfer

The instruction

```
JMP MAXN
```

transfers control to `MAXN`, matching the second step of `JSJ MAXN`. Since MIX executes instructions sequentially, no intermediate side effects occur between setting `J` and jumping.

Thus the control flow matches exactly:

- original: set $J \leftarrow k+1$, then jump to `MAXN`,
- replacement: explicitly performs the same two operations in sequence.

### 3. Preservation of return mechanism

The subroutine `MAXN` relies on the J-register to return via `JP EXIT` or equivalent linkage code.

Since the replacement ensures:

$$
J = k+1
$$

at entry, all subsequent return logic behaves identically to the original `JSJ`-based call.

No global or memory-based linkage state is required, and repeated or nested calls remain valid because each call site independently sets the correct return address.

## Conclusion

The correct substitution for `JSJ MAXN` is:

```
        ENJ k+1
        JMP MAXN
```

where $k+1$ is the address of the instruction following the original `JSJ`.

This exactly reproduces both components of `JSJ`: setting the return address in the J-register and transferring control to the subroutine.
