**Exercise 6.** [*26*] Suppose MIX did not have a `MOVE` operator. Write a subroutine entitled MOVE such that the calling sequence `JMP MOVE; NOP A,I(F)` has an effect just the same as `MOVE A,I(F)` if the latter were admissible. The only differences should be the effect on register `J` and the fact that a subroutine naturally consumes more time and space than a hardware instruction does.

**Verified:** no  
**Solve time:** 4m26s  

---

The original attempt fails because it tries to update memory-resident variables with `INCX`/`DECX`, which in MIX affect only register $X$. The corrected construction must therefore represent all evolving state (source pointer, destination pointer, and loop counter) in registers or index registers and avoid any “increment memory cell” idiom.

A correct solution uses index registers as pointers and register $X$ as the loop counter.

## Correct solution

We implement a subroutine `MOVE` so that

$$
\texttt{JMP MOVE; NOP A,I(F)}
$$

has the same effect as the hypothetical instruction `MOVE A,I(F)`:

$$
M(D+k) \leftarrow M(S+k), \quad 0 \le k \le F-1,
$$

where

$$
S = A, \quad
D =
\begin{cases}
A & I = 0, \\
rI_I & 1 \le I \le 6.
\end{cases}
$$

## 1. Save return address and fetch instruction word

After

```
MOVE   STJ SAVE
```

the register $J$ contains the address of the next instruction (`NOP A,I(F)`), so we store it.

```
       STJ  SAVE
       LDA  0,SAVE(0:5)
       STA  INST
```

Now `INST` holds the full calling instruction word.

## 2. Extract fields $A$, $I$, $F$

Using MIX field selection:

```
       LDA  INST(0:2)
       STA  AVAL

       LDA  INST(3:3)
       STA  IVAL

       LDA  INST(4:5)
       STA  CNT
```

Thus:

- `AVAL = A`
- `IVAL = I`
- `CNT = F`

## 3. Initialize pointers in index registers

We use:

- $rI_1$ as source pointer
- $rI_2$ as destination pointer

### Source pointer

```
       LDA  AVAL
       STA  0,1
```

So $rI_1 \leftarrow A$.

### Destination pointer

If $I = 0$, destination is $A$. Otherwise it is $rI_I$.

```
       LDX  IVAL
       JNZ  NONZERO

       LDA  AVAL
       STA  0,2
       JMP  INITCNT
```

For $I \ne 0$:

```
NONZERO LDX  IVAL
        STX  0,2
```

Thus $rI_2$ is correctly initialized.

## 4. Initialize loop counter

```
INITCNT LDX  CNT
```

Now $X \leftarrow F$.

If $F = 0$, we terminate immediately:

```
        JXZ  DONE
```

## 5. Block transfer loop

We now copy word by word using index registers.

```
LOOP    LDA  0,1
        STA  0,2

        INC1 0
        INC2 0

        DECX 0
        JNZ  LOOP
```

Explanation:

- `0,1` addresses $M(rI_1)$
- `0,2` addresses $M(rI_2)$
- `INC1 0` increments index register 1
- `INC2 0` increments index register 2
- `DECX 0` decrements register $X$

Each iteration performs:

$$
M(rI_2) \leftarrow M(rI_1), \quad rI_1 \leftarrow rI_1+1, \quad rI_2 \leftarrow rI_2+1.
$$

## 6. Return

```
DONE    JMP SAVE
```

This restores control to the instruction following the call.

## Correctness justification

### 1. No illegal memory increments

All pointer updates use:

- `INC1`, `INC2` for index registers
- `DECX` for loop control

No memory cell is incorrectly treated as a mutable counter.

### 2. Correct pointer semantics

Initially:

$$
rI_1 = A, \quad rI_2 = D
$$

After $k$ iterations:

$$
rI_1 = A+k, \quad rI_2 = D+k
$$

So iteration $k$ copies:

$$
M(A+k) \to M(D+k)
$$

### 3. Correct termination

Register $X$ is initialized to $F$ and decremented once per iteration. The loop executes exactly $F$ times.

### 4. Correct handling of $I$

- If $I = 0$, destination is explicitly set to $A$
- If $I \ne 0$, destination is loaded from index register $rI_I$

### 5. Correct linkage

`STJ SAVE` preserves the return address, and `JMP SAVE` restores execution exactly after the calling sequence.

## Conclusion

The corrected program implements a valid MIX `MOVE` subroutine using only legitimate register operations. It performs a faithful block transfer from $A$ to $D$ for $F$ words, with correct handling of indexing, termination, and return linkage, differing from a hardware `MOVE` only in time and space overhead.
