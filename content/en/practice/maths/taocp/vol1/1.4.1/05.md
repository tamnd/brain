**Exercise 5.** [*21*] Suppose MIX did not have a J-register. Invent a means for subroutine linkage that does not use register `J`, and give an example of your invention by writing a MAX100 subroutine effectively equivalent to (1). State the characteristics of this subroutine in a fashion similar to (4). (Retain MIX's conventions of self-modifying code.)

**Verified:** yes  
**Solve time:** 1m16s

## Solution

In MIX without a J-register, subroutine linkage must be achieved by explicitly storing the return address in a general register or memory cell before transferring control to the subroutine, and then returning by an indexed jump through that stored address.

A convenient choice is to reserve index register $rI4$ as the linkage register. The calling sequence must place the return address into $rI4$ and then jump to the subroutine. The return address is supplied directly by the assembler as the label of the instruction following the call.

### Calling mechanism

Let the call to the subroutine MAX100 occur at some location $L$. The next instruction after the call is labeled $L^*$. The calling sequence is written as

```
ENT4 L*
JMP  MAX100
```

The instruction `ENT4 L*` loads $rI4 \leftarrow L^*$, so $rI4$ contains the return address at the moment control enters the subroutine.

### Subroutine MAX100 without a J-register

The subroutine uses $rI4$ as its sole linkage mechanism. All other behavior is identical to the standard MAX100 in Section 1.4.1, except that the final return is performed by an indexed jump through $rI4$.

```
MAX100  ST3  SAVEI3
        ST2  SAVEI2

        ENT3 100          n = 100
        JMP  2F           continue as in (1)

        ...               (MAX100 body identical to (1))
        ...               (search loop computing max)

        LD2  SAVEI2
        LD3  SAVEI3

EXIT    JMP  *,4
```

The instruction `JMP *,4` transfers control to the address contained in $rI4$, thereby returning to the instruction immediately following the original call.

### Correctness of linkage

At entry, $rI4$ contains the address $L^_$ of the instruction following the call. The subroutine does not modify $rI4$, so this value remains unchanged throughout execution. When execution reaches `JMP *,4`, control is transferred exactly to $L^_$, which reproduces the behavior of the original J-register-based linkage.

### Characteristics of the subroutine

The calling sequence and behavior can be summarized in the same format as (4):

```
Calling sequence: ENT4 L*; JMP MAX100.
Entry conditions: rI4 = L*; assume n = 100.
Exit conditions:
  rA  = max_{1 ≤ k ≤ 100} CONTENTS(X + k)
      = CONTENTS(X + rI2);
  rI3 = 0; rI4 unchanged and used for return;
  rJ and comparison indicator are affected.
```

No memory locations external to the subroutine are altered except those explicitly used for computation inside MAX100, and the linkage mechanism requires no modification of the instruction stream during execution.

This completes the construction of subroutine linkage without a J-register. ∎
