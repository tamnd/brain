Algorithm E constructs the set of nodes reachable from the designated list heads by repeatedly following the structural fields `DLINK` and `RLINK`, marking each visited node by setting its mark bit `S`, and then performs a second pass that returns every unmarked node to `AVAIL`. The validity proof separates into completeness of marking and soundness of collection.

Let $R$ denote the set of nodes reachable from the initial list heads by following a finite sequence of `DLINK` and `RLINK` pointers. Every step of the marking phase moves along one such pointer and marks the endpoint; therefore every node inserted into the marked set is reachable from a head node, so it lies in $R$. Conversely, consider any node $x \in R$. There exists a finite chain of pointers from a head node to $x$. The algorithm begins at each head node and proceeds by repeatedly applying the same pointer-following rule used to define reachability. By induction on the length of the chain, the node at each stage is eventually visited and marked, since whenever a marked node is processed all its outgoing pointers are examined and their targets are marked if not already marked. Hence every node in $R$ is marked, so the marked set coincides with $R$.

In the sweep phase, a node is returned to `AVAIL` if and only if its mark bit is not set, so precisely the complement of $R$ in the memory graph is reclaimed. Any node not in $R$ is not reachable from any list head, hence cannot be accessed by any list operation defined in Section 2.3.5, so its reclamation does not affect the correctness of subsequent list manipulation. This establishes that Algorithm E marks exactly the reachable nodes and collects exactly the unreachable nodes. This completes the proof. ∎
