**Exercise 8.** [**] [27] (L. P. Deutsch.) Design a marking algorithm similar to Algorithms D
and E in that it uses no auxiliary memory for a stack, but modify the method so
that it works with nodes of variable size and with a variable number of pointers
having the following format: The first word of a node has two fields `MARK` and
`SIZE`; the `MARK` field is to be treated as in Algorithm E, and the `SIZE`
field contains a number $n\ge 0$. This means that there are $n$ consecutive
words after the first word, each containing two fields `MARK` (which is zero and
should remain so) and `LINK` (which is `\Lambda` or points to the first word of
another node). For example, a node with three pointers would comprise four
consecutive words:

```text
First word    MARK = 0 (will be set to 1)    SIZE = 3
Second word   MARK = 0    LINK = first pointer
Third word    MARK = 0    LINK = second pointer
Fourth word   MARK = 0    LINK = third pointer.
```

Your algorithm should mark all nodes reachable from a given node `P0`.

**Solve time:** 6m14s

Let $P_0$ denote the initial node. The algorithm maintains a single pointer $p$ to the current node being processed and a single pointer $r$ that threads the traversal of edge-words forming an implicit return chain. Each node is marked exactly once via its `MARK` field, and the edges are explored by scanning the consecutive words determined by the `SIZE` field.

A node $p$ is first tested; if $\operatorname{MARK}(p)=1$, control returns to the previously suspended position through the chain stored in edge-word `LINK` fields. Otherwise $\operatorname{MARK}(p)\leftarrow 1$ and the traversal of its outgoing pointers begins at the first edge word immediately following the node. If $\operatorname{SIZE}(p)=n$, these are the $n$ consecutive words $(w_1,\dots,w_n)$, each containing a `LINK` to a successor node.

During the scan of a node, suppose the current edge word is $w_k$ and its `LINK` field contains a pointer $q$. Before moving to $q$, the algorithm performs the local replacement $w_k.\mathrm{LINK}\leftarrow r$, and then sets $r\leftarrow w_k$. This action constructs a singly linked chain of edge words representing the suspended continuation points of the depth-first traversal. The pointer $q$ is preserved in a local variable during this update and is then used as the next current node, namely $p\leftarrow q$.

When a node $p$ has no unprocessed edge words left, the algorithm backtracks by removing the most recent edge word $w$ from the chain $r$. The node containing $w$ is recovered as the parent context, and the next unprocessed edge word of that parent is obtained by continuing the sequential scan from the position immediately following $w$ within its block of $n$ edge words. This reconstruction is valid because the edge words of a node occupy consecutive locations determined by `SIZE`, so the successor position is determined solely by address increment within the node layout.

Since each node is marked when first entered and each edge word is processed exactly once in either forward exploration or backtracking reconstruction, the procedure visits exactly the set of nodes reachable from $P_0$ and assigns `MARK = 1` to each of them. No auxiliary stack is used beyond the modification of existing `LINK` fields of edge words, which serve as the transient return structure of the traversal. This completes the marking of all nodes reachable from $P_0$. ∎
