**Exercise 5.** [**] [25] (Schorr and Waite.) Give a marking algorithm that combines

A precise combined Schorr–Waite marking procedure is obtained by running a depth-first traversal in which the stack stores continuation states exactly as in Algorithm B, and the pointer-reversal mechanism of Algorithm E is used only to simulate stack frames when overflow occurs. The mark bit $S$ and link-reversal discipline are those of Algorithm E.

Each node is assumed to contain a mark bit $S$ and link fields as in Algorithm E. The stack entries have the form $(P, p)$, where $P$ is a node and $p$ identifies which link of $P$ is to be explored next. For a node with two outgoing links (as in the binary-tree form underlying Algorithm E), $p \in {0,1}$ indicates whether the left or right link is next to be processed.

The algorithm begins by setting $P \leftarrow \text{root}$. For every node $P$, the condition $S(P)=0$ means unvisited and $S(P)=1$ means already marked.

When $P \neq \Lambda$ and $S(P)=0$, the node is marked by setting $S(P) \leftarrow 1$. The traversal then enters a loop in which it repeatedly tries to store the current continuation state in the stack before descending.

If $p=0$, the next node to visit is $Q \leftarrow \text{DLINK}(P)$, and the next continuation state is $p \leftarrow 1$ for the same node $P$. If $p=1$, the next node is $Q \leftarrow \text{RLINK}(P)$, and no further continuation remains at $P$.

At each descent step, the algorithm attempts to execute

$$
\text{STACKTOP} \leftarrow \text{STACKTOP}+1,\quad \text{STACK}[\text{STACKTOP}] \leftarrow (P,p).
$$

If this increment keeps $\text{STACKTOP} \le N$, then the algorithm moves to the child node by setting $P \leftarrow Q$ and $p \leftarrow 0$, and the process repeats.

When the attempt to push would give $\text{STACKTOP} = N+1$, stack overflow occurs and the mechanism of Algorithm E is activated for the current node $P$.

In this E-phase, no stack operations are performed. Instead, the link that would normally store the return information is reversed exactly as in Algorithm E: if the traversal is about to move from $P$ to $Q$, the link $(P,Q)$ is replaced by a backward link to the previously visited node, and the mark bit $S$ is used to distinguish already visited nodes so that no node is marked twice. The reversal encodes the fact that the current state $(P,p)$ cannot be stored on the stack.

Formally, during overflow at node $P$ with next node $Q$, the algorithm performs the same transformation as Algorithm E: the appropriate link field of $P$ is redirected to point back toward the predecessor in the traversal path, and the traversal continues with $P \leftarrow Q$ while preserving reachability of $P$ via reversed links. All further backtracking in this phase follows reversed links until a node is reached that still has unprocessed links.

Whenever backtracking in the E-phase reaches a node $R$ for which an unprocessed continuation exists and the stack is no longer full, the algorithm returns to B-mode by restoring the invariant that the traversal state is again stored in $(P,p)$ and resuming stack-based execution. This transition is well defined because E-phase always leaves a reversible path structure identical to Algorithm E, so the current node and its next unexplored link are recoverable without stack information.

The key invariant is that every unprocessed subtree is represented either by a stored stack frame $(P,p)$ or by a reversed link chain exactly as in Algorithm E. The stack contains a faithful simulation of DFS backtracking whenever space is available, and the E-mechanism preserves the same backtracking information whenever it is not.

Every node is marked exactly once because $S(P)$ is set to $1$ immediately upon first encounter and neither stack re-entry nor reversed-link traversal revisits a node with $S(P)=1$ as an unmarked node. Stack operations never duplicate states because a state is pushed only when it has not yet been executed, and E-phase traversal never creates new unexplored configurations, only encodes pending ones structurally.

Termination follows because each step either removes one unprocessed continuation from the stack or eliminates one link from further consideration via reversal, and both structures are finite. When the stack becomes empty and no reversed links lead to unprocessed continuations, every node has been completely explored and marked, so the algorithm halts. This completes the construction of the combined Schorr–Waite marking algorithm. ∎
