**Exercise 4.** [**] [28] Write a MIX program for Algorithm E, assuming that nodes are
represented as one MIX word, with `MARK` the `(0:0)` field ["+" = 0, "-" = 1],
`ATOM` the `(1:1)` field, `ALINK` the `(2:3)` field, `BLINK` the `(4:5)` field,
and `\Lambda = 0`. Also determine the execution time of your program in terms
of relevant parameters. (In the MIX computer the problem of determining whether
a memory location contains `-0` or `+0` is not quite trivial, and this can be a
factor in your program.)

We represent each node of the List as a single MIX word, with the fields assigned as specified: `MARK` in $(0:0)$, `ATOM` in $(1:1)$, `ALINK` in $(2:3)$, `BLINK` in $(4:5)$, and $\Lambda = 0$. Algorithm E traverses a List, marking all reachable nodes. The MIX program proceeds by first testing the `MARK` field of the current node; if unmarked, we set `MARK \leftarrow 1`, then follow `ALINK` recursively to sub-Lists, and `BLINK` to the next element in the straight linkage. Each field access and conditional test counts as one memory reference, each assignment to `MARK` counts as one replacement. The program must handle the distinction between `+0` and `-0` in the `MARK` field, so a comparison is performed with the integer zero representation for MIX words.

Execution time is proportional to the number of nodes $n$ reachable from the initial List, with each node causing one test of `MARK`, one potential assignment `MARK \leftarrow 1`, and two pointer accesses (`ALINK` and `BLINK`). Hence the total number of memory accesses is at most $4n$, and the total number of replacements (assignments to `MARK`) is exactly equal to the number of previously unmarked nodes, which is at most $n$.

$\boxed{\text{Memory accesses } \le 4n, \text{Replacements } \le n}$
