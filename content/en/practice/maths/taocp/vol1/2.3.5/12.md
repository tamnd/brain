The previous solution failed because it implicitly allowed uncontrolled queue duplication and did not establish a genuine worst-case per-operation bound independent of heap size and history. We now rebuild the argument in a way that matches a real-time discipline: bounded work per List operation, strictly bounded auxiliary structure behavior, and a monotone global progress measure.

## 1. Data representation and invariants

Each node is in exactly one of three states:

- White: not yet discovered by the collector
- Gray: discovered but not fully scanned
- Black: fully scanned

We maintain the fundamental invariant:

> (**Tri-color invariant**) No pointer from a black node points to a white node.

Formally,

$$
x \text{ is black and } x \to y \;\Rightarrow\; y \text{ is not white}.
$$

All reachable nodes are initially white except roots, which are gray.

## 2. Queue discipline without duplication

A central correction is required: each gray node appears in the work queue at most once.

We enforce this by coupling color change and queue insertion:

- A node is enqueued **only at the moment it changes from white to gray**.
- There is no other enqueue operation.

Thus:

- If a node is already gray, it is never enqueued again.
- If a node is black, it is never enqueued.
- If a node is white, it is enqueued exactly once when first discovered.

This removes the need for a separate “in-queue” flag, because:

$$
\text{enqueued} \;\Leftrightarrow\; \text{just turned gray}.
$$

Hence queue operations are strictly bounded.

## 3. Collector algorithm

The collector repeatedly performs the following atomic step:

1. Remove one node $v$ from the queue.
2. For each pointer $v \to w$:

- If $w$ is white:

- set $w$ to gray
- enqueue $w$
3. Set $v$ to black.

Each collector step performs $O(1)$ work per outgoing pointer, and in TAOCP List structures the degree of a node is bounded by a constant (fixed representation), so each step is $O(1)$.

## 4. Mutator with write barrier

Each List operation (mutator step) performs two kinds of actions:

### (a) Normal pointer manipulation and allocation

- Allocation creates a new white node.

### (b) Write barrier

For each pointer assignment $x.f := y$:

- If $x$ is black and $y$ is white, then:

- set $y$ to gray
- enqueue $y$

This is the only situation in which the mutator enqueues nodes.

## 5. Preservation of the tri-color invariant

We show the invariant is preserved under both collector and mutator actions.

### Collector preservation

When a gray node $v$ is processed:

- Every outgoing edge $v \to w$ is examined.
- If $w$ is white, it is immediately turned gray.

Thus after processing $v$, no edge from $v$ (now black) points to a white node.

Since previously processed black nodes already satisfied the invariant, it is preserved.

### Mutator preservation

Consider an assignment $x.f := y$.

The only dangerous case is:

- $x$ is black and $y$ is white.

The write barrier immediately turns $y$ gray, eliminating any black-to-white edge.

Thus no black node can point to a white node after the assignment completes.

Hence the invariant is preserved under all interleavings.

## 6. Global potential function

To control real-time behavior, define the potential:

$$
\Phi = \#(\text{gray nodes}) + \#(\text{white nodes reachable from roots}).
$$

Interpretation:

- Gray nodes represent unfinished collector work.
- Reachable white nodes represent undiscovered live structure.

### Effect of collector step

A collector step processes one gray node:

- Gray count decreases by 1
- Each newly discovered white neighbor becomes gray, increasing gray count but strictly decreasing reachable white count

Net effect:

$$
\Phi \text{ decreases by at least } 1.
$$

### Effect of mutator step

A mutator step can:

- allocate $O(1)$ new white nodes
- create a constant number of pointers

Thus it can increase $\Phi$ by at most a fixed constant $c$.

## 7. Real-time scheduling discipline

We now define the execution rule:

> Each List operation performs at most $k$ collector steps, for a fixed constant $k$, followed by its own bounded mutator work.

Thus each operation performs:

$$
O(1) \text{ mutator work} + O(1) \text{ collector work}.
$$

All queue operations are $O(1)$, since each node is enqueued at most once.

## 8. Bounded backlog and worst-case guarantee

Let $n$ be the number of reachable nodes.

Initially, $\Phi = O(n)$.

Each operation changes $\Phi$ by at most:

- $+c$ from mutation
- $-k$ from collection

So over $t$ operations:

$$
\Phi(t) \le \Phi(0) + ct - kt.
$$

Choosing $k > c$, we obtain:

$$
\Phi(t) \to 0 \text{ linearly in } t.
$$

Thus:

- the collector cannot fall behind indefinitely
- the amount of unfinished work remains uniformly bounded by a constant multiple of live structure size
- no unbounded queue growth occurs

Most importantly, every operation performs a fixed bounded number of steps independent of $n$ or execution history.

## 9. Liveness

When $\Phi = 0$:

- no reachable white nodes remain
- no gray nodes remain
- all reachable nodes are black

Hence all unreachable nodes are exactly the remaining white nodes and may be reclaimed.

Since $\Phi$ decreases by at least a fixed amount per operation (after accounting for bounded mutation), eventual completion follows.

## 10. Conclusion

We have constructed a real-time garbage collection scheme with:

1. A strict tri-color invariant preserved under all interleavings
2. A queue discipline ensuring each node is enqueued at most once
3. Constant-time queue and color operations
4. A global potential function proving bounded lag between mutation and collection
5. A fixed constant amount of collector work per List operation

Therefore each List operation has a uniform worst-case time bound independent of heap size and history, while garbage collection proceeds concurrently and reliably in real time. ∎
