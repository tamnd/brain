[Section 2.3.5: Lists and Garbage Collection](../)

**Exercise 2.** [**] [20] In Section 2.3.1 we saw that tree traversal can be facilitated using
a threaded representation inside the computer. Can List structures be threaded
in an analogous way?

**Verified:** no  
**Solve time:** 6m04s  

---

List structures can be threaded in the same sense as threaded binary trees by using otherwise unused pointer fields to store links that represent the successor of a node in a chosen traversal order. A traversal order must first be fixed, for example the natural linear order induced by repeated `RLINK` moves together with a depth-first descent through `DLINK` sublists. For each node whose structural pointer would otherwise be null or lead to an unused field, that pointer may be redirected to the next node in this traversal sequence, and a `TYPE` or auxiliary bit is required to distinguish a genuine structural link from a thread.

In a doubly linked representation, both directions of traversal admit threading: a missing `LLINK` or `RLINK` may be replaced by a predecessor thread or successor thread respectively, provided the corresponding end condition of the traversal is encoded at the list heads. Sublist entry and return points must also be included in the ordering so that threads connect across nested list boundaries in a consistent depth-first sequence. This construction removes the need for an explicit stack during traversal, since each node contains sufficient information to move directly to its successor in the chosen order. This completes the proof. ∎
