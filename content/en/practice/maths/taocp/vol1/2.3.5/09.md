**Exercise 9.** [**] [28] (D. Edwards.) Design an algorithm for the second phase of garbage
collection that "compacts storage" in the following sense: Let
`NODE(1), ..., NODE(M)` be one-word nodes with fields `MARK`, `ATOM`, `ALINK`,
and `BLINK`, as described in Algorithm E. Assume that `MARK = 1` in all nodes
that are not garbage. The desired algorithm should relocate the marked nodes, if
necessary, so that they all appear in consecutive locations `NODE(1), ...,
NODE(K)`, and at the same time the `ALINK` and `BLINK` fields of nonatomic
nodes should be altered if necessary so that the List structure is preserved.

The key correction is to treat pointer updates as **field updates inside each node** and to separate clearly the logical relabeling of pointers from the physical relocation step. We also make explicit the invariant that no live pointer refers to a garbage node after marking.

## Corrected Solution

Let the marking phase have already been completed, so that each node `NODE(i)` has `MARK(i) ∈ {0,1}`, with `MARK(i)=1` exactly for reachable (non-garbage) nodes. Let `Λ` denote the null pointer.

Let

$$
K = |\{ i : \mathrm{MARK}(i)=1 \}|.
$$

Define the ranking function $f$ by a single scan:

initialize $j \leftarrow 0$, and for $i=1,2,\dots,M$,

if $\mathrm{MARK}(i)=1$ then set

$$
j \leftarrow j+1,\quad f(i) \leftarrow j.
$$

Thus $f$ is a bijection from marked indices onto $\{1,\dots,K\}$.

## Phase 2: Relabel pointer fields

We now update pointers **inside each surviving node**.

For each $i$ with $\mathrm{MARK}(i)=1$, do:

- If $\mathrm{ALINK}(i) \neq \Lambda$, replace

$$
\mathrm{ALINK}(i) \leftarrow f(\mathrm{ALINK}(i)).
$$
- If $\mathrm{BLINK}(i) \neq \Lambda$, replace

$$
\mathrm{BLINK}(i) \leftarrow f(\mathrm{BLINK}(i)).
$$

### Justification of well-definedness

After the marking phase, a standard invariant holds: every pointer reachable from a marked node points either to $\Lambda$ or to another marked node. Otherwise, that node would have been reachable and therefore marked. Hence $f$ is defined on every non-null link used above.

Thus all pointer updates are well-defined and map surviving nodes into $\{1,\dots,K\}$.

## Phase 3: Physical compaction

Allocate an auxiliary array $\mathrm{NODE}'(1),\dots,\mathrm{NODE}'(K)$.

For each $i$ with $\mathrm{MARK}(i)=1$, place:

$$
\mathrm{NODE}'(f(i)) \leftarrow \mathrm{NODE}(i).
$$

Finally, replace the active portion of the storage by:

$$
\mathrm{NODE}(j) \leftarrow \mathrm{NODE}'(j), \quad 1 \le j \le K.
$$

Nodes with $\mathrm{MARK}(i)=0$ occupy positions $K+1,\dots,M$ and may be left unchanged or placed on a free list depending on implementation.

## Correctness argument

1. **Bijection of live nodes to compact segment**

The function $f$ maps each marked node to a unique index in $\{1,\dots,K\}$, so every surviving node is assigned exactly one new location, and all such locations are filled.
2. **Pointer preservation**

After Phase 2, every pointer field in every surviving node is transformed from an old index $i$ to $f(i)$. Since Phase 3 places node $i$ exactly at location $f(i)$, all links now correctly refer to relocated nodes.
3. **No dangling references among live nodes**

By the marking invariant, no pointer from a live node leads to an unmarked node, so all structural references are preserved under $f$.
4. **Structural preservation**

For any surviving node $i$, its adjacency structure is preserved because:

- its links are relabeled consistently via $f$,
- and its physical position is moved consistently to $f(i)$.

Thus the directed graph defined by `ALINK` and `BLINK` among live nodes is isomorphic before and after compaction.

## Conclusion

The algorithm correctly compacts all marked nodes into contiguous storage locations $\mathrm{NODE}(1),\dots,\mathrm{NODE}(K)$ and preserves all list structure by consistently relabeling internal pointers via $f$ before relocation.
