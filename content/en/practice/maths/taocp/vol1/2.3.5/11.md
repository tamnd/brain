### Corrected Solution

#### 1. Formal model of List structures

We model a List structure as a finite directed, rooted, ordered graph.

Each node is of exactly one of two kinds:

1. **Atom node**

Labeled by an atomic value $a \in \mathcal{A}$. It has no outgoing edges.
2. **List node**

Represents an ordered sequence (possibly empty) of elements. Each element is either:

- an atom node, or
- a pointer to another list node.

Formally, each list node $v$ has an ordered finite sequence of successors

$$
\operatorname{succ}(v) = (v_1, v_2, \dots, v_k)
$$

where each $v_i$ is a node (atom or list node).

This is exactly the structure implicitly described by expressions such as

$(a:C, b, a:(b:D))$, where colon denotes a reference into a sublist.

The **fully expanded tree** of a node is obtained by repeatedly replacing each list node by its ordered sequence of successors. If the structure is recursive, this unfolding is infinite.

Two list structures are equivalent in the sense of the problem if their fully expanded (possibly infinite) ordered trees are isomorphic as rooted ordered trees with atom labels preserved.

This is exactly bisimulation equivalence on rooted ordered labeled graphs.

#### 2. Characterization of equivalence

Let $p$ and $q$ be nodes. Define a relation $R$ such that $(p,q) \in R$ means “$p$ and $q$ represent equivalent unfolded structures”.

Then $R$ is the greatest relation satisfying:

1. If $p$ and $q$ are atoms, then they must have the same label.
2. If exactly one of $p,q$ is an atom, they are not related.
3. If both are list nodes, then:

- they must have the same number of successors
- and their corresponding successors must also be related:

$$
(p_i, q_i) \in R \quad \text{for all } i
$$

Thus equivalence is greatest bisimulation on this structure.

#### 3. Algorithm (worklist bisimulation test)

We maintain a worklist of node pairs and a visited set.

**Input:** root nodes $A_0, B_0$

Initialize:

$$
S \leftarrow \{(A_0, B_0)\}, \quad V \leftarrow \varnothing
$$

While $S \neq \varnothing$:

1. Remove a pair $(p,q)$ from $S$.
2. If $(p,q) \in V$, continue.
3. Insert $(p,q)$ into $V$.
4. **Type check**

- If exactly one of $p,q$ is an atom, reject.
- If both are atoms:

- accept this pair only if their labels are equal
- continue
5. **List node case**

Suppose both are list nodes.

Let

$$
\operatorname{succ}(p) = (p_1,\dots,p_k), \quad
\operatorname{succ}(q) = (q_1,\dots,q_m)
$$

- If $k \neq m$, reject.
- Otherwise, for each $i = 1,\dots,k$, push $(p_i, q_i)$ onto $S$.

If the algorithm finishes without rejection, accept equivalence.

#### 4. Correctness proof

We prove soundness and completeness.

### Lemma 1 (Invariant)

At every stage, every pair $(p,q) \in V$ is such that $p$ and $q$ are equivalent under all constraints already verified, and any unexplored structural obligations are contained in $S$.

**Proof.**

Initialization is trivial. Each step removes a pair and only inserts successor pairs that must match in any valid bisimulation. Rejected pairs violate either label equality or structural arity equality, which are necessary conditions for equivalence. Thus no invalid pair enters $V$. ∎

### Lemma 2 (Soundness)

If the algorithm accepts, then the two root structures have identical full expansions.

**Proof.**

Acceptance implies all reachable pairs satisfy:

- identical atom labels, and
- identical branching structure, with all corresponding successors paired.

Thus $V$ is a bisimulation relation. Bisimulation on finite labeled ordered graphs implies equivalence of all finite unfoldings. Therefore all finite prefixes of the two expanded trees are isomorphic. Hence the infinite unfoldings, if cyclic, are identical as infinite trees. ∎

### Lemma 3 (Completeness)

If the two structures have identical full expansions, the algorithm accepts.

**Proof.**

Assume equivalence under full unfolding. Then the relation consisting of all corresponding nodes in the unfolding is a bisimulation. Every pair of corresponding nodes satisfies label equality and matching successor structure.

The algorithm explores exactly the reachable pairs under this bisimulation starting from $(A_0,B_0)$. No rule of rejection is triggered, since equivalence guarantees equal labels and equal arity at every list node. Hence no pair is rejected, and all required successor pairs are eventually processed. ∎

### Lemma 4 (Termination)

The algorithm terminates.

**Proof.**

The underlying structure has finitely many nodes. Therefore there are only finitely many ordered pairs of nodes. Each pair enters $V$ at most once. Each iteration either adds a new pair or removes one already processed. Hence the worklist procedure must terminate. ∎

#### 5. Conclusion

The algorithm correctly decides equivalence of List structures under full expansion by computing the greatest bisimulation relation on the induced finite directed ordered graph. This equivalence exactly matches equality of the possibly infinite unfolded trees.

$$
\boxed{\text{Two List structures are equivalent iff the algorithm accepts.}}
$$
