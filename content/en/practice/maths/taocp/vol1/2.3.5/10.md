**Exercise 10.** [**] [28] Design an algorithm that copies a List structure, assuming that an
internal representation like that in (7) is being used. (Thus, if your procedure
is asked to copy the List whose head is the node at the upper left corner of
(7), a new set of Lists having 14 nodes, and with structure and information
identical to that shown in (7), should be created.)

Assume that the List structure is stored in memory using `S`, `T`, `REF`, and
`RLINK` fields as in (9), and that `NODE(P0)` is the head of the List to be
copied. Assume further that the `REF` field in each List head node is
`\Lambda`; to avoid the need for additional memory space, your copying
procedure should make use of the `REF` fields (and reset them to `\Lambda`
again afterwards).

The copying procedure is defined on nodes reachable from $P_0$ using a depth-first traversal in which the field $\mathrm{REF}$ temporarily stores the correspondence between an original node $x$ and its copy. For each node $x$, the algorithm first tests whether $\mathrm{REF}(x)\ne \Lambda$. If this holds, the node has already been copied and $\mathrm{REF}(x)$ is returned as its image. Otherwise a new node $y$ is allocated and $\mathrm{REF}(x)\leftarrow y$ is executed, establishing the mapping from $x$ to its copy. The fields of $y$ are then initialized according to the type field $T$ of $x$. If $T>1$, $x$ is an atom, and the information field is duplicated into $y$; no structural recursion is required. If $T=0$ or $T=1$, the node is a list head or sublist node, and the structural links are followed: for each relevant pointer $p$ among $\mathrm{RLINK}(x)$ and, when $T=1$, also the list pointer stored in $\mathrm{REF}(x)$ according to (9), the algorithm ensures that the corresponding node is copied by evaluating $\mathrm{REF}(p)$ if it is already defined, otherwise recursively invoking the procedure on $p$, and the resulting node is assigned to the corresponding link field of $y$.

After all reachable nodes have been processed in this manner, a second traversal over the same reachable subgraph restores all auxiliary markings by replacing $\mathrm{REF}(x)\leftarrow \Lambda$ for every node $x$. The structure of the copy is preserved because every pointer in each original node is translated through the mapping $x\mapsto \mathrm{REF}(x)$ at the moment of assignment, so adjacency relations among list heads, sublists, and atoms are preserved exactly under the induced image mapping. This completes the construction of a duplicate list system isomorphic to the one rooted at $P_0$, using $\mathrm{REF}$ as temporary storage without requiring additional memory. ∎
