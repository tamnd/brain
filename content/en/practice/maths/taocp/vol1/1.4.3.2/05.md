[Section 1.4.3.2: Trace Routines](../)

**Exercise 5.** [*28*] In a manner similar to that used to solve the previous exercise, consider the situation in which two copies of the trace routine are placed in different places in memory, and each is set up to trace the other. What would happen?

**Verified:** yes  
**Solve time:** 1m10s

## Setup

Let $T_A$ and $T_B$ denote two physically distinct copies of the trace routine in Section 1.4.3.2 placed at disjoint regions of memory, with separate entry points. Each routine performs the same control logic: it intercepts execution of a target instruction, simulates register effects, optionally logs state, and then resumes control via the cycle loop.

Assume $T_A$ is set to trace execution occurring in the region containing $T_B$, and $T_B$ is set to trace execution occurring in the region containing $T_A$. The execution of either routine therefore treats the other routine as the “program being traced” whenever control enters the other routine’s memory region.

The problem is to determine the global behavior of execution under this mutual tracing configuration.

## Solution

Execution begins when control enters one of the trace routines, say $T_A$. The routine immediately assumes it is supervising a traced program and proceeds to execute its interception logic. During this process, control transfers through the simulated instruction cycle, including instruction fetch, classification of jumps, and execution at location `INST`.

Since $T_B$ lies in a different memory region, any transfer of control into $T_B$ occurs as part of normal execution of instructions within that region. The moment control enters $T_B$, the same trace mechanism is triggered again, because $T_B$ is active as a trace routine for the region containing $T_A$. Thus the entry into $T_B$ initiates a second level of tracing applied to execution that already originated from $T_A$.

Within $T_B$, the simulated execution of instructions of $T_A$ (or instructions reached via return paths from $T_A$’s cycle loop) causes a symmetric situation. Whenever control returns to $T_A$, it is again interpreted as traced program activity by $T_A$, reactivating its interception logic.

This produces a nested sequence of activations of the form

$$
T_A \rightarrow T_B \rightarrow T_A \rightarrow T_B \rightarrow \cdots
$$

where each transition arises from the fact that entering the memory region of one trace routine is itself an executable event subject to tracing by the other routine.

Each activation requires completion of the full trace cycle before returning control to the previous level. However, each trace cycle contains no terminating condition that distinguishes “external program execution” from “trace routine execution” when both reside in traced regions. Therefore, no activation level is ever discharged before a deeper level is created.

Because every step of simulated execution necessarily passes through the instruction dispatch at `INST` and the cycle advance at `INCP`, and because both routines intercept these transitions in their respective regions, the nesting depth increases without bound. No finite prefix of execution reaches a stable state in which either routine executes a traced instruction without triggering the other.

Thus the system does not simulate two independent traces. Instead, it generates an unbounded mutual re-entry process with no terminal configuration.

## Verification

Each invocation of a trace routine is triggered by control entering its designated memory region. Since both $T_A$ and $T_B$ are themselves located in traced regions, every return to either routine reactivates tracing logic rather than terminating it.

No instruction in either routine provides a mechanism to recognize that the current execution context originated from the other trace routine rather than an external program. The only available state information consists of register simulation and instruction classification, which is identical in both directions of tracing.

Therefore, no step in the control flow halts the creation of a new trace activation level. Every attempted completion of a trace cycle necessarily re-enters the other routine, producing infinite alternation.

This establishes that no finite execution segment corresponds to a completed trace of a stable external program.

## Conclusion

Two mutually tracing copies of the trace routine generate an unbounded alternation of control between the two routines, with each activation initiating another before the previous completes. The result is non-terminating execution with no meaningful separation between “program” and “trace mechanism”.

This completes the analysis. ∎
