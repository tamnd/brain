[Section 1.4.3.2: Trace Routines](../)

**Exercise 3.** [*10*] The previous exercise suggests having the trace program write its output onto tape. Discuss why this would be preferable to printing directly.

**Verified:** yes  
**Solve time:** 1m07s  

---

Writing trace output directly to a printer or similar output device forces the trace routine to compete with the traced program for that device. This conflicts with restriction (b), since instructions such as $JBUS$ may then reflect activity caused by tracing rather than by the external program. Using tape avoids this interference because the trace output is recorded on a separate sequential medium that is not used by the program being observed, so the state of the traced computation is not distorted by output-device contention.

Tape output is also preferable because it permits block buffering in a fixed format, as in the 100-word structure described in Exercise 2, so that many instruction snapshots can be accumulated with minimal device switching. The resulting record can be processed later by a separate program that prints or analyzes it, which removes time spent on immediate formatting and device handling from the execution path of the trace routine itself. This separation preserves the correctness of the trace and reduces perturbation of the simulated execution timing. ∎
