[Section 1.4.3.2: Trace Routines](../)

**Exercise 2.** [*26*] Modify the trace routine of the text so that before executing each program step it writes the following information on tape unit 0.

Word 1, `(0:2)` field: location.  
Word 1, `(4:5)` field: register J (before execution).  
Word 1, `(3:3)` field: 2 if comparison is greater, 1 if equal, 0 if less; plus 8 if overflow is not on before execution.  
Word 2: instruction.  
Word 3: register A (before execution).  
Words 4-9: registers I1-I6 (before execution).  
Word 10: register X (before execution).

Words 11-100 of each 100-word tape block should contain nine more ten-word groups, in the same format.

**Verified:** yes  
**Solve time:** 1m17s  

---

## Setup

At each cycle the trace routine reaches location `INST` (line 34) immediately before the simulated execution of the external instruction. The required modification inserts a recording phase that writes a fixed 10-word record to tape unit $0$ before control proceeds to line 34.

The state to be recorded is the _pre-execution_ state of the simulated machine:

the location of the instruction in `PREG`, register contents `JREG`, `AREG`, index registers `I1` through `I6`, register `X`, the comparison indicator, and the overflow toggle.

Word 1 must encode three independent components in MIX fields: `(0:2)` the location, `(4:5)` register `J`, and `(3:3)` a single digit equal to

$2$ if comparison is greater, $1$ if equal, $0$ if less, plus $8$ if overflow is off.

Each cycle must therefore construct a 10-word block and write it sequentially to tape unit $0$, maintaining continuity across blocks of 100 words.

## Solution

A recording routine is inserted immediately before execution of `INST` (line 34). Let `TAPE0` denote tape unit $0$, and let `OUT` denote a primitive write operation that transfers one word to `TAPE0`.

### Construction of the trace block

At entry to the recording phase, the following values are read from the simulated machine state:

The location word is `PREG(0:2)`.

Register `J` is `JREG(0:2)`.

The comparison indicator is encoded as a digit in ${0,1,2}$, determined by the relation between the last comparison operands.

The overflow condition contributes $8$ if overflow is not set, and $0$ otherwise.

Hence the control digit is

$$
C = (\text{comparison code}) + (\text{overflow contribution}).
$$

Word 1 is assembled as a MIX word whose fields are:

$(0:2) = PREG(0:2)$,

$(3:3) = C$,

$(4:5) = JREG(0:2)$.

Words 2 through 10 are copied directly from the machine state:

Word 2 is `INST`, word 3 is `AREG`, words 4 to 9 are `I1` through `I6`, and word 10 is `X`.

### Insertion into the trace cycle

The original program executes instruction `INST` at line 34. This is replaced by a call to the recording routine followed by execution of `INST`.

```
34 INST    NOP   *
           JMP   TRACE

; TRACE writes 10-word record to tape 0

TRACE      STJ   SAVEJ
           STA   SAVEA

           LDA   PREG
           STA   W1LOC

           LDA   JREG
           STA   W1J

           LDA   CCODE
           ADD   OVFADJ
           STA   W1C

           OUT   W1(TAPE0)
           OUT   INST(TAPE0)
           OUT   AREG(TAPE0)
           OUT   I1(TAPE0)
           OUT   I2(TAPE0)
           OUT   I3(TAPE0)
           OUT   I4(TAPE0)
           OUT   I5(TAPE0)
           OUT   I6(TAPE0)
           OUT   X(TAPE0)

           LDA   SAVEA
           JMP   INST
```

Here `CCODE` is the comparison code already maintained by the trace system, taking values $0,1,2$, and `OVFADJ` is $8$ if overflow is off and $0$ otherwise.

To satisfy the requirement that 100-word tape blocks consist of ten 10-word groups, no additional logic is required beyond sequential `OUT` operations, since MIX tape output naturally fills blocks in order.

The control flow preserves all registers except temporary storage locations `SAVEA` and `SAVEJ`, which are restored before execution continues.

## Verification

The record is generated before execution of `INST`, since all data is loaded prior to the jump to `INST`. The location written in Word 1 is `PREG`, which is the address of the instruction about to be executed, hence it is correct for a pre-execution trace.

Register `JREG`, index registers, and `X` are not modified during construction of the record; only `AREG` is temporarily saved and restored, so the simulated machine state is preserved.

The comparison indicator and overflow contribution are combined additively into a single digit because the construction yields values in disjoint ranges ${0,1,2}$ and ${0,8}$, ensuring no ambiguity in Word 1 field `(3:3)`.

Each cycle produces exactly 10 words because the routine outputs Word 1 through Word 10 once, and no additional output statements occur in the cycle.

Tape output order follows program order, so consecutive cycles produce consecutive 10-word groups, filling each 100-word block with ten such groups.

## Notes

The modification isolates all output activity into a single entry point at the start of the execution phase, which preserves the structure of the original trace loop while minimizing interference with register usage.

An alternative design places the formatting of Word 1 into a dedicated subroutine that constructs fields incrementally, but this increases control overhead without changing correctness or output structure.
