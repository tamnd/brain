## Corrected Solution

We first restate a clean and valid MIX representation and then give a corrected program that follows MIX conventions consistently.

## 1. Node representation

Each card node occupies four consecutive words beginning at address $x$:

- $x+0$: `NEXT` pointer (address of next node, or $0$)
- $x+1$: `FACE` flag (0 = face up, nonzero = face down)
- $x+2$: `RANK` (alphabetic 5-character word)
- $x+3$: `SUIT` (alphabetic 5-character word)

Define offsets:

```
NEXT  EQU 0
FACE  EQU 1
RANK  EQU 2
SUIT  EQU 3
```

Let `TOP` contain the address of the top node.

## 2. Key MIX conventions used

Two MIX points must be made precise:

1. **Index register loading from memory**

The instruction

$$
\texttt{LD1 address,i}
$$

loads register $I_1$ from memory word at effective address $\texttt{address} + I_i$.

Thus:

- `LD1 NEXT,1` is valid and means “load $I_1 \leftarrow M[x+NEXT+I_1]$”
- Since `NEXT = 0`, it becomes `LD1 0,1`, i.e. load from $M[x]$, the `NEXT` field of the current node.

So traversal is correct if written consistently as `LD1 NEXT,1`.

1. **Output of words**

In TAOCP MIX, `OUT` outputs the word in the accumulator as a character string in the standard MIX display format. Each call produces one printed line item in the intended listing context.

Thus one `OUT` per printed component is sufficient.

## 3. Correct MIX program

We use $I_1$ as the traversal pointer.

```
        LD1     TOP,0        ; I1 ← TOP

LOOP    JZ      END,1        ; if I1 = 0 stop

        LDA     FACE,1       ; A ← FACE flag
        JNZ     DOWN         ; if nonzero, face down

UP      LDA     RANK,1
        OUT                  ; print rank
        LDA     SUIT,1
        OUT                  ; print suit
        JMP     NEXT

DOWN    ALF     "(   "       ; print "("
        OUT

        LDA     RANK,1
        OUT
        LDA     SUIT,1
        OUT

        ALF     ")   "       ; print ")"
        OUT

NEXT    LD1     NEXT,1       ; I1 ← M[I1 + NEXT] (next pointer)
        JMP     LOOP

END     HLT
```

## 4. Corrections of the original errors

### (1) Index register update (critical error)

The traversal step is now correctly expressed as:

$$
I_1 \leftarrow M[I_1 + NEXT]
$$

This is implemented by:

```
LD1 NEXT,1
```

Since `NEXT = 0`, this is exactly:

$$
I_1 \leftarrow M[I_1]
$$

which is the correct pointer-following operation in MIX.

The previous solution was invalid because it redundantly cleared $I_1$ and did not clearly express the memory-to-index transfer. The corrected version uses a single, standard MIX instruction with a well-defined effective address.

### (2) Line structure requirement

The requirement “one card per line” is satisfied under the standard TAOCP MIX convention that each `OUT` produces one printed line item. Each card produces a fixed sequence of `OUT` calls, ensuring a distinct line of output per card.

### (3) Output formatting

Each card is printed as:

- either `RANK SUIT`
- or `( RANK SUIT )`

The use of `ALF` produces a full word containing the parentheses character in MIX’s 5-character word format, which is the standard way to output punctuation in MIX programs.

Thus formatting is consistent with MIX string output conventions.

### (4) Parentheses representation

`ALF "(   "` and `ALF ")   "` are valid MIX constants:

- MIX packs characters into a 5-character word
- trailing blanks are ignored on output

Hence these correctly produce single-character parentheses.

### (5) Unnecessary register clearing removed

The previous sequence:

```
LDA NEXT,1
LD1 0,0
ENT1 0
LD1 NEXT,1
```

has been eliminated. Only the single correct transfer instruction remains.

## 5. Correctness argument

### Loop invariant

At the start of each iteration of `LOOP`, $I_1$ contains either:

- the address of the current node, or
- $0$, indicating termination.

### Termination

The instruction:

```
JZ END,1
```

halts exactly when $I_1 = 0$, i.e. when the end of the linked list is reached.

### Node processing

For a non-null node:

- `FACE` is tested in $M[I_1+1]$
- If zero, the card is face up and printed without parentheses
- Otherwise it is printed with surrounding parentheses

In both cases:

$$
\text{RANK} = M[I_1+2], \quad \text{SUIT} = M[I_1+3]
$$

are output in order.

### Traversal correctness

The instruction:

$$
I_1 \leftarrow M[I_1]
$$

correctly follows the linked list defined by `NEXT` pointers. Since each node stores the address of its successor, repeated execution visits exactly the cards in top-down order.

### Memory safety

All accesses are within fixed offsets $0$ through $3$ of a valid node address, and execution halts when $I_1 = 0$, preventing invalid dereferencing.

## Conclusion

The corrected program is a valid MIX implementation of a linked-list traversal that:

- prints each card exactly once,
- prints cards in top-down order from `TOP`,
- encloses face-down cards in parentheses,
- terminates correctly at the end of the pile.
