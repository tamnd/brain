**Exercise 4.** [*18*] Give an algorithm analogous to Algorithm A, except that it puts the new card *face down* at the *bottom* of the pile. (The pile may be empty.)

**Verified:** yes  
**Solve time:** 1m21s  

---

## Solution

Let $TOP$ denote the address of the first card in the pile, and let $NEXT(X)$ denote the link field of card $X$, with value $0$ meaning “no successor.” Let $NEWCARD$ be the address of a newly allocated card whose fields are initially undefined.

The required operation places $NEWCARD$ at the bottom of the pile, preserving the existing order.

Define the following algorithm.

**Algorithm B** (Insert at bottom of pile, face down).

**B1.** If $TOP = 0$, set $NEXT(NEWCARD) \leftarrow 0$, set $TOP \leftarrow NEWCARD$, and terminate.

**B2.** Set $NEXT(NEWCARD) \leftarrow 0$, set $P \leftarrow TOP$.

**B3.** If $NEXT(P) = 0$, set $NEXT(P) \leftarrow NEWCARD$ and terminate.

**B4.** Set $P \leftarrow NEXT(P)$ and go to B3.

The correctness of the algorithm rests on the fact that the iteration in B3–B4 advances $P$ along the unique chain

$$
TOP, \ NEXT(TOP), \ NEXT(NEXT(TOP)), \ldots
$$

until reaching the unique node $P$ such that $NEXT(P)=0$, which is exactly the bottom card of the pile when $TOP \ne 0$. In the empty case $TOP=0$, B1 creates a one-element list whose next pointer is correctly set to $0$, so the single card is simultaneously top and bottom. In the nonempty case, B2 initializes $P$ at the top and preserves the invariant that $P$ is always a node on the existing list; B3 halts precisely when $P$ is the last node, at which point attaching $NEWCARD$ via $NEXT(P) \leftarrow NEWCARD$ extends the list without altering any existing links.

This completes the construction of an algorithm that inserts a new card at the bottom of the pile. ∎
