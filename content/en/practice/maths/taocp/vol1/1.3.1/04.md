[Section 1.3.1: Description of MIX](../)

**Exercise 4.** [*00*] The last example in (5) is `LDA -2000,4`. How can this be legitimate, in view of the fact that memory addresses should not be negative?

**Verified:** yes  
**Solve time:** 1m10s  

---

The sign is part of the address field in a MIX instruction word, so the quantity written as $\pm AA$ may be negative even though memory locations themselves are numbered $0$ through $3999$.

In `LDA -2000,4`, the value stored in the address field is $-2000$. During execution, the effective address is formed as

$$
M = -2000 + rI4.
$$

This computation is performed before any memory reference is made. If the resulting value $M$ lies between $0$ and $3999$, the instruction refers to the corresponding memory cell in the usual way. If it does not lie in this range, the definition of $M$ is explicitly left undefined by the rules of indexing in Section 1.3.1.

Thus the negativity refers only to the encoding of the instruction, not to a memory location itself. The address field is a signed quantity, while valid memory references are determined only after indexing produces $M$. This permits negative intermediate addresses in instruction representation without contradicting the restriction that memory locations are nonnegative. ∎
