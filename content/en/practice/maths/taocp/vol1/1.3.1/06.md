**Exercise 6.** [*10*] Assume that location 3000 contains

```text
+ 5 1 200 15
```

What is the result of the following instructions? (State if any of them are undefined or only partially defined.) (a) `LDAN 3000`; (b) `LD2N 3000(3:4)`; (c) `LDX 3000(1:3)`; (d) `LD6 3000`; (e) `LDXN 3000(0:0)`.

**Verified:** yes  
**Solve time:** 3m51s  

---

We work in standard MIX conventions from TAOCP: each word consists of a sign and 5 bytes, and each byte must satisfy $0 \le \text{byte} \le 63$. Any memory word containing a byte outside this range is not a valid MIX word, and any operation that depends on such data is undefined or only partially defined.

### Given memory content

$$
\text{M}(3000) = +\,5\;1\;200\;15\;0
$$

Bytes are:

- byte 1 = 5
- byte 2 = 1
- byte 3 = 200
- byte 4 = 15
- byte 5 = 0

Since $200 \notin [0,63]$, location 3000 does **not contain a valid MIX word**.

Therefore, any instruction that reads any field involving byte 3 is operating on invalid data.

## (a) `LDAN 3000`

This instruction loads the full word $(0:5)$ into $rA$, then negates it.

However, the source word contains an invalid byte (200), so the loaded value is not a valid MIX number representation. Although the sign and field positions are syntactically defined, the magnitude is not representable as a MIX value.

**Conclusion:** the operation is **undefined** in MIX terms.

## (b) `LD2N 3000(3:4)`

Field $(3:4)$ consists of bytes:

$$
(200,\;15)
$$

Since byte 3 is invalid, the extracted field does not correspond to any valid MIX value.

Negation is therefore applied to an undefined magnitude.

**Conclusion:** **undefined**.

## (c) `LDX 3000(1:3)`

Field $(1:3)$ consists of:

$$
(5,\;1,\;200)
$$

Again, byte 3 is invalid. Even though the field extraction rule is syntactically clear, the resulting value is not a valid MIX number.

**Conclusion:** **undefined**.

## (d) `LD6 3000`

This loads the full word $(0:5)$ into index register $rI6$, which can only hold a sign and two bytes.

Since the source word is already invalid (contains byte 200), the loaded value is not a valid MIX quantity. Any truncation rule assumes a valid sequence of bytes, which is not satisfied here.

**Conclusion:** **undefined**.

## (e) `LDXN 3000(0:0)`

Field $(0:0)$ refers only to the sign of the word.

The sign of location 3000 is $+$, which is well-defined and does not depend on byte values.

Thus:

$$
\text{field} = +0
$$

`LDXN` negates this value, giving:

$$
rX \leftarrow -0
$$

This operation does not access any invalid byte, so it is fully well-defined.

**Conclusion:** **defined**, with result $rX = -0$.

## Final Answers

- **(a)** undefined
- **(b)** undefined
- **(c)** undefined
- **(d)** undefined
- **(e)** $rX \leftarrow -0$ (defined)
