**Exercise 16.** [*20*] Write a program that sets memory cells `0000`-`0099` all to zero and is (a) as short a program as possible; (b) as fast a program as possible. [Hint: Consider using the `MOVE` command.]

The previous solution fails because it builds the argument around a nonexistent need to construct a zero word and around an incorrect use of `MOVE`. Once the actual MIX instruction set is used correctly, the structure of the solution changes completely.

## 1. Correct primitive operation

MIX includes the instruction

$$
\text{STZ } A
$$

which stores a zero word directly into memory location $A$. This eliminates the need to construct a zero in a register or to propagate a seed value.

So the fundamental operation for this problem is simply:

- one `STZ` per memory cell that must be cleared, or
- a loop that repeatedly executes `STZ` with indexed addressing.

## 2. Correct way to address 0000–0099

Let index register $X$ be used as a counter from $0$ to $99$. Then the effective address

$$
0000 + X
$$

ranges exactly over the required block.

We assume the standard MIX indexed form:

$$
\text{STZ } 0000,3
$$

meaning “store zero at address $0000 + X_3$”.

## 3. Correct program (loop form)

We use:

- $X_3$ as loop index
- initialize $X_3 = 100$
- decrement to 0

### MIX program

```
ENTA 100
STX  1003
L3:   STZ 0000,3
      DEC3 1
      J3P  L3
HLT
```

## 4. Correctness proof

### Initialization

- `ENTA 100` sets register $A = 100$.
- `STX 1003` stores $100$ into index register $X_3$.

Thus $X_3 = 100$.

### Loop invariant

At the start of each iteration of label $L3$:

- $X_3 = k$, where $0 \le k \le 100$
- Locations $0000$ through $0099-k$ have been set to zero.

### Loop step

Each iteration executes:

1. `STZ 0000,3`

This stores zero at address $0000 + X_3$.
2. `DEC3 1`

Decrements $X_3$ by 1.
3. `J3P L3`

If $X_3 > 0$, repeat.

Thus exactly one new memory cell is zeroed per iteration.

### Termination

When $X_3 = 0$, all addresses

$$
0000, 0001, \dots, 0099
$$

have been set to zero, and the program halts.

## 5. Correct optimality analysis

### Key correction to previous argument

The previous solution incorrectly claimed:

- a zero value must be constructed
- propagation via `MOVE` is required

This is false because:

- `STZ` already writes zero directly
- `MOVE` is not needed for initialization or broadcasting

### Lower bound

Each of the 100 memory cells must be written at least once.

Since MIX has no instruction that writes more than one independently addressed memory location with guaranteed zeroing semantics, any correct program must perform at least 100 effective stores.

Thus:

$$
\text{at least } 100 \text{ zero-store operations are necessary.}
$$

## 6. (a) Shortest program

To minimize instruction count, we must minimize loop overhead.

A standard minimal MIX loop requires:

- load constant 100
- initialize index register
- loop body with `STZ`, decrement, and conditional jump

This yields a tight implementation:

```
ENTA 100
STX  1003
L3:   STZ 0000,3
      DEC3 1
      J3P  L3
HLT
```

This is minimal in structure because:

- removing initialization requires assuming register state (not allowed)
- removing the loop requires 100 explicit `STZ` instructions
- any alternative loop still needs at least these three operations

So the shortest correct program uses 5 instructions (excluding label/HLT formatting variations).

## 7. (b) Fastest program

The fastest program has the same structure.

Reason:

- Each memory cell must be explicitly written
- `STZ` is already the minimal-cost write operation
- loop overhead is unavoidable for a compact program
- unrolling the loop increases instruction fetch and execution time significantly

Thus the optimal time solution is the same indexed `STZ` loop.

## 8. Final corrected answer

### Shortest and fastest correct MIX program

```
ENTA 100
STX  1003
L3:   STZ 0000,3
      DEC3 1
      J3P  L3
HLT
```

## 9. Summary of corrections

- `STZ` directly writes zero, eliminating the need for construction
- `MOVE` is unnecessary and was incorrectly applied in the previous solution
- the problem reduces to a counted loop of 100 stores
- optimality is based on the unavoidable need to write 100 memory locations

This fully corrects both the construction and the optimality argument.
