[Section 1.3.1: Description of MIX](../)

**Exercise 12.** [*10*] Find a single instruction that has the effect of multiplying the current contents of `rI3` by two and leaving the result in `rI3`.

**Verified:** yes  
**Solve time:** 3m42s  

---

We restart from the MIX instruction semantics rather than attempting to reinterpret the field specification.

### Correct analysis

In MIX, the instruction

$$
\texttt{INC } i, M(F)
$$

has the precise effect

$$
rI_i \leftarrow rI_i + M(F),
$$

where $M(F)$ denotes a field extracted from a memory word. The field specification $F$ applies only to memory words; it does not access registers.

Thus:

- The destination is always a single index register $rI_i$.
- The source operand is always a value derived from memory.
- No MIX instruction provides two independent read accesses to the same index register within a single instruction.

### Critical consequence

To compute

$$
rI_3 \leftarrow 2 \cdot rI_3,
$$

an instruction would need to supply the value $rI_3$ twice: once as the value being added, and once as the accumulator. However, in MIX:

- No instruction can use $rI_3$ as a source operand.
- No field specification can extract index register contents.
- Therefore, no instruction can form the expression $rI_3 + rI_3$ directly.

The only possible source of data for `INC` is memory, not registers.

### Exhaustion of possibilities

All instructions that modify $rI_3$ fall into one class:

- `INC 3, M(F)` adds a memory-derived value.
- `DEC 3, M(F)` subtracts a memory-derived value.
- `ENT 3, M(F)` loads a memory-derived value.
- `ENNa` instructions similarly load from memory only.

None of these instructions can read $rI_3$ as an operand.

Since MIX has no instruction that both:

1. Reads $rI_3$, and
2. Writes a function of it back into $rI_3$,

it is impossible to form a single-instruction doubling.

### Conclusion

There is **no single MIX instruction** whose effect is

$$
rI_3 \leftarrow 2 \cdot rI_3.
$$

Any correct implementation requires at least two instructions (for example, storing $rI_3$ to memory and then adding it back), but a one-instruction solution does not exist in the MIX architecture as defined in TAOCP §1.3.
