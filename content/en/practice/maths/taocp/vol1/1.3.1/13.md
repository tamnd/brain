**Exercise 13.** [*10*] Suppose location 1000 contains the instruction `JOV 1001`. This instruction turns off the overflow toggle if it is on (and the next instruction executed will be in location 1001, in any case). If this instruction were changed to `JNOV 1001`, would there be any difference? What if it were changed to `JOV 1000` or `JNOV 1000`?

**Verified:** no  
**Solve time:** 3m36s

Let the instruction at location $1000$ be either $\mathrm{JOV}\ 1001$, $\mathrm{JNOV}\ 1001$, $\mathrm{JOV}\ 1000$, or $\mathrm{JNOV}\ 1000$. The next sequential location after $1000$ is $1001$.

Recall MIX semantics:

- $\mathrm{JOV}\ M$: if overflow toggle is 1, transfer to $M$ and set overflow to 0; otherwise proceed to the next instruction.
- $\mathrm{JNOV}\ M$: if overflow toggle is 0, transfer to $M$; otherwise proceed to the next instruction. Overflow toggle is unchanged.

Only the behavior under iteration can determine whether execution terminates or loops.

## 1. Replace by $\mathrm{JOV}\ 1001$

At location $1000$:

- If overflow is 1: transfer to $1001$, overflow is set to 0.
- If overflow is 0: fall through to $1001$.

In both cases execution continues at $1001$.

There is no possibility of repetition of location $1000$, since neither branch returns to $1000$.

**Conclusion:** No change in control flow relative to the original instruction. The only effect is that overflow is cleared exactly when it is 1 upon entry to $1000$.

## 2. Replace by $\mathrm{JNOV}\ 1001$

At location $1000$:

- If overflow is 0: transfer to $1001$.
- If overflow is 1: fall through to $1001$.

In both cases execution continues at $1001$, and overflow is unchanged.

**Conclusion:** No change in control flow relative to case 1 or the original instruction, except that overflow is never altered.

## 3. Replace by $\mathrm{JOV}\ 1000$

Consider repeated execution.

### Case A: overflow initially 0

- $\mathrm{JOV}\ 1000$ does not jump.
- Execution proceeds immediately to $1001$.

### Case B: overflow initially 1

- At $1000$, control transfers to $1000$ and overflow is cleared to 0.
- On the next execution of $1000$, overflow is now 0, so the instruction falls through to $1001$.

Thus the system behaves as follows:

- If overflow is initially 1: the instruction at $1000$ executes twice, then control moves to $1001$.
- If overflow is initially 0: control moves to $1001$ immediately.

**Conclusion:** The instruction may repeat at most once. After at most one iteration, overflow becomes 0, guaranteeing termination of the self-loop.

## 4. Replace by $\mathrm{JNOV}\ 1000$

Consider repeated execution.

### Case A: overflow initially 0

- At $1000$, $\mathrm{JNOV}\ 1000$ transfers control to $1000$.
- Overflow remains 0.
- The same condition holds again at the next execution.

Hence execution repeats indefinitely at $1000$.

This produces an **infinite loop at location $1000$**.

### Case B: overflow initially 1

- $\mathrm{JNOV}\ 1000$ does not jump.
- Control falls through to $1001$.
- Overflow remains 1.

Thus execution continues at $1001$ with no repetition.

**Conclusion:** This case is qualitatively different from the others. Depending on the initial overflow state, execution either

- enters an infinite loop at $1000$ (if overflow is 0), or
- proceeds to $1001$ immediately (if overflow is 1).

## Final conclusion

- $\mathrm{JOV}\ 1001$ and $\mathrm{JNOV}\ 1001$ have identical control flow: execution always proceeds to $1001$. They differ only in whether overflow is cleared.
- $\mathrm{JOV}\ 1000$ may repeat once, but always eventually proceeds to $1001$; no non-termination occurs.
- $\mathrm{JNOV}\ 1000$ can cause non-termination: if overflow is initially 0, execution loops forever at $1000$; otherwise it proceeds to $1001$.
