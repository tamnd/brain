**Exercise 9.** [*15*] List all the MIX operators that can possibly affect the setting of the overflow toggle. (Do not include floating point operators.)

**Verified:** yes  
**Solve time:** 2m43s  

---

We restart from the MIX specification of the overflow toggle.

The overflow toggle is affected only by execution of instructions whose defined arithmetic result cannot be represented in the destination register or register pair. In MIX, this occurs only for arithmetic operations; it is not triggered by data movement or by loading immediate values.

We examine the integer (non-floating-point) instruction set.

## 1. Arithmetic instructions on the accumulator

The accumulator $rA$ holds a single MIX word. The following operations compute a new value from existing register contents and store it back into $rA$:

- **ADD $M$**: $rA \leftarrow rA + M$
- **SUB $M$**: $rA \leftarrow rA - M$

If the computed sum or difference is outside the representable range of a MIX word, the overflow toggle is set.

Thus, **ADD** and **SUB** can affect overflow.

## 2. Multiplication and division

- **MUL $M$**: $rAX \leftarrow rA \cdot M$

The product is formed in the double register $rAX$. If the resulting value does not fit the signed interpretation of the $2w$-byte product representation, overflow is set.

- **DIV $M$**: $rA \leftarrow rAX / M,\; rX \leftarrow rAX \bmod M$

Overflow is set if division is invalid (for example, division by zero) or if the quotient cannot be represented in $rA$.

Thus, **MUL** and **DIV** can affect overflow.

## 3. Increment and decrement instructions

For each register $r \in \{A, X, I_1, \dots, I_6\}$, MIX includes:

- **INCr**: $r \leftarrow r + M$
- **DECr**: $r \leftarrow r - M$

These are genuine arithmetic updates of fixed-size registers. If the result exceeds the representable range of the target register, overflow is set.

Thus, all **INC** and **DEC** instructions can affect overflow.

This includes:

- INC A, DEC A
- INC X, DEC X
- INC $I_i$, DEC $I_i$, $1 \le i \le 6$

## 4. Excluded instruction classes

We now justify completeness by excluding all other integer MIX operators.

### (a) Load immediate instructions

Instructions such as:

$$
\text{ENTA},\ \text{ENNA},\ \text{ENTX},\ \text{ENNX},\ \text{ENT}I_i,\ \text{ENN}I_i
$$

do not compute from existing register contents. They simply load a constant value into a register. Since overflow in MIX is defined as arising from arithmetic computation, not from assignment of constants, these instructions do **not** affect the overflow toggle.

### (b) Load/store, comparison, control, shifts, I/O

- Load/store instructions ($LDA, LDX, LD I_i, STA, STX, ST I_i$) transfer values without arithmetic.
- Comparison instructions ($CMPA, CMPX, CMP I_i$) only set the comparison indicator.
- Control transfer instructions do not produce arithmetic results.
- Shift instructions ($SLA, SRA, SLAX, SRAX, SLC, SRC$) move bits or bytes without arithmetic overflow in the MIX definition.
- Input/output instructions do not affect arithmetic state.

None of these can set overflow.

## 5. Complete list

Therefore, the integer MIX operators that can possibly affect the overflow toggle are exactly:

### Arithmetic on registers

- ADD
- SUB
- MUL
- DIV

### Increment/decrement family

For $r \in \{A, X, I_1, \dots, I_6\}$:

- INC $r$
- DEC $r$

## Final answer

$$
\boxed{
\text{ADD, SUB, MUL, DIV, INC } r, \text{ DEC } r \ (r \in \{A, X, I_1,\dots,I_6\})
}
$$

These are all and only the integer MIX operators that can affect the overflow toggle. ∎
