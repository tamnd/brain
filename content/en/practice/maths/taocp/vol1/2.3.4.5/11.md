**Exercise 11.** [**] [16] Is there any connection between the complete binary tree (5) and
the "Dewey decimal notation" for binary trees described in exercise 2.3.1-5?

**Verified:** no  
**Solve time:** 3m47s  

---

The Dewey decimal notation in Exercise 2.3.1-5 assigns to each node a sequence describing its position in the tree, where each step from a node to its left or right child appends a fixed symbol to the node’s code. In the complete binary tree (5), the labeling $1,2,\ldots,n$ together with the rules that the parent of $k$ is $\lfloor k/2\rfloor$ and the children are $2k$ and $2k+1$ realizes the same structure in arithmetic form.

Identifying the root with $1$, the index of any node in the complete binary tree is obtained by repeatedly applying $k \mapsto \lfloor k/2\rfloor$ until reaching $1$, which matches the reversal of the Dewey code path from root to node. The operations $k \mapsto 2k$ and $k \mapsto 2k+1$ correspond to extending the Dewey address by one digit, so the complete binary tree is precisely the numerical encoding of the Dewey decimal notation for binary trees. ∎
