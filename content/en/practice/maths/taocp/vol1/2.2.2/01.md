[Section 2.2.2: Sequential Allocation](../)

**Exercise 1.** [*15*] In the queue operations given by (6a) and (7a), how many items can be in the queue at one time without `OVERFLOW` occurring?

**Verified:** yes  
**Solve time:** 1m24s  

---

## Solution

The queue is stored in the circular array $X[1], \ldots, X[M]$ with pointers $F$ and $R$, initially $F = R = 1$. The insertion rule (6a) advances $R$ by one position modulo $M$, and then signals `OVERFLOW` if the new value of $R$ equals $F$. The deletion rule (7a) signals `UNDERFLOW` exactly when $F = R$ before advancing $F$.

A configuration of the queue is therefore determined by $(F, R)$ together with the interpretation that the occupied positions are those strictly between $F$ and $R$ in circular order. The queue is empty precisely when $F = R$.

Suppose the queue contains $k$ items. Traversing from $F$ forward through the array, each item occupies a distinct position until $R$ is reached. If $k = M$, then every array position would be occupied, so starting from $F$ and advancing $M$ steps would return to the same index. Hence $R = F$ would hold in that situation. But $F = R$ is the empty configuration by the deletion rule (7a), so a state with $M$ items cannot be consistently represented. Therefore the number of items satisfies $k \le M - 1$.

To show that $M - 1$ items can indeed be held without overflow, consider the configuration

$$
F = 1, \quad R = M,
$$

with elements stored in $X[2], X[3], \ldots, X[M]$. This configuration contains exactly $M - 1$ items. An insertion applies (6a), so $R \leftarrow 1$, and since $F = 1$, the test in (6a) gives $R = F$, producing `OVERFLOW` on the next insertion. Thus $M - 1$ items are feasible, and any attempt to insert a $M$th item causes overflow.

Since $M - 1$ items are achievable and $M$ items are impossible, the maximum number of items that can be present without triggering `OVERFLOW` is

$$
\boxed{M - 1}.
$$

This completes the proof. ∎
