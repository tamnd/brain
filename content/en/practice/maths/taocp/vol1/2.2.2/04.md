**Exercise 4.** [*25*] Assume that MIX has been extended as in exercise 3. Show how to give a *single instruction* (plus auxiliary constants) for each of the following actions:

a) To loop indefinitely because indirect addressing never terminates.

b) To bring into register A the value `LINK(LINK(x))`, where the value of link variable $x$ is stored in the `(0:2)` field of the location whose symbolic address is $x$, the value of `LINK(x)` is stored in the `(0:2)` field of location $x$, etc., assuming that the `(3:3)` fields in these locations are zero.

c) To bring into register A the value `LINK(LINK(LINK(x)))`, under assumptions like those in (b).

d) To bring into register A the contents of location `rI1 + rI2 + rI3 + rI4 + rI5 + rI6`.

e) To quadruple the current value of `rI6`.

We adopt the MIX indirect addressing semantics from the extension in Exercise 3:

- An effective address $a$ may be modified by index additions $rI_i$.
- If the indirect modifier $7$ is present, the address field is replaced by the contents of the link field $(0:2)$.
- This replacement is part of the effective address evaluation itself, and it may be applied repeatedly as long as the resulting address again specifies indirect addressing.
- Thus effective address evaluation is a _semantic process_, not a syntactically bounded one, and it may fail to terminate if the induced link structure contains a cycle.

## (a) Infinite loop via indirect addressing

Let the instruction be

$$
\text{LDA } x \quad \text{with indirect addressing (modifier }7\text{)}.
$$

Let $a_0 = x$. Effective address evaluation performs:

$$
a_{k+1} = \operatorname{LINK}(a_k),
$$

whenever the current effective address specifies further indirection through the $(0:2)$ field.

If the memory configuration satisfies a cycle

$$
x \to \operatorname{LINK}(x) \to \operatorname{LINK}(\operatorname{LINK}(x)) \to \cdots \to x,
$$

then the sequence $(a_k)$ never reaches a terminal address.

Hence the effective address evaluation does not terminate, and the machine loops forever before execution of the load completes.

Thus a single instruction can cause nontermination purely through indirect addressing.

## (b) Computing $\operatorname{LINK}(\operatorname{LINK}(x))$

We use a single load instruction with repeated indirect evaluation:

$$
\text{LDA } x \quad (\text{with indirect addressing }7:7).
$$

Evaluation proceeds stepwise:

$$
x \;\to\; \operatorname{LINK}(x) \;\to\; \operatorname{LINK}(\operatorname{LINK}(x)).
$$

Since each application of modifier $7$ performs one level of link following, two applications yield exactly the second iterated link. The value in register $A$ becomes:

$$
A \leftarrow \operatorname{LINK}(\operatorname{LINK}(x)).
$$

## (c) Computing $\operatorname{LINK}(\operatorname{LINK}(\operatorname{LINK}(x)))$

We extend the same mechanism to three levels of indirection:

$$
\text{LDA } x \quad (\text{with indirect addressing }7:7:7).
$$

Effective address evaluation yields:

$$
x \to \operatorname{LINK}(x) \to \operatorname{LINK}(\operatorname{LINK}(x)) \to \operatorname{LINK}(\operatorname{LINK}(\operatorname{LINK}(x))).
$$

Therefore the single instruction loads into $A$:

$$
A \leftarrow \operatorname{LINK}(\operatorname{LINK}(\operatorname{LINK}(x))).
$$

There is no restriction limiting the number of successive indirect modifiers beyond the instruction format itself, so three nested applications are permitted.

## (d) Address $rI_1 + \cdots + rI_6$

We use a base address $0$ and apply all index modifications:

$$
\text{LDA } 0 \quad (1:2:3:4:5:6).
$$

Effective address computation:

$$
0 \mapsto 0 + rI_1 + rI_2 + rI_3 + rI_4 + rI_5 + rI_6.
$$

Hence register $A$ receives the contents of location

$$
rI_1 + rI_2 + rI_3 + rI_4 + rI_5 + rI_6.
$$

## (e) Quadrupling $rI_6$

We compute $4rI_6$ via repeated indexing in the effective address:

$$
\text{ENT6 } 0 \quad (6:6:6:6).
$$

Let the intermediate effective addresses be:

$$
a_0 = 0,\quad a_{k+1} = a_k + rI_6.
$$

Then:

$$
a_1 = rI_6,\quad a_2 = 2rI_6,\quad a_3 = 3rI_6,\quad a_4 = 4rI_6.
$$

Thus register $I_6$ is set to:

$$
rI_6 \leftarrow 4rI_6.
$$

## Summary of corrections

- (a) Indirect addressing may lead to nontermination when link pointers form a cycle.
- (b) Two successive indirect applications yield $\operatorname{LINK}(\operatorname{LINK}(x))$.
- (c) Three successive applications yield $\operatorname{LINK}(\operatorname{LINK}(\operatorname{LINK}(x)))$.
- (d) Indexed addressing composes additively across all six index registers.
- (e) Repeated indexing accumulates linearly, enabling multiplication by 4 of $rI_6$.
