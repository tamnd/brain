[Section 2.2.2: Sequential Allocation](../)

**Exercise 16.** [*20*] The text illustrates how two stacks can be located so they grow towards each other, thereby making efficient use of a common memory area. Can two *queues*, or a stack and a queue, make use of a common memory area with the same efficiency?

**Verified:** yes  
**Solve time:** 1m29s  

---

## Solution

Let the memory consist of locations $L_0 < L \le L_\infty$. For two stacks, a single partition point suffices because each stack modifies only one end of its occupied segment. If stack $1$ grows upward and stack $2$ grows downward, the invariant is that each stack occupies a contiguous block, and a single moving boundary between them preserves full utilization of memory. Each insertion or deletion changes only one endpoint, so no relocation of existing elements is required.

Consider now two queues $Q_1$ and $Q_2$ stored in the same memory. For each queue, both a front and a rear position are required. Let $F_1, R_1$ denote the front and rear of $Q_1$, and $F_2, R_2$ denote those of $Q_2$. Each insertion modifies a rear pointer, and each deletion modifies a front pointer.

If each queue is required to occupy a contiguous segment of memory at all times, then $Q_1$ occupies an interval $[F_1, R_1]$ and $Q_2$ occupies an interval $[F_2, R_2]$, disjoint except possibly at boundaries. The memory must then maintain four moving endpoints. Unlike the stack case, no reduction to two boundaries is possible, since both ends of each interval are active.

Suppose instead that a two-boundary “toward each other” scheme is attempted, so that all free space lies between two moving barriers and both queues grow into this region. Then whenever a deletion occurs at the front of a queue, the corresponding interval shortens at the lower end while leaving a gap inside its previous region. Preserving contiguity would require shifting remaining elements toward the front boundary to eliminate gaps. Such shifting may involve moving a number of elements proportional to the current queue length, so a single deletion cannot in general be performed in $O(1)$ time without additional structure.

If one allows noncontiguous storage using links or a free-list mechanism, both queues can share memory with full utilization, but then the representation is no longer a simple sequential allocation with moving boundaries as in (6a) and (7a). The simplicity of the two-stack scheme is lost, since availability must be tracked at the granularity of individual nodes rather than contiguous segments.

Therefore, unlike two stacks, two queues cannot be stored in a single sequential array with only two moving boundaries while preserving both full utilization of memory and constant-time queue operations without relocation or auxiliary structure. A stack and a queue together also require at least three active boundaries for contiguous storage, so the same limitation applies.

This completes the solution. ∎
