**Exercise 2.** [*22*] Generalize the method of (6a) and (7a) so that it will apply to any deque with fewer than $M$ elements. In other words, give specifications for the other two operations, "delete from rear" and "insert at front."

## Solution

We extend the circular array representation $X[1],\ldots,X[M]$ used in (6a) and (7a), with pointers $F$ (front) and $R$ (rear), where the queue is empty exactly when $F = R$. The same convention is kept: after any insertion, the index is advanced before testing overflow, and overflow occurs precisely when the next position would coincide with the opposite end.

The additional operations for a deque are defined by symmetry.

For insertion at the front, the front pointer must move backward in circular fashion before the new item is stored. Thus

$$
X \Leftarrow Y \text{ (insert at front): }
\begin{cases}
\text{if } F = 1,\ \text{then } F \leftarrow M,\ \text{otherwise } F \leftarrow F - 1;\\
\text{if } F = R,\ \text{then OVERFLOW;}\\
X[F] \leftarrow Y.
\end{cases}
$$

The first line performs the cyclic decrement of $F$ so that the new front position is created. The second line tests whether this move has caused the structure to fill completely, in which case no free location remains. The third line places the element at the newly created front position.

For deletion from the rear, the element currently at position $R$ must be removed before the rear pointer is moved backward. Thus

$$
Y \Leftarrow X \text{ (delete from rear): }
\begin{cases}
\text{if } F = R,\ \text{then UNDERFLOW;}\\
Y \leftarrow X[R];\\
\text{if } R = 1,\ \text{then } R \leftarrow M,\ \text{otherwise } R \leftarrow R - 1.
\end{cases}
$$

The first line detects the empty structure condition. The second line extracts the rear element prior to pointer modification. The third line performs the cyclic decrement of $R$, preserving the circular ordering of occupied positions.

These two specifications, together with (6a) and (7a), yield a complete implementation of a deque in a circular array of size $M$, in which at most $M-1$ elements can be stored without ambiguity between empty and full configurations. ∎
