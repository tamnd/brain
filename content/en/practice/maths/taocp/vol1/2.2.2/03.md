**Exercise 3.** [*21*] Suppose that MIX is extended as follows: The I-field of each instruction is to have the form $8I_1 + I_2$, where $0 \le I_1 \le 8$, $0 \le I_2 \le 8$. In assembly language one writes '`OP ADDRESS,I1:I2`' or (as presently) '`OP ADDRESS,I2`' if $I_1 = 0$. The meaning is to perform first the "address modification" $I_1$ on `ADDRESS`, then to perform the "address modification" $I_2$ on the resulting address, and finally to perform the `OP` with the new address. The address modifications are defined as follows:

0: $M = A$

1: $M = A + rI1$

2: $M = A + rI2$

$\ldots$

6: $M = A + rI6$

7: $M =$ resulting address defined from the '`ADDRESS,I1:I2`' fields found in location $A$. The case $I_1 = I_2 = 7$ in location $A$ is not allowed. (The reason for the latter restriction is discussed in exercise 5.)

Here $A$ denotes the address before the operation, and $M$ denotes the resulting address after the address modification. In all cases the result is undefined if the value of $M$ does not fit in two bytes and a sign. The execution time is increased by one unit for each "indirect-addressing" (modification 7) operation performed.

As a nontrivial example, suppose that location 1000 contains '`NOP 1000,1:7`'; location 1001 contains '`NOP 1000,2`'; and index registers 1 and 2 respectively contain 1 and 2. Then the command '`LDA 1000,7:2`' is equivalent to '`LDA 1004`', because

$$
1000,7:2 = (1000,1:7),2 = (1001,7),2 = (1000,2),2 = 1002,2 = 1004.
$$

a) Using this indirect addressing feature (if necessary), show how to simplify the coding on the right-hand side of (8) so that two instructions are saved per reference to the table. How much faster is your code than (8)?

b) Suppose there are several tables whose base addresses are stored in locations `BASE + 1`, `BASE + 2`, `BASE + 3`, ...; how can the indirect addressing feature be used to bring the Ith element of the Jth table into register A in one instruction, assuming that `I` is in `rI1` and `J` is in `rI2`?

c) What is the effect of the instruction '`ENT4 X,7`', assuming that the `(3:3)`-field in location $X$ is zero?

### (a)

Let (8) denote the standard MIX sequence that performs table access via a relocatable base and indexing, of the form

$$
\texttt{LD1 I},\quad
\texttt{LDA BASE},\quad
\texttt{STA *+1},\quad
\texttt{LDA *,1}.
$$

Its effect is to compute the address

$$
\text{BASE} + rI_1
$$

and then fetch the corresponding table entry. The inefficiency is that the effective address is constructed by self-modifying code: one instruction builds the modified instruction and another executes it.

In the extended MIX, the same effective address can be formed directly by composing addressing modifications.

Assume `BASE` contains the relocatable base address $B$. Then:

- modification $7$ replaces the address field by the contents of that location, i.e. $A \mapsto M[A]$,
- modification $1$ adds $rI_1$.

Thus the single instruction

$$
\texttt{LDA BASE,7:1}
$$

computes:

$$
A = B,\quad
A \xrightarrow{7} M[B],\quad
A \xrightarrow{1} M[B] + rI_1,
$$

which is exactly the effective address produced by (8) but without constructing and executing a modified instruction.

Hence, per table reference, the original code spends two instructions on address formation plus one on execution, while the new code uses only one instruction for both. Therefore:

- old cost per reference: 3 instructions,
- new cost per reference: 1 instruction,

so **2 instructions are saved per reference**. The saving is real because the self-modifying sequence is entirely eliminated.

### (b)

We are given:

- $I$ in $rI_1$,
- $J$ in $rI_2$,
- `BASE + j` contains the address of the $j$-th table.

We want:

$$
\text{contents}(\text{contents}(\text{BASE} + J) + I).
$$

#### Key constraint

Each instruction allows exactly two sequential modifications:

$$
I = 8I_1 + I_2,
$$

meaning the address is transformed by applying modification $I_1$, then $I_2$. Each modification is one of $0,\dots,7$.

Thus a single instruction can perform at most two of:

- add index register,
- indirect through memory,
- add another index register.

We need three conceptual operations:

1. add $J$,
2. indirect to get table base,
3. add $I$.

This cannot be done in one instruction because it requires three distinct address transformations, but only two are permitted.

#### Formal impossibility argument

Let the initial address be $A = \texttt{BASE}$. Any single instruction produces:

$$
A \xrightarrow{I_1} f_{I_1}(A) \xrightarrow{I_2} f_{I_2}(f_{I_1}(A)),
$$

where each $f_k$ is either:

- addition of a single index register,
- or a single indirection through memory.

Each such function introduces at most one of the required dependencies on $I$ or $J$, or one level of memory indirection.

To obtain

$$
\text{contents}(\text{contents}(\texttt{BASE}+J)+I),
$$

we require:

- dependence on $J$ to select a table,
- one indirection to obtain the table base,
- dependence on $I$ to index into the table.

These are three independent address transformations, but only two stages are available. Therefore no valid single MIX instruction exists that performs the full operation under the given restriction.

#### Correct optimal coding

The best achievable form is therefore a two-instruction sequence, for example:

1. obtain table base:

$$
\texttt{LDA BASE,2:7}
$$

This computes $\text{contents}(\texttt{BASE}+J)$.
2. then index by $I$:

$$
\texttt{LDA *,1}
$$

This achieves the required result in two instructions, and this is optimal under the model.

### (c)

Consider:

$$
\texttt{ENT4 X,7}.
$$

The instruction ENT loads index register $rI_4$ with the **effective address** of its operand.

1. Start with address field $X$.
2. Apply modification $7$: replace $X$ by $\text{contents}(X)$.

Thus the effective address is:

$$
EA = \text{contents}(X).
$$

Therefore:

$$
rI_4 \leftarrow \text{contents}(X).
$$

The condition that the $(3:3)$-field of $X$ is zero is irrelevant to the semantics of `ENT`. It does not affect address evaluation; modification $7$ already specifies a single level of indirection, and ENT uses only the resulting effective address.

### Final answers

- **(a)** `LDA BASE,7:1` replaces the self-modifying sequence and saves **two instructions per table reference**.
- **(b)** A single instruction cannot perform the required access because three independent address transformations are needed but only two are permitted; the correct solution requires at least two instructions.
- **(c)** `ENT4 X,7` sets $rI_4 = \text{contents}(X)$; the extra field condition is irrelevant.
