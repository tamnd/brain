**Exercise 5.** [*35*] The extension of MIX suggested in exercise 3 has an unfortunate restriction that "`7:7`" is not allowed in an indirectly addressed location.

a) Give an example to indicate that, without this restriction, it would probably be necessary for the MIX hardware to be capable of maintaining a long internal stack of three-bit items. (This would be prohibitively expensive hardware, even for a mythical computer like MIX.)

b) Explain why such a stack is not needed under the present restriction; in other words, design an algorithm with which the hardware of a computer could perform the desired address modifications without much additional register capacity.

c) Give a milder restriction than that of exercise 3 on the use of `7:7` that alleviates the difficulties of exercise 4(c), yet can be cheaply implemented in computer hardware.

### Corrected Solution

Let MIX effective address modification be governed by an $I$-field taking values $0,1,\dots,7$, where $0,\dots,6$ are ordinary modifications and $7$ denotes indirect addressing. In the extended model, a word may itself contain an $I$-field that requires further interpretation during effective-address evaluation.

The problematic situation is that a word reached by indirection may again demand indirection in a way that forces nested, unresolved evaluations. The symbol $7:7$ indicates precisely such a nested indirect dependency.

## (a) Necessity of an unbounded stack without the restriction

We construct a concrete family of MIX memory words that forces unbounded nesting of indirect address evaluation.

Let there be words

$$
X_0, X_1, X_2, \dots
$$

such that each $X_k$ has the following structure:

- Its $I$-field is $7$, so it demands indirect addressing.
- The address part of $X_k$ contains the address of $X_{k+1}$.

Thus the effective address evaluation starting from $X_0$ proceeds as:

$$
X_0 \to X_1 \to X_2 \to \cdots
$$

Now consider what the machine must do during evaluation.

When processing $X_k$, the machine encounters $I=7$. At this point:

- The current evaluation of the effective address of the original instruction is not complete, because the result of the indirect fetch will still need to be combined with whatever modification steps remain in the original instruction context.
- Therefore, the machine must suspend the current computation and remember its state, including:

- the partially processed instruction context,
- the pending modification sequence (encoded in part by a 3-bit $I$-field or equivalent control state),
- the return point after the indirect fetch completes.

It then continues evaluation at $X_{k+1}$, which may again require suspension.

Because the chain $X_0, X_1, \dots, X_k$ can be arbitrarily long, the number of suspended evaluation contexts can also be arbitrarily large. Each suspended context requires storage of a constant-size control item (3 bits for the $I$-field plus minimal control state).

Hence the hardware must support an unbounded pushdown structure of 3-bit items. In other words, effective address evaluation becomes a recursive process of unbounded depth, forcing an unbounded internal stack.

## (b) Why the restriction removes the need for a stack

Now assume the restriction that the configuration $7:7$ is not allowed in any indirectly accessed location.

We analyze effective-address evaluation more carefully.

During evaluation of a MIX effective address, the machine processes a sequence of modification steps determined by the $I$-field(s). The key issue is whether evaluation can require _nested suspension_: that is, whether a second indirect evaluation can begin before the first one has completed.

Under unrestricted rules, this can occur if a word reached by indirection again contains an indirect specification that itself triggers another suspension before the previous one resolves.

The restriction forbids exactly this situation in indirectly accessed words.

### Key consequence of the restriction

If a word $W$ is obtained by indirect addressing, then within $W$ there is no occurrence of a structure that would require _further indirect suspension before completion of the current effective-address evaluation phase_.

Thus:

- An indirect step may occur, but once it occurs, the resulting word does not itself initiate another unresolved indirect dependency chain during the same evaluation episode.
- Therefore, indirect evaluation does not nest.

### Evaluation algorithm without stack

The hardware can evaluate effective addresses using only a small fixed set of registers:

- $A$: current address being evaluated,
- $I$: current $I$-field,
- a small control bit indicating whether an indirect fetch is being executed.

Algorithm:

1. Fetch the word at address $A$, obtain its $I$-field.
2. If $I \in \{0,\dots,6\}$, apply the corresponding modification to $A$, and proceed to the next modification step.
3. If $I = 7$, replace $A$ by the address field of the fetched word and continue.
4. Because of the restriction, after executing step 3, the newly fetched word cannot trigger another unresolved indirect suspension within the same evaluation phase. Hence the machine continues linearly without needing to remember multiple pending contexts.

### Why no stack is required

The crucial point is not that indirect addressing is absent, but that **indirect addressing cannot re-enter itself before the current evaluation step completes**.

Thus there is never a situation requiring the machine to store multiple “return points” for suspended indirect computations. At every moment:

- either a single indirect fetch is in progress, or
- a direct modification is being applied,

but never two nested indirect suspensions.

Hence a single working register suffices, and no stack is required.

## (c) A milder restriction

We now give a weaker condition than forbidding $7:7$ outright, while still preventing the stack difficulty.

### Milder restriction (bounded indirection depth per evaluation)

Allow all memory configurations, including those containing $7:7$, but impose a hardware rule on effective-address evaluation:

> During the evaluation of a single effective address, at most one indirect step is permitted. If an indirect reference has already been taken once, any further occurrence of $I=7$ encountered during that same evaluation is treated as a no-op (or equivalently, as $I=0$).

### Why this is a genuine restriction

- Words containing $7:7$ are still legal in memory.
- The restriction applies only at execution time during effective-address evaluation.
- No memory representation is disallowed.

Thus it is strictly weaker than forbidding $7:7$ syntactically.

### Why it eliminates stack growth

This rule enforces that effective-address evaluation has **depth at most one in indirect addressing**:

- The first $7$ may trigger an indirection.
- Any subsequent $7$ cannot trigger another nested suspension.

Therefore, no chain of suspended evaluation contexts can form. The computation is forced to remain iterative, and no pushdown storage is needed.

## Conclusion

- **(a)** Without restrictions, one can construct arbitrarily long chains of indirect references, each forcing suspension of the current evaluation context. This requires an unbounded stack of constant-size (3-bit) control items.
- **(b)** The restriction on $7:7$ in indirectly accessed locations prevents nested indirect suspensions, ensuring that evaluation never requires storing more than one active context. Hence a stack is unnecessary.
- **(c)** A weaker alternative is to allow all memory contents but enforce at runtime that at most one indirect step is effective per evaluation. This preserves expressiveness of memory while eliminating the need for stack-based evaluation.
