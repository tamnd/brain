**Exercise 8.** [*20*] True or false: Whenever line 010 of the simulator program is executed, we have $0 \le rI6 < \text{BEGIN}$.

## Solution

We are asked to determine the truth of the statement: _Whenever line 010 of the simulator program is executed, we have $0 \le rI6 < \text{BEGIN}$._ Line 010 reads the next instruction into the simulated A-register:

$\text{010: } LDA\ 0,6 \quad \text{(rA} \gets \text{memory[rI6])}.$

By the program, $rI6$ is defined as the location of the next instruction to execute. Its value is initially set in line 006:

$\text{006: } ENT6\ 0,$

so that $rI6 \gets 0$. Therefore, at the very first iteration, $rI6 = 0$, which satisfies $0 \le rI6 < \text{BEGIN}$, since $\text{BEGIN} = 3500$ is the start of the simulator program itself.

Next, we examine how $rI6$ changes. In line 012:

$\text{012: } INC6\ 1,$

$rI6$ is incremented by 1 after reading the instruction at the previous $rI6$. Therefore, $rI6$ always points to the _next instruction_ to simulate.

The simulated memory range is defined as locations $0$ through $\text{BEGIN}-1$, i.e., $0 \le rI6 \le \text{BEGIN}-1$, because the program explicitly places the simulator itself starting at location $\text{BEGIN} = 3500$. By line 008, the execution time of the previous instruction is stored, and by lines 010–012, the instruction at $rI6$ is read and $rI6$ is incremented. At no point does the simulator increment $rI6$ beyond the upper bound of the simulated memory.

All checks that could prevent an illegal memory reference occur either in the `MEMORY` subroutine (lines 126–134) or via explicit jump conditions, e.g., `J5N MEMERROR` and `JGE MEMERROR`. The simulator never fetches instructions from memory locations $\ge \text{BEGIN}$; any attempt to do so results in a jump to `MEMERROR`. Therefore, under normal execution, the invariant

$0 \le rI6 < \text{BEGIN}$

holds each time line 010 is executed.

Finally, we consider whether any instruction could alter $rI6$ in an unexpected way. Only branching instructions (`JMP`, conditional jumps) modify the program counter, but by the definition of `rI6` and the control structure, any such jump within the _simulated program_ still targets addresses in the range $0 \le rI6 < \text{BEGIN}$. Jumps to the simulator code at $\text{BEGIN}$ or higher are handled as part of the simulation itself and do not affect the value of `rI6` in the simulated memory space. Therefore, no execution path allows $rI6$ to exceed the bounds of the simulated memory when line 010 is reached.

Hence, the statement is **true**.

$\boxed{\text{True}}$

This completes the proof.
