## Solution

Let $M$ and $F$ denote the address and field of the instruction, already placed in $rI5$ and $rI3$ by the control routine, and let $X$ denote the index register contents stored in $XREG$. The MOVE instruction transfers a block of $F$ consecutive words between memory locations while updating $XREG$ to the final destination pointer.

The MIX specification for MOVE is as follows. If $F=0$, no transfer occurs. If $M \ge 0$, the words at locations $M+1, M+2, \dots, M+F$ are copied into locations $X+1, X+2, \dots, X+F$, and $XREG$ becomes $X+F$. If $M < 0$, the transfer proceeds in reverse order, copying $M+F, M+F-1, \dots, M+1$ into $X+F, X+F-1, \dots, X+1$, and $XREG$ becomes $X-F$. The reverse direction prevents overwrite when the source and destination blocks overlap.

Let $i$ be a loop counter in $rI4$, $S=M$ stored in a working register $rI2$, and $D=XREG$ stored in $rI6$.

### MOVE routine

```
MOVE    STJ  9F              save return link
        LD2  M               rI2 <- M
        LD6  XREG            rI6 <- X
        LD4  INST(3:3)       rI4 <- F
        J4Z  9F              if F = 0, exit

        LDA  M
        JGE  POS             branch on sign of M

* --------, M < 0 : reverse transfer --------, NEG     ENT1 0              i <- 0
        ENTA F
        INC1 1              i <- F, but use decrement loop via comparison

NEG1    LDA  F
        SUB  =1=
        STA  F              temporary decrement of counter
        LDA  M
        ADD  F
        INC5 1              rI5 <- M + F
        JMP  MEMORY         fetch word

        STA  TEMP           save magnitude
        STX  TEMPS          save sign

        LDA  XREG
        ADD  F
        INC5 1              rI5 <- X + F
        LDX  TEMPS
        LDA  TEMP
        JMP  STORE          store word

        LDA  F
        JNZ  NEG1

        LDA  XREG
        SUB  INST(3:3)
        STA  XREG
        JMP  9F

* --------, M ≥ 0 : forward transfer --------, POS     ENT1 0

POS1    LDA  M
        ADD  I1
        INC5 1              rI5 <- M + i
        JMP  MEMORY

        STA  TEMP
        STX  TEMPS

        LDA  XREG
        ADD  I1
        INC5 1              rI5 <- X + i
        LDX  TEMPS
        LDA  TEMP
        JMP  STORE

        INC1 1
        CMP1 INST(3:3)
        JLE  POS1

        LDA  XREG
        ADD  INST(3:3)
        STA  XREG

9H      JMP  *              exit
```

The forward case maintains the invariant that after $k$ iterations of the loop, the words at $M+1$ through $M+k$ have been copied to $X+1$ through $X+k$, and $XREG = X+k$. Each iteration increments both source and destination indices consistently, so no overwritten data is read.

The reverse case preserves correctness under overlap by starting at $M+F$ and $X+F$ and moving downward. The loop invariant states that after $k$ iterations counting down from $F$, the words at $M+F, \dots, M+F-k+1$ have been copied to $X+F, \dots, X+F-k+1$, and $XREG = X-k$.

In both cases, termination occurs exactly when all $F$ words have been transferred, and $XREG$ is updated by $\pm F$ accordingly. This completes the construction of the MOVE routine. ∎
