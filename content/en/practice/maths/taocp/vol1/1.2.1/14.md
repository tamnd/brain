**Exercise 14.** [*50*] (R. W. Floyd.) Prepare a computer program that accepts, as input, programs in some programming language together with optional assertions, and that attempts to fill in the remaining assertions necessary to make a proof that the computer program is valid. (For example, strive to get a program that is able to prove the validity of Algorithm E, given only assertions *A1*, *A4*, and *A6*. See the papers by R. W. Floyd and J. C. King in the IFIP Congress proceedings, 1971, for further discussion.)

**Verified:** yes  
**Solve time:** 2m51s

A correct solution must avoid any assumption that Floyd verification conditions are decidable by semantic truth tables. The proper approach is to separate:

1. generation of candidate annotations, and
2. formal proof search for verification conditions in a fixed deductive system.

The result is a **dovetailed semi-decision procedure**, not a decision procedure.

## Correct Solution

### 1. Formal model of programs and assertions

Represent a program as a finite flow graph $G = (V,E)$. Each node $v \in V$ carries a command:

- assignment $x \leftarrow E$,
- conditional with guard $B$,
- or skip.

Each node $v$ is associated with an assertion $P_v$, a formula in a fixed first-order language $L$ over program variables (for example arithmetic over integers with equality and order).

The Floyd verification conditions are standard:

- **Assignment edge** $u \to v$, $x \leftarrow E$:

$$
P_u \Rightarrow P_v[x \leftarrow E].
$$
- **Conditional node** $u$ with guards $B$, $\neg B$:

$$
P_u \Rightarrow (B \Rightarrow P_{v_1}) \land (\neg B \Rightarrow P_{v_2}).
$$
- **Initial/final conditions** as usual.
- **Loops** are handled by invariants placed at loop entry nodes, with preservation conditions encoded as verification conditions of the same form.

Thus correctness reduces to the validity of a finite set of first-order formulas.

### 2. Key correction: validity is not decided semantically

The previous solution incorrectly assumed that implication between assertions can be checked by truth tables. This is false because:

- assertions are first-order formulas over infinite structures (e.g. integers),
- validity of such formulas is not decidable in general.

Therefore, the algorithm must not attempt semantic validity checking.

Instead, we replace it with:

### 3. Replacement: proof-theoretic checking

Fix once and for all a **formal deductive system** $\mathcal{D}$ for first-order logic (with whatever arithmetic theory is chosen, e.g. Peano Arithmetic or Robinson arithmetic).

We require only:

- proofs in $\mathcal{D}$ are finite strings,
- there is an effective procedure to check whether a string is a valid proof.

Thus:

$$
\text{“} \varphi \text{ is valid”} \quad \leadsto \quad \text{“there exists a proof of } \varphi \text{ in } \mathcal{D}”.
$$

This is a **recursively enumerable** notion, sufficient for a Floyd-style synthesis procedure.

### 4. Enumeration of candidate assertions

Construct a computable enumeration:

$$
A_1, A_2, A_3, \dots
$$

of all formulas in $L$, for example by enumerating all finite strings and filtering syntactically well-formed formulas.

A candidate annotation is a function:

$$
\alpha : V \to \{A_i\}.
$$

We enumerate all finite assignments $\alpha$ by standard dovetailing.

### 5. Verification condition checking via proof search

For a fixed candidate annotation $\alpha$, each Floyd verification condition becomes a concrete formula $\varphi_j(\alpha)$.

Instead of deciding whether $\varphi_j(\alpha)$ is true, we search for a formal proof.

We dovetail over all finite proof objects $\pi$ in $\mathcal{D}$ and check mechanically whether:

$$
\pi \text{ is a valid proof of } \varphi_j(\alpha).
$$

Since proof checking is decidable, this is effective.

Thus each condition is a **semi-decidable predicate**:

- it is confirmed when a proof is found,
- it may run forever otherwise.

### 6. Global dovetailed algorithm

The program performs a triple dovetailing:

1. over candidate annotations $\alpha_1, \alpha_2, \dots$,
2. over verification conditions within each annotation,
3. over candidate proofs in $\mathcal{D}$.

Procedure:

- For $s = 1, 2, 3, \dots$:

- enumerate first $s$ annotations,
- for each annotation $\alpha_i$, consider first $s$ verification conditions,
- for each such condition, search among proofs of length $\le s$,
- if all verification conditions of some $\alpha_i$ have certified proofs, output $\alpha_i$ and halt.

### 7. Correctness argument

**Soundness of checking step.**

If a proof $\pi$ is accepted, then by correctness of the proof system $\mathcal{D}$, the corresponding verification condition is logically valid.

**Completeness of search.**

If there exists a correct Floyd annotation $\alpha^\*$ such that every verification condition is provable in $\mathcal{D}$, then:

- $\alpha^\*$ appears in the enumeration,
- each required proof appears at some finite stage of the dovetailing,
- hence the algorithm eventually halts and outputs $\alpha^\*$.

### 8. What this fixes from the flawed solution

- Removes all reliance on truth-table or propositional reasoning.
- Avoids any assumption of decidability of first-order validity.
- Replaces semantic implication checking with syntactic proof search.
- Ensures every step is recursively enumerable and mechanically checkable.
- Restores the intended Floyd–King style synthesis as an enumeration of annotations plus proofs of verification conditions.

### Conclusion

The program is a dovetailed search over:

1. all possible assertion labelings of the flow graph, and
2. all formal proofs of their Floyd verification conditions in a fixed deductive system.

It halts exactly when it finds a complete annotation whose correctness is provable. This correctly captures Floyd’s specification without assuming any decidability of first-order logic. ∎
