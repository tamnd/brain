**Exercise 20.** [*24*] Algorithm `T` uses `F`, `R`, and the `QLINK` table to obtain the effect of a queue that contains those nodes whose `COUNT` field has become zero but whose successor relations have not yet been removed. Could a stack be used for this purpose instead of a queue? If so, compare the resulting algorithm with Algorithm `T`.

**Verified:** yes  
**Solve time:** 1m10s  

---

## Solution

Algorithm $T$ maintains a collection of nodes whose $\text{COUNT}$ field has just become $0$ but whose successor arcs have not yet been fully processed. The queue represented by $(F, R, \text{QLINK})$ enforces first-in-first-out removal of such nodes.

Replace the queue by a stack by introducing a single pointer variable $T$ for the top of the stack and using the same link field $\text{QLINK}$ as the stack link. The fundamental change is that insertion uses

$$
\text{QLINK}(P) \leftarrow T,\qquad T \leftarrow P,
$$

while deletion uses

$$
P \leftarrow T,\qquad T \leftarrow \text{QLINK}(P).
$$

In Algorithm $T$, every node is inserted exactly when its $\text{COUNT}$ first becomes $0$, and each inserted node is removed exactly once to have its successors processed. Replacing the queue by a stack preserves this one-time insertion and one-time deletion invariant, since neither correctness condition depends on the order in which eligible nodes are processed.

The effect on the main loop is that a node whose $\text{COUNT}$ becomes $0$ is placed on top of the stack immediately instead of being appended at the rear of a queue. Consequently, the next node to be processed is always the most recently activated one, rather than the earliest activated one.

Correctness is unchanged because the algorithm only requires that a node be processed after all decrement operations affecting its $\text{COUNT}$ have occurred. Once $\text{COUNT} = 0$, no later step increases it, so the moment of removal from the auxiliary structure is unconstrained with respect to future correctness. Therefore any discipline that ensures each zero-$\text{COUNT}$ node is eventually removed exactly once yields the same final processing of all nodes.

The resulting algorithm differs from Algorithm $T$ only in traversal order. The queue version processes nodes in increasing order of discovery time of the event $\text{COUNT} \leftarrow 0$, producing a breadth-first style propagation through the dependency structure. The stack version processes most recently discovered zero-$\text{COUNT}$ nodes first, producing a depth-first style propagation.

Both versions perform the same sequence of decrement operations on $\text{COUNT}$ fields and the same tests for the condition $\text{COUNT} = 0$. The set of nodes eventually removed from the auxiliary structure is identical, while the order of their removal differs according to FIFO versus LIFO discipline.

This completes the solution. ∎
