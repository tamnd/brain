[Section 2.2.3: Linked Allocation](../)

**Exercise 7.** [*23*] Design an algorithm to "invert" a linked linear list such as (1), that is, to change its links so that the items appear in the opposite order. [If, for example, the list (1) were inverted, we would have `FIRST` linking to the node containing item 5; that node would link to the one containing item 4; etc.]. Assume that the nodes have the form (3).

**Verified:** yes  
**Solve time:** 1m02s

## Solution

Let $P$ be a pointer to the current node being processed, initially set to $FIRST$, let $Q$ be a pointer to the already reversed portion of the list, initially set to $\Lambda$, and let $R$ be an auxiliary pointer.

The transformation proceeds by repeatedly detaching the node at $P$ and inserting it at the front of the list headed by $Q$. The algorithm is:

$$
P \leftarrow FIRST,\qquad Q \leftarrow \Lambda.
\tag{1}
$$

$$
\text{while } P \ne \Lambda \text{ do}
$$

$$
R \leftarrow LINK(P),\qquad LINK(P) \leftarrow Q,\qquad Q \leftarrow P,\qquad P \leftarrow R.
\tag{2}
$$

$$
FIRST \leftarrow Q.
\tag{3}
$$

At any moment during execution of (2), the nodes originally reachable from $FIRST$ are partitioned into two disjoint segments determined by $P$ and $Q$. The pointer $Q$ references a list consisting exactly of the nodes already removed from the original list, and each of these nodes has its link field redirected to the preceding node in the original order. The pointer $P$ references the remaining unprocessed suffix, and $R$ preserves the original successor of $P$ before relinking occurs.

The loop invariant states that $Q$ is the head of a linked list containing exactly the nodes already processed, in reverse order of their appearance in the original list, and that following $Q$ by successive $LINK$ references reaches exactly the nodes processed so far until $\Lambda$, while $P$ is the head of the remaining original list with all original links unchanged except those already redirected.

Initialization holds because $Q=\Lambda$ corresponds to an empty reversed list and $P=FIRST$ represents the full original list. Each iteration preserves the invariant since the node $P$ is removed from the front of the unprocessed portion, its successor is stored in $R$, its link is redirected to the previously reversed portion $Q$, and then $Q$ is advanced to $P$ while $P$ advances to $R$. This maintains disjointness of processed and unprocessed segments while extending the reversed portion by one node in correct order reversal.

Termination occurs when $P=\Lambda$, at which point all nodes have been transferred into the structure headed by $Q$, which therefore contains all nodes in reversed order. Assignment (3) makes $FIRST$ point to this final head node.

This completes the proof. ∎
