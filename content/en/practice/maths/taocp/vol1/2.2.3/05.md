**Exercise 5.** [*24*] Operations (14) and (17) give the effect of a queue; show how to define the further operation "insert at front" so as to obtain all the actions of an output-restricted deque. How could the operation "delete from rear" be defined (so that we would have a general deque)?

**Solve time:** 2m19s

Let the linked representation be the standard TAOCP structure with node fields $\mathrm{INFO}$ and $\mathrm{LINK}$, and with pointers $F$ and $R$ to the front and rear of the queue. The free list is headed by $AVAIL$. The empty structure satisfies $F = R = \Lambda$.

Operations (14) and (17) implement insertion at rear and deletion at front, giving a queue. We now extend this to obtain:

1. an output-restricted deque by adding insertion at the front,
2. a general deque by adding deletion from the rear.

## Insert at front (output-restricted deque)

A new node $P$ is taken from the free list using the standard allocation step:

$$
P \leftarrow AVAIL,\qquad AVAIL \leftarrow \mathrm{LINK}(P).
$$

The value to be inserted is stored:

$$
\mathrm{INFO}(P) \leftarrow Y.
$$

The new node is linked in front of the current list. The pointer $F$ is not yet modified at this stage, so $\mathrm{LINK}(P)$ must use the old value of $F$:

$$
\mathrm{LINK}(P) \leftarrow F.
$$

Now update the external pointers. If the structure was empty, the new node is both front and rear; otherwise only the front pointer changes:

$$
\text{if } F = \Lambda \text{ then } R \leftarrow P \text{ else } \mathrm{(no change to } R\mathrm{)},
\qquad F \leftarrow P.
$$

This preserves the invariant that $F$ points to the first node and $R$ to the last node.

## Delete from rear (general deque)

If $F = \Lambda$, the structure is empty and underflow occurs.

### Case 1: single node ($F = R$)

The only node is removed and returned to the free list:

$$
Y \leftarrow \mathrm{INFO}(R),
$$

$$
\mathrm{LINK}(R) \leftarrow AVAIL,\qquad AVAIL \leftarrow R,
$$

$$
F \leftarrow \Lambda,\qquad R \leftarrow \Lambda.
$$

The assignment $\mathrm{LINK}(R) \leftarrow AVAIL$ is essential to preserve the free list.

### Case 2: at least two nodes ($F \ne R$)

Find the predecessor of the rear node:

$$
Q \leftarrow F,
$$

$$
\text{while } \mathrm{LINK}(Q) \ne R \text{ do } Q \leftarrow \mathrm{LINK}(Q).
$$

Now $Q$ is the node immediately preceding $R$. Remove the rear node and return it to the free list:

$$
Y \leftarrow \mathrm{INFO}(R),
$$

$$
\mathrm{LINK}(R) \leftarrow AVAIL,\qquad AVAIL \leftarrow R.
$$

Relink the structure and update the rear pointer:

$$
\mathrm{LINK}(Q) \leftarrow \Lambda,\qquad R \leftarrow Q.
$$

## Result

- Insertion at the front together with the original queue operations yields an output-restricted deque (insert at both ends, delete only at front).
- Adding deletion from the rear yields a general deque.

All operations preserve both the queue structure invariant and the integrity of the free list $AVAIL$.
