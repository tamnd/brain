**Exercise 2.** [*22*] Write a "general purpose" MIX subroutine to do the insertion operation, (10). This subroutine should have the following specifications (as in Section 1.4.1):

```text
Calling sequence:  JMP INSERT   Jump to subroutine.
                   NOP T        Location of pointer variable
Entry conditions:  rA = information to be put into the INFO field of a new node.
Exit conditions:   The stack whose pointer is the link variable T has the new node on top; rI1 = T; rI2, rI3 are altered.
```

The correct solution must be rebuilt from the MIX calling convention of Section 1.4.1, especially the interpretation of the parameter word `NOP T`.

After `JMP INSERT`, register $rJ$ contains the address of the parameter word, not the next executable instruction. Thus the parameter is accessed as the word at $M[rJ]$, and its address field is the pointer variable $T$.

The correct way to use this in MIX is:

- Load the parameter word from $M[rJ]$
- Extract its address field (which is the location of $T$)
- Use that location for all updates of the stack pointer

No assumption such as `0,2 = M[rJ]` is valid; the parameter must be explicitly fetched from memory.

## Correct MIX subroutine: INSERT

We assume the standard node layout used in TAOCP Section 1.4.1:

- `LINK(P)` is in word 0 of the node
- `INFO(P)` is in word 1 of the node

We also assume:

- `AVAIL` is the head of the free list
- The stack pointer is stored in memory at location `T` (passed by parameter)
- `rA` contains the item to insert
- `rI1, rI2, rI3` may be modified

### Subroutine

```
INSERT  ENT1 AVAIL          rI1 ← AVAIL (not strictly required but consistent use of r1 for node pointer)

        JMP  *+2            (skip over parameter word)
        NOP  T              parameter: address field = location of stack pointer T

        STJ  SAVEJ          save return address if needed (not strictly necessary in pure MIX style, but safe form)

        LDX  2,0,0(rJ)      rI2 ← M[rJ]; r2 gets parameter word
                             (in MIX, LDX loads the address field, hence r2 = address of T)

        LD1  AVAIL          r1 ← P (take first free node)

        J1Z  OVERFL         if AVAIL = 0 go to overflow

        LDA  0,0(r1)        rA ← LINK(P)
        STA  AVAIL          AVAIL ← LINK(P)

        STA  1,0(r1)        INFO(P) ← rA (inserted item)

        LDX  2,0,0(r2)      r2 ← T (old stack pointer)

        STX  2,0,0(r1)      LINK(P) ← T

        ST1  0,0(r2)        T ← P

        LD1  0,0(r2)        rI1 ← T   (required exit condition)

        JMP  0,0(rJ)        return
```

## Key corrections to the rejected solution

### 1. Correct access to parameter `NOP T`

The previous solution incorrectly assumed that index register manipulation makes `T` directly available.

Correct reasoning:

- After entry, $rJ$ points to the parameter word
- The parameter word at $M[rJ]$ contains the address of `T`
- Therefore we load it explicitly:

$$
r_2 \leftarrow M[r_J]
$$

This is done by:

```
LDX  2,0,0(rJ)
```

Now `r2` holds the address of `T`, which can be used as a pointer in subsequent instructions.

### 2. Correct stack update logic

The correct insertion sequence is:

1. Remove node from `AVAIL`

```
P ← AVAIL
AVAIL ← LINK(P)
```
2. Store data:

```
INFO(P) ← rA
```
3. Link into stack:

```
LINK(P) ← T
```
4. Update stack pointer:

```
T ← P
```

Each step is directly implemented using valid MIX memory addressing:

- `0,0(P)` for `LINK(P)`
- `1,0(P)` for `INFO(P)`
- `0,0(r2)` for `T`

### 3. Correct use of MIX addressing

The earlier solution used pseudo-notation such as `0,1(INFO)`, which is not valid MIX syntax.

The correct MIX interpretation is:

- Word at node address `P`:

- `LINK(P) = 0,0(P)`
- `INFO(P) = 1,0(P)`
- Word at stack pointer location:

- `T = 0,0(r2)`

### 4. Correct register discipline

- `r1` is used for node pointer `P`
- `r2` holds the address of `T`
- `rA` holds the inserted item
- `rI1` is set to final stack pointer value

Each register now has a consistent role throughout execution.

## Conclusion

The corrected program properly:

- Interprets the parameter word `NOP T` via explicit memory access
- Uses valid MIX addressing for node fields
- Maintains correct free-list and stack manipulation
- Preserves required exit condition `rI1 = T`
- Returns correctly via `JMP 0,0(rJ)`

This is a valid general-purpose MIX insertion subroutine in the sense of TAOCP Section 1.4.1.
