[Section 2.2.3: Linked Allocation](../)

**Exercise 3.** [*22*] Write a "general purpose" MIX subroutine to do the deletion operation, (11). This subroutine should have the following specifications:

```text
Calling sequence:  JMP DELETE      Jump to subroutine.
                   NOP T           Location of pointer variable
                   JMP UNDERFLOW   First exit, if UNDERFLOW sensed
Entry conditions:  None
Exit conditions:   If the stack whose pointer is the link variable T is empty, the first exit is taken; otherwise the top node of that stack is deleted, and exit is made to the third location following 'JMP DELETE'. In the latter case, rI1 = T and rA is the contents of the INFO field of the deleted node. In either case, rI2 and rI3 are used by this subroutine.
```

**Verified:** no  
**Solve time:** 4m14s

## Solution

Use the standard representation employed in §2.2.3. Let each node occupy two consecutive words,

```
P      LINK(P)
P+1    INFO(P)
```

and let `AVAIL` contain the head of the available-space list. Then deletion operation (11) is

$$
P\leftarrow T,
$$

underflow if $P=\Lambda$, otherwise

$$
T\leftarrow \operatorname{LINK}(P),\qquad
Y\leftarrow \operatorname{INFO}(P),\qquad
\operatorname{AVAIL}\Leftarrow P.
$$

The required calling sequence is

```
          JMP DELETE
          NOP T
          JMP UNDERFLOW
```

When `JMP DELETE` is executed, the jump register `rJ` receives the address of the parameter word `NOP T`. Hence the subroutine must first save this value. A convenient method is to use `STJ`, storing the contents of `rJ` in a temporary word. From that address the parameter word itself may be fetched.

In a MIX instruction word, bytes $0{:}2$ contain the signed address field. Therefore, after the word `NOP T` has been loaded and stored, the instruction

```
LD2 PARM(0:2)
```

loads into `rI2` the address of the pointer variable $T$.

The deleted node address must remain in `rI1`, and the information field of that node must remain in `rA` upon normal exit.

A correct MIXAL program is therefore

```
DELETE  STJ RETADR         Save address of parameter word

        LDA RETADR         Fetch parameter word address
        STA PTR

        LD3 PTR            rI3 ← address of parameter word
        LDA 0,3            Load word NOP T
        STA PARM

        LD2 PARM(0:2)      rI2 ← address of T
        LD1 0,2            rI1 ← P ← T
        J1Z EMPTY          Underflow if P = Λ

        LDA 0,1            A ← LINK(P)
        STA 0,2            T ← LINK(P)

        LDA 1,1            A ← INFO(P)
        STA TEMP           Preserve INFO(P)

        LDA AVAIL
        STA 0,1            LINK(P) ← AVAIL
        ST1 AVAIL          AVAIL ← P

        LDA TEMP           Restore INFO(P)

        LDJ RETADR         Restore original return address
        INCJ 2             Advance to third location after JMP DELETE
        JMP RETURN

EMPTY   LDJ RETADR         Restore original return address
        INCJ 1             Advance to JMP UNDERFLOW
        JMP RETURN

RETURN  JMP 0

RETADR  CON 0
PTR     CON 0
PARM    CON 0
TEMP    CON 0
```

The return mechanism is as follows.

Immediately after

```
JMP DELETE
```

the jump register contains the address of

```
NOP T .
```

Suppose this address is $a$. Then the calling sequence occupies

```
a-1     JMP DELETE
a       NOP T
a+1     JMP UNDERFLOW
a+2     next instruction
```

Hence:

- on underflow, the subroutine restores $a$ to `rJ`, increments it by one, and returns. Control therefore reaches location $a+1$, namely

```
JMP UNDERFLOW
```

which constitutes the first exit;

- on successful deletion, the subroutine restores $a$, increments it by two, and returns. Control therefore reaches location $a+2$, which is the third location following `JMP DELETE`, as required.

The node address $P$ remains in `rI1`, because `rI1` is loaded once with the original stack pointer and is not altered thereafter. The information field $\operatorname{INFO}(P)$ is restored to `rA` after the available-space list has been updated.

Thus the subroutine satisfies all specifications:

- if the stack is empty, the first exit is taken;
- otherwise the top node is deleted and linked onto `AVAIL`;
- `rI1=T=P`, where $P$ is the deleted node;
- `rA=\operatorname{INFO}(P));
- only `rI2` and `rI3` are used as auxiliary index registers. ∎
