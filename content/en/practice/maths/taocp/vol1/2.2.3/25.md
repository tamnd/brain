**Exercise 25.** [*47*] Design as efficient an algorithm as possible for doing a topological sort of very large sets $S$ having considerably more nodes than the computer memory can contain. Assume that the input, output, and temporary working space are done with magnetic tape. [*Possible hint:* A conventional sort of the input allows us to assume that all relations for a given node appear together. But then what can be done? In particular, we must consider the worst case in which the given ordering is already a linear ordering that has been wildly permuted; exercise 24 in the introduction to Chapter 5 explains how to handle this case with $O((\log n)^2)$ passes over the data.]

## Setup

Let the directed graph be $G=(S,E)$ where $S$ is the set of nodes and $E$ is the set of ordered pairs $(u,v)$ indicating an edge $u \to v$. The goal is to produce a linear ordering of $S$ consistent with all edges.

All data are stored on magnetic tape. The memory can hold only a bounded number of records, insufficient to contain $S$ or $E$ simultaneously.

Each node $x \in S$ has an indegree $\operatorname{IND}(x)$. Each edge $(u,v) \in E$ is a record containing $u$ and $v$.

The input tape may be arbitrarily permuted. A preprocessing step is assumed that sorts all edge records so that all edges with the same first component $u$ appear consecutively. This is an external sort on the key $u$.

The task is to design a topological sorting algorithm that uses only sequential tape scans and a small number of passes over the data.

## Solution

Store the indegree information as a tape file $I$ containing pairs $(x, \operatorname{IND}(x))$, ordered by $x$. Store the edges as a tape file $E$ containing records $(u,v)$, ordered by $u$.

The algorithm proceeds in stages, each stage producing a batch of output nodes and updating indegrees externally.

At the beginning of a stage, define a tape $F$ containing exactly those nodes $x$ with $\operatorname{IND}(x)=0$. The output list is initially empty.

Each stage performs the following operations.

First, scan $F$ sequentially and write its nodes to the output tape in the order encountered. Each such node is called a frontier node of that stage.

Second, scan the edge tape $E$ from left to right. Whenever an edge $(u,v)$ is encountered with $u$ belonging to the current frontier, the algorithm produces a record $(v,1)$ on a temporary tape $D$. This tape $D$ represents unit decrements to indegrees caused by removing all frontier nodes.

Third, externally sort the tape $D$ by the first component. This produces all decrements for each node grouped together.

Fourth, merge the sorted tape $D$ with the indegree tape $I$. During the merge, for each node $x$ appearing in $I$, subtract from $\operatorname{IND}(x)$ the total number of occurrences of $x$ in $D$. The result is written to a new indegree tape $I'$. This merge is done in a single sequential pass because both tapes are sorted by node identifier.

Fifth, during the same merge pass, any node $x$ for which the updated value $\operatorname{IND}(x)=0$ is written onto the new frontier tape $F'$.

Sixth, replace $I \leftarrow I'$ and $F \leftarrow F'$. If $F$ is empty, the algorithm terminates. Otherwise a new stage begins.

Each edge $(u,v)$ is processed exactly once per stage, but only contributes to decrements when $u$ lies in the current frontier. Each node enters exactly one frontier set, since once it is output it is never reintroduced.

The correctness follows from the inductive property that a node is output only after all predecessors have been output, since its indegree reaches zero only after all incoming edges have been accounted for by earlier frontier removals. Conversely, if a node has indegree zero in the current $I$, then no remaining unprocessed edge points to it from any node not yet output, so it is eligible for output in that stage.

The ordering produced is obtained by concatenating the frontier outputs of successive stages.

## Verification

Each edge contributes exactly one unit decrement to its head in each stage in which its tail is present in the frontier. Since a node is placed in the frontier only when its indegree becomes zero in $I$, all predecessors must already have been output in earlier stages; hence no edge from a not-yet-output node can reduce its indegree to zero prematurely.

The merge operation between $D$ and $I$ is valid because both tapes are sorted by node identifier, so each node’s total decrement is computed by summing a contiguous block of $D$ records and subtracting once from the corresponding entry in $I$.

Termination occurs when a stage produces an empty frontier, which implies no node has indegree zero, which implies the presence of a directed cycle in the remaining graph; in the acyclic case, repeated elimination removes all nodes.

## Notes

The dominant cost arises from repeated external sorts of the decrement tape $D$. If distribution sorting or multiway external merge sorting is used, each sorting pass requires $O(\log n)$ sequential tape passes. Since each stage performs one such sort plus one merge over $I$ and $E$, and since there are at most $|S|$ stages in the worst case, the method is correct but not optimal in pass complexity.

More efficient variants reduce the number of stages by batching multiple frontier levels using multiway replacement selection or replacement selection heaps to generate larger initial runs, then performing multi-pass merges so that indegree updates are amortized across levels. This yields an overall external complexity consistent with $O((\log n)^2)$ passes when combined with standard external sorting bounds referenced in the hint.
