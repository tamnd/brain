**Exercise 23.** [*27*] When the topological sort algorithm cannot proceed because it has detected a loop in the input (see step `T8`), it is usually of no use to stop and say, "There was a loop." It is helpful to print out one of the loops, thereby showing part of the input that was in error. Extend Algorithm `T` so that it will do this additional printing of a loop when necessary. ## Setup

Algorithm $T$ performs a topological sort on a directed graph with vertices $1,2,\dots,n$, using the array $\text{COUNT}$ and the queue controlled by $F$ and $R$. Step $T8$ halts with $N>0$ exactly when no vertex has $\text{COUNT}=0$ while some vertices remain unprinted, which implies that the remaining subgraph contains a directed cycle.

The task is to extend Algorithm $T$ so that, when step $T8$ is reached, the program prints the vertices of one directed cycle contained in the remaining input graph.

## Solution

Assume execution has reached step $T8$ with $N>0$. Let $S$ be the set of vertices $v$ such that $\text{COUNT}(v)>0$ at this moment. By the argument leading to $T8$, every vertex in $S$ has at least one incoming edge from a vertex in $S$, hence for each $v \in S$ there exists at least one $u \in S$ such that $u \prec v$ is an input relation.

Define a partial predecessor function $\text{PRED}$ on $S$ by choosing, for each $v \in S$, one vertex $\text{PRED}(v)$ such that $\text{PRED}(v) \in S$ and $\text{PRED}(v) \prec v$. This can be implemented by scanning the input relations (or the adjacency structure used in Algorithm $T$) until such a predecessor is found.

Starting from an arbitrary vertex $v_0 \in S$, construct a sequence

$$
v_0, v_1, v_2, \dots
$$

by the rule $v_{i+1} = \text{PRED}(v_i)$. Since $S$ is finite and every $v_i$ lies in $S$, some vertex must repeat: there exist $i<j$ such that $v_i = v_j$. The segment

$$
v_i, v_{i+1}, \dots, v_{j-1}, v_j=v_i
$$

forms a directed cycle, since each step follows an input relation from a vertex in $S$ to another vertex in $S$.

To extract the cycle explicitly, maintain a marker array $\text{MARK}(v)$ initialized to 0 for all $v \in S$. Repeatedly apply the predecessor step $v \leftarrow \text{PRED}(v)$ while marking each visited vertex. When a vertex $v$ is encountered with $\text{MARK}(v)=1$, a cycle has been detected. The cycle is then printed by starting at $v$ and repeatedly applying $\text{PRED}$ until $v$ is encountered again.

The extension of Algorithm $T$ is therefore:

When step $T8$ detects $N>0$, execute the following procedure.

Choose any vertex $v \in S$.

Set $\text{MARK}(v) \leftarrow 1$.

Repeat the following operations:

Find a vertex $u \in S$ such that $u \prec v$ and $\text{COUNT}(u)>0$, and set $v \leftarrow u$.

If $\text{MARK}(v)=1$, then a cycle has been found; go to printing.

Otherwise set $\text{MARK}(v) \leftarrow 1$ and continue.

For printing, let $w$ be the repeated vertex. Set $x \leftarrow w$ and repeatedly output $x$, then set $x \leftarrow \text{PRED}(x)$, stopping when $x=w$.

## Verification

Each vertex in $S$ has $\text{COUNT}>0$, hence at least one incoming edge from another vertex in $S$, so the predecessor selection step is always defined within $S$.

Because $S$ is finite, the sequence generated by repeated application of $\text{PRED}$ cannot be injective indefinitely, so a repeated vertex must occur.

When a repetition occurs, the construction follows directed edges of the original graph, since each $\text{PRED}(v)$ satisfies $\text{PRED}(v) \prec v$. The repeated segment therefore forms a directed closed walk.

Since all vertices in the walk lie in $S$ and each has at least one predecessor in $S$, no vertex in the constructed segment can be terminal; thus the closed walk contains a directed cycle. The marking rule ensures that the first repeated vertex closes a directed cycle, and the printing phase outputs exactly the vertices along this cycle in order.

This completes the proof. ∎

## Notes

The construction depends only on the fact that in the remaining graph at step $T8$, every vertex has positive remaining indegree, which guarantees the existence of a predecessor inside the same set. This avoids any need for reverse adjacency lists by selecting predecessors via a scan of the input relation structure, consistent with the storage assumptions of Section 2.2.3.
