**Exercise 22.** [*23*] Program `T` assumes that its input tape contains valid information, but a program that is intended for general use should always make careful tests on its input so that clerical errors can be detected, and so that the program cannot "destroy itself." For example, if one of the input relations for $k$ were negative, Program `T` may erroneously change one of its own instructions when storing into `X[k]`. Suggest ways to modify Program `T` so that it is suitable for general use.

## Solution

Program $T$ becomes unsafe when an input value is used as a subscript without verification, since a negative or excessively large value of $k$ causes an access such as $X[k]$ or $\text{COUNT}[k]$ to reference memory outside the intended table, potentially overwriting instructions. A correct modification therefore consists of inserting a validation phase before every indirect reference through an input index.

Let the program input each relation in the form $j \prec k$, where $j$ and $k$ are intended to satisfy $1 \le j \le n$ and $1 \le k \le n$. The modified program reads a pair $(j,k)$ and immediately tests both components before any use as subscripts. If either value fails the test, control transfers to a non-returning error routine that halts execution without altering any table entry.

The required modification replaces every direct use of $X[k]$ and $\text{COUNT}[k]$ by a guarded access. After reading $k$, the program executes a range test:

$$
\text{if } k < 1 \text{ or } k > n,\ \text{then ERROR;}
$$

otherwise execution continues with the original instruction sequence that updates $\text{COUNT}[k]$ and later stores $k$ into $\text{X}[j]$. The same test is applied symmetrically to $j$ before any reference to $\text{COUNT}[j]$ or queue insertion keyed by $j$.

In MIX implementation terms, each update of the form $\text{COUNT}[k] \leftarrow \text{COUNT}[k] - 1$ is preceded by a comparison of $k$ with the bounds $1$ and $n$, using conditional jumps to an error exit if either bound is violated. The same protection is applied before any instruction that stores into $\text{QLINK}[k]$ or enqueues $k$ into the queue structure, since these operations also use $k$ as a node address.

To prevent accidental corruption of program storage, the array regions containing $X$, $\text{COUNT}$, and $\text{QLINK}$ are assumed to be disjoint from the instruction area, and all indexing is restricted to these declared bounds. The error routine performs no memory modification and simply terminates execution or returns control to a supervisory system.

With these additions, every access to a table entry depends on a prior proof of validity of its index, so no input value can cause indirect addressing outside the designated storage pool. This eliminates self-modification caused by erroneous subscripts while preserving the original logic of Program $T$. This completes the solution. ∎
