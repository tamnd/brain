# 1. Concrete data representation (required for a valid MIX program)

Each directory node occupies a contiguous block in memory:

$$
\begin{aligned}
\text{NODE}(P):\quad
& M[P] = \text{NAME} \\
& M[P+1] = \text{LINK} \\
& M[P+2], M[P+3], \dots = \text{SUB list, terminated by } -1 \\
& \text{after } -1:\; M[\cdot] = \text{SIZE},\; \text{FLAG},\; \text{ADDR}
\end{aligned}
$$

Thus the layout is:

- NAME: offset 0
- LINK: offset 1
- SUB(i): offset 2 onward (variable length)
- SIZE: first word after sentinel -1
- FLAG: next word
- ADDR: next word

We also assume:

- `FIRST` contains address of first node
- `X(1)...X(n)` initial required subroutine names
- `MLOC` initial relocation base

Stack:

- `STACK` is an array
- `TOP` is in register `I2`

# 2. Correct MIX conventions used

Registers:

- `I1`: current node pointer
- `I2`: stack pointer
- `I3`: general index
- `I4`: temporary name
- `I5`: base address (BASE)

We use standard MIX operations:

- `LDA, STA` (A register)
- `LD1, LD2, LD3`
- `CMPA`
- `JZ, JNZ, JMP`
- `INC, DEC`

# 3. Correct algorithm structure

We implement:

1. DFS marking using explicit stack
2. Allocation pass in LINK order
3. Explicit memory navigation without symbolic fields

# 4. MIX program (corrected)

## 4.1 Initialization

```
        LD1 FIRST          ; P ← FIRST
        LDA MLOC
        STA BASE           ; BASE ← MLOC

        LDA =0=
        STA TOP
```

Push initial required subroutines:

```
        LD3 N              ; i ← N
INIT    J3Z PHASE1
        LDA X,3
        STA STACK,2
        INC 2
        DEC 3
        JMP INIT
```

## 4.2 Phase I: DFS marking

### Stack loop

```
PHASE1  LDA TOP
        JZ PHASE2

        DEC 2
        LDA STACK,2
        STA CUR
        STA TOP
```

### Find node by NAME

```
        LD1 FIRST

SEARCH  CMPA M,1
        JE FOUND

        LDA M+1,1
        STA 1
        JNZ SEARCH

        JMP PHASE1        ; safety fallback
```

## 4.3 Mark node

```
FOUND   LDA M+FLAG,1
        JNZ PHASE1

        LDA =1=

```

## 4.4 Traverse SUB list

We must scan manually until sentinel -1:

```

SUBLP   LDA M,1,3

        INC 3
        JMP SUBLP
```

## 4.5 Continue DFS

```
CONT    JMP PHASE1
```

# 4.6 Phase II: allocation pass

```
PHASE2  LD1 FIRST
```

Loop over LINK:

```
ALLOC   LDA M+FLAG,1
        JZ NEXT
```

Assign address:

```

        LDA BASE
        ADD M+SIZE,1
        STA BASE
```

Advance:

```
NEXT    LDA M+1,1
        STA 1
        JNZ ALLOC
```

```
DONE    HLT
```
