[Section 2.2.3: Linked Allocation](../)

**Exercise 26.** [*29*] (*Subroutine allocation.*) Suppose that we have a tape containing the main subroutine library in relocatable form, for a 1960s-style computer installation. The loading routine wants to determine the amount of relocation for each subroutine used, so that it can make one pass through the tape to load the necessary routines. The problem is that some subroutines require others to be present in memory. Infrequently used subroutines (which appear toward the end of the tape) may call on frequently used subroutines (which appear toward the beginning of the tape), and we want to know all of the subroutines that are required, before passing through the tape.

One way to tackle this problem is to have a "tape directory" that fits in memory. The loading routine has access to two tables:

a) The tape directory. This table is composed of variable-length nodes having the form

![Tape-directory node forms for exercise 26.](images/page_0293_ex26_tables.png)

where `SPACE` is the number of words of memory required by the subroutine; `LINK` is a link to the directory entry for the subroutine that appears on the tape following this subroutine; `SUB1`, `SUB2`, ..., `SUBn` ($n \ge 0$) are links to the directory entries for any other subroutines required by this one; `B = 0` on all words except the last, `B = -1` on the last word of a node. The address of the directory entry for the first subroutine on the library tape is specified by the link variable `FIRST`.

b) The list of subroutines directly referred to by the program to be loaded. This is stored in consecutive locations `X[1]`, `X[2]`, ..., `X[N]`, where `N \ge 0` is a variable known to the loading routine. Each entry in this list is a link to the directory entry for the subroutine desired.

The loading routine also knows `MLOC`, the amount of relocation to be used for the first subroutine loaded.

As a small example, consider the following configuration:

![Example tape directory and required-subroutine list.](images/page_0294_example_tables.png)

The tape directory in this case shows that the subroutines on tape are `1002`, `1010`, `1006`, `1000`, `1005`, `1003`, and `1007` in that order. Subroutine `1007` takes 200 locations and implies the use of subroutines `1005`, `1002`, and `1006`; etc. The program to be loaded requires subroutines `1003` and `1010`, which are to be placed into locations $\ge 2400$. These subroutines in turn imply that `1000`, `1006`, and `1002` must also be loaded.

The subroutine allocator is to change the `X`-table so that each entry `X[1]`, `X[2]`, `X[3]`, ... has the form

```text
+ | 0 | BASE | SUB
```

(except the last entry, which is explained below), where `SUB` is a subroutine to be loaded, and `BASE` is the amount of relocation. These entries are to be in the order in which the subroutines appear on tape. One possible answer for the example above would be

```text
BASE   SUB
X[1]:  2400  1002
X[2]:  2430  1010
X[3]:  2450  1006
X[4]:  2510  1000
X[5]:  2530  1003
X[6]:  2730  0
```

The last entry contains the first unused memory address.

(Clearly, this is not the only way to treat a library of subroutines. The proper way to design a library is heavily dependent upon the computer used and the applications to be handled. Large modern computers require an entirely different approach to subroutine libraries. But this is a nice exercise anyway, because it involves interesting manipulations on both sequential and linked data.)

The problem in this exercise is to design an algorithm for the stated task. Your allocator may transform the tape directory in any way as it prepares its answer, since the tape directory can be read in anew by the subroutine allocator on its next assignment, and the tape directory is not needed by other parts of the loading routine.

**Verified:** yes  
**Solve time:** 1m36s

## Setup

For each directory node let

$$
\operatorname{SPACE}(P)
$$

denote the number of words required by subroutine $P$, and let

$$
\operatorname{NEXT}(P)
$$

denote the link to the directory entry of the next subroutine on the tape. The links

$$
\operatorname{SUB}_1(P),\operatorname{SUB}_2(P),\ldots
$$

refer to the subroutines required by $P$. The last word of a node is distinguished by the mark $B=-1$.

The initial requests are

$$
X[1],X[2],\ldots ,X[N].
$$

Every subroutine that is directly requested, or that is required recursively by another requested subroutine, must be loaded. The output table is to contain all such subroutines in tape order. If

$$
P_1,P_2,\ldots ,P_t
$$

are the required subroutines in tape order, their relocation bases must satisfy

$$
\begin{aligned}
\operatorname{BASE}(P_1)&=MLOC,\\
\operatorname{BASE}(P_{i+1})
&=\operatorname{BASE}(P_i)+\operatorname{SPACE}(P_i),
\qquad 1\le i<t.
\end{aligned}
$$

The last entry of the $X$-table must contain the first unused address.

The tape directory may be altered arbitrarily during the computation.

## Solution

The problem separates naturally into three stages.

### Stage 1. Mark all required subroutines

A depth-first search is performed on the graph whose vertices are directory nodes and whose arcs are the requirement links.

The directory itself supplies storage for the marks. Introduce a field

$$
\operatorname{MARK}(P),
$$

initially $0$ for every node.

For each entry $X[i]$, $1\le i\le N$, execute procedure $R(P)$, where $P=X[i]$.

Procedure $R(P)$:

R1. If $\operatorname{MARK}(P)=1$, terminate this invocation.

R2. Set

$$
\operatorname{MARK}(P)\leftarrow1.
$$

R3. Successively examine

$$
\operatorname{SUB}_1(P),\operatorname{SUB}_2(P),\ldots
$$

until the word having $B=-1$ is reached. For every non-null dependency $Q$, invoke $R(Q)$.

When all invocations terminate,

$$
\operatorname{MARK}(P)=1
$$

exactly for the subroutines that must be loaded.

### Stage 2. Scan the tape order and assign relocation bases

Set

$$
LOC\leftarrow MLOC,
\qquad
K\leftarrow1,
\qquad
P\leftarrow FIRST.
$$

Then repeat the following steps.

S1. If

$$
P=\Lambda,
$$

go to S5.

S2. If

$$
\operatorname{MARK}(P)=0,
$$

set

$$
P\leftarrow\operatorname{NEXT}(P)
$$

and return to S1.

S3. Store in the next output position

$$
X[K]=(+\mid0\mid LOC\mid P).
$$

S4. Replace

$$
LOC\leftarrow LOC+\operatorname{SPACE}(P),
\qquad
K\leftarrow K+1,
\qquad
P\leftarrow\operatorname{NEXT}(P),
$$

and return to S1.

S5. Store

$$
X[K]=(+\mid0\mid LOC\mid0).
$$

Since the scan follows the chain beginning with $FIRST$, the entries appear in tape order automatically.

### Example

In the example of the statement, the initial requests are

$$
1003,\;1010.
$$

The recursive marking stage reaches

$$
1003,\;1000,\;1006,\;1002,\;1010.
$$

Scanning the tape order

$$
1002,\;1010,\;1006,\;1000,\;1005,\;1003,\;1007
$$

and retaining only marked nodes yields

$$
1002,\;1010,\;1006,\;1000,\;1003.
$$

Starting from

$$
LOC=2400,
$$

the successive bases are

$$
2400,\;2430,\;2450,\;2510,\;2530,
$$

and the final unused address is

$$
2730.
$$

Hence one obtains

$$
\begin{array}{c|c}
\text{BASE}&\text{SUB}\\
\hline
2400&1002\\
2430&1010\\
2450&1006\\
2510&1000\\
2530&1003\\
2730&0
\end{array}
$$

which agrees with the sample answer.

## Verification

Every subroutine produced by the algorithm is required.

A node is entered into the output table only during Stage 2, and only nodes satisfying

$$
\operatorname{MARK}(P)=1
$$

are selected. A mark can be created only in procedure $R$, beginning with the original requests and propagating along requirement links. Therefore every output node belongs to the transitive closure of the requested set.

Every required subroutine appears in the output.

Suppose $P$ is required. By definition there exists a chain of requirement links beginning with one of the original entries $X[i]$ and ending at $P$. Procedure $R$ traverses every such link once before terminating, unless the endpoint has already been marked. Hence

$$
\operatorname{MARK}(P)=1.
$$

During Stage 2 the scan visits every directory node exactly once in tape order. Since $P$ is marked, it is entered into the output table. Thus no required subroutine is omitted.

The order is correct.

The values

$$
P,\operatorname{NEXT}(P),\operatorname{NEXT}(\operatorname{NEXT}(P)),\ldots
$$

are precisely the order of appearance on the tape. Stage 2 follows this chain without rearrangement. Therefore the output order is the tape order.

The relocation bases are correct.

For the first subroutine,

$$
\operatorname{BASE}(P_1)=MLOC.
$$

After recording $P_i$, the variable $LOC$ is increased by

$$
\operatorname{SPACE}(P_i),
$$

so before recording $P_{i+1}$,

$$
\operatorname{BASE}(P_{i+1})
=
\operatorname{BASE}(P_i)
+
\operatorname{SPACE}(P_i).
$$

Consequently each subroutine begins immediately after the preceding one, and the final value of $LOC$ equals the first unused memory location. The last table entry therefore has the required form.

This completes the proof.

∎

## Notes

The recursive marking stage is equivalent to Algorithm T of Section 2.2.3 applied to the directed graph of subroutine dependencies. The present problem requires only the transitive closure of the requested set, not a topological ordering, because the tape order is prescribed independently by the links beginning with `FIRST`.
