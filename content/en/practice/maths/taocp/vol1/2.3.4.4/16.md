**Exercise 16.** [**] [25] Design an algorithm (as efficient as possible) that converts from the canonical representation of an oriented tree to a conventional computer representation using `PARENT` links.

Suppose the canonical representation of an oriented tree with $n$ vertices is given as the sequence $x_1,x_2,\ldots,x_{n-1}$, where $1\le x_j\le n$. We first determine the vertices $V_1,V_2,\ldots,V_n$ in the order in which they were removed during the canonical construction. Initialize a set $S$ of available vertices containing ${1,2,\ldots,n}$. For $j$ from $1$ to $n-1$, let $V_j$ be the smallest element of $S$ that does not appear in $x_j,\ldots,x_{n-1}$; then remove $V_j$ from $S$. Finally, let $V_n$ be the remaining element of $S$.

Having obtained the permutation $V_1,\ldots,V_n$, we define the conventional `PARENT` representation by performing $n-1$ assignments: for each $j$ from $1$ to $n-1$, execute

$$
\text{PARENT}[V_j] \leftarrow x_j.
$$

Each assignment corresponds to one replacement operation, so the total number of replacements is $n-1$. Vertex $V_n$ is the root and does not receive a parent. This algorithm is optimal in the sense that every edge must be recorded, requiring at least $n-1$ assignments. The procedure produces a standard computer representation of the oriented tree directly from its canonical representation, preserving all structure.

This completes the proof.
