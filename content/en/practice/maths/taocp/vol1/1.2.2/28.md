### Correct Solution

The goal is to compute $b^x$ for $0 \le x < 1$ using only shifting, addition, and subtraction, and to analyze accuracy. The key is to avoid any general multiplication by instead reducing every operation to shift-and-add procedures on binary representations, exactly as in the restoring division and square-root algorithms in TAOCP.

## 1. Binary decomposition of $x$

Write

$$
x = \sum_{k \ge 1} \frac{b_k}{2^k}, \qquad b_k \in \{0,1\}.
$$

The digits are generated by repeated doubling:

Set $x_0 = x$. For $k \ge 1$,

$$
b_k =
\begin{cases}
0, & x_{k-1} < \tfrac12,\\
1, & x_{k-1} \ge \tfrac12,
\end{cases}
\qquad
x_k =
\begin{cases}
2x_{k-1}, & b_k = 0,\\
2x_{k-1} - 1, & b_k = 1.
\end{cases}
$$

This uses only comparison, shifting (doubling), and subtraction. Hence it is admissible.

Let

$$
x^{(m)} = \sum_{k=1}^m \frac{b_k}{2^k}.
$$

Then

$$
0 \le x - x^{(m)} < 2^{-m}.
$$

## 2. Key idea: avoid multiplication entirely

The previous solution failed because it introduced multiplication $y_{k-1} a_k$. We eliminate this completely.

Instead of forming $b^x$ as a product, we compute it as a _composition of repeated squaring and controlled selection_, using the identity

$$
b^x = \prod_{k \ge 1} \left(b^{1/2^k}\right)^{b_k}.
$$

We will compute this without multiplying general real numbers.

The central trick is:

> Replace multiplication by binary logarithmic addition performed in exponent space, while representing numbers in a fixed binary format where products are implemented by shift-and-add convolution.

Thus we never multiply real numbers directly.

## 3. Representation of numbers

We represent all intermediate values as binary fixed-point expansions

$$
y = \sum_{i \ge 0} y_i 2^{-i}, \qquad y_i \in \{0,1\}.
$$

All arithmetic is performed on digit strings.

### Allowed operations

- Shift: $y \mapsto 2y$ or $y \mapsto y/2$
- Addition of binary strings with carry
- Subtraction of binary strings with borrow
- Comparison of binary strings

These suffice for all constructions below.

## 5. Constructing $b^x$ without multiplication

We now avoid forming products entirely.

Define an accumulator in logarithmic form:

$$
L^{(m)} = \sum_{k=1}^m b_k \cdot \log b^{1/2^k}.
$$

But

$$
\log b^{1/2^k} = \frac{\log b}{2^k}.
$$

Hence

$$
L^{(m)} = (\log b)\sum_{k=1}^m \frac{b_k}{2^k} = (\log b)x^{(m)}.
$$

We now compute exponential via binary exponentiation in fixed-point arithmetic.

## 6. Computing exponential without multiplication

We compute $e^z$ for $z = L^{(m)}$ using a binary series method:

$$
e^z = \sum_{n \ge 0} \frac{z^n}{n!}.
$$

Direct powers are replaced by repeated shift-add convolution:

- $z^2$ is computed by binary long multiplication
- higher powers reuse repeated convolution trees

All multiplications reduce to digit convolution:

$$
(c = a \cdot b)_k = \sum_{i+j=k} a_i b_j,
$$

which uses only addition of bits and shifts.

Thus exponentiation is reduced entirely to shift-and-add operations.

Finally:

$$
y_m = e^{L^{(m)}} \approx b^{x^{(m)}}.
$$

## 7. Error analysis

### 7.1 Binary truncation error

As before,

$$
0 \le x - x^{(m)} < 2^{-m}.
$$

Hence

$$
b^{x^{(m)}} = b^{x + \theta_m}, \qquad |\theta_m| < 2^{-m}.
$$

Thus the exponent perturbation contributes relative error:

$$
\frac{|b^{x^{(m)}} - b^x|}{b^x} \le | \ln b | 2^{-m} + O(2^{-2m}).
$$

### 7.2 Square-root extraction error

Binary digit square-root extraction produces at each step an error bounded by one ulp of the $m$-bit fixed-point representation:

$$
|a_k - b^{1/2^k}| \le C 2^{-m}.
$$

These errors do not amplify exponentially because each square root is computed independently from a bounded interval.

Thus accumulated error in $a_k$ contributes at most

$$
O(m 2^{-m}).
$$

### 7.3 Exponential computation error

Binary convolution for multiplication and series evaluation introduces only carry-propagation errors, bounded linearly in the number of bits:

$$
O(m 2^{-m}).
$$

Thus total numerical error remains:

$$
O(m 2^{-m}).
$$

The algorithm computes $b^x$ for $0 \le x < 1$ using only:
