**Exercise 4.** [*20*] Show how coroutine linkage analogous to (1) can be given for real-life computers you are familiar with.

## Solution

Consider a conventional stored-program computer with a program counter $\mathrm{PC}$ and memory cells that can hold addresses. Let $A$ and $B$ denote two coroutines, and let $A1$ and $B1$ denote their respective entry points into their principal code sections.

On such a machine, the role of the MIX instruction $\mathrm{STJ}$ in linkage (1) is performed by storing the current return address, namely the value of $\mathrm{PC}$ at the moment of transfer, into a dedicated memory location associated with the coroutine that is about to resume later. The role of $\mathrm{JMP}$ is performed by loading $\mathrm{PC}$ with the address of the target entry point.

Introduce two memory locations $AX$ and $BX$, intended to store the continuation points of coroutines $A$ and $B$, respectively.

When coroutine $A$ transfers control to $B$, the linkage is written in the form

$$
AX \leftarrow \mathrm{PC}; \quad \mathrm{PC} \leftarrow \text{address}(B1).
$$

This stores the return position in $A$ and transfers execution to the beginning of $B$'s main body.

When coroutine $B$ transfers control back to $A$, the symmetric linkage is

$$
BX \leftarrow \mathrm{PC}; \quad \mathrm{PC} \leftarrow \text{address}(A1).
$$

This stores the return position in $B$ and resumes execution of $A$ at $A1$.

The essential mechanism is that each coroutine overwrites its own saved continuation each time it suspends, and resumes by indirect transfer through the stored address. When $A$ is later reactivated, execution continues from the address stored in $AX$, and when $B$ is reactivated, execution continues from the address stored in $BX$. This implements the same control behavior as the MIX sequence in (1), since each transfer both saves the current return point and restores the previously saved return point of the other coroutine.

In a machine with a hardware stack, the same effect is obtained by treating each coroutine as having its own activation record containing a saved program counter; suspension pushes the current continuation into that record, and resumption restores it into $\mathrm{PC}$. This realizes the same symmetric control structure as the MIX linkage.

This completes the solution. ∎
