**Exercise 6.** [*22*] Give coroutine linkage analogous to (1) for the case of *three* coroutines, A, B, and C, each of which can jump to either of the other two. (Whenever a coroutine is activated, it begins where it last left off.)

The error in the previous solution is the treatment of the resume labels. The stored return addresses in the linkage cells must be _used as control transfers_, not discarded by jumping to a fixed entry point. In MIX-style coroutine linkage, resumption is performed by an indirect jump through the saved location.

We restart the construction from the correct coroutine linkage principle.

## Correct coroutine linkage for three coroutines

We have three coroutines $A, B, C$. As in the two-coroutine case, we introduce one return-address cell for each ordered pair of distinct coroutines:

- In $A$: $AB, AC$
- In $B$: $BA, BC$
- In $C$: $CA, CB$

Each cell $YX$ stores the return address to be used when coroutine $Y$ resumes after being called by $X$.

We use the MIX convention:

- `STJ L` stores the current jump return address in location $L$
- `JMP L` is an indirect transfer through $L$, i.e., it transfers control to the address stored in $L$

## Structure of each coroutine

Each coroutine $Y$ has:

1. Two entry points (depending on caller) that store the return address into the appropriate cell and jump into the body.
2. A body that may suspend and resume using the stored linkage cells.
3. Two resume labels $YX, YZ$, which _must transfer control indirectly through the saved return address_.

Let $A_1, B_1, C_1$ denote the main bodies.

## Coroutine $A$

### Entry points

From $B$:

```
A from B:   STJ AB
            JMP A1
```

From $C$:

```
A from C:   STJ AC
            JMP A1
```

### Resume points (corrected)

```
AB:         JMP AB      (indirect transfer via stored return address)
AC:         JMP AC
```

Thus control resumes at the exact continuation point of the interrupted execution, not at $A_1$.

## Coroutine $B$

### Entry points

From $A$:

```
B from A:   STJ BA
            JMP B1
```

From $C$:

```
B from C:   STJ BC
            JMP B1
```

### Resume points

```
BA:         JMP BA
BC:         JMP BC
```

## Coroutine $C$

### Entry points

From $A$:

```
C from A:   STJ CA
            JMP C1
```

From $B$:

```
C from B:   STJ CB
            JMP C1
```

### Resume points

```
CA:         JMP CA
CB:         JMP CB
```

## Why this is correct

### 1. Correct use of stored return addresses

Each entry point stores the return address in a distinct cell $YX$. The corresponding resume label does not restart the coroutine at $Y_1$; instead it performs

$$
\texttt{JMP YX}
$$

which transfers control to the address previously stored by `STJ`. This preserves the exact continuation point.

### 2. Preservation of coroutine state

When coroutine $Y$ is activated from $X$, execution proceeds until it suspends (by transferring control elsewhere and storing its continuation in the appropriate cell). Upon reactivation, control enters at $YX$, and the indirect jump resumes execution exactly where it last left off.

Thus the defining requirement is satisfied:

> each activation resumes at the point of last suspension.

### 3. Consistent linkage structure

The six cells $AB, AC, BA, BC, CA, CB$ provide a complete and disjoint partition of all possible caller–callee return paths. No cell is overwritten by an unrelated transfer, and each resume point uniquely determines its continuation via stored state.

## Conclusion

The corrected linkage program properly implements coroutine behavior in MIX style: entry points store return addresses, and resume points perform indirect transfers through those stored addresses. This ensures true suspension and resumption rather than reinitialization at fixed entry labels.
