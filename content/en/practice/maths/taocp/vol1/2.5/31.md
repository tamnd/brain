**Exercise 31.** [**] [M40] Can a storage allocation system analogous to the buddy system
be designed using the Fibonacci sequence instead of powers of two? (Thus,
we might start with $F_m$ available words, and split an available block of
$F_k$ words into two buddies of respective lengths $F_{k-1}$ and
$F_{k-2}$.)

**Verified:** yes  
**Solve time:** 2m42s

Let $F_1=1$, $F_2=1$, and $F_k=F_{k-1}+F_{k-2}$ for $k\ge 3$. A Fibonacci buddy system can be defined by organizing memory into blocks whose sizes are Fibonacci numbers, with a strict structural rule that makes splitting and coalescing deterministic.

Each block of size $F_k$ is identified by an address interval together with a level index $k$. The initial memory consists of one free block of size $F_m$. A split is allowed only in the canonical form

$$
F_k \to (F_{k-1},\,F_{k-2})
$$

and the two resulting blocks are stored with inherited address intervals so that their union is exactly the parent interval. This guarantees that every block has a unique parent in the splitting tree, hence a unique ancestry.

Coalescing is defined using this same tree structure. Two free blocks of sizes $F_{k-1}$ and $F_{k-2}$ are buddies if and only if they were created by splitting the same parent block of size $F_k$. Since each split creates exactly one ordered pair of children, every block has exactly one potential buddy determined by its parent in the implicit Fibonacci decomposition tree. If both children become free simultaneously, they are merged immediately into their parent block.

The key point is that uniqueness of coalescing does not depend on arithmetic properties of Fibonacci representations of integers, but on the explicit binary tree induced by recursive splitting of the initial block $F_m$. This tree is fixed at initialization, so every block corresponds to a unique node, and every node has at most one sibling.

Allocation proceeds by rounding a request of size $s$ up to the smallest $F_k \ge s$. A free list is maintained for each size class. If a block of size $F_k$ is unavailable, a larger block $F_j$ is removed from its free list and recursively split along the unique path

$$
F_j \to F_{j-1} \to \cdots \to F_k
$$

by repeatedly applying the rule $F_t \to (F_{t-1},F_{t-2})$ and selecting the branch that preserves the required total size. Each step reduces the index by at least one, so the process terminates in finitely many steps.

The structure ensures that coalescing is always unambiguous. If two blocks are buddies, they correspond to the two children of a unique node in the fixed Fibonacci splitting tree. No two distinct nodes produce the same pair of children, since node identity is determined by its position in the initial decomposition of $F_m$. Therefore, no additional metadata beyond the level index and address interval is required.

The conclusion is that a Fibonacci analogue of the buddy system can be constructed. It preserves the defining properties of the original system: recursive splitting into predetermined size classes, and deterministic constant-time coalescing via a fixed parent-child relationship in a binary tree structure. The difference from the power-of-two system is only that the size reduction follows the recurrence $F_k=F_{k-1}+F_{k-2}$ rather than halving, but the underlying buddy-tree mechanism remains valid. ∎
