**Exercise 7.** [**] [10] The example (1) shows that first-fit can sometimes be definitely
superior to best-fit. Give a similar example that shows a case where
best-fit is superior to first-fit.

**Verified:** no  
**Solve time:** 4m25s  

---

Suppose the `AVAIL` list contains three free blocks, in order of location, with sizes

$100,\ 20,\ 20,$

and suppose two requests arrive consecutively, first for a block of size $18$, then for a block of size $100$.

With first-fit, the request for $18$ is satisfied from the first block of size $100$, leaving free blocks of sizes

$82,\ 20,\ 20.$

The subsequent request for $100$ cannot be satisfied, since no free block has size at least $100$.

With best-fit, the request for $18$ is satisfied from one of the blocks of size $20$, leaving free blocks of sizes

$100,\ 2,\ 20.$

The subsequent request for $100$ is then satisfied exactly by the remaining block of size $100$.

Thus first-fit fails to satisfy both requests, while best-fit satisfies both. This gives a case in which best-fit is definitely superior to first-fit. ∎
