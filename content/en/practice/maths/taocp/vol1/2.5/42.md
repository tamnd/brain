**Exercise 42.** [**] [M40] (J. M. Robson, 1975.) Let $N_{\rm BF}(n,m)$ be the amount of
memory needed to guarantee non-overflow when the best-fit method is used
for allocation as in exercise 38. Find an attacking strategy to show that
$$
N_{\rm BF}(n,m)\geq mn-O(n+m^2).
$$

**Verified:** no  
**Solve time:** 4m  

---

Let the memory be a linear array of locations. There are $m$ processes $P_1,\dots,P_m$. Each process alternates forever between “holding” a block of size $n$ and being completely free. At any time, a process either owns one contiguous allocated block or owns nothing. The adversary controls the order of requests and releases.

The attack is constructed in rounds. In each round $t=1,2,\dots$, exactly one process performs a full build of a block of size $n$, while all other processes are forced to participate in controlled releases that create fragmentation.

### Construction of one round

Fix a round with active process $P_i$. At the beginning of the round, all processes are empty.

The process $P_i$ requests $n$ blocks of size $1$, one at a time. After each allocation of size $1$, the adversary forces the following action: every process $P_j$ with $j<i$ releases exactly one of its previously allocated unit blocks if it has any, and every process $P_j$ with $j>i$ does nothing.

Thus during the growth of $P_i$, only processes with smaller indices actively release, and each release is of size $1$.

This guarantees two properties. First, the system always contains at most $m$ processes holding at most one block each, so the total number of free intervals created by releases is controlled by $m$. Second, every release produces a hole of size $1$, and these holes are repeatedly reinserted into the global free list at scattered positions.

### Behavior of best-fit

Each request is for a single unit, so best-fit always chooses a hole of minimum size. Since all holes are created by unit releases, the only holes ever present are unit holes plus occasional larger holes formed by merging adjacent free unit holes.

A key structural fact is that after any prefix of the construction of $P_i$, the free memory consists of:

1. At most $n$ unit holes created during the current round.
2. At most one boundary region adjacent to each process block, contributing at most $O(m)$ additional holes.
3. At most one interaction region between any ordered pair $(P_i,P_j)$, since a hole created by $P_j$ can be absorbed into the growth of $P_i$ at most once before it is displaced by subsequent unit releases.

This last point is the only nontrivial combinatorial bound. It follows because a hole created by a unit release from $P_j$ is either used immediately by the next unit request of $P_i$, or it is merged with at most one adjacent unit hole. After such a merge, its boundary with respect to $P_i$'s growing block is destroyed and cannot reappear in the same form during the same round. Since there are $m$ processes and each ordered pair can produce at most one such boundary interaction per round, the total number of persistent interaction-induced holes is $O(m^2)$ per round.

### Wasted space accounting

We now bound the difference between total memory used and useful memory $mn$.

Each process contributes a contiguous block of size $n$, so the useful memory is exactly $mn$.

Wasted memory comes only from holes. These are of three types:

First, boundary holes internal to the construction of each process contribute at most $O(n)$ total space, since at most one new boundary can arise per unit increase in a process’s block size.

Second, interaction holes between distinct processes contribute at most $O(m^2)$ total space, since each ordered pair $(P_i,P_j)$ can generate at most a constant number of persistent holes during a full round.

Third, unit holes are transient and are always immediately reused by best-fit in subsequent allocations, so they do not contribute to persistent wasted space.

Thus the total wasted memory is bounded by

$$
O(n+m^2).
$$

### Conclusion

Since the useful occupied memory is $mn$ and the wasted memory is at most $O(n+m^2)$, any memory that avoids overflow under this adversary must satisfy

$$
N_{\mathrm{BF}}(n,m) \ge mn - O(n+m^2).
$$

This completes the construction of the required attacking strategy. ∎
