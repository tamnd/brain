**Exercise 19.** [**] [24] Suppose that blocks of memory have the form of (7), but without
the `TAG` or `SIZE` fields required in the last word of the block.
Suppose further that the following simple algorithm is being used to make
a reserved block free again:
$$
Q\leftarrow AVAIL,\quad LINK(P0)\leftarrow Q,\quad
LINK(P0+1)\leftarrow LOC(AVAIL),
$$
$$
LINK(Q+1)\leftarrow P0,\quad AVAIL\leftarrow P0,\quad
TAG(P0)\leftarrow "-".
$$
(This algorithm does nothing about collapsing adjacent areas together.)
Design a reservation algorithm, similar to Algorithm A, that does the
necessary collapsing of adjacent free blocks while searching the `AVAIL`
list, and at the same time avoids any unnecessary fragmentation of memory
as in (2), (3), and (4).

The key difficulty is that without `TAG` or `SIZE` in the last word of a block, physical adjacency cannot be inferred from list structure alone. The algorithm must therefore use address comparison against the ordered `AVAIL` list while scanning, exactly as in Algorithm A, and must perform coalescing by explicit removal and reinsertion in the free list.

Let the free list be maintained in strictly increasing address order by `LINK`, as in Algorithm A. Each free block $P$ is identified by its starting address, and its length is known from the block representation (7), since the block occupies a contiguous region and its size is stored in the block header. From a block starting at address $P$, the address of its immediate physical successor in memory is $P + \text{size}(P)$.

The reservation algorithm performs a first-fit scan with a predecessor pointer $P_0$ and current pointer $P$. For each block, the algorithm first performs maximal coalescing with any physically adjacent free block before testing whether it is large enough.

The algorithm is:

$P_0 \leftarrow 0,\quad P \leftarrow AVAIL$

E1: If $P = 0$, the search fails.

E2: Set $R \leftarrow LINK(P)$.

E3: Compute the physical successor address $S \leftarrow P + \text{size}(P)$.

E4: If $R \neq 0$ and $LOC(R) = S$, then the successor in the free list is physically adjacent to $P$, so coalescing is performed:

Remove $R$ from the free list:

$LINK(P) \leftarrow LINK(R).$

Extend $P$ to absorb $R$ by increasing its stored size:

$\text{size}(P) \leftarrow \text{size}(P) + \text{size}(R).$

Return to E2, since further coalescing may now be possible.

E5: If the request size $n$ satisfies $n \le \text{size}(P)$, proceed to allocation.

Otherwise set $P_0 \leftarrow P,\quad P \leftarrow LINK(P)$ and return to E1.

E6: Allocation from $P$:

If $\text{size}(P) = n$, remove $P$ from the free list by

$LINK(P_0) \leftarrow LINK(P).$

If $\text{size}(P) > n$, split the block. Let $Q \leftarrow P + n$. The first $n$ words are allocated, and the remainder becomes a new free block starting at $Q$ with size $\text{size}(P) - n$.

Insert $Q$ into the free list in the same position as $P$ was removed from, preserving address order:

$LINK(Q) \leftarrow LINK(P),\quad LINK(P_0) \leftarrow Q.$

The correctness of coalescing follows from the fact that every time two free blocks become physically adjacent during the scan, their starting addresses differ exactly by the size of the left block, so the equality test $LOC(R) = P + \text{size}(P)$ detects adjacency. Because the scan always proceeds in increasing address order, any newly formed larger block remains in correct position relative to later blocks, and future adjacency created by earlier merges is encountered immediately when the scan reaches that region.

Since every examined block is fully merged with all physically adjacent free blocks before allocation is attempted, no two adjacent free blocks can remain unmerged at any time during the scan. This eliminates fragmentation patterns where separable adjacent free regions persist, because any such configuration is immediately collapsed during traversal. The first-fit scan order is preserved, so allocation remains identical in structure to Algorithm A, except that each candidate block is maximally consolidated before being tested.
