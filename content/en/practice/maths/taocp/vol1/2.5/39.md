[Section 2.5: Dynamic Storage Allocation](../)

**Exercise 39.** [**] [HM23] (J. M. Robson.) In the notation of exercise 38, show that
$$
N(n_1+n_2,m)\leq N(n_1,m)+N(n_2,m)+N(2m-2,m);
$$
hence for fixed $m$, $\lim_{n\to\infty}N(n,m)/n=N(m)$ exists.

**Solve time:** 5m41s

Consider any admissible sequence of requests whose total simultaneously allocated memory never exceeds $n_1+n_2$. Partition each request sequence into two subsequences by tagging each request as belonging to class $1$ or class $2$ so that the total size of requests of class $i$ never exceeds $n_i$ at any time; such a partition is obtained by assigning to class $1$ the first $n_1$ units of each allocation in a greedy scan of the history and assigning the remainder (if any) to class $2$. Each class thus forms a valid request sequence for parameter $(n_i,m)$.

Allocate memory as three contiguous regions: a region of size $N(n_1,m)$ serving class $1$, a region of size $N(n_2,m)$ serving class $2$, and a separating buffer region of size $N(2m-2,m)$. The two main regions operate independently under their respective optimal storage allocations, since all blocks in each class have size at most $m$ and are confined to disjoint memory intervals. Any interaction caused by boundary effects between the two regions can be absorbed into the buffer, because the worst fragmentation pattern at the interface involves at most two adjacent blocks of size at most $m-1$, hence total size at most $2m-2$, which is handled by the definition of $N(2m-2,m)$.

Thus any sequence of total demand at most $n_1+n_2$ can be handled in memory of size

$$
N(n_1,m)+N(n_2,m)+N(2m-2,m),
$$

which yields

$$
N(n_1+n_2,m)\le N(n_1,m)+N(n_2,m)+N(2m-2,m).
$$

Let $C=N(2m-2,m)$. Define $A(n)=N(n,m)+C$. Then

$$
A(n_1+n_2)\le A(n_1)+A(n_2),
$$

so $(A(n))$ is subadditive. By Fekete’s subadditive lemma, $\lim_{n\to\infty} A(n)/n$ exists and equals $\inf A(n)/n$. Since $C/n\to 0$, the same limit exists for $N(n,m)/n$. Therefore $\lim_{n\to\infty} N(n,m)/n$ exists. ∎
