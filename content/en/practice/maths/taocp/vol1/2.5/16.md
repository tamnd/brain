**Exercise 16.** [**] [24] Write a MIX program for Algorithm C, incorporating the ideas of
exercise 15.

**Verified:** no  
**Solve time:** 4m35s  

---

Algorithm C can be coded directly in MIX by following the structure of steps C1, C6 and incorporating the optimization of exercise 15. The essential idea is to distinguish the four possibilities determined by the signs of `TAG(P0-1)` and `TAG(P0+SIZE(P0))` before any links are changed:

1. Neither adjacent block is free.
2. Only the left adjacent block is free.
3. Only the right adjacent block is free.
4. Both adjacent blocks are free.

In each case the program performs only the link updates required for the resulting free-list structure. Case 1 inserts the newly released block into the free list. Cases 2 and 3 enlarge an existing free block by coalescing with one neighbor. Case 4 merges three consecutive regions into a single free block. Since the final free-list configuration is different in each case, at least the corresponding indispensable link modifications must be made; the optimization consists of avoiding any temporary deletion and reinsertion of blocks that are already correctly linked.

The MIX program therefore follows Algorithm C with a four-way branch after examining the two tags. Separate code sequences handle the four cases, updating only the necessary `LINK` and `SIZE` fields before returning. This yields a somewhat longer program because of the additional branching, but it reduces the number of memory references and link assignments performed during storage release.
