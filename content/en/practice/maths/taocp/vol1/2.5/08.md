[Section 2.5: Dynamic Storage Allocation](../)

**Exercise 8.** [**] [21] Show how to modify Algorithm A in a simple way to obtain an
algorithm for the best-fit method, instead of first-fit.

**Verified:** yes  
**Solve time:** 57s  

---

Algorithm A performs a first-fit search in the `AVAIL` list by scanning nodes in order and selecting the first block whose `SIZE` is at least the requested value `N`. To obtain the best-fit method, the selection rule must be changed from “first block satisfying the constraint” to “block of minimal size among all blocks satisfying the constraint.”

The simplest modification is to replace the single linear scan with a full scan that maintains a current best candidate. During the traversal of the `AVAIL` list, two variables are maintained: a pointer `BEST` to the best block found so far, and a value `B` equal to its `SIZE`. Initially `BEST ← Λ` and `B ← ∞` (or any value larger than all possible block sizes). For each block `P` in the list, the test is performed: if `SIZE(P) ≥ N` and `SIZE(P) < B`, then update `BEST ← P` and `B ← SIZE(P)`. After the entire list has been scanned, `BEST` is the selected block.

In MIX terms, this is obtained by modifying Algorithm A so that the loop is not exited upon the first satisfactory block. Instead, the conditional transfer in step A4 is replaced by a comparison against the current best size, and the continuation link is always followed until `Λ < 0` terminates the list. The final allocation step uses `BEST` rather than the first qualifying `P`.

This modification preserves the structure of the `AVAIL` traversal while changing only the selection criterion from first admissible element to minimum admissible element, which implements the best-fit policy. The resulting algorithm is correct because every admissible block is examined, and the smallest admissible one is retained throughout the scan. ∎
