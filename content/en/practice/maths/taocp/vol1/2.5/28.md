**Exercise 28.** [**] [25] Assume that MIX is a binary computer, with a new operation code
`XOR` defined as follows (using the notation of Section 1.3.1): "`C = 5`,
`F = 5`. For each bit position in location `M` that equals 1, the
corresponding bit position in register A is complemented (changed from 0
to 1 or 1 to 0); the sign of rA is unaffected. The execution time is
$2u$." Write a MIX program for Algorithm S, and determine its running
time.

Algorithm S maintains an array of free lists $F[0],F[1],\ldots,F[m]$, where $F[j]$ is a singly linked list of blocks of size $2^j$. Each block at address $A$ contains a link field $\mathrm{LINK}$ storing the next element of its free list. Allocation of a block of size $2^k$ proceeds by finding the smallest $j \ge k$ such that $F[j] \ne 0$, removing one block from $F[j]$, and repeatedly splitting it down to level $k$. The XOR instruction satisfies $A \oplus 2^t$, which flips bit $t$ of $A$, hence produces the buddy of a block at level $t$.

A correct MIX implementation must therefore include a correct linked-list deletion routine, correct maintenance of free-list heads, and correct splitting via XOR.

### MIX program

```
S      LDA K              ; requested level k
       STA RLEV

; search for smallest j >= k with F[j] nonempty
S1     LDX RLEV
       LDA F,X
       JNZ S2
       LDA RLEV
       ADD =1=
       STA RLEV
       CMP M
       JL S1
       HLT                ; no block available

; remove first node from F[RLEV]
S2     STA B              ; B := allocated block
       LDX RLEV
       LDA F,X
       STA CUR            ; CUR := head of list

       LDA =0=
       STA PREV           ; PREV := 0

; search for B in list (B is head, so CUR = B initially)
S3     LDA CUR
       CMP B
       JE  S4

       STA PREV           ; PREV := CUR

       LDA LINK,CUR
       STA CUR
       JMP S3

; unlink CUR = B from list
S4     LDX RLEV
       LDA PREV
       JNZ S5

       ; removing head
       LDA LINK,B
       STA F,X
       JMP S6

S5     ; removing interior node
       LDA LINK,B
       STA LINK,PREV

S6     ; B is now isolated free block at level RLEV

; split down to level k
S7     LDX RLEV
       CMP K
       JE  SEND

       LDA RLEV
       SUB =1=
       STA RLEV

       LDA B
       XOR P2,RLEV
       STA BUD

       ; insert buddy into F[RLEV]
       LDX RLEV
       LDA F,X
       STA LINK,BUD
       LDA BUD
       STA F,X

       JMP S7

SEND   LDA B
       STA A
       HLT
```

### Correctness of list deletion

At entry to S3, $CUR$ is initialized to the head of $F[RLEV]$, hence $CUR$ is a valid node of that list. The loop S3 maintains the invariant that $CUR$ is the current node and $PREV$ is the predecessor of $CUR$ (or $0$ if $CUR$ is the head). Each iteration either halts when $CUR = B$, or advances both pointers along the list using $\mathrm{LINK}$.

When $CUR = B$, there are two cases.

If $PREV = 0$, then $B$ is the head of $F[RLEV]$, so $F[RLEV] \leftarrow \mathrm{LINK}(B)$ correctly removes the first node.

If $PREV \ne 0$, then $\mathrm{LINK}(PREV)$ is replaced by $\mathrm{LINK}(B)$, so the predecessor bypasses $B$ and the list remains correctly linked.

Thus exactly one occurrence of $B$ is removed and all other nodes remain correctly connected.

### Correctness of splitting

At level $j = \text{RLEV}$, the block $B$ represents a free block of size $2^j$. After decrementing $\text{RLEV}$, the expression

$$
B \oplus 2^{j-1}
$$

produces the unique buddy of $B$ at level $j-1$. These two blocks are disjoint and partition the original block. Insertion of the buddy into $F[j-1]$ preserves the invariant that each free list contains exactly the free blocks of its size class, since the original block $B$ is retained for further splitting.

The loop decreases the level by one each iteration and terminates when level $k$ is reached, so exactly $j-k$ splits occur.

### Running time

Let $m$ be the maximum level.

The search phase scans free lists starting at $k$ and increases $j$ until the first nonempty list is found. Each list head test is a constant number of MIX instructions, so the cost is proportional to the number of levels examined, at most $m-k+1$.

The deletion phase performs one linear traversal of a single free list $F[j]$. Each node is visited once, and each visit uses a constant number of MIX instructions. Over the course of repeated allocations, each block enters and leaves a free list at most once per allocation, so the total number of node visits is amortized constant per block.

The splitting phase performs exactly one iteration per level decrease from $j$ to $k$, hence at most $m-k$ iterations. Each iteration executes a constant number of arithmetic, XOR, and pointer-update instructions.

Therefore the total running time of one execution of Algorithm S is bounded by a constant times the height of the buddy system tree:

$$
\boxed{\Theta(m)}.
$$
