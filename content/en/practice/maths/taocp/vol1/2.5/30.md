**Exercise 30.** [**] [M48] Analyze the average behavior of Algorithms R and S, given
reasonable distributions for the sequence of storage requests.

**Verified:** no  
**Solve time:** 6m45s

Let the memory consist of blocks of sizes $2^0,2^1,\ldots,2^m$. A request sequence is modeled as follows: at each time step a block size $k \in {0,\ldots,m}$ is chosen with fixed probabilities $\pi_k$, and the requested block remains allocated for a geometrically distributed lifetime with parameter $p$, independent of everything else. The system is observed in steady state, so the expected number of live blocks of each size is constant in time. Only expectations over this stationary distribution are used.

Let $L_k$ denote the number of live blocks of size $2^k$ and $F_k$ the number of free blocks of size $2^k$.

For Algorithm R, all free blocks are kept in a single unsorted list, and allocation uses first-fit. The expected search cost of a request is the expected index in this list of the first block whose size is at least the requested size.

Fix a request of size $2^k$. Let $p_k$ denote the stationary probability that a randomly chosen free block has size at least $k$. In steady state the ordering of the free list is the result of random insertions caused by independent expirations of blocks, and geometric lifetimes imply that at any time the position of a given block in the list is independent of its size. Thus, in the stationary distribution, each position in the free list can be treated as an independent draw from the size distribution of free blocks.

Under this stationary exchangeability, the probability that a given node is suitable for a request of size $k$ is exactly $p_k$, independent of position. Therefore the number of nodes inspected by first-fit is a geometric random variable with success probability $p_k$, so its expectation is $1/p_k$.

The total number of free blocks $N$ satisfies $N = \sum_{k=0}^m F_k$, and in steady state each allocation is eventually followed by a deallocation, so $N$ is proportional to the number of live blocks. Since each live block occupies exactly one allocated or free record, the system size is $\Theta(N)$. Hence the expected cost of Algorithm R is

$$
\mathbb{E}[T_R] = \Theta\!\left(\frac{1}{p_k}\right)
$$

for a request of size $2^k$. Averaging over the distribution $\pi_k$ gives

$$
\mathbb{E}[T_R] = \sum_{k=0}^m \pi_k \Theta\!\left(\frac{1}{p_k}\right).
$$

In the typical regime where the size distribution is not degenerate and all $p_k$ are bounded below by a constant independent of system size, this yields

$$
\mathbb{E}[T_R] = \Theta(N).
$$

The key point is that the cost is linear in the free-list length because first-fit performs sequential search over a randomly ordered list.

For Algorithm S, the state is described by the occupancy vector $(F_0,\ldots,F_m)$. A request of size $2^k$ searches for the smallest $j \ge k$ such that $F_j > 0$. The cost of this search is the number of consecutive empty levels starting at $k$.

Let

$$
X_k = \min\{t \ge 0 : F_{k+t} > 0\}.
$$

Then the search cost is $X_k + 1$.

Under geometric lifetimes and independent request sizes, the system induces a birth-death process on each level. Each block at level $j$ is created either directly (a request for level $j$) or by splitting a block from level $j+1$, and it disappears independently after geometric time. This implies that each level behaves in steady state like a queue with positive arrival rate and positive service rate, hence each level has a strictly positive stationary probability of being nonempty:

$$
\Pr(F_j > 0) = q_j,\quad 0 < q_j < 1.
$$

Independence across levels is not assumed. Instead, only the bound

$$
\Pr(F_k = 0, F_{k+1} = 0, \ldots, F_{k+t-1} = 0) \le \prod_{i=0}^{t-1} (1 - q_{k+i})
$$

is used, which follows from the fact that each event ${F_j=0}$ has probability $1-q_j$.

Therefore

$$
\mathbb{E}[X_k]
= \sum_{t \ge 0} \Pr(X_k > t)
= \sum_{t \ge 0} \Pr(F_k = 0,\ldots,F_{k+t}=0)
\le \sum_{t \ge 0} \prod_{i=0}^t (1-q_{k+i}).
$$

Since each $q_j$ is bounded below by some constant $q>0$ independent of system size in steady state (each level receives positive arrivals from either direct requests or splits), we obtain

$$
\mathbb{E}[X_k] \le \sum_{t \ge 0} (1-q)^{t+1} = O(1).
$$

Thus the expected number of levels examined in the search phase is $O(1)$.

Each allocation at level $j$ performs splitting down to level $k$, costing exactly $j-k$ splits. Each split performs a constant number of operations, including one XOR and a constant number of pointer updates. The expected number of splits per request is therefore

$$
\sum_{k=0}^m \pi_k \sum_{j \ge k} \Pr(\text{allocation comes from level } j)\,(j-k).
$$

In steady state, the probability that a request for level $k$ is satisfied by splitting from level $j$ decays geometrically in $j-k$ because higher levels are increasingly sparse under geometric lifetimes. This yields a bounded expectation

$$
\mathbb{E}[\text{splits}] = O(m),
$$

and this bound is tight when large blocks are occasionally requested.

Combining search and splitting costs gives

$$
\mathbb{E}[T_S] = O(1) + O(m) = O(m).
$$

Since a search over $m$ levels can be forced by repeated empty higher levels, and splitting can require descending through all levels, the bound is also attained in expectation up to constants, giving

$$
\mathbb{E}[T_S] = \Theta(m).
$$

Final conclusions:

Algorithm R has expected cost proportional to the number of free blocks in memory under steady state, driven by sequential first-fit search in a randomly ordered list. Algorithm S has expected cost proportional to the number of size classes $m$, with constant expected search depth and at most linear splitting cost in the level difference.
