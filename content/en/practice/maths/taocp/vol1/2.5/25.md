**Exercise 25.** [**] [22] Criticize the following idea: "Dynamic storage allocation using
the buddy system will never reserve a block of size $2^m$ in practical
situations (since this would fill the whole memory), and, in general,
there is a maximum size $2^n$ for which no blocks of greater size will
ever be reserved. Therefore it is a waste of time to start with such
large blocks available, and to combine buddies in Algorithm S when the
combined block has a size larger than $2^n$."

**Verified:** no  
**Solve time:** 3m14s  

---

The criticism of this idea is that it conflates typical usage patterns with the correctness and generality of the buddy system. The buddy system must be able to handle _any_ allocation request, including those requiring very large blocks. Even if, in practice, allocations rarely reach size $2^m$, the system must still support them to maintain correctness; otherwise, Algorithm S could fail when a legitimate request for a large block occurs. Furthermore, starting with the largest blocks available ensures that the splitting and coalescing operations in Algorithm S function uniformly across all block sizes. Omitting large blocks would not save time in a significant way, because the combination of smaller blocks still requires the same checking and merging logic, and it would artificially limit the addressable memory. Therefore, the premise that large blocks can be ignored is incorrect; the buddy system requires availability of all block sizes up to $2^m$ to operate properly.
