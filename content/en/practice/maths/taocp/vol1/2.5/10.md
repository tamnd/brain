[Section 2.5: Dynamic Storage Allocation](../)

**Exercise 10.** [**] [22] Show how to modify Algorithm B so that the block of `N`
consecutive cells beginning at location `P0` is made available, without
assuming that each of these `N` cells is currently unavailable; assume,
in fact, that the area being freed may actually overlap several blocks
that are already free.

**Verified:** yes  
**Solve time:** 1m37s  

---

Let the procedure for freeing storage be the same as Algorithm B except that it must not assume that the cells $P_0, P_0+1, \ldots, P_0+N-1$ are all currently allocated.

The modification is to process each cell of the interval ${P_0, P_0+1, \ldots, P_0+N-1}$ independently using the standard “return-to-AVAIL” mechanism, while allowing coalescing whenever adjacency with an already free block occurs.

For $i = 0,1,\ldots,N-1$, let $P \leftarrow P_0+i$. For each such $P$, first test whether $P$ is already in `AVAIL`. If it is, no change is made. If it is not, then insert $P$ into `AVAIL` as a new free block of length $1$, and immediately merge it with its predecessor or successor block in memory if either is free, exactly as in Algorithm B. The merging step is applied repeatedly until no adjacent free blocks remain.

Because every cell is treated individually, any pattern of overlap between the interval ${P_0,\ldots,P_0+N-1}$ and existing free blocks is resolved locally: already-free segments are ignored, newly freed cells are inserted, and adjacent free blocks are coalesced. Since Algorithm B already guarantees correct coalescing of adjacent free blocks, repeated application over the interval produces a correct representation of the union of the newly freed region with the existing `AVAIL` structure.

The result is that the entire interval becomes free regardless of its previous state, and no assumption about initial occupancy is required.
