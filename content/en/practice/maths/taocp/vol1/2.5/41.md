[Section 2.5: Dynamic Storage Allocation](../)

**Exercise 41.** [**] [M27] The purpose of this exercise is to consider the worst-case
memory usage of the buddy system. A particularly bad case occurs, for
example, if we start with an empty memory and proceed as follows: First
reserve $n=2^{r+1}$ blocks of length 1, which go into locations 0 through
$n-1$; then for $k=1,2,\ldots,r$, liberate all blocks whose starting
location is not divisible by $2^k$, and reserve $2^{-k-1}n$ blocks of
length $2^k$, which go into locations ${1\over2}(1+k)n$ through
${1\over2}(2+k)n-1$. This procedure uses $1+{1\over2}r$ times as much
memory as is ever occupied.

Prove that the worst case cannot be substantially worse than this: When
all requests are for block sizes $1,2,\ldots,2^r$, and if the total space
requested at any time never exceeds $n$, where $n$ is a multiple of
$2^r$, the buddy system will never overflow a memory area of size
$(r+1)n$.

**Solve time:** 5m13s

Let level $k$ denote blocks of size $2^k$, for $0 \le k \le r$. At any time let $A_k$ be the number of allocated blocks of size $2^k$. Since each such block contributes $2^k$ units of requested space, the hypothesis on the total request size gives
$$
\sum_{k=0}^r A_k 2^k \le n.
$$

In the buddy system every allocated block is represented by a node in the binary partition tree, and every time a block is split both halves remain present in memory as free or allocated units. Fix a level $k$. Every block at this level is contained in a unique interval of length $2^k$, and these intervals are disjoint except possibly for blocks belonging to a single partially split ancestor region. The total length of all level-$k$ intervals that contain allocated blocks is exactly $\sum A_k 2^k$, and every additional fragment created by splitting can occupy space only inside a region whose total size is bounded by the same sum, hence by $n$. Therefore the total memory occupied by all blocks at level $k$, including both allocated blocks and all buddies created above them, is at most $n$.

Summing this bound over all levels $k=0,1,\ldots,r$ shows that the total memory used by the buddy system is at most $(r+1)n$. This completes the proof. ∎
