[Section 2.5: Dynamic Storage Allocation](../)

**Exercise 1.** [**] [20] What simplifications can be made to the reservation and
liberation algorithms of this section, if storage requests always appear
in a "last-in-first-out" manner, that is, if no reserved block is freed
until after all blocks that were reserved subsequently have already been
freed?

## Exercises

**Verified:** no  
**Solve time:** 5m52s  

---

Under the last-in-first-out restriction, the dynamic storage problem degenerates into stack allocation. If each reservation is liberated before any earlier reservation can be liberated, the free storage always consists of one contiguous block at the top of the currently used area. Fragmentation cannot occur, and there is no need to maintain an available-space list, boundary tags, predecessor links, successor links, or any search for a suitably large free block.

A single pointer to the current top of storage is sufficient. To reserve a block of size $n$, save the current top address and advance the pointer by $n$. To liberate a block, restore the pointer to the value it had immediately before that reservation. Thus reservation and liberation reduce to stack push and pop operations, each requiring only a constant number of operations and no list manipulation.
