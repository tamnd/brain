**Exercise 33.** [**] [28] (*Garbage collection and compacting.*) Assume that memory
locations $1,2,\ldots,AVAIL-1$ are being used as a storage pool for nodes
of varying sizes, having the following form: The first word of `NODE(P)`
contains the fields

```text
SIZE(P) = number of words in NODE(P);
   T(P) = number of link fields in NODE(P); T(P) < SIZE(P);
LINK(P) = special link field for use only during garbage collection.
```

The node immediately following `NODE(P)` in memory is
`NODE(P + SIZE(P))`. Assume that the only fields in `NODE(P)` that are
used as links to other nodes are `LINK(P + 1), LINK(P + 2), ...,
LINK(P + T(P))`, and each of these link fields is either $\Lambda$ or the
address of the first word of another node. Finally, assume that there is
one further link variable in the program, called `USE`, and it points to
one of the nodes.

Design an algorithm that (i) determines all nodes accessible directly or
indirectly from the variable `USE`, (ii) moves these nodes into memory
locations 1 through `K - 1`, for some `K`, changing all links so that
structural relationships are preserved, and (iii) sets `AVAIL <- K`.

For example, consider the following contents of memory, where `INFO(L)`
denotes the contents of location `L`, excluding `LINK(L)`:

```text
1:  SIZE = 2, T = 1       6:  SIZE = 2, T = 0       AVAIL = 11,
2:  LINK = 6, INFO = A    7:  CONTENTS = D          USE = 3.
3:  SIZE = 3, T = 1       8:  SIZE = 3, T = 2
4:  LINK = 8, INFO = B    9:  LINK = 8, INFO = E
5:  CONTENTS = C         10:  LINK = 3, INFO = F
```

Your algorithm should transform this into

```text
1:  SIZE = 3, T = 1       4:  SIZE = 3, T = 2       AVAIL = 7,
2:  LINK = 4, INFO = B    5:  LINK = 4, INFO = E    USE = 1.
3:  CONTENTS = C          6:  LINK = 1, INFO = F
```

Use three passes. In the first pass, mark all accessible nodes and simultaneously assign their future addresses. Let $K$ denote the first free compacted location, initially $K\leftarrow1$. Starting from `USE`, perform a graph traversal using `LINK(P)` as the mark field. When a node at address $P$ is first reached, set `LINK(P)` to $K$, then set $K\leftarrow K+\mathrm{SIZE}(P)$. Every reachable node is marked exactly once, and the value stored in `LINK(P)` is the address that node will occupy after compaction. Continue recursively through the fields `LINK(P+1),\ldots,\mathrm{LINK}(P+T(P))`.

In the second pass, traverse the reachable nodes again. For every nonnull link field `LINK(Q)` that contains the address $R$ of a reachable node, replace it by the new address stored in the mark field of that node, namely `LINK(R)`. Thus every pointer is translated from an old address to its compacted address. Also replace `USE` by `LINK(USE)`. In the third pass, scan memory from location $1$ to `AVAIL-1`; whenever a marked node is found at address $P$, copy its $\mathrm{SIZE}(P)$ words to the address stored in `LINK(P)`. Since the destination intervals $[,\mathrm{LINK}(P),,\mathrm{LINK}(P)+\mathrm{SIZE}(P)-1,]$ were assigned consecutively during the first pass, the reachable nodes are packed without gaps into locations $1,\ldots,K-1$. After all copies have been completed, set `AVAIL \leftarrow K`. Unreachable nodes are ignored and disappear from the compacted storage pool. This produces exactly the required transformation in the example, with the node originally at address $3$ moved to address $1$, the node originally at address $8$ moved to address $4$, `USE` changed to $1$, and `AVAIL` changed to $7$.
