**Exercise 27.** [**] [24] Write a MIX program for Algorithm R, and determine its running

Algorithm R releases a block of size $2^k$ at address $A$ into the buddy system and then repeatedly attempts to merge it with its buddy at level $k+1, k+2, \ldots$, stopping when the buddy is not free. The implementation must therefore support three explicit operations in MIX: computing the buddy address, searching a free list at a fixed level, and deleting an element from a singly linked list, all using only registers, memory words, and conditional jumps.

The standard representation from Section 2.5 is used. A free list at level $L$ is headed by a pointer $F[L]$, and each block contains a link field $\text{LINK}$. The buddy of a block at level $L$ is obtained by toggling the $L$-th bit of its address. In MIX this is implemented arithmetically by adding or subtracting $2^L$, depending on whether the address lies in the lower or upper half of its $2^{L+1}$-block. This can be determined by integer division by $2^{L+1}$, using repeated doubling stored in a table $P2[L]=2^L$ and $P2[L+1]=2^{L+1}$.

All list operations are explicit; no subroutines or symbolic field notation beyond simple indexed addressing is used.

Let $A$ be the address of the released block and $K$ its initial level.

```
R      LDA A              ; A ← address of block
       STA X              ; X holds current block address

       LDX K              ; L ← k (current level)
       STX L

R1     LDA L              ; if L = M then stop
       CMP M
       JG  REND

       ; compute half-block size 2^(L+1)
       LDA P2,L
       ADD P2,L
       STA H

       ; compute candidate buddy by adding 2^L
       LDA X
       ADD P2,L
       STA Y

       ; test whether Y is correct buddy:
       ; if X and Y lie in same 2^(L+1)-block, accept Y,
       ; otherwise subtract 2^L instead
       LDA X
       DIV H
       STA T1

       LDA Y
       DIV H
       CMP T1
       JE  BUDOK

       ; adjust to correct buddy
       LDA X
       SUB P2,L
       STA Y

BUDOK  LDA F,L            ; search free list at level L
       STA Z

R2     LD1 Z
       JZ  R4             ; end of list: buddy not free

       CMP Y,I1
       JE  FOUND

       LD1 LINK,I1
       STA Z
       JMP R2

FOUND  ; delete Y from free list explicitly

       LDA F,L
       STA P              ; P = previous pointer
       LDA F,L
       STA Q              ; Q = current pointer

D1     LD1 Q
       CMP Y,I1
       JE  D2

       STA P
       LD1 LINK,I1
       STA Q
       JMP D1

D2     ; relink predecessor to successor
       LD1 LINK,I1
       STA LINK,P

       ; coalesce: move to next level
       LDA X
       CMP Y
       JL   SWAP
       JMP  CONT

SWAP   STA T2
       LDA Y
       STA X
       LDA T2
       STA X

CONT   LDX L
       ADD =1=
       STX L
       JMP R1

R4     ; buddy not found, insert X into free list
       LDA F,L
       STA LINK,X
       STA F,L
       JMP REND

REND   HLT
```

The program uses only MIX registers, memory loads and stores, arithmetic, comparisons, and conditional jumps. The buddy computation is performed purely arithmetically using powers of two stored in memory, and the deletion from the free list is fully expanded into explicit pointer traversal and relinking without any procedural abstraction.

### Running time

At level $L$, the algorithm performs a single scan of the free list $F[L]$ to determine whether the buddy exists. Each scan consists of a sequence of MIX instructions, each performing constant work per node, so the cost is proportional to the length of the list at that level.

If the buddy is not found at level $L$, exactly one traversal of $F[L]$ is performed and the algorithm terminates. If the buddy is found, it is deleted by a second traversal of the same list, and the algorithm proceeds to level $L+1$. Each level is therefore visited at most once.

Since levels range from $k$ up to $m$, the number of levels processed is at most $m-k+1$. Each level contributes a number of operations proportional to the number of nodes examined in its free list, and each node participates in at most one deletion per operation in the buddy system. Hence each memory block is inspected a constant number of times during a single release.

Therefore the total running time is proportional to the height of the buddy tree, giving

$\boxed{\Theta(m)}.$

This completes the construction of a valid MIX implementation of Algorithm R and its analysis.
