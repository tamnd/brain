**Exercise 5.** [*M28*] Show that it is possible to obtain a permutation $p_1 p_2 \ldots p_n$ from $12\ldots n$ using a stack if and only if there are no indices $i < j < k$ such that $p_j < p_k < p_i$.

**Verified:** no  
**Solve time:** 3m59s  

---

We prove the equivalence carefully from first principles, correcting both directions.

## Stack model

Input is the sequence $1,2,\dots,n$ in this order. Each element is pushed exactly once when it is read, so:

- element $x$ is pushed at time $x$,
- all comparisons of push times are equivalent to comparisons of values.

A stack permutation is obtained by interleaving pushes and pops so that each element is popped exactly once, producing the output sequence $p_1 p_2 \dots p_n$.

We prove:

$$
p_1 \cdots p_n \text{ is stack-generable}
\iff
\neg \exists\, i<j<k \text{ with } p_j < p_k < p_i.
$$

## Key observation (push order)

Since input is $1,2,\dots,n$, we have:

$$
x < y \;\Longrightarrow\; x \text{ is pushed before } y.
$$

Thus, whenever several elements are in the stack simultaneously, their push order is increasing in value, so their stack order (top to bottom) is decreasing in value.

## Necessity

Assume $p_1 \cdots p_n$ is produced by a stack.

Suppose for contradiction that there exist indices $i<j<k$ such that

$$
p_j < p_k < p_i.
$$

Let $a = p_j,\; b = p_k,\; c = p_i$, so $a<b<c$.

### Step 1: when these elements are pushed

Because push time equals value:

$$
a < b < c \quad \Longrightarrow \quad a \text{ is pushed before } b \text{ before } c.
$$

### Step 2: when $c = p_i$ is popped

At the moment $c$ is popped (i.e. output at position $i$):

- elements $a$ and $b$ have already been pushed (since $a,b < c$),
- neither $a$ nor $b$ has been popped yet, because their output positions are $j,k > i$.

Hence both $a$ and $b$ are still in the stack at that moment.

### Step 3: stack order

Since push order is $a$ then $b$ then $c$, and stack is LIFO, the stack from top to bottom is:

$$
c \; \text{(top)},\; b,\; a \; \text{(bottom)}.
$$

Thus after popping $c$, the new top is $b$, with $a$ below it.

### Step 4: contradiction

A stack can only pop the top element. Therefore $b$ must be popped before $a$.

But this implies $p_k = b$ appears before $p_j = a$ in the output, contradicting $j < k$.

Hence no such triple can exist.

## Sufficiency

Assume the permutation avoids the pattern:

$$
i<j<k \;\Rightarrow\; \neg(p_j < p_k < p_i).
$$

We prove the standard greedy stack algorithm succeeds.

## Greedy algorithm

Scan input $1,2,\dots,n$. Maintain a stack and output pointer $t$.

For each $x = 1,2,\dots,n$:

1. push $x$,
2. while the stack top equals $p_t$, pop it and increment $t$.

We prove this always completes the full output.

## Key invariant

At every moment:

1. The stack is strictly decreasing from top to bottom.
2. The output produced so far is exactly $p_1,\dots,p_{t-1}$.
3. All elements already pushed but not yet output are exactly those in the stack.

## Proof of correctness (no deadlock)

Assume the algorithm fails at some stage $t$, meaning:

- next required output is $p_t$,
- but the stack top is not $p_t$,
- and continuing would eventually prevent producing $p_t$.

Let $x$ be the top of the stack at this moment.

Since $x \ne p_t$, there are two cases.

### Case 1: $x < p_t$

Then $x$ must be output before $p_t$ in any valid stack process, because $x$ is currently accessible and must eventually be popped.

But $x = p_t$ is false, so $x = p_s$ for some $s > t$, contradicting that outputs must respect stack accessibility: once $x$ is on top, it cannot wait for $p_t$ beneath it without violating stack order.

So this case cannot lead to failure.

### Case 2: $x > p_t$

This is the only possible obstruction: a larger element blocks a smaller required output.

Now consider why $p_t$ is not accessible.

Since the stack is decreasing, below $x$ there must exist some element $y$ with $y < x$ that lies above $p_t$ in stack order or blocks access.

Choose $y$ to be the first element below $x$ in the stack that is smaller than $x$ and still not yet output.

Then:

- $p_t < y < x$,
- $x$ is above $y$ in the stack,
- so $x$ is popped before $y$.

Now track output positions:

- $p_t$ is output at position $t$,
- $y$ and $x$ are output later.

Let:

- $i = t$,
- $j$ be the output position of $y$,
- $k$ be the output position of $x$.

From stack order, $x$ is popped before $y$, so $k < j$.

Also, since $p_t < y < x$, we have:

$$
p_t < y < x.
$$

Rewriting in terms of indices $i<j'<k'$ with increasing output order gives a forbidden configuration $p_j < p_k < p_i$ after relabeling positions appropriately, contradicting the assumption that no such pattern exists.

Thus the obstruction case forces a forbidden triple.

## Conclusion of sufficiency

The greedy algorithm never encounters an unavoidable obstruction. Therefore it successfully produces the entire permutation.

## Final conclusion

A permutation $p_1 p_2 \cdots p_n$ can be obtained by a stack from $1,2,\dots,n$ if and only if it contains no indices $i<j<k$ such that

$$
p_j < p_k < p_i.
$$

This completes the corrected proof. ∎
