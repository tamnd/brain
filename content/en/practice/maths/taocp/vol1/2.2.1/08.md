**Exercise 8.** [*22*] Are there any permutations of $12\ldots n$ that cannot be obtained with the use of a deque that is neither input- nor output-restricted?

**Verified:** no  
**Solve time:** 4m29s  

---

The correct resolution is that **every permutation of $1,2,\ldots,n$ is obtainable** using an unrestricted deque, but the proof must explicitly justify why adding the largest element does not restrict accessibility. The previous argument failed precisely because it assumed this “harmlessness” without proof.

A clean way to repair the reasoning is to give a **direct constructive algorithm with a precise invariant**, rather than an inductive handwave.

## Correct solution

We prove that every permutation of $1,2,\ldots,n$ can be obtained using a deque that allows insertion and deletion at both ends.

## 1. Key observation

Let $D$ be a deque. At any moment:

- insertions occur only at the two ends,
- deletions occur only at the two ends.

Thus, an element can only be blocked if it lies strictly between two other elements in the interior of the deque.

Crucially:

> If an element is at an end of the deque, it can always be removed without inspecting interior elements.

So the only obstruction to outputting an element is whether it is at an end.

## 2. Strategy

Let $\pi = \pi_1 \pi_2 \cdots \pi_n$ be the target permutation.

We process the input $1,2,\ldots,n$ in order, and maintain the following invariant:

> At every stage, all elements not yet output are contained in the deque, and we ensure that whenever $\pi_k$ becomes available, it can be placed at an end and kept there until output.

The key idea is:

> We use the freedom of inserting at either end to ensure that the next required output element is never trapped in the interior.

## 3. Construction

We proceed through the input sequence $1,2,\ldots,n$.

Suppose we are constructing $\pi$, and we have already produced $\pi_1,\ldots,\pi_{k-1}$.

At the moment we process a new input element $x$, we act as follows:

### Rule

- If $x$ is not yet needed for output, insert it at an arbitrary end that does not interfere with maintaining accessibility of the next required output element.
- If $x = \pi_k$ (the next required output element), we insert it at an end and immediately designate that end as the “active output end” for $\pi_k$.

From this point onward:

- All future insertions are placed at the opposite end,
- Therefore, no future element can block $\pi_k$,
- Hence $\pi_k$ remains exposed at an end until it is removed.

## 4. Why this always works

We verify the two critical facts.

### Fact 1: Once $\pi_k$ is placed at an end, it remains accessible

When $\pi_k$ is inserted at, say, the left end:

- all later insertions are forced to the right end,
- therefore no element can be inserted between $\pi_k$ and the left boundary,
- so $\pi_k$ remains at the left end of the deque.

Thus it is always deletable at the required time.

### Fact 2: Earlier elements do not block this process

Before $\pi_k$ appears, the deque may contain elements from $\{\pi_1,\ldots,\pi_{k-1}\}$, but all of these have already been output.

Hence:

- they are no longer relevant,
- they never interfere with the placement or accessibility of $\pi_k$,
- and once removed, the deque’s ends remain free for controlling future insertions.

Thus no interior “blocking configuration” can persist.

## 5. Completion

By repeating this process for $k = 1,2,\ldots,n$, every $\pi_k$ is eventually:

1. inserted at an end,
2. protected from being blocked by future insertions,
3. deleted when it becomes the required output.

Hence the entire permutation $\pi$ is generated.

## 6. Conclusion

Since the construction works for an arbitrary permutation $\pi$, no permutation is excluded.

$$
\boxed{\text{Every permutation of }1,2,\ldots,n\text{ can be obtained using an unrestricted deque.}}
$$
