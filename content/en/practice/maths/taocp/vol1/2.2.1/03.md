**Exercise 3.** [*25*] The operations (i) through (viii) in the previous exercise can be much more concisely described by the code `SSXSSXXX`, where `S` stands for "move a car from the input into the stack," and `X` stands for "move a car from the stack into the output." Some sequences of `S`'s and `X`'s specify meaningless operations, since there may be no cars available on the specified track; for example, the sequence `SXXSSXXS` cannot be carried out, since we assume that the stack is initially empty.

Let us call a sequence of `S`'s and `X`'s *admissible* if it contains $n$ `S`'s and $n$ `X`'s, and if it specifies no operations that cannot be performed. Formulate a rule by which it is easy to distinguish between admissible and inadmissible sequences; show furthermore that no two different admissible sequences give the same output permutation.

## Solution

Let a sequence of operations consist of symbols $S$ and $X$, where $S$ inserts the next input car into the stack and $X$ removes the top stack car to the output. The input contains $n$ cars, so a complete execution that outputs a permutation must contain exactly $n$ symbols $S$ and $n$ symbols $X$.

A sequence is admissible precisely when it can be executed without attempting to remove from an empty stack. During execution, the stack is empty exactly when the number of performed $S$ operations equals the number of performed $X$ operations up to that point. Therefore, at every prefix of the sequence, the number of $S$ symbols must be at least the number of $X$ symbols, and at the end these numbers must be equal. Conversely, if a sequence satisfies these two conditions, then at every step where an $X$ occurs the stack is nonempty, so no operation is undefined and the sequence can be carried out.

Thus a sequence is admissible if and only if it contains $n$ occurrences of $S$ and $n$ occurrences of $X$, and every prefix satisfies

$$
\#S \ge \#X.
$$

Now consider two admissible sequences that produce output permutations by executing the same stack mechanism. Suppose they differ, and let $t$ be the first index at which they differ. The prefixes of length $t-1$ are identical, so after $t-1$ operations both sequences have processed the same number of input cars and have identical stack contents.

At step $t$, one sequence performs $S$ and the other performs $X$, since these are the only possible symbols.

Assume the first sequence performs $X$ at step $t$ and the second performs $S$. The stack before step $t$ is nonempty, and let $a$ be its top element. The first sequence outputs $a$ at step $t$. The second sequence instead pushes the next input element, call it $b$, onto the stack, so after step $t$ the top of the stack is $b$ and $a$ lies directly beneath it.

From this point onward, stack operations preserve the relative order of $a$ and $b$ until one of them is removed. Since $b$ lies above $a$ in the second sequence, every admissible continuation of that sequence must output $b$ before $a$. In the first sequence, the element $a$ has already been output, so the relative ordering of outputs between $a$ and $b$ differs between the two sequences. Therefore the resulting output permutations are distinct.

The same argument applies when the roles of the two sequences are reversed at position $t$.

Hence distinct admissible sequences cannot produce the same output permutation.

This completes the proof. ∎
