**Exercise 13.** &#9654; [**] [22] Why were circular lists used in Fig. 14 instead of straight linear lists? Could Algorithm `S` be rewritten so that it does not make use of the circular linkage?

**Verified:** no  
**Solve time:** 53s  

---

Circular lists are used in Fig. 14 so that each row and each column forms a closed cycle with its head node. Traversal of a row or column can proceed by repeatedly following a link field until the head node is encountered again; no special case is needed for the end of the list. This removes the need for null pointers in the link structure and eliminates boundary tests during insertion, deletion, and traversal, since every node, including the head, has both a predecessor and a successor in the circular ordering. The head nodes `BASEROW[i]` and `BASECOL[j]` act as permanent representatives of their lists, so that empty rows or columns are represented by a single self-linked head node.

Straight linear lists could be used instead by terminating each row and column with null links and storing explicit pointers to the first and last elements. Algorithm `S` can then be rewritten to follow linear linkage by replacing each test for return to a head node with a test for a null link. The resulting formulation requires additional conditional checks to detect the end of a row or column and separate cases for inserting or deleting the first or last element of a list, since such operations must update external head or tail pointers instead of relying on cyclic structure.
