**Exercise 7.** [*10*] (a) What is the meaning of `4B` in line `34` of Program `P`? (b) What effect, if any, would be caused if the location of line `15` were changed to `2H` and the address of line `20` were changed to `2B`?

**Verified:** no  
**Solve time:** 6m19s  

---

(a) In MIXAL, a label of the form $kB$ denotes a backward reference to the most recent preceding occurrence of the local label $kH$. Thus $4B$ in line 34 refers to the nearest line above line 34 whose LOC field is $4H$. In Program $P$, this is line 15, so the instruction `J5P 4B` transfers control to line 15.

(b) If the location of line 15 is changed from $4H$ to $2H$, then there is no longer any occurrence of $4H$ above line 34. The reference $4B$ in line 34 would therefore have no defined target within the program text, since no preceding $4H$ exists from which to count backward. The jump in line 34 becomes undefined in the sense of MIXAL assembly, and the program cannot be correctly assembled as written.

If, in addition, the address of line 20 is changed from $4B$ to $2B$, then line 20 no longer returns to the $4H$ location. The operand $2B$ refers to the most recent preceding line labeled $2H$ above line 20, which is line 12. Thus when line 20 is executed with $R = 0$, control transfers to line 12 instead of the original $4H$ target at line 15. This redirects the flow from step P4 back into step P2, causing the program to re-enter the “prime found” update phase rather than advancing $N$, thereby altering the logical structure of Algorithm $P$ and breaking its intended iteration over candidates. ∎
