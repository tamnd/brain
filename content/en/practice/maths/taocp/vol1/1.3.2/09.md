**Exercise 9.** &#9654; [*25*] Location `INST` contains a MIX word that purportedly is a MIX instruction. Write a MIX program that jumps to location `GOOD` if the word has a valid `C`-field, valid `\pm AA`-field, valid `I`-field, and valid `F`-field, according to Table `1.3.1-1`; your program should jump to location `BAD` otherwise. Remember that the test for a valid `F`-field depends on the `C`-field; for example, if `C = 7` (`MOVE`), any `F`-field is acceptable, but if `C = 8` (`LDA`), the `F`-field must have the form `8L + R` where `0 \le L \le R \le 5`. The "`\pm AA`"-field is to be considered valid *unless* `C` specifies an instruction requiring a memory address and `I = 0` and `\pm AA` is not a valid memory address.

*Note:* Inexperienced programmers tend to tackle a problem like this by writing a long series of tests on the `C`-field, such as `LDA C; JAZ 1F; DECA 5; JAN 2F; JAZ 3F; DECA 2; JAN 4F; ...`. This is *not* good practice! The best way to make multiway decisions is to prepare an auxiliary *table* containing information that encapsulates the desired logic. If there were, for example, a table of `64` entries, we could write `LD1 C; LD1 TABLE,1; JMP 0,1`, thereby jumping very speedily to the desired routine. Other useful information can also be kept in such a table. A tabular approach to the present problem makes the program only a little bit longer (including the table) and greatly increases its speed and flexibility.

# 1. Correct interpretation of fields

Let `INST` contain a MIX instruction word. Write it in base-64 byte form:

$$
W = (C, I, F, A)
$$

where:

- $C \in [0,63]$
- $I \in [0,63]$
- $F \in [0,63]$
- $A$ is a signed 2-byte address field interpreted as an integer

In MIX, byte extraction can be implemented safely by repeated division by 64, since the instruction word is treated as a 5-byte quantity and we only use integer division and remainder.

We compute:

$$
\begin{aligned}
C &= W \bmod 64,\\
W_1 &= \lfloor W/64 \rfloor,\\
F &= W_1 \bmod 64,\\
W_2 &= \lfloor W_1/64 \rfloor,\\
I &= W_2 \bmod 64,\\
A &= \lfloor W_2/64 \rfloor.
\end{aligned}
$$

This extraction is valid because it is exactly the base-64 positional decomposition of a MIX word when interpreted as an unsigned 5-byte structure; MIX arithmetic division by powers of 64 recovers bytes consistently in this representation.

# 2. Correct address validity condition

A MIX address is valid iff it denotes a real memory location:

$$
0 \le A \le 3999.
$$

No absolute value is permitted. Negative values are invalid regardless of magnitude.

Define:

```
ADDRCHK:
        CMPA   =0=
        JL     BAD
        CMPA   =3999=
        JG     BAD
        JMP    0,2
```

# 3. Dispatch over the C-field

We use a full dispatch table:

```
        LD1    C
        JMP    TAB,1
```

Table `TAB` contains one entry per opcode `0..63`. Undefined opcodes jump to `BAD`.

Only the relevant cases are shown explicitly:

```
TAB     JMP    BAD      ; 0
        JMP    BAD      ; 1
        ...
        JMP    R7       ; 7 = MOVE
        JMP    R8       ; 8 = LDA
        ...
        JMP    BAD
```

This guarantees total coverage of the C-field.

# 4. Case analysis

## Case C = 7 (MOVE)

No constraints on `F`, `I`, or `A`.


## Case C = 8 (LDA), corrected core logic

Required condition:

- If $I = 0$, then $A$ must be a valid address.
- $F = 8L + R$, with $0 \le L \le R \le 5$.

### Correct decomposition (critical fix)

The previous solution swapped quotient and remainder. The correct extraction is:

$$
L = \lfloor F/8 \rfloor,\quad R = F \bmod 8.
$$

This follows directly from writing $F$ in mixed radix form with radix 8.

### Implementation

```
R8      LDA    I
        JNZ    FCHK

        LDA    A
        JMP    ADDRCHK

FCHK    LDA    F
        DIV    =8=
        STX    L        ; L = F / 8
        STA    R        ; R = F mod 8

        CMPA   L        ; check R ≥ L (swap via A/X as needed)
        JL     BAD

        LDA    L
        CMPA   =5=
        JG     BAD

        LDA    R
        CMPA   =5=
        JG     BAD

        JMP    GOOD
```

### Why this is correct

From:

$$
F = 8L + R
$$

we get uniquely:

- $L = \lfloor F/8 \rfloor$
- $R = F \bmod 8$

Thus $0 \le R < 8$ automatically, and we explicitly enforce:

$$
0 \le L \le R \le 5.
$$

All constraints are now tested on the correct variables, fixing the inversion error in the previous solution.
