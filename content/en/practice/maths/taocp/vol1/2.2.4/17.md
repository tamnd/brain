**Exercise 17.** [*22*] What advantage is there in representing polynomials with a circular list as in this section, instead of with a straight linear linked list terminated by $\Lambda$ as in the previous section?

## Solution

Let a polynomial be represented as in Section 2.2.4, where each node contains a term ordered by decreasing $ABC$ field and a special node with $ABC=-1$ and $COEF=0$ links back to the first term, forming a circular list.

In the linear representation of Section 2.2.3, each list terminates with $\Lambda$, so every traversal requires detecting the last node by testing whether `LINK(P) = Λ`. Every operation that scans or modifies the end of the list depends on a distinguished terminal condition that must be handled separately from ordinary nodes.

In the circular representation, the link structure contains no $\Lambda$ within the list itself. Instead, every node has a successor, and termination of traversal is detected by returning to a known reference node, typically the special node with $ABC=-1$. Thus a scan over a polynomial starting from any node proceeds by repeated application of `P ← LINK(P)` until $P$ returns to the starting node, and no structural distinction between intermediate nodes and terminal nodes is required.

The advantage is that all list operations that depend on traversal become uniform in form, since the same `LINK` operation applies at every step without checking for a null pointer. In particular, procedures such as polynomial addition (Algorithm A) and similar merging algorithms require comparisons of ordered keys and pointer advancement only; they do not require separate logic for detecting the end of a list. In the linear representation, termination requires repeated testing against $\Lambda$, which introduces an additional structural case at every step of traversal. In the circular representation, termination is absorbed into the equality test against the distinguished node, which is already required in polynomial processing to detect completion of a full cycle through the ordered terms.

A further advantage appears in concatenation and splitting operations. In a linear list, concatenating two lists requires locating the last node of the first list, which is only identifiable by traversal until $\Lambda$ is encountered. In a circular list with a right pointer `PTR`, concatenation of two nonempty lists can be performed by exchanging links at the two pointer positions as in equation (3), since `LINK(PTR)` always exists and no search for a terminal node is needed. The operation depends only on local pointer manipulation.

The representation with a special node also eliminates the empty-list case. Since the special node is always present, the structure never becomes $\Lambda$. This removes the need for separate initialization and underflow cases that arise when a pointer becomes undefined after deleting the last element in a linear list. In the linear model, repeated deletion may produce $\Lambda$, after which further operations require explicit tests. In the circular model, deletion operations such as (c) preserve a valid pointer until the list reduces to the single special node, which then serves as both head and sentinel for termination.

These properties imply that circular representation converts end-of-list conditions into ordinary node conditions and replaces structural termination by cyclic termination. The resulting uniformity reduces the number of distinct pointer cases required in list-processing algorithms and allows list concatenation, traversal, and deletion to be expressed solely in terms of `LINK` and a fixed reference node.
