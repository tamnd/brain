**Exercise 18.** [*25*] Devise a way to represent circular lists inside a computer in such a way that the list can be traversed efficiently in both directions, yet only one link field is used per node. [*Hint:* If we are given two pointers, to two successive nodes $x_{i-1}$ and $x_i$, it should be possible to locate both $x_{i+1}$ and $x_{i-2}$.]

**Verified:** yes  
**Solve time:** 1m06s

## Solution

Each node $x_i$ stores a single link field $\mathrm{LINK}(x_i)$ defined as the exclusive-or of the addresses of its two neighbors in the circular order. Let $\oplus$ denote bitwise exclusive-or on machine words representing addresses. The representation is defined by

$$
\mathrm{LINK}(x_i) = \mathrm{addr}(x_{i-1}) \oplus \mathrm{addr}(x_{i+1}),
$$

where indices are taken modulo $n$ in a circular list of nodes $x_0, x_1, \ldots, x_{n-1}$.

Suppose two successive nodes $x_{i-1}$ and $x_i$ are known via pointers $P_{i-1}$ and $P_i$. The value stored in $\mathrm{LINK}(x_i)$ satisfies

$$
\mathrm{LINK}(x_i) = P_{i-1} \oplus P_{i+1}.
$$

Applying $\oplus P_{i-1}$ to both sides yields

$$
\mathrm{LINK}(x_i) \oplus P_{i-1} = (P_{i-1} \oplus P_{i+1}) \oplus P_{i-1} = P_{i+1},
$$

using associativity and cancellation $a \oplus a = 0$. Hence the successor of $x_i$ is computed as

$$
P_{i+1} = \mathrm{LINK}(x_i) \oplus P_{i-1}.
$$

To obtain the predecessor $x_{i-2}$, use $\mathrm{LINK}(x_{i-1})$. By definition,

$$
\mathrm{LINK}(x_{i-1}) = P_{i-2} \oplus P_i.
$$

Applying $\oplus P_i$ gives

$$
P_{i-2} = \mathrm{LINK}(x_{i-1}) \oplus P_i.
$$

Thus both the next and previous nodes relative to the directed pair $(x_{i-1}, x_i)$ are computable using only one stored link field per node.

Traversal of the circular list proceeds by maintaining two pointers to consecutive nodes. Given $(P_{i-1}, P_i)$, the forward step replaces this pair by $(P_i, P_{i+1})$, where $P_{i+1}$ is computed as above. The backward step replaces $(P_{i-1}, P_i)$ by $(P_{i-2}, P_{i-1})$, where $P_{i-2}$ is computed as above.

The representation is consistent on a circular list because each node has exactly one predecessor and one successor, so the definition of $\mathrm{LINK}(x_i)$ is well defined for all $i$ modulo $n$. ∎
