**Exercise 1.** [*21*] The text suggests at the beginning of this section that an empty circular list could be represented by `PTR = Λ`. It might be more consistent with the philosophy of circular lists to have `PTR = LOC(PTR)` indicate an empty list. Does this convention facilitate operations (a), (b), or (c) described at the beginning of this section?

**Verified:** yes  
**Solve time:** 1m08s

## Solution

Let the proposed convention be that an empty circular list is represented by $PTR = LOC(PTR)$, while a nonempty list is represented as in the text, where $PTR$ is the address of the rightmost node and $LINK(PTR)$ is defined.

The value $LOC(PTR)$ is a fixed memory address distinct from all node addresses and from $\Lambda$. Hence the test for emptiness becomes a single comparison $PTR = LOC(PTR)$ instead of $PTR = \Lambda$.

Operation (a) requires the assignment

$$
P \leftarrow AVAIL,\quad INFO(P) \leftarrow Y,\quad LINK(P) \leftarrow LINK(PTR),\quad LINK(PTR) \leftarrow P.
$$

When the list is empty under the new convention, $PTR = LOC(PTR)$ and there is no node designated by $PTR$. The expression $LINK(PTR)$ is then undefined because $PTR$ does not point to a node record containing a $LINK$ field. Therefore operation (a) cannot be applied uniformly; it requires a separate case in which the first inserted node must be made to point to itself and $PTR$ must be reset to that node. This introduces at least one additional conditional structure beyond the single test for emptiness.

Operation (c) requires accessing $P \leftarrow LINK(PTR)$ and then updating $LINK(PTR) \leftarrow LINK(P)$. When $PTR = LOC(PTR)$, neither $LINK(PTR)$ nor $LINK(P)$ is defined, since no list structure exists. Thus deletion from an empty list must still be excluded by an explicit test, and the empty case again requires a separate convention, namely generation of an underflow condition before any pointer manipulation.

Operation (b) inherits all difficulties of (a), since it is defined through (a) followed by $PTR \leftarrow P$. In the empty case, the same special-case insertion rule is required before the pointer can be interpreted as a node address.

The only simplification provided by the convention $PTR = LOC(PTR)$ is that emptiness can be detected without comparing against $\Lambda$. No structural simplification occurs in the pointer manipulation steps, since every operation still requires a distinct handling of the empty case where no node fields exist.

The additional burden introduced is that $PTR$ is no longer uniformly a node address in the empty case, so any expression involving $LINK(PTR)$ must be guarded by a condition excluding $PTR = LOC(PTR)$. This reduces uniformity of the algorithms and increases the number of exceptional branches relative to the $\Lambda$ convention.

Hence the proposed representation does not facilitate operations (a), (b), or (c), since it replaces a single special value $\Lambda$ with a special self-referential value that still forces separate cases and further invalidates direct pointer dereferencing in the empty state. ∎
