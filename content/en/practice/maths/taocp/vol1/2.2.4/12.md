**Exercise 12.** [*21*] Compare the running time of the program in exercise 11 with that of Program `A` when `polynomial(Q) = 0`.

## Solution

Let $P$ contain $n$ nonzero terms, and let $\operatorname{polynomial}(Q)=0$. In the representation of Section 2.2.4, this means that $Q$ consists of a single special node whose field $ABC=-1$ and whose link points to itself.

### Running time of Exercise 11 (COPY)

The COPY subroutine traverses the circular list of $P$ exactly once, starting from the node $\operatorname{LINK}(P)$ and returning when it reaches the starting node again.

During this traversal, each visited node contributes exactly one new node to the output polynomial via one allocation from `AVAIL`, followed by a fixed number of field assignments:

$$
COEF,\quad ABC,\quad LINK.
$$

Each iteration of the traversal performs a constant number of pointer updates and comparisons to determine whether the traversal has returned to the starting node. Since exactly $n$ non-sentinel nodes are visited, COPY performs $n$ iterations of this pattern, and therefore performs exactly $n$ allocations and a number of field assignments proportional to $n$.

Hence the running time of COPY is linear in $n$.

### Running time of Algorithm A when $\operatorname{polynomial}(Q)=0$

In this case $Q$ initially contains only the sentinel node with $ABC=-1$. After step $A1$, the variables satisfy

$$
P \leftarrow \operatorname{LINK}(P), \quad Q \leftarrow \operatorname{LINK}(Q), \quad Q_1 \leftarrow Q.
$$

Since $\operatorname{LINK}(Q)=Q$, the pointer $Q$ remains at the sentinel node.

At each execution of step $A2$, the comparison is between $ABC(P)$ and $ABC(Q)=-1$. For every non-sentinel node of $P$, the relation

$$
ABC(P) > ABC(Q)
$$

holds, so control proceeds to step $A5$.

In step $A5$, one node is removed from `AVAIL` and inserted into $Q$. The operations in that step consist of one allocation from `AVAIL`, a fixed number of assignments to `COEF`, `ABC`, and `LINK`, and one update of the list links via `LINK(Q1)`. After execution of $A5$, the pointer $P$ advances once to its successor node.

This pattern repeats once for each of the $n$ non-sentinel nodes of $P$, since each iteration consumes exactly one node of $P$ and inserts exactly one node into $Q$. After the final node is processed, control reaches the sentinel condition in step $A3$ when $ABC(P)<0$.

Thus Algorithm A performs exactly $n$ executions of the insertion step $A5$, each requiring a constant number of primitive list operations, together with a constant amount of overhead per iteration from step $A2$ comparisons.

Hence the running time of Algorithm A is also linear in $n$.

### Comparison

Both procedures perform exactly one node allocation per nonzero term of $P$, and both perform a traversal that advances through each node of $P$ exactly once. The COPY subroutine performs only direct traversal and copying operations, while Algorithm A performs, in addition, repeated comparisons in step $A2$ against a fixed sentinel node and repeated execution of the insertion mechanism in step $A5$.

Both running times are therefore proportional to $n$, but COPY uses fewer pointer manipulations and fewer control transfers per processed node.

Therefore COPY has strictly smaller constant factor in its linear running time than Algorithm A in the case $\operatorname{polynomial}(Q)=0$.

$$
\boxed{\text{Both algorithms run in } \Theta(n)\text{ time; COPY has smaller constant overhead.}}
$$

This completes the solution. ∎
