## Solution

Let the given circular list be nonempty and let $PTR$ point to its rightmost node, so that $F = LINK(PTR)$ is the leftmost node in the cyclic order induced by following `LINK` pointers.

If $LINK(PTR) = PTR$, the list consists of a single node, and reversing all arrows leaves the unique pointer unchanged; thus $PTR$ is unchanged.

Assume $LINK(PTR) \ne PTR$. Let

$$
P_0 = LINK(PTR)
$$

be the first node encountered when traversing the list from the leftmost position. The algorithm maintains two pointers, $prev$ and $current$, together with a temporary pointer $next$. Initially set

$$
prev \leftarrow PTR,\quad current \leftarrow P_0.
$$

Each iteration processes one node by redirecting its `LINK` field to its predecessor in the reversed orientation. The update step is

$$
next \leftarrow LINK(current),\quad LINK(current) \leftarrow prev,\quad prev \leftarrow current,\quad current \leftarrow next.
$$

The iteration is repeated until $current = P_0$ holds. At that moment every node has been visited exactly once because traversal of a finite circular list starting at $P_0$ returns to $P_0$ only after passing through every node exactly once. The stopping condition ensures that the final occurrence of $P_0$ as a traversal state is not processed again, so each node’s `LINK` field is modified exactly once.

When the loop terminates, $prev$ equals the original $PTR$, since the traversal visits the nodes in the order starting at $P_0$ and ending at $PTR$ immediately before returning to $P_0$. The assignment performed when processing $P_0$ has already set $LINK(P_0) \leftarrow PTR$, so the edge incident to $P_0$ is already reversed correctly. For every other node $x$, its `LINK` field was set to the preceding node in the traversal order, which is exactly the predecessor of $x$ in the reversed circular order.

Finally, the pointer representing the rightmost node of the reversed list must be updated to the original leftmost node, since that node becomes last in the reversed orientation. Thus set

$$
PTR \leftarrow P_0.
$$

After this reassignment, following `LINK(PTR)` from the new $PTR$ traverses the entire list in reversed order and returns to $PTR$, so the structure is again a properly linked circular list with all arrows reversed.

This completes the construction of the reversal algorithm. ∎
