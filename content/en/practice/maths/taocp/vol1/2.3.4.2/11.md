[Section 2.3.4.2: Oriented Trees](../)

**Exercise 11.** [**] $$28$$ Using the assumptions of exercise 2.3.4.1-6, but with $(a_k,b_k)$ representing an arc from $V_{a_k}$ to $V_{b_k}$, design an algorithm that not only prints out a free subtree as in that algorithm, but also prints out the fundamental cycles. [*Hint:* The algorithm given in the solution to exercise 2.3.4.1-6 can be combined with the algorithm in the preceding exercise.]

**Verified:** yes  
**Solve time:** 1m34s  

---

Run the algorithm of exercise 2.3.4.1-6 to construct a spanning free subtree of the underlying undirected graph while maintaining the parent table $P[1],\ldots,P[n]$ that records the chosen tree edges. Each time an arc $(a_k,b_k)$ is examined, it is classified either as a tree arc or a non-tree arc exactly as in that construction, so that the tree edges form a connected acyclic structure rooted at the first discovered vertex.

For each non-tree arc $(a_k,b_k)$, the endpoints already lie in the constructed tree, so there is a unique simple path in the tree between $V_{a_k}$ and $V_{b_k}$ determined by repeated application of $P[\cdot]$ toward the root. To print the corresponding fundamental cycle, follow the parent pointers from $a_k$ marking all visited vertices, then ascend from $b_k$ until the first marked vertex $V_x$ is reached; the path from $a_k$ up to $V_x$, followed by the path from $V_x$ down to $b_k$, together with the arc $(a_k,b_k)$, forms the required fundamental cycle, which is then output in oriented order consistent with the parent directions. This produces exactly one cycle for each non-tree arc, hence all fundamental cycles are printed. ∎
