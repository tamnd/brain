[Section 2.3.4.2: Oriented Trees](../)

**Exercise 4.** [**] $$M20$$ The concept of *topological sorting* can be defined for any finite directed graph $G$ as a linear arrangement of the vertices $V_1V_2\ldots V_n$ such that $\operatorname{init}(e)$ precedes $\operatorname{fin}(e)$ in the ordering for all arcs $e$ of $G$. (See Section 2.2.3, Figs. 6 and 7.) Not all finite directed graphs can be topologically sorted; which ones can be? (Use the terminology of this section to give the answer.)

**Verified:** yes  
**Solve time:** 1m49s  

---

A finite directed graph admits a topological sorting if and only if it contains no oriented cycle.

If a topological ordering $V_1V_2\ldots V_n$ exists and the graph contained an oriented cycle $(e_1,\ldots,e_m)$ with vertices $V_{i_1},\ldots,V_{i_m}$, then each arc would force strict inequalities

$$
V_{i_1}<V_{i_2}<\cdots<V_{i_m}<V_{i_1},
$$

which is impossible in a linear order. Hence no oriented cycle can exist.

Conversely, assume the directed graph has no oriented cycles. Then each vertex is contained in a finite acyclic relation induced by reachability. A vertex with in-degree $0$ must exist, since otherwise following incoming arcs indefinitely would produce an oriented cycle. Choose such a vertex as $V_1$ and remove it together with all outgoing arcs. The remaining graph is still finite and has no oriented cycles, so the same argument produces $V_2$, and iteration yields a linear ordering $V_1V_2\ldots V_n$. Every arc goes from an earlier vertex to a later vertex, since otherwise it would create a cycle at the moment its endpoints are first separated in the construction. This gives a valid topological sorting.

Therefore a finite directed graph can be topologically sorted exactly when it is acyclic, that is, when it contains no oriented cycles. ∎
