[Section 2.3.4.2: Oriented Trees](../)

**Exercise 25.** [**] $$26$$ Design a computer representation for directed graphs that generalizes the right-threaded binary tree representation of a tree. Use two link fields `ALINK`, `BLINK` and two one-bit fields `ATAG`, `BTAG`; and design the representation so that: (i) there is one node for each *arc* of the directed graph (not for each vertex); (ii) if the directed graph is an oriented tree with root $R$, and if we add an arc from $R$ to a new vertex $H$, then the representation of this directed graph is essentially the same as a right-threaded representation of this oriented tree (with some order imposed on the children in each family), in the sense that `ALINK`, `BLINK`, `BTAG` are respectively the same as `LLINK`, `RLINK`, `RTAG` in Section 2.3.2; and (iii) the representation is symmetric in the sense that interchanging `ALINK`, `ATAG`, with `BLINK`, `BTAG` is equivalent to changing the direction on all the arcs of the directed graph.

Each arc $e$ of the directed graph is represented by a node whose identity is $e$ itself, with fields $\text{ALINK}$, $\text{BLINK}$ and one-bit tags $\text{ATAG}$, $\text{BTAG}$. For a fixed vertex $V$, the set of arcs with $\operatorname{init}(e)=V$ is organized into a singly threaded cyclic list in $\text{ALINK}$ order, where $\text{ALINK}(e)$ is the next arc in a predetermined cyclic ordering of all arcs leaving $V$, and $\text{ATAG}(e)=1$ indicates that $\text{ALINK}(e)$ is a genuine successor in this out-list while $\text{ATAG}(e)=0$ indicates a thread skipping to the next arc in the global traversal order. Symmetrically, for each vertex $V$, the arcs with $\operatorname{fin}(e)=V$ form a cyclic list in $\text{BLINK}$, where $\text{BLINK}(e)$ is the next arc entering $V$ in a fixed ordering, and $\text{BTAG}(e)=1$ distinguishes a true in-list successor from a thread.

In an oriented tree with root $R$, each vertex $V\ne R$ has exactly one outgoing arc $e[V]$, so every $\text{ALINK}$-chain degenerates to a linear threaded structure, while each vertex except $R$ has exactly one incoming arc, so the $\text{BLINK}$-structure forms the unique parent chain. If an extra arc $R\to H$ is added, the resulting structure has exactly one incoming and one outgoing arc at each vertex in the sense required to emulate the right-threaded binary tree representation, with $\text{ALINK},\text{BLINK},\text{BTAG}$ playing the roles of $\text{LLINK},\text{RLINK},\text{RTAG}$ under the identification of arcs with nodes. The construction is symmetric because exchanging $\text{ALINK}$ with $\text{BLINK}$ and $\text{ATAG}$ with $\text{BTAG}$ interchanges the roles of initial and final vertices of every arc, which corresponds precisely to reversing all arc directions in the directed graph. ∎
