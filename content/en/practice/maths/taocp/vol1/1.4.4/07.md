**Exercise 7.** [*22*] Write a subroutine called `WORDIN` that is essentially like (4) except that it does not make use of a sentinel.

**Verified:** no  
**Solve time:** 4m32s  

---

## Solution

Subroutine (4) detects the end of a buffer by placing a sentinel in the 101st word of each buffer. To eliminate the sentinel, the routine must keep an explicit count of the words already used from the current buffer.

Let index register 6 continue to point to the last word returned, exactly as in (4). Introduce a counter in index register 5. The value of register 5 will be the number of words remaining in the current buffer. Whenever a word is delivered, register 5 is decreased by one. When it becomes zero, the current buffer has been exhausted; a new input operation is started into that buffer, the other buffer is selected, and the counter is reset to 100.

The links between buffers used in lines 11 and 14 of (4) are retained, since they provide the most compact method for buffer swapping.

A suitable replacement for (4) is therefore:

```
01 WORDIN STJ  1F                  Store the exit location.
02        INC6 1                   Advance to next word.
03        DEC5 1                   One word less remains.
04 1H     JG5  *                   If buffer not exhausted,
05                                   return.
06        IN   -99,6(U)           Refill this buffer.
07        LD6  1,6                 Switch to other buffer.
08        ENT5 100                 New buffer contains
09                                   100 words.
10        JMP  1B
11 INBUF1 ORIG *+100              First buffer
12        CON  *+1                Address of other buffer
13 INBUF2 ORIG *+100              Second buffer
14        CON  INBUF1             Address of other buffer
```

To verify correctness, suppose register 6 points to the last word delivered from the current buffer and register 5 contains the number of unread words remaining in that buffer. After line 02, register 6 points to the next word. After line 03, register 5 is reduced by one. If the new value of register 5 is positive, there are still unread words in the current buffer, so line 04 returns immediately with register 6 pointing to the desired word.

When register 5 becomes zero, exactly 100 words have been taken from the current buffer since the most recent swap. The current buffer is therefore exhausted. Line 06 begins reading the next tape block into that exhausted buffer. Line 07 follows the stored link to the other buffer, which already contains the anticipated input. Line 08 sets the count of unread words in that new buffer to 100. Control then returns through line 04, and the word addressed by register 6 is delivered.

Thus the subroutine performs the same buffering and buffer-swapping operations as (4), while using an explicit word count instead of a sentinel.

$$
\boxed{\text{A counter of remaining words replaces the sentinel test; the above subroutine is a correct implementation.}}
$$
