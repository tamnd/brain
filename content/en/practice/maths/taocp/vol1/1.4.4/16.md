**Exercise 16.** [*29*] Formulate the "green-yellow-red-purple" algorithm, suggested by Fig. 26, in the manner of the algorithms for multiple buffering given in the text, using three coroutines (one to control the input device, one for the output device, and one for the computation).

## Setup

The exercise asks for a formulation of the "green-yellow-red-purple" buffering scheme of Fig. 26 in the same style as the multiple-buffer algorithms of Section 1.4.4.

The figure suggests four states for each buffer:

$$
\text{green} \rightarrow \text{yellow} \rightarrow \text{red} \rightarrow \text{purple} \rightarrow \text{green}.
$$

A green buffer is empty and available for input. A yellow buffer is being filled by the input device. A red buffer contains completed input and is available for computation. A purple buffer contains computed results waiting to be written by the output device.

Three independent activities are present:

1. Input of records from the source device.
2. Computation on completed input records.
3. Output of completed results.

Accordingly, three coroutines are required:

$$
\text{INPUT},\qquad \text{COMPUTE},\qquad \text{OUTPUT}.
$$

The objective is to maintain the buffer-state cycle

$$
\text{green} \to \text{yellow} \to \text{red} \to \text{purple} \to \text{green}
$$

while allowing input, computation, and output to proceed concurrently whenever buffers are available.

## Solution

Let the buffers form a circular list. Let

$$
G=\text{number of green buffers},
\qquad
R=\text{number of red buffers},
\qquad
P=\text{number of purple buffers}.
$$

A yellow buffer is uniquely associated with the input device whenever an input operation is in progress. Similarly, at most one purple buffer is actively being written by the output device at any instant.

The coroutines communicate only by changing buffer colors.

### INPUT coroutine

I1. If no green buffer exists, transfer control to COMPUTE.

I2. Select a green buffer $B$.

I3. Change the color of $B$ from green to yellow.

I4. Initiate input into $B$.

I5. When the input operation terminates, change the color of $B$ from yellow to red.

I6. Transfer control to COMPUTE.

### COMPUTE coroutine

C1. If no red buffer exists, transfer control to INPUT.

C2. Select a red buffer $B$.

C3. Perform the required computation using the data in $B$.

C4. Change the color of $B$ from red to purple.

C5. Transfer control to OUTPUT.

### OUTPUT coroutine

O1. If no purple buffer exists, transfer control to COMPUTE.

O2. Select a purple buffer $B$.

O3. Initiate output of the contents of $B$.

O4. When the output operation terminates, change the color of $B$ from purple to green.

O5. Transfer control to INPUT.

The initialization places every buffer in the green state. INPUT begins by selecting one or more green buffers and starting the first input operations. The process then continues according to steps I1-I6, C1-C5, and O1-O5.

Whenever a buffer leaves the green state, it enters the sequence

$$
\text{green}
\rightarrow
\text{yellow}
\rightarrow
\text{red}
\rightarrow
\text{purple}
\rightarrow
\text{green},
$$

which is precisely the cycle indicated by Fig. 26.

## Verification

Each buffer changes color exactly four times during one complete passage through the system.

The transition green $\rightarrow$ yellow occurs only in step I3. Since input is initiated immediately afterward in step I4, every yellow buffer is associated with an outstanding input operation.

The transition yellow $\rightarrow$ red occurs only after completion of that input operation in step I5. Hence every red buffer contains a complete input record.

The transition red $\rightarrow$ purple occurs only in step C4, after computation has been performed in step C3. Hence every purple buffer contains processed output data.

The transition purple $\rightarrow$ green occurs only in step O4, after the corresponding output operation has completed. Therefore no buffer is returned to the pool of available buffers until its contents have been written successfully.

No coroutine modifies a buffer in a state that belongs to another coroutine. INPUT acts only on green and yellow buffers, COMPUTE acts only on red buffers, and OUTPUT acts only on purple buffers. Consequently, no race exists between the three activities.

The algorithm therefore implements concurrent input, computation, and output while preserving the required color cycle

$$
\boxed{\text{green} \rightarrow \text{yellow} \rightarrow \text{red} \rightarrow \text{purple} \rightarrow \text{green}}.
$$

## Notes

The ordinary multiple-buffer scheme of Section 1.4.4 uses the cycle

$$
\text{available} \rightarrow \text{input} \rightarrow \text{compute} \rightarrow \text{available}.
$$

The green-yellow-red-purple method inserts an additional stage for output. Input, computation, and output are thereby represented symmetrically by three coroutines, each responsible for exactly one color transition of the cycle.
