**Exercise 19.** [*M28*] Special considerations arise when input or output involves short blocks on a rotating device like a magnetic disk. Suppose a program works with $n \geq 2$ consecutive blocks of information in the following way: Block $k$ begins to be input at time $t_k$, where $t_1 = 0$. It is assigned for processing at time $u_k \geq t_k + T$ and released from its buffer at time $v_k = u_k + C$. The disk rotates once every $P$ units of time, and its reading head passes the start of a new block every $L$ units; so we must have $t_k \equiv (k - 1)L \pmod P$. Since the processing is sequential, we must also have $u_k \geq v_{k-1}$ for $1 < k \leq n$. There are $N$ buffers, hence $t_k \geq v_{k-N}$ for $N \leq k < n$.

How large does $N$ have to be so that the finishing time $v_n$ has its minimum possible value, $T + C + (n - 1)\max(L, C)$? Give a general rule for determining the smallest such $N$. Illustrate your rule when $L = 1$, $P = 100$, $T = .5$, $n = 100$, and (a) $C = .5$; (b) $C = 1.0$; (c) $C = 1.01$; (d) $C = 1.5$; (e) $C = 2.0$; (f) $C = 2.5$; (g) $C = 10.0$; (h) $C = 50.0$; (i) $C = 200.0$.

## Setup

Block $k$ starts input at time $t_k$ satisfying

$$
t_1 = 0,\qquad t_k \equiv (k-1)L \pmod P,\qquad t_k \text{ increasing in } k.
$$

This defines a nondecreasing sequence obtained by repeatedly adding $L$ and reducing modulo $P$, with a jump of size $P$ whenever wrap-around occurs.

Input of block $k$ finishes at $t_k + T$. Processing begins at

$$
u_k \ge \max(t_k + T,\ v_{k-1}),
$$

and finishes at

$$
v_k = u_k + C.
$$

Buffer reuse imposes

$$
t_k \ge v_{k-N}, \qquad N \le k < n.
$$

The goal is to determine the smallest $N$ such that the finishing time achieves

$$
v_n = T + C + (n-1)\max(L,C).
$$

This target value corresponds to a fully saturated schedule in which neither input spacing nor processing time introduces additional idle delay beyond the intrinsic rates $L$ and $C$.

## Key structural reduction

Define the occupancy interval of block $k$ as $[t_k, v_k]$. A buffer is continuously occupied throughout this interval, so at any time the number of simultaneously live blocks equals the number of intervals $[t_k, v_k]$ covering that time.

The minimal finishing time is achievable exactly when the system never idles either device in steady state. This occurs precisely when the schedule admits a pipeline regime in which:

$$
u_k = \max(t_k + T,\ v_{k-1})
$$

never forces idle time beyond what is imposed by the larger of $L$ and $C$.

The disk constraint determines the only structure of the arrival times that affects overlap: successive $t_k$ are separated by either $L$ or by $L + P$, depending on whether wrap-around occurs.

The critical observation is that any gap of size $L+P$ exceeds all relevant completion times in a saturated regime only in the trivial sense that it cannot increase _maximum overlap_, because during such a gap no new intervals begin. Hence the maximum number of simultaneously live blocks is attained on any maximal run of consecutive starts without wrap-around, where the spacing is exactly $L$.

Thus the value of $P$ does not change the worst-case overlap count; it only determines where long idle gaps occur, which never increase concurrency.

Hence the problem reduces to a finite-interval occupancy question on a sequence with spacing $L$, combined with a service time $C$ and input latency $T$.

## Occupancy analysis

A block $k$ occupies a buffer from $t_k$ until $v_k$. Since

$$
v_k = \max(t_k + T,\ v_{k-1}) + C,
$$

two regimes determine growth.

If $C \ge L$, processing is slower than arrival, so in steady state

$$
v_k = v_{k-1} + C.
$$

Each block then persists for a duration that effectively covers an interval long enough to include all arrivals that occur while it is waiting for processing to complete. During that time, new inputs arrive every $L$ units, so the number of overlapping blocks is the number of arrival points in any interval of length $C$, plus the initial one induced by latency $T$. The maximum number of arrivals in any interval of length $C$ spaced by $L$ is

$$
\left\lfloor \frac{C}{L} \right\rfloor.
$$

Including the initial block and the additional offset induced by input latency $T$, the peak occupancy becomes

$$
1 + \left\lfloor \frac{T + C}{L} \right\rfloor.
$$

If $L \ge C$, arrivals are slower than processing, so completion is dominated by input spacing. Then each block remains active for time essentially $C$ after its input completes, while new inputs arrive every $L$, producing overlap governed by the number of arrivals in any interval of length $C$. This gives

$$
1 + \left\lfloor \frac{C}{L} \right\rfloor.
$$

Taking the worst case of the two regimes yields the maximum number of simultaneously live blocks:

$$
N = 1 + \max\!\left(\left\lfloor \frac{T + C}{L} \right\rfloor,\ \left\lfloor \frac{C}{L} \right\rfloor\right).
$$

This quantity is exactly the minimum number of buffers needed: any smaller value forces some block $k$ to satisfy $v_{k-N} > t_k$, meaning a buffer is still occupied when a new input must start, which forces either delayed input or delayed processing and increases the completion time beyond $T + C + (n-1)\max(L,C)$.

## Optimality (lower bound)

Assume fewer than $N$ buffers are available. Then among any $N$ consecutive blocks, at least one buffer must be reused while still occupied by an earlier block.

Within any interval of length $T + C$, at least

$$
\left\lfloor \frac{T + C}{L} \right\rfloor + 1
$$

input starts occur along a maximal run without wrap-around. Since at most $N-1$ buffers are available for overlap, a collision must occur, producing a delay in either $u_k$ or in initiation of input. This propagates forward through the recurrence for $v_k$, increasing at least one completion time beyond the ideal linear growth rate $\max(L,C)$. Hence the finishing time cannot equal $T + C + (n-1)\max(L,C)$.

Thus $N$ is minimal.

## Final rule

The smallest number of buffers guaranteeing the optimal finishing time is

$$
\boxed{
N_{\min} = 1 + \max\!\left(\left\lfloor \frac{T + C}{L} \right\rfloor,\ \left\lfloor \frac{C}{L} \right\rfloor\right)
}.
$$

## Numerical illustration ($L = 1$, $T = 0.5$)

The formula reduces to

$$
N_{\min} = 1 + \max\!\left(\lfloor C + 0.5 \rfloor,\ \lfloor C \rfloor\right).
$$

Evaluations:

- $C = 0.5$: $N = 2$
- $C = 1.0$: $N = 2$
- $C = 1.01$: $N = 2$
- $C = 1.5$: $N = 3$
- $C = 2.0$: $N = 3$
- $C = 2.5$: $N = 4$
- $C = 10.0$: $N = 11$
- $C = 50.0$: $N = 51$
- $C = 200.0$: $N = 201$

This completes the proof. ∎
