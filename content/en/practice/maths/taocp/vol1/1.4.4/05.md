**Exercise 5.** [*M21*] Generalize the situation of the preceding exercise to the case when the program refers to $n$ I/O devices instead of just one.

## Solution

Let $n$ be the number of I/O devices referred to by the program, and let $T_i$ denote the time required to perform a complete unbuffered I/O operation on device $i$, for $1 \le i \le n$. Let $C$ denote the computation time required between successive accesses to the same device in the unbuffered scheme. In the single-device case discussed in the preceding exercise, the total running time per tape block was $T + C$ for unbuffered I/O and $\max(T, C)$ for buffered I/O using two buffers, assuming that computation and I/O could overlap completely.

We generalize this argument to $n$ devices. Suppose the program accesses each device in a cyclic manner, so that after performing an I/O operation on device $i$, it performs I/O on device $i+1$, and after device $n$ it returns to device $1$. Let $C_i$ denote the computation time performed between successive accesses to device $i$ in this cycle.

In the unbuffered case, each access to device $i$ is followed by a busy wait of duration $C_i$, during which the device remains idle. Therefore, the running time for a single cycle of $n$ devices is

$$
R_\text{unbuffered} = \sum_{i=1}^{n} (T_i + C_i).
$$

Introduce a buffer of size sufficient to hold one full block of I/O for each device, and initiate the next I/O on device $i$ before computation with the data of device $i$ is complete. Let $B_i$ denote the buffer for device $i$. Using anticipated input/output, the program can perform the computation corresponding to device $i$ while the I/O on device $i$ proceeds in parallel. The running time for device $i$ then becomes the maximum of computation and I/O:

$$
R_i^\text{buffered} = \max(T_i, C_i).
$$

Since computation can overlap with all devices simultaneously in principle, the total running time per cycle becomes

$$
R_\text{buffered} = \sum_{i=1}^{n} \max(T_i, C_i) + \text{overhead}.
$$

The overhead consists of the time required to switch buffers and possibly to perform the initial I/O to fill each buffer, analogous to the $202u$ of equation (3) for two buffers. Denote this overhead by $O$, which is negligible relative to large $T_i$ or $C_i$ in practical applications.

To analyze the maximal possible speedup, consider the case in which $T_i = C_i$ for all $i$. Then

$$
R_\text{unbuffered} = \sum_{i=1}^{n} (T_i + C_i) = 2 \sum_{i=1}^{n} T_i,
$$

while

$$
R_\text{buffered} = \sum_{i=1}^{n} \max(T_i, C_i) = \sum_{i=1}^{n} T_i.
$$

Thus the running time is cut by a factor of exactly two.

No better factor is possible. Suppose that $\max(T_i, C_i) < \frac{1}{2}(T_i + C_i)$ for some $i$. Then both $T_i$ and $C_i$ would be strictly less than $\frac{1}{2}(T_i + C_i)$, which is impossible. Therefore, the speedup due to buffering cannot exceed a factor of two.

The argument applies independently to each device. With $n$ devices, the running time of the program using unbuffered I/O is

$$
\sum_{i=1}^{n} (T_i + C_i),
$$

while using double-buffering (or multiple-buffering) it is

$$
\sum_{i=1}^{n} \max(T_i, C_i) + O.
$$

Hence, in favorable circumstances, the running time can be reduced by almost a factor of two, and it cannot be reduced by more than a factor of two. The generalization requires one buffer per device to overlap computation and I/O, or, equivalently, a total of $2n$ memory areas for buffer swapping, one active and one anticipatory for each device.

This completes the proof.
