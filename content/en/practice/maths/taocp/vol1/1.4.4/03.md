**Exercise 3.** [*22*] Write a buffer-swapping output subroutine analogous to (4). The subroutine, called `WORDOUT`, should store the word in rA as the next word of output, and if a buffer is full it should write 100 words onto tape unit V. Index register 5 should be used to refer to the current buffer position. Show the layout of buffer areas and explain what instructions (if any) are necessary at the beginning and end of the program to ensure that the first and last blocks are properly written. The final block should be filled out with zeros if necessary.

## Solution

The output analogue of (4) should keep one buffer available for writing by the program while the other buffer is being written onto tape. Since output proceeds from the program to memory and then from memory to tape, the natural end-of-buffer test is based on a position count rather than on a sentinel.

Let index register 5 contain the address of the most recently filled word. Each buffer consists of $100$ data words followed by one linking word containing the address of the other buffer. Thus the layout is

```
OUTBUF1 ORIG *+100               First buffer
        CON  *+1                 Address of other buffer
OUTBUF2 ORIG *+100               Second buffer
        CON  OUTBUF1             Address of other buffer
```

The subroutine is:

```
01 WORDOUT STJ  1F               Store the exit location.
02         STA  1,5              Store next output word.
03         INC5 1                Advance buffer position.
04         CMP5  =OUTBUF1+100=
05         JL   1F               Not yet at end of buffer.
06         CMP5  =OUTBUF2+100=
07         JL   2F               End of first buffer.
08 3H      OUT  -100,5(V)        Write second buffer.
09         LD5  1,5              Switch buffers.
10 1H      JNE  *                Return.
11 2H      OUT  -100,5(V)        Write first buffer.
12         LD5  1,5              Switch buffers.
13         JMP  1B
```

The operation is as follows. Suppose $rI_5$ addresses the last occupied location of the current buffer. Line 02 stores the word from $rA$ into the next position. Line 03 advances $rI_5$. If fewer than $100$ words have been stored in the current buffer, control returns immediately. When the hundredth word has been stored, $rI_5$ becomes equal to the linking word of that buffer. The instruction

$$
\texttt{OUT -100,5(V)}
$$

addresses the beginning of the just-filled buffer, because $rI_5-100$ is its first word. Hence exactly the $100$ data words are written onto tape. The linking word itself is not part of the output block. After the output operation has been initiated, line 09 or 12 loads into $rI_5$ the address of the other buffer, thereby preparing for subsequent output.

No `JBUS` instruction is required inside the subroutine. The next $100$ output words are placed into the alternate buffer while the previous block is being written, exactly as in the input routine (4).

To start the process correctly, the program must initialize $rI_5$ to the address immediately preceding the first word of the first buffer. Thus an initialization such as

```
ENT5 OUTBUF1-1
```

is required before the first call on `WORDOUT`.

At termination, suppose that $m$ words have been placed into the current buffer, where $0 \le m < 100$. These $m$ words have not yet been written, because output occurs only when a buffer becomes full. The remaining $100-m$ locations of the current buffer must therefore be filled with zeros. After this padding has been performed, one final output operation writes the completed block.

If $rI_5$ points to the next unused position of the current buffer, the program repeatedly stores zero into successive locations until the linking word is reached. Then it executes

```
OUT -100,5(V)
JBUS *(V)
```

The `OUT` instruction writes the final $100$-word block, and the `JBUS` instruction guarantees that the write operation has actually finished before the program terminates.

Consequently, every output word produced by the program appears exactly once on tape, the tape is written in blocks of $100$ words, buffer swapping overlaps computation with output, and the final partial block is completed with zeros as required.

This completes the proof.
