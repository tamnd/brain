**Exercise 15.** [*22*] Write a complete MIX program that copies 100 blocks from tape unit 0 to tape unit 1, using just three buffers. The program should be as fast as possible.

**Verified:** no

## Solution

Let the three buffers be `BUF1`, `BUF2`, and `BUF3`, each consisting of 100 words. Since the task is merely to copy tape blocks, no computation is performed on the data. Therefore the fastest possible program is one that keeps both tape units continuously occupied after the initial startup.

The input unit can be reading block $k+1$ while the output unit is writing block $k$, provided that a third buffer is available. Thus three buffers are sufficient to form a pipeline:

$$
\text{read} \rightarrow \text{buffer} \rightarrow \text{write}.
$$

At every stage after startup, one buffer is being filled from unit $0$, one buffer contains a completed block waiting to be written or currently being written to unit $1$, and one buffer is free for the next transfer.

A complete MIXAL program is:

```
        ORIG 3000

START   IN   BUF1(0)          First input.
        IN   BUF2(0)          Second input.

        ENT1 98              Number of remaining blocks
                             after the first two.

        JBUS *(0)            Wait for BUF1 complete.
        OUT  BUF1(1)         Start first output.

        JBUS *(0)            Wait for BUF2 complete.
        IN   BUF3(0)         Read third block.

LOOP    JBUS *(1)            Wait until previous output ends.
        OUT  BUF2(1)         Output second buffer.

        JBUS *(0)            Wait until BUF3 is full.
        IN   BUF1(0)         Reuse BUF1.

        JBUS *(1)
        OUT  BUF3(1)

        JBUS *(0)
        IN   BUF2(0)

        JBUS *(1)
        OUT  BUF1(1)

        JBUS *(0)
        IN   BUF3(0)

        DEC1 3
        J1P  LOOP

        JBUS *(1)
        OUT  BUF2(1)

        JBUS *(0)
        OUT  BUF3(1)

        JBUS *(1)
        JBUS *(0)

        HLT

BUF1    ORIG *+100
BUF2    ORIG *+100
BUF3    ORIG *+100

        END  START
```

To justify optimality, let $T_i$ denote the time required to read one 100-word tape block and let $T_o$ denote the time required to write one 100-word tape block.

A copy operation for one block cannot be completed in less than

$$
\max(T_i,T_o),
$$

because every block must be read once and written once. Hence any program that copies $100$ blocks requires at least

$$
100\max(T_i,T_o)
$$

time, apart from unavoidable startup and shutdown delays.

The program above overlaps every read with a write after the pipeline is filled. No buffer-copying operations are performed, and no tape unit is left idle except during the initial filling of the pipeline and the final draining of it. Consequently the steady-state rate is exactly one block every

$$
\max(T_i,T_o)
$$

time units, which attains the lower bound.

Therefore the program is as fast as possible using three buffers.

$$
\boxed{\text{Three-buffer pipelined copy with continuous overlap of input and output}}
$$
