**Exercise 12.** [*24*] Suppose that the multiple buffering algorithm in the text is being used for card input, and suppose the input is to terminate as soon as a card with "." in column 80 has been read. Show how the CONTROL coroutine (Algorithm B and Program B) should be changed so that input is shut off in this way.

**Verified:** no  
**Solve time:** 4m50s  

---

## Solution

In the multiple buffering scheme, anticipated input is used: whenever a buffer is released for processing, the CONTROL coroutine immediately initiates reading of another card into a free buffer. Therefore a special provision is needed when a card containing `.` in column 80 is encountered, since no further input is to be requested after that card has been read.

The text preceding exercise 12 points out that anticipated input becomes possible if an identifying mark occurs in a fixed position of the final record. Here the identifying mark is the character `.` in column 80. Since the entire card has already been read into a buffer before the card is processed, CONTROL can examine column 80 when the card is assigned to the consumer coroutine.

Let the buffer structure be unchanged. Add a Boolean variable, say `FINAL`, initially `false`. Whenever CONTROL obtains a card from an input buffer, it inspects column 80 of that card.

If column 80 is not `.`, CONTROL behaves exactly as before. The card is passed to the consumer coroutine, and when the corresponding buffer becomes free, that buffer is returned to the circle of available buffers and another `IN` operation is initiated.

If column 80 is `.`, CONTROL sets

$$
\textit{FINAL} \leftarrow \textit{true},
$$

and passes this card to the consumer coroutine in the normal way. After this moment no further input operations are initiated. The remaining buffers that are already in the system continue to be processed normally. Since the card containing `.` is the final card of the deck, every card that must be processed has already been read. When the buffer containing that card is finally released, the input side of the buffering system is exhausted and CONTROL terminates in the usual manner.

In Algorithm B, the change is therefore confined to the step at which a newly filled card buffer is assigned for processing. Immediately before the assignment, test column 80. If it contains `.`, set the flag `FINAL`. Thereafter, whenever CONTROL would ordinarily execute an `IN` operation on a newly available buffer, it first tests `FINAL`; if `FINAL = \textit{true}`, the `IN` operation is suppressed.

The corresponding modification of Program B is analogous. After a card has been read and before another `IN` instruction is started for the released buffer, the program examines the character stored for column 80. If that character is `.`, it sets a flag. Every subsequent point at which Program B would initiate another card read is preceded by a test of this flag; when the flag is set, control bypasses the `IN` instruction.

This modification guarantees that the card containing `.` in column 80 is delivered to the consumer coroutine, that no card following it is requested from the input device, and that all previously initiated input operations are completed correctly. Hence input is shut off exactly when the final card has been read.

$$
\boxed{\text{Set a final-card flag when column 80 contains '.'; thereafter suppress all further }IN\text{ operations.}}
$$

This completes the proof.
