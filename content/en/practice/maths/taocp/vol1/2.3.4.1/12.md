**Exercise 12.** [**] $$29$$ The algorithm of exercise 11 is not stated in a fashion suitable for direct computer implementation. Reformulate that algorithm, specifying in more detail the operations that are to be done, in such a way that a computer program can carry out the process with reasonable efficiency.

**Verified:** yes  
**Solve time:** 3m17s  

---

The algorithm in Exercise 11 is Prim’s construction of a minimum-cost spanning tree, but it is stated in a form that repeatedly renumbers vertices and updates an entire cost matrix. A direct implementation avoids renumbering by keeping a fixed labeling of vertices and maintaining, for each vertex not yet in the tree, the cheapest known connection to the current tree.

Start with an arbitrary root vertex $r$. Maintain a set $S$ of vertices already chosen, initially $S={r}$. For each vertex $v\notin S$, maintain a value $d[v]=\min_{u\in S} c(u,v)$ together with a pointer $p[v]\in S$ achieving this minimum. At each step, select a vertex $v\notin S$ with minimum $d[v]$, add it to $S$, and add the edge $(p[v],v)$ to the tree. Then update all remaining vertices $w\notin S$ by setting $d[w]\leftarrow \min(d[w],c(v,w))$ and updating $p[w]$ accordingly when a smaller value is found. This process continues until all vertices are included.

This reformulation is equivalent to the original algorithm because at every stage the vertex chosen next is exactly the one whose cheapest connection to the current chosen set is minimal, and the update step preserves the invariant that $d[v]$ always represents the best available connection from $v$ into $S$. The resulting edges form a spanning tree since each added vertex is connected to an already connected set, and optimality follows from the cut property: at each step the selected edge is the minimum-weight edge crossing the cut $(S, V\setminus S)$, so no lighter spanning tree can replace it without violating connectivity.
