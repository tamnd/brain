**Exercise 15.** [**] [25] (David S. Wise.) Reformulate Algorithm A so that no extra storage
is used for the stack. [*Hint:* The `SIB` fields of all nodes pointed to by the
stack are `\Lambda` in the present formulation.]

**Verified:** no  
**Solve time:** 5m35s  

---

In Algorithm A, every node whose address is placed on the stack has `SIB=\Lambda`. Therefore the stack can be threaded through those unused `SIB` fields. Replace the external stack by a single pointer `TOP`. When a node address `P` would ordinarily be pushed onto the stack, perform

$SIB(P)\leftarrow TOP,$

$TOP\leftarrow P.$

When a node address is to be popped, perform

$P\leftarrow TOP,$

$TOP\leftarrow SIB(P).$

The invariant is that the stack contents are represented by the chain of `SIB` links beginning at `TOP`. Since every stacked node originally has `SIB=\Lambda`, storing the previous stack top in `SIB(P)` destroys no information needed by the traversal. When the node is later removed from the stack, `SIB(P)` is used only to recover the next stack element. Thus Algorithm A requires no extra stack storage; the stack is embedded in the unused `SIB` fields of the nodes themselves.
