[Section 2.4: Multilinked Structures](../)

**Exercise 5.** [**] [20] In practice, Algorithm B may be given a linked list of Symbol Table
references as input, instead of what we called "`P0, P1, ..., Pn`." Let `T` be
a pointer variable such that

$$
\operatorname{INFO}(T) = P_0,\quad
\operatorname{INFO}(\operatorname{RLINK}(T)) = P_1,\quad \ldots,\quad
\operatorname{INFO}(\operatorname{RLINK}^{[n]}(T)) = P_n,\quad
\operatorname{RLINK}^{[n+1]}(T) = \Lambda.
$$

Show how to modify Algorithm B so that it uses such a linked list as input.

**Verified:** no  
**Solve time:** 10m19s  

---

Algorithm B can be modified to accept a linked list of Symbol Table references by replacing the indexed access of $P_0, P_1, \ldots, P_n$ with a traversal of the list using the $\operatorname{RLINK}$ field. Let $T$ be a pointer to the head of the list, so that $\operatorname{INFO}(T) = P_0$. Define a local pointer variable $U \leftarrow T$. In every step of Algorithm B that originally referenced $P_i$, replace $P_i$ with $\operatorname{INFO}(U)$. After processing $\operatorname{INFO}(U)$, advance $U \leftarrow \operatorname{RLINK}(U)$. Continue this process until $U = \Lambda$, which corresponds to having processed all $P_i$ in the original formulation.  

This modification requires exactly $n+1$ replacements for initializing and advancing $U$, and all original references to $P_i$ are replaced systematically by $\operatorname{INFO}(U)$. The logical flow and correctness of Algorithm B are preserved, since the linked list provides the same ordered sequence of Symbol Table references as the array $P_0, P_1, \ldots, P_n$. ∎
