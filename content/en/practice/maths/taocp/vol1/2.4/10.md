If there were no `NAME` link in the Data Table nodes, the test `NAME(S) = P_k` in step B6 could be performed by traversing the `DATAP` chain from `S` to find the data field corresponding to the name and comparing it to `P_k`. Specifically, one would follow the `DATAP` link from `S` to each entry in the data table until the entry with the matching name is found, then compare its value with `P_k`. This replaces a single pointer dereference with a linear search through the data table, but the logical comparison remains valid.

That is, for entries `P` and `Q`, follow the `DATAP` links to access the actual name fields stored within the nodes and compare these values directly. This ensures equality is correctly determined without an explicit `NAME` link, though each comparison requires accessing the data field explicitly.
