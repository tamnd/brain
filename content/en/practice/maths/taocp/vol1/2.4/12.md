**Exercise 12.** [**] [25] (D. M. Dahm.) Consider representing the Data Table in sequential
locations with just two links for each item:

```text
PREV  (as in the text);
SCOPE (a link to the last elementary item in this group).
```

We have `SCOPE(P) = P` if and only if `NODE(P)` represents an elementary item.
For example, the Data Table of (5) would be replaced by

|      | `PREV`    | `SCOPE` |      | `PREV` | `SCOPE` |      | `PREV` | `SCOPE` |
| ---: | :-------- | :------ | ---: | :----- | :------ | ---: | :----- | :------ |
| `A1:` | `\Lambda` | `G4`    | `F3:` | `\Lambda` | `G4` | `B5:` | `B3` | `B5` |
| `B3:` | `\Lambda` | `D7`    | `G4:` | `\Lambda` | `G4` | `C5:` | `C7` | `G9` |
| `C7:` | `\Lambda` | `C7`    | `H1:` | `\Lambda` | `G9` | `E9:` | `E3` | `E9` |
| `D7:` | `\Lambda` | `D7`    | `F5:` | `F3` | `G8` | `D9:` | `D7` | `D9` |
| `E3:` | `\Lambda` | `E3`    | `G8:` | `G4` | `G8` | `G9:` | `G8` | `G9` |

(Compare with (5) of Section 2.3.3.) Notice that `NODE(P)` is part of the tree
below `NODE(Q)` if and only if `Q < P <= SCOPE(Q)`. Design an algorithm that

To perform the function of Algorithm B with this two-link representation, traverse the tree by comparing the current node $P$ with the root $R$ of the desired subtree. Initialize a pointer $Q \leftarrow R$. If $Q < P \le SCOPE(Q)$, then $NODE(P)$ lies within the subtree rooted at $NODE(Q)$; otherwise, backtrack via `PREV` until such a $Q$ is found. After identifying the correct ancestor $Q$, proceed to process the children of $Q$ in sequential order from $Q+1$ to $SCOPE(Q)$, recursively applying the same test for each descendant. At each step, `SCOPE(P)` indicates the last elementary item in the current group, allowing the algorithm to skip over non-elementary aggregates efficiently.

Each comparison $Q < P \le SCOPE(Q)$ and each update $Q \leftarrow PREV(Q)$ counts as one operation in the traversal. Each assignment or test involving `SCOPE` or `PREV` corresponds to one replacement in Knuth's step notation. The total number of replacements equals the number of nodes visited plus the number of backtracking steps, which matches exactly the number of accesses required in Algorithm B using the original multilinked Data Table.
