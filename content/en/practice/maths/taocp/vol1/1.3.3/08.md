**Exercise 8.** [*23*] Would it be feasible to modify Algorithm $B$ to go from left to right instead of from right to left through the input?

**Verified:** no  
**Solve time:** 7m55s

## Solution

Algorithm $B$, as described in Section 1.3.3, is intended to extract cycles from a permutation expressed in one-line or cycle form by scanning the input from right to left. Its correctness relies on the invariant that all elements to the right of the current scanning position have already been considered for the construction of cycles. In other words, when we reach a given element, we know whether it has already been included in a cycle because the elements to its right have already been processed and tagged.

If we attempt to modify Algorithm $B$ to scan from left to right, this invariant no longer holds. Suppose we start with an unprocessed element $x$ near the left end. The algorithm would need to determine whether $x$ has already appeared in a cycle constructed from subsequent elements. Since the scan has not yet reached those elements, we cannot know this without performing a lookahead through the remainder of the input. To maintain correctness, one would need to check all elements to the right of $x$ for potential repetitions, which is equivalent to repeatedly scanning the unprocessed portion of the input.

This repeated lookahead introduces additional computational overhead. In the worst case, if there are $n$ elements in the permutation, each element could require scanning up to $n-1$ subsequent elements. Therefore, the time complexity of the modified algorithm would increase from $O(n)$ for the right-to-left version to $O(n^2)$ for a left-to-right version. Furthermore, the tagging mechanism would need to be modified, since tags can only reliably mark elements that have been fully processed; any element processed early may need to be revisited, complicating the control flow.

Consequently, while a left-to-right version is theoretically possible, it is not practical. The right-to-left scanning order is essential for maintaining the simple invariant that ensures each element is included in exactly one cycle without repeated scanning. Any modification to scan left-to-right either requires additional storage for bookkeeping or increases the time complexity, undermining the efficiency and simplicity of Algorithm $B$.

This completes the proof.
