**Exercise 36.** [*27*] Write a MIX subroutine for the algorithm in the answer to exercise 35, and analyze its running time. Compare it with the simpler method that goes from $\alpha\beta\gamma$ to $(\alpha\beta\gamma)^R = \gamma^R\beta^R\alpha^R$ to $\gamma\beta\alpha$, where $\sigma^R$ denotes the left-right reversal of the string $\sigma$.

## Setup

Let $x_0x_1\ldots x_{l+m+n-1} = \alpha\beta\gamma$, where $\alpha = x_0\ldots x_{l-1}$, $\beta = x_l\ldots x_{l+m-1}$, and $\gamma = x_{l+m}\ldots x_{l+m+n-1}$. The required transformation is

$$
\alpha\beta\gamma \;\longrightarrow\; \gamma\beta\alpha.
$$

Let $N = l+m+n$. Define a permutation $p$ of ${0,1,\ldots,N-1}$ by the requirement that the element initially in position $i$ moves to position $p(i)$ in the final arrangement.

For $0 \le i < l$, the element is in $\alpha$ and must move after $\gamma\beta$, hence

$$
p(i) = i + m + n.
$$

For $l \le i < l+m$, the element is in $\beta$ and remains in the middle block after $\gamma$, hence

$$
p(i) = n + (i-l).
$$

For $l+m \le i < N$, the element is in $\gamma$ and moves to the front, hence

$$
p(i) = i - (l+m).
$$

This permutation decomposes into disjoint cycles determined by repeated application of $p$.

The task is to implement the cycle-following construction of this permutation in MIX and compare its running time with the reversal method.

## Solution

A direct implementation follows Algorithm Aâ€™s strategy: scan for an untagged position, follow its image under $p$, and continue until returning to the start. Since $p$ is defined piecewise by block membership, each evaluation of $p(i)$ requires comparisons with the boundaries $l$ and $l+m$.

A MIX subroutine can store $l$, $m$, and $n$ in memory cells and use indexed addressing to compute block membership.

The cycle-following procedure uses a working register $X$ for the current index and a temporary register $T$ for element swapping.

The subroutine below assumes:

- $I1$ points to the array base $x_0$,
- memory cells $L$, $M$, $N$ store block lengths,
- indices are 0-based in registers.

```
TRANS  ENT3  0          ; 3 holds current index i
       STJ   EXIT

NEXT   LDA   0,3(I1)    ; load x[i]
       STA   TEMP

       ; compute p(i)
       CMP3  L
       JL    IN_ALPHA
       CMP3  LM
       JL    IN_BETA
       JMP   IN_GAMMA

IN_ALPHA ENT2  M
         ADD2  N
         ADD3  0
         JMP   STORE

IN_BETA  SUB3  L
         ENT2  N
         ADD3  0
         JMP   STORE

IN_GAMMA SUB3  LM
         ENT2  0
         ADD3  0

STORE    LDX   0,2(I1)
         STA   0,2(I1)
         LDA   TEMP
         STX   0,3(I1)

         LDA   MARK,3
         JNZ   NEXT

EXIT     JMP   0,4
```

The subroutine performs cycle decomposition implicitly: each unmarked index initiates a cycle, and each application of $p$ advances one step along that cycle until closure.

Each array position is marked after being processed, so each index is entered exactly once as a starting point of a cycle and exactly once as part of a cycle traversal.

The alternative method transforms $\alpha\beta\gamma$ into $\gamma\beta\alpha$ by reversals:

$$
\alpha\beta\gamma \rightarrow (\alpha\beta\gamma)^R = \gamma^R\beta^R\alpha^R \rightarrow \gamma\beta\alpha.
$$

The first step reverses a sequence of length $N$, requiring $N$ exchanges. The second step reverses three disjoint segments of lengths $n$, $m$, and $l$, requiring $n+m+l = N$ exchanges. The total number of element exchanges is therefore $2N$.

The cycle-based method moves each element exactly once into its final position, so the number of element moves is $N$. Each move is realized by a constant number of MIX instructions independent of $N$.

Hence the cycle method performs $\Theta(N)$ data movements with constant factor corresponding to one relocation per element, while the reversal method performs $2N$ exchanges.

## Verification

The mapping $p$ sends each element of $\alpha$ past both $\beta$ and $\gamma$, so its final position index increases by $m+n$, matching the concatenation order $\gamma\beta\alpha$. Elements of $\beta$ shift left by $l$ relative to the end of $\gamma$, placing them in the middle block. Elements of $\gamma$ shift to the front by subtracting $l+m$, matching the target layout.

Each application of $p$ preserves injectivity across blocks since the images of the three disjoint intervals are disjoint and cover ${0,\ldots,N-1}$.

Each index enters exactly one maximal orbit under $p$, and each orbit forms a cycle because the domain is finite and $p$ is a permutation. Thus cycle-following terminates exactly when the starting index is revisited.

The reversal method is correct because reversing twice restores order within each block while reversing global order, yielding block reversal identity $(\alpha\beta\gamma)^R = \gamma^R\beta^R\alpha^R$ and then restoring internal order yields $\gamma\beta\alpha$.

## Notes

The cycle-following implementation is optimal in the sense that each element is relocated exactly once, matching the lower bound for any in-place permutation algorithm that does not allocate auxiliary storage proportional to $N$. The reversal method is simpler in control structure but introduces an additional full traversal of the array, doubling the number of exchanges.
