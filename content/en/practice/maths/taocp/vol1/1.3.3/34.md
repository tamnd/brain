**Exercise 34.** [*M25*] (*Transposing blocks of data.*) One of the most common permutations needed in practice is the change from $\alpha\beta$ to $\beta\alpha$, where $\alpha$ and $\beta$ are substrings of an array.

In other words, if $x_0x_1\ldots x_{m-1} = \alpha$ and $x_mx_{m+1}\ldots x_{m+n-1} = \beta$, we want to change the array $x_0x_1\ldots x_{m+n-1} = \alpha\beta$ to the array $x_mx_{m+1}\ldots x_{m+n-1}x_0x_1\ldots x_{m-1} = \beta\alpha$; each element $x_k$ should be replaced by $x_{p(k)}$ for $0 \le k < m + n$, where $p(k) = (k + m) \bmod (m + n)$. Show that every such "cyclic-shift" permutation has a simple cycle structure, and exploit that structure to devise a simple algorithm for the desired rearrangement.

## Solution

Let $N = m + n$. The permutation is $p(k) = (k + m) \bmod N$ on the set ${0,1,\ldots,N-1}$.

Iterating $p$ gives

$$
p^t(k) \equiv k + tm \pmod N.
$$

The cycle containing $k$ is the set of all $p^t(k)$ until the first return to $k$, which occurs when

$$
k + tm \equiv k \pmod N,
$$

so

$$
tm \equiv 0 \pmod N.
$$

Let $d = \gcd(m,N)$. Write $m = dm_1$ and $N = dN_1$ with $\gcd(m_1,N_1)=1$. The congruence $tm \equiv 0 \pmod N$ becomes

$$
dN_1 \mid tdm_1,
$$

equivalently

$$
N_1 \mid tm_1.
$$

Since $\gcd(m_1,N_1)=1$, this holds exactly when $N_1 \mid t$. The least positive such $t$ is $t = N_1 = \frac{N}{d}$. Hence every cycle has length $\frac{N}{d}$.

The cycle starting at $k$ is therefore

$$
(k,\; k+m,\; k+2m,\; \ldots,\; k+(N/d-1)m),
$$

with all arithmetic taken modulo $N$. Two such cycles are either identical or disjoint, since equality of any elements implies equality of starting points modulo the subgroup generated by $m$ in $\mathbb{Z}_N$. The number of distinct cycles is $d$, since each cycle has length $N/d$ and the cycles partition a set of size $N$.

This shows that the permutation decomposes into $d = \gcd(m,m+n) = \gcd(m,n)$ disjoint cycles, each formed by repeated addition of $m$ modulo $N$.

To perform the rearrangement in-place, each cycle is processed independently. For each $k$ not yet placed into its final position, the value at position $k$ is saved as a temporary variable $t$, and the cycle is followed by repeated application of the rule $k \leftarrow p(k)$, writing each displaced value into its predecessor’s position until returning to the starting index.

Concretely, for a fixed starting index $k$, set $i \leftarrow k$ and $t \leftarrow x_i$. Then repeatedly set $j \leftarrow (i + m) \bmod N$, exchange $x_j$ with $t$, and continue with $i \leftarrow j$. The process terminates when $i$ returns to $k$, at which point the initial value has been placed in its final position and the cycle is complete. Repeating this procedure for each unvisited index produces the full transformation $x_k \leftarrow x_{p(k)}$ for all $k$.

Each array position is visited exactly once within its cycle because the cycles partition ${0,1,\ldots,N-1}$, and each step advances strictly along the orbit of $p$. This completes the construction of an algorithm that exploits the cycle structure of the cyclic-shift permutation. ∎
