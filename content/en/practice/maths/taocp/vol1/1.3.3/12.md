**Exercise 12.** [*M27*] (*Transposing a rectangular matrix*.) Suppose an $m \times n$ matrix $(a_{ij})$, $m \ne n$, is stored in memory in a fashion like that of exercise 1.3.2-10, so that the value of $a_{ij}$ appears in location $L + n(i - 1) + (j - 1)$, where $L$ is the location of $a_{11}$. The problem is to find a way to *transpose* this matrix, obtaining an $n \times m$ matrix $(b_{ij})$, where $b_{ij} = a_{ji}$ is stored in location $L + m(i - 1) + (j - 1)$. Thus the matrix is to be transposed "on itself." (a) Show that the transposition transformation moves the value that appears in cell $L + x$ to cell $L + (mx \bmod N)$, for all $x$ in the range $0 \le x < N = mn - 1$. (b) Discuss methods for doing this transposition by computer.

The error occurs precisely at the point where a congruence modulo $mn-1$ is turned into an equality without first identifying the correct residue system and without using the correct domain restriction. A correct argument must treat the mapping as a permutation on $0,\dots,mn-2$ with a separate fixed point at $mn-1$.

## (a) Correct proof of the transposition mapping

Let the entries of the $m\times n$ matrix be stored in row-major order, so that $a_{ij}$ is in location

$$
L + n(i-1) + (j-1).
$$

Write

$$
x = n(i-1) + (j-1),
\quad 0 \le x \le mn-1.
$$

Set $q=i-1$ and $r=j-1$, so $0 \le q < m$, $0 \le r < n$, and

$$
x = nq + r.
$$

After transposition, $a_{ij}$ becomes $b_{ji}$, which is stored at

$$
y = m(j-1) + (i-1) = mr + q.
$$

We analyze the mapping $x \mapsto y$.

### Step 1: Key congruence

Compute

$$
mx = m(nq+r) = mnq + mr.
$$

Modulo $mn-1$, we use

$$
mn \equiv 1 \pmod{mn-1},
$$

so

$$
mnq \equiv q \pmod{mn-1}.
$$

Hence

$$
mx \equiv q + mr = y \pmod{mn-1}.
$$

So we have shown

$$
y \equiv mx \pmod{mn-1}.
$$

### Step 2: Correct domain split

The index set $\{0,1,\dots,mn-1\}$ is not a full residue system modulo $mn-1$, since that modulus has residues $0,1,\dots,mn-2$. Therefore we separate the last index.

#### Case 1: $x = mn-1$

This corresponds to $q=m-1$, $r=n-1$. Then

$$
y = m(n-1) + (m-1) = mn-1.
$$

So $mn-1$ is a fixed point.

#### Case 2: $0 \le x \le mn-2$

This excludes $(q,r)=(m-1,n-1)$, so either $q \le m-2$ or $r \le n-2$. Then

$$
y = mr + q \le m(n-1) + (m-2) = mn-2.
$$

Also clearly $y \ge 0$. Hence

$$
0 \le y \le mn-2.
$$

Thus for $x < mn-1$, both $x$ and $y$ lie in the complete residue system $\{0,\dots,mn-2\}$ modulo $mn-1$.

### Step 3: Conclude equality

For $0 \le x \le mn-2$, we have:

- $y \equiv mx \pmod{mn-1}$,
- both $y$ and the least nonnegative residue of $mx$ lie in $\{0,\dots,mn-2\}$,

so they must be equal:

$$
y = (mx) \bmod (mn-1).
$$

For $x = mn-1$, both sides equal $mn-1$, so the same formula holds trivially if we interpret the last element as fixed.

### Final statement for (a)

The transposition on indices is:

$$
x \mapsto
\begin{cases}
mx \bmod (mn-1), & 0 \le x \le mn-2,\\[4pt]
mn-1, & x = mn-1.
\end{cases}
$$

This is a permutation consisting of a single fixed point and a cycle structure on the remaining $mn-1$ elements.

## (b) Methods for performing the transposition

The key fact is that the mapping on $\{0,\dots,mn-2\}$ is a permutation given by multiplication by $m$ modulo $mn-1$. Since $\gcd(m,mn-1)=1$, this permutation is invertible and decomposes into disjoint cycles.

A standard in-place method follows these cycles directly. One scans the array positions $x$ from $0$ to $mn-2$, and whenever an unvisited position is found, one follows the chain

$$
x,\; mx \bmod (mn-1),\; m^2x \bmod (mn-1), \dots
$$

until returning to $x$. The values along the cycle are rotated using a single temporary storage cell. A separate mark (or implicit marking via cycle completion) ensures each position is processed exactly once.

The element at index $mn-1$ is untouched, since it is a fixed point.

This is precisely the standard cycle-following algorithm for permutations, specialized to the multiplication permutation induced by the row-major to column-major index conversion.

A second viewpoint is to explicitly interpret indices as pairs $(i,j)$, compute their target $(j,i)$, and move elements accordingly. This is conceptually direct but typically requires either auxiliary storage or careful cycle handling to avoid overwriting data.

Both approaches are equivalent in structure: the transposition is a permutation of the $mn$ locations, and any permutation can be executed in place by decomposing it into disjoint cycles.
