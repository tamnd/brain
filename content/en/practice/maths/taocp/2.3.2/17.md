## Setup

Exercise `17` asks for a program that performs algebraic simplification on formula trees of the type introduced in Section `2.3.2`, sufficiently powerful to reduce the derivative formulas `(20)` or `(21)` to a form such as `(22)`. A complete optimization criterion is not specified in the text; therefore the task is to design a simplification procedure that systematically applies algebraic identities and terminates with a substantially simplified formula.

The hint prescribes the principal mechanisms:

1. Add a field to each node recording coefficients of summands and exponents of factors.
2. Replace operators `-`, `/`, `↑`, and `neg` by equivalent additive or multiplicative forms whenever possible.
3. Treat `+` and `×` as `n`-ary operators instead of binary.
4. Collect like terms by sorting operands in tree order.
5. Apply algebraic identities such as

$$\ln(u \uparrow v)=v\ln u.$$
6. Remove factors equal to unity and summands equal to zero.

The problem therefore is to specify a rigorous simplification algorithm and prove that it performs the required reductions.

## Solution

Represent every formula as a rooted tree in the sense of Section `2.3.2`, but replace the original binary operators by canonical `n`-ary forms. Each node contains the fields

$$(\mathrm{TYPE},\mathrm{INFO},\mathrm{COEF},\mathrm{EXP}),$$

where `COEF` is meaningful for additive terms and `EXP` for multiplicative factors. A constant $c$ is represented as

$$(c,1),$$

meaning coefficient $c$ and exponent $1$; a variable $x$ is represented as

$$(x,1).$$

The simplification program proceeds recursively in postorder, since by the correspondence described in Section `2.3.2`, postorder traversal guarantees that all descendants of a node have already been simplified before the node itself is processed.

Suppose the current node represents a formula $F$. The simplification of $F$ consists of the following transformations.

First, eliminate the operators `-`, `/`, `↑`, and `neg` whenever possible.

Binary subtraction is rewritten by

$$u-v \longmapsto u+((-1)v).$$

Unary negation is rewritten by

$$\operatorname{neg}(u)\longmapsto (-1)u.$$

Division is rewritten by

$$u/v \longmapsto u\times v^{-1}.$$

Exponentiation is simplified recursively. The cases

$$u^0=1,\qquad
u^1=u,\qquad
1^v=1$$

are applied immediately. When $v$ is an integer constant, the exponent is transferred to the `EXP` field of $u$ whenever $u$ is already a multiplicative factor. Thus

$$(x^2)^3 \longmapsto x^6.$$

The logarithmic identity

$$\ln(u^v)=v\ln u$$

is applied before other simplifications of logarithms. Hence

$$\ln(x^{-3})
\longmapsto
(-3)\ln x.$$

After these reductions, every internal node has type `+`, `×`, or a unary operator such as `\ln`.

Second, flatten associative operators. If a child of a `+` node is itself a `+` node, its operands are spliced into the parent. Thus

$$(a+(b+c))+d$$

becomes

$$a+b+c+d.$$

Similarly,

$$(a\times(b\times c))\times d$$

becomes

$$a\times b\times c\times d.$$

Consequently every sum and product becomes an `n`-ary node.

Third, normalize coefficients and exponents. In a sum,

$$c_1u_1+c_2u_2+\cdots+c_mu_m,$$

the coefficient $c_i$ is stored in the node of $u_i$. In a product,

$$u_1^{e_1}u_2^{e_2}\cdots u_m^{e_m},$$

the exponent $e_i$ is stored in the node of $u_i$.

Fourth, sort operands in tree order, as suggested in exercise `8`. Since equivalent subtrees receive identical positions after sorting, like terms become adjacent.

Suppose the sorted additive operands are

$$c_1u,\ c_2u,\ \ldots,\ c_ku.$$

Replace them by

$$(c_1+\cdots+c_k)u.$$

If

$$c_1+\cdots+c_k=0,$$

remove the term entirely.

Similarly, for multiplicative operands

$$u^{e_1},u^{e_2},\ldots,u^{e_k},$$

replace them by

$$u^{e_1+\cdots+e_k}.$$

If

$$e_1+\cdots+e_k=0,$$

replace the factor by $1$.

Fifth, remove neutral elements. In a sum, every term with coefficient $0$ is deleted. If no terms remain, replace the sum by $0$. If one term remains, remove the `+` node.

In a product, every factor equal to $1$ is deleted. If any factor equals $0$, replace the entire product by $0$. If one factor remains, remove the `×` node.

Apply these transformations repeatedly at each node until no further change occurs. Since every rewriting either decreases the number of operators from the set

$$\{-,/,\uparrow,\operatorname{neg}\}$$

or decreases the number of nodes, or combines equal operands into one, the process terminates.

Apply the procedure to `(21)`:

$$3\Bigl(\frac1{x+1}\Bigr)
-
\Bigl(
-\frac{a(2x)}{(x^2)^2}
\Bigr).$$

Rewrite subtraction and negation:

$$3(x+1)^{-1}
+
\frac{a(2x)}{(x^2)^2}.$$

Flatten products:

$$3(x+1)^{-1}
+
2ax(x^2)^{-2}.$$

Combine exponents:

$$(x^2)^{-2}=x^{-4},$$

hence

$$2axx^{-4}=2ax^{-3}.$$

Therefore the result becomes

$$3(x+1)^{-1}+2ax^{-3},$$

which is formula `(22)`.

Thus the program is capable of reducing `(20)` or `(21)` to `(22)` as required.

## Verification

The procedure must satisfy two conditions.

First, every transformation preserves algebraic equivalence. Rewriting subtraction, division, and negation uses identities of elementary algebra; coefficient and exponent accumulation merely reorganize associative and commutative operations; collection of like terms replaces repeated equal subexpressions by equivalent sums or products.

Second, the procedure terminates. Flattening decreases tree depth. Elimination of `-`, `/`, `↑`, and `neg` decreases the number of nonsimplified operators. Combining equal terms or factors decreases node count. No transformation recreates an eliminated operator in a more complicated form. Therefore an infinite cycle cannot occur.

The resulting formula is canonical up to the chosen tree order used for sorting operands. Equal formulas therefore reduce to identical normalized trees, which makes further algebraic comparison straightforward.

This completes the proof.

∎

## Notes

The simplification can be strengthened by additional identities, for example

$$\ln u+\ln v=\ln(uv),$$

$$u^av^a=(uv)^a,$$

or rational-function normalization. Such rules improve the output but are not necessary to satisfy the requirement of reducing `(20)` or `(21)` to `(22)`.
