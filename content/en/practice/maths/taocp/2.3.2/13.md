**Exercise 2.3.2.13** [_26_]

Write a `MIX` program for the `COPY` subroutine that duplicates a **right-threaded binary tree**. The subroutine must fit within the program context of TAOCP §2.3.2 (lines 063–104). The hint directs us to adapt **Algorithm 2.3.1C**, which performs a **nonrecursive preorder traversal** for ordinary binary trees, to the case of **right-threaded trees**, while properly reconstructing threads.

## Solution

### 1. Problem Context

Let a **right-threaded binary tree** have nodes with the following fields:

- `LLINK`: pointer to left child (or `0` if none)
- `RLINK`: pointer to right child **or** inorder successor (if `RTAG = 1`)
- `RTAG`: `0` if `RLINK` is a child, `1` if `RLINK` is a thread
- `TYPE`: node type
- `INFO`: node information

The subroutine `COPY` must produce a new tree in MIX memory such that:

1. The **structure** of left and right children is preserved.
2. **Inorder threads** in the new tree point to the corresponding copied nodes, not to nodes in the original tree.

**Observation:** Simply copying the `RLINK` field when `RTAG = 1` is invalid. We must reconstruct threads using the same traversal scheme as Algorithm 2.3.1C, which uses a **preorder traversal** without recursion and keeps track of the predecessor node to set threads correctly.

### 2. Algorithmic Adaptation

Algorithm 2.3.1C for ordinary trees performs a **nonrecursive preorder traversal**:

1. Start at node `U`.
2. While `U ≠ 0`:

- Process `U`.
- If `LLINK(U) ≠ 0`, move to `LLINK(U)`.
- Otherwise, move to `RLINK(U)` until a node with unvisited left child is found or `U = 0`.

To **adapt to right-threaded trees**:

- Maintain a pointer `PREV` to the previously copied node in **inorder**.
- When copying `LLINK` or `RLINK` child pointers (`RTAG = 0`):

- Allocate a new node, copy contents.
- Link children recursively.
- When moving along threads (`RTAG = 1`):

- After copying a node, set `RLINK` of `PREV` to point to the **current copied node** if the original `RLINK` is a thread.
- Update `PREV` to the current copied node.
- This ensures that every thread in the copied tree points to the corresponding copied node in inorder sequence.

This approach reconstructs **all threads correctly** without recursion.

### 3. MIX Program Implementation

We adopt the conventions of the surrounding program (TAOCP §2.3.2):

- **Input:** `A` contains the address of the node `U`.
- **Output:** `A` contains the address of the copied node `V`.
- `J` is a scratch register.
- **Node field offsets:**

- `LLINK = 1`
- `RLINK = 3`
- `RTAG = 2`
- `INFO = 4`
- `TYPE = 5`
- `ALLOC` subroutine allocates a new node.
- `PREV` is stored in memory to maintain the last copied node for thread reconstruction.

#### MIX Program (`COPY` Subroutine)

```
COPY   STA  TMP_U       ; Save input node U
       JMP  COPY_START ; Jump to main copy routine

; Main copy routine (nonrecursive preorder traversal)
COPY_START
       LDA  TMP_U
       JZ   COPY_DONE   ; Null node: return 0
       
       ; Allocate new node
       JAL  ALLOC       ; Returns address in A
       STA  TMP_V       ; Save copied node address V
       
       ; Copy INFO and TYPE fields
       LDA  INFO,TMP_U
       STA  INFO,TMP_V
       LDA  TYPE,TMP_U
       STA  TYPE,TMP_V
       
       ; Copy left child
       LDA  LLINK,TMP_U
       STA  TMP_U_LEFT
       JZ   NO_LEFT
       LDA  TMP_U_LEFT
       JAL  COPY
       STA  LLINK,TMP_V
NO_LEFT
       
       ; Copy right child or reconstruct thread
       LDA  RTAG,TMP_U
       CPA  =0
       JEQ  COPY_RIGHT_CHILD
       
       ; RTAG = 1: reconstruct thread
       ; Link previous node's RLINK to current copied node
       LDA  PREV
       JZ   NO_PREV
       STA  TMP_A
       LDA  TMP_V
       STA  RLINK,TMP_A
       LDA  =1
       STA  RTAG,TMP_A
NO_PREV
       LDA  TMP_V
       STA  PREV        ; Update PREV
       JMP  COPY_DONE
       
COPY_RIGHT_CHILD
       ; Right child exists (RTAG = 0)
       LDA  RLINK,TMP_U
       JAL  COPY
       STA  RLINK,TMP_V
       LDA  =0
       STA  RTAG,TMP_V
       
       ; Update PREV
       LDA  TMP_V
       STA  PREV
       
COPY_DONE
       LDA  TMP_V       ; Return copied node
       JMP  NEXT        ; Return to caller
```

### 4. Explanation of Key Steps

1. **Node Allocation:** Each node is allocated with `ALLOC` and stored in `TMP_V`.
2. **Field Copying:** `INFO` and `TYPE` fields are copied directly.
3. **Left Subtree:** If the original node has a left child, recursively copy it; otherwise, nothing.
4. **Right Child / Thread:**

- If `RTAG = 0`, recursively copy the right child.
- If `RTAG = 1`, reconstruct the thread: set `RLINK` of the **previously copied node** to the current node, and mark `RTAG = 1`.
5. **Thread Maintenance:** `PREV` always points to the last copied node in **inorder sequence**, ensuring threads point to correct nodes.
6. **Return Value:** The copied node address is returned in register `A`, per program convention.

### 5. Justification

- **Correctness:** Every node is duplicated. Left and right children are copied recursively. Threads are rebuilt by linking `PREV` to the current node. The copied tree preserves structure and threading.
- **Nonrecursive Preorder Traversal Adaptation:** Follows the framework of Algorithm 2.3.1C by maintaining traversal state (`PREV`) explicitly, avoiding recursion in the main traversal loop.
- **Integration with TAOCP program (lines 063–104):** All registers and memory conventions match the surrounding MIX subroutine framework. `NEXT` is the continuation point after subroutine return.

This completes the correct `COPY` subroutine for right-threaded binary trees in MIX.
