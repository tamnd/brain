**Exercise 3.** [*20*] Change Algorithm E (for the sake of efficiency) so that all trivial replacement operations such as “$m \leftarrow n$” are avoided. Write this new algorithm in the style of Algorithm E, and call it Algorithm F.

**Verified:** no  
**Solve time:** 7m20s

## Solution

The purpose of Algorithm F is to avoid all trivial replacement operations in Algorithm E, namely assignments of the form $x \leftarrow y$ when $x$ already has the value $y$. Since Algorithm E repeatedly performs the pair of assignments

$$m \leftarrow n, \qquad n \leftarrow r,$$

we seek a formulation in which every replacement necessarily changes the value of the variable being assigned.

The key observation is that after the first execution of step E1, one always has $m>n$ at the beginning of the loop (Exercise 1.1.2). Hence, when the remainder $r\neq 0$,

$$0 \le r < n < m.$$

Thus both assignments

$$m \leftarrow n, \qquad n \leftarrow r$$

are automatically nontrivial, since $m\neq n$ and $n\neq r$. The only possible trivial replacement in Algorithm E occurs at the beginning, if initially $m=n$. This may be avoided by testing equality first.

We therefore obtain the following modification.

**Algorithm F** (_Euclid’s algorithm avoiding trivial replacements_). Given two positive integers $m$ and $n$, find their greatest common divisor.

**F1.** [Equal?] If $m=n$, terminate; the answer is $m$.

**F2.** [Ensure $m>n$.] If $m<n$, exchange $m \leftrightarrow n$.

**F3.** [Find remainder.] Divide $m$ by $n$ and let $r$ be the remainder:

$$m = qn + r, \qquad 0 \le r < n.$$

**F4.** [Is it zero?] If $r=0$, terminate; the answer is $n$.

**F5.** [Reduce.] Set

$$m \leftarrow n, \qquad n \leftarrow r,$$

and return to step F3.

We verify that every replacement is nontrivial.

In step F2, the exchange occurs only when $m<n$, hence it necessarily changes both variables. If $m=n$, the algorithm terminates in F1, so no trivial exchange is performed.

At the beginning of F3, after the first pass, one has $m>n$. Since $r<n$, whenever $r\neq 0$,

$$r<n<m.$$

Therefore $m\leftarrow n$ in F5 is nontrivial because $m\neq n$, and $n\leftarrow r$ is nontrivial because $r<n$, hence $r\neq n$.

The algorithm performs exactly the same sequence of Euclidean divisions as Algorithm E, except that it terminates immediately when $m=n$ instead of carrying out the trivial replacement $m\leftarrow n$. Therefore the correctness proof is identical to that of Algorithm E, and all trivial replacement operations have been eliminated.

$\square$
