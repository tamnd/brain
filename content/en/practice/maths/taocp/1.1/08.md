## Solution

Equation (3) defines a formal algorithm by a finite set of quadruples

$$(\theta_j,\phi_j,a_j,b_j),$$

where $\theta_j$ is a predicate deciding whether rule $j$ applies, $\phi_j$ is the associated operation, and $a_j,b_j$ determine the next rule according as $\theta_j$ is false or true.

Represent the state by strings of the form

$$a^m b^n,$$

with $m,n>0$. The intended meaning is that the number of $a$'s equals the current value of $m$, and the number of $b$'s equals the current value of $n$).

The algorithm repeatedly replaces $(m,n)$ by

$$(|m-n|,\min(m,n)),$$

which preserves the greatest common divisor:

$$(m,n) = (|m-n|,\min(m,n)).$$

When the two numbers become equal, that common value is the greatest common divisor.

We use the following rules.

Let the current string be denoted by $x$.

Rule 1:

$$\theta_1(x) \equiv \text{$x$ contains the substring $ab$}.$$

If $\theta_1$ is true, perform $\phi_1$ by deleting one occurrence of $ab$ from the string. Thus

$$a^m b^n \mapsto
\begin{cases}
a^{m-n}, & m>n,\\[4pt]
b^{n-m}, & n>m.
\end{cases}$$

If $\theta_1$ is false, the string has the form $a^k$ or $b^k$ for some $k>0$. In this case the computation terminates.

The remaining rules convert a pure $a$-string or pure $b$-string into the required output form.

Rule 2:

$$\theta_2(x) \equiv \text{$x$ contains no $b$}.$$

If $\theta_2$ is true, replace

$$a^k \mapsto a^k b^k.$$

Rule 3:

$$\theta_3(x) \equiv \text{$x$ contains no $a$}.$$

If $\theta_3$ is true, replace

$$b^k \mapsto a^k b^k.$$

The control table is therefore

$$\begin{array}{c|c|c|c}
j & \theta_j & a_j & b_j \\ \hline
1 & \text{contains }ab & 2 & 1 \\
2 & \text{contains no }b & 3 & 0 \\
3 & \text{contains no }a & 0 & 0
\end{array}$$

where state $0$ denotes termination.

The operations are

$$\phi_1:\ a^m b^n \mapsto
\begin{cases}
a^{m-n}, & m>n,\\
b^{n-m}, & n>m,
\end{cases}$$

$$\phi_2:\ a^k \mapsto a^k b^k,$$

$$\phi_3:\ b^k \mapsto a^k b^k.$$

To prove correctness, begin with the input $a^m b^n$. Each application of $\phi_1$ replaces $(m,n)$ by $(|m-n|,\min(m,n))$, and

$$(m,n) = (|m-n|,\min(m,n))$$

by the argument underlying Algorithm E. Since $m+n$ decreases whenever $m\ne n$, repeated application eventually reaches a state with equal numbers. At that moment the string becomes either $a^d$ or $b^d$, where

$$d=(m,n).$$

Rule 2 or Rule 3 then converts this to

$$a^d b^d,$$

and the algorithm terminates.

Hence the output represents the greatest common divisor of the original integers $m$ and $n$. The algorithm is effective because each predicate and each operation is completely specified and mechanically executable.

Therefore the formal algorithm computes the greatest common divisor correctly. This completes the proof.
