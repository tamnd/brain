## Solution

We are asked to give insertion and deletion operations that implement a _stack_ using the circular list representation with a list head, as in diagram (4) of Section 2.2.4. In a stack, the last element inserted is the first to be removed (LIFO behavior). Let the list head be denoted by `HEAD`. The stack will be represented as the circular list whose first node after `HEAD` is the top of the stack. All operations will maintain the circularity of the list, and the invariant that `LINK(HEAD)` points to the top node (or back to `HEAD` if the stack is empty).

**Insertion (Push)**. Let `X` be the element to be pushed onto the stack. We take a node from `AVAIL` and insert it immediately after `HEAD`, thereby making it the new top of the stack.

1. `P ⇐ AVAIL`
2. `COEF(P) ⇐ X`  (or `INFO(P) ⇐ X` in the general list)
3. `LINK(P) ⇐ LINK(HEAD)`
4. `LINK(HEAD) ⇐ P`

After these four assignments, `P` is the new first node after `HEAD`, which is the top of the stack.

**Deletion (Pop)**. To remove the top element from the stack, we remove the node immediately following `HEAD` and return its value in `X`.

1. Test if `LINK(HEAD) = HEAD`; if so, signal `UNDERFLOW`.
2. `P ⇐ LINK(HEAD)`
3. `X ⇐ COEF(P)`  (or `INFO(P)` for general list)
4. `LINK(HEAD) ⇐ LINK(P)`
5. `AVAIL ⇐ P`

After these operations, the top element is removed, and the stack remains a circular list with `HEAD` as sentinel.

**Correctness.** The invariant maintained is that `LINK(HEAD)` always points to the top of the stack. Each push inserts a new node immediately after `HEAD`, and each pop removes exactly that node. The circularity ensures that traversals starting at `HEAD` remain valid, and the list head provides a convenient sentinel to detect an empty stack.

**Complexity.** Each push or pop requires exactly four or five pointer operations, all constant time. No traversal of the list is necessary.

This completes the solution.
