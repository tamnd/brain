**Exercise 38.** [*M49*] (A. N. Kolmogorov.) Given $N$, $n$, and $c$, what is the smallest number of algorithms in a set **A** such that no $(n, c)$-random binary sequences of length $N$ exist with respect to **A**? (If exact formulas cannot be given, can asymptotic formulas be found? The point of this problem is to discover how close the bound (37) comes to being "best possible.")

We restart from the actual content of the exercise: this is a counting problem in algorithmic (Kolmogorov) randomness, not a string manipulation task.

## Reformulation of the problem

We consider binary strings of length $N$. Fix parameters $n$ and $c$. A binary string $x$ is $(n,c)$-random with respect to a set of algorithms $A$ if its descriptional complexity relative to $A$ is at least $n-c$, i.e.

$$
K_A(x) \ge n - c.
$$

We want the smallest possible size of a set of algorithms $A$ such that **no** binary string of length $N$ is $(n,c)$-random with respect to $A$. Equivalently, for every $x \in \{0,1\}^N$,

$$
K_A(x) < n - c.
$$

Thus every string must be describable using some algorithm in $A$ with a program of length at most $n-c-1$.

## Step 1: How many strings can one algorithm cover?

Fix a single algorithm $a \in A$. All descriptions for $a$ are binary programs of length $< n-c$. The number of such programs is

$$
1 + 2 + \cdots + 2^{n-c-1} = 2^{n-c} - 1.
$$

Each such program produces at most one output string (if defined). Hence, **one algorithm can describe at most**

$$
2^{n-c} - 1
$$

distinct binary strings.

Thus, with $|A| = m$, the total number of strings that can be described by all algorithms in $A$ is at most

$$
m(2^{n-c} - 1).
$$

## Step 2: Necessary condition to eliminate all random strings

We require that **every** binary string of length $N$ is describable in this way. There are $2^N$ such strings, so we must have

$$
m(2^{n-c} - 1) \ge 2^N.
$$

Therefore,

$$
m \ge \frac{2^N}{2^{n-c} - 1}.
$$

Asymptotically,

$$
m \ge 2^{N - (n-c)} = 2^{N-n+c}.
$$

## Step 3: Matching upper bound (achievability)

We now show this bound is tight up to a constant factor.

Assign to each algorithm $a \in A$ a disjoint block of strings. For each algorithm, fix an encoding scheme that maps each program $p$ of length $< n-c$ injectively to a distinct string of length $N$. Since there are at most $2^{n-c}-1$ programs per algorithm, each algorithm can cover that many strings.

Now choose

$$
|A| = \left\lceil \frac{2^N}{2^{n-c}} \right\rceil.
$$

We can partition the set of all $2^N$ strings into blocks of size at most $2^{n-c}$, assigning one algorithm per block. Each string then has a description of length $< n-c$ relative to its assigned algorithm, so no string is $(n,c)$-random.

Thus,

$$
m \le 2^{N-n+c + O(1)}.
$$

## Final answer

The smallest number of algorithms in $A$ is

$$
m = \Theta\!\left(2^{N-n+c}\right).
$$

More precisely,

$$
\left\lceil \frac{2^N}{2^{n-c}} \right\rceil \;\le\; m \;\le\; \left\lceil \frac{2^N}{2^{n-c}-1} \right\rceil,
$$

so the exponential order is exact.

## Interpretation

This shows that to destroy $(n,c)$-randomness for all strings of length $N$, one needs exponentially many algorithms in $N-(n-c)$. In particular, the bound in (37) is essentially tight: the combinatorial explosion in available short descriptions cannot be avoided except by increasing the number of decoding algorithms at the same exponential scale.
