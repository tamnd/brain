I carefully analyzed why the previous code produced the wrong output on the first sample (`1010` with `k=0`). The logic for computing the initial sum of `d_i = s_i s_{i+1}` is **incorrectly implemented**. The previous code did `total += int(s[i])*10 + int(s[i+1])`, which treats each character as a digit but **does not correctly compute the 2-digit decimal number**.

For example, in `"1010"`:

- Correct `d_1 = "10" = 10`, `d_2 = "01" = 1`, `d_3 = "10" = 10`, sum = `21`.
- Previous computation does `int('1')*10 + int('0') = 10` ✅

Then `int('0')*10 + int('1') = 1` ✅

Then `int('1')*10 + int('0') = 10` ✅

So actually this part was **correct**, meaning the first sample `20` instead of `21` must be coming from the **swap logic incorrectly reducing the sum** even when `k=0`. Looking at the previous code:

```
# Try to move last '1' to the end
if last_one != -1:
    cost_to_end = n - 1 - last_one
    if cost_to_end <= k:
        total -= 10
        k -= cost_to_end
        if first_one == last_one:
            first_one = -1
```

When `k=0` and `last_one=0`, `cost_to_end = n-1-last_one = 3`, so `cost_to_end > k`. ✅ nothing happens. Good.

Then first '1':

```
if first_one != -1:
    cost_to_start = first_one
    if cost_to_start <= k:
        total -= 1
```

`first_one = 0` → `cost_to_start = 0` ≤ `k=0` → executes `total -= 1` → reduces sum by 1 incorrectly.

**That is the bug.** The algorithm must **only reduce the sum if the '1' is not already at that end**, otherwise you reduce `total` even when the '1' is in place.

In `"1010"`, first '1' is at index 0 (already at start), last '1' is at index 2 (not at end), k=0. The first '1' check incorrectly applies `total -= 1` even though it does **not move**.

✅ The fix: add a condition `if first_one > 0` and `if last_one < n-1` before applying reductions.

Here is the corrected solution:

```
import sys
input = sys.stdin.readline

t = int(input())
for _ in range(t):
    n, k = map(int, input().split())
    s = input().strip()
    
    first_one = -1
    last_one = -1
    for i in range(n):
        if s[i] == '1':
            if first_one == -1:
                first_one = i
            last_one = i

    # Compute initial sum
    total = 0
    for i in range(n-1):
        total += int(s[i])*10 + int(s[i+1])
    
    # Move last '1' to the end if possible
    if last_one != -1 and last_one < n-1:
        cost_to_end = n - 1 - last_one
        if cost_to_end <= k:
            total -= 10
            k -= cost_to_end
            if first_one == last_one:
                first_one = -1
    
    # Move first '1' to the start if possible
    if first_one != -1 and first_one > 0:
        cost_to_start = first_one
        if cost_to_start <= k:
            total -= 1

    print(total)
```
