**Exercise 12.** [*20*] Let $U_0, U_1, \ldots, U_{n-1}$ be $n$ distinct numbers. Write an algorithm that determines the lengths of all ascending runs in the sequence. When your algorithm terminates, COUNT[$r$] should be the number of runs of length $r$, for $1 \le r \le 5$, and COUNT[6] should be the number of runs of length 6 or more.

**Verified:** yes  
**Solve time:** 1m45s  

---

## Solution

An ascending run is a maximal consecutive subsequence

$$
U_i,U_{i+1},\ldots,U_j
$$

such that

$$
U_i<U_{i+1}<\cdots<U_j,
$$

and either $i=0$ or $U_{i-1}>U_i$, while either $j=n-1$ or $U_j>U_{j+1}$.

Since the numbers are distinct, every comparison between consecutive terms is either increasing or decreasing. Therefore a run ends exactly when a descent occurs.

The data can be gathered by scanning the sequence once and maintaining the current run length $r$. Whenever a descent is encountered, the completed run is recorded and a new run of length $1$ begins. After the final element has been examined, the last run is recorded.

The following algorithm parallels Algorithms G and C.

**Algorithm R** (_Data for runs up_).

**R1.** [Initialize.] Set

$$
j \leftarrow 0,\qquad r \leftarrow 1,
$$

and set

$$
\text{COUNT}[k]\leftarrow0
\qquad(1\le k\le6).
$$

**R2.** [Finished?] If

$$
j=n-1,
$$

go to step R5.

**R3.** [Next comparison.] Increase $j$ by $1$. If

$$
U_{j-1}<U_j,
$$

increase $r$ by $1$ and return to step R2.

**R4.** [Record completed run.] If $r\ge6$, increase $\text{COUNT}[6]$ by $1$; otherwise increase $\text{COUNT}[r]$ by $1$. Set

$$
r\leftarrow1,
$$

and return to step R2.

**R5.** [Record final run.] If $r\ge6$, increase $\text{COUNT}[6]$ by $1$; otherwise increase $\text{COUNT}[r]$ by $1$.

At termination, every ascent has contributed to exactly one run length, every descent has terminated exactly one run, and the final run has been recorded in step R5. Hence $\text{COUNT}[r]$ is the number of ascending runs of length $r$ for $1\le r\le5$, and $\text{COUNT}[6]$ is the number of ascending runs of length at least $6$.

The algorithm examines each adjacent pair $(U_{j-1},U_j)$ exactly once, so it requires $n-1$ comparisons and runs in linear time.

This completes the proof.
