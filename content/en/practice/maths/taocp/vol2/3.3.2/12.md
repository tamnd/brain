**Exercise 12.** [*20*] Let $U_0, U_1, \ldots, U_{n-1}$ be $n$ distinct numbers. Write an algorithm that determines the lengths of all ascending runs in the sequence. When your algorithm terminates, COUNT[$r$] should be the number of runs of length $r$, for $1 \le r \le 5$, and COUNT[6] should be the number of runs of length 6 or more.

Let $U_0, U_1, \ldots, U_{n-1}$ be $n$ distinct numbers. An ascending run is a maximal contiguous subsequence $U_i, U_{i+1}, \ldots, U_j$ such that $U_k < U_{k+1}$ for all $i \le k < j$. Maximality means that either $i = 0$ or $U_{i-1} > U_i$, and either $j = n-1$ or $U_j > U_{j+1}$ fails, which is equivalent to $j = n-1$ or $U_j > U_{j+1}$ being false, hence $j = n-1$ or $U_j < U_{j+1}$ does not hold. Since all values are distinct, equality never occurs, so every adjacent pair is either strictly increasing or strictly decreasing.

We consider the following single-pass procedure that partitions the sequence into maximal ascending runs and records their lengths.

We initialize $\text{COUNT}[r] = 0$ for $1 \le r \le 5$ and $\text{COUNT}[6] = 0$. We set $i \leftarrow 0$. While $i < n$, we begin a new run at position $i$ by setting $j \leftarrow i$ and $\ell \leftarrow 1$. We then extend the run as long as it remains strictly increasing, that is, while $j < n-1$ and $U_j < U_{j+1}$, we set $j \leftarrow j+1$ and $\ell \leftarrow \ell+1$. When this condition fails, the segment $U_i, \ldots, U_j$ is recorded as one run. If $1 \le \ell \le 5$, we increment $\text{COUNT}[\ell]$, and if $\ell \ge 6$, we increment $\text{COUNT}[6]$. Finally we set $i \leftarrow j+1$ and continue.

To prove correctness, we must show that this procedure produces exactly the maximal ascending runs and that these runs form a partition of the index set $\{0, 1, \ldots, n-1\}$.

First we show that every run produced by the algorithm is an ascending run and is maximal. By construction, during extension we maintain $U_k < U_{k+1}$ for all $k$ between $i$ and $j-1$, so the segment is strictly increasing. The loop terminates either because $j = n-1$ or because $U_j > U_{j+1}$. In the first case, no extension to the right is possible. In the second case, the strict decrease at $j$ implies that appending $U_{j+1}$ would violate the increasing property. Hence in both cases the segment $U_i, \ldots, U_j$ cannot be extended while preserving strict increase, so it is maximal.

Next we show that these runs cover the entire sequence and are disjoint. The algorithm starts at $i = 0$, so the first run begins at the first element. Assume inductively that after finishing a run ending at index $j$, the next starting index is set to $i = j+1$. This ensures disjointness because no index is ever included in two different runs.

It remains to show that every index belongs to some run, which also guarantees termination and completeness. Consider any index $t$. If $t = 0$, it is the start of the first run. If $t > 0$, then when the algorithm moves from $t-1$ to $t$, there are two possibilities. If $U_{t-1} < U_t$, then the inner extension procedure will not stop at $t-1$, so $t$ lies in the same run as $t-1$. If instead $U_{t-1} > U_t$, then the condition for extending a run fails at $t-1$, so the algorithm must have ended the previous run at some index $j \le t-1$ and restarted at $i = t$. In this case $t$ is the first element of a new run. Thus every index either continues an existing run or starts a new one, and therefore belongs to exactly one run. This establishes that the runs form a partition of $\{0,1,\ldots,n-1\}$.

Finally, since each run is processed exactly once and its length is counted immediately when the run is closed, every maximal ascending run contributes exactly one increment to the appropriate entry of $\text{COUNT}$. The mapping of lengths into buckets $1$ through $5$ and the saturation into $\text{COUNT}[6]$ for all lengths at least $6$ matches the specification, so the array $\text{COUNT}$ is correctly computed.

This completes the proof of correctness.
