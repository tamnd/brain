[Section 3.2.1.1: Choice of Modulus](../)

**Exercise 6.** &#9654; [*28*] $[28]$ The previous exercise suggests that subtraction mod $m$ is easier to perform than addition mod $m$. Discuss sequences generated by the rule

$$X_{n+1} = (a X_n - c) \bmod m.$$

Are these sequences essentially different from linear congruential sequences as defined in the text? Are they more suited to efficient computer calculation?

**Verified:** yes  
**Solve time:** 14m32s  

---

## Setup

Let $m$ be a positive integer modulus and let $a, c, X_0$ be integers with $0 \le X_0 < m$. Define a sequence by

$X_{n+1} = (aX_n - c) \bmod m.$

A standard linear congruential sequence has the form

$X_{n+1} = (aX_n + d) \bmod m,$

for some constant $d$ with $0 \le d < m$.

The goal is to compare these two recurrences and determine whether the subtraction form defines a genuinely different class of sequences and whether it offers computational advantages.

## Solution

Define the integer $d$ by the congruence

$d \equiv -c \pmod m,\qquad 0 \le d < m.$

Equivalently,

$d = (m - (c \bmod m)) \bmod m.$

For every integer $X_n$, the difference

$(aX_n - c) - (aX_n + d) = -(c + d)$

is a multiple of $m$ because $d \equiv -c \pmod m$ implies $c + d \equiv 0 \pmod m$. Hence

$aX_n - c \equiv aX_n + d \pmod m.$

Taking both sides modulo $m$ yields

$(aX_n - c) \bmod m = (aX_n + d) \bmod m.$

Substituting into the recurrence gives

$X_{n+1} = (aX_n + d) \bmod m,$

so every sequence defined by subtraction with parameter $c$ coincides term-for-term with a standard linear congruential sequence with parameter $d = (-c \bmod m)$, provided the same initial value $X_0$ is used.

The transformation is reversible because $c \equiv -d \pmod m$ determines $c$ uniquely in $[0,m-1]$ from $d$ in $[0,m-1]$. Hence the two formulations generate exactly the same set of sequences.

In computational terms, when arithmetic is performed modulo the word size $w$ as in MIX, the operation $(aX_n - c) \bmod w$ is implemented by multiplication followed by a subtraction in the accumulator. The subtraction step produces wrap-around modulo $w$ in the same way as addition, since the machine arithmetic already reduces results modulo $w$.

Thus the subtraction form does not enlarge the class of generators and does not change their period structure or statistical properties. It only replaces the additive constant $d$ by its congruent representative $-c \bmod m$, which may match the native machine operation more directly when subtraction is simpler to implement than addition with carry handling.

This completes the proof. ∎

## Verification

The key congruence step uses $d \equiv -c \pmod m$, which implies $c + d = km$ for some integer $k$. Substituting into $aX_n - c$ gives

$aX_n - c = aX_n + d - km,$

so both expressions differ by a multiple of $m$. Reduction modulo $m$ eliminates the term $km$, validating equality of residues.

The construction of $d$ ensures $0 \le d < m$, so the transformed recurrence remains within the standard definition of a linear congruential generator.

## Notes

The subtraction form is algebraically redundant, since it corresponds exactly to a change of parameter $c \mapsto (-c \bmod m)$. Its only potential advantage lies in implementation on machines where subtraction and modular wrap-around interact more naturally than addition with carry correction, but it introduces no new theoretical behavior and no new family of sequences.
