## Setup

Let $m$ be a positive integer less than the computer word size $w$, and let $x$ and $y$ be nonnegative integers satisfying $0 \le x, y < m$. We are asked to exhibit efficient MIX code to compute the modular difference

$$
(x - y) \bmod m
$$

and the modular sum

$$
(x + y) \bmod m
$$

using a minimal number of instructions, and without performing a division operation. The goal is to count and justify each instruction and show that the result is correctly reduced modulo $m$. Registers A and X are available for computation.

We follow the conventions of Section 3.2.1.1: addition or subtraction modulo $w$ is performed automatically in the A-register, provided that overflow is managed properly, and conditional jumps allow correction when the preliminary result falls outside $[0, m-1]$.

## Solution

### Modular difference

We first compute $x - y \bmod m$. Observe that the straightforward subtraction

$$
rA \leftarrow x - y
$$

produces a signed integer in the range $-(m-1) \le rA \le m-1$, since $x$ and $y$ are both less than $m$. If $rA \ge 0$, no further adjustment is needed; if $rA < 0$, we can add $m$ to obtain a nonnegative remainder modulo $m$.

This procedure requires four MIX instructions:

```

ADD  M       rA ŌåÉ rA + m

```

Instruction 1 loads $x$ into $rA$, instruction 2 subtracts $y$, instruction 3 jumps if the result is nonnegative, instruction 4 adds $m$ if the result is negative, and instruction 5 is a label for the exit point. Each instruction corresponds to exactly one fundamental operation in MIX.

The subtraction produces an intermediate value in $[-(m-1), m-1]$. Adding $m$ in the negative case ensures that the final value lies in $[0, m-1]$, as required.

### Modular sum

For the sum $x + y \bmod m$, compute the ordinary sum in the A-register:

$$
rA \leftarrow x + y
$$

This sum lies in $[0, 2m - 2]$. If $rA \ge m$, subtract $m$ to reduce the value modulo $m$; otherwise, leave it unchanged. This procedure is implemented in four instructions:

```
LDA  X       rA ŌåÉ x
ADD  Y       rA ŌåÉ rA + y
SUB  M       rA ŌåÉ rA ŌłÆ m
JPOS **1     jump to end if rA Ōēź 0
ADD  M       rA ŌåÉ rA + m
**1 NOP      end
```

Here, we first perform $rA \leftarrow x + y - m$. If the result is nonnegative, it already lies in $[0, m-1]$. If negative, adding $m$ restores it to the correct range. This method requires only four or five instructions depending on the exact branching strategy, and it avoids any division operation.

## Verification

For the difference, consider the cases $x \ge y$ and $x < y$.

- If $x \ge y$, then $rA = x - y \ge 0$, the jump is taken, and the final value is $x - y$, which equals $(x - y) \bmod m$ because $x - y < m$.
- If $x < y$, then $rA = x - y < 0$, the jump is not taken, $m$ is added, producing $rA = x - y + m$. Since $0 \le x < m$ and $0 \le y < m$, we have $0 < x - y + m < m$, as required.

For the sum, consider $x + y < m$ and $x + y \ge m$:

- If $x + y < m$, then $rA \leftarrow x + y - m < 0$, so $m$ is added, giving $rA = x + y$, which is already modulo $m$.
- If $x + y \ge m$, then $rA \leftarrow x + y - m \ge 0$, the jump is taken, and the value remains $rA = x + y - m$, which equals $(x + y) \bmod m$.

In each case, the instructions produce the correct result with minimal operations, and no division is required.

## Notes

An alternative strategy for the sum is to compute $rA \leftarrow x + y$ first and then use the conditional jump `JGE` to subtract $m$ if $rA \ge m$. This approach also uses four instructions and is simpler in branching logic. For both sum and difference, the general principle is to bring the intermediate result into the range $[-(m-1), 2m-2]$ and then apply a single conditional correction.

This completes the proof.
