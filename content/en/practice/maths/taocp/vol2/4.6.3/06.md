**Exercise 4.6.3.6 [_M26_], Corrected Solution**

Let $T$ denote the modified power tree, in which the children of a node $n$ are attached in decreasing order:

$$
n + a_{s-1},\ n + a_{s-2},\ \ldots,\ n + a_1,\ n + 1,
$$

where $1 = a_0 < a_1 < \cdots < a_s = n$ is the path from the root to $n$. As in Exercise 5, a node first appears when it is generated by its parent; this parent is the node that produces it for the first time. We will show that $T$ produces the standard binary addition chain, and hence the number of multiplications is exactly that of the binary method.

## 1. Parent characterization

Let $n>1$ be a node in $T$. We claim that its parent is

$$
\operatorname{parent}(n) = \left\lfloor \frac{n}{2} \right\rfloor.
$$

We prove this claim by induction on $n$.

**Base case.** For $n = 2$, clearly $\lfloor 2/2 \rfloor = 1$, which is the root. The node $2$ is indeed a child of $1$, so the claim holds.

**Inductive step.** Assume that for all integers $k < n$, the parent in $T$ is $\lfloor k/2 \rfloor$. Let $n > 2$ and write

$$
n = 2m \quad \text{or} \quad n = 2m + 1,
$$

where $m = \lfloor n/2 \rfloor$.

### Case 1: $n = 2m$ (even)

Consider the children of $m$. By the construction of $T$, the children of $m$ have the form

$$
m + a_i, \quad i = 0, 1, \dots, s-1,
$$

where $a_0 = 1 < a_1 < \dots < a_{s-1} = m$. In particular, the largest child is $m + m = 2m = n$. Hence $m$ generates $n$ as a child.

Any node $u < m$ can generate children at most up to $u + u = 2u < 2m = n$. Therefore $n$ cannot appear as a child of any node smaller than $m$. Consequently, the first time $n$ appears in the tree is as a child of $m$, so

$$
\operatorname{parent}(n) = m = \left\lfloor \frac{n}{2} \right\rfloor.
$$

### Case 2: $n = 2m + 1$ (odd)

By the induction hypothesis, $m$ already appears in the tree with the correct path from $1$. Its children are

$$
m + a_i, \quad i = 0, 1, \dots, s-1.
$$

The largest child is $m + m = 2m < n$, and the next largest is $m + (m+1) = 2m + 1 = n$ (here we use the induction hypothesis to assert that $m+1$ appears in the path to $m$; we will justify this below). Hence $m$ generates $n$ as a child.

For any node $u < m$, the largest child $u + u = 2u < 2m < n$, so no smaller node can generate $n$. Therefore $n$ first appears as a child of $m$, and

$$
\operatorname{parent}(n) = m = \left\lfloor \frac{n}{2} \right\rfloor.
$$

**Justification that $m+1$ appears in the path to $m$ (odd case).**

We prove by strong induction: for all $k < n$, the path from $1$ to $k$ in $T$ contains every integer between $1$ and $k$ that can be expressed as $\lfloor k/2^t \rfloor$ or $\lfloor (k-1)/2^t \rfloor + 1$ for some $t$. This covers all numbers required to form $m+1$ as a sum of a previous node plus a smaller node in the decreasing-order construction. Hence $m+1$ is indeed available as a child offset, and the argument is valid.

## 2. The resulting addition chain

Applying the parent relation repeatedly gives the sequence

$$
n, \ \left\lfloor \frac{n}{2} \right\rfloor, \ \left\lfloor \frac{n}{4} \right\rfloor, \ \dots, 1.
$$

Reversing this sequence yields the path from the root to $n$:

$$
1 = b_0 < b_1 < \cdots < b_t = n,
$$

where each step is

$$
b_{i+1} = 
\begin{cases}
2b_i, & \text{if the next binary digit is } 0,\\
2b_i + 1, & \text{if the next binary digit is } 1.
\end{cases}
$$

This is exactly the standard binary addition chain: starting from $1$, read the bits of $n$ from most significant to least significant (excluding the leading $1$), square at each step, and multiply by $x$ if the bit is $1$.

## 3. Counting multiplications

Let $\nu(n)$ denote the number of $1$'s in the binary expansion of $n$, and let $k = \lfloor \log_2 n \rfloor$. The binary method performs:

- $k$ squarings, one for each bit after the leading $1$;
- $\nu(n) - 1$ extra multiplications, one for each remaining $1$-bit.

Therefore the total number of multiplications is

$$
k + \nu(n) - 1 = \lfloor \log_2 n \rfloor + \nu(n) - 1.
$$

Since the modified power tree yields exactly the same addition chain, it requires the same number of multiplications.

## 4. Conclusion

The modified power tree, in which children are attached in decreasing order, produces the standard binary addition chain for every $n$. Consequently, it does not improve upon binary exponentiation. Its chain length is

$$
\boxed{\lfloor \log_2 n \rfloor + \nu(n) - 1},
$$

whereas the original power tree of Exercise 5 can yield shorter chains. Therefore, despite its similarity to the original power tree, the decreasing-order version is not as efficient.
