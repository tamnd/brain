**Exercise 5.** &#9654; [*24*] [24] Figure 14 shows the first eight levels of the "power tree." The $(k+1)$st level of this tree is defined as follows, assuming that the first $k$ levels have been constructed: Take each node $n$ of the $k$th level, from left to right in turn, and attach below it the nodes

$$n+1,\ n+a_1,\ n+a_2,\ \ldots,\ n+a_{k-1} = 2n$$

(in this order), where $1, a_1, a_2, \ldots, a_{k-1}$ is the path from the root of the tree to $n$; but discard any node that duplicates a number that has already appeared in the tree.

Design an efficient algorithm that constructs the first $r+1$ levels of the power tree. [*Hint:* Make use of two sets of variables LINK0[$j$], LINK1[$j$] for $0 \le j \le 2^r$; these point upwards and to the right, respectively, if $j$ is a number in the tree.]

## Solution

We are asked to construct the first $r+1$ levels of the "power tree" as defined in Figure 14 and in Exercise 4.6.3.5. We proceed by carefully translating the tree-construction rule into an efficient algorithm using the suggested LINK0 and LINK1 arrays.

Let the levels be numbered from $0$ to $r$, with level $0$ consisting of the root node $1$. Suppose we have already constructed levels $0,1,\dots,k$ and wish to construct level $k+1$. Each node $n$ at level $k$ has a _path_ from the root, consisting of the sequence $1, a_1, a_2, \dots, a_{k-1}, n$. The children of $n$ are defined as

$$
n+1,\ n+a_1,\ n+a_2,\ \dots,\ n+a_{k-1},\ 2n
$$

and any duplicate nodes are discarded.

We aim for an algorithm that avoids recomputing paths repeatedly. To this end, we store the path implicitly using two arrays, LINK0 and LINK1, which are pointers that allow reconstruction of the path from any node $n$ in the tree. Specifically, LINK0[$n$] points to the parent node along the vertical link (upwards), and LINK1[$n$] points to the node immediately to the left of $n$ at the same level, or zero if $n$ is the leftmost node.

### Invariants

1. At any stage, we maintain a set TREE of all numbers already included in the tree. This allows O(1) duplicate detection per candidate node.
2. For each node $n$, LINK0[$n$] points to its parent, LINK1[$n$] points to the previous node at its level. These arrays suffice to reconstruct the path from the root to $n$ by following LINK0 links.
3. The nodes are constructed level by level, from left to right, ensuring that the children are attached in the prescribed order.

### Algorithm

We describe the algorithm in pseudocode that uses only the suggested arrays:

```
Algorithm PowerTree(r)
Input: Integer r ≥ 0
Output: TREE[1..M] containing all nodes up to level r, LINK0[1..M], LINK1[1..M]

1. Initialize
    TREE ← {1}
    LINK0[1] ← 0
    LINK1[1] ← 0
    NEXT ← 2     // Next available index for TREE
    LEVEL_START[0] ← 1
    LEVEL_END[0] ← 1

2. For k ← 0 to r-1 do
       // Process level k
       i ← LEVEL_START[k]
       while i ≤ LEVEL_END[k] do
           n ← TREE[i]
           
           // Reconstruct path from root to n
           PATH ← empty list
           j ← n
           while j ≠ 1 do
               append j to PATH
               j ← LINK0[j]
           reverse PATH to get sequence from root to n
           
           // Generate children in order: n+1, n+a_1, ..., n+a_{k-1}, 2n
           CHILD_CANDIDATES ← [n+1]
           for t ← 2 to length(PATH) do
               CHILD_CANDIDATES.append(n + PATH[t])
           CHILD_CANDIDATES.append(2*n)
           
           // Insert non-duplicates
           for c in CHILD_CANDIDATES do
               if c not in TREE then
                   TREE[NEXT] ← c
                   LINK0[NEXT] ← n
                   LINK1[NEXT] ← TREE[NEXT-1] if NEXT > LEVEL_END[k] else 0
                   NEXT ← NEXT + 1
           i ← i + 1
       
       LEVEL_START[k+1] ← LEVEL_END[k] + 1
       LEVEL_END[k+1] ← NEXT - 1

3. Return TREE, LINK0, LINK1
```

### Justification

Step 1 initializes the tree with the root node $1$ and sets all pointers to zero. Step 2 iterates through levels $0$ to $r-1$, ensuring that each level is fully constructed before proceeding. For each node $n$ at level $k$, the path from the root to $n$ is reconstructed using LINK0 pointers. The candidate children are generated according to the rule

$$
n+1,\ n+a_1,\ n+a_2,\ \dots,\ n+a_{k-1},\ 2n
$$

where $a_1,\dots,a_{k-1}$ are the entries in the path from the root to $n$, excluding the root itself. Duplicate detection is performed by checking membership in TREE. Each new node is appended to TREE, and its LINK0 and LINK1 pointers are set appropriately. After processing all nodes in level $k$, LEVEL_START and LEVEL_END delimit the next level.

The algorithm ensures that:

1. Each node appears exactly once, satisfying the discard-duplicates rule.
2. Nodes are attached in left-to-right order, preserving the tree structure.
3. Reconstruction of paths requires at most $O(\text{level})$ steps per node; the total cost is proportional to the number of nodes in the first $r+1$ levels.

This completes the algorithm.

### Complexity

Let $N_r$ denote the number of nodes in the first $r+1$ levels. Each node is visited once, and for each node we reconstruct its path of length at most $r$. Each candidate child requires a membership check in TREE. Using a hash table or a balanced search tree, duplicate detection can be done in $O(\log N_r)$ or expected $O(1)$. Therefore, the algorithm runs in time $O(r \cdot N_r)$ using linear search or $O(N_r \log N_r)$ with efficient membership checks. Memory requirement is $O(N_r)$ for TREE, LINK0, LINK1, and LEVEL_START/LEVEL_END arrays.

### Example

For $r = 3$, level 0 contains ${1}$. The path from root to $1$ is empty. Children: $2, 2$ (duplicate discarded), $2$ (duplicate discarded). Level 1: ${2}$. Node $2$ path: ${1,2}$. Children: $3, 3$, $4$; duplicates discarded. Level 2: ${3,4}$, and so on. This reproduces the first four levels of the power tree exactly as shown in Figure 14.

The procedure generalizes efficiently for arbitrary $r$, producing the first $r+1$ levels with correct parent-child relationships and without duplicates.

This completes the solution.
