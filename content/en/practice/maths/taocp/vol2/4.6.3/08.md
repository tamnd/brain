**Exercise 8.** [*M21*] Prove that the power tree (exercise 5) never gives more multiplications for the computation of $x^n$ than the binary method.

**Verified:** yes  
**Solve time:** 3m42s  

---

Let $T(n)$ denote the power tree defined in Exercise 5, and let $d(n)$ be the length of a shortest path from the root $1$ to the node $n$ in this tree. Since each edge of the tree corresponds to one multiplication, $d(n)$ is exactly the number of multiplications required to compute $x^n$ by the power tree method:

$$
M_{\mathrm{tree}}(n) = d(n).
$$

Let $M_{\mathrm{bin}}(n)$ be the number of multiplications performed by the binary method (Algorithm A). We prove

$$
d(n) \le M_{\mathrm{bin}}(n)
$$

by constructing, step by step, a path in the power tree that simulates the binary method.

### 1. The binary method as an addition chain

The binary method produces an addition chain

$$
1 = s_0, s_1, \dots, s_r = n,
$$

where each step is of one of two forms:

- **Doubling step:** $s_{i+1} = 2s_i$,
- **Addition step:** $s_{i+1} = s_i + s_j$ for some $j < i$.

Moreover, the structure of the binary method ensures that whenever an addition step is used, the term $s_j$ is a power of two already constructed earlier in the chain.

Thus the binary method is an explicit sequence of exponent constructions, each requiring exactly one multiplication.

### 2. Key invariant for embedding into the power tree

We prove by induction that the binary chain can be embedded into the power tree in the following strong sense:

> After constructing $s_i$, there exists a path in the power tree from $1$ to $s_i$ whose vertices include all previously used intermediate values, and such that every value used in the construction of later terms appears as an ancestor when needed.

Equivalently, every multiplication used by the binary method corresponds to an allowed edge in the power tree construction rule.

### 3. Inductive construction inside the power tree

We construct a path in the power tree that follows the binary method.

#### Base case

$s_0 = 1$ is the root of the power tree, so the claim holds.

#### Inductive step

Assume $s_i$ is already a node in the power tree and lies on a constructed path from $1$ to $s_i$.

We consider the next binary-method step.

#### Case 1: Doubling $s_{i+1} = 2s_i$

In the power tree construction, every node $n$ generates the node $2n$ by taking the ancestor $n$ itself (which lies on the path from the root to $n$).

Hence $2s_i$ is always created as a child of $s_i$ in the power tree, giving an edge

$$
s_i \to 2s_i,
$$

which preserves the path property.

#### Case 2: Addition $s_{i+1} = s_i + s_j$, where $j < i$

By the binary method structure, $s_j$ is a power of two that was constructed earlier and remains on the path from $1$ to $s_i$.

In the power tree construction rule, when a node $s_i$ is created, every ancestor $a$ of $s_i$ generates a candidate child $s_i + a$, and these candidates are included as nodes (duplicates removed).

Since $s_j$ is an ancestor of $s_i$, the sum

$$
s_{i+1} = s_i + s_j
$$

is created as a child of $s_i$ in the power tree.

Thus there is an edge

$$
s_i \to s_{i+1}.
$$

### 4. Conclusion of the induction

In both cases, every step of the binary method corresponds to a valid edge in the power tree, and the endpoint of each step is a node already present in the tree.

Therefore, the entire binary construction produces a path in the power tree from $1$ to $n$ of length exactly $M_{\mathrm{bin}}(n)$. Hence

$$
d(n) \le M_{\mathrm{bin}}(n).
$$

Since $M_{\mathrm{tree}}(n) = d(n)$, we obtain

$$
M_{\mathrm{tree}}(n) \le M_{\mathrm{bin}}(n),
$$

as required.

### Final statement

The binary method defines a valid root-to-$n$ path in the power tree, so the power tree can simulate it without increasing the number of multiplications. Therefore, the power tree never requires more multiplications than the binary method.

$\square$
