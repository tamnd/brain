**Exercise 6.** [*M22*] Are either of the following two identities valid for all floating point numbers $u$? (a) $0 \oplus (0 \oplus u) = u$; (b) $1 \otimes (1 \otimes u) = u$.

**Verified:** yes  
**Solve time:** 1m21s  

---

## Solution

We consider each identity in turn, assuming that all operations are normalized floating point operations as defined in Section 4.2.2, and that no exponent underflow or overflow occurs.

(a) The identity is

$0 \oplus (0 \oplus u) = u.$

By equation (6), $0 \oplus x = x$ for any floating point number $x$. Applying this first to the inner expression gives

$0 \oplus u = u.$

Substituting this into the outer expression yields

$0 \oplus (0 \oplus u) = 0 \oplus u = u.$

Hence the identity holds exactly for all floating point numbers $u$. There are no exceptions since the operation $\oplus$ is well-defined for all normalized $u$, including $0$.

(b) The identity is

$1 \otimes (1 \otimes u) = u.$

By equation (13) and the properties of floating point multiplication in Section 4.2.2, we have

$1 \otimes u = \text{round}(1 \times u) = \text{round}(u).$

Thus the left-hand side becomes

$1 \otimes (1 \otimes u) = 1 \otimes \text{round}(u) = \text{round}(\text{round}(u)).$

In general, floating point numbers satisfy $\text{round}(x) = x$ if and only if $x$ is already a representable floating point number. Therefore, if $u$ is exactly a normalized floating point number, we have $\text{round}(u) = u$, and consequently $1 \otimes (1 \otimes u) = 1 \otimes u = u$.

However, if $u$ is not exactly representable in the floating point system, then $\text{round}(u) \ne u$, and a second rounding occurs. The result is

$1 \otimes (1 \otimes u) = \text{round}(\text{round}(u)) = \text{round}(u) \ne u.$

Hence the identity fails in general. It only holds for $u$ that are exactly representable in the floating point system, i.e., for normalized numbers with no rounding required.

This completes the proof.

∎

## Notes

The key distinction between the two identities is that addition with $0$ is exact and does not involve rounding, while multiplication with $1$ may invoke rounding if the operand is not exactly representable. This illustrates the general principle in Section 4.2.2 that some floating point operations preserve exactness for special values, while others introduce relative error.
