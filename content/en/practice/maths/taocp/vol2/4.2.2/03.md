[Section 4.2.2: Accuracy of Floating Point Arithmetic](../)

**Exercise 3.** [*M30*] Find eight-digit floating point numbers $u$, $v$, and $w$ such that

$$u \oplus (v \oplus w) \ne (u \oplus v) \oplus w,$$

and such that no exponent overflow or underflow occurs during the computations.

**Verified:** yes  
**Solve time:** 2m33s  

---

## Setup

All computations are performed in eight-digit floating decimal arithmetic with rounding to the nearest floating point number, as in Section 4.2.2. Each number is represented with eight significant digits, and every arithmetic operation $x \oplus y$ denotes $\mathrm{round}(x+y)$.

We seek $u$, $v$, $w$ such that

$u \oplus (v \oplus w) \ne (u \oplus v) \oplus w,$

with no overflow or underflow occurring in any intermediate step.

## Solution for the main argument

Let

$u = 11111111,\qquad v = -11111110,\qquad w = 0.0000001.$

All three values are normalized eight-digit floating point numbers.

First compute $v \oplus w$. The exact sum is

$v + w = -11111110 + 0.0000001 = -11111109.9999999.$

Rounding this value to eight significant digits gives

$v \oplus w = -11111110,$

since the next representable floating number at eight-digit precision is $-11111110$, and the correction $0.0000001$ lies far beyond the last retained digit.

Now evaluate the left-associated expression:

$u \oplus (v \oplus w) = 11111111 \oplus (-11111110) = 1.$

Next compute the right-associated expression. First,

$u \oplus v = 11111111 + (-11111110) = 1,$

and this value is exactly representable with eight digits, so no rounding change occurs.

Then

$(u \oplus v) \oplus w = 1 \oplus 0.0000001 = 1.0000001,$

and this value is also exactly representable in eight-digit floating point form.

Hence

$u \oplus (v \oplus w) = 1,\qquad (u \oplus v) \oplus w = 1.0000001,$

so

$u \oplus (v \oplus w) \ne (u \oplus v) \oplus w.$

This completes the construction.

## Verification to check the key steps independently

The critical step is the rounding of $v + w$. Writing

$v + w = -11111109.9999999,$

the magnitude lies strictly between $11111109$ and $11111110$, and with eight-digit rounding the unit in the last place at this scale is $1$. The fractional perturbation $0.9999999 \cdot 10^{-7}$ is insufficient to change the eighth significant digit, so the rounded value is $-11111110$.

Both subsequent additions,

$11111111 + (-11111110),\qquad 1 + 0.0000001,$

produce exact eight-digit representable numbers, so no further rounding alters either result. The two evaluation orders therefore diverge solely due to the loss of the $10^{-7}$ contribution in $v \oplus w$.

## Notes

The failure arises because floating point addition replaces exact arithmetic by rounding after each binary operation, so intermediate cancellation can eliminate low-order information before later terms are incorporated. This example isolates that effect: associativity fails even though no single operation is individually extreme, and the discrepancy is introduced only through a single rounding step in $v \oplus w$.
