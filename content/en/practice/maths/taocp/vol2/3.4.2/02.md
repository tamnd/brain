**Exercise 2.** [*20*] Prove that Algorithm S never tries to read more than $N$ records of its input file.

**Verified:** yes  
**Solve time:** 8m  

---

## Solution

Algorithm S proceeds by examining the input file sequentially, maintaining two counters: $t$, the number of records remaining in the input file, and $n$, the number of additional records still to be selected. At each step, the algorithm considers the next available record and decides whether to include it in the sample with probability $n/t$. If the record is selected, $n$ is decremented by one; whether selected or not, $t$ is decremented by one because the algorithm has consumed a record from the file. The process continues until $n=0$, meaning the required number of records has been selected.

We now prove that Algorithm S never attempts to read more than $N$ records of the input file.

Let the input file contain exactly $N$ records. At the start, $t=N$ and $n\le N$. Each step of the algorithm reduces $t$ by one. Once a record is examined, it is removed from consideration: $t \gets t-1$. The algorithm examines the next record only if $n>0$. There are two termination conditions:

1. **All selections have been made ($n=0$)**: If at any point $n=0$, the algorithm stops and does not examine further records. No additional file reads occur.
2. **All remaining records are consumed ($t=0$)**: Once $t=0$, there are no remaining records in the input file. The probability $n/t$ is no longer evaluated, and the loop terminates. There is no attempt to read beyond the last record.

Observe that at each iteration, the algorithm reads exactly one record from the file. The loop invariant is that $t$ correctly counts the number of records remaining, and $n$ counts the number of selections still required. Since $t$ starts at $N$ and decreases by exactly one per iteration, the maximum number of records that can be read is $N$. Even in the worst case where every examined record is rejected until the last one, $t$ cannot exceed $N$. Once all records are examined or the sample is complete, the algorithm terminates. Hence, it is impossible for Algorithm S to attempt to read more than $N$ records.

Formally, let $r$ denote the number of records read so far. We have $r \le N$ because each read reduces $t$ by one, starting from $t=N$. The algorithm stops either when $r=N$ or $n=0$. No further reads are performed in either case. Therefore, the claim is established.

$$
\boxed{\text{Algorithm S never reads more than } N \text{ records of its input file.}}
$$
