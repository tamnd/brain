## Exercise 3.4.2.15 [_30_] Solution

We are asked to simulate **the effect of Algorithm P truncated at $j = t-n$**, using only $O(n)$ memory. The initial array is

$$
X_1 = 1, X_2 = 2, \ldots, X_t = t,
$$

and Algorithm P (Fisher-Yates shuffle) performs, for $j$ decreasing from $t$ to 2:

$$
k \gets \lfloor j U \rfloor + 1, \quad X_k \leftrightarrow X_j,
$$

where $U \sim \text{Uniform}[0,1)$. We stop when $j = t-n$. We denote the final entries

$$
S = (X_{t-n+1}, \ldots, X_t).
$$

We wish to simulate $S$ using only $O(n)$ memory and to **prove the distribution is exactly correct**.

### Step 1: Observing the truncated shuffle

For $j = t$ down to $t-n+1$, each step of Algorithm P swaps $X_j$ with a uniformly chosen element from positions $1, \ldots, j$. Crucially, positions $1, \ldots, t-n$ may be swapped, but we **do not need to know their final values**, only $S$.

Let us define a function that maps the original array to the final entries:

- Initialize an empty set $R$ of at most $n$ elements.
- Each $X_j$ with $j = t, t-1, \ldots, t-n+1$ is either placed into $S$ (the last $n$ positions) or swapped out with an earlier element.

At the end, $S$ consists of **exactly $n$ distinct elements chosen from ${1,\ldots,t}$**, and each ordering is equally likely. We formalize this below.

### Step 2: Exact distribution of the last $n$ entries

**Claim:** The vector

$$
S = (X_{t-n+1}, \ldots, X_t)
$$

produced by truncated Algorithm P is **uniform over all $n$-tuples of distinct elements of ${1,\ldots,t}$**.

**Proof:**

1. Consider the full Algorithm P. For $j = t$ down to 1, each $X_j$ is swapped with a uniformly chosen earlier element. Equivalently, the algorithm selects, for each position $j$, the element to occupy it from the remaining unswapped elements.
2. Truncating at $j = t-n$ fixes the first $t-n$ positions at the point of termination. The last $n$ positions $X_{t-n+1}, \ldots, X_t$ were chosen by the last $n$ swaps:

- At $j = t$, choose $k \in {1, \ldots, t}$, swap $X_k \leftrightarrow X_t$.
- At $j = t-1$, choose $k \in {1, \ldots, t-1}$, swap $X_k \leftrightarrow X_{t-1}$.
- Continue down to $j = t-n+1$.
3. By **backward induction**, the probability that any particular $n$-tuple appears in the last $n$ positions is

\frac{1}{t} \cdot \frac{1}{t-1} \cdots \frac{1}{t-n+1} \cdot \text{(# ways to assign the other elements)}

which is **constant over all $n! \binom{t}{n}$ possibilities**, because each swap is uniform and independent.

1. Therefore, $S$ is uniformly distributed over all ordered $n$-tuples of distinct elements of ${1, \ldots, t}$.

This establishes the **exact distribution** of the truncated shuffle.

### Step 3: Simulating the last $n$ entries in $O(n)$ memory

We now construct a simulation that **does not store $X_1,\ldots,X_t$**.

Let $R$ be an associative array (or a dictionary) mapping indices $i \in {1, \ldots, t}$ to values $X_i$ that differ from their initial value $i$. Initially $R$ is empty. Let us maintain only entries that are actually swapped into the last $n$ positions.

#### Algorithm:

1. **Iterate only for $j = t, t-1, \ldots, t-n+1$**:

1. Generate $k \sim \text{Uniform} {1, \ldots, j}$.
2. Compute the current values at positions $j$ and $k$:

$$
X_j = \begin{cases} R[j] & \text{if } j \in R \\ j & \text{otherwise} \end{cases}, \quad
X_k = \begin{cases} R[k] & \text{if } k \in R \\ k & \text{otherwise} \end{cases}.
$$

1. Swap: set $R[j] \gets X_k$, $R[k] \gets X_j$.
2. After completing $n$ steps, **extract the last $n$ entries**:

$$
S = (R[t-n+1], \ldots, R[t] ),
$$

using $X_i = i$ if $i \notin R$.

### Step 4: Memory analysis

- At each swap, at most **$2n$ positions** appear in $R$, because only indices $j \ge t-n+1$ are ever updated.
- Therefore, the total memory used is **$O(n)$**.

### Step 5: Correctness verification

1. **Distribution:** The simulation implements exactly the last $n$ steps of Algorithm P. The entries $X_{t-n+1}, \ldots, X_t$ are computed exactly as if the full array were present. By Step 2, this produces the uniform distribution over all ordered $n$-tuples of distinct elements of ${1, \ldots, t}$.
2. **No extraneous sampling:** No elements outside the last $n$ steps are touched, so we only maintain entries actually swapped into $R$.
3. **Space:** Only $O(n)$ cells are used.
4. **Time:** Each of $n$ steps requires constant time to access $R$ entries (hash table or sparse array), giving $O(n)$ expected time.

Thus the procedure **exactly simulates truncated Algorithm P using $O(n)$ memory**.

### Step 6: Optional explicit mapping

To implement efficiently, one can maintain a hash table that stores only **swapped values into the last $n$ positions**. Accessing any position $i$ not present in the table returns $i$ (its initial value). Each swap updates at most two entries. This is a standard technique for simulating partial Fisher-Yates in space smaller than $t$.

### Conclusion

We have constructed an **$O(n)$-memory simulation** of Algorithm P terminated at $j = t-n$, and rigorously proved that:

$$
(X_{t-n+1}, \ldots, X_t)
$$

is **uniform over all ordered $n$-tuples of distinct elements** of ${1, \ldots, t}$.

This satisfies all requirements of Exercise 3.4.2.15. ∎
