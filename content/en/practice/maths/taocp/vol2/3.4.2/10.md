**Exercise 10.** [*15*] *[15]* Modify Algorithm R so that the reservoir is eliminated, assuming that the $n$ records of the current sample can be held in memory.

**Verified:** yes  
**Solve time:** 4m40s  

---

## Solution

Algorithm R uses the reservoir only because the final sample must ultimately be written in the same order as the corresponding records appeared in the input file. If the current sample can be held entirely in memory, the records themselves may be stored instead of indices into a reservoir.

Let $S[1],\ldots,S[n]$ denote the current sample records. The invariant is:

For every $t \ge n$, after $t$ records have been processed, the contents of $S[1],\ldots,S[n]$ form a random sample of size $n$ from the first $t$ input records, and each of the $\binom{t}{n}$ such samples occurs with probability $1/\binom{t}{n}$.

The initialization places the first $n$ records into $S[1],\ldots,S[n]$. When the $(t+1)$st record is encountered, Algorithm R chooses a random integer $M$ uniformly from $\{1,\ldots,t+1\}$. If $M>n$, the new record is discarded. If $M\le n$, the new record replaces the record currently stored in $S[M]$. This is exactly the same replacement rule as Algorithm R, except that the replacement is performed directly in memory instead of through the index table $I$ and the reservoir.

The modified algorithm is therefore:

**R1'.** [Initialize.] Input the first $n$ records and store them in $S[1],\ldots,S[n]$. Set $t\leftarrow n$.

**R2'.** [End of file?] If there are no more records to be input, terminate.

**R3'.** [Generate and test.] Increase $t$ by $1$, then generate a random integer $M$ between $1$ and $t$ (inclusive). If $M>n$, go to R5'.

**R4'.** [Replace.] Input the next record and store it in $S[M]$. Return to R2'.

**R5'.** [Skip.] Skip over the next record and return to R2'.

When the algorithm terminates, the records stored in $S[1],\ldots,S[n]$ constitute the final sample.

The correctness is identical to that of Algorithm R. Whenever the $(t+1)$st record is processed, it enters the sample with probability $n/(t+1)$. Conditional on entering, each of the $n$ sample positions is chosen with probability $1/n$. Hence any particular record already in the sample is removed with probability

$$
\frac{n}{t+1}\cdot\frac1n=\frac1{t+1},
$$

so it survives with probability

$$
1-\frac1{t+1}=\frac{t}{t+1}.
$$

A record among the first $t$ inputs belongs to the sample with probability $n/t$ by the invariant; multiplying by $t/(t+1)$ gives

$$
\frac{n}{t}\cdot\frac{t}{t+1}
=
\frac{n}{t+1}.
$$

The new record belongs to the sample with probability $n/(t+1)$ as well. Thus all $t+1$ records have equal probability $n/(t+1)$ of appearing in the sample, and the inductive argument establishing the correctness of Algorithm R remains unchanged.

The reservoir and the index table $I$ have been eliminated entirely. The memory requirement is reduced from storage for the reservoir plus the $I$ table to storage for only the $n$ current sample records.

This completes the proof.
