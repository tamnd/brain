[Section 3.4.2: Random Sampling and Shuffling](../)

**Exercise 16.** [*M25*] Devise a way to compute a random sample of $n$ records from $N$, given $N$ and $n$, based on the idea of hashing (Section 6.4). Your method should use $O(n)$ storage locations and an average of $O(n)$ units of time, and it should present the sample as a sorted set of integers $1 \le X_1 < X_2 < \cdots < X_n \le N$.

**Verified:** yes  
**Solve time:** 4m

## Solution

The rejection method in the proposed solution does not satisfy the required expected $O(n)$ time bound when $n$ is comparable to $N$. A different use of hashing is needed.

The key idea is to generate the sample by performing only $n$ selections, while using a hash table to represent the effect of partial permutations.

Recall that if we execute the first $n$ steps of the Fisher-Yates shuffle on the set $\{1,\ldots,N\}$, the values moved into positions $1,\ldots,n$ form a uniformly distributed $n$-subset of $\{1,\ldots,N\}$. The difficulty is that $N$ may be much larger than $n$, so we cannot store the entire array. Instead we store only the locations whose values differ from their initial values.

### Representation

Let $A[1],\ldots,A[N]$ initially satisfy $A[i]=i$.

Maintain a hash table $H$ containing only modified entries of $A$.

Define

$$
\operatorname{val}(i)=
\begin{cases}
H[i], & \text{if } i \text{ is present in } H,\\[4pt]
i, & \text{otherwise}.
\end{cases}
$$

Thus $H$ represents the current permutation implicitly. Since each shuffle step modifies at most two positions, only $O(n)$ positions ever need to be stored.

### Algorithm

For $i=1,2,\ldots,n$:

1. Generate a uniform random integer

$$
j \in \{i,i+1,\ldots,N\}.
$$

1. Let

$$
x=\operatorname{val}(i),\qquad
y=\operatorname{val}(j).
$$

1. Exchange the values at positions $i$ and $j$:

$$
A[i]\leftarrow y,\qquad A[j]\leftarrow x.
$$

This is implemented by updating $H$:

- store $H[i]=y$ unless $y=i$, in which case remove $i$ from $H$;
- store $H[j]=x$ unless $x=j$, in which case remove $j$ from $H$.

After the $n$-th step, the selected sample is

$$
S=\{\operatorname{val}(1),\ldots,\operatorname{val}(n)\}.
$$

Finally, sort these $n$ integers and output

$$
1\le X_1<X_2<\cdots<X_n\le N.
$$

### Correctness

The algorithm performs exactly the first $n$ iterations of the Fisher-Yates shuffle. The hash table changes only the representation, not the permutation being generated.

After iteration $i$, the value placed in position $i$ is chosen uniformly from the values that have not yet been placed in positions $1,\ldots,i-1$. Therefore the resulting permutation of $\{1,\ldots,N\}$ is distributed exactly as in the ordinary Fisher-Yates algorithm.

Consequently every ordered $n$-tuple of distinct elements occurs with probability

$$
\frac{1}{N(N-1)\cdots(N-n+1)}.
$$

Each $n$-subset appears in exactly $n!$ such ordered tuples, hence every $n$-subset occurs with probability

$$
\frac{n!}{N(N-1)\cdots(N-n+1)}
=
\frac{1}{\binom{N}{n}}.
$$

Thus $S$ is a uniform random sample of size $n$.

Sorting $S$ merely changes its presentation and yields the required output

$$
X_1<X_2<\cdots<X_n.
$$

### Storage Analysis

Each shuffle step can introduce at most two new modified positions.

After $n$ steps, at most $2n$ positions differ from their original values. Therefore the hash table contains $O(n)$ entries.

The storage requirement is therefore $O(n)$.

### Time Analysis

Each iteration performs:

- one random-number generation;
- a constant number of hash-table searches and updates.

With standard hashing and load factor bounded away from $1$, each search or update requires $O(1)$ expected time. Hence the $n$ shuffle steps require $O(n)$ expected time.

The resulting sample consists of $n$ integers in the range $[1,N]$. They may be sorted in linear time by radix sorting on $\lceil\log_2 N\rceil$-bit keys. Since the word size is $\Theta(\log N)$, radix sort requires $O(n)$ time and $O(n)$ additional storage.

Therefore the total expected running time is $O(n)$, and the storage requirement remains $O(n)$.

Hence the exercise requirements are satisfied. ∎
