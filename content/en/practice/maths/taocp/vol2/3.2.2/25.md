**Exercise 25.** [*26*] $[26]$ Discuss the alternative to Program A that changes all 55 entries of the $Y$ table every 55th time a random number is required.

**Verified:** no

## Solution

In Program A of Section 3.2.2, the table

$$
Y[1],Y[2],\ldots ,Y[55]
$$

is updated one position at a time. If the current position is $k$, the program computes

$$
Y[k]\leftarrow (Y[k]+Y[k+31])\pmod{2^e},
$$

with subscripts taken modulo $55$, and the newly computed value is output. After $55$ calls every entry has been updated once.

The exercise asks about a different generator in which no entry is altered until $55$ numbers are needed. At that moment all $55$ entries are replaced simultaneously by

$$
Y_i'=(Y_i+Y_{i+31})\pmod{2^e},
\qquad 1\le i\le55,
$$

where every right-hand side uses the old table.

This generator is not equivalent to Program A. In Program A, later updates in a cycle use some values that were already changed earlier in the same cycle. Hence one complete round of $55$ sequential updates corresponds to a certain linear transformation $S$, whereas the simultaneous replacement above corresponds to another linear transformation $P$. In general,

$$
S\neq P.
$$

Therefore the resulting state after $55$ outputs is different, and the generated sequence itself is different.

The alternative generator is nevertheless easy to analyze. If the state vector is

$$
Y=(Y_1,\ldots ,Y_{55}),
$$

then one block transformation is

$$
Y'=(I+R^{31})Y,
$$

where $R$ denotes cyclic rotation by one place. Since $R^{55}=I$, the eigenvalues of $R$ are the $55$-th roots of unity $\omega^j$, and the eigenvalues of the block transformation are

$$
1+\omega^{31j},
\qquad 0\le j<55.
$$

Because $55$ is odd, no $55$-th root of unity equals $-1$. Hence none of these eigenvalues vanishes, and the transformation is nonsingular. Thus every state has a unique predecessor, and the nonzero states are permuted among themselves.

The amount of computation is reduced. During a block of $55$ outputs only $55$ additions are performed, namely one addition per table entry. Thus the arithmetic cost remains one addition per generated number on average. Moreover, all $55$ additions are independent, since each uses only old table values. Consequently they can be carried out simultaneously, which makes this scheme attractive on machines possessing parallel arithmetic units.

On the other hand, the sequence produced is different from that of Program A, and there is no reason to expect exactly the same period or the same statistical behavior. The alternative generator should therefore be regarded as a distinct additive generator whose main advantage is that the $55$ additions required for a block can be performed in parallel.

$\square$
