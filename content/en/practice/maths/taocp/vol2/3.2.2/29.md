**Exercise 3.2.2.29** [_40_]

**Problem:** Find a good way to compute the function $f(x_1,\dots,x_k)$ defined by Martin's sequence in Exercise 17, given only the $k$-tuple $(x_1,\dots,x_k)$.

### Solution

Let the alphabet size be $m$, i.e., $\{0,1,\dots,m-1\}$. The sequence defined in Exercise 17 is the **prefer-max de Bruijn sequence of order $k$** over this alphabet. Let

$$
f(x_1,\dots,x_k) = x_{k+1}
$$

be the function giving the next symbol in the sequence. The goal is to compute $f(w)$ **directly from the tuple** $w = x_1\cdots x_k$, without generating the whole sequence.

### Step 1: Characterize the successor

Martin's construction is the **prefer-max algorithm**, which extends a current word $w$ by the **largest possible symbol** that does not produce a repeated $k$-tuple. Knuth shows that this is equivalent to the **necklace-head characterization**.

Let $v \neq (m-1)^k$ be a $k$-word. Write

$$
v = (m-1)^t u \sigma, \qquad \sigma < m-1,
$$

where $t$ is maximal. Then $v$ is called a **necklace head** if

$$
u\sigma (m-1)^t
$$

is the **lexicographically smallest rotation** of $v$, that is, the **expanded Lyndon word** in its necklace. Every word except $(m-1)^k$ has exactly one necklace head.

The prefer-max successor $f(w)$ is then:

$$
f(w) =
\begin{cases}
\sigma + 1, & \text{if } \sigma < m-1 \text{ and } z\sigma \text{ is a necklace head},\\[1mm]
\min\{\tau < m-1 : z\tau \text{ is a necklace head}\}, & \text{if } \sigma = m-1 \text{ and such a } \tau \text{ exists},\\[1mm]
m-1, & \text{if } w = (m-1)^k,
\end{cases}
$$

where $z = x_2\cdots x_k$ and $\sigma = x_1$.

This gives a theoretical characterization, but to compute $f$ **directly from the tuple**, we must describe how to detect necklace heads.

### Step 2: Efficient detection of necklace heads

Let $v = v_1\cdots v_k$ be a $k$-word. A word $v$ is a **necklace head** if the following condition holds:

$$
v \le \rho^i(v) \quad \text{for all rotations } \rho^i(v) = v_{i+1}\cdots v_k v_1\cdots v_i, \quad 1 \le i < k.
$$

Equivalently, $v$ is **lexicographically minimal among its rotations**.

To check this efficiently without enumerating all rotations, use the **Duval algorithm** for Lyndon factorization:

1. Let $i = 1$, $j = 2$, and $k_0 = 0$.
2. Compare $v_i$ with $v_j$.

- If $v_i = v_j$, increment $i$ and $j$.
- If $v_i < v_j$, set $j = j + 1$, $i = 1$.
- If $v_i > v_j$, set $i = 1$, $k_0 = j - i$, $j = k_0 + 1$.
3. If $k_0 = 0$ at the end, the word is lexicographically minimal.

This algorithm runs in **linear time $O(k)$** and uses **constant memory** if implemented carefully. It allows testing whether $z\tau$ is a necklace head for a given symbol $\tau$.

### Step 3: Computation of $f(w)$

Given $w = x_1\cdots x_k$:

1. Let $z = x_2\cdots x_k$ and $\sigma = x_1$.
2. If $\sigma < m-1$:

- Check if $z\sigma$ is a necklace head.
- If yes, return $f(w) = \sigma + 1$.
3. If $\sigma = m-1$:

- Find the **smallest $\tau < m-1$** such that $z\tau$ is a necklace head.
- If such $\tau$ exists, return $f(w) = \tau$.
4. If no such $\tau$ exists, $w = (m-1)^k$ and return $f(w) = m-1$.

This procedure **requires only the $k$-tuple $w$** and does **not generate the entire sequence**. Using the linear-time test for necklace heads, the successor function can be computed in $O(k m)$ time in the worst case (checking up to $m$ candidates), but often faster in practice.

### Step 4: Correctness justification

1. Every $k$-tuple except $(m-1)^k$ has a unique predecessor in the prefer-max sequence, and the above procedure follows the **prefer-max rule**.
2. The "otherwise" case corresponds exactly to $w = (m-1)^k$, which is the **last tuple in the sequence**, hence the successor is $m-1$.
3. The necklace-head check guarantees that the chosen next symbol does not create a repeated $k$-tuple, satisfying the **de Bruijn property**.

### Step 5: Complexity

- Testing a single candidate $\tau$ for a necklace head takes $O(k)$.
- In the worst case, we may check all $\tau < m-1$, giving $O(k m)$.
- Space complexity is $O(k)$ for storing $w$ and temporary comparisons.
- No sequence reconstruction is required.

### Conclusion

The function $f(x_1,\dots,x_k)$ can be computed **directly from the tuple** using the above procedure:

1. Extract $\sigma = x_1$ and $z = x_2\cdots x_k$.
2. Use the linear-time **necklace head test** to determine the correct next symbol.
3. Handle the last tuple $(m-1)^k$ separately.

This gives a **practical, efficient, and fully justified method** to compute $f$ from the $k$-tuple alone.

This completes the solution.
