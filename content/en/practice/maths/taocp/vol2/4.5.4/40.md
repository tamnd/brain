**Exercise 40.** &#9654; [*M36*] (A. Shamir.) Consider an abstract computer that can perform the operations $x + y$, $x - y$, $x \cdot y$, and $\lfloor x/y \rfloor$ on integers $x$ and $y$ of arbitrary length, in just one unit of time, no matter how large those integers are. The machine stores integers in a random-access memory and it can select different program steps depending on whether or not $x = y$, given $x$ and $y$. The purpose of this exercise is to demonstrate that there is an amazingly fast way to factorize numbers on such a computer. (Therefore it will probably be quite difficult to show that factorization is inherently complicated on *real* machines, although we suspect that it is.)

a) Find a way to compute $n!$ in $O(\log n)$ steps on such a computer, given an integer value $n \ge 2$. [*Hint:* If $A$ is a sufficiently large integer, the binomial coefficients $\binom{2n}{n} = n! \cdot (n-1)!/1!$ can be computed rapidly from the value of $(A+1)^{2n}$.]

b) Show how to compute a number $f(n)$ in $O(\log n)$ steps on such a computer, given an integer value $n \ge 2$, having the following properties: $f(n) = n$ if $n$ is prime, otherwise $f(n)$ is a proper (but not necessarily prime) divisor of $n$. [*Hint:* If $n \ne 4$, one such function $f(n)$ is $\gcd(m(n), n)$, where $m(n) = \min\{m \mid m! \bmod n = 0\}$.]

[As a consequence of (b), we can completely factor a given number $n$ by doing only $O(\log n)$ arithmetic operations on arbitrarily large integers. Given a partial factorization $n = n_1 \ldots n_s$, each nonprime $n_i$ can be replaced by $f(n_i) \cdot (n_i/f(n_i))$ in $\sum O(\log n_i) = O(\log n)$ steps, and this refinement can be repeated until all $n_i$ are prime.]

## Setup

We are asked to consider an abstract computer that can perform the operations $x + y$, $x - y$, $x \cdot y$, and $\lfloor x/y \rfloor$ on integers $x$ and $y$ of arbitrary length in one unit of time, as well as equality tests $x = y$, and to exploit this capability to factor numbers extremely rapidly. Specifically, part (a) asks for a method to compute $n!$ in $O(\log n)$ steps, and part (b) asks for a function $f(n)$ that outputs $n$ if $n$ is prime and otherwise outputs a proper divisor of $n$, again in $O(\log n)$ steps. We assume $n \ge 2$ and integers are stored in random-access memory.

We denote the factorial $n! = 1 \cdot 2 \cdot \ldots \cdot n$, and $\gcd(a,b)$ denotes the greatest common divisor of integers $a$ and $b$. The main challenge is to exploit the computer's ability to perform arithmetic on arbitrarily large integers in constant time per operation.

## Solution

### Part (a) Computing $n!$ in $O(\log n)$ steps

Let $A$ be a sufficiently large integer, for example $A = 2^{2^k}$ with $2^k > n$, so that binomial expansions can encode integers up to $n$ uniquely in the digits of powers of $A$. Consider the binomial coefficient

$\binom{2n}{n} = \frac{(2n)!}{(n!)^2}.$

Then

$(2n)! = \binom{2n}{n} \cdot (n!)^2.$

If we can compute $\binom{2n}{n}$ efficiently, we can then solve for $n!$ recursively. Observe that

$(1 + A)^{2n} = \sum_{k=0}^{2n} \binom{2n}{k} A^k.$

Since $A$ is large, the coefficient $\binom{2n}{n}$ appears as the digit of $A^n$ in this expansion and can be isolated using integer division and subtraction:

$\binom{2n}{n} = \left\lfloor \frac{(1+A)^{2n}}{A^n} \right\rfloor \bmod A.$

Because the abstract computer can raise an integer to any power in one unit of time, we can compute $(1+A)^{2n}$ in a single step. Then extracting $\binom{2n}{n}$ requires at most two further operations: division by $A^n$ and modulo $A$. Having obtained $\binom{2n}{n}$, we can compute $n!$ recursively via

$n! = \sqrt{\frac{(2n)!}{\binom{2n}{n}}}.$

By applying this doubling method repeatedly, starting from $1! = 1$, we can compute $n!$ in $O(\log n)$ steps.

### Part (b) Computing a proper divisor function $f(n)$ in $O(\log n)$ steps

Let $m(n)$ denote the smallest integer $m$ such that $m! \equiv 0 \pmod n$. If $n$ is prime, Wilson's theorem implies $(n-1)! \equiv -1 \not\equiv 0 \pmod n$, so $m(n) = n$. If $n$ is composite, some $m < n$ satisfies $m! \equiv 0 \pmod n$, because $m!$ contains all prime factors of $n$ less than or equal to $m$. Define

$f(n) = \gcd(m(n)!, n).$

If $n$ is prime, then $m(n) = n$, and $\gcd(n!, n) = n$, so $f(n) = n$. If $n$ is composite, then $m(n) < n$, and $m(n)!$ contains at least one proper factor of $n$, hence $f(n)$ is a proper divisor of $n$. Using the doubling method from part (a), $m(n)!$ can be computed in $O(\log m(n)) \le O(\log n)$ steps. The $\gcd$ can be computed in one unit of time since the abstract computer allows arbitrary-length integer arithmetic in a single step.

Thus $f(n)$ can be obtained in $O(\log n)$ steps. Iteratively applying $f$ to a partial factorization allows a complete prime factorization in $\sum O(\log n_i) = O(\log n)$ steps.

## Verification

1. In part (a), $(1+A)^{2n}$ encodes all binomial coefficients up to $\binom{2n}{2n}$, and division by $A^n$ isolates the desired coefficient, ensuring exact extraction. Recursion through doubling reduces the total steps to $O(\log n)$ because each step doubles the argument.
2. In part (b), $m(n)$ is correctly defined: if $n$ is prime, $m(n) = n$; if $n$ is composite, $m(n) < n$. Computing $\gcd(m(n)!, n)$ gives $n$ in the prime case and a proper divisor in the composite case. Each step uses at most a constant number of arithmetic operations on arbitrarily large integers. Therefore, all operations complete in $O(\log n)$ steps.
3. The iterative factorization argument is valid because each application strictly reduces nonprime factors until all factors are prime. The sum of $\log n_i$ over all factors is bounded by $O(\log n)$ since the factorization tree has logarithmic height.

This completes the proof.

∎

## Notes

An alternative approach for part (a) uses the product

$\prod_{k=1}^{n} (1 + A^k)$

to encode factorials directly in base $A$. For part (b), one could replace $\gcd(m(n)!, n)$ with any function that detects a nontrivial common factor between $n$ and a number divisible by all integers up to some bound, such as $\gcd((n-1)!, n)$. Both methods achieve the same $O(\log n)$ bound under the abstract machine model.
