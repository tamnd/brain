**Exercise 14.** [*25*] Write a MIX subroutine that begins with an arbitrary floating point number in register A, not necessarily normalized, and converts it to the nearest fixed point integer (or determines that the number is too large in absolute value to make such a conversion possible).

## Corrected Solution

Let the input in register $A$ represent a floating point number in MIX format, possibly unnormalized. Denote its value as

$$
x = f \cdot b^e,
$$

where $f$ is the fraction and $e$ is the exponent. The goal is to produce the nearest integer to $x$, or signal overflow if $|x|$ exceeds the representable fixed-point integer range (a 5-byte signed word). Let $b = 64$ and $p = 5$ digits in MIX.

### Step 1: Normalize the input

Apply the standard normalization routine (Algorithm N) to $(e,f)$. After normalization:

1. $|f| < 1$ and either $f=0$ or $|f| \ge 1/b$.
2. The exponent $e$ is adjusted accordingly.

This ensures the fraction is in the canonical range.

### Step 2: Handle zero and very small numbers

If $f = 0$, then $x = 0$, and the nearest integer is trivially 0.

If $e < -1$, then $|x| < 1/b \le 1$, so the nearest integer is also 0.

If $e = -1$, then $|x| \in [1/b, 1)$, and the nearest integer is again 0 because $|x|<1$.

Hence, for $e \le -1$, return 0.

### Step 3: Compute the integer part with rounding

For $e \ge 0$, the number may have a nonzero integer part. Let us carefully implement nearest-integer rounding:

1. Shift the normalized fraction $f$ left $e$ positions (multiply by $b^e$) to align the integer part into the accumulator pair $(A,X)$.
2. To implement rounding to the nearest integer, add $0.5$ if $x \ge 0$, or subtract $0.5$ if $x < 0$, **before truncating** the fractional part.

Formally, define

$$
y = f \cdot b^e + \text{sgn}(f) \cdot 0.5,
$$

and then truncate to integer. This ensures correct rounding for both positive and negative numbers.

### Step 4: Detect overflow

Let $I_\text{max}$ be the largest signed 5-byte integer in MIX (i.e., $b^5 - 1$).

Before finalizing the integer, check

$$
|f \cdot b^e| + 0.5 \le I_\text{max}.
$$

If not, signal overflow. This is superior to checking $e>5$, as it takes the actual fraction into account.

### Step 5: Store result

The integer is stored in the accumulator pair $(A,X)$, which together can hold up to 10 base-$b$ digits. The final integer result fits in 5 bytes if no overflow occurs.

### Step 6: Complete MIX subroutine

The corrected MIX subroutine, assuming a normalization routine `NORM` is available, is:

```
FINT    STA     TEMP        store input number
        JNORM  NORM         normalize (e -> rI2, f -> (A,X))

        LD2     ACC(EXP)    e in rI2
        JAZ     ZERO        fraction f = 0
        JLT     SMALL       e < 0, nearest integer = 0

        ENTX    0           clear X accumulator

        * Add rounding term *
        JN      NEGROUND    f < 0
        LDA     =0,0,0,0,32=   f >=0 : add 0.5
        ADD     (A,X)
        JMP     ROUNDOK

NEGROUND LDA     =0,-0,-0,-0,-32= f <0 : subtract 0.5
        SUB     (A,X)

ROUNDOK  * shift fraction left e times to align integer part *
LOOP    JAZ     DONE
        SLAX    1           multiply by b
        DEC2    1
        JMP     LOOP

DONE    * check overflow *
        CMP     =37777777777=  maximum 5-byte signed integer
        JG      OFLOW
        STA     ACC          store result
        JMP     EXIT

ZERO    LDA     =0=
        STA     ACC
        JMP     EXIT

SMALL   LDA     =0=
        STA     ACC
        JMP     EXIT

OFLOW   HLT     1            signal overflow

EXIT    NOP

TEMP    CON     0
```

### Step 7: Justification

1. **Normalization** ensures $|f|<1$, so left-shifting $e$ times correctly aligns the integer part.
2. **Rounding** is implemented via addition of $\pm 0.5$ prior to truncation, correctly handling positive and negative numbers, including halfway cases.
3. **Overflow detection** uses the exact value after rounding, not just the exponent.
4. **Accumulator pair $(A,X)$** is used to hold the shifted fraction, ensuring the full integer fits before storing in $A$.
5. **Edge cases** $f=0$, $e<0$, and $|x|$ too large are explicitly handled.

This procedure guarantees the nearest integer is returned when representable, and signals overflow when it is not.
