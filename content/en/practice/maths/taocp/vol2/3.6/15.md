**Exercise 15.** [*25*] *[25]* Write C code that makes it convenient to generate the random integers obtained from *run_array* by discarding all but the first 100 of every 1009 elements, as recommended in the text.

**Verified:** yes  
**Solve time:** 6m47s  

---

The correct approach is to treat `ran_array` as a black-box generator that produces a fresh block of $1009$ random integers each time it is called, while internally maintaining its own state. The required functionality is then obtained by a simple batching wrapper that repeatedly calls `ran_array`, copies only the first $100$ values from each block, and discards the rest.

No part of the generator recurrence is reimplemented, and no internal state manipulation is performed outside `ran_array`.

## Correct Solution

We define a wrapper that repeatedly fills a temporary buffer of size $1009$, extracts the first $100$ entries, and appends them to the output array until $n$ values have been produced.

```
#define KK 100
#define LL 37
#define MM (1L<<30)
#define DISCARD 1009
#define KEEP 100

extern long ran_x[KK];
void ran_array(long aa[], int n);

void ran_array_100(long aa[], int n)
{
    long tmp[DISCARD];
    int i = 0;

    while (i < n) {
        ran_array(tmp, DISCARD);

        int k = KEEP;
        if (n - i < KEEP)
            k = n - i;

        for (int j = 0; j < k; j++)
            aa[i + j] = tmp[j];

        i += k;
    }
}
```

## Correctness Argument

Each call to `ran_array(tmp, 1009)` produces exactly the next block of $1009$ random integers in the sequence defined by the generator, while correctly updating the generator state stored in `ran_x`.

From each such block, the function retains exactly the first $100$ values, as required by the exercise specification, and discards the remaining $909$. This selection is implemented by a direct copy loop with no modification of the generated values.

The process repeats until $n$ output values have been written. The final block may require fewer than $100$ values, handled by the truncation `k = n - i`.

## Why the previous approach was incorrect

The previous solution failed because it attempted to reproduce the internal recurrence of `ran_array`, introducing unnecessary and error-prone state manipulation. In a correct solution, the generator is never reimplemented or partially duplicated. The wrapper must depend solely on the published interface of `ran_array`, ensuring that the sequence remains consistent with Knuth’s specification and that correctness reduces entirely to the correctness of the original generator.

This completes the corrected solution.
