**Exercise 12.** &#9654; [*M21*] *[M21]* What random number generator would be suitable for a minicomputer that does arithmetic only on integers in the range $[-32768\,.\,.\,32767]$?

**Verified:** yes  
**Solve time:** 2m19s

## Solution

The machine performs integer arithmetic only in the range $[-32768,.,.,32767]$, which corresponds to a word size of $2^{16}$. In two’s complement arithmetic, overflow implements reduction modulo $2^{16}$ automatically, so a linear congruential generator of the form

$X \leftarrow (aX + c) \bmod 2^{16}$

can be implemented directly in the machine’s native arithmetic.

Choose

$m = 2^{16} = 65536.$

To ensure maximal period when $m$ is a power of $2$, Section 3.6(iii) requires $a \bmod 8 = 5$, and Section 3.6(v) requires $c$ to be relatively prime to $m$, hence $c$ must be odd. A convenient choice satisfying the size requirement $0.01m < a < 0.99m$ is

$a = 40013,\quad c = 1.$

Indeed, $40013 \bmod 8 = 5$, and $1$ is coprime to $2^{16}$.

Define the recurrence

$X \leftarrow (40013X + 1) \bmod 2^{16}.$

Each application of this rule produces a new value of $X$ in the same word, with overflow performing the reduction modulo $2^{16}$ exactly. The resulting sequence cycles through all $2^{16}$ possible residues before repeating, since the conditions $a \bmod 8 = 5$ and $c$ odd ensure full period for modulus a power of $2$ as stated in Section 3.6(iii)–(v).

To interpret outputs in the required signed range $[-32768,.,.,32767]$, each word produced by the recurrence is read in two’s complement form; this identifies the same bit pattern with a unique integer in that interval, so no additional correction is needed.

The initial seed $X_0$ may be any odd integer in $[-32768,.,.,32767]$, since different seeds only shift the same full-period cycle.

This completes the construction of a suitable random number generator for the given machine. ∎
