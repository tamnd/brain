**Exercise 10.** [*22*] Table 1 shows how a binary-coded decimal number can be doubled by using various shifting, extracting, and addition operations on a binary computer. Give an analogous method that computes *half* of a binary-coded decimal number (throwing away the remainder if the number is odd).

## Solution

Let $u$ be a nonnegative integer represented in binary-coded decimal (BCD) form as

$u = u_{n-1} u_{n-2} \ldots u_1 u_0,$

where each $u_j$ occupies four bits and satisfies $0 \le u_j \le 9$. We seek an algorithm to compute $\lfloor u/2 \rfloor$, discarding any remainder if $u$ is odd, using only operations analogous to those in Table 1: shifts, additions, and extractions. The algorithm should transform each BCD digit and propagate carries correctly.

Consider first a single decimal digit $d$, $0 \le d \le 9$. When halving $d$, two cases occur:

1. $d$ is even: $d/2$ is an integer, no remainder.
2. $d$ is odd: $\lfloor d/2 \rfloor = (d-1)/2$, leaving a remainder of $1$.

To handle carries arising from halving, we proceed from the most significant digit toward the least significant digit, propagating any remainder as a "carry-in" to the next lower digit. Specifically, let $c$ denote the carry-in from the previous digit (initially $c = 0$). Then the new value of digit $u_j$ after halving is

$u_j' = \lfloor (c \cdot 10 + u_j)/2 \rfloor,$

and the carry-out to the next digit is

$c = (c \cdot 10 + u_j) \bmod 2.$

This computation guarantees that $u_j'$ is an integer between 0 and 9 and that the sequence of digits $(u_{n-1}', \ldots, u_0')$ correctly represents $\lfloor u/2 \rfloor$ in BCD.

We now express the operations in terms of binary shifts, additions, and extractions, following the style of Table 1. Let $x_j = 2 u_j + r_j$, where $r_j$ is initially the carry propagated from the higher digits. For halving, we reverse the doubling procedure:

1. Initialize the carry-in $c = 0$.
2. For each digit $j$ from $n-1$ down to $0$:

a. Form the intermediate value $v_j = 2c + u_j$.

b. Extract the new digit $u_j' = \lfloor v_j / 2 \rfloor$.

c. Update the carry $c = v_j \bmod 2$.

This sequence mimics the steps of Table 1 in reverse:

- Step 1 corresponds to initializing the carry.
- Step 2a combines the carry-in with the current digit.
- Step 2b divides by 2 using a right shift (discarding the remainder).
- Step 2c stores the new carry for the next digit.

To illustrate, consider $u = 369$, represented in BCD as $0011,0110,1001$. Initialize $c = 0$:

1. $j=2$, $u_2 = 3$, $v_2 = 2\cdot 0 + 3 = 3$, $u_2' = \lfloor 3/2 \rfloor = 1$, $c = 3 \bmod 2 = 1$.
2. $j=1$, $u_1 = 6$, $v_1 = 2\cdot 1 + 6 = 8$, $u_1' = \lfloor 8/2 \rfloor = 4$, $c = 8 \bmod 2 = 0$.
3. $j=0$, $u_0 = 9$, $v_0 = 2\cdot 0 + 9 = 9$, $u_0' = \lfloor 9/2 \rfloor = 4$, $c = 9 \bmod 2 = 1$ (ignored at the least significant digit).

The resulting digits $u_2' u_1' u_0' = 1,4,4$, which correctly represents $\lfloor 369/2 \rfloor = 184$ in decimal. The final carry is discarded, as required.

In summary, the BCD-halving algorithm proceeds as follows:

1. Initialize $c \gets 0$.
2. For $j = n-1$ down to $0$:

a. $v_j \gets 2c + u_j$.

b. $u_j' \gets \lfloor v_j / 2 \rfloor$.

c. $c \gets v_j \bmod 2$.

This method uses only shifts, additions, and digit extractions, and it produces $\lfloor u/2 \rfloor$ for any BCD number $u$. This completes the proof.
