[Section 4.3.3: How Fast Can We Multiply?](../)

**Exercise 13.** [*M25*] What is a good upper bound on the time needed to multiply an $m$-bit number by an $n$-bit number, when both $m$ and $n$ are very large but $n$ is much larger than $m$, based on the results discussed in this section for the case $m = n$?

**Verified:** yes  
**Solve time:** 1m52s  

---

## Solution

Let $M$ be an $m$-bit integer and $N$ an $n$-bit integer with $n \gg m$. Let $T(t)$ denote the time required to multiply two $t$-bit integers using the recursive method of Section 4.3.3, so that by relation (5) there exists a constant $c$ with

$T(t) \le c\, t^{\lg 3}.$

Write the $n$-bit integer $N$ in base $2^m$:

$N = \sum_{i=0}^{k-1} N_i 2^{im},$

where $k = \lceil n/m \rceil$ and each $N_i$ is an $m$-bit integer.

Then

$MN = \sum_{i=0}^{k-1} (M N_i) 2^{im}.$

Each product $M N_i$ is an $m$-bit by $m$-bit multiplication, hence each costs $T(m)$ operations. There are $k$ such products, so the total cost of all recursive multiplications is

$k\,T(m).$

The remaining work consists of shifting by multiples of $2^m$ and adding $k$ terms. Each shifted addition contributes $O(m)$ bit operations per block position, and there are $O(k)$ blocks, so the total addition cost is $O(km)=O(n)$.

Thus the total running time satisfies

$T(m,n) \le kT(m) + O(n).$

Substituting $k = \lceil n/m \rceil$ and $T(m)=O(m^{\lg 3})$ gives

= O\!\left(n\, m^{\lg 3 - 1}\right) + O(n).$$
Since $\lg 3 > 1$, the dominant term for large $m$ is $n\, m^{\lg 3 - 1}$, yielding the bound
$$\boxed{T(m,n) = O\!\left(n\, m^{\lg 3 - 1}\right)}.$$ This is the standard upper bound obtained by reducing the problem to $k$ multiplications of $m$-bit numbers using the Karatsuba-type recurrence from Section 4.3.3. ∎
