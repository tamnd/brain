[Section 4.3.3: How Fast Can We Multiply?](../)

**Exercise 15.** [*M49*] (S. A. Cook.) A multiplication algorithm is said to be *online* if the $(k+1)$st input bits of the operands, from right to left, are not read until the $k$th output bit has been produced. What are the fastest possible online multiplication algorithms achievable on various species of automata?

We restart from a clean formulation and give a model-independent analysis of what “fastest online multiplication” means in each automaton class.

Let inputs be binary streams

$$
u = \sum_{i\ge 0} u_i 2^i,\qquad v = \sum_{i\ge 0} v_i 2^i,
$$

read least significant bit first. The output stream is

$$
w_k = \sum_{i=0}^k u_i v_{k-i} \pmod 2
$$

together with carries, but the key structural fact is that the $k$th output bit depends only on pairs $(i,j)$ with $i+j=k$.

We assume a standard sequential online model: at step $k$, after reading $u_0,\dots,u_k$ and $v_0,\dots,v_k$, the machine must output $w_k$ before reading $u_{k+1}, v_{k+1}$.

# 1. Finite-state automata

### Claim

No finite-state automaton can perform unbounded online multiplication. Hence the fastest possible behavior is only for bounded input length; there is no asymptotic online multiplication algorithm.

### Proof (rigorous indistinguishability)

Fix a finite-state machine $M$ with state set $S$, $|S|=s$.

For each $k$, consider inputs where all bits are zero except possibly pairs $(i,k-i)$. Define, for each subset $A \subseteq \{0,\dots,k\}$, inputs $u^{(A)}, v^{(A)}$ by

$$
u_i^{(A)} = 1 \iff i \in A,\qquad v_j^{(A)} = 1 \iff j \in \{k-i : i \in A\},
$$

and all other bits zero.

Then the $k$th output bit satisfies

$$
w_k^{(A)} \equiv |A| \pmod 2.
$$

In particular, for different choices of $A$, the correct output depends on global information about the prefix of length $k$. More importantly, consider prefixes up to time $k$: there are $2^k$ distinct assignments to $u_0,\dots,u_{k-1}$, and these induce $2^k$ distinct required “future behaviors” for correct computation of $w_k$ and subsequent bits.

If two distinct prefixes lead $M$ to the same state, then by feeding appropriate continuations that isolate a single contributing pair, one can force different required outputs while $M$ behaves identically, contradicting correctness.

Formally, define two prefixes equivalent if all continuations produce identical future output streams. Online correctness requires that each distinct prefix induces a distinct state. Hence the number of states must satisfy $s \ge 2^k$ for all $k$, impossible for fixed $s$.

Thus no finite-state automaton computes unbounded online multiplication.

# 2. Pushdown automata and Turing machines

We treat PDA and TM together because both have unbounded memory, but only sequential access to input.

We determine the optimal time per output bit.

## 2.1 Upper bound: $O(k)$ time per output bit

At time $k$, we can compute $w_k$ by explicitly evaluating all contributing pairs:

$$
w_k = \sum_{i=0}^k u_i v_{k-i} \pmod 2,
$$

together with carry computation.

A direct online algorithm stores all past bits on the stack (PDA) or tape (TM). To compute output $k$, it scans indices $i=0,\dots,k$, retrieves $u_i$ and $v_{k-i}$, and accumulates the sum.

Each output bit is therefore computable in $O(k)$ time. Since $k \le n$, this gives $O(n)$ per bit in the worst case.

## 2.2 Lower bound: $\Omega(k)$ per output bit

We now show this is optimal.

### Key idea

At time $k$, the output depends on $k+1$ _independent potential contributions_

$$
(u_0,v_k), (u_1,v_{k-1}), \dots, (u_k,v_0).
$$

Each pair can independently flip the output bit.

### Adversary argument

Fix $k$. Consider $k+1$ input families $I^{(t)}$, $t=0,\dots,k$, defined by:

- all bits are zero except
- $u_t = 1$, $v_{k-t} = 1$.

Then for each $t$,

$$
w_k^{(t)} = 1,
$$

while the all-zero input gives $w_k = 0$.

Now consider any algorithm that produces $w_k$ online.

To be correct, at time $k$ it must distinguish the all-zero input from each $I^{(t)}$. The only way to distinguish $I^{(t)}$ is to read at least one of the bits $u_t$ or $v_{k-t}$.

Since these $k+1$ cases involve disjoint critical positions, any algorithm that has not examined $\Omega(k)$ of these positions cannot rule out all alternatives.

Thus any correct algorithm must perform at least $k+1$ essential bit-inspections by time $k$, giving a time lower bound

$$
\Omega(k).
$$

This bound is independent of whether the machine has a stack or tape: extra memory does not reduce the number of distinct input symbols whose contribution must be resolved.

## 2.3 Tightness

The upper bound $O(k)$ and lower bound $\Omega(k)$ match, so:

$$
\text{optimal time per output bit} = \Theta(k).
$$

In particular, for operands of length $n$, the worst-case per-bit time is $\Theta(n)$.

# 3. Conclusion by automaton class

### Finite-state automata

- Impossible to compute unbounded online multiplication.
- Formal reason: finite-state indistinguishability fails because prefixes induce exponentially many inequivalent future behaviors.

### Pushdown automata

- Optimal online multiplication time per output bit:

$$
\Theta(k)
$$

- Achieved by explicit recomputation of all contributing pairs.
- Lower bound follows from necessity of resolving $k+1$ independent cross-pair contributions.

### Turing machines

- Same optimal bound:

$$
\Theta(k)
$$

- Tape or stack does not remove the need to process $k+1$ contributions per output bit.

# Final answer

- Finite-state automata: unbounded online multiplication is impossible.
- Pushdown automata: optimal time per output bit is $\Theta(k)$, hence $\Theta(n)$ in the worst case.
- Turing machines: same optimal bound $\Theta(k)$, hence $\Theta(n)$ worst case.

No sequential online model can asymptotically beat linear work per produced output bit, because each output bit depends on $\Theta(k)$ independent input interactions that must be resolved explicitly.
