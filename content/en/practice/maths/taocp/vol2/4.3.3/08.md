**Exercise 8.** [*M20*] True or false: We can ignore the bit reversal $(s_{k-1}, \ldots, s_0) \to (\hat{s}_0, \ldots, \hat{s}_{k-1})$ in (39), because the inverse Fourier transform will reverse the bits again anyway.

## Solution

The statement is **false**. The bit reversal $(s_{k-1}, \ldots, s_0) \mapsto (\hat{s}_0, \ldots, \hat{s}_{k-1})$ in equation (39) of Section 4.3.3 is essential and cannot be omitted, even if the inverse Fourier transform is applied afterward.

To justify this, recall that in the Cooley–Tukey fast Fourier transform (FFT) algorithm, the input sequence is permuted according to the bit-reversed order so that the recursive decomposition of the discrete Fourier transform aligns with the in-place computation of the butterfly operations. Specifically, if $s_j$ denotes the $j$th input coefficient, the FFT rearranges these coefficients to $(s_{\hat{j}})$, where $\hat{j}$ is the integer obtained by reversing the $k$-bit binary representation of $j$.

Omitting this step would produce an output sequence in which the results of the recursive multiplications and additions are stored in the wrong positions. The inverse Fourier transform applied to this misordered sequence would **not** automatically correct the bit-reversal; instead, it would yield the values of the polynomial evaluated at the powers of the primitive root $\omega$ in an order corresponding to the un-reversed indices. Consequently, the final reconstruction of coefficients from the FFT would be scrambled.

Formally, let $X_j$ denote the FFT of the original sequence $x_j$. The in-place FFT algorithm satisfies

$X_{\hat{j}} = \sum_{m=0}^{N-1} x_m \omega^{jm},$

where $\hat{j}$ is the bit-reversal of $j$ and $N = 2^k$. If the bit-reversal is skipped, the computed array will contain $X_j$ in the position $j$ rather than $\hat{j}$. When the inverse transform is applied to this array, one obtains

$x'_m = \frac{1}{N} \sum_{j=0}^{N-1} X_j \omega^{-jm},$

which differs from the original $x_m$ because the order of the $X_j$ has not been permuted correctly. Therefore, the bit-reversal step cannot be ignored.

This completes the proof.
