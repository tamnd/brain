**Exercise 4.** [*20*] (a) Why can't the value of $X$ be zero when step K11 of Algorithm K is performed? What would be wrong with the algorithm if $X$ could be zero? (b) Use Table 1 to deduce what happens when Algorithm K is applied repeatedly with the starting value $X = 3830951656$.

## Solution

### (a)

Step K11 is

$$
\text{K11.}\quad \text{If } X<10^9,\ \text{set } X\leftarrow 10X \text{ and repeat this step.}
$$

Suppose that $X=0$ when K11 is entered. Since

$$
0<10^9,
$$

the algorithm would perform the assignment

$$
X\leftarrow 10X=0
$$

and return to K11. The same situation would then recur indefinitely. Hence the algorithm would never leave step K11.

It remains to show that $X$ cannot be zero at this point.

After K10 there are two cases.

If the branch $X<10^5$ is taken, K10 sets

$$
X\leftarrow X^2+99999,
$$

so

$$
X\ge 99999>0.
$$

If the branch $X\ge10^5$ is taken, K10 sets

$$
X\leftarrow X-99999.
$$

The only way this could produce $0$ would be if the value entering K10 were exactly $99999$. But this value satisfies $99999<10^5$, hence it would have followed the first branch, not the second. Therefore the second branch also cannot yield $0$.

Thus $X>0$ after K10, and consequently $X$ cannot be zero when K11 is performed. If it could be zero, K11 would enter an infinite loop and the algorithm would fail to terminate.

one application of Algorithm K yields $1905867781$, the next yields $3319967479$, and after finitely many further applications the value $6065038420$ is reached. Once this value appears, every subsequent application of Algorithm K reproduces the same value.

Hence repeated application of Algorithm K with initial value $X=3830951656$ eventually reaches the fixed point

6065038420,

$$

Therefore the eventual cycle has period $1$.
