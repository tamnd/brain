**Exercise 5.** [*15*] Explain why, in any case, Algorithm K should not be expected to provide infinitely many random numbers, in the sense that (even if the coincidence given in Table 1 had not occurred) one knows in advance that any sequence generated by Algorithm K will eventually be periodic.

## Solution

Algorithm K generates each new value of $X$ by a fixed deterministic rule applied to the preceding value. The value of $X$ is always a 10-digit number, hence there are only

$10^{10}$

possible states.

Let

$X_0,X_1,X_2,\ldots$

be the sequence produced by repeated application of Algorithm K. Consider the first

$10^{10}+1$

terms

$X_0,X_1,\ldots,X_{10^{10}}.$

Since there are only $10^{10}$ possible values for $X$, two of these terms must be equal, by the pigeonhole principle. Thus there exist integers $i<j$ such that

$X_i=X_j.$

Because the algorithm is deterministic, equal values have equal successors. Therefore

$X_{i+1}=X_{j+1}.$

Applying the same argument repeatedly gives

$X_{i+k}=X_{j+k}\qquad(k\ge0).$

Let

$p=j-i.$

Then

$X_{i+p}=X_i,$

and hence

$X_{i+mp}=X_i\qquad(m\ge0).$

Therefore the sequence repeats with period $p$ from the point $X_i$ onward. The sequence may have a nonperiodic initial segment $X_0,\ldots,X_{i-1}$, but after the first repeated state it is necessarily periodic.

Consequently, regardless of the particular behavior exhibited in Table 1, every sequence generated by Algorithm K must eventually enter a cycle. Since a periodic sequence can contain at most finitely many distinct values, Algorithm K cannot be expected to provide infinitely many random numbers in the sense of producing infinitely many distinct outputs.

$$
\boxed{\text{Every sequence generated by Algorithm K is eventually periodic.}}
$$
