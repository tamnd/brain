**Exercise 22.** [*21*] *[21]* (H. Rolletschek.) Would it be a good idea to generate random numbers by using the sequence $f(0), f(1), f(2), \ldots$, where $f$ is a random function, instead of using $x_0, f(x_0), f(f(x_0))$, etc.?

**Verified:** yes  
**Solve time:** 2m54s  

---

Let $f$ be a random function on a (typically infinite) set, meaning that for each input $x$, the value $f(x)$ is chosen according to a fixed distribution on outputs, and the random variables $\{f(x)\}_{x}$ are independent for distinct inputs $x$.

We compare two ways of generating a sequence of random values.

### 1. The sequence $f(0), f(1), f(2), \ldots$

Fix distinct indices $i_1, \ldots, i_k$. Then the inputs $i_1, \ldots, i_k$ are distinct arguments of the random function. By the definition of a random function, the values $f(i_1), \ldots, f(i_k)$ are independent random variables. Moreover, each $f(i)$ has the same distribution as $f(x)$ for any fixed $x$, since the distribution does not depend on the input.

Therefore the sequence $\{f(n)\}_{n \ge 0}$ is an independent identically distributed sequence with the desired marginal distribution. There is no reuse of function values across different indices, so no dependence is introduced by the construction.

### 2. The iterative sequence $x_{n+1} = f(x_n)$

Now consider a sequence defined by iteration:

$$
x_0 \text{ given}, \quad x_{n+1} = f(x_n).
$$

Each term $x_{n+1}$ is obtained by evaluating $f$ at a random input $x_n$. The key point is that the same value $f(x)$ is reused whenever the process revisits a previously seen state $x$. This creates dependence between terms.

To make this precise, consider $x_0, x_1, x_2$. We have

$$
x_1 = f(x_0), \quad x_2 = f(f(x_0)).
$$

If $x_2$ happens to satisfy $x_2 = x_0$, then $f(x_0)$ influences both $x_1$ and $x_2$. More generally, even when no immediate repetition occurs, the randomness in later states is always coupled through previously sampled values of $f$.

A clean way to see the dependence is to note that the collection of all values $\{f(x)\}$ is fixed once sampled. The process $x_n$ does not resample $f$ independently at each step; instead it adaptively queries a fixed random table. Hence the sequence $\{x_n\}$ is a function of a single random mapping and is not a sequence of independent draws.

### 3. Correct structural behavior of the iterated process

The earlier argument incorrectly invoked eventual periodicity, which holds only for functions on finite sets. For an infinite domain, no such conclusion is valid: the trajectory $x_0, x_1, \ldots$ may or may not repeat values, and there is no general guarantee of cycles.

What is always true is weaker but sufficient for randomness analysis: the process is a deterministic walk on a random directed graph where each vertex has outdegree $1$. The structure of this graph induces correlations because paths that merge share all future evolution.

In particular, if two different starting points ever reach the same value $x$, then their future evolutions are identical from that point onward:

$$
x_n = y_m \;\Rightarrow\; x_{n+k} = y_{m+k} \text{ for all } k \ge 0.
$$

This “coalescence” property shows strong dependence between different parts of the sequence and prevents any interpretation of the iterates as independent samples.

### 4. Comparison of the two methods

The sequence $f(0), f(1), f(2), \ldots$ uses disjoint arguments of $f$. By independence of the random function at distinct inputs, each term is independent of the others, giving an i.i.d. sequence.

The iterative sequence $x_{n+1} = f(x_n)$ repeatedly evaluates the same random function on values that depend on previous evaluations. This reuse of random data introduces correlations, because the underlying randomness is shared across time through the fixed function $f$.

### Conclusion

The sequence $f(0), f(1), f(2), \ldots$ is a correct source of independent identically distributed random variables (assuming independent values of a random function at distinct inputs).

The iterative method $x_{n+1} = f(x_n)$ does not produce independent samples because it adaptively reuses a single random function, creating persistent dependence between values.

Therefore, while both methods use the same underlying randomness, only the first yields independent random numbers in the intended sense.
