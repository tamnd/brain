## Solution to Exercise 4.3.2.7

### Step 1: Context and original formula

Equation (24) in _The Art of Computer Programming_, Volume 2, Section 4.3.2, describes a method to convert residues modulo $m_1, \ldots, m_r$ into a unique integer modulo $M = m_1 m_2 \cdots m_r$ using constants $c_{ij}$. Explicitly, for residues $u_1, \ldots, u_r$, the sequence $v_1, \ldots, v_r$ is given recursively by

$$
v_1 = u_1 \bmod m_1,
$$

$$
v_2 = (u_2 - v_1)c_{12} \bmod m_2,
$$

$$
v_3 = \Big(u_3 - \big(v_1 + m_1 v_2\big)\Big) c_{13} c_{23} \bmod m_3,
$$

$$
\vdots
$$

$$
v_r = \Big(u_r - \big(v_1 + m_1(v_2 + m_2(v_3 + \cdots + m_{r-2}v_{r-1}) \cdots)\big)\Big) c_{1r} c_{2r} \cdots c_{r-1,r} \bmod m_r.
$$

Each constant $c_{ij}$ satisfies

$$
c_{ij} m_i \equiv 1 \pmod{m_j}, \quad 1 \le i < j \le r.
$$

The original method in (24) required storing all $c_{ij}$, totaling $r(r-1)/2$ constants.

### Step 2: Rewriting in terms of cumulative constants

Define cumulative constants for each $j = 2, \ldots, r$:

$$
C_j = c_{1j} c_{2j} \cdots c_{j-1,j} \bmod m_j.
$$

Then each $v_j$ can be rewritten as

$$
v_j = \Big(u_j - \sum_{i=1}^{j-1} m_1 m_2 \cdots m_{i-1} v_i\Big) C_j \bmod m_j,
$$

where the sum inside the parentheses is the standard nested expansion:

$$
v_1 + m_1 \big(v_2 + m_2 (v_3 + \cdots + m_{j-2} v_{j-1}) \cdots \big).
$$

For $j=2$ this reduces to $v_2 = (u_2 - v_1) C_2 \bmod m_2$, exactly as before. For $j=3$ it becomes $v_3 = (u_3 - (v_1 + m_1 v_2)) C_3 \bmod m_3$, and so on.

This rewriting shows that instead of storing each individual $c_{ij}$, we only need $r-1$ cumulative constants $C_2, C_3, \ldots, C_r$ (plus $v_1 = u_1 \bmod m_1$ with no multiplier).

### Step 3: Verification by induction

Base case $r=2$:

$$
v_2 = (u_2 - v_1)c_{12} = (u_2 - v_1) C_2 \bmod m_2, \quad C_2 = c_{12}.
$$

Inductive step: assume formula holds for $j-1$. Then

$$
v_j = \Big(u_j - \sum_{i=1}^{j-1} m_1 \cdots m_{i-1} v_i\Big) c_{1j} c_{2j} \cdots c_{j-1,j} \bmod m_j = (u_j - \text{nested sum}) C_j \bmod m_j,
$$

which is exactly the rewritten form. Therefore the formula is valid for all $j = 1, \ldots, r$.

### Step 4: Computational merits

1. **Memory usage**: Using the original formula (24) requires storing $r(r-1)/2$ constants $c_{ij}$. In contrast, the rewritten form requires storing only $r-1$ cumulative constants $C_j$. For large $r$, this is a substantial saving in memory.
2. **Multiplication cost**: In the rewritten formula, the products $c_{1j} \cdots c_{j-1,j}$ are precomputed modulo $m_j$. This reduces the number of modular multiplications during the actual computation of $v_j$. The tradeoff is the one-time precomputation of $C_j$.
3. **Numerical stability**: Precomputing the cumulative constants $C_j$ ensures that all subsequent arithmetic uses fewer intermediate large numbers. Each operation is modulo $m_j$, which can be beneficial on computers with fixed-precision arithmetic.
4. **Simplicity of indexing**: The rewritten formula is conceptually simpler: each $v_j$ is computed with a single multiplicative factor $C_j$, rather than chaining $j-1$ individual $c_{ij}$ constants at runtime. This reduces potential indexing errors and improves code readability.
5. **Tradeoff**: If the application involves frequent changes in the moduli $m_j$ or the residues $u_j$, recomputation of $C_j$ may be needed. In such cases, the savings are less significant. Otherwise, for fixed moduli, the cumulative constants approach is clearly superior.

### Step 5: Summary

- Each $v_j$ can be expressed as

$$
v_1 = u_1 \bmod m_1, \quad v_j = \Big(u_j - (v_1 + m_1(v_2 + \cdots + m_{j-2} v_{j-1}))\Big) C_j \bmod m_j, \quad 2 \le j \le r.
$$

- The cumulative constants $C_j = c_{1j} c_{2j} \cdots c_{j-1,j} \bmod m_j$ replace the individual constants $c_{ij}$.
- Computational advantages include reduced memory usage, fewer multiplications during evaluation, and simpler indexing.
- The formula is correct by induction, starting from $v_2$ and extending to $v_r$.
