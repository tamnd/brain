## Exercise 4.5.2.39 [_M28_]

**Problem.** (V. R. Pratt) Extend Algorithm B to an Algorithm Y that is analogous to Algorithm X.

## Setup

Let $u$ and $v$ be positive integers. Algorithm B, described in Section 4.5.2, is the classical _subtractive Euclidean algorithm_, which repeatedly replaces the larger of the two integers by its difference with the smaller until equality is reached. Algorithm X, introduced in Section 4.5.2, is a variant that reduces the number of steps by using quotient estimates: it replaces the subtraction of one number from another multiple times by a single division with remainder.

The task is to define **Algorithm Y**, which extends Algorithm B in a way analogous to the extension from Algorithm A to Algorithm X: that is, Algorithm Y should replace multiple subtractions in B6 of Algorithm B by a single division, while preserving correctness. The goal is to describe Algorithm Y rigorously, justify that it terminates with $\gcd(u, v)$, and show that the number of steps is reduced compared with Algorithm B.

We adopt the same notation as in Section 4.5.2: the integers $u$ and $v$ correspond to $A$ and $C$ in Algorithm B, the remainder after division is denoted $r$, and temporary replacements are indicated using $\leftarrow$.

## Solution

**Algorithm Y** (_Extended subtractive algorithm with division_).

Given positive integers $A$ and $C$, we define the steps:

**Y1.** [Initialize.] If $A < C$, exchange $A$ and $C$ so that $A \ge C > 0$.

**Y2.** [Check divisibility.] If $C \mid A$, terminate; $\gcd(A, C) = C$.

**Y3.** [Compute quotient and remainder.] Compute

$q \leftarrow \lfloor A / C \rfloor, \quad r \leftarrow A - q \cdot C.$

**Y4.** [Replace.] Set $A \leftarrow C$, $C \leftarrow r$.

**Y5.** [Repeat.] Return to **Y2**.

**Justification.**

At each iteration, $r = A - q C$ satisfies $0 \le r < C$, so the pair $(A, C)$ is strictly decreasing in the second component. Step **Y2** ensures termination when the remainder is zero.

We claim that Algorithm Y terminates with $\gcd(A, C)$:

1. Let $d$ be any common divisor of $A$ and $C$ at the beginning of an iteration. Then $d$ divides $A - q C = r$, so $d$ divides both $C$ and $r$.
2. Conversely, any common divisor of $C$ and $r$ also divides $A = q C + r$.

Therefore, by induction on the number of iterations, the final nonzero value of $C$ is the greatest common divisor of the original $A$ and $C$.

**Step count reduction.**

Algorithm B replaces $A$ by $A - C$ repeatedly, executing the subtraction step multiple times. Algorithm Y computes $q = \lfloor A / C \rfloor$ and performs a single replacement $A \leftarrow A - q C$, which is equivalent to $q$ subtractions in Algorithm B. Consequently, the number of iterations in Algorithm Y is at most the number of nonzero remainders in the classical Euclidean algorithm, which is $\mathcal{O}(\log \max(A, C))$, instead of potentially $\mathcal{O}(\max(A, C))$ in Algorithm B.

## Verification

Consider the pair $(A, C) = (55, 21)$:

1. **Iteration 1:** $q = \lfloor 55/21 \rfloor = 2$, $r = 55 - 2 \cdot 21 = 13$, then $(A, C) \leftarrow (21, 13)$.
2. **Iteration 2:** $q = \lfloor 21/13 \rfloor = 1$, $r = 21 - 1 \cdot 13 = 8$, then $(A, C) \leftarrow (13, 8)$.
3. **Iteration 3:** $q = \lfloor 13/8 \rfloor = 1$, $r = 13 - 8 = 5$, then $(A, C) \leftarrow (8, 5)$.
4. **Iteration 4:** $q = \lfloor 8/5 \rfloor = 1$, $r = 8 - 5 = 3$, then $(A, C) \leftarrow (5, 3)$.
5. **Iteration 5:** $q = \lfloor 5/3 \rfloor = 1$, $r = 2$, then $(A, C) \leftarrow (3, 2)$.
6. **Iteration 6:** $q = \lfloor 3/2 \rfloor = 1$, $r = 1$, then $(A, C) \leftarrow (2, 1)$.
7. **Iteration 7:** $q = \lfloor 2/1 \rfloor = 2$, $r = 0$, algorithm terminates with $\gcd(55, 21) = 1$.

This reproduces the classical Euclidean sequence but consolidates multiple subtractions into single division steps.

## Notes

Algorithm Y is equivalent to the _modern Euclidean algorithm using division with remainder_ (Algorithm X), but it is formally derived as an extension of the subtractive Algorithm B. Any bounds on the number of steps from Section 4.5.2, such as $1 + \lfloor \lg \max(u, v) \rfloor$ for the subtractive form, can be replaced by the number of nonzero quotients in the sequence of divisions, which is always less than or equal to that bound.

Algorithm Y can be further optimized by using binary division techniques or Lehmer’s method for large integers, but the correctness argument remains identical, relying only on the property that common divisors are preserved under the transformation $(A, C) \mapsto (C, A - q C)$.

This completes the construction of Algorithm Y. ∎

**Answer (algorithmic form)**

$$
\begin{aligned}
&\text{Given positive integers } A, C: \\
&\quad \text{If } A < C, \text{ swap } A \text{ and } C \\
&\quad \text{While } C \ne 0: \\
&\quad\quad q \leftarrow \lfloor A / C \rfloor \\
&\quad\quad r \leftarrow A - q \cdot C \\
&\quad\quad A \leftarrow C, \quad C \leftarrow r \\
&\quad \text{Return } A \text{ as } \gcd(A, C)
\end{aligned}
$$
