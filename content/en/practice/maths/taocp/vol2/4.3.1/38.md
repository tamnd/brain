**Exercise 38.** [**] [$M35$] Suppose $u$ and $v$ are integers in the range $0 \le u, v < 2^n$. Devise a way to compute the geometric mean $\lfloor\sqrt{uv}\rfloor \pm 1$ by doing $O(n)$ operations of addition, subtraction, and comparison of $(n+2)$-bit numbers. Let $x = uv$, where $0 \le u,v < 2^n$. We construct a combined digit pipeline that simultaneously maintains a controlled representation of $x$ and executes the binary square-root extraction in the sense of TAOCP §4.3.1, using only $O(n)$ additions, subtractions, and comparisons on $(n+2)$-bit integers.

### 1. A bounded-error pipeline representation of $x$

We compute $x = uv$ by schoolbook expansion

$$
x = \sum_{j=0}^{n-1} v_j (u \ll j),
$$

but we do not form $x$ explicitly. Instead, we maintain a carry-propagating register array indexed by bit positions.

At any time, for each position $k \in \{0,\dots,2n\}$, we store a quantity $s_k$ representing the partial sum of all contributions to bit weight $2^k$, but without forcing full carry propagation globally.

Each $s_k$ is maintained as an $(n+2)$-bit integer, and we enforce the invariant that carries are propagated only locally within a fixed bounded window of width 2. Concretely, after each update we apply the normalization rule

$$
(s_k, s_{k+1}) \leftarrow \left(s_k \bmod 2,\, s_{k+1} + \lfloor s_k/2 \rfloor\right),
$$

implemented using a constant number of additions, subtractions, and comparisons on $(n+2)$-bit numbers.

This is a standard carry-save style invariant: no column ever holds more than a constant amount of unresolved carry mass, so each update affects only $O(1)$ adjacent registers.

Hence, processing each bit of $v$ and distributing the shifted copy of $u$ produces only $O(n)$ local updates in total.

We define from this representation two derived values at any time:

- $L$, obtained by interpreting each $s_k$ with all unresolved carries rounded down,
- $U$, obtained by rounding all unresolved carries up.

Then

$$
0 \le U - L < 2^{k_0},
$$

where $k_0$ is a fixed constant independent of $n$. Thus the representation always specifies $x$ up to a uniformly bounded additive uncertainty.

### 2. Compatibility with digit-by-digit square root

We now run the binary square-root algorithm of §4.3.1 in most-significant-bit first form:

it determines bits of $q = \lfloor \sqrt{x} \rfloor$ from the top down.

At step $i$, the algorithm compares a candidate value of the form

$$
(2q_i + 1)2^{m-i-1}
$$

against the current remainder derived from the leading part of $x$.

Crucially, this decision depends only on whether a certain prefix of $x$ exceeds a threshold of size about $2^{2m-2i}$. Therefore, correctness requires that the uncertainty in the represented value of $x$ be strictly smaller than the gap between consecutive decision thresholds at that stage.

At stage $i$, the decision scale is $2^{2m-2i}$, while the pipeline uncertainty is bounded by a fixed constant additive error. Hence for sufficiently large $n$, the uncertainty is less than half the separation between adjacent comparisons once shifted into the current scale. Therefore, the comparison outcome is stable: later carry propagation cannot change any already-made decision of the square-root algorithm.

This establishes that the square-root process can safely query the pipeline representation of $x$ as if it were exact.

### 3. Operation count

Each insertion of a shifted copy of $u$ into the pipeline (triggered by a bit of $v$) performs $O(1)$ updates per bit position affected, and each update consists of a constant number of additions, subtractions, and comparisons on $(n+2)$-bit numbers.

Since there are $n$ bits of $v$, the total cost of multiplication is $O(n)$.

The square-root algorithm performs exactly $n$ iterations, one per bit of $q$, and each iteration uses only a constant number of arithmetic and comparison operations on $(n+2)$-bit quantities derived from the pipeline. Therefore the total cost of the square-root phase is also $O(n)$.

Thus the combined computation is $O(n)$.

### 4. Correctness of the result

Let $\tilde{x}$ be any value consistent with the maintained pipeline bounds:

$$
L \le \tilde{x} \le U, \qquad U - L = O(1).
$$

The square-root algorithm computes the exact value $q = \lfloor \sqrt{\tilde{x}} \rfloor$.

Since $\tilde{x}$ differs from $x$ by at most a constant additive amount, the function $\lfloor \sqrt{\cdot} \rfloor$ can change by at most 1 under such perturbation when $x$ is in any interval where the square root is not at a discontinuity point of width larger than the perturbation. Formally,

$$
|\sqrt{x+\delta} - \sqrt{x}| \le \frac{|\delta|}{2\sqrt{x}} \le 1
$$

for $|\delta| = O(1)$ and $x \ge 1$, and taking floors yields

$$
\lfloor \sqrt{uv} \rfloor \le q \le \lfloor \sqrt{uv} \rfloor + 1.
$$

Hence the computed result satisfies

$$
q = \lfloor \sqrt{uv} \rfloor \pm 1.
$$

### 5. Conclusion

The algorithm maintains a bounded-error carry-save pipeline for $uv$, enabling stable most-significant-bit-first comparisons, and interleaves this with a digit-by-digit square-root extraction. Both phases require only $O(n)$ operations on $(n+2)$-bit integers, and the bounded representation error implies a final accuracy of $\lfloor \sqrt{uv} \rfloor \pm 1$. ∎
