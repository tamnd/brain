## Solution

We are asked to write a MIX program that evaluates a fifth-degree polynomial according to scheme (11) in Section 4.6.4, using floating-point arithmetic operators FADD and FMUL. Scheme (11) expresses a fifth-degree polynomial

$$
u(x) = a_5 x^5 + a_4 x^4 + a_3 x^3 + a_2 x^2 + a_1 x + a_0
$$

in nested form, also called Horner's rule, as

$$
u(x) = ((((a_5 x + a_4)x + a_3)x + a_2)x + a_1)x + a_0. \eqno(11)
$$

This form requires exactly five multiplications and five additions to evaluate, assuming all coefficients are nonzero. The MIX program should implement this directly, while taking care to use FADD and FMUL instructions for floating-point operations.

We assume the coefficients $a_0, \ldots, a_5$ and the variable $x$ are stored in consecutive memory locations for convenience. Let us define:

- $A_0$ at address `2000`
- $A_1$ at address `2001`
- $A_2$ at address `2002`
- $A_3$ at address `2003`
- $A_4$ at address `2004`
- $A_5$ at address `2005`
- $X$ at address `2010`

We will evaluate $u(x)$ in the accumulator `A` and store the result in `2020`.

The program proceeds as follows:

```
* Evaluate u(x) = ((((a5 x + a4)x + a3)x + a2)x + a1)x + a0
* Coefficients: A0..A5 at 2000..2005
* Variable x at 2010
* Result stored at 2020
        ORIG 1000
        LDA 2005,0      * A = a5
        FMP 2010,0      * A = A * x
        FADD 2004,0     * A = A + a4
        FMP 2010,0      * A = A * x
        FADD 2003,0     * A = A + a3
        FMP 2010,0      * A = A * x
        FADD 2002,0     * A = A + a2
        FMP 2010,0      * A = A * x
        FADD 2001,0     * A = A + a1
        FMP 2010,0      * A = A * x
        FADD 2000,0     * A = A + a0
        STA 2020,0      * Store result
        HLT
```

Each instruction corresponds exactly to one operation in the nested evaluation of (11). The order of operations preserves Horner's rule and ensures that partial results are reused without storing intermediate values elsewhere. This program uses five multiplications (FMUL) and five additions (FADD), matching the theoretical minimum.

To make the program slightly more efficient, one could load $x$ into an index register and use indirect addressing to avoid repeated memory fetches. However, since MIX has only a single floating-point accumulator, the present approach is the most straightforward while keeping all variables in memory.

This completes the solution.
