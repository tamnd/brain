## Solution

Let $N = s+t$. The genlex listing of $(s,t)$-combinations $a_{n-1}\dots a_1 a_0$ produces all $\binom{N}{t}$ binary strings with $t$ ones in lexicographic order. Method (39) is the next-combination procedure underlying this listing, and the variable $j$ is repeatedly increased in Step L3 while scanning a maximal suffix of positions satisfying the saturation condition $c_{j}+1=c_{j+1}$.

Each execution of the algorithm visits exactly one combination, and consecutive visits are separated by a single application of Step L5. The operation $j \leftarrow j+1$ occurs only inside Step L3, and only when the equality $c_j+1=c_{j+1}$ forces the algorithm to move one position leftward in the scan. Thus every occurrence of $j \leftarrow j+1$ corresponds to one unit step in the upward propagation of this scan through a maximal block of saturated inequalities.

Fix a combination $C = c_t \dots c_1$ that is not the final combination in lexicographic order. Let $j(C)$ be the index determined by Step L3 at the moment $C$ is processed, so that

$$
c_{j(C)}+1 \ne c_{j(C)+1}
$$

and for all $i<j(C)$ one has $c_i+1=c_{i+1}$. The computation of $j(C)$ begins at $j=1$ and performs the operation $j \leftarrow j+1$ exactly once for each index passed before reaching $j(C)$. Hence the number of increments of $j$ executed while processing $C$ is exactly $j(C)-1$.

The key structural fact is that each nonfinal combination $C$ contributes exactly one “first non-saturated position” $j(C)$, and this position is uniquely determined by $C$. Conversely, for every index $i \in {1,\dots,t}$ and every combination $C$ for which the scan reaches position $i$, there is exactly one increment $j \leftarrow j+1$ executed at level $i$ during the processing of $C$. These increments are disjoint across different combinations because the scan state at Step L3 depends only on the current combination and is completely reset at Step L1 for the next visit.

The process of generating all combinations can be viewed as a traversal of a rooted ordered tree whose nodes are all $\binom{N}{t}$ combinations in lexicographic order, where each step from one node to the next corresponds to a single application of Step L5. The executions of $j \leftarrow j+1$ occur exactly during the backward scan that determines the next branch point in this traversal. Each such increment corresponds to passing one internal boundary in this tree traversal, and every combination except the last contributes exactly one such boundary crossing in the global execution of the algorithm.

Since the traversal visits all $\binom{N}{t}$ combinations exactly once, it performs exactly $\binom{N}{t}-1$ transitions between consecutive combinations. Each transition contributes exactly one unit of net upward movement of the scan index $j$, and no other source of $j$ increments exists. Therefore the total number of executions of $j \leftarrow j+1$ equals the total number of transitions.

Hence the number of times method (39) performs the operation $j \leftarrow j+1$ is

$$
\binom{N}{t}-1 = \binom{s+t}{t}-1.
$$

This completes the proof. ∎
