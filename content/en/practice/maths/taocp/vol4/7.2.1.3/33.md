**Exercise 33.** [*HM33*] [HM33] How many of the genlex listings in exercise 31(b) are near-perfect?

**Verified:** no

## Setup

Let $n=s+t$ and consider genlex listings of $(s,t)$-combinations in index-list form $c_t\cdots c_1$, with

$n>c_t>c_{t-1}>\cdots>c_1\ge 0.$

A genlex listing is produced by Algorithm $L$ in Section 7.2.1.3, and Exercise 31(b) counts all possible genlex listings in this representation.

A listing is near-perfect when each transition between consecutive combinations changes exactly one coordinate of the index-list by $\pm 1$, with all other coordinates unchanged.

The goal is to count all genlex listings of index-list $(s,t)$-combinations that satisfy this condition.

## Solution

A transition in Algorithm $L$ is determined by Step L3. From a state $c_t\cdots c_1$, the algorithm finds the smallest index $j$ such that $c_j$ can be increased while preserving strict decrease, then sets

$c_j \leftarrow c_j+1$

and resets all $c_1,\dots,c_{j-1}$ to their minimal values $0,1,\dots,j-2$.

The near-perfect condition requires that every transition modifies exactly one coordinate. This forces that no reset of any coordinate $c_k$ with $k<j$ can occur at any step, hence necessarily $j=1$ at every transition.

Therefore, every intermediate configuration must satisfy that the rightmost entry $c_1$ is always the only coordinate eligible for change. This is possible only if the constraint $c_2>c_1$ never blocks successive increments of $c_1$.

For $t=0$, the listing contains a single empty combination, so the condition holds vacuously, contributing one valid listing.

For $t=1$, the index-list consists of a single coordinate $c_1$ with $0\le c_1\le n-1$. Algorithm $L$ produces the sequence

$0,1,2,\dots,n-1,$

and each transition is

$c_1 \leftarrow c_1+1,$

so exactly one coordinate changes in every step. Any genlex listing in this case is uniquely determined, since no branching is possible in a single-coordinate system. Hence exactly one near-perfect listing exists.

For $t\ge 2$, consider the first time a configuration occurs with $c_2=c_1+1$. At that point, increasing $c_1$ is forbidden, so Algorithm $L$ must increase $c_2$, forcing $j=2$. The transition then resets $c_1$ to $0$, producing a change in two coordinates simultaneously. This violates the near-perfect condition.

Since every complete genlex listing of $(s,t)$-combinations must realize configurations with arbitrarily large $c_1$ until saturation of the constraint $c_2>c_1$, such a forced $j=2$ event occurs in every listing. Therefore no genlex listing satisfies the near-perfect condition when $t\ge 2$.

Thus the number of near-perfect genlex listings is

1, & t\le 1,\\ 0, & t\ge 2. \end{cases}}$$ ## Verification For $t=0$, there is exactly one state and no transitions, so the condition holds. For $t=1$, every step in Algorithm $L$ is a single increment of $c_1$, and no reset variables exist, so each transition changes exactly one coordinate. For $t\ge 2$, the inequality constraint $c_2>c_1$ implies that once $c_1$ reaches $c_2-1$, the next increase forces $j=2$, and Step L3 triggers a reset of $c_1$. This produces at least one transition with multiple coordinate changes in every complete enumeration, so the near-perfect property fails globally. This completes the proof. ∎
