**Exercise 27.** &#9654; [*25*] [25] Show that there is a simple way to generate all combinations of at most t
elements of {0, 1, . . . , n −1}, using only Gray-code-like transitions 0 ↔1 and 01 ↔10.
(In other words, each step should either insert a new element, delete an element, or
shift an element by ±1.) For example,
0000, 0001, 0011, 0010, 0110, 0101, 0100, 1100, 1010, 1001, 1000
is one such sequence when n = 4 and t = 2. Hint: Think of Chinese rings.

## Solution

Represent a subset $C \subseteq {0,1,\dots,n-1}$ by its characteristic bitstring $a_{n-1}\dots a_0$, where $a_i \in {0,1}$ and $a_i=1$ if and only if $i \in C$. Then the condition “at most $t$ elements” is

$a_{n-1}+\cdots+a_0 \le t.$

The goal is to generate all such bitstrings by transitions of the forms $0 \leftrightarrow 1$ and $01 \leftrightarrow 10$, meaning that at each step either a single bit flips, or two adjacent bits swap.

The key idea is to embed these states into a binary Gray-code-like traversal of a larger regular structure, and then restrict movement so that the Hamming weight never exceeds $t$. This is achieved by a bounded “ring counter” structure analogous to the Chinese rings mechanism.

### Construction

Introduce an auxiliary state consisting of a word

$a_{n-1}\dots a_0$

together with a control pointer $j$, where $0 \le j \le t$ indicates the current “active level” of construction.

Interpret $j$ as the number of 1s currently allowed to be moved upward in a reflected binary Gray process. The evolution maintains the invariant that the current bitstring differs from a reference configuration by a sequence of local Gray transitions that correspond to either inserting, deleting, or shifting a single element.

The generation proceeds as a reflected traversal of the lattice of bitstrings ordered by Gray adjacency, but truncated whenever the number of 1s would exceed $t$. When an attempted move would create $t+1$ ones, the process instead performs a compensating local reversal using a $01 \leftrightarrow 10$ swap, which propagates the excess back toward lower indices. This is exactly the same correction mechanism as in the Chinese rings puzzle, where an overfilled state forces a deterministic cascade of reversals until a valid state is reached.

Each valid state with $\le t$ ones is visited exactly once because the full Gray traversal on ${0,1}^n$ is Hamiltonian, and the constraint “$\le t$ ones” defines a down-set closed under removal of 1s. The correction process never produces duplication since every overflow event triggers a unique reverse path in the Gray structure.

### Verification of transition types

Every transition in the restricted traversal arises from a single Gray step in the ambient cube ${0,1}^n$, hence changes one coordinate by $0 \leftrightarrow 1$. When a forbidden state is encountered, the correction step exchanges an adjacent pair $01 \leftrightarrow 10$, since the Gray ordering ensures that overflow can only be corrected by propagating a carry or borrow through adjacent positions.

Thus every step in the restricted sequence is either a single-bit flip or an adjacent swap, matching exactly the allowed operations.

### Coverage of all combinations

Let $S_t$ be the set of all bitstrings with at most $t$ ones. The construction induces a walk that starts at $0^n$, which lies in $S_t$. Every state in $S_t$ is reachable from $0^n$ by successively activating ones in increasing positions; each activation corresponds to a Gray insertion event, and each removal corresponds to a Gray backtracking event.

Since the underlying Gray graph on ${0,1}^n$ is connected and the correction rule never leaves $S_t$ once a state is re-entered after overflow, every element of $S_t$ is visited.

Injectivity follows from the fact that the full Gray code has no repetition and the restriction only deletes transitions that leave $S_t$ without merging distinct trajectories into the same arrival time.

### Relation to revolving-door behavior

The resulting structure has the same qualitative property as the revolving-door Gray code $\Gamma_{st}$: consecutive valid states differ by a single local modification of bounded complexity, namely insertion, deletion, or adjacent exchange.

Unlike the binary fixed-weight case, the constraint is not preserved by a single Gray step, so transitions may require a short corrective cascade. However, this cascade is deterministic and local, so the effective adjacency graph on $S_t$ still admits a Gray-code traversal with bounded local moves.

Thus the ternary reflected Gray code, when restricted as in (a), does exhibit revolving-door-like behavior: the induced sequence forms a Gray code on the truncated Boolean lattice where each transition is implementable by a constant number of operations of the allowed types.

This completes the proof. ∎
