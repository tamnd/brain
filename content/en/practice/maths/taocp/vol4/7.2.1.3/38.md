## Setup

Let $(s,t)$ be fixed with $n=s+t$. Let $C_{st}$ denote the lexicographic sequence of all $(s,t)$-combinations in the form $c_t \dots c_1$, ordered as in (3). Let $C^R_{st}$ denote the reverse sequence of $C_{st}$.

Algorithm C is the genlex method (39) specialized to lexicographic order: at each step it selects the rightmost index $j$ whose component can be increased while preserving the strict inequalities $n>c_t>\cdots>c_1\ge 0$, performs $c_j \leftarrow c_j+1$, then resets $c_{j-1}\dots c_1$ to the lexicographically least completion consistent with the constraint, namely $c_i \leftarrow i-1$.

The task is to construct the corresponding genlex algorithm generating $C^R_{st}$.

## Solution

The reverse sequence $C^R_{st}$ is obtained by reversing the total order induced by lexicographic comparison of decreasing tuples $c_t \dots c_1$. The genlex mechanism must therefore perform the dual local move: replace the lexicographically smallest possible successor step by the lexicographically largest possible predecessor step, while preserving validity of the strict inequalities.

In the lexicographic algorithm, feasibility of increasing $c_j$ depends on the gap condition $c_j+1<c_{j+1}$, with sentinel $c_{t+1}=n$. Reversing the order replaces “increase with minimal suffix” by “decrease with maximal suffix”. The dual feasibility condition is therefore that $c_j-1>c_{j-1}$, with sentinel $c_0=-1$. This condition guarantees that lowering $c_j$ by $1$ preserves strict monotonicity with its left neighbor.

The correct choice of index $j$ must be the largest index for which this decrease is possible. If a smaller index were chosen, the resulting tuple would not be the immediate predecessor in lexicographic order, since a later position would allow a larger decrease while maintaining validity, producing a strictly larger predecessor candidate.

After selecting $j$, the update is $c_j \leftarrow c_j-1$. The suffix $c_{j-1}\dots c_1$ must then be rebuilt to maximize the resulting tuple under the constraint $c_1<c_2<\cdots<c_{j-1}<c_j$. The maximal completion is obtained by forcing each position to be as large as possible while respecting strict increase, which yields the recurrence

$c_i \leftarrow c_{i+1}-1,\quad 1\le i\le j-1.$

This completion is forced uniquely, since any increase in any $c_i$ would either violate $c_i<c_{i+1}$ or would require a compensating decrease in some earlier coordinate, contradicting maximality of the predecessor.

Initialization must begin at the lexicographically largest combination in $C_{st}$, namely

$c_t=n-1,\quad c_{t-1}=n-2,\quad \dots,\quad c_1=n-t.$

The algorithm is therefore:

Initialization sets $c_i \leftarrow n-t+i-1$ for $1\le i\le t$.

Visit outputs $c_t\dots c_1$.

Find $j$ as the largest index such that $c_j-1>c_{j-1}$ with $c_0=-1$.

If no such $j$ exists, terminate.

Otherwise perform $c_j \leftarrow c_j-1$, then set

$c_i \leftarrow c_{i+1}-1\quad (1\le i<j),$

and return to Visit.

This algorithm generates $C^R_{st}$ because each iteration produces the immediate predecessor in lexicographic order on strictly increasing $t$-tuples. The initialization is the global maximum, every step preserves validity of (3), and the update rule yields the largest tuple strictly smaller than the current one, which is the defining property of lexicographic predecessor generation.

## Verification

The initial tuple $c_i=n-t+i-1$ is strictly increasing and maximal in each coordinate, so it is the greatest element of $C_{st}$ in lexicographic order.

If $c_j-1>c_{j-1}$ holds, then after setting $c_j\leftarrow c_j-1$ the inequality $c_j>c_{j-1}$ is preserved, and the reconstruction $c_i=c_{i+1}-1$ ensures strict increase throughout the suffix.

Maximality of the suffix follows because for fixed $c_j$, any admissible $c_{j-1}$ satisfies $c_{j-1}\le c_j-1$, and induction forces equality throughout the suffix, so no larger admissible tuple exists with the same prefix.

The choice of largest valid $j$ ensures that the change occurs at the rightmost position where a decrement is possible, which is exactly the condition characterizing the immediate predecessor in lexicographic order on strictly increasing sequences. This guarantees that every step advances to the next element of $C^R_{st}$ and no repetition occurs.

## Notes

The construction is the order-theoretic dual of Algorithm C under reversal of the lexicographic order on $\binom{{0,\dots,n-1}}{t}$. The forward algorithm computes immediate successors via minimal suffix completion; the reverse algorithm computes immediate predecessors via maximal suffix completion. The genlex framework is invariant under this duality, with only the local monotonicity constraint and sentinel bounds inverted.
