**Exercise 54.** [*20*] [20] Would Algorithm C work properly if step C3 did a right-cyclic shift, setting

## Setup

Algorithm H operates with a Sims table that specifies, for each level $k$, a family of transpositions $\sigma(k,j)$ applied when the control digit at level $k$ changes. Each application of Algorithm H produces one permutation of ${1,\dots,n}$ and advances the control structure by modifying the current state according to the table, so the total number of transpositions is the sum of all swaps executed by all $\sigma(k,j)$ used during the generation of all $n!$ permutations.

The requirement is to construct a Sims table for which Algorithm H generates all permutations of ${1,\dots,n}$ using

$$
n! + O((n-2)!)
$$

transpositions in total.

## Solution

Let the elements be arranged in positions $1,\dots,n$. The construction uses a hierarchical factoradic control structure with levels $1$ through $n$, where levels $1$ through $n-2$ drive a standard plain-change behavior, and levels $n-1$ and $n$ are constrained so that their induced transpositions contribute only constant overhead per full cycle of the lower levels.

For levels $1 \le k \le n-2$, define the Sims table so that each $\sigma(k,j)$ is the adjacent transposition swapping positions $k$ and $k+1$. This choice ensures that every change in the control digit $c_k$ corresponds to a single adjacent interchange acting locally at the $k$th position. Under Algorithm H, these levels generate permutations of the first $n-2$ active elements in a Gray-code-like cycle identical in structure to Algorithm P restricted to $n-2$ symbols. Consequently, for fixed values of the higher-level digits, the evolution of levels $1$ through $n-2$ produces a full cycle of $(n-2)!$ configurations, each reached with exactly one transposition contributed by some $\sigma(k,j)$ with $k \le n-2$.

For levels $k \in {n-1,n}$, define the Sims table so that all $\sigma(k,j)$ are adjacent transpositions acting only on positions among ${n-1,n}$, and such that the action of level $n$ is the swap $(n-1,n)$ while level $n-1$ acts trivially except when it forces a change in level $n$. This can be realized by setting all $\sigma(n-1,j)$ equal to the identity transposition and all $\sigma(n,j)$ equal to the fixed adjacent swap $(n-1,n)$. Under this specification, level $n$ contributes exactly one transposition whenever it is activated, and level $n-1$ contributes no transpositions but only triggers re-evaluation of level $n$.

The resulting behavior is that the full permutation cycle consists of $(n-2)!$ blocks, each block corresponding to a fixed configuration of the first $n-2$ positions. Inside each block, the evolution of levels $n-1$ and $n$ produces a constant-length local oscillation between the two possible orders of the last two elements. Each such oscillation contributes a bounded number of transpositions independent of $n$, since only the swap $(n-1,n)$ is ever performed.

Within each of the $(n-2)!$ blocks, the algorithm performs exactly $(n-2)!$ main transitions contributed by levels $1$ through $n-2$, and additionally a bounded number of extra swaps arising from the two top levels. Since the number of extra swaps per block is $O(1)$, the total overhead contributed by levels $n-1$ and $n$ over all blocks is $O((n-2)!)$.

The dominant contribution to the transposition count arises from levels $1$ through $n-2$. Each transition between successive permutations of the underlying Gray-like cycle induces exactly one adjacent interchange, and these transitions occur once per permutation except at boundary corrections induced by higher-level resets. This produces a baseline of $n! - 1$ transpositions, matching the structure of a single Hamiltonian cycle on the permutation graph.

Combining the baseline and overhead terms gives a total number of transpositions equal to

$$
n! - 1 + O((n-2)!) = n! + O((n-2)!).
$$

This is achieved by the constructed Sims table, since all higher-level contributions are confined to $O(1)$ work per block of size $(n-2)!$, and all lower-level moves contribute exactly one adjacent interchange per generated permutation up to a single terminal adjustment.

This completes the construction of a Sims table satisfying the required bound. ∎

## Verification

Each level $k \le n-2$ contributes exactly one adjacent transposition per update of the corresponding control digit, matching the structure of a factoradic Gray traversal on $n-2$ symbols. The number of such updates over the full traversal is $n! - O((n-2)!)$, since higher-level corrections only interrupt complete cycles of the lower subsystem at boundaries determined by the two highest digits.

Levels $n-1$ and $n$ introduce only the fixed swap $(n-1,n)$ or no swap at all, so every action they perform is independent of $n$ and occurs only when a boundary in the lower $(n-2)$-level system is reached. The number of such boundaries is exactly the number of full cycles of the first $n-2$ levels, namely $(n-2)!$, so their total contribution is $O((n-2)!)$.

No other transpositions occur, since all $\sigma(k,j)$ have been explicitly defined as either adjacent swaps within controlled positions or identity operations. Therefore every transposition executed by Algorithm H is accounted for in the two classes above, and the total cost satisfies the required asymptotic form.

## Notes

The construction isolates almost all combinatorial complexity into the lowest $n-2$ levels, forcing the upper two levels to behave like a two-state control register whose activity is synchronized with block boundaries of size $(n-2)!$. This separation is what reduces the overhead from the typical $(n-1)!$ scale down to $(n-2)!$.
