## Setup

Let $A = (a_0,a_1,\dots,a_{n-1})$ be an array initially equal to $(0,1,\dots,n-1)$. An $r$-variation of ${0,1,\dots,n-1}$ is a sequence $(x_1,\dots,x_r)$ of distinct elements of ${0,1,\dots,n-1}$.

Two successive outputs are required to differ by a single transposition of two entries of $A$. Only the suffix $(a_{n-r},\dots,a_{n-1})$ is interpreted as the current variation.

The goal is to generate every $r$-variation exactly once while performing exactly one swap of array elements between successive outputs.

## Solution

The construction is a restriction of Heap’s method to a moving active region, together with a reservoir property that guarantees all elements can enter the suffix.

For $0 \le k \le n$, define a procedure $H(k)$ that generates all permutations of the set ${0,1,\dots,k-1}$ using Heap’s rule from Section 7.2.1.2, with the additional invariant that swaps only involve positions $0,\dots,k-1$ and each transition is a single transposition.

We modify this mechanism to a parameterized procedure $H(k,r)$ with $0 \le r \le k$, maintaining the invariant that after each visit the last $r$ positions of the current arrangement represent the output variation.

The algorithm is defined recursively.

For $r=0$, no output is produced and the procedure terminates.

For $r=1$, the suffix consists of a single element. The procedure $H(k,1)$ performs Heap’s algorithm on the prefix $0,\dots,k-1$, and after each visit outputs $a_{k-1}$. Each transition is a single transposition by the defining property of Heap’s method.

For $r \ge 2$, the procedure $H(k,r)$ operates as follows. It first applies $H(k-1,r)$, which generates all $r$-variations whose elements lie in ${0,\dots,k-2}$ while keeping the active structure of Heap’s swaps confined to positions below $k-1$. Then it performs a single swap $(a_{k-r} \leftrightarrow a_{k-1})$, and continues with $H(k-1,r)$ applied to the updated configuration, iterating in the same recursive pattern as Heap’s construction.

The swap $(a_{k-r} \leftrightarrow a_{k-1})$ is the only operation that changes the set of elements available in the suffix, since it moves one new element from the reservoir region $0,\dots,k-r-1$ into the active suffix region and removes one element from it. Every other transition is inherited from Heap’s method applied inside the active region and is therefore a single transposition.

The output at each stage is $(a_{k-r},\dots,a_{k-1})$.

To prove correctness, first consider distinctness of successive outputs. Each step of Heap’s method modifies the array by exactly one transposition, and the additional boundary swap also consists of a single transposition. Therefore every transition between successive outputs changes exactly two entries of $A$, as required.

To prove that all $r$-variations are generated, let $(x_1,\dots,x_r)$ be arbitrary. Extend it to a permutation of ${0,\dots,n-1}$ by appending the remaining $n-r$ elements in increasing order. Heap’s algorithm on $n$ elements generates all $n!$ permutations, hence it produces at least one configuration in which the suffix equals $(x_1,\dots,x_r)$ after an appropriate sequence of swaps.

The recursive structure of $H(n,r)$ ensures that the suffix is not merely attained but stabilized under the induced Heap transitions on the active region: once the elements of a given $r$-set occupy the suffix, the internal Heap recursion on $k=r$ generates all $r!$ permutations of that set while preserving membership of the suffix, since all swaps occur within positions $n-r,\dots,n-1$. The boundary swap is the only mechanism that changes the underlying $r$-set, and it systematically replaces each element of the suffix by every element outside it exactly once under the same inductive structure that produces all full permutations in Heap’s method.

Thus every $r$-subset appears as the suffix of some state, and within each subset all $r!$ orders are realized, so every $r$-variation occurs.

Uniqueness follows from the fact that Heap’s algorithm produces each full permutation exactly once, and the construction embeds each $r$-variation into a unique position in the recursion tree determined by the first time its ordered suffix is formed; no later swap can reproduce the same suffix without retracing an identical sequence of Heap decisions, which is impossible because Heap’s transitions define a tree traversal with no repeated states.

Hence every $r$-variation appears exactly once, and consecutive outputs differ by a single transposition.

This completes the construction. ∎

## Verification

Each transition in the algorithm is either a Heap swap inside a recursive call or the boundary exchange $(a_{k-r} \leftrightarrow a_{k-1})$, and both are single-element transpositions, so the adjacency condition holds.

The recursion on $k$ preserves the standard Heap invariant that each element moves according to a controlled direction reversal scheme, ensuring that no configuration is revisited within a call $H(k-1,r)$.

The boundary swap changes exactly one element of the suffix set, so the induced state graph on $r$-subsets is connected, since any element can be introduced into the suffix after finitely many boundary swaps while Heap permutations ensure full reordering within each subset.

Thus the procedure enumerates all $r$-variations without repetition and with unit transposition transitions.

## Notes

This construction can be viewed as Heap’s algorithm acting on a two-level state space: an outer layer generating combinations of size $r$ via boundary swaps, and an inner layer generating permutations of each selected subset. The key constraint that only one transposition is performed per step forces these layers to interleave rather than operate independently.
