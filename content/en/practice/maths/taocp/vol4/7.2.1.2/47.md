## Solution

Let Algorithm X generate permutations by building prefixes $a_1,\dots,a_k$ subject to the constraints $t_j(a_1,\dots,a_j)$ for $1 \le j \le k$. For each $k$, let $N_k$ be the number of prefixes $a_1,\dots,a_k$ that satisfy all these constraints. Thus $N_k$ counts the number of nodes at depth $k$ in the implicit search tree of Algorithm X, and $N_0$ is the number of valid empty prefixes, hence $N_0=1$.

Each execution of Algorithm X corresponds to a traversal of this tree in depth-first fashion. Every time a prefix of length $k$ is valid, the algorithm performs the local test associated with level $k$ exactly once for that prefix, and every such prefix contributes exactly one occurrence of the step that verifies whether the current prefix satisfies $t_k(a_1,\dots,a_k)$.

Hence the step at level $k$ that checks the condition $t_k(a_1,\dots,a_k)$ is executed exactly $N_k$ times, since it is executed once for each valid prefix of length $k$ and no other prefixes of length $k$ are ever visited.

The transition from level $k-1$ to level $k$ occurs whenever a valid prefix of length $k-1$ is extended in all possible ways allowed by the algorithm. Each valid prefix of length $k-1$ is entered exactly once, and from it the algorithm attempts extensions until all resulting valid prefixes of length $k$ are generated. The number of successful extensions from level $k-1$ to level $k$ is therefore exactly the number of nodes at depth $k$, namely $N_k$. Consequently, the total number of times the extension mechanism at level $k-1$ is activated equals $N_{k-1}$, since each valid prefix of length $k-1$ is processed once.

The rejection or backtracking actions at level $k$ occur precisely when an attempted extension of a valid prefix of length $k-1$ fails to produce a valid prefix of length $k$. Each prefix of length $k-1$ generates some number of failed attempts equal to the number of non-viable choices at that level, and these failures account exactly for the difference between all attempted extensions and successful ones. Since every prefix of length $k-1$ produces a fixed number of total trials distributed into successful and unsuccessful outcomes, the total number of rejection steps at level $k$ is determined by the discrepancy between the number of attempted extensions and the number of valid extensions, hence it is expressed entirely in terms of $N_{k-1}$ and $N_k$.

Specifically, every valid prefix of length $k-1$ induces a full cycle of attempts at level $k$, so the total number of extension attempts at level $k$ is $N_{k-1}$. Among these, exactly $N_k$ succeed, since each successful attempt corresponds to a distinct valid prefix of length $k$. Therefore the number of failures, and hence the number of executions of the failure or backtracking step at level $k$, is

$$
N_{k-1} - N_k.
$$
