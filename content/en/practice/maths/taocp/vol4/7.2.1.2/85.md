[Section 7.2.1.2: Generating All Permutations](../)

**Exercise 85.** &#9654; [*25*] [25] Assume that n is small enough that n! fits in a computer word. What’s a
good way to convert a given permutation α = a1 . . . an of {1, . . . , n} into an integer
k = r(α) in the range 0 ≤k < n!? Both functions k = r(α) and α = r[−1](k) should
be computable in only O(n) steps.

## Solution

Let $\alpha = a_1 a_2 \dots a_n$ be a permutation of ${1,2,\dots,n}$. The standard way to encode permutations in a mixed-radix system compatible with Section 7.2.1.2 is to use its inversion table, but written in the forward direction so that it can be accumulated into a single integer in factorial number system form.

For each position $j$ define

$$
c_j = \#\{\,k : j < k \le n,\ a_k < a_j\,\}.
$$

This is exactly the inversion table definition from Section 5.1.1–7 specialized to the symbol $j$ in Knuth’s formulation of permutations as a sequence of distinct elements.

Each $c_j$ satisfies $0 \le c_j < n-j+1$, since among the elements to the right of position $j$ there are exactly $n-j$ positions. The vector $(c_1,c_2,\dots,c_n)$ is a mixed radix representation with radices $(n,n-1,\dots,1)$.

Define the rank function

$$
r(\alpha) = \sum_{j=1}^{n} c_j \cdot (n-j)!.
$$

Since $0 \le c_j < n-j+1$, this is exactly the factorial number system representation of an integer in $[0,n!)$, hence $0 \le r(\alpha) < n!$.

Correctness follows from uniqueness of the inversion table representation. Given $\alpha$, the values $c_j$ uniquely determine $\alpha$, and conversely every sequence satisfying $0 \le c_j < n-j+1$ arises from exactly one permutation. The mapping between mixed radix digits and integers in factorial base is bijective, so $r(\alpha)$ is well-defined and injective, hence a permutation of $n!$ values.

To compute $r(\alpha)$ in $O(n)$ steps under the word-RAM assumptions implicit in the exercise, maintain a dynamic representation of the set of unused symbols. Let $U$ initially be ${1,2,\dots,n}$. At step $j$, we must compute $c_j$, the number of elements in $U$ that are less than $a_j$.

Assume a word-sized bitset representation of $U$ together with a constant-time rank operation $\operatorname{rank}(x,U)$ returning the number of set bits in $U$ strictly less than $x$. Under this assumption, each $c_j$ is computed as

$$
c_j = \operatorname{rank}(a_j,U).
$$

After computing $c_j$, we remove $a_j$ from $U$.

The rank is accumulated as

$$
r \leftarrow r \cdot (n-j+1) + c_j
$$

which is equivalent to Horner’s form of $\sum c_j (n-j)!$ and avoids explicit factorial precomputation.

Each step performs $O(1)$ word operations, so the total time is $O(n)$.

Now consider the inverse mapping. Given an integer $k$ with $0 \le k < n!$, we construct $\alpha = r^{-1}(k)$.

Maintain a set $U = {1,2,\dots,n}$. For each position $j$ from $1$ to $n$, define

$$
b_j = (n-j)!.
$$

At step $j$, compute

$$
q = \left\lfloor \frac{k}{b_j} \right\rfloor,
\qquad
k \leftarrow k \bmod b_j.
$$

The value $q$ satisfies $0 \le q \le n-j$, and we select $a_j$ as the $(q+1)$-th smallest element of $U$.

Under the same word-RAM assumption with a constant-time order-statistics operation $\operatorname{select}(U,t)$ returning the $t$-th smallest element of $U$, we set

$$
a_j = \operatorname{select}(U,q+1),
$$

and remove $a_j$ from $U$.

Correctness follows from the factorial number system representation. At step $j$, the interval of size $(n-j)!$ partitions the range of possible suffix permutations according to the first unused element, and $q$ selects the correct block corresponding to the next element in the permutation. Inductively, the residual value of $k$ encodes the remaining suffix in the same representation.

Each step performs a constant number of arithmetic operations and one selection and deletion from $U$, so the total complexity is $O(n)$.

This completes the construction of inverse bijections $r$ and $r^{-1}$, both computable in $O(n)$ steps under word-RAM operations supporting constant-time rank and select on a set of size $n$. ∎
