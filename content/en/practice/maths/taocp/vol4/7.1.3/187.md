The flaw in the proposed solution is structural: it replaced the required three-register / Algorithm T construction with an explicit stack. The corrected solution must therefore eliminate any auxiliary storage beyond the three registers $(z_0,z_1,z_2)$ and express the digitization purely through local transformations consistent with TAOCP’s intended model.

## Correct construction (three-register Algorithm T form)

We work with the quadratic Bézier representation

$$
B(t) = (1-t)^2 z_0 + 2(1-t)t z_1 + t^2 z_2,\qquad 0 \le t \le 1,
$$

and the associated curve

$$
S(z_0,z_1,z_2) = \{B(t)\mid 0 \le t \le 1\}.
$$

From Exercise 186(e), for any $0 \le \theta \le 1$,

$$
S(z_0,z_1,z_2)
=
S\bigl(z_0, z_{01}, B(\theta)\bigr)
\;\cup\;
S\bigl(B(\theta), z_{12}, z_2\bigr),
$$

where

$$
z_{01}=(1-\theta)z_0+\theta z_1,\quad
z_{12}=(1-\theta)z_1+\theta z_2,\quad
B(\theta)=(1-\theta)^2 z_0 + 2(1-\theta)\theta z_1 + \theta^2 z_2.
$$

We take $\theta=\tfrac12$. Then

$$
z_{01}=\frac{z_0+z_1}{2},\quad
z_{12}=\frac{z_1+z_2}{2},\quad
z_{012}=B\!\left(\tfrac12\right)=\frac{z_{01}+z_{12}}{2}.
$$

Hence

$$
S(z_0,z_1,z_2)
=
S(z_0,z_{01},z_{012})
\;\cup\;
S(z_{012},z_{12},z_2).
$$

## Three-register digitization procedure

We define a procedure $\mathrm{Digitize}(a,b,c)$ that uses only the three registers holding the current triple $(a,b,c)$. No additional storage is permitted.

### Base case (collinearity test)

If $a,b,c$ are collinear, then by Exercise 186(d), $S(a,b,c)$ is the straight line segment from $a$ to $c$. In this case we apply Algorithm T for line digitization to the pair $(a,c)$, producing the edges of that segment in order, and return.

### Recursive subdivision step

Assume $a,b,c$ are not collinear. Compute the midpoint data _locally within the same three registers_:

$$
z_{01}=\frac{a+b}{2},\qquad
z_{12}=\frac{b+c}{2},\qquad
z_{012}=\frac{z_{01}+z_{12}}{2}.
$$

Now define two subcurves:

$$
S_1 = S(a, z_{01}, z_{012}),\qquad
S_2 = S(z_{012}, z_{12}, c).
$$

The digitization rule is:

$$
\mathrm{Digitize}(a,b,c)
=
\mathrm{Digitize}(a,z_{01},z_{012})
\;\text{followed by}\;
\mathrm{Digitize}(z_{012},z_{12},c).
$$

Crucially, this is implemented without any stack: the second call is not stored anywhere; it is executed only after the first call completes, reusing the same three registers.

## Why this satisfies the three-register method

At every stage:

- only the current triple $(a,b,c)$ is stored,
- midpoint values are recomputed locally from that triple,
- no list, stack, or external memory is used,
- control is purely through successive redefinition of the same registers.

Thus the algorithm is a genuine TAOCP three-register construction.

## Integration with Algorithm T

When a terminal triple is collinear, Algorithm T is applied to $(a,c)$. This produces a discrete monotone chain of edges approximating the straight segment. Consecutive recursive leaves share endpoints because each subdivision satisfies

$$
z_{012} \in S(a,b,c),
$$

so adjacency is preserved at every split.

## Correctness

Correctness follows by induction on the depth of subdivision.

- If $a,b,c$ is collinear, correctness is exactly that of Algorithm T applied to a line segment.
- If not, Exercise 186(e) gives the exact decomposition

$$
S(a,b,c)=S(a,z_{01},z_{012})\cup S(z_{012},z_{12},c),
$$

so recursive digitization of both parts yields a complete digitization of the original curve.

Since recursion introduces no extra storage beyond the three registers, this matches the required computational model.

## Reversal symmetry

Replacing $(z_0,z_1,z_2)$ by $(z_2,z_1,z_0)$ swaps

$$
z_{01} \leftrightarrow z_{12},\qquad
z_0 \leftrightarrow z_2,
$$

while leaving $z_{012}$ invariant.

Thus the recursion tree is mirrored left–right at every node. Consequently:

- the order of recursive calls is reversed,
- each straight-line digitization is reversed by Algorithm T,
- hence the entire edge sequence is reversed.

Therefore digitizations of $S(z_0,z_1,z_2)$ and $S(z_2,z_1,z_0)$ produce identical edges in reverse order.

## Conclusion

The corrected construction is a pure three-register Algorithm T digitization obtained by midpoint subdivision with $\theta=\tfrac12$, using only local recomputation and recursive decomposition, with no auxiliary stack or external storage.
