## Setup

Let $X$ be an $M \times N$ bitmap with pixel set ${(i,j) \mid 1 \le i \le M,, 1 \le j \le N}$. Adjacency is rookwise: $(i,j)$ and $(i',j')$ are adjacent when $|i-i'|+|j-j'|=1$.

A connected component means a rookwise connected component of either black or white pixels, as in the constructions leading to the surroundedness tree in (162). The surroundedness tree has one node for each white connected component, and a directed edge from a white component $W_1$ to $W_2$ when $W_2$ is immediately enclosed inside a region determined by $W_1$ via the black connectivity structure described in (162).

The task is to construct this tree while scanning $X$ row by row in increasing order of $i$, using only information from the current and previous rows.

We maintain the standard sweep-line invariant: after processing rows $1,\dots,i$, every connected component of the induced subgraph on these rows is represented by a live structure encoding its boundary intersection with row $i$.

## Solution

During the scan, each row is decomposed into maximal contiguous monochromatic intervals. Each such interval belongs to exactly one connected component of its color when restricted to rows $\le i$, except that components may merge via vertical adjacency with intervals in row $i-1$.

We maintain a union-find structure over active component identifiers. Each active component $C$ stores two pieces of data: its color (black or white), and the set of indices $j$ such that $(i,j)$ is currently part of $C$ or adjacent to it from row $i-1$.

When processing row $i$, we first create singleton components for each maximal monochromatic run in row $i$. We then perform unions between a new interval and any interval in row $i-1$ that shares a column adjacency, namely when a pixel in row $i$ is vertically adjacent to a pixel already assigned to a component from row $i-1$.

This produces the connected components of the subgraph induced by rows $\le i$ restricted to those components that intersect row $i$.

To maintain the surroundedness structure, each component $C$ carries an active boundary signature consisting of the set of components in row $i-1$ with which it had vertical adjacency. When a component in row $i-1$ fails to appear in row $i$, its boundary is considered closed at level $i-1$, and the component is finalized.

For each finalized white component $W$, we determine its parent in the tree by inspecting the black components that bordered it during its lifetime. The parent is the unique black component $B$ that shares a boundary with $W$ and remains connected around $W$ in the sense of (162), namely that any path leaving $W$ must pass through $B$. This is computed incrementally: whenever a white component $W$ first becomes adjacent to a black component $B$, we record $B$ as a candidate parent; unions of black components ensure that this parent pointer remains well defined under merging.

When a white component $W$ is finalized, we attach it as a child of its recorded parent black region in the evolving tree structure. If multiple candidate parents exist due to merging of black components during the scan, union-find guarantees that they lie in the same black component at finalization, so the parent is unique.

The algorithm outputs a node for each white component at the moment it is closed, and all edges to its parent are fixed at that time. The scan continues until row $M$ is processed, after which all remaining active components are finalized.

Correctness follows from the invariant that at every stage $i$, the union-find structure exactly represents connected components in the subgraph induced by rows $\le i$, since all horizontal adjacencies are captured within rows and all vertical adjacencies are explicitly unioned between consecutive rows. The boundary tracking ensures that a component is finalized exactly when it no longer intersects any future row, which is equivalent to it being separated from the unprocessed portion of the bitmap. The parent assignment matches the definition of surroundedness because any separation of a white region from the exterior must occur through a black region already connected along the boundary, and this connectivity is preserved under the union operations.

Thus the procedure constructs exactly the surroundedness tree while scanning each row once and maintaining only local boundary information.

## Verification

Each union operation corresponds to an actual rook adjacency in $X$, so no spurious connectivity is introduced. Every rookwise adjacency is eventually encountered either within a row or between two consecutive rows, so no genuine adjacency is missed.

A component is finalized only when no pixel belonging to it appears in any later row. At that point, no further unions involving that component are possible, so its connectivity class in the full bitmap is fixed.

For each white component $W$, every black region that can separate it from the exterior must be adjacent to it at some row during the scan. The union-find invariant ensures that all such black regions belong to a single merged black component at the time of finalization, so the parent pointer is well defined and unique.

Since each white component is created once and finalized once, and every adjacency is processed exactly at the row where it occurs, the constructed graph is acyclic and connects each node to exactly one parent, hence it is a tree.

This completes the proof. ∎
