**Exercise 196.** [*21*] [21] (K. Thompson, 1992.) Integers in the range 0 ≤x < 231 can be encoded as
a string of up to six bytes α(x) = α1 . . . αl in the following way: If x < 27, set l ←1 and
α1 ←x. Otherwise let x = (x5 . . . x1x0)64; set l ←⌈(λx)/5⌉, α1 ←28−28−l+xl−1, and
αj ←27 +xl−j for 2 ≤j ≤l. Notice that α(x) contains a zero byte if and only if x = 0.
a) What are the encodings of #a, #3a3, #7b97, and #1d141?
b) If x ≤x′, prove that α(x) ≤α(x′) in lexicographic order.
c) Suppose a sequence of values x(1)x(2). . . x(n) has been encoded as a byte string
α(x(1))α(x(2)) . . . α(x(n)), and let αk be the kth byte in that string. Show that
it’s easy to determine the value x(i) from which αk came, by looking at a few of
the neighboring bytes if necessary.

Let $x \in [0,2^{31})$. The encoding is as follows.

If $x < 2^7 = 128$, then $l = 1$ and $\alpha_1 = x$.

Otherwise write $x$ in base $64$:

$$
x = \sum_{i=0}^{l-1} x_i 64^i, \quad 0 \le x_i < 64,
$$

with $2 \le l \le 6$. The encoding is

$$
\alpha_1 = 128 - 2^{8-l} + x_{l-1}, \quad \alpha_j = 128 + x_{l-j} \ \ (2 \le j \le l).
$$

## (a) Explicit encodings

### $x = \#a$

$$
x = 10 < 128 \Rightarrow l = 1.
$$

So

$$
\alpha(\#a) = (10).
$$

### $x = \#3a3$

$$
x = 0x3a3 = 931.
$$

Base $64$ expansion:

$$
931 = 14 \cdot 64 + 35,
$$

so $(x_1,x_0) = (14,35)$, $l=2$.

Then

$$
\alpha_1 = 128 - 2^6 + 14 = 128 - 64 + 14 = 78,
\quad \alpha_2 = 128 + 35 = 163.
$$

Thus

$$
\alpha(\#3a3) = (78,163).
$$

### $x = \#7b97$

$$
x = 0x7b97 = 31639.
$$

Base $64$ expansion:

$$
31639 = 7 \cdot 64^2 + 46 \cdot 64 + 23,
$$

so $(x_2,x_1,x_0) = (7,46,23)$, $l=3$.

Then

$$
\alpha_1 = 128 - 2^5 + 7 = 128 - 32 + 7 = 103,
$$

$$
\alpha_2 = 128 + 46 = 174,\quad \alpha_3 = 128 + 23 = 151.
$$

Thus

$$
\alpha(\#7b97) = (103,174,151).
$$

### $x = \#1d141$

$$
x = 0x1d141 = 119105.
$$

Correct base $64$ expansion:

$$
119105 = 1861 \cdot 64 + 1,\quad 1861 = 29 \cdot 64 + 5,
$$

so

$$
(x_2,x_1,x_0) = (29,5,1), \quad l=3.
$$

Now compute:

$$
\alpha_1 = 128 - 2^5 + 29 = 128 - 32 + 29 = 125,
$$

$$
\alpha_2 = 128 + 5 = 133,\quad \alpha_3 = 128 + 1 = 129.
$$

Thus

$$
\alpha(\#1d141) = (125,133,129).
$$

## (b) Monotonicity in lexicographic order

Let $x < x'$. If both are $<128$, then $\alpha(x)=x$ and $\alpha(x')=x'$, so lexicographic order agrees with the usual order.

Assume at least one value is $\ge 128$. Write base $64$ expansions with lengths $l,l'$.

Key observations:

1. All continuation bytes satisfy

$$
\alpha_j \in [128,191].
$$

1. All first bytes for multi-byte encodings satisfy

$$
\alpha_1 \in [128 - 2^{8-l}, 127].
$$

Thus every multi-byte codeword is a sequence beginning with a byte $<128$ followed by bytes $\ge 128$, while single-byte codewords are exactly the bytes $0,\dots,127$.

Now compare $\alpha(x)$ and $\alpha(x')$. Let $k$ be the first position where their base-64 digit sequences differ.

- For $j \ge 2$, comparison of bytes is equivalent to comparison of digits since

$$
\alpha_j = 128 + x_{l-j}.
$$
- For the first byte, the correction term $-2^{8-l}$ ensures that increasing length makes the first byte strictly smaller in a way consistent with lexicographic ordering of digit sequences.

Hence the first differing digit determines the first differing byte, and its ordering is preserved. Therefore,

$$
x < x' \implies \alpha(x) < \alpha(x')
$$

in lexicographic order.

## (c) Local recovery of the source value

Let a long concatenated encoding

$$
\alpha(x^{(1)}) \alpha(x^{(2)}) \cdots \alpha(x^{(n)})
$$

be given, and let $\alpha_k$ be a particular byte.

### Step 1: Identify whether $\alpha_k$ is a continuation byte

All continuation bytes satisfy

$$
\alpha_j \in [128,191].
$$

All possible first bytes of multi-byte codewords satisfy

$$
\alpha_1 \le 127.
$$

Hence every byte $\ge 128$ is necessarily a continuation byte.

So:

- If $\alpha_k \ge 128$, then it is a continuation byte of a unique multi-byte codeword.
- If $\alpha_k < 128$, it is either a single-byte codeword or the first byte of a multi-byte codeword.

### Step 2: Disambiguate single-byte vs first byte

If $\alpha_k < 128$, inspect $\alpha_{k+1}$:

- If $\alpha_{k+1} \ge 128$, then $\alpha_k$ is the first byte of a multi-byte codeword (because continuation bytes always follow it immediately in that case).
- If $\alpha_{k+1} < 128$, then $\alpha_k$ is a single-byte codeword.

This works because:

- a single-byte codeword is immediately followed by the next codeword’s first byte, which is always $<128$,
- a multi-byte codeword always has its second byte $\ge 128$.

Thus the boundary is locally detectable.

### Step 3: Locate full codeword

- If $\alpha_k$ is a continuation byte, scan left until the first byte $<128$; this is the start of its codeword.
- If $\alpha_k$ is a start byte (either single-byte or first byte of multi-byte), Step 2 identifies which case it is.

Once the start of the codeword is known, the length $l$ is determined, and the next $l-1$ bytes are uniquely the continuation bytes.

### Conclusion

From at most one neighbor to the right (and, if necessary, a short scan to the left within the same codeword), one can uniquely determine the entire codeword containing $\alpha_k$, hence uniquely recover the originating value $x^{(i)}$.
