**Exercise 62.** &#9654; [*22*] Let $N = 2^d$. We can obviously represent a permutation $\pi$ of $\{0, 1, \ldots, N-1\}$ by storing a table of $N$ numbers, $d$ bits each. With this representation we have instant access to $y = x\pi$, given $x$; but it takes $\Omega(N)$ steps to find $x = y\pi^-$ when $y$ is given.

Show that, with the same amount of memory, we can represent an arbitrary permutation in such a way that $x\pi$ and $y\pi^-$ are both computable in $O(d)$ steps.

## Solution

Let $N = 2^d$ and let $\pi$ be a permutation of ${0,1,\ldots,N-1}$. The goal is to encode $\pi$ using $O(Nd)$ bits, the same order as an explicit table of $d$-bit values, while enabling computation of both $x\pi$ and $y\pi^{-}$ in $O(d)$ steps.

The construction uses a Beneš network on $N$ wires. A Beneš network for $N = 2^d$ consists of $2d-1$ stages of $N/2$ two-input switches arranged so that each stage connects wires in disjoint pairs. Each switch has one control bit specifying whether it connects its inputs straight or crossed. A configuration of all switch bits determines a bijection on ${0,1,\ldots,N-1}$.

A standard property of the Beneš network is that it is rearrangeable: for every permutation $\pi$ of ${0,1,\ldots,N-1}$, there exists an assignment of switch settings that realizes $\pi$. This follows by induction on $d$ using the recursive decomposition of the network into two smaller Beneš networks of size $2^{d-1}$, together with a middle stage that routes elements between the two halves. The induction constructs routing so that any permutation is decomposed into subpermutations on the top and bottom halves, with appropriate crossing choices in the middle stage.

To represent $\pi$, store all switch settings of a Beneš network realizing $\pi$. The network contains $(2d-1)\cdot N/2$ switches, each storing one bit. The total storage is therefore $(2d-1)N/2$ bits, which is $\Theta(Nd)$ bits, matching the stated memory constraint up to a constant factor.

To compute $x\pi$, place a token at input wire $x$ and simulate its traversal through the network. At each of the $2d-1$ stages, the token passes through exactly one switch and the next wire index is determined by that switch's stored bit. Since each stage performs a constant-time local update, the total number of steps is proportional to the number of stages, namely $2d-1$, hence $O(d)$.

To compute $y\pi^{-}$, use the same network in reverse. The inverse permutation is obtained by running the same switch configuration from outputs back to inputs. Each stage again contributes one constant-time update, and the computation traverses $2d-1$ stages, giving $O(d)$ time.

Both directions use the same stored representation, since the inverse of a fixed switch configuration is obtained by reversing the order of stages without changing switch bits. Therefore the representation supports forward and inverse evaluation within the required time bound.

This completes the proof. ∎
