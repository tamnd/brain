**Exercise 137.** [*21*] [21] Show that the operation in exercise 136(c) is simpler with packed vectors
like (131) than with the unpacked form (130).

## Solution

Let the set ${a,b,c}$ be encoded by fixed binary codes as in the representation (130), and let a packed vector as in (131) represent a sequence of symbols by storing one code per fixed-size field in a single machine word. In this representation, a vector of length $m$ occupies one word partitioned into $m$ independent bit-fields, and every operation on ${a,b,c}$ is applied coordinatewise by acting uniformly on all fields.

The operation in exercise 136(c) is defined by the table

$$
\begin{array}{c|ccc}
\cdot & a & b & c \\
\hline
a & a & b & a \\
b & a & a & c \\
c & a & b & c
\end{array}
$$

so for each fixed coordinate, the result depends only on the pair of input symbols in that coordinate and is independent of all other coordinates.

In the unpacked representation (130), each coordinate is stored as an isolated symbol, so evaluating the operation requires selecting two input symbols, applying the above table lookup, and writing the result back for each coordinate separately. This forces a separate constant-time decision process per coordinate, hence a loop over all coordinates with one table evaluation per entry.

In the packed representation (131), each word contains all coordinates simultaneously, and each coordinate occupies a fixed set of bits. The operation on ${a,b,c}$ induces a Boolean function on these bit-fields that depends only on the local two-bit codes of the inputs. Since the encoding assigns distinct bit patterns to $a$, $b$, and $c$, the table entries correspond to a fixed Boolean combination of the input bits applied uniformly across all fields. Therefore the entire vector transformation is realized by a constant number of word-level operations, each acting on all coordinates in parallel, without per-coordinate branching or per-coordinate table lookup.

Thus the packed representation replaces a coordinatewise evaluation of the table, requiring a number of elementary steps proportional to the vector length, by a bounded number of word operations independent of the length. This shows that the operation in 136(c) is simpler in packed form than in unpacked form. ∎
