**Exercise 84.** [*25*] Given $n$-bit numbers $z = (z_{n-1} \ldots z_1 z_0)_2$ and $\chi = (\chi_{n-1} \ldots \chi_1 \chi_0)_2$, explain how to calculate the "stretched" quantities $\overline{z} = \overline{\chi} = (\overline{z}_{(n-1)\leftarrow\chi} \ldots \overline{z}_{1\leftarrow\chi} \overline{z}_{0\leftarrow\chi})$ and

$z \to \chi = (z_{l(n-1)\to\chi} \ldots z_{1\to\chi} z_{0\to\chi})_2$, where

$$j \leftarrow \chi = \max\{k \mid k \le j \text{ and } \chi_k = 1\}, \qquad j \to \chi = \min\{k \mid k \ge j \text{ and } \chi_k = 1\};$$

we let $z_{j\to\chi} = 0$ if $\chi_k = 0$ for $0 \le k \le j$, and $z_{j\to\chi} = 0$ if $\chi_k = 0$ for $n \ge k \ge j$. For example, if $n = 11$ and $\chi = (01101110010)_2$, then $z \leftarrow \chi = (z_9 z_9 z_8 z_6 z_6 z_5 z_4 z_1 z_1 z_0)_2$ and $z \to \chi = (0z_9 z_8 z_8 z_6 z_5 z_4 z_4 z_4 z_1 z_1)_2$.

The previous solution fails because it treats the operation as bitwise propagation, while the definition is fundamentally a **selection of whole values indexed by a control mask**, not a bitwise merge. The correct construction must preserve the identity of the chosen index $k$, not mix contributions from multiple indices.

## 1. Correct interpretation of the operation

Let indices be $j = 0,1,\dots,n-1$.

Define the set of active positions of $\chi$:

$$
S = \{k \mid \chi_k = 1\}.
$$

For each $j$, the definitions are:

- Left stretch:

$$
j \leftarrow \chi = \max\{k \in S \mid k \le j\},
$$

if no such $k$ exists, the result is undefined and contributes $0$.

- Right stretch:

$$
j \to \chi = \min\{k \in S \mid k \ge j\},
$$

if no such $k$ exists, the result is $0$.

The output is then:

$$
(z \leftarrow \chi)_j = z_{j \leftarrow \chi}, \qquad (z \to \chi)_j = z_{j \to \chi},
$$

with the convention that missing indices yield $0$.

This is an **index-selection problem**, not a bitwise transformation.

## 2. Construction of $z \leftarrow \chi$

### Step 1: compute predecessor array

Define an auxiliary array $p_j$ by

$$
p_j =
\begin{cases}
\max\{k \le j \mid \chi_k = 1\}, & \text{if such } k \text{ exists}, \\
-1, & \text{otherwise}.
\end{cases}
$$

This can be computed by a single left-to-right scan:

- Initialize $p_{-1} = -1$.
- For each $j$:

$$
p_j =
\begin{cases}
j, & \chi_j = 1, \\
p_{j-1}, & \chi_j = 0.
\end{cases}
$$

### Step 2: generate output

Now define:

$$
(z \leftarrow \chi)_j =
\begin{cases}
z_{p_j}, & p_j \ne -1, \\
0, & p_j = -1.
\end{cases}
$$

This exactly implements “copy the value of the nearest previous 1 in $\chi$”.

## 3. Construction of $z \to \chi$

### Step 1: compute successor array

Define:

$$
q_j =
\begin{cases}
\min\{k \ge j \mid \chi_k = 1\}, & \text{if such } k \text{ exists}, \\
n, & \text{otherwise}.
\end{cases}
$$

Compute it by a right-to-left scan:

- Initialize $q_n = n$.
- For $j = n-1, \dots, 0$:

$$
q_j =
\begin{cases}
j, & \chi_j = 1, \\
q_{j+1}, & \chi_j = 0.
\end{cases}
$$

### Step 2: generate output

$$
(z \to \chi)_j =
\begin{cases}
z_{q_j}, & q_j \ne n, \\
0, & q_j = n.
\end{cases}
$$

This correctly assigns each position the value of the next 1 in $\chi$.

## 4. Why the previous OR construction is wrong

The previous solution attempted to compute:

$$
\bigvee_{i \le j} (z_i \,\&\, \chi_i),
$$

but this destroys positional information.

The operation:

- mixes bits from multiple different $z_i$,
- cannot recover a single selected index $p_j$,
- and therefore does not implement the required mapping $j \mapsto p_j$.

A concrete failure occurs whenever two different $z_i$ contribute different bits: OR merges them, while the definition requires selecting exactly one $z_i$.

Thus the earlier method computes a **bitwise union over a set**, while the problem requires a **function application through an index map**.

## 5. Final correct formulation

Let $p_j$ and $q_j$ be defined as above. Then:

$$
\boxed{
(z \leftarrow \chi)_j =
\begin{cases}
z_{p_j}, & p_j \ne -1, \\
0, & \text{otherwise},
\end{cases}
\qquad
(z \to \chi)_j =
\begin{cases}
z_{q_j}, & q_j \ne n, \\
0, & \text{otherwise}.
\end{cases}
}
$$

These constructions correctly implement the “stretched” operators as **nearest-1 index selection followed by table lookup**, preserving the full value $z_k$ without bitwise interference.
