## Setup

Let the CRC procedure (183) be defined for a fixed generator polynomial $g(x)$ over $\mathbb{F}_2$ of degree $w$, with corresponding bit mask $G$ in MMIX representation, where the coefficient of $x^w$ is $1$ and the lower coefficients occupy bits $0,\dots,w-1$.

Let a message be a sequence of bytes $B_1,\dots,B_t$, interpreted as a polynomial

$M(x)=\sum_{i=0}^{8t-1} m_i x^i \in \mathbb{F}_2[x].$

The CRC value is the remainder $R(x)$ of the division

$x^w M(x) \bmod g(x).$

Let a register $r$ store the current remainder, and let $M$ denote the fixed bit mask corresponding to the coefficients of $g(x)$ excluding $x^w$.

The task is to implement the iterative update defined in (183) on MMIX using bitwise operations.

## Solution

Write the remainder update in bitwise form. Let the current remainder be $r$. Processing one input bit $b \in {0,1}$ corresponds to forming

$r \leftarrow (r \ll 1) \oplus b,$

followed by reduction modulo $g(x)$ when the shifted-out top bit equals $1$.

Let $\operatorname{msb}(r)$ denote the coefficient of $x^w$ in the extended register after shifting. The reduction step is

$r \leftarrow r \oplus M \quad \text{if } \operatorname{msb}(r)=1.$

Thus each bit update consists of a left shift, insertion of the input bit, and conditional XOR with $M$.

Processing a byte $B$ proceeds from the most significant bit to the least significant bit. If $b_i$ is the $i$th bit of $B$, then the loop executes the same update for each $b_i$.

The MMIX implementation uses one register for the state and one for the generator mask.

```
        LOC     Data_Segment

Mask    IS      $0          ; CRC generator mask M
R       IS      $1          ; remainder register
Byte    IS      $2          ; current input byte
Bit     IS      $3          ; extracted bit
Count   IS      $4          ; bit counter

        LOC     #100

CRC     SET     R,0         ; r ← 0

LoopB   LDB     Byte,$1,0   ; load next byte from memory
        ADD     $1,$1,1     ; advance pointer

        SET     Count,8

LoopBit SL     R,R,1        ; r ← r << 1
        SRU     Bit,Byte,7  ; extract MSB of Byte
        AND     Bit,Bit,1
        SL      Byte,Byte,1 ; shift next bit into position

        XOR     R,R,Bit     ; insert input bit

        ; if top bit of (previous) shift is 1, reduce
        ; MSB test corresponds to bit shifted out of width w
        ; assume MSB is in overflow position
        BZ      Bit,NoRed
        XOR     R,R,Mask

NoRed   SUB     Count,Count,1
        PBP     Count,LoopBit

        CMP     $5,$1,$2    ; end of message test (abstract)
        PBNZ    $5,LoopB

        TRAP    0,Halt,0
```

The code assumes byte-wise streaming input from memory and maintains the CRC state in $R$.

The reduction step uses XOR with $Mask$, which encodes the polynomial coefficients of $g(x)$.

## Verification

After processing $k$ input bits, let $r_k$ denote the register content. The initialization $r_0=0$ matches the remainder of the zero polynomial.

Assume $r_k$ equals the remainder of $x^w$ times the prefix polynomial of length $k$. Processing the next bit applies the transformation

$r_{k+1} = (2r_k + b_k) \bmod g(x),$

since left shift corresponds to multiplication by $x$ and XOR insertion corresponds to addition in $\mathbb{F}_2$.

The conditional XOR with $M$ implements subtraction of $g(x)$ when the coefficient of $x^w$ becomes $1$, because

$x^w \equiv g(x) - \sum_{i=0}^{w-1} g_i x^i \pmod{g(x)},$

and over $\mathbb{F}_2$ subtraction equals addition.

Thus each step preserves the invariant that $r_k$ is the remainder of $x^w$ times the processed prefix polynomial modulo $g(x)$. After all bits are processed, the register contains the CRC remainder required by (183).

This completes the proof.
