## Solution

Assume representation (128) encodes each ternary value $x \in {0,_,1}$ as a pair

$(x^-,x^+)$

with $x^- \le x^+$, where $0=(0,0)$, $_=(0,1)$, $1=(1,1)$, and all operations on pairs are performed bitwise on the two components using the Boolean operations of Section 7.1.3. A ternary value is thus identified with the interval $[x^-,x^+]$.

In this representation, the order is componentwise:

$x \le y \iff x^- \le y^- \ \text{and}\ x^+ \le y^+,$

and conjunction is intersection of intervals:

$x \wedge y = (x^- \wedge y^-,\, x^+ \wedge y^+).$

Negation interchanges $0$ and $1$ and fixes $*$, so it acts by endpoint reversal followed by complementation:

$\neg(x^-,x^+) = (\overline{x^+},\,\overline{x^-}).$

Implication in ≈Åukasiewicz three-valued logic is determined by the condition that $x \Rightarrow y$ is the least value $z$ such that $x \wedge z \le y$. In interval form this yields

$x \Rightarrow y = (\overline{x^+} \vee y^-,\, \overline{x^-} \vee y^+).$

### Negation, possibility, necessity, equivalence

Let $x=(x^-,x^+)$.

The possibility operator is defined by

$\Diamond x = \neg x \Rightarrow x.$

Substituting the representation of negation gives $\neg x=(\overline{x^+},\overline{x^-})$. Applying the implication formula,

$\Diamond x = (\overline{\overline{x^-}} \vee x^-,\, \overline{\overline{x^+}} \vee x^+).$

Using $\overline{\overline{t}}=t$,

$\Diamond x = (x^- \vee x^-,\, x^+ \vee x^+).$

Hence each component simplifies under idempotence of $\vee$ to

$\Diamond x = (x^-,\,x^+).$

The intermediate step shows the structural cancellation in each coordinate induced by $\neg x \Rightarrow x$.

The necessity operator is defined by

$\Box x = \neg \Diamond \neg x.$

First compute $\neg x=(\overline{x^+},\overline{x^-})$, then

$\Diamond(\neg x) = (\overline{x^+},\,\overline{x^-}).$

Applying negation again,

$\Box x = (\overline{\overline{x^-}},\,\overline{\overline{x^+}}) = (x^-,x^+).$

Thus both modal operators preserve the interval representation:

$\Diamond x = \Box x = (x^-,x^+).$

This reflects that in representation (128) the values already encode lower and upper bounds, so possibility and necessity do not refine the interval further.

### Equivalence

Equivalence is defined by

$x \Leftrightarrow y = (x \Rightarrow y) \wedge (y \Rightarrow x).$

Compute each implication:

$x \Rightarrow y = (\overline{x^+} \vee y^-,\, \overline{x^-} \vee y^+),$

$y \Rightarrow x = (\overline{y^+} \vee x^-,\, \overline{y^-} \vee x^+).$

Taking componentwise conjunction,

$$
x \Leftrightarrow y =
\big((\overline{x^+} \vee y^-) \wedge (\overline{y^+} \vee x^-),\;
(\overline{x^-} \vee y^+) \wedge (\overline{y^-} \vee x^+)\big).
$$

Each coordinate is expressed using only the Boolean operations $\wedge,\vee,\overline{\cdot}$ applied bitwise to the endpoints of $x$ and $y$, hence is directly implementable in representation (128) without leaving the interval encoding.
