[Section 7.1.3: Bitwise Tricks and Techniques](../)

**Exercise 128.** [*M46*] [M46] Can (νx) mod 2 be computed in O(log n/log log n) broadword steps?

**Verified:** no  
**Solve time:** 4m36s  

---

The previous argument fails because it conflates “AC⁰-definable word transformations” with a single TAOCP broadword step and then implicitly allows the description size of a step to grow with $n$. Once the model is stated correctly, the claimed speedup cannot be justified.

We now restate the correct model and derive a valid consequence.

## 1. Correct broadword model

A broadword step on an $n$-bit word is a _fixed_, $n$-independent instruction chosen from a finite repertoire such as:

- bitwise Boolean operations ($\wedge,\vee,\oplus,\neg$),
- shifts by constants (or by $O(1)$-bit constants encoded in the instruction),
- masking by fixed patterns.

Crucially:

- Each step has **constant description size**, independent of $n$.
- Each step applies a uniform operation to the entire word.
- A computation consists of a sequence of such steps.

Thus a $t$-step broadword computation corresponds to a _uniform circuit of depth $t$_, where each layer is composed of a constant number of word-level AC⁰ primitives.

## 2. What the previous argument incorrectly used

The flawed solution attempted to use a single step of the form

$$
x \leftarrow x \oplus \bigoplus_{i=1}^{\Theta(\log n)} ((x \gg d_i)\wedge M^{(i)}),
$$

which implicitly allows:

- $\Theta(\log n)$ distinct shifts,
- $\Theta(\log n)$ masks,
- a growing fan-in within one step.

This violates the model because a single step cannot encode an instruction whose size grows with $n$. That construction is not one broadword step; it is already a circuit of logarithmic size.

## 3. Correct reduction to circuit complexity

After $t$ broadword steps:

- Each output bit is a Boolean function of the input bits.
- This function is computable by a **uniform Boolean circuit of depth $O(t)$**.
- Each layer has **bounded fan-in and constant size expansion**, since each step is a fixed combination of shifts, masks, and bitwise operations.

Hence the parity computation after $t$ steps yields a uniform circuit for parity of depth $O(t)$ and size at most exponential in $t$, but crucially still within AC⁰-style bounded-depth structure when $t$ is small.

More precisely, each step preserves membership in AC⁰, and composition of $t$ steps yields an AC⁰ circuit of depth $O(t)$ with size $n^{O(1)}$ for fixed $t$.

## 4. Known lower bound for parity

A classical result from circuit complexity (Håstad’s switching lemma) implies:

> Any AC⁰ circuit computing parity on $n$ bits must have depth

$$
\Omega\!\left(\frac{\log n}{\log \log n}\right).
$$

Equivalently:

- parity cannot be computed by uniform bounded-fan-in circuits of smaller depth,
- and this bound applies to any model that compiles into such circuits with constant-size layers.

Our broadword model fits this framework because each step is a constant-size uniform AC⁰ transformation.

## 5. Consequence for broadword steps

If parity were computable in $t$ broadword steps, then it would be computable by a depth-$O(t)$ AC⁰ circuit.

Therefore we must have

$$
t = \Omega\!\left(\frac{\log n}{\log \log n}\right).
$$

This rules out any $o(\log n / \log \log n)$-step computation in the TAOCP broadword model.

## 6. Final conclusion

The proposed $O(\log n / \log \log n)$-step construction fails because it illegally embeds a growing number of operations into a single step.

Once the model is enforced correctly, each broadword step corresponds to a constant-size layer in a uniform AC⁰ circuit, and parity requires super-logarithmic depth in this sense.

$$
(\nu x) \bmod 2 \text{ cannot be computed in } O\!\left(\frac{\log n}{\log \log n}\right) \text{ broadword steps.}
$$

In fact,

$$
\text{any such computation requires } \Omega\!\left(\frac{\log n}{\log \log n}\right) \text{ steps.}
$$

∎
