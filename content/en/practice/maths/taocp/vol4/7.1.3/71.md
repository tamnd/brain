**Exercise 71.** [*20*] Explain how to *reverse* the procedure of (80), going from the compact value $y = (y_{r-1} \ldots y_1 y_0)_2$ to a number $z = (z_{63} \ldots z_1 z_0)_2$ that has $z_{j_i} = y_i$ for $0 \le i < r$.

## Solution

Let the compression procedure (80) select indices $0 \le j_0 < j_1 < \cdots < j_{r-1} < 64$, and let the packed value

$y = (y_{r-1}\cdots y_1 y_0)_2$

satisfy $y_i = z_{j_i}$ for the original word $z = (z_{63}\cdots z_1 z_0)_2$. The task is to reconstruct a word $z$ such that $z_{j_i} = y_i$ for $0 \le i < r$.

The binary expansion of $y$ gives each bit $y_i$ as the $i$th bit of $y$, hence

$y_i = (y \gg i) \,\&\, 1.$

Define a word $z$ by initializing all bits to $0$ and then setting, for each $i$ with $0 \le i < r$,

$z \leftarrow z \mid (y_i \ll j_i).$

Each update inserts a single bit at position $j_i$ since $y_i \in {0,1}$ implies $y_i \ll j_i$ has a 1-bit only at position $j_i$ when $y_i = 1$, and is $0$ otherwise. The bitwise OR accumulates these contributions without interference between different indices because distinct $j_i$ correspond to distinct bit positions.

For correctness, fix $k$ with $0 \le k < 64$. If $k = j_i$ for some $i$, then the only term contributing to position $k$ in the construction of $z$ is $y_i \ll j_i$, so $z_k = y_i$. If $k \ne j_i$ for all $i$, then no term contributes at position $k$, hence $z_k = 0$. This matches the required specification $z_{j_i} = y_i$.

Thus the constructed word satisfies all prescribed bit placements, and no other bits are set.

This completes the proof. ∎
