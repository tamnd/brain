## (1) Fredman–Willard procedure (as in (95))

The Fredman–Willard technique is a word-RAM method for performing many comparisons in parallel by encoding combinatorial structure into a constant number of word operations. The key idea is that a set of ordered keys can be reduced to short “sketches” so that order is preserved, and these sketches can then be manipulated in parallel using field arithmetic inside a single machine word.

### Sketch construction (distinguishing bits)

Let $S = \{y_0 < y_1 < \dots < y_{t-1}\}$ be a set of keys, each a $w$-bit integer. Consider the binary trie on $S$. The relevant bits for distinguishing keys are the bit positions corresponding to branching nodes in this trie. Let these positions be

$$
b_1 < b_2 < \cdots < b_r,
$$

where $r \le t-1$.

For each key $y$, define its sketch $\sigma(y)$ by extracting exactly these bits:

$$
\sigma(y) = (y[b_1], y[b_2], \dots, y[b_r]) \in [0, 2^r).
$$

A crucial property is that the sketches preserve order:

$$
y_1 < y_2 \;\Longleftrightarrow\; \sigma(y_1) < \sigma(y_2),
$$

because the first differing trie bit appears among the selected distinguishing positions.

### Packing into a word

Each sketch is stored in a fixed-width field of $r+1$ bits inside a word. The extra guard bit prevents carries from crossing field boundaries.

Define the packed representation

$$
W = \sum_{i=0}^{t-1} \sigma(y_i)\,2^{i(r+1)}.
$$

Each block $\sigma(y_i)$ occupies its own isolated field.

The guard bit ensures that when additions or subtractions are performed on the word, no carry propagates from one field into another, since every field is separated by at least one zero bit.

### Parallel comparison via multiplication

Given a query key $x$, compute its sketch $\sigma(x)$. Form the replicated word

$$
X = \sigma(x)\cdot \left(\sum_{i=0}^{t-1} 2^{i(r+1)}\right).
$$

This works because multiplication by a constant distributes $\sigma(x)$ into every field position:

$$
X = \sum_{i=0}^{t-1} \sigma(x)\,2^{i(r+1)}.
$$

Now compute

$$
D = W - X.
$$

Within each field, the value is

$$
\sigma(y_i) - \sigma(x),
$$

and because of guard bits, no borrow propagates between fields.

Hence, the sign of each field encodes the comparison result:

- negative: $\sigma(y_i) < \sigma(x)$
- zero: equal
- positive: greater

### Extracting predecessor / successor

To find the predecessor of $x$, we need the largest index $i$ such that $\sigma(y_i) \le \sigma(x)$.

This reduces to identifying the rightmost field whose subtraction result is nonpositive.

Standard word-RAM techniques then apply:

- isolate sign bits of each field,
- compress them into a bitmask,
- find the most significant set bit (using standard constant-time bit tricks in the word-RAM model),
- convert that position back to the index $i$.

Thus predecessor or successor search is completed in $O(1)$ word operations.

## (2) Computing $2^{\lambda x}$ without left shifts

We must compute

$$
2^{\lambda x}
$$

in the word-RAM model without using left-shift instructions, using only arithmetic operations such as addition and multiplication.

### Step 1: eliminate shifts from exponent construction

Observe that

$$
2^{\lambda x} = (2^\lambda)^x.
$$

Let

$$
a = 2^\lambda.
$$

Since $\lambda$ is a fixed constant, $a$ can be constructed from $1$ using repeated multiplication:

$$
1 \xrightarrow{\times 2} 2 \xrightarrow{\times 2} \cdots \xrightarrow{\times 2} 2^\lambda.
$$

This uses only multiplication, not shifting.

### Step 2: compute $a^x$ in the word-RAM model

Now compute $a^x$ using exponentiation by squaring:

- if $x$ is even, $a^x = (a^{x/2})^2$,
- if $x$ is odd, $a^x = a \cdot a^{x-1}$.

Each step uses only:

- multiplication of $w$-bit words,
- division by 2 (bit test and shift-free parity extraction via arithmetic),
- conditional branching.

No left-shift instruction is ever used; all growth of magnitude is handled by multiplication.

### Step 3: complexity justification

In the word-RAM model, a word has $w = O(\log n)$ bits, and arithmetic operations on words are unit cost.

The exponentiation algorithm performs $O(\log x)$ multiplications. Since $x$ is a $w$-bit number, $\log x = O(w)$, and in the word-RAM model this is $O(1)$ time because $w$ is fixed machine precision.

Thus the entire computation runs in $O(1)$ word-RAM time.

### Step 4: correctness

By construction,

$$
a^x = (2^\lambda)^x = 2^{\lambda x}.
$$

Therefore the algorithm computes the required value exactly, using only multiplication and no left shifts.

## Conclusion

The Fredman–Willard method works by compressing keys into order-preserving sketches and using field-packed arithmetic with guard bits so that a single subtraction evaluates many comparisons in parallel, with the result decoded via bit extraction.

The computation of $2^{\lambda x}$ is obtained by first forming $2^\lambda$ using multiplication only, then computing its $x$-th power via exponentiation by squaring. In the word-RAM model this uses only $O(1)$ word operations and avoids all shift instructions, since shifts are replaced entirely by multiplication-based growth of values.
