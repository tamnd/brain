**Exercise 147.** &#9654; [*M20*] [M20] What values of πv, βv, αv, and τj occur in Algorithm V when the forest is
a) the empty digraph with vertices {v1, . . . , vn} and no arcs?
b) the oriented path vn −−→· · · −−→v2 −−→v1?

**Verified:** yes  
**Solve time:** 1m53s  

---

We restate Algorithm V as the standard depth-first search from TAOCP: vertices are scanned in the outer loop in order $v_1, v_2, \ldots, v_n$. When an unmarked vertex is encountered, a DFS-visit is started, which assigns discovery time $\beta_v$, finishing time $\alpha_v$, sets $\pi_v$ for tree edges, and appends each newly discovered tree arc to $\tau_j$.

Let the time counter start at $0$ and increase by $1$ at each assignment of a discovery or finishing time.

## a) Empty digraph on $\{v_1,\ldots,v_n\}$

There are no arcs, so every DFS-visit immediately terminates at the vertex where it starts.

The outer loop still processes every vertex in order.

For each $v_i$, when it is first encountered it is unmarked, so a DFS-visit is initiated:

- $\beta_{v_i} = 2i - 1$
- $\alpha_{v_i} = 2i$

This follows because each vertex contributes exactly one discovery event and one finishing event, and no recursive calls occur to interleave the timestamps.

Since no edges exist, no vertex ever discovers another vertex:

- $\pi_v = 0$ for all $v$
- $\tau_j = \varnothing$

Thus:

$$
\pi_v = 0,\quad \beta_{v_i} = 2i-1,\quad \alpha_{v_i} = 2i,\quad \tau_j = \varnothing.
$$

## b) Oriented path $v_n \to v_{n-1} \to \cdots \to v_1$

The directed edges are:

$$
(v_{i+1}, v_i), \quad 1 \le i \le n-1.
$$

Crucial point: DFS does not start at $v_n$. It begins at $v_1$ because the outer loop processes vertices in order $v_1, v_2, \ldots, v_n$.

### Step 1: Start at $v_1$

- $v_1$ has no outgoing edges.
- DFS-visit$(v_1)$ completes immediately:

$$
\beta_{v_1} = 1,\quad \alpha_{v_1} = 2.
$$

### Step 2: Process $v_2$

When $v_2$ is reached in the outer loop, $v_1$ is already marked, so:

- DFS-visit$(v_2)$ runs.
- It examines the only outgoing edge $(v_2, v_1)$, but $v_1$ is already finished, so no recursive call occurs.

Thus:

$$
\beta_{v_2} = 3,\quad \alpha_{v_2} = 4,\quad \pi_{v_2} = 0.
$$

### Induction step

Assume $v_1,\ldots,v_{k-1}$ are already finished when $v_k$ is processed. Then all of them are marked, so DFS-visit$(v_k)$ sees no unvisited successors. Hence:

$$
\beta_{v_k} = 2k-1,\quad \alpha_{v_k} = 2k,\quad \pi_{v_k} = 0.
$$

This holds for all $k = 1,\ldots,n$.

### Tree structure $\tau_j$

No recursive calls are ever made, because every potential child is already visited by the time its predecessor is processed in the outer loop. Therefore no tree edges are generated:

$$
\tau_j = \varnothing.
$$

## Final values for part (b)

$$
\pi_v = 0 \text{ for all } v,
\quad
\beta_{v_i} = 2i - 1,
\quad
\alpha_{v_i} = 2i,
\quad
\tau_j = \varnothing.
$$

## Conclusion

In both (a) and (b), the structure of the graph does not affect the DFS forest because the outer-loop order $(v_1,\ldots,v_n)$ ensures that every vertex is already visited before any possible predecessor in the directed path is processed. Thus all vertices become separate roots, all $\pi_v$ are zero, and all timestamps are assigned sequentially without recursion.
