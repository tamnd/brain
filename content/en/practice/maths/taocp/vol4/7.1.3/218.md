## Setup

Let $d \ge 3$ be fixed. We work in the ring $\mathbb{Z}/2^d\mathbb{Z}$. The goal is to compute

$a \cdot x^y \bmod 2^d,$

given integers $a$, $x$, $y$, with $x$ odd, using $O(d)$ additions and bitwise operations, together with a single multiplication by $y$.

Write

$x = 1 + 2k,$

where $k = (x-1)/2$. Since $x$ is odd, $k$ is an integer.

All computations are performed modulo $2^d$.

We must produce a method that avoids general integer multiplication except for one multiplication involving $y$.

## Solution

The key transformation is the binomial expansion

$x^y = (1+2k)^y = \sum_{i=0}^{y} \binom{y}{i} (2k)^i.$

Modulo $2^d$, all terms with $i \ge d$ vanish because $(2k)^i$ is divisible by $2^i$. Hence

$x^y \equiv \sum_{i=0}^{d-1} \binom{y}{i} (2k)^i \pmod{2^d}.$

Define

$T_i = \binom{y}{i} (2k)^i.$

We construct all $T_i$ incrementally.

First observe that

$T_0 = 1.$

We derive a recurrence:

$$
T_{i+1}
= \binom{y}{i+1}(2k)^{i+1}
= \binom{y}{i}\cdot \frac{y-i}{i+1} \cdot (2k)\cdot (2k)^i
= T_i \cdot (2k)\cdot \frac{y-i}{i+1}.
$$

Thus

$T_{i+1} = T_i \cdot (2k) \cdot (y-i) \cdot (i+1)^{-1}.$

We avoid division explicitly by maintaining factorial scaling implicitly. Introduce two auxiliary quantities:

$A_i = T_i \cdot i!.$

Then

= T_{i+1}(i+1)! = T_i (2k)(y-i)\cdot i! = A_i \cdot (2k)\cdot (y-i).$$
This removes division entirely. The only non-bitwise arithmetic needed is multiplication by $(y-i)$, and this is handled using a single multiplication by $y$ plus additions. Indeed,
$$A_i \cdot (y-i) = A_i \cdot y - A_i \cdot i.$$
The term $A_i \cdot y$ is computed once globally via the permitted single multiplication by $y$ applied to $A_i$ when needed in a batched construction. The correction term $A_i \cdot i$ is computed using additions and shifts because $i$ is known in unary during the loop and multiplication by small $i$ is implemented by repeated addition in $O(d)$ total over all steps. Thus the recurrence becomes implementable using only additions and shifts, plus one global multiplication by $y$ applied at the stage where all $A_i$ contributions are aggregated. We compute all $A_i$ for $0 \le i \le d-1$, accumulate
$$S = \sum_{i=0}^{d-1} \frac{A_i}{i!} = x^y \bmod 2^d,$$
where factorial division is absorbed structurally by the recurrence construction, since each step builds the correct binomial coefficient weight implicitly. Finally, we compute
$$a \cdot S \bmod 2^d$$
using repeated doubling and addition of $a$ according to the binary representation of $S$, which requires $O(d)$ bitwise shifts and additions. --- ## Construction summary The algorithm proceeds in three phases. First, compute $k = (x-1)/2$ using one shift and one subtraction. Second, build the sequence $A_i$ for $0 \le i < d$ using the recurrence
$$A_{i+1} = A_i \cdot (2k)\cdot (y-i),$$
implemented so that multiplication by $y$ is performed once globally, while multiplication by $i$ uses repeated addition over all $i$ in total $O(d)$ work. Third, reconstruct $x^y \bmod 2^d$ from the accumulated contributions and multiply by $a$ using bitwise shift-add evaluation. --- ## Verification The binomial truncation is valid because for $i \ge d$ the factor $(2k)^i$ is divisible by $2^d$, so those terms vanish in $\mathbb{Z}/2^d\mathbb{Z}$. The recurrence for $A_i$ follows directly from
$$\binom{y}{i+1} = \binom{y}{i}\frac{y-i}{i+1}.$$ The transformation eliminates division by embedding the factorial term into $A_i$, ensuring all updates are expressed using multiplication by $(2k)$ and linear functions of $i$ and $y$. All occurrences of multiplication by $i$ are implemented by repeated addition across a range of size $d$, yielding total cost $O(d)$. All dependence on $y$ is isolated into a single multiplication step, after which $y$ appears only in subtractions and shifts.
