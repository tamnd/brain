**Exercise 142.** &#9654; [*33*] [33] A subcube such as ∗10∗1∗01 can be represented by asterisk codes 10010100
and bit codes 01001001, as in (85); but many other encodings are also possible. What

Let variables be $x_1,\dots,x_n$. A cube (subcube) $C$ is a conjunction of literals in which each index $i$ is either fixed to $1$, fixed to $0$, or left free.

## 1. What the consensus algorithm actually requires

In Exercise 7.1.1–31, the **consensus operation** is a _binary resolution step on a single pivot variable_:

Given two cubes $C$ and $D$, a consensus is formed only if there exists an index $k$ such that:

- $x_k \in C$ and $\bar{x}_k \in D$, or vice versa,
- and for every other variable $i \ne k$, the cubes are compatible (they do not force opposite values on any variable).

The resulting cube (the resolvent) is obtained by:

- deleting the pivot variable $x_k$,
- keeping all other literals that are consistent.

So the representation must make three operations efficient:

1. Detect complementary literals,
2. Verify that there is exactly one resolving variable and no other conflicts,
3. Construct the merged cube with that variable removed.

## 2. Representation scheme

The best representation is to encode each cube $C$ by two bit vectors of length $n$:

$$
p_i =
\begin{cases}
1 & \text{if } x_i \text{ appears in } C \\
0 & \text{otherwise}
\end{cases}
\qquad
n_i =
\begin{cases}
1 & \text{if } \bar{x}_i \text{ appears in } C \\
0 & \text{otherwise}
\end{cases}
$$

with the invariant:

$$
p \,\&\, n = 0.
$$

Thus:

- $p$ marks positive literals,
- $n$ marks negative literals,
- free variables have $p_i = n_i = 0$.

This is equivalent to ternary encoding $\{0,1,*\}$, but implemented in two machine words.

## 3. Compatibility and pivot detection

Given cubes $C=(p,n)$ and $D=(p',n')$:

### Opposite-literal conflicts

$$
\text{conflict}(C,D) = (p \,\&\, n') \,\lor\, (n \,\&\, p').
$$

- If this is zero, the cubes are compatible (no contradictions).
- If it is nonzero, every 1-bit indicates a variable where they disagree.

### Consensus condition (critical correction)

A valid consensus exists only if:

1. There is exactly one resolving variable:

$$
k \text{ is the unique index in } (p \,\&\, n') \,\lor\, (n \,\&\, p').
$$

1. No other conflicts exist.

Equivalently:

$$
(p \,\&\, n') \,\lor\, (n \,\&\, p') = 2^k.
$$

This ensures the resolution is a _single-variable pivot_, not a multi-conflict merge.

## 4. Correct consensus (resolvent) construction

Assume $k$ is the unique pivot variable.

The resolvent cube $E$ is formed by removing the pivot literal and combining all remaining literals:

$$
p_E = (p \,\lor\, p') \,\&\, \overline{2^k},
\qquad
n_E = (n \,\lor\, n') \,\&\, \overline{2^k}.
$$

This is correct because:

- All non-conflicting literals are preserved,
- The pivot variable is removed entirely,
- No other variables can be contradictory by the validity condition.

This exactly matches the definition of consensus in cube calculus: resolution on one variable only.

## 5. Why this representation is optimal for the algorithm

This two-bit representation is optimal for a consensus-driven prime implicant algorithm because every required operation becomes a small number of word-level instructions:

### (a) Detecting resolution opportunity

$$
(p \,\&\, n') \,\lor\, (n \,\&\, p')
$$

is one AND, one AND, one OR.

### (b) Checking uniqueness of pivot

Testing whether the result is a power of two can be done with:

$$
x \ne 0 \;\land\; (x \,\&\, (x-1)) = 0.
$$

### (c) Constructing resolvent

Two ORs and two ANDs.

All operations are constant-time on machine words.

## 6. Why other encodings are worse

- A ternary encoding $\{0,1,*\}$ per variable still requires at least two bits per variable in hardware to represent three states.
- Sparse literal lists require scanning and comparison of all literals, making consensus $O(m)$ per step instead of word-parallel $O(1)$.
- Any multi-state packed encoding still reduces to separate masks for “positive” and “negative” occurrences when implementing conflict detection.

Thus every practical encoding collapses to the same minimal structure: two disjoint bit masks.

## 7. Conclusion

The best representation scheme for subcubes in the consensus-based algorithm is:

- a pair of bit vectors $(p,n)$,
- encoding positive and negative literals separately,
- with disjoint support $p \& n = 0$.

It is optimal because:

- compatibility reduces to bitwise AND,
- consensus detection reduces to detecting a single-bit conflict,
- resolvent construction is bitwise masking and OR,
- all operations are word-parallel and constant-time.

This matches exactly the structure required for efficient generation of prime implicants via repeated consensus.
