**Exercise 105.** [*30*] Given $x = (x_7 \ldots x_0)_{256}$ and $y = (y_7 \ldots y_0)_{256}$, discuss bitwise operations that will *sort* the bytes into order, so that $x_0 \le y_0 \le \cdots \le x_7 \le y_7$ afterwards.

The flaw in the previous solution is the assumption that per-byte arithmetic is available “for free.” In the standard TAOCP word model, all operations act on the full 64-bit word, and carries or borrows do propagate across byte boundaries unless we explicitly prevent it. Therefore, we must _construct_ a bytewise comparator using only legal word operations.

The correct approach is to simulate 8-bit arithmetic inside each byte by preventing inter-byte carry propagation at every step.

## 1. Bit model and byte isolation

Let a 64-bit word be decomposed into bytes

$$
A = (a_7,\dots,a_0)_{256}, \quad B = (b_7,\dots,b_0)_{256}.
$$

Define the byte mask

$$
R = (0x7f,0x7f,\dots,0x7f)_{256}, \quad H = (0x80,0x80,\dots,0x80)_{256}.
$$

The key idea is:

- Bit 7 of each byte is reserved to block carry propagation.
- We enforce this invariant after every shift step.

Thus, each byte behaves like a 7-bit cell with an isolated carry boundary.

## 2. Constructing per-byte subtraction without cross-byte borrows

We compute $A - B$ using the standard bitwise subtraction identity:

$$
d = A \oplus B,\quad c = (\neg A \,\&\, B)
$$

then iteratively propagate borrows via shifts:

$$
c \leftarrow (c \ll 1), \quad d \leftarrow d \oplus c
$$

However, this normally allows borrows to cross byte boundaries. To prevent this, after each shift we enforce:

$$
c \leftarrow c \,\&\, R
$$

so any borrow bit entering the high bit of a byte is deleted.

Thus the iteration becomes:

1. $c \leftarrow (\neg A \,\&\, B)$
2. Repeat 8 times:

$$
t \leftarrow c \ll 1,\quad t \leftarrow t \,\&\, R
$$

$$
d \leftarrow d \oplus t
$$

$$
c \leftarrow (\neg d \,\&\, t)
$$

Because the mask $R$ removes bit 7 of every byte at each step, no borrow can propagate between bytes. Within each byte, the computation is exactly standard binary subtraction.

Hence after 8 iterations:

$$
d_i \equiv a_i - b_i \pmod{256}
\quad \text{computed independently for each byte } i.
$$

## 3. Extracting a per-byte comparison mask

We now determine whether $a_i < b_i$.

As in standard 8-bit arithmetic, after computing $d = A - B$, the sign of each byte is given by its high bit.

Define:

$$
S = d \,\&\, H.
$$

Then each byte satisfies:

- $S_i = 0x80$ iff $a_i < b_i$
- $S_i = 0$ otherwise

Convert this into a full-byte mask:

$$
M = (S \gg 7)\cdot 255.
$$

So:

- $M_i = 0xFF$ if $a_i < b_i$
- $M_i = 0x00$ otherwise

This is a valid word-level construction using only shifts, AND, and multiplication by a constant mask.

## 4. Bytewise compare–exchange

Using the mask $M$, define:

$$
\min(A,B) = (A \,\&\, M)\;|\;(B \,\&\, \neg M),
$$

$$
\max(A,B) = (A \,\&\, \neg M)\;|\;(B \,\&\, M).
$$

This works because each byte of $M$ is uniform (all 0 or all 1 bits), so selection is independent per byte.

Thus we obtain a correct compare–exchange primitive:

$$
\mathrm{CE}(A,B) = (\min(A,B), \max(A,B)).
$$

## 5. Sorting structure

We now apply the same deterministic network as in the original attempt.

We want:

$$
x_0 \le y_0 \le x_1 \le y_1 \le \cdots \le x_7 \le y_7.
$$

This is enforced in two stages.

### Stage 1: intra-pair ordering

For each $i$:

$$
(x_i, y_i) \leftarrow \mathrm{CE}(x_i, y_i).
$$

So:

$$
x_i \le y_i \quad \forall i.
$$

### Stage 2: inter-pair coupling

For each $i = 0,\dots,6$:

$$
(y_i, x_{i+1}) \leftarrow \mathrm{CE}(y_i, x_{i+1}).
$$

So:

$$
y_i \le x_{i+1}.
$$

## 6. Correctness

We now have the chain:

$$
x_i \le y_i \le x_{i+1}
\quad \text{for all } i.
$$

Chaining these inequalities yields:

$$
x_0 \le y_0 \le x_1 \le y_1 \le \cdots \le x_7 \le y_7.
$$

Thus the sequence is fully sorted in alternating order as required.

## 7. Resolution of the original defect

The original error was the assumption that bytewise subtraction is automatically independent. The corrected construction removes this assumption entirely by:

- explicitly simulating subtraction via bitwise operations
- preventing cross-byte borrow propagation using a fixed mask after every shift
- deriving per-byte comparison only after correctness of isolated arithmetic is established

All steps now operate within the standard TAOCP word-bit model using only bitwise logic, shifts, and word masks.

This yields a valid compare–exchange primitive and therefore a correct sorting construction.
