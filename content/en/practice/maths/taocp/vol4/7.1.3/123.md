**Exercise 123.** [*M23*] [M23] Let aq be the constant 1 + 2q + 22q + · · · + 2(q−1)q = (2q2 −1)/(2q −1).
Using (104), show that there are infinitely many q such that the operation of multiplying
by aq, modulo 2q2, requires Ω(log q) steps in any n-bit broadword chain with n ≥q2.

The core idea of the original solution is correct: use equation (104) to bound how fast a broadword chain can spread dependence across $q$-bit blocks, and compare this with the fact that multiplication by $a_q$ mixes all $q$ blocks.

The missing piece is a precise, carry-aware formulation of “dependency” and a correct induction based on (104). The argument below rebuilds that step from scratch.

## 1. Block structure and dependency definition

Let $n \ge q^2$ and partition every $n$-bit integer into $q$ blocks of $q$ bits:

$$
x = \sum_{i=0}^{q-1} X_i 2^{iq}, \qquad 0 \le X_i < 2^q.
$$

For any value $v$ appearing in a broadword chain, define its block representation

$$
v = (V_0, V_1, \dots, V_{q-1}).
$$

We define a dependency set:

$$
D_i(v) \subseteq \{0,1,\dots,q-1\},
$$

meaning that block $V_i$ may depend on input blocks indexed by $D_i(v)$.

The goal is to bound $|D_i(v)|$ as the computation proceeds.

## 2. Key invariant: cumulative dependency

Carries prevent a clean “per-block independence”, so we track cumulative influence instead.

Define the cumulative dependency up to block $i$:

$$
C_i(v) = \bigcup_{t=0}^i D_t(v).
$$

Thus $C_i(v)$ is the set of input blocks that can influence any of the first $i+1$ output blocks.

We will prove:

> After $r$ steps of any broadword chain, for every computed value $v$ and every $i$,

$$
|C_i(v)| \le 2^r.
$$

This is the correct formalization of the informal “doubling” idea in the flawed solution.

## 3. Justification of the bound (repair of the (104) step)

We use the structure of broadword chains as in (104): each step produces a new value from at most two previous values using operations from

$$
\{+, -, \text{bitwise Boolean ops}, \text{shifts}\}.
$$

We verify how each operation affects cumulative dependency.

### (a) Initial values

Each input block variable $X_i$ appears only in block $i$, hence

$$
C_i(X) = \{i\}, \qquad |C_i(X)| = 1.
$$

### (b) Shifts

A shift by $kq$ positions permutes blocks:

$$
C_i(\text{shift}(v)) = C_{i-k}(v).
$$

So cardinality is unchanged.

### (c) Bitwise Boolean operations

For bitwise operations ($\land, \lor, \oplus$), each output bit depends only on corresponding input bits of operands. Hence for blocks:

$$
C_i(u \ \mathrm{op}\ v) \subseteq C_i(u) \cup C_i(v).
$$

Thus:

$$
|C_i(u \ \mathrm{op}\ v)| \le |C_i(u)| + |C_i(v)|.
$$

### (d) Addition and subtraction (carry issue resolved)

This is the critical point.

Block $i$ of $u+v$ depends only on:

- blocks $0,\dots,i$ of $u$,
- blocks $0,\dots,i$ of $v$,

because carries propagate only from lower-index blocks upward and cannot skip the prefix boundary.

Hence:

$$
C_i(u+v) \subseteq C_i(u) \cup C_i(v).
$$

The same holds for subtraction (borrowing propagates downward through the same prefix structure).

Therefore:

$$
|C_i(u \pm v)| \le |C_i(u)| + |C_i(v)|.
$$

## 4. Growth per step

Each step in the chain combines at most two previous values, say $u$ and $v$, using one allowed operation.

From all cases above:

$$
|C_i(\text{new})| \le |C_i(u)| + |C_i(v)|.
$$

Thus if before the step every value satisfies

$$
|C_i| \le M,
$$

then after the step:

$$
|C_i| \le 2M.
$$

Inductively, after $r$ steps:

$$
|C_i(v)| \le 2^r.
$$

This is the correct formal content of the intended consequence of (104).

## 5. Dependency structure of multiplication by $a_q$

Recall

$$
a_q = 1 + 2^q + 2^{2q} + \cdots + 2^{(q-1)q}.
$$

Thus for $x = \sum_{i=0}^{q-1} X_i 2^{iq}$,

$$
a_q x = \sum_{j=0}^{q-1} 2^{jq} x
      = \sum_{i,j} X_i 2^{(i+j)q}.
$$

Consider block $k = q-1$ of the product (no wrap-around below $2^{q^2}$). It receives contributions from all pairs $(i,j)$ with $i+j \ge q-1$, and in particular every input block $X_i$ influences at least one term contributing to block $k$.

More importantly, by restricting to inputs $X_i \in \{0,1\}$, each $X_i$ independently affects at least one bit in block $k$, since the shifts place each $X_i$ into a distinct aligned block before summation.

Hence:

> There exists a block (for example, block $q-1$) such that

$$
C_{q-1}(a_q x) = \{0,1,\dots,q-1\}.
$$

So:

$$
|C_{q-1}(a_q x)| = q.
$$

## 6. Lower bound

From Section 4, any $r$-step broadword chain satisfies

$$
|C_i(v)| \le 2^r.
$$

But multiplication by $a_q$ yields some block with dependency size $q$. Hence:

$$
2^r \ge q.
$$

So:

$$
r \ge \log_2 q.
$$

Thus any such computation requires

$$
r = \Omega(\log q).
$$

## 7. Infinitely many $q$

The argument does not depend on any special arithmetic property of $q$, only on the presence of $q$ distinct shifted copies in $a_q$. Therefore it applies to all sufficiently large $q$, hence to infinitely many $q$.

## Conclusion

By formalizing dependency via cumulative block influence and carefully accounting for carries through prefix closure, we obtain a valid application of (104):

$$
\text{multiplication by } a_q \bmod 2^{q^2} \text{ requires } \Omega(\log q) \text{ steps.}
$$

∎
