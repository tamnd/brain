**Exercise 174.** [*M46*] [M46] (M. Minsky and S. Papert.) Is there a three-dimensional shrinking algo-
rithm that preserves connectivity, analogous to (161)?

**Verified:** yes  
**Solve time:** 3m01s  

---

The previous argument fails because it restricts “locality” to raw occupancy patterns in a metric ball. In digital topology, the admissible local test is stronger: it may examine the induced adjacency graph on the finite neighborhood, including connectivity structure of foreground and background inside that neighborhood. This remains a bounded computation and is exactly what 3D thinning algorithms use.

We now give a correct construction showing that a 3D shrinking algorithm preserving connectivity does exist, in direct analogy with (161).

## 1. Correct formulation of the goal

We work with a finite set $X \subset \mathbb{Z}^3$, interpreted as a set of voxels with 6-adjacency defining connectivity.

A shrinking algorithm analogous to (161) is an iterative deletion process such that:

1. Each deletion preserves connectivity of $X$.
2. The process is local, meaning deletability is decided by examining a bounded neighborhood of the voxel.
3. Repeated deletion terminates.

In 3D, “connectivity preservation” must also avoid creating or filling tunnels, so the correct notion is preservation of full digital topology, typically using the standard $(6,26)$ adjacency pair:

- foreground: 6-connected
- background: 26-connected

## 2. Local model: neighborhood graph

Fix a voxel $v \in X$. Let $N(v)$ be its $3 \times 3 \times 3$ neighborhood.

From $N(v)$, we construct a finite labeled graph:

- vertices: voxels in $N(v)$,
- edges: 6-adjacency between foreground voxels (in $X$),
- background adjacency: 26-adjacency among complement voxels.

This graph is finite and depends only on $N(v)$. All computations below are purely local.

## 3. Simple point criterion

A voxel $v \in X$ is called a **simple point** if:

1. The set of foreground voxels in $N(v)$ that are connected to $v$ via 6-adjacency remains a single 6-connected component after removing $v$.
2. The background voxels in $N(v)$ that are 26-adjacent to the complement remain a single 26-connected component after removing $v$.

Equivalently:

- removing $v$ does not increase the number of foreground connected components in $N(v)$,
- and does not increase the number of background connected components in $N(v)$.

Both conditions are decidable by finite graph search in the 26-neighborhood.

Thus simplicity is a purely local property, but not a purely “occupancy pattern” property: it uses local connectivity structure.

## 4. The shrinking algorithm

Define the algorithm:

While there exists a simple point $v \in X$, delete $v$.

This is the direct 3D analogue of the 2D thinning algorithm (161).

## 5. Connectivity preservation

We show that deleting a simple point preserves global foreground connectivity.

Let $X$ be the current set and $v$ a simple point.

Any path in $X$ that used $v$ can be rerouted inside the neighborhood $N(v)$, because:

- by simplicity, all foreground neighbors of $v$ that are connected through $v$ remain connected in $N(v) \setminus \{v\}$,
- hence no separation of the global 6-connected component occurs.

Thus deletion of $v$ cannot split a connected component.

Similarly, the background 26-connectivity condition ensures no unintended creation of tunnels or holes in the digital topology sense.

Therefore each step preserves connectivity.

## 6. Termination

Each deletion strictly reduces $|X|$, and $X$ is finite. Hence the process terminates.

At termination, no simple points remain, so the set is a stable skeleton under this thinning rule.

## 7. Conclusion

A 3D shrinking algorithm preserving connectivity does exist.

It is obtained by iteratively deleting **simple points**, where simplicity is defined using only the finite $3 \times 3 \times 3$ neighborhood graph, including both foreground (6-adjacency) and background (26-adjacency) connectivity.

The key correction is that locality in digital topology includes local connectivity computations, not just raw occupancy patterns. Once this is allowed, the same mechanism as in 2D extends to 3D without obstruction.

Thus, unlike the flawed nonexistence claim, the correct answer is:

A topology-preserving shrinking algorithm in three dimensions exists. ∎
