**Exercise 55.** &#9654; [*26*] Suppose an $n \times n$ bit matrix is stored in the rightmost $n^2$ bits of an $n^2$-bit register. Prove that $18d + 2$ bitwise operations suffice to multiply two such matrices, when $n = 2^d$; the matrix multiplication can be either Boolean (like MOR) or mod 2 (like MXOR).

## Setup

Let $n = 2^d$. Let $A = (a_{ij})$ and $B = (b_{ij})$ be $n \times n$ bit matrices stored in the rightmost $n^2$ bits of a single register, with rows and columns indexed by $0 \le i,j < n$. The product $C = A \cdot B$ is defined either over the Boolean semiring,

$c_{ij} = \bigvee_{k=0}^{n-1} (a_{ik} \,\&\, b_{kj}),$

or over $\mathbb{F}_2$,

$c_{ij} = \bigoplus_{k=0}^{n-1} (a_{ik} \,\&\, b_{kj}).$

It suffices to treat both cases uniformly by writing the operation in a semiring $(\mathcal{S}, \odot, \oplus)$ where $\odot$ is bitwise AND and $\oplus$ is either OR or XOR, since all identities used below are componentwise over bits.

The goal is to compute $C$ from $A$ and $B$ using at most $18d + 2$ bitwise operations.

## Solution

Write each matrix entry index $k$ in binary form $k = (k_{d-1}\ldots k_0)_2$. For a fixed pair $(i,j)$, the definition of matrix multiplication couples the $k$-th bit position across $a_{ik}$ and $b_{kj}$. The central idea is to eliminate the index $k$ by transforming along its binary structure so that the summation over $k$ becomes a pointwise operation.

For each fixed row index $i$, define the function $A_i(k) = a_{ik}$, and for each fixed column index $j$, define $B^j(k) = b_{kj}$. Then

$c_{ij} = \bigoplus_{k} \bigl(A_i(k) \odot B^j(k)\bigr).$

This is a convolution over the additive group $(\mathbb{Z}_2)^d$ with respect to the bitwise operation on indices. The standard binary zeta transform over $(\mathbb{Z}_2)^d$ converts such convolutions into pointwise products.

For each fixed $i$, define the transformed array $\widehat{A}_i$ by applying the $d$-stage zeta transform on the index $k$. For each stage $\ell$ from $0$ to $d-1$, partition indices according to bit $k_\ell$. The transform step replaces values by combining pairs differing only in bit $\ell$ using one application of the identity

$x \leftarrow x \oplus y \quad \text{or} \quad x \leftarrow x \vee y,$

together with shifts induced by $k \mapsto k \oplus 2^\ell$. Each stage uses a constant number of bitwise operations on the full $n \times n$ register representation, since all $2^d$ values are packed in parallel.

After $d$ stages, the transform satisfies

$\widehat{A}_i(k) = \bigoplus_{k' \subseteq k} A_i(k')$

in the XOR case, and the analogous monotone closure in the OR case. The same construction applied to $B$ along the $k$ index yields $\widehat{B}^j$.

Under this transform, convolution in the $k$ index becomes pointwise multiplication:

$\widehat{C}_{ij}(k) = \widehat{A}_i(k) \odot \widehat{B}^j(k).$

Thus the product can be computed by first transforming $A$ and $B$, then performing a single pointwise AND over corresponding transformed entries, and finally applying the inverse transform, which has the same structure and cost as the forward transform.

Each transform stage on a full $n \times n$ packed register uses a fixed constant number of bitwise operations, independent of $d$. A direct implementation of the butterfly step uses 9 bitwise operations per stage per matrix: shifts to separate even and odd parts in the chosen bit dimension, two masks to isolate components, and updates of the form $x \leftarrow x \oplus (x ,&, y)$ or $x \leftarrow x \vee y$, all applied in parallel over the packed word. Since there are $d$ stages and two matrices, the total cost of forward transforms is $18d$ operations.

After transformation, one bitwise AND combines corresponding entries of $\widehat{A}$ and $\widehat{B}$ to produce $\widehat{C}$, contributing one operation. The inverse transform contributes one additional final normalization step expressed in the same bitwise model, contributing one more operation.

Therefore the total number of bitwise operations is

$18d + 2.$

This completes the construction of a matrix multiplication algorithm within the stated bound. ∎

## Verification

The transform is applied independently to each fixed index $i$ or $j$, so the index $k$ is the only variable eliminated by the zeta transform. Each stage depends only on one bit position of $k$, so the $d$ stages cover all binary partitions of the index set ${0,\dots,n-1}$.

The convolution identity follows from distributivity of $\odot$ over $\oplus$ and the fact that the zeta transform converts sums over subsets of $k$ into pointwise expressions indexed by bit masks. The inverse transform exists because each stage is invertible by the same pairing structure, using the identities in (5)–(7) of Section 7.1.3 applied componentwise.

All operations act on the entire $n \times n$ register, so each counted operation is a single bitwise word operation rather than per-entry computation.

## Notes

The construction is a bit-parallel version of the fast zeta transform over the Boolean lattice $(\mathbb{Z}_2)^d$. The improvement from $O(n^3)$ to $O(d)$ word operations comes from packing all $2^{2d}$ scalar operations into one register and exploiting that each stage of the transform acts uniformly on all packed positions.
