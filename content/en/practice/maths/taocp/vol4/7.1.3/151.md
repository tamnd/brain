**Exercise 151.** [*22*] [22] Conversely, show that any algorithm for range minimum queries can be used
to find nearest common ancestors, with essentially the same efficiency.

**Verified:** yes  
**Solve time:** 59s

## Solution

Let $T$ be a rooted tree with vertices $1,\ldots,n$. The nearest common ancestor of two vertices $u,v$ is denoted $\mathrm{lca}(u,v)$.

Construct an Euler tour of $T$ starting at the root. Let the resulting sequence of visited vertices be

$E[1],E[2],\ldots,E[2n-1].$

Let $D[k]$ be the depth of $E[k]$ in $T$. For each vertex $v$, let $\mathrm{first}(v)$ be the least index $k$ such that $E[k]=v$.

For any vertices $u,v$, consider the interval

$[\mathrm{first}(u),\mathrm{first}(v)] \quad \text{if } \mathrm{first}(u)\le \mathrm{first}(v),$

and the reversed interval otherwise. Let $k^*$ be an index in this interval minimizing $D[k]$:

$D[k^\*]=\min \{D[k]\mid k \in [\mathrm{first}(u),\mathrm{first}(v)]\}.$

Let $w=E[k^*]$. The vertex $w$ lies on every root-to-$u$ path segment appearing after $\mathrm{first}(u)$ and before $\mathrm{first}(v)$ in the tour, since the Euler tour descends and ascends along tree edges and depth strictly decreases only when moving toward ancestors. The minimum depth in the interval is attained at the unique vertex closest to the root among those visited between the first occurrences of $u$ and $v$, hence $w$ is an ancestor of both $u$ and $v$. Any vertex deeper than $w$ cannot be an ancestor of both endpoints, since its occurrence in the Euler tour forces a return above it before reaching the other endpoint. Thus $w$ is the deepest vertex that is an ancestor of both $u$ and $v$, so $w=\mathrm{lca}(u,v)$.

An RMQ instance on the array $D$ with indices restricted to Euler-tour positions therefore returns $\mathrm{lca}(u,v)$ after mapping indices through $E$ and $\mathrm{first}$, so nearest common ancestor queries reduce to range minimum queries with identical query complexity.

Conversely, let $A_1,\ldots,A_n$ be an array. Construct its Cartesian tree $C$ as follows. The inorder traversal of $C$ lists the nodes in the order $1,2,\ldots,n$. The heap property is enforced so that for every node $i$, the value at $i$ is less than the values at all nodes in its subtree, so $A_i$ is the minimum value in the subtree of $i$.

For any interval $[i,j]$, let $t$ be the index attaining $\min{A_k \mid i\le k\le j}$. In the inorder structure of $C$, all nodes in the subtree of $t$ form a contiguous interval containing $t$. If $i\le t\le j$, both $i$ and $j$ lie in the subtree of $t$ or in distinct subtrees attached along the search path in the inorder structure. Every common ancestor of nodes $i$ and $j$ must lie on the unique path in $C$ connecting them, and among these ancestors the node $t$ has strictly minimal value, since any ancestor has value smaller than all values in its subtree only through the heap ordering and cannot exceed $A_t$ without contradicting minimality of $t$ in $[i,j]$. Hence $t$ is the lowest common ancestor of $i$ and $j$ in $C$, so

$\mathrm{lca}_C(i,j)=t=\arg\min_{k\in[i,j]} A_k.$

Thus a data structure that answers nearest common ancestor queries in $O(1)$ time after preprocessing answers range minimum queries with the same asymptotic efficiency by building the Cartesian tree in linear time and translating each RMQ query into one LCA query.

This completes the proof. ∎
