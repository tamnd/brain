**Exercise 85.** [*22*] [22] (K. D. Tocher, 1954.) Imagine that you have a vintage 1950s computer with a drum memory for storing data, and that you need to do some computations with a $32 \times 32 \times 32$ array $a[i, j, k]$, whose subscripts are 5-bit integers in the range $0 \le i, j, k < 32$. Unfortunately your machine has only a very small high-speed memory. You can access only 128 consecutive elements of the array in fast memory at any time. Since your application usually moves from $a[i,j,k]$ to a neighboring position $a[i',j',k']$, where $|i - i'| + |j - j'| + |k - k'| = 1$, you have decided to allocate the array so that, if $i = (i_4 i_3 i_2 i_1 i_0)_2$, $j = (j_4 j_3 j_2 j_1 j_0)_2$, and $k = (k_4 k_3 k_2 k_1 k_0)_2$, the array entry $a[i, j, k]$ is stored in drum location $(k_4 j_4 i_4 k_3 j_3 i_3 k_2 j_2 i_2 k_1 j_1 i_1 k_0 j_0 i_0)_2$. By interleaving the bits in this way, a small change to $i$, $j$, or $k$ will cause only a small change in the address.

Discuss the implementation of this addressing function: (a) How does it change when $i$, $j$, or $k$ changes by $\pm 1$? (b) How would you handle a random access to $a[i,j,k]$, given $i$, $j$, and $k$? (c) How would you detect a "page fault" (namely, the condition that a new segment of 128 elements must be swapped into fast memory from the drum)?

Let the address function be

$$
A(i,j,k) = (k_4 j_4 i_4\, k_3 j_3 i_3\, k_2 j_2 i_2\, k_1 j_1 i_1\, k_0 j_0 i_0)_2.
$$

It is convenient to index bit positions from least significant bit (position $0$). Then the interleaving is

$$
\begin{aligned}
A_0 &= i_0,\quad A_1 = j_0,\quad A_2 = k_0,\\
A_3 &= i_1,\quad A_4 = j_1,\quad A_5 = k_1,\\
&\;\;\vdots\\
A_{3b} &= i_b,\quad A_{3b+1} = j_b,\quad A_{3b+2} = k_b.
\end{aligned}
$$

Thus each coordinate occupies every third bit position of the 15-bit word.

## (a) Effect of changing $i$, $j$, or $k$ by $\pm 1$

### Change in $i \mapsto i \pm 1$

The value $i$ is stored in the sparse bit string

$$
(i_4\, i_3\, i_2\, i_1\, i_0)_2
$$

embedded into bit positions $0,3,6,9,12$ of the address.

Adding $1$ to $i$ is therefore equivalent to adding $1$ to a 5-bit binary number whose bits are spaced three apart in the full address.

#### Carry structure

A carry propagates only through the $i$-bit positions:

- If $i_0 = 0$, then only $A_0$ flips and no higher $i$-bit changes.
- If $i_0 = 1$, then $A_0$ becomes $0$ and a carry is generated to $i_1$, i.e., position $3$.
- In general, if $i_0 = i_1 = \cdots = i_{r-1} = 1$ and $i_r = 0$, then:

$$
i_0,\dots,i_{r-1} \text{ flip to } 0,\quad i_r \text{ flips to } 1,
$$

and all higher $i_b$ for $b > r$ remain unchanged.

In the full address, this means:

- Exactly the bits at positions $0,3,6,\dots,3r$ are modified,
- The number of affected address bits is $r+1 \le 5$,
- No bits in the $j$- or $k$-lanes are changed.

#### Worst case

If $i = (11111)_2$, then incrementing produces:

$$
(00000)_2 \text{ with carry out of the } i\text{-field},
$$

so all five $i$-bit positions of the address flip. Thus up to 5 address bits change.

### Change in $i \mapsto i - 1$

Borrow propagation is identical in structure but reversed:

- If $i_0 = 1$, only position $0$ changes.
- If $i_0 = 0$, borrow propagates through consecutive $i$-bit positions:

$$
i_0,i_1,\dots,i_{r-1} = 0 \to 1,\quad i_r: 1 \to 0.
$$

Thus again only the positions $0,3,6,9,12$ are affected, with a contiguous prefix of these positions toggled. In the worst case, all five $i$-bits change.

### Changes in $j$ and $k$

The same analysis applies verbatim:

- $j \pm 1$ affects only bit positions $1,4,7,10,13$,
- $k \pm 1$ affects only bit positions $2,5,8,11,14$,
- Each behaves as a 5-bit binary counter embedded in interleaved positions,
- Carry/borrow propagates only within its own 5-bit subsequence.

### Summary of (a)

A unit change in any coordinate:

- modifies only one of the three disjoint bit-lanes,
- causes a binary ripple carry/borrow within that lane,
- affects a contiguous run of that coordinate’s 5 bits,
- changes between $1$ and $5$ bits of the 15-bit address,
- never mixes bits between $i$, $j$, and $k$.

## (b) Random access computation of $A(i,j,k)$

The address is formed by bit interleaving. For each $b = 0,\dots,4$,

place bits $(i_b,j_b,k_b)$ into positions $(3b,3b+1,3b+2)$.

A direct implementation is:

$$
A = \sum_{b=0}^{4} \left(i_b \cdot 2^{3b} + j_b \cdot 2^{3b+1} + k_b \cdot 2^{3b+2}\right).
$$

Algorithmically:

- extract bits of $i,j,k$,
- shift each into position $3b,3b+1,3b+2$,
- combine by bitwise OR.

Equivalently, using precomputed “spread” tables $I[i],J[j],K[k]$ that expand each 5-bit index into a 15-bit word with zeros inserted between bits:

$$
A(i,j,k) = I[i] \,\lor\, J[j] \,\lor\, K[k].
$$

This gives constant-time random access using a fixed number of shifts, masks, and logical OR operations.

## (c) Page fault detection for 128-word fast memory

Fast memory holds $128 = 2^7$ consecutive drum locations.

Let

$$
A = A(i,j,k).
$$

Then:

- page number: $\;p = \lfloor A / 128 \rfloor = A \gg 7$,
- offset: $\;A \bmod 128$.

A page fault occurs exactly when the required page differs from the resident page $P$:

$$
(A \gg 7) \ne P.
$$

Equivalently:

$$
A \notin [128P,\,128P+127].
$$

On a page fault, the system loads the block starting at:

$$
128 \cdot (A \gg 7).
$$

## Final observation

This addressing scheme converts each coordinate into a separate 5-bit Gray-free binary counter embedded in disjoint bit lanes. As a result, nearest-neighbor moves in $(i,j,k)$-space induce highly localized changes in the 15-bit address, with bounded carry propagation confined to a single coordinate lane.
