**Exercise 198.** &#9654; [*21*] [21] Unicode characters are often represented as strings of bytes using a scheme
called UTF-8, which is the encoding of exercise 196 restricted to integers in the range
0 ≤x < 220+216. Notice that UTF-8 efficiently preserves the standard ASCII character
set (the codepoints with x < 27), and that it is quite different from UTF-16.
Let α1 be the first byte of a UTF-8 string α(x). Show that there are reasonably
small integer constants a, b, and c such that only four bitwise operations
(a ≫((α1 ≫b) & c)) & 3
suffice to determine the number l −1 of bytes between α1 and the end of α(x).

We restart from the UTF-8 leading-byte structure.

For valid UTF-8 leading bytes (exercise restriction $0 \le x < 2^{20} + 2^{16}$), the first byte $\alpha_1$ has exactly one of the following forms:

$$
\begin{aligned}
0xxxxxxx &\quad\Rightarrow\quad l-1 = 0,\\
110xxxxx &\quad\Rightarrow\quad l-1 = 1,\\
1110xxxx &\quad\Rightarrow\quad l-1 = 2,\\
11110xxx &\quad\Rightarrow\quad l-1 = 3.
\end{aligned}
$$

### Step 1. Choose a correct coarse classification

Let

$$
y = \alpha_1 \gg 4.
$$

This keeps the high 4 bits of the byte. We compute the possible ranges:

- ASCII: $0x00$–$0x7F$ gives $y \in \{0,\dots,7\}$,
- 2-byte leading: $0xC0$–$0xDF$ gives $y \in \{12,13\}$,
- 3-byte leading: $0xE0$–$0xEF$ gives $y = 14$,
- 4-byte leading: $0xF0$–$0xF7$ gives $y = 15$.

These classes are already disjoint in $y$. No further masking is required, and introducing one would only destroy information.

Hence we set

$$
b = 4,\qquad c = 15,
$$

so that

$$
(\alpha_1 \gg b)\,\&\,c = y.
$$

### Step 2. Define the lookup function

We define a function $f(y) = l-1$ on $0 \le y \le 15$. From the classification above:

$$
\begin{aligned}
f(y) &= 0 && \text{for } y = 0,1,\dots,7,\\
f(12) &= 1,\quad f(13)=1,\\
f(14) &= 2,\\
f(15) &= 3.
\end{aligned}
$$

All other values of $y$ never occur for valid UTF-8 leading bytes, so they may be assigned arbitrarily, for instance $0$.

### Step 3. Encode the lookup table into a shift constant

We use Knuth’s shift-table construction: choose $a$ such that

$$
(a \gg y)\,\&\,3 = f(y).
$$

A standard construction is base-4 digit encoding:

$$
a = \sum_{y=0}^{15} f(y)\,4^y.
$$

Substituting the nonzero values:

$$
a = 1\cdot 4^{12} + 1\cdot 4^{13} + 2\cdot 4^{14} + 3\cdot 4^{15}.
$$

This is a valid integer constant.

### Step 4. Verification of correctness

Each UTF-8 leading-byte class maps to a unique value of $y$:

- ASCII maps to $y \in [0,7]$, all giving $f(y)=0$,
- 2-byte leading bytes map to $y \in \{12,13\}$, both giving $f(y)=1$,
- 3-byte leading bytes map to $y=14$, giving $f(y)=2$,
- 4-byte leading bytes map to $y=15$, giving $f(y)=3$.

Thus $f(y)$ is well-defined and consistent on all valid cases, and the shift-table identity guarantees the required result.

### Step 5. Final expression

Take

$$
b = 4,\qquad c = 15,\qquad
a = 4^{12} + 4^{13} + 2\cdot 4^{14} + 3\cdot 4^{15}.
$$

Then for every valid UTF-8 leading byte $\alpha_1$,

$$
(a \gg ((\alpha_1 \gg 4)\,\&\,15))\,\&\,3 = l-1.
$$

### Final answer

$$
\boxed{
b = 4,\quad c = 15,\quad
a = 4^{12} + 4^{13} + 2\cdot 4^{14} + 3\cdot 4^{15}
}
$$
