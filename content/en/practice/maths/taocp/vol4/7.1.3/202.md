**Exercise 202.** [*20*] [20] What two instructions will change an octabyte’s nonzero wydes to #ffff?

**Verified:** no  
**Solve time:** 4m27s

The failure in the proposed solution comes entirely from an incorrect understanding of `MOR`. Once the correct MMIX semantics are used, the construction becomes simple and valid.

## Key MMIX fact (correct semantics)

In MMIX, `MOR` is not an ordinary bytewise register OR. It is a _wyde-wise folding operation_:

For each wyde (16-bit block), `MOR` computes the bitwise OR of the corresponding bytes within that wyde. Thus it collapses information inside each wyde but does not mix data between different wydes.

Consequently:

- A wyde of `x` is zero if and only if all its bytes are zero.
- After `MOR $t,$x,$x`, each wyde of `t` is zero exactly when the corresponding wyde of `x` is zero, and otherwise it becomes nonzero in every byte position where any byte of the wyde was nonzero.

This is precisely the “nonzero wyde detector” that the previous solution failed to justify.

## Step 1: detect nonzero wydes

```
MOR $t,$x,$x
```

For each wyde:

- If the wyde of $x$ is 0, then all bytes are 0, so the result is 0.
- If the wyde of $x$ is nonzero, then at least one byte is nonzero, so the OR within the wyde produces a nonzero byte pattern.

Thus each wyde of $t$ is zero if and only if the corresponding wyde of $x$ is zero.

We now have a correct per-wyde nonzero indicator.

## Step 2: expand nonzero wydes to `#ffff`

```
MOR $t,$t,#ffff
```

Now we use the same per-wyde OR behavior with a constant.

Within each wyde:

- If the wyde of $t$ is 0, then OR with `#ffff` leaves it unchanged as `#0000`.
- If the wyde of $t$ is nonzero, then at least one byte position in the wyde is nonzero, so OR with `#ffff` forces both bytes of the wyde to become `#ff`, yielding `#ffff`.

This works because the constant `#ffff` supplies ones in both byte positions of every wyde, and the wyde-wise OR propagates that only in those wydes that were already nonzero.

## Correctness

After the first instruction, each wyde encodes whether the corresponding wyde of $x$ is zero or not. The second instruction then replaces every nonzero wyde by `#ffff` while leaving zero wydes unchanged.

Thus the transformation is exactly:

$$
x^{(i)} \mapsto
\begin{cases}
\#0000 & \text{if } x^{(i)} = 0,\\
\#ffff & \text{if } x^{(i)} \neq 0.
\end{cases}
$$

which is equivalent to the required specification.

## Final answer

```
MOR $t,$x,$x
MOR $t,$t,#ffff
```
