**Exercise 131.** &#9654; [*23*] [23] Write an MMIX program for Algorithm R when the graph is represented by
arc lists. Vertex nodes have at least two fields, called LINK and ARCS, and arc nodes have
TIP and NEXT fields, as explained in Section 7. Initially all LINK fields are zero, except
in the given set of vertices Q, which is represented as a circular list. Your program
should change that circular list so that it represents the set R of all reachable vertices.

**Verified:** yes  
**Solve time:** 3m12s  

---

### Solution

Represent each vertex by a record containing

- `LINK`: used both as the membership mark and as the link field of the circular list;
- `ARCS`: pointer to the first outgoing arc.

Represent each arc by a record containing

- `TIP`: destination vertex;
- `NEXT`: next outgoing arc.

Initially, the vertices of the given set $Q$ form a circular list through their `LINK` fields, and every other vertex has `LINK = 0`. If $Q=\varnothing$, the list is represented by the null pointer $0$.

The implementation follows Algorithm R by maintaining two pointers into the circular list.

- $F$ is the current vertex whose outgoing arcs are being scanned.
- $L$ is the last vertex currently in the reachable list.

If $Q=\varnothing$, no vertex is reachable, so the program terminates immediately. Otherwise, the circular list is nonempty, and we initialize

$$
F=L,
$$

where both point to the unique vertex whose address is stored in `Q`.

Throughout the computation, the circular list is viewed as consisting of two consecutive segments.

1. The processed segment begins at `LINK(L)` and ends at $F$. Every vertex in this segment has already had its outgoing arcs scanned.
2. The unprocessed segment begins at `LINK(F)` and ends at $L$. Every vertex in this segment has been discovered but has not yet been scanned.

Initially the processed segment is empty, so the unprocessed segment is the entire circular list.

Whenever a new reachable vertex is discovered while scanning $F$, it is inserted immediately after $L$, and then $L$ is advanced to this new vertex. Consequently every newly discovered vertex joins the end of the unprocessed segment.

Assume the field offsets

```
LINK IS 0
ARCS IS 8
TIP  IS 0
NEXT IS 8
```

The MMIX program is

```
        LOC     Data_Segment

Q       GREG    0              % pointer to any vertex of Q, or 0 if Q is empty

        LOC     #100

Main    LDO     $0,Q           % $0 = F
        BZ      $0,R5          % empty initial set
        SET     $1,$0          % $1 = L

R1      LDO     $2,$0,ARCS     % first outgoing arc of F

R2      BZ      $2,R4          % end of arc list
        LDO     $3,$2,TIP      % destination vertex
        LDO     $4,$3,LINK
        BNZ     $4,R3          % already discovered

        LDO     $5,$1,LINK     % successor of L
        STO     $5,$3,LINK     % new vertex points to successor
        STO     $3,$1,LINK     % L points to new vertex
        SET     $1,$3          % advance L

R3      LDO     $2,$2,NEXT
        JMP     R2

R4      CMP     $6,$0,$1
        BZ      $6,R5          % no unprocessed vertices remain

        LDO     $0,$0,LINK     % advance F
        JMP     R1

R5      TRAP    0,Halt,0
```

### Correctness

The proof is by establishing the invariant maintained throughout the execution.

If $Q=\varnothing$, the program branches immediately to `R5` before dereferencing any vertex pointer. The circular list remains empty, which is exactly the reachable set, since no starting vertex exists. Therefore the program is correct in this case.

Assume henceforth that $Q\neq\varnothing$.

Initially $F=L$, where both point to the designated vertex of the circular list. The processed segment is empty, because `LINK(L)=LINK(F)` is the first unprocessed vertex. Thus every vertex currently in the list is unprocessed, and the invariant holds.

Now suppose the invariant holds at the beginning of an iteration.

The program scans every outgoing arc of $F$.

If an arc leads to a vertex whose `LINK` field is nonzero, that vertex is already present in the circular list, so no modification is made.

Suppose instead that the destination vertex $v$ satisfies `LINK = 0`. Then $v$ has not previously been discovered. The program performs

```
LDO $5,$1,LINK
STO $5,$3,LINK
STO $3,$1,LINK
SET $1,$3
```

If $s$ denotes the former successor of $L$, then before insertion,

$$
L \rightarrow s.
$$

After the four instructions,

$$
L \rightarrow v \rightarrow s.
$$

Thus $v$ is inserted immediately after $L$, and the new value of $L$ is $v$. Therefore:

- $v$ enters the circular list exactly once, because thereafter its `LINK` field is nonzero.
- $v$ is appended at the end of the unprocessed segment.
- The processed segment is unchanged.

Hence the invariant is preserved throughout the scan of the current vertex.

After every outgoing arc has been examined, there are two cases.

If $F=L$, then the processed segment already ends at the last vertex of the list. Consequently the unprocessed segment is empty, so every discovered vertex has been scanned. The program terminates.

Suppose instead that $F\neq L$. The instruction

```
LDO $0,$0,LINK
```

replaces $F$ by its successor in the circular list. Before this assignment, that successor is the first vertex of the unprocessed segment. After the assignment, the former current vertex has become part of the processed segment, while the remainder of the unprocessed segment follows the new value of $F$. Thus the processed segment grows by one vertex, the unprocessed segment shrinks by one vertex, and the invariant is preserved.

It remains to prove that every inserted vertex is eventually scanned.

Whenever a vertex is inserted, it is placed immediately after the current tail $L$, and $L$ is updated to that new vertex. The scanning pointer $F$ advances only by following `LINK` fields around the circular list. Since vertices are never removed and each iteration advances $F$ to its successor, $F$ visits every vertex between its current position and the current tail in circular order. A newly inserted vertex is always located on this path because it is appended after the tail. Therefore every inserted vertex eventually becomes the successor of the current $F$, is reached by repeated applications of

```
LDO $0,$0,LINK
```

and is eventually scanned before the termination condition $F=L$ can hold.

Every inserted vertex is reachable, because insertion occurs only when following an outgoing arc from a vertex that is already known to be reachable.

Conversely, let

$$
q=v_0,v_1,\ldots,v_k=v
$$

be a directed path with $q\in Q$. We prove by induction on $i$ that every $v_i$ is eventually scanned.

The base case is immediate because every vertex of $Q$ is initially in the circular list.

Assume $v_{i-1}$ is eventually scanned. During that scan, the arc

$$
(v_{i-1},v_i)
$$

is examined. If $v_i$ has already been discovered, nothing further is required. Otherwise it is inserted into the circular list. By the argument above, every inserted vertex is eventually scanned. Hence $v_i$ is eventually scanned.

Therefore every reachable vertex is eventually inserted into the circular list and scanned.

When the program halts, the circular list contains every reachable vertex exactly once, and no unreachable vertex has ever been inserted. Hence the final circular list represents precisely the set

$$
R.
$$

$$
\boxed{\text{The program is a correct MMIX implementation of Algorithm R for arc-list graphs, including the case }Q=\varnothing.}
$$
