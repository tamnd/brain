**Exercise 199.** &#9654; [*23*] [23] A person might try to encode #a as #c08a or #e0808a or #f080808a in
UTF-8, because the obvious decoding algorithm produces the same result in each case.
But such unnecessarily long forms are illegal, because they could lead to security holes.
Suppose α1 and α2 are bytes such that α1 ≥#80 and #80 ≤α2 < #c0. Find
a branchless way to decide whether α1 and α2 are the first two bytes of at least one
legitimate UTF-8 string α(x).

The original solution fails because it ignores that the legality of the second byte depends on the first byte in boundary cases (E0, ED, F0, F4), where UTF-8 imposes non-uniform constraints to avoid overlong encodings, surrogate halves, and values above $U+10FFFF$.

We restart from the correct characterization of all UTF-8 strings whose first two bytes are fixed as $\alpha_1,\alpha_2$, with

$$
0x80 \le \alpha_1,\qquad 0x80 \le \alpha_2 < 0xC0.
$$

Only the cases where $\alpha_1$ is a valid leading byte matter. Thus $\alpha_1 \in [0xC2,0xF4]$. The remaining issue is whether there exists at least one completion byte (and a full continuation) producing a valid Unicode scalar value.

## 1. Structure of the problem

We examine each UTF-8 length class:

### (i) Two-byte sequences: $\texttt{C2–DF}$

For $\alpha_1 \in [0xC2,0xDF]$, any $\alpha_2 \in [0x80,0xBF]$ is valid, and completion is always possible. No further restriction.

### (ii) Three-byte sequences: $\texttt{E0–EF}$

Write:

$$
x = ((\alpha_1 \,\&\, 0x0F) \ll 12) + ((\alpha_2 \,\&\, 0x3F) \ll 6) + \beta,
\quad 0x80 \le \beta < 0xC0.
$$

The only restriction affecting existence is the _first continuation boundary_, which prevents overlong encodings:

- $\alpha_1 = \texttt{E0}$: must have $\alpha_2 \ge 0xA0$
- $\alpha_1 = \texttt{ED}$: must have $\alpha_2 \le 0x9F$ (to avoid surrogate range)
- otherwise ($\texttt{E1–EC, EE–EF}$): no restriction beyond $\alpha_2 \in [0x80,0xBF]$

For each admissible pair, a suitable $\beta$ always exists.

### (iii) Four-byte sequences: $\texttt{F0–F4}$

Now:

$$
x = ((\alpha_1 \,\&\, 0x07) \ll 18)
+ ((\alpha_2 \,\&\, 0x3F) \ll 12)
+ (\gamma \ll 6)
+ \delta,
$$

with $0x80 \le \gamma,\delta < 0xC0$.

Boundary constraints:

- $\texttt{F0}$: $\alpha_2 \ge 0x90$ (no overlong encoding)
- $\texttt{F1–F3}$: no restriction beyond continuation byte range
- $\texttt{F4}$: $\alpha_2 \le 0x8F$ (ensures $x \le U+10FFFF$)

Again, for any admissible prefix, suitable $\gamma,\delta$ exist.

## 2. Reduction to a prefix feasibility test

Thus the problem reduces to checking whether $(\alpha_1,\alpha_2)$ satisfies:

1. Valid leading byte:

$$
0xC2 \le \alpha_1 \le 0xF4
$$

1. Second-byte constraints in boundary cases:

- if $\alpha_1 = 0xE0$: $\alpha_2 \ge 0xA0$
- if $\alpha_1 = 0xED$: $\alpha_2 \le 0x9F$
- if $\alpha_1 = 0xF0$: $\alpha_2 \ge 0x90$
- if $\alpha_1 = 0xF4$: $\alpha_2 \le 0x8F$

All other cases automatically allow any $\alpha_2 \in [0x80,0xBF]$, which is given.

## 3. Branchless formulation

We encode comparisons as masks.

Define indicator-style predicates (0 or 1 values):

$$
\begin{aligned}
e0 &= (\alpha_1 = 0xE0),\\
ed &= (\alpha_1 = 0xED),\\
f0 &= (\alpha_1 = 0xF0),\\
f4 &= (\alpha_1 = 0xF4).
\end{aligned}
$$

These can be expressed branchlessly using XOR and zero tests in the usual Knuth style, but we keep them symbolic for clarity.

Now define violation conditions:

$$
\begin{aligned}
V_1 &= e0 \land (\alpha_2 < 0xA0),\\
V_2 &= ed \land (\alpha_2 > 0x9F),\\
V_3 &= f0 \land (\alpha_2 < 0x90),\\
V_4 &= f4 \land (\alpha_2 > 0x8F).
\end{aligned}
$$

Let:

$$
V = V_1 \lor V_2 \lor V_3 \lor V_4.
$$

Also require:

$$
L = (0xC2 \le \alpha_1 \le 0xF4).
$$

## 4. Final branchless decision

The required predicate is:

$$
\boxed{
\text{valid} = L \land \neg V
}
$$

## 5. Fully branchless bitwise form

A standard arithmetic encoding uses the fact that comparisons can be converted into sign bits of differences:

Let:

$$
[x < y] = (x - y) \gg 31,\quad \text{for 32-bit signed arithmetic.}
$$

Then each constraint becomes a mask expression, and:

$$
V = (e0 \,\&\, (\alpha_2 - 0xA0) \gg 31)
\;\lor\;
(ed \,\&\, (0x9F - \alpha_2) \gg 31)
\;\lor\;
(f0 \,\&\, (\alpha_2 - 0x90) \gg 31)
\;\lor\;
(f4 \,\&\, (0x8F - \alpha_2) \gg 31).
$$

Finally:

$$
\boxed{
\text{valid} =
(\alpha_1 \in [0xC2,0xF4]) \,\&\,
\neg V.
}
$$

## Conclusion

The correct solution must incorporate **context-dependent second-byte constraints for E0, ED, F0, and F4**, since UTF-8 validity is not a per-byte property but a constrained prefix system ensuring uniqueness, no overlong encodings, and bounded Unicode scalar values.

The earlier reduction to a single interval $[0xC2,0xF4]$ is insufficient because it ignores exactly these boundary interactions.
