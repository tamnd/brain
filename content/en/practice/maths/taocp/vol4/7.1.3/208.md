**Exercise 208.** &#9654; [*23*] [23] What’s a fast way for MMIX to transpose an 8 × 8 Boolean matrix?

**Verified:** no

Represent the $8\times 8$ Boolean matrix as a single 64-bit MMIX register $x$, stored in row-major order:

$$
x = \sum_{i=0}^{7}\sum_{j=0}^{7} a_{ij} 2^{8i+j},
\quad a_{ij}\in\{0,1\}.
$$

The transpose maps bit position $(i,j)$ to $(j,i)$, i.e.,

$$
8i + j \longmapsto 8j + i.
$$

Thus the problem is a fixed permutation of the 64 bits, realizable by a small bit-swap network.

A fast MMIX method uses a standard SWAR-style three-stage butterfly of XOR-shifts and masks.

## Algorithm

Let $x$ be the input 64-bit word. Compute:

### Stage 1 (swap within 1-bit neighborhoods inside each 8-bit group)

$$
t = (x \oplus (x \ll 7)) \;\&\; \texttt{0xAA00AA00AA00AA00}
$$

$$
x = x \oplus t \oplus (t \gg 7)
$$

### Stage 2 (swap across 2-bit groups within each 16-bit block)

$$
t = (x \oplus (x \ll 14)) \;\&\; \texttt{0xCCCC0000CCCC0000}
$$

$$
x = x \oplus t \oplus (t \gg 14)
$$

### Stage 3 (swap across 4-bit groups, completing the $8\times 8$ transpose)

$$
t = (x \oplus (x \ll 28)) \;\&\; \texttt{0xF0F0F0F0F0F0F0F0}
$$

$$
x = x \oplus t \oplus (t \gg 28)
$$

The resulting $x$ contains the transposed matrix in the same row-major encoding.

## Correctness argument

Write each bit position as a 6-bit index:

$$
k = 8i + j = (i_2 i_1 i_0 j_2 j_1 j_0)_2.
$$

Transposition swaps row and column indices:

$$
(i_2 i_1 i_0 j_2 j_1 j_0) \mapsto (j_2 j_1 j_0 i_2 i_1 i_0).
$$

Each stage performs a controlled swap of increasingly large bit fields:

- Stage 1 exchanges the least significant bit of $i$ and $j$.
- Stage 2 exchanges the middle bit of $i$ and $j$.
- Stage 3 exchanges the most significant bit of $i$ and $j$.

The masks isolate exactly those bit pairs that differ in the relevant bit position, and the XOR-shift form implements a conditional swap without branching. After all three stages, the 3-bit halves are fully exchanged, giving the transpose permutation.

## Efficiency

Each stage uses:

- one shift left,
- one shift right,
- one mask,
- a constant number of XORs.

Total cost is $O(1)$ word operations (specifically 3 stages), so the transpose is performed in a small fixed number of MMIX instructions, independent of any loop over entries.

## MMIX interpretation

On MMIX, each line corresponds to a few instructions of the form:

- shift left: `SL`
- shift right: `SR`
- bitwise and: `AND`
- xor: `XOR`

So the entire transpose runs in constant time using only a handful of instructions, making it a fast in-register matrix transpose.
