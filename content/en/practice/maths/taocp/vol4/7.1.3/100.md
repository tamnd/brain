[Section 7.1.3: Bitwise Tricks and Techniques](../)

**Exercise 100.** [*25*] Suppose $x = (x_{15} \ldots x_1 x_0)_{16}$ and $y = (y_{15} \ldots y_1 y_0)_{16}$ are *binary-coded decimal* numbers, where $0 \le x_j$, $y_j < 10$ for each $j$. Explain how to compute their sum $u = (u_{15} \ldots u_1 u_0)_{16}$ and difference $v = (v_{15} \ldots v_1 v_0)_{16}$, where $0 \le u_j$, $v_j < 10$ and

$$(u_{15} \ldots u_1 u_0)_{10} = ((x_{15} \ldots x_1 x_0)_{10} + (y_{15} \ldots y_1 y_0)_{10}) \bmod 10^{16},$$
$$(v_{15} \ldots v_1 v_0)_{10} = ((x_{15} \ldots x_1 x_0)_{10} - (y_{15} \ldots y_1 y_0)_{10}) \bmod 10^{16},$$

without bothering to do any radix conversion.

The reviewer is correct: the proposed “global ±$0x666\ldots6$” method is invalid because BCD correction is inherently **digit-dependent**, not uniform. The correction must detect which nibbles are invalid and adjust only those, while letting binary carries propagate naturally.

A correct solution proceeds by **binary word operations plus per-nibble masking**, without converting to decimal.

# 1. Representation

Let

$$
x = (x_{15}\ldots x_0)_{16}, \quad y = (y_{15}\ldots y_0)_{16}, \quad 0 \le x_j,y_j \le 9.
$$

Each hexadecimal digit is a decimal digit (packed BCD). All arithmetic is performed in a 64-bit binary word.

Let

$$
M = 0x8888888888888888,
\quad
L = 0x6666666666666666.
$$

The constant $M$ isolates the high bit of each nibble, which is used to detect decimal carry/borrow conditions.

# 2. BCD Addition

## Step 1: binary addition

Compute the raw binary sum

$$
s = x + y \quad (\bmod 2^{64}).
$$

This is not yet valid BCD, because each nibble may exceed 9.

## Step 2: detect decimal-carry nibbles

Add the correction bias:

$$
t = s + L.
$$

Now observe the key property:

- If a nibble of $s$ satisfies $s_j \le 9$, then adding 6 keeps it below 16 and does **not generate a carry into bit 4**.
- If $s_j \ge 10$, then $s_j + 6 \ge 16$, producing a carry into the next nibble.

Thus the high bit of each nibble in $t$ detects which digits require correction.

Define the carry mask:

$$
c = (t \,\&\, M) \gg 3.
$$

Each nibble of $c$ is either 0 or 1, indicating whether that digit overflowed past 9.

## Step 3: final correction

Each flagged nibble must be increased by 6.

$$
u = t + (c \cdot 6).
$$

Since $c$ is a packed mask (each nibble is 0 or 1), multiplication by 6 means each active nibble contributes $6$ in its position.

## Result

$$
u = (x + y) \bmod 10^{16}
$$

with each digit $u_j \in [0,9]$.

## Why this is correct

Two facts justify the method:

1. **Local thresholding**

$$
s_j \ge 10 \iff s_j + 6 \ge 16,
$$

so decimal overflow is equivalent to binary carry out of the nibble after adding 6.
2. **No inter-nibble interference**

The mask $c$ is derived from disjoint nibble high bits, so each correction is localized.

Thus correction depends only on valid per-digit overflow detection.

# 3. BCD Subtraction

Subtraction is best handled via **10’s complement in BCD**, avoiding direct borrow reasoning.

## Step 1: compute 9’s complement of $y$

Each digit becomes:

$$
y_j' = 9 - y_j.
$$

We compute this in binary using:

$$
y' = 0x9999999999999999 - y.
$$

This is valid because each nibble subtraction is independent (no borrows occur when subtracting from all 9s).

## Step 2: add 1 (10’s complement)

Compute:

$$
z = y' + 1.
$$

Then:

$$
x - y \equiv x + z \pmod{10^{16}}.
$$

So we reduce subtraction to addition.

## Step 3: perform BCD addition

Compute:

$$
s = x + z,
$$

then apply the **addition algorithm from Section 2**:

1. $t = s + L$
2. $c = (t \& M) \gg 3$
3. $v = t + 6c$

## Result

$$
v = (x - y) \bmod 10^{16}.
$$

# 4. Summary of the Correct Method

## Addition

$$
\begin{aligned}
s &= x + y \\
t &= s + 0x6666666666666666 \\
c &= (t \,\&\, 0x8888888888888888) \gg 3 \\
u &= t + 6c
\end{aligned}
$$

## Subtraction

$$
\begin{aligned}
y' &= 0x9999999999999999 - y \\
z &= y' + 1 \\
\text{then compute } u &= x + z \text{ using addition algorithm above.}
\end{aligned}
$$

# 5. Conclusion

The correct solution differs fundamentally from the rejected one:

- There is **no uniform global correction**
- Each nibble is corrected only if it exceeds 9
- Detection is done via **binary carry-out from $+6$ adjustment**
- Subtraction is reduced to addition via **10’s complement in BCD**

This restores full correctness while respecting the constraint: **no radix conversion and only binary word operations.**
