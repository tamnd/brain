**Exercise 35.** &#9654; [*M26*] According to Reitwiesner's theorem, exercise 4.1–34, every integer $n$ has a unique representation $n = n^+ - n^-$ such that $n^+ \mathbin{\&} n^- = (n^+ \mid n^-) \mathbin{\&} ((n^+ \mid n^-) \gg 1) = 0$. Show that $n^+$ and $n^-$ can be calculated quickly with bitwise operations. *Hint:* Prove the identity $(x \oplus 3x) \mathbin{\&} ((x \oplus 3x) \gg 1) = 0$.

The proposed solution fails because it never constructs the Reitwiesner representation from $n$. The correct approach is to directly compute the unique non-adjacent signed-digit expansion guaranteed by Reitwiesner’s theorem, and then observe that $n^+$ and $n^-$ are simple bit masks derived from those digits.

## 1. Reitwiesner (non-adjacent signed-digit) form

Reitwiesner’s theorem states that every integer $n$ has a unique expansion

$$
n = \sum_{k \ge 0} d_k 2^k,
\qquad d_k \in \{-1,0,1\},
$$

such that no two consecutive digits are nonzero:

$$
d_k \ne 0 \implies d_{k+1} = 0.
$$

Define bit masks

$$
n^+ = \sum_{d_k = 1} 2^k,
\qquad
n^- = \sum_{d_k = -1} 2^k,
\qquad
u = n^+ \mid n^-.
$$

Then automatically:

$$
n = n^+ - n^-,
\qquad
n^+ \,\&\, n^- = 0,
\qquad
u \,\&\, (u \gg 1) = 0.
$$

Thus the problem reduces to computing the digits $d_k$ quickly.

## 2. Local digit rule (key fact)

Let $r$ be the current residual value during construction. At each step:

- If $r$ is even, set $d_0 = 0$.
- If $r$ is odd, examine $r \bmod 4$:

$$
d_0 =
\begin{cases}
+1, & r \equiv 1 \pmod 4,\\
-1, & r \equiv 3 \pmod 4.
\end{cases}
$$

Then update:

$$
r \leftarrow \frac{r - d_0}{2}.
$$

### Why this works

If $r \equiv 3 \pmod 4$, choosing $d_0 = -1$ forces

$$
r - d_0 = r + 1 \equiv 0 \pmod 4,
$$

so division by 2 introduces no carry interaction in the next digit. This is exactly what enforces the “no adjacent nonzero digits” property.

## 3. Bitwise implementation

We compute $n^+$ and $n^-$ simultaneously while scanning bits.

Let:

$$
r \leftarrow n,\quad
p \leftarrow 0,\quad
m \leftarrow 0,
$$

where $p = n^+$, $m = n^-$.

We iterate over machine words using only shifts, masks, and arithmetic:

### Step per iteration

1. If $r = 0$, stop.
2. Let

$$
d = r \,\&\, 3
\quad (\text{extract two least significant bits}).
$$
3. Determine digit:

- if $d = 1$, set $d_0 = +1$,
- if $d = 3$, set $d_0 = -1$.
4. Update masks:

$$
p \leftarrow p \mid (d_0 = +1 ? 1 : 0)\cdot 2^k,
\quad
m \leftarrow m \mid (d_0 = -1 ? 1 : 0)\cdot 2^k.
$$
5. Update residual:

$$
r \leftarrow (r - d_0) \gg 1.
$$

Each step removes one binary digit of the signed expansion.

## 4. Fast bitwise formulation

We can express the digit extraction compactly using bit operations.

Define:

$$
\text{odd}(r) = r \,\&\, 1.
$$

Then:

$$
d_0 =
(\text{odd}(r)) \cdot \bigl(2 - (r \,\&\, 3)\bigr).
$$

This yields:

- $r \& 3 = 01 \Rightarrow d_0 = +1$,
- $r \& 3 = 11 \Rightarrow d_0 = -1$.

Thus all decisions are constant-time bitwise operations.

## 5. Construction of $n^+$ and $n^-$

We accumulate:

- If $d_0 = +1$, set the current bit in $n^+$.
- If $d_0 = -1$, set the current bit in $n^-$.

Since digits are non-adjacent, these assignments never conflict, and we obtain:

$$
n^+ \,\&\, n^- = 0,
\qquad
u := n^+ \mid n^- \text{ has no adjacent 1s.}
$$

## 6. Correctness

### (i) Representation

At each step,

$$
r = d_0 + 2r',
$$

so unfolding gives

$$
n = \sum_k d_k 2^k = n^+ - n^-.
$$

### (ii) Non-adjacency

If $d_k \ne 0$, the update forces $r$ to become even before the next step, so $d_{k+1} = 0$. Hence:

$$
u \,\&\, (u \gg 1) = 0.
$$

### (iii) Uniqueness

Reitwiesner’s theorem guarantees uniqueness of such digit sequences, so the constructed $n^+, n^-$ are the canonical ones.

## 7. Complexity

Each iteration uses only:

- bitwise AND,
- shifts,
- a constant number of additions/subtractions.

Hence for a word of size $w$, the computation is:

$$
O(w) \text{ bit operations} = O(1) \text{ word operations}.
$$

## 8. Role of the hint

The identity

$$
(x \oplus 3x) \,\&\, ((x \oplus 3x) \gg 1) = 0
$$

is a structural fact showing how local parity interactions prevent adjacent 1-bits. It reflects the same mechanism as the carry-elimination step in the signed-digit construction above: enforcing that after each digit choice, the next position becomes independent.

## Final result

We compute the Reitwiesner decomposition by the standard signed-digit recurrence, producing bit masks $n^+$ and $n^-$ such that:

$$
n = n^+ - n^-,
\qquad
n^+ \,\&\, n^- = 0,
\qquad
(n^+ \mid n^-) \,\&\, ((n^+ \mid n^-) \gg 1) = 0,
$$

using only constant-time bitwise operations per digit.
