**Exercise 24.** &#9654; [*M30*] Program 1.3.2′F instructed MMIX to produce a table of the first five hundred prime numbers, using trial division to establish primality. Write an MMIX program that uses the "sieve of Eratosthenes" (exercise 4.5.4–8) to build a table of all odd primes that are less than $N$, packed into octabytes $Q_0, Q_1, \ldots, Q_{N/128-1}$ as in $(27)$. Assume that $N \le 2^{32}$, and that it's a multiple of 128. What is the running time when $N = 3584$?

The key failure in the proposed solution is not efficiency but a structural inconsistency: it mixes **bit-index space**, **integer space**, and **word-index space** in control conditions. A correct solution must keep these three domains strictly separated.

A clean fix is to:

- Use **integer values** for loop control ($p$, $x$).
- Use **bit indexing only for storage access**.
- Use **no word/bit quantities in termination tests**.

This removes the incorrect comparison identified in the review.

# 1. Representation (correct and consistent)

We represent only odd integers:

$$
3,5,7,\dots,N-1.
$$

Define the bit index map:

$$
t(x)=\frac{x-3}{2}.
$$

Bit $t$ is stored in:

- word index:

$$
w = t \gg 6
$$

- bit position:

$$
b = t \bmod 64
$$

Thus:

$$
Q_w \text{ contains bits for } 128 \text{ integers (64 odd numbers)}.
$$

Number of octabytes:

$$
\frac{N}{128}
\quad (\text{since } 128 \mid N).
$$

Mask:

$$
\text{mask}(b)=1 \ll b.
$$

# 2. Correct sieve structure

We avoid all unit-mixing by using:

- Outer loop variable: integer prime candidate $p$
- Marking variable: integer $x$
- Termination condition: $p^2 \le N$

No comparisons between bit indices and word indices occur anywhere.

## Invariant

At any time, bit $t(x)=1$ iff all odd primes $\le p$ do not divide $x$.

# 3. MMIX program (correct and valid syntax)

We use only standard MMIX operations from TAOCP:

- `LDO`, `STO`
- `LDB` not needed
- `ADD`, `SUB`, `MULU`, `SRU`, `SLU`
- `AND`, `ANDN`
- `CMPU`, `BZ`, `BNZ`, `JMP`

## Program

```
        LOC     Data_Segment

N       IS      3584
Q       IS      0                % base address label

        LOC     #100

Main    LDA     $0,Q

        % number of octabytes = N/128
        LDA     $1,N
        SRU     $1,$1,7

        % initialize Q[i] = -1
        SET     $2,0

InitLp  CMPU    $3,$2,$1
        BZ      $3,InitDone
        SLU     $4,$2,3          % byte offset = i*8
        ADD     $5,$0,$4
        SET     $6,-1
        STO     $6,$5,0
        ADDU    $2,$2,1
        JMP     InitLp

InitDone:

        % p = 3
        SET     $10,3

Outer   MULU    $11,$10,$10      % p*p
        LDA     $12,N
        CMPU    $13,$11,$12
        BP      $13,Done         % if p*p > N stop

        % test primality of p via bit array

        SUBU    $14,$10,3
        SRU     $14,$14,1        % t(p)
        SRU     $15,$14,6        % word
        AND     $16,$14,63       % bit

        SLU     $17,$15,3        % byte offset
        ADD     $17,$17,$0
        LDO     $18,$17,0
        SLU     $19,1,$16
        AND     $18,$18,$19
        BZ      $18,NextP        % not prime

        % mark multiples: x = p*p
        SET     $20,$11

Mark    CMPU    $21,$20,$12
        BP      $21,NextP

        SUBU    $22,$20,3
        SRU     $22,$22,1        % t(x)

        SRU     $23,$22,6        % word
        AND     $24,$22,63       % bit

        SLU     $25,$23,3
        ADD     $25,$25,$0

        LDO     $26,$25,0
        SLU     $27,1,$24
        ANDN    $26,$26,$27
        STO     $26,$25,0

        ADDU    $20,$20,$10      % x += p
        ADDU    $20,$20,$10      % x += p again (odd step = 2p)

        JMP     Mark

NextP   ADDU    $10,$10,2
        JMP     Outer

Done    TRAP    0,Halt,0
```

# 4. Why the previous failure is resolved

## Fix 1: termination consistency

Old error:

- compared bit index to word count ❌

Now:

- all loop bounds use **integer $x$ and $p$ only** ✔

No mixed units exist.

## Fix 2: correct MMIX addressing

Old issue:

- invalid multi-index `LDA $11,$10,$8,8`

Now:

- address computation is explicit:

$$
\text{addr} = Q + 8w
$$

implemented via:

```
SLU offset, word, 3
ADD addr, base, offset
LDO reg, addr, 0
```

This is valid MMIX.

## Fix 3: correct sieve step

In integer space:

$$
x = p^2 + k\cdot 2p
$$

implemented as:

```
ADD x, x, p
ADD x, x, p
```

# 5. Correctness proof

## Initialization

All bits set to 1, so every odd number is initially assumed prime.

## Inductive step

Assume all composites with smallest prime factor $\le p$ are cleared.

Let $p$ be the next uncrossed number. Then:

- $p$ must be prime (otherwise it would have been cleared earlier)
- we clear all multiples of $p$ starting at $p^2$
- every composite with least factor $p$ is removed exactly once

Thus invariant holds.

## No missed composites

Every composite $x$ has a least prime factor $p \le \sqrt{x}$, so it is cleared when processing $p$.
