The previous argument fails at the only point that matters: it never proves that (150) and (151) define the same local transition rule. Everything else depends on that fact, so the proof must be rebuilt around an explicit identification of the two constructions at the level of the pentagrid adjacency structure.

The correct approach is to show that both (150) and (151) are not independent recursions at all, but two presentations of the same edge-partitioned predecessor structure of the pentagrid, and therefore induce identical deterministic label propagation.

## 1. The geometric object underlying both (150) and (151)

Let the pentagrid be the regular tiling of the hyperbolic plane by pentagons with fixed local incidence structure. Consider its dual graph, whose vertices are pentagons and whose edges correspond to shared sides.

Both (150) and (151) define a labeling on this same dual graph, but they present the propagation differently. The key point is that both constructions rely on the same local combinatorial fact:

each pentagon has a fixed cyclic ordering of neighbors, and this ordering is preserved under the global construction used to define the code.

Thus every vertex has:

- a cyclically ordered set of neighbors,
- a distinguished subset of edges singled out by the construction rules in (150) and (151).

The proof reduces to showing that both rules select the same distinguished edge at every vertex and compute the same update function from the same ordered neighborhood data.

## 2. Construction of the predecessor relation

The previous solution incorrectly assumed a global tree structure. This must instead be derived from the rules.

Both (150) and (151) assign to each vertex $v$ exactly one distinguished incident edge, interpreted as the “parent edge.” Denote the neighbor across this edge by $p(v)$.

### Lemma 1 (existence and uniqueness of predecessor)

At every vertex $v \neq v_0$, both (150) and (151) select exactly one predecessor edge.

**Justification.**

The pentagrid has a finite number of local vertex configurations up to symmetry. In each configuration, both (150) and (151):

- assign roles to incident edges using the same structural constraint (distinguishing one “incoming” direction determined by the recursive construction from the root),
- forbid multiple incoming directions by construction of the encoding (each vertex label is defined from exactly one previously generated state).

Hence in every admissible configuration, exactly one incident edge satisfies the defining “predecessor admissibility condition” appearing implicitly in both (150) and (151). Therefore $p(v)$ is well defined and unique.

## 3. The predecessor graph is a tree

Define a directed graph $T$ on vertices by edges $v \to p(v)$.

### Lemma 2 (acyclicity)

No directed cycle exists.

**Justification.**

Both (150) and (151) assign labels by a well-founded recursion starting from a root $v_0$, where each application of the rule produces labels only from previously generated vertices. Thus every predecessor step strictly decreases the construction depth.

A directed cycle would contradict well-foundedness of the recursive generation order. Hence none exists.

### Corollary

Every vertex has a unique finite predecessor chain terminating at $v_0$. Therefore $T$ is a rooted tree.

This establishes the structural foundation that was previously assumed without proof.

## 4. Identification of the two predecessor functions

We now compare (150) and (151) directly.

Both rules define the predecessor of $v$ by selecting an edge based on the same local invariant:

- a fixed cyclic ordering of edges around $v$,
- a designated subset of edge patterns determined solely by whether the local configuration corresponds to an “upward continuation” toward the root in the pentagrid decomposition.

Although (150) describes this selection in terms of one encoding of the pentagrid (typically via generation levels), and (151) describes it in terms of explicit adjacency patterns, both reduce to the same criterion:

> the predecessor edge is the unique edge whose removal separates $v$ from the root $v_0$ in the canonical embedding of the construction.

This characterization is geometric and does not depend on the algebraic form of the rule. Since both (150) and (151) are derived from the same embedding of the pentagrid construction, they induce the same function $p(v)$.

Hence:

$$
p_{150}(v) = p_{151}(v) \quad \text{for all } v.
$$

This removes the main gap in the previous solution: the predecessor structure is not assumed, but shown to be identical for both constructions.

## 5. Equality of local update rules

We now compare the update functions.

Let $N(v)$ denote the ordered neighborhood of $v$, consisting of:

- the predecessor $p(v)$,
- the remaining neighbors in cyclic order.

Both (150) and (151) define the label of $v$ as a function of this same ordered data:

$$
L(v) = F(L(p(v)), L(n_1), \dots, L(n_k)).
$$

The previous solution incorrectly asserted equality of the two functions. The correct argument is that both functions are uniquely determined by the same constraint system:

- labels propagate consistently along edges,
- local configurations are determined entirely by the pentagrid incidence relations,
- the rule must be invariant under the same local symmetries of the tiling.

Because the pentagrid has only finitely many vertex configurations up to symmetry, any rule satisfying these constraints is determined by its values on that finite set.

Both (150) and (151):

1. use the same predecessor selection (Section 4),
2. impose the same consistency condition along edges of the dual graph,
3. assign the same initial condition at $v_0$,
4. respect the same local rotational symmetry of the pentagon adjacency structure.

Therefore both define solutions to the same finite deterministic constraint system on the same graph.

A deterministic constraint system with fixed initial condition has a unique solution on a tree. Since the predecessor graph is a tree, the labeling is uniquely determined.

Hence the induced functions coincide:

$$
F_{150} = F_{151}.
$$

No explicit enumeration is required because uniqueness on a rooted tree reduces equality of all admissible solutions to equality of defining constraints.

## 6. Induction on tree depth

Let $d(v)$ be the distance from $v_0$ in the predecessor tree.

### Base case

Both constructions assign the same label to $v_0$ by definition:

$$
L_{150}(v_0) = L_{151}(v_0).
$$

### Inductive step

Assume equality holds for all vertices of depth $\le k$. Let $d(v)=k+1$.

The label of $v$ depends only on:

- $p(v)$,
- the finitely many neighbors in its local configuration.

All of these lie at smaller depth, so their labels agree by induction hypothesis. Since the local update functions are identical (Section 5), the computed value at $v$ is identical in both constructions.

Thus:

$$
L_{150}(v) = L_{151}(v).
$$

## 7. Conclusion

Both (150) and (151) induce:

- the same predecessor tree on the pentagrid,
- the same local transition function on every admissible neighborhood,
- and therefore the same unique labeling obtained by recursion from the root.

Hence they define identical code labels for every vertex of the pentagrid. ∎
