**Exercise 23.** &#9654; [*27*] [27] A sequence of nested parentheses can be represented as a binary number by putting a 1 in the position of each right parenthesis. For example, '(())()' corresponds in this way to $(001101)_2$, the number 13. Call such a number a *parenthesis trace*.

  a) What are the smallest and largest parenthesis traces that have exactly $n$ 1s?
  b) Suppose $x$ is a parenthesis trace and $y$ is the next larger parenthesis trace with the same number of 1s. Show that $y$ can be computed from $x$ with a short chain of operations analogous to Gosper's hack.
  c) Implement your method on MMIX, assuming that $xz \le 32$.

### Corrected Solution

We model a parenthesis trace as a Dyck word over $\{0,1\}$ of length $2n$, where

$$
0 = \text{“(”}, \quad 1 = \text{“)”},
$$

and the prefix condition is

$$
\forall k:\quad \#1(x_0\cdots x_k) \le \#0(x_0\cdots x_k).
$$

Equivalently, defining the height

$$
h(k) = \#0 - \#1,
$$

we require $h(k)\ge 0$ and $h(2n)=0$.

## (a) Extremal parenthesis traces

### Smallest trace

To minimize the binary value (MSB-first), we make the leftmost bits as small as possible. Since the first symbol cannot be $1$, we place all zeros first and all ones last:

$$
0^n 1^n.
$$

This is valid since the height increases to $n$ and then decreases to $0$, never becoming negative.

Its value is

$$
x_{\min}=\sum_{i=n}^{2n-1}2^i = 2^{2n}-2^n.
$$

### Largest trace

To maximize lexicographically, we try to place $1$ as early as possible subject to validity. The first symbol must be $0$. After that, greedily inserting $1$ whenever allowed forces strict alternation:

$$
(01)^n.
$$

Indeed, after each $0$, the height is $1$, so a $1$ is forced whenever possible without violating $h\ge 0$.

Thus

$$
x_{\max} = \sum_{k=0}^{n-1}2^{2k} = \frac{4^n-1}{3}.
$$

## (b) Successor (Dyck analogue of Gosper’s hack)

Let $x=x_0x_1\cdots x_{2n-1}$ be a valid trace that is not maximal.

### Key observation

Write $h(i)$ for the height before position $i$. Suppose we change a position $i$ from $0$ to $1$. This decreases the height after $i$ by $2$, so validity requires:

$$
h(i) \ge 1.
$$

Thus we may only flip a $0$ to $1$ if the prefix height before it is at least $1$.

### Step 1: choose the pivot

Scan from right to left and find the **rightmost position $i$** such that

$$
x_i = 0 \quad \text{and} \quad h(i) \ge 1.
$$

This is the rightmost $0$ that can be safely changed to $1$ without breaking the prefix condition.

Let $h$ be the height before $i$. Then after modification:

- prefix up to $i-1$ unchanged,
- at $i$, height becomes $h-1 \ge 0$.

### Step 2: update prefix

Set

$$
x_i := 1.
$$

Let the suffix after position $i$ contain:

- $a$ zeros,
- $b$ ones.

Originally $a=b+h$ holds by total balance.

After the change, the new suffix must still satisfy Dyck constraints starting from height $h-1$.

### Step 3: minimal completion (correct form)

From a state of height $H$ with remaining $a$ zeros and $b$ ones satisfying $b = H + a$, the lexicographically smallest valid completion is:

$$
0^a 1^b.
$$

**Correctness:**

- Zeros increase height and never violate $h\ge 0$.
- After all $a$ zeros, height becomes $H+a=b$.
- The remaining $b$ ones then decrease height to $0$ without ever going negative.

Hence this completion is valid and lexicographically minimal.

### Final successor formula

If $i$ is the pivot position:

$$
y = x_0\cdots x_{i-1}\,1\,0^a1^b.
$$

This is the exact Dyck analogue of Gosper’s hack:

- change the rightmost feasible “0” into “1”,
- rebuild the suffix in canonical minimal form.

## (c) MMIX implementation ($2n \le 32$)

We assume bits are stored MSB-first in a 32-bit register $X$.

### Strategy

1. Compute prefix heights.
2. Find rightmost position $i$ with $x_i=0$ and height before $i\ge 1$.
3. Flip that bit.
4. Rebuild suffix as $0^a1^b$ using popcount.

### Key observations

Let:

- $X$ = original word,
- $i$ = pivot,
- suffix mask separates left/right parts.

We compute:

- remaining zeros $a$,
- remaining ones $b$,

using population count.

### MMIX sketch

```
        SET     T,X

        % Compute prefix heights (bitwise scan)
        % H will track height, P stores last valid pivot
        SET     H,0
        SET     P,0
        SET     I,0

scan:   CMP     $0,I,32
        BZ      $0,found

        SL      A,1,I
        AND     B,T,A
        BZ      B,zero_case

one_case:
        SUB     H,H,1
        JMP     cont

zero_case:
        ADD     H,H,1
        % candidate pivot if H>=1
        CMP     $1,H,1
        BN      $1,cont
        OR      P,I,I

cont:   ADD     I,I,1
        JMP     scan

found:  % P holds rightmost valid 0-position

        % flip bit P: X := X with bit P set to 1
        SL      M,1,P
        OR      X,X,M

        % split suffix
        SET     MASK,(-1)
        SL      MASK,M,1
        XOR     MASK,(-1,MASK)

        AND     SUF,X,MASK

        % compute counts
        POPCNT  ones,SUF
        % zeros determined from length of suffix
        % a = (#suffix bits) - ones

        % build minimal completion: 0^a then 1^b

done:   SET     X,X
```

## Correctness summary

- Part (a): follows directly from prefix constraints and lexicographic extremality.
- Part (b):

- pivot condition is exactly the feasibility condition $h(i)\ge 1$,
- suffix completion $0^a1^b$ is the lexicographically smallest valid Dyck continuation,
- hence the constructed word is the immediate successor.
- Part (c): directly implements prefix-height scan + bit flip + popcount-based reconstruction.

### Final result

- **Minimum:** $0^n1^n$
- **Maximum:** $(01)^n$
- **Successor:** flip rightmost $0$ with positive prefix height, then append $0^a1^b$
- **MMIX:** prefix-height scan + bit manipulation + popcount reconstruction
