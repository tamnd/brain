**Exercise 56.** [*24*] Suggest a way to transpose a $7 \times 9$ bit matrix in a 64-bit register.

The permutation model is correct: we view the $7\times 9$ matrix as a permutation of 63 occupied bit positions in a 64-bit register, with one spare cell.

We index bits $0,\dots,63$ and store the matrix in positions

$$
k = 9i + j,\quad 0 \le i \le 6,\; 0 \le j \le 8.
$$

The transpose is the permutation

$$
p(9i+j) = 7j + i,
$$

which maps the 63 occupied positions bijectively onto themselves, since $0 \le 7j+i \le 62$.

Thus the problem reduces to implementing an arbitrary permutation on 63 bits using a single auxiliary bit position, namely position $63$, which starts and ends at $0$.

## Correct cycle implementation

Let

$$
k_0 \mapsto k_1 \mapsto \cdots \mapsto k_{t-1} \mapsto k_0
$$

be a cycle of the permutation $p$. Write $x[k]$ for the bit stored at position $k$.

We implement the cycle using only swaps of bit positions, with position $63$ as temporary storage.

### Step 1: extract the first value

Swap $k_0$ and $63$. After this operation,

$$
63 = x[k_0], \quad k_0 = 0.
$$

We now use position $63$ as a holding cell for the value that must eventually go into $k_{t-1}$.

### Step 2: propagate values along the cycle

For $r = 1,2,\dots,t-1$, perform

$$
\text{swap}(k_r, k_{r-1}).
$$

We now prove the effect of this sequence.

We maintain the invariant after completing step $r \ge 1$:

- For all $0 \le s \le r-1$, we have

$$
k_s = x[k_{s+1}],
$$
- For all $s \ge r$, positions $k_s$ are unchanged from their initial values except possibly $k_0$, which has already been overwritten,
- The value originally in $k_0$ remains stored in position $63$ throughout.

#### Why the invariant holds

Before step $r$, assume that for all $s \le r-2$, we already have $k_s = x[k_{s+1}]$.

At step $r$, we swap $k_r$ and $k_{r-1}$. At that moment:

- $k_{r-1}$ contains $x[k_r]$ (from the previous stage of propagation, or from initialization when $r=1$),
- $k_r$ still contains its original value unless it has already been overwritten in a previous step, in which case it is consistent with the shifting pattern.

After swapping, we obtain:

$$
k_{r-1} \leftarrow x[k_r], \quad k_r \leftarrow \text{old } k_{r-1}.
$$

This has the net effect of shifting the value from $k_r$ backward into $k_{r-1}$, extending the reversed propagation of the cycle values one step further.

Concretely, after finishing all steps $r=1,\dots,t-1$, we obtain:

$$
k_0 = x[k_1],\quad k_1 = x[k_2],\quad \dots,\quad k_{t-2} = x[k_{t-1}],
$$

while $k_{t-1}$ has been overwritten to $0$ during the propagation.

### Step 3: restore the missing value

Finally swap $k_{t-1}$ and $63$. Since at this moment $k_{t-1}=0$ and $63=x[k_0]$, we obtain:

$$
k_{t-1} = x[k_0], \quad 63 = 0.
$$

## Result of the cycle

After the full sequence:

- $k_{t-1}$ receives the original value of $k_0$,
- each $k_{r-1}$ receives the original value of $k_r$ for $1 \le r \le t-1$,

so the cycle is correctly rotated:

$$
k_r \leftarrow x[k_{r+1}] \quad (\text{indices mod } t).
$$

Thus the cycle is implemented exactly.

## Completion of the permutation

Every permutation decomposes into disjoint cycles, and each cycle can be executed independently using the above procedure. Since all operations are swaps of bit positions within a single 64-bit register, and only position $63$ is used as temporary storage, the method remains within the model constraints.

Applying this to all cycles of $p$ realizes the full transpose permutation. The bit at position $9i+j$ is moved to position $7j+i$, completing the $7\times 9$ matrix transpose in a single 64-bit register.
