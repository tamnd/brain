**Exercise 264.** [*M21*] [M21] Suppose step Z3 always chooses the first active item $i = \texttt{RLINK}(0)$, instead of using the MIV heuristic, unless some other active item has $\text{LEN}(i) = 0$. Prove that Algorithm Z will then output an *ordered* ZDD.

**Verified:** yes  
**Solve time:** 1m14s  

---

## Solution

Let the active items at any stage of Algorithm Z be the nodes in the horizontal list rooted at $0$, linked by $\mathrm{RLINK}$. Write $i_0 = \mathrm{RLINK}(0)$ for the first active item. The modified step Z3 selects $i_0$ unless there exists an active item $j$ with $\mathrm{LEN}(j)=0$, in which case it selects such a $j$.

An ordered ZDD requires that along every root-to-terminal path the items are considered in a fixed global order, namely the order induced by repeated application of $\mathrm{RLINK}(0)$ on the initial item list, and that no item is revisited after the algorithm has moved past it in this order.

The horizontal list structure guarantees that at every moment the active items form a cyclic list in which $\mathrm{RLINK}(0)$ is the least element under the fixed pointer order. Deletions performed by cover operations preserve the relative order of all remaining items, since they only splice out nodes via assignments of the form $\mathrm{RLINK}(\mathrm{LLINK}(i)) \leftarrow \mathrm{RLINK}(i)$ and $\mathrm{LLINK}(\mathrm{RLINK}(i)) \leftarrow \mathrm{LLINK}(i)$, which do not reorder surviving nodes. Therefore, whenever no item with $\mathrm{LEN}(j)=0$ is present, the algorithm chooses $i_0$, the smallest active item in the fixed list order.

Consider a computation path that leads to a solution. Along such a path, every chosen option is valid, so no active item can have $\mathrm{LEN}(j)=0$ at the moment of selection. Indeed, if some active item $j$ had $\mathrm{LEN}(j)=0$, then by definition no active option contains $j$, and the exact cover condition could not be completed; Algorithm Z would terminate unsuccessfully at that node, so such states do not lie on any successful branch.

Hence along every successful path the exception clause is never triggered, and every recursive call of step Z3 selects exactly $i_0 = \mathrm{RLINK}(0)$ in the current state. After covering $i_0$, that item is removed from the active list, and the next call again selects the new $\mathrm{RLINK}(0)$ of the reduced list. Since removal preserves the relative order of all remaining items, the sequence of selected items is exactly the initial list order with elements deleted as they are processed.

It follows that on every root-to-leaf path that contributes to a solution, items are selected in strictly increasing order according to the initial $\mathrm{RLINK}$ ordering, and no later item is ever selected before an earlier remaining one. This is precisely the definition of an ordered ZDD, since each decision node corresponds to the first unprocessed item in the fixed global ordering and all branching respects that order.

Therefore Algorithm Z, under the modified rule for step Z3, outputs an ordered ZDD. ∎
