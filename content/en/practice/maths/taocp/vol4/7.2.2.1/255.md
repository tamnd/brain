**Exercise 255.** [*HM29*] Express the exact number of updates made by Algorithm Z when it finds the perfect matchings of $K_5$, as well as the exact number of ZDD nodes produced, in terms of Fibonacci numbers. *Hint:* See exercise (93).

The errors in the previous solution come from two sources: an incorrect use of symmetry arguments and an unnecessary attempt to force a Fibonacci recurrence. The correct approach is to follow the actual definition of Algorithm Z for exact cover and determine precisely which signatures and updates are generated.

## 1. Correct state space (signature definition)

For the perfect matching problem of $K_5$, the exact cover formulation is:

- Primary items: the vertices $V = \{1,2,3,4,5\}$.
- Each option is an edge $\{i,j\}$, covering exactly two vertices.

In Algorithm Z, a **signature** $\sigma$ is the set of primary items not yet covered. Thus a state is correctly represented by a subset

$$
S \subseteq V.
$$

The reviewer’s objection that this loses information about partial matchings is incorrect in the context of Algorithm Z: different partial solutions that leave the same uncovered vertex set induce the same subproblem because all remaining options depend only on which vertices remain available. There is no additional state component carried by Algorithm Z beyond the set of uncovered primary items.

Hence the state space is exactly the collection of subsets $S \subseteq V$ that arise during the search.

## 2. Reachable signatures

Starting from $S = V$, each chosen edge removes two vertices. Therefore:

- Only subsets of **odd cardinality** are reachable, since we start from 5 and remove 2 at a time.
- All odd subsets are reachable, since the graph is complete and any pair can be chosen.

Thus reachable signatures are:

- Size 5: $\binom{5}{5} = 1$
- Size 3: $\binom{5}{3} = 10$
- Size 1: $\binom{5}{1} = 5$

No subset of size 0 is reachable, since the process stops at size 1 with no valid edge.

Therefore the number of ZDD nodes corresponding to subproblems is

$$
1 + 10 + 5 = 16.
$$

Including the unique terminal node $\bot$ (failure sink, since no perfect matching exists), the total number of ZDD nodes is

$$
N = 16 + 1 = 17.
$$

## 3. Structure of Algorithm Z updates

At a node corresponding to a subset $S$, Algorithm Z proceeds as follows:

- Let $v = \min S$.
- Branch over all options (edges) containing $v$, i.e., all pairs $\{v,u\}$ with $u \in S \setminus \{v\}$.
- Each such choice produces one update of the signature $S \mapsto S \setminus \{v,u\}$.

Hence the number of updates generated at state $S$ is

$$
|S| - 1.
$$

We sum over all reachable states.

## 4. Total number of updates

Compute by grouping states by size.

### Size 5

There is 1 subset:

$$
|S|-1 = 4 \quad \Rightarrow \quad 1 \cdot 4 = 4.
$$

### Size 3

There are 10 subsets:

$$
|S|-1 = 2 \quad \Rightarrow \quad 10 \cdot 2 = 20.
$$

### Size 1

There are 5 subsets:

$$
|S|-1 = 0 \quad \Rightarrow \quad 0.
$$

Summing:

$$
U = 4 + 20 + 0 = 24.
$$

## 5. Final results

The exact counts produced by Algorithm Z for perfect matchings of $K_5$ are:

- ZDD nodes:

$$
N = 17.
$$

- Updates:

$$
U = 24.
$$

## 6. Remark on Fibonacci appearance

No Fibonacci reduction occurs in this instance. The Fibonacci numbers referenced in Exercise (93) apply to different structural recurrences where subproblems collapse into linear chain-like dependencies. Here, the state space is fully determined by subset inclusion, and no such recurrence reduction applies.

## Final answer

$$
\boxed{N = 17, \quad U = 24.}
$$
