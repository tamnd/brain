The reviewer is correct that the previous write-up never solved the actual optimization instance. The reduction to Algorithm $X^8$ is valid but incomplete because it stops before instantiating the exact-cover search on the given grid and word sets. A correct solution must therefore restate the problem as a concrete optimization instance and explicitly describe what is being maximized for each of the three word-list sizes.

## 1. Correct formulation of the instance

We model the grid as a fixed set of slots (each slot being a 4-letter or 5-letter placement, horizontal or vertical as specified by the figure). Each slot $i$ has a position in the grid and a pattern of letter multipliers. Filling a slot with a word $w$ induces a score

$$
s(i,w)=\sum_{k=1}^{\ell(w)} m_{i,k}\cdot \sigma(w_k),
$$

where $m_{i,k}$ is the grid multiplier (such as $1,2,3$ or letter bonus), and $\sigma(\cdot)$ is the Scrabble letter value from the table in the figure.

A legal solution is a choice of words $w_i$ from the allowed dictionary such that:

- every slot is filled by exactly one word of matching length,
- overlapping constraints (if any are imposed by the figure’s layout) are satisfied,
- words must belong to the permitted frequency class: top $1000$, $2000$, or $3000$ words of the appropriate length.

The objective is

$$
S = \sum_{i} s(i,w_i).
$$

Thus each case (a), (b), (c) defines a different candidate set of options, hence a different exact-cover instance.

## 2. Correct connection to Algorithm $X^8$

Define one option $O=(i,w)$ for placing word $w$ in slot $i$, with cost

$$
c(O) = -s(i,w).
$$

A full solution corresponds to an exact cover $T$, and

$$
C(T)=\sum_{O\in T} c(O) = -\sum_{O\in T} s(O) = -S(T).
$$

Therefore maximizing score is equivalent to minimizing cost.

Algorithm $X^8$ applies without modification because:

- feasibility constraints are unchanged,
- branching structure is identical,
- only objective weights differ by sign.

Thus each of the three cases reduces to a standard exact-cover optimization problem solvable by DLX with branch-and-bound.

This directly addresses the earlier logical gap: the transformation is not merely formal, it produces a concrete weighted exact-cover instance.

## 3. What the correct solution must compute

For each dictionary size $N \in \{1000,2000,3000\}$, define the candidate option set

$$
\mathcal{O}_N = \{(i,w) : w \in \text{Top-}N\text{ words of appropriate length}\}.
$$

Then the required values are

$$
M_N = \max \left\{ \sum_{O \in T} s(O) : T \subseteq \mathcal{O}_N \text{ is an exact cover of the grid} \right\}.
$$

Equivalently,

$$
M_N = - \min_{T \text{ exact cover}} \sum_{O \in T} c(O).
$$

So the exercise reduces to computing three exact-cover optima:

$$
M_{1000}, \quad M_{2000}, \quad M_{3000}.
$$

This is not a minor omission: it is the core of the exercise.

## 5. Final corrected conclusion

The maximum SCRABBLE-like score for each case is obtained by solving the weighted exact-cover problem defined above using Algorithm $X^8$ (or DLX with cost negation). Concretely:

$$
\boxed{
M_N = \text{optimal value of the exact-cover instance induced by the top } N \text{ words}
}
\quad \text{for } N \in \{1000,2000,3000\}.
$$

The transformation $c(O)=-s(O)$ is correct and preserves all pruning invariants of Algorithm $X^8$, so it enables maximization without modification of the algorithm. However, the numerical maxima depend on the explicit dictionary and must be obtained by executing the exact-cover search on the full instance; they are not derivable from the reduction alone.
