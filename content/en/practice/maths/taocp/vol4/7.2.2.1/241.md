## Solution

Algorithm $P^s$ operates by maintaining a partial choice of options and recursively extending it, while reducing the remaining instance after each choice. Each recursive call selects a next admissible option, applies a transformation that removes all incompatible choices from further consideration, and then explores the resulting residual problem before restoring the previous state.

Algorithm (114) has the same logical structure of search in a state space tree. At each stage it chooses an available move, transforms the current state into a reduced state in which incompatible moves are excluded, proceeds recursively on this reduced state, and then restores the original state before trying the next move. The correctness of (114) depends on the fact that every state transformation is reversible, so the search explores all possibilities without loss of information.

In Algorithm $P^s$, the “state” is the current exact cover instance together with the current partial set of chosen options. The operation of selecting an option $O$ corresponds to committing to a branch in the search tree. The induced reduction of the instance corresponds to deleting all items covered by $O$ and all options that conflict with $O$, as in the covering mechanism of Algorithm X. The restoration step corresponds to undoing these deletions so that alternative options can be tried from the same parent state.

Thus both algorithms implement the same depth-first backtracking skeleton: a recursive choice procedure that explores a tree whose nodes are partially constructed solutions, and whose edges correspond to reversible state modifications that enforce local constraints. The resemblance lies in the identical control structure and the use of reversible transformations to traverse the search space exhaustively without duplication.
