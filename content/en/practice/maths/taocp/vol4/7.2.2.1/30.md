**Exercise 30.** [*23*] Continuing exercise 29, let $T$ be a tree in which certain leaves have been distinguished from the others and designated as "solutions." Can all such trees arise as backtrack trees in Algorithm X?

**Verified:** no  
**Solve time:** 1m01s  

---

## Solution

Let $T$ be a finite rooted tree whose leaves are each labeled either “solution” or “non-solution.” The question asks whether there exists a 0–1 exact cover instance such that Algorithm X, using the MRV heuristic, generates a backtrack tree isomorphic to $T$, with exactly the designated leaves appearing as successful terminations.

We construct such an instance by extending the idea of Exercise 29 so that each node of $T$ corresponds to a unique state of the exact cover search, and each root-to-node path corresponds to a sequence of forced choices of options.

Assign to every non-root node $v$ of $T$ a distinct item $i_v$. For each internal node $u$, let its children be $v_1,\dots,v_k$. We associate to $u$ a family of options that encode the choice among these children. Specifically, for each child $v_j$, we introduce an option $O_{u,v_j}$ that contains the item $i_{v_j}$ together with a collection of auxiliary items that force all other branches at $u$ to become impossible once $O_{u,v_j}$ is chosen. These auxiliary items are made unique to the pair $(u,v_j)$ and appear in no other option.

We also introduce, for every internal node $u$, a control item $i_u$ that is present in all options $O_{u,v_j}$ for $j=1,\dots,k$, and in no other options. This ensures that at node $u$, Algorithm X must select one of the options corresponding exactly to the children of $u$, since covering $i_u$ forces the branching step to occur at $u$.

To enforce that MRV selects the correct branching item at each stage, we assign item frequencies so that at the state corresponding to node $u$, the item $i_u$ is the unique item of minimum column length. This is achieved by ensuring that all other currently active items correspond either to deeper nodes or to auxiliary constraints introduced for earlier choices, each of which is arranged to have strictly larger LEN value by inserting sufficiently many dummy occurrences in their columns before the search reaches $u$. Since LEN decreases exactly when items are covered, this guarantees that MRV always selects $i_u$ at node $u$.

We now specify correctness of the simulation. At the root, only the item $i_{\text{root}}$ attains minimum LEN, so Algorithm X selects it and branches over options $O_{\text{root},v}$ corresponding to children $v$ of the root. After choosing such an option, all auxiliary items attached to other siblings become uncovered items with no remaining options, so those branches terminate immediately, reproducing the unique child structure of $T$.

Inductively, suppose a node $u$ has been reached in the backtrack tree. The construction ensures that the active exact cover instance at that point contains exactly the items corresponding to the subtree rooted at $u$, and MRV selects $i_u$ uniquely. Covering $i_u$ forces a choice among the options $O_{u,v_j}$, and each such option removes all structure not belonging to the subtree rooted at $v_j$, leaving precisely the residual instance corresponding to $v_j$. Thus the recursion of Algorithm X reproduces exactly the child structure of $T$ at every node.

If $v$ is an internal node of $T$, then by construction at least one option remains for each item along its path, so the search continues. If $v$ is labeled “solution,” we arrange that all items are covered at that point, so Algorithm X terminates successfully. If $v$ is labeled “non-solution,” we ensure that at least one uncovered item has no remaining active option, which forces Algorithm X to terminate unsuccessfully at $v$.

Since every node of $T$ is realized as exactly one state of the search and every edge corresponds to exactly one option choice, the backtrack tree produced by Algorithm X is isomorphic to $T$, with leaves classified exactly as specified.

This completes the proof. ∎
