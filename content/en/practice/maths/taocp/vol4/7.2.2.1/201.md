**Exercise 201.** &#9654; [*M30*] Consider the bipartite matching problem that has $3n$ options, '$X_j Y_k$' for $1 \le j, k \le n$ and $(j - k) \bmod n \in \{0, 1, n - 1\}$. (Assume that $n \ge 3$.)

a) What "natural, intuitively obvious" problem is equivalent to this one?

b) How many solutions does this problem have?

c) How many updates does Algorithm X make when finding all solutions, if the items are ordered $X_1, Y_1, \ldots, X_n, Y_n$, and if exercise 9 is used in step X3?

### (a) Natural equivalent problem

Each option is an edge $X_jY_k$ with

$$
k \equiv j,\; j+1,\; j-1 \pmod n.
$$

Thus we are choosing a bijection $p:\{1,\dots,n\}\to\{1,\dots,n\}$ such that

$$
p(j)-j \in \{-1,0,1\}\pmod n.
$$

Equivalently, consider the cycle graph $C_n$ on vertices $1,\dots,n$ with edges $j\sim j\pm 1$. We are selecting a permutation $p$ such that every vertex is matched either to itself or to an adjacent vertex, with every vertex used exactly once as image and preimage.

So the natural equivalent problem is:

> Count permutations of an $n$-cycle whose graph lies in the graph consisting of loops and the cycle edges, i.e. every vertex is mapped either to itself or to a neighbor, forming a bijection.

Equivalently:

> Count decompositions of $C_n$ into directed cycles using edges $j\to j$, $j\to j+1$, $j\to j-1$.

### (b) Number of solutions

We classify all valid permutations.

#### Step 1: structure of allowed components

A vertex $j$ can be used in exactly one of three ways:

- fixed point: $j\to j$,
- 2-cycle with neighbor: $j\leftrightarrow j+1$ (or $j-1$),
- part of a directed cycle using only $+1$ steps,
- part of a directed cycle using only $-1$ steps.

If a single vertex uses $j\to j+1$, then consistency of bijectivity forces a chain:

$$
j \to j+1 \to j+2 \to \cdots
$$

which can only close on the finite cycle if every vertex uses $+1$. Hence this produces exactly one permutation: the rotation $j\mapsto j+1$. Similarly for $j\mapsto j-1$.

Thus there are exactly:

- 2 global cyclic permutations (all $+1$, all $-1$),
- all remaining solutions use only fixed points and disjoint adjacent transpositions.

#### Step 2: matchings part

If no global shift is used, the permutation is a disjoint union of:

- fixed points $j\mapsto j$,
- 2-cycles $(j\, j+1)$, where edges are disjoint on the cycle $C_n$.

Thus we choose a matching on the cycle graph $C_n$.

The number of matchings in $C_n$ is the classical value:

$$
M_n = F_{n-1} + F_{n+1},
$$

where $F_n$ is the Fibonacci sequence with $F_0=0, F_1=1$.

This includes the empty matching (identity permutation).

#### Step 3: add the two cyclic shifts

These are not included in matchings (they are single $n$-cycles), so we add 2.

#### Final formula

$$
\boxed{
a_n = F_{n-1} + F_{n+1} + 2
}
$$

Check $n=3$:

$$
F_2 + F_4 + 2 = 1 + 3 + 2 = 6,
$$

correct.

### (c) Number of Algorithm X updates

Algorithm X performs one update per node in the search tree up to constant overhead (Knuth’s implementation with Exercise 9 ensures bounded branching with exact-cover propagation on this banded cyclic matrix).

The search structure mirrors exactly the combinatorics in (b):

- all matchings on $C_n$, counted by $M_n = F_{n-1}+F_{n+1}$,
- plus two additional linear chains corresponding to the two full cyclic shifts.

Thus the total number of search nodes, and hence total updates, is

$$
U(n) = \Theta(F_{n-1}+F_{n+1}).
$$

Since $F_n = \Theta(\varphi^n)$ where $\varphi = \frac{1+\sqrt5}{2}$,

$$
\boxed{
U(n) = \Theta(\varphi^n)
}
$$

### Final answers

**(a)** Permutations of an $n$-cycle where each element maps to itself or an adjacent element; equivalently matchings on $C_n$ plus two global shifts.

**(b)**

$$
\boxed{a_n = F_{n-1} + F_{n+1} + 2.}
$$

**(c)**

$$
\boxed{U(n) = \Theta(\varphi^n).}
$$
