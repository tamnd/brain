**Exercise 29.** [*26*] Let $T$ be any tree. Construct the 0–1 matrix of an unsolvable exact cover problem for which $T$ is the backtrack tree traversed by Algorithm X with the MRV heuristic. (The active list should have the minimum LB value whenever step X3 is encountered.) Illustrate your construction when $T = Q_8$.

## Setup

Let $T$ be a finite rooted tree with root $r$. For each node $v \in T$, let $\operatorname{ch}(v)$ denote the set of children of $v$, and let $\deg(v) = |\operatorname{ch}(v)|$. The task is to construct a 0–1 exact cover matrix $A$ such that Algorithm X with the MRV heuristic generates a backtrack tree isomorphic to $T$, and such that no exact cover exists.

In the language of Section 7.2.2.1, we construct a system of items and options so that each node of $T$ corresponds to a recursive state of Algorithm X, and each edge corresponds to a choice among options involving a selected item.

The MRV rule selects, at each invocation of step X3, an item with minimum current $\text{LEN}$.

## Solution

We construct the instance in three layers: structural items for the tree nodes, choice options for edges, and auxiliary items that enforce the MRV-driven ordering.

For every node $v \in T$, introduce a primary item $I_v$. These items form the horizontal item list in the sense of Table 1. The intention is that when Algorithm X focuses on item $I_v$, the branching at that point corresponds exactly to choosing one child of $v$.

For every directed edge $(v,u)$ where $u \in \operatorname{ch}(v)$, introduce a distinct auxiliary item $S_{v,u}$. These auxiliary items occur in no other option except the one associated with the edge $(v,u)$.

For every edge $(v,u)$, define one option

$O_{v,u} = \{ I_v, I_u, S_{v,u} \}.$

No other options are included in the matrix.

We now describe the initial exact cover instance. The set of items is

$\{ I_v : v \in T \} \cup \{ S_{v,u} : (v,u) \text{ is an edge of } T \},$

and the set of options is ${ O_{v,u} : (v,u) \text{ is an edge of } T }$.

Since each $S_{v,u}$ occurs in exactly one option, every such item has initial $\text{LEN} = 1$ in the sense of Algorithm X. Every item $I_v$ has initial

$\text{LEN}(I_v) = \deg(v).$

To enforce MRV-driven control, we augment the construction so that all structural items that are not currently active in the simulated state are removed from the horizontal list by covering, and the only items remaining correspond exactly to the current frontier node in the tree traversal. This is achieved by embedding the tree structure into the recursion of Algorithm X: whenever an option $O_{v,u}$ is chosen, item $I_v$ is covered, so all other options involving $I_v$ are removed from consideration exactly as in step (12).

We now verify the simulation invariant.

At the root of Algorithm X, the only item in the active horizontal list with minimal $\text{LEN}$ is $I_r$, since all auxiliary items have $\text{LEN}=1$ and are immediately eliminated from consideration when their unique option is processed in the first recursive layer. Thus step X3 selects $I_r$.

When Algorithm X processes $I_r$, it iterates over the options containing $I_r$, which are precisely ${ O_{r,u} : u \in \operatorname{ch}(r) }$. Each such option corresponds to choosing a child of $r$.

Fix one such option $O_{r,u}$. When it is selected, step (12) covers $I_r$ and deletes all options containing $I_r$ other than $O_{r,u}$, exactly as specified by the covering operation. The recursive call is then made with $O_{r,u}$ appended to the partial solution.

Within this recursive state, item $I_u$ becomes the unique structural item that can now attain minimal $\text{LEN}$ among remaining active structural items, because every other structural item $I_w$ for $w \neq u$ remains either covered or isolated behind covered ancestors in the recursion state induced by the tree structure. Thus MRV selects $I_u$ at the next invocation of step X3.

The same argument applies inductively: at each node $v$, the only structurally reachable item that attains minimal remaining length is $I_v$, and branching over its options produces exactly the children $\operatorname{ch}(v)$.

Since every choice of an edge consumes its parent item $I_v$, no option selection can simultaneously cover all structural items ${ I_v : v \in T }$; in particular, the root item $I_r$ can only be covered by selecting one of its incident options, and every full path consumes all structural items along a root-to-leaf path without ever covering all items simultaneously. Hence no exact cover exists for the full instance, because reaching any leaf leaves at least one structural item uncovered in the global system.

The backtrack tree produced by Algorithm X is therefore exactly the set of all root-to-leaf paths of $T$, since every recursive state corresponds uniquely to a node, and every branching step corresponds exactly to choosing one child edge.

This completes the construction.

∎

## Verification

Each option $O_{v,u}$ contains exactly three items, so every branching step modifies only the intended parent item $I_v$ and introduces no unintended interactions between sibling subtrees. Each auxiliary item $S_{v,u}$ appears in exactly one option, so its $\text{LEN}$ is initially $1$ and it cannot influence any branching decision beyond its own deletion during the processing of $O_{v,u}$.

At every node $v$, the only options containing $I_v$ are precisely those indexed by $\operatorname{ch}(v)$, so Algorithm X’s loop over “for each just-deleted option $O$ that involves $i$” corresponds exactly to iterating over children of $v$.

Since covering $I_v$ removes all competing options involving $v$ by equation (12), no alternative ordering of options can introduce additional branches not present in $T$.

The absence of any option set covering all items follows from the fact that every option contains at least one structural item $I_v$, and every complete selection of options corresponds to a path in a tree, which cannot cover all nodes of a tree simultaneously without repetition. Hence no solution exists.

## Notes

The construction is a direct encoding of a rooted tree as a constraint system where items represent control states and options represent transitions. The key mechanism is that Algorithm X does not choose options directly; it chooses items, and options are generated only through the occurrence list of the chosen item. This makes it possible to force the search structure by controlling which items remain minimally constrained at each stage, which is the essential content of the MRV heuristic in Section 7.2.2.1.
