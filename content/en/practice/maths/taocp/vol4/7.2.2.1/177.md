**Exercise 177.** [*M21*] [M21] Algorithm 7.2.1.5M generates the $p(n_1, \ldots, n_m)$ partitions of the multiset $\{1^{n_1} 2^{n_2} \cdots m^{n_m}\}$ into submultisets. Consider the special cases where $n_1 = \cdots = n_k = 1$ and $n_{k+1} = \cdots = n_m = 2$ and $s + l = m$.

a) Generate those partitions with Algorithm M, using the previous two exercises.
b) Also generate the $q(n_1, \ldots, n_m)$ multipartitions into *distinct* submultisets.

Let

$$
n_1=\cdots=n_s=1,\qquad n_{s+1}=\cdots=n_{s+l}=2,\qquad m=s+l,
$$

and let the underlying multiset be

$$
\{1,\ldots,s,(s+1)^2,\ldots,(s+l)^2\}.
$$

Introduce the ordinary set of occurrences

$$
U=\{1,\ldots,s\}\ \cup\ \{(s+1)_1,(s+1)_2,\ldots,(s+l)_1,(s+l)_2\},
$$

so $|U|=s+2l$.

A partition of the multiset into submultisets is equivalent to a partition of $U$ into blocks, with the understanding that each block $B\subseteq U$ is interpreted as the submultiset obtained by collapsing $(s+t)_1,(s+t)_2\mapsto s+t$.

Thus the problem is an MCC instance whose primary items are the elements of $U$.

## (a) Generation via Algorithm M (Exercises 175–176)

### MCC formulation (explicit)

We use the formulation of Algorithm 7.2.1.5M where:

- **Items (columns):** the elements of $U$.
- **Options (rows):** all admissible blocks $B\subseteq U$.

Since each element of $U$ occurs once, an admissible block is simply any subset of $U$. The multiplicity restriction is already enforced by splitting doubled elements into two distinct items.

Hence the MCC matrix $A=(a_{Bx})$ is defined by

$$
a_{Bx}=
\begin{cases}
1 & x\in B,\\
0 & x\notin B,
\end{cases}
\qquad (B\subseteq U).
$$

A solution to the MCC instance is a selection of rows whose supports form a partition of $U$, i.e., an exact cover.

### Connection with Exercises 175–176

Exercises 175–176 establish that:

1. Partitions of a set correspond bijectively to exact covers of the incidence matrix of subsets of $U$.
2. Algorithm M generates all solutions of the MCC by recursively choosing, for each uncovered item $x$, a block containing $x$, and then removing all conflicting options.

In this special case, the recursion specializes to:

- Pick the least uncovered element $x\in U$.
- For each subset $B\subseteq U$ with $x\in B$, recursively attempt to complete a partition after selecting $B$.
- Remove all items of $B$ and all rows intersecting $B$.

This is exactly the partition-generation procedure of Algorithm 7.2.1.5M.

### Structural simplification for this exercise

Because every element of $U$ is distinct, this MCC instance is precisely the classical set-partition problem on a set of size $s+2l$. Hence Algorithm M generates all partitions of $U$.

Finally, each partition of $U$ maps to a unique partition of the original multiset by collapsing pairs $(s+t)_1,(s+t)_2$ into a single element $s+t$. This mapping is a bijection between:

- partitions of $U$, and
- partitions of $\{1^{n_1}\cdots m^{n_m}\}$.

Therefore Algorithm M generates exactly all required multiset partitions.

## (b) Multipartitions into distinct submultisets

A multipartition differs from an ordinary partition in that we now consider the family of blocks as a multiset of submultisets, and we require:

> no two blocks are identical as multisets over $\{1,\ldots,m\}$.

In the representation via $U$, different blocks may project to the same collapsed multiset. This is the key issue absent in part (a).

### Type structure of blocks

For each block $B\subseteq U$, define its **type**

$$
\tau(B)=\big(\#(1\in B),\ldots,\#(s\in B),\#(s+1\in B),\ldots,\#(s+l\in B)\big),
$$

where for $t>s$,

$$
\#(t\in B)\in\{0,1,2\}
$$

counts how many of $(t)_1,(t)_2$ lie in $B$.

Two blocks are identical submultisets iff they have the same type vector $\tau(B)$.

### Augmented MCC instance

We refine the MCC formulation by introducing **secondary (type) constraints** as in the framework underlying Exercises 175–176:

- Primary items: elements of $U$ (as in part (a)).
- Secondary items: all feasible block types $\tau$.

Each option $B\subseteq U$ is augmented with the secondary item corresponding to $\tau(B)$.

We require:

- each primary item is covered exactly once (partition condition),
- each secondary item is used **at most once** (distinct-block condition).

This is implemented in Algorithm M by treating secondary columns as “at-most-once” constraints.

### Correctness argument

- Primary constraints ensure the selected blocks form a partition of $U$, hence a partition of the multiset.
- Secondary constraints ensure that no two selected blocks have the same type $\tau(B)$, hence no two blocks represent the same submultiset of $\{1,\ldots,m\}$.
- Conversely, any multipartition into distinct submultisets yields a unique selection of options satisfying both constraints.

Thus there is a bijection between:

$$
\text{multipartitions into distinct submultisets}
\quad\leftrightarrow\quad
\text{solutions of the augmented MCC instance}.
$$

Algorithm M (with secondary constraints as in the exact-cover framework of Exercises 175–176) therefore generates exactly the required objects.

## Conclusion

- (a) reduces to exact covers of the incidence system of the set $U$, hence Algorithm M generates all partitions of the multiset via the bijection $U \leftrightarrow \{1^{n_1}\cdots m^{n_m}\}$.
- (b) requires augmentation with secondary “type” constraints so that each submultiset pattern can appear at most once, yielding exactly multipartitions into distinct submultisets.
