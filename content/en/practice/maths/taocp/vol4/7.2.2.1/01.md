[Section 7.2.2.1: Dancing Links](../)

**Exercise 1.** &#9654; [*M25*] A doubly linked list of $n$ elements, with a list head at 0, begins with $\text{LLINK}(k) = k - 1$ and $\text{RLINK}(k - 1) = k$ for $1 \le k \le n$; furthermore $\text{LLINK}(0) = n$ and $\text{RLINK}(n) = 0$, as in (3). But after we use operation (1) to delete elements $a_1, a_2, \ldots, a_n$, where $a_1 a_2 \ldots a_n$ is a permutation of $\{1, 2, \ldots, n\}$, the list will be empty and the links will be entangled as in (4).

a) Show that the final settings of LLINK and RLINK can be described in terms of the binary search tree that is obtained when the keys $a_n, \ldots, a_2, a_1$ (in reverse order) are inserted by Algorithm 6.2.2T into an initially empty tree.

b) Say that permutations $a_1 a_2 \ldots a_n$ and $b_1 b_2 \ldots b_n$ are equivalent if they both yield the same LLINK and RLINK values after deletion. How many distinct equivalence classes arise, for a given value of $n$?

c) How many of those equivalence classes contain just one permutation?

## Solution

### (a)

Let $a_1,a_2,\ldots,a_n$ be a permutation of ${1,2,\ldots,n}$ and apply operation (1) successively to delete $a_1,a_2,\ldots,a_n$. Let $L_k$ and $R_k$ denote the values of LLINK and RLINK after deleting $a_1,\ldots,a_k$.

Construct a binary search tree $T_k$ by inserting the keys $a_n,a_{n-1},\ldots,a_{k+1}$ into an initially empty binary search tree using Algorithm 6.2.2T, and write $T=T_0$.

We prove by induction on $k$ that the set of nodes not yet deleted, together with the restriction of LLINK and RLINK, determines a binary search tree isomorphic to $T_k$ when interpreted in inorder order.

For $k=n$, no insertions are performed, and all nodes have been deleted. Both structures are empty.

Assume the correspondence holds for $k-1$. At step $k$, node $a_k$ is removed using (1), so its predecessor $p$ and successor $s$ in the current circular list are joined by

$\text{RLINK}(p)\leftarrow s,\qquad \text{LLINK}(s)\leftarrow p.$

In the BST constructed from $a_n,\ldots,a_{k+1}$, inserting $a_{k}$ as the next reverse-insertion key places it into the unique position determined by comparisons with previously inserted keys. This insertion partitions the inorder sequence exactly at $a_k$, producing the same left and right substructures as the update above produces by splicing out $a_k$ from the circular inorder list.

Thus the effect of deleting $a_k$ corresponds exactly to inserting $a_k$ into the BST built from later elements in reverse order, preserving inorder adjacency relations. The correspondence is maintained at each step.

After all deletions, the final LLINK and RLINK structure encodes the binary search tree obtained by inserting $a_n,\ldots,a_1$.

This completes the proof. ∎

### (b)

Two permutations are equivalent exactly when they produce the same final LLINK and RLINK values, hence the same structure described in part (a). Therefore equivalence classes correspond to binary search tree shapes on the key set ${1,2,\ldots,n}$ with fixed inorder labeling.

Every binary search tree on $n$ distinct ordered keys corresponds uniquely to a full binary tree shape with inorder labeling $1,2,\ldots,n$, and every such shape arises from some permutation. Hence the number of equivalence classes equals the number of binary search tree shapes on $n$ nodes.

Let $C_n$ denote this number. Standard decomposition at the root $r$ yields a left subtree on $k$ nodes and a right subtree on $n-1-k$ nodes, giving the recurrence

$C_n=\sum_{k=0}^{n-1} C_k C_{n-1-k},\qquad C_0=1.$

Thus $C_n$ is the $n$th Catalan number,

$C_n=\frac{1}{n+1}\binom{2n}{n}.$

Hence the number of equivalence classes is $\frac{1}{n+1}\binom{2n}{n}$.

### (c)

A class contains exactly one permutation precisely when the BST shape admits only one insertion order consistent with producing that shape under reverse insertion.

For a fixed BST shape, the number of permutations producing it equals the number of ways to interleave the insertion sequences of its left and right subtrees at every node while preserving internal orders. At a node with left subtree size $L$ and right subtree size $R$, this contributes a multinomial factor $\binom{L+R}{L}$ to the total count.

A class has size $1$ only when every such factor equals $1$, which requires $L=0$ or $R=0$ at every node. Thus every node has at most one child, so the tree is a single path.

There are exactly two such BST shapes on $n$ nodes: the completely left-skewed chain and the completely right-skewed chain.

Each of these shapes corresponds to exactly one permutation: increasing order and decreasing order respectively.

Hence the number of equivalence classes containing exactly one permutation is $2$.
