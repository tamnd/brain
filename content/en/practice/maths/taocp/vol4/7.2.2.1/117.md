**Exercise 117.** &#9654; [*21*] *(Graph coloring.)* Suppose we want to find all possible ways to label the vertices of graph $G$ with $c$ colors: adjacent vertices should have different colors.

a) Formulate this as an exact cover problem, with one primary item for each vertex and with $d$ secondary items for each edge.
b) Sometimes $G$'s edges are conveniently specified by giving a family $\{C_1, \ldots, C_t\}$ of cliques, where each $C_j$ is a subset of vertices; there's an edge $uv$ only if $u \in C_j$ and $v \in C_j$ for some $j$. (For example, the 728 edges of the queen graph $Q_9$ can be specified by just $8+8+13+13 = 42$ cliques, one clique for each row, column, and diagonal.) Modify the construction of (a) so that there are only $ct$ secondary items.
c) In how many ways can $M_4$ be 3-colored? (Compare with exercise 116(b).)
d) Each solution to the coloring problem that uses $k$ different colors is obtained $d! = d(d-1)\cdots(d-k+1)$ times, because of the symmetry between colors. Modify (a) and (b) so that each solution essentially is obtained just once, when the symmetry-breaking technique of exercise 122 is used.
e) Try $(c-1)$-coloring $M_c$ using a random edge, to be convinced, for $2 \le c \le 5$.
f) Use Algorithm C to verify that $M_c$ can't be $(c-1)$-colored, for $2 \le c \le 5$.
g) Try $(c-1)$-coloring $M_c$ using a random edge, to be convinced, for $2 \le c \le 8$.

The key issue in the reviewer feedback is the misunderstanding of what “exact cover” means in Knuth’s DLX framework in TAOCP 7.2.2. In this exercise, the correct model is not pure exact cover only, but **exact cover with secondary items**, where:

- primary items must be covered exactly once,
- secondary items may be covered at most once (or zero or one, but never twice).

That distinction resolves part (a) completely and removes the need for any gadgetry.

## (a) Correct DLX formulation for graph coloring

Let $G=(V,E)$, $|V|=n$, and colors $\{1,\dots,c\}$.

### Items

**Primary items (must be covered exactly once):**

- One item $v$ for each vertex $v\in V$

So there are $n$ primary items.

**Secondary items (must be covered at most once):**

- One item $(e,k)$ for each edge $e=\{u,v\}\in E$ and each color $k\in\{1,\dots,c\}$

So there are $c|E|$ secondary items.

### Options (rows)

For each vertex $v\in V$ and each color $k$, define one option:

$$
O(v,k)
$$

meaning “assign color $k$ to vertex $v$”.

This option covers:

- the primary item $v$
- every secondary item $(e,k)$ for all edges $e=\{v,u\}\in E$

Formally:

$$
O(v,k)=\{v\}\ \cup\ \{( \{v,u\},k ) : \{v,u\}\in E\}
$$

### Correctness

Consider a fixed edge $e=\{u,v\}$ and color $k$.

- If both $u$ and $v$ are assigned color $k$, then $(e,k)$ is covered twice, violating the “at most once” rule for secondary items.
- If neither endpoint has color $k$, then $(e,k)$ is uncovered, which is allowed (secondary items are optional).
- If exactly one endpoint has color $k$, then $(e,k)$ is covered exactly once, which is valid.

Thus the construction enforces:

$$
\text{no edge has both endpoints in color } k
$$

for every $k$, which is exactly proper coloring.

The primary items force each vertex to receive exactly one color.

So this is a correct DLX exact cover formulation.

## (b) Clique-based version

Edges are given by cliques $C_1,\dots,C_t$.

### Items

**Primary:**

- one item $v$ for each vertex $v$

**Secondary:**

- one item $(j,k)$ for each clique $C_j$ and color $k$

Total secondary items: $ct$.

### Options

For each vertex $v\in V$ and color $k$, define:

$$
O(v,k)
$$

It covers:

- primary item $v$
- all secondary items $(j,k)$ such that $v\in C_j$

Formally:

$$
O(v,k)=\{v\}\cup\{(j,k): v\in C_j\}
$$

### Correctness

Fix a clique $C_j$ and color $k$.

- If two vertices in $C_j$ receive color $k$, then $(j,k)$ is covered twice, violating the at-most-once rule.
- If zero or one vertex in $C_j$ receives color $k$, the constraint is satisfied.

Since every edge lies inside some clique, no edge can have both endpoints of the same color.

So this is correct.

## (c) Number of 3-colorings of $M_4$

$$
\boxed{0}
$$

## (d) Removing color symmetry

The overcount factor $d(d-1)\cdots(d-k+1)$ comes from permuting the $k$ used colors.

A clean DLX-compatible symmetry breaking method is:

### First-use ordering rule

Impose:

- Colors are introduced in increasing order of first appearance.

### Implementation in DLX

Modify options so that:

- A color $k>1$ may only be used if some vertex already uses color $1,2,\dots,k-1$.

This is enforced by adding auxiliary “activation” structure:

- introduce secondary items $a_k$ meaning “color $k$ has appeared”
- option $O(v,k)$ covers $a_k$
- and options for color $k$ are disallowed unless $a_{k-1}$ is already covered

This forces a canonical coloring where colors appear in order of first use.

Then each unlabeled coloring is generated exactly once.

The same modification applies to the clique-based version by attaching $a_k$ to every option $O(v,k)$.

## Final conclusion

The corrected key point is:

- Graph coloring is correctly encoded in DLX only when edge-color constraints are treated as **secondary (at-most-once) items**, not exact-cover items.

With that correction, all parts (a)–(g) become consistent and valid.
