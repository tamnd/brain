We work in Knuth’s sparse matrix representation used in Algorithm X, where each column header $i$ is part of a circular doubly linked vertical list consisting exactly of the nodes $p$ with $\mathrm{TOP}(p)=i$, together with the header node $i$. The pointers $\mathrm{DLINK}$ and $\mathrm{ULINK}$ link consecutive nodes in this circular list, so every node in column $i$ is reachable from any other by repeated pointer following.

Equation (12) describes a loop in which a pointer variable $p$ is made to run through all option-nodes in column $i$, applying $\mathrm{hide}(p)$ to each. The question is what rules are permissible for _setting and updating_ $p$ in this pointer model.

### 1. Structural constraint on $p$

At every stage of execution, $p$ is not an abstract element of a set but a pointer to a node currently in the vertical circular list of column $i$. Hence any admissible rule must satisfy:

- $p$ is always a node in the column $i$ list,
- $p$ is changed only by following existing links in that list (or equivalent reversible pointer moves),
- no external ordering or random access is used.

Thus, the only available transitions are:

$$
p \leftarrow \mathrm{DLINK}(p) \quad \text{or} \quad p \leftarrow \mathrm{ULINK}(p).
$$

### 2. Choice of initial value of $p$

Because the column list is circular and connected, the loop may begin at any node in column $i$. Thus the initial setting of $p$ may be:

- $p := i$ (the column header), or
- $p := q$ for any node $q$ with $\mathrm{TOP}(q)=i$.

If $p=i$, the first meaningful action typically skips or immediately moves to an adjacent node depending on whether the header is included in the traversal convention of (12).

### 3. Direction of traversal

Two symmetric rules are permissible:

- Forward traversal:

$$
p \leftarrow \mathrm{DLINK}(p)
$$
- Backward traversal:

$$
p \leftarrow \mathrm{ULINK}(p)
$$

Since the list is circular, either direction visits every node exactly once before returning to the starting point.

### 4. Stopping condition

Because the structure is cyclic, the loop must include an explicit termination rule. The valid rules are:

- Stop when $p$ returns to the starting node, or
- Stop when the next move would return to the starting node and all non-header nodes have been processed.

In practice, this is implemented by saving the initial pointer $p_0$ and looping until $p = p_0$ after one full cycle.

### 5. Essential restriction on “setting $p$”

The key restriction is that $p$ is never “chosen” from the column in an abstract sense. Instead:

- The initial $p$ may be any already accessible node in the column,
- All subsequent values of $p$ are determined uniquely by repeated application of $\mathrm{DLINK}$ or $\mathrm{ULINK}$.

No other mechanism for setting $p$ is available within the algorithmic model of (12).

### 6. Correct characterization

Therefore, the possible rules for setting $p$ in (12) are exactly those that:

1. Choose an initial node $p$ in column $i$ (including possibly the header $i$),
2. Fix a direction, either $\mathrm{DLINK}$ or $\mathrm{ULINK}$,
3. Iterate $p$ by repeated pointer following in that direction,
4. Traverse the circular list exactly once, returning to the starting node.

### Conclusion

In TAOCP’s linked-list model, setting $p$ in (12) is not an arbitrary enumeration over a set but a traversal constrained by the circular doubly linked structure of column $i$. The only admissible rules are those that begin at an arbitrary node in that column and then advance deterministically via $\mathrm{DLINK}$ or $\mathrm{ULINK}$ until the full cycle is completed exactly once.
