**Exercise 167.** [*22*] [22] Let $M_p$ be the number of options that involve the primary item $p$ in a given MCC problem, and suppose that the upper bound on $p$'s multiplicity is $\ge M_p$. Does the precise value of this upper bound affect the behavior of Algorithm M? (In other words, does $v_p = \infty$ lead to the same running time as $v_p = M_p$?)

**Verified:** yes  
**Solve time:** 1m02s  

---

## Solution

Let $M_p$ be the number of options that contain the primary item $p$, and let $v_p$ be the prescribed upper bound on the number of chosen options involving $p$ in the MCC instance processed by Algorithm M.

Every option that involves $p$ is represented exactly once in the input, so any feasible partial solution can include at most one copy of each such option. Hence the total number of distinct options involving $p$ that can appear in any solution is at most $M_p$.

Let $S$ be any partial solution generated during the execution of Algorithm M. The number of selected options in $S$ that involve $p$ equals the number of distinct options involving $p$ that have been chosen along the current search path. Since no option can be chosen more than once on a single path of the search tree, this number is bounded above by $M_p$ for every $S$.

Assume $v_p \ge M_p$. Then for every partial solution $S$, the inequality

$\#\{O \in S : p \in O\} \le M_p \le v_p$

holds, so the constraint imposed by $v_p$ is never violated. Consequently, no branch in the search tree is pruned due to the bound $v_p$.

Replacing $v_p = \infty$ by any finite value $v_p$ with $v_p \ge M_p$ does not introduce additional pruning conditions and does not eliminate any node of the search tree generated by Algorithm M. The set of admissible partial solutions, the sequence of covering and uncovering operations, and the branching structure remain unchanged.

Therefore the running time and search behavior of Algorithm M are identical when $v_p = \infty$ and when $v_p = M_p$.

This completes the proof. ∎
