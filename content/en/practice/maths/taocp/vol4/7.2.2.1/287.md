**Exercise 287.** &#9654; [*23*] [23] A princess asks you to pack an $m \times n$ box with pentominoes, rewarding you with $\$c \cdot (i + j)$ if you've covered cell $(i, j)$ with piece $c$, where $c \in \{1, 2, \ldots, 12\}$ for pieces $(O, P, \ldots, Z)$ in some reasonable packing order (perhaps closest to alphabetic order?).

Use Algorithm X$^\dagger$ to maximize your bounty when packing boxes of sizes $4 \times 15$, $5 \times 12$, $6 \times 10$, $10 \times 6$, $12 \times 5$, $15 \times 4$. Consider also the princess's circle-shaped subset of a $9 \times 9$ box, where you are to cover only the 69 cells whose distances from the center are between 1 and $\sqrt{18}$. How do the running times of Algorithm X$^\dagger$ compare to the amounts of time that Algorithm X would take to find *all* solutions?

## Solution

Each placement of a pentomino is an option in an exact cover instance in which the items are the cells of the board together with the requirement that each pentomino is used exactly once. In the weighted version of the problem, each option $O$ acquires a weight equal to the sum of its cell contributions. If a placement of piece $c$ covers cells $(i_1,j_1),\dots,(i_5,j_5)$, then the option weight is

$w(O)=\sum_{k=1}^5 c\cdot(i_k+j_k).$

A solution of the exact cover instance is a set of disjoint options whose union covers every item exactly once. Its value is the sum of the weights of its selected options. The task is to find a maximum-weight solution.

Algorithm $X^\dagger$ is obtained from Algorithm X by attaching a running score to the partial solution and maintaining an admissible upper bound during search. The underlying state of the dancing-links structure, including $\text{cover}(i)$ and $\text{uncover}(i)$, is unchanged, and only the search strategy is modified.

Let $S$ denote the total weight of the options already chosen in the current partial solution. For each item $j$, define

$M(j)=\max\{w(O): O \text{ is an active option that contains item } j\}.$

Since every feasible completion must cover each item exactly once, any completion contributes at most $M(j)$ to the total weight from the coverage of item $j$. Summing over all currently uncovered items gives an upper bound

$B=\sum_{j \text{ uncovered}} M(j).$

Every complete extension of the current partial solution has weight at most $S+B$. If $S+B$ is not strictly larger than the best solution weight already found, the search from this node cannot improve the optimum and the branch is discarded.

The recursive structure of Algorithm X is preserved. At a node where item $i$ is selected, each option $O$ containing $i$ is tried in turn. After choosing $O$, the algorithm updates $S$ by adding $w(O)$ and performs the usual sequence of $\text{cover}(j)$ operations for all $j\in O$. After recursion, $\text{uncover}(j)$ restores the state and $S$ is restored by subtracting $w(O)$. The only additional condition before descending into a branch is the test $S+B>\text{best}$.

The cases $4\times 15$, $5\times 12$, $6\times 10$, $10\times 6$, $12\times 5$, and $15\times 4$ correspond to different exact cover instances with identical option sets up to rotation or transposition of the board. The values $c(i+j)$ depend only on geometry and the fixed labeling of pieces, so each instance induces a different set of option weights, but the combinatorial structure of feasible covers is unchanged.

The circular region of the $9\times 9$ board restricted to cells at distance between $1$ and $\sqrt{18}$ produces a smaller exact cover instance in which the number of items is $69$. Each option is again a placement of a pentomino contained entirely in the region, and its weight is defined by the same formula.

The running time of Algorithm $X^\dagger$ depends on the effectiveness of the bound $B$. In the worst case, when every partial assignment remains consistent with the bound, no pruning occurs and the search explores exactly the same recursion tree as Algorithm X applied to the same instance. In that situation the running time equals the time needed by Algorithm X to enumerate all exact covers, up to the additional constant overhead of maintaining $S$ and evaluating $B$.

In typical instances arising from pentomino packing, the bound $B$ becomes strict early in the search because each item has a limited set of covering placements and their weights vary substantially with position and piece label. This causes many branches that are feasible for exact cover to be eliminated before completion. The search tree explored by Algorithm $X^\dagger$ is therefore a pruned subtree of the full backtracking tree of Algorithm X, and its size can be significantly smaller when high-weight placements concentrate early in the search.

The comparison between the two methods is therefore that Algorithm X has running time proportional to the number of partial nodes generated while enumerating all solutions, whereas Algorithm $X^\dagger$ has running time proportional to the number of partial nodes whose bound exceeds the current best solution. Since this set is instance-dependent and may range from all nodes to a small fraction of them, the improvement can vary from negligible to exponential in the size of the pruned search space. This completes the analysis. ∎
