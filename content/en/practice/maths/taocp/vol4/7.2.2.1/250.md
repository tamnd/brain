**Exercise 250.** [*21*] [21] Users of Algorithm C\* are allowed to supply hints that speed up the computation, by specifying (i) a set $Z$ of characters, such that every element of $Z$ is the first character of exactly one primary item in every option; also (ii) a number $z > 0$, meaning that every option contains exactly $z$ primary items whose names *don't* begin with a character in $Z$. (For example, $Z = \{p, r, c, b\}$ in the sudoku options (30); $z = 1$ in options (16).) In the options (16) for Langford pairs, we could change the name of each numeric item $i$ to '$i$', then let $Z = \{!\}$ and $z = 2$.) Explain how to use these hints to supply an early-cutoff test at the beginning of step C3\*, as explained in the text.

## Solution

Let the current state of Algorithm C* at the beginning of step C3* consist of a partial solution $P$ and a residual exact-cover instance in which certain primary items remain uncovered.

For each character $c \in Z$, let $S_c$ denote the set of currently active primary items whose names begin with $c$, and let $s_c = |S_c|$. Let $U$ denote the set of currently active primary items whose names do not begin with any character in $Z$, and let $u = |U|$.

The hypothesis on the hints states that every option contains exactly one primary item from each $S_c$ (for every $c \in Z$), and also contains exactly $z$ primary items from $U$.

Assume a complete solution extending $P$ exists, and let $t$ denote the number of additional options selected in such a completion. Each selected option contributes exactly one covered item to each set $S_c$, so every $S_c$ must be exhausted after exactly $t$ further choices. This yields the identity $s_c = t$ for every $c \in Z$, hence all integers $s_c$ must be equal. If there exist $c, d \in Z$ with $s_c \ne s_d$, no completion is possible.

When all $s_c$ are equal, define

$$
t = s_c \quad (c \in Z).
$$

Each of the $t$ remaining chosen options contributes exactly $z$ items from $U$, so the total number of covered elements of $U$ in any completion must equal $zt$. Since each item of $U$ must be covered exactly once, a necessary condition for feasibility is

$$
u = zt.
$$

If $u \ne zt$, no completion exists.

These two conditions together provide a test that depends only on the residual instance induced at the beginning of step C3*, without exploring any branches.

In Algorithm C*, the early cutoff is inserted immediately before enumerating candidate options for the current recursive level in step C3*. The algorithm computes the values $s_c$ for all $c \in Z$, verifies that they are all equal, sets $t$ to this common value, and then computes $u$. If either inequality $s_c \ne s_d$ for some $c, d \in Z$ holds or $u \ne zt$, the procedure terminates unsuccessfully at that node of the search tree without executing the remainder of step C3*.

This completes the construction of the early cutoff test. ∎
