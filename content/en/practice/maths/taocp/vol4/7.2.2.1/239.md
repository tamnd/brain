**Exercise 239.** &#9654; [*M27*] [M27] Given a family $\{S_1, \ldots, S_m\}$ of subsets of $\{1, \ldots, n\}$, together with positive weights $(w_1, \ldots, w_m)$, the *optimum set cover* problem asks for a minimum-weight way to cover $\{1, \ldots, n\}$ with a union of $S_j$'s. Formulate this problem as an optimum *exact cover* problem, suitable for solution by Algorithm X$^c$. *Hint:* Maximize the weight of all sets that do not participate in the cover.

The reviewer correctly identifies that the previous construction fails because it uses the same set-column $j$ inside multiple incidence rows, violating the exact cover rule that each column may be covered exactly once. The key point is that this is not an artifact of presentation but a structural impossibility in that encoding.

A correct solution must therefore avoid introducing any column whose multiplicity is meant to track “how many elements use a set.” Exact cover cannot represent that kind of shared bookkeeping. The right approach is to eliminate sharing entirely and push all coupling into element-duplication.

## Correct formulation (restart)

Let

$$
U = \{1,\dots,n\}, \quad S_1,\dots,S_m \subseteq U, \quad w_1,\dots,w_m > 0.
$$

We reduce weighted set cover to an instance of **optimum exact cover** for Algorithm $X^c$.

The core idea is:

> Each element must choose exactly one set that covers it, and each chosen set is charged once via row weight.
> 
> No column is ever shared between different elements and different sets in a way that creates conflicts.

## Columns

We introduce only **element-choice columns**:

For each element $i \in U$ and each set $j$ with $i \in S_j$, create a column

$$
(i,j).
$$

Interpretation: “element $i$ is assigned to set $j$.”

Additionally, we introduce **element constraints**:

For each $i \in U$, we require that exactly one column among

$$
(i,j) \quad \text{for all } j \text{ with } i \in S_j
$$

is covered.

Thus each element is assigned to exactly one covering set.

There are no set-columns at all.

## Rows

There are two types of rows.

### 1. Assignment rows

For each pair $(i,j)$ with $i \in S_j$, create a row

$$
r_{i,j} = \{(i,j)\}.
$$

This row means: “assign element $i$ to set $j$.”

Each such row has weight $0$.

### 2. Set-selection rows

For each set $S_j$, create a row

$$
a_j = \{(i,j) : i \in S_j\}.
$$

This row represents selecting set $j$ as an active covering set.

Assign weight:

$$
w(a_j) = w_j.
$$

## Exact cover interpretation

We now interpret Algorithm $X^c$ in the standard way:

- Each column $(i,j)$ must be covered **exactly once**.
- Selected rows form a partition of all chosen assignment columns.

### Feasible structure of solutions

A solution consists of:

- choosing, for each element $i$, exactly one assignment row $r_{i,j}$,
- selecting exactly those set rows $a_j$ consistent with chosen assignments.

But here is the key consistency mechanism:

- If any assignment $r_{i,j}$ is chosen, then row $a_j$ must also be included to cover all columns $(i,j)$ used by that set.
- If no assignment uses set $j$, then $a_j$ is not forced.

This enforces:

> A set is selected if and only if it is used by at least one element.

## Why this avoids the reviewer’s error

The previous construction failed because it reused a single column $j$ across multiple incidence rows.

Here:

- Every column is of the form $(i,j)$, uniquely tied to a single element.
- No column is shared between different elements.
- Therefore multiple elements using the same set do not conflict.

This restores the fundamental requirement of exact cover: column uniqueness is never violated.

## Correctness

### (1) Set cover → exact cover

Given a set cover $C$:

- For each element $i$, choose one $j \in C$ with $i \in S_j$, and select row $r_{i,j}$.
- For each $j \in C$, select row $a_j$.

Then every column $(i,j)$ corresponding to the chosen assignment is covered exactly once.

Thus we obtain a valid exact cover.

### (2) Exact cover → set cover

Given a solution:

- For each element $i$, exactly one column $(i,j)$ is covered, so assign $i$ to that set $j$.
- Let $C$ be the set of all such $j$.

Then every element is covered, so $C$ is a set cover.

## Objective preservation

The total weight is

$$
\sum_{j \in C} w_j,
$$

because:

- assignment rows have weight $0$,
- each used set $j$ forces inclusion of exactly one weighted row $a_j$.

Thus minimizing exact cover weight is equivalent to minimizing set cover weight.

Equivalently, by Knuth’s hint, this is the same as maximizing the weight of unused sets:

$$
\sum_{j=1}^m w_j - \sum_{j \in C} w_j.
$$

## Conclusion

The corrected construction avoids shared set-columns entirely by:

- duplicating element constraints into independent columns $(i,j)$,
- representing set usage through weighted rows rather than shared columns.

This restores the exact cover condition and yields a valid reduction suitable for Algorithm $X^c$.
