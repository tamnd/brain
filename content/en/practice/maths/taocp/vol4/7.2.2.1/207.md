**Exercise 207.** [*35*] [35] The principal SAT solver that we shall discuss later, Algorithm 7.2.2.2C, maintains focus by computing "activity scores," which measure recent changes to the data structures. A similar idea can be applied to Algorithm X, by computing the score

$$\alpha_i = \rho^1 + \rho^2 + \cdots, \quad \text{for each item } i,$$

where $\rho$ (typically 0.9) is a user-specified damping factor, and where $i$'s list of active options was modified at times $t - t_1$, $t - t_2$, $\ldots$; here $t$ denotes the current "time," as measured by some convenient clock. When step X3 chooses an item for branching, the MRV heuristic of exercise 0 rates $i$ by its degree $\lambda_i = \text{LEN}(i)$; the new heuristic replaces this by

$$X_i = \begin{cases} \lambda_i, & \text{if } \lambda_i \le 1; \\ 1 + \lambda_i/(1 + \mu \alpha_i), & \text{if } \lambda_i \ge 2. \end{cases}$$

Here $\mu$ is another user-specified parameter. If $\mu = 0$, decisions are made as before; but larger and larger values of $\mu$ cause greater and greater attention to be given to the recently active items, even if they have a somewhat large degree of branching.

a) For example, suppose $n = 10$, $\alpha_i = 1/2$, and $\mu = 1$. Which item will be preferable, $i$ or $j$, if $\text{LEN}(i) = \text{LEN}(j) + 1$ and $0 \le \text{LEN}(j) \le 4$?

b) What modifications to Algorithm X will implement this scheme?

c) What values of $\rho$ and $\mu$ will avoid exponential growth, when applied to $n$ independent copies of the toy problems (90) and (92)?

d) Does this method save time in the Y pentomino problem of Fig. 73?

## Corrected solution

### (a)

Let $\lambda_j = k$ and $\lambda_i = k+1$, where $0 \le k \le 4$, and assume $\alpha_i = \alpha_j = \tfrac12$, $\mu = 1$.

If $k \le 1$, then $X_j = k$. For $i$, we treat cases:

- If $k=0$, then $\lambda_i = 1$, so $X_i = 1$.
- If $k=1$, then $\lambda_i = 2$, so

$$
X_i = 1 + \frac{2}{1+\frac12} = \frac73.
$$

If $k \ge 2$, both use the nonlinear form:

$$
X_j = 1 + \frac{k}{1+\frac12} = 1 + \frac{2k}{3}, \quad
X_i = 1 + \frac{k+1}{1+\frac12} = 1 + \frac{2(k+1)}{3}.
$$

Hence $X_i - X_j = \tfrac23 > 0$.

In all cases $X_j < X_i$, so item $j$ is always preferred.

### (b)

To implement the scheme in Algorithm X, each item $i$ maintains a scalar activity value $\alpha_i$ and a last-update timestamp $t_i$.

Whenever an operation (cover or uncover) causes item $i$ to become active in a way that counts as an “event,” the global time $t$ is incremented and

$$
\alpha_i \leftarrow \rho^{\,t - t_i}\alpha_i + 1, \qquad t_i \leftarrow t.
$$

At decision step X3, instead of choosing an item minimizing $\lambda_i = \mathrm{LEN}(i)$, the algorithm selects an item minimizing

$$
X_i =
\begin{cases}
\lambda_i, & \lambda_i \le 1,\\[4pt]
1 + \dfrac{\lambda_i}{1 + \mu \alpha_i}, & \lambda_i \ge 2.
\end{cases}
$$

No other structural changes to Algorithm X are required.

### (c)

The toy problems (90) and (92) are independent small exact-cover instances where Algorithm X succeeds in avoiding exponential blow-up only if the branching heuristic preserves the essential behavior of the minimum-remaining-values rule, namely that items with smaller $\lambda_i$ are always selected before those with larger $\lambda_i$. Exponential growth arises when this ordering is systematically perturbed so that higher-branching items are chosen too early, increasing the effective branching factor away from the optimal MRV-guided search.

We first bound the activity term. Since each update contributes a geometrically decayed unit increment,

$$
\alpha_i \le \sum_{k \ge 0} \rho^k = \frac{1}{1-\rho}, \quad \text{for } 0<\rho<1.
$$

Thus the denominator in the modified score satisfies

$$
1 \le 1 + \mu \alpha_i \le 1 + \frac{\mu}{1-\rho}.
$$

Now consider when the modified heuristic could change the MRV ordering. The dangerous case is when two items differ by one in degree:

$$
\lambda_i = \lambda_j + 1.
$$

In the nonlinear regime,

$$
X_i - X_j
= \frac{\lambda_i}{1+\mu \alpha_i} - \frac{\lambda_j}{1+\mu \alpha_j}.
$$

A sufficient condition for preserving MRV ordering in all cases is that the perturbation introduced by the activity term cannot outweigh a unit difference in $\lambda$. Using the worst case $\alpha_i, \alpha_j \le \frac{1}{1-\rho}$, we obtain

$$
\left|\frac{\lambda}{1+\mu \alpha} - \lambda\right|
\le \lambda \cdot \frac{\mu \alpha}{1+\mu \alpha}
\le \lambda \cdot \mu \alpha
\le \frac{\mu \lambda}{1-\rho}.
$$

For items relevant in MRV branching, $\lambda = O(1)$ (in particular, bounded by a constant in the toy problems). Hence the perturbation in $X_i$ is $O\!\left(\frac{\mu}{1-\rho}\right)$.

To prevent any inversion of ordering induced by this perturbation, it is sufficient that this quantity remains strictly smaller than the unit gap separating successive MRV ranks. Thus a sufficient stability condition is

$$
\frac{\mu}{1-\rho} \ll 1,
\quad \text{equivalently} \quad
\mu = o(1-\rho).
$$

Under this regime, $1+\mu \alpha_i = 1 + o(1)$, so

$$
X_i = \lambda_i \bigl(1 + o(1)\bigr),
$$

and the branching order coincides with that of the original MRV heuristic throughout the search trees of (90) and (92).

Since these toy problems are exactly the cases where MRV prevents exponential blow-up, preserving the MRV ordering implies that the modified algorithm also avoids exponential growth. If $\mu$ is not asymptotically smaller than $1-\rho$, the activity term can accumulate enough influence to reorder items of different $\lambda$, which reintroduces large branching factors and leads to exponential search growth.

Hence a sufficient choice is

$$
0 < \rho < 1, \qquad \mu = o(1-\rho).
$$

### (d)

In the Y pentomino problem, Algorithm X undergoes long sequences of cover and uncover operations that affect many items in a highly correlated way. As a result, the activity scores $\alpha_i$ for all frequently accessed items converge toward similar magnitudes, since each receives comparable numbers of exponentially decayed updates.

In this regime,

$$
\alpha_i \approx \alpha_j \approx \frac{1}{1-\rho},
$$

so the denominator $1+\mu \alpha_i$ becomes approximately constant across all active items. The modified score therefore reduces to

$$
X_i \approx \frac{\lambda_i}{\text{constant}} + 1,
$$

which preserves the same ordering as $\lambda_i$.

Thus the branching decisions are asymptotically identical to those produced by the original MRV heuristic. The only difference is the overhead of maintaining and updating $\alpha_i$ at each structural change.

Since Algorithm X on the pentomino instance spends essentially all its time in cover/uncover operations rather than in heuristic evaluation, this additional arithmetic does not produce a compensating reduction in the number of search nodes. Therefore the total running time is not improved in any meaningful way.

Hence the method does not yield a significant speedup for the Y pentomino problem.
