Let Algorithm C denote a DLX-style depth-first exact-cover search over a sparse incidence matrix. Its execution forms a search tree whose nodes are partial choices of rows, with explicit ordering of decisions. Space usage is $O(d + m)$, where $d$ is recursion depth and $m$ is the sparse matrix plus bookkeeping. Time is proportional to the number of visited search nodes.

Let Algorithm Z denote construction of a ZDD representing the family of partial or complete solutions under a fixed variable ordering. A ZDD node does not represent a canonical CSP state in general. It represents a subfunction of the remaining decisions induced by the ordering and reduction rules. This distinction is important: ZDD sharing is strong when many search paths induce identical subfunctions, but it is not a perfect quotient by “residual CSP isomorphism.”

The key comparison is therefore:

- Algorithm C: enumerates ordered sequences of choices, so identical partial configurations can be revisited many times in different orders.
- Algorithm Z: merges nodes that induce identical remaining decision functions under the chosen ordering, eliminating much ordering redundancy and some structural redundancy.

## (a) 16 queens

The exact cover formulation uses columns for rows, columns, and the two diagonal families.

A partial placement is determined by which rows, columns, and diagonals are already occupied. The main redundancy in Algorithm C comes from permutations of placing the same set of queens in different orders.

### Algorithm C

C explores all orderings of placing $k$ queens in distinct rows. The same partial set of $k$ placements can appear in many different sequences, producing substantial combinatorial repetition in the search tree.

The redundancy is exponential in $k$ and includes large multiplicative factors from ordering, but it should not be described as purely factorial in a strict sense because pruning and column constraints remove many permutations early.

Time: exponential in $n$, with significant ordering-induced duplication.

Space: $O(n)$ recursion depth.

### Algorithm Z

A ZDD node corresponds to the remaining exact-cover choices consistent with already fixed occupied columns and diagonals, under the variable ordering. Many different placement orders that lead to the same occupied constraint set collapse into a single subfunction.

However, ZDD nodes are not purely indexed by the set of occupied constraints; they are indexed by induced residual decision structure. Still, for queens, this structure depends mostly on the occupied columns and diagonals, so strong sharing occurs.

Time: exponential, but reduced relative to C due to elimination of ordering redundancy.

Space: exponential in distinct constraint-induced subfunctions, typically far smaller than the search tree.

Conclusion: strong compression from symmetry in placement order.

## (b) pentominoes

Columns represent grid cells and pieces; rows represent legal placements.

### Algorithm C

C suffers from repeated exploration of equivalent partial tilings differing only in placement order. However, pruning from geometric conflicts occurs early in many branches, so not all permutations survive deep exploration.

Time: exponential, dominated by search over placements, with moderate to large ordering redundancy.

Space: linear in depth.

### Algorithm Z

ZDD nodes represent subfunctions determined by remaining uncovered cells and remaining pieces, but again only up to equivalence induced by ordering and reduction. Many different placement orders do not lead to identical residual boards because geometry couples decisions strongly.

Thus, fewer merges than in highly symmetric CSPs.

Time: exponential, somewhat reduced compared to C.

Space: exponential in distinct geometric residual configurations.

Conclusion: limited but nontrivial sharing; geometry reduces ZDD compression effectiveness compared to simpler constraint systems.

## (c) MacMahon’s triangle

The problem is a layered constraint system on a triangular array with local consistency constraints between adjacent entries.

A crucial structural fact is the size of the frontier separating assigned and unassigned variables.

### Correct frontier size

At depth $k$, the boundary consists of all variables along a cut through the triangular lattice. In the natural parameterization, this boundary has width proportional to the side length of the triangle, hence

$$
\text{frontier size} = O(n),
$$

not $O(\sqrt{n})$. The earlier claim underestimated the true width of the active constraint interface.

### Algorithm C

C repeatedly reaches identical partial boundary configurations via different internal assignment orders. Since it does not memoize subproblems, it recomputes equivalent boundary states many times.

Time: exponential in $n$, with repeated exploration of identical boundary assignments.

Space: $O(n)$ recursion depth.

### Algorithm Z

ZDD nodes correspond to induced subfunctions over the remaining unassigned region given a particular boundary assignment pattern under the variable ordering.

Because the frontier has size $O(n)$, the number of possible boundary assignments is exponential in $n$. This corrects the earlier incorrect implication of subexponential state space.

Thus the correct picture is:

- ZDD does not reduce the boundary complexity below exponential in $n$,
- but it eliminates recomputation of identical boundary-induced subproblems.

Time: proportional to number of distinct boundary subfunctions, which is exponential in $n$ in the worst case but often far smaller than the full search tree.

Space: similarly exponential in the number of distinct boundary states.

Conclusion: the improvement is due to dynamic-programming-style reuse over an $O(n)$-width frontier, not a small $O(\sqrt{n})$ boundary.

## (d) generalized de Bruijn sequences

This corresponds to Eulerian cycle construction in a de Bruijn graph.

A residual state can be viewed as the current vertex together with the remaining multiset of unused edges.

### Algorithm C

C explores permutations of edge traversal orders. The same remaining edge set can be reached by many different traversal histories, producing large redundancy.

The redundancy is exponential, but it should be described as arising from permutation of traversal histories rather than a clean factorial factor.

Time: exponential in number of edges, with large ordering redundancy.

Space: $O(d)$.

### Algorithm Z

ZDD nodes correspond to subfunctions over remaining edges and current position under the variable ordering. Many traversal histories collapse to the same residual function because only the remaining availability of edges matters, not the order in which they were consumed.

However, ZDD nodes are not uniquely identified with the set of remaining edges; they represent the induced decision structure.

Time: exponential in edges, reduced relative to C by eliminating traversal-order duplication.

Space: exponential but significantly smaller than the raw search tree.

Conclusion: strong reduction from order symmetry of edge traversal.

## (e) right word stair

A layered selection problem with adjacency constraints between successive layers.

### Algorithm C

C explores many orders of selecting words that lead to identical remaining candidate sets in later layers.

Time: exponential with repeated recomputation of equivalent suffix feasibility states.

Space: $O(d)$.

### Algorithm Z

ZDD nodes represent subfunctions determined by current layer and remaining word availability profiles under ordering.

Many paths collapse when they induce identical remaining candidate sets for future layers.

Time: proportional to number of distinct availability-induced subfunctions.

Space: similarly reduced.

Conclusion: strong DP-like behavior due to repeated availability profiles.

## (f) $6 \times 6$ word search

Exact cover over word placements on a grid.

### Algorithm C

C explores different orders of placing words, often reaching identical uncovered grid configurations via different sequences.

Time: exponential with significant ordering redundancy.

Space: linear.

### Algorithm Z

ZDD nodes merge states that induce identical residual coverage constraints and remaining word choices, under the variable ordering.

Because independent placements in disjoint regions are common, substantial merging occurs, but full canonical state equivalence is not guaranteed.

Time: exponential, reduced by independence-based sharing.

Space: exponential but smaller than C in typical instances.

Conclusion: moderate to strong compression depending on spatial independence.

## (g) Kakuro

Each state consists of partial digit assignments with remaining sum constraints and domain restrictions.

### Algorithm C

C recomputes identical constraint configurations arising from different orders of filling a run.

Time: exponential with substantial recomputation of equivalent constraint states.

Space: linear depth.

### Algorithm Z

ZDD nodes correspond to subfunctions over remaining admissible digit assignments consistent with current partial sums under ordering.

Many partial assignments collapse because they induce identical remaining domain constraints, but again this is not a perfect quotient of CSP states.

Time: exponential in worst case, but reduced by constraint reuse.

Space: exponential but significantly smaller than C in structured instances.

## Overall comparison

- Algorithm C: explicit backtracking search tree over ordered decisions; exponential time; linear space.
- Algorithm Z: DAG of subfunctions induced by variable ordering; exponential worst-case time and space, but often much smaller due to elimination of ordering redundancy and reuse of identical induced subproblems.

Key correction points:

1. In MacMahon’s triangle, the frontier is $O(n)$, not $O(\sqrt{n})$, so the state space is exponential in $n$, not subexponential.
2. ZDD nodes correspond to induced residual subfunctions under an ordering, not exact canonical residual CSP states.
3. The redundancy in Algorithm C is exponential due to ordering effects, but should not be described as purely factorial in general.
