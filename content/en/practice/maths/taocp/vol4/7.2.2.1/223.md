## Solution

Step P10 prints a complete solution stored on the stack $S$ produced by Algorithm P. Each element of $S$ is an option node, and each option is represented internally as a cyclic sequence of nodes connected through the $\text{ULINK}$ and $\text{DLINK}$ fields, with spacer nodes separating consecutive options. The output phase must traverse every selected option and print the names of all items contained in it.

Let $O$ be an option node taken from $S$. The nodes belonging to $O$ form a cycle in memory in which each node $p$ satisfies $\text{TOP}(p) > 0$, and consecutive nodes are linked so that repeated application of $\text{DLINK}$ returns to the starting node. Spacer nodes are characterized by $\text{TOP}(x) \le 0$ and serve only to connect the last node of one option to the first node of the next; they do not represent items and must be ignored during printing.

The output procedure processes one option at a time. When an option node $O$ is removed from $S$, a traversal of its cyclic structure begins at $O$ itself. A pointer $p$ is initialized to $O$, and the cycle is followed using $\text{DLINK}$ until the traversal returns to $O$. For each visited node $p$, if $\text{TOP}(p) > 0$, the item name $\text{NAME}(\text{TOP}(p))$ is printed. If $\text{TOP}(p) \le 0$, the node is a spacer and is ignored, and traversal continues.

Formally, Step P10 can be written as follows. While $S \ne \Lambda$, remove its top element $O$. Set $p \leftarrow O$. Repeat: if $\text{TOP}(p) > 0$, output $\text{NAME}(\text{TOP}(p))$; set $p \leftarrow \text{DLINK}(p)$; continue until $p = O$. After completing the traversal of $O$, output a record separator indicating completion of one option, then proceed to the next element of $S$.

Correctness follows from the structure of the representation. Each solution of the exact cover problem corresponds to a set of pairwise disjoint options stored in $S$, so every item appears in exactly one option of the solution. Within each option, every node with $\text{TOP}(p) > 0$ corresponds to exactly one item in that option, and every such node is visited exactly once during the cyclic traversal because the $\text{DLINK}$ pointers define a single cycle over the option’s nodes. Spacer nodes are excluded because they satisfy $\text{TOP}(p) \le 0$, so they do not contribute spurious output. Therefore each solution is printed as the union of its option-wise item lists, with no omissions and no repetitions.

This completes the implementation of Step P10. ∎
