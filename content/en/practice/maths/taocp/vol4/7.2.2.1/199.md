**Exercise 199.** [*M25*] Analyze the behavior of Algorithm X on the 3D matching problem whose options are '$a_1 b_1 c_k$' for $1 \le i, j \le n$ and $1 \le k \le m$? $m = 1; n$).

## Solution

Let the items be $A = {a_1,\ldots,a_n}$, $B = {b_1,\ldots,b_n}$, and $C = {c_1,\ldots,c_m}$. Each option has the form $a_i b_j c_k$, so every option covers exactly one item from each of the three sets.

A solution to the exact cover problem must select a set of options in which every item in $A \cup B \cup C$ is covered exactly once. Hence the number of selected options must simultaneously equal $n$ (to cover $A$), $n$ (to cover $B$), and $m$ (to cover $C$). Therefore a necessary condition for any solution is $m = n$.

If $m \ne n$, no exact cover exists unless $n = m = 0$. In particular, for $n > 0$ and $m = 1$ with $n > 1$, no solution exists because one selected option would cover exactly one element of $C$, leaving all other $c_k$ uncovered.

Assume from this point that $m = n$. Then every solution consists of exactly $n$ options $a_i b_j c_k$, and each $a_i$, $b_j$, and $c_k$ appears exactly once. The choice of options induces a bijection between $A$, $B$, and $C$. More precisely, each solution can be described by permutations $\pi$ and $\sigma$ of ${1,\ldots,n}$ such that the selected options are

$$
a_i b_{\pi(i)} c_{\sigma(i)}, \quad 1 \le i \le n.
$$

Conversely, any pair of permutations $\pi,\sigma$ produces a valid exact cover, since each $a_i$, each $b_j$, and each $c_k$ appears exactly once. Hence the number of solutions is $(n!)^2$.

The behavior of Algorithm X is governed by the column degrees. Each item $a_i$ occurs in exactly $n^2$ options, since $j$ and $k$ are free. Each item $b_j$ also occurs in $n^2$ options. Each item $c_k$ occurs in exactly $n^2$ options as well. Therefore all item columns have equal initial length, and the choice of branching item is symmetric.

Suppose Algorithm X selects an item $a_i$ to cover. The operation $\text{cover}(a_i)$ removes all options containing $a_i$, namely all triples $a_i b_j c_k$ for $1 \le j,k \le n$. Thus the branching step iterates over $n^2$ options, each specified by a pair $(j,k)$.

Fix one such option $a_i b_j c_k$. The subsequent covering of $b_j$ and $c_k$ forces their removal from all remaining options. After these operations, no remaining option may use $a_i$, $b_j$, or $c_k$. The residual instance is again a 3D matching problem on sets

$$
A' = A \setminus \{a_i\}, \quad B' = B \setminus \{b_j\}, \quad C' = C \setminus \{c_k\},
$$

and every remaining option is exactly of the same form $a_{i'} b_{j'} c_{k'}$ with $i' \ne i$, $j' \ne j$, $k' \ne k$.

Thus each recursive level removes exactly one element from each of $A$, $B$, and $C$, and every choice corresponds to selecting a triple $(i,j,k)$ consistent with previously chosen indices. The recursion constructs a perfect matching between the three sets.

At the top level there are $n^2$ branches from fixing $a_1$ (or any chosen $a_i$), and each branch reduces the problem to size $n-1$ in all three dimensions. The same structure repeats at every level, so the search tree enumerates all ways of pairing each $a_i$ with a unique pair $(b_j,c_k)$, which is equivalent to choosing two independent permutations of ${1,\ldots,n}$. Hence Algorithm X explores a search space of size $(n!)^2$, with uniform branching symmetry at each level up to isomorphism of the remaining instance.

For $m = 1$ and $n > 1$, the item $c_1$ must be covered exactly once, so exactly one option can be chosen. Any such option covers exactly one $a_i$ and one $b_j$, leaving $n-1$ items in each of $A$ and $B$ uncovered. The residual problem has no further options involving $c_1$, so no continuation is possible. Thus Algorithm X terminates at depth $1$ with failure in every branch. The only exception is $n = 1$, where the unique option $a_1 b_1 c_1$ yields the unique solution.

This completes the analysis. ∎
