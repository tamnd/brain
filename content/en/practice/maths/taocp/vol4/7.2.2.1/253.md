**Exercise 253.** &#9654; [*21*] [21] Extend Algorithm Z so that it reports the total number of solutions.

## Solution

Algorithm Z performs a depth-first traversal of the ZDD corresponding to the exact cover search space, generating each solution by recursive extension of partial choices. To modify it so that it reports the total number of solutions, each recursive call must return an integer equal to the number of solutions in its subproblem, and the global answer must be the value returned by the initial call.

Let $Z(P)$ denote the number of solutions of a problem instance $P$. The modification preserves the structure of Algorithm Z, changing only the terminal behavior and the aggregation at each branching step.

When no item remains to be covered, the current partial solution is complete and valid. In this case there is exactly one way to extend it, namely by choosing no further options. Therefore the recursion must return $1$.

When some item $i$ remains but no active option involves $i$, no completion is possible from the current state. The recursion must return $0$.

In the general case, Algorithm Z selects an item $i$ and iterates over all active options $O$ that contain $i$. For each such option $O$, the algorithm performs the covering operations on all items in $O$, including $i$, producing a residual problem $P_O$. Each residual instance contributes independently to the total number of solutions, so the total count is the sum of the counts of all residual instances.

Formally, if $\mathcal{O}(i)$ denotes the set of active options containing item $i$, and if $P_O$ denotes the residual instance obtained after choosing option $O$, then

$$
Z(P) \;=\; \sum_{O \in \mathcal{O}(i)} Z(P_O).
$$

To implement this within the dancing-links framework, the covering and uncovering operations from (12)–(14) are used symmetrically around the recursive calls. The modification does not change the structure of $\text{cover}(i)$ or $\text{uncover}(i)$; it only changes the recursion to accumulate an integer result.

The resulting augmented Algorithm Z is therefore:

$$
\text{Z}(P) =
\begin{cases}
1, & \text{if no item remains to be covered},\\
0, & \text{if some item } i \text{ has no active option},\\
\text{otherwise:}
\end{cases}
$$

Select an item $i$ that remains to be covered and set $t \leftarrow 0$. For each active option $O$ containing $i$, perform $\text{cover}(j)$ for every item $j \in O$, then set $t \leftarrow t + \text{Z}(P_O)$, and finally perform $\text{uncover}(j)$ for every item $j \in O$ in reverse order of covering. After all such options have been processed, return $t$.

The correctness follows from the fact that each recursive call enumerates exactly the solutions consistent with a fixed choice of the option $O$ covering item $i$, and distinct options lead to disjoint subspaces of the search tree. Therefore every solution is counted exactly once in exactly one summand, and every summand counts only valid completions of its corresponding partial state. This completes the proof. ∎
