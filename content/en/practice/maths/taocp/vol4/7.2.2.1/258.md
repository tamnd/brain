**Exercise 258.** [*HM21*] How many updates does Algorithm Z perform, in that extreme problem?

**Verified:** no  
**Solve time:** 4m27s  

---

The previous solution failed because it treated Algorithm Z as if each incidence is processed a constant number of times independent of the recursive search. In DLX (Algorithm Z), every hide/uncover is driven by the search tree, so the correct approach is to count operations per search state and then sum over all states.

We restart from the actual dynamics of Algorithm Z.

## 1. Structure of the extreme instance

There are $n$ items and one option for every nonempty subset of $\{1,\dots,n\}$.

So every column $i$ contains exactly the rows (options) that include $i$, i.e. $2^{n-1}$ rows.

A row corresponding to a subset $R$ has size $|R|$, and there are $\binom{n}{|R|}$ such rows.

## 2. What Algorithm Z actually does

Algorithm Z (DLX) performs a recursive search for exact covers (set partitions in this instance).

At each recursive step:

- A row $R$ is chosen.
- Every column $i \in R$ is **covered**.
- Covering column $i$ removes:

- the column header links, and
- every row containing $i$, by unlinking those row nodes from their other columns.
- On backtracking, the same structure is restored via **uncover**, reversing all pointer changes.

Thus updates come from two sources:

1. Column header removals/restorations.
2. Unlink/relink of row nodes in other columns.

Crucially, these operations occur once per _search-tree event_, not once per static incidence.

## 3. Key structural fact: how often a column is covered

In this instance, Algorithm Z explores all set partitions of $\{1,\dots,n\}$.

Each complete solution (leaf of the search tree) is a partition, and every element $i$ is assigned exactly once per partition.

Hence:

- For a fixed item $i$, column $i$ is covered exactly once per partition.
- The number of partitions of an $n$-element set is the Bell number $B_n$.

Therefore:

$$
\text{Each column is covered } B_n \text{ times.}
$$

By symmetry, this holds for every column $i$.

## 4. Cost of one column cover

Fix a column $i$. When it is covered:

### 4.1 Header update cost

Removing the column header from the active list costs:

$$
2 \text{ pointer updates.}
$$

### 4.2 Row removal cost

Every row containing $i$ must be removed from all other columns.

For a row $R\ni i$:

- For each $j \in R \setminus \{i\}$, we unlink one node from column $j$.
- Each unlink costs $2$ pointer updates.

So cost contributed by row $R$ is:

$$
2(|R|-1).
$$

Now sum over all rows containing $i$.

For each $k$, there are $\binom{n-1}{k-1}$ rows of size $k$ containing $i$. Hence:

$$
\sum_{R\ni i} (|R|-1)
= \sum_{k=1}^n (k-1)\binom{n-1}{k-1}
= \sum_{m=0}^{n-1} m\binom{n-1}{m}.
$$

Using the standard identity:

$$
\sum_{m=0}^{n-1} m\binom{n-1}{m}
= (n-1)2^{n-2}.
$$

Thus total row-unlink updates per cover of $i$ are:

$$
2 (n-1)2^{n-2}.
$$

### 4.3 Total per column cover

$$
C_{\text{cover}}(i)
= 2 + 2 (n-1)2^{n-2}
= 2\bigl(1 + (n-1)2^{n-2}\bigr).
$$

## 5. Include uncover operations

Each cover is later reversed by an uncover, which performs the same number of pointer updates.

So per cover–uncover pair:

$$
C_{\text{pair}}(i)
= 4\bigl(1 + (n-1)2^{n-2}\bigr).
$$

## 6. Total over all columns

Each column is involved in $B_n$ cover–uncover pairs.

There are $n$ columns, so total updates:

$$
U(n)
= n \cdot B_n \cdot 4\bigl(1 + (n-1)2^{n-2}\bigr).
$$

## 7. Final result

$$
\boxed{
U(n)
= 4n B_n \bigl(1 + (n-1)2^{n-2}\bigr)
}
$$

## 8. Where the previous solution failed

- It assumed each incidence is hidden/unhidden exactly once, which ignores the search tree entirely.
- In Algorithm Z, operations are tied to **recursive column covers**, not static data structure membership.
- The correct frequency is governed by the number of partitions of the set, i.e. the Bell number $B_n$, not by incidence counts alone.

The corrected analysis replaces the incorrect “per incidence once” assumption with a “per column per partition” accounting, which matches the actual DLX execution model.
