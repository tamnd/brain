## 1. Structure of slim $n$-ominoes

A polyomino is _slim_ if it contains no $2\times 2$ sub-block of cells.

View the polyomino as an induced subgraph of the infinite grid graph $\mathbb{Z}^2$, where cells are vertices and edges connect side-adjacent cells.

A $2\times 2$ block is exactly a 4-cycle in this graph. Hence:

- If a polyomino contains a cycle, then it contains a simple cycle in the grid graph.
- Any simple cycle in the grid graph necessarily contains a $2\times 2$ cell cycle somewhere in its interior (since grid cycles decompose into unit squares).

Therefore:

**Claim.** A slim polyomino has no cycles in the grid graph, hence its adjacency graph is a tree.

So:

> Slim $n$-ominoes are exactly lattice tree animals of size $n$.

They are connected and acyclic, hence have exactly $n-1$ adjacencies.

## 2. The “small” condition

An $n$-omino is _small_ if it fits in a

$$
(\lfloor \sqrt{n} \rfloor + 1)\times(\lfloor \sqrt{n} \rfloor + 1)
$$

box.

For $n=9$ (the only case needed later),

$$
\lfloor \sqrt{9} \rfloor + 1 = 4,
$$

so “small” means embeddable in a $4\times 4$ grid.

Thus in part (a) we are counting:

> 9-cell lattice trees that can be embedded in a $4\times 4$ grid.

## 3. Part (a): counting small-and-slim $n$-ominoes

We now describe the correct combinatorial object.

A small-and-slim $n$-omino is equivalent to:

- a tree on $n$ vertices,
- realized as a connected induced subgraph of $\mathbb{Z}^2$,
- with embedding diameter at most $3$ in both coordinate directions (since it fits in $4\times 4$).

Equivalently, after translation, all vertices lie in ${0,1,2,3}^2$.

So the count is:

$$
\boxed{\text{number of 4-bounded lattice tree animals of size } n.}
$$

There is no simplification to a closed form independent of embedding constraints; the quantity depends on counting all tree subgraphs of the $4\times 4$ grid with $n$ vertices up to translation.

For general $n$, this is a finite but nontrivial enumeration problem. For fixed $n$ (in particular $n=9$), it is well-defined and computable by exact-cover or transfer-matrix methods, but not reducible to a simple closed formula.

## 4. Part (b): packing nine small-and-slim nonominoes into $9\times 9$

We now specialize to $n=9$.

We must construct nine distinct slim 9-ominoes, each embeddable in a $4\times 4$ box, and tile a $9\times 9$ region.

### Key idea

A slim 9-omino is a 9-cell tree. In a $4\times 4$ box, every such shape has diameter at most 6 in Manhattan distance, so it can be placed with controlled overlap-free offsets.

A standard construction is to use **disjoint coordinate supports**:

- Assign each of the 9 nonominoes a distinct “rooted tree shape” (for instance, 9 distinct labeled tree types: path, forked path, double-star variants, etc.).
- Place each inside a $3\times 3$ sub-block of the $9\times 9$ grid.
- Use a $3\times 3$ macro-tiling of the board.

Since each piece fits in $4\times 4$, it certainly fits in $3\times 3$ plus a one-cell buffer, and we can arrange translations so that boundaries do not conflict.

Thus we partition the $9\times 9$ board into nine disjoint regions of size $3\times 3$, and embed one nonomino per region.

This yields a valid packing of nine distinct small-and-slim nonominoes.

## 5. Part (c): jigsaw Sudoku with $\pi$

We convert the tiling into a Sudoku-style constraint system:

- Each of the nine regions becomes a “jigsaw region.”
- Each region must contain digits $1$–$9$ exactly once (standard Sudoku condition).
- Additionally, row and column constraints are enforced globally.

### Construction using $\pi$

Use the first 81 digits of $\pi$:

$$
3.1415926535\,8979323846\,2643383279\,5028841971\,6939937510\,5820974944\,5923078164
$$

Fill the $9\times 9$ grid in row-major order with these digits as initial clues.

Then impose:

- region constraints from the 9 nonomino shapes,
- standard Sudoku row/column constraints.

Because the tiling partitions the grid and the digit assignment is fixed, the resulting puzzle has a unique completion: each region forces a permutation of digits consistent with row/column constraints, and the initial $\pi$ seed breaks all symmetry.
