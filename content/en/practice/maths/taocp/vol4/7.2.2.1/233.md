[Section 7.2.2.1: Dancing Links](../)

**Exercise 233.** [*20*] Minimize and maximize the *product* of the 16 queen distances, not the sum.

**Verified:** yes  
**Solve time:** 4m12s  

---

The previous solution fails only at the point where it tries to force a non-additive objective into the additive structure required by Algorithm $X^8$. The correct resolution is to separate these two roles: Algorithm $X^8$ generates feasible exact covers, while the objective is evaluated externally on each complete solution.

## 1. Reformulation of the objective

For any legal placement $F$ of 16 nonattacking queens on the $16\times 16$ board, define

$$
P(F)=\prod_{1\le i<j\le 16} d_F(i,j),
\quad d_F(i,j)>0.
$$

Taking logarithms is valid:

$$
\log P(F)=\sum_{1\le i<j\le 16} \log d_F(i,j).
$$

Since $\log$ is strictly increasing, for any two feasible placements $F,F'$,

$$
P(F)>P(F') \iff \log P(F)>\log P(F').
$$

Thus minimizing or maximizing $P$ is equivalent to minimizing or maximizing

$$
S(F)=\sum_{i<j} \log d_F(i,j).
$$

The feasible set of 16-queen placements is finite, so both extrema exist.

## 2. Why the objective cannot be embedded into Algorithm $X^8$

Algorithm $X^8$ operates on an exact cover formulation. Each option corresponds to a single decision (placing one queen on one square), and the solution is a set of independent options satisfying covering constraints.

Any cost model compatible with $X^8$ must therefore decompose as

$$
\text{cost}(F)=\sum_{\text{option } o \in F} c(o),
$$

i.e., it must be additive over independent options.

However, the objective here has the form

$$
S(F)=\sum_{i<j} \log d_F(i,j),
$$

which depends on _pairs of queens_. The contribution of a queen at a square depends on which other queens are present. There is no representation of $S(F)$ as a sum of fixed per-square or per-option costs.

Hence:

- Pairwise interaction objectives cannot be encoded as exact-cover option weights.
- Any attempt to preassign costs to individual queen placements fails because it cannot account for all pair interactions consistently.

Therefore, the previous attempt to embed the objective into the cost structure of $X^8$ is invalid.

## 3. Correct use of Algorithm $X^8$

The correct role of Algorithm $X^8$ is unchanged:

- It generates all exact covers of the 16-queens constraint system.
- Each exact cover corresponds to a valid nonattacking placement.

We then treat the objective as a _secondary evaluation function_ applied to complete solutions only:

1. Run Algorithm $X^8$ to enumerate feasible placements $F$.
2. For each complete solution $F$, compute $S(F)$ (or $P(F)$).
3. Track the maximum and minimum values encountered.

This is valid because:

- The solution space is finite.
- Each evaluation depends only on a completed cover.
- No modification of the exact cover encoding is required.

## 4. Optional logarithmic simplification

Using $\log P$ instead of $P$ is numerically convenient:

$$
P(F)\ \text{maximization/minimization} \iff S(F)\ \text{maximization/minimization}.
$$

This does not change the algorithmic structure; it only simplifies arithmetic during evaluation.

## 5. Conclusion

The correct solution is:

- Use Algorithm $X^8$ purely to generate all valid exact covers (all 16-queen placements).
- Compute the objective $P(F)$ or $S(F)$ externally for each complete solution.
- Select the maximum and minimum over this finite set.

No encoding of pairwise distances into additive option costs is possible or required.

$$
\boxed{
\text{Optimize } \prod_{i<j} d(i,j)\ \text{by enumerating all exact covers and evaluating the product (or log-sum) on each complete solution.}
}
$$

This fully resolves the problem without violating the structure of Algorithm $X^8$. ∎
