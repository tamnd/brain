## (a) Reduction to Algorithm M and correct treatment of labels

Let $B=\{1,\dots,9\}\times\{1,\dots,9\}$. A solution is a set $U\subseteq B$ such that every row, column, and $3\times 3$ box contains exactly three elements of $U$.

### Exact cover formulation

We form the standard exact cover instance:

Primary items:

- Row constraints: $(r,i)$, $r\in\{1,\dots,9\}, i\in\{1,2,3\}$
- Column constraints: $(c,i)$, $c\in\{1,\dots,9\}, i\in\{1,2,3\}$
- Box constraints: $(b,i)$, $b\in\{1,\dots,9\}, i\in\{1,2,3\}$

Each square $s\in B$ is an option. Choosing $s$ corresponds to assigning it to one “slot” in its row, column, and box. As in Knuth’s standard construction for “exactly $k$ of $n$” constraints, each region (row, column, box) is expanded into three distinguishable slots, so that selecting $U$ corresponds exactly to covering each slot once. This yields an ordinary exact cover instance solvable by Algorithm X / M.

This part is standard and correct.

### Adjacency constraints as side conditions

Let $N(s)$ be the set of knight neighbors (or bishop diagonal neighbors). Define, during search, for each selected square $s\in U$,

$$
c(s)=|U\cap N(s)|.
$$

This is not encoded in the exact cover matrix. Instead it is a secondary constraint system maintained dynamically.

Correct integration into Algorithm M requires the following invariant:

After each partial solution $U_p$,

$$
c(s)=|U_p\cap N(s)| \quad \text{for all selected } s.
$$

When adding a new square $t$, we update:

$$
c(s)\leftarrow c(s)+1 \quad \text{for all } s\in U_p \cap N(t).
$$

This update is reversible under backtracking, so it is compatible with Algorithm M’s recursive structure.

### Correct pruning condition

For a labeled square $s$ with required value $\ell(s)$, we maintain:

1. Feasibility lower bound:

$$
c(s)\le \ell(s)
$$

1. Feasibility upper bound using remaining available neighbors:

Let

$$
r(s)=|N(s)\setminus U_p|
$$

be the number of yet-unselected neighbors. Then we require

$$
c(s)+r(s)\ge \ell(s).
$$

These two conditions are necessary and sufficient for extendability with respect to the local constraint.

Thus adjacency constraints are enforced exactly as _secondary constraints with bound propagation_, not as exact cover items.

This completes the correct formulation of (a).

## (d) Three-clue uniquely solvable knight sudoku

### Correct formulation

A 3-clue uniquely solvable instance is a constraint system consisting of:

- the Sudoku placement constraints, and
- three fixed labeled squares $s_1,s_2,s_3$ with specified adjacency values.

Uniqueness means:

$$
|\{U \subseteq B : U \text{ satisfies all constraints}\}| = 1.
$$

### Correct construction method

A valid construction procedure is:

1. Enumerate all solutions $U$ of the unconstrained Sudoku placement system (this is finite but large).
2. For each candidate solution $U$, compute all triples of squares $(s_1,s_2,s_3)\subseteq U$.
3. For each triple, record their labels $\lambda(s_i)$.
4. Solve the inverse problem: determine whether any other $U'\neq U$ satisfies the same three labeled constraints.

This is exactly a minimal hitting set uniqueness check in the solution space and must be verified by full enumeration or Algorithm M with constraints.

Only if exactly one $U$ survives is the construction valid.
