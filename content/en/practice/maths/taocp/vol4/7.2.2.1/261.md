**Exercise 261.** &#9654; [*23*] [23] Let $G$ be a directed acyclic graph, with source vertices $S$ and sink vertices $T$.
a) Use Algorithm C (or Z) to find all sets of $m$ vertex-disjoint paths from $S$ to $T$.
b) Also find all such sets of paths from $s_k$ to $t_k$ for $1 \le k \le m$, given $s_k$ and $t_k$.
c) Apply (a) to find all sets of $n-1$ disjoint paths that enter an $n \times n$ square at the north or east edge, proceed by south and/or west steps, and exit at the south or west edge, avoiding the corners. (A random $16 \times 16$ example is shown.)

## Solution

A set of vertex-disjoint directed paths in a DAG can be represented as a partial selection of directed edges in which each vertex is used in at most one path, and every selected internal vertex has exactly one incoming selected edge and exactly one outgoing selected edge. Algorithm C or Algorithm Z enforces such constraints by treating each vertex constraint as an item in an exact cover instance and each admissible local choice as an option that covers the relevant items.

For part (a), introduce one item for every vertex $v \in V$, representing the condition that $v$ is used in a path. For every directed edge $u \to v$, introduce an option corresponding to using this edge in a path. When this option is chosen, vertex $u$ is committed to having an outgoing selected edge, and vertex $v$ is committed to having an incoming selected edge. In the language of Algorithm C, this is enforced by refining the state so that when an edge $u \to v$ is chosen, vertex $u$ is immediately constrained to choose exactly one outgoing edge and vertex $v$ is constrained to choose exactly one incoming edge among the remaining compatible edges. The recursion of Algorithm C ensures that once a vertex is covered by one incident edge in a given direction, all conflicting incident edges are removed.

To enforce vertex-disjointness, the item corresponding to each vertex is covered as soon as it is included in a path, which prevents it from appearing in any other path. A path from a source $s \in S$ to a sink $t \in T$ is then generated as a maximal sequence of consistent edge selections produced by the recursive extension of Algorithm C starting from $s$ until reaching some $t$. A complete solution consisting of $m$ vertex-disjoint $S$–$T$ paths is obtained by selecting $m$ disjoint starting vertices in $S$ and extending each into a complete path; disjointness is enforced by the covering mechanism on vertices, so no vertex can be reused across different recursive branches.

Thus Algorithm C naturally generates all families of $m$ vertex-disjoint $S$–$T$ paths by repeated choice of a start vertex in $S$, followed by deterministic backtracking search over outgoing edges, with vertex covering enforcing disjointness.
