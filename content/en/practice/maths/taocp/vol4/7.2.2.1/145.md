**Exercise 145.** &#9654; [*M28*] $[M28]$ Many problems that involve an $l \times m \times n$ cuboid require a good internal representation of its $(l+1)(m+1)(n+1)$ vertices, its $l(m+1)(n+1) + (l+1)m(n+1) + (l+1)(m+1)n$ edges, and its $lm(n+1)+l(m+1)n+(l+1)mn$ faces, in addition to its $lmn$ individual cells. Show that there's a convenient way to do this with integer coordinates $(x, y, z)$ whose ranges are $0 \le x \le 2l$, $0 \le y \le 2m$, $0 \le z \le 2n$.

**Verified:** yes  
**Solve time:** 1m20s  

---

## Setup

Consider the integer lattice box

$0 \le x \le 2l,\quad 0 \le y \le 2m,\quad 0 \le z \le 2n,$

and define four classes of lattice points:

Vertices:

$V = \{(x,y,z) : x,y,z \text{ are even}\}.$

Edges:

$E = \{(x,y,z) : \text{exactly one of } x,y,z \text{ is odd}\}.$

Faces:

$F = \{(x,y,z) : \text{exactly two of } x,y,z \text{ are odd}\}.$

Cells:

$C = \{(x,y,z) : x,y,z \text{ are odd}\}.$

The task is to show that these sets encode respectively the vertices, edges, faces, and cells of an $l \times m \times n$ cuboid, and that their cardinalities match the formulas given in the problem statement.

## Solution

### 1. Vertices

A point $(x,y,z)$ has even coordinates exactly when

$x = 2i,\quad y = 2j,\quad z = 2k,$

with integers

$0 \le i \le l,\quad 0 \le j \le m,\quad 0 \le k \le n.$

Thus vertices correspond bijectively to integer triples $(i,j,k)$ in this range. The number of such triples is

$(l+1)(m+1)(n+1).$

This matches the required vertex count.

### 2. Edges

An edge of the cuboid is determined by fixing two coordinates at vertices and varying the third between them. Under the encoding:

- An edge parallel to the $x$-axis has midpoint $(2i+1,2j,2k)$.
- An edge parallel to the $y$-axis has midpoint $(2i,2j+1,2k)$.
- An edge parallel to the $z$-axis has midpoint $(2i,2j,2k+1)$.

Each such midpoint has exactly one odd coordinate.

Counting:

- $x$-parallel edges: choose $j,k$ with $0 \le j \le m$, $0 \le k \le n$, and $i$ with $0 \le i \le l-1$, giving $l(m+1)(n+1)$.
- $y$-parallel edges: $(l+1)m(n+1)$.
- $z$-parallel edges: $(l+1)(m+1)n$.

Summing gives

$l(m+1)(n+1) + (l+1)m(n+1) + (l+1)(m+1)n,$

which matches the required formula.

### 3. Faces

A face is determined by fixing one coordinate at a vertex position and allowing the other two to vary over an elementary square. In this encoding, face centers occur at points with exactly two odd coordinates:

- Face perpendicular to $x$-axis:

$(2i+1,2j+1,2k), \quad 0 \le i \le l-1,\; 0 \le j \le m-1,\; 0 \le k \le n.$

Count: $l m (n+1)$.
- Face perpendicular to $y$-axis:

$(2i+1,2j,2k+1), \quad 0 \le i \le l-1,\; 0 \le j \le m,\; 0 \le k \le n-1.$

Count: $l (m+1) n$.
- Face perpendicular to $z$-axis:

$(2i,2j+1,2k+1), \quad 0 \le i \le l,\; 0 \le j \le m-1,\; 0 \le k \le n-1.$

Count: $(l+1)m n$.

Summing yields

$lm(n+1) + l(m+1)n + (l+1)mn,$

which matches the required expression.

### 4. Cells

A cell corresponds to a unit cube indexed by $(i,j,k)$ with

$0 \le i \le l-1,\quad 0 \le j \le m-1,\quad 0 \le k \le n-1.$

Its center is encoded as

$(2i+1,2j+1,2k+1),$

which has all coordinates odd.

The number of such triples is

$lmn,$

matching the required count.

## Verification

Every lattice point in the box has each coordinate either even or odd, so the four parity classes listed partition the set of all integer points.

No vertex point can have an odd coordinate, since all vertices are of the form $(2i,2j,2k)$. No edge midpoint can have more than one odd coordinate, since edges vary in exactly one coordinate direction. No face center can have fewer than two odd coordinates, since it lies between two coordinate directions. No cell center can fail to have three odd coordinates, since it lies strictly inside a unit cube.

Each class corresponds bijectively to a geometric element of the subdivided cuboid, and the counts agree with the combinatorial construction in each case.
