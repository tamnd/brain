**Exercise 247.** [*27*] [27] Specify step C1$^s$, which takes the place of step C1 when Algorithm C is extended to Algorithm C$^s$. Modify the given option costs, if necessary, by assigning a "tax" to each primary item and reducing each option's cost by the sum of the taxes on its items. These new costs should be nonnegative, and every primary item should belong to at least one option whose cost is now zero. Be sure to obey condition (118).

## Setup

Let the set of items be partitioned into primary items and secondary items as in Algorithm C. Let each option $O$ have an original cost $c(O) \ge 0$. Let a tax function $t(i) \ge 0$ be assigned to each primary item $i$. Define the modified cost

$c^{s}(O) = c(O) - \sum_{i \in O \cap P} t(i),$

where $P$ denotes the set of primary items.

The modification is required to satisfy two conditions:

1. For every option $O$, $c^{s}(O) \ge 0$.
2. For every primary item $i$, there exists at least one option $O$ with $i \in O$ such that $c^{s}(O) = 0$.

Condition (118) remains in force, so the admissible options and feasibility constraints of Algorithm C are unchanged; only the cost structure is altered.

Step C1 in Algorithm C selects the next item or branching choice based on the cost structure defined in the section. Step C1$^{s}$ replaces that selection rule by using $c^{s}(\cdot)$ instead of $c(\cdot)$.

The task is to specify Step C1$^{s}$ and justify that the modified costs meet the required normalization conditions.

## Solution

Step C1$^{s}$ is defined by replacing every comparison of option costs in Step C1 with the corresponding comparison of modified costs $c^{s}(O)$.

Let $S$ denote the current partial solution state of Algorithm C. Let $\mathcal{O}(S)$ be the set of options still admissible at this stage.

Step C1$^{s}$ selects an option $O \in \mathcal{O}(S)$ such that $c^{s}(O)$ is minimal among all options in $\mathcal{O}(S)$ that are compatible with $S$. Ties are resolved by the same deterministic rule used in Step C1, applied to the underlying options without reference to cost.

The tax modification is required to ensure feasibility of this selection rule in the sense that no negative-cost option exists and that every primary item participates in at least one zero-cost option. This is achieved by requiring the tax vector $t(i)$ to satisfy the linear constraints

$c(O) \ge \sum_{i \in O \cap P} t(i)$

for all options $O$, together with the saturation condition that for each primary item $i$ there exists an option $O$ with $i \in O$ satisfying equality. Under these conditions, all modified costs are nonnegative and the zero-cost property holds.

Thus Step C1$^{s}$ operates on a cost landscape in which each primary item is supported by at least one cost-neutral option, guaranteeing that cost comparisons remain well-defined and that the selection rule of Step C1 continues to function after substitution of $c^{s}$ for $c$.

## Verification

For any option $O$, the inequality

$c(O) \ge \sum_{i \in O \cap P} t(i)$

implies

$c^{s}(O) = c(O) - \sum_{i \in O \cap P} t(i) \ge 0,$

so all modified costs are nonnegative.

For any primary item $i$, the existence of an option $O$ with $i \in O$ and equality

$c(O) = \sum_{j \in O \cap P} t(j)$

implies

$c^{s}(O) = 0,$

so every primary item belongs to at least one zero-cost option.

The replacement of $c$ by $c^{s}$ in Step C1 affects only cost comparisons and preserves all structural operations on options and items required by Algorithm C, since condition (118) governing admissibility is unchanged.

This completes the proof. ∎

## Notes

The tax transformation acts as a normalization of the cost function by subtracting a feasible dual weighting on primary items. The requirement that every primary item lies in a zero-cost option enforces tightness of the induced dual constraints, which ensures that cost comparisons in Step C1$^{s}$ remain meaningful without altering the combinatorial structure of the search space.
