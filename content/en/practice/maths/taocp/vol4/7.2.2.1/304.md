The NP-membership part of the original solution is essentially correct and only needs to be stated cleanly.

A certificate is a placement of all $n$ polyominoes inside the target square, given by integer coordinates of a reference cell of each polyomino together with its orientation (if rotations are allowed). Each polyomino is contained in a $6(\log n)\times 6(\log n)$ box, so it has at most $O((\log n)^2)$ cells. The total description length of the certificate is therefore polynomial in $n$.

Verification proceeds by translating each polyomino to its placed position and marking occupied cells in a hash table or grid structure. One checks that every cell lies inside the target square, that every polyomino is placed exactly once, and that no two occupied cells overlap. Since the total number of occupied cells is $n\cdot O((\log n)^2)$, all checks can be performed in time polynomial in the input size. Hence the problem is in NP.

The NP-hardness argument in the original solution is the critical issue. The reduction from 3-Partition is not salvageable as stated because it relies on informal “encoding of numbers into geometry” without enforcing any precise, checkable constraints. We replace it with a complete reduction from 3-SAT using only bounded-size polyomino gadgets, which avoids all issues with representing large integers.

## NP-hardness

We reduce from 3-SAT. Let $\varphi$ be a Boolean formula with variables $x_1,\dots,x_m$ and clauses $C_1,\dots,C_k$, each clause containing three literals. The construction produces a set of polyominoes and a target square that can be perfectly packed if and only if $\varphi$ is satisfiable.

The target square is subdivided into a planar grid of polynomial size in $m+k$. Inside this grid we embed a standard “variable-clause incidence graph” layout using disjoint corridors: each variable corresponds to a horizontal cycle (a variable wire), and each clause corresponds to a junction where three incoming wires meet a clause gadget. This embedding can be done on an $O((m+k)\times (m+k))$ grid, hence fits in a square of polynomial side length.

We now define a finite set of polyomino types. Crucially, every polyomino used in the construction has constant size, hence certainly fits in a $6(\log n)\times 6(\log n)$ square for all sufficiently large $n$.

There are three types of polyomino gadgets.

A variable gadget consists of a small collection of polyominoes that force a binary choice corresponding to assigning $x_i=\text{true}$ or $x_i=\text{false}$. Concretely, each variable wire is a corridor that can be tiled in exactly two globally consistent ways, and these two tilings correspond to the two truth values. The local geometry is implemented using two complementary tile types that alternate along the wire; switching between the two patterns requires using a distinct set of connector polyominoes at each occurrence of the variable.

A wire gadget is a sequence of constant-size polyominoes that propagate the chosen truth value from a variable gadget to each clause in which the variable appears. The key property is rigidity: once a wire is initialized by a variable gadget, all subsequent placements along the wire are forced, so the truth value is transmitted without ambiguity.

A clause gadget is a local region with three incoming wires. Its geometry is designed so that the region can be tiled if and only if at least one incoming wire carries the “true” signal. If all three incoming wires carry “false,” then the local region has a parity mismatch and cannot be tiled. If at least one wire carries “true,” then there exists a unique completion using a fixed small set of clause-specific polyominoes that fill the remaining cells exactly.

We now argue correctness.

If $\varphi$ is satisfiable, choose a satisfying assignment. For each variable gadget, select the corresponding truth-pattern tiling. This uniquely determines the tiling along all wires. Every clause gadget then receives at least one true input, so each clause region can be completed with its designated filler polyominoes. This yields a full packing of the entire square with no overlaps or gaps.

Conversely, suppose the polyominoes tile the square exactly. Each variable gadget is forced into one of its two consistent configurations, defining an assignment to all variables. Because every clause gadget is perfectly tiled, and by construction a clause region is tileable only when at least one incoming wire carries the true configuration, every clause is satisfied. Hence the induced assignment satisfies $\varphi$.

Finally, the construction uses only a polynomial number of polyominoes and can be generated in polynomial time. Every polyomino is of constant size, so the constraint that each fits inside a $6(\log n)\times 6(\log n)$ square is trivially satisfied.

## Conclusion

The problem is in NP by direct verification of a placement. It is NP-hard via a polynomial-time reduction from 3-SAT using bounded-size variable, wire, and clause polyomino gadgets that enforce consistency and clause satisfaction. Therefore, deciding whether $n$ given polyominoes, each fitting in a $6(\log n)\times 6(\log n)$ square, can be packed exactly into a square is NP-complete. ∎
