\mathcal{P} = \{P_1, P_2, \dots, P_M\},

Thus MacMahon’s problem (59) is exactly the exact-cover instance $(U,\mathcal{P})$.

## 2. Solutions and their exact characterization

A solution of MacMahon’s problem is a subset of placements

$$
\mathcal{S} \subseteq \mathcal{P}
$$

such that:

1. No overlap:

$$
P \cap Q = \varnothing \quad \text{for distinct } P,Q \in \mathcal{S}.
$$

1. Full coverage:

$$
\bigcup_{P \in \mathcal{S}} P = U.
$$

Hence the solution set is

$$
\mathfrak{S} = \{\mathcal{S}_1, \dots, \mathcal{S}_K\},
$$

where each $\mathcal{S}_i$ is a distinct exact cover of $U$.

### Key point: what Algorithm C enumerates

Algorithm C (TAOCP backtracking for exact cover) constructs solutions by:

- choosing the first uncovered item $u \in U$ in the fixed coordinate order,
- branching over all placements $P \in \mathcal{P}$ containing $u$,
- recursively continuing on the reduced instance.

This induces a search tree whose terminal nodes are exactly the solutions $\mathcal{S}_1, \dots, \mathcal{S}_K$.

Therefore:

$$
\text{Solutions of MacMahon (59)} \;\equiv\; \text{terminal nodes of Algorithm C on } (U,\mathcal{P}).
$$

No solution is missed and no solution is duplicated, because every recursive step fixes the smallest-index uncovered item and chooses exactly one placement covering it. This forces a canonical representation of each exact cover.

Thus each solution corresponds to exactly one root-to-leaf path in the search tree.

## 3. Structure of Algorithm C on this instance

Let a state of the algorithm be a subset $U' \subseteq U$ of uncovered items.

At each step:

1. Select the least $u \in U'$.
2. Form the candidate list

$$
\mathcal{P}(u) = \{P \in \mathcal{P} : u \in P\}.
$$
3. Branch over all $P \in \mathcal{P}(u)$.
4. Recurse on

$$
U' \setminus P.
$$

The branching factor at $u$ is therefore $|\mathcal{P}(u)|$, determined entirely by the geometry of the coordinate system and piece placements.

## 5. Cost of Algorithm C on this instance

In Algorithm C, each recursive node performs two expensive operations:

### (i) Candidate search cost

To compute $\mathcal{P}(u)$, the algorithm scans the global option set:

$$
\Theta(M_u) \quad \text{where } M_u = |\mathcal{P}(u)|.
$$

This scan is repeated independently at many nodes, even when most options become invalid due to earlier choices.

### (ii) Consistency checking cost

Each trial placement $P$ requires checking intersection with the current partial solution, costing $\Theta(|P|)$, repeated many times.

Thus node cost is:

$$
c_C(u) = \Theta\!\left(M_u + \sum_{P \in \mathcal{P}(u)} |P|\right).
$$

Aggregating over the search tree, Algorithm C repeatedly traverses the same incidence information.

## 6. Improved algorithm of exercise 122

Exercise 122 replaces the explicit set scans with a fully dynamic covering structure (Dancing Links style):

- Each item $u$ maintains a doubly linked list of incident options.
- Each option $P$ maintains pointers into the items it covers.
- Covering an item removes all conflicting options in time proportional only to what is actually removed.
- Uncovering restores structure in reverse order.

### Crucial change

Each incidence pair $(u,P)$ is **touched at most once per level of the search path**, rather than being repeatedly scanned.

Thus:

- Algorithm C: repeated scanning of option lists.
- Improved algorithm: each incidence is removed and restored via pointer updates only when necessary.

## 7. Correct time comparison

Let

$$
I = \sum_{P \in \mathcal{P}} |P|
$$

be the total number of incidences in the exact-cover matrix.

### Algorithm C time

Because option lists are repeatedly scanned, the same incidences are revisited many times across different nodes. The total time has the form:

$$
T_C = \Theta\!\left(\sum_{\text{nodes}} \sum_{u \in U'} M_u\right),
$$

which effectively reprocesses large portions of the incidence structure multiple times.

### Improved algorithm time

Each incidence $(u,P)$ is deleted and restored only when it is actually affected by a choice. Each such operation is constant time.

Hence:

$$
T_{122} = \Theta(I + \text{search overhead}).
$$

The key point is that the incidence structure is no longer repeatedly scanned.

## 8. Time saved

The saving is the elimination of repeated traversal of option lists.

Define the average multiplicity of scanning in Algorithm C as:

$$
\sigma = \frac{\text{total times incidences are scanned in C}}{\text{number of incidences } I}.
$$

Then:

$$
T_C = \Theta(\sigma \cdot I + \text{backtracking overhead}), \quad
T_{122} = \Theta(I + \text{backtracking overhead}).
$$

Therefore the speedup factor is:

$$
\frac{T_C}{T_{122}} = \Theta(\sigma).
$$
