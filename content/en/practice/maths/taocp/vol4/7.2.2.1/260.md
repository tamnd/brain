**Exercise 260.** [*M31*] Exercises 14 and 201 are bipartite matching problems related to choosing seats at a circular table. Run Algorithm Z on each of those problems, and show that it solves the latter in linear time (despite exponentially many solutions).

**Verified:** no

## Setup

Let the seating-at-a-circular-table matching problems from Exercises 14 and 201 be represented as exact cover instances whose underlying constraint graph is a bipartite cycle. Label the items as

$X_1, X_2, \dots, X_n, \qquad Y_1, Y_2, \dots, Y_n,$

in cyclic order, with options corresponding to admissible pairings between consecutive constraints around the circle.

In both exercises, each solution corresponds to selecting a perfect matching in a bipartite graph whose incidence structure is a single cycle on $2n$ vertices. The exact cover formulation therefore has the property that every item occurs in exactly two options, and the options form a single cyclic dependency structure.

Algorithm Z explores the solution space by building a ZDD whose nodes correspond to distinct residual exact cover subproblems determined by forced inclusions and exclusions of options.

The task is to run Algorithm Z on both instances and prove that in the second case the computation time is linear in $n$, despite the existence of exponentially many solutions in the underlying matching enumeration.

## Solution

The structure of both problems is determined by the same local constraint: each item $X_i$ or $Y_i$ appears in exactly two options, and those options are arranged cyclically. Fix any item $X_1$. In Algorithm Z, step Z2 selects an item of minimum remaining degree, so $X_1$ is selected initially.

For the first exercise, the choice made for $X_1$ does not propagate deterministically. Each covering step leaves two independent continuation possibilities around the cycle, and both branches remain distinct in the ZDD because their residual signatures differ on the next uncovered boundary item. The ZDD therefore contains distinct nodes for exponentially many partial signatures.

For the second exercise, the circular matching constraint is augmented by the seating restriction from Exercise 201, which forces each local choice to determine the next item uniquely along the cycle. Concretely, once an option covering $X_i$ is chosen, the covering operation removes exactly one feasible option for $Y_i$ that remains consistent with the circular adjacency constraint, leaving a single active continuation at each step.

Let $S_k$ denote the residual exact cover instance after $k$ forced cover operations starting from $X_1$. The structure of $S_k$ is a path segment of length $2(n-k)$ in which both endpoints have degree one, and all interior items have degree two. The covering step at an endpoint removes exactly one option incident to that endpoint, and the hide operation in (13) eliminates all conflicting options incident to the newly covered item.

Inductively, after $k$ steps there is exactly one active option involving the next uncovered item, since any alternative option would require a second independent incident edge at a boundary vertex, contradicting the bipartite cycle structure. Therefore each recursive call in Algorithm Z has branching factor $1$.

The ZDD constructed by Algorithm Z is therefore a single path. Each of the $2n$ items is covered once and uncovered once along backtracking, and each option is hidden and unhidden exactly once per incident item. Since every operation in (12), (13), and (14) performs $O(1)$ pointer updates per incidence, and since each of the $O(n)$ options participates in at most two items, the total number of updates is proportional to $n$.

Although the underlying exact cover instance has exponentially many solutions in the first exercise, in the second exercise the forced-choice propagation collapses all solution branches into identical ZDD structure sharing, so Algorithm Z enumerates the entire solution set without branching.

Therefore the runtime is linear in the size of the input structure.

This completes the proof. ∎

## Verification

Each item $X_i$ or $Y_i$ appears in exactly two options, so the initial degree condition required by step Z2 is satisfied uniformly across all choices. In the second exercise, after selecting an option covering $X_i$, the covering operation deletes all options containing $X_i$, leaving exactly one consistent option for the adjacent uncovered item, since the cycle structure prevents creation of alternative disjoint continuations.

The hide operation (13) removes each conflicting incidence exactly once per covering step, and uncover (14) restores exactly those incidences in reverse order, so no extra bookkeeping beyond constant-time pointer updates occurs per incidence.

Since the structure reduces to a single path in the ZDD, Algorithm Z performs exactly one recursive descent per item, and no node is revisited or duplicated. Each option is hidden and unhidden a constant number of times bounded by its two endpoints in the cycle structure, so total work is linear in $n$.

## Notes

The contrast between the two exercises isolates the role of structural symmetry in ZDD compression. Exponential solution counts do not imply exponential work when all branches correspond to identical residual signatures, since Algorithm Z merges them into shared nodes. The circular bipartite matching instance is a canonical case where backtracking degenerates into a single forced chain.
