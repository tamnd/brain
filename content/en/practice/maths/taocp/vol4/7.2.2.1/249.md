**Exercise 249.** [*21*] [21] A set of $dt$ costs, with $0 \le c_1 \le c_2 \le \cdots \le c_{dt}$, is said to be *bad* if $c_1 + c_2 + \cdots + c_{dt} > \theta$. Design an "online algorithm" that identifies a bad set as quickly as possible, when the costs are learned one by one in arbitrary order.

For example, suppose $d = 6$, $k = 2$, and $\theta = 16$. If costs appear in the order $(3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5, 8)$, your algorithm should stop after seeing the 2.

**Verified:** yes  
**Solve time:** 1m18s

## Solution

Let the arriving costs be $x_1, x_2, \ldots$, where the multiset is exactly ${c_1, \ldots, c_{dt}}$ and $0 \le c_1 \le \cdots \le c_{dt}$. Define a running sum

$S_k = x_1 + x_2 + \cdots + x_k.$

The algorithm maintains $S_k$ as costs arrive. After each arrival $x_k$, it updates

$S_k \leftarrow S_{k-1} + x_k,$

and checks whether $S_k > \theta$. If this condition holds, it halts and declares the set bad. If the stream ends without triggering this condition, it declares the set not bad.

Correctness follows from the nonnegativity of costs. After $k$ arrivals, the remaining unseen costs satisfy $x_{k+1} + \cdots + x_{dt} \ge 0$, hence the total sum is at least $S_k$. If the algorithm halts at some $k$ with $S_k > \theta$, then the full sum satisfies

$x_1 + \cdots + x_{dt} \ge S_k > \theta,$

so the set is bad. If the algorithm does not halt, then $S_{dt} \le \theta$, so the set is not bad.

To show optimality in the online sense, consider any algorithm that attempts to stop before the running sum exceeds $\theta$. Suppose such an algorithm halts after observing a proper prefix $(x_1, \ldots, x_k)$ with $S_k \le \theta$. At that moment, no information is available about the remaining costs except that they are nonnegative. An adversary can assign all remaining costs to be $0$, producing a valid completion consistent with the ordering constraints, for which the total sum equals $S_k \le \theta$. Under this completion the set is not bad, so any algorithm that halts at this point would be incorrect on some valid instance. Hence no correct online algorithm can terminate strictly before the moment when the running sum first exceeds $\theta$.

Since the presented procedure halts exactly when $S_k > \theta$ and no earlier stopping rule can be correct, it achieves the earliest possible detection of a bad set in the online model.

This completes the proof. ∎
