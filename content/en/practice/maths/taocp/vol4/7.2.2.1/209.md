## Setup

Let the exact cover instance be restricted to the _bipartite_ form considered in search (93), where the set of items is partitioned into two disjoint classes $U$ and $V$, and every option contains exactly one item of $U$ and exactly one item of $V$. Thus each option has the form $(u,v)$ with $u \in U$ and $v \in V$.

An exact cover is a set of options $C$ such that every item in $U \cup V$ occurs in exactly one option of $C$.

Search (93) applies Algorithm X in its general form: repeatedly select an item $i$, branch over options containing $i$, and recursively cover all items in each chosen option.

The goal is to modify this search in the same way that (100) improves (91), by exploiting the bipartite restriction to reduce branching and propagation overhead.

## Solution

Let $i \in U \cup V$ be an arbitrary item. In the bipartite setting, every option containing $i$ pairs $i$ with a unique item in the opposite part. If $i \in U$, each option is of the form $(i,v)$; if $i \in V$, each option is of the form $(u,i)$.

The key modification is to avoid treating both sides symmetrically during branching. Instead, branching is performed only on items in $U$, while items in $V$ are handled implicitly by forced pairing.

The modified search proceeds as follows.

Choose an item $u \in U$ that is still uncovered. If no such item exists, the remaining items in $V$ are automatically determined, and a solution is obtained exactly when every item in $V$ is also covered.

If no option contains $u$, backtrack.

Otherwise, for each option $(u,v)$, perform a single branch in which $(u,v)$ is selected. In that branch, both $u$ and $v$ are covered simultaneously. Every option containing either $u$ or $v$ is deleted from the active structure, since no further option may reuse either item.

After covering $u$ and $v$, the residual problem remains bipartite on the induced subinstance obtained by removing $u$, $v$, and all incident options. The recursion continues on this smaller instance.

This modification eliminates any need to branch on items in $V$. Each item in $V$ is determined uniquely by the choice of its partner in $U$.

The correctness follows from the bijection between solutions and selections of disjoint pairs $(u,v)$ covering all items: every valid cover selects exactly one option incident to each $u \in U$, and each such selection determines a unique covering of $V$. Conversely, any consistent selection of disjoint options covering all of $U$ necessarily covers all of $V$ exactly once, since every option contributes exactly one $v \in V$ and no two options share a $v$.

This completes the construction of the improved search.

## Verification

Each recursive step removes exactly one item from $U$ and exactly one item from $V$, since every option contains one of each. Hence after $k$ recursive selections, exactly $k$ items of $U$ and $k$ items of $V$ have been removed, preserving the invariant that uncovered items in $V$ correspond precisely to those not yet paired with chosen items in $U$.

No option containing a covered item can appear in a valid solution, since any such option would reuse an already covered element of either $U$ or $V$, contradicting the exact cover condition.

Every complete branch that covers all items in $U$ necessarily covers all items in $V$, since each selection removes exactly one item of $V$, and the number of selections equals $|U|$ at termination.

Thus the modified search produces exactly the same solution set as search (93), while reducing branching to choices over $U$ only.

This completes the proof. ∎

## Notes

The improvement mirrors the reduction in (100), where structural constraints remove symmetric branching. Here the bipartite restriction converts the general exact cover search into a matching-style search in which one side of the bipartition is eliminated from the branching process. This is equivalent to viewing the problem as selecting a perfect matching in a bipartite graph, with Algorithm X acting as a backtracking matcher driven solely by choices on one partition.
