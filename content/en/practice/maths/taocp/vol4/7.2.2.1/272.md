**Exercise 272.** [*23*] [23] Continuing exercise 271, explain how to find special kinds of packings:
a) Those that decompose into $6 \times k$ and $6 \times (10-k)$.
b) Those that have all twelve pentominoes touching the outer boundary.
c) Those with all pentominoes touching that boundary *except* for V, which doesn't.
d) Same as (c), with each of the other eleven pentominoes in place of V.
e) Those with the minimum number of pentominoes touching the outer boundary.
f) Those that are characterized by Arthur C. Clarke's description, as quoted below. That is, the X pentomino should touch only the F (aka R), the N (aka S), the U, and the V, no others.

> *Very gently, he replaced the titanite cross*
> *in its setting between the F, N, U, and V pentominoes,*
>, ARTHUR C. CLARKE, *Imperial Earth* (1976)

Let the standard exact cover formulation from Exercise 271 be used: items are the $60$ board cells together with the $12$ pentomino names, and each option corresponds to a legal placement of one pentomino covering five cells.

A solution is a set of options that covers every cell exactly once and uses each pentomino exactly once.

## (a) Decomposition into $6\times k$ and $6\times (10-k)$

Fix $k$, $0 \le k \le 10$. Let the vertical cut split the board into two disjoint cell sets

$$
B_L = \{1,\dots,k\}\times \{1,\dots,6\}, \quad
B_R = \{k+1,\dots,10\}\times \{1,\dots,6\}.
$$

In the exact cover model, a placement is valid for the constrained problem if and only if all its covered cells lie entirely in $B_L$ or entirely in $B_R$.

Thus we construct, for each $k$, a reduced instance obtained by deleting every placement option that intersects both $B_L$ and $B_R$.

Correctness follows because:

- Every remaining option lies wholly in one side.
- Exact cover still enforces full coverage of all cells in both $B_L$ and $B_R$.
- No placement can straddle the cut, so the solution decomposes into two independent exact covers of the two sub-boards.

Running Algorithm X on each $k$ and taking the union of solutions produces exactly all such decompositions.

## (b) All twelve pentominoes touch the boundary

Let $B$ be the set of boundary cells of the $6\times 10$ rectangle.

A placement option $O$ is allowed if it satisfies

$$
\operatorname{cells}(O)\cap B \ne \varnothing.
$$

We delete all options that do not touch $B$. Then every selected pentomino must have at least one boundary cell, so all twelve pentominoes touch the boundary.

Exact cover constraints on pentomino items ensure each piece is used exactly once, so the condition holds globally.

## (c) All except $V$ touch the boundary

For pentomino $V$, allow only placements avoiding the boundary:

$$
\operatorname{cells}(O_V)\cap B = \varnothing.
$$

For every other pentomino $P \ne V$, restrict to placements satisfying

$$
\operatorname{cells}(O)\cap B \ne \varnothing.
$$

Deleting all violating options yields an exact cover instance whose solutions are exactly those in which every pentomino except $V$ touches the boundary.

## (d) Any chosen pentomino avoids the boundary

For each pentomino $P_0$, define the instance:

- For $P_0$: keep only placements disjoint from $B$,
- For all $P \ne P_0$: keep only placements intersecting $B$.

Running Algorithm X separately for each $P_0$ produces all cases in which exactly that pentomino fails to touch the boundary.

## (e) Minimum number of boundary-touching pentominoes

For each placement option $O$, define

$$
t(O)=
\begin{cases}
1 & \text{if } O \cap B \ne \varnothing,\\
0 & \text{otherwise.}
\end{cases}
$$

During Algorithm X search, maintain a partial solution $S$ and define

$$
T(S)=\sum_{O\in S} t(O).
$$

For each remaining pentomino $P$, define the optimistic lower bound

$$
m(P)=\min_{O\in \mathcal{O}_P^{\mathrm{rem}}} t(O).
$$

Any completion must choose one remaining option per pentomino, so a valid lower bound on the final number of boundary-touching pentominoes is

$$
T(S)+\sum_{P\in \mathcal{P}_{\mathrm{rem}}} m(P).
$$

Let $T_{\min}$ be the best complete solution found so far. During search, prune any node satisfying

$$
T(S)+\sum_{P\in \mathcal{P}_{\mathrm{rem}}} m(P)\ge T_{\min}.
$$

With standard initialization $T_{\min}=+\infty$, this yields all solutions, and after the first solution is found, it correctly restricts search to those achieving the minimum possible boundary-touching count.

## (f) Clarke constraint on adjacency of $X$

The condition is:

> The $X$ pentomino may touch only $F, N, U, V$ (and the outer boundary).

This is a **pairwise adjacency constraint between chosen pentomino placements**, not a property of a single placement alone. Therefore it cannot be enforced by filtering individual options in a pure exact cover formulation.

### Correct formulation

We keep the standard exact cover structure for cell coverage and pentomino usage, but augment the search with an additional compatibility test.

For each placement of $X$, precompute the set of grid cells it occupies. During the search, whenever $X$ is placed, we maintain a dynamic set

$$
A_X = \{P : \text{a chosen placement of } P \text{ is edge-adjacent to a cell of } X\}.
$$

This adjacency set is updated incrementally:

- When adding a placement of a pentomino $P$, check all edges between its occupied cells and already placed cells.
- If any adjacency involves $X$, add $P$ to $A_X$.

We enforce the constraint by pruning any partial solution satisfying

$$
A_X \nsubseteq \{F,N,U,V\}.
$$

This is valid because adjacency depends only on already fixed geometric placements in the partial solution, and once a forbidden adjacency is created it can never be removed.

### Why this resolves the issue

- Exact cover still guarantees correct tiling of the board.
- The adjacency rule is enforced as an additional constraint over partial solutions.
- No illegal solution can survive because forbidden adjacencies are detected when they first occur.
- No incorrect pre-elimination of placements is performed.

Thus the problem is correctly handled by extending Algorithm X with a stateful constraint check rather than attempting to encode adjacency purely at the level of individual options.

## Conclusion

Parts (a)–(e) are handled by restricting or augmenting the exact cover instance in standard ways.

Part (f) requires extending Algorithm X with an additional adjacency-tracking constraint, since adjacency between pentominoes is inherently a global property of the chosen solution and cannot be reduced to local option filtering.
