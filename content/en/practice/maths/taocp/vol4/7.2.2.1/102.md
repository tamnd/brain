The error in the previous solution comes from attempting to use secondary (colored) items to enforce consistency. In XCC / Algorithm C, secondary items do **not** propagate constraints by being “covered”; they only impose an **at-most-once usage condition** across chosen options. The previous argument incorrectly relied on a non-existent operational effect.

A correct solution avoids this entirely and uses a standard XCC encoding for Japanese arrow puzzles based only on primary items.

# Correct Solution

A Japanese arrow puzzle consists of a set of disjoint runs of cells. Each run $r$ has a target sum $S(r)$. The cells in each run must receive distinct digits from $\{1,\dots,9\}$ whose sum equals $S(r)$. Every cell belongs to exactly one run.

We encode this as an instance of XCC for Algorithm C.

## 1. Items

We use only **primary items**; no secondary items are required.

### Primary items

1. **Cell items**

$$
\{c : c \in \mathcal{C}\}
$$

Each cell must be assigned exactly one digit.

1. **Run items**

$$
\{r : r \in \mathcal{R}\}
$$

Each run must be chosen exactly once, ensuring exactly one assignment per run.

## 2. Options

Let a run $r \in \mathcal{R}$ contain ordered cells

$$
\mathcal{C}(r) = (c_1,\dots,c_k).
$$

For each assignment of digits

$$
(x_1,\dots,x_k) \in \{1,\dots,9\}^k
$$

satisfying:

1. $x_i \neq x_j$ for $i \neq j$,
2. $x_1 + \cdots + x_k = S(r)$,

we define one option

$$
O(r, x_1,\dots,x_k).
$$

This option represents choosing this digit pattern for run $r$, and it consists of the items:

- the run item $r$,
- the cell items $c_1,\dots,c_k$.

Formally,

$$
O(r, x_1,\dots,x_k)
=
\{r, c_1,\dots,c_k\}.
$$

## 3. Interpretation of XCC constraints

In XCC / Algorithm C:

- Primary items must be covered **exactly once**.
- Each option covers all items it contains.
- Each item may appear in multiple options, but is usable only if it is not already covered.

No additional mechanism is required.

## 4. Correctness

### (a) Each run is assigned exactly one pattern

Each run item $r$ appears in every option associated with run $r$. Since $r$ is a primary item, Algorithm C enforces that exactly one such option is chosen.

Thus each run is used exactly once.

### (b) Each cell receives exactly one digit

Each cell item $c$ appears in exactly one option within the chosen run (since runs are disjoint and each run is selected exactly once).

Because $c$ is a primary item, Algorithm C enforces that exactly one option covering $c$ is selected.

Therefore each cell belongs to exactly one chosen run-option, hence receives exactly one digit.

### (c) Consistency of digit assignments

There is no need for secondary items.

Consistency is enforced structurally:

- A cell belongs to exactly one run.
- Within that run, exactly one option is chosen.
- Each option assigns exactly one digit to each of its cells.

Thus no cell can receive conflicting digit assignments.

The previous attempt incorrectly tried to enforce this via secondary items, but XCC semantics already guarantee it.

### (d) Validity of run constraints

By construction of options, each option satisfies:

- digits are distinct within the run,
- digits sum to $S(r)$.

Hence every chosen option automatically satisfies the arrow constraint.

## 5. Bijection between solutions and exact covers

### From puzzle solution to XCC solution

Given a valid puzzle solution:

- For each run $r$, the assigned digits on its cells form a tuple $(x_1,\dots,x_k)$.
- This tuple satisfies the constraints, hence corresponds to exactly one option $O(r,x_1,\dots,x_k)$.
- Selecting all such options covers every run and every cell exactly once.

Thus every puzzle solution induces an exact cover.

### From XCC solution to puzzle solution

Given a set of chosen options forming a solution of XCC:

- Each run item appears in exactly one selected option, so each run is assigned exactly one digit pattern.
- Each cell item appears in exactly one selected option, so each cell receives exactly one digit.
- The digits are read directly from the unique option covering each cell.

Thus the selected options define a valid arrow-puzzle assignment.

## 6. Conclusion

The correct formulation is a pure XCC instance with:

- primary items: cells and runs,
- one option per valid run digit assignment,
- no secondary items.

The previous solution failed because it introduced secondary items but incorrectly assumed they actively enforce consistency via a non-existent “covering” mechanism. In Algorithm C, secondary items do not propagate constraints dynamically; correctness must come entirely from primary-item coverage structure, which the corrected encoding achieves directly. ∎
