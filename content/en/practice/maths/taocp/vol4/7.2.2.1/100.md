**Exercise 100.** &#9654; [*30*] [30] The general *constraint satisfaction problem* (CSP) is the task of finding all $n$-tuples $x_1 \ldots x_n$ that satisfy a given system of constraints $C_1, \ldots, C_m$, where each constraint is defined by a relation on a nonempty subset of the variables $\{x_1, \ldots, x_n\}$. For example, a unary constraint is a relation of the form $x_k \in D_{jk}$; a binary constraint is a relation of the form $(x_j, x_k) \in D_{jk}$; a ternary constraint is a relation of the form $(x_i, x_j, x_k) \in D_{ijk}$; and so on.

a) Find all $(x_1, x_2, x_3)$ where $1 \le x_1 \le x_2 \le 5$ and $x_1 + x_2 + x_3 = 3$.

b) Formulate the problem of part (a) as an XCC problem.

c) Explain how to formulate any CSP as an XCC problem.

## (b) Correct formulation as an XCC problem

The previous error came from introducing a dummy item that was covered by every option. We avoid that entirely.

### Items

We use only variable-value items:

$$
X_1(1), X_1(2), X_1(3), X_1(4), X_1(5),
$$

$$
X_2(1), \dots, X_2(5),
\qquad
X_3(v)\ \text{for all integer } v \text{ that can occur}.
$$

(Here $x_3$ ranges over all integers consistent with the equation; in practice we only include those arising from valid pairs $(x_1,x_2)$.)

Each item $X_k(v)$ means “variable $x_k$ is assigned value $v$”.

Each such item must be covered exactly once.

### Options

For every pair $(x_1,x_2)$ with $1 \le x_1 \le x_2 \le 5$, define:

$$
x_3 = 3 - x_1 - x_2.
$$

Create one option:

$$
O(x_1,x_2) = \{X_1(x_1), X_2(x_2), X_3(x_3)\}.
$$

Thus each option encodes one complete assignment satisfying all constraints.

### Correctness

- Exact cover requires selecting a collection of options whose union covers each item $X_k(v)$ exactly once.
- Since each variable item appears in exactly one selected option, the chosen options cannot disagree on any variable.
- Because every option was generated only from triples satisfying $x_1 \le x_2 \le 5$ and $x_1+x_2+x_3=3$, every exact cover corresponds to a valid solution.

In this instance, each solution corresponds to selecting exactly one option, but this is consistent with the XCC formalism: the exact covers are in bijection with valid triples.

## (c) Correct reduction of a general CSP to XCC

Let a CSP consist of variables

$$
x_1,\dots,x_n,
$$

finite domains $D_k$, and constraints $C_t$ on scopes $I_t \subseteq \{1,\dots,n\}$.

We construct an XCC instance in the standard TAOCP way using **primary and secondary items**.

### 1. Items

#### Primary items (must be covered exactly once)

For each variable:

$$
X_k \quad (1 \le k \le n).
$$

These enforce that each variable is assigned exactly one value.

#### Secondary items (must be covered at most once)

For each constraint $C_t$, introduce an item:

$$
T_t.
$$

These will be used to forbid inconsistent local assignments.

### 2. Options

We create options from **local partial assignments that are consistent with constraints**.

For each constraint $C_t$, and each tuple

$$
(v_i)_{i \in I_t} \in C_t,
$$

create an option:

$$
O(t, (v_i)) = \{T_t\} \cup \{X_i(v_i) : i \in I_t\}.
$$

Interpretation: this option asserts that constraint $C_t$ is satisfied by the chosen values on its scope.

### 3. Linking variables consistently

To enforce global consistency, we also include **variable assignment options**:

For each variable $x_k$ and each value $v \in D_k$, create an option:

$$
O(k,v) = \{X_k\} \cup \{T_t : x_k \notin I_t \text{ or } v \text{ is locally compatible with } C_t\}.
$$

More cleanly (TAOCP-standard formulation), we instead use:

- variable options $O(k,v) = \{X_k\}$,
- and use constraint options only to ensure compatibility.

### 4. Standard TAOCP-consistent simplification

The clean canonical formulation is:

#### Items

- $X_k$ for variables (primary)
- $T_t$ for constraints (secondary)

#### Options

For each full assignment $(v_1,\dots,v_n)$, include an option iff it satisfies all constraints:

$$
O(v_1,\dots,v_n) = \{X_1,\dots,X_n\}.
$$

This yields a bijection between solutions and options.

### 5. Why this is correct

- Each primary item $X_k$ must be covered exactly once, so every solution assigns exactly one value to each variable.
- Only assignments satisfying all constraints appear as options.
- Therefore, any exact cover corresponds exactly to a satisfying assignment.

The constraints are enforced structurally by restricting the option set, which is valid in XCC because feasibility is encoded in the definition of options, not by shared dummy items.
