We restart from the structure established in Exercise 169.

Let $G = (V, E)$ be the graph constructed in Exercise 169, where the independent sets are encoded via the selection of compatible primary and secondary items under the given constraints. In that formulation, a feasible set corresponds to a set of vertices containing no adjacent pair in $G$.

We are required to generate all $t$-element kernels of $G$, meaning all maximal independent sets of size $t$.

## Step 1: Reformulation of the goal

A set $K \subseteq V$ is a kernel (maximal independent set) if and only if:

1. (Independence) No two vertices in $K$ are adjacent in $G$.
2. (Maximality) For every vertex $v \in V \setminus K$, the set $K \cup \{v\}$ is not independent, equivalently every such $v$ is adjacent to at least one vertex in $K$.
3. $|K| = t$.

Thus, we must generate all independent sets of cardinality $t$ that dominate all remaining vertices.

## Step 2: Augmented formulation from Exercise 169

In Exercise 169, each feasible configuration is built by choosing items subject to mutual exclusion constraints encoded by edges of $G$. To extend that formulation, we introduce an additional status requirement for each vertex $v \in V$:

- $x_v \in \{0,1\}$, where $x_v = 1$ indicates inclusion in the independent set.
- The independence constraints remain:

$$
x_u + x_v \le 1 \quad \text{for every } \{u,v\} \in E.
$$
- Cardinality constraint:

$$
\sum_{v \in V} x_v = t.
$$
- Maximality constraint becomes:

$$
\forall v \in V,\quad x_v = 0 \implies \sum_{u \in N(v)} x_u \ge 1,
$$

where $N(v)$ is the neighborhood of $v$.

This reformulation converts the problem into generating all feasible solutions of a constraint system with both packing and covering conditions.

## Step 3: Backtracking generation scheme

We generate all kernels by a depth-first search over partial independent sets.

Define a recursive procedure $\mathrm{Gen}(S, U)$, where:

- $S$ is the current independent set,
- $U$ is the set of vertices not yet decided,
- $S$ is always independent,
- we maintain $|S| \le t$.

### Transition rules

At each step, select the least-indexed vertex $v \in U$. We branch as follows:

### Case 1: Include $v$

We may include $v$ only if it is not adjacent to any vertex in $S$. Then:

- $S' = S \cup \{v\}$,
- remove $v$ and all neighbors $N(v)$ from $U$,
- proceed with $\mathrm{Gen}(S', U \setminus (\{v\} \cup N(v)))$.

### Case 2: Exclude $v$

We exclude $v$, but then we must ensure it can still be dominated later:

- $S' = S$,
- proceed with $\mathrm{Gen}(S', U \setminus \{v\})$.

## Step 4: Enforcing maximality and size exactly $t$

Two pruning conditions are enforced.

### Size constraint

If $|S| > t$, abort the branch.

If $|S| = t$, we do not continue branching; instead we test maximality.

### Maximality test at leaves

When $U = \varnothing$ or $|S| = t$, we verify:

$$
\forall v \in V \setminus S,\quad N(v) \cap S \neq \varnothing.
$$

If this holds, output $S$ as a $t$-kernel.

## Step 5: Correctness proof

### Lemma 1 (Independence invariant)

Every recursive call maintains that $S$ is independent.

**Proof.** A vertex $v$ is added to $S$ only if no neighbor of $v$ is in $S$. Therefore no edge is ever introduced inside $S$. ∎

### Lemma 2 (Completeness of search)

Every independent set $S$ of size $t$ is generated in some branch.

**Proof.** For each vertex $v$, the recursion eventually makes an include/exclude decision. Thus every subset of vertices is represented by exactly one path in the decision tree, and all independent subsets of size $t$ are encountered. ∎

### Lemma 3 (Maximality filtering is correct)

A set $S$ is output only if it is maximal independent.

**Proof.** The explicit check ensures that no vertex outside $S$ is non-adjacent to all vertices in $S$. Hence no vertex can be added while preserving independence, so $S$ is maximal. ∎
