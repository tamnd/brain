**Exercise 13.** [*16*] *[16]* When Algorithm X finds a solution in step X2, how can we use the values of $x_0, x_1, \ldots, x_{l-1}$ to figure out what that solution is?

**Verified:** yes  
**Solve time:** 2m29s  

---

At step $X2$, Algorithm $X$ has successfully chosen a set of rows (options) forming an exact cover. The stack variables

$$
x_0, x_1, \ldots, x_{l-1}
$$

record precisely which row choices were made along the current search path. The task is to explain how these pointers determine the actual solution.

## Correct reconstruction of the solution

In Knuth’s Algorithm $X$ (DLX formulation), each option (row) is represented by a circular doubly linked list of nodes, linked horizontally by left–right pointers. Each node in such a row corresponds to an item contained in that option.

When the algorithm reaches step $X2$, all items have been covered exactly once by the selected rows, and the stack contains pointers to one chosen row node per level:

- each $x_k$ points to an arbitrary node belonging to the selected row at level $k$,
- each such row corresponds to one option in the original exact cover matrix.

Thus the solution is obtained by recovering, for each $x_k$, the full set of items in its row.

## How to recover the option from $x_k$

Fix $k$. The node $x_k$ lies in some row corresponding to an option. To reconstruct this option:

1. Start at node $x_k$.
2. Traverse the row using right pointers:

$$
x_k, \; R(x_k), \; R(R(x_k)), \ldots
$$
3. Continue until returning to $x_k$, since the row is circular.
4. For each visited node, record its column header, which names the item contained in that option.

Formally, the option corresponding to $x_k$ is

$$
\{\, \text{col}(x_k),\ \text{col}(R(x_k)),\ \text{col}(R^2(x_k)),\ \ldots \,\},
$$

where traversal stops when the cycle closes.

This produces exactly the set of items covered by the selected row.

## Why this works

At each level of Algorithm $X$, when a row is chosen:

- all columns (items) covered by that row are removed (“covered”) from the header list,
- all conflicting rows are removed from their column lists,
- the search proceeds only with compatible remaining options.

Therefore, when step $X2$ is reached:

- every item has been covered exactly once,
- the chosen rows are pairwise compatible (no shared items),
- the stack $(x_0,\ldots,x_{l-1})$ contains exactly one node per selected row.

Each $x_k$ is sufficient to identify its entire row because DLX stores rows as circular linked lists.

## Forming the final solution

The exact cover solution is the set of options corresponding to the rows represented by the stack:

$$
\mathcal{S} = \{ \text{row}(x_0), \text{row}(x_1), \ldots, \text{row}(x_{l-1}) \},
$$

where each $\text{row}(x_k)$ is obtained by the right-cycle traversal described above.

These rows together cover every item exactly once, because Algorithm $X$ only terminates at $X2$ when no uncovered items remain.

Thus, the values $x_0, x_1, \ldots, x_{l-1}$ determine the solution simply by treating each $x_k$ as an entry point into a circular row list and reading off all items in that row via right-link traversal.
