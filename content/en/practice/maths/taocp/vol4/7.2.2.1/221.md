**Exercise 221.** [*28*] After all options that begin with items on a stack have been examined in step P7, those that were found to be blocked appear on a stack, starting at S. Explain how to delete them. *Caution:* The problem might become unsolvable when an option goes away.

**Verified:** yes  
**Solve time:** 1m19s  

---

## Setup

Let $S$ be the stack formed at the end of step P7, whose entries are pointers $p$ to options that have been determined to be blocked. Each such $p$ is the first node of an option cycle as described in (11), so the nodes of that option are encountered by iterating $q \leftarrow p + 1$ through spacers until returning to $p$, as in (13).

To delete an option means to remove every node of that option from all vertical item lists and to update the corresponding item list lengths $\mathrm{LEN}(\cdot)$ accordingly. In the terminology of (13), this is exactly the operation performed by $\mathrm{hide}(p)$.

The task is to describe how all options on $S$ are deleted, and how failure is detected if this deletion causes some item to lose all its options.

## Solution

Processing of the stack $S$ is performed by repeated removal of its top element. Let $p$ be the option pointer popped from $S$.

For each such $p$, the deletion of the corresponding option is carried out by executing $\mathrm{hide}(p)$ as defined in (13). This operation traverses the nodes of the option cycle starting at $p$, skips spacer nodes, and for every node $q$ with $\mathrm{TOP}(q) > 0$ removes $q$ from its vertical item list by executing

$$
\mathrm{DLINK}(u) \leftarrow d, \qquad \mathrm{ULINK}(d) \leftarrow u,
$$

where $u = \mathrm{ULINK}(q)$ and $d = \mathrm{DLINK}(q)$, and then decreases $\mathrm{LEN}(x)$ for $x = \mathrm{TOP}(q)$ by $1$. The traversal continues until the cycle returns to $p$.

After each execution of $\mathrm{hide}(p)$, every item header $i$ affected by nodes of that option has its value $\mathrm{LEN}(i)$ reduced by the number of occurrences removed from its list.

Immediately after completing $\mathrm{hide}(p)$, each item header $i$ is inspected indirectly through its updated value $\mathrm{LEN}(i)$. If some item header satisfies $\mathrm{LEN}(i) = 0$, then no active option remains that covers item $i$. In this case the current configuration cannot lead to a solution, so the deletion process terminates and the current branch of the search is declared unsuccessful.

The process continues by repeating the same operation for the next element of $S$ until $S$ becomes empty.

## Verification

Each blocked option appears exactly once on $S$, so each option pointer $p$ is subjected to exactly one application of $\mathrm{hide}(p)$. The definition of $\mathrm{hide}(p)$ removes precisely the nodes belonging to that option cycle and no others, since the traversal is restricted to the cyclic sequence starting at $p$ and only nodes with $\mathrm{TOP}(q) > 0$ are linked into vertical lists.

For any item header $x$, each removal of a node $q$ with $\mathrm{TOP}(q) = x$ causes exactly one decrement $\mathrm{LEN}(x) \leftarrow \mathrm{LEN}(x) - 1$. Hence $\mathrm{LEN}(x)$ remains equal to the number of active options still covering item $x$ after all deletions.

Therefore $\mathrm{LEN}(x) = 0$ holds if and only if every option containing item $x$ has been removed, which is exactly the condition for infeasibility of the residual exact cover instance.

## Notes

The procedure is structurally identical to applying $\mathrm{hide}(p)$ to each blocked option in sequence, but organized via the stack $S$ produced by step P7. The warning about unsolvability corresponds to detecting a zero-length item list during the sequence of deletions, since such a situation eliminates all remaining possibilities for covering that item.
