### Corrected Solution

We construct the initial memory contents for the exact cover representation used by Algorithm X (Dancing Links), following the structure of Table 1. The input consists of:

- A first line listing item names.
- Subsequent lines, one per option, listing the items contained in that option.

Let the items be processed in input order, and let the options be processed sequentially.

We build three structures simultaneously:

1. A circular doubly linked list of column (item) headers via $\mathrm{LLINK}, \mathrm{RLINK}$.
2. For each column, a circular doubly linked vertical list of nodes via $\mathrm{ULINK}, \mathrm{DLINK}$.
3. For each option, a circular doubly linked row of nodes via $\mathrm{LEFT}, \mathrm{RIGHT}$.

No spacer nodes are required for Algorithm X; they are not part of the DLX data structure.

## 1. Creation of item header nodes

We first read the first line and create one header node per item name.

Create a special root header node $0$. Let the item header nodes be $1,2,\dots,n$ in the order of appearance in the input line (this is a construction order, not a memory-address assumption).

For each item header node $i$:

- $\mathrm{NAME}(i)$ is set to the corresponding item name.
- $\mathrm{LEN}(i) \gets 0$.
- Initialize vertical links:

$$
\mathrm{ULINK}(i) \gets i, \quad \mathrm{DLINK}(i) \gets i.
$$

### Horizontal header list

We maintain a pointer $p$ to the most recently created header.

Set:

$$
\mathrm{LLINK}(0) \gets 0, \quad \mathrm{RLINK}(0) \gets 0.
$$

For each item header $i = 1,2,\dots,n$:

- Insert $i$ to the right of $p$:

$$
\mathrm{RLINK}(p) \gets i, \quad \mathrm{LLINK}(i) \gets p.
$$
- Temporarily set:

$$
\mathrm{RLINK}(i) \gets 0, \quad \mathrm{LLINK}(0) \gets i.
$$
- Update $p \gets i$.

This yields a circular doubly linked list:

$$
0 \leftrightarrow 1 \leftrightarrow 2 \leftrightarrow \cdots \leftrightarrow n \leftrightarrow 0.
$$

This corrects the earlier invalid assumption that headers are stored at consecutive memory addresses.

## 2. Mapping item names to headers

Since input items are given by name, we maintain a dictionary:

$$
\text{index}(\text{name}) = \text{pointer to corresponding header node}.
$$

This mapping is built during header creation and used to locate the correct column for each option entry.

## 3. Processing options and constructing nodes

We now process each option line. Suppose option $k$ contains items:

$$
i_{k,1}, i_{k,2}, \dots, i_{k,t_k}.
$$

We allocate one node per incidence (item, option pair). Each node $x$ stores:

- $\mathrm{TOP}(x)$: pointer to its column header.

We also maintain row pointers:

- $\mathrm{rowStart}_k$: first node of option $k$.
- $\mathrm{prev}_k$: most recently created node in this option.

Initialize:

$$
\mathrm{rowStart}_k \text{ undefined initially}, \quad \mathrm{prev}_k \text{ undefined}.
$$

## 4. Construction of vertical column lists

For each item $i_{k,s}$:

Let $c = \text{index}(i_{k,s})$ be its column header.

Create node $x$ and set:

$$
\mathrm{TOP}(x) \gets c.
$$

### Insert into column $c$

We insert $x$ at the bottom of the circular column list:

Let $u = \mathrm{ULINK}(c)$ (last node in column).

$$
\mathrm{DLINK}(u) \gets x, \quad \mathrm{ULINK}(x) \gets u,
$$

$$
\mathrm{DLINK}(x) \gets c, \quad \mathrm{ULINK}(c) \gets x.
$$

Increment:

$$
\mathrm{LEN}(c) \gets \mathrm{LEN}(c) + 1.
$$

This preserves the circular vertical structure required by DLX.

## 5. Construction of horizontal row lists (critical correction)

Each option must form a circular doubly linked list using $\mathrm{LEFT}, \mathrm{RIGHT}$.

For the first node $x$ in option $k$:

- Set:

$$
\mathrm{rowStart}_k \gets x, \quad \mathrm{prev}_k \gets x
$$
- Initialize:

$$
\mathrm{LEFT}(x) \gets x, \quad \mathrm{RIGHT}(x) \gets x.
$$

For each subsequent node $x$ in the same option:

Let $p = \mathrm{prev}_k$.

Insert $x$ to the right of $p$:

$$
\mathrm{RIGHT}(p) \gets x, \quad \mathrm{LEFT}(x) \gets p.
$$

Close temporarily:

$$
\mathrm{RIGHT}(x) \gets \mathrm{rowStart}_k, \quad \mathrm{LEFT}(\mathrm{rowStart}_k) \gets x.
$$

Update:

$$
\mathrm{prev}_k \gets x.
$$

After processing all nodes in option $k$, the row is circular:

$$
x_1 \leftrightarrow x_2 \leftrightarrow \cdots \leftrightarrow x_{t_k} \leftrightarrow x_1.
$$

This corrects the major omission in the previous solution.

## 6. No spacer nodes

The previous solution introduced spacer nodes linking options in memory. This is not part of Algorithm X or Knuth’s DLX structure.

All required structure is fully captured by:

- header list (RLINK, LLINK),
- column lists (ULINK, DLINK),
- row lists (LEFT, RIGHT).

Memory order is irrelevant to correctness.

## 7. Final correctness statement

The resulting structure satisfies:

1. Each column header participates in a circular doubly linked list of headers.
2. Each column is a circular doubly linked vertical list including its header.
3. Each option is a circular doubly linked horizontal list of nodes.
4. Each node belongs to exactly one column and one row.
5. All links are symmetric and locally consistent, enabling correct cover and uncover operations in Algorithm X.

The construction uses exactly one node per incidence of an item in an option, plus one header node per item, matching the DLX representation requirements.

This completes the correct initialization algorithm. ∎
