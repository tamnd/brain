**Exercise 220.** [*28*] Step P5 of Algorithm P needs to emulate step C5 of Algorithm C; to see if some primary item will lose all of its options. Describe in detail what needs to be done.

**Verified:** yes  
**Solve time:** 1m11s  

---

## Setup

Algorithm C step C5 halts the search when some primary item has no remaining active options. In the dancing-links representation of Algorithm P, each item $i$ is represented by a header node whose vertical list consists of all active option-nodes $p$ with $\text{TOP}(p)=i$. The list is empty exactly when $\text{DLINK}(i)=i$ (equivalently $\text{ULINK}(i)=i$), or equivalently when $\text{LEN}(i)=0$ in the header fields as described in Table 1.

Step P5 must detect this situation after each modification produced by $\text{cover}(i)$ and its calls to $\text{hide}(p)$ in (12) and (13).

## Solution

During execution of $\text{cover}(i)$, the procedure $\text{hide}(p)$ removes every node in option $p$ from all vertical lists of items $x=\text{TOP}(q)$ appearing in that option. For each such item header $x>0$, the operation

$\text{DLINK}(u)\leftarrow d,\quad \text{ULINK}(d)\leftarrow u$

removes exactly one node from the vertical list of $x$, and the instruction

$\text{LEN}(x)\leftarrow \text{LEN}(x)-1$

decrements the number of active nodes in that list.

Thus, after each execution of $\text{hide}(p)$, the invariant holds that $\text{LEN}(x)$ equals the number of nodes currently in the vertical list of item $x$.

Step C5 in Algorithm C requires detection of any primary item whose remaining option set becomes empty. In the dancing-links structure this occurs precisely when, for some primary item $x$,

$\text{LEN}(x)=0.$

At that moment the vertical list of $x$ contains no nodes, hence no active option includes $x$.

To emulate C5, step P5 must therefore maintain a test during the execution of $\text{hide}(p)$: whenever a decrement

$\text{LEN}(x)\leftarrow \text{LEN}(x)-1$

is performed for a primary item header $x$, the algorithm immediately checks whether $\text{LEN}(x)=0$. If this equality holds for any primary item, then no active option contains that item, and the current branch of the search cannot be extended. The algorithm must then terminate the current recursion level unsuccessfully and begin the corresponding sequence of $\text{uncover}$ operations in reverse order.

An equivalent formulation uses the pointer structure alone. Since a vertical list is circular, it is empty exactly when

$\text{DLINK}(x)=x.$

Hence P5 may alternatively be implemented by testing, after each deletion step in $\text{hide}(p)$, whether any affected primary item header satisfies $\text{DLINK}(x)=x$.

Because each deletion in $\text{hide}(p)$ touches exactly one occurrence of each item in option $p$, the only items whose emptiness status can change during that call are precisely those items $x$ with a node removed in that iteration. Therefore P5 does not require a global scan of all primary items; it suffices to test each such affected header immediately after its corresponding $\text{LEN}(x)$ update.

This reproduces exactly the effect of C5, since both formulations detect and react to the existence of a primary item with no remaining active options.

This completes the required description. ∎

## Verification

Each decrement of $\text{LEN}(x)$ corresponds to removal of exactly one node from the vertical list of item $x$ by (13), so $\text{LEN}(x)=0$ if and only if no nodes remain in that list. The condition $\text{DLINK}(x)=x$ characterizes the same state because the list is circular and its only node in the empty case is the header itself. The check is triggered only on items explicitly modified in $\text{hide}(p)$, and every item whose membership can change during $\text{cover}(i)$ is modified in some call to $\text{hide}(p)$ for an option containing it. Therefore no failure case escapes detection.

## Notes

The implementation choice between $\text{LEN}(x)=0$ and $\text{DLINK}(x)=x$ is equivalent in correctness; maintaining $\text{LEN}(x)$ gives a direct constant-time test, while pointer-based testing avoids an explicit counter but requires dereferencing links. Both realize the same logical condition required by C5.
