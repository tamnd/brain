## Solution

Let $G=(V,E)$ be the underlying graph used in Exercise 7.2.2–78, and let a subgraph be specified by a subset $X \subseteq V$ of chosen vertices together with the induced edges. The algorithm of Exercise 7.2.2–78 constructs subgraphs by backtracking over choices that progressively determine whether vertices belong to $X$, while maintaining the current connected components of the induced subgraph.

We extend this process by maintaining, in addition to the current partial subgraph, the partition of $X$ into connected components under the induced adjacency relation, together with the size of each component. For each component $C$, let $\sigma(C)$ denote its current size, and let $N(C)$ denote the set of vertices not yet decided that are adjacent to at least one vertex of $C$ in $G$.

A component $C$ is called closed when $N(C)=\varnothing$. At the moment a component becomes closed, no future step of the backtracking algorithm can enlarge it, so its final size is fixed and must satisfy the admissibility constraint.

Let $\mathcal{R} \subseteq \mathbb{Z}_{\ge 0}$ be the set of all integers that can be expressed as a sum of elements of $[L..U]$. Since all summands lie in $[L..U]$, the standard unbounded reachability recurrence defines $\mathcal{R}$ uniquely:

$0 \in \mathcal{R}, \qquad s \in \mathcal{R} \;\Rightarrow\; s+t \in \mathcal{R} \text{ for all } t \in [L..U].$

Because every component size is at most $|V|$, it suffices to precompute the predicate $\mathrm{ok}(s)$ for $0 \le s \le |V|$, where $\mathrm{ok}(s)$ holds exactly when $s \in \mathcal{R}$.

During the backtracking construction, a necessary feasibility condition arises from each connected component independently: whenever a component becomes closed, its final size must lie in $\mathcal{R}$. In addition, partial components can already be ruled out when even maximal possible future growth cannot reach any feasible size.

For a component $C$, define

$T(C) = |N(C)|,$

since each vertex in $N(C)$ is the only remaining possibility to join $C$, and no other vertices can later connect to $C$ once they are permanently excluded or assigned elsewhere.

Let

$I(C) = \{ s \mid \sigma(C) \le s \le \sigma(C)+T(C),\ \mathrm{ok}(s)\}.$

If $I(C)=\varnothing$, then no completion of the current partial subgraph can assign a valid final size to $C$, so the search branch is discarded.

The modified algorithm therefore differs from Exercise 7.2.2–78 only by inserting a feasibility test after every update of the component structure.

When a vertex choice is made and components are updated accordingly, the algorithm performs the following checks for each affected component $C$.

If $C$ is closed, the condition $\mathrm{ok}(\sigma(C))$ is verified. If it fails, the construction cannot lead to a valid solution and backtracking occurs.

If $C$ is not closed, the interval test $I(C)\ne \varnothing$ is applied. If it fails, no extension of the current partial construction can complete $C$ to an admissible size, so the branch is eliminated.

Correctness follows from the fact that every connected component in the final subgraph is disjoint from all others and its vertex set can only grow by incorporating vertices from $N(C)$ during the construction. The value of $\sigma(C)$ is monotone nondecreasing, and $N(C)$ is the full set of vertices that can still be added to $C$, so every attainable final size of $C$ lies in the interval $[\sigma(C),\sigma(C)+T(C)]$. The set $\mathcal{R}$ characterizes exactly the admissible final sizes, so any branch violating the stated conditions cannot be extended to a valid solution. Conversely, any valid solution preserves all component-wise feasibility conditions throughout construction, since each final component size lies in $\mathcal{R}$ and each intermediate state satisfies the interval constraint by monotonicity.

This completes the proof. ∎
