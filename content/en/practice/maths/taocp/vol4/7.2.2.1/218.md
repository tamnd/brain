**Exercise 218.** [*20*] Why must $i$ be uncolored, in the definition of "forcing" for Algorithm P?

**Verified:** yes  
**Solve time:** 1m11s  

---

## Solution

In Algorithm P, a variable $i$ is said to be _forced_ when the current state of the partial construction leaves exactly one admissible choice for $i$, so that the algorithm may assign the corresponding value without branching.

The requirement that $i$ be uncolored is necessary because the definition of forcing is intended to apply only to variables whose value has not yet been fixed in the current partial solution. If $i$ is already colored, then its value is no longer determined by the remaining admissible choices, since it is determined by the history of earlier assignments.

Formally, let the state of the algorithm encode, for each item $i$, the set $C(i)$ of colors still compatible with the current partial assignment. For an uncolored item $i$, the meaning of forcing is that $|C(i)|=1$, so there exists a unique color $c \in C(i)$, and assigning $i \leftarrow c$ preserves consistency because no alternative assignment remains.

If $i$ is already colored with some value $c_0$, then the invariant is that $c_0$ is fixed independently of the current reduced candidate set. In that situation $C(i)$ is either irrelevant or may even have been altered by deletions performed for other items, and it need not contain $c_0$. Applying the forcing rule to such an $i$ would therefore mix two incompatible notions: the historical assignment $c_0$ and the current residual constraint set $C(i)$.

This leads to a logical failure in the deduction mechanism used by Algorithm P. If forcing were allowed on colored items, then a later reduction could make $C(i)$ empty or singleton in a way that contradicts the already established assignment, causing either spurious backtracking or an attempt to reassign $i$, both of which violate the invariant that colored items remain fixed throughout the search path.

Restricting forcing to uncolored items ensures that every forced step corresponds exactly to the transition from an unassigned variable to a uniquely determined assignment, and no previously fixed value is ever reconsidered or revalidated against a changing constraint structure.

This completes the proof. ∎
