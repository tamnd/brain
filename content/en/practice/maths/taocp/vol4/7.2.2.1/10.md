**Exercise 10.** [*20*] *[20]* In some applications the MRV heuristic of exercise 9 leads the search astray, because certain primary items have short lists yet convey little information about desirable choices. Modify answer 9 so that an item $p$ whose name does not begin with the character '#' will be chosen only if LEN$(p) \le 1$ or no other choices exist. (This tactic is called the "sharp preference" heuristic.)

## Solution

Let $i$ range over the currently active items in step X3 of Algorithm X. For each item $p$, define its admissibility status by

$$
\text{adm}(p) =
\begin{cases}
1, & \text{if NAME}(p) \text{ begins with } \# \\
1, & \text{if NAME}(p) \text{ does not begin with } \# \text{ and LEN}(p) \le 1 \\
0, & \text{otherwise.}
\end{cases}
$$

Let $S_1$ be the set of all items $p$ such that $\text{adm}(p)=1$. The modified branching rule replaces the original MRV selection in step X3 as follows.

If $S_1 \ne \emptyset$, choose an item $i \in S_1$ such that LEN$(i)$ is minimum over all $p \in S_1$. If several items in $S_1$ achieve this minimum value, select among them according to the same tie-breaking rule used in exercise 9, preserving the original ordering criterion of Algorithm X.

If $S_1 = \emptyset$, then no item satisfies the sharp preference constraint. In this case, discard the admissibility restriction and choose an item $i$ among all active items such that LEN$(i)$ is minimum, again using the same tie-breaking rule as in exercise 9 when several items achieve this minimum.

This modification enforces that any item $p$ whose NAME does not begin with $#$ and satisfies LEN$(p) > 1$ is excluded from consideration whenever at least one admissible item exists, while preserving the original MRV behavior whenever no admissible item is available. This completes the modified specification of step X3. ∎
