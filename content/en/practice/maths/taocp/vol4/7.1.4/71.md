## Solution

Let Algorithm S denote the BDD construction procedure described in Section 7.1.4, which recursively decomposes a Boolean function by Shannon expansion in a fixed variable order and constructs ordered, reduced binary decision diagrams with binary branching at each node.

To extend Algorithm S to ternary operators, the underlying model is replaced by a 3-valued decision structure in which each variable $x_k$ takes values in ${0,1,2}$ and each internal node has three outgoing edges corresponding to these values.

The truth table of a function $f(x_1,\dots,x_n)$ is replaced by a ternary truth table of length $3^n$, ordered lexicographically by $(x_1,\dots,x_n)$. For a fixed variable $x_k$, every table $\tau$ of order $n-k+1$ decomposes uniquely as

$$
\tau = \tau_0 \tau_1 \tau_2,
$$

where $\tau_i$ is the subtable obtained by fixing $x_k = i$ and letting $(x_{k+1},\dots,x_n)$ vary.

Algorithm S is modified by replacing each binary decomposition step with this ternary decomposition. Each node corresponding to a subfunction $f(x_k,\dots,x_n)$ is represented as a triple-branch node labeled by $k$, with fields

$$
V,\quad LO_0,\quad LO_1,\quad LO_2,
$$

where $LO_i$ points to the node representing the subfunction $f(x_k=i,x_{k+1},\dots,x_n)$.

The ordering constraint is preserved by requiring that if a pointer goes from a node labeled $i$ to a node labeled $j$, then $i < j$, so that variables are tested in strictly increasing index order along every path.

Reduction rules are extended from the binary case. A node is eliminated whenever all three successors are identical, since then the node does not depend on $x_k$. Two nodes are merged whenever their quadruples $(V,LO_0,LO_1,LO_2)$ are identical, ensuring uniqueness of representation. Accessibility from the root is preserved by construction since nodes are created only during recursive decomposition of reachable subtables.

The construction procedure proceeds exactly as in Algorithm S: starting from the full truth table, at stage $k$ each distinct subtable of order $n-k+1$ is examined. If it depends on $x_k$, it generates a new node labeled $k$ whose three children are the nodes corresponding to its three subtables of order $n-k$. If it does not depend on $x_k$, it is represented directly by the already existing node for its common subtable, so no new node is created.

Correctness follows from the fact that every subfunction is uniquely determined by its decomposition into three smaller subfunctions, and that reduction and sharing preserve uniqueness of representation in the same way as in the binary case. The resulting structure is a directed acyclic graph in which each non-sink node has exactly three outgoing edges and variables are tested in fixed order, so it is a ternary ordered reduced decision diagram.

This completes the construction of the ternary extension of Algorithm S. ∎
