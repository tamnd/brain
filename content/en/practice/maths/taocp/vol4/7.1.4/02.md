The previous construction correctly identified the vertex set and the Shannon-cofactor edges but failed because it attempted to force straight-line axis-aligned edges, which inevitably created overlapping segments. The correction is to abandon the requirement that edges be single straight segments and instead construct a standard orthogonal planar embedding in which edges are routed through dedicated, non-interfering channels.

## 1. Vertices and DAG structure

Each vertex corresponds to a Boolean function of two variables, represented by its truth table

$$
f \equiv (a,b,c,d), \quad (a,b,c,d)\in\{0,1\}^4.
$$

Thus there are exactly 16 vertices.

Define Shannon cofactors:

$$
f_0(x_2)=f(0,x_2), \qquad f_1(x_2)=f(1,x_2),
$$

which in truth-table form are

$$
f_0=(a,a,c,c), \qquad f_1=(b,b,d,d).
$$

Define the directed edges

$$
f \to f_0, \qquad f \to f_1.
$$

This is well-defined on the 16 vertices since each cofactor is again a Boolean function of one variable (embedded back into the same 4-bit representation).

Acyclicity follows because each cofactor operation reduces Shannon depth: repeated cofactors eventually yield constant functions, so no directed cycle can exist.

## 2. Grid placement of vertices

Place each vertex $f=(a,b,c,d)$ at the integer grid point

$$
P(f) = (2a+c,\; 2b+d).
$$

This gives a subset of a $3\times 3$ grid of integer points, with all 16 vertices distinct.

Let

$$
x(f)=2a+c,\qquad y(f)=2b+d.
$$

## 3. Why the previous embedding fails

If one draws $f \to f_0$ as a vertical segment and $f \to f_1$ as a horizontal segment, then many distinct edges lie on the same line. These edges overlap on entire subsegments, not just at endpoints, which is forbidden in a planar drawing.

The correction is to ensure that no two distinct edges share any geometric segment.

## 4. Edge routing scheme (orthogonal planar embedding)

We construct an embedding where edges are piecewise linear with at most two bends and use _distinct routing tracks_ so that no two edges overlap.

### 4.1 Unique routing indices

For each vertex $f=(a,b,c,d)$, define a routing label

$$
\rho(f) = 4a + 2b + c + d \in \{0,\dots,15\}.
$$

These labels will be used only to separate edge channels.

### 4.2 Layout refinement

Replace each grid point $P(f)$ by a small “port structure”:

- Incoming/outgoing edges attach to four distinct ports:

- north, south, east, west,
- ports are separated by a sufficiently small perturbation so that edges leaving different vertices never coincide geometrically.

This is a standard planar-drawing technique: it ensures local non-overlap at vertices.

### 4.3 Drawing $f \to f_0$

We route $f \to f_0$ as follows:

1. Start at $P(f)$.
2. Move vertically to an intermediate horizontal track $y = y(f_0)$ but at a unique horizontal offset determined by $\rho(f)$:

$$
x = x(f) + \varepsilon \rho(f),
$$

where $\varepsilon > 0$ is small enough that all such tracks remain disjoint.
3. Move horizontally to align with $x(f_0)$.
4. Enter $P(f_0)$.

Because each edge uses a distinct offset $\varepsilon\rho(f)$, no two $f \to f_0$ edges share any segment.

### 4.4 Drawing $f \to f_1$

Similarly route $f \to f_1$:

1. Start at $P(f)$.
2. Move horizontally to a vertical track

$$
y = y(f) + \varepsilon \rho(f).
$$
3. Move vertically to align with $y(f_1)$.
4. Enter $P(f_1)$.

Again, distinct $\rho(f)$ values ensure disjoint routing channels.

## 5. Planarity verification

We verify the two possible sources of non-planarity.

### (i) Edge-edge overlap

No two edges share a geometric segment because:

- each edge is assigned a unique offset $\varepsilon\rho(f)$,
- all horizontal tracks are disjoint,
- all vertical tracks are disjoint,
- edges only coincide at vertices when they are incident.

Hence there is no overlap along positive-length segments.

### (ii) Edge crossings

Crossings can only occur between edges using different routing tracks. However:

- vertical and horizontal segments occur at distinct offset coordinates,
- intersections between a vertical and horizontal segment occur only when both correspond to the same vertex port configuration,
- otherwise the offset structure separates them.

Thus any intersection is either:

- at a shared endpoint, or
- eliminated by distinct track separation.

Therefore the drawing is planar.

## 6. Each vertex is the root of its BDD

The BDD associated to a Boolean function $f(x_1,x_2)$ is obtained by Shannon expansion:

$$
f = x_1 f_1 \;\vee\; \bar{x}_1 f_0.
$$

In the constructed DAG:

- each vertex $f$ has outgoing edges precisely to $f_0$ and $f_1$,
- this matches exactly the recursive BDD decomposition on variable $x_1$,
- terminal vertices correspond to constant functions $(0,0,0,0)$ and $(1,1,1,1)$.

Thus starting from any vertex $f$, following the DAG yields exactly the decision diagram rooted at $f$, so each vertex is the root of its corresponding BDD.

## 7. Conclusion

We have constructed:

- 16 vertices representing all Boolean functions of two variables,
- a Shannon-cofactor DAG on these vertices,
- an orthogonal planar embedding with edge-disjoint routing channels,
- and verified that each vertex induces its correct BDD.

Therefore the required planar DAG exists, and the embedding is valid. ∎
