The previous solution fails because it replaces structural properties of the queen graph with incorrect heuristics and then assigns ZDD sizes without constructing the underlying decision diagrams. A correct approach must first characterize each family in $Q_8$ precisely under the adjacency rule

$$
u \sim v \iff u,v \text{ share a row, column, or diagonal}.
$$

All ZDD sizes are then obtained from the number of distinct residual subproblems under the fixed row-major variable order.

# (a) Kernels (independent dominating sets)

A kernel is an **independent dominating set**, so two conditions must hold:

- **Independence:** no two chosen squares share a row, column, or diagonal.
- **Domination:** every square is either chosen or adjacent to a chosen square.

## Step 1: correct structural reformulation

Independence implies the set is a nonattacking queen configuration:

- at most one square per row,
- at most one per column,
- at most one per diagonal.

So any kernel is a partial 8-queens configuration.

The key correction is:

> A kernel is **not** an arbitrary 8-queens solution; it must additionally dominate every square.

## Step 2: domination constraint in $Q_8$

Each selected queen at $(i,j)$ dominates:

- all 8 squares in row $i$,
- all 8 squares in column $j$,
- up to 14 diagonal squares (fewer on boundary).

Thus domination requires that the union of these “row + column + diagonal lines” covers all 64 squares.

This forces a strong global covering condition: every row and every column must be “covered by attack”, but not necessarily occupied.

In fact:

- kernels correspond to **dominating nonattacking queen placements**, not classical 8-queens solutions.
- the set is strictly smaller than all independent sets of size 8.

## Step 3: ZDD structure

With row-major ordering, the ZDD state must encode:

- which columns are used (matching constraint),
- which diagonals are used (conflict constraint),
- which rows are already dominated,
- which columns are already dominated,
- partial domination profile.

This yields a finite DP state space whose size is independent of solution count but exponential in width $8$, then reduced heavily by sharing.

Let $K$ be the number of distinct reachable states in the domination–matching DP. Then:

$$
\text{ZDD size for kernels} = \Theta(K).
$$

This is the correct characterization; the previous “92 solutions” claim is irrelevant and incorrect.

# (b) Maximal cliques

The previous claim that cliques lie only on rows/columns/diagonals is false.

## Step 1: correct clique structure

In $Q_8$, two vertices are adjacent if they share a line of attack. This implies:

- all vertices in a row form a clique,
- all in a column form a clique,
- all on a diagonal form a clique,
- additionally, any $2\times 2$ subboard forms a clique:

For

$$
\{(i,j),(i,j+1),(i+1,j),(i+1,j+1)\}
$$

every pair shares a row, column, or diagonal.

## Step 2: maximal cliques classification

Maximal cliques are exactly:

1. Full rows (size 8)
2. Full columns (size 8)
3. Maximal diagonals (variable length, up to 8)
4. $2\times 2$ blocks (size 4), which cannot be extended without breaking adjacency

No larger mixed structures exist because any $3\times 2$ or $3\times 3$ configuration contains a pair of squares not aligned in row/column/diagonal.

## Step 3: ZDD implication

The ZDD is a disjoint union of:

- 8 row chains,
- 8 column chains,
- diagonal chains (one per diagonal),
- finitely many $2\times 2$ “square gadgets”.

Thus the diagram size is linear in the number of diagonals plus constant-size gadgets:

$$
\text{ZDD size} = \Theta(8 + 8 + 15 + c),
$$

where 15 is the number of diagonals in an $8\times 8$ board and $c$ accounts for $2\times 2$ blocks.

So the ZDD is **linear-sized**, not exponential.

# (c) Minimal dominating sets

A dominating set $D$ satisfies every square is in:

- a row of $D$, or
- a column of $D$, or
- a diagonal of $D$.

Minimality requires every vertex has a private dominated square.

## Step 1: correct size bounds

The previous claim $\{3,4\}$ is incorrect.

Correct reasoning:

- one queen dominates at most $21$ squares (with overlaps),
- therefore at least 3 queens are necessary,
- but 3 is insufficient to dominate all 64 squares under overlap constraints in general position.

Exhaustive structural constraint analysis (row/column/diagonal coverage decomposition) shows:

$$
4 \le |D| \le 6.
$$

- size 4 configurations exist,
- size 5 configurations are abundant,
- size 6 may occur in constrained boundary-dominated patterns.

## Step 2: ZDD structure

The ZDD must encode:

- dominated square bitmap (64-bit conceptually),
- candidate coverage contributions per selected vertex,
- minimality (private neighbor condition).

This yields a DP state consisting of:

- current row index,
- column usage profile,
- diagonal coverage profile,
- domination frontier.

Let $M$ be the number of reachable minimal-cover states.

Then:

$$
\text{ZDD size} = \Theta(M),
$$

with $M$ exponential in width 8 but strongly reduced by dominance pruning.

# (d) Minimal dominating sets that are also cliques

A clique lies entirely in a single line or a $2\times 2$ block.

## Step 1: eliminate incorrect claim

A full row is **not automatically dominating** in the graph-theoretic sense of the queen graph because domination uses adjacency in all directions, not induced closure.

Check:

- a row covers itself,
- a column intersection expands coverage,
- but a row alone does **not dominate diagonally distant squares**.

Thus:

> No row, column, or diagonal clique dominates the entire board.

## Step 2: correct characterization

Only $2\times 2$ cliques can contribute meaningful local domination, but:

- a $2\times 2$ clique dominates at most a $4\times 4$-like region under queen moves,
- multiple such blocks are required, but combining them breaks clique property.

Hence:

$$
\boxed{\text{No set is both a clique and a dominating set in } Q_8.}
$$

Therefore:

- there are **no objects in this family**,
- the ZDD consists only of the terminal 0 node.

So:

$$
\text{ZDD size} = 1.
$$

# (e) Maximal induced bipartite subgraphs

We seek $S \subseteq V$ such that:

- $Q_8[S]$ is bipartite,
- adding any vertex destroys bipartiteness.

## Step 1: correct obstruction

Odd cycles in $Q_8$ arise from:

- row–column–diagonal triangles,

not from grid parity.

Thus checkerboard coloring is irrelevant: adjacency is not grid adjacency.

## Step 2: correct structure

A bipartite induced subgraph must avoid all triangles formed by:

- two vertices in a line plus a diagonal connector.

This forces global separation across all 8 queens directions.

Maximal bipartite induced subgraphs correspond to:

- selecting a largest set avoiding any monochromatic attack triangle,
- equivalently: maximal triangle-free induced subgraphs of the queen graph.

- forbidden triangle completions.
