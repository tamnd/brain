## Solution

Let Algorithm R produce a reduced ordered BDD $G$ whose nodes are the usual triples $(V,LO,HI)$ together with sink nodes $\bot,\top$. Let $r$ be the root. The goal is to replace the AVAIL-based storage discipline by a fresh sequential representation

$I_{s-1}, I_{s-2}, \ldots, I_0$

where each instruction has the compact form $(\bar v_k ? \ell_k : h_k)$, meaning node $k$ has variable index $v_k$ and outgoing edges to nodes $\ell_k$ and $h_k$, with $\ell_k,h_k<k$, while $\bot$ and $\top$ are represented as fixed terminal indices.

Define a partial map $\rho$ from nodes of $G$ to integers by constructing it in a reverse topological order forced by the ordering constraint of BDDs. Since every arc goes from $x_i$ to $x_j$ with $i<j$, every directed path strictly increases variable indices, hence the graph is acyclic and admits a numbering where children always receive smaller indices than their parents.

Construct $\rho$ recursively on reachable nodes from the root $r$ as follows. For each sink node, assign fixed indices $\rho(\bot)=0$ and $\rho(\top)=1$. For any branch node $u$ with label $V(u)=k$, define first the values $\rho(LO(u))$ and $\rho(HI(u))$, and only then assign a fresh index to $u$.

To make this definition executable without repetition, perform a depth-first traversal with memoization. For each visited node $u$, if $\rho(u)$ is already defined, return it. Otherwise compute recursively the images of its two successors, then assign $\rho(u)$ as the next unused integer.

After traversal, let $s$ be the number of distinct reachable nodes including sinks. Define the array $I[0],\ldots,I[s-1]$ by setting

$I_k = (V(u_k), \rho(LO(u_k)), \rho(HI(u_k)))$

where $u_k$ is the unique node such that $\rho(u_k)=k$.

The AVAIL stack in Algorithm R serves only to recycle discarded nodes during intermediate reductions. In this modified construction no node is physically deleted, since reduction is enforced by the uniqueness condition of BDD nodes before numbering. Consequently, AVAIL becomes unnecessary: all reachable nodes are retained until final renumbering, after which only the compact array $I$ is kept.

Correctness follows from two structural invariants. First, every BDD path respects increasing variable indices, so the recursive numbering never requires a child index that has not already been defined when the parent is assigned. Second, since Algorithm R ensures reduction, no two distinct nodes have identical triples $(V,LO,HI)$, hence the traversal encounters each reachable subfunction exactly once and $\rho$ is well defined.

The resulting representation is a BDD because each node retains its variable label and pointers to lower-index nodes, preserving orderedness. It is reduced because no duplication is introduced during renumbering. It is consecutive because every reachable node is assigned exactly one integer in ${0,\ldots,s-1}$ and stored in a single array.

This completes the proof. ∎
