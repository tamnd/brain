[Section 7.1.4: Binary Decision Diagrams](../)

**Exercise 83.** [*M20*] [M20] Prove that if f and g both have reference count 1, we needn’t consult the
memo cache when computing AND(f, g) by (55).

**Verified:** yes  
**Solve time:** 1m15s  

---

## Solution

Let Algorithm (55) compute $\operatorname{AND}(f,g)$ by the standard recursive decomposition on BDD nodes, where each call on a pair $(u,v)$ first tests terminal cases, then consults a memo cache for a previously computed value, and otherwise recurses on the low and high successors and stores the result.

Assume that the root nodes representing $f$ and $g$ both have $\operatorname{REF} = 1$.

The meaning of the $\operatorname{REF}$ field in a BDD base is that each node is referenced by exactly that many parent pointers from other nodes or from external roots. Hence $\operatorname{REF}(f)=1$ and $\operatorname{REF}(g)=1$ imply that no node reachable from either $f$ or $g$ can have more than one incoming edge from within the reachable subgraph, since any such additional incoming edge would increase the reference count of that node beyond $1$. Therefore the subgraphs rooted at $f$ and at $g$ are trees rather than dags, and every node in each subgraph is reached by a unique path from its root.

Consider the execution of (55) on input $(f,g)$. Every recursive call is of the form $\operatorname{AND}(f',g')$, where $f'$ is a node in the tree rooted at $f$ and $g'$ is a node in the tree rooted at $g$. Because each node in each tree has a unique parent chain, each pair $(f',g')$ determines a unique pair of root-to-node paths in the two trees.

Suppose, for contradiction, that some pair $(u,v)$ is encountered twice during the recursion. Then there are two distinct call paths producing $(u,v)$. Projecting these call paths onto the first component yields two distinct paths in the tree of $f$ that end at $u$, and similarly two distinct paths in the tree of $g$ that end at $v$. This contradicts the fact that both structures are trees, since a node in a tree has a unique path from the root. Hence no pair $(f',g')$ can arise more than once.

Since the memo cache in (55) is only used to avoid recomputing results for previously encountered pairs, and no pair is ever repeated, every cache lookup would fail and every cache insertion would never be reused. Therefore the memo cache is not needed for correctness or efficiency in this case, and all cache consultations may be omitted without changing the computation.

This completes the proof. ∎
