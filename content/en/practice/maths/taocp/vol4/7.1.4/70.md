## Solution

Let $LCOUNT[l]$ denote the number of nodes that Algorithm S allocates at level $l$, and let $b$ be the number of bits used in step S4 to encode the index of a node at that level. The value $\lg LCOUNT[l]$ measures the information-theoretic minimum bit-length needed to distinguish all nodes at level $l$.

If $b = \lceil \lg LCOUNT[l] \rceil$, then

$$
2^b \ge LCOUNT[l].
$$

Hence there are at least as many distinct binary codes as nodes. The indexing scheme used in S4 can assign a unique code to every node at level $l$ without collisions, so the representation remains injective.

If $b = \lfloor \lg LCOUNT[l] \rfloor$, then

$$
2^b \le LCOUNT[l].
$$

If $LCOUNT[l]$ is not an exact power of $2$, then $2^b < LCOUNT[l]$, so there are strictly fewer available codes than nodes. In that case any fixed-width encoding using $b$ bits must identify at least two distinct nodes with the same code, which violates the requirement that each node be uniquely addressable in the BDD representation. The only case where no conflict occurs is when $LCOUNT[l] = 2^b$, equivalently when $\lg LCOUNT[l]$ is an integer.

Algorithm S relies on unique representation of nodes at each level for correctness of pointer structure and for the reduction condition that equal triples $(V,LO,HI)$ correspond to the same node. If collisions are introduced at the level-index encoding stage, distinct nodes may be merged implicitly, which can destroy the intended structure of the BDD and invalidate later comparisons and lookups.

Therefore choosing $b = \lceil \lg LCOUNT[l] \rceil$ is necessary to guarantee sufficient address space for level $l$ in all cases, while $b = \lfloor \lg LCOUNT[l] \rfloor$ is safe only in the degenerate case where $LCOUNT[l]$ is already a power of $2$.

This completes the proof. ∎
