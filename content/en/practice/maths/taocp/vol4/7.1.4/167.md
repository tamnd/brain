**Exercise 167.** [*21*] [21] Design an efficient algorithm that computes a permutation π for which both
B(f π) and B(f π, ¯f π) are minimized, given any read-once function f(x1, . . . , xn).

The previous proof fails at the point where it tries to force OBDD duplication via interleaving. The correct approach is different: optimality is not obtained by arguing that bad orders necessarily duplicate nodes locally, but by proving a global lower bound on the number of distinct cofactors that any OBDD must represent for a read-once function.

## 1. Structure of read-once functions

Let $f(x_1,\dots,x_n)$ be read-once. Then there exists a formula tree $T$ such that

- each leaf is a distinct variable,
- each internal node is a Boolean operation on disjoint variable sets,
- every node $v \in T$ computes a subfunction $f_v$ on variable set $X(v)$.

A key property is:

> For every node $v$, the function $f_v$ is a cofactor of $f$ obtained by assigning all variables outside $X(v)$.

Indeed, since the variables are disjoint across subtrees, fixing all variables not in $X(v)$ reduces $f$ exactly to $f_v$.

## 2. Construction of the permutation

Define a permutation $\pi$ by a depth-first traversal of $T$, ordering children arbitrarily but keeping each subtree contiguous. Equivalently, every set $X(v)$ forms an interval in $\pi$.

This construction is computable in $O(n)$ time.

## 3. Upper bound: size of the OBDD under $\pi$

We show:

$$
B(f^\pi) = |T| + 2.
$$

### Lemma 1 (subtree isolation)

For every node $v \in T$, the variables $X(v)$ appear consecutively in $\pi$. Hence, during evaluation of the OBDD, once computation enters the first variable of $X(v)$, it does not return to variables outside $X(v)$.

### Lemma 2 (exact representation of subfunctions)

We prove by induction on $T$ that under $\pi$, every node $v$ corresponds to exactly one OBDD node computing $f_v$.

- **Leaf case:** immediate; each variable yields one decision node.
- **Internal node:** let $v$ have children $v_1,\dots,v_k$. Since each $X(v_i)$ is an interval disjoint from others, the OBDD evaluates each subfunction $f_{v_i}$ completely before proceeding to the next block. Thus no interleaving occurs inside subcomputations.

Moreover, within a read-once formula under this ordering, two distinct nodes $u \neq v$ cannot induce identical residual functions during evaluation, because their variable sets differ and the structure of evaluation preserves the identity of each subtree function.

Thus each node in $T$ contributes exactly one OBDD node.

Adding the two terminal nodes gives:

$$
B(f^\pi) = |T| + 2.
$$

## 4. Key correction: why optimality is not about “interleaving duplication”

The previous argument attempted to show that non-contiguous orders increase size by forcing repeated occurrences of subfunctions. This is false because:

If $x \notin X(v)$, then $f_v \mid x=0 = f_v \mid x=1$, so no duplication of $f_v$ is forced.

Optimality must therefore be proved by a different method: a lower bound on the number of _distinct cofactors of $f$_ that any OBDD must represent, independent of the variable order.

## 5. Lower bound for arbitrary permutations

Let $\sigma$ be any permutation. We prove:

$$
B(f^\sigma) \ge |T| + 2.
$$

### Lemma 3 (each subtree function appears as a cofactor)

For every node $v \in T$, there exists an assignment $a_v$ to variables outside $X(v)$ such that:

$$
f \mid a_v = f_v.
$$

This holds because variables are read-once and subtrees are independent: fixing all other subtrees to constants reduces the formula exactly to $v$.

Thus each $f_v$ is a cofactor of $f$.

### Lemma 4 (distinctness of all subtree functions)

If $u \neq v$, then $f_u \neq f_v$.

Indeed, their variable sets differ, and since variables are read-once, no substitution of irrelevant variables can make them identical as Boolean functions on their respective domains.

Hence all $f_v$ are pairwise distinct.

### Lemma 5 (OBDD must realize all cofactors)

In any reduced OBDD for $f^\sigma$, every distinct cofactor of $f$ must appear as a node label somewhere in the diagram.

Reason: along the computation path determined by $\sigma$, once all variables outside $X(v)$ have been assigned according to $a_v$, the residual function being computed is exactly $f_v$. Therefore the OBDD must contain a node representing $f_v$, otherwise it could not represent the correct continuation of computation on that branch.

Thus the OBDD must contain at least one node for each $f_v$.

### Lemma 6 (terminal nodes)

The functions constantly $0$ and $1$ must also appear as terminal nodes.

### Conclusion of lower bound

All nodes of $T$ contribute distinct required OBDD nodes, plus two terminals:

$$
B(f^\sigma) \ge |T| + 2.
$$

## 6. Optimality and construction

From Sections 3 and 5:

- For the constructed permutation $\pi$:

$$
B(f^\pi) = |T| + 2.
$$
- For any permutation $\sigma$:

$$
B(f^\sigma) \ge |T| + 2.
$$

Hence $\pi$ is optimal.

## 7. Complementation

For any reduced OBDD, complementing the function swaps the terminal nodes $0$ and $1$ without changing internal structure. Therefore:

$$
B(f^\pi, \overline{f^\pi}) = B(f^\pi).
$$

## Final result

An optimal permutation $\pi$ is obtained by any depth-first traversal ordering of the formula tree $T$. It can be computed in $O(n)$ time and satisfies:

$$
B(f^\pi) = B(f^\pi, \overline{f^\pi}) = |T| + 2,
$$

and no permutation yields a smaller OBDD.
