**Exercise 23.** &#9654; [*M20*] [M20] All paths from the root of the BDD in Fig. 22 to ⊤have exactly eight
solid arcs. Why is this not a coincidence?

**Verified:** yes  
**Solve time:** 2m09s  

---

Let the BDD in Fig. 22 be the standard ordered BDD for the exact-cardinality function

$$
f(x_1,\ldots,x_n)=1 \iff \sum_{j=1}^n x_j = 8,
$$

constructed in the usual layered way used for combinatorial counting problems.

## 1. Structural meaning of the nodes

In this construction each nonterminal node can be viewed as representing a state

$$
(i,k),
$$

meaning:

- $i$: how many variables $x_1,\ldots,x_i$ have been tested,
- $k$: how many of these have been assigned value $1$.

The root is $(0,0)$. The terminal $\top$ is reached only from states with $k=8$ after all variables have been processed.

The ordered nature of the BDD ensures that from any node $(i,k)$, the next decision is on $x_{i+1}$, and:

- the LO arc (value $0$) goes to $(i+1,k)$,
- the solid HI arc (value $1$) goes to $(i+1,k+1)$.

Thus each solid arc increases the counter $k$ by exactly $1$, while each LO arc leaves $k$ unchanged.

## 2. Invariant along every path

Along any root-to-node path, if $h$ denotes the number of solid (HI) arcs taken, then the reached state has the form

$$
(i,h),
$$

because each HI arc contributes exactly one unit to the count of chosen variables.

This is an invariant of the construction and follows immediately from the transition rules above.

## 3. Condition for reaching $\top$

A root-to-$\top$ path corresponds to a full assignment satisfying $f=1$. By definition of the function,

$$
f(x_1,\ldots,x_n)=1 \iff \sum_{j=1}^n x_j = 8,
$$

so any accepting path must reach a state with

$$
k=8
$$

after the final variable is processed.

Since $k$ equals exactly the number of HI arcs on the path, every root-to-$\top$ path must contain exactly eight solid arcs.

## 4. Why this is not a coincidence

The constancy of the number of solid arcs is not a property of particular assignments or of the drawing, but of the way the BDD is constructed:

- the diagram encodes a counting process for the number of selected variables,
- HI arcs implement the increment operation $k \mapsto k+1$,
- acceptance is defined solely by the condition $k=8$.

Therefore every accepting computation must perform exactly eight increments, independent of the order in which variables take value $1$. This forces every root-to-$\top$ path to contain exactly eight solid arcs.

Thus the observed uniformity of eight solid arcs reflects the underlying exact-cardinality structure of the BDD, not a feature specific to individual paths or to the particular instance drawn in Fig. 22. ∎
