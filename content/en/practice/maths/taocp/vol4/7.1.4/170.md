## Solution

A skinny BDD has exactly one branch node for each variable $x_j$, and in each such node exactly one outgoing edge continues to the next branch node while the other goes to a sink. Since the BDD is ordered, the branch nodes appear in the fixed order $1,2,\dots,n$ along every root-to-sink path.

Let the unique branch node for $x_j$ be $k_j$. Write its two outgoing edges as LO and HI. Exactly one of these edges leads to the next node $k_{j+1}$ (for $j<n$), while the other leads to a sink in ${\bot,\top}$. For the last node $k_n$, both outgoing edges go directly to sinks.

For each $j<n$, define a bit $c_j\in{0,1}$ by declaring that the edge labeled $x_j=c_j$ is the continuation edge to $k_{j+1}$. The opposite value $x_j=1-c_j$ sends the evaluation immediately to a sink, denoted $s_j(1-c_j)\in{\bot,\top}$. At $k_n$, define sink values $s_n(0),s_n(1)\in{\bot,\top}$.

Along evaluation of $f(x_1,\dots,x_n)$, the computation proceeds through the sequence $k_1,k_2,\dots$ as long as $x_j=c_j$. If $x_j\neq c_j$ at the first index $j$, the value returned is $s_j(x_j)$. If no such index exists, the value returned is determined at $k_n$ as $s_n(x_n)$.

Thus a skinny Boolean function is completely determined by the data

$$
(c_1,\dots,c_n)\in\{0,1\}^n,\qquad (s_j(0),s_j(1))\in\{\bot,\top\}\ \text{for }1\le j\le n.
$$

## a)

For each of the $n$ indices $j$, the choice of $c_j$ has $2$ possibilities. For each $j$, the pair $(s_j(0),s_j(1))$ has $4$ possibilities in ${\bot,\top}^2$. These choices are independent across all $j$.

Hence the number of skinny Boolean functions equals

$$
2^n\cdot 4^n = 2^{3n}.
$$

$$
\boxed{2^{3n}}
$$

## b)

A function is monotone only if changing any input bit from $0$ to $1$ never decreases the output in the order $\bot<\top$.

If some node $k_j$ has $c_j=0$, then inputs with $x_j=1$ immediately terminate at $s_j(1)$, while inputs with $x_j=0$ proceed further. Fix all later variables to $1$ so that continuation reaches $k_n$ and yields $s_n(1)$. Monotonicity forces $s_j(1)\ge s_n(1)$. If simultaneously later a mismatch occurs at some $k_i$, the same comparison forces consistency of all sink values along the chain. Propagating these constraints backward shows that all sink values must agree and the function must be constant, contradicting dependence on all variables.

Hence every monotone skinny function must satisfy $c_j=1$ for all $j$, since any $c_j=0$ creates a forced decrease when flipping $x_j$ from $0$ to $1$.

With $c_j=1$ for all $j$, the function proceeds through all nodes exactly when all variables are $1$. Any mismatch occurs when some $x_j=0$. Monotonicity forces all mismatch sinks to be $\bot$, otherwise flipping a later variable could increase the output. The final node must satisfy $s_n(1)=\top$ and $s_n(0)=\bot$, since the all-ones input must be maximal and all-zeros minimal.

This yields exactly the conjunction $x_1\land\cdots\land x_n$, and no other assignment is consistent with monotonicity.

$$
\boxed{1}
$$

## c)

Let $a=x_1x_2\cdots x_n$ be the integer whose binary representation is $(x_1\ldots x_n)_2$. The function

$$
f_t(x_1,\dots,x_n) = [a^2 \ge t]
$$

with $t$ odd and $0<t<2^n$ depends on whether the binary square $a^2$ exceeds an odd threshold.

Since $t$ is odd, comparison with $a^2$ is determined without ambiguity from the least significant bit once carry propagation is resolved. The squaring map preserves ordering of binary prefixes in the sense that if two inputs first differ at position $j$, then the contribution of higher-order bits dominates all lower-order contributions in $a^2$ because all cross terms involving positions $>j$ exceed the total contribution of positions $<j$.

Thus the decision $a^2\ge t$ is resolved at the first index $j$ where $x_j$ differs from the critical pattern determined by $t$, and all later bits become irrelevant once this comparison is determined. This produces exactly one branch per variable, and at each node the continuation corresponds to matching the required bit while the opposite branch forces immediate comparison to a fixed threshold outcome. Hence the BDD is a single chain and therefore skinny.

$$
\boxed{\text{$f_t$ is skinny for odd }t}
$$

## d)

The dual of a Boolean function is obtained by interchanging $\bot$ and $\top$ in its BDD representation. Therefore the dual of

$$
f_t(x_1,\dots,x_n) = [a^2 \ge t]
$$

is

$$
f_t^\ast(x_1,\dots,x_n) = [a^2 < t],
$$

equivalently

$$
f_t^\ast = \neg f_t = [a^2 \le t-1].
$$

$$
\boxed{[a^2 < t]\ \text{or equivalently}\ [a^2 \le t-1]}
$$

## e)

A skinny BDD is a chain of $n$ branch nodes. Each root-to-sink path fixes a unique earliest index $j$ where the computation terminates. Therefore every assignment either reaches a sink at some $j$ or reaches the terminal node $k_n$.

For a CNF, each root-to-$\bot$ path yields a clause excluding exactly the assignments that follow that path. Since each such path fixes a single variable condition $x_j\neq c_j$ at the termination point, each clause contains exactly the corresponding literal that blocks that failure. Collecting all such clauses produces a CNF whose size is linear in the number of branch nodes.

For a DNF, each root-to-$\top$ path yields a term specifying the sequence of variable choices that avoids all early sink transitions and reaches a $\top$ sink. Each such path fixes exactly one assignment pattern along the chain, so each term is a conjunction of literals $x_j=c_j$ for all variables up to the first $\top$ termination point. The number of such terms is also linear in $n$.

Thus both the CNF and DNF are obtained directly from the single chain structure by listing, respectively, all failure paths and all accepting paths, each of which corresponds to a unique branching position in the BDD.
