**Exercise 139.** [*22*] [22] Generalize the algorithm of exercise 138 so that (i) it computes a common

The original solution fails because it tries to introduce algebraic linearity and artificial weight propagation that are not part of the definition of a profile chart in Exercise 138. The correct approach is to stay entirely within the traversal semantics of QDDs: a profile chart is defined by accumulated local contributions induced by structure-dependent traversal of the diagram, not by function-space operations.

The correct fix is therefore to reconstruct both generalizations purely at the level of traversals of the shared decision structure.

# Correct solution

Let $\mathcal{B}$ be a BDD base with roots $R=\{r_1,\dots,r_m\}$ in a fixed variable order $x_1<\cdots<x_n$. Let $Q$ be the corresponding QDD (shared DAG) containing all nodes reachable from any root.

Assume Exercise 138 defines a profile chart construction as follows:

- Each traversal step through a node $v$ at level $i$ contributes a local quantity $c(v,i)$,
- The profile chart is the sum of all contributions induced by the traversal of the QDD from the given starting root, respecting the variable ordering.

Crucially, this is a traversal-based definition over a directed acyclic structure, not an algebraic functional of Boolean functions.

# (i) Common profile chart for a BDD base

## Correct viewpoint: traversal union, not function combination

The profile chart of a single root $r$ is the result of running the algorithm of Exercise 138 on the QDD starting at $r$, generating a multiset of visited node-level contributions.

For a base $R=\{r_1,\dots,r_m\}$, the correct object is simply the union of these traversals:

$$
P(\mathcal{B}) \;=\; \sum_{r \in R} P(r),
$$

but this is not justified by linearity of Boolean functions. It is justified because the definition of $P(\cdot)$ is itself a sum over traversal contributions, and the base semantics is that each root generates an independent traversal process.

There is no interaction between roots except shared storage in the QDD.

## Single-pass computation without linearity assumptions

We compute the common profile chart directly on the shared QDD without duplicating work as follows.

We run the algorithm of Exercise 138 on the _forest with multiple sources_, implemented by initializing the traversal with all roots:

- Maintain a worklist (or recursion stack) seeded with all $r_i$.
- Each root is processed exactly as in Exercise 138.
- When traversal reaches a node $v$, contributions $c(v,i)$ are generated exactly as prescribed by the original algorithm.

The key point is:

- The algorithm of Exercise 138 is defined on a rooted QDD traversal,
- Extending it to multiple roots means we run the same traversal procedure independently from each root,
- Shared nodes in the DAG are naturally reused structurally, but their contributions are counted once per root-induced traversal, exactly as dictated by the definition.

No “weight propagation” is introduced. No assumption of linearity of Boolean functions is required.

Thus the correct common profile chart is:

$$
P(\mathcal{B}) \;=\; \sum_{i=1}^m P(r_i),
$$

where each $P(r_i)$ is computed by the original algorithm, and sharing is exploited only as implementation-level memoization of node structure, not as algebraic superposition.

# (ii) Restriction to variables $\{x_a,\dots,x_b\}$

## Step 1: semantic meaning of restriction

We do not “eliminate variables algebraically.” Instead, we modify the QDD so that the traversal induced by Exercise 138 only encounters decision points labeled by variables in the interval $[a,b]$.

Variables outside this range act as fixed cofactors determined by their position in the ordered structure and are collapsed structurally.

## Step 2: construction of the restricted QDD

We construct a reduced QDD $Q_{[a,b]}$ by structural elimination:

For every node $v$ labeled by $x_i$:

### Case 1: $i < a$

These variables are above the active interval. Replace $v$ by its Shannon-reduced cofactor:

$$
f_v \;=\; (\neg x_i \wedge f_{v_0}) \lor (x_i \wedge f_{v_1})
$$

but since $x_i$ is not part of the active chart, we recursively collapse $v$ into a node representing the function already resolved with respect to the fixed upper structure.

Operationally:

- repeatedly eliminate nodes labeled $x_1,\dots,x_{a-1}$,
- splice their cofactors upward until only dependence on $x_a$ remains.

### Case 2: $a \le i \le b$

Keep node unchanged.

### Case 3: $i > b$

These variables lie below the active interval. They are not involved in chart construction, so they are collapsed downward into terminal structure exactly as in standard BDD restriction: their contributions appear only in terminal cofactors and do not generate chart entries.

## Step 3: key invariance principle (correct form)

We now justify correctness.

The profile chart algorithm of Exercise 138 depends only on:

- the structure of the QDD over the variables it actually traverses,
- the local node-level contributions induced during traversal.

Eliminating variables outside $[a,b]$ preserves the induced traversal on the active part of the diagram in the following sense:

- Every root-to-sink traversal in the original QDD projects uniquely onto a traversal in $Q_{[a,b]}$,
- The eliminated variables contribute only through cofactors that are absorbed before reaching the active interval,
- No new branching structure over $[a,b]$-variables is created or destroyed by elimination, only compressed.

Thus there is a bijective correspondence between:

- traversal segments contributing to active-variable chart entries in $Q$,
- and traversal segments in $Q_{[a,b]}$.

Therefore the accumulated chart over active variables is preserved.

## Step 4: running the algorithm

We now apply Exercise 138 directly:

1. Construct $Q_{[a,b]}$ by eliminating all nodes labeled outside $[a,b]$,
2. Initialize traversal from all roots $r_1,\dots,r_m$,
3. Run the original profile chart algorithm unchanged,
4. Accumulate contributions only from nodes labeled in $[a,b]$.

# Final result

The generalized profile chart is obtained by:

$$
P_{[a,b]}(\mathcal{B})
\;=\;
\sum_{i=1}^m P_{[a,b]}(r_i),
$$

where $P_{[a,b]}(r_i)$ is computed by running the original Exercise 138 algorithm on the restricted QDD $Q_{[a,b]}$ starting from root $r_i$.

Equivalently:

- build the QDD for the base,
- eliminate variables outside $[a,b]$ by structural cofactoring,
- run the original traversal-based chart algorithm from all roots,
- sum the resulting traversal contributions.

No linearity of Boolean functions is used. No artificial node weights are introduced. The correctness follows solely from preservation of traversal structure under variable elimination and from the fact that the profile chart is defined as a traversal-induced accumulation over the QDD.
