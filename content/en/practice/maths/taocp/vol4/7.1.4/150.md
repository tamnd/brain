The original argument fails at three points: the meaning of the state $g_{v,i}$, the validity of the “no-split” Case 3b, and the cost analysis of restriction. The corrected proof restarts the construction with a precise invariant and replaces the flawed cost reasoning with explicit memoization and charging to the input and output BDD sizes.

## Correct solution

Let $F$ be represented by a reduced ordered BDD $G$ under variable order $\prec$, and let $\prec_\pi$ be the target order

$$
x_{\pi(1)} \prec_\pi x_{\pi(2)} \prec_\pi \cdots \prec_\pi x_{\pi(n)}.
$$

Let $F^\pi$ denote the reduced OBDD of $F$ under $\prec_\pi$. We write $B(F)$ and $B(F^\pi)$ for their numbers of nodes.

## 1. Well-defined state functions

For every node $v$ of the original BDD and every index $i\in\{1,\dots,n+1\}$, define the Boolean function

$$
g_{v,i} \;=\; f_v \big|_{x_{\pi(1)}=a_1,\dots,x_{\pi(i-1)}=a_{i-1}},
$$

where $a_j\in\{0,1\}$ are arbitrary fixed values.

The key point is that in a reduced OBDD, variables not yet tested are independent of earlier choices, so all choices of $a_1,\dots,a_{i-1}$ yield the same reduced subfunction up to equivalence. Hence $g_{v,i}$ is well-defined as a Boolean function.

We will construct the reduced OBDD for $g_{v,1}=F$ in order $\prec_\pi$.

## 2. Memoized construction

We define a procedure

$$
\mathrm{Build}(v,i)
$$

returning the unique node in the target OBDD representing $g_{v,i}$. Memoization is used on pairs $(v,i)$.

Let $x = x_{\pi(i)}$.

We assume a global unique table for nodes $(x,u_0,u_1)$.

## 3. Recursive cases and correctness

### Case 1: terminal nodes

If $v\in\{\bot,\top\}$, return it.

### Case 2: variable already current

If $\operatorname{var}(v)=x$, then

$$
g_{v,i} = x \cdot g_{v_1,i+1} + \bar{x}\cdot g_{v_0,i+1},
$$

so we compute

$$
u_0=\mathrm{Build}(v_0,i+1),\quad u_1=\mathrm{Build}(v_1,i+1),
$$

and return $\mathrm{node}(x,u_0,u_1)$.

### Case 3: variable in original node comes earlier in target order

Assume $\operatorname{var}(v)=y\neq x$.

Let $\mathrm{rank}(y)=\pi^{-1}(y)$.

#### Case 3a: $y \prec_\pi x$, i.e. $\mathrm{rank}(y) < i$

Then $y$ must be tested before $x$ in the target order. Hence

$$
g_{v,i} = y \cdot g_{v_1,i} + \bar{y}\cdot g_{v_0,i}.
$$

Correctness reason: along any assignment, $y$ is encountered before $x_{\pi(i)}$, so Shannon expansion in the target order must split on $y$ first.

Thus we compute

$$
u_0=\mathrm{Build}(v_0,i),\quad u_1=\mathrm{Build}(v_1,i),
$$

and return $\mathrm{node}(y,u_0,u_1)$.

This fixes the previous incorrect “index shift” justification: the index does not change because no target variable has been consumed.

#### Case 3b: $x \prec_\pi y$, i.e. $\mathrm{rank}(y) > i$

Then all occurrences of $x_{\pi(i)}$ must be resolved before reaching $y$. We therefore compute cofactors with respect to $x$:

$$
v_0' = f_v|_{x=0}, \quad v_1' = f_v|_{x=1}.
$$

Then

$$
g_{v,i} = x \cdot g_{v_1',i+1} + \bar{x}\cdot g_{v_0',i+1}.
$$

This is valid because in any path, $x_{\pi(i)}$ precedes $y$, so Shannon expansion at $x$ is forced in the target order.

We compute

$$
u_0=\mathrm{Build}(v_0',i+1),\quad u_1=\mathrm{Build}(v_1',i+1),
$$

and return $\mathrm{node}(x,u_0,u_1)$.

## 4. Correctness

We prove:

**Lemma 1.** $\mathrm{Build}(v,i)$ computes a reduced OBDD for $g_{v,i}$.

Proof is by induction on the pair $(\text{number of variables remaining in }g_{v,i}, \text{structure of }v)$.

- Terminal case is trivial.
- If $\operatorname{var}(v)=x$, correctness follows from Shannon decomposition.
- If $\operatorname{var}(v)=y \prec_\pi x$, then $y$ must be the next variable in the target order among those present in $g_{v,i}$, so Case 3a matches Shannon expansion.
- If $x \prec_\pi y$, then $x$ is the next variable in the target order, so Case 3b performs the correct Shannon split.

Thus each recursive step matches the Shannon expansion under $\prec_\pi$. Reduction is ensured by the unique table. ∎

## 5. Cost of restriction

We eliminate the earlier incorrect claim of linear traversal without memoization.

We explicitly memoize restriction as well.

Define a cache for $(v,x)$ computing $f_v|_{x=b}$. Each pair is evaluated once.

Each original edge $(v,v_0),(v,v_1)$ participates in at most two restriction results per variable, but due to memoization each is processed once per $(v,x)$ pair. Since each node has at most two children, total restriction work over all calls is

$$
O(B(F)).
$$

No exponential recomputation occurs.

## 6. Output-size bound

We now fix the previous flawed injectivity argument.

**Lemma 2.** Each call $\mathrm{Build}(v,i)$ returns a node of the reduced OBDD $F^\pi$, and two calls that produce identical Boolean functions return the same memoized node.

Thus the number of distinct results of $\mathrm{Build}$ is at most $B(F^\pi)$.

**Reason.**

- By Lemma 1, every state computes a Boolean function consistent with Shannon expansion under $\prec_\pi$.
- The unique table ensures that two states producing the same function correspond to the same node.
- Since $F^\pi$ is the canonical reduced OBDD, each distinct function corresponds to exactly one node.

Hence the number of memoized states that produce new nodes is bounded by $B(F^\pi)$. ∎

## 7. Total complexity

We separate costs:

### (1) Restriction cost

Each restriction operation is charged to a pair $(v,x)$, giving

$$
O(B(F)).
$$

### (2) Construction cost

Each distinct memoized state corresponds to a node in the output BDD, hence at most $B(F^\pi)$ states. Each state performs $O(1)$ work aside from recursive calls.

Thus construction cost is

$$
O(B(F^\pi)).
$$

## 8. Final bound

Combining both parts,

$$
T = O(B(F) + B(F^\pi)).
$$

## Conclusion

The corrected algorithm performs variable jumps directly by recursively enforcing Shannon expansion in the target order, with restriction computed via memoization and all node creation tied to canonical subfunctions of the target OBDD. The earlier incorrect state semantics, invalid Case 3b reasoning, and unjustified restriction cost are removed. The resulting method achieves the required bound

$$
O(B(F) + B(F^\pi)).
$$

∎
