## Solution

Let the swap-in-place procedure of Exercise 147 be the standard recursive transformation that, given a reduced ordered BDD and two adjacent levels $j_u$ and $j_v$, replaces every node labeled $j_u$ above level $j_v$ by an equivalent structure in which the variables are interchanged, using recursive calls on the LO and HI children and a memoization table to ensure each reachable node is processed once.

For ZDDs the same structural recursion applies, since both BDDs and ZDDs are ordered rooted DAGs with identical sharing discipline based on a unique table indexed by $(V,LO,HI)$. The only change required is that reduction and terminal propagation differ, so the reconstruction step after swapping levels must enforce the ZDD invariants instead of the BDD invariants.

In a ZDD, a node $k_j$ with variable $x_j$ and children $LO$ and $HI$ represents a family of sets in which the contribution of $x_j$ is excluded via $LO$ and included via $HI$. The defining reduction rule states that if $HI = 0$, then the node is eliminated and replaced by $LO$, since no set in the family contains $x_j$. Together with the uniqueness condition on $(V,LO,HI)$, this is the only reduction needed beyond sharing.

During the swap, consider a recursive call on a node $p$ whose variable lies at one of the two levels being interchanged. The recursion produces two transformed children $p_{LO}'$ and $p_{HI}'$ corresponding to the swapped subproblems. A candidate node $q$ is formed with variable index updated according to the swapped levels and with children $(p_{LO}',p_{HI}')$.

In a BDD swap, $q$ would then be reduced by the condition $q.LO = q.HI$. In a ZDD this condition is not part of the reduction system, so it must be removed entirely. The only structural simplification permitted after node construction is the ZDD deletion rule: if $q.HI = 0$, then $q$ is replaced by $q.LO$.

Thus the swap-in-place algorithm is modified at exactly one structural point: the reduction test applied after node construction. Every occurrence of the BDD rule

$$
LO = HI \;\Rightarrow\; \text{delete node}
$$

is replaced by the ZDD rule

$$
HI = 0 \;\Rightarrow\; \text{delete node}.
$$

All other parts of the algorithm remain unchanged. The ordering invariant is preserved because the swap is applied only between adjacent levels $j_u$ and $j_v$, and recursive calls ensure that children are already transformed before the parent node is rebuilt, so all edges continue to respect increasing variable indices after the interchange.

Memoization keyed by node identity and level pair remains valid because ZDD canonicality still guarantees that each pair $(V,LO,HI)$ determines a unique node, and swapping does not introduce ambiguity in variable indices, only in their placement in the order.

Therefore the swap-in-place algorithm for ZDD bases is obtained from the BDD version by replacing the BDD reduction step with the single ZDD elimination rule $HI = 0 \Rightarrow$ bypass the node, while leaving recursion, memoization, and uniqueness-table insertion unchanged.

This completes the proof. ∎
