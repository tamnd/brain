## Solution

Algorithm S constructs a BDD for a composed function of the form $f \circ g$, using Shannon decomposition and reduction. Step S1 is the terminal and shortcut case handling stage, where trivial outcomes are detected before any recursive construction or node allocation.

If during step S1 the evaluation of $f \circ g$ shows that the resulting function is constant, then both Shannon cofactors with respect to the next variable are identical constants. In that situation no further decomposition is required, since a BDD node would otherwise be introduced whose LO and HI successors are identical, which is forbidden by reduction.

If the constant value is $0$, the result is the unique sink node $\perp$. If the constant value is $1$, the result is the unique sink node $\top$. No new branch node is created, and no recursive calls are issued beyond the detection of constancy.

This handling is consistent with the reduction rule that forbids nodes with equal LO and HI pointers, since a constant function would otherwise induce such a node at every level of attempted expansion. It also ensures that the representation remains minimal, because any non-sink structure would strictly increase $B(f \circ g)$ without adding distinguishable computational behavior.

This completes the explanation of step S1. ∎
