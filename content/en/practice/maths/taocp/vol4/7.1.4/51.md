Let the variables be ordered

$$
x_1 \prec x_2 \prec \cdots \prec x_{2n},
$$

where column $k$ consists of the pair $(x_{2k-1},x_{2k})$. The addition is

$$
(x_{2n-1}\cdots x_3 x_1)_2 + (x_{2n}\cdots x_4 x_2)_2
= (f_{n+1}f_n\cdots f_1)_2.
$$

## 1. Carry formulation and local recurrence

Define the carry $c_k \in \{0,1\}$ entering column $k$, with

$$
c_1 = 0.
$$

For each $k$ the full-adder equation is

$$
x_{2k-1} + x_{2k} + c_k = f_k + 2c_{k+1},
$$

where $f_k \in \{0,1\}$ and $c_{k+1} \in \{0,1\}$ are uniquely determined by $(x_{2k-1},x_{2k},c_k)$.

Define the transition function

$$
\delta(x,y,c) = c' \quad \text{and} \quad \varphi(x,y,c) = f,
$$

where $c',f$ are determined by the above binary addition rule.

## 2. Definition of the BDD base via suffix subfunctions

For $k \in \{1,\dots,n+1\}$ and $c \in \{0,1\}$, define the suffix subfunction

$$
F_{k,c}(x_{2k-1},\dots,x_{2n}) = (f_k,f_{k+1},\dots,f_n,f_{n+1}),
$$

computed by starting with carry $c_k=c$ and applying the recurrence for columns $k,k+1,\dots,n$.

This is the formal notion of a node in the BDD base: two pairs $(k,c)$ and $(k',c')$ represent the same node if they define identical suffix functions.

## 3. Fundamental uniqueness lemma

**Lemma.** The suffix function $F_{k,c}$ depends only on $(k,c)$ and not on the history of the inputs.

**Proof.** We proceed by induction on the number of remaining columns $n-k+1$.

For $k=n+1$, no variables remain. The computation has produced a final carry $c_{n+1}=c$, and there are no further sum bits. Hence the suffix function is the constant function

$$
F_{n+1,c} = (c),
$$

so it depends only on $c$.

Now assume the claim holds for $k+1$. Fix $(k,c)$. The next output bit and carry are

$$
f_k = \varphi(x_{2k-1},x_{2k},c), \quad c_{k+1} = \delta(x_{2k-1},x_{2k},c).
$$

After processing column $k$, the remaining output is $F_{k+1,c_{k+1}}$. By the induction hypothesis, this depends only on $(k+1,c_{k+1})$, which itself depends only on $(x_{2k-1},x_{2k},c)$. Hence the entire suffix function depends only on $(k,c)$, completing the induction. ∎

Thus the distinct BDD base nodes are in one-to-one correspondence with the pairs $(k,c)$.

## 4. Structure of decision nodes under the ordering

The ordering alternates variables within each column:

$$
x_{2k-1} \prec x_{2k} \prec x_{2k+1}.
$$

From a node $(k,c)$, the BDD first tests $x_{2k-1}$. After this test, the next level tests $x_{2k}$, and only then is the carry updated.

More explicitly, for fixed $c$, define the intermediate function after testing $x_{2k-1}=a$:

$$
g_a(x_{2k},c) = (f_k, c_{k+1}) \text{ computed from } (a,x_{2k},c).
$$

This respects the ordered BDD rule because each node depends on exactly one variable, and the “two-variable full-adder” behavior is realized through two successive decision levels, not a single combined gadget.

Thus the structure is a standard ordered BDD with two decision levels per column, not a grouped super-node.

## 5. Terminal nodes

After processing all variables, we reach level $n+1$ with no remaining inputs. At this point the only remaining quantity is the final carry.

Hence:

- Node $(n+1,0)$ represents the constant function $0$,
- Node $(n+1,1)$ represents the constant function $1$.

These are the only terminal nodes in the BDD base, and they correspond exactly to the possible values of the final carry.

This resolves the earlier conflation: the carry is not an external variable but the value of the final output bit $f_{n+1}$, so the terminal subfunctions are genuine constant Boolean functions.

## 6. The BDD base

From the lemma, every subfunction arising from any suffix restriction is uniquely represented by a pair $(k,c)$, where:

- $k \in \{1,\dots,n+1\}$ is the current column index,
- $c \in \{0,1\}$ is the incoming carry.

No other distinctions arise, since different input histories leading to the same carry produce identical future behavior.

Therefore the BDD base consists exactly of:

$$
\{(k,c) \mid 1 \le k \le n+1,\ c \in \{0,1\}\},
$$

together with the interpretation:

- each node $(k,c)$ encodes the suffix addition process starting at column $k$ with carry $c$,
- transitions are given by the full-adder rule under the fixed variable ordering,
- terminal nodes $(n+1,c)$ are constants.

## Conclusion

The BDD base for binary addition under right-to-left interleaved numbering consists precisely of two states per column index, indexed by the carry value, with terminal constants $0$ and $1$. The correctness rests on the inductive fact that the suffix function depends only on the current column and carry, and all transitions respect the ordered single-variable BDD structure.
