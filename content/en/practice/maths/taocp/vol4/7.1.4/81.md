**Exercise 81.** &#9654; [*20*] [20] By modifying (55), explain how to compute f ⊕g in a BDD base.

## Solution

Let $f$ and $g$ be represented by BDD nodes $u$ and $v$. Algorithm (55) for $f \wedge g$ is an instance of the standard APPLY procedure: it evaluates the Boolean operation at sinks and otherwise recurses on cofactors determined by the smallest variable index present at the current nodes, while reusing already computed pairs of subproblems.

To compute $f \oplus g$, the same control structure is used, but the terminal operation is replaced by exclusive-or.

Define a procedure $\operatorname{Apply}_\oplus(u,v)$ that returns the BDD node representing $f \oplus g$.

At sink nodes the Boolean value is determined directly from the operation table of $\oplus$:

$$
0 \oplus 0 = 0,\quad 0 \oplus 1 = 1,\quad 1 \oplus 0 = 1,\quad 1 \oplus 1 = 0.
$$

Thus if $u \in {\bot,\top}$ and $v \in {\bot,\top}$, the result is the corresponding sink node.

If $u$ and $v$ are branch nodes, let $i = V(u)$ and $j = V(v)$ and set $k = \min(i,j)$. The Shannon decomposition with respect to $x_k$ is used exactly as in (55). Three cases occur.

If $i = j = k$, then both functions depend on $x_k$, so both cofactors are combined:

$$
u_0 = \operatorname{LO}(u),\quad u_1 = \operatorname{HI}(u),\quad v_0 = \operatorname{LO}(v),\quad v_1 = \operatorname{HI}(v).
$$

The recursive definition is

$$
\operatorname{Apply}_\oplus(u,v)
=
\operatorname{Node}\!\bigl(k,\ \operatorname{Apply}_\oplus(u_0,v_0),\ \operatorname{Apply}_\oplus(u_1,v_1)\bigr).
$$

If $i < j$, then $u$ splits on $x_k$ while $v$ does not vary with $x_k$, so $v$ is used in both cofactors:

$$
\operatorname{Apply}_\oplus(u,v)
=
\operatorname{Node}\!\bigl(i,\ \operatorname{Apply}_\oplus(u_0,v),\ \operatorname{Apply}_\oplus(u_1,v)\bigr).
$$

If $j < i$, the symmetric rule holds:

$$
\operatorname{Apply}_\oplus(u,v)
=
\operatorname{Node}\!\bigl(j,\ \operatorname{Apply}_\oplus(u,v_0),\ \operatorname{Apply}_\oplus(u,v_1)\bigr).
$$

In all cases the procedure uses the BDD reduction rule from Section 7.1.4: if both recursive results are identical, that node is returned directly; otherwise a unique node $\operatorname{Node}(k,\cdot,\cdot)$ is created or retrieved from the unique table, ensuring no duplicate triples $(V,LO,HI)$ occur.

The memoization mechanism from (55) is retained unchanged, storing computed pairs $(u,v)$ to ensure each subproblem is evaluated once.

Correctness follows from structural induction on the pair $(u,v)$ ordered by BDD depth. At sinks the returned value matches the definition of $\oplus$. For branch nodes, the Shannon decomposition of $f$ and $g$ with respect to $x_k$ implies

$$
(f \oplus g)|_{x_k=0} = f_0 \oplus g_0,\quad (f \oplus g)|_{x_k=1} = f_1 \oplus g_1,
$$

or the corresponding identities when one operand does not depend on $x_k$. The recursive construction therefore produces a BDD whose cofactors agree with those of $f \oplus g$ at every variable, and uniqueness of reduced ordered BDDs implies equivalence of the resulting graph.

This completes the proof. ∎
