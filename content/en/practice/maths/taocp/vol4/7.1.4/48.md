## 1. Hidden weighted bit function and representation

Let $x_1,\ldots,x_n\in\{0,1\}$. Define the prefix sums

$$
s_k=\sum_{i=1}^k x_i,\qquad 1\le k\le n,
$$

so that $0\le s_k\le k$. Let $x_0$ be a fixed constant input (as in Fig. 23, the sink value).

The hidden weighted bit function is

$$
\mathrm{HWB}(x_1,\ldots,x_n)=x_{s_n}.
$$

Thus the computation consists of two logically separate tasks:

1. Compute the integer $s_n$.
2. Use $s_n$ as an address selecting the input $x_{s_n}$.

## 2. Module implementation (Fig. 23 style network)

We construct a chain of modules $G_1,\ldots,G_n$, where each $G_k$ transforms

$$
(s_{k-1}, x_k)\mapsto s_k.
$$

### 2.1 Adder-control module

Each module $G_k$ computes

$$
s_k = s_{k-1} + x_k.
$$

This is implemented as a constant-size BDD module because $x_k\in\{0,1\}$, so the update is a controlled increment:

- if $x_k=0$, pass $s_{k-1}$,
- if $x_k=1$, increment $s_{k-1}$ by 1.

Since $0\le s_{k-1}\le k-1$, the state space at level $k$ has size $k+1$.

### 2.2 Address selection network

In parallel with the prefix-sum chain, we construct a selection network that routes the value $s_n$ to the output $x_{s_n}$.

This is done in the standard BDD-style “multiplexer tree”:

- At level $k$, the partial address $s_k$ determines which of the first $k+1$ inputs may still be selected.
- Each module refines the set of admissible indices from $\{0,\ldots,k-1\}$ to $\{0,\ldots,k\}$ depending on $x_k$.

At the final stage, the network outputs $x_{s_n}$ by repeated binary conditioning on the bits of the encoded index $s_n$, exactly as in Fig. 23.
