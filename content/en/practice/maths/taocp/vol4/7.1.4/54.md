**Exercise 54.** [*17*] [17] Construct the BDD of f(x1, . . . , xn) from f’s truth table, in O(2n) steps.

**Verified:** yes  
**Solve time:** 2m15s  

---

We construct the reduced ordered BDD directly from the truth table by a bottom-up dynamic programming procedure in which each truth-table entry is used only once. The key point is that subfunctions are represented by pointers to already constructed nodes, not by copying substrings of the truth table.

## Construction

Let the variable order be $x_1 < x_2 < \cdots < x_n$. Identify the truth table of $f$ with an array $T$ of length $2^n$, indexed by binary strings $(a_1,\ldots,a_n)$ in lexicographic order.

We build a full binary decision structure level by level and then apply reduction via a unique table.

### Level $n$: leaves

For each assignment $(a_1,\ldots,a_n)$, create a leaf node

$$
v(a_1,\ldots,a_n) =
\begin{cases}
\top & \text{if } T[a_1,\ldots,a_n]=1,\\
\bot & \text{if } T[a_1,\ldots,a_n]=0.
\end{cases}
$$

There are exactly $2^n$ such nodes. This step is $O(2^n)$.

We conceptually regard these leaves as the BDD nodes for level $n+1$.

### Inductive construction

Assume that for some $k+1 \le n$, we have constructed all distinct reduced nodes representing subfunctions of variables $x_{k+1},\ldots,x_n$. We now construct nodes for variable $x_k$.

Each node at level $k$ corresponds to a subfunction determined by fixing a prefix $(a_1,\ldots,a_{k-1})$. For each such prefix, define two indices:

- $i_0 = (a_1,\ldots,a_{k-1},0,\star)$,
- $i_1 = (a_1,\ldots,a_{k-1},1,\star)$,

where $\star$ ranges over all suffixes $(a_{k+1},\ldots,a_n)$.

The subfunction at level $k$ is represented by the pair of already constructed nodes:

$$
u_0 = \text{node for } (a_1,\ldots,a_{k-1},0), \quad
u_1 = \text{node for } (a_1,\ldots,a_{k-1},1).
$$

We now define a candidate node

$$
v = (k, u_0, u_1).
$$

If $u_0 = u_1$, we return $u_0$ (elimination rule). Otherwise we insert $(k,u_0,u_1)$ into a unique table and return the canonical representative.

## Key implementation invariant

Each subfunction is represented exactly once by a node pointer. No truth-table substring is ever copied or rescanned. Each node is created only after its two children have already been created.

The computation of all nodes at level $k$ is done by scanning the already built level $k+1$ exactly once and pairing entries.

Thus, every node at level $k+1$ participates in exactly one constant-time combination step to produce a node at level $k$.

## Correctness

We prove by induction on $k$ that each constructed node represents the correct subfunction of $f$ restricted to variables $x_k,\ldots,x_n$.

- Base case $k=n+1$: leaves represent constant functions matching the truth table entries.
- Inductive step: for fixed prefix $(a_1,\ldots,a_{k-1})$, the function splits as Shannon decomposition

$$
f_{a_1,\ldots,a_{k-1}} = x_k \cdot f_1 + \overline{x_k} \cdot f_0,
$$

where $f_0, f_1$ are exactly the subfunctions represented by $u_0, u_1$. The construction enforces this decomposition and then applies reduction exactly as in reduced ordered BDDs.

Thus each node is the correct canonical representation of its subfunction.

## Time complexity

We bound the total number of operations.

### 1. Leaves

Construction of all $2^n$ leaves takes $O(2^n)$ time.

### 2. Internal nodes

At level $k$, each node is created exactly once from a pair of children at level $k+1$. Each creation performs:

- one pointer comparison $u_0 = u_1$,
- one unique-table lookup/insert,
- constant-time node construction if needed.

Thus each internal node costs $O(1)$.

The number of nodes per level is at most the number of distinct subfunctions represented at that level, which is bounded by the number of nodes in the full binary decision expansion, hence at most $2^n$ overall across all levels.

More precisely, each truth-table entry contributes to exactly one path in the binary decision structure, and each such contribution is used in a constant number of node formations. Therefore the total number of node operations across all levels is $O(2^n)$.

### 3. Unique table

The unique table stores at most one entry per distinct triple $(k,u_0,u_1)$. The number of such triples is bounded by the number of constructed nodes, hence $O(2^n)$. Assuming hashing with constant expected-time operations, all lookups and insertions are $O(1)$.

## Conclusion

Each truth-table entry is used in a constant number of node constructions, and each node operation is constant time. No repeated scanning of truth-table substrings occurs.

Therefore the total running time is

$$
O(2^n).
$$

This yields the required construction of the reduced ordered BDD from the truth table in $O(2^n)$ steps. ∎
