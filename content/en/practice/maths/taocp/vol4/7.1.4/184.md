**Exercise 184.** [*M23*] [M23] What are the BDD and ZDD profiles of the permutation function Pm?

Let $P_m$ be the Boolean function on variables $x_{ij}$ $(1 \le i,j \le m)$ that is $1$ iff $(x_{ij})$ is a permutation matrix. The variables are ordered lexicographically

$$
x_{11},x_{12},\dots,x_{1m},x_{21},\dots,x_{mm}.
$$

A partial assignment to the first $k$ variables is naturally interpreted row by row. Write

$$
k = tm + s,\qquad 0 \le s < m.
$$

Thus the first $t$ rows are completely exposed, and row $t+1$ is partially exposed in its first $s$ columns.

The key point is that a BDD or ZDD node represents a residual Boolean function, so two partial assignments are equivalent exactly when they induce the same set of completions.

## 1. Correct state description

### Completed rows

After processing $t$ full rows, each row $i \le t$ must contain exactly one 1. Let $U \subseteq \{1,\dots,m\}$ be the set of columns already used in these rows.

Once $U$ is fixed, the future constraints are independent of how the columns in $U$ were distributed among the first $t$ rows. Indeed, for every future row $i > t$, the only requirement is:

- choose exactly one column in $\{1,\dots,m\} \setminus U$,
- avoid columns already used.

Thus the residual function depends only on $U$, not on the bijection between $\{1,\dots,t\}$ and $U$.

Hence at level $tm$, BDD states are in one-to-one correspondence with subsets $U \subseteq [m]$ of size $t$, so the number of states is

$$
\binom{m}{t}.
$$

### Partially exposed row

At level $k = tm + s$, $0 < s < m$, row $t+1$ is partially specified.

There are two qualitatively different situations:

**(A) No 1 has appeared in the first $s$ variables of row $t+1$.**

Then row $t+1$ is still “open”: it must place its unique 1 among the remaining columns $\{s+1,\dots,m\}$, avoiding $U$.

The state is fully determined by $U$, with $|U|=t$.

**(B) Exactly one 1 has already appeared in the prefix of row $t+1$.**

Suppose this 1 is in column $j \le s$. Then $j \in U$, and row $t+1$ is already satisfied; all remaining entries in that row must be 0. The remaining constraint is again determined solely by the set $U$, now of size $t+1$.

Importantly, different choices of $j \in U$ lead to the same residual function, since future behavior depends only on which columns are forbidden, not where they were first used.

Thus case (B) is indexed only by $U \subseteq [m]$ with $|U| = t+1$.

## 2. BDD profile

Combining the two disjoint cases:

- states with $|U|=t$ (row $t+1$ still open),
- states with $|U|=t+1$ (row $t+1$ already fixed),

we obtain for $k = tm+s$, $0 < s < m$,

$$
B_k^{\mathrm{BDD}} = \binom{m}{t} + \binom{m}{t+1}.
$$

For boundary levels $s=0$,

$$
B_{tm}^{\mathrm{BDD}} = \binom{m}{t}.
$$

(With the convention $\binom{m}{t+1}=0$ for $t+1>m$.)

## 3. ZDD profile

For ZDDs, the function is viewed as a family of sets of selected pairs $(i,j)$ forming perfect matchings.

Under the same lexicographic ordering, the ZDD reduction rule changes the representation but not the underlying equivalence of residual families: two partial assignments induce the same ZDD node exactly when they admit the same set of completions.

The same structural observation applies:

- only the set $U$ of already used columns matters,
- and at the boundary between rows, whether the current row is already satisfied or still open matters.

No additional distinguishing structure survives ZDD reduction, since all remaining choices depend only on available columns and row completeness.

Hence the ZDD state space at level $k$ is identical to the BDD state space, and

$$
B_k^{\mathrm{ZDD}} = \binom{m}{t} + \binom{m}{t+1}, \qquad k = tm+s,\ 0<s<m,
$$

with

$$
B_{tm}^{\mathrm{ZDD}} = \binom{m}{t}.
$$

## 4. Final result

Let $k = tm+s$, $0 \le s < m$. Then

$$
B_k^{\mathrm{BDD}} = B_k^{\mathrm{ZDD}} =
\begin{cases}
\binom{m}{t}, & s=0,\\[6pt]
\binom{m}{t} + \binom{m}{t+1}, & 0 < s < m.
\end{cases}
$$

The profiles coincide because, under this ordering, both representations collapse exactly to tracking (i) the set of used columns and (ii) whether the current row has already been satisfied within the partially exposed block.
