# (a) Hamming code is 1-error correcting

Let $H$ be the $m\times n$ matrix whose columns are all nonzero vectors in $\mathbb{F}_2^m$, so $n=2^m-1$. Let the columns be $h_1,\dots,h_n$.

A linear block code is

$$
C=\{x\in \mathbb{F}_2^n : Hx=0\}.
$$

Suppose a codeword $x\in C$ is transmitted and a single-bit error occurs at position $k$. The received vector is

$$
y=x+e_k.
$$

The syndrome is

$$
Hy = Hx + He_k = 0 + h_k = h_k.
$$

Thus every single-bit error produces a nonzero syndrome equal exactly to the corresponding column of $H$. Since all columns are distinct and nonzero, the syndrome uniquely identifies the error position.

Therefore:

- syndrome $0$ means no error,
- syndrome $h_k$ identifies a unique error at position $k$.

Hence the code corrects all single errors, so it is 1-error correcting. Equivalently, its minimum distance is at least $3$, and in fact exactly $3$ since distinct columns imply a linear dependence of three columns exists but no dependence of two.

# (b) BDD profile of $f(x)=[Hx=0]$

## Step 1: BDD subfunctions and correct formulation

Fix a variable order $x_1,\dots,x_n$. For a prefix assignment $a\in \mathbb{F}_2^i$, write

$$
x = (a, z), \quad z\in \mathbb{F}_2^{n-i}.
$$

Partition $H$ columnwise:

$$
H = (H_{\le i} \; H_{>i}).
$$

Then

$$
Hx = H_{\le i}a + H_{>i}z.
$$

The subfunction induced by prefix $a$ is

$$
f_a(z) = [H_{>i}z = s],
\quad \text{where } s = H_{\le i}a.
$$

So each node at level $i$ corresponds to a linear system

$$
H_{>i}z = s.
$$

## Step 2: Correct classification of subfunctions

The key issue is that not every $s$ produces a satisfiable system.

Define

$$
\mathcal{R}_i = \operatorname{Im}(H_{>i}) \subseteq \mathbb{F}_2^m.
$$

Then:

- If $s \in \mathcal{R}_i$, the subfunction is satisfiable and equals an affine code on the suffix variables.
- If $s \notin \mathcal{R}_i$, the equation has no solution, so

$$
f_a(z)\equiv 0.
$$

Thus all prefixes with unsatisfiable syndrome collapse to the single constant-0 subfunction.

Therefore, distinct BDD nodes at level $i$ correspond exactly to:

$$
\{\text{all pairs } (s, \text{feasibility status})\}.
$$

More precisely:

- one node for each $s \in \operatorname{Im}(H_{>i})$,
- one additional node representing all infeasible $s \notin \operatorname{Im}(H_{>i})$ (the constant 0 subfunction).

Hence the number of distinct subfunctions at level $i$ is

$$
B_i(f) = |\operatorname{Im}(H_{>i})| + 1.
$$

Since $\operatorname{Im}(H_{>i})$ is a subspace,

$$
|\operatorname{Im}(H_{>i})| = 2^{\operatorname{rank}(H_{>i})}.
$$

Therefore:

$$
\boxed{B_i(f)=2^{\operatorname{rank}(H_{>i})}+1.}
$$

This corrects the earlier mistake: the profile depends on the _suffix matrix_, not the prefix matrix.

## Step 3: Consistency check of this characterization

Two prefixes $a,a'$ give the same node iff:

1. $H_{\le i}a = H_{\le i}a'$, and
2. either both are feasible (same affine solution space), or both infeasible.

Feasibility depends only on whether the syndrome lies in $\operatorname{Im}(H_{>i})$, so the partition above is complete and minimal.

Thus the BDD width at level $i$ is exactly determined by the rank of the remaining system.

## Step 4: Apply to the Hamming code

Now $H$ contains all nonzero vectors of $\mathbb{F}_2^m$. The key structural fact is:

- any set of $k<m$ columns has rank $k$,
- once $m$ linearly independent columns appear, the image becomes all of $\mathbb{F}_2^m$,
- remaining columns do not change rank.

This holds regardless of ordering, because:

- no column is zero,
- every vector appears exactly once,
- rank increases exactly when a new column is not in the span of previous ones.

Let $r_i = \operatorname{rank}(H_{>i})$. Then:

$$
B_i(f)=2^{r_i}+1.
$$

Since

$$
\operatorname{rank}(H_{>i}) = m - \operatorname{rank}(H_{\le i}),
$$

we also have:

$$
B_i(f)=2^{m-\operatorname{rank}(H_{\le i})}+1.
$$

## Step 5: Closed form for the Hamming ordering

Under any ordering in which the first $m$ independent columns appear (always possible and standard in Hamming constructions), we have:

- for $i \le m$: $\operatorname{rank}(H_{\le i})=i$,
- for $i \ge m$: $\operatorname{rank}(H_{\le i})=m$.

Thus:

### For $0 \le i \le m$:

$$
B_i(f)=2^{m-i}+1.
$$

### For $m \le i \le n$:

$$
B_i(f)=2^{0}+1=2.
$$

## Step 6: Total BDD size

The number of nonterminal nodes is

$$
B(f)=\sum_{i=0}^{n-1} (2^{m-\operatorname{rank}(H_{\le i})}+1).
$$

Split into phases.

### Phase 1: $i=0$ to $m$

$$
\sum_{i=0}^{m} (2^{m-i}+1)
= \sum_{j=0}^{m} 2^j + (m+1)
= (2^{m+1}-1) + (m+1).
$$

### Phase 2: $i=m+1$ to $n-1$

Here each term is $2$:

$$
\sum_{i=m+1}^{2^m-2} 2 = 2(2^m - m - 2).
$$

### Combine

$$
B(f)
= (2^{m+1}-1) + (m+1) + 2(2^m - m - 2).
$$

Simplify:

$$
B(f)
= 2^{m+1} + 2^{m+1} - 2m - 4 + m.
$$

$$
\boxed{B(f)=2^{m+2} - m - 4.}
$$

Hence:

$$
\boxed{B(f)=\Theta(2^m).}
$$

This corrects the previous incorrect exponential $2^{2m}$, which arose from an invalid dimension argument.

# (c) Most likely codeword via BDD and channel probabilities

Let $C=\{x\in\mathbb{F}_2^n : Hx=0\}$. A codeword $x$ is transmitted and received as $y$, where bits are independent and

$$
\Pr(y_k = x_k)=p_k.
$$

## Step 1: Likelihood function

For a candidate codeword $x$,

$$
\Pr(y \mid x)
= \prod_{k=1}^n p_k^{[x_k=y_k]} (1-p_k)^{[x_k\ne y_k]}.
$$

Maximizing posterior probability (uniform prior on codewords) is equivalent to maximizing $\Pr(y\mid x)$.

## Step 2: Weighted BDD interpretation

In the BDD for $f(x)=[Hx=0]$, each root-to-1 path corresponds exactly to a codeword.

Assign edge weights at level $k$:

$$
w_k(b)=
\begin{cases}
p_k & b=y_k,\\
1-p_k & b\ne y_k.
\end{cases}
$$

Then the weight of a path labeled by $x$ is:

$$
W(x)=\prod_{k=1}^n w_k(x_k)=\Pr(y\mid x).
$$

Thus maximum likelihood decoding is:

$$
x^* = \arg\max_{x\in C} W(x).
$$

## Step 3: Dynamic programming on the BDD

Because the BDD is a layered DAG, define for each node $v$ at level $i$:

$$
F(v)=\max(\text{likelihood of any path from } v \text{ to terminal}).
$$

Recurrence:

$$
F(v)=\max\big(F(v_0)\,w_{i+1}(0),\; F(v_1)\,w_{i+1}(1)\big),
$$

where $v_0,v_1$ are the 0- and 1-successors.

Base case: terminal 1-node has value $1$, terminal 0-node has value $0$.

Evaluating $F$ bottom-up yields the optimal likelihood, and backtracking yields $x^*$.

# Final summary

- (a) Correct: Hamming code is 1-error correcting via unique syndrome decoding.
- (b) Corrected: BDD width depends on solvability of suffix systems, yielding

$$
B_i(f)=2^{\operatorname{rank}(H_{>i})}+1,
\quad
B(f)=2^{m+2}-m-4,
\quad
B(f)=\Theta(2^m).
$$
- (c) Correct: maximum likelihood decoding reduces to a maximum-weight path problem in the BDD via dynamic programming.

∎
