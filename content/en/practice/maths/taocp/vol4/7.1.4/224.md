Let $D$ be a DAG in which every non-source vertex has in-degree $1$. Such a DAG is a rooted directed forest: every vertex belongs to a unique source-rooted tree, and each vertex has a unique directed path from its source.

Let $\mathcal{P}(v)$ denote the set of all directed paths from $v$ to any sink.

The ZDD of Fig. 28 is constructed by recursion on these path families: a nonterminal node represents a family $\mathcal{P}(v)$, and the structure of the node encodes the decomposition of paths according to the first choice of outgoing edge.

## 1. Structure of path families in a forest DAG

Let $v$ be a vertex with outgoing neighbors $v_1,\dots,v_k$ in the fixed order used by the construction.

Every path $p \in \mathcal{P}(v)$ begins by choosing exactly one outgoing edge $(v,v_i)$, and then continues with a path in $\mathcal{P}(v_i)$. Hence

$$
\mathcal{P}(v)
=
\bigcup_{i=1}^k (v \to v_i)\circ \mathcal{P}(v_i),
$$

and this union is disjoint because the first edge of a path is uniquely determined.

If $v$ is a sink, then $\mathcal{P}(v)=\{\epsilon\}$, the singleton set containing the empty path.

## 2. Well-defined ZDD nodes as subproblems

Define a node $N(v)$ to represent the subproblem $\mathcal{P}(v)$ as produced by the ZDD construction.

We first show that this definition is canonical.

If $v \neq w$, then $\mathcal{P}(v) \neq \mathcal{P}(w)$. Indeed, in a forest DAG each vertex lies in a unique rooted tree, and $\mathcal{P}(v)$ consists exactly of paths contained in the subtree rooted at $v$, while $\mathcal{P}(w)$ is contained in a different subtree unless $v=w$. Thus no two distinct vertices induce identical path families.

Hence each vertex determines a unique ZDD subproblem, so we may identify ZDD nodes with vertices of $D$ without ambiguity of merging.

## 3. ZDD recursion in the forest case

For a non-sink vertex $v$, the ZDD node $N(v)$ is defined by the ordered decomposition of $\mathcal{P}(v)$. Conceptually, it is a decision among the alternatives

$$
(v,v_1), (v,v_2), \dots, (v,v_k).
$$

In the standard binary ZDD encoding of such an ordered choice, this is represented as a binary decision chain:

- choosing $v_1$ corresponds to the 0-branch at the first decision,
- skipping $v_1$ corresponds to the 1-branch leading to the next choice,
- and so on.

Formally, the ZDD node satisfies the recursive structure

$$
N(v) = \operatorname{node}\big(v_1,\; N(v_1),\; \operatorname{node}(v_2,\dots,v_k)\big),
$$

which unrolls into a binary decision chain selecting exactly one outgoing edge.

For a sink $v$, $N(v)$ is the terminal node.

## 4. Induced structure of a forest

Because every non-source vertex has in-degree $1$, each vertex $v$ has a unique parent (except sources), and the subgraph reachable from any vertex is a rooted tree.

Thus each $N(v)$ depends only on the subtree rooted at $v$, and the recursion for $N(v)$ propagates independently down each child $v_i$.

No sharing between different branches occurs except through identical subtrees, but in a forest DAG these subtrees are disjoint, so the ZDD construction produces a node for each vertex with no forced identifications beyond structural equality.

## 5. Correspondence with the binary-tree representation

Recall the “natural correspondence between forests and binary trees” (Eqs. 2.3.2–(1) to (3)):

- the first child of a node becomes its left child,
- the next sibling becomes its right child,
- a forest is represented as a binary tree by chaining siblings through right pointers.

We now match this exactly with the ZDD recursion.

At a vertex $v$ with ordered children $v_1,\dots,v_k$:

- the choice of the first outgoing option $v_1$ corresponds to descending into the left branch,
- moving from $v_i$ to $v_{i+1}$ corresponds to following right-sibling links,
- termination at $v_k$ corresponds to the end of the sibling chain.

Thus the ZDD node for $v$ encodes precisely a left-child/right-sibling representation of the ordered list $v_1,\dots,v_k$.

Inductively, the same correspondence holds for every descendant $v_i$, since their ZDD nodes encode the same ordered decomposition of their outgoing edges.

## 6. Conclusion

The ZDD for all source-to-sink paths in $D$ is obtained by applying a uniform recursive construction to each vertex $v$, where each node encodes an ordered choice among outgoing edges and recursively invokes the same construction on the targets.

In a forest DAG, these recursive subproblems are independent and correspond exactly to rooted subtrees. The resulting structure is therefore identical, node-for-node, with the binary tree obtained from the forest via the standard left-child/right-sibling encoding.

Hence the ZDD is essentially the same as the binary tree representation of the forest described by Eqs. 2.3.2–(1) through (3). ∎
