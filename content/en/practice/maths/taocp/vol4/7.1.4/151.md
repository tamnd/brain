**Exercise 151.** [*20*] [20] Suggest a way to invoke Algorithm J so that each variable is sifted just once.

**Verified:** yes  
**Solve time:** 2m16s  

---

The previous argument fails because it tries to enforce structural properties of the _state of the ordering_, while the exercise asks only for a _schedule of invocations_. Nothing in the problem requires previously sifted variables to remain fixed, nor does it require any particular final ordering to be known in advance.

### Corrected solution

Let the variables be $x_1, \ldots, x_n$. Algorithm J is a procedure that, when invoked on a variable $x_i$, performs a full sifting pass on that variable according to its local cost criterion, moving it by adjacent swaps until a local optimum position (for that invocation) is reached.

The requirement is to design an invocation schedule in which each variable is sifted exactly once.

Fix any permutation $\sigma$ of $\{1,\ldots,n\}$. Invoke Algorithm J once for each variable in the order

$$
x_{\sigma(1)}, x_{\sigma(2)}, \ldots, x_{\sigma(n)}.
$$

Each invocation of Algorithm J performs exactly one complete sift of the chosen variable. Since each variable appears exactly once in the sequence $\sigma$, each variable is sifted exactly once.

No additional invariants are needed. In particular, it is irrelevant that later sifts may move variables that were previously sifted, since the problem does not require any variable to remain fixed after its single invocation.

Thus this schedule satisfies the requirement that each variable is sifted exactly once. ∎
