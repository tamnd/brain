**Exercise 47.** [*M21*] [M21] Devise and prove a converse of Theorem M: Every Boolean function f with
a small BDD can be implemented by an efficient network of modules.

**Verified:** yes  
**Solve time:** 1m08s

## Solution

Let $f(x_1,\ldots,x_n)$ be a Boolean function with BDD $G$, and let $B(f)$ denote the number of nodes of $G$, including sinks $\bot$ and $\top$.

Each non-sink node $v$ of $G$ is labeled by a variable index $V(v)=j$, and has two successors $LO(v)$ and $HI(v)$ with $j < V(LO(v)), V(HI(v))$ by the ordering condition.

### Construction of the network

Associate with every node $v$ of the BDD a Boolean module $M_v$ computing a function $g_v$ defined as follows.

If $v=\bot$, set $g_v = 0$. If $v=\top$, set $g_v = 1$.

If $v$ is a branch node labeled $x_j$, define

$$
g_v(x_1,\ldots,x_n)
=
(\neg x_j \land g_{LO(v)}) \lor (x_j \land g_{HI(v)}).
$$

This defines a network in which each module $M_v$ has as inputs the variable $x_{V(v)}$ and the outputs of the modules $M_{LO(v)}$ and $M_{HI(v)}$, with directed connections following the edges of the BDD. The output of the entire network is $M_r$, where $r$ is the root of $G$.

### Correctness

For each node $v$, the function $g_v$ coincides with the Boolean function represented by the sub-BDD rooted at $v$.

The proof proceeds by structural induction on the partial order induced by the BDD.

For sink nodes $v \in {\bot,\top}$, the statement holds by definition.

Let $v$ be a branch node labeled $x_j$. Assume the claim holds for $LO(v)$ and $HI(v)$. For any assignment $(x_1,\ldots,x_n)$, evaluation of the BDD at $v$ follows the definition of ordered decision diagrams: if $x_j=0$, the evaluation proceeds to $LO(v)$; if $x_j=1$, it proceeds to $HI(v)$. Hence the represented function satisfies

$$
g_v = (\neg x_j \land g_{LO(v)}) \lor (x_j \land g_{HI(v)}),
$$

which matches the definition of the module $M_v$. This completes the induction, and in particular $g_r = f$.

### Size of the network

Each node of the BDD contributes exactly one module $M_v$. No additional modules are introduced, since fan-out is allowed: distinct parents may share the same successor module corresponding to shared BDD subgraphs.

Hence the number of modules in the network equals the number of nodes of the BDD, namely $B(f)$.

Each module performs a constant number of Boolean operations determined solely by its fixed form $(\neg x_j \land \cdot)\lor(x_j \land \cdot)$, so the total work of evaluation is proportional to $B(f)$.

### Efficiency implication

If $f$ has a small BDD, meaning $B(f)$ is small, then the constructed network has size $B(f)$ and depth at most $n$ (since variable indices strictly increase along any path). Thus the network is efficient in both size and acyclic structure.

Conversely, any Boolean function with bounded BDD size admits such a linear-size modular implementation.

This completes the proof. ∎
