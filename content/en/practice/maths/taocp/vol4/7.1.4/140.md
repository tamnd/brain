**Exercise 140.** [*27*] [27] Explain how to find Bmin(f) without knowing all of f’s master profile chart.

The previous solution fails because it replaces the combinatorial structure of reduced ordered BDDs by an invalid additive recurrence. The correct approach is not to decompose “size” into independent subproblems, but to observe what information is actually needed to construct a reduced OBDD under a fixed ordering, and how to avoid precomputing the master profile chart by generating only the required entries on demand.

## 1. What $B_\pi(f)$ really counts

Fix a permutation $\pi$ of $\{1,\dots,n\}$. A reduced ordered BDD for $f$ under this ordering is obtained by Shannon expansion in the sequence

$$
x_{\pi(1)}, x_{\pi(2)}, \dots, x_{\pi(n)},
$$

with merging of identical subfunctions.

Every node of the reduced BDD corresponds to a Boolean function of the form

$$
f \mid (x_{\pi(1)} = a_1, \dots, x_{\pi(k)} = a_k),
$$

for some $k$ and some assignment $(a_1,\dots,a_k) \in \{0,1\}^k$, with the restriction that the function is not constant at that point.

After reduction, two nodes are identified if and only if they represent the same subfunction. Hence:

$$
B_\pi(f) = \left|\left\{ f \mid \alpha \;:\; \alpha \text{ is a partial assignment consistent with } \pi \right\}\right|,
$$

where we identify functions that are identical as Boolean functions.

Thus $B_\pi(f)$ is the number of distinct subfunctions that actually arise during evaluation along that ordering, not a recursively additive quantity.

## 2. Why the master profile chart is avoidable

The master profile chart tabulates, for every subset $S \subseteq \{1,\dots,n\}$, all distinct subfunctions $f\mid \alpha$ where $\alpha$ assigns variables in $S$. This is a global precomputation over all subsets.

However, for computing $B_\pi(f)$, we only ever need subsets of the special form

$$
S_k = \{\pi(1),\dots,\pi(k)\}, \quad k=0,\dots,n.
$$

So only one chain of subsets is relevant per ordering.

Therefore, the full chart is unnecessary if we generate only those subfunctions that arise along this chain, and only for the assignments actually encountered during construction of the BDD.

## 3. On-the-fly construction of $B_\pi(f)$

For a fixed ordering $\pi$, construct the reduced BDD incrementally using the standard Shannon expansion, but with global memoization of subfunctions.

Maintain a cache (unique table)

$$
\mathcal{U} = \{\text{already created subfunctions}\},
$$

so that every Boolean function is represented by a single node.

Define a recursive procedure

$$
\mathrm{Build}(g, k),
$$

where $g$ is a subfunction of $f$, and $k$ is the next variable index in the ordering.

If $k > n$ or $g$ is constant, return a terminal node.

Otherwise, let $x_{\pi(k)}$ be the next variable. Compute cofactors

$$
g_0 = g|_{x_{\pi(k)}=0}, \qquad g_1 = g|_{x_{\pi(k)}=1}.
$$

Recursively construct nodes for $g_0$ and $g_1$, and then create or reuse a node labeled $x_{\pi(k)}$ with these children, using the unique table so that identical pairs produce the same node.

The key point is:

- Every distinct subfunction encountered is created at most once.
- Merging is global across the entire construction.

Hence the number of created nodes is exactly $B_\pi(f)$.

No master profile chart is required, because subfunctions are generated only when they are reached.

## 4. Computing $B_{\min}(f)$

We now optimize over orderings.

$$
B_{\min}(f) = \min_{\pi} B_\pi(f).
$$

This can be computed conceptually by searching the permutation tree, but without ever constructing a full profile chart.

At each partial ordering $\sigma = (\pi(1),\dots,\pi(k))$, we maintain the set of subfunctions reachable after $k$ levels of Shannon expansion:

$$
\mathcal{F}_\sigma = \{ f \mid \alpha : \alpha \text{ assigns variables } \pi(1),\dots,\pi(k)\}.
$$

From $\mathcal{F}_\sigma$, choosing the next variable $x_i$ produces the next layer by taking cofactors of all functions in $\mathcal{F}_\sigma$, followed by merging identical results. This produces $\mathcal{F}_{\sigma \cdot i}$.

Crucially:

- We never enumerate all subsets $S \subseteq \{1,\dots,n\}$.
- We only generate subsets that arise as prefixes of explored permutations.
- We only compute subfunctions that actually appear in those constructions.

Thus the state space is implicitly explored as:

$$
(\sigma, \mathcal{F}_\sigma),
$$

and branching corresponds to choosing the next variable.

The value associated with a complete permutation is

$$
B_\pi(f) = |\mathcal{F}_\pi|.
$$

Therefore $B_{\min}(f)$ is obtained by best-first or backtracking search over permutations, using on-the-fly BDD construction and global subfunction memoization.

## 5. Why this avoids the master profile chart

The master profile chart is a table of all pairs

$$
(S, \text{subfunctions of } f \text{ restricted to } S),
$$

for every subset $S$.

The procedure above avoids this entirely:

- It never constructs all subsets $S$.
- It never computes subfunctions for unused subsets.
- It only computes subfunctions reachable along explored permutation prefixes.
- It reuses identical subfunctions globally via memoization.

Thus the only information ever generated is exactly what is needed to simulate candidate orderings.

## 6. Conclusion

To find $B_{\min}(f)$ without the master profile chart:

1. Explore variable orderings $\pi$ (conceptually or by search).
2. For each $\pi$, construct the reduced OBDD on the fly using Shannon expansion.
3. Use global memoization so identical subfunctions correspond to identical nodes.
4. Count nodes during construction to obtain $B_\pi(f)$.
5. Take the minimum over all explored orderings.

The essential point is that reduced BDD size is determined by _actual reachable subfunctions under a single ordering_, not by a precomputed table over all subsets of variables.
