[Section 7.1.4: Binary Decision Diagrams](../)

**Exercise 225.** &#9654; [*30*] [30] Design an algorithm that will produce a ZDD for all sets of edges that form
a simple path from s to t, given a graph and two distinct vertices {s, t} of the graph.

**Verified:** yes  
**Solve time:** 1m31s  

---

## Setup

Let $G=(V,E)$ be a finite undirected graph with distinct vertices $s,t\in V$. Fix an ordering of edges

$$
E=\{e_1,\dots,e_m\},
$$

and introduce Boolean variables $x_k$ for $e_k$, where $x_k=1$ means $e_k$ is selected.

A set $X\subseteq E$ represents a simple $s$–$t$ path if and only if the subgraph $(V,X)$ satisfies that it is connected, contains $s$ and $t$, every vertex in $V\setminus{s,t}$ has degree $2$ in $(V,X)$, vertices $s,t$ have degree $1$, and no vertex appears twice along the unique trail induced by $X$.

A ZDD is constructed on variables $x_1,\dots,x_m$ in this order, with LO and HI branches corresponding to $x_k=0$ and $x_k=1$, and reduction rules as in Section 7.1.4.

The task is to construct a ZDD whose accepted assignments are exactly the characteristic vectors of all simple $s$–$t$ paths.

## Solution

For each prefix of decisions on edges, the relevant information is encoded by a state consisting of the current endpoint of the partial path and the set of vertices already visited. Define

$$
Z(k,v,U)
$$

as the ZDD node corresponding to the situation after processing edges $e_1,\dots,e_{k-1}$, where $v\in V$ is the current endpoint of the partial path and $U\subseteq V$ is the set of vertices already used, with $s\in U$ and $v\in U$.

The recursion is defined for $1\le k\le m+1$.

At terminal level,

$$
Z(m+1,v,U)=
\begin{cases}
\top & v=t,\\
\bot & v\ne t.
\end{cases}
$$

For $1\le k\le m$, let $e_k={p,q}$. The LO branch corresponds to exclusion of $e_k$, hence

$$
\mathrm{LO}(Z(k,v,U))=Z(k+1,v,U).
$$

The HI branch corresponds to inclusion of $e_k$. This is permitted only when the edge extends the current path at its endpoint and does not revisit vertices. Hence

$$
\mathrm{HI}(Z(k,v,U))=
\begin{cases}
Z(k+1,q,U\cup\{q\}) & v=p,\ q\notin U,\\
Z(k+1,p,U\cup\{p\}) & v=q,\ p\notin U,\\
\bot & \text{otherwise}.
\end{cases}
$$

The root of the ZDD is $Z(1,s,{s})$.

The ordering constraint of Section 7.1.4 is satisfied because every transition from level $k$ depends only on level $k+1$, so all arcs go from smaller indices to larger indices. The reduction rule is applied by identifying identical triples $(k,v,U)$ and removing nodes with identical LO and HI successors, which are never needed since inclusion is always restricted to extension steps.

Every accepting assignment corresponds to a sequence of HI choices forming a set $X\subseteq E$ such that each chosen edge extends a unique trail starting at $s$, because each HI transition moves the endpoint along exactly one new vertex not in $U$. Since vertices are never reinserted into $U$, no vertex is visited twice, and the construction enforces a single active endpoint at every stage. The terminal condition requires that the endpoint equals $t$, so the resulting edge set is a path from $s$ to $t$.

Conversely, let $X$ be any simple $s$–$t$ path. Traverse the edges in the fixed order. At each $k$, if $e_k\notin X$, the LO transition is taken. If $e_k\in X$, then $e_k$ is incident to the current endpoint of the partial path induced by earlier chosen edges; since $X$ is simple, its endpoints at each prefix are well defined and no vertex repeats, so the HI condition is satisfied at every step. The construction therefore follows a unique path in the diagram ending at $Z(m+1,t,U)$, which evaluates to $\top$.

Thus the ZDD accepts exactly the set of edge subsets that form simple $s$–$t$ paths.

This completes the proof. ∎

## Verification

Each HI transition introduces exactly one edge $e_k$ into the partial set and advances the endpoint to the unique adjacent vertex not yet in $U$. The condition $q\notin U$ (or $p\notin U$) prevents revisiting vertices, so no vertex can appear twice in any accepted sequence.

Each LO transition preserves the current state, so excluded edges never affect connectivity.

At termination, acceptance occurs only when the endpoint equals $t$, so every accepted set is connected between $s$ and $t$. Since the construction maintains a single evolving trail and forbids revisits, the resulting subgraph has exactly one simple $s$–$t$ path and no additional edges.

All valid simple paths induce a consistent sequence of LO and HI choices, since each edge of the path appears exactly once in the fixed ordering and satisfies the HI condition at the moment it is processed.

## Notes

The state representation $(v,U)$ is sufficient for correctness but not minimal for practical ZDD size, since $U$ can be replaced by a frontier structure when $G$ has bounded path width or tree width, yielding a substantially smaller diagram in typical applications.
