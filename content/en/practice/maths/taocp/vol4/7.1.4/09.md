**Exercise 9.** [*16*] [16] Given the BDD for a function f(x) = f(x1, . . . , xn), represented sequentially
as in (8), explain how to determine the lexicographically largest x such that f(x) = 0.

## Solution

Let each node of the BDD be either a sink node $\bot,\top$ or a branch node labeled by a variable index $V(v)=j$ with two outgoing arcs $\mathrm{LO}(v)$ and $\mathrm{HI}(v)$. A bit vector $x=(x_1,\dots,x_n)$ satisfies $f(x)=0$ exactly when the evaluation path starting at the root ends in $\bot$.

Define a predicate $R(v)$ on BDD nodes, where $R(v)$ holds when there exists an assignment to the remaining variables along any path starting from $v$ that reaches $\bot$. This is defined recursively by

$R(\bot)$ true, $R(\top)$ false, and for a branch node $v$, $R(v)$ is true exactly when $R(\mathrm{LO}(v))$ or $R(\mathrm{HI}(v))$ is true.

This recursion is well defined because the BDD is ordered, so every arc goes from a node with smaller variable index to a node with larger variable index, hence the evaluation graph is acyclic and the recursion terminates at sinks.

Construct a path from the root by processing variables in increasing order of their indices as encountered in the BDD. At a branch node $v$ labeled $j$, both outgoing arcs correspond to assigning $x_j=0$ or $x_j=1$. Among all assignments leading to $\bot$, the lexicographically largest assignment is obtained by maximizing $x_1$, then $x_2$, and so on. Therefore at node $v$ the choice $x_j=1$ is admissible exactly when $R(\mathrm{HI}(v))$ holds, since this means there exists an extension of the partial assignment with $x_j=1$ that reaches $\bot$. If $R(\mathrm{HI}(v))$ fails, the only possible choice consistent with reaching $\bot$ is $x_j=0$, and then the next node is $\mathrm{LO}(v)$.

Inductively along the path, after fixing $x_1,\dots,x_{j-1}$ according to this rule, the current node represents exactly the subfunction restricted by those assignments, and $R$ correctly characterizes feasibility of reaching $\bot$ from that node. The lexicographic maximality follows because at the first index $j$ where the constructed assignment differs from any other assignment reaching $\bot$, the construction chooses $x_j=1$ whenever feasible, while any alternative lexicographically larger choice would require $x_j=0$, contradicting maximality at that coordinate.

Thus the algorithm produces an assignment $x$ with $f(x)=0$, and no assignment with a larger lexicographic value can also satisfy $f(x)=0$.
