## Solution

Let each node $v$ of the BDD represent a Boolean function $f_v(x_{i},\dots,x_n)$, where $i = V(v)$, with sink nodes representing constant functions $0$ and $1$. The algorithm of exercise 138 computes the $z$-profile of a single function by evaluating recursively on the LO and HI subfunctions and combining the resulting profiles according to whether the current variable is fixed to $0$ or $1$.

To construct the master $z$-profile chart, the computation is extended from a single root evaluation to all distinct subfunctions represented in the BDD, while preserving the same recurrence used in exercise 138.

For each node $v$, define $Z(v)$ to be the $z$-profile of the function represented by $v$. If $v$ is a sink node, $Z(v)$ is the profile of the constant function it represents. If $v$ is a branch node with variable index $i$, LO successor $v_0$, and HI successor $v_1$, then $Z(v)$ is computed from $Z(v_0)$ and $Z(v_1)$ using exactly the same combination rule as in exercise 138, applied at level $i$ of the decomposition. This recurrence is well-defined because the ordering property of BDDs ensures that all successors correspond to functions in variables strictly later than $x_i$.

The modification consists of replacing the single recursive computation by a memoized traversal over the directed acyclic graph. During a depth-first traversal starting at the root, each node $v$ is processed exactly once. When a node is first encountered, its two successor nodes are processed recursively if they have not yet been assigned a $z$-profile, and then $Z(v)$ is computed from the stored values of $Z(v_0)$ and $Z(v_1)$ by the same rule as in exercise 138. The result $Z(v)$ is stored in a table indexed by node identity.

The master $z$-profile chart is the collection of all pairs $(v, Z(v))$ over all nodes $v$ reachable from the root. Because reduction guarantees that each Boolean subfunction corresponds to exactly one node, this chart assigns exactly one $z$-profile to each distinct subfunction represented in the BDD.

Since each node has exactly two outgoing edges and is evaluated once, and since each evaluation performs a constant amount of work beyond recursive lookup of successors, the total work is proportional to the number of nodes in the BDD.

This completes the construction of the master $z$-profile chart. ∎
