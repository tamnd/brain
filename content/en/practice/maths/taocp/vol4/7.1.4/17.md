**Exercise 17.** [*32*] [32] If possible, improve the algorithm of exercise 16 so that its running time is
O(B(f)) + O(N) when there are N solutions.

**Verified:** no

## Setup

Let $f(x_1,\dots,x_n)$ be a Boolean function represented by a reduced ordered binary decision diagram $D$ as defined in Section 7.1.4. Each internal node $v$ of $D$ is labeled by a variable index $V(v)\in{1,\dots,n}$ and has two outgoing arcs, $\mathrm{LO}(v)$ and $\mathrm{HI}(v)$, corresponding to setting $x_{V(v)}=0$ and $x_{V(v)}=1$. Sink nodes are $\bot$ and $\top$.

Exercise 16 asks for an algorithm that generates all vectors $(x_1,\dots,x_n)$ such that $f(x_1,\dots,x_n)=1$ given the BDD of $f$.

The goal here is to improve that algorithm so that the total running time is

$$
O(B(f)) + O(N),
$$

where $B(f)$ is the number of nodes in the BDD and $N$ is the number of solutions output.

Each solution is an assignment $x=(x_1,\dots,x_n)$ with $f(x)=1$.

## Solution

The algorithm performs a depth-first traversal of the BDD while maintaining a single global assignment array $x_1,\dots,x_n$. When a node $v$ is entered, the variable $x_{V(v)}$ is assigned according to the branch taken, and the recursion proceeds to the corresponding child.

A direct recursive formulation is as follows.

At any node $v$, assume variables $x_1,\dots,x_{V(v)-1}$ are already fixed by the current path. If $v=\bot$, the search stops with no output. If $v=\top$, the current partial assignment is extended arbitrarily to a full vector by assigning all unset variables consistently with the last decisions along the path, and the resulting vector is output.

For an internal node $v$ with $V(v)=j$, the procedure explores the two outgoing edges:

When the $\mathrm{LO}(v)$ edge is followed, the value $x_j$ is set to $0$, and the algorithm continues from $\mathrm{LO}(v)$. When the $\mathrm{HI}(v)$ edge is followed, the value $x_j$ is set to $1$, and the algorithm continues from $\mathrm{HI}(v)$.

Because the BDD is ordered, variable indices strictly increase along every path, so each variable is assigned at most once along any root-to-sink path.

To achieve the required complexity bound, the traversal is executed iteratively with an explicit stack of nodes in place of recursion. Each stack entry stores a node $v$ together with the next unexplored outgoing edge. Each edge of the BDD is pushed and popped at most a constant number of times, since once both outgoing edges of a node are explored, the node is never revisited.

During traversal, the algorithm maintains the current assignment array $x[1..n]$. When descending through a node $v$ with $V(v)=j$, the value $x_j$ is written exactly once per traversal of that edge. When backtracking, no restoration is required beyond restoring the previous stack state, since the next exploration of a sibling branch overwrites $x_j$.

When a sink node $\top$ is reached, the current array $(x_1,\dots,x_n)$ is output as one solution.

Since each solution corresponds to exactly one root-to-$\top$ path in the BDD, each solution is output exactly once.

## Verification

Each internal node of the BDD is processed a constant number of times, since each of its two outgoing edges is traversed once and then marked as exhausted in the stack discipline. Thus the total number of node-processing steps is proportional to $B(f)$.

Each edge traversal performs at most one assignment to an array entry $x_j$, and each such assignment is induced by a traversal of that edge. Since edges are traversed a constant number of times, the total number of assignments is $O(B(f))$.

Each solution corresponds to a distinct visit of $\top$, and each such visit produces exactly one output vector. Hence the total number of outputs is $N$.

All non-output work during traversal is bounded by a constant amount per node or per edge, and all output work is bounded by a constant amount per solution once the final assignment is formed along the root-to-$\top$ path.

Thus the total running time is

$$
O(B(f)) + O(N).
$$

This completes the proof. ∎
