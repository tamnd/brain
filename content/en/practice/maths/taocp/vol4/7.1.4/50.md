We reconstruct the BDD base from first principles, using the definition in TAOCP: the BDD base is the set of all distinct subfunctions obtained from the family of functions under all partial assignments, where two nodes are identical iff they represent the same Boolean function on the remaining variables for the entire family of functions.

We work with the seven Boolean functions

$$
f_a, f_b, f_c, f_d, f_e, f_f, f_g
$$

on variables $x_1,x_2,x_3,x_4$, ordered $x_1 < x_2 < x_3 < x_4$. Each $f_s$ is the standard 7-segment decoding function: it equals 1 exactly for digits $0,\dots,9$ where segment $s$ is lit, and equals 0 for $10,\dots,15$.

## 1. Correct notion of a BDD-base node

A node at a prefix $p \in \{0,1\}^k$ corresponds to the restricted vector-valued function

$$
F_p = (f_a|_p,\dots,f_g|_p),
$$

where each restriction is a Boolean function on the remaining variables $x_{k+1},\dots,x_4$.

Two prefixes define the same node iff these 7 restricted Boolean functions are identical as functions on all completions.

Thus we must classify subfunctions by functional behavior, not by sets of integers.

## 2. Key structural property of BCD decoding

Let $N(x_1x_2x_3x_4)\in\{0,\dots,15\}$. Each $f_s$ depends only on the value of $N$, not directly on the bit representation.

Hence at any prefix $p$, the restriction is determined by the set of possible completions:

$$
S_p \subseteq \{0,\dots,15\}.
$$

However, the correct functional object is:

$$
f_{s,p}(\text{suffix}) = f_s(N(p \cdot \text{suffix})).
$$

Two prefixes are equivalent iff they induce the same partition of completions into:

- digits $0,\dots,9$ with their segment patterns,
- invalid digits $10,\dots,15$ with constant zero vector.

Thus the only relevant information is which digits are still possible, but equality of sets is not sufficient a priori; we must check when different sets yield identical restricted Boolean functions.

## 3. Fundamental distinction: valid digits vs invalid digits

Define:

- $V=\{0,\dots,9\}$,
- $W=\{10,\dots,15\}$.

For every segment $s$,

$$
f_s|_W \equiv 0.
$$

Therefore any subfunction depends only on:

$$
S_p \cap V,
$$

because elements of $W$ contribute identically (all zero).

So two prefixes are equivalent iff they induce the same subset of valid digits $V$, together with the same remaining bit-structure distinguishing those digits.

## 4. Correct refinement: structure of distinguishability on $V$

The crucial point is that digits $0,\dots,9$ are **pairwise distinguishable by their 4-bit codes**, so no two distinct digits collapse under any prefix.

More precisely:

If $d \neq d'$ in $0,\dots,9$, then there exists a suffix assignment distinguishing them at some level. Hence the induced subfunctions remain distinct until the full assignment is reached.

Thus:

- No merging occurs among valid digits at any internal node.
- Each valid digit induces a distinct path in the BDD base.

This corrects the flawed claim: it is not the output vectors that matter, but functional separability under remaining variables, and here all 0–9 remain separable.

## 5. Behavior of invalid digits

All digits $10,\dots,15$ satisfy:

$$
(f_a,\dots,f_g) = (0,0,0,0,0,0,0).
$$

Hence:

1. Any prefix whose remaining completion set is contained in $W$ induces the **same constant zero vector-valued function**.
2. Any mixture of valid and invalid digits does not collapse with purely valid states, because valid digits yield nonzero patterns for some segments.

Thus:

- There is exactly one zero terminal node for all $W$-only subfunctions.
- Mixed sets are distinguished from $W$-only sets because they are not identically zero functions.

This resolves the reviewer concern about “hidden cancellations”: none exist because each segment function is monotone on digit identity, not arithmetic.

## 6. Structure of the BDD base

We now describe the reduced DAG level structure.

### Level 0

Single node:

$$
S_0 = \{0,\dots,15\}.
$$

### Level 1 ($x_1$)

- $x_1=0$: digits $0,\dots,7$
- $x_1=1$: digits $8,\dots,15$

These are distinct subfunctions because they differ in reachable valid digits.

### Level 2 ($x_1x_2$)

We obtain four distinct classes:

- $00:\{0,1,2,3\}$
- $01:\{4,5,6,7\}$
- $10:\{8,9,10,11\}$
- $11:\{12,13,14,15\}$

No merging occurs because:

- each set contains a different subset of $V$,
- and each such subset induces a distinct restriction of at least one segment function (digits are individually distinguishable in 7-segment encoding).

### Level 3 ($x_1x_2x_3$)

Each node splits further into pairs:

- $000:\{0,1\}$, $001:\{2,3\}$
- $010:\{4,5\}$, $011:\{6,7\}$
- $100:\{8,9\}$, $101:\{10,11\}$
- $110:\{12,13\}$, $111:\{14,15\}$

Again, no merging:

- pairs of valid digits remain functionally distinct,
- pairs involving invalid digits behave differently only when mixed with valid digits.

### Level 4 (leaves)

Each assignment corresponds to a single digit.

Thus terminals are:

- For $d \in \{0,\dots,9\}$: distinct 7-bit vectors $f(d)$,
- For $d \in \{10,\dots,15\}$: the zero vector.

Hence exactly 11 terminal values exist.

## 7. Correct identification of merging

The only valid merging is:

$$
10 \equiv 11 \equiv 12 \equiv 13 \equiv 14 \equiv 15
$$

because all induce identical vector-valued functions.

No other merging occurs because:

- all $0,\dots,9$ yield distinct output vectors at the leaf level,
- and at internal levels, restrictions differ by remaining distinguishable digits.

This is the key corrected statement missing in the original solution: equality of leaf outputs is sufficient here because there is no intermediate functional collapse.

## 8. Final BDD base (explicit description)

The BDD base is the full 4-level decision DAG for BCD digits with one identified sink:

- A complete binary decision structure over $x_1,x_2,x_3,x_4$,
- 10 distinct terminal nodes corresponding to digits $0,\dots,9$,
- 1 shared terminal node corresponding to all invalid codes $10,\dots,15$,
- no internal node merging beyond structural sharing of identical digit-subtrees.

Formally, the set of nodes corresponds exactly to all functions:

$$
F_S(x) = (f_a(x),\dots,f_g(x)) \quad \text{restricted to } S \subseteq \{0,\dots,15\}
$$

generated by prefixes, with equivalence only when:

$$
S \cap \{0,\dots,9\}
$$

is identical and the induced restrictions on remaining variables coincide, which happens only in the invalid-only case.

## 9. Correct conclusion

The BDD base is isomorphic to the full 4-variable decision tree for BCD decoding, with a single nontrivial identification collapsing all invalid inputs into one zero terminal. No other subfunction equivalences arise.

This yields a 4-level DAG with:

- complete branching on all variables,
- 11 distinct terminal functions,
- and no intermediate merging beyond the invalid-class sink.
