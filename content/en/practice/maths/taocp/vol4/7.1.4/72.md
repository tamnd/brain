## Solution

Algorithm S uses a unique-table mechanism to ensure that no two distinct nodes share the same triple $(V,LO,HI)$. In the original formulation this lookup is implemented by hashing. To eliminate hashing, the unique-table must be replaced by a deterministic search structure that supports membership testing and insertion on the key $(V,LO,HI)$.

For each variable index $V$, introduce a separate search structure $T[V]$ containing all nodes $k_j$ with $V(k_j)=V$. Each $T[V]$ stores records keyed by the pair $(LO,HI)$, since $V$ is fixed within the table. The entries in $T[V]$ are maintained in lexicographic order of $(LO,HI)$, where pointers are compared by their underlying node identifiers.

When Algorithm S constructs a candidate node with fields $(V,LO,HI)$, it performs a search in $T[V]$ for a record whose key equals $(LO,HI)$. If such a record exists, the algorithm returns the associated node pointer and discards the candidate. If no such record exists, a new node $k_j$ is created, inserted into $T[V]$ at the appropriate position to preserve order, and its pointer is returned.

The structure $T[V]$ may be implemented either as a balanced binary search tree or as a dynamically maintained sorted array with binary search and logarithmic insertion. In both cases, the search procedure examines exactly those previously constructed nodes with the same variable index $V$, and it determines equality of triples by equality of $(LO,HI)$ together with equality of $V$.

Correctness follows from the invariant that each table $T[V]$ contains at most one entry for each pair $(LO,HI)$. When a node is created, it is inserted only if no identical pair already exists in $T[V]$, so no duplicates can be introduced. Conversely, whenever a node with a given triple is required later in the construction, the search in $T[V]$ necessarily finds it, since the pair $(LO,HI)$ uniquely determines membership in the table at level $V$. This preserves the reduction condition of BDDs that forbids distinct nodes with identical $(V,LO,HI)$.

Ordering of the BDD is unaffected, since Algorithm S already guarantees that pointers in $LO$ and $HI$ reference nodes from strictly higher-index levels. Hence all comparisons and searches in $T[V]$ are well-defined at the time of insertion.

This replaces all hash-based accesses in Algorithm S by deterministic searches in ordered per-level structures while preserving the uniqueness and reduced properties required for BDD construction. ∎
