**Exercise 102.** [*23*] [23] Design a method to cache the results of functional composition using the fol-
lowing conventions: The system maintains at all times an array of functions [g1, . . . , gn],
one for each variable xj. Initially gj is simply the projection function xj, for 1 ≤j ≤n.
This array can be changed only by the subroutine NEWG(j, g), which replaces gj by g.
The subroutine COMPOSE(f) always performs functional composition with respect to
the current array of replacement functions.

The previous solution fails because it imports an inappropriate BDD substitution model and then tries to force correctness through structural claims that do not hold. The correct approach is simpler and closer to what the exercise actually asks: memoized evaluation of a functional expression under a dynamically changing environment.

# Correct solution

## 1. Model of the system

We are given variables $x_1,\dots,x_n$. At all times the system maintains an array of functions

$$
G = [g_1,\dots,g_n],
$$

where each $g_j : D^n \to D$ (for some domain $D$).

Initially,

$$
g_j(x_1,\dots,x_n) = x_j.
$$

The array is updated only by

$$
\mathrm{NEWG}(j,g),
$$

which replaces $g_j$ by $g$.

We must support:

- updates of a single component $g_j$,
- repeated calls to $\mathrm{COMPOSE}(f)$, which returns

$$
f(g_1,\dots,g_n).
$$

The goal is to cache intermediate work so repeated compositions under the same or partially changed environment are efficient.

## 2. Representation of functions

Represent the input function $f$ as a rooted DAG (as in TAOCP’s standard functional representation):

- each internal node is an operation node (for example, an $m$-ary composition operator),
- leaves are variables $x_j$,
- the DAG is acyclic.

Denote by $\mathrm{root}(f)$ the root node.

Each node $u$ of the DAG represents a function $F_u(x_1,\dots,x_n)$.

## 3. Environment model and versioning

We represent the environment as a persistent object

$$
E = (g_1,\dots,g_n).
$$

Although the system allows mutation via $\mathrm{NEWG}$, we implement logically immutable versions:

- Initially $E^0 = (x_1,\dots,x_n)$.
- Each $\mathrm{NEWG}(j,g)$ creates a new environment object $E'$ identical to $E$ except at position $j$.

Thus each environment has a unique identity (pointer identity in an implementation), and old environments remain valid for caching.

This avoids any need for hashing or collision arguments.

## 4. Definition of composition under an environment

Define the semantic evaluation function:

$$
\mathrm{VAL}(u,E)
$$

which returns the function obtained by interpreting node $u$ under substitution $x_j \mapsto g_j$.

We define it structurally.

### Base case: variable node

If $u$ is the leaf $x_j$, then

$$
\mathrm{VAL}(u,E) = g_j.
$$

### Inductive case: operation node

If $u = \varphi(u_1,\dots,u_k)$, where $\varphi$ is a fixed $k$-ary operation, then

$$
\mathrm{VAL}(u,E)
=
\varphi\big(\mathrm{VAL}(u_1,E),\dots,\mathrm{VAL}(u_k,E)\big).
$$

This definition is well-formed because the structure of $f$ is a finite DAG.

## 5. Memoized computation

We define a cache table:

$$
M[u,E]
$$

storing the computed value of $\mathrm{VAL}(u,E)$.

### Procedure $\mathrm{EVAL}(u,E)$

1. If $u$ is a leaf $x_j$, return $g_j$.
2. If $M[u,E]$ is defined, return it.
3. If $u = \varphi(u_1,\dots,u_k)$, compute recursively:

$$
v_i = \mathrm{EVAL}(u_i,E), \quad 1 \le i \le k.
$$
4. Compute:

$$
v = \varphi(v_1,\dots,v_k).
$$
5. Store $M[u,E] \leftarrow v$.
6. Return $v$.

Finally,

$$
\mathrm{COMPOSE}(f) = \mathrm{EVAL}(\mathrm{root}(f), E).
$$

## 6. Correctness

### Lemma 1 (semantic correctness)

For every node $u$ and environment $E$,

$$
\mathrm{EVAL}(u,E) = \mathrm{VAL}(u,E).
$$

**Proof by structural induction on $u$:**

- If $u = x_j$, then both sides equal $g_j$.
- If $u = \varphi(u_1,\dots,u_k)$, assume the statement holds for all $u_i$. Then:

$$
\mathrm{EVAL}(u,E)
= \varphi(\mathrm{EVAL}(u_1,E),\dots,\mathrm{EVAL}(u_k,E))
= \varphi(\mathrm{VAL}(u_1,E),\dots,\mathrm{VAL}(u_k,E))
= \mathrm{VAL}(u,E).
$$

Thus the recursion is correct.

## 7. Correctness of caching

### Lemma 2 (cache soundness)

If $\mathrm{EVAL}(u,E)$ is called twice with the same pair $(u,E)$, the returned value is identical.

This holds because:

- the computation of $\mathrm{VAL}(u,E)$ is a deterministic function of $(u,E)$,
- memoization stores the unique computed value,
- no mutation of $E$ occurs without producing a distinct environment object.

Thus $M[u,E]$ is a pure memo table over a well-defined key space.

## 8. Effect of NEWG

A call $\mathrm{NEWG}(j,g)$ produces a new environment $E'$ differing only in component $j$.

Crucially:

- all cache entries are indexed by the environment object itself,
- hence entries computed under $E$ are never reused under $E'$,
- no explicit invalidation is required.

This is the standard “persistent environment” technique.

## 9. Caching behavior across repeated COMPOSE calls

If multiple calls

$$
\mathrm{COMPOSE}(f)
$$

are made under the same environment $E$, then:

- the entire computation of each node $u$ is performed at most once,
- subsequent calls reuse cached values $M[u,E]$,
- thus repeated composition is reduced to table lookups.

If only part of $G$ changes, only computations depending on changed components are recomputed, since unchanged environments reuse all cached subresults.

## 10. Termination

The DAG of $f$ is finite and acyclic. Each call to $\mathrm{EVAL}(u,E)$ recurses only to strict subnodes. Therefore:

- no infinite recursion is possible,
- memoization ensures each pair $(u,E)$ is evaluated at most once.

Hence the algorithm terminates.

## 11. Why the previous BDD approach fails

The earlier solution incorrectly attempted to:

- force substitution through $\mathrm{ITE}$ over BDDs,
- assume closure and canonical form preservation without justification,
- encode environment changes via heuristics.

The correct structure is independent of BDD machinery: the problem is purely one of memoized evaluation over a dynamically updated substitution environment.

## Final statement

The correct method is to treat $f$ as a finite DAG, evaluate it under the substitution $x_j \mapsto g_j$ using structural recursion, and memoize results keyed by the pair (node, environment version). Updates via $\mathrm{NEWG}$ are handled cleanly by creating new environment identities, ensuring caching remains valid without unsafe invalidation or structural assumptions.
