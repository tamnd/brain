The previous solution fails because it never uses the actual definition of $h_{\pi,n}$. The key correction is that $h_{\pi,n}$ is not an arbitrary Boolean function with some vague “recursive structure,” but a function obtained from a fixed underlying structure by permuting variable names. Once this is used, the optimization problem becomes a symmetry argument rather than a structural search.

## Step 1: Use the definition of $h_{\pi,n}$

In TAOCP §7.1.4, the function $h_{\pi,n}$ is obtained from a fixed Boolean function $h_n$ by permuting its arguments. Concretely, there exists a base function $h_n(x_1,\dots,x_n)$ such that

$$
h_{\pi,n}(x_1,\dots,x_n)
=
h_n(x_{\pi(1)}, x_{\pi(2)}, \dots, x_{\pi(n)}).
$$

Thus changing $\pi$ does not change the underlying logical structure of the function, it only renames variables.

## Step 2: What BDD size depends on

The size $B(h_{\pi,n})$ is the number of nodes in the reduced ordered BDD of $h_{\pi,n}$ with respect to the fixed variable ordering

$$
x_1, x_2, \dots, x_n.
$$

So the only effect of $\pi$ is that it changes which variable of the original function appears at each level of the BDD.

Equivalently, $h_{\pi,n}$ is the pullback of $h_n$ under a permutation of coordinates.

## Step 3: Key structural observation (isomorphism of BDDs)

Let $\sigma$ be any permutation of $\{1,\dots,n\}$. Define a relabeling of variables by

$$
x_i \mapsto x_{\sigma(i)}.
$$

This induces a bijection between subfunctions of $h_n$ and subfunctions of $h_{\pi,n}$, because fixing any set of variables in $h_{\pi,n}$ corresponds exactly to fixing a permuted set of variables in $h_n$.

More precisely:

- Every restriction of $h_{\pi,n}$ by assignments to $k$ variables corresponds to a restriction of $h_n$ by assignments to $k$ variables.
- The resulting residual functions are identical up to renaming of free variables.
- Therefore the directed acyclic graph obtained in the BDD construction is preserved up to isomorphism.

Hence the reduced ordered BDDs of all $h_{\pi,n}$ are isomorphic graphs.

## Step 4: Consequence for BDD size

Isomorphic reduced OBDDs have the same number of nodes. Therefore,

$$
B(h_{\pi,n}) = B(h_n)
\quad \text{for every permutation } \pi.
$$

In particular, for $n=100$,

$$
B(h_{\pi,100}) \text{ is independent of } \pi.
$$

## Step 5: Minimization

Since all permutations yield exactly the same BDD size, every permutation is optimal. There is no strict improvement possible by reordering variables.

Thus the minimum value of $B(h_{\pi,100})$ is achieved by all permutations $\pi$.

## Final answer

Every permutation of $\{1,2,\dots,100\}$ minimizes $B(h_{\pi,100})$. In particular, one valid minimizing choice is

$$
\boxed{\pi = (1,2,3,\dots,100)}.
$$

## Remarks on the flawed solution

The original argument failed because it:

- ignored that $h_{\pi,n}$ is defined by variable permutation of a fixed function,
- treated the problem as if $\pi$ changed the logical structure, not just labels,
- attempted a “structural simplicity” argument with no connection to the actual function definition.

Once the permutation-invariance is used correctly, the problem collapses to an isomorphism argument and the minimization becomes trivial.
