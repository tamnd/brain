**Exercise 24.** [*M22*] [M22] Suppose twelve weights (w12, w13, . . . , w89) have been assigned to the edges
of the grid in Fig. 22. Explain how to find a minimum spanning tree in that graph
(namely, a spanning tree whose edges have minimum total weight), by applying Algo-
rithm B to the BDD shown there.

## Solution

Let $G$ be the $3 \times 3$ grid graph whose vertices are ${1,2,\dots,9}$ arranged in the natural layout, and whose edges are the twelve grid edges with weights

$$
w_{12}, w_{13}, w_{23}, w_{45}, w_{46}, w_{56}, w_{78}, w_{79}, w_{89}, w_{14}, w_{25}, w_{36}, w_{47}, w_{58}, w_{69},
$$

with the obvious restriction to the twelve edges actually present in Fig. 22. Each edge corresponds to one decision variable in the BDD, representing whether that edge is included in the spanning subgraph.

The BDD in Fig. 22 represents exactly the family of spanning trees of $G$. Every root-to-$\top$ path encodes a subset of edges satisfying the spanning-tree constraints, and every spanning tree corresponds to exactly one such path because the BDD is reduced and ordered, so each edge variable is tested exactly once along any path.

Algorithm B assigns to each edge variable $x_e$ a contribution equal to its weight when the HI branch is taken, and zero when the LO branch is taken. For a node $v$, let $C(v)$ denote the minimum total weight among all paths from $v$ to $\top$. For the sink nodes, the values are

$$
C(\top)=0, \qquad C(\bot)=+\infty.
$$

For every branch node $v$ labeled by an edge variable $x_e$, with LO successor $v_0$ and HI successor $v_1$, Algorithm B evaluates

$$
C(v)=\min\bigl(C(v_0),\; w_e + C(v_1)\bigr).
$$

The first term corresponds to excluding edge $e$, and the second corresponds to including $e$ and adding its weight. Because the BDD is ordered, every edge is processed exactly once along any path, so the accumulated sum along a path is exactly the total weight of the corresponding edge set.

Processing the nodes in reverse topological order from sinks to root computes $C(v)$ for every node. At the root $r$, the value $C(r)$ equals the minimum total weight over all root-to-$\top$ paths, hence equals the minimum total weight of a spanning tree of $G$. The corresponding tree is obtained by selecting at each node $v$ the choice (LO or HI) that attains the minimum in the defining recurrence, and collecting all edges whose HI branch is chosen along the resulting path.

Because every root-to-$\top$ path encodes a spanning tree and every spanning tree appears as exactly one such path, the minimum over paths is exactly the minimum spanning tree weight. No spanning tree can have smaller weight than $C(r)$, since its path weight would contradict the definition of $C(r)$ as a global minimum over all feasible paths.

This completes the proof. ∎
