**Exercise 86.** &#9654; [*21*] [21] The routine MUX in (62) refers to “obvious” values. What are they?

**Verified:** yes

## Solution

The MUX operation in (62) is the Boolean choice function

$\operatorname{MUX}(f,g,h) = (f \land g) \lor (\neg f \land h).$

Equivalently, it returns $g$ when $f = \top$ and returns $h$ when $f = \bot$.

The “obvious values” in the routine are the cases where this definition collapses without recursion.

When $f = \bot$, substitution into the defining expression gives

$\operatorname{MUX}(\bot,g,h) = (\bot \land g) \lor (\top \land h) = \bot \lor h = h.$

When $f = \top$, substitution gives

$\operatorname{MUX}(\top,g,h) = (\top \land g) \lor (\bot \land h) = g \lor \bot = g.$

When $g = h$, substitution into the definition yields

$\operatorname{MUX}(f,g,g) = (f \land g) \lor (\neg f \land g) = (f \lor \neg f)\land g = \top \land g = g.$

When $g = \top$ and $h = \bot$, substitution yields

$\operatorname{MUX}(f,\top,\bot) = (f \land \top) \lor (\neg f \land \bot) = f \lor \bot = f.$

When $g = \bot$ and $h = \top$, substitution yields

$\operatorname{MUX}(f,\bot,\top) = (f \land \bot) \lor (\neg f \land \top) = \neg f.$

Each of these cases eliminates the need to construct a new BDD node because the result is already represented by one of the inputs or its complement. These identities exhaust the situations in which the defining Boolean expression reduces to a simpler form without further decomposition in the MUX routine. ∎
