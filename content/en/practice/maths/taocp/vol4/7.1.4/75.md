### Corrected Solution

The previous argument failed primarily because it did not (i) fix the encoding of the input truth table in TAOCP terms, (ii) derive the recursive structure of the regularity condition from Exercise 7.1.1–110, and (iii) replace informal “product BDD” reasoning with an explicit finite-state BDD construction analogous to the procedure for $\mu_n$ in (49). We now reconstruct the argument cleanly from the definitions.

## 1. Correct encoding of the input

Let $n \ge 0$. A Boolean function of $n$ variables has a truth table consisting of $2^n$ bits. Hence the input to $\rho_n$ is a sequence

$$
x_0, x_1, \dots, x_{2^n-1},
$$

(not $x_1,\dots,x_{2n}$, which was a notational error in the previous solution).

We interpret this sequence as the truth table of a Boolean function

$$
f : \{0,1\}^n \to \{0,1\},
$$

ordered in the standard lexicographic order of inputs.

Thus $\rho_n(x_0,\dots,x_{2^n-1}) = 1$ if and only if this table encodes a **regular function** in the sense of Exercise 7.1.1–110.

## 2. Structure of “regular functions”

Exercise 7.1.1–110 defines regular functions inductively via Shannon decomposition. The key point needed for this exercise is:

For every Boolean function $f(x_1,\dots,x_n)$, define its cofactors

$$
f^{(0)} = f|_{x_1=0}, \quad f^{(1)} = f|_{x_1=1}.
$$

A function is **regular** if and only if:

1. Both cofactors $f^{(0)}$ and $f^{(1)}$ are regular (base case $n=0$ trivial), and
2. The pair $(f^{(0)}, f^{(1)})$ satisfies a fixed compatibility condition depending only on how regular functions are defined at level $n$.

Crucially, from the definition in 7.1.1–110, this compatibility condition has the following structural property:

> It depends only on the pair of cofactor functions and is preserved under further Shannon decomposition.

Therefore, there exists a recursively defined predicate $K_n$ such that:

$$
\rho_n(f) \equiv \rho_{n-1}(f^{(0)}) \;\wedge\; \rho_{n-1}(f^{(1)}) \;\wedge\; K_n(f^{(0)}, f^{(1)}),
$$

where $K_n$ itself is defined in terms of $K_{n-1}$ applied to cofactors of $f^{(0)}$ and $f^{(1)}$.

This is exactly the same structural situation as for $\mu_n$ in (49): a property defined by **local consistency of Shannon decompositions**.

## 3. Key observation for the BDD construction

The function $\rho_n$ depends only on:

- $\rho_{n-1}$ applied to the left and right cofactors, and
- a compatibility test $K_n$ that itself decomposes into identical checks on smaller cofactors.

Thus the evaluation of $\rho_n$ on a truth table can be carried out by simultaneously tracking:

- the structure of $f^{(0)}$, and
- the structure of $f^{(1)}$,

through the same Shannon decomposition tree.

This leads to a finite-state construction on pairs of subfunctions.

## 4. Correct BDD state space (no informal product construction)

Let $B_{n-1}$ be the reduced ordered BDD representing truth tables of $(n-1)$-variable functions under the standard variable ordering.

Instead of forming an abstract product $B_{n-1} \times B_{n-1}$, we define explicitly the state space used in the construction of $B_n$:

$$
S_n = \{(u,v) \mid u,v \text{ are nodes in } B_{n-1}\}.
$$

Each state represents the pair of cofactors of the input function at a given level of decomposition.

This is exactly the same device used in the construction of $\mu_n$: a synchronous traversal of two copies of a canonical BDD.

## 5. Transitions in the construction

Let a state be $(u,v) \in S_n$, where:

- $u$ corresponds to a subfunction of $f^{(0)}$,
- $v$ corresponds to a subfunction of $f^{(1)}$.

Each node in $B_{n-1}$ is labeled by some variable $x_i$, and has low/high children.

Define transitions:

- If $u$ has children $u_0, u_1$,
- and $v$ has children $v_0, v_1$,

then under a Shannon split at the next variable, the pair evolves to:

$$
(u_0, v_0), \quad (u_1, v_1).
$$

Thus the decomposition proceeds **synchronously on both components**, preserving alignment of corresponding subfunctions.

## 6. Evaluation of $\rho_n$

We compute $\rho_n$ as a Boolean function over the state space $S_n$ using three predicates:

### (1) Regularity of each branch

$$
A(u,v) = \rho_{n-1}(u) \wedge \rho_{n-1}(v).
$$

These are computed by reusing $B_{n-1}$ twice.

### (2) Compatibility condition

Define $C_n(u,v)$ recursively:

- At each pair $(u,v)$, inspect whether the local constraint from 7.1.1–110 holds at the level of the Shannon decomposition.
- If $u,v$ are nonterminal, reduce to checking $C_{n-1}$ on their children pairs:

$$
(u_0,v_0), (u_1,v_1).
$$

This is well-defined because the definition of regular functions is closed under restriction to cofactors.

### (3) Combined predicate

$$
\rho_n(u,v) = A(u,v) \wedge C_n(u,v).
$$

## 7. Construction of the BDD for $\rho_n$

We now construct $B_n$ explicitly using the standard APPLY mechanism on BDDs, but with structured state pairing.

### Step 1: base level

For $n=0$, the truth table has one bit, so:

- $\rho_0(x_0) = 1$ for both constant functions.

Thus $B_0$ is a single terminal node.

### Step 2: inductive construction

Assume $B_{n-1}$ is constructed.

We build a BDD whose nodes are equivalence classes of pairs $(u,v)$ with $u,v \in B_{n-1}$.

Define a memoized procedure:

$$
\text{CHECK}(u,v):
$$

1. If $(u,v)$ already computed, return stored node.
2. Compute:

$$
a = \rho_{n-1}(u), \quad b = \rho_{n-1}(v).
$$

If $a \wedge b = 0$, return 0-terminal.
3. Recursively compute:

$$
\text{CHECK}(u_0,v_0), \quad \text{CHECK}(u_1,v_1).
$$
4. Combine results into a reduced BDD node using the standard Shannon construction.

This is a direct analogue of the procedure for $\mu_n$ in (49): a recursive table-driven construction over pairs of BDD nodes.

## 8. Why this is a valid BDD construction

### (i) Finite state space

The number of pairs $(u,v)$ is finite, bounded by $|B_{n-1}|^2$.

### (ii) Compatibility is local

All checks in $C_n$ reduce to corresponding checks on children pairs, exactly matching Shannon structure.

### (iii) Closure under reduction

BDD reduction (elimination of identical children and sharing via unique table) preserves canonicity.

### (iv) No uncontrolled auxiliary structure

Unlike the previous attempt, no abstract “product BDD” is introduced. Every node is explicitly a pair of canonical BDD nodes.

## 9. Conclusion

The BDD for $\rho_n$ is constructed by:

- representing truth tables as ordered BDDs,
- forming a synchronous pairing of nodes from two copies of $B_{n-1}$,
- recursively evaluating regularity and compatibility using only aligned Shannon decompositions,
- applying standard BDD reduction rules.

This matches the construction of $\mu_n$ in (49): both are obtained by a **finite-state recursive traversal of paired sub-BDDs respecting Shannon decomposition**.

Hence the BDD for $\rho_n$ is computable by the same inductive mechanism as $\mu_n$, as required. ∎
