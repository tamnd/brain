**Exercise 113.** [*20*] [20] Instead of having two sink nodes, one for each Boolean constant, we could
have 216 sinks, one for each Boolean function of four variables. Then a BDD could stop
four levels earlier, after branching on xn−4. Would this be a good idea?

## Solution

Let $f(x_1,\ldots,x_n)$ be represented by an ordered reduced BDD as defined in Section 7.1.4, with two sink nodes $\bot$ and $\top$. The proposal replaces these sinks by $2^{16}$ distinct sink nodes, one for each Boolean function of four variables, and allows the BDD to terminate after branching only down to level $x_{n-4}$.

A sink at depth $n-4$ would have to represent the residual subfunction

$g(x_{n-3},x_{n-2},x_{n-1},x_n)$

induced by a partial assignment to $(x_1,\ldots,x_{n-4})$. Such a sink must therefore encode the truth table of an arbitrary Boolean function of four variables, since $g$ is an unrestricted Boolean function on four inputs. There are exactly $2^{2^4} = 2^{16}$ such functions, which matches the proposed number of sinks.

The structural effect of this modification is that every path in the decision diagram is shortened by four levels, since no branching is performed on $x_{n-3},x_{n-2},x_{n-1},x_n$.

However, the representation now moves all combinatorial complexity from internal nodes into the sink layer. Each sink is no longer a constant value in ${\bot,\top}$ but a full truth table of size $16$. To evaluate the represented Boolean function on a full input $(x_1,\ldots,x_n)$, reaching a sink after processing $x_1,\ldots,x_{n-4}$ still requires evaluating the stored 4-variable function on $(x_{n-3},x_{n-2},x_{n-1},x_n)$, so the terminal is not an atomic value in the sense used for $\bot$ and $\top$.

The canonicity argument for standard BDDs relies on the fact that each node represents a unique subfunction and that equality of subfunctions is identified by node sharing with constant-time terminal comparison. With $2^{16}$ terminal types, equality of sinks is equality of 16-bit truth tables, and thus no longer a constant-time structural identity unless an additional normalization or hashing mechanism is assumed. This breaks the simplicity of reduction rules, since sink comparison is no longer trivial.
