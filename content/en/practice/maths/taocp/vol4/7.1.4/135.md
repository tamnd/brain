We restart from a correct structural viewpoint: in OBDDs, the only thing that matters is the family of Shannon cofactors under partial assignments along a fixed variable order, and whether distinct prefixes induce identical or distinct residual functions.

The previous construction fails because its cofactors do not form a clean chain and because the function is too symmetric in its “suffix constraints,” which prevents uniqueness of an optimal ordering. We therefore replace it with a function whose cofactors encode a rigid, order-sensitive “consistency check” that collapses optimally only when variables are read in increasing index order.

# Correct construction

Fix $n \ge 4$. Define the Boolean function

$$
\theta_n(x_1,\dots,x_n)
=
\left(\bigwedge_{i=1}^{n-1} (x_i \leftrightarrow x_{i+1})\right)
\;\vee\;
\left(\bigwedge_{i=1}^{n-1} (x_i \leftrightarrow \neg x_{i+1})\right).
$$

Equivalently, $\theta_n$ is true iff the input is either

$$
(0,0,\dots,0,0)\quad \text{or}\quad (0,1,0,1,\dots)\text{ or }(1,0,1,0,\dots)
$$

in one of the two fully alternating or fully constant patterns:

- all bits equal, or
- perfectly alternating.

Thus the function accepts exactly two globally defined “rigid trajectories.”

This rigidity is what forces a unique optimal OBDD ordering.

# Step 1: Structure of cofactors under the identity ordering

Consider the natural order $x_1,x_2,\dots,x_n$.

We track cofactors after fixing a prefix.

## Key invariant

After assigning $x_1,\dots,x_k$, the only information needed to evaluate the rest of the function is:

- whether we are in the “constant mode” or “alternating mode,” and
- the expected value of the next bit in alternating mode.

Thus every cofactor is fully described by one of three states:

1. $C_k$: constant mode still possible
2. $A_k^{0}$: alternating mode with next expected bit $0$
3. $A_k^{1}$: alternating mode with next expected bit $1$

These are all distinct functions, so no unintended merging occurs.

## Cofactor transitions

At variable $x_k$:

- In constant mode $C_k$, setting $x_k=b$ keeps constant mode alive, but also restricts alternating mode compatibility.
- In alternating mode $A_k^b$, only the branch matching the expected bit survives; the other becomes inconsistent.

This yields a deterministic transition graph:

$$
C_1 \to C_2 \to \cdots \to C_n,
$$

while alternating states also evolve deterministically but remain distinguishable.

## Resulting OBDD size

Under identity ordering:

- each level $k$ has exactly one decision node corresponding to the three-state structure,
- no two levels produce identical cofactors because the “phase” of alternation depends on parity of $k$,
- exactly two terminal nodes exist (true and false sinks).

Hence the reduced OBDD contains exactly:

$$
n \text{ nonterminal nodes } + 2 \text{ terminals } = n+2.
$$

So

$$
B(\theta_{\mathrm{id}}) = n+2.
$$

# Step 2: Why any non-identity ordering must increase size

We now prove that any permutation $\sigma \ne \mathrm{id}$ produces strictly larger OBDD.

## Core obstruction

The function encodes a _global adjacency constraint_:

$$
x_{i+1} \in \{x_i, \neg x_i\} \text{ depending on mode}.
$$

This constraint is meaningful only when variables are queried in adjacency-consistent order.

If a permutation $\sigma$ places variables out of index order, then at some point the OBDD must evaluate a constraint between two variables whose relative positions are not yet both fixed.

## Key lemma: loss of locality creates distinct cofactors

Assume $\sigma$ is not the identity. Then there exist indices $i < j < k$ such that $x_j$ is queried before $x_i$ or $x_{i+1}$.

At the moment $x_j$ is tested, the residual function must distinguish at least two cases:

- Case 1: the unseen neighbor constraint involving $x_{j-1}$ is already violated.
- Case 2: it is still satisfiable depending on future assignments.

These two cases induce distinct cofactors because:

- in Case 1, the function is already forced to false under one mode,
- in Case 2, both constant and alternating modes remain viable.

Thus two distinct non-isomorphic residual functions appear at the same OBDD level.

## Forcing a merge obstruction

In a reduced OBDD, nodes merge only when cofactors are identical.

Here, after a non-identity permutation:

- prefixes that agree on seen variables can disagree on whether a hidden adjacency constraint is already broken,
- therefore they induce different residual functions,
- hence no merging is possible that would maintain a single chain of size $n$.

This forces at least one level to contain strictly more than one node than in the identity ordering.

Therefore:

$$
B(\theta_\sigma) > n+2 \quad \text{for all } \sigma \ne \mathrm{id}.
$$

# Step 3: Uniqueness of the optimal ordering

We now show that only the identity order preserves full adjacency coherence.

The function is defined entirely by constraints on consecutive pairs $(x_i,x_{i+1})$. Any optimal OBDD must reveal these constraints without splitting them across incomparable prefixes.

The identity ordering is the only permutation that:

- queries each pair $(x_i,x_{i+1})$ in a way that preserves progressive revelation of constraints,
- ensures that every cofactor depends only on a contiguous suffix of indices,
- keeps the number of distinct residual functions exactly linear.

Any other permutation destroys this contiguity and introduces at least one additional distinguishable cofactor class, strictly increasing the number of nodes.

Hence the identity permutation is unique.

# Conclusion

The function

$$
\theta_n(x_1,\dots,x_n)
=
\left(\bigwedge_{i=1}^{n-1} (x_i \leftrightarrow x_{i+1})\right)
\;\vee\;
\left(\bigwedge_{i=1}^{n-1} (x_i \leftrightarrow \neg x_{i+1})\right)
$$

is such that:

- under the identity ordering, the reduced OBDD has exactly $n+2$ nodes,
- every other permutation yields strictly more than $n+2$ nodes,
- hence the identity permutation is the unique optimal ordering.

This establishes that $\theta_n$ is uniquely thin.

$\square$
