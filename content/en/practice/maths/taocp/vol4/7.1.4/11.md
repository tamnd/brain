[Section 7.1.4: Binary Decision Diagrams](../)

**Exercise 11.** [*20*] [20] Does Algorithm C give the correct answer if it is applied to a binary decision
diagram that is (a) ordered but not reduced? (b) reduced but not ordered?

**Verified:** yes  
**Solve time:** 1m13s  

---

## Solution

Algorithm C for binary decision diagrams is designed under the assumption that each Boolean function is represented by a canonical structure. In particular, the argument in Section 7.1.4 shows that reduced ordered BDDs (ROBDDs) give a one-to-one correspondence between Boolean functions and diagram structure, so equality of functions can be tested by structural identity of their representations.

Let $G(f)$ denote the diagram produced for a function $f$.

The correctness of Algorithm C depends on the property that

$$
G(f) = G(g) \quad \Longleftrightarrow \quad f = g.
$$

This equivalence holds only when the representation is both ordered and reduced.

If the BDD is ordered but not reduced, distinct nodes may represent identical subfunctions, since reduction forbids two distinct nodes with the same triple $(V, LO, HI)$. Without this restriction, the same Boolean function may admit multiple distinct directed acyclic graph representations even under a fixed variable ordering. In that case, two identical functions $f$ and $g$ can yield structurally different diagrams because redundant nodes may be inserted in one construction but not the other. Algorithm C, which relies on structural comparison of nodes rather than semantic evaluation over all inputs, may therefore return inequality for equal functions. Hence it does not always give the correct answer in case (a).

If the BDD is reduced but not ordered, then repeated variables along a root-to-sink path are allowed, since the restriction $i < j$ for edges from node $k_i$ to node $k_j$ is violated. In this situation, the same Boolean function can be represented by many different DAG structures depending on where variable tests are placed along paths. Even if reduction merges identical triples, the absence of a fixed variable ordering destroys the canonical form property. Consequently, two equivalent functions can again yield non-isomorphic reduced graphs, so Algorithm C may incorrectly report inequality. Hence it does not give the correct answer in case (b).

Since correctness of Algorithm C relies on both uniqueness of structure (ensured by reduction) and consistency of variable ordering (ensured by ordering), failure of either condition breaks the equivalence between functional equality and structural identity.

This completes the proof. ∎
