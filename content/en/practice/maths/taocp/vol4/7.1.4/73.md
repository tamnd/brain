**Exercise 73.** &#9654; [*25*] [25] Discuss the use of “virtual addresses” instead of actual addresses as the links
of a BDD: Each pointer p has the form π(p)2e + σ(p), where π(p) = p ≫e is p’s “page”
and σ(p) = p mod 2e is p’s “slot”; the parameter e can be chosen for convenience. Show
that, with this approach, only two fields (LO, HI) are needed in BDD nodes, because
the variable identifier V (p) can be deduced from the virtual address p itself.

A correct argument must define a _virtual-address representation scheme_ in which the variable index is recoverable from the address itself, without assuming any particular physical memory layout of BDD nodes.

## Corrected Solution

Let BDD nodes be identified by _virtual addresses_ $p$. Fix a parameter $e \ge 0$, and decompose every address as

$$
p = \pi(p)\,2^e + \sigma(p),
$$

where

$$
\pi(p) = p \gg e, \qquad \sigma(p) = p \bmod 2^e.
$$

Thus $\sigma(p)$ selects one of $2^e$ slots inside a page, and $\pi(p)$ is the page number.

### 1. Construction of the virtual-address layout

We define the virtual memory layout explicitly so that pages encode BDD variable indices.

For each BDD variable $x_j$, allocate a contiguous block of page numbers

$$
B_j = \{ j \cdot K, j \cdot K + 1, \dots, j \cdot K + K - 1 \},
$$

where $K$ is a fixed constant large enough that all nodes of level $j$ can be placed in $K$ pages if needed.

Each node associated with variable $x_j$ is stored in some page $b \in B_j$. The allocator ensures that only nodes of level $j$ are placed in pages belonging to $B_j$. Inside each page, nodes are assigned slots $0,1,\dots,2^e-1$ via $\sigma(p)$.

This scheme is part of the _definition of the virtual addressing representation_, not a property of general memory allocation.

### 2. Recovery of the variable index from the address

From the construction, every page number $\pi(p)$ lies in exactly one block $B_j$. Hence the mapping

$$
V(p) = j \quad \Longleftrightarrow \quad \pi(p) \in B_j
$$

is well-defined and unique, because the blocks $B_j$ are disjoint.

Equivalently, define a purely arithmetic recovery function

$$
V(p) = \left\lfloor \frac{\pi(p)}{K} \right\rfloor.
$$

Since $\pi(p)$ is obtained directly from $p$, the variable index $V(p)$ is determined entirely by the virtual address $p$.

### 3. Independence from node representation

Each BDD node stored at address $p$ contains only its two outgoing links:

$$
(\mathrm{LO}(p), \mathrm{HI}(p)).
$$

No explicit variable field is stored.

The variable index is recoverable from the address via

$$
V(p) = \left\lfloor \frac{p \gg e}{K} \right\rfloor,
$$

so it is not stored inside the node itself.

### 4. Consistency with BDD structure

In a BDD, each node is associated with a fixed variable index determined by its level in the decision diagram. The construction above respects this invariant by ensuring that all nodes created at level $j$ are allocated only within the address region corresponding to $x_j$. Thus the addressing scheme is consistent with the BDD ordering constraint.

## Conclusion

By encoding the variable index into disjoint ranges of page numbers in the virtual address space, the function $V(p)$ becomes computable from $p$ alone. Therefore each node requires only two stored fields, $(\mathrm{LO}, \mathrm{HI})$, since the variable index is implicit in the address. ∎
