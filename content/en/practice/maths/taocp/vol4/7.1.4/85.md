[Section 7.1.4: Binary Decision Diagrams](../)

**Exercise 85.** [*16*] [16] Compare the size of a BDD base for the 32 functions of 16×16-bit binary mul-
tiplication with the alternative of just storing a complete table of all possible products.

**Verified:** yes  
**Solve time:** 1m13s  

---

## Solution

Let the inputs be two 16-bit numbers

$x = (x_{15}\ldots x_0)_2,\qquad y = (y_{15}\ldots y_0)_2,$

so the multiplication function produces a 32-bit output

$z = x\cdot y = (z_{31}\ldots z_0)_2.$

Each output bit $z_k$ is a Boolean function of the $32$ variables $x_0,\ldots,x_{15},y_0,\ldots,y_{15}$. Hence the BDD base for multiplication consists of $32$ Boolean functions.

A complete truth-table representation for multiplication stores the value of $x\cdot y$ for every pair $(x,y)$. Since there are $2^{16}$ choices for $x$ and $2^{16}$ choices for $y$, there are

$2^{16}\cdot 2^{16} = 2^{32}$

input pairs. Each product is a 32-bit word, so the full table requires

$32\cdot 2^{32}$

bits of storage.

For the BDD representation, each output bit $z_k$ can be expressed using the standard binary multiplication identity

$z_k = \bigoplus_{i+j=k} (x_i \wedge y_j) \oplus c_k,$

where $c_k$ is the carry propagated from lower positions. The structure of the carry satisfies a linear recurrence built from full-adders, so each carry bit is a Boolean function of bounded fan-in depending only on lower-index bits.

The BDD for a full-adder (sum and carry of three inputs) has constant size under a fixed variable ordering that interleaves bits of $x$ and $y$. The carry propagation through $n=16$ bit positions therefore builds a chain of $O(n)$ levels, and each level introduces only a constant number of new BDD nodes up to sharing of identical subfunctions. The partial products $x_i \wedge y_j$ contribute $O(n^2)$ distinct conjunctions, but each such term feeds into the same addition-and-carry structure, so the total number of distinct subfunctions remains $O(n^2)$ for each output bit.

Hence each $z_k$ has a BDD of size bounded by $O(n^2)$ nodes. With $n=16$, this yields a bound on the order of $O(256)$ nodes per output function. The BDD base shares identical subfunctions among all $z_k$, since the same partial products and carry structures reappear across bit positions shifted in index. Therefore the total size of the base remains $O(n^2)$ rather than $32\cdot O(n^2)$.

The full BDD base therefore requires $O(16^2)=O(256)$ to $O(10^3)$ nodes in typical reduced form, whereas the truth-table representation requires $32\cdot 2^{32}$ bits.

The ratio of storage scales as

$\frac{32\cdot 2^{32}}{O(n^2)} = \Theta\!\left(\frac{2^{32}}{n^2}\right),$

which is exponential in the input width.

Thus the BDD base is polynomial in $n$, while the complete product table is exponential in $2n$.

$$
\boxed{\text{BDD base size } = O(n^2)\ \text{versus full table } = \Theta(2^{2n+5})\ \text{bits}}
$$

This completes the comparison. ∎
