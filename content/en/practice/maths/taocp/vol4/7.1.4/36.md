### Corrected Solution

Let $F$ be an FBDD with internal nodes $k$, each labeled by a variable $x_{V(k)}$, with low and high successors $\mathrm{LO}(k)$ and $\mathrm{HI}(k)$, and sinks $\bot,\top$. Along every root-to-sink path, no variable is repeated.

Let $\circ$ and $\bullet$ be binary operations on a set $A$ with distinguished elements $\bot,\top \in A$, such that:

- $\circ,\bullet$ are associative and commutative,
- $\bullet$ distributes over $\circ$,
- $\circ$ distributes over $\bullet$,
- $\bot$ is the identity for $\circ$,
- $\top$ is the identity for $\bullet$.

We compute an **elaborated truth table** as a canonical algebraic object in the free expression system generated from variables $x_i$, their negations $\neg x_i$, and constants $\bot,\top$, using operations $\circ,\bullet$.

### 1. Elaboration as path expansion

For any root-to-sink path $P$ in the FBDD, define its **path term**:

- For each node labeled $x_i$, if the path goes to $\mathrm{LO}$, include $\neg x_i$; if it goes to $\mathrm{HI}$, include $x_i$, combined by $\bullet$.
- Let the terminal sink value be $t(P)\in\{\bot,\top\}$.
- Define the path contribution

$$
T(P) \;=\; \Bigl(\mathop{\bullet}_{v \in P} \ell(v)\Bigr)\bullet t(P),
$$

where $\ell(v)\in\{x_i,\neg x_i\}$.

The **elaborated truth table of node $k$** is the aggregation over all such paths starting at $k$:

$$
E(k) \;=\; \bigcirc_{P \in \mathcal{P}(k)} T(P),
$$

where $\mathcal{P}(k)$ is the set of all root-to-sink paths starting at $k$.

Because $\circ$ is commutative and associative, this is well defined independently of the order in which paths are enumerated.

### 2. Structural reduction using FBDD decomposition

We now show how to compute $E(k)$ without enumerating paths.

Fix an internal node $k$ labeled $x_j$. Every path from $k$ begins with a choice of low or high edge, hence partitions as:

$$
\mathcal{P}(k) \;=\; \mathcal{P}_{0}(k)\;\cup\;\mathcal{P}_{1}(k),
$$

where paths in $\mathcal{P}_0(k)$ take the low edge and those in $\mathcal{P}_1(k)$ take the high edge.

For any path in $\mathcal{P}_0(k)$, the first step contributes $\neg x_j$ via $\bullet$, followed by a path in $\mathcal{P}(\mathrm{LO}(k))$. Hence:

$$
\bigcirc_{P \in \mathcal{P}_0(k)} T(P)
=
(\neg x_j \bullet E(\mathrm{LO}(k))).
$$

Similarly,

$$
\bigcirc_{P \in \mathcal{P}_1(k)} T(P)
=
(x_j \bullet E(\mathrm{HI}(k))).
$$

Combining both classes of paths and using commutativity and associativity of $\circ$, we obtain:

$$
E(k)
=
(\neg x_j \bullet E(\mathrm{LO}(k))) \circ (x_j \bullet E(\mathrm{HI}(k))).
$$

For sinks:

$$
E(\bot)=\bot,\qquad E(\top)=\top.
$$

### 3. Why distributivity is essential

The crucial point is that different paths may encounter variables in different relative orders in an FBDD. Without algebraic laws, expressions corresponding to different paths would depend on these orders.

However:

- Commutativity and associativity of $\bullet$ allow all literals along a path to be combined into a single product independent of order.
- Commutativity and associativity of $\circ$ allow aggregation over all paths without ordering.
- Distributivity ensures that when paths merge at shared subgraphs, we can factor and reuse subexpressions:

$$
(a \bullet b) \circ (a \bullet c) = a \bullet (b \circ c),
$$

enabling sharing of subcomputations in the DAG.

This is exactly what permits a canonical “elaborated” form rather than a path-dependent one.

### 4. Bottom-up computation on the FBDD

Because the FBDD is acyclic, compute $E(k)$ for all nodes in reverse topological order:

1. For sinks, assign $E(\bot)=\bot$, $E(\top)=\top$.
2. For each internal node $k$, once $E(\mathrm{LO}(k))$ and $E(\mathrm{HI}(k))$ are known, compute

$$
E(k)
=
(\neg x_{V(k)} \bullet E(\mathrm{LO}(k)))
\circ
(x_{V(k)} \bullet E(\mathrm{HI}(k))).
$$

Each node is processed once, and shared subgraphs are reused, so the computation depends only on the DAG structure, not on the exponential set of paths.

### 5. Role of commutativity in FBDDs (key extension over BDDs)

In a BDD, variables appear in a fixed order along all paths. In an FBDD, different paths may encounter variables in different orders or omit variables entirely.

Commutativity of $\bullet$ ensures that within each path term, the order of encountered literals is irrelevant. Hence every path contributes a well-defined product of literals independent of traversal order.

Distributivity then guarantees that merging such unordered path contributions through shared subgraphs yields the same global expression regardless of how paths are grouped.

Thus the elaborated truth table is well defined even without a global variable ordering.

### 6. Conclusion

The elaborated truth table of the FBDD is the algebraic expression $E(\mathrm{root})$ computed by the above bottom-up recurrence. It represents the sum (via $\circ$) of all path contributions, where each contribution is the product (via $\bullet$) of literals along the path and the terminal value.

The commutative, associative, and distributive laws ensure:

- path terms are order-independent,
- merging at shared nodes is well defined,
- bottom-up DAG evaluation produces a canonical result.

This yields an efficient computation of the elaborated truth table for any FBDD, extending the BDD method of Exercise 31. ∎
