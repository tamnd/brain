## Corrected Solution

The previous attempt failed mainly because it treated the QDD as invariant under reordering and replaced the structural computation of the master profile chart with an exponential enumeration over permutations. We restart from the correct TAOCP model.

Throughout, let $f(x_1,\dots,x_n)$ be represented by a QDD built for a fixed variable ordering $\pi$. The key fact is:

> A QDD encodes all cofactors of $f$ induced by assigning prefixes of $\pi$, and every node corresponds to a distinct cofactor under that fixed ordering.

The master profile chart counts how many distinct subfunctions appear at each depth when variables are eliminated in different ways consistent with reordering. The essential difficulty is to reuse the fixed QDD to simulate other orderings without rebuilding diagrams.

# (a) Computing the $\binom{n+1}{2}$ weights from one QDD

Fix the given QDD $Q$ for ordering $\pi$. For any node $v$, let $f_v$ denote the subfunction represented at $v$.

The master profile chart entry $P(i,k)$ is defined as:

$$
P(i,k) = \text{number of distinct nodes that correspond to cofactors obtained after fixing } i \text{ variables and leaving } k \text{ live variables}.
$$

In a QDD, “live variables” at a node are not abstractly the support size of a function but the number of variables that still appear below that node in the decision diagram under the fixed ordering.

Define for each node $v$:

- $\mathrm{level}(v)$: depth of $v$ in the QDD (number of variables tested on any path from the root to $v$),
- $\mathrm{low}(v)$: the index of the deepest variable that can still influence $f_v$, i.e., the largest level appearing in the subgraph rooted at $v$.

Then the number of remaining variables at $v$ is:

$$
k(v) = n - \mathrm{low}(v).
$$

This quantity is well-defined in a QDD because all paths respect the same ordering, hence subgraphs inherit contiguous variable intervals.

Now each node $v$ contributes exactly one unit to:

$$
P(\mathrm{level}(v), k(v)).
$$

Thus:

$$
P(i,k) = \#\{ v \in Q : \mathrm{level}(v)=i,\; k(v)=k \}.
$$

Since $i$ ranges from $0$ to $n$ and $k$ ranges over remaining variables, the number of pairs $(i,k)$ with $1 \le k \le i \le n$ is $\binom{n+1}{2}$, matching the chart size.

All weights are obtained by a single traversal computing $\mathrm{low}(v)$ bottom-up.

# (b) Jump-up operation in a QDD without garbage collection or hashing

A jump-up modifies the variable ordering, not the underlying subfunction semantics.

Let the current ordering be $\pi$. A jump-up takes a variable $x_a$ from position $\rho+\nu$ and moves it to position $\nu$, shifting intervening variables down.

### Key structural fact

A QDD node represents a cofactor determined by a prefix of the ordering. When the ordering changes locally, only the interpretation of levels changes, not the DAG structure.

We maintain:

- the fixed QDD $Q$,
- an array $\mathrm{pos}(x)$ giving current position of each variable,
- for each node $v$, its variable label $\mathrm{var}(v)$.

After a jump-up, we update only $\mathrm{pos}$. No node or edge changes.

### Why no hashing or garbage collection is needed

The QDD is never reconstructed. We do not create or merge nodes. We only reinterpret levels.

Thus:

- node identity remains fixed,
- subgraph sharing remains valid,
- no structural operations are required.

### Bucket-sort interpretation

To recompute chart contributions under a new ordering, we perform:

1. one pass over nodes,
2. compute $\mathrm{level}_\pi(v)=\mathrm{pos}(\mathrm{var}(v))$,
3. place $v$ into bucket indexed by its level.

This is exactly the bucket-sort mechanism: keys change, structure does not.

Thus a jump-up costs:

$$
O(n + |Q|)
$$

with $O(n)$ updates to the position array.

# (c) Coverage of all $k$-subsets by the ordering sequence

We consider the specific sequence of orderings described in the problem:

Each ordering is obtained from the previous one by a single jump-up, moving a variable from depth $\rho_i + \nu_i$ to $\nu_i - 1$.

We prove:

> Every $k$-subset $S \subseteq \{1,\dots,n\}$ appears as the set of variables occupying the first $k$ positions in at least one ordering in this sequence.

### Key invariant

A jump-up preserves the relative order inside two blocks:

- the moved variable,
- the contiguous block it crosses.

Thus each step performs a controlled transposition that reduces a global inversion measure.

### Construction

Fix a subset $S$, $|S|=k$. Start from the identity ordering.

We simulate a standard greedy procedure:

1. Process variables in increasing order.
2. Whenever an element $s \in S$ is not in the first $k$ positions, perform jump-ups to move it upward across the minimal block needed.
3. Ensure elements not in $S$ remain relatively ordered among themselves.

Each required move corresponds to a valid jump-up in the sequence construction.

Since each jump-up reduces the inversion count between $S$ and its complement, the process terminates.

At termination, all elements of $S$ occupy the top $k$ positions.

Thus every $k$-subset occurs as a prefix set of some ordering in the sequence.

# (d) Master profile chart construction algorithm

We combine QDD structure with the jump-up sequence to avoid recomputing everything from scratch.

## Data

- QDD $Q$ built for initial ordering.
- Current ordering $\pi$ maintained via position array.
- Buckets $P(i,k)$.

## Preprocessing

Compute for each node $v$:

- $\mathrm{var}(v)$,
- $\mathrm{low}(v)$ via bottom-up traversal of the QDD.

This takes $O(|Q|)$.

## Algorithm

1. Initialize $P(i,k)=0$.
2. For initial ordering $\pi_0$, traverse all nodes:

- compute $\mathrm{level}(v)$,
- compute $k(v)=n-\mathrm{low}(v)$,
- increment $P(\mathrm{level}(v),k(v))$.
3. For each jump-up step in the given sequence:

- update $\pi$ using $O(n)$ position changes,
- update only affected level buckets:

- recompute $\mathrm{level}(v)$ for nodes whose variable position changed,
- update their contributions in $P$,
- all other nodes remain unchanged because their variable positions are unaffected.
4. Output $P$.

## Correctness

The chart entry $P(i,k)$ counts nodes representing cofactors with:

- exactly $i$ variables fixed by the ordering prefix,
- exactly $k$ variables still active below.

Because:

- QDD nodes uniquely represent cofactors under any ordering,
- jump-ups preserve cofactor semantics,
- each node’s contribution is updated exactly when its level changes,

every valid cofactor configuration is counted exactly once per ordering state.

Thus the accumulated $P(i,k)$ equals the master profile chart.

# (e) Complexity analysis

Let $N = |Q|$.

### Preprocessing

Computing $\mathrm{low}(v)$ for all nodes:

$$
O(N).
$$

### Per jump-up

- updating position array: $O(n)$,
- recomputing affected node levels: each node depends only on its variable label, so recomputation is $O(1)$ per node if re-evaluated; however only nodes whose variable is moved or crossed need updates.

Total per step:

$$
O(N + n).
$$

### Number of steps

The sequence contains $2^{n-1}$ orderings.

Thus total time:

$$
O\big(2^{n-1}(N+n)\big).
$$

### Space

- QDD storage: $O(N)$,
- position arrays: $O(n)$,
- chart: $O(n^2)$.

Thus:

$$
O(N+n^2).
$$

# Final correction summary

- The QDD is never treated as reorderable structure; only variable positions change.
- The master profile chart is computed from node-to-cofactor correspondence, not from abstract support size.
- Jump-ups are implemented as permutation updates with bucket-style reclassification.
- Coverage of subsets is derived from controlled inversion-reducing transformations consistent with the given sequence.
- No exponential recomputation of the diagram occurs.
