## Solution

Let Algorithm R construct the reduced ordered BDD by recursively building or retrieving nodes of the form $k_j = (V, LO, HI)$ under the variable ordering $1 < 2 < \cdots < n$, and assume that steps $R1$ and $R2$ are the points where a node labeled by variable $v$ is considered and its two sub-BDDs are formed before possible insertion into the unique table.

To compute the restriction of a Boolean function with respect to the partial assignment encoded by $FIX[v]$, the construction is modified so that every node whose variable is fixed is eliminated and replaced by the appropriate sub-BDD.

Between steps $R1$ and $R2$, suppose Algorithm R is currently processing a node associated with variable $v = V(k_j)$ and has obtained the two recursive results corresponding to its children, say $p$ for the LO branch and $q$ for the HI branch. The additional actions are as follows.

If $FIX[v] \ge 0$, then the variable $x_v$ is fixed and no branching on $v$ is permitted in the restricted function. If $FIX[v] = 0$, the value of $x_v$ is forced to $0$, so the node contributes only its LO behavior and the entire construction for this node is replaced by $p$. If $FIX[v] = 1$, the value of $x_v$ is forced to $1$, so the node is replaced by $q$. In both cases the node $(v, p, q)$ is not created and control returns immediately with the chosen sub-BDD.

If $FIX[v] < 0$, no restriction applies to $x_v$, and the algorithm proceeds exactly as in the original Algorithm R, but with $p$ and $q$ already representing restricted subfunctions.

After this selection, before the creation or lookup of the node in the unique table, a further necessary reduction condition is enforced: if $p = q$, then the node labeled $v$ is redundant after restriction, since both outcomes of $x_v$ yield the same subfunction. In this case the construction again returns $p$ instead of creating $(v, p, p)$.

Only when $FIX[v] < 0$ and $p \ne q$ does Algorithm R proceed to step $R2$ to create or retrieve the node $(v, p, q)$ in the usual way.

This modification ensures that restriction is applied locally at every decision point, eliminating fixed variables from the BDD while preserving canonicity under the given variable ordering. ∎
