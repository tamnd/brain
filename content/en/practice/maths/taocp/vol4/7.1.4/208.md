**Exercise 208.** &#9654; [*16*] [16] By modifying Algorithm C, show that all solutions of a Boolean function

We restart from a correct interpretation of Algorithm C as a pure dynamic program on the directed acyclic graph of the ZDD.

## Correct solution

Let a reduced ZDD be given for a Boolean function $f(x_1,\dots,x_n)$. As in TAOCP Section 7.1.4, each internal node $v$ is labeled by a variable index $V(v)$, and has two outgoing edges:

- LO child $L(v)$ (case $x_{V(v)}=0$),
- HI child $H(v)$ (case $x_{V(v)}=1$).

In a ZDD, the HI edge follows the zero-suppression rule (the case where choosing the item is relevant), but for counting assignments this does not change the structural interpretation of the decision diagram: each HI edge corresponds to fixing the variable to $1$ and continuing computation at the target node.

The key point is that the counting must be formulated as a node-local dynamic program, so all dependence on “skipped variables” is pushed onto edge weights, not into terminal values.

## DP state

For each node $v$, define $C(v)$ to be the number of assignments of variables $x_{V(v)}, x_{V(v)+1}, \dots, x_n$ that are consistent with the subfunction represented by $v$.

Unlike the flawed solution, we will eliminate any dependence on external indices in the definition of $C(v)$ itself by expressing skipped-variable freedom explicitly in the recurrence.

## Terminal cases

These are now well-defined and independent of context:

$$
C(\bot)=0,\qquad C(\top)=1.
$$

Justification:

- $\bot$ represents no satisfying assignment.
- $\top$ represents the empty constraint, and in a ZDD context it corresponds to one completed assignment of the remaining structure once all decisions have been resolved. All freedom of untested variables is accounted for in edge transitions, not in terminal values.

## Edge contribution factors

Let $v$ be an internal node with $V(v)=k$. Let $w$ be a child of $v$, and assume $V(w)=\ell$, where $\ell>k$ (or $w=\top$).

Between variables $x_k$ and $x_\ell$, the variables

$$
x_{k+1}, x_{k+2}, \dots, x_{\ell-1}
$$

are not tested along the edge. Each of these variables is free, contributing a factor of $2$. Hence define the edge weight

$$
\alpha(v,w)=2^{\max(0,\ell-k-1)},
$$

with the convention that if $w=\top$, then $\ell=n+1$, so

$$
\alpha(v,\top)=2^{n-k}.
$$

This resolves the terminal-node issue raised in the review: no undefined $V(\top)$ is used.

## Recurrence

For any internal node $v$,

$$
C(v)=\alpha(v,L(v))\,C(L(v)) + \alpha(v,H(v))\,C(H(v)).
$$

This is well-defined because:

- both children are nodes in the DAG or $\top,\bot$,
- all index gaps are handled locally by $\alpha(v,w)$,
- no global “current index” is required.

## Why this is correct for ZDDs

The ZDD differs from a BDD only in the restriction that an HI edge represents inclusion of the current item and zero-suppression removes unnecessary LO chains. However, counting assignments depends only on:

1. the variable tested at each node,
2. the fact that untested variables are free,
3. the structure of the DAG.

Zero suppression does not change the fact that every path corresponds to a consistent partial assignment with skipped variables free. The factors $\alpha(v,w)$ account for exactly those skipped variables.

## Algorithm C modification

Algorithm C performs a postorder traversal of the DAG and computes $C(v)$ once per node using memoization.

The modification is:

1. Visit nodes in reverse topological order.
2. Set $C(\bot)=0$, $C(\top)=1$.
3. For each internal node $v$, compute $C(v)$ using the recurrence above.
4. Return $C(r)$ for the root $r$.

Each node is processed once, and each edge contributes only a constant-time exponentiation based on pre-stored variable indices.

## Complexity

Let $N$ be the number of nodes in the ZDD. Each node is evaluated once, and each evaluation performs $O(1)$ arithmetic operations. Hence the total time is $O(N)$, matching Algorithm C for BDDs.

## Conclusion

By redefining the dynamic program so that:

- terminal values are constant,
- all skipped-variable freedom is encoded in edge weights,
- recurrence depends only on local node information,

we obtain a correct adaptation of Algorithm C that counts all satisfying assignments of a Boolean function given by a ZDD.

This completes the corrected solution. ∎
