**Exercise 45.** [*22*] [22] Give precise specifications for the Boolean modules that compute the three-
in-a-row function as in (33) and (34), and show that the network is well defined.

The previous solution fails because it tries to encode the problem as a two-state run-length machine and never introduces a mechanism that can distinguish “exactly two consecutive 1s” from “three consecutive 1s”. The correct construction must explicitly remember enough local history to test a window of length three.

A direct and faithful way to match the style of Knuth’s constructions in (33) and (34) is to build a feedforward network where each module propagates a sliding window of the last two input bits together with a running detection flag.

Let the input be $x_1,\dots,x_n \in \{0,1\}$. We define a sequence of Boolean modules $M_1,\dots,M_n$. Each module $M_i$ produces a state triple

$$
(a_i,b_i,d_i),
$$

where $a_i$ and $b_i$ store the two most recent inputs, and $d_i$ records whether a three-in-a-row occurrence has already been seen among $x_1,\dots,x_i$.

### Initial module

We initialize with a fictitious zero history:

$$
a_0 = 0,\quad b_0 = 0,\quad d_0 = 0.
$$

### Module specification

For each $i \ge 1$, module $M_i$ takes inputs $(a_{i-1}, b_{i-1}, d_{i-1}, x_i)$ and produces outputs $(a_i,b_i,d_i)$ defined by the Boolean equations

$$
a_i = b_{i-1},
$$

$$
b_i = x_i,
$$

$$
d_i = d_{i-1} \lor (a_{i-1} \land b_{i-1} \land x_i).
$$

The intended meaning is that $a_{i-1} = x_{i-2}$ and $b_{i-1} = x_{i-1}$, so the conjunction $a_{i-1} \land b_{i-1} \land x_i$ is exactly the predicate that $x_{i-2} = x_{i-1} = x_i = 1$.

### Output of the network

The final output is

$$
F = d_n.
$$

This correctly computes the three-in-a-row function, since $d_n = 1$ if and only if there exists some index $i \le n-2$ such that $x_i = x_{i+1} = x_{i+2} = 1$, and once such an occurrence is detected it is permanently propagated by the disjunction in the recurrence for $d_i$.

### Correctness

We prove by induction on $i$ that after module $M_i$, the variables satisfy:

1. $a_i = x_{i-1}$ for $i \ge 2$,
2. $b_i = x_i$,
3. $d_i = 1$ if and only if there exists $j \le i-2$ such that $x_j x_{j+1} x_{j+2} = 1$.

The base case $i=1$ holds by definition of the initial state, since $a_1 = b_0 = 0$, $b_1 = x_1$, and no triple can yet exist, so $d_1 = 0$.

For the inductive step, assume correctness for $i-1$. The updates $a_i = b_{i-1}$ and $b_i = x_i$ correctly maintain the sliding window invariant. For $d_i$, there are two cases. If a three-in-a-row occurrence already exists among $x_1,\dots,x_{i-1}$, then $d_{i-1} = 1$ and hence $d_i = 1$. Otherwise, $d_{i-1} = 0$, and the only possible new occurrence ending at position $i$ is exactly $x_{i-2} = x_{i-1} = x_i = 1$, which is captured by $a_{i-1} \land b_{i-1} \land x_i$. This establishes the invariant.

### Well-definedness of the network

Each module $M_i$ defines $(a_i,b_i,d_i)$ as a Boolean function of $(a_{i-1}, b_{i-1}, d_{i-1}, x_i)$. No variable depends on itself or on any variable at a higher index. All dependencies strictly decrease the index of state variables, so the dependency graph is acyclic.

Therefore the values of all internal variables are uniquely determined by forward substitution from $i=1$ to $i=n$. This shows the network is well defined in the sense of Knuth’s construction: every output is a uniquely specified Boolean function of the input vector $(x_1,\dots,x_n)$.

This completes the corrected solution.
