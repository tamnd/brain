### (a) Unateness in first-order logic

Let $f(x_1,\dots,x_n)$ and define polarities $y_i \in \{0,1\}$. Define the flipped function

$$
h(x_1,\dots,x_n)=f(x_1\oplus y_1,\dots,x_n\oplus y_n).
$$

$f$ is unate with polarities $y$ iff $h$ is monotone increasing:

$$
\forall x,x' \in \{0,1\}^n \; \Bigl(x \le x' \Rightarrow h(x)\le h(x')\Bigr),
$$

where $x \le x'$ means $x_i \le x'_i$ for all $i$.

Substituting $h$, this becomes:

$$
\exists y_1\cdots \exists y_n \; \forall x \forall x' \;
\Bigl(x \le x' \Rightarrow f(x\oplus y)\le f(x'\oplus y)\Bigr).
$$

Equivalently, monotonicity can be reduced to single-bit increases:

$$
\exists y \;\forall x \;\forall i \;
\Bigl(f(x_1\oplus y_1,\dots,x_i\oplus y_i=0,\dots,x_n\oplus y_n)
\le
f(x_1\oplus y_1,\dots,x_i\oplus y_i=1,\dots,x_n\oplus y_n)\Bigr).
$$

This expresses unateness using existential quantification over polarities and universal quantification over inputs.

### (b) $O(B(f)^2)$ BDD algorithm for testing unateness

Let $B(f)$ be the number of nodes in the reduced ordered BDD for $f$.

#### Key fact

For a fixed variable $x_i$, define cofactors:

- $f_i^0 = f|_{x_i=0}$
- $f_i^1 = f|_{x_i=1}$

Then:

- $x_i$ is **positive-unate** iff $f_i^0 \le f_i^1$
- $x_i$ is **negative-unate** iff $f_i^1 \le f_i^0$

Thus $f$ is unate iff every variable admits at least one of these two directions.

#### Subroutine: BDD implication test

We use a standard memoized procedure

$$
\mathrm{LE}(u,v) \equiv \text{BDD for } f_u \le f_v.
$$

It is computed by DFS on pairs $(u,v)$ with memoization. Each pair is processed once, so over all calls the total number of visited pairs is $O(B(f)^2)$.

#### Algorithm

We process each variable $x_i$ independently.

Let $u$ be a BDD node labeled by $x_i$ with low and high children $u_0,u_1$.

Define:

- $x_i$ is **positive candidate** if for all nodes $u$ labeled $x_i$,

$$
\mathrm{LE}(u_0,u_1) = \text{true}.
$$
- $x_i$ is **negative candidate** if for all nodes $u$ labeled $x_i$,

$$
\mathrm{LE}(u_1,u_0) = \text{true}.
$$

Algorithm:

For each variable $x_i$:

1. check whether all $x_i$-nodes satisfy $\mathrm{LE}(u_0,u_1)$
2. check whether all $x_i$-nodes satisfy $\mathrm{LE}(u_1,u_0)$
3. if neither holds, return “not unate”
4. otherwise set polarity:

- $y_i=0$ if positive holds
- $y_i=1$ if negative holds (or either if both hold)

Return all $y_i$.

#### Correctness

If $x_i$ is positive-unate, then for every assignment to other variables, increasing $x_i$ cannot decrease $f$. In a BDD, every occurrence of $x_i$ represents exactly the Shannon decomposition

$$
f_u = (x_i=0 ? f_{u_0} : f_{u_1}),
$$

so monotonicity in $x_i$ is equivalent to $f_{u_0} \le f_{u_1}$ for every $x_i$-labeled node $u$. The negative case is symmetric.

Thus the local BDD edge condition is necessary and sufficient.

If every variable admits one polarity, the flipped function $f(x\oplus y)$ is monotone in each coordinate, hence monotone overall.

#### Complexity

There are at most $B(f)$ nodes and $O(B(f))$ edges.

Each call to $\mathrm{LE}(u,v)$ is memoized globally over all pairs $(u,v)$. Hence every pair is evaluated at most once, giving total cost

$$
O(B(f)^2).
$$

Each variable triggers only constant-many scans over its incident BDD nodes, and these scans only invoke cached $\mathrm{LE}$ calls. Therefore the total work across all variables remains bounded by the number of distinct $(u,v)$ pairs, which is $O(B(f)^2)$.

#### Polarities

The algorithm returns $y_i$ directly from which direction succeeds:

- $y_i=0$ if $f_i^0 \le f_i^1$
- $y_i=1$ if $f_i^1 \le f_i^0$

If both hold, $x_i$ is irrelevant (don’t care polarity).

### Result

The corrected approach avoids per-node polarity propagation and reduces unateness testing to globally validated BDD implication checks on cofactors. The resulting algorithm correctly decides unateness and produces polarities in $O(B(f)^2)$ time.
