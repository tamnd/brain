The previous argument fails because it tries to compare subtables living on different variable sets using an undefined notion of truth-table equality. The corrected proof rebuilds the construction so that every object being compared lives in a fixed, well-defined function space.

## Corrected solution

### 1. Fixed variable order and cofactors

Let $f(x_1,\dots,x_n)$ be a Boolean function with fixed variable order $x_1 < \cdots < x_n$.

For each $k \in \{0,\dots,n\}$ and each assignment $a \in \{0,1\}^k$, define the **cofactor**

$$
f_a(x_{k+1},\dots,x_n) := f(a_1,\dots,a_k,x_{k+1},\dots,x_n).
$$

Thus $f_a$ is a Boolean function on exactly $n-k$ variables.

The crucial point is that we never compare functions with different numbers of free variables unless they are first placed into a level-specific space.

### 2. Level-wise function spaces

For each $k$, let

$$
\mathcal{F}_k := \{ g : \{0,1\}^{n-k} \to \{0,1\} \}.
$$

Each cofactor $f_a$ at level $k$ lies in $\mathcal{F}_k$.

Define equivalence only within a fixed level:

$$
g \equiv_k h \quad \Longleftrightarrow \quad g(x_{k+1},\dots,x_n) = h(x_{k+1},\dots,x_n)\ \text{for all inputs}.
$$

This is well defined because both functions have the same domain.

A **bead at level $k$** is an equivalence class $[f_a]_k$ under $\equiv_k$.

### 3. Node set of the BDD

The node set is

$$
V = \{\, [f_a]_k \mid k \in \{0,\dots,n\},\ a \in \{0,1\}^k \,\} \ \cup \ \{\bot,\top\}.
$$

Each node is therefore a pair of:

- a level $k$,
- a Boolean function on the suffix variables $x_{k+1},\dots,x_n$.

This avoids any cross-level identification problems.

### 4. Transition functions

Let $v = [g]_k$, where $g \in \mathcal{F}_k$. Define cofactors in $\mathcal{F}_{k+1}$ by

$$
g_0(x_{k+2},\dots,x_n) := g(0,x_{k+2},\dots,x_n),
\quad
g_1(x_{k+2},\dots,x_n) := g(1,x_{k+2},\dots,x_n).
$$

Define edges:

$$
\mathrm{LO}(v) := [g_0]_{k+1}, \quad \mathrm{HI}(v) := [g_1]_{k+1}.
$$

This is well defined: if $g \equiv_k g'$, then $g_0 \equiv_{k+1} g'_0$ and $g_1 \equiv_{k+1} g'_1$, since equality of functions is preserved under substitution of variables.

### 5. Degenerate (elimination) criterion

A node $v = [g]_k$ is **degenerate** if

$$
g_0 = g_1 \quad \text{as elements of } \mathcal{F}_{k+1}.
$$

In this case, the Shannon decomposition

$$
g = (x_{k+1} \ ?\ g_1 : g_0)
$$

does not depend on $x_{k+1}$, so the test is redundant.

In a reduced BDD, such a node is eliminated by redirecting it to $[g_0]_{k+1}$.

### 6. Acyclicity and ordering

Every edge goes from level $k$ to level $k+1$. Hence along every directed path, the level strictly increases from $0$ to $n$. Therefore the graph is acyclic and respects the fixed variable ordering.

### 7. Correct evaluation

We prove by induction on $k$:

**Claim.** If at level $k$ the current node is $[g]_k$, then under any assignment to $x_{k+1},\dots,x_n$, the BDD evaluates to $g$.

Base case: at level $n$, functions are constant, so the claim holds at terminals $\bot,\top$.

Inductive step: at node $[g]_k$,

- if $x_{k+1}=0$, computation moves to $[g_0]_{k+1}$,
- if $x_{k+1}=1$, computation moves to $[g_1]_{k+1}$.

By definition of cofactors, this matches exactly the Shannon decomposition of $g$. The inductive hypothesis applies.

Thus the BDD computes $f$.

### 8. No duplicate nodes (canonicity within levels)

If $[g]_k$ and $[h]_k$ have identical children, then

$$
[g_0]_{k+1} = [h_0]_{k+1}, \quad [g_1]_{k+1} = [h_1]_{k+1}.
$$

So $g_0 = h_0$ and $g_1 = h_1$, hence $g = h$ as functions in $\mathcal{F}_k$. Therefore the equivalence classes coincide, so no two distinct nodes can represent the same Boolean function at a fixed level.

This avoids circularity because equality is taken in the semantic function space $\mathcal{F}_k$, not defined via the graph.

### 9. Bijection between nodes and cofactors

Every node is constructed as $[f_a]_k$ for some prefix assignment $a$, so every node corresponds to a cofactor of $f$.

Conversely, every cofactor $f_a$ determines exactly one equivalence class $[f_a]_k$.

Thus there is a one-to-one correspondence:

$$
\{\text{BDD nodes}\} \;\longleftrightarrow\; \{ [f_a]_k \}.
$$

Hence the node count satisfies

$$
B(f) = \left| \{ [f_a]_k \mid k,a \} \right|.
$$

### 10. Uniqueness of the reduced ordered BDD

Any reduced ordered BDD identifies nodes exactly when they compute the same function on the suffix variables and eliminates nodes with identical cofactors. This is precisely the equivalence relation defining the beads. Therefore any two reduced ordered BDDs for $f$ induce the same partition of cofactors and are isomorphic.

### Conclusion

By defining cofactors as level-specific Boolean functions and restricting equivalence to fixed domains $\mathcal{F}_k$, all comparisons become well defined. The bead construction is then a direct quotient of semantic function spaces, yielding a well-defined, acyclic, reduced ordered BDD and completing the proof of Theorem W. ∎
