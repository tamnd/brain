**Exercise 206.** [*M46*] [M46] What are the worst-case running times of the algorithms in exercise 205?

Let $N_f$ and $N_g$ denote the numbers of nonterminal nodes in the reduced ZDDs representing the families $f$ and $g$, respectively. Include the two terminal nodes in each ZDD.

The algorithms in Exercise 205 are all Apply-style recursive procedures on ZDDs. Each procedure is defined by case analysis on the top variable of the current ZDD nodes and makes recursive calls only on the children of those nodes. A memoization table stores results indexed by pairs of ZDD nodes.

## 1. Structure of the state space

For every algorithm in Exercise 205, the recursion is invoked on arguments of the form $(u,v)$, where $u$ is a node of $f$ (or a terminal) and $v$ is a node of $g$ (or a terminal).

No additional parameters (such as accumulators or visited sets) are carried by the recursion. Therefore the identity of a subproblem is completely determined by the ordered pair $(u,v)$.

Hence the number of distinct possible recursive states is bounded by

$$
(N_f+2)(N_g+2).
$$

## 2. Memoization bound

Each pair $(u,v)$ is computed at most once, because after the first evaluation its value is stored in the memoization table and reused on subsequent visits.

Therefore the total number of recursive evaluations is $O(N_f N_g)$.

## 3. Cost of processing a single state

Fix a pair $(u,v)$. The algorithm inspects whether $u$ or $v$ is terminal, compares their top variables, and then performs a constant number of operations:

- a constant number of comparisons of variable indices,
- a constant number of table lookups in the memoization structure,
- and a constant number of recursive calls (at most two or three, depending on the operation).

Each recursive call is on strictly smaller ZDD nodes (children of $u$ or $v$), so no additional traversal or scanning of the graph is performed at this step.

Thus the nonrecursive work per state is $O(1)$.

## 4. Binary operations: union, intersection, symmetric difference, quotient

For

$$
f \sqcup g,\quad f \sqcap g,\quad f \triangle g,\quad f/g,
$$

each algorithm is a binary Apply-style recursion on pairs $(u,v)$. As established above:

- state space size is $O(N_f N_g)$,
- each state is evaluated once,
- each evaluation costs $O(1)$.

Hence each of these four operations has worst-case running time

$$
O(N_f N_g).
$$

## 5. Remainder operation

The remainder operation in Exercise 205 is also defined as a recursive procedure on pairs of ZDD nodes. Although its semantic definition differs from the previous operations, its implementation follows the same structural pattern:

- the recursion is invoked on a pair $(u,v)$,
- branching is determined only by the top variables of $u$ and $v$,
- recursive calls are made only on children of $u$ and/or $v$,
- results are memoized by the pair $(u,v)$.

No additional state beyond the node pair is introduced by the algorithm.

Therefore the same state-space argument applies: every reachable subproblem is represented by some pair of nodes from $f$ and $g$, and there are at most $(N_f+2)(N_g+2)$ such pairs.

Each state performs only constant local work, so the remainder algorithm also runs in

$$
O(N_f N_g)
$$

worst-case time.

## 6. Conclusion

All five algorithms of Exercise 205 are memoized recursive procedures whose subproblems are uniquely identified by ordered pairs of ZDD nodes, with no auxiliary state. Each pair is evaluated at most once, and each evaluation takes constant time apart from recursive calls.

Therefore, in the worst case, every algorithm in Exercise 205 has running time

$$
\boxed{O(N_f N_g)}.
$$
