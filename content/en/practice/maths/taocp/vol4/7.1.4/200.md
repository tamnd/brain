We restart from the correct operational model for ZDDs in TAOCP: all constructions must be expressed using the ITE (apply) framework, with no assumption of Boolean negation as a primitive operation. The goal is to implement

$$
\operatorname{MUX}(f,g,h),
$$

which is defined semantically by

$$
\operatorname{MUX}(f,g,h)(x) =
\begin{cases}
g(x) & \text{if } f(x)=1,\\
h(x) & \text{if } f(x)=0.
\end{cases}
$$

Hence,

$$
\operatorname{MUX}(f,g,h) = \operatorname{ITE}(f,g,h).
$$

The correct task is therefore to implement ITE directly on ZDDs using the apply paradigm.

## 1. Structural principle

We construct a reduced ordered ZDD for $\operatorname{ITE}(f,g,h)$ by recursion on triples $(f,g,h)$, using the global variable ordering.

For any ZDD node $u$, let $\operatorname{var}(u)$ be its root variable, and define

$$
\operatorname{var}(\bot)=\operatorname{var}(\top)=+\infty.
$$

At each step we choose the smallest variable appearing in any argument:

$$
x = \min(\operatorname{var}(f),\operatorname{var}(g),\operatorname{var}(h)).
$$

This guarantees adherence to the global ordering.

## 2. Cofactor definition (ZDD-safe form)

For each $u \in \{f,g,h\}$, define cofactors with respect to $x$:

- If $\operatorname{var}(u)=x$, let $u_0, u_1$ be the low and high children.
- If $\operatorname{var}(u)>x$, then $x$ does not appear in $u$, and we set

$$
u_0 = u_1 = u.
$$

This is valid because no path in $u$ can test $x$ before its root variable.

## 3. Recursive definition of MUX

Define recursively:

$$
r_0 = \operatorname{MUX}(f_0,g_0,h_0), \qquad
r_1 = \operatorname{MUX}(f_1,g_1,h_1).
$$

These correspond to the Shannon expansion of the ITE operator at variable $x$.

## 4. Base cases

If $f,g,h \in \{\bot,\top\}$, then we evaluate directly from the ITE meaning:

- If $f=\top$, then $\operatorname{MUX}(f,g,h)=g$.
- If $f=\bot$, then $\operatorname{MUX}(f,g,h)=h$.

This follows immediately from the defining semantics of ITE and requires no further decomposition.

## 5. Node construction without negation

We now construct the ZDD node for variable $x$, ensuring correctness under ZDD semantics.

### Case 1: identical branches

If

$$
r_0 = r_1,
$$

then variable $x$ does not affect the result, so we return

$$
\operatorname{MUX}(f,g,h) = r_0.
$$

### Case 2: distinct branches

Otherwise we create a node

$$
u = \text{node}(x,r_0,r_1).
$$

### ZDD zero-suppression rule

If $r_1 = \bot$, then no solution includes $x$, so we may still return $r_0$ directly. This is consistent with ZDD semantics where absence of the high branch encodes exclusion of $x$.

Unique-table lookup ensures canonicity of $(x,r_0,r_1)$.

## 6. Correctness proof

### 6.1 Equivalence to ITE semantics

We prove by induction on the recursion structure that the procedure computes $\operatorname{ITE}(f,g,h)$.

For fixed $x$, any assignment splits into two cases:

- $x=0$: all arguments reduce to $f_0,g_0,h_0$, so the result is $r_0$.
- $x=1$: all arguments reduce to $f_1,g_1,h_1$, so the result is $r_1$.

Thus the constructed node exactly represents Shannon expansion:

$$
\operatorname{ITE}(f,g,h)
=
(x=0 ? r_0) \cup (x=1 ? r_1).
$$

### 6.2 Elimination of negation dependency

No step uses Boolean complement. All operations are:

- restriction to cofactors,
- recursive ITE evaluation,
- ZDD node construction.

Thus the algorithm is fully expressed in the apply/ITE framework, consistent with TAOCP requirements.

### 6.3 Ordering preservation

Since $x$ is the minimum variable among all arguments at each step:

- no variable smaller than $x$ appears in any cofactor,
- recursion strictly increases variable indices along any path.

Hence the result is an ordered ZDD.

### 6.4 ZDD canonicity

- Unique-table merging ensures identical subproblems share nodes.
- Redundant tests ($r_0=r_1$) are eliminated.
- Zero-suppression removes nodes with empty high branches.

Therefore the representation is reduced and canonical.

## 7. Conclusion

The correct implementation of MUX for ZDDs is obtained by recognizing that

$$
\operatorname{MUX}(f,g,h)=\operatorname{ITE}(f,g,h),
$$

and implementing ITE directly via global-minimum variable decomposition and Shannon expansion on ZDD cofactors.

No Boolean negation or derived complement operation is required. The resulting algorithm is a standard apply procedure specialized to the ternary ITE operator and fully consistent with TAOCP ZDD constructions. ∎
