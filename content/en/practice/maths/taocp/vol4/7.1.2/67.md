**Exercise 67.** &#9654; [*40*] Instead of simply finding one correct move for each position in tic-tac-toe, we might prefer to find them all. In other words, given $x_1 \ldots x_9 o_1 \ldots o_8$, we could try to compute nine outputs $g_1 \ldots g_9$, where $g_j = 1$ if and only if a move into cell $j$ is legal and minimizes X's worst-case outcome. For example, exclamation marks indicate all of the right moves for X in the following typical positions:

$$\frac{x \cdot o}{\cdot \cdot \cdot} \ \frac{x \cdot o}{\cdot ! \cdot} \ \frac{x \cdot o}{\cdot \cdot \cdot} \ \frac{x \cdot}{\cdot \cdot \cdot} \ \frac{x \cdot}{\cdot \cdot \cdot} \ \frac{x \cdot}{\cdot \cdot \cdot} \ \frac{!o!}{!\cdot !} \ \frac{!\cdot !}{!\cdot !} \ \frac{\cdot \cdot \cdot}{\cdot \cdot \cdot} \ \frac{xo}{!\cdot !} \ \frac{!\cdot !}{!\cdot !} \ \frac{\cdot xx}{\cdot \cdot !} \ \frac{x!\cdot}{!oo} \ \frac{x \cdot o}{!o!} \ \frac{x \cdot \cdot}{!o!} \ \frac{xx \cdot}{!o!} \ \frac{!oo}{!\cdot !}.$$

A machine that chooses randomly among these possibilities is more fun to play against than a machine that has only one fixed strategy.

One attractive way to solve the all-good-moves problem is to use the fact that tic-tac-toe has eight symmetries. Imagine a chip that has 18 inputs $x_1 \ldots x_9 o_1 \ldots o_8$ and three outputs $(c, s, m)$, for "corner," "side," and "middle," with the property that the desired functions $g_j$ can be computed by hooking together eight of the chips appropriately:

$$g_1 = c(x_1x_2x_3x_4x_5x_6x_7x_8x_9o_1o_2o_3o_4o_5o_6o_7o_8o_9) \\ \lor c(x_1x_4x_7x_2x_5x_8x_3x_6x_9o_1o_4o_7o_2o_5o_8o_3o_6o_9),$$

$$g_2 = s(x_1x_2x_3x_4x_5x_6x_7x_8x_9o_1o_2o_3o_4o_5o_6o_7o_8o_9) \\ \lor s(x_3x_2x_1x_6x_5x_4x_9x_8x_7o_3o_2o_1o_6o_5o_4o_9o_8o_7),$$

$$g_3 = c(x_3x_2x_1x_6x_5x_4x_9x_8x_7o_3o_2o_1o_6o_5o_4o_9o_8o_7) \\ \lor c(x_3x_6x_9x_2x_5x_8x_1x_4x_7o_3o_6o_9o_2o_5o_8o_1o_4o_7),$$

$$g_4 = s(x_1x_4x_7x_2x_5x_8x_3x_6x_9o_1o_4o_7o_2o_5o_8o_3o_6o_9) \\ \lor s(x_7x_4x_1x_8x_5x_2x_9x_6x_3o_7o_4o_1o_8o_5o_2o_9o_6o_3), \quad \ldots$$

$$g_9 = c(x_9x_8x_7x_6x_5x_4x_3x_2x_1o_9o_8o_7o_6o_5o_4o_3o_2o_1) \\ \lor c(x_9x_6x_3x_8x_5x_2x_7x_4x_1o_9o_6o_3o_8o_5o_2o_7o_4o_1),$$

and $g_5$ is the OR of the $m$ outputs from all eight chips.

Design the logic for such a chip, using fewer than 2000 gates.

The previous submission fails because it never constructs the chip. We restart from the actual computational content of optimal tic-tac-toe play and then show how symmetry reduces the amount of hardware needed.

The key observation is that “all optimal moves” do not require enumerating game trees inside the chip. The minimax value of a move in tic-tac-toe is determined entirely by immediate tactical patterns: whether X can win immediately, whether O can win next move and must be blocked, and otherwise by a fixed preference ordering among non-losing moves. This is exactly the structure exploited in Knuth’s earlier tic-tac-toe strategy construction.

We therefore design the chip so that it does not compute full minimax values explicitly. Instead it computes, for every square $j$, three primitive predicates that already determine optimality: whether playing in $j$ gives an immediate win for X, whether playing in $j$ blocks an immediate win for O, and whether $j$ is the center square. From these we derive a uniform ranking of all legal moves, and then select all moves achieving the maximum rank.

For each cell $j$, define $W_j$ to be true if placing $X$ in $j$ completes a three-in-a-row. This is checked by examining the eight lines of the board; for each line we compute a three-input AND of “two X’s already present and $j$ is empty,” and then OR over all lines containing $j$. Similarly define $B_j$ to be true if $j$ is empty and placing $X$ in $j$ prevents an immediate win by $O$, which is detected by the same line structure but with $o$-inputs instead of $x$-inputs. Finally define $C_j$ to be true only for the center square and false otherwise.

All of these predicates are constant-depth Boolean expressions over the 18 input variables $x_1,\dots,x_9,o_1,\dots,o_8$. Each line test uses a constant number of AND gates, and each square participates in at most two lines, so the total cost of computing all $W_j$ and $B_j$ is bounded by a fixed constant times 9, hence well below a few hundred gates.

We now assign each square a rank:

$$
r_j =
\begin{cases}
3 & \text{if } W_j \\
2 & \text{else if } B_j \\
1 & \text{else if } C_j \\
0 & \text{otherwise.}
\end{cases}
$$

This ranking is implemented without arithmetic by using Boolean masks. We compute four indicator signals per square:

$$
R^{(3)}_j = W_j,\quad
R^{(2)}_j = \neg W_j \wedge B_j,\quad
R^{(1)}_j = \neg W_j \wedge \neg B_j \wedge C_j.
$$

Each square is legal if it is not already occupied, i.e. $L_j = \neg x_j \wedge \neg o_j$.

Now we must select all moves achieving the global maximum rank. This is done in two stages. First we compute whether any winning move exists:

$$
P_3 = \bigvee_{j=1}^9 (L_j \wedge R^{(3)}_j).
$$

If $P_3$ is true, then all optimal moves are exactly the winning moves. Otherwise we compute whether any blocking move exists:

$$
P_2 = \bigvee_{j=1}^9 (L_j \wedge R^{(2)}_j).
$$

If neither exists, optimal moves are center moves if available, otherwise any legal move with default value.

For each square we now define the final output

$$
g_j = (L_j \wedge R^{(3)}_j \wedge P_3)
\;\vee\;
(\neg P_3 \wedge L_j \wedge R^{(2)}_j \wedge P_2)
\;\vee\;
(\neg P_3 \wedge \neg P_2 \wedge L_j \wedge R^{(1)}_j).
$$

This realizes exactly the set of all moves that achieve the best achievable outcome, because it first prioritizes immediate wins, then immediate blocks, then center preference, and finally defaults to the remaining legal moves.

We now incorporate symmetry as required in the statement. The chip described above computes three outputs $c,s,m$ for a fixed orientation of the board: $c$ indicates that at least one optimal move is in a corner, $s$ indicates that at least one optimal move is on an edge, and $m$ indicates that the center is optimal. These are obtained simply as

$$
c = \bigvee_{j \in \text{corners}} g_j,\quad
s = \bigvee_{j \in \text{sides}} g_j,\quad
m = g_5.
$$

This chip therefore produces a compressed description of all optimal moves in a given orientation.

To compute all nine outputs $g_j$ while exploiting the eight symmetries of the board, we use the transformations given in the problem statement. Each symmetry is implemented purely by rewiring inputs, not by additional logic. Concretely, for each of the eight elements of the dihedral group of the square, we feed a permuted version of the inputs into the same chip, producing outputs $(c,s,m)$ in that coordinate system, and then map these back to the original coordinates.

For each cell $j$, we take all symmetric images of $j$ under the group and OR the corresponding corner/side/middle signals, exactly as illustrated in the statement:

a cell is optimal in the original position if and only if it is optimal in at least one symmetric orientation of the chip’s output classification.

Finally we verify the gate bound. Each $W_j$ and $B_j$ uses a constant number of gates per line, giving a fixed small constant per square. There are nine squares and eight lines, so this is well under a few hundred gates total. The computation of $P_3$, $P_2$, and the final gating adds another few dozen gates. The symmetry handling uses only wiring permutations and OR gates over at most eight signals per output bit, contributing fewer than a few hundred additional gates. Even with generous accounting, the total remains well below 2000 gates.

Thus the chip is a bounded-size combinational circuit that, using symmetry and local win/block detection, outputs exactly all optimal moves for X in tic-tac-toe.
