**Exercise 59.** [*26*] A ZDD with 3,174,197 nodes can be constructed to find (almost) all simple corner-to-corner king paths on a chessboard, using the method of exercise 7.1.4–225. Explain how to use this ZDD to compute (a) the total length of all paths; (b) the number of paths that touch any given subset of the center and/or corner points.

Part (a) is already correct; only part (b) requires repair. The issue is that the DP state must be tied to the **ZDD variable (edge) at each node**, so that the update of visited vertices is well-defined.

## (a) Total length of all paths

The given recurrence is valid.

Let each node $v$ have 0-child $v_0$ and 1-child $v_1$. Define:

$$
N(v)=\#\{\text{paths in sub-ZDD at } v\},\qquad
L(v)=\sum_{P} \ell(P).
$$

Base:

$$
N(\mathbf{1})=1,\qquad L(\mathbf{1})=0.
$$

Recurrence:

$$
N(v)=N(v_0)+N(v_1),
$$

$$
L(v)=L(v_0)+L(v_1)+N(v_1).
$$

Each 1-edge contributes exactly one additional move, hence the additive term $N(v_1)$. Evaluating bottom-up gives $L(\text{root})=\sum_{P\in\mathcal P}\ell(P)$.

## (b) Paths hitting prescribed subsets of $S$

### Step 1: Correct interpretation of ZDD variables

In the ZDD for king paths (as in Exercise 7.1.4–225), each node $v$ is labeled by a **specific edge variable**

$$
e(v),
$$

meaning:

- the 0-edge: exclude this move,
- the 1-edge: include this move.

Let this edge be

$$
e(v) = \{x(v), y(v)\},
$$

where $x(v), y(v)$ are its endpoints on the chessboard.

Define the contribution of this decision to the distinguished set $S$:

$$
\delta(v) = \{x(v), y(v)\}\cap S.
$$

Thus the visited-set update is completely determined by the node label.

### Step 2: DP state

For each node $v$ and subset $M \subseteq S$, define:

$$
F_v(M)
=
\#\{\text{partial completions from } v \text{ whose visited set in } S \text{ equals } M\}.
$$

Here “visited set” means the union of endpoints of all included edges along with the fixed start corner (if it lies in $S$, it is included in the initial state).

### Step 3: Base case

At the terminal:

$$
F_{\mathbf{1}}(M)=
\begin{cases}
1, & M=\varnothing,\\
0, & M\neq\varnothing.
\end{cases}
$$

(Any required inclusion of the start corner can be handled by initializing the root state accordingly; it does not affect the transition.)

### Step 4: Correct transition

At a nonterminal node $v$ with children $v_0, v_1$, consider the variable $e(v)$.

- If we take the 0-edge: no change in visited set.
- If we take the 1-edge: we add the endpoints in $\delta(v)$.

Hence:

$$
F_v(M)
=
F_{v_0}(M)
+
\sum_{\substack{M' \subseteq S:\\ M' \cup \delta(v)=M}} F_{v_1}(M').
$$

Since $\delta(v)$ is fixed at node $v$, this is equivalently written in the invertible form:

$$
F_v(M)
=
F_{v_0}(M)
+
F_{v_1}(M \setminus \delta(v)),
$$

where the term is interpreted as $0$ whenever $\delta(v)\nsubseteq M$.

This inversion is now valid because the added set $\delta(v)$ is **completely determined by the ZDD variable at $v$**.

### Step 5: Correctness

Every root-to-terminal path in the ZDD corresponds to exactly one selection of edges forming a simple king path.

For each such path:

- the sequence of ZDD decisions uniquely determines a sequence of sets $\delta(v)$,
- the visited set in $S$ is exactly the union of these contributions,
- each root-to-terminal path is counted exactly once in exactly one $F_{\text{root}}(M)$.

Disjointness follows from the fact that distinct subsets $M \subseteq S$ partition all paths by their induced visited set.

Thus:

$$
F_{\text{root}}(M)
=
\#\{P \in \mathcal P : \mathrm{vis}(P)\cap S = M\}.
$$

### Step 6: Answer to the query in (b)

The number of paths that touch every point in a subset $T \subseteq S$ is:

$$
\#\{P : T \subseteq \mathrm{vis}(P)\}
=
\sum_{M \supseteq T} F_{\text{root}}(M).
$$

Each path is counted exactly once according to its exact intersection $M$, and the condition $T \subseteq M$ captures precisely those paths that visit all required points.

### Step 7: Complexity

For each ZDD node we maintain $2^{|S|}$ states, and each transition costs $O(2^{|S|})$. Hence:

$$
O(|Z|\,2^{|S|})
$$

time and $O(2^{|S|})$ working memory per node layer (or memoized per node).

Since $S$ consists only of corners and center points, this overhead is small relative to the ZDD size.

## Final conclusion

- Part (a): correct as stated.
- Part (b): corrected by explicitly tying the DP transition to the ZDD edge variable $e(v)$, making the visited-set update well-defined and invertible.
