**Exercise 43.** [*20*] $[20]$ Suppose you're using the undoing scheme (26) and the operation $\sigma \leftarrow \sigma + 1$ has just bumped the current stamp $\sigma$ to zero. What should you do?

**Verified:** yes  
**Solve time:** 1m17s  

---

## Solution

In the undoing scheme (26), each marked element stores a _stamp_ equal to the current global counter $\sigma$, and membership is tested by comparing a stored stamp with $\sigma$. The value $0$ is reserved for the unmarked state.

Assume $\sigma \leftarrow \sigma + 1$ has just produced $\sigma = 0$. This implies that $\sigma$ has overflowed modulo the register modulus, so the value $0$ is now simultaneously the current stamp and the designated value for “unmarked”.

At this moment, the predicate used by the scheme loses meaning: for any location $i$, the test “$i$ is marked” is implemented as

$\text{stamp}[i] = \sigma.$

If $\sigma = 0$, then this condition becomes identical to the condition for an unmarked cell, since unmarked cells also carry value $0$. Hence every entry with $\text{stamp}[i] = 0$ would be interpreted as both marked and unmarked, and the invariant that distinguishes the current level of the backtracking search from untouched memory states fails.

To restore correctness, the representation must be reinitialized so that the invariant “$\text{stamp}[i] = 0$ iff $i$ is unmarked in the current configuration” holds again with a fresh nonzero current stamp. The only consistent repair is to discard all old stamp values and restart the stamping mechanism:

set every $\text{stamp}[i] \leftarrow 0$ for all array locations used by the scheme, and reset $\sigma \leftarrow 1$ before any further marking operations.

After this reset, all membership tests again distinguish unmarked cells (value $0$) from marked cells (value $\sigma = 1$), and the undoing scheme continues to function without ambiguity.

This completes the proof. ∎
