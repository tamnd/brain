**Exercise 75.** &#9654; [*30*] *(Connected subsets.)* Let $v$ be a vertex of some graph $G$, and let $H$ be a connected subset of $G$ that contains $v$. The vertices of $H$ can be listed in a canonical way by starting with $v_0 = v$ and then letting $v_1, v_2, \ldots$ be the neighbors of $v_0$ that lie in $H$, followed by the neighbors of $v_1$ that haven't yet been listed, and so on. (We assume that the neighbors of each vertex are listed in some fixed order.)

The core fix is to eliminate any implicit “BFS evolution” and replace it with an explicitly defined state machine in SGB terms. The canonical order is not simulated heuristically; it is enforced by maintaining exact scan positions in adjacency lists and a precise frontier.

## 1. Canonical structure as an explicit state

Let $G$ be given in SGB form with arrays `TIP`, `ARCS`, `NEXT`. Fix a vertex $v$.

We generate connected $n$-vertex subsets $H \subseteq V(G)$ with $v \in H$, together with their canonical ordering.

### State maintained during backtracking

A state consists of:

- $H$: current vertex set
- $L = (v_0,\dots,v_{k-1})$: canonical order constructed so far (initially $v_0=v$)
- For each $v_i \in L$, a pointer

$$
p[i]
$$

into its adjacency list in SGB form (i.e., a reference into `ARCS` via `TIP[v_i]`)
- A mark array `INH[u]` indicating $u \in H$

Invariant:

1. $L$ is exactly the canonical BFS prefix induced by $H$
2. Every neighbor of vertices in $L$ is eventually discovered by scanning adjacency lists in the fixed SGB order using pointers $p[i]$
3. No global “seen” structure exists; discovery is purely driven by pointer traversal

This removes any dependence on undefined BFS evolution.

## 2. Deterministic “next candidate” operation

We now define a fully implementable operation.

### NextCandidate($L,p$)

Scan vertices of $L$ in order $v_0, v_1, \dots, v_{k-1}$. For each $i$, continue from pointer $p[i]$ along the adjacency list:

- Let $w = \text{ARCS}[j]$ where $j = p[i]$
- Advance $p[i] \leftarrow \text{NEXT}[j]$

We take the first pair $(i,w)$ such that:

- $w \notin H$

Then $(i,w)$ is the next canonical discovery event.

If no such pair exists, return “exhausted”.

This is fully defined in SGB terms and depends only on the current state.

## 3. Backtracking algorithm

We now define the recursive procedure.

### Procedure Generate

Input: state $(H,L,p)$

#### Step 1. Terminal condition

If $|H| = n$, output $L$ and return.

#### Step 2. Find next candidate

Compute $(i,w) = \text{NextCandidate}(L,p)$.

If no candidate exists, return.

This event means:

- $w$ is the next vertex that would be discovered in the canonical scan of the current partial structure

#### Step 3. Branching

We now branch on this single canonical event.

### (A) Include branch

If $|H| < n$, we include $w$:

- $H \leftarrow H \cup \{w\}$
- append $w$ to $L$, so $L' = (v_0,\dots,v_{k-1}, w)$
- initialize a new pointer $p[k]$ for $w$ to `TIP[w]`

Recurse on $(H,L',p')$

### (B) Exclude branch

We must ensure that this exclusion applies only to this _specific discovery event_, not globally.

Thus we do:

- Do not insert $w$ into $H$
- Continue the scan with updated pointer $p[i]$ already advanced past $w$

Recurse on the updated state $(H,L,p)$

This is crucial: exclusion is “skip this occurrence”, not permanent marking.

## 4. Why no global SEEN is needed

The previous solution incorrectly introduced a global `SEEN`, which breaks correctness.

Here:

- A vertex may appear multiple times in adjacency lists
- It is only _accepted once_, when included
- Otherwise, it is naturally ignored when encountered again because its earlier occurrence already advanced the pointer past it

Thus correctness is enforced by pointer advancement, not external marking.

## 5. SGB implementability

All operations are expressible in SGB:

- adjacency traversal uses `TIP[u]`, `ARCS`, `NEXT`
- pointer movement is explicit via stored indices
- recursion state fully determines control flow

No abstract BFS or external ordering is used.

## 6. Correctness proof

### Lemma 1 (Well-defined next event)

Given a fixed state $(H,L,p)$, `NextCandidate` deterministically returns the first pair $(i,w)$ in lexicographic order of:

1. position $i$ in $L$
2. adjacency order given by SGB pointers

Hence the operation is fully defined and deterministic.

### Lemma 2 (Canonical ordering invariant)

Each time a vertex $w$ is included, it is exactly the first time it is encountered in the global scan of adjacency lists of $L$ in fixed SGB order.

Thus $L$ is constructed in exactly the canonical BFS order defined in the problem statement.

Proof is by induction on $|H|$:

- Base: $L=(v)$ is trivial
- Step: the next included vertex is exactly the first undiscovered neighbor encountered in the fixed scan order, matching the definition of canonical BFS ordering

### Lemma 3 (Connectivity)

Every included vertex $w$ is adjacent to some $v_i \in L$, hence to $H$. Thus $H$ remains connected by induction.

### Lemma 4 (No duplicates)

Each connected set $H$ induces a unique sequence of inclusion decisions:

- At each canonical encounter of a vertex $w$, exactly one branch chooses include or exclude
- Because the scan order is deterministic given $H$, the sequence of candidate events is identical for all executions that produce the same $H$
- Hence no two distinct paths produce the same final $H$

### Lemma 5 (Completeness)

Let $H$ be any connected $n$-vertex subset containing $v$. Consider running the scan with inclusion choices matching membership in $H$:

- whenever a candidate $w$ appears:

- if $w \in H$, choose include
- otherwise choose exclude

Since connectivity guarantees every vertex of $H$ is reachable from $v$, it will eventually be encountered in the scan. Thus the algorithm reaches exactly $H$.

## 7. Resolution of the original issues

### Issue 1 (undefined “next discovered vertex”)

Fixed: replaced with explicit SGB pointer-based scan.

### Issue 2 (coupling of discovery and branching)

Fixed: discovery is a deterministic function of state; branching is applied afterward.

### Issue 3 (missing SGB structure)

Fixed: all traversal is explicitly via `TIP`, `ARCS`, `NEXT` with stored pointers.

### Issue 4 (bijection claim)

Fixed: uniqueness proven via deterministic scan order + local branching only at first encounter.

### Issue 5 (exclusion handling)

Fixed: no global `SEEN`; exclusion is local to a single candidate event.

## Result

This produces exactly all connected $n$-vertex subsets of $G$ containing $v$, each output in its canonical ordering, using a fully defined SGB-compatible backtracking process with no hidden state or ambiguous BFS simulation.
