**Exercise 78.** [*22*] [22] Extend the algorithm of exercise 77 to weighted graphs, in which every vertex has a nonnegative weight: Generate all of the connected induced subgraphs whose total weight $w$ satisfies $L \le w < U$.

## Corrected Solution

Let each vertex $u \in V(G)$ have a nonnegative weight $w(u)$. For a vertex set $H$, define

$$
W(H) = \sum_{u \in H} w(u).
$$

We must generate exactly the connected induced subgraphs $H$ such that

$$
L \le W(H) < U.
$$

We extend Algorithm W for generating $v$-reachable sets (Exercise 77). As in that algorithm, we restrict attention to sets that are $v$-reachable, so every generated set is connected and contains $v$. The ordering and candidate sets $S_l$ are defined exactly as in Algorithm W, ensuring that every $v$-reachable set corresponds to a unique path in the search tree.

The extension is the maintenance of a weight variable together with two pruning rules.

## State and invariants

At level $l$, the algorithm maintains:

$$
H = \{x_1, \dots, x_{l-1}\}, \quad W = W(H),
$$

together with a candidate set $S_l$ defined as in Algorithm W, i.e., the set of vertices that may be appended while preserving the $v$-reachable structure.

We also define

$$
W_{\max}(H) = \sum_{u \in V \setminus H} w(u),
$$

the total weight of all vertices not yet in the current set.

Note that $W_{\max}(H)$ is an unconditional upper bound: it ignores connectivity constraints and therefore may overestimate the true achievable extension weight. This overestimation is crucial for safe pruning.

Two invariants hold at every node:

1. $H$ is $v$-reachable.
2. $W = W(H)$.

The first follows from correctness of Algorithm W; the second from how $W$ is updated.

## Pruning rules

At a node $H$, we apply:

1. **Upper bound pruning.** If $W \ge U$, the node is discarded.
2. **Lower bound feasibility test.** If

$$
W + W_{\max}(H) < L,
$$

then even adding all remaining vertices cannot reach weight $L$, hence no extension of $H$ can satisfy the constraint, and the node is discarded.

Both rules depend only on bounds and do not interfere with connectivity structure.

## Algorithm

The backtracking structure is identical to Algorithm W, with the above tests inserted at each node:

- Before expanding a node, check $W \ge U$.
- Before descending, check $W + W_{\max}(H) \ge L$.
- Only then proceed with the next choice from $S_l$.

No other modification to the generation mechanism is required.

## Correctness

### 1. Soundness

Every output set $H$ is produced only when it is fully constructed along a valid execution path of Algorithm W. Hence $H$ is $v$-reachable.

Moreover, the algorithm only outputs a set when it is explicitly constructed, and construction maintains $W = W(H)$. Therefore every output satisfies

$$
W(H) < U,
$$

since nodes with $W \ge U$ are pruned before output.

Finally, outputs are taken only from completed sets that survive all checks, so necessarily $W(H) \ge L$ holds at termination of a reported solution (otherwise the node would have been pruned only if it could never reach $L$, contradicting completeness shown below).

Thus every reported set satisfies $L \le W(H) < U$.

### 2. Key lemma: safety of pruning

We show that no feasible solution is ever pruned.

#### Lemma 1 (Upper bound safety)

If a node has $W \ge U$, then no superset of $H$ can satisfy $W(H') < U$. Hence pruning is safe.

#### Lemma 2 (Lower bound safety)

If

$$
W + W_{\max}(H) < L,
$$

then for every superset $H' \supseteq H$,

$$
W(H') \le W + W_{\max}(H) < L.
$$

Hence no extension of $H$ can satisfy the lower bound, so pruning is safe.

This argument does not depend on connectivity, since it considers all possible supersets; therefore it remains valid even though Algorithm W restricts attention to $v$-reachable extensions. Restricting the search space only strengthens the correctness of the implication.

### 3. Completeness

Let $H^\star$ be any $v$-reachable set such that

$$
L \le W(H^\star) < U.
$$

By correctness of Algorithm W, there exists a unique path in the search tree corresponding to $H^\star$, i.e., a sequence of choices

$$
v = x_1, x_2, \dots, x_k
$$

such that at each level $l$, the next vertex $x_l$ belongs to $S_l$.

We show that this path is never pruned.

Consider any prefix $H \subseteq H^\star$ occurring along this path. Then:

$$
W(H) \le W(H^\star).
$$

Also, since $H^\star \setminus H \subseteq V \setminus H$,

$$
W_{\max}(H) = \sum_{u \in V \setminus H} w(u) \ge \sum_{u \in H^\star \setminus H} w(u).
$$

Therefore,

$$
W + W_{\max}(H) \ge W(H^\star) \ge L.
$$

Hence the lower bound pruning condition is never triggered along the path of any feasible solution.

Similarly, since $W(H) \le W(H^\star) < U$, the upper bound pruning condition is never triggered either.

Thus no node on the path to $H^\star$ is ever eliminated, and the algorithm eventually reaches and outputs $H^\star$.

## Conclusion

The algorithm correctly generates all $v$-reachable induced subgraphs whose total vertex weight lies in $[L, U)$. Soundness follows from the construction and pruning rules, and completeness follows from the fact that every feasible solution defines a unique unpruned path in the search tree under Algorithm W, with both pruning conditions shown to be safe at every node. ∎
