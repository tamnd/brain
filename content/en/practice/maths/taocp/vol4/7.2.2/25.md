**Exercise 25.** &#9654; [*25*] *[25]* Explain how to use *orthogonal lists*, as in Fig. 13 of Section 7.2.6, so that it's easy to visit all 5-letter words whose $k$th character is $c$, given $1 \le k \le 5$ and $a \le c \le z$. Use those sublists to speed up the algorithm of exercise 24.

## Solution

Let $W$ be the set of all 5-letter words over ${a,\ldots,z}$. For each pair $(k,c)$ with $1 \le k \le 5$ and $a \le c \le z$, define the subset

$$
W_{k,c} = \{\, w \in W \mid w_k = c \,\}.
$$

Construct orthogonal lists in the sense of Fig. 13 of Section 7.2.6 by representing each word $w \in W$ as a single node that belongs simultaneously to five position lists and to twenty-six letter classes at each position. Concretely, for each fixed pair $(k,c)$, all words in $W_{k,c}$ are linked into a doubly linked list, and each word node contains five pointers, one for each position class it belongs to.

Each node representing a word $w = w_1 w_2 w_3 w_4 w_5$ is inserted into exactly five lists $W_{1,w_1}, \ldots, W_{5,w_5}$. The lists are made circular or terminated with sentinels so that deletion and reinsertion of a node can be done by pointer adjustment in constant time.

Consider now the backtracking construction of a $5 \times 6$ word rectangle as in exercise 24. At level $l$, a partial rectangle determines constraints on candidate words for the next column or row, expressed as fixed character conditions. Suppose the algorithm requires a word whose $k$th character is $c$. Instead of scanning all words and testing the condition, the algorithm accesses the list $W_{k,c}$ directly.

When multiple constraints are active, say positions $k_1, \ldots, k_r$ with prescribed letters $c_1, \ldots, c_r$, the candidate set is

$$
\bigcap_{i=1}^r W_{k_i,c_i}.
$$

This intersection is evaluated by iterating through the smallest of these lists and testing membership in the others using back-pointers into the orthogonal structure. Each membership test is reduced to checking whether the node is currently present in the corresponding list, which is a constant-time operation because deletion in orthogonal lists removes all incident pointers consistently.

During backtracking, when a word is chosen for placement in the rectangle, the algorithm temporarily removes that node from all lists $W_{k,w_k}$ for $1 \le k \le 5$. This ensures that no word is reused in the same rectangle and that future intersections automatically exclude the chosen word. Restoration on backtracking reinstates the node into the same lists in reverse order of deletion, using stored neighbor pointers.

The speed improvement in exercise 24 arises from replacing global enumeration of all words by localized traversal of constraint lists. In step B3 of the underlying backtrack algorithm, instead of testing all words in $W$, the algorithm iterates only through $W_{k,c}$ for the next required constraint. When several constraints are simultaneously present, the algorithm selects a starting list $W_{k,c}$ of minimal current length and filters its elements by constant-time membership tests in the remaining lists.

The orthogonal list structure guarantees that each candidate word is examined only when it satisfies at least one active positional constraint, and each violation is detected without scanning unrelated words. Thus the work per node in the search tree becomes proportional to the size of the smallest active constraint list rather than to $|W|$.

This completes the construction of the data structure and its integration into the backtracking algorithm for $5 \times 6$ word rectangles. ∎
