**Exercise 61.** [*HM26*] Let $P_n$ be the number of integer sequences $x_1 \ldots x_n$ such that $x_1 = 1$ and $1 \le x_{k+1} \le 2x_k$ for $1 \le k < n$. (The first few values are 1, 2, 6, 20, 76, 2280, ...; this sequence was introduced by A. Cayley in *Philosophical Magazine* (4) **13** (1857), 245–248, who showed that $P_n$ enumerates the partitions of $2^n - 1$ into powers of 2.)

a) Show that $P_n$ is the number of different profiles that are possible for a binary tree of height $n$.

b) Find an efficient way to compute $P_n$ for large $n$. *Hint:* Consider the more general sequence $P_n^{(m)}$, defined similarly but with $x_1 = m$.

c) Use the estimation of exercise 57 (Theorem E') to show that $P_n \ge \left\lfloor\frac{2^n}{3}/(n-1)\right\rfloor$.

## (a) Profiles of binary trees

Let a binary tree of height $n$ be given, and let $x_k$ denote the number of nodes at level $k$. Then:

- The root gives $x_1 = 1$.
- Each node has at most two children, hence $x_{k+1} \le 2x_k$.
- Height $n$ implies every level is nonempty, so $x_k \ge 1$.

Thus every profile satisfies

$$
x_1 = 1, \qquad 1 \le x_{k+1} \le 2x_k.
$$

Conversely, given a sequence satisfying these inequalities, construct level by level. At level $k$, there are $x_k$ nodes, hence at most $2x_k$ available child slots. Since $x_{k+1} \le 2x_k$, we can assign each node at level $k+1$ a distinct parent at level $k$, distributing children so that exactly $x_{k+1}$ nodes are created. Proceeding inductively yields a binary tree with profile $(x_1,\dots,x_n)$.

This construction is reversible, hence the sequences are in bijection with binary tree profiles of height $n$. Therefore $P_n$ equals the number of such profiles.

## (b) Efficient computation of $P_n$

Define the generalized quantity $P_n^{(m)}$ as the number of sequences $x_1,\dots,x_n$ such that

$$
x_1 = m, \qquad 1 \le x_{k+1} \le 2x_k.
$$

Then $P_n = P_n^{(1)}$.

From a state $m$, the next value $x_2$ can be any integer in $[1,2m]$, hence

$$
P_n^{(m)} = \sum_{x=1}^{2m} P_{n-1}^{(x)}, \qquad P_1^{(m)} = 1.
$$

Let $A^{(n)}(m) = P_n^{(m)}$. Then

$$
A^{(n)}(m) = \sum_{x \le 2m} A^{(n-1)}(x).
$$

If we maintain prefix sums

$$
S^{(n-1)}(t) = \sum_{x \le t} A^{(n-1)}(x),
$$

then

$$
A^{(n)}(m) = S^{(n-1)}(2m).
$$

Thus each level is computable from the previous one by prefix-sum preprocessing followed by constant-time queries per state. Since all reachable values satisfy $x_k \le 2^{k-1}$, the state space at level $k$ has size $O(2^k)$. Therefore the total running time is

$$
\sum_{k=1}^n O(2^k) = O(2^n).
$$

This gives an efficient dynamic programming algorithm for $P_n$.

## (c) Lower bound on $P_n$

We construct an explicit large family of valid sequences.

Starting from $x_1 = 1$, at each step choose

$$
x_{k+1} \in \{x_k,\, 2x_k\}.
$$

Both choices satisfy the constraint $1 \le x_{k+1} \le 2x_k$, since $x_k \ge 1$. Hence every binary string $(\varepsilon_1,\dots,\varepsilon_{n-1}) \in \{0,1\}^{n-1}$, interpreted as

$$
x_{k+1} =
\begin{cases}
x_k & \varepsilon_k = 0,\\
2x_k & \varepsilon_k = 1,
\end{cases}
$$

produces a valid sequence.

This mapping is injective, so it yields $2^{n-1}$ distinct valid sequences. Therefore

$$
P_n \ge 2^{n-1}.
$$

Finally, for $n \ge 2$,

$$
2^{n-1} = \frac{2^n}{2} \ge \frac{2^n}{3(n-1)},
$$

since $3(n-1) \ge 2$. Hence

$$
P_n \ge \left\lfloor \frac{2^n}{3}/(n-1) \right\rfloor.
$$

## Conclusion

- (a) The profile constraints exactly characterize binary tree level structures, giving a bijection.
- (b) A dynamic programming recurrence with prefix sums computes $P_n$ in $O(2^n)$ time.
- (c) A direct injection from binary strings yields $P_n \ge 2^{n-1}$, which implies the required bound.
