**Exercise 1.** &#9654; [**] [22] Explain how the tasks of generating (i) $n$-tuples, (ii) permutations of distinct items, (iii) combinations, (iv) integer partitions, (v) set partitions, and (vi) nested parentheses can all be regarded as special cases of backtrack programming, by presenting suitable domains $D_k$ and cutoff properties $P_l(x_1, \ldots, x_l)$ that satisfy (1) and (2).

We formalize each of the six generation tasks as an instance of Algorithm B in the sense of TAOCP: we construct sequences $(x_1,\dots,x_n)$ where each $x_k$ ranges over a finite domain $D_k$, and we define cutoff predicates $P_l(x_1,\dots,x_l)$ that guarantee that every partial sequence that violates the defining structural constraints is rejected immediately, while every valid object has at least one feasible extension to a full solution.

In each case, a solution corresponds to a complete sequence of length $n$ (or a fixed length depending on the object), and acceptance is defined by reaching a terminal level where the sequence encodes exactly the desired combinatorial structure.

### (i) $n$-tuples

We take a fixed positive integer $n$ and define for every level $k$ the domain

$$
D_k = \{1,2,\dots,n\}.
$$

A full solution is any sequence $(x_1,\dots,x_n)$.

The cutoff predicate is trivial:

$$
P_l(x_1,\dots,x_l) \equiv \text{true}.
$$

Since no constraint is imposed at any prefix, every partial sequence is extendable to a full sequence, and every node in the search tree is expanded until depth $n$. Thus Algorithm B generates exactly all $n$-tuples over ${1,\dots,n}$.

### (ii) Permutations of distinct items

We again take

$$
D_k = \{1,2,\dots,n\}, \quad 1 \le k \le n,
$$

and we interpret a solution as a sequence $(x_1,\dots,x_n)$.

The cutoff predicate enforces distinctness:

$$
P_l(x_1,\dots,x_l) \equiv (\forall\, i<j \le l,\; x_i \ne x_j).
$$

A partial sequence is feasible exactly when it contains no repeated values. A node at level $l<n$ is expanded only if it can still be completed without repetition; a node at level $n$ is a solution exactly when it contains all $n$ distinct elements. Hence Algorithm B generates all permutations of ${1,\dots,n}$ exactly once.

### (iii) Combinations of size $k$ from ${1,\dots,n}$

We fix integers $1 \le k \le n$. A combination is represented by a strictly increasing sequence $(x_1,\dots,x_k)$.

Thus we set

$$
D_l = \{1,2,\dots,n\} \quad \text{for } 1 \le l \le k.
$$

The cutoff predicate for a partial sequence $(x_1,\dots,x_l)$ is

$$
P_l(x_1,\dots,x_l)
$$

holds if and only if both of the following conditions are satisfied:

First, strict increase is maintained:

$$
x_1 < x_2 < \cdots < x_l.
$$

Second, the sequence can still be completed to length $k$ without exceeding $n$. This is enforced by the standard feasibility bound

$$
x_l \le n - (k-l).
$$

(For $l=0$, no constraint is imposed.)

A node is terminal exactly when $l=k$. In that case, the sequence encodes a $k$-element subset. Every $k$-subset has a unique strictly increasing representation, and every partial increasing sequence satisfying the bound extends to at least one full valid combination. Hence Algorithm B generates all combinations of size $k$ exactly once.

### (iv) Integer partitions of a fixed integer $N$

A partition is represented in nonincreasing form as a sequence $(x_1,\dots,x_l)$ of positive integers with

$$
x_1 \ge x_2 \ge \cdots \ge x_l, \quad x_i \ge 1, \quad x_1+\cdots+x_l = N.
$$

We allow variable length, so we take

$$
D_k = \{1,2,\dots,N\} \quad \text{for all } k.
$$

A partial sequence $(x_1,\dots,x_l)$ is feasible if it satisfies two conditions:

First, monotonicity:

$$
x_1 \ge x_2 \ge \cdots \ge x_l.
$$

Second, the partial sum does not exceed $N$:

$$
x_1 + \cdots + x_l \le N.
$$

Thus we define

$$
P_l(x_1,\dots,x_l) \equiv \left(x_1 \ge \cdots \ge x_l\right) \land \left(\sum_{i=1}^l x_i \le N\right).
$$

The terminal condition is that a node is accepted if and only if its sum equals $N$:

$$
\sum_{i=1}^l x_i = N.
$$

No further extension is possible at such a node, since all parts are positive. Conversely, any sequence with sum strictly less than $N$ is extended only if the next part does not violate monotonicity and does not force the sum to exceed $N$. Therefore Algorithm B generates each integer partition of $N$ exactly once.

### (v) Set partitions of ${1,2,\dots,n}$

We encode a set partition by a restricted growth sequence $(x_1,\dots,x_n)$, where $x_i$ indicates the block containing element $i$. To ensure canonical representation, we require:

$$
x_i \in \{1,2,\dots\}.
$$

We set

$$
D_k = \{1,2,\dots,k\}.
$$

The cutoff predicate $P_l(x_1,\dots,x_l)$ is defined by two constraints:

First, restricted growth:

$$
x_1 = 1,
$$

and for all $i \ge 2$,

$$
x_i \le 1 + \max(x_1,\dots,x_{i-1}).
$$

Second, feasibility of extension is automatic from this rule, since any new block label is introduced only as the next unused integer, guaranteeing canonical labeling.

Thus

$$
P_l(x_1,\dots,x_l)
$$

holds if and only if the restricted growth condition is satisfied for all indices up to $l$.

A complete sequence of length $n$ corresponds uniquely to a set partition of ${1,\dots,n}$, and every set partition induces exactly one such restricted growth sequence. Therefore Algorithm B generates all set partitions exactly once.

### (vi) Nested parentheses (Dyck words)

We represent a Dyck word of semilength $n$ as a sequence of length $2n$ over

$$
D_k = \{(,)\}.
$$

We encode $(x_1,\dots,x_{2n})$ where $($ corresponds to $1$ and $)$ corresponds to $0$.

The cutoff predicate $P_l(x_1,\dots,x_l)$ requires two conditions.

First, the prefix balance condition:

$$
\#\{i \le l : x_i = (\} \;\ge\; \#\{i \le l : x_i = )\}.
$$

Second, a feasibility condition ensuring completion to length $2n$. If $o_l$ is the number of opening parentheses and $c_l$ the number of closing parentheses in the prefix, then we must have

$$
o_l \le n, \quad c_l \le n.
$$

Thus we define

$$
P_l(x_1,\dots,x_l) \equiv (o_l \ge c_l) \land (o_l \le n) \land (c_l \le n).
$$

A node at depth $2n$ is accepted exactly when

$$
o_{2n} = c_{2n} = n.
$$

The prefix condition guarantees that no partial sequence can violate balance, and the bounds guarantee that completion to a full balanced word remains possible. Every Dyck word corresponds to exactly one such sequence, and every feasible path reaches a valid completion if and only if it respects the cutoff conditions. Hence Algorithm B generates all nested parenthesis sequences of length $2n$.

This completes the formulation of all six combinatorial generation tasks as instances of backtrack programming under Algorithm B.
