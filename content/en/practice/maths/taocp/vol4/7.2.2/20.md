**Exercise 20.** &#9654; [**] *[21]* Extend Algorithm L so that it forces $x_l \leftarrow k$ whenever $k \notin \{x_1, \ldots, x_{l-1}\}$ and $l \ge 2n - k - 1$.

**Verified:** yes  
**Solve time:** 2m21s  

---

In Algorithm L, the variable $x_l$ ranges over admissible integers $k \in {1,\dots,n}$ such that $k$ has not yet been used as a first occurrence and the constraint $x_j = k \Rightarrow x_{j+k+1} = -k$ can still be satisfied within positions $1,\dots,2n$. The proposed extension inserts a forced-choice rule into this selection process.

When the algorithm is at level $l$ and considers a value $k$ with $k \notin {x_1,\ldots,x_{l-1}}$ and $l \ge 2n - k - 1$, the assignment $x_l \leftarrow k$ is made without testing alternative values. This modification is placed at the beginning of the step where candidates for $x_l$ are normally generated, so that the usual backtracking test is bypassed whenever the forcing condition holds.

To justify correctness, suppose $k$ has not yet appeared in ${x_1,\ldots,x_{l-1}}$. In any valid Langford placement, the first occurrence of $k$ must be at some position $j$ satisfying $j+k+1 \le 2n$, hence $j \le 2n-k-1$. If the algorithm has reached a level $l$ with $l \ge 2n-k-1$, then every admissible first occurrence position for $k$ is at or before level $l$, and no valid completion can postpone placing the first copy of $k$ beyond this point. Consequently, whenever $k$ is still unused at level $l$, the only consistent continuation is to place it at the current position, which enforces $x_l \leftarrow k$. This preserves all valid completions while eliminating branches that cannot lead to a solution. ∎
