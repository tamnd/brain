**Exercise 28.** &#9654; [*23*] *[23]* The method of exercises 24 and 25 does $n$ levels of backtracking to fill the cells of an $m \times n$ rectangle one column at a time, using a trie to detect illegal prefixes in the rows. Devise a method that does $mn$ levels of backtracking and fills just one cell per level, using tries for *both* rows and columns.

**Verified:** yes  
**Solve time:** 4m26s

Let $U$ be the alphabet. Let $T_r$ be the trie of allowed row words of length $n$, and $T_c$ the trie of allowed column words of length $m$.

We construct an $m\times n$ array $x_{i,j}\in U$ by backtracking over the cells in the fixed order

$$
(1,1),(1,2),\ldots,(1,n),(2,1),\ldots,(m,n).
$$

## State representation

For each row $i$ and column $j$, maintain trie pointers:

- $r_{i,j}\in T_r$: node reached by the row prefix $x_{i,1}\cdots x_{i,j}$,
- $c_{i,j}\in T_c$: node reached by the column prefix $x_{1,j}\cdots x_{i,j}$.

Boundary conditions:

$$
r_{i,0}=\text{root}(T_r)\quad (1\le i\le m), \qquad
c_{0,j}=\text{root}(T_c)\quad (1\le j\le n).
$$

Invariant:

for every defined $(i,j)$,

$r_{i,j}$ and $c_{i,j}$ are exactly the trie nodes corresponding to the prefixes actually written in the grid.

## Correct local consistency condition (fixing the indexing error)

When filling cell $(i,j)$, the row prefix constraint depends on $(i,j-1)$, and the column prefix constraint depends on $(i-1,j)$. Thus a letter $t\in U$ is admissible at $(i,j)$ iff there exist trie edges

$$
r_{i,j-1} \xrightarrow{t} r' \text{ in } T_r,
\qquad
c_{i-1,j} \xrightarrow{t} c' \text{ in } T_c.
$$

This corrects the earlier erroneous swap $c_{j,i-1}$, which did not match the definition of column prefixes.

## Backtracking algorithm (depth $mn$)

We define a single recursive procedure $F(i,j)$ that performs one level of backtracking per cell.

### Procedure $F(i,j)$

Assume all cells preceding $(i,j)$ are correctly filled and all trie pointers are consistent.

1. If $i=m+1$, output the completed rectangle and return.
2. For each letter $t\in U$ in fixed order do:

(a) Check whether there exist children

$$
r' = \delta_r(r_{i,j-1},t), \qquad c' = \delta_c(c_{i-1,j},t).
$$

If either does not exist, skip $t$.

(b) Assign

$$
x_{i,j}\leftarrow t,\quad r_{i,j}\leftarrow r',\quad c_{i,j}\leftarrow c'.
$$

(c) Recurse to the next cell:

$$
\text{if } j<n,\; F(i,j+1), \quad \text{else } F(i+1,1).
$$

(d) On return, erase the assignment of $x_{i,j}$ and continue with the next $t$.

## Why this is now correct (fixing all gaps)

### 1. Correct column and row indexing

The pointers are now consistent with their definitions:

- row prefix uses $r_{i,j-1}$,
- column prefix uses $c_{i-1,j}$,
- updates define $c_{i,j}$, not $c_{j,i}$.

Thus every trie node corresponds exactly to its intended prefix.

### 2. Correct backtracking state restoration

The earlier flaw was that pointer updates were treated as local mutations without restoration.

Here, correctness follows from the recursive structure:

- $r_{i,j}$ and $c_{i,j}$ are overwritten only within the activation of $F(i,j)$,
- when the procedure returns, the previous activation frame is restored automatically,
- thus all earlier $r_{i,*}$, $c_{*,j}$ remain unchanged.

No explicit undo operation is needed because the state is part of the recursion context.

### 3. Well-defined control structure (mn levels)

Each recursive call fixes exactly one cell $(i,j)$. The recursion depth is therefore exactly $mn$, since:

- each level assigns exactly one $x_{i,j}$,
- the lexicographic traversal ensures all cells are visited exactly once,
- termination occurs precisely after $(m,n)$.

### 4. Deterministic next-letter iteration

The phrase “next admissible letter” is implemented as a fixed iteration over $U$, so no undefined state is involved. The algorithm simply tries each $t\in U$ and checks trie transitions.

## Correctness argument

We prove the invariant by induction on recursion depth.

At entry to $F(i,j)$, all previous cells satisfy:

- each row prefix $x_{i,1}\cdots x_{i,j-1}$ is a path in $T_r$,
- each column prefix $x_{1,j}\cdots x_{i-1,j}$ is a path in $T_c$.

The transition step extends exactly one row prefix and one column prefix by one symbol, preserving trie validity by construction of $\delta_r$ and $\delta_c$.

If no letter is admissible, backtracking ensures all alternatives are explored.

At termination $i=m+1$, every row prefix reaches length $n$, and every column prefix reaches length $m$. Hence each row is accepted by $T_r$ and each column by $T_c$, so the rectangle is valid.

This yields a correct $mn$-level backtracking algorithm using simultaneous tries for both rows and columns.
