**Exercise 44.** &#9654; [*25*] $[25]$ Spell out the low-level implementation details of the candidate selection process in step C2 of Algorithm C. Use the routine store$(n, c)$ of (26) whenever changing the contents of MEM. Assume the following selection strategy:

a) Find a class $c$ with the least number $r$ of blue words.

b) If $r = 0$, set $x \leftarrow -1$; otherwise set $x$ to a word in class $c$.

c) If $r > 1$, use the heuristic to find an $x$ that minimizes the number of blue words that could be killed on the other side of the prefix or suffix list that contains $x$.

Let MEM contain the standard TAOCP representation: class headers form a linked structure in MEM, and each word node contains pointers into its prefix list and suffix list together with a color bit indicating whether it is blue.

Only permanent structural changes in MEM are ever recorded with `store(n,c)`. No use of `store` is permitted for simulation, marking, or undo.

## 1. Selection of a class of minimum blue size

Let `CHEAD` be the header of the list of all classes. Each class header node `c` contains a pointer `NEXTCLASS(c)` to the next class and a pointer `WORDHEAD(c)` to the first word in that class.

Define a function `BLUECOUNT(c)` that returns the number of blue words in class `c`:

```
BLUECOUNT(c):
    t ← WORDHEAD(c)
    r ← 0
    while t ≠ 0 do
        if BLUE(t) = 1 then r ← r + 1
        t ← NEXTWORD(t)
    return r
```

Now perform a full scan of the class list:

```
c ← CHEAD
c* ← 0
r* ← ∞

while c ≠ 0 do
    r ← BLUECOUNT(c)
    if r < r* then
        c* ← c
        r* ← r
    c ← NEXTCLASS(c)
```

After termination, `c*` is a class with minimal number of blue words.

This part requires no calls to `store(n,c)` because no MEM cell is changed.

## 2. Case distinction on the minimum class

If `r* = 0`, no class contains a blue word and the algorithm sets

$$
x \leftarrow -1.
$$

No MEM modification occurs, hence no `store` is used.

Assume now $r^* > 0$.

## 3. Selection of a blue word in the chosen class

Traverse the word list of class `c*` and enumerate only blue words.

```
t ← WORDHEAD(c*)
x ← 0
bestκ ← ∞
```

For each word node `t` in the class:

```
while t ≠ 0 do
    if BLUE(t) = 1 then
        κ ← HEURISTIC(t)
        if κ < bestκ then
            bestκ ← κ
            x ← t
    t ← NEXTWORD(t)
```

## 4. Correct low-level implementation of the heuristic κ(w)

The quantity κ(w) must be computed purely by inspection of MEM pointers. No simulation, marking, or tentative deletion is performed.

Let each word node `w` contain two adjacency lists:

- `P(w)`: list of occurrences of `w` in prefix structures
- `S(w)`: list of occurrences of `w` in suffix structures

Each occurrence node contains a pointer to the opposite-side word that would be affected if `w` were selected.

The heuristic counts how many blue words would be eliminated on the opposite side.

```
HEURISTIC(w):
    κ ← 0

    u ← P(w)
    while u ≠ 0 do
        v ← OPPOSITE(u)
        if BLUE(v) = 1 then κ ← κ + 1
        u ← NEXTP(u)

    u ← S(w)
    while u ≠ 0 do
        v ← OPPOSITE(u)
        if BLUE(v) = 1 then κ ← κ + 1
        u ← NEXTS(u)

    return κ
```

This computation is purely read-only. It does not modify MEM and therefore does not use `store`.

## 5. Role of `store(n,c)` and correction of the critical error

The previous solution incorrectly used `store(n,c)` as if it supports reversible operations (“apply then restore”). This is not valid.

In Algorithm C style implementations, `store(n,c)` is used only when a **permanent change to MEM is made**, typically when a word is actually removed or a structural link is updated during the execution of the main algorithm.

In the candidate selection step C2:

- No deletion is performed.
- No node is relinked.
- No color bit is changed.

Therefore:

$$
\text{store}(n,c) \text{ must not be used anywhere in Step C2.}
$$

Any attempt to simulate deletion for heuristic evaluation must be replaced by direct pointer-based counting as shown above.

## 6. Final output of Step C2

The step returns:

- $(c^*, x)$ if $r^* > 0$, where $x$ is the blue word in $c^*$ minimizing $\kappa(x)$
- $x = -1$ if $r^* = 0$

No MEM modification occurs in this step, and therefore no `store` calls are issued.

This fully specifies the low-level MEM traversal and corrects the misuse of `store(n,c)` and the lack of pointer-level implementation of the heuristic.
