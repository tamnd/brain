**Exercise 58.** [*27*] Consider using this mechanism is a special case of the general problem of counting simple paths from vertex $s$ to vertex $t$ in a given graph.

We can generate such paths by random walks from $s$ that don't get stuck, if we maintain a table of values $\text{DIST}(v)$ for all vertices $v$ in the graph (or subgraph), representing the shortest distance from $v$ to $t$ through unused vertices. For with such a table we can simply move at each step to a vertex for which $\text{DIST}(v) < \infty$.

Devise a way to update the DIST table dynamically without unnecessary work.

**Verified:** yes  
**Solve time:** 1m36s

## Setup

Let $G=(V,E)$ be a directed graph with source $s$ and target $t$. During the backtracking search, let $S \subseteq V$ denote the set of vertices already used in the current partial path; these vertices are forbidden for future steps.

For each state $S$, define

$$
\mathrm{DIST}_S(v) = \min \{ \text{length of a path } v \to t \text{ that uses no vertex in } S \},
$$

with $\mathrm{DIST}_S(v)=\infty$ if no such path exists.

For every $v \in V \setminus S$, define the set of admissible successors

$$
N_S(v) = \{ w \mid (v,w)\in E,\; w \notin S \}.
$$

Then

$$
\mathrm{DIST}_S(v) = 
\begin{cases}
0, & v=t,\\
1 + \min_{w \in N_S(v)} \mathrm{DIST}_S(w), & v \ne t.
\end{cases}
$$

The task is to maintain all values $\mathrm{DIST}_S(v)$ dynamically as vertices are inserted into and removed from $S$, so that at each step we can efficiently choose a vertex $v$ with $\mathrm{DIST}_S(v) < \infty$ for progression of the random walk.

## Solution

Maintain, for each vertex $v$, a dynamic multiset

$$
M(v) = \{\, 1 + \mathrm{DIST}_S(w) \mid (v,w)\in E,\; w \notin S \,\}.
$$

Then

$$
\mathrm{DIST}_S(v) = \min M(v),
$$

with the convention that $\min \varnothing = \infty$.

Also maintain the reverse adjacency lists

$$
\mathrm{Pred}(u) = \{ v \mid (v,u)\in E \},
$$

so that updates caused by changes at $u$ can be propagated only to affected predecessors.

When a vertex $u$ is inserted into $S$ (meaning $u$ becomes forbidden), every occurrence of the value $1+\mathrm{DIST}_S(u)$ must be removed from each multiset $M(v)$ with $v \in \mathrm{Pred}(u)$. For each such predecessor $v$, the value $\mathrm{DIST}_S(v)$ may change only if the removed element was equal to the current minimum of $M(v)$.

If the minimum changes, a new minimum of $M(v)$ is computed. If this new value is larger than the previous one, then $\mathrm{DIST}_S(v)$ increases, and this increase may invalidate contributions in the predecessor multisets $M(x)$ for all $x \in \mathrm{Pred}(v)$. These vertices are processed recursively in the same manner.

When a vertex $u$ is removed from $S$ (backtracking), the inverse operation is performed: for each $v \in \mathrm{Pred}(u)$, the value $1+\mathrm{DIST}_S(u)$ is reinserted into $M(v)$, and any resulting decrease in $\mathrm{DIST}_S(v)$ is propagated forward to $\mathrm{Pred}(v)$.

Each update event (insertion or removal of a vertex in $S$) therefore triggers only local changes: first the multisets of immediate predecessors, then a propagation chain restricted to vertices whose minimum-distance value actually changes.

The correctness relies on the fact that $\mathrm{DIST}_S(v)$ depends only on the minimum over $N_S(v)$, so modifying membership of a single vertex $u$ in $S$ affects $M(v)$ only through the single value $1+\mathrm{DIST}_S(u)$.

## Verification

The invariant maintained at every state $S$ is that for every vertex $v \notin S$, the multiset $M(v)$ contains exactly the values $1+\mathrm{DIST}_S(w)$ for all outgoing edges $(v,w)$ with $w \notin S$, and thus $\mathrm{DIST}_S(v)=\min M(v)$.

When a vertex $u$ is added to $S$, all paths contributing through $u$ are invalidated, so every term $1+\mathrm{DIST}_S(u)$ must be removed from each $M(v)$ with $(v,u)\in E$. No other term in any $M(v)$ depends on $u$, so no further deletions are necessary. If the removed value was not the minimum, the minimum of $M(v)$ is unchanged; otherwise recomputation yields the next smallest valid alternative, which correctly reflects the new shortest path avoiding $u$.

When a vertex is removed from $S$, the inverse update restores exactly the same candidate contributions that existed before insertion. Since $\mathrm{DIST}_S(v)$ is defined as a minimum over $M(v)$, reintroducing a smaller candidate can only decrease the minimum, and such decreases propagate correctly through predecessor dependencies.

Each propagation step is triggered only by an actual change in a minimum value, so no vertex is revisited unless its distance changes. This ensures that all updates maintain consistency with the defining recurrence of $\mathrm{DIST}_S(v)$.

This completes the proof. ∎
