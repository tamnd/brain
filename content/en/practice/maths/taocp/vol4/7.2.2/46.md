**Exercise 46.** [*M35*] $[M35]$ Is there a binary ($m = 2$) commafree code with one codeword in each of the $\bigl(\sum_{j,n} \phi(d) 2^{j/k}\bigr)/n$ cycle classes, for every word length $n$?

**Verified:** yes  
**Solve time:** 2m44s  

---

The previous proof fails because it tries to force a contradiction via incorrect claims about cyclic equivalence. A correct argument does not use those substrings at all. It instead exploits a forced interaction between a single nontrivial cycle class and the constant word $0^n$.

## Correct solution

Assume $n \ge 2$. Let $C \subset \{0,1\}^n$ be a binary commafree code that contains exactly one representative from each cycle class (necklace).

The total number of cycle classes includes the class of constant words, so in particular the singleton word

$$
0^n \in C.
$$

Now consider the cycle class containing

$$
u = 0^{n-1}1, \qquad v = 10^{n-1}.
$$

These are cyclic shifts of each other, hence they belong to the same cycle class. Since $C$ contains exactly one representative from each cycle class, it contains exactly one of $u$ or $v$. Let this chosen word be $w$.

We now show that either choice leads to a violation of the commafree property.

## Case 1: $w = 0^{n-1}1$

Consider the concatenation $0^n w$:

$$
0^n \, 0^{n-1}1.
$$

Look at the substring of length $n$ starting at position $2$. This substring consists of:

- the last $n-1$ symbols of $0^n$, which are $0^{n-1}$,
- followed by the first symbol of $w$, which is $0$.

Hence the substring is

$$
0^{n-1}0 = 0^n.
$$

This is a codeword in $C$, but it occurs starting at position $2$, i.e. not at the boundary position $1$. This contradicts the commafree property.

## Case 2: $w = 10^{n-1}$

Consider the concatenation $w\,0^n$:

$$
10^{n-1} \, 0^n.
$$

Now examine the substring of length $n$ starting at position $2$. This substring consists of:

- the last $n-1$ symbols of $w$, which are $0^{n-1}$,
- followed by the first symbol of $0^n$, which is $0$.

Thus again the substring is

$$
0^{n-1}0 = 0^n.
$$

This is a codeword in $C$, and it occurs at position $2$, violating the commafree condition.

## Conclusion

In both cases, the assumption that $C$ contains exactly one word from each cycle class forces an internal occurrence of the codeword $0^n$ inside a concatenation of two codewords.

This contradicts the definition of a commafree code.

Therefore, no binary commafree code of length $n \ge 2$ can contain exactly one representative from each cycle class.

For $n=1$, the statement is trivial.

$$
\boxed{\text{No such binary commafree code exists for } n \ge 2.}
$$
