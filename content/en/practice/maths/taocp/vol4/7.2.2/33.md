**Exercise 33.** [*21*] *[21]* Puzzlers who tried to construct word squares by hand found long ago that it was easier to work from bottom to top. They often compiled "reverse dictionaries," whose words appear in colex order. Does this idea speed up computer experiments?

## Solution

Let a word square be an $n \times n$ array $(w_{ij})$ such that each row $w_{i1}\ldots w_{in}$ and each column $w_{1j}\ldots w_{nj}$ is a word from the given dictionary.

In Algorithm B style backtracking, a partial solution at level $l$ consists of the first $l$ rows, and property $P_l$ requires that every column prefix formed by these $l$ rows is consistent with at least one dictionary word prefix.

The proposed idea replaces top-to-bottom construction with bottom-to-top construction, and replaces a forward dictionary with a reverse dictionary ordered in colexicographic order, that is, ordering words primarily by their last letter, then the preceding letter, and so on.

Let each word $w = a_1 a_2 \ldots a_n$ be mapped to its reversal $w^R = a_n \ldots a_2 a_1$. A word square $(w_{ij})$ satisfies the column condition if and only if the reversed array $(w^R_{ij})$ satisfies the row condition, since column equality

$w_{1j} w_{2j} \ldots w_{nj} \in D$

is equivalent to row equality in the reversed representation.

Thus a bottom-to-top construction in the original square corresponds exactly to a top-to-bottom construction in the reversed square, with rows replaced by reversed words.

In a standard implementation, the dominant cost in step B3 is filtering the set of dictionary words compatible with the current partial column constraints. When constructing from the top, these constraints are prefix constraints on the words in the columns. When constructing from the bottom, these become suffix constraints on the original words, which are prefix constraints on reversed words.

If the dictionary is stored in colex order, then words sharing long suffixes in the original representation appear in contiguous blocks. After reversal, this becomes the usual lexicographic ordering by prefixes. Hence the candidate set for a partial column can be maintained by restricting attention to a contiguous interval in the reversed dictionary, exactly as in standard prefix-trie pruning.

At each level $l$, the set of admissible choices is therefore obtained by intersecting $n$ prefix-constraint sets, one for each column position. With reversed words, each such constraint is implemented as a prefix filter, and the intersection is computed over contiguous dictionary intervals determined by these prefixes.

This changes the computational structure of step B3. In the forward construction, each extension typically requires repeated access to arbitrary positions within words to test column compatibility. In the reversed-colex representation, compatibility testing reduces to prefix-range restriction in a preordered list, so the work per candidate becomes proportional to locating and intersecting contiguous blocks rather than scanning individual words.

The backtrack tree defined by $P_l$ is unchanged, since reversal is a bijection on words and preserves feasibility of every partial assignment. Hence the number of nodes visited by Algorithm B is invariant under this transformation.

The improvement therefore lies entirely in the cost of computing and updating the sets analogous to $S_l$. In the reversed-colex representation these sets are maintained by interval refinement on ordered lists, which reduces per-node overhead in the same way that lexicographic ordering improves prefix dictionary lookup in standard word-square construction.

Thus using reverse dictionaries and building bottom-to-top does not reduce the search tree size, but it improves the efficiency of each step of the backtracking process by aligning constraints with contiguous structure in memory.

This completes the proof. ∎
