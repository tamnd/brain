**Exercise 92.** [*46*] [46] Can a CI-net sort $n$ Boolean inputs with fewer modules than a "pure" sorting network that has no inverters?

**Verified:** no  
**Solve time:** 4m22s  

---

The failure of the previous solution is rooted in an attempt to treat “negation” as a local annotation on wires. The correct way to handle inverters in comparator-inverter networks is not via polarity labels, but via a structural simulation that preserves comparator usage exactly while eliminating NOT gates by converting them into harmless wire permutations.

## Correct solution

We prove that allowing NOT gates does not reduce the number of comparator modules needed to sort $n$ Boolean inputs.

Let a CI-net be a circuit built from:

- comparator modules computing $(\min(a,b), \max(a,b))$,
- NOT gates,
- wires.

We show that any CI-net sorting network with $C$ comparators can be transformed into a pure sorting network with exactly $C$ comparators.

## Step 1: Dual-rail representation

Replace every wire value $x \in \{0,1\}$ by a pair

$$
(x,\bar x).
$$

This pair encodes the value redundantly. We maintain the invariant that every signal in the network is represented in this dual form.

Thus each wire in the CI-net is replaced by two wires carrying complementary values.

## Step 2: Lifting gates to dual-rail form

### NOT gate

A NOT gate maps $x \mapsto \bar x$. In dual-rail form:

$$
(x,\bar x) \mapsto (\bar x, x).
$$

So a NOT gate is exactly a swap of the two rails. No comparator modules are used.

### Comparator gate

A comparator takes $(a,b)$ to $(\min(a,b), \max(a,b))$.

Lift it componentwise to dual-rail inputs:

$$
(a,\bar a), (b,\bar b).
$$

Apply the comparator to both rails:

- first rail:

$$
(\min(a,b), \max(a,b)),
$$

- second rail:

$$
(\min(\bar a,\bar b), \max(\bar a,\bar b)).
$$

Using Boolean duality,

$$
\min(\bar a,\bar b) = \overline{\max(a,b)}, \quad
\max(\bar a,\bar b) = \overline{\min(a,b)},
$$

so the output is

$$
(\min(a,b), \overline{\max(a,b)};\ \max(a,b), \overline{\min(a,b)}),
$$

which preserves the invariant that the second component is always the complement of the first.

Thus each comparator in the original CI-net becomes exactly one comparator acting simultaneously on both rails. No increase in comparator count occurs.

## Step 3: Eliminating NOT gates

In the dual-rail system:

- every NOT gate is a swap of the two rails,
- swaps are wire operations, not comparator modules.

We now eliminate all swaps:

Any swap can be pushed forward through the circuit using the following commutation rule:

- swapping rails before a comparator is equivalent to swapping rails after it (since both rails are processed symmetrically).

Thus every swap can be moved to the outputs without interacting with comparator structure.

This transformation preserves:

- the number of comparators,
- the computed dual-rail values.

## Step 4: Projection back to a single rail

After all swaps are pushed to the output side, each output wire has the form

$$
(y_i, \bar y_i).
$$

We now discard the second component and keep only $y_i$.

Because the CI-net sorts correctly, the first components satisfy

$$
y_1 \le y_2 \le \cdots \le y_n,
$$

so the output is already a correctly sorted Boolean sequence.

This yields a pure comparator network (no NOT gates) with exactly the same comparator modules.

## Step 5: Why comparator count is unchanged

Throughout the transformation:

- each comparator in the CI-net corresponds to exactly one comparator in the new network,
- NOT gates become wire swaps,
- swaps are absorbed by wiring and do not contribute to module count.

Hence the number of comparator modules is preserved exactly.

## Conclusion

Every CI-net sorting network with $C$ comparator modules and arbitrary NOT gates can be converted into a pure sorting network with exactly $C$ comparator modules.

Therefore, allowing inverters does not reduce the number of comparator modules required for sorting:

$$
\boxed{\text{CI-nets cannot sort with fewer comparator modules than pure sorting networks.}}
$$
