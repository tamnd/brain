## Solution

The required order is obtained by traversing the same mixed-radix number system as Algorithm M but replacing increment-with-carry by decrement-with-borrow.

Let $(a_1,\dots,a_n)$ represent a mixed-radix number with radices $m_1,\dots,m_n$ as in condition (1). The smallest tuple is $(0,\dots,0)$ and the largest is $(m_1-1,\dots,m_n-1)$. The task is to generate all tuples in strictly decreasing lexicographic (mixed-radix) order, starting from the largest.

Define the modified algorithm.

Algorithm $M'$ (Reverse mixed-radix generation).

$M'1.$ [Initialize.] Set $a_j \leftarrow m_j - 1$ for $1 \le j \le n$.

$M'2.$ [Visit.] Visit the $n$-tuple $(a_1,\dots,a_n)$.

$M'3.$ [Prepare to subtract one.] Set $j \leftarrow n$.

$M'4.$ [Borrow if necessary.] If $a_j = 0$, set $a_j \leftarrow m_j - 1$, set $j \leftarrow j - 1$, and repeat this step.

$M'5.$ [Decrease, unless done.] If $j = 0$, terminate the algorithm. Otherwise set $a_j \leftarrow a_j - 1$ and go back to $M'2$.

Correctness follows from interpreting $(a_1,\dots,a_n)$ as a mixed-radix integer

$$
\sum_{k=1}^n a_k \prod_{t=k+1}^n m_t,
$$

so each execution of $M'4$ implements borrowing in base $m_j$ exactly as in ordinary subtraction. Step $M'5$ performs a single unit decrement in the first position from the right where a borrow is not required. Each iteration therefore produces the predecessor of the current tuple in lexicographic order with respect to the usual ordering $0 < 1 < \cdots < m_j-1$, and every tuple is reached exactly once until the state $(0,\dots,0)$ is produced and the next borrow propagates past position $1$, triggering termination in $M'5$.
