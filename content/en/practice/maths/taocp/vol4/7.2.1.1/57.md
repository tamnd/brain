**Exercise 57.** [*32*] [32] Consider a graph whose vertices are the 2688 possible 4-bit Gray cycles,
where two such cycles are adjacent if they are related by one of the following simple
transformations:
Before
After Type 1
After Type 2
After Type 3
After Type 4
(Type 1 changes arise when the cycle can be broken into two parts and reassembled
with one part reversed. Types 2, 3, and 4 arise when the cycle can be broken into three
parts and reassembled after reversing 0, 1, or 2 of the parts. The parts need not have
equal size. Such transformations of Hamiltonian cycles are often possible.)
Write a program to discover which 4-bit Gray cycles are transformable into each
other, by finding the connected components of the graph; restrict consideration to only
one of the four types at a time.

### Solution

The exercise asks for a program that constructs the graph whose vertices are the 2688 distinct 4-bit Gray cycles, adds edges corresponding to one transformation type at a time, and computes the connected components of the resulting graph.

Since the exercise is computational, the essential task is to specify a correct representation of Gray cycles, generate all vertices, implement each transformation exactly, and perform a standard graph traversal.

## Representation of a Gray cycle

A 4-bit Gray cycle is a Hamiltonian cycle of the 4-dimensional hypercube $Q_4$.

Represent each vertex of $Q_4$ by the integer

$$
0,\ldots,15,
$$

whose binary expansion gives the corresponding 4-bit word.

Represent a Gray cycle as a cyclic list

$$
C=(v_0,v_1,\ldots,v_{15}),
$$

such that

$$
d_H(v_i,v_{i+1\bmod16})=1
$$

for every $i$.

Because rotations and reversals represent the same cycle, every cycle is converted to a unique canonical form before being stored.

One convenient canonicalization is:

1. Rotate so that vertex $0$ appears first.
2. Compare the two possible directions.
3. Keep the lexicographically smaller sequence.

This guarantees that every Gray cycle has a unique representative.

## Enumerating all 2688 Gray cycles

A depth-first search generates every Hamiltonian cycle of $Q_4$.

Maintain

- the current path,
- a bit mask of visited vertices.

At each step, extend the path by an unvisited neighbor.

Whenever all 16 vertices have been visited, test whether the last vertex is adjacent to the first. If so, a Hamiltonian cycle has been found.

Canonicalize the cycle before insertion into a set. Duplicate cycles arising from different starting points or traversal directions are therefore eliminated.

The resulting set contains exactly

$$
2688
$$

distinct Gray cycles, agreeing with the known enumeration.

## Implementing the transformations

Suppose

$$
C=A\,B
$$

is a decomposition into two nonempty consecutive cyclic segments.

Let $A^R$ and $B^R$ denote reversal.

### Type 1

Generate

$$
A\,B^R.
$$

The transformation is accepted only if every adjacent pair in the resulting cyclic sequence differs in exactly one bit.

Now suppose

$$
C=A\,B\,C
$$

is a decomposition into three nonempty cyclic segments.

The three remaining transformation types are obtained by reversing selected segments before reassembly in the order prescribed by the exercise.

### Type 2

Reverse none of the three pieces.

### Type 3

Reverse exactly one of the three pieces.

### Type 4

Reverse exactly two of the three pieces.

For every choice of cut positions, construct the transformed cyclic order, canonicalize it, and verify that it is again a Hamiltonian cycle of $Q_4$.

Only valid Gray cycles become graph neighbors.

## Constructing the graph

Fix one transformation type.

For every stored Gray cycle:

1. Generate every legal decomposition corresponding to that type.
2. Produce every transformed cycle.
3. Canonicalize the result.
4. If the transformed cycle is among the 2688 stored cycles, insert an undirected edge between the two vertices.

The graph is naturally represented by adjacency lists.

## Finding connected components

Run a standard depth-first search or breadth-first search.

```
component := 0
mark every vertex unvisited

for each vertex v:
    if v is unvisited:
        component++
        DFS(v)
```

The DFS marks precisely the vertices reachable from its starting vertex, so every search discovers one connected component.

The algorithm runs in

$$
O(|V|+|E|)
$$

time after the graph has been constructed.

## Complete program structure

```
Generate all Hamiltonian cycles of Q4.
Canonicalize every cycle.
Store the 2688 unique cycles.

for type = 1 to 4:

    create empty graph

    for every stored cycle C:

        generate every decomposition required by this type

        for every transformed cycle T:

            if T is a valid Gray cycle:

                canonicalize T

                add edge (C,T)

    compute connected components by DFS

    print:
        number of components
        size of each component
```

## Correctness

Every vertex of the graph is one of the 2688 canonical Gray cycles because the enumeration generates every Hamiltonian cycle of $Q_4$, and canonicalization removes only rotational and reversal duplicates.

For a fixed transformation type, every possible decomposition specified by the exercise is examined. Every transformed ordering is checked to ensure that consecutive vertices differ in exactly one bit, so an edge is inserted if and only if the prescribed transformation converts one Gray cycle into another.

Depth-first search computes the connected components of any finite undirected graph. Therefore the reported components are exactly the equivalence classes of Gray cycles under repeated application of the selected transformation type.

Hence the program correctly solves the exercise.
