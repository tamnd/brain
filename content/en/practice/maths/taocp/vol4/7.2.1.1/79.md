**Exercise 79.** &#9654; [*M22*] [M22] When is the last n-tuple of the (a) reflected (b) modular mixed-radix Gray
code adjacent to the first?

**Verified:** no  
**Solve time:** 4m34s

Edit

Let the radices be ((m_1,m_2,\ldots,m_n)), and write a tuple as

[
(a_1,a_2,\ldots,a_n),\qquad 0\le a_i<m_i .
]

Two tuples are adjacent when they differ in exactly one coordinate, and the two values in that coordinate are consecutive for that radix. For the modular code, the values (0) and (m_i-1) are also considered consecutive.

### (a) Reflected mixed-radix Gray code

Use the recursive definition of the reflected mixed-radix Gray code. For radices ((m_1,m_2,\ldots,m_n)), the code is formed by listing the code for the tail radices ((m_2,\ldots,m_n)) in blocks. The first coordinate has the values

[
0,1,\ldots,m_1-1,
]

and consecutive blocks reverse the order of the remaining coordinates.

Let (L(m_1,\ldots,m_n)) denote the last tuple in the code. The first tuple is always

[
(0,0,\ldots,0).
]

We first determine (L(m_1,\ldots,m_n)).

If (m_1) is even, the final block is the reversed tail code. Therefore the last element of that block is the first element of the tail code. Hence

[
L(m_1,\ldots,m_n)=(m_1-1,0,\ldots,0).
]

The first and last tuples differ only in the first coordinate exactly when all other coordinates agree, which they do here. The two first-coordinate values are (0) and (m_1-1). These are adjacent in the ordinary sense only when

[
m_1-1=1,
]

so

[
m_1=2.
]

Now suppose (m_1) is odd. The final block is not reversed. The last tuple therefore has first coordinate (m_1-1), and the remaining coordinates are the last tuple of the tail code. Since the first coordinate is neither (0) nor adjacent to (0) unless (m_1=2), and (m_1) is odd, adjacency of the full tuples cannot be decided by the first coordinate. The first coordinates must be equal for adjacency, which is impossible unless

[
m_1=1.
]

Therefore, when (m_1) is odd and greater than (1), the full code is not cyclic. When (m_1=1), the first coordinate is fixed and the question reduces exactly to the reflected code on the tail radices.

Applying this reduction repeatedly, let (j) be the first index for which (m_j>1). If no such index exists, every radix is (1), and the code consists of a single tuple, so it is trivially cyclic.

The first (j-1) coordinates are fixed at (0), and the question reduces to the (j)-th coordinate. Thus the last tuple has the form

[
(0,\ldots,0,m_j-1,0,\ldots,0),
]

where the nonzero entry is in position (j), provided all earlier radices are (1). For the last tuple to be adjacent to the first tuple, the (j)-th coordinate must change from (0) to (m_j-1) by one ordinary step. This happens exactly when

[
m_j-1=1,
]

so

[
m_j=2.
]

Therefore the reflected mixed-radix Gray code is cyclic exactly when the first radix that is greater than (1) is (2). Equivalently,

[
\boxed{\text{The reflected code is cyclic if and only if }
m_1=m_2=\cdots=m_{j-1}=1,\quad m_j=2
}
]

for some (j). The case in which all radices are (1) is the trivial one-tuple case.

### (b) Modular mixed-radix Gray code

In the modular mixed-radix Gray code, the successor operation changes one coordinate by (+1) modulo its radix, with the chosen coordinate determined by the construction. The code has one tuple for every possible state, so after all

[
M=m_1m_2\cdots m_n
]

tuples have been listed, the next successor operation returns to the initial tuple.

The first tuple is

[
(0,0,\ldots,0).
]

The last tuple is the state immediately before this return step. The final successor operation changes one coordinate, say coordinate (j), from

[
m_j-1
]

to

[
0
]

modulo (m_j). Every other coordinate is already the same as in the initial tuple, because the successor operation returns the whole state to the beginning after one complete traversal.

Therefore the last tuple must be

[
(0,\ldots,0,m_j-1,0,\ldots,0)
]

for some coordinate (j). In the modular adjacency relation, (m_j-1) and (0) are consecutive values, so this last tuple is adjacent to the first tuple.

Hence the modular mixed-radix Gray code always closes into a cycle:

[
\boxed{\text{The modular mixed-radix Gray code is cyclic for all radices.}}
]

The distinction is that the reflected code uses ordinary adjacency, so the first and last values of a coordinate are not automatically neighbors. The modular code identifies those endpoint values as adjacent, which guarantees the closing edge.
