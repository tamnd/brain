**Exercise 15.** &#9654; [*25*] [25] Design a loopless algorithm to generate the strings of the previous exercise.
All strings of the same length should be visited in lexicographic order as before, but
strings of different lengths can be intermixed in any convenient way.

## Solution

Let $T$ be the rooted tree whose nodes are all strings $a_1\ldots a_j$ with $0 \le j \le n$ and $0 \le a_i < m_i$, where the root is $\epsilon$. Each node of length $j<n$ has ordered children

$$
a_1\ldots a_j 0,\ a_1\ldots a_j 1,\ \ldots,\ a_1\ldots a_j (m_{j+1}-1),
$$

ordered lexicographically by the last symbol. The required condition “strings of the same length are visited in lexicographic order” is exactly the statement that, for fixed $j$, children are scanned in increasing $a_{j+1}$.

A traversal that preserves lexicographic order within each level is obtained by visiting each node before its descendants, and ensuring that children are generated in increasing order. The remaining freedom in the exercise allows arbitrary interleaving of different depths, so no constraint is placed on whether depth $j$ nodes are visited before or after depth $j+1$ nodes outside a fixed branch.

Define a current node as a pair $(a_1,\ldots,a_j)$ with $0 \le j \le n$, maintained in an array $a_1,\ldots,a_n$ together with a length variable $j$. The invariant is that the current node is always exactly $(a_1,\ldots,a_j)$, and positions $a_{j+1},\ldots,a_n$ are irrelevant.

The successor operation is defined by three mutually exclusive cases.

At a node of length $j<n$, if $a_{j+1}$ is not yet defined, it is interpreted as $0$ when descending; hence a first descent is always possible. If $j<n$, the algorithm may always move to the child

$$
(a_1,\ldots,a_j,0)
$$

by setting $a_{j+1} \leftarrow 0$ and $j \leftarrow j+1$. This introduces a new node exactly one level deeper.

If no descent is taken, the algorithm attempts to increase the last coordinate. When $j>0$ and $a_j < m_j-1$, the next sibling in lexicographic order within level $j$ is obtained by

$$
a_j \leftarrow a_j + 1,
$$

with $j$ unchanged.

If $j>0$ and $a_j = m_j-1$, no further node exists in the current level, so the algorithm moves upward by one level,

$$
j \leftarrow j-1,
$$

and the state becomes $(a_1,\ldots,a_{j-1})$. The next iteration will then either extend this prefix or increment at level $j-1$, depending on feasibility.

The initial state is $\epsilon$ with $j=0$. The output operation is “visit $(a_1,\ldots,a_j)$”.

The ordering condition for fixed length follows directly from the increment rule at level $j$, since successive values of $a_j$ are generated by repeated application of $a_j \leftarrow a_j+1$ before any backtracking occurs to level $j-1$. The lexicographic order within level $j$ is therefore preserved.

The absence of loops follows from the structure of the transition rules: each step performs exactly one of the operations “extend by one level”, “increment last coordinate”, or “decrease length by one”. No operation depends on repeated iteration over levels; repeated upward movement occurs across successive applications of the transition rule rather than within a single step.

This produces a valid traversal of all nodes in $T$ because every node is reached from its parent either by extension or by repeated backtracking until an ancestor with an available increment is found, and every increment exhausts all remaining siblings before backtracking continues. Hence every string $a_1\ldots a_j$ is eventually visited exactly when its prefix is reached and its last coordinate is generated.

This completes the construction of a loopless generation scheme satisfying the stated constraints. ∎
