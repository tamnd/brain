**Exercise 14.** [*20*] [20] Design an algorithm to generate all strings a1 . . . aj such that 0 ≤j ≤n and
0 ≤ai < mi for 1 ≤i ≤j, in lexicographic order. For example, if m1 = m2 = n = 2,
your algorithm should successively visit ϵ, 0, 00, 01, 1, 10, 11.

## Solution

Let $\Sigma_j = {0,1,\dots,m_j-1}$ for $1 \le j \le n$. The required objects are all strings $(a_1,\dots,a_j)$ with $0 \le j \le n$ and $a_i \in \Sigma_i$ for $1 \le i \le j$. The lexicographic order is the usual dictionary order: $(a_1,\dots,a_j)$ precedes $(b_1,\dots,b_k)$ when for the least index $t$ with $a_t \ne b_t$ one has $a_t < b_t$, or when no such $t$ exists and $j < k$.

Each valid string corresponds to a node in a rooted ordered tree in which the root is $\varepsilon$, every node at depth $j$ is labeled by $(a_1,\dots,a_j)$, and every node $(a_1,\dots,a_j)$ with $j < n$ has children $(a_1,\dots,a_j,v)$ for $v \in \Sigma_{j+1}$ in increasing order of $v$. The lexicographic order on strings is exactly the preorder traversal order of this tree.

Algorithm $L$ performs this traversal.

Algorithm $L$.

L1. [Visit.] Visit the current string $(a_1,\dots,a_j)$.

L2. [Extend.] If $j = n$, return. Otherwise set $v \leftarrow 0$.

L3. [Next child.] Form the string $(a_1,\dots,a_j,v)$ and apply Algorithm $L$ to it.

L4. Increase $v \leftarrow v+1$. If $v < m_{j+1}$, return to L3. Otherwise return.

The initial call is $L(\varepsilon)$ with $j=0$.

Correctness follows from the structure of the ordered tree. Every valid string is produced exactly once because each string is reached by a unique sequence of choices $a_1 \in \Sigma_1, \dots, a_j \in \Sigma_j$, and the recursion enumerates all such choices without repetition. The termination condition $j=n$ prevents extension beyond valid length, so no invalid string is generated.

Lexicographic order follows from the child ordering. Suppose two distinct strings $x=(a_1,\dots,a_j)$ and $y=(b_1,\dots,b_k)$. Let $t$ be the first index with $a_t \ne b_t$. The recursion explores all descendants of $(a_1,\dots,a_{t-1},a_t)$ completely before moving to $(a_1,\dots,a_{t-1},a_t+1)$, since all children of a node are exhausted in increasing order before the next sibling is taken. Hence all strings beginning with prefix $(a_1,\dots,a_{t-1},a_t)$ appear before those beginning with $(a_1,\dots,a_{t-1},a_t+1)$, which matches lexicographic ordering. If one string is a prefix of the other, the algorithm visits the shorter string at step L1 before any extension occurs, so the shorter precedes the longer.

The example $m_1 = m_2 = n = 2$ yields

$$
\varepsilon,\ 0,\ 00,\ 01,\ 1,\ 10,\ 11,
$$

since the recursion first expands $0$-subtree completely, then $1$-subtree, and within each level the digits are taken in increasing order.

The algorithm therefore generates exactly all required strings in lexicographic order.

$$
\boxed{\text{Algorithm } L \text{ correctly generates all required strings in lexicographic order}}
$$
