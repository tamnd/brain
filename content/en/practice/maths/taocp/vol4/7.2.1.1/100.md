**Exercise 100.** [*40*] [40] Do the de Bruijn cycles of exercise 97 provide a useful source of pseudo-
random bits when n is large?

**Verified:** yes

## Setup

Let $(f_n(k))_{k \ge 0}$ be the binary sequence obtained from a de Bruijn cycle of order $n$ as constructed in Exercise 97, interpreted cyclically with period $2^n$, so that every binary $n$-tuple occurs exactly once as a consecutive block in one period.

The question asks whether these cycles provide a useful source of pseudo-random bits when $n$ is large.

A useful pseudo-random generator is expected to produce sequences that are computationally unpredictable from short prefixes and that exhibit statistical behavior close to independent fair coin tosses on finite windows.

## Solution

The sequence $(f_n(k))$ is completely determined by a finite state consisting of the last $n-1$ bits together with the position in the cycle. More precisely, the de Bruijn property implies that each length-$n$ block occurs exactly once in a full period of length $2^n$. Hence each $(n-1)$-tuple occurs as a prefix of exactly two $n$-tuples, one ending in $0$ and one ending in $1$, but only one of these is realized at each position of the cycle. Therefore, once a window of $n$ consecutive bits is known, the next bit is uniquely determined by the requirement that the resulting $(n+1)$-tuple must correspond to the unique continuation consistent with the cycle ordering.

This implies that the process is a deterministic finite-state automaton with exactly $2^n$ states, each state corresponding to one of the $n$-bit substrings in the cycle. The transition function is a permutation of these states, since the cycle is Hamiltonian on the de Bruijn graph.

Consider predictability. If an observer knows $n$ consecutive output bits, then the current state in the cycle is uniquely identified. From that state, the next bit is uniquely determined. Hence the sequence has zero unpredictability in the information-theoretic sense once the window of length $n$ is known. A sequence with this property cannot model independent random bits, since in a truly random process the conditional distribution of the next bit given the previous $n$ bits is still uniform on ${0,1}$.

Consider statistical independence. For a sequence of independent fair bits, every binary word of length $m$ occurs with frequency $2^{-m}$ in the limit. In the de Bruijn cycle, every word of length $n$ occurs exactly once per period, but words of length $m < n$ do not occur with independent frequencies. Their occurrences are constrained by the global Hamiltonian structure of the underlying de Bruijn graph, and the overlaps between consecutive $n$-windows force long-range correlations. In particular, knowing one occurrence of an $m$-tuple restricts the location of all other occurrences, since it lies inside a unique $n$-tuple state.

Consider correlation structure. Let $X_k = f_n(k)$. Then $(X_k, X_{k+1}, \dots, X_{k+n-1})$ takes each binary $n$-vector exactly once per period. This implies strong constraints on joint distributions of overlapping blocks. For example, the event that a given pattern appears at position $k$ determines the next $n-1$ bits up to the cycle structure, so overlapping samples are maximally dependent rather than independent.

Consider finite-state complexity. The sequence is generated by a deterministic cycle on $2^n$ states, hence its linear complexity and state complexity are at most exponential in $n$ but completely deterministic. A pseudo-random source intended for simulation or cryptographic use typically requires that no short description or small state machine can reproduce the sequence. Here the entire sequence is produced by a fixed algorithm with $n$ bits of state, so the structure is maximally compressible.

The only apparent random-like feature is uniform coverage of all $n$-bit blocks. However, this property alone does not imply randomness for longer windows or independence. In fact, it is compatible with extreme regularity, since the sequence is a single Eulerian traversal of a finite graph.

Therefore, while the cycle has optimal coverage properties for length-$n$ substrings, it fails the independence and unpredictability requirements that characterize useful pseudo-random bit sources.

## Verification

The key implication used is that each state corresponds to an $n$-bit block, and the de Bruijn property guarantees a unique successor for each state in the fixed cycle, so the next bit is a deterministic function of the current $n$-state. This follows directly from the definition of the cycle as a Hamiltonian cycle in the de Bruijn graph of order $n$.

The claim that every $n$-tuple appears exactly once ensures that the state space has size $2^n$ and that the process is a permutation on these states, so no branching occurs in the evolution. This justifies the absence of conditional randomness.

The conclusion that statistical independence fails follows from the deterministic dependence of all future outputs on a finite state of size $n$, which contradicts the defining property of independent Bernoulli sequences.

## Conclusion

The de Bruijn cycles provide sequences with perfect uniform coverage of length-$n$ blocks, but they are deterministic finite-state permutations with complete predictability from any window of length $n$. This structural dependence prevents them from serving as a useful source of pseudo-random bits when $n$ is large. ∎
