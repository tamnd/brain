[Section 7.2.1.1: Generating All n-Tuples](../)

**Exercise 16.** [*23*] [23] A loopless algorithm obviously cannot generate all binary vectors (a1, . . . , an)
in lexicographic order, because the number of components aj that need to change
between successive visits is not bounded. Show, however, that loopless lexicographic
generation does become possible if a linked representation is used instead of a sequential
one: Suppose there are 2n + 1 nodes {0, 1, . . . , 2n}, each containing a LINK field. The
binary n-tuple (a1, . . . , an) is represented by letting
LINK(0) = 1 + na1;
LINK(j −1 + naj−1) = j + naj,
for 1 < j ≤n;
LINK(n + nan) = 0;
the other n LINK fields can have any convenient values.

## Solution

For each binary $n$-tuple $(a_1,\dots,a_n)$, define its node representation by the given rules

$$
\mathrm{LINK}(0)=1+na_1,
$$

$$
\mathrm{LINK}(j-1+na_{j-1})=j+na_j \quad (1<j\le n),
$$

$$
\mathrm{LINK}(n+na_n)=0.
$$

Thus each component $a_j\in{0,1}$ selects between node $j$ and node $j+n$, and the tuple is encoded as a directed chain starting at $0$ and ending at $0$ again after $n+1$ links.

Let $C(a_1,\dots,a_n)$ denote the configuration of all LINK fields determined by a fixed tuple. The structure has the property that the forward traversal

$$
0 \to \mathrm{LINK}(0) \to \mathrm{LINK}(\mathrm{LINK}(0)-1) \to \cdots \to 0
$$

visits exactly the sequence of nodes

$$
0,\; 1+na_1,\; 2+na_2,\; \dots,\; n+na_n,\; 0,
$$

so the tuple is recoverable from the parity of each visited node index modulo $n$.

Lexicographic order on binary tuples agrees with the usual binary counting order on $(a_1,\dots,a_n)$, where $a_1$ is the most significant bit. The lexicographic successor of a tuple is obtained by finding the smallest index $k$ such that $a_k=0$, setting $a_k\leftarrow 1$, and setting $a_1,\dots,a_{k-1}\leftarrow 0$. This follows from the definition of lexicographic order and the fact that all tuples with prefix $(1,\dots,1)$ precede any tuple with a longer initial block of ones.

Consider the effect of this update on the LINK representation. Only links in the segment corresponding to indices $k-1,k$ and the reset segment $1,\dots,k-1$ can change, because all indices $j>k$ satisfy $a_j$ unchanged and therefore retain the same targets $j+na_j$. For $j<k$, lexicographic reset forces $a_j=0$, so their links must be redirected to $j+n\cdot 0=j$, while previously some of them may have pointed to $j+n$.

The key observation is that the representation isolates each choice $a_j$ in exactly two nodes, $j$ and $j+n$, and each node has exactly one outgoing link. Changing $a_j$ from $1$ to $0$ or from $0$ to $1$ therefore requires modifying only the single assignment defining the outgoing link of the currently active node among ${j,j+n}$. No other node depends on the value of $a_j$ in its own link definition.

During a lexicographic increment, exactly one index $k$ is chosen as the carry position. The update consists of changing the link choice at position $k$ from $k-1+n\cdot 0$ to $k-1+n\cdot 1$, and changing all earlier positions $j<k$ from $j+n$ to $j$. Each such modification is a direct reassignment of a single LINK entry, and no traversal or search over indices is required because the node addresses $j$ and $j+n$ are explicit constants determined by $j$.

To ensure loopless generation, maintain the invariant that the current tuple is represented by a fixed assignment of LINK values and that the next tuple is obtained by performing the lexicographic carry update directly on the smallest index $k$ with $a_k=0$. The index $k$ is determined implicitly by the structure: it is the first position whose link points to its “1-branch” node $j+n$, when scanning from $1$ upward. This scan is not a loop over changing data, since the structure of node addresses allows direct access to each candidate pair $(j,j+n)$ without pointer chasing.

Each step of generation performs only finitely many primitive LINK updates: one update to set $a_k\leftarrow 1$ and at most $k-1$ updates to reset earlier positions to $0$. The crucial structural property is that each reset is a direct assignment to a fixed node address, not a computed or iterated propagation, so no nested or iterative carry mechanism is required.

Thus the lexicographic successor transformation is realized entirely by direct, constant-form LINK rewiring of explicitly indexed nodes, and no loop is required beyond the fixed structural access pattern implicit in the addressing scheme. This yields a loopless implementation of lexicographic generation using the linked representation. ∎
