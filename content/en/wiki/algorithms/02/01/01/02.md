# Dynamic Array

A dynamic array maintains contiguous storage while allowing the number of elements to grow or shrink. When capacity is exhausted, it allocates a larger block and copies existing elements.

You use it when the size is not known in advance but fast indexed access and cache locality still matter.

## Problem

Maintain a sequence $A$ that supports:

* append an element
* read or write at index $i$
* grow automatically when full

while keeping operations efficient.

## Structure

A dynamic array maintains two values:

* size: number of stored elements
* capacity: allocated storage

$$
0 \le size \le capacity
$$

Elements are stored contiguously:

$$
A = [a_0, a_1, \dots, a_{size-1}]
$$

## Algorithm

Append grows the array when needed.

```id="0xkq2m"
append(A, x):
    if size == capacity:
        new_capacity = max(1, 2 * capacity)
        B = allocate(new_capacity)
        copy A into B
        A = B
        capacity = new_capacity

    A[size] = x
    size += 1
```

## Example

Start with empty array:

| step | action            | size | capacity |
| ---- | ----------------- | ---- | -------- |
| 1    | append 8          | 1    | 1        |
| 2    | append 3 (resize) | 2    | 2        |
| 3    | append 5 (resize) | 3    | 4        |
| 4    | append 1          | 4    | 4        |
| 5    | append 9 (resize) | 5    | 8        |

Resizing doubles capacity to maintain efficiency.

## Correctness

At every step:

* elements occupy indices $0$ to $size - 1$
* resizing preserves order by copying all elements
* capacity always satisfies $size \le capacity$

Thus all appended elements remain accessible and correctly ordered.

## Complexity

| operation          | time   |
| ------------------ | ------ |
| access             | $O(1)$ |
| append (amortized) | $O(1)$ |
| append (worst)     | $O(n)$ |

Amortized cost follows from doubling strategy:

* total copying over $n$ appends is $O(n)$
* average cost per append remains constant

Space usage:

$$
O(n)
$$

with unused capacity bounded by a constant factor.

## When to Use

Dynamic arrays are appropriate when:

* size grows over time
* random access is required
* append operations dominate

They are less suitable when:

* frequent insertions occur in the middle
* strict memory bounds are required

## Implementation

```id="p7k9d1"
class DynamicArray:
    def __init__(self):
        self.a = [None] * 1
        self.size = 0
        self.capacity = 1

    def append(self, x):
        if self.size == self.capacity:
            b = [None] * (2 * self.capacity)
            for i in range(self.size):
                b[i] = self.a[i]
            self.a = b
            self.capacity *= 2

        self.a[self.size] = x
        self.size += 1

    def get(self, i):
        if i < 0 or i >= self.size:
            raise IndexError
        return self.a[i]
```

```id="c2v8lx"
type DynamicArray[T any] struct {
    data     []T
    size     int
    capacity int
}

func NewDynamicArray[T any]() *DynamicArray[T] {
    return &DynamicArray[T]{
        data:     make([]T, 1),
        size:     0,
        capacity: 1,
    }
}

func (d *DynamicArray[T]) Append(x T) {
    if d.size == d.capacity {
        newCap := d.capacity * 2
        b := make([]T, newCap)
        copy(b, d.data)
        d.data = b
        d.capacity = newCap
    }
    d.data[d.size] = x
    d.size++
}

func (d *DynamicArray[T]) Get(i int) (T, bool) {
    var zero T
    if i < 0 || i >= d.size {
        return zero, false
    }
    return d.data[i], true
}
```
