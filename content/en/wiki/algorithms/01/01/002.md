# Sentinel Linear Search

Sentinel linear search removes the explicit boundary check inside the loop by placing the target value at the end of the array as a sentinel. This reduces one comparison per iteration and improves constant factors while preserving correctness.

You use it when you want a tight inner loop and can safely modify the array temporarily.

## Problem

Given an array $A$ of length $n$ and a value $x$, find an index $i$ such that

$$
A[i] = x
$$

If no such index exists, return $-1$.

## Idea

Append the target value $x$ as a sentinel at the end of the array. Then scan without checking bounds. The loop is guaranteed to terminate because the sentinel matches.

After the loop, determine whether the match occurred within the original array or only at the sentinel.

## Algorithm

```id="k3v8s2"
sentinel_linear_search(A, x):
    n = length(A)
    last = A[n - 1]
    A[n - 1] = x

    i = 0
    while A[i] != x:
        i = i + 1

    A[n - 1] = last

    if i < n - 1:
        return i
    if last == x:
        return n - 1
    return -1
```

## Example

Let

$$
A = [8, 3, 5, 1, 9]
$$

and

$$
x = 7
$$

We overwrite the last element with $7$:

$$
A = [8, 3, 5, 1, 7]
$$

Now the loop runs without boundary checks:

| step | index | value | match |
| ---- | ----- | ----- | ----- |
| 1    | 0     | 8     | no    |
| 2    | 1     | 3     | no    |
| 3    | 2     | 5     | no    |
| 4    | 3     | 1     | no    |
| 5    | 4     | 7     | yes   |

We restore the last element:

$$
A = [8, 3, 5, 1, 9]
$$

Since the match occurred at the sentinel and the original last element was not $7$, the result is:

$$
-1
$$

## Correctness

The sentinel guarantees termination because the loop condition eventually becomes true. If the algorithm stops at index $i < n - 1$, then $A[i] = x$ holds in the original array. If it stops at $n - 1$, then either the original last element matched or only the sentinel matched. The final check distinguishes these cases.

## Complexity

| case  | comparisons | time   |
| ----- | ----------- | ------ |
| best  | $1$         | $O(1)$ |
| worst | $n$         | $O(n)$ |

Space usage remains constant:

$$
O(1)
$$

The advantage lies in removing the loop condition check $i < n$ at each iteration.

## When to Use

Sentinel linear search is useful when:

* inner loop performance matters
* the array can be modified temporarily
* you want to reduce branch overhead

It is common in low level or performance critical code.

## Implementation

```id="z81c0a"
def sentinel_linear_search(a, x):
    n = len(a)
    last = a[-1]
    a[-1] = x

    i = 0
    while a[i] != x:
        i += 1

    a[-1] = last

    if i < n - 1:
        return i
    if last == x:
        return n - 1
    return -1
```

```id="p2x7qm"
func SentinelLinearSearch(a []int, x int) int {
    n := len(a)
    last := a[n-1]
    a[n-1] = x

    i := 0
    for a[i] != x {
        i++
    }

    a[n-1] = last

    if i < n-1 {
        return i
    }
    if last == x {
        return n - 1
    }
    return -1
}
```
