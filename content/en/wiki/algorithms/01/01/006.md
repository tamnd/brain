# First Occurrence Search

First occurrence search finds the smallest index where a target value appears. In unsorted data, this is naturally obtained by scanning from left to right and stopping at the first match.

This operation is fundamental when duplicates exist and you care about stable positions or earliest matches.

## Problem

Given a sequence $A$ of length $n$ and a value $x$, find the smallest index $i$ such that

$$
A[i] = x
$$

If no such index exists, return $-1$.

## Algorithm

Scan from the beginning and stop at the first match.

```id="q9m2xp"
first_occurrence_search(A, x):
    for i from 0 to length(A) - 1:
        if A[i] == x:
            return i
    return -1
```

## Example

Let

$$
A = [5, 3, 7, 3, 9]
$$

and

$$
x = 3
$$

The scan proceeds:

| step | index | value | match |
| ---- | ----- | ----- | ----- |
| 1    | 0     | 5     | no    |
| 2    | 1     | 3     | yes   |

Return index $1$, which is the first occurrence.

## Correctness

The algorithm examines elements in increasing index order. The first time it encounters $A[i] = x$, all earlier elements have already been checked and are not equal to $x$. Therefore, $i$ is the smallest valid index.

If no match is found, then no occurrence exists in the sequence.

## Complexity

| case  | comparisons | time   |
| ----- | ----------- | ------ |
| best  | $1$         | $O(1)$ |
| worst | $n$         | $O(n)$ |

Space usage:

$$
O(1)
$$

## When to Use

Use first occurrence search when:

* duplicates exist and order matters
* you need stable or earliest matches
* working with unsorted data

In sorted arrays, this task is typically solved more efficiently using a modified binary search.

## Implementation

```id="m2x7zr"
def first_occurrence_search(a, x):
    for i, value in enumerate(a):
        if value == x:
            return i
    return -1
```

```id="p6k1wn"
func FirstOccurrenceSearch[T comparable](a []T, x T) int {
    for i, v := range a {
        if v == x {
            return i
        }
    }
    return -1
}
```
