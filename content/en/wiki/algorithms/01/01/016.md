# Boyer Moore Majority Vote

Boyer Moore majority vote finds a candidate that may appear more than half the time using a single pass and constant space. It relies on a cancellation principle: pairs of different elements cancel each other.

A second pass verifies whether the candidate truly satisfies the majority condition.

## Problem

Given a sequence $A$ of length $n$, find a value $x$ such that

$$
|{, i \mid A[i] = x ,}| > \frac{n}{2}
$$

If no such value exists, return `none`.

## Idea

Maintain a current candidate and a counter.

* when the counter is zero, choose the current element as the new candidate
* when the current element equals the candidate, increment the counter
* otherwise decrement the counter

Intuition: non matching elements cancel out matching ones. If a majority exists, it survives the cancellations.

## Algorithm

```id="m2k9xp"
boyer_moore_majority_vote(A):
    candidate = none
    count = 0

    for value in A:
        if count == 0:
            candidate = value
            count = 1
        else if value == candidate:
            count = count + 1
        else:
            count = count - 1

    # verification
    count = 0
    for value in A:
        if value == candidate:
            count = count + 1

    if count > length(A) / 2:
        return candidate
    return none
```

## Example

Let

$$
A = [3, 3, 4, 2, 3, 3, 3]
$$

First pass:

| step | value | candidate | count |
| ---- | ----- | --------- | ----- |
| 1    | 3     | 3         | 1     |
| 2    | 3     | 3         | 2     |
| 3    | 4     | 3         | 1     |
| 4    | 2     | 3         | 0     |
| 5    | 3     | 3         | 1     |
| 6    | 3     | 3         | 2     |
| 7    | 3     | 3         | 3     |

Candidate becomes $3$.

Second pass confirms frequency $5 > 7/2$.

## Correctness

Each decrement cancels a pair of different elements. If a majority element exists, its excess occurrences cannot be fully canceled. Therefore it remains as the final candidate. The second pass ensures correctness when no majority exists.

## Complexity

| phase          | time   |
| -------------- | ------ |
| candidate scan | $O(n)$ |
| verification   | $O(n)$ |

Total time:

$$
O(n)
$$

Space usage:

$$
O(1)
$$

## When to Use

Use this method when:

* you need a majority element in linear time
* memory must remain constant
* the dataset is large

It is optimal for the strict majority condition.

## Implementation

```id="v7x3qp"
def boyer_moore_majority_vote(a):
    candidate = None
    count = 0

    for value in a:
        if count == 0:
            candidate = value
            count = 1
        elif value == candidate:
            count += 1
        else:
            count -= 1

    if candidate is None:
        return None

    if a.count(candidate) > len(a) // 2:
        return candidate
    return None
```

```id="k2m8vn"
func BoyerMooreMajorityVote[T comparable](a []T) (T, bool) {
    var candidate T
    count := 0

    for _, v := range a {
        if count == 0 {
            candidate = v
            count = 1
        } else if v == candidate {
            count++
        } else {
            count--
        }
    }

    freq := 0
    for _, v := range a {
        if v == candidate {
            freq++
        }
    }

    if freq > len(a)/2 {
        return candidate, true
    }

    var zero T
    return zero, false
}
```
