# Linear Search

Linear search scans a sequence element by element until it finds a match with the target. It works on any data layout, including unsorted arrays, streams, or linked structures.

You use it when no ordering or indexing structure exists, or when the dataset is small enough that preprocessing would cost more than scanning.

## Problem

Given a sequence $A$ of length $n$ and a value $x$, find an index $i$ such that

$$
A[i] = x
$$

If no such index exists, return $-1$.

## Algorithm

Check each position in order. Stop immediately when a match appears.

```
linear_search(A, x):
    for i from 0 to length(A) - 1:
        if A[i] == x:
            return i
    return -1
```

## Example

Let

$$
A = [8, 3, 5, 1, 9]
$$

and

$$
x = 1
$$

The scan proceeds sequentially:

| step | index | value | match |
| ---- | ----- | ----- | ----- |
| 1    | 0     | 8     | no    |
| 2    | 1     | 3     | no    |
| 3    | 2     | 5     | no    |
| 4    | 3     | 1     | yes   |

Return index $3$.

## Correctness

The loop visits every index in increasing order. If it returns at index $i$, then $A[i] = x$ holds by direct comparison. If it finishes without returning, every element has been tested and none equals $x$, so the result is correct.

## Complexity

| case    | comparisons   | time   |
| ------- | ------------- | ------ |
| best    | $1$           | $O(1)$ |
| worst   | $n$           | $O(n)$ |
| average | $\approx n/2$ | $O(n)$ |

Space usage stays constant:

$$
O(1)
$$

## When to Use

Linear search is appropriate when:

* the data is unsorted
* the dataset is small
* the search runs once or a few times
* the cost of sorting or indexing is unjustified

For repeated queries on large data, prefer binary search on sorted arrays or hash based lookup.

## Implementation

```
def linear_search(a, x):
    for i, value in enumerate(a):
        if value == x:
            return i
    return -1
```

```
func LinearSearch[T comparable](a []T, x T) int {
    for i, v := range a {
        if v == x {
            return i
        }
    }
    return -1
}
```
