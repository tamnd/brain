# Min Max Search

Min max search finds both the minimum and maximum values in one traversal. Instead of running two separate scans, it maintains both extrema simultaneously.

This reduces constant overhead and serves as a base for more advanced selection algorithms.

## Problem

Given a sequence $A$ of length $n$, find indices $i$ and $j$ such that

$$
A[i] = \min_{0 \le k < n} A[k], \quad A[j] = \max_{0 \le k < n} A[k]
$$

## Algorithm

Track both minimum and maximum while scanning.

```id="k7p2xm"
min_max_search(A):
    if length(A) == 0:
        return (-1, -1)

    min_index = 0
    max_index = 0

    for i from 1 to length(A) - 1:
        if A[i] < A[min_index]:
            min_index = i
        if A[i] > A[max_index]:
            max_index = i

    return (min_index, max_index)
```

## Example

Let

$$
A = [7, 3, 5, 2, 9]
$$

The scan proceeds:

| step | index | value | min index | max index |
| ---- | ----- | ----- | --------- | --------- |
| 1    | 0     | 7     | 0         | 0         |
| 2    | 1     | 3     | 1         | 0         |
| 3    | 2     | 5     | 1         | 0         |
| 4    | 3     | 2     | 3         | 0         |
| 5    | 4     | 9     | 3         | 4         |

Return:

$$
(3, 4)
$$

## Correctness

At each step, the algorithm maintains two invariants:

* `min_index` points to the smallest element seen so far
* `max_index` points to the largest element seen so far

After processing all elements, both invariants imply global correctness.

## Complexity

| case      | comparisons | time   |
| --------- | ----------- | ------ |
| all cases | $2(n - 1)$  | $O(n)$ |

Space usage:

$$
O(1)
$$

## When to Use

Use min max search when:

* both extrema are required
* avoiding multiple passes matters
* as a building block for selection algorithms

Further optimization is possible using pairwise comparison, which reduces comparisons to about $1.5n$.

## Implementation

```id="v3k8qp"
def min_max_search(a):
    if not a:
        return -1, -1

    min_index = 0
    max_index = 0

    for i in range(1, len(a)):
        if a[i] < a[min_index]:
            min_index = i
        if a[i] > a[max_index]:
            max_index = i

    return min_index, max_index
```

```id="n2x7zr"
func MinMaxSearch[T constraints.Ordered](a []T) (int, int) {
    if len(a) == 0 {
        return -1, -1
    }

    minIndex := 0
    maxIndex := 0

    for i := 1; i < len(a); i++ {
        if a[i] < a[minIndex] {
            minIndex = i
        }
        if a[i] > a[maxIndex] {
            maxIndex = i
        }
    }

    return minIndex, maxIndex
}
```
