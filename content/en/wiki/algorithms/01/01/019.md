# Local Minimum Linear Search

Local minimum search finds an index where the element is smaller than its immediate neighbors. Unlike global minimum, it does not require scanning for the smallest value in the entire sequence.

A linear scan guarantees finding one local minimum.

## Problem

Given a sequence $A$ of length $n$, find an index $i$ such that

$$
A[i] \le A[i-1] \quad \text{and} \quad A[i] \le A[i+1]
$$

with boundary conditions:

* for $i = 0$: $A[0] \le A[1]$
* for $i = n - 1$: $A[n-1] \le A[n-2]$

Return any such index.

## Idea

Check each position and test whether it satisfies the local minimum condition relative to its neighbors.

## Algorithm

```id="x8k2qp"
local_minimum_linear_search(A):
    n = length(A)
    if n == 0:
        return -1

    if n == 1:
        return 0

    if A[0] <= A[1]:
        return 0

    for i from 1 to n - 2:
        if A[i] <= A[i - 1] and A[i] <= A[i + 1]:
            return i

    return n - 1
```

## Example

Let

$$
A = [9, 6, 3, 5, 7]
$$

The scan proceeds:

| index | value | condition       | local minimum |
| ----- | ----- | --------------- | ------------- |
| 0     | 9     | 9 ≤ 6           | no            |
| 1     | 6     | 6 ≤ 9 and 6 ≤ 3 | no            |
| 2     | 3     | 3 ≤ 6 and 3 ≤ 5 | yes           |

Return index $2$.

## Correctness

A local minimum must exist in any finite sequence. If the first element is not a local minimum, then the sequence must decrease at least once. At the point where it stops decreasing, a local minimum appears. The algorithm checks all candidates and therefore finds one.

## Complexity

| case  | comparisons | time   |
| ----- | ----------- | ------ |
| worst | $n$         | $O(n)$ |

Space usage:

$$
O(1)
$$

## When to Use

Use local minimum search when:

* a global minimum is not required
* working with unimodal or partially structured data
* as a step in optimization problems

More efficient methods exist using binary search when additional structure is known.
