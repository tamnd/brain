# Pairwise Min Max Search

Pairwise min max search improves over the basic min max scan by reducing the number of comparisons. It processes elements in pairs, comparing them with each other first, then updating global minimum and maximum.

This reduces comparisons from about $2n$ to about $1.5n$.

## Problem

Given a sequence $A$ of length $n$, find indices $i$ and $j$ such that

$$
A[i] = \min_{0 \le k < n} A[k], \quad A[j] = \max_{0 \le k < n} A[k]
$$

## Idea

Instead of comparing each element independently with both min and max, compare elements in pairs:

* compare the pair internally
* smaller goes to min comparison
* larger goes to max comparison

This saves one comparison per pair.

## Algorithm

```id="w8k2xm"
pairwise_min_max_search(A):
    if length(A) == 0:
        return (-1, -1)

    if length(A) is even:
        if A[0] < A[1]:
            min_index = 0
            max_index = 1
        else:
            min_index = 1
            max_index = 0
        i = 2
    else:
        min_index = 0
        max_index = 0
        i = 1

    while i < length(A) - 1:
        if A[i] < A[i + 1]:
            local_min = i
            local_max = i + 1
        else:
            local_min = i + 1
            local_max = i

        if A[local_min] < A[min_index]:
            min_index = local_min
        if A[local_max] > A[max_index]:
            max_index = local_max

        i = i + 2

    return (min_index, max_index)
```

## Example

Let

$$
A = [7, 3, 5, 2, 9, 1]
$$

Pairs:

| pair  | values | local min | local max |
| ----- | ------ | --------- | --------- |
| (0,1) | (7,3)  | 3         | 7         |
| (2,3) | (5,2)  | 2         | 5         |
| (4,5) | (9,1)  | 1         | 9         |

Final result:

$$
\text{min} = 1,\quad \text{max} = 9
$$

## Correctness

Each pair comparison determines relative order within the pair. Only the smaller element is compared with the global minimum, and only the larger element is compared with the global maximum. Since every element participates in exactly one pair comparison and one global comparison, correctness follows.

## Complexity

| case     | comparisons        | time   |
| -------- | ------------------ | ------ |
| even $n$ | $\frac{3n}{2} - 2$ | $O(n)$ |
| odd $n$  | $\frac{3(n-1)}{2}$ | $O(n)$ |

Space usage:

$$
O(1)
$$

This is optimal for finding both min and max.

## When to Use

Use pairwise min max search when:

* minimizing comparisons is important
* working in performance sensitive environments
* designing optimal selection primitives

## Implementation

```id="z1m8qp"
def pairwise_min_max_search(a):
    n = len(a)
    if n == 0:
        return -1, -1

    if n % 2 == 0:
        if a[0] < a[1]:
            min_i, max_i = 0, 1
        else:
            min_i, max_i = 1, 0
        i = 2
    else:
        min_i = max_i = 0
        i = 1

    while i < n - 1:
        if a[i] < a[i + 1]:
            local_min, local_max = i, i + 1
        else:
            local_min, local_max = i + 1, i

        if a[local_min] < a[min_i]:
            min_i = local_min
        if a[local_max] > a[max_i]:
            max_i = local_max

        i += 2

    return min_i, max_i
```

```id="q7x2vk"
func PairwiseMinMaxSearch[T constraints.Ordered](a []T) (int, int) {
    n := len(a)
    if n == 0 {
        return -1, -1
    }

    var minI, maxI, i int

    if n%2 == 0 {
        if a[0] < a[1] {
            minI, maxI = 0, 1
        } else {
            minI, maxI = 1, 0
        }
        i = 2
    } else {
        minI, maxI = 0, 0
        i = 1
    }

    for i < n-1 {
        var localMin, localMax int
        if a[i] < a[i+1] {
            localMin, localMax = i, i+1
        } else {
            localMin, localMax = i+1, i
        }

        if a[localMin] < a[minI] {
            minI = localMin
        }
        if a[localMax] > a[maxI] {
            maxI = localMax
        }

        i += 2
    }

    return minI, maxI
}
```
