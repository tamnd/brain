# Second Minimum Search

Second minimum search finds the element that is strictly larger than the minimum and as small as possible. It extends minimum search by tracking two candidates.

This is useful in ranking, selection, and tournament based algorithms.

## Problem

Given a sequence $A$ of length $n$, find an index $i$ such that

$$
A[i] = \min {, A[j] \mid A[j] > \min(A) ,}
$$

If no such element exists, return $-1$.

## Idea

Maintain two values:

* smallest element seen so far
* second smallest element seen so far

Update both as the scan progresses.

## Algorithm

```id="x7k2qp"
second_minimum_search(A):
    if length(A) < 2:
        return -1

    min_index = 0
    second_index = -1

    for i from 1 to length(A) - 1:
        if A[i] < A[min_index]:
            second_index = min_index
            min_index = i
        else if A[i] > A[min_index]:
            if second_index == -1 or A[i] < A[second_index]:
                second_index = i

    return second_index
```

## Example

Let

$$
A = [7, 3, 5, 2, 9]
$$

The scan proceeds:

| step | index | value | min | second min |
| ---- | ----- | ----- | --- | ---------- |
| 1    | 0     | 7     | 7   | —          |
| 2    | 1     | 3     | 3   | 7          |
| 3    | 2     | 5     | 3   | 5          |
| 4    | 3     | 2     | 2   | 3          |
| 5    | 4     | 9     | 2   | 3          |

Return index of value $3$.

## Correctness

At each step:

* `min_index` holds the smallest element seen
* `second_index` holds the smallest element greater than the minimum

Whenever a new minimum appears, the old minimum becomes a candidate for second minimum. When a value lies between the current minimum and second minimum, it updates the second.

After scanning all elements, the invariant ensures correctness.

## Complexity

| case      | comparisons | time   |
| --------- | ----------- | ------ |
| all cases | $n - 1$     | $O(n)$ |

Space usage:

$$
O(1)
$$

## When to Use

Use second minimum search when:

* selecting top two smallest elements
* ranking or filtering tasks
* building tournament based algorithms

For general k-th selection, use selection algorithms such as quickselect.

## Implementation

```id="m8q3xp"
def second_minimum_search(a):
    if len(a) < 2:
        return -1

    min_i = 0
    second_i = -1

    for i in range(1, len(a)):
        if a[i] < a[min_i]:
            second_i = min_i
            min_i = i
        elif a[i] > a[min_i]:
            if second_i == -1 or a[i] < a[second_i]:
                second_i = i

    return second_i
```

func SecondMinimumSearch[T constraints.Ordered](a []T) int {
    if len(a) < 2 {
        return -1
    }

    minI := 0
    secondI := -1

    for i := 1; i < len(a); i++ {
        if a[i] < a[minI] {
            secondI = minI
            minI = i
        } else if a[i] > a[minI] {
            if secondI == -1 || a[i] < a[secondI] {
                secondI = i
            }
        }
    }

    return secondI
}
