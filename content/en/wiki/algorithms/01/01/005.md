# Recursive Linear Search

Recursive linear search expresses the same logic as iterative linear search using recursion. At each step, the problem reduces to searching a smaller suffix of the array.

This form is mainly useful for reasoning, proofs, or when working in a recursive framework. In practice, iteration is usually preferred due to lower overhead.

## Problem

Given a sequence $A$ of length $n$ and a value $x$, find an index $i$ such that

$$
A[i] = x
$$

If no such index exists, return $-1$.

## Idea

At position $i$, either:

* $A[i] = x$, so return $i$
* or search the remaining subarray starting at $i + 1$

This leads to a natural recursive definition.

## Algorithm

```id="g5n8q1"
recursive_linear_search(A, x, i):
    if i == length(A):
        return -1

    if A[i] == x:
        return i

    return recursive_linear_search(A, x, i + 1)
```

Initial call:

```id="k2m4zp"
recursive_linear_search(A, x, 0)
```

## Example

Let

$$
A = [4, 7, 2, 9]
$$

and

$$
x = 2
$$

The recursion unfolds:

| call | index | value | result   |
| ---- | ----- | ----- | -------- |
| 1    | 0     | 4     | recurse  |
| 2    | 1     | 7     | recurse  |
| 3    | 2     | 2     | return 2 |

The final result propagates back through the call stack.

## Correctness

The base case handles the situation where all elements have been checked. Each recursive step either finds the target at position $i$ or reduces the problem to a smaller suffix. By induction on the remaining length, the algorithm is correct.

## Complexity

| case  | calls | time   |
| ----- | ----- | ------ |
| best  | $1$   | $O(1)$ |
| worst | $n$   | $O(n)$ |

Space usage depends on recursion depth:

$$
O(n)
$$

due to the call stack.

## When to Use

Recursive linear search is appropriate when:

* working in a recursive algorithm design
* teaching or proving correctness
* integrating with other recursive structures

For performance critical code, iterative versions are preferred.

## Implementation

```id="v1z8cp"
def recursive_linear_search(a, x, i=0):
    if i == len(a):
        return -1
    if a[i] == x:
        return i
    return recursive_linear_search(a, x, i + 1)
```

```id="n7k3qy"
func RecursiveLinearSearch[T comparable](a []T, x T, i int) int {
    if i == len(a) {
        return -1
    }
    if a[i] == x {
        return i
    }
    return RecursiveLinearSearch(a, x, i+1)
}
```
